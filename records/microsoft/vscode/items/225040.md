# Issue #225040 — vscode-oss build can't change icon or message

[https://github.com/microsoft/vscode/issues/225040](https://github.com/microsoft/vscode/issues/225040)

## Summary

Fixed on main: the root cause was rcedit 0.3.0 bundled in @vscode/gulp-electron, which was upgraded to rcedit ^4.0.1 in gulp-electron 1.37.0. VS Code bumped to that version in June 2025, so a fresh build from current main uses the modern rcedit that correctly modifies the exe icon and metadata on fresh Windows installs.

## Change summary

Building VS Code OSS from source on a fresh Windows 10 install fails to modify the executable icon and product name because the old rcedit 0.3.0 in @vscode/gulp-electron lacks required dependencies.

## Best solution

Keep the shipped dependency bump — @vscode/gulp-electron ≥1.37.0 with rcedit ^4.0.1 resolves the issue. No further change needed; building from current main should produce correctly branded executables on fresh Windows installs.
