# Issue #276999 — Dragging-n-dropping a file with tilde in the name into terminal

[https://github.com/microsoft/vscode/issues/276999](https://github.com/microsoft/vscode/issues/276999)

## Summary

Still reproduces on main: the escaping logic in terminalEnvironment.ts strips shell-special characters instead of quoting/escaping them. Multiple open PRs attempt to fix it but none have merged.

## Change summary

Dragging a file with shell-special characters (~ $ & # ! etc.) into the terminal silently drops those characters from the path.

## Best solution

Replace the bannedChars stripping regex in `escapeNonWindowsPath` (src/vs/platform/terminal/common/terminalEnvironment.ts ~line 71) with proper shell-aware quoting — wrapping the entire path in single quotes handles all listed characters safely. One of the open PRs (e.g. https://github.com/microsoft/vscode/pull/316392) should be reviewed and landed.
