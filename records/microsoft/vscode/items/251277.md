# Issue #251277 — Function of shortcut key of block comment in c/c++ not strong enough

[https://github.com/microsoft/vscode/issues/251277](https://github.com/microsoft/vscode/issues/251277)

## Summary

This is a feature request to enhance block-comment toggling so it also strips the interior `* ` decoration from JSDoc/Doxygen-style comments. Current behavior only removes `/*` and `*/` delimiters, which is by design for the basic toggle; the interior stripping would be new functionality.

## Change summary

Toggling off a block comment with Shift+Alt+A should also strip the leading ` * ` prefix from each interior line of a Doxygen/JSDoc-style comment.

## Best solution

Enhance `_createRemoveBlockCommentOperations` in `src/vs/editor/contrib/comment/browser/blockCommentCommand.ts` to detect and strip the common ` * ` line prefix from interior lines when removing a block comment. This is a non-trivial enhancement: it must avoid stripping content when the `*` is semantically meaningful (e.g. multiplication). An alternative is an extension or a separate command. The current `editor-autoindent` label seems mis-routed; `editor-comments` would be more fitting if it existed, but `editor` is appropriate.
