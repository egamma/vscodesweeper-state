# Issue #309881 — run_in_terminal cancellation can hang indefinitely in basic and none execute strategies under continuous terminal output

[https://github.com/microsoft/vscode/issues/309881](https://github.com/microsoft/vscode/issues/309881)

## Summary

Team-authored debt issue, still present on main. The `waitForIdle` helper in `executeStrategy.ts` has no cancellation path, confirmed by source inspection — both `NoneExecuteStrategy` and `BasicExecuteStrategy` still await uncancellable idle waits before checking the cancellation token.

## Change summary

Cancelling a `run_in_terminal` tool invocation can hang indefinitely in basic and none execute strategies when the terminal produces continuous output, because `waitForIdle` has no cancellation support.

## Best solution

Add a `CancellationToken` parameter to `waitForIdle` (and `waitForIdleWithPromptHeuristics`) so the debounce promise rejects on cancellation. Then thread the token through all call sites in `basicExecuteStrategy.ts` and `noneExecuteStrategy.ts`. This avoids the indefinite hang without changing the idle-detection semantics for the normal (non-cancelled) path.
