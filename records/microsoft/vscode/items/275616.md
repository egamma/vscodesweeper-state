# Issue #275616 — Keystrokes for GitHub Copilot "Continue to iterate" prompt Continue/Pause options.

[https://github.com/microsoft/vscode/issues/275616](https://github.com/microsoft/vscode/issues/275616)

## Summary

Still valid and on the 'On Deck' milestone. The generic ChatConfirmationContentPart (used for the 'Continue to iterate' prompt) has no keybinding wiring, unlike the tool-confirmation parts that already use IKeybindingService.appendKeybinding. A team member (roblourens) confirmed in December 2025 that keybindings were intended but not implemented.

## Change summary

Add keyboard shortcuts (e.g. Ctrl+Enter for Continue) to the 'Continue to iterate?' agent-mode confirmation prompt.

## Best solution

Wire IKeybindingService into ChatConfirmationContentPart / SimpleChatConfirmationWidget (src/vs/workbench/contrib/chat/browser/widget/chatContentParts/chatConfirmationContentPart.ts and chatConfirmationWidget.ts) following the same pattern used by AbstractToolConfirmationSubPart and ChatElicitationContentPart, which call keybindingService.appendKeybinding to show the shortcut on button tooltips. Register a keybinding (likely reusing the existing AcceptToolConfirmationActionId or a new action) so Ctrl+Enter accepts the primary button.
