# Issue #178994 — `terminal.goToRecentDirectory` is not shell-aware

[https://github.com/microsoft/vscode/issues/178994](https://github.com/microsoft/vscode/issues/178994)

## Summary

Valid feature request — the `goToRecentDirectory` command hardcodes `cd` (confirmed at terminalRunRecentQuickPick.ts:355) and has no mechanism for non-POSIX shells or REPLs to register a custom cd template. The feature remains unimplemented on main.

## Change summary

Allow REPL-like programs using shell integration to register a custom directory-change command template instead of the hardcoded `cd $dir`.

## Best solution

Implement a `cdTemplate` shell integration property (via OSC 633;P) that overrides the hardcoded `cd` in `src/vs/workbench/contrib/terminalContrib/history/browser/terminalRunRecentQuickPick.ts` line 355. This would benefit Julia, cmd.exe (`cd /d`), and other non-standard shells. The terminal maintainer (Tyriar) already endorsed the approach in comments.
