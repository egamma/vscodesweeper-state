# Issue #318189 — Chat: Implement icon-first design when pane is narrow in size

[https://github.com/microsoft/vscode/issues/318189](https://github.com/microsoft/vscode/issues/318189)

## Summary

Valid polish request. The chat input already has a compact/icon-only mode for narrow widths (< 280px) that works for the model picker and mode picker, but the permission picker (Default Approvals) does not implement icon-only rendering when compact — it always shows the text label. The infrastructure is in place; the gap is in permissionPickerActionItem.ts.

## Change summary

Chat input picker buttons (model, mode, approvals) truncate their text labels at narrow pane widths instead of collapsing to icon-only buttons.

## Best solution

In `src/vs/workbench/contrib/chat/browser/widget/input/permissionPickerActionItem.ts`, update `renderLabel` to check `this.pickerOptions.compact.get()` and skip the text `<span>` when compact is true and an icon is available — matching the pattern already used in `modePickerActionItem.ts` (line 312) and `chatModelPicker.ts` (line 1538). The responsive toolbar overflow already handles items that don't fit, but the remaining visible items should be icon-only at narrow widths for consistency.
