# Issue #307760 — Archiving sessions in slow

[https://github.com/microsoft/vscode/issues/307760](https://github.com/microsoft/vscode/issues/307760)

## Summary

Team-authored performance bug — still reproduces on main. The archive-all-sessions flow calls `archiveSession` in a serial loop, firing `_onDidArchiveSession` per session, which triggers a re-render after each one instead of batching. Keeping open for the author to address.

## Change summary

Archiving multiple sessions blocks the UI (~350ms) because each session archive fires an individual event and re-render instead of batching.

## Best solution

Add a batch `archiveSessions(sessions[])` method to `SessionsManagementService` that performs all provider calls, then fires a single bulk event (or a single re-render trigger). Update the action call sites in `sessionsViewActions.ts` (lines ~594, ~653, ~802) and `agentSessionsActions.ts` (line ~518) to use the batch API.
