# Issue #307760 — Archiving sessions in slow

[https://github.com/microsoft/vscode/issues/307760](https://github.com/microsoft/vscode/issues/307760)

## Summary

Team-authored performance bug with an attached trace and a clear diagnosis (re-rendering after each archive instead of batching). Milestone is 'On Deck' — keeping open per policy.

## Change summary

Archiving multiple sessions in a folder blocks the UI for ~350ms because the view re-renders after each individual archive instead of batching them.

## Best solution

Batch the archive operations so the tree/view re-renders once after all sessions in the folder are archived, eliminating the per-item layout thrash visible in the trace.
