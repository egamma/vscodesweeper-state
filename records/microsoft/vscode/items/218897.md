# Issue #218897 — Option to automatically change the debug toolbar's selected active session when launching a new debug session

[https://github.com/microsoft/vscode/issues/218897](https://github.com/microsoft/vscode/issues/218897)

## Summary

Valid feature request, not yet implemented. The current focus logic in debugService.ts explicitly skips focusing newly launched sessions when another session is already focused. A team member acknowledged the request makes sense but noted complexity around child/automatic sessions. Mislabeled as `bug` — this is a feature request.

## Change summary

Add an option to automatically switch the debug toolbar's selected session to a newly launched debug session.

## Best solution

Add a `debug.focusNewSession` setting (or similar) that, when enabled, calls `focusStackFrame(undefined, undefined, session)` for user-initiated (non-child, non-auto) sessions in `launchOrAttachToSession` around line 714 of `src/vs/workbench/contrib/debug/browser/debugService.ts`. The tricky part is distinguishing user-initiated launches from child/automatic sessions, as roblourens noted.
