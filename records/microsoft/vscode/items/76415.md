# Issue #76415 — Allow for configuration files in .devcontainer

[https://github.com/microsoft/vscode/issues/76415](https://github.com/microsoft/vscode/issues/76415)

## Summary

The core need — container-specific launch and task configurations — is already supported via `settings.launch` and `settings.tasks` in devcontainer.json, as confirmed by a team member in comment 6. The specific UX of separate files in .devcontainer/ was never implemented, but the underlying capability exists.

## Change summary

Allow tasks.json and launch.json equivalents to live in .devcontainer/ and be picked up automatically when opening in a container.

## Best solution

Close as designed: the `settings.launch` and `settings.tasks` keys in devcontainer.json already provide container-specific launch and task configurations without polluting the workspace .vscode folder. The devcontainer spec (now at devcontainers/spec) is the appropriate venue for any further file-layout enhancements.

- **Reason:** Dev container configuration UX is tangential to the current product direction; the devcontainer spec is now community-governed outside VS Code core.
