# Issue #214896 — Quoting issue with command sent to the terminal

[https://github.com/microsoft/vscode/issues/214896](https://github.com/microsoft/vscode/issues/214896)

## Summary

The quoting responsibility lies with the debug adapter (debugpy) per the DAP spec. VS Code's `prepareCommand` correctly quotes args when `argsCanBeInterpretedByShell` is false; debugpy must either not set that flag or pre-quote paths with spaces. This is a `*caused-by-extension` issue.

## Change summary

Python debugging fails when Python is installed in a path with spaces (e.g. `C:\Program Files\`) because the command sent to the terminal is not properly quoted.

## Best solution

File this against microsoft/vscode-python or microsoft/debugpy — debugpy should either send `argsCanBeInterpretedByShell: false` so VS Code handles quoting, or pre-quote paths containing spaces when `argsCanBeInterpretedByShell: true`. VS Code's `prepareCommand` (in `src/vs/workbench/contrib/debug/node/terminals.ts`) correctly handles quoting for args when the flag is false.
