# Issue #321103 — Copilot Chat sends full active file even when implicit context is disabled

[https://github.com/microsoft/vscode/issues/321103](https://github.com/microsoft/vscode/issues/321103)

## Summary

The reported behavior is by design: when a user explicitly attaches a selection from a file, the prompt construction in `FileVariable` (line 238-246) intentionally includes a CodeSummary of the full file alongside the selected excerpt (`alwaysIncludeSummary=true`). The implicit context settings the user disabled control a different mechanism (the `CurrentEditor` component / `ChatImplicitContexts`). This is a valid request for a setting to control whether the full-file summary is included when a selection is explicitly attached.

## Change summary

Copilot Chat sends the full active file as additional context alongside a user's explicit selection, even when implicit context settings are disabled.

## Best solution

Add a setting or respect existing implicit-context settings to control whether `FileVariable` includes a `CodeSummary` of the full file when the user explicitly attaches only a selection. The fix would be in `extensions/copilot/src/extension/prompts/node/panel/fileVariable.tsx` (lines 238-246) and the callers that set `alwaysIncludeSummary=true` in `chatVariables.tsx`. Issue https://github.com/microsoft/vscode/issues/324279 is a duplicate of this one and should be consolidated here.
