## Summary

Team-authored feature request to show logpoint messages as inline decorations in the editor. Not yet implemented on main; keeping open for maintainer judgment.

## Change summary

Show the logpoint message text as an inline decoration at the end of the line where the logpoint is set, making it easier to correlate debug console output with source code.

## Best solution

Add inline decorations to the debug breakpoint decoration logic (likely in `src/vs/workbench/contrib/debug/browser/breakpointEditorContribution.ts`) that render the logpoint's `logMessage` as after-content text, similar to how inline values or CodeLens annotations appear. This would also benefit AI-generated logpoints by showing their message without requiring the editor to be visible.
