# Issue #212853 — Can't click on links in streaming chat response

[https://github.com/microsoft/vscode/issues/212853](https://github.com/microsoft/vscode/issues/212853)

## Summary

Team-authored issue; the core problem — click events lost when DOM elements are replaced during streaming — remains on the default rendering path. An experimental incremental rendering feature exists but is off by default and doesn't resolve this for the standard code path.

## Change summary

Clicking links or file references in a streaming chat response does nothing because the DOM element is replaced between mousedown and mouseup, swallowing the click event.

## Best solution

Either land the incremental rendering approach as default (avoiding full DOM replacement during streaming), or implement the mousedown/mouseup workaround discussed in comments 11–14 on the default rendering path in `src/vs/workbench/contrib/chat/browser/widget/chatListRenderer.ts` and `chatMarkdownContentPart.ts`.
