# Issue #225488 — Notification that a user is running emulated should be a blocking modal dialog

[https://github.com/microsoft/vscode/issues/225488](https://github.com/microsoft/vscode/issues/225488)

## Summary

Still open on main: the emulated-build warning at `src/vs/workbench/electron-browser/window.ts:256–273` uses `notificationService.prompt` with `Severity.Warning` and `URGENT` priority, but it is not a blocking modal dialog as requested. Team-authored — keeping open.

## Change summary

Change the Rosetta/emulation notification from a dismissible warning notification to a blocking modal dialog, since the emulated experience is severely degraded.

## Best solution

Convert the `notificationService.prompt(Severity.Warning, …)` call in `src/vs/workbench/electron-browser/window.ts` (line ~257) to use `IDialogService.prompt()` or `IDialogService.show()` as a modal dialog. This ensures users cannot miss the warning and are guided to download the native ARM64 build.
