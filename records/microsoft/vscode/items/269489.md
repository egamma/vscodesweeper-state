# Issue #269489 — `LanguageModelChatInformation#capabilities` should be optional

[https://github.com/microsoft/vscode/issues/269489](https://github.com/microsoft/vscode/issues/269489)

## Summary

Active API design discussion between core team members about whether `LanguageModelChatInformation#capabilities` should be optional. Two commenters explained the current required behavior was a deliberate API-sync decision to prevent authors from accidentally overlooking capabilities like `toolCalling`. The `under-discussion` label is appropriate — this needs a team-level resolution, not external triage.

## Change summary

Make `LanguageModelChatInformation#capabilities` optional so extension authors don't need to spell out an empty object when a model lacks all capabilities.

## Best solution

The team should resolve the API ergonomics trade-off in an API sync: either keep `capabilities` required (as-designed, forcing intentional consideration) or make it optional with documentation emphasizing that authors should review capability flags. The current `under-discussion` label correctly captures this state.
