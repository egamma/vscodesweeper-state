# Issue #235670 — Run Recent Command does not cover multi-line Python REPL commands

[https://github.com/microsoft/vscode/issues/235670](https://github.com/microsoft/vscode/issues/235670)

## Summary

Still reproduces on main: the Python REPL continuation prompt (`... `) is not handled in command detection, so multi-line Python commands are not stored as a single command for 'Run Recent Command'.

## Change summary

Multi-line Python REPL commands (using `... ` continuation prompt) are only partially captured by 'Run Recent Command', showing only the last line.

## Best solution

Extend the command detection in `commandDetectionCapability.ts` to recognize the Python continuation prompt (`... `) and join continuation lines with the preceding `>>> ` command into a single multi-line command entry. The `_getWindowsPrompt` method (which also handles non-Windows prompts despite the name) already detects `>>> ` — it needs a companion mechanism to treat `... ` lines as continuations rather than new commands.
