# Issue #147794 — SCM - screenreader not reading git commit formatting warnings

[https://github.com/microsoft/vscode/issues/147794](https://github.com/microsoft/vscode/issues/147794)

## Summary

Still reproduces on main: the SCM input validation message (e.g. commit line length warning) is rendered as a visual-only context view popup with no ARIA announcement or screen reader notification. The bug remains unfixed.

## Change summary

SCM commit message validation warnings (e.g. line exceeds 72 characters) are not announced to screen readers.

## Best solution

In `renderValidation()` in `scmInput.ts`, add an ARIA announcement (e.g. via `status()` from `src/vs/base/browser/ui/aria/aria.ts` or an `aria-live` region) when a validation message is set, so screen readers are informed of the warning/error text.
