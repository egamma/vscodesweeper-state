# Issue #311542 — [Error] potential listener LEAK — documentSemanticTokens

[https://github.com/microsoft/vscode/issues/311542](https://github.com/microsoft/vscode/issues/311542)

## Summary

Still reproduces on main: each ModelSemanticColoring instance subscribes to themeService.onDidColorThemeChange (line 178), leaking listeners when many models are created (e.g., multi-diff editor). The singleton already hoisted provider-change listeners (PR #298512) but missed this one.

## Change summary

Opening a multi-diff editor with many files triggers a 'potential listener LEAK' error because per-model themeService.onDidColorThemeChange subscriptions accumulate on the shared emitter.

## Best solution

Hoist the themeService.onDidColorThemeChange listener from ModelSemanticColoring into DocumentSemanticTokensFeature (the singleton), adding a method like `handleThemeChange()` on ModelSemanticColoring that clears tokens and reschedules — mirroring how handleRegistryChange/handleProviderDidChange already work. Issue #312368 is a duplicate of this one.
