# Issue #198451 — Editor tab tooltips for TextDocumentContentProvider schemes have backslashes on Windows and don't show the scheme

[https://github.com/microsoft/vscode/issues/198451](https://github.com/microsoft/vscode/issues/198451)

## Summary

Still reproduces on main: when no resourceLabelFormatter is registered for a custom scheme, the label service falls through to `getPathLabel` which uses `resource.fsPath`, stripping the scheme and applying OS-native path separators. The workaround (registering a resourceLabelFormatter) exists but the default behavior for non-file schemes is incorrect.

## Change summary

Editor tab tooltips for TextDocumentContentProvider custom schemes drop the scheme and show backslashes on Windows because the fallback label path uses `resource.fsPath`.

## Best solution

In `getPathLabel` (src/vs/base/common/labels.ts), when the resource scheme is not `file` (and not a known workspace scheme), the label should preserve the URI form (e.g. `scheme:/path`) instead of converting via `fsPath`. Alternatively, guard in `doGetUriLabel` (src/vs/workbench/services/label/common/labelService.ts) to produce a URI-style label for unknown schemes. The `label-provider` label is already set — this belongs with whoever owns that area.
