# Issue #310037 — Agents: The run button's behavior is a little unpredictable

[https://github.com/microsoft/vscode/issues/310037](https://github.com/microsoft/vscode/issues/310037)

## Summary

Team-authored polish issue about the agents window run button selecting the wrong action when both a run action and a worktree-init action exist. Keeping open for maintainer judgment.

## Change summary

The top-level run button should prefer the run action over the worktree-init action when both exist, since the worktree init would have already been executed.

## Best solution

Update the run-button action selection logic in the agents window to deprioritize worktree-init actions when a regular run action is available, so clicking the button without the split menu does the expected thing.
