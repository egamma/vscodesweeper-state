# Issue #153844 — Line Has Inline Suggestion Audio Cue — Pain Point Solution

[https://github.com/microsoft/vscode/issues/153844](https://github.com/microsoft/vscode/issues/153844)

## Summary

The rapid-fire audio cue pain point has been partially addressed by the experimental `accessibility.signalOptions.debouncePositionChanges` setting (with configurable delays), but it remains opt-in and off by default — the issue's core ask (that debouncing be the default behavior) is still unresolved.

## Change summary

Reduce rapid-fire repetition of the inline suggestion audio cue by debouncing or throttling playback during fast keystrokes.

## Best solution

Graduate the experimental `accessibility.signalOptions.debouncePositionChanges` to a stable, on-by-default setting (or at least on-by-default for `lineHasInlineSuggestion`), so the inline-suggestion signal is debounced without user discovery. The delay defaults (400ms sound, 3000ms announcement) in `accessibilityConfiguration.ts` already exist and could become the baseline.
