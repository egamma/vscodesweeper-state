# Issue #206908 — Programmatically setting focus causes custom hover to show

[https://github.com/microsoft/vscode/issues/206908](https://github.com/microsoft/vscode/issues/206908)

## Summary

Still reproduces on main: the hover service's onFocus handler in hoverService.ts does not distinguish programmatic focus from user-initiated (keyboard) focus, so programmatically calling .focus() triggers the custom hover. Partial fixes exist for specific components (ActionList) but the general issue in the hover service remains.

## Change summary

Custom hovers appear when focus is set programmatically (e.g. revealing a view), not just when a user Tabs to an element.

## Best solution

Add a mechanism in `src/vs/platform/hover/browser/hoverService.ts` to suppress hover on programmatic focus — either by detecting FocusEvent characteristics (no relatedTarget plus a programmatic-focus flag), or by adding a short suppression window that callers can trigger before calling .focus(). The ActionList fix (ba9425ba6ba) shows a per-component pattern, but a general solution in the hover service would prevent each component from needing its own workaround.
