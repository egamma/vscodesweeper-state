# Issue #206908 — Programmatically setting focus causes custom hover to show

[https://github.com/microsoft/vscode/issues/206908](https://github.com/microsoft/vscode/issues/206908)

## Summary

Still reproducible on main: the managed hover's focus handler in hoverService.ts triggers on any non-null relatedTarget focus event, including programmatic focus changes (e.g. after dropdown selection). Recent fixes (83bc012e666) only guard against null relatedTarget and hover-originating focus, not the broader programmatic-focus case.

## Change summary

Programmatically setting focus on an element (e.g. after selecting from a dropdown) causes its custom hover tooltip to appear unexpectedly.

## Best solution

Add a mechanism to distinguish keyboard-driven focus from programmatic focus in the managed hover's `onFocus` handler (src/vs/platform/hover/browser/hoverService.ts ~line 654). One approach: suppress hover on focus events that occur within a short window after a known programmatic focus call, or use a flag/signal from the component that triggers the programmatic focus. The ActionList fix (7f1f8807f60) shows one pattern — suppress hover during known programmatic focus changes — that could be generalized in the hover service.
