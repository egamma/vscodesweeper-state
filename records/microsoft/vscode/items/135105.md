# Issue #135105 — Different breakpoint marker for unverified breakpoint before debug session start

[https://github.com/microsoft/vscode/issues/135105](https://github.com/microsoft/vscode/issues/135105)

## Summary

Still valid: current main shows breakpoints as fully verified (red) before any debug session starts, with no distinct 'unknown/pending' state. The feature request remains unimplemented, and the team discussion shows cautious interest but no consensus on the right UX approach.

## Change summary

Show a visually distinct breakpoint marker (icon or hover) for breakpoints set before a debug session starts, to indicate they have not yet been validated by a debug adapter.

## Best solution

Decide on the UX approach discussed in comments — either a new 'unknown' icon state, or keeping the red icon but adding a checkmark for verified breakpoints, or just an informative hover. The logic lives in `getBreakpointMessageAndIcon()` in `src/vs/workbench/contrib/debug/browser/breakpointsView.ts` (line ~1821), which currently only distinguishes states when `debugActive` is true. A new pre-session state would need an additional branch before the `debugActive` checks.
