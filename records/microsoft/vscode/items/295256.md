# Issue #295256 — Power save block on active chat session should have an indicator

[https://github.com/microsoft/vscode/issues/295256](https://github.com/microsoft/vscode/issues/295256)

## Summary

Team-authored feature request for a UI indicator when VS Code is actively blocking system power save (e.g., during an agent/chat session). The power save blocker infrastructure exists in `src/vs/workbench/services/power/` but has no corresponding workbench UI to surface the blocking state to the user. Keeping open per team-member policy.

## Change summary

Add a visible indicator (e.g., status bar item) when VS Code is actively blocking system sleep via a power save blocker, so users can identify which window is preventing sleep.

## Best solution

Add a status bar indicator (or similar UI surface) in the workbench that activates when `startPowerSaveBlocker` is in effect, likely in a new contribution under `src/vs/workbench/contrib/` that listens to `IPowerService` state and renders an icon similar to browser audio/camera indicators. The referenced Copilot chat PR (vscode-copilot-chat#3483) likely triggers the blocker during active sessions.
