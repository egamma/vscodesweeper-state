# Issue #100611 — Fail to load timeline info when activate timeline view using keyboard shortcut

[https://github.com/microsoft/vscode/issues/100611](https://github.com/microsoft/vscode/issues/100611)

## Summary

Team-authored issue on the 'On Deck' milestone — keeping open per policy. The bug is still present on main: `files.openTimeline` only calls `service.setUri(arg)` when arg is a URI, so invoking it via keybinding (no arg) is effectively a no-op that doesn't focus the view or load the active editor's timeline.

## Change summary

Using `files.openTimeline` via a keyboard shortcut (without a URI argument) fails to show or load timeline information.

## Best solution

Fix the `files.openTimeline` command handler in `src/vs/workbench/contrib/timeline/browser/timeline.contribution.ts` to fall back to the active editor's resource URI when no argument is provided, and also focus/reveal the timeline pane — or deprecate it in favor of `timeline.focus` and document that as the recommended keybinding target.
