# Issue #288450 — Git actions fail to work when Git for Windows is installed with Use external OpenSSH option

[https://github.com/microsoft/vscode/issues/288450](https://github.com/microsoft/vscode/issues/288450)

## Summary

Real bug: VS Code sets SSH_ASKPASS to a .sh script with SSH_ASKPASS_REQUIRE=force, but Windows' native OpenSSH (used when Git for Windows is installed with 'Use external OpenSSH') cannot execute .sh scripts, causing CreateProcessW error 193. The workaround is setting git.useIntegratedAskPass to false. Routing to the git area for triage.

## Change summary

Git SSH operations (push/pull) fail with CreateProcessW error 193 when Git for Windows is configured to use external (native Windows) OpenSSH, because VS Code's SSH_ASKPASS is a .sh script that Windows OpenSSH cannot execute.

## Best solution

The ssh-askpass scripts need a Windows-compatible counterpart (e.g. a .bat or .cmd wrapper, or a direct Node invocation) so that native Windows OpenSSH can execute the askpass program. The fix would live in `extensions/git/src/askpass.ts` (env setup) and new Windows-specific askpass scripts alongside the existing .sh files. Alternatively, detect when external OpenSSH is in use and skip setting SSH_ASKPASS_REQUIRE=force to let the SSH agent handle auth.

## Evidence

- **ssh-askpass.sh is a shell script:** extensions/git/src/ssh-askpass.sh is a POSIX shell script — incompatible with Windows native OpenSSH which expects a .exe or .cmd.
- **workaround exists:** Setting git.useIntegratedAskPass to false (extensions/git/src/askpass.ts line 163) prevents VS Code from setting SSH_ASKPASS and SSH_ASKPASS_REQUIRE, allowing the Windows SSH agent to handle authentication.
