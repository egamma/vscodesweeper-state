# Issue #258173 — Split terminal with inherited paths does not work when starting from a WSL terminal

[https://github.com/microsoft/vscode/issues/258173](https://github.com/microsoft/vscode/issues/258173)

## Summary

Already fixed on main and shipped: commit da7155bb4f5 prepends '/' to the URI path when splitting a terminal with a remote authority (WSL), which is exactly the error this issue reports. Shipped in 1.121.0.

## Change summary

Splitting a WSL terminal with inherited CWD fails with a UriError because the Windows path lacks a leading slash when an authority is present.

## Best solution

Keep the shipped fix (commit da7155bb4f5, release 1.121.0); no further change needed. The reporter should update to 1.121.0+ to resolve the issue.
