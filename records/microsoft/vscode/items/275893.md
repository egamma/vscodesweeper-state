# Issue #275893 — Chat confirmation buttons should be a toolbar for accessibility

[https://github.com/microsoft/vscode/issues/275893](https://github.com/microsoft/vscode/issues/275893)

## Summary

Team-authored accessibility bug still open on main: the confirmation approve/deny buttons are rendered as individual Button instances in a plain container, not inside a toolbar widget, so keyboard navigation uses Tab instead of arrow keys. Hard keep-open (MEMBER author).

## Change summary

Chat confirmation approve/deny buttons should be grouped in a toolbar so screen reader users can navigate them with arrow keys instead of Tab.

## Best solution

Wrap the confirmation buttons (approve/deny/dropdown) in `chatConfirmationWidget.ts` inside a proper toolbar or `role="toolbar"` container with arrow-key navigation, similar to how `MenuWorkbenchToolBar` already handles the adjacent actions toolbar.
