# Issue #300748 — Model picker can get in bad state

[https://github.com/microsoft/vscode/issues/300748](https://github.com/microsoft/vscode/issues/300748)

## Summary

Legitimate bug in the model picker's filter logic — a separator remains visible and keyboard focus escapes when filter text matches nothing. The issue has an 'On Deck' milestone (hard keep-open rule applies).

## Change summary

Model picker shows a stray separator and loses keyboard focus when filtering produces no matching items.

## Best solution

Fix the `_applyFilter` logic in `src/vs/platform/actionWidget/browser/actionList.ts` to suppress trailing/orphaned separators when `showAlways` items are the only visible entries, and prevent Down Arrow from moving focus out of the filter input when the list is empty.
