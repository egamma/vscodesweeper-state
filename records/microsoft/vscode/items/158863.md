# Issue #158863 — Merge Editor Highlighting Bug

[https://github.com/microsoft/vscode/issues/158863](https://github.com/microsoft/vscode/issues/158863)

## Summary

Team-authored issue; the underlying viewport-boundary bug still exists on main — `blockDecorations.ts` still uses `ctx.getDecorationsInViewport()` without fetching decorations one line above/below, so block decorations can disappear at the scroll edge.

## Change summary

Merge editor conflict block highlighting disappears when the decoration's anchor line scrolls just out of the viewport.

## Best solution

Either expand `getDecorationsInViewport()` to include one extra line above/below, or have `BlockDecorations.render()` query decorations independently for block-decorated lines near the viewport boundary, as alexdima suggested in the comments.
