# Issue #172482 — explorer.excludeGitIgnore is not working in a multiroot workspace when .gitignore is outside the workspace

[https://github.com/microsoft/vscode/issues/172482](https://github.com/microsoft/vscode/issues/172482)

## Summary

Verified duplicate of #189951 — both report the same root cause: the explorer's gitignore filter only processes .gitignore files encountered during tree traversal within the workspace folder, missing parent .gitignore files above the workspace root. #189951 is the more general framing (any subfolder-opened workspace), has `help wanted`, and absorbs this multiroot-specific report.

## Change summary

explorer.excludeGitIgnore does not respect .gitignore files located above the workspace folder root in multiroot workspaces.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/189951 which tracks the same defect with a clearer repro. The fix would involve walking up from the workspace folder to the git repo root to collect ancestor .gitignore files during ignore-tree initialization in `FilesFilter.updateConfiguration()`.
