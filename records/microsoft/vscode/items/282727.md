# Issue #282727 — A bottom shadow shows incorrectly in chat

[https://github.com/microsoft/vscode/issues/282727](https://github.com/microsoft/vscode/issues/282727)

## Summary

Still unfixed on main: the chat input editor's scrollable element shows a bottom shadow when the input is scrolled, but only a top shadow is appropriate here. The related issue #282942 was closed as a duplicate of this one, confirming this is the canonical tracker.

## Change summary

When the chat input has enough text to scroll, a bottom scroll shadow appears incorrectly alongside the expected top shadow.

## Best solution

Suppress the bottom shadow on the chat input's scrollable element — either by setting `useShadows: false` on the editor's scrollbar options and applying a top-only shadow via CSS, or by targeting the `.shadow.bottom` element in the chat input's scrollable with `display: none`. The fix belongs in `src/vs/workbench/contrib/chat/browser/widget/input/chatInputPart.ts` (editor options) or `src/vs/workbench/contrib/chat/browser/widget/media/chat.css`.
