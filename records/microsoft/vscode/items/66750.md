# Issue #66750 — VS Code holds onto parent console when launched via command line

[https://github.com/microsoft/vscode/issues/66750](https://github.com/microsoft/vscode/issues/66750)

## Summary

This is a genuine upstream issue in Electron/Chromium where `RouteStdioToConsole` calls `AttachConsole`, keeping the parent console attached. VS Code already sets `ELECTRON_NO_ATTACH_CONSOLE` but Electron's own startup path runs before it takes effect. The upstream PR (electron/electron#18698) was closed without merging; the root cause persists in current Electron 42.x. Keeping open as a tracked upstream issue.

## Change summary

When VS Code is launched from a Windows command line, the Electron process holds onto the parent console, preventing `exit` from closing the shell and spamming debug output.

## Best solution

The fix must come from Electron/Chromium — either by not calling `AttachConsole`/`RouteStdioToConsole` for GUI apps, or by properly detaching afterward. VS Code should re-engage with the Electron team (the original upstream PR https://github.com/electron/electron/pull/18698 was closed) to get a proper fix landed in a future Electron release. No viable workaround exists purely within VS Code's codebase.

## Evidence

- **root cause identified:** Chromium's `base::RouteStdioToConsole` calls `AttachConsole` during Electron startup (https://github.com/electron/electron/blob/master/atom/app/atom_main.cc), attaching to the parent console before VS Code's `ELECTRON_NO_ATTACH_CONSOLE` env var can take effect.
- **VS Code workaround present:** VS Code sets `ELECTRON_NO_ATTACH_CONSOLE=1` in `src/vs/code/node/cli.ts:226` and `src/vs/platform/shell/node/shellEnv.ts:115`, but this doesn't prevent the initial attach in Electron's own startup code path.
- **upstream PR stalled:** https://github.com/electron/electron/pull/18698 was created to fix this in Electron but was closed without merging.
- **current Electron version:** VS Code currently uses Electron 42.5.0; the issue persists as no upstream fix has landed.
- **Windows console team confirmation:** DHowett from the Windows console team confirmed via stack trace that `electron.exe` calls `AttachConsole` during startup (comment from 2019-01-23).
