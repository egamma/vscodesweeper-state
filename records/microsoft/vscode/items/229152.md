# Issue #229152 — Enabling the `issueReporter.experimental.auxWindow` setting prevents Issue Reporter window from being focused

[https://github.com/microsoft/vscode/issues/229152](https://github.com/microsoft/vscode/issues/229152)

## Summary

Still reproducible on main: when the Issue Reporter aux window is already open but behind another window, calling 'Help: Report Issue...' again calls `window.focus()` in `hasToReload()`, which does not reliably bring the auxiliary window to the foreground on macOS. No fix has been committed.

## Change summary

Re-invoking 'Help: Report Issue...' when the Issue Reporter window is behind another window does not bring it to the foreground.

## Best solution

In `hasToReload()` (src/vs/workbench/contrib/issue/browser/issueFormService.ts), supplement the `window.focus()` call with a native window raise/focus mechanism — on Electron, use `BrowserWindow.fromBrowserView` or the native host service to bring the auxiliary window to the front. The wizard editor-pane path (nativeIssueFormService.ts) may also need a `revealIfOpened` check when re-opening.
