# Issue #81231 — Remote terminals don't resolve variables when restored on startup

[https://github.com/microsoft/vscode/issues/81231](https://github.com/microsoft/vscode/issues/81231)

## Summary

Fixed on main since late 2020: commit 861e7bcc1b2 added an await on `whenRemoteConfigurationLoaded()` before resolving variables in the remote terminal creation path, which is exactly the race Tyriar diagnosed. A contributor confirmed it no longer reproduces on 1.129. Safe to close as completed.

## Change summary

Remote terminals failed to resolve `${workspaceFolder}` and similar variables when restored on startup because the configuration resolver wasn't ready yet.

## Best solution

Keep the shipped fix (await remote configuration before variable resolution on the remote createProcess path). No further change needed — close as completed.
