# Issue #286513 — Title bar visibility regression in 1.107.1

[https://github.com/microsoft/vscode/issues/286513](https://github.com/microsoft/vscode/issues/286513)

## Summary

Confirmed upstream regression caused by the Electron 37→39 update. The issue is already correctly labeled `electron-39-update`. The window creation code in VS Code hasn't changed — when `titleBarStyle: native` is set, the native frame is shown by the OS, and the Electron 39 upgrade changed how this interacts with `customTitleBarVisibility: never` on Linux/GNOME. Keeping open for the team to track against Electron.

## Change summary

On Linux with `window.titleBarStyle: native` and `window.customTitleBarVisibility: never`, the native window decoration (title bar with close button) is visible in 1.107.1 but was hidden in 1.106.

## Best solution

Root-cause this in Electron 39's window decoration handling on Linux. Either file an upstream Electron issue for the behavior change, or add a VS Code-side workaround in `defaultBrowserWindowOptions` (e.g. setting `frame: false` when both `titleBarStyle` is native and `customTitleBarVisibility` is `never` on Linux). The `electron-39-update` label is already applied.
