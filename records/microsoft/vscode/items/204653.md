# Issue #204653 — Some Hebrew letters appear with a square in JSON

[https://github.com/microsoft/vscode/issues/204653](https://github.com/microsoft/vscode/issues/204653)

## Summary

Still reproduces on main: Hebrew characters ו, ט, י, ס are in the ambiguous confusable list and the word-context guard in `shouldHighlightNonBasicASCII` doesn't fire when all characters in a word happen to be ambiguous (no "non-confusable non-basic-ASCII" anchor exists to prove the word is non-Latin).

## Change summary

Certain Hebrew letters (ו, ט, י, ס) get falsely highlighted with a confusable-character box when the word they form consists entirely of ambiguous code points, or when mixed with ASCII digits.

## Best solution

The word-context heuristic in `shouldHighlightNonBasicASCII` needs an additional check: when ALL characters in a word are non-basic-ASCII (even if individually ambiguous), the word should not be flagged — it is clearly not a homoglyph attack. Alternatively, add a Hebrew (`he`) locale to the ambiguous-character data that removes these code points, or use Unicode script detection to recognize a uniform-script word. The fix lives in `src/vs/editor/common/services/unicodeTextModelHighlighter.ts` (the word-context logic around line 216).
