# Issue #242187 — Notebook find filters icon is always "enabled"

[https://github.com/microsoft/vscode/issues/242187](https://github.com/microsoft/vscode/issues/242187)

## Summary

Team-authored polish request to improve the notebook find filter icon UX. The filter icon uses toggle-active styling (border/background) via `applyStyles()` whenever `isModified()` is true, which makes it look permanently "enabled" even though it's just a dropdown. The suggestion to switch to an ellipsis icon and merge the two overflow menus is a reasonable UX improvement. Keeping open per team-author policy.

## Change summary

The notebook find widget's filter icon always appears visually "enabled" (with active toggle styling) because it uses toggle decoration for a simple dropdown action.

## Best solution

In `src/vs/workbench/contrib/notebook/browser/contrib/find/notebookFindReplaceWidget.ts`, remove the toggle-style active decoration from `NotebookFindInputFilterButton.applyStyles()` so the icon no longer looks "checked", and consider replacing the filter codicon with an ellipsis icon. Optionally merge the filter dropdown and the existing `...` overflow menu into a single dropdown to reduce clutter.
