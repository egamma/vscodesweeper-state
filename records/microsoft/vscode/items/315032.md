# Issue #315032 — Subagent fails to resolve custom OAI model although available for main agent

[https://github.com/microsoft/vscode/issues/315032](https://github.com/microsoft/vscode/issues/315032)

## Summary

Custom OAI model fails to resolve in subagent context. A maintainer has already engaged with debugging guidance. The root cause may be that the custom model lacks required metadata (toolCalling/agentMode capabilities) or isn't registered in the model cache at subagent invocation time. Needs further investigation or reporter follow-up.

## Change summary

Custom OpenAI model registered for the main agent cannot be selected when launching a subagent, even though it works as the primary model.

## Best solution

Either the subagent model resolution should include custom OAI models that the main agent can already use (fixing the filter in `resolveSubagentModel`/`getAvailableModelsInfo` in `src/vs/workbench/contrib/chat/common/tools/builtinTools/runSubagentTool.ts`), or the error message should clearly state which capability the model is missing so the user can fix their model configuration.
