# Issue #235890 — TreeWidget nesting performance

[https://github.com/microsoft/vscode/issues/235890](https://github.com/microsoft/vscode/issues/235890)

## Summary

Still reproduces on main: deeply nested trees cause UI freeze due to O(n²) indent guide rendering (each row creates depth-many DOM elements). Related tracking issues #213839 and #250787 remain open with no fix merged.

## Change summary

Deeply nested tree items (e.g. 500+ levels in Outline view) cause VS Code to freeze due to per-row indent guide DOM element creation scaling with depth.

## Best solution

The fix tracked by https://github.com/microsoft/vscode/issues/250787 (avoid individual indent guides via a single background-image or CSS approach) would resolve this. The tree renderer in abstractTree.ts creates one `.indent-guide` div per nesting level per visible row — replacing that with a scalable rendering strategy eliminates the quadratic DOM cost.
