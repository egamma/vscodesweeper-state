# Issue #175330 — Diagnostics jump around in the minimap when scrolling

[https://github.com/microsoft/vscode/issues/175330](https://github.com/microsoft/vscode/issues/175330)

## Summary

Still plausible on main: the minimap layout's `startLineNumber` is computed via `Math.floor` and can oscillate by ±1 during fine-grained touchpad scrolling, causing decorations (diagnostics) to visually jump. The existing stabilization heuristic (monotonic startLineNumber w.r.t. scroll direction) helps but doesn't fully eliminate the jitter for small sub-line scroll deltas.

## Change summary

Diagnostics in the minimap shift by approximately ±1 line when scrolling slowly with a touchpad.

## Best solution

Improve the `startLineNumber` stabilization in `MinimapLayout.create` (around line 377–396 of `src/vs/editor/browser/viewParts/minimap/minimap.ts`) so that it resists oscillation during fine-grained scroll events — e.g. by adding a hysteresis threshold or snapping to the previous value when the delta is sub-line.
