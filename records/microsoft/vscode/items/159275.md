# Issue #159275 — Sticky keys and context menu keybinding (Shift F10)

[https://github.com/microsoft/vscode/issues/159275](https://github.com/microsoft/vscode/issues/159275)

## Summary

Still present on main: the context menu keyboard shortcut check in listWidget.ts requires `e.shiftKey` to be true on the F10 keydown/keyup event, which may not be reported correctly by Chromium/Electron when Windows Sticky Keys is active and the keys are pressed sequentially rather than simultaneously.

## Change summary

Shift+F10 context menu shortcut does not work when Windows Sticky Keys is enabled and keys are pressed sequentially.

## Best solution

This likely requires understanding how Electron/Chromium reports modifier state under Windows Sticky Keys. If the browser simply doesn't set `shiftKey` on the F10 event when sticky keys are active, the fix would need to track latched modifier state independently — but that may be an upstream Chromium/Electron concern. Investigate whether Electron correctly reports `shiftKey` in this scenario before attempting a VS Code-level workaround.
