# Issue #159275 — Sticky keys and context menu keybinding (Shift F10)

[https://github.com/microsoft/vscode/issues/159275](https://github.com/microsoft/vscode/issues/159275)

## Summary

Keep open: the report is a clear Windows accessibility bug and current list context-menu handling still appears to depend on `event.shiftKey` being present on the F10 event. I do not see enough evidence for a close or duplicate.

## Change summary

With Windows Sticky Keys enabled, pressing Shift then F10 separately should open the focused list/tree item's context menu in VS Code.

## Best solution

Investigate the shared list/tree context menu keyboard path in `src/vs/base/browser/ui/list/listWidget.ts` and make the Shift+F10 trigger work with Windows Sticky Keys without regressing the normal ContextMenu key or simultaneous Shift+F10 behavior.
