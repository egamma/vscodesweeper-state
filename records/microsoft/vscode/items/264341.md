# Issue #264341 — stage/revert changes options missing from menu when editor has lost focus

[https://github.com/microsoft/vscode/issues/264341](https://github.com/microsoft/vscode/issues/264341)

## Summary

Confirmed on current main and milestoned On Deck — keeping open. The `isInDiffRightEditor` context key is toggled by editor widget focus/blur, so clicking the toolbar menu (which blurs the editor) sets it to false, hiding menu items that depend on it.

## Change summary

Git stage/revert/unstage menu items in the diff editor title bar disappear when the editor loses focus (e.g., after opening the menu once).

## Best solution

The `editor/title` menu contributions should not rely on `isInDiffRightEditor` (a focus-dependent context key). Either the when-clause should use a non-focus-dependent condition (e.g. checking the active editor type), or the context key itself should stick while the diff editor is the active editor rather than tracking widget focus. The fix touches `extensions/git/package.json` (when clauses) and/or `src/vs/editor/browser/widget/diffEditor/components/diffEditorEditors.ts` (context key lifecycle).
