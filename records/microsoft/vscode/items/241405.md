# Issue #241405 — Window title bar remains in -inFocus color even when multiple windows open at the same time.

[https://github.com/microsoft/vscode/issues/241405](https://github.com/microsoft/vscode/issues/241405)

## Summary

Confirmed bug: `isInactive` defaults to `false` (active) in titlebarPart.ts, so windows restored on startup that never receive focus remain styled as active. An open PR (#247773) addresses this. Keeping open pending that PR's resolution.

## Change summary

When multiple VS Code windows are restored on startup, non-focused windows incorrectly display the active title bar color because they never receive a blur event.

## Best solution

Land or iterate on the open PR https://github.com/microsoft/vscode/pull/247773 which fixes the initialization of `isInactive` on window restore. The fix likely needs to check `document.hasFocus()` at construction time in `src/vs/workbench/browser/parts/titlebar/titlebarPart.ts` line 293 rather than defaulting to `false`.
