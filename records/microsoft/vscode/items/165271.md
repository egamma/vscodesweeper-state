# Issue #165271 — 100% CPU with explorer and code moving files to folders

[https://github.com/microsoft/vscode/issues/165271](https://github.com/microsoft/vscode/issues/165271)

## Summary

Still valid on current main: the explorer's file-change handler uses a 500ms RunOnceScheduler that triggers a full refresh on each batch, so a flood of 15k file moves causes continuous refreshes and high CPU. A team member acknowledged the problem but no fix was implemented.

## Change summary

Explorer causes 100% CPU when an external script rapidly moves thousands of files in the open workspace.

## Best solution

Introduce adaptive throttling or coalescing in `ExplorerService.onFileChangesScheduler` (src/vs/workbench/contrib/files/browser/explorerService.ts) — when the volume of incoming file-change events exceeds a threshold within the debounce window, back off to a longer delay (e.g. 2–5s) or skip intermediate refreshes entirely, resuming normal cadence once the burst subsides.
