# Issue #274135 — renaming top terminal UI

[https://github.com/microsoft/vscode/issues/274135](https://github.com/microsoft/vscode/issues/274135)

## Summary

Team-authored visual bug that still reproduces on main: the rename input's focus outline on the topmost terminal tab is clipped by `overflow: hidden` on the parent split-view container. No fix found in git history.

## Change summary

Renaming the top terminal tab causes the focus/selection outline to be cut off by the terminal container's overflow:hidden.

## Best solution

Add a small top padding or margin to the tabs list (or set `overflow: visible` on the relevant axis) so the focus outline of the first item isn't clipped. The fix likely lives in `src/vs/workbench/contrib/terminal/browser/media/terminal.css` around line 169–172 where `.split-view-view` sets `overflow: hidden`.
