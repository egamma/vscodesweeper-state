# Issue #162639 — Wrong AppData folder

[https://github.com/microsoft/vscode/issues/162639](https://github.com/microsoft/vscode/issues/162639)

## Summary

An upstream Electron/Chromium sandbox issue affecting UNC-path AppData in enterprise VDI environments. The last response (July 2023) tested Insiders with --disable-chromium-sandbox but results were inconclusive. The issue has been stale for over two years with no confirmation whether the problem persists on current Electron 42 / VS Code Insiders, and the confirmation-pending label was never resolved.

## Change summary

VS Code resolves a UNC-based %APPDATA% path incorrectly (replacing the server name with C:\) since ~1.69, breaking enterprise VDI setups that redirect AppData to a network share.

## Best solution

The reporter and commenters should test on the latest VS Code Insiders (now on Electron 42, far newer than the 2022-era Electron). If the issue persists, an upstream Chromium/Electron bug should be filed with a minimal repro. If it's resolved by the many Electron upgrades since, this can be closed as fixed upstream.
