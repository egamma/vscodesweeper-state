# Issue #319574 — VS Code Agents App - New sessions claims to not have tool access

## Summary

The AI model incorrectly claims it lacks tool access on the first turn in the Agents window, but tools work when the user insists. This suggests a prompting or tool-listing timing issue specific to the Agents window. Multiple tool-related fixes have landed since 1.122.1 but none clearly address this exact first-turn behavior.

## Change summary

In the Agents window, the AI model claims it doesn't have access to GitHub tools on the first turn but uses them successfully after being told it has them.

## Best solution

Investigate whether the system prompt or tool list sent on the first turn in the Agents window differs from the regular VS Code chat — likely the tool descriptions or system instructions need to explicitly inform the model of its available tools, or there's a race where tools aren't fully registered before the first request is sent.
