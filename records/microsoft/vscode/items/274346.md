# Issue #274346 — super + pageup / super + pagedown acts like constant pageup / pagedown press

[https://github.com/microsoft/vscode/issues/274346](https://github.com/microsoft/vscode/issues/274346)

## Summary

Upstream mutter bug causing stuck key-repeat when switching GNOME workspaces to an Electron window. Already fixed in mutter/GNOME 50 — updating the compositor resolves it. Classifying as upstream since no VS Code or Electron change is needed.

## Change summary

GNOME workspace-switching shortcuts (Super+PageUp/Down, Ctrl+Alt+Arrow) cause VS Code to behave as if a key is held down, requiring another keypress to stop.

## Best solution

Close as upstream once GNOME 50 has been widely available for a release cycle. The fix lives in mutter (https://gitlab.gnome.org/GNOME/mutter/-/merge_requests/4737). Users on GNOME 49 should update their compositor. No VS Code-side workaround is needed or possible.
