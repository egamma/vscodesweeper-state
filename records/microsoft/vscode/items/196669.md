# Issue #196669 — Cannot read properties of undefined (reading 'isInHiddenArea')

## Summary

Still present on main: `this._zones[visibleWhitespace.id]` at line 366 of viewZones.ts is accessed without a guard, crashing when the whitespace viewport data references a zone ID that has already been removed from `this._zones` (race during render after inline completion accept in a notebook markup cell).

## Change summary

TypeError crash when rendering view zones because a whitespace ID from viewport data references a zone that no longer exists in the `_zones` map.

## Best solution

Add a null guard at line 366 of `src/vs/editor/browser/viewParts/viewZones/viewZones.ts` — if `this._zones[visibleWhitespace.id]` is undefined, `continue` (skip the stale whitespace entry). This is consistent with how other lookups in the same file handle potentially-stale IDs.
