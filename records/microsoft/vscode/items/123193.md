# Issue #123193 — Support fragment in Uri label formatter

[https://github.com/microsoft/vscode/issues/123193](https://github.com/microsoft/vscode/issues/123193)

## Summary

Still valid: the label formatter regex at labelService.ts:80 only supports scheme, authority, authoritySuffix, path, and query — fragment is not present. The gap remains on current main, and the request comes from a contributor working on notebook cell URIs.

## Change summary

Allow `${fragment}` (or `${fragment.xxx}`) as a template variable in ResourceLabelFormatter so notebook cell URIs and similar fragment-bearing URIs can customize their display labels.

## Best solution

Add `fragment` support to the label matching regex and substitution logic in `src/vs/workbench/services/label/common/labelService.ts` (line ~80 and ~440–486). Small, localized change — extend the regex to accept `fragment` and handle it alongside `query` in the switch block. The label-provider area owner should decide whether plain `${fragment}` suffices or whether JSON-parsed `${fragment.key}` (mirroring query) is also needed.
