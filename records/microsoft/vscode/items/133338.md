# Issue #133338 — Add code lens option/alternative for native testing

[https://github.com/microsoft/vscode/issues/133338](https://github.com/microsoft/vscode/issues/133338)

## Summary

Team-authored feature request still unimplemented on main — testing has gutter decorations but no code lens option to run/debug tests. The author revived it in Jan 2024 via #203913; keeping open per hard keep-open rules for MEMBER-authored issues.

## Change summary

Add an optional code lens (as an alternative to gutter icons) for running and debugging tests inline.

## Best solution

Implement a `testing.codeLens.enabled` setting in `src/vs/workbench/contrib/testing/common/configuration.ts` and a CodeLensProvider in the testing contribution that offers Run/Debug/Coverage actions above test functions, off by default. The existing `testingDecorations.ts` gutter infrastructure provides the model; the code lens would surface the same actions in a different presentation.
