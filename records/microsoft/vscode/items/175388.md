# Issue #175388 — Error notification shown in addition to modal error dialog

[https://github.com/microsoft/vscode/issues/175388](https://github.com/microsoft/vscode/issues/175388)

## Summary

Team-authored issue on the On Deck milestone — keeping open. The duplicate notification is still reproducible from source: `remoteAgentService.ts` shows a notification from its independent `getRawEnvironment()` error path, which receives a different error instance than the one the SSH extension marked as handled.

## Change summary

When reconnecting to a remote SSH host without internet, both a modal error dialog (from the remote-SSH extension) and a notification (from the remote agent service) are shown, when only the modal should appear.

## Best solution

Ensure `remoteAgentService.ts`'s `getRawEnvironment()` error path respects the handled state — either by propagating the handled flag through `_setResolvedAuthorityError` so downstream consumers can detect it, or by suppressing the notification when the resolver has already reported the error. The fix likely touches `src/vs/workbench/services/remote/electron-browser/remoteAgentService.ts` and possibly the resolver error propagation in `src/vs/platform/remote/common/remoteAuthorityResolver.ts`.
