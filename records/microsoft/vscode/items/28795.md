# Issue #28795 — Editor: scroll jumps randomly (related to Chrome, Electron, xinput)

[https://github.com/microsoft/vscode/issues/28795](https://github.com/microsoft/vscode/issues/28795)

## Summary

Long-standing upstream Chromium/Electron bug on Linux (X11) where scroll events queue while the window is unfocused and replay on refocus. Already correctly labeled upstream with 456 upvotes. Four open community PRs attempt a VS Code-level workaround but none have been merged — keeping open for maintainer review of those PRs.

## Change summary

On Linux/X11, switching back to VS Code causes the editor to jump because queued scroll events from other windows replay on focus.

## Best solution

Review the four open community PRs (https://github.com/microsoft/vscode/pull/316951, https://github.com/microsoft/vscode/pull/317464, https://github.com/microsoft/vscode/pull/317465, https://github.com/microsoft/vscode/pull/317477) that attempt to drop stale wheel events after focus return. If a VS Code-level workaround is viable, land the best PR; otherwise, continue tracking upstream Chromium fix (https://bugs.chromium.org/p/chromium/issues/detail?id=608246).
