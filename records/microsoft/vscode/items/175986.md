# Issue #175986 — Allow VS Code extensions to trigger audio cues

[https://github.com/microsoft/vscode/issues/175986](https://github.com/microsoft/vscode/issues/175986)

## Summary

Valid feature request with strong accessibility use cases. The internal IAccessibilitySignalService exists but no extension API has been exposed. The issue has 22+ upvotes, is in Backlog, and has active team engagement — should remain open.

## Change summary

Allow VS Code extensions to trigger accessibility signals (audio cues) via a public extension API, including both built-in signals and potentially custom tones.

## Best solution

Design and ship a proposed extension API surface that lets extensions declare and play accessibility signals — likely modeled on IAccessibilitySignalService with semantic signal types (not raw sound reuse). The API design discussion in comment #11 by hediet outlines a sound approach: extensions declare new signals with default sounds, restricted to their own contributions. Implementation would live in a new proposed API file and corresponding extHost/mainThread pair under src/vs/workbench/api/.
