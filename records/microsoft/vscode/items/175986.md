# Issue #175986 — Allow VS Code extensions to trigger audio cues

[https://github.com/microsoft/vscode/issues/175986](https://github.com/microsoft/vscode/issues/175986)

## Summary

Valid feature request with 22 upvotes (past the 20-upvote threshold) and clear accessibility use cases. No extension API for accessibility signals exists on current main — the internal IAccessibilitySignalService is not exposed. The issue remains unimplemented and should stay open in the backlog.

## Change summary

Allow VS Code extensions to trigger accessibility audio signals (existing built-in sounds and custom tones) via a new extension API.

## Best solution

Design a proposed extension API (as outlined by @hediet in comment 11) that lets extensions declare semantic audio cues with default sounds, then play only their own contributed cues. This prevents misuse of built-in sounds while enabling accessibility-focused extensions like indent-level sonification and bookmark navigation feedback. The API would live alongside the existing internal IAccessibilitySignalService in the accessibility-signal area.
