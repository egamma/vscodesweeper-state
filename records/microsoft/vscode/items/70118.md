# Issue #70118 — default run task

[https://github.com/microsoft/vscode/issues/70118](https://github.com/microsoft/vscode/issues/70118)

## Summary

Valid, unimplemented feature request for a 'Run' task group alongside the existing Build and Test groups. Team members discussed it positively in 2019 and it remains unimplemented on current main — the TaskGroup enum still only defines Clean, Build, Rebuild, and Test. Keeping open.

## Change summary

Add a default 'Run' task group (build + execute) analogous to the existing default Build and Test task groups.

## Best solution

Add a `Run` entry to `TaskGroup` in `src/vs/workbench/contrib/tasks/common/tasks.ts`, wire up a 'Tasks: Run Task (Run Group)' command with a default keybinding in the task contribution, and surface 'Configure Default Run Task' in the command palette — mirroring the Build/Test pattern. Team members @alexr00 and @weinand already agreed this fits as a task group rather than a debugger-less launch config.
