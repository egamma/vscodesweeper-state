# Issue #221470 — Accessibility issue with screen reader and vscode keep saying name of project and WSL env (program title bar value)

[https://github.com/microsoft/vscode/issues/221470](https://github.com/microsoft/vscode/issues/221470)

## Summary

This appears to be an upstream NVDA issue — NVDA re-announces the window title bar when focus moves to Quick Open or IntelliSense widgets. The VS Code team couldn't reproduce with NVDA on Windows (without WSL) or with JAWS/VoiceOver. An open issue exists in the NVDA repo tracking this behavior.

## Change summary

NVDA screen reader redundantly announces the full window title bar text when Quick Open or IntelliSense popups appear in VS Code with WSL.

## Best solution

Track this as upstream (NVDA). The root cause is likely in how NVDA interprets focus transitions between VS Code's `role="application"` container and popup widgets. The open NVDA issue (nvaccess/nvda#18385) owns the fix. VS Code could explore a workaround (e.g. adjusting ARIA roles during popup display), but the team couldn't reproduce and it's specific to NVDA + WSL.
