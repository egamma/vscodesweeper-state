# Issue #324389 — Agents View: Symlinks decorations render multiple times

[https://github.com/microsoft/vscode/issues/324389](https://github.com/microsoft/vscode/issues/324389)

## Summary

Still reproduces on main: SessionsExplorerView extends ExplorerView, which registers an ExplorerDecorationsProvider per instance — when both views exist, symlink decorations are applied multiple times.

## Change summary

Symlink decorations (the ↷ badge) render multiple times in the Agents window file explorer because the decorations provider is registered once per ExplorerView instance.

## Best solution

Make ExplorerDecorationsProvider registration singleton (e.g. guard it so only one instance registers with the decoration service), or have SessionsExplorerView skip re-registering when the workbench ExplorerView already registered the same provider. The fix lives in `src/vs/workbench/contrib/files/browser/views/explorerView.ts` around line 839–841.
