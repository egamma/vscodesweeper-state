# Issue #238210 — Inconsistent line copy-paste on Chromebook

[https://github.com/microsoft/vscode/issues/238210](https://github.com/microsoft/vscode/issues/238210)

## Summary

Keep this open as a real editor clipboard bug: the report has clear steps, extensions-disabled confirmation, and a later confirmation that it still reproduced on 1.100.3. Current main has had clipboard/paste refactors, but I did not find a fixing commit tied to this Chromebook/Linux line-paste behavior, so a close would be premature.

## Change summary

Copying with an empty selection and pasting on Chromebook/Linux inserts the copied line at the cursor instead of as a new line.

## Best solution

Verify the repro on current Insiders/main in the editor clipboard path, especially whether empty-selection copy metadata survives the native Linux/Chromebook clipboard round trip; if it still reproduces, fix the `pasteOnNewLine` metadata propagation in the editor clipboard/copy-paste controller flow and add a non-skipped regression test for the Linux/native path.

## Evidence

- **report details:** https://github.com/microsoft/vscode/issues/238210 reports VS Code 1.96.4 on Linux arm64/Chromebook with extensions disabled; Ctrl+C with an empty selection followed by Ctrl+V pastes into the middle of the line instead of as a new line.
- **still reproduced:** A later comment on https://github.com/microsoft/vscode/issues/238210 says the issue was still present on VS Code 1.100.3 and appeared Linux-specific; another commenter also reported similar behavior on macOS 15.1.
- **current code path:** Current main stores empty-selection copy state as `isFromEmptySelection` in `src/vs/editor/browser/controller/editContext/clipboardUtils.ts` and consumes it as `pasteOnNewLine` in `src/vs/editor/contrib/dropOrPasteInto/browser/copyPasteController.ts`; this is the right area to investigate rather than an extension-caused issue.
- **not proven fixed:** Commit 8aead9b9ba3834bc30b5da4a99150f8b96b429fe changed native paste handling and is contained in 1.100.3, but the issue was later reported still present on 1.100.3. Commit 91eeae05dd2c02b7e79c0f27ea1a1437a40611bd refactored copy/paste event routing in 2026, but it does not reference https://github.com/microsoft/vscode/issues/238210 or the Chromebook/Linux line-copy bug.
- **test gap:** `extensions/vscode-api-tests/src/singlefolder-tests/documentPaste.test.ts` has a test named `Copy with empty selection should copy entire line`, but the containing suite is `suite.skip` with a note about Linux Desktop flakiness and web errors, so it does not provide active regression coverage for this report.
