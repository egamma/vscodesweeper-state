# Issue #159095 — VS Code is removed from anchor layout

[https://github.com/microsoft/vscode/issues/159095](https://github.com/microsoft/vscode/issues/159095)

## Summary

Upstream Electron bug (snap layout lost on minimize/restore) remains unfixed. The linked Electron issue was closed with a partial fix (hide/show), but commenters confirm the original minimize/restore scenario still reproduces on Electron 37+. VS Code is now on Electron 42 but the root cause persists upstream.

## Change summary

VS Code is removed from Windows snap/anchor layout when minimizing and restoring a snapped window group.

## Best solution

The root cause is in Electron's window management on Windows. The Electron fix in https://github.com/electron/electron/pull/46006 only addressed hide()/show(), not minimize/restore. A new upstream issue should be filed against Electron to track the remaining scenario, since the original was closed prematurely.
