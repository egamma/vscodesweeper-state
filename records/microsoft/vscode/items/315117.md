# Issue #315117 — Release validation using quality metrics

[https://github.com/microsoft/vscode/issues/315117](https://github.com/microsoft/vscode/issues/315117)

## Summary

This is a broad build/release pipeline feature request, not a product bug, and there is not enough evidence to close it as already implemented or out of scope. Route it to the build/release owners for a decision on whether an additional readiness-metrics gate belongs in the VS Code release pipeline.

## Change summary

Add a CI/CD validation stage that gates release readiness using quality metrics such as coverage, build stability, critical issues, regressions, and Insiders errors.

## Best solution

Have the build/release maintainers decide whether to extend `build/azure-pipelines/product-build.yml` or `build/azure-pipelines/product-publish.yml` with an explicit readiness gate, or document that the existing Quality, Publish, and SanityTests stages are the intended release validation model.

- **remaining gap:** A repository search did not show an existing adaptive readiness score or gate over the requested metrics such as unresolved critical issues, coverage thresholds, regression failures, or Insiders error signals.
