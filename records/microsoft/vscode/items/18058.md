# Issue #18058 — Evaluate selected expression on hover while debugging?

[https://github.com/microsoft/vscode/issues/18058](https://github.com/microsoft/vscode/issues/18058)

## Summary

Valid feature request still unimplemented on main — the debug hover only uses EvaluatableExpressionProvider or regex extraction at a position and never checks the editor selection. 15 upvotes, below the 20-vote Backlog promotion threshold but in the Backlog milestone; keeping open.

## Change summary

Evaluate the editor's selected text expression on hover during a debug session, rather than only evaluating the single symbol under the cursor.

## Best solution

Add selection-aware logic in the debug hover compute path (src/vs/workbench/contrib/debug/browser/debugHover.ts): when the hover position falls inside the editor's current non-empty selection, use the selected text as the expression to evaluate instead of calling getEvaluatableExpressionAtPosition. This is a small, self-contained change in the debug hover widget.
