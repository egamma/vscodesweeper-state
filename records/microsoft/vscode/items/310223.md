# Issue #310223 — Allow github.copilot.chat.commitMessageGeneration.instructions to read files outside of the workspace

[https://github.com/microsoft/vscode/issues/310223](https://github.com/microsoft/vscode/issues/310223)

## Summary

Valid feature request — the `_collectInstructionsFromFile` method in the Copilot extension only resolves instruction file paths relative to workspace folders, confirming the limitation. The gap remains on current main.

## Change summary

Allow instruction settings (commitMessageGeneration, reviewSelection, pullRequestDescriptionGeneration) to resolve file paths outside the workspace (absolute paths and `~/` expansion).

## Best solution

Extend `_collectInstructionsFromFile` in `extensions/copilot/src/platform/customInstructions/common/customInstructionsService.ts` to detect absolute paths and `~/`-prefixed paths (similarly to how the instruction-location config already does at line 157–161), reading them directly instead of joining with workspace folder URIs.
