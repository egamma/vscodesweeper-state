# Issue #325381 — Copilot agent regex search returns zero results for files outside workspace root

[https://github.com/microsoft/vscode/issues/325381](https://github.com/microsoft/vscode/issues/325381)

## Summary

Team-authored issue — keeping open per policy. The Copilot CLI SDK's grep/rg tool scopes searches to workspace folders; files at absolute paths outside the workspace return zero results. The issue is real and traces to how the SDK invokes ripgrep with workspace-root scoping. Opening the file in an editor buffer works around it because VS Code's in-memory model search can find the content.

## Change summary

The Copilot agent's grep/regex search tool returns zero results for files outside the workspace root, even when given an absolute path, causing the agent to thrash on valid files in sibling directories.

## Best solution

The fix likely belongs in the Copilot CLI SDK's ripgrep invocation layer — when a tool call specifies an explicit absolute path outside the workspace, the search should honor that path rather than silently scoping to workspace folders. The ripgrep shim is in `extensions/copilot/src/extension/chatSessions/copilotcli/node/ripgrepShim.ts` but the actual search scoping is in the `@github/copilot/sdk` package. Coordinate with the SDK team to allow out-of-workspace searches when an explicit path is provided.
