Verified duplicate of #295150, which is already triaged as a feature-request in the Backlog milestone with 6 upvotes and tracks the same request — letting users pick a cheaper model for compaction.

## Change summary

Allow users to configure a separate, cheaper model for context compaction (/compact) instead of always using the active chat model.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/295150 — it already owns this work, is triaged with `feature-request` and `chat-summarization` labels, and sits in the Backlog.
