# Issue #131502 — Source code downloaded via the debuggers sourceRequest creates lots of duplicate open files across debug sessions

[https://github.com/microsoft/vscode/issues/131502](https://github.com/microsoft/vscode/issues/131502)

## Summary

Still reproduces on main: debug source URIs embed `sessionId` and `sourceReference` in the query string, so each debug session creates unique URIs for identical sources. The `closeReadonlyTabsOnEnd` setting (PR #199898) is a partial mitigation but deduplication across sessions and history cleanup were never implemented. A team member expressed intent to work on this but no follow-up landed.

## Change summary

Source files downloaded via the debugger's `sourceRequest` create duplicate open editor tabs across debug sessions because each session produces a unique URI for the same content.

## Best solution

Implement cross-session deduplication in `getUriFromSource()` (e.g. by omitting or normalizing the session component for sources with a stable path), and call `IHistoryService.removeFromHistory()` for debug-scheme URIs when a session ends. The DAP-level discussion about a stable source identity field (https://github.com/microsoft/debug-adapter-protocol/issues/444) may also be relevant for a longer-term solution.
