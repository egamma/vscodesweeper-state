# Issue #225358 — Multiline selection in linux

[https://github.com/microsoft/vscode/issues/225358](https://github.com/microsoft/vscode/issues/225358)

## Summary

Keep open for editor multicursor triage: current main still has a Linux-specific default path that prevents middle-button column selection while the Linux selection clipboard is enabled. This is not enough for a close as fixed or as-designed without maintainer confirmation of the intended Linux tradeoff.

## Change summary

Middle-mouse drag column or multiline selection works on Windows but not on Ubuntu with the reporter's copied shortcuts.

## Best solution

Decide in the editor multicursor/selection code whether Linux middle-button drag should support column selection even when `editor.selectionClipboard` is enabled, or document the supported workaround of disabling `editor.selectionClipboard` or using Column Selection Mode.

## Evidence

- **current code:** In `src/vs/editor/browser/view/viewController.ts`, `dispatchMouse` computes `selectionClipboardIsOn = platform.isLinux && editor.selectionClipboard`; middle-button column selection only runs when that is false.
- **default setting:** `src/vs/editor/common/config/editorOptions.ts` registers `editor.selectionClipboard` with default `true` and `included: platform.isLinux`, so the reporter's Ubuntu default would bypass middle-button column selection.
- **related fix not sufficient:** Commit `6f5ee3a73fc9948ac37c8282064590a57e8e0d58` restored default middle-button column selection when selection clipboard is not active, but it preserved the Linux selection-clipboard exclusion.
- **discussion:** The only later comment reports Column Selection Mode as a partial workaround, but notes it does not auto-toggle with the mouse wheel button like Windows.
