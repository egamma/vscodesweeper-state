# Issue #203379 — Too much ghost text can cause scroll event that pushes cursor off screen

[https://github.com/microsoft/vscode/issues/203379](https://github.com/microsoft/vscode/issues/203379)

## Summary

Still appears reproducible on main: the `keepCursorStable` mechanism in `AdditionalLinesWidget` only adjusts scroll when ghost text view zones are added *above* the cursor (`lineNumber < cursorLineNumber`), but does not handle the case where large ghost text at the cursor line causes scroll that pushes the cursor off the visible viewport.

## Change summary

Large inline completion ghost text (multi-line) at the cursor position can cause a scroll event that moves the cursor off screen when the user continues editing without accepting.

## Best solution

Extend the scroll stability logic in `AdditionalLinesWidget.keepCursorStable` (or the editor's view zone handling) to ensure the cursor remains visible when large ghost text view zones are added/removed at the cursor line. Alternatively, the editor could reveal the cursor position after ghost text dismissal.
