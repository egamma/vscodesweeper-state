# Issue #194412 — Settings editor width overflows space in 2x2 grid

[https://github.com/microsoft/vscode/issues/194412](https://github.com/microsoft/vscode/issues/194412)

## Summary

Still reproduces on main: the settings editor's hard 500px minimum width causes its inner splitview to overflow when placed in a narrow grid cell. An open PR (#307246) exists but hasn't merged.

## Change summary

Settings editor's inner splitview overflows its bounding grid cell in a 2x2 layout because of its 500px minimum width constraint.

## Best solution

Review and land the open PR https://github.com/microsoft/vscode/pull/307246, or alternatively make the settings editor responsive below 500px (e.g., collapse the ToC earlier and allow horizontal scroll or a narrower body layout).
