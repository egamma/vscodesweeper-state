# Issue #52777 — [HTML] Auto tag-closing doesn't work with undo/redo and programmatic changes

[https://github.com/microsoft/vscode/issues/52777](https://github.com/microsoft/vscode/issues/52777)

## Summary

Team-authored issue. The undo/redo part was fixed (commit 801dff2f3b6, `TextDocumentChangeReason` API), but the programmatic-change part remains unresolved — `reason` is `undefined` for programmatic edits, so extensions still cannot distinguish them from user input.

## Change summary

HTML auto tag-closing fires on programmatic document changes (e.g. Live Share sync), duplicating closing tags for every participant.

## Best solution

Extend `TextDocumentChangeReason` (or add a new signal on `TextDocumentChangeEvent`) to distinguish programmatic/applied edits from direct user keyboard input, then guard auto-insertion in `extensions/html-language-features/client/src/autoInsertion.ts` accordingly.
