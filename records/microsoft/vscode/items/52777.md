# Issue #52777 — [HTML] Auto tag-closing doesn't work with undo/redo and programmatic changes

[https://github.com/microsoft/vscode/issues/52777](https://github.com/microsoft/vscode/issues/52777)

## Summary

Team-authored issue. The undo/redo part was fixed (TextDocumentChangeReason check on line 47 of autoInsertion.ts), but the programmatic-change detection (e.g. Live Share edits) remains unresolved — there is still no API to distinguish keyboard input from programmatic workspace edits, as noted in comment #16.

## Change summary

HTML auto tag-closing fires on undo/redo and programmatic (Live Share) document changes, producing duplicate closing tags.

## Best solution

Expose a new VS Code extension API (or extend TextDocumentChangeReason) that lets extensions distinguish user-keyboard edits from programmatic workspace edits, then use it in the auto-insertion logic. The undo/redo half is already handled; only the programmatic-change half remains.
