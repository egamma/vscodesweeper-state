# Issue #306023 — Profiles editor ignores workbench.editor.useModal="off" setting

[https://github.com/microsoft/vscode/issues/306023](https://github.com/microsoft/vscode/issues/306023)

## Summary

Fixed on main and shipped in 1.124.0 — the `editorGroupFinder` now respects `workbench.editor.useModal: 'off'` for editors with the `RequiresModal` capability, including the Profiles editor.

## Change summary

The Profiles editor always opens as a modal editor even when the user has set `workbench.editor.useModal` to `false`/`off`.

## Best solution

Keep the shipped fix (commit ea8e43b23fd, 1.124.0). The `RequiresModal` capability is now gated on the user's `useModal` setting — no further change needed.
