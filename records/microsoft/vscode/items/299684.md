# Issue #299684 — Insiders Sessions Window fails to create or destroy worktrees too often

[https://github.com/microsoft/vscode/issues/299684](https://github.com/microsoft/vscode/issues/299684)

## Summary

Team-authored issue (MEMBER) — keeping open. Significant worktree reliability work has landed since this was filed (timeout increases, clearer error messages, checkout.workers speedup, branch metadata fixes), but no single commit explicitly resolves the ~50% creation failure rate described here, and the reporter has not confirmed resolution.

## Change summary

Sessions Window worktree creation fails roughly half the time, and worktree deletion also fails frequently, with unhelpful error messages in both cases.

## Best solution

The error-messaging side has been addressed by https://github.com/microsoft/vscode/commit/d9eb8ee3a90 (clearer timeout errors) and https://github.com/microsoft/vscode/commit/bea4e89a0ca (increased timeout + checkout.workers). The remaining gap is the root cause of the high failure rate itself — likely disk-I/O contention or locked build artifacts in the worktree. The team should confirm whether the timeout/perf improvements have brought the failure rate to an acceptable level, and whether worktree deletion needs to handle locked files (e.g. node_modules) more gracefully. Relevant code lives in `src/vs/platform/agentHost/node/agentHostGitService.ts` and `src/vs/platform/agentHost/node/copilot/copilotAgent.ts`.
