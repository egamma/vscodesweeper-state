# Issue #112608 — Git - Allow configuring git to use current VS Code version for editing commits

[https://github.com/microsoft/vscode/issues/112608](https://github.com/microsoft/vscode/issues/112608)

## Summary

The core request — use the currently running VS Code version as the git editor for terminal commands — is implemented via the `git.terminalGitEditor` setting (shipped mid-2022). When enabled, `GIT_EDITOR` is set via `environmentVariableCollection` using `process.execPath`, so the current instance handles git editor requests regardless of stable/insiders.

## Change summary

Allow git commands in the integrated terminal to use the currently running VS Code instance (stable or insiders) as the editor, rather than hardcoding `code`.

## Best solution

The shipped `git.terminalGitEditor` setting already delivers this: it injects `GIT_EDITOR` into the terminal environment pointing at the running instance's process. No further change needed.
