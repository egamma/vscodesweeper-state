# Issue #240774 — Font renders badly on light background with Experimental GPU Acceleration enabled

[https://github.com/microsoft/vscode/issues/240774](https://github.com/microsoft/vscode/issues/240774)

## Summary

Still present on main: the GPU glyph rasterizer's `_clearColor` uses exact-match background removal and the subpixel antialiasing path doesn't handle light backgrounds well — confirmed by multiple users across macOS, Windows, and Linux, and the code still carries a TODO acknowledging incomplete font aliasing support.

## Change summary

Font text renders with visible artifacts (thin, washed-out, or garbled glyphs) on light themes when the experimental GPU acceleration is enabled in the editor.

## Best solution

Fix the glyph rasterizer's background-color clearing logic in `src/vs/editor/browser/gpu/raster/glyphRasterizer.ts` — the `_clearColor` exact-match approach loses antialiased edge pixels on light backgrounds. The existing TODO on line 47 ('Support workbench.fontAliasing correctly') acknowledges this gap. A threshold-based or alpha-premultiplication approach for background removal would likely resolve the light-theme rendering artifacts.
