# Issue #239577 — Profile "Default" not showing text "All profiles"

[https://github.com/microsoft/vscode/issues/239577](https://github.com/microsoft/vscode/issues/239577)

## Summary

Still reproduces on main: the 'All Profiles' label display inconsistency in the Default profile's extensions view is a real state-synchronization bug in the profile editor model's handling of `isApplicationScoped`. The code path in `extensionsResource.ts` correctly reads the flag, but the toggle and refresh cycle appears to have timing/state issues that cause the Default profile view to show stale data.

## Change summary

The Default profile's extensions view does not consistently show the 'All Profiles' label for application-scoped extensions, and toggling 'Apply Extension to all Profiles' doesn't always update the display correctly.

## Best solution

Fix the state synchronization in `UserDataProfileElement.getChildrenForResourceType` and the event handling in `AbstractUserDataProfileElement` constructor (lines 133-150 of `userDataProfilesEditorModel.ts`) to ensure the extensions tree is reliably refreshed after `toggleApplicationScope` completes — particularly for the Default profile where application-scoped extensions live.
