# Issue #232590 — Panel: no drop zone for views after the last panel

[https://github.com/microsoft/vscode/issues/232590](https://github.com/microsoft/vscode/issues/232590)

## Summary

Still unfixed on main. The panel's empty space after the last composite bar item is consumed by the actions container (flex-grow), not covered by a drop target — unlike the sidebar where the activity bar element spans the full empty area. Filed by a contributor with a clear reproduction, acknowledged by a team member.

## Change summary

Dragging a view container to the end of the panel is difficult because the empty space after the last tab doesn't act as a drop zone.

## Best solution

Extend the drop target registration in the panel's composite bar area to cover the flex-grown actions container, or add a secondary drop target on the empty space element. The fix likely lives in src/vs/workbench/browser/parts/compositeBar.ts or the panel-specific composite bar setup in src/vs/workbench/browser/parts/paneCompositeBar.ts.
