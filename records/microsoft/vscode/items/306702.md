# Issue #306702 — Updating extension to newly published version picks the universal VSIX instead of platform one

[https://github.com/microsoft/vscode/issues/306702](https://github.com/microsoft/vscode/issues/306702)

## Summary

Real bug with 'On Deck' milestone — must stay open per scheduling rules. The platform-preference logic in filterLatestExtensionVersionsForTargetPlatform only replaces universal with platform-specific when version numbers match, but the update flow may encounter versions in an order where this replacement doesn't trigger.

## Change summary

When updating an extension that newly publishes both universal and platform-specific VSIXs, VS Code picks the universal VSIX instead of the platform-specific one.

## Best solution

Fix the version resolution in the gallery service update path (likely in `extensionGalleryService.ts`) to always prefer platform-specific versions over universal ones during updates, not just during fresh installs. The `sortExtensionVersions` and `filterLatestExtensionVersionsForTargetPlatform` functions need to ensure the platform-specific VSIX wins regardless of publishing order.
