## Change summary

Background agent session results are hard to apply, push, or turn into PRs when the local workspace has uncommitted changes.

## Best solution

Improve the apply/push/PR flow for background session worktrees: allow applying changes even when local workspace is dirty (via merge/cherry-pick semantics), surface worktree branches in the SCM view, and streamline PR creation from background sessions. The work likely lives in the chat session actions and SCM integration layers.
