# Issue #314020 — chat plugins: Claude-format plugin frontmatter (rules `paths`, agents `tools`/`model`) is not transformed when installed via marketplace or pluginLocations

[https://github.com/microsoft/vscode/issues/314020](https://github.com/microsoft/vscode/issues/314020)

## Summary

Current main still appears to have the reported gap: Claude plugin format is detected during plugin discovery, but prompt-file parsing still infers Claude behavior from `.claude` paths. Route to the chat area; the linked PR is relevant but still open, so this is not completed yet.

## Change summary

Claude-format plugins installed through marketplaces or `chat.pluginLocations` lose Claude-specific `rules.paths`, agent `tools`, and agent `model` frontmatter transformations because parsing falls back to path-based/Copilot semantics.

## Best solution

Propagate the detected plugin format from agent plugin discovery into plugin prompt paths, then treat plugin-rooted Claude rules and agents as `Target.Claude` in prompt discovery and parsing. The open PR https://github.com/microsoft/vscode/pull/318877 looks like the likely place to land that fix.

- **agent parsing path:** `src/vs/workbench/contrib/chat/common/promptSyntax/languageProviders/promptFileAttributes.ts` returns `Target.Claude` for agents only when the parent path ends with the Claude agents folder; plugin-installed paths do not satisfy that condition, preventing `mapClaudeModels()` and `mapClaudeTools()` from running in `promptsServiceImpl.ts`.
