# Issue #314020 ‚Äî chat plugins: Claude-format plugin frontmatter (rules `paths`, agents `tools`/`model`) is not transformed when installed via marketplace or pluginLocations

[https://github.com/microsoft/vscode/issues/314020](https://github.com/microsoft/vscode/issues/314020)

## Summary

The bug is real and still present on main: `isInClaudeRulesFolder()` checks for the literal `.claude/rules/` path substring, and `getTarget()` checks for `/.claude/agents` ‚Äî neither matches marketplace-installed plugin paths. The detected `PluginFormat` is never propagated to the prompt-file parser.

## Change summary

Claude-format plugins installed via marketplace or pluginLocations have their `paths` frontmatter (rules) and `tools`/`model` frontmatter (agents) silently dropped because the parser uses path-based detection that doesn't match plugin install paths.

## Best solution

Carry the `PluginFormat` (already detected by `detectPluginFormat()`) on `IPluginPromptPath`, then amend `isInClaudeRulesFolder()` checks in `promptsServiceImpl.ts` (line 1377) and `getTarget()` in `promptFileAttributes.ts` (line 415) to also return Claude semantics when `pluginFormat === PluginFormat.Claude`.
