# Issue #310884 — Each time the remote server window is opened, the agent session becomes unread.

[https://github.com/microsoft/vscode/issues/310884](https://github.com/microsoft/vscode/issues/310884)

## Summary

The core scenario — sessions becoming unread on window reopen due to remote host disconnect/reconnect — was fixed in commit e84ab99125e and shipped in 1.123.0. The read set itself is persisted in profile storage, and the fix prevents the session provider from re-announcing cached sessions as new additions after a reconnect.

## Change summary

Agent sessions become unread every time a remote server window (or a window without a workspace) is reopened.

## Best solution

Keep the shipped fix (1.123.0). If the reporter still sees this on the latest build, a new issue with more specific reproduction steps would be appropriate.
