# Issue #215462 — Tree Sticky Scroll does not support Touch

[https://github.com/microsoft/vscode/issues/215462](https://github.com/microsoft/vscode/issues/215462)

## Summary

Still present on main: the StickyScrollFocus class only handles mouse events (onMouseDown) and never registers touch support (Gesture.addTarget or onTouchStart/onTap listeners) on the sticky scroll container, so touch input is ignored on sticky scroll elements.

## Change summary

Tree sticky scroll elements in the Explorer ignore touchscreen taps, preventing folder expand/collapse via touch.

## Best solution

Register `Gesture.addTarget` on the sticky scroll container DOM node and add an `onTap`/`onTouchStart` listener in `StickyScrollFocus` (analogous to the existing `onMouseDown` handler) so touch events trigger the same expand/collapse behavior. The fix is localized to `src/vs/base/browser/ui/tree/abstractTree.ts` in the `StickyScrollFocus` and `StickyScrollWidget` classes.
