# Issue #284196 — Implementing a plan leads to unhelpful session title

[https://github.com/microsoft/vscode/issues/284196](https://github.com/microsoft/vscode/issues/284196)

## Summary

The reported problem — session titles being set to the literal user prompt (e.g. 'Start implementation') — has been comprehensively addressed on main by AI-generated session titles and first-turn context refinement.

## Change summary

When handing off a plan to a background CLI agent with a generic prompt like 'Start implementation', the session title was set to that unhelpful prompt text instead of reflecting the plan's purpose.

## Best solution

Keep the shipped AI-generated title pipeline (commits c3f4c30 and d689633); no further change needed. The title is now seeded from the first message, then refined by a utility model using the full first-turn context (user request + agent response), producing a meaningful title even when the user prompt is generic.
