# Issue #205142 — usability: callstack UI should not scroll

[https://github.com/microsoft/vscode/issues/205142](https://github.com/microsoft/vscode/issues/205142)

## Summary

Valid UX improvement request for the debug callstack view with very long frame names (e.g. C++ templates). The layout has not been changed on main — filename/line-number elements are still pushed off-screen by long labels, and no sticky/pinned positioning has been added. A team member acknowledged the problem but noted lists aren't optimized for horizontal scrolling.

## Change summary

Debug callstack view becomes unusable with long frame names: filenames, line numbers, and 'Load More' are pushed off-screen or hidden.

## Best solution

Pin the file/line-number column to the right edge of the viewport (CSS `position: sticky` or an overlay approach) and center 'Load More Stack Frames' relative to the view, not the scrollable content. The fix lives in `src/vs/workbench/contrib/debug/browser/callStackView.ts` (template rendering) and `src/vs/workbench/contrib/debug/browser/media/debugViewlet.css` (layout). This is a polish/layout change scoped to the debug callstack, not a broader list infrastructure change.
