# Issue #169002 — Editor triggers keyboard open on scroll on mobile

[https://github.com/microsoft/vscode/issues/169002](https://github.com/microsoft/vscode/issues/169002)

## Summary

Keep open as a valid editor touch bug: a VS Code member confirmed it on vscode.dev, and current main still has the touch tap path that focuses the editor textarea. I found a candidate fix commit, but it is only on `origin/copilot/fix-virtual-keyboard-issue`, not contained in `main`, so this is not completed on main.

## Change summary

On Android/mobile browsers, scrolling the editor can be interpreted as a tap and open the virtual keyboard.

## Best solution

Suppress tap/contextmenu dispatch after a touchmove scroll before the editor calls `focusTextArea()`, likely in `src/vs/base/browser/touch.ts` with the editor touch path in `src/vs/editor/browser/controller/pointerHandler.ts` as the consumer.
