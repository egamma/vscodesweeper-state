# Issue #169002 — Editor triggers keyboard open on scroll on mobile

[https://github.com/microsoft/vscode/issues/169002](https://github.com/microsoft/vscode/issues/169002)

## Summary

Still reproduces on main: the touch gesture handler in `src/vs/base/browser/touch.ts` dispatches a Tap event (which triggers editor focus and keyboard open) when touch movement is <30px, without checking whether scroll (Change) events were already dispatched during the gesture. A fix PR (#292179) was opened but closed without merging.

## Change summary

Scrolling on the editor on mobile Android triggers a Tap event that focuses the textarea and opens the virtual keyboard.

## Best solution

Track whether a Change (scroll) event was dispatched during the current touch gesture in `src/vs/base/browser/touch.ts` and suppress the Tap dispatch if so. The closed PR #292179 had the right approach — a boolean flag set on Change dispatch, checked before Tap dispatch.
