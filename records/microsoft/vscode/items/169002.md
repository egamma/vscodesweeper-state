## Change summary

Scrolling a large editor on mobile/touch devices can focus the editor and reopen the virtual keyboard.

## Best solution

Keep this routed to the editor/touch area for investigation in Monaco/editor pointer handling; the desired end state is that touch scroll gestures do not focus the editor or show the virtual keyboard unless the user explicitly taps to place the cursor.
