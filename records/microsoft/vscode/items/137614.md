# Issue #137614 — Multiple quiet login requests (different scopes) from an extension to a single AuthenticationProvider are indistinguishable on the Accounts menu

[https://github.com/microsoft/vscode/issues/137614](https://github.com/microsoft/vscode/issues/137614)

## Summary

The gap remains on current main: quiet sign-in menu items still show only provider label and extension name with no scope/hint differentiation. The `AuthenticationGetSessionPresentationOptions.detail` field only applies to modal dialogs, not to the accounts-menu entries created for `createIfNone: false`. Keeping open as the canonical tracker per TylerLeonhardt's comment.

## Change summary

Multiple quiet login requests (createIfNone: false) from the same extension to the same auth provider produce identical, indistinguishable menu items in the Accounts menu.

## Best solution

Extend the quiet sign-in menu item creation in `authenticationExtensionsService.ts` (around line 471) to incorporate either the requested scopes or an extension-supplied hint string into the menu label, so users can distinguish between multiple pending sign-in requests. The `AuthenticationGetSessionPresentationOptions.detail` mechanism could be reused or extended for the quiet path.
