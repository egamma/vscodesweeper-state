# Issue #282541 — zsh themes that use `RPROMPT` seem to break suggest feature of integrated terminal

[https://github.com/microsoft/vscode/issues/282541](https://github.com/microsoft/vscode/issues/282541)

## Summary

Still reproduces on main: the prompt input model reads from commandStartX to end-of-line, including RPROMPT text, because it never consults `commandRightPromptStartX` to truncate. This causes terminal suggest to receive garbled input.

## Change summary

zsh themes with RPROMPT cause terminal suggest to include the right-prompt text in the parsed command input, breaking suggestions.

## Best solution

In `promptInputModel.ts`'s `_doSync`, when building `commandLine` from the buffer line, truncate at `commandRightPromptStartX` (already tracked by the command detection capability) so RPROMPT content is excluded from the prompt input value fed to terminal suggest.
