# Issue #327250 — Support boolean AND/OR/NOT search in the Search view

[https://github.com/microsoft/vscode/issues/327250](https://github.com/microsoft/vscode/issues/327250)

## Summary

Legitimate feature request for boolean AND/OR/NOT search operators in the Search view. Regex partially covers OR (alternation) and same-line AND (lookaheads), but cross-file AND (file must contain both terms anywhere) has no built-in equivalent. No exact duplicate found; leaving for the community upvote process.

## Change summary

Add boolean AND/OR/NOT operators to the Search view so users can find files containing multiple independent terms without resorting to regex.

## Best solution

If implemented, this would likely live in `src/vs/workbench/contrib/search/` as a query parser that decomposes boolean expressions into multiple ripgrep passes and intersects/unions the file results. Alternatively, the existing regex toggle already enables OR via `term1|term2`; a proposed comment could point users at that workaround while the request sits in Backlog Candidates.
