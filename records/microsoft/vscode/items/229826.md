## Change summary

Add an option to display SCM Graph commits in strict date order (date-order, author-date-order, topo-order) instead of only the default topological order.

## Best solution

Expose a setting or view menu toggle in the SCM History view pane (likely in `src/vs/workbench/contrib/scm/browser/scmHistoryViewPane.ts` and the history provider API) to pass `--date-order` / `--author-date-order` / `--topo-order` through to git log, similar to the existing repository sort-order options.
