# Issue #326365 — Expensive fetch in Copilot code search for repo tracking

[https://github.com/microsoft/vscode/issues/326365](https://github.com/microsoft/vscode/issues/326365)

## Summary

Valid performance bug: the `getGithubRemoteFromSshConfig` fallback in `repoTracker.ts` runs `git fetch --dry-run` which performs full network operations (SSH handshake, object negotiation) on large repos. The code is still present on current main. The reporter's suggestion of `git ls-remote` is a reasonable alternative. Related to https://github.com/microsoft/vscode/issues/313471 (SSH key prompts from the same code path) but not a duplicate — that issue tracks the UX prompt, this one tracks the performance cost.

## Change summary

Copilot code search's repo tracker performs an expensive `git fetch --dry-run` to discover GitHub remotes from SSH config, causing significant network overhead on large repos.

## Best solution

Replace the `git fetch --dry-run` call in `getGithubRemoteFromSshConfig` (extensions/copilot/src/platform/workspaceChunkSearch/node/codeSearch/repoTracker.ts:386) with `git ls-remote --get-url` or parse SSH config directly to avoid full fetch negotiation. Consider also fixing https://github.com/microsoft/vscode/issues/313471 in the same change since both stem from this code path.
