# Issue #285141 — Copilot chat always defaults to Agent mode (and doesn't remember the last selected mode)

[https://github.com/microsoft/vscode/issues/285141](https://github.com/microsoft/vscode/issues/285141)

## Summary

Both aspects of this issue — remembering the last-used chat mode across restarts and providing a setting to configure the default — are now implemented on main. The `chat.newSession.defaultMode` setting lets users pin a default mode (e.g. "Ask"), and commit 87b6a1aef52 adds full session-type preference persistence via storage.

## Change summary

Copilot chat always reverts to Agent mode on restart instead of remembering the user's last-selected mode (e.g. Ask).

## Best solution

Keep the shipped implementation: `chat.newSession.defaultMode` setting (introduced 74fcb3b75e4) plus the session-type remembering logic in `chatSessionTypePreference.ts` (introduced 87b6a1aef52). No further change needed — once these ship in the next stable release, both the "remember last mode" and "configure a default" requests are satisfied.
