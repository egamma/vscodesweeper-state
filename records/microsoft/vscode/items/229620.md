# Issue #229620 — Focus is lost when it enters an image preview

[https://github.com/microsoft/vscode/issues/229620](https://github.com/microsoft/vscode/issues/229620)

## Summary

Team-authored accessibility bug about focus being lost when previewing an attached image in chat. No evidence of a fix on main — the image carousel service (`chatImageCarouselService.ts`) has no focus trapping or restoration logic. Keeping open per team-author rule.

## Change summary

When an attached image is previewed via keyboard in the chat panel, focus escapes to the start of the window instead of returning to the chat widget.

## Best solution

The image carousel/preview overlay needs focus trapping while open and focus restoration to the originating element on close. Likely files are `chatImageCarouselService.ts` and possibly `chatAttachmentsContentPart.ts`. This is an accessibility gap in the chat image preview flow.
