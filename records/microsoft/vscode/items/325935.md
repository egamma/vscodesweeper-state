# Issue #325935 — Disable .copilot folder creation

[https://github.com/microsoft/vscode/issues/325935](https://github.com/microsoft/vscode/issues/325935)

## Summary

Valid feature request: when all AI features are disabled (`chat.disableAIFeatures: true`), VS Code should not spawn the agent host process or create the `~/.copilot` directory. Currently `chat.agentHost.enabled` (which defaults to true on Insiders) independently gates the agent host spawn, and nothing connects `chat.disableAIFeatures` to that gate.

## Change summary

Prevent the `~/.copilot` directory from being created when the user has disabled all AI features.

## Best solution

Gate the agent host spawn (in `isAgentHostEnabled` at `src/vs/platform/agentHost/common/agentService.ts:49`) on `chat.disableAIFeatures` being false, so that setting `chat.disableAIFeatures: true` prevents the Copilot CLI process from starting and creating `~/.copilot`. The user should also be able to set `chat.agentHost.enabled: false` as a direct override. Area: `agent-host` or the chat configuration surface.

## Evidence

- **agent host default:** `chat.agentHost.enabled` defaults to `true` on non-stable builds (src/vs/platform/agentHost/common/agentHost.config.contribution.ts:40), so the Copilot CLI is spawned on Insiders regardless of other AI settings.
- **directory origin:** The `~/.copilot` directory is created by the Copilot CLI process itself (src/vs/platform/agentHost/node/copilot/copilotAgentSession.ts:93 defines `COPILOT_HOME_DIRECTORY = '.copilot'`); the CLI writes `config.json` and logs there on startup.
- **no disableAIFeatures gate:** `isAgentHostEnabled` (src/vs/platform/agentHost/common/agentService.ts:49-50) only checks `chat.agentHost.enabled` and `isWeb` — it does not check `chat.disableAIFeatures`, so disabling AI features does not prevent the agent host from spawning.
- **workaround:** Setting `"chat.agentHost.enabled": false` should prevent the agent host from spawning and the directory from being created, but this is not discoverable from the `chat.disableAIFeatures` setting.
