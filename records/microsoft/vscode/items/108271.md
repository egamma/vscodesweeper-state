# Issue #108271 — Add an explorer/title menu contribution point

[https://github.com/microsoft/vscode/issues/108271](https://github.com/microsoft/vscode/issues/108271)

## Summary

Still unimplemented on main. The `viewContainer/title` contribution point exists as a proposed API but is currently restricted to the debug view container only — no extension can contribute to the Explorer's title bar. The issue carries the 'On Deck' milestone, which is a scheduling decision that must be respected.

## Change summary

Allow extensions to contribute actions to the Explorer view container's title bar menu.

## Best solution

Finalize the proposed `viewContainer/title` contribution point (`contribViewContainerTitle`) and remove the debug-only restriction so extensions can target any view container including the Explorer. The validation in `src/vs/workbench/services/actions/common/menusExtensionPoint.ts` currently gates `viewContainer/title` to `viewContainer == workbench.view.debug` — broadening this would address the request. The overflow/ellipsis pattern from SCM (as commenters suggested) would manage clutter.
