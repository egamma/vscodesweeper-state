# Issue #167921 — Debug attach - launch.json pickProcess command launches before preLaunchTask

[https://github.com/microsoft/vscode/issues/167921](https://github.com/microsoft/vscode/issues/167921)

## Summary

Still valid on current main: `substituteVariables` (line 511 of debugService.ts) resolves command variables like `${command:pickProcess}` before `preLaunchTask` runs (line 547). The ordering gap remains exactly as described, with 10 upvotes and continued community interest. Keeping open in Backlog.

## Change summary

When a debug attach configuration uses `${command:pickProcess}` and a `preLaunchTask`, the process picker appears before the task runs, preventing users from picking a process that the task is supposed to start.

## Best solution

Reorder the flow in `src/vs/workbench/contrib/debug/browser/debugService.ts` so that `preLaunchTask` runs before `substituteVariables`. As roblourens noted, variables used *inside* the task itself may need early resolution — a two-pass approach (resolve task variables → run task → resolve remaining variables) would handle both cases. This is a core debug contribution change, not extension territory.
