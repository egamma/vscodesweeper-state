# Issue #201650 — Issue Reporter doesn't show me any errors when submitting doesn't work

[https://github.com/microsoft/vscode/issues/201650](https://github.com/microsoft/vscode/issues/201650)

## Summary

Still reproduces on main: when the GitHub API rejects an issue creation (e.g. 403 from being blocked), `submitToGitHub` silently logs to console and returns `false` — no user-visible error is ever shown. The related issue #203522 was closed as a duplicate of this one, confirming this is the canonical tracker.

## Change summary

The Issue Reporter silently fails with no user-facing error message when the GitHub API rejects the submission (e.g. user is blocked from the org).

## Best solution

In `baseIssueReporterService.ts`, surface the GitHub API error to the user — show a notification or inline error banner when `submitToGitHub` gets a non-200 response, ideally including the HTTP status or GitHub error message. The fix is localized to `submitToGitHub()` and its callers in the same file.
