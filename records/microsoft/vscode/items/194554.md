# Issue #194554 — Forced mouse selection in terminal cleared after mouse movement under 1003 (all events) and 1006 (SGR) combination

[https://github.com/microsoft/vscode/issues/194554](https://github.com/microsoft/vscode/issues/194554)

## Summary

Confirmed upstream xterm.js bug — forced mouse selection is cleared when SGR mode 1006 is active because xterm.js doesn't properly suppress mouse reports during forced selection in that mode. The VS Code terminal passes `macOptionClickForcesSelection` directly to xterm.js; the fix belongs there. Issue is correctly labeled and linked upstream.

## Change summary

Forced mouse selection in the integrated terminal is immediately cleared by any mouse movement when both mode 1003 (all events) and mode 1006 (SGR extended) are active.

## Best solution

Wait for the upstream fix in xterm.js (tracked at https://github.com/xtermjs/xterm.js/issues/4297) and pick it up in a future xterm.js dependency bump. The `macOptionClickForcesSelection` option is passed through directly; no VS Code-side workaround is practical.
