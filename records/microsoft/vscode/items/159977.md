# Issue #159977 — Keybinding doesn't work.

[https://github.com/microsoft/vscode/issues/159977](https://github.com/microsoft/vscode/issues/159977)

## Summary

Still valid on main: `deleteWordLeft` is an editor command (precondition: `EditorContextKeys.writable`) that doesn't fire in the Command Palette's native HTML input. `alt+backspace` only works there because macOS natively handles Option+Backspace as word-delete. The `help wanted` label is appropriate.

## Change summary

Editor word-deletion commands (like `deleteWordLeft`) don't work in the Command Palette's input box when bound to non-native key combos, because the quick input uses a plain HTML input where `textInputFocus` is false.

## Best solution

Either make the quick input's native input respond to `deleteWordLeft`/`deleteWordRight` commands (e.g., by handling them in `quickInputActions.ts`), or switch the quick input to use a Monaco editor instance so editor keybindings apply uniformly. The `help wanted` label is correctly placed for a community contributor to pick this up.
