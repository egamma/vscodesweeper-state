# Issue #273286 — Git - Optionally use git commits with notes for agent checkpoint

[https://github.com/microsoft/vscode/issues/273286](https://github.com/microsoft/vscode/issues/273286)

## Summary

Legitimate feature request to surface agent checkpoints as real git commits with git notes instead of hidden refs. The current implementation in `agentHostCheckpointService` deliberately uses hidden `refs/agents/` refs that never appear to the user. Low community traction (2 upvotes, no comments) — keeping open in Backlog for the community process.

## Change summary

Option to have agent checkpoints create real git commits with attached git notes instead of hidden internal refs, making AI-assisted changes visible in standard git history.

## Best solution

This is a product-design decision for the agent-host team: the current hidden-ref approach (`refs/agents/<sid>/checkpoints/turn/<N>` in `src/vs/platform/agentHost/node/agentHostCheckpointService.ts`) was an intentional choice to keep checkpoints invisible. Adding an opt-in mode to create real commits with git notes would require rethinking the checkpoint lifecycle (cleanup, squashing, push behavior). Best evaluated alongside broader agent-session UX decisions; if implemented, it would likely be a setting gating the checkpoint strategy.
