# Issue #141815 — Highlight current word in json sometimes matches a single double quote

[https://github.com/microsoft/vscode/issues/141815](https://github.com/microsoft/vscode/issues/141815)

## Summary

Still reproduces on main: the windowSize=15 algorithm in getWordAtText can match just a trailing quote when positioned after a long JSON string value. The root cause diagnosed by aeschli in 2022 remains unfixed — the word-matching window logic in wordHelper.ts is unchanged.

## Change summary

Word highlighting in JSON files incorrectly highlights all double quotes as the 'current word' when the cursor is after a long string value's closing quote.

## Best solution

Fix the word-matching window logic in `src/vs/editor/common/core/wordHelper.ts` so that when the backward window lands mid-token (e.g. on an isolated quote that's actually part of a longer construct), it either expands the window or declines to match a degenerate single-separator result.
