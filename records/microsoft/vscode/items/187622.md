# Issue #187622 — Default log level does not work in vscode.dev

[https://github.com/microsoft/vscode/issues/187622](https://github.com/microsoft/vscode/issues/187622)

## Summary

Still reproduces on main: the DefaultLogLevelsService stores the user-set default in argv.json and displays it correctly, but on startup/reload it never reads argv.json to apply the stored default as the active log level — the active level resets to Info.

## Change summary

Setting default log level in vscode.dev persists and displays correctly, but after reload the active log level reverts to Info instead of honoring the stored default.

## Best solution

In `src/vs/workbench/services/log/common/defaultLogLevels.ts`, the constructor should proactively read argv.json (via `_parseLogLevelsFromArgv`) and apply the stored default to the logger service's active level, similar to what `setDefaultLogLevel` does after a user action. An initialization path or a workbench contribution that reads the stored default and calls `loggerService.setLogLevel()` on startup would close the gap.
