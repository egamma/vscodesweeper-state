# Issue #54248 — [json] package.json: complete package versions from scopes or private registries

[https://github.com/microsoft/vscode/issues/54248](https://github.com/microsoft/vscode/issues/54248)

## Summary

Partially addressed — version completion now uses `npm view` CLI (respects private registries) for the latest version, but does not offer all available versions, and name completion for private/scoped registries still hardcodes npmjs.org. The feature request remains valid for full version listing.

## Change summary

Request to autocomplete package versions (not just names) from scoped or private npm registries using `npm view`.

## Best solution

Extend `collectValueSuggestions` in `extensions/npm/src/features/packageJSONContribution.ts` to fetch and offer all available versions (via `npm view <pkg> versions`) rather than only the latest, leveraging the existing `runNpmCommand` infrastructure that already respects `.npmrc`.
