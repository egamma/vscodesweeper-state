# Issue #54248 — [json] package.json: complete package versions from scopes or private registries

[https://github.com/microsoft/vscode/issues/54248](https://github.com/microsoft/vscode/issues/54248)

## Summary

The core request — version autocompletion for packages on private registries and scoped packages — is implemented on main. The npm extension now uses `npm view` (which respects .npmrc) as the primary path for fetching version info, falling back to the public registry only when npm CLI is unavailable.

## Change summary

Request to autocomplete package versions for scoped packages and packages hosted on private registries in package.json.

## Best solution

Keep the shipped behavior: `fetchPackageInfo` uses `npm view` which inherits the user's npm/npmrc configuration (including private registry mappings). No further change needed for version completion. If name-search against private registries is also desired, that would be a separate issue.
