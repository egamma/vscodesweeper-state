# Issue #283560 — "Chat took too long to get ready" - Remote setups (WSL, Dev Containers, Remote SSH)

[https://github.com/microsoft/vscode/issues/283560](https://github.com/microsoft/vscode/issues/283560)

## Summary

Active tracking issue for proxy misconfiguration in remote setups (WSL, Dev Containers, SSH) causing Copilot Chat timeouts. Team members confirmed the root cause and designated this as the canonical tracking issue — keep open.

## Change summary

Copilot Chat times out in remote environments (WSL, Dev Containers) because the Windows-side proxy listening on 127.0.0.1 is unreachable from the Linux/container side, and VS Code's local proxy configuration is not properly bridged.

## Best solution

Improve proxy resolution for remote environments so that `http.useLocalProxyConfiguration` correctly handles the localhost address mismatch between Windows host and WSL/container. chrmarti is actively working on proxy improvements in `src/vs/workbench/api/node/proxyResolver.ts` (recent commits include proxy type recording and telemetry). The workaround of disabling `HTTP > Use Local Proxy Configuration` in remote settings or using WSL `networkingMode=mirrored` should be documented more prominently until a proper fix lands.
