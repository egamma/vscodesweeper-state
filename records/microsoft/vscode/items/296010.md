# Issue #296010 — Git - detecting filepath+line link in console diff not working for all formats

[https://github.com/microsoft/vscode/issues/296010](https://github.com/microsoft/vscode/issues/296010)

## Summary

Valid bug: the git diff hunk link regex in the terminal multi-line link detector requires `,COUNT` after the line number (`+LINE,COUNT`), so it doesn't match the column-less format (`+LINE`) that git produces for single-line hunks or with `--unified=0`. Still present on main.

## Change summary

Terminal link detection for git diff `@@` hunk headers fails when the hunk format omits the count (e.g. `@@ -8 +8 @@` instead of `@@ -8,1 +8,1 @@`).

## Best solution

Make `toFileCount` optional in the `gitDiffMatchers` regex in `src/vs/workbench/contrib/terminalContrib/links/browser/terminalMultiLineLinkDetector.ts` (line 43) — change the regex to `/^(?<link>@@ .+ \+(?<toFileLine>\d+)(?:,(?<toFileCount>\d+))? @@)/` and default `toFileCount` to `"1"` when absent.
