# Issue #231927 — Tools declaration still supports `toolReferenceName`

[https://github.com/microsoft/vscode/issues/231927](https://github.com/microsoft/vscode/issues/231927)

## Summary

The API debt is still present on current main: `toolReferenceName` remains a standalone property in the tool contribution schema and `IToolData`, separate from `canBeReferencedInPrompt`. The team discussed consolidation options but deferred. Author is a contributor, so keeping open for maintainer judgment.

## Change summary

The `toolReferenceName` property in the tools package.json contribution point should be removed or merged with the `canBeReferencedInPrompt` property to reduce API surface confusion.

## Best solution

Consolidate `toolReferenceName` into `canBeReferencedInPrompt` (e.g. `canBeReferencedInPrompt?: boolean | string` per jrieken's suggestion, or a nested object per roblourens's alternative). The work touches `src/vs/workbench/contrib/chat/common/tools/languageModelToolsContribution.ts` (schema + `IRawToolContribution`), `languageModelToolsService.ts` (`IToolData`), the ext-host protocol files, and consumers that read `toolReferenceName`. Commit 9102c10c7f1 cleaned up the *tool-set*-level `toolReferenceName` but left the tool-level property untouched.
