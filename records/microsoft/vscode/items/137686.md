# Issue #137686 — Git - Sync Changes button pushes to different branch than current branch

[https://github.com/microsoft/vscode/issues/137686](https://github.com/microsoft/vscode/issues/137686)

## Summary

Confirmed on current main: the sync and push commands hardcode the push target to the upstream tracking branch (`head.upstream.remote`/`head.upstream.name`), ignoring `push.default`, `remote.pushDefault`, and `branch.<name>.pushRemote` git config settings. An open PR (https://github.com/microsoft/vscode/pull/323650) is already addressing this.

## Change summary

The Git 'Sync Changes' and 'Push' actions ignore `push.default` and `pushRemote` git configuration, always pushing to the upstream tracking branch rather than the configured push target.

## Best solution

Land or iterate on the open PR https://github.com/microsoft/vscode/pull/323650 which adds push-target resolution. The fix needs to touch `_sync()` and `push()` in `extensions/git/src/repository.ts` (lines ~2355–2400) to consult `push.default`, `remote.pushDefault`, and `branch.<name>.pushRemote` before computing the push refspec, and update `syncTooltip` (line ~3257) to display the correct push destination.
