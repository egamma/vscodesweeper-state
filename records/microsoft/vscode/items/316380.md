# Issue #316380 — Unable to Permanently Delete Archived AI Agent Conversations in VS Code Chat

[https://github.com/microsoft/vscode/issues/316380](https://github.com/microsoft/vscode/issues/316380)

## Summary

The inability to delete chat sessions is already tracked by https://github.com/microsoft/vscode/issues/307452 (same core problem: delete doesn't work or isn't available). Delete functionality exists in the codebase (`DeleteAgentSessionAction` via right-click context menu) but has known gaps across session types; multiple open issues track specific facets (#316929 intermittent deletion, #325673 Claude sessions resurrecting). This issue adds no distinct requirement beyond what #307452 already captures.

## Change summary

Users cannot permanently delete archived AI agent conversations in VS Code Chat; only archive/unarchive is available, with no visible delete option for some session types.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/307452 which covers the same inability to delete chat sessions. The delete action exists in `src/vs/workbench/contrib/chat/browser/agentSessions/agentSessionsActions.ts` (`DeleteAgentSessionAction`) but its `when` clause limits visibility to local and agent-host session types — extending coverage and fixing reliability is the fix for both issues.
