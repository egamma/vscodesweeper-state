## Summary

Team-authored accessibility bug about voice-command interaction with expand/collapse controls requiring two activations instead of one. Kept open per hard keep-open rule (MEMBER author) and because it's a valid accessibility concern.

## Change summary

Expand/Collapse controls require two voice commands (show numbers + click) to toggle state, instead of responding to a single command like mouse click does.

## Best solution

Investigate why the expand/collapse control in the extensions view requires a double activation via voice command — likely the first activation moves focus/selects the control and only the second toggles it. The fix would ensure the click handler fires fully on the first programmatic activation, matching mouse-click behavior. Likely in the tree widget or the extensions view's checkbox/twistie handling.
