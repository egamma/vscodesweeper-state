# Issue #159749 — Workspace Trust - The redundancy settings in UI State sync file

[https://github.com/microsoft/vscode/issues/159749](https://github.com/microsoft/vscode/issues/159749)

## Summary

Fixed on main and shipped in 1.87.0. The workspace trust status bar entry was changed from a per-workspace ID to a single static ID, and the lifecycle was reworked to add/remove the entry instead of hide/show — preventing future accumulation of redundant entries in globalState.

## Change summary

Workspace trust created a separate status bar entry per workspace, causing redundant entries to accumulate in the UI state sync file (globalState.json) that could not be cleaned up.

## Best solution

Keep the shipped fix from commit 1b58747581e (1.87.0). The root cause — per-workspace status bar entry IDs — is resolved. Old orphaned entries in existing user profiles are harmless and will be ignored since nothing references them.
