# Issue #306717 — Chat: add 'tiers' frontmatter and 'tier' parameter to runSubagent for semantic model tier routing

[https://github.com/microsoft/vscode/issues/306717](https://github.com/microsoft/vscode/issues/306717)

## Summary

This is a well-articulated feature request for semantic model tier routing in subagent invocations. The companion `model` parameter (explicit override) has shipped, but the semantic `tiers` abstraction layer proposed here has not been implemented on main. Keeping open as an unresolved feature request.

## Change summary

Allow agent frontmatter to declare named model tiers (quick/capable/thorough) so orchestrators can request capability levels without hardcoding model IDs.

## Best solution

If the team wants semantic tier routing, implement `tiers` parsing in the agent file loader and add a `tier` parameter to `RunSubagentTool.invoke()` slotting into the existing `resolveSubagentModel()` resolution chain. Alternatively, this could remain an extension-level concern built atop the already-shipped explicit `model` parameter. Needs product decision on whether VS Code core should own tier semantics.
