# Issue #234353 — Terminal suggest: Support resolving environment variables in directories

[https://github.com/microsoft/vscode/issues/234353](https://github.com/microsoft/vscode/issues/234353)

## Summary

Feature not yet implemented: terminal suggest does not resolve environment variables (like $HOME) in typed paths to provide directory completions. The code has _getEnvVar() and CDPATH support but no logic to expand $VAR prefixes in the user's input for path completion.

## Change summary

Terminal suggest should resolve environment variables like $HOME in directory paths to show completions from the resolved directory.

## Best solution

Implement env-var expansion in `resolveResources` within `src/vs/workbench/contrib/terminalContrib/suggest/browser/terminalCompletionService.ts`: detect `$VAR` or `${VAR}` patterns in `lastWordFolder`, resolve them via the existing `_getEnvVar` helper, and use the resolved path for directory listing and detail display.
