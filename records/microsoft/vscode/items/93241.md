# Issue #93241 — Should we use namespacing in the CLI

[https://github.com/microsoft/vscode/issues/93241](https://github.com/microsoft/vscode/issues/93241)

## Summary

The subcommand architecture has been adopted for newer CLI features (chat, tunnel, agent, serve-web), partially validating this proposal. However, the original extension-management flags remain flat and the broader namespacing policy discussion (including whether to migrate existing flags) is unresolved — keeping open as under-discussion.

## Change summary

Adopt subcommand namespacing in the VS Code CLI to organize growing functionality (extensions, terminal, etc.) instead of flat --flags.

## Best solution

The subcommand pattern is now proven in the codebase (chat, tunnel, agent). The remaining work is deciding whether to migrate legacy extension-management flags to a `code ext` subcommand and establishing a formal CLI review/approval process for new arguments. This belongs with the CLI area owners to finalize the migration policy.
