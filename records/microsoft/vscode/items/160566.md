# Issue #160566 — Adjust settings dialog suggests language, not a script

[https://github.com/microsoft/vscode/issues/160566](https://github.com/microsoft/vscode/issues/160566)

## Summary

Still present on main: the unicode highlighter's 'Adjust settings' dialog labels locale codes as 'language' (e.g., "ru") when the user expects a script name (e.g., "Cyrillic"). The feature request has no upvotes and is in Backlog Candidates territory; keeping open for potential future improvement.

## Change summary

The unicode highlight 'Adjust settings' quick-pick suggests allowing characters common in a 'language' (locale code) rather than a 'script' (e.g., Cyrillic), which would be more accurate for multi-language scripts.

## Best solution

Refactor the ambiguous-character resolution in `src/vs/editor/contrib/unicodeHighlighter/browser/unicodeHighlighter.ts` to map locale codes to their Unicode script names (or at least display a human-readable locale name instead of a raw code), so the label reads something like 'Allow characters common in the Cyrillic script' rather than 'Allow unicode characters that are more common in the language "ru"'.
