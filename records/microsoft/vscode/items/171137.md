# Issue #171137 — Terminal - vscode.dev - remote tunnels - No shell integration in default terminal that is opened

[https://github.com/microsoft/vscode/issues/171137](https://github.com/microsoft/vscode/issues/171137)

## Summary

Still a valid bug: shell integration is injected only at terminal spawn time, so a pre-existing terminal reconnected via remote tunnels in vscode.dev will not have it. No fix found on main for this specific scenario.

## Change summary

The default terminal opened automatically when connecting to a remote tunnel via vscode.dev lacks shell integration because the process was already running before the web client connected.

## Best solution

Implement late/deferred shell integration injection for reconnected remote terminals — likely in the attachToProcess/revive flow in remoteTerminalBackend.ts and the pty host reconnection path — so that shell integration scripts are re-injected (or their state detected) when a client reattaches to an already-running terminal process.
