# Issue #75005 — Quickpick loses active item if items are updated

[https://github.com/microsoft/vscode/issues/75005](https://github.com/microsoft/vscode/issues/75005)

## Summary

Still reproduces on main: setting `items` always triggers `trySelectFirst()` via the default `ItemActivation.FIRST`, resetting the active item even when the array contains the same objects. The workaround (manually re-setting `activeItems`) exists but the original request for automatic preservation remains unaddressed.

## Change summary

QuickPick loses the active (focused) item when the `items` array is reassigned, even if the array contains the same item objects.

## Best solution

In the `itemsUpdated` branch of `QuickPick.update()` (src/vs/platform/quickinput/browser/quickInput.ts ~line 1088), before calling `trySelectFirst()`, check whether the previously active item still exists in the new items array (by identity) and restore focus to it instead of resetting to first.
