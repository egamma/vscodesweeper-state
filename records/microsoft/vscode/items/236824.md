# Issue #236824 — Crash with Illegal Instruction under Rosetta2 emulation in remote container

[https://github.com/microsoft/vscode/issues/236824](https://github.com/microsoft/vscode/issues/236824)

## Summary

The crash is upstream: the core dump (comment 5) shows it occurs in `/run/rosetta/rosetta /usr/local/bin/git git status -z -uall` — git compiled for x64 hitting instructions Rosetta2 cannot emulate, not in VS Code or Node.js. Related to #214630 (AVX in Node) but distinct: here the crashing binary is git itself under Rosetta2 in a container. Keeping open as an upstream issue.

## Change summary

Running an x64 dev container on an ARM Mac causes 'Illegal Instruction' crashes when VS Code triggers git operations under Rosetta2 emulation.

## Best solution

This is a Rosetta2/Docker emulation limitation for x64 git binaries on ARM. VS Code triggers `git status` which crashes, but the fix belongs in the git binary, the container image (use ARM-native images), or Apple's Rosetta2. The existing label `upstream` is appropriate. A pragmatic workaround note pointing users to ARM-native containers would help. Relates to but is distinct from https://github.com/microsoft/vscode/issues/214630 (Node AVX512).
