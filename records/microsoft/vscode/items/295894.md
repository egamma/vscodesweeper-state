# Issue #295894 — per-model(s) instruction files

[https://github.com/microsoft/vscode/issues/295894](https://github.com/microsoft/vscode/issues/295894)

## Summary

Valid feature request for model-specific instruction files. Not implemented on main — the instruction file system currently supports only `applyTo` (glob-based file matching) and `excludeAgent`, with no model-filtering attribute.

## Change summary

Allow instruction files to target specific AI models via a frontmatter attribute (e.g. `applyToModel: 'gpt-*-codex'`) so different models receive different instructions.

## Best solution

Add an optional `applyToModel` (or similar) glob attribute to the instruction file YAML frontmatter, processed in `src/vs/workbench/contrib/chat/common/promptSyntax/computeAutomaticInstructions.ts` alongside the existing `applyTo` logic, so instructions are only injected when the active model matches the pattern.
