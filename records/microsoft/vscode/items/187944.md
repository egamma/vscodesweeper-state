# Issue #187944 ŌĆö `editor.detectIndentation` produces incorrect values for empty files when `editor.insertSpaces` is `true`

[https://github.com/microsoft/vscode/issues/187944](https://github.com/microsoft/vscode/issues/187944)

## Summary

Still reproduces on main: when detectIndentation cannot determine indentation (e.g. empty file), it ignores the user's configured `indentSize` and hardcodes it to follow `tabSize`. The TODO comments in the code acknowledge this gap.

## Change summary

When `editor.detectIndentation` falls back (e.g. on empty files), `editor.indentSize` is ignored and set to `tabSize` instead of the user's configured value.

## Best solution

Pass the configured `indentSize` into `detectIndentation` and use it as the fallback when indentation cannot be guessed. The fix touches `src/vs/editor/common/model/textModel.ts` (the `detectIndentation` method signature and `resolveOptions`) and possibly the `guessIndentation` return type.
