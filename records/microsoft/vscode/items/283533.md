# Issue #283533 — Strange white line at bottom of window after hiding taskbar

[https://github.com/microsoft/vscode/issues/283533](https://github.com/microsoft/vscode/issues/283533)

## Summary

This is an upstream Chromium/Electron ANGLE rendering issue — commenters confirm the same white line appears in Chrome and Edge, and switching the ANGLE backend to OpenGL resolves it. Not a VS Code-specific bug. Classifying as upstream and keeping open for tracking.

## Change summary

A thin white line appears at the bottom (or side) of the VS Code window when maximized with the Windows taskbar set to auto-hide, caused by the default ANGLE rendering backend in Chromium.

## Best solution

Label as `upstream` + `windows` + `electron`. The root cause is in Chromium's ANGLE D3D11 rendering of maximized window borders on Windows 11 with auto-hidden taskbar. VS Code cannot fix this directly; it needs a fix in Chromium or Electron. The `--use-angle=gl` flag is a viable workaround users can apply now. A prior near-duplicate (https://github.com/microsoft/vscode/issues/285089) was closed for lack of info — this issue has better context and should be kept as the tracking item.
