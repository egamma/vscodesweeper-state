# Issue #326102 — Non-paged pool memory leak (UCOD tag grows continuously when VS Code is running)

[https://github.com/microsoft/vscode/issues/326102](https://github.com/microsoft/vscode/issues/326102)

## Summary

The UCOD kernel pool tag belongs to Windows Code Integrity (ci.dll), a kernel-mode component — not VS Code. The reporter is also on Windows 10 1909, which has been out of support since May 2020. This is most likely an OS-level or driver-level issue triggered by code-signing verification when Electron loads modules, not a defect in VS Code's own code. Requesting more info to rule out environment factors before any further action.

## Change summary

Non-paged kernel pool memory (UCOD tag) grows continuously at ~2 MB/s while VS Code is running, eventually causing system instability.

## Best solution

This is a Windows kernel Code Integrity (ci.dll) issue, not a VS Code bug. The reporter should update from the unsupported Windows 10 1909 to a current build, check for third-party security/AV/filter drivers interfering with CI, and verify the issue persists on a supported OS. If it does, it should be reported to Microsoft's Windows team or Electron upstream.
