# Issue #311547 — [Error] potential listener LEAK — contextKeyService

[https://github.com/microsoft/vscode/issues/311547](https://github.com/microsoft/vscode/issues/311547)

## Summary

Still present on main: the changesViewRenderer creates a ScopedContextKeyService per template row, each subscribing to the parent's onDidChangeContext emitter — exceeding the leak-detection threshold when enough templates accumulate in the row cache. No fix has landed.

## Change summary

The Agents window changes tree triggers a 'potential listener LEAK' error because each pooled template row creates a scoped context key service that subscribes to the parent emitter without raising the listener threshold.

## Best solution

Either raise the listener threshold on the parent context key service's onDidChangeContext emitter for this tree (acknowledging pooled templates are expected), or restructure the renderer to share a single scoped context key service across templates and update context keys in renderElement instead of renderTemplate.
