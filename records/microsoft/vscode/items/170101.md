# Issue #170101 — npm.packageManager=auto incorrectly assumes yarn as package manager for running tasks

[https://github.com/microsoft/vscode/issues/170101](https://github.com/microsoft/vscode/issues/170101)

## Summary

Still reproduces on current main: `isYarnPreferred` uses `find-yarn-workspace-root` which only checks for `packageJson.workspaces`, falsely detecting yarn in npm/pnpm workspace projects. Two open PRs attempt fixes but neither has merged.

## Change summary

Package manager auto-detection incorrectly identifies yarn when a project uses npm or pnpm workspaces (the `workspaces` field in package.json triggers false yarn detection).

## Best solution

Land one of the open fix PRs (https://github.com/microsoft/vscode/pull/243250 or https://github.com/microsoft/vscode/pull/302916) that replaces or guards the `find-yarn-workspace-root` check — prioritizing the `packageManager` field in package.json and requiring an actual yarn.lock before concluding yarn is preferred.
