# Issue #113541 — QuickPick API: When input value changes, first item becomes active item

[https://github.com/microsoft/vscode/issues/113541](https://github.com/microsoft/vscode/issues/113541)

## Summary

Still reproduces on main: when the input value changes, `trySelectFirst()` is called before `onDidChangeValueEmitter.fire()`, causing a visual flash to the first item before extensions can set their desired active item.

## Change summary

QuickPick API forces focus to first item on every input value change, causing a flash when extensions want to control active item themselves.

## Best solution

When `matchOnLabel` is false and `sortByLabel` is false (extension is managing filtering/active state itself), skip the `trySelectFirst()` call in the value-change path at `src/vs/platform/quickinput/browser/quickInput.ts:603`. Alternatively, respect `itemActivation = NONE` in the filter path, not just the items-updated path.
