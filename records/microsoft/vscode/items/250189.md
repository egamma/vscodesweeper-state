# Issue #250189 — Bug in global search field

[https://github.com/microsoft/vscode/issues/250189](https://github.com/microsoft/vscode/issues/250189)

## Summary

Still reproduces on main: programmatic `setValue` via Ctrl+Shift+F directly assigns `this.input.value` on the DOM input element, which clears the browser's native undo stack. The defect is in the `InputBox.set value()` setter and the `FindInput.setValue()` method.

## Change summary

Using Ctrl+Shift+F to populate the global search field from an editor selection destroys the search input's Ctrl+Z undo history.

## Best solution

The `InputBox.set value()` setter should use a DOM-undo-preserving mechanism (e.g. focusing the input, selecting all, and using `document.execCommand('insertText', ...)` or the Clipboard API) instead of directly assigning `.value`, so that programmatic text insertion preserves the native undo stack. The fix would be in `src/vs/base/browser/ui/inputbox/inputBox.ts` and potentially `src/vs/base/browser/ui/findinput/findInput.ts`. Care is needed to avoid breaking other callers of `setValue` that may not expect undo-stack side effects.
