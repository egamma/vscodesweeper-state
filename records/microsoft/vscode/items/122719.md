# Issue #122719 — Git - Cannot push branch after branch is deleted from the remote

[https://github.com/microsoft/vscode/issues/122719](https://github.com/microsoft/vscode/issues/122719)

## Summary

Still reproduces on main: after deleting a remote branch and renaming the local branch, the stale upstream tracking config persists, causing push to fail with 'no commits to push'. The `renameBranch` implementation does a bare `git branch -m` without clearing or updating the upstream, and the push path blindly uses the old upstream name.

## Change summary

After deleting a published branch from the remote and renaming the local branch, VS Code's push/publish fails because the stale upstream tracking configuration still points to the old (deleted) remote branch name.

## Best solution

After `git branch -m` in `renameBranch()`, unset the upstream tracking ref (via `git branch --unset-upstream`) so the branch is treated as unpublished. The push flow already handles `NoUpstreamBranch` by offering to publish — clearing the stale upstream would let that existing codepath kick in. Fix is localized to `extensions/git/src/git.ts` (`renameBranch`) and possibly `extensions/git/src/repository.ts`.
