# Issue #318982 — Agent plugin marketplace mangles SSH `source.url` when user is not `git` (breaks GHE / self-hosted)

[https://github.com/microsoft/vscode/issues/318982](https://github.com/microsoft/vscode/issues/318982)

## Summary

Valid bug: the marketplace reference and plugin source URL parsers on current main do not handle `git+ssh://` scheme at all (rejected at line 225 of marketplaceReference.ts), and SCP-style URLs with non-`git` users may produce incorrect cache paths via URI.parse fallback. GHE environments where the SSH user is not `git` are blocked from using SSH-based plugin sources.

## Change summary

Agent plugin marketplace mangles or rejects SSH `source.url` values when the SSH user is not `git`, preventing plugin installation on GHE instances with non-standard SSH users.

## Best solution

Extend `parseUriMarketplaceReference` in `marketplaceReference.ts` to accept the `git+ssh` scheme (line 225), and ensure `_gitUrlCacheSegments` in `pluginSources.ts` handles SCP-style URLs gracefully (the current catch fallback at line 346 may suffice, but should be verified). The error message at `pluginDirNotFound` should also not surface when a URL source fails to clone — the user should see a clone failure, not a missing-directory error.

## Evidence

- **scheme rejection:** src/vs/workbench/contrib/chat/common/plugins/marketplaceReference.ts line 225: `if (scheme !== 'http' && scheme !== 'https' && scheme !== 'ssh') { return undefined; }` — `git+ssh` is not accepted.
- **SCP regex handles any user:** marketplaceReference.ts line 287: regex `/^([^@\s]+)@([^:\s]+):(.+?\.git)(?:#(.+))?$/i` does accept any user for SCP-style, but only for marketplace references — not for `parsePluginSource` which delegates the raw URL string directly.
- **cache path fallback:** pluginSources.ts line 338-348: `_gitUrlCacheSegments` calls `URI.parse(url)` which may misinterpret SCP URLs, but falls back to a sanitized string in the catch block.
