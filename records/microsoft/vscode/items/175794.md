# Issue #175794 — Find in files: Inconsistent search results when using regex and whole word matching

[https://github.com/microsoft/vscode/issues/175794](https://github.com/microsoft/vscode/issues/175794)

## Summary

Still reproduces on main: the `createRegExp` function in `strings.ts` applies `\b` to only the first/last characters of the whole regex string, so alternations like `Foo|Bar` become `\bFoo|Bar\b` instead of `\b(?:Foo|Bar)\b`, producing incorrect whole-word matches. A prior Copilot-authored PR (#253061) attempted a fix but was closed without merge.

## Change summary

Regex whole-word search applies word boundaries incorrectly to alternation patterns, matching partial words that shouldn't match.

## Best solution

Fix `createRegExp` in `src/vs/base/common/strings.ts` (lines 209–216) to wrap alternation patterns in a non-capturing group before applying `\b` word boundaries, so `Foo|Bar` becomes `\b(?:Foo|Bar)\b` rather than `\bFoo|Bar\b`. The prior approach in PR #253061 (splitting each alternative and applying boundaries individually) is also valid but more complex.
