# Issue #238151 — FREETYPE_PROPERTIES="autofitter:no-stem-darkening=0" messes with product icons rendering

[https://github.com/microsoft/vscode/issues/238151](https://github.com/microsoft/vscode/issues/238151)

## Summary

This is an upstream Chromium/FreeType interaction issue — the same distortion reproduces in Chrome on vscode.dev (confirmed by a team member in #209801). VS Code uses an icon font (codicons) with `text-rendering: auto`, and FreeType's `no-stem-darkening` setting applies text hinting to the icon glyphs. A CSS mitigation (`text-rendering: geometricPrecision`) was proposed in #246110 but closed without reaching the upvote threshold. Keeping open as upstream since a team member (aeschli) asked for input and no final disposition was made on this specific issue.

## Change summary

FreeType's `no-stem-darkening` environment variable distorts codicon icon font rendering on Linux because Chromium applies text hinting to icon glyphs.

## Best solution

The cleanest VS Code-side mitigation would be changing `text-rendering: auto` to `text-rendering: geometricPrecision` in `src/vs/base/browser/ui/codicons/codicon/codicon.css` for the `.codicon` selector. However, since the root cause is in Chromium's FreeType integration (confirmed to reproduce in Chrome on vscode.dev), a team member should decide whether to apply this CSS workaround or label as upstream and defer.
