# Issue #318516 — Add Bootstrap Support for Git Worktrees

[https://github.com/microsoft/vscode/issues/318516](https://github.com/microsoft/vscode/issues/318516)

## Summary

Valid feature request for post-creation bootstrap hooks when creating Git worktrees. No such capability exists on current main — the createWorktree flow has no hook/script mechanism. Routing to the git area for triage.

## Change summary

Add configurable post-creation bootstrap support (hooks, file copy/symlink, dependency install) for newly created Git worktrees.

## Best solution

Consider a `git.worktree.postCreateScript` setting or a task-based hook in the git extension's `_createWorktree` flow (extensions/git/src/commands.ts). Alternatively, this could be an extension-candidate if VS Code's task/hook system is deemed sufficient — a marketplace extension could listen for worktree creation events via the git extension API.
