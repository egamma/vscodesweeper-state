[https://github.com/microsoft/vscode/issues/286361](https://github.com/microsoft/vscode/issues/286361)

## Summary

Still reproduces on main — the disposed context key service is accessed during a toolbar update triggered by `setDiffModel(null)` inside a transaction when closing a split diff editor. No fix has landed. Additionally, the 'On Deck' milestone is a hard keep-open.

## Change summary

Closing a split diff editor triggers an error because the context key service is accessed after disposal during a toolbar menu update inside the setDiffModel transaction.

## Best solution

Guard the toolbar update path (likely in `menuService.ts` or `toolbar.ts`) against a disposed context key service, or restructure the `clearInput` → `setDiffModel(null)` flow in `textDiffEditor.ts` / `editorPanes.ts` so the context key service outlives the transaction that nulls the model. The diff editor's `setDiffModel` fires context-key changes inside a subtransaction that triggers menu recomputation after the scoped context key service is already disposed.
