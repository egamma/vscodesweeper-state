# Issue #304555 — Copilot Chat: child_process ENOPRO when file:// URIs are passed to commands

[https://github.com/microsoft/vscode/issues/304555](https://github.com/microsoft/vscode/issues/304555)

## Summary

The bug lives in the Copilot Chat extension (command-runner boundary in vscode-copilot-chat), not in VS Code core — an open fix PR already exists in that repo. Proposing close as *caused-by-extension.

## Change summary

Copilot Chat extension passes file:// URIs directly to child_process spawn/execFile instead of converting them to filesystem paths, causing ENOPRO errors on Linux.

## Best solution

The fix belongs in the Copilot Chat extension (https://github.com/microsoft/vscode-copilot-chat/pull/4666 is already open addressing this). The reporter should track that PR for resolution.
