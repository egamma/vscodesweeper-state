# Issue #304555 — Copilot Chat: child_process ENOPRO when file:// URIs are passed to commands

[https://github.com/microsoft/vscode/issues/304555](https://github.com/microsoft/vscode/issues/304555)

## Summary

Valid bug in the hook executor: when `hook.command` contains file:// URI strings (as opposed to URI objects), they are passed directly to the shell without path conversion, causing ENOPRO on Linux. The code on main converts URI objects in cwd and JSON input but not URI strings in the command text itself. No fix has landed on main.

## Change summary

Hook commands that contain file:// URI strings in their command text fail with ENOPRO because the executor doesn't normalize URI strings to filesystem paths before spawning.

## Best solution

Apply URI-to-path normalization for file:// strings in the command text at the spawn boundary in `extensions/copilot/src/platform/chat/node/hookExecutor.ts`, similar to the existing `uriToFsPath` handling for cwd and JSON input. The related open PR in the deprecated vscode-copilot-chat repo suggests the team is aware; the fix should land in-tree.
