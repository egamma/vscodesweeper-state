# Issue #291679 — Cursor in the middle of terminal path

[https://github.com/microsoft/vscode/issues/291679](https://github.com/microsoft/vscode/issues/291679)

## Summary

Real terminal persistence bug on Linux where the revived session places the cursor at the column offset of the previous project's path. Not fixed on current main — the only related cursor-position fix (1d3e82fd38b) is Windows-specific and targets pty host restart, not persistent session revive across workspaces.

## Change summary

When a terminal session is revived via `persistentSessionReviveProcess`, the cursor appears at the wrong column — stuck at the position matching the length of the previously opened project's path.

## Best solution

Fix the persistent session revive logic (likely in ptyService.ts or terminalService.ts) to reset the cursor column when restoring a session into a new workspace context on Linux/bash. The reporter's workaround (disabling the setting) confirms the revive path is the culprit.
