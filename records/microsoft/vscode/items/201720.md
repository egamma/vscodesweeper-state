# Issue #201720 — Abnormal syntax highlight for Perl in Search Editor ("search.action.openNewEditor")

[https://github.com/microsoft/vscode/issues/201720](https://github.com/microsoft/vscode/issues/201720)

## Summary

Real but niche visual defect: the search-result TextMate grammar embeds `source.perl` per-line, so multi-line Pod constructs (`=head1`…`=cut`) lose their section-header highlighting because the Perl grammar needs cross-line state to recognize Pod blocks. Still present on main — no fix found.

## Change summary

Perl Pod section headers (e.g. `=head1`) are not syntax-highlighted in the Search Editor, though they render correctly in the regular Text Editor.

## Best solution

The search-result grammar in `extensions/search-result/syntaxes/searchResult.tmLanguage.json` embeds languages line-by-line, which breaks multi-line constructs like Perl Pod. A targeted fix could add explicit patterns for Pod directives (`=head1`, `=head2`, `=over`, `=item`, `=back`, `=cut`, `=pod`, `=encoding`, `=for`, `=begin`, `=end`) within the Perl block of the search-result grammar so they are recognized independently of cross-line state. This is a narrow grammar improvement in the `search-result` built-in extension.
