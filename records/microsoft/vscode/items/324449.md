# Issue #324449 — Chat input box's placeholder not dimmed anymore

[https://github.com/microsoft/vscode/issues/324449](https://github.com/microsoft/vscode/issues/324449)

## Summary

Team-authored bug report (MEMBER) — the chat input placeholder text renders at full foreground color instead of the dimmed `input.placeholderForeground`. The decoration-based placeholder correctly sets the color via `getInputPlaceholderColor()`, but the CSS rule on `[class^="ced-chat-session-detail"]` or the inline color may be overridden. Keeping open per hard keep-open rule for team-authored issues.

## Change summary

The chat input box placeholder text appears at full opacity instead of being dimmed like standard placeholder text.

## Best solution

Investigate whether the decoration's inline `color` from `getInputPlaceholderColor()` (which resolves `input.placeholderForeground` at 50% opacity) is being overridden — likely by the `.chat-editor-container .monaco-editor .mtk1 { color: var(--vscode-input-foreground) }` rule at line 1607 of `chat.css`, which sets the full foreground color on all `.mtk1` spans including the placeholder decoration's text node. The fix likely needs to ensure the placeholder decoration's color takes precedence over the `.mtk1` foreground rule.
