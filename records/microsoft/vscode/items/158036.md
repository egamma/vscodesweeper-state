# Issue #158036 — Overlapping scrollbar doesn't scroll outer scrollbar

[https://github.com/microsoft/vscode/issues/158036](https://github.com/microsoft/vscode/issues/158036)

## Summary

Still present on main: inner scrollable elements (list/tree widgets) default to `alwaysConsumeMouseWheel: true`, preventing scroll propagation to parent scrollable containers when inner scroll reaches its boundary. The debug sidebar's nested views exhibit this when the window is short enough to trigger the outer scrollbar.

## Change summary

When scrolling reaches the end of a nested scrollable in the debug sidebar, the scroll event does not propagate to the outer scrollbar.

## Best solution

Either fix the minimum body sizes of debug views so the outer scrollbar doesn't appear (as @roblourens suggested), or allow scroll propagation at boundaries by changing the scroll consumption logic in `scrollableElement.ts` to not stop propagation when `didScroll` is false. The second approach would be a broader fix benefiting all nested scrollable areas.
