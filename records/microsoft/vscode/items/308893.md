# Issue #308893 — Agents App: Changes reports `git diff --stat` instead of `git diff --stat -w`

[https://github.com/microsoft/vscode/issues/308893](https://github.com/microsoft/vscode/issues/308893)

## Summary

Team-authored issue; the Changes panel's diff stats count whitespace-only changes because the underlying data (from agent host / copilot sessions providers) does not apply a whitespace-ignoring flag. The discussion suggests honoring existing settings like `diffEditor.ignoreTrimWhitespace`. Still needs a design decision and implementation.

## Change summary

The Agents window Changes panel shows inflated diff stats (insertions/deletions) because whitespace-only line changes are counted, unlike `git diff --stat -w`.

## Best solution

When computing diff stats for session changesets, honor `diffEditor.ignoreTrimWhitespace` (or `scm.diffDecorationsIgnoreTrimWhitespace`) so that whitespace-only line changes are excluded from the reported insertions/deletions counts. The stats flow through `agentHostDiffs.ts` / `agentHostSessionFiles.ts` and the copilot sessions changesets provider; a `-w` equivalent needs to be applied at the point where the server or client computes per-file line counts.
