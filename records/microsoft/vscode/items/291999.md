# Issue #291999 — GitHub Copilot Connection Pool Issue with Forticlient VPN

[https://github.com/microsoft/vscode/issues/291999](https://github.com/microsoft/vscode/issues/291999)

## Summary

The connection timeout originates in the Copilot extension's own fetch/undici connection pool, not in VS Code core's request infrastructure. VS Code core uses Node.js http/https (src/vs/platform/request/node/requestService.ts), while the stack trace points to undici within the Copilot extension host. This belongs in the Copilot extension's issue tracker.

## Change summary

Copilot requests fail with UND_ERR_CONNECT_TIMEOUT after idle periods when using Forticlient VPN, because stale pooled connections are reused instead of creating fresh ones.

## Best solution

File this in microsoft/vscode-copilot-release — the connection pool behavior and retry logic live in the Copilot extension, not VS Code core. The extension could implement connection health checks, shorter pool idle timeouts, or automatic retry on UND_ERR_CONNECT_TIMEOUT with a fresh socket.
