# Issue #204074 — Multi diff editor: provide right context for current resource

[https://github.com/microsoft/vscode/issues/204074](https://github.com/microsoft/vscode/issues/204074)

## Summary

Still an open gap on main: the multi-diff editor's per-item toolbar context does not populate ResourceContextKey with the current file's URI, making when-clause filtering on resource properties impossible for extension-contributed menu items.

## Change summary

Multi-diff editor should set resource context keys (resourceScheme, resourcePath, etc.) to the focused/active resource's URI rather than the synthetic multi-diff-editor URI.

## Best solution

In `src/vs/editor/browser/widget/multiDiffEditor/diffEditorItemTemplate.ts`, after creating the scoped context key service (~line 218), instantiate a `ResourceContextKey` bound to the modified (or original) URI of the current diff item. This would let extensions use `resourceScheme`, `resourceLangId`, etc. in `multiDiffEditor/resource/title` when clauses.
