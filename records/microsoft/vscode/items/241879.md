# Issue #241879 — `git.diagnosticsCommitHook.Sources` doesn't document valid keys

[https://github.com/microsoft/vscode/issues/241879](https://github.com/microsoft/vscode/issues/241879)

## Summary

Valid bug: the setting keys are matched against `Diagnostic.source` strings (e.g. `"ts"`, `"eslint"`), not globs, but the setting description doesn't explain this. The issue has milestone 'On Deck' and an open PR (#312589) addressing it — keeping open.

## Change summary

The `git.diagnosticsCommitHook.sources` setting doesn't document that its keys must be exact diagnostic source names (not glob patterns), confusing users.

## Best solution

Land the open PR https://github.com/microsoft/vscode/pull/312589 which clarifies valid keys in the setting description. The code in `extensions/git/src/commands.ts` matches keys via exact `Object.keys(sourceSeverity).includes(d.source)`, so valid keys are literal diagnostic source strings like `"ts"`, `"eslint"`, plus the `"*"` wildcard.
