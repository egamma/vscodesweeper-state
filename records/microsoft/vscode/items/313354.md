## Summary

Team-authored bug report — kept open per policy. The issue describes a mobile-specific interaction conflict where long-pressing the chat input box to paste instead triggers a context menu from a parent/sibling element, making paste impossible. No fix has landed on main since the report was filed.

## Change summary

On mobile, long-pressing the Agents window chat input box to paste triggers a different element's context menu instead of the native paste menu.

## Best solution

The long-press gesture installed on chat rows (or a parent container) likely captures the pointer event before the input box's native context menu can fire. The fix should either exclude the input editor area from the long-press target, or let the native `contextmenu` event through when the touch target is inside the chat input's editable area. Likely files: `src/vs/sessions/browser/parts/mobile/longPress.ts` and wherever `installLongPress` is wired to chat row containers. The input box itself is in `src/vs/sessions/contrib/chat/browser/newChatInput.ts`.
