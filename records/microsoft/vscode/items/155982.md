## Change summary

Replace hand-crafted NLS localization IDs with random auto-generated IDs to reduce developer friction and eliminate meaningless naming decisions.

## Best solution

If pursued, adopt incrementally: new strings use generated IDs, old strings migrate over time via tooling. The discussion between hash-based vs pure-random IDs and translator context (comments vs IDs) still needs resolution.
