# Issue #146547 — Add an `onExtensionActivated` activation event

[https://github.com/microsoft/vscode/issues/146547](https://github.com/microsoft/vscode/issues/146547)

## Summary

Valid, unimplemented feature request for an activation event that triggers when a specific other extension activates. The use case (chaining extension activation beyond what extensionDependencies provides) is clear and distinct from existing mechanisms. Low traction (4 upvotes) but still a real gap.

## Change summary

Allow an extension to declare an activation event like `onExtensionActivated:<extensionId>` so it activates whenever the named extension activates, without duplicating activation events.

## Best solution

If pursued, implement a new `onExtensionActivated:<id>` activation event in the extension host's activation machinery (likely `src/vs/workbench/services/extensions/common/abstractExtensionService.ts` and related activation-event handling). However, given low traction, leaving it in Backlog for community signal is reasonable.
