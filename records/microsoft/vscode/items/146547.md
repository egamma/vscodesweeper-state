# Issue #146547 — Add an `onExtensionActivated` activation event

[https://github.com/microsoft/vscode/issues/146547](https://github.com/microsoft/vscode/issues/146547)

## Summary

Keep open because the issue is assigned to the Backlog milestone, which requires explicit maintainer handling. The request is a clear extension activation feature request rather than a cleanup candidate.

## Change summary

Add an `onExtensionActivated:<extensionId>` activation event so one extension can activate when another extension activates.

## Best solution

Extension activation/API owners should decide whether VS Code should support activation based on another extension's activation, or document an intended alternative to avoid duplicating activation events across dependent extensions.
