# Issue #146547 — Add an `onExtensionActivated` activation event

[https://github.com/microsoft/vscode/issues/146547](https://github.com/microsoft/vscode/issues/146547)

## Summary

Keep this open as an unscheduled extension-activation feature request. The existing `extensionDependencies` behavior only activates dependencies before the dependent extension, and current main does not appear to provide an `onExtensionActivated:<id>` activation event.

## Change summary

Add an activation event that activates one extension when another named extension is activated.

## Best solution

The extension activation area should decide whether to support a reverse-dependency activation event or a different extension-to-extension coordination pattern; if accepted, it likely belongs in the activation event schema and extension activation dispatch path.
