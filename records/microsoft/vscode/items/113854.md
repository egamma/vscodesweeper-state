# Issue #113854 — "Cannot edit in read-only editor" tooltip may not show in web version

[https://github.com/microsoft/vscode/issues/113854](https://github.com/microsoft/vscode/issues/113854)

## Summary

Keep open because this is a team-authored issue from a VS Code member, and the sweeper rules reserve those for explicit maintainer judgment. The discussion also identifies a plausible web editor root cause in the textarea readonly handling, so this remains a real editor/web bug rather than a cleanup close.

## Change summary

Typing into a read-only editor in VS Code for Web may fail to show the expected "Cannot edit in read-only editor" tooltip when adding text.

## Best solution

Have the editor/web owners decide whether the web textarea should avoid the `readonly` attribute on non-mobile browsers, or replace that behavior with a web-safe keyboard suppression mechanism such as `inputmode="none"` where supported.
