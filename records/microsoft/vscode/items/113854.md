## Change summary

In VS Code web, typing into a read-only editor may fail to show the expected "Cannot edit in read-only editor" tooltip when adding text.

## Best solution

Editor maintainers should decide whether the web textarea still needs the `readonly` attribute for keyboard suppression, or whether a web/mobile-specific alternative such as `inputmode="none"` can preserve mobile behavior while allowing the editor command path to surface the read-only tooltip.
