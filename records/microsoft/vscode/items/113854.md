# Issue #113854 — "Cannot edit in read-only editor" tooltip may not show in web version

[https://github.com/microsoft/vscode/issues/113854](https://github.com/microsoft/vscode/issues/113854)

## Summary

Fixed on main since March 2021 — commit 02b394a1a5c introduced `domReadOnly` to decouple the textarea `readonly` attribute from the editor's read-only state, so input events now fire on web and the tooltip appears. Shipped in 1.55.0.

## Change summary

The 'Cannot edit in read-only editor' tooltip did not appear when typing into a read-only file in VS Code for Web because the `readonly` HTML attribute on the textarea suppressed input events.

## Best solution

Keep the shipped fix (commit 02b394a1a5c, VS Code 1.55.0); no further change is needed — the tooltip now shows correctly on web.
