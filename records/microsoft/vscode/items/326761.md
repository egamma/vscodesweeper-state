# Issue #326761 — Key to copy text from Debug Console different from Termal

[https://github.com/microsoft/vscode/issues/326761](https://github.com/microsoft/vscode/issues/326761)

## Summary

The difference is intentional: the integrated terminal uses Ctrl+Shift+C on Linux because Ctrl+C must send SIGINT to the running process — this matches every other Linux terminal emulator. The Debug Console has no shell, so it uses the standard Ctrl+C. Proposing close as designed.

## Change summary

User finds it confusing that copy is Ctrl+C in the Debug Console but Ctrl+Shift+C in the Terminal on Linux.

## Best solution

Keep the current behavior. The Terminal's Ctrl+Shift+C follows universal Linux terminal convention (Ctrl+C = SIGINT). The Debug Console's Ctrl+C follows standard editor/panel convention. Users can customize keybindings if they prefer a different mapping.

  > Thanks for the report — I understand how the difference can feel surprising at first.

  > This is actually intentional and matches how every Linux terminal emulator works: in a terminal, **Ctrl+C** must send the interrupt signal (SIGINT) to the running process, so copy uses **Ctrl+Shift+C** instead. ## Evidence

- **terminal convention:** Every Linux terminal emulator (GNOME Terminal, Konsole, xterm, etc.) uses Ctrl+Shift+C for copy because Ctrl+C is reserved for SIGINT. VS Code's integrated terminal follows this universal convention.
- **debug console rationale:** The Debug Console is not a shell — it has no running process to send SIGINT to, so it uses the standard platform copy shortcut (Ctrl+C), consistent with every other non-terminal panel in VS Code.
- **customizable:** Users who want uniform shortcuts can rebind terminal.copySelection in Keyboard Shortcuts to Ctrl+C (at the cost of losing Ctrl+C as SIGINT in the terminal).
