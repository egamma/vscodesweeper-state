# Issue #314769 — PowerShell in Integrated  Terminal: Ctrl+Spacebar not working when shell integration is disabled

[https://github.com/microsoft/vscode/issues/314769](https://github.com/microsoft/vscode/issues/314769)

## Summary

Confirmed from source: when shell integration is disabled, the Ctrl+Space keybinding that sends the MenuComplete sequence to PowerShell is inactive (gated on `terminalShellIntegrationEnabled`), and no fallback passes the keystroke through — so Ctrl+Space does nothing.

## Change summary

Ctrl+Space does nothing in the integrated terminal's PowerShell session when shell integration is disabled, preventing PSReadLine MenuComplete from triggering.

## Best solution

The sendSequence keybinding at `src/vs/workbench/contrib/terminalContrib/sendSequence/browser/terminal.sendSequence.contribution.ts:157` sends an F12,a escape that the shell integration script remaps to Ctrl+Space. Without shell integration, this indirection doesn't work. A potential fix would add a separate lower-priority keybinding that sends the raw `\x00` (NUL) byte — the standard encoding of Ctrl+Space — when shell integration is disabled and suggest is off, so the keystroke reaches PowerShell's native MenuComplete handler.

## Evidence
