# Issue #226531 — VS Code installs  en-US-10-1.bdic

[https://github.com/microsoft/vscode/issues/226531](https://github.com/microsoft/vscode/issues/226531)

## Summary

Confirmed by team members (lszomoru, deepak1556) as a real issue needing a build-time fix to prevent Electron's spellchecker component from installing dictionary files. The runtime-level `spellcheck: false` in BrowserWindow options (src/vs/platform/windows/electron-main/windows.ts:155) doesn't prevent the .bdic file from being written. No fix has landed yet.

## Change summary

VS Code's Electron runtime installs an en-US spellcheck dictionary file (en-US-10-1.bdic) on startup even though VS Code doesn't use native spellchecking, interfering with spell-checking extensions.

## Best solution

Disable the spellchecker component at Electron build time as deepak1556 indicated — the BrowserWindow-level `spellcheck: false` suppresses the UI but doesn't prevent dictionary installation. This likely requires a build configuration change in the Electron packaging step.
