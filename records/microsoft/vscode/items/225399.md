# Issue #225399 — Add setting to control rendering of multi-line inline completions

[https://github.com/microsoft/vscode/issues/225399](https://github.com/microsoft/vscode/issues/225399)

## Summary

Team-authored feature request (hediet/MEMBER) for a setting to control multi-line inline completion rendering. No such setting exists on current main — the IInlineSuggestOptions interface has no multi-line rendering toggle. Keeping open per policy.

## Change summary

Add a user-facing setting to control whether multi-line inline completions are rendered (e.g. disable them while keeping single-line suggestions).

## Best solution

Add an option to IInlineSuggestOptions (e.g. `editor.inlineSuggest.multiLine` with values like `always`/`never`/`auto`) and wire it into the inline-completions model/view layer so users can suppress multi-line ghost text while retaining single-line suggestions.
