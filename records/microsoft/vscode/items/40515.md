# Issue #40515 — Tasks with target = TaskScope.Global as the scope do not show up in task picker UI

[https://github.com/microsoft/vscode/issues/40515](https://github.com/microsoft/vscode/issues/40515)

## Summary

Still unimplemented on current main — tasks with TaskScope.Global are silently dropped in _getGroupedTasks because the code only adds tasks that have a workspace folder. The 'On Deck' milestone also prevents proposing a close.

## Change summary

Tasks provided via the extension API with TaskScope.Global as scope do not appear in the task picker UI.

## Best solution

Extend _getGroupedTasks in abstractTaskService.ts (around line 2377–2383) to handle tasks without a workspace folder (Global scope) by storing them under a synthetic key (e.g. an empty-string or dedicated global bucket), and ensure the task quick-pick renders them. Variable resolution for ${workspaceFolder} in global tasks needs a decision (leave literal or resolve to empty).
