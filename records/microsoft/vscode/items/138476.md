# Issue #138476 — Misleading free memory reporting on MacOS in Issue Reporter

[https://github.com/microsoft/vscode/issues/138476](https://github.com/microsoft/vscode/issues/138476)

## Summary

Still reproduces on main: the issue reporter uses Node.js os.freemem() which on macOS reports only truly free pages, excluding purgeable/cached memory the system would reclaim on demand — making free memory look far lower than what's actually available. Author is a CONTRIBUTOR (not MEMBER), so the hard keep-open rule does not apply, but the bug remains valid and unfixed.

## Change summary

macOS free memory reporting in the Issue Reporter is misleadingly low because os.freemem() excludes purgeable/cached file memory.

## Best solution

On Darwin, replace os.freemem() with a platform-specific call (e.g. parsing `vm_stat` or using `sysctlbyname`) that includes purgeable + inactive memory as available, giving a number closer to what Activity Monitor reports as 'Memory Available'. The fix would touch `src/vs/platform/diagnostics/node/diagnosticsService.ts` (lines 205 and 277).
