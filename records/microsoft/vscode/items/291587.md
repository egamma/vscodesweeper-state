# Issue #291587 — MCP client authenticate using Kerberos/NTLM if the server replies with WWW-Authenticate: Negotiate

[https://github.com/microsoft/vscode/issues/291587](https://github.com/microsoft/vscode/issues/291587)

## Summary

Valid feature request from a team member: MCP HTTP transport should handle WWW-Authenticate: Negotiate (Kerberos/NTLM) for on-prem servers. A concrete reuse path exists via the existing proxy auth credential lookup. Keeping open per team-authored policy.

## Change summary

VS Code's MCP client should perform Kerberos/NTLM authentication when an HTTP MCP server responds with 401 and WWW-Authenticate: Negotiate, instead of falling through to the OAuth/.well-known flow.

## Best solution

Extend the MCP HTTP transport's auth handling to detect a 401 with WWW-Authenticate: Negotiate and reuse the existing credential-lookup mechanism from `src/vs/workbench/api/node/proxyResolver.ts` (the same path used for 407 proxy-authenticate). This keeps the fix localized to the MCP/auth layer without new dependencies.
