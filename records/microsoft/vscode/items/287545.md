# Issue #287545 — Insiders log constantly warns about models already being registered

[https://github.com/microsoft/vscode/issues/287545](https://github.com/microsoft/vscode/issues/287545)

## Summary

The warning at `src/vs/workbench/contrib/chat/common/languageModels.ts:1170` (`[LM] Model ${model.identifier} is already registered. Skipping.`) is still present on current main and can fire when multiple groups for the same vendor produce models with the same identifier. No commit references this issue. Routing to chat-lm-management for triage.

## Change summary

The VS Code Insiders dev console is constantly flooded with warnings about language models already being registered, drowning out useful log output.

## Best solution

Either deduplicate `allModels` before the cache-insert loop in `_resolveLanguageModels` (around line 1168 of `src/vs/workbench/contrib/chat/common/languageModels.ts`) so the warning only fires once per session, or downgrade the log level from `warn` to `trace` for duplicate model identifiers that are expected when multiple groups resolve the same model.
