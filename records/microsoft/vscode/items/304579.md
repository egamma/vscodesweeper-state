# Issue #304579 — Rebase from worktree window operates on wrong worktree and wrong branch, causing data loss

[https://github.com/microsoft/vscode/issues/304579](https://github.com/microsoft/vscode/issues/304579)

## Summary

Well-documented data-loss bug where rebase from a worktree window dispatches operations to wrong worktrees and rebases the wrong branch. No fix found on main. The reflog evidence is detailed but the exact reproduction steps (which UI path triggered it) remain underspecified — routing to the git area for investigation.

## Change summary

Rebasing from a worktree VS Code window operates on the wrong branch and scatters git operations across unrelated worktrees, destroying uncommitted changes.

## Best solution

The git extension's rebase/checkout path needs worktree isolation: operations initiated from a worktree window must execute exclusively within that worktree's Repository instance. Likely involves how the `{ repository: true }` command decorator resolves which Repository to use, and whether fetch/pull pre-steps incorrectly target the main repo root. Related to https://github.com/microsoft/vscode/issues/290708 (same class of bug). Needs maintainer investigation of the exact UI flow that triggered cross-worktree dispatch.
