# Issue #176361 — Git - Problem with Placeholder in settings.json using git.path

[https://github.com/microsoft/vscode/issues/176361](https://github.com/microsoft/vscode/issues/176361)

## Summary

Variable substitution (e.g. ${env:VAR}) is only supported in tasks.json and launch.json, not in general settings.json values. The git extension reads git.path via workspace.getConfiguration().get() which returns the raw string — this is by design.

## Change summary

Using environment variable placeholders like ${env:VAR} in the git.path setting in settings.json does not resolve the variable, causing VS Code to look for git at the literal placeholder path.

## Best solution

Close as designed. Variable substitution in settings.json is not a supported feature — it only works in tasks.json and launch.json. Users should use a literal absolute path in git.path, or set the PATH environment variable before launching VS Code.

  > Thanks for the detailed report. Variable substitution like `${env:VAR}` in settings.json is only supported in `tasks.json` and `launch.json` configurations — it's not a general settings feature. When you use `${env:PORTABLEGIT}` in `git.path`, the extension receives the literal placeholder string, not the resolved value.
  > 
  > To work around this, you can either:
  > 1. Use the full literal path in `git.path` (e.g. `"git.path": "C:\\temp\\portablegit\\bin\\git.exe"`)
  > 2. Set your `PATH` environment variable in the startup script before launching VS Code, so git is found automatically.
  > 
  > Closing as this is the current design for settings.json.
