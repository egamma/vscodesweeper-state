# Issue #292239 — When I have a git changes editor open, if I change files in git, the editor steals focus

[https://github.com/microsoft/vscode/issues/292239](https://github.com/microsoft/vscode/issues/292239)

## Summary

Team-authored issue with 'On Deck' milestone — keeping open. The bug is real: when the git multi-diff changes editor is open and the underlying git state changes (e.g. from a terminal checkout), the editor steals focus, likely because the editor open/refresh path doesn't pass preserveFocus appropriately.

## Change summary

The git multi-diff changes editor steals focus from the terminal (or other active view) when git state changes underneath it.

## Best solution

Ensure that when the SCM multi-diff editor is already open and its content refreshes due to git state changes, the refresh path passes `preserveFocus: true` so focus remains in the previously active component. The fix likely involves the `_workbench.openMultiDiffEditor` command or whatever re-triggers `setInput` on the multi-diff editor when resources change — `src/vs/workbench/contrib/multiDiffEditor/browser/multiDiffEditor.ts` already supports `preserveFocus` in `setInput`.
