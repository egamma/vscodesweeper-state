# Issue #133727 — Git - extension API shows incorrect remote when there's an extra push url

[https://github.com/microsoft/vscode/issues/133727](https://github.com/microsoft/vscode/issues/133727)

## Summary

Still reproduces on main — the git extension's remote-parsing logic cannot correctly represent multiple push URLs, and depending on the code path (FS config parse vs `git remote -v`), either the wrong URL or only the first/last URL is reported. Team-authored issue; keeping open.

## Change summary

The Git extension API reports incorrect fetchUrl/pushUrl when a remote has multiple push URLs configured via `git remote set-url --add`.

## Best solution

The `Remote` interface in `extensions/git/src/api/git.d.ts` models fetchUrl/pushUrl as single optional strings, which can't represent multiple push URLs. A proper fix would either (a) extend the API to support arrays (breaking change) or (b) ensure the single-value fields reflect the primary URL consistently across both parsing paths (`parseGitRemotes` in git.ts:909 and `getRemotesGit` in git.ts:3124). The FS path (`GitConfigParser`) only keeps the first `url` property (line 810 deduplicates), while the git CLI path overwrites pushUrl with the last encountered push line. Both paths need alignment, and the API shape likely needs a `pushUrls`/`fetchUrls` array extension.
