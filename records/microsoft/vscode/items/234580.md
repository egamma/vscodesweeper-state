# Issue #234580 — Editor GPU: Support non-regular inline decorations

[https://github.com/microsoft/vscode/issues/234580](https://github.com/microsoft/vscode/issues/234580)

## Summary

Still needed: current main explicitly rejects non-Regular inline decorations in the GPU renderer (falls back to DOM). The plan-item tracking this work remains valid.

## Change summary

Support rendering Before, After, and RegularAffectingLetterSpacing inline decoration types in the GPU editor renderer.

## Best solution

Implement GPU-side rendering for non-regular inline decoration types (Before, After, RegularAffectingLetterSpacing) in the viewport and full-file render strategies, removing the current fallback-to-DOM path in `viewGpuContext.ts`.
