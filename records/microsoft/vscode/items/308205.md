# Issue #308205 — [Mac] Unable to input a backslash in the terminal using the fish shell with a Japanese keyboard

[https://github.com/microsoft/vscode/issues/308205](https://github.com/microsoft/vscode/issues/308205)

## Summary

Legitimate bug report: fish shell's kitty keyboard protocol likely interferes with macOS yen-to-backslash key remapping in the VS Code terminal. No fix found on main, no duplicate identified. Routing to the terminal area for investigation.

## Change summary

On macOS with a Japanese keyboard configured to send backslash for the ¥ key, the VS Code terminal with fish shell sends U+00A5 (¥) instead of U+005C (\), while other terminals and other shells in VS Code work correctly.

## Best solution

Investigate how the kitty keyboard protocol (enabled by fish) interacts with macOS keyboard remapping of the yen key in xterm.js/VS Code terminal input handling. The fix likely involves ensuring the yen-to-backslash OS-level remapping is respected even when the kitty keyboard protocol is active. The relevant code path is in `src/vs/workbench/contrib/terminal/browser/xterm/xtermTerminal.ts` and possibly upstream in xterm.js.
