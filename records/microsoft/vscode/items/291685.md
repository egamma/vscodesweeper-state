# Issue #291685 — Make sharing of copilot instructions, skills etc globally easy for organizations

[https://github.com/microsoft/vscode/issues/291685](https://github.com/microsoft/vscode/issues/291685)

## Summary

Valid feature request for better organizational sharing of copilot instructions across repos. Existing settings (`chat.instructionsFilesLocations`) provide a partial workaround but don't fully address cross-repo/cross-machine scenarios (Codespaces, GitHub UI). Community interest is present but upvote count is unclear.

## Change summary

Users want a native mechanism to share copilot instructions, skills, and agents across multiple repositories in an organization without manual syncing or local file copies.

## Best solution

A first-class organization-level or multi-repo instruction sharing mechanism — possibly via a remote URL source in `chat.instructionsFilesLocations`, a GitHub org-level `.github` repo convention, or an extension-based distribution model. The partial workarounds (local folders, node_modules, extensions) exist but don't cover Codespaces or the GitHub UI Copilot surface.
