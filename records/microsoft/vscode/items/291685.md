# Issue #291685 — Make sharing of copilot instructions, skills etc globally easy for organizations

[https://github.com/microsoft/vscode/issues/291685](https://github.com/microsoft/vscode/issues/291685)

## Summary

Valid feature request for easier cross-repo/org-wide sharing of Copilot instructions. Current workarounds (chat.instructionsFilesLocations pointing to synced folders or node_modules) exist but the user and commenters argue they're too fragile or complex for teams. Under discussion with maintainer engagement but no resolution yet.

## Change summary

Users want a first-class way to share Copilot instructions across multiple repos in an organization without manual syncing or per-repo duplication.

## Best solution

Consider supporting remote/URL-based instruction file sources (e.g., a git repo URL or org-level config endpoint) in the `chat.instructionsFilesLocations` setting, so teams can point to a single canonical source without local sync infrastructure. Alternatively, a VS Code extension gallery-based distribution (already suggested by maintainer) could be better documented as the recommended path.
