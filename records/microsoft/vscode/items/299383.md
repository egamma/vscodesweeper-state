# Issue #299383 — Copilot: Support firewalled or airgapped remote machines via SSH

[https://github.com/microsoft/vscode/issues/299383](https://github.com/microsoft/vscode/issues/299383)

## Summary

Valid feature request already triaged into the Backlog with appropriate labels. Routing Copilot LLM requests through the local machine for airgapped remote SSH sessions is a real gap — no implementation exists on main. The comments also surface a concrete path-handling bug in agent mode over Remote-SSH, which adds signal. Keeping open for the community process.

## Change summary

Route Copilot/LLM requests through the local machine when connected via Remote-SSH to a server with no internet access, enabling full AI-assisted development in firewalled environments.

## Best solution

Implement local-side proxying of LLM API calls so that when Copilot's extension host runs on the remote, model traffic is tunneled through the SSH connection to the local machine's network. The comments also flag a separate path-handling bug (Linux paths treated as Windows paths in agent-mode tools over Remote-SSH) that should be tracked independently — see https://github.com/microsoft/vscode/issues/284417 and https://github.com/microsoft/vscode/issues/303402. Issue https://github.com/microsoft/vscode/issues/292758 appears to be a less-detailed duplicate of this request and could be consolidated here.
