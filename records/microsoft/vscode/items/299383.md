# Issue #299383 — Copilot: Support firewalled or airgapped remote machines via SSH

[https://github.com/microsoft/vscode/issues/299383](https://github.com/microsoft/vscode/issues/299383)

## Summary

Valid feature request for routing Copilot/LLM traffic through the local machine when connected to a firewalled remote via SSH. Already in Backlog with 9 upvotes — below the 20-vote promotion threshold but a real scenario with active community interest and a user-contributed patch. Keep open for the community process.

## Change summary

Route Copilot AI requests (language model calls, tool invocations) through the local VS Code client when the remote SSH host has no internet access.

## Best solution

Implement a relay/proxy mechanism so that when Copilot Agent mode runs tools on a remote SSH host, LLM API calls are forwarded through the local client's network connection. The comment from joserfg2 also identifies a concrete path-handling bug (Linux paths treated as Windows paths in Remote-SSH context) that may be a separable fix in the chat tool infrastructure. The broader routing feature likely belongs in the remote/chat plumbing layer.
