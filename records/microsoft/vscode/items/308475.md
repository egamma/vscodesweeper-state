# Issue #308475 — git.detectSubmodules fails to nest submodules with NFD Unicode paths on macOS

[https://github.com/microsoft/vscode/issues/308475](https://github.com/microsoft/vscode/issues/308475)

## Summary

Confirmed on current main: the git extension's `normalizePath` in `util.ts` lowercases and strips trailing separators but never performs Unicode normalization (NFC/NFD), so submodule path comparisons fail on macOS APFS where filesystem paths are NFD but `.gitmodules` stores NFC.

## Change summary

Git submodules with non-ASCII (e.g. umlaut) directory names are not recognized as nested under the parent repository on macOS because NFD filesystem paths don't match NFC `.gitmodules` paths.

## Best solution

Add `String.prototype.normalize('NFC')` (or 'NFD') to the `normalizePath` function in `extensions/git/src/util.ts` so that `isDescendant`, `pathEquals`, and `getRepositoryForSubmodule` all compare Unicode-equivalent paths correctly. Also normalize in `getRepositoryForSubmodule`'s direct `===` comparison at line 1022.
