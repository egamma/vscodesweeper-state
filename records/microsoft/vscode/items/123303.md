# Issue #123303 — Cursors are not merged in this case

[https://github.com/microsoft/vscode/issues/123303](https://github.com/microsoft/vscode/issues/123303)

## Summary

Team-authored issue; the bug remains unfixed on main. The related PR #123896 was closed without merging. The cursor merging logic in cursorCollection.ts skips merging entirely when `multiCursorMergeOverlapping` is false (line 188–190), but Cmd+D (Add Selection to Next Find Match) should still merge cursors that end up selecting the same range — the `continue` is too broad.

## Change summary

When `editor.multiCursorMergeOverlapping` is false, pressing Cmd+D creates duplicate overlapping cursors on the same word instead of merging them.

## Best solution

The fix likely lives in `src/vs/editor/common/cursor/cursorCollection.ts` around line 188: instead of unconditionally skipping all merging when `multiCursorMergeOverlapping` is false, Cmd+D selections that result in identical ranges should still be merged. The closed PR #123896 attempted this from the find-controller side; an alternative is to handle it in the cursor collection's merge logic with a more nuanced condition.
