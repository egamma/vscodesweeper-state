# Issue #243533 — Search results appear for excluded files when file is opened

[https://github.com/microsoft/vscode/issues/243533](https://github.com/microsoft/vscode/issues/243533)

## Summary

Still reproduces on main: the open-editor search path in `getOpenEditorResults` calls `pathIncludedInQuery`, which only checks the top-level `excludePattern` (from the UI exclude box) but ignores per-folder exclude patterns from `search.exclude` and `files.exclude` settings. Files excluded by settings still appear in results if they are open in an editor.

## Change summary

Search results include matches from files that are excluded via `search.exclude` or `files.exclude` settings when those files are currently open in the editor.

## Best solution

Fix `pathIncludedInQuery` (or the call site in `getOpenEditorResults`) to also check the per-folder `folderQueries[].excludePattern` entries, which carry the `search.exclude` and `files.exclude` settings. The fix should match the open file's resource against the exclude patterns of its containing folder query.
