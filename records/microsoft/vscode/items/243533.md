# Issue #243533 — Search results appear for excluded files when file is opened

[https://github.com/microsoft/vscode/issues/243533](https://github.com/microsoft/vscode/issues/243533)

## Summary

Confirmed regression on current main: open/dirty files bypass `search.exclude` and `files.exclude` settings because `getOpenEditorResults` uses `pathIncludedInQuery`, which only checks the top-level `excludePattern` (user-typed excludes from the search box), not the per-folder `excludePattern` where settings-based excludes live.

## Change summary

Search results include matches from files that are configured in `search.exclude`/`files.exclude` when those files are currently open in an editor.

## Best solution

Fix `getOpenEditorResults` in `src/vs/workbench/services/search/common/searchService.ts` to also check each open file's path against the per-folder `excludePattern` from `folderQueries`, not just the global `excludePattern`. The `pathIncludedInQuery` function in `src/vs/workbench/services/search/common/search.ts` needs to incorporate folder-level excludes for the matching resource.
