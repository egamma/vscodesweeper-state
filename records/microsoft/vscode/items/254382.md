# Issue #254382 — The column number calculation in the status bar has issues when meeting the full-width characters and emoji characters

[https://github.com/microsoft/vscode/issues/254382](https://github.com/microsoft/vscode/issues/254382)

## Summary

Confirmed in source: `toStatusbarColumn` in cursorColumns.ts counts all non-tab characters as 1 column, ignoring full-width/emoji visual width. The internal `_nextVisibleColumn` handles these correctly (+2), but the status bar method deliberately doesn't use it. Whether this is a bug or as-designed depends on what 'Col' should mean — there's an inconsistency since tabs ARE expanded to visual width while CJK/emoji are not.

## Change summary

Status bar column number does not account for CJK full-width characters or emoji occupying 2 visual columns.

## Best solution

Decide whether status bar 'Col' should reflect visual width (like `visibleColumnFromColumn`) or code-point count. If visual width: add `isFullWidthCharacter`/`isEmojiImprecise` checks to `toStatusbarColumn` (mirroring `_nextVisibleColumn`). If by design: close as `*as-designed` and document the distinction. The inconsistency with tab expansion suggests the visual-width interpretation is more coherent.
