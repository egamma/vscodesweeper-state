# Issue #254382 — The column number calculation in the status bar has issues when meeting the full-width characters and emoji characters

[https://github.com/microsoft/vscode/issues/254382](https://github.com/microsoft/vscode/issues/254382)

## Summary

Confirmed on current main: `toStatusbarColumn` in `cursorColumns.ts` does not account for full-width or emoji character widths — it increments by 1 for all non-tab code points, unlike `visibleColumnFromColumn` which correctly uses `_nextVisibleColumn` (adding 2 for full-width/emoji). Already labeled bug + editor-core by the team.

## Change summary

The status bar column indicator counts full-width CJK characters and emoji as 1 column instead of 2, inconsistent with their visual width.

## Best solution

Update `CursorColumns.toStatusbarColumn` in `src/vs/editor/common/core/cursorColumns.ts` to call `_nextVisibleColumn` (or replicate its full-width/emoji logic) so the status bar column reflects display width, matching the behavior of `visibleColumnFromColumn`.
