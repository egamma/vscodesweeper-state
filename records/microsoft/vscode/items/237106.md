# Issue #237106 — Configure coverage testing profile

[https://github.com/microsoft/vscode/issues/237106](https://github.com/microsoft/vscode/issues/237106)

## Summary

VS Code already supports configuring multiple coverage profiles via the dropdown on the coverage button — the mechanism exists when an extension registers 2+ coverage profiles. The issue is that the Go extension only registers one coverage profile; the missing dropdown is a result of that, not a VS Code limitation.

## Change summary

Request to add a profile-selection dropdown to the coverage run button, similar to how run/debug profiles can be selected.

## Best solution

This is already supported: `testingExplorerView.ts` (line ~424) shows a dropdown with all registered coverage profiles when there are 2+. The Go extension should register multiple `TestRunProfileKind.Coverage` profiles (e.g., 'Coverage (package)' and 'Coverage (project)') via `createRunProfile`. This belongs as a feature request against `golang/vscode-go`, not VS Code core.
