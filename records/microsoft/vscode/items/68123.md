# Issue #68123 — Specify terminal group for integrated terminal in launch configuration

[https://github.com/microsoft/vscode/issues/68123](https://github.com/microsoft/vscode/issues/68123)

## Summary

Still a valid, unimplemented feature request: the DAP `runInTerminal` protocol has no presentation/group property, and VS Code's debug infrastructure has no terminal grouping support for launch configs. The newer #310020 is a duplicate of this issue, not the reverse. With 16 upvotes and active community interest, this should remain open under discussion.

## Change summary

Allow debug launch configurations to specify terminal grouping/splitting behavior (analogous to the `presentation.group` property available in tasks) so compound debug sessions can launch terminals side by side.

## Best solution

The path forward, as outlined by @weinand in comment 8, is to add an opaque VS Code-specific option bag to the DAP `runInTerminal` request that can carry presentation hints (group, split direction). This requires a DAP spec change plus VS Code implementation in the debug adapter manager and terminal integration. Close https://github.com/microsoft/vscode/issues/310020 as a duplicate of this issue to consolidate discussion. This issue is the canonical tracker for the feature.
