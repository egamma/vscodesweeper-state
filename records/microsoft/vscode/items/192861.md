# Issue #192861 — [Bug] Opening new editor from a diff editor resets the location

[https://github.com/microsoft/vscode/issues/192861](https://github.com/microsoft/vscode/issues/192861)

## Summary

Still present on main: the DIFF_OPEN_SIDE command opens the editor without passing cursor position/selection options. The issue is correctly labeled as feature-request and routed to diff-editor.

## Change summary

When opening a file from a diff editor via Ctrl+K Shift+O, the opened editor should preserve the cursor position from the diff view instead of jumping to the top.

## Best solution

In `src/vs/workbench/browser/parts/editor/diffEditorCommands.ts`, the `DIFF_OPEN_SIDE` handler should read the cursor position from the focused diff editor side (original or modified) and pass it as `options.selection` to `editorService.openEditor()`.
