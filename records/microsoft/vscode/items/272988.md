# Issue #272988 — Agent handoffs: prompt should not overriden existing chat input text

[https://github.com/microsoft/vscode/issues/272988](https://github.com/microsoft/vscode/issues/272988)

## Summary

Team-authored bug, still reproducible on main. The handoff logic in `executeHandoff` unconditionally overwrites input text via `setValue(promptToUse, false)` instead of appending.

## Change summary

Clicking a handoff option replaces any existing text in the chat input with the handoff prompt instead of appending it.

## Best solution

In `executeHandoff` (chatWidget.ts ~lines 1409 and 1425), check if the input already has content and, if so, append the handoff prompt on a new line rather than replacing.
