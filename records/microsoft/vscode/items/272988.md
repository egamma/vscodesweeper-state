# Issue #272988 — Agent handoffs: prompt should not overriden existing chat input text

[https://github.com/microsoft/vscode/issues/272988](https://github.com/microsoft/vscode/issues/272988)

## Summary

Team-authored issue; the bug is confirmed on current main — executeHandoff uses setValue which overwrites existing chat input text. No fix has landed.

## Change summary

Clicking a handoff option resets the chat input to just the handoff prompt, discarding any text the user had already typed.

## Best solution

In `src/vs/workbench/contrib/chat/browser/widget/chatWidget.ts` `executeHandoff()`, check whether the input already has text before calling `setValue`; if so, append the handoff prompt on a new line rather than replacing.
