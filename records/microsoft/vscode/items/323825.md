# Issue #323825 — Feature Request: Allow telemetry.telemetryLevel to be overridden at workspace scope (or new workspace-scope setting)

[https://github.com/microsoft/vscode/issues/323825](https://github.com/microsoft/vscode/issues/323825)

## Summary

Legitimate feature request for workspace-scoped telemetry control. The setting is intentionally APPLICATION-scoped today (confirmed in source). The proposal is well-reasoned with a monotonic-decrease safety model, but needs product/security team judgment on whether per-workspace telemetry clamping is desirable.

## Change summary

Allow telemetry.telemetryLevel to be overridden at workspace scope so developers on mixed public/regulated projects can disable telemetry per-workspace without toggling the user-level setting.

## Best solution

Route to the telemetry area owner for a product decision. The proposal's option (a) — widening scope to WINDOW with restricted:true and a monotonic-decrease constraint — is technically sound (the setting at `src/vs/platform/telemetry/common/telemetryService.ts` line 309 just needs `ConfigurationScope.WINDOW`), but the design choice to keep it APPLICATION-scoped was intentional and changing it has implications for enterprise policy enforcement and telemetry consistency guarantees. A maintainer should decide whether per-workspace opt-out fits the product's telemetry contract.
