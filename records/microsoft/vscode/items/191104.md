# Issue #191104 — Consider persisting results for quick search

[https://github.com/microsoft/vscode/issues/191104](https://github.com/microsoft/vscode/issues/191104)

## Summary

Team-authored feature request for persisting Quick Search results and position across opens, analogous to quick chat state retention. The existing `search.quickAccess.preserveInput` setting partially addresses this by restoring the query text, but results are re-fetched each time — the full ask (retained results, scroll position) remains unimplemented.

## Change summary

Quick Search should retain its results and selection position across opens, similar to how quick chat preserves its state.

## Best solution

Extend `TextSearchQuickAccess` to cache the last search model and pick-list position so re-opening the picker restores results immediately without re-running the search. The implementation would live in `src/vs/workbench/contrib/search/browser/quickTextSearch/textSearchQuickAccess.ts`, likely gated behind a new setting or extending the existing `search.quickAccess.preserveInput` behavior.
