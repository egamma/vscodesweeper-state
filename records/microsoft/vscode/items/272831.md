# Issue #272831 — Chat response: mix of ✅ in response

[https://github.com/microsoft/vscode/issues/272831](https://github.com/microsoft/vscode/issues/272831)

## Summary

The inconsistent emoji rendering in chat responses is a minor visual papercut. The author (bpasero) noted in December 2025 that it may have improved with thinking content being collapsed, but didn't confirm it fully resolved. Since the author is a CONTRIBUTOR and the issue sits in the Backlog milestone, it should stay open for maintainer judgment.

## Change summary

Chat responses display two visually different styles of ✅ checkmark emoji, creating an inconsistent look.

## Best solution

If the issue persists, the chat response rendering could normalize emoji usage — either by post-processing model output to use consistent emoji variants, or by instructing the model (via system prompt) to use codicons instead of emoji as mrleemurray suggested. The fix would likely touch chat markdown rendering in `src/vs/workbench/contrib/chat/browser/`.
