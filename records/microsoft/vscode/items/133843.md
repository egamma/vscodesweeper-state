## Summary

Valid, distinct feature request acknowledged by a team member (alexdima) — improving nested block comment handling in languages without line-comment tokens (JSX, CSS, HTML). No fix has landed on main; the issue remains relevant.

## Change summary

Toggle Line Comment produces invalid nested block comments in JSX/CSS/HTML; should escape inner comments or use per-line commenting instead.

## Best solution

Implement comment escaping or per-line block commenting in `src/vs/editor/contrib/comment/` so that toggling comments in languages without line-comment tokens handles already-commented regions gracefully (similar to PyCharm's approach). This is distinct from #32541 (toggle without selection) — it's about nested comment correctness.
