# Issue #133843 — Toggle Line Comment can produce invalid block comment in JSX and CSS (should use single-line comments instead)

[https://github.com/microsoft/vscode/issues/133843](https://github.com/microsoft/vscode/issues/133843)

## Summary

Keep open as a Backlog feature request in editor comments. The current implementation still falls back to raw block-comment insertion for languages without line comments, and the maintainer discussion clarified this is distinct from the initially linked related issues.

## Change summary

Improve Toggle Line Comment for JSX, HTML, and CSS so commenting overlapping block-comment ranges does not create prematurely terminated or invalid comments.

## Best solution

Decide whether the editor comment command should support escaping/unescaping nested block-comment delimiters for block-only languages, likely in src/vs/editor/contrib/comment/browser/lineCommentCommand.ts and the editor-comments area. If accepted, add focused tests for JSX tag comments, HTML comments, and CSS block comments with overlapping selections.
