# Issue #124182 — Workspace Trust - Extension Development Host 

[https://github.com/microsoft/vscode/issues/124182](https://github.com/microsoft/vscode/issues/124182)

## Summary

Team-authored issue — must keep open for explicit maintainer judgment. The bug reports that Workspace Trust does not properly govern extension activation in the Extension Development Host: extensions activate regardless of trust declarations, and trust transitions during a debug session fail to re-activate extensions. No fixing commit was found on main; the workspace trust extension enablement logic does not appear to have special handling for the EDH scenario.

## Change summary

Extensions in the Extension Development Host ignore workspace trust declarations — they activate in untrusted workspaces even without the `untrustedWorkspaces` capability, and trust transitions mid-session fail to re-activate them.

## Best solution

The extension enablement service (`src/vs/workbench/services/extensions/common/extensionManifestPropertiesService.ts`) and the extension host manager need to account for the Extension Development Host scenario: respect `untrustedWorkspaces` capability for sideloaded extensions and properly handle trust transitions during a debug session. A second commenter from the Python team confirmed the same behavior, suggesting it remains relevant for extension authors.
