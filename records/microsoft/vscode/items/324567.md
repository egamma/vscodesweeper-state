# Issue #324567 — When sessions.list.showEmptyDefaultGroups is disabled, unclear how to launch chats in agent window

[https://github.com/microsoft/vscode/issues/324567](https://github.com/microsoft/vscode/issues/324567)

## Summary

Team-authored issue (MEMBER) reporting a UX gap: when sessions.list.showEmptyDefaultGroups is disabled and no active chats exist, there's no discoverable way to start a new chat. Keeping open per policy.

## Change summary

Disabling sessions.list.showEmptyDefaultGroups hides the chat section entirely, removing the ability to launch a new chat when no active chats exist.

## Best solution

When the chat group is hidden due to no active chats, provide an alternative entry point (e.g. a persistent 'New Chat' action in the sessions list toolbar or welcome content) so users can always initiate a chat. Likely involves `src/vs/sessions/contrib/sessions/browser/views/sessionsList.ts` and the sessions contribution registration.
