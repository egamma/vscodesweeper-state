# Issue #254350 — Copilot doesn't understand Git renames

[https://github.com/microsoft/vscode/issues/254350](https://github.com/microsoft/vscode/issues/254350)

## Summary

The commit message generation is handled entirely by the Copilot extension (extensions/copilot/src/extension/prompt/node/gitCommitMessageGenerator.ts). VS Code's Git extension correctly tracks renames with INDEX_RENAMED status and renameUri — the issue is in how the Copilot extension presents diff context to the model, making this *caused-by-extension.

## Change summary

Copilot's commit message generation describes renamed/moved files as additions instead of renames.

## Best solution

File this against the Copilot extension (https://github.com/microsoft/vscode-copilot-release) where the commit message prompt construction lives. The Git extension in VS Code core already surfaces rename status correctly via INDEX_RENAMED and renameUri; the fix belongs in gitCommitMessageGenerator.ts or the prompt-building layer in the Copilot extension.
