# Issue #254350 — Copilot doesn't understand Git renames

[https://github.com/microsoft/vscode/issues/254350](https://github.com/microsoft/vscode/issues/254350)

## Summary

This should stay open: current main still appears to omit explicit rename context from Copilot commit-message generation, so a staged rename can be presented to the model like an added file.

## Change summary

Copilot-generated commit messages describe staged file renames or moves as additions instead of renames.

## Best solution

Teach the Copilot git commit-message context to preserve rename metadata, likely by passing status/original/new paths through the prompt and/or generating a rename-aware diff for INDEX_RENAMED changes in the Copilot git diff service.
