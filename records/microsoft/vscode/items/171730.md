# Issue #171730 — "Resolve in Merge Editor" button initially missing

[https://github.com/microsoft/vscode/issues/171730](https://github.com/microsoft/vscode/issues/171730)

## Summary

Still a valid race condition bug: the 'Resolve in Merge Editor' button depends on context keys set by `onDidChangeActiveTextEditor`, which may fire before the repository's merge group is populated or before the document text is fully available. The Oct 2025 fix (c6d8b3ab0bd) refined the when-clause but did not address the underlying timing issue — confirmed by the June 2026 reporter still seeing it.

## Change summary

The 'Resolve in Merge Editor' button fails to appear when first opening a file with merge conflicts due to a race between editor activation and repository state initialization.

## Best solution

Fix the race in `extensions/git/src/model.ts` — when a file with merge conflicts is opened, ensure the context key `git.activeResourceHasMergeConflicts` is re-evaluated after both the repository merge group is populated AND the document text is available (e.g., listen to `onDidChangeTextDocument` or debounce the check).
