# Issue #326523 — Local file attachments do not work in remote Agent Host sessions

[https://github.com/microsoft/vscode/issues/326523](https://github.com/microsoft/vscode/issues/326523)

## Summary

Team-authored bug report with a clear root cause analysis and suggested fix — keeping open for the owning team to address.

## Change summary

Local file attachments fail in remote Agent Host sessions because implicit read grants are not registered for resource attachment URIs in outgoing chat actions.

## Best solution

Extend `RemoteAgentHostProtocolClient._grantImplicitReadsForOutgoingAction` to register implicit read grants for `MessageResourceAttachment` URIs carried by `ChatTurnStarted` and `ChatPendingMessageSet` actions, matching the existing pattern for customization URIs. Add test coverage for the grant-before-read flow including reconnect replay.
