# Issue #130992 — Debugger fails with SyntaxError when using an external directory for TypeScript (outDir)

[https://github.com/microsoft/vscode/issues/130992](https://github.com/microsoft/vscode/issues/130992)

## Summary

Confirmed search API bug: glob patterns containing `../` in `RelativePattern` are not normalized, so `findTextInFiles` fails to match files outside the workspace base. The debugger side was worked around in vscode-js-debug#1486, but the core search API defect in `glob.ts` remains on current main. Keeping open as a confirmed, team-triaged Backlog item.

## Change summary

The search API's `findTextInFiles` does not support relative patterns with `../` segments, causing failures when searching outside the workspace base directory.

## Best solution

Normalize `../` segments in `RelativePattern` by resolving them into the `base` before matching, likely in `src/vs/base/common/glob.ts` (around `parsePattern`/`wrapRelativePattern`). The fix should align with Node.js glob package behavior as connor4312 suggested.
