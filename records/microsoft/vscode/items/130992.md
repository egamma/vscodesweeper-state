# Issue #130992 — Debugger fails with SyntaxError when using an external directory for TypeScript (outDir)

[https://github.com/microsoft/vscode/issues/130992](https://github.com/microsoft/vscode/issues/130992)

## Summary

Confirmed search API bug: RelativePattern does not handle '../' (parent-directory) patterns in `findTextInFiles`, which breaks the JS debugger's source-map resolution when outDir is outside the workspace. The core glob/search issue remains unfixed on main; the debugger side was worked around in vscode-js-debug#1486 but the underlying search API gap persists.

## Change summary

The search API's `findTextInFiles` fails to match files when a RelativePattern's pattern component contains '../' segments pointing outside the workspace base.

## Best solution

Normalize '../' segments in RelativePattern by resolving them into the base URI before passing the pattern to the glob matcher, so that `findTextInFiles` can match files outside the original workspace root. The fix likely belongs in the search extension conversion layer (`src/vs/workbench/services/search/common/searchExtConversionTypes.ts`) or in the RelativePattern handling in `src/vs/base/common/glob.ts`. The debugger-side workaround in vscode-js-debug#1486 mitigates the symptom but does not fix the API contract.
