# Issue #136387 — Git - "Stage Change / Revert Change" hotkey doesn't work

[https://github.com/microsoft/vscode/issues/136387](https://github.com/microsoft/vscode/issues/136387)

## Summary

The `git.stageChange` and `git.revertChange` commands are internal commands that require programmatic arguments (uri, changes array, index) passed by the inline diff widget buttons — they are intentionally hidden from the command palette (`when: false`) and are not designed for keybinding invocation. The user's need is served by `git.stageSelectedRanges` / `git.revertSelectedRanges` and `git.diff.stageHunk`, which work from keyboard shortcuts.

## Change summary

User-assigned keybindings for `git.stageChange` and `git.revertChange` do nothing because the commands require programmatic arguments not available from a keybinding context.

## Best solution

Close as designed — these are internal commands. Users should bind `git.stageSelectedRanges` / `git.revertSelectedRanges` (for range-based staging in diff editors) or `git.diff.stageHunk` (for hunk-based staging) instead. A polish improvement could hide these commands from the Keyboard Shortcuts UI entirely, but that's a separate concern.
