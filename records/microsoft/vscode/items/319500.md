[https://github.com/microsoft/vscode/issues/319500](https://github.com/microsoft/vscode/issues/319500)

## Summary

Valid feature request to hide coverage-related context menu items; no existing setting controls this. Currently in the 60-day community voting period with 2/20 upvotes — let the process run.

## Change summary

Request an option to hide 'Run with Coverage' and related context menu items in the test explorer for users who don't use coverage.

## Best solution

Add a setting (e.g. `testing.coverageActionsEnabled`) that, when false, hides coverage-related menu contributions in the test explorer and gutter decorations. The `when` clauses on the menu registrations in `testExplorerActions.ts` and `testingDecorations.ts` would check this setting via a `config.testing.coverageActionsEnabled` context key.
