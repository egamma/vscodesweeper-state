# Issue #301152 — Add `model` field to Copilot agent hook inputs

[https://github.com/microsoft/vscode/issues/301152](https://github.com/microsoft/vscode/issues/301152)

## Summary

Partially addressed: `model` is already included in the `SessionStartHookInput` but not in the common hook input fields shared by all hook types. The request for a `model` field in every hook's payload (PreToolUse, PostToolUse, UserPromptSubmit, etc.) remains unimplemented. Keeping open as a valid, actionable feature request.

## Change summary

Add the active model identifier to the common input fields sent to all Copilot agent hook types, not just SessionStart.

## Best solution

Add `model` to the `commonInput` object in `extensions/copilot/src/extension/chat/vscode-node/chatHookService.ts` (around line 126) so every hook receives it. The model ID is already available at session start time; threading it through `executeHook` (or storing it on the service instance) would make it available to PreToolUse, PostToolUse, UserPromptSubmit, Stop, etc. This is a small, localized change in the hook service.
