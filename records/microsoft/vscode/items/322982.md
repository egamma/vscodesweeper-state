# Issue #322982 — AH remote access UI is a bit odd

[https://github.com/microsoft/vscode/issues/322982](https://github.com/microsoft/vscode/issues/322982)

## Summary

Team-authored UI polish feedback on the agent-host remote access indicator — active discussion among team members on icon and color changes. Keeping open for the team to land the agreed-upon fixes.

## Change summary

The remote access status-bar indicator has UX issues: it auto-launches sign-in on click, relies solely on color to distinguish states (accessibility concern), and uses yellow for 'on' which reads as a warning.

## Best solution

Switch the active-state color from yellow/dandelion to blue (or accent color), add an icon variant toggle (tower with/without radio bands) so state isn't conveyed by color alone, and gate the sign-in flow behind an explicit user action rather than triggering on first click.
