# Issue #321729 — Workspace becomes invalid after running Copilot CLI

[https://github.com/microsoft/vscode/issues/321729](https://github.com/microsoft/vscode/issues/321729)

## Summary

Still-valid bug: agent-host sessions expose workspace folders with schemes (`vscode-agent-host://`, `unknown:/`) that lack registered file system providers in the classic window, greying out the workspace. A partial mitigation landed (1d2c9c20a35) but the root cause persists. Confirmed by a second user; the `agent-host` label is already correct.

## Change summary

Agent sessions make the workspace appear invalid/grey because file system providers for agent-host URI schemes are not registered in the classic workbench window.

## Best solution

The agent-host file system provider needs to be registered (or proxied) in the classic window when an agent session exposes workspace folders with the `vscode-agent-host://` scheme. The non-git correlation (comment 4) suggests the issue may be in how workspace folder resolution falls back when no SCM context is available. Likely fix area: `src/vs/platform/agentHost/` or `src/vs/sessions/` where workspace folders are propagated to the classic window.
