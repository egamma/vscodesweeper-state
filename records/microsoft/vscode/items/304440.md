# Issue #304440 — Vision model usage in disabled organization breaks the agent session

[https://github.com/microsoft/vscode/issues/304440](https://github.com/microsoft/vscode/issues/304440)

## Summary

Fixed on main: the session-blocking bug was resolved by guarding tool-call images against unsupported vision, and the reporter confirmed sessions are no longer stuck as of 1.115. Further improvements (image placeholders, GA vision) have since shipped.

## Change summary

When vision is disabled by organization policy and the agent model generates a tool call requiring vision, the session becomes permanently stuck with no way to recover.

## Best solution

Keep the shipped fix in commit ab31dc087b9 and the follow-up improvements (4fa2ff42e96, f170fd1f23d). No further change is needed — vision-unsupported images are now replaced with text placeholders instead of blocking the session.
