# Issue #290272 — Skill changes in skill.md don’t apply until VS Code restart or new chat session

[https://github.com/microsoft/vscode/issues/290272](https://github.com/microsoft/vscode/issues/290272)

## Summary

Team-authored issue requesting hot-reload of skill.md changes mid-session. A maintainer confirmed the current behavior is technically as-designed (skills are loaded once via read_file) but expressed interest in revisiting it. Keeping open for product decision.

## Change summary

Editing and saving skill.md during a chat session does not pick up the latest changes until VS Code is restarted or a new session is started.

## Best solution

Implement a file-watcher or invalidation mechanism so that skill.md edits are reflected in subsequent skill invocations within the same chat session, similar to hot-reload in Claude Code or `/skills reload` in Gemini CLI. The decision on UX (automatic vs. explicit reload command) is still under discussion per pwang347's comment.
