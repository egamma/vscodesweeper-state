# Issue #264983 — Hide 0x multiplier for custom models?

[https://github.com/microsoft/vscode/issues/264983](https://github.com/microsoft/vscode/issues/264983)

## Summary

Author is a CONTRIBUTOR (hard keep-open). The bug is still present on current main: the model picker description line and hover show a '0x' multiplier for custom/extension-provided models that don't have meaningful multiplier pricing. The `isMultiplierPricing` check only tests `multiplierNumeric !== undefined` without filtering out zero values or non-Copilot models.

## Change summary

Custom models contributed via the LM API show a meaningless '0x' multiplier in the model picker dropdown, even though other parts of the UI already hide it.

## Best solution

Guard the multiplier display in `chatModelPicker.ts` to suppress the pricing string when `multiplierNumeric` is `0` or when the model is extension-contributed (e.g. `isBYOK`), making the behavior consistent across both the dropdown description line and the hover.
