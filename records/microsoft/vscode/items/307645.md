# Issue #307645 — [Git] Unable to commit during rebase

[https://github.com/microsoft/vscode/issues/307645](https://github.com/microsoft/vscode/issues/307645)

Verified duplicate of #61877 — the exact same problem (VS Code treats commits during interactive rebase as `rebase --continue` instead of allowing normal `git commit`) is tracked there with 12 comments, labeled `bug`/`git`, and in the Backlog.

## Change summary

During an interactive rebase with 'edit' stops, VS Code's Source Control pane runs `git rebase --continue` instead of allowing a normal `git commit`, making commit splitting impossible through the GUI.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/61877 — that canonical issue owns the work to restore proper `git commit` support during interactive rebase edit stops.
