# Issue #121204 — Add a `modifier` flag to the `onDidAccept` event of quick picks

[https://github.com/microsoft/vscode/issues/121204](https://github.com/microsoft/vscode/issues/121204)

## Summary

The internal quick pick already tracks modifier keys (IKeyMods with ctrlCmd/alt/shift) and uses them in the internal pickerQuickAccess framework, but this information is not surfaced to extensions — the extension API onDidAccept remains Event<void>. The request to expose modifiers to extension consumers remains unimplemented.

## Change summary

Expose modifier key state (ctrl, alt, shift) on the onDidAccept event of quick picks in the extension API so extensions can vary behavior based on how an item was accepted.

## Best solution

Add an event payload to the extension API's QuickPick.onDidAccept that includes modifier key state, mirroring the internal IKeyMods already tracked by the quick input widget. This would likely touch src/vscode-dts/vscode.d.ts (or a proposed API first), src/vs/workbench/api/common/extHostQuickOpen.ts, and the main-thread quick-open proxy to forward the modifier state.
