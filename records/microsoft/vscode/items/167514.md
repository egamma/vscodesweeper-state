## Summary

Team-authored feature request that remains unimplemented — the extension host log at extHostExtensionService.ts:480 still does not distinguish implicit from explicit activation events.

## Change summary

The Extension Host output log should indicate when an extension is activated via an implicit (generated) activation event vs. an explicitly declared one.

## Best solution

Enhance the log line in `src/vs/workbench/api/common/extHostExtensionService.ts` (line 480) to include an `(implicit)` annotation when the activation event was generated by `ImplicitActivationEvents` rather than declared in the extension's `package.json`. This requires threading that information through `ExtensionActivationReason`.
