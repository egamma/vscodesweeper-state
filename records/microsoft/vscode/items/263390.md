# Issue #263390 — Chat: Skipped fetch tool response still has some success text

[https://github.com/microsoft/vscode/issues/263390](https://github.com/microsoft/vscode/issues/263390)

## Summary

Still reproduces on main: when the internal fetch tool is skipped, the outer `copilot_fetchWebPage` tool's invoke() treats the skip message as valid web content and wraps it with the 'Here is some relevant context' prefix, contaminating subsequent prompts.

## Change summary

Skipping the fetch tool confirmation still prepends a bogus success message ('Here is some relevant context from the web page...') to the skip response in the prompt sent to the model.

## Best solution

In `extensions/copilot/src/extension/tools/vscode-node/fetchWebPageTool.tsx`, detect when the internal tool result is a skip message (or propagate a skip signal from the tools service) and return only the skip text without the 'Here is some relevant context' wrapper. Alternatively, handle this in `languageModelToolsService.ts` by ensuring the outer tool's invoke() is never called when the internal tool is skipped.
