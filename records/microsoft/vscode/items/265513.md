# Issue #265513 — Folding range breaks minimap highlight

[https://github.com/microsoft/vscode/issues/265513](https://github.com/microsoft/vscode/issues/265513)

## Summary

This appears fixed on current main by a concrete minimap rendering fix after the report was filed. The issue describes minimap inline highlights breaking after folding, and the current code now computes each highlighted minimap line's y-position from that line rather than reusing the decoration start line.

## Change summary

Minimap inline highlights from errors or find results become visually incorrect after toggling a fold when `editor.minimap.size` is `fill`.

## Best solution

Treat this as completed by commit 6a3b90260e379120bdca24d1053cdc5361a76c4a from https://github.com/microsoft/vscode/pull/276370; current minimap rendering should keep inline decoration highlights aligned per line after folding/layout changes.

- **fix commit:** Commit 6a3b90260e379120bdca24d1053cdc5361a76c4a (2025-11-09), merged via https://github.com/microsoft/vscode/pull/276370, changes `src/vs/editor/browser/viewParts/minimap/minimap.ts` so `_renderDecorationsLineHighlights` calls `layout.getYForLineNumber(line, minimapLineHeight)` instead of `layout.getYForLineNumber(startLineNumber, minimapLineHeight)`.
- **current code:** Current main at `src/vs/editor/browser/viewParts/minimap/minimap.ts:1661` loops over every intersecting decoration line and now draws each highlight at that specific line's minimap y-coordinate.
- **affected decorations:** Find matches and marker errors both use minimap inline decorations: `src/vs/editor/contrib/find/browser/findDecorations.ts` registers find minimap decorations with `MinimapPosition.Inline`, and `src/vs/editor/common/services/markerDecorationsService.ts` does the same for info/warning/error markers.
