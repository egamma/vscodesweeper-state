# Issue #252765 — Mouse cursor disappears when I try to hover in the editor and terminal.

[https://github.com/microsoft/vscode/issues/252765](https://github.com/microsoft/vscode/issues/252765)

## Summary

This is an upstream Chromium/GPU-driver cursor rendering issue (tracked at https://issues.chromium.org/issues/40239916) that also reproduces in Edge and other Chromium apps. The commenter on this issue confirms the same. Multiple similar open issues exist (#239422, #236842, #239283). VS Code's only lever is the existing `disable-hardware-acceleration` workaround. Keeping open — a human should consolidate these duplicates and decide whether to track upstream.

## Change summary

Mouse cursor disappears when moving into the VS Code editor/terminal area on certain AMD GPU + driver combinations with hardware acceleration enabled.

## Best solution

Consolidate this with the cluster of similar open reports (#239422, #236842, #239283) into one canonical upstream-tracking issue. The root cause is Chromium bug https://issues.chromium.org/issues/40239916; VS Code cannot fix it in core. The existing `disable-hardware-acceleration` runtime argument is the correct workaround until the Chromium/driver fix lands.
