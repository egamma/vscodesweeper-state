# Issue #154866 — Expand object in debug panel messes up with scroll top

[https://github.com/microsoft/vscode/issues/154866](https://github.com/microsoft/vscode/issues/154866)

## Summary

Still reproduces on main: the debug console's `onDidChangeContentHeight` handler in `repl.ts` scrolls to the bottom whenever a tree node is expanded and the user was previously at or near the bottom (or the content hadn't exceeded the viewport yet). No fix has landed for this issue.

## Change summary

Expanding a large object in the debug console/REPL auto-scrolls to the end of the expanded content instead of keeping the scroll position stable.

## Best solution

As roblourens suggested, the autoscroll-to-bottom behavior should be smarter — ideally moved into the tree/list widget itself, only triggering when new output appends (not when a user-initiated expand changes content height). A localized fix in `repl.ts` could differentiate between content-height changes from new REPL output vs. user-driven expand/collapse and skip the `revealLastElement` call for the latter.
