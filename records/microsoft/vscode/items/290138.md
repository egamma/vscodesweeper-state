# Issue #290138 — Minimap behaves strangely when scrolling past large section of code deleted in chat

[https://github.com/microsoft/vscode/issues/290138](https://github.com/microsoft/vscode/issues/290138)

## Summary

Keep open under discussion: a maintainer confirmed the behavior follows the current minimap/view-zone model, but also described the result as strange and undecided rather than a settled as-designed close.

## Change summary

The minimap viewport indicator jumps unexpectedly while scrolling through chat-deleted code rendered as editor view zones.

## Best solution

Editor minimap/rendering should make an explicit product decision for view-zone-only viewport ranges, likely in `src/vs/editor/browser/viewParts/minimap/minimap.ts`: either represent view-zone gaps in the minimap or keep the nearest real-line indicator stable while scrolling through them.
