# Issue #290138 — Minimap behaves strangely when scrolling past large section of code deleted in chat

[https://github.com/microsoft/vscode/issues/290138](https://github.com/microsoft/vscode/issues/290138)

## Summary

Still unresolved on main — no minimap commits address view zone viewport slider behavior since the issue was filed. The under-discussion label is appropriate as alexdima acknowledged the problem but is uncertain about the right fix.

## Change summary

Minimap viewport slider jumps erratically when scrolling through large view zones (e.g. deleted code in chat) because view zones are not represented in the minimap.

## Best solution

The minimap slider should hold its position stable while the viewport is entirely within view zones, rather than jumping to unrelated lines. The fix likely lives in `src/vs/editor/browser/viewParts/minimap/minimap.ts` in the slider position calculation logic. Needs a design decision from alexdima on how to represent viewport-in-viewzone state.
