# Issue #282485 — Make default color decorator be followed only by string or empty space before determining if its a color

[https://github.com/microsoft/vscode/issues/282485](https://github.com/microsoft/vscode/issues/282485)

## Summary

Team-authored issue (MEMBER); the bug is real and traceable in current source — the hex-color regex in `defaultDocumentColorsComputer.ts` uses `\b` as its trailing anchor, which matches between `#add` and `(` so private TypeScript methods like `#add()` are incorrectly decorated as colors.

## Change summary

The default color decorator's hex regex matches short hex-like tokens (e.g. `#add`) even when followed by non-whitespace/non-quote characters like `(`, causing false-positive color decorations on TypeScript private methods.

## Best solution

Replace the trailing `\b` in the hex-color branches of `initialValidationRegex` (line 104 of `src/vs/editor/common/languages/defaultDocumentColorsComputer.ts`) with a lookahead such as `(?=[\s'";,}\])]|$)` that only accepts whitespace, string delimiters, or end-of-line after the hex digits.
