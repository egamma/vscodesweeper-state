# Issue #201999 — The extensions list should have the ARIA listbox role; not the list role

[https://github.com/microsoft/vscode/issues/201999](https://github.com/microsoft/vscode/issues/201999)

## Summary

Still reproduces on main: the extensions list (and most other lists) defaults to role="list" because no getWidgetRole override is provided. The infrastructure to set role="listbox" exists since 2020 but hasn't been applied broadly.

## Change summary

The extensions list uses ARIA role "list" instead of "listbox", causing aria-activedescendant to be invalid and confusing screen readers like Orca.

## Best solution

Either change the default role in listWidget.ts from 'list' to 'listbox' (as isidorn suggested in comment #2 for all lists that use aria-activedescendant), or add getWidgetRole() returning 'listbox' to each affected accessibility provider. The broader approach (changing the default) is more impactful but needs careful regression testing with screen readers.
