# Issue #226000 — Diff Editor: in-file search should reveal only a small fragment of unchanged region

[https://github.com/microsoft/vscode/issues/226000](https://github.com/microsoft/vscode/issues/226000)

## Summary

Valid feature request still unaddressed on main. The diff editor's `showModifiedLine`/`showOriginalLine` reveals everything from the edge of the hidden region to the target line, rather than just a small context window. No commits since filing have changed this behavior.

## Change summary

When in-file search finds a match inside a collapsed unchanged region in the diff editor, the entire region from one edge to the match is revealed instead of just a small context window around the match.

## Best solution

Modify the reveal logic triggered by cursor/search navigation in `hideUnchangedRegionsFeature.ts` so that when a search match lands in a hidden region, only `contextLineCount` lines around the match are revealed (similar to how the region initially collapses with context). The key files are `src/vs/editor/browser/widget/diffEditor/diffEditorViewModel.ts` (the `showModifiedLine`/`showOriginalLine` methods) and the cursor-change handlers in `hideUnchangedRegionsFeature.ts`.
