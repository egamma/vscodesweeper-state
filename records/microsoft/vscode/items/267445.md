# Issue #267445 — Accessibility/A11y: quickInputBox aria role is textbox when it should be combobox

[https://github.com/microsoft/vscode/issues/267445](https://github.com/microsoft/vscode/issues/267445)

## Summary

The issue requests role="combobox" on the quick input. A recent commit (424ef28d9b7) improved accessibility by dynamically adding aria-haspopup/aria-autocomplete when the list is active, but does not set role="combobox" — so the core request (proper combobox role per WAI-ARIA) remains partially unaddressed. An open PR from the reporter exists; keeping open for maintainer judgment on the correct semantic approach.

## Change summary

The quick input box should announce as a combobox (not textbox) to screen readers when it has list/autocomplete behavior.

## Best solution

Decide whether the input should dynamically switch to role="combobox" when a listbox popup is active (as TylerLeonhardt noted, it's not always a combobox). The open PR #267447 or a revised version that conditionally applies role="combobox" alongside the existing dynamic aria-haspopup/aria-autocomplete in `src/vs/platform/quickinput/browser/quickInputBox.ts` would complete the fix.
