# Issue #205620 — Unable to login using the new Fine-grained PATs, even with all open permissions

[https://github.com/microsoft/vscode/issues/205620](https://github.com/microsoft/vscode/issues/205620)

## Summary

Still valid: fine-grained PATs (PATv2) remain unsupported on current main — the scope validation in flows.ts still reads X-OAuth-Scopes (which fine-grained PATs don't populate the same way) and there's no detection or handling of github_pat_ prefixed tokens. Team member acknowledged the gap and proposed a solution direction.

## Change summary

Support GitHub fine-grained PATs (PATv2) for authentication, or at minimum inform users that only classic PATs are supported.

## Best solution

In `extensions/github-authentication/src/flows.ts`, detect `github_pat_` prefix tokens early, either skip the classic scope validation for them (with a warning about unverifiable permissions) or inform the user that fine-grained PATs are not yet supported. TylerLeonhardt's suggestion of a user-facing message is the pragmatic first step; full PATv2 scope mapping is the long-term fix.
