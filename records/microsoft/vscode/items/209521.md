# Issue #209521 — [Bug]: Incorrect indentation when object nested inline on the same line as parent object

[https://github.com/microsoft/vscode/issues/209521](https://github.com/microsoft/vscode/issues/209521)

## Summary

Team-authored issue (MEMBER) tracking a known limitation in the editor auto-indent regex logic for nested inline objects. Must remain open for maintainer judgment.

## Change summary

Pressing Enter after an inline nested object (e.g. `const a = { b : {},`) does not increase indentation because the indent-increase regex fails to match lines with balanced inner braces.

## Best solution

Enhance the indent-increase regex (or the tree-sitter autoindent path, per the `tree-sitter-autoindent-candidate` label) to correctly detect unclosed braces even when inner braces are balanced on the same line. The author notes recursive regex to depth ~3 as a viable approach for the regex path.
