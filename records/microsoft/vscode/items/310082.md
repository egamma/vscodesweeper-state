# Issue #310082 — Launch VS Code with a specific Profile when opening certain file types from OS file explorer (file association / “Open with”)

[https://github.com/microsoft/vscode/issues/310082](https://github.com/microsoft/vscode/issues/310082)

## Summary

A well-written feature request for file-type-based profile auto-selection from OS file associations. Related but distinct from #231974 (folder-based profile association, closed for lack of upvotes). Should be routed to the user-profiles area for triage consideration.

## Change summary

Allow VS Code to automatically select a profile based on the file type or extension when opened from the OS file explorer.

## Best solution

Route to the user-profiles area owner for evaluation. The `--profile` CLI flag already exists; the gap is a declarative mapping (file pattern → profile) and/or per-profile OS file-association registration. If accepted, implementation would likely touch `src/vs/workbench/services/userDataProfile/` and the platform-specific launch handlers. Alternatively, this could be considered out-of-scope if the team's position is that OS-level integration per profile is too complex to maintain cross-platform.
