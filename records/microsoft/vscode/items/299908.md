# Issue #299908 — Extensions list takes long time to open.

[https://github.com/microsoft/vscode/issues/299908](https://github.com/microsoft/vscode/issues/299908)

## Summary

The reporter traced a ~20s Extensions view delay to a fetch timeout in emergencyAlert, but the emergency alert runs async at WorkbenchPhase.Eventually and shouldn't block the Extensions view. The real cause is likely that all Chromium network requests (including marketplace queries for the Extensions view) time out due to the reporter's macOS IPv6/DNS configuration. Report is 132 days old on VS Code 1.110.1 with no other confirmations — needs verification on latest Insiders and clarification on whether the Extensions view delay persists with marketplace requests specifically.

## Change summary

Opening the Extensions sidebar takes ~20 seconds when IPv6 DNS resolution causes Chromium-based network requests to time out on macOS.

## Best solution

Clarify whether this is an environment-specific IPv6/DNS issue affecting all Chromium network requests on macOS, or a VS Code-specific regression. If it's Chromium's IPv6 handling, this should be tracked upstream (Electron/Chromium) rather than in VS Code core. The emergency alert timeout is a symptom, not the cause of the Extensions view delay — the marketplace data fetch is the likely blocker.
