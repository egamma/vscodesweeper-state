# Issue #123332 — integrated terminal assumes GNU bash is used as user shell, passes --login

[https://github.com/microsoft/vscode/issues/123332](https://github.com/microsoft/vscode/issues/123332)

## Summary

The immediate crash for non-bash/zsh shells was fixed by scoping `--login` to bash/zsh only (commit e70eb682c51), so mksh no longer receives `--login`. However, the broader request — using the proper POSIX mechanism (argv[0] hyphen-prepend) instead of `--login` for login shell signaling — remains unimplemented, blocked on upstream https://github.com/microsoft/node-pty/issues/472.

## Change summary

VS Code should use the POSIX-standard argv[0] hyphen-prepend mechanism to signal login shells instead of the bash-specific `--login` flag.

## Best solution

Once https://github.com/microsoft/node-pty/issues/472 lands (argv[0] control), replace the `--login` argument in `terminalProfileResolverService.ts` with the proper hyphen-prepend mechanism and expose a `loginShell` boolean in terminal profiles. Until then, the workaround (custom profile without `--login`) suffices for non-bash/zsh shells.
