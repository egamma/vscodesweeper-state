# Issue #302460 — MCP Registry URL not retreived although properly set in Enterprise settings (Github Enterprise Managed Users)

[https://github.com/microsoft/vscode/issues/302460](https://github.com/microsoft/vscode/issues/302460)

## Summary

Server-side issue: the GitHub Copilot MCP registry API endpoint is not provisioned for Enterprise Managed Users tenants with EU data residency. The VS Code client correctly attempts the fetch but receives a 404 from GitHub's infrastructure. A team member already triaged it as server-side; the reporter found a workaround via VS Code Enterprise policies.

## Change summary

GitHub's /copilot/mcp_registry API endpoint returns 404 for EMU tenants with EU data residency, preventing MCP server configuration from reaching the VS Code client.

## Best solution

The root fix lives on GitHub's server side (provisioning the MCP registry endpoint for EU EMU tenants). On the VS Code side, the workaround via enterprise policies already works. This should stay open as an upstream tracking issue until GitHub resolves the server-side provisioning gap, or be closed if the team decides the enterprise-policy workaround is sufficient and the server issue belongs entirely in GitHub's backlog.
