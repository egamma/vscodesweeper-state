# Issue #211519 — SCM - actions on highlighted files

[https://github.com/microsoft/vscode/issues/211519](https://github.com/microsoft/vscode/issues/211519)

## Summary

Valid feature request: SCM keyboard actions operate on selection only (not focus), unlike the Explorer. The gap is confirmed on current main — `getSelectedResources()` uses only `tree.getSelection()`, while `getSelectedRepositories()` already uses both focus and selection. Still needs a product decision on whether to align the behavior or add a setting.

## Change summary

SCM view keyboard shortcuts (e.g. Discard Changes, Stage Changes) should act on the focused/highlighted file(s), not only on the selected file whose diff is open.

## Best solution

Align `getSelectedResources()` in `src/vs/workbench/contrib/scm/browser/scmViewPane.ts` to include focused resources (as `getSelectedRepositories()` already does), or add a configuration option. The related issues https://github.com/microsoft/vscode/issues/234488 and https://github.com/microsoft/vscode/issues/136387 cover overlapping ground and could be consolidated.
