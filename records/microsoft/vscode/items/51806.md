## Change summary

Add a word-aware DecorationRangeBehavior that expands a decoration only when word characters are typed at its edges, not when non-word characters are typed.

## Best solution

This would require adding a new TrackedRangeStickiness mode in the editor text buffer that is word-boundary-aware, plus exposing it as a new DecorationRangeBehavior enum value in the extension API. The implementation lives in `src/vs/editor/common/model.ts` (TrackedRangeStickiness) and `src/vscode-dts/vscode.d.ts` (DecorationRangeBehavior). Given the very low demand (2 upvotes in 7 years), this could be left in the backlog indefinitely or closed via the community process — but the request itself is coherent and valid.
