# Issue #319717 — Create PR spinner doesn't stop after PR creation (copilot cli extension host)

[https://github.com/microsoft/vscode/issues/319717](https://github.com/microsoft/vscode/issues/319717)

## Summary

Team-authored bug, keeping open. The spinner is driven by `hasGitOperationInProgress` from session metadata; after PR creation via the copilot CLI extension host, this flag is never reset to false, so the loading indicator persists indefinitely.

## Change summary

The Create PR button spinner in the Agents window Changes view never stops after PR creation via the copilot CLI extension host.

## Best solution

The copilot CLI extension host (or the `copilotChatSessionsProvider`) needs to emit a metadata update with `hasGitOperationInProgress: false` once the PR creation operation completes. The fix likely belongs in the extension code that drives the `createPullRequestCopilotCLIAgentSession.createPR` action, or in how the session provider processes the completion callback.
