# Issue #302689 — Restore previous Ctrl Shift C behavior

[https://github.com/microsoft/vscode/issues/302689](https://github.com/microsoft/vscode/issues/302689)

## Summary

Valid feature request: the multi-root folder picker for Ctrl+Shift+C was intentionally introduced in PR #295073 (implementing #279137), and the reporter wants a setting to restore the previous behavior of always opening at the last active workspace root without a picker.

## Change summary

Request a setting to skip the workspace folder picker when opening an external terminal via Ctrl+Shift+C in multi-root workspaces, restoring the prior behavior.

## Best solution

Add a boolean setting (e.g. `terminal.external.openInLastActiveFolder`) that, when enabled, skips the quick pick in multi-root workspaces and uses `historyService.getLastActiveWorkspaceRoot()` directly — a small guard in `src/vs/workbench/contrib/externalTerminal/electron-browser/externalTerminal.contribution.ts` around the `folders.length > 1` branch.
