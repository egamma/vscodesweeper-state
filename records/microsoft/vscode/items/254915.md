# Issue #254915 — Language Auto-Detection Misclassifies Full HTML File as CSS When Large <style> Block Is Present

[https://github.com/microsoft/vscode/issues/254915](https://github.com/microsoft/vscode/issues/254915)

## Summary

This is a specific case of the known language auto-detection limitation tracked by the canonical meta-issue #131450, which collects samples where the ML model misclassifies content. The detection samples the first 10,000 characters and feeds them to an ML model; when a large CSS block dominates that sample in an HTML file, the model picks CSS. No structural-tag heuristic exists on current main.

## Change summary

Language auto-detection misclassifies an untitled HTML file as CSS when a large inline <style> block dominates the first 10,000 characters.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/131450, which tracks language detection failures and model retraining needs. A fix would require either retraining the ML model with more HTML-with-embedded-CSS samples, or adding a structural heuristic for DOCTYPE/html tags before the ML pass.
