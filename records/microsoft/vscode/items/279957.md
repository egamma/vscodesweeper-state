[https://github.com/microsoft/vscode/issues/279957](https://github.com/microsoft/vscode/issues/279957)

## Summary

Valid tech-debt item: model picker filtering is still hardcoded inside ChatInputPart on current main. The related PR #280081 was closed without merging, and the TODO comment at line 2869 of chatInputPart.ts remains. The debt is real and unresolved — keep open.

## Change summary

Make the model/tool picker filter in ChatInputPart configurable from outside the widget (e.g. via IChatWidgetViewOptions) instead of hardcoding it internally.

## Best solution

Add a model-picker filter callback to `IChatWidgetViewOptions` (as suggested by roblourens) so consumers can define filtering externally. The work lives in `src/vs/workbench/contrib/chat/browser/widget/input/chatInputPart.ts` and `src/vs/workbench/contrib/chat/browser/chat.ts`. The closed PR https://github.com/microsoft/vscode/pull/280081 sketched the approach but was never landed.
