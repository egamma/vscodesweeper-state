# Issue #227592 — [json] Intellisense doesn't suggest objects as a possibility when next to string enums

[https://github.com/microsoft/vscode/issues/227592](https://github.com/microsoft/vscode/issues/227592)

## Summary

Valid feature request to offer object completion snippets in JSON IntelliSense when the schema uses anyOf with both string enums and object types. Still unimplemented on main; the completion logic lives in the upstream vscode-json-languageservice package.

## Change summary

JSON IntelliSense should suggest object completions (e.g. `{ ... }`) alongside string enums when the schema allows both via anyOf.

## Best solution

Implement object-type completion proposals in the vscode-json-languageservice completion provider so that anyOf/oneOf schemas including object alternatives surface a snippet (e.g. `{}`) in the suggestion list. The fix would land in https://github.com/microsoft/vscode-json-languageservice and flow into VS Code via the dependency bump.
