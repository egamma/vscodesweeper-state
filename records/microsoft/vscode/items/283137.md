# Issue #283137 — Option to "pin" attached context files in Copilot Chat for persistent usage

[https://github.com/microsoft/vscode/issues/283137](https://github.com/microsoft/vscode/issues/283137)

## Summary

Valid community-backed feature request (31 upvotes, promoted to Backlog). A maintainer asked for clarification on the underlying problem — the existing behavior keeps prior context in the conversation history, but users want an explicit guarantee that specific files remain actively included. No implementation of persistent/pinned user-attached file context exists on main; the existing 'Pin selection' button in implicitContextAttachment.ts only converts implicit selection context to a regular attachment, which is different from the requested cross-message persistence.

## Change summary

Allow users to pin explicitly attached context files so they persist reliably across consecutive chat messages in the same session.

## Best solution

Clarify the product gap with the community: the maintainer's question (comment 8) about whether prior-turn context already solves this is unanswered. The real ask seems to be ensuring specific files are always included in the LLM context window (not just in conversation history). If validated, add a 'pinned' state to user-attached context entries in the chat input model so they survive across turns, with a visual indicator. Likely touches `src/vs/workbench/contrib/chat/browser/attachments/` and the chat input widget.
