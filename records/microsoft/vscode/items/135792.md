# Issue #135792 — Deleted files from explorer should match styling from deletions on disk

[https://github.com/microsoft/vscode/issues/135792](https://github.com/microsoft/vscode/issues/135792)

## Summary

This issue carries the 'On Deck' milestone, which is a real scheduling decision — deferring to the maintainer who placed it there. The request to align SCM-deleted file tab styling with on-disk-deleted styling remains a valid polish item; the SCM tree already passes `strikethrough` through to labels, but the editor tab treatment for SCM-deleted files may still differ from the on-disk-deleted treatment introduced in #130526.

## Change summary

When a file is deleted via the explorer and opened through SCM, the editor tab shows a plain 'Deleted' label instead of the newer strikethrough decoration treatment used for on-disk deletions.

## Best solution

Align the SCM-deleted file editor tab decoration with the on-disk-deleted styling introduced in #130526 — likely in the editor tab label/decoration plumbing (around `src/vs/workbench/browser/parts/editor/`) so the same strikethrough treatment applies regardless of how the deletion was detected. The `help wanted` label makes this a good community contribution candidate.
