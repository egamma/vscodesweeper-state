# Issue #209123 — VSCode Web - Safari - File drag and drop in explorer panel

[https://github.com/microsoft/vscode/issues/209123](https://github.com/microsoft/vscode/issues/209123)

## Summary

The root cause — drag-image styles were scoped to list-specific suffix selectors and never reached the element when appended to document.body in Safari — was addressed by commit 7248ff1cc24, which moved applyDragImage into ui/dnd and added global default styles on .monaco-drag-image using CSS variables. Shipped in 1.99.0.

## Change summary

In VS Code Web on Safari, drag-and-drop in the explorer shows a blank drag handle instead of the file name pill visible in Chrome.

## Best solution

Keep the shipped fix from commit 7248ff1cc24 (1.99.0). The .monaco-drag-image class now carries global default background-color, color, and outline via CSS variables, so the drag image renders correctly even when appended outside the list's DOM tree. No further change needed.
