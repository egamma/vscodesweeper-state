# Issue #323072 — Problems panel filter status chip blocks input area and makes debugging hard in Chinese locale

[https://github.com/microsoft/vscode/issues/323072](https://github.com/microsoft/vscode/issues/323072)

## Summary

Still reproduces on main: the filter badge in the Problems panel uses a hardcoded 150px right-padding and position:absolute controls container, which causes overlap when the badge text is long (e.g. Chinese locale) and the panel is narrow.

## Change summary

The Problems panel filter badge text overlaps and blocks the input area when using a Chinese locale with a narrow bottom panel width.

## Best solution

Make `adjustInputBox()` in `src/vs/workbench/browser/parts/views/viewFilter.ts` measure the actual badge element width instead of hardcoding 150px, or add `max-width` with `text-overflow: ellipsis` to the `.viewpane-filter-badge` class in `src/vs/workbench/browser/parts/views/media/views.css` so it truncates gracefully when space is limited.
