# Issue #201805 — Minimap view state

[https://github.com/microsoft/vscode/issues/201805](https://github.com/microsoft/vscode/issues/201805)

## Summary

Keep open as an actionable editor-minimap state sync bug; current main still has separate minimap context-menu update code and View/Appearance checked-state wiring, and I found no fixing commit tied to this issue.

## Change summary

Toggling the minimap from the minimap context menu hides it, but the View/Appearance menu still shows Minimap as checked.

## Best solution

Unify the minimap context menu toggle with the workbench `editor.action.toggleMinimap` path, or otherwise ensure `config.editor.minimap.enabled` updates the MenubarAppearanceMenu checked state immediately after the minimap context menu changes `editor.minimap.enabled`.
