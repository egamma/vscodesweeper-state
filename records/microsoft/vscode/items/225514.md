# Issue #225514 — History graph diff always shows horizontal scroll bar

[https://github.com/microsoft/vscode/issues/225514](https://github.com/microsoft/vscode/issues/225514)

## Summary

Still reproducible on main: the multi-diff-editor's outer scrollable element shows a spurious horizontal scrollbar in inline diff mode because `maxScroll` can be slightly positive even when there's no real horizontal overflow to scroll.

## Change summary

The multi-diff-editor (history graph diff, incoming/outgoing commits) always shows a horizontal scrollbar that does nothing when using inline diff mode.

## Best solution

Guard the `scrollWidth` calculation in `multiDiffEditorWidgetImpl.ts` (around line 237) so that a `maxScroll` value of 0 or near-zero doesn't inflate `scrollWidth` beyond the viewport width, or ensure the inline diff editor's content width is reported accurately to avoid the off-by-one that triggers `ScrollbarVisibility.Auto`.
