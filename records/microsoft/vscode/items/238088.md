# Issue #238088 — Terminal GPU acceletion causing GPU page faults on AMD (hangs, soft-timeouts and resets)

[https://github.com/microsoft/vscode/issues/238088](https://github.com/microsoft/vscode/issues/238088)

## Summary

Confirmed upstream issue: GPU page faults in the amdgpu kernel driver triggered by WebGL rendering in Chromium/Electron's GPU process. Multiple users reproduce across RDNA 3/3.5/4 GPUs on Linux. Already correctly labeled `upstream` with a known workaround. Should stay open to track upstream progress.

## Change summary

Terminal GPU acceleration via WebGL causes amdgpu page faults, hangs, and soft-resets on AMD Radeon GPUs under Linux.

## Best solution

Track upstream fixes in the amdgpu kernel driver, Mesa radeonsi, and Chromium's WebGL/ANGLE implementation. If a specific Chromium bug is identified, link it here. The existing workaround (`terminal.integrated.gpuAcceleration: "off"`) is correct. VS Code could potentially detect AMD GPUs on Linux and auto-disable WebGL for the terminal, but that would be a separate feature decision in `src/vs/workbench/contrib/terminal/browser/xterm/xtermTerminal.ts`.

## Evidence

- **multiple confirmations:** Reproduced by 5+ users across AMD 760M, 780M, 890M, and RX 7900 XTX GPUs on various Linux distros (NixOS, Fedora, Alpine, Zorin/Ubuntu), confirming it spans RDNA 3/3.5/4 and is not iGPU-specific.
- **upstream root cause:** The kernel logs show amdgpu gfxhub page faults (SQC data client, PERMISSION_FAULTS:0x3) in the GPU's compute shader queue used by Chromium's WebGL process — this is a GPU driver/firmware-level fault, not a VS Code code defect.
- **team awareness:** VS Code team member @Tyriar suggested WebGL2 may be enabled where it shouldn't be (comment 3), and @deepak1556 asked about Chromium browser reproduction (comment 7) — team is aware and tracking.
- **related firmware issue:** https://bugzilla.redhat.com/show_bug.cgi?id=2420062 — a related amdgpu firmware revert was suggested for RDNA 3.5 devices, indicating the root cause spans GPU firmware and kernel driver.
- **workaround exists:** Setting `terminal.integrated.gpuAcceleration: "off"` disables WebGL rendering and avoids the GPU faults entirely.
- **WebGL addon code:** The terminal WebGL rendering path is in src/vs/workbench/contrib/terminal/browser/xterm/xtermTerminal.ts via the @xterm/addon-webgl addon — VS Code loads it but the rendering itself is handled by xterm.js and Chromium's GPU process.
