# Issue #271138 — Link MCP servers with Marketplace extensions

[https://github.com/microsoft/vscode/issues/271138](https://github.com/microsoft/vscode/issues/271138)

## Summary

Team-authored feature request in Backlog milestone tracking deduplication between MCP marketplace servers and VS Code Marketplace extensions. Kept open per hard keep-open rule (author is MEMBER).

## Change summary

Detect when an MCP server from the MCP marketplace duplicates functionality already provided by an installed VS Code extension, and let users enable it from only one source.

## Best solution

Design a linking/matching mechanism between MCP server entries and Marketplace extensions (likely in the MCP management layer under `src/vs/workbench/contrib/chat/browser/` or the MCP service). Needs a product decision on matching criteria (command-based vs. explicit metadata) given concerns raised about extensions providing additional flags beyond what a standalone MCP server offers.
