# Issue #253260 — Perf: changeLineHeights leads to long tasks on a decently sized file

[https://github.com/microsoft/vscode/issues/253260](https://github.com/microsoft/vscode/issues/253260)

## Summary

Team-authored perf issue about changeLineHeights blocking the main thread on large files. Still under discussion with no shipped fix specifically targeting this critical-path concern; recent linesLayout.ts work (queuing, batching) addresses related areas but the core question of offloading to a worker remains open.

## Change summary

Opening a large file with many special-font lines causes the editor to freeze for several seconds due to expensive synchronous changeLineHeights/commit calls on the critical path.

## Best solution

Either offload the initial changeLineHeights computation to a worker thread, or restructure the commit path in `src/vs/editor/common/viewLayout/linesLayout.ts` to batch/defer work so it doesn't block the main thread on first render of large files.
