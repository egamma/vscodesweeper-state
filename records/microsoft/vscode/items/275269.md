# Issue #275269 — `LanguageModelDataPart.json` emits a surprising mimetype

[https://github.com/microsoft/vscode/issues/275269](https://github.com/microsoft/vscode/issues/275269)

## Summary

Team-authored issue; the bug is confirmed on current main — `text/x-json` is still the default mime in `LanguageModelDataPart.json`. Keeping open per policy.

## Change summary

`LanguageModelDataPart.json()` defaults to the legacy `text/x-json` MIME type instead of the IANA-standard `application/json`.

## Best solution

Change the default mime from `'text/x-json'` to `'application/json'` in `LanguageModelDataPart.json()` and the corresponding `NotebookCellOutputItem.json()` helpers across `src/vs/workbench/api/common/extHostTypes.ts`, `src/vs/workbench/api/common/extHostTypes/notebooks.ts`, and the copilot extension copy, then update the three assertions in `extHostTypes.test.ts`.
