# Issue #264737 — Terminal suggestions feel slow to appear over remote

[https://github.com/microsoft/vscode/issues/264737](https://github.com/microsoft/vscode/issues/264737)

## Summary

Team-authored performance bug about terminal suggestions latency over remote connections. A Windows-specific perf fix was mentioned but the reporter confirmed they were not on Windows, so the issue likely remains open for non-Windows remote scenarios.

## Change summary

Terminal suggestions have a noticeable delay when working in a remote window compared to local, despite other remote operations feeling snappy.

## Best solution

Investigate the roundtrip path for terminal completions over remote — the trace logs show ~200ms for the completion provider plus ~50ms for resource resolution, which compounds over network latency. Consider caching completion results or prefetching on the remote side to reduce perceived latency.
