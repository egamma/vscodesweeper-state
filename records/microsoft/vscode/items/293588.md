# Issue #293588 — Feature request: Enhance the GitHub Copilot integration to allow users to spawn multiple simultaneous Chat/CLI instances directly into the editor area with automated, intelligent layout distribution.

[https://github.com/microsoft/vscode/issues/293588](https://github.com/microsoft/vscode/issues/293588)

## Summary

The core need — multiple simultaneous AI sessions with automated layout — is now substantially addressed by the Agents Window, which provides a dedicated multi-session surface with an internal grid of session views. The specific request for editor-area layout distribution is a different UX approach; keeping open for a maintainer to decide whether the Agents Window fully supersedes this or whether editor-area multi-chat remains a gap.

## Change summary

Allow users to spawn N simultaneous Copilot Chat/CLI instances in the editor area with automated grid layout distribution (side-by-side, 2x2, etc.).

## Best solution

The Agents Window (`src/vs/sessions/`) already delivers concurrent multi-session chat with an internal horizontal grid and layout management. A maintainer should evaluate whether the Agents Window fully supersedes this request (close as `*as-designed`) or whether editor-area multi-chat embedding remains a distinct, desired capability. If the latter, the workbench's existing editor group split APIs could power the layout distribution.
