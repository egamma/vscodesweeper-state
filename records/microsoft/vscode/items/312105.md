# Issue #312105 — fix(auth): OAuth2 redirect fails when "code" is the first query parameter

[https://github.com/microsoft/vscode/issues/312105](https://github.com/microsoft/vscode/issues/312105)

## Summary

Confirmed on current main: the regex at line 741 of extHostAuthentication.ts requires a leading `?` or `&` before `code=`, but URI.query never includes the leading `?`, so when `code` is the first parameter the match fails.

## Change summary

OAuth2 authorization code extraction fails when `code` is the first query parameter because the regex doesn't handle start-of-string.

## Best solution

Fix the regex on line 741 of `src/vs/workbench/api/common/extHostAuthentication.ts` to also match at the start of the string, e.g. `/(?:^|[?&])code=([^&]+)/`. Add a unit test covering `code` as the first and non-first query parameter.
