# Issue #276240 — When Copilot Code Review (CCR) no-ops, there's no notice to user as to why

[https://github.com/microsoft/vscode/issues/276240](https://github.com/microsoft/vscode/issues/276240)

## Summary

Team-authored bug report about missing user feedback when Copilot Code Review silently no-ops. The related fix PR (#296444) was closed without merging, so the issue remains unaddressed on main. Keeping open per team-author policy.

## Change summary

When Copilot Code Review is invoked without a selection (or with a selection that yields no reviewable content), nothing happens and the user receives no feedback explaining why.

## Best solution

Show a notification or information message when CCR determines there is nothing to review — either because no text is selected or the selection yields no actionable content. The fix likely belongs in the code review action handler within the chat-review contribution area. The closed PR https://github.com/microsoft/vscode/pull/296444 attempted this approach and may serve as a reference.
