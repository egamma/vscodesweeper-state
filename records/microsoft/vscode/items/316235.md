# Issue #316235 — Feature Request - Ability to specify OAuth scopes in the MCP config for static client registration

[https://github.com/microsoft/vscode/issues/316235](https://github.com/microsoft/vscode/issues/316235)

## Summary

Valid feature request: the MCP OAuth flow currently sends all `scopes_supported` from the resource metadata, with no way for users to restrict which scopes are requested. The `McpServerTransportHTTPOAuth` interface only has `clientId` and `enterpriseManaged` — no `scopes` field. Other MCP clients already support this. Routing to the appropriate area.

## Change summary

Allow users to specify a subset of OAuth scopes in the MCP server configuration for static client registration, so the authorization request only includes the desired scopes rather than all `scopes_supported`.

## Best solution

Add an optional `scopes` array to the `McpServerTransportHTTPOAuth` interface and the mcp.json schema (in `src/vs/workbench/contrib/mcp/common/mcpConfiguration.ts`), then pass it through to the `createAuthMetadata` flow in `src/vs/workbench/api/common/extHostMcp.ts` so that configured scopes override the discovered `scopes_supported`. This is a small localized change to the config schema and auth metadata initialization.
