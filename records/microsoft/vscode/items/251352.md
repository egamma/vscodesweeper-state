# Issue #251352 — Bug: Profile Extensions UI incorrectly displays extensions after switching source under that profile.

[https://github.com/microsoft/vscode/issues/251352](https://github.com/microsoft/vscode/issues/251352)

## Summary

Still reproduces on main: `UserDataProfileElement.getChildrenForResourceType` always fetches extensions from `this.profile` without checking `this.getFlag(resourceType)`, so switching the Extensions source radio to "Default" doesn't update the displayed list.

## Change summary

In the Profiles editor UI, switching the Extensions source between "Default" and the current profile does not correctly update the displayed extension list.

## Best solution

In `UserDataProfileElement.getChildrenForResourceType` (line ~519 of `userDataProfilesEditorModel.ts`), add a `this.getFlag(resourceType)` check before fetching extensions — when the flag is set, use `this.userDataProfilesService.defaultProfile` instead of `this.profile` as the source, mirroring the logic in `getChildrenFromProfile`.
