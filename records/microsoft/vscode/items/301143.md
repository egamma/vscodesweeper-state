[https://github.com/microsoft/vscode/issues/301143](https://github.com/microsoft/vscode/issues/301143)

## Summary

This requests a read-only event for observing tool-approval prompts from extensions — a strict subset of the API surface proposed in #302362, which the team already closed as *out-of-scope. The narrower read-only variant has sat 127 days with zero upvotes and no team engagement, and the precedent from the broader request applies.

## Change summary

Expose an extension API event that fires when the chat UI presents a tool-approval dialog, so notification extensions can alert users.

## Best solution

Keep the current behavior where tool-approval lifecycle is internal to the chat subsystem. The team already declined exposing this surface in https://github.com/microsoft/vscode/issues/302362; the read-only observation subset does not change the scope calculus. Extensions like Copilot Chime can use existing signals (terminal execution events, chat response events) for partial coverage.
