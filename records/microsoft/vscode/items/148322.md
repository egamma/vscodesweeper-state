# Issue #148322 — Git - Out of workspace diff does not survive a reload

[https://github.com/microsoft/vscode/issues/148322](https://github.com/microsoft/vscode/issues/148322)

## Summary

Still reproducible on main: the git file-system provider's `getOrOpenRepository` only auto-opens repositories in the empty-window or agent-sessions-window case, not when an out-of-workspace file is opened in a window that already has a workspace. The 'On Deck' milestone also prevents a close proposal.

## Change summary

Git diff editors for files outside the workspace fail to restore after a window reload because the repository is not rediscovered on startup.

## Best solution

Extend `getOrOpenRepository` in `extensions/git/src/fileSystemProvider.ts` to also attempt repository discovery when the file's path is outside all open workspace folders — not just when `workspaceFolders` is undefined. The existing `openRepository` call (line 148) is the right mechanism; the guard on line 144 just needs to also cover the out-of-workspace case.
