# Issue #244880 — Bug with auto-indent rule: "onEnterRules" + "indentOutdent" + "AppendText" do not insert indentation and string at the END as expected.

[https://github.com/microsoft/vscode/issues/244880](https://github.com/microsoft/vscode/issues/244880)

## Summary

Confirmed on current main: when `indentAction` is `IndentOutdent` and `appendText` is provided, the tab character is not prepended to `appendText` (line 50 of enterAction.ts only checks for `Indent`). The bug persists unfixed.

## Change summary

Using `onEnterRules` with `indentOutdent` + `appendText` fails to prepend indentation to the appended text, placing it at column 0 instead of after the indent.

## Best solution

Add `|| (indentAction === IndentAction.IndentOutdent)` to the `else if` condition at line 50 of `src/vs/editor/common/languages/enterAction.ts`, so that `appendText` gets a tab prepended for `IndentOutdent` just as it does for `Indent`. Add a unit test covering this combination.
