# Issue #139485 — Cannot show full file path for long symbol signature in Go To Symbol

[https://github.com/microsoft/vscode/issues/139485](https://github.com/microsoft/vscode/issues/139485)

## Summary

Still reproduces on main: workspace symbol picks in `symbolsQuickAccess.ts` don't set a `tooltip` on items, so the row-level hover in `quickInputList.ts` (which requires `saneTooltip`) never fires; when a long label hides the description element, there's no way to see the file path.

## Change summary

In Go to Symbol in Workspace (Ctrl+T), when a symbol's signature is too long, the file path description is clipped and there is no hover to reveal it.

## Best solution

Set a `tooltip` on workspace symbol quick pick items (in `src/vs/workbench/contrib/search/browser/symbolsQuickAccess.ts`) containing the container/file path, so the existing row-level hover in `quickInputList.ts` fires and reveals the full path even when the description element is pushed offscreen by a long label.
