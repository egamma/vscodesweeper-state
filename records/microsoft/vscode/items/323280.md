# Issue #323280 — Clicking links doesn't work in the markdown editor

[https://github.com/microsoft/vscode/issues/323280](https://github.com/microsoft/vscode/issues/323280)

## Summary

Team-authored issue; the markdown editor webview has no link-click handling wired up — the EditorView likely emits link-open events but neither the webview-side editor.ts nor the extension-side markdownEditorProvider.ts forwards them to the VS Code URI opener.

## Change summary

Clicking links in the WYSIWYG markdown editor does nothing until after an alt-tab round-trip.

## Best solution

Wire a link-open callback in the EditorView options (or listen for an openLink message from `@vscode/markdown-editor`) in `extensions/markdown-language-features/markdown-editor-src/editor.ts`, post it to the extension host, and handle it in `markdownEditorProvider.ts` by calling `vscode.env.openExternal` or `vscode.commands.executeCommand('vscode.open', …)`. The alt-tab workaround suggests a focus/activation gating issue in the webview's click dispatch that should also be investigated.
