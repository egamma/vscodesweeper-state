# Issue #298401 — After updating to the latest version of VS Code, the coding process has become extremely laggy, and memory usage spikes dramatically.

[https://github.com/microsoft/vscode/issues/298401](https://github.com/microsoft/vscode/issues/298401)

## Summary

Root cause was the C/C++ extension calling `selectChatModels` on every hover, causing high CPU and memory in the remote extension host. The cpptools team identified and fixed this in their PR #14327 (merged 2026-03-30). The reporter confirmed the workaround resolves the issue.

## Change summary

After updating VS Code, coding in Remote-WSL with C/C++ and Copilot extensions caused severe lag and memory spikes due to excessive `selectChatModels` API calls on every hover.

## Best solution

The fix shipped in the C/C++ extension via https://github.com/microsoft/vscode-cpptools/pull/14327 (merged 2026-03-30), which minimizes calls to `lm.selectChatModels`. Users should update to C/C++ extension 1.32.0+. The cpptools team noted the `selectChatModels` API could be more resilient to frequent calls, but the primary fix is in the extension.
