# Issue #170314 — Hidden Menu Bar Appears After Fast Scrolling

[https://github.com/microsoft/vscode/issues/170314](https://github.com/microsoft/vscode/issues/170314)

## Summary

Confirmed upstream Electron/Chromium issue: Alt key release during fast-scroll triggers native menu bar focus via Chromium's focus manager. Already correctly labeled and triaged by team members; no VS Code-side fix is feasible — this needs to be addressed in Electron's menu bar key handling.

## Change summary

On Linux with native title bar, releasing Alt after fast-scrolling (Alt+scroll) causes the hidden menu bar to appear, because Chromium's focus manager interprets the Alt key-up as a menu-bar activation.

## Best solution

Track upstream in Electron — the root cause is in Chromium's focus manager callback for Alt key release (https://github.com/electron/electron/blob/644243efd61b75c22707605d9d0dbc6ade4fe458/shell/browser/ui/views/menu_bar.cc#L133-L157). VS Code could file or link an Electron issue requesting that Alt key-up is suppressed when it was consumed as a modifier during scrolling. No VS Code-side code change can fix this.
