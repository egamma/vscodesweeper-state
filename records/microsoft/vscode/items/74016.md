# Issue #74016 — Expose vscode remote reconnecting UI.

[https://github.com/microsoft/vscode/issues/74016](https://github.com/microsoft/vscode/issues/74016)

## Summary

A 6-year-old feature request with zero community upvotes and a contributor explicitly pushing back on the core concept of letting extensions block the UI. Still technically open for the community process, but unlikely to gain traction in its current form.

## Change summary

Expose VS Code's remote reconnecting UI as an extension API so extensions like Live Share can block the UI during their own network reconnection.

## Best solution

This likely stays unimplemented as described — blocking the UI from extensions conflicts with VS Code's values around lightweight, responsive UX. If a reconnection UX is needed, extensions should use their own non-blocking notification patterns (e.g., status bar items, notifications). The community process can retire this if no further interest materializes.
