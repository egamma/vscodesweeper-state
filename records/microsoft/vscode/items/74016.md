# Issue #74016 — Expose vscode remote reconnecting UI.

[https://github.com/microsoft/vscode/issues/74016](https://github.com/microsoft/vscode/issues/74016)

## Summary

This is an old API feature request asking whether extensions should be able to reuse the Remote reconnecting/blocking UI. The only maintainer signal in the thread is pushback against allowing extensions to block the workbench UI, but that is not enough by itself to propose a policy close.

## Change summary

Expose an extension API that lets Live Share-like extensions show or reuse VS Code's Remote "Reconnecting" UI while network state is being restored.

## Best solution

API owners should make an explicit product/API decision: either design a constrained non-blocking connection-state surface for extensions, or close as out of scope if extensions must not be able to block the workbench during reconnect.
