# Issue #128527 — editor.scrollbar.scrollByPage doesn't work correctly when Option-clicking (Mac)

[https://github.com/microsoft/vscode/issues/128527](https://github.com/microsoft/vscode/issues/128527)

## Summary

Keep open: the requested macOS Option-click behavior does not appear to be implemented in the shared scrollbar code, and the issue is already classified as a Backlog feature request in the editor scrollbar area.

## Change summary

Option-clicking the editor scrollbar track on macOS should jump the thumb to the clicked position even when `editor.scrollbar.scrollByPage` is enabled.

## Best solution

Decide whether the shared scrollbar should special-case macOS `altKey` track clicks in `src/vs/base/browser/ui/scrollbar/abstractScrollbar.ts` so `scrollByPage` remains the default click behavior but Option-click jumps to the absolute offset. The Keyboard Shortcuts scrollbar note in the follow-up comment should be split or separately verified if still reproducible.

- **code path:** `src/vs/editor/browser/viewParts/editorScrollbar/editorScrollbar.ts` passes `editor.scrollbar.scrollByPage` through to `SmoothScrollableElement`, so editor scrollbar track-click behavior is owned by the shared scrollbar implementation.
- **current implementation:** `src/vs/base/browser/ui/scrollbar/abstractScrollbar.ts` `_onPointerDown` chooses between `getDesiredScrollPositionFromOffsetPaged(offset)` and `getDesiredScrollPositionFromOffset(offset)` solely from `_scrollByPage`; it does not inspect `e.altKey` for Option-click track jumps.
- **related behavior:** `src/vs/base/browser/ui/scrollbar/scrollableElement.ts` uses `browserEvent.altKey` for mouse-wheel fast scrolling, not for scrollbar track pointer-down behavior.
