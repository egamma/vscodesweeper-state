# Issue #128527 — editor.scrollbar.scrollByPage doesn't work correctly when Option-clicking (Mac)

[https://github.com/microsoft/vscode/issues/128527](https://github.com/microsoft/vscode/issues/128527)

## Summary

Valid feature request for Mac-native scrollbar behavior (Option-click to invert scrollByPage). Still unimplemented on current main — the scrollbar click handler in abstractScrollbar.ts has no altKey check. Low community traction (0 upvotes) but correctly labeled and in Backlog; keeping open.

## Change summary

Option-clicking the scrollbar track on Mac should jump the thumb to the clicked position when scrollByPage is enabled (inverting the default page-scroll behavior), matching native macOS conventions.

## Best solution

Add an `e.altKey` (Option on Mac) check in `_onPointerDown` in `src/vs/base/browser/ui/scrollbar/abstractScrollbar.ts` that inverts the `scrollByPage` logic — when altKey is held, use jump-to-offset if scrollByPage is true, and page-scroll if it's false. This matches native macOS scrollbar behavior.
