# Issue #128527 — editor.scrollbar.scrollByPage doesn't work correctly when Option-clicking (Mac)

[https://github.com/microsoft/vscode/issues/128527](https://github.com/microsoft/vscode/issues/128527)

## Summary

Keep open: this is already classified as a Backlog feature request and has an assigned milestone, which is a hard keep-open condition. Current scrollbar code still appears to route track clicks through the `scrollByPage` setting without a macOS Option-click exception, so the requested behavior remains a real editor-scrollbar gap.

## Change summary

On macOS, Option-clicking the editor scrollbar track should jump the thumb to the clicked position even when `editor.scrollbar.scrollByPage` is enabled.

## Best solution

If the editor team wants native macOS parity here, handle `altKey`/Option-click in `src/vs/base/browser/ui/scrollbar/abstractScrollbar.ts` so track clicks can use absolute positioning while ordinary clicks continue to honor `scrollByPage`; also decide whether the same shared scrollbar behavior should apply outside the text editor, such as Keyboard Shortcuts.
