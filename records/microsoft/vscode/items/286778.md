# Issue #286778 — (Windows 2026) Excessive rg.exe proccesses spawning when opening any folder(even if empty) whose root has brackets, and is located in Users folder.

[https://github.com/microsoft/vscode/issues/286778](https://github.com/microsoft/vscode/issues/286778)

## Summary

Verified duplicate: the root cause (ripgrep misinterpreting brackets in folder paths as glob characters, causing overly broad searches) is the same bug tracked by #282852, which is open with 'bug' and 'search' labels. This issue adds the extension-triggered variant but the underlying fix is the same.

## Change summary

Opening a workspace folder whose path contains square brackets causes ripgrep to interpret brackets as glob characters, searching from a parent directory and spawning excessive rg.exe processes.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/282852 — fixing glob-escaping of folder paths in `src/vs/workbench/services/search/node/ripgrepFileSearch.ts` (specifically `getAbsoluteGlob`) resolves both the direct search case and the `workspace.findFiles()` extension API path.
