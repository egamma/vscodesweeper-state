# Issue #310863 — [Bug] Enabling "GitHub MCP Server: Enabled" automatically writes entry to mcp.json causing duplicate tool names

[https://github.com/microsoft/vscode/issues/310863](https://github.com/microsoft/vscode/issues/310863)

## Summary

The issue involves the Copilot extension's 'GitHub MCP Server: Enabled' setting causing duplicate tool registrations that break Claude model requests. VS Code core has alias-mapping infrastructure (githubMCPServerAliases in languageModelToolsService.ts) to normalize GitHub MCP server names, but the deduplication may not fully prevent duplicate tool names from being sent to the model API. The setting and its mcp.json-writing behavior are owned by the Copilot extension.

## Change summary

Enabling the 'GitHub MCP Server: Enabled' Copilot setting causes the GitHub MCP Server to be registered twice (natively and via mcp.json), producing duplicate tool names that fail Claude API requests with a 400 error.

## Best solution

The Copilot extension should either stop writing to mcp.json when the native setting is enabled, or the tool deduplication in `src/vs/workbench/contrib/chat/browser/tools/languageModelToolsService.ts` (which already maps `io.github.github/github-mcp-server` → `github`) needs to suppress duplicate tool names before sending them to model APIs. The fix likely spans both the Copilot extension and the core tool service.
