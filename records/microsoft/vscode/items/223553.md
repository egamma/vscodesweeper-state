# Issue #223553 — Node process in terminal not killed when VSCode is closed (Windows)

[https://github.com/microsoft/vscode/issues/223553](https://github.com/microsoft/vscode/issues/223553)

## Summary

This is the canonical issue for terminal child processes not being killed on Windows when the last VS Code window closes. Multiple other issues (#232384, #234986, #308450) have been closed as duplicates of this one. The bug remains unresolved on current main — the graceful-kill setting attempt (commit 0e566b9b075) was reverted (75ea5730ccc). Keep open as the active tracker.

## Change summary

Node child processes spawned in the integrated terminal survive after the last VS Code window is closed on Windows, leaving orphaned processes.

## Best solution

Fix the terminal process tree cleanup on Windows when the last window shuts down. The `shutdown()` method in `src/vs/platform/terminal/node/terminalProcess.ts` intentionally avoids immediate kill on Windows (mitigation for #71966/conpty unresponsiveness), but the deferred kill path may not execute when the last window closes and the pty host process exits. A reliable process-tree teardown for the last-window-close path is needed.
