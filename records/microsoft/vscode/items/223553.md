# Issue #223553 — Node process in terminal not killed when VSCode is closed (Windows)

[https://github.com/microsoft/vscode/issues/223553](https://github.com/microsoft/vscode/issues/223553)

## Summary

This is the canonical issue for terminal child processes surviving VS Code shutdown on Windows — at least three other issues (#232384, #234986, #308450) have been duped to it. No targeted fix has landed on main; the graceful-kill setting (0e566b9b075) is opt-in and addresses a different use case. Keeping open as the active tracker.

## Change summary

Node child processes spawned in the integrated terminal are not killed when the last VS Code window closes on Windows, leaving orphan processes that hold ports.

## Best solution

Investigate the terminal shutdown path in `terminalService.ts` (the `ShutdownReason.CLOSE` / last-window branch) to ensure child process trees are terminated on Windows when the final window closes. The `confirmation-pending` label suggests the team needs a reliable reproduction — the ARM/Snapdragon detail may be a red herring given x64 reports in duplicates.
