# Issue #198146 — File explorer performance degrades when the number of files in workspaces is large

[https://github.com/microsoft/vscode/issues/198146](https://github.com/microsoft/vscode/issues/198146)

## Summary

Team-authored issue (MEMBER); the explorer still has no pagination or child-count limit for large directories, and community comments from 2025 confirm the problem persists on current builds. Keeping open.

## Change summary

File explorer hangs or becomes very slow when a workspace contains directories with tens or hundreds of thousands of files.

## Best solution

Introduce a cap on the number of children loaded per directory in the explorer tree (e.g. lazy/paginated expansion), likely in `src/vs/workbench/contrib/files/browser/views/explorerView.ts` and `src/vs/workbench/contrib/files/common/explorerModel.ts`. The tree data source should limit initial child enumeration and offer a 'Show more…' node, similar to how other large-list UIs in VS Code handle unbounded data.
