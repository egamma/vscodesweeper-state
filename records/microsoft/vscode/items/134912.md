# Issue #134912 — Test API: Cursor should be placed prior to inline error message

[https://github.com/microsoft/vscode/issues/134912](https://github.com/microsoft/vscode/issues/134912)

## Summary

Still present on main: the test inline error message decoration does not set `cursorStops`, so the cursor can land after the injected text. The `InjectedTextCursorStops` API exists and is already used in the same contrib (code coverage decorations use `None`), so the fix is trivial.

## Change summary

Cursor can be placed after inline test error messages, which is inconsistent with other injected-text decorations that suppress cursor stops.

## Best solution

Add `cursorStops: InjectedTextCursorStops.None` to the `options.after` block in `src/vs/workbench/contrib/testing/browser/testingDecorations.ts` (~line 1304), matching how code coverage decorations already handle this.
