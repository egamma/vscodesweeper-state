# Issue #33449 — [folding] Automatic function/method folding

[https://github.com/microsoft/vscode/issues/33449](https://github.com/microsoft/vscode/issues/33449)

## Summary

This is a legitimate, accepted feature request (Backlog milestone) for automatic fold-all-but-current-function behavior. No implementation exists on main — folding only unfolds when the cursor enters a hidden region but never auto-folds regions the cursor leaves. Community interest remains active.

## Change summary

Request to automatically fold all functions except the one containing the cursor, expanding/collapsing as the cursor moves between functions.

## Best solution

If pursued, add an editor option (e.g. `editor.foldingAutoCollapse`) that keeps all top-level folding regions collapsed except the one containing the cursor, implemented in `src/vs/editor/contrib/folding/browser/folding.ts` around the existing `onCursorPositionChanged` handler. Alternatively, this could remain an extension-level feature (an extension like `felicio.vscode-fold` already explores this space).
