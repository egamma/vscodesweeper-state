# Issue #205369 — Git - Unable to see buttons when list of conflicting tag(s) is too long.

[https://github.com/microsoft/vscode/issues/205369](https://github.com/microsoft/vscode/issues/205369)

## Summary

Fixed on main: commit d481546f4c4 added max-height: 90vh to the dialog box and made the message content scrollable, preventing the dialog from overflowing the screen and hiding buttons — exactly the behavior this issue reports.

## Change summary

When a Git pull produces many conflicting tags, the modal error dialog grows taller than the screen, hiding the action buttons and pushing the VS Code window off-screen.

## Best solution

Keep the shipped fix in d481546f4c4 (1.111.0). The dialog now caps at 90vh with a scrollable message area, so buttons always remain visible regardless of tag count.
