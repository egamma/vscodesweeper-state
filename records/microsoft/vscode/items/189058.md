# Issue #189058 — Ghost text hover sometimes has empty rendering

[https://github.com/microsoft/vscode/issues/189058](https://github.com/microsoft/vscode/issues/189058)

## Summary

Team-authored bug that is still present on main. The ghost text hover widget renders with an empty count label when the inline completion provider hasn't yet responded to the explicit trigger, producing the broken appearance shown in the screenshot.

## Change summary

Ghost text hover sometimes shows an empty/broken rendering because the suggestion count is undefined until the provider responds to the explicit trigger.

## Best solution

Guard the hover rendering so it either shows a loading indicator or hides the prev/next navigation entirely until `inlineCompletionsCount` is defined (non-undefined). The fix lives in `src/vs/editor/contrib/inlineCompletions/browser/hintsWidget/inlineCompletionsHintsWidget.ts` around lines 262–280 and/or in the hover participant's `renderHoverParts`.
