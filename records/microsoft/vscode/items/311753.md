# Issue #311753 — Explorer panel: add vertical scroll support (mouse wheel / touchpad)

[https://github.com/microsoft/vscode/issues/311753](https://github.com/microsoft/vscode/issues/311753)

## Summary

The Explorer tree has always supported mouse-wheel and touchpad scrolling — it uses VS Code's standard SmoothScrollableElement infrastructure with explicit mouseWheelScrollSensitivity support. The report is not reproducible against current main and is likely user-specific (OS driver, extension, or configuration issue). Requesting more info.

## Change summary

Reporter claims the Explorer panel does not scroll with mouse wheel or touchpad on macOS Tahoe 26.4.1.

## Best solution

Almost certainly not a VS Code bug. The Explorer's tree widget (WorkbenchCompressibleAsyncDataTree) inherits scroll support from the base ListView, which wires up a SmoothScrollableElement with full mouse-wheel handling. The reporter should test with extensions disabled (`--disable-extensions`) and check `workbench.list.mouseWheelScrollSensitivity` settings. If it persists, a screen recording and `Developer: Toggle Developer Tools` console output would be needed.
