# Issue #226391 — VSC crashes at launch installed ver: code_1.92.2-1723660989_amd64.deb

[https://github.com/microsoft/vscode/issues/226391](https://github.com/microsoft/vscode/issues/226391)

## Summary

Already correctly triaged as upstream/mitigated. The crash is in Chromium's ANGLE/EGL GPU initialization with certain AMD GPU drivers on Linux — not in VS Code's own code. A workaround exists (`--disable-accelerated-video-decode`). The linked PR #307548 to change the desktop file Exec path was closed without merging; the build still substitutes the raw binary path (`/usr/share/code/code`) rather than the wrapper script.

## Change summary

VS Code crashes at launch on Linux with AMD GPU drivers due to Chromium GPU process ANGLE/EGL initialization failure (exit_code=139).

## Best solution

The root cause is in Chromium's GPU process initialization and would need to be fixed upstream in Electron/Chromium. The desktop file Exec path concern from comment 4 (pointing to the raw binary instead of the wrapper script at `/usr/bin/code`) is a valid improvement tracked in the build at `build/gulpfile.vscode.linux.ts:54` — a maintainer could evaluate whether changing that substitution is worthwhile independently of the upstream GPU fix.
