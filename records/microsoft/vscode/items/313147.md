# Issue #313147 — agents app- worktree changes not visible

[https://github.com/microsoft/vscode/issues/313147](https://github.com/microsoft/vscode/issues/313147)

## Summary

Team-authored issue reporting that the Agents window sidebar shows only branch-level changes but not uncommitted worktree changes, making it hard to verify whether the agent actually implemented requested edits. Keeping open per team-member policy.

## Change summary

Agents window sidebar does not display uncommitted worktree changes, only showing branch diff changes.

## Best solution

The Agents window's changes/PR review panel should surface uncommitted (dirty) worktree changes alongside the branch diff, so users can distinguish new agent edits from pre-existing branch changes without forcing a commit first.
