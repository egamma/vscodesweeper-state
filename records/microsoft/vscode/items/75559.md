# Issue #75559 — Builtin support for reading files from their zip archives

[https://github.com/microsoft/vscode/issues/75559](https://github.com/microsoft/vscode/issues/75559)

## Summary

Well-established feature request (644 upvotes, in Backlog since 2020) for built-in zip archive file reading. No built-in implementation exists on current main; the author's ZipFS extension provides a partial workaround but lacks seamless integration. The request remains valid and should stay open.

## Change summary

Add built-in support for reading files from within zip archives in VS Code's file explorer and editor, enabling seamless ctrl+click, Quick Open, and CLI workflows for zip-contained files.

## Best solution

The strongest path is improving the extension API surface (virtual filesystem search providers, URI resolution) so that extensions like ZipFS can integrate seamlessly — or, if demand continues, implementing a lightweight built-in zip filesystem provider in `src/vs/platform/files/`. The Yarn PnP-specific pain points (TypeScript Go to Definition) have been largely mitigated by editor SDKs, but the general zip-browsing capability gap remains real.
