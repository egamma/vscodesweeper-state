# Issue #158575 — Add "Disconnect single session" command

[https://github.com/microsoft/vscode/issues/158575](https://github.com/microsoft/vscode/issues/158575)

## Summary

Still unimplemented on main. The request is for a 'Disconnect Single Session' command that overrides compound `stopAll` behavior, letting a user disconnect one debug session without triggering termination of all sibling sessions. A team member (roblourens) agreed the command makes sense. Keeping open in Backlog.

## Change summary

Add a command to disconnect a single debug session from a compound launch without triggering the `stopAll` cascade that terminates all sibling sessions.

## Best solution

Add a 'Disconnect Single Session' command in `src/vs/workbench/contrib/debug/browser/debugCommands.ts` that calls `session.disconnect()` directly, bypassing the `DebugCompoundRoot.sessionStopped()` cascade in `src/vs/workbench/contrib/debug/browser/debugSession.ts`. This is a small, localized change in the debug contribution.
