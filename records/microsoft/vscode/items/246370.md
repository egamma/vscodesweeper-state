# Issue #246370 — Explorer: Ctrl + Drag-and-drop file does not copy file

[https://github.com/microsoft/vscode/issues/246370](https://github.com/microsoft/vscode/issues/246370)

## Summary

The VS Code drag-and-drop code correctly checks `originalEvent.ctrlKey` to determine copy vs. move (lines 1642 and 1854 in explorerViewer.ts), and the list view correctly sets `dropEffect` based on the result (line 1326 in listView.ts). The root cause is that Electron/Chromium on Linux does not reliably report `ctrlKey: true` on `DragEvent` objects during drag operations. A team member already pinged the Electron specialist (deepak1556). Keeping open as an upstream issue.

## Change summary

Ctrl+drag-and-drop in the Explorer does not copy files on Linux — the modifier key is not detected, so it falls back to move behavior.

## Best solution

This needs investigation on the Electron/Chromium side — `DragEvent.ctrlKey` is not reliably set on Linux during drag-and-drop. The Electron specialist was pinged in comment 18 but no follow-up occurred. Next step is to file or find the upstream Chromium/Electron bug and track it. A VS Code-side workaround could involve listening for `keydown`/`keyup` to track Ctrl state independently of the DragEvent modifier flags.
