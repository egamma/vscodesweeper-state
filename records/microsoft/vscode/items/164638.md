# Issue #164638 — Server process leaks after closing VS Code with Remote Tunnel Access On

[https://github.com/microsoft/vscode/issues/164638](https://github.com/microsoft/vscode/issues/164638)

## Summary

Team-authored issue (alexdima, MEMBER) — must keep open per policy. The bug describes a real resource leak: the VS Code server process isn't terminated when the desktop app quits with Remote Tunnel Access enabled. The comment from aeschli notes a 5-minute auto-shutdown delay exists, but the report implies the process persists beyond that.

## Change summary

Quitting VS Code with Remote Tunnel Access on leaves the server process running indefinitely instead of shutting it down.

## Best solution

Ensure the tunnel lifecycle management in `src/vs/server/node/remoteExtensionHostAgentServer.ts` properly triggers shutdown when the owning desktop process exits, rather than relying solely on the 5-minute idle timeout after connection close.
