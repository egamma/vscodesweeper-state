# Issue #109977 — Git - Expose git.publish as a command through Git API

[https://github.com/microsoft/vscode/issues/109977](https://github.com/microsoft/vscode/issues/109977)

## Summary

Team-authored feature request to properly expose git.publish through the Git extension API so errors propagate correctly to callers. The command exists but error handling when invoked programmatically is inadequate. Hard keep-open: author is MEMBER.

## Change summary

Expose git.publish as a proper Git extension API method so that errors (e.g. publishing a repo with no commits) propagate to the caller instead of surfacing as unhandled system modal errors.

## Best solution

Add a `publish` method to the `Repository` API surface in `extensions/git/src/api/git.d.ts` and `api1.ts` that wraps the git.publish command logic and properly propagates errors, rather than requiring consumers to use `commands.executeCommand('git.publish')` which swallows errors. The `help wanted` label makes this a good community contribution candidate.
