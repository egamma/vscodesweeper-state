# Issue #271438 — Images do not render in Chat from coding agent in Sessions

[https://github.com/microsoft/vscode/issues/271438](https://github.com/microsoft/vscode/issues/271438)

## Summary

Still reproduces on main: remote images in chat are unconditionally blocked. The `remoteImageIsAllowed` callback in the chat markdown renderer always returns `false`, preventing any remote image from rendering. Team-authored issue — keeping open.

## Change summary

Images from coding agent tasks (e.g. Playwright screenshots) do not render in the chat view because the chat markdown sanitizer blocks all remote images.

## Best solution

Change the `_remoteImageDisallowed` callback in `src/vs/workbench/contrib/chat/browser/widget/chatContentMarkdownRenderer.ts` to allow remote images (or a scoped subset, e.g. GitHub user-attachments URLs) so that agent-produced screenshots render in chat. The second sub-issue (private-repo image URLs 404-ing) may need authenticated image fetching or a proxy, which is a separate concern.
