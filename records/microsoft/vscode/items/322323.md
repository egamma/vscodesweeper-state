# Issue #322323 — Start Menu pinned tile becomes invisible (transparent) after every auto-update on   Windows

[https://github.com/microsoft/vscode/issues/322323](https://github.com/microsoft/vscode/issues/322323)

## Summary

Legitimate Windows 10 bug: the Inno Setup installer does not call SHChangeNotify or invalidate the icon cache after replacing Code.exe during auto-update, causing Start Menu pinned tiles to go transparent. A team member has engaged and suggested a possible duplicate of #311174, but the reporter clarified this is a distinct issue (Start Menu tiles only, not desktop shortcuts). Keeping open for the team to decide.

## Change summary

After VS Code auto-updates on Windows 10, the pinned Start Menu tile becomes invisible/transparent because the icon cache is not invalidated post-install.

## Best solution

Add a SHChangeNotify(SHCNE_ASSOCCHANGED) call in the CurStepChanged post-install step in build/win32/code.iss, or invoke ie4uinit.exe -ClearIconCache after the update completes. The team should also decide whether this is truly a duplicate of https://github.com/microsoft/vscode/issues/311174 (desktop shortcut icon path breaks due to hash folder rename) — the root causes differ: #311174 is about stale icon paths in shortcuts, while this issue is about the Windows icon cache not being invalidated even though paths are correct.
