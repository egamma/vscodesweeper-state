# Issue #60669 — `extensions/extension-editing/src/extensionLinter.ts` does not respect `repository` shorthand in package.json

[https://github.com/microsoft/vscode/issues/60669](https://github.com/microsoft/vscode/issues/60669)

## Summary

Still reproduces on main: the linter only reads `repository.url` (object form) and ignores npm shorthand strings like `"github:user/repo"`. An open PR (https://github.com/microsoft/vscode/pull/318073) is in flight to fix this.

## Change summary

The extension linter warns about relative image URLs even when `package.json` specifies a valid repository using npm shorthand syntax (e.g. `"repository": "github:user/repo"`).

## Best solution

Land https://github.com/microsoft/vscode/pull/318073 (or an equivalent fix) that parses npm repository shorthand strings in `extensions/extension-editing/src/extensionLinter.ts` at line 410, expanding them to HTTPS URLs before the `hasHttpsRepository` check.
