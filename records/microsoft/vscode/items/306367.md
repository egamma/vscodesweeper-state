# Issue #306367 — Failures when running tests for a newly-created extension project in a folder with spaces in the name

[https://github.com/microsoft/vscode/issues/306367](https://github.com/microsoft/vscode/issues/306367)

## Summary

The bug is in `@vscode/test-cli` (the `vscode-test` binary from `microsoft/vscode-test`), not in VS Code core. It was fixed by PR #335 in that repo, merged 2026-05-13 — updating the package resolves the issue.

## Change summary

Running extension tests via `vscode-test` in a folder with spaces in the path on Windows fails because the workspace/extension paths are split at space characters.

## Best solution

Close as resolved externally — the fix landed in `microsoft/vscode-test` (commit 6e17df7ed031ca7a613515846cec94e8499a49df). Users should update `@vscode/test-cli` to the version shipping that fix.
