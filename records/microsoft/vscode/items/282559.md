# Issue #282559 — Model Picker: Show Full Model Tags (e.g. mistral:7b) to Avoid Duplicates

[https://github.com/microsoft/vscode/issues/282559](https://github.com/microsoft/vscode/issues/282559)

## Summary

Valid feature request for improved model disambiguation in the picker when multiple variants of the same model family are registered by a third-party provider (Ollama). The model picker displays `model.metadata.name` as-is from the language model provider; the fix could live in core (deduplication detection / suffix display) or in the providing extension. Keeping open as it has the correct area label and milestone.

## Change summary

When multiple Ollama model variants share a base name (e.g. qwen2.5-coder:0.5b vs :32b), the model picker shows only the base name, making them look like duplicates.

## Best solution

Either the language-model provider extension (Ollama/AI Toolkit) should supply the full qualified tag as the `name` in `ILanguageModelChatMetadata`, or the model picker in `src/vs/workbench/contrib/chat/browser/widget/input/chatModelPicker.ts` should detect duplicate base names and append a disambiguating suffix (e.g. the parameter count). The simpler path is for the provider extension to include the tag — making this partially an extension-side fix — but a core fallback for deduplication display would benefit all providers.
