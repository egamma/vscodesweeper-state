# Issue #48063 — Search and replace across files corrupts files with CR line endings

[https://github.com/microsoft/vscode/issues/48063](https://github.com/microsoft/vscode/issues/48063)

## Summary

Still reproduces on main: the search infrastructure uses `/\r?\n/g` to split lines, which ignores CR-only (`\r`) line endings, causing line-number mismatches and file corruption during replace-in-files. Author is a MEMBER — keeping open per policy.

## Change summary

Search and replace across files corrupts files that use CR-only line endings because the search engine does not recognize standalone CR as a line terminator.

## Best solution

The line-splitting regex in `src/vs/workbench/services/search/common/getFileResults.ts` (line 51) and any parallel paths need to handle standalone `\r` as a line ending — or, at minimum, the replace path should detect CR-only files and refuse to apply replacements with a user-facing warning (as explored in the closed https://github.com/microsoft/vscode/issues/146026). The upstream feature request for full CR support is https://github.com/microsoft/vscode/issues/35797.
