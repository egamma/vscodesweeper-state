# Issue #314937 — Explorer File Tree Sort Order Bug

[https://github.com/microsoft/vscode/issues/314937](https://github.com/microsoft/vscode/issues/314937)

## Summary

The default Explorer sort uses natural/numeric sorting via Intl.Collator with `numeric: true`, which intentionally compares numeric substrings as numbers (so `10` < `106`, placing `Z10X` before `Z106`). This is as-designed; pure ASCII ordering is available via the `explorer.sortOrderLexicographicOptions: "unicode"` setting.

## Change summary

Explorer default sort order treats numeric substrings numerically rather than character-by-character, causing files like `file_Z10X.txt` to appear before `file_Z106.txt`.

## Best solution

Close as designed. The user can set `explorer.sortOrderLexicographicOptions` to `"unicode"` to get pure Unicode/ASCII character ordering, which matches the `ls` behavior they expect.
