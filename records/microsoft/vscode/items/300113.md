# Issue #300113 — Personal CLAUDE.md is attached even though its empty

[https://github.com/microsoft/vscode/issues/300113](https://github.com/microsoft/vscode/issues/300113)

## Summary

Team-authored issue; the bug appears still present on main — `promptsServiceImpl.ts` discovers `~/.claude/CLAUDE.md` without checking if file content is empty before attaching it to the prompt. No fix commit found.

## Change summary

An empty personal `~/.claude/CLAUDE.md` file is attached to the system prompt, producing an empty `<attachment>` element that confuses the agent into requesting file access.

## Best solution

When resolving prompt instruction files (in `promptsServiceImpl.ts` or during variable resolution), skip files whose trimmed content is empty so they are never attached as empty `<attachment>` elements. The fix likely lives in `src/vs/workbench/contrib/chat/common/promptSyntax/service/promptsServiceImpl.ts` (discovery) or wherever the file content is rendered into the prompt attachment XML.
