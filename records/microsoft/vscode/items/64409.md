# Issue #64409 — Git - Gracefully handle git hooks

[https://github.com/microsoft/vscode/issues/64409](https://github.com/microsoft/vscode/issues/64409)

## Summary

Partially addressed — VS Code now has noVerify commit commands and a general progress indicator during commits — but the core ask (a distinct UI indicator specifically for hook execution, and inline hook output) remains unimplemented. Valid feature request with 44 upvotes, properly in Backlog with `help wanted`.

## Change summary

Show a dedicated UI indicator when git hooks (pre-commit, commit-msg, etc.) are running during a commit via the source control panel, and optionally allow skipping them.

## Best solution

The skip-hooks half is already shipped (`git.commitNoVerify` commands + `git.allowNoVerifyCommit` setting). The remaining gap is surfacing hook-specific progress/output — e.g., distinguishing 'running pre-commit hook' from the commit itself, and streaming hook stdout/stderr in the Git output channel or a notification. Implementation would likely touch `extensions/git/src/git.ts` (spawn handling) and `extensions/git/src/repository.ts` (progress reporting). Good `help wanted` candidate.
