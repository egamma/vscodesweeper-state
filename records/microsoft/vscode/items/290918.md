## Summary

Valid telemetry instrumentation gap tracked as part of the broader chat-submit telemetry effort (#287920, labeled under-discussion). The request is for emitting request/response events when a user encounters a disabled model but does not enable it, to avoid SRMs in A/B experiments. Keeping open — the parent issue is still active and under discussion.

## Change summary

Emit telemetry request/response events when a user selects or is served a disabled model in Chat Panel but does not enable it, so A/B experiment analysis isn't skewed by missing data.

## Best solution

If the parent issue fully absorbs this scenario, it could also be closed as a duplicate of #287920, but that depends on maintainer judgment since #287920 is still under discussion.
