# Issue #284175 — Copilot Edits: Arrow up should not change attached files

[https://github.com/microsoft/vscode/issues/284175](https://github.com/microsoft/vscode/issues/284175)

## Summary

Team-authored issue requesting that arrow-up/down history navigation in chat not restore attached file context. The behavior is still present on main — `navigateHistory()` unconditionally calls `restoreAttachments()`. Keeping open per team-member policy.

## Change summary

Arrow up/down in Copilot Edits chat restores the full attached-files context from the previous request, making it hard to retry with different context.

## Best solution

Modify `navigateHistory()` in `src/vs/workbench/contrib/chat/browser/widget/input/chatInputPart.ts` to skip `restoreAttachments()` (line 2381), restoring only the input text. This is a UX design decision the team should finalize — whether to never restore attachments, or to offer an opt-in mechanism.
