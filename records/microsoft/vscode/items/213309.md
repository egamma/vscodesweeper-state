# Issue #213309 — Very thin hollow/transparent gap/background between mini-map and scrollbar (on split editor)

[https://github.com/microsoft/vscode/issues/213309](https://github.com/microsoft/vscode/issues/213309)

## Summary

Already correctly triaged as upstream (Chromium zoom rounding). A team member explained the root cause lives in Chromium's sub-pixel rendering at non-default zoom levels; the existing labels are accurate and no action is needed beyond keeping it open as an upstream tracker.

## Change summary

A thin transparent gap appears between the minimap and scrollbar when zoom level is set to -1, exposing underlying text.

## Best solution

This is a Chromium rendering artifact at non-integer zoom levels. VS Code positions elements at pixel-perfect CSS coordinates, but Chromium's zoom rounding introduces sub-pixel gaps. The issue is correctly labeled `upstream` and `zoom`; it stays open as a known limitation until Chromium improves zoom rendering fidelity. Users can work around it by keeping zoom at 0 and adjusting font size instead.
