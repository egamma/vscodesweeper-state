# Issue #234812 — Don't import XYZ.contribution-files from non contribution/entry-files

[https://github.com/microsoft/vscode/issues/234812](https://github.com/microsoft/vscode/issues/234812)

## Summary

Valid engineering debt item from a core team member requesting lint rules to prevent importing contribution files from non-contribution/entry files. No progress has been made — the custom eslint plugin has no such rule yet.

## Change summary

Add lint rules to enforce that contribution files are not imported by normal (non-entry) files, preventing unintended side effects.

## Best solution

Implement a custom eslint rule in `.eslint-plugin-local/` (e.g. `code-no-contribution-import.ts`) that flags imports of `*.contribution.*` files from files that are not themselves contribution or entry-point files. The rule would scan import paths for the `.contribution.` segment and report when the importing file doesn't match the same pattern.
