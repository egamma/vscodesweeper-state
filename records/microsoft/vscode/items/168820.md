# Issue #168820 — SCM - Sort `Source control repositories` by number of changes

[https://github.com/microsoft/vscode/issues/168820](https://github.com/microsoft/vscode/issues/168820)

## Summary

The core request — sorting SCM repositories by number of changes — remains unimplemented on main. The badge-visibility sub-topic (hiding zero-count badges) was addressed by PR #194696, but the sort key was never added. With 25 upvotes and Backlog status, this stays open as a valid feature request.

## Change summary

Add a sort-by-number-of-changes option for the Source Control Repositories view so modified repos cluster together in poly-repo projects.

## Best solution

Add a new `ISCMRepositorySortKey` value (e.g. `ChangeCount`) in `src/vs/workbench/contrib/scm/common/scm.ts`, wire it into `scm.repositories.sortOrder` in `scm.contribution.ts`, and implement the comparator in `SCMViewService.compareRepositories()` in `scmViewService.ts` — sorting by the sum of resource-group counts descending.
