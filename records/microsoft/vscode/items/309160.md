# Issue #309160 — Terminal: Missing devicePixelRatio listener causes misalignment on zoom

[https://github.com/microsoft/vscode/issues/309160](https://github.com/microsoft/vscode/issues/309160)

## Summary

Confirmed in source: the terminal reads `devicePixelRatio` for canvas layout calculations but never subscribes to `PixelRatio.onDidChange` or zoom-level events, so zoom changes leave the canvas misaligned until a manual relayout is triggered. An open community PR (#309165) addresses this.

## Change summary

Terminal canvas misaligns on zoom level change because no listener triggers relayout when devicePixelRatio changes.

## Best solution

Review and land the community PR https://github.com/microsoft/vscode/pull/309165 (or implement an equivalent fix): subscribe to `PixelRatio.onDidChange` in the terminal instance and trigger a relayout + font re-measurement when it fires.
