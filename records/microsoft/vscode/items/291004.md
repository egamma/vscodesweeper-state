# Issue #291004 — MCP Resources Not Accessible to AI Agents - Breaking Intended Use Cases

[https://github.com/microsoft/vscode/issues/291004](https://github.com/microsoft/vscode/issues/291004)

## Summary

Valid feature request for programmatic agent access to MCP resources. Partial progress exists via the agent-host gateway (McpGatewayToolBrokerChannel exposes listResourcesForServer/readResourceForServer), but standard chat-agent sessions still lack direct resource access. The related issue #251747 covers adjacent ground (auto-attach for MCP prompts/resources) and is under discussion.

## Change summary

Allow AI agents to programmatically list and read MCP server resources during task execution, rather than requiring manual user attachment.

## Best solution

The agent-host path already has resource access via McpGatewayToolBrokerChannel. The remaining gap is exposing resources as callable tools (or auto-injected context) in standard Copilot chat sessions. This work sits naturally alongside https://github.com/microsoft/vscode/issues/251747 and should be resolved as part of that broader effort to make MCP prompts and resources available to agents.
