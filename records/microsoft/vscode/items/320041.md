# Issue #320041 — Git extension falsely detects repository in empty/new window after updating to latest version

[https://github.com/microsoft/vscode/issues/320041](https://github.com/microsoft/vscode/issues/320041)

## Summary

Traceable to an intentional Jan 2026 change (7c845943300) that allows opening repositories in empty workspaces for the agents/sessions workflow, but which also bypasses the `openRepositoryInParentFolders` prompt when a parent directory has a .git folder. Routing to the git area owner for evaluation of whether empty-window detection should still respect the parent-folder config.

## Change summary

Opening a new empty window causes the Git extension to detect and open a parent-directory repository without prompting, flooding Source Control with irrelevant changes.

## Best solution

In `isRepositoryOutsideWorkspace` (extensions/git/src/model.ts ~line 1116), the empty-workspace bypass should still respect the `openRepositoryInParentFolders` setting — when set to `never` or `prompt`, the parent-folder repo should not be silently opened. The workaround is setting `git.openRepositoryInParentFolders` to `never`.
