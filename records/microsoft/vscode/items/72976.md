# Issue #72976 — Source Control: "Accept All Incoming" for "deleted by them" should delete the file from working copy and stage the change

[https://github.com/microsoft/vscode/issues/72976](https://github.com/microsoft/vscode/issues/72976)

## Summary

Valid feature request with 27 upvotes (above the 20-upvote backlog threshold). The "Accept All Incoming" codelens in the editor doesn't handle delete-vs-modify conflicts because there are no textual merge markers — only the Stage Changes context menu offers the keep/delete choice. The gap is real and still present on main.

## Change summary

"Accept All Incoming" on a "deleted by them" merge conflict should delete the file and stage the change, instead of showing "No merge conflicts found."

## Best solution

When opening a file with status DELETED_BY_THEM and using the merge conflict codelens actions, the git extension (or the merge-conflict extension at `extensions/merge-conflict`) should detect the delete-vs-modify state and offer to delete+stage or keep+stage, rather than reporting no conflicts found. The fix likely spans `extensions/merge-conflict/src/mergeDecorator.ts` (to detect deletion conflicts) and `extensions/git/src/commands.ts` (where the DELETED_BY_THEM staging logic at ~line 1595 already exists and could be reused). With 27 upvotes this belongs in the Backlog.
