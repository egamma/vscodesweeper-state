# Issue #307292 — Terminal confirmation disclaimer renders as KaTeX math when path contains `$`

[https://github.com/microsoft/vscode/issues/307292](https://github.com/microsoft/vscode/issues/307292)

## Summary

Confirmed on current main: file paths containing `$` are inserted unescaped into the disclaimer MarkdownString, allowing unintended KaTeX math pairing with the `$(info)` theme icon prefix. The bug remains unfixed.

## Change summary

Terminal confirmation disclaimer renders as KaTeX math when file write paths contain `$` characters that pair with the theme icon syntax.

## Best solution

Escape `$` characters in file path strings (e.g. replace `$` with `\$`) in `commandLineFileWriteAnalyzer.ts` line 196 before they are inserted into the backtick-wrapped markdown, so they cannot pair with the `$(info)` icon prefix to form a math span.
