# Issue #147526 — [folding] Increase number of foldable regions

[https://github.com/microsoft/vscode/issues/147526](https://github.com/microsoft/vscode/issues/147526)

## Summary

The 65535 limit is still present on main — MAX_FOLDING_REGIONS = 0xFFFF in foldingRanges.ts, and the setting cap remains 65000. The architectural constraint (16-bit parent index storage) has not been lifted.

## Change summary

Request to raise the maximum number of foldable regions beyond 65000 for users working with very large files.

## Best solution

Refactor parent-index storage in `src/vs/editor/contrib/folding/browser/foldingRanges.ts` to use a wider representation (separate Uint32Array or relative offsets as aeschli suggested), then raise MAX_FOLDING_REGIONS and the EditorIntOption upper bound in `src/vs/editor/common/config/editorOptions.ts`.
