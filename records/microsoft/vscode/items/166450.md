# Issue #166450 — Debugging: Support single-thread execution requests

[https://github.com/microsoft/vscode/issues/166450](https://github.com/microsoft/vscode/issues/166450)

## Summary

Still unimplemented: VS Code's debug adapter client never sends the `singleThread` flag on execution requests (continue, next, stepIn, etc.), despite the DAP protocol supporting it since the capability was added. The feature request remains valid with ongoing community interest (most recent comment May 2026).

## Change summary

Support the DAP `singleThread` property on execution requests so that debug adapters advertising `supportsSingleThreadExecutionRequests` can resume/step only the selected thread.

## Best solution

Implement `singleThread` support in the debug session's execution request paths (likely `src/vs/workbench/contrib/debug/browser/debugSession.ts` and the call stack view). When a DA advertises `supportsSingleThreadExecutionRequests`, the inline thread-level continue/step buttons in the Call Stack view should send `singleThread: true`, while the main debug toolbar continues to resume all threads. This is a core debugger feature, not an extension candidate.
