# Issue #305195 — Model Selection Submenu cannot show

[https://github.com/microsoft/vscode/issues/305195](https://github.com/microsoft/vscode/issues/305195)

## Summary

The model picker submenu (e.g. thinking-level low/medium selector) reportedly flashes and disappears in standalone/auxiliary chat windows but works correctly in the main window. No fix found on current main. The issue is plausible — the actionWidget's blur handler in `actionWidget.ts` (line 210–216) uses `getActiveElement()` and `.closest()` to check whether focus moved to a submenu, and this could race or misfire in auxiliary windows. Routing to the chat area for investigation.

## Change summary

Model picker submenu items with nested actions (like thinking-level selection) flash once and disappear when chat is opened in a standalone/auxiliary window.

## Best solution

Investigate the focus-blur interaction in `src/vs/platform/actionWidget/browser/actionWidget.ts` (the `focusTracker.onDidBlur` handler at ~line 210) and `src/vs/platform/actionWidget/browser/actionList.ts` submenu show/hide scheduling in the context of auxiliary windows. The `getActiveElement()` call may return the wrong element or `null` when the submenu opens in an auxiliary window's DOM tree, causing the widget to immediately dismiss. Related issue https://github.com/microsoft/vscode/issues/298167 covers tooltip flickering in the same area.
