# Issue #241055 — Accessibility MacOS: Pressing a routing button over a character in editor does not move the cursor

[https://github.com/microsoft/vscode/issues/241055](https://github.com/microsoft/vscode/issues/241055)

## Summary

Legitimate accessibility bug — the editor doesn't expose the macOS accessibility API property needed for Braille display cursor routing. No fix found on current main. Requires specialized hardware to reproduce, so reproduction status is unclear from a code-only inspection, but the reporter's explanation is technically credible.

## Change summary

Pressing a routing button on a physical Braille display over a character in the editor does not move the cursor; VoiceOver treats it as an activation instead of a cursor move.

## Best solution

The editor's accessibility implementation (likely in `src/vs/editor/browser/controller/editContext/`) needs to expose the appropriate macOS accessibility action or attribute so that VoiceOver can route the cursor via Braille display routing buttons, similar to how native TextEdit handles this.
