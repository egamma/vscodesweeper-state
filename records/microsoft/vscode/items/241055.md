# Issue #241055 — Accessibility MacOS: Pressing a routing button over a character in editor does not move the cursor

[https://github.com/microsoft/vscode/issues/241055](https://github.com/microsoft/vscode/issues/241055)

## Summary

Still open on main — no fix has landed for the missing macOS accessibility API property that would allow VoiceOver Braille routing buttons to move the cursor in the editor. Requires physical Braille hardware to reproduce.

## Change summary

Pressing a Braille display routing button over a character in the VS Code editor does not move the cursor; VoiceOver treats it as an activation instead of a cursor-positioning action.

## Best solution

The editor's accessible DOM element (in editContext/native or textArea) needs to expose the appropriate macOS accessibility attributes (likely AXSelectedTextMarkerRange or similar) so VoiceOver can perform cursor routing rather than element activation. Investigation is needed in `src/vs/editor/browser/controller/editContext/` to determine which accessibility property is missing. This may also involve Electron/Chromium's accessibility bridge for macOS.
