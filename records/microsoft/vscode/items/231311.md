# Issue #231311 — Closing a view while dragging border makes the editor view unresponsive to mouse interaction

[https://github.com/microsoft/vscode/issues/231311](https://github.com/microsoft/vscode/issues/231311)

## Summary

Still reproduces on main: closing a split view mid-sash-drag leaves remaining view items with `pointer-events: none` because `onSashEnd` never fires to re-enable them.

## Change summary

Closing an editor group while dragging the sash border between splits leaves the editor area unresponsive to mouse interaction.

## Best solution

In `SplitView.removeView()`, detect an active `sashDragState` when the dragged sash is disposed, clean it up, and re-enable all remaining view items' pointer events.
