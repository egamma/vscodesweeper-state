# Issue #231311 — Closing a view while dragging border makes the editor view unresponsive to mouse interaction

[https://github.com/microsoft/vscode/issues/231311](https://github.com/microsoft/vscode/issues/231311)

## Summary

Still reproduces on main: when a sash is disposed mid-drag (e.g. closing the view with Ctrl+W), the drag-session cleanup never runs — the forced-cursor stylesheet and iframe pointer-events suppression persist, making the editor unresponsive to mouse interaction.

## Change summary

Closing an editor view while dragging its split border leaves the workspace unresponsive to mouse interaction in the editor area.

## Best solution

Track the drag-session disposable store on the Sash instance (or register it in the sash's own DisposableStore) so that `Sash.dispose()` cleans up the forced cursor stylesheet and re-enables pointer events on iframes even when the pointer-up event never fires.
