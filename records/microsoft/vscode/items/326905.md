# Issue #326905 — Module not found error in Output panel when running Python script via Remote SSH, despite successful execution in the integrated terminal

[https://github.com/microsoft/vscode/issues/326905](https://github.com/microsoft/vscode/issues/326905)

## Summary

The reporter confirmed the issue does not occur with all extensions disabled, and the behavior ("Run Python File" button, Output panel execution, interpreter selection) is entirely owned by the Python extension (microsoft/vscode-python), not VS Code core. This should be moved to the Python extension's tracker.

## Change summary

The "Run Python File" action uses a different Python environment than the integrated terminal when connected via Remote SSH with a Conda environment.

## Best solution

Move this issue to https://github.com/microsoft/vscode-python — the "Run Python File" command, its interpreter resolution, and the Output-panel execution path are all owned by the Python extension. The Python extension team can investigate why the run-button path doesn't pick up the same Conda environment that the integrated terminal activates.
