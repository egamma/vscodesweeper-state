# Issue #208492 — git conflicts - highlight for the base for diff3 conflict format

[https://github.com/microsoft/vscode/issues/208492](https://github.com/microsoft/vscode/issues/208492)

## Summary

Valid incremental feature request: the merge-conflict extension already parses diff3 `|||||||` markers and applies background highlighting for the base/common-ancestor section, but the header decoration is missing the descriptive label text (e.g. "(Base)") that the Current Change and Incoming Change headers display. Low upvote count (1) and niche audience — keeping open in Backlog for community traction.

## Change summary

Add a descriptive inline label like "(Base)" to the diff3 common-ancestor header marker, matching the existing "(Current Change)" and "(Incoming Change)" labels.

## Best solution

Add an `after: { contentText: ' (Base)', color: ... }` property to the `commonAncestors.header` decoration in `extensions/merge-conflict/src/mergeDecorator.ts` (around line 95), mirroring the pattern used for `current.header` (line 90) and `incoming.header` (line 120). The background highlighting already works via `merge.commonHeaderBackground` / `merge.commonContentBackground` theme colors — the gap is only the missing label text.
