# Issue #230136 — [json] Add support for extensions associating JSON schemas with documents dynamically (via API)

[https://github.com/microsoft/vscode/issues/230136](https://github.com/microsoft/vscode/issues/230136)

## Summary

Valid feature request for a dynamic JSON schema association API. A team member (aeschli) engaged with design considerations, and an open community PR (#239701) is actively pursuing this. Below the 20-upvote threshold but has active discussion and related work in progress.

## Change summary

Extensions cannot programmatically associate JSON schemas with documents at runtime; only static file-match patterns are supported.

## Best solution

Provide a contribution-point-level mechanism (as aeschli suggested) or an extension API to dynamically register JSON schema associations, avoiding tight coupling to `json-language-features`. The open PR https://github.com/microsoft/vscode/pull/239701 and issue https://github.com/microsoft/vscode/issues/252610 are pursuing this space — consolidate the design there.
