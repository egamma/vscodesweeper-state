# Issue #299480 — Does not load copilot-instructions.md

[https://github.com/microsoft/vscode/issues/299480](https://github.com/microsoft/vscode/issues/299480)

## Summary

The reported behavior — which instruction files get loaded depending on the selected agent type — is controlled by the GitHub Copilot extension (version 0.38.0 at time of report), not VS Code core. The extension's `agentInstructionsLocator.ts` and `automaticInstructionsCollector.ts` own the logic for discovering and attaching `copilot-instructions.md`.

## Change summary

Copilot does not load `.github/copilot-instructions.md` when using certain agent types ("Local") but does load `CLAUDE.md` when switching to Claude.

## Best solution

This should be filed against the GitHub Copilot extension (https://github.com/microsoft/vscode-copilot-release) where the instruction-file discovery logic lives. The VS Code core prompt-syntax infrastructure provides the framework, but agent-type-specific loading decisions are made by the extension.
