# Issue #171837 — Add prelaunch Command as well for launch configuration

[https://github.com/microsoft/vscode/issues/171837](https://github.com/microsoft/vscode/issues/171837)

## Summary

Valid feature request with 21 upvotes, properly in the Backlog. Not implemented on main — no preLaunchCommand property exists in the debug schema. The gap remains real: users cannot inline a shell command in launch.json without defining a separate task in tasks.json.

## Change summary

Add a preLaunchCommand property to launch.json that directly invokes a shell command without requiring a tasks.json entry, enabling variable sharing between the launch config and its pre-launch step.

## Best solution

Implement a `preLaunchCommand` (or allow inline task definitions in `preLaunchTask`) in `src/vs/workbench/contrib/debug/common/debugSchemas.ts` and wire execution through `debugTaskRunner.ts`. Alternatively, improve variable sharing between launch configs and referenced tasks so the underlying pain point (sharing inputs like IP addresses) is resolved without a new schema property.
