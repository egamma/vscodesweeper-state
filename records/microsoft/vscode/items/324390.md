## Summary

The VS Code client correctly sends `gpt-5.4-mini` as the requested model — aeschli confirmed this. The unexpected resolution to a non-mini backend label (`gpt-5.4-2026-03-05` or `gpt-5.3-codex`) happens on the Copilot API backend, making this an upstream issue. ## Change summary

When a sub-agent's frontmatter requests `gpt-5.4-mini`, the Copilot backend resolves it to a non-mini model identifier, raising concerns about whether the mini SKU is actually used for billing and inference.

## Best solution

The fix belongs in the Copilot API backend (model routing/pinning layer), not in the VS Code client. The client's `normalizeResponseModel` already expects mini responses to include 'mini' in the resolved name (e.g. `gpt-5.4-mini-2026-03-17`). The related `github/copilot-api` issue on model pinning likely owns this work — resolution should confirm whether the backend is correctly routing to the mini deployment and whether the response model label accurately reflects the served model.
