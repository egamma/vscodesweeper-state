# Issue #124954 — Allow going inside graphemes (with an explicit gesture)

[https://github.com/microsoft/vscode/issues/124954](https://github.com/microsoft/vscode/issues/124954)

## Summary

Keep open as a Backlog feature request: the default grapheme-cluster behavior is intentional, but the issue asks for an explicit escape hatch to move/edit inside graphemes and it reached the community upvote threshold.

## Change summary

Allow an explicit editor gesture or command to place the cursor inside a Unicode grapheme cluster for fine-grained text editing.

## Best solution

Preserve Unicode grapheme-aware cursor movement by default, but make an editor-core decision on whether to add a deliberate codepoint-level movement/positioning command or setting. If implemented, it likely belongs around cursor movement and column mapping code that currently uses GraphemeIterator.
