# Issue #264340 — git diff  view not updated after unstaging changes of new  file

[https://github.com/microsoft/vscode/issues/264340](https://github.com/microsoft/vscode/issues/264340)

## Summary

Valid bug with an open community PR (#292153) awaiting review. The diff view fails to update after unstaging a new file because the `~` ref resolution in the git file system provider doesn't properly fall back when the file leaves the index. Keeping open pending PR review.

## Change summary

After unstaging changes to a newly added file, the diff view does not refresh to reflect the updated state.

## Best solution

Review and land (or iterate on) the open PR https://github.com/microsoft/vscode/pull/292153, which adds index→HEAD→empty fallback logic for the `~` ref in `extensions/git/src/fileSystemProvider.ts`. The fix touches `stat` and `readFile` to handle the case where a new file is unstaged and no longer in the index.
