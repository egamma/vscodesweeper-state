# Issue #289657 — GitHub Copilot Agent cannot execute commands in WSL/bash terminals - only PowerShell supported

[https://github.com/microsoft/vscode/issues/289657](https://github.com/microsoft/vscode/issues/289657)

## Summary

Valid feature request for the chat agent's run_in_terminal tool to auto-detect WSL context or reuse the active terminal profile. A workaround exists via `chat.tools.terminal.terminalProfile.windows`, but the default behavior doesn't adapt to project context. Already triaged with `confirmation-pending` and appropriate area labels.

## Change summary

The Copilot Agent's run_in_terminal tool defaults to PowerShell on Windows even when WSL terminals are active and the project requires Unix toolchains.

## Best solution

Improve terminal profile selection in `TerminalProfileFetcher.getCopilotProfile()` (in `src/vs/workbench/contrib/terminalContrib/chatAgentTools/browser/tools/runInTerminalTool.ts`) to consider the active terminal's profile or workspace context (e.g., presence of Makefiles, remote WSL authority) when choosing the shell. In the interim, users can set `chat.tools.terminal.terminalProfile.windows` to their WSL profile.
