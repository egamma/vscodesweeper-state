# Issue #311174 — Icon theme missing after update to 1.116.0

[https://github.com/microsoft/vscode/issues/311174](https://github.com/microsoft/vscode/issues/311174)

## Summary

Still reproduces on main: the versioned resource folder hash changes with each update, invalidating cached Windows icon/tile paths. A team member (dmitrivMS) is actively engaged. A related, more detailed report exists at #322323 covering the same root cause.

## Change summary

VS Code's desktop/Start Menu icons become blank after each Windows update because the versioned resource folder name (commit hash) changes, breaking cached icon paths.

## Best solution

The installer should invalidate the Windows icon cache after updating (e.g., call `SHChangeNotify(SHCNE_ASSOCCHANGED, ...)` or `ie4uinit.exe -ClearIconCache` post-install in `build/win32/code.iss`). The VisualElements manifest path also needs to survive the folder hash change — either by using a stable symlink/junction or by notifying Windows to re-read the manifest. Issue https://github.com/microsoft/vscode/issues/322323 provides a concrete suggested fix in Inno Setup. Consider consolidating both issues.
