# Issue #47816 — Use fresh environment block on new terminals in Windows

[https://github.com/microsoft/vscode/issues/47816](https://github.com/microsoft/vscode/issues/47816)

## Summary

Still valid: on Windows, VS Code's pty host returns its own inherited `process.env` and `resolveShellEnv()` explicitly skips Windows. New terminals cannot pick up environment variable changes made after VS Code launched. The issue has 123 upvotes, well above the Backlog threshold.

## Change summary

New terminal instances on Windows should use a fresh environment block reflecting system/user environment variable changes made after VS Code was launched.

## Best solution

Implement a Windows-specific mechanism (e.g. reading from the registry via `HKCU\Environment` and `HKLM\...\Environment`, similar to how cmder and other tools do it) to construct a fresh environment block when spawning new terminal processes, rather than inheriting the stale `process.env` of the pty host. The fix likely lives in `src/vs/platform/terminal/node/ptyService.ts` (the `getEnvironment()` method) or a new Windows-specific environment resolver in `src/vs/platform/shell/node/shellEnv.ts`.
