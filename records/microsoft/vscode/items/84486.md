# Issue #84486 — [themes] provide command to change theme

[https://github.com/microsoft/vscode/issues/84486](https://github.com/microsoft/vscode/issues/84486)

## Summary

The core request — a command that accepts a theme name as an argument for direct keybinding — is not implemented on main. The toggle command partially addresses the use case but only switches between preferred light/dark, not arbitrary named themes.

## Change summary

Request a command that sets the color theme by name (passed as an argument) so users can bind specific themes to keyboard shortcuts.

## Best solution

Add an optional `theme` argument to `workbench.action.selectTheme` (or register a new `workbench.action.setColorTheme` command) that, when provided, immediately applies the named theme without opening the picker. This would live in `src/vs/workbench/contrib/themes/browser/themes.contribution.ts`.
