# Issue #151715 — Extension variables are not expanded in integrated terminal profiles

[https://github.com/microsoft/vscode/issues/151715](https://github.com/microsoft/vscode/issues/151715)

## Summary

Still reproduces on main: profile.env values are never passed through _resolveVariables in terminalProfileResolverService.ts — only path and args get variable resolution. Confirmed by a team member and labeled accordingly; keeping open as a real, unfixed bug.

## Change summary

Variables in terminal profile `env` values (e.g. `${workspaceFolder}`) are not resolved, leaving literal placeholder strings in the environment.

## Best solution

In `_resolveProfile` (terminalProfileResolverService.ts), iterate over `profile.env` values and pass each through `_resolveVariables` the same way `profile.path` and `profile.args` are resolved today.
