# Issue #286611 — Slow chat switching- keep ChatModel in memory for a bit

[https://github.com/microsoft/vscode/issues/286611](https://github.com/microsoft/vscode/issues/286611)

## Summary

Still relevant on main — the ChatModel is only kept alive during in-progress requests, not across session switches. A TODO at line 2537 of chatModel.ts explicitly notes the missing timeout-based retention. Team-authored issue; keeping open.

## Change summary

Switching away from a large chat session and back forces a full re-deserialization of the ChatModel each time, causing noticeable slowness.

## Best solution

Add a timeout-based retention mechanism (e.g. 5-minute LRU or MutableDisposable with setTimeout) around the self-reference pattern already in `src/vs/workbench/contrib/chat/common/model/chatModel.ts` (line ~2537) so the ChatModel stays alive briefly after losing its last external reference, avoiding repeated deserialization on quick session switches.
