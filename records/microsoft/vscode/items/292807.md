## Summary

Team-authored issue; the diff timeout notification in diffEditorHelper.ts still fires unconditionally during agent sessions on current main — no fix or suppression has landed. Keeping open.

## Change summary

The 'Diff Algorithm has stopped after 5000 ms' notification repeatedly appears during agent sessions when large files are being edited, even though the diff is only for background visual display.

## Best solution

Suppress or downgrade the notification when the diff is triggered by an agent session (e.g. check whether the edit originated from a chat/agent tool call). The logic lives in `src/vs/workbench/contrib/codeEditor/browser/diffEditorHelper.ts` around line 58–70; a context-aware guard or silent fallback for agent-driven edits would resolve the annoyance without changing the default timeout.
