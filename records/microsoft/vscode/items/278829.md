# Issue #278829 — Feature Request: Ignore specific folders/files in Copilot

[https://github.com/microsoft/vscode/issues/278829](https://github.com/microsoft/vscode/issues/278829)

## Summary

Valid feature request with strong community support (57 upvotes, promoted to Backlog). The Copilot extension already supports `.copilotignore` files and remote content exclusions, but the user specifically requests a VS Code settings-based approach (`github.copilot.ignoredPaths`). This represents a real UX gap — a settings-driven exclusion mechanism would be more discoverable and consistent with existing VS Code patterns. Keeping open as a legitimate backlog item.

## Change summary

Add a VS Code setting (e.g. `github.copilot.ignoredPaths`) to exclude specific files/folders from Copilot context by glob pattern, without disabling Copilot entirely.

## Best solution

The Copilot extension already has the infrastructure for file exclusion via `.copilotignore` (in `extensions/copilot/src/platform/ignore/`) and remote content exclusions. A settings-based approach would extend this by reading glob patterns from a configuration setting and feeding them into the existing `IIgnoreService.isCopilotIgnored()` pipeline. This belongs in the Copilot extension itself (not VS Code core), likely as an enhancement to `BaseIgnoreService` in `extensions/copilot/src/platform/ignore/node/ignoreServiceImpl.ts`. Related issue https://github.com/microsoft/vscode/issues/283189 overlaps on the exclusion ask but bundles a second unrelated feature (persistent context attachment) and has minimal traction (1 upvote); this issue is the stronger signal.
