# Issue #318294 — Integrated terminal inserts Fcitx candidate selection number into shell input on Linux

[https://github.com/microsoft/vscode/issues/318294](https://github.com/microsoft/vscode/issues/318294)

## Summary

Reporter has only reproduced in Cursor, not vanilla VS Code. The likely root cause is in xterm.js's CompositionHelper (keydown returns true for number keys during composition), but we need a confirmed reproduction in official VS Code before triaging further.

## Change summary

Fcitx IME candidate selection number keys are passed through to the terminal as raw digits in addition to committing the selected candidate.

## Best solution

Confirm reproduction in vanilla VS Code 1.120+. If confirmed, file upstream at xtermjs/xterm.js — the fix belongs in CompositionHelper.keydown() which should suppress number keys that finalize a composition. The logic at node_modules/@xterm/xterm/src/browser/input/CompositionHelper.ts line ~110 calls _finalizeComposition(false) but then falls through to return true instead of false.
