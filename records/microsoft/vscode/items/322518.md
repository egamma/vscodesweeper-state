# Issue #322518 — Runaway thread/process growth when opening a workspace on a slow network share

[https://github.com/microsoft/vscode/issues/322518](https://github.com/microsoft/vscode/issues/322518)

## Summary

The report describes runaway resource growth on a slow network share but lacks the diagnostics needed to identify which VS Code component is responsible — no process explorer output, no logs, no extension list, and no test with extensions disabled. Requesting more info.

## Change summary

VS Code triggers unbounded thread/process/handle growth when a workspace is on a slow network share, especially when saving during ongoing directory scanning.

## Best solution

The reporter should provide a VS Code Process Explorer snapshot (Help > Open Process Explorer), the output of `code --status`, whether the issue reproduces with `--disable-extensions`, and ideally the file watcher logs (`Developer: Set Log Level > File Watcher > Trace`). Without this, the root cause could be file watching, search (ripgrep), git, an extension, or the network client itself. Once narrowed, the fix likely lives in the file watcher throttling or retry logic in `src/vs/platform/files/node/watcher/`.
