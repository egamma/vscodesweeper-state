# Issue #315972 — Invisible and inaccessible window under Wayland

[https://github.com/microsoft/vscode/issues/315972](https://github.com/microsoft/vscode/issues/315972)

## Summary

Verified duplicate: this reports the same NVIDIA + native Wayland blank/invisible window caused by Chromium GPU detection failure (VENDOR=0x0000) already tracked by the open canonical issue #280464, which covers the same Electron-level root cause.

## Change summary

VS Code window is invisible under native Wayland with NVIDIA GPU because Chromium fails to detect the GPU (VENDOR=0x0000, DEVICE=0x0000), while XWayland works fine.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/280464, which tracks the NVIDIA + Wayland blank screen after the Electron update. The root cause is in Chromium's GPU initialization under native Wayland with NVIDIA drivers; the fix will come from upstream Electron/Chromium. Workaround: use `--ozone-platform=x11` or `--disable-gpu`.
