# Issue #130995 — Blank space above integrated terminal when using decimal value for window.zoomLevel

[https://github.com/microsoft/vscode/issues/130995](https://github.com/microsoft/vscode/issues/130995)

## Summary

Still present on main: the terminal row calculation in getXtermScaledDimensions uses Math.floor, which at certain devicePixelRatio values (induced by fractional window.zoomLevel) leaves unused pixel space that appears as blank space above/below terminal content. Confirmed by a team member and still in Backlog.

## Change summary

Fractional window.zoomLevel values cause blank space above integrated terminal content due to rounding in row-height calculations.

## Best solution

The fix likely belongs in `src/vs/workbench/contrib/terminal/browser/xterm/xtermTerminal.ts` in `getXtermScaledDimensions` — either by adjusting the terminal container height to exactly match `rows * scaledLineHeight` (eliminating leftover pixels), or by having xterm.js distribute the remainder across rows. This is a long-standing layout precision issue that may require coordination with xterm.js upstream.
