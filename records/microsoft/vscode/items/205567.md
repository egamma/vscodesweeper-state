# Issue #205567 — Multi file diff editor: Last focused editor is not remembered

[https://github.com/microsoft/vscode/issues/205567](https://github.com/microsoft/vscode/issues/205567)

## Summary

Team-authored issue (hard keep-open). The multi-diff editor still lacks logic to remember and restore the last-focused inner editor when the tab regains focus. The related #204076 only added basic focus/hasFocus plumbing, not focus restoration.

## Change summary

Switching away from and back to the multi-diff editor loses focus on the previously-focused inner editor.

## Best solution

Add focus-memory state to the multi-diff editor widget: track which inner diff editor last held focus and restore it when the multi-diff editor regains focus, likely in `src/vs/workbench/contrib/multiDiffEditor/browser/multiDiffEditor.ts` and the underlying widget.
