# Issue #226194 — Scroll lists by page

[https://github.com/microsoft/vscode/issues/226194](https://github.com/microsoft/vscode/issues/226194)

## Summary

Team-authored feature request for page-scrolling in lists with long items (especially chat). The gap remains on current main — existing `list.focusPageUp/Down` only moves focus by index, not by viewport height, and `list.scrollUp/Down` scrolls by a small fixed amount. A related PR (#283973) was closed without merging. Hard keep-open: author is MEMBER.

## Change summary

Add a paging scroll command for lists that scrolls by viewport height rather than by item index, so long list items (e.g. chat responses) can be paged through efficiently.

## Best solution

Extend `focusPageUp/Down` in `src/vs/base/browser/ui/list/listWidget.ts` (or add new `list.scrollPageUp/Down` commands in `src/vs/workbench/browser/actions/listCommands.ts`) to scroll the list view by one page height when a single item spans multiple pages, rather than jumping focus to the next item index. This is particularly important for the chat view where responses are very tall.
