# Issue #115953 — Search: Arrow navigation broken

[https://github.com/microsoft/vscode/issues/115953](https://github.com/microsoft/vscode/issues/115953)

## Summary

Still reproduces on main: the search widget's clear action does not call `addToHistory()` before clearing the input, so the last search term is lost from arrow-key history navigation. Confirmed by a team member (andreamah) and correctly labeled.

## Change summary

After clearing search results, arrow-key history navigation skips the most recent search term because it was never added to the history before the input was cleared.

## Best solution

In `SearchWidget.clear()` (src/vs/workbench/contrib/search/browser/searchWidget.ts), call `this.searchInput.inputBox.addToHistory()` before clearing the input value, so the last search term is preserved in the arrow-key navigation history.
