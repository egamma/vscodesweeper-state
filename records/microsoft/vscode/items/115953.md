# Issue #115953 — Search: Arrow navigation broken

[https://github.com/microsoft/vscode/issues/115953](https://github.com/microsoft/vscode/issues/115953)

## Summary

Still reproduces on main: clearing search results does not save the current search string to the input history before wiping it, so arrow-key navigation cannot recover the last term. Confirmed by a team member in 2022 and the code path is unchanged.

## Change summary

After clearing search results, the last search string is lost from arrow-key history navigation because it is not added to the input history before the value is cleared.

## Best solution

In `SearchWidget.clear()` (src/vs/workbench/contrib/search/browser/searchWidget.ts), call `this.searchInput.inputBox.addToHistory()` before `this.searchInput.clear()` so the current value is preserved in the navigation history. A similar guard may be needed in `FindInput.clear()` (src/vs/base/browser/ui/findinput/findInput.ts).
