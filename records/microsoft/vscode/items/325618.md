# Issue #325618 — Regression: Copilot Chat MCP fails on Dataverse server due to strict tool schema validation (array parameter missing items)

[https://github.com/microsoft/vscode/issues/325618](https://github.com/microsoft/vscode/issues/325618)

## Summary

Confirmed in source: the toolSchemaNormalizer throws a fatal error for arrays missing `items`, breaking the entire MCP session. The rule should either fix the schema gracefully (like other rules do) or skip the single invalid tool instead of failing all tools.

## Change summary

A single MCP tool with an array parameter missing `items` causes a fatal throw that breaks the entire Copilot Chat MCP session, even though other tools are valid.

## Best solution

Change the array-items validation rule in `extensions/copilot/src/extension/tools/common/toolSchemaNormalizer.ts` (line ~103) to either gracefully add a default `items: {}` (like other normalizer rules fix issues in-place) or change the outer `catch` to skip the failing tool and continue processing the rest. The latter approach (skip and warn) matches the reporter's expected behavior.
