# Issue #200773 — Git - Branch on new local repo fails

[https://github.com/microsoft/vscode/issues/200773](https://github.com/microsoft/vscode/issues/200773)

## Summary

Still reproduces on main — the `branch()` method in `extensions/git/src/git.ts` unconditionally passes `--no-track` to `git checkout -b`, which fails in repos with no commits because HEAD is not a valid commit. An open community PR (#247394) exists. Author is a team member, so keeping open.

## Change summary

Creating a new Git branch via the command palette fails in a freshly initialized repo that has no commits yet.

## Best solution

Land or supersede the open PR https://github.com/microsoft/vscode/pull/247394 — the fix is to omit `--no-track` (or the implicit HEAD ref) when the repository has no commits. The change is localized to `extensions/git/src/git.ts:2174-2181`.
