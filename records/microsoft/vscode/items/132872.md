# Issue #132872 — Cannot properly run multiple instances of VS Code within separate RDP sessions

[https://github.com/microsoft/vscode/issues/132872](https://github.com/microsoft/vscode/issues/132872)

## Summary

Valid upstream issue in Electron/Chromium's single-instance lock, which does not distinguish between separate RDP sessions under the same Windows user. Already acknowledged by a team member (deepak1556) and correctly labeled — should remain open as an upstream tracker.

## Change summary

Launching VS Code in a second RDP session (same Windows user) focuses the window in the first session instead of opening a new instance.

## Best solution

Track upstream in Electron/Chromium for per-session singleton awareness on Windows RDP. The workaround (`--user-data-dir` per session) is functional but not ergonomic. A proper fix would require Electron to incorporate the Windows session ID into its single-instance lock, or VS Code to detect RDP session boundaries when deciding whether to reuse an existing instance. File or link an Electron issue if one doesn't exist.
