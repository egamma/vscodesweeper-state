# Issue #187716 — "unable to get local issuer certificate" attempting to install extensions into dev container by code-server on dev container build

[https://github.com/microsoft/vscode/issues/187716](https://github.com/microsoft/vscode/issues/187716)

## Summary

This is a specific manifestation of the long-standing CLI proxy/certificate support gap tracked by #29910. The core problem — code-server/CLI not respecting system or custom CA certificates when installing extensions — is exactly the work that canonical issue owns. A commenter in this thread (#8) already identified #29910 as the tracking issue, and the team-member workaround (NODE_EXTRA_CA_CERTS) confirms it's a known limitation rather than a novel defect.

## Change summary

The `code-server` process used during dev container builds does not respect custom/corporate CA certificates (e.g. Zscaler) when installing extensions via CLI, producing 'unable to get local issuer certificate' errors.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/29910, which tracks CLI proxy support including certificate handling. Implementing proper proxy and CA certificate propagation in CLI/server mode resolves this report. The NODE_EXTRA_CA_CERTS workaround documented by chrmarti covers the gap until then.
