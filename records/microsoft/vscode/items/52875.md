# Issue #52875 — Dragging files and folders onto an empty explorer ignores the files

[https://github.com/microsoft/vscode/issues/52875](https://github.com/microsoft/vscode/issues/52875)

## Summary

Still reproduces on current main: when files and folders are dropped together onto the empty explorer, `handleWorkspaceDrop` opens the folder(s) and returns `true`, causing `handleDrop` to exit before opening the files as editors. The bug is in `src/vs/workbench/browser/dnd.ts`.

## Change summary

Dragging a mix of files and folders onto the empty explorer ignores the files — only the folder is opened.

## Best solution

After `handleWorkspaceDrop` opens the folder(s), the remaining non-folder, non-workspace-file resources should still be opened as editors. The fix belongs in `ResourcesDropHandler.handleDrop` in `src/vs/workbench/browser/dnd.ts` — instead of returning early when `isWorkspaceOpening` is true, filter out the workspace/folder entries and continue to open the remaining files.
