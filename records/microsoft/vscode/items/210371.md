# Issue #210371 — minimap markers include `-->` in the end of the display

[https://github.com/microsoft/vscode/issues/210371](https://github.com/microsoft/vscode/issues/210371)

## Summary

This should stay open: maintainers already confirmed the current folding marker parsing only uses a prefix, and the latest maintainer comment points to a likely core fix using block comment metadata rather than closing or redirecting it.

## Change summary

Minimap folding marker labels include trailing block-comment delimiters such as `-->`, `--->`, or `*/` instead of displaying only the marker name.

## Best solution

Teach the editor folding/minimap marker label extraction to strip language block-comment suffixes using the language configuration `blockComment` metadata, likely in the folding marker parsing path, while preserving existing prefix-only marker behavior for line comments.
