# Issue #276096 — Debug API: `vscode.debug.breakpoints` returns duplicate breakpoints for the same line after deleting code blocks

[https://github.com/microsoft/vscode/issues/276096](https://github.com/microsoft/vscode/issues/276096)

## Summary

Still reproduces on main: sortAndDeDup() silently drops duplicate breakpoints without firing a 'removed' event, leaving ghost entries visible to extensions via vscode.debug.breakpoints. An open PR (#310779) addresses this.

## Change summary

vscode.debug.breakpoints returns duplicate SourceBreakpoint entries for the same line after code deletion merges breakpoint positions, because deduplicated entries are never reported as removed.

## Best solution

Land the fix in PR https://github.com/microsoft/vscode/pull/310779 (or an equivalent approach): when sortAndDeDup() removes entries via distinct(), fire _onDidChangeBreakpoints with the removed breakpoints so the extension API stays consistent.
