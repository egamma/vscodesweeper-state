# Issue #191094 — International settings in VS Code

[https://github.com/microsoft/vscode/issues/191094](https://github.com/microsoft/vscode/issues/191094)

## Summary

Valid feature request for an independent locale/date-time format setting separate from the UI language. Still unimplemented on current main — the code passes the UI `language` directly to `Intl.DateTimeFormat` with no user-overridable locale. Keeping open in the backlog.

## Change summary

Allow users to configure date/time display locale independently from the VS Code UI language.

## Best solution

Introduce a setting (e.g. `locale.dateTimeFormat` or expose `platform.locale` as a user-configurable override) that feeds into `Intl.DateTimeFormat` calls across the workbench, so users can get localized date/time formatting without switching the entire UI language. The l10n-platform area owns this.
