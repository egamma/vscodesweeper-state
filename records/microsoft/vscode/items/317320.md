## Summary

Verified duplicate of #306770, which is the open canonical issue tracking the same Ctrl+Backspace data-loss scenario in the chat input. The keybinding and confirmation dialog code are still present on main unchanged.

## Change summary

Ctrl+Backspace in the chat input discards all pending edits instead of deleting a word when the prompt text is empty, risking accidental data loss.

Consolidate on https://github.com/microsoft/vscode/issues/306770 — that issue is assigned to justschen in the Backlog with the `bug` label, and owns the work to make the Ctrl+Backspace binding less accident-prone (e.g. remove the keybinding, require a harder confirmation, or only trigger on a deliberate gesture).
