# Issue #149667 — [scss] don't autocomplete in comments

[https://github.com/microsoft/vscode/issues/149667](https://github.com/microsoft/vscode/issues/149667)

## Summary

Still unfixed on main — the `doComplete` method in `vscode-css-languageservice` has no guard for comment contexts. Team member aeschli acknowledged the gap in 2022 but no fix has landed.

## Change summary

SCSS autocomplete triggers inside comments, inserting unwanted suggestions when the user presses Enter.

## Best solution

Add a comment-context check at the top of `CSSCompletion.doComplete()` in the upstream `vscode-css-languageservice` package (microsoft/vscode-css-languageservice) to return an empty completion list when the offset falls inside a comment token.
