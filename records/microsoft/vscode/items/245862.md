# Issue #245862 — When using the command to cycle through completions, indicate if there are no alternatives

[https://github.com/microsoft/vscode/issues/245862](https://github.com/microsoft/vscode/issues/245862)

## Summary

Valid feature request: when cycling inline completions with Alt+]/Alt+[, there's no user feedback when only one alternative exists. The hints widget shows '1/1' and disables buttons, but the keyboard-only path is silent. Still unimplemented on main.

## Change summary

Provide user feedback (notification, sound, or status indicator) when cycling inline suggestions reveals no alternatives.

## Best solution

Add feedback in `ShowNextInlineSuggestionAction`/`ShowPreviousInlineSuggestionAction` (src/vs/editor/contrib/inlineCompletions/browser/controller/commands.ts) when the suggestion count is 1 — either an accessibility signal, a brief status-bar message, or enabling the toolbar hint widget automatically on cycle attempt. The hints widget already computes this state (`suggestionCount`); the keyboard commands just don't surface it.
