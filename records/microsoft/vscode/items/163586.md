## Summary

Team-authored polish request still unimplemented on main — the search tree uses WorkbenchCompressibleAsyncDataTree but lacks the CompressedNavigationController that the explorer uses for individual folder selection on compressed nodes. Keeping open per hard keep-open rule (MEMBER author).

## Change summary

Allow selecting individual folders within compressed folder paths in the search results tree, matching the explorer's behavior.

## Best solution
