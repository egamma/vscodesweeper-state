# Issue #320654 — MCP server connection keep prompting for authentication using dynamic client registration when attempting to fetch resource metadata.

[https://github.com/microsoft/vscode/issues/320654](https://github.com/microsoft/vscode/issues/320654)

## Summary

Legitimate bug: when a user provides a static Authorization header in MCP server config, VS Code still attempts OAuth/DCR discovery on 401 responses (e.g. missing session ID), causing repeated authentication prompts that block usage.

## Change summary

MCP HTTP client triggers OAuth dynamic client registration prompts even when the user has configured a static Authorization header, making the server unusable.

## Best solution

In `_fetchWithAuthRetry` (src/vs/workbench/api/common/extHostMcp.ts), when user-supplied headers already include an Authorization value, skip the `createAuthMetadata` OAuth discovery flow and do not prompt for DCR. The retry logic at line 829–834 should also not force a new registration when the original auth was user-supplied rather than OAuth-derived.
