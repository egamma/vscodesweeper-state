# Issue #156013 — I can't scroll screen using Windows Ink

[https://github.com/microsoft/vscode/issues/156013](https://github.com/microsoft/vscode/issues/156013)

## Summary

Keep open: this is an accepted Backlog feature request with enough community upvotes, and current editor pointer handling still treats pen drag as selection rather than scroll. There is no confirmed duplicate or fixing commit in the provided discussion or local history search.

## Change summary

Allow scrolling editor content with a Windows Ink pen/stylus drag instead of only selecting text.

## Best solution

Decide in the editor input/scroll area whether pen gestures should pan the editor, be configurable, or remain selection-oriented; the likely implementation point is `src/vs/editor/browser/controller/pointerHandler.ts` where touch gestures scroll but pen gestures dispatch selection.
