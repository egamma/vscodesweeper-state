# Issue #156013 — I can't scroll screen using Windows Ink

[https://github.com/microsoft/vscode/issues/156013](https://github.com/microsoft/vscode/issues/156013)

## Summary

Valid community-promoted feature request (24 upvotes) for Windows Ink pen scrolling support. No fix or implementation found on current main — the scrollbar infrastructure handles wheel events and pointer events for dragging, but doesn't translate pen pointer-move gestures into scroll actions.

## Change summary

Scrolling with a Windows Ink pen device does not work in VS Code editors and views.

## Best solution

Likely needs handling in the scrollable element layer (`src/vs/base/browser/ui/scrollbar/scrollableElement.ts`) to recognize pen pointer events as scroll gestures, or may require an Electron/Chromium-level fix for how Windows Ink pointer events are dispatched as wheel events. Investigation should determine whether Chromium already translates these events and VS Code is suppressing them, or whether explicit pen-scroll handling is needed.
