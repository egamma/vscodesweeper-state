# Issue #311990 — SCM inline toolbar actions leak between providers on template reuse

[https://github.com/microsoft/vscode/issues/311990](https://github.com/microsoft/vscode/issues/311990)

## Summary

Confirmed on current main: connectPrimaryMenu initializes its action cache as empty arrays, so when a recycled template moves to a provider with zero inline actions, equals([],[]) returns true and the stale toolbar is never cleared.

## Change summary

SCM inline toolbar actions from one provider leak onto another provider's resource items on template reuse when the new provider contributes no inline actions.

## Best solution

Initialize cachedPrimary/cachedSecondary as undefined in connectPrimaryMenu (src/vs/workbench/contrib/scm/browser/util.ts line 90–91) so the first invocation always falls through to the callback, clearing stale actions.
