# Issue #170347 — iPad: Cmd-C Cmd-X shortcuts for copying the current line don't work

[https://github.com/microsoft/vscode/issues/170347](https://github.com/microsoft/vscode/issues/170347)

## Summary

Upstream WebKit/Safari bug: iPad Safari does not fire the `copy` event when Cmd-C is pressed with no text selected, so VS Code's 'copy current line' feature cannot work. The reporter filed Apple FB11920278. No VS Code-side workaround has been implemented.

## Change summary

On iPad, Cmd-C/Cmd-X with no selection should copy/cut the current line, but the browser never fires the copy event in that scenario.

## Best solution

This is blocked on WebKit fixing the missing copy event on iPadOS (Apple FB11920278). A VS Code workaround could intercept the keyboard shortcut directly and programmatically trigger the copy with the expanded line selection, but that would be fragile. Best tracked here until Apple resolves it or the team decides to implement a keyboard-event-based workaround in the editor's iPad input handling.
