# Issue #170347 — iPad: Cmd-C Cmd-X shortcuts for copying the current line don't work

[https://github.com/microsoft/vscode/issues/170347](https://github.com/microsoft/vscode/issues/170347)

## Summary

Keep open as an iPadOS/WebKit clipboard interoperability bug rather than a cleanup close. The reporter provided a minimal Safari/iPadOS repro and current VS Code web clipboard handling still depends on the browser firing the copy/cut event for this path.

## Change summary

Cmd-C/Cmd-X on vscode.dev for iPad should copy or cut the current line when the editor selection is empty.

## Best solution

Track this under iOS/iPadOS editor clipboard behavior, with the preferred fix coming from WebKit/Safari dispatching the expected textarea copy/cut event. If VS Code decides to work around it, the likely area is the editor web clipboard command path in src/vs/editor/contrib/clipboard/browser/clipboard.ts and the edit-context clipboard event handling.

## Evidence

- **issue repro:** The report is specifically vscode.dev Preview on iPadOS 16.1: pressing Cmd-C or Cmd-X with no selection does not put the current line on the pasteboard.
- **upstream signal:** The reporter's follow-up says a minimal textarea copy-event repro fails on iPad Safari but works on desktop Safari, and they filed Apple Feedback FB11920278 for inconsistent Safari textarea copy event behavior.
- **current code:** src/vs/editor/contrib/clipboard/browser/clipboard.ts:81-88 still intentionally does not bind copy keybindings in the browser because browsers handle those shortcuts and this avoids security prompts.
- **event dependency:** src/vs/editor/browser/controller/editContext/textArea/textAreaEditContextInput.ts:389-407 and src/vs/editor/browser/controller/editContext/native/nativeEditContext.ts:118-133 populate editor clipboard data from copy events; if iPad Safari does not fire that event, this path is not reached.
- **no completed-fix proof:** The current fallback in src/vs/editor/contrib/clipboard/browser/clipboard.ts:194-208 only writes directly to the clipboard for platform.isNative when execCommand('copy') fails, so there is no concrete current-main fix for vscode.dev/iPadOS to cite.
