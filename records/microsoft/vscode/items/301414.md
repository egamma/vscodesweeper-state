# Issue #301414 — Shared Copilot Chat history across multiple workspaces/projects

[https://github.com/microsoft/vscode/issues/301414](https://github.com/microsoft/vscode/issues/301414)

## Summary

Valid feature request with strong community support (33 upvotes, promoted to Backlog). Chat sessions are stored per-workspace (StorageScope.WORKSPACE in chatSessionStore.ts), so there is no current mechanism for cross-workspace chat access. Related issue #285106 covers management/visibility of orphaned chat data but is distinct enough — this one focuses on active cross-workspace session access for workflow continuity.

## Change summary

Allow Copilot Chat history to be accessed across multiple workspaces/projects rather than being siloed per workspace.

## Best solution

Introduce a global or profile-scoped chat session index (moving from StorageScope.WORKSPACE to StorageScope.APPLICATION or PROFILE for the session index in `src/vs/workbench/contrib/chat/common/model/chatSessionStore.ts`) so users can browse and resume sessions from any workspace. This likely involves changes to session storage, the session picker UI, and possibly the agents window session list. Consider linking sessions to repository identity rather than workspace path.
