# Issue #216626 — Deleting lines when editor.linkedEditing is enabled leads to inconsistent results

[https://github.com/microsoft/vscode/issues/216626](https://github.com/microsoft/vscode/issues/216626)

## Summary

Still reproduces on main: when a whole-line delete removes an HTML closing tag, the linked-editing sync logic propagates a corrupted edit to the opening tag. No fix has landed.

## Change summary

Deleting a line containing a linked-editing range (e.g. an HTML closing tag) incorrectly mutates the paired opening tag.

## Best solution

Guard _syncRanges in src/vs/editor/contrib/linkedEditing/browser/linkedEditing.ts against whole-line deletions that destroy the decoration range entirely — when the reference range is deleted (not just edited), clear ranges instead of propagating a sync edit.
