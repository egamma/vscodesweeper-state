# Issue #167467 — Diff Editor: Support Go To Line In Original Text Editor

[https://github.com/microsoft/vscode/issues/167467](https://github.com/microsoft/vscode/issues/167467)

## Summary

Valid accessibility-motivated feature request. The issue remains on current main: `getCodeEditor()` in `editorBrowser.ts` always resolves to `getModifiedEditor()` for diff editors, so Ctrl+G never targets the original (left) side. No fix or workaround has been implemented. The item is low-traction (0 upvotes) but sits in Backlog with accessibility relevance.

## Change summary

Support Go to Line (Ctrl+G) in the original (left-side) editor of the diff view, not only the modified (right-side) editor.

## Best solution

The Go to Line quick access provider should respect which side of the diff editor currently has focus. When the original editor has focus, `getCodeEditor` (or the provider's editor resolution) should route to the original editor rather than always defaulting to the modified editor. The UX challenge noted by bpasero (the picker floats above both sides with no visual indicator of which it targets) would need a design decision — either scoping the picker visually, or simply respecting keyboard focus as the routing signal.
