# Issue #178577 — terminal.integrated.env.linux.PATH and terminal.integrated.env.osx.PATH settings break tasks

[https://github.com/microsoft/vscode/issues/178577](https://github.com/microsoft/vscode/issues/178577)

## Summary

Still reproduces on main: `terminal.integrated.env.linux` (and osx) with `${env:PATH}` correctly enriches PATH for regular integrated terminals but tasks receive a reduced/basic PATH. The task terminal creation path in `terminalTaskSystem.ts` sets `useShellEnvironment: true` but the variable resolution context for `${env:PATH}` may differ from what the regular terminal uses, particularly in WSL or remote scenarios.

## Change summary

Setting `terminal.integrated.env.linux.PATH` (with `${env:PATH}`) correctly extends PATH for integrated terminals but tasks receive only the basic system PATH.

## Best solution

Investigate why the variable resolver used during `createTerminalEnvironment` for task-spawned terminals resolves `${env:PATH}` to a reduced PATH compared to regular terminals. The fix likely belongs in `src/vs/workbench/contrib/tasks/browser/terminalTaskSystem.ts` or in how the terminal process manager constructs `envForResolver` when `shellLaunchConfig.isFeatureTerminal` is true. Ensure tasks and regular terminals use the same resolution context for `terminal.integrated.env.*` settings.
