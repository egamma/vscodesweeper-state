# Issue #324809 — Copilot Chat leaks the window renderer to OOM (-536870904) on idle Remote-SSH; disabling Copilot Chat is a confirmed fix (1.127.0)

[https://github.com/microsoft/vscode/issues/324809](https://github.com/microsoft/vscode/issues/324809)

## Summary

Well-documented report of an idle Remote-SSH window renderer OOM crash driven by the bundled Copilot Chat extension. The reporter isolated the leak with process-level measurements and confirmed disabling Copilot Chat eliminates it. Related to several open issues (#303387, #313762, #294050) but distinct in trigger (idle + restored chat session, not active chat). Needs routing to the Copilot area for investigation.

## Change summary

The VS Code window renderer leaks memory to OOM (~0.6–1.2 GB/min) on idle Remote-SSH sessions when Copilot Chat is enabled and a chat-session view is restored, crashing with code -536870904 every ~4 minutes.

## Best solution

Route to the Copilot team to investigate the idle-session memory leak path in the bundled @github/copilot SDK and its chat-session restoration logic. The reporter's fingerprint (384 tokio-rt-worker threads, timer-driven `setItems` error, RSS growth with zero clients) gives a concrete starting point. Related open issues #303387 and #294050 may share a root cause but have different triggers; consolidation should happen after the Copilot team triages.
