# Issue #241525 — Feature request: Fuzzy search files including file path

[https://github.com/microsoft/vscode/issues/241525](https://github.com/microsoft/vscode/issues/241525)

## Summary

Legitimate feature request for improved multi-term fuzzy file search against full paths. The underlying scoring infrastructure already supports space-separated multi-term matching against paths, but the file search pipeline has practical limitations in large workspaces: only the first term is used for the initial file search query (limited to 512 results), and relevant files may not surface when the first term is too broad. With 32 upvotes and promotion to Backlog, this should remain open.

## Change summary

Improve Quick Open (Ctrl+P) fuzzy file search so that multiple space-separated terms reliably match against the full file path, especially in large codebases where the search result limit truncates relevant matches.

## Best solution

Improve the file search pipeline in `anythingQuickAccess.ts` to better leverage all query terms when issuing the underlying file search (currently only the first of multiple terms is sent as `filePattern`). The fuzzy scoring in `fuzzyScorer.ts` already handles multi-term path matching correctly; the gap is in the initial file-search query construction and result-limit handling. Related ranking improvements tracked in https://github.com/microsoft/vscode/issues/25925 and https://github.com/microsoft/vscode/issues/27317 overlap but don't fully absorb this specific multi-term path-matching ask.
