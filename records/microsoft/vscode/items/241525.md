# Issue #241525 — Feature request: Fuzzy search files including file path

[https://github.com/microsoft/vscode/issues/241525](https://github.com/microsoft/vscode/issues/241525)

## Summary

This request for better multi-term fuzzy matching against full file paths in Quick Open is a specific case of the long-standing canonical tracking issue #27317, which explicitly lists path-matching and ranking improvements. Proposing a duplicate close to consolidate signal.

## Change summary

Improve Quick Open (⌘+P) to better support multi-term fuzzy search matching against the full file path, not just the filename.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/27317 which tracks all quick open ranking and matching improvements, including the sub-issues about path matching (#116021, #25925, #30404). The current implementation already splits space-separated terms and scores each against the full path (label + description) via `doScoreItemFuzzyMultiple` in `fuzzyScorer.ts`, but the file search backend (`getFileSearchResults`) only sends the first term to ripgrep, which can miss results in large workspaces — this limitation is part of the broader ranking work tracked there.
