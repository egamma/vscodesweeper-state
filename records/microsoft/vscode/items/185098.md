# Issue #185098 — Add proxy support to `http2` requests in the extension host

[https://github.com/microsoft/vscode/issues/185098](https://github.com/microsoft/vscode/issues/185098)

## Summary

Team-authored tracking issue for http2 proxy support in the extension host. The maintainer's latest comment (May 2025) indicates no current work is planned for the `http2` module directly, since proxy support was added to `fetch` (which itself supports http2 via undici). Keeping open per team-authored rule.

## Change summary

Add proxy server support for Node.js `http2` module requests made by extensions in the extension host.

## Best solution

This remains a valid tracking issue for a niche gap — extensions using the raw `http2` module behind a proxy still lack support. The maintainer has deprioritized it in favor of the `fetch`-based path. Keep open as a backlog feature request; if `fetch` with http2 enabled (pending upstream undici work) covers the remaining use cases, the maintainer may close it as resolved-by-alternative.
