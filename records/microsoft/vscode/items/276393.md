# Issue #276393 — VSCode sandbox broken - fails to access remote devices, copilot and other networking on macOS 26.1

[https://github.com/microsoft/vscode/issues/276393](https://github.com/microsoft/vscode/issues/276393)

## Summary

Active investigation by a team member (deepak1556) into macOS 26.1 sandbox violations blocking network access for Code Helper processes. The root cause is a macOS platform change tightening sandbox enforcement; VS Code may need to adjust its entitlements or sandbox configuration. Keeping open as under-discussion.

## Change summary

VS Code's sandbox configuration fails to allow network access on macOS 26.1, with Code Helper and Code Helper (GPU) receiving deny violations for file-read-data and mach-lookup operations.

## Best solution

deepak1556 is actively diagnosing the specific sandbox violations. The fix likely involves updating the helper entitlements in `build/azure-pipelines/darwin/helper-entitlements.plist` and/or `helper-gpu-entitlements.plist` to accommodate macOS 26.1's stricter sandbox enforcement. The current helper entitlements only grant `com.apple.security.cs.allow-jit` — additional entitlements for network access or mach-lookup may be needed.
