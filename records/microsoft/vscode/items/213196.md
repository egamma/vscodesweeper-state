# Issue #213196 — workbench.explorer.fileView.toggleVisibility does not work

[https://github.com/microsoft/vscode/issues/213196](https://github.com/microsoft/vscode/issues/213196)

## Summary

The command does nothing because the Explorer file view is intentionally registered with `canToggleVisibility: false` — it is the primary view in the Explorer sidebar and hiding it would leave the container empty. This is as-designed.

## Change summary

The command `workbench.explorer.fileView.toggleVisibility` does nothing when invoked.

## Best solution

Close as designed. The Explorer file view (`workbench.explorer.fileView`) sets `canToggleVisibility: false` because it is the only non-optional view in the Explorer sidebar — hiding it would leave the container non-functional. The `toggleVisibility` command exists generically for all registered views, but its precondition is `ContextKeyExpr.false()` for views that disallow toggling. Users wanting to hide the Explorer sidebar entirely should use the sidebar toggle command instead.
