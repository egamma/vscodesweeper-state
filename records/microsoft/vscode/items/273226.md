## Change summary

HTML completions in unclosed string literals delete text beyond the intended replacement range when the scanner finds an unrelated closing quote later in the file.

## Best solution

Land the upstream fix (microsoft/vscode-html-languageservice#247) which clamps the replacement range when an unescaped `<` appears between the cursor and the matched closing quote, then bump the dependency in vscode.
