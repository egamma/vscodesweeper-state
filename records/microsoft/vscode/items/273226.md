# Issue #273226 — [html] completions: better handle unclosed string literals

[https://github.com/microsoft/vscode/issues/273226](https://github.com/microsoft/vscode/issues/273226)

## Summary

Mid-verification (`verification-found`): the initial fix in vscode-html-languageservice#239 was incomplete — it still reproduces when another `"` exists later in the file. A follow-up fix (vscode-html-languageservice#247) is open but not yet merged.

## Change summary

Accepting an HTML attribute completion inside an unclosed string literal deletes subsequent markup (e.g. `</th>`) when the scanner latches onto a later unrelated quote character.

## Best solution

Merge the follow-up fix in https://github.com/microsoft/vscode-html-languageservice/pull/247 (clamp the completion range when an unescaped `<` appears between the cursor and the matched closing quote), then bump the vscode-html-languageservice dependency in VS Code and re-verify.
