# Issue #280076 — when downvoting copilot chat response multi selection is not possible

[https://github.com/microsoft/vscode/issues/280076](https://github.com/microsoft/vscode/issues/280076)

## Summary

This is a polish request to allow multi-selecting reasons when downvoting a Copilot Chat response. The current downvote is a simple thumbs-down toggle with no reason-picker UI at all — the request is for richer feedback granularity, not a broken feature. Already labeled `polish`, which fits.

## Change summary

Allow selecting multiple reasons when downvoting a Copilot Chat response, rather than a single toggle.

## Best solution

The feedback UX for downvotes could be enhanced with a multi-select reason picker (e.g. 'inaccurate', 'incomplete', 'irrelevant') in `src/vs/workbench/contrib/chat/browser/actions/chatTitleActions.ts` and the `IChatVoteAction` interface. This is a product decision about how much feedback granularity to collect — worth discussing with the Copilot Chat team.
