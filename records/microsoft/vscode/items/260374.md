# Issue #260374 — Chat: Setting to gitignore files that Copilot can still access

[https://github.com/microsoft/vscode/issues/260374](https://github.com/microsoft/vscode/issues/260374)

## Summary

Team-authored feature request for a chat-specific gitignore setting, decoupled from explorer.excludeGitIgnore. The gap is confirmed on current main — chat file resolution still reads `explorer.excludeGitIgnore` directly with no chat-specific override. Keeping open per team-author policy.

## Change summary

Provide a chat/agent-specific setting to control whether gitignored files are accessible to Copilot, independent of the explorer's excludeGitIgnore setting.

## Best solution

Introduce a new setting (e.g. `chat.excludeGitIgnore`) that controls whether chat/agent file resolution respects .gitignore, defaulting to off (as the issue suggests). The main touch points are `src/vs/workbench/contrib/chat/common/promptSyntax/utils/promptFilesLocator.ts` (lines 677, 718) and `src/vs/workbench/contrib/search/browser/searchChatContext.ts` (line 245), which currently read `explorer.excludeGitIgnore` and should fall back to the new chat-specific setting.
