# Issue #69335 — Investigation: do not restart Extension host when first folder changes

[https://github.com/microsoft/vscode/issues/69335](https://github.com/microsoft/vscode/issues/69335)

## Summary

Still valid on current main: `WorkspaceChangeExtHostRelauncher` restarts the extension host when the first workspace folder changes (with a sessions-window carve-out added in Feb 2026). The original 2019 fix was reverted and no replacement has shipped for the general case.

## Change summary

Avoid restarting the extension host when the first folder in a multi-root workspace changes, since `workspace.rootPath` is deprecated.

## Best solution

Remove `WorkspaceChangeExtHostRelauncher` for the general workbench case (not just sessions window) once the ecosystem has migrated off `workspace.rootPath`. The API-proposal and under-discussion labels are appropriate — this needs a product decision on the remaining extension ecosystem impact before proceeding.
