# Issue #153500 — Git - Should focus return to the SCM view after closing COMMIT_EDITMSG?

[https://github.com/microsoft/vscode/issues/153500](https://github.com/microsoft/vscode/issues/153500)

## Summary

Still relevant: the git editor handler in `gitEditor.ts` resolves its promise when COMMIT_EDITMSG closes but performs no focus restoration to the SCM view. The sibling issue #153499 (focus return to terminal) is also still open. Keeping open as a valid polish item.

## Change summary

After saving and closing the COMMIT_EDITMSG editor, focus should return to the SCM view input area so users can access actions like Publish Branch via keyboard.

## Best solution

After the tab-close promise resolves in `extensions/git/src/gitEditor.ts` `handle()`, programmatically focus the SCM view's input area. The sibling issue https://github.com/microsoft/vscode/issues/153499 asks for the same behavior for the terminal — both could share a pattern where the originating view is remembered and re-focused on close.
