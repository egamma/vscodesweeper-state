# Issue #242712 — Line wrapping in terminal search

[https://github.com/microsoft/vscode/issues/242712](https://github.com/microsoft/vscode/issues/242712)

## Summary

Still reproduces on main — the terminal find widget's flex container lacks a `nowrap` or `overflow` guard, so the matches-count text can force a line break when the result count string is wide. Milestone is 'On Deck', so keeping open per policy.

## Change summary

The terminal find widget's 'X of Y' result count text causes a line break in the find bar when the count string is long enough.

## Best solution

Add `flex-wrap: nowrap` (or `white-space: nowrap` plus `overflow: hidden`) to the `.simple-find-part` container in `src/vs/workbench/contrib/codeEditor/browser/find/simpleFindWidget.css` to prevent the matches count from wrapping the widget layout. The fixed-width `.matchesCount` (73px) may also need widening or text-overflow handling for very large result counts.
