# Issue #318208 — Files in files.exclude appear as untracked in the explorer

[https://github.com/microsoft/vscode/issues/318208](https://github.com/microsoft/vscode/issues/318208)

## Summary

Feature request: when an excluded file is opened and becomes visible in the explorer, its git decoration (e.g. green 'untracked') overrides the deemphasized styling. The user wants `files.exclude` to suppress git status decorations for those files. Current behavior is as-designed — the explorer decoration provider dims excluded items, but the git decoration provider contributes independently.

## Change summary

Files matched by `files.exclude` that are opened via ⌘+O appear in the explorer with git status decorations (green untracked) instead of staying deemphasized.

## Best solution

The decoration compositing in `src/vs/workbench/services/decorations/browser/decorationsService.ts` would need to let the explorer's 'excluded' decoration take priority over (or suppress) git status decorations for `isExcluded` items. Alternatively, this could be a new setting like `files.excludeGitDecorations`. Worth discussing whether this belongs in the explorer filter or the decoration compositing layer. Area: `file-explorer` and `file-decorations`.
