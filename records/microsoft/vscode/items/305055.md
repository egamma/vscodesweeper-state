# Issue #305055 — Sash hover briefly clears before both sashes highlight when approaching a T-junction from the through-going edge

[https://github.com/microsoft/vscode/issues/305055](https://github.com/microsoft/vscode/issues/305055)

## Summary

Confirmed visual bug in sash hover at T-junctions: the through-going sash loses hover when the pointer enters the orthogonal drag handle overlay. An open PR (#305279) exists to fix it — keeping open pending that PR's resolution.

## Change summary

Sash hover briefly flickers off when approaching a T-junction from the through-going edge due to independent mouseenter/mouseleave handlers and the orthogonal drag handle's z-stacking.

## Best solution

Land the fix in PR https://github.com/microsoft/vscode/pull/305279, which addresses this flicker. The fix likely needs to either suppress the through-going sash's mouseleave when the pointer moves into the orthogonal drag handle, or unify the hover state tracking across the junction.
