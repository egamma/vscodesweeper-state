# Issue #267221 — cd prefix simplification isn't handling absolute paths

[https://github.com/microsoft/vscode/issues/267221](https://github.com/microsoft/vscode/issues/267221)

## Summary

The cd prefix simplification bug remains unfixed on main — no commit references this issue, and the string comparison in CommandLineCdPrefixRewriter still lacks proper absolute path resolution against the cwd URI. Keeping open as a valid bug.

## Change summary

The terminal chat's cd prefix simplification fails to remove a redundant `cd <absolute-path>` prefix when the absolute path matches the terminal's cwd.

## Best solution

Fix the path comparison in `src/vs/workbench/contrib/terminalContrib/chatAgentTools/browser/tools/commandLineRewriter/commandLineCdPrefixRewriter.ts` — the rewriter should resolve the extracted directory against the cwd URI (similar to how the display code at line 1196–1201 of `runInTerminalTool.ts` already does with `isAbsolute` + `URI.from`/`URI.joinPath`) before comparing, rather than relying on a raw string equality between the command's path text and `cwd.fsPath`.
