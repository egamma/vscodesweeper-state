# Issue #241968 — `i` should be marked as an alias of `install`

[https://github.com/microsoft/vscode/issues/241968](https://github.com/microsoft/vscode/issues/241968)

## Summary

Team-authored feature request still open — subcommand alias marking (e.g. npm `i` → `install`) is not yet implemented on main. The completion model handles alias sorting but does not mark subcommand aliases in suggestions.

## Change summary

Terminal suggest should display `i` as an alias of `install` for npm subcommands.

## Best solution

Extend the terminal suggest completion model to support alias annotations on subcommands (not just top-level commands), likely in terminalCompletionModel.ts and the npm completion spec. Tyriar's comment outlines the approach: apply alias treatment to subcommands where the distinction is clear (e.g. npm help doesn't list `i`).
