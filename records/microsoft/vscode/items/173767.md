## Summary

Still open on main: the command palette sorting logic in `compareEntries` uses `compareAnything` (alphabetical with basic boost) but does not prioritize exact word matches or shorter labels that fully contain the query. Team-authored issue with ongoing maintainer discussion — keeping open.

## Change summary

Command palette sorting shows alphabetically-earlier but less relevant commands above exact/prefix matches (e.g. typing 'Profiles: ' shows 'Developer: Reset...' first).
