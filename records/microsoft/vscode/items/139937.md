# Issue #139937 — Add arguments property to npm tasks

[https://github.com/microsoft/vscode/issues/139937](https://github.com/microsoft/vscode/issues/139937)

## Summary

Still valid: the npm task type in `extensions/npm/src/tasks.ts` defines `INpmTaskDefinition` with only `script` and `path` — no `args` property exists. The feature remains unimplemented on current main, and the related open issue #176277 tracks a closely related (but Mac-specific) bug rather than this general args-property request.

## Change summary

Add an `args` property to npm-type tasks so users can pass command-line arguments to npm scripts without converting to shell tasks.

## Best solution

Add an `args?: string[]` property to `INpmTaskDefinition` in `extensions/npm/src/tasks.ts` and wire it into the `ShellExecution` construction (line ~345) so that arguments are appended after `--`. Issue #176277 tracks the same underlying gap on Mac and could be closed as a duplicate of this one once the feature lands.
