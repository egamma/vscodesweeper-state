# Issue #179203 — vscode.workspace.findFiles returns nothing if executed early in extension startup

[https://github.com/microsoft/vscode/issues/179203](https://github.com/microsoft/vscode/issues/179203)

## Summary

Team-authored, confirmed bug that remains unfixed on main — a race condition where `vscode.workspace.findFiles` returns empty results when called early in extension activation. Still reported as late as v1.118.0. Keeping open per hard keep-open rules (MEMBER author) and because no fix has landed.

## Change summary

`vscode.workspace.findFiles` returns no results when invoked early during extension activation, likely due to a race with search provider readiness.

## Best solution

The file search path needs a readiness barrier so that `findFiles` calls issued during early extension activation wait for the file search provider to be fully initialized before executing the query. The fix likely belongs in the search service layer (`src/vs/workbench/services/search/`) or the ext-host workspace proxy (`src/vs/workbench/api/common/extHostWorkspace.ts`). The search area label is already applied.
