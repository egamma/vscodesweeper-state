# Issue #26338 — [html] Javascript intellisense in inline scripts from external references

[https://github.com/microsoft/vscode/issues/26338](https://github.com/microsoft/vscode/issues/26338)

## Summary

Still a valid open feature request. The HTML language server's JS mode hardcodes only the current document URI and 'jquery' as script file names — no resolution of external <script src> references or project-level imports exists on current main.

## Change summary

JavaScript intellisense inside inline <script> tags does not resolve symbols from externally referenced scripts (via <script src> or imports).

## Best solution

Extend the TypeScript language service host in `extensions/html-language-features/server/src/modes/javascriptMode.ts` to parse <script src="..."> tags from the HTML document and include those resolved file paths in `getScriptFileNames`, enabling cross-file intellisense within inline script blocks. Alternatively, integrate with jsconfig.json/tsconfig.json project context for the embedded JS virtual document.
