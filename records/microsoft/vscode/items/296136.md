# Issue #296136 — Diff editor select and git stage issues with code deletions

[https://github.com/microsoft/vscode/issues/296136](https://github.com/microsoft/vscode/issues/296136)

## Summary

The issue reports two problems: (1) inline diff selection for deleted lines behaves like plain-text selection rather than VS Code's native selection model — this is a distinct report from the original feature (#8226, shipped Oct 2025) and needs area-owner attention; (2) inability to stage a subset of deleted lines, which is already tracked by #9475. Routing to the diff-editor area for the selection behavior part.

## Change summary

In the inline diff view, selecting deleted (red) lines uses plain-text–style selection instead of VS Code's native selection model, and deleted-line ranges cannot be staged via the context menu.

## Best solution

The diff-editor area owner should investigate the inline-view selection model for deleted content (rendered via view zones in `src/vs/editor/browser/widget/diffEditor/components/diffEditorViewZones/`). The selection in inline deleted zones may need to integrate with the editor's native selection rather than relying on browser text selection. The staging limitation is already tracked by https://github.com/microsoft/vscode/issues/9475 and should be referenced in a reply.
