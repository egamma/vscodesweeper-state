# Issue #202642 — When using monospaced Chinese fonts, the auto line-wrapping in simple mode is incorrect.

[https://github.com/microsoft/vscode/issues/202642](https://github.com/microsoft/vscode/issues/202642)

## Summary

The report describes a real editor wrapping mismatch for CJK-width fonts when `editor.wrappingStrategy` is `simple`, and there is not enough evidence in the provided discussion or local code history to close it as fixed, duplicate, or as-designed.

## Change summary

Auto line wrapping in the editor's simple wrapping strategy miscalculates line breaks when a font makes one Chinese character exactly two Latin character widths.

## Best solution

Keep this in the editor wrapping area and have the editor owner decide whether `simple` wrapping should account for full-width CJK glyphs or whether documentation/settings UI should more explicitly direct these font cases to `advanced` wrapping.
