# Issue #312578 — workbench.secondarySideBar.defaultVisibility is ignored in Agents

[https://github.com/microsoft/vscode/issues/312578](https://github.com/microsoft/vscode/issues/312578)

## Summary

Confirmed on current main: the Agents window's desktop layout controller never consults `workbench.secondarySideBar.defaultVisibility` — it defaults the aux bar to visible when no per-session state exists, ignoring the user's setting.

## Change summary

Opening a new Copilot session in the Agents window always shows the secondary sidebar, even when `workbench.secondarySideBar.defaultVisibility` is set to `hidden`.

## Best solution

In `desktopSessionLayoutController.ts`, when `_newSessionViewState` is undefined (no prior explicit user toggle), read `workbench.secondarySideBar.defaultVisibility` from the configuration service and use it to decide whether to open or hide the aux bar for a new session.
