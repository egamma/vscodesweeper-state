# Issue #323388 — Feature Request: Native support ${num:$Snippet(prefix)} to nest other code snippets & trigger snippets inside quotes/brackets

[https://github.com/microsoft/vscode/issues/323388](https://github.com/microsoft/vscode/issues/323388)

## Summary

Legitimate feature request for declarative snippet composition (referencing one snippet from another by prefix). No such syntax exists in the native snippet grammar; the existing nesting support is purely runtime (inserting a snippet while another is active). Keeping open for the community process.

## Change summary

Request native snippet body syntax to reference and inline other snippets by prefix, plus allow snippet triggering inside quotes/brackets.

## Best solution

This is a multi-part request: (1) a new snippet grammar construct for snippet-to-snippet references, (2) expanding suggest trigger contexts inside strings/brackets, and (3) independent placeholder navigation. Part 2 may already be partially addressable via `editor.quickSuggestions` string settings. Parts 1 and 3 would require significant changes to the snippet parser (`src/vs/editor/contrib/snippet/browser/snippetParser.ts`) and session. Given 0 upvotes and niche audience, let the community process decide; alternatively, an extension using the CompletionItemProvider API could approximate this.
