# Issue #175935 — Quick open renders icons from file names

[https://github.com/microsoft/vscode/issues/175935](https://github.com/microsoft/vscode/issues/175935)

## Summary

Still reproduces on main: quick open's IconLabel is created with `supportIcons: true`, so file names containing codicon syntax (e.g. `$(zap).ts`) are rendered as icons instead of literal text. No fix has been applied.

## Change summary

File names containing codicon syntax like `$(zap)` are rendered as actual icons in Quick Open instead of showing the literal file name.

## Best solution

Escape codicon sequences in file-name labels before passing them to the quick pick. The label assigned in `createAnythingPick` (src/vs/workbench/contrib/search/browser/anythingQuickAccess.ts ~line 1027) should pass through `escapeIcons()` from `vs/base/common/iconLabels`. Alternatively, the QuickInputList renderer could disable icon rendering for file-sourced labels, though that's a broader change.
