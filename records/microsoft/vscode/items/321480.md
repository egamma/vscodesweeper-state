# Issue #321480 — Pasting occurs before confirmation

[https://github.com/microsoft/vscode/issues/321480](https://github.com/microsoft/vscode/issues/321480)

## Summary

Team-authored bug, must keep open. The defect is traceable on current main: when pasting between remote windows, two paste paths fire — the DOM paste-event listener (line 337–344 of explorerView.ts, which triggers the native confirmation dialog) and the keybinding-dispatched command (which reads from clipboardService without the fileList parameter and therefore bypasses the `confirmPasteNative` guard). The result is that the file is pasted via the non-native path regardless of the confirmation answer.

## Change summary

Pasting a file between remote VS Code windows proceeds even when the user declines the confirmation dialog.

## Best solution

Ensure only one paste path fires per Cmd+V, or gate the clipboard-service path behind the same confirmation. The recent guard attempt (3d616509df4, reverted by 8ff8f516aa8 because it broke external pastes) shows this is a known area of tension — a more robust solution needs to coordinate the DOM paste listener in `explorerView.ts` with the `pasteFileHandler` in `fileActions.ts` so the confirmation is authoritative.

## Evidence

- **dual paste paths:** src/vs/workbench/contrib/files/browser/views/explorerView.ts:337-344 listens for DOM paste events and calls `filesExplorer.paste` with `event.clipboardData.files`. The keybinding for Cmd+V also dispatches `filesExplorer.paste` without a fileList, so `pasteFileHandler` (fileActions.ts:1123) runs twice — once with native files (confirmation shown) and once via clipboardService (confirmation skipped because `hasNativeFilesToPaste` is false).
- **guard attempt and revert:** Commit 3d616509df4 ('Fix explorer paste #323002') added a guard, but 8ff8f516aa8 reverted it on main because the async await invalidated the DataTransfer, breaking external OS file pastes (#323858). The double-fire issue remains unresolved.
- **test plan item:** Filed while testing #321354 (remote-to-local copy/paste refactor by lramos15), confirming this is a regression from that work.
