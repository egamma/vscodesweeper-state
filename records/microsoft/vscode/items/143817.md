# Issue #143817 — Allow for disabling certain languages from auto language detection

[https://github.com/microsoft/vscode/issues/143817](https://github.com/microsoft/vscode/issues/143817)

## Summary

Team-authored feature request — must keep open. The requested capability (excluding specific languages from detection output) is not yet implemented; the existing per-language override only prevents switching *away from* a language, not removing candidates from results.

## Change summary

Allow users to exclude specific languages (e.g., CoffeeScript) from the set of candidates that auto language detection can return.

## Best solution

Add a setting (e.g., `workbench.editor.languageDetection.excludedLanguages`) that feeds into the `supportedLangs` filter already accepted by `ILanguageDetectionService.detectLanguage`. The plumbing exists — `detectLanguage` takes an optional `supportedLangs` parameter — but no user-facing configuration exposes it. Implementation would live in `src/vs/workbench/services/languageDetection/browser/languageDetectionWorkerServiceImpl.ts` and the setting registration in `src/vs/workbench/browser/workbench.contribution.ts`.
