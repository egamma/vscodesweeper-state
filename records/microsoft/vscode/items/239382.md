[https://github.com/microsoft/vscode/issues/239382](https://github.com/microsoft/vscode/issues/239382)

## Summary

Still present on main: the CLI uses native-tls (system OpenSSL) and panics via .unwrap() when the system openssl.cnf loads providers incompatible with the linked version. No fix or mitigation has been applied.

## Change summary

The VS Code CLI (Rust binary) panics on startup when the system's openssl.cnf configures providers that are incompatible with the linked OpenSSL version.

## Best solution

Either switch reqwest to the `rustls-tls` feature (avoiding system OpenSSL entirely), or replace the `.unwrap()` on `ClientBuilder::build()` in main.rs with graceful error handling that sets OPENSSL_CONF=/dev/null as a fallback, or at minimum produces a user-readable error instead of a panic.
