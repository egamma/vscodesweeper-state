# Issue #11688 — Feature request: 'Ignore comments' checkbox in the 'Find' dialog

[https://github.com/microsoft/vscode/issues/11688](https://github.com/microsoft/vscode/issues/11688)

## Summary

Well-established feature request with 194 upvotes — far above the 20-upvote Backlog threshold — and no implementation on current main. Multiple duplicates have been closed against this canonical issue. Keeping open as the community-endorsed canonical tracker.

## Change summary

Add an option to exclude comments (and optionally strings) from Find/Search results, so users can search only in actual code.

## Best solution

Implement a comment/string scope filter for both the editor Find widget and the Search viewlet. The technical path noted by the team (comment #9) is to leverage semantic tokenization or TextMate scopes to identify comment/string ranges, then filter matches against them. This requires a language-aware tokenization layer fast enough for interactive search. The feature belongs in core (editor-find + search), not an extension, because it needs deep integration with the find/replace pipeline.
