# Issue #321498 — Testing: Linux SSH session file paste via keybinding causes false error notification

[https://github.com/microsoft/vscode/issues/321498](https://github.com/microsoft/vscode/issues/321498)

## Summary

Team-authored bug report — keeping open. The paste handler's catch block (line 1280 of fileActions.ts) treats all errors, including EEXIST from mkdir on an already-existing directory, as 'files deleted or moved,' producing a false error notification even though the paste succeeded. Still present on main.

## Change summary

Pasting files via keybinding in a Linux SSH session between two remote folders shows a false 'file(s) deleted or moved' error when a target directory already exists (EEXIST).

## Best solution

In `src/vs/workbench/contrib/files/browser/fileActions.ts` around line 1280, the catch block should either ignore EEXIST errors (since the directory already existing is not a failure) or the upstream `mkdir` call should use `{ recursive: true }` to avoid throwing on existing directories. The generic error message should also be refined to distinguish transient filesystem errors from genuine 'files deleted' scenarios.
