# Issue #255946 — Output cant be cleared

[https://github.com/microsoft/vscode/issues/255946](https://github.com/microsoft/vscode/issues/255946)

## Summary

Verified duplicate of #224516 — both report that LogOutputChannel.clear() fails to clear content because the offset tracking in ExtHostOutputChannel.log() only counts raw message bytes while the underlying logger adds timestamp/level formatting, causing a mismatch.

## Change summary

Calling clear() on a LogOutputChannel (created with { log: true }) does not fully clear the output because the byte offset is underestimated.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/224516 which is the canonical tracking issue for the same root cause (offset mismatch in log output channels). The fix belongs in src/vs/workbench/api/common/extHostOutput.ts — the offset calculation in ExtHostOutputChannel.log() needs to account for the formatting bytes the logger adds.
