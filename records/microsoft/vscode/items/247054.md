# Issue #247054 — broken window border with wayland

[https://github.com/microsoft/vscode/issues/247054](https://github.com/microsoft/vscode/issues/247054)

## Summary

Still a live upstream issue: VS Code's custom titlebar on Wayland lacks compositor-drawn window borders/shadows. Multiple users confirmed the problem persists through Electron 35→42. Already correctly labeled `upstream`; keeping open to track resolution.

## Change summary

When using custom titlebar on Wayland (GNOME and KDE), VS Code windows have no visible border or shadow, making it hard to distinguish adjacent windows.

## Best solution

Track the upstream Electron fix (https://github.com/electron/electron/issues/44540 covers CSD borders/shadows on Wayland). Once Electron ships a proper fix for client-side-decorated windows requesting compositor shadows on Wayland, bump the Electron dependency. No VS Code-side code change can solve this — the compositor integration lives in Chromium/Electron's Ozone Wayland backend.
