# Issue #257287 — Git - prune branches when deleting a worktree

[https://github.com/microsoft/vscode/issues/257287](https://github.com/microsoft/vscode/issues/257287)

## Summary

Team-authored feature request for pruning branches when deleting a Git worktree — keeping open per hard keep-open rules for MEMBER-authored issues. The request is well-scoped and belongs in the built-in Git extension.

## Change summary

When deleting a Git worktree via VS Code, offer to also delete (prune) the corresponding local and/or remote branches.

## Best solution

Add a prompt or option in the Git worktree deletion flow (in `extensions/git/`) to prune the associated local/remote branch. Community comments also suggest a standalone 'Prune' command under Source Control options for `git remote prune origin`, and a reverse flow (delete worktree when deleting branch). These could be addressed incrementally.
