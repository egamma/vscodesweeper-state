# Issue #247137 — UX: Custom model picker doesn't behave like native

[https://github.com/microsoft/vscode/issues/247137](https://github.com/microsoft/vscode/issues/247137)

## Summary

Several of the six reported UX issues have been partially addressed (mouse-down handler added at line 1272, 'Manage Models' is now a regular action item), but the deeper behavioral gaps — single-gesture drag-select, active button styling while open, and hover deselection on mouse-leave — stem from the ActionWidget infrastructure and likely remain on current main. Keeping open as legitimate polish work.

## Change summary

The custom model picker dropdown does not match native menu behavior in several ways: mouse-down open, drag-to-select, active button styling, hover deselection, and tab/focus handling.

## Best solution

Address the remaining native-menu-parity gaps in the ActionWidget infrastructure (`src/vs/platform/actionWidget/browser/`) and/or the model picker's show logic in `src/vs/workbench/contrib/chat/browser/widget/input/chatModelPicker.ts` — specifically: (1) support single-gesture drag-select (mouse-down opens, drag highlights, mouse-up selects), (2) keep the trigger button styled as active/pressed while the dropdown is open, (3) clear item highlight when the mouse leaves the menu area. The mouse-down open and 'Manage Models as action item' items appear already addressed.
