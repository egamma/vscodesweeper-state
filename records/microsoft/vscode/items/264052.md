# Issue #264052 — Pressing 'return' with file explorer context menu open starts renaming file

[https://github.com/microsoft/vscode/issues/264052](https://github.com/microsoft/vscode/issues/264052)

## Summary

Still open on main. The bug is real — on macOS, pressing Enter to accept a context menu item causes the keyup event to fire on the re-focused explorer item, triggering rename. An open PR (https://github.com/microsoft/vscode/pull/288498) exists but has not been merged, so the issue should stay open.

## Change summary

On macOS, pressing Enter on a file explorer context menu item triggers the folder rename action instead of the selected menu command.

## Best solution

Review and land (or replace) the open PR https://github.com/microsoft/vscode/pull/288498, which suppresses spurious keyup events after context menu action triggers in `contextMenuHandler.ts`. The fix approach of capturing keyup at the window level is reasonable but needs human review for edge cases.
