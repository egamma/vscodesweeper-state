# Issue #308852 — Agents App: Changes are still in the panel even after running merge

[https://github.com/microsoft/vscode/issues/308852](https://github.com/microsoft/vscode/issues/308852)

## Summary

Team-authored issue with confirmed reproduction and active design discussion about improving the post-merge UX in the Agents window — must stay open for maintainer resolution.

## Change summary

After merging changes in the Agents window (worktree or folder flow), the Branch Changes view still shows the changes until a push to the remote is performed, confusing users into thinking the merge didn't work.

## Best solution

Either default the merge action to 'Merge & Sync' (push to remote), or update the Changes view to compare against the local branch post-merge, or surface a clear indicator that a push is still needed. The team is actively discussing the right UX in the comments.
