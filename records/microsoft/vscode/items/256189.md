# Issue #256189 — Impove Source Control to fast review ai generated code: option to set hotkey for stage/unstage block

[https://github.com/microsoft/vscode/issues/256189](https://github.com/microsoft/vscode/issues/256189)

## Summary

Valid feature request: the `git.diff.stageHunk` command ("Stage Block") has no default keybinding, making keyboard-only diff review workflows impossible for that action. Navigation between changed files already exists via multiDiffEditor commands (Alt+F5). The request is reasonable but has zero community upvotes after a year.

## Change summary

Add default keybindings for the Stage Block / Revert Block gutter actions in the diff editor, and improve keyboard-driven navigation between changed files in the SCM review flow.

## Best solution

Assign default keybindings to `git.diff.stageHunk` and the corresponding revert/unstage hunk commands so keyboard-only review is possible from the diff editor gutter. The `multiDiffEditor.goToNextChange` (Alt+F5) already handles cross-file navigation, so the remaining gap is hunk-level stage/unstage via keyboard. Changes would live in `extensions/git/package.json` (keybindings section).
