# Issue #317485 — agents: create pull request fails on worktree session

[https://github.com/microsoft/vscode/issues/317485](https://github.com/microsoft/vscode/issues/317485)

## Summary

Valid bug: the in-memory worktree properties cache in ChatSessionWorktreeService can return undefined for a valid worktree session, causing the 'create pull request' action to fail with a misleading error. Restarting the agents window re-populates the cache, confirming stale state. No fix found on current main.

## Change summary

Creating a pull request from the agents window intermittently fails with 'Creating a pull request is only supported for worktree-based sessions' even though the session does use a worktree.

## Best solution

Fix the race in `ChatSessionWorktreeService.getWorktreeProperties()` (extensions/copilot/src/extension/chatSessions/vscode-node/chatSessionWorktreeServiceImpl.ts). The `_sessionWorktrees` in-memory map can miss entries when the session was created but the worktree properties were not yet cached — the fallback to the metadata store file may also fail if persistence was delayed. The fix should ensure the worktree properties are reliably written to the in-memory map during session initialization, or retry the metadata-store lookup before showing the error.
