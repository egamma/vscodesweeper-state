# Issue #169011 — Don't initialize with "Manual Resolution"

[https://github.com/microsoft/vscode/issues/169011](https://github.com/microsoft/vscode/issues/169011)

## Summary

Team-authored issue; the bug is still present on main — `computeState()` falls through to `ModifiedBaseRangeState.unrecognized` whenever the initial result text doesn't match one of the six recognized combinations, incorrectly labeling it 'Manual Resolution' at init time.

## Change summary

The merge editor marks conflicts as 'Manual Resolution' on first open even when no manual editing has occurred, because it cannot distinguish an initial unrecognized state from a user-modified one.

## Best solution

Distinguish initial state from user-edited state in `computeState` (or the caller in `mergeEditorModel.ts`), e.g. by introducing a separate `initialUnrecognized` kind or suppressing the 'handled' flag until the user actually edits the result for that range. The fix lives in `src/vs/workbench/contrib/mergeEditor/browser/model/mergeEditorModel.ts` around the `computeState`/`modifiedBaseRangeResultStates` logic.
