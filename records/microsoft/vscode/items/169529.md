# Issue #169529 — Parts that are hidden should not render DOM nodes

[https://github.com/microsoft/vscode/issues/169529](https://github.com/microsoft/vscode/issues/169529)

## Summary

Issue has milestone 'On Deck' (a scheduling milestone), so it must stay open per triage rules. Partial progress exists — a NativeTitlebarPart class was introduced and commit 8b511732138 removed JS from the titlebar — but the broader request (all hidden parts should not render DOM) appears unfinished on current main.

## Change summary

Workbench parts that are hidden or disabled still render DOM nodes, causing unnecessary style recalculations and potential perf impact.

## Best solution

Complete the work so that each hideable part (titlebar in native mode, activity bar, status bar, sidebar, auxiliary bar, panel) removes its DOM subtree when hidden rather than just toggling CSS visibility. The titlebar case was partially addressed in 8b511732138; the remaining parts need the same treatment in `src/vs/workbench/browser/layout.ts` and the individual part implementations.
