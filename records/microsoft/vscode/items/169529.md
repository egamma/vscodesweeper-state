# Issue #169529 — Parts that are hidden should not render DOM nodes

[https://github.com/microsoft/vscode/issues/169529](https://github.com/microsoft/vscode/issues/169529)

## Summary

Still open on main: hidden parts (especially the titlebar in native mode) continue to render DOM nodes. The issue carries the 'On Deck' milestone, which is a scheduling decision — keeping open.

## Change summary

Workbench parts that are hidden or disabled still add DOM nodes to the document, causing unnecessary style recalculations (especially env(titlebar-area-*) for window-controls-container).

## Best solution

Implement lazy DOM creation for parts that are disabled or hidden by default (especially the titlebar part when native title is enabled). The proposed NativeTitlebarPart (a cheap serializable view providing ITitleService without DOM) from comment 6 was never built. The fix lives in `src/vs/workbench/browser/parts/titlebar/titlebarPart.ts` and possibly `src/vs/workbench/browser/layout.ts`.
