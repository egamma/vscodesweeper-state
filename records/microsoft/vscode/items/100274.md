# Issue #100274 — Git - Support `subst` drives

[https://github.com/microsoft/vscode/issues/100274](https://github.com/microsoft/vscode/issues/100274)

## Summary

Still reproduces on current VS Code (1.101.0, June 2025 report). Partial fixes exist for UNC/symlink paths (commit 1d4130b8b39, relative-path improvements in 04260cb17b5/82839a3b65c), but the subst-drive scenario still shows path mismatches in explorer decorations, source control paths, and open-file navigation. The maintainer is actively engaged and asked for details. Keeping open as the canonical issue.

## Change summary

Git file decorations, source control paths, and editor navigation break when a workspace is opened via a Windows `subst` drive because `git rev-parse --show-toplevel` resolves through the substitution.

## Best solution

Extend the `getRepositoryRoot()` fallback in `extensions/git/src/git.ts` (and the `realpath` comparison logic in `extensions/git/src/model.ts`) to detect when a subst drive is in use and normalize the repository root back to the subst path, similar to the existing UNC-path normalization block. The upstream git-for-windows issue (git-for-windows/git#2705) may eventually fix `--show-toplevel` itself, but VS Code should handle the mismatch defensively.
