# Issue #124584 — Terminal.creationOptions API shows default options in remotes

[https://github.com/microsoft/vscode/issues/124584](https://github.com/microsoft/vscode/issues/124584)

## Summary

Still present on main: UI-created terminals in remote environments expose resolved defaults in creationOptions rather than an empty/minimal object, because $acceptTerminalOpened builds creationOptions from the full shellLaunchConfigDto without filtering out defaults.

## Change summary

Terminal.creationOptions exposes resolved default options (shell path, args, etc.) for UI-created terminals in remote windows, instead of reflecting only user-specified options.

## Best solution

In the ext host terminal service, distinguish between user-specified creation options and resolved defaults when constructing creationOptions for UI-created terminals — likely by passing a flag or separate DTO field from the main thread indicating which options were explicitly set vs. resolved as defaults.
