# Issue #326670 — "getWslProfiles" blocks main thread via synchronous "CreateProcessW"

[https://github.com/microsoft/vscode/issues/326670](https://github.com/microsoft/vscode/issues/326670)

## Summary

Valid performance bug: PtyHostService.getProfiles() calls detectAvailableProfiles (and thus cp.exec('wsl.exe -l -q')) directly in the main Electron process rather than via the pty host proxy, so the synchronous CreateProcessW in libuv's uv_spawn can stall the main event loop. The code is unchanged on current main.

## Change summary

WSL terminal profile detection via cp.exec('wsl.exe -l -q') blocks the main Electron process event loop during CreateProcessW, causing UI freezes on Windows with WSL installed.

## Best solution

Move the getProfiles() call in PtyHostService to go through the pty host proxy (this._proxy.getProfiles()) instead of calling detectAvailableProfiles directly in the main process, or spawn wsl.exe on a worker thread. The relevant code is in src/vs/platform/terminal/node/ptyHostService.ts lines 303-306 and src/vs/platform/terminal/node/terminalProfiles.ts line 347-358.
