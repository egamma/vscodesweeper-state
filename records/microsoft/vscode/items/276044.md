# Issue #276044 — Tasks misses basic editor support

[https://github.com/microsoft/vscode/issues/276044](https://github.com/microsoft/vscode/issues/276044)

## Summary

Valid feature request: the tasks.json problem matcher schema doesn't declare required fields (like `beginsPattern`/`endsPattern` inside `background`), so users get no inline editor validation — only a cryptic error on save. The gap remains on current main.

## Change summary

Tasks.json editing should show inline JSON schema validation errors for missing required properties (like launch.json does), rather than only showing a generic error on save.

## Best solution

Add `required: ['beginsPattern', 'endsPattern']` to the `background` object schema in `src/vs/workbench/contrib/tasks/common/problemMatcher.ts` (and review other task schema objects for similar missing `required` arrays). This would give parity with launch.json's inline validation experience.
