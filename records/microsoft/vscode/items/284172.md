# Issue #284172 — Attached files do not update on file events

[https://github.com/microsoft/vscode/issues/284172](https://github.com/microsoft/vscode/issues/284172)

## Summary

Still present on main. The file attachment watcher in chatAttachmentModel.ts only handles FileChangeType.DELETED — it does not update the attachment's URI or name on rename. Team-authored issue, keeping open.

## Change summary

File attachments in the chat input do not update their name/URI when the attached file is renamed in the Explorer.

## Best solution

Extend _watchAttachment in src/vs/workbench/contrib/chat/browser/attachments/chatAttachmentModel.ts to detect renames (delete + create pair, or listen to IWorkingCopyFileService.onDidRunWorkingCopyFileOperation for rename events) and update the attachment entry's URI and name accordingly, rather than removing it.
