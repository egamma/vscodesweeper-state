# Issue #168676 — Extension commands do not show any status indicator while waiting for activation

[https://github.com/microsoft/vscode/issues/168676](https://github.com/microsoft/vscode/issues/168676)

Keep open: this is a MEMBER-authored Backlog feature request, and the discussion identifies a real UX gap before the existing extension activation status item appears. The issue already has the right extension-area labels and needs maintainer product/implementation judgment rather than cleanup closure.

## Change summary

Show immediate feedback when a command invocation is waiting for extension host startup or extension activation before the command can run.

## Best solution

Decide whether command-triggered activation should surface a command-specific progress/status indication in the command service or whether extension host startup should expose an earlier renderer status item, preserving timeouts so stuck activations do not leave persistent UI.
