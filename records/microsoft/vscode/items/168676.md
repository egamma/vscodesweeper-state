# Issue #168676 — Extension commands do not show any status indicator while waiting for activation

[https://github.com/microsoft/vscode/issues/168676](https://github.com/microsoft/vscode/issues/168676)

## Summary

Team-authored feature request that remains valid: the existing 'Activating Extensions...' status bar message only fires after the extension host is ready and `activateByEvent` is called, but there's no indicator during the extension host startup gap itself. The discussion concluded with agreement to add command-service-level progress, which hasn't been implemented.

## Change summary

Show a status indicator immediately when an extension command is invoked, covering the extension host startup delay before activation begins.

## Best solution

Add progress reporting in the command service (or a similar central place) that covers the full span from command invocation through extension host startup to activation completion — as discussed by jrieken and alexdima in the thread. The current `ExtensionActivationProgress` in `src/vs/workbench/contrib/extensions/browser/extensionsActivationProgress.ts` only covers the activation phase after the host is ready.
