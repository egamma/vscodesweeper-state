# Issue #229571 — [tunnels] Remote Tunnels stays active after logging out

[https://github.com/microsoft/vscode/issues/229571](https://github.com/microsoft/vscode/issues/229571)

## Summary

Team-authored polish issue: Remote Tunnels should deactivate when the user logs out, but the extension doesn't listen to authentication change events. Keeping open per team-member rule.

## Change summary

Remote Tunnels remains active after the user logs out of their account, requiring manual deactivation.

## Best solution

The remote tunnels extension should listen to the `onDidChangeSessions` authentication event and automatically stop the tunnel when the user's session ends. This likely lives in the remote tunnels extension code that manages tunnel lifecycle.
