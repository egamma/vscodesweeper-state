# Issue #193832 — Collapsed diff breadcrumbs don't show in timeline diffs

[https://github.com/microsoft/vscode/issues/193832](https://github.com/microsoft/vscode/issues/193832)

## Summary

Not a duplicate of #193831 — that issue is about breadcrumbs overflowing and not being scrollable, while this one is about breadcrumbs not appearing at all in timeline diffs. The root cause (per hediet's comment) is that language services don't report document symbols for the file schemes used in timeline diffs. Still a valid, unresolved gap.

## Change summary

Collapsed unchanged-region breadcrumbs don't appear when viewing diffs from the Timeline view, because document symbol providers aren't active for timeline file schemes.

## Best solution

Ensure the diff editor breadcrumbs source (in `src/vs/editor/contrib/diffEditorBreadcrumbs/browser/contribution.ts`) can resolve symbols for timeline diff documents — likely by making the outline model service work with the `git`/timeline URI schemes, or by mapping those models to their on-disk counterparts for symbol resolution. The area owner (diff-editor + breadcrumbs) should confirm whether this is feasible without performance impact.
