# Issue #312940 — Custom OpenAI-compatible BYOK providers proxying Anthropic models cannot use prompt caching (no way to declare backend type)

[https://github.com/microsoft/vscode/issues/312940](https://github.com/microsoft/vscode/issues/312940)

## Summary

Valid feature request: custom OpenAI-compatible BYOK endpoints that proxy Anthropic models have no way to declare their backend type, so prompt caching is never activated. The related OpenRouter-specific issue (#312939) was fixed by PR #312435, but that fix is scoped to the built-in OpenRouter provider — the general custom-endpoint case requested here remains unaddressed on main.

## Change summary

Add a `backendType` (or equivalent) field to the custom OpenAI-compatible model config so that Anthropic-style prompt caching can be applied for proxied Claude models.

## Best solution

Extend `_CustomOAIModelConfig` in `extensions/copilot/src/extension/byok/vscode-node/customOAIProvider.ts` with an optional `backendType` field (e.g. `'anthropic' | 'openai'`), and wire it into the endpoint layer so that when set to `'anthropic'`, the request builder emits `cache_control` fields analogous to the native Anthropic and OpenRouter providers. This keeps the fix localized to the BYOK custom-OAI path.
