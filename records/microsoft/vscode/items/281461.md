# Issue #281461 — shellIntegration.ps1 caused Python Debugger with Garbled Text

[https://github.com/microsoft/vscode/issues/281461](https://github.com/microsoft/vscode/issues/281461)

## Summary

Still reproduces on main: the OSC 633;E sequence sends escaped command text on Windows 10, which can leak into terminal output as garbled text. An open draft PR (#284298) exists to fix this but has not been merged.

## Change summary

On Windows 10, long PowerShell commands in the integrated terminal intermittently display garbled escaped text (e.g., `\x5c` for backslashes) due to the OSC 633;E shell integration sequence.

## Best solution

Review and land the open draft PR https://github.com/microsoft/vscode/pull/284298, which disables the OSC 633;E sequence on Windows 10 to prevent the escaped command text from leaking into terminal output. The nonce is already gated on non-Windows-10 (line 225 of shellIntegration.ps1), so extending that gate to the entire E sequence is a natural fix.
