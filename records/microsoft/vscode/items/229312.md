# Issue #229312 — Do not present ghost text when code is deleted or undone

[https://github.com/microsoft/vscode/issues/229312](https://github.com/microsoft/vscode/issues/229312)

## Summary

Still open and valid: the inline completions model intentionally re-surfaces ghost text on undo (line ~436 of inlineCompletionsModel.ts), and team members are actively discussing heuristics to suppress it in unwanted cases. Issue #255141 was closed as a duplicate of this one, confirming this is the canonical tracker.

## Change summary

Ghost text / inline completions reappear after deleting text or undoing, creating non-deterministic editing behavior.

## Best solution

Implement an intent-based suppression heuristic in `src/vs/editor/contrib/inlineCompletions/browser/model/inlineCompletionsModel.ts` — suppress re-showing completions after undo/delete unless the user explicitly re-engaged (e.g. paused or typed forward). The team discussion in comments 4–5 outlines the shape: suppress explicitly-rejected suggestions while allowing previously-ignored ones to resurface.
