# Issue #228750 — URI not properly serialized when sent to webworker

[https://github.com/microsoft/vscode/issues/228750](https://github.com/microsoft/vscode/issues/228750)

## Summary

Team-authored technical debt issue that remains unfixed on main. The workaround (manual URI.revive calls) is in place, but the underlying structured-clone serialization gap for URIs sent to web workers has not been addressed architecturally.

## Change summary

URIs sent to web workers via structured clone lose their `$mid` field, preventing `revive()` from properly deserializing them without manual workarounds.

## Best solution

Design an integration point for the structured clone algorithm used by web workers so that URI objects include `$mid` on arrival, eliminating the need for per-callsite manual `URI.revive()` workarounds. The fix likely belongs in the worker protocol layer (`src/vs/base/common/worker/`) or the marshalling utilities.
