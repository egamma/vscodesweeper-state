# Issue #228750 — URI not properly serialized when sent to webworker

[https://github.com/microsoft/vscode/issues/228750](https://github.com/microsoft/vscode/issues/228750)

## Summary

Keep open for maintainer judgment: this is a team-authored issue by a VS Code member and is already tracked in Backlog with debt/URI engineering labels. The report describes a real serialization gap in the web worker path rather than a cleanup candidate with a safe close reason.

## Change summary

URI values in web-worker search requests arrive without `$mid`, so `revive(folderQuery)` cannot deserialize `folderQuery.folder` as a URI.

## Best solution

Investigate the URI marshaling boundary for web workers, especially the local file search worker path around `localFileSearch.ts` and structured-clone transport, then either add durable URI revival support there or document why that boundary intentionally cannot preserve URI metadata.
