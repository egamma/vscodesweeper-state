# Issue #316713 — Shell integration cannot be enabled for executable "/app/bin/host-spawn" and args undefined

[https://github.com/microsoft/vscode/issues/316713](https://github.com/microsoft/vscode/issues/316713)

## Summary

Valid request — shell integration cannot detect the actual shell when the terminal profile uses a wrapper executable like host-spawn or flatpak-spawn. The reporter has an open PR (#316714) addressing this. Keeping open pending PR review.

## Change summary

Shell integration fails for terminal profiles using host-spawn or flatpak-spawn wrappers because the shell type is derived from the executable basename, which doesn't match any known shell.

## Best solution

Review and land (or iterate on) the author's open PR https://github.com/microsoft/vscode/pull/316714, which adds optional `shellType` to terminal profiles and auto-detects wrapper executables. The fix belongs in `src/vs/platform/terminal/node/terminalEnvironment.ts` around the shell-detection logic at line 84.
