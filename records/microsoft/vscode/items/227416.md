# Issue #227416 — Markdown language detection is overly aggressive and cannot be disabled

[https://github.com/microsoft/vscode/issues/227416](https://github.com/microsoft/vscode/issues/227416)

## Summary

Valid feature request: the existing `workbench.editor.languageDetection` setting is language-overridable to prevent switching *away from* a language, but there is no mechanism to exclude a language from being a *detection target*. The request for per-language exclusion from auto-detection results remains unimplemented on current main. Low traction (0 upvotes, no comments) but properly categorized.

## Change summary

Automatic language detection too aggressively identifies untitled/new files as Markdown, and there is no setting to exclude specific languages from being detection targets.

## Best solution

Add a mechanism (e.g. a list of excluded target languages) to the language detection service so users can prevent specific languages like Markdown from being auto-assigned. The logic lives in `src/vs/workbench/services/languageDetection/browser/languageDetectionWorkerServiceImpl.ts` and the contribution in `src/vs/workbench/contrib/languageDetection/browser/languageDetection.contribution.ts`. Alternatively, improve the ML model's Markdown confidence threshold.
