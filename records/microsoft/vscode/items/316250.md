# Issue #316250 — Copilot CLI agent panel: git worktree creation path defaults to inside workspace folder, causing silent duplicate loading in plugin-scan runtimes

[https://github.com/microsoft/vscode/issues/316250](https://github.com/microsoft/vscode/issues/316250)

## Summary

The issue's core premise — that the worktree lands inside the workspace folder — is incorrect on current main: `getCopilotWorktreesRoot` places worktrees one level above the repo root (`../<repo>.worktrees/`). The underlying request for configurability of the worktree base path is already tracked by #293884.

## Change summary

Request to place agent-created git worktrees outside the workspace folder (or make the path configurable) to avoid host-runtime duplicate-loading.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/293884, which tracks making the worktree directory configurable. The specific scenario described (worktree inside workspace) does not reproduce on current main — the default path is already a sibling of the repo root.
