# Issue #296007 — Git - on displaying .patch files, for --word-diff format text is NOT colored as for typical format

[https://github.com/microsoft/vscode/issues/296007](https://github.com/microsoft/vscode/issues/296007)

## Summary

The built-in diff grammar (extensions/diff/syntaxes/diff.tmLanguage.json) only handles unified/context/normal diff formats — it has no patterns for git's --word-diff markers ([-OLD-]{+NEW+}). This is a valid feature request to extend the grammar.

## Change summary

Add syntax highlighting for git --word-diff format ([-OLD-]{+NEW+} markers) in .patch files.

## Best solution

Extend the diff TextMate grammar at extensions/diff/syntaxes/diff.tmLanguage.json with patterns matching \[-...-\] (markup.deleted) and \{\+...\+\} (markup.inserted) inline tokens. Alternatively, this could ship as a Marketplace extension since it's a niche format.
