# Issue #229970 — SCM - Going up past the start of a commit message goes into history and sometimes loses the commit message you were typing

[https://github.com/microsoft/vscode/issues/229970](https://github.com/microsoft/vscode/issues/229970)

## Summary

Still an open design question on current main — the up-arrow history navigation in SCM input is unchanged, and a team member engaged but no resolution was reached. The related issue #222005 was closed as a duplicate of this one, making this the canonical tracker. Worth keeping open for a product decision.

## Change summary

Pressing up-arrow past the top of a multiline SCM commit message triggers history navigation, which can disorienting and sometimes loses the in-progress message.

## Best solution

Decide on a UX change in `src/vs/workbench/contrib/scm/browser/scm.contribution.ts` and `src/vs/workbench/contrib/scm/common/scmService.ts` — options include: (1) add a setting to disable up/down-arrow history navigation, (2) change the default keybinding to Alt+Up/Down (the force variants already exist), or (3) improve the save/restore logic so the in-progress message is never lost. A commenter's workaround of unbinding `scm.viewPreviousCommit` via keybindings.json confirms the feature is separable.
