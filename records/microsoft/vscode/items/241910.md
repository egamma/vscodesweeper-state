# Issue #241910 — Copy paste of image from safari inserts attachment and url text

[https://github.com/microsoft/vscode/issues/241910](https://github.com/microsoft/vscode/issues/241910)

## Summary

Still reproduces on main: the PasteImageProvider only declares handledMimeType for the image MIME type, so on Safari (whose clipboard includes both image data and a text/plain URL), the default text paste provider also fires, inserting the URL alongside the image attachment.

## Change summary

Pasting an image from Safari into chat input inserts both the image attachment and the URL as text, whereas only the attachment should be inserted.

## Best solution

In `PasteImageProvider.provideDocumentPasteEdits`, when an image is successfully handled, also declare the text/plain MIME type as handled (or add `text/plain` to the handled types) so the default text paste provider is suppressed. Alternatively, add `text/html` handling as well, since Safari may also include HTML in the clipboard for copied images.
