# Issue #177925 — Case-sensitive files/folders on Windows are not shown correctly in vscode

[https://github.com/microsoft/vscode/issues/177925](https://github.com/microsoft/vscode/issues/177925)

## Summary

Still reproduces on main: DiskFileSystemProvider only sets PathCaseSensitive on Linux (line 61–63 of diskFileSystemProvider.ts), so Windows directories marked case-sensitive via fsutil are treated as case-insensitive, causing duplicate-cased entries to collapse in the explorer. A maintainer explicitly reopened this to track the Windows-specific flag separately from the macOS issue.

## Change summary

Files and folders that differ only by case in a Windows directory marked case-sensitive via `setCaseSensitiveInfo` are collapsed to a single entry in the explorer.

## Best solution

The DiskFileSystemProvider needs per-directory (or per-workspace-root) case-sensitivity detection on Windows and macOS, rather than the current global `isLinux` check. This is architecturally related to https://github.com/microsoft/vscode/issues/94307 (macOS) but the Windows mechanism (`fsutil setCaseSensitiveInfo`) differs. Both issues should stay open until the file system provider can query per-path case sensitivity from the OS.

- **URI comparison:** src/vs/base/common/resources.ts line 343: `extUriBiasedIgnorePathCase` returns `!isLinux` for file-scheme URIs, meaning all non-Linux file paths are compared case-insensitively.
