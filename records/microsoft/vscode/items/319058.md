# Issue #319058 — Shell integration for fish breaks command status code return

[https://github.com/microsoft/vscode/issues/319058](https://github.com/microsoft/vscode/issues/319058)

## Summary

Valid bug confirmed from source: when fish_mode_prompt is empty/disabled (e.g. starship), the else branch in __init_vscode_shell_integration calls __vsc_fish_prompt_start before __vsc_fish_prompt, and the escape-sequence printf inside it clobbers $status. An open PR (#319065) exists to fix it.

## Change summary

Fish shell integration's escape-sequence emission in __vsc_fish_prompt_start overwrites $status before the user's prompt function can read it, breaking status-code indicators when fish_mode_prompt is disabled.

## Best solution

Land https://github.com/microsoft/vscode/pull/319065 (or an equivalent fix) that saves and restores $status across the escape-sequence emission in __vsc_fish_prompt_start, so the user's prompt sees the real last-command exit code.
