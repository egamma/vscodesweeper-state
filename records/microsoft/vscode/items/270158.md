# Issue #270158 — [Bug] IME fails in Integrated Terminal when "Screen Reader Optimized" mode is active

[https://github.com/microsoft/vscode/issues/270158](https://github.com/microsoft/vscode/issues/270158)

## Summary

Still reproduces on main: xterm.js's screenReaderMode intentionally skips InputEvent processing for 'insertText' events, which drops IME-committed text. The bug is at the xterm.js level but needs a VS Code-side or upstream fix to reconcile screen reader mode with IME composition.

## Change summary

IME input is silently dropped in the integrated terminal when Screen Reader Optimized mode is active, because xterm.js's screenReaderMode disables the insertText InputEvent path.

## Best solution

Fix in xterm.js's CoreBrowserTerminal._inputEvent to allow IME composition commits (compositionend-triggered insertText events) even when screenReaderMode is active, or add a VS Code-side workaround that intercepts composition events before they reach xterm's filtered path. The fix likely belongs upstream in xtermjs/xterm.js.
