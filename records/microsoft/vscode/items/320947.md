# Issue #320947 — Copilot Chat add extra characters to terminal commands

[https://github.com/microsoft/vscode/issues/320947](https://github.com/microsoft/vscode/issues/320947)

## Summary

Real bug: the basic and none execute strategies send Ctrl+U (\x15) to clear pending input before executing agent commands, but PowerShell sometimes renders it as a literal `^U` prefix on the next command, causing failures. The maintainer (meganrogge) is actively investigating. No fix on main yet.

## Change summary

Agent-sent terminal commands are intermittently prefixed with `^U` (Ctrl+U / U+0015), causing the command to fail in PowerShell.

## Best solution

The Ctrl+U line-kill sent in `basicExecuteStrategy.ts` (line 207) and `noneExecuteStrategy.ts` (line 84) needs to be made safe for PowerShell — either by waiting for the shell to consume it before sending the command, gating it on shell type, or using a PowerShell-specific clear mechanism. The reporter confirmed the issue persists even after upgrading to rich shell integration, so the rich strategy path should also be checked.
