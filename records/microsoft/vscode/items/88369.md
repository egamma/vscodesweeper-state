## Summary

Still reproduces on main: `revertSelectedRanges` filters whole hunks by intersection rather than using `intersectDiffWithRange` for line-level granularity like `stageSelectedRanges` does — the inconsistency remains unfixed.

## Change summary

Git 'Revert Selected Ranges' reverts entire hunks instead of only the selected lines, inconsistent with 'Stage Selected Ranges' which correctly stages only selected lines.
