## Summary

Still valid: user tasks in multi-root workspaces always execute in the first workspace folder (`_getAFolder()` returns `workspaceFolders[0]`). No folder-picking prompt exists for user tasks. The feature has 30 upvotes and is in the On Deck milestone.

## Change summary

User tasks in multi-root workspaces should prompt for which workspace folder to execute in, rather than always using the first folder.

## Best solution

When a user task is run in a multi-root workspace, present a folder picker (similar to how workspace tasks resolve ambiguity). The fix would live in the task execution path in `src/vs/workbench/contrib/tasks/browser/abstractTaskService.ts`, likely around the `_getAFolder()` usage at line 2601/2609 or the task-run resolution logic.
