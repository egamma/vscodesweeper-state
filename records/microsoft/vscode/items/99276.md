# Issue #99276 — Tasks: Pick folder for user tasks in multi-root-workspaces

[https://github.com/microsoft/vscode/issues/99276](https://github.com/microsoft/vscode/issues/99276)

## Summary

Still valid on current main — user tasks in multi-root workspaces still fall back to the first workspace folder (hardcoded `folders[0]`) rather than prompting the user to pick. The 'On Deck' milestone is a scheduling decision; keeping open.

## Change summary

User tasks in multi-root workspaces should prompt the user to select which workspace folder to execute in, rather than always using the first folder.

## Best solution

Add a folder-picker prompt in `_getAFolder()` or the execution path in `terminalTaskSystem.ts` (line ~815) when a user task has no explicit workspace folder and multiple folders exist, similar to how other multi-root features prompt for a folder. The fallback to `workspaceFolders[0]` should become a picker when triggered from user tasks in a multi-root workspace.
