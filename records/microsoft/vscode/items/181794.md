# Issue #181794 — Vscode Sets breakpoint in both sessions in multicore debugging with single Debug Configuration 

[https://github.com/microsoft/vscode/issues/181794](https://github.com/microsoft/vscode/issues/181794)

## Summary

Valid feature request acknowledged by two team members (roblourens, connor4312). The underlying behavior — breakpoints sent to all sessions via `sendToOneOrAllSessions` — is unchanged on current main. The request for session-scoped breakpoints remains unimplemented and relevant for multicore/disassembly debugging.

## Change summary

In multicore debugging, breakpoints are always sent to all debug sessions; request to scope breakpoints to the user-selected session only.

## Best solution

Introduce session- or config-scoped breakpoints in the debug model so that `sendBreakpoints` in `src/vs/workbench/contrib/debug/browser/debugService.ts` can filter by target session. This is a bigger architectural change (per connor4312's comment) and may also require DAP-level coordination (see the linked DAP issue https://github.com/microsoft/debug-adapter-protocol/issues/445). Keep open for future planning.
