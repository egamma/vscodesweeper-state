# Issue #228742 — 'search' option in fileLocation of problemMatcher for Task does not work in Remote Development

[https://github.com/microsoft/vscode/issues/228742](https://github.com/microsoft/vscode/issues/228742)

## Summary

Still reproduces on current main: the `search` fileLocation mode in problem matchers always queries the local file system provider (`NetworkSchemas.file`) and constructs `URI.file(dir)` URIs, so in remote scenarios the directory scan hits the host machine rather than the remote.

## Change summary

The `search` fileLocation option in task problemMatchers scans the local (host) file system instead of the remote file system when using Remote SSH or WSL.

## Best solution

Fix `searchForFileLocation` in `src/vs/workbench/contrib/tasks/common/problemMatcher.ts` to use the remote-aware file service (or use the `uriProvider`/task system info to construct URIs with the correct authority) rather than hardcoding `NetworkSchemas.file` and `URI.file()`. The `uriProvider` pattern already exists for other fileLocation modes — extend it to cover the search case.
