# Issue #301437 — Always show me "Language model unavailable"

## Summary

The 'Language model unavailable' error is caused by the GitHub Copilot extension's inability to establish TLS connections through the user's proxy when connected via SSH remote — this is an extension/service-side issue, not a VS Code core bug.

## Change summary

GitHub Copilot Chat shows 'Language model unavailable' when connected to a remote SSH host with proxy configuration.

## Best solution

This belongs with the Copilot extension team (github.com/microsoft/vscode-copilot-release) — the TLS/proxy failure in the chat diagnostics points to the extension's network handling over SSH remote, not VS Code core. The user should file at the Copilot extension repo or check their proxy/TLS configuration on the remote host.
