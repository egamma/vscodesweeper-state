[https://github.com/microsoft/vscode/issues/301437](https://github.com/microsoft/vscode/issues/301437)

## Summary

This is a network/proxy configuration issue specific to the reporter's SSH remote environment — the diagnostics show TLS handshake failures through a local proxy (127.0.0.1:7890) when the Copilot extension runs on the remote host, not a bug in VS Code or Copilot Chat itself.

## Change summary

Copilot Chat shows 'Language model unavailable' when connected to an SSH remote server that routes through a local proxy causing TLS failures.

## Best solution

This is a user-specific network configuration issue — the proxy on the remote host is terminating TLS connections before they complete. The reporter needs to configure their proxy to allow TLS passthrough to Copilot API endpoints, or adjust the proxy/certificate settings on the remote machine.
