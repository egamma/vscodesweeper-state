# Issue #269854 — NES doesn't render when jumping to next problem with F8

[https://github.com/microsoft/vscode/issues/269854](https://github.com/microsoft/vscode/issues/269854)

## Summary

Team-authored feature request with active design discussion about how to render NES alongside the peek/problem view zone. Still unresolved — the team is exploring UX approaches (collapsed mode, long-distance edit widget, resizing the peek view).

## Change summary

NES suggestions should render when the user jumps to the next problem with F8, even though a peek view zone is currently visible.

## Best solution

Resolve the UX conflict between NES rendering and the peek view zone — the most promising direction from the discussion is rendering a collapsed/long-distance-edit widget while the view zone is visible, expanding to full NES when the user interacts (e.g. Tab). Implementation likely touches the NES rendering gate in the inline edit controller and the view zone visibility check.
