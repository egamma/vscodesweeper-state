# Issue #270585 — Task commands retrieved from a command won't be recognized as absolute

[https://github.com/microsoft/vscode/issues/270585](https://github.com/microsoft/vscode/issues/270585)

## Summary

Verified duplicate of #160891: same root cause — process task command variables from `${command:...}` are not resolved before the `isAbsolute` check in `_resolveAndFindExecutable`, causing the workspace folder to be prepended to the resolved absolute path.

## Change summary

Process tasks using `${command:...}` to retrieve an absolute executable path have the workspace folder incorrectly prepended because the variable isn't resolved at the path-check stage.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/160891 which is the canonical issue (assigned, Backlog milestone, `help wanted`). The fix belongs in `_resolveAndFindExecutable` in `terminalTaskSystem.ts` — it must use the already-resolved command value from the variable mapping rather than calling `resolveAsync` which cannot resolve `${command:...}` variables.

- **root cause in source:** In src/vs/workbench/contrib/tasks/browser/terminalTaskSystem.ts, `_resolveAndFindExecutable` (line ~687) calls `resolveAsync` on the raw command string. `resolveAsync` calls `evaluateSingleVariable` without a `commandValueMapping`, so `resolveFromMap` (variableResolver.ts:335-344) returns the literal `${command:...}` string unchanged. Then `path.isAbsolute` fails on the literal and `path.join(cwd, command)` prepends the workspace folder.
