# Issue #158911 — Don't allow extension to annoy users with auth asks

[https://github.com/microsoft/vscode/issues/158911](https://github.com/microsoft/vscode/issues/158911)

## Summary

The proposed fix from the Dec 2022 discussion — gating modal auth prompts on command invocation or specific activation events — has not been implemented on current main. The `createIfNone` flow in `mainThreadAuthentication.ts` still unconditionally triggers a modal dialog regardless of whether the call originates from `activate()` or a user-initiated command. The issue remains valid and actionable.

## Change summary

Extensions can repeatedly show modal authentication dialogs during activation, annoying users who have already declined, because `createIfNone` is not gated on user-initiated context.

## Best solution

Implement the plan from the Dec 2022 discussion: only show a modal auth dialog when `createIfNone` is used inside a command invocation or `onFileSystem` activation event; otherwise fall back to a less intrusive UX (notification or account badge). The change would live in `src/vs/workbench/api/browser/mainThreadAuthentication.ts` around the `createIfNone` modal flow (line ~470). Related open issue https://github.com/microsoft/vscode/issues/233424 covers a narrower Copilot-specific case but this issue owns the broader platform fix.
