# Issue #162126 — Code action widget accessibility strangeness

[https://github.com/microsoft/vscode/issues/162126](https://github.com/microsoft/vscode/issues/162126)

## Summary

Still present on main: the action list widget moves keyboard focus to the hovered item via `onListHover` → `setFocus`, so hovering steals the blue focus indicator from the keyboard-navigated item. Author is a CONTRIBUTOR (hard keep-open). The behavior is confirmed in current source.

## Change summary

Hovering over code action widget entries moves the keyboard focus indicator to the hovered item, making keyboard navigation unreliable.

## Best solution

Decouple mouse hover highlight from keyboard focus in `ActionList.onListHover` (src/vs/platform/actionWidget/browser/actionList.ts ~line 1927) — show a hover style without calling `setFocus`, similar to how QuickOpen handles it. This keeps the blue focus ring stable during keyboard navigation while still providing visual hover feedback.
