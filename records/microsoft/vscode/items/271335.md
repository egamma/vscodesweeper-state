# Issue #271335 — Chat response: changes overview is counting wrong

[https://github.com/microsoft/vscode/issues/271335](https://github.com/microsoft/vscode/issues/271335)

## Summary

Still present on main: the changes overview in chat responses counts files with 0 insertions and 0 deletions toward the total file count, inflating the reported number of changed files.

## Change summary

Chat response changes overview shows files with +0 -0 in the file count, making it appear more files changed than actually did.

## Best solution

Filter out zero-change entries (where `added === 0 && removed === 0`) before computing `diffs.length` for the file count label in `chatChangesSummaryPart.ts`, and exclude those entries from the file list rendering. The filtering should happen in the `renderHeader` autorun (around line 120) and the `renderFilesList` autorun (around line 222), or upstream in `_getDiffsForFilesAtEpochs` in `chatEditingCheckpointTimelineImpl.ts`.
