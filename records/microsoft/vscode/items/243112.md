# Issue #243112 — Code snippet editor has many editor features that are not working

[https://github.com/microsoft/vscode/issues/243112](https://github.com/microsoft/vscode/issues/243112)

## Summary

Keep open: the original noisy context menu has been partially cleaned up, but a maintainer follow-up still identifies questionable remaining entries in chat code snippet editors. This is not a clear broken-language-feature bug anymore; it is a UX polish decision about which context-menu commands should be shown for embedded chat code blocks.

## Change summary

Reduce the context menu shown in chat code snippet editors to commands that make sense and work for an in-memory embedded editor.

## Best solution

Have the chat/editor owners decide the intended minimal command set for `vscode-chat-code-block` / embedded editors, then hide inappropriate menu contributions with `resourceScheme != 'vscode-chat-code-block'` or `!isInEmbeddedEditor` where applicable.
