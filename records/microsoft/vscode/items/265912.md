# Issue #265912 — MCP servers added via McpServerDefinitionProvider should display in the MCP Servers list

[https://github.com/microsoft/vscode/issues/265912](https://github.com/microsoft/vscode/issues/265912)

## Summary

Valid feature request — the MCP Servers list view only queries config-file-based servers via mcpManagementService.getInstalled(), missing extension-contributed servers registered through mcpRegistry.registerCollection(). The gap remains on current main. Two related issues (#258549, #260778) were closed as duplicates of this one, confirming it's the canonical tracker.

## Change summary

MCP servers contributed via McpServerDefinitionProvider (extension API) should appear in the Extensions sidebar MCP Servers list, alongside config-file-based servers.

## Best solution

Bridge the McpServersListView (or its backing McpWorkbenchService.queryLocal) to also surface servers registered through IMcpRegistry collections, so extension-contributed servers appear in the installed MCP servers panel. The likely files are src/vs/workbench/contrib/mcp/browser/mcpWorkbenchService.ts (queryLocal) and src/vs/workbench/contrib/mcp/browser/mcpServersView.ts.
