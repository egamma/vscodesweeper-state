# Issue #292374 — .gitignore explorer files aren't dimmed

[https://github.com/microsoft/vscode/issues/292374](https://github.com/microsoft/vscode/issues/292374)

## Summary

Still reproduces on main: the GitIgnoreDecorationProvider in the built-in git extension only fires decoration refreshes on .gitignore save or repository open/close events. If the initial checkIgnore call fails or races with git initialization, there's no retry — files stay un-dimmed until the user manually saves .gitignore, which matches the reported workaround.

## Change summary

Git-ignored files in the Explorer are not dimmed when first opening a folder, requiring a manual .gitignore save to trigger the decoration.

## Best solution

Add a retry or delayed re-check in `GitIgnoreDecorationProvider` (extensions/git/src/decorationProvider.ts) so that when the initial `checkIgnore` batch fails or the repository isn't ready yet, decorations are re-requested after git initialization completes. The `model.onDidOpenRepository` event already triggers a refresh but may fire before the repository is fully usable for `checkIgnore`.
