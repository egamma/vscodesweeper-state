# Issue #325616 — Feature request: User-level default Docker volume mounts for Dev Containers

[https://github.com/microsoft/vscode/issues/325616](https://github.com/microsoft/vscode/issues/325616)

## Summary

This is a feature request for the Dev Containers extension (ms-vscode-remote.remote-containers), not VS Code core. The requested settings (`dev.containers.defaultMounts`, `dev.containers.defaultRunArgs`) would be implemented in the Dev Containers extension or the devcontainer spec, and should be filed in microsoft/vscode-remote-release.

## Change summary

Add user-level default Docker volume mounts and runArgs that are automatically merged into all Dev Containers without requiring per-repository configuration.

## Best solution

File this in https://github.com/microsoft/vscode-remote-release (the issue tracker for the Remote Development extensions including Dev Containers) or https://github.com/devcontainers/spec (the devcontainer spec repository). The Dev Containers extension owns container lifecycle configuration; VS Code core has no involvement in Docker mount handling.
