# Issue #150083 — When Debug: Inline Values is set together with Word wrap, breakpoints are not correctly focused 

[https://github.com/microsoft/vscode/issues/150083](https://github.com/microsoft/vscode/issues/150083)

## Summary

Fixed on main: commit 9aaf087e6bb explicitly preserves the cursor's vertical viewport position when inline value decorations are applied with word wrap enabled, which is exactly the scroll-away problem described here.

## Change summary

Debug inline values combined with word wrap cause the breakpoint line to scroll out of view when the debugger pauses, because decorations are applied after the reveal.

## Best solution

Keep the shipped fix in commit 9aaf087e6bb (VS Code ~1.90); no further change needed. The fix preserves the cursor's vertical position relative to the viewport when applying inline value decorations under word wrap.
