# Issue #119481 — Data breakpoints: support modifying current breakpoint access type

[https://github.com/microsoft/vscode/issues/119481](https://github.com/microsoft/vscode/issues/119481)

## Summary

Still a valid gap: data breakpoints can be created with an access type, but there is no GUI to modify the access type after creation. The model field is readonly and the update interface only supports condition/hitCondition changes. Keeping open as a niche but reasonable debug feature request.

## Change summary

Allow users to change the access type (read/write/readWrite) of an existing data breakpoint through the breakpoints view UI.

## Best solution

Extend the data breakpoint update path to support changing `accessType`: make `DataBreakpoint.accessType` mutable in `src/vs/workbench/contrib/debug/common/debugModel.ts`, add `accessType` to the update interface in `debug.ts`/`debugService.ts`, and surface it via a context menu action or inline edit in `breakpointsView.ts`. The access type picker (quick-pick) already exists for creation and could be reused.
