# Issue #235391 — Git - Improve algorithm for picking the base branch it uses when populating Source Control Graph in Auto mode

[https://github.com/microsoft/vscode/issues/235391](https://github.com/microsoft/vscode/issues/235391)

## Summary

Valid feature request — the base-branch algorithm in `getDefaultRemote()` still hardcodes preference for 'origin' on current main, so fork contributors tracking an upstream remote don't get the ideal merge-base in Auto mode. No fix or tracking issue found.

## Change summary

When working with a forked repo that has an 'upstream' remote, the Source Control Graph Auto mode picks origin/main instead of upstream/main as the merge base.

## Best solution

Enhance `getDefaultRemote()` in `extensions/git/src/repository.ts` (or the `getBranchBase` flow) to detect fork topologies and prefer the upstream remote when determining the merge base — potentially gated by a setting. The git extension owner (@lszomoru) should decide whether to always prefer upstream or make it configurable.
