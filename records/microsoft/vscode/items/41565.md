# Issue #41565 — Git - Detect git repositories under ignored paths

[https://github.com/microsoft/vscode/issues/41565](https://github.com/microsoft/vscode/issues/41565)

## Summary

Both scenarios described in this issue are addressed by existing settings on current main: `git.repositoryScanMaxDepth` (raise past nesting depth to discover sub-repos under ignored paths — the scan does not consult .gitignore) and `git.openRepositoryInParentFolders` (prevents ancestor-repo ignore decorations from bleeding into a child folder). The last commenter confirmed both work on 1.129.

## Change summary

Git repositories nested inside gitignored paths are not detected by VS Code's SCM, so diffs and source control features are unavailable for those repos.

## Best solution

No further code change needed. The workspace scan in `extensions/git/src/model.ts` (`traverseWorkspaceFolder`) already ignores `.gitignore` — it traverses by filesystem only, gated on `repositoryScanMaxDepth`. Increasing that setting picks up sub-repos under ignored paths. The ancestor-repo greying-out case is handled by `openRepositoryInParentFolders` (default `prompt`). Close as completed.
