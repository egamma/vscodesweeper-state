# Issue #236512 — Selected tab underline thickness is hard to see

[https://github.com/microsoft/vscode/issues/236512](https://github.com/microsoft/vscode/issues/236512)

## Summary

Valid feature request for customizable active tab border thickness. The color is customizable via theme tokens, but thickness is hardcoded. No existing setting or capability covers this; 16 upvotes (approaching the 20-upvote promotion threshold).

## Change summary

Allow customizing the active tab underline thickness (currently hardcoded at 1px) across editor tabs, panel tabs, and activity bar indicators.

## Best solution

Introduce a workbench setting (e.g. `workbench.tabs.activeBorderWidth`) or a theme color token dimension that controls the border thickness for active tabs, activity bar indicators, and panel tabs. The implementation would touch `src/vs/workbench/browser/parts/editor/multiEditorTabsControl.ts` (where `TAB_ACTIVE_BORDER` is applied) and the corresponding panel/activity bar styling. Alternatively, simply increase the default from 1px to 2px as a polish pass, since the accessibility argument for visibility is strong.
