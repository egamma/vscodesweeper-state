# Issue #236512 — Selected tab underline thickness is hard to see

[https://github.com/microsoft/vscode/issues/236512](https://github.com/microsoft/vscode/issues/236512)

## Summary

Reasonable feature request with 16 upvotes asking to make active tab underline thicker (or configurable). The border is currently hardcoded at 1px in CSS; color is customizable via `tab.activeBorder` but thickness is not. Touches accessibility (visibility of active state). Keeping open for community traction.

## Change summary

Allow customization of the active tab underline thickness, which is currently hardcoded to 1px and difficult to notice in light themes.

## Best solution

Either increase the default active-tab border thickness (e.g. to 2px) across the board for better visibility, or expose a setting/theme token for thickness. The hardcoded `height: 1px` lives in `src/vs/workbench/browser/parts/editor/media/multieditortabscontrol.css` (line ~324). A uniform default increase is more aligned with VS Code's 'simple, uniform UI' value than a per-component setting.
