# Issue #213149 — Speech To Text in VS code is awkward on MacOS

[https://github.com/microsoft/vscode/issues/213149](https://github.com/microsoft/vscode/issues/213149)

## Summary

Keep open: the report has concrete macOS Dictation repro videos and maps to the editor textarea/composition input path, but I did not find a specific fixing commit or confirmed duplicate.

## Change summary

macOS built-in Dictation can render overlapping text with word wrap and can duplicate, delete, or mis-capitalize dictated text in the VS Code editor.

## Best solution

Route this through editor input on macOS and decide whether VS Code can improve its textarea/composition handling for system text services, or explicitly document that users should use VS Code's own editor dictation command when a speech provider is installed.
