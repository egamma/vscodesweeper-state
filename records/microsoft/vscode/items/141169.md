# Issue #141169 — Trimming final newline is not POSIX standards compliant

[https://github.com/microsoft/vscode/issues/141169](https://github.com/microsoft/vscode/issues/141169)

## Summary

Actively scheduled item (On Deck milestone) requesting that `files.insertFinalNewline` default to `true` on POSIX platforms. The default remains `false` on current main; the discussion is live and the team has signaled intent to address it.

## Change summary

Change the default of `files.insertFinalNewline` to `true` on Linux/macOS to comply with POSIX text-file conventions.

## Best solution

Change the platform-conditional default of `files.insertFinalNewline` to `true` on Linux (and likely macOS) in `src/vs/workbench/contrib/files/browser/files.contribution.ts`, paired with the already-shipped `editor.renderFinalNewline: dimmed` on Linux so users understand the implicit trailing newline. The related PR #175597 attempted this but remains open — landing it (or a successor) resolves the issue.
