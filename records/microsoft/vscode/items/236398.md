# Issue #236398 — Clicking URLs in terminal while connecting are opened only after connecting

[https://github.com/microsoft/vscode/issues/236398](https://github.com/microsoft/vscode/issues/236398)

## Summary

Still reproduces on main: `MainThreadUriOpeners.getOpeners()` calls `activateByEvent` with default `ActivationKind.Normal`, which awaits `_installedExtensionsReady` — a barrier that stays closed during remote authority resolution. URLs clicked in the terminal during connect (e.g. 2FA prompts) block until the connection completes. An open community PR (#310834) targets this exact issue but has not merged.

## Change summary

Clicking URLs printed in the terminal during remote connection (e.g. 2FA links) does nothing until the connection finishes, then all deferred opens fire at once.

## Best solution

Review and land PR https://github.com/microsoft/vscode/pull/310834, which switches `activateByEvent` to `ActivationKind.Immediate` in `mainThreadUriOpeners.ts` so the opener falls through to the default external handler when the remote extension host hasn't resolved yet. The fix is small and well-scoped.
