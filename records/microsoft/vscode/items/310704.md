# Issue #310704 — Change colour of status bar while attached to running process when broken on exception/breakpoint

[https://github.com/microsoft/vscode/issues/310704](https://github.com/microsoft/vscode/issues/310704)

## Summary

Valid feature request: VS Code's status bar currently uses a single color for the entire debug session (Running and Stopped states alike). The request asks for a distinct color when the debugger is stopped at a breakpoint or exception, similar to Visual Studio. No duplicate found, and the capability does not exist on current main.

## Change summary

Add a distinct status bar color when the debugger is stopped at a breakpoint or exception, differentiating it from the running-debug color.

## Best solution

Add new theme color tokens (e.g. `statusBar.debuggingStoppedBackground`, `statusBar.debuggingStoppedForeground`, `statusBar.debuggingStoppedBorder`) and update `StatusBarColorProvider.update()` in `src/vs/workbench/contrib/debug/browser/statusbarColorProvider.ts` to apply them when `State.Stopped` is active. This belongs in core since the status bar color override mechanism is already built into the debug contribution.
