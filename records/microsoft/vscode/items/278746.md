# Issue #278746 — `cannot set privileged capabilities: Operation not permitted` error for Snap installed packages

[https://github.com/microsoft/vscode/issues/278746](https://github.com/microsoft/vscode/issues/278746)

## Summary

Legitimate snap packaging issue where VS Code's integrated terminal inherits constraints that prevent other snap-installed programs from setting privileged capabilities. A team member has already engaged; the root cause is in snap's security model interaction with VS Code's process spawning, not in VS Code's own code.

## Change summary

Running snap-installed programs (e.g. shellcheck) from VS Code's integrated terminal fails with 'cannot set privileged capabilities: Operation not permitted' because the spawned processes cannot elevate capabilities within the snap context.

## Best solution

The fix likely requires changes to VS Code's snap packaging (e.g. migrating from core20 to core24 per https://github.com/microsoft/vscode/issues/241636, or adjusting how the terminal spawns child processes to avoid inheriting snap capability restrictions). The team member (@deepak1556) who owns Electron/platform integration is already tagged for investigation.
