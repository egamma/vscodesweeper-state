# Issue #278746 — `cannot set privileged capabilities: Operation not permitted` error for Snap installed packages

[https://github.com/microsoft/vscode/issues/278746](https://github.com/microsoft/vscode/issues/278746)

## Summary

Real issue caused by Electron's Chromium sandbox restricting child processes from setting privileged capabilities, which breaks snap-confined binaries launched from VS Code's integrated terminal. A team member is already investigating. Keeping open as an upstream/sandbox interaction problem.

## Change summary

Snap-installed packages fail with 'cannot set privileged capabilities: Operation not permitted' when run from VS Code's integrated terminal, because Electron's sandbox restrictions propagate to child processes.

## Best solution

The root cause is the interaction between Electron's Chromium sandbox (which sets `PR_SET_NO_NEW_PRIVS` via seccomp) and snap confinement requiring elevated capabilities. The fix likely lives in how the pty host or terminal child processes inherit sandbox flags — either in Electron upstream or in VS Code's pty host spawning. deepak1556 (already pinged) is the right person to assess whether this needs an Electron-side fix or a VS Code workaround in the terminal process spawning.
