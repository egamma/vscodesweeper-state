# Issue #315199 — Suggestion: check .gitattributes for linguist-language before giving up on syntax highlighting

[https://github.com/microsoft/vscode/issues/315199](https://github.com/microsoft/vscode/issues/315199)

## Summary

Valid feature request to read .gitattributes linguist-language overrides for file-type detection. No existing implementation or duplicate found. Low traction so far (1 upvote); leaving open for community process.

## Change summary

Use .gitattributes linguist-language annotations to determine syntax highlighting / language mode for files that lack a recognized extension.

## Best solution

This could be implemented in the language detection service (src/vs/workbench/services/languageDetection/) by parsing .gitattributes for linguist-language entries and feeding them into the file-association resolution. However, given VS Code's value of keeping the core lightweight and its existing per-workspace `files.associations` setting that already solves this use case manually, this might be better served as an extension (reading .gitattributes and calling `vscode.languages.setTextDocumentLanguage`). Leave open for community traction and maintainer judgment.
