# Issue #52131 — cursor up/down is confused by selection

[https://github.com/microsoft/vscode/issues/52131](https://github.com/microsoft/vscode/issues/52131)

## Summary

The current behavior is intentional and confirmed by a team member: down moves relative to the end of the selection, up moves relative to the beginning. This is consistent with SublimeText and is by design. The request for an option has had zero upvotes over 7+ years.

## Change summary

After making a leftward shift-selection and releasing shift, pressing down should move relative to the visual cursor column (selection start), not the selection end.

## Best solution

Close as designed. The current convention — down/right anchor to selection end, up/left anchor to selection start — is intentional, documented in the code, and consistent with SublimeText. An option to change this behavior has attracted no community interest in over seven years.
