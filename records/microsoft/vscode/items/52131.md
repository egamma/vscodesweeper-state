# Issue #52131 — cursor up/down is confused by selection

[https://github.com/microsoft/vscode/issues/52131](https://github.com/microsoft/vscode/issues/52131)

## Summary

Hard keep-open: issue carries the 'On Deck' milestone. Additionally, alexdima confirmed the current behavior is intentional (down collapses to end of selection), and the reporter acknowledged it as a preference, requesting an option.

## Change summary

Request for a configurable option controlling whether cursor-down after a leftward shift-selection collapses to the selection start or end.

## Best solution

If the team decides to act on this, a setting (e.g. 'editor.cursorDownAtSelectionStart') in `src/vs/editor/browser/coreCommands.ts` or the cursor move logic could toggle the collapse point for up/down movement. Given alexdima's comment that the current behavior is intentional and consistent with SublimeText, it may also be closed as `*as-designed` — but that decision belongs to the milestone owner.
