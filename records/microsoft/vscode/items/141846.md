# Issue #141846 — GitHub - Bad credentials is shown when the github extension fails to fetch repositories

[https://github.com/microsoft/vscode/issues/141846](https://github.com/microsoft/vscode/issues/141846)

## Summary

Team-authored issue (MEMBER); the bug is still present on current main — the GitHub extension's `getRemoteSources` path has no error handling for expired/invalidated credentials and does not retry with `forceNewSession`. Keeping open per hard keep-open rules and because the defect remains.

## Change summary

When a GitHub SAML session is invalidated, the GitHub extension shows a raw 'Bad credentials' error instead of re-prompting for authentication.

## Best solution

Add error handling in `extensions/github/src/remoteSourceProvider.ts` (around the `getRemoteSources`/`getUserRemoteSources` calls) and/or in `extensions/github/src/auth.ts` (`getOctokit`) to catch 401/Bad-credentials responses from Octokit and retry with `forceNewSession: true` on `authentication.getSession`, so the user is re-prompted instead of seeing a raw error.
