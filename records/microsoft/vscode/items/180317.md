# Issue #180317 — Git - publish branch behave confusingly after git shallow clone

[https://github.com/microsoft/vscode/issues/180317](https://github.com/microsoft/vscode/issues/180317)

## Summary

Valid request to warn or handle the confusing 'Publish Branch' button behavior in shallow clones. An open PR (https://github.com/microsoft/vscode/pull/296052) is actively attempting to address this — keep open pending that PR's review.

## Change summary

After a shallow clone, publishing a branch succeeds but VS Code keeps showing 'Publish Branch' because remote tracking refs aren't fetched, making the upstream invisible to the git extension.

## Best solution

Land or close the open PR https://github.com/microsoft/vscode/pull/296052, which adds a warning when publishing from a shallow clone. The fix belongs in `extensions/git/src/` — the publish flow in `commands.ts` and/or the upstream detection in `statusbar.ts`/`actionButton.ts` need to detect shallow state (via `git rev-parse --is-shallow-repository`) and either warn the user or suggest `git fetch --unshallow`.
