# Issue #202418 — Folder collapses after rename when changing letter case

[https://github.com/microsoft/vscode/issues/202418](https://github.com/microsoft/vscode/issues/202418)

## Summary

Still reproduces on current main. When renaming a folder with only a case change on macOS, the tree identity (based on the case-sensitive URI string) changes, causing the tree to lose expansion state. The bug is in ExplorerItem.getId() and the refresh path after rename.

## Change summary

Renaming a folder with only a letter-case change (e.g. `name` → `Name`) on macOS causes the folder to collapse in the Explorer, losing its expanded state.

## Best solution

In the rename path in `explorerService.ts` (line ~395), preserve the tree's expansion state before calling `modelElement.rename()` and restore it afterward, since `rename()` changes the resource URI (and thus `getId()` output) even though the node is logically the same. Alternatively, the `refresh()` call on the parent could be made aware that a case-only rename should transfer expansion state from the old identity to the new one.

## Evidence

- **identity source:** ExplorerItem.getId() at src/vs/workbench/contrib/files/common/explorerModel.ts:192 returns `root.resource.toString() + '::' + this.resource.toString()`, which is case-sensitive. A case-only rename changes the URI string, so the tree sees a new identity after refresh.
- **rename path:** src/vs/workbench/contrib/files/browser/explorerService.ts:395-400 handles same-parent moves (renames): it calls `modelElement.rename(newElement)` which updates the resource URI, then `view.refresh(shouldDeepRefresh, modelElement.parent)`. The tree refresh sees a child with a changed identity and does not preserve its expansion.
