# Issue #295305 — Source Control Graph fails to refresh when git.autorefresh is false

[https://github.com/microsoft/vscode/issues/295305](https://github.com/microsoft/vscode/issues/295305)

## Summary

Valid bug: the SCM Graph update path depends on observable changes fired by `GitHistoryProvider.onDidRunWriteOperation`, but when `git.autorefresh` is false the intermediate `onFileChange` → `updateWhenIdleAndWait` path is skipped. While `Git: Refresh` does trigger `updateModelState` and should fire the history events, there's a plausible scenario where the delta computation yields no changes (e.g., after a fetch where refs look unchanged) leaving the graph stale. The graph's own refresh button should unconditionally re-fetch but the reporter says it doesn't — this needs investigation.

## Change summary

Source Control Graph fails to refresh reliably when `git.autorefresh` is set to false, even after manual refresh actions.

## Best solution

Investigate whether the graph refresh button (`workbench.scm.action.graph.refresh`) reliably triggers a full data re-fetch from the history provider regardless of observable state. The fix likely involves ensuring `view.refresh()` or `Git: Refresh` forces the graph data source to call `provideHistoryItems` with fresh data from the git extension even when the observable delta is empty. Likely files: `extensions/git/src/historyProvider.ts` (the delta/event emission) and `src/vs/workbench/contrib/scm/browser/scmHistoryViewPane.ts` (the refresh path).
