# Issue #323908 — Starting MCP servers block starting sessions

[https://github.com/microsoft/vscode/issues/323908](https://github.com/microsoft/vscode/issues/323908)

## Summary

Team-authored issue (hard keep-open). The core blocking behavior is SDK-side (tracked in copilot-mcp-core#1892); a UX affordance (progress hint) was merged in PR #324103, but the underlying 30s blocking startup remains unresolved.

## Change summary

MCP server initialization blocks agent host session startup for ~30 seconds, delaying the first model request.

## Best solution

The SDK (copilot-mcp-core) should allow sessions to start without waiting for MCP servers, or at least signal progress so the host can stream status. The merged PR (aaad32718ff) is a partial mitigation showing a hint; the real fix is non-blocking MCP initialization tracked upstream in github/copilot-mcp-core#1892.
