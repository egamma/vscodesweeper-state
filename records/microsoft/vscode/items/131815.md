## Summary

Still a valid feature request but blocked on upstream: Electron closed the corresponding issue (electron/electron#31028) as won't-fix, and VS Code's trash operation calls `shell.trashItem` with no progress callback. With 16 upvotes (below the 20 threshold) and no upstream path forward, this remains under-discussion but isn't actionable today.

## Change summary

Show a percentage-based progress indicator when deleting (trashing) a large directory in the file explorer.

## Best solution

This is blocked on Electron providing progress information from `shell.trashItem`. The upstream request was declined (https://github.com/electron/electron/issues/31028). A workaround could be an indeterminate progress notification (already partially done for long operations), but percentage-based progress as requested is not feasible without upstream support. Keep open as `under-discussion` for now; if community interest grows past 20 upvotes, promote to Backlog.
