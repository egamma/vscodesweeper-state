# Issue #310726 — CLI for tunnels not accepting GitHub PATs for unclear reasons

[https://github.com/microsoft/vscode/issues/310726](https://github.com/microsoft/vscode/issues/310726)

## Summary

The CLI accepts a GitHub PAT via `--access-token` and validates it against the GitHub API, but the Dev Tunnels service rejects it with 401 because it expects OAuth tokens issued through the registered client_id flow — the CLI should either reject unsupported token types at login time or document this limitation. A team member has already routed this internally.

## Change summary

Using a GitHub PAT with `code tunnel user login --provider github --access-token` succeeds at login but fails with 401 when creating a tunnel.

## Best solution

Either validate at login time that the token was issued via the correct OAuth app (and surface a clear error explaining PATs aren't supported for tunnels), or document the limitation. The fix likely touches `cli/src/auth.rs` around the `get_tunnel_authentication` path or the login flow. Since the Dev Tunnels API is a separate service, the CLI-side fix is better error messaging rather than making PATs work.
