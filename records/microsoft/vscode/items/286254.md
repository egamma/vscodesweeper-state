# Issue #286254 — Chat suggest isn't positioned correctly when editing an old prompt in a chat editor

Team-authored issue (Tyriar is a core VS Code contributor). Keeping open per team-authored policy.

## Change summary

The suggest (autocomplete) widget appears at the wrong position (bottom-right of screen) when editing a previous prompt inline within the chat list view.

## Best solution

When the inline editing input is created inside the chat list row (the `!isInput` branch in `chatWidget.ts` around line 1766–1774), ensure the suggest widget uses the `editorOverflowWidgetsDomNode` correctly or that `fixedOverflowWidgets` is set so the suggest popup is positioned relative to the window rather than the scrolled/transformed tree container. The likely files are `src/vs/workbench/contrib/chat/browser/widget/chatWidget.ts` and `src/vs/workbench/contrib/chat/browser/widget/input/chatInputPart.ts`.
