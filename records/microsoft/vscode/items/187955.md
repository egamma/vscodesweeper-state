# Issue #187955 — Incorrect $PATH order in task shell

[https://github.com/microsoft/vscode/issues/187955](https://github.com/microsoft/vscode/issues/187955)

## Summary

Still reproduces on main: task terminals use `useShellEnvironment = true` which resolves the base env via a login shell (`process.shellEnv()`), then launches the task command in another login shell (`-l`), causing `path_helper` to re-order PATH with system paths first. No fix has been committed.

## Change summary

VS Code task shells have incorrect $PATH ordering — system paths precede user-prepended paths (e.g. from .zshrc), unlike the integrated terminal.

## Best solution

The task shell launch should either not use a login shell for the command invocation (avoiding double `path_helper` invocation), or the environment resolution should be aligned with how the integrated terminal handles PATH ordering. The fix likely touches `src/vs/workbench/contrib/tasks/browser/terminalTaskSystem.ts` (the `-l` flag in shell args) and possibly `src/vs/workbench/contrib/terminal/browser/terminalProcessManager.ts` (the env merge logic).
