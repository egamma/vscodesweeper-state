## Change summary

Accepting pending changes from an earlier agent session silently overwrites all work done in later sessions, causing data loss.

## Best solution

Implement a staleness guard in the chat editing accept flow (likely in `src/vs/workbench/contrib/chat/browser/chatEditing/`) that detects when a file's content has changed since the edit was proposed, and either blocks the accept with a warning or marks the old session's edits as superseded.
