# Issue #244379 — cursorSurroundingLines in v1.98.2 causes text to be selected when clicking near the region

[https://github.com/microsoft/vscode/issues/244379](https://github.com/microsoft/vscode/issues/244379)

## Summary

The reported scroll on mouse click is the documented behavior when `editor.cursorSurroundingLinesStyle` is set to `all`; current `main` still intentionally applies cursor surrounding-line padding to mouse-triggered reveals in that mode. This should be closed as designed, with the workaround of using the default style if mouse clicks should not enforce scroll-off.

## Change summary

Clicking near the top or bottom cursor-surrounding region with `cursorSurroundingLinesStyle: "all"` scrolls the editor and can begin text selection while the mouse button is held.

## Best solution

Keep the documented `all` behavior in editor-core; users who want mouse clicks to avoid scroll-off should use the default `cursorSurroundingLinesStyle`, while any delayed-until-mouseup behavior would need a separate feature request.

- **Proposed comment:**

  > Thanks for the report. With `editor.cursorSurroundingLinesStyle: "all"`, this is the intended behavior: VS Code enforces `editor.cursorSurroundingLines` for mouse-triggered cursor moves too, so clicking near the top or bottom scroll-off region can scroll while the mouse button is still down. To avoid that behavior, remove the override or set `editor.cursorSurroundingLinesStyle` back to `default`, where mouse clicks do not enforce the surrounding-lines padding. Closing as designed.

## Evidence

- **report configuration:** The issue reproduces with `editor.cursorSurroundingLines: 8` and `editor.cursorSurroundingLinesStyle: "all"`, which explicitly opts into enforcing cursor surrounding lines for all cursor movements.
- **setting contract:** `src/vs/editor/common/config/editorOptions.ts:6210-6219` documents `cursorSurroundingLinesStyle.default` as enforcing only for keyboard/API, and `cursorSurroundingLinesStyle.all` as enforcing always.
- **current implementation:** `src/vs/editor/browser/viewParts/viewLines/viewLines.ts:720-730` ignores scroll-off for mouse/minimal reveal only when the style is `default`; with `all`, it applies padding from `cursorSurroundingLines`, causing the scroll described in the report.
- **history:** The `cursorSurroundingLinesStyle` switch was introduced in commit ea49afd15e387eb5a7e6b0cbbe70b9b35e8c568d (`Fix #79935. editor.cursorSurroundingLinesStyle.`), making the mouse-enforced behavior an explicit opt-in mode rather than the default.
