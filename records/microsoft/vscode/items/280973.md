# Issue #280973 — Chat Interface Stealing Focus / Opening By Default

[https://github.com/microsoft/vscode/issues/280973](https://github.com/microsoft/vscode/issues/280973)

## Summary

The described behavior — the secondary sidebar (containing Chat) opening automatically when a workspace is opened for the first time — is controlled by the `workbench.secondarySideBar.defaultVisibility` setting, which defaults to `visibleInWorkspace`. Setting it to `hidden` gives the user the experience they want. This setting has been available since v1.104 (September 2025), before this issue was filed. The identical complaint in #268973 was resolved by pointing to the same setting.

## Change summary

The Chat view opens by default in the secondary sidebar when opening a new workspace, overriding the user's expectation that Source Control would be shown instead.

## Best solution

Set `workbench.secondarySideBar.defaultVisibility` to `hidden` in user settings. This controls whether the secondary sidebar (which defaults to Chat) is shown when opening workspaces for the first time. The per-workspace view state is then restored on subsequent opens.
