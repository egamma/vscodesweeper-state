# Issue #153522 — Replacing with capture groups writes the first match's values to all matches in that line

[https://github.com/microsoft/vscode/issues/153522](https://github.com/microsoft/vscode/issues/153522)

## Summary

Still reproduces on main: when multiple regex matches with capture groups appear on the same line and need surrounding context (e.g. lookahead), `getReplaceString` always returns the first match's capture group values for all matches because `_regExp.exec(text)` finds only the first match in the full-line text.

## Change summary

Search-and-replace with capture groups writes the first match's captured values to all matches on the same line when the regex requires surrounding context (e.g. lookahead).

## Best solution

Fix `ReplacePattern.getReplaceString` (or the `replaceString` getter in `match.ts`) so that when the full surrounding text is passed for context, the regex is positioned to match at the correct offset — not always the first match. One approach: pass the match offset so the regex can start from the right position, or slice the text to start at the match's column.
