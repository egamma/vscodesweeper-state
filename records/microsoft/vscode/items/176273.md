# Issue #176273 — Cannot highlight U+200B and unhighlight multiple languages at the same time

[https://github.com/microsoft/vscode/issues/176273](https://github.com/microsoft/vscode/issues/176273)

## Summary

Still reproduces on main: the word-context early-return in `shouldHighlightNonBasicASCII` (line 216–221) suppresses invisible character highlighting when U+200B appears in a "word" containing non-confusable non-basic-ASCII characters (e.g. CJK). The invisible check at line 223 is never reached. This manifests in markdown/plaintext because those modes don't tokenize content as strings/comments, so full text sequences form single words via DEFAULT_WORD_REGEXP (which doesn't treat U+200B as whitespace since `\s` excludes it).

## Change summary

U+200B (zero-width space) is not highlighted in markdown/plaintext when `invisibleCharacters` is true but `nonBasicASCII` is false, because the word-context heuristic early-exits before the invisible character check.

## Best solution

Move the invisible character check (lines 223–228) above the word-context early-return (lines 216–221) in `src/vs/editor/common/services/unicodeTextModelHighlighter.ts`, so invisible characters are always flagged regardless of word composition. Invisible characters are a security concern and should never be suppressed by the 'this word looks intentionally non-Latin' heuristic.
