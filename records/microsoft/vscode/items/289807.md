# Issue #289807 — Really long thinking descriptions for cloud agents

[https://github.com/microsoft/vscode/issues/289807](https://github.com/microsoft/vscode/issues/289807)

## Summary

Team-authored issue (MEMBER) about overly long thinking descriptions for cloud agents. The summarization prompt has been improved since filing (now enforces 'under 10 words' with extensive formatting rules), but there is no hard length truncation enforced on the generated title, so the issue may still occur if the LLM does not comply with the prompt constraints.

## Change summary

Cloud agent thinking part titles can be excessively long, producing poor UX in the chat widget.

## Best solution

Add a hard character/word-count truncation on the generated title in `chatThinkingContentPart.ts` (around `setFinalizedTitle` / after `generateTitleViaLLM`) so that even when the LLM ignores the prompt's length constraint, the displayed title is capped. The prompt improvements from justschen help but are not a guarantee.
