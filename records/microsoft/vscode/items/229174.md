Hard keep-open: the issue carries the 'On Deck' milestone (a real scheduling milestone). Additionally, this is an upstream Electron issue with multi-monitor DPI scaling — no fix has landed on main, and the root cause lives in Electron's display bounds calculations.

## Change summary

VS Code window opens at its minimum size on a multi-monitor Windows setup with different display scaling factors (100% and 200%).

## Best solution

The root cause is in Electron's handling of display bounds across monitors with different DPI (electron/electron#10862, electron/electron#29605). The fix likely needs to happen upstream in Electron or be worked around in VS Code's window state serialization/restoration code in `src/vs/platform/windows/electron-main/windowImpl.ts`. A team member (deepak1556) was investigating and identified the callsite affected by display configuration at shutdown.
