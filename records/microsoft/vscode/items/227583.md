# Issue #227583 — Underscores not visible in search in Ubuntu 24.04.1 LTS

[https://github.com/microsoft/vscode/issues/227583](https://github.com/microsoft/vscode/issues/227583)

## Summary

Correctly triaged as upstream. The root cause is the new Ubuntu font (shipped with Ubuntu 24.04) whose underscore glyph descends below the baseline farther than the classic font, causing clipping in standard HTML input elements rendered by Chromium/Electron. The issue is already labeled `upstream` and `upstream-issue-pending`, and team members have confirmed the Electron/Chromium dependency. Keep open as an upstream tracker.

## Change summary

Underscores are visually clipped (invisible) in search input fields and other standard HTML inputs on Ubuntu 24.04 due to the new Ubuntu font's underscore glyph metrics.

## Best solution

Track until the upstream fix lands — either in the Ubuntu font package (adjusting underscore metrics), or in Chromium's text rendering (better descender handling for input elements). The workaround (`fonts-ubuntu-classic`) is documented in the thread. If the upstream fix stalls, VS Code could consider a CSS workaround (e.g. `overflow: visible` or slight padding increase on Linux input elements), but that's a band-aid — the real fix belongs upstream.
