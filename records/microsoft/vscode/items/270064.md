# Issue #270064 — Git Commit keeps suggesting to create a branch, even though main isn't protected

[https://github.com/microsoft/vscode/issues/270064](https://github.com/microsoft/vscode/issues/270064)

## Summary

Team-authored bug with an 'On Deck' milestone; the maintainer (lszomoru) is actively investigating — commit b604f475c59 (Apr 2026) fixed a related workspace-folder scope issue, and commit 75e023baf01 (May 2026) added tracing telemetry specifically to diagnose this class of false-positive branch-protection prompts. Keeping open per both the MEMBER and milestone keep-open rules.

## Change summary

Git commit action button incorrectly triggers a 'create new branch' picker on an unprotected branch when neither `git.branchProtection` nor GitHub branch-protection rules are configured.

## Best solution

The root cause likely lies in a race or stale-cache path in `GitHubBranchProtectionProvider` (globalState restoration at line 128 of `extensions/github/src/branchProtection.ts` serves potentially stale data before the async update completes) or in multi-folder configuration scoping for `GitBranchProtectionProvider`. The recently added tracing telemetry (commit 75e023baf01) should help narrow this down. A fix would ensure `isBranchProtected()` never returns true before the async GitHub-ruleset fetch completes, or clears the globalState cache when rules come back empty.
