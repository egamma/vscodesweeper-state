# Issue #134322 — Git - git setting `core.excludesFile` is ignored if placed inside an include.path of ~/.gitconfig

[https://github.com/microsoft/vscode/issues/134322](https://github.com/microsoft/vscode/issues/134322)

## Summary

This is an upstream ripgrep limitation: ripgrep does not parse gitconfig `[include]` directives, so `core.excludesFile` set via an included config file is invisible to VS Code's search. The upstream maintainer explicitly declined to fix this (ripgrep won't bundle a full git config parser). Keeping open as upstream.

## Change summary

VS Code search ignores `core.excludesFile` when that setting is defined inside a gitconfig `[include]`-referenced file rather than directly in `~/.gitconfig`.

## Best solution

The root cause is in ripgrep (https://github.com/BurntSushi/ripgrep/issues/1014, closed won't-fix). VS Code could work around it by resolving `git config --include --global core.excludesFile` and passing the result to ripgrep via `--ignore-file`, but that's a non-trivial enhancement. The pragmatic workaround is using the default `~/.config/git/ignore` location. If VS Code wants to own a fix, it would live in the ripgrep search integration layer (e.g. `src/vs/workbench/services/search/`).
