# Issue #136867 — ICodeEditor.setHiddenAreas optimizes away valid update

[https://github.com/microsoft/vscode/issues/136867](https://github.com/microsoft/vscode/issues/136867)

## Summary

Still reproduces on main: the setHiddenAreas optimization compares incoming ranges against decoration-tracked ranges, but decorations don't expand when onModelLinesInserted marks new lines as hidden at fold boundaries — so a valid update is skipped. Author is MEMBER; keeping open.

## Change summary

setHiddenAreas incorrectly skips an update when a line inserted at a fold boundary was marked hidden by onModelLinesInserted but the backing decoration didn't expand.

## Best solution

Stop relying on decorations to track the current hidden-area truth. Either update decorations in onModelLinesInserted to reflect the actual expanded state, or compare against the real projection state (modelLineProjections visibility) rather than stale decoration ranges. The fix is in src/vs/editor/common/viewModel/viewModelLines.ts around the setHiddenAreas equality check.
