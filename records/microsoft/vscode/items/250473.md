# Issue #250473 — Retry experience not good

[https://github.com/microsoft/vscode/issues/250473](https://github.com/microsoft/vscode/issues/250473)

## Summary

This is a UX design discussion from a contributor (isidorn) about improving the chat retry experience — icon choice and whether retry should merge with the edit-prompt flow. The issues raised are still present on current main (icon is still Codicon.refresh, retry is still a standalone button). A community comment adds an accidental-click concern. Keeping open as an active design discussion.

## Change summary

The chat retry button uses a generic refresh icon and exists as a separate action from editing the prompt, which may not be the ideal UX.

## Best solution

The chat team should decide whether to (a) update the retry icon to something more specific, (b) merge retry into the edit-prompt flow so users naturally re-submit after editing, and/or (c) add a confirmation dialog or undo for accidental retries (a confirmation already exists for edit-mode retries that modify files, but not for simple chat retries). The relevant code is in `src/vs/workbench/contrib/chat/browser/actions/chatTitleActions.ts` (RetryChatAction, line ~168).
