# Issue #323783 — GitHub Copilot model selector doesn't indicate model context window size if unconfigurable in v1.126

[https://github.com/microsoft/vscode/issues/323783](https://github.com/microsoft/vscode/issues/323783)

## Summary

The model picker hover already renders context window size on current main (line 1944 of chatModelPicker.ts), but only when the model's metadata includes maxInputTokens/maxOutputTokens. Whether Claude Haiku 4.5 populates that metadata depends on the Copilot extension and its server-side model catalog — if the data isn't provided, no context size appears. This is a reasonable request to ensure all models display context window info in the simplified hover.

## Change summary

The simplified model hover in the model picker does not show context window size for models like Claude Haiku 4.5 that lack configurable context options and cost data.

## Best solution

Ensure the Copilot extension's model catalog always populates maxInputTokens/maxOutputTokens metadata for every model, so the existing context-size rendering in chatModelPicker.ts (line ~1944) fires for all models. If some models genuinely lack this server-side data, the model picker could fall back to showing a qualitative context-size indicator. The fix likely spans both the Copilot extension (model metadata population) and possibly chatModelPicker.ts (fallback display).
