# Issue #191897 — [css] Wrong selector hover when using :has() 

[https://github.com/microsoft/vscode/issues/191897](https://github.com/microsoft/vscode/issues/191897)

## Summary

Still reproduces: the selector hover for `:has()` is handled in the upstream `vscode-css-languageservice` package's `selectorPrinting.ts`, which currently renders `:has` as an HTML attribute (e.g. `<article :has>`) rather than showing nested element context. The test file on the upstream `main` branch confirms this behavior is unchanged.

## Change summary

Hovering a CSS selector using `:has()` shows an incorrect HTML example that renders the pseudo-class as an attribute (e.g. `<article :has>`) instead of illustrating the relational selector with nested elements.

## Best solution

Fix the `selectorToMarkedString` method in `vscode-css-languageservice/src/services/selectorPrinting.ts` to handle `:has()` (and likely other relational pseudo-classes like `:is()`, `:where()`, `:not()`) by rendering the argument selector as nested child elements rather than treating the pseudo-class name as an attribute. Once fixed upstream, bump the dependency in `extensions/css-language-features/server/package.json`.
