# Issue #191897 — [css] Wrong selector hover when using :has() 

[https://github.com/microsoft/vscode/issues/191897](https://github.com/microsoft/vscode/issues/191897)

## Summary

Still reproduces on main. The bug lives in the upstream `vscode-css-languageservice` package's `selectorPrinting.js` — the `PseudoSelector` case in `toElement` doesn't handle pseudo-class functions like `:has()` with arguments, so it renders them as bare attributes in the hover HTML preview.

## Change summary

Hovering a CSS selector using `:has()` shows incorrect HTML example (e.g., `<article :has>` instead of demonstrating a matching DOM structure).

## Best solution

Fix the `toElement` function in `microsoft/vscode-css-languageservice` (`src/services/selectorPrinting.ts`) to handle pseudo-class functions (`:has()`, `:is()`, `:not()`, `:where()`) by either omitting them from the HTML preview or rendering a representative child structure. The `help wanted` label is appropriate — this is a well-scoped contribution opportunity in the upstream service.
