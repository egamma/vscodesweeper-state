# Issue #148093 — Language guessing: Allow paste contents to override `files.defaultLanguage`?

[https://github.com/microsoft/vscode/issues/148093](https://github.com/microsoft/vscode/issues/148093)

## Summary

Still unfixed on main: `files.defaultLanguage` causes `hasLanguageSetExplicitly` to be true, blocking paste-based language override. The feature request from team members remains valid and unimplemented.

## Change summary

Allow paste-from-known-source to override `files.defaultLanguage` when the untitled editor's language was set only by that setting, not by an explicit user action.

## Best solution

Distinguish `files.defaultLanguage` from explicit user language changes when setting `_languageChangeSource` in `untitledTextEditorModel.ts` (line ~165). Introduce a new source like `'setting'` so `hasLanguageSetExplicitly` remains false for setting-driven language, allowing paste override in `textResourceEditor.ts`. Also relax the `currentLanguageId !== PLAINTEXT_LANGUAGE_ID` guard to allow override when the language was set only by the default setting. A new setting to make this configurable (as discussed in comments) would be ideal.
