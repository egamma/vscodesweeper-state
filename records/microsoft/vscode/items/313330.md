# Issue #313330 — Race between replace_string_in_file and terminal commands in chat

[https://github.com/microsoft/vscode/issues/313330](https://github.com/microsoft/vscode/issues/313330)

## Summary

The race between the agent's file-edit tool and subsequent terminal commands is a real orchestration concern that lives in the Copilot extension layer. The issue is already correctly labeled `chat-ext-issue` and should stay open for the extension team to evaluate whether the tool pipeline should auto-save before terminal execution.

## Change summary

The chat agent edits a file via replace_string_in_file but the changes are not flushed to disk before the agent runs a terminal command against that file, causing the terminal to see stale content.

## Best solution

The Copilot extension's tool orchestration (in `extensions/copilot/src/extension/tools/`) should ensure edited files are saved to disk before a terminal command executes against them — either by auto-saving after replace_string_in_file completes or by adding a save step in the terminal tool's pre-execution path. This is an extension-internal fix, not a core VS Code change.
