# Issue #189960 — [Accessibility] Add a setting for users to choose whether to hear partial or the complete token in inline suggestion alert

[https://github.com/microsoft/vscode/issues/189960](https://github.com/microsoft/vscode/issues/189960)

## Summary

Valid accessibility feature request — not yet implemented. The `renderForScreenReader` method in `ghostText.ts` currently only returns the ghost text (partial token), with no setting to announce the complete token. Keeping open as a legitimate accessibility enhancement.

## Change summary

Add a setting to let screen reader users choose whether inline suggestion announcements read only the new (partial) text or the full completed token.

## Best solution

Add a configuration option (e.g. under `editor.screenReaderAnnounceInlineSuggestion` or `accessibility.verbosity`) that controls whether `renderForScreenReader` in `src/vs/editor/contrib/inlineCompletions/browser/model/ghostText.ts` returns the partial ghost text or the full line text including the already-typed prefix. The change is localized to that method and the controller that calls it.
