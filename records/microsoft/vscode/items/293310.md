# Issue #293310 — Terminal command edits are silently ignored when using Claude Agent SDK (works correctly with regular Copilot)

[https://github.com/microsoft/vscode/issues/293310](https://github.com/microsoft/vscode/issues/293310)

## Summary

Fixed on main: commit 4ca8211277d adds userEditedInput plumbing through the agent host session handler and ConfirmTerminalCommandTool so user edits to terminal commands are no longer silently discarded. Issue #307956 was closed as a duplicate of this one, confirming it's the canonical report.

## Change summary

User edits to agent-proposed terminal commands were silently ignored when using the agent host / Claude Agent SDK path, while working correctly in regular Copilot sessions.

## Best solution

Keep the shipped fix (commit 4ca8211277d); it adds `userEditedInput` to the tool-call-approved protocol action, threads it through the reducer and session handler, and returns it from `ConfirmTerminalCommandTool.invoke()`. No further change needed — confirm on the next stable release.
