# Issue #232726 — Hover: Consider changing defaults

[https://github.com/microsoft/vscode/issues/232726](https://github.com/microsoft/vscode/issues/232726)

## Summary

Still valid: compact is not yet the global default for IHoverService — it's only defaulted when a HoverStyle (Pointer/Mouse) is explicitly set. The non-compact styling also still exists. Team members agreed on the direction but the work hasn't landed.

## Change summary

Make compact the default hover appearance (or remove non-compact entirely) so consumers don't need to opt in.

## Best solution

Make `compact: true` the default in the hover widget when no explicit `appearance.compact` is provided (or remove the non-compact path entirely as discussed). Likely touches `src/vs/platform/hover/browser/hoverWidget.ts` and potentially removes explicit `compact: true` from ~25 call sites across the workbench.
