# Issue #300855 — Failed to load native module: pty.node

[https://github.com/microsoft/vscode/issues/300855](https://github.com/microsoft/vscode/issues/300855)

## Summary

The failing native module (`pty.node`) and the mismatched `prebuilds/win32-x64` directory live entirely inside the Copilot Chat extension's bundled SDK (`github.copilot-chat/node_modules/@github/copilot/sdk`), not in VS Code core. VS Code's own source has no `prebuilds` reference and its PTY handling is separate. This is a packaging/deployment bug in the Copilot Chat extension when targeting remote Linux hosts.

## Change summary

The Copilot Chat extension installs Windows native binaries (win32-x64) instead of Linux binaries on remote Linux targets (WSL2, SSH, DevContainers), breaking agent/background features.

## Best solution

This should be tracked and fixed in the Copilot Chat extension (github.copilot-chat) — the extension's platform-specific VSIX packaging or its remote installation logic is deploying the wrong platform's native prebuilds. The reporter should file at https://github.com/microsoft/vscode-copilot-release/issues.
