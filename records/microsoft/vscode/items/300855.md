# Issue #300855 — Failed to load native module: pty.node

[https://github.com/microsoft/vscode/issues/300855](https://github.com/microsoft/vscode/issues/300855)

## Summary

Fixed on main: the packaging pipeline now strips wrong-platform prebuilds (cb1bf5f3514, 1.113.0), and a runtime nodePtyShim was restored for the marketplace install route to copy the correct platform's pty.node at startup (17c5c32122f, 1.119.0). Both fixes address the reported scenario of Windows binaries appearing in WSL/remote Linux environments.

## Change summary

Copilot Chat extension loads Windows native binaries (pty.node) instead of Linux ones when running in WSL2 or remote Linux environments.

## Best solution

Keep the shipped fixes — the packaging filter (1.113.0) and the runtime nodePtyShim for marketplace installs (1.119.0) together resolve the wrong-platform binary problem for both built-in and marketplace extension routes on remote servers.
