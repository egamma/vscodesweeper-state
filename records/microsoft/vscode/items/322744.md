# Issue #322744 — Agent Host file linkifier seems broken

[https://github.com/microsoft/vscode/issues/322744](https://github.com/microsoft/vscode/issues/322744)

## Summary

Team-authored bug (hard keep-open). The file linkifier in chat tool-invocation messages appears broken — markdown file links render as raw text instead of clickable anchors. A commenter notes this affects thinking text rendering more broadly, not just agent host.

## Change summary

Markdown file links (e.g. `[foo.ts](file:///path)`) in agent host tool progress messages render as raw text instead of clickable links.

## Best solution

Investigate the markdown rendering path for tool invocation/thinking text content parts — likely in `src/vs/workbench/contrib/chat/browser/widget/chatContentParts/chatThinkingContentPart.ts` or the markdown decorations renderer (`chatMarkdownDecorationsRenderer.ts`) / inline anchor widget (`chatInlineAnchorWidget.ts`). The `file:` URI scheme links produced by tool display (e.g. `Editing [foo.ts](file:///repo/src/foo.ts)`) need to be properly linkified in whatever content part renders them.
