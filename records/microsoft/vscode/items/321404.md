# Issue #321404 — Copy/Paste to Windows file explorer not working

[https://github.com/microsoft/vscode/issues/321404](https://github.com/microsoft/vscode/issues/321404)

## Summary

Team-authored issue (MEMBER) — must keep open. The Copy command in the VS Code file explorer writes to a custom clipboard format (`code/file-list`) that Windows Explorer cannot read. The attempt to add native file clipboard support in PR #320685 was merged and then reverted (#323490), leaving this gap. The author notes it also fails on stable 1.124.2, suggesting this never worked for native-to-native paste.

## Change summary

Copying a file in VS Code's explorer and pasting in Windows file explorer does nothing — the native clipboard never receives the file paths in a format the OS recognizes.

## Best solution

Re-land the native file clipboard support from PR #320685 (by @lramos15) with the fixes that motivated the revert. The desktop `NativeClipboardService.writeResources` in `src/vs/workbench/services/clipboard/electron-browser/clipboardService.ts` needs to write file paths to the system clipboard in a format the OS file manager understands (e.g. via Electron's `clipboard.write({ files: [...] })` or equivalent).
