# Issue #239748 — Sort installed extensions by "Last Released"

[https://github.com/microsoft/vscode/issues/239748](https://github.com/microsoft/vscode/issues/239748)

## Summary

Valid feature request: there's no way to sort installed extensions by their marketplace last-release date. The gallery API already has `SortBy.LastUpdatedDate` but it's never exposed as a user-facing sort option for the installed list. Keeping open for the community process (6 upvotes, below the 20-vote backlog threshold).

## Change summary

Add a sort option for installed extensions by the date the extension was last released/updated on the marketplace.

## Best solution

Expose the gallery's `LastUpdatedDate` sort as a new `@sort:lastReleasedDate` (or similar) option in the extensions view query parser (`extensionQuery.ts` and `extensionsViews.ts`). The sort would use each extension's gallery `lastUpdated` metadata rather than the local install timestamp, letting users identify stale/abandoned extensions.

## Evidence

- **no existing sort:** src/vs/workbench/contrib/extensions/browser/extensionsViews.ts lines 243-248 show the available sort options: installs, rating, name, publishedDate, updateDate. No 'lastReleasedDate' or 'LastUpdatedDate' mapping exists.
- **gallery API supports it:** src/vs/platform/extensionManagement/common/extensionManagement.ts line 297 defines `SortBy.LastUpdatedDate` but it's never wired to a user-facing query sort command.
- **updateDate is local-only:** The existing `@sort:updateDate` sorts by `local.installedTimestamp` (line 850-853 in extensionsViews.ts), which reflects when the user installed/reinstalled, not when the publisher released a new version.
