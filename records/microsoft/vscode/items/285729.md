# Issue #285729 — Problem in VS Code task with -Command parameter

[https://github.com/microsoft/vscode/issues/285729](https://github.com/microsoft/vscode/issues/285729)

## Summary

Cannot reproduce the Set-Location behavior from the VS Code source: the task system sets cwd via shellLaunchConfig.cwd (OS-level), not via Set-Location, and -Command is added as a shell argument to powershell.exe. The Set-Location in the error must come from the user's environment or a terminal extension. Requesting more information to diagnose.

## Change summary

Running a shell task with a cwd option on Windows PowerShell produces a Set-Location error because -Command is misinterpreted as a parameter to Set-Location.

## Best solution

Determine the source of the Set-Location command — it is not generated by terminalTaskSystem.ts (which uses shellLaunchConfig.cwd at the OS process level). The user should provide their terminal automation profile settings, PowerShell version, and installed terminal extensions to identify whether the Set-Location is injected by an extension, a custom PowerShell module, or a terminal profile configuration.
