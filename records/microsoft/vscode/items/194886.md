# Issue #194886 — Screencast Mode doesn't reflect raw key strokes when input method is enabled

[https://github.com/microsoft/vscode/issues/194886](https://github.com/microsoft/vscode/issues/194886)

## Summary

Still reproduces on main: the screencast mode keydown handler returns early for IME keys (line 316) without displaying the raw physical key, and the compositionUpdate handler shows composed characters instead of raw keystrokes.

## Change summary

Screencast Mode shows composed IME characters (e.g. '한글') instead of the raw keystrokes typed (e.g. 'h', 'a', 'n', 'g', 'e', 'u', 'l') when an input method is active.

## Best solution

When an IME keydown is detected, use `e.code` to display the physical key label (e.g. 'KeyH' → 'H') in the keyboard overlay instead of returning early. The fix is localized to the keydown handler in `src/vs/workbench/browser/actions/developerActions.ts` around line 316.
