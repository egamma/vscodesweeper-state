# Issue #229154 — Window size becomes the size of the monitor when transferring maximized window across differently scaled monitors

[https://github.com/microsoft/vscode/issues/229154](https://github.com/microsoft/vscode/issues/229154)

## Summary

Team-authored upstream Electron issue, already correctly labeled. The root cause is that Electron lacks a `setNormalBounds()` API to update un-maximized bounds when a maximized window moves across monitors with different DPI scaling. Keeping open per hard keep-open rule (CONTRIBUTOR author) and because the upstream dependency remains unresolved.

## Change summary

When a maximized VS Code window is transferred across monitors with different DPI scaling, restoring (un-maximizing) it produces a window sized to the full monitor rather than the original pre-maximize dimensions scaled for the new monitor's DPI.

## Best solution

Report the missing `setNormalBounds()` capability to the Electron project (as bpasero suggested in the discussion). VS Code cannot fix this without Electron providing an API to update normalized bounds when a maximized window changes displays. Once Electron supports it, the fix would live in `src/vs/platform/windows/electron-main/windowImpl.ts` around the bounds-saving logic. Track the upstream Electron issue here.
