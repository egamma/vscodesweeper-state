## Summary

Valid feature request still unaddressed on main — the 1000-character limit in unicodeTextModelHighlighter.ts is hardcoded with no user-facing configuration. Only 3 upvotes and 621 days old, so it remains in the community process as a backlog candidate.

## Change summary

Make the hardcoded 1000-character limit for disabling unicode highlighting configurable via a user setting.

## Best solution

Add a new setting (e.g. `editor.unicodeHighlight.maximumHighlightedCharacters`) that feeds into the `MAX_RESULT_LENGTH` constant in `src/vs/editor/common/services/unicodeTextModelHighlighter.ts`, and wire it through `unicodeHighlightConfigKeys` in `editorOptions.ts`. Alternatively, allow a special value like `-1` or `Infinity` to mean 'no limit'.
