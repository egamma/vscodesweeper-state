# Issue #258864 — Cancelled terminal tool inputs should be readonly

[https://github.com/microsoft/vscode/issues/258864](https://github.com/microsoft/vscode/issues/258864)

## Summary

Still unfixed on main: after denying a terminal tool confirmation, the code editor remains editable. The `isReadOnly` flag in `chatTerminalToolConfirmationSubPart.ts` is only set when `presentationOverrides` is present, with no logic to make it readonly after cancellation/denial.

## Change summary

Cancelled/denied terminal tool inputs in chat should become readonly to signal that the command won't run.

## Best solution

After the confirmation widget resolves with a denial (ToolConfirmKind.Denied), set the editor to readonly in `chatTerminalToolConfirmationSubPart.ts` so the user cannot edit a command that will never execute.
