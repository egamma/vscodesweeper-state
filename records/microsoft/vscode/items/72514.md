# Issue #72514 — index.js/index.ts file should show at top of file picker when searching for the name of the directory

[https://github.com/microsoft/vscode/issues/72514](https://github.com/microsoft/vscode/issues/72514)

## Summary

Valid feature request with 74 upvotes, promoted to Backlog. The fuzzy scorer still has no directory-aware boosting for index files — this remains unimplemented on current main.

## Change summary

When typing a directory name in Quick Open, rank index.ts/index.js files in that directory near the top of results.

## Best solution

Enhance the fuzzy scorer or the quick-open ranking layer (src/vs/base/common/fuzzyScorer.ts or src/vs/workbench/contrib/search/browser/anythingQuickAccess.ts) to boost files whose parent directory matches the query when the filename is a conventional index file. Consider an extensible mechanism (as suggested in the comments) so frameworks like SvelteKit (+page.svelte) can also benefit.
