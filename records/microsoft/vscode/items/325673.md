# Issue #325673 — Agent host: Delete... on a Claude session doesn't delete the SDK transcript — session resurrects on next list refresh

[https://github.com/microsoft/vscode/issues/325673](https://github.com/microsoft/vscode/issues/325673)

## Summary

Confirmed bug: the agent-host Claude session delete path (`disposeSession`) tears down in-memory state and the VS Code overlay DB but never calls the SDK's `deleteSession`, so `listSessions` rediscovers the surviving transcript on the next refresh. An open PR (#325683) is already addressing this — keep open until it lands.

## Change summary

Deleting a Claude agent-host session appears to succeed but the session resurrects on the next list refresh because the SDK transcript is never removed.

## Best solution

Land or review https://github.com/microsoft/vscode/pull/325683, which should wire `this._sdkService.deleteSession(sessionId)` into `ClaudeAgent._teardownEntry` (or `disposeSession`), mirroring the existing call in `_removeAllTurns` at line 800 of `claudeAgent.ts`. The fix should also decide whether to delete transcripts of externally-created CLI sessions when the user explicitly deletes them from the Agents window.
