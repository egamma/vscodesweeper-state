# Issue #275262 — Keyboard shortcuts, diffs can be selected as context but don't actually get added

[https://github.com/microsoft/vscode/issues/275262](https://github.com/microsoft/vscode/issues/275262)

## Summary

Team-authored bug, still present on main. Diff editors and the Keyboard Shortcuts editor pass the `isSupportedChatFileScheme` filter because their underlying URIs use the `file` scheme, so they appear as selectable context items but silently fail to attach meaningful content. The filter needs to account for these editor types, either by excluding them or by properly resolving their content.

## Change summary

Selecting a diff view or the Keyboard Shortcuts editor as chat context shows them in the picker but does not actually attach their content.

## Best solution

The context picker's filter logic in `chatContextActions.ts` (line ~565) and `chatContext.ts` (line ~182) should detect diff editors and virtual editor tabs (like Keyboard Shortcuts) whose URI resolves to a real `file` scheme resource but whose content isn't that file. Either exclude these editor types from the picker entirely, or resolve them to meaningful content (e.g., both sides of a diff). The `chatAlwaysUnsupportedFileSchemes` set in `src/vs/workbench/contrib/chat/common/constants.ts` may also need the `vscodeDefaultKeybindings` scheme if one exists, or the filter should check editor input type rather than just URI scheme.
