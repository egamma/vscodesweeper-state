## Summary

Valid feature request for a setting to disable bidirectional text rendering. No such setting exists on current main; the team member engaged with a workaround suggestion (TM grammar isolation) but no resolution. Keeping open as under-discussion with only 3 upvotes and active dialogue.

## Change summary

Add a user setting to disable or force LTR bidirectional text rendering in the editor, particularly for fixed-format languages like COBOL with embedded RTL content.

## Best solution

The reporter already built an extension (NLTC.free-bidi) as a workaround. A core setting like `editor.bidiTextRendering` could live in `src/vs/editor/common/config/editorOptions.ts` and be consumed by the view line renderer, but the team may prefer keeping this in extension territory given the niche audience. The discussion between alexdima and the reporter should continue to determine if a core API gap exists that blocks the extension approach.
