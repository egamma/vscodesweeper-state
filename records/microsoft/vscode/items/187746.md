# Issue #187746 — tasks.json: Operating system specific properties can't be combined with the dependsOn property

[https://github.com/microsoft/vscode/issues/187746](https://github.com/microsoft/vscode/issues/187746)

## Summary

Confirmed bug still present on main: OS-specific blocks inside a task definition only override command and problemMatcher properties; `dependsOn` (and other configuration properties like `group`, `isBackground`) specified inside `linux`/`windows`/`osx` blocks are silently ignored.

## Change summary

Placing `dependsOn` inside a task's OS-specific block (`linux`, `windows`, `osx`) has no effect — the dependency is not applied.

## Best solution

Extend `ConfigurationProperties.from` in `src/vs/workbench/contrib/tasks/common/taskConfiguration.ts` to read and merge `dependsOn` (and other applicable configuration properties) from the OS-specific block matching the current platform, similar to how `ProblemMatcherConverter.fromWithOsConfig` already handles `problemMatcher`.
