# Issue #235106 — Test filter is hard to follow

[https://github.com/microsoft/vscode/issues/235106](https://github.com/microsoft/vscode/issues/235106)

## Summary

Still reproduces on main: the test explorer filter splits input on commas via `splitGlobAware(globText, ',')`, so searching for a suite named "IPC, Child Process" yields two independent OR-filters ("ipc" and "child process"), surfacing many unrelated matches. No fix has landed.

## Change summary

Searching for a test suite whose name contains a comma shows too many unrelated results because the filter text is split on commas into separate OR-terms.

## Best solution

Add a quoting mechanism (e.g. double-quote support) to the filter text parser in `testExplorerFilterState.ts` so that comma-containing suite names can be searched as a single term, or reconsider whether comma is the right separator when fuzzy matching is disabled.
