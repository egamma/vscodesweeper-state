# Issue #291104 — pulling from main ends up in edit session

[https://github.com/microsoft/vscode/issues/291104](https://github.com/microsoft/vscode/issues/291104)

## Summary

Team-authored issue (MEMBER) — must keep open for maintainer judgment. The bug describes a real UX problem where git operations outside an agent session bleed into the session's pending-files tracking, producing a spurious edit session.

## Change summary

Pulling from main during an agent session causes the git-pulled changes to appear as a huge edit session when the user skips the pending files prompt.

## Best solution

The agent session's file-change tracking should scope its dirty-file detection to only changes made by the agent, not external git operations like pulls or stashes. The fix likely lives in the edit session or agent session file-tracking layer — the component that decides which files to surface in the pending-files prompt needs to distinguish agent-authored changes from external VCS operations.
