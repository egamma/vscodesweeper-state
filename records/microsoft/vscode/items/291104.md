# Issue #291104 — pulling from main ends up in edit session

[https://github.com/microsoft/vscode/issues/291104](https://github.com/microsoft/vscode/issues/291104)

## Summary

Team-authored issue on the 'On Deck' milestone — keeping open for maintainer judgment. The bug describes Copilot's pending-files tracking incorrectly capturing a `git pull` as an edit session when the user skips changes during a background task.

## Change summary

Running `git pull` in the terminal while a Copilot background task has pending files, then selecting 'skip changes', incorrectly treats the pulled changes as an edit session.

## Best solution

The pending-files/edit-session tracking in the Copilot background task flow should scope its file-change detection to only changes made by the agent, not external VCS operations. The fix likely lives in the edit session or working copy diff logic that powers the 'pending files' prompt.
