# Issue #130880 — Allow key bindings to work on tree view items

[https://github.com/microsoft/vscode/issues/130880](https://github.com/microsoft/vscode/issues/130880)

## Summary

Valid API feature request on the 'On Deck' milestone — keybindings fired against tree views should pass the selected/focused tree item(s) as command arguments, matching context menu behavior. No implementation has landed; active design discussion exists among team members.

## Change summary

When a keybinding triggers a command registered in `view/item/context`, the focused/selected tree item should be passed as a command argument, just as it is when invoked from the context menu.

## Best solution

Implement a mechanism in the tree view keybinding dispatch path (likely in `src/vs/workbench/browser/parts/views/treeView.ts`) that resolves the currently selected/focused tree items and passes them as arguments when a keybinding fires a command associated with a focused tree view. The opt-in design discussed by connor4312 (tree must opt in, `when` clause includes view focus) seems like the right direction. Issue #236413 was correctly closed as a duplicate of this.
