# Issue #228718 — [106eb] potential listener LEAK detected, having 187 listeners already. MOST frequent listener (13):

[https://github.com/microsoft/vscode/issues/228718](https://github.com/microsoft/vscode/issues/228718)

## Summary

Still reproducible on main: each inline diff view zone registers its own onMouseMove listener on the modified editor. With many diff hunks, this triggers the leak detector. The fix is to use a single delegating listener or increase the threshold for this pattern.

## Change summary

Diff editor triggers a potential listener leak warning when a file has many inline diff hunks, because each InlineDiffDeletedCodeMargin registers its own onMouseMove/onMouseDown listeners on the shared modified editor.

## Best solution

Refactor InlineDiffDeletedCodeMargin to share a single onMouseMove listener at the DiffEditorViewZones level that dispatches to the active view zone, rather than each margin registering its own listener on the modified editor. This eliminates the O(n) listener accumulation per diff hunk.
