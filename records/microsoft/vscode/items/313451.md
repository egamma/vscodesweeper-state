# Issue #313451 — Avoid using `instanceof Promise` due to incompatibility with patched Promise implementations (e.g. zone.js)

[https://github.com/microsoft/vscode/issues/313451](https://github.com/microsoft/vscode/issues/313451)

## Summary

Keep open as an actionable compatibility cleanup: current main still has multiple runtime `instanceof Promise` checks, and VS Code already has an `isThenable` helper for this pattern. There is not enough evidence for a close because the reported pattern remains present and no canonical duplicate or fixing commit was found.

## Change summary

Replace brittle `instanceof Promise` checks with a thenable check where VS Code code needs to accept Promise-like values.

## Best solution

Audit the remaining non-test `instanceof Promise` checks and replace those that are meant to detect async results with `isThenable` from `src/vs/base/common/async.ts`, leaving only cases that intentionally require native `Promise` instances.

- **scope:** The issue is a broad code health and interoperability request rather than a narrowly reproduced product failure; some matches are tests or may intentionally assert native Promise behavior, so an audit is needed before changing all occurrences mechanically.
