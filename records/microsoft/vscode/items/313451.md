# Issue #313451 — Avoid using `instanceof Promise` due to incompatibility with patched Promise implementations (e.g. zone.js)

[https://github.com/microsoft/vscode/issues/313451](https://github.com/microsoft/vscode/issues/313451)

## Summary

VS Code already provides `isThenable()` (duck-type check) in `src/vs/base/common/async.ts` for places that need broad compatibility. The remaining `instanceof Promise` usages are intentional in internal code paths where VS Code controls promise creation and needs to distinguish native promises from arbitrary thenables for correctness or performance.

## Change summary

Replace all `instanceof Promise` checks with duck-type thenable checks to support patched Promise implementations like zone.js.

## Best solution

The existing `isThenable()` utility already solves this for extension-facing and API-boundary code. The remaining internal usages are deliberate — zone.js compatibility in Angular apps embedding Monaco is an extension/consumer concern, not a core VS Code concern. No blanket replacement is warranted.
