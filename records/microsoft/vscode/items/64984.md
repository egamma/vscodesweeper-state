# Issue #64984 — Antimalware Service Executable is still spiking when  some project is loaded 

[https://github.com/microsoft/vscode/issues/64984](https://github.com/microsoft/vscode/issues/64984)

## Summary

This is a long-standing upstream interaction between ripgrep (rg.exe) and Windows Defender's real-time scanning. The issue is already correctly labeled `upstream` — the root cause is in Windows Defender scanning files as ripgrep reads them, not in VS Code's own code. The parent issue https://github.com/microsoft/vscode/issues/63070 was closed as completed but this report documents the problem persisting. With the `upstream` label and `Backlog` milestone, this serves as a tracking issue for a problem VS Code cannot fully control.

## Change summary

Windows Defender's Antimalware Service Executable spikes CPU when VS Code's ripgrep search engine reads files, making search slow on Windows.

## Best solution

This remains an upstream Windows Defender interaction that VS Code cannot fully resolve in its own code. The known workaround (adding rg.exe to Windows Defender's process exclusion list) is the best user-facing mitigation. The issue serves as a valid tracking point for this platform-level friction; if ripgrep or Windows Defender changes resolve it, the issue can be closed then.
