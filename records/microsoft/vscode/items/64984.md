# Issue #64984 — Antimalware Service Executable is still spiking when  some project is loaded 

[https://github.com/microsoft/vscode/issues/64984](https://github.com/microsoft/vscode/issues/64984)

## Summary

This is a genuine upstream issue — Windows Defender scans file I/O generated by ripgrep (rg.exe) during search, causing CPU spikes. Already correctly labeled `upstream` and `confirmed`. The parent issue https://github.com/microsoft/vscode/issues/63070 was fixed and verified (specific rg.exe invocation patterns), but the residual interaction between ripgrep's file scanning and Windows Defender remains outside VS Code's control. Keeping open as the upstream tracker.

## Change summary

Windows Defender's Antimalware Service Executable spikes CPU when VS Code's ripgrep runs file or text searches on large projects.

## Best solution

The root cause is Windows Defender scanning ripgrep's high-volume file I/O. VS Code cannot fix Defender's behavior. The best end state is either (a) Microsoft's Defender team whitelists signed ripgrep binaries shipped with VS Code, or (b) ripgrep upstream finds a way to reduce the scan surface. The known user workaround is adding rg.exe to the Windows Defender process exclusion list. This issue serves as the canonical tracker for that upstream interaction.
