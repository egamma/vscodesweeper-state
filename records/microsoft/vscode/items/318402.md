# Issue #318402 — Terminal suggest completes incorrectly when text is wrapped

[https://github.com/microsoft/vscode/issues/318402](https://github.com/microsoft/vscode/issues/318402)

## Summary

Still reproduces on main: when terminal input wraps to a new line, the suggest acceptance logic sends backspaces based on replacement text length without accounting for how the terminal handles backspace across wrapped lines, leading to incomplete replacement.

## Change summary

Terminal suggest completes incorrectly when the typed command text wraps to a new line — the replacement doesn't fully remove the previous text.

## Best solution

The acceptance logic in `terminalSuggestAddon.ts` (around line 1015) needs to account for line wrapping when computing the backspace sequence. When text wraps, the shell may need different escape sequences (e.g., cursor-up + end-of-line) rather than plain backspaces to delete across a line boundary.
