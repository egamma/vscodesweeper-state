# Issue #229008 — Adopt new call stack widget for exceptions

[https://github.com/microsoft/vscode/issues/229008](https://github.com/microsoft/vscode/issues/229008)

## Summary

A team member (connor4312) explicitly adopted this issue to track adopting testing's call stack widget for debug exceptions. The exception widget (`exceptionWidget.ts`) still uses a plain `ZoneWidget` with link detection — it has not been ported to the richer `CallStackWidget`. Keeping open as team-owned tracked work.

## Change summary

Adopt the testing call stack widget to render debug exception stack traces with rich, clickable, highlighted frames instead of the current plain-text link-detection approach.

## Best solution

Replace the stack-trace rendering in `src/vs/workbench/contrib/debug/browser/exceptionWidget.ts` with the `CallStackWidget` (already living at `src/vs/workbench/contrib/debug/browser/callStackWidget.ts`) to provide the same rich stack-frame UI that testing's peek view offers. connor4312 owns this work.
