# Issue #182560 — Empty environment variables passed to vscode are unset

[https://github.com/microsoft/vscode/issues/182560](https://github.com/microsoft/vscode/issues/182560)

## Summary

Still open and valid: the root cause is in Chromium's utility process launch API which treats empty-string env values as 'unset'. A team member (deepak1556) confirmed it can be fixed in Electron. No fix has landed in either Electron or VS Code — the issue remains relevant.

## Change summary

Empty environment variables (e.g. `EMPTY_VAR=""`) passed to VS Code are dropped/unset in the extension host process due to Chromium's utility process launch API treating empty strings as 'remove this variable'.

## Best solution

The fix belongs in Electron's utility process fork: Electron should preserve empty-string environment variable values instead of treating them as unset. deepak1556 acknowledged this path. Track the upstream Electron fix and, once available, update the Electron dependency. Alternatively, a VS Code-side workaround in `utilityProcess.ts`'s `createEnv` could encode empty strings as a sentinel value, but the clean fix is upstream.
