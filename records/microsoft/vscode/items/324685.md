# Issue #324685 — Git: Always shows the same tag when multiple tags point to the same commit

[https://github.com/microsoft/vscode/issues/324685](https://github.com/microsoft/vscode/issues/324685)

## Summary

Confirmed in source: `getHEADRef()` uses `tags.find()` which picks the first tag matching the commit SHA, ignoring which tag was actually checked out. A real bug, but limited to users with multiple tags on one commit. Routing to the git area.

## Change summary

When multiple tags point to the same commit, VS Code's git integration always displays the first matching tag instead of the one that was explicitly checked out.

## Best solution

In `extensions/git/src/git.ts` `getHEADRef()`, when HEAD is detached and multiple tags match the commit, consult the git reflog (`git reflog -1 --format=%gs`) to determine which tag was actually checked out, and prefer that tag. Falls back to the current `find()` behavior when the reflog doesn't indicate a tag checkout.
