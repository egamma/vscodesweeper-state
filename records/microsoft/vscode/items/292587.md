# Issue #292587 — [Feature Request] User-Contributed Accessibility Help System

[https://github.com/microsoft/vscode/issues/292587](https://github.com/microsoft/vscode/issues/292587)

## Summary

Valid feature request for user-contributed accessibility help text via settings. Not implemented on main — the existing system only supports extension-contributed help content. The issue is in Backlog with 0 upvotes but the author tagged a team member for discussion, suggesting some internal interest.

## Change summary

Allow users to contribute custom Alt+F1 accessibility help text through VS Code settings rather than requiring an extension.

## Best solution

Decide whether this belongs as a user setting (adding custom help text per-view via configuration) or whether the existing extension contribution point (`accessibilityHelpContent` on view descriptors) is sufficient. If settings-based, it would likely touch `src/vs/workbench/contrib/accessibility/browser/extensionAccesibilityHelp.contribution.ts` and `src/vs/workbench/contrib/accessibility/browser/accessibilityConfiguration.ts`.
