# Issue #164209 — Git - .gitignore does not work for Hangul

[https://github.com/microsoft/vscode/issues/164209](https://github.com/microsoft/vscode/issues/164209)

## Summary

VS Code delegates .gitignore matching to `git check-ignore`; the encoding handling of non-ASCII filenames in .gitignore is a git behavior. The commenter confirms this is likely git's issue. Keeping open as upstream — VS Code could potentially work around it, but the root cause is in git's .gitignore parsing on Windows with non-UTF-8 system encodings.

## Change summary

.gitignore entries containing Hangul (or other non-ASCII/CJK characters) are not respected in VS Code's Git integration on Windows when the system encoding differs from UTF-8.

## Best solution

Classify as upstream (git). VS Code's git extension calls `git check-ignore` directly — if git itself doesn't honor the .gitignore entry due to encoding mismatch, VS Code can't easily fix this. The real fix is in git's handling of UTF-8 .gitignore on non-UTF-8 Windows systems (or enabling `core.quotePath`/UTF-8 codepage on the user's machine). A workaround note in the issue would help affected users.
