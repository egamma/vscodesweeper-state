# Issue #149236 — Git - Prune On Fetch

[https://github.com/microsoft/vscode/issues/149236](https://github.com/microsoft/vscode/issues/149236)

## Summary

The report conflates remote tracking branch pruning with local branch deletion. `git fetch --prune` (which `pruneOnFetch` enables) removes stale remote tracking refs — it does not delete local branches. The code on current main correctly passes `--prune` to `git fetch` when the setting is enabled. Without clarification of what exactly stopped working, this cannot be verified.

## Change summary

Reporter expects that enabling `git.pruneOnFetch` will automatically remove local branches whose upstream was deleted on the remote.

## Best solution

Request clarification: does the reporter expect local branch deletion (not what `--prune` does) or is `git fetch --prune` genuinely not running? If the former, this is as-designed — `pruneOnFetch` removes remote tracking refs, not local branches. If the latter, current code at `extensions/git/src/repository.ts:2278-2283` correctly reads the setting. A separate feature request for auto-deleting gone local branches would be a distinct issue.
