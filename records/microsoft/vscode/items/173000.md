# Issue #173000 — GitHub - Add support for `.issuetracker` file in Source Control feature

[https://github.com/microsoft/vscode/issues/173000](https://github.com/microsoft/vscode/issues/173000)

## Summary

A niche feature request to support .issuetracker/bugtraq conventions in the SCM view, with only 6 upvotes and no existing implementation or duplicate. This could be built as an extension; keeping open for the community process.

## Change summary

Add support for a .issuetracker file convention so that issue references in commit messages (e.g. ABC-123, #456) render as clickable links in the Source Control view.

## Best solution

This is a strong extension candidate — an extension could read a .issuetracker or .gitbugtraq config file and use the SCM API (or a LinkProvider over the SCM input/history) to render issue links. If the current extension API is insufficient for this, the gap should be filed as a separate API request. Keeping open for community traction, but the bar for adding this to core is high given the niche audience and the extension path.
