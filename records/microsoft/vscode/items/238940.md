Team-authored issue (bpasero) reporting that quick open overlaps window controls when zoomed in. No fix found on current main; the quick input controller lacks zoom-aware positioning relative to window controls. Keeping open per hard keep-open rule for team-authored issues.

## Change summary

Quick Open widget can overlap native window controls when the user has zoomed in multiple times.
