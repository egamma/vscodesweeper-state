# Issue #227798 — Reveal  active file in file explorer  and ctrl+shift+ e  should work in diff editor

[https://github.com/microsoft/vscode/issues/227798](https://github.com/microsoft/vscode/issues/227798)

## Summary

Still reproduces on main: when both sides of a diff editor use non-file schemes (e.g. git: URIs from GitLens ref comparison), the reveal-in-explorer command gets a URI that fails the `isInsideWorkspace` check and falls through to the open-editors view instead of mapping back to the workspace file path.

## Change summary

"Reveal Active File in File Explorer" and Ctrl+Shift+E don't reveal the file when the active editor is a diff comparing git refs (non-file-scheme URIs).

## Best solution

In the REVEAL_IN_EXPLORER handler (src/vs/workbench/contrib/files/browser/fileCommands.ts), when the resolved URI has a non-file scheme but carries a `path` that corresponds to a workspace file, attempt to map it back to the file-scheme workspace URI before checking `isInsideWorkspace`. Alternatively, use `EditorResourceAccessor.getCanonicalUri` with a `file` scheme filter as a fallback.
