# Issue #192789 — Quick pick: Support searching hangul treating last character as partially formed

[https://github.com/microsoft/vscode/issues/192789](https://github.com/microsoft/vscode/issues/192789)

## Summary

Still a real gap on main: the quick pick filter handles Hangul alternate QWERTY representations (#196203) but does not treat a standalone jamo consonant (like ㅌ) as a prefix match for syllables beginning with that consonant (like 터). The described behavior remains unimplemented.

## Change summary

Quick pick should treat the last Hangul character in a search query as potentially partially-formed, matching syllables whose initial consonant matches that jamo.

## Best solution

Extend the filter logic in `src/vs/base/common/filters.ts` (and/or `src/vs/base/common/naturalLanguage/korean.ts`) so that when the last character of the query is a standalone initial consonant (jamo in the ㄱ–ㅎ range), it matches any full Hangul syllable whose initial consonant is equivalent. This is a localized change to the existing alternate-character infrastructure.
