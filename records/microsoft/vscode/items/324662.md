# Issue #324662 — Copilot Chat history item silently fails to load when chatSessions JSONL is missing but session-store.db has transcript

[https://github.com/microsoft/vscode/issues/324662](https://github.com/microsoft/vscode/issues/324662)

## Summary

Team-authored issue describing a real UX gap: when the per-workspace JSONL session file is missing but metadata persists in workspace state, clicking the history item silently fails. The code in `chatSessionStore.ts` returns `undefined` on FILE_NOT_FOUND with no user-facing feedback. Keeping open for maintainer judgment.

## Change summary

Clicking a Chat history item silently fails when its backing `chatSessions/<sessionId>.jsonl` file is missing, even though session data exists in the Copilot extension's global `session-store.db`.

## Best solution

In `ChatSessionStore.readSessionFromLocation`, when the session file is not found and `readSession` returns `undefined`, the caller in `chatServiceImpl.ts` should either surface a user-visible notification explaining the session is unavailable, prune the stale index entry, or attempt recovery from the Copilot extension's global session store. The silent `return undefined` at line 661 in `src/vs/workbench/contrib/chat/common/model/chatSessionStore.ts` is the immediate gap.
