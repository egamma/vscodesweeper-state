# Issue #231173 — [BUG] [1.94 Regression] VS Code uses the wrong, deprecated file/folder picker on Gnome.

[https://github.com/microsoft/vscode/issues/231173](https://github.com/microsoft/vscode/issues/231173)

## Summary

This is an upstream Electron issue, already acknowledged by a VS Code team member (deepak1556). The root cause is Electron's portal version requirement for the `defaultPath` feature — until xdg-desktop-portal ships a compatible release and Electron adopts it, VS Code falls back to the older GTK dialog. VS Code is now on Electron 42; the issue persists because the fix lives in Electron/Chromium's file-dialog portal integration, not in VS Code's own code.

## Change summary

VS Code uses the deprecated GTK file chooser on GNOME instead of the native portal-based file picker, a regression since 1.94.

## Best solution

Track upstream progress on Electron's xdg-desktop-portal support (electron/electron#43819). Once Electron ships a version that lowers or adjusts the portal version requirement (or xdg-desktop-portal releases a version meeting it), upgrading Electron in VS Code should resolve this. No VS Code-side code change can fix the dialog selection — it is entirely within Electron/Chromium's `select_file_dialog_linux` implementation.
