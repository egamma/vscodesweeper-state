# Issue #282076 — NES rendering for minor changes (i.e. missing letter or parenthesis)

[https://github.com/microsoft/vscode/issues/282076](https://github.com/microsoft/vscode/issues/282076)

## Summary

Team-authored feature request about improving visual distinctiveness of NES for very minor edits (single missing characters, closing braces). Kept open per hard keep-open rule (MEMBER author) and because the UX gap remains valid.

## Change summary

NES suggestions for tiny edits (a missing letter or closing parenthesis) are too visually subtle, causing users to miss them and not realize their code has errors.

## Best solution

Explore a more prominent visual treatment for single-character or very-short NES insertions — possibly a stronger highlight, a distinct decoration, or a complementary diagnostic-style cue — so users reliably notice the suggestion rather than mistaking the ghost text for already-committed code.
