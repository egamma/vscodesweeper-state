# Issue #282076 — NES rendering for minor changes (i.e. missing letter or parenthesis)

[https://github.com/microsoft/vscode/issues/282076](https://github.com/microsoft/vscode/issues/282076)

## Summary

Team-authored feature request for improved visual rendering of minor NES suggestions (single-character insertions, missing brackets). Stays open per keep-open rules for MEMBER-authored issues; also aligns with current NES UX priorities.

## Change summary

NES suggestions for very small edits (a missing letter or closing brace) are too visually subtle, causing users to overlook them and not realize their code has errors.

## Best solution

Explore differentiated rendering for minimal NES edits — e.g. a more prominent highlight, inline marker, or combined diagnostic cue — so single-character fixes are noticeable without being intrusive. Work likely lives in the inline-completions rendering layer.
