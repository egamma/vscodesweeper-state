# Issue #282076 — NES rendering for minor changes (i.e. missing letter or parenthesis)

[https://github.com/microsoft/vscode/issues/282076](https://github.com/microsoft/vscode/issues/282076)

## Summary

Team-authored feature request for improving the visual distinctiveness of very small NES suggestions (single missing characters, closing braces). Kept open per team-member rule; the UX concern is valid and already labeled appropriately.

## Change summary

NES suggestions for tiny edits (one missing letter or closing paren) are too visually subtle, causing users to overlook them and not realize their code has an error.

## Best solution

Explore a differentiated rendering treatment for very small NES insertions — possibly stronger contrast, a subtle animation, or a distinct inline marker — so users notice them rather than mistaking the suggestion for already-present code. This lives in the inline-completions/NES-ux area.
