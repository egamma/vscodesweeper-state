# Issue #158076 — Support no-code language server integration

[https://github.com/microsoft/vscode/issues/158076](https://github.com/microsoft/vscode/issues/158076)

## Summary

Valid feature request with 24 upvotes, accepted into Backlog. No implementation exists on current main — the declarative language server contribution point has not been built. Keeping open as an accepted backlog item.

## Change summary

Allow extensions to declare language server integrations declaratively in package.json instead of writing boilerplate activation/deactivation code.

## Best solution

Implement a `contributes.languageServer` extension point that lets extensions declare server module, transport, document selectors, and file watchers in package.json, with VS Code handling client lifecycle automatically. This reduces extension size and boilerplate for the common case while still allowing programmatic clients for advanced scenarios.
