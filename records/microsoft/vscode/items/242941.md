## Summary

Valid tech-debt item: the `matchFilenameCasing` method in `anythingQuickAccess.ts` still uses `IFileService.resolve` directly to match filename casing, which is a layer concern that could be encapsulated in a lower-level service. The code remains unchanged on current main.

## Change summary

The anything picker directly resolves parent folder children via IFileService to match filename casing, breaking layer separation.
