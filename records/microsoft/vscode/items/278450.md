# Issue #278450 — GHCP Code Review does not respect Git worktree boundaries

[https://github.com/microsoft/vscode/issues/278450](https://github.com/microsoft/vscode/issues/278450)

## Summary

Confirmed in the source: when the SCM resource-group review button is clicked for a specific worktree, the commands pass a bare 'workingTree' or 'index' group string, and both `collectDiffChanges` and `CurrentChange.getCurrentChanges` iterate all `git.repositories` without filtering to the originating repository. The bug is real and still present on main.

## Change summary

The SCM review button for a specific Git worktree reviews changes across all worktrees instead of scoping to the clicked worktree's repository.

## Best solution

The `scm/resourceGroup/context` menu commands receive a `SourceControlResourceGroup` argument that carries the parent repository's `rootUri`. The review commands (`github.copilot.chat.review.stagedChanges`, `github.copilot.chat.review.unstagedChanges`) should accept this argument, thread it through `ReviewSession.review` and the `ReviewGroup` type, and filter `git.repositories` in `collectDiffChanges` / `CurrentChange.getCurrentChanges` to only the matching repository. The fix touches `extensions/copilot/src/extension/inlineChat/vscode-node/inlineChatCommands.ts` (command registration), `extensions/copilot/src/extension/review/node/doReview.ts` (ReviewGroup / review function), `extensions/copilot/src/extension/review/node/githubReviewAgent.ts` (collectDiffChanges), and `extensions/copilot/src/extension/prompts/node/feedback/currentChange.tsx` (getCurrentChanges).
