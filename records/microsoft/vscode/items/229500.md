# Issue #229500 — SCM Graph - Picked wrong repo not from active editor

[https://github.com/microsoft/vscode/issues/229500](https://github.com/microsoft/vscode/issues/229500)

## Summary

Milestone is 'On Deck' (a real scheduling milestone), so this must stay open per policy. The bug is still traceable in the current source: the `latestChangedValue` race between `firstRepository` (resolved immediately as a constObservable) and `graphRepository` (derived from the active editor) means the first repo in the workspace wins the initial read when both resolve in the same transaction.

## Change summary

In a multi-root workspace, the SCM Graph view picks the first repository instead of the one matching the active editor.

## Best solution

Fix the repository selection priority in `SCMHistoryViewModel` (scmHistoryViewPane.ts ~line 1167) so that `graphRepository` (active-editor-derived) takes precedence over `firstRepository` during the initial synchronous read. The `latestChangedValue` combinator currently lets the const-resolved first repo win the race.
