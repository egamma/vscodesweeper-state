# Issue #298255 — "Compact conversation" replaces current message

[https://github.com/microsoft/vscode/issues/298255](https://github.com/microsoft/vscode/issues/298255)

## Summary

Verified duplicate of #314664, which describes the same bug (unsent chat input lost when 'Compact Conversation' is triggered) and already has an assignee. The defect is still present on main — the action calls `widget?.acceptInput('/compact')` which overwrites the user's in-progress input.

## Change summary

Clicking 'Compact Conversation' in the Copilot Chat statistics menu replaces the user's unsent chat input with `/compact` and sends it, discarding the draft.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/314664, which is assigned and tracks the same fix: the compact action should either save/restore the user's draft or send the `/compact` command without going through the input box.
