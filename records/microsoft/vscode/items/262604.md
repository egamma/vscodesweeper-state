# Issue #262604 — Contributed Chat Participants don't work in Web Extensions

[https://github.com/microsoft/vscode/issues/262604](https://github.com/microsoft/vscode/issues/262604)

## Summary

Confirmed bug: chat participants registered by web extensions never get their request handler invoked — the chat UI hangs on 'Working...' indefinitely. A MEMBER confirmed the reproduction. No fix has been committed on main.

## Change summary

Chat participants contributed by web extensions are registered but their request handlers are never called, leaving the chat spinner stuck forever.

## Best solution

Investigate the activation/invocation path for chat participants in the web extension host. The static registration (`chatParticipantExtensionPoint`) and `ExtHostChatAgents2` are in the `common` layer and should work, but something in the activation event routing (`onChatParticipant:`) or the `$invokeAgent` proxy dispatch to the web worker extension host is broken. The fix likely lives in the chat service invocation path (`src/vs/workbench/contrib/chat/common/chatService/chatServiceImpl.ts`) or the extension host kind routing for the `MainThreadChatAgents2` customer.
