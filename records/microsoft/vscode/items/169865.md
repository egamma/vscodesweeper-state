# Issue #169865 — When zoomed out in PWA mode, the title bar gets smaller than WCO

[https://github.com/microsoft/vscode/issues/169865](https://github.com/microsoft/vscode/issues/169865)

## Summary

Team-authored issue; the underlying problem (browser zoom level unknown in web/PWA) persists on current main — the counter-zoom logic in titlebarPart.ts relies on getZoomFactor which is only set for VS Code's own zoom, not the browser's native zoom. PR #169866 was a partial workaround, not a full fix.

## Change summary

When zoomed out in PWA mode, the titlebar shrinks below the fixed-height Window Controls Overlay because VS Code cannot detect browser-level zoom.

## Best solution

Detect browser zoom (e.g. via visualViewport API or matchMedia heuristics) in the web environment and feed it into getZoomFactor so counter-zoom works correctly for WCO in PWA mode.
