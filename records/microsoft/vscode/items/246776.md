# Issue #246776 — Feature request: Reuse existing window when launching into a given profile, if profiles match

[https://github.com/microsoft/vscode/issues/246776](https://github.com/microsoft/vscode/issues/246776)

## Summary

Valid feature request with 22 upvotes, promoted to Backlog. The requested profile-aware window reuse is not yet implemented on main — the forceReuseWindow path in windowsMainService.ts does not consider profile matching. #183347 was closed as a duplicate of this issue, confirming it as canonical.

## Change summary

When launching VS Code with --profile, reuse an existing window that already has the matching profile open instead of always opening a new window.

## Best solution

Implement profile-aware window selection in the reuse-window logic of `src/vs/platform/windows/electron-main/windowsMainService.ts` — when `--profile` is passed with `--reuse-window`, prefer a window already running the specified profile. This may also involve the window finder logic and CLI argument handling.
