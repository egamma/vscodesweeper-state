# Issue #312768 — SCM lost its getting started buttons/contents

[https://github.com/microsoft/vscode/issues/312768](https://github.com/microsoft/vscode/issues/312768)

## Summary

Team contributor report with On Deck milestone — hard keep-open. The SCM welcome view buttons (Clone Repository, Open Folder) no longer appear in an empty window. The welcome content is still registered in the git extension's package.json but something prevents it from rendering.

## Change summary

SCM viewlet no longer shows getting-started buttons (Clone Repository, Open Folder, Initialize Repository) when opened in an empty VS Code window.

## Best solution

Investigate whether the 'Agents - refactor SCM contributions' commit (d0b379e4344) or the git.enabled override (ba6e70744a4) broke the context key chain that gates the welcome view content. The when-clause depends on `workbenchState == empty && git.state == initialized && scm.providerCount == 0` — one of those keys may no longer be set correctly after the refactor. Fix likely lives in `src/vs/workbench/contrib/scm/browser/scm.contribution.ts` or the git extension's activation path.
