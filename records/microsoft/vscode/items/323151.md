# Issue #323151 — Allow opening the Merge Editor for multiple selected files simultaneously

[https://github.com/microsoft/vscode/issues/323151](https://github.com/microsoft/vscode/issues/323151)

## Summary

Genuine feature request: the git.openMergeEditor command currently accepts a single URI and there is no multi-select batch-open path in the SCM context menu or command handler. No duplicate found. Routing to the merge-editor and git area owners.

## Change summary

Allow multi-selecting conflicting files in the Source Control view and batch-opening them all in Merge Editor tabs.

## Best solution

Extend `git.openMergeEditor` in `extensions/git/src/commands.ts` to accept an array of URIs (or a `...resources` rest parameter following the SCM multi-select pattern), and register a context-menu entry in the SCM resource-group that is visible when multiple merge-change items are selected. The menu contribution in `extensions/git/package.json` would need a multi-select `when` clause. This is a small, self-contained enhancement in the built-in git extension.
