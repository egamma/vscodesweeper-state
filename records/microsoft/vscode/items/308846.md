# Issue #308846 — Source Control Graph gets stuck on "All" and history reference picker becomes unclickable

[https://github.com/microsoft/vscode/issues/308846](https://github.com/microsoft/vscode/issues/308846)

## Summary

Still reproducible on main: the history reference picker action has `precondition: SCMHistoryItemCount.notEqualsTo(0)`, so when 'All' returns zero items the button is disabled and the user is stuck — no way to switch back to 'Auto'.

## Change summary

Source Control Graph's history reference picker becomes unclickable after switching to 'All' if zero history items are returned, because the action's precondition disables it when the item count is 0.

## Best solution

Remove or relax the `SCMHistoryItemCount.notEqualsTo(0)` precondition on `PICK_HISTORY_ITEM_REFS_ACTION_ID` so the picker remains clickable even when the graph is empty. The welcome-content message can still show, but the user must always be able to change the filter. Fix likely lives in `src/vs/workbench/contrib/scm/browser/scmHistoryViewPane.ts` around line 194.
