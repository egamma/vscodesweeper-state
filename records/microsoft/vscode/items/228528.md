# Issue #228528 — [json] builtin json formatter doesn't support format-on-paste

[https://github.com/microsoft/vscode/issues/228528](https://github.com/microsoft/vscode/issues/228528)

## Summary

The JSON range formatter is registered but likely cannot format arbitrary paste ranges correctly because JSON formatting requires full-document context for indentation. The issue is correctly triaged as a feature-request with help-wanted in Backlog.

## Change summary

The built-in JSON formatter does not reformat pasted content via editor.formatOnPaste, even though formatOnSave works.

## Best solution

Improve the JSON language server's range formatting to handle sub-document ranges correctly (computing indentation from context), or implement a dedicated paste-formatting path in the json-language-features extension. The fix likely lives in the JSON language server's formatting logic that handles `DocumentRangeFormattingRequest`.
