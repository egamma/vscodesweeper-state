# Issue #178334 — Weird code indentation C++

[https://github.com/microsoft/vscode/issues/178334](https://github.com/microsoft/vscode/issues/178334)

## Summary

Known auto-indentation bug where indentation is based on the previous line's indentation rather than the scope level. A team member confirmed it in May 2024 and noted tree-sitter as the intended fix path; the label `tree-sitter-autoindent-candidate` already tracks this.

## Change summary

Auto-indent in C++ (and other languages) incorrectly indents based on the previous line's whitespace rather than the logical scope, producing double-indentation after multi-line function signatures.

## Best solution

Resolve via the tree-sitter-based indentation work already planned under the `tree-sitter-autoindent-candidate` label, which would make the indentation engine scope-aware rather than line-above-based.
