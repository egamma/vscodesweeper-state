# Issue #326041 — AH: agent edit does not show in a dirty file in the editor

[https://github.com/microsoft/vscode/issues/326041](https://github.com/microsoft/vscode/issues/326041)

## Summary

Team-authored bug (MEMBER): agent edits to dirty or untitled files are written to disk but not reflected in the editor buffer — keep open for the owning team to address.

## Change summary

When a file has unsaved changes in the editor, agent-host edits land on disk but the editor buffer is not updated; for untitled files the edit is lost entirely.

## Best solution

The agent-host changeset/edit path needs to detect dirty editor buffers and apply the edit through the editor's text model (or trigger a revert-to-disk / merge) rather than writing only to the filesystem. For untitled files, the edit must go through the in-memory model since there is no disk backing. Likely involves `src/vs/platform/agentHost/node/agentHostChangesetService.ts` or `agentHostChangesetCoordinator.ts` coordinating with the workbench's `ITextFileService` / `IEditorService`.
