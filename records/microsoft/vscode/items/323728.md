# Issue #323728 — slash command blocks input - Copilot Chat with Agent Host, Claude Agent SDK

[https://github.com/microsoft/vscode/issues/323728](https://github.com/microsoft/vscode/issues/323728)

## Summary

Confirmed bug: slash commands from the agent host fail because `resolvePromptSlashCommand` calls `parseNew` on a `vscode-agent-host://` URI that the file service cannot resolve. A team member has already reproduced and engaged.

## Change summary

Sending a bare slash command (e.g. `/code-review`) in Copilot Chat with the Agent Host silently fails because the prompt service tries to read the agent-host URI as a file and throws.

## Best solution

The prompt resolution path in `promptsServiceImpl.ts#resolvePromptSlashCommand` (or the `parseNew` call it delegates to) needs to handle agent-host-sourced slash commands that don't back onto a readable file URI — either by skipping file read for those schemes or by registering a file-system provider that serves the prompt content for `vscode-agent-host` URIs.
