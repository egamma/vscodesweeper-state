# Issue #263288 — Chat: Edited message disapears from chat

[https://github.com/microsoft/vscode/issues/263288](https://github.com/microsoft/vscode/issues/263288)

## Summary

Still open on main — the editing cancellation flow in `finishedEditing()` does not properly handle the case where the edited message has scrolled out of the virtualizing list's viewport. The team member (justschen) acknowledged the bug and indicated they would investigate.

## Change summary

Edited chat message disappears from the list when the user tabs out of the inline edit input (scrolling it out of view) and then cancels editing.

## Best solution

Fix the `finishedEditing()` method in `chatWidget.ts` to ensure the inline input container is properly detached and the edited item is correctly restored in the virtualizing tree after the editing state is cleared, even when the item has been recycled due to scrolling out of the viewport. The `getTemplateDataForRequestId` call at line ~1861 may return undefined for recycled items, and the subsequent DOM cleanup and tree refresh need to handle that gracefully.
