# Issue #325227 — Fish Shell Tab Completion Not Work

[https://github.com/microsoft/vscode/issues/325227](https://github.com/microsoft/vscode/issues/325227)

## Summary

Fish shell tab completion reportedly broken when `editor.accessibilitySupport` is set to `on`, which enables xterm.js `screenReaderMode`. The reporter self-diagnosed the setting as the root cause. Zsh tab completion is unaffected. Needs routing to the terminal area for investigation — the interaction between xterm's screen reader mode and fish's completion protocol may need a fix in VS Code's terminal layer or upstream in xterm.js.

## Change summary

Fish shell tab completion stops working in the integrated terminal when `editor.accessibilitySupport` is set to `on`.

## Best solution

The terminal team should investigate how xterm.js `screenReaderMode` (set in `src/vs/workbench/contrib/terminal/browser/terminalInstance.ts:1988`) interferes with fish shell's tab completion. If the issue is in xterm.js's handling of Tab key events in screen reader mode, it may need an upstream fix. If VS Code's terminal layer can work around it (e.g. ensuring Tab is still forwarded to the PTY in screen reader mode for fish), a local fix may suffice.
