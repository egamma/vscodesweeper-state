# Issue #306568 — runSubagent ignores "agents" config by omitting "agentName" parameter

[https://github.com/microsoft/vscode/issues/306568](https://github.com/microsoft/vscode/issues/306568)

## Summary

Valid feature request to make `agentName` always required (or conditionally required when `agents` config restricts subagent choices) in the `runSubagent` tool schema, so models that ignore optional parameters cannot bypass agent restrictions.

## Change summary

When a model omits the optional `agentName` parameter in `runSubagent`, the subagent inherits the parent's agent profile, potentially bypassing `disable-model-invocation: true` restrictions set in `.agent.md`.

## Best solution

Make `agentName` required in the tool's input schema whenever the parent agent has an `agents` config restricting allowed subagents. The logic in `runSubagentTool.ts` (lines 123-126) already conditionally requires it for `generalPurposeAgentEnabled`; extending that to also require it when the invoking agent has restricted subagent lists would close the gap.
