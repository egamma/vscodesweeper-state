# Issue #156630 — Can the inputbox show a red outline for invalid filenames?

[https://github.com/microsoft/vscode/issues/156630](https://github.com/microsoft/vscode/issues/156630)

## Summary

Team-authored polish request that remains unimplemented on main. The "New File..." quick pick (`newFile.contribution.ts`) does not validate the entered filename — it creates a "Create New File (…)" entry and lets the user accept it without checking for invalid characters or path segments.

## Change summary

The "New File..." input box should show a red outline with an error message when the user types an invalid filename, and should prevent acceptance while invalid.

## Best solution

Add filename validation to the `onDidChangeValue` handler in `src/vs/workbench/contrib/welcomeViews/common/newFile.contribution.ts`, reusing the existing `validateFileName` utility from `src/vs/workbench/contrib/files/browser/fileActions.ts`. When invalid, show the quick pick's validation message (red outline) and suppress the "Create New File" entry or disable acceptance.
