# Issue #78004 — [folding] When you cut a collapsed code block it should stays collapsed when you paste/drop it.

[https://github.com/microsoft/vscode/issues/78004](https://github.com/microsoft/vscode/issues/78004)

## Summary

Accepted feature request in the Backlog — preserving collapsed state across cut/paste has not been implemented on main. No related code or commits found in the folding contribution.

## Change summary

When cutting and pasting a collapsed code block, the pasted block should remain collapsed instead of expanding.

## Best solution

Implement folding state preservation in the paste/drop handler within `src/vs/editor/contrib/folding/`. The folding ranges of the cut region would need to be captured before the cut and re-applied after the paste at the new offset. This is a non-trivial editor feature that requires design consideration around edge cases (partial folds, nested folds, cross-file paste).
