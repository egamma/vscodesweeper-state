# Issue #294883 — Background agent: Worktree

[https://github.com/microsoft/vscode/issues/294883](https://github.com/microsoft/vscode/issues/294883)

## Summary

Valid request to make the background agent's worktree location configurable or place it at the same directory depth as the original repo, so that relative paths to sibling repositories are preserved. The current `getCopilotWorktreesRoot` in `copilotAgent.ts` places worktrees under `<repo>.worktrees/`, adding an extra directory level that breaks cross-repo relative references.

## Change summary

Background agent worktree creation breaks relative project references to sibling repositories because the worktree is nested one level deeper than the original repo.

## Best solution

Allow users to configure the worktree base path (or default to a sibling-level layout like `<parent>/.worktrees/<repo-name>/`) so that relative paths from the worktree to neighboring repositories resolve the same way as from the original repo. The key function is `getCopilotWorktreesRoot` in `src/vs/platform/agentHost/node/copilot/copilotAgent.ts`. Issue https://github.com/microsoft/vscode/issues/277754 tracks a related worktree-location problem (permissions in devcontainers) and a unified configurable-location solution could address both.
