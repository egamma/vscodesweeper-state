# Issue #183852 — New File: `alt+crtl+N` does not show that it requires `super` 

[https://github.com/microsoft/vscode/issues/183852](https://github.com/microsoft/vscode/issues/183852)

## Summary

Team-authored issue (hard keep-open). The bug is in Electron's native menu accelerator rendering on Linux — VS Code's ElectronAcceleratorLabelProvider correctly emits 'Super' for the meta key, but Electron's native menu strips it. Confirmed by sbatten as an Electron menus bug.

## Change summary

Native menus on Linux do not display the Super/Meta modifier key in keybinding accelerator labels, while custom menus and Windows native menus display it correctly.

## Best solution

This is an upstream Electron bug — Electron's native menu implementation on Linux (GTK) drops the Super/Meta modifier from accelerator labels. The fix belongs in Electron's menu accelerator rendering. VS Code's code in `src/vs/base/common/keybindingLabels.ts` already correctly maps metaKey to 'Super' for the Electron accelerator label provider. A workaround within VS Code would be to avoid passing Meta-containing accelerators to Electron's native menus on Linux, but the proper fix is upstream.
