# Issue #265654 — Crash in renderer when capturing performance profile with enhanced tracing enabled

[https://github.com/microsoft/vscode/issues/265654](https://github.com/microsoft/vscode/issues/265654)

## Summary

Team-authored upstream V8/Chromium crash bug with a confirmed reproduction and a Chromium bug filed (https://issues.chromium.org/issues/377497451). VS Code landed a workaround (disabling renderer profiling) but the root cause lives in V8's TraceScriptRundown. Keeping open as the upstream tracker — hard keep-open applies (MEMBER author).

## Change summary

The renderer crashes in V8's TraceScriptRundown when capturing a CPU performance profile because a native script lacks an execution context.

## Best solution

The upstream V8 fix (null-context guard in TraceScriptRundown) needs to land in Chromium and flow into the Electron version VS Code ships. Once the upstream fix ships, the rendererProfiling feature can be re-enabled. Track the Chromium bug at https://issues.chromium.org/issues/377497451.
