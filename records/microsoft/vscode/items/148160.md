# Issue #148160 — extension.extensionKind is wrong for extensions not running on the host

[https://github.com/microsoft/vscode/issues/148160](https://github.com/microsoft/vscode/issues/148160)

## Summary

Still reproduces on main: the TODO@alexdima comments at lines 611 and 629 of extHost.api.impl.ts confirm the extensionKind is hardcoded to the host's own kind and never flipped for foreign extensions. Team-authored with milestone On Deck — keeping open.

## Change summary

extension.extensionKind returns Workspace for all extensions in allAcrossExtensionHosts, even those running on the local/UI host.

## Best solution

When constructing Extension objects for foreign (different-host) extensions, derive extensionKind from whether the foreign extension's host is remote or local, rather than reusing the caller's own extensionKind. The fix is localized to src/vs/workbench/api/common/extHost.api.impl.ts around lines 611 and 629.
