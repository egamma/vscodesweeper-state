# Issue #250764 — onDidEndTerminalShellExecution not received for commands that contain newlines

[https://github.com/microsoft/vscode/issues/250764](https://github.com/microsoft/vscode/issues/250764)

## Summary

Still reproduces on main: the bash shell integration script's `633;D` completion marker relies on PROMPT_COMMAND firing after command execution, but multi-line commands sent programmatically via executeCommand fail to trigger it. No fix has landed; related issues #316556 and #324392 are also open.

## Change summary

The `onDidEndTerminalShellExecution` event and `executeCommand().read()` stream never complete for commands containing newlines in bash.

## Best solution

Fix the bash shell integration script (`src/vs/workbench/contrib/terminal/common/scripts/shellIntegration-bash.sh`) or the `executeCommand` API path to ensure the `]633;D` marker is emitted after multi-line command execution completes. This likely requires changes to how multi-line input is handled in the preexec/precmd hooks, or how the API sends multi-line text to the terminal. Related issues #316556 and #324392 track the same root cause from different angles.
