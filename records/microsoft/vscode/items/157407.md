# Issue #157407 — Neo2 Layout Keybinding wrong key used

[https://github.com/microsoft/vscode/issues/157407](https://github.com/microsoft/vscode/issues/157407)

## Summary

Root cause is in Chromium's keyboard event reporting: it emits the same keyCode (88) for both 'ö' and 'x' on the Neo2 layout under Linux, making them indistinguishable at the keyCode level. This is an upstream limitation VS Code cannot fix without Chromium changes.

## Change summary

On Linux with Neo2 layout and keyboard.dispatch=keyCode, the umlaut 'ö' key is reported as 'x' because Chromium assigns the same keyCode to both.

## Best solution

Track as upstream (Chromium/Electron keyboard event limitation for non-standard layouts on Linux). A fix would require Chromium to correctly differentiate key events for remapped layouts, or VS Code to implement a workaround using the `key` field when `keyCode` collides — but that would need careful design work in the keybinding dispatch layer (`src/vs/workbench/services/keybinding/`).
