# Issue #326185 — http.noProxy config not work for agent host when request BYOK model

[https://github.com/microsoft/vscode/issues/326185](https://github.com/microsoft/vscode/issues/326185)

## Summary

Real bug: when the agent host injects HTTP_PROXY/HTTPS_PROXY into the Copilot SDK subprocess env, it does not also inject NO_PROXY from the user's `http.noProxy` setting, so BYOK model requests to localhost/127.0.0.1 are incorrectly routed through the corporate proxy.

## Change summary

Requests from the agent host to a BYOK model at 127.0.0.1 ignore the `http.noProxy` setting and are sent through the configured proxy.

## Best solution

In `_configureProxyEnv` (src/vs/platform/agentHost/node/copilot/copilotAgent.ts), when setting HTTP_PROXY/HTTPS_PROXY, also set the `NO_PROXY` env var from the `http.noProxy` configuration value so the SDK subprocess respects the same bypass list.
