## Summary

Community-promoted feature request (115 upvotes) in the Backlog milestone, requesting a `code devcontainer` CLI subcommand. No implementation exists on main. The earlier related issue #111230 was closed by the community process for lack of upvotes — this newer issue succeeded and should remain open.

## Change summary

Add a `code devcontainer up` CLI subcommand to open a project directly in a dev container without the intermediate WSL/local window step.

## Best solution

Implement a `code devcontainer` subcommand in the VS Code CLI (likely in the Rust CLI under `cli/`) that orchestrates devcontainer startup and connects VS Code directly, skipping the manual reopen-in-container flow. This would coordinate with the Dev Containers extension and the existing devcontainers/cli. The scope spans the CLI, the remote connection bootstrapping, and the Dev Containers extension — likely needs input from the remote-development team.
