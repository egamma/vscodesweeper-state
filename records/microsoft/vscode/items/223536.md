# Issue #223536 — Ability to set log levels by output channel

[https://github.com/microsoft/vscode/issues/223536](https://github.com/microsoft/vscode/issues/223536)

## Summary

Valid feature request with 34 upvotes (above the 20-vote threshold) in the Backlog milestone. The API gap remains on current main — LogOutputChannel.logLevel is read-only with no programmatic setter, and createOutputChannel has no log-level parameter.

## Change summary

Allow extensions to programmatically set the log level of individual LogOutputChannels via the VS Code API.

## Best solution

Add a `setLogLevel(level: LogLevel)` method to `LogOutputChannel` and optionally accept an initial log level in `createOutputChannel` options. This belongs in the extension API surface (src/vscode-dts/vscode.d.ts) with implementation in the output channel service.
