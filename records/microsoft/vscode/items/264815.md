# Issue #264815 — Find: “Search wraps” off jumps to previous match and gives no accessible feedback when Loop set to false

[https://github.com/microsoft/vscode/issues/264815](https://github.com/microsoft/vscode/issues/264815)

## Summary

Confirmed source-reproducible on current main: the non-loop boundary guards in _moveToNextMatch and _moveToPrevMatch intentionally refocus in the wrong direction instead of staying put. An open PR #308019 exists but has not been merged.

## Change summary

When editor.find.loop is disabled, pressing F3 past the last match jumps backward (and Shift+F3 before the first match jumps forward) instead of staying in place, with no accessible feedback.

## Best solution

Remove the 'refocus' logic in the `!canNavigateForward()` and `!canNavigateBack()` guards in `src/vs/editor/contrib/find/browser/findModel.ts` so the cursor stays put, and add an accessible announcement (via AccessibilitySignal or aria alert) indicating no further results. PR https://github.com/microsoft/vscode/pull/308019 addresses the navigation part; the accessibility announcement may still need work.
