# Issue #316651 — Machine scoped configuration properties do not respect the correct scope

[https://github.com/microsoft/vscode/issues/316651](https://github.com/microsoft/vscode/issues/316651)

## Summary

Legitimate bug report about machine-scoped configuration not being filtered when reading from the extension host on a remote. A team member (sandy081) is actively investigating and the reporter provided a sample extension. Keep open for the ongoing investigation.

## Change summary

Machine-scoped extension configuration properties leak their local user value to the remote extension host when the property isn't set on the remote.

## Best solution

The root cause likely involves timing: when the local user settings are parsed before the extension's configuration contribution is registered, the scope is unknown and the `shouldInclude` filter in `configurationModels.ts` allows it through (line 477: `scope === undefined` returns true). The fix belongs in `src/vs/platform/configuration/common/configurationModels.ts` or the ext host forwarding in `src/vs/workbench/api/common/extHostConfiguration.ts`. sandy081 is the right owner; the sample repro extension has been provided.
