# Issue #322078 — Ctrl-Clicking a link in the terminal opens the file path on the line above instead

[https://github.com/microsoft/vscode/issues/322078](https://github.com/microsoft/vscode/issues/322078)

## Summary

Still reproduces on main: the TerminalMultiLineLinkDetector's line-number regex matches timestamps (e.g. `10:44:38`) as line:col, then resolves an existing file path from the line above, stealing priority from the URI link detector that would correctly match the URL on the same line.

## Change summary

Ctrl-clicking a URL in the terminal opens a file path from the previous line when that line contains an existing file and the current line starts with a timestamp matching the `line:col` regex.

## Best solution

Tighten the `lineNumberPrefixMatchers` regex in `terminalMultiLineLinkDetector.ts` to avoid matching timestamps — e.g. require end-of-string or whitespace after the match, or reject patterns where a second colon follows (like `10:44:38`). Alternatively, when a URI link is also detected on the same line, prefer it over a multiline file-path link.
