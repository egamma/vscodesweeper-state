# Issue #200259 — Widget navigation steals editor focus

[https://github.com/microsoft/vscode/issues/200259](https://github.com/microsoft/vscode/issues/200259)

## Summary

The bug remains unfixed on main — only diagnostic logging was added (PR #209357). The `navigableContainerFocused` context key can get stuck as `true` when focus returns to the editor, causing `widgetNavigation.focusNext` to steal cmd+down. No reliable reproduction steps exist yet, and no fix commit has landed.

## Change summary

The `navigableContainerFocused` context key sometimes stays true after focus leaves a navigable container, causing cmd+up/down to invoke widgetNavigation instead of editor commands.

## Best solution

The focus-tracking logic in `src/vs/workbench/browser/actions/widgetNavigationCommands.ts` (handleFocusEventsGroup) likely has a race where blur events are missed or arrive out of order. A fix should ensure the context key is reliably reset when DOM focus moves to the editor. The debug logging setting (`workbench.navigibleContainer.enableDebug`) is in place to collect evidence from affected users — someone needs to reproduce with logging enabled and trace the stale state.
