# Issue #286822 — GitHub Copilot more likely to summarize converation when switching to Agent mode

[https://github.com/microsoft/vscode/issues/286822](https://github.com/microsoft/vscode/issues/286822)

## Summary

Valid report: switching to Agent mode injects tool definitions and heavier system prompts, pushing the context-window fill ratio past the ~0.78–0.82 summarization threshold in `shouldKickOffBackgroundSummarization`, even when the conversation itself hasn't grown. This is a real UX problem — summarizing at the exact moment the user wants to implement a plan — but the mechanism is working as coded. Routing to the chat-summarization area for a timing/strategy improvement.

## Change summary

Switching from a non-agent mode (e.g. a custom 'Discuss' agent) to Agent mode frequently triggers conversation summarization immediately, losing planning context right when the user needs it most for implementation.

## Best solution

The background summarizer's trigger in `extensions/copilot/src/extension/prompts/node/agent/backgroundSummarizer.ts` fires based purely on context-ratio thresholds (~0.78–0.90). When switching modes changes the token budget allocation (Agent mode adds tool definitions), the ratio spikes and triggers compaction. A mode-switch-aware heuristic — e.g. deferring compaction for one turn after a mode switch, or accounting for the delta caused by tool-definition injection rather than treating it as conversation growth — would address the reported behavior. This overlaps with the broader summarization-strategy work tracked in https://github.com/microsoft/vscode/issues/283446 and https://github.com/microsoft/vscode/issues/289220.
