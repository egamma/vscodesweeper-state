# Issue #322943 — PHP Nowdoc syntax misinterpreted when labels contain UTF8

[https://github.com/microsoft/vscode/issues/322943](https://github.com/microsoft/vscode/issues/322943)

## Summary

Confirmed bug in the PHP TextMate grammar: the nowdoc begin pattern only matches ASCII label characters, so UTF-8 labels cause the nowdoc block to never close, leaking its scope into subsequent code. An open PR (#323831) is already addressing it.

## Change summary

PHP nowdoc syntax highlighting breaks when the delimiter label contains non-ASCII (UTF-8) characters, causing subsequent code to appear commented out.

## Best solution

Land the open PR https://github.com/microsoft/vscode/pull/323831 (or pull the upstream fix from https://github.com/KapitanOczywisty/language-php/pull/53) to widen the nowdoc label regex to accept Unicode identifiers, matching the heredoc pattern's behavior.
