# Issue #304516 — 'GitHub Copilot Suggestions' webview does not have a global keyboard listener that captures every key press

[https://github.com/microsoft/vscode/issues/304516](https://github.com/microsoft/vscode/issues/304516)

## Summary

The report describes Alt+key mnemonics firing alongside custom keybindings when a webview panel is focused — a keyboard double-dispatch issue in webviews. This is closely related to existing confirmed issues about webview keyboard handling inconsistencies. Routing to webview and keybindings areas for triage.

## Change summary

When the Copilot Suggestions webview panel is focused, Alt+H triggers both the Help menu mnemonic and the user's custom keybinding, causing dual behavior.

## Best solution

This likely belongs in the same fix as the general webview keyboard dispatch coordination tracked in https://github.com/microsoft/vscode/issues/241801 and the Alt-mnemonic handling tracked in https://github.com/microsoft/vscode/issues/121379 (On Deck). A maintainer should decide whether the fix belongs in core webview keyboard infrastructure or whether the Copilot extension's webview should install its own keyboard handlers.
