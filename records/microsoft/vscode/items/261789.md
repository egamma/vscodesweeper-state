# Issue #261789 — keyword suggestion message overrides search results count

[https://github.com/microsoft/vscode/issues/261789](https://github.com/microsoft/vscode/issues/261789)

## Summary

Confirmed on current main: the keyword suggestion UI clears the search result count message and prevents it from being re-rendered. An open PR (#307994) addresses this — keeping the issue open until that PR lands.

## Change summary

When search keyword suggestions appear, the 'X results in Y files' message is replaced instead of being shown alongside the suggestions.

## Best solution

Land (or close) the open PR https://github.com/microsoft/vscode/pull/307994, which removes the early return in `updateSearchResultCount()` and appends keyword suggestions alongside the result count instead of replacing it. The fix is localized to `searchView.ts`.

## Evidence

- **early return on main:** src/vs/workbench/contrib/search/browser/searchView.ts line 2058-2060: `updateSearchResultCount()` returns immediately when `_cachedKeywords.length > 0`, preventing the result count from ever rendering alongside keywords.
- **clearMessage call:** src/vs/workbench/contrib/search/browser/searchView.ts line 2142: `updateKeywordSuggestionUI()` calls `this.clearMessage()` on the first keyword, wiping the existing result count message before rendering the suggestion.
- **open fixing PR:** https://github.com/microsoft/vscode/pull/307994 is open and targets this exact issue with `Fixes #261789` — it removes the early return and appends keywords to the existing message element.
- **community confirmation:** A second user confirmed the issue on VS Code 1.105.1 (Windows), noting it diminishes the usefulness of the `search.searchView.keywordSuggestions` setting.
