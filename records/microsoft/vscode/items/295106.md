# Issue #295106 — For the extended LanguageModelChatProvider, context calculations are never precise.

[https://github.com/microsoft/vscode/issues/295106](https://github.com/microsoft/vscode/issues/295106)

## Summary

This request for accurate token/context tracking via response-level usage data (instead of the current provideTokenCount approach) is a less specific formulation of the problem already tracked in detail by #314722, which names the exact root cause and proposes a concrete fix. Proposing duplicate close to consolidate discussion.

## Change summary

Extension-contributed LanguageModelChatProvider models have inaccurate context/token tracking because provideTokenCount is imprecise and there is no way to report actual usage from the response.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/314722, which identifies the exact hardcoded-zero root cause in ExtensionContributedChatEndpoint and proposes a backward-compatible fix via a usage mime type on LanguageModelDataPart. The author's suggestion here (return usage from provideLanguageModelChatResponse) aligns with the same end goal.
