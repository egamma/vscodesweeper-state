# Issue #175097 — Task beginsPattern does not work

[https://github.com/microsoft/vscode/issues/175097](https://github.com/microsoft/vscode/issues/175097)

## Summary

Team-member issue (hard keep-open). The bug appears to still exist on main: ConPTY's cursor-addressed output causes the LineDataEventAddon's CSI H handler to fire line data events before the full line content ('Starting up http-server') lands in a single buffer row, so the beginsPattern regex never matches a complete line.

## Change summary

Background task `beginsPattern` fails to match on Windows because ConPTY cursor positioning splits expected text across multiple line-data events.

## Best solution

The LineDataEventAddon's Windows CSI H handler (lineDataEventAddon.ts:60-64) fires line data on every cursor move, which can emit partial lines. A debounce or buffering strategy that coalesces rapid cursor-move events before matching against beginsPattern could fix this. Alternatively, the problem collector could accumulate recent lines and re-test patterns against a sliding window.
