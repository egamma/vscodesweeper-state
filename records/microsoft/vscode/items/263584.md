## Change summary

The merge editor fails to handle add/add conflicts (files added in both branches) — it marks the entire file as conflicting and the 'Complete Merge' button never appears.

## Best solution

Fix the merge editor's conflict detection logic to properly handle the case where a file exists in neither base (empty base) but is added in both input branches. The diff computation likely needs to handle empty-base scenarios so that only the actual differing lines are marked as conflicts rather than the entire file content. Key area: `src/vs/workbench/contrib/mergeEditor/browser/model/`.
