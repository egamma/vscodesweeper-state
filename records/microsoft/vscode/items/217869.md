# Issue #217869 — Override less specific code action with more specific code action

[https://github.com/microsoft/vscode/issues/217869](https://github.com/microsoft/vscode/issues/217869)

## Summary

Team-authored issue (MEMBER), must keep open. The bug is confirmed by a second team member and still present on main: the exclude mechanism in `createCodeActionsOnSave` filters by action kind strings, but when a provider returns actions with the generic `source.fixAll` kind rather than a provider-specific sub-kind like `source.fixAll.eslint`, the `never` override cannot distinguish them from other fixAll actions.

## Change summary

Setting `source.fixAll.eslint: "never"` alongside `source.fixAll: "always"` does not prevent eslint-provided fix-all actions from running on save, because the specificity override is not honored.

## Best solution

The `CodeActionOnSaveParticipant` in `src/vs/workbench/contrib/codeEditor/browser/saveParticipants.ts` needs to implement proper specificity resolution: when a more-specific key (e.g. `source.fixAll.eslint`) is set to `never`, it should override a less-specific key (`source.fixAll`) set to `always`, regardless of the action kind string the provider actually returns. This likely requires changes in both `createCodeActionsOnSave` and `filteredSaveList` construction to treat `never` sub-keys as authoritative overrides.
