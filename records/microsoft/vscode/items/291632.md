# Issue #291632 — Right-Click > "Copy" on image in Copilot Chat copies the entire history instead of just the screenshot

[https://github.com/microsoft/vscode/issues/291632](https://github.com/microsoft/vscode/issues/291632)

## Summary

Team-authored bug — right-clicking an image in Copilot Chat and selecting "Copy" invokes the CopyItemAction which calls `stringifyItem()` to copy the full response text, rather than copying the image binary to the clipboard. Keeping open per team-author policy.

## Change summary

Right-clicking and copying an image shown in Copilot Chat copies the entire chat item text instead of copying just the image.

## Best solution

The CopyItemAction in `chatCopyActions.ts` should detect when the right-click target is an image element and use `IClipboardService` to write the image data (or the image's data URI/blob) to the clipboard instead of falling through to `stringifyItem()`. Alternatively, a separate "Copy Image" context menu action could be registered for image elements in the chat, gated on a context key like `chatImageElement`.
