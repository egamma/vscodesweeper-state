# Issue #325537 — sub-comand code serve-web fail

[https://github.com/microsoft/vscode/issues/325537](https://github.com/microsoft/vscode/issues/325537)

## Summary

Confirmed in current source: `new_script_command` on Windows passes the executable path unquoted to `cmd /C`, so paths with spaces (e.g. user profile directories) are truncated. Still present on main.

## Change summary

`code serve-web` fails on Windows when the user profile path contains a space, because the CLI's `cmd /C` invocation does not quote the script path.

## Best solution

In `cli/src/util/command.rs`, the Windows `new_script_command` should wrap the script argument in double quotes before passing it to `cmd /C` — e.g. `cmd.arg(format!("\"{}\"" , script.as_ref().to_string_lossy()))` or use `cmd.raw_arg` with proper quoting. This also affects `code_server.rs` and `agent_host.rs` which use the same helper.
