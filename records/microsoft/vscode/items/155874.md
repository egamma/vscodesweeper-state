# Issue #155874 — File Decorations for Search Folders 

[https://github.com/microsoft/vscode/issues/155874](https://github.com/microsoft/vscode/issues/155874)

## Summary

Team-authored feature request still open on main — folder entries in search results still don't display file decorations. The FolderMatchRenderer does not pass fileDecorations to setFile, unlike the FileMatchRenderer. Keeping open per hard keep-open rule (MEMBER author).

## Change summary

Add file decorations (colors/badges) to folder entries in search results that reflect the decorations of the contained search result files.

## Best solution

In `FolderMatchRenderer.renderElement` and `renderCompressedElements`, pass `fileDecorations` to `setFile` — but with a custom decoration provider that aggregates only the decorations of files visible in the search results under that folder, rather than all files in the directory. The design question (comment 2) about how to merge multiple child decorations into one folder decoration needs resolution first.
