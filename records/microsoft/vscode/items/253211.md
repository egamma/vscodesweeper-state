# Issue #253211 — Support Instruction Breakpoints in the VS Code API

[https://github.com/microsoft/vscode/issues/253211](https://github.com/microsoft/vscode/issues/253211)

## Summary

Valid feature request — InstructionBreakpoint is not exposed in the VS Code extension API (stable or proposed). The internal model exists but has no public counterpart. Keeping open as it's a real gap for debug adapter extension authors.

## Change summary

Expose InstructionBreakpoint in the VS Code extension API so extensions can programmatically add, remove, and observe instruction breakpoints.

## Best solution

Add an `InstructionBreakpoint` class extending `Breakpoint` to the extension API (similar to `SourceBreakpoint` and `FunctionBreakpoint`), wire it through extHostDebugService, and include it in the `breakpoints` array and change events. Could be bundled with the DataBreakpoint API work in PR #226735 or done separately.
