# Issue #312407 — renaming a forked chat will rename the original

[https://github.com/microsoft/vscode/issues/312407](https://github.com/microsoft/vscode/issues/312407)

## Summary

Team-authored bug (MEMBER) — must keep open. The forked chat rename appears to target the original session's resource rather than the fork's, causing the original chat's title to change. No fix found on current main.

## Change summary

Renaming a forked chat session renames the original chat instead of the fork.

## Best solution

The rename flow (likely in `chatViewTitleControl.ts` or the view pane title actions) needs to resolve the correct session resource for the currently-displayed forked model rather than the source session. The fork action in `chatForkActions.ts` correctly generates a new `sessionId`, so the bug is likely in how the title-rename UI resolves which model to rename — it may be holding a stale reference to the source session's resource. A secondary issue noted by the author is that the pencil/rename, trash, and archive icons disappeared from the title widget entirely.
