# Issue #254424 — Mousewheel Zoom is extremely slow on zoomLevel 1

[https://github.com/microsoft/vscode/issues/254424](https://github.com/microsoft/vscode/issues/254424)

## Summary

Still reproduces on main: at window.zoomLevel=1 (pageZoomFactor=1.2), the MouseWheelClassifier misidentifies physical mouse wheels as trackpads because the zoom-factor-scaled deltas fail the _isAlmostInt check, routing to the slow trackpad zoom path (accumulated delta / 5). The epsilon fix in e9df78f7473 was insufficient.

## Change summary

Ctrl+mousewheel editor zoom is extremely slow when window.zoomLevel is set to 1, because the MouseWheelClassifier misclassifies the physical mouse wheel as a trackpad.

## Best solution

Fix the _isAlmostInt tolerance or adjust the pageZoomFactor compensation in MouseWheelClassifier.acceptStandardWheelEvent so that integer deltas scaled by common zoom factors (like 1.2) are still recognized as physical mouse wheel inputs. The fix belongs in src/vs/base/browser/ui/scrollbar/scrollableElement.ts.
