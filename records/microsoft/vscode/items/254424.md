# Issue #254424 — Mousewheel Zoom is extremely slow on zoomLevel 1

[https://github.com/microsoft/vscode/issues/254424](https://github.com/microsoft/vscode/issues/254424)

## Summary

Already fixed and shipped. The mousewheel classifier's `_isAlmostInt` tolerance was too tight, causing the physical-mouse-wheel detection to fail when `window.zoomLevel = 1` because the zoom-factor compensation produced values just barely off from integers due to floating-point math.

## Change summary

Editor CTRL+mousewheel zoom is extremely slow when `window.zoomLevel` is set to 1, requiring many scroll notches to change the font size.

## Best solution

Keep the shipped fix in commit e9df78f7473 (VS Code 1.103.0). No further change needed — updating to 1.103+ resolves it.
