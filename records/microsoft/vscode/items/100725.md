# Issue #100725 — Git - git sync issue on an empty repo

[https://github.com/microsoft/vscode/issues/100725](https://github.com/microsoft/vscode/issues/100725)

## Summary

Still reproduces on current main: the sync button in the status bar requires HEAD.commit to be truthy (statusbar.ts:241), so on a freshly-cloned empty repo with new remote commits, the button is unclickable and invoking sync manually hits a rev-parse failure path. PR #114163 was opened in Jan 2021 but never merged.

## Change summary

Git sync button is non-functional on repositories cloned empty, even after remote commits become available.

## Best solution

Land a fix that (1) allows the status bar sync button to show a command even when HEAD.commit is undefined but an upstream exists, and (2) handles the rev-parse failure on empty repos gracefully in the sync flow. The open PR https://github.com/microsoft/vscode/pull/114163 attempted this but has gone stale — either revive it or implement a fresh fix in extensions/git/src/statusbar.ts and extensions/git/src/git.ts.
