# Issue #108954 — Matching bracket decoration and :after decoration clash

[https://github.com/microsoft/vscode/issues/108954](https://github.com/microsoft/vscode/issues/108954)

## Summary

Keep open: this is a concrete editor rendering clash with a focused Extension API reproduction, and I did not find a confirmed duplicate or fixing commit that resolves the `after` decoration interaction.

## Change summary

An extension `after` decoration placed next to a matching bracket can be visually included in the bracket-match highlight.

## Best solution

Keep this routed to editor rendering and either add a focused rendering regression test for bracket-match decorations adjacent to `afterContentClassName`/injected text, or have the editor rendering owner decide whether bracket-match styling should exclude attachment content.
