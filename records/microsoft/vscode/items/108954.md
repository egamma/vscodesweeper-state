# Issue #108954 — Matching bracket decoration and :after decoration clash

[https://github.com/microsoft/vscode/issues/108954](https://github.com/microsoft/vscode/issues/108954)

## Summary

Still reproducible on main: the bracket-match `className` decoration applies a background that visually extends over `::after` pseudo-element decorations from extensions, because the rendering layer doesn't isolate inline after-content from whole-range background decorations.

## Change summary

Bracket matching highlight background bleeds into extension-created `::after` text decorations adjacent to a bracket.

## Best solution

The bracket-match decoration's `className` background covers the full decoration node including after-content. A fix would need to either scope the background more tightly (e.g. use only `inlineClassName` styling that doesn't bleed into pseudo-elements) or adjust the decoration rendering so `::after` content from other decorations is not visually covered by a sibling decoration's background. Likely files: `src/vs/editor/contrib/bracketMatching/browser/bracketMatching.css` and possibly the view-line decoration rendering in `src/vs/editor/browser/viewParts/decorations/`.
