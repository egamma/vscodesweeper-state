# Issue #235512 — Characters go into each other with fontLigatures and RTL characters.

[https://github.com/microsoft/vscode/issues/235512](https://github.com/microsoft/vscode/issues/235512)

## Summary

Still present on main: when a line contains both RTL characters and ligature-eligible sequences (like `===`), the renderer skips ligature-aware token splitting for the entire line (line 509 of viewLineRenderer.ts), but the browser still renders the ligature glyphs, causing overlapping characters.

## Change summary

Characters overlap when a line contains both RTL characters and font ligatures (e.g., `سلام ===` with FiraCode).

## Best solution

The `splitLargeTokens` logic (or equivalent ligature-aware splitting) should handle tokens that do NOT contain RTL even when the overall line contains RTL. The guard at line 509 currently skips splitting for the entire line; it could be refined to only protect RTL-containing tokens while still splitting LTR tokens that benefit from ligature-aware handling.
