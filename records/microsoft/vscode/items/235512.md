# Issue #235512 — Characters go into each other with fontLigatures and RTL characters.

[https://github.com/microsoft/vscode/issues/235512](https://github.com/microsoft/vscode/issues/235512)

Keep open as an actionable editor rendering bug: the report has a concrete reproduction and there is no confirmed issue-specific fix on current main. Later bidi-rendering changes may have affected this case, but they do not provide enough evidence to close it as completed.

## Change summary

Mixed RTL Arabic text and a Fira Code ligature sequence like `سلام ===` overlap or render incorrectly when `editor.fontLigatures` is enabled.

## Best solution

Verify the exact `سلام ===` + Fira Code + `editor.fontLigatures` case on current Insiders/main, then add focused renderer coverage in `src/vs/editor/common/viewLayout/viewLineRenderer.ts` if it still reproduces. The likely fix belongs in editor rendering by isolating mixed-direction/operator runs correctly, or otherwise avoiding ligature shaping that corrupts bidi layout.
