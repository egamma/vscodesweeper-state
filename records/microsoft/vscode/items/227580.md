# Issue #227580 — [Accessibility] The Explorer section view needs role and name information.

[https://github.com/microsoft/vscode/issues/227580](https://github.com/microsoft/vscode/issues/227580)

## Summary

The sidebar part is intentionally set to role="none" (commit ab43469e12b, issue #114892) to reduce screen reader verbosity, but the reporter's concern about missing tabpanel/aria-labelledby semantics for Explorer views is a valid WCAG gap. This is a real accessibility tension that needs maintainer judgment — keeping open for the accessibility team.

## Change summary

The Explorer sidebar container uses role="none" and lacks tabpanel/aria-labelledby attributes, making it harder for screen reader users to understand the tab/panel relationship of views.

## Best solution

The accessibility team should evaluate whether the sidebar (and other role="none" parts) can adopt tabpanel semantics without reintroducing the chattiness that #114892 fixed. The fix would touch `src/vs/workbench/browser/workbench.ts` (part creation) and potentially `src/vs/workbench/browser/parts/paneCompositePart.ts` for aria-labelledby wiring. This likely requires hands-on screen reader testing to find the right balance.
