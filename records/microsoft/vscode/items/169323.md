# Issue #169323 — Search editor can randomize the result order and also the file count(?) when changing the context lines

[https://github.com/microsoft/vscode/issues/169323](https://github.com/microsoft/vscode/issues/169323)

## Summary

Still reproduces on main: changing context lines in the search editor triggers a full re-search (line 163 of searchEditor.ts), which can produce non-deterministic result ordering and varying file counts due to async search results arriving in different orders or hitting result limits differently.

## Change summary

Toggling context lines in the search editor re-runs the entire search, causing result order randomization and inconsistent file counts.

## Best solution

When context lines change, re-serialize the existing search results with the new context-line count instead of triggering a full re-search. The search model already holds the results; `serializeSearchResultForEditor` accepts the `contextLines` parameter — the fix is to call serialization with updated context lines on the cached results rather than invoking `triggerSearch`. This would be in `src/vs/workbench/contrib/searchEditor/browser/searchEditor.ts` around line 163.
