# Issue #169323 — Search editor can randomize the result order and also the file count(?) when changing the context lines

[https://github.com/microsoft/vscode/issues/169323](https://github.com/microsoft/vscode/issues/169323)

## Summary

The bug appears to still be present on main. Changing context lines in the search editor triggers a full re-search, and the result order can change between runs due to non-deterministic file-system result ordering. No commit references this issue and no fix was found in the search editor serialization or search model code.

## Change summary

Search editor randomizes result order and possibly file count when changing context line count.

## Best solution

When context lines change, the search editor should ideally re-serialize from the existing search model results rather than re-running the full search query. Alternatively, ensure that the serialization path produces a stable, deterministic ordering regardless of result arrival order. The fix likely belongs in `src/vs/workbench/contrib/searchEditor/browser/searchEditor.ts` (the trigger path) and possibly `searchEditorSerialization.ts` (the serialization path).
