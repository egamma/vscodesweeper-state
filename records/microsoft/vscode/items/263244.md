# Issue #263244 — "Workspace folder does not exist: ." if trailing "." in "files to exclude"

[https://github.com/microsoft/vscode/issues/263244](https://github.com/microsoft/vscode/issues/263244)

## Summary

Confirmed bug on current main: bare '.' in search exclude/include falls through to the workspace-folder-name lookup and throws a misleading error. An open PR (#309140) already addresses it.

## Change summary

Entering '.' in the 'files to exclude' field in a multi-root workspace throws 'Workspace folder does not exist: .' instead of treating it as the current folder shorthand.

## Best solution

Land the open fix PR https://github.com/microsoft/vscode/pull/309140, which extends the existing './' and '.\' short-circuit on line 513 of queryBuilder.ts to also cover bare '.'. Review and merge that PR.
