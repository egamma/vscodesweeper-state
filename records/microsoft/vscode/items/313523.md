# Issue #313523 — MCP Authentication Dialog overflows and hides action buttons when URL is very long (Linux)

[https://github.com/microsoft/vscode/issues/313523](https://github.com/microsoft/vscode/issues/313523)

## Summary

The dialog overflow fix (d481546f4c4, 1.111.0) added max-height/scroll constraints to the generic dialog component, and word-wrap: break-word already exists — but the reporter's version (1.118.1) post-dates that fix, so the MCP authentication URL dialog may trigger a different code path or the horizontal expansion still occurs for very long unbreakable query-param tokens. Keeping open as a likely real visual bug in the MCP auth flow.

## Change summary

MCP authentication dialog overflows and hides action buttons when the OAuth URL contains extremely long tokens.

## Best solution

Investigate whether the MCP auth URL confirmation dialog (likely routed through `trustedDomainsValidator.ts` line 89's `detail` field with an untruncated string) needs the same ~60-char truncation applied to MCP auth URLs, or whether the dialog's word-wrap: break-word is failing on long query-string tokens. A fix would likely be in `src/vs/workbench/contrib/url/browser/trustedDomainsValidator.ts` (ensure truncation applies regardless of resource type) or in dialog CSS (add `overflow-wrap: break-word` alongside `word-wrap`).
