# Issue #324049 — Allow Repository inheritence of MCP Servers like Skills

[https://github.com/microsoft/vscode/issues/324049](https://github.com/microsoft/vscode/issues/324049)

## Summary

Valid feature request for MCP server discovery to walk parent directories (like skills/instructions do), reducing duplication in monorepos. No existing implementation or duplicate found; routing to the MCP area.

## Change summary

Allow .mcp.json discovery to traverse parent directories up to the repository root, similar to how chat.useCustomizationsInParentRepositories works for other customization files.

## Best solution

Extend WorkspaceDotMcpDiscovery (src/vs/workbench/contrib/mcp/common/discovery/workspaceDotMcpDiscovery.ts) to optionally walk parent directories when resolving .mcp.json, gated behind a setting analogous to chat.useCustomizationsInParentRepositories. Merge discovered servers from all levels.
