# Issue #224562 — Issues with Multi-cursor and Selection Drag Mouse Functionality

[https://github.com/microsoft/vscode/issues/224562](https://github.com/microsoft/vscode/issues/224562)

## Summary

The reported behavior matches the editor's intentional column-selection anchor semantics: Shift+Alt/Option drag extends from the existing cursor or selection start rather than resetting the anchor to the mouse-down point.

## Change summary

Shift+Option multi-cursor drag should start the rectangular selection from the mouse-down location instead of the current cursor position.

## Best solution

Keep the existing Shift+Alt/Option column-selection behavior unless a separate feature request proposes an alternate gesture or setting; changing this directly would break the square-selection behavior covered by the existing regression test.

- **Proposed comment:**

  > Thanks for the report. The current Shift+Alt/Option column-selection behavior is anchored from the existing cursor or selection start, not from the mouse-down point. That behavior is covered by the regression test in src/vs/editor/test/browser/controller/cursor.test.ts:1007 and traces back to commit 2d853212c4642664f9e2006d9072579e49af54f8 for https://github.com/microsoft/vscode/issues/20087, so I would treat this as as-designed rather than a current bug.

## Evidence

- **current routing:** src/vs/editor/browser/view/viewController.ts:277 routes Shift plus the multicursor modifier to _columnSelect(..., true), meaning it extends the active column-selection anchor instead of creating a new anchor at the mouse-down position.
- **anchor implementation:** src/vs/editor/browser/coreCommands.ts:469 uses prevColumnSelectData.fromViewLineNumber/fromViewVisualColumn when doColumnSelect is true; src/vs/editor/common/cursor/cursor.ts:285 derives that initial data from the primary cursor selection start when there is no active column selection.
- **regression test:** src/vs/editor/test/browser/controller/cursor.test.ts:1007 explicitly tests the same shape as this report: cursor at line 10, column 10, then column-selecting toward line 1, column 1 produces selections anchored at column 10 on each line.
- **historical fix:** Commit 2d853212c4642664f9e2006d9072579e49af54f8 ('Fixes #20087: Broken Square Select') introduced this selection-start anchoring behavior for https://github.com/microsoft/vscode/issues/20087.
