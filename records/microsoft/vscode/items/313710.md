# Issue #313710 — MCP file fetch tools return inaccessible resource URIs

[https://github.com/microsoft/vscode/issues/313710](https://github.com/microsoft/vscode/issues/313710)

## Summary

Fixed on main: the root cause was that MCP tool invocations used the wrong call ID for resource permalinks, so vscode-chat-response-resource:// URIs couldn't be resolved back to their content. Commit 2f36dfd5431 switched to chatStreamToolCallId and shipped in 1.120.0.

## Change summary

MCP file fetch tools returned vscode-chat-response-resource:// URIs that were immediately inaccessible in the same chat session.

## Best solution

Keep the shipped fix in commit 2f36dfd5431 (1.120.0); no further change needed. The broader feature request for programmatic MCP resource access by agents is tracked separately in https://github.com/microsoft/vscode/issues/291004.
