# Issue #260898 — `Keybinding` for `Git: Open Containing Folder` doesn't work

[https://github.com/microsoft/vscode/issues/260898](https://github.com/microsoft/vscode/issues/260898)

## Summary

Confirmed on current main: the git reveal commands early-return when no `resourceState` argument is provided, which is the case for keybinding invocations. An open PR (#297211) already addresses this. The 'On Deck' milestone means this stays open for the maintainer.

## Change summary

Keybinding invocations of `git.revealFileInOS.*` and `git.revealInExplorer` silently do nothing because the commands expect a `SourceControlResourceState` argument only provided by context menus.

## Best solution

Review and land https://github.com/microsoft/vscode/pull/297211, which adds a fallback to `getSCMResource()` / active editor URI when no `resourceState` is passed — matching the pattern already used by `openFile` and `ignore` in the same file.
