# Issue #305876 — chat.promptFilesLocations ordering ignored

[https://github.com/microsoft/vscode/issues/305876](https://github.com/microsoft/vscode/issues/305876)

## Summary

Valid bug: `computeListPromptFiles` hard-codes user-storage prompts before local-storage, so when the same prompt name exists in both locations, the user-level file wins regardless of the ordering in `chat.promptFilesLocations`. No fix has landed on main.

## Change summary

When a prompt file with the same name exists in both a local workspace folder and a user-level folder, the user-level version always wins for `/` slash commands, ignoring the configured order in `chat.promptFilesLocations`.

## Best solution

In `computeListPromptFiles` (promptsServiceImpl.ts), respect the config-defined ordering when flattening results from user and local storage, and add deduplication logic for regular prompts (similar to the existing skill dedup) so the first-listed location wins. Alternatively, expose an explicit precedence rule in the setting docs.
