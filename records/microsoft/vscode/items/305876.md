# Issue #305876 — chat.promptFilesLocations ordering ignored

[https://github.com/microsoft/vscode/issues/305876](https://github.com/microsoft/vscode/issues/305876)

## Summary

Confirmed in source: `getSourceFoldersInDiscoveryOrder()` hardcodes user-before-local ordering, ignoring the user's configured `chat.promptFilesLocations` order. An open PR (#308171) exists to fix this but has not yet been merged.

## Change summary

The ordering of entries in `chat.promptFilesLocations` is ignored — user-scope prompts always take precedence over workspace-scope prompts regardless of configuration order.

## Best solution

Merge the fix in PR #308171 (or an equivalent) so that `getSourceFoldersInDiscoveryOrder` respects the configured ordering from `chat.promptFilesLocations` rather than hardcoding `[...userFolders, ...localFolders]`.
