# Issue #188134 — Inconsistent naming for WSL projects in right-click taskbar menu

[https://github.com/microsoft/vscode/issues/188134](https://github.com/microsoft/vscode/issues/188134)

## Summary

Still present on main: when a recent workspace entry has no stored `label`, the jump list fallback in `getWindowsJumpListLabel` uses only `basename(workspace)` and never appends the remote authority suffix (e.g. [WSL]), causing inconsistent naming between entries that have a stored label and those that don't.

## Change summary

WSL (and other remote) projects in the Windows taskbar jump list sometimes lack the [WSL] suffix, appearing inconsistently alongside entries that do show it.

## Best solution

In `getWindowsJumpListLabel` (workspacesHistoryMainService.ts), when falling back to `basename(workspace)` because no stored label exists, also check `recent.remoteAuthority` and append the remote suffix (e.g. `[WSL: Ubuntu]`) to match the format of entries that have a stored label.
