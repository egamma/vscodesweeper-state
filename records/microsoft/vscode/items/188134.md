# Issue #188134 — Inconsistent naming for WSL projects in right-click taskbar menu

[https://github.com/microsoft/vscode/issues/188134](https://github.com/microsoft/vscode/issues/188134)

## Summary

Still reproduces on main: `getWindowsJumpListLabel` only shows the [WSL] suffix when `recentLabel` is already set; when `recentLabel` is undefined (e.g. entries added via `onDidEnterWorkspace`), it falls through to `basename(workspace)` without using the available `remoteAuthority` field to append the remote suffix.

## Change summary

WSL/SSH folders in the Windows taskbar jump list sometimes lack the [WSL: ...] or [SSH: ...] suffix, making them indistinguishable from local folders with the same name.

## Best solution

In `getWindowsJumpListLabel` (workspacesHistoryMainService.ts), when `recentLabel` is undefined but `remoteAuthority` is present, append a `[remoteAuthority]` suffix to the title (similar to what the window title logic does). This ensures consistent labeling regardless of how the entry was added to the recents list.
