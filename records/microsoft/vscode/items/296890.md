# Issue #296890 — Agent mode: sending message mid-response sometimes forks session into two parallel agents, causing conflicting edits and JSONL corruption

[https://github.com/microsoft/vscode/issues/296890](https://github.com/microsoft/vscode/issues/296890)

## Summary

Plausible race condition in cancellation logic: `cancelCurrentRequestForSession` uses `raceTimeout(responseCompletePromise, 1000)` — if the in-flight response does not settle within 1 second, the cancel returns and the new request is dispatched while the old one is still running. No fix has landed on main; an open PR (#309286) addresses corruption recovery but not the root cause.

## Change summary

Sending a new message while an agent is mid-response can sometimes fail to cancel the in-progress response, resulting in two parallel agent threads making conflicting edits and corrupting the session JSONL file.

## Best solution

The cancellation path in `chatServiceImpl.ts` (`cancelCurrentRequestForSession`) should guarantee the previous response is fully terminated before the new request is dispatched — the 1-second `raceTimeout` at line 2011 is insufficient for long-running tool invocations. Additionally, the session JSONL writer should enforce single-writer semantics. The open PR https://github.com/microsoft/vscode/pull/309286 addresses recovery from corruption but not the root cause race.
