# Issue #203734 — [ux] `window.titleBarStyle: "custom"` causes rightmost pixels to resize window instead of actioning scrollbar

[https://github.com/microsoft/vscode/issues/203734](https://github.com/microsoft/vscode/issues/203734)

## Summary

Confirmed upstream Electron bug — the resize border overlapping the scrollbar edge is an Electron window-management issue, not a VS Code defect. Already tracked at electron/electron#45916 by bpasero. Keeping open to track the upstream fix landing.

## Change summary

With custom title bar, the rightmost pixel column triggers window resize instead of scrollbar interaction, breaking Fitts's Law for tiled windows.

## Best solution

The fix belongs in Electron's resize-border handling (https://github.com/electron/electron/issues/45916). Keep this issue open as the VS Code tracking item until Electron ships a fix and VS Code picks up the updated Electron version. No VS Code-side code change is expected.
