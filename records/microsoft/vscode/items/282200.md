# Issue #282200 — Vscode minimizes when titlebar is middle clicked

## Summary

Verified duplicate: this reports the same middle-click-on-custom-titlebar misbehavior on Linux that is already tracked by the open canonical issue #231292, which is labeled upstream and linked to an Electron issue. Proposing duplicate close.

## Change summary

Middle-clicking VS Code's custom titlebar on Linux lowers the window but alt-tab cycling doesn't restore it properly, requiring two alt-tabs.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/231292, which tracks the upstream middle-click titlebar regression on Linux and is linked to Electron and GNOME/Mutter bugs. The user's immediate workaround is `"window.titleBarStyle": "native"` as benibenj already suggested.
