# Issue #314326 — Unescaped char for pending command in session list

[https://github.com/microsoft/vscode/issues/314326](https://github.com/microsoft/vscode/issues/314326)

## Summary

Two valid, unfixed bugs in the session list approval row: (1) raw escape characters from terminal command lines are rendered visibly because the label is passed verbatim into a markdown code block with no stripping, and (2) the status text can show 'running' (from the session description) when the command is actually waiting for user approval. Both issues are traceable in the current source and no fixing commit was found.

## Change summary

In the Agents window session list, a pending terminal command approval row shows raw escape characters and misleading 'running' status text instead of indicating that user input is needed.

## Best solution

Strip ANSI/shell escape sequences from the approval label before rendering in `sessionsList.ts` (around line 647–656), and ensure the session description or status text reflects 'waiting for approval' rather than 'running' when a terminal tool invocation is in `WaitingForConfirmation` state. The approval model (`agentSessionApprovalModel.ts` line 126) and/or the session list renderer are the likely fix sites.
