# Issue #250116 — Wrong/previous version of the extension is still used for debugger features after Install From VSIX... and Restart Extension

[https://github.com/microsoft/vscode/issues/250116](https://github.com/microsoft/vscode/issues/250116)

## Summary

Team-authored issue with an identified root cause (IExtensionPoint.setHandler not surfacing changes during extension updates) and an active design discussion among maintainers — keeping open.

## Change summary

After installing a newer VSIX and restarting extensions, debugger-related extension points still use stale contributions from the previous version until a full window reload.

## Best solution

Add a `changed` field to the `IExtensionPointDelta` so that `setHandler` consumers can detect when an extension's contributions were replaced in-place during an update, without requiring a full window reload. This likely touches `src/vs/workbench/services/extensions/common/extensionsRegistry.ts` and its consumers in the debug layer.
