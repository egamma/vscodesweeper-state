# Issue #262610 — Edit telemetry causes serious typing lag on large-ish documents

[https://github.com/microsoft/vscode/issues/262610](https://github.com/microsoft/vscode/issues/262610)

## Summary

Still reproduces on main: edit telemetry was re-enabled (default: true) after the refactoring in Aug 2025, and the core performance issue — unbounded accumulation of edits via `_edits.compose(e)` in `DocumentEditSourceTracker` — remains present. The issue also carries the `On Deck` milestone, which is a hard keep-open.

## Change summary

Edit telemetry causes ~0.5–1s typing lag on very large documents (e.g. whatwg/html `source` file) because each keystroke composes against the full accumulated edit history.

## Best solution

Bound the accumulated edit history in `DocumentEditSourceTracker` — either periodically flush/compact the `_edits` AnnotatedStringEdit, or switch to an incremental summary that doesn't grow with every keystroke. The fix belongs in `src/vs/workbench/contrib/editTelemetry/browser/telemetry/editTracker.ts`.
