# Issue #307216 — Electron fetch in the utility process doesn't respect HTTP cache headers

[https://github.com/microsoft/vscode/issues/307216](https://github.com/microsoft/vscode/issues/307216)

## Summary

Team-authored upstream bug with an 'On Deck' milestone and an assigned owner — both are hard keep-open conditions. The root cause is confirmed in Electron's utility process network context (disk cache disabled), and a related Electron PR has merged to add session support to utility process requests.

## Change summary

Electron's utility process URLLoader uses a system network context that disables disk cache, so VS Code's fetch calls from the utility process ignore HTTP cache headers.

## Best solution

Track the upstream Electron fix (https://github.com/electron/electron/pull/51279) and adopt it once it lands in a stable Electron release used by VS Code. The fix lives in Electron's `system_network_context_manager.cc`; VS Code's side likely just needs an Electron version bump.
