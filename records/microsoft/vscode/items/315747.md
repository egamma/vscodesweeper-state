# Issue #315747 — Plan content is different when editing

[https://github.com/microsoft/vscode/issues/315747](https://github.com/microsoft/vscode/issues/315747)

## Summary

Team-filed UX polish issue: the plan review widget renders a summary but doesn't label it as such, making the transition to 'Edit' (which shows the full plan) confusing. Keeping open for the team to decide how to communicate the summary vs. full-plan distinction.

## Change summary

The rendered plan content in the chat widget is a summary, but clicking Edit reveals different (full) content with no indication the initial view was summarized.

## Best solution

Either label the rendered markdown as a summary (e.g. a 'Summary' header or subtitle), or provide a way to view the full rendered plan alongside the summary. The relevant widget code is in `src/vs/workbench/contrib/chat/browser/widget/chatContentParts/chatPlanReviewPart.ts` and the tool definition in `src/vs/workbench/contrib/chat/common/tools/builtinTools/reviewPlanTool.ts`. @DonJayamanne was cc'd for input.
