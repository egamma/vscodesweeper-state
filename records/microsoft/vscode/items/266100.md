# Issue #266100 — Deleted files in VS Code are being moved to Snap trash, consuming excessive disk space

[https://github.com/microsoft/vscode/issues/266100](https://github.com/microsoft/vscode/issues/266100)

## Summary

The trash behavior is delegated to Electron's `shell.trashItem()` API — VS Code has no control over where the OS or Snap sandbox places trashed files. This is an upstream Electron/Snap platform issue, not something VS Code's own code can fix. Keeping open with the existing `electron` label.

## Change summary

Files deleted from the VS Code explorer on Ubuntu Snap accumulate in the Snap-specific trash directory, consuming excessive disk space.

## Best solution

This is an Electron/Snap platform interaction issue. VS Code calls `shell.trashItem()` (src/vs/platform/files/electron-main/diskFileSystemProviderServer.ts:49) and has no say in where the OS routes trashed items. The upstream fix belongs in Electron's trash implementation for Snap-confined environments, or in the Snap confinement configuration. Users can work around it by setting `files.enableTrash` to `false` (permanently deletes instead of trashing) or periodically clearing the Snap trash.
