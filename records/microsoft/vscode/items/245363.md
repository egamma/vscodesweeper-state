# Issue #245363 — SCM input placeholder text is not vertically centered

[https://github.com/microsoft/vscode/issues/245363](https://github.com/microsoft/vscode/issues/245363)

## Summary

Still present on main: the placeholder overlay widget positions itself at `getTopForLineNumber(0)` (which returns `paddingTop`), matching where line-1 text starts — but when the SCM input container is visually taller than its single-line content, the placeholder appears top-aligned rather than vertically centered within the visible input area.

## Change summary

SCM commit-message input's placeholder text appears visually higher than center when the input box is taller than one line of text.

## Best solution

Adjust `PlaceholderTextContribution` (or the SCM input's editor options) so the placeholder vertically centers within the editor viewport when the content is a single empty line and the editor height exceeds the line height. This likely lives in `src/vs/editor/contrib/placeholderText/browser/placeholderTextContribution.ts` and may need a layout-aware offset calculation.
