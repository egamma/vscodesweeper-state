# Issue #320530 — Copilot tool list showing wrong name for tools

[https://github.com/microsoft/vscode/issues/320530](https://github.com/microsoft/vscode/issues/320530)

## Summary

Confirmed regression from PR #318638: when multiple MCP servers announce the same serverInfo.name (e.g. a shared proxy binary), the tool prefix uses that announced name instead of the distinct mcp.json key, causing indistinguishable tool names in the picker.

## Change summary

MCP tools list shows the server-announced binary name instead of the user-configured JSON key, causing name collisions when multiple servers share the same binary.

## Best solution

The preferredName logic in `src/vs/workbench/contrib/mcp/common/mcpServer.ts` (line 578) should detect collisions from announced names and fall back to `definition.label` (the JSON key) as a disambiguator, or prefer the user-configured label when the announced name would create duplicates.
