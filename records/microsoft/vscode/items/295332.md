[https://github.com/microsoft/vscode/issues/295332](https://github.com/microsoft/vscode/issues/295332)

## Summary

Valid feature request to auto-deny or add a retry-hint mechanism for agent file access outside the workspace. The area is actively evolving — `ChatExternalPathConfirmationContribution` already gates external-path reads with allowlists, but no auto-deny option or agent-facing hint exists yet. Route to the chat-agent area for triage.

## Change summary

Add an option to automatically deny (or prompt the agent to self-correct) when Copilot Chat attempts to read files outside the current workspace, avoiding unnecessary approval pauses.

## Best solution

Add a setting (e.g. `chat.agent.autoDenyExternalPaths`) that auto-rejects file reads outside workspace/allowlisted folders, or returns a tool-level error message telling the agent the path is outside the workspace so it can self-correct before escalating to user approval. The external-path confirmation logic in `src/vs/workbench/contrib/chat/common/tools/builtinTools/chatExternalPathConfirmation.ts` is the natural place for this. This should be evaluated alongside the broader agent sandbox/security work.
