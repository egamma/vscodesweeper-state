# Issue #143643 — Git - `git.openFile` command doesn't work properly with Replace Preview

[https://github.com/microsoft/vscode/issues/143643](https://github.com/microsoft/vscode/issues/143643)

## Summary

Still reproducible on main: git.openFile's fallback path uses activeTextEditor.document.uri without checking whether the scheme is actionable, so it passes the replacePreview URI to vscode.open, producing a duplicate/broken editor tab.

## Change summary

Running `git.openFile` from a Replace Preview editor opens a broken editor tab instead of the underlying file.

## Best solution

In the fallback branch of `openFile` (extensions/git/src/commands.ts ~line 1319), convert or filter the activeTextEditor URI to extract the underlying file resource when the scheme is not `file` or `git` — the `toFileResource` helper in the search contrib already does this conversion for `replacePreview` URIs.
