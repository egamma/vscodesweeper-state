# Issue #235210 — Clear Search Results also clears the filter unexpectedly

[https://github.com/microsoft/vscode/issues/235210](https://github.com/microsoft/vscode/issues/235210)

## Summary

Still reproduces on main: the 2-stage clear logic in `clearSearchResults()` clears file include/exclude patterns when the search input is already empty, which is unexpected behavior for a button labeled 'Clear Search Results'.

## Change summary

The 'Clear Search Results' button unexpectedly clears include/exclude file filters when pressed after the search input is already empty.

## Best solution

Remove or gate the 2-stage clearing logic in `SearchView.clearSearchResults()` (line ~1416 of `src/vs/workbench/contrib/search/browser/searchView.ts`) so that `clearFilePatternFields()` is not called from the clear-results action. If separate pattern clearing is desired, it should be a distinct UI action.
