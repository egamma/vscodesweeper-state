# Issue #235210 — Clear Search Results also clears the filter unexpectedly

[https://github.com/microsoft/vscode/issues/235210](https://github.com/microsoft/vscode/issues/235210)

## Summary

Still reproduces on main: the Clear Search Results button clears include/exclude file filters on a second press (when search fields are already empty), because `clearSearchResults()` calls `clearFilePatternFields()` whenever `allSearchFieldsClear()` is true. Filed by a contributor and confirmed by a team member.

## Change summary

Clicking 'Clear Search Results' when the search box is already empty also clears the include/exclude file pattern filters.

## Best solution

Guard the `clearFilePatternFields()` call in `searchView.ts#clearSearchResults()` so it only runs when the user explicitly intends to clear filters (e.g., require a third deliberate action, or never auto-clear filters from the 'Clear Search Results' button). The fix is localized to `src/vs/workbench/contrib/search/browser/searchView.ts` around lines 1415–1418.
