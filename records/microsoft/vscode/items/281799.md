# Issue #281799 — Chat: Custom instruction causes fetch and Allow dialog

[https://github.com/microsoft/vscode/issues/281799](https://github.com/microsoft/vscode/issues/281799)

## Summary

Team-authored issue reporting that custom instruction files trigger an unnecessary fetch/Allow dialog and then fail because the model strips the `vscode-userdata:` scheme. Keeping open per team-author policy.

## Change summary

Custom instruction files referenced via `vscode-userdata:` scheme cause a fetch permission dialog and then fail to be read because the model uses the bare filesystem path instead of the full URI.

## Best solution

Ensure the `read_file` tool (or the system prompt guiding models) correctly handles `vscode-userdata:` URIs for prompt/instruction files, either by reinforcing the scheme in the system prompt or by having the tool gracefully resolve userdata paths without requiring a fetch dialog.
