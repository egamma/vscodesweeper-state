# Issue #281799 — Chat: Custom instruction causes fetch and Allow dialog

[https://github.com/microsoft/vscode/issues/281799](https://github.com/microsoft/vscode/issues/281799)

## Summary

Team-authored bug about custom instructions triggering a fetch/Allow dialog and then failing because the read_file tool strips the vscode-userdata: scheme. No clear fix found on main for this specific issue.

## Change summary

Custom instructions with vscode-userdata: URIs trigger a fetch Allow dialog and then fail when the LLM's read_file tool call omits the URI scheme.

## Best solution

The read_file tool (or the instruction-loading path) should handle vscode-userdata: URIs transparently — either by not exposing them to the model as fetchable URLs, or by recognizing the scheme when the model attempts to read them. Related work in the copilot extension's instruction handling and the prompt system prompt should ensure the model knows to use the full URI with scheme.
