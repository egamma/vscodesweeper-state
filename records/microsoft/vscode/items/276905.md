# Issue #276905 — Very hard to resize split panes

[https://github.com/microsoft/vscode/issues/276905](https://github.com/microsoft/vscode/issues/276905)

## Summary

The default sash size is intentionally 4px, with a built-in setting (`workbench.sash.size`) designed for users who find it hard to resize. No code changes around v1.105 affected sash behavior — the issue describes existing default behavior, not a regression.

## Change summary

User finds it difficult to grab the 4px sash to resize split panes on macOS.

## Best solution

Close as as-designed: the `workbench.sash.size` setting (range 1–20, default 4) exists for exactly this complaint. The setting description says "Set it to a larger value if you feel it's hard to resize views using the mouse." The `workbench.sash.hoverDelay` (default 300ms) can also be reduced.

  >

  > - **`workbench.sash.hoverDelay`** — reduce it (default 300ms) so the hover highlight appears faster
