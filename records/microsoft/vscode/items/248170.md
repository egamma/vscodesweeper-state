# Issue #248170 — Cannot connect to MCP server via SSE using trusted developer certificate

[https://github.com/microsoft/vscode/issues/248170](https://github.com/microsoft/vscode/issues/248170)

## Summary

Active upstream Electron issue with deep team engagement (chrmarti, connor4312, deepak1556, JamesNK). An Electron distro fix for self-signed cert handling landed in April 2026 (commit 015aef6054c) but the team hasn't closed this, and the broader system-cert tracking issue #287739 remains open on On Deck — verification is needed to confirm the dotnet dev-cert scenario now works.

## Change summary

Connecting to an MCP server over HTTPS with a trusted .NET developer certificate fails with 'fetch failed' because Electron's Node.js TLS validation is stricter than standalone Node.js for self-signed certificates.

## Best solution

Verify whether the Electron distro bump in commit 015aef6054c ('fix: tls.getCACertificates result for self-signed and server auth', April 2026) resolves the dotnet dev-cert scenario on current Insiders. If it does, close this issue and note the fix in the broader tracking issue https://github.com/microsoft/vscode/issues/287739. If it doesn't, the remaining gap is in Electron's TLS handling and should be escalated via deepak1556.
