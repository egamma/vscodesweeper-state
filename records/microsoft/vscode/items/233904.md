# Issue #233904 — vscode window is large then it seems. transparent unclickable area around all windows.

[https://github.com/microsoft/vscode/issues/233904](https://github.com/microsoft/vscode/issues/233904)

## Summary

Likely an upstream Electron issue: when VS Code uses `frame: false` for the custom title bar on Linux/Wayland (src/vs/platform/windows/electron-main/windows.ts, line 214), Electron creates oversized invisible resize borders (~50px) that capture clicks outside the visible window bounds. The issue is specific to Fedora 41 GNOME/Wayland and may interact with compositor behavior. Keeping open as upstream.

## Change summary

On Linux (Fedora 41, GNOME Wayland), VS Code windows have a ~50px transparent area around all edges that captures mouse clicks, preventing interaction with windows underneath.

## Best solution

Investigate the Electron resize border behavior when `frame: false` on Wayland/GNOME — the input region (or the invisible resize handle) appears oversized. A workaround for the reporter is to try `"window.titleBarStyle": "native"` in settings, which avoids `frame: false`. The root fix likely belongs in Electron's Linux/Wayland window management code. Consider filing upstream at electron/electron if not already tracked. Related VS Code issues: https://github.com/microsoft/vscode/issues/203734 (resize border stealing scrollbar clicks) and https://github.com/microsoft/vscode/issues/225073 (GNOME Wayland shadow with custom title bar).
