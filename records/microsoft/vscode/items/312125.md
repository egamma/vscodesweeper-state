# Issue #312125 — Insiders Update Fails - Windows - MoveFile failed; code 5. Access is denied

[https://github.com/microsoft/vscode/issues/312125](https://github.com/microsoft/vscode/issues/312125)

## Summary

Well-diagnosed bug: the auto-updater intentionally sets `__COMPAT_LAYER: 'RunAsInvoker'` when spawning Inno Setup, which on certain Windows configurations strips the subprocess token to `User privileges: None`, preventing writes to the user's own install directory. The code is confirmed on current main at lines 442 and 620 of the update service. Should be routed to the install-update area for triage.

## Change summary

VS Code Insiders auto-update fails on Windows with 'MoveFile failed; code 5. Access is denied' because the Inno Setup subprocess inherits a restricted token from the `RunAsInvoker` compatibility layer.

## Best solution

Investigate whether `__COMPAT_LAYER: 'RunAsInvoker'` should be conditionally applied (e.g. only for system installs where UAC elevation must be suppressed, not for per-user installs where the user already has write access). The fix lives in `src/vs/platform/update/electron-main/updateService.win32.ts` at the two spawn sites (lines ~442 and ~620). This needs a product decision since removing `RunAsInvoker` unconditionally could reintroduce UAC prompts.
