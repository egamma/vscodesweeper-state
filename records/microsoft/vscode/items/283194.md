# Issue #283194 — Gemini 2.5 pro context window seems to be much less than 1M

[https://github.com/microsoft/vscode/issues/283194](https://github.com/microsoft/vscode/issues/283194)

## Summary

Active, well-upvoted feature request (31 👍) for larger Copilot model context windows, especially Gemini 2.5 Pro. Maintainers have confirmed the limitation is intentional due to capacity/engineering constraints and have stated this is on the plan. Keeping open as a valid community-tracked request above the 20-upvote Backlog threshold.

## Change summary

Increase the context window size for Gemini 2.5 Pro (and other models) in GitHub Copilot Chat to be closer to the models' native limits, rather than the current ~128K cap.

## Best solution

This is a service-side infrastructure and engineering effort owned by the Copilot team (tracked internally via https://github.com/microsoft/vscode-copilot/issues/16480). The VS Code client should also surface context window limits transparently in the model picker UI, as isidorn suggested. No VS Code client-side code change alone resolves the core ask — the Copilot API must expose larger context windows per model.
