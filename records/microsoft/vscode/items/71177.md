# Issue #71177 — No clue that server ready action is running

## Summary

Valid feature request: the debug-server-ready extension still performs its action (opening browser, starting debug session) silently without any user-visible feedback. The gap remains on current main.

## Change summary

When a serverReadyAction fires during debugging, there is no visible indication to the user that the action is running — no debug console message, no notification.

## Best solution

Add a debug console message (e.g. 'Detected pattern match, launching browser at http://localhost:3000...') in `extensions/debug-server-ready/src/extension.ts` when the pattern matches and the action fires. As weinand noted, an informational notification or debug console append would be appropriate — not terminal output.
