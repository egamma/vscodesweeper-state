# Issue #210624 — Unable to resize with mouse, right half of window not clickable Ubuntu 24.04

[https://github.com/microsoft/vscode/issues/210624](https://github.com/microsoft/vscode/issues/210624)

## Summary

Upstream mutter/Wayland bug already correctly labeled. The root cause (mouse clicks falling through to the window behind after resize) was fixed in mutter 46.2 (LP: #2026194, shipped in Ubuntu 24.04.1). The issue is already well-triaged with `upstream`, `electron`, `wayland` labels and `help wanted`; keeping open for any remaining edge cases or users on older mutter versions.

## Change summary

On Ubuntu 24.04 with Wayland, resizing or maximizing VS Code causes the right half of the window to become unclickable, with clicks falling through to windows behind.

## Best solution

The upstream mutter fix (46.2, LP: #2026194) resolves the core problem. A maintainer could close this with a comment pointing users to update their system (Ubuntu 24.04.1+) and noting the `--ozone-platform=wayland` or `window.titleBarStyle: custom` workarounds for those stuck on older versions. Issue https://github.com/microsoft/vscode/issues/235751 tracks a related but distinct Wayland native-titlebar resize problem.
