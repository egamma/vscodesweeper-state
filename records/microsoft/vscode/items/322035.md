# Issue #322035 — Disable Font Smoothing (Font Antialiasing)

[https://github.com/microsoft/vscode/issues/322035](https://github.com/microsoft/vscode/issues/322035)

## Summary

The requested option already exists on current main, so this can be closed as completed rather than tracked as a new feature request.

## Change summary

Add a VS Code option to disable font smoothing/font antialiasing.

## Best solution

Treat the existing `workbench.fontAliasing` setting as the shipped solution; users who want disabled smoothing can set it to `none`.

- **Proposed comment:**

  > Thanks for the request. This is already supported through the `workbench.fontAliasing` setting: set it to `none` to disable font smoothing. The setting was added in commit 7a78eee3cfaaa11c7fc701d7632e19da63986c21 and is still present on current main, so I believe this can be closed as completed.

## Evidence

- **existing setting:** Current main registers `workbench.fontAliasing` in `src/vs/workbench/browser/workbench.contribution.ts` with enum values `default`, `antialiased`, `none`, and `auto`; the `none` description says it disables font smoothing.
- **current behavior:** `src/vs/workbench/browser/workbench.ts` reads `workbench.fontAliasing` and applies the `monaco-font-aliasing-none` class when the setting is `none`.
- **styling implementation:** `src/vs/workbench/browser/media/style.css` defines `.monaco-workbench.mac.monaco-font-aliasing-none` with `-webkit-font-smoothing: none` and `-moz-osx-font-smoothing: unset`.
- **implementation commit:** Commit 7a78eee3cfaaa11c7fc701d7632e19da63986c21 added the `workbench.fontAliasing` configuration option, including the `none` mode for font antialiasing.
- **issue discussion:** The only comment on https://github.com/microsoft/vscode/issues/322035 notes that https://github.com/microsoft/vscode/issues/195256 indicates this is already possible.
