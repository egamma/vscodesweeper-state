## Change summary

Screen readers do not read filename reference links in chat responses — the accessible viewer shows an empty bulleted list where filenames should be.

## Best solution

Verify with the reporter whether the issue persists on the latest Insiders build (which includes 334b6709af4). If it does, the gap is likely in how file references embedded inside `markdownContent` parts (as `<a>` tags with empty text) are rendered in the accessible view — the `useLinkFormatter` option on `renderAsPlaintext` should produce basenames, but specific Copilot response formats may not hit that path. A targeted fix would be in `chatResponseAccessibleView.ts`.
