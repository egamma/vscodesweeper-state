# Issue #241011 — Moving a line down in Go code messes up indentation in VS Code

[https://github.com/microsoft/vscode/issues/241011](https://github.com/microsoft/vscode/issues/241011)

## Summary

Still reproduces on main — the move-lines-down autoindent logic misapplies indentation rules when moving a line past bracket-heavy Go constructs. The existing `tree-sitter-autoindent-candidate` label correctly identifies this as awaiting the tree-sitter indentation work.

## Change summary

Moving a line down with Alt+Down in Go code progressively corrupts indentation of subsequent lines.

## Best solution

The autoindent logic in `moveLinesCommand.ts` uses regex-based indentation rules that fail on complex bracket patterns (} else if / } else). The tree-sitter autoindent work (tracked by the label) would provide structure-aware indentation that handles these cases correctly. Until then, the virtual model's token mapping (acknowledged TODO at line 170) is the root cause.
