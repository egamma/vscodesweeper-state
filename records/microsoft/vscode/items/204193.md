# Issue #204193 — Editor.cursorSurroundingLines broken with update to 1.86

[https://github.com/microsoft/vscode/issues/204193](https://github.com/microsoft/vscode/issues/204193)

## Summary

Still reproduces on main: `getViewRangeWithCursorPadding` in viewModelImpl.ts uses the static `stickyScroll.maxLineCount` for both top and bottom padding instead of the actual current sticky line count. PR #292158 (open, not merged) partially addresses the bottom padding but the core issue remains.

## Change summary

When sticky scroll is enabled, `editor.cursorSurroundingLines` is overridden by `stickyScroll.maxLineCount` for both top and bottom padding, even when fewer (or zero) sticky lines are actually displayed.

## Best solution

In `src/vs/editor/common/viewModel/viewModelImpl.ts` line 732, separate top and bottom padding: top should use `max(cursorSurroundingLines, currentlyVisibleStickyLineCount)` (queried dynamically from the sticky scroll controller), and bottom should use only `cursorSurroundingLines`. PR https://github.com/microsoft/vscode/pull/292158 addresses part of this but needs extension for the dynamic top calculation.
