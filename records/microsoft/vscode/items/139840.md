# Issue #139840 — Use modern (CSS color level 4) syntax for colors in the color picker

[https://github.com/microsoft/vscode/issues/139840](https://github.com/microsoft/vscode/issues/139840)

## Summary

Still valid: the color picker's output/presentation uses legacy comma-based syntax (rgb(r, g, b), hsl(h, s%, l%)) rather than CSS Color Level 4 space-separated syntax, and oklch/oklab/lch/lab color spaces are not yet supported. Partial progress exists for detection but the core request remains unmet.

## Change summary

Update the color picker to output modern CSS Color Level 4 syntax (space-separated rgb/hsl) and add support for oklch, oklab, lch, and lab color spaces.

## Best solution

Update `Color.Format.CSS.formatRGB/formatHSL` in `src/vs/base/common/color.ts` to emit space-separated Level 4 syntax, and update `defaultDocumentColorProvider.ts` to offer oklch/oklab/lch/lab presentations. PR https://github.com/microsoft/vscode/pull/310768 is in flight for oklch/oklab support — once merged, the remaining gap is the rgb/hsl output format.
