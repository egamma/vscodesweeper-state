# Issue #318254 — OSC52 unsupported in `code serve-web`

[https://github.com/microsoft/vscode/issues/318254](https://github.com/microsoft/vscode/issues/318254)

## Summary

Valid bug: OSC52 clipboard writes in serve-web go to in-memory storage instead of the system clipboard because BrowserClipboardService short-circuits when a `type` parameter is provided. An open PR (#318255) addresses it; keeping open pending that fix.

## Change summary

OSC52 escape sequences in `code serve-web` terminal fail to write to the system clipboard because the browser clipboard service treats any typed write as in-memory only.

## Best solution

Land PR https://github.com/microsoft/vscode/pull/318255 (or an equivalent fix) that lets `type: 'clipboard'` fall through to the real system clipboard path in `src/vs/platform/clipboard/browser/clipboardService.ts`. The area label `terminal` is appropriate; the fix touches the clipboard platform service.
