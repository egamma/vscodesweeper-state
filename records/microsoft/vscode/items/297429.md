# Issue #297429 — Grey out unavailable functions for certain languages in right-click menu

[https://github.com/microsoft/vscode/issues/297429](https://github.com/microsoft/vscode/issues/297429)

## Summary

VS Code intentionally hides context menu items whose preconditions are unmet rather than greying them out — this is a deliberate, long-standing UX choice to keep menus clean and contextually relevant. The behavior is *as-designed*.

## Change summary

Show unavailable editor actions (e.g. Rename when no rename provider exists) as greyed-out context menu entries with tooltips, instead of hiding them.

## Best solution

Keep the current as-designed behavior. VS Code's menu system explicitly merges `precondition` into the `when` clause for context menu items (see `src/vs/editor/browser/editorExtensions.ts` line 375), hiding items when preconditions aren't satisfied. This keeps context menus short, relevant, and consistent — a core UX principle. Showing all possible actions greyed out would bloat context menus significantly across all languages.
