## Summary

Valid feature request with an open community PR (#273679) implementing it. The no-verify commit commands already exist in the dropdown, but the specific ask — surfacing a 'Commit without Verify' option in the hook-failure error dialog — is not yet on main. ## Change summary

Show a 'Commit without Verify' option in the error dialog when git pre-commit hooks fail, so users can retry without hooks directly from the failure prompt.
