# Issue #235816 — Highlighted line in terminal moving with resizing the window

[https://github.com/microsoft/vscode/issues/235816](https://github.com/microsoft/vscode/issues/235816)

## Summary

Confirmed terminal bug where text selection coordinates are not updated during buffer reflow on resize. Still reproduces on current main — the xterm.js reflow logic in BufferReflow.ts does not adjust selection positions, and VS Code does not clear selection on resize either.

## Change summary

Terminal text selection drifts to wrong lines when the terminal panel is resized vertically.

## Best solution

Either xterm.js's reflow should preserve/adjust selection coordinates during rewrap, or VS Code should clear the selection on resize as a simpler interim fix (in terminalInstance.ts where resize is handled). The root cause is likely upstream in @xterm/xterm's BufferReflow.ts which has no selection awareness.
