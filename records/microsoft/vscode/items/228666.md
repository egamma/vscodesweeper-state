# Issue #228666 — Utility process: Simplify the command line and make it obvious which process is doing what

[https://github.com/microsoft/vscode/issues/228666](https://github.com/microsoft/vscode/issues/228666)

## Summary

Team-authored (MEMBER) request to simplify utility process command lines for better identification in OS process views. No fix found on main; must keep open per policy.

## Change summary

VS Code utility processes have opaque command lines, making it hard to identify which process (e.g., extension host, pty host) is which in OS-level process viewers.

## Best solution

Add a human-readable identifier (e.g. `--type=extensionHost`) as the first argument in utility process command lines, and consider moving static arguments to an argv file or environment variables to reduce noise. The spawn logic lives in `src/vs/platform/utilityProcess/electron-main/utilityProcess.ts` and callers like `extensionHostStarter.ts`, `electronPtyHostStarter.ts`, and `sharedProcess.ts`.
