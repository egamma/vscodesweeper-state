# Issue #164900 — Solve merge conflict in git submodule update using new 3 way merge editor

[https://github.com/microsoft/vscode/issues/164900](https://github.com/microsoft/vscode/issues/164900)

## Summary

Still present on main: submodule merge conflicts open the 3-way merge editor but the editor cannot resolve them because submodule entries are commit-hash references, not text files with conflict markers. No fix has been committed.

## Change summary

Opening a conflicted git submodule in the merge editor shows no accept current/incoming buttons, making it impossible to resolve the conflict from VS Code.

## Best solution

The git extension's `resolveChangeCommand` (extensions/git/src/repository.ts ~line 530) and `getResources` (~line 570) need a submodule-aware code path during merge conflicts that either (a) provides a specialized UI to pick between the two commit SHAs, or (b) offers 'accept theirs'/'accept ours' actions via `git checkout --theirs/--ours` for the submodule path. This likely belongs in the git extension's merge handling rather than the merge editor itself.
