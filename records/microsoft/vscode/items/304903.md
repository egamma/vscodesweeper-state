## Change summary

The Problems pane continuously re-evaluates frontmatter diagnostics for custom agent/instruction/skill/prompt files, causing excessive CPU and memory usage due to missing debounce, full re-parse on every event, and cascading event loops.

## Best solution

Complete the incremental fix @aeschli noted in comment 6: reduce change-event emissions (done in 1.114), then add debouncing and incremental parsing so only changed files are re-parsed. Symlink deduplication would further help workspaces like hve-core.
