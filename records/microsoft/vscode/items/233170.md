# Issue #233170 — Support comment toggling for languages that do not support content on the first column

[https://github.com/microsoft/vscode/issues/233170](https://github.com/microsoft/vscode/issues/233170)

## Summary

Implemented on main: commit d9145a291dc introduced a generic `noIndent` option for `lineComment` in language configuration, allowing any language extension to specify that comments must be placed at column 0 rather than indented. This directly addresses the feature request.

## Change summary

Support comment toggling for languages that require the comment token at column 0 (not indented).

## Best solution

Keep the shipped fix (commit d9145a291dc). Language extensions for languages with first-column comment requirements can now use `lineComment: { comment: "#", noIndent: true }` in their language configuration. No further core change is needed.
