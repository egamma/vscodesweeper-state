# Issue #170829 — Perf: Tasks problem matcher and JSON schemas block startup

[https://github.com/microsoft/vscode/issues/170829](https://github.com/microsoft/vscode/issues/170829)

## Summary

Still reproduces on main: the JSON schema construction and registerSchema call remain at module top-level in task.contribution.ts (lines 437–444), executing eagerly at import time rather than deferred to a lifecycle phase. The PR #183039 that attempted to fix this was closed without merging.

## Change summary

Tasks problem matcher and JSON schema parsing run at module load time, blocking startup.

## Best solution

Move the schema construction (lines 437–444 in task.contribution.ts) and the schema imports into the `TaskRegistryContribution` class body so they execute at `WorkbenchPhase.AfterRestored` instead of at module import time. The default exports from jsonSchema_v1.ts and jsonSchema_v2.ts should become lazily-evaluated factory functions to avoid triggering schema creation on import.
