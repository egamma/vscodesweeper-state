## Summary

Team-authored feature request for user control over extension-contributed instructions in agent mode. Keep open per team-author policy; the comment from @aeschli suggests the immediate pain point may be an extension-side fix (Azure extension misusing `applyTo: '**'`), but the broader UX ask for user-removable instructions remains valid.

## Change summary

Users cannot disable or remove extension-contributed instruction files from their agent-mode prompt context.

## Best solution

Two-pronged: (1) ensure extensions like Azure follow best practices by not forcing instructions via `applyTo: '**'` (extension-side guidance), and (2) expose a UX in the chat prompt attachments area that lets users toggle off individual extension instructions, similar to how attached files can be removed.
