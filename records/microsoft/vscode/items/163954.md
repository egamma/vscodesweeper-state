# Issue #163954 — API to disable bracket pair colorization for specific ranges

[https://github.com/microsoft/vscode/issues/163954](https://github.com/microsoft/vscode/issues/163954)

## Summary

Team-authored feature request for an extension API to disable bracket pair colorization in specific ranges. No such API exists on current main; the gap remains real. Hard keep-open (MEMBER author).

## Change summary

Extensions like C/C++ need an API to exclude specific text ranges (e.g. inactive preprocessor regions) from bracket pair colorization so unpaired brackets don't produce mismatched highlights.

## Best solution

Expose an extension API (or a mechanism through semantic tokens / decoration options) that lets language extensions mark ranges where bracket pair colorization should be suppressed. Related to #149552 (bracket colorization with semantic tokens). The implementation would likely touch `src/vs/editor/common/model/bracketPairsTextModelPart/` and the extension host API surface.
