# Issue #163748 — Repositories - svg icons stay dark when the repository row is selected.

[https://github.com/microsoft/vscode/issues/163748](https://github.com/microsoft/vscode/issues/163748)

## Summary

Still unfixed on main. Static SVG icons contributed by extensions don't adapt their fill color when a list row is selected in light themes, making dark icons invisible on the blue selection background. The 'On Deck' milestone is a scheduling decision — keeping open.

## Change summary

Extension-contributed SVG icons in tree/list views stay dark when a row is selected in light themes, making them invisible against the blue selection background.

## Best solution

The list/tree widget needs a mechanism to invert or adapt static SVG icon fills when a row is selected and the selection background is dark. This likely lives in the base list rendering layer (`src/vs/base/browser/ui/list/`) or the icon/decoration rendering. A CSS `filter: invert()` on selected rows' custom SVG icons, or dynamically switching to the dark-theme icon variant on selection, are plausible approaches. Already confirmed as a general list widget issue by @lszomoru, not SCM-specific.
