# Issue #316791 — Agents App: git commit hangs when commit signing (gpg.format=ssh) is enabled due to missing GIT_ASKPASS environment variable

[https://github.com/microsoft/vscode/issues/316791](https://github.com/microsoft/vscode/issues/316791)

## Summary

Valid bug, confirmed in source: agentHostGitService._runGit never injects GIT_ASKPASS, so SSH commit signing hangs indefinitely in the non-TTY agent environment. The recent removal of --no-gpg-sign (8c7f8a1409f) makes this directly triggerable.

## Change summary

Git commit hangs indefinitely in the Agents App when SSH commit signing is enabled because GIT_ASKPASS is not injected into the agent's git execution environment.

## Best solution

Inject GIT_ASKPASS (and supporting VSCODE_GIT_ASKPASS_* variables) into the environment used by AgentHostGitService._runGit, mirroring what extensions/git already does. The fix belongs in src/vs/platform/agentHost/node/agentHostGitService.ts (the _runGit method's env setup) and potentially agentHostTerminalManager.ts for the bash tool path.
