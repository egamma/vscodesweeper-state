## Change summary

Network fetch failures during background extension auto-update propagate as unhandled promise rejections to the global error handler.

## Best solution

Add proper error handling (`.then(undefined, err => null)` or try/catch) around the `autoUpdateBuiltinExtensions()` fire-and-forget calls in `extensionsWorkbenchService.ts` (lines 1192, 1206), similar to how `eventuallyAutoUpdateExtensions` already swallows errors at line 2247. The fix should NOT reclassify the error as OfflineError (per @sandy081's feedback on the first PR).
