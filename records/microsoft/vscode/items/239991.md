# Issue #239991 — VScode auto detects HTML/JS as Dart on unsaved files

[https://github.com/microsoft/vscode/issues/239991](https://github.com/microsoft/vscode/issues/239991)

## Summary

Still reproducible on main: the ML-based language detection model can misclassify JS/HTML/JSON as Dart in unsaved files because Dart is not in the negative confidence correction list in `adjustLanguageConfidence`. The issue is correctly labeled and milestoned; no fix has landed.

## Change summary

Pasting HTML/JS or JSON into an unsaved file causes VS Code's auto language detection to incorrectly identify it as Dart.

## Best solution

Add `'dart'` to the negative confidence correction bucket in `src/vs/workbench/services/languageDetection/browser/languageDetectionWebWorker.ts` (similar to how `bat`, `ini`, `sql` etc. are penalized), or consider a user-facing setting to exclude languages from auto-detection. The fix is small and localized in `adjustLanguageConfidence`.
