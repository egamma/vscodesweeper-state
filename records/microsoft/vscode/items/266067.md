# Issue #266067 — Chat: Invalid/failed tool calls are never shown to the user

[https://github.com/microsoft/vscode/issues/266067](https://github.com/microsoft/vscode/issues/266067)

## Summary

Team-authored feature request for surfacing failed/invalid tool calls in the chat UI. No evidence this has been implemented on main — the tool invocation content parts don't appear to have dedicated error-state rendering. Keeping open per team-member policy and because the gap remains real.

## Change summary

Failed or invalid tool calls (e.g. bad JSON schema for MCP, failed edit-file calls) are silently swallowed in the chat output, leaving users confused about agent struggles.

## Best solution

Add error-state rendering in the tool invocation content parts (`src/vs/workbench/contrib/chat/browser/widget/chatContentParts/toolInvocationParts/`) so that failed tool calls display inline with a clear error indicator and collapsed details. Design input is needed per connor4312's comment. This is core chat UX, not an extension candidate.
