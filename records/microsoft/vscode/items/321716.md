# Issue #321716 — Fcitx5 Hangul preedit text is discarded when clicking another editor position on native Wayland

[https://github.com/microsoft/vscode/issues/321716](https://github.com/microsoft/vscode/issues/321716)

## Summary

This is an upstream Chromium/Electron issue with Wayland IME preedit commit on pointer click. VS Code's editor correctly handles composition events, but Chromium's native Wayland backend fails to commit preedit text before processing the click, discarding it instead. The issue does not reproduce under XWayland or in native GTK apps.

## Change summary

Fcitx5 Hangul preedit text is discarded instead of committed when clicking another editor position on native Wayland.

## Best solution

Track as upstream — the root cause is in Chromium's Wayland IME integration (ozone/wayland input method handling). A workaround in VS Code could force-commit composition on mousedown in nativeEditContext.ts, but the proper fix belongs in Chromium. Consider filing or linking to a Chromium bug for Wayland IME preedit commit on pointer events.
