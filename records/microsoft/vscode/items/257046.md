# Issue #257046 — SSH public key password saving issue

[https://github.com/microsoft/vscode/issues/257046](https://github.com/microsoft/vscode/issues/257046)

## Summary

Confirmed in source: the SSH host-key authenticity prompt returns a localized string (e.g. Chinese "是") to SSH, which expects the literal English "yes" or "no". This causes SSH to reject the answer and re-prompt in a loop. Still unfixed on main.

## Change summary

When VS Code is set to a non-English locale and SSH asks to verify a host key, the confirmation dialog returns a translated string instead of the English "yes"/"no" SSH expects, causing an infinite prompt loop.

## Best solution

In `extensions/git/src/askpass.ts` line 157–158, display the localized labels to the user but return the English "yes"/"no" to SSH. Use a QuickPickItem with a `label` for display and map the selection back to the English literal before returning.
