# Issue #309414 — File write detected `/dev/null`

[https://github.com/microsoft/vscode/issues/309414](https://github.com/microsoft/vscode/issues/309414)

## Summary

The requested /dev/null exception was implemented in PR #276263 (merged November 2025) and shipped in 1.107.0. The commandLineFileWriteAnalyzer now maps /dev/null to a nullDevice symbol and explicitly allows it through auto-approve. Tests confirm the behavior.

## Change summary

Terminal auto-approve should not block commands that write to /dev/null since it is not a real file.

## Best solution

Keep the shipped fix from PR #276263; if commenters still see the issue on latest stable, they should file a new issue with specific reproduction steps (the core /dev/null path is handled, but edge cases like piped commands might need separate treatment).
