# Issue #238943 — Quick open can appear out of view with multiple window sizes

[https://github.com/microsoft/vscode/issues/238943](https://github.com/microsoft/vscode/issues/238943)

## Summary

Still reproduces on main: when the quick pick's stored position ratio is rendered on a smaller window, the pixel left offset is not clamped to the viewport bounds, so part of the widget can render off-screen.

## Change summary

Quick open can appear partially off-screen when a stored drag position from a larger window is restored on a smaller window.

## Best solution

Clamp the computed left (and top) pixel values in the layout else-branch at line 958 of `quickInputController.ts` to `[0, dimension.width - width]`, mirroring the clamping already applied during drag in the `_layout` method.
