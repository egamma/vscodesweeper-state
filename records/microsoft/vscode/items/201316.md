# Issue #201316 — Sort Lines Ascending command messes up trailing commas

[https://github.com/microsoft/vscode/issues/201316](https://github.com/microsoft/vscode/issues/201316)

## Summary

Keep this open under discussion rather than closing as a bug: current Sort Lines Ascending is a generic line-text command, and the requested comma repair would be new language-aware behavior.

## Change summary

Make Sort Lines Ascending reorder comma-separated TypeScript/JavaScript lists without leaving trailing commas attached to the wrong item.

## Best solution

Decide whether VS Code wants a language-aware list-sorting command/code action, likely owned by editor commands or TS/JS language features; the existing generic Sort Lines command should stay predictable unless deliberately expanded.

## Evidence

- **implementation:** src/vs/editor/contrib/linesOperations/browser/sortLinesCommand.ts reads full line contents with model.getLineContent, sorts those raw strings with Intl.Collator, and replaces the selected line range with data.after.join('\n'); it does not inspect language syntax or punctuation.
- **registration:** src/vs/editor/contrib/linesOperations/browser/linesOperations.ts registers editor.action.sortLinesAscending as the generic 'Sort Lines Ascending' editor action.

- **discussion:** A contributor comment notes that the command is not language-aware and suggests external formatters such as Prettier for trailing-comma cleanup.
