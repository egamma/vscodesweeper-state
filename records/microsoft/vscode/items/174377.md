# Issue #174377 — Pressing enter on list view find widget doesn't navigate to next

[https://github.com/microsoft/vscode/issues/174377](https://github.com/microsoft/vscode/issues/174377)

## Summary

Still reproduces on main: Enter in the tree find widget calls `this.tree.domFocus()` (abstractTree.ts:899) instead of navigating to the next match. The commit 7e2a93ab5d9 added Alt+Down/Up for match-to-match navigation as a partial workaround, but the core Enter inconsistency with other find widgets remains unresolved. Labeled under-discussion and involves a UX design tradeoff noted by joaomoreno.

## Change summary

Pressing Enter in the list/tree find widget moves DOM focus to the tree instead of navigating to the next match, unlike other find widgets in VS Code.

## Best solution

Change Enter in FindWidget (abstractTree.ts ~line 894) to navigate to the next find match while keeping focus in the input, consistent with the editor find widget. The challenge joaomoreno noted — list focus not rendering without DOM focus — may now be less relevant given the newer find/filter infrastructure and match highlighting added in 2024.
