# Issue #232199 — Multi-diff editor is not screen reader accessible

[https://github.com/microsoft/vscode/issues/232199](https://github.com/microsoft/vscode/issues/232199)

## Summary

Team-authored accessibility issue for the multi-diff editor — the original side of a diff is not navigable by screen reader. Keeps open per team-author policy; the work aligns with both accessibility commitments and current priorities.

## Change summary

In the multi-diff editor, screen reader users can only access the modified line but cannot navigate to the original (left-side) line of a diff.

## Best solution

Extend the multi-diff editor's accessible view or ARIA tree to expose both sides of each diff hunk so screen reader users can navigate original and modified content. Likely touches `src/vs/workbench/contrib/multiDiffEditor/browser/` and its accessible content provider.
