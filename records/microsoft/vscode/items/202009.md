# Issue #202009 — ListError [codeActionWidget] Invalid start index: 2

[https://github.com/microsoft/vscode/issues/202009](https://github.com/microsoft/vscode/issues/202009)

## Summary

Team-authored error-telemetry issue; the underlying race condition (stale index after an async hover callback) is still present in current main. Keeping open for maintainer judgment.

## Change summary

The code action widget throws a ListError when splice is called with a stale index after an async hover operation completes and the list has been modified concurrently.

## Best solution

Guard the splice at `src/vs/platform/actionWidget/browser/actionList.ts` line ~1957: after the `await this._delegate.onHover(...)`, validate that `e.index` is still within `this._list.length` before calling `this._list.splice(e.index, 1, [element])`. A bounds check or re-lookup of the element's current index would prevent the invalid start index error.
