# Issue #283096 — Error activating the extension in tests of another extension

[https://github.com/microsoft/vscode/issues/283096](https://github.com/microsoft/vscode/issues/283096)

## Summary

Team-authored bug still present on main: `getScopedEnvironmentVariableCollection` calls `this._register()` on a possibly-disposed `UnifiedEnvironmentVariableCollection`, leaking the scoped collection. Keeping open per hard keep-open rule (MEMBER author).

## Change summary

Calling `getEnvironmentVariableCollection` after the terminal service's disposable store has been disposed throws a DisposableStore error and leaks the added object.

## Best solution

Guard `getScopedEnvironmentVariableCollection` against being called after disposal — either check `this._store.isDisposed` before `this._register()`, or prevent the extension host from calling into this path after the terminal service is torn down. The fix is localized to `src/vs/workbench/api/common/extHostTerminalService.ts` around line 1095.
