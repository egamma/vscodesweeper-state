# Issue #324180 — Extension Signature Verification Failed: Vue (Official), Codex & Claude

[https://github.com/microsoft/vscode/issues/324180](https://github.com/microsoft/vscode/issues/324180)

## Summary

NixOS-specific signature verification failure (internal code 159) in the native vsce-sign binary, confirmed by multiple reporters on NixOS. The root cause is in the vsce-sign native binary's interaction with NixOS's non-standard filesystem layout. Already labeled extension-signature and part of a known class of Linux packaging issues.

## Change summary

Extension signature verification fails with UnknownError (internal code 159) on NixOS, blocking all extension installs/updates.

## Best solution

The fix belongs in the vsce-sign native binary (likely a dynamic linker or certificate path issue specific to NixOS's /nix/store layout). The extension-signature team (sandy081) should investigate whether vsce-sign can be made more robust to non-standard Linux environments, or whether NixOS packaging needs to patch the binary's library paths. Workaround: `extensions.verifySignature: false`.
