# Issue #269340 — `MCP: Open Remote User Configuration` open incorrect file

[https://github.com/microsoft/vscode/issues/269340](https://github.com/microsoft/vscode/issues/269340)

## Summary

Still reproduces on main: `OpenRemoteUserMcpResourceCommand` opens the remote profile's per-profile `mcpResource`, but actual MCP configuration reading/writing for USER_REMOTE doesn't use profile-specific paths — it only reads from `environment.settingsPath`. The mismatch means the command opens the wrong file when a non-default profile is active.

## Change summary

The 'MCP: Open Remote User Configuration' command opens a profile-specific mcp.json that is not the file actually read by the MCP configuration system on the remote.

## Best solution

Align the Open command with what the configuration system actually reads. Either make `OpenRemoteUserMcpResourceCommand` always resolve to the default remote profile's `mcpResource` (matching what `RemoteUserConfiguration` reads), or extend `RemoteUserConfiguration`/`configurationEditing.ts` to handle `MCP_CONFIGURATION_KEY` for `USER_REMOTE` the same way `USER_LOCAL` does (profile-aware). The latter is the more correct long-term fix. Likely files: `src/vs/workbench/contrib/mcp/browser/mcpCommands.ts` and `src/vs/workbench/services/configuration/browser/configuration.ts`.
