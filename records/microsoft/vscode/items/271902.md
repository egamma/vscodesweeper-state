# Issue #271902 — Investigate SIGINT from PromptInputModel being incorrectly used in chat terminal tool result

[https://github.com/microsoft/vscode/issues/271902](https://github.com/microsoft/vscode/issues/271902)

## Summary

The spurious exit-130 cascade from SIGINT being sent at idle prompts (the p10k forwarding scenario) was fixed in commit 8eaab1c0893 and shipped in 1.119.0. The fix filters out stale onCommandFinished events and avoids sending Ctrl+C when the prompt is idle, using Ctrl+U instead.

## Change summary

Chat terminal tool incorrectly reports exit code 130 when powerlevel10k forwards SIGINT from a cancelled PromptInputModel interaction to the next command.

## Best solution

Keep the shipped fix (8eaab1c0893, 1.119.0) which filters stale command-finished events and replaces idle-prompt SIGINT with Ctrl+U. No further change needed — reporters should verify on the latest build.
