# Issue #319591 — ${workspaceFolderBasename} variable not assign correctly when .code-workspace is active

[https://github.com/microsoft/vscode/issues/319591](https://github.com/microsoft/vscode/issues/319591)

## Summary

The behavior described is as-designed: in a multi-root workspace, ${workspaceFolderBasename} requires scoping via the `:folderName` syntax (e.g., ${workspaceFolderBasename:MyFolder}). The reporter's suggestion to add a workspace-level property is a feature request for changed semantics, but the current design is intentional and documented.

## Change summary

Request to add a top-level `workspaceFolderBasename` property to .code-workspace files so the variable resolves without folder scoping.

## Best solution

Close as as-designed. The existing scoping syntax `${workspaceFolderBasename:folderName}` already solves the user's problem in multi-root workspaces. The variable is intentionally ambiguous without scoping and VS Code throws a helpful error directing users to the correct syntax.

- **Proposed comment:**

  > Thanks for the report. This is working as designed — in a multi-root workspace, `${workspaceFolderBasename}` is intentionally ambiguous (which folder?), so VS Code requires you to scope it with the folder name using the syntax `${workspaceFolderBasename:FolderName}`.
  > 
  > For example, if your workspace folder is named "🕹️ project files", you'd use `${workspaceFolderBasename:🕹️ project files}`.
  > 
  > See the [Variables Reference documentation](https://code.visualstudio.com/docs/editor/variables-reference#_variables-scoped-per-workspace-folder) for details on scoping variables in multi-root workspaces.
