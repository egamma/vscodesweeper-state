# Issue #247003 — vscode/proxy-agent uses untrusted certificates on macOS

[https://github.com/microsoft/vscode/issues/247003](https://github.com/microsoft/vscode/issues/247003)

## Summary

Still reproduces on main: the bundled @vscode/proxy-agent v0.32.0 reads all macOS keychain certificates via `/usr/bin/security find-certificate -a -p` without checking per-certificate trust settings, so untrusted certificates are included in the CA bundle. A team member (chrmarti) acknowledged the issue and identified potential solutions but no fix has landed.

## Change summary

On macOS, vscode-proxy-agent includes untrusted keychain certificates in the CA bundle because it reads all certs without checking trust settings.

## Best solution

Either filter certificates through `security verify-cert` as chrmarti suggested, or adopt Node.js's new `--use-system-ca` / native OS certificate support (now available in the project's Node 24 baseline) to replace the manual keychain reading in `@vscode/proxy-agent`. The fix belongs in https://github.com/microsoft/vscode-proxy-agent — specifically the `readMacCaCertificates()` function.
