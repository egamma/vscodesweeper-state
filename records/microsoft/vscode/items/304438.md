# Issue #304438 — Copilot CLI "Apply" button appears after all changes are already committed

[https://github.com/microsoft/vscode/issues/304438](https://github.com/microsoft/vscode/issues/304438)

## Summary

The Apply button logic for Copilot CLI worktree handoff lives entirely in the Copilot extension (chatSessionWorktreeServiceImpl.ts). The extension fails to detect that changes are already committed/pushed and still shows the Apply panel — this is a Copilot extension bug, not a VS Code core issue.

## Change summary

Copilot CLI's Apply panel persists after changes have already been committed and pushed, offering to redundantly re-apply them.

## Best solution

File this against the Copilot extension (github/copilot-chat or the built-in copilot extension). The fix belongs in `extensions/copilot/src/extension/chatSessions/vscode-node/chatSessionWorktreeServiceImpl.ts` — the `getWorktreeChanges` or diff-computation logic should detect when the worktree branch's changes are already on the target branch and suppress the Apply UI.
