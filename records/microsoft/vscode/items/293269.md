# Issue #293269 — [Chrome OS] Chinese IME not working with native Chrome OS IME (GTK_IM_MODULE=cros)

[https://github.com/microsoft/vscode/issues/293269](https://github.com/microsoft/vscode/issues/293269)

## Summary

This is an upstream Electron/Chromium issue: the `cros` GTK_IM_MODULE is not supported when running under the Wayland (ozone) backend. VS Code has no special IME-module handling — it delegates entirely to Electron/Chromium. The `--ozone-platform=x11` workaround confirms the root cause is in Chromium's Wayland IME path, not VS Code-specific code.

## Change summary

Chinese IME (GTK_IM_MODULE=cros) does not work in VS Code on Chrome OS Crostini when using the Wayland display protocol.

## Best solution

Track as upstream (Electron/Chromium). The fix belongs in Chromium's Wayland IME integration to properly support the `cros` input method module. In the meantime, the `--ozone-platform=x11` workaround is effective. VS Code could consider documenting this workaround for Chrome OS users, or defaulting to X11 on Chrome OS if feasible.
