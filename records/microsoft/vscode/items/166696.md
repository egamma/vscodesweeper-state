# Issue #166696 — Explorer should show which folders are gitignored when multiple folders are collapsed into one item

[https://github.com/microsoft/vscode/issues/166696](https://github.com/microsoft/vscode/issues/166696)

## Summary

Team-authored feature request for better gitignore visibility in compact (compressed) folder rendering in the Explorer. No implementation found on main; keeping open per hard keep-open rule for team-authored issues.

## Change summary

When multiple folders are compressed into a single Explorer item, individual folders' gitignored status is not visually distinguished — all segments inherit the same decoration.

## Best solution

The compressed tree node rendering in the Explorer (`explorerViewer.ts`) should apply per-segment decorations so that gitignored folders within a compressed path show the dimmed/ignored style while non-ignored parent folders remain normal. This likely involves the `CompressedTreeNodeWrapper` and how decorations are resolved for each URI segment in a compressed node.
