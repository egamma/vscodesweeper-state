# Issue #251780 — Implement Searching Bar as Overlay

[https://github.com/microsoft/vscode/issues/251780](https://github.com/microsoft/vscode/issues/251780)

## Summary

The Find Widget already floats as an overlay (position: absolute). The 'line break' the reporter sees is the intentional view zone added by the `editor.find.addExtraSpaceOnTop` setting (default: true), which prevents the widget from obscuring the first line of code. Setting it to false removes the extra space.

## Change summary

The editor Find Widget pushes content down when opened, and the reporter wants it to float without affecting the code layout.

## Best solution

Close as designed — the Find Widget is already an overlay. The `editor.find.addExtraSpaceOnTop` setting (default `true`) adds a view zone so the first line isn't hidden behind the widget; setting it to `false` removes the extra space. The reporter can use this existing setting to get the behavior they want.
