# Issue #251780 — Implement Searching Bar as Overlay

[https://github.com/microsoft/vscode/issues/251780](https://github.com/microsoft/vscode/issues/251780)

## Summary

Confirmed bug with an assignee (osortega) — the Find Widget's view zone pushes editor content down when `editor.find.addExtraSpaceOnTop` is true (the default). The reporter wants the widget to float as a pure overlay without displacing content. Already triaged and assigned; keeping open.

## Change summary

The editor Find Widget pushes code content down (inserts a view zone) instead of floating as a pure overlay, which disrupts formatted code layout.

## Best solution

Investigate whether the Find Widget can avoid pushing content via the `FindWidgetViewZone` when it overlaps the first line, or improve the behavior so the content shift is less disruptive. The relevant code is in `src/vs/editor/contrib/find/browser/findWidget.ts` (the `FindWidgetViewZone` class and `_showViewZone`/`_layoutViewZone` methods). The setting `editor.find.addExtraSpaceOnTop` already controls this — the workaround is to set it to `false` — but the confirmed bug suggests the default behavior needs refinement.
