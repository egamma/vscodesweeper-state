# Issue #218210 — minimap does not show conflicting changes when the change are empty lines

[https://github.com/microsoft/vscode/issues/218210](https://github.com/microsoft/vscode/issues/218210)

## Summary

Still appears valid on main: when a merge conflict range is empty (one side is a pure insertion/deletion), the minimap gutter decoration may not render because the decoration is placed on a collapsed zero-width range or a line beyond the model content. No fix commit found.

## Change summary

Minimap does not show conflict gutter decorations for merge conflicts where one side has no content (empty line ranges).

## Best solution

In `src/vs/workbench/contrib/mergeEditor/browser/view/editors/baseCodeEditorView.ts`, the decoration range for empty conflict regions (via `toInclusiveRangeOrEmpty()`) produces a point-range at column 1 on a single line. The minimap code handles this correctly in isolation, so the root cause likely involves how `blockIsAfterEnd` interacts with the minimap rendering when the conflict sits at the document boundary, or how the decoration is filtered before reaching the minimap. The fix would ensure minimap gutter decorations always render for conflict ranges regardless of emptiness.
