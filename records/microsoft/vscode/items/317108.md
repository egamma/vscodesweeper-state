# Issue #317108 — vscode_listCodeUsages reports incomplete results to Copilot without a mechanism to warn it the results are incomplete

[https://github.com/microsoft/vscode/issues/317108](https://github.com/microsoft/vscode/issues/317108)

## Summary

Team-authored feature request (hard keep-open). The `IToolResult` interface lacks a mechanism for tools to signal that results are incomplete (e.g. workspace parsing not yet finished), so Copilot cannot know to retry or warn the user. No such mechanism has been added on current main.

## Change summary

The `vscode_listCodeUsages` tool has no way to tell Copilot that its results may be incomplete (e.g. when C/C++ workspace parsing is still in progress), potentially leading to inferior agent responses.

## Best solution

Add an `isIncomplete` or similar field to `IToolResult` (in `src/vs/workbench/contrib/chat/common/tools/languageModelToolsService.ts`) so tool implementations can signal partial results. The Copilot extension's tool harness would then need to handle this — either by retrying the tool call or surfacing the incompleteness to the model. The DevTools extension's deferral approach (via `toolDeferralService.ts` in `extensions/copilot`) is a reference pattern, but a first-class API in `IToolResult` would make it available to all tools uniformly.
