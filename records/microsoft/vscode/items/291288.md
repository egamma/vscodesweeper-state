# Issue #291288 — Read folder uses search icon, read file uses book icon?

[https://github.com/microsoft/vscode/issues/291288](https://github.com/microsoft/vscode/issues/291288)

## Summary

Still reproduces on main: the `getToolInvocationIcon` function assigns `Codicon.search` to folder-listing tools (via `lowerToolId.includes('list')`) and `Codicon.book` to file-reading tools (via `lowerToolId.includes('read')`), producing inconsistent icons for two conceptually similar read operations. A small alignment fix is warranted.

## Change summary

The thinking-widget icon for 'read folder' (list_dir) shows a search icon while 'read file' shows a book icon; they should use the same icon since both are read operations.

## Best solution

Unify the icon for read-family tools in `getToolInvocationIcon` (src/vs/workbench/contrib/chat/browser/widget/chatContentParts/chatThinkingContentPart.ts) — either move `list_dir`/`listDir` into the read/book branch or give both a shared 'file-read' icon. A targeted one-line change to the match order or conditions.
