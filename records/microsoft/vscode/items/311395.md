# Issue #311395 — Unable to view diff after reloading

[https://github.com/microsoft/vscode/issues/311395](https://github.com/microsoft/vscode/issues/311395)

## Summary

Team-authored bug report (hard keep-open). The git file system provider cannot resolve a git:/ URI referencing a specific commit after window reload in an agents/worktree session, likely because the git extension's file system provider doesn't survive reload for that repository context.

## Change summary

After reloading the window, viewing a diff in a worktree CLI session fails because the git:/ URI (with a commit ref) cannot be resolved.

## Best solution

The git extension's file system provider needs to handle the case where a diff editor is restored after reload — either by re-registering the worktree's repository context on reload, or by gracefully handling the missing ref. Likely involves `extensions/git/src/` (the git file system provider) and possibly the sessions layer's diff editor restoration logic.
