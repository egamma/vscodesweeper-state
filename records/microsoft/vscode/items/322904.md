# Issue #322904 — Extensions Auto Update Delay not applied by GPO

[https://github.com/microsoft/vscode/issues/322904](https://github.com/microsoft/vscode/issues/322904)

## Summary

Confirmed bug in the @vscode/policy-watcher native module (microsoft/vscode-policy-watcher): NumberPolicy.cc only accepts REG_QWORD but the ADMX template generates <decimal>, which Windows writes as REG_DWORD. Multiple reporters confirm, and one identified the exact root cause and workaround (using QWORD). Keeping open as an upstream dependency issue.

## Change summary

The ExtensionsAutoUpdateDelay GPO policy is ignored because the native policy reader rejects the REG_DWORD registry type that Windows writes for ADMX decimal values.

## Best solution

Fix in microsoft/vscode-policy-watcher: NumberPolicy.cc should accept both REG_DWORD and REG_QWORD in its supported types set. Alternatively, VS Code's ADMX generation could emit a longDecimal instead of decimal, but fixing the native reader is the correct approach since REG_DWORD is the standard type for GPO decimal values.
