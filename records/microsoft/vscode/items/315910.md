# Issue #315910 — Integrated terminal restores incorrect height when VS Code is launched via code command in fullscreen

[https://github.com/microsoft/vscode/issues/315910](https://github.com/microsoft/vscode/issues/315910)

## Summary

The panel maximized state (`PANEL_WAS_LAST_MAXIMIZED`) is stored with `StorageScope.WORKSPACE`, so opening a new window via `code` (a fresh workspace context) defaults to non-maximized. This is likely by-design per-workspace scoping, but the reporter expects it to carry over. Routing to the workbench area for a maintainer to decide if this is intentional or if the state should be profile-scoped.

## Change summary

Terminal panel does not restore to maximized height when a new VS Code window is opened via the `code` CLI command.

## Best solution

A maintainer should decide whether `PANEL_WAS_LAST_MAXIMIZED` should remain workspace-scoped or be promoted to `StorageScope.PROFILE` (like `PANEL_LAST_NON_MAXIMIZED_HEIGHT` already is). If workspace-scoping is intentional, close as `*as-designed` and point the user to `workbench.panel.opensMaximized: always`. If it's a gap, change the storage scope.
