# Issue #140915 — explorer.sortOrder default doesn't work well for hashes

[https://github.com/microsoft/vscode/issues/140915](https://github.com/microsoft/vscode/issues/140915)

## Summary

Valid feature request for a sort option that avoids numeric collation on hash-like filenames. The existing `unicode` option disables numeric sorting but also changes case sensitivity, which isn't what the reporter wants. Already labeled `help wanted` in Backlog — keeping open as a legitimate enhancement.

## Change summary

Explorer default sort order treats hex-hash filenames as numbers, breaking alphabetical order for files like PNG test suite hashes.

## Best solution

Add a new `sortOrderLexicographicOptions` variant (or a separate setting) that uses case-insensitive collation without `numeric: true`, giving alphabetical sorting that still treats `a` and `A` the same. The fix lives in `src/vs/base/common/comparers.ts` (new collator without `numeric`) and `src/vs/workbench/contrib/files/browser/views/explorerViewer.ts` (wire it up). Related to but distinct from https://github.com/microsoft/vscode/issues/27759.
