# Issue #264028 — Git - Stage Selected Ranges problem

[https://github.com/microsoft/vscode/issues/264028](https://github.com/microsoft/vscode/issues/264028)

## Summary

Still present on main — the `intersectDiffWithRange` heuristic in `extensions/git/src/staging.ts` (lines 134–141) keeps the full original range when original and modified line counts differ, which can cause unselected original lines to be included in the staged patch. Milestone is 'On Deck', so keeping open per scheduling rule.

## Change summary

"Git: Stage Selected Ranges" stages more changes than the user selected when the diff hunk has a different number of original and modified lines.

## Best solution

Fix the else-branch of `intersectDiffWithRange` in `extensions/git/src/staging.ts` (lines 134–141) to proportionally clip the original range to match the selected modified sub-range, rather than using the full original range. The existing test file at `extensions/git/src/test/staging.test.ts` (if present) or a new test should verify that only the selected sub-range of a mixed insertion/deletion hunk is staged.
