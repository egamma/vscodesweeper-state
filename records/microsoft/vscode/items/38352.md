# Issue #38352 — Autoclosing pairs should be configurable

[https://github.com/microsoft/vscode/issues/38352](https://github.com/microsoft/vscode/issues/38352)

## Summary

Well-established feature request (147 upvotes, milestoned in Backlog) asking users to configure which auto-closing pairs apply per language — still unimplemented on main. Current settings only control *when* pairs fire, not *which* pairs exist.

## Change summary

Allow users to add, remove, or override language-defined auto-closing pairs via settings.json.

## Best solution

Expose a per-language user setting (e.g. `editor.autoClosingPairs`) that merges with or overrides the extension-contributed `language-configuration.json` pairs. Related canonical tracking: https://github.com/microsoft/vscode/issues/102574 (user settings for language configurations) covers the broader surface but this issue is the specific, high-signal tracker for the auto-closing subset.
