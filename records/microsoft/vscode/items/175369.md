## Summary

Still reproduces on main: the onDidOptionChange handler in searchWidget.ts (line 570) guards with `if (!viaKeyboard)`, so keyboard-triggered Preserve Case toggles never fire `_onPreserveCaseChange` and the search model ignores the state change. The linked fix PR was closed without merging.

## Change summary

Toggling Preserve Case via keyboard shortcut or Enter key visually activates the toggle but does not update the search model, so replacements ignore the setting.

## Best solution

Remove or invert the `if (!viaKeyboard)` guard on the `onDidOptionChange` handler in `src/vs/workbench/contrib/search/browser/searchWidget.ts` (around line 570) so that `_onPreserveCaseChange` fires for both keyboard and mouse toggles. The original PR #175373 attempted this but was closed without merge; its approach can be revisited.
