# Issue #175369 — Preserve Case toggle in Search does not react correctly to keyboard events

[https://github.com/microsoft/vscode/issues/175369](https://github.com/microsoft/vscode/issues/175369)

## Summary

Still reproduces on main: the `if (!viaKeyboard)` guard in `searchWidget.ts` line 571 blocks the `_onPreserveCaseChange` event for keyboard-triggered toggles, so `viewModel.preserveCase` is never updated when using the keyboard shortcut — replace operations silently ignore the Preserve Case state.

## Change summary

Toggling Preserve Case via keyboard shortcut (Cmd+Opt+P / Alt+P) visually activates the toggle but does not update the viewModel, so replace operations ignore the preserve-case setting.

## Best solution

Remove or invert the `if (!viaKeyboard)` guard in `searchWidget.ts` at the `onDidOptionChange` handler so that `_onPreserveCaseChange` fires regardless of input method. The keyboard-shortcut command path (`togglePreserveCase()` in `searchView.ts`) already calls `triggerQueryChange`, but that only re-runs the search — the replace preview depends on `viewModel.preserveCase`, which is only set via `onPreserveCaseChange`. The guard should be removed so the event fires for both mouse and keyboard.
