# Issue #165165 — Support custom "reverse requests" from debug adapters

[https://github.com/microsoft/vscode/issues/165165](https://github.com/microsoft/vscode/issues/165165)

## Summary

Still a valid, unimplemented API gap. Custom reverse requests from debug adapters are explicitly rejected in current main (rawDebugSession.ts line 723–726 returns an error for unknown request commands). A team member agreed it would be useful, and it already carries appropriate labels (api-proposal, under-discussion). Keeping open for continued discussion and potential API proposal work.

## Change summary

Allow extensions to register handlers for custom DAP reverse requests from debug adapters, analogous to the existing onDidReceiveDebugSessionCustomEvent for custom events.

## Best solution

Add a proposed extension API (e.g. a registerDebugSessionCustomRequestHandler or similar) that lets extensions intercept and respond to non-standard reverse requests in rawDebugSession.ts's dispatchRequest, instead of returning 'unknown request'. The api-proposal label is already in place; the next step is an API design discussion and a proposed-API .d.ts file.
