# Issue #185957 — Moved code UI is bit busy and full of lines and boxes

[https://github.com/microsoft/vscode/issues/185957](https://github.com/microsoft/vscode/issues/185957)

## Summary

The visual busyness concern raised here remains unaddressed on current main — the moved blocks feature still uses border-based boxes and connector lines without a background color option or the gradient/flat styling jrieken proposed. The related issue #187432 was closed with partial improvements that didn't satisfy the original request (comment 5). An open PR #319846 proposes configurable moved-code colors, which partially addresses this.

## Change summary

The moved code detection UI in the diff editor is too visually busy with boxes, frames, and connecting lines that clash with VS Code's flat design language.

## Best solution

Simplify the moved-blocks visual treatment: replace or reduce the border/box decorations with subtler background coloring and a cleaner connector line (possibly a gradient). The open PR https://github.com/microsoft/vscode/pull/319846 addresses part of this (configurable colors) but the core concern is the visual weight of the boxes themselves in `src/vs/editor/browser/widget/diffEditor/features/movedBlocksLinesFeature.ts`.
