# Issue #184829 — JSX Linked Editing not working with Vim extension

[https://github.com/microsoft/vscode/issues/184829](https://github.com/microsoft/vscode/issues/184829)

## Summary

Still reproduces on main. The linked editing sync logic at line 162 of linkedEditing.ts checks whether changes intersect the reference range, but when the entire word is deleted (Vim `cw` or backspacing the whole tag name), the range collapses and subsequent typing triggers a full range re-query instead of syncing the mirror ranges.

## Change summary

JSX linked editing (tag rename sync) breaks when the entire tag word is deleted before retyping, as Vim's change-word does.

## Best solution

The `onDidChangeModelContent` handler in `LinkedEditingContribution.reinitialize` should handle the case where a deletion collapses the reference range to zero-width: keep the synced-region session active and continue syncing subsequent insertions at that position, rather than falling through to `triggerRangeUpdate()` which re-queries the provider on now-empty content.
