# Issue #18064 — Allow extension commands to define the behaviour when Alt modifier is pressed

[https://github.com/microsoft/vscode/issues/18064](https://github.com/microsoft/vscode/issues/18064)

## Summary

Still a valid, unimplemented API request with 27 upvotes (above the 20-vote Backlog threshold). The internal QuickInput infrastructure already tracks keyMods (IKeyMods with ctrlCmd/alt/shift) but this is not exposed to the extension API — onDidAccept remains Event<void>.

## Change summary

Allow extensions to detect modifier keys (Ctrl, Alt, Shift) when a QuickPick item is accepted, enabling alternative behaviors like 'open in new window'.

## Best solution

Expose keyMods in the extension QuickPick API — either by changing onDidAccept to pass an event object containing modifier state, or by adding a readable keyMods property on QuickPick. The internal plumbing (IKeyMods, onKeyMods) already exists in src/vs/platform/quickinput/common/quickInput.ts; the gap is only the public vscode.d.ts surface and the extension host proxy layer.
