# Issue #312749 — Can't copy text in integrated terminal on Safari (Code Server Web on VM)

[https://github.com/microsoft/vscode/issues/312749](https://github.com/microsoft/vscode/issues/312749)

## Summary

Confirmed on current main: on Safari over insecure origins (HTTP), the WebKit clipboard workaround intercepts writeText and returns early, preventing the execCommand fallback from firing. The reporter has identified the exact issue and drafted a fix.

## Change summary

Copying text from the integrated terminal fails on Safari when VS Code Web is served over HTTP (insecure origin) because the WebKit clipboard workaround blocks the fallback path.

## Best solution

Guard the WebKit workaround installation (line 35–37 of clipboardService.ts) so it only activates when `navigator.clipboard` is actually available (i.e., on secure contexts), or adjust the `writeText` early-return at line 134 to fall through to the execCommand fallback when the clipboard.write() call fails. The reporter has a draft commit ready for a PR.
