# Issue #286102 — Reopen Closed Repositories > All Repositories does not reopen worktrees

[https://github.com/microsoft/vscode/issues/286102](https://github.com/microsoft/vscode/issues/286102)

## Summary

Team-authored bug with 'On Deck' milestone — hard keep-open. The defect appears to still exist on main: when 'All Repositories' is selected, `openRepository` resolves each worktree's git root (which may resolve to the main repo), causing the existing-repo early-return to skip reopening the worktree.

## Change summary

Selecting 'All Repositories' in 'Reopen Closed Repositories' does not reopen git worktrees that were previously closed.

## Best solution

Fix the `reopenClosedRepositories` command or `openRepository` in `extensions/git/src/model.ts` so that worktree paths are preserved through the reopen flow rather than being resolved to the main repository root, which triggers the duplicate-repository guard.
