# Issue #319418 — GitHub Copilot sees "compilerError" for diagnostics instead of the "C/C++ IntelliSense" diagnostic source

[https://github.com/microsoft/vscode/issues/319418](https://github.com/microsoft/vscode/issues/319418)

## Summary

Team-authored issue; the defect is confirmed on current main — the Tag at line 94 of diagnosticsContext.tsx hardcodes 'compileError' and never includes the diagnostic source, misleading Copilot about the origin of diagnostics from extensions like C/C++.

## Change summary

Copilot's inline diagnostics context labels all diagnostics as 'compileError' and omits the vscode.Diagnostic.source, causing Copilot to misidentify extension-reported IntelliSense errors as compiler errors.

## Best solution

Include `d.source` in the prompt context (e.g. as a Tag attribute or separate field), rename the 'compileError' tag to something neutral like 'diagnostic' or 'problem', and consider an opt-out mechanism (e.g. a `copilotVisibility` property on Diagnostic or a new API) so extensions can hide known-noisy diagnostics from Copilot. Work lives in `extensions/copilot/src/extension/prompts/node/inline/diagnosticsContext.tsx` and potentially the VS Code Diagnostic API surface.
