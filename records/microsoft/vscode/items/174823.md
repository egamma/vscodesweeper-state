# Issue #174823 — Workspace Trust - should be made accessible more easily.

[https://github.com/microsoft/vscode/issues/174823](https://github.com/microsoft/vscode/issues/174823)

## Summary

Valid UX feature request: easier access to the Workspace Trust management page when the workspace is already trusted. Current main still only shows the shield icon for untrusted workspaces and hides it once trusted — the gap remains. Low traction (0 upvotes in 2+ years) but already classified as under-discussion, which is appropriate.

## Change summary

Request to show a persistent Workspace Trust indicator in the status bar and/or a menu item even when the workspace is already trusted, so users can easily re-access the management page.

## Best solution

Keep open as under-discussion. The status bar entry is intentionally cleared for trusted workspaces (see `updateStatusbarEntry` in `src/vs/workbench/contrib/workspace/browser/workspace.contribution.ts`). A related issue https://github.com/microsoft/vscode/issues/170664 asks for the ability to pin the trust icon — if that lands, it would partially address this request. The remaining ask (a settings-menu item) is a small UX addition. Consider consolidating with #170664 if the team agrees they are close enough.
