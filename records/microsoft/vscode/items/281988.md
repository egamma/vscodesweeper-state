# Issue #281988 — Git - Commit Message Doesn't Overwrite User Message

[https://github.com/microsoft/vscode/issues/281988](https://github.com/microsoft/vscode/issues/281988)

## Summary

Valid feature request to preserve user-typed commit message prefixes when generating commit messages with Copilot. This is the canonical issue (https://github.com/microsoft/vscode/issues/248123 was closed as a duplicate of it). No evidence of implementation on current main; keeping open in its Backlog milestone.

## Change summary

When generating a commit message with Copilot, existing user-typed text in the commit input box is overwritten instead of being preserved and extended.

## Best solution

Modify the commit message generation flow (likely in the Copilot extension's gitCommitMessageGenerator and the SCM input widget integration) to detect existing text in the input box and either append the generated message after it or use the existing text as context/prefix for generation. This supports conventional-commit workflows where users type a prefix like `fix:` before generating.
