# Issue #277185 — Consider option for Go To File.../Quick Open not to scroll to symbols in unfocused editors / splits

[https://github.com/microsoft/vscode/issues/277185](https://github.com/microsoft/vscode/issues/277185)

## Summary

Still unfixed on main: the Quick Open picker uses `revealIfOpened: true` which scrolls editors in other groups without restoring their view state. PR #288041 attempted a fix but was closed without merging. The team agreed on Solution C (restore view states on picker close) but it hasn't landed yet.

## Change summary

Go To File / Quick Open scrolls editors in unfocused splits to arbitrary symbols while typing, losing the user's place, and doesn't restore the view state when the picker closes.

## Best solution

Land the approach from the closed PR #288041 (restore view states of revealed editors in other groups when Quick Open closes) — the fix touches `src/vs/workbench/browser/quickaccess.ts` and `src/vs/workbench/contrib/search/browser/anythingQuickAccess.ts`. The team already agreed on Solution C in the discussion.
