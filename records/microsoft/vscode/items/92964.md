## Summary

Issue is still present on main — linked editing creates separate undo stops for the primary edit and the synced-region edit. Milestone is 'On Deck' (scheduling decision), so keeping open.

## Change summary

Option-delete or paste in a linked editing region (rename on type) creates 2 undo stops instead of 1, requiring two undos to restore original content.

## Best solution

The linked editing contribution should group both the primary edit and the synced-region edit into a single undo stop. The fix likely lives in `src/vs/editor/contrib/linkedEditing/browser/linkedEditing.ts` where edits are pushed to the model — they should share a single undo group.
