# Issue #92964 — [rename on type] option-delete and pasting creates 2 undo stops

[https://github.com/microsoft/vscode/issues/92964](https://github.com/microsoft/vscode/issues/92964)

## Summary

The undo-grouping issue in linked editing (synced regions) still appears present on current main — `popUndoStop()` before `executeEdits` at line 243 of linkedEditing.ts doesn't fully handle multi-character operations like option-delete or paste. The bug is real but low-impact; keeping open.

## Change summary

Option-delete or paste inside a linked editing range creates two undo stops instead of one, requiring two Cmd+Z presses to restore the original content.

## Best solution

Fix the undo grouping in `src/vs/editor/contrib/linkedEditing/browser/linkedEditing.ts` around the `_syncRanges` method (lines 242–250) so that the mirror edit and the triggering edit (option-delete, paste) are merged into a single undo stop. This likely requires ensuring `popUndoStop` correctly collapses the undo stack for non-typing edit operations, or using a different undo-grouping strategy for linked edits.
