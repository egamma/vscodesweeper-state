# Issue #253643 — Unable to setup multiple ports to forward

[https://github.com/microsoft/vscode/issues/253643](https://github.com/microsoft/vscode/issues/253643)

## Summary

Still reproduces on main: the race condition in tunnel-forwarding port updates remains unfixed. PR #259930 was closed without merging, and the current source has no debounce guard on `updateActivePortsIfRunning()`.

## Change summary

Changing port visibility or protocol for one forwarded port reverts settings on another port due to a race condition in the tunnel forwarding extension.

## Best solution

Add a debounce to `updateActivePortsIfRunning()` in `extensions/tunnel-forwarding/src/extension.ts` so that dispose+create cycles complete before sending the port list to the CLI process. The closed PR #259930 had the right idea but was never merged.
