# Issue #191369 — Terminal title on Windows may not change when switched shortly after creating

[https://github.com/microsoft/vscode/issues/191369](https://github.com/microsoft/vscode/issues/191369)

## Summary

The race condition around terminal title updates on Windows appears to still exist on main — the title listener is attached asynchronously via setTimeout (terminalInstance.ts:1513), and no commit directly addresses the reported scenario of quick input after creation. Keeping open as a valid low-severity bug.

## Change summary

Terminal tab title on Windows may not reflect the running process when a command is typed quickly after terminal creation.

## Best solution

Tighten the async title-listener setup in `terminalInstance.ts` so that title-change events fired during the `setTimeout` gap are not lost. The fix likely lives in the `_setupProcessListeners` method around line 1511–1519.
