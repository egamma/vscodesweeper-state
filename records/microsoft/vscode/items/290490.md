Team-authored tracking issue for chat session performance improvements (tree re-rendering, markdown rendering costs, retained ChatModels). Hard keep-open: authored by a team member.

## Change summary

Chat session rendering has multiple perf inefficiencies: unnecessary full tree re-renders, slow markdown rendering for simple strings, and ChatModel retention after disposal.

## Best solution

Work through the listed optimizations: observe chat viewmodel changes for incremental tree updates instead of full re-renders, skip markdown rendering for plain strings, audit markdown renderer feature flags, and fix ChatModel disposal leaks. These are performance debt items owned by the chat area team.
