# Issue #277735 — Agent: Prompts should not persist tool/model selection after chat response

[https://github.com/microsoft/vscode/issues/277735](https://github.com/microsoft/vscode/issues/277735)

## Summary

Team-authored issue on the 'On Deck' milestone describing desired prompt-file lifecycle behavior (tools/model selection should not persist after a prompt completes). Multiple related issues confirm community demand. Keeping open per policy.

## Change summary

Agent prompt files persist tool and model selection after their chat response completes, when they should be cleaned up (one-shot semantics).

## Best solution

Ensure that tool/model selections activated by a prompt file are scoped to that single chat turn and reverted once the response completes, reinforcing the one-shot prompt vs. multi-turn agent distinction.
