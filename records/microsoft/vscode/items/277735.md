# Issue #277735 — Agent: Prompts should not persist tool/model selection after chat response

[https://github.com/microsoft/vscode/issues/277735](https://github.com/microsoft/vscode/issues/277735)

## Summary

Team-authored issue on 'On Deck' milestone requesting that prompt files scope their tool/model selection to a single turn rather than persisting. Multiple related community issues (https://github.com/microsoft/vscode/issues/290363, https://github.com/microsoft/vscode/issues/307602) were closed as duplicates or info-needed pointing back here. The current `scopedTools` derived in chatWidget.ts still snapshots tools per-session, not per-turn — the requested one-shot scoping for prompts (vs. multi-turn for custom agents) does not appear implemented on main.

## Change summary

Prompt files that specify tools or model should scope those selections to the single chat turn, not persist them for subsequent interactions.

## Best solution

Implement per-turn tool/model scoping for prompt files in the chat widget's request pipeline — likely in `chatWidget.ts` (`scopedTools` derived) and wherever prompt-file tool selections are applied — so that after the prompt's response completes, the tool/model state resets to the user's prior selection. This distinguishes prompts (one-shot) from custom agents (multi-turn).
