# Issue #237087 — [Accessibility, Mouse]: When using the mouse to navigate a menu, the content behind the menu is reported to the screen reader.

[https://github.com/microsoft/vscode/issues/237087](https://github.com/microsoft/vscode/issues/237087)

## Summary

The reporter's pushback against the duplicate close is valid — this is about menu overlays not blocking screen reader mouse-tracking from reading background content, which is distinct from #224704 (Electron mouse hover regression). However, the scenario is niche (NVDA mouse tracking + third-party add-on) and reproduction details are thin.

## Change summary

When using NVDA mouse tracking to navigate VS Code menus, content behind the open menu is reported to the screen reader.

## Best solution

Investigate whether adding `inert` to the workbench container (or `aria-modal` on the menu overlay) when a context/menubar menu is open would prevent mouse-tracking screen readers from reading background content. The fix would likely live in `src/vs/base/browser/ui/contextview/` or `src/vs/base/browser/ui/menu/menubar.ts`. Needs confirmation from the accessibility team on whether this NVDA mouse-tracking scenario is in scope.
