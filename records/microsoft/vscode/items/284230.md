# Issue #284230 — Agent sessions keeps asking for input even when not needed after restart

[https://github.com/microsoft/vscode/issues/284230](https://github.com/microsoft/vscode/issues/284230)

## Summary

Acknowledged, unfixed bug: after a reload, a chat session that had a pending tool-call confirmation still shows 'input needed' but the confirmation is lost, producing an error. A team member (connor4312) explained the root cause and left it open intentionally — no fix has landed on main.

## Change summary

After restarting VS Code, agent sessions with pending tool-call confirmations still indicate input is needed, but the confirmation is gone and an error is shown instead.

## Best solution

Fix the hydration path so that serialized sessions with pending tool-call confirmations either re-present the confirmation after reload or gracefully clear the 'input needed' state. connor4312 noted this is tied to a larger refactoring to support confirming tool calls across reloads; the interim fix would be to detect unresolvable pending confirmations on hydration and mark them as cancelled/failed rather than leaving them in a broken 'needs input' state.
