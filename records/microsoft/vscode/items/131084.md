# Issue #131084 — Search files by name is very slow even for cases that could be optimised

[https://github.com/microsoft/vscode/issues/131084](https://github.com/microsoft/vscode/issues/131084)

## Summary

The relative path resolution feature exists in current main (getRelativePathFileResults, line 769), but unlike absolute path matches which short-circuit the entire file search (line 578-593), relative path matches still run in parallel with the full file search and don't provide an early-exit optimization. The enhancement request for short-circuiting on relative path matches remains partially unaddressed.

## Change summary

When pasting a relative path like 'foo/bar/file.php' into Quick Open, VS Code should resolve it against workspace folders first and return immediately instead of running the full slow file search.

## Best solution

Mirror the absolute-path short-circuit pattern (lines 578-593) for relative path matches: when getRelativePathFileResults resolves to a file, return it immediately as the sole result in getFilePicks without waiting for the full file search. This would be a small change in src/vs/workbench/contrib/search/browser/anythingQuickAccess.ts.
