# Issue #317500 — Language model per-model settings are saved to the wrong provider group when multiple groups share the same vendor

[https://github.com/microsoft/vscode/issues/317500](https://github.com/microsoft/vscode/issues/317500)

## Summary

Already fixed on main: commit fa3b663f723 (PR #322974) added model→group resolution so setModelConfiguration writes to the group that actually defines the model, not just the first vendor match. Not yet in a released build.

## Change summary

Per-model settings (e.g. thinkingEffort) are saved to the wrong provider group when multiple groups share the same vendor, because the fallback picks the first vendor match.

## Best solution

Keep the shipped fix in commit fa3b663f723; no further change needed. The reporter should verify on the next Insiders build that includes this commit.
