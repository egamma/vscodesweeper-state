# Issue #313703 — Opening a chat session can silently dirty editor buffers and cause data loss

[https://github.com/microsoft/vscode/issues/313703](https://github.com/microsoft/vscode/issues/313703)

## Summary

Still reproduces on main: the unconditional restore in resetDocumentValues has no on-disk content check. Team-authored, milestoned On Deck, with an open fix PR — keeping open.

## Change summary

Opening a chat session can silently restore stale agent edits into editor buffers, overwriting newer on-disk content and causing data loss.

## Best solution

Land the open PR https://github.com/microsoft/vscode/pull/313793 which adds mtime/hash guards to the restore path, or implement an equivalent check in chatEditingTextModelChangeService.ts before calling resetDocumentValues.
