# Issue #177747 — Notebook Search: regex enabled on default toggle state

[https://github.com/microsoft/vscode/issues/177747](https://github.com/microsoft/vscode/issues/177747)

## Summary

Still reproduces on main: notebook find passes `regex: true` to the text-model search but the webview output search (line 2631 of notebookEditorWidget.ts) only forwards `caseSensitive` and `wholeWord` — regex is silently ignored for output matches. Team-authored issue on 'On Deck' milestone; keeping open.

## Change summary

Notebook search default state enables regex, but regex doesn't actually apply to output searches because the webview find path ignores the regex flag.

## Best solution

Either disable regex in the default notebook find toggle state when outputs are included, or clearly communicate that regex doesn't apply to output results (the webview uses `window.find()` which has no regex support). The related issue https://github.com/microsoft/vscode/issues/183882 already acknowledged this limitation for formatted text. A proper fix would either disable the regex toggle when output search is active, or find an alternative to `window.find()` for output search.
