# Issue #177747 — Notebook Search: regex enabled on default toggle state

[https://github.com/microsoft/vscode/issues/177747](https://github.com/microsoft/vscode/issues/177747)

## Summary

Team-authored confirmed bug — notebook output search uses the browser's `window.find()` API in the webview, which does not support regex, so enabling regex by default creates a misleading UX where regex appears active but silently doesn't apply to output matches. Still present on main.

## Change summary

When notebook search has regex enabled (the default toggle state), output matches via the webview's `window.find()` ignore the regex flag, giving incorrect results.

## Best solution

Either disable the regex toggle when output search is active (with a clear indicator explaining why), or implement regex-capable output searching in the webview layer. The related closed issue https://github.com/microsoft/vscode/issues/183882 proposed expressing this limitation to the user — that approach could be revisited. Key files: `src/vs/workbench/contrib/notebook/browser/contrib/find/findModel.ts` and the webview-side find implementation.
