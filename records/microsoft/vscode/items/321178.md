# Issue #321178 — @restartrequired tag "Show" link doesn't refresh when new extensions are updated

[https://github.com/microsoft/vscode/issues/321178](https://github.com/microsoft/vscode/issues/321178)

## Summary

Still reproduces on main: the 'Show' link calls `search(query)` which short-circuits when the search box already holds the same value, so newly-added restart-required extensions aren't displayed until the user manually clears and re-clicks.

## Change summary

Clicking the 'Show' link for restart-required extensions doesn't refresh the list when the search box already contains `@restartrequired`.

## Best solution

In `ExtensionsViewlet.search()`, when the value is already set, still call `this.triggerSearch()` (or `doSearch(true)`) so the view refreshes with the current set of restart-required extensions. The fix is localized to `src/vs/workbench/contrib/extensions/browser/extensionsViewlet.ts` around line 751.
