# Issue #174791 — Tasks interfering despite sequence order

[https://github.com/microsoft/vscode/issues/174791](https://github.com/microsoft/vscode/issues/174791)

## Summary

Still reproduces on main: tasks using `${command:...}` as the command resolve instantly because the editor command is fired during variable resolution, making `dependsOrder: "sequence"` ineffective for sequential editor commands.

## Change summary

Tasks with `dependsOrder: "sequence"` don't properly await completion of `${command:...}` editor commands before starting the next dependent task.

## Best solution

The task system needs to distinguish between variable-resolution commands (whose result is consumed as a string) and fire-and-forget editor commands used as the task's sole action. For the latter pattern, the task should await the command's returned promise before signaling completion. Likely fix in `src/vs/workbench/contrib/tasks/browser/terminalTaskSystem.ts` around the empty-task early-exit path.
