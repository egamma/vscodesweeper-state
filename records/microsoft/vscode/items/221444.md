# Issue #221444 — Minor indentation inconsistency - trimAutoWhitespace - difference between tab and shift-tab

[https://github.com/microsoft/vscode/issues/221444](https://github.com/microsoft/vscode/issues/221444)

## Summary

Still reproduces on main: the ShiftCommand (outdent/shift-tab) does not set `insertsAutoWhitespace`, so the resulting indentation is not marked as auto-whitespace and won't be trimmed on the next edit. Tab (indent) correctly marks it via ReplaceCommand.

## Change summary

Shift-Tab (outdent) revokes auto-whitespace status of a line's indentation, causing trimAutoWhitespace to not trim it, while Tab (indent) preserves the auto-whitespace marking.

## Best solution

Add `insertsAutoWhitespace` support to `ShiftCommand` (in `src/vs/editor/common/commands/shiftCommand.ts`) so that outdent operations preserve auto-whitespace status the same way indent operations do via `ReplaceCommand`.
