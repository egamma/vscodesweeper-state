# Issue #149175 — Problems without column information are always shown as column 1,1

[https://github.com/microsoft/vscode/issues/149175](https://github.com/microsoft/vscode/issues/149175)

## Summary

Milestone is 'On Deck' (a real scheduling milestone), triggering the hard keep-open rule. The discussion between team members ended without a clear resolution on how to represent 'no column' in the marker model.

## Change summary

When a task problem matcher produces no column, the Problems view always displays column 1,1 instead of indicating the whole line.

## Best solution

Resolve the design question raised by jrieken: task problem matchers without a column match should set a large end-column (or a sentinel) so the marker spans the whole line, and the Problems view / editor should clamp to actual line length. The fix likely touches the task problem matcher in `src/vs/workbench/contrib/tasks/` and possibly the marker service consumers.
