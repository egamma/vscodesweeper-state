Team-authored issue (hard keep-open). The core ask — moving notifications away from the chat input — is already addressed by the configurable notification positions feature shipped in 1.110 (`workbench.notifications.position` setting with bottom-left and top-right options). ## Change summary

Notifications overlay the Copilot chat input box, blocking typing when multiple notifications fire simultaneously.

## Best solution

The `workbench.notifications.position` setting (shipped in 1.110 via commit ff21980e60c) lets users move notifications to bottom-left or top-right, which resolves the overlap with chat input. A maintainer should confirm this addresses the request and close the issue as completed.
