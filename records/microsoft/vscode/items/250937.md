# Issue #250937 — The diff mod is slow, maybe infinite loop.

[https://github.com/microsoft/vscode/issues/250937](https://github.com/microsoft/vscode/issues/250937)

## Summary

Still reproducible on main: the default Myers diff algorithm checks the timeout only once per edit-distance increment, so pathological inputs can far exceed the 5-second timeout before the check fires. No fix has landed for this issue.

## Change summary

Opening a diff between two specific files causes the diff editor to hang for 2+ minutes instead of respecting the 5-second timeout.

## Best solution

Make the timeout check more granular in the Myers inner loop (src/vs/editor/common/diff/defaultLinesDiffComputer/algorithms/myersDiffAlgorithm.ts), or switch the default algorithm to the external @vscode/diff package which was adopted in commit 5fc781f799f but is not yet the default.
