# Issue #283507 — run_in_terminal fails with Oh My Zsh RPS1 (right-side prompt) - returns only prompt instead of command output CAUSE FOUND

[https://github.com/microsoft/vscode/issues/283507](https://github.com/microsoft/vscode/issues/283507)

## Summary

The reported false-positive prompt detection with Oh My Zsh RPS1 remains a valid gap. The `detectsCommonPromptPattern` regex table in `executeStrategy.ts` can match decorated prompt lines that include right-side prompt content (e.g. exit codes, special characters). Multiple fixes have landed since the report but none specifically address RPS1 right-side prompt interference. Related issue #309370 acknowledges the fragility.

## Change summary

The `run_in_terminal` tool returns only the prompt (no command output) when Oh My Zsh's RPS1 right-side prompt displays exit codes, because the prompt-detection regex falsely matches the decorated line.

## Best solution

The prompt detection regex table in `src/vs/workbench/contrib/terminalContrib/chatAgentTools/browser/executeStrategy/executeStrategy.ts` needs to account for right-side prompt content (RPS1 decorations) that may be appended after the actual prompt marker. The consolidation tracked by https://github.com/microsoft/vscode/issues/309370 would be the natural vehicle for this fix — adding test cases for Oh My Zsh themes with RPS1 and ensuring the regex doesn't match exit-code markers like `1 ↵` on the right side of the prompt line.
