# Issue #168192 — Terminal can essentially become bricked after pressing ctrl+s

[https://github.com/microsoft/vscode/issues/168192](https://github.com/microsoft/vscode/issues/168192)

## Summary

The requested feature — a status indicator warning when Ctrl+S flow control has paused the PTY, with an action to send Ctrl+Q — has not been implemented on main. The related PR #211289 was closed without merging, and no terminalContrib for flow control detection exists. Still a valid, unresolved feature request.

## Change summary

Show a warning status indicator when Ctrl+S pauses the terminal PTY and provide an action to resume it (send Ctrl+Q).

## Best solution

Implement a terminalContrib that detects when xoff/flow-control pause is active (may need xterm.js API) and surfaces a status bar item with a 'Resume' action that sends Ctrl+Q. The contrib would live in src/vs/workbench/contrib/terminalContrib/flowControl/.
