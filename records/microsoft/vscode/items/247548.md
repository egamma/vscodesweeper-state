# Issue #247548 — When using the trackpad to zoom in and out on a MacBook, the zoom acceleration is too fast

[https://github.com/microsoft/vscode/issues/247548](https://github.com/microsoft/vscode/issues/247548)

## Summary

Valid request for smoother, finer-grained zoom when using trackpad pinch gestures on macOS. VS Code currently uses integer zoom levels via Electron, each step being a large ~20% change, while Chrome supports smooth fractional zooming natively. This is a polish/feature gap, not a bug — the zoom works as designed, just coarsely.

## Change summary

Trackpad pinch-to-zoom on macOS jumps in large integer steps, making the zoom feel too fast and jerky compared to Chrome's smooth fractional zooming.

## Best solution

Implement fractional or finer-grained zoom steps for trackpad pinch gestures at the window level, similar to how the editor font zoom already accumulates delta and divides by 5 in `src/vs/editor/browser/controller/mouseHandler.ts`. This would require changes in `src/vs/platform/window/electron-browser/window.ts` and possibly `src/vs/workbench/electron-browser/actions/windowActions.ts` to support sub-integer zoom levels for trackpad input. The challenge is Electron's `webFrame.setZoomLevel` supports fractional levels but VS Code intentionally rounds to integers (`Math.round` in `BaseZoomAction`). A trackpad-specific path could bypass this rounding.
