# Issue #265630 — File Search does not work when using the cache with glob patterns and sortByScore

[https://github.com/microsoft/vscode/issues/265630](https://github.com/microsoft/vscode/issues/265630)

## Summary

Team-authored issue, still reproduces on main. The cached file search path in rawSearchService.ts always uses fuzzy matching (default `fuzzy=true` in `isFilePatternMatch`) instead of glob matching when filtering cached results, causing zero results on cache hits for glob-pattern queries. An open PR (#308685) is in progress to fix this.

## Change summary

File search cache filtering always uses fuzzy matching instead of glob matching, causing cached queries with glob patterns to return zero results.

## Best solution

Land the open PR https://github.com/microsoft/vscode/pull/308685 which switches the cached result filtering to use glob matching. The fix site is `src/vs/workbench/services/search/node/rawSearchService.ts` lines 322–326, where `isFilePatternMatch` is called with the default `fuzzy=true` parameter — it should use `fuzzy=false` (glob mode) when the original query used glob patterns.
