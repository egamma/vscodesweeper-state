# Issue #265630 — File Search does not work when using the cache with glob patterns and sortByScore

[https://github.com/microsoft/vscode/issues/265630](https://github.com/microsoft/vscode/issues/265630)

## Summary

Team-authored bug, still reproduces on main. The cached file search path in rawSearchService.ts always uses fuzzy matching via `isFilePatternMatch` with `fuzzy=true` (the default), ignoring glob patterns — so re-running the same query against the cache returns 0 results when the query contains glob characters. An open PR (#308685) exists to fix this but has not merged.

## Change summary

File search returns 0 results on cached queries when glob patterns and sortByScore are used, because the cache filtering always applies fuzzy matching instead of glob matching.

## Best solution

Land the open fix PR https://github.com/microsoft/vscode/pull/308685, which switches the cached-result filtering in `rawSearchService.ts` to use glob matching when appropriate. The core defect is at line 326: `isFilePatternMatch(entry, normalizedSearchValueLowercase)` always defaults `fuzzy=true`, so glob-pattern queries never match cached entries.
