# Issue #283248 — Fetch tool doesn't grab javadocs content

[https://github.com/microsoft/vscode/issues/283248](https://github.com/microsoft/vscode/issues/283248)

## Summary

Team-authored bug that remains open: the iframe fix (PR #286367, shipped 1.109.0) did not fully resolve the issue — a team member verified on 2026-01-28 that the fetch tool still cannot extract javadoc content. The root cause is JavaScript-rendered content that the current fetch mechanism cannot process.

## Change summary

The fetch tool cannot retrieve actual documentation content from javadoc.io pages because the site renders content dynamically via JavaScript/iframes.

## Best solution

The WebPageLoader needs to wait for or execute JavaScript rendering before extracting content — either via a headless browser approach (Playwright-like wait-for-network-idle), or by detecting and following iframe src URLs to fetch the static content directly (javadoc.io has static equivalents at /static/ paths). The sandbox reverts (PR #317095) suggest this may have been attempted and rolled back; the next approach should handle JS-rendered sites without sandbox complications.
