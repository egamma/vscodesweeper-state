# Issue #3553 — Get selected file/folder in the explorer view

## Summary

This is the canonical feature request for an API to retrieve the explorer selection, with 125 upvotes and an 'On Deck' milestone — it stays open as a scheduling decision by the team.

## Change summary

Provide an extension API to programmatically retrieve the currently selected file(s)/folder(s) in the explorer view, especially when invoked via keybinding rather than context menu.

## Best solution

Implement a new extension API surface (e.g. `window.activeExplorerSelection` or a command returning selected URIs) in `src/vs/workbench/api/` that exposes the explorer tree's current selection to extensions. The related issue https://github.com/microsoft/vscode/issues/139313 was already closed as a duplicate of this one, confirming it as the canonical tracker.
