# Issue #3553 — Get selected file/folder in the explorer view

[https://github.com/microsoft/vscode/issues/3553](https://github.com/microsoft/vscode/issues/3553)

## Summary

This is the canonical, well-supported (126 upvotes) feature request for an extension API to read the Explorer view selection programmatically. The internal mechanism exists (`IExplorerService.getContext()`) but no extension API has been added. The request remains valid and is properly tracked on the On Deck milestone.

## Change summary

Expose the file Explorer's current selection (file/folder URIs) to extensions via a public API, so commands invoked from keybindings or the Command Palette can act on the selected resources.

## Best solution

Add a public extension API (e.g. on `vscode.window`) that exposes the Explorer view's current selection as `Uri[]` plus a change event, backed by the existing internal `IExplorerService.getContext()`. The newer issue https://github.com/microsoft/vscode/issues/316820 is a duplicate of this one and should be consolidated here.
