# Issue #226068 — Git - VS Code doesn't show "Merge branch" commits in "Timeline" and shows incorrect diffs for subsequent commits

[https://github.com/microsoft/vscode/issues/226068](https://github.com/microsoft/vscode/issues/226068)

## Summary

Still reproduces on main: `git log --follow` in the timeline provider suppresses merge commits, causing them to be missing from the Timeline view and producing incorrect diffs for subsequent commits. The attempted fix PR #324684 was closed because `--topo-order` alone doesn't recover merge commits hidden by `--follow`. The bug remains confirmed and unresolved.

## Change summary

Git Timeline omits merge commits and shows incorrect diffs for commits following a merge because `git log --follow` excludes merge commits from the file-level history.

## Best solution

The fix needs to reconcile `--follow` (needed for rename tracking) with merge-commit visibility in `extensions/git/src/timelineProvider.ts`. Possible approaches: drop `--follow` and handle renames separately, use `--follow --diff-merges=on`, or run a supplementary `git log` pass without `--follow` to recover merge commits. The area owner (@lszomoru) identified the regression source as PR #187174.
