# Issue #293013 — VSCode portable has folder with random characters

[https://github.com/microsoft/vscode/issues/293013](https://github.com/microsoft/vscode/issues/293013)

## Summary

The versioned layout (commit-hash folder) is intentional per MEMBER deepak1556, who stated 'The version based layout at this point is fixed.' The secondary breakages the reporter cited (#293714, #307217) are now closed. The remaining concern is portable-mode ergonomics for users who browse built-in extensions — a valid discussion point the team left open with `under-discussion`.

## Change summary

VS Code portable mode's new versioned directory layout places resources behind a commit-hash folder that changes every release, breaking workflows that reference built-in extension paths.

## Best solution

The team should decide whether to expose a stable symlink or well-known path to built-in extensions in portable mode, or document the `code --version` → first-10-chars convention as the official workaround and close as `*as-designed`. The issue is correctly parked under `under-discussion` for that call.
