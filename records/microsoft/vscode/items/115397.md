# Issue #115397 — Git - clicking on a path segment for a file on git index opens empty picker

[https://github.com/microsoft/vscode/issues/115397](https://github.com/microsoft/vscode/issues/115397)

## Summary

Still reproduces on current main — the git file system provider's `stat()` always returns `FileType.File` and `readDirectory()` throws 'Method not implemented', so breadcrumb path-segment clicks yield an empty picker. The 'On Deck' milestone means this stays open for maintainer scheduling.

## Change summary

Clicking a breadcrumb path segment for a file in the git index opens an empty picker because the git file system provider does not implement directory listing.

## Best solution

Implement `readDirectory()` and fix `stat()` in `extensions/git/src/fileSystemProvider.ts` to resolve directory entries from the git tree, so the breadcrumb picker can list siblings. The related PR https://github.com/microsoft/vscode/pull/115464 (closed without merge) attempted a breadcrumb-side workaround, but jrieken identified the root cause as the git FS provider itself.
