# Issue #232226 — All <span> tags in hover have a 4px bottom margin which causes nested spans to have stacking bottom margins

[https://github.com/microsoft/vscode/issues/232226](https://github.com/microsoft/vscode/issues/232226)

## Summary

The 4px bottom margin on hover spans was fully removed in commit 200701b2f39 (shipped in 1.107.0), resolving both the stacking-margin and gap problems reported here.

## Change summary

Nested <span> tags in editor hovers accumulated 4px bottom margins, causing unintended gaps and expanding heights.

## Best solution

Keep the shipped removal (commit 200701b2f39). No further change needed — the special-case span margin no longer exists in hoverWidget.css.
