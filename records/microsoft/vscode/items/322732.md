# Issue #322732 — Cannot copy selected log fragment from Debug Console on macOS

[https://github.com/microsoft/vscode/issues/322732](https://github.com/microsoft/vscode/issues/322732)

## Summary

Still reproduces on main: right-clicking in the debug console tree triggers `MouseController.onMouseDown()` (which calls `domFocus()`) and `onContextMenu()` (which calls `list.setFocus()`), both of which clear the native DOM text selection before the `debug.replCopy` action reads it via `getSelection()`. The reporter's description is accurate — selection is lost on right-click.

## Change summary

Right-clicking a text selection in the Debug Console clears the selection, so the Copy context menu action copies the whole element instead of the selected fragment.

## Best solution

Preserve the native DOM text selection across right-click in the tree widget's mouse controller — either skip `domFocus()` on right-click in `onMouseDown`, or save/restore the selection range around the focus change. The fix likely belongs in `src/vs/base/browser/ui/list/listWidget.ts` (`MouseController.onMouseDown` and/or `onContextMenu`). The `debug.replCopy` action in `src/vs/workbench/contrib/debug/browser/repl.ts` already handles selection correctly — it's the tree layer clearing it too eagerly.
