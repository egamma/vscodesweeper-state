# Issue #248664 — VSCode is unresponsive on startup

[https://github.com/microsoft/vscode/issues/248664](https://github.com/microsoft/vscode/issues/248664)

## Summary

Reporter self-diagnosed this as a Chromium bug with KWallet on KDE/Wayland (https://bugs.kde.org/show_bug.cgi?id=504014). The root cause is upstream in Chromium's interaction with KWallet when the wallet is not enabled — VS Code inherits the issue via Electron. Reclassifying as upstream; the workaround is enabling KWallet.

## Change summary

VS Code is unresponsive for ~50 seconds on startup on Linux KDE/Wayland when KWallet is not enabled, due to a Chromium bug in the password-store integration.

## Best solution

Reclassify as `upstream` since the root cause is in Chromium's KWallet integration (KDE bug 504014). VS Code already exposes a `--password-store` CLI flag (see `src/main.ts` and `src/vs/platform/environment/node/argv.ts`) that can work around keyring issues. The fix belongs in Chromium/Electron; no VS Code code change is needed. Consider closing with a note pointing users to the `--password-store` flag as a workaround until the upstream fix lands.
