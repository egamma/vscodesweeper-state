# Issue #275808 — keybound merge-conflict.accept.both complains cursor not within a merge

[https://github.com/microsoft/vscode/issues/275808](https://github.com/microsoft/vscode/issues/275808)

## Summary

Confirmed on main: the merge-conflict extension's keyboard-shortcut path for accept commands relies on `findConflictContainingSelection`, which fails when the cursor isn't strictly inside the conflict body — unlike the CodeLens buttons which pass the conflict object directly via `'known-conflict'` args. No fix has been applied.

## Change summary

Keyboard shortcut for `merge-conflict.accept.both` fails with 'cursor not within a merge conflict' even when the cursor is at/near the conflict, because it doesn't receive the conflict context that the CodeLens button provides.

## Best solution

The `accept()` method in `extensions/merge-conflict/src/commandHandler.ts` should, when no `'known-conflict'` argument is passed and `findConflictContainingSelection` returns null, fall back to finding the nearest conflict (or expand the range check to include the marker lines). This would make keyboard shortcuts work like the CodeLens buttons.
