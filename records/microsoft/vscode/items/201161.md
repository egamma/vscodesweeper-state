# Issue #201161 — Tooltips are semantically static/poor

[https://github.com/microsoft/vscode/issues/201161](https://github.com/microsoft/vscode/issues/201161)

## Summary

Valid polish request: the diff editor's whitespace toggle button tooltip is static and does not reflect the current toggled state. The button uses a `toggled` property for visual state but keeps the same title text regardless. Low traction (0 upvotes, no comments in 932 days) but a legitimate UX improvement.

## Change summary

The diff editor's 'Show Leading/Trailing Whitespace Differences' toggle button tooltip remains unchanged after toggling, instead of reflecting whether whitespace differences are currently shown or hidden.

## Best solution

Update the menu item registration at `src/vs/workbench/browser/parts/editor/editor.contribution.ts` (line ~686) to provide a `toggled.title` that reflects the active state (e.g., 'Hide Leading/Trailing Whitespace Differences' when active). The Action2/MenuRegistry infrastructure supports `toggled: { condition, title }` for exactly this pattern.
