# Issue #275281 — Source control changes disappear

[https://github.com/microsoft/vscode/issues/275281](https://github.com/microsoft/vscode/issues/275281)

## Summary

Team-authored issue (MEMBER); must keep open. The original bug resolved as a discoverability problem — repos were silently disabled in the Repositories view. The discussion pivoted to a UX improvement: making hidden-repo state more visible so users aren't confused by a blank SCM panel. Several related commits have improved badge counts and visibility handling since filing, but no change specifically addresses the discoverability gap the reporter described.

## Change summary

When SCM repositories are disabled in a multi-root workspace, the Source Control panel goes blank with no indication that repos are hidden, confusing users.

## Best solution

Add a visible indicator or notification in the SCM view when repositories are hidden but have pending changes — e.g., an inline message or info bar in the empty SCM panel pointing users to the Repositories view. The relevant files are `src/vs/workbench/contrib/scm/browser/scmViewPane.ts` (main view rendering) and `src/vs/workbench/contrib/scm/browser/scmViewService.ts` (visibility state).
