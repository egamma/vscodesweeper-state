# Issue #197637 — '--no-proxy-server' command line argument does not work

[https://github.com/microsoft/vscode/issues/197637](https://github.com/microsoft/vscode/issues/197637)

## Summary

Still reproduces on main. The minimist arg parser converts `--no-proxy-server` into `proxy-server=false`, triggering the 'requires a non empty value' error. An open PR (https://github.com/microsoft/vscode/pull/313185) has the fix but is not yet merged.

## Change summary

The `--no-proxy-server` CLI flag is mangled by minimist's `--no-*` prefix stripping, making it impossible to disable proxy via the command line.

## Best solution

Merge https://github.com/microsoft/vscode/pull/313185 which pre-processes `--no-KEY` args before minimist to prevent the mangling. The fix is in `src/vs/platform/environment/node/argv.ts`.
