# Issue #197637 — '--no-proxy-server' command line argument does not work

[https://github.com/microsoft/vscode/issues/197637](https://github.com/microsoft/vscode/issues/197637)

## Summary

Still reproduces on main: minimist's --no-* prefix handling converts --no-proxy-server into proxy-server:false, which then triggers the 'requires a non empty value' error because proxy-server is a string option. The no-sandbox alias workaround was never applied here.

## Change summary

The --no-proxy-server CLI flag (a Chromium switch to disable proxy) is silently ignored with an error because minimist's --no-* parsing conflicts with the proxy-server string option.

## Best solution

In parseArgs (src/vs/platform/environment/node/argv.ts), add handling so that when a boolean --no-X option coexists with a string X option, minimist's false-assignment to X is suppressed. Alternatively, filter out the spurious proxy-server:false after parsing when no-proxy-server is true. A team member (chrmarti) already identified the root cause in comment #2.
