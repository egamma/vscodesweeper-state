# Issue #294543 — GPT models always read every instruction file

[https://github.com/microsoft/vscode/issues/294543](https://github.com/microsoft/vscode/issues/294543)

## Summary

The system prompt wording that encouraged GPT models to eagerly read all instruction files was rewritten in commit 51d818fec64 (April 2026) to explicitly say 'Only load instruction files when they are relevant to the current task. Do not eagerly load all instructions upfront.' This directly addresses the reported behavior.

## Change summary

GPT models eagerly read every instruction file via tool calling regardless of applyTo relevance, wasting tokens and requiring unnecessary tool approvals.

## Best solution

Keep the shipped fix (commit 51d818fec64). The system prompt was rewritten to discourage eager reading and emphasize relevance-based loading. Further work in fd7f7ce0e1d experimented with disabling the readFile tool entirely for newer GPT models. No additional change needed.
