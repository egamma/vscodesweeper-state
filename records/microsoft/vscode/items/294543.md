# Issue #294543 — GPT models always read every instruction file

[https://github.com/microsoft/vscode/issues/294543](https://github.com/microsoft/vscode/issues/294543)

## Summary

The system prompt instructs models to only load relevant instruction files, but GPT models ignore this guidance and eagerly read all of them via tool calls. The instruction text exists at line 366 of computeAutomaticInstructions.ts but is apparently insufficient for GPT model families. This is a real behavioral gap that has not been addressed on current main.

## Change summary

GPT models eagerly read every instruction file via tool calls regardless of applyTo patterns, wasting tokens and requiring unnecessary tool approvals.

## Best solution

Either strengthen the prompt guidance for GPT models specifically (model-conditional prompting), or change the architecture so that instruction files without a matching applyTo pattern are not listed as available tool targets for GPT models. The fix likely lives in `src/vs/workbench/contrib/chat/common/promptSyntax/computeAutomaticInstructions.ts` around the `_getCustomizationsIndex` method.
