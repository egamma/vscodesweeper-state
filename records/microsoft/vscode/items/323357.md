# Issue #323357 — Copilot Chat /compact triggers MCP server startup (unexpected side effect)

[https://github.com/microsoft/vscode/issues/323357](https://github.com/microsoft/vscode/issues/323357)

## Summary

The MCP autostart fires unconditionally for all agent requests when `model.canUseTools` is true (chatServiceImpl.ts:1579-1586), with no exemption for `/compact` which never needs tools. This is a real inefficiency — compaction explicitly uses `tool_choice: 'none'` but the autostart runs before the agent sees the request.

## Change summary

Running `/compact` in Copilot Chat triggers MCP server startup unnecessarily because the chat service autostarts MCP servers for all agent requests regardless of whether tools are needed.

## Best solution

Guard the MCP autostart in `chatServiceImpl.ts` (line ~1580) to skip when the parsed request is a known tool-free operation like `/compact`, or let the agent signal that tools aren't needed before the autostart runs. The fix likely touches `src/vs/workbench/contrib/chat/common/chatService/chatServiceImpl.ts` and possibly the request parsing to expose whether tools are required.
