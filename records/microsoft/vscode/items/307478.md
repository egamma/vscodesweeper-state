# Issue #307478 — Plugin hooks can't invoke relative bundled scripts

[https://github.com/microsoft/vscode/issues/307478](https://github.com/microsoft/vscode/issues/307478)

## Summary

Confirmed bug: Copilot-format plugin hooks resolve relative paths against the workspace root instead of the plugin's install directory. The Claude/OpenPlugin formats handle this via `interpolateHookPluginRoot` and a `${PLUGIN_ROOT}` token, but the Copilot format's `parseHooks` ignores `pluginUri`. A team member has reproduced the issue.

## Change summary

Hooks in Copilot-format plugins installed from the marketplace cannot invoke scripts bundled inside the plugin because relative paths resolve against the workspace, not the plugin installation directory.

## Best solution

Apply the same `interpolateHookPluginRoot` pattern (or equivalent `${PLUGIN_ROOT}` token expansion) in the `COPILOT_FORMAT.parseHooks` implementation in `src/vs/platform/agentPlugins/common/pluginParsers.ts`, so that relative paths and the `cwd` field resolve against the plugin's install directory rather than the workspace root.
