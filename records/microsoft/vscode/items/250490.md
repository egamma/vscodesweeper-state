# Issue #250490 — UX bug: unexpected hover behaviour - attached file

[https://github.com/microsoft/vscode/issues/250490](https://github.com/microsoft/vscode/issues/250490)

## Summary

Team-authored polish issue about inconsistent hover treatment on attached file chips in the chat input. The team agreed in comments to try improved hover styling in a future iteration — still open work.

## Change summary

Attached file chips in the chat input lack a cohesive hover background, making them feel inconsistent with the 'Add Context' button and unclear about click targets.

## Best solution

Add a hover background to the icon + filename portion of the attachment chip (excluding the X button), matching the treatment on the 'Add Context' button. The relevant CSS is in `src/vs/workbench/contrib/chat/browser/widget/media/chat.css` around the `.chat-attached-context-attachment` selectors.
