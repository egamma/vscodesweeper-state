# Issue #294558 — Treat "Working Tree" as a selectable node for comparisons in Git Graph

[https://github.com/microsoft/vscode/issues/294558](https://github.com/microsoft/vscode/issues/294558)

## Summary

Valid feature request already accepted into Backlog: add a Working Tree node to the built-in SCM history graph so users can compare uncommitted changes against any commit or branch. No implementation exists on current main; the gap remains real.

## Change summary

Allow comparing uncommitted working tree changes against any commit or branch in the built-in Git Graph view.

## Best solution

Add a virtual 'Working Tree' node to the SCM history graph (src/vs/workbench/contrib/scm/browser/scmHistoryViewPane.ts) that participates in the multi-select compare flow when uncommitted changes exist. This requires extending the history item model and the comparison action to accept a working-tree reference.
