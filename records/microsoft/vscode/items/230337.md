# Issue #230337 — Support for search engines

[https://github.com/microsoft/vscode/issues/230337](https://github.com/microsoft/vscode/issues/230337)

## Summary

Valid, community-promoted feature request (35 upvotes) for extensible search engine support in VS Code's search UI. The related TextSearchProvider APIs (#59921, #73524) cover custom file-system search but not custom search modes with custom UI — a real gap remains. Keep open in the Backlog.

## Change summary

Allow extensions to register custom search engine types (e.g. AST search) in VS Code's built-in search bars, with the ability to add custom filter options and search modes alongside text and regex.

## Best solution

Extend the search contribution API to allow extensions to register custom search modes in both the workspace search sidebar and the in-editor find widget, building on the existing TextSearchProvider2 proposal and the notebook filter-button precedent. This likely involves work in `src/vs/workbench/contrib/search/` for the UI surface and `src/vs/workbench/api/` for the extension API. Stabilizing TextSearchProvider (https://github.com/microsoft/vscode/issues/59921) is a prerequisite or closely related track.
