# Issue #320271 — Middle click scroll and drag causes text to be deleted

[https://github.com/microsoft/vscode/issues/320271](https://github.com/microsoft/vscode/issues/320271)

## Summary

This is as-designed: middle-click triggers column selection when `editor.scrollOnMiddleClick` is disabled (the default). The Firefox-style autoscroll the reporter wants already exists behind the `editor.scrollOnMiddleClick` setting, which prevents the column-select behavior and enables scroll-on-drag.

## Change summary

Middle-click drag selects/deletes text instead of triggering Firefox-style autoscroll.

## Best solution

Close as designed — the `editor.scrollOnMiddleClick` setting already provides exactly the autoscroll behavior the reporter wants. The column-selection on middle-click is intentional multi-cursor functionality when that setting is off.
