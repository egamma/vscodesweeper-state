# Issue #314556 — Chat session persistence inlines image and tool binary data as base64, causing unbounded session growth

[https://github.com/microsoft/vscode/issues/314556](https://github.com/microsoft/vscode/issues/314556)

## Summary

Still reproduces on main: all base64 inline persistence paths described in the issue remain in place — no resource-store indirection has been added. A team member confirmed the problem. Keeping open as a real, unaddressed bug.

## Change summary

Chat session JSONL files grow without bound because image and tool binary payloads are stored inline as base64 instead of as external resource references.

## Best solution

Introduce a session-scoped resource store so binary payloads are persisted separately and JSONL stores lightweight references. Key files to change: `src/vs/workbench/contrib/chat/common/attachments/chatVariableEntries.ts` (the `$base64` encode/decode in `toExport`/`fromExport`), `src/vs/workbench/contrib/chat/common/model/chatProgressTypes/chatToolInvocation.ts` (`base64Data` serialization), and `src/vs/workbench/contrib/chat/browser/tools/languageModelToolsService.ts` (`encodeBase64` in embed output). Session lifecycle and garbage collection must be wired to the new store.
