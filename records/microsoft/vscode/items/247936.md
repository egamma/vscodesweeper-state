# Issue #247936 — No option for commit --amend if working with detached HEAD

[https://github.com/microsoft/vscode/issues/247936](https://github.com/microsoft/vscode/issues/247936)

## Summary

Confirmed bug with an open PR already addressing it. On detached HEAD, `getCommitActionButtonSecondaryCommands()` returns `[]`, hiding the amend dropdown. The issue has an 'On Deck' milestone and an open fix PR — keeping open for that PR to land.

## Change summary

The commit button's dropdown (including Commit Amend) is hidden when working on a detached HEAD.

## Best solution

Land or iterate on the open PR https://github.com/microsoft/vscode/pull/305034, which adds the amend option to the secondary commands when on a detached HEAD. The fix is in `extensions/git/src/actionButton.ts`, lines 189–192, where the early `return []` suppresses all secondary commands for non-branch refs.
