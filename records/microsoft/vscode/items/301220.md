# Issue #301220 ‚Äî Locked `Explore.agent.md` frontmatter appears editable using `Configure Tools...` but actually isn't

[https://github.com/microsoft/vscode/issues/301220](https://github.com/microsoft/vscode/issues/301220)

## Summary

Confirmed on current main: the "Configure Tools..." code lens appears on read-only/locked prompt files (like the built-in Explore agent) and lets the user pick tools via the picker, but the edit is silently dropped because the model is read-only ‚Äî no guard exists in either the code lens provider or the rewriter.

## Change summary

The "Configure Tools..." code lens on locked/built-in agent prompt files lets users select tools via the picker, but changes are silently discarded because the underlying model is read-only.

## Best solution

Either suppress the "Configure Tools..." code lens when the model is read-only (check `model.isReadonly()` in `PromptToolsCodeLensProvider.provideCodeLenses`), or have `PromptFileRewriter.rewriteTools` / `rewriteAttribute` check the model's readonly state and surface a user-facing notification when the edit cannot be applied. The same fix should be applied to `PromptToolSetsCodeLensProvider`.
