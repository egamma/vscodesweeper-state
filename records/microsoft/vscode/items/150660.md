# Issue #150660 — Loading 256kb file takes 15 seconds on a powerful machine

[https://github.com/microsoft/vscode/issues/150660](https://github.com/microsoft/vscode/issues/150660)

## Summary

Hard keep-open: issue carries the `important` label and is assigned the `On Deck` milestone. The underlying long-line rendering performance problem (LayoutNG + DOM-based character positioning for long lines) appears to still be present on main, though GPU rendering work may partially mitigate it.

## Change summary

Opening a 256KB file with very long lines causes multi-second freezes due to expensive DOM character positioning queries triggered by decoration rendering.

## Best solution

The GPU-accelerated line rendering path (`viewLinesGpu`) may already help for lines that can use `FastRenderedViewLine`. The remaining fix likely involves either switching to multiplication-based positioning for long lines (avoiding DOM queries) or disabling expensive decoration features when lines exceed a threshold. The root cause (LayoutNG + `RenderedViewLine` DOM queries) is in `src/vs/editor/browser/viewParts/viewLines/` and the rendering context.
