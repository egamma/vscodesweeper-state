# Issue #210709 — Git - Execute multiple git operations at the same time on different repositories

[https://github.com/microsoft/vscode/issues/210709](https://github.com/microsoft/vscode/issues/210709)

## Summary

Still reproduces on main: the global `operationInProgress` context key is set to true when ANY repository has a running operation, disabling git commands for ALL repositories. This is a real bug in multi-root workspaces with no fix on current main.

## Change summary

Git operations in one repository disable all git UI commands across all other repositories in a multi-root workspace.

## Best solution

Make `operationInProgress` per-repository (or scope the enablement to the active/selected repository) so that operations in one repo don't block commands targeting a different repo. The fix lives in `extensions/git/src/model.ts` (the `updateOperationInProgressContext` closure) and the 138 `enablement` clauses in `extensions/git/package.json`.
