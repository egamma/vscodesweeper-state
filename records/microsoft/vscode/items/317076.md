# Issue #317076 — Visible editor does not refresh document links until tab switch or reopen

[https://github.com/microsoft/vscode/issues/317076](https://github.com/microsoft/vscode/issues/317076)

## Summary

Valid feature gap: the DocumentLinkProvider API has no onDidChange event, so extensions cannot signal the editor to re-query links when external state changes. The internal LinkDetector only recomputes on content/model/language/registry changes. Routing to the editor-api area for consideration.

## Change summary

Extensions providing document links have no way to trigger a link refresh in a visible editor when external dependencies change without editing the document.

## Best solution

Add an optional `onDidChangeDocumentLinks` event to the `DocumentLinkProvider` interface (similar to `CodeLensProvider.onDidChangeCodeLenses`), and wire it into the internal `LinkDetector` to schedule a recomputation. This would be a new extension API surface — likely starts as a proposed API.
