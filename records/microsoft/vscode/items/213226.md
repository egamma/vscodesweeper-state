# Issue #213226 — In diff mode, user is forced to interact with a dialog instead of direct hitting the revert button

[https://github.com/microsoft/vscode/issues/213226](https://github.com/microsoft/vscode/issues/213226)

## Summary

Still reproduces on main: in the diff editor gutter toolbar, when `_isSmall` is true the overflow `maxItems` is set to 1, but the toolbar's `count >= maxItems` logic (toolbar.ts:184) moves items to overflow at count=1, meaning zero items are shown directly — the sole "Revert Block" action ends up behind a three-dot menu.

## Change summary

In diff mode with a small hunk starting at line 1, the gutter shows a three-dot overflow menu containing a single "Revert Block" action instead of showing the action directly.

## Best solution

Change `maxItems: 1` to `maxItems: 2` in gutterFeature.ts line 260 for the `_isSmall` case, so that one action is shown directly. Alternatively, fix the off-by-one in the toolbar overflow logic (`count >= maxItems` → `count > maxItems`), but that would affect all callers and needs broader review.
