# Issue #281299 — `Authorize Visual Studio Code` doesn't distinguish VS Code from VS Code insider

[https://github.com/microsoft/vscode/issues/281299](https://github.com/microsoft/vscode/issues/281299)

## Summary

Valid cosmetic request but blocked on GitHub platform constraints — both VS Code and Insiders share one OAuth client ID, and GitHub's authorization page shows the icon registered against that client ID. A team member acknowledged this needs discussion with GitHub to resolve. Keeping open as an accepted but unscheduled enhancement.

## Change summary

GitHub OAuth authorization page shows the stable VS Code logo even when authenticating from VS Code Insiders.

## Best solution

Coordinate with GitHub to either register a separate OAuth client ID for Insiders (with GHES compatibility) or request GitHub support per-flow icon selection on their authorization page. This is primarily a GitHub-platform dependency, not a VS Code code change.
