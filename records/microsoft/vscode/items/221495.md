# Issue #221495 — Search editor shows the wrong results when running multiple

[https://github.com/microsoft/vscode/issues/221495](https://github.com/microsoft/vscode/issues/221495)

## Summary

Confirmed race condition in concurrent search editors — each SearchEditor creates its own SearchModelImpl, but when two searches complete concurrently, both editors display the same results. The bug is unfixed on current main and appropriately placed in Backlog.

## Change summary

When running two project-wide searches concurrently with `search.mode: newEditor`, both search editor tabs display the results of whichever search completes last, rather than each showing its own results.

## Best solution

Investigate the race condition in `src/vs/workbench/contrib/searchEditor/browser/searchEditor.ts` around `doRunSearch()` — each editor has its own `SearchModelImpl`, so the cross-contamination likely involves either the shared `ISearchService` result streaming, the `SearchEditorInput` model resolution, or a subtle reference-sharing path in the serialization at line 632. The guard in `onSearchComplete` (lines 615–620) checks input identity and config equality but may not catch all concurrent-completion races.
