# Issue #221495 — Search editor shows the wrong results when running multiple

[https://github.com/microsoft/vscode/issues/221495](https://github.com/microsoft/vscode/issues/221495)

## Summary

Still present on main: the search editor's `doRunSearch` uses `this.searchModel` which is per-pane, but when VS Code reuses or recycles editor panes across tabs in the same group, concurrent searches can overwrite each other's results. No fixing commit found.

## Change summary

When two search editors run concurrently with `search.mode: "newEditor"`, both display the same results instead of their respective query results.

## Best solution

The `SearchEditor` pane's `searchModel` is instance-scoped but panes can be recycled. The fix should either scope the search model per-input (not per-pane), or add a generation counter / cancellation guard so that stale results from a recycled pane's model are never written to the wrong input's results model. Likely files: `src/vs/workbench/contrib/searchEditor/browser/searchEditor.ts`.
