Team-authored feature request (MEMBER) — must stay open for maintainer judgment. The chat variable leader is still hardcoded as `#` in `chatParserTypes.ts`; no `@` aliasing exists on current main. ## Change summary

Alias `@` to `#` in the chat input so users with muscle memory from other tools (e.g. Cursor) can use `@` to reference context.

## Best solution

Implement `@` as an alternative trigger for the chat variable/context completions in `src/vs/workbench/contrib/chat/common/requestParser/chatParserTypes.ts` and the completion provider in `src/vs/workbench/contrib/chat/browser/widget/input/editor/chatInputCompletions.ts`. This requires a product decision on whether `@` fully replaces `#` or acts as a secondary alias — the team comment (+1 from @jo-oikawa) suggests appetite exists.
