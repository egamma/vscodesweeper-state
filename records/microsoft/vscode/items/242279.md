## Summary

Legitimate feature request from a core contributor to add view zone support to the multi-file diff editor. No implementation exists on current main; the feature remains a real gap. Keeping open in Backlog.

## Change summary

Add support for rich markdown view zones in the multi-file diff editor, enabling use cases like rendering commit details (author, date, description, issue links) atop the editor when opening a commit from git blame.

## Best solution

Implement view zone support in the multi-diff editor widget (likely in `src/vs/editor/browser/widget/multiDiffEditor/`), allowing consumers to inject rich content zones above individual diff entries. The git-blame commit-details use case would be a first consumer. This is a meaningful enhancement to the multi-diff editor's extensibility.
