# Issue #298054 — Merge Conflict Bug - cannot stage change even no conflict

[https://github.com/microsoft/vscode/issues/298054](https://github.com/microsoft/vscode/issues/298054)

## Summary

This reports the same underlying gap as #250857: when merge conflicts are resolved outside the merge editor (e.g. in the regular text editor or via context-menu accept), VS Code does not automatically `git add` the file, leaving it in an 'unmerged' state. The canonical issue tracks this.

## Change summary

Resolving merge conflicts in the regular editor does not auto-stage the file, leaving git in an 'unmerged files' state that blocks commit.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/250857 — implementing auto-stage after conflict resolution outside the merge editor delivers exactly what this reporter needs.
