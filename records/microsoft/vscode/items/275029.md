# Issue #275029 — Terminal Completion: `commandLine` context sometimes includes the shell indicator and the working directory

[https://github.com/microsoft/vscode/issues/275029](https://github.com/microsoft/vscode/issues/275029)

## Summary

Team-authored issue (MEMBER) — must stay open for maintainer judgment. The bug is about the terminal completion `commandLine` context including the shell prompt prefix (e.g. `PS C:\Code\vscode>`) when the panel is resized. A related fix landed in commit 66717e67900 (fixes #275546, swapping `prefix` for `value`), but that addresses a different aspect of the same area; whether it also resolves this specific resize-triggered prompt-leaking behavior is unclear.

## Change summary

Terminal completion's `context.commandLine` incorrectly includes the shell indicator and working directory (e.g. `PS C:\Code\vscode> helloWorld` instead of `helloWorld`), triggered by panel resizing.

## Best solution

Investigate whether `_currentPromptInputState.value` still includes the shell prompt text after a panel resize in `terminalSuggestAddon.ts`. The prompt-detection logic in the terminal suggest addon may need to re-parse the command line boundary when the terminal is resized, since resizing can cause the prompt and command to reflow onto the same reported line.
