[https://github.com/microsoft/vscode/issues/246841](https://github.com/microsoft/vscode/issues/246841)

## Summary

The most recent commenter confirmed that updating the Cortex-Debug extension to its latest release (from GitHub, not Marketplace) resolved the issue. This is a Cortex-Debug compatibility problem that was triggered by VS Code 1.86 changes but owned by the extension.

## Change summary

Debugger UI remains open after stopping a debug session when using the Cortex-Debug extension with a Restart→Stop sequence.

## Best solution

Close as caused-by-extension. The Cortex-Debug extension needed to adapt to debug session lifecycle changes in 1.86+; updating to the latest Cortex-Debug release resolves it. The reporter should update their extension and report remaining issues to the Cortex-Debug project.
