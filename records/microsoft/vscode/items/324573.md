# Issue #324573 — Agents window detail panel: Not prompted about migrating local changes

[https://github.com/microsoft/vscode/issues/324573](https://github.com/microsoft/vscode/issues/324573)

## Summary

Team-authored bug report about inconsistent behavior when migrating local uncommitted changes to worktree sessions in the Agents window — must remain open for maintainer judgment.

## Change summary

When creating a worktree session from the Agents window on a repo with local uncommitted changes, the user is not prompted to migrate those changes and the Changes tab remains empty, unlike folder sessions created from the editor window.

## Best solution

Ensure the worktree session creation path in the Agents window prompts the user about migrating local uncommitted changes (matching the editor-window folder-session behavior), and populates the Changes tab accordingly when the user opts in.
