# Issue #183117 — Select previous word in JSON string sometimes also selects `"`, but not always

[https://github.com/microsoft/vscode/issues/183117](https://github.com/microsoft/vscode/issues/183117)

## Summary

As-designed: word navigation uses `editor.wordSeparators` where both `?` and `"` are separators. In `"??"` the entire content is separators, so cursor-left jumps past the opening quote to the next word boundary. Maintainer aeschli already explained this in the thread.

## Change summary

CTRL+SHIFT+LEFT in a JSON string like `"??"` selects the opening quote, but in `"N_FP"` it does not.

## Best solution

Close as designed — the behavior is consistent with `editor.wordSeparators` treating `?` and `"` as the same character class (separator). The user can customize `editor.wordSeparators` to change this behavior.

As Martin explained, this is the expected behavior of `editor.wordSeparators` — `?` and `"` are both classified as word separators, so in `"??"` the entire content (including quotes) forms a single separator-class word unit, and cursor-word-left jumps past it.
