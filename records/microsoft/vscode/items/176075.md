## Summary

Still valid on current main. The inconsistency persists: New File/Folder falls back to `explorerService.roots[0]` when no context is available, while other commands use the previously selected folder. Already triaged with `help wanted` and `polish` — keeping open for a community contributor.

## Change summary

Explorer context menu commands behave inconsistently when right-clicking in empty space: New File/Folder targets the first workspace folder while other commands target the previously selected folder.

## Best solution

Align `openExplorerAndCreate` in `src/vs/workbench/contrib/files/browser/fileActions.ts` (line ~937) to use the last-focused/selected folder instead of always falling back to `roots[0]`, matching the behavior of other context menu commands like Reveal in File Explorer and Copy Path.
