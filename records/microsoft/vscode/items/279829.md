# Issue #279829 — Feature Request: Add Rich Text Editor Support in Copilot Chat Window

[https://github.com/microsoft/vscode/issues/279829](https://github.com/microsoft/vscode/issues/279829)

## Summary

Feature request for WYSIWYG rich-text editing in the Copilot Chat input. The current input is a Monaco editor where users type plain text/markdown; responses already render markdown. Zero upvotes after ~8 months — keeping open for the community process but unlikely to gain traction given VS Code's uniform UI values.

## Change summary

Add a rich text (WYSIWYG) editor to the Copilot Chat input box supporting formatted markdown, embedded images, tables, and a toggle between rich and plain text modes.

## Best solution

The current chat input already accepts markdown syntax and responses render it fully (headings, code blocks, lists, links). A full WYSIWYG editor in the input box would add significant complexity for marginal benefit — users can already type markdown. If there's a specific gap (e.g. image attachment, drag-and-drop), a narrower issue would be more actionable. This request is best left for the community process to gauge demand; if it doesn't reach 20 upvotes, it will age out naturally.
