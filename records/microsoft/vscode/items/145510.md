# Issue #145510 — Feature to disable auto-expanding on sub-folders.

[https://github.com/microsoft/vscode/issues/145510](https://github.com/microsoft/vscode/issues/145510)

## Summary

Valid, community-backed feature request (20 upvotes, at Backlog promotion threshold). The `autoExpandSingleChildren: true` is hardcoded in the explorer view with no user-facing setting to disable it. The related PR was closed unmerged when its author left the team, and the behavior remains unchanged on current main.

## Change summary

Add a setting to disable the explorer's automatic expansion of folders that contain only a single child subfolder.

## Best solution

Add an `explorer.autoExpandSingleChildren` boolean setting (default `true` for backward compat) that gates the hardcoded `autoExpandSingleChildren: true` in `src/vs/workbench/contrib/files/browser/views/explorerView.ts` (line ~500). The tree infrastructure already supports this option — it just needs to be wired to a user setting. The abandoned PR https://github.com/microsoft/vscode/pull/145606 is a useful reference for the approach.
