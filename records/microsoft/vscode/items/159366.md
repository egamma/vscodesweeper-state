# Issue #159366 — Cursor no longer selects entire tab (accessibility issue)

[https://github.com/microsoft/vscode/issues/159366](https://github.com/microsoft/vscode/issues/159366)

## Summary

Still present on main: block cursor on a tab intentionally renders at 1ch width (line 220-221 of viewCursor.ts). The team discussed this as a deliberate change matching SublimeText behavior but left the issue open to collect feedback. It has accessibility implications for users relying on the visual cue.

## Change summary

Block cursor no longer covers the full tab width, making it harder for visually-impaired users to distinguish tabs from spaces.

## Best solution

Add a setting (e.g. `editor.cursorBlockFullTab`) to restore the old behavior where the block cursor spans the full tab width. The fix would be localized to `src/vs/editor/browser/viewParts/viewCursors/viewCursor.ts` around lines 219-224. Alternatively, the team may decide the extension-decoration approach isidorn suggested is sufficient.
