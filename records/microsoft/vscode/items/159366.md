# Issue #159366 — Cursor no longer selects entire tab (accessibility issue)

[https://github.com/microsoft/vscode/issues/159366](https://github.com/microsoft/vscode/issues/159366)

## Summary

Keep open: maintainers reproduced the regression, identified it as fallout from https://github.com/microsoft/vscode/issues/151959, and explicitly chose to keep the issue open to collect feedback rather than close it as designed or extension-only.

## Change summary

Block cursor on a tab now renders as one column wide instead of covering the full tab width, reducing a visual accessibility cue for the reporter.

## Best solution

Editor rendering should either intentionally keep the one-column block cursor behavior with a documented accessibility-friendly extension/decorator path, or add a core option if maintainer feedback shows the full-tab block cursor is an accessibility requirement.
