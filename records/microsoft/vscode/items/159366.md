# Issue #159366 — Cursor no longer selects entire tab (accessibility issue)

[https://github.com/microsoft/vscode/issues/159366](https://github.com/microsoft/vscode/issues/159366)

## Summary

This should stay open: a VS Code maintainer reproduced the regression and identified it as fallout from https://github.com/microsoft/vscode/issues/151959, but the team then chose to keep the issue open to collect feedback rather than close it as designed.

## Change summary

Block cursors on tab characters now render as one column wide instead of spanning the full tab width, reducing visibility for the reporter's accessibility workflow.

## Best solution

Decide in the editor rendering/accessibility area whether block cursor width on tabs should remain one column, be restored to the pre-1.70 full-tab rendering, or become a narrowly scoped accessibility/editor option. If extension support is preferred, core would still need to expose enough cursor or decoration control for an extension to reproduce the old affordance cleanly.
