# Issue #287059 — Workspace folder name integrated terminal title not preserved on window reload

[https://github.com/microsoft/vscode/issues/287059](https://github.com/microsoft/vscode/issues/287059)

## Summary

Confirmed on main: during terminal reconnection after reload, `_workspaceFolder` is never derived from `attachPersistentProcess.cwd`, so `${workspaceFolderName}` in the title template resolves to the last-active workspace root for all terminals instead of each terminal's actual workspace folder.

## Change summary

Terminal title variable `${workspaceFolderName}` loses its per-terminal value after a window reload in multi-root workspaces, defaulting all terminals to the same name.

## Best solution

In the constructor of `TerminalInstance`, also check `attachPersistentProcess.cwd` (in addition to `shellLaunchConfig.cwd`) when determining `_workspaceFolder`. Additionally, re-derive `_workspaceFolder` in the `ProcessPropertyType.InitialCwd` handler so it stays correct throughout the terminal's lifetime.
