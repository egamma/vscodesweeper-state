# Issue #178184 — Difference in hover sequence results between run-time and test-time

[https://github.com/microsoft/vscode/issues/178184](https://github.com/microsoft/vscode/issues/178184)

## Summary

Still reproduces on main: the `_executeHoverProvider` command returns hovers in promise-resolution order, while the UI sorts them by ordinal (provider score). The discrepancy is a real bug in the command's output contract.

## Change summary

The `vscode.executeHoverProvider` command returns hover results in non-deterministic resolution order instead of the score-sorted order shown in the hover widget UI.

## Best solution

Sort the results in `getHoversPromise` by the `ordinal` field before returning, matching the `hoverParts.sort(compareBy(hover => hover.ordinal, numberComparator))` pattern already used in `markdownHoverParticipant.ts`.
