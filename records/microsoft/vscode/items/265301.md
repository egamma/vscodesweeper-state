# Issue #265301 — When WSL is closed / crashes, all open instances of VSCode have to be closed completely - reload window and attempting to reconnect fails

[https://github.com/microsoft/vscode/issues/265301](https://github.com/microsoft/vscode/issues/265301)

## Summary

Valid bug: when WSL is force-killed, VS Code's reconnection flow fails because the PersistentConnection uses a reconnection token tied to the now-dead server process. No fix found on current main. Keeping open as a real WSL reconnection resilience gap.

## Change summary

After WSL crashes, VS Code's 'Reconnect' and 'Reload Window' actions fail to establish a new connection; only a full Dev Tools refresh works.

## Best solution

The reconnection logic in `src/vs/platform/remote/common/remoteAgentConnection.ts` should detect when the remote server process is entirely gone (not just temporarily unreachable) and fall back to a fresh connection with a new reconnection token, rather than permanently failing. The WSL extension (`ms-vscode-remote.remote-wsl`) may also need changes to handle the case where a new WSL instance spawns a new server.
