# Issue #289051 — Editor Hover Overflow Issue In Small Screen Devices

[https://github.com/microsoft/vscode/issues/289051](https://github.com/microsoft/vscode/issues/289051)

## Summary

Still reproducible on main: the hover's default CSS max-width of 500px exceeds small mobile viewports, causing overflow before the JS-based width adjustment can kick in. No fix has been committed for this scenario.

## Change summary

Editor hover overflows the viewport on small-screen devices (e.g. Android phones) when hovering over large types.

## Best solution

The hover widget's initial max-width (500px CSS default) should be clamped to the available viewport width. A fix in `src/vs/base/browser/ui/hover/hoverWidget.css` (e.g. `max-width: min(var(--vscode-hover-maxWidth, 500px), calc(100vw - 14px))`) or in the JS sizing logic in `contentHoverWidget.ts` to set the CSS variable before the first layout would prevent overflow on narrow viewports.
