## Summary

Crash in the native sqlite3 module (`Statement::Finalize_`) during Code OSS main-process shutdown when stopping a debug session. Author is a CONTRIBUTOR — keeping open per policy. No evidence of a fix on current main; the `@vscode/sqlite3` native binding at 5.1.12-vscode still ships and the crash path in native code is outside the TypeScript layer.

## Change summary

Code OSS crashes (SIGABRT) in the sqlite3 native module's Statement::Finalize_ during main-process shutdown when stopping a debug session.

## Best solution

Investigate the native `@vscode/sqlite3` shutdown path — the C++ `Statement::Finalize_` is throwing a C++ exception that propagates to `std::terminate`. Either the statement or database handle is already destroyed by the time finalize runs. The fix likely belongs in the `@vscode/sqlite3` native addon (ensuring graceful finalization order on process exit) or in the storage layer's shutdown sequence (`src/vs/base/parts/storage/`) to guarantee all statements are finalized before the database handle closes.
