# Issue #202916 — Git - Please add an option to use English for SCM UI even when a localization pack is used

[https://github.com/microsoft/vscode/issues/202916](https://github.com/microsoft/vscode/issues/202916)

## Summary

Valid feature request to keep English git terminology in the SCM UI regardless of language pack. Not yet implemented — an earlier 2018 attempt was reverted. Issue #232002 was closed as a duplicate of this one, making this the canonical tracker. Low upvotes but a maintainer acknowledged the request and routed it to the git extension owner.

## Change summary

Add a setting to keep English strings for SCM/git UI terms even when a localization pack is active.

## Best solution

Implement a setting in the built-in git extension (e.g. `git.useEnglishUI`) that forces English strings for git-specific terms. The setting would live in `extensions/git/` and override NLS lookups for the git extension's UI strings. A maintainer acknowledged this path in the comments.
