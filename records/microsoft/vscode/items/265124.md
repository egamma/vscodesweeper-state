# Issue #265124 — Visual Studio Code cannot run from filesystem which is case-sensitive

[https://github.com/microsoft/vscode/issues/265124](https://github.com/microsoft/vscode/issues/265124)

## Summary

The fatal crash originates in Chromium's GPU process DLL loading (`content/browser/gpu/gpu_process_host.cc`), which fails when Electron is run from a case-sensitive filesystem on Windows — this is an Electron/Chromium upstream issue, not a VS Code code defect. Keeping open for upstream tracking.

## Change summary

VS Code crashes on launch when installed on a case-sensitive Windows filesystem (e.g. SMB, WinBTRFS, NFS) because Chromium's GPU process cannot find its DLLs due to filename case mismatches.

## Best solution

File this upstream against Electron (or Chromium) as a bug in GPU process DLL resolution on case-sensitive Windows filesystems. VS Code has no control over how Electron locates and loads its GPU subprocess binaries. Until fixed upstream, users should install VS Code on a case-insensitive filesystem or use `--disable-gpu` as a workaround.
