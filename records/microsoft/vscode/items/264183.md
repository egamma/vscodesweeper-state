# Issue #264183 — Console host / terminal tab will not exit after using `code` to open the first Code window, until all Code windows are closed

[https://github.com/microsoft/vscode/issues/264183](https://github.com/microsoft/vscode/issues/264183)

## Summary

The report describes a real behavioral issue on Windows where the `code` CLI, when opening the first VS Code window, prevents the console host / terminal tab from closing until all VS Code windows exit. The CLI launcher (`src/vs/code/node/cli.ts`) spawns the Electron GUI process with `detached: true` but never calls `child.unref()`, and the deeper cause likely involves Windows console host job-object tracking of Electron/Chromium child processes. A team member attempted reproduction without success and requested PATH info; the reporter provided it and confirmed the issue persists (albeit reduced) on 1.120.0. Keeping open as it needs further investigation on Windows.

## Change summary

Running `code; exit` in PowerShell blocks the console host / terminal tab from closing when the `code` command opens the first VS Code window, until all VS Code windows are closed.

## Best solution

Investigate whether adding `child.unref()` after the `spawn()` call in `src/vs/code/node/cli.ts` (around line 502) resolves the issue on Windows, or whether the root cause is Electron/Chromium's console-detach behavior on Windows (upstream). A maintainer with a Windows environment should reproduce and test the `unref()` mitigation. If the cause is Electron's job-object inheritance, file upstream and track here.
