# Issue #234032 — "Accept all changes from .." button in merge editor duplicates non-conflicting changes

[https://github.com/microsoft/vscode/issues/234032](https://github.com/microsoft/vscode/issues/234032)

## Summary

Still reproduces on main: `acceptAll` in the merge editor view model iterates all `modifiedBaseRanges` without skipping non-conflicting ones that are already handled, causing non-conflicting changes to be duplicated when 'Accept All' is clicked.

## Change summary

The merge editor's 'Accept All Changes' button duplicates non-conflicting changes by toggling state on ranges that are already correctly handled.

## Best solution

Guard the `acceptAll` loop in `src/vs/workbench/contrib/mergeEditor/browser/view/viewModel.ts` to skip `modifiedBaseRanges` where `!range.isConflicting && model.isHandled(range)` is already true, so only actual conflicts are affected.
