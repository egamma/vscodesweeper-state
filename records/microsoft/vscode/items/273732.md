# Issue #273732 — Terminal suggest hijacks listing files

[https://github.com/microsoft/vscode/issues/273732](https://github.com/microsoft/vscode/issues/273732)

## Summary

Still open and unfixed on main. The bug is that with `runOnEnter: always`, when aliases/commands fail to appear in terminal suggest completions, file-name suggestions get accepted and executed on Enter — hijacking the intended command. The maintainer acknowledged the issue but no fix has landed.

## Change summary

Terminal suggest with `runOnEnter: always` accepts and executes a file-name completion instead of running a typed alias/command when aliases aren't populated in the suggestion list.

## Best solution

Fix the terminal suggest completions to always include aliases/commands (or prioritize them) so that `runOnEnter` doesn't accidentally submit a file-name completion in place of a valid alias. The root cause appears to be aliases not showing up in the completion provider for certain shell configurations; the fix likely lives in `extensions/terminal-suggest/src/terminalSuggestMain.ts` and/or the shell integration scripts that export aliases.
