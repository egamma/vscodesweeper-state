# Issue #302109 — Sessions: Terminal and command output is not visible when the agent runs commands, showing "The terminal is no longer available"

[https://github.com/microsoft/vscode/issues/302109](https://github.com/microsoft/vscode/issues/302109)

## Summary

Team-authored issue (MEMBER) — must remain open for explicit maintainer judgment. The request for persistent, inspectable terminal output in agent sessions is an active UX concern with community engagement and related work in progress.

## Change summary

Terminal command output run by the agent disappears or shows 'The terminal is no longer available', preventing developers from inspecting logs and diagnostics inline.

## Best solution

The sessions terminal contribution (`src/vs/sessions/contrib/terminal/browser/sessionsTerminalContribution.ts`) needs a strategy that preserves command output for inspection after completion — either by keeping terminal content accessible in the session history, streaming output inline, or retaining terminal instances until the user dismisses them. Related issues (https://github.com/microsoft/vscode/issues/322466, https://github.com/microsoft/vscode/issues/290977) track adjacent cleanup and lifecycle concerns that should be coordinated.
