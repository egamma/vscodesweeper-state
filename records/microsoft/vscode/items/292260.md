# Issue #292260 — Notebook tools added without being enabled by the user

[https://github.com/microsoft/vscode/issues/292260](https://github.com/microsoft/vscode/issues/292260)

## Summary

Team-authored bug, still open on main. The `extension_installed_by_tool` tag bypass in toolsService.ts (line 349) auto-enables tools from pre-installed extensions (like Jupyter) even when not selected by the user. No fix has landed.

## Change summary

Notebook tools from the Jupyter extension are auto-enabled in chat agent sessions even when the user only requested `web` tools, because the `extension_installed_by_tool` tag bypasses the tool picker filtering.

## Best solution

Remove or narrow the `extension_installed_by_tool` auto-enable path in `extensions/copilot/src/extension/tools/vscode-node/toolsService.ts` (line 349) so it only fires for tools from extensions installed *during* the current session, not pre-installed ones. Additionally, respect `when: "false"` clauses during request-time tool filtering.
