# Issue #292260 — Notebook tools added without being enabled by the user

[https://github.com/microsoft/vscode/issues/292260](https://github.com/microsoft/vscode/issues/292260)

## Summary

Still reproduces on main — the `extension_installed_by_tool` tag bypass in toolsService.ts auto-enables tools (e.g. Jupyter notebook tools) even when the user hasn't selected them. The related fix PR #303663 is still open and unmerged.

## Change summary

Notebook-related tools from the Jupyter extension are sent to the LLM even when the user's agent configuration only enables specific tools or none at all.

## Best solution

Remove or gate the `extension_installed_by_tool` auto-enable path in `extensions/copilot/src/extension/tools/vscode-node/toolsService.ts` (line 349) so it only applies to tools installed *during* the current session run, not pre-installed extensions. Additionally, respect `when: "false"` clauses at request time (the second code path described in the issue). PR https://github.com/microsoft/vscode/pull/303663 appears to address the `when` clause part.
