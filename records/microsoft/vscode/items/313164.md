# Issue #313164 — agents: ssh ProxyCommand

[https://github.com/microsoft/vscode/issues/313164](https://github.com/microsoft/vscode/issues/313164)

## Summary

Team-authored feature request (hard keep-open). SSH ProxyCommand is not implemented in the Agents window's SSH flow on current main — the gap remains. Two related issues (#316493, #317527) were closed as duplicate/info-needed, pointing back here as the canonical tracker.

## Change summary

The Agents window SSH flow does not honor SSH config ProxyCommand directives, causing connections to fail for hosts that rely on proxy commands (e.g. GitHub Codespaces via GH CLI).

## Best solution

Implement ProxyCommand (and ideally ProxyJump) support in the agent host's SSH connection logic under `src/vs/platform/agentHost/`. This is the canonical issue for the work — the two related closed issues both point here. The workaround noted by a team member (local SSH proxy without ProxyCommand) confirms the gap is real and scoped to proxy-style directives.
