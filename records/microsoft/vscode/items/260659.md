# Issue #260659 — Only provide task output from `beginsPattern` when possible

[https://github.com/microsoft/vscode/issues/260659](https://github.com/microsoft/vscode/issues/260659)

## Summary

Team-authored feature request to truncate task output at the `beginsPattern` boundary so agents only see the latest build cycle's output, not stale errors from earlier runs. Active discussion between team members on implementation approach — keeping open for maintainer judgment.

## Change summary

When retrieving task output for chat agents, truncate to the most recent `beginsPattern` match so stale build errors from previous cycles are excluded.

## Best solution

Implement truncation of task output at the last `beginsPattern` match (when one is defined for the task's problem matcher) so the `get_task_output` tool returns only the current build cycle's output. The char-cap in https://github.com/microsoft/vscode/issues/261272 is a complementary mitigation but doesn't address the semantic boundary.
