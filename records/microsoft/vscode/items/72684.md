## Summary

Partially addressed — the `never` option was added in 2022, but the specific request for a mode that hides *all* folding controls (including collapsed indicators) on non-hover while still allowing mouseover interaction remains unimplemented. Low engagement; keep open as a backlog feature request.

## Change summary

Request for a `showFoldingControls` mode that hides both expanded and collapsed fold indicators in the gutter unless the mouse hovers over it.

## Best solution

Add a fourth enum value (e.g. `mouseoverOnly`) to `editor.showFoldingControls` in `src/vs/editor/common/config/editorOptions.ts` and handle it in `foldingDecorations.ts` to auto-hide even collapsed-region icons unless hovered. Alternatively, confirm that the existing `mouseover` + `never` options suffice and close as covered.
