# Issue #290498 — Support linking paths/URIs in the exception popup window

[https://github.com/microsoft/vscode/issues/290498](https://github.com/microsoft/vscode/issues/290498)

## Summary

Valid feature request: the exception widget already uses LinkDetector with built-in regex patterns (WEB_LINK_REGEX, PATH_LINK_REGEX), but doesn't support extension-provided link providers like the terminal does. The gap remains on current main.

## Change summary

Allow extension-provided link providers (like those in the terminal) to linkify paths/URIs in the debug exception popup widget.

## Best solution

Extend the exception widget in `src/vs/workbench/contrib/debug/browser/exceptionWidget.ts` to either improve the built-in `LinkDetector` regex patterns to cover more path formats (e.g. Dart stack traces), or expose a mechanism for debug adapters/extensions to provide link information for exception stack traces. The existing `LinkDetector` in `src/vs/workbench/contrib/debug/browser/linkDetector.ts` handles web URLs and some file paths already, but the regex may not match all stack trace formats across languages.
