# Issue #226835 — SCM Graph - add support for async filtering

[https://github.com/microsoft/vscode/issues/226835](https://github.com/microsoft/vscode/issues/226835)

## Summary

The async find/filter infrastructure for async data trees was added in commit c7e0153e527 (September 2024), but the SCM graph view has not yet adopted it. The feature request to wire up async filtering in the SCM graph remains unimplemented.

## Change summary

The SCM Graph's type-to-filter only searches already-loaded items; it should load additional items asynchronously when the viewport isn't full after filtering.

## Best solution

Adopt the `IAsyncFindProvider` interface (from `src/vs/base/browser/ui/tree/asyncDataTree.ts`) in the SCM history view's `WorkbenchCompressibleAsyncDataTree` setup in `src/vs/workbench/contrib/scm/browser/scmHistoryViewPane.ts`, so that filtering triggers async data loading when results are sparse.
