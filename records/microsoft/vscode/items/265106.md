# Issue #265106 — why does organize import suggests the true and false if it only accepts the string

[https://github.com/microsoft/vscode/issues/265106](https://github.com/microsoft/vscode/issues/265106)

## Summary

Still present on main: the deprecated `true`/`false` values are listed in the `codeActionsOnSave` schema enum alongside their string replacements. A team member acknowledged this and suggested migrating users, but no change has landed yet.

## Change summary

The `editor.codeActionsOnSave` setting suggests deprecated `true` and `false` values in completions even though string equivalents (`"always"`/`"explicit"`/`"never"`) are the intended values.

## Best solution

Remove `true` and `false` from the `enum` arrays in `createCodeActionsAutoSave` and `createNotebookCodeActionsAutoSave` in `src/vs/workbench/contrib/codeActions/browser/codeActionsContribution.ts`, and add a migration in `src/vs/editor/browser/config/migrateOptions.ts` to silently convert existing `true`→`"explicit"` and `false`→`"never"` values. The team member's comment already outlines this plan.
