# Issue #226802 — SCM Graph - Improve name/description of theme keys

[https://github.com/microsoft/vscode/issues/226802](https://github.com/microsoft/vscode/issues/226802)

## Summary

The core concern — removing the 'Group' terminology — was addressed in commit c2c032e0da1 (shipped in 1.94.0), which renamed `historyItemGroupLocal`/`Remote`/`Base` to `historyItemRefColor`/`RemoteRefColor`/`BaseRefColor`. However, some of the issue's broader naming suggestions remain unaddressed: the 'history' prefix is still present in all keys, and the suffix convention is inconsistent (`Color` on some keys vs `Foreground`/`Background` on others). Keeping open as an under-discussion polish item for the SCM area owner.

## Change summary

Improve the naming and descriptions of SCM graph theme color keys to be more consistent and descriptive.

## Best solution

The SCM area owner should decide whether to do a second naming pass — standardizing the suffix convention (e.g. always using `Foreground`/`Background` instead of `Color`) and potentially dropping the `historyItem` prefix from the `scmGraph.*` keys. This is a breaking change for theme authors, so it should ideally happen before the color keys are considered stable API.
