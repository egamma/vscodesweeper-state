# Issue #89850 — [html] HTML breadcrumbs don't work right when optional end tags omitted

[https://github.com/microsoft/vscode/issues/89850](https://github.com/microsoft/vscode/issues/89850)

## Summary

Still an open gap — the HTML language service parser does not handle optional end tags, and the upstream tracking issue (vscode-html-languageservice#63) remains open with no fix merged.

## Change summary

HTML breadcrumbs treat elements with omitted optional end tags as deeply nested rather than as siblings, making breadcrumbs unusable for tidied HTML.

## Best solution

Implement optional-end-tag awareness in the HTML parser at https://github.com/microsoft/vscode-html-languageservice (issue #63). The fix belongs in the upstream `htmlParser.ts` where tag closing logic lives; once shipped there it flows into VS Code's built-in HTML extension.
