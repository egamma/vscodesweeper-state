# Issue #89850 — [html] HTML breadcrumbs don't work right when optional end tags omitted

[https://github.com/microsoft/vscode/issues/89850](https://github.com/microsoft/vscode/issues/89850)

## Summary

Still unresolved — the vscode-html-languageservice parser has no support for optional end tags. The upstream issue (microsoft/vscode-html-languageservice#63) remains open, and the current bundled parser code confirms no such logic exists on main.

## Change summary

HTML breadcrumbs nest incorrectly when optional end tags (e.g. </p>, </li>) are omitted, because the parser treats unclosed elements as containing all subsequent siblings.

## Best solution

Implement optional/implicit end tag awareness in the HTML parser at https://github.com/microsoft/vscode-html-languageservice (tracked upstream as microsoft/vscode-html-languageservice#63). The fix belongs in `htmlParser.ts` of that service, adding knowledge of which tags auto-close when a sibling or parent opens.
