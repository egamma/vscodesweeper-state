# Issue #255503 — Unable to abandon task with ${input} variable type command

[https://github.com/microsoft/vscode/issues/255503](https://github.com/microsoft/vscode/issues/255503)

## Summary

Confirmed bug still present on main: the command-type input handler at line 343-344 wraps undefined in an object instead of returning bare undefined, preventing task cancellation. An open PR (#307980) addresses it.

## Change summary

Tasks using ${input} variables of type 'command' cannot be abandoned when the command returns undefined, because the result is wrapped in an object instead of propagating the cancellation signal.

## Best solution

Land PR https://github.com/microsoft/vscode/pull/307980 (or equivalent) that returns `undefined` from `showUserInput` when the command result is undefined/null, matching the behavior of pickString and promptString inputs.

## Evidence

- **bug on main:** src/vs/workbench/services/configurationResolver/browser/baseConfigurationResolverService.ts line 343-344: `return { value: result, input: info }` wraps undefined in an object, so the cancellation check at line 194 (`if (result === undefined)`) never fires for command inputs.
- **contrast with pickString:** Line 285 correctly returns bare `undefined` when the user cancels a pickString/promptString input: `return resolvedInput !== undefined ? { value: resolvedInput, input: info } : undefined`.
- **partial fix shipped:** Commit 0a1e0c9205c (Feb 2026) fixed cancellation for debug launches via `resolveWithInteractionReplace`, but did not fix the underlying `showUserInput` return for command-type inputs.
