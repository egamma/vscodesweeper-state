## Summary

Team-authored issue requesting a cross-editor find experience in the multi-diff editor. No implementation exists on current main — the multi-diff editor widget has no aggregated find/search support; Cmd+F only triggers the standard per-editor FindWidget.

## Change summary

Add a find/search experience that spans all editors in the multi-diff editor, rather than being scoped to a single embedded editor.

## Best solution

Implement a multi-diff-level find widget (or find controller) that searches across all visible diff editors in the multi-diff view, likely in `src/vs/editor/browser/widget/multiDiffEditor/` with coordination from a new find model that aggregates matches from each embedded editor.
