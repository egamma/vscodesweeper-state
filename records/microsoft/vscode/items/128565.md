# Issue #128565 — [semantic highlighting] Source language suffix should be added to semantic token fallback scopes when looked up

[https://github.com/microsoft/vscode/issues/128565](https://github.com/microsoft/vscode/issues/128565)

## Summary

Team-authored feature request in Backlog; the enhancement to append language ID suffixes to semantic token fallback scopes has not been implemented on main. Extensions can work around it by providing explicit per-language scopes.

## Change summary

VS Code should append the source language suffix (e.g. `.cpp`, `.cs`) to TextMate scopes when looking up semantic token fallback scopes, so themes with language-specific colorization rules work with semantic highlighting.

## Best solution

Add a language-to-scope-suffix mapping (possibly via the `semanticTokenScopes` contribution point as aeschli suggested) and use it in `colorThemeData.ts` when resolving fallback TextMate scopes for semantic tokens. This primarily benefits theme authors targeting language-specific scopes.
