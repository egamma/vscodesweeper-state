# Issue #244271 — Pasting text from one note into chat adds image attachment 

[https://github.com/microsoft/vscode/issues/244271](https://github.com/microsoft/vscode/issues/244271)

## Summary

Still reproduces on main: PasteImageProvider in chatPasteProviders.ts never sets yieldTo, so when clipboard contains both text and image (as OneNote does), the image attachment is created alongside the text instead of deferring to the text paste.

## Change summary

Pasting text from OneNote (or any app that puts both text and image on the clipboard) into chat creates an unwanted image attachment alongside the pasted text.

## Best solution

Add `edit.yieldTo = [{ kind: HierarchicalKind.Empty.append('text', 'plain') }]` to the PasteImageProvider's edit (line 144 of chatPasteProviders.ts), mirroring the pattern already used by PasteCodeAttachmentProvider at line 401. This ensures text/plain is preferred when both text and image are on the clipboard, while still allowing explicit image paste.
