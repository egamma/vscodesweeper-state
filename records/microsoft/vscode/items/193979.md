# Issue #193979 — Diff Editor v2: setting for expanded unchanged regions by default

[https://github.com/microsoft/vscode/issues/193979](https://github.com/microsoft/vscode/issues/193979)

## Summary

Team-authored feature request for a diff editor setting to expand all unchanged regions by default. No such setting exists on current main — the existing `hideUnchangedRegions.enabled` controls whether regions are hidden, but there's no option to start with all collapsed regions expanded. Keeping open per team-author rule.

## Change summary

Add a setting so the diff editor shows all unchanged regions expanded by default, even when hideUnchangedRegions is enabled.

## Best solution

Add a new option (e.g. `diffEditor.hideUnchangedRegions.defaultExpanded`) in the diff editor configuration that, when true, renders all unchanged regions in their expanded state on open. The implementation would live in `src/vs/editor/browser/widget/diffEditor/features/hideUnchangedRegionsFeature.ts` and the schema in `src/vs/editor/common/config/editorConfigurationSchema.ts`.
