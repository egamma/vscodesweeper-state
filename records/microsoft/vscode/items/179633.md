# Issue #179633 — Need a UI message when an extension fails to initialize

[https://github.com/microsoft/vscode/issues/179633](https://github.com/microsoft/vscode/issues/179633)

## Summary

Keeping this open because it is a team-authored issue already labeled `under-discussion`; VS Code policy requires explicit maintainer judgment rather than cleanup closure. The thread also shows maintainers intentionally debating whether activation failures should become notifications, stay in the Extensions UI, or wait for newer extension UI work.

## Change summary

Request to make extension activation failures visible in UI for regular users instead of only logging them to the Developer Tools console in retail builds.

## Best solution

Extension activation failures should have a clearer, user-understandable surface in the Extensions area or a future extension notification center without making users think VS Code itself failed; the extension-host and Extensions UI owners should decide the product treatment.
