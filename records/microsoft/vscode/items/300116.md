# Issue #300116 — Agent using gh CLI always opens alt buffer

[https://github.com/microsoft/vscode/issues/300116](https://github.com/microsoft/vscode/issues/300116)

## Summary

Team-authored issue, still reproduces on main: `toolTerminalCreator.ts` sets `GIT_PAGER` but not `GH_PAGER`, so `gh` CLI commands open the alt buffer in agent terminals. An open WIP PR exists but has not merged.

## Change summary

When the agent runs `gh` CLI commands in the tool terminal, the gh pager opens the alternate screen buffer, causing chaos in the terminal output.

## Best solution

Add `GH_PAGER: 'cat'` (or `''`) to the env block in `src/vs/workbench/contrib/terminalContrib/chatAgentTools/browser/toolTerminalCreator.ts`, mirroring what `agentHostTerminalManager.ts` already does. The open WIP PR https://github.com/microsoft/vscode/pull/300525 is the implementation candidate.
