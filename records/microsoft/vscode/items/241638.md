# Issue #241638 — Inconsistent Debug Console Hover Styling: list.hoverBackground Applies, but list.hoverForeground Does Not

[https://github.com/microsoft/vscode/issues/241638](https://github.com/microsoft/vscode/issues/241638)

## Summary

Still reproduces on main: the debug console's per-severity foreground color rules have higher CSS specificity than the generic list hover foreground rule, so `list.hoverForeground` is overridden while `list.hoverBackground` (which targets a different property not overridden by debug styles) works fine.

## Change summary

Setting `list.hoverForeground` in `workbench.colorCustomizations` does not apply to Debug Console entries on hover, even though `list.hoverBackground` does.

## Best solution

In `src/vs/workbench/contrib/debug/browser/debugColors.ts`, the hover-state foreground needs to be respected: either add `:hover` overrides that yield to `list.hoverForeground` when set, or register dedicated `debugConsole.hoverForeground`/`debugConsole.hoverBackground` color tokens. The fix is localized to the debug console's theme contribution.
