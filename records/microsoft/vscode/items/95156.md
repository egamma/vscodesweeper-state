## Summary

Valid feature request for a 'regions only' folding strategy. The current `editor.foldingStrategy` setting only offers `auto` and `indentation` — no option exists to restrict folding exclusively to region markers. Zero community upvotes and over five years old with no traction, but it's in Backlog so it stays open.

## Change summary

Request to add a folding mode that uses only region markers (e.g., #region/#endregion) and disables all other folding.

## Best solution

Add a third value (e.g. `'regions'`) to the `editor.foldingStrategy` enum in `src/vs/editor/common/config/editorOptions.ts` and wire it into the folding contribution so that only the region-based range provider is used. Alternatively, this could be an `under-discussion` candidate for community interest assessment.
