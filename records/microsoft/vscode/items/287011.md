# Issue #287011 — Decoration not applied to diff editors when "fontWeight" decoration is set

[https://github.com/microsoft/vscode/issues/287011](https://github.com/microsoft/vscode/issues/287011)

## Summary

Confirmed on current main: the diff editor sets `allowVariableFonts = false`, which causes `filterFontDecorations` to completely exclude any decoration with `affectsFont` (triggered by fontWeight/fontSize/fontFamily/fontStyle) from the rendering pipeline — even though the decoration may have a valid `inlineClassName` for non-font styling like backgroundColor.

## Change summary

Decorations that include `fontWeight` (or other font properties) are completely invisible in diff editors, even though non-font aspects like background color should still render.

## Best solution

The diff editor should either not filter out the entire decoration but only suppress the font-affecting rendering aspects, or it should allow decorations with `affectsFont` to pass through the pipeline while ignoring the `hasVariableFonts` flag for font measurement. The fix would likely be in `src/vs/editor/browser/widget/diffEditor/components/diffEditorEditors.ts` (removing the blanket `allowVariableFonts = false`) or in the filtering logic in `src/vs/editor/common/model/intervalTree.ts` to separate font-filtering from decoration-filtering.
