# Issue #287011 — Decoration not applied to diff editors when "fontWeight" decoration is set

[https://github.com/microsoft/vscode/issues/287011](https://github.com/microsoft/vscode/issues/287011)

## Summary

Confirmed bug: the diff editor sets `allowVariableFonts: false`, causing the decoration filtering logic to drop the *entire* decoration (not just the font properties) when `affectsFont` is set. This means any decoration that includes `fontWeight` (or `fontSize`/`fontFamily`/`fontStyle`) is completely invisible in diff editors, even though non-font aspects (like backgroundColor) should still render.

## Change summary

Decorations with `fontWeight` set are entirely filtered out in diff editors instead of having only the font-related CSS stripped while preserving other visual properties.

## Best solution

The `filterFontDecorations` path in the interval tree search should not wholesale exclude the decoration node; instead, the rendering layer should strip only font-affecting CSS properties (`fontWeight`, `fontSize`, `fontFamily`, `fontStyle`) from the resolved decoration options while preserving className, backgroundColor, inlineClassName, etc. The fix likely lives in how `getNodeAffectsFont` filtering interacts with `intervalSearch`/`search` in `src/vs/editor/common/model/intervalTree.ts` and how decorations are resolved into CSS in `abstractCodeEditorService.ts`.
