# Issue #272827 — Chat response: fold multiple changes in a file into one pill

[https://github.com/microsoft/vscode/issues/272827](https://github.com/microsoft/vscode/issues/272827)

## Summary

Team-authored polish request that remains unaddressed. The chat response renders a separate pill for each edit to a file (keyed by URI + undoStopId), so multiple edits to the same file produce multiple pills. The requested consolidation — one pill per file — has not been implemented on current main.

## Change summary

When a chat response edits the same file multiple times, each edit renders as a separate pill instead of being folded into a single per-file pill.

## Best solution

Consolidate the external-edit pills by file URI rather than by URI+undoStopId. The rendering logic in `chatListRenderer.ts` (around line 3379) builds a partId that includes `undoStopId`, causing per-edit granularity. Grouping by URI alone (or aggregating diff stats per file) and rendering one pill per file would match the mockup bpasero provided. The `ChatThinkingContentPart` appendItem dedup and the `ChatExternalEditContentPart` would need coordinated changes.
