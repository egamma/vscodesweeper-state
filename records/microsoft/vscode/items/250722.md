# Issue #250722 — C# wrong extra indentation in latest release

[https://github.com/microsoft/vscode/issues/250722](https://github.com/microsoft/vscode/issues/250722)

## Summary

Real auto-indentation bug for C# that remains unfixed on main. The C# language configuration lacks explicit indentationRules, relying on generic bracket heuristics that fail in multi-block editing scenarios. Already triaged with tree-sitter-autoindent-candidate label.

## Change summary

Auto-indentation incorrectly adds extra indentation to lines following edited blocks in C#, particularly when wrapping existing code in new control structures.

## Best solution

The long-term fix is tree-sitter-based auto-indentation (as the label suggests). Short-term, adding proper indentationRules to extensions/csharp/language-configuration.json could mitigate some cases. The editor-autoindent area owner should assess whether the generic indent logic in src/vs/editor/common/languages/autoIndent.ts can better handle the 'wrap existing code in a new block' pattern.
