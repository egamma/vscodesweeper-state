# Issue #293800 — Remote Tunnels fails to open workspace when path contains Chinese(non-ascii) characters.

[https://github.com/microsoft/vscode/issues/293800](https://github.com/microsoft/vscode/issues/293800)

## Summary

Valid bug: non-ASCII characters in tunnel workspace paths are not decoded before workspace validation, causing 'Workspace does not exist' errors. No fix found on main; related open issue in vscode-remote-release confirms this is unresolved.

## Change summary

Remote Tunnels fails to open workspaces when the path contains non-ASCII (e.g., CJK) characters because the URL-encoded path is not decoded before the workspace existence check.

## Best solution

The tunnel path needs URI-decoding before being passed to the workspace resolver. The fix likely lives in how vscode.dev parses the tunnel URL path into a folder URI — the path segment should be decoded (similar to the `decodeURIComponent` already used in `webClientServer.ts` for resource paths). May partly live in the tunnel extension or vscode.dev frontend rather than this repo.
