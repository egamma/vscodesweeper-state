# Issue #281116 — Feature Request: Allow handsoff with resetting context window for custom agent

[https://github.com/microsoft/vscode/issues/281116](https://github.com/microsoft/vscode/issues/281116)

## Summary

This is a legitimate feature request for context/history control during agent handoffs. The capability does not exist on current main — handoff frontmatter supports only label, agent, prompt, send, showContinueOn, and model. Milestone is Backlog (accepted), so it stays open.

## Change summary

Allow custom agent handoffs to control whether conversation history is preserved, summarized, or cleared when delegating to another agent.

## Best solution

Add a `clearHistory` or `historyMode` property to the handoff frontmatter schema (in `src/vs/workbench/contrib/chat/common/promptSyntax/`) supporting modes like 'preserve', 'summarize', and 'clear'. The execution path in `chatExecuteActions.ts` would need to respect this when initiating the handoff session.
