# Issue #252640 — Dragging the View Container title in a singleton View Contain should move the View Container, not the View

[https://github.com/microsoft/vscode/issues/252640](https://github.com/microsoft/vscode/issues/252640)

## Summary

Valid UX polish request: dragging the title bar of a singleton View Container should move the entire container, not just the single view inside it. Still unfixed on main; already labeled under-discussion and workbench-views, which is correct routing.

## Change summary

When a View Container has only one view, dragging the merged title bar moves only the view rather than the whole container, causing confusing behavior when additional views are later added.

## Best solution

Update the drag logic in viewPaneContainer.ts (or the compositeBar/dnd handling) so that when a View Container is in singleton mode (merged title), dragging the title initiates a container-level move rather than a view-level move. This keeps the mental model consistent with multi-view containers.
