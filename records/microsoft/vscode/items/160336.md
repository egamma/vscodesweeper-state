# Issue #160336 — Connecting via code-server shows the cwd in the terminal tab incorrectly

[https://github.com/microsoft/vscode/issues/160336](https://github.com/microsoft/vscode/issues/160336)

## Summary

Still reproducible on main: the terminal tab cwdFolder comparison uses client-side `path.resolve` and `URI.from` which can produce incorrect results when connecting cross-OS via code-server (macOS client → Windows remote), causing the cwd to be shown in the tab even when it matches the workspace root.

## Change summary

Terminal tab incorrectly displays the cwd when connecting cross-platform via code-server because the path comparison uses client-OS path resolution on a remote-OS path.

## Best solution

Fix the cwdFolder comparison in `TerminalLabelComputer.computeLabel` to use the remote OS path semantics (or compare normalized URIs directly) instead of relying on client-side `path.resolve`. The fix belongs in `src/vs/workbench/contrib/terminal/browser/terminalInstance.ts` around line 2785–2796.
