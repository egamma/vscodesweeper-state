# Issue #318355 — In Agents sidebar, Allow buttons push session list down, causing misclicks on Done

[https://github.com/microsoft/vscode/issues/318355](https://github.com/microsoft/vscode/issues/318355)

## Summary

Team-authored issue describing a real UX hazard: dynamically appearing Allow buttons in the sessions list push rows down, causing misclicks on the destructive Done action. Keeping open for maintainer judgment.

## Change summary

In the Agents sidebar, when a session's Allow button appears while hovering another session's Allow button, the layout shift moves the Done button under the cursor, leading to accidental archival.

## Best solution

Reserve stable vertical space for the approval row in each session list item (or use a fixed-height slot) so that Allow buttons appearing on other sessions do not shift sibling rows. Alternatively, add a confirmation step or undo mechanism for the Done action to mitigate the consequences of misclicks. The approval button rendering is in `src/vs/sessions/contrib/sessions/browser/views/sessionsList.ts` around line 372 (the `approvalRow` / `approvalButtonContainer` template elements).
