Well-established feature request with 168 upvotes and multiple closed duplicates pointing here — this is the canonical issue for showing file metadata (size, date modified) in the explorer. No implementation work found on current main; the feature gap remains real. Should stay open in the Backlog.

## Change summary

Show file size, last modified date, and other file attributes in the file explorer sidebar.

## Best solution

The explorer tree currently has no support for displaying auxiliary file metadata columns. Implementation would likely touch the explorer view model and renderers in `src/vs/workbench/contrib/files/browser/views/` to add optional inline or column-based metadata display, gated behind a user setting. The UX question raised by a team member (comment 8 — hover vs. inline vs. columns, configurability) remains the key design decision before implementation.
