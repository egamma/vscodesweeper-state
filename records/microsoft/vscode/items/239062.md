# Issue #239062 — Keyboard Enter for "Force Checkout" not working

[https://github.com/microsoft/vscode/issues/239062](https://github.com/microsoft/vscode/issues/239062)

## Summary

Still reproduces on main — the dialog's Enter key handling returns early ('leave default handling') at line 363 of dialog.ts without explicitly activating the focused button, so non-default buttons like 'Force Checkout' don't fire on Enter. Two open PRs address this; milestone is 'On Deck'.

## Change summary

Pressing Enter on a focused non-default button in a modal dialog (e.g. 'Force Checkout') does not activate it; it falls through or triggers the wrong action.

## Best solution

Land a fix in `src/vs/base/browser/ui/dialog/dialog.ts` around line 350–363 to explicitly resolve the dialog with the currently focused button's index when Enter is pressed and no input field has focus. Two open PRs (#308735, #319482) offer competing approaches — evaluate and merge one.
