# Issue #325912 — Niri desktop environment not found when using verbose command

[https://github.com/microsoft/vscode/issues/325912](https://github.com/microsoft/vscode/issues/325912)

## Summary

This is the same Niri desktop-environment-not-detected issue that was already filed as #285777 and closed as a duplicate of the canonical upstream tracking issue #187338. The "desktop environment: (unknown)" log line comes from Chromium's os_crypt component, not VS Code code. Proposing duplicate close against #187338.

## Change summary

Niri compositor is not recognized as a desktop environment by Chromium's os_crypt, causing keyring/password storage detection to fail.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/187338, which tracks the upstream Chromium/Electron issue for unrecognized Linux desktop environments and keyring detection. The fix belongs in Chromium's `key_storage_util_linux.cc`, not in VS Code.
