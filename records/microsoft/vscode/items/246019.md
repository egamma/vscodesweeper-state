# Issue #246019 — lsp signature_helper Activity parameter highlight position error

[https://github.com/microsoft/vscode/issues/246019](https://github.com/microsoft/vscode/issues/246019)

## Summary

Real bug, still present on main: when `ParameterInformation.label` is a string, the regex in `getParameterLabelOffsets` finds only the first occurrence in the signature, which can be wrong when the parameter name appears earlier (e.g., in the function name or a preceding parameter). Milestone is 'On Deck' — must keep open.

## Change summary

Signature help highlights the wrong parameter position when the parameter name string appears multiple times in the signature label.

## Best solution

Fix `getParameterLabelOffsets` in `src/vs/editor/contrib/parameterHints/browser/parameterHintsWidget.ts` to account for the active parameter index when doing string-based matching — e.g., skip past earlier parameters' positions before matching the Nth parameter. Alternatively, document that language servers should use the `[number, number]` tuple form for unambiguous highlighting.
