## Summary

Filed in 2020 against VS Code 1.51; since then, the SSH_ASKPASS handler (commit a6272c94161, Sept 2022) was implemented to intercept SSH prompts and prevent hangs. The report is too old to assume it still reproduces on current builds.

## Change summary

Git push hangs indefinitely in WSL 2 when the SSH key has a non-default filename, even though the key is loaded in ssh-agent.

## Best solution

The SSH_ASKPASS handler (shipped ~1.72) should have resolved the hang by intercepting SSH prompts. Ask the reporter to confirm on a current build; if the issue persists, it's likely an SSH_AUTH_SOCK propagation problem in the WSL environment rather than a VS Code bug.
