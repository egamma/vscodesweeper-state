# Issue #303444 — Large multi-file diff editor freezes entire workbench for seconds

[https://github.com/microsoft/vscode/issues/303444](https://github.com/microsoft/vscode/issues/303444)

## Summary

Team-authored issue; the freeze is real and traced in code (no limit on DocumentDiffItemViewModel creation for large file sets). A fix branch exists but hasn't merged to main yet — keep open until merged.

## Change summary

Opening a multi-file diff with thousands of files freezes the workbench because all DocumentDiffItemViewModels and text models are created eagerly.

## Best solution

Merge the work from `copilot/limit-text-models-to-100` (commits 80310599482, 65fb6d91b7f, c534c71df01) which caps the multi-diff editor at a configurable maxFileCount and shows a notification when the limit is hit, preventing eager creation of thousands of text models.
