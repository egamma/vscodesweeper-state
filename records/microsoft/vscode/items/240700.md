# Issue #240700 — Ability to change Github Sign Ins between profiles

[https://github.com/microsoft/vscode/issues/240700](https://github.com/microsoft/vscode/issues/240700)

## Summary

This is the canonical issue for per-profile GitHub account preferences. The team has discussed and agreed on an implementation path (StorageScope.PROFILE), but it has not been implemented yet on main — multiple duplicates (282986, 290237, 301084, 325096) were closed pointing here. Keeping open as the tracked work item.

## Change summary

Allow GitHub account preferences to be stored per VS Code profile rather than globally, so switching profiles also switches the active GitHub account for extensions like Copilot.

## Best solution

Implement the change discussed in comments 7–8: in `src/vs/workbench/services/authentication/browser/authenticationExtensionsService.ts`, change the fallback storage scope from `StorageScope.APPLICATION` to `StorageScope.PROFILE` so account preferences persist per profile. Additionally, surface this in the Profile editor UI as Tyler suggested in comment 4.
