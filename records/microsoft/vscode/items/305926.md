# Issue #305926 тАФ Git is watching all worktrees recursively

[https://github.com/microsoft/vscode/issues/305926](https://github.com/microsoft/vscode/issues/305926)

## Summary

The core issue тАФ Git auto-detecting and recursively watching all worktrees by default тАФ was fixed by changing the `git.detectWorktrees` default to `false` in commit f92a4853f7f, shipped in 1.115.0. The expensive recursive file watching of worktree directories no longer happens out of the box.

## Change summary

Git extension auto-detected worktrees and set up expensive recursive file watchers on all of them, causing performance bottlenecks.

## Best solution

Keep the shipped default (`git.detectWorktrees: false`) from commit f92a4853f7f. If the Copilot Chat extension code path that opens worktree repositories still needs optimization, that should be tracked as a separate, more targeted issue.
