# Issue #287012 — Selection decoration incorrectly drawn below custom decorations, hindering usability

[https://github.com/microsoft/vscode/issues/287012](https://github.com/microsoft/vscode/issues/287012)

## Summary

Confirmed on current main: the overlay rendering order in view.ts places SelectionsOverlay before DecorationsOverlay, so extension-provided background decorations paint on top of the selection highlight, making selection invisible when decorations use opaque backgrounds.

## Change summary

Selection highlight is invisible behind extension-provided decorations that use opaque background colors, because decorations render on top of selections.

## Best solution

The fix likely belongs in the decoration rendering system — either reorder overlays so selections paint last (risky for search/word highlights), introduce a z-index layer that accounts for selection vs custom decorations, or require custom decorations to use semi-transparent backgrounds. Area owner decision needed in `src/vs/editor/browser/view.ts` (overlay ordering) and potentially `src/vs/editor/browser/viewParts/decorations/decorations.ts`.
