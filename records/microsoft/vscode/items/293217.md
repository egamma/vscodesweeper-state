# Issue #293217 — Enable word wrap for tool invocation parameters in GitHub Copilot Chat

[https://github.com/microsoft/vscode/issues/293217](https://github.com/microsoft/vscode/issues/293217)

## Summary

Feature request to have tool invocation parameter editors respect the `chat.editor.wordWrap` setting or default to word wrap on. The current compact-then-expand UX is intentional, but the request to honor the user's wrap preference is reasonable polish.

## Change summary

Tool invocation parameter displays ignore the `chat.editor.wordWrap` setting and require horizontal scrolling until the user clicks 'See more'.

## Best solution

Have the tool confirmation input editor in `chatToolConfirmationSubPart.ts` (line ~196) read `chat.editor.wordWrap` from configuration instead of hardcoding `'off'`, or at minimum apply wrap in the initial compact view when the content exceeds the viewport width.
