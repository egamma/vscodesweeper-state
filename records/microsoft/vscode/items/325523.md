# Issue #325523 — Copilot chat shows ignored-files troubleshooting text in grep search tool completion UI

[https://github.com/microsoft/vscode/issues/325523](https://github.com/microsoft/vscode/issues/325523)

## Summary

Confirmed on current main: the grep-style search tool leaks internal agent troubleshooting text into the user-visible `toolResultMessage` when no matches are found and `includeIgnoredFiles` is false. No fix has landed.

## Change summary

The grep search tool's no-results UI shows internal LLM troubleshooting guidance ('Your search pattern might be excluded completely by either the search.exclude settings or .*ignore files…') to the end user in the chat completion widget.

## Best solution

In `extensions/copilot/src/extension/tools/node/findTextInFilesTool.tsx`, split the `noMatchInstructions` into two: keep the detailed troubleshooting text in the tool result content (for the LLM to act on), but set `toolResultMessage` to a short user-facing string like 'No matches found.' only. The `errorResult` call at line 187 should pass the user-facing message to `toolResultMessage` and the agent guidance as prompt content.
