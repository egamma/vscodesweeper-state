# Issue #200916 — SCM - Sorting Changes by FileSystem's "Modified Date"

[https://github.com/microsoft/vscode/issues/200916](https://github.com/microsoft/vscode/issues/200916)

## Summary

Legitimate backlog feature request with 39 upvotes, well above the 20-vote threshold. The SCM changes view currently supports sorting by Path, Name, and Status only — no filesystem "Modified Date" sort exists on current main. Issue #204283 was closed as a duplicate of this one, confirming this is the canonical tracker.

## Change summary

Add the ability to sort SCM changes by filesystem modification date.

## Best solution

Add a new `ViewSortKey` value (e.g. `ModifiedDate`) to the SCM view sort enum in `src/vs/workbench/contrib/scm/browser/scmViewPane.ts`, wire it into the sort menu and `scm.defaultViewSortKey` configuration, and implement the comparator using `IFileService.stat()` to retrieve file modification timestamps. The sort option should appear alongside the existing Path/Name/Status options in the SCM view sort dropdown.
