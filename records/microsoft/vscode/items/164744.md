# Issue #164744 — [scss] Sass modules intellisense when using @use at-rules.

[https://github.com/microsoft/vscode/issues/164744](https://github.com/microsoft/vscode/issues/164744)

## Summary

Legitimate feature request for Sass builtin module completions via @use. The built-in CSS language service does not currently provide this. A third-party extension (Some Sass by wkillerud) covers it, but the request for built-in support remains valid.

## Change summary

Request intellisense/autocomplete for Sass built-in modules (e.g. sass:math) when using @use at-rules in .scss files.

## Best solution

Either enhance vscode-css-languageservice to provide completions for Sass builtin modules referenced via @use, or consider whether the Some Sass extension (https://github.com/wkillerud/vscode-scss) adequately fills this gap making this *extension-candidate*. Given the low community traction, leaving as a backlog feature request with the existing workaround noted is reasonable.
