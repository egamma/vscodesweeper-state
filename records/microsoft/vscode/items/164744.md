## Summary

Feature request for deeper @use intellisense in the built-in SCSS language service remains unimplemented on main. An extension (Some Sass) provides this functionality. With only 5 upvotes and an available extension workaround, this stays open in Backlog as-is.

## Change summary

The built-in SCSS language service does not provide completions for Sass built-in modules (e.g. sass:math) or symbols imported via @use at-rules.

## Best solution
