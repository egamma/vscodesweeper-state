# Issue #300484 — Proper RTL (Right-to-Left) Support for Chat Responses

[https://github.com/microsoft/vscode/issues/300484](https://github.com/microsoft/vscode/issues/300484)

## Summary

Verified duplicate: this requests RTL/BiDi support in chat responses, which is already tracked by the open canonical issue https://github.com/microsoft/vscode/issues/273696 (Backlog milestone, feature-request). An open community PR https://github.com/microsoft/vscode/pull/314365 lists both issues in its Fixes list and addresses the exact capabilities requested here.

## Change summary

Chat responses in RTL languages (Hebrew, Arabic) render incorrectly due to LTR-locked containers, causing broken word order and misplaced punctuation in mixed-language text.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/273696, which owns the same RTL-in-chat work and is already in the Backlog. The open PR https://github.com/microsoft/vscode/pull/314365 introduces a configurable `editor.textDirection` setting and propagates it to chat surfaces, covering automatic direction detection, BiDi support, and manual override — all three points this issue requests.
