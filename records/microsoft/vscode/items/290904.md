# Issue #290904 — Expanding thinking container is disorienting

[https://github.com/microsoft/vscode/issues/290904](https://github.com/microsoft/vscode/issues/290904)

## Summary

Team-authored UX polish issue about disorienting scroll behavior when expanding large thinking containers. A `FixedScrolling` mode with max-height and internal scrolling exists (since Oct 2025) but is opt-in — the default `Collapsed` mode still exhibits both reported problems. Keeping open per team-authored rule.

## Change summary

Expanding a large thinking container in chat auto-scrolls the clicked label out of view and dumps too much content inline, making it hard to navigate.

## Best solution

For problem 1: suppress auto-scroll when the user manually expands a thinking container (likely in `chatListRenderer.ts` or the thinking part's expand handler). For problem 2: make the expanded thinking container use a viewport-relative max-height (~75%) with internal scrolling by default — the `FixedScrolling` mode's scrollable element pattern already does this, so promoting that behavior to the default expand path (or making it the default `thinkingStyle`) would address both. A sticky header (as suggested by justschen) would further help navigation.
