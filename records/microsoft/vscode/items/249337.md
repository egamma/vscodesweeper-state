# Issue #249337 — drag and move file twice will cause file disappear

[https://github.com/microsoft/vscode/issues/249337](https://github.com/microsoft/vscode/issues/249337)

## Summary

Real race-condition bug still present on main: dragging a file to a folder twice on slow storage can cause the file to disappear because the overwrite-retry path doesn't verify the source still exists after the first move already relocated it.

## Change summary

Dragging a file from folder A to folder B twice (before the first move completes on slow storage) and confirming the overwrite dialog causes the file to vanish.

## Best solution

Guard the overwrite-retry path in `doHandleExplorerDropOnMove` (explorerViewer.ts ~line 2016) to verify the source file still exists before retrying with overwrite. If the source no longer exists, skip the retry or inform the user the move already completed. Also consider preventing a second drag-drop of the same file while a move is in progress.
