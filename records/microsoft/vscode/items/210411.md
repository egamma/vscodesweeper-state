# Issue #210411 — Portable version of VSCode uses absolute path for location entry in data\extensions\extensions.json

[https://github.com/microsoft/vscode/issues/210411](https://github.com/microsoft/vscode/issues/210411)

## Summary

The absolute path in extensions.json is stored for backward compatibility only — VS Code already reads from `relativeLocation` at runtime, so portable installs relocate correctly. The issue is a polish request to stop writing the redundant absolute path in portable mode, which a team member acknowledged as 'clean' but not functionally required.

## Change summary

In portable mode, extensions.json stores absolute paths in the `location` field alongside the functional `relativeLocation`, making the file look non-portable even though it works correctly when moved.

## Best solution

In `src/vs/platform/extensionManagement/common/extensionsProfileScannerService.ts`, the write path (line ~289-297) could omit or relativize the `location` field when running in portable mode, since `relativeLocation` is the authoritative field. This is a cosmetic/polish improvement — the current behavior is functionally correct for portability.
