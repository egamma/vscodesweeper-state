# Issue #314838 — Add auto-generated interactive dependency map for codebase architecture

[https://github.com/microsoft/vscode/issues/314838](https://github.com/microsoft/vscode/issues/314838)

## Summary

The request asks for an interactive dependency graph visualization integrated into CI. VS Code already ships a robust layering checker (`build/checker/layersChecker.ts`) and CI pipeline checks that block invalid imports. The interactive visualization/web-page aspect is tooling that belongs outside the core repo — it's a developer-tooling concern buildable with existing third-party tools against the public source.

## Change summary

Add an auto-generated interactive dependency map visualization published from CI, plus automated PR checks for invalid dependencies.

## Best solution

The CI-enforcement half already exists via `valid-layers-check` and the layers checker. The interactive visualization half (a web page generated by dependency-cruiser or madge) is a developer tooling concern that can be built as an external project or community contribution outside the VS Code core repository — it doesn't belong in the product itself.
