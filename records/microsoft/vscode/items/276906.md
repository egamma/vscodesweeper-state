[https://github.com/microsoft/vscode/issues/276906](https://github.com/microsoft/vscode/issues/276906)

## Summary

Team-authored plan item on the 'On Deck' milestone — must stay open for maintainer judgment. The request is to consolidate the `run_task` tool into an execute strategy on `run_in_terminal`, reducing tool surface for agents.

## Change summary

Remove the separate `run_task` chat tool and instead run tasks via an execute strategy on the `run_in_terminal` tool.

## Best solution

Implement the execute-strategy approach described in the comments: when the terminal tool detects a command line matching a configured task, run it as a task internally. This also removes the `create_task` tool (edit tool handles that). The work lives in `src/vs/workbench/contrib/chat/` (tool registrations) and `src/vs/workbench/contrib/terminal/` (terminal tool execute strategies).
