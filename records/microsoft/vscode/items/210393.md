# Issue #210393 — Wrong text decoration locations on vscode 1.88.1

[https://github.com/microsoft/vscode/issues/210393](https://github.com/microsoft/vscode/issues/210393)

## Summary

Still present on main: commit 5faa55a1735 added `position: absolute` to `.view-line > span`, which changed the containing block for absolutely-positioned text decorations from document-level to line-level. This breaks extensions relying on absolute positioning in decoration CSS. No fix has landed.

## Change summary

Text decorations using CSS `position: absolute` are now positioned relative to the line rather than the document, breaking extensions that relied on document-level absolute positioning.

## Best solution

Either provide a way for decoration pseudo-elements to escape the line-level containing block (e.g. a wrapper that doesn't establish a positioning context), or document that `position: absolute` in decorations is now line-scoped and extensions should adapt. The architectural change in `viewLines.css` was intentional for line-height flexibility, so a compatibility shim or API guidance is the likely path.
