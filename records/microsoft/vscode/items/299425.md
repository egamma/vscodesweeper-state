# Issue #299425 — Error: Uncaught Error: !!! NLS MISSING: 16444 !!! when starting VS Code 1.110.0 in serve-web mode via the CLI

[https://github.com/microsoft/vscode/issues/299425](https://github.com/microsoft/vscode/issues/299425)

## Summary

Confirmed serve-web NLS mismatch bug — still broken on main. A draft PR (#320945) exists with a fix but has not merged yet; keeping open until it lands.

## Change summary

VS Code serve-web mode crashes with 'NLS MISSING' errors for all non-English locales because the bundled nls.messages.js has server-web combined string indices that don't match the CDN locale translation indices.

## Best solution

Land the fix in draft PR https://github.com/microsoft/vscode/pull/320945 which splits NLS collectors so serve-web's browser-facing nls.messages.js uses CDN-compatible indices. The build/NLS infrastructure in `build/next/` is the likely area.
