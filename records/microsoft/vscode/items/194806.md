# Issue #194806 — [folding] Editor expands when a folding provider comes in with delay

[https://github.com/microsoft/vscode/issues/194806](https://github.com/microsoft/vscode/issues/194806)

## Summary

Still reproduces on main: when an LSP folding provider arrives after the indentation-based fallback, ranges with differing endLineNumber are unfolded during sanitizeAndMerge. The core condition at foldingRanges.ts:339 has not been changed to handle the delayed-provider scenario.

## Change summary

Folded regions expand (unfold) when a language server folding provider registers with delay and its ranges differ in end line from the initial indentation-based ranges.

## Best solution

Modify the merge logic in `sanitizeAndMerge` (src/vs/editor/contrib/folding/browser/foldingRanges.ts ~line 339) to preserve `isCollapsed` when the incoming provider range starts at the same line but differs in end line, particularly during the initial provider registration phase. Alternatively, implement @dbaeumer's suggestion (comment 10) to store the provider ID with persisted ranges and wait for the matching provider before falling back.
