# Issue #194806 — [folding] Editor expands when a folding provider comes in with delay

[https://github.com/microsoft/vscode/issues/194806](https://github.com/microsoft/vscode/issues/194806)

## Summary

Still reproduces on main: the sanitizeAndMerge logic at foldingRanges.ts:339 unfolds regions when end line numbers differ, which always happens when the indentation-based provider fills in before the LSP provider registers. No fix has landed.

## Change summary

Persisted folding regions are expanded when a delayed folding provider (e.g. LSP) registers and its ranges differ in end-line from the indentation-based fallback.

## Best solution

Modify the merge logic in `src/vs/editor/contrib/folding/browser/foldingRanges.ts` (line 339) to preserve collapsed state during the initial provider transition — either by relaxing the end-line equality check on first provider arrival, or by reapplying initial folding state when the first real provider registers and no user interaction has occurred (as aeschli proposed in comment 16).
