# Issue #185212 — Allow safestorage api to detect presence of libsecret or kwallet if the desktop environment is unknown

[https://github.com/microsoft/vscode/issues/185212](https://github.com/microsoft/vscode/issues/185212)

## Summary

Upstream Electron/Chromium issue with safestorage backend detection on Linux when the desktop environment is unknown. Both upstream Electron issues were closed as completed in Feb 2026, but the fix hasn't been confirmed as landed in VS Code's current Electron 42. The issue also carries the 'On Deck' milestone, which is a hard keep-open.

## Change summary

On Linux with an unknown desktop environment, Electron's safestorage API fails to detect available secret providers like libsecret or kwallet, forcing VS Code to fall back to in-memory storage and losing session persistence across restarts.

## Best solution

Verify whether the upstream Electron fix (issues electron/electron#39789 and electron/electron#47436, both closed as completed 2026-02-15) has landed in VS Code's current Electron 42 dependency. If so, confirm the behavior is fixed and close. If not, track the Electron version bump that will bring the fix. The open sibling issue https://github.com/microsoft/vscode/issues/187338 tracks a broader variant of the same problem and may be the better canonical home.
