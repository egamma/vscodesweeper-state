## Summary

Still valid: the GitHub extension's Git credential provider remains hardcoded to github.com on current main; GitHub Enterprise hosts are not served credentials despite the auth provider already supporting GH Enterprise authentication.

## Change summary

Extend the GitHub extension's Git credential provider to supply credentials for GitHub Enterprise hosts, not just github.com.

## Best solution

Update `extensions/github/src/credentialProvider.ts` to detect GitHub Enterprise hosts (via the `github-enterprise` auth provider and the `github-enterprise.uri` setting) and supply credentials for them, handling the optional `workflow` scope for older GHE versions. The auth infrastructure already exists in `extensions/github-authentication/`; the gap is only in the credential provider bridge.
