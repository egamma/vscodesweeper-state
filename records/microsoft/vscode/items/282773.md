# Issue #282773 — Search / Replace does not work if I use undo and try again

[https://github.com/microsoft/vscode/issues/282773](https://github.com/microsoft/vscode/issues/282773)

## Summary

Still reproducible on main: after Replace All, `textSearchHeading.replaceAll()` calls `this.clear()` which disposes all file match objects and their model listeners. When the user undoes, changes go undetected and the Replace All button stays grayed out. The PR that attempted a fix (#283061) was closed without merging.

## Change summary

After performing Replace All in the search view and then undoing, the Replace All button remains grayed out because the file match listeners were disposed by `clear()` during the replacement.

## Best solution

In `textSearchHeading.ts`, the `replaceAll()` method should not call `this.clear()` after replacement completes. Instead it should force a re-search / re-match so that if the user undoes, the model listeners are still alive to detect restored matches and re-enable the Replace All button. The closed PR #283061 had the right diagnosis — replace the `clear()` call with a `forceUpdateMatches()` or similar approach that preserves listeners.
