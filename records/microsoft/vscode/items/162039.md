# Issue #162039 — Merge Editor: A way to look through automatically resolved "conflicts"

[https://github.com/microsoft/vscode/issues/162039](https://github.com/microsoft/vscode/issues/162039)

## Summary

Valid feature request — the merge editor currently only navigates unhandled conflicts (filtered by `!isHandled`); no command exists to cycle through all modified base ranges including resolved ones. Still unimplemented on main.

## Change summary

Request navigation arrows in the merge editor that cycle through all conflicts (both resolved and unresolved), not just unhandled ones.

## Best solution

Add a pair of commands (e.g. `merge.goToNextConflict` / `merge.goToPreviousConflict`) that call `goToNextModifiedBaseRange` with a truthy predicate (no filter), giving users a way to review all merge regions for a final pass. The plumbing in the viewModel already supports an arbitrary predicate.
