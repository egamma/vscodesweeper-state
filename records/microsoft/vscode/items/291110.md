Team-authored issue â€” keeping open per policy. The welcome view's session-target picker does not filter out session types (like codex) that are incompatible with the welcome chat input, because the welcome view delegate does not supply an `isSessionTypeVisible` filter to scope the dropdown.

## Change summary

Selecting 'codex' in the welcome view's session-target picker breaks input and the send button, because codex requires an editor-based session but the welcome view uses an inline chat input.

## Best solution

The welcome view's `sessionTypePickerDelegate` should implement `isSessionTypeVisible` to hide session types that require an editor-based session (like codex). The filtering logic in `sessionTargetPickerActionItem.ts` already supports this via the delegate pattern â€” the welcome view just needs to provide the filter. Alternatively, as benibenj suggested, the command dispatched when a session type is selected should have enough context to open the right surface (editor vs inline) regardless of where the picker lives.
