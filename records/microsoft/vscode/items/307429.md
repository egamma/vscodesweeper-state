# Issue #307429 — certain regexp became really really slow in the newest version

[https://github.com/microsoft/vscode/issues/307429](https://github.com/microsoft/vscode/issues/307429)

## Summary

The regex `(?<="([^"”]|\n)*)'` contains a classic catastrophic-backtracking alternation (`[^"\u201d]|\n` where `\n` is already matched by the character class, creating exponential branching). No VS Code search code changed between 1.113 and 1.114, and both ship the same Electron 39.8.3 / V8 14.2. Without a concrete reproducible sample and a confirmed prior working version, more info is needed.

## Change summary

A user-supplied regex with a redundant alternation in a lookbehind causes the editor find widget to freeze on ~2000 characters of Chinese text.

## Best solution

Ask the reporter for the exact previous version that worked and a shareable reproduction text. If a genuine V8 regression is confirmed, file upstream against V8/Chromium. If the same regex always backtracked, close as a question and suggest rewriting the regex as `(?<="[^"”\n]*)'([^"'\u2018\u2019]*)'` to eliminate the alternation.
