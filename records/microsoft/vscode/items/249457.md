# Issue #249457 — "Clear all Results" is rather slow (20s?) and blocks the UI when there are a lot of tests

[https://github.com/microsoft/vscode/issues/249457](https://github.com/microsoft/vscode/issues/249457)

## Summary

Still reproduces on main: the 'Clear All Results' action fires a synchronous event whose listeners (especially treeProjection) iterate and refresh all items in O(n log n) on the UI thread, blocking it for large test suites.

## Change summary

Clearing all test results blocks the UI for ~20 seconds when there are many tests due to synchronous per-item state recalculation.

## Best solution

Batch or defer the state recalculation in treeProjection.ts (line ~133) when results are cleared — either reset all items to Unset in bulk without per-item fireChange(), or schedule the refresh across animation frames. The same pattern in listProjection.ts likely needs the same treatment.
