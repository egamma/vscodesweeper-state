# Issue #147450 — filename of numbers without extension is classified as JavaScript depending on content

[https://github.com/microsoft/vscode/issues/147450](https://github.com/microsoft/vscode/issues/147450)

## Summary

The reported behavior — language detection running on extension-less files and being influenced by viewing history — was explicitly confirmed as intentional by a team member. The user can disable it with `workbench.editor.languageDetection: false`. The separate Open Editors icon bug was split into #147733.

## Change summary

Extension-less files with numeric names get auto-detected as JavaScript based on content and viewing history, and don't revert to Plain Text when content changes back.

## Best solution

Close as designed: language detection on extension-less files is intentional (supports pipe-to-code workflows), history-based biasing is intentional, and plain text cannot be positively 'detected'. The setting `workbench.editor.languageDetection: false` disables this for users who don't want it.
