# Issue #286352 — Wrapped file edit pills don't look great

[https://github.com/microsoft/vscode/issues/286352](https://github.com/microsoft/vscode/issues/286352)

## Summary

Team-contributed polish issue that remains unfixed on main. The file edit pill CSS still allows text wrapping inside individual pills (`text-wrap: wrap` on `.icon-label`), producing the awkward multi-line pill layout shown in the screenshots.

## Change summary

When file edit pills wrap to multiple lines in a narrow chat panel, the visual result is cluttered — the pills should either truncate with ellipsis or the container should wrap pills onto new rows more cleanly.

## Best solution

Adjust the pill layout in `src/vs/workbench/contrib/chat/browser/widget/chatContentParts/media/chatCodeBlockPill.css` — either change `.icon-label` from `text-wrap: wrap` to `text-wrap: nowrap` with `text-overflow: ellipsis` / `overflow: hidden` to truncate long filenames, or rework the container's flex layout to wrap pills onto new rows while keeping each pill single-line, matching the preferred designs in the screenshots.
