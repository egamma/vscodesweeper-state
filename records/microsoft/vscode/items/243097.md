# Issue #243097 — UX: Quick chat resizing bug

[https://github.com/microsoft/vscode/issues/243097](https://github.com/microsoft/vscode/issues/243097)

## Summary

Team-authored bug, still reproducible on main. The quick chat sash resize logic enforces a 200px minimum height that exceeds the empty-state height, causing an abrupt jump when resizing begins.

## Change summary

Quick chat height abruptly jumps to a preset minimum (200px) when attempting to resize from the empty state.

## Best solution

In `src/vs/workbench/contrib/chat/browser/widgetHosts/chatQuick.ts`, make the sash minimum height responsive to the actual content height so the widget doesn't jump when the empty state is shorter than `DEFAULT_MIN_HEIGHT`. Possibly compute a dynamic min based on the widget's current content rather than a fixed 200px constant.
