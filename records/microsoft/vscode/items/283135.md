# Issue #283135 — Poor experience if you accidentally close the sign in window

[https://github.com/microsoft/vscode/issues/283135](https://github.com/microsoft/vscode/issues/283135)

## Summary

Team-authored issue (MEMBER) — must stay open for maintainer judgment. The bug describes a real UX gap in the GitHub authentication sign-in flow when the browser window is closed or lost.

## Change summary

Clicking 'Sign in to Github.com' again after closing the sign-in browser window does nothing, with no feedback or way to restart the flow.

## Best solution

The authentication flow should either re-focus the existing browser window or restart the sign-in flow when the user clicks the sign-in button again. A progress spinner in the chat UI and proper state reset on cancel would complete the fix. Likely lives in the GitHub authentication provider and chat sign-in contribution.
