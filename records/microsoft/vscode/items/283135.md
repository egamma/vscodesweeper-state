# Issue #283135 — Poor experience if you accidentally close the sign in window

[https://github.com/microsoft/vscode/issues/283135](https://github.com/microsoft/vscode/issues/283135)

## Summary

Team-authored issue (MEMBER) describing a real UX gap in the GitHub sign-in flow — no fix found on main. Keeping open per policy.

## Change summary

Clicking 'Sign in to GitHub.com' again after accidentally closing the sign-in browser window does nothing, with no feedback to the user.

## Best solution

Make the sign-in button either restart the OAuth flow or refocus the existing browser window when clicked while a session is already pending; add a progress spinner in the chat UI so users know a flow is active.
