## Summary

Valid accessibility feature request with team support (meganrogge endorsed using an icon). Not yet implemented on main. Assigned to benibenj who owns tabs; keeping open for implementation.

## Change summary

Add a non-color indicator (e.g. warning icon) on tabs with problems so colorblind users can distinguish them from modified-file tabs.

## Best solution

Add an optional icon decoration (warning/error codicon) next to the file name in tabs that have diagnostics, gated behind a setting or always-on in accessibility mode, implemented in `src/vs/workbench/browser/parts/editor/multiEditorTabsControl.ts`.
