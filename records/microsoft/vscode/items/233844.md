## Change summary

Add a non-color indicator (e.g. warning icon) to editor tabs for files with problems, so colorblind users can distinguish them from modified-file tabs.

## Best solution

Add an optional icon decoration (e.g. warning codicon) to editor tabs when files have diagnostics, likely in `src/vs/workbench/browser/parts/editor/multiEditorTabsControl.ts` and related tab rendering code. A setting could control the behavior. This aligns with existing accessibility investment.
