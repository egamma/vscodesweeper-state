# Issue #52116 — [Feature Request] Extension Permissions, Security Sandboxing & Update Management Proposal

[https://github.com/microsoft/vscode/issues/52116](https://github.com/microsoft/vscode/issues/52116)

## Summary

A highly-upvoted (332 👍) long-standing feature request for extension permissions and sandboxing. No implementation exists on current main beyond Workspace Trust (which addresses a narrow subset). Well above the 20-upvote Backlog threshold — should remain open.

## Change summary

Request a permissions system for VS Code extensions allowing users to approve/deny capabilities (filesystem, network, process execution) with sandboxing enforcement, plus per-extension update controls.

## Best solution

This remains a valid architectural aspiration. The current agent-security workstream (sandboxing for local agents) could eventually inform extension sandboxing patterns. The issue serves as the canonical tracker for this broad capability request. Keep open as accepted Backlog; implementation would span extension-host isolation, a declared-permissions manifest schema, and UI for permission management.
