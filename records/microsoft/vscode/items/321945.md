# Issue #321945 — [Regression] Unknown tokenizer: undefined

[https://github.com/microsoft/vscode/issues/321945](https://github.com/microsoft/vscode/issues/321945)

## Summary

Real regression in the built-in Copilot extension — object spread drops prototype getters on getter-based endpoints, causing `acquireTokenizer` to throw. An open bot-generated PR (#321947) has the fix but hasn't merged yet; keeping open for a human reviewer to land it.

## Change summary

Copilot's `_provideLanguageModelResponse` uses object spread to override `modelMaxPromptTokens`, which strips prototype getters like `tokenizer` from getter-based endpoints, causing `Unknown tokenizer: undefined` at runtime.

## Best solution

Review and land https://github.com/microsoft/vscode/pull/321947, which replaces the object spread with `_endpoint.cloneWithTokenOverride(tokenLimit)` in `extensions/copilot/src/extension/conversation/vscode-node/languageModelAccess.ts`. The fix is small, well-scoped, and uses an existing idiomatic API.
