# Issue #280716 — Fetch 'Allow and review' UI is confusing

[https://github.com/microsoft/vscode/issues/280716](https://github.com/microsoft/vscode/issues/280716)

## Summary

The confusing 'Allow and Review Once' button label and related UX issues in the fetch tool confirmation dialog remain on current main — the exact string is still rendered in chatToolConfirmationSubPart.ts. Valid polish item that needs a design pass.

## Change summary

The fetch tool's pre-execution confirmation dialog uses unclear language ('Allow and Review Once', 'Skip') and shows the injection warning at the wrong stage.

## Best solution

Redesign the fetch confirmation wording in `src/vs/workbench/contrib/chat/browser/widget/chatContentParts/toolInvocationParts/chatToolConfirmationSubPart.ts` and `src/vs/workbench/contrib/chat/electron-browser/builtInTools/fetchPageTool.ts` to clarify what is being allowed, what 'review' means (and when), what 'skip' does, and move the injection warning to the result-confirmation stage where the risk actually applies.
