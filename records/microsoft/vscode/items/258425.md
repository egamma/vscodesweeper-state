# Issue #258425 — A better way to manage prompts/instructions/agents with the same name.

[https://github.com/microsoft/vscode/issues/258425](https://github.com/microsoft/vscode/issues/258425)

## Summary

Valid feature request for better handling of same-name prompts/instructions across user and workspace levels. Some dedup logic exists for built-in items but doesn't address the user-vs-workspace override/precedence UX the reporter describes.

## Change summary

User wants a precedence or deduplication mechanism when prompts/instructions/agents share the same name at user-level and workspace-level, to avoid SSOT violations and UI clutter.

## Best solution

Implement a precedence model (workspace overrides user, or vice-versa) for same-name prompts in the AI Customization layer, suppressing duplicates in the picker UI. The logic would likely live in `src/vs/workbench/contrib/chat/browser/aiCustomization/aiCustomizationItemSource.ts` alongside the existing built-in dedup.
