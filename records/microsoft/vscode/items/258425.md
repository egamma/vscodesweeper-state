# Issue #258425 — A better way to manage prompts/instructions/agents with the same name.

[https://github.com/microsoft/vscode/issues/258425](https://github.com/microsoft/vscode/issues/258425)

## Summary

Valid feature request. Current main has deduplication logic for skills (workspace > personal precedence) but not for prompts/instructions/chat modes. The user's ask — unified management of same-named prompts across scopes — remains unaddressed.

## Change summary

When a prompt/instruction/chat mode file exists at both user and workspace levels with the same name, both appear in the UI as duplicates with no precedence or merge behavior.

## Best solution

Extend the existing skill dedup/precedence logic (workspace > personal > plugin) to prompts and instructions in `promptsServiceImpl.ts`, and surface scope information in the UI so users can see which level a prompt comes from without duplication in the picker.
