# Issue #108256 — resources#relativePath does not honour path casing strategy

[https://github.com/microsoft/vscode/issues/108256](https://github.com/microsoft/vscode/issues/108256)

## Summary

Still reproduces on current main: the `file` scheme special case in `relativePath` bypasses `_ignorePathCasing`, exactly as reported. The prepared fix (commit 9997c23708a / PR #108796) was never merged.

## Change summary

`resources#relativePath` short-circuits with `paths.relative` for `file:` URIs, ignoring the path-casing strategy that `IExtUri` is supposed to honour.

## Best solution

Remove the `file` scheme special case in `ExtUri.relativePath` (src/vs/base/common/resources.ts ~line 238) and rely on `_ignorePathCasing` for all schemes, adding drive-letter mismatch detection. Audit callers to ensure they pass canonical URIs or use `IUriIdentityService.extUri`.
