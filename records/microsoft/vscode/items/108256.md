# Issue #108256 — resources#relativePath does not honour path casing strategy

[https://github.com/microsoft/vscode/issues/108256](https://github.com/microsoft/vscode/issues/108256)

## Summary

Still unfixed on main: `relativePath` in `src/vs/base/common/resources.ts` (lines 239–241) still has the `file` scheme special case that bypasses `_ignorePathCasing`, exactly as reported. PR #108796 was closed without merging due to the complexity of ensuring all callers use canonical URIs.

## Change summary

`resources#relativePath` uses `paths.relative` for `file:` URIs, ignoring the `_ignorePathCasing` strategy, so case-insensitive file systems get incorrect relative-path results.

## Best solution

Remove the `file` scheme special case in `ExtUri.relativePath` (lines 239–241 of `src/vs/base/common/resources.ts`) so it falls through to the `_ignorePathCasing`-aware code path. The prerequisite work identified in the 2020 discussion — ensuring all callers pass canonical URIs or use `IUriIdentityService.extUri` — may still apply; audit callers before merging.
