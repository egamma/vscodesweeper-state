# Issue #121203 — Folding: sometimes can not be folded and touch registered as double click

[https://github.com/microsoft/vscode/issues/121203](https://github.com/microsoft/vscode/issues/121203)

## Summary

Still reproduces on main: PointerEventHandler listens to both `pointerup` (line 51) and inherits `mouseup` from MouseHandler (line 114), so touch triggers `_onMouseUp` twice — the folding code sees down+up+up which can look like a double-click fold/unfold cycle.

## Change summary

On touch devices, tapping a folding gutter icon sometimes fires the fold/unfold action twice (once via pointerup, once via mouseup), causing the fold to immediately re-expand.

## Best solution

Guard `_onMouseUp` in `PointerEventHandler` the same way `_onMouseDown` is guarded — skip when `pointerType === 'touch'` — or deduplicate the up events in `mouseHandler.ts` so only one path fires per gesture. The fix is localized to `src/vs/editor/browser/controller/pointerHandler.ts`.
