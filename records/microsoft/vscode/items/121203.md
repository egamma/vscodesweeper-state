# Issue #121203 — Folding: sometimes can not be folded and touch registered as double click

[https://github.com/microsoft/vscode/issues/121203](https://github.com/microsoft/vscode/issues/121203)

## Summary

Team-authored issue; still reproduces on main. The PointerHandler registers `onPointerUp` while the parent MouseHandler also registers `onMouseUp` — on touch devices both fire for the same gesture, causing a double-invocation of `_onMouseUp` that the folding code interprets as expand+collapse.

## Change summary

On iPad/touch devices, tapping a folded region's decoration fires both `pointerup` and `mouseup`, causing a double mouse-up that expands then immediately re-folds the region.

## Best solution

In the PointerHandler (src/vs/editor/browser/controller/pointerHandler.ts), suppress the parent MouseHandler's `onMouseUp` listener when the pointer type is `touch` (or `pen`), so only the `pointerup` event drives the interaction — or deduplicate at the EditorMouseEventFactory level. The fix is localized to the pointer/mouse handler coordination in the editor controller layer.
