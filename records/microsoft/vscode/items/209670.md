# Issue #209670 — Terminal is zoomed in and out to be very large or small with the mouse wheel while holding Ctrl

[https://github.com/microsoft/vscode/issues/209670](https://github.com/microsoft/vscode/issues/209670)

## Summary

Still reproduces on current Insiders after the bounds-checking fix (PR #253010). The core defect — first Ctrl+scroll jump being dramatically oversized — is distinct from the negative-font-size issue that PR addressed.

## Change summary

First Ctrl+mouse wheel zoom in the terminal produces a large font size jump instead of the expected ±1 increment.

## Best solution

Investigate the MouseWheelClassifier's first-event behavior in `src/vs/workbench/contrib/terminalContrib/zoom/browser/terminal.zoom.contribution.ts`. On the very first scroll the classifier may not yet know the device type, so it falls into the gesture path where `gestureAccumulatedDelta` gets a large raw `deltaY` value from the browser event, causing the big initial jump. The fix likely needs to handle the first-event classification edge case or cap the initial gesture delta.

There, `gestureStartFontSize` is initialized at field level but `gestureAccumulatedDelta` starts at 0, then `gestureAccumulatedDelta += browserEvent.deltaY` applies the raw browser deltaY (e.g. 120) TWICE — once at line 97 and again at line 105 inside the zoom-modifiers block — so the first gesture calculation uses a large accumulated delta producing a big font jump. The fix: remove the duplicate `gestureAccumulatedDelta += browserEvent.deltaY` on line 105 (it's already accumulated on line 97 before the if-block), OR cap the initial delta calculation for the first event.
