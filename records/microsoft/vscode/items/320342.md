# Issue #320342 — Allow users to hide extension-contributed `viewsWelcome` content (welcome view)

[https://github.com/microsoft/vscode/issues/320342](https://github.com/microsoft/vscode/issues/320342)

## Summary

Genuine feature gap — no user-facing mechanism exists to hide extension-contributed viewsWelcome content on built-in views. Currently labeled as `bug` but this is a feature request; the current behavior (extensions control visibility via `when` clauses) is by design, though the lack of a user override is a real UX gap.

## Change summary

Allow users to hide or suppress extension-contributed viewsWelcome content in built-in views without disabling the extension.

## Best solution

Reclassify as `feature-request`. The implementation would likely live in `src/vs/workbench/contrib/welcomeViews/common/viewsWelcomeContribution.ts` and `src/vs/workbench/common/views.ts`, adding a user-persisted suppression list (by extension ID or content descriptor) that filters results in `getViewWelcomeContent`. A context-menu action on welcome items (option 1 in the report) feels most discoverable and consistent with VS Code's existing hide-view pattern.
