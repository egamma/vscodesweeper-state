# Issue #257923 — Managed hovers leak `hoverwidget` disposables

[https://github.com/microsoft/vscode/issues/257923](https://github.com/microsoft/vscode/issues/257923)

## Summary

Team-authored debt issue identifying a real disposable leak in the managed hover widget path. The team is aware and plans to address it either via the new hover API migration or a targeted fix.

## Change summary

ManagedHoverContent's hover widget is cleared but never disposed when the mouse leaves the originating element, leaking disposables.

## Best solution

Add proper disposal of the hover widget in the new hover API (which is replacing the deprecated managed hover path), or fix the leak in the legacy code at `src/vs/editor/browser/services/hoverService/hoverService.ts` around line 397–400 before the old API is fully removed.
