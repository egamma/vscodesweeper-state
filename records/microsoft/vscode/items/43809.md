# Issue #43809 — Git - Support prompting for GPG password

[https://github.com/microsoft/vscode/issues/43809](https://github.com/microsoft/vscode/issues/43809)

## Summary

Valid, well-supported feature request (101 upvotes) for native GPG passphrase prompting. VS Code's askpass mechanism handles SSH passphrases but not GPG, which uses the separate pinentry protocol. The gap remains on current main — no GPG pinentry integration exists in the git extension.

## Change summary

VS Code should natively prompt for GPG passphrases when commit signing is enabled, instead of requiring users to configure external pinentry programs.

## Best solution

Implement a VS Code-native pinentry provider (or GPG_TTY/loopback-mode bridge) in the git extension's askpass infrastructure so that GPG passphrase prompts surface as VS Code input boxes, similar to how SSH passphrases are already handled. This would eliminate the need for platform-specific pinentry configuration and solve the problem in containerized/remote/WSL environments. The `help wanted` label is appropriate — this is a good community contribution target.
