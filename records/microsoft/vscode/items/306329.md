# Issue #306329 — Hope for “Go to File” to Check Path Existence When Inputting Full Relative Path

[https://github.com/microsoft/vscode/issues/306329](https://github.com/microsoft/vscode/issues/306329)

## Summary

The quick open already checks path existence for both absolute and relative paths via `fileService.stat`. However, for relative paths the result is gated behind `Promise.all` with the general file search, so on slow network shares (Samba) the user sees no results until the full search completes. This is a valid polish/improvement request to surface relative-path stat results independently of the general search.

## Change summary

When typing a full relative path in Quick Open on a slow filesystem, show the file immediately if the path exists rather than waiting for the full file search to complete.

## Best solution

In `src/vs/workbench/contrib/search/browser/anythingQuickAccess.ts`, restructure `doFileSearch` so that relative-path file results from `getRelativePathFileResults` can surface independently and immediately (similar to how absolute paths short-circuit at line 578–593), without waiting for the general file search to finish. This would help users on slow network shares see exact-path matches instantly.
