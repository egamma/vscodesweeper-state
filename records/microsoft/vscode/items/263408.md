# Issue #263408 — Git - Differentiate between submodules and normal repos on virtual drives (Winows)

[https://github.com/microsoft/vscode/issues/263408](https://github.com/microsoft/vscode/issues/263408)

## Summary

Kept open — the issue carries the 'On Deck' milestone, which is a scheduling decision that defers to maintainer judgment. The reported bug is plausible: on Windows virtual/mapped drives, `git rev-parse --show-superproject-working-tree` likely returns the real path rather than the mapped drive letter, so the parent-child hierarchy breaks because the paths don't match.

## Change summary

When a Git repository with submodules is opened from a Windows virtual/mapped drive, the Source Control Repository view shows submodules as flat siblings instead of nested children of the parent repository.

## Best solution

The path comparison in `extensions/git/src/repository.ts` (~line 953–961) that matches `superProjectPath` to parent repositories likely fails because the superproject path resolves to the real filesystem path while the repository root uses the mapped drive letter. A path normalization (resolving both to real paths via `repositoryRootRealPath`) before comparison should fix this. The `extensions/git/src/git.ts` `getRepositoryDotGit()` method may also need to normalize `superProjectPath` against `repositoryRootRealPath`.
