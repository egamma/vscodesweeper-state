# Issue #317481 — Feature Request: [Chronicle] Surface chat session titles and deep-links in standup/search output

[https://github.com/microsoft/vscode/issues/317481](https://github.com/microsoft/vscode/issues/317481)

## Summary

Valid feature request to improve Chronicle output with session titles and clickable deep-links. Chronicle already stores session summaries and session IDs, but the SKILL.md output template only renders raw IDs. No duplicate found. Keeping open for area routing — the feature lives in the Copilot extension's chronicle skill.

## Change summary

Chronicle outputs should display human-readable session titles (from the `summary` field) and wrap session IDs in clickable deep-links for direct navigation.

## Best solution

Update the Chronicle SKILL.md output templates (standup, search, tips) to render the session `summary` alongside the ID, and — once a `vscode://` URI scheme for opening chat sessions exists — wrap the ID in a clickable link. The title portion is straightforward (the data is already in the `sessions.summary` column); the deep-link portion depends on whether a chat-session URI handler exists in `src/vs/workbench/contrib/chat/browser/pluginUrlHandler.ts` or similar. Both changes are localized to `extensions/copilot/assets/prompts/skills/chronicle/SKILL.md` and potentially the session-store SQL tool output formatting.
