# Issue #298673 — Feature: Anchor file names or folder names directly to chat while typing automatically

[https://github.com/microsoft/vscode/issues/298673](https://github.com/microsoft/vscode/issues/298673)

## Summary

Valid feature request to insert file/selection references inline at the chat input cursor position (Cursor-style), rather than as separate context attachments. No duplicate found; the related issue #298568 covers inline text insertion but not file-reference anchoring. Keeping open for community traction.

## Change summary

When using 'Add File to Chat' from the explorer or editor context menu, insert the file reference inline at the current cursor position in the chat input, rather than adding it as a separate attachment above the input.

## Best solution

Enhance the chat input widget so that 'Add File to Chat' (and similar context actions) can optionally insert a `#file:` dynamic variable reference at the current cursor position in the chat input editor, in addition to (or instead of) adding a context attachment chip. The implementation would touch `src/vs/workbench/contrib/chat/browser/actions/chatContextActions.ts` and the chat input editor's dynamic variable handling in `src/vs/workbench/contrib/chat/browser/attachments/chatDynamicVariables.ts`. Related but distinct from https://github.com/microsoft/vscode/issues/298568 (inline text insertion).
