# Issue #293758 — Shell integration breaks grml-zsh-config by setting ZDOTDIR

[https://github.com/microsoft/vscode/issues/293758](https://github.com/microsoft/vscode/issues/293758)

## Summary

Valid bug — VS Code's ZDOTDIR injection sets ZDOTDIR to a temp directory before /etc/zsh/zshrc runs, so grml-zsh-config's zrclocal() looks for .zshrc.local in the wrong path. The script on current main still uses the same injection approach; no fix has landed.

## Change summary

VS Code's zsh shell integration sets ZDOTDIR to a temp directory during shell startup, causing grml-zsh-config's zrclocal() to miss ~/.zshrc.local because it evaluates ${ZDOTDIR:-${HOME}}/.zshrc.local against the temp path.

## Best solution

The ZDOTDIR injection in shellIntegration-rc.zsh needs to ensure that system scripts (/etc/zsh/zshrc) see the original ZDOTDIR. This is architecturally challenging since ZDOTDIR must be set before zsh starts to redirect .zshrc loading. A possible approach: the injected .zshenv (which runs before /etc/zsh/zshrc) could unset/restore ZDOTDIR after doing its work, or use a .zshrc-only injection mechanism that doesn't require ZDOTDIR manipulation during system script execution. Related to https://github.com/microsoft/vscode/issues/231313 (HISTFILE) which shares root cause.
