# Issue #16082 — [folding] retain folded state on copy/paste

[https://github.com/microsoft/vscode/issues/16082](https://github.com/microsoft/vscode/issues/16082)

## Summary

Legitimate feature request to retain folding state on copy/paste. Not implemented on current main; has steady community interest over many years. Parked in Backlog — keep open.

## Change summary

When copying folded code and pasting it elsewhere, the pasted code unfolds; users want it to stay folded.

## Best solution

Implement folding-state preservation in the editor paste handler (likely in `src/vs/editor/contrib/folding/`). The paste logic would need to capture fold ranges from the source selection and re-apply them at the target offset after insertion. This is a non-trivial editor feature that needs maintainer design input on how fold metadata travels through the clipboard.
