# Issue #186517 — Git - Performance Improvement: Revert Selected Ranges Should Not Replace The Entire Document

[https://github.com/microsoft/vscode/issues/186517](https://github.com/microsoft/vscode/issues/186517)

## Summary

Team-authored debt issue (hard keep-open). The diff editor now has "Revert Block" / "Revert Selection" actions (per lszomoru's comment), but the git extension's `applyLineChanges` still reconstructs the entire document as a string and replaces it wholesale. The remaining work — surfacing actions in the normal editor and optimizing the edit to touch only changed lines — is still valid.

## Change summary

The git extension's revert-selected-ranges command replaces the entire document instead of applying targeted edits, causing unnecessary retokenization and bracket-pair-AST invalidation.

## Best solution

Refactor `applyLineChanges` in `extensions/git/src/staging.ts` to build a `WorkspaceEdit` with per-range replacements instead of a single full-document replace, and surface the diff editor's Revert Block/Selection actions in the normal editor as lszomoru suggested. The remaining next-steps outlined in the latest comment are a reasonable roadmap.
