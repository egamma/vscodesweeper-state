# Issue #251725 — MCP `resources/read` doesn't support multiple Resources in result

[https://github.com/microsoft/vscode/issues/251725](https://github.com/microsoft/vscode/issues/251725)

## Summary

Fixed on main — the directory support commit (6e14077b6de) added navigation into MCP resources that return multiple contents, resolving the 'file is a directory' error when browsing such resources.

## Change summary

Selecting an MCP resource whose `resources/read` returns multiple child resources showed 'The file is not displayed in the text editor because it is a directory' instead of displaying the children.

## Best solution

Keep the shipped fix in commit 6e14077b6de (1.107.0); the Browse Resources command now navigates into directory-like resources and displays their children as picks.
