# Issue #284302 — Allow MDM to prevent modifying list of allow/denied commands for chat auto-approval

[https://github.com/microsoft/vscode/issues/284302](https://github.com/microsoft/vscode/issues/284302)

## Summary

The request for MDM policy control over the terminal auto-approve command list remains unimplemented on main. The related PRs (#285687 and #285732) were both closed without merging. The `chat.tools.terminal.autoApprove` setting currently lacks both `restricted` and `policy` properties. Issue carries milestone 'On Deck', enforcing keep-open.

## Change summary

Allow MDM administrators to lock down the `chat.tools.terminal.autoApprove` setting so users cannot override the allowed/denied command list.

## Best solution

Add `restricted: true` and a `policy` block to the `chat.tools.terminal.autoApprove` setting registration in `src/vs/workbench/contrib/terminalContrib/chatAgentTools/common/terminalChatAgentToolsConfiguration.ts`, matching the pattern already used for `EnableAutoApprove`. Also add the corresponding entry to `build/lib/policies/policyData.jsonc` and update platform-specific policy fixtures. Consider whether `ignoreDefaultAutoApproveRules` also needs a policy (as discussed in the issue comments).
