# Issue #241391 — Sync Settings fails with Microsoft login and Run as different user - Error Code: -2147023584

[https://github.com/microsoft/vscode/issues/241391](https://github.com/microsoft/vscode/issues/241391)

## Summary

Confirmed upstream issue in MSAL/WAM (Windows authentication broker) when VS Code is launched via 'Run as different user'. Team member TylerLeonhardt has triaged this correctly — VS Code depends on MSAL fixing the broker behavior; the `classic` setting is the interim workaround.

## Change summary

Settings Sync fails with MSAL Error Code -2147023584 when VS Code is run as a different Windows user than the logged-in session.

## Best solution

Keep tracking until Microsoft Identity/MSAL team addresses the WAM broker bug with cross-user scenarios (also tracked in https://github.com/microsoft/winget-cli/issues/6186). The `"microsoft-authentication.implementation": "classic"` workaround remains available until then.
