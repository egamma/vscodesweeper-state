# Issue #63397 — Git - Staging when ignoring whitespace, doesn't stage changed whitespace

[https://github.com/microsoft/vscode/issues/63397](https://github.com/microsoft/vscode/issues/63397)

## Summary

Still reproduces on main: the diff information piped to the git extension for staging comes from the diff editor's computed result, which excludes whitespace-only changes when `ignoreTrimWhitespace` is true. This means 'Stage selected ranges' silently skips whitespace changes, producing incomplete commits. 16 upvotes and a recent confirmation (Feb 2026) show this remains a real pain point.

## Change summary

When `diffEditor.ignoreTrimWhitespace` is enabled (the default), 'Stage selected ranges' does not stage whitespace-only changes because they are excluded from the diff computation results used by the staging logic.

## Best solution

The staging path in `mainThreadEditors.ts` (around line 185) should provide diff information computed without `ignoreTrimWhitespace` to the git extension — or the git extension should request a separate, unfiltered diff for staging purposes. The display setting should only affect rendering, not what gets staged. This likely requires changes in `src/vs/workbench/api/browser/mainThreadEditors.ts` (where diff results are gathered for the API) and possibly `extensions/git/src/staging.ts`.
