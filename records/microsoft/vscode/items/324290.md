# Issue #324290 — Copying multiple empty selections adds blank lines when pasted into a single cursor

[https://github.com/microsoft/vscode/issues/324290](https://github.com/microsoft/vscode/issues/324290)

## Summary

Confirmed regression from commit 20e65fd2: empty-selection multicursor entries now include a trailing newline, but the join in clipboardUtils adds another, producing blank lines on single-cursor paste. Fix PR #324294 is open but not yet merged.

## Change summary

Copying multiple empty selections and pasting into a single cursor produces spurious blank lines between entries (regression since 1.108.0).

## Best solution

Merge the fix from PR https://github.com/microsoft/vscode/pull/324294 (or equivalent): either stop appending `newLineCharacter` to each multicursorText entry in `getPlainTextToCopy`, or strip trailing newlines before the `join` in `clipboardUtils.ts`. The root is in `src/vs/editor/common/viewModel/viewModelImpl.ts` line 1033.
