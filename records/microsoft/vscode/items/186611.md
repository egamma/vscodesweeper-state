# Issue #186611 — Grid: distribute split views evenly even when one view requires a larger width

[https://github.com/microsoft/vscode/issues/186611](https://github.com/microsoft/vscode/issues/186611)

## Summary

Still reproduces on main: `distributeViewSizes()` in splitview.ts distributes equal sizes clamped to each view's min/max, but doesn't redistribute the leftover space when one view's minimum prevents it from shrinking to the equal share — the deficit is effectively stolen from the last view.

## Change summary

When splitting editor groups with a view that has a larger minimum width, the distribute logic doesn't redistribute surplus evenly, leaving the last group undersized.

## Best solution

Fix `distributeViewSizes()` in `src/vs/base/browser/ui/splitview/splitview.ts` to iteratively redistribute surplus space when clamping prevents equal distribution — similar to a multi-pass proportional allocation that accounts for minimum-size constraints.
