# Issue #41909 — Provide API to access and change editor tab labels

[https://github.com/microsoft/vscode/issues/41909](https://github.com/microsoft/vscode/issues/41909)

## Summary

The core use case (disambiguating index.js tabs) is now partially addressed by the `workbench.editor.customLabels.patterns` setting shipped in 2024, but the original request — a programmatic extension API to dynamically compute and set editor tab labels — remains unimplemented. With 389 upvotes and real extension-author demand (Dart, Joplin, partial-diff, SvelteKit), this stays open.

## Change summary

Provide an extension API that lets extensions programmatically read and change editor tab labels, beyond what the static glob-based `workbench.editor.customLabels.patterns` setting offers.

## Best solution

Design a `vscode.Tab` write API (or a label-provider registration point) so extensions can dynamically compute tab labels at runtime — e.g. showing package names for Dart imports, or context-aware renaming beyond what static glob patterns can express. The `Tab.label` property in `vscode.d.ts` is currently `readonly`; the API work would lift that restriction or add a provider. The existing `CustomEditorLabelService` in `src/vs/workbench/services/editor/common/customEditorLabelService.ts` is the natural integration point.
