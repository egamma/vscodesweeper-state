# Issue #41909 — Provide API to access and change editor tab labels

[https://github.com/microsoft/vscode/issues/41909](https://github.com/microsoft/vscode/issues/41909)

## Summary

The core ask — an extension API to programmatically set editor tab labels — remains unimplemented. The shipped `workbench.editor.customLabels.patterns` setting addresses the user-facing index.js problem via static patterns but does not expose an API for extensions. The 'On Deck' milestone signals active scheduling intent; keeping open.

## Change summary

Provide an extension API that allows extensions to read and change editor tab labels programmatically, enabling use cases like renaming index.js tabs to their parent folder name.

## Best solution

Design and ship an extension API (e.g. a `ResourceLabelFormatter`-like registration or a writable `Tab.label`) that lets extensions customize editor tab labels dynamically. The existing `workbench.editor.customLabels.patterns` setting handles static glob-based renaming, but extensions need a programmatic hook for context-aware labels (e.g. package names for Dart, framework-specific patterns). The 'On Deck' milestone suggests this is queued for implementation.
