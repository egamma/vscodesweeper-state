# Issue #150353 — SCM - provide state information for a repository

[https://github.com/microsoft/vscode/issues/150353](https://github.com/microsoft/vscode/issues/150353)

## Summary

Team-authored feature request — must keep open. The git extension already tracks merge/rebase/cherry-pick state internally and shows partial indicators (a conflicts icon and '(Rebasing)' text in the status bar branch item), but there is no prominent SCM view indication equivalent to SmartGit's 'Merging' state banner with an abort action. A meaningful UX gap remains.

## Change summary

Surface repository operation state (merging, rebasing, cherry-picking) prominently in the SCM view, with easy access to abort actions.

## Best solution

Add a visible state banner or badge in the SCM view (source control panel) when the repository is in a merging, rebasing, or cherry-picking state, similar to SmartGit's approach. The git extension already tracks `mergeInProgress`, `cherryPickInProgress`, and `rebaseCommit` in `extensions/git/src/repository.ts` and has abort commands registered — the missing piece is a dedicated SCM view UI element that surfaces this state and links to the abort/continue actions. This could be done via the SCM input box area, a view badge, or a dedicated SCM contribution widget.
