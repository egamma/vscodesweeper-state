# Issue #137737 — Add a command to run/debug tests with the current filter from anywhere

[https://github.com/microsoft/vscode/issues/137737](https://github.com/microsoft/vscode/issues/137737)

## Summary

Still valid: the `testing.runSelected` and `testing.debugSelected` commands remain `ViewAction<TestingExplorerView>` on current main, requiring the Test Explorer to be visible. The requested ability to run filtered tests from anywhere is not yet implemented.

## Change summary

Allow running/debugging tests with the current Test Explorer filter applied, even when the Test Explorer view is not focused.

## Best solution

Decouple the filtered-run logic from the view: store the active filter state in a service (or accept a filter parameter on the command), so `testing.runSelected`/`testing.debugSelected` can execute without `TestingExplorerView` being visible. The change would be in `src/vs/workbench/contrib/testing/browser/testExplorerActions.ts` (the `ExecuteSelectedAction` class).
