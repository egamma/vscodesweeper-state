# Issue #150113 — Duplicate output in devtools when extension writes to process.stdout

[https://github.com/microsoft/vscode/issues/150113](https://github.com/microsoft/vscode/issues/150113)

## Summary

The duplicate output in devtools is an intentional consequence of the stdout-to-console forwarding mechanism that makes extension stdout visible in the debug console. A maintainer already noted devtools is not the primary extension debugging surface.

## Change summary

Writing to process.stdout from an extension causes duplicated output in the VS Code developer tools console.

## Best solution

Keep the current behavior — the `_wrapStream` in `extHostConsoleForwarder.ts` intentionally forwards stdout writes to `console.log` so they appear in the debug console/CLI. The duplicate in devtools is a side effect of that design. The debug console is the supported debugging surface for extensions.
