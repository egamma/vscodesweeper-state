# Issue #150113 — Duplicate output in devtools when extension writes to process.stdout

[https://github.com/microsoft/vscode/issues/150113](https://github.com/microsoft/vscode/issues/150113)

## Summary

This looks like an intentional side effect of extension-host console forwarding rather than an actionable product bug: stdout is deliberately written to the original stream and also forwarded through console output so it reaches the debugger/CLI.

## Change summary

An extension writing to process.stdout in development mode produces duplicate lines in Chromium DevTools.

## Best solution

Keep the current extension-host forwarding behavior and direct extension authors to the VS Code Debug Console as the supported debugging surface; changing this would risk dropping stdout visibility in debugger/CLI paths.

## Proposed action

- **Triage action:** propose-close
- **Item type:** bug
- **Proposed label:** *as-designed
- **Area labels:** extension-host
- **Confidence:** 0.82
- **Proposed comment:**

  > Thanks for the report. The current extension-host implementation intentionally forwards stdout/stderr by both preserving the original stream write and mirroring complete lines through console output so they are available to the debugger/CLI. Since the VS Code Debug Console shows the output normally and DevTools is not the primary extension debugging surface, I think this should be treated as as-designed.

## Evidence

- **current implementation:** src/vs/workbench/api/node/extHostConsoleForwarder.ts wraps process.stdout/process.stderr and, for complete lines, calls console[severity](...) before also calling the original stream.write(...). The file comment says this is done so stream output is transmitted to the renderer or CLI and made available to the debugger/CLI.
- **current wiring:** src/vs/workbench/api/node/extHostExtensionService.ts creates ExtHostConsoleForwarder before running extensions, so this forwarding still applies on current main.
- **maintainer context:** In https://github.com/microsoft/vscode/issues/150113#issuecomment-1139343611, bpasero notes that the debug window in VS Code shows the output normally and that DevTools should not be treated as the main way to debug an extension.
