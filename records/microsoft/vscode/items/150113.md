# Issue #150113 — Duplicate output in devtools when extension writes to process.stdout

[https://github.com/microsoft/vscode/issues/150113](https://github.com/microsoft/vscode/issues/150113)

## Summary

Keep open under extension-host discussion: current main still intentionally forwards extension-host stdout both through the console path and the original stream, and the only maintainer response frames DevTools as secondary rather than declaring the duplicate output fixed or out of scope.

## Change summary

An extension calling `process.stdout.write` in development mode produces duplicate output in VS Code Developer Tools.

## Best solution

Have the extension-host owners decide whether renderer DevTools should de-duplicate direct stdout while preserving Debug Console and CLI visibility. The likely touch points are `src/vs/workbench/api/node/extHostConsoleForwarder.ts` and `src/vs/workbench/services/extensions/electron-browser/localProcessExtensionHost.ts`.

## Evidence

- **current code:** `src/vs/workbench/api/node/extHostConsoleForwarder.ts` wraps `process.stdout.write`, calls `console[severity](...)` for complete lines, and then calls the original stream write, matching the two output paths described in the report.
- **renderer forwarding:** `src/vs/workbench/services/extensions/electron-browser/localProcessExtensionHost.ts` maps extension-host stdout/stderr to renderer console output, while `src/vs/workbench/api/browser/mainThreadConsole.ts` also logs forwarded extension-host console messages to the local console.
- **maintainer discussion:** In https://github.com/microsoft/vscode/issues/150113#issuecomment-1139335226, bpasero noted that the normal extension debugging experience is the VS Code debug window and that DevTools is a nice-to-have, but did not mark the issue as fixed or explicitly as-designed.
- **history check:** `git log --grep=150113` found no commit explicitly closing this issue; commit f9acb97f3c75a9c4626942d7ed7f5cd0e1621f7f changed console forwarding after the report but retained the documented stdout-to-console plus original-write behavior.
