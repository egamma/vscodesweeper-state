# Issue #240780 — Failed to execute 'createScriptURL' on 'TrustedTypePolicy'

[https://github.com/microsoft/vscode/issues/240780](https://github.com/microsoft/vscode/issues/240780)

## Summary

Team-authored (MEMBER) bug still under active investigation — the Trusted Types error during F5 debugging was confirmed reproducible even with the js-debug nightly fix, and the root cause appears to involve Chromium/Blink executing scripts while the page is in a paused debugger state. Must keep open for maintainer judgment.

## Change summary

F5 debugging triggers 'Failed to execute createScriptURL on TrustedTypePolicy: The provided callback is no longer runnable' errors from amdX.ts when Chromium runs scripts during a debugger-paused state.

## Best solution

The root cause is in the Chromium/Blink debugger lifecycle — scripts execute via `amdX.ts` line 163 while the page's execution context considers the TrustedTypePolicy callback no longer runnable. The js-debug fix (vscode-js-debug#2196) addressed a related aux-window case but not this core F5 scenario. Resolution likely requires either a workaround in `src/vs/amdX.ts` (deferring script loading until the page is unpaused) or an upstream Chromium fix for the `generated_code_helper.cc` pause-state check. deepak1556's Chromium-level investigation in comment 2 is the best lead.
