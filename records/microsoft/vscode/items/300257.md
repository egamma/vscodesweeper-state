# Issue #300257 — Syntax highlighting breaks for LESS CSS variables starting with "@container"

[https://github.com/microsoft/vscode/issues/300257](https://github.com/microsoft/vscode/issues/300257)

## Summary

Confirmed upstream grammar bug in radium-v/Better-Less: the `at-container` rule's begin pattern `(?=\s*@container)` matches LESS variables starting with `@container` (e.g. `@container-min-width`), breaking subsequent highlighting. The fix belongs upstream.

## Change summary

LESS variables starting with `@container` (like `@container-min-width`) are incorrectly tokenized as CSS `@container` at-rules, breaking syntax highlighting for the rest of the file.

## Best solution

File a bug at https://github.com/radium-v/Better-Less to add a word boundary or negative lookahead (e.g. `(?=\s*@container(?![\w-]))`) to the `at-container` rule's begin pattern. Once fixed upstream, update VS Code's grammar via `extensions/less/build/update-grammar.js`.
