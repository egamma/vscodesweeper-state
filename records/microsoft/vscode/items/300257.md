# Issue #300257 — Syntax highlighting breaks for LESS CSS variables starting with "@container"

[https://github.com/microsoft/vscode/issues/300257](https://github.com/microsoft/vscode/issues/300257)

## Summary

Confirmed upstream grammar bug in the Better-Less TextMate grammar (radium-v/Better-Less). The `@container` at-rule pattern greedily matches LESS variables that start with `@container`, breaking subsequent highlighting. No fix has landed on main.

## Change summary

LESS variables starting with `@container` (e.g. `@container-min-width`) break syntax highlighting because the grammar's at-rule pattern matches them as CSS container queries.

## Best solution

Fix should go upstream in https://github.com/radium-v/Better-Less — the `at-container` rule's begin pattern (`(?=\s*@container)`) needs a word-boundary or negative lookahead (e.g. `(?=\s*@container\s)`) to avoid matching variable names. Once fixed upstream, update the vendored grammar in `extensions/less/syntaxes/less.tmLanguage.json`.
