# Issue #310382 — Diff view crash/failure

[https://github.com/microsoft/vscode/issues/310382](https://github.com/microsoft/vscode/issues/310382)

## Summary

Confirmed on current main: the move-extension loop in computeMovedLines.ts lacks a lower-bound check, so origLine/modLine can reach 0, indexing the lines array at -1 and crashing areLinesSimilar with undefined.

## Change summary

Diff view crashes when computing moved lines because the extend-to-top loop doesn't guard against origLine or modLine going below 1.

## Best solution

Add `origLine < 1 || modLine < 1` to the bounds check on line 210 (and the analogous check would protect line 236 for the extend-to-bottom loop, though that path is less likely to underflow). A unit test with the reporter's XML samples should validate the fix.
