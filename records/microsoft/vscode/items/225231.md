# Issue #225231 — Git - Source Control / File Watcher does not respect settings

[https://github.com/microsoft/vscode/issues/225231](https://github.com/microsoft/vscode/issues/225231)

## Summary

Confirmed in source: when `git.autoRepositoryDetection` is `false`, `scanWorkspaceFolders()` returns early at line 366, so `git.scanRepositories` entries are never processed. Also, `git.ignoredRepositories` requires exact paths (not globs), which the second commenter misused. The fundamental limitation — no exclusive scan mode — is still present on main.

## Change summary

Setting `git.autoRepositoryDetection: false` with `git.scanRepositories` paths does not limit scanning to only those paths; instead it disables scanning entirely, ignoring the scanRepositories list.

## Best solution

Fix `scanWorkspaceFolders` in `extensions/git/src/model.ts` so that when `autoRepositoryDetection` is `false`, `git.scanRepositories` entries are still honored (only the automatic traversal is skipped). This gives users an explicit-only scan mode. The `ignoredRepositories` glob limitation is a separate enhancement.
