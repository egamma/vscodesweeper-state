# Issue #284742 — [BUG]: VSCode incorrect syntax highlighting problem

[https://github.com/microsoft/vscode/issues/284742](https://github.com/microsoft/vscode/issues/284742)

## Summary

Already triaged by a team member as a feature-request and placed in the Backlog Candidates 60-day voting process. The 512-color limit is a known architectural constraint of the 9-bit foreground color index in encoded token attributes. No action needed — let the community process run.

## Change summary

Themes with more than ~512 unique foreground colors produce corrupted syntax highlighting because the foreground color index is stored as a 9-bit value in the packed token representation.

## Best solution

If the community votes it up, the fix would be to widen the foreground color bits in the token encoding (e.g. a 12/6 fg/bg split as RedCMD suggested) in both `src/vs/editor/common/encodedTokenAttributes.ts` and the upstream `vscode-textmate` package. Alternatively, surface a warning when a theme exceeds the color limit. Already in the correct voting pipeline.
