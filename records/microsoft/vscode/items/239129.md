# Issue #239129 — vscode is not respecting users choice to not be part of the experiments.

[https://github.com/microsoft/vscode/issues/239129](https://github.com/microsoft/vscode/issues/239129)

## Summary

VS Code core properly gates all experimentation traffic on `workbench.enableExperiments` — when set to false, no TAS client is created and no calls to `default.exp-tas.com` are made from VS Code itself. The traffic is almost certainly originating from an installed extension with its own experimentation service. No follow-up from the reporter in over 17 months.

## Change summary

Network requests to `default.exp-tas.com` continue even when `workbench.enableExperiments` is set to false.

## Best solution

Identify the specific extension making the calls. The reporter should launch with `--disable-extensions` and confirm the traffic stops, then use extension bisect to find the culprit and report the issue to that extension's publisher.
