# Issue #322678 — Feature request: allow per-option descriptions in snippet choice placeholders (`${1|...|}`)

[https://github.com/microsoft/vscode/issues/322678](https://github.com/microsoft/vscode/issues/322678)

## Summary

Well-articulated feature request to add per-option descriptions in snippet choice placeholders. No duplicate found; the snippet grammar currently has no support for this. Keeping open for community traction.

## Change summary

Allow snippet choice placeholder options to carry descriptions shown in the dropdown, analogous to how snippet-level descriptions appear in IntelliSense.

## Best solution

This would require extending the snippet grammar in `src/vs/editor/contrib/snippet/browser/snippetParser.ts` (the `Choice` class and its `options: Text[]`) and the choice completion provider in `snippetSession.ts` to surface descriptions. A richer JSON object form in the snippet definition (mapping values to descriptions) is more realistic than an inline separator syntax. Worth discussing whether the TextMate snippet grammar compatibility is a blocker.
