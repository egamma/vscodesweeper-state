# Issue #283298 — Cannot read properties of undefined (reading 'normalizePosition')

[https://github.com/microsoft/vscode/issues/283298](https://github.com/microsoft/vscode/issues/283298)

## Summary

Still present on main: `normalizePosition` in `viewModelLines.ts` accesses `modelLineProjections[index]` without guarding against undefined during a race where `_constructLines` rebuilds the array while cursor events fire synchronously. PR #285602 attempted a fix but was closed without merging.

## Change summary

Race condition in ViewModelLines causes TypeError when `normalizePosition` is called on an incomplete `modelLineProjections` array during reconstruction.

## Best solution

Add defensive null/bounds checks (or an `isValidLineIndex` guard) in `normalizePosition()` and adjacent access points in `src/vs/editor/common/viewModel/viewModelLines.ts`, similar to the approach in closed PR #285602 and orphaned commit c27f49f3a7c. Alternatively, prevent the race by deferring cursor state updates until `_constructLines` completes.
