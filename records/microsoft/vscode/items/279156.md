## Summary

Fixed on main: the listener leak from chatInlineAnchorWidget creating ResourceContextKey (with global model listeners) was resolved by introducing StaticResourceContextKey, which skips the listeners entirely. Shipped in 1.109.0.

## Change summary

Chat inline anchor widgets created many ResourceContextKey instances that each registered global model-change listeners, triggering a listener leak warning.

## Best solution

Keep the shipped fix (StaticResourceContextKey) in commit 141b5fe8ddc; no further change needed. The two related issues (#255830, #260082) were already closed as duplicates of #249666, which is also closed.
