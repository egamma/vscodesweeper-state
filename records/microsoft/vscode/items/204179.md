# Issue #204179 — Docs mistake: `FindTextInFilesOptions.useDefaultExcludes` defaults to false, not true.

[https://github.com/microsoft/vscode/issues/204179](https://github.com/microsoft/vscode/issues/204179)

## Summary

Still reproduces on current main: the `findTextInFiles` API's `useDefaultExcludes` effectively defaults to `false` (line 877 of extHostWorkspace.ts sets `disregardExcludeSettings: true` when the option is undefined), contradicting the JSDoc that says 'Defaults to true'. Team-authored issue, keeping open.

## Change summary

The `findTextInFiles` API's `useDefaultExcludes` option defaults to false at runtime but the docs claim it defaults to true.

## Best solution

Either change the runtime default so `disregardExcludeSettings` is `false` when `useDefaultExcludes` is undefined (making excludes respected by default, matching the docs), or update the JSDoc to say 'Defaults to false'. Since this is still a proposed API, changing behavior to match the docs (defaulting to true) is the better fix — aligns with user expectations.
