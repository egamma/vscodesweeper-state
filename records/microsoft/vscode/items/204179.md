# Issue #204179 — Docs mistake: `FindTextInFilesOptions.useDefaultExcludes` defaults to false, not true.

[https://github.com/microsoft/vscode/issues/204179](https://github.com/microsoft/vscode/issues/204179)

## Summary

Still reproduces on main: the d.ts docs claim useDefaultExcludes defaults to true, but the runtime (extHostWorkspace.ts:877) sets disregardExcludeSettings to true when the option is undefined, meaning excludes are effectively not applied — the real default is false. This is a team-authored issue on a proposed API, so keeping open.

## Change summary

The `findTextInFiles` API option `useDefaultExcludes` defaults to `false` at runtime but the d.ts documentation says it defaults to `true`.

## Best solution

Either fix the runtime default so `disregardExcludeSettings` is `false` when `useDefaultExcludes` is undefined (making the behavior match the docs), or update the d.ts comment to say "Defaults to false." Since this is still a proposed API, changing the runtime to default to `true` (matching docs) is the better path, as the author originally noted.
