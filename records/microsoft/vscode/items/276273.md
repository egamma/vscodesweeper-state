# Issue #276273 — Publish Branch dialog closes immediately when connected to WSL and Windows proxy is set

[https://github.com/microsoft/vscode/issues/276273](https://github.com/microsoft/vscode/issues/276273)

## Summary

Still present on main: the publish-branch flow in the git extension lacks error handling around the `publishRepository` and `pushTo` calls, so a network error (like ECONNREFUSED from an unreachable WSL proxy) causes the dialog to close silently instead of showing an error. A team member already reassigned it for error-handling work.

## Change summary

The 'Publish Branch' dialog closes immediately without showing an error when a network/proxy failure occurs during publishing in WSL.

## Best solution

Add try-catch error handling around the `publisher.publishRepository()` and `repository.pushTo()` calls in the `publish` command in `extensions/git/src/commands.ts` (around lines 4378, 4385, 4404, 4409) to surface a user-visible error message instead of silently closing the dialog.
