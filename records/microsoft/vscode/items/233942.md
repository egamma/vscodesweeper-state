# Issue #233942 — Semantic tokens are causing a large changed range onTokensChanged event to fire

[https://github.com/microsoft/vscode/issues/233942](https://github.com/microsoft/vscode/issues/233942)

## Summary

Team-authored follow-up requesting that the semantic tokens onTokensChanged event be tightened to only fire for ranges that actually change. No fix has landed on main; the issue remains valid.

## Change summary

Semantic tokens fire a large onTokensChanged range covering the entire viewport when scrolling stops, even though no tokens actually changed.

## Best solution

Tighten the semantic token write path to diff incoming tokens against existing ones and only emit changed ranges in the event, reducing unnecessary cache invalidation in GPU rendering and other consumers.
