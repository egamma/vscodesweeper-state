# Issue #233942 — Semantic tokens are causing a large changed range onTokensChanged event to fire

[https://github.com/microsoft/vscode/issues/233942](https://github.com/microsoft/vscode/issues/233942)

## Summary

Keep open as a Backlog semantic-tokens feature request. Current main still routes viewport semantic-token refreshes through a full requested range, and I did not find a concrete fixing commit or canonical duplicate for https://github.com/microsoft/vscode/issues/233942.

## Change summary

Tighten semantic-token token-change events so unchanged viewport-sized ranges do not invalidate every visible line after scrolling stops.

## Best solution

If accepted, implement narrower changed-range computation in `src/vs/editor/common/tokens/sparseTokensStore.ts` / `src/vs/editor/common/model/tokens/tokenizationTextModelPart.ts`, likely by comparing replaced sparse token pieces and emitting only lines whose semantic tokens actually changed after viewport refresh.
