# Issue #82794 — Provision trusted domains for enterprise setup

[https://github.com/microsoft/vscode/issues/82794](https://github.com/microsoft/vscode/issues/82794)

## Summary

Still a valid, unimplemented feature request. Trusted domains currently come from the product config, browser environment options, and per-user storage — there is no configuration setting or group policy to pre-configure them for enterprise deployment. The issue has 18 upvotes and ongoing community interest (most recent comment September 2025), approaching the backlog promotion threshold.

## Change summary

Allow enterprises to pre-configure a list of trusted domains via a VS Code setting or group policy, so link-protection prompts are suppressed for known-safe domains across managed deployments.

## Best solution

Add a `workbench.trustedDomains.list` (or similar) configuration setting with `policy` support so enterprise admins can push trusted domains via group policy (Windows GPO / macOS managed preferences). The setting would feed into `readStaticTrustedDomains()` in `src/vs/workbench/contrib/url/browser/trustedDomains.ts` alongside the existing `productService.linkProtectionTrustedDomains` and storage-based sources. This aligns with the existing group-policy infrastructure (`src/vs/platform/policy/`).
