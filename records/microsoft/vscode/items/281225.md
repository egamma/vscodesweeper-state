[https://github.com/microsoft/vscode/issues/281225](https://github.com/microsoft/vscode/issues/281225)

## Summary

The reported mouse-click-and-drag behavior when moving between an elevated VS Code and a non-elevated Chrome window is consistent with Windows' User Interface Privilege Isolation (UIPI), which restricts input messages between processes at different integrity levels. The reporter confirms it stopped as soon as they launched VS Code without admin rights, and no stable reproduction exists. Needs more info to determine if this is anything beyond the known OS-level behavior.

## Change summary

Moving the mouse from an admin-elevated VS Code to a non-elevated Chrome window on a dual-monitor setup causes an unwanted click and ~600px downward offset.

## Best solution

This is very likely a Windows UIPI (User Interface Privilege Isolation) artifact — mixing elevated and non-elevated windows is known to cause mouse/input anomalies on Windows. Prior VS Code issues with admin + mouse problems (e.g. https://github.com/microsoft/vscode/issues/165375) were closed as `upstream`/`chromium`. If the reporter can provide a stable reproduction on the latest Insiders build, the issue could be re-evaluated; otherwise it should follow the `info-needed` auto-close path.
