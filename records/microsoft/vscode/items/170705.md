# Issue #170705 — PHP formatting on typing

[https://github.com/microsoft/vscode/issues/170705](https://github.com/microsoft/vscode/issues/170705)

## Summary

Keep this open: a maintainer already reproduced and traced the bad PHP/HTML indentation to mixed-language auto-indent handling rather than an extension or user setting. I did not find concrete evidence of a fixing commit for this issue on current main.

## Change summary

Typing an HTML closing tag inside a PHP file can remove its indentation and move it to column 1.

## Best solution

Fix the embedded-language auto-indent path in `src/vs/editor/common/languages/autoIndent.ts`, likely by computing inherited indentation with the relevant scoped language for prior lines, and add a regression test for HTML indentation inside PHP.

## Evidence

- **maintainer diagnosis:** In the issue discussion, roblourens identified that `.php` inherits HTML indentation rules, but inherited indentation was being computed using the model language `php` for a previous line, causing over-deindentation.
- **current code:** `src/vs/editor/common/languages/autoIndent.ts:378-396` still gets the typed line's language at the cursor and then calls `getInheritIndentForLine`; `src/vs/editor/common/languages/autoIndent.ts:86` and `:152` use the model/tokenization-level indent rules for inherited indentation metadata.
- **history check:** `git log -- src/vs/editor/common/languages/autoIndent.ts` shows later indentation fixes such as `8ca1b8c195b`, `d309e11579e`, and `bb5150c70d8`, but I found no commit or PR explicitly fixing https://github.com/microsoft/vscode/issues/170705.
- **workaround only:** The maintainer-suggested workaround was lowering `editor.autoIndent` below `Full`, which avoids the path but also disables expected automatic indentation behavior.
