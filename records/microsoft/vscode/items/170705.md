# Issue #170705 — PHP formatting on typing

[https://github.com/microsoft/vscode/issues/170705](https://github.com/microsoft/vscode/issues/170705)

## Summary

Still reproduces on main: getInheritIndentForLine uses model.tokenization.getLanguageId() (PHP) instead of the position-based language (HTML), so the decreaseIndentPattern is applied with the wrong language's indent rules. The root cause identified by roblourens in comment 3 remains unfixed.

## Change summary

Typing HTML closing tags in a PHP file causes them to lose indentation because inherited indentation is computed using PHP's language rules instead of the embedded HTML language at that position.

## Best solution

Fix getInheritIndentForLine in src/vs/editor/common/languages/autoIndent.ts to accept and use a position-based languageId (via getLanguageIdAtPosition) rather than the model's top-level language when computing inherited indentation for embedded languages.
