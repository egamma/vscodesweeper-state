# Issue #291504 — GitHub Copilot Chat: Account is not changed in the chat sidebar, when changed using the Account Preferences menu

[https://github.com/microsoft/vscode/issues/291504](https://github.com/microsoft/vscode/issues/291504)

## Summary

Real bug confirmed by two users: Copilot Chat does not respect account preference changes when multiple GitHub accounts are signed in. Info was already provided (logs, screenshots) so the `info-needed` label is stale. A June 2026 auth-hardening commit (1717758dcb7) may address this but cannot be conclusively verified against this scenario.

## Change summary

Changing the GitHub account via 'Account Preferences' for Copilot Chat does not take effect — the chat continues using the previously selected account until the unwanted account is fully signed out.

## Best solution

Verify whether commit 1717758dcb7 ('Harden the auth change event', June 2026) resolves this multi-account scenario. If so, close as fixed. If not, the Copilot Chat extension needs to properly react to `onDidAuthenticationChange` when the preferred account switches, re-acquiring the token for the newly selected account without requiring a full sign-out/restart.
