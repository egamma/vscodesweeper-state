# Issue #250399 — Cmd+Backspace should also work if focus is on elements of the chat output.

[https://github.com/microsoft/vscode/issues/250399](https://github.com/microsoft/vscode/issues/250399)

## Summary

Team-authored bug about Cmd+Backspace (or Ctrl+Backspace on Windows) not working to undo a chat request when focus is on elements within chat output — keeping open per team-member policy. The keybinding exists but its `when` clause may not fire correctly when focus lands on interactive output elements (code blocks, buttons). A third-party commenter reports it working on Mac, but the original report was Windows-specific.

## Change summary

Cmd+Backspace (Mac) / Delete (Windows) to undo a chat request does not trigger when keyboard focus is on interactive elements within the chat response output.

## Best solution

Investigate whether the `when` clause on the RemoveAction / RestoreCheckpointAction keybindings (in `src/vs/workbench/contrib/chat/browser/chatEditing/chatEditingActions.ts`) correctly evaluates when focus is on output elements like code blocks or rendered content. The `ChatContextKeys.inChatSession` context key may not propagate into child scoped context key services used by embedded editors or output elements. The fix likely involves ensuring the keybinding `when` clause covers the output-focused state, or adjusting context key scoping so `inChatSession` remains true when focus is within output elements.

## Evidence

- **keybinding definition:** The RemoveAction keybinding at src/vs/workbench/contrib/chat/browser/chatEditing/chatEditingActions.ts:447-451 binds Delete (all platforms) and Cmd+Backspace (mac only), with `when: ChatContextKeys.inChatSession && !EditorContextKeys.textInputFocus && !inChatQuestionCarousel && !readOnly`.
- **context key scope:** `ChatContextKeys.inChatSession` is set at src/vs/workbench/contrib/chat/browser/widget/chatWidget.ts:473, bound to the widget's scoped context key service. Output elements with their own context key scopes may not inherit this.
- **parent test plan:** Filed while testing #250357 (Chat UI/UX Refresh TPI), specifically step 4 about keyboard navigation for undoing requests.
- **platform:** Reporter (dbaeumer) is on Windows with both Backspace and Delete keys; only Delete worked. On Windows the primary binding is Delete (no Ctrl+Backspace equivalent is bound for Windows).
