# Issue #262823 — code.desktop on linux should not have execution permissions

[https://github.com/microsoft/vscode/issues/262823](https://github.com/microsoft/vscode/issues/262823)

## Summary

Valid polish issue: `resources/linux/code.desktop` is tracked in git as mode 100755 (executable), while the freedesktop.org Desktop Entry spec does not require `.desktop` files to be executable. The sibling `code-url-handler.desktop` correctly uses 100644. The executable bit was added in commit 10c24b07734 as a side-effect of the icon-path fix (#95459), likely unintentionally.

## Change summary

The `code.desktop` file shipped in Linux packages has unnecessary execute permissions (mode 755 instead of 644).

## Best solution

Remove the executable bit from `resources/linux/code.desktop` in git (`git update-index --chmod=-x resources/linux/code.desktop`) to match `code-url-handler.desktop` and the freedesktop.org spec. No functional change expected — desktop environments read `.desktop` files from `/usr/share/applications/` regardless of the execute bit.

## Evidence

- **git file mode:** `git ls-files -s resources/linux/code.desktop` shows mode 100755; `git ls-files -s resources/linux/code-url-handler.desktop` shows mode 100644 — inconsistent within the same directory.
- **exec bit origin:** Commit https://github.com/microsoft/vscode/commit/10c24b07734cb6920ff51f76d1cea88b647c03d8 (2020-04-25) changed the file from 100644 to 100755 as 'Part of #95459', but that issue was about the icon path, not permissions — the exec bit was likely incidental.
- **freedesktop spec:** The freedesktop.org Desktop Entry Specification does not require .desktop files in /usr/share/applications/ to be executable; the execute bit is unnecessary.
