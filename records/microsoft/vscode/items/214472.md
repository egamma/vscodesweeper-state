# Issue #214472 — Investigate Editor Placeholder Adoption

[https://github.com/microsoft/vscode/issues/214472](https://github.com/microsoft/vscode/issues/214472)

## Summary

Team-authored tracking issue for adopting the editor's built-in placeholder API across the workbench. Two of four original items are done (SCM, comment widget), but the chat input editor and SuggestEnabledInput still use custom placeholder implementations, and the empty editor hint (raised by bpasero) also remains unadopted. Remaining work is valid.

## Change summary

Migrate remaining custom placeholder implementations (chat input, SuggestEnabledInput, empty editor hint) to the editor's built-in placeholder API.

## Best solution

Complete the remaining adoption: (1) chat input (`src/vs/workbench/contrib/chat/browser/widget/input/editor/chatInputEditorContrib.ts`) still uses decoration-based placeholders; (2) `SuggestEnabledInput` (`src/vs/workbench/contrib/codeEditor/browser/suggestEnabledInput/suggestEnabledInput.ts`) overlays a DOM element; (3) empty editor hint (`src/vs/workbench/contrib/codeEditor/browser/emptyTextEditorHint/emptyTextEditorHint.ts`) uses a custom overlay widget. Each should migrate to the editor placeholder API, resolving the related width/scrollbar issue (https://github.com/microsoft/vscode/issues/228111) along the way.
