# Issue #326263 — hovering over run buttons sometimes moves the gutter and prevents test run buttons from working

[https://github.com/microsoft/vscode/issues/326263](https://github.com/microsoft/vscode/issues/326263)

## Summary

Legitimate bug: the glyph margin resizes dynamically when decoration lane count changes after a test run completes, shifting the run button away from the cursor. The issue is intermittent and the reporter confirms ~20% reproduction rate but cannot provide exact steps.

## Change summary

Hovering over test run gutter buttons after a test completes sometimes causes the glyph margin to resize, moving the button out from under the cursor and preventing interaction.

## Best solution

The root cause is likely in `src/vs/editor/browser/view.ts` where `_shouldRecomputeGlyphMarginLanes` triggers a layout change via `setGlyphMarginDecorationLaneCount` during the render cycle when test result decorations change. The fix should either stabilize the lane count during hover interaction, or debounce/delay the glyph margin resize to avoid stealing the click target. This belongs in the `testing` and/or `editor-gutter` area.
