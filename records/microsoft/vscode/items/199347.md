# Issue #199347 — Use empty view zone for foreign view zone caused by find

[https://github.com/microsoft/vscode/issues/199347](https://github.com/microsoft/vscode/issues/199347)

## Summary

Team-authored polish request for the diff editor's find widget view zone alignment — must stay open for maintainer judgment.

## Change summary

When the find widget inserts a view zone in one diff editor side, the other side's corresponding hashed view zone should use an empty view zone instead of displaying hashed content.

## Best solution

Special-case the find widget's view zone in the diff editor so the opposing side uses an empty (blank) view zone rather than the default hashed pattern, improving visual clarity when the find widget is open.
