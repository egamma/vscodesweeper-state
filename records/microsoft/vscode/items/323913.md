# Issue #323913 — Feature Request: Per-agent MCP server scoping to reduce context overhead

[https://github.com/microsoft/vscode/issues/323913](https://github.com/microsoft/vscode/issues/323913)

## Summary

Valid feature request for per-agent MCP server scoping to reduce context overhead. No existing implementation found on main; the tools: field in .agent.md restricts invocation but not schema loading. Keeping open for community traction and team consideration.

## Change summary

Allow MCP servers in mcp.json to declare which agents they load for, so unused servers don't consume context tokens for unrelated agents.

## Best solution

Consider an `agents` field in mcp.json server entries (or inline MCP defs in .agent.md) that gates both server startup and schema injection per agent. This touches the MCP server sync logic in `src/vs/workbench/contrib/chat/browser/agentSessions/agentHost/` and the customization bundler. Alternatively, make the existing `tools:` restriction in .agent.md also suppress schema loading — a lighter-weight approach.
