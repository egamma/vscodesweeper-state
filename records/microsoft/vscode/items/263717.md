# Issue #263717 — Selecting text longer that window is impossible

[https://github.com/microsoft/vscode/issues/263717](https://github.com/microsoft/vscode/issues/263717)

## Summary

Valid polish issue acknowledged by a team member. The chat view does not auto-scroll when drag-selecting text beyond the visible viewport. A workaround exists (right-click → Copy All), and a team member noted interest in addressing range selection. No fix has landed on main.

## Change summary

Dragging to select text in a Copilot chat response does not auto-scroll the chat view when the mouse moves beyond the viewport edge, making it impossible to select text spanning more than one screen.

## Best solution

Implement pointer-drag auto-scrolling in the chat list widget so that when a user drags a text selection past the top or bottom edge of the chat viewport, the container scrolls automatically — similar to how code editors and other scrollable text containers behave. The fix likely belongs in `src/vs/workbench/contrib/chat/browser/widget/chatListWidget.ts` or the underlying list view layer. A team member already acknowledged the gap.
