# Issue #246946 — Git - Partial Git repositories cause problems in VS Code Source Control panel

[https://github.com/microsoft/vscode/issues/246946](https://github.com/microsoft/vscode/issues/246946)

## Summary

Issue is milestoned 'On Deck' — a real scheduling milestone — so it must stay open per triage policy, regardless of other factors. The report describes a real gap: the Git extension assumes complete history, causing hangs and errors with shallow/grafted repos.

## Change summary

VS Code's Source Control panel hangs or throws errors when working with shallow-cloned (grafted) Git repositories that have incomplete history.

## Best solution

The Git extension (`extensions/git/`) should gracefully handle grafted/shallow repositories — detect the shallow state and avoid operations that assume full history (e.g. amend/rebase reaching beyond the graft point). The existing `unshallow` support in pull is a starting point but the broader interaction surface (diff viewing, amend, rebase) needs guarding.
