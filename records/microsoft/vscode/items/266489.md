# Issue #266489 — Shell integration fails with set -u

[https://github.com/microsoft/vscode/issues/266489](https://github.com/microsoft/vscode/issues/266489)

## Summary

Mid-verification (`verification-found` label): PR #270310 merged a partial fix but verification found the issue persists. Must stay open per verification lifecycle rules.

## Change summary

Shell integration scripts fail with unbound variable errors when the user's shell has `set -u` (nounset) enabled.

## Best solution

Audit remaining unguarded variable references in the shell integration scripts (shellIntegration-bash.sh and shellIntegration-rc.zsh) — particularly around `vsc_aa_env` array accesses at lines 181/212/216 in the zsh script which still reference array elements without `:-` default guards under nounset mode. A follow-up fix should ensure every variable expansion uses `${var:-}` or equivalent.
