# Issue #279068 — Fedora 43 (Rawhide): VS Code fails to render window without --no-sandbox and GPU overrides

[https://github.com/microsoft/vscode/issues/279068](https://github.com/microsoft/vscode/issues/279068)

## Summary

This is an upstream Electron/Chromium compatibility issue with Fedora 43 Rawhide's newer kernel, glibc, and pure-Wayland session. Both the sandbox failure and the GPU rendering black screen originate in Chromium's seccomp sandbox and GPU compositing layers, not in VS Code's own code. Routing to the linux/wayland area for upstream tracking alongside the related open issue #282662.

## Change summary

VS Code fails to render its window on Fedora 43 (Rawhide) under pure Wayland unless launched with --no-sandbox and GPU override flags.

## Best solution

Track this as an upstream Electron/Chromium issue. The sandbox conflict with newer glibc/kernel and the Wayland GPU rendering failure both need fixes in Chromium's sandbox and compositor code, which will reach VS Code through Electron updates. The closely related https://github.com/microsoft/vscode/issues/282662 (black screen on Debian, already labeled upstream/linux/gpu/wayland) covers the GPU rendering half; the sandbox half may warrant a separate upstream Chromium bug if not already filed. No VS Code code change will resolve this.
