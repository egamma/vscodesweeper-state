# Issue #188706 — Editor does not re-evaluate indentation after quickfix 

[https://github.com/microsoft/vscode/issues/188706](https://github.com/microsoft/vscode/issues/188706)

## Summary

Team-authored bug, still reproduces on main: indentation detection only runs at file open or via explicit command — no re-detection after code actions modify indentation. Keeping open per policy.

## Change summary

After a quickfix changes file indentation (e.g. ESLint fix spaces→tabs), saving re-formats with the stale detected indentation because the model never re-evaluates.

## Best solution

After a code action applies edits that change indentation characters, trigger `model.detectIndentation()` so subsequent format-on-save uses the updated indentation style. The fix likely lives in the code-action application path or in the model service's response to bulk edits that alter whitespace patterns.
