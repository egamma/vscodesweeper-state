# Issue #312107 — Copilot Autopilot (Preview) mode answers its own questions without waiting for user input

[https://github.com/microsoft/vscode/issues/312107](https://github.com/microsoft/vscode/issues/312107)

## Summary

Autopilot mode is explicitly designed to run autonomously without blocking on user questions. The askQuestions tool intentionally auto-responds in Autopilot mode, telling the model to 'work autonomously and make good decisions.' This is as-designed; the reporter wants a different behavior (blocking or configurable delay), which is a feature request against an intentional design choice.

## Change summary

In Autopilot (Preview) mode, the askQuestion tool auto-responds instead of blocking for user input, so the agent proceeds without waiting.

## Best solution

Close as designed. Autopilot mode's core purpose is autonomous operation. The request for configurable blocking/delay (comment 4/5) is a valid feature discussion but would need a separate, scoped feature request — e.g. a setting to control whether askQuestions blocks in Autopilot mode. The current behavior was deliberately simplified in commits 121ed5f8651 and 06d480b4f97.
