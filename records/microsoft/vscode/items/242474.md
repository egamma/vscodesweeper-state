# Issue #242474 — Following link from Jasmine testing framework breaks search

[https://github.com/microsoft/vscode/issues/242474](https://github.com/microsoft/vscode/issues/242474)

## Summary

Still reproduces on main: terminal links with URI query parameters (like `?__quibble=...`) cause `createAndConfigureFileMatch` to throw because `isEqualOrParent` requires matching query strings, so the file URI with a query component fails the descendant check against the workspace folder. Same root cause as #296608 (fragments). Not yet fixed.

## Change summary

Opening a terminal link containing a query string (e.g. `?__quibble=12`) breaks multi-file search with a 'not a descendant' error.

## Best solution

Strip query and fragment from file URIs before the `uriHasParent` check in `createAndConfigureFileMatch` (in `src/vs/workbench/contrib/search/browser/searchTreeModel/folderMatch.ts`), or normalize URIs at the point they enter the search model. This also fixes the related #296608 (fragment variant). Alternatively, the terminal link handler could strip query/fragment before opening, but the search model should be defensive regardless.
