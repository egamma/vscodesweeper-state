# Issue #227484 — Custom controls need to provide proper textual name, role, and state information 

[https://github.com/microsoft/vscode/issues/227484](https://github.com/microsoft/vscode/issues/227484)

## Summary

Still reproduces on main: folding gutter controls are rendered as CSS decoration icons without ARIA roles, aria-expanded state, or accessible names — screen readers cannot identify them as interactive expand/collapse controls.

## Change summary

Folding expand/collapse controls in the editor gutter lack accessible name, role, and aria-expanded state for screen readers.

## Best solution

The folding gutter controls in `src/vs/editor/contrib/folding/browser/foldingDecorations.ts` use `firstLineDecorationClassName` with icon CSS — they need proper ARIA semantics (role=button, aria-expanded, aria-label). This likely also involves the glyph margin rendering in `src/vs/editor/browser/` to support interactive ARIA attributes on decoration elements.
