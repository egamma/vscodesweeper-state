# Issue #227484 — Custom controls need to provide proper textual name, role, and state information 

[https://github.com/microsoft/vscode/issues/227484](https://github.com/microsoft/vscode/issues/227484)

## Summary

Still valid on main: folding gutter controls are rendered as plain icons without role, aria-label, or aria-expanded attributes, making them invisible to screen readers.

## Change summary

Folding expand/collapse controls in the editor gutter lack accessible name, role, and state (aria-expanded) for screen reader users.

## Best solution

Add role="button", aria-label (e.g. "Fold region" / "Unfold region"), and aria-expanded="true/false" to the folding glyph margin elements in `src/vs/editor/contrib/folding/browser/foldingDecorations.ts` or the glyph margin rendering layer.
