# Issue #227484 — Custom controls need to provide proper textual name, role, and state information 

[https://github.com/microsoft/vscode/issues/227484](https://github.com/microsoft/vscode/issues/227484)

## Summary

Still valid on main: folding gutter controls are rendered as decoration icons without ARIA roles, `aria-expanded`, or accessible names — screen readers cannot identify them as expand/collapse buttons.

## Change summary

Folding controls in the editor gutter lack accessible name, role, and state information for screen reader users.

## Best solution

Add `role="button"`, `aria-expanded="true/false"`, and `aria-label` to the folding gutter decoration elements in `src/vs/editor/contrib/folding/browser/foldingDecorations.ts` and the glyph margin rendering layer, so screen readers announce them as toggle buttons with state.
