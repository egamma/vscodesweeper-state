# Issue #286578 — Defer InlineCompletionsModel to after the editor has opened

[https://github.com/microsoft/vscode/issues/286578](https://github.com/microsoft/vscode/issues/286578)

## Summary

Still open on main — the InlineCompletionsModel is eagerly created in a derivedDisposable as soon as the editor model is available (inlineCompletionsController.ts:99–115), with no deferral. No commits reference this issue.

## Change summary

Defer InlineCompletionsModel initialization until after the editor has fully opened to reduce editor-switch latency.

## Best solution

Introduce a deferral gate (e.g. idle callback or a one-frame delay) before the derivedDisposable in `inlineCompletionsController.ts` creates the InlineCompletionsModel, so it doesn't run in the microtask queue of the editor-opening mousedown event.
