# Issue #275010 — Copilot Chat edit approval modal causes UI freeze when VS Code loses focus

[https://github.com/microsoft/vscode/issues/275010](https://github.com/microsoft/vscode/issues/275010)

## Summary

Upstream Windows OS bug in the toast notification service causing a UI-thread deadlock in Electron apps; both the Electron-side mitigation and the Windows OS fix (KB5086672) have shipped. A team member (deepak1556) is actively tracking and waiting for user confirmation that the OS update resolves the issue — keep open until that confirmation arrives.

## Change summary

VS Code freezes when Copilot Chat triggers a notification while the window is out of focus, due to the Windows toast notification COM service hanging on the UI thread.

## Best solution

Wait for user confirmation that the Windows OS fix (KB5086672, builds 26100.8117 / 26200.8117) resolves the hang, then close as upstream-fixed. If users on patched Windows still reproduce, escalate back to the Electron/Windows teams. The Electron-side fix (moving the COM call off the UI thread) already shipped via the electron@39.2.7 bump in VS Code.

## Evidence

- **root cause (team diagnosis):** deepak1556 confirmed via ETW trace that the Windows toast notification service becomes unresponsive and blocks Electron's UI thread via a synchronous COM call: https://github.com/microsoft/vscode/issues/275010#issuecomment-3612854954
- **Electron fix:** Electron commit https://github.com/electron/electron/commit/90e338df507f7dc7ef0cac1a09b62d2c97c95f72 moves the notification COM request off the UI thread. Shipped to VS Code via merged PR https://github.com/microsoft/vscode/pull/283786 (electron@39.2.7).
- **Windows OS fix:** KB5086672 (March 31, 2026) for OS builds 26100.8117 and 26200.8117 addresses the toast notification service hang: https://support.microsoft.com/en-us/topic/march-31-2026-kb5086672-os-builds-26200-8117-and-26100-8117-out-of-band-45cc1666-b34f-4ea6-bc93-f67defff8c38
- **workaround:** Disabling notification settings (chat.notifyWindowOnConfirmation, chat.notifyWindowOnResponseReceived, task.notifyWindowOnTaskCompletion) avoids the freeze.
- **awaiting confirmation:** deepak1556's latest comment (May 8, 2026) asks users to update Windows and confirm — no user has responded yet.
