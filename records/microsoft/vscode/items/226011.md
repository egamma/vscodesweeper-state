# Issue #226011 — Git - integration displays unhelpful error message when pushing large file

[https://github.com/microsoft/vscode/issues/226011](https://github.com/microsoft/vscode/issues/226011)

## Summary

Still reproduces on main: pushing a file exceeding the remote's size limit falls through to the generic PushRejected handler, showing the misleading 'Try running Pull first' message. An open PR (#304798) is attempting to fix this.

## Change summary

Pushing a large file (>100MB on GitHub) shows a misleading error suggesting the user pull first, instead of indicating the file size limit was exceeded.

## Best solution

Review and land the open PR https://github.com/microsoft/vscode/pull/304798, which adds a FileSizeLimitExceeded error code and detects large-file patterns in stderr before the generic PushRejected fallback in `extensions/git/src/git.ts` and `extensions/git/src/commands.ts`.
