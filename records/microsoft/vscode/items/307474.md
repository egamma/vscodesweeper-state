# Issue #307474 — NAV Process Tree memory issue

[https://github.com/microsoft/vscode/issues/307474](https://github.com/microsoft/vscode/issues/307474)

## Summary

The memory consumption is caused by the AL Language extension's own language server process (Microsoft.Dynamics.Nav.EditorServices.Host), not by VS Code core. The reporter's own process list confirms the heavy process is a child of the extension host, spawned by ms-dynamics-smb.al. This belongs in the extension's issue tracker.

## Change summary

Microsoft.Dynamics.Nav.EditorServices.Host processes from the AL Language extension consume excessive memory (20–35 GB).

## Best solution

Report to the AL Language extension repository (https://github.com/microsoft/AL) where the same issue is already tracked as https://github.com/microsoft/AL/issues/5354. VS Code core has no control over an extension's language server memory behavior.
