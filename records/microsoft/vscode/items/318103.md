# Issue #318103 — Agents Window: support extension development workflow (extensionDevelopmentPath + debugger)

[https://github.com/microsoft/vscode/issues/318103](https://github.com/microsoft/vscode/issues/318103)

## Summary

Valid feature request: the extension development workflow (extensionDevelopmentPath + debugger) has no support for the Agents Window. The gap is confirmed in the source — the extensionHost debug adapter and the Agents Window launch path don't integrate. Keeping open for the team to address as the Agents Window matures.

## Change summary

Support launching the Agents Window as the Extension Development Host with debugger attached, and propagate extensionDevelopmentPath to the Agents Window's extension host.

## Best solution

Wire up the `--agents` flag in the extensionHost debug adapter (src/vs/workbench/contrib/debug/browser/extensionHostDebugService.ts) so it opens the Agents Window instead of a regular editor, and propagate the extensionDevelopmentPath from the parent host when the Agents Window is spawned from an Extension Development Host (src/vs/sessions/electron-browser/sessions.ts). This would unblock extension authors targeting the Agents Window.
