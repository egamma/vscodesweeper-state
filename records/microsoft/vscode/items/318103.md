# Issue #318103 — Agents Window: support extension development workflow (extensionDevelopmentPath + debugger)

[https://github.com/microsoft/vscode/issues/318103](https://github.com/microsoft/vscode/issues/318103)

## Summary

Valid feature request for extension development workflow support in the Agents Window. No implementation exists on current main — the extensionHost debug type doesn't handle the agents window, and the sessions layer only minimally references extensionDevelopmentPath (for devtools keybindings, not extension loading). Already correctly labeled and milestoned.

## Change summary

Support launching an Agents Window as an Extension Development Host with debugger attached and extensions loaded from extensionDevelopmentPath.

## Best solution

Extend the extensionHost debug adapter (src/vs/workbench/contrib/debug/browser/extensionHostDebugService.ts) to recognize an --agents flag and open the sessions window instead of a regular editor window. Additionally, wire the sessions window's extension host to respect extensionDevelopmentPath when launched from a dev host. This is a core gap that blocks extension developers targeting the Agents Window.
