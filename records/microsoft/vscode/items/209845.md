# Issue #209845 — Diff editor: comparing moved code scrolls somewhere off

[https://github.com/microsoft/vscode/issues/209845](https://github.com/microsoft/vscode/issues/209845)

## Summary

Still appears present on main: the moved-text comparison scroll logic in diffEditorViewZones.ts adjusts padding and scroll offsets without first revealing the moved block's line range, which can cause the viewport to jump to an unrelated position.

## Change summary

Clicking 'Compare' on a moved code block in the diff editor scrolls the viewport to an unexpected position instead of revealing the moved code.

## Best solution

The scroll-alignment logic in `diffEditorViewZones.ts` (lines ~425–454) should reveal the moved block's start line into the viewport before applying the deltaOrigToMod padding adjustment, so both sides show the moved code rather than jumping off-screen.
