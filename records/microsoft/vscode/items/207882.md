# Issue #207882 — Git - Add an option to display untracked files like git's default behavior, -unormal

[https://github.com/microsoft/vscode/issues/207882](https://github.com/microsoft/vscode/issues/207882)

## Summary

Valid feature request — the git extension still hardcodes `-uall` or `-uno` on current main; there is no `-unormal` (git's default) option. The gap remains real and the referenced closed issues did not deliver this specific behavior.

## Change summary

Add a `git.untrackedChanges` option that uses git's default `-unormal` behavior, showing untracked directories as collapsed entries instead of listing every individual file.

## Best solution

Add a fourth enum value (e.g. `normal`) to the `git.untrackedChanges` setting in `extensions/git/package.json` and wire it to pass `-unormal` (or no `-u` flag) in `extensions/git/src/git.ts` → `getStatus()`. The trailing `/` on directory names from git output would need to be stripped. This is a small, localized change in the built-in git extension.
