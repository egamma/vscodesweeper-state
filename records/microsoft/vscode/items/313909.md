# Issue #313909 — Copilot Apply In Editor to New Untitled File Overwrites Existing Untitled File

[https://github.com/microsoft/vscode/issues/313909](https://github.com/microsoft/vscode/issues/313909)

## Summary

Still present on main: the 'Apply in Editor' → 'New untitled editor' flow constructs an untitled URI that can collide with an existing untitled file, causing the content to be applied to the wrong editor instead of a fresh one.

## Change summary

Choosing 'New untitled editor' in the Apply in Editor quick pick overwrites an existing untitled file instead of creating a new one.

## Best solution

In `codeBlockOperations.ts` line 253, the `newUntitledFile` case should not reuse `resource.path` for the untitled URI. It should either omit the path entirely or generate a unique one, similar to how 'Insert into New File' in `chatCodeblockActions.ts` (line 394) passes `resource: undefined` to create a genuinely new untitled editor.
