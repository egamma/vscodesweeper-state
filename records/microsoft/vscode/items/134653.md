# Issue #134653 — Deleting large files with the UI takes very long

[https://github.com/microsoft/vscode/issues/134653](https://github.com/microsoft/vscode/issues/134653)

## Summary

Milestone 'On Deck' is a scheduling decision — deferring to maintainer judgment. The core complaint (slow delete due to undo backup) has been partially mitigated: files >5MB already skip undo content backup (commit ff0785571f7, Nov 2020), and `explorer.enableUndo` (shipped via #117621, Jan 2022) lets users disable undo entirely. A remaining gap may exist for remote/SSH scenarios where even trash-based delete is slow for large files.

## Change summary

Deleting large files via the Explorer UI is very slow compared to CLI `rm`, especially over SSH remotes, because VS Code backs up file contents for undo.

## Best solution

Evaluate whether the existing mitigations (5MB undo-content cap and `explorer.enableUndo` setting) fully address the reported scenario. If the bottleneck is now the trash operation over remote SSH rather than the undo backup, consider optimizing remote file deletion or adding a size-based prompt offering a direct (non-trash) delete. The area owner should decide whether this still warrants further work or can be closed as addressed.
