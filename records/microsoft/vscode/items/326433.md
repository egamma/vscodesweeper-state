# Issue #326433 — Add an option to display Copilot-edited files in either List View or Tree View

[https://github.com/microsoft/vscode/issues/326433](https://github.com/microsoft/vscode/issues/326433)

## Summary

Valid feature request to add a list/tree view toggle for Copilot-edited files shown inline in the chat panel, similar to the SCM view's toggle. The Agents window's Changes view already supports this (ChangesViewMode.List / ChangesViewMode.Tree in src/vs/sessions/contrib/changes/), but the workbench chat panel's modified-files confirmation widget (chatModifiedFilesConfirmationSubPart.ts) renders a flat CollapsibleListPool only.

## Change summary

Add an option to display Copilot-edited files in either flat list or folder-structured tree view within the chat panel.

## Best solution

Extend the modified-files presentation in the chat panel (chatModifiedFilesConfirmationSubPart.ts / chatReferencesContentPart.ts) to support a tree view mode toggle, similar to how the Agents window Changes view already does it via ChangesViewMode in src/vs/sessions/contrib/changes/. Alternatively, consider this already addressed for users of the Agents window and close if the older inline chat editing surface is being deprecated.
