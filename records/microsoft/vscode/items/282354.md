## Summary

Team-authored issue tracking a UX problem where multiple MCP servers sharing the same identity provider create indistinguishable duplicate account entries. Hard keep-open applies (author is MEMBER).

## Change summary

Multiple MCP servers using the same OAuth identity provider cause duplicate, indistinguishable account entries in the accounts list.

## Best solution

Deduplicate or merge account entries when multiple resources share the same identity provider session, likely in the authentication account storage layer (src/vs/workbench/services/authentication/). The UX should show one account with info about which resources use it, rather than N identical-looking entries.
