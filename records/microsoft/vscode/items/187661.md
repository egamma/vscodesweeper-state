# Issue #187661 — C# Dev Kit: None of the ShellQuoting values work in all cases to run dotnet build command using ShellExecution

[https://github.com/microsoft/vscode/issues/187661](https://github.com/microsoft/vscode/issues/187661)

## Summary

Team-authored issue (MEMBER) — must stay open. The core problem (no single ShellQuoting value works across all shells for all path characters) remains structurally present despite recent quoting improvements in eaaeaefe435 and 781c214442a, which address double-quoting and cmd.exe wrapping but don't fully resolve the cross-shell gap for paths with parentheses + multiple spaces.

## Change summary

ShellExecution with ShellQuoting fails for dotnet build commands when paths contain parentheses (unquoted), multiple consecutive spaces (Strong quoting in PowerShell), or when the default terminal is cmd (Escape quoting).

## Best solution

Extend the task shell quoting logic in `terminalTaskSystem.ts` to handle the combination of parentheses and multiple-space paths across cmd.exe and PowerShell — likely by using Strong quoting with a PowerShell-specific fix to preserve whitespace runs, or by detecting the active shell and selecting the appropriate quoting strategy automatically.
