# Issue #173306 — Git - Close repository (Git) doesn't forget openened parent repository

[https://github.com/microsoft/vscode/issues/173306](https://github.com/microsoft/vscode/issues/173306)

## Summary

Still reproduces on main: closing a parent repository does not clear the globalState `parentRepository:` flag, so it reopens next session. The issue also carries the 'On Deck' milestone, which is a hard keep-open.

## Change summary

Closing a git parent repository via 'Close Repository' only persists the closure in workspace state (session-scoped), but the globalState flag that remembers the parent repo is never cleared, causing it to reopen on the next session.

## Best solution

In `Model.close()` (extensions/git/src/model.ts), when closing a repository that was opened as a parent repository (has a `parentRepository:${root}` globalState entry), clear that globalState flag so the repo is not re-remembered next session. Also consider adding a command to re-list discovered parent repositories for reopening, as the reporter requests.
