# Issue #201772 — Allow "Debug Console" editors

[https://github.com/microsoft/vscode/issues/201772](https://github.com/microsoft/vscode/issues/201772)

## Summary

Valid, well-supported feature request (34 upvotes, past the 20-vote Backlog threshold). The debug console is still implemented solely as a panel view (`repl.ts`), with no EditorInput/EditorPane registration — only the DisassemblyView has one. The requested capability remains unimplemented on current main.

## Change summary

Allow the Debug Console (REPL) to be opened as an editor pane so it can be floated into auxiliary windows, like terminals and output channels already can.

## Best solution

Implement an EditorInput and EditorPane for the Debug Console, similar to how the terminal editor (`TerminalEditorInput`/`TerminalEditor`) works, enabling it to open in the editor area and float into auxiliary windows. The REPL view in `src/vs/workbench/contrib/debug/browser/repl.ts` would need to support multiple instantiation and an editor-pane wrapper registered via `EditorPaneDescriptor` in `debug.contribution.ts`.
