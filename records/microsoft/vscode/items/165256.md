# Issue #165256 — tasks: empty arguments are not properly passed to the command

[https://github.com/microsoft/vscode/issues/165256](https://github.com/microsoft/vscode/issues/165256)

## Summary

Still reproduces on main: empty string arguments in task configurations are not properly quoted when building the shell command line, causing them to be silently dropped from the executed command.

## Change summary

Task args that are empty strings (or meant to represent empty shell arguments) get swallowed because they pass through command-line construction without quoting.

## Best solution

In `_buildShellCommandLine`, the `quoteIfNecessary` function should treat empty strings as needing strong quotes (e.g. return `["\"\"", true]` for an empty value), ensuring the shell receives the intended empty argument.
