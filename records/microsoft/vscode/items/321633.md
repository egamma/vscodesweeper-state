# Issue #321633 — vs code 1.124.2 breaks extensions

[https://github.com/microsoft/vscode/issues/321633](https://github.com/microsoft/vscode/issues/321633)

## Summary

The available evidence points to an exception thrown by the Trailing Spaces extension while handling a VS Code event, not to an actionable VS Code core bug. Without a reduced extension-independent API regression, this should be handled by the extension publisher.

## Change summary

After upgrading to VS Code 1.123/1.124, the third-party `shardulm94.trailing-spaces` extension no longer highlights or trims trailing whitespace on save.

## Best solution

The Trailing Spaces extension should update its event/save handling for current VS Code. VS Code core should only track this if someone provides a minimal extension or API-level repro showing VS Code broke a documented extension API contract.

## Evidence

- **extension-specific repro:** The steps require installing `shardulm94.trailing-spaces`, and the reported failure is that this extension no longer highlights/trims whitespace.
- **reported stack:** The Extension Host output begins with `ExtensionError: Error in extension shardulm94.trailing-spaces: FAILED to handle event`, directly identifying the failing extension.
- **source inspection:** `src/vs/workbench/api/common/extHost.api.impl.ts` wraps extension event listeners and emits `new ExtensionError(extension.identifier, err, 'FAILED to handle event')` only when the extension listener throws.
- **error classification:** `src/vs/platform/extensions/common/extensions.ts` defines `ExtensionError` as an error clearly from an extension, identified by its `ExtensionIdentifier`.
