# Issue #132058 — `debug.startDebugging()` can't find configuration from `.code-workspace` file

[https://github.com/microsoft/vscode/issues/132058](https://github.com/microsoft/vscode/issues/132058)

## Summary

Still a valid gap on current main: `debug.startDebugging()` resolves configurations only from the folder-level launch (`.vscode/launch.json`) and never falls back to the workspace-level launch (`.code-workspace`). The issue is correctly labeled as a feature-request in the debug area and has community interest.

## Change summary

The `debug.startDebugging()` API does not find debug configurations defined in `.code-workspace` files, only those in `.vscode/launch.json`.

## Best solution

In `mainThreadDebugService.ts` (`$startDebugging`), after `getLaunch(folderUri)` returns a launch that lacks the named configuration, fall back to `getWorkspaceLaunch()` and check its configurations. Similarly, in `debugService.ts` (`startDebugging`), when `launch.getConfiguration(configOrName)` returns nothing at line 377, try `configurationManager.getWorkspaceLaunch()?.getConfiguration(configOrName)` before throwing. The fix touches `src/vs/workbench/api/browser/mainThreadDebugService.ts` and/or `src/vs/workbench/contrib/debug/browser/debugService.ts`.
