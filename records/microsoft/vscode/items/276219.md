## Summary

Reasonable UX polish request: the smart commit dialog still uses generic Yes/Always/Never buttons on current main. Already labeled under-discussion in the Backlog — keeping open for maintainer judgment on whether to adopt more descriptive button labels.

## Change summary

The smart commit warning dialog uses generic Yes/Always/Never buttons instead of descriptive action labels like 'Stage All & Commit', making it harder to quickly parse.

## Best solution

Replace the Yes/Always/Never button labels in `extensions/git/src/commands.ts` (around line 2428–2430) with descriptive labels like 'Stage All & Commit' / 'Always Stage & Commit' / 'Don't Ask Again', following Microsoft's own dialog guidelines for specific action responses. The 'Never' behavior (silently disabling the commit button) could also benefit from a brief explanatory tooltip or mention of the setting name.
