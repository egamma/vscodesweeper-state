# Issue #123615 — [api][commands] Open new window and a file

[https://github.com/microsoft/vscode/issues/123615](https://github.com/microsoft/vscode/issues/123615)

## Summary

Team-authored feature request on the 'On Deck' milestone — must remain open for maintainer judgment. The request for passing extra arguments (e.g. files to open) to the `vscode.newWindow` command remains unimplemented.

## Change summary

Extensions need a way to open a new window on a remote authority and simultaneously open a file, without polluting recently-opened entries with synthetic remote authority strings.

## Best solution

Extend the `vscode.newWindow` command (or a new API surface) to accept additional arguments — files/notebooks to open, options like preserveFocus/preview, and custom extension-provided resolver data — so extensions don't have to encode extra state into the remoteAuthority URI.
