# Issue #123615 — [api][commands] Open new window and a file

[https://github.com/microsoft/vscode/issues/123615](https://github.com/microsoft/vscode/issues/123615)

## Summary

Team-authored feature request on the 'On Deck' milestone — must remain open for maintainer judgment. The request is for a cleaner API to open a new window on a remote authority with additional arguments (e.g. files to open), avoiding polluted recently-opened entries.

## Change summary

Extensions need a way to open a new remote window and a file without encoding extra data in the remoteAuthority URI, which creates spurious recently-opened entries and workspace trust prompts.

## Best solution

Extend the `vscode.newWindow` command (or add a new API) to accept structured options — files/notebooks to open, extra resolver arguments — so extensions like Azure ML don't have to smuggle data through the remote authority string. This likely touches `src/vs/workbench/api/browser/mainThreadWindow.ts` and the command registration in `src/vs/workbench/api/common/extHostCommands.ts`.
