# Issue #293750 — Plan with Gemin 3 Pro -> Implement with gpt-5.1-codex-mini doesn't work since `replace_string_in_file` is used

[https://github.com/microsoft/vscode/issues/293750](https://github.com/microsoft/vscode/issues/293750)

## Summary

Valid bug in the Copilot extension's prompt/history management: model-specific reminderInstructions (e.g. 'use replace_string_in_file') leak into the chat history and influence a different model when the user switches mid-session. The tool selection prompt should be scoped to the active model, not baked into persistent history.

## Change summary

Switching models mid-session causes the new model to follow stale tool-use instructions from the previous model's reminderInstructions in chat history.

## Best solution

The Copilot extension's prompt assembly should strip or re-derive model-specific reminderInstructions (tool preference prompts) per-turn based on the currently active model's capabilities, rather than leaving them as persistent chat history entries. The fix likely belongs in the prompt rendering layer under `extensions/copilot/src/extension/prompts/`.
