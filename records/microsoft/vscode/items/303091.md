# Issue #303091 — VS Code consistently crashes/closes when attempting to "Discard Changes" for a file via the Git Source Control view. Deleting the file directly from the explorer or disk works fine.

[https://github.com/microsoft/vscode/issues/303091](https://github.com/microsoft/vscode/issues/303091)

## Summary

The crash occurs in the native OS dialog path (Electron's showMessageBox) when Git's discard-changes confirmation is displayed. The workaround of setting `window.dialogStyle` to `custom` confirms the issue is in the native dialog layer, not in VS Code's TypeScript code. This is likely an Electron/Chromium/OS interaction bug on Windows. The reporter provided crash dumps; the issue needs a team member to analyze them and potentially file upstream.

## Change summary

VS Code crashes when clicking 'Discard Changes' in the Git Source Control view because the native modal confirmation dialog triggers a crash on Windows.

## Best solution

A team member should analyze the attached crash dump (comment 2) to identify the faulting module. If the crash is in Electron's native dialog code or a Windows shell DLL (e.g. explorerframe.dll), file upstream against Electron. In the meantime, the `window.dialogStyle: custom` workaround is effective. Consider whether the default dialog style should be `custom` on affected Electron/OS combinations, or whether this is the same root cause as https://github.com/microsoft/vscode/issues/325433 (explorerframe.dll crashes).
