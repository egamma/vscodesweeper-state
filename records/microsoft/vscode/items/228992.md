# Issue #228992 — Terminal message displays Gibberish upon creation by window.createTerminal() API randomly

[https://github.com/microsoft/vscode/issues/228992](https://github.com/microsoft/vscode/issues/228992)

## Summary

Intermittent rendering glitch when terminal initialText is written before the xterm canvas is fully ready. No fix commit found on main, but the report is thin on reproduction detail (20% random occurrence, macOS only, older version) and has zero community engagement.

## Change summary

Terminal message set via `window.createTerminal({ message })` randomly displays garbled text on initial render.

## Best solution

The race likely lives in the timing between `_writeInitialText` and xterm's DOM attachment/layout in `terminalInstance.ts`. A proper fix would ensure the initial text write occurs after the terminal canvas has completed its first layout pass, or force a re-render after writing. The issue needs confirmation on current builds before investing investigation time.
