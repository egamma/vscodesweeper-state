# Issue #324686 — Split terminal

[https://github.com/microsoft/vscode/issues/324686](https://github.com/microsoft/vscode/issues/324686)

## Summary

Genuine feature request to hide or collapse individual panes within a split terminal group to temporarily give more width to the active pane. No existing command covers this — the closest are resize pane (which still splits the space) and unsplit (which permanently moves the instance to a new tab). Keeping open for community traction.

## Change summary

Request the ability to temporarily hide individual panes in a split terminal to give more width to the focused pane.

## Best solution

A "maximize pane" or "toggle hide pane" command within a terminal split group (in `terminalGroup.ts` / `terminalActions.ts`) that temporarily collapses sibling panes and restores them on toggle. Alternatively, this could be an extension-candidate if the terminal API surface allows it. The existing resize-pane commands don't cover this because they can't fully collapse a pane. Worth evaluating whether the generic editor-group maximize behavior could be extended to terminal split panes.
