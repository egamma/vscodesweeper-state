# Issue #259160 — Cannot use access token for gemini

[https://github.com/microsoft/vscode/issues/259160](https://github.com/microsoft/vscode/issues/259160)

## Summary

The Gemini BYOK integration intentionally supports API keys only, not OAuth access tokens from the Gemini extension's credential store. A team member confirmed this is the current design and suggested the Gemini extension should manage its own models for richer auth flows.

## Change summary

User wants to authenticate with Gemini models in Copilot Chat using an OAuth access token from the Gemini extension's credentials, rather than an API key.

## Best solution

Keep the current API-key-only design for Gemini BYOK in Copilot Chat. OAuth/credential-based auth for Gemini models is better handled by the Gemini extension itself registering as a language model provider, not by Copilot Chat reimplementing Google's auth flows.
