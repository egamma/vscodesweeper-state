# Issue #173602 — Graceful disconnect to remote Management Host

[https://github.com/microsoft/vscode/issues/173602](https://github.com/microsoft/vscode/issues/173602)

## Summary

This appears implemented on current main: shutdown now awaits a graceful remote-agent disconnect before stopping extension hosts, so the management connection has a chance to send and drain its disconnect message.

## Change summary

Ensure a remote client shutdown sends the disconnect message to the remote Management Host before the window closes.

## Best solution

Treat this as completed by the graceful remote disconnect work in commit cacaccb42fc85c8622c444c6ff4da2f1e8c3211a, with the follow-up remote extension host shutdown work in 7e805145f76dea04d774cb14b7bc85366c02e79d.

## Evidence

- **fix commit:** Commit cacaccb42fc85c8622c444c6ff4da2f1e8c3211a (https://github.com/microsoft/vscode/pull/218972) added graceful remote disconnect on shutdown: AbstractExtensionService joins shutdown with remoteAgentService.endConnection(), and AbstractRemoteAgentService.endConnection() sends protocol.sendDisconnect() and awaits protocol.drain().
- **current code:** src/vs/workbench/services/extensions/common/abstractExtensionService.ts currently awaits this._remoteAgentService.endConnection() before _doStopExtensionHosts(), with the comment that the management connection must send its disconnection message before the local extension host terminates the underlying tunnel.
- **follow-up:** Commit 7e805145f76dea04d774cb14b7bc85366c02e79d (https://github.com/microsoft/vscode/pull/223422) extended the same graceful disconnect pattern to the remote extension host by sending Terminate, sendDisconnect(), and awaiting drain().
