# Issue #237469 — Deleting a folder ending in a dot, actually deletes a different folder

[https://github.com/microsoft/vscode/issues/237469](https://github.com/microsoft/vscode/issues/237469)

## Summary

Still open on main: VS Code's file deletion path passes the raw filesystem path to Node.js APIs without guarding against Windows stripping trailing dots, so deleting `folder.` actually deletes `folder`. A team member has acknowledged it but hasn't confirmed on Windows yet. The `confirmation-pending` label is appropriate — keeping open.

## Change summary

Deleting a folder whose name ends with a dot on Windows silently deletes a different folder (the one without the trailing dot), causing data loss.

## Best solution

Add a Windows-specific guard in the file deletion (and possibly rename/move) code paths — either reject operations on paths with trailing dots, or use the `\\?\` extended-length path prefix to bypass Windows path normalization. The fix likely lives in `src/vs/platform/files/node/diskFileSystemProvider.ts` and possibly `src/vs/base/node/pfs.ts`. The team member's Windows confirmation should come first.
