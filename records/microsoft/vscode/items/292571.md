# Issue #292571 — Installing extension auto closes welcome page

[https://github.com/microsoft/vscode/issues/292571](https://github.com/microsoft/vscode/issues/292571)

## Summary

Team-authored bug, still reproduces on main. Line 588 of extensionsActions.ts calls `extensionsWorkbenchService.open()` during install, which replaces the active editor (e.g. the welcome page). A prior fix attempt (91cafa504) was reverted (a45b02bfda6), so the behavior persists. Keeping open per team-authored rule.

## Change summary

Installing an extension from the extensions sidebar replaces the currently active editor (e.g. the welcome page) with the extension details editor.

## Best solution

Modify the `extensionsWorkbenchService.open()` call at line 588 of `src/vs/workbench/contrib/extensions/browser/extensionsActions.ts` in `InstallAction.run()` to open the extension editor without replacing the active editor — e.g. by using `preserveFocus: true` or opening in a background tab. A previous attempt (commit 91cafa504) simply removed the `open` call but was reverted (a45b02bfda6), likely because the extension details editor is still wanted — it just shouldn't steal focus from the current editor. The same pattern at line 827 (`InstallInOtherServerAction`) should be addressed.
