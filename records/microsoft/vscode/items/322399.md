# Issue #322399 — [Error] unhandlederror-ModelService: Cannot add model because it already exists!

[https://github.com/microsoft/vscode/issues/322399](https://github.com/microsoft/vscode/issues/322399)

## Summary

Confirmed regression in 1.125: `loadModel()` in `outputChannelModel.ts` calls `modelService.createModel` without guarding against an already-existing model for the same URI. An open PR (https://github.com/microsoft/vscode/pull/322407) addresses this but has not merged — keeping open for that PR to land.

## Change summary

Opening an output channel can throw 'Cannot add model because it already exists' when `loadModel` is called while a model for the same URI is already registered.

## Best solution

Land or refine https://github.com/microsoft/vscode/pull/322407, which makes `loadModel` idempotent by checking for an existing model before calling `createModel`. The fix belongs in `src/vs/workbench/contrib/output/common/outputChannelModel.ts` around line 496.
