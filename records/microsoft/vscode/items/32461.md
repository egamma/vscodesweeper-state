# Issue #32461 — Cleanup ExtensionContext.workspaceState when a workspace no longer exists

[https://github.com/microsoft/vscode/issues/32461](https://github.com/microsoft/vscode/issues/32461)

## Summary

Team-authored feature request on the 'On Deck' milestone with 60 upvotes — keep open. The request for automatic or user-assisted cleanup of orphaned workspaceStorage remains unimplemented and has strong community demand.

## Change summary

Provide a mechanism to clean up ExtensionContext.workspaceState and workspace storage folders when the associated workspace no longer exists or is stale.

## Best solution

Implement a UI affordance (e.g. a management view or command) that lists workspace storage entries with their last-used date and disk usage, letting the user select which to remove. The heuristic-based auto-delete approach was ruled out early due to network/mount edge cases, so user-driven cleanup with size/staleness hints is the agreed direction (per the discussion between dbaeumer and bpasero).
