# Issue #32461 — Cleanup ExtensionContext.workspaceState when a workspace no longer exists

[https://github.com/microsoft/vscode/issues/32461](https://github.com/microsoft/vscode/issues/32461)

## Summary

Still valid: the existing `UnusedWorkspaceStorageDataCleaner` only removes storage for empty windows, not for real workspaces whose folders have been deleted. The requested UI for managing/cleaning stale workspace storage remains unimplemented. 61 upvotes and strong community demand support keeping this open.

## Change summary

Provide a way to clean up workspace storage data for workspaces whose folders no longer exist on disk.

## Best solution

Implement a user-facing UI (as agreed in comments 6/18/19) that lists workspace storage folders with their last-used date and disk usage, letting users select which to remove. The cleanup logic in `src/vs/code/electron-utility/sharedProcess/contrib/storageDataCleaner.ts` currently skips real workspaces entirely — extending it with an interactive prompt (rather than silent deletion) avoids the network-mount/VM false-positive problem discussed in early comments.
