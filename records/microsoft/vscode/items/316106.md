# Issue #316106 — Agent defaults to partial scope execution on multi-file operations, producing incomplete migrations, regressions, and wasted compute time

[https://github.com/microsoft/vscode/issues/316106](https://github.com/microsoft/vscode/issues/316106)

## Summary

The core complaint is about the Copilot agent's planning and execution strategy during multi-file operations — it processes a subset of targets rather than all. This is agent behavior quality, not a VS Code editor bug. The secondary concern about lost session history is a separate issue. Routing to agent-behavior for triage by the team that owns agent planning logic.

## Change summary

The Copilot agent should perform scope discovery, execute on all matching targets, and verify completion when asked to do multi-file operations like codebase-wide migrations.

## Best solution

The agent-behavior team should evaluate whether the agent's planning prompts can be improved to include a scope-discovery step (grep for all affected files), full execution across all targets, and a verification pass. This is fundamentally about agent prompting/orchestration quality in the Copilot extension, not a core VS Code code change. The session-history concern (chat history lost on panel close) is a separate issue and should be filed independently if not already tracked.
