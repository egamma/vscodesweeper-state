# Issue #240410 — `IConfigurationChangeEvent.affectsConfiguration` does not support `overrides` properly for default changes

[https://github.com/microsoft/vscode/issues/240410](https://github.com/microsoft/vscode/issues/240410)

## Summary

Still reproduces on main: `compareAndUpdateDefaultConfiguration` produces empty override key arrays when extension-contributed defaults update a language override that was already partially registered, causing `affectsConfiguration` with an override identifier to incorrectly return `false`.

## Change summary

When extensions contribute configuration defaults with language overrides (e.g. `[markdown]`), `IConfigurationChangeEvent.affectsConfiguration` with an `overrideIdentifier` returns `false` because the change event's override keys are empty.

## Best solution

Fix `compareAndUpdateDefaultConfiguration` in `src/vs/platform/configuration/common/configurationModels.ts` to properly compute override key diffs when the `keys` parameter is provided from the registry update — the comparison logic at lines 872–883 can produce empty arrays when both old and new models already share the same override keys but with different values.
