# Issue #145701 — [css] Inconsistent results when formatting range/selection

[https://github.com/microsoft/vscode/issues/145701](https://github.com/microsoft/vscode/issues/145701)

## Summary

Still open on main — the CSS range formatting delegates directly to vscode-css-languageservice's format() with the raw selection range, which can produce semantically incorrect results for ranges that don't align with syntax boundaries. No fix has landed.

## Change summary

CSS 'Format Selection' produces incorrect one-line output when the selection excludes the enclosing curly braces of a rule block.

## Best solution

The fix belongs in vscode-css-languageservice (upstream): its `format()` function should expand the given range to the nearest enclosing syntactic node before applying formatting rules, similar to how other language formatters handle partial selections.
