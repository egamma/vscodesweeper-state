# Issue #145701 — [css] Inconsistent results when formatting range/selection

[https://github.com/microsoft/vscode/issues/145701](https://github.com/microsoft/vscode/issues/145701)

## Summary

Still present on main. The CSS range formatting delegates directly to vscode-css-languageservice's `format()` with the user-selected range; the language service doesn't validate that the range aligns with a syntactic boundary, producing collapsed output for partial selections.

## Change summary

CSS "Format Selection" produces incorrect single-line output when the selected range doesn't align to a complete CSS block boundary.

## Best solution

Fix in vscode-css-languageservice: the `format()` method should expand or clamp the range to the nearest enclosing syntactic node before reformatting, ensuring partial-block selections don't collapse structure. Alternatively, the VS Code CSS server (cssServer.ts `onFormat`) could pre-expand the range to a valid CSS AST boundary before passing it downstream.
