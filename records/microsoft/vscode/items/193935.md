# Issue #193935 — Git commit message enforcement

[https://github.com/microsoft/vscode/issues/193935](https://github.com/microsoft/vscode/issues/193935)

## Summary

Long-standing UX feature request for commit button state management. A team member explained the current design rationale (secondary actions don't require a message). Low community traction (1 upvote) and already labeled under-discussion — leaving it in Backlog for the community process.

## Change summary

Disable the commit button (or provide clearer feedback) when no commit message has been entered, to avoid confusion when Git rejects an empty-message commit.

## Best solution

If addressed, add inline validation or a brief notification when a user attempts a commit action that requires a message but none is provided — rather than disabling the button entirely (which would break Commit Amend). The git extension's `commands.ts` commit flow would need a pre-check with user-facing feedback. Alternatively, this remains under-discussion until community demand grows.
