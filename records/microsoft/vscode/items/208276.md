# Issue #208276 — [linked rename] Linked Editing does't work well

[https://github.com/microsoft/vscode/issues/208276](https://github.com/microsoft/vscode/issues/208276)

## Summary

Still reproduces on current main. PR #242993 fixed the race-condition desync but the 'deleting entire tag name with attributes' variant persists — the word-pattern check in `_syncRanges` calls `clearRanges()` when the reference value becomes empty. An open PR #310383 targets this specific remaining issue, and a team member confirmed it still needs a fix.

## Change summary

Linked editing deletes the closing tag (instead of syncing it) when the opening tag name is fully removed while attributes are present.

## Best solution

Land or iterate on PR https://github.com/microsoft/vscode/pull/310383 which addresses the word-pattern guard in `_syncRanges` (src/vs/editor/contrib/linkedEditing/browser/linkedEditing.ts lines 194-200). The fix should allow the reference value to be empty without clearing the synced ranges, so that typing a new name still propagates to the closing tag.
