## Summary

Still appears to be unfixed on main: the save dialog's default path is not scoped per-session in the agents window, so switching sessions retains the previous session's folder. No fixing commit found.

## Change summary

The 'New file' save dialog in the agents window defaults to the previous session's folder instead of updating to the current session's workspace.
