# Issue #265510 — Submodule auto selected every time

[https://github.com/microsoft/vscode/issues/265510](https://github.com/microsoft/vscode/issues/265510)

## Summary

Still present on main: when a submodule repository's provider key isn't in the saved `previousState.all` array on reload, `onDidAddRepository` treats it as a brand-new repo and force-selects all repositories. The save/restore logic in `scmViewService.ts` has not been fixed for this scenario.

## Change summary

Submodule repositories are auto-selected (made visible) in the SCM view on every VS Code restart, ignoring the user's previous selection.

## Best solution

Fix the `onDidAddRepository` method in `src/vs/workbench/contrib/scm/browser/scmViewService.ts` — when a repository key is not found in `previousState.all` (line 364), it currently selects all repositories (lines 369–386). Instead, a new-but-not-previously-known repo should be added without force-selecting everything, or the provider storage key for submodules should be stabilized so it matches across sessions.
