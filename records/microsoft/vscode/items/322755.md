# Issue #322755 — [Error] [GitHub.copilot-chat] unhandlederror-t.with is not a function

[https://github.com/microsoft/vscode/issues/322755](https://github.com/microsoft/vscode/issues/322755)

## Summary

Confirmed bug in core: `DiagnosticCollection.delete()` fires the `onDidChangeDiagnostics` event without validating its `uri` argument (unlike `set()` which guards with `URI.isUri`), causing a TypeError when a non-URI value is passed. An open PR already targets this — keeping open for that PR to land.

## Change summary

Copilot Chat's diagnostics listener crashes with `TypeError: t.with is not a function` because `DiagnosticCollection.delete()` broadcasts an unvalidated URI in the change event.

## Best solution

Land the open fix PR (https://github.com/microsoft/vscode/pull/324184) which adds a URI guard to `delete()` in `src/vs/workbench/api/common/extHostDiagnostics.ts`, mirroring the existing `URI.isUri()` check in `set()`. The second error variant (null `.value`) noted in Comment 3 may need a separate guard in the Copilot extension's `diagnosticsChanged.ts`.
