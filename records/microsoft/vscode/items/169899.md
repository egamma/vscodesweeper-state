# Issue #169899 — Investigate performance of `menubar.updateOverflowAction`

[https://github.com/microsoft/vscode/issues/169899](https://github.com/microsoft/vscode/issues/169899)

## Summary

The performance concern remains valid on current main: `updateOverflowAction` still forces synchronous layout via `container.offsetWidth` and per-button `offsetWidth` reads (lines 484, 493 of menubar.ts). While the `update()` path already schedules via `scheduleAtNextAnimationFrame`, the underlying forced-layout pattern persists. Author is a CONTRIBUTOR (team member), so keeping open.

## Change summary

The `menubar.updateOverflowAction` method forces synchronous layout during window resize by reading `offsetWidth` on the container and each menu button element.

## Best solution

Adopt the measure/modify pattern (as jrieken suggested) or batch the offsetWidth reads into a single measurement pass to avoid forced synchronous layouts. The fix belongs in `src/vs/base/browser/ui/menu/menubar.ts` around lines 484–493.
