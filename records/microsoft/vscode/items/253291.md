# Issue #253291 — chat.editRequests should not be a user visible setting

[https://github.com/microsoft/vscode/issues/253291](https://github.com/microsoft/vscode/issues/253291)

## Summary

The setting `chat.editRequests` still exists on current main with four enum values (inline, hover, input, none). The request to remove it as a user-visible setting and default to inline-only remains unaddressed.

## Change summary

Remove the `chat.editRequests` user-visible setting and hardcode inline editing as the only mode.

## Best solution

Once the inline edit-requests feature is stable, remove the `chat.editRequests` configuration registration in `src/vs/workbench/contrib/chat/browser/chat.shared.contribution.ts`, hardcode the inline behavior, and clean up the enum-based branching in `chatListRenderer.ts`, `chatWidget.ts`, `chatExecuteActions.ts`, and `chatEditingActions.ts`.
