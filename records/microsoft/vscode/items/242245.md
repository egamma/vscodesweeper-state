## Summary

Team-authored issue (MEMBER) — must keep open. The bug appears to still exist on main: while `_activateAddedExtensionIfNeeded` handles `workspaceContains` checks for dynamically added extensions, there may be a timing/path issue where newly installed extensions don't go through this flow before the extension host is ready.

## Change summary

Newly installed extensions with `workspaceContains` activation events don't activate until the window is reloaded.

## Best solution

Ensure that the extension installation path (both UI-installed and server-startup) triggers `_activateAddedExtensionIfNeeded` with a proper `workspaceContains` check, likely in `abstractExtensionService.ts` or the extension management service's post-install flow.
