# Issue #149393 — Sort options directly in Explorer menu

[https://github.com/microsoft/vscode/issues/149393](https://github.com/microsoft/vscode/issues/149393)

## Summary

The underlying sort settings exist (including the reverse-sort option added in 2024), but the requested UI — a sort-order button/menu directly in the Explorer toolbar for quick access — has not been implemented. This reached 20 upvotes and was promoted to the Backlog; it remains a valid, unimplemented feature request.

## Change summary

Add a sort-order button or menu directly in the Explorer view toolbar so users can change sort attribute, direction, and options without navigating to Settings.

## Best solution

Add a toolbar action in the Explorer view header (similar to how other views expose filter/sort controls) that opens a dropdown or submenu with the existing sort-order settings (`explorer.sortOrder`, `explorer.sortOrderReverse`, `explorer.sortOrderLexicographicOptions`). The likely files are `src/vs/workbench/contrib/files/browser/views/explorerView.ts` for registering the toolbar action and `src/vs/workbench/contrib/files/browser/fileActions.contribution.ts` for menu contributions.
