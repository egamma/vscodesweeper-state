# Issue #288698 — Tool picker doesn't work properly when editing a prompt

[https://github.com/microsoft/vscode/issues/288698](https://github.com/microsoft/vscode/issues/288698)

## Summary

Still unfixed on main. The editing flow in chatWidget.ts restores message text, context, and model but does not properly sync tool picker changes back into the request when re-submitting an edited prompt.

## Change summary

Removing a tool via the tool picker while editing a previous prompt does not take effect when the prompt is re-submitted.

## Best solution

In the `clickedRequest`/editing flow in `src/vs/workbench/contrib/chat/browser/widget/chatWidget.ts`, restore the tool selection state from the edited request and ensure that tool picker modifications during editing are properly reflected in the `enabledTools` passed to `chatService.sendRequest`. The `chatSelectedTools.ts` model likely needs integration with the edit-and-resend path.
