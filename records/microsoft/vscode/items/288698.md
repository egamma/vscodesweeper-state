# Issue #288698 — Tool picker doesn't work properly when editing a prompt

[https://github.com/microsoft/vscode/issues/288698](https://github.com/microsoft/vscode/issues/288698)

## Summary

Still reproduces on main: when editing a previous prompt's tool selection (e.g. removing `execute`), the tool changes are lost because `finishedEditing()` disposes the inline input before `getModeRequestOptions()` captures the updated tool selection in `_acceptInput`.

## Change summary

When editing a previous chat prompt and modifying the tool picker (removing/adding tools), the tool changes are ignored on re-submission — the original tools are used instead.

## Best solution

In `_acceptInput` (chatWidget.ts), capture the mode request options (including `userSelectedTools`) from `this.input` *before* calling `finishedEditing()`, which disposes the inline input and switches `this.input` back to the bottom input part. Then use the captured options in the `sendRequest` call.
