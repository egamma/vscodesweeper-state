# Issue #299143 — Being able to select multi-root workspace folder in chat

[https://github.com/microsoft/vscode/issues/299143](https://github.com/microsoft/vscode/issues/299143)

## Summary

Valid feature request for a dedicated `#workspace-folder` chat context variable with autocomplete, distinct from existing file/folder completions and the agent-session folder picker. Currently in the Backlog Candidates community voting process with 0 upvotes — letting the 60-day window play out.

## Change summary

Request to reference multi-root workspace folder roots as first-class chat context variables (e.g. `#root-folder-name`) with autocomplete support, to scope the model's focus to a specific root.

## Best solution

If implemented, this would likely live in `src/vs/workbench/contrib/chat/browser/widget/input/editor/chatInputCompletions.ts` as a new completion category that surfaces workspace root folders (from `IWorkspaceContextService.getWorkspace().folders`) as attachable context items. The existing `addFileAndFolderEntries` already searches folders but at the file-picker level — a root-folder-scoped variable would be a higher-level concept. Alternatively, the existing folder picker for agent sessions (`OpenAgentHostFolderPickerAction`) could be extended to non-agent chat modes. Let the community process decide demand.
