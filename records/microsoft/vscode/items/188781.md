# Issue #188781 — Too many icons in the Search title area

[https://github.com/microsoft/vscode/issues/188781](https://github.com/microsoft/vscode/issues/188781)

## Summary

Valid polish request to declutter the Search view title bar by moving less-used actions (View as Tree/List) into an overflow menu. The related issue #228953 was closed as a duplicate of this one, making this the canonical tracker. The View as Tree/List actions are still registered inline (`group: 'navigation'` in `MenuId.ViewTitle`) on current main — not yet implemented.

## Change summary

Move less-popular Search view title bar actions (View as Tree/List toggle) into an overflow ('...') menu to reduce icon clutter.

## Best solution

Move the View as Tree/List toggle from the inline `navigation` group to a non-inline group in `MenuId.ViewTitle` in `src/vs/workbench/contrib/search/browser/searchActionsTopBar.ts`, following the pattern used in Terminal and Explorer views. Telemetry data from 2023 already supports this — the toggle had ~72K uses vs. 2.2M for Collapse.
