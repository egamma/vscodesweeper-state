# Issue #188781 — Too many icons in the Search title area

[https://github.com/microsoft/vscode/issues/188781](https://github.com/microsoft/vscode/issues/188781)

## Summary

Still valid: all search view title-bar actions remain in the `navigation` group on current main — none have been moved to an overflow menu. Issue #228953 was closed as a duplicate of this one, consolidating the request here.

## Change summary

Move less-popular search view toolbar icons (e.g. View as Tree/List toggle) into a '...' overflow menu to reduce visual clutter.

## Best solution

Move the View as Tree/List toggle (and possibly Refresh and Open New Search Editor) out of the `navigation` group into the default overflow in `src/vs/workbench/contrib/search/browser/searchActionsTopBar.ts`, following the pattern the Explorer already uses.
