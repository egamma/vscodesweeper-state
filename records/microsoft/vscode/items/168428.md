# Issue #168428 — Consider a sticky scroll provider API

[https://github.com/microsoft/vscode/issues/168428](https://github.com/microsoft/vscode/issues/168428)

## Summary

Valid, popular feature request (48 upvotes, well above the 20-upvote Backlog threshold) with no implementation yet — the extension API has no dedicated sticky scroll provider. Keep open.

## Change summary

Extensions need a dedicated sticky scroll provider API to control which lines are shown, rather than relying on DocumentSymbolProvider or folding information.

## Best solution

Introduce a proposed `DocumentStickyScrollProvider` API in `src/vscode-dts/vscode.proposed.*.d.ts` that lets extensions supply custom sticky line ranges and display text, decoupled from DocumentSymbolProvider. The internal plumbing already exists in `src/vs/editor/contrib/stickyScroll/browser/stickyScrollProvider.ts` (`IStickyLineCandidateProvider`), which could serve as the foundation.
