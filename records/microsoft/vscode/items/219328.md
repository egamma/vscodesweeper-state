# Issue #219328 — Recognize paths with ~ on Windows

[https://github.com/microsoft/vscode/issues/219328](https://github.com/microsoft/vscode/issues/219328)

## Summary

Valid feature request to expand ~ on Windows CLI paths. No implementation exists on current main — the CLI path resolution does not preprocess tilde. Keeping open in Backlog as an unaddressed gap.

## Change summary

VS Code's Windows CLI should resolve ~ to the user's home directory when paths are passed as arguments.

## Best solution

Add tilde expansion in the CLI argument parsing (likely in `src/vs/platform/environment/node/argvHelper.ts` or the path resolution layer) so that `code ~\.ssh` resolves to the user's home directory on Windows, matching shell behavior on macOS/Linux.
