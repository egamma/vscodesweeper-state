# Issue #304443 — copilot-cli sessions trigger save right on startup

[https://github.com/microsoft/vscode/issues/304443](https://github.com/microsoft/vscode/issues/304443)

## Summary

Hard keep-open: carries `important` label and milestone `On Deck`. The bug is real and unfixed on main — `_doAccept` unconditionally triggers save participants even during session restore.

## Change summary

Restoring a stale copilot-cli session at startup triggers unnecessary file saves and save participants (e.g. ESLint) for worktree files.

## Best solution

Guard `_doAccept` in `chatEditingModifiedDocumentEntry.ts` (or the caller in the remote-session restore path) to skip save participants when restoring/replaying a session rather than performing a live accept. The restore codepath should pass `skipSaveParticipants: true` similar to the discard path at line 299.
