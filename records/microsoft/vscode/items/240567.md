# Issue #240567 — Editor setting `editor.tabSize` does not apply on empty files after startup

[https://github.com/microsoft/vscode/issues/240567](https://github.com/microsoft/vscode/issues/240567)

## Summary

Still open on main: the workaround (hardcoding known language defaults in core modesRegistry.ts) is in place, but the underlying issue — that extension-contributed tabSize/insertSpaces defaults aren't available early enough at startup and can't be dynamically applied to an already-created model — remains unfixed.

## Change summary

editor.tabSize from extension-contributed configuration defaults does not apply to empty files opened before extensions activate after a reload.

## Best solution

Either make the editor model's tabSize/insertSpaces dynamically updatable after creation (in editorConfiguration.ts / EditorOption), or ensure the configuration cache for extension defaults is reliably warm before editors open. The workaround only covers a handful of hardcoded languages; all other extension-contributed overrides still regress.
