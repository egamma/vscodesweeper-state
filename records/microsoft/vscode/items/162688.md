# Issue #162688 — Code action widget status bar border does not match editor hovers

[https://github.com/microsoft/vscode/issues/162688](https://github.com/microsoft/vscode/issues/162688)

## Summary

Still present on main: the code action widget (actionWidget.css) uses full-opacity `--vscode-editorHoverWidget-border` for its separators and status bar border, while the editor hover applies `.transparent(0.5)` to the same color via JS. Author is a MEMBER — keeping open for maintainer judgment.

## Change summary

The code action widget's internal separator/status-bar border uses the full editorHoverWidget border color, whereas the editor hover uses a 50%-transparent version, creating a visual inconsistency.

## Best solution

Either introduce a dedicated `editorHoverWidget.separatorBorder` theme color (as the author suggested in comment 1), or apply the same `.transparent(0.5)` treatment in `src/vs/platform/actionWidget/browser/actionWidget.css` lines that reference `--vscode-editorHoverWidget-border` for separator/status-bar borders. The fix is small and localized to actionWidget.css and possibly hoverContribution.ts / hoverService.ts.
