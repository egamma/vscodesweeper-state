# Issue #284495 — Remember Copilot Results expand state

[https://github.com/microsoft/vscode/issues/284495](https://github.com/microsoft/vscode/issues/284495)

## Summary

Valid feature request from a team member. The AI search results node is force-expanded on every search completion without remembering user preference. Hard keep-open: author is MEMBER.

## Change summary

Remember the user's expand/collapse state of the Copilot Results node in Search across searches, instead of force-expanding it every time.

## Best solution

Track the user's collapse preference for the AI results heading (e.g. a boolean flag on the search view or AI heading model) and respect it in the post-search expansion logic in `searchView.ts` around line 1826. Only auto-expand when the user hasn't explicitly collapsed it.
