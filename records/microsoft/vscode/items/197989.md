# Issue #197989 — Workspace Trust - Untrusted files don't open in the same window via Open Recent

[https://github.com/microsoft/vscode/issues/197989](https://github.com/microsoft/vscode/issues/197989)

## Summary

Still reproducible on main: 'Open Recent' for files uses OpenContext.API which on Windows/Linux defaults to openFilesInNewWindow=true, bypassing the user's workspace trust 'untrustedFiles: open' setting that should keep untrusted files in the same window. No fix has been applied.

## Change summary

Opening an untrusted file via 'Open Recent' opens a new window instead of reusing the current untrusted window, despite the trust setting being set to 'open'.

## Best solution

The window-opening path for 'Open Recent' files (OpenContext.API in windowsMainService.ts) needs to consult the workspace trust 'untrustedFiles' setting before deciding to open in a new window. The fix likely involves passing trust-aware context from the renderer or checking the trust setting in the main process shouldOpenNewWindow logic. Files: src/vs/platform/windows/electron-main/windowsMainService.ts (shouldOpenNewWindow), src/vs/workbench/browser/actions/windowActions.ts (OpenRecentAction).
