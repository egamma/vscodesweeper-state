## Change summary

Minimizing and restoring a maximized VS Code window on a mixed-DPI multi-monitor Windows setup causes the window to stretch across monitors.

## Best solution

This is an Electron/Chromium window-management bug with mixed-DPI monitors on Windows. The fix belongs upstream in Electron's `BrowserWindow` restore logic. Keep open as an upstream tracking issue; if Electron resolves the underlying DPI-aware restore, a VS Code Electron update will pick it up. Related but distinct from https://github.com/microsoft/vscode/issues/284296 (overflow) and https://github.com/microsoft/vscode/issues/202517 (sleep restore).
