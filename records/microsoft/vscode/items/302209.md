# Issue #302209 — Not seeing `github.copilot.chat.agentDebugLog.enabled` after chat extension update until after restart

[https://github.com/microsoft/vscode/issues/302209](https://github.com/microsoft/vscode/issues/302209)

## Summary

Team-authored bug — the settings editor does not pick up new configuration schema contributed by an extension that was updated via 'reload extensions' (without a full window reload). The settings editor listens for install/uninstall events but not for extension-host restarts that bring new configuration contributions.

## Change summary

After updating an extension and reloading extensions (not a full VS Code reload), newly contributed settings don't appear in the Settings editor until a full window reload.

## Best solution

The settings editor (settingsEditor2.ts) should listen for configuration-schema changes triggered by extension-host restarts — likely by reacting to `onDidChangeConfiguration` with `source === ConfigurationTarget.DEFAULT` or by listening on a schema-change event from the configuration registry after the extension host re-registers contributions. The fix likely lives in `src/vs/workbench/contrib/preferences/browser/settingsEditor2.ts` and possibly in `src/vs/workbench/services/configuration/`.
