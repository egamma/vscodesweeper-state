# Issue #232258 — Powershell 7 Ctrl + G from Terminal fails for paths with space

[https://github.com/microsoft/vscode/issues/232258](https://github.com/microsoft/vscode/issues/232258)

## Summary

Still reproduces on main: `preparePathForShell` unconditionally prepends the PowerShell call operator `&` for paths with spaces, which is correct for invoking executables but breaks `cd` commands. The bug is confirmed, well-diagnosed, and the code path is unchanged.

## Change summary

Ctrl+G (Go to Directory) in the terminal produces an invalid `cd & 'path with spaces'` command on PowerShell because the path-escaping helper adds the call operator `&` unconditionally.

## Best solution

The fix is in `preparePathForShell` (`src/vs/workbench/contrib/terminal/common/terminalEnvironment.ts`): it needs a parameter or variant that controls whether to emit the `&` call operator, since the caller in `terminalRunRecentQuickPick.ts` uses the result as a `cd` argument (where `&` is wrong), not as an executable invocation. The quick-pick call site should request quoting-only escaping without the call operator.
