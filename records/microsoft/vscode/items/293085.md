# Issue #293085 — For the extended LanguageModelChatProvider, the context indicator is always 0.

[https://github.com/microsoft/vscode/issues/293085](https://github.com/microsoft/vscode/issues/293085)

## Summary

Confirmed real API gap: the LanguageModelChatProvider progress callback has no usage-part type, so third-party providers cannot report token counts to the context-usage widget. An open PR (#320550) adds LanguageModelResponseUsagePart to address this.

## Change summary

Extensions implementing LanguageModelChatProvider cannot report token usage back to VS Code, so the context-window indicator always shows 0 for custom models.

## Best solution

Land https://github.com/microsoft/vscode/pull/320550, which adds LanguageModelResponseUsagePart to the chatProvider proposed API and wires it through the ext-host bridge so usage data flows to the context-usage widget.
