# Issue #218102 — Profile/metadata feedback on testing

[https://github.com/microsoft/vscode/issues/218102](https://github.com/microsoft/vscode/issues/218102)

## Summary

Team-authored discussion issue about testing profile/metadata feedback, specifically around debug session restart not preserving test run context. Hard keep-open: author is MEMBER.

## Change summary

Restarting a debug session tied to a test run loses the test run association, so results aren't recorded and context is missing on re-run.

## Best solution

The testing team should resolve whether VS Code's restart flow should preserve the test run association (and any extension-provided context/parameters), or whether extensions need an API to re-supply that context. This lives in the testing contribution area and may require changes to how debug session restarts interact with `TestRunProfile`.
