# Issue #237345 — Git - remote name should be considered when checking out a branch

[https://github.com/microsoft/vscode/issues/237345](https://github.com/microsoft/vscode/issues/237345)

## Summary

Valid feature request: when checking out a remote branch via the Git extension, the local branch name is derived by stripping the remote prefix (git --track default). With multiple remotes sharing branch names, this causes conflicts. No setting or mechanism exists on current main to prepend the remote name to the local branch. Keeping open for area owner consideration.

## Change summary

Allow prepending the remote name to local branch names when checking out remote branches, to avoid conflicts when multiple remotes share the same branch names.

## Best solution

Add a setting (e.g. `git.checkoutRemotePrefix`) that, when enabled, creates local branches as `{remoteName}/{branchName}` instead of just `{branchName}` during remote branch checkout. The change would be localized to `CheckoutRemoteHeadItem.run()` in `extensions/git/src/commands.ts` and `checkoutTracking` in `extensions/git/src/repository.ts`, switching from `git checkout --track` to `git checkout -b {remote}/{branch} --track {remote}/{branch}`.
