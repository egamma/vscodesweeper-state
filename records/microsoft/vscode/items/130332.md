# Issue #130332 — Editor > Minimap: Size = "fit" setting incorrectly stretches minimap on small files after an integrated terminal panel is initiated and closed from a larger file.

[https://github.com/microsoft/vscode/issues/130332](https://github.com/microsoft/vscode/issues/130332)

## Summary

Keep open as a plausible editor minimap layout bug with concrete reproduction steps and a second user confirmation; I did not find a concrete fixing commit for this exact panel/tab-switching fit-vs-fill behavior on current main.

## Change summary

With `editor.minimap.size` set to `fit`, switching from a large file to a small file after opening and closing the integrated terminal can make the small file's minimap stretch like `fill`.

## Best solution

The editor minimap owner should verify this against current Insiders and, if it still reproduces, add a focused layout test around `EditorLayoutInfoComputer._computeMinimapLayout` for `fit` after editor height changes and model/view line count changes.
