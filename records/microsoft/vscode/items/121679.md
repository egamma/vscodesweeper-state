# Issue #121679 — Include options in Issue Reporter are not persistent

[https://github.com/microsoft/vscode/issues/121679](https://github.com/microsoft/vscode/issues/121679)

## Summary

Still reproduces on main: the Issue Reporter creates a fresh IssueReporterModel with all include options defaulting to true on every open, with no persistence to storage. The bug remains valid and unfixed.

## Change summary

Unchecking 'Include' options in the Issue Reporter is not remembered when the reporter is reopened.

## Best solution

Persist the include-option checkbox states (includeSystemInfo, includeProcessInfo, includeWorkspaceInfo, includeExtensions, includeExperiments, includeExtensionData) via IStorageService in the IssueReporterModel initialization, and restore them on reopen. The fix belongs in `src/vs/workbench/contrib/issue/browser/baseIssueReporterService.ts` and possibly `issueReporterModel.ts`.
