Team-authored UX feedback on the agents window changes view — three concrete issues with badge/notification behavior and empty-state messaging. Keeping open for maintainer judgment.

## Change summary

The agents window changes view shows confusing badge indicators that don't clear on file click, shows diff view for newly created files with nothing to diff against, and has an unhelpful empty state when no git repo is present.

## Best solution

Address the three UX gaps in the agents window changes contribution: (1) show created-file indicators instead of diff badges for new files, (2) clear the badge/notification after the user views a changed file, (3) improve the empty-state copy when no git repo is found.
