# Issue #249257 — Inline completion that just inserts tab is indistinguishable from a tab that is in the text buffer

[https://github.com/microsoft/vscode/issues/249257](https://github.com/microsoft/vscode/issues/249257)

## Summary

Still open and unfixed on main. The root cause is well-understood (Copilot re-suggests deleted whitespace as ghost text that's visually identical to buffer content), and hediet linked the related issue #229312 noting the fix belongs in the inline completions provider. Keeping open as under-discussion.

## Change summary

Ghost-text inline completions that consist only of whitespace (like a tab) are visually indistinguishable from actual buffer content, confusing users who think their edit didn't take effect.

## Best solution

The inline completions provider (Copilot side) should avoid suggesting completions that are purely whitespace when the user just deleted that whitespace, or the ghost-text renderer should visually distinguish whitespace-only suggestions more clearly (e.g. different styling). Related work tracked in https://github.com/microsoft/vscode/issues/229312.
