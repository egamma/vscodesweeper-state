# Issue #212696 — Port is being added twice to Microsoft dev tunnel in VsCode with absolute URLs

[https://github.com/microsoft/vscode/issues/212696](https://github.com/microsoft/vscode/issues/212696)

## Summary

The port-duplication in redirect URLs is caused by how the dev tunnels proxy rewrites HTTP Location headers — it doesn't strip the port from `localhost:3000` when mapping to the tunnel URL that already encodes the port in its subdomain. No fix has landed in VS Code's tunnel-forwarding extension or tunnel service code on main. Keeping open as an upstream/tunnel-service issue.

## Change summary

Dev tunnel URLs include the port twice (both in the subdomain and as :port) when an app issues a redirect using an absolute localhost URL.

## Best solution

The fix belongs in the dev tunnels proxy/CLI layer that rewrites HTTP redirect Location headers — it should strip the explicit port when the tunnel URL already encodes it in the subdomain. The VS Code tunnel-forwarding extension (extensions/tunnel-forwarding) merely launches the tunnel CLI and doesn't control header rewriting. This may need to be filed against the dev tunnels service itself (https://github.com/AzureAD/microsoft-authentication-library-for-js or the devtunnels CLI).
