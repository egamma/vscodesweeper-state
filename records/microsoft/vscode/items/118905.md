# Issue #118905 — Some emojis not displaying correctly in VSCode macOS

[https://github.com/microsoft/vscode/issues/118905](https://github.com/microsoft/vscode/issues/118905)

## Summary

Keep open as an upstream font-rendering issue: a VS Code maintainer already traced the editor rendering behavior to Chromium with Menlo on macOS, and the issue is already labeled `upstream`/`upstream-issue-linked`. Later comments indicate variants of the emoji rendering problem still occur, so there is not enough evidence for a completed or not-reproducible close.

## Change summary

Certain emoji variation-sequence characters render as monochrome, incorrectly sized, or inconsistently spaced in the editor and terminal on macOS.

## Best solution

Continue tracking the Chromium/Electron font fallback behavior for the editor via the linked upstream Chromium issue, while treating terminal width/spacing behavior separately through the terminal/xterm.js Unicode width handling if maintainers decide that part needs its own issue.
