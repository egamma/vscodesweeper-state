# Issue #118905 — Some emojis not displaying correctly in VSCode macOS

[https://github.com/microsoft/vscode/issues/118905](https://github.com/microsoft/vscode/issues/118905)

## Summary

Well-documented upstream Chromium issue where certain emoji characters (those requiring Variation Selector-16 fallback) render as monochrome glyphs when a programming font like Menlo contains non-emoji versions. Already correctly labeled as upstream with a linked Chromium bug. Should remain open as a tracker.

## Change summary

Emojis composed of a base character plus U+FE0F (Variation Selector-16) render as small black-and-white glyphs instead of color emoji in the editor and terminal on macOS when using fonts like Menlo.

## Best solution

This remains an upstream Chromium text-rendering issue (https://bugs.chromium.org/p/chromium/issues/detail?id=1190093). Keep open as a tracker. The workaround of adding 'Apple Color Emoji' to the font family or removing Menlo is documented in comments. A core fix would require Chromium to improve its font fallback for VS-16 sequences, or VS Code to inject a CSS @font-face override for emoji unicode ranges — the latter was prototyped in comment 7 but never landed.
