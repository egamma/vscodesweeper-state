# Issue #158724 — Have UI expose data in VariablePresentationHint

## Summary

Still unimplemented on main. The debug expression renderer applies CSS classes for `virtual` and `internal` presentation hints, but does not surface visibility, kind, or attributes in the hover tooltip. The request to enrich the variable hover with VariablePresentationHint data remains a valid gap.

## Change summary

Expose VariablePresentationHint fields (visibility, kind, attributes) in the Variables pane hover tooltip so debug adapter authors can surface access modifiers and other metadata to users.

## Best solution

Extend the hover in `src/vs/workbench/contrib/debug/browser/debugExpressionRenderer.ts` (around line 197) to include presentationHint fields (visibility, kind, attributes) when present on the variable, likely as additional lines in the rich hover content. This is a small, self-contained enhancement in the debug area.
