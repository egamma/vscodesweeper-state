# Issue #144513 — Remote extension fails to activate in web worker extension host 

[https://github.com/microsoft/vscode/issues/144513](https://github.com/microsoft/vscode/issues/144513)

## Summary

Still open on main: the web worker extension host fetches remote extension code via a URI that triggers CORS failures when the worker runs on a different origin (Codespaces/Gitpod). The proposed fix PR #144515 was closed unmerged, and no subsequent fix has landed. Milestone is 'On Deck' — keeping open per scheduling rules.

## Change summary

Remote extensions fail to activate in the web worker extension host due to CORS errors when the worker iframe runs on a different origin than the VS Code Server.

## Best solution

Implement the approach alexdima described in comment 6: for embedders like Codespaces where the worker runs cross-origin, proxy the fetch request back to the main thread (which shares the server's origin). The fix likely lives in `src/vs/workbench/api/worker/extHostExtensionService.ts` around the `_loadCommonJSModule` method, with a conditional path when the fetch would be cross-origin.
