## Change summary

Remote extensions running in the web worker extension host fail to activate in hosted web environments when extension entrypoint fetches cross origins.

## Best solution

Resolve this in the extension host/remote loading path by implementing an embedder-aware main-thread fetch or equivalent CORS-safe resource loading path for Codespaces/Gitpod-style web workers, likely around `src/vs/workbench/api/worker/extHostExtensionService.ts` and the `$asBrowserUri` flow.
