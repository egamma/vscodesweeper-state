## Summary

The logs and follow-up comments confirm this is a network/ISP connectivity issue, not a VS Code or WSL bug. The Copilot Chat output shows fetch failures reaching GitHub APIs from within WSL, and two commenters independently confirmed that using a VPN resolved it — one confirmed their ISP was the root cause.

## Change summary

GitHub Copilot Chat shows 'Language model unavailable' when used in a WSL remote session due to network fetch failures reaching GitHub APIs.

## Best solution

No VS Code change needed. The root cause is network connectivity between the WSL environment and GitHub's APIs, likely blocked or degraded by the user's ISP. Using a VPN (e.g. Cloudflare WARP) is the established workaround, and contacting the ISP resolved it for at least one commenter.
