# Issue #240968 — MSAL Brokering does not work with PAM

[https://github.com/microsoft/vscode/issues/240968](https://github.com/microsoft/vscode/issues/240968)

## Summary

Correctly triaged as upstream — the MSAL broker flow doesn't support PAM accounts that require a separate elevated browser session. A team member is already investigating with the Microsoft Identity team. Keeping open awaiting upstream guidance.

## Change summary

MSAL brokering prevents PAM users from authenticating via an external elevated browser, as the new flow opens an in-app popup tied to the Windows-signed-in account.

## Best solution

Await guidance from the Microsoft Identity team (TylerLeonhardt's action item). If MSAL cannot support this scenario, VS Code may need a fallback path that bypasses the broker for accounts not signed in to Windows, or a setting to force the classic browser-based auth flow.
