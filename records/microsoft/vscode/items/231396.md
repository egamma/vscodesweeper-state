# Issue #231396 — Bug on Keyboard shortcut `multiDiffEditor.goToFile`

[https://github.com/microsoft/vscode/issues/231396](https://github.com/microsoft/vscode/issues/231396)

## Summary

Still reproduces on main: GoToFileAction.run() blindly casts args[0] as URI without guarding against undefined; when triggered via keyboard shortcut (no args), downstream calls crash with 'Cannot read properties of undefined'. Already tracked by the collection issue #282733.

## Change summary

Using a keyboard shortcut for multiDiffEditor.goToFile throws a runtime error because the action doesn't handle missing URI args.

## Best solution

Guard the undefined case in GoToFileAction.run() by deriving the focused document URI from the active MultiDiffEditor widget (similar to how GoToNextChangeAction uses resolveCommandsContext) when args[0] is not provided. Already tracked by https://github.com/microsoft/vscode/issues/282733.
