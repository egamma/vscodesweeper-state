# Issue #241871 — Dragging from Firefox doesn't work

[https://github.com/microsoft/vscode/issues/241871](https://github.com/microsoft/vscode/issues/241871)

## Summary

Team-authored issue (MEMBER) — must stay open per policy. The bug appears still present on main: the drag-and-drop handler in chatDragAndDrop.ts has conditions on MIME types (line 284) that may not match Firefox's drag data format, and no fixing commit was found.

## Change summary

Dragging a URL/image from Firefox into the VS Code chat panel shows 'Attach URL as context' but the drop is silently ignored.

## Best solution

Investigate Firefox's drag event MIME type payloads (especially `text/uri-list`, `text/html`, `text/plain` availability) in `src/vs/workbench/contrib/chat/browser/widget/chatDragAndDrop.ts` around line 284 and in `extractUrlsFromDragEvent`. Firefox may not set the same combination of MIME types as Chrome/Electron, causing the condition `containsDragType(e, Mimes.uriList) && (containsDragType(e, Mimes.html) || containsDragType(e, Mimes.text))` to fail or the extracted data to be empty.
