## Summary

Confirmed bug with no fix found on main. The 'Reveal in Explorer View' command fails to properly reveal and select the file when the Explorer view has been moved to the Panel (middle column). The issue is already labeled `confirmed` and `file-explorer`; it should stay open for an area owner to address.

## Change summary

## Best solution

The `openView` call resolves the container location dynamically, but something in the composite opening or view activation path may not handle the Panel location correctly for the Explorer view. The fix likely lives in `fileCommands.ts` or the `viewsService.ts` open-view path.
