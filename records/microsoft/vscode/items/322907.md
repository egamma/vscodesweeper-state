# Issue #322907 — [Explorer/Outline] Add filter to show only exported members

[https://github.com/microsoft/vscode/issues/322907](https://github.com/microsoft/vscode/issues/322907)

## Summary

Valid feature request to filter Outline by export/visibility. Closely related to previously closed issues, but the core capability (SymbolTag for exported/public) was never implemented — only `Deprecated` exists today. The request spans language servers and the Outline view, requiring both an API extension and filter UI. Keeping open for community traction.

## Change summary

Add a filter to the Outline view to show only exported/public members, requiring new SymbolTag values and Outline filter support.

## Best solution

This requires two pieces: (1) extending `SymbolTag` with an `Exported`/`Public` tag so language extensions can annotate symbols, and (2) adding a filter toggle in the Outline view that uses that tag. As @jrieken noted in https://github.com/microsoft/vscode/issues/146937, the Outline itself doesn't know about export/visibility — that information must come from the language extension via the API. The closely related https://github.com/microsoft/vscode/issues/146937 was closed as `*out-of-scope`, and https://github.com/microsoft/vscode/issues/52202 was also closed. A maintainer should decide whether this specific framing (exported-only filter) warrants reopening the discussion or whether the prior `*out-of-scope` verdict still applies.
