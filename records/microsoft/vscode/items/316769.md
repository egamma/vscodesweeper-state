## Change summary

VS Code UI freezes after restoring from minimized when an external process rapidly creates/deletes files in the workspace.

## Best solution

Analyze the attached perf trace to identify whether the bottleneck is in the file watcher event coalescing (src/vs/platform/files/common/watcher.ts), the explorer tree refresh on focus-regain (src/vs/workbench/contrib/files/browser/explorerService.ts line 133), or the rendering pipeline. Consider throttling/coalescing queued file events more aggressively when the window is not visible, or deferring explorer tree updates until the window is restored.
