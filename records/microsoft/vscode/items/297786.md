# Issue #297786 — Git: revParse reads ref files from worktree gitdir instead of common gitdir, causing ENOENT loop in linked worktrees

[https://github.com/microsoft/vscode/issues/297786](https://github.com/microsoft/vscode/issues/297786)

## Summary

Confirmed on current main: `revParse` in `extensions/git/src/git.ts` reads ref files from `this.dotGit.path` (the worktree-specific gitdir) instead of `this.dotGit.commonPath` (the shared gitdir), causing ENOENT loops for remote refs in linked worktrees. An open PR exists to fix this.

## Change summary

In linked git worktrees, the Source Control panel spins indefinitely because `revParse` looks for shared refs (e.g. remote tracking branches) in the worktree-specific gitdir instead of the common gitdir.

## Best solution

Land https://github.com/microsoft/vscode/pull/304787, which should change line 3356 of `extensions/git/src/git.ts` to use `this.dotGit.commonPath ?? this.dotGit.path` — the same pattern already used at lines 3028 and 3120 in the same file.
