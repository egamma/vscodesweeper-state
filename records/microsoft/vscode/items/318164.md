# Issue #318164 — The hint in Copilot Chat contains invalid Markdown formatting in Traditional Chinese (Taiwan)

[https://github.com/microsoft/vscode/issues/318164](https://github.com/microsoft/vscode/issues/318164)

## Summary

Valid localization bug in the zh-tw translation of the chat tip prefix. The source string in chatTipService.ts is `"**Tip:** {0}"` — the zh-tw translation incorrectly places the colon inside the bold markers (`**提示：**`) instead of outside (`**提示**：`). This is a translation fix in the localization pack (vscode-loc), but the source string's structure arguably invites the error.

## Change summary

The Copilot Chat tip prefix renders broken Markdown bold in Traditional Chinese because the colon is inside the bold markers in the zh-tw translation.

## Best solution

Fix the zh-tw translation in the vscode-loc repository (microsoft/vscode-loc) so that `tipPrefix` reads `**提示**：{0}` instead of `**提示：**{0}`. Alternatively, refactor the source string in `src/vs/workbench/contrib/chat/browser/chatTipService.ts` to separate the bold and colon so translators cannot misplace the markers (e.g. use two localized segments or move the bold styling out of the localized string).
