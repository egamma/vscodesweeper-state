# Issue #127212 — Custom editors: implement proper untitled editor factory

[https://github.com/microsoft/vscode/issues/127212](https://github.com/microsoft/vscode/issues/127212)

## Summary

Both requested improvements remain unimplemented on current main: the _untitledCounter only increments (never decrements on editor close), and createUntitledEditorInput does not handle an associated resource (preserving path, dirty state, or save-without-prompt behavior). Keeping open as valid debt.

## Change summary

Custom editors should properly manage untitled editor counters (decrement on close) and support untitled editors with associated resources.

## Best solution

Implement counter tracking in `src/vs/workbench/contrib/customEditor/browser/customEditors.ts` so `_untitledCounter` reuses freed numbers, and extend `createUntitledEditorInput` to forward the associated resource URI and mark the editor dirty when one is provided. The notebook equivalent (issue #125028) may have patterns to follow.
