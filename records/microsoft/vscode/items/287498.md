# Issue #287498 — PHP autoindent behavior is broken (since 1.108)

[https://github.com/microsoft/vscode/issues/287498](https://github.com/microsoft/vscode/issues/287498)

## Summary

Still reproduces on main. The autoindent logic in cursorTypeEditOperations.ts falls through to bracket-based indentation only when indentationRules are absent; PHP has indentationRules that don't match the scenario, so the result is incorrect indentation on Enter. No fix has landed since the issue was filed.

## Change summary

Pressing Enter before existing text in PHP unindents/pushes lines incorrectly instead of preserving indentation, because indentationRules override bracket-based indentation even when they don't match.

## Best solution

Fix the fallback logic in `src/vs/editor/common/languages/autoIndent.ts` (around line 173) and/or `src/vs/editor/common/cursor/cursorTypeEditOperations.ts` (around line 818) so that when indentationRules exist but produce no match for the current line context, the bracket-based indentation still applies rather than returning a null/zero-indent result.
