# Issue #310714 — When updating VSC if one window has an ongoing copilot session only that window is re-opened when VSC restarts after update

[https://github.com/microsoft/vscode/issues/310714](https://github.com/microsoft/vscode/issues/310714)

## Summary

Team-authored bug report — keeping open per policy. The issue describes a real window-restore regression during the update flow when a Copilot session is active.

## Change summary

When updating VS Code with multiple windows open and one has a running Copilot task, only the window with the Copilot session is restored after restart — the other windows are lost.

## Best solution

The update-restart flow should preserve all open windows in the session state, not just the final window that was closed (the one with the Copilot task). The fix likely lives in the lifecycle/update handling code that persists window state before restart.
