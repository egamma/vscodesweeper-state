# Issue #146453 — Enable bracket colorization within Strings

[https://github.com/microsoft/vscode/issues/146453](https://github.com/microsoft/vscode/issues/146453)

## Summary

Valid feature request with 29 upvotes (above the 20-upvote Backlog threshold). The bracket colorizer intentionally skips non-`Other` token types (strings, comments, regex) by design. Adding an option to colorize brackets within strings would require a setting to bypass this filter.

## Change summary

Request an option to enable bracket pair colorization inside string tokens (e.g., template literals, snippets).

## Best solution

Add a setting (e.g., `editor.bracketPairColorization.colorizeInStrings`) that relaxes the `isOther` guard in `src/vs/editor/common/model/bracketPairsTextModelPart/bracketPairsTree/tokenizer.ts` line 208, allowing bracket pairs to be detected and colorized within string tokens. The feature is well-scoped and community-validated.
