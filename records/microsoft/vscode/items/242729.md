# Issue #242729 — `zsh` `shellIntegration.env` is undefined in earlier .onDidChangeShellIntegration

[https://github.com/microsoft/vscode/issues/242729](https://github.com/microsoft/vscode/issues/242729)

## Summary

Team-filed debt issue documenting a real timing problem: `shellIntegration.env` is undefined in early `onDidChangeShellIntegration` events for zsh, causing commands relying on PATH to fail. Filed by a team member, confirmed by another — keeping open for the team to address the lifecycle ordering.

## Change summary

In zsh, `shellIntegration.env` is undefined during early `onDidChangeShellIntegration` firings, causing consumers that rely on the environment (e.g. terminal suggest) to fail.

## Best solution

Ensure the shell integration env is populated before (or concurrently with) the first `onDidChangeShellIntegration` event, or document/enforce the contract that consumers must wait for a later event where `env` is defined. The fix likely lives in the zsh shell integration script (`shellIntegration-rc.zsh`) and/or the TypeScript shell integration event pipeline under `src/vs/workbench/contrib/terminal/`.
