# Issue #283423 — Ctrl+drag to copy editor tab between windows no longer works in 1.107.0

[https://github.com/microsoft/vscode/issues/283423](https://github.com/microsoft/vscode/issues/283423)

## Summary

Confirmed regression from the Electron 39 update (commit f2531f1c562). The VS Code copy-on-Ctrl-drag logic in editorDropTarget.ts is intact and correct — it checks e.ctrlKey properly — but Electron/Chromium appears to have changed how modifier keys are propagated in cross-window drag events. No fix has landed on main. Already correctly labeled as upstream/Electron regression.

## Change summary

Ctrl+drag to copy an editor tab between windows stopped working after the Electron 39 update in VS Code 1.107.0; the tab is moved instead of copied.

## Best solution

Track upstream with the Electron/Chromium team — the root cause is in how Electron 39 propagates modifier key state during cross-window drag-and-drop events. VS Code's isCopyOperation() logic (src/vs/workbench/browser/parts/editor/editorDropTarget.ts:373) is correct. If Electron cannot fix this, a workaround (e.g. capturing ctrlKey at drag start and carrying it through the transfer data) could be explored in editorDropTarget.ts and multiEditorTabsControl.ts.
