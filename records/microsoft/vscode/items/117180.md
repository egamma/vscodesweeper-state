# Issue #117180 — The initial position of the VSCode is shifted down when the native titlebar is on

[https://github.com/microsoft/vscode/issues/117180](https://github.com/microsoft/vscode/issues/117180)

## Summary

Confirmed upstream Electron bug on Linux with native titlebars — still present on main. An open PR (#315493) proposes a workaround. Keeping open since the PR has not merged.

## Change summary

On Linux with native titlebars, the window drifts downward on each restart because Electron's getPosition() returns content-area coordinates instead of frame coordinates.

## Best solution

Land the workaround in https://github.com/microsoft/vscode/pull/315493 (switching `getBounds()` in `windowImpl.ts` to delegate to `this._win.getBounds()` instead of composing from `getPosition()`/`getSize()`), and file or track the upstream Electron bug so the root cause is eventually fixed in Electron itself.
