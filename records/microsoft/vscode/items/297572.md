# Issue #297572 — Add Visual Distinction for Cross-Tab Search Results in Chat Customizations

[https://github.com/microsoft/vscode/issues/297572](https://github.com/microsoft/vscode/issues/297572)

## Summary

Valid polish request for the Chat Customizations editor: when searching, results from other tabs (Skills, Prompts, etc.) lack visual indicators of their origin. The Chat Customizations UI has seen significant development since filing but no evidence of this specific cross-tab search labeling being addressed. Keeping open as a reasonable UX improvement.

## Change summary

Add section headers or badges to cross-tab search results in the Chat Customizations panel so users can identify which tab (Skills, Prompts, etc.) each result belongs to.

## Best solution

Add a source-tab badge or group header to search results that originate from tabs other than the currently active one in the Chat Customizations editor. The relevant code lives in `src/vs/workbench/contrib/chat/browser/aiCustomization/` — likely `aiCustomizationManagementEditor.ts` (search orchestration) and the individual list widgets. The existing `CustomizationGroupHeaderRenderer` pattern could be reused to visually group cross-tab results by their source tab.
