# Issue #245355 — "Open Containing Folder" Fails Unless File Browser is Already Open

[https://github.com/microsoft/vscode/issues/245355](https://github.com/microsoft/vscode/issues/245355)

## Summary

The "Open Containing Folder" command delegates entirely to Electron's `shell.showItemInFolder()` (one-line call in nativeHostMainService.ts:645). The reported failure — file browser not opening unless already running — is an Electron/platform-level issue on certain Linux configurations, part of the broader family of Linux native file dialog problems tracked under the `native-file-dialog` label.

## Change summary

"Open Containing Folder" does not launch the system file browser on certain Linux setups (NixOS, Pop!_OS) unless a file browser window is already open.

## Best solution

This belongs upstream in Electron's `shell.showItemInFolder` implementation for Linux. The issue sits in the same family as https://github.com/microsoft/vscode/issues/231173 and https://github.com/microsoft/vscode/issues/243575 — all rooted in Electron's portal/xdg-open handling. Keep open under `native-file-dialog` as a data point; resolution depends on Electron fixes (see https://github.com/electron/electron/issues/43819 for upstream context).
