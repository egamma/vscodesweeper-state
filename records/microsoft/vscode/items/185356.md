# Issue #185356 — Use `ThemeIcon.color` for `QuickInputButton`

[https://github.com/microsoft/vscode/issues/185356](https://github.com/microsoft/vscode/issues/185356)

## Summary

Still unresolved on main: IQuickInputButton has no ThemeIcon support (only iconPath/iconClass), so ThemeIcon.color cannot be used for quick input buttons. A team member (dmitrivMS) actively picked this up in Oct 2025 but was unable to land it in the toggle PR — further changes to core quick pick item and button implementations are needed.

## Change summary

QuickInputButton should support ThemeIcon.color so extension authors can color button icons to reflect toggle state.

## Best solution

Plumb ThemeIcon (with its color property) through the QuickInputButton pipeline — from the extension host API conversion in extHostQuickOpen.ts through to the rendering in quickInputList.ts and quickInput.ts — rather than converting to plain CSS class strings. Related open issue https://github.com/microsoft/vscode/issues/308704 covers the broader QuickPick icon color gap.
