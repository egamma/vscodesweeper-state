# Issue #58980 — Relative problem paths don't take `cwd` into account

[https://github.com/microsoft/vscode/issues/58980](https://github.com/microsoft/vscode/issues/58980)

## Summary

Still valid and unimplemented on main — the problem matcher's filePrefix defaults to ${workspaceFolder} and never considers the task's cwd option. Team-authored issue in Backlog with 12 upvotes.

## Change summary

When a task specifies a custom `cwd`, the problem matcher should resolve relative file paths against that cwd rather than always using the workspace folder.

## Best solution

In `_resolveMatchers` (terminalTaskSystem.ts), when a problem matcher has no explicit `fileLocation` set by the user, default `filePrefix` to the resolved task cwd instead of `${workspaceFolder}`. This is a small change but potentially breaking for existing tasks, so it may need an opt-in or a new `fileLocation` value like `"taskCwd"`.
