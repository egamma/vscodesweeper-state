# Issue #325812 — Cannot set value in Watch by double click

[https://github.com/microsoft/vscode/issues/325812](https://github.com/microsoft/vscode/issues/325812)

## Summary

The Watch panel's double-click handler only supports Expression (top-level) and VisualizedExpression — Variable children (e.g. struct members) are not handled. Double-click on a child variable does nothing; 'Set Value' is only available via the right-click context menu. A prior feature request (#133268) for this was closed by the community process, and a team member raised design concerns about double-click ambiguity in list rows.

## Change summary

Double-clicking a child variable's value in the Watch panel should open an inline input box to set the value, but currently does nothing.

## Best solution

Extend the `onMouseDblClick` handler in `src/vs/workbench/contrib/debug/browser/watchExpressionsView.ts` (line ~210) to also handle `Variable` instances by calling `setSelectedExpression(element, true)` (with `settingWatch: true` to trigger value-setting mode). However, the prior team discussion in https://github.com/microsoft/vscode/issues/133268 raised concerns about double-click ambiguity in list rows — a maintainer should decide whether to adopt this behavior or keep the right-click-only approach.
