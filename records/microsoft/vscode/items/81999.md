# Issue #81999 — The result of proxyResolver.request is wrong

[https://github.com/microsoft/vscode/issues/81999](https://github.com/microsoft/vscode/issues/81999)

## Summary

The bug still exists in the current `@vscode/proxy-agent@0.32.0` shipped with VS Code: when a plain options object carries a `searchParams` property (spread from a `URL` instance), the proxy patch misidentifies it as a URL object, dropping `method` and other request options. The fix belongs in the upstream `vscode-proxy-agent` package. Author is a MEMBER, so keeping open.

## Change summary

The proxy resolver's `http(s).request` patch incorrectly treats a plain options object with a `searchParams` property as a URL object, causing the HTTP method and other options to be lost.

## Best solution

Fix the URL-detection heuristic in `@vscode/proxy-agent` (https://github.com/microsoft/vscode-proxy-agent) — the `patched()` function in `src/index.ts` should use `instanceof URL` instead of duck-typing via `searchParams`. The fix lives upstream in that package, then gets pulled into VS Code via a dependency bump.
