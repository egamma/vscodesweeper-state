## Summary

Valid feature request to suppress Copilot inline completions during merge conflict resolution. The requested setting doesn't exist today; the work likely belongs in the Copilot extension or could leverage existing editor context (conflict markers / merge editor) to conditionally disable completions.

## Change summary

Add a setting to automatically suppress Copilot inline completions when merge conflicts are active or the merge editor is open.

## Best solution

This likely belongs in the Copilot extension (`extensions/copilot/`) rather than VS Code core — the extension already controls when inline completions are provided and could check for conflict markers or the merge-editor context. If VS Code core needs to expose additional context (e.g., a context key for 'file has unresolved conflicts'), that would be a small core contribution feeding an extension-side decision. Routing to both the inline-completions and merge-editor areas for visibility.
