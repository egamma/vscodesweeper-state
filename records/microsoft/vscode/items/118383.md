# Issue #118383 — Can’t debug VS Code extensions on iPadOS PWA

[https://github.com/microsoft/vscode/issues/118383](https://github.com/microsoft/vscode/issues/118383)

## Summary

This is a genuine upstream platform limitation: iPadOS PWA (Add to Home Screen) does not support multiple tabs or windows, so the Extension Development Host window cannot open during debugging. The issue is already correctly labeled `upstream` and parked in Backlog. No code change on VS Code's side can work around the Safari PWA constraint, and no progress has been made upstream. Keeping open as a known platform limitation.

## Change summary

Debugging VS Code extensions on iPadOS PWA fails because the Extension Development Host window cannot open — PWA mode does not support multiple tabs/windows, so the new window replaces the original codespace.

## Best solution

This remains blocked on Safari/WebKit adding multi-window or multi-tab support for PWAs on iPadOS. No VS Code code change can fix this. Keep it open as a tracked upstream limitation so it can be revisited if Apple ships PWA windowing capabilities. If iPadOS PWA multi-window support lands, the debug flow in `src/vs/workbench/contrib/debug/` would need to be retested.
