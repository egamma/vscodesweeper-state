# Issue #270016 — Selected launch option changes when more than one source exists and either changes

[https://github.com/microsoft/vscode/issues/270016](https://github.com/microsoft/vscode/issues/270016)

## Summary

Still reproduces on main: when a launch configuration file changes and multiple launch sources exist, `selectConfiguration(undefined)` re-resolves the launch from the last active workspace root, which can differ from the currently selected launch source, causing the selection to reset to the first config of the wrong source.

## Change summary

Editing a launch configuration in a multi-source workspace (workspace file + folder launch.json) resets the debug dropdown selection to the first config of the folder instead of preserving the current selection.

## Best solution

In `registerListeners` (line 348–353 of `debugConfigurationManager.ts`), when `launch` config changes, preserve the current `selectedLaunch` and `selectedName` instead of calling `selectConfiguration(undefined)` — pass `this.selectedLaunch` and `this.selectedName` so the selection is only reset if the previously selected config no longer exists.
