# Issue #227121 — Editor GPU: Support editor.fontVariations

[https://github.com/microsoft/vscode/issues/227121](https://github.com/microsoft/vscode/issues/227121)

## Summary

Team-authored plan-item tracking a real gap: the GPU renderer's glyph rasterizer does not apply font-variation-settings. Keeping open per hard keep-open rule (CONTRIBUTOR author) and because it's a labeled plan-item.

## Change summary

The GPU-accelerated editor renderer should honor the editor.fontVariations setting when rasterizing glyphs.

## Best solution

In `src/vs/editor/browser/gpu/raster/glyphRasterizer.ts`, apply the resolved `fontVariations` value (from EditorOption.fontVariations) to the canvas 2D context's font string or via the CSS font-variation-settings equivalent when drawing glyphs. The tracked decorations in `viewGpuContext.ts` may also need a new entry for variation-settings similar to the existing `font-weight` handling.
