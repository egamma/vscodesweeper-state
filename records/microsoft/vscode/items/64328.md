# Issue #64328 — Wasted space in the multiline search input

[https://github.com/microsoft/vscode/issues/64328](https://github.com/microsoft/vscode/issues/64328)

## Summary

Team-authored UX improvement request for multiline search input layout — the design discussion (moving toggle buttons below when text wraps, à la Facebook Messenger) remains unimplemented. The related fix in commit 0aca34b47c5 only addressed a narrower AI-button padding issue (#209264), not the original multiline layout concern.

## Change summary

When the search input text wraps to multiple lines, the toggle buttons (case-sensitive, whole-words, regex) waste horizontal space on the first line, leaving an awkward gap.

## Best solution

Implement the design discussed in the comments: when text wraps past the first line, move the toggle controls below the textarea (or shift them to a second row), so the first line's full width is available for text. The relevant files are `src/vs/workbench/contrib/search/browser/searchFindInput.ts` and `src/vs/workbench/contrib/search/browser/media/searchview.css`. The area owner should decide on the final layout approach from the options Miguel and Rob discussed.
