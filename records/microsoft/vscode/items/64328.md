# Issue #64328 — Wasted space in the multiline search input

[https://github.com/microsoft/vscode/issues/64328](https://github.com/microsoft/vscode/issues/64328)

## Summary

Team-authored UX polish request for the multiline search input layout. The core issue — text wrapping around absolutely-positioned toggle controls, leaving dead space — remains unaddressed on main. The related fix (0aca34b) only removed extra AI button width, not the fundamental layout problem discussed here.

## Change summary

When the search input wraps to multiple lines, the toggle buttons (case sensitive, whole words, regex) are positioned top-right, leaving unused space below them.

## Best solution

Implement the layout approach discussed in comment 6/10: when text wraps, measure the first line at reduced width and after wrapping, measure at full width with controls on a separate line below. The changes would likely touch `src/vs/base/browser/ui/inputbox/inputBox.ts` (paddingRight/mirror logic) and `src/vs/workbench/contrib/search/browser/media/searchview.css`.
