# Issue #67397 — Support filter on type should in search tree

[https://github.com/microsoft/vscode/issues/67397](https://github.com/microsoft/vscode/issues/67397)

## Summary

Team-authored issue (MEMBER) requesting filter-on-type in the search tree — hard keep-open applies. The feature is still missing on current main: the search tree has no `keyboardNavigationLabelProvider` and type navigation remains disabled. A related open issue (https://github.com/microsoft/vscode/issues/197634) from another team member tracks the same gap; the two could be consolidated.

## Change summary

Enable filter-on-type (type navigation) in the search results tree so users can narrow results by typing file names.

## Best solution

Consider consolidating with https://github.com/microsoft/vscode/issues/197634 ("Search: Support tree type navigation and find widget"), which is the newer, cleaner statement of the same gap. The implementation would add a `keyboardNavigationLabelProvider` to the search tree in `src/vs/workbench/contrib/search/browser/searchView.ts`, returning labels for file-level nodes and `undefined` for text match nodes (per the original discussion) so only files participate in filtering.
