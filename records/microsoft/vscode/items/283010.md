# Issue #283010 — Improve codelens and selection/decoration behaviour

[https://github.com/microsoft/vscode/issues/283010](https://github.com/microsoft/vscode/issues/283010)

## Summary

Still present on main: codelens widgets are IContentWidgets rendered as separate DOM elements that don't participate in the editor's selection background highlighting. No fix has been applied since the issue was filed.

## Change summary

When selecting code that spans a codelens line, the codelens background color does not change to reflect the selection highlight.

## Best solution

The codelens content widget (or the view zone it occupies) needs to respond to the editor's selection range — when the selection spans the codelens line, apply the selection background color to the codelens DOM node. The fix likely lives in `src/vs/editor/contrib/codelens/browser/codelensWidget.ts` (the `CodeLensContentWidget` class) and possibly `codelensWidget.css`.
