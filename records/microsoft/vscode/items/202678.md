# Issue #202678 — resourceLabelFormatters include a leading backslash on Windows paths and stripPathStartingSeparator doesn't affect this

[https://github.com/microsoft/vscode/issues/202678](https://github.com/microsoft/vscode/issues/202678)

## Summary

Still reproduces on main: `stripPathStartingSeparator` compares the URI path's leading `/` against `formatting.separator` (which is `\` on Windows), so the strip never fires for Windows-style formatters. The bug is in `formatUri` in labelService.ts.

## Change summary

When a `resourceLabelFormatter` uses `separator: '\\'` on Windows, `stripPathStartingSeparator` fails to strip the leading path separator because URI paths always use `/` internally, but the check compares against the formatter's `\` separator.

## Best solution

In `formatUri`, the `stripPathStartingSeparator` check at line 470–471 should compare against `/` (the URI-internal separator) rather than — or in addition to — `formatting.separator`. This is a small fix in `src/vs/workbench/services/label/common/labelService.ts`. A test with `separator: '\\'` and `stripPathStartingSeparator: true` should be added to confirm.
