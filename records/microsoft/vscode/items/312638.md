# Issue #312638 — Installing plugin from custom marketplace fails as local git clone isn't updated

[https://github.com/microsoft/vscode/issues/312638](https://github.com/microsoft/vscode/issues/312638)

## Summary

Team-authored bug, keeping open. The install flow for relative-path plugins (from a registered marketplace repo) calls `ensureRepository` (clone-only) but never `pullRepository` before resolving the plugin directory, so newly added plugins on the remote aren't visible.

## Change summary

Installing a plugin from a custom marketplace fails when the local git clone is stale because the install flow doesn't pull before checking for the plugin directory.

## Best solution

In `_installRelativePathPlugin`, call `pullRepository` (or `updatePluginSource`) after `ensureRepository` succeeds so the local clone is up-to-date before resolving the plugin directory. The `IAgentPluginRepositoryService` already exposes `pullRepository` for this purpose.
