# Issue #185208 — Operating the mouse wheel in another window will cause the mouse wheel to operate abnormally when editing code

[https://github.com/microsoft/vscode/issues/185208](https://github.com/microsoft/vscode/issues/185208)

## Summary

The report describes mouse wheel events from another window accumulating and then being applied in VS Code, but the mechanism is unclear — VS Code should not receive wheel events when unfocused. No reproduction from others, no comments, very old Linux X11-specific report. Needs more information to verify.

## Change summary

Mouse wheel scrolling in another window causes unexpected large scroll jumps when returning to the VS Code editor.

## Best solution

Request the reporter to confirm on the latest build and clarify the exact window manager/compositor setup. If reproducible, this is likely an upstream Electron/Chromium issue on X11 where focus handling delivers stale wheel events. The VS Code scrollbar code (src/vs/base/browser/ui/scrollbar/scrollableElement.ts) handles events as they arrive — it cannot distinguish delayed delivery from fresh input.
