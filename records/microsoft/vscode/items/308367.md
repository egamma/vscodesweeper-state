# Issue #308367 — prompts-diagnostics-provider: false "File not found" errors for existing .md links in multi-root workspace

[https://github.com/microsoft/vscode/issues/308367](https://github.com/microsoft/vscode/issues/308367)

## Summary

The original multi-root path resolution complaint was unfounded (code already uses dirname of the document URI). However, the real bug — stale diagnostics not refreshing when referenced files are created/restored — is confirmed in the source: the ModelTracker only re-validates on content changes, tool changes, and model changes, but not on file system events or editor focus. Maintainer aeschli acknowledged this and suggested re-validating on focus.

## Change summary

Prompt file diagnostics report 'File not found' errors that persist after the referenced file is created or restored, because validation is not re-triggered on file system changes.

## Best solution

Add a file-system change listener (or an editor-focus listener as aeschli suggested) in `promptFileContributions.ts` that calls `validateAll()` when workspace files change, so stale 'File not found' diagnostics are cleared when referenced files appear.
