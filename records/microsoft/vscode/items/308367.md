# Issue #308367 — prompts-diagnostics-provider: false "File not found" errors for existing .md links in multi-root workspace

[https://github.com/microsoft/vscode/issues/308367](https://github.com/microsoft/vscode/issues/308367)

## Summary

The original multi-root false-positive was not reproduced, but the code confirms a real remaining issue: prompt diagnostics are never re-validated when referenced files are created/deleted on disk, only on content edits or tool/mode changes. A team member (aeschli) identified this and suggested re-validating on editor focus.

## Change summary

Prompt file diagnostics report stale 'File not found' errors when a referenced file is restored or created after the prompt file was opened, because no filesystem-change listener triggers re-validation.

## Best solution

Add a file-system watcher (or re-validate on editor focus as aeschli suggested) in `promptFileContributions.ts` so that `ModelTracker.validate()` is called when files in the workspace change, clearing stale 'File not found' markers.
