# Issue #306490 — bg terminal fails to start in other directory

[https://github.com/microsoft/vscode/issues/306490](https://github.com/microsoft/vscode/issues/306490)

## Summary

Team-authored issue reporting a dangerous but non-reproducible bug where a background terminal's `cd` prefix was incorrectly stripped, causing a command to run in the wrong directory. The root cause likely lives in the Copilot CLI SDK's command simplification logic. Still under investigation by the team.

## Change summary

Background terminal commands sometimes have their `cd` prefix stripped even when the terminal isn't already in the target directory, causing commands to execute in the wrong location.

## Best solution

Investigate whether background terminals should skip the `cd`-stripping logic entirely (as roblourens suggested in comment 5), or ensure the SDK's working directory comparison accounts for all edge cases. The presentation-only `getCdPresentationOverrides` in `extensions/copilot/src/extension/chatSessions/copilotcli/common/copilotCLITools.ts` is safe (display-only), but the actual execution-side stripping happens in the SDK. The fix likely needs to land in `@github/copilot/sdk`.
