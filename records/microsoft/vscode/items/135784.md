# Issue #135784 — [PWA] Register vscode.dev as file handler for well-supported file types

[https://github.com/microsoft/vscode/issues/135784](https://github.com/microsoft/vscode/issues/135784)

## Summary

Team-authored feature request for PWA File Handling API registration — still unimplemented on main (no file_handlers in the web manifest). Keeping open per team-member authorship rule and because the capability genuinely does not exist yet.

## Change summary

Register vscode.dev as a file handler for common code/text file types via the PWA File Handling API so files can be opened directly from the OS file explorer.

## Best solution

Add a `file_handlers` section to `resources/server/manifest.json` listing common code/text extensions and wire up the `launchQueue` consumer in the web workbench startup to open the handled files. The existing file-extension lists used for Windows/macOS associations could seed the handler set.
