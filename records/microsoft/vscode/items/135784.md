# Issue #135784 — [PWA] Register vscode.dev as file handler for well-supported file types

[https://github.com/microsoft/vscode/issues/135784](https://github.com/microsoft/vscode/issues/135784)

## Summary

Unimplemented feature request from a team member. The PWA manifest at `resources/server/manifest.json` has no `file_handlers` entry and there is no `launchQueue` API usage anywhere in the codebase — the File Handling API integration has not been built. Hard keep-open applies (MEMBER author).

## Change summary

Register vscode.dev as a PWA file handler so users can open files directly from the OS file explorer into vscode.dev.

## Best solution

Add a `file_handlers` block to the PWA manifest listing well-supported text/code file extensions and implement `launchQueue` consumption in the web workbench startup path to open the handled files. The related issue https://github.com/microsoft/vscode/issues/143936 was already closed as a duplicate of this one, confirming this is the canonical tracker.
