# Issue #295389 — Opening Copilot Chat after crash leads to error.

[https://github.com/microsoft/vscode/issues/295389](https://github.com/microsoft/vscode/issues/295389)

## Summary

This was fixed by PR #289779 (commit 7393b0ec607) which added migration logic to ChatSessionStore for workspace ID changes, shipped in 1.110.1. The reporter was on 1.106.3 and said they'd re-test on a newer version.

## Change summary

After a crash, the workspace ID changes and chat session files become inaccessible because the storage path includes the workspace ID.

## Best solution

Keep the shipped fix in commit 7393b0ec607 (1.110.1); the ChatSessionStore now migrates sessions when the workspace ID changes. No further change is needed — the reporter should upgrade from 1.106.3.
