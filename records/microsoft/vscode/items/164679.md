# Issue #164679 — browser password auto complete function working unexpectedly on Quick Pick Input

[https://github.com/microsoft/vscode/issues/164679](https://github.com/microsoft/vscode/issues/164679)

## Summary

This is a known Chromium/browser limitation where password autofill leaks into non-password input fields. The related PR #167191 was closed without merging, the issue is correctly labeled upstream-issue-pending and help wanted, and remains valid — no fix has landed on main.

## Change summary

Browser password autofill popups appear on all Quick Pick inputs in vscode.dev/code-server after saving credentials via a git credential prompt.

## Best solution

A workaround via input `name` attributes or other HTML hints could be attempted (as PR #167191 tried), but the root cause is Chromium's autofill heuristics ignoring `autocomplete=off`. Keep open with `upstream-issue-pending` until Chromium provides a reliable opt-out mechanism, or a creative workaround lands via `help wanted`.
