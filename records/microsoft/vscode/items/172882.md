# Issue #172882 — Replace in files doesn't work as expected with match containing newline

[https://github.com/microsoft/vscode/issues/172882](https://github.com/microsoft/vscode/issues/172882)

## Summary

Still reproduces on main: multiline regex matches in 'Replace in Files' fall back to the literal replacement string (e.g. `$1`) instead of performing capture-group substitution, because all four `getReplaceString` attempts fail and the fallback at line 121 returns the raw pattern.

## Change summary

When a regex match in 'Replace in Files' spans multiple lines (e.g. uses a lookahead that crosses a newline), capture-group substitution ($1, $2, etc.) is replaced literally instead of with the captured value.

## Best solution

Fix the fallback path in `src/vs/workbench/contrib/search/browser/searchTreeModel/match.ts` (around line 120) so that when `getReplaceString` returns `null` for a multiline match, the code still performs capture-group substitution against the match text rather than returning the raw replacement pattern string. The `ReplacePattern` class in `src/vs/workbench/services/search/common/replace.ts` may also need adjustment to handle multiline match text correctly with its regex.
