# Issue #161347 — A misalignment of a few menu's shortcut keys (macOS)

[https://github.com/microsoft/vscode/issues/161347](https://github.com/microsoft/vscode/issues/161347)

## Summary

Still present on main: non-native keybindings (chords, custom bindings) are rendered inline as `[shortcut]` in native macOS menus because Electron lacks support for arbitrary accelerator labels. The fallback is explicit in `menubar.ts` lines 833–841. This is an upstream Electron limitation and remains valid.

## Change summary

On macOS, some keyboard shortcuts in native menus appear inline in brackets rather than right-aligned as proper accelerators.

## Best solution

The fix belongs in Electron — adding support for rendering arbitrary labels as accelerator-style text in native menus. Until Electron exposes that capability, VS Code's fallback (bracket notation) is the only option. Keep open as an upstream tracking issue.
