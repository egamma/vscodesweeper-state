# Issue #309506 — Should the model be encouraged to use a small subagent for multi-stage inputs?

[https://github.com/microsoft/vscode/issues/309506](https://github.com/microsoft/vscode/issues/309506)

## Summary

Team-authored discussion about using a lightweight subagent for multi-stage terminal inputs; must remain open for maintainer judgment. The merged PR #309904 improved the terminal subagent experience but the comments indicate architectural constraints (lack of async subagents, missing input-needed/command-finished notifications) that may leave this partially unresolved.

## Change summary

Use a small/fast subagent model instead of the main agent loop for multi-stage terminal input flows.

## Best solution

Evaluate whether async subagents with access to terminal input-needed and command-finished notifications are feasible; if so, route multi-stage terminal inputs through a cheaper model to reduce latency and cost in the agent loop.
