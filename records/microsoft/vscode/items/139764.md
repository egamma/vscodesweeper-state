# Issue #139764 — Build does not wait for files to save

[https://github.com/microsoft/vscode/issues/139764](https://github.com/microsoft/vscode/issues/139764)

## Summary

Still reproduces on main: `saveAllBeforeDebugStart` in debugUtils.ts awaits `editorService.saveAll()` but ignores its return value, so debugging proceeds even when a save fails (e.g. during privilege escalation). The partial fix in commit 42a43eb made saveAll correctly return false on failure, but the caller was never updated to act on it.

## Change summary

When a file save requires privilege escalation (sudo), the debug session starts before the save completes, because the save-before-debug code does not check whether saving actually succeeded.

## Best solution

Update `saveAllBeforeDebugStart` in `src/vs/workbench/contrib/debug/common/debugUtils.ts` to check the return value of `editorService.saveAll()`. When it returns `false`, either abort the debug launch or show a confirmation prompt ('Save failed — start debugging anyway?') as roblourens suggested in the discussion. The prompt approach avoids blocking on non-modal notifications while still giving the user a choice.
