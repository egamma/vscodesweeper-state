# Issue #227026 — Infinite loading when filesystem has no registered provider

[https://github.com/microsoft/vscode/issues/227026](https://github.com/microsoft/vscode/issues/227026)

## Summary

Still reproduces on current main. The bug is in the team member's own area (search) and the milestone is On Deck — keeping open per policy. The deferred promise in waitForProvider never resolves when no provider registers for the scheme.

## Change summary

File picker search hangs infinitely when a filesystem has no registered file search provider.

## Best solution

Add a timeout or cancellation guard to `waitForProvider` in `src/vs/workbench/services/search/common/searchService.ts` so it resolves with an empty result (or rejects with a clear error) instead of hanging forever when no provider registers. Alternatively, check `schemeHasFileSearchProvider` before initiating the search for that scheme.
