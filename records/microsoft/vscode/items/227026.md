# Issue #227026 — Infinite loading when filesystem has no registered provider

[https://github.com/microsoft/vscode/issues/227026](https://github.com/microsoft/vscode/issues/227026)

## Summary

Team-authored issue with On Deck milestone; the infinite-wait path in searchService.ts is still present on main when no file search provider is registered for a scheme and no other scheme has a provider either.

## Change summary

Quick-pick file search hangs indefinitely when a filesystem has no registered file search provider.

## Best solution

Add a timeout or early-exit in `searchService.ts` `waitForProvider` so the search resolves (empty or with an error) instead of hanging forever when no provider will ever register for the scheme. The `schemeHasFileSearchProvider` utility added in 62ad06ad638 could be used to short-circuit earlier.
