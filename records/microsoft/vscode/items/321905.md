# Issue #321905 — Chat implicit "active selection" context stays stuck after its source document is closed/deleted

[https://github.com/microsoft/vscode/issues/321905](https://github.com/microsoft/vscode/issues/321905)

## Summary

Real bug on current main: the implicit context preservation logic in `updateImplicitContext` does not check whether the backing document's model/editor still exists. When a non-code editor is active after closing the source document, stale context persists because `hasActiveEditor` is true while `newValue` is undefined, and the guard on line 287 preserves existing values.

## Change summary

Closing or deleting a document leaves its implicit "active selection" context chip stuck in the Chat input, silently sending stale context on every subsequent turn.

## Best solution

In `ChatImplicitContextContribution.updateImplicitContext()` (src/vs/workbench/contrib/chat/browser/attachments/chatImplicitContext.ts), when `newValue` is undefined and existing implicit context has a value, validate that the backing URI still has a live text model or open editor before preserving the stale values. If the backing resource is gone, clear the implicit context. Also listen for `editorService.onDidCloseEditor` to trigger `updateImplicitContext()` when the source editor is closed.

## Evidence

- **source trace:** In src/vs/workbench/contrib/chat/browser/attachments/chatImplicitContext.ts, line 287: `if (newValue !== undefined || !widget.input.implicitContext.hasValue || !hasActiveEditor)` — when `newValue` is undefined (no code editor), `hasValue` is true (stale context), and `hasActiveEditor` is true (non-code editor like chat panel), the condition is false and stale values are preserved.
- **prior partial fix:** Commit addd8a9e3c1 ('Fix detected attachment sticking around after editor is closed', #297400) added the `!hasActiveEditor` guard, which fixes the case where ALL editors are closed but not the case where a non-code editor is active while the source document is gone.
- **no subsequent fix:** No commits to chatImplicitContext.ts since June 2026; the stale-context-with-active-non-code-editor path remains unguarded on current main.
- **missing event:** The contribution listens to `onDidActiveEditorChange` and selection/scroll events but not `onDidCloseEditor`, so closing the source document without switching editors may not trigger a re-evaluation.
