# Issue #296608 — Error when searching after opening file by clicking link in terminal with line number

[https://github.com/microsoft/vscode/issues/296608](https://github.com/microsoft/vscode/issues/296608)

## Summary

Still reproduces on main: `createAndConfigureFileMatch` in `folderMatch.ts` does not strip URI fragments before the `isEqualOrParent` descendant check, so any open editor with a fragment (e.g. `#L65` from a terminal or Markdown link) causes the search to throw. No fix commit exists.

## Change summary

Opening a file via a terminal link with a line number (e.g. `file.php#L65`) causes search to error out because the URI fragment is included in the workspace-descendant path check.

## Best solution

Strip the URI fragment (e.g. `rawFileMatch.resource.with({ fragment: '' })`) before the `uriHasParent` check in `createAndConfigureFileMatch` in `src/vs/workbench/contrib/search/browser/searchTreeModel/folderMatch.ts`. Alternatively, the root cause may be upstream: whatever surfaces open-editor URIs to the search should normalize them first. Both the terminal-link and Markdown-link entry points are affected.

It calls `this.uriHasParent(this.resource, rawFileMatch.resource)` which delegates to `isEqualOrParent` — but when `rawFileMatch.resource` contains a fragment like `#L65`, the comparison fails because the fragment is part of the URI string.
