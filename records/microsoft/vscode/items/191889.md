# Issue #191889 — ILanguageService events only fire once

[https://github.com/microsoft/vscode/issues/191889](https://github.com/microsoft/vscode/issues/191889)

## Summary

Team-authored debt issue; the problem is still present on main — the events are guarded by a Set so they fire at most once per language, and late subscribers never receive them.

## Change summary

ILanguageService's onDidRequestBasicLanguageFeatures and onDidRequestRichLanguageFeatures fire only once per language; late listeners are never called.

## Best solution

Convert these to replay-style events (e.g. fire for already-requested languages when a new listener attaches), or expose a synchronous query for already-requested languages alongside the event, in src/vs/editor/common/services/languageService.ts.
