# Issue #320882 — Use WebView2 instead of Electron V8 on the Windows platform.

[https://github.com/microsoft/vscode/issues/320882](https://github.com/microsoft/vscode/issues/320882)

## Summary

This is a duplicate of previously triaged requests to replace Electron with WebView2 on Windows. Issue #138343 (the same ask) was closed as *out-of-scope, and #142287 (nearly identical) was closed as NOT_PLANNED. VS Code is an Electron application by design; swapping the rendering engine to WebView2 is a fundamental architectural change that has been declined.

## Change summary

Allow users to switch VS Code's rendering engine from Electron to WebView2 on Windows for better performance and rendering quality.

## Best solution

Consolidate on the prior decision in https://github.com/microsoft/vscode/issues/138343, which was closed *out-of-scope. VS Code's architecture is built on Electron, and replacing it with WebView2 is not in scope for the project. The request conflates V8 (JavaScript engine) with the rendering layer and does not present evidence of a concrete deficiency that WebView2 would solve.

## Vision fit

- **Fit:** rejected
- **Reason:** VS Code's core value is 'lightweight and fast' via Electron — replacing the runtime with WebView2 is a fundamental platform swap that conflicts with the established architecture and the principle that the power lives in extensions, not in swapping internals.
- **Evidence:** Value: 'Out of the box, VS Code is a text editor: lightweight and fast.' The Electron platform is the foundation of VS Code's cross-platform story; replacing it with a Windows-only runtime contradicts the uniform architecture.
