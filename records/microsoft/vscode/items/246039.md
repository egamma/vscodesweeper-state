# Issue #246039 — Adding an MCP Server from a private NPM doesn't work

[https://github.com/microsoft/vscode/issues/246039](https://github.com/microsoft/vscode/issues/246039)

## Summary

Valid feature gap: while custom registryBaseUrl support exists in the MCP management service for gallery-sourced servers, there is no way for users to specify a private npm registry (or have .npmrc respected) when adding an MCP server manually by package name. The validation step in the Copilot extension still hardcodes registry.npmjs.org.

## Change summary

Allow MCP server discovery and installation from private npm registries (via .npmrc or explicit registry configuration) rather than only the public npmjs.org registry.

## Best solution

Extend the 'Add MCP Server' flow to either read the user's .npmrc for registry configuration or provide a UI field to specify a custom registry URL. The Copilot extension's validation in `extensions/copilot/src/extension/mcp/vscode-node/commands.ts` (line 326) currently hardcodes `https://registry.npmjs.org/` and needs to support alternative registries. The platform-level installation code in `src/vs/platform/mcp/common/mcpManagementService.ts` already passes `--registry` when `registryBaseUrl` is set, so the missing piece is surfacing that to user-initiated flows.
