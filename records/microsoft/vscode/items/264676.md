# Issue #264676 — Should I be asked if I want to open the worktree?

[https://github.com/microsoft/vscode/issues/264676](https://github.com/microsoft/vscode/issues/264676)

## Summary

Team-authored polish issue on the 'On Deck' milestone — kept open per policy. The request to prompt users to open a newly created worktree is a reasonable UX improvement to the git worktree quick-pick flow.

## Change summary

After creating a git worktree via the quick pick, the user is not offered the option to open it in the current window, a new window, or not at all.

## Best solution

Add a follow-up quick-pick step in the 'create worktree' flow (likely in `extensions/git/src/commands.ts` or the worktree-related command handler) that offers 'Open in Current Window / Open in New Window / Don't Open', similar to how clone/init flows prompt to open the resulting folder.
