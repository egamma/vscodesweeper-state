# Issue #223077 — Make the `TextDocument.version` properly monotonically increasing during the whole application lifecycle

[https://github.com/microsoft/vscode/issues/223077](https://github.com/microsoft/vscode/issues/223077)

## Summary

A well-reasoned extension-API behavior proposal still awaiting a second opinion from the team (alexdima was pinged but never responded). The PR was closed but the issue was explicitly reopened for discussion. Current code still resets versions when disposed models are evicted from the heap-limited cache.

## Change summary

Make TextDocument.version never reset within the VS Code lifecycle, so extensions can use it as a unique document-state identifier across close/reopen cycles.

## Best solution

A team member (alexdima or equivalent) should weigh in on the trade-off: the memory cost of retaining (uri, version, sha1) indefinitely vs. the extension-developer benefit. If accepted, the fix lives in `src/vs/editor/common/services/modelService.ts` around the `_disposedModels` eviction logic. If rejected, close as `*out-of-scope` with a note that extensions should track versions themselves.
