# Issue #133397 — Embedded languages use outer language auto-closing characters

[https://github.com/microsoft/vscode/issues/133397](https://github.com/microsoft/vscode/issues/133397)

## Summary

Still reproduces on main: CursorConfiguration in cursorCommon.ts always uses the outer languageId for autoClosingPairs/surroundingPairs, ignoring embedded languages. An open PR (#309365) addresses this but hasn't merged yet.

## Change summary

Auto-closing pairs inside embedded language regions (e.g. C++ in LaTeX, math in Markdown) incorrectly use the parent language's configuration instead of the embedded language's.

## Best solution

Merge or iterate on PR https://github.com/microsoft/vscode/pull/309365 which makes cursor typing operations query the language configuration at the cursor position rather than always using the outer language. The fix lives in `src/vs/editor/common/cursorCommon.ts` and `src/vs/editor/common/cursor/cursorTypeEditOperations.ts`.
