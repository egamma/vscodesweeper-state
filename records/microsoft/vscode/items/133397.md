# Issue #133397 — Embedded languages use outer language auto-closing characters

[https://github.com/microsoft/vscode/issues/133397](https://github.com/microsoft/vscode/issues/133397)

## Summary

Keep open: this is a confirmed editor auto-closing bug, and current main still appears to use the outer model language configuration for auto-closing pairs rather than the embedded language at the cursor. The issue is in the Backlog, which does not by itself block cleanup, but there is no evidence of a fixing commit or duplicate owner.

## Change summary

Auto-closing characters in embedded languages should come from the embedded language configuration instead of the outer language configuration.

## Best solution

Fix the editor auto-closing path to resolve language configuration from scoped tokens at each cursor position, including multi-cursor behavior, and add focused unit coverage under the editor cursor/typing tests.
