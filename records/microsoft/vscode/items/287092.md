[https://github.com/microsoft/vscode/issues/287092](https://github.com/microsoft/vscode/issues/287092)

## Summary

The error originates entirely within the ms-python.debugpy extension's bundled pydevd library (PySide6/Qt GUI event loop integration), not in VS Code core. This should be reported to the debugpy extension.

## Change summary

Debug breakpoints trigger a Qt GUI event loop integration error from the debugpy extension's bundled pydevd library when using PySide6.

## Best solution

Report this to the debugpy extension repository (https://github.com/microsoft/debugpy) — the stack trace is entirely within `ms-python.debugpy`'s vendored `pydevd` code trying to create a `QApplication` via `PySide6.QtGui` instead of `PySide6.QtWidgets`.
