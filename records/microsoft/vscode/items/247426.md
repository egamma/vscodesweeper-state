## Summary

Valid API feature request for dynamic, model-aware tool descriptions. No implementation exists on main — the `LanguageModelToolInformation.description` remains static. A team member (roblourens) engaged with a clarifying question, and the reporter provided a reasonable use case. Low community traction (1 upvote) so not yet eligible for Backlog promotion; keeping open for the community process.

## Change summary

Allow extension-registered language model tools to provide dynamic, model-specific descriptions instead of a single static `modelDescription`.

## Best solution

Consider adding an optional `resolveDescription` method on `LanguageModelTool<T>` (or a hook in tool registration) that receives the model identifier and returns a tailored description string. This would live in the `chat-tools` API surface (`src/vs/workbench/api/common/extHostLanguageModelTools.ts` and the corresponding `vscode.d.ts` types). The current low traction suggests waiting for more community signal before designing the API shape.
