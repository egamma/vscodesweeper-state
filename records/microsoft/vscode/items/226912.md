# Issue #226912 — Second layer of filtering after returning from `FileSearchProviderNew`'s `provideFileSearchResults(...)`

[https://github.com/microsoft/vscode/issues/226912](https://github.com/microsoft/vscode/issues/226912)

## Summary

Team-authored polish issue tracking documentation improvements for the FileSearchProvider API's post-processing behavior. The API docs already hint at relaxed matching (lines 80–83 of the proposed API type) but don't explicitly document the two layers of post-filtering (glob re-application and fuzzy picker filtering) that andreamah described. Keeping open per team-author rule.

## Change summary

Document the two layers of post-processing (glob re-application and fuzzy picker filtering) that occur after a FileSearchProvider returns results.

## Best solution

Update the JSDoc on `provideFileSearchResults` in the proposed `FileSearchProvider2` API (`src/vscode-dts/vscode.proposed.fileSearchProvider2.d.ts`) to explicitly note that returned results may be further filtered by glob patterns and by the quick-open picker's fuzzy scoring, so extension authors understand the provider is not the sole arbiter of what the user sees.
