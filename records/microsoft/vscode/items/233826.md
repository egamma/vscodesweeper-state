# Issue #233826 — When hover contains one hover provider result, increase/decrease verbosity commands should work on that provider result

[https://github.com/microsoft/vscode/issues/233826](https://github.com/microsoft/vscode/issues/233826)

## Summary

Team-authored polish issue requesting smarter default targeting for hover verbosity commands. Still valid on main — the commands require explicit focus on a specific hover provider result even when only one exists.

## Change summary

Hover verbosity increase/decrease commands should auto-target the sole hover provider result (or the only verbosity-capable one) without requiring the user to explicitly focus it.

## Best solution

Update the hover verbosity command handlers in `src/vs/editor/contrib/hover/` to auto-select the target hover part when there is only one provider result, or only one result that supports verbosity changes, removing the need for an extra focus gesture.
