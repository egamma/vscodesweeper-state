# Issue #88158 — Provide a way to debug extensions to hint syntax for advanced breakpoints

[https://github.com/microsoft/vscode/issues/88158](https://github.com/microsoft/vscode/issues/88158)

## Summary

Team-authored feature request still under discussion. The breakpoint widget placeholders remain hardcoded and generic on current main — no DAP capability or API lets adapters hint at their syntax for hit conditions or logpoint expressions. The later `breakpointModes` DAP addition is related but orthogonal.

## Change summary

Allow debug adapters to provide syntax hints (placeholder text, docs URL, or description) for hit-count expressions and logpoint message format in the breakpoint widget.

## Best solution

If still desired, extend DAP capabilities (or the VS Code extension API) so adapters can supply per-context placeholder text or a documentation URL for the breakpoint widget's condition, hit-count, and log-message inputs. The UI side lives in `src/vs/workbench/contrib/debug/browser/breakpointWidget.ts` (the `placeholder` getter). Requires coordination with the DAP spec maintainers.
