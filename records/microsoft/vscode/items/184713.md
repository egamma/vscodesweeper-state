# Issue #184713 — MacOS: "no job control in this shell" prevents Ctrl+C

[https://github.com/microsoft/vscode/issues/184713](https://github.com/microsoft/vscode/issues/184713)

## Summary

This bug only affected macOS High Sierra (10.13) and Mojave (10.14), which VS Code no longer supports. Current VS Code (Electron 42) requires macOS 12 (Monterey) minimum, so the affected platform is completely outside the support matrix and the issue cannot reproduce on any supported version.

## Change summary

Bash terminals on macOS High Sierra/Mojave showed 'no job control in this shell' and Ctrl+C could not interrupt processes.

## Best solution

Close as not reproducible on supported platforms. VS Code's minimum macOS version is now 12 (Monterey), well past the affected 10.13/10.14 versions. No code change is needed.
