# Issue #192719 — MacOS scrolling inertia is not cancelled when stopped

[https://github.com/microsoft/vscode/issues/192719](https://github.com/microsoft/vscode/issues/192719)

## Summary

Real macOS-specific bug where OS-level trackpad inertia events continue arriving after a viewport jump (e.g., typing causes reveal-cursor). The codebase has no mechanism to detect or discard momentum-phase wheel events because the web platform doesn't reliably expose macOS's momentumPhase. Keeping open as a valid low-severity platform interaction issue.

## Change summary

On macOS, trackpad scrolling inertia is not cancelled when the viewport programmatically jumps to the cursor position, causing unwanted continued scrolling.

## Best solution

Implement a heuristic in the scroll handler (likely `src/vs/base/browser/ui/scrollbar/scrollableElement.ts`) that detects when `setScrollPositionNow` programmatically changes the scroll position and temporarily suppresses incoming wheel events for a short window (~200ms), or resets the internal inertial scroll state. Alternatively, track whether a programmatic scroll just occurred and ignore small-delta wheel events that look like tail-end inertia.
