# Issue #272091 — [html] Auto-complete in embedded JavaScript does not work when no space is present

[https://github.com/microsoft/vscode/issues/272091](https://github.com/microsoft/vscode/issues/272091)

## Summary

Still present on main: embedded JavaScript completions fail at the boundary of a closing tag when no space separates the cursor from the tag. The HTML language service's region boundary detection or the `getModeAtPosition` logic returns 'html' instead of 'javascript' in this edge case.

## Change summary

Auto-complete for embedded JavaScript in HTML files does not trigger when the cursor is immediately adjacent to a closing tag with no intervening space.

## Best solution

Fix the embedded region boundary detection in `extensions/html-language-features/server/src/modes/embeddedSupport.ts` (or upstream in `vscode-html-languageservice`) so that the cursor position immediately before a closing tag within a script region is correctly identified as JavaScript. Alternatively, adjust the completion trigger in `htmlServer.ts` to look one character back when at a region boundary.
