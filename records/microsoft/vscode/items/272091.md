## Summary

Valid bug report about embedded JavaScript completions failing at region boundaries in HTML files. No fix has landed on main. The report could benefit from clarification on the exact scenario (inline event handler vs script tag), but the described behavior is consistent with known boundary-detection limitations in the HTML language server's embedded completion logic.

## Change summary

Auto-complete for embedded JavaScript in HTML does not trigger when the cursor is immediately before a closing tag with no space separating them.

## Best solution

Investigate the completion trigger logic in `extensions/html-language-features/server/src/modes/embeddedSupport.ts` and `javascriptMode.ts` to ensure completions are offered at the exact boundary between embedded JS content and closing HTML tags. May also involve the upstream `vscode-html-languageservice` scanner's token boundary calculations.
