[https://github.com/microsoft/vscode/issues/302261](https://github.com/microsoft/vscode/issues/302261)

## Summary

Still reproduces on main: the debug integrated terminal path in extHostDebugService.ts only extracts `path` and `args` from the automation profile via getDefaultShell/getDefaultShellArgs, completely ignoring the profile's `env` property when creating the terminal options.

## Change summary

The `terminal.integrated.automationProfile.windows` env object is ignored when the debugger launches an integrated terminal via `runInTerminal`.

## Best solution

In `src/vs/workbench/api/node/extHostDebugService.ts`, add a method like `getDefaultEnv(useAutomationShell)` to `IExtHostTerminalService` that returns `profile.env`, and pass it into the terminal options at line ~100. The task system already handles this correctly via `convertProfileToShellLaunchConfig`.
