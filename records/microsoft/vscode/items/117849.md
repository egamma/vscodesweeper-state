# Issue #117849 — [remote cli] support opening `file` URIs

[https://github.com/microsoft/vscode/issues/117849](https://github.com/microsoft/vscode/issues/117849)

## Summary

The initial BROWSER variable issue was partially addressed in 1.54, but the broader feature request — making file:// URI opening work seamlessly from remote terminals (e.g. Python's webbrowser.open) — remains unimplemented. Keeping open as a valid feature request in the Backlog.

## Change summary

Opening file:// URIs from a remote/WSL terminal should launch the host browser with the correct translated path.

## Best solution

Extend the remote CLI's openExternal path (src/vs/server/node/server.cli.ts) to translate file:// URIs from the remote filesystem to host-accessible paths (e.g. \\wsl$\... on Windows) before forwarding to the host opener, and re-enable a BROWSER-like mechanism that tools like Python's webbrowser module can use.
