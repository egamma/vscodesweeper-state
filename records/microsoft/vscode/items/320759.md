# Issue #320759 — Agents Window: Agent dropdown only shows “Agent” despite multiple agents being configured

[https://github.com/microsoft/vscode/issues/320759](https://github.com/microsoft/vscode/issues/320759)

## Summary

The Agents Window mode picker intentionally shows only 'Agent' plus custom modes whose target matches — it does not include Ask or Plan (those are regular chat panel modes). Custom agents may not appear due to target filtering. Needs more info to determine whether the custom agents are configured with the correct target, or whether the reporter is conflating the regular chat panel's mode switcher with the Agents Window's agent picker.

## Change summary

The Agents Window dropdown shows only 'Agent' instead of the expected Ask, Plan, and custom agent options.

## Best solution

Clarify whether the reporter expects Ask/Plan in the Agents Window (which is by design — the Agents Window is agent-mode only) or whether their configured custom agents genuinely fail to appear. If the latter, the target filtering in `ModePickerModel.getAvailableModes()` (`src/vs/sessions/contrib/providers/copilotChatSessions/browser/modePicker.ts`, lines 95–116) should be investigated — custom modes only appear when their `target` matches `effectiveTarget` or is `Target.Undefined`.
