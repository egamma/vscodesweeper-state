# Issue #299889 — In WSL, Export Chat as Zip default location fails: attempts to write to a Windows-like version of the actual path that doesn't exist

[https://github.com/microsoft/vscode/issues/299889](https://github.com/microsoft/vscode/issues/299889)

## Summary

Team-authored issue — keeping open. The bug is traceable in the source: `createZipFile` calls `zipPath.fsPath` in the Electron main process (Windows side), which converts a WSL Unix path like `/home/user/...` into `C:\home\user\...` instead of using the remote file system.

## Change summary

Exporting chat as zip in a WSL remote session fails because the URI's Unix path is converted to a Windows path in the main process.

## Best solution

The fix should write via `IFileService` (which respects the remote authority) instead of `zipPath.fsPath` in `nativeHostMainService.ts`, or the action should use a file-service-aware write path. Likely files: `src/vs/workbench/contrib/chat/electron-browser/actions/chatExportZip.ts` and possibly a new remote-aware zip helper.
