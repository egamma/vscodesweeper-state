# Issue #264731 — Chat - Can't change chat inline code font style

[https://github.com/microsoft/vscode/issues/264731](https://github.com/microsoft/vscode/issues/264731)

## Summary

Still present on main: inline `<code>` elements in chat markdown responses do not pick up `chat.editor.fontFamily`. The CSS variable `--vscode-chat-font-family` is set on the widget container but no rule targets inline code to use the editor/monospace font. The discussion (lszomoru + mjbvz) agreed on using `chat.editor.fontFamily` for inline code but not `chat.editor.fontSize`; no implementation has landed.

## Change summary

Inline code spans in chat responses ignore the `chat.editor.fontFamily` setting, rendering in the chat body font rather than the configured monospace font.

## Best solution

Add a CSS rule targeting `.interactive-item-container .value .rendered-markdown code` (non-fenced inline code) to set `font-family: var(--vscode-chat-font-family, var(--monaco-monospace-font))`. Per the discussion, do NOT override `font-size` for inline code to avoid size mismatches. The fix belongs in `src/vs/workbench/contrib/chat/browser/widget/media/chat.css`.
