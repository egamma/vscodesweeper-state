# Issue #269269 — Git - Do not see chat sparkle icon when merge conflict is generated

[https://github.com/microsoft/vscode/issues/269269](https://github.com/microsoft/vscode/issues/269269)

## Summary

Team-authored feature request (keep-open by policy). The chat sparkle icon for merge conflicts currently only appears in the EditorContent menu when chat setup is NOT completed (the `scm.editor.triggerSetup` action). Once chat is set up, the resolve-conflicts action is only available via context menu, not inline. The request is to add an inline sparkle entry point when chat is already installed.

## Change summary

Show the chat sparkle icon inline in the editor when a merge conflict is detected, even when Copilot Chat is already set up — not just in the right-click context menu.

## Best solution

Add a second `EditorContent` menu registration (or adjust the existing one) in `src/vs/workbench/contrib/scm/browser/scm.contribution.ts` that shows the sparkle icon when `ChatContextKeys.Setup.completed` is true and the file has merge conflicts. The area owner (lszomoru) has acknowledged the gap and would need to decide whether the inline entry point is warranted.
