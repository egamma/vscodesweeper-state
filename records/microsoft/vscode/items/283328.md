## Summary

Team-authored, milestone On Deck, labeled important — hard keep-open. Multiple partial fixes have merged (PRs #286866, #292366, #307938), but the root cause (elicitation state not transitioning out of 'pending' on dismiss) appears only worked around, not fully resolved. The issue remains valid.

## Change summary

The stop button disappears or becomes non-functional during agent sessions, leaving the user unable to interrupt the AI without reloading the window.

## Best solution

Fix the root cause: ensure elicitation state transitions out of 'pending' when the user clicks 'focus terminal' or otherwise dismisses the elicitation widget, rather than only working around the missing stop button. The fix likely lives in `src/vs/workbench/contrib/chat/common/model/chatModel.ts` around the elicitation state management (line ~1339) and the terminal elicitation dismissal flow.
