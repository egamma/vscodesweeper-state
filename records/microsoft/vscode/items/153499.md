# Issue #153499 — Git - git.terminalGitEditor should return focus to the terminal after being closed

[https://github.com/microsoft/vscode/issues/153499](https://github.com/microsoft/vscode/issues/153499)

## Summary

Still reproducible on main: the git editor handler in extensions/git/src/gitEditor.ts resolves after the tab closes but never returns focus to the terminal. The polish request remains valid.

## Change summary

After closing a file opened via git.terminalGitEditor (or `code --wait`), focus should return to the terminal that spawned the editor.

## Best solution

After the tab-close promise resolves in `GitEditor.handle()`, focus the terminal instance that initiated the request. The fix likely involves calling the terminal focus API (or signaling back to the originating terminal) in `extensions/git/src/gitEditor.ts` and possibly `src/vs/workbench/contrib/terminal/` for the broader `--wait` case the commenter describes.
