# Issue #221423 — Refreshing semantic tokens does not visually update 

[https://github.com/microsoft/vscode/issues/221423](https://github.com/microsoft/vscode/issues/221423)

## Summary

The bug describes document-level semantic tokens not visually refreshing until scroll. PR #271419 fixed a related issue for viewport/range tokens, but no commit explicitly fixes the document-level refresh path described here. The mechanism looks correct in code, but the issue may involve timing/debounce interactions that are hard to verify without runtime reproduction. Keeping open — the report is thin on reproduction steps for the core bug vs the extension-specific scenario, but a real gap may remain.

## Change summary

When an extension fires `onDidChange` on a document semantic tokens provider and sends updated tokens, the editor view does not repaint until the user scrolls or types.

## Best solution

Confirm whether this still reproduces on latest Insiders with the C#/Razor extension. If it does, the fix likely belongs in `src/vs/editor/contrib/semanticTokens/browser/documentSemanticTokens.ts` — possibly canceling the in-flight request and immediately re-fetching when `handleProviderDidChange` fires during a running request, rather than deferring to `rescheduleIfNeeded`. If the Razor extension now uses range-based tokens (fixed by PR #271419), this may already be resolved in practice.
