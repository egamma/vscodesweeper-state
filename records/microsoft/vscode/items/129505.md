# Issue #129505 — Git - Images should not be `textconv`ed when pulled from Git for display in diff viewer

[https://github.com/microsoft/vscode/issues/129505](https://github.com/microsoft/vscode/issues/129505)

## Summary

Still reproduces on main: `extensions/git/src/git.ts` unconditionally passes `--textconv` to `git show` in the `buffer()` and `detectObjectType()` methods, so images with a configured textconv filter are converted to text instead of being served as raw binary for the image diff viewer.

## Change summary

Images with a git textconv filter configured are displayed as corrupted/text in the diff viewer instead of as images.

## Best solution

When retrieving file content for binary/image display (as opposed to text diff), omit `--textconv` from the `git show` command. The fix is localized to `extensions/git/src/git.ts` — either remove `--textconv` from the `buffer()` method (and add it only for text content retrieval) or add a parameter to control whether textconv is applied.
