# Issue #218033 — RangeError: Maximum call stack size exceeded.

[https://github.com/microsoft/vscode/issues/218033](https://github.com/microsoft/vscode/issues/218033)

## Summary

The recursive close pattern likely still exists on current main: the `beforeunload` handler in `issueFormService.ts` line 498–499 calls `auxiliaryWindow.window.close()` on the same window, which can re-trigger `beforeunload` in Safari. The web issue reporter still uses this auxiliary-window path. Keeping open as the bug appears to remain.

## Change summary

Opening and closing the issue reporter in Safari triggers a RangeError: Maximum call stack size exceeded due to recursive `close` calls.

## Best solution

Guard the `beforeunload` handler in `src/vs/workbench/contrib/issue/browser/issueFormService.ts` (line ~498) against re-entrancy — e.g. with a boolean flag or by removing the `auxiliaryWindow.window.close()` call from inside `beforeunload`, since the window is already unloading at that point. The `dispose()` and null-assignment are the meaningful cleanup; the redundant `close()` is what causes the recursion on Safari.
