# Issue #322340 — Allow customizing Git Blame date formatting

[https://github.com/microsoft/vscode/issues/322340](https://github.com/microsoft/vscode/issues/322340)

## Summary

Valid feature request to allow customizing Git Blame date formatting. Currently `authorDate` uses `toLocaleString()` and `authorDateAgo` uses `fromNow()` with no user-configurable format — the gap is confirmed on current main.

## Change summary

Allow users to customize the date format used in Git Blame editor decorations and status bar, independently of the system locale.

## Best solution

Add a `git.blame.dateFormat` setting (or support inline format specifiers like `${authorDate:yyyy-MM-dd}` in the template) in `extensions/git/src/blame.ts`, modifying `formatBlameInformationMessage()` to apply the user's chosen format to the `authorDate` and `authorDateAgo` tokens.
