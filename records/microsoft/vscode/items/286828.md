# Issue #286828 — Shortcut for Terminal in Linux no longer works correctly for non-default US keyboards

[https://github.com/microsoft/vscode/issues/286828](https://github.com/microsoft/vscode/issues/286828)

## Summary

Verified duplicate of #282752 — both report that dead-key / compose-key shortcuts (Ctrl+backtick to toggle terminal) broke on Linux with the Electron 39 update (Wayland IME handling change), affecting international keyboard layouts.

## Change summary

Ctrl+backtick shortcut to toggle the terminal no longer works on Linux with non-US keyboard layouts that use dead/compose keys, since approximately VS Code 1.107.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/282752, which is the canonical tracking issue for this Electron/Wayland dead-key regression, already labeled `upstream` and `electron-39-update`. The fix likely needs to come from the Electron/Chromium layer's Wayland IME handling.
