# Issue #227099 — Editor GPU: Reduce memory use in the atlas glyph storage buffers

[https://github.com/microsoft/vscode/issues/227099](https://github.com/microsoft/vscode/issues/227099)

## Summary

Still valid: the atlas glyph storage buffer remains fixed at 5,000 entries per page on current main. This is a team-authored plan item for GPU renderer memory optimization.

## Change summary

The GPU renderer allocates a fixed-size glyph storage buffer (5,000 entries per page) regardless of actual usage, wasting memory.

## Best solution

Make the glyph storage GPUBuffer dynamically sized (start small, grow on demand) in `src/vs/editor/browser/viewParts/viewLinesGpu/viewLinesGpu.ts` and update the WGSL shader binding in `fullFileRenderStrategy.wgsl.ts` to match. This reduces baseline memory for editors that use fewer unique glyphs.
