# Issue #216 — [grammars] provide alternative to TextMate grammars

[https://github.com/microsoft/vscode/issues/216](https://github.com/microsoft/vscode/issues/216)

## Summary

This is a long-standing feature request for alternatives to TextMate grammars. Tree-sitter support is being actively developed (experimental settings exist on main for CSS, TypeScript, INI, regex), but the extension API for custom tokenizers that this issue specifically requests does not yet exist. The issue remains valid.

## Change summary

Extensions should be able to provide custom tokenizers/grammars as an alternative to TextMate, enabling better syntax highlighting for complex languages.

## Best solution

The broader tree-sitter exploration is tracked in https://github.com/microsoft/vscode/issues/210475. This issue's specific ask — an extension API for custom tokenization — remains unresolved. Once tree-sitter stabilizes, an extension-facing tokenization API could close this. Until then, this remains a valid open request alongside #210475.
