# Issue #216 — [grammars] provide alternative to TextMate grammars

[https://github.com/microsoft/vscode/issues/216](https://github.com/microsoft/vscode/issues/216)

## Summary

This long-standing feature request for alternatives to TextMate grammars remains valid. Tree-sitter support is actively being explored (experimental settings exist for CSS, TypeScript, INI, regex) but is not yet GA or extensible to third-party extensions — the core ask (an extension API for custom tokenizers) is unresolved.

## Change summary

Provide an extension API or alternative engine (beyond TextMate) for language tokenization so extensions can implement richer syntax highlighting.

## Best solution

Continue tracking under https://github.com/microsoft/vscode/issues/210475 as the canonical tree-sitter exploration issue. This issue could eventually be closed as a duplicate of #210475 once tree-sitter ships with extension API support, but that hasn't happened yet. Keep open as the broader 'alternative tokenizer API' request.
