# Issue #258410 — Incorrect value of resource parameter is sent during authentication when two MCP servers have the same host

[https://github.com/microsoft/vscode/issues/258410](https://github.com/microsoft/vscode/issues/258410)

## Summary

This is the same bug as #272047, which was fixed in commit 3fe7649e8a9 (shipped in 1.107.0). The fix ensures auth provider lookup uses the full resource server URI, not just the auth server base URI, preventing incorrect session reuse across MCP servers sharing a host.

## Change summary

When two MCP servers share the same OAuth authorization server host, VS Code incorrectly reuses the first server's auth session and sends the wrong `resource` parameter for the second server.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/272047 — the fix there (commit 3fe7649e8a9) resolves this exact scenario by matching auth providers on the full resource server URI. Further improvements shipped in d8dd0206e5c (RFC 8707 resource indicator forwarding).
