# Issue #145677 — [themes] Preserve my search for theme search in marketplace

[https://github.com/microsoft/vscode/issues/145677](https://github.com/microsoft/vscode/issues/145677)

## Summary

Still valid on main: the marketplace theme picker's `openQuickPick` receives the search `value` from the installed-themes picker but never applies it to the new quickpick widget, so the user's search text is lost when transitioning to marketplace browsing.

## Change summary

When selecting 'Browse Additional Color Themes' from the theme picker, the user's search text should be preserved in the marketplace search quick pick.

## Best solution

In `MarketplaceThemesPicker.openQuickPick`, set `quickpick.value = value` after creating the quickpick (and trigger the initial search with that value). The fix is localized to `src/vs/workbench/contrib/themes/browser/themes.contribution.ts`.
