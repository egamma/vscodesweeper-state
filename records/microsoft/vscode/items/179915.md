# Issue #179915 — notebook search: output matches not disappearing when dismissed 

[https://github.com/microsoft/vscode/issues/179915](https://github.com/microsoft/vscode/issues/179915)

## Summary

Still reproduces on main: dismissing an individual notebook output search result updates the model and editor decorations but does not remove the corresponding webview highlight, because no API exists to remove a single output highlight. Team-authored issue — keeping open.

## Change summary

Dismissing a notebook output search result in the search sidebar does not remove the orange highlight decoration in the rendered output.

## Best solution

Add a mechanism to remove individual webview output highlights (not just all-or-nothing via `findStop`). The `removeMatch` path in `notebookSearchModel.ts` calls `setNotebookFindMatchDecorationsUsingCellMatches` which updates editor decorations but leaves webview highlights untouched. Either extend `setAllFindMatchesDecorations` to reconcile webview highlights, or call `stopWebviewFind` followed by re-highlighting the remaining matches. The fix likely touches `findMatchDecorationModel.ts`, `notebookSearchModel.ts`, and potentially `webviewPreloads.ts` / `backLayerWebView.ts`.
