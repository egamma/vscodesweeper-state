# Issue #306017 — Copilot does not realize it's in ask mode

[https://github.com/microsoft/vscode/issues/306017](https://github.com/microsoft/vscode/issues/306017)

## Summary

The reporter could not provide the request logs the maintainer asked for, and the report is too thin to verify against current main. Meanwhile, the codebase has since added explicit tool-permission filtering in Ask mode (`isPermitted` in `languageModelToolsService.ts`) that restricts tools to read/search/web toolsets when agent mode is disabled — which directly addresses the described symptom. Keeping as info-needed to confirm whether the issue persists on a current build.

## Change summary

When switching a Copilot session from Agent to Ask mode mid-conversation, Copilot still attempted to call file-editing tools and got confused by the resulting errors.

## Best solution

The `isPermitted` gate in `src/vs/workbench/contrib/chat/browser/tools/languageModelToolsService.ts` now blocks execute/edit toolsets when agent mode is disabled, which should prevent the model from attempting file edits in Ask mode. Ask the reporter to confirm on the latest Insiders build — if it still reproduces, request logs per https://github.com/microsoft/vscode/wiki/Copilot-Issues#language-model-requests-and-responses.
