# Issue #206099 — Git - Open File should be in core, not the git extension

[https://github.com/microsoft/vscode/issues/206099](https://github.com/microsoft/vscode/issues/206099)

## Summary

Team-authored debt issue; the 'Open File' command remains in extensions/git/src/commands.ts on current main and has not been moved to core. Still valid.

## Change summary

Move the diff-editor 'Open File' command from the git extension into the core so it works for all diff scenarios, not just git-sourced ones.

## Best solution

Implement a core 'Open File' action in the diff-editor contribution (likely src/vs/workbench/contrib/multiDiffEditor/ or the diff-editor contrib area) that resolves the underlying resource, then have the git extension delegate to it or remove its version. This eliminates the coupling that caused bugs like #196084.
