# Issue #206099 — Git - Open File should be in core, not the git extension

[https://github.com/microsoft/vscode/issues/206099](https://github.com/microsoft/vscode/issues/206099)

## Summary

Team-authored (hediet) architectural debt issue — the 'Open File' command in the diff editor is still implemented in the git extension rather than core. Hard keep-open applies.

## Change summary

Move the diff editor's 'Open File' command from the git extension into core so it works for all diff scenarios, not just git-related ones.

## Best solution

Implement a core 'Open File' command in the diff editor contribution (likely `src/vs/workbench/contrib/multiDiffEditor/` or `src/vs/editor/browser/widget/diffEditor/`) that resolves the original (non-diff) resource and opens it. The git extension's `git.openFile` command can then delegate to or be replaced by the core version. This prevents bugs like #196084 where the command fails for non-git diff scenarios.
