# Issue #250513 — navigating through prompt history with a mcp image resource will cause a delay

[https://github.com/microsoft/vscode/issues/250513](https://github.com/microsoft/vscode/issues/250513)

## Summary

Still present on main: navigateHistory() awaits restoreAttachments() which re-fetches image binary data over the network before displaying the prompt text, causing a visible delay. Team-authored issue — keeping open.

## Change summary

Navigating through prompt history with an MCP image resource attachment causes a delay because image binary data is re-fetched synchronously before the prompt text is displayed.

## Best solution

In chatInputPart.ts's navigateHistory(), decouple the text restoration from the image re-fetch: set the input text and cursor position immediately, then restore attachments asynchronously in the background so the user sees the prompt text without waiting for the network round-trip.
