# Issue #41860 — Allow to "Scope to this" in explorer

[https://github.com/microsoft/vscode/issues/41860](https://github.com/microsoft/vscode/issues/41860)

## Summary

Well-established feature request with 171 upvotes, in Backlog milestone, and serving as the canonical issue for explorer folder scoping. Not implemented on main — keeping open.

## Change summary

Allow right-clicking a folder in the Explorer to scope/isolate the tree view to only that folder's contents.

## Best solution

Implement a native 'Scope to This' context menu action in the file explorer that filters the tree view to the selected folder, with a breadcrumb or button to navigate back up. This should not manipulate `files.exclude` (as the community extension workaround does) but use a dedicated view filter. Consider also scoping Quick Open (Ctrl+P) and new terminal CWD when a scope is active, as commenters suggest.
