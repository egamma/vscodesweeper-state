# Issue #78893 — [html] autoclose not working properly with multicursor

[https://github.com/microsoft/vscode/issues/78893](https://github.com/microsoft/vscode/issues/78893)

## Summary

Still reproduces on main: the autoclose provider is called with only the last change's position, then its single result is inserted at all cursor positions, so all cursors get the same closing tag.

## Change summary

HTML auto-close with multicursor inserts the same closing tag (from the last cursor's context) at all cursor positions instead of computing each independently.

## Best solution

In `extensions/html-language-features/client/src/autoInsertion.ts`, the `doAutoInsert` function should call the provider for each cursor position independently and insert per-cursor results, rather than computing once and broadcasting.
