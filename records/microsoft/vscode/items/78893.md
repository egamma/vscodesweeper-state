# Issue #78893 — [html] autoclose not working properly with multicursor

[https://github.com/microsoft/vscode/issues/78893](https://github.com/microsoft/vscode/issues/78893)

## Summary

Still reproduces on main: the auto-close logic only queries the provider for one cursor position (the last change) and applies the same closing tag text to all selections, so multicursor autoclosing always produces the tag from the last cursor.

## Change summary

HTML auto-close with multicursor inserts the same closing tag (from the last cursor's context) at all cursor positions instead of computing the correct tag per cursor.

## Best solution

Refactor `doAutoInsert` in `extensions/html-language-features/client/src/autoInsertion.ts` to call the provider for each cursor position independently and apply position-specific closing tags, likely via `WorkspaceEdit` rather than a single `insertSnippet` call.
