# Issue #78893 — [html] autoclose not working properly with multicursor

[https://github.com/microsoft/vscode/issues/78893](https://github.com/microsoft/vscode/issues/78893)

## Summary

Still reproduces on main: the autoclose provider is called once for a single cursor position and the same closing-tag text is inserted at all selections, so multicursor autoclose always uses the tag from one position.

## Change summary

HTML auto-closing tags with multicursor inserts the same closing tag (from one cursor) at all cursor positions instead of computing each independently.

## Best solution

In `extensions/html-language-features/client/src/autoInsertion.ts`, call the provider for each selection's position independently, then apply per-position results (likely via a WorkspaceEdit rather than a single snippet insertion).
