# Issue #155795 — Remove workaround in launch config arg escaping

[https://github.com/microsoft/vscode/issues/155795](https://github.com/microsoft/vscode/issues/155795)

## Summary

Team-authored debt item tracking removal of a `< >` escaping workaround in debug terminal arg building. The workaround is still present on current main (lines 115, 153, 188 of terminals.ts), and the upstream dependency (vscode-cpptools#10654) may still be pending. Keeping open per team-author rule.

## Change summary

Remove the special-case workaround that skips escaping args that are bare `<` or `>` in debug launch config terminal commands, once debug adapters have adopted `argsCanBeInterpretedByShell`.

## Best solution

Once the cpptools adapter (and other relevant debug adapters) have adopted `argsCanBeInterpretedByShell` from DAP, remove the `a === '<' || a === '>'` guards on lines 115, 153, and 188 of `src/vs/workbench/contrib/debug/node/terminals.ts` so all args go through the normal `quote()` path unless `argsCanBeInterpretedByShell` is set.
