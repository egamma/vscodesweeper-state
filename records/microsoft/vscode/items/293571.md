# Issue #293571 — Restart causes race condition in diff editor

[https://github.com/microsoft/vscode/issues/293571](https://github.com/microsoft/vscode/issues/293571)

## Summary

Still reproduces on main: ScmHistoryItemResolver.resolveDiffSource does not wait for the SCM repository to become available, unlike its sibling ScmMultiDiffSourceResolver which uses waitForState. On restart, this race yields empty results shown as 'No Changed Files'.

## Change summary

Restarting VS Code with a git commit diff editor open shows 'No Changed Files' instead of waiting for the git extension to register its repository.

## Best solution

Add a waitForState call in ScmHistoryItemResolver.resolveDiffSource (similar to ScmMultiDiffSourceResolver) so it waits for the repository to be registered before querying the history provider.
