## Best solution

The reporter or a reproducer should test with `--disable-extensions` to rule out the Python extension's terminal environment modifications (the most plausible VS Code-side cause per the July 2026 comment). If it reproduces without extensions, a `VSCODE_TERMINAL_*` environment diff between a working external terminal and VS Code's terminal would help isolate the core defect. Without that, this is likely `*caused-by-extension` (ms-python.python) or a Zephyr SDK/CMake toolchain issue.
