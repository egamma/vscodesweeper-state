# Issue #129530 — Disassembly view: do not piggy back on existing step commands, introduce new ones

[https://github.com/microsoft/vscode/issues/129530](https://github.com/microsoft/vscode/issues/129530)

## Summary

Still valid on current main: all four step commands (stepOver, stepInto, stepOut, stepBack) check CONTEXT_DISASSEMBLY_VIEW_FOCUS inside their handlers to branch between instruction and source stepping. The requested refactor to separate commands has not been done.

## Change summary

Refactor disassembly view stepping to use dedicated step commands instead of piggy-backing on existing step commands with runtime context checks.

## Best solution

Introduce dedicated instruction-stepping commands (e.g. stepOverInstruction, stepIntoInstruction, stepOutInstruction, stepBackInstruction) with their own when-clauses scoped to CONTEXT_DISASSEMBLY_VIEW_FOCUS, and remove the context-key checks from the existing step command handlers in `src/vs/workbench/contrib/debug/browser/debugCommands.ts`.
