# Issue #302346 — Support opening file/line references in an external editor from chat responses in github copilot

[https://github.com/microsoft/vscode/issues/302346](https://github.com/microsoft/vscode/issues/302346)

## Summary

This requests that VS Code route its own internal file references (from Copilot Chat) to an external editor — which runs counter to VS Code's core UX model. VS Code is the editor; opening its own references externally is out of scope for core, and no community traction supports it.

## Change summary

Add a configurable option to open file/line references from Copilot Chat responses in an external editor instead of VS Code's built-in editor.

## Best solution

This is fundamentally at odds with VS Code's design as a self-contained editor — Copilot Chat file references are a core navigation mechanism and should open in VS Code. An extension could theoretically intercept editor open calls and redirect them, but this does not belong in core. Close as out-of-scope.
