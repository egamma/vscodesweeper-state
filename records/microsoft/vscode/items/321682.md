# Issue #321682 — It says files are ignored when they are not.

[https://github.com/microsoft/vscode/issues/321682](https://github.com/microsoft/vscode/issues/321682)

## Summary

Valid feature request: the error message 'File … is configured to be ignored by Copilot' doesn't tell the user *why* or *which rule* caused the exclusion. The user spent significant effort debugging before discovering the rules came from their GitHub organization's content exclusion policy. Requesting better transparency — listing active rules and their source (organization vs repository vs .copilotignore).

## Change summary

Surface the source of content exclusion rules (organization, repository, or .copilotignore) in error messages and provide a way for users to list active exclusion rules.

## Best solution

Enhance the error message in `extensions/copilot/src/extension/tools/node/toolUtils.ts` (`assertFileNotContentExcluded`) to include the rule source (organization/repository name), which is already available in the `ContentExclusionRule.source` field in `remoteContentExclusion.ts`. Additionally, consider exposing the active exclusion rules via the existing `ignoreInfoFileContentProvider.ts` mechanism or a dedicated command so users can inspect them without resorting to dev tools.
