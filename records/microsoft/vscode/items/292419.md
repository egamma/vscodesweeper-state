# Issue #292419 — Workspace Trust - Double trust request for untrusted workspace, different dialogue content

[https://github.com/microsoft/vscode/issues/292419](https://github.com/microsoft/vscode/issues/292419)

## Summary

The core complaint — a redundant startup trust dialog appearing after the user already declined trust when adding a folder — has been resolved on main. Commit b049a8623ca changed the default for `security.workspace.trust.startupPrompt` from `'once'` to `'never'`, so the startup dialog no longer appears by default. Shipped in 1.126.0.

## Change summary

When a user declines trust for a newly added folder, VS Code shows a second, different trust dialog on the next restart, creating redundant prompting with inconsistent dialog content.

## Best solution

Keep the shipped default change (startupPrompt → 'never') from commit b049a8623ca. The startup prompt is now suppressed by default, eliminating the double-dialog scenario. Users who want the startup prompt can still opt in via the setting.
