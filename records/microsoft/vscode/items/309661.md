# Issue #309661 — YAML support for VSCode tasks

[https://github.com/microsoft/vscode/issues/309661](https://github.com/microsoft/vscode/issues/309661)

## Summary

This is a re-request of the YAML configuration format support that was explicitly declined in #4851 as a product decision (JSON is the chosen format). The rationale — uniform tooling, schema validation, IntelliSense — still applies. Proposing duplicate close against the canonical decision.

## Change summary

Allow VS Code tasks to be defined in a YAML file (tasks.yml) as an alternative to tasks.json.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/4851 — this is the same request revisited. The product decision to use JSON for configuration files was deliberate and the ecosystem (JSON Schema, IntelliSense, JSONC comments) is built around it. If this decision were to be revisited, it would be a broad product-direction call, not a per-file-type addition.

## Vision fit

- **Fit:** rejected
- **Reason:** Adding an alternative configuration format adds complexity and diverges from VS Code's uniform UI/interaction model; the existing extension API could theoretically support this as a third-party contribution.
- **Evidence:** Values: 'Simple, uniform UI and UI interactions' and 'The power lives in extensions' — alternative config formats add surface area that could live in an extension rather than core.
