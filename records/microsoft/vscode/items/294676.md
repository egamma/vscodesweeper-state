# Issue #294676 — I set `chat.disableAIFeatures = true`, but it's still constantly requesting `https://api.github.com/copilot/mcp_registry` and it's still stuck on a 429 status. How can I disable it?

[https://github.com/microsoft/vscode/issues/294676](https://github.com/microsoft/vscode/issues/294676)

## Summary

Valid bug: on current main, `chat.disableAIFeatures` does not gate the MCP registry fetch in `defaultAccount.ts`, and 429 responses are treated as 'no registry available' (clearing cached org config). A fix exists on `origin/dev/dmitriv/mcp-registry-negative-cache` but has not merged to main.

## Change summary

Setting `chat.disableAIFeatures = true` does not suppress MCP registry requests to `api.github.com/copilot/mcp_registry`, and 429 rate-limit responses incorrectly clear cached org MCP configuration.

## Best solution

Land the pending work on the `dev/dmitriv/mcp-registry-negative-cache` branch which adds an `isAIDisabled()` gate before the MCP fetch, moves the fetch to a non-blocking background path, and caches negative results for 6 hours. The 429-clears-cache problem should also be addressed by treating 429 as a transient failure that preserves existing cached data rather than a definitive 'no registry' signal.
