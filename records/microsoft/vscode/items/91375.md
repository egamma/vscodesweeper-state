[https://github.com/microsoft/vscode/issues/91375](https://github.com/microsoft/vscode/issues/91375)

## Summary

This should stay open for maintainer judgment because it is a team-authored Backlog feature request about exposing semantic token providers through an execute command. The discussion also shows the API shape was unresolved, especially how callers would receive or interpret the semantic token legend.

## Change summary

Expose a `vscode.executeSemanticHighlightProvider`-style command so extensions can request semantic highlighting for documents such as virtual embedded-language documents.

## Best solution

Decide in the semantic tokens API area whether a programmatic execute command should exist and, if so, define a result shape that includes enough legend information for extensions to decode returned tokens safely.
