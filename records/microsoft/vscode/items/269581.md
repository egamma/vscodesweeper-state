# Issue #269581 — sed: command not found shows up when opening gitbash terminal

[https://github.com/microsoft/vscode/issues/269581](https://github.com/microsoft/vscode/issues/269581)

## Summary

Still reproduces on main: the shell integration script's IFS=':' splitting of VSCODE_ENV_REPLACE/PREPEND/APPEND corrupts PATH on Windows Git Bash (MSYS2) because Windows paths contain colons. Team-authored issue — keeping open.

## Change summary

Shell integration bash script corrupts PATH on Git Bash (Windows) by splitting environment variable collections on ':' which breaks Windows-format paths containing drive letters.

## Best solution

The env var processing in shellIntegration-bash.sh (lines 72–98) needs a platform-aware delimiter or encoding scheme that won't collide with Windows path colons. The sed references on lines 228 and 416 are a secondary symptom — once PATH is fixed, sed will be found. The fix lives in `src/vs/workbench/contrib/terminal/common/scripts/shellIntegration-bash.sh` and possibly the TypeScript code that serializes VSCODE_ENV_REPLACE/PREPEND/APPEND (the encoding side).
