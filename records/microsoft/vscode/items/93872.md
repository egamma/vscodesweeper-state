# Issue #93872 — [rename on type] feedback incorrect after Rename

[https://github.com/microsoft/vscode/issues/93872](https://github.com/microsoft/vscode/issues/93872)

## Summary

Team-authored bug on the On Deck milestone; the linked editing decoration issue after Rename (F2) appears still present — no fix found in git history for this specific interaction. Keeping open per policy.

## Change summary

After performing F2 Rename on a linked-editing (synced region) tag, the decoration feedback no longer highlights the entire renamed region correctly.

## Best solution

The linked editing contribution in `src/vs/editor/contrib/linkedEditing/browser/linkedEditing.ts` needs to detect when a Rename operation completes and force a full `updateRanges()` refresh so decorations are recomputed against the post-rename text. The `_syncRanges` edit minification (common prefix/suffix trimming at lines 219–227) causes stale decoration ranges when an external edit (like Rename) replaces the same region.
