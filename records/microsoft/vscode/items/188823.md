# Issue #188823 — SVGs don't seem to work for `iconPath` in `QuickPickItem`

[https://github.com/microsoft/vscode/issues/188823](https://github.com/microsoft/vscode/issues/188823)

## Summary

Team-authored issue; still reproduces on main — remote SVGs set via background-image in the quick pick icon path receive no special handling, and CSP or content-type sniffing may block them. Keeping open per hard keep-open rule (MEMBER author).

## Change summary

Remote SVG URIs passed as `iconPath` in `QuickPickItem` don't render, while PNGs work fine.

## Best solution

The quick pick icon renderer in `src/vs/platform/quickinput/browser/quickInputList.ts` uses `background-image: url(...)` which may be blocked for SVGs by CSP or Electron's content-type handling. A fix could use an `<img>` element for remote URIs or explicitly handle SVG content types, similar to how other icon renderers in the workbench handle SVGs.
