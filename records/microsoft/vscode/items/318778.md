# Issue #318778 — [Regression] [Error] unhandlederror-Edits must be disjoint and sorted

[https://github.com/microsoft/vscode/issues/318778](https://github.com/microsoft/vscode/issues/318778)

## Summary

Confirmed regression on main: `offsetEditFromContentChanges` uses a naive `reverse()` instead of sorting by offset, which breaks when undo/redo delivers content changes in non-descending order. 540 users affected on stable. An open PR (#318782) has the fix but it has not been merged yet.

## Change summary

Undo/redo triggers 'Edits must be disjoint and sorted' error because `offsetEditFromContentChanges` assumes content changes arrive in descending offset order.

## Best solution

Merge PR https://github.com/microsoft/vscode/pull/318782 (or equivalent) which replaces the naive `reverse()` with a proper sort-by-offset in `src/vs/editor/common/model/textModelStringEdit.ts`. The fix is small and localized.
