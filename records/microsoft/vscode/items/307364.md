# Issue #307364 — macOS: Child processes cannot access TCC-protected resources (Photos, Speech Recognition, Location, Contacts, etc.)

[https://github.com/microsoft/vscode/issues/307364](https://github.com/microsoft/vscode/issues/307364)

## Summary

Valid request with an open PR (#321762) by a team member taking the TCC-disclaim approach instead of adding more Info.plist usage descriptions. Keeping open while the PR is in flight.

## Change summary

macOS child processes spawned from VS Code's terminal/debug adapter cannot access TCC-protected resources (Photos, Contacts, Location, etc.) because VS Code's Info.plist lacks the corresponding NS*UsageDescription keys.

## Best solution

Land https://github.com/microsoft/vscode/pull/321762, which disclaims TCC responsibility for the pty host process — the approach deepak1556 endorsed over adding more usage descriptions to VS Code's Info.plist. Once merged, child processes should inherit TCC responsibility themselves rather than VS Code acting as the responsible process.
