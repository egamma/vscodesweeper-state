# Issue #209622 — Test Coverage Reading

[https://github.com/microsoft/vscode/issues/209622](https://github.com/microsoft/vscode/issues/209622)

## Summary

The test coverage inline decorations in the editor rely on visual-only color indicators (gutter/background) with no ARIA or accessible alternative for screen readers to convey per-line coverage status. The coverage tree view has ARIA labels, but the editor decorations do not. The 'Go to Next/Previous Uncovered Line' commands (added Oct 2025) partially help navigation but still don't announce coverage state to screen readers.

## Change summary

Screen readers cannot convey per-line test coverage status (covered/uncovered/partially covered) because the editor decorations are purely visual.

## Best solution

Add accessible descriptions or ARIA annotations to the coverage editor decorations in `src/vs/workbench/contrib/testing/browser/codeCoverageDecorations.ts` so screen readers can announce line coverage status. Alternatively, integrate with the Accessible View (Alt+F1) to provide a coverage summary for the current line/file. The related issue #221647 was closed as a duplicate of this one, confirming this is the canonical tracking issue.
