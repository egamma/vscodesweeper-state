# Issue #20467 — search in files focus resets when line changed

[https://github.com/microsoft/vscode/issues/20467](https://github.com/microsoft/vscode/issues/20467)

## Summary

Long-standing confirmed bug with 49 upvotes and an open community PR (https://github.com/microsoft/vscode/pull/255670) actively addressing it. The bug is still present on main — F4 search navigation resets position after editing a matched line. Keeping open for the PR to land.

## Change summary

When editing a file causes a search result to disappear, the focused search result resets to the beginning instead of advancing to the next valid result.

## Best solution

Review and land (or iterate on) the open community PR https://github.com/microsoft/vscode/pull/255670, which implements tracking of previous navigation positions so that F4 navigates to the correct next result after a match disappears due to editing.
