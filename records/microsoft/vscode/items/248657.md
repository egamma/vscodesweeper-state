[https://github.com/microsoft/vscode/issues/248657](https://github.com/microsoft/vscode/issues/248657)

## Summary

The spacing concern has been partially addressed by the compact toggle added in Feb 2026, but the request for a configurable number of visible icons (with overflow menu for the rest) in the top activity bar remains unimplemented. Keeping open as an incremental polish/feature request with 7 upvotes — below the 20-upvote promotion threshold.

## Change summary

Request to increase spacing between top-positioned activity bar icons and allow configuring the number of visible icons with an overflow menu for the rest.

## Best solution

The compact toggle (`workbench.activityBar.compact`) already provides a denser/roomier option. The remaining gap is user-configurable visible icon count with explicit overflow — this is a niche layout customization that could be added to the existing composite bar overflow logic in `activitybarPart.ts`, or considered part of a broader UI density exploration (see issue #168671).
