# Issue #251081 — Text editor shows unmonospaced Japanese Hiragana when using Noto Sans JP font

[https://github.com/microsoft/vscode/issues/251081](https://github.com/microsoft/vscode/issues/251081)

## Summary

Confirmed, still-open visual bug where certain Japanese hiragana characters render at proportional widths when a non-monospace fallback font (Noto Sans JP, Yu Gothic) provides the glyphs. A team member has engaged and the reporter confirmed it reproduces on Insiders and vscode.dev. No fix has landed on main.

## Change summary

Certain Japanese hiragana characters (e.g. って) render at non-monospace widths in the editor when a proportional font like Noto Sans JP is the active fallback.

## Best solution

The editor's character-width measurement and monospace layout assumptions need to handle proportional fallback fonts for CJK characters more gracefully — likely in `src/vs/editor/browser/config/fontMeasurements.ts` and the view-line rendering path. Related to https://github.com/microsoft/vscode/issues/315426 (ambiguous-width characters) but distinct: this is about proportional-font fallback glyphs, not East Asian Ambiguous Width classification. Both may benefit from the same underlying work.
