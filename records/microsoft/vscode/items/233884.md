# Issue #233884 — Vscode doesn't respect the login shell on MacOS

[https://github.com/microsoft/vscode/issues/233884](https://github.com/microsoft/vscode/issues/233884)

## Summary

Still reproduces on main: the fallback default profile selection in terminalProfileResolverService.ts matches by basename only (`path.parse(e.path).name`), so when two shells with the same basename exist in /etc/shells, VS Code picks the first one rather than the one matching the actual login shell path from `$SHELL`.

## Change summary

When multiple shells with the same basename exist in /etc/shells (e.g. two different xonsh installations), VS Code uses the wrong one instead of the actual login shell.

## Best solution

In `_getUnresolvedFallbackDefaultProfile` (src/vs/workbench/contrib/terminal/browser/terminalProfileResolverService.ts, ~line 245), prefer an exact path match against `executable` before falling back to basename matching. This ensures the correct binary is selected when multiple shells share the same name.
