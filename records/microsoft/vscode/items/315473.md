# Issue #315473 — VS Code on Windows remains attached to terminal: Ctrl+C or closing terminal kills VS Code, log output continues in terminal

[https://github.com/microsoft/vscode/issues/315473](https://github.com/microsoft/vscode/issues/315473)

## Summary

Verified duplicate of the long-standing canonical issue tracking VS Code holding onto the parent console on Windows. The described symptoms — Ctrl+C killing VS Code, log output bleeding into the terminal, closing the terminal closing VS Code — are exactly the same bug.

## Change summary

On Windows, launching VS Code from a terminal keeps the process attached to the parent console, so Ctrl+C or closing the terminal kills VS Code and log output leaks into the terminal.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/66750, which is the canonical upstream/Electron issue tracking this exact behavior. The root cause involves Electron reconnecting to the parent console on Windows; the fix likely requires changes in Electron or in how the CLI launcher detaches the process.
