## Summary

Mouse offset on Linux with fractional scaling is a known class of Electron/Chromium compositor issues, especially on Crostini/Wayland. No fix on VS Code's main, but the root cause likely lives upstream. Requesting more info to confirm the exact environment and whether Electron/Chromium workarounds help.

## Change summary

Mouse clicks register at the wrong position in VS Code on Linux (Crostini and Arch) when display scaling is below 100%.

## Best solution
