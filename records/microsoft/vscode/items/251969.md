# Issue #251969 — SCM Graph - Allow git branches graph to set auto with other branches

[https://github.com/microsoft/vscode/issues/251969](https://github.com/microsoft/vscode/issues/251969)

## Summary

Valid feature request — the SCM graph branch picker currently treats 'Auto' and 'All' as exclusive radio-style selections that clear any other chosen branches. The request to combine 'Auto' with additional pinned branches is not yet implemented on main and no canonical issue owns this work.

## Change summary

Allow combining the 'Auto' graph filter with additional manually selected branches so the graph shows auto-suggested branches plus user-pinned important branches.

## Best solution

Extend the `HistoryItemRefsFilter` type and the quick-pick selection logic in `src/vs/workbench/contrib/scm/browser/scmHistoryViewPane.ts` (around lines 1560–1585) to allow 'auto' to coexist with specific `ISCMHistoryItemRef[]` entries, then merge the auto-resolved refs with the pinned refs when building the graph query. This is a small, self-contained change in the SCM history view pane.
