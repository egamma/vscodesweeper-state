# Issue #321676 — Revisit pricing properties for Agent Host Copilot CLI/SDK scenarios

[https://github.com/microsoft/vscode/issues/321676](https://github.com/microsoft/vscode/issues/321676)

## Summary

Team-authored design discussion about lifting pricing properties out of the loosely-typed `_meta` bag into a strongly-typed shape on `IAgentModelInfo`. The current implementation in `agentModelPricing.ts` works but uses runtime type guards to read from an open `Record<string, unknown>` bag — the issue asks whether these should be first-class typed fields. Keeping open for maintainer judgment.

## Change summary

Lift pricing metadata from the untyped `_meta` property bag on agent model info into strongly-typed, dedicated properties.

## Best solution

Promote the `IAgentModelPricingMeta` fields from the open `_meta: Record<string, unknown>` bag on `IAgentModelInfo`/`SessionModelInfo` into first-class optional properties (or a dedicated `pricing` sub-object) on those interfaces. This would eliminate the runtime type-guarding in `readAgentModelPricingMeta` and `normalizeCAPIBilling`, and the same refactor could address the related ask (comment #1) to lift CAPI-specific error codes out of `_meta` as well. The work lives in `src/vs/platform/agentHost/common/agentModelPricing.ts`, `agentService.ts`, and the agent implementations under `node/copilot/` and `node/claude/`.
