# Issue #180833 — Firefox: Files in menu items (such as explorer) are opened on mouseup, not onclick

[https://github.com/microsoft/vscode/issues/180833](https://github.com/microsoft/vscode/issues/180833)

## Summary

Still reproduces on main: Firefox fires a click on mouseup after a cross-element drag, and no guard exists in the list widget or workbench button handlers. The open PR #211245 has not been merged. Keeping open with existing labels.

## Change summary

In Firefox on vscode.dev, releasing a text-selection drag over sidebar/explorer items triggers an unintended click that opens a file.

## Best solution

Review and land (or supersede) https://github.com/microsoft/vscode/pull/211245 — the fix needs a Firefox-specific guard that suppresses click events originating from a cross-element drag (mouseup without a prior mouseenter). The fix belongs in the list widget layer (src/vs/base/browser/ui/list/) or a shared DOM utility. The PR is community-authored and labeled `help wanted`, so a maintainer review would unblock it.
