# Issue #226828 — `FindFiles2OptionsNew.useIgnoreFiles` is quite complicated

[https://github.com/microsoft/vscode/issues/226828](https://github.com/microsoft/vscode/issues/226828)

## Summary

Team-authored API design discussion about simplifying the `findFiles2New` search API surface — must stay open for maintainer judgment. The discussion between two team members is ongoing and touches proposed API finalization.

## Change summary

The `useIgnoreFiles` and `useExcludeSettings` options in the `findFiles2New` extension API are overly granular; extension authors lack context to choose the right combination.

## Best solution

Simplify the `findFiles2New` API before finalization: collapse the granular ignore-file and exclude-settings options into a high-level toggle (e.g. a boolean `respectUserConfiguration` or a two-value enum) that extension authors can reason about without knowing the user's personal ignore-file setup. The search-api area owner should decide which granularity is actually needed by real extension consumers.
