## Summary

Team-authored API design discussion about simplifying the `findFiles2New` options surface ‚Äî must stay open for maintainer judgment.

## Change summary

The `useIgnoreFiles` and `useExcludeSettings` options in the proposed `findFiles2New` API expose too much internal complexity for extension authors to use effectively.

## Best solution

Simplify the `FindFiles2OptionsNew` interface: collapse the granular `useIgnoreFiles` sub-properties into a single boolean (or a simpler enum like 'respect all user filters' vs 'raw disk'), and rationalize `useExcludeSettings` to avoid exposing internal setting distinctions that extension authors cannot meaningfully choose between. The discussion in https://github.com/microsoft/vscode/issues/205692 is prior art on this same design question.
