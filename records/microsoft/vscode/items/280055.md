# Issue #280055 — Orchestrator "learns" to use unauthorized subAgents -> Restrict which Subagents can be used by runSubagent

[https://github.com/microsoft/vscode/issues/280055](https://github.com/microsoft/vscode/issues/280055)

## Summary

This feature request is implemented on main and shipped: agents can now define which subagents they are allowed to use via the `agents` header attribute in `.md` agent files, with filtering enforced in the customizations index.

## Change summary

Allow agent authors to restrict which subagents can be invoked by runSubagent, preventing the orchestrator from discovering and using unauthorized agents.

## Best solution

Keep the shipped implementation from commit 0fb4f779292. The `agents` header attribute in agent `.md` files accepts a list of allowed subagent names (or `*` for all), and the system filters accordingly. No further change needed.
