# Issue #275628 — Every tool is selected after reloading if I open the tools list too early during startup

[https://github.com/microsoft/vscode/issues/275628](https://github.com/microsoft/vscode/issues/275628)

## Summary

Team-authored bug that is still reproducible from source on main. The tool picker's `set()` method overwrites global state with only the currently-visible tools, losing disabled state for extension-provided tools that haven't loaded yet during startup.

## Change summary

Opening and closing the chat tools picker before extensions finish loading resets all tool enablement to 'selected' because the save overwrites stored state with a partial tool set.

## Best solution

The `ChatSelectedTools.set()` method (or the picker result handling) should merge the new enablement state with the existing stored state, preserving disabled entries for tools not currently present in the picker. The fix belongs in `src/vs/workbench/contrib/chat/browser/widget/input/chatSelectedTools.ts`.
