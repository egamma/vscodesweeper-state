# Issue #275628 — Every tool is selected after reloading if I open the tools list too early during startup

[https://github.com/microsoft/vscode/issues/275628](https://github.com/microsoft/vscode/issues/275628)

## Summary

Team-authored bug report — must remain open for maintainer judgment. The issue describes a race condition where disabling tools is reset when the tools list is opened before extensions finish loading.

## Change summary

Opening the chat tools list before extensions load causes all tools to become selected, overriding previously disabled tools.

## Best solution

The tools list UI should preserve the user's disabled-tools state during the extension-loading window, likely by deferring the tool selection sync until extensions have fully contributed their tools, or by not overwriting persisted state with the incomplete tool set.
