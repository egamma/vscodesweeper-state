# Issue #163080 — Support nesting threads in the debugger

[https://github.com/microsoft/vscode/issues/163080](https://github.com/microsoft/vscode/issues/163080)

## Summary

Valid feature request dependent on an upstream DAP protocol change (microsoft/debug-adapter-protocol#339) that is still open. No parent-thread support exists in the current debug code. Zero upvotes after nearly 3 years, but it's a legitimate request tied to a real protocol proposal — keeping open in Backlog.

## Change summary

Show debug threads in a tree (parent-child hierarchy) when the debug adapter exposes nesting information, useful for languages with green threads like Go goroutines.

## Best solution

This is blocked on the upstream DAP proposal (https://github.com/microsoft/debug-adapter-protocol/issues/339). Once that protocol change is accepted, VS Code's debug callstack view (`src/vs/workbench/contrib/debug/`) would need to render threads as a tree and the debug adapter protocol types would need a `parentId` field on `Thread`. Until the DAP proposal lands, this stays parked.
