# Issue #185200 — Allow EnvironmentVariableCollection to unset variables

[https://github.com/microsoft/vscode/issues/185200](https://github.com/microsoft/vscode/issues/185200)

## Summary

Valid, unimplemented feature request: EnvironmentVariableMutatorType on current main only has Replace/Append/Prepend — no Unset capability exists. The request has 14 upvotes (approaching the 20-upvote backlog threshold) and real-world pain points confirmed by multiple commenters.

## Change summary

Allow extensions to unset (remove) environment variables via EnvironmentVariableCollection, rather than only replacing, appending, or prepending.

## Best solution

Add an `Unset` value to the `EnvironmentVariableMutatorType` enum and wire it through the terminal environment mutation pipeline (environmentVariable.ts, extHostTerminalService.ts, and the shell integration scripts which already use `unset` for internal variables). This is a stable API addition — the existing enum is purely additive, and the shell integration scripts already demonstrate `unset` mechanics. A maintainer API proposal (proposed API file) would be the first step.
