# Issue #245221 — When the keyboard focus is on the "OK" button and the TAB key is pressed, the focus moves to the subscription list, but focus is not visible: A11y_VS Code_Appling filter on subscription_keyboard

[https://github.com/microsoft/vscode/issues/245221](https://github.com/microsoft/vscode/issues/245221)

## Summary

Team-authored accessibility bug still open on main — the quick-pick list's focus ring depends on `.monaco-list:focus .monaco-list-row.focused` CSS, but tabbing from the OK/Cancel button back to the list doesn't reliably restore visual focus on the previously active item. No fix found in git history.

## Change summary

In a multi-select quick-pick, tabbing from the OK button back to the subscription list does not show a visible focus indicator on the focused list item.

## Best solution

When the quick-pick list regains DOM focus via Tab, ensure the previously focused row gets a visible focus indicator. The fix likely involves ensuring `domFocus()` is called on the tree (or that the tree element receives focus) when Tab returns focus to the list area, in `src/vs/platform/quickinput/browser/quickInputController.ts` or the tab-order handling around the OK/Cancel buttons. The CSS already styles `.monaco-list:focus .monaco-list-row.focused` correctly — the issue is that DOM focus isn't landing on the `.monaco-list` element.
