# Issue #135231 — Troublesome focus behavior after Ctrl+Enter to stage/commit changes when multiple repos are open. Could also be avoided by adding a Git command to stage/commit tracked changes in multiple (all) open repos at once

[https://github.com/microsoft/vscode/issues/135231](https://github.com/microsoft/vscode/issues/135231)

## Summary

The core ask — auto-focus the next repo's SCM input after Ctrl+Enter commit — remains unimplemented on current main. The `scm.acceptInput` handler delegates to the provider command without any post-commit focus logic. Manual `focusNextInput`/`focusPreviousInput` commands were added in May 2024 (https://github.com/microsoft/vscode/commit/994893cbba3) as a partial mitigation but don't fire automatically after commit. The multi-repo stage/commit-all request is a distinct feature request. Already triaged in Backlog with correct area label; keeping open.

## Change summary

After Ctrl+Enter commits in one repo's SCM input, focus should automatically jump to the next repo's input box; additionally, a command to stage/commit across all open repos at once is requested.

## Best solution

Wire post-commit focus advancement into the `scm.acceptInput` flow (or let the SCM view pane listen for successful commits and call `focusNextInput` automatically). The multi-repo commit-all request is a separate, lower-priority feature that could reuse the existing `focusNextInput` infrastructure. Both changes live in `src/vs/workbench/contrib/scm/browser/`.
