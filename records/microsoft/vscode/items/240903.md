# Issue #240903 — When GPU rendering is enabled, editor content elements are positioned incorrectly after zooming with the mouse

[https://github.com/microsoft/vscode/issues/240903](https://github.com/microsoft/vscode/issues/240903)

## Summary

GPU rendering positioning bug after zoom is still unfixed on main. The GPU renderer tracks devicePixelRatio via window resize events and clears the atlas, but there's no specific handling for repositioning content overlays after zoom-level changes. The issue is correctly labeled and in Backlog.

## Change summary

Editor content elements rendered via GPU become mispositioned after using Ctrl+scroll zoom in/out.

## Best solution

The GPU renderer in `src/vs/editor/browser/gpu/` needs to properly recompute layout/positioning of DOM overlay elements (cursors, selections, decorations) relative to the GPU canvas after a zoom-level change. The fix likely belongs in the render strategy files or the view layer that coordinates DOM elements with the GPU canvas.
