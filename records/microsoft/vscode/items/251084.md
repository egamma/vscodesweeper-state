# Issue #251084 — [LEAKED DISPOSABLE] in extensions view

[https://github.com/microsoft/vscode/issues/251084](https://github.com/microsoft/vscode/issues/251084)

## Summary

Still unfixed on main: the DisposableStore created in queryLocal (line 338) can leak when the query is cancelled before the result is assigned to this.queryResult, since the cancelled promise discards the returned object without disposing it.

## Change summary

DisposableStore created in ServerInstalledExtensionsView.queryLocal leaks when the query is cancelled mid-flight.

## Best solution

Guard the DisposableStore in queryLocal against cancellation — either pass the cancellation token into the method and dispose on cancel, or use a finally block in the show() method to dispose the result's disposables when the cancelable promise rejects without assigning to this.queryResult.
