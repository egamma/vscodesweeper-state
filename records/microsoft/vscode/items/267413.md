# Issue #267413 — When using user defined tool sets shows wrong count

[https://github.com/microsoft/vscode/issues/267413](https://github.com/microsoft/vscode/issues/267413)

## Summary

Still reproducible on main: the tool count in the Configure Tools picker includes both the toolset entry and its expanded member tools, inflating the displayed count. The bug is acknowledged by a team member (comment 4) and tracked here. No fix has landed.

## Change summary

Selecting a user-defined toolset shows double the actual tool count because both the toolset and its expanded member tools are counted in the displayed total.

## Best solution

The tool count display should deduplicate: count each underlying tool once regardless of how many toolsets reference it, and exclude toolset entries themselves from the 'N tools' count. Fix likely lives in `chatToolActions.ts` (telemetry count) and wherever the badge/label computes the displayed number from `entriesMap.size`.
