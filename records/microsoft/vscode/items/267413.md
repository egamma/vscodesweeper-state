# Issue #267413 — When using user defined tool sets shows wrong count

[https://github.com/microsoft/vscode/issues/267413](https://github.com/microsoft/vscode/issues/267413)

## Summary

The tool picker count-doubling issue (showing toolset tools counted individually alongside the toolset) remains a valid UX bug. While the tool limit warning was removed (reducing practical impact) and qualified names support was added (PR #320265), the core count-display issue appears not fully resolved on main.

## Change summary

When a user-defined tool set is selected, the tool picker shows double the actual tool count (e.g., 38 instead of 19) because tools within a selected toolset are listed both under the toolset and individually.

## Best solution

The tool picker should count each underlying tool only once when computing the displayed total, regardless of how many toolsets reference it. The fix likely belongs in `src/vs/workbench/contrib/chat/browser/actions/chatToolActions.ts` (line 202, `tools.size`) or in how `entriesMap` in `chatSelectedTools.ts` reports its size — filtering out toolset entries from the reported count.
