# Issue #317771 — [BUG] No results returned for the `fileSearch` tool when querying a network drive, works fine when ommited

[https://github.com/microsoft/vscode/issues/317771](https://github.com/microsoft/vscode/issues/317771)

## Summary

Source-traceable bug: the `fileSearch` tool's `inputGlobToPattern` function in the Copilot extension doesn't properly handle UNC-style network drive paths. When the model generates a query like `/192.168.1.50/path/to/folder/**`, `URI.file()` fails to produce a URI matching the workspace folder, and the raw absolute path falls through as an unresolved glob that matches nothing.

## Change summary

The `fileSearch` tool returns zero results when the opened workspace is on a network drive and the model generates a UNC-style path query, because the path-to-glob conversion doesn't recognize network paths.

## Best solution

Fix `inputGlobToPattern` in `extensions/copilot/src/extension/tools/node/toolUtils.ts` to handle UNC paths: when `URI.file(query)` produces a URI with a non-empty authority (the network host), match it against workspace folders using authority-aware comparison. The `resolveInput` method in `findFilesTool.tsx` may also need adjustment so that UNC paths (e.g. `//192.168.1.50/...` or `\\server\...`) are recognized and not treated as plain absolute paths. Add test cases for UNC paths in the existing `inputGlobToPattern` test suite.
