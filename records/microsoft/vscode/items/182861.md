# Issue #182861 — Merge Editor overrides manually resolved conflicts

[https://github.com/microsoft/vscode/issues/182861](https://github.com/microsoft/vscode/issues/182861)

## Summary

Still reproduces on main: when opening a partially-resolved conflict file in the merge editor, the `resetResult` flag is set based on whether *any* conflict markers remain (line 332 of mergeEditorInputModel.ts), triggering a full `reset()` that overwrites all manual resolutions with `computeAutoMergedResult()`.

## Change summary

Opening a file with some manually-resolved conflicts in the merge editor resets the entire result, discarding previous manual resolutions.

## Best solution

The `reset()` logic should preserve regions the user already resolved manually. Either skip resetting ranges where the result text already differs from what auto-merge would produce, or only reset ranges that still contain conflict markers — not the entire document. The fix lives in `src/vs/workbench/contrib/mergeEditor/browser/model/mergeEditorModel.ts` (`reset()` method) and potentially `mergeEditorInputModel.ts` (the `resetResult` decision).
