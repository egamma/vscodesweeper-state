# Issue #224140 — Slashes in `terminal.integrated.cwd` converted to backslashes on Linux remote host and Windows host

[https://github.com/microsoft/vscode/issues/224140](https://github.com/microsoft/vscode/issues/224140)

## Summary

This is a duplicate of #172099, which tracks the same root cause: path variable resolution uses Windows separators when the frontend is Windows but the target is a Linux remote/WSL. The commenter's reproduction with `${relativeFileDirname}` in a task cwd is explicitly covered there. The original report about a literal cwd path being backslash-converted is the same path-normalization issue in the terminal launch flow.

## Change summary

Forward slashes in `terminal.integrated.cwd` and resolved path variables like `${relativeFileDirname}` are converted to backslashes when the VS Code client is on Windows but the remote host is Linux/WSL.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/172099, which is confirmed and tracks the variable-resolution path-separator bug for remote/WSL targets. The fix likely lives in the variable resolver (src/vs/workbench/services/configurationResolver/common/variableResolver.ts) which uses the local platform's `paths` module regardless of the target OS.
