# Issue #242814 — Settings Sync automatically logs out when using Personal Microsoft account - Error Code 3399614467

[https://github.com/microsoft/vscode/issues/242814](https://github.com/microsoft/vscode/issues/242814)

## Summary

Actively tracked upstream MSAL/WAM issue affecting personal Microsoft account token refresh. Maintainer is engaged, workaround exists, and the root cause lives in the OS-provided auth layer — keeping open as the tracking point.

## Change summary

Settings Sync loses authentication with personal Microsoft accounts after ~1 hour due to MSAL/WAM token refresh failures (Error Code 3399614467).

## Best solution

Continue tracking upstream with the Microsoft Identity team. The workaround (`"microsoft-authentication.implementation": "classic"`) unblocks users. Resolution depends on MSAL fixing the token refresh path for personal accounts; VS Code may need to improve error handling or fallback behavior in the microsoft-authentication extension once the root cause is identified upstream.
