# Issue #242735 — SCM Graph - Allow branch merge in git graph

[https://github.com/microsoft/vscode/issues/242735](https://github.com/microsoft/vscode/issues/242735)

## Summary

Valid feature request — the SCM Graph context menu currently offers checkout, cherry-pick, compare, and delete actions on history items, but no merge, revert, or reset. These are standard git operations that would complement the existing graph interactions. Only 4 upvotes so far; keeping open for community traction.

## Change summary

Add 'Merge into current branch', 'Revert', and 'Reset' context menu actions to the SCM Graph view's history item context menu.

## Best solution

Implement `git.graph.merge`, `git.graph.revert`, and `git.graph.reset` commands in the git extension (alongside the existing `git.graph.cherryPick`, `git.graph.checkout`, etc.) and register them in `MenuId.SCMHistoryItemContext` / `MenuId.SCMHistoryItemRefContext`. The plumbing for the underlying git operations already exists in the git extension's `commands.ts` and `repository.ts`.
