# Issue #283856 — Chat input part must not depend on `IAgentSessionsService.model`

[https://github.com/microsoft/vscode/issues/283856](https://github.com/microsoft/vscode/issues/283856)

## Summary

The cyclic dependency on `IAgentSessionsService` in the chat input part still exists on current main. The related PR #289277 was closed without merging, so the issue remains valid and actionable.

## Change summary

Remove the cyclic dependency where `chatInputPart.ts` depends on `IAgentSessionsService.model` instead of using its own chat model.

## Best solution

Surface `IChatSessionFileChange[]` (or equivalent) on the chat model itself, as connor4312 suggested — the chat input part should listen to chat model changes directly rather than reaching into the agent sessions service. The fix touches `src/vs/workbench/contrib/chat/browser/widget/input/chatInputPart.ts` (lines 4340–4465 use `agentSessionsService`) and the chat model interface.
