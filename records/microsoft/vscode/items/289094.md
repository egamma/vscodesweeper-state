# Issue #289094 — Incorrect warning "The model wants to edit files outside of your workspace" (if using symlinks?)

[https://github.com/microsoft/vscode/issues/289094](https://github.com/microsoft/vscode/issues/289094)

## Summary

The symlink-specific workspace containment bug remains unresolved. The related issue #292424 fixed a different case (non-existent files) in vscode-copilot-chat. The symlink path comparison problem — where workspace folders accessed via symlinks fail the isEqualOrParent check — is a distinct gap that likely lives in the Copilot Chat extension's toolUtils.ts rather than VS Code core.

## Change summary

Files inside symlinked workspace folders are incorrectly flagged as external when the Copilot agent tries to edit them, triggering a false 'outside your workspace' confirmation prompt.

## Best solution

The fix likely belongs in microsoft/vscode-copilot-chat's `isFileExternalAndNeedsConfirmation()` (toolUtils.ts), which should resolve symlinks before comparing paths — or VS Code core's `getWorkspaceFolder()` should canonicalize URIs through symlinks. Given the `*caused-by-extension` pattern, this may warrant transfer to the copilot-chat repo, but since VS Code core's workspace containment API is involved, keeping it here for routing is reasonable.
