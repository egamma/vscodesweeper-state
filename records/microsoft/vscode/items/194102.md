# Issue #194102 — Git - `git.requireGitUserConfig` setting has no effect

[https://github.com/microsoft/vscode/issues/194102](https://github.com/microsoft/vscode/issues/194102)

## Summary

Confirmed on current main: the bug is real and an open PR (#304983) already addresses it. The setting correctly gates `user.useConfigOnly=true` on the commit args, but `handleCommitError` independently and unconditionally checks user.name/user.email on any commit failure, masking the actual error. Keeping open pending the linked PR.

## Change summary

Setting `git.requireGitUserConfig` to `false` still shows a user.name/user.email error on commit failures because the error handler ignores the setting.

## Best solution

Land or refine open PR https://github.com/microsoft/vscode/pull/304983, which threads `requireUserConfig` into `handleCommitError` so the user-config check is skipped when the setting is `false`. The fix touches `extensions/git/src/git.ts`.
