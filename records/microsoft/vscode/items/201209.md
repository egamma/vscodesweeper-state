# Issue #201209 — When closing editors in the "Open Editors" view, the cursor should not move all the way up

[https://github.com/microsoft/vscode/issues/201209](https://github.com/microsoft/vscode/issues/201209)

## Summary

Team-authored issue (MEMBER) — must stay open for maintainer judgment. The reported behavior is traceable in current main: closing an editor in the Open Editors view triggers `updateWholeList()` followed by `focusActiveEditor()` which calls `list.reveal(index)`, scrolling away from the user's current position.

## Change summary

When closing editors via the Open Editors sidebar, the scroll position jumps to the newly active editor instead of staying in place.

## Best solution

In `openEditorsView.ts`, the `EDITOR_CLOSE` path in `onDidModelChange` calls `updateWholeList()` unconditionally. After the list refresh, `focusActiveEditor()` reveals the active editor, resetting scroll. The fix should either skip the `reveal()` call when the close was initiated from the Open Editors view, or preserve and restore the scroll position across the list refresh. The `blockFocusActiveEditorTracking` flag already exists for a similar purpose and could be extended.
