# Issue #292708 — inaccessible Source Control Graph View branch colors

[https://github.com/microsoft/vscode/issues/292708](https://github.com/microsoft/vscode/issues/292708)

## Summary

The bug is traceable in source: `historyItemHoverLabelForeground` defaults to `PANEL_BACKGROUND`, so when a theme sets a dark panel background in light mode, the branch label text inside colored chips (e.g. yellow `#FFB000`) becomes dark-on-dark and unreadable. This is a real accessibility/contrast issue that should stay open for the SCM area owner.

## Change summary

SCM Graph branch label text is unreadable in light themes with dark panel backgrounds because the label foreground color defaults to the panel background color.

## Best solution

The `scmGraph.historyItemHoverLabelForeground` color token (line 38 of `src/vs/workbench/contrib/scm/browser/scmHistory.ts`) should use a contrast-safe default instead of `PANEL_BACKGROUND` — either a per-theme light/dark split (e.g. `{ light: '#FFFFFF', dark: PANEL_BACKGROUND, hcDark: ..., hcLight: ... }`) or a computed foreground that guarantees sufficient contrast against the graph foreground colors used as chip backgrounds. The same token is consumed in `scmHistoryViewPane.ts` line 565 for badge rendering.
