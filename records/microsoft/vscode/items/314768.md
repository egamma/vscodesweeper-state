## Summary

The merge editor uses plain CodeEditorWidgets (not DiffEditorWidget), so it reads `editor.wordWrap` rather than `diffEditor.wordWrap`. The report only confirms setting `diffEditor.wordWrap` to `off` — need to know the user's `editor.wordWrap` value to determine if this is a real bug or expected behavior.

## Change summary

Word wrap appears enabled in the merge editor's side-by-side view despite `diffEditor.wordWrap` being set to `off`.

## Best solution

Clarify whether `editor.wordWrap` is also set to `off`. If so, this is a real bug in the merge editor's option propagation (likely in `src/vs/workbench/contrib/mergeEditor/browser/view/mergeEditor.ts` → `applyOptions`). If `editor.wordWrap` is set to `on`/`bounded`/`wordWrapColumn`, the behavior is as-designed — the merge editor isn't a diff editor and doesn't consult `diffEditor.wordWrap`. A possible enhancement would be to have the merge editor also respect `diffEditor.wordWrap`.
