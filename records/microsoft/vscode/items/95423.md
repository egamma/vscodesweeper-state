# Issue #95423 — ${file} - must be the current opened file, but points to the name of output pane

[https://github.com/microsoft/vscode/issues/95423](https://github.com/microsoft/vscode/issues/95423)

## Summary

Still reproduces on main: the ext host variable resolver (`extHostVariableResolverService.ts`) calls `editorService.activeEditor()` without filtering by URI scheme, so `${file}` resolves to the output pane when it has focus. The renderer-side resolver correctly filters by scheme but the ext host path used by debug does not. A 2020 fix (f9c6fa31616) was partially reverted (ab8267f7f4a) because it broke WSL remote, and the core defect was never re-addressed.

## Change summary

`${file}` in launch.json resolves to the output pane name instead of the active code editor file when the output pane has focus.

## Best solution

Add URI scheme filtering in `src/vs/workbench/api/common/extHostVariableResolverService.ts` `getActiveUri()` to skip non-file editors (mirroring the `filterByScheme` approach in `baseConfigurationResolverService.ts`), or fall back to the active tab API when `activeEditor()` returns a non-file editor. Must not regress remote/WSL scenarios (the reason the 2020 fix was reverted).
