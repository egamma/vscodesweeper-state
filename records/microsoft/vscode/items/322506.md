# Issue #322506 — [Feature Request] Allow configuring Copilot agent/skill/instruction discovery paths via settings

[https://github.com/microsoft/vscode/issues/322506](https://github.com/microsoft/vscode/issues/322506)

## Summary

Valid feature request — the agent host discovery paths in sessionCustomizationDiscovery.ts are hardcoded constants with no user-facing setting to customize them. The workbench side already has configurable paths (chat.agent.fileLocations), but the agent host side lacks an equivalent.

## Change summary

Allow users to configure the directories the agent host scans for agents, skills, instructions, and hooks via a VS Code setting.

## Best solution

Expose searchRoots and fixedDiscoveryFiles in sessionCustomizationDiscovery.ts as a restricted workspace-trust setting (similar to chat.agent.fileLocations), defaulting to the current hardcoded values so behavior is preserved without configuration.
