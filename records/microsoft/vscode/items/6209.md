# Issue #6209 — isWatching/background task as preLaunchTask in launch.json

[https://github.com/microsoft/vscode/issues/6209](https://github.com/microsoft/vscode/issues/6209)

## Summary

The core request — allowing background begin/end patterns at the task level without requiring a dummy problemMatcher — remains unimplemented on current main. With 61 upvotes, On Deck milestone, and team acknowledgment, this should stay open.

## Change summary

Allow defining background task begin/end patterns (beginsPattern, endsPattern) directly on the task definition instead of requiring an artificial problemMatcher with a dummy pattern.

## Best solution

Add a `background` property to the task schema (as sketched by @dbaeumer in comment 14) with `activeOnStart`, `beginsPattern`, and `endsPattern` fields, parsed in `src/vs/workbench/contrib/tasks/common/taskConfiguration.ts` alongside the existing `isBackground` handling. This eliminates the need for a dummy problemMatcher pattern when using background preLaunchTasks.
