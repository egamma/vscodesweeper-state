[https://github.com/microsoft/vscode/issues/198122](https://github.com/microsoft/vscode/issues/198122)

## Summary

Team-authored architectural feature request about systematic z-index management in the editor; must remain open for maintainer judgment. The related issue https://github.com/microsoft/vscode/issues/229941 suggests this design problem is still active.

## Change summary

Editor widgets (sticky scroll, inline chat, PR comments) compete for z-index stacking order with no systematic assignment mechanism.

## Best solution

Introduce a centralized z-index context system in the editor layer so that important widgets (sticky scroll) always stack above other editor widgets without manual index bumping. The open tracking issue https://github.com/microsoft/vscode/issues/229941 may be the canonical place for this work.
