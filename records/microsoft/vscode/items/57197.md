# Issue #57197 — Identation rules are broken

[https://github.com/microsoft/vscode/issues/57197](https://github.com/microsoft/vscode/issues/57197)

## Summary

The auto-indentation logic in moveLinesCommand.ts still applies indentation adjustments via getIndentMetadata/getGoodIndentForLine when moving lines, even when a language's decreaseIndentPattern should never match. The bug remains unfixed on main and is correctly tagged as a tree-sitter-autoindent-candidate.

## Change summary

Moving a line up or down changes indentation unexpectedly, even when the language's decreaseIndentPattern is set to a regex that can never match.

## Best solution

The move-lines auto-indent logic in `src/vs/editor/contrib/linesOperations/browser/moveLinesCommand.ts` needs to properly respect the language's indentation rules — specifically, `getIndentMetadata` at line 284 applies the DECREASE_MASK without validating whether the decreaseIndentPattern actually matched the content. This is likely to be addressed holistically by the tree-sitter autoindent work (per the label).
