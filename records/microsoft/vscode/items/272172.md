# Issue #272172 — VS Code User Installer on Windows 11 always opens as Administrator despite normal install

[https://github.com/microsoft/vscode/issues/272172](https://github.com/microsoft/vscode/issues/272172)

## Summary

VS Code correctly detects and displays elevated status via the `native-is-elevated` module — the "(Administrator)" label reflects a real OS-level elevation. The report lacks the Windows configuration details needed to determine why the process is being elevated; requesting more info.

## Change summary

VS Code User Installer on Windows 11 always launches as Administrator, showing "(Administrator)" in the title bar despite no explicit elevation.

## Best solution

This is almost certainly a Windows environment issue (UAC disabled, built-in Administrator account, or a group policy). The reporter needs to confirm their UAC level, whether they're using the built-in Administrator account, and whether other apps also show elevated. If confirmed as a VS Code-side issue, the relevant code is in `src/vs/platform/native/electron-main/nativeHostMainService.ts` (the `native-is-elevated` check) and `src/vs/workbench/electron-browser/window.ts` (where `isAdmin` is surfaced to the title bar).
