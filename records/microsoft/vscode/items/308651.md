# Issue #308651 — `⎇` → ` Alley`

[https://github.com/microsoft/vscode/issues/308651](https://github.com/microsoft/vscode/issues/308651)

## Summary

Copilot inline edits model is misinterpreting the Unicode character ⎇ (U+2387, Alternative Key Symbol) and repeatedly suggesting to replace it with " Alley". This is a model/service-side quality issue rather than a VS Code code defect, but it's in-scope since Copilot ships with VS Code. Routing to the inline suggestions quality area.

## Change summary

Copilot inline edits repeatedly suggests replacing the Unicode character ⎇ with " Alley" throughout the codebase.

## Best solution

This is a model tokenization/quality issue — the model doesn't properly handle the ⎇ character and hallucinates a replacement. The inline-suggestions-quality team should investigate whether there's a filtering/guardrail that can suppress edits that only replace Unicode characters with ASCII text, or flag this to the model/science team as a tokenization defect.
