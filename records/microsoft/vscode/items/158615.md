# Issue #158615 — "Run VS Code outside the Developer Command Prompt" instructions fail if opened file's basename and/or dirname contains space characters

[https://github.com/microsoft/vscode/issues/158615](https://github.com/microsoft/vscode/issues/158615)

## Summary

The bug remains on current main — cmd.exe shell quoting in tasks breaks when both the shell args (VsDevCmd.bat path) and task args (file paths) contain spaces, because the outer-quote wrapping logic at line 1717 of terminalTaskSystem.ts only accounts for the built command line, not the prepended shell args that are injected separately.

## Change summary

Task shell commands with cmd.exe fail when both shell option args and task args contain paths with spaces, due to incorrect nested quoting.

## Best solution

Fix the cmd.exe quoting logic in `_buildShellCommandLine` (or the shell-args assembly around line 1269) to properly wrap the entire composed command line — including prepended shell args — in outer quotes when both segments contain quoted strings. The workaround comment (#5) of adding manual opening/closing quotes confirms the missing outer-quote layer.
