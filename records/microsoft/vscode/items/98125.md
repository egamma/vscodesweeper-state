# Issue #98125 — Allow extensions to fully control workspace symbol search (matching and highlights)

[https://github.com/microsoft/vscode/issues/98125](https://github.com/microsoft/vscode/issues/98125)

## Summary

Still valid: the workspace symbol API has no mechanism for extensions to supply their own match highlights or bypass VS Code's built-in fuzzy scoring. The request has 82 upvotes and is in Backlog — it should stay open.

## Change summary

Allow extensions providing workspace symbols to return match positions/highlights so VS Code doesn't re-filter results using only the `name` field.

## Best solution

Extend the `WorkspaceSymbolProvider` API (or `SymbolInformation`) with optional match ranges or a `filterText` field, and let VS Code skip its own fuzzy scoring when the provider supplies pre-computed highlights. The work lives in `src/vs/workbench/contrib/search/browser/symbolsQuickAccess.ts` (consumer) and `src/vscode-dts/vscode.d.ts` (API surface).
