## Change summary

Source Control view in Remote-WSL does not reflect file changes on /mnt/c paths until a manual Developer: Reload Window, even though the file explorer updates correctly.

## Best solution

Continue the active triage: the problem is narrowed to SCM/git change detection on WSL /mnt/c mounts, not the general file watcher (Explorer works). The maintainer should determine whether the git file watcher (distinct from the workspace file watcher) is missing events on drvfs mounts, or whether the SCM refresh is not being triggered. Likely involves `src/vs/workbench/contrib/scm/` or the built-in `extensions/git/` extension's file-system watcher configuration for WSL paths.
