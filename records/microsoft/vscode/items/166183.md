# Issue #166183 — Wrong session restored after main window closed while another instance was running in debugger

[https://github.com/microsoft/vscode/issues/166183](https://github.com/microsoft/vscode/issues/166183)

## Summary

Still reproducible on main: when the debug (extension development) window is the last to close, its state is stored as `lastClosedState` without checking `isExtensionDevelopmentHost`, which causes it to be restored as the `lastActiveWindow` on next launch.

## Change summary

Closing the main window while an extension development host window is running causes the wrong session (the debug window's) to be restored on next launch.

## Best solution

Guard the `lastClosedState` assignment in `windowsStateHandler.ts` line ~254 to skip extension development host windows, similar to the guard already present at lines 234 and 165–166.
