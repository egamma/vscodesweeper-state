## Summary

Team-authored bug (MEMBER) — must keep open per policy. No commit since Sep 2024 specifically addresses this double-render / wrong-position issue; the suspected missing-dispose path remains plausible.

## Change summary

Merge editor code lenses render twice or at the wrong location after scrolling, likely due to a missing dispose when triggered from the Pull Request Overview's Resolve Conflicts button.

## Best solution

Investigate the lifecycle of CommandViewZone instances created in `src/vs/workbench/contrib/mergeEditor/browser/view/viewZones.ts` — ensure previous view-zone widgets are properly disposed before recomputation, especially in the PR-triggered flow. The ConflictActionsFactory.createWidget return value likely needs tracking in a DisposableStore that's cleared on each recompute cycle.
