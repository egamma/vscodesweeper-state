## Summary

Legitimate, well-supported feature request (81 upvotes) that has not been implemented on main. A community PR (#274988) was closed without merging. The diff editor still has no EOL-change visualization.

## Change summary

The diff editor should visualize line-ending differences (CRLF vs LF) rather than silently hiding them.

## Best solution

Implement EOL-change highlighting in the diff editor (likely in `src/vs/editor/browser/widget/diffEditor/`) when `diffEditor.ignoreTrimWhitespace` is false, similar to the approach in the closed PR #274988. The feature has strong community demand and real workflow impact for mixed-EOL repositories.
