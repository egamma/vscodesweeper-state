# Issue #142796 — Task that runs on folderOpen also always opens new terminal

[https://github.com/microsoft/vscode/issues/142796](https://github.com/microsoft/vscode/issues/142796)

## Summary

Still a valid race condition bug: when a task with `runOn: folderOpen` triggers, `_initializeTerminal` in terminalView.ts can fire before the task's terminal group is created, causing `groups.length === 0` to be true and spawning an unwanted shell terminal. Confirmed by a team member (aeschli) in Nov 2023 after the earlier partial fix.

## Change summary

A task configured with `runOn: folderOpen` causes an unwanted extra shell terminal to open alongside the task terminal due to a race condition in terminal initialization.

## Best solution

The fix belongs in `src/vs/workbench/contrib/terminal/browser/terminalView.ts` — the `_initializeTerminal` method needs to be aware that a task is about to create a terminal group (e.g. by checking pending automatic tasks or deferring terminal creation until after task startup completes). The `hideOnStartup: 'whenEmpty'` setting is a workaround but not a proper fix.
