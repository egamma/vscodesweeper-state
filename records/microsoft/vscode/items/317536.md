# Issue #317536 — GitHub Copilot Chat extension is sometimes present in extension list, sometimes not

[https://github.com/microsoft/vscode/issues/317536](https://github.com/microsoft/vscode/issues/317536)

## Summary

Real unfixed inconsistency: `--list-extensions` shows builtin auto-update extensions (like copilot-chat) only when a user-installed copy exists (post-update), but not after a VS Code update ships a newer bundled version and removes the stale user copy. The `listExtensions` method has no filter for `builtInExtensionsEnabledWithAutoUpdates`.

## Change summary

`code --list-extensions` inconsistently includes `github.copilot-chat` depending on whether the auto-updated user copy or the bundled version is active.

## Best solution

Filter out extensions in `builtInExtensionsEnabledWithAutoUpdates` from the `listExtensions` output in `extensionManagementCLI.ts`, similar to how install/uninstall already guards against them (commit a6e49120535). Alternatively, always include them — either way, make it consistent.
