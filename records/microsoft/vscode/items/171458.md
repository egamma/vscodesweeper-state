# Issue #171458 — Leaving duplicate workspace nodes When refreshing

[https://github.com/microsoft/vscode/issues/171458](https://github.com/microsoft/vscode/issues/171458)

## Summary

Still unfixed on main: the explorer service reacts to FS provider deregistration during page refresh without checking if the workbench is shutting down, causing a transient visual glitch (duplicate/broken workspace nodes). The root cause was thoroughly analyzed in the comments but no guard was added.

## Change summary

Refreshing the page on vscode.dev causes the file explorer to briefly show duplicate or broken workspace nodes because the FS provider disposal triggers a tree re-render just before the page unloads.

## Best solution

Guard the `onDidChangeFileSystemProviderRegistrations` handler in `src/vs/workbench/contrib/files/browser/explorerService.ts` (line ~114) to skip `setTreeInput` when the workbench lifecycle is in a shutdown phase — e.g. check `ILifecycleService.phase` or listen for `onBeforeShutdown` and set a flag. This is a small, localized fix.
