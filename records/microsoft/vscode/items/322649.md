# Issue #322649 — Guardrail status in the Agent window

[https://github.com/microsoft/vscode/issues/322649](https://github.com/microsoft/vscode/issues/322649)

## Summary

The Agents window already ships substantial guardrail/check-status infrastructure on current main (CIStatusWidget, BlockedSessions, session input banners for CI failures, a /fix-ci command, and blocked-session reasons). Several of the listed bullets — whether checks ran, pass/fail status, why the agent cannot finish — appear largely addressed. However, the request for distinguishing advisory vs blocking checks and surfacing what was auto-fixed may represent real remaining gaps worth tracking. Keeping open for area-owner judgment on the delta.

## Change summary

Show visible guardrail status in the Agents window so users can see whether required checks ran, passed/failed, what was auto-fixed, and why an agent cannot finish.
