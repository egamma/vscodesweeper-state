# Issue #269812 — Git - Respect submodule names defined in superrepo's `.gitmodules` file

[https://github.com/microsoft/vscode/issues/269812](https://github.com/microsoft/vscode/issues/269812)

## Summary

Valid feature request with an open PR (#269902) that addresses it. The issue already carries correct labels (`feature-request`, `git`, `help wanted`) and sits in the Backlog milestone. Keep open while the PR is under review.

## Change summary

When a submodule has a custom name in `.gitmodules` that differs from its directory basename, the SCM panel should display the configured name instead of just the directory name.

## Best solution

Review and land (or close) the open PR https://github.com/microsoft/vscode/pull/269902, which wires the configured submodule name from `.gitmodules` into `ISCMProvider.name` when the name differs from the path. The submodule name is already parsed correctly by `parseGitmodules` in `extensions/git/src/git.ts`; the gap is in how the SCM provider label is set (currently `path.basename(repository.root)` in `extensions/git/src/model.ts`).
