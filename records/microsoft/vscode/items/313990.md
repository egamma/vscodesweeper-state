  > Thanks for the question! The mechanism you're looking for already exists — it's the `chat.defaultConfiguration` setting rather than `chat.permissions.default`. The two settings are intentionally separate:
  > 
  > - **`chat.permissions.default`** controls permission levels for **Local** sessions only.
  > - **`chat.defaultConfiguration`** (with its `approvals` property) controls the starting approval behavior for **agent sessions** like Copilot CLI.
  > 
  > You can set `"chat.defaultConfiguration": { "approvals": "autoApprove" }` (or `"default"`) in your user settings to control the default permission level for new Copilot CLI sessions, including those created by handoff.
  > 
  > If this doesn't address your scenario, please let us know and we can reopen. Note: issues labeled `info-needed` auto-close after 7 days without a reply.
