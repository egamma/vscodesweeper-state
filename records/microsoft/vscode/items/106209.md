# Issue #106209 — Line highlight on search results doesn't work

[https://github.com/microsoft/vscode/issues/106209](https://github.com/microsoft/vscode/issues/106209)

## Summary

Hard keep-open: milestone is 'On Deck'. The bug is confirmed still present on main — `RangeHighlightDecorations` removes the search line highlight on any `onDidChangeDecorations` event, so any extension (or VS Code itself) that updates decorations immediately clears it.

## Change summary

Search result line highlight disappears immediately when any other decoration changes on the model.

## Best solution

Replace the `onDidChangeDecorations` listener in `src/vs/workbench/contrib/search/browser/searchTreeModel/rangeDecorations.ts` with an `onDidChangeCursorPosition` approach (as used by the editor's own `RangeHighlightDecorations` in `src/vs/workbench/browser/codeeditor.ts`). This requires moving the logic to `/browser/` or injecting a cursor-change callback, as discussed in comment 14–15.
