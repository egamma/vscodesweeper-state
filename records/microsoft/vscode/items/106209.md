# Issue #106209 — Line highlight on search results doesn't work

[https://github.com/microsoft/vscode/issues/106209](https://github.com/microsoft/vscode/issues/106209)

## Summary

Still reproduces on current main: the search range-highlight decoration is immediately cleared by any other decoration change because the search-specific RangeHighlightDecorations in rangeDecorations.ts listens on `onDidChangeDecorations` instead of `onDidChangeCursorPosition`. The root cause identified in 2021 has never been fixed.

## Change summary

When stepping through multi-file search results, the whole-line range highlight flashes briefly then disappears because any concurrent decoration change (from extensions or the editor itself) clears it.

## Best solution

Replace the `onDidChangeDecorations` listener in `src/vs/workbench/contrib/search/browser/searchTreeModel/rangeDecorations.ts` with `onDidChangeCursorPosition` on the editor (matching the pattern in `src/vs/workbench/browser/codeeditor.ts` RangeHighlightDecorations). The search RangeHighlightDecorations is already in `/browser/` and the call site in searchView.ts already has the editor control — pass the editor instead of the model and use cursor-based clearing.
