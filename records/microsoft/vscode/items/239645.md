# Issue #239645 — `stopAll` in compound launch.json should show default action as stop, not disconnect

[https://github.com/microsoft/vscode/issues/239645](https://github.com/microsoft/vscode/issues/239645)

## Summary

Still reproduces on main: the debug toolbar determines Stop vs Disconnect default purely from `CONTEXT_FOCUSED_SESSION_IS_ATTACH`, without considering the compound config's `stopAll` property. No fix has been applied.

## Change summary

When `stopAll` is specified in a compound launch config, the debug toolbar should show Stop as the default action instead of Disconnect for attach sessions.

## Best solution

Add a context key reflecting whether the active compound configuration has `stopAll` set, and factor it into the toolbar's stop/disconnect primary-action logic in `src/vs/workbench/contrib/debug/browser/debugToolBar.ts` (lines 448–449) so Stop becomes the default when `stopAll` is active.
