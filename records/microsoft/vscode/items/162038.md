# Issue #162038 — Merge Editor: "Accept all change from Left / Right" Available all the time, and isn't really undoable

[https://github.com/microsoft/vscode/issues/162038](https://github.com/microsoft/vscode/issues/162038)

## Summary

Still present on main: the Accept All commands in the merge editor have no precondition checking whether changes are already accepted — the button stays enabled unconditionally. The undo concern (code lenses not restored) is a separate but related gap.

## Change summary

Merge Editor's 'Accept All Changes from Left/Right' remains enabled after all changes are already accepted, and the action is not properly undoable via code lenses.

## Best solution

Add a context key (e.g. `ctxHasUnacceptedInput1`/`ctxHasUnacceptedInput2`) that tracks whether any modified base range lacks the given input's acceptance, and use it as a precondition/when-clause for the Accept All commands. Undo integration is a harder second step — likely needs the merge editor's state-tracking to hook into the result editor's undo stack.
