# Issue #225073 — [GNOME Linux] No window decoration (shadow) if a window title bar style is custom

[https://github.com/microsoft/vscode/issues/225073](https://github.com/microsoft/vscode/issues/225073)

## Summary

Confirmed upstream Electron/Chromium issue — the custom title bar path (OpaqueBrowserFrameView) does not implement CSD shadows on Wayland. A team member (deepak1556) already traced the root cause to Chromium's implementation. No fix has landed on main; the issue remains valid and well-understood.

## Change summary

On Linux with Wayland and custom title bar style, VS Code windows have no window decoration shadow, making overlapping windows hard to distinguish.

## Best solution

The fix belongs in Electron's opaque_frame_view.cc (ported from Chromium's OpaqueBrowserFrameView). Track upstream progress at https://issues.chromium.org/issues/40186366 and https://github.com/electron/electron/issues/43721. In the meantime, a VS Code-side workaround (a subtle CSS border on the window frame, similar to JetBrains IDEs) could be considered as a polish item if upstream remains stalled.
