# Issue #225073 — [GNOME Linux] No window decoration (shadow) if a window title bar style is custom

[https://github.com/microsoft/vscode/issues/225073](https://github.com/microsoft/vscode/issues/225073)

## Summary

Confirmed upstream Electron/Chromium issue — custom titlebar on Linux (Wayland and X11) lacks window shadow/decoration because the opaque frame view implementation in Chromium doesn't enable CSD shadows. No fix exists on main or upstream; keeping open as an upstream tracker.

## Change summary

No window shadow/decoration appears on Linux (GNOME/KDE, Wayland/X11) when using custom title bar style.

## Best solution

Track upstream progress in Chromium/Electron (https://issues.chromium.org/issues/40186366, https://github.com/electron/electron/issues/43721). A VS Code-side workaround adding a CSS border to the window frame (similar to IntelliJ) could be considered as a polish item if upstream remains stalled, but the real fix belongs in Electron's opaque_frame_view implementation.
