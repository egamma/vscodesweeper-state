# Issue #314630 — Agents: Long terminal names expand past the viewport and cause buttons to go missing

[https://github.com/microsoft/vscode/issues/314630](https://github.com/microsoft/vscode/issues/314630)

## Summary

Team-filed visual bug that still reproduces on main: terminal tabs in the Agents window with very long titles expand indefinitely because the tab CSS uses `flex: 0 0 auto` with no max-width cap, pushing subsequent buttons off-screen. The `text-overflow: ellipsis` on the label is ineffective without a constraining width.

## Change summary

Long terminal names in the Agents window editor tab bar expand past the viewport and hide action buttons that follow.

## Best solution

Add a `max-width` constraint to `.agent-sessions-workbench .part.editor .tabs-container > .tab.sizing-fit` (e.g. `max-width: 200px` or a sensible cap) so that text-overflow ellipsis activates for long titles while still allowing tabs to size naturally below that limit. File: `src/vs/sessions/browser/media/style.css` around line 341.
