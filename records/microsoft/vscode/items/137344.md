## Summary

Still valid on current main: on macOS, explicit 'Check for Updates' immediately downloads via Electron's autoUpdater instead of first presenting the user with a choice. The infrastructure to check-without-downloading exists (`checkForUpdateNoDownload`) but is only used for metered connections, not for explicit manual checks. In Backlog with 19 upvotes.

## Change summary

When update.mode is set to manual and the user clicks 'Check for Updates' on macOS, the update immediately downloads instead of first informing the user and offering a choice.

## Best solution

Extend the Darwin update service's `doCheckForUpdates` to use the existing `checkForUpdateNoDownload` path for explicit checks (not just metered connections), transitioning to `AvailableForDownload` state so the user can decide whether to proceed. The plumbing already exists in `src/vs/platform/update/electron-main/updateService.darwin.ts`.
