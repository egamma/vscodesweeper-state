# Issue #292462 — Mouse pointer remains persistently visible over text, blocking view of typed characters

[https://github.com/microsoft/vscode/issues/292462](https://github.com/microsoft/vscode/issues/292462)

## Summary

This is a duplicate of the closed issue #79915, which was triaged as an upstream Chromium/Electron limitation. The Chromium bug (issues.chromium.org/issues/41021563) remains open since 2013. VS Code has no control over this behavior — it is inherited from the Chromium rendering engine that Electron uses.

## Change summary

Mouse pointer does not auto-hide when typing in VS Code, unlike native Windows apps that respect the SPI_GETMOUSEVANISH system setting.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/79915 — this is an upstream Chromium limitation (Chromium issue 41021563). Until Chromium implements SPI_GETMOUSEVANISH support, VS Code cannot address this without a custom workaround. The community extension 'Banish Pointer' exists as a partial mitigation.
