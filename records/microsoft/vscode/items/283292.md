# Issue #283292 — Make sure the ref is set before accessing the element. Maybe wrong initialization order?

[https://github.com/microsoft/vscode/issues/283292](https://github.com/microsoft/vscode/issues/283292)

## Summary

Still reproduces on current main — `triggerAnimation()` reads `_iconRef.element` before the icon DOM is mounted. A fix exists on branch `bryanchen-d/fix-inline-edit-gutter-ref-race` (commit ea3a2516c17) but has not yet landed on main. Milestone 'On Deck' also blocks a close proposal.

## Change summary

Accessing `_iconRef.element` in `triggerAnimation()` throws when the gutter icon DOM node hasn't been constructed yet due to a race between observable transactions.

## Best solution

Merge the fix from branch `bryanchen-d/fix-inline-edit-gutter-ref-race` (commit ea3a2516c17) which gates `isVisible` on actual icon-node mount via an `obsRef` observable, preventing `triggerAnimation()` from being called before the ref is bound.
