# Issue #98597 — Seeing output running out of sources related to CoreText

[https://github.com/microsoft/vscode/issues/98597](https://github.com/microsoft/vscode/issues/98597)

## Summary

This is correctly classified as an upstream Chromium bug (already labeled `upstream` + `chromium`). The CoreText warning originates from Chromium's use of `CTFontCreateWithName` to access the `.NewYork-Regular` system font, which Apple deprecated in Xcode 11+. The Chromium bug (crbug 1131559) appears still unresolved. Nothing actionable on the VS Code side — no VS Code CSS references `.NewYork` — so this stays open as an upstream tracker.

## Change summary

macOS CoreText warnings about `.NewYork-Regular` font access flood the console when VS Code runs, caused by Chromium using deprecated font APIs.

## Best solution

This is owned by Chromium (crbug 1131559 / migrated to issues.chromium.org). VS Code's codebase has no direct `.NewYork` font references — the warning comes from Chromium's internal font fallback code. Keep tracking upstream; it resolves when Chromium switches to proper CoreText APIs (`CTFontCreateUIFontForLanguage`).
