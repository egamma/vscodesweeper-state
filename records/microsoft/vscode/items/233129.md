# Issue #233129 — The "Source Control" panel is often out of sync with my git repo

[https://github.com/microsoft/vscode/issues/233129](https://github.com/microsoft/vscode/issues/233129)

## Summary

Still reproduces on main: the `onFileChange` guard in `repository.ts` silently drops file-system events when any operation (including Status/Refresh) is running, with no follow-up refresh scheduled. PR #303205 diagnosed and proposed a fix but was closed without merging. The bug remains.

## Change summary

Source Control panel becomes stale after git operations (especially interactive rebases or large batches of file changes) because file-system change events are silently dropped when a Status operation is already running.

## Best solution

Apply the approach from the closed PR #303205: in `onFileChange` (extensions/git/src/repository.ts ~line 3185), when `isIdle()` is false but only a Status or Refresh operation is running, fall through and schedule a debounced follow-up refresh via `eventuallyUpdateWhenIdleAndWait()` instead of returning early. This ensures file changes during a running `git status` are not permanently lost.
