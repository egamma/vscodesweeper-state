# Issue #214909 — Terminal does not detect failure in activated environments when activated using PS1 script

[https://github.com/microsoft/vscode/issues/214909](https://github.com/microsoft/vscode/issues/214909)

## Summary

Still reproduces on main: when Python's activate.ps1 wraps VS Code's Prompt function, the wrapper resets $? before VS Code's shell integration captures it, so exit codes always read as 0. Filed by a team member — keeping open per policy.

## Change summary

Terminal shows a blue dot (success) instead of a red dot (failure) after a failed command when a Python venv is activated via activate.ps1 in PowerShell.

## Best solution

The Prompt function in shellIntegration.ps1 captures exit status via `$FakeCode = [int]!$global:?` (line 107), but when activate.ps1 wraps it, the wrapper's own code resets `$?` before VS Code's Prompt runs. The fix should either capture `$?` / `$LASTEXITCODE` before the wrapper can clobber it (e.g. via a PSReadLine handler or a pre-prompt hook) or re-wrap Python's prompt after activation so VS Code's capture comes first in the call chain. The auto-activation path on line 84 already re-captures OriginalPrompt post-activation, but the manual activation path (user running activate.ps1 directly) has no such recovery.
