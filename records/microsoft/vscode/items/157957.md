# Issue #157957 — Visual Studio Code launch can't find custom prelaunch task

[https://github.com/microsoft/vscode/issues/157957](https://github.com/microsoft/vscode/issues/157957)

## Summary

Team-authored bug still present on main: the object-form preLaunchTask snippet is offered by IntelliSense but doesn't resolve built-in tasks because their _key is a UUID that never matches an ITaskIdentifier constructed from label+type. Keeping open per team decision in comment 11.

## Change summary

Using the object form `{task, type}` for `preLaunchTask` (suggested by IntelliSense) fails to find configured tasks because built-in task identifiers use UUIDs internally.

## Best solution

Either make `TaskService#getTask` resolve built-in tasks when given a partial ITaskIdentifier (matching by label+type), or remove the misleading `defaultSnippets: [{ body: { task: '', type: '' } }]` from the preLaunchTask/postDebugTask schema in `src/vs/workbench/contrib/debug/browser/debugAdapterManager.ts` so IntelliSense doesn't suggest a broken format.
