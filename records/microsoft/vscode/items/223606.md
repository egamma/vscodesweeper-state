# Issue #223606 — Auto indent issue: Lua multiline quotes `[[...]]` conflict with #210641 which removes brackets before passing text to indent rules

[https://github.com/microsoft/vscode/issues/223606](https://github.com/microsoft/vscode/issues/223606)

## Summary

The bracket-stripping logic in indentationLineProcessor.ts (line 218) still replaces bracket characters in string/comment tokens with empty string rather than preserving token boundaries, causing false-positive indent matches for languages like Lua with bracket-based multiline quotes. The issue remains open on main.

## Change summary

Lua multiline quotes `[[...]]` have their brackets stripped before indent-rule evaluation, exposing inner content (e.g. keywords) that triggers false-positive indentation.

## Best solution

Modify `getProcessedTokens` in `src/vs/editor/common/languages/supports/indentationLineProcessor.ts` so that string/comment/regex tokens are replaced entirely with whitespace (preserving length or using a single space) rather than only stripping bracket characters. This avoids exposing inner content to indent regex evaluation while still neutralizing bracket-matching interference. The proposed approach in comment 3 (replace token text with space when the line contains other token types) is a reasonable starting point.
