## Summary

Legitimate feature request already triaged by a team member and placed in Backlog with appropriate labels. No reason to close — it requests per-chat-mode-file control over `chat.agent.maxRequests`, which is a reasonable enhancement to the chat mode system.

## Change summary

Allow the `chat.agent.maxRequests` setting to be overridden per chat mode file, so different modes can have different continuous-action limits before prompting the user.

## Best solution

Extend the chat mode file schema to accept an optional `maxRequests` (or similar) field that overrides the global `chat.agent.maxRequests` setting for that mode. Implementation would likely touch the chat mode file parsing and the request-counting logic in the agent chat flow.
