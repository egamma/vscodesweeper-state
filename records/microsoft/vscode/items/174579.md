# Issue #174579 — Background Task With No Output Never Completes

[https://github.com/microsoft/vscode/issues/174579](https://github.com/microsoft/vscode/issues/174579)

## Summary

Team-authored feature request for handling background tasks with no output in the problem matcher / preLaunchTask flow. Hard keep-open rule applies (author is MEMBER). The gap remains: background preLaunchTasks still require a problem matcher with matching output to signal completion.

## Change summary

Background preLaunchTasks that produce no output never signal completion, blocking dependent launch configs from starting.

## Best solution

Add a mode for background preLaunchTasks that only requires the task to have started (process spawned) rather than matching an end pattern in output. This would live in `src/vs/workbench/contrib/tasks/` (task execution/problem matcher logic). Alternatively, allow an empty problem matcher to immediately signal active state.
