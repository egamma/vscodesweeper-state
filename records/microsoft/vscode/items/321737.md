# Issue #321737 — Quick search (%) is sluggish during typing

[https://github.com/microsoft/vscode/issues/321737](https://github.com/microsoft/vscode/issues/321737)

## Summary

Valid performance report: Quick Search (%) triggers a full text search on every keystroke with no input debounce — only the preview navigation is debounced (75ms). On larger workspaces this can freeze the UI, as the synchronous search path blocks the main thread.

## Change summary

Quick Search (%) freezes the application on each keystroke while typing because no input debounce is applied before starting a text search.

## Best solution

Add a debounce to the input value change handler in the PickerQuickAccessProvider or override it in TextSearchQuickAccess so that text search only fires after the user pauses typing (e.g. 150–300ms). The sync results path in `_getPicks` should also be reviewed to ensure it doesn't block the main thread for large workspaces.
