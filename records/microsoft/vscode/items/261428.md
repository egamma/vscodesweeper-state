## Change summary

File search (Ctrl+P) accumulates unbounded IRawFileMatch objects in memory during sequential searches in very large projects (1M+ files), eventually causing OOM crashes.

## Best solution

Investigate the file search result accumulation path in `src/vs/workbench/services/search/node/fileSearch.ts` and `src/vs/workbench/services/search/common/fileSearchManager.ts` — the `{base, relativePath, searchPath}` objects matching `IRawFileMatch` grow without bound across sequential queries. The cache state in `FileQueryCacheState` and how previous query results are disposed when new queries start needs review. The `info-needed` label should be removed since the reporter responded.
