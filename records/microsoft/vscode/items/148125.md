# Issue #148125 — Support DAP's ValueFormat for formatting values in variables list

[https://github.com/microsoft/vscode/issues/148125](https://github.com/microsoft/vscode/issues/148125)

## Summary

Still unimplemented on main: the DAP protocol types for ValueFormat exist in debugProtocol.d.ts, but VS Code never reads supportsValueFormattingOptions or sends format options in Variables/Evaluate/StackTrace requests. The feature request remains valid and is correctly tracked here.

## Change summary

Add built-in UI support for DAP's ValueFormat (e.g. hex display) in the variables list, so debug adapters that advertise supportsValueFormattingOptions can receive format preferences without custom workarounds.

## Best solution

Implement ValueFormat support in the debug adapter communication layer: check supportsValueFormattingOptions in the adapter's capabilities, add a UI toggle (e.g. a toolbar button or context menu in the Variables view) for hex/decimal display, and pass the ValueFormat object in Variables, Evaluate, SetVariable, and StackTrace requests. The work lives in src/vs/workbench/contrib/debug/ — likely debugSession.ts (request construction), debugModel.ts (format state), and the variables view contribution.
