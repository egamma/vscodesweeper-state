# Issue #3352 — [folding] Collapse ending brace to the same line

[https://github.com/microsoft/vscode/issues/3352](https://github.com/microsoft/vscode/issues/3352)

## Summary

This long-standing feature request to collapse the ending brace onto the folded line (showing `function a() { ... }` instead of leaving `}` on its own line) has not been implemented on main. The hidden range model still hides from startLine+1 to endLine with no option to merge the last line visually.

## Change summary

When code is folded, the closing brace/bracket should collapse onto the same line as the opening, showing a compact single-line representation.

## Best solution

Implement a setting (e.g. `editor.foldingConsumeLastLine`) that, when the folding range's last line is just a closing bracket, visually merges it onto the collapsed first line. The main change would be in `src/vs/editor/contrib/folding/browser/hiddenRangeModel.ts` (to extend the hidden range by one line) and the folding decoration rendering to show `{ … }` inline. This is a non-trivial editor layout change involving hidden ranges and inline decorations.
