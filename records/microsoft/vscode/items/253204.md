# Issue #253204 — VSCode instances started through "Relaunch VS Code" button are started with PR_SET_NO_NEW_PRIVS

[https://github.com/microsoft/vscode/issues/253204](https://github.com/microsoft/vscode/issues/253204)

## Summary

Real, source-confirmed bug on Linux: Electron's app.relaunch() inherits the PR_SET_NO_NEW_PRIVS flag, breaking sudo in the terminal after a version-mismatch relaunch. An open team PR (https://github.com/microsoft/vscode/pull/296711) addresses it but has not yet merged — keep open until it lands.

## Change summary

VS Code instances relaunched via the 'Relaunch VS Code' button on Linux inherit PR_SET_NO_NEW_PRIVS, preventing sudo from working in the integrated terminal.

## Best solution

Land https://github.com/microsoft/vscode/pull/296711, which replaces Electron's app.relaunch() with a spawned process on Linux to avoid inheriting the no-new-privileges flag. Review and merge that PR.
