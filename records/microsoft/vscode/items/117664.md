# Issue #117664 — Diff editor scrolls horizontally when dragging selection vertically

[https://github.com/microsoft/vscode/issues/117664](https://github.com/microsoft/vscode/issues/117664)

## Summary

Keep open for maintainer judgment: this is a team-authored issue with an `On Deck` milestone, both of which block cleanup closure. The discussion also identifies a plausible editor behavior improvement rather than a resolved or duplicate issue.

## Change summary

Dragging a vertical selection across a diff editor view zone can jump the cursor to the end of the line above, causing unwanted horizontal scrolling.

## Best solution

Adjust editor mouse-coordinate handling around view zones so, when the adjacent line is visible, the cursor column follows the mouse's horizontal offset instead of defaulting to the line end; this belongs in the editor rendering/view zone selection logic.
