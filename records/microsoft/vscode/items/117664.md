# Issue #117664 — Diff editor scrolls horizontally when dragging selection vertically

[https://github.com/microsoft/vscode/issues/117664](https://github.com/microsoft/vscode/issues/117664)

## Summary

Still open and unfixed on main: the view-zone mouse-position logic snaps to end-of-line instead of using the horizontal offset, causing unwanted horizontal scroll in the diff editor during vertical drag selection. Team-authored with 'On Deck' milestone — keeping open.

## Change summary

Dragging a selection vertically in the diff editor causes unwanted horizontal scrolling when the mouse crosses a view zone, because the cursor snaps to end-of-line rather than preserving the horizontal offset.

## Best solution

In the mouse-target hit-testing code (mouseTarget.ts or the view-zone position resolution path), when the line above a view zone is visible, compute the logical column from the mouse's actual horizontal coordinate rather than defaulting to end-of-line. alexdima's comment outlines the fix direction.
