# Issue #283224 — Canceled mcp tool shows "Canceled: canceled"

[https://github.com/microsoft/vscode/issues/283224](https://github.com/microsoft/vscode/issues/283224)

## Summary

Team-authored bug with 'On Deck' milestone — keep open. The defect is still present on main: when a session is canceled mid-MCP-confirmation, the CancellationError's message ("Canceled") is stored as toolResultError and displayed alongside the state label, producing the redundant "Canceled: canceled" text.

## Change summary

Canceling a chat session while an MCP tool awaits confirmation shows the redundant message "Canceled: canceled" instead of a clean cancellation state.

## Best solution

In `src/vs/workbench/contrib/chat/browser/tools/languageModelToolsService.ts` (around line 772–784), detect `isCancellationError(err)` before writing the error message into `toolResultError`, and either skip setting `toolResultError` entirely for cancellation or set a user-friendly localized message. The tool invocation part UI should also avoid rendering the error string when the tool is already in a Cancelled state.
