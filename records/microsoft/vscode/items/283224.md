# Issue #283224 — Canceled mcp tool shows "Canceled: canceled"

[https://github.com/microsoft/vscode/issues/283224](https://github.com/microsoft/vscode/issues/283224)

## Summary

Still reproduces on main: when a session is cancelled mid-tool-execution, CancellationError's message ("Canceled") is stored as toolResultError and displayed in the UI as "Canceled: canceled". Team-authored issue with On Deck milestone — keeping open.

## Change summary

Cancelling a chat session while an MCP tool confirmation is pending shows an unhelpful "Canceled: canceled" error message in the tool result UI.

## Best solution

In the catch block at languageModelToolsService.ts:772–777, skip setting `toolResultError` when `isCancellationError(err)` is true (or set a user-friendly localized message instead). The `cancelFromStreaming` path in chatModel.ts:1500 already transitions the tool to cancelled state gracefully — the error-path display is redundant for cancellation.
