# Issue #290266 — Need a way to inspect every tool call to understand how the LLM operates / what bugs the tools have

[https://github.com/microsoft/vscode/issues/290266](https://github.com/microsoft/vscode/issues/290266)

## Summary

Team-authored feature request on the On Deck milestone requesting better tool-call inspection in the chat UI. Active internal discussion with concrete design ideas — keep open for the owning team.

## Change summary

Provide a lightweight way to inspect tool call inputs/outputs directly from the chat window, rather than requiring users to dig through the log view.

## Best solution

Add a quick-link or hover affordance on tool-call elements in the chat UI that opens the full tool input/output in an editor or navigates to the debug log view. The discussion between hediet, pwang347, and justschen is converging on a hover-based link to avoid cluttering the chat surface. Implementation likely touches chat response rendering in `src/vs/workbench/contrib/chat/browser/` and possibly the Agent Debug Panel integration.
