# Issue #188557 — Closed notebook search: opening the notebook at a result might not scroll properly to it

[https://github.com/microsoft/vscode/issues/188557](https://github.com/microsoft/vscode/issues/188557)

## Summary

Team-authored confirmed bug about notebook search scroll positioning after layout shifts from output loading. No fix commit found on main; hard keep-open rule applies (author is MEMBER).

## Change summary

Opening a closed notebook at a search result near the end doesn't scroll to the correct position because outputs loading cause layout shifts.

## Best solution

The notebook cell list needs to account for layout shifts from async output rendering when scrolling to a search result. The fix likely belongs in the notebook search reveal/scroll logic (around `src/vs/workbench/contrib/search/` and `src/vs/workbench/contrib/notebook/browser/`) — possibly deferring the final scroll until outputs have settled, or using a scroll-anchor mechanism similar to what the notebook already uses for cell resizes above the viewport.
