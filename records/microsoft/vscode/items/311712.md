## Summary

Active working copy leak affecting thousands of users across multiple stable releases (1.116–1.128) with no fix merged. Multiple code paths (chat editing restore, multi-diff editor, workspace-edit apply, git editor input) accumulate working copies past the 512 threshold. Still reproduces on main.

## Change summary

Working copy leak detection fires (513+ copies) during chat editing session restore and other paths that create TextFileEditorModels without proper disposal.

## Best solution

The primary leak path is in `chatEditingSession.ts`'s `_initEntries` which creates model references during session restore without bounding or disposing them when sessions are torn down mid-restore. Fix should ensure model references acquired in the restore loop are properly disposed on session disposal, and investigate the secondary paths (multiDiffEditorInput, workspace-edit apply, git editor input). Area: `workbench-working-copy` or `chat`.
