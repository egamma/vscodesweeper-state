# Issue #211159 — Github Querry limit exceeded when tying the issue title

[https://github.com/microsoft/vscode/issues/211159](https://github.com/microsoft/vscode/issues/211159)

## Summary

Team-authored issue — keeping open. The bug is still present on main: both the legacy and new issue reporter implementations use unauthenticated GitHub search API calls (10 req/min limit) with only a 300ms debounce, with no rate-limit detection or back-off.

## Change summary

GitHub API rate limit is exceeded when typing in the issue reporter title field, because the unauthenticated search endpoint fires too frequently.

## Best solution

Add rate-limit awareness to the GitHub search calls in the issue reporter: detect 403/429 responses with `X-RateLimit-Remaining` headers, back off gracefully, and show a user-facing message instead of silently failing. The fix touches `searchGitHubIssues` in `issueReporterOverlay.ts` and `searchGitHub` in `baseIssueReporterService.ts`. Alternatively, authenticate the requests or increase the debounce interval.
