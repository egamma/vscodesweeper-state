# Issue #257959 — git.openChange stopped to work in changed files

[https://github.com/microsoft/vscode/issues/257959](https://github.com/microsoft/vscode/issues/257959)

## Summary

Still reproduces on main: when `git.openChange` is invoked via keyboard shortcut from the SCM view without an active editor, `getSCMResource()` returns `undefined` because it only checks `window.activeTextEditor` — it never consults the SCM view's current selection. The context menu works because it passes the resource as an argument. This is a real gap that affects multiple SCM commands.

## Change summary

The `git.openChange` command (and similar git commands) silently does nothing when triggered via keyboard shortcut while a file is selected in the SCM view but no editor is active, because the fallback `getSCMResource()` only checks the active text editor, not the SCM view selection.

## Best solution

Update `getSCMResource()` in `extensions/git/src/commands.ts` to also check the SCM view's selected resources when no URI is provided and no active editor exists. Alternatively, the `openChange` command handler could query the SCM view's selection directly. The commenter confirms this also affects `Open File` and `Reveal in Explorer` commands that share the same fallback pattern.
