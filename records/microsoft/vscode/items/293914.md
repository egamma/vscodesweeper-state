# Issue #293914 — MCP servers installed from registry disabled on reload

[https://github.com/microsoft/vscode/issues/293914](https://github.com/microsoft/vscode/issues/293914)

## Summary

Still reproduces on main: when the MCP gallery query fails or returns empty during early initialization (before enterprise auth completes), `syncInstalledMcpServersWithGallery` clears the `.gallery` field and disables the server. The linked PR #293917 is still open and unmerged. Team members are actively investigating.

## Change summary

MCP servers installed from an enterprise registry are permanently disabled after a window reload when the initial gallery validation call fails due to an authentication race condition.

## Best solution

The open PR https://github.com/microsoft/vscode/pull/293917 should be reviewed and landed, or an alternative fix that either defers `syncInstalledMcpServers` until auth is ready, retries on transient failures, or preserves the installed state when the gallery query returns empty for auth-gated registries. The fix likely belongs in `src/vs/workbench/contrib/mcp/browser/mcpWorkbenchService.ts` around the sync logic and possibly in the workbench gallery manifest service.
