# Issue #99295 — Allow for "random access" undos

[https://github.com/microsoft/vscode/issues/99295](https://github.com/microsoft/vscode/issues/99295)

## Summary

Team-authored feature request for smarter undo that can skip unrelated edits in other files. Keeping open per team-member policy; still a valid enhancement in the undo-redo area.

## Change summary

Allow undo operations to succeed even when unrelated changes have been made in other files, by detecting that the affected range was not modified.

## Best solution

Enhance the undo stack in `src/vs/platform/undoRedo/` to support conflict-free selective undo — when an undo target's affected range hasn't been touched by intervening edits, allow it to proceed rather than failing. This is a non-trivial architectural change to the undo/redo service.
