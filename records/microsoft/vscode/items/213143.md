# Issue #213143 — Safari 14 no longer able to load vscode.dev, affecting shipped WebKit of macOS 11 and iOS 15

[https://github.com/microsoft/vscode/issues/213143](https://github.com/microsoft/vscode/issues/213143)

## Summary

VS Code's web build targets ES2024 (confirmed in `src/tsconfig.base.json` and `build/next/index.ts`), which Safari 14 (macOS 11 / iOS 15) does not support. macOS 11 has been end-of-life since late 2023, and Safari 14 dates to 2020. Supporting it would require downgrading the JS target, conflicting with VS Code's performance-first values. This is out of scope.

## Change summary

vscode.dev fails to load on Safari 14 (macOS 11 / iOS 15) due to ES2024 syntax unsupported by that browser version.

## Best solution

Close as out-of-scope — vscode.dev requires a modern browser engine, and Safari 14 on macOS 11 (EOL) falls below the project's JS target (ES2024). Users should upgrade to a supported macOS/iOS version or use a modern browser.

## Evidence

- **JS target:** src/tsconfig.base.json sets `"target": "ES2024"` and build/next/index.ts confirms esbuild target is `es2024`. Safari 14 does not support ES2024 syntax (e.g. class static blocks, RegExp named groups, etc.).
- **reporter error:** Comment 1 confirms a `SyntaxError: Unexpected token '{'` in `workbench.web.main.js` — a classic symptom of modern JS syntax hitting an older engine.

- **related issue:** https://github.com/coder/code-server/issues/6805 reports the same symptom in code-server, confirming this is a platform-level JS engine limitation, not a VS Code-specific regression.
