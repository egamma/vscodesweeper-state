# Issue #299442 — Terminal font broken on latest stable

[https://github.com/microsoft/vscode/issues/299442](https://github.com/microsoft/vscode/issues/299442)

## Summary

Still reproduces on main: commit f32f3306fd3 introduced `allowTransparency: config.enableImages` in xtermTerminal.ts, which degrades font rendering on macOS when images are enabled. No fix has landed.

## Change summary

Enabling `terminal.integrated.enableImages` forces xterm's `allowTransparency` mode, degrading terminal font anti-aliasing on macOS.

## Best solution

Remove the implicit coupling between `enableImages` and `allowTransparency` in `src/vs/workbench/contrib/terminal/browser/xterm/xtermTerminal.ts` (lines 269 and 573). The image addon should work without forcing transparency mode on all terminal text rendering.
