# Issue #129487 — [css] Wrong selector specificity values with :where

[https://github.com/microsoft/vscode/issues/129487](https://github.com/microsoft/vscode/issues/129487)

## Summary

Fixed in the upstream vscode-css-languageservice — the current version (6.3.4) correctly gives `:where()` zero specificity by skipping its contents in the calculation.

## Change summary

`:where(.foo) li` incorrectly reported specificity (0, 2, 1) instead of (0, 0, 1) because `:where()` was not handled as a zero-specificity pseudo-class.

## Best solution

Keep the shipped fix in vscode-css-languageservice; no further change needed. The specificity calculation in `selectorPrinting.js` correctly handles `:where` with `continue elementLoop`.
