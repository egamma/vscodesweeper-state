## Summary

Team-authored design discussion about workspace trust for virtual workspaces — hard keep-open applies. The `canonicalUriProvider` API remains proposed-only and the broader virtual-workspace trust story discussed here appears unresolved on current main.

## Change summary

Investigate and design an API so virtual file system providers can supply trustable URI aliases to the workspace trust system.

## Best solution

A maintainer decision is needed on whether to pursue a dedicated trust-URI API for file system providers (the `WorkspaceTrustUriProvider` proposal in this issue), extend the existing proposed `canonicalUriProvider` API, or declare the current RemoteHub-specific handling in core sufficient. The workspace-trust area owner should evaluate whether the virtual-workspace trust gap still matters given current usage patterns.
