# Issue #319286 — [folding] Format large file unfold all code

[https://github.com/microsoft/vscode/issues/319286](https://github.com/microsoft/vscode/issues/319286)

## Summary

Legitimate bug report: formatting a file causes all folded regions to unfold. The folding model's `sanitizeAndMerge` logic (line 339 of foldingRanges.ts) uncollapse ranges when the selection/cursor moves inside them during the format operation. Commit 5b133aa65c8 addressed a similar case for organize-imports but not general formatting. No fix found on main.

## Change summary

Formatting a large file with folded code regions causes all folds to expand, losing the user's folding state.

## Best solution

The folding controller's `onDidChangeModelContent` → `triggerFoldingModelChanged` path should preserve fold state during format operations even when the cursor moves. A likely fix is to snapshot the folded ranges before the format, or to not use the post-format cursor position for the selection-based uncollapse check in `sanitizeAndMerge`. The fix would touch `src/vs/editor/contrib/folding/browser/folding.ts` and possibly `foldingRanges.ts`.
