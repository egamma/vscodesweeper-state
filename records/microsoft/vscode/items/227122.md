# Issue #227122 — Editor GPU: Support editor.fontWeight

[https://github.com/microsoft/vscode/issues/227122](https://github.com/microsoft/vscode/issues/227122)

## Summary

Still needed: the GPU renderer does not yet consume the `editor.fontWeight` setting. Only decoration-level font-weight (bold/normal) is partially supported via CSS rules; the base editor font weight configuration remains unimplemented in the GPU path.

## Change summary

Support the `editor.fontWeight` setting (e.g., weight 350 for Cascadia Code) in the GPU-accelerated editor renderer.

## Best solution

Implement `editor.fontWeight` consumption in the GPU glyph rasterizer so that variable-weight fonts like Cascadia Code render at the configured weight. The rasterizer needs to pass the configured weight to the canvas font string used for glyph texture generation.

## Vision fit

- **Fit:** aligned
- **Reason:** The GPU renderer is part of the editor performance story, and font rendering fidelity is a core editor quality concern — both are durable themes.
- **Evidence:** Durable theme: Performance — the GPU renderer improves editor rendering performance; completing its feature parity (font weight) serves the 'performance trumps functionality' value without sacrificing functionality.
