With GPU rendering enabled, editor rulers render over the line numbers gutter area instead of being clipped to the content region.

## Best solution

The existing `setScissorRect` fix in `src/vs/editor/browser/gpu/rectangleRenderer.ts` clips the ruler draw call but apparently doesn't cover all code paths. The rectangle renderer or the ruler-specific drawing logic needs a second pass to identify the remaining leak — likely a viewport offset or a secondary render pass that bypasses the scissor.
