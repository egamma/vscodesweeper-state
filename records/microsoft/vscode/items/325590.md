# Issue #325590 — Agents window send fails when Git LFS is unavailable

[https://github.com/microsoft/vscode/issues/325590](https://github.com/microsoft/vscode/issues/325590)

## Summary

Team-authored bug that still reproduces on main — the related fix PR (https://github.com/microsoft/vscode/pull/325591) was closed without merging, so the LFS-missing worktree checkout failure remains. Keep open for maintainer judgment.

## Change summary

Agents window worktree-isolated sessions fail on first send when git-lfs is not on PATH, because the checkout inherits the repository's required LFS filter and exits with code 128.

## Best solution

In `agentHostGitService.ts`, disable or downgrade the LFS clean/smudge filter (e.g. `git -c filter.lfs.process= -c filter.lfs.smudge= -c filter.lfs.clean=`) when creating the agent worktree, so checkout leaves LFS pointer files intact instead of failing. The closed PR #325591 was the right direction — a revised version should land.
