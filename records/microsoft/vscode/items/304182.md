# Issue #304182 — Profiles: spaces in profile name not sanitized in Open New Window command ID

[https://github.com/microsoft/vscode/issues/304182](https://github.com/microsoft/vscode/issues/304182)

## Summary

Team-authored issue with a confirmed source-level bug: the replace call uses a string literal instead of a regex, so spaces in profile names are never sanitized. An open PR (#304153) already addresses it; keeping open pending maintainer review and merge.

## Change summary

Profile names with spaces produce command IDs with unescaped spaces because `.replace('/\s+/', '_')` uses a string literal instead of a regex.

## Best solution

Land the fix in PR https://github.com/microsoft/vscode/pull/304153 (change the string literal to a proper regex `/\s+/g`) after addressing the reviewer's question about concrete breakage.
