# Issue #240040 — Git - Blame not working on remote repositories

[https://github.com/microsoft/vscode/issues/240040](https://github.com/microsoft/vscode/issues/240040)

## Summary

Git blame is intentionally limited to local/file-scheme URIs because it relies on a local git repository to run `git blame`. Supporting virtual filesystems (Remote Repositories) would require an alternative blame data source (e.g., a GitHub API-based blame provider). This is a valid feature request that remains unimplemented on main.

## Change summary

Git blame (editor decoration and status bar item) does not work when browsing remote repositories via the Remote Repositories extension.

## Best solution

Blame for remote repositories would likely need a separate blame information provider that uses the GitHub API (or similar) rather than the local `git blame` CLI command. This could be implemented in the Remote Repositories extension itself or via an extension API that allows extensions to contribute blame data. The current `isResourceSchemeSupported` guard in `extensions/git/src/blame.ts` is the gating check. The best path is likely an extension-API contribution point for blame providers, or support within the Remote Repositories / GitHub Repositories extension.

- **architectural gap:** The built-in git extension's blame relies on running `git blame` against a local repository checkout. Remote Repositories provides a virtual FS with no local .git directory, so the underlying command cannot run.
