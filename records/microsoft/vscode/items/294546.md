# Issue #294546 — Updates not finishing

[https://github.com/microsoft/vscode/issues/294546](https://github.com/microsoft/vscode/issues/294546)

## Summary

The fix from PR #313791 was partially reverted in PR #316264, and issue #318029 now tracks the same root cause (dllhost.exe blocking inno_updater during update) with additional evidence of a PowerShell quoting bug in the remaining fix. This issue should consolidate into #318029, which is the active tracking issue.

## Change summary

VS Code updates on Windows hang indefinitely because dllhost.exe (COM Surrogate) locks code_explorer_command_x64.dll, preventing inno_updater from deleting the old commit folder.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/318029, which tracks the same dllhost.exe blocking problem post-revert and includes evidence of the PowerShell quoting bug in the KillContextMenuComSurrogate procedure in build/win32/code.iss. The fix needs to correctly terminate the COM surrogate before inno_updater runs, and the PowerShell command escaping must be corrected so it doesn't exit with code 1.
