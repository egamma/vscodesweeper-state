## Summary

Team-authored performance improvement request for the multi diff editor — progressive rendering to avoid a perceived 1s+ white-screen delay. Kept open per hard keep-open rule (MEMBER author) and because the proposed incremental-render approach hasn't landed yet.

## Change summary

Opening the multi diff editor shows a blank white screen for ~1–2 seconds before content appears; request to render progressively.

## Best solution

Implement the progressive rendering approach described in the comments: show the multi diff editor shell immediately with a loading spinner, then append each diff editor as its diff computation completes, eliminating the white-flash delay.
