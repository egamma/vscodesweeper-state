## Change summary

Ensure that running multiple VS Code versions simultaneously does not cause IPC conflicts, shared user-data-dir corruption, or CLI commands executing against a mismatched version during updates.

## Best solution

Implement the three safeguards outlined: (1) version-scoped IPC connections, (2) user-facing error when a second instance targets an in-use user-data-dir, and (3) CLI version-mismatch check before update-sensitive commands.
