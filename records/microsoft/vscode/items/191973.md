# Issue #191973 — Sticky Scrolling Makes Scrolling Stutter

[https://github.com/microsoft/vscode/issues/191973](https://github.com/microsoft/vscode/issues/191973)

## Summary

The core performance issue remains on main: `scrollWidth` is still read per-line during sticky scroll rendering (line 521), causing forced reflows. Some related improvements have landed (DOM refactoring, memory leak fix, hover listener fix) but the layout-thrashing root cause persists.

## Change summary

Sticky scroll causes dropped frames during scrolling due to forced reflow from reading `scrollWidth` per rendered sticky line.

## Best solution

Cache or batch the `scrollWidth` reads in `RenderedStickyLine` construction — either defer the read to after all DOM mutations are complete, or use a `ResizeObserver` / cached value to avoid synchronous layout queries inside the render loop.
