# Issue #190151 — useInlineViewWhenSpaceIsLimited - only do this when there is horizontal scroll bar

[https://github.com/microsoft/vscode/issues/190151](https://github.com/microsoft/vscode/issues/190151)

## Summary

Team-authored feature request for smarter inline-view toggling in the diff editor — only switch when long lines cause a horizontal scrollbar, not merely when horizontal space is tight. Hard keep-open: author is a team member.

## Change summary

The diff editor's `useInlineViewWhenSpaceIsLimited` heuristic should only trigger when the content actually has long lines (producing a horizontal scrollbar), not unconditionally when horizontal space is small.

## Best solution

Refine the inline-view switching logic in the diff editor widget to measure content width (or detect horizontal scrollbar presence) before toggling to inline mode, likely in `src/vs/editor/browser/widget/diffEditor/`. The noted challenges (content width unknown before render, avoiding toggling on keystrokes) suggest a debounced post-layout check.
