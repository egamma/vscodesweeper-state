# Issue #286724 — Chat: support an action to send a chat request and clear widget

[https://github.com/microsoft/vscode/issues/286724](https://github.com/microsoft/vscode/issues/286724)

## Summary

The requested 'send and clear to empty' behavior (fire-and-forget) is distinct from the existing 'Send to New Chat' action, which sends input into a new session the user then watches. PR #286767 was closed without merge, and the design discussion (comments 5–6) remains unresolved — keeping open for a product decision.

## Change summary

Add an action that sends a chat request in the background and immediately returns the widget to an empty state, without showing the response.

## Best solution

Resolve the UX semantics raised in comment 5 (roblourens) and comment 6 (jo-oikawa): decide whether this is a variant of Send to New Chat or a distinct 'work in background' action, then implement the chosen approach in `chatExecuteActions.ts`. The existing `SendToNewChatAction` class is the natural extension point.
