# Issue #286724 — Chat: support an action to send a chat request and clear widget

[https://github.com/microsoft/vscode/issues/286724](https://github.com/microsoft/vscode/issues/286724)

## Summary

The existing SendToNewChatAction sends input to a new session where the user sees the response. The original request — fire off a request that processes in the background while the user gets a blank chat — remains unimplemented. The related PR #286767 was closed without merge, and the UX design is still under discussion (comments 5–6).

## Change summary

Add a chat action that sends a request to process in the background and immediately returns the user to an empty chat input.

## Best solution

Resolve the open design question (comment 5: whether this replaces or complements 'send to new chat'; comment 6: a 'work in background' UX option) and implement accordingly in `src/vs/workbench/contrib/chat/browser/actions/chatExecuteActions.ts`, likely as a new action or an option on the existing SendToNewChatAction.
