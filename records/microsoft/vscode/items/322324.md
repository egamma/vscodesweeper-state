# Issue #322324 — Window keeps crashing on build

[https://github.com/microsoft/vscode/issues/322324](https://github.com/microsoft/vscode/issues/322324)

## Summary

This is a recurrence of the @xterm/addon-ligatures OOM/crash when parsing certain fonts (PragmataPro Mono Liga). The same reporter hit this before in https://github.com/microsoft/vscode/issues/284510, which was rooted in the upstream issue https://github.com/microsoft/vscode/issues/239541. That fix shipped in 1.108 but has regressed — the reporter confirms 1.124.2 works but 1.125.1 and 1.126.0 crash. The root cause lives in the @xterm/addon-ligatures package; this needs upstream investigation and likely another addon-ligatures update.

## Change summary

Window freezes and crashes (OOM in the renderer process) when terminal ligatures are enabled with the PragmataPro Mono Liga font.

## Best solution

Investigate the regression in @xterm/addon-ligatures between the versions shipped with VS Code 1.124 and 1.125. The prior fix for https://github.com/microsoft/vscode/issues/239541 appears to have regressed for PragmataPro Mono Liga. The fix will likely need to land upstream in the xterm.js addon-ligatures package. Label as `upstream` and `terminal-ligatures` and route to the terminal area owner.
