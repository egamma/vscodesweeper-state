  > Thanks for the detailed write-up. The API proposals you're seeing errors for (`chatDebug`, `chatHooks`, etc.) are **proposed VS Code APIs** that only ship in official VS Code builds — they aren't part of the open-source base that code-server packages. Copilot Chat relies on these proposed APIs and is only officially supported on:
  > 
  > - VS Code Desktop
  > - vscode.dev / github.dev
  > - GitHub Codespaces
  > - Official VS Code Server (Remote Tunnels)
  > 
  > Since code-server is a separate third-party project, compatibility questions are best directed to the [code-server repository](https://github.com/coder/code-server). For remote/server scenarios with full Copilot support, the official [VS Code Server](https://code.visualstudio.com/docs/remote/tunnels) or Codespaces are the recommended paths.
