# Issue #267695 — Microsoft Authentication repeatly asks for sign in

[https://github.com/microsoft/vscode/issues/267695](https://github.com/microsoft/vscode/issues/267695)

## Summary

Active upstream issue in the MSAL native broker (Azure AD identity platform) causing repeated sign-in prompts. Team members are investigating and have identified a workaround; the root cause lives in the broker/identity stack, not VS Code itself.

## Change summary

Microsoft Authentication repeatedly prompts for sign-in due to MSAL native broker errors (invalid_grant / session invalid) even without password changes.

## Best solution

Continue tracking upstream with the Azure Identity team. The workaround (`"microsoft-authentication.implementation": "msal-no-broker"`) helps some users but not all (comment 10). Resolution depends on a fix in the MSAL native broker or the Azure AD platform; VS Code's role is to relay logs and potentially improve fallback behavior when the broker fails.
