# Issue #150285 — `$msCompile` problem matcher has issues with wrapped lines

[https://github.com/microsoft/vscode/issues/150285](https://github.com/microsoft/vscode/issues/150285)

## Summary

Still a valid limitation on current main: when programs (or ConPTY) insert hard line breaks based on terminal width, the problem matcher receives fragmented lines and fails to match. The xterm soft-wrap case is handled (LineDataEventAddon concatenates wrapped lines), but program-inserted hard breaks remain unsolved. Keeping open given community interest (8 upvotes) and multiple affected tools beyond MSBuild.

## Change summary

Problem matchers fail when programs insert hard line breaks based on terminal/console buffer width, producing truncated or phantom diagnostics in the Problems panel.

## Best solution

A general solution would require either buffering and reassembling lines that match a partial pattern (in the problem matcher layer), or providing a mechanism for tasks to pipe output through a file rather than the terminal (as suggested in comment #10). The `LineDataEventAddon` already handles xterm soft wraps correctly; the gap is program-injected hard breaks. The `ForceNoAlign` workaround exists for MSBuild specifically but not for arbitrary tools.
