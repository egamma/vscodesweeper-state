# Issue #161560 — Merge editor: restore original contents if aborting and allow to undo

[https://github.com/microsoft/vscode/issues/161560](https://github.com/microsoft/vscode/issues/161560)

## Summary

Partially addressed but not fully resolved — the workspace merge editor mode saves before opening (so 'Don't save' reverts), and an undo stack was added inside the merge editor, but the full request (undo in the regular text editor after closing with auto-save ON) remains unimplemented. Keeping open as under-discussion.

## Change summary

Restore original file contents when aborting the merge editor, and allow undo in the text editor afterward.

## Best solution

The remaining gap is allowing undo in the regular text editor after the merge editor closes with auto-save ON. The merge editor's workspace input model (`src/vs/workbench/contrib/mergeEditor/browser/mergeEditorInputModel.ts`) would need to preserve the undo stack across the open/close transition so the underlying text model's edits are reversible. This likely belongs to the merge-editor area owner to decide on UX feasibility.
