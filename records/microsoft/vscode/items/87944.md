# Issue #87944 — Diff editor: Differentiate unimportant differences

[https://github.com/microsoft/vscode/issues/87944](https://github.com/microsoft/vscode/issues/87944)

## Summary

Valid, well-upvoted feature request (69 👍) for grammar/regex-aware 'unimportant difference' handling in the diff editor. Not implemented on current main — only whitespace trimming exists. Belongs in backlog as-is.

## Change summary

Allow the diff editor to differentiate or filter out unimportant differences (comments, case, regex-matched patterns) via distinct coloring or hiding.

## Best solution

Implement a configurable mechanism (e.g. regex array or language-grammar hooks) to classify diff hunks as 'unimportant' and render them with reduced visual weight rather than full insert/delete coloring. This lives in `src/vs/editor/browser/widget/diffEditor` and the diff computation layer. Given the complexity (language-aware semantics, UI for toggling filters), this is a significant feature — but with 69 upvotes and clear use cases it warrants staying in the backlog for future planning.
