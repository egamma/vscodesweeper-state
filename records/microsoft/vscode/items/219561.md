# Issue #219561 — Minimap section headers missing for`jsonc` #region 

[https://github.com/microsoft/vscode/issues/219561](https://github.com/microsoft/vscode/issues/219561)

## Summary

This still appears to be a valid editor-minimap bug on current main: JSON/JSONC has language-server region folding, but minimap section headers only read language-configuration folding markers.

## Change summary

Named `//#region` blocks in `jsonc` fold but do not appear as minimap section headers.

## Best solution

Either add JSON folding markers to `extensions/json/language-configuration.json` as the narrow fix, or preferably teach `src/vs/editor/contrib/sectionHeaders/browser/sectionHeaders.ts` to consume language-service/LSP folding ranges of kind `region` so section headers work consistently across languages.

## Evidence

- **current code:** `src/vs/editor/contrib/sectionHeaders/browser/sectionHeaders.ts` builds section-header options from `ILanguageConfigurationService.getLanguageConfiguration(languageId).foldingRules`; it does not query folding providers or LSP folding ranges.
- **current code:** `src/vs/editor/common/services/findSectionHeaders.ts` only collects region headers when `options.foldingRules?.markers` exists, then matches `options.foldingRules.markers.start` against text lines.
- **json configuration:** `extensions/json/language-configuration.json` on current main defines comments, brackets, indentation, and on-enter rules, but no `folding.markers` entry for `#region` / `#endregion`.
- **history check:** `git log -- extensions/json/language-configuration.json` shows recent changes for comment continuation and JSON indentation, but no commit adding JSON region folding markers after the 2024-07-01 report.
