# Issue #174295 — Multiple Import Organizers results in corrupted code

[https://github.com/microsoft/vscode/issues/174295](https://github.com/microsoft/vscode/issues/174295)

## Summary

Still reproduces on main. The core defect is in `applyOnSaveActions`: `getCodeActions` fetches edits from ALL providers against the same document snapshot for a given kind, then applies them sequentially — but the second provider's edits were computed against the pre-first-edit state, causing corruption. An open PR (#306045) exists but has not merged. Author is a MEMBER and milestone is On Deck — hard keep-open.

## Change summary

When multiple code action providers register for the same kind (e.g. `source.organizeImports`), running code actions on save corrupts the file because all providers receive the same document version rather than cascading changes.

## Best solution

Review and land the open PR (https://github.com/microsoft/vscode/pull/306045) or implement a fix that re-queries remaining providers after each provider's edits are applied, so each provider sees the updated document. The fix lives in `src/vs/workbench/contrib/codeEditor/browser/saveParticipants.ts` in `applyOnSaveActions` — after applying each action from `actionsToRun`, the model should be re-queried for the next provider's actions. A UI affordance to select or order providers (like the formatter picker) would also help.
