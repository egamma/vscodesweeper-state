# Issue #74342 — [themes] Allow to set a background color for settings / webview editors

[https://github.com/microsoft/vscode/issues/74342](https://github.com/microsoft/vscode/issues/74342)

## Summary

This feature request for a dedicated background color for the settings editor and webview-based editors remains unimplemented on main. It has the `help wanted` label and sits in Backlog, so it should stay open for community contribution.

## Change summary

Allow theme authors to set a separate background color for the settings editor and webview/rich-text editors, distinct from `editor.background`.

## Best solution

Add a `settings.background` color token (defaulting to `editor.background`) in `src/vs/workbench/contrib/preferences/common/settingsEditorColorRegistry.ts` and apply it to the settings editor pane. A broader webview background token could also be considered but would need design discussion.
