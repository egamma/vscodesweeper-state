[https://github.com/microsoft/vscode/issues/324136](https://github.com/microsoft/vscode/issues/324136)

## Summary

Team-authored feature request to restore granular session-scoped and persistent terminal auto-approve controls in the Agent Host — hard keep-open per policy.

## Change summary

Bring back sticky terminal auto-approve controls (per-session and always-allow-exact-command) to the Agent Host's command auto-approver, matching the workbench side's existing session-rule support.

## Best solution

Extend `src/vs/platform/agentHost/node/commandAutoApprover.ts` to support session-scoped approval rules (mirroring `_getSessionRules` from the workbench `commandLineAutoApprover.ts`) and an 'always allow exact command' option that persists to the `chat.tools.terminal.autoApprove` setting. The 'allow all commands in session' option needs alignment with the permissions/risk-assessment model before implementation.
