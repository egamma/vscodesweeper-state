# Issue #325611 — Self messaging works but loops can still form with two sessions

[https://github.com/microsoft/vscode/issues/325611](https://github.com/microsoft/vscode/issues/325611)

## Summary

Team-authored bug report identifying a real gap in cross-session loop detection. A maintainer (sandy081) has already acknowledged the issue and described planned work to add hop-count/provenance-based detection. Keeping open as active work.

## Change summary

Two agent sessions can form an infinite messaging loop (A↔B) because the self-loop guard only blocks trivial A→A messaging, not cross-session cycles.

## Best solution

Implement lightweight cross-session loop detection via a hop-count or provenance marker propagated through `send_message`, complementing the existing spawn-depth mechanism in `create_session` and keeping the 50-message cap as a final backstop.
