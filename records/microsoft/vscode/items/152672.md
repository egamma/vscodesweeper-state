# Issue #152672 — Don't select inlay hint UI when it's next to a selected variable or have a parameter to enable that

[https://github.com/microsoft/vscode/issues/152672](https://github.com/microsoft/vscode/issues/152672)

## Summary

Team-authored issue (MEMBER). The visual selection of inlay hint text when double-clicking an adjacent variable remains unaddressed on current main — inlay hints use InjectedTextCursorStops but nothing guards against the injected text appearing selected during word-select.

## Change summary

Double-clicking a variable causes adjacent inlay hint decorations to appear visually selected, which looks incorrect.

## Best solution

Either exclude injected inlay-hint text from the selection highlight in the view-model rendering (likely in the decoration/view-line rendering layer), or expose an option on InjectedTextOptions to opt out of selection painting. The fix likely touches `src/vs/editor/contrib/inlayHints/browser/inlayHintsController.ts` and possibly the view-line decoration rendering in `src/vs/editor/browser/viewParts/`.
