# Issue #262129 — "open with code" automatically with wsl vscode server

[https://github.com/microsoft/vscode/issues/262129](https://github.com/microsoft/vscode/issues/262129)

## Summary

Valid feature request acknowledged by a maintainer (aeschli) as difficult due to WSL being an extension that cannot modify OS registries. Keeping open as a recognized gap.

## Change summary

Requests that opening a WSL path folder via the Windows Explorer 'Open with Code' context menu automatically opens it in WSL mode without requiring a manual 'Reopen in WSL' step.

## Best solution

The WSL extension (or VS Code's Windows installer integration) would need to detect when a path is under a WSL mount (e.g. `\\wsl$\`) and automatically connect to the WSL remote. This likely requires changes in the installer's shell integration or in the WSL extension's activation logic. Difficult because the extension cannot modify OS registries for context menu entries.
