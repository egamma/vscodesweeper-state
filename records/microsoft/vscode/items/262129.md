# Issue #262129 — "open with code" automatically with wsl vscode server

[https://github.com/microsoft/vscode/issues/262129](https://github.com/microsoft/vscode/issues/262129)

## Summary

Valid feature request acknowledged by a team member (aeschli) as a real gap — the Windows Explorer 'Open with Code' context menu doesn't auto-detect WSL paths and reopen in WSL mode. Noted as difficult because WSL functionality lives in an extension that cannot modify OS registries.

## Change summary

Opening a WSL path via the Windows Explorer 'Open with Code' context menu should automatically connect to WSL instead of requiring a manual 'Reopen Folder in WSL' step.

## Best solution

The WSL extension (or the shell integration installer) would need to detect that a path is under `\\wsl$\` or `\\wsl.localhost\` at startup and automatically trigger the WSL remote connection. The challenge is that the context menu entry is registered by the core installer, not the extension, so coordination is needed between the two.
