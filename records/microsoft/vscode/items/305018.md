# Issue #305018 — Ctrl+T (toggle reasoning display) does not work in Sessions UI

[https://github.com/microsoft/vscode/issues/305018](https://github.com/microsoft/vscode/issues/305018)

## Summary

Team-authored issue; Ctrl+T is consumed by the Sessions UI's own 'new chat tab' keybinding (sessionsActions.ts line 423) and never reaches the CLI process. Still unresolved on main — no fix commit found.

## Change summary

Ctrl+T documented as 'toggle reasoning display' in Copilot CLI never reaches the CLI process inside the Sessions window because VS Code's keybinding layer intercepts it.

## Best solution

Either remap the CLI reasoning toggle to a non-conflicting key when running inside Sessions, or add a `/reasoning` slash command as the reporter suggests, or pass the key through to the terminal when the CLI is focused. The conflict lives in `src/vs/sessions/contrib/sessions/browser/sessionsActions.ts` (Cmd/Ctrl+T = new chat tab).
