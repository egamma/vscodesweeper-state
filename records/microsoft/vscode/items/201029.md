# Issue #201029 — [Remote-SSH Bug]: Error: AttachConsole failed, could not find pty on pty host

[https://github.com/microsoft/vscode/issues/201029](https://github.com/microsoft/vscode/issues/201029)

## Summary

Team-authored issue (MEMBER); the root cause is an upstream race in node-pty's ConPTY kill path where AttachConsole is called against already-exited short-lived diagnostic shells. The try/catch added in node-pty v1.2.0-beta.11 is a workaround, not a full fix. Keeping open for the upstream fix to land.

## Change summary

Remote-SSH to Windows hosts produces AttachConsole failed errors and 'Could not find pty on pty host' spew from node-pty when short-lived diagnostic shells exit before the kill path runs.

## Best solution

The fix belongs in microsoft/node-pty: the kill() path should check whether the shell process has already exited before calling AttachConsole(shellPid). The existing try/catch (commit 2b25c76) suppresses the crash but the redundant call and error cascade remain. A proper fix would skip the console-process-list enumeration entirely when the process exit has already been handled. Track upstream and bump node-pty once a complete fix ships.
