# Issue #144389 — A white line at the top of the window on macOS Monterey

[https://github.com/microsoft/vscode/issues/144389](https://github.com/microsoft/vscode/issues/144389)

## Summary

This is an Electron/macOS upstream issue: macOS draws native window chrome (including the top border) based on the OS color scheme, not the app's content theme. VS Code added the `window.systemColorTheme` setting (commit 3148688541a, Dec 2023) which mitigates this when set to `auto`, but the default remains `default` (follow OS), so users still hit this out of the box. The underlying Electron issue (electron/electron#13164) remains relevant. Keeping open as an upstream issue with a known workaround.

## Change summary

A white line appears at the top of the VS Code window on macOS when the OS is in light mode and VS Code uses a dark theme, because macOS renders native window chrome in the OS color scheme.

## Best solution

Consider changing the default of `window.systemColorTheme` from `'default'` to `'auto'` so that VS Code's native chrome automatically matches the active color theme, eliminating the mismatched border for the majority of users. This is a one-line change in `src/vs/platform/theme/electron-main/themeMainServiceImpl.ts`. Alternatively, keep tracking the upstream Electron fix (electron/electron#13164, electron/electron#51015) and document the workaround more prominently.

## Evidence

- **workaround setting:** The `window.systemColorTheme` setting was added in commit 3148688541a (Dec 2023, PR #200144). When set to `auto`, it sets `electron.nativeTheme.themeSource` to match the VS Code theme, which fixes the white border. Default is `'default'` (follows OS).
- **code path:** src/vs/platform/theme/electron-main/themeMainServiceImpl.ts line 46: `SYSTEM_COLOR_THEME = new Setting<'default' | 'auto' | 'light' | 'dark'>('window.systemColorTheme', 'default')`. Lines 115-137 show the logic that sets `electron.nativeTheme.themeSource`.
- **upstream Electron issue:** https://github.com/electron/electron/issues/13164 (closed) and https://github.com/electron/electron/issues/51015 (closed) track this on the Electron side — macOS draws native window borders based on the system theme, not the app theme.
- **team confirmation:** Comment by rzhao271 (MEMBER) confirmed the issue is in Electron since version 9. Comment by bpasero (CONTRIBUTOR) noted uncertainty about whether it's from VS Code's custom title or a general Electron/OS issue.
- **still reported:** Comment from March 2026 confirms the issue persists on macOS 26.4.
