# Issue #205013 — VSCode terminal seems to construct environment in a strange way

[https://github.com/microsoft/vscode/issues/205013](https://github.com/microsoft/vscode/issues/205013)

## Summary

Still reproduces on main: buildUserEnvironment spreads the resolved shell env (from a spawned login shell) over the actual process.env, so /etc/profile.d values override user-exported variables. Team members identified the cause but no fix landed.

## Change summary

When using `code serve-web`, terminal environment variables exported before launching are overridden by values from /etc/profile.d because shell environment resolution results take precedence over the inherited process environment.

## Best solution

Fix the environment merge order in `buildUserEnvironment` (src/vs/server/node/extensionHostConnection.ts) so that the parent process's environment wins over the resolved shell environment for variables already present, or ensure the `VSCODE_CLI` flag is properly set in `serve-web` to skip redundant shell env resolution.
