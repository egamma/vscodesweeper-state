# Issue #292969 — Up arrow in PowerShell terminals types the wrong command

[https://github.com/microsoft/vscode/issues/292969](https://github.com/microsoft/vscode/issues/292969)

## Summary

The described symptom — agent-run commands appearing in the user's shell history in Chat-created terminals — was fixed on main by commit 84be8379, which sets VSCODE_PREVENT_SHELL_HISTORY and prepends a leading space to agent-host shell commands. The fix shipped in 1.117.0.

## Change summary

Pressing up arrow in a terminal created by the Chat agent replays agent-run commands instead of the user's most recent command.

## Best solution

Keep the shipped fix (commit 84be8379, 1.117.0). No further change needed — agent-host commands are now excluded from shell history via HISTCONTROL/PSReadLine handlers. If the broader non-agent PSReadLine history issue persists, that is tracked by https://github.com/microsoft/vscode/issues/289100.
