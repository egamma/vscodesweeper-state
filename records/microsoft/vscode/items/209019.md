# Issue #209019 — Wrong indentation when editing text block in Python

[https://github.com/microsoft/vscode/issues/209019](https://github.com/microsoft/vscode/issues/209019)

## Summary

Still reproduces on main. The `(` before `"""` is not inside the string token, so the bracket-stripping logic from PR #210641 doesn't remove it before indentation evaluation. The Enter handler sees an unmatched `(` and indents the new line to align after it. The `tree-sitter-autoindent-candidate` label correctly identifies this as needing tree-sitter-level awareness to detect the cursor is inside a multiline string and skip indentation rules.

## Change summary

Pressing Enter inside a Python triple-quoted string incorrectly indents the new line based on the opening line's bracket structure instead of preserving the current column.

## Best solution

The EnterOperation._enter path needs to detect when the cursor is inside a string token (like the paste path does at line 419 of indentation.ts) and skip indentation rules in that case, or the tree-sitter autoindent implementation should handle this natively. The bracket `(` on `txt = ("""` is outside the string token, so the existing IndentationContextProcessor bracket-stripping doesn't help here.

## Evidence

The `(` on `txt = ("""` is tokenized as non-string, so IndentationContextProcessor (src/vs/editor/common/languages/supports/indentationLineProcessor.ts line 204-207) does NOT strip it.
- **paste path has guard:** src/vs/editor/contrib/indentation/browser/indentation.ts line 419 checks isStartOrEndInString for paste operations and bails out, but no equivalent guard exists in the Enter/typing path.
- **partial fix shipped:** PR #210641 (commit d309e11579e, May 2024) strips brackets from strings/comments/regex before evaluating indentation, but the `(` here is NOT inside a string token — it precedes the triple-quote that starts the string.
