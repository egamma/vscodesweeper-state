# Issue #318198 — Default themes should not include markdown suffix

[https://github.com/microsoft/vscode/issues/318198](https://github.com/microsoft/vscode/issues/318198)

## Summary

Valid feature request to make default theme scopes more generic (removing `.markdown` suffix) so other markup languages benefit from the same token colors. No change has been made on main.

## Change summary

Default themes use markdown-specific TextMate scopes (e.g. `punctuation.definition.quote.begin.markdown`) instead of generic markup scopes, preventing other markup languages from inheriting theme colors.

## Best solution

Update the token rules in `extensions/theme-defaults/themes/dark_vs.json` (and sibling themes) to use the generic `markup.*` scopes already defined in `colorThemeSchema.ts`, or at minimum drop the `.markdown` suffix so the rules match any markup grammar that follows the same naming hierarchy.
