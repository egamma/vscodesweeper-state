## Summary

Still reproduces on main: workspace symbol filtering breaks on Windows when the query contains `/` because `normalizeQuery` converts `/` to `\`, causing fuzzy scoring to fail against symbol names that legitimately contain forward slashes (like Spring Boot URL mappings). Issue #300999 is a duplicate of this one.

## Change summary

Ctrl+T workspace symbol search returns no results when typing `/` on Windows, because the fuzzy scorer's path normalization converts `/` to `\` before scoring against symbol names containing `/`.

## Best solution

Guard the path normalization in `normalizeQuery` (fuzzyScorer.ts line 898) so that it does not apply when scoring workspace symbols — or make `scoreFuzzy2` in `symbolsQuickAccess.ts` pass the original (un-path-normalized) query for symbol-name matching. The fix is localized to the interaction between `symbolsQuickAccess.ts` and `fuzzyScorer.ts`. Close #300999 as a duplicate of this issue.
