# Issue #292838 — "Open File" from diff editor should be in the context menu and command palette

[https://github.com/microsoft/vscode/issues/292838](https://github.com/microsoft/vscode/issues/292838)

## Summary

Valid discoverability request: the Git extension's `git.openFile` command is shown as an icon in the diff editor title bar but is not available in the tab context menu, and its command palette visibility is limited by a `when` clause that excludes the diff-editor scenario (requires `resourceScheme == file && scmActiveResourceHasChanges`).

## Change summary

Make the 'Open File' action from the diff editor accessible in the tab context menu and reliably visible in the command palette.

## Best solution

Add `git.openFile` to the `editor/title/context` menu in the Git extension's `package.json` (when `isInDiffEditor && resourceScheme =~ /^git$|^file$/`), and relax the command palette `when` clause to also match the diff-editor case (e.g. add an `|| isInDiffEditor` branch). The fix lives in `extensions/git/package.json`.

## Evidence

- **icon source:** The 'Open File' icon in the diff editor title bar is contributed by the Git extension at extensions/git/package.json line 2757: `git.openFile` in `editor/title` menu with when `isInDiffEditor && resourceScheme =~ /^git$|^file$/`.
- **command palette limitation:** The `commandPalette` entry for `git.openFile` requires `resourceScheme == file && scmActiveResourceHasChanges`, which excludes the diff editor view where resourceScheme is often `git`.
- **tab context menu absent:** No entry for `git.openFile` exists in the `editor/title/context` menu contribution in extensions/git/package.json.
- **core diff command exists but is different:** `DIFF_OPEN_SIDE` (`workbench.action.compareEditor.openSide`) is in the command palette as 'Open Active Diff Side' — but it opens whichever side has focus, not specifically the modified file, which is what the toolbar icon does.
