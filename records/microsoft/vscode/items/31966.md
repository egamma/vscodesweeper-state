# Issue #31966 — [folding] show comment first line for folded sections

[https://github.com/microsoft/vscode/issues/31966](https://github.com/microsoft/vscode/issues/31966)

## Summary

This feature request is still unimplemented on main. The folding decoration system still renders the first line of the range with an ellipsis; there is no logic to substitute a 'summary' line from comment blocks. The prerequisite (language-aware folding via #70794) was completed, but no work on this specific UX enhancement has landed.

## Change summary

When folding a multi-line doc-comment, show the first descriptive line (e.g., the summary text) instead of just the opening delimiter (`/**`).

## Best solution

Implement an optional folding range label provider (or leverage the existing `FoldingRangeProvider` API's `collapsedText` field if exposed) so that language extensions can supply a summary string for collapsed comment ranges. The decoration logic in `src/vs/editor/contrib/folding/browser/foldingDecorations.ts` would need to render this label instead of the generic `…` placeholder.
