# Issue #312980 — Support marketplace specification in extensions.json

[https://github.com/microsoft/vscode/issues/312980](https://github.com/microsoft/vscode/issues/312980)

## Summary

This request is about protecting users of alternative marketplaces (Open VSX / VSCodium) from name-squatting across marketplaces. VS Code itself only uses the Visual Studio Marketplace, so marketplace disambiguation in extensions.json is not something VS Code core needs to solve — it's a concern for forks that swap the marketplace endpoint.

## Change summary

Allow extensions.json to specify which marketplace an extension should be installed from, to prevent cross-marketplace name-squatting attacks.

## Best solution

This belongs in the downstream forks (VSCodium, etc.) or in the Open VSX ecosystem tooling. VS Code only connects to its own marketplace; the cross-marketplace trust problem doesn't arise in the canonical product. The related Open VSX issue (EclipseFdn/open-vsx.org#9961) confirms this is an Open VSX ecosystem concern.

## Vision fit

- **Fit:** rejected
- **Reason:** VS Code connects to a single marketplace; adding multi-marketplace plumbing to the core product adds complexity for a scenario that only affects forks, cutting against the 'lightweight and simple' value.
- **Evidence:** Values: 'Out of the box, VS Code is a text editor: lightweight and fast' and 'Simple, uniform UI and UI interactions.' Adding marketplace source resolution adds complexity for a scenario outside the canonical product.
