# Issue #266331 — `gitDecoration.ignoredResourceForeground` does not respect `git.decorations.enabled`

[https://github.com/microsoft/vscode/issues/266331](https://github.com/microsoft/vscode/issues/266331)

## Summary

Confirmed on current main: `GitIgnoreDecorationProvider` is unconditionally created in the `GitDecorations` constructor (line 285), bypassing the `git.decorations.enabled` toggle that controls the other decoration providers.

## Change summary

Disabling `git.decorations.enabled` still shows ignored-file coloring because `GitIgnoreDecorationProvider` is created unconditionally.

## Best solution

Move `GitIgnoreDecorationProvider` creation from the unconditional `disposables` list into the `enable()`/`disable()` lifecycle in `extensions/git/src/decorationProvider.ts`, so it is toggled alongside `GitDecorationProvider` and `GitIncomingChangesFileDecorationProvider`.
