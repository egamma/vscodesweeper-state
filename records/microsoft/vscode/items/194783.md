# Issue #194783 — Symlinks stops working in workspace

[https://github.com/microsoft/vscode/issues/194783](https://github.com/microsoft/vscode/issues/194783)

## Summary

The root cause — atomic writes destroying symlinks — was fixed by bpasero in commit e933c1d7b29 (cherry-picked as 956d8cba7d7 for 1.83.1). The `canWriteFileAtomic` guard now detects symlinks and falls back to non-atomic writes, preserving the link. This can be closed as completed.

## Change summary

Symlinks in the workspace stop being recognized and display their raw link content instead of the target file.

## Best solution

Keep the shipped fix (atomic writes disabled for symlinks, commit e933c1d7b29). No further change is needed — the reporter should verify on any build ≥ 1.83.1.
