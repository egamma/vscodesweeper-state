# Issue #240641 — Debug hover now is offset at the start of the full expression - far from pointer

[https://github.com/microsoft/vscode/issues/240641](https://github.com/microsoft/vscode/issues/240641)

## Summary

Still reproduces on main: the debug hover widget is positioned at the start of the evaluatable expression range rather than near the mouse cursor, causing it to appear far from the hovered token for long chained expressions.

## Change summary

Debug hover appears at the start of a long chained expression (far from the mouse pointer) instead of near the hovered token.

## Best solution

In `debugHover.ts` line 287, pass the original `position` (the cursor location) to `doShow` instead of `result.range.getStartPosition()`, so the content widget renders near where the user is hovering while still highlighting the full expression range.
