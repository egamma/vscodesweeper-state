# Issue #163918 — [Bug] Failed to open a jpg file in mergeEditor

[https://github.com/microsoft/vscode/issues/163918](https://github.com/microsoft/vscode/issues/163918)

## Summary

Valid feature request for binary/image file support in the merge editor. The merge editor is fundamentally text-based (extends AbstractTextResourceEditorInput) and has no binary file handling. Recent work on #324804 improved binary diff handling for custom editors, but the merge editor remains unaddressed. Keeping open as-labeled.

## Change summary

The merge editor fails to display binary image files during merge conflicts, showing an error instead of a usable image comparison.

## Best solution

Extend the merge editor to detect binary files and either delegate to a custom editor capable of rendering binary diffs/merges, or offer a simple 'choose theirs/ours' UI for binary conflicts. The recent #324804 work on binary diff editor fallbacks could inform the approach. This likely touches `src/vs/workbench/contrib/mergeEditor/browser/mergeEditorInput.ts` and the editor resolver integration.
