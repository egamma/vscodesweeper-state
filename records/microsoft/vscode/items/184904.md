# Issue #184904 — [json] Schema changes not reflected if schema file has jsonc extension.

[https://github.com/microsoft/vscode/issues/184904](https://github.com/microsoft/vscode/issues/184904)

## Summary

Still reproduces on main: the JSON language client file watcher only watches `**/*.json`, so changes to `.jsonc` schema files never notify the server to re-validate instance documents.

## Change summary

JSON schema changes in `.jsonc` files are not reflected in instance document validation until VS Code is restarted.

## Best solution

Extend the file system watcher glob on line 292 of `extensions/json-language-features/client/src/jsonClient.ts` from `'**/*.json'` to `'**/*.{json,jsonc}'` so that `.jsonc` schema file changes trigger re-validation.
