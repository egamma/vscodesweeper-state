# Issue #184904 — [json] Schema changes not reflected if schema file has jsonc extension.

[https://github.com/microsoft/vscode/issues/184904](https://github.com/microsoft/vscode/issues/184904)

## Summary

Still reproduces on main: the JSON language client file watcher only monitors `**/*.json`, so schema changes in `.jsonc` files never trigger revalidation until restart.

## Change summary

Schema changes in `.jsonc` files are not reflected in instance file validation without restarting VS Code.

## Best solution

Extend the file system watcher in `extensions/json-language-features/client/src/jsonClient.ts` line 292 to also watch `**/*.jsonc` files, so that schema revalidation is triggered when `.jsonc` schema files change.
