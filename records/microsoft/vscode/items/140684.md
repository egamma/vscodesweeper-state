# Issue #140684 — Web: Allow non-repository, non-filesystem temporary folders

[https://github.com/microsoft/vscode/issues/140684](https://github.com/microsoft/vscode/issues/140684)

## Summary

Valid feature request with 21 upvotes in the Backlog. The underlying `tmp` scheme and InMemoryFileSystemProvider infrastructure exists, but no user-facing command exposes 'create a temporary folder/workspace' in vscode.dev — the central ask remains unmet.

## Change summary

Allow users on vscode.dev to create an in-memory temporary folder workspace without requiring filesystem access or an existing GitHub repository.

## Best solution

Expose a user-facing command (e.g. 'New Temporary Workspace') in vscode.dev that opens an in-memory workspace backed by the existing `tmp` scheme / InMemoryFileSystemProvider, with upload-from-archive and download-as-archive options. The plumbing in `src/vs/workbench/browser/web.main.ts` (line 555) and the `isTemporaryWorkspace` utility already exist; the gap is the UI entry point and the archive import/export layer.
