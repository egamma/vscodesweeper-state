# Issue #265841 — Unable to drag image into chat prompt

[https://github.com/microsoft/vscode/issues/265841](https://github.com/microsoft/vscode/issues/265841)

## Summary

Team-authored feature request (MEMBER) about improving image drag-and-drop into chat when using external agents. Image drag-and-drop infrastructure exists in `chatDragAndDrop.ts` but the experience may not work correctly for all agent types. Hard keep-open applies.

## Change summary

Dragging an image into the chat prompt with an external agent opens the image in its own panel instead of attaching it as context.

## Best solution

Ensure that the image drag-and-drop path in `src/vs/workbench/contrib/chat/browser/widget/chatDragAndDrop.ts` works for external agent sessions (label `chat-external-agent`), so images are attached as context rather than opened as editor tabs. The gating on `isProposedApiEnabled(ext, 'chatReferenceBinaryData')` (line ~200) may be the root cause — if the external agent's extension doesn't enable that proposed API, the image type is not recognized and falls through to the default editor-open behavior.
