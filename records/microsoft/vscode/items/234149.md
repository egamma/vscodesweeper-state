# Issue #234149 — Allow extensions to augment an existing language's configuration, such as surroundingPairs

[https://github.com/microsoft/vscode/issues/234149](https://github.com/microsoft/vscode/issues/234149)

## Summary

Still a valid gap on main: language configuration entries overwrite rather than merge surroundingPairs (line 261 of languageConfigurationRegistry.ts), and surroundingPairs is not part of the public extension API. Keeping open as a legitimate extension-API enhancement request.

## Change summary

Allow extensions to augment (merge into) an existing language's surroundingPairs rather than replacing the entire set.

## Best solution

Change the merging logic in `src/vs/editor/common/languages/languageConfigurationRegistry.ts` (around line 254) to concatenate arrays for `surroundingPairs` (and potentially `autoClosingPairs`) rather than overwrite, and expose `surroundingPairs` in the `LanguageConfiguration` type in `vscode.d.ts` so extensions can contribute pairs without hacks.
