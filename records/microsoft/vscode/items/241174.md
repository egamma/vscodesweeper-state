# Issue #241174 — Ctrl+Shift+C does not open external terminal on Linux snap

[https://github.com/microsoft/vscode/issues/241174](https://github.com/microsoft/vscode/issues/241174)

## Summary

Active fix in progress — PR #298237 (snap core22 upgrade) explicitly lists Fixes #241174, and the issue carries the 'On Deck' milestone. Keeping open for the PR to land.

## Change summary

Ctrl+Shift+C fails to open an external terminal on Linux when VS Code is installed as a snap, because the snap's GTK_PATH export causes a glibc symbol lookup error in the host's gnome-terminal.

## Best solution

Land https://github.com/microsoft/vscode/pull/298237 which upgrades the snap to core22 base — this should resolve the library incompatibility that causes x-terminal-emulator to fail when GTK_PATH points into the snap's bundled GTK libraries.

## Evidence

- **root cause in source:** resources/linux/snap/electron-launch line 254 appends $SNAP/usr/lib/$ARCH/gtk-3.0 to GTK_PATH; when the host's x-terminal-emulator (gnome-terminal) loads, it picks up the snap's bundled libpthread from core20, causing a GLIBC_PRIVATE symbol lookup error.
- **open fixing PR:** https://github.com/microsoft/vscode/pull/298237 by dmitrivMS explicitly lists 'Fixes #241174' and upgrades the snap base from core20 to core22, which should resolve the library mismatch.
