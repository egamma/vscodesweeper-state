# Issue #193831 — Overflowing collapsed diff breadcrumbs aren't scrollable

[https://github.com/microsoft/vscode/issues/193831](https://github.com/microsoft/vscode/issues/193831)

## Summary

Still reproduces on main: the collapsed diff region breadcrumbs container uses `overflow: hidden` with `text-overflow: ellipsis` and no scroll mechanism, so long breadcrumb trails are simply clipped. The issue is valid and correctly labeled.

## Change summary

When collapsed diff region breadcrumbs overflow the editor width, they are clipped with no way to scroll or access the hidden portion.

## Best solution

Add horizontal scrolling (overflow-x: auto or a scroll-on-wheel handler) to the `.diff-hidden-lines .center` container or the `@others` breadcrumb div in `src/vs/editor/browser/widget/diffEditor/features/hideUnchangedRegionsFeature.ts` and `style.css`, matching the behavior of the file breadcrumb bar.
