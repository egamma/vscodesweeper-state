# Issue #323295 — Unicode support in embedded PowerShell terminal upon "git status" command

[https://github.com/microsoft/vscode/issues/323295](https://github.com/microsoft/vscode/issues/323295)

## Summary

This is git's default behavior, not a VS Code bug — git quotes non-ASCII characters in filenames with octal escapes when `core.quotePath` is true (the default). The terminal faithfully displays git's output; the fix is a git config change.

## Change summary

Running `git status` in the integrated terminal shows octal escape sequences instead of Unicode (Devanagari) filenames.

## Best solution

Close as a support question. The reporter needs to run `git config --global core.quotePath false` to make git display Unicode filenames directly. VS Code's file explorer shows filenames correctly because the git extension handles unquoting internally, but the terminal shows raw git output as expected.
