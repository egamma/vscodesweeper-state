# Issue #234708 — VSCode will silently never auto-update if it is not explicitly moved from the "Downloads" directory to "Application"

[https://github.com/microsoft/vscode/issues/234708](https://github.com/microsoft/vscode/issues/234708)

## Summary

Real, community-confirmed macOS bug where quarantine/app-translocation silently blocks auto-updates. Already assigned to update-service team members (deepak1556, dmitrivMS) and actively triaged — keeping open.

## Change summary

On macOS, VS Code silently fails to auto-update when the app is quarantined or translocated (e.g. run from Downloads), with no proactive notification to the user.

## Best solution

Detect the quarantine/translocation state at startup or when an auto-update attempt fails, and surface a clear notification telling the user to move the app to /Applications and remove the quarantine attribute — rather than failing silently. The error-handling path in `updateService.darwin.ts` (onError) currently swallows silent auto-update failures. The `preprocessError` function in `updateUtils.ts` is dead code and should be wired in or replaced with proactive detection.
