# Issue #326274 — VS Code update silently changed Copilot Agent model and consumed 2,100 AI credits

[https://github.com/microsoft/vscode/issues/326274](https://github.com/microsoft/vscode/issues/326274)

## Summary

Already triaged with `info-needed` by a team member who couldn't reproduce. The model-selection persistence logic in `chatModelSelectionLogic.ts` has an intentional design where persisted-as-default selections follow default changes (`shouldRestorePersistedModel`), which could explain the behavior if Sonnet 5 was the default when the user first used it and the default later changed to Fable 5. Keeping open — the existing `info-needed` triage is appropriate while awaiting version details.

## Change summary

User's Copilot Agent model selection changed from Claude Sonnet 5 to Claude Sonnet Fable 5 after a VS Code update, without notification, resulting in unexpected high credit consumption.

## Best solution

Once the reporter provides the previous version, the team can determine whether the `shouldRestorePersistedModel` logic in `src/vs/workbench/contrib/chat/browser/widget/input/chatModelSelectionLogic.ts` correctly distinguished an explicit user pick from a default-following selection. If the user explicitly chose Sonnet 5 and it was overridden, that's a bug in persistence; if it was the default and the default changed, the behavior is as-designed but the UX concern about silent model switches with cost implications is valid feedback. Credit refunds are outside VS Code's scope (GitHub Copilot billing).
