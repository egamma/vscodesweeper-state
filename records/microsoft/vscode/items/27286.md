# Issue #27286 — Explorer / Opened Editors Sorting

[https://github.com/microsoft/vscode/issues/27286](https://github.com/microsoft/vscode/issues/27286)

## Summary

This is a well-established tracking issue for Explorer and Open Editors sorting enhancements, assigned to the 'On Deck' milestone with 191 upvotes and an open community PR (#183155). The milestone is a scheduling decision that must be preserved, and the open PR means this stays with its human owners.

## Change summary

Provide richer sorting options in the Explorer (e.g. platform-aligned sort, custom file ordering via API, per-directory sort rules) and in the Open Editors list.

## Best solution

Several sort modes already ship (default, mixed, filesFirst, type, modified, foldersNestsFiles, plus lexicographic options). The remaining open items center on (1) platform-aligned sorting (#27759), (2) an extension API for custom sort orders (#12345), and (3) per-pattern/per-directory sort rules (#23231, #25724, #40679). The open community PR https://github.com/microsoft/vscode/pull/183155 proposes custom file explorer ordering. A maintainer should evaluate that PR against the remaining checklist items and decide whether to land, request changes, or close it in favor of an API-driven approach.
