# Issue #245187 — Multitudes of conhost zombies

[https://github.com/microsoft/vscode/issues/245187](https://github.com/microsoft/vscode/issues/245187)

## Summary

Conhost zombie processes accumulating on Windows after normal terminal use remains unresolved on main. While related agent-specific terminal accumulation (#298890) was fixed, and conpty was updated to 1.25, no targeted fix addresses the general conhost zombie scenario reported here. The issue has community confirmation and warrants investigation.

## Change summary

Closing or terminating VS Code terminals on Windows leaves orphan conhost.exe processes that accumulate over time, consuming memory.

## Best solution

Investigate whether conhost orphans survive `node-pty` kill on Windows with current conpty 1.25 — if they do, the pty host shutdown/dispose path in `src/vs/platform/terminal/node/terminalProcess.ts` may need to explicitly kill the conhost process tree. Alternatively, this may need an upstream fix in node-pty's Windows conpty backend.
