# Issue #209601 — code serve web --server-base-path PWA app does not include base path

[https://github.com/microsoft/vscode/issues/209601](https://github.com/microsoft/vscode/issues/209601)

## Summary

Still reproduces on main: the PWA manifest.json has a hardcoded `start_url: "/"` and is served as a static file without rewriting for the configured server-base-path, so PWA install/launch navigates to the wrong URL.

## Change summary

When `code serve-web --server-base-path` is set, the PWA app's start_url does not include the base path, causing PWA install and launch to fail.

## Best solution

Serve `manifest.json` dynamically (like the workbench HTML) so that `start_url` and icon `src` paths are prefixed with the resolved `basePath`. The fix belongs in `src/vs/server/node/webClientServer.ts` — either add a dedicated handler for the manifest route that rewrites the JSON, or template it similarly to how `workbench.html` uses `{{WORKBENCH_WEB_BASE_URL}}`.
