# Issue #209601 — code serve web --server-base-path PWA app does not include base path

[https://github.com/microsoft/vscode/issues/209601](https://github.com/microsoft/vscode/issues/209601)

## Summary

Still reproduces on main: the PWA manifest.json has a hardcoded `"start_url": "/"` that ignores `--server-base-path`. The manifest is served as a static file (resources/server/manifest.json) and is never rewritten to include the base path, so the PWA install/launch navigates to `/` instead of the configured base path.

## Change summary

When using `code serve-web --server-base-path`, the PWA manifest's `start_url` doesn't include the base path, so PWA installs launch to the wrong URL.

## Best solution

Either serve the manifest dynamically (rewriting `start_url` to include the base path, similar to how `WORKBENCH_WEB_BASE_URL` is injected into the HTML template), or change `start_url` to a relative value like `./` so it resolves relative to the manifest's own URL (which already includes the base path via `staticRoute`).
