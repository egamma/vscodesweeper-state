## Summary

Confirmed on current main: the `cp.exec('wsl.exe -l -q')` call in `terminalProfiles.ts` is missing `windowsHide: true`, causing a visible console window on systems without WSL. The fix is a one-line addition.

## Change summary

VS Code's WSL profile detection spawns a visible console window on Windows systems without WSL installed, interrupting the user.

## Best solution

Add `windowsHide: true` to the `cp.exec` options in `src/vs/platform/terminal/node/terminalProfiles.ts` line 352, matching the pattern already used in `src/vs/platform/agentHost/node/wslRemoteAgentHostHelpers.ts` and `src/vs/platform/update/electron-main/updateService.win32.ts`.
