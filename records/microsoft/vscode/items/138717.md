# Issue #138717 — Disable language detection for large files

[https://github.com/microsoft/vscode/issues/138717](https://github.com/microsoft/vscode/issues/138717)

## Summary

Still reproducible on main: language detection runs on large extensionless files without any size guard, potentially causing a freeze when the detected language triggers heavy tokenization. The gap remains unaddressed.

## Change summary

Opening a large file without an extension triggers language detection and can freeze the editor when the detected language activates expensive tokenization.

## Best solution

Add a size/line-count guard in `textFileEditorModel.ts#autoDetectLanguage` (or in the base `textEditorModel.ts#doAutoDetectLanguage`) that skips detection when the model exceeds the large-file threshold (aligned with `isTooLargeForTokenization`). Alternatively, expose a setting like `workbench.editor.languageDetectionMaxFileSize` similar to other large-file settings.
