# Issue #196143 — [Accessibility] quick fix includes headers in element count for screen reader users

[https://github.com/microsoft/vscode/issues/196143](https://github.com/microsoft/vscode/issues/196143)

## Summary

Still present on main: the ActionList does not provide custom getSetSize/getPosInSet to exclude headers and separators, so screen readers announce inflated item counts. Keeping open as a real accessibility bug.

## Change summary

Quick fix widget includes non-actionable header/separator items in the ARIA set size announced to screen readers, making the count incorrect.

## Best solution

Implement custom `getSetSize` and `getPosInSet` in the `ActionList` accessibility provider (src/vs/platform/actionWidget/browser/actionList.ts ~line 780) that count only `ActionListItemKind.Action` items and compute position relative to those, excluding headers and separators from the announced total.
