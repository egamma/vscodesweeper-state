# Issue #169951 — Right To Left text with numbers

[https://github.com/microsoft/vscode/issues/169951](https://github.com/microsoft/vscode/issues/169951)

## Summary

The underlying issue — incorrect visual ordering of RTL text with embedded numbers — stems from the editor's default LTR base direction. Recent work (decoration-based RTL support in commit 4846a428224, unicode-bidi:isolate in 18554b91593) has significantly improved RTL handling, but automatic per-line base direction detection is still not built into the core editor without extension-provided decorations. The issue remains relevant as a tracking item for full RTL support.

## Change summary

Hebrew (RTL) text containing numbers displays in wrong visual order because the editor defaults to LTR base direction.

## Best solution

The decoration-based RTL infrastructure (commit 4846a428224) now exists — the remaining gap is either a built-in heuristic for automatic line direction detection or clear documentation guiding users to extensions that provide RTL decorations. This could be addressed via a language-aware extension or a core `editor.defaultDirection` setting with an `auto` option.
