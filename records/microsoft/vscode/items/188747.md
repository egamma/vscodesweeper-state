# Issue #188747 — ptyHost starts when there is only a terminal renderer

[https://github.com/microsoft/vscode/issues/188747](https://github.com/microsoft/vscode/issues/188747)

## Summary

Team-authored performance bug about ptyHost unnecessarily starting when only a terminal renderer (no backing process) is opened. The issue remains valid — no fix commit was found on main, and the lazy connection pattern in localTerminalBackend.ts doesn't fully guard against this case.

## Change summary

The ptyHost process starts unnecessarily when a terminal with only a renderer (no process) is opened, wasting resources.

## Best solution

Guard the ptyHost startup path so that opening a renderer-only terminal (e.g. via Dev Containers: Show Container Log) does not trigger the ptyHost launch. The fix likely involves ensuring that terminal activation code paths that don't need a process don't call into _connectToDirectProxy or other ptyHost-launching APIs.
