# Issue #74237 — Add "type" to textDocument/references results (icon, filtering)

[https://github.com/microsoft/vscode/issues/74237](https://github.com/microsoft/vscode/issues/74237)

## Summary

Team-authored feature request for typed/categorized references in the references API (icons, filtering). Hard keep-open: author is a MEMBER. The C/C++ extension worked around it with a custom tree view, but the core API gap remains for other language extensions.

## Change summary

Allow language extensions to return typed/categorized references (confirmed, comment, string, inactive, unconfirmed) with distinct icons and user-facing filtering in the references panel.

## Best solution

This requires both an LSP protocol change (tracked at https://github.com/microsoft/language-server-protocol/issues/763) and a VS Code API/UI change to surface reference categories with icons and filtering in the Peek References and Find All References views. The C/C++ team built a custom tree view workaround, but a core solution would benefit all language extensions. Needs a product decision on whether to bring this into the shared references UI or leave it to per-extension tree views.
