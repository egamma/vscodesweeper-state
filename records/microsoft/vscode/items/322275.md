# Issue #322275 — GitHub API limit frequently exhausted

[https://github.com/microsoft/vscode/issues/322275](https://github.com/microsoft/vscode/issues/322275)

## Summary

The report is credible and traceable in source: for an AOSP-scale workspace with hundreds of git repos, the content exclusion fetch in `RemoteContentExclusion` issues one API call per 10-repo batch every 30 minutes, which can plausibly exhaust the GitHub REST API rate limit. This is a real bug in the Copilot extension's content exclusion fetching strategy for large multi-repo workspaces.

## Change summary

Working in a workspace with a very large number of Git repositories (e.g. AOSP) causes the Copilot extension's content exclusion rule fetching to exhaust the GitHub API rate limit, breaking Copilot Chat sessions.

## Best solution

The content exclusion fetching logic in `extensions/copilot/src/platform/ignore/node/remoteContentExclusion.ts` should be made more conservative for large multi-repo workspaces — e.g. by capping the number of repos queried per refresh cycle, increasing the staleness threshold, or coalescing fetches across repos that share the same GitHub organization. The 30-minute refresh interval (line 123) and per-10-batch loop (line 298) are the key levers.

## Evidence

- **source trace:** extensions/copilot/src/platform/ignore/node/remoteContentExclusion.ts line 123 triggers a full re-fetch when Date.now() - _lastRuleFetch > 30 minutes. Line 298 batches repos in groups of 10, each requiring a separate API call. For AOSP (~900+ repos), that's ~90+ API calls per refresh cycle.
- **rate limit mechanism:** GitHub REST API rate limit is 5,000 requests/hour for authenticated users. With ~90 batches every 30 minutes (2 cycles/hour = ~180 calls) plus other Copilot API calls (token refresh, model metadata, chat requests), the budget can be exhausted in workspaces with many repos.
- **related closed issue:** https://github.com/microsoft/vscode/issues/322915 was a similar rate-limit exhaustion bug (BYOK models triggering GitHub API rate limit) fixed in July 2026, confirming the Copilot extension can trigger this class of problem.
- **user evidence:** Reporter's logs mention content exclusion rule fetches taking up to 10 minutes, consistent with sequential batched API calls for hundreds of repos, with 35-minute recovery matching the GitHub rate limit reset window.
