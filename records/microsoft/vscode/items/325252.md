# Issue #325252 — "Generate Commit Message" does not work with BYOK

[https://github.com/microsoft/vscode/issues/325252](https://github.com/microsoft/vscode/issues/325252)

## Summary

Valid feature request: the 'Generate Commit Message' SCM button requires Copilot setup (GitHub sign-in) even when BYOK models are configured and working for other chat features. The button's action (`SetupAction`) always gates through `CHAT_SETUP_SUPPORT_ANONYMOUS_ACTION_ID` and then delegates to `product.defaultChatAgent?.generateCommitMessageCommand` — there's no BYOK-aware path in the SCM input widget. Routing to the SCM and AI areas for triage.

## Change summary

The 'Generate Commit Message' button in the SCM input box prompts for GitHub sign-in instead of using configured BYOK models, even though other AI features (chat summarization) work with BYOK.

## Best solution

The SCM input widget's commit-message generation flow should detect when BYOK models are available and configured as utility models (`chat.utilityModel`/`chat.utilitySmallModel`) and bypass the Copilot-specific setup gate. This likely requires changes in `src/vs/workbench/contrib/scm/browser/scmInput.ts` (the `SetupAction` registration and its `when` clause) and coordination with whoever owns `generateCommitMessageCommand` in the Copilot extension to support BYOK model resolution.
