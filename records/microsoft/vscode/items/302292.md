# Issue #302292 — Allow custom-guardrails for autopilot's report `task_complete`

[https://github.com/microsoft/vscode/issues/302292](https://github.com/microsoft/vscode/issues/302292)

## Summary

Team-authored feature request for extensible task-completion guardrails in autopilot — still unimplemented on main, keep open for maintainer judgment.

## Change summary

Allow users to define custom guardrails (linting, tests, builds) that the agent must verify before reporting task_complete.

## Best solution

Extend the `task_complete` tool in `src/vs/workbench/contrib/chat/common/tools/builtinTools/taskCompleteTool.ts` and the continuation message in the same file to support user-configurable completion criteria — either via appended prompt instructions, additional required input fields on the tool schema, or a setting that injects guardrail checks. The copilot-setup-steps / hooks infrastructure could also be a natural integration point.
