# Issue #291791 — Expose git.closeRepository and git.openRepository commands for programmatic use

[https://github.com/microsoft/vscode/issues/291791](https://github.com/microsoft/vscode/issues/291791)

## Summary

Valid feature request to expose a `closeRepository` method on the git extension API. The `openRepository` API already exists in `git.d.ts`, but there is no corresponding `close` method — the internal `git.close` command takes a `Repository` SCM object, not a path, and is not part of the public API. Routes to the git area owner.

## Change summary

Expose a programmatic `closeRepository` method on the git extension API so extensions can close repositories by path, complementing the existing `openRepository` API.

## Best solution

Add a `closeRepository(root: Uri): Promise<void>` method to the git extension API in `extensions/git/src/api/git.d.ts` and implement it in `extensions/git/src/api/api1.ts`, mirroring the existing `openRepository` pattern. The internal plumbing already exists via `Model.close()`. The `git.close` command could also be updated to accept a path string for programmatic callers, but the extension API is the cleaner surface.
