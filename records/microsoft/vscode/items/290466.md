# Issue #290466 — Restoring a sync state is failing

[https://github.com/microsoft/vscode/issues/290466](https://github.com/microsoft/vscode/issues/290466)

## Summary

Still reproduces on main: the `replace` method in abstractSynchronizer.ts calls `applyResult` with `force: false` but lacks the retry logic that `performSync` has for 412 PreconditionFailed errors, causing the restore action to fail when remote data changes between fetch and write.

## Change summary

Restoring a sync state via 'Show Synced Data → Restore' fails with a 412 Precondition Failed error because the replace flow doesn't retry on remote data conflicts.

## Best solution

Add retry logic to the `replace` method in `src/vs/platform/userDataSync/common/abstractSynchronizer.ts` similar to what `performSync` already does for `PreconditionFailed` — refetch latest remote data and retry the apply. Alternatively, pass `force: true` to `applyResult` in the replace flow since the user explicitly chose to restore.
