# Issue #166752 — Typing when simple filtering is enabled in the explorer should center the item

[https://github.com/microsoft/vscode/issues/166752](https://github.com/microsoft/vscode/issues/166752)

## Summary

Still valid on current main: the tree find/filter code only centers the matched element when it is completely off-screen, not when it is visible but near the viewport edge. The requested always-center behavior has not been implemented.

## Change summary

When typing to filter in the explorer (filter mode), the matched item should be centered in the viewport to provide more context below, instead of only scrolling when the item is off-screen.

## Best solution

In `src/vs/base/browser/ui/tree/abstractTree.ts`, the `applyPattern` method (around line 1227) currently guards `reveal(element, 0.5)` behind `getRelativeTop(element) === null`. Removing or relaxing that guard so the focused match is always revealed at 0.5 would deliver the requested centering behavior. A small, localized change in the tree find widget.
