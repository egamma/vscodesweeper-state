# Issue #248962 — Bad tab sizing with Editor Tab Wrap enabled

[https://github.com/microsoft/vscode/issues/248962](https://github.com/microsoft/vscode/issues/248962)

## Summary

The last tab in each wrapped row intentionally grows to fill remaining space via `flex-grow: 1` (CSS line referencing #113801). A maintainer confirmed this is as-designed in the related issue #270581, and a subsequent feature request (#325587) to change this behavior was also closed.

## Change summary

When wrapping tabs with `tabSizing: fit`, the last tab in each row stretches to fill remaining horizontal space, appearing oversized.

## Best solution

Keep current behavior as designed. The `flex-grow: 1` on `.sizing-fit.last-in-row` creates a homogeneous look across wrapped rows. Users wanting narrower tabs can use `tabSizing: shrink` or hide editor actions to free space.

  > Thanks for the report. The behavior you're seeing — the last tab in each wrapped row stretching to fill the remaining space — is intentional. The CSS uses `flex-grow: 1` on the last tab in each row (except the final row) to create a more uniform, homogeneous look across wrapped rows (see [#113801](https://github.com/microsoft/vscode/issues/113801)).
  > 
  > A maintainer confirmed this as designed in https://github.com/microsoft/vscode/issues/270581. If you'd prefer narrower tabs, you can try `"workbench.editor.tabSizing": "shrink"`, or right-click the editor actions area to hide some actions — this frees up space so fewer tabs need to wrap.
  > 
  > Closing as the current behavior is intentional.
