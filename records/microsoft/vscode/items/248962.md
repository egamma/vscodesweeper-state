# Issue #248962 — Bad tab sizing with Editor Tab Wrap enabled

[https://github.com/microsoft/vscode/issues/248962](https://github.com/microsoft/vscode/issues/248962)

## Summary

Valid visual bug — when tab wrapping is enabled, the CSS flex layout causes tabs to expand beyond their natural width. No fix has been applied on main since the issue was filed; the behavior is traceable in the current CSS.

## Change summary

Editor tabs with wrap enabled grow to fill available row width, appearing much larger than needed.

## Best solution

Adjust the flex-grow rules in `src/vs/workbench/browser/parts/editor/media/multieditortabscontrol.css` for wrapped tabs so that tabs (particularly with `sizing-fit`) don't expand beyond their content width except intentionally for the last tab in the last row. The fix belongs in the workbench-tabs area.
