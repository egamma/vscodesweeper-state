# Issue #321656 — Accessibility: Built-in VSCode webviews  fails with WCAG 2.1 SC 2.4.2 Page Titled (Level A) - Missing <title> element in <head> element

[https://github.com/microsoft/vscode/issues/321656](https://github.com/microsoft/vscode/issues/321656)

## Summary

Confirmed from source: the outer webview host document (index.html) has no <title> element in its <head>, violating WCAG 2.4.2. The inner iframe gets a title attribute dynamically, but the outer frame does not. This is a real, localized accessibility gap.

## Change summary

The outer webview host HTML document lacks a <title> element, failing WCAG 2.1 SC 2.4.2 (Page Titled, Level A).

## Best solution

Add a <title> element to the <head> of src/vs/workbench/contrib/webview/browser/pre/index.html (e.g. 'VS Code Webview') and wire it to update dynamically via the existing 'set-title' message handler so it reflects the webview's purpose.
