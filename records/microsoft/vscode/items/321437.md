# Issue #321437 — The JSON language server's NPM package (vscode-json-languageserver) is five years out of date

[https://github.com/microsoft/vscode/issues/321437](https://github.com/microsoft/vscode/issues/321437)

## Summary

Keep open as maintenance debt: current main still advertises and builds `vscode-json-languageserver` as an npm-consumable package, but the repo metadata remains at 1.3.4 and the report says npm has not been republished in years. This needs JSON extension/package-owner judgment rather than a triage close.

## Change summary

Republish or otherwise update the public `vscode-json-languageserver` npm package so external consumers can use the current JSON language server without building VS Code from source.

## Best solution

The JSON language feature owners should decide whether `vscode-json-languageserver` remains a supported standalone npm package. If yes, update the package version/release process and publish a current package; if no, update the README/package metadata so external consumers are not directed to stale npm bits.
