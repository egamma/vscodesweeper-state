## Summary

This appears to be an upstream Chromium/Electron color-profile handling difference between Windows and Linux — the same class of issue as the previously-resolved #68001. VS Code already exposes a `force-color-profile` runtime argument as a workaround. The report is old (2023, VS Code 1.78), has zero upvotes, and no one else confirmed the problem. ## Change summary

Syntax highlighting colors appear less saturated on Windows compared to Linux due to differences in how Chromium handles color profiles across platforms.

## Best solution

This is an Electron/Chromium platform difference in color profile handling, not a VS Code bug. The existing `force-color-profile` setting in `argv.json` (settable to `srgb`) is the intended workaround. If the reporter confirms it persists on current builds even with that workaround, escalate to an Electron upstream issue.
