# Issue #317183 — Agents: Steering brings me to the new session screen

[https://github.com/microsoft/vscode/issues/317183](https://github.com/microsoft/vscode/issues/317183)

## Summary

Team-authored issue with a confirmed reproduction and an open PR (#319202) that was deferred pending Agent Host work. Keeping open per policy.

## Change summary

Steering (sending a follow-up message) during a Copilot CLI session in the Agents window redirects the user to the new session screen instead of staying in the active session.

## Best solution

Once Agent Host is integrated, revisit the session-commit timeout handling in the Copilot CLI provider so that steering during an in-progress session preserves the active session rather than treating the timeout as a removal. The open PR https://github.com/microsoft/vscode/pull/319202 contains one approach; a maintainer decision on timing is needed.
