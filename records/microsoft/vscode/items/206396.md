# Issue #206396 — Can not install vscode on Debian 10

[https://github.com/microsoft/vscode/issues/206396](https://github.com/microsoft/vscode/issues/206396)

## Summary

The libxkbfile1 (>= 1:1.1.0) dependency in the .deb package breaks the stated Debian 10 compatibility (glibc 2.28 minimum), since Debian 10 only ships libxkbfile1 1:1.0.9. Debian 10 reached EOL in June 2024, so the practical impact is diminishing, but the gap between documented support and actual installability remains a valid packaging bug. The test/sanity matrix still lists Debian 10 as a target.

## Change summary

VS Code .deb package cannot be installed on Debian 10 because it requires libxkbfile1 >= 1:1.1.0, which is unavailable in that distribution.

## Best solution

Either lower the libxkbfile1 version requirement in `build/linux/debian/dep-lists.ts` to match what Debian 10 provides (if the binary is actually compatible), or formally drop Debian 10 from the supported platforms documentation and remove it from the test/sanity matrix — given Debian 10 EOL, the latter is probably the cleaner path.
