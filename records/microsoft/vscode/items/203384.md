# Issue #203384 — In File Save as Dialog, clicking on an file with an extension longer than 4 characters only brings puts file name, no extension, as the default file name 

[https://github.com/microsoft/vscode/issues/203384](https://github.com/microsoft/vscode/issues/203384)

## Summary

Confirmed upstream issue in macOS native file dialog behavior (via Electron API). Already correctly labeled `upstream` with a workaround (`files.simpleDialog.enable`). Keeping open as the canonical tracker for this platform limitation.

## Change summary

macOS native Save As dialog strips file extensions longer than 4 characters when clicking an existing file to populate the filename field.

## Best solution

This is owned by macOS/Electron's native dialog behavior. The workaround (`files.simpleDialog.enable`) is already documented. If an upstream Electron fix lands, VS Code gets it for free. Otherwise, this remains a known platform limitation.
