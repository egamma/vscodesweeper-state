# Issue #251251 — Feature Request: Add Temperature for Custom Agent Profiles

[https://github.com/microsoft/vscode/issues/251251](https://github.com/microsoft/vscode/issues/251251)

## Summary

Valid, highly-upvoted feature request (84 👍) that has been promoted to Backlog. Team members have stated no current plans to expose temperature in chat modes, but the issue remains open and under active community discussion — particularly around BYOK scenarios where hardcoded temperature values break certain models.

## Change summary

Allow users to configure LLM temperature (and potentially other model parameters) in custom chat modes or BYOK model settings.

## Best solution

The team should decide whether temperature belongs in chat mode files, in per-model BYOK configuration, or in a separate power-user settings surface. The immediate pain point (hardcoded temperature=0.1 breaking o3-mini/o4-mini/gpt-5 BYOK endpoints) could be addressed independently by omitting temperature for models that don't support non-default values — that narrower fix may warrant a separate bug issue (referenced as https://github.com/microsoft/vscode/issues/254675).
