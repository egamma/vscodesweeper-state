# Issue #319934 — Markdown `editor.colorDecorators: false` not honored on startup; likely race condition

[https://github.com/microsoft/vscode/issues/319934](https://github.com/microsoft/vscode/issues/319934)

## Summary

Fixed on main via PR #323090 (commit ddf88c30572, merged 2026-06-26). The issue carries milestone 1.127.0 and a related internal verification item, indicating it is likely mid-verification. Keeping open per milestone and verification rules.

## Change summary

Color decorators appear on startup in Markdown files despite `editor.colorDecorators: false`, due to a race where decorations aren't cleared when the setting is off.

## Best solution

The fix in commit ddf88c30572 adds a `removeAllDecorations()` call when `_isColorDecoratorsEnabled` is false during the update cycle, which should ship in 1.128. Verify on Insiders and close once confirmed.
