# Issue #320120 — Powershell doesn't open correctly for Next.js app router files

[https://github.com/microsoft/vscode/issues/320120](https://github.com/microsoft/vscode/issues/320120)

## Summary

This is the same PowerShell square-bracket cwd issue previously tracked and closed as upstream in #234935 and #70432. The root cause is a PowerShell bug (PowerShell/PowerShell#5752), not a VS Code defect.

## Change summary

PowerShell terminal fails to open at the correct directory when the path contains square brackets (common in Next.js app router).

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/70432 — this is a known PowerShell upstream limitation. The fix must come from PowerShell itself; VS Code cannot safely work around their path-handling bug without breaking other scenarios.

  > Thanks for the report. This is a known upstream PowerShell limitation — PowerShell interprets square brackets in paths as glob/wildcard characters, which causes `Set-Location` to fail when the working directory contains `[` or `]` (common in Next.js app router paths like `[lang]` or `[...slug]`).
  > 
  > This has been tracked before in https://github.com/microsoft/vscode/issues/234935 and the canonical VS Code issue https://github.com/microsoft/vscode/issues/70432. The root cause lives in PowerShell itself: https://github.com/PowerShell/PowerShell/issues/5752.
  > 
  > As a workaround, you can use Git Bash, Command Prompt, or WSL as you noted — those shells handle brackets in paths correctly.
