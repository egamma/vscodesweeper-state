# Issue #304888 — Inconsistency in tools definition for Copilot in VSCode and GitHub Copilot CLI

[https://github.com/microsoft/vscode/issues/304888](https://github.com/microsoft/vscode/issues/304888)

## Summary

Confirmed via source: VS Code's custom agent tools parser has no wildcard (`*`) handling — `tools: ["*"]` is treated as a literal tool name lookup that matches nothing, silently disabling all tools. The author is a team member; keeping open for their judgment on the fix approach.

## Change summary

Setting `tools: ["*"]` in a custom agent header should enable all tools (matching GitHub Copilot CLI behavior), but instead disables all tools because `*` is treated as a literal tool name.

## Best solution

Add wildcard support in the tools resolution path: in `PromptHeader.tools` (promptFileParser.ts) or in `toToolAndToolSetEnablementMap` (languageModelToolsService.ts), detect `*` and expand it to mean "all available tools". The validator in `promptValidator.ts` (`validateVSCodeTools`) should also recognize `*` as valid rather than flagging it as an unknown tool. Align the documentation to explicitly state the wildcard behavior.
