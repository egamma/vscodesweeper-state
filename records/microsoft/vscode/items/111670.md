# Issue #111670 — Consider adding a clear condition option

[https://github.com/microsoft/vscode/issues/111670](https://github.com/microsoft/vscode/issues/111670)

## Summary

Team-authored feature request for a 'Clear Condition' context menu action on breakpoints. Not yet implemented on main — no such action exists in the current codebase. Keeping open per team-author policy.

## Change summary

Add a 'Clear Condition' context menu option to breakpoints so users don't have to edit-then-backspace-then-enter to remove a condition.

## Best solution

Add a 'Clear Condition' action in the `DebugBreakpointsContext` menu (alongside the existing 'Edit Condition...' action in `src/vs/workbench/contrib/debug/browser/breakpointsView.ts`) that calls `IDebugService.updateBreakpoints` to set the condition to `undefined`. Small, localized change in the debug breakpoints view.
