# Issue #311729 — Accessibility: Make chat action buttons persistently visible (or add setting) for voice/eye-tracker users

[https://github.com/microsoft/vscode/issues/311729](https://github.com/microsoft/vscode/issues/311729)

## Summary

Well-articulated accessibility feature request asking for persistently visible chat code block action buttons (copy, insert, run) for voice coders who cannot discover hover-only UI. The code block toolbar in `codeBlockPart.css` is indeed hidden by default (`opacity: 0`) and only revealed on `:hover`, `:focus-within`, or `.focused` — no accessibility-mode override exists. This is a real gap; routing to chat and accessibility areas.

## Change summary

Make chat code block action buttons (copy, insert, run in terminal) persistently visible or provide a setting so voice coders without eye trackers can discover and target them.

## Best solution

Add a CSS rule (or a setting-gated class) that makes `.interactive-result-code-block-toolbar` persistently visible when the user is in screen-reader-optimized mode or when an explicit setting like `accessibility.verbosity.chatCodeBlockActions` is enabled. The toolbar visibility is controlled in `src/vs/workbench/contrib/chat/browser/widget/chatContentParts/media/codeBlockPart.css` (lines 11–14, 49–54); a similar pattern for the response footer toolbar lives in `src/vs/workbench/contrib/chat/browser/widget/media/chat.css` (lines 276–284). The `.force-visibility` class already exists (line 56–59 of `codeBlockPart.css`) and could be toggled from TypeScript when accessibility mode is active.
