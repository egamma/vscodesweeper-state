# Issue #159135 — Some global properties in tasks.json don't  take effect

[https://github.com/microsoft/vscode/issues/159135](https://github.com/microsoft/vscode/issues/159135)

## Summary

Source-reproducible on current main: `TaskParser.from` calls `CustomTask.fillGlobals` for custom tasks but never applies globals to configuring tasks (contributed types like `cppbuild`), so global `presentation` and `options` are ignored for contributed task types.

## Change summary

Global `presentation` and `options` properties in tasks.json are not inherited by contributed/configuring tasks (e.g. type: cppbuild), only by custom tasks (type: shell/process).

## Best solution

In `TaskParser.from` (src/vs/workbench/contrib/tasks/common/taskConfiguration.ts ~line 1818), apply global `presentation` and `options` to configuring tasks the same way `CustomTask.fillGlobals` does for custom tasks. The fix is localized to that file.
