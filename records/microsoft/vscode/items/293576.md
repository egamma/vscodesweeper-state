# Issue #293576 — Support for ZAI Coding plan / Any Other Model as a custom model provider for GitHub Copilot

[https://github.com/microsoft/vscode/issues/293576](https://github.com/microsoft/vscode/issues/293576)

## Summary

Adding every third-party AI provider as a first-class built-in option is not scalable; Z.AI's OpenAI-compatible API can already be used via the existing 'OpenAI Compatible' custom endpoint option in BYOK configuration, which is the intended path for providers not bundled in the dropdown.

## Change summary

Add Z.AI Coding Plan models (GLM-4.5, GLM-4.7) as a built-in custom model provider in the Copilot provider dropdown.

## Best solution

Use the existing OpenAI-compatible custom endpoint BYOK option to connect to Z.AI's API. If Z.AI's API is not OpenAI-compatible, that gap is better addressed by improving the generic custom-endpoint support (already tracked via issues like https://github.com/microsoft/vscode/issues/312940 and https://github.com/microsoft/vscode/issues/319968) rather than adding a provider-specific integration to core.
