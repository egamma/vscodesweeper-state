# Issue #289101 — Proposal: Allow for a `reason` to be included in a `TextEdit`

[https://github.com/microsoft/vscode/issues/289101](https://github.com/microsoft/vscode/issues/289101)

## Summary

Valid architectural proposal to enrich `TextEdit` with source/reason metadata so the `TextModelEditSource` tracking system can identify the originating extension at edit-creation time rather than threading it separately. Keeping open as an under-discussion design proposal with an associated draft PR.

## Change summary

Add a `reason` (edit-source metadata) field to `TextEdit` so format providers and similar callers can annotate edits with their origin at creation time.

## Best solution

Decide whether `TextEdit` should carry optional source metadata (e.g. a `ProviderId` or lightweight reason tag) or whether the format/codeAction call sites should return `{ edits, providerId }` tuples instead. The relevant files are `src/vs/editor/common/languages.ts` (TextEdit interface), `src/vs/editor/common/textModelEditSource.ts` (EditSources registry), and `src/vs/editor/contrib/format/browser/format.ts` (consumer). The draft PR #289177 explores the latter approach.
