# Issue #131128 — Allow quick pick items to always be visible and sticky

[https://github.com/microsoft/vscode/issues/131128](https://github.com/microsoft/vscode/issues/131128)

## Summary

The core request — a positionally sticky section at the bottom of the quick pick exposed as an extension API — remains unimplemented. The existing `alwaysShow` prevents filtering-out but doesn't make items visually pinned to a fixed position. The internal pinning (PR #158037) is terminal-specific, not a general API. Keeping open as an under-discussion feature request below the 20-upvote backlog threshold.

## Change summary

Allow quick pick items to be grouped into a visually sticky section at the bottom of the picker, always visible regardless of scrolling or filtering.

## Best solution

Implement a general-purpose sticky-section concept in the quick pick API (e.g. a new `QuickPickItemKind` or a dedicated items-array slot) so extensions can declare items that remain pinned at the bottom. The internal `quickPickPin.ts` pattern could inform the design but the API needs to be extension-facing and position-aware, not just storage-based.
