# Issue #224882 — Custom title bar not hiding on startup in Zen mode

[https://github.com/microsoft/vscode/issues/224882](https://github.com/microsoft/vscode/issues/224882)

## Summary

Still reproducible on main: zen mode restoration on startup does not preserve the user's hidden-titlebar preference. The Oct 2024 fix (788e7891eb7) for the opposite problem (#215408) removed zen mode's influence on titlebar visibility entirely, but didn't address this scenario where the titlebar reappears on restart.

## Change summary

Custom title bar reappears on startup when zen mode is active, even though the user explicitly hid it before entering zen mode.

## Best solution

The zen mode restore path in `src/vs/workbench/browser/layout.ts` (around line 1189) should save and restore the user's `window.customTitleBarVisibility` state as part of `zenModeExitInfo`, similar to how it already saves sidebar/panel/auxiliaryBar visibility.
