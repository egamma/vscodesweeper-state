# Issue #236828 — 🚨 Stop adding comment when enter is pressed

[https://github.com/microsoft/vscode/issues/236828](https://github.com/microsoft/vscode/issues/236828)

## Summary

Still reproduces on main: PR #321230 regressed the onEnter comment-continuation rule by removing the `\S` guard, causing `//` inside strings/URLs to trigger comment insertion. PR #322668 is open but not yet merged.

## Change summary

Pressing Enter after `//` in non-comment contexts (URLs, template literals, string values) incorrectly inserts `//` on the new line.

## Best solution

Merge the pending fix PR https://github.com/microsoft/vscode/pull/322668 (or a corrected version) that restores the `\S` requirement in the `beforeText` regex in `extensions/typescript-basics/language-configuration.json` (line 263). The rule should only fire when the cursor is inside actual comment body text, not at the start of `//`.
