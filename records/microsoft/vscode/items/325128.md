# Issue #325128 — AutoPilot mode for agent with handoffs results in infinite loop

[https://github.com/microsoft/vscode/issues/325128](https://github.com/microsoft/vscode/issues/325128)

## Summary

Confirmed in source: autopilot mode auto-triggers `send: true` handoffs without guarding against self-referential loops. When a custom agent's handoff targets itself, every response triggers another handoff indefinitely, freezing the chat.

## Change summary

Autopilot mode with a custom agent whose handoffs point back to the same agent creates an infinite submit loop.

## Best solution

Add a guard in `chatWidget.ts` around line 1350 to detect and break self-referential handoff cycles — e.g. skip auto-send when the handoff target matches the current response agent, or cap consecutive auto-sends to the same agent. The fix is small and localized to the autopilot auto-send path.
