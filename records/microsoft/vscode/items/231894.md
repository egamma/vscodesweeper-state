# Issue #231894 — enable the edit > show writing tools menu for editor

[https://github.com/microsoft/vscode/issues/231894](https://github.com/microsoft/vscode/issues/231894)

## Summary

Still valid: the upstream Electron fix (electron/electron#45138) landed in Electron 36 and VS Code ships Electron 42, but VS Code hasn't adopted the new `frame` option in `menu.popup()` to enable Writing Tools in context menus, nor wired up the Edit menu integration. The issue has 31 upvotes and remains actionable.

## Change summary

macOS Writing Tools menu items are not functional in VS Code's editor — neither in the Edit menu nor in the right-click context menu — despite the upstream Electron API now being available.

## Best solution

Pass the `frame` option (from the renderer's `WebFrameMain`) to `menu.popup()` in `src/vs/base/parts/contextmenu/electron-main/contextmenu.ts` so macOS Writing Tools, Autofill, and Services appear and work in context menus. The Edit menu integration in `src/vs/platform/menubar/electron-main/menubar.ts` may also need adjustment. This is a VS Code-side integration task now that the upstream blocker is resolved.
