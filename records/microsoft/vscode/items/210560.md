# Issue #210560 — msCompile problem matcher does not match note: lines which are important for context

[https://github.com/microsoft/vscode/issues/210560](https://github.com/microsoft/vscode/issues/210560)

## Summary

Still valid on main: the msCompile problem matcher regex only matches `error|warning|info` severities, not `note:` lines. The December 2025 update (f9c4d40c2fb) improved the matcher but did not add `note` support. This is more accurately a feature request — the matcher works as defined, but MSVC `note:` context lines would be valuable to surface.

## Change summary

The built-in msCompile problem matcher should also match MSVC `note:` diagnostic lines to provide instantiation context for template errors.

## Best solution

Add `note` to the severity alternation in the msCompile regex in `src/vs/workbench/contrib/tasks/common/problemMatcher.ts` (line 1517), mapping it to an info-level diagnostic. This gives users the template instantiation context that MSVC emits alongside errors.
