# Issue #316996 — [Bug] Incomplete Sanitization in conversationFeature.ts: Double-Escaping Bug Allows Shell Argument Injection in Git Commit Message

## Summary

Security-sensitive issue — keeping open for the team to handle per their internal processes. The reported ordering bug in the escape sequence is confirmed in current main source.

## Change summary

Shell argument injection possible via double-escaping bug in git commit message sanitization: escaping quotes before backslashes causes `\"` to become `\\"` which breaks out of the quoted argument.

## Best solution

Swap the replacement order in line 299 of conversationFeature.ts so backslashes are escaped first, then double quotes, then `$`. The CodeQL suppression comment is incorrect — it claims the ordering is safe but the second replace re-escapes the backslash introduced by the first. A team member should handle this per SECURITY.md processes.
