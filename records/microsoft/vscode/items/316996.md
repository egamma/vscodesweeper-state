# Issue #316996 — [Bug] Incomplete Sanitization in conversationFeature.ts: Double-Escaping Bug Allows Shell Argument Injection in Git Commit Message

[https://github.com/microsoft/vscode/issues/316996](https://github.com/microsoft/vscode/issues/316996)

## Summary

Security-sensitive issue — shell argument injection via incorrect escape ordering is confirmed present on current main. A team member (rwoll) has already engaged and directed toward SECURITY.md. Keeping open for maintainer handling.

## Change summary

The commit-message sanitization in conversationFeature.ts escapes double quotes before backslashes, producing `\"` which splits the shell argument and allows injection.

## Best solution

Swap the escape order at line 299 so backslashes are escaped first, then quotes, then dollar signs. The correct sequence is `.replace(/\\/g, '\\\\').replace(/"/g, '\\"').replace(/\$/g, '\\$')`. Alternatively, avoid shell-string construction entirely and use a structured API (e.g., pass args as an array to the terminal).
