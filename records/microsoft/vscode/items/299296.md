# Issue #299296 — Failed to render content: ModelService: Cannot add model because it already exists!

[https://github.com/microsoft/vscode/issues/299296](https://github.com/microsoft/vscode/issues/299296)

## Summary

Team-authored bug on the On Deck milestone — keeping open. The underlying model-collision defect in the chat code block renderer appears to still exist on main: while the pool now uses `generateUuid()` for URIs, the re-render path for large code blocks can race or re-create a model before the previous one is disposed.

## Change summary

Chat renderer throws 'Cannot add model because it already exists' when re-rendering large inline code blocks (20KB+), breaking all subsequent message rendering.

## Best solution

Guard `CodeBlockPart` model creation against an existing model at the same URI — either check `modelService.getModel(uri)` before calling `createModel`, or ensure the pool's release/dispose path fully removes the old model before a new one is created for the same logical code block. The fix likely lives in `src/vs/workbench/contrib/chat/browser/widget/chatContentParts/codeBlockPart.ts` and possibly `chatContentCodePools.ts`.
