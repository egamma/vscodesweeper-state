# Issue #224477 — Cannot use `\p{sc=...}` and lookahead/lookbehind simultaneously in the search panel with regular expressions

[https://github.com/microsoft/vscode/issues/224477](https://github.com/microsoft/vscode/issues/224477)

## Summary

This is a ripgrep upstream issue: `--engine auto` fails when a pattern requires PCRE2 (lookahead/lookbehind) but also contains `\p{sc=...}` syntax, because the Rust regex engine and PCRE2 use incompatible Unicode script property syntax (`=` vs `:`). VS Code passes `--engine auto` to ripgrep for all regex searches.

## Change summary

Combining `\p{sc=...}` Unicode script properties with lookahead/lookbehind in the search panel produces a regex parse error.

## Best solution

Either fix upstream in ripgrep's `--engine auto` to translate Unicode property syntax when falling back to PCRE2, or add a VS Code-side workaround in `getRgArgs()` (in `ripgrepTextSearchEngine.ts`) to rewrite `\p{sc=X}` to a PCRE2-compatible form when lookahead/lookbehind is detected in the pattern. The existing `unicodeEscapesToPCRE2` function shows the pattern for such transformations.
