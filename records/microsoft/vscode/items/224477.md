# Issue #224477 — Cannot use `\p{sc=...}` and lookahead/lookbehind simultaneously in the search panel with regular expressions

[https://github.com/microsoft/vscode/issues/224477](https://github.com/microsoft/vscode/issues/224477)

## Summary

Legitimate bug where ripgrep's `--engine auto` fails to handle patterns combining `\p{sc=...}` Unicode script properties with lookahead/lookbehind. VS Code always passes `--engine auto` for regex searches. The Rust regex engine supports `\p{sc=...}` but not lookahead; PCRE2 supports both, but the auto-detection in older ripgrep versions doesn't gracefully fall back. The ripgrep dependency has been bumped multiple times since this was filed (now `@vscode/ripgrep-universal` ^1.18.0), which may have fixed this, but I cannot confirm without running the binary.

## Change summary

Using `\p{sc=...}` Unicode script properties together with lookahead or lookbehind in the search panel regex produces a parse error.

## Best solution

Verify whether the current ripgrep version (shipped with `@vscode/ripgrep-universal` ^1.18.0) resolves this. If not, VS Code could mitigate in `ripgrepTextSearchEngine.ts` by detecting patterns containing both `\p{...}` and lookahead/lookbehind and explicitly using `--pcre2` instead of `--engine auto`. The relevant code is at line ~490 of `src/vs/workbench/services/search/node/ripgrepTextSearchEngine.ts`.
