# Issue #274211 — Signature help doesn't provide enough information for a screenreader user, at least in languages supporting overloads

[https://github.com/microsoft/vscode/issues/274211](https://github.com/microsoft/vscode/issues/274211)

## Summary

Confirmed on current main: the parameter hints aria announcement only reads the parameter label substring and documentation, omitting the full signature, parameter type, and overload context. No AccessibleView provider exists for signature help. This is a valid accessibility gap that remains unaddressed, and a team member has already engaged.

## Change summary

Signature help's screen reader announcement lacks the full overload signature and parameter type information, making it insufficient for blind users navigating overloaded methods.

## Best solution

Add an AccessibleView provider for parameter hints (similar to what exists for hover and suggest) that presents the full signature, current parameter with its type, documentation, and overload navigation context. The aria alert in `parameterHintsWidget.ts` could also be enriched to include the parameter type when available. The accessibility team (already tagged) should own the design decision on verbosity.
