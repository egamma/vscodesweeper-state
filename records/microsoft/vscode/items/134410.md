# Issue #134410 — Possible to disable automatic insertion of periods when using Codespaces on macOS?

[https://github.com/microsoft/vscode/issues/134410](https://github.com/microsoft/vscode/issues/134410)

## Summary

This should stay open because it has an assigned `On Deck` milestone, which is an explicit scheduling decision under the sweeper rules. The discussion also shows a current product gap/regression path around macOS double-space period insertion with the web/editor input stack, including a temporary `editor.editContext: false` workaround rather than a completed fix.

## Change summary

Allow VS Code/Codespaces on macOS to suppress or control automatic period insertion from double-space in editor input without requiring users to disable the macOS setting globally.

## Best solution

Keep this with the editor input/web owners to decide whether VS Code should opt the editor out of macOS text substitution when using EditContext, expose a per-editor setting, or add an Electron-backed substitutions menu/control for desktop. The workaround noted in the thread should not be treated as resolution unless maintainers decide the current platform behavior is intentional.
