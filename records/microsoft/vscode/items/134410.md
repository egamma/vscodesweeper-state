# Issue #134410 — Possible to disable automatic insertion of periods when using Codespaces on macOS?

[https://github.com/microsoft/vscode/issues/134410](https://github.com/microsoft/vscode/issues/134410)

## Summary

Valid feature request with 20 upvotes, assigned to 'On Deck' milestone — must stay open. The issue is that macOS's 'Add period with double-space' text substitution fires in the editor (especially since EditContext was enabled by default); no suppression logic has been implemented on main.

## Change summary

Suppress macOS's automatic period-on-double-space text substitution in the code editor, as other code editors do.

## Best solution

Implement suppression of macOS text replacement (specifically double-space → period) in the NativeEditContext path, similar to how other editors opt out. The workaround is `editor.editContext: false`, but a proper fix would disable this OS substitution at the input layer without losing other EditContext benefits. Likely touch `src/vs/editor/browser/controller/editContext/native/nativeEditContext.ts` or the Electron shell configuration.
