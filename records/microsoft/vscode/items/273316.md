# Issue #273316 — Clicking and dragging a folder in the explorer closes session

[https://github.com/microsoft/vscode/issues/273316](https://github.com/microsoft/vscode/issues/273316)

## Summary

Still reproducible on main: dragging a folder from the explorer tree and accidentally dropping it on the editor area triggers `ResourcesDropHandler.handleWorkspaceDrop`, which opens the folder via `hostService.openWindow` without confirmation — replacing the current workspace and destroying terminal sessions. The code path is intact and unguarded.

## Change summary

Dragging a folder in the explorer and accidentally dropping it outside the tree opens that folder as a workspace, closing the current session without confirmation.

## Best solution

Guard the workspace-opening path in `ResourcesDropHandler.handleWorkspaceDrop` (or the editor drop target) so that drops originating from within the explorer tree do not trigger workspace replacement. Alternatively, add a confirmation dialog before replacing an active workspace with running terminals. The fix would likely involve `src/vs/workbench/browser/dnd.ts` (around the `handleWorkspaceDrop` method) and possibly `src/vs/workbench/browser/parts/editor/editorDropTarget.ts`.
