# Issue #219210 — Quick Pick imprecise filtering breaks after 60 characters

[https://github.com/microsoft/vscode/issues/219210](https://github.com/microsoft/vscode/issues/219210)

## Summary

Still reproduces on main: the camelCase filter in filters.ts truncates the target word at 60 characters, causing fuzzy matching to ignore content beyond that limit. The code has a TODO acknowledging this should be revisited.

## Change summary

Quick Pick fuzzy/imprecise filtering stops working beyond the 60th character of item labels due to a hard-coded truncation in the camelCase filter.

## Best solution

Bump or remove the 60-character limit at `src/vs/base/common/filters.ts:308-309`. The `_maxLen` for the fuzzy scorer is already 128, so the camelCase path is the bottleneck. Any increase should be profiled for performance impact on large pick lists.
