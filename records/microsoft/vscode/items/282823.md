  > Thanks for raising this concern — controlling what data extensions can access is important. However, this is a question about GitHub Copilot extension configuration rather than a VS Code core feature request.
  > 
  > To restrict what Copilot can access:
  > 1. **Content exclusion** — your GitHub organization admin can configure [content exclusion policies](https://docs.github.com/en/copilot/managing-copilot/managing-github-copilot-in-your-organization/setting-policies-for-copilot-in-your-organization/excluding-content-from-github-copilot) to prevent Copilot from accessing specific files or repositories.
  > 2. **Per-language toggle** — use the `github.copilot.enable` setting to disable Copilot for specific languages.
  > 3. **Workspace Trust** — VS Code's Workspace Trust feature can restrict extension capabilities in untrusted workspaces.
  > 
  > For Copilot-specific privacy questions, please see [GitHub Copilot privacy docs](https://docs.github.com/en/copilot/responsible-use-of-github-copilot-features/github-copilot-general-privacy-notice) or open a discussion at https://github.com/orgs/community/discussions.
