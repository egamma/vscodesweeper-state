# Issue #168662 — Git - The `git submodule deinit` command is not reflected in the GUI

[https://github.com/microsoft/vscode/issues/168662](https://github.com/microsoft/vscode/issues/168662)

## Summary

Still reproduces on main. The git extension reads submodules from `.gitmodules` (which `deinit` does not modify) and never closes already-opened submodule repositories when their working directories are cleared. The 'On Deck' milestone triggers the hard keep-open rule.

## Change summary

After `git submodule deinit`, VS Code's Source Control panel still shows the deinitialized submodule until a full window reload.

## Best solution

The git extension's `Model.open()` opens submodule repos based on `.gitmodules`, but never re-evaluates whether a submodule repo is still valid (i.e. has an initialized working directory). The fix would need to either (a) watch for `.git/config` changes and close repositories for deinitialized submodules, or (b) during refresh, check whether each submodule path still contains an initialized git repo and close stale ones. Likely files: `extensions/git/src/model.ts` (submodule open/close logic) and `extensions/git/src/git.ts` (`getSubmodules` — could check init state).
