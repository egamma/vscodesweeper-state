# Issue #239300 — VSCode 1.96.4 Fails to Function Correctly After Installing NVIDIA GameReady Driver 572.16

[https://github.com/microsoft/vscode/issues/239300](https://github.com/microsoft/vscode/issues/239300)

## Summary

This is an upstream Chromium/Electron GPU rendering issue triggered by a specific NVIDIA driver version (572.16). The workaround (disabling hardware acceleration) confirms it lives in Chromium's GPU compositing layer, not in VS Code's own code. Should be labeled as upstream with gpu and chromium labels.

## Change summary

VS Code renders a black screen on mouse movement after installing NVIDIA GameReady Driver 572.16; disabling hardware acceleration works around it.

## Best solution

Label as upstream/gpu/chromium. The root cause is in Chromium's GPU compositor interacting with the specific NVIDIA driver. A commenter confirms the same symptom in Brave (also Chromium-based), reinforcing that this is not VS Code-specific. The established workaround (`disable-hardware-acceleration`) is available. Monitor for an Electron/Chromium update that resolves the driver incompatibility.
