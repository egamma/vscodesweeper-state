# Issue #284747 — Failure with symlinks

[https://github.com/microsoft/vscode/issues/284747](https://github.com/microsoft/vscode/issues/284747)

## Summary

This is a re-filing of the closed canonical issue #74104, which was triaged by a team member (bpasero) as upstream — VS Code receives the resolved path from the Electron/system dialog and cannot control symlink resolution at that layer. No new technical evidence changes that analysis.

## Change summary

When opening a workspace via a symlinked path on Linux, VS Code resolves the symlink, causing path mismatches that break extensions like vscode-java-test.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/74104 — the root cause is in Electron's handling of the system file dialog on Linux. If the community wants progress, an upstream Electron issue (or a VS Code feature to optionally preserve symlink paths from the dialog) would be the path forward.
