# Issue #263027 — feature request for editor cursor

[https://github.com/microsoft/vscode/issues/263027](https://github.com/microsoft/vscode/issues/263027)

## Summary

Both requested features — a distinct overtype cursor color and an inactive editor cursor style — remain unimplemented on main. The issue reached 20 upvotes and was promoted to Backlog; it should stay open.

## Change summary

Allow separate color customization for the overtype cursor and a distinct cursor style when the editor is unfocused.

## Best solution

Implement `editorCursorOvertype.foreground` as a new color token in `src/vs/editor/common/core/editorColorRegistry.ts` (defaulting to `editorCursor.foreground`) and wire it through the cursor rendering path. For `editor.cursorStyleInactive`, add a new EditorOption similar to `overtypeCursorStyle` and apply it in the cursor controller when the editor loses focus. Both are localized to the editor-rendering area.
