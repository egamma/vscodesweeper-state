# Issue #283164 — Copilot Chat Only Fetches the First #fetch URL in Prompts

[https://github.com/microsoft/vscode/issues/283164](https://github.com/microsoft/vscode/issues/283164)

## Summary

The issue describes inconsistent multi-URL fetch behavior that team members cannot reliably reproduce — it's model-dependent. The team has acknowledged the problem (tool schema shape may not optimally guide models) but no concrete fix has landed. Keeping open as a known limitation being tracked.

## Change summary

When multiple #fetch directives are used in a prompt, the LLM sometimes only invokes the fetch tool for the first URL.

## Best solution

Improve the fetch tool's schema definition to better guide models toward invoking it for all referenced URLs, potentially mirroring Claude's web-fetch-tool or OpenAI's fetch-tool shapes as Tyler suggested. Alternatively, implement pre-submit URL extraction so #fetch acts as a deterministic attachment rather than a model-mediated suggestion.
