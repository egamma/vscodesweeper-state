# Issue #280500 — Unable to drag an editor into a split editor

[https://github.com/microsoft/vscode/issues/280500](https://github.com/microsoft/vscode/issues/280500)

## Summary

The Shift modifier during drop is intentionally mapped to 'drop into editor' (insert content as text). The reporter expects it to open the file in a split, but VS Code explicitly prompts 'Hold Shift to drop into editor' — meaning insert data into the active editor. Without Shift, the file opens as a tab. This is as-designed.

## Change summary

When dragging a file from the OS file explorer and holding Shift before dropping into a split editor panel, the file path is inserted as text instead of opening the file in the panel.

## Best solution

Close as as-designed: Shift+drop deliberately triggers 'drop into editor content' (text insertion) rather than opening the file as a tab. Without Shift, the file opens normally. The UI prompt in the overlay states this. If the reporter wants the file opened in a specific split, they should drop without holding Shift.
