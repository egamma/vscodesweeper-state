# Issue #192462 — New terminal not opening in default project path in remote connection

[https://github.com/microsoft/vscode/issues/192462](https://github.com/microsoft/vscode/issues/192462)

## Summary

Confirmed and reproducible bug where new terminals in a remote (WSL) connection open with '/' as their cwd instead of the project directory. The maintainer (Tyriar) confirmed the pty host isn't receiving the cwd, and the TODO comment in `remoteTerminalBackend.ts` (`cwd: string, // TODO: This is ignored`) is still present on main — this is still an open defect.

## Change summary

New terminals opened via a WSL remote connection default to '/' instead of the workspace project directory.

## Best solution

Fix the cwd propagation in the remote terminal backend so the `cwd` parameter passed to `createProcess` in `remoteTerminalBackend.ts` is actually forwarded to the remote pty host. The `// TODO: This is ignored` comment at line 156 marks the exact gap. The remote channel's `createProcess` call already sends `shellLaunchConfig.cwd` via the DTO, so the fix likely lives on the server side where the pty is spawned — ensure the remote pty service uses the shell launch config's cwd when no custom cwd override is set.
