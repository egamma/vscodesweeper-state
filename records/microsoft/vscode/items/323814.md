# Issue #323814 — Native menus are incorrectly scaled on KDE Plasma 6.6 with Global Menu enabled

[https://github.com/microsoft/vscode/issues/323814](https://github.com/microsoft/vscode/issues/323814)

## Summary

Native menu scaling on KDE Plasma is an upstream issue — VS Code delegates native menu rendering entirely to Electron, which in turn uses GTK for menu drawing on Linux. The mismatch between VS Code's own UI scale and the GTK-rendered native menus is a known class of Electron/Chromium DPI handling problems. Keeping open for routing and upstream tracking.

## Change summary

Native context menus render at a larger scale than the rest of the VS Code UI on KDE Plasma 6.6 when using `window.menuStyle: native`.

## Best solution

Route to the menus/linux area owners for upstream tracking. The root cause is Electron/Chromium's GTK menu DPI handling — VS Code sets `window.menuStyle` to native and Electron draws the menus via GTK, which picks up a different scale factor than the Chromium content area. Related upstream work is tracked in https://github.com/microsoft/vscode/issues/41345 (heterogeneous DPI on Linux) and https://github.com/microsoft/vscode/issues/224515 (context menu sizing). The fix must come from Electron or Chromium.
