# Issue #305990 — Terminal in editor area prevents new tabs from opening in the expected editor group

[https://github.com/microsoft/vscode/issues/305990](https://github.com/microsoft/vscode/issues/305990)

## Summary

The reported behavior — new tabs opening in the editor group that contains the focused terminal — is how ACTIVE_GROUP routing works by design. However, there's a reasonable UX argument that non-terminal editors should prefer the non-terminal group. No fix found on main; keeping open for team confirmation.

## Change summary

When a terminal in the editor area has focus in a split group, opening a new tab places it in that terminal's group instead of the other editor group.

## Best solution

The terminal-editors team should confirm whether this is intended ACTIVE_GROUP behavior or whether new untitled file/tab actions should prefer a non-terminal editor group when the terminal has focus. If a fix is warranted, it would likely involve the editor group routing logic in the editor service or the new-file action handler.
