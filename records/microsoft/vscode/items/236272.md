# Issue #236272 — Title bar too big in some screens, does not scale with zoom

[https://github.com/microsoft/vscode/issues/236272](https://github.com/microsoft/vscode/issues/236272)

## Summary

Correctly triaged as an upstream Electron/XWayland issue. A team member confirmed the root cause is in the runtime's handling of per-monitor DPI under XWayland, and suggested filing upstream. The issue remains valid for tracking.

## Change summary

The custom title bar does not scale correctly when moving VS Code between monitors with different DPI on Linux (XWayland/Xorg), and zoom level changes do not affect the title bar.

## Best solution

File or link a corresponding Electron issue for WCO (Window Controls Overlay) scaling under XWayland. The fix lives in Electron's handling of per-monitor DPI changes; VS Code core cannot address it. For native Wayland users, https://github.com/microsoft/vscode/issues/207033 tracks enabling Wayland natively which sidesteps the XWayland path.
