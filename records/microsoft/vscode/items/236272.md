# Issue #236272 — Title bar too big in some screens, does not scale with zoom

[https://github.com/microsoft/vscode/issues/236272](https://github.com/microsoft/vscode/issues/236272)

## Summary

Already correctly triaged as an upstream Electron/XWayland scaling issue. A VS Code team member confirmed this is a runtime issue that needs an Electron bug report. No fix has landed on main; the issue should remain open as an upstream tracker.

## Change summary

Title bar does not rescale when moving VS Code between monitors with different DPI on Linux (XWayland/Xorg), and zoom level changes do not affect the title bar size.

## Best solution

File an upstream Electron issue with a minimal WCO repro (as deepak1556 suggested). For users, native Wayland mode or setting GDK_SCALE / --force-device-scale-factor are workarounds. The native Wayland default tracked in https://github.com/microsoft/vscode/issues/207033 may eventually make this moot for Wayland users.
