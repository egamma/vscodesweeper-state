# Issue #206777 — File nesting for enumeration of files with same basename

[https://github.com/microsoft/vscode/issues/206777](https://github.com/microsoft/vscode/issues/206777)

## Summary

Valid feature request for enhanced file nesting patterns to support grouping enumerated files (e.g. `basename_0.ext`, `basename_1.ext`) under a parent file. The current nesting pattern system does not support this kind of numeric-suffix grouping. Already labeled `under-discussion` and `file-nesting`, which is appropriate — keeping open for the community process.

## Change summary

Allow file nesting patterns to group files with numeric suffixes (e.g. `solution_00000000_0.pvu`) under a common parent file (e.g. `solution_00000000.pvtu`).

## Best solution

Extend the file nesting pattern engine in `src/vs/workbench/contrib/files/common/explorerModel.ts` to support numeric wildcard or range captures, so patterns like `$(basename)_*.$(extension)` can nest enumerated files under a parent. Related issue https://github.com/microsoft/vscode/issues/207717 asks for similar prefix-based grouping and could potentially share an implementation approach, though the mechanisms differ (virtual folders vs. nesting).
