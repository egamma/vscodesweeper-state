# Issue #283739 — VScode AI chat function doesn't work when opening code files from a remote SSH server

[https://github.com/microsoft/vscode/issues/283739](https://github.com/microsoft/vscode/issues/283739)

## Summary

This is a network configuration question, not a VS Code bug. The diagnostic logs clearly show the remote SSH server cannot reach Copilot API endpoints (ECONNRESET errors on api.individual.githubcopilot.com and proxy.individual.githubcopilot.com), while the local Windows machine can. The user's rebuttal that it works locally actually confirms the diagnosis: the Copilot extension runs on the remote server in SSH scenarios and uses that server's network path, which is blocked.

## Change summary

Copilot Chat AI models are unavailable when connected to a remote SSH server because the remote host cannot reach Copilot API endpoints.

## Best solution

The user needs to ensure their remote SSH server's network/firewall allows outbound HTTPS connections to the Copilot service endpoints listed in GitHub's documentation (https://docs.github.com/en/copilot/troubleshooting-github-copilot/troubleshooting-firewall-settings-for-github-copilot). This is a server-side network configuration issue, not a VS Code defect.
