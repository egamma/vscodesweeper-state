# Issue #297843 — Fix "Directory Already Exists" Error When Re-adding a Plugin Marketplace Repository in GitHub Copilot

[https://github.com/microsoft/vscode/issues/297843](https://github.com/microsoft/vscode/issues/297843)

## Summary

Fixed on main: concurrent/repeated marketplace repo clones are now serialized via SequencerByKey, and the clone path checks for an existing directory before attempting to clone. Shipped in 1.112.0.

## Change summary

Re-adding a plugin marketplace repository URL causes a fatal Git clone error because the directory already exists locally.

## Best solution

Keep the shipped fix (commit e7e6cbe3a9e, 1.112.0). The `ensureRepository` method now checks `repoExists` inside a sequencer queue and returns early if the directory is already present. No further change needed.
