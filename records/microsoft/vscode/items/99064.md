Team-authored issue (MEMBER) about contextual title display inconsistency in views moved to the panel — must keep open per policy. The panel and view title code has evolved significantly since 2020 but the containerTitle logic still exists; unclear if the specific inconsistency persists.

## Change summary

When a view with a contextual title is moved to the horizontal panel, the full title is not shown until the view is clicked, creating an inconsistent display.

## Best solution

Verify whether this inconsistency still reproduces with the current panel/view architecture (substantial rework has happened since 2020).
