# Issue #322201 — Context window dial assumes 1M tokens when 200K tokens are selected

[https://github.com/microsoft/vscode/issues/322201](https://github.com/microsoft/vscode/issues/322201)

## Summary

Fixed on main: the context-usage gauge denominator now resolves from the user's selected context-size tier (e.g. 200K) instead of the model's full native window (1M). Multiple commits in late June addressed this exact scenario.

## Change summary

The context window dial uses the model's full 1M native window as the denominator instead of the user-selected 200K context-size tier, making the gauge appear nearly empty at 78K.

## Best solution

Keep the shipped fix from commit 4972ac9c099 and its follow-ups; no further change is needed. The `resolveContextWindowInputTokens` function now correctly resolves configured contextSize → schema default tier → full native window, so the gauge denominator matches the request path.
