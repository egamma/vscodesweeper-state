# Issue #322409 — Context window size in donut doesn't match selection

[https://github.com/microsoft/vscode/issues/322409](https://github.com/microsoft/vscode/issues/322409)

## Summary

Team-authored issue filed during test-plan validation (#322259). The context-window picker shows the input-only budget (272K) while the donut widget shows input + output (272K + 128K = 400K), creating a confusing discrepancy. Keeping open for the team to decide whether the donut label or the picker label should be aligned.

## Change summary

The context-window size shown in the donut/circular progress widget does not match the size shown in the model picker — the donut includes maxOutputTokens while the picker shows input tokens only.

## Best solution

Decide on one consistent number to surface across both the picker and the donut. If the donut intentionally shows the full window (input + output), the picker or its tooltip should clarify that its value is input-only; alternatively, show the same denominator in both places with the output reserve clearly broken out. The relevant code is in `resolveContextWindow()` in `src/vs/workbench/contrib/chat/browser/widgetHosts/viewPane/chatContextUsageWidget.ts` (line ~406: `totalContextWindow = maxInputTokens + maxOutputTokens`).
