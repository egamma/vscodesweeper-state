# Issue #236165 — Customize shortcut to show the context menu for the highlighted File/Folder in the Files Explorer

[https://github.com/microsoft/vscode/issues/236165](https://github.com/microsoft/vscode/issues/236165)

## Summary

Valid feature request: the context menu trigger in the explorer tree is handled at the DOM level in the base list widget (Shift+F10 / ContextMenu key), not via a VS Code keybinding command — so users cannot customize it. No duplicate found. Keeping open under discussion.

## Change summary

Allow users to customize the keyboard shortcut for showing the context menu on the focused item in the Files Explorer.

## Best solution

Expose a keybinding-customizable command (e.g. `filesExplorer.action.showContextMenu` or a generic `list.showContextMenu`) that triggers the context menu on the focused tree item. The current Shift+F10 handling lives in `src/vs/base/browser/ui/list/listWidget.ts` (line ~1458–1482) as a raw DOM keydown/keyup listener, so the fix would involve surfacing this as a proper VS Code command registered with the keybinding service. A generic `list.showContextMenu` scoped to list/tree focus contexts would benefit all tree views, not just the explorer.
