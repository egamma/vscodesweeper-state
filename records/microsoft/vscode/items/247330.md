# Issue #247330 — Next edit suggestion needs a unique verbosity key

[https://github.com/microsoft/vscode/issues/247330](https://github.com/microsoft/vscode/issues/247330)

## Summary

Team-authored issue, still open on main: the inline completions accessible view provider uses a single verbosity key for both inline suggestions and next edit suggestions, with no separation. Keeping open.

## Change summary

Next edit suggestions share the same accessibility verbosity setting key as inline completions, preventing independent management of the hint.

## Best solution

Add a new `AccessibleViewProviderId` (e.g. `InlineEdits`) and a separate verbosity setting key (e.g. `accessibility.verbosity.inlineEdits`) in the accessibility configuration, then split the accessible view content provider in `inlineCompletionsAccessibleView.ts` so that the `inlineEditVisible` context uses the new provider with its own verbosity key.
