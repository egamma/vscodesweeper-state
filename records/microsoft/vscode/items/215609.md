## Change summary

Deleting or renaming a large directory (100k files) in the explorer causes VS Code to hang, whereas the equivalent CLI operation completes without freezing the UI.

## Best solution

The underlying issue likely involves the file watcher emitting events for each file in the deleted directory, causing the explorer service (`src/vs/workbench/contrib/files/browser/explorerService.ts`) to process a torrent of change events and refresh the tree repeatedly. A fix would involve coalescing or suppressing watcher events during known bulk file operations (delete/rename) initiated from the explorer, or throttling the explorer refresh more aggressively. However, the report needs an updated reproduction on a current build with a CPU profile before a targeted fix can be pursued.
