# Issue #231304 — Folders in explorer disappear when clicked

[https://github.com/microsoft/vscode/issues/231304](https://github.com/microsoft/vscode/issues/231304)

## Summary

Still reproduces on main: a TypeScript parameter-property pitfall in the IgnoreFile constructor causes nested .gitignore patterns to match the containing directory itself, hiding it from the explorer.

## Change summary

When `explorer.excludeGitIgnore` is enabled and a subdirectory contains a .gitignore whose pattern matches the subdirectory's own name, expanding that directory causes it to disappear from the file explorer.

## Best solution

Fix the IgnoreFile constructor in `src/vs/workbench/services/search/common/ignoreFile.ts` so that `parseIgnoreFile` receives the slash-normalized location. Line 24 passes `this.location` (set before the local `location += '/'` on line 22), so the `startsWith` guard on line 118 matches the directory itself. Changing line 24 to use the local `location` (or normalizing `this.location` before the parameter property is set) ensures patterns scope only to children of the .gitignore's directory.

## Evidence

- **root cause (source trace):** In src/vs/workbench/services/search/common/ignoreFile.ts, the constructor declares `private readonly location: string` as a parameter property. TypeScript sets `this.location` before the body executes. Line 22 (`location += '/'`) only rebinds the local variable; `this.location` keeps the original value without a trailing slash. Line 24 passes `this.location` to `parseIgnoreFile`, so `dirPath` in `isPathIgnored` (line 118) lacks the trailing `/`. `path.startsWith(dirPath)` then matches the directory itself (e.g. `/workspace/bfdd`.startsWith(`/workspace/bfdd`) → true), allowing the glob `**/bfdd` to hide it.
- **confirmed by maintainer:** Comment by lramos15 (MEMBER) on 2024-11-06: 'Yes this is a bug on our end and I have marked it as such.'
- **no fix on main:** git log on ignoreFile.ts shows no commit addressing this issue; the constructor still passes `this.location` on line 24 of current main.
- **additional prefix bug:** The missing trailing slash also means sibling directories whose names share a prefix (e.g. `bfdd_other`) could be incorrectly matched, since `/workspace/bfdd_other`.startsWith(`/workspace/bfdd`) is true.
