This is almost certainly a kernel or hardware issue on the reporter's Debian 12 host (faulty RAM, disk, or a kernel bug in the 6.1.0-10-amd64 build). VS Code runs entirely in userspace and cannot trigger a kernel panic. The reporter should check dmesg/kernel logs after a crash, test with an updated kernel, and run memtest86. If reproducible with a newer kernel, it should be filed against the Debian kernel package. Close as upstream/not-reproducible if no further info arrives.
