# Issue #313092 — GPT 5.5 doesn't detect agent changes properly

[https://github.com/microsoft/vscode/issues/313092](https://github.com/microsoft/vscode/issues/313092)

## Summary

Team-authored issue (MEMBER) — must keep open for maintainer judgment. Reports that GPT 5.5 fails to detect agent switches in multi-agent sessions, while Opus models work correctly. Likely a model-level or prompt-construction issue in how agent identity/permissions context is conveyed to the model.

## Change summary

GPT 5.5 does not recognize agent changes in multi-agent sessions, continuing to act as the previous agent's persona instead of adopting the new agent's permissions and identity.

## Best solution

Investigate whether the agent-switch context (system prompt, tool permissions) is being properly reconstructed or conveyed when the model changes to GPT 5.5 — the issue may be in how the conversation history or system message is formatted for that model family versus Opus. If GPT 5.5 handles context windows or system prompts differently, the agent harness may need model-specific adjustments. Could also be an upstream model behavior issue worth reporting to OpenAI.
