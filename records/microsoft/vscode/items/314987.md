# Issue #314987 — Keyboard shortcut do delete a word at the caret

[https://github.com/microsoft/vscode/issues/314987](https://github.com/microsoft/vscode/issues/314987)

## Summary

Keep this open as a small editor feature request: VS Code already has a keybindable Delete Word command for the word at the cursor, but current main does not show a default keybinding on that action.

## Change summary

Add a default keyboard shortcut for deleting the word at the caret.

## Best solution

Route to the editor area to decide whether `deleteInsideWord` should receive a default `kbOpts` binding, or whether the existing keybindable command is sufficient documentation/customization surface.

  > Thanks for the request. ## Evidence

- **existing command:** Current main registers `deleteInsideWord` in `src/vs/editor/contrib/wordOperations/browser/wordOperations.ts` with label "Delete Word" and metadata description "Delete the word at the cursor".
- **no default shortcut in action:** `DeleteInsideWord` has no `kbOpts` in its `EditorAction` registration, while nearby word-delete commands such as `deleteWordRight` do define `kbOpts`.
- **behavior covered by tests:** `src/vs/editor/contrib/wordOperations/test/browser/wordOperations.test.ts` has `deleteInsideWord` tests for empty lines, whitespace, punctuation, words, and the `onlyWord` argument.
- **recent related change:** Commit `2539ea18ae1` added an `onlyWord` argument for `deleteInsideWord` via https://github.com/microsoft/vscode/pull/283433, confirming the command is actively maintained.
