## Summary

Team-authored feature request for improving partial accept of inline completions to autoclose brackets/quotes. Still open and valid — no fix has landed on main.

## Change summary

When partially accepting an inline completion, brackets and string quotes should be autoclosed just as they are during normal typing.

## Best solution

Modify the partial-accept path in the inline completions model to route text through the typing/autoclose machinery (or replicate its bracket-closing logic) so that accepted partial text gets matching close characters. The PoC patch in the issue body shows the direction; the challenge is handling edge cases like pre-existing nearby close brackets.
