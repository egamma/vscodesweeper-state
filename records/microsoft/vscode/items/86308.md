# Issue #86308 — [themes] Minimal icon theme should use icon font

[https://github.com/microsoft/vscode/issues/86308](https://github.com/microsoft/vscode/issues/86308)

## Summary

Team-authored feature request (MEMBER). The minimal icon theme still uses SVG icon definitions on current main — no migration to codicon/icon font has occurred. The infrastructure for fontCharacter in file icon themes exists in fileIconThemeData.ts, but the minimal theme hasn't been updated to use it.

## Change summary

The built-in minimal icon theme should use codicon/icon font instead of SVGs so icons render with LCD text rendering consistency.

## Best solution

Update `extensions/theme-defaults/fileicons/vs_minimal-icon-theme.json` to replace SVG `iconPath` definitions with `fontCharacter`/`fontId` references to codicons. The infrastructure already exists in `src/vs/workbench/services/themes/browser/fileIconThemeData.ts`. This depends on whether the codicon API for file icon themes (https://github.com/microsoft/vscode/issues/90911) is sufficiently complete.
