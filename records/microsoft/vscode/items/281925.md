# Issue #281925 — Apply the hover.hidingDelay to when hover.sticky is false

[https://github.com/microsoft/vscode/issues/281925](https://github.com/microsoft/vscode/issues/281925)

## Summary

Team-authored feature request to honor `editor.hover.hidingDelay` even when `editor.hover.sticky` is false. Keeping open per policy for member-filed issues.

## Change summary

The `editor.hover.hidingDelay` setting is ignored (treated as 0ms) when `editor.hover.sticky` is set to false.

## Best solution

Update the hover hiding logic in the editor hover controller so that `hidingDelay` is respected independently of the `sticky` setting, giving users a brief grace period before the hover dismisses even in non-sticky mode.
