# Issue #305969 — Go to Repeating Prompt with Each Character Typed When Using a Screen Reader

[https://github.com/microsoft/vscode/issues/305969](https://github.com/microsoft/vscode/issues/305969)

## Summary

Still reproduces on main. Each keystroke in Go to Line (Ctrl+G) replaces picker.items, which updates the aria-live visibleCount region and re-triggers aria-activedescendant, causing screen readers to re-announce the prompt. No fix has been applied.

## Change summary

When using a screen reader, each character typed in the Go to Line dialog causes the prompt and result to be announced repeatedly instead of just the updated result.

## Best solution

In `src/vs/editor/contrib/quickAccess/browser/gotoLineQuickAccess.ts`, avoid replacing `picker.items` on every keystroke when only the label/ariaLabel text changes — update the existing item in place, or suppress the aria-live announcement during rapid typing. The `visibleCountContainer` (`aria-live: polite`) in `src/vs/platform/quickinput/browser/quickInputController.ts` also re-announces '1 Results' on each item replacement; the quick input layer could debounce or suppress redundant aria-live updates when the count hasn't changed.
