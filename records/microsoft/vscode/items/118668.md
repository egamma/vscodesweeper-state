# Issue #118668 — No way to "deny" a sign in auth request

[https://github.com/microsoft/vscode/issues/118668](https://github.com/microsoft/vscode/issues/118668)

## Summary

Still valid on current main: sign-in requests registered via `requestNewSession` add menu items and a badge to the Accounts icon, but there is no 'Deny' or 'Dismiss' action to remove them without signing in. The gap remains.

## Change summary

Allow users to dismiss/deny an extension's sign-in request in the Accounts menu without being forced to sign in.

## Best solution

Add a 'Deny' or 'Dismiss' action to sign-in request menu items in `authenticationExtensionsService.ts` (around the `requestNewSession` method, lines 427–507), mirroring how `showGetSessionPrompt` already offers Allow/Deny for access requests. Dismissing should dispose the menu item, remove the extension from `_signInRequestItems`, update the badge count, and persist the denial so the request doesn't reappear.
