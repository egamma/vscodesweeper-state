# Issue #25898 — [html] provide translations of tag documentation

[https://github.com/microsoft/vscode/issues/25898](https://github.com/microsoft/vscode/issues/25898)

## Summary

Keep open: this is a long-standing HTML/l10n feature request with 48 community upvotes and a Backlog milestone, so it should not be closed as stale or insufficiently popular. The current discussion frames it as missing translations for HTML language-service tag documentation, not a resolved defect.

## Change summary

HTML tag hover and IntelliSense documentation should be shown in the configured display language, such as Korean, instead of only English.

## Best solution

Keep this in the HTML and localization backlog until maintainers decide whether to localize the HTML language-service data through the VS Code localization pipeline, seed it from historical translations, or consume localized upstream data such as MDN data. The implementation likely belongs around the HTML language service data/hover pipeline rather than the workbench UI.
