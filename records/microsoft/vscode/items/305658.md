# Issue #305658 — Text with small custom font size is rendered below expected position

[https://github.com/microsoft/vscode/issues/305658](https://github.com/microsoft/vscode/issues/305658)

## Summary

Real visual rendering bug when using tokenColorCustomizations with small fontSize multipliers. The related fix PR #315533 is still open, confirming this is unresolved on main. The reporter has clarified the issue in comment 2, so info-needed could be removed.

## Change summary

Text decorated with a small custom font-size multiplier via textMateRules renders below the expected line position because the line div's font-size doesn't account for inline decoration sizes.

## Best solution

The fix requires the line-height/positioning logic in the editor view layer to account for the largest inline font-size decoration on a line. PR https://github.com/microsoft/vscode/pull/315533 is a prerequisite; once merged, the actual positioning fix can land. The info-needed label can likely be removed since the reporter clarified.
