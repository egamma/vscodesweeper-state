# Issue #132085 — Editor hint in new file can't be reached by tabbing

[https://github.com/microsoft/vscode/issues/132085](https://github.com/microsoft/vscode/issues/132085)

## Summary

Still reproduces on main: the empty editor hint links ("Select a language", "don't show") are rendered as anchor elements without tabIndex, so they cannot be reached via keyboard Tab navigation. Multiple community PRs have attempted a fix but none has been merged.

## Change summary

The action links in the empty/untitled editor hint cannot be focused via keyboard Tab, making them inaccessible to keyboard-only and screen reader users.

## Best solution

Add `tabIndex = 0` (and corresponding keydown handlers for Enter/Space) to the anchor elements produced in `EmptyTextEditorHintContentWidget.getHint()` at `src/vs/workbench/contrib/codeEditor/browser/emptyTextEditorHint/emptyTextEditorHint.ts`, or apply the fix in `renderFormattedText` so all action links are keyboard-accessible by default.
