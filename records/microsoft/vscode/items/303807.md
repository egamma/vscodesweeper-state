# Issue #303807 — In a multi-root workspace, all user tasks are injected into request context multiple times (for each workspace folder)

[https://github.com/microsoft/vscode/issues/303807](https://github.com/microsoft/vscode/issues/303807)

## Summary

Confirmed from source: the TasksService uses `getConfiguration('tasks', workspaceFolder)` which returns the merged user+folder config, so user-level tasks appear under every workspace folder in the chat context. No fix has landed on main.

## Change summary

In multi-root workspaces, user-scoped tasks are duplicated in the chat workspace_info context once per workspace folder, wasting context tokens.

## Best solution

In `extensions/copilot/src/platform/tasks/vscode/tasksService.ts`, deduplicate tasks across folders — either by reading user-level tasks separately and emitting them once outside any folder tag, or by comparing task definitions across folders and only including a task under the first folder that defines it. The rendering in `AgentTasksInstructions` (agentPrompt.tsx) may also need adjustment to emit a top-level section for user tasks.
