# Issue #248246 — Hover on top of file goes away when cursor moves to click on + and - buttons

[https://github.com/microsoft/vscode/issues/248246](https://github.com/microsoft/vscode/issues/248246)

## Summary

Team-authored issue (MEMBER); must remain open per policy. The bug describes a real UX problem with expandable hovers near the top of a file where repositioning moves the hover away from the cursor.

## Change summary

Expanding a hover near the top of the file causes the popup to reposition so the cursor is outside it, and moving the cursor back dismisses the hover.

## Best solution

The content hover controller should preserve the hover when it repositions after expansion — either by keeping the hover alive during the transition or by adjusting the mouse-leave detection to account for the reposition. Likely lives in `src/vs/editor/contrib/hover/browser/contentHoverWidget.ts` or the resize/reposition logic in the hover controller.
