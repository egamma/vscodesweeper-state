# Issue #239351 — Renaming paired tags doesn't work if you clear the tag completely

[https://github.com/microsoft/vscode/issues/239351](https://github.com/microsoft/vscode/issues/239351)

## Summary

Still reproduces on main: linked editing session is lost when the tag name is fully deleted because the synced-range logic clears decorations when the reference range collapses or the word pattern rejects an empty value. Two open PRs attempt to fix this.

## Change summary

Linked editing (paired tag rename) breaks if the user fully deletes the tag name before retyping — the synced session is dropped and the paired tag is not updated.

## Best solution

Keep the linked-editing session alive when the reference range becomes empty (zero-length). The fix likely involves guarding the word-pattern check and position-containment logic in `src/vs/editor/contrib/linkedEditing/browser/linkedEditing.ts` to tolerate an empty range. Two community PRs are already open addressing this.
