# Issue #259754 — Multi-Root Workspaces: glob and file search not working

[https://github.com/microsoft/vscode/issues/259754](https://github.com/microsoft/vscode/issues/259754)

## Summary

The core problem — search tools prepending `**/` and breaking multi-root workspace folder scoping — was fixed by commit 5478618cc66, which added `inputGlobToPattern` in toolUtils.ts to detect folder-name-prefixed patterns (including stripping a leading `**/`) and rewrite them into `RelativePattern` scoped to the correct workspace folder. Shipped in 1.116.0.

## Change summary

Copilot's file and text search tools unconditionally prepend `**/` to glob patterns, preventing multi-root workspace searches from targeting a specific workspace root.

## Best solution

Keep the shipped fix from commit 5478618cc66 (1.116.0). The `inputGlobToPattern` function now correctly resolves multi-root workspace folder names from glob patterns. No further change is needed beyond confirming on the latest build.
