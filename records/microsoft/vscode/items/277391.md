# Issue #277391 — Delay before starting npm script in debug terminal

[https://github.com/microsoft/vscode/issues/277391](https://github.com/microsoft/vscode/issues/277391)

## Summary

The 5-7 second delay is caused by the MaxDelay=5000ms fallback in the shell initialization wait logic (introduced in commit 8653e209cc0 to fix #228191). When shell integration doesn't activate quickly, the code falls through to this timeout. The issue is real and still present on main.

## Change summary

First launch of a debug terminal for npm scripts incurs a 5-7 second delay before the script executes, because the shell-readiness detection falls back to a 5-second timeout.

## Best solution

Investigate why shell integration isn't activating promptly on macOS for debug terminals (the `forceShellIntegration: true` option is set). A faster deterministic readiness signal would eliminate the fallback timeout. The fix likely lives in `src/vs/workbench/api/node/extHostDebugService.ts` around the `giveShellTimeToInitialize` block (lines 125-151).
