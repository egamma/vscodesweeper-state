# Issue #326171 — [Error] unhandlederror-Cannot read properties of undefined (reading '0')

[https://github.com/microsoft/vscode/issues/326171](https://github.com/microsoft/vscode/issues/326171)

## Summary

Real IPC crash from empty MessagePort frames reaching the protocol reader unguarded — the defect path is confirmed in current source. An open bot-authored PR (#326172) already targets the fix; keeping the issue open until that PR is reviewed and landed by a human.

## Change summary

Zero-length MessagePort frames forwarded as VSBuffer.alloc(0) cause `deserialize` to return undefined, crashing `header[0]` in both ChannelClient.onBuffer and ChannelServer.onRawMessage.

## Best solution

Review and land (or rework) the open PR https://github.com/microsoft/vscode/pull/326172, which drops empty frames at the transport boundary in `ipc.mp.ts` before they reach the channel readers. The fix owner should be the MessagePort transport maintainer (deepak1556 is suggested by the PR).
