# Issue #320989 — [Feature]Stable installation path required: commit ID breaks third-party integrations

[https://github.com/microsoft/vscode/issues/320989](https://github.com/microsoft/vscode/issues/320989)

## Summary

Legitimate feature request for a stable installation path on Windows. The commit-hash-based `VersionedResourcesFolder` in the Inno Setup installer is intentional (enables background updates), but the request to provide a stable reference path for third-party tools is reasonable. Already correctly labeled `install-update` and `under-discussion`.

## Change summary

The Windows installer places resources under a commit-hash directory (e.g. `1b50d58d73/resources/app/out/cli.js`) that changes every update, breaking third-party tools like SumatraPDF that reference those paths.

## Best solution

Provide a stable symlink, junction, or well-known registry key pointing to the current versioned directory so third-party tools can reference a fixed path. The change would live in `build/win32/code.iss` and the Inno Setup update scripts. Alternatively, document a stable entry point (e.g. `code.cmd` on PATH) that third-party tools should use instead of hardcoded internal paths.
