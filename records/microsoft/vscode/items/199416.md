# Issue #199416 — Nope emoji in quotes breaks cursor position 

[https://github.com/microsoft/vscode/issues/199416](https://github.com/microsoft/vscode/issues/199416)

## Summary

The ZWJ emoji cursor positioning bug likely still exists on main — VS Code's simplified GB11 grapheme break implementation (noted in the code) combined with variable emoji visual widths can cause cursor/column misalignment. No targeted fix has landed since the report.

## Change summary

Cursor position becomes incorrect when typing around the 🙅‍♂️ (ZWJ emoji sequence) inside quotes.

## Best solution

The root cause is likely in how the editor maps model columns to visual positions for multi-code-point emoji. The grapheme segmentation in `src/vs/base/common/strings.ts` (simplified GB11 at line ~1061) and the character width measurement in the view line rendering need to correctly handle ZWJ emoji sequences. A fix would ensure the cursor model treats the full ZWJ sequence as a single visual unit with its measured width.
