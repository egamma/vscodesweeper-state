# Issue #322670 — Git inline diff decorations persist even after "scm.diffDecorations" set to "none"

[https://github.com/microsoft/vscode/issues/322670](https://github.com/microsoft/vscode/issues/322670)

## Summary

The inline decorations described (strikethrough deleted lines, arrow-prefixed new lines) are not produced by VS Code's built-in diff decorations — those are gutter/minimap/overview-ruler indicators. The described visual style matches GitLens's inline change annotations, which the reporter has installed.

## Change summary

Setting `scm.diffDecorations` to `none` does not remove inline strikethrough/arrow diff decorations from the editor.

## Best solution

Close as caused-by-extension. The reporter should disable GitLens's inline change decorations via `gitlens.changes.locations` or GitLens's own settings UI, or file an issue on the GitLens repository.
