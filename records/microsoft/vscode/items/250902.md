# Issue #250902 — Debug Toolbar missing project in command center

[https://github.com/microsoft/vscode/issues/250902](https://github.com/microsoft/vscode/issues/250902)

## Summary

Valid feature request: the session-focus dropdown is intentionally excluded from the command center toolbar (line 456 of debugToolBar.ts gates it with `CONTEXT_TOOLBAR_COMMAND_CENTER.negate()`). The command center mode is still marked experimental, so this is a known gap rather than a bug.

## Change summary

When `debug.toolBarLocation` is set to `commandCenter`, there is no way to switch between debug sessions (multi-project) like the floating toolbar provides.

## Best solution

Add a session picker (or equivalent quick-pick action) to the command center debug toolbar submenu so users can switch focused debug sessions without reverting to the floating toolbar. The fix would likely involve removing or adjusting the `CONTEXT_TOOLBAR_COMMAND_CENTER.negate()` guard on line 456 of `src/vs/workbench/contrib/debug/browser/debugToolBar.ts`, or contributing a separate command-center-appropriate session selector.
