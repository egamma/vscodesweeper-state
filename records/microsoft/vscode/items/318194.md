# Issue #318194 — TAB bricked by copilot inline autocomplete

[https://github.com/microsoft/vscode/issues/318194](https://github.com/microsoft/vscode/issues/318194)

## Summary

Tab accepting inline suggestions instead of indenting is intentional VS Code behavior. The keybinding is guarded by context keys (suggestion must be visible, tabMovesFocus not set, etc.) and users can dismiss suggestions with Escape first, or toggle `editor.tabCompletion` / rebind the accept action.

## Change summary

User wants Tab to indent/move focus rather than accept inline completions when a suggestion is showing.

## Best solution

Close as designed. The Tab→accept binding is intentional and well-guarded. Users who prefer Tab for indentation can press Escape to dismiss the suggestion first, disable `editor.inlineSuggest.enabled`, or rebind `editor.action.inlineSuggest.commit` to a different key.
