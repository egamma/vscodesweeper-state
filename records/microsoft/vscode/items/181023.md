# Issue #181023 — some snippet show suggestion preview, but some not

[https://github.com/microsoft/vscode/issues/181023](https://github.com/microsoft/vscode/issues/181023)

## Summary

Still present on main: computeGhostText returns undefined (no preview) when the diff between the typed prefix and the snippet body involves character replacements rather than pure insertions, which happens when the snippet trigger doesn't match the start of the snippet body text.

## Change summary

Some snippets show inline suggest preview ghost text while others with the same single-line structure do not, depending on whether the typed prefix matches the start of the completion text.

## Best solution

The ghost text computation in `computeGhostText.ts` bails out (line 96-98) when a diff change has `originalLength > 0`. For suggest preview specifically, the mode could be relaxed to allow showing the non-replaced portion as ghost text, or a fallback to show the full replacement text. The fix would be localized to `src/vs/editor/contrib/inlineCompletions/browser/model/computeGhostText.ts`.
