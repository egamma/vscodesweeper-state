# Issue #224704 — [Accessibility] Reading is no longer possible when hovering the mouse over items in the tab and explorer areas of the editor.

[https://github.com/microsoft/vscode/issues/224704](https://github.com/microsoft/vscode/issues/224704)

## Summary

Verified duplicate of #277895, which tracks the same NVDA mouse-tracking regression caused by upstream Chromium accessibility changes. The newer issue has deeper investigation, upstream links, and is already labeled upstream with a Backlog milestone — consolidating there.

## Change summary

NVDA mouse tracking stopped reading text under cursor in VS Code after Chromium updated its accessibility hit-test behavior, breaking the legacy IAccessible path NVDA uses by default.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/277895 which already tracks the upstream Chromium/NVDA interop issue with detailed investigation. The fix path is in NVDA updating its UIA opt-in heuristics for newer Chromium/Electron builds (tracked at https://github.com/nvaccess/nvda/issues/17108). VS Code has no actionable code change — the workaround is forcing UIA in NVDA advanced settings.
