# Issue #174488 — Annoying warnings in launch.json for configs that don't apply to current platform. 

[https://github.com/microsoft/vscode/issues/174488](https://github.com/microsoft/vscode/issues/174488)

## Summary

Still reproduces on current main: disabled-platform debuggers surface a deprecationMessage warning in the Problems panel for launch.json configs that are valid on another OS. A team member acknowledged the issue and suggested reducing the severity. Keeping open as a valid bug in the debug area.

## Change summary

launch.json shows Problems-panel warnings for debug configurations whose type is installed but not supported on the current platform, even though the config is valid on another OS.

## Best solution

Change the disabled-debugger indicator in `src/vs/workbench/contrib/debug/common/debugger.ts` (line 264) from `deprecationMessage` (which surfaces as a warning) to a lighter mechanism — for example, a hint-level diagnostic or an informational decoration — so cross-platform launch.json files don't produce noisy Problems-panel entries. Rob Lourens's comment 4 suggested something 'similar to unrecognized settings' as the model.
