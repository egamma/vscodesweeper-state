# Issue #55253 — Problem matchers should support creating related diagnostic information

[https://github.com/microsoft/vscode/issues/55253](https://github.com/microsoft/vscode/issues/55253)

## Summary

Team-authored feature request on the 'On Deck' milestone — both are hard keep-open triggers. The capability (related diagnostic information from problem matchers) has not been implemented on main; no code in the tasks contribution references relatedInformation.

## Change summary

Allow problem matchers to produce DiagnosticRelatedInformation so compilers that emit multi-line diagnostic chains (e.g. 'included from', 'defined here') can surface linked context in the Problems panel.

## Best solution

Extend the problem matcher schema and runtime in `src/vs/workbench/contrib/tasks/` to support a child/related pattern that populates `DiagnosticRelatedInformation` on the parent diagnostic. The programmatic task API (see referenced #9635) would also solve this more flexibly.
