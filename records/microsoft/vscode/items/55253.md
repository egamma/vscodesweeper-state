# Issue #55253 — Problem matchers should support creating related diagnostic information

[https://github.com/microsoft/vscode/issues/55253](https://github.com/microsoft/vscode/issues/55253)

## Summary

Team-authored feature request (MEMBER). The marker model already supports `relatedInformation` (IRelatedInformation in markers.ts), but problem matchers have no mechanism to populate it — the feature gap remains real. Keeping open per hard keep-open rule and because it is still a valid unimplemented request.

## Change summary

Allow problem matchers to produce related diagnostic information (e.g. "defined here", "included from") when parsing compiler output, populating the existing IRelatedInformation on IMarkerData.

## Best solution

Extend the problem matcher schema to support child/related patterns that map to IRelatedInformation entries on the resulting diagnostic. The infrastructure already exists in the marker model (IMarkerData.relatedInformation); the gap is in the problem matcher parsing and contribution-point schema. A programmatic problem-matching API (referenced in issue #9635) would also solve this more flexibly.
