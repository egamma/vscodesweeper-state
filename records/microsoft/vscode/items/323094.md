# Issue #323094 — AGENTS.md discovery still runs `rg --no-ignore --follow` over the whole tree (regression sibling of #304676)

[https://github.com/microsoft/vscode/issues/323094](https://github.com/microsoft/vscode/issues/323094)

## Summary

The report describes a real performance hazard in the optional nested AGENTS.md discovery path, and a maintainer confirmed it only happens behind `chat.useNestedAgentsMdFiles`, which is disabled by default because of this cost. Keep open for chat/agent owners to decide whether the non-default nested scan should be made safer rather than closing as designed.

## Change summary

Nested AGENTS.md discovery can repeatedly run ripgrep with `--no-ignore --follow` over large pnpm workspaces, traversing node_modules symlink forests and consuming high CPU.

## Best solution

Either keep `chat.useNestedAgentsMdFiles` explicitly off by default and document the tradeoff, or harden the nested AGENTS.md discovery so it avoids following ignored symlink-heavy directories and scopes/debounces the search in the chat agent instruction-file discovery path.
