# Issue #323094 — AGENTS.md discovery still runs `rg --no-ignore --follow` over the whole tree (regression sibling of #304676)

[https://github.com/microsoft/vscode/issues/323094](https://github.com/microsoft/vscode/issues/323094)

## Summary

The reported behavior only triggers with the non-default opt-in setting `chat.useNestedAgentsMdFiles`. A maintainer already noted this, suggesting it's somewhat expected. However, the user's point that `--no-ignore --follow` is overly aggressive even for an opt-in scan may still be a valid improvement request. Keeping open for maintainer judgment on whether the scan flags should be tightened.

## Change summary

When `chat.useNestedAgentsMdFiles` is enabled, the AGENTS.md discovery scan uses `--no-ignore --follow` across the entire workspace, causing severe CPU usage in large pnpm monorepos with symlink forests.

## Best solution

If the nested scan is kept as an opt-in feature, it should still respect `.gitignore` (drop `--no-ignore`) and avoid following symlinks (drop `--follow`) to prevent pathological behavior in pnpm workspaces. The core `promptFilesLocator.ts` already uses the search service with exclude patterns — the discrepancy may be in the ripgrep file-search provider's flags when invoked for this particular query. Area: `src/vs/workbench/services/search/` or the copilot extension's locator.
