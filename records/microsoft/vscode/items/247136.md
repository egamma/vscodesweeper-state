# Issue #247136 — UX: Copilot pickers need alignment

[https://github.com/microsoft/vscode/issues/247136](https://github.com/microsoft/vscode/issues/247136)

## Summary

The model picker has been intentionally evolved into a feature-rich custom widget (ActionWidgetDropdown) with grouped models, per-model configuration, cache hints, and restricted-mode handling — capabilities that exceed native menus. All chat input pickers now share a common ChatInputPickerActionViewItem base class, achieving alignment through consistent custom widgets rather than native menus.

## Change summary

The chat view's model picker used a custom dropdown while the other pickers used native controls, creating an inconsistent look.

## Best solution

The pickers are now aligned on a shared custom widget base class (ChatInputPickerActionViewItem → ActionWidgetDropdownActionViewItem). The feature-rich model picker requirements (grouped models, configuration, tooltips, restricted-mode UI) justify the custom approach. No further change needed.
