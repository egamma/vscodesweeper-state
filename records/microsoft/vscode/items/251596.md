# Issue #251596 — Using `${activeRepositoryName}` inside the `window: title` while inside a worktree displays the current worktree branch

[https://github.com/microsoft/vscode/issues/251596](https://github.com/microsoft/vscode/issues/251596)

## Summary

Still reproducible on main: `${activeRepositoryName}` resolves via `basename(_rootUri)` in `mainThreadSCM.ts`, which for a worktree returns the worktree directory name (often the branch name) rather than the main repository name. An open PR (#267692) addresses this, and the maintainer has explicitly kept this under discussion for further feedback.

## Change summary

When a git worktree is opened, the `${activeRepositoryName}` window title variable shows the worktree directory name instead of the actual repository name.

## Best solution

Review and land https://github.com/microsoft/vscode/pull/267692, which improves worktree hierarchy detection. The fix likely needs to resolve the parent repository name when `repository.kind === 'worktree'` — the SCM provider name logic in `src/vs/workbench/api/browser/mainThreadSCM.ts` (lines 363–370) should detect the worktree case and use the parent repository's root basename instead of the worktree's own `_rootUri`.
