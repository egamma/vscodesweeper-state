# Issue #275152 ‚Äî During composition everything is filtered

[https://github.com/microsoft/vscode/issues/275152](https://github.com/microsoft/vscode/issues/275152)

## Summary

Still unfixed on main: the tree find widget filters during IME composition because `InputBox.onValueChange` fires on every input event without checking composition state. An open PR (#275866) addresses this but has not merged.

## Change summary

During IME composition in the tree find widget, intermediate keystrokes trigger filtering prematurely, hiding all results until composition completes.

## Best solution

Land the open PR https://github.com/microsoft/vscode/pull/275866 (or an equivalent fix) that suppresses filtering while `imeSessionInProgress` is true in the tree's find controller. The `FindInput` already tracks composition state via `compositionstart`/`compositionend` ‚Äî the tree just needs to gate `onDidChangeValue` on it.
