# Issue #275152 — During composition everything is filtered

[https://github.com/microsoft/vscode/issues/275152](https://github.com/microsoft/vscode/issues/275152)

## Summary

Still reproduces on main: the tree find widget fires filter events during IME composition because it uses `onDidChange` instead of `onInput` (which respects composition boundaries). An open PR exists but has not merged.

## Change summary

During IME composition in the tree find widget, intermediate characters trigger filtering and hide all items prematurely.

## Best solution

Land a fix in `src/vs/base/browser/ui/tree/abstractTree.ts` (line ~927) to gate filtering on composition state — switch from `findInput.onDidChange` to `findInput.onInput` or guard with the IME session flag. The open PR https://github.com/microsoft/vscode/pull/275866 attempts this but needs human review before merging.
