# Issue #323226 — High idle CPU: PR-icon model thrash in Chat Sessions list (create/dispose churn)

[https://github.com/microsoft/vscode/issues/323226](https://github.com/microsoft/vscode/issues/323226)

## Summary

Team-authored performance investigation with detailed root-cause analysis of idle CPU waste from PR-model create/dispose churn driven by ChatSessionsService over-resolution. Keeping open for maintainer action.

## Change summary

Idle high CPU caused by ChatSessionsService resolving 162 sessions every few seconds, triggering GitHub PR extension to create+dispose PR models in a tight loop (476 cycles in 4 min for 7 unique PRs).

## Best solution

Two fixes needed: (1) GitHubPullRequestModelReferenceCollection/PollingContribution should hold stable references for tracked sessions rather than acquire-and-release per refresh pass; (2) ChatSessionsService resolution should be throttled/deduped to avoid re-resolving all 162 sessions every few seconds. The PR-icon thrash is likely in the GitHub Pull Requests extension (microsoft/vscode-pull-request-github), while the ChatSessionsService throttling belongs in src/vs/workbench/contrib/chat or the sessions layer.
