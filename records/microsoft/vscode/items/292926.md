# Issue #292926 — Auto model visibility

[https://github.com/microsoft/vscode/issues/292926](https://github.com/microsoft/vscode/issues/292926)

## Summary

Partially addressed by recent work (auto-mode resolution content part showing "Routed to <model>" in responses, and auto hover in the model picker), but the picker label itself still reads "Auto" without showing the resolved model upfront. The core ask — immediate visibility of which model Auto selected — retains a real gap.

## Change summary

When "Auto" is selected as the chat model, the resolved model should be immediately visible rather than only discoverable after the response.

## Best solution

Update the model picker label or add a subtitle/badge so that when Auto resolves to a specific model, the picker shows e.g. "Auto → Claude Sonnet" before or as the response streams, in addition to the existing collapsible "Routed to" content part. Likely touches `chatModelPicker.ts` (`_renderLabel`) and possibly `chatInputPart.ts`.
