# Issue #155794 — Adopt argsCanBeInterpretedByShell for external terminals

[https://github.com/microsoft/vscode/issues/155794](https://github.com/microsoft/vscode/issues/155794)

## Summary

Team-authored debt issue — the gap is still present on main. The external terminal code path (`IExternalTerminalService.runInTerminal`) does not accept or propagate the `argsCanBeInterpretedByShell` flag from the DAP `RunInTerminalRequest`, so escaping behavior for external terminals is incomplete compared to the integrated terminal path.

## Change summary

Adopt the `argsCanBeInterpretedByShell` DAP flag in the external terminal code path so argument escaping is consistent with integrated terminals.

## Best solution

Thread the `argsCanBeInterpretedByShell` parameter through `IExternalTerminalService.runInTerminal` (in `src/vs/platform/externalTerminal/common/externalTerminal.ts`) and its three platform implementations (`node/externalTerminalService.ts`), and update the call site in `extHostDebugService.ts` line 231 to pass `args.argsCanBeInterpretedByShell`. The Linux/Windows implementations can leverage `prepareCommand` from `debug/node/terminals.ts` which already handles the flag; the macOS AppleScript path may need separate consideration.
