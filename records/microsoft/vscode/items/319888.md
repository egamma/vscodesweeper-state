# Issue #319888 — vscode snap fails to start podman due to database static dir mismatch

[https://github.com/microsoft/vscode/issues/319888](https://github.com/microsoft/vscode/issues/319888)

## Summary

This is a self-answered workaround note for a snap/podman path mismatch, not an actionable VS Code issue. The author explicitly states they are sharing a snippet for future reference. The root cause is snap revision-based paths breaking podman's database config, which is outside VS Code's code.

## Change summary

Podman fails to start from the VS Code snap because snap revision numbers change the storage path between updates, causing a database static dir mismatch.

## Best solution

Close as a support question. The underlying issue is a snap confinement / podman interaction, not a VS Code code defect. The workaround (pinning `graphroot` in `~/.config/containers/storage.conf` to a stable `$HOME`-relative path) is the correct user-side fix. If there is a real snap packaging concern, it belongs on the snap packaging repo or with the containers/podman project, not in VS Code's core issue tracker.
