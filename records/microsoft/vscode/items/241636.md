## Summary

Real packaging bug — the snap's core20 base bundles a glibc too old for system-installed Java 21, causing symbol lookup crashes. An open PR (#298237) upgrades to core22 and explicitly fixes this issue; keep open until it merges.

## Change summary

Java 21 applications crash with a __libc_pthread_init symbol lookup error when launched from VS Code's integrated terminal in the snap package because the snap bundles core20's ancient glibc.
