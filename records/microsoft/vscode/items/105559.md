# Issue #105559 — Windows Magnifier not following active menuitem

[https://github.com/microsoft/vscode/issues/105559](https://github.com/microsoft/vscode/issues/105559)

## Summary

Still an open issue with no fix on main. Likely an Electron/Chromium bug (doesn't reproduce on web). The sibling issue #105558 (caret tracking) was fixed, but this menu-specific magnifier tracking remains unresolved. Keeping open as upstream.

## Change summary

Windows Magnifier does not follow the focused menu item when navigating custom menus with keyboard arrows.

## Best solution

Investigate whether newer Electron versions (currently on Electron 34+) resolve the issue, or whether an explicit accessibility workaround in the menu widget (e.g., forcing a system caret position update on menu item focus) is needed. The root cause is likely in Chromium's accessibility tree not properly reporting focused items in shadow-DOM-like custom menu containers to the Windows UIA Magnifier API.
