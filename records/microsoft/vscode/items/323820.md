# Issue #323820 — Ability to limit search to within the current function scope

[https://github.com/microsoft/vscode/issues/323820](https://github.com/microsoft/vscode/issues/323820)

## Summary

Valid feature request for automatically scoping editor find/replace to the current function/symbol without manually selecting it first. No duplicate found; the existing 'Find in Selection' requires manual selection, which is the exact UX gap this requests closing.

## Change summary

Add an option to scope in-editor find/replace results to the enclosing function or symbol scope automatically.

## Best solution

This could be implemented as a new toggle in the find widget that uses document symbols (from the language service) to constrain matches to the enclosing function/method at cursor. The feature lives at the intersection of editor-find and language intelligence. Alternatively, it's an extension-candidate — an extension could select the enclosing symbol range and enable 'Find in Selection' programmatically. Worth keeping open for community traction.
