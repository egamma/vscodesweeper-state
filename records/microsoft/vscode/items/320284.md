# Issue #320284 — "Open File" dialog does not open to current file directory

[https://github.com/microsoft/vscode/issues/320284](https://github.com/microsoft/vscode/issues/320284)

## Summary

Recurring upstream Electron/Chromium regression on Linux: the native file dialog ignores the `defaultPath` parameter after the Electron v42 upgrade in 1.123.0. VS Code correctly computes and passes the path — the bug is in Electron's GTK/portal dialog integration. Should be tracked and reported upstream.

## Change summary

On Linux, Ctrl+O opens the native file dialog to the 'Recent' tab instead of the directory of the currently open file.

## Best solution

File an upstream Electron issue for the `defaultPath` regression in Electron 42's Linux file dialog (GTK/xdg-desktop-portal). VS Code's code in `src/vs/workbench/services/dialogs/electron-browser/fileDialogService.ts` correctly passes `defaultPath` — the fix must come from the Electron/Chromium layer. Add the `upstream` and `electron` and `native-file-dialog` labels so the team can track it alongside the established pattern from https://github.com/microsoft/vscode/issues/213780.

Each time it was an Electron/Chromium upgrade breaking Linux native file dialog defaultPath.
