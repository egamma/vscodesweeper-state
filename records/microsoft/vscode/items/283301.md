# Issue #283301 — Cannot read properties of undefined (reading 'toString')

[https://github.com/microsoft/vscode/issues/283301](https://github.com/microsoft/vscode/issues/283301)

## Summary

Still reproduces on current main: `borderColor.toString()` at line 104 of inlineEditsLongDistanceHint.ts has no null guard, so if `getEditorBlendedColor(...).read(reader)` returns undefined (e.g. during disposal or theme initialization race), a TypeError is thrown.

## Change summary

The inline edits long-distance hint throws 'Cannot read properties of undefined (reading toString)' when `borderColor` is undefined during the derived style computation.

## Best solution

Add a null guard before calling `.toString()` on `borderColor` in the `_styles` derived (line 104 of `inlineEditsLongDistanceHint.ts`), returning a fallback color string when the value is undefined — likely during teardown or a theme initialization race.
