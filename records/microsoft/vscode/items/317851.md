# Issue #317851 — TitleBarAccountWidget leak

[https://github.com/microsoft/vscode/issues/317851](https://github.com/microsoft/vscode/issues/317851)

## Summary

Fixed on main by commit 841d3b6bbe4, which adds an `isDisposed` guard in `refreshAvatar` to prevent the leaked disposable when the widget is disposed during an async account info resolution. Shipped in 1.123.0.

## Change summary

TitleBarAccountWidget.refreshAvatar leaks a disposable when the widget is disposed while an async account-info request is in flight.

## Best solution

Keep the shipped fix (841d3b6bbe4); no further change needed. The early-return guard prevents the `toDisposable` from being created after the store is already disposed.
