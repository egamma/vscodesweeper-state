# Issue #324548 — [Error] unhandlederror-cannot add a decoration for a command {} with no marker

[https://github.com/microsoft/vscode/issues/324548](https://github.com/microsoft/vscode/issues/324548)

## Summary

Confirmed regression in 1.127.0: `_flushPendingHandleCommandStartTask` fires `onCommandStartedEmitter` with a potentially undefined `commandStartMarker`, causing the decoration addon to throw. An open PR (#324557) addresses this but hasn't merged yet.

## Change summary

Terminal throws an unhandled error when a command-start decoration is registered but the command has no marker, affecting 1,436 users on 1.127.0 stable.

## Best solution

Land the open PR https://github.com/microsoft/vscode/pull/324557 (or an equivalent guard) — either guard the `onCommandStartedEmitter.fire` call in `commandDetectionCapability.ts` line 805 to skip when `commandStartMarker` is undefined, or guard `decorationAddon.ts` line 231's listener to bail out when the command has no marker instead of throwing.
