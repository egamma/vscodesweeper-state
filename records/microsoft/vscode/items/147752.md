# Issue #147752 — Line Endings Selector showing wrong Line Endings in Diff View

[https://github.com/microsoft/vscode/issues/147752](https://github.com/microsoft/vscode/issues/147752)

## Summary

Still reproduces on main: the EOL status bar indicator doesn't correctly reflect the line endings of the original (left) side in a diff view because `onEOLChange` skips read-only editors and no focus-change listener updates the status when switching between diff panes.

## Change summary

The status bar EOL indicator shows the wrong (or stale) line endings when focusing the original side of a diff editor in the Source Control view.

## Best solution

Fix the EOL status logic in `editorStatus.ts` to either (a) show EOL for read-only editors in diff views, or (b) listen for focus changes within the diff editor and re-query the focused pane's model. The readOnly guard at line 877 intentionally hides the EOL picker for read-only files, but in a diff context the display should still be informational.
