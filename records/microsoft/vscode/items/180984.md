# Issue #180984 — Reopening a file (notebook) should open it as a notebook instead of as a text file

[https://github.com/microsoft/vscode/issues/180984](https://github.com/microsoft/vscode/issues/180984)

## Summary

Team-authored issue (hard keep-open). The underlying UX gap remains: quick open history and recently-closed-editors store the old `editorId`, so reopening a file after installing a new editor extension (e.g. a notebook provider) silently uses the stale editor type. No fix has landed on main.

## Change summary

Reopening a file from quick open history after installing a notebook extension opens it as plain text instead of as a notebook, because the history persists the old editor ID.

## Best solution

When the set of registered editors for a resource changes (extension install/uninstall), invalidate or ignore the stored `editorId` in `historyService.ts` recently-closed and quick-open history entries so the resolver picks the best available editor. The discussion in comment 10 sketches the approach: only persist an explicit editor ID when the user actively chose 'Reopen With…', not when the editor was opened normally.
