# Issue #285278 — Missing cursor after activating it using Alt + Click

[https://github.com/microsoft/vscode/issues/285278](https://github.com/microsoft/vscode/issues/285278)

## Summary

This matches the editor's configured default behavior: overlapping multiple cursors are merged, including touching collapsed cursors after navigation. The issue is asking whether that behavior is expected, and the code plus maintainer comment support closing as designed.

## Change summary

Multiple cursors created with Alt+Click can collapse into one cursor when vertical navigation brings them to the same position.

## Best solution

Keep the default `editor.multiCursorMergeOverlapping: true` behavior; users who want to preserve overlapping cursors can disable that setting.

- **Proposed comment:**

  > Thanks for the report. This is the current intended editor behavior: with `editor.multiCursorMergeOverlapping` enabled, VS Code merges overlapping multiple cursors, and collapsed cursors are merged once navigation brings them to the same position. You can set `editor.multiCursorMergeOverlapping` to `false` if you want to preserve overlapping cursors.

## Evidence

- **current implementation:** In `src/vs/editor/common/cursor/cursorCollection.ts`, `CursorCollection.normalize()` merges cursors when `multiCursorMergeOverlapping` is enabled; for collapsed selections it explicitly merges touching cursors with `isBeforeOrEqual`.
- **default setting:** `src/vs/editor/common/config/editorOptions.ts` registers `editor.multiCursorMergeOverlapping` with default `true` and description "Merge multiple cursors when they are overlapping."
- **maintainer context:** A maintainer commented on 2026-01-05 that this looks intended because there is not always room for multiple cursors on the same line.
