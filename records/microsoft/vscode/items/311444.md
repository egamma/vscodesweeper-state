# Issue #311444 — [Error] a.getId is not a function — explorerView / asyncDataTree

[https://github.com/microsoft/vscode/issues/311444](https://github.com/microsoft/vscode/issues/311444)

## Summary

Still present on main: the `getDataNode` error-reporting path at asyncDataTree.ts:1077 unsafely casts `element as T` and calls `identityProvider.getId()` on it, but when the element is the root input (e.g. an `ExplorerItem[]` array from a race between `_updateChildren` and `setInput`), `getId()` doesn't exist. The race condition is the root cause; the crashing error path is the proximate bug.

## Change summary

The async data tree's `getDataNode` error-reporting path crashes with `a.getId is not a function` when called with a root input element that lacks an `getId` method, triggered by a race between `_updateChildren` and `setInput`.

## Best solution

Two-part fix: (1) Guard the error-reporting path at asyncDataTree.ts:1077 so `getId` is only called when `element` is actually a `T` (not `TInput`), e.g. wrap in try/catch or check `typeof element.getId === 'function'`. (2) Address the underlying race in `_updateChildren` by checking whether the root has changed after awaiting `root.refreshPromise` and bailing out if so.
