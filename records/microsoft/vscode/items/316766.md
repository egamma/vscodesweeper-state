## Summary

Valid feature request for a cancel button on ongoing git push operations in the Agents window. No implementation found on current main; the issue is correctly labeled and in Backlog.

## Change summary

Allow users to cancel an in-progress git commit & sync (particularly the push stage) in the Agents window.

## Best solution

Add a cancel/abort affordance for the git push phase of Commit & Sync in the Agents window SCM integration. The push is the network-bound bottleneck and is safe to abort without corrupting the repository. A commenter has already identified the safe boundary (aborting push, not commit).
