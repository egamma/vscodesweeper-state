# Issue #254835 — When using macOS with the VoiceOver screen reader, moving word-by-word across a sign character causes the word to be spoken twice

[https://github.com/microsoft/vscode/issues/254835](https://github.com/microsoft/vscode/issues/254835)

## Summary

Confirmed and properly triaged by the team. The root cause is identified (WordRight stops on delimiters like `:` causing VoiceOver to re-read the preceding word), and it's accepted as a feature request in the Backlog. No fix has landed on main.

## Change summary

Moving word-by-word with VoiceOver causes words before sign characters (`:`, `=`) to be spoken twice because VS Code's WordRight command stops on delimiters differently than other editors.

## Best solution

Modify WordRightCommand logic in `src/vs/editor/common/cursor/cursorWordOperations.ts` to skip delimiter-only stops (like `:` and `=`) so VoiceOver doesn't re-announce the preceding word. The team analysis by aiday-mar already identified this path.
