# Issue #254835 — When using macOS with the VoiceOver screen reader, moving word-by-word across a sign character causes the word to be spoken twice

[https://github.com/microsoft/vscode/issues/254835](https://github.com/microsoft/vscode/issues/254835)

## Summary

Confirmed and already triaged by team members. The WordRightCommand stops on delimiters like `:` and `=`, causing VoiceOver to re-read the preceding word. Properly labeled as a feature-request in Backlog; no action needed.

## Change summary

Moving word-by-word with Option+Right Arrow causes VoiceOver to announce a word twice when the cursor stops on a delimiter character after the word.

## Best solution

Modify the WordRightCommand logic to skip delimiter-only stops (`:`, `=`) when in accessibility/screen-reader mode, aligning cursor-stop behavior with Microsoft Word's approach. The fix would likely live in the word operations or cursor controller in `src/vs/editor/common/cursor/` or `src/vs/editor/contrib/wordOperations/`.
