## Summary

Still unimplemented on main — CSS/LESS/SCSS format-on-type has no registration in the extension. Low traction but a legitimate backlog feature request.

## Change summary

Request to add format-on-type support (e.g. on `}` and `;`) to the CSS/LESS/SCSS language features extension.

## Best solution

Implement a `DocumentOnTypeFormattingProvider` in `extensions/css-language-features/server/src` that leverages vscode-css-languageservice's range-formatting to format the enclosing block on `}` and the statement on `;`. The upstream vscode-css-languageservice already supports range formatting, so the server just needs to wire it to onTypeFormatting triggers.
