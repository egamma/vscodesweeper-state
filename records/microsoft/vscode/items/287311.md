# Issue #287311 — chat request fails with `Unexpected token '<', "<!DOCTYPE "... is not valid JSON`

[https://github.com/microsoft/vscode/issues/287311](https://github.com/microsoft/vscode/issues/287311)

## Summary

Transient service outage caused HTML error pages to be returned instead of JSON, resulting in an unhandled parse error. A team member confirmed the root cause and flagged that the error should be rendered in a user-friendly way — this remains an open UX improvement.

## Change summary

Chat requests fail with a raw JSON parse error when the Copilot backend returns an HTML error page during a service outage instead of a JSON response.

## Best solution

Improve error handling in the Copilot extension's networking/fetch layer (likely `extensions/copilot/src/platform/networking/` or `extensions/copilot/src/platform/endpoint/`) to detect non-JSON responses (e.g. HTML error pages) and surface a user-friendly message like 'The service is temporarily unavailable' instead of exposing the raw SyntaxError.
