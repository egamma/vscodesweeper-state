# Issue #234402 — Allow selecting extensions when copying a profile

[https://github.com/microsoft/vscode/issues/234402](https://github.com/microsoft/vscode/issues/234402)

## Summary

Valid feature request with 29 upvotes, promoted to Backlog. The profile copy UI currently only toggles Extensions as a whole (all or none) — individual extension selection during copy is not implemented on current main.

## Change summary

Allow selecting individual extensions (not just all-or-nothing) when copying a profile.

## Best solution

Extend the profile contents tree in `userDataProfilesEditorModel.ts` to include Extensions in the expandable-children list (line 263, alongside Settings/Keybindings/Tasks/Mcp), so users see checkboxes for each extension when copying. The `ExtensionsResourceExportTreeItem` and `getChildrenFromProfile` already support enumerating extensions — the gap is purely in the `getChildren` method that excludes Extensions from the expanded set.
