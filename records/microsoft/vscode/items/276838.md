# Issue #276838 — Copilot writes to wrong location at C:\f\

[https://github.com/microsoft/vscode/issues/276838](https://github.com/microsoft/vscode/issues/276838)

## Summary

Fixed on main: the Copilot extension's path resolver now rectifies unix-style paths on Windows that lack a drive letter, preventing the model from writing to the wrong location (e.g. C:\f\ instead of F:\).

## Change summary

Copilot agent writes files to wrong location (C:\f\...) instead of the workspace drive (F:\...) because the model outputs unix-style paths without proper Windows drive letters.

## Best solution

Keep the shipped fix in bafbf12c59b (1.116.0) which rectifies drive-letter-less unix paths against workspace folder drive letters in PromptPathRepresentationService. No further change needed.
