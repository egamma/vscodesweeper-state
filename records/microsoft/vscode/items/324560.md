# Issue #324560 — terminal tool resize doesn't resize like it does in the actual terminal

[https://github.com/microsoft/vscode/issues/324560](https://github.com/microsoft/vscode/issues/324560)

## Summary

Team-authored issue (MEMBER); the terminal tool's detached mirror terminal does not resize/reflow text when the chat panel width changes, confirmed by source inspection — no resize handling exists in the mirror class. Keeping open per hard keep-open rule for team issues.

## Change summary

Terminal output in the chat terminal tool does not reflow text on resize like the actual terminal does.

## Best solution

Add resize/reflow handling to `DetachedTerminalCommandMirror` so that when the chat panel width changes, the detached xterm instance is resized to match the new available columns, triggering xterm's built-in text reflow. The `_handleResize` path in `ChatTerminalToolProgressPart` already fires on container resize — it needs to propagate the new column width down to the mirror's detached terminal.
