# Issue #119047 — [themes] use both English and UI language to search color theme names

[https://github.com/microsoft/vscode/issues/119047](https://github.com/microsoft/vscode/issues/119047)

## Summary

This feature request remains unimplemented on main. The theme quick pick (`toEntry` in themes.contribution.ts) only uses the single localized `theme.label` for display and filtering — there is no mechanism to also match against the original English name when the UI is in another language.

## Change summary

When VS Code uses a non-English UI language, users cannot type English theme names (like 'dark' or 'light') to filter color themes in the quick pick because only the localized names are searchable.

## Best solution

Store the original English label alongside the localized label in the theme data model, and surface it as an additional searchable field (e.g. via the `detail` or a hidden filter string) in the theme quick pick so that both English and localized names match user input.
