# Issue #119047 — [themes] use both English and UI language to search color theme names

[https://github.com/microsoft/vscode/issues/119047](https://github.com/microsoft/vscode/issues/119047)

## Summary

Valid feature request with low community traction (1 upvote). The theme quick pick only filters on the localized `theme.label`; no English/original label is stored or matched. Still unimplemented on main.

## Change summary

When VS Code UI is in a non-English language, searching color themes by their English name (e.g. 'dark', 'light') no longer matches because only the translated label is used for filtering.

## Best solution

Store the original English label alongside the localized one on theme data, then set `filterText` (or use `matchOnDescription` with the English name in description) on ThemeItem entries in `toEntry()` so the quick pick matches both the localized and English names.
