# Issue #265627 — extensions.allowed setting should support an indirection

[https://github.com/microsoft/vscode/issues/265627](https://github.com/microsoft/vscode/issues/265627)

## Summary

Valid enterprise feature request with documented customer demand (UBS, Bank of America, Barclays, Bloomberg). Not implemented on main — the extensions.allowed setting still only accepts inline configuration, with no URL/file indirection support. Keeping open for prioritization.

## Change summary

Allow extensions.allowed setting to reference an external URL or file path for the allowed-extensions list, enabling enterprises to update it without heavy group-policy rollouts.

## Best solution

Implement URL/file indirection for the `extensions.allowed` setting in `src/vs/platform/extensionManagement/common/extensionManagement.ts` and related services, so the setting value can be a string pointing to a remote JSON file. This enables lightweight, frequent updates for enterprise admins without GPO changes.
