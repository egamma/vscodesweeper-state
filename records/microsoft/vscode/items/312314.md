# Issue #312314 — Agent Hooks dont work when Windows Powershell is restricted on workstation

[https://github.com/microsoft/vscode/issues/312314](https://github.com/microsoft/vscode/issues/312314)

## Summary

The hook executor deliberately hardcodes Windows PowerShell 5.1 as the shell on Windows (commit 04c1359472e), and the recent ExecutionPolicy Bypass fix (1f7db286554) doesn't help when the executable itself is EPERM-blocked by corporate policy. The request to honor user-configured terminal/shell settings is a valid feature request with no duplicate.

## Change summary

Agent hooks should honor the user's configured shell (e.g. PowerShell 7+) instead of hardcoding Windows PowerShell 5.1, which fails with EPERM in restricted corporate environments.

## Best solution

Update `getShellCommand` in `extensions/copilot/src/platform/chat/node/hookExecutor.ts` to check user-configured terminal profile settings (e.g. `chat.tools.terminal.terminalProfile.windows` or `terminal.integrated.defaultProfile.windows`) before falling back to the hardcoded PowerShell 5.1 path. This would let corporate users who have PowerShell 7+ configured actually run hooks.
