# Issue #84620 — Git Merge Resolve Dialog

[https://github.com/microsoft/vscode/issues/84620](https://github.com/microsoft/vscode/issues/84620)

## Summary

Valid feature request to auto-stage files after all merge conflicts are resolved. Not implemented on current main — the merge-conflict extension detects and resolves conflicts but has no integration with git staging. Only 1 upvote and no community traction after ~5 years, but the request touches a real workflow gap.

## Change summary

When all merge conflict markers are resolved in a file, VS Code should offer to mark (stage) the file as resolved in Git.

## Best solution

The merge-conflict extension (`extensions/merge-conflict/`) already tracks when conflicts are resolved via `containsConflict`; it could detect the transition from conflicted to conflict-free and prompt the user to stage the file via the git extension's API. Given the low community traction, this is a reasonable candidate for the community backlog process. An extension could also implement this via existing APIs.
