# Issue #201841 — `onDidChangeTabs` fires before `vscode.window.activeTextEditor` has been updated

[https://github.com/microsoft/vscode/issues/201841](https://github.com/microsoft/vscode/issues/201841)

## Summary

Still present on main: `onDidChangeTabs` and `activeTextEditor` are updated via separate, uncoordinated IPC paths (`$acceptEditorTabModel` vs `$acceptDocumentsAndEditorsDelta`), so the tab event can fire before the active editor is updated. Filed by a team contributor; keeping open.

## Change summary

When switching editor tabs, the `onDidChangeTabs` event fires before `vscode.window.activeTextEditor` reflects the new tab, because the two APIs are driven by independent event/IPC paths with no ordering guarantee.

## Best solution

Coordinate the IPC ordering so that the documents-and-editors delta (which updates `activeTextEditor`) is guaranteed to arrive before the tab-change event fires on the ext host side. The likely approach is either batching both updates into a single IPC call or sequencing them in `mainThreadEditorTabs.ts` / `mainThreadDocumentsAndEditors.ts`. The fix would touch both main-thread dispatchers and possibly the ext host tab model.
