# Issue #179639 — Ghost text background is not covering the whole line height

[https://github.com/microsoft/vscode/issues/179639](https://github.com/microsoft/vscode/issues/179639)

## Summary

Still reproduces on main: ghost text background is applied via `inlineClassName` on inline decorations, which only cover the text span height, not the full editor line height. No fix has landed.

## Change summary

Ghost text background color does not extend to the full line height, leaving a visible gap.

## Best solution

The ghost text decoration's background styling needs to account for full line height. This likely requires either setting a `line-height` or `min-height` on the decoration span in `ghostTextView.css`, or switching to a whole-line decoration class for the background. The fix lives in `src/vs/editor/contrib/inlineCompletions/browser/view/ghostText/ghostTextView.css` and possibly the decoration setup in `ghostTextView.ts`.
