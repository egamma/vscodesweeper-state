# Issue #209354 — Terminal buttons pushed outside of the app window boundary when resizing the panel

[https://github.com/microsoft/vscode/issues/209354](https://github.com/microsoft/vscode/issues/209354)

## Summary

Confirmed by maintainer (benibenj, Comment 7) and still unfixed on main. The composite bar layout calculation in paneCompositePart.ts can fail to account for toolbar width, causing right-aligned buttons to overflow the panel boundary.

## Change summary

When the panel is narrow, right-aligned terminal toolbar buttons get pushed outside the visible boundary instead of the left-aligned composite bar tabs collapsing into an overflow menu.

## Best solution

Fix the timing/calculation in `layoutCompositeBar()` (src/vs/workbench/browser/parts/paneCompositePart.ts line 725) — likely `getToolbarWidth()` returns 0 before the toolbar has rendered, so the composite bar consumes all available width. A CSS-level fix ensuring the composite-bar-container has `overflow: hidden` and `flex-shrink: 1` while title-actions/global-actions have `flex-shrink: 0` would prevent the visual overflow regardless of JS layout timing.
