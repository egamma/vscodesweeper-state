# Issue #192943 — Terminal profile settings not reflected in terminals created using VSCode API

[https://github.com/microsoft/vscode/issues/192943](https://github.com/microsoft/vscode/issues/192943)

## Summary

Still valid on main: when an extension calls `createTerminal()` without specifying `shellPath`, the default terminal profile's env vars and other settings are not merged because `mainThreadTerminalService.$createTerminal` always passes a non-null config object, bypassing the default-profile fallback in `terminalService.createTerminal`. The maintainer (Tyriar) acknowledged this as a valid enhancement in the discussion.

## Change summary

Terminal profile settings (env vars, args, etc.) from the default profile are not applied to terminals created via the `vscode.window.createTerminal()` extension API when `shellPath` is not explicitly set.

## Best solution

When `shellPath` and `shellArgs` are both undefined in the extension API's `createTerminal` call, merge the default profile's properties (env, args, etc.) into the shell launch config — likely by calling `ITerminalProfileResolverService` or `ITerminalProfileService.getDefaultProfile()` in `mainThreadTerminalService.$createTerminal` before constructing the `shellLaunchConfig`, as Tyriar suggested in the discussion.
