# Issue #70283 —  errors in dependsOn background tasks do not prevent subsequent tasks from executing

[https://github.com/microsoft/vscode/issues/70283](https://github.com/microsoft/vscode/issues/70283)

## Summary

Still valid on main: the dependency promise for background tasks always resolves with exitCode 0 regardless of problem matcher errors; commit ac8b3ac9416 added hasErrors tracking/reporting but doesn't block subsequent tasks. Milestone 'On Deck' also prevents any close.

## Change summary

Errors detected by problem matchers in background dependsOn tasks should prevent subsequent dependent tasks from executing.

## Best solution

Modify `_createInactiveDependencyPromise` in `src/vs/workbench/contrib/tasks/browser/terminalTaskSystem.ts` to resolve with a non-zero exitCode when ProblemMatcherFoundErrors fires for the dependency task, so the existing `exitCode !== 0` break in the sequence loop stops subsequent tasks. The hasErrors tracking infrastructure from ac8b3ac9416 is already in place — it just needs to feed into the promise resolution.
