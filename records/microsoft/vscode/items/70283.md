# Issue #70283 —  errors in dependsOn background tasks do not prevent subsequent tasks from executing

[https://github.com/microsoft/vscode/issues/70283](https://github.com/microsoft/vscode/issues/70283)

## Summary

Still a valid gap on current main: background dependency tasks always resolve with exitCode 0 when they go inactive, so problem-matcher errors never block subsequent tasks. The error-tracking infrastructure exists but isn't wired into the inactive-dependency promise. 35 upvotes confirm community demand.

## Change summary

When a background task used in `dependsOn` reports errors via its problem matcher, subsequent dependent tasks should not execute.

## Best solution

Wire `_taskHasErrors` into `_createInactiveDependencyPromise` (in `src/vs/workbench/contrib/tasks/browser/terminalTaskSystem.ts`) so that when a background dependency goes inactive with problem-matcher errors, the returned `ITaskSummary` carries a non-zero `exitCode`, which the existing sequencing logic at lines 569–573 and 586–591 already checks. This is a small, localized change.
