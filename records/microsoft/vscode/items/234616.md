# Issue #234616 — Git - Git blame editor decoration appears before the folding ... one

[https://github.com/microsoft/vscode/issues/234616](https://github.com/microsoft/vscode/issues/234616)

## Summary

Still present on main: the folding 'inline-folded' CSS pseudo-element and the git blame extension API 'after' decoration have no defined relative ordering, so the blame text renders before the folding ellipsis. The issue is valid, in Backlog, and awaits a decision on decoration ordering.

## Change summary

Git blame inline decoration appears before the folding '...' indicator on collapsed lines, when it should appear after.

## Best solution

Introduce decoration ordering/priority for `afterContentClassName` (CSS ::after) vs extension API `after` decorations in the editor's line rendering, so that built-in folding indicators always appear closer to the line content than extension-contributed inline decorations. The fix likely spans `src/vs/editor/contrib/folding/browser/foldingDecorations.ts` and the view-line renderer that merges after-content decorations.
