# Issue #234616 — Git - Git blame editor decoration appears before the folding ... one

[https://github.com/microsoft/vscode/issues/234616](https://github.com/microsoft/vscode/issues/234616)

## Summary

Still present on main: the git blame `after` decoration lacks ordering relative to the folding `afterContentClassName: 'inline-folded'` placeholder, so blame text renders before the '...' indicator on collapsed lines.

## Change summary

Git blame editor decoration appears before the folding '...' indicator on collapsed lines instead of after it.

## Best solution

Set a lower `after` ordering/priority on the blame decoration in `extensions/git/src/blame.ts`, or raise the folding placeholder's priority in `src/vs/editor/contrib/folding/browser/foldingDecorations.ts`, so the '...' always renders immediately after the line content and blame info follows it.
