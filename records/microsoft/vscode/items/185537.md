# Issue #185537 — Cannot select between a sequence of unicode flags

[https://github.com/microsoft/vscode/issues/185537](https://github.com/microsoft/vscode/issues/185537)

## Summary

Still reproduces on main: the grapheme break logic in strings.ts explicitly skips proper GB12/GB13 implementation for Regional Indicator sequences, treating all consecutive flag emoji as one cluster and preventing cursor navigation between them.

## Change summary

Cursor cannot stop between consecutive Unicode flag emoji (e.g. 🇩🇲🇩🇴) because the grapheme break rules treat the entire RI sequence as a single cluster.

## Best solution

Implement proper GB12/GB13 counting in `breakBetweenGraphemeBreakType` (or the callers `nextCharLength`/`prevCharLength`) in `src/vs/base/common/strings.ts` so that breaks occur at even-numbered RI boundaries within a sequence. This requires tracking state (counting preceding RI code points) which the current stateless pair-wise check intentionally avoids.
