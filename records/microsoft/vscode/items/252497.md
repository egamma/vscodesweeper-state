# Issue #252497 — VSCode does not use XDG portals for opening files/folders on COSMIC desktop environment

[https://github.com/microsoft/vscode/issues/252497](https://github.com/microsoft/vscode/issues/252497)

## Summary

Still reproduces on main: VS Code unconditionally sets `xdg-portal-required-version` to 4, which breaks native file dialogs on COSMIC (which supports v4 features but whose portal backend may advertise v3). An open PR from the reporter exists but has not been merged.

## Change summary

On the COSMIC desktop environment, opening files/folders shows a GTK fallback dialog instead of the native COSMIC file chooser because VS Code requires XDG portal version 4.

## Best solution

Review and land (or close) the open PR https://github.com/microsoft/vscode/pull/252498, which adds COSMIC detection in `desktopEnvironmentInfo.ts` and conditionally falls back to portal version 3 in `src/main.ts`. The approach is small and localized.
