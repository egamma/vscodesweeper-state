# Issue #299540 — `git commit` with git config `core.editor = code --wait` opens in window for main git worktree instead of active window

[https://github.com/microsoft/vscode/issues/299540](https://github.com/microsoft/vscode/issues/299540)

## Summary

Valid bug: `findWindowOnFile` in windowsFinder.ts has no awareness of git linked worktrees, so files stored under the main worktree's `.git/worktrees/<name>/` directory (commit messages, rebase editors) are matched to the main worktree window instead of the linked worktree window. No fix exists on current main.

## Change summary

When using `core.editor = code --wait` with git linked worktrees, commit/rebase editor files open in the main worktree's VS Code window instead of the active linked worktree's window.

## Best solution

Augment `findWindowOnFile` in `src/vs/platform/windows/electron-main/windowsFinder.ts` to detect when a file path falls under a `.git/worktrees/<worktree-name>/` directory and resolve the owning linked worktree path, then prefer a window opened on that linked worktree. This is a focused change in one file with a clear heuristic (parse the gitdir link or match `.git/worktrees/` path segments). Related issue https://github.com/microsoft/vscode/issues/304579 (data-loss rebase bug) shares this root cause of operations dispatched to the wrong worktree.
