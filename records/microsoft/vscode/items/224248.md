# Issue #224248 — search error

## Summary

Still open and unresolved on main. The issue is milestoned 'On Deck' (a scheduling milestone) and confirmed by a team member — keeping open per policy.

## Change summary

Search in open editors fails on vscode.dev when files are opened outside the current workspace, because the file service reports them as non-existent and the query builder skips non-absolute-path URIs.

## Best solution

Fix `commonQueryFromFileList` in `src/vs/workbench/services/search/common/queryBuilder.ts` to handle out-of-workspace files on vscode.dev — the `isAbsolutePath(file)` guard at line 311 drops files that don't resolve to absolute local paths, and the fileService's `exists` check (referenced by the maintainer) returns false for out-of-workspace remote files. The fix likely needs to handle non-file-scheme URIs (e.g. virtual FS) as valid search targets.
