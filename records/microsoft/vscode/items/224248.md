[https://github.com/microsoft/vscode/issues/224248](https://github.com/microsoft/vscode/issues/224248)

## Summary

Still reproduces on main: the exists-check in searchService.ts filters out open-editor folder queries for out-of-workspace files on vscode.dev, producing empty search results. Already confirmed by a team member with root cause identified but not yet fixed.

## Change summary

Searching only in open editors on vscode.dev fails when files are opened from outside the current workspace — no results are returned and an error may appear.

## Best solution

The fix should modify `src/vs/workbench/services/search/common/searchService.ts` around line 199 to not filter out folder queries derived from open editors via `fileService.exists()` when searching only in open editors, or ensure the file service can resolve existence for out-of-workspace files opened in the editor on vscode.dev. The root cause was already identified by @andreamah in the last comment.

- **Fix prompt:**

  > In `src/vs/workbench/services/search/common/searchService.ts` around line 199, the `textSearch` method filters folder queries by checking `fileService.exists(query.folder)` for each one. When searching only in open editors (`onlyOpenEditors`), each open editor URI becomes a folder query via `queryBuilder.ts` line 280-286 (`commonQueryFromFileList`). On vscode.dev, files opened from outside the workspace (e.g. via 'Open File...') have URIs whose parent folders are not registered with any file system provider, so `exists()` returns `false` and all those folder queries are silently removed — producing zero results or an error. The fix should skip or special-case the `exists` check for folder queries that originate from open editors (i.e., when `query.onlyOpenEditors` is true or the folder queries are file-level queries from the open editor list). Do NOT remove the exists check entirely — it is needed for normal workspace folder validation. Validate with a test that searches open editors containing out-of-workspace URIs and asserts non-empty results. Related issue: https://github.com/microsoft/vscode/issues/224248
