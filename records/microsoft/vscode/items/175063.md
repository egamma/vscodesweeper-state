# Issue #175063 — Search results come from all the folders in the workspace when the `files to include` is an aliased folder

[https://github.com/microsoft/vscode/issues/175063](https://github.com/microsoft/vscode/issues/175063)

## Summary

Still reproduces on main: the mismatch between `escapeGlobPattern` (escapes only `?*[]`) and `splitGlobFromPath` (treats `(){}` as glob chars too) causes search path splitting to fail for folder aliases containing parentheses or braces, falling back to searching all workspace folders. For aliases with `[]`, the escaped pattern also fails to match the raw `folder.name` in `expandOneSearchPath`.

## Change summary

Right-clicking an aliased workspace folder and choosing 'Find in Folder…' returns results from all workspace folders instead of only the selected one, when the alias contains glob-special characters like `(`, `)`, `{`, `}`, `[`, or `]`.

## Best solution

Harmonize `escapeGlobPattern` and `splitGlobFromPath` in `src/vs/workbench/services/search/common/queryBuilder.ts`: either extend `escapeGlobPattern` to also escape `(`, `)`, `{`, `}` and teach `expandOneSearchPath` to unescape the folder-name portion before matching; or stop treating `()` as glob characters in `splitGlobFromPath` (ripgrep doesn't use them as glob syntax). Add multi-root workspace tests with aliased folders containing special characters.
