# Issue #175063 — Search results come from all the folders in the workspace when the `files to include` is an aliased folder

[https://github.com/microsoft/vscode/issues/175063](https://github.com/microsoft/vscode/issues/175063)

## Summary

Still reproduces on main: the search include path is built using the workspace folder's alias name, which the search engine cannot resolve back to the actual filesystem path. Milestone 'On Deck' mandates keep-open.

## Change summary

"Find in Folder" on an aliased workspace folder returns results from all folders instead of just the targeted one.

## Best solution

Fix `resolveResourcesForSearchIncludes` in `src/vs/workbench/services/search/common/queryBuilder.ts` to use the folder's actual filesystem-relative path (or URI) rather than the display alias when constructing the search include glob pattern. The query builder must resolve the include against the folder's real URI, not `owningFolder.name`.
