# Issue #209658 — Changed lines incorrectly highlighted

[https://github.com/microsoft/vscode/issues/209658](https://github.com/microsoft/vscode/issues/209658)

## Summary

The gutter dirty-diff decorations reportedly don't match the actual diff shown in the diff editor. A July 2025 commenter still experiences this, but the exact reproduction conditions are unclear — could be whitespace-related or an algorithmic edge case in how changes map to line ranges.

## Change summary

SCM gutter change indicators highlight incorrect lines compared to the actual diff shown in the diff editor or inline peek.

## Best solution

Investigate whether the mismatch stems from the `scm.diffDecorationsIgnoreTrimWhitespace` setting (defaults to `'false'`, meaning whitespace changes appear in gutter but not in the diff editor which defaults to ignoring whitespace), or from a genuine algorithmic bug in `quickDiffModel.ts`'s change-to-line mapping. A reproduction case with specific file content is needed to diagnose.
