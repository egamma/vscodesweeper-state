# Issue #295655 — Spawn UNKNOWN in Search/Chat

[https://github.com/microsoft/vscode/issues/295655](https://github.com/microsoft/vscode/issues/295655)

## Summary

This is the canonical issue for ripgrep spawn failures caused by Windows WDAC/AppLocker policies blocking rg.exe execution. Multiple users confirm the root cause, and team members (roblourens, dmitrivMS) are actively investigating. Related issues were closed as duplicates of this one.

## Change summary

Search and Copilot Agent mode fail with 'spawn UNKNOWN' on Windows machines where WDAC/Code Integrity policies block the bundled ripgrep binary from executing.

## Best solution

VS Code needs a graceful fallback or improved error messaging when rg.exe is blocked by WDAC/AppLocker policies. Options include: signing rg.exe so it passes code integrity checks, providing a user-facing error explaining the policy block, or offering a configuration to point to a pre-approved rg.exe path. The search service code lives in src/vs/workbench/services/search/.
