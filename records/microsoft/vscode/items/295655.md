# Issue #295655 — Spawn UNKNOWN in Search/Chat

[https://github.com/microsoft/vscode/issues/295655](https://github.com/microsoft/vscode/issues/295655)

## Summary

Actively investigated by team members (roblourens, dmitrivMS, joshspicer). Root cause confirmed: WDAC/AppLocker enterprise policies block rg.exe from spawning, breaking both Search and Copilot Chat agent mode. No fix on main yet; keeping open for the ongoing investigation.

## Change summary

On Windows machines with WDAC or AppLocker enforcement, VS Code's bundled ripgrep binary is blocked from executing, causing 'spawn UNKNOWN' errors in Search and Copilot Chat agent mode.

## Best solution

Improve error handling when rg.exe fails to spawn due to security policies — surface a clear diagnostic message pointing users/admins to the WDAC/AppLocker cause, and consider a fallback search path or documentation for enterprise allowlisting of VS Code bundled binaries. The search service code in `src/vs/workbench/services/search/` would need better error classification for UNKNOWN spawn failures.
