# Issue #176795 — Variables within a configuration snippet defined in package.json file are not being allowed

[https://github.com/microsoft/vscode/issues/176795](https://github.com/microsoft/vscode/issues/176795)

## Summary

Still reproduces on main: debug configurationSnippets pass the body directly as defaultSnippets to the JSON schema system, where `${command:...}` values are interpreted as snippet variables instead of literal text. No escaping mechanism is documented or properly supported. The related issues #186618 and #133614 were closed without a fix landing.

## Change summary

Extension-contributed debug configurationSnippets cannot include literal `${command:...}` or `${workspaceFolder}` text in their body — these are incorrectly interpreted as snippet variables.

## Best solution

The JSON-to-snippet conversion (where `IJSONSchemaSnippet.body` is stringified into snippet syntax) needs to either escape `${}` sequences that are not intended as snippet placeholders, or provide a documented escaping mechanism for extension authors. The fix likely lives in how `defaultSnippets` bodies are serialized — possibly in `src/vs/workbench/contrib/debug/browser/debugAdapterManager.ts` (lines 217–219) where configurationSnippets are forwarded, or deeper in the JSON language service's snippet-body-to-text conversion. A documented `\$` escape or a `bodyText` alternative for debug snippets would resolve this.
