The discussion identifies a real lifecycle gap where extension-set editor cursor style is not reset after the extension host/profile state changes.

## Change summary

Switching back from a profile where the Vim extension set a block cursor leaves the default profile using that block cursor after the extension is disabled.

## Best solution

Editor/extension-host ownership should decide whether extension-contributed cursor style needs restoration when the extension host stops or an extension is disabled across profile switches; the likely end state is a reset path for extension-set editor options when their owning extension is no longer active.
