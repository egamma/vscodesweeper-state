# Issue #191855 — Block cursor remains after switching to profile with enabled vim extension and back

[https://github.com/microsoft/vscode/issues/191855](https://github.com/microsoft/vscode/issues/191855)

## Summary

Team-authored issue, still valid on main. When an extension sets cursorStyle via the API and the extension host stops (e.g., profile switch), nothing resets the editor option back to the user's configured value — the stale override persists.

## Change summary

Block cursor set by an extension (e.g., Vim) persists after switching to a profile where that extension is disabled, because extension-host teardown doesn't reset editor options.

## Best solution

On extension-host dispose/restart, reset editor options that were overridden by extensions (cursorStyle, lineNumbers) back to their configured defaults. The fix likely belongs in `src/vs/workbench/api/browser/mainThreadEditor.ts` (MainThreadTextEditor.dispose or a new restore method) or in the MainThreadEditors orchestration layer that manages the lifecycle.
