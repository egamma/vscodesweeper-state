# Issue #150938 — Add nvm support

[https://github.com/microsoft/vscode/issues/150938](https://github.com/microsoft/vscode/issues/150938)

## Summary

Valid feature request for nvm/.nvmrc support in VS Code's task runner. The associated PR was closed unmerged and no nvm support exists on current main. With 18 upvotes and community interest, it should remain open for the community process.

## Change summary

VS Code tasks should respect .nvmrc files and use the correct Node version managed by nvm when launching npm scripts.

## Best solution

Implement .nvmrc-aware Node resolution in the task execution layer (likely via nvm-exec or reading .nvmrc to resolve the correct Node binary path before spawning tasks). The related closed PR #150940 may serve as a starting point. Alternatively, a `runtimeVersion` property for tasks (as suggested in comments) could unify with the existing debug configuration approach.
