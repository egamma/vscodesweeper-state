Well-supported feature request with 91 upvotes, promoted to Backlog by the community process. No implementation exists on current main. Two team members have commented positively. The earlier issue https://github.com/microsoft/vscode/issues/121635 was closed by the triage bot for insufficient upvotes — this issue is the active canonical tracker and should stay open.

## Change summary

Add support for organizing uncommitted changes into multiple named changelists in the Source Control view, similar to IntelliJ IDEs.

## Best solution

Implement a changelist-style grouping mechanism in the SCM view (likely in `src/vs/workbench/contrib/scm/`) that lets users create named groups of uncommitted changes, move files between them, and commit from each independently — all without branches or stashes. The SCM API may also need extension points so third-party SCM providers can participate.
