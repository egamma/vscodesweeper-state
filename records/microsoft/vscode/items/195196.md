# Issue #195196 — VS Code in Linux freezes when copying file name from file dialog and pasting it into the editor

[https://github.com/microsoft/vscode/issues/195196](https://github.com/microsoft/vscode/issues/195196)

## Summary

Confirmed Linux-only freeze when pasting text copied from a native file dialog into the editor. A team member reproduced it, and the original reporter confirmed it persists on v1.106.2 (November 2025). No fix has been committed. The bug remains valid and unresolved.

## Change summary

VS Code on Linux freezes (becomes unresponsive, requires force-close) when pasting text that was copied from a native file dialog (e.g. Save As) into the editor.

## Best solution

Investigate the interaction between native file dialog clipboard content and the editor paste handler pipeline on Linux. The status bar shows 'running paste handlers' during the freeze, suggesting the paste handler chain in `src/vs/editor/contrib/dropOrPasteInto/browser/copyPasteController.ts` hangs — likely during `mergeInDataFromCopy` or when Electron reads the system clipboard (`readResources` / `readClipboardBuffer`). A workaround exists: enabling the simple file dialog avoids the native dialog clipboard issue. The root cause may be in Electron/Chromium's X11 clipboard integration, making this partially upstream, but the paste handler should not hang indefinitely regardless of clipboard content — adding a timeout or guard would be a reasonable mitigation.
