# Issue #294751 — Claude model error: `temperature` and `top_p` cannot both be specified for this model

[https://github.com/microsoft/vscode/issues/294751](https://github.com/microsoft/vscode/issues/294751)

## Summary

Fixed on main: the Claude language model server now explicitly guards against sending both `temperature` and `top_p` simultaneously, which resolves the reported 400 error.

## Change summary

Claude API requests from Copilot Chat failed with a 400 error when both `temperature` and `top_p` were included in the request body.

## Best solution

Keep the shipped fix in commit c1c606c16dd — the guard in `claudeLanguageModelServer.ts` (lines 876–881) deletes both parameters from the base request when the SDK specifies either, preventing the conflict.
