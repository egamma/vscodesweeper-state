# Issue #209475 — SCM Input - Allow configuring scm input as a specific language

[https://github.com/microsoft/vscode/issues/209475](https://github.com/microsoft/vscode/issues/209475)

## Summary

Valid feature request that remains unimplemented. The SCM input currently uses a hardcoded `scminput` language ID with no setting to override it. The `ISCMInput` interface has no `languageId` property, and no `scm.inputLanguage` configuration exists. Keeping open in the Backlog as a legitimate enhancement.

## Change summary

Allow configuring the SCM input editor to use a specific language (e.g. markdown) for syntax highlighting, bracket matching, snippets, and IntelliSense.

## Best solution

Add a `scm.inputLanguage` setting (defaulting to `scminput`) that lets users override the language mode of the SCM input editor. The implementation would touch `src/vs/workbench/contrib/scm/browser/scm.contribution.ts` (setting registration) and the SCM input model/widget to apply the configured language to the editor model. This enables markdown highlighting, snippets, and IntelliSense in commit message boxes.
