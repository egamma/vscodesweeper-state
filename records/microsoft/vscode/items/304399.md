# Issue #304399 — Copilot ripgrep EACCES error

[https://github.com/microsoft/vscode/issues/304399](https://github.com/microsoft/vscode/issues/304399)

## Summary

An EACCES error on the ripgrep binary on one specific remote Linux server strongly suggests a local filesystem permission issue (execute bit lost), not a VS Code bug. Needs confirmation from the reporter.

## Change summary

Copilot's ripgrep search fails with EACCES on a specific remote Linux server that was previously working.

## Best solution

The reporter should check permissions on the ripgrep binary (`ls -la ~/.vscode-server/cli/servers/Stable-*/server/node_modules/@vscode/ripgrep/bin/rg`) and restore execute permission with `chmod +x` if missing. If the problem persists after that, a fresh server install (`Remote-SSH: Uninstall VS Code Server from Host`) would reset the binaries.
