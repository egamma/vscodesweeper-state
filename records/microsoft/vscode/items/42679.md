# Issue #42679 — [json] schema Validation/Intellisense very slow when JSON deep and Schema Complex

[https://github.com/microsoft/vscode/issues/42679](https://github.com/microsoft/vscode/issues/42679)

## Summary

Known performance limitation in JSON schema validation with deeply nested schemas and many alternatives — acknowledged by maintainer but unfixed. No time budget allocated as of 2023. Still valid on current main.

## Change summary

JSON schema validation and intellisense become extremely slow with deeply nested JSON and complex schemas with many alternatives.

## Best solution

Optimize the schema validation in vscode-json-languageservice to short-circuit child validation early when the parent type clearly doesn't match, avoiding combinatorial explosion of alternatives. The fix likely lives in the upstream `vscode-json-languageservice` package (schema validation logic), not in this repo directly.
