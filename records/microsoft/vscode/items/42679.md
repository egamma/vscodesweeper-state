# Issue #42679 — [json] schema Validation/Intellisense very slow when JSON deep and Schema Complex

[https://github.com/microsoft/vscode/issues/42679](https://github.com/microsoft/vscode/issues/42679)

## Summary

Acknowledged performance bug in JSON schema validation with deeply nested/recursive schemas. Team member explained root cause (combinatorial explosion testing alternatives) but no fix has landed. Still valid.

## Change summary

JSON schema validation and intellisense become extremely slow or freeze the editor when schemas are complex/recursive and documents are deeply nested.

## Best solution

Optimize the JSON language service's schema validation to short-circuit child validation when a node's type clearly doesn't match, avoiding the combinatorial explosion. The fix belongs in the upstream `vscode-json-languageservice` library (schema validation logic). Alternatively, allow intellisense to respond independently of full validation completing.
