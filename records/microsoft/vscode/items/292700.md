## Summary

Team-authored issue describing a known UX gap: when an extension invokes an LM tool outside a chat request (unset toolInvocationToken), the confirmation falls back to a modal dialog instead of the inline chat confirmation flow. connor4312 confirmed this is an unpolished scenario. ## Change summary

## Best solution
