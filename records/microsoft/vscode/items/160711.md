# Issue #160711 — Given VS Code processes unique icons and titles on Windows

[https://github.com/microsoft/vscode/issues/160711](https://github.com/microsoft/vscode/issues/160711)

## Summary

Team-contributor-authored feature request for unique process icons/titles in Windows Task Manager. Hard keep-open rule applies (author is CONTRIBUTOR). No implementation found on main, and the request remains valid — VS Code processes are still difficult to differentiate in Task Manager.

## Change summary

Give each VS Code process (main, renderer, extension host, etc.) a unique icon and descriptive title in the Windows Task Manager, similar to how Chromium-based browsers like Edge identify their processes.

## Best solution

Investigate using Windows APIs (e.g. SetCurrentProcessExplicitAppUserModelID or similar per-process metadata) to assign distinguishable names/icons to VS Code's child processes. This likely involves changes in the Electron main process startup and shared process/extension host spawning code. Needs maintainer/platform-owner decision on feasibility and Electron support.
