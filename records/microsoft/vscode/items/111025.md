# Issue #111025 — [html] HTML line wrapping on paste occurs before auto-indenting

[https://github.com/microsoft/vscode/issues/111025](https://github.com/microsoft/vscode/issues/111025)

## Summary

Still present on main: the HTML range formatter (used by formatOnPaste) passes the selection to js-beautify without adjusting wrapLineLength for the indentation context, so wrapped lines exceed the limit after indentation is applied.

## Change summary

HTML formatOnPaste wraps lines as if content is at root indentation level, causing incorrect line lengths when content is deeply nested.

## Best solution

When performing range formatting, either subtract the current indentation width from wrapLineLength before passing to js-beautify, or skip wrapLineLength entirely for range-only formats. The fix likely belongs in `extensions/html-language-features/server/src/modes/formatting.ts` or the html mode's format call that delegates to js-beautify.
