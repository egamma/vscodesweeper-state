# Issue #111025 — [html] HTML line wrapping on paste occurs before auto-indenting

[https://github.com/microsoft/vscode/issues/111025](https://github.com/microsoft/vscode/issues/111025)

## Summary

Still unfixed on main. The root cause (identified by aeschli) is that formatOnPaste runs the HTML formatter on just the pasted range without accounting for surrounding indentation context, causing wrapLineLength to wrap at the wrong column. The fix likely needs to happen in vscode-html-languageservice.

## Change summary

HTML formatOnPaste wraps lines at wrong column because it ignores the indentation depth of the paste target.

## Best solution

The HTML range formatter (delegated to vscode-html-languageservice via `htmlLanguageService.format()` in `extensions/html-language-features/server/src/modes/htmlMode.ts`) should either subtract the current indentation from wrapLineLength when formatting a range, or ignore wrapLineLength entirely during range-only formatting as aeschli suggested.
