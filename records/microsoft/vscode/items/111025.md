# Issue #111025 — [html] HTML line wrapping on paste occurs before auto-indenting

[https://github.com/microsoft/vscode/issues/111025](https://github.com/microsoft/vscode/issues/111025)

## Summary

Still reproducible on main. The HTML range formatter (used by formatOnPaste) wraps lines using wrapLineLength without accounting for the indentation depth of the paste target, causing subsequent full-document formats to re-wrap.

## Change summary

HTML formatOnPaste wraps lines as if content is at root indentation level, ignoring the actual nesting depth.

## Best solution

When performing range formatting, the HTML formatter in vscode-html-languageservice should subtract the current indentation width from wrapLineLength, or the VS Code HTML extension should pass an adjusted wrapLineLength for range formats. The fix likely lives in vscode-html-languageservice's htmlFormatter (upstream dependency) rather than in this repo's formatting.ts wrapper.
