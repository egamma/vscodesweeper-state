# Issue #301893 — Function name, and first line in function overlap when Editor: Line Height != 0, and tokenColors:fontSize are used

[https://github.com/microsoft/vscode/issues/301893](https://github.com/microsoft/vscode/issues/301893)

## Summary

The reported overlap between function declarations and subsequent lines when combining editor.lineHeight < 1, tokenColors fontSize > 1, and bold/underline styles appears to be a genuine rendering issue in the font-token-customization feature. The fix at 47819e962a7 (setting lineHeight = fontSize when unspecified) was already shipped in the user's version but doesn't fully resolve edge cases with reduced global editor.lineHeight.

## Change summary

Lines overlap when tokenColors fontSize (e.g. 1.4) exceeds the global editor.lineHeight setting (e.g. 1.15), especially with bold/underline styles.

## Best solution

The line height computation in `src/vs/editor/common/viewLayout/lineHeights.ts` (line 461) should account for the case where the custom per-token lineHeight multiplier needs to produce enough space for the enlarged font even when the base editor lineHeight is reduced. The fix should ensure that decorated lines always have sufficient height for their actual rendered font size.
