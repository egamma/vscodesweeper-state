# Issue #270437 — Bad terminal history navigation after terminal clear command

[https://github.com/microsoft/vscode/issues/270437](https://github.com/microsoft/vscode/issues/270437)

## Summary

Fixed on main: the workaround (`terminal.integrated.windowsUseConptyDll: true`) that resolved the issue for both reporters is now the default, shipped in 1.121.0.

## Change summary

After clearing the terminal, arrow-up history navigation displays commands at wrong positions in the terminal view on Windows.

## Best solution

Keep the shipped default (`windowsUseConptyDll: true` since 1.121.0); no further change needed. The newer conpty.dll bundled with VS Code handles the clear+history interaction correctly.
