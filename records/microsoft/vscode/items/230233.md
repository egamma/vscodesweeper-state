# Issue #230233 — Terminal search highlights update really slowly

[https://github.com/microsoft/vscode/issues/230233](https://github.com/microsoft/vscode/issues/230233)

## Summary

Still unfixed on main. The terminal find widget delegates to xterm's search addon with a 20,000-match highlight limit; no debounce or throttle guards the incremental search path in `_onInputChanged`, so every keystroke triggers a full `findPrevious` with `incremental: true`, causing visible lag when many matches exist. The issue remains valid and should stay open.

## Change summary

Terminal search highlights update very slowly when typing or editing the search query in a terminal with many matches.

## Best solution

Add debouncing or throttling to the incremental search path in `TerminalFindWidget._onInputChanged` (or in the xterm search addon itself) so that highlight decoration updates don't block on every keystroke. The `highlightLimit` (currently 20,000) may also need tuning. The fix likely spans `src/vs/workbench/contrib/terminalContrib/find/browser/terminalFindWidget.ts` and possibly upstream xterm.js search addon work.
