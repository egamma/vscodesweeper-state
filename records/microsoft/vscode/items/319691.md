# Issue #319691 — Copilot Chat: Editing model on historical message is ignored when regenerating — always falls back to bottom selector model

[https://github.com/microsoft/vscode/issues/319691](https://github.com/microsoft/vscode/issues/319691)

## Summary

Confirmed source-reproducible on main: the retry/resend flow always reads the model from the bottom input widget rather than from the request's own `modelId`, so a per-turn model override on a historical message is silently ignored.

## Change summary

When editing a historical chat message's model and regenerating, the regenerated response uses the bottom model selector's model instead of the per-message model override.

## Best solution

In the `RetryChatAction` (chatTitleActions.ts ~L255), the rerun-with-agent path (chatListWidget.ts ~L349), and `rerunLastRequest` (chatWidget.ts ~L2463), prefer `request.modelId` over `widget.input.currentLanguageModel` when the request carries a model override. The request's `modelId` field already stores the per-turn model — it just needs to be consulted during resend.
