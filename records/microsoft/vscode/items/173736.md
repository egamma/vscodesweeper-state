# Issue #173736 — Code action not working even though command is invoked

[https://github.com/microsoft/vscode/issues/173736](https://github.com/microsoft/vscode/issues/173736)

## Summary

Still reproduces on main: when a code action's command triggers another code action request (e.g. `editor.action.organizeImports`), the retrigger in `applyCodeAction` cancels the in-flight request. Author is a MEMBER and milestone is On Deck — keeping open.

## Change summary

Code actions that delegate to another code action command (e.g. a Pylint quick fix invoking `editor.action.organizeImports`) silently fail when triggered from the editor lightbulb, because the retrigger cancels the delegated request.

## Best solution

In `codeActionController.ts`, the `retrigger` call at line 194-196 should either be deferred until any command-initiated code action request completes, or the retrigger should not cancel an already-running code action request that was spawned by the applied action's command. The fix is localized to `applyCodeAction` in `src/vs/editor/contrib/codeAction/browser/codeActionController.ts` and the cancellation logic in `codeActionModel.ts`.
