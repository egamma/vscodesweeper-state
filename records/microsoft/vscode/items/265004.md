# Issue #265004 — Add a warning message in output window if mcp.json is not well formed.

[https://github.com/microsoft/vscode/issues/265004](https://github.com/microsoft/vscode/issues/265004)

## Summary

Team-authored feature request for better user feedback when mcp.json is malformed. The current code silently swallows parse errors, so the request remains valid and unimplemented.

## Change summary

Surface a warning in the MCP output channel when mcp.json cannot be parsed, informing the user that the last known good configuration is being used.

## Best solution

Add a warning log to the MCP output channel in the discovery layer (e.g. `workspaceDotMcpDiscovery.ts` and the settings-based MCP configuration parsing paths) when JSON parsing fails, so users understand why newly added headers or config changes are not taking effect.
