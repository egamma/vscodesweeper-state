# Issue #316946 — [Copilot Chat] Feature request: Mobile companion experience for monitoring and refining in-progress chat / agent sessions

[https://github.com/microsoft/vscode/issues/316946](https://github.com/microsoft/vscode/issues/316946)

## Summary

Team-authored feature request for a mobile companion experience to monitor and steer in-progress Copilot Chat / Agent sessions. Kept open for explicit maintainer judgment — the request is well-scoped and identifies a concrete API gap, but the mobile delivery surface and relay architecture require a product decision well beyond triage.

## Change summary

Provide a first-party mobile experience (app or PWA) that lets developers monitor, refine, and approve/reject actions in their running VS Code Copilot Chat / Agent sessions from a phone.

## Best solution

This needs a product decision at the intersection of VS Code, GitHub Mobile, and Copilot platform teams. The immediate actionable piece for VS Code is the API gap the author identifies: no public API to enumerate, subscribe to, or inject prompts into Copilot Chat sessions beyond @-mentioned participants. The related issue https://github.com/microsoft/vscode/issues/318855 (chat session observation API) covers part of this gap. A product owner should decide whether the mobile surface belongs in GitHub Mobile, a standalone PWA, or is out of scope — and whether the underlying session-observation API should be prioritized independently of the mobile story.
