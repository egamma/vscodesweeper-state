# Issue #326309 — 1.129.0 renderer crashes repeatedly (SIGSEGV/SIGBUS/SIGTRAP) when attaching to a Dev Container; downgrade to 1.128.1 fixes it

[https://github.com/microsoft/vscode/issues/326309](https://github.com/microsoft/vscode/issues/326309)

## Summary

Well-documented crash regression report (SIGSEGV/SIGBUS/SIGTRAP in the renderer) between 1.128.1 and 1.129.0 during Dev Container attach over SSH-tunneled Docker. The crash is inside the `code` binary itself, not extensions or drivers. Routing to electron and freeze-slow-crash-leak areas for investigation — likely related to the Electron 42.5.0 bump that landed between those releases.

## Change summary

VS Code 1.129.0 renderer crashes repeatedly with SIGSEGV/SIGBUS/SIGTRAP when attaching to a Dev Container over an SSH-tunneled Docker context; 1.128.1 does not crash.

## Best solution

Investigate whether the Electron 42.5.0 bump (commit 4b6f5e55bb8) or another change between 1.128.1 and 1.129.0 introduced a V8 CHECK() regression on the Dev Container attach path. The crash signature (SIGTRAP at address 0x0 inside the `code` binary) matches a V8/Chromium internal assertion — may need Electron-side bisection or a Crashpad symbolicated stack. Issue https://github.com/microsoft/vscode/issues/314075 documents an identical failure class in a prior release.

- **prior similar issue:** https://github.com/microsoft/vscode/issues/314075 documents the same failure class (deterministic SIGTRAP / V8 CHECK() failure, fixed by downgrading one version) in VS Code 1.118.1, still open.
- **GPU ruled out:** Reporter confirmed the crash reproduces identically with hardware acceleration disabled (disable-hardware-acceleration in argv.json), ruling out Mesa/driver correlation.
