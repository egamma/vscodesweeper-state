Real accessibility bug: screen reader announces 'Action' instead of the model name when selecting BYOC models in the chat model picker. The current code on main has accessibility infrastructure (getAriaLabel, isChecked, getRole as menuitemradio) but it's unclear whether the specific BYOC model path was broken at report time or remains broken.
