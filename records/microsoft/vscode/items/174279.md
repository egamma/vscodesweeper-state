# Issue #174279 — All extension network traffic breaks when system proxy settings are changed

[https://github.com/microsoft/vscode/issues/174279](https://github.com/microsoft/vscode/issues/174279)

## Summary

Still reproduces on current main: the proxy resolution cache in @vscode/proxy-agent has no invalidation mechanism when system proxy settings change at runtime. A team member (chrmarti) acknowledged the caching as the root cause. The fix belongs in the @vscode/proxy-agent package.

## Change summary

Extension network traffic breaks permanently when system proxy settings change while VS Code is running, because the proxy-agent caches resolved proxies per URL with no TTL or invalidation.

## Best solution

Add a cache invalidation mechanism in @vscode/proxy-agent (microsoft/vscode-proxy-agent) — either a TTL on cached entries, a listener for system proxy changes, or an explicit resetProxyCache() API callable from VS Code when network conditions change. The proxy cache Map in createProxyResolver (node_modules/@vscode/proxy-agent/out/index.js ~line 66) needs to be clearable.

## Evidence

- **no invalidation on main:** grep for resetCaches/proxy change detection in src/vs/workbench/api/node/proxyResolver.ts shows no mechanism to clear the proxy resolution cache on system proxy changes. The existing resetCaches export only clears certificate caches, not proxy caches.
