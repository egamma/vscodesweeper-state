# Issue #157275 — Unable to connect to VS Code server: Error in request - ENOENT /run/user/1000/vscode-ipc-*.sock

[https://github.com/microsoft/vscode/issues/157275](https://github.com/microsoft/vscode/issues/157275)

## Summary

Still present on main: `server.cli.ts` uses `VSCODE_IPC_HOOK_CLI` without checking socket liveness. When the server restarts or reconnects, existing terminals hold a stale env var pointing to a dead socket, causing ENOENT. No fix or fallback has been implemented.

## Change summary

Running `code <file>` in a remote terminal fails with ENOENT when the IPC socket referenced by VSCODE_IPC_HOOK_CLI no longer exists after a server restart/reconnect.

## Best solution

Add a socket-existence check (e.g. `fs.existsSync` or `fs.access`) in `src/vs/server/node/server.cli.ts` before making the HTTP request, and either surface a clearer error message or attempt to discover a live socket from the known directory (`/run/user/$UID/` or `/tmp/`). Alternatively, the terminal channel could re-inject a fresh `VSCODE_IPC_HOOK_CLI` into existing terminals on reconnect.
