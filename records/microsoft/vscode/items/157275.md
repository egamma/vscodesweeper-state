# Issue #157275 — Unable to connect to VS Code server: Error in request - ENOENT /run/user/1000/vscode-ipc-*.sock

[https://github.com/microsoft/vscode/issues/157275](https://github.com/microsoft/vscode/issues/157275)

## Summary

Still unfixed on main: the CLI reads VSCODE_IPC_HOOK_CLI at startup and passes it directly as socketPath without validating the socket exists, so stale env values from reconnected sessions cause ENOENT. The bug persists with recent confirmations (June 2026).

## Change summary

Running `code <file>` in a remote/WSL terminal fails with ENOENT when VSCODE_IPC_HOOK_CLI points to a stale IPC socket that no longer exists.

## Best solution

In `src/vs/server/node/server.cli.ts`, before using `cliPipe` as the `socketPath`, check whether the socket file exists (e.g. `fs.existsSync(cliPipe)`). If it doesn't exist, either search for an active vscode-ipc socket in the expected directory or emit a clearer error message guiding the user to reconnect.
