# Issue #284990 — Command Palette cannot find related commands when Keyboard Shortcuts view is open

[https://github.com/microsoft/vscode/issues/284990](https://github.com/microsoft/vscode/issues/284990)

## Summary

Team-authored issue (MEMBER); the behavior is confirmed by TylerLeonhardt as caused by EditorAction's implicit `when` clause filtering commands from the Command Palette when no editor is focused. Needs a maintainer decision on whether to surface these commands regardless of editor focus.

## Change summary

EditorAction-based commands (e.g. font zoom) disappear from the Command Palette when no code editor is focused, such as when the Keyboard Shortcuts view is active.

## Best solution

Either relax the Command Palette `when` clause for EditorActions that have `precondition: undefined` (so they show even without an active editor), or accept as by-design. The relevant logic is in `src/vs/editor/browser/editorExtensions.ts` where EditorCommand registers menu items with the precondition as the `when` clause. A targeted fix could add a separate CommandPalette registration without the editor-focus requirement for commands with no precondition.
