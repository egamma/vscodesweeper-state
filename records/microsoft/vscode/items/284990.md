# Issue #284990 — Command Palette cannot find related commands when Keyboard Shortcuts view is open

[https://github.com/microsoft/vscode/issues/284990](https://github.com/microsoft/vscode/issues/284990)

## Summary

Team-authored issue, so it should stay open for explicit maintainer judgment even though the current behavior is plausibly by design because the commands are editor-scoped by when-clause/context rules.

## Change summary

Make editor-related commands such as Decrease Editor Font Size discoverable in the Command Palette even when Keyboard Shortcuts is the active editor.

## Best solution

Have the editor/command-palette owners decide whether unavailable editor actions should remain searchable as disabled or lower-ranked entries, or whether the current when-clause filtering is the intended product boundary.
