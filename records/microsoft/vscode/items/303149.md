# Issue #303149 — Git sync doesn't work after squash + merge PR

[https://github.com/microsoft/vscode/issues/303149](https://github.com/microsoft/vscode/issues/303149)

## Summary

Still reproduces on main: when a remote has both a tag and a branch with the same name (e.g. `main`), VS Code's `git pull --tags origin main` resolves the branch argument as a tag, causing the pull to silently fetch nothing. The bare branch name on line 2473 of `extensions/git/src/git.ts` should be disambiguated with `refs/heads/` prefix.

## Change summary

Git sync/pull silently fails when `git.pullTags` is enabled and a remote tag shares a name with the branch being pulled.

## Best solution

In `extensions/git/src/git.ts` line 2473, disambiguate the branch argument by pushing `refs/heads/${branch}` instead of the bare `branch` name. The same pattern should be checked in `sync()` which also passes a bare `pullBranch` (repository.ts line 2399). This avoids git's ambiguous refname resolution when `--tags` is used alongside an explicit branch.
