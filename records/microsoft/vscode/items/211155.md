# Issue #211155 — `HoverContext` should pass back the specific type of hover I provided

[https://github.com/microsoft/vscode/issues/211155](https://github.com/microsoft/vscode/issues/211155)

## Summary

This is a valid API design debt item: the proposed hover verbosity API still uses a non-generic `HoverContext` with `previousHover?: Hover` rather than returning the specific hover type. The related PR #211223 was closed without merging, and the issue is correctly labeled `on-hold` / `debt`. It should remain open.

## Change summary

Make `HoverProvider` generic so `HoverContext.previousHover` returns the specific hover subtype, avoiding an extra cast.

## Best solution

When the `editorHoverVerbosityLevel` proposed API is ready to finalize, make `HoverProvider<T extends Hover = Hover>` generic and type `HoverContext.previousHover` as `T`. The change lives in `src/vscode-dts/vscode.proposed.editorHoverVerbosityLevel.d.ts` and the corresponding internal provider wiring.
