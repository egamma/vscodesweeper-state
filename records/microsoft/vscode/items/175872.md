# Issue #175872 — setBreakpoints is called with sourceModified:false when source is dirty and new breakpoint is added

[https://github.com/microsoft/vscode/issues/175872](https://github.com/microsoft/vscode/issues/175872)

## Summary

Still present on main: `sendBreakpoints` in `debugService.ts` never checks whether the file is dirty when adding/removing/toggling breakpoints — it always defaults `sourceModified` to `false`. Only the on-save path (line 1355) passes `true`. The bug is confirmed in the source.

## Change summary

When a breakpoint is added to a dirty (unsaved) file during a debug session, `setBreakpoints` is sent to the debug adapter with `sourceModified: false` instead of `true`.

## Best solution

In `debugService.ts`, callers of `sendBreakpoints` (addBreakpoints, removeBreakpoints, enableOrDisableBreakpoints, updateBreakpoints) should check whether the model for the given URI is dirty and pass `sourceModified: true` when it is. This requires injecting or accessing the text model service to query dirty state. The DAP spec semantics should also be clarified (tracked in https://github.com/microsoft/debug-adapter-protocol/issues/376).
