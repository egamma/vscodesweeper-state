# Issue #66148 — [html] code complete for href anchors

[https://github.com/microsoft/vscode/issues/66148](https://github.com/microsoft/vscode/issues/66148)

## Summary

Valid feature request for HTML anchor/id completion in href attributes. Not implemented on current main — the HTML language server has path completion for file references but no support for completing in-document fragment identifiers (#id). Low engagement (2 upvotes over 5+ years) but a legitimate gap.

## Change summary

Request autocomplete suggestions for in-document `#id` anchors when typing `href="#"` in HTML files.

## Best solution

Implement fragment identifier completion in the HTML language server (likely in vscode-html-languageservice, the upstream LS library) that collects all `id` attributes in the current document and offers them as completions when the user types `href="#`.
