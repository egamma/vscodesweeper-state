# Issue #90301 — Allow to use decorations in quick input (e.g. for editors picker)

[https://github.com/microsoft/vscode/issues/90301](https://github.com/microsoft/vscode/issues/90301)

## Summary

Still unimplemented on main: the quick input API lacks file decoration support (git status badges/colors). Active design discussion with prototypes from team members, but no solution landed.

## Change summary

Show git decoration status (modified/untracked badges and colors) on file items in the quick input picker (e.g. Ctrl+Tab editor switcher).

## Best solution

Resolve the design questions raised in the discussion (badge placement, color contrast, alignment with buttons) and implement decoration support in the quick pick item renderer — likely touching `src/vs/platform/quickinput/browser` for rendering and `src/vs/workbench/browser/parts/editor/editorQuickAccess.ts` for applying decorations to editor picks.
