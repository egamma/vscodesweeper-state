# Issue #224890 — "Add a folder to Workspace" stops working when opening multiple folders

[https://github.com/microsoft/vscode/issues/224890](https://github.com/microsoft/vscode/issues/224890)

## Summary

Milestone 'On Deck' requires keep-open per policy. The bug is plausible: the dialog queue/lock mechanism in dialogMainService.ts could get permanently stuck if the Electron native GTK dialog on Linux errors or hangs when `canSelectMany: true` + `canSelectFolders: true` are combined, blocking all subsequent dialogs for that window until reload.

## Change summary

Selecting multiple folders in the native 'Add Folder to Workspace' dialog on Linux silently fails and permanently breaks the dialog until VS Code is reloaded.

## Best solution

Investigate the dialog lock/queue interaction in `src/vs/platform/dialogs/electron-main/dialogMainService.ts` — if the Electron `showOpenDialog` promise rejects or hangs with multi-folder selection on GTK, the per-window queue stays blocked. A timeout or error guard on the queued dialog call would prevent the permanent hang. May also be an upstream Electron bug with GTK multi-folder selection.
