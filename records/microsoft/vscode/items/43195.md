# Issue #43195 — [json] launch.json completion for "type" does not include expected values

[https://github.com/microsoft/vscode/issues/43195](https://github.com/microsoft/vscode/issues/43195)

## Summary

Still an open, confirmed bug in JSON schema completion with anyOf/oneOf. The attempted fixes in vscode-json-languageservice were not merged, and a related issue (#201111) was closed as a duplicate of this one. The bug remains unfixed on main.

## Change summary

JSON schema completion omits valid enum values when multiple schema branches (anyOf/oneOf) apply, particularly for launch.json "type" field.

## Best solution

The fix belongs in `microsoft/vscode-json-languageservice` — the schema matching algorithm needs to correctly aggregate completion candidates across all applicable anyOf/oneOf branches. PR https://github.com/microsoft/vscode-json-languageservice/pull/237 is still open and represents an ongoing attempt at the fix.
