# Issue #43195 — [json] launch.json completion for "type" does not include expected values

[https://github.com/microsoft/vscode/issues/43195](https://github.com/microsoft/vscode/issues/43195)

## Summary

Still a valid bug — the JSON language service incorrectly narrows completions when schema uses oneOf/anyOf and partial properties are filled. Confirmed by a team member in 2018, and a user reported it still reproduces in 2024. The root cause is in the schema-matching logic of the vscode-json-languageservice library.

## Change summary

JSON completions for launch.json "type" field (and other oneOf/anyOf schemas) incorrectly filter out valid values when sibling properties are present.

## Best solution

Fix the schema-matching/narrowing logic in the upstream `vscode-json-languageservice` library (https://github.com/microsoft/vscode-json-languageservice) so that oneOf/anyOf branches are not prematurely eliminated based on partial property matches. The debug adapter schema in `src/vs/workbench/contrib/debug/browser/debugAdapterManager.ts` uses `items.oneOf` for configurations (line 214), and the completion narrowing bug lives in the language service's completion resolver.
