## Summary

This is a duplicate of the well-analyzed canonical issue tracking unbounded webview cache growth. The reporter's 450 GB WebStorage cache matches the same root cause: media-preview cache-busts with Date.now() and the webview service worker never evicts cached entries.

## Change summary

VS Code's WebStorage cache grows unboundedly to hundreds of gigabytes because the webview service worker caches every resource response but never evicts, and the media-preview extension cache-busts with Date.now() on every render.
