# Issue #185515 — files.Readonly has weak graphical support

[https://github.com/microsoft/vscode/issues/185515](https://github.com/microsoft/vscode/issues/185515)

## Summary

Valid polish request: readonly files/directories in the explorer have no visual decoration (badge, color, or dimming) despite the model tracking readonly status. The gap remains on current main.

## Change summary

Files and directories configured as readonly via `files.readonlyInclude` (or filesystem-readonly) lack any visual indicator in the file explorer, making them indistinguishable from writable items.

## Best solution

Add a readonly decoration in `src/vs/workbench/contrib/files/browser/views/explorerDecorationsProvider.ts` — e.g. a lock badge character and/or `listDeemphasizedForeground` color — when `fileStat.isReadonly` is truthy, similar to the existing symlink and excluded-file decorations. Separately, exposing readonly status to the `FileDecorationProvider` extension API would let icon-theme extensions style readonly items themselves, but that's a broader API discussion.
