# Issue #292810 — Vs code terminal process incorrect behaviour

[https://github.com/microsoft/vscode/issues/292810](https://github.com/microsoft/vscode/issues/292810)

## Summary

Plausible terminal input regression on Linux after PR #283065 removed PTY write chunking, but the report lacks a minimal reproduction without the CMake Tools extension and no maintainer has investigated. Routing to terminal area for triage.

## Change summary

After PR #283065 removed terminal input throttling, CMake Tools extension's compile command fails because g++ receives no input files — suggesting command-line data is lost or truncated when written to the PTY.

## Best solution

The terminal area owner should determine whether large single-write PTY input on Linux can lose data without the old chunking (similar to the macOS 1024-byte buffer issue in #296955). If confirmed, a Linux-appropriate chunking guard in `src/vs/platform/terminal/node/terminalProcess.ts` input() would fix it. If it's purely a CMake Tools issue, redirect to microsoft/vscode-cmake-tools.
