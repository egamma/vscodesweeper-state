# Issue #311028 — Agents: Increase visibility of branch/worktree

[https://github.com/microsoft/vscode/issues/311028](https://github.com/microsoft/vscode/issues/311028)

## Summary

Team-authored UX request; must stay open for maintainer judgment. While recent work added a workspace folder pill to the session header (with branch info in its hover), the core request — making branch/worktree persistently and prominently visible at all times — remains unaddressed. The branch name is still only accessible via hover, not displayed inline.

## Change summary

Display the active branch and worktree prominently and persistently in the agents window so users always know which branch they are on and where merges will land.

## Best solution

Extend the session header meta row (likely in `src/vs/sessions/contrib/files/browser/workspaceFolderActions.ts` or a new branch pill action) to show the branch name inline — either as part of the workspace pill label or as a separate `$(git-branch) <name>` pill — so it remains visible without hovering. Additionally, surface merge-target information before commit/merge approval actions.
