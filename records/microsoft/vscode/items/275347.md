# Issue #275347 — Chat: Insert into Terminal should not auto trigger Suggestion

[https://github.com/microsoft/vscode/issues/275347](https://github.com/microsoft/vscode/issues/275347)

## Summary

Still reproduces on main: the terminal suggest addon's `isPasting` guard only covers clipboard paste (via `clipboardContrib.onWillPaste`), not programmatic text insertion from chat's 'Insert into Terminal' action which uses `terminal.runCommand` → `sendText`. The `_sync` method triggers quick suggestions when the prompt input model changes, regardless of whether the input came from a user keystroke or programmatic insertion.

## Change summary

Inserting chat-suggested terminal commands via 'Insert into Terminal' auto-triggers the terminal suggest widget, obscuring the inserted text.

## Best solution

Extend the `isPasting` suppression in the suggest addon to also cover programmatic `sendText`/`runCommand` calls — either by signaling from `terminalInstance.runCommand` that a programmatic insertion is in progress, or by checking in `_sync` whether the prompt input changed without a corresponding `onKey` event (no recent `_lastUserDataTimestamp` update). The fix likely touches `src/vs/workbench/contrib/terminalContrib/suggest/browser/terminalSuggestAddon.ts` and possibly `terminal.suggest.contribution.ts`.
