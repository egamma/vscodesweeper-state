# Issue #70614 — [html] embedded java script grammar doesn't end because of missing semicolon

[https://github.com/microsoft/vscode/issues/70614](https://github.com/microsoft/vscode/issues/70614)

## Summary

Keep open as the canonical tracker for an HTML embedded-JavaScript TextMate grammar limitation. Maintainer discussion points to upstream textmate/html.tmbundle behavior rather than a simple VS Code-side duplicate or completed fix.

## Change summary

Missing semicolons in inline HTML `<script>` blocks can leave following HTML treated as JavaScript, causing Toggle Line Comment to insert `//` outside the script tag.

## Best solution

Keep this as the canonical VS Code tracker while either following https://github.com/textmate/html.tmbundle/issues/85 or deciding to stop relying on TextMate grammar scopes for HTML embedded range detection. The likely ownership remains the HTML grammar/tokenization path, not the comment command itself.
