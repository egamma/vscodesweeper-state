# Issue #70614 — [html] embedded java script grammar doesn't end because of missing semicolon

[https://github.com/microsoft/vscode/issues/70614](https://github.com/microsoft/vscode/issues/70614)

## Summary

This is a known TextMate grammar limitation where the embedded JS grammar doesn't detect `</script>` as its boundary when a missing semicolon causes `<` to be parsed as a JS operator. The root cause is in the TextMate engine/grammar design, not in VS Code-specific code.

## Change summary

When a semicolon is omitted in a `<script>` tag, the HTML grammar fails to end the JS embedded region, causing subsequent HTML to be treated as JavaScript.

## Best solution

This requires either an upstream fix in the TextMate HTML grammar (textmate/html.tmbundle#85) or switching away from TextMate for detecting embedded language ranges. The Tree-sitter integration work may eventually resolve this class of issues. Keep tracking as upstream/grammar limitation.
