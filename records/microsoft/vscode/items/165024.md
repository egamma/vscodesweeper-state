# Issue #165024 — Keybindings for side-by-side diff editor arrow functions

[https://github.com/microsoft/vscode/issues/165024](https://github.com/microsoft/vscode/issues/165024)

## Summary

Implemented on main: the `diffEditor.revert` command now works from the keyboard (using cursor position or selection to determine which hunk to revert), which is exactly the keybinding-driven revert workflow requested here.

## Change summary

Request for a keyboard-bindable command to revert diff hunks without using the mouse arrow button.

## Best solution

Keep the shipped `diffEditor.revert` keyboard support from commit 6efb3110dee; no further change needed. Users can bind it to any key and combine with next/previous change navigation for a fully keyboard-driven workflow.
