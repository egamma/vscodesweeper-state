# Issue #311001 — Agent Plugins: Add support for "userConfig" to inject inputs into MCP configs.

[https://github.com/microsoft/vscode/issues/311001](https://github.com/microsoft/vscode/issues/311001)

## Summary

Valid feature request: agent plugins that define MCP servers with `userConfig` in their plugin.json cannot prompt users for configurable inputs (API keys, endpoints) because VS Code's plugin MCP discovery doesn't map `userConfig` entries to its existing input/variable-replacement infrastructure. The gap is confirmed on current main.

## Change summary

Support the `userConfig` object from plugin.json so that MCP servers contributed by agent plugins can prompt users for configurable inputs (e.g. API keys) via the `${input:...}` syntax.

## Best solution

Extend `PluginMcpDiscovery` (in `src/vs/workbench/contrib/mcp/common/discovery/pluginMcpDiscovery.ts`) and the plugin manifest parsing (`src/vs/workbench/contrib/chat/common/plugins/agentPluginServiceImpl.ts`) to read `userConfig` entries, register them as input definitions in the MCP registry's input storage, and expose them through the existing variable-replacement flow so users are prompted when the server starts.
