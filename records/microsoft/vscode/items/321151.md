# Issue #321151 — Make extensions trusted publishers configurable, to optionally include Microsoft, GitHub and Open AI in delayed auto-update

[https://github.com/microsoft/vscode/issues/321151](https://github.com/microsoft/vscode/issues/321151)

## Summary

Valid feature request with 30 upvotes, promoted to Backlog. The auto-update delay bypass for product-level trusted publishers (Microsoft, GitHub, OpenAI) is hardcoded in `isFromTrustedPublisher` and not configurable by users — the gap is real and still present on main.

## Change summary

Allow users to configure which publishers bypass the extension auto-update delay, rather than hardcoding Microsoft, GitHub, and OpenAI as always-trusted.

## Best solution

Extend `isFromTrustedPublisher` in `extensionsWorkbenchService.ts` to check whether the user has opted out of specific product-level trusted publishers (e.g. via a setting or by integrating with the existing user-trusted-publishers storage). The existing 'Manage Trusted Extension Publishers' command could be augmented to also show/manage the product-level publishers for the auto-update delay context.
