# Issue #305994 — execution_subagent doesn't have skill context

[https://github.com/microsoft/vscode/issues/305994](https://github.com/microsoft/vscode/issues/305994)

## Summary

Team-authored issue — must keep open. The execution_subagent still does not receive skill context on current main; no fix has landed. A community member confirms real-world impact.

## Change summary

The execution_subagent tool spawns a subagent without passing skill/instruction context, so the subagent cannot follow project-specific CLI guidance (e.g. test runner conventions).

## Best solution

Bring `ComputeAutomaticInstructions` (or equivalent skill-loading logic) into the execution subagent's prompt assembly in `extensions/copilot/src/extension/prompt/node/executionSubagentToolCallingLoop.ts` and `extensions/copilot/src/extension/tools/node/executionSubagentTool.ts`, so the subagent inherits the same skill context the outer agent has. The linked issue https://github.com/microsoft/vscode/issues/297437 may be relevant for converting execution_subagent to a custom agent, which would get skills automatically.
