# Issue #305994 — execution_subagent doesn't have skill context

[https://github.com/microsoft/vscode/issues/305994](https://github.com/microsoft/vscode/issues/305994)

## Summary

Team-authored issue tracking a real gap: the execution_subagent tool doesn't receive skill context (project instructions for CLIs, test scripts, etc.), causing it to misuse tools. Still open with active discussion and a linked tracking issue.

## Change summary

The execution_subagent doesn't have access to skill/instruction context, so it fails to follow project-specific conventions (e.g. test runner usage).

## Best solution

Bring ComputeAutomaticInstructions into the extension so that when the execution_subagent is spawned it receives the same skill context the outer agent has. Per comment 3, this was already planned. Related tracking: https://github.com/microsoft/vscode/issues/297437.
