# Issue #305994 — execution_subagent doesn't have skill context

[https://github.com/microsoft/vscode/issues/305994](https://github.com/microsoft/vscode/issues/305994)

## Summary

Team-authored issue identifying a real gap: the execution_subagent tool doesn't receive skill context (project instructions for CLIs/test scripts), causing it to misuse tools. Discussion points toward bringing ComputeAutomaticInstructions into the extension. Keeping open per policy.

## Change summary

The execution_subagent tool spawns a sub-agent without passing along skill/instruction context, so it doesn't know how to properly use project-specific CLIs.

## Best solution

Pass computed automatic instructions (skills) into the execution_subagent's spawned context, likely by integrating ComputeAutomaticInstructions into the extension as discussed in comments. Related tracking at https://github.com/microsoft/vscode/issues/297437.
