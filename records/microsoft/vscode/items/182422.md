# Issue #182422 — Not able to fully select an Arabic text inside quotation 

[https://github.com/microsoft/vscode/issues/182422](https://github.com/microsoft/vscode/issues/182422)

## Summary

Keep open as an editor-core RTL selection bug. The issue has recent user confirmations, a maintainer only linked a related issue rather than a duplicate, and current main still contains code that explicitly avoids the string-content selection path for RTL text.

## Change summary

Mouse selection of Arabic text inside quoted JavaScript strings is difficult and can miss the final character.

## Best solution

Editor-core should investigate RTL/Bidi hit-testing and selection behavior around quoted strings, likely in the editor view mouse selection path such as src/vs/editor/browser/view/viewController.ts and related mouse target/view rendering code. If https://github.com/microsoft/vscode/issues/185537 is the canonical RTL selection tracker, this issue can later be folded into it, but the provided evidence only marks it as related.
