# Issue #312815 — Terminal color changes (via OSC 10/11/12 sequences) do not persist correctly

[https://github.com/microsoft/vscode/issues/312815](https://github.com/microsoft/vscode/issues/312815)

## Summary

Still reproduces on main: _updateTheme() unconditionally overwrites the terminal theme without preserving OSC 10/11/12-modified colors. An open PR from the reporter addresses this.

## Change summary

Terminal colors set via OSC 10/11/12 escape sequences are lost when switching tabs in the editor area or changing the VS Code color theme.

## Best solution

Review and land (or iterate on) the open PR https://github.com/microsoft/vscode/pull/312856, which preserves OSC-modified colors across theme refreshes by comparing current colors against restore values before applying the new theme.
