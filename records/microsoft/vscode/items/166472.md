# Issue #166472 — [Accessibility] Add an option to allow Alt+F5 to jump to the next --word-diff instead of the whole line

[https://github.com/microsoft/vscode/issues/166472](https://github.com/microsoft/vscode/issues/166472)

## Summary

Valid accessibility feature request with active team interest (meganrogge, hediet engaged in comments). Not yet implemented on main — the accessible diff viewer still navigates by line-level groups only, with no word-level granularity. Keeping open in Backlog as the team has explicitly accepted it.

## Change summary

Allow Alt+F5 in the diff editor to navigate to individual word-level (inline) changes rather than jumping by whole changed lines, so screen reader users can quickly identify exact modifications.

## Best solution

Extend the AccessibleDiffViewer in `src/vs/editor/browser/widget/diffEditor/components/accessibleDiffViewer.ts` to support word-level navigation within diff groups, optionally gated by a setting. The diff model already computes inner/character changes — surface those as navigable stops and announce insertion/deletion/modification type on hover or focus.
