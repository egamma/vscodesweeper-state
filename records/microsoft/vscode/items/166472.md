# Issue #166472 — [Accessibility] Add an option to allow Alt+F5 to jump to the next --word-diff instead of the whole line

[https://github.com/microsoft/vscode/issues/166472](https://github.com/microsoft/vscode/issues/166472)

## Summary

Valid accessibility feature request with active maintainer engagement (meganrogge and hediet discussed implementation details as recently as Jan 2024). Belongs in Backlog awaiting planning and implementation.

## Change summary

Allow Alt+F5 diff navigation to land on individual word-level changes rather than whole lines, so screen reader users can identify exact modifications without reading entire lines.

## Best solution

Implement word-level diff navigation in the diff editor's accessible diff viewer (likely in `src/vs/editor/browser/widget/diffEditor/`). When a setting is enabled, Alt+F5 should cycle through individual word-diff hunks within changed lines. The hover should also indicate whether each change is an insertion, deletion, or modification, as discussed by hediet.
