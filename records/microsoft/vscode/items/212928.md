[https://github.com/microsoft/vscode/issues/212928](https://github.com/microsoft/vscode/issues/212928)

## Summary

Team-authored feature request still open on main — the Share submenu in tab context menu is explicitly disabled for multiselect via `MultipleEditorsSelectedInGroupContext.negate()`. Keeping open per hard keep-open rule (MEMBER author).

## Change summary

Enable the 'Share' context menu action when multiple editor tabs are selected.

## Best solution

Remove the `MultipleEditorsSelectedInGroupContext.negate()` guard on the Share submenu in `src/vs/workbench/browser/parts/editor/editor.contribution.ts` (line 408) and ensure share contributions can handle multiple editor resources. This may also allow moving the contribution back down from the workbench layer as noted in the issue.
