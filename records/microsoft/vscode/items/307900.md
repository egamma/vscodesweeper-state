# Issue #307900 — Copilot Chat fails when image file has .jpeg extension but actual content is PNG

[https://github.com/microsoft/vscode/issues/307900](https://github.com/microsoft/vscode/issues/307900)

## Summary

Verified duplicate of #311732 — both report MIME-type mismatch when image file content doesn't match its declared MIME type (derived from extension). The canonical issue is broader (covers MCP tool results and conversation poisoning) and already has an assignee.

## Change summary

Copilot Chat sends image/jpeg MIME type based on file extension even when the actual file bytes are PNG, causing a 400 error from the backend.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/311732 — implementing content-based MIME detection (magic-byte sniffing) there fixes both the attachment and MCP-tool code paths. The existing `readImageDimensions` in `src/vs/base/common/image.ts` already parses magic bytes for dimensions and could be extended to also return the detected MIME type, which `chatImageExtraction.ts` and the attachment resolve service would then use instead of trusting the file extension.

## Evidence

- **root cause traced in source:** In `src/vs/workbench/contrib/chat/browser/attachments/chatAttachmentResolveService.ts` line 188, `getMimeTypeFromPath` derives MIME type purely from the file extension via `CHAT_ATTACHABLE_IMAGE_MIME_TYPES` (defined in `src/vs/workbench/contrib/chat/common/model/chatModel.ts` line 94). No content sniffing is performed.
- **MIME type passed unchecked to API:** In `extensions/copilot/src/extension/byok/common/anthropicMessageConverter.ts` line 58, the `part.mimeType` is cast directly to `media_type` and sent to Anthropic, which validates actual bytes against it.
- **canonical issue:** https://github.com/microsoft/vscode/issues/311732 is open, assigned to sandy081, and covers the same MIME-mismatch root cause from a broader angle (MCP tools + conversation poisoning). Both request content-based MIME detection.
- **existing magic-byte infrastructure:** `src/vs/base/common/image.ts` already has `readImageDimensions` with magic-byte detection for JPEG/PNG/GIF/WebP, and `extensions/copilot/src/util/common/imageUtils.ts` has `getMimeType` that sniffs base64 prefixes — neither is wired into the attachment MIME resolution.
