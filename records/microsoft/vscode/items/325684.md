# Issue #325684 — Agent Sessions: clicking inline tool-result images opens sidecar tab instead of Image Carousel

[https://github.com/microsoft/vscode/issues/325684](https://github.com/microsoft/vscode/issues/325684)

## Summary

Confirmed gap: the Agent Sessions window imports the image carousel contribution but its chat rendering path doesn't wire tool-result image clicks through chatInlineAnchorWidget's carousel logic. A team member acknowledged the report. No fix on main yet.

## Change summary

Clicking inline tool-result images in the Agent Sessions window opens a plain editor tab instead of the Image Carousel, unlike the classic Copilot Chat panel.

## Best solution

Wire the Agent Sessions chat rendering path to use the same chatImageCarouselService click handler that chatInlineAnchorWidget.ts uses (line ~312) when images from tool invocations are rendered, so clicking opens the carousel when imageCarousel.chat.enabled is true.
