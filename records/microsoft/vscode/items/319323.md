# Issue #319323 — BidirectionalMap.set() and delete() cannot handle undefined as a stored value

[https://github.com/microsoft/vscode/issues/319323](https://github.com/microsoft/vscode/issues/319323)

## Summary

Confirmed on current main: BidirectionalMap.delete() and set() use `=== undefined` as a sentinel, which breaks when undefined is a legitimate stored value. Additionally, set() never cleans stale reverse-map entries regardless of value type. Real but niche — current callers store strings, not undefined.

## Change summary

BidirectionalMap.set() leaks stale reverse mappings, delete() fails for undefined-valued entries, and getOrSet() overwrites existing undefined values.

## Best solution

Replace the `=== undefined` sentinel checks with `Map.has()` in BidirectionalMap.set(), BidirectionalMap.delete(), and getOrSet() in src/vs/base/common/map.ts. The set() method also needs to delete the old reverse-map entry before inserting the new one (this bug exists for all value types, not just undefined).
