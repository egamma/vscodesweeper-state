# Issue #248862 — chat.implicitContext.enabled setting needs more documentation

[https://github.com/microsoft/vscode/issues/248862](https://github.com/microsoft/vscode/issues/248862)

## Summary

The setting's value-side documentation was improved (enum + enumDescriptions for never/first/always), but the key-side (valid chat location keys: panel, terminal, notebook, editor) remains undocumented — the original ask is partially addressed but a real gap remains. Team discussion about removing the setting didn't reach a conclusion.

## Change summary

The `chat.implicitContext.enabled` setting is an object type but doesn't document valid keys (chat locations), making it hard for users to discover what to configure.

## Best solution

Either add a `propertyNames` constraint with an enum of the valid location keys (`panel`, `terminal`, `notebook`, `editor`) to the JSON Schema registration in `src/vs/workbench/contrib/chat/browser/chat.shared.contribution.ts`, or — as team members discussed — decide whether to remove this experimental setting entirely. The schema fix is small and localized.
