# Issue #323997 — Bug: The Explorer cannot paste

[https://github.com/microsoft/vscode/issues/323997](https://github.com/microsoft/vscode/issues/323997)

## Summary

Real bug confirmed in current source: NativeClipboardService.readResources() only reads VS Code's internal 'code/file-list' clipboard format, so files copied in the OS file manager are invisible to the Explorer paste command. An earlier fix (PR #320685) was reverted on June 29 (commit 26d36469b04), reintroducing the gap. An open community PR (#324610) attempts to re-fix it. Keeping open since the fix PR has not merged.

## Change summary

Pasting files copied from the OS file manager (Finder/Windows Explorer) into the VS Code Explorer does nothing — only VS Code-internal copies are recognized.

## Best solution

Land a fix in `src/vs/workbench/services/clipboard/electron-browser/clipboardService.ts` that falls back to OS clipboard file formats (platform-specific: `public.file-url` on macOS, `FileNameW` on Windows, `text/uri-list` on Linux) when the internal `code/file-list` buffer is empty. Review and iterate on the open PR https://github.com/microsoft/vscode/pull/324610 which implements this approach.
