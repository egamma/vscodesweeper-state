# Issue #161242 — Dead debug state

[https://github.com/microsoft/vscode/issues/161242](https://github.com/microsoft/vscode/issues/161242)

## Summary

Still a valid bug — the debug UI can get stuck in a debugging state when the extension host crashes or the debuggee is killed externally. A team member confirmed a reliable repro (kill the extension host), and the cleanup code still has a known gap (`TODO@roblourens` at `session.dispose()` in `debugService.ts:808`). No evidence of a targeted fix on main.

## Change summary

Debug UI remains in an active-debugging state after the debuggee is killed externally or the extension host crashes, leaving a "dead" debug session.

## Best solution

Investigate the cleanup chain when the extension host crashes: `MainThreadDebugService` fires adapter errors on dispose (line 82–86 of `mainThreadDebugService.ts`), which should cascade through `RawDebugSession.shutdown` → `DebugSession.onDidEndAdapter` → `DebugService.registerSessionListeners`. A race or missed path likely leaves the session in the model without firing `onDidEndAdapter`. The `TODO@roblourens` at `debugService.ts:808` (`session.dispose()`) suggests known incomplete cleanup. The related open issue https://github.com/microsoft/vscode/issues/305208 describes a similar stuck-RUNNING symptom in a different scenario.
