# Issue #224863 — Search highlight not cleared sometimes

[https://github.com/microsoft/vscode/issues/224863](https://github.com/microsoft/vscode/issues/224863)

## Summary

Confirmed bug (acknowledged by maintainer) on the 'On Deck' milestone — search highlights persist when a regex becomes invalid mid-typing due to a mismatch between V8's regex engine (used for open files) and ripgrep's engine (used for file search). Hard keep-open: assigned scheduling milestone.

## Change summary

Search view highlights are not cleared when a regex transitions from valid to invalid during incremental typing.

## Best solution

Fix the search highlight clearing logic so that when the regex engine returns an error (e.g. ripgrep rejects `[^]`), any stale decorations from the previous valid-regex pass are removed. Likely in the search result refresh/decoration path in `src/vs/workbench/contrib/search/browser/`.
