# Issue #224863 — Search highlight not cleared sometimes

[https://github.com/microsoft/vscode/issues/224863](https://github.com/microsoft/vscode/issues/224863)

## Summary

Confirmed bug acknowledged by a team member (andreamah) — search highlights persist when a regex transitions from valid (V8 engine) to invalid (ripgrep engine) during typing. No fix has landed on main; the issue remains valid.

## Change summary

Search result highlights in open editors are not cleared when an incrementally-typed regex becomes invalid due to a mismatch between V8's and ripgrep's regex parsers.

## Best solution

When a search query returns a regex parse error from ripgrep, the search result model should clear existing in-editor highlight decorations. The fix likely belongs in the search view or search result model code that handles search completion/error, ensuring `toggleHighlights(false)` or equivalent is called on error paths. Files: `src/vs/workbench/contrib/search/browser/searchView.ts` and `src/vs/workbench/contrib/search/browser/searchTreeModel/textSearchHeading.ts`.
