# Issue #271268 — If you have logged into your Microsoft account, close VSCode and then reopen it - you will need to log in again.

[https://github.com/microsoft/vscode/issues/271268](https://github.com/microsoft/vscode/issues/271268)

## Summary

Confirmed upstream MSAL Runtime broker issue where refresh tokens are not being renewed properly on Windows. A team member is actively investigating with the MSAL Runtime team; the workaround (msal-no-broker) is known. Keep open pending upstream resolution.

## Change summary

Microsoft account refresh tokens expire prematurely when using the Windows authentication broker, requiring re-login after closing and reopening VS Code.

## Best solution

Resolution depends on the MSAL Runtime team fixing the broker's token renewal behavior on Windows. VS Code should track upstream progress and potentially update the bundled MSAL packages once a fix lands. The workaround (`"microsoft-authentication.implementation": "msal-no-broker"`) remains viable in the meantime.
