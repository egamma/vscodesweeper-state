# Issue #320369 — Unable to use certain fonts in the editor (Consolas) since the last update

[https://github.com/microsoft/vscode/issues/320369](https://github.com/microsoft/vscode/issues/320369)

## Summary

Upstream Chromium/Skia issue: the switch to the fontations bridge for fontconfig scanning (landed between Electron versions backing VS Code 1.122 and 1.123) likely breaks font resolution for specific Consolas .ttf files on Linux. A VS Code team member (deepak1556) is actively investigating with the reporter. Keep open as an upstream tracking issue.

## Change summary

Consolas font on Linux stopped resolving correctly in VS Code 1.123, falling back to Noto Sans Mono despite fontconfig matching correctly.

## Best solution

Root-cause and fix (or report) the fontations bridge regression in Chromium (https://chromium-review.googlesource.com/c/chromium/src/+/7232260). Once the Chromium fix ships, it will flow through Electron into VS Code. In the meantime, the reporter's workaround is to copy the font with a modified family name. deepak1556 is already comparing font metadata tables to isolate the mismatch.
