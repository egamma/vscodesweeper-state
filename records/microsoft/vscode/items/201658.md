# Issue #201658 — Laggy rendering due to NVidia "Background Application Max Framerate"

[https://github.com/microsoft/vscode/issues/201658](https://github.com/microsoft/vscode/issues/201658)

## Summary

Confirmed upstream issue: NVidia drivers classify VS Code (Electron) as a background application and throttle its framerate. The root cause is in NVidia's driver behavior and/or Chromium/Electron's window classification. Already correctly labeled as upstream. A user-side workaround exists (per-app NVidia profile override). No VS Code code change can fix this.

## Change summary

NVidia 'Background Application Max Framerate' setting throttles VS Code's rendering because Electron windows are classified as background applications.

## Best solution

This lives in NVidia's driver heuristics and Chromium/Electron's window classification. The practical path is documenting the workaround (create a per-application NVidia profile for VS Code with that setting disabled) or filing upstream on Electron/Chromium to ensure their windows are properly classified as foreground applications. No VS Code core change addresses this.
