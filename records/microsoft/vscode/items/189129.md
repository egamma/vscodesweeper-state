# Issue #189129 — [themes] Theme is getting switched on switching profiles

[https://github.com/microsoft/vscode/issues/189129](https://github.com/microsoft/vscode/issues/189129)

## Summary

Team-authored bug report (sandy081/MEMBER) about theme resetting when switching profiles after making a theme setting apply to all profiles. Hard keep-open per policy.

## Change summary

Theme resets to default dark when switching to a profile whose custom theme extension gets removed/unloaded during the profile switch.

## Best solution

The theme service should gracefully handle the case where a profile's configured theme extension is temporarily unavailable during a profile switch (or when a setting is promoted to apply across all profiles), rather than falling back to the default dark theme. Likely involves `src/vs/workbench/services/themes/browser/workbenchThemeService.ts` and its interaction with the profile/extension lifecycle.
