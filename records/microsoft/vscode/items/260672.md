# Issue #260672 — Instructions: Allow filtering instructions to specific modes

[https://github.com/microsoft/vscode/issues/260672](https://github.com/microsoft/vscode/issues/260672)

## Summary

Team-authored feature request. The backend infrastructure for session-type filtering exists (`matchesSessionType`, `sessionTypes` on `IPromptPathBase`), but there is no front-matter attribute in `.instructions.md` files that lets workspace authors filter instructions to specific modes. The gap remains.

## Change summary

Allow `.instructions.md` files to declare a `mode` (or similar) front-matter field so workspace authors can scope instructions to specific modes like agent, code-review, etc.

## Best solution

Add a new `PromptHeaderAttributes` entry (e.g. `sessionTypes` or `applyToMode`) in `promptFileParser.ts`, expose it as a parsed header getter, and wire it into the `IPromptPath.sessionTypes` field during instruction discovery in `promptsServiceImpl.ts` so that workspace-authored `.instructions.md` files can filter by mode without needing the extension API.
