# Issue #259467 — MCP code exchange request not going through due to proxy

[https://github.com/microsoft/vscode/issues/259467](https://github.com/microsoft/vscode/issues/259467)

## Summary

Team-authored bug, still open. The MCP OAuth token exchange in extHostAuthentication.ts uses the global Node.js `fetch` rather than a proxy-aware fetch, causing failures for users behind reverse proxies. The code path is unchanged on current main. Keeping open per hard keep-open rule (MEMBER author) and because the underlying issue persists.

## Change summary

MCP OAuth code exchange request to the token endpoint fails when the MCP server is behind a reverse proxy (e.g. traefik), because the fetch call in extHostAuthentication.ts does not go through VS Code's proxy support.

## Best solution

Align the `fetch` calls in `McpDynamicAuthProvider.exchangeCodeForToken` and `exchangeRefreshTokenForToken` (src/vs/workbench/api/common/extHostAuthentication.ts, lines ~776 and ~826) with how `github-authentication` handles fetch — using a proxy-aware fetch implementation instead of the bare global `fetch`. The `network` area owner should decide whether Electron fetch or the extension host's proxy-patched fetch is the right path.
