# Issue #240023 — Indent/unindent reformats code when I don't want it to

[https://github.com/microsoft/vscode/issues/240023](https://github.com/microsoft/vscode/issues/240023)

## Summary

This is a real bug in the ShiftCommand's `useTabStops` path: it aligns each line to its own nearest tab stop rather than uniformly shifting all lines by the same amount, breaking continuation-line alignment. The issue is already assigned to aiday-mar and labeled as a tree-sitter autoindent candidate — it should remain open.

## Change summary

Indent/unindent (Tab/Shift+Tab) with selection reformats continuation lines by aligning each line to its own tab stop instead of uniformly shifting all lines.

## Best solution

Fix the `useTabStops` path in `src/vs/editor/common/commands/shiftCommand.ts` (lines 114–196) to uniformly shift all selected lines by the same amount (one indentSize) while preserving relative alignment between lines, rather than independently computing each line's next/prev tab stop. The tree-sitter autoindent work may provide a better heuristic for distinguishing structural indentation from continuation alignment.
