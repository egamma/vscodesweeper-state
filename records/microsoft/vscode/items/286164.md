# Issue #286164 — Remote host extension repeatedly restarting

[https://github.com/microsoft/vscode/issues/286164](https://github.com/microsoft/vscode/issues/286164)

## Summary

The crash originates in the GitHub Copilot Chat extension (v0.35.2/0.35.3), which accesses the global `navigator` object in the Node.js extension host. VS Code intentionally intercepts this access and throws a `PendingMigrationError` to guide extensions through the Node.js v21+ migration. The fix belongs in the Copilot Chat extension, not in VS Code core.

## Change summary

Remote extension host repeatedly crashes when GitHub Copilot Chat extension accesses `navigator` global in Node.js, triggering VS Code's migration guard.

## Best solution

The Copilot Chat extension needs to stop relying on `navigator` for environment detection in the Node.js extension host. The reporter should file this against the `microsoft/vscode-copilot-release` repository or update to a newer Copilot Chat version that likely addresses this. VS Code's migration guard in `extensionHostProcess.ts` is working as intended.
