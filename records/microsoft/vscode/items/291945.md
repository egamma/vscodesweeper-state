# Issue #291945 — SCM: staged typechanges do not show up as staged

[https://github.com/microsoft/vscode/issues/291945](https://github.com/microsoft/vscode/issues/291945)

## Summary

Team-authored bug, confirmed in source: staged typechanges are silently dropped because the index status parser doesn't handle the 'T' status code. An open PR (#316008) addresses this. Keeping open per team-author rule and because the PR hasn't merged yet.

## Change summary

When a file's type changes (e.g. regular file → symlink) and is staged, it does not appear in the SCM staged changes group.

## Best solution

Land the open PR https://github.com/microsoft/vscode/pull/316008 which adds 'T' handling to the index status switch and introduces an INDEX_TYPE_CHANGED status enum value, mirroring the existing working-tree TYPE_CHANGED support.

## Evidence

- **missing case in parser:** extensions/git/src/repository.ts line 3071-3077: the switch on `raw.x` (index/staged status) handles M, A, D, R, C but not 'T' (typechange). The working-tree switch at line 3084 does handle 'T'.
- **no INDEX_TYPE_CHANGED enum:** extensions/git/src/api/git.constants.ts defines TYPE_CHANGED (11) for working tree but has no INDEX_TYPE_CHANGED counterpart, so staged typechanges have no representable status.
- **open fix PR:** https://github.com/microsoft/vscode/pull/316008 — 'Git: show staged typechanges in the index group' — is open and targets this exact issue.
- **related older issue:** https://github.com/microsoft/vscode/issues/91100 — 'Git Typechanges do not appear in source control panel' — is a broader/older issue also open with help-wanted; this issue is specifically about the staged (index) case.
