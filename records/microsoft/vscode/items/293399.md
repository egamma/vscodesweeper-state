# Issue #293399 — "Cannot launch a terminal process in an empty workspace" restriction is problematic

[https://github.com/microsoft/vscode/issues/293399](https://github.com/microsoft/vscode/issues/293399)

## Summary

Still present on main: the empty-workspace cwd check at terminalInstance.ts:1601 blocks legitimate extension-initiated terminal processes when no workspace is open and the cwd differs from userHome. A team member acknowledged the restriction should likely be removed.

## Change summary

Terminal process launch is blocked in an empty workspace when cwd differs from userHome, preventing extensions from running tasks or shells with a non-home cwd.

## Best solution

Remove or relax the empty-workspace cwd guard in `src/vs/workbench/contrib/terminal/browser/terminalInstance.ts` (lines 1601–1605). Extensions are inherently trusted, so blocking their terminal requests based on a cwd mismatch is overly restrictive. The area owner (@meganrogge) already agreed the line should likely be removed.
