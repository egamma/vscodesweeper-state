# Issue #221435 — can not show marker in python minimap

[https://github.com/microsoft/vscode/issues/221435](https://github.com/microsoft/vscode/issues/221435)

## Summary

Current main already supports minimap section headers for Python-style comments, including MARK comments and named regions, so this can be closed as completed rather than tracked as a new editor-minimap bug.

## Change summary

Python MARK or region comments should appear as section headers in the editor minimap.

## Best solution

Treat the existing minimap section-header implementation as the shipped solution: commit c800bf9216ee64bdd87db41b6e6101df1b89e897 added the feature for https://github.com/microsoft/vscode/issues/74843, and the current Python language configuration provides the comment and region metadata it needs.

  > Thanks for the report. This appears to be covered by the minimap section-header implementation from commit c800bf9216ee64bdd87db41b6e6101df1b89e897, which implemented https://github.com/microsoft/vscode/issues/74843. Current main has `editor.minimap.showMarkSectionHeaders` and `editor.minimap.showRegionSectionHeaders` enabled by default, and Python contributes `#` comments plus `# region` / `# endregion` folding markers, so `# MARK:` and named regions should render in the minimap. I believe this can be closed as completed; please open a new issue with exact Python text and version details if it still fails on the latest build.

## Evidence

- **fix commit:** Commit c800bf9216ee64bdd87db41b6e6101df1b89e897 is titled "Introduce minimap section headers, a la Xcode (#190759)" and its commit message says it adds minimap section headers for https://github.com/microsoft/vscode/issues/74843.
- **current settings:** src/vs/editor/common/config/editorOptions.ts defines editor.minimap.showRegionSectionHeaders and editor.minimap.showMarkSectionHeaders as enabled by default, with the default MARK parser regex '\\bMARK:\\s*(?<separator>\-?)\\s*(?<label>.*)$'.
- **current implementation:** src/vs/editor/contrib/sectionHeaders/browser/sectionHeaders.ts creates minimap decorations with MinimapPosition.Inline and sectionHeaderText, after validating MARK headers are inside comments.
- **python support:** extensions/python/language-configuration.json defines '#' as the Python line comment and folding markers '^\\s*#\\s*region\\b' / '^\\s*#\\s*endregion\\b', which are the inputs used by the section-header detector.
