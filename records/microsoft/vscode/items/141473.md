# Issue #141473 — Auto save behaviour might be unexpected here

[https://github.com/microsoft/vscode/issues/141473](https://github.com/microsoft/vscode/issues/141473)

## Summary

Team-authored issue with 'On Deck' milestone — keep open. The infrastructure to disable auto-save per editor exists (disableAutoSave API) and is used by chat editing, but the debug memory editor does not use it yet.

## Change summary

Auto save should be disabled when editing memory values during a debug session.

## Best solution

Call `disableAutoSave` from `IFilesConfigurationService` in the debug memory editor contribution so that memory edits don't trigger auto-save. The pattern already exists in `src/vs/workbench/contrib/chat/browser/chatEditing/chatEditingModifiedFileEntry.ts`.
