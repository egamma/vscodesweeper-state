# Issue #314264 — chat.tools.terminal.autoApproveWorkspaceNpmScripts does not auto-approve npm scripts defined in workspace subfolders

[https://github.com/microsoft/vscode/issues/314264](https://github.com/microsoft/vscode/issues/314264)

## Summary

Confirmed on main: the npm auto-approver only checks `package.json` at the exact CWD, never traversing up the directory tree to find the nearest `package.json` the way npm itself does. Monorepo subfolder scripts are never auto-approved.

## Change summary

The npm script auto-approver fails to auto-approve scripts defined in subfolder package.json files in monorepo workspaces.

## Best solution

Extend `_getPackageJsonScripts` in `npmScriptAutoApprover.ts` to walk up the directory tree (bounded by the workspace root) looking for the nearest `package.json` containing the requested script, mirroring npm's own resolution behavior. Also consider parsing `cd <dir> &&` prefixes in the command to resolve the effective CWD before checking scripts.
