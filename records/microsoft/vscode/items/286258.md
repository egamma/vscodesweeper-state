# Issue #286258 — Opening a Worktree should still have a mention of the workspace in the window title

[https://github.com/microsoft/vscode/issues/286258](https://github.com/microsoft/vscode/issues/286258)

## Summary

Team-authored issue requesting that the window title surface the parent workspace name when a git worktree is opened. No worktree awareness exists in the window title or label service on current main; the request is valid and unaddressed.

## Change summary

When opening a git worktree in a new window, the window title shows only the worktree folder name with no reference to the parent workspace.

## Best solution

Teach `getWorkspaceLabel` (or the `rootName` variable in `windowTitle.ts`) to detect when the opened folder is a git worktree and append or incorporate the parent repository/workspace name — e.g. `q_master [ai-editor]`. The git service already knows about worktrees; the label service and title template logic in `src/vs/workbench/browser/parts/titlebar/windowTitle.ts` and `src/vs/workbench/services/label/common/labelService.ts` would need to consume that information.
