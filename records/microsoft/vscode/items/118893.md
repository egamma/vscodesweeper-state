# Issue #118893 — Using key commands to scroll to top or bottom of document does not incorporate editor.paddingTop and editor.paddingButtom

[https://github.com/microsoft/vscode/issues/118893](https://github.com/microsoft/vscode/issues/118893)

## Summary

Keep open: the current editor command/reveal path still appears to use minimal cursor reveal rather than explicitly honoring configured top/bottom editor padding for cursorTop/cursorBottom. There is not enough evidence for a duplicate or completed-fix close.

## Change summary

Keyboard commands that move to the start or end of the buffer should preserve the configured editor top/bottom padding in the final scroll position.

## Evidence

- **current command path:** src/vs/editor/browser/coreCommands.ts defines cursorTop/cursorBottom and both call viewModel.revealAllCursors(args.source, true) with minimal reveal after moving the cursor.
- **current reveal behavior:** src/vs/editor/browser/viewParts/viewLines/viewLines.ts:693 computes reveal scroll positions from the cursor/range box and cursorSurroundingLines; the minimalReveal path ignores scroll-off padding and does not explicitly account for editor.paddingTop/editor.paddingBottom as requested.
