# Issue #118893 — Using key commands to scroll to top or bottom of document does not incorporate editor.paddingTop and editor.paddingButtom

[https://github.com/microsoft/vscode/issues/118893](https://github.com/microsoft/vscode/issues/118893)

## Summary

Still reproduces on current main: the reveal-cursor logic scrolls to make line 1 (or the last line) visible but never scrolls further to expose the editor padding region above/below it.

## Change summary

Keyboard commands to go to document top/bottom (Cmd+Up/Down) ignore `editor.padding.top` and `editor.padding.bottom`, pinning the first/last line at the viewport edge instead of showing the configured padding.

## Best solution

In `_computeScrollTopToRevealRange` (src/vs/editor/browser/viewParts/viewLines/viewLines.ts), when the target is line 1, expand `boxStartY` to 0 (include the top padding region); when the target is the last line, expand `boxEndY` to include the bottom padding. This ensures keyboard-driven reveals match mouse-scroll behavior at document boundaries.
