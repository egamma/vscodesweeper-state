# Issue #118893 — Using key commands to scroll to top or bottom of document does not incorporate editor.paddingTop and editor.paddingButtom

[https://github.com/microsoft/vscode/issues/118893](https://github.com/microsoft/vscode/issues/118893)

## Summary

Keep open: the report has clear reproduction steps and current main still routes `cursorTop`/`cursorBottom` through the generic minimal cursor reveal path rather than explicitly preserving configured editor top/bottom padding.

## Change summary

Keyboard commands for jumping to the start or end of a file ignore `editor.padding.top` and `editor.padding.bottom` even though mouse scrolling can show that padding.

## Best solution

Handle document-boundary cursor reveal in the editor command/reveal pipeline so `cursorTop` and `cursorBottom` land at the maximum scroll positions that preserve configured editor padding, with focused coverage in `src/vs/editor/browser/coreCommands.ts` or `src/vs/editor/browser/viewParts/viewLines/viewLines.ts`.
