# Issue #234393 — Codes Crashes after about 15 Minutes

[https://github.com/microsoft/vscode/issues/234393](https://github.com/microsoft/vscode/issues/234393)

## Summary

Confirmed OOM crash from unbounded PTY output buffering in the main process. An open PR (#323980) directly fixes this issue — keep open until it merges.

## Change summary

VS Code main process OOMs after ~15 minutes when terminal produces heavy output, because `ProxyChannel.fromService` eagerly buffers `onProcessData` events that never get consumed.

## Best solution

Land https://github.com/microsoft/vscode/pull/323980 which adds `disableEventBuffering` to the LocalPty channel registration, preventing unbounded accumulation of PTY data in the main process heap.
