# Issue #319379 — Agents view: session resume fails with "Error during execution" when no git repo

[https://github.com/microsoft/vscode/issues/319379](https://github.com/microsoft/vscode/issues/319379)

## Summary

Confirmed regression on current main: the `unknown:///` fallback URI at line 1289 of copilotChatSessionsProvider.ts is still present, and the open fix PR #319380 has not yet merged. Keeping open for that PR to land.

## Change summary

Resuming a CLI-created agent session in the Agents view fails with 'Error during execution' when the session has no associated git repository, because `unknown:///` is registered as a workspace folder and yields an invalid cwd.

## Best solution

Land PR https://github.com/microsoft/vscode/pull/319380, which filters out non-file-scheme workspace folders in `getFolderInfoForSession()` so that virtual/unknown URIs don't propagate as cwd to the Claude SDK.
