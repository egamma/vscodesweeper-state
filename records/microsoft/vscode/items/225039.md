# Issue #225039 — Context menu shows up on left click for deleted lines, not right click

[https://github.com/microsoft/vscode/issues/225039](https://github.com/microsoft/vscode/issues/225039)

## Summary

Still present on main: the inline diff deleted-code view zone has `suppressMouseDown: false` but no proper `contextmenu` event handler, so mouse interactions on deleted lines behave incorrectly (left click triggers context menu via the lightbulb path, right click does nothing).

## Change summary

In the inline diff editor, left-clicking deleted lines shows a context menu while right-clicking does not — the expected behavior is the reverse.

## Best solution

Add a `contextmenu` event listener on the deleted code DOM node (in `inlineDiffDeletedCodeMargin.ts` or the view zone setup in `diffEditorViewZones.ts`) that shows the diff actions menu, and remove or guard the `mousedown` handler so the lightbulb alone triggers on left click without also firing from left clicks on the deleted text.
