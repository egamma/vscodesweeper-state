# Issue #167146 — Softlink icon is not updated for folder

[https://github.com/microsoft/vscode/issues/167146](https://github.com/microsoft/vscode/issues/167146)

## Summary

Still reproduces on main: when a folder is deleted and replaced with a symlink of the same name, the explorer does not refresh the symlink decoration because the file-change handler skips ADDED events when the model already has a child with that name. An open PR (#168707) from the reporter exists but has not been merged.

## Change summary

Explorer does not update the symlink icon overlay when a regular folder is replaced by a symbolic link of the same name until the window is reloaded.

## Best solution

Review and land (or close with feedback) the existing PR https://github.com/microsoft/vscode/pull/168707, which addresses this. Multiple duplicate issues have been closed against this one, so the demand is real. The fix likely needs to ensure that UPDATED or replaced-in-place file events trigger a re-resolve of symlink status in the explorer model.
