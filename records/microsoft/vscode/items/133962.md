# Issue #133962 — Terminal toggle size to content width acts unexpectedly wrt viewport dimensions, scroll bar and padding

[https://github.com/microsoft/vscode/issues/133962](https://github.com/microsoft/vscode/issues/133962)

## Summary

The terminal toggle-size-to-content-width feature still has viewport dimension and padding issues on current main. The initial fix (9974f744e6a) and follow-up (fcdbfd08f0d) addressed some symptoms, but the core problem of the viewport scooting up and extra bottom padding when toggling remains, as evidenced by the author's continued reports and two related open issues (#235776, #235777).

## Change summary

Toggling terminal 'size to content width' repeatedly causes the viewport to shift up, extra bottom padding to appear, and scroll bar misbehavior.

## Best solution

The scrollbar and viewport dimension logic in `terminalInstance.ts` `_addScrollbar`/`_removeScrollbar`/`toggleSizeToContentWidth` needs to properly account for padding and viewport height when toggling. Consider consolidating this with the fixes needed for https://github.com/microsoft/vscode/issues/235776 and https://github.com/microsoft/vscode/issues/235777 since they share the same root cause area.
