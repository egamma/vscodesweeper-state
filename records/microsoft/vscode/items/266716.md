# Issue #266716 — Copilot OOM (Node.js)

[https://github.com/microsoft/vscode/issues/266716](https://github.com/microsoft/vscode/issues/266716)

## Summary

Confirmed OOM in the renderer when restoring a large ChatEditingSessions folder (264MB, 2.4k files). Root cause identified by a team member: chatEditingSessionStorage.ts loads all file contents into an in-memory map during restoreState, blowing the 4GB JS heap. No fix commit found on current main — the code still loads everything eagerly.

## Change summary

VS Code renderer crashes with OOM when Copilot agent mode attempts to restore a large ChatEditingSessions storage folder on window load.

## Best solution

Add size limits or lazy loading to `chatEditingSessionStorage.ts` restoreState, or implement automatic pruning of stale/large ChatEditingSessions folders so the storage doesn't grow unbounded. The fix likely belongs in `src/vs/workbench/contrib/chat/browser/chatEditing/chatEditingSessionStorage.ts` and possibly a companion cleanup mechanism in the service.
