# Issue #187284 — Unlock keyring showing up at startup

[https://github.com/microsoft/vscode/issues/187284](https://github.com/microsoft/vscode/issues/187284)

## Summary

Team-authored debt issue (MEMBER). The Settings Sync side was addressed by PR #192264 (lazy session fetch). The Cloud Changes / edit sessions side appears to also use lazy initialization now. Kept open per hard keep-open rule for team-authored issues — a maintainer should verify whether remaining eager keyring access paths exist and close if fully resolved.

## Change summary

VS Code eagerly reads from the OS keyring at startup (for Settings Sync and Cloud Changes enablement), triggering an 'Unlock keyring' dialog on Linux even when no auth-dependent features are in use.

## Best solution

A maintainer should audit whether any startup-path code still eagerly reads from the keyring (secret storage / authentication sessions) when Settings Sync and Cloud Changes are not enabled. If PR https://github.com/microsoft/vscode/pull/192264 (Settings Sync) and the existing lazy-init pattern in `editSessionsStorageService.ts` fully cover the issue, close it as completed. Otherwise, identify and defer the remaining eager access paths.
