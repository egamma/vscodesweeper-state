# Issue #316587 — Open file from editor without navigating into other scopes

[https://github.com/microsoft/vscode/issues/316587](https://github.com/microsoft/vscode/issues/316587)

## Summary

Valid feature request to detect and make clickable bare filesystem paths in editor content (like the terminal already does). No existing implementation on main; the editor's LinkComputer only handles URL-scheme links (http/https/file). Keeping open for community traction.

## Change summary

Detect bare filesystem paths in editor text and make them ctrl+clickable to open, similar to terminal link detection.

## Best solution

This would require a built-in or extension-provided LinkProvider that detects absolute/relative filesystem paths in editor content. Could potentially be an extension-candidate, but the terminal already does this in core, so parity in the editor is reasonable. Related to https://github.com/microsoft/vscode/issues/149153 (file link detection in output). Should be classified as a feature-request in the editor-links area.
