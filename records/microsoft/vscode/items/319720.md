# Issue #319720 — Agent window: Cannot select Auto for model

[https://github.com/microsoft/vscode/issues/319720](https://github.com/microsoft/vscode/issues/319720)

## Summary

The report lacks detail on which agent provider is being used — the Auto model option is intentionally per-provider (currently only the Copilot CLI harness declares `supportsAutoModel`). Without knowing which session type the reporter is in, it's unclear whether this is expected behavior or a missing capability. Requesting clarification.

## Change summary

User wants the 'Auto' model option available in the Agents window model picker, but it's absent for their session type.

## Best solution

Clarify which agent host provider the reporter is using. If it's Claude or Codex, Auto is intentionally unavailable because those harnesses don't declare `supportsAutoModel`. A broader feature request to enable Auto for more providers would touch `agentHostProviderSupportsAutoModel()` in `src/vs/workbench/contrib/chat/browser/agentSessions/agentHost/agentHostLanguageModelProvider.ts` — but this is gated by whether each agent host protocol actually supports model routing. The existing TODO in that file notes the intent to hoist this capability onto the agent host protocol itself.
