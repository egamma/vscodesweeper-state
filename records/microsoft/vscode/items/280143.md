# Issue #280143 — Give extension the ability to know where the current debug session is stopped

[https://github.com/microsoft/vscode/issues/280143](https://github.com/microsoft/vscode/issues/280143)

## Summary

Active discussion between the reporter and a team member about whether EvaluatableExpressionProvider should receive debugger stop context. The team member suggested workarounds (customRequest for stackTrace, or handling scope in the debug adapter), but the reporter raised valid concerns that these don't fit the PHP/Xdebug case. Current labels (under-discussion, info-needed) are appropriate.

## Change summary

Provide debug session stop location context to EvaluatableExpressionProvider so extensions can determine if a variable is in scope before evaluating it.

## Best solution

Keep under discussion. The team may decide to either (a) add optional debugger context to `provideEvaluatableExpression` similar to how `InlineValuesProvider` receives `InlineValuesContext`, or (b) confirm the current design stands and extensions should use `customRequest` to query DAP directly. The discussion needs a final product decision from the debug area owner.
