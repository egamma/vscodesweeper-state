# Issue #274859 — Unable to open multiple copies of vscode from windows explorer

[https://github.com/microsoft/vscode/issues/274859](https://github.com/microsoft/vscode/issues/274859)

## Summary

This is as-designed behavior rooted in a Windows security constraint: a non-elevated process cannot connect to the IPC pipe of an elevated (admin) VS Code instance, resulting in an EPERM error. VS Code correctly detects this and shows the warning dialog. The user's Explorer context is running non-elevated while an admin instance holds the pipe.

## Change summary

Opening files via Windows Explorer fails with 'another instance is already running as administrator' when an elevated VS Code instance is already running.

## Best solution

Keep the current behavior — this is a Windows OS security boundary (non-admin processes cannot connect to admin-owned named pipes). The workaround is to either run Explorer elevated, or close the admin VS Code instance before opening files from a non-admin Explorer. This was previously discussed and closed as out-of-scope in https://github.com/microsoft/vscode/issues/6560.

  > - Use the command line (`code -n <file>`) from an elevated terminal.
