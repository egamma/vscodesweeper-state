Keep open: this is a team-authored Backlog feature request in the editor folding area, so it needs maintainer judgment rather than cleanup closure. The issue tracks improving folding responsiveness while language-server folding data is not yet available.

## Change summary

Folding controls should remain useful while the language server is still starting instead of doing nothing until folding ranges arrive.

## Best solution

Editor folding should ideally fall back to an available local/indentation-based folding model until language-server ranges are ready, or the editor-folding owners should explicitly decide that the startup delay is acceptable.
