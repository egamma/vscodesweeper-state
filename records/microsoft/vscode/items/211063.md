# Issue #211063 — Verbose Hover: Hover jumps when there is not enough space

[https://github.com/microsoft/vscode/issues/211063](https://github.com/microsoft/vscode/issues/211063)

## Summary

The fix (PR #235851) was merged then immediately reverted due to a regression; no replacement fix has landed. The issue remains valid and open, marked as a feature request with 'help wanted'.

## Change summary

Verbose hover jumps from above to below when it grows beyond available space, disrupting the user experience when clicking ±  buttons.

## Best solution

Implement a solution in the content-hover widget (src/vs/editor/contrib/hover/browser/contentHoverWidget.ts and contentHoverWidgetWrapper.ts) that locks the hover's above/below placement once expanded, or caps growth to the available space. The reverted approach (always render on the same side) was correct in intent but caused regressions — a more careful version that avoids the regression is needed.
