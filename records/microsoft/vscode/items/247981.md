# Issue #247981 — Screenshot can include parts of VS Code

[https://github.com/microsoft/vscode/issues/247981](https://github.com/microsoft/vscode/issues/247981)

## Summary

Mid-verification issue (`verification-found`) filed by a team member — two hard keep-open rules apply. The merged PR (https://github.com/microsoft/vscode/pull/247993) addressed out-of-bounds clipping, but a commenter confirmed the screenshot can still capture VS Code UI elements (e.g. command palette) overlaying the target element, so the fix is incomplete.

## Change summary

Screenshots of UI elements can include parts of the VS Code chrome when overlapping UI (like the command palette) is visible.

## Best solution

The screenshot capture logic needs to either dismiss transient overlays before capture or mask/clip them out. The contributor comment indicates this requires signaling to close overlays before `webContents.capturePage` runs — likely in the screenshot contribution or the integrated browser's capture path. Remains open for the next iteration.
