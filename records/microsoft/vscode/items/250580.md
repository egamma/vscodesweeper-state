# Issue #250580 — Git - remember an added git repository that is outside the parent folder of current workspace

[https://github.com/microsoft/vscode/issues/250580](https://github.com/microsoft/vscode/issues/250580)

## Summary

Valid bug with an open community PR (https://github.com/microsoft/vscode/pull/269536) already targeting it. The git extension lacks persistence for repositories manually opened outside the workspace — they vanish on restart. Keeping open for PR review.

## Change summary

Repositories added via 'Git: Open Repository' that are outside the workspace folder are not remembered across VS Code sessions.

## Best solution

Review and land (or iterate on) the open PR https://github.com/microsoft/vscode/pull/269536, which introduces an `OpenedRepositoriesManager` in `extensions/git/src/model.ts` to persist externally-opened repos in workspace state and restore them on startup.
