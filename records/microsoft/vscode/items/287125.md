# Issue #287125 — No heading is defined on every copilot response: A11y_Visual Studio Code_Github Copilot_Screenreader

[https://github.com/microsoft/vscode/issues/287125](https://github.com/microsoft/vscode/issues/287125)

## Summary

Team-authored accessibility bug, still present on main. The h3 username heading in each chat response is hidden for Copilot responses, leaving no heading landmark for screen reader H-key navigation. Keeping open per team-author policy.

## Change summary

Screen reader users cannot use H / Shift+H heading navigation to jump between Copilot chat responses because the only heading element (h3.username) is hidden when the response comes from Copilot.

## Best solution

In the chat list renderer, ensure a visible (or at least screen-reader-accessible) heading element is present on every Copilot response even when the username is visually hidden. One approach: instead of toggling the h3 to `display:none`, use a visually-hidden (sr-only) CSS class so the heading remains in the accessibility tree, or add a separate aria heading role on the response container. The fix likely lives in `src/vs/workbench/contrib/chat/browser/widget/chatListRenderer.ts` (around lines 649 and 920–922) and the associated CSS in `chat.css`.
