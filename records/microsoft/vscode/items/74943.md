[https://github.com/microsoft/vscode/issues/74943](https://github.com/microsoft/vscode/issues/74943)

## Summary

Still valid feature request — no mechanism exists on current main to override or exclude extension-contributed JSON schema associations. User-configured schemas are merged alongside extension ones rather than replacing them.

## Change summary

Allow users to override or disable built-in/extension-contributed JSON schema associations for specific file patterns (e.g., package.json).

## Best solution

Add a priority or exclusion mechanism in the JSON language features client/server so that user-configured `json.schemas` entries can override extension-contributed `jsonValidation` associations for the same fileMatch pattern, or support a negation pattern (e.g., `!`) to exclude files from a schema.
