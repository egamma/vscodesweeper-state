# Issue #247375 — Add MCP Allow List to MDM Policy

[https://github.com/microsoft/vscode/issues/247375](https://github.com/microsoft/vscode/issues/247375)

## Summary

Valid enterprise feature request. Current main has a coarse MCP access policy (none/registry/all) but no granular per-server allow list by hostname or identifier as requested. The need remains unmet.

## Change summary

Add an MDM-manageable MCP server allow list so enterprise admins can restrict developers to specific approved MCP servers.

## Best solution

Extend the existing `chat.mcp.discovery.access` policy (in `src/vs/platform/mcp/common/mcpManagement.ts` and `chat.shared.contribution.ts`) to support a per-server allow/block list — likely a new policy-managed array setting alongside the current enum. The `registry_only` mode partially addresses this for enterprises hosting their own registry, but a hostname-based allow list would cover orgs that need finer control.
