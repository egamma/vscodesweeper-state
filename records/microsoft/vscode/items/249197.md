# Issue #249197 — findFiles doesn't find newly added files in a virtual workspace

[https://github.com/microsoft/vscode/issues/249197](https://github.com/microsoft/vscode/issues/249197)

## Summary

Confirmed bug: findFiles in virtual workspaces returns stale results because the remotehub extension's fileSearchCache is not invalidated when files are created. The issue is confirmed by a team member, has detailed reproduction steps and tracing, and no fix has landed on main.

## Change summary

vscode.workspace.findFiles does not find newly added files in virtual workspaces (e.g. GitHub repos on vscode.dev) due to stale caching in the file search provider.

## Best solution

The root cause is split: the remotehub extension (GitHub.remotehub, outside this repo) caches file search results without invalidating on file create/delete events, and VS Code core's search service has no mechanism to signal cache invalidation to search providers. The fix likely involves either (1) the remotehub extension listening to workspace file events to invalidate its fileSearchCache, or (2) VS Code core providing a cache-invalidation signal in the FileSearchProvider contract. A team member (mjbvz) also confirmed this affects #codebase with memfs, suggesting the core search service may need changes too. Route to the search area owner for a design decision on where the invalidation belongs.
