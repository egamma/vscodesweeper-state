# Issue #187254 — [scss] Class names in .scss files does not show on hover when @container query has a variable

[https://github.com/microsoft/vscode/issues/187254](https://github.com/microsoft/vscode/issues/187254)

## Summary

The bug lives in the vscode-css-languageservice parser — when an @container query contains an SCSS interpolation (`#{$var}`), the parser fails to recover cleanly, breaking selector path computation for subsequent rules. This is an upstream issue in that library, still present on main.

## Change summary

Hovering an SCSS class selector after a `@container` rule that uses an interpolated variable fails to show the concatenated class name.

## Best solution

Fix the @container at-rule parsing in vscode-css-languageservice (https://github.com/microsoft/vscode-css-languageservice) to handle SCSS interpolation expressions without breaking the node tree. The fix belongs upstream, not in this repo.
