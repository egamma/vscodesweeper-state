# Issue #187254 — [scss] Class names in .scss files does not show on hover when @container query has a variable

[https://github.com/microsoft/vscode/issues/187254](https://github.com/microsoft/vscode/issues/187254)

## Summary

Still reproduces on main: the SCSS parser in `vscode-css-languageservice` (v6.3.4 installed) does not override `_parseContainerQueryInParens` or `_parseMediaFeatureValue` to handle SCSS interpolation (`#{$var}`), so a `@container` query with a variable breaks the AST for subsequent selectors. The upstream repo has a test for this case in its latest source, but the fix hasn't shipped to the installed dependency yet.

## Change summary

Hovering class names after a `@container` query containing SCSS interpolation (`#{$variable}`) fails to show the concatenated selector because the parser breaks on the interpolation.

## Best solution

Update the `vscode-css-languageservice` dependency to a version that includes the container-query interpolation fix (the upstream test already exists in the latest source). The fix belongs in `microsoft/vscode-css-languageservice` — specifically adding interpolation support in the container query value parsing path of `scssParser.ts`.
