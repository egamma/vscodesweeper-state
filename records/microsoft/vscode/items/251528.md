# Issue #251528 — Regression: findMatch Foreground colors break RTL text

[https://github.com/microsoft/vscode/issues/251528](https://github.com/microsoft/vscode/issues/251528)

## Summary

Still reproduces on main: the `inlineClassName: 'findMatchInline'` on find-match decorations splits text runs, breaking RTL text flow and letter joining. PR #253063 attempted a fix but was closed without merging; no RTL-aware handling exists in the current find decorations code.

## Change summary

Find-match foreground color decorations break RTL text rendering by splitting inline text runs, causing disjoined letters and disrupted reading order.

## Best solution

The find-match decoration in `src/vs/editor/contrib/find/browser/findDecorations.ts` uses `inlineClassName` which creates `<span>` boundaries that break BiDi text shaping. The fix should either use a decoration approach that doesn't split text runs for RTL content, or apply Unicode BiDi isolation/embedding marks around the inline spans. The closed PR #253063 had a relevant approach that could be revisited.
