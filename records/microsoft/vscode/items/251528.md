# Issue #251528 — Regression: findMatch Foreground colors break RTL text

[https://github.com/microsoft/vscode/issues/251528](https://github.com/microsoft/vscode/issues/251528)

## Summary

Still reproduces on main: the `inlineClassName: 'findMatchInline'` on find-match decorations disrupts RTL text flow by wrapping matched segments in inline spans that break bidirectional rendering. The attempted fix PR #253063 was closed without merging, and no RTL-aware handling exists in the current code.

## Change summary

Find-match foreground color decorations break RTL text rendering by disrupting bidirectional text flow and dis-joining connected letters.

## Best solution

Add RTL-aware find decorations in `src/vs/editor/contrib/find/browser/findDecorations.ts` that omit `inlineClassName` for matches containing RTL text, preserving the background highlight and overview ruler but avoiding inline spans that break bidi layout. The closed PR #253063 had the right approach (separate RTL decoration options without `inlineClassName`); it needs to be reworked and landed.
