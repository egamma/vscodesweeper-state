# Issue #204333 — implicit onLanguage extension activation event not happening if language is declared, but has no language-configuration.json

[https://github.com/microsoft/vscode/issues/204333](https://github.com/microsoft/vscode/issues/204333)

## Summary

Behavior is intentional and confirmed as-designed by a team member (alexdima): implicit onLanguage activation only fires for extensions that declare a language-configuration.json. The workaround is to explicitly list `onLanguage` in activationEvents.

## Change summary

Extensions declaring a language without a language-configuration.json no longer get an implicit onLanguage activation event since VS Code 1.78.

## Best solution

Close as designed. The workaround is straightforward (explicitly declare `onLanguage:` in `activationEvents`). The remaining documentation gap belongs in a PR to microsoft/vscode-docs, not a core code change.
