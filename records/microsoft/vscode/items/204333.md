# Issue #204333 — implicit onLanguage extension activation event not happening if language is declared, but has no language-configuration.json

[https://github.com/microsoft/vscode/issues/204333](https://github.com/microsoft/vscode/issues/204333)

## Summary

Keep open as a documentation/extension-activation follow-up rather than closing as a product bug. The runtime behavior is intentional, but the current activation-events docs still state the broader rule without the `configuration`/main-language nuance.

## Change summary

Clarify that implicit `onLanguage` activation for contributed languages only applies when the language contribution has a `configuration`, and that extensions without one should declare `onLanguage:<id>` explicitly.

## Best solution

Document the current behavior in https://github.com/microsoft/vscode-docs/blob/main/api/references/activation-events.md under `onLanguage`, including the recommended explicit `activationEvents` workaround for language contributions without `configuration`.

## Evidence

- **intentional change:** Commit 413f98e4995d0c509c3b02bfd7498cbdd317b90c (https://github.com/microsoft/vscode/pull/179287) changed the languages activation generator from `if (languageContribution.id)` to `if (languageContribution.id && languageContribution.configuration)`, matching the reported behavior.
- **current code:** Current main in `src/vs/workbench/services/language/common/languageService.ts` still yields `onLanguage:${languageContribution.id}` only when both `id` and `configuration` are present.
- **maintainer comment:** alexdima explained that VS Code avoids activating extensions that contribute additional language mappings but are not the main language extension, and agreed with explicitly adding `onLanguage` as the workaround.
- **docs gap:** The current `onLanguage` docs still say that, beginning with VS Code 1.74.0, languages contributed by an extension do not require a corresponding `onLanguage` declaration, without mentioning the `configuration` requirement. Source: https://github.com/microsoft/vscode-docs/blob/main/api/references/activation-events.md.
