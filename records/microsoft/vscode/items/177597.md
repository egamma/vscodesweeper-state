# Issue #177597 — French keyboard "backtick symbol" is converted to the "lower than sign" in the native menus

[https://github.com/microsoft/vscode/issues/177597](https://github.com/microsoft/vscode/issues/177597)

## Summary

Keep open as an upstream Electron/macOS keyboard-layout issue; VS Code is passing Electron a native menu accelerator, and the mis-rendering appears to happen in Electron's native menu handling.

## Change summary

On macOS with a French keyboard layout, VS Code's native menu shows the default terminal shortcut with the wrong physical key label.

## Best solution

Track this against Electron's accelerator handling for non-US macOS keyboard layouts, especially https://github.com/electron/electron/issues/19747; VS Code should only need a core change if Electron exposes a corrected accelerator path or VS Code decides to suppress/avoid native accelerators for affected bindings.
