# Issue #121805 — "Reopen folder using..." should use a more dedicated ux solution, not notifications

[https://github.com/microsoft/vscode/issues/121805](https://github.com/microsoft/vscode/issues/121805)

## Summary

The core request — replace startup notifications for workspace/container reopening with dedicated UX (explorer buttons, badges, welcome view) — remains unimplemented. Incremental improvements were made (silent notifications for known workspaces via commit 4f3f4319082), but the workspace file notification still uses `INotificationService.prompt` on current main. Keeping open as an active under-discussion UX improvement.

## Change summary

Replace notification-based 'Reopen in Container' and 'Open Workspace' prompts with a more integrated UX surface such as explorer buttons, badges, or welcome-view actions.

## Best solution

Design and implement a non-notification UX surface for workspace/container reopen prompts — likely in the explorer (title action or empty-area button) or the remote indicator. The workspace notification code lives in `src/vs/workbench/contrib/workspaces/browser/workspaces.contribution.ts`; the dev container side is in the Remote Containers extension. Both should converge on the same pattern. Needs a UX decision before implementation.
