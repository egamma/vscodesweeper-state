## Summary

A well-documented race condition in concurrent auto-attach debug sessions. The bug likely lives in the js-debug extension's session lifecycle handling. No fix found on current main; the issue remains valid and well-described with clear repro steps, though intermittent (~1 in 10).

## Change summary

When multiple Node.js debug sessions start concurrently via Auto Attach, some sessions remain in RUNNING state after the target process exits normally.

## Best solution

The race likely lives in microsoft/vscode-js-debug's session termination/disconnect handling when multiple concurrent attach sessions fire. The debug adapter may miss or drop a 'terminated' event under concurrent load. Investigate the session lifecycle in js-debug's auto-attach machinery — particularly how concurrent DAP sessions handle the exited/terminated event sequence.
