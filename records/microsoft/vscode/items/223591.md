# Issue #223591 — Warning when running in vscode terminal 'xdg-open "https://..."'

[https://github.com/microsoft/vscode/issues/223591](https://github.com/microsoft/vscode/issues/223591)

## Summary

Still present on main: the Wayland socket detection in the snap electron-launch script only checks `$XDG_RUNTIME_DIR/../$wdisplay`, missing the direct `$XDG_RUNTIME_DIR/$wdisplay` path used on many distros. An open PR (#269449) attempts to fix this — keeping open for that PR to land.

## Change summary

Running `xdg-open` from VS Code's integrated terminal on a Wayland session produces a spurious 'Failed to open Wayland display, fallback to X11' warning because GDK_BACKEND is not correctly set to 'wayland'.

## Best solution

Review and land the open PR https://github.com/microsoft/vscode/pull/269449 which adds a fallback check for the Wayland socket at `$XDG_RUNTIME_DIR/$wdisplay` in `resources/linux/snap/electron-launch`. The fix is a ~3-line shell script change at lines 154–164.
