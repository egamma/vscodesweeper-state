# Issue #213520 — Git - VS Code Cannot Push to Git via SSH

[https://github.com/microsoft/vscode/issues/213520](https://github.com/microsoft/vscode/issues/213520)

## Summary

This is the same stale-SSH_AUTH_SOCK-on-reconnect problem tracked in the closed #168202, which was itself closed as a duplicate of microsoft/vscode-remote-release#10274 (also now closed/fixed). The root cause lives in the Remote SSH extension, not in VS Code core's git extension. A team member (lszomoru) initially routed it to the remote team, and the remote team confirmed the fix in the extension's pre-release.

## Change summary

SSH agent forwarding breaks after reconnecting a Remote SSH session because the SSH_AUTH_SOCK environment variable is not updated, causing git SSH operations to fail.

## Best solution

Close as duplicate of https://github.com/microsoft/vscode/issues/168202 — the underlying fix was delivered in the Remote SSH extension (tracked at https://github.com/microsoft/vscode-remote-release/issues/10274, also closed). Users hitting this on older builds should update the Remote SSH extension.
