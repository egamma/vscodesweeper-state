# Issue #309941 — Bad chat loading rendering

[https://github.com/microsoft/vscode/issues/309941](https://github.com/microsoft/vscode/issues/309941)

## Summary

Still present on main: the chat editor loading overlay (`chat-loading-overlay`) is appended to the DOM without any CSS rules, so it participates in normal document flow instead of overlaying — pushing tabs up and rendering the spinner below the input box.

## Change summary

Restoring a chat editor from a contributed session shows a broken loading state — tabs are pushed up and the spinner appears below the input box instead of as a centered overlay.

## Best solution

Add CSS rules for `.chat-loading-overlay` and `.chat-loading-content` in the chat editor stylesheet (e.g. `position: absolute; inset: 0; display: flex; align-items: center; justify-content: center`) so the loading indicator overlays the editor area without affecting layout flow.
