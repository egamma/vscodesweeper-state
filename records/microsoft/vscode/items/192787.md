# Issue #192787 — Output channel names are not being localized on the remote side

[https://github.com/microsoft/vscode/issues/192787](https://github.com/microsoft/vscode/issues/192787)

## Summary

Still reproduces on main: the pty host process is never given VSCODE_NLS_CONFIG in its env, so output channel names created there remain unlocalised on the remote side. The l10n-platform label is appropriate.

## Change summary

Output channel names from the pty host (and possibly other child processes like network requests) are not localised because those processes don't receive VSCODE_NLS_CONFIG.

## Best solution

Pass `VSCODE_NLS_CONFIG` (built from the client's configured language) into the pty host env in `src/vs/platform/terminal/node/nodePtyHostStarter.ts`, mirroring how the extension host connection does it in `src/vs/server/node/extensionHostConnection.ts`. Other non-extension-host processes (network requests) may need the same treatment.
