# Issue #198021 — In inline diff view of files with >= 1000 lines, the last digits of the left line numbers are sometimes truncated 

[https://github.com/microsoft/vscode/issues/198021](https://github.com/microsoft/vscode/issues/198021)

## Summary

Same root cause as #223410 (inline diff layout not reactive to decorationsLeft changes), fixed in commit 9dc000f7d79 and shipped in 1.92.0. The fix makes the original-editor width in inline diff properly track line-number width changes, which resolves the truncation at >=1000 lines.

## Change summary

In inline diff view, the last digit of left-side line numbers is clipped when files have 1000+ lines because the diff layout doesn't react to the wider line-number gutter.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/223410 — the fix there (commit 9dc000f7d79, shipped 1.92.0) makes the inline diff layout reactive to decorationsLeft changes, which directly addresses this truncation.
