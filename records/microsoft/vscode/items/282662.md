# Issue #282662 — VS Code 1.107.0 shows black screen on Debian unless launched with --disable-gpu

[https://github.com/microsoft/vscode/issues/282662](https://github.com/microsoft/vscode/issues/282662)

## Summary

Properly triaged upstream Chromium bug in DRM render node detection on Wayland. A team member already identified the root cause and provided a workaround; the issue serves as a useful tracker for affected users until the upstream fix lands.

## Change summary

VS Code shows a black screen on Debian 13 with Wayland due to incorrect DRM render node selection in Chromium's ozone platform layer.

## Best solution

Track the upstream Chromium fix in `ui/ozone/platform/wayland/common/drm_render_node_path_finder.cc` and pick it up via a future Electron update. The `--render-node-override` workaround is sufficient in the interim. No VS Code core change needed.
