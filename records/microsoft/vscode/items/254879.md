# Issue #254879 — Multiple edits to the same file should just be rendered as one

[https://github.com/microsoft/vscode/issues/254879](https://github.com/microsoft/vscode/issues/254879)

## Summary

Valid feature request to collapse multiple edits to the same file into a single rendered entry in the chat UI. No implementation found on main; a team member commented with possible approaches but no work has landed yet.

## Change summary

When an agent makes multiple sequential edits to the same file, the chat UI should render them as a single grouped entry rather than showing each edit separately.

## Best solution

Implement grouping/collapsing of consecutive tool-call results targeting the same file in the chat response rendering, likely in `src/vs/workbench/contrib/chat/browser/widget/chatContentParts/`. Alternatively, provide agents with a batch-edit tool so multiple edits arrive as one operation. The roblourens comment suggests both approaches are under consideration.
