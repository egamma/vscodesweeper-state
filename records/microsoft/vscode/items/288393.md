# Issue #288393 — memory leak in extension host editor tabs

[https://github.com/microsoft/vscode/issues/288393](https://github.com/microsoft/vscode/issues/288393)

## Summary

Fixed on main via commit 95bb7480752 (PR #321936, shipped in 1.126.0), which reuses ExtHostEditorTab and ExtHostEditorTabGroup instances during model resyncs instead of recreating them — directly addressing the reported leak. The separate community PR #288404 is now superseded.

## Change summary

ExtHostEditorTab and ExtHostEditorTabGroup objects were recreated on every full tab-model resync, leaking the old instances and any Map/WeakMap/Set collections keyed by them.

## Best solution

Keep the shipped fix in commit 95bb7480752 (1.126.0). The open community PR https://github.com/microsoft/vscode/pull/288404 is now superseded and can be closed.
