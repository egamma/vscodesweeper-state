# Issue #138905 — in Persian language the words (آ) and (ه) are displayed in the highlighted form 

[https://github.com/microsoft/vscode/issues/138905](https://github.com/microsoft/vscode/issues/138905)

## Summary

Still reproducible on main: the ambiguous character data in `src/vs/base/common/strings.ts` has no Persian/Arabic ("fa"/"ar") locale entry, so characters like ه (U+0647, mapped to 'o') are highlighted as confusable even when the OS locale is Persian. The partial fix in commit 58861695500 suppresses highlighting only in purely non-ASCII word contexts, but doesn't address the missing locale data.

## Change summary

Persian characters آ and ه are incorrectly highlighted as ambiguous unicode confusables for users with a Persian OS locale.

## Best solution

Add a Persian/Arabic locale ("fa"/"ar") entry to `AmbiguousCharacters.ambiguousCharacterData` in `src/vs/base/common/strings.ts` (generated via https://github.com/hediet/vscode-unicode-data) that excludes Arabic-script characters common in Persian from the confusable set, so the existing `_os` locale detection suppresses highlighting for Persian users.

- **character in _common set:** U+0647 (ه, heh) is mapped to 111 ('o') in the _common ambiguous set and U+0627 (alef, 1575) is mapped to 108 ('l'); these trigger highlighting regardless of OS locale since no Persian locale exclusion exists.

- **OS locale mechanism exists:** src/vs/editor/contrib/unicodeHighlighter/browser/unicodeHighlighter.ts line 166-168: `_os` locale resolves via Intl.NumberFormat but has no matching data to use for Persian.
