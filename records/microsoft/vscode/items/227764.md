# Issue #227764 — [html] highlighting does not detect end script tag in string literals

[https://github.com/microsoft/vscode/issues/227764](https://github.com/microsoft/vscode/issues/227764)

## Summary

Acknowledged by maintainer aeschli as a real limitation: the embedded JavaScript TextMate grammar doesn't treat `</script>` as a stop token inside string literals. Kept open per maintainer's implicit intent (commented but didn't close).

## Change summary

HTML syntax highlighting fails to end the `<script>` block when `</script>` appears inside a JavaScript string literal, diverging from how browsers parse it.

## Best solution

The HTML TextMate grammar (or the embedded JavaScript grammar rules for HTML) would need special-case handling to recognize `</script>` as a block terminator even within string contexts — matching browser parsing behavior. This likely requires custom JavaScript tokenization rules specific to HTML-embedded scripts.
