# Issue #227764 — [html] highlighting does not detect end script tag in string literals

[https://github.com/microsoft/vscode/issues/227764](https://github.com/microsoft/vscode/issues/227764)

## Summary

Still present on main — the HTML TextMate grammar's script-end pattern matches `</script>` regardless of JavaScript string context, as acknowledged by aeschli. This is a known grammar limitation with no fix applied.

## Change summary

HTML syntax highlighting incorrectly ends the script block when `</script>` appears inside a JavaScript string literal.

## Best solution

The HTML grammar's script-block end rules in `extensions/html/syntaxes/html.tmLanguage.json` need context-aware patterns that avoid matching `</script>` inside JS string literals — or a separate embedded-JS grammar variant that handles this. This is fundamentally difficult with TextMate grammars but could be approximated with more specific end patterns.
