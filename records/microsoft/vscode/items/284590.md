# Issue #284590 — Editor GPU characters are sometimes far too large

[https://github.com/microsoft/vscode/issues/284590](https://github.com/microsoft/vscode/issues/284590)

## Summary

Intermittent GPU rendering bug where characters render at incorrect (too large) size, likely related to DPR handling across multi-monitor setups with different scaling. No fix found on main; the issue remains valid and needs investigation.

## Change summary

Editor GPU renderer sometimes draws characters at an oversized scale, possibly triggered by mixed-DPR multi-monitor configurations.

## Best solution

Investigate the glyph rasterizer's devicePixelRatio handling in `src/vs/editor/browser/gpu/raster/glyphRasterizer.ts` and the texture atlas slab allocator to ensure DPR is correctly resolved per-window when moving between monitors with different scaling factors.
