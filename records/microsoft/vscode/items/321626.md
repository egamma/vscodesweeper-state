# Issue #321626 — [Error] unhandlederror-Timed out waiting for authentication provider 'github' to register.

[https://github.com/microsoft/vscode/issues/321626](https://github.com/microsoft/vscode/issues/321626)

## Summary

Still reproduces on main: the 5s timeout in `tryActivateProvider` is present with its own TODO comment acknowledging it needs a proper fix. High-volume error (~171K users on 1.127.0) especially in remote/Codespaces scenarios. Team member has already investigated the telemetry breakdown; this needs the cross-cutting activation-handshake refactor described in the analysis.

## Change summary

Authentication provider 'github' times out during startup because the 5s `raceTimeout` in `tryActivateProvider` fires before the extension-host RPC delivers the provider registration, especially in remote sessions.

## Best solution

Replace the tactical 5s timeout with a proper activation handshake that guarantees the auth provider registers during `activate()` before consumers call `getSessions`. The fix lives in `src/vs/workbench/services/authentication/browser/authenticationService.ts` (the `tryActivateProvider` method) and likely requires coordination with the extension activation lifecycle. The remote-authority skew (Codespaces dominates) suggests the timeout should at minimum be adaptive to connection latency, but the TODO says to remove it entirely.
