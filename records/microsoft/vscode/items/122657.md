# Issue #122657 — [folding] region ranges not restored when editor is opened the first time.

[https://github.com/microsoft/vscode/issues/122657](https://github.com/microsoft/vscode/issues/122657)

## Summary

Confirmed bug acknowledged by maintainer @aeschli: region-based folding ranges depend on language configuration that isn't available when restoreViewState runs on first editor open. The timing issue remains unfixed on current main.

## Change summary

Folded region ranges are not restored when a file is reopened because language settings aren't available yet during view state restoration.

## Best solution

Defer or re-apply the folding memento after the first folding range computation completes (in `triggerFoldingModelChanged`), so that region-based ranges that depend on language configuration are properly matched. The fix belongs in `src/vs/editor/contrib/folding/browser/folding.ts`.
