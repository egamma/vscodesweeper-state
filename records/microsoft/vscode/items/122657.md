# Issue #122657 — [folding] region ranges not restored when editor is opened the first time.

[https://github.com/microsoft/vscode/issues/122657](https://github.com/microsoft/vscode/issues/122657)

## Summary

Still valid on main. The timing issue acknowledged by aeschli — language settings not available when folded regions are first restored — remains present in the current folding controller code. Region-based folding ranges rely on language-specific markers that may not be recognized at restore time.

## Change summary

Region-based folded ranges (#region/#endregion) are not properly restored when an editor is opened for the first time because language settings are not yet available.

## Best solution

The `restoreViewState`/`applyMemento` path needs to account for region markers not yet being recognized. One approach: defer memento application until after the first syntax-based range computation completes, or re-apply the memento when folding providers become available. Fix likely lives in `src/vs/editor/contrib/folding/browser/folding.ts` around the interaction between `restoreViewState`, `triggerFoldingModelChanged`, and `onFoldingStrategyChanged`.
