# Issue #225315 — Search Editor Progress Bar Animates Endlessly After Search Completed (intermittent)

[https://github.com/microsoft/vscode/issues/225315](https://github.com/microsoft/vscode/issues/225315)

## Summary

Confirmed, still-reproducing bug on the On Deck milestone — the search editor progress bar can get stuck spinning after toggling between valid and invalid regex input. PR #225549 was a partial fix; the race condition remains. Hard keep-open due to the On Deck milestone.

## Change summary

The search editor's blue progress bar animates endlessly after a search completes when the user toggles between valid and invalid regex patterns.

## Best solution

The progress-bar lifecycle in `doRunSearch` (src/vs/workbench/contrib/searchEditor/browser/searchEditor.ts, lines 540–612) needs a more robust guard: if `searchModel.search(query)` throws synchronously for an invalid regex pattern, or if the `Delayer`-scheduled `doRunSearch` runs after the input becomes invalid (reading stale-then-current widget state), `ongoingOperations` is incremented but never decremented, leaving the progress bar stuck. The fix should ensure `searchOperation.stop()` is called on every exit path from `doRunSearch` that follows `searchOperation.start()`.
