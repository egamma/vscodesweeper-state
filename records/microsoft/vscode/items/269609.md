# Issue #269609 — Terminal dictation while typing overlaps text

[https://github.com/microsoft/vscode/issues/269609](https://github.com/microsoft/vscode/issues/269609)

## Summary

Team-authored bug about terminal dictation overlapping typed text — must keep open per policy. The interaction between voice dictation ghost text and concurrent typing in the terminal needs a resolution strategy (accept or drop pending dictation on keystroke).

## Change summary

When typing in the terminal while dictation is processing, the dictated text overlaps with typed text instead of being accepted or dropped.

## Best solution

The terminal voice/dictation integration should commit or discard pending ghost text when the user begins typing, preventing overlap. Likely lives in the intersection of `workbench-voice` and `terminal` contributions.
