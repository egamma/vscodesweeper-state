# Issue #246030 — Race Condition in Merge Complete button when the file is unsaved?

[https://github.com/microsoft/vscode/issues/246030](https://github.com/microsoft/vscode/issues/246030)

## Summary

The reported race condition is not evident in the current code — the save is properly awaited before staging. The reporter hasn't confirmed on the latest build, and the reproduction steps are environment-specific and unconfirmed by others.

## Change summary

Clicking 'Complete Merge' on a large unsaved file allegedly stages the conflict-containing version because git add runs before save completes.

## Best solution

Request confirmation on the latest VS Code build with concrete reproduction evidence. The code path in `extensions/git/src/commands.ts` (line 1848–1850) properly awaits `mergeEditor.acceptMerge` (which awaits save) before calling `repository.add()`. If the issue persists, it may be a file-system flush timing issue on Windows with large files under memory pressure — that would need a targeted investigation in the text file service layer.
