# Issue #314075 — Extension host crashes deterministically with SIGTRAP (code 133, INT3 at fixed binary offset 0x431ea0d) on 1.118.1 — downgrade to 1.117.0 fixes it

[https://github.com/microsoft/vscode/issues/314075](https://github.com/microsoft/vscode/issues/314075)

## Summary

The report has concrete crash signatures and later comments narrow the trigger to Copilot Chat enumerating chat session providers, with additional confirmation on 1.120. This should stay open for the chat/extension-host owners to investigate or route upstream if the fixed INT3 offset is proven to be Electron/V8.

## Change summary

VS Code 1.118+ extension host repeatedly exits with SIGTRAP/code 133 on Linux when Copilot Chat interacts with registered chat session providers, while 1.117.0 does not crash.

## Best solution

Investigate the chat session provider enumeration path and Copilot Chat activation around `vscode.chat.registerChatSessionItemProvider`; if the deterministic INT3 is in Electron/V8, keep this as an upstream-tracked crash with a minimal repro and dependency version mapping rather than closing it from current evidence.

## Evidence

- **issue evidence:** Reporter provided repeated extension host exits with code 133 and kernel INT3 traps at the same binary offset `0x431ea0d` on VS Code 1.118.1, and stated downgrading to 1.117.0 stopped the crash with the same workspace and configuration.
- **version confirmation:** Reporter later reproduced the same fixed offset on VS Code 1.119.0, and another user confirmed SIGTRAP still occurs on 1.120.
- **trigger narrowed:** A later reporter update ties the crashes to GitHub Copilot Chat activation and enumeration of third-party chat session providers through `vscode.chat.registerChatSessionItemProvider`, specifically mentioning Anthropic Claude Code as registering `claude-code`.
- **maintainer context:** A VS Code contributor reported seeing the same behavior, noted that disabling the chat extension avoids it, and linked similar prior issues at https://github.com/microsoft/vscode/issues/292644 and https://github.com/microsoft/vscode/issues/292167, but did not identify a confirmed canonical duplicate.
