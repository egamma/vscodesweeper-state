# Issue #109954 — SCM - support for filesystem path using ssh://

[https://github.com/microsoft/vscode/issues/109954](https://github.com/microsoft/vscode/issues/109954)

## Summary

Legitimate community-promoted feature request (75 upvotes, Backlog milestone) for SCM to support ssh:// filesystem paths. No implementation exists on current main, and no canonical issue supersedes it. Keeping open.

## Change summary

VS Code's Source Control should support git repositories accessed via ssh:// filesystem paths (e.g., through SSH FS extensions) rather than requiring Remote-SSH, which doesn't support all architectures.

## Best solution

The built-in git extension would need to understand non-local filesystem schemes (ssh://) when scanning for and interacting with repositories. This is a non-trivial change to the SCM/git integration layer. Given the strong community signal (75 upvotes), it should remain in the Backlog for prioritization. Alternatively, clearer extension API hooks for custom filesystem-aware SCM providers could let extensions like vscode-sshfs solve this themselves.
