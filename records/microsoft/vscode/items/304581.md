# Issue #304581 — [BUG] VS Code Hangs when elicitation is requested with empty schema (i.e. user info not requested)

[https://github.com/microsoft/vscode/issues/304581](https://github.com/microsoft/vscode/issues/304581)

## Summary

Team-authored issue; the bug is source-reproducible on main — when `requestedSchema.properties` is empty, `_convertSchemaToQuestions` returns zero questions, the carousel's DeferredPromise never resolves, and the MCP tool call hangs indefinitely. No fix has landed.

## Change summary

VS Code hangs indefinitely on an MCP elicitation request when the schema has no properties (empty object), instead of showing a yes/no confirmation or resolving immediately.

## Best solution

Guard `_elicitForm` in `mcpElicitationService.ts`: when `requestedSchema.properties` is empty (no questions to ask), immediately resolve the carousel with an accept (empty content) or show a simple confirm/cancel prompt, matching the prior behavior.
