# Issue #311614 — Integrated terminal rendering becomes corrupted after macOS lock/unlock

[https://github.com/microsoft/vscode/issues/311614](https://github.com/microsoft/vscode/issues/311614)

## Summary

Active, confirmed bug with team engagement (meganrogge, anthonykim1, doubleyewdee) and multiple independent reproductions on macOS Apple Silicon. The WebGL renderer's existing `onContextLoss` handler doesn't cover all corruption scenarios triggered by macOS desktop switching / lock-unlock. No fix has landed on main. This is the best-diagnosed report for this cluster and should remain the canonical tracking issue.

## Change summary

The integrated terminal's WebGL-rendered text becomes garbled/corrupted on macOS after lock/unlock, desktop switching, or extended use — recoverable only by zoom toggle, reload, or terminal restart.

## Best solution

Investigate whether the WebGL context invalidation on macOS (particularly Apple Silicon + Metal ANGLE backend) can be detected and force a redraw/re-render without a full context loss event. The fix likely lives in `src/vs/workbench/contrib/terminal/browser/xterm/xtermTerminal.ts` around the WebGL addon lifecycle, or may require an upstream xterm.js change to handle partial GPU state corruption. Issue https://github.com/microsoft/vscode/issues/288682 is a satellite of this one and could be closed as a duplicate once this is confirmed as canonical.
