# Issue #283526 — [Windows] VS Code shows blank white window and exits when launched via “Run as different user”/"Run as Administrator"; fixed by --disable-gpu-sandbox (1.107 / Electron 39)

[https://github.com/microsoft/vscode/issues/283526](https://github.com/microsoft/vscode/issues/283526)

## Summary

Already correctly triaged as an upstream Chromium/Electron regression. The GPU sandbox crashes under Windows secondary logon tokens after the Electron 39 upgrade. No fix has landed on main; the root cause is in Chromium's GPU process initialization, not in VS Code's own code. Keeping open to track upstream resolution.

## Change summary

VS Code shows a blank white window and exits when launched via 'Run as different user' on Windows due to GPU sandbox incompatibility with secondary logon tokens, introduced by the Electron 39 upgrade.

## Best solution

Track the upstream Chromium/Electron fix for GPU sandbox behavior under Windows secondary logon tokens. VS Code's `src/main.ts` already disables GPU sandbox when `--no-sandbox` is used, but the default sandboxed path has no fallback for this scenario. Once Chromium/Electron ships a fix, updating Electron will resolve it. In the meantime, `--disable-gpu-sandbox` is the documented workaround.
