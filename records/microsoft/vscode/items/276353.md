# Issue #276353 — Checkpoint restoration only reverts first file when multiple files modified across separate requests

[https://github.com/microsoft/vscode/issues/276353](https://github.com/microsoft/vscode/issues/276353)

## Summary

The checkpoint/snapshot system has been significantly refactored since 1.106.0-insider (Nov 2025) with a new timeline-based architecture that explicitly tracks all file operations including text edits across multiple files. The reported bug likely no longer reproduces, but no single commit explicitly fixes this scenario — requesting confirmation on a current build.

## Change summary

Restoring a checkpoint when multiple files were modified across separate requests only reverts the first file, leaving subsequent file changes in place.

## Best solution

Ask the reporter to verify on the latest Insiders build. The checkpoint system now uses `ChatEditingCheckpointTimelineImpl` which tracks all `FileOperationType.TextEdit` operations and restores all affected URIs via `_reconstructAllFileContents`. If it still reproduces, the bug is in the epoch/operation tracking for multi-request scenarios.
