# Issue #231313 — Custom HISTFILE not being honored

[https://github.com/microsoft/vscode/issues/231313](https://github.com/microsoft/vscode/issues/231313)

## Summary

Verified duplicate of #169264, which tracks the same HISTFILE-clobbering defect and already has an open fix PR (#314615). Proposing duplicate close to consolidate discussion.

## Change summary

Zsh shell integration unconditionally overwrites a custom HISTFILE set in ~/.zshenv, forcing it back to the default ~/.zsh_history location.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/169264 and land https://github.com/microsoft/vscode/pull/314615, which preserves a pre-existing HISTFILE via an env-var handoff. This reporter's workaround (disabling shell integration) and the commenter's suggested `${HISTFILE:=...}` fix both confirm the same root cause.
