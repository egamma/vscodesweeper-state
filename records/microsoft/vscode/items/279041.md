# Issue #279041 — Titlebar overlay buttons have the wrong size when increasing Text size

[https://github.com/microsoft/vscode/issues/279041](https://github.com/microsoft/vscode/issues/279041)

## Summary

Confirmed and still unfixed on main: the titlebar overlay height is hardcoded to 29px on Windows and doesn't account for OS-level text size scaling, causing a mismatch between the overlay buttons and the rest of the titlebar when Windows accessibility text size is increased.

## Change summary

Windows titlebar overlay buttons don't scale with the OS-level text size accessibility setting, causing visual mismatch.

## Best solution

The initial overlay height (hardcoded at 29 in `src/vs/platform/windows/electron-main/windows.ts:230`) and the dynamic height passed to `setTitleBarOverlay` in `windowImpl.ts` need to account for the Windows text scale factor. This may require reading the OS text scale factor via Electron APIs or system calls, and may partially depend on Electron's handling of `setTitleBarOverlay` with scaled values.
