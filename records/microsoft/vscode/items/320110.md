# Issue #320110 — Copilot Chat fails to process images with third-party OpenAI-compatible models (DeepSeek V4): unknown variant image_url, expected text

[https://github.com/microsoft/vscode/issues/320110](https://github.com/microsoft/vscode/issues/320110)

## Summary

The error originates from the third-party extension 'OAI Compatible Provider for Copilot' (johnny-zhao.oai-compatible-copilot), which is responsible for translating VS Code's language model API message parts into the OpenAI-compatible format that DeepSeek expects. VS Code core correctly delivers image data as `LanguageModelDataPart` through the extension API; the extension must convert it to the standard `{type: "image_url"}` format before forwarding to the model endpoint.

## Change summary

Pasting an image in Copilot Chat with a third-party OpenAI-compatible model (DeepSeek V4, via the OAI Compatible Provider extension) returns a 400 error: 'unknown variant image_url, expected text'.

## Best solution

File this as a bug on the extension's repository (johnny-zhao.oai-compatible-copilot). The extension needs to handle `LanguageModelDataPart` instances in the message content array and convert them to the OpenAI-standard `{type: "image_url", image_url: {url: "data:..."}}` format before sending to the third-party endpoint.
