# Issue #210341 — Folders cointaining only `.gitignore`d files are sometimes not hidden, disappear only on click

[https://github.com/microsoft/vscode/issues/210341](https://github.com/microsoft/vscode/issues/210341)

## Summary

Still present on main: `.gitignore` files inside unresolved folders are only processed lazily when the folder is expanded, so folders containing exclusively gitignored files remain visible until clicked. The canonical issue #66971 was closed for a different fix (explorer find), not for this filtering gap.

## Change summary

Folders whose contents are all covered by a nested `.gitignore` are not hidden by `explorer.excludeGitIgnore` until the user expands them, because the gitignore is only read on folder resolution.

## Best solution

The filter in `explorerViewer.ts` processes nested `.gitignore` files lazily (line ~1388). To hide folders whose entire contents are gitignored, the explorer would need to eagerly read `.gitignore` files in unresolved directories — or re-filter parent folders after a nested `.gitignore` is processed. This is a non-trivial change in the explorer's resolution/filter pipeline and needs area-owner judgment on the performance trade-off.
