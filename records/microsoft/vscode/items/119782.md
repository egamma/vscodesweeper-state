# Issue #119782 — Git - No more surrounding or auto-closing in commit msg box

[https://github.com/microsoft/vscode/issues/119782](https://github.com/microsoft/vscode/issues/119782)

## Summary

Still reproduces on main: the `scminput` language registered for the SCM input box has no language configuration (no surroundingPairs, no autoClosingPairs, no brackets), so selecting text and typing a bracket/quote character replaces the selection instead of surrounding it. Team members confirmed the root cause in comments.

## Change summary

Auto-surrounding and auto-closing pairs (backticks, quotes, brackets) stopped working in the SCM commit message input box since VS Code 1.45.

## Best solution

Register a language configuration for the `scminput` language with `surroundingPairs`, `autoClosingPairs`, and `brackets` matching what `plaintext` provides — either via `ILanguageConfigurationService` in `src/vs/workbench/contrib/scm/browser/scm.contribution.ts` or by adding a `scminput.language-configuration.json` under `extensions/git-base/languages/`. The `help wanted` label is already set, making this a good community contribution target.
