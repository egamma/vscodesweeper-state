# Issue #166741 — Sticky scroll messes up accurate markdown scroll sync

[https://github.com/microsoft/vscode/issues/166741](https://github.com/microsoft/vscode/issues/166741)

## Summary

Still present on main: the editor's `visibleRanges` API doesn't account for lines occluded by sticky scroll, causing markdown preview scroll sync to desynchronize. alexdima confirmed this is an architectural gap with no straightforward fix applied since.

## Change summary

Sticky scroll causes editor `visibleRanges` to include occluded lines, breaking markdown preview scroll sync.

## Best solution

The editor core needs to expose an adjusted visible range that excludes lines hidden behind sticky scroll (or provide a separate API for the "true" first visible line). The fix likely lives in the view model's visible-range computation (`src/vs/editor/common/viewModel/viewModelImpl.ts`) or the extension host's `visibleRanges` relay (`src/vs/workbench/api/common/extHostTextEditor.ts`). This is non-trivial given the contribution-based architecture of sticky scroll.
