# Issue #234814 — Git - Command `git.openFile` does not do the same thing as the "Open File" icon button on a file that has been committed in diff view

[https://github.com/microsoft/vscode/issues/234814](https://github.com/microsoft/vscode/issues/234814)

## Summary

Still reproduces on main: when `git.openFile` is invoked via keybinding on a committed file's diff view, `arg` is undefined, `getSCMResource()` finds nothing (file is no longer in working tree/index/merge groups), and the fallback at line 1319 uses the active editor's git-scheme URI directly — opening a readonly editor instead of the working-tree file. Milestone is On Deck, keeping open.

## Change summary

The `git.openFile` command opens a readonly editor when invoked via keybinding on a committed file's diff view, instead of opening the writable working-tree file like the toolbar button does.

## Best solution

In `openFile()` (extensions/git/src/commands.ts ~line 1319), when falling back to `window.activeTextEditor.document.uri`, check if it's a git URI and convert it to a file URI (similar to lines 1302–1303). A closely analogous fix was recently made for `stageFile`/`unstageFile` in commit 61deced4744.
