# Issue #277632 — VSCode 1.106 / repository new icon problem

[https://github.com/microsoft/vscode/issues/277632](https://github.com/microsoft/vscode/issues/277632)

## Summary

The filled vs outline repo icon difference is intentional — the code deliberately uses `Codicon.repoSelected` for multi-repo workspaces to indicate which repository is visible/active. This is a polish request to use a consistent icon style. Keeping open as it's labeled `under-discussion`.

## Change summary

SCM view uses different icon styles (outline vs filled) for repository entries depending on whether one or multiple repositories are present.

## Best solution

The SCM team should decide whether the filled 'selected' icon adds enough clarity in multi-repo scenarios to justify the visual inconsistency. If not, remove the `showSelectedIcon` branch in `src/vs/workbench/contrib/scm/browser/scmRepositoryRenderer.ts` (lines 119-123) and always use `Codicon.repo`.
