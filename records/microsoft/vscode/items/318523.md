# Issue #318523 — Feature Request: File-only Navigate Back Command

[https://github.com/microsoft/vscode/issues/318523](https://github.com/microsoft/vscode/issues/318523)

## Summary

Valid feature request for a file-level-only navigation back command. The existing GoFilter.NAVIGATION only tracks explicit 'Go to definition'-style jumps, not all file switches — this asks for something distinct that is not yet implemented.

## Change summary

Request for a navigate-back command that skips within-file cursor changes and only moves between different files.

## Best solution

Add a new GoFilter value (e.g. GoFilter.FILE) in src/vs/workbench/services/history/common/history.ts that records entries only when the file (resource) changes, plus a corresponding command in editorActions.ts. The history service already supports filter-based navigation stacks, so the pattern is well-established.

- **existing filter architecture:** src/vs/workbench/services/history/common/history.ts defines GoFilter enum with NONE, EDITS, NAVIGATION — no file-only filter exists.
