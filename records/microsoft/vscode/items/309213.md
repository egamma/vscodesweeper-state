# Issue #309213 — Extension allowlist by publisher fails for remote extensions after version upgrade due to missing publisherDisplayName metadata

[https://github.com/microsoft/vscode/issues/309213](https://github.com/microsoft/vscode/issues/309213)

## Summary

Confirmed in source: when remote extensions are installed with source 'vsix' (server migration), publisherDisplayName metadata is absent, so the GPO publisher allowlist check at line 126 of allowedExtensionsService.ts falls through to the raw publisher ID which doesn't match the org name. The reporter's extensions.json evidence confirms the gap. The maintainer asked for info and got a detailed reply — this is in active triage.

## Change summary

Remote extensions installed via VSIX during server setup lack publisherDisplayName metadata, causing the GPO publisher allowlist to block them until a window reload backfills the metadata.

## Best solution

The fallback logic in `allowedExtensionsService.ts` line 126 should also attempt to match the raw publisher ID prefix against `extensionPublisherOrgs` (or the metadata should be preserved/backfilled during VSIX-based remote installs in `extensionManagementService.ts`). The area owner (sandy081) is already engaged.
