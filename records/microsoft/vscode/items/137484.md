# Issue #137484 — Optimize ripgrep glob patterns for file search

[https://github.com/microsoft/vscode/issues/137484](https://github.com/microsoft/vscode/issues/137484)

## Summary

Team-authored issue, still confirmed valid as of Dec 2024. The glob optimization (groupBy + spreadGlobComponents) exists in ripgrepTextSearchEngine.ts but has not been ported to ripgrepFileSearch.ts. Keeping open per hard keep-open rule for MEMBER-authored issues.

## Change summary

Port the include-glob optimization (splitting non-** globs via spreadGlobComponents) from text search to file search, improving performance for workspaceContains and findFiles.

## Best solution

Apply the same glob-rewriting logic from `ripgrepTextSearchEngine.ts` (lines 415–432, using `groupBy` and `spreadGlobComponents`) to `ripgrepFileSearch.ts` so that file search also avoids scanning the entire workspace for patterns like `*/foo`.
