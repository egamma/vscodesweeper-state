# Issue #137484 — Optimize ripgrep glob patterns for file search

[https://github.com/microsoft/vscode/issues/137484](https://github.com/microsoft/vscode/issues/137484)

## Summary

Team-authored polish issue that remains valid: the text search glob optimization (splitting non-** includes into path components to narrow the search directory) has not been ported to the file search engine. Author confirmed it is still needed in Dec 2024.

## Change summary

Port the ripgrep include-glob optimization from text search to file search so patterns like `*/foo` narrow the search directory instead of scanning the whole workspace.

## Best solution

Port the `doubleStarIncludes`/`otherIncludes` grouping and `spreadGlobComponents` logic from `ripgrepTextSearchEngine.ts` into `ripgrepFileSearch.ts`'s `getRgArgs` function, so file search (used by `workspaceContains` and `findFiles`) benefits from the same directory-narrowing optimization that text search already has.
