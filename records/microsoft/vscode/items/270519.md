# Issue #270519 — The search results cannot be copied in a tree structure

[https://github.com/microsoft/vscode/issues/270519](https://github.com/microsoft/vscode/issues/270519)

## Summary

Still reproduces on main: the copy functions in searchActionsCopy.ts have no tree-awareness — they always produce flat output regardless of the tree/list toggle. An open PR (#310767) is already attempting a fix.

## Change summary

"Copy All" in the Search view ignores the tree layout setting and always copies results in a flat list format.

## Best solution

Review and land (or close) the open PR https://github.com/microsoft/vscode/pull/310767, which threads an `isTree` flag through the copy helpers to emit indented tree-structured output when the view is in tree mode.
