# Issue #315131 — Completion-inserted `()` no longer tracked as auto-closed pair

[https://github.com/microsoft/vscode/issues/315131](https://github.com/microsoft/vscode/issues/315131)

## Summary

This should stay open and be routed to the editor area as a plausible core regression: current main still only tracks inserted auto-closing pairs when the edit source is `snippet`, while completion snippets now reach the cursor as `suggest`. The report has concrete repro steps and the code path matches the claimed behavior, though I did not run an interactive reproduction.

## Change summary

Completion items that insert snippets like `foo($0)` should keep `()` registered as an auto-closed pair so Backspace/overtype remove or replace both characters.

## Best solution

Fix the editor/suggest snippet insertion path so `_findAutoClosingPairs` is applied when a suggest completion inserts snippet text that creates an auto-closing pair, likely around `src/vs/editor/common/cursor/cursor.ts` or the source passed through `SnippetSession.insert`. Add focused coverage for completion-accepted snippets inserting `()` and then Backspace/overtype behavior.

## Evidence

- **current cursor behavior:** In `src/vs/editor/common/cursor/cursor.ts`, `executeEdits` still calls `_findAutoClosingPairs(edits)` only under `if (source === 'snippet')`; otherwise no auto-closed ranges are pushed for Backspace/overtype handling.
- **suggest completion path:** In `src/vs/editor/contrib/suggest/browser/suggestController.ts`, accepting a suggestion passes `reason: EditSources.suggest({ providerId: ... })` into `snippetController.insert(...)`.
- **source propagation:** `src/vs/editor/contrib/snippet/browser/snippetSession.ts` calls `this._editor.executeEdits(editReason ?? EditSources.snippet(), ...)`, and `src/vs/editor/browser/widget/codeEditor/codeEditorWidget.ts` converts a `TextModelEditSource` to `sourceStr = source.metadata.source`, so `EditSources.suggest` reaches the cursor as source `suggest`.
- **regression source:** Commit `b1689799b2e73c6eb87d3a67b72113305733ffc4` / PR https://github.com/microsoft/vscode/pull/252666 changed snippet/suggest edit reason reporting by passing a suggest edit reason through the suggestion acceptance path.
