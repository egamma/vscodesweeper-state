# Issue #125270 — Toggling wrap during comparison moves the view on the open files.

[https://github.com/microsoft/vscode/issues/125270](https://github.com/microsoft/vscode/issues/125270)

## Summary

This appears to have been fixed on main by making Toggle Word Wrap apply to both sides of a side-by-side diff editor, which addresses the likely scroll-sync mismatch shown in the recording. The fix shipped starting in 1.65.0, so this can be closed as completed rather than with a triage close label.

## Change summary

Toggling word wrap in one side of a diff editor caused the compared files to jump away from the current viewport.

## Best solution

Treat the issue as fixed by commit 2ecb5c0b8f5a24b0310c34e90607540e565c7fee, which synchronizes transient word-wrap state across both diff panes; no further core change is needed unless a fresh reproduction exists on current builds.
