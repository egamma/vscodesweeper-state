# Issue #325607 — Quick Open is slow to open an exact relative path in large workspaces

[https://github.com/microsoft/vscode/issues/325607](https://github.com/microsoft/vscode/issues/325607)

## Summary

Valid performance issue: relative paths in Quick Open go through a full workspace file search even when the exact file can be resolved immediately via stat, unlike absolute paths which have a fast path. An open PR from the reporter addresses this.

## Change summary

Quick Open waits for a full fuzzy file search before showing results for an exact workspace-relative path, causing multi-second delays in large workspaces.

## Best solution

Review and land PR https://github.com/microsoft/vscode/pull/325609, which adds an early-exit fast path for relative paths matching the existing absolute-path pattern in `getAbsolutePathFileResult` / `getFilePicks`.
