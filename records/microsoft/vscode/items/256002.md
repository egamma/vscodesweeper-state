# Issue #256002 — Chat: UI controls are still operational even though they look disabled

[https://github.com/microsoft/vscode/issues/256002](https://github.com/microsoft/vscode/issues/256002)

## Summary

PR #260910 fixed the mouse-click aspect of this bug by intercepting clicks on the disabled overlay, but the keyboard-navigation issue raised in Comment 1 (Tab still reaches greyed-out controls) remains unaddressed on current main. The `setEditing` method only sets context keys without applying `inert` or disabling Tab focus on the underlying controls.

## Change summary

When editing a previous chat message, the greyed-out UI controls below (mode/model pickers, tools quick pick) are still reachable and operational via keyboard Tab navigation, despite appearing disabled.

## Best solution

Apply the `inert` HTML attribute (or equivalent focus-trap) to the chat input and list regions that should be non-interactive during edit mode, so that both mouse and keyboard access is properly blocked. The likely fix area is `chatInputPart.ts` (`setEditing` method) and/or the chat list overlay setup in `chatListRenderer.ts`.
