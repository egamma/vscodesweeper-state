# Issue #315740 — `Go to symbol` open to side button does nothing

[https://github.com/microsoft/vscode/issues/315740](https://github.com/microsoft/vscode/issues/315740)

## Summary

Still reproduces on main: the agents window has no override of `gotoLocation` to handle the `forceSideBySide` path — the base implementation in `gotoSymbolQuickAccess.ts` fires the event but nothing in `src/vs/sessions/` implements the side-by-side open.

## Change summary

Clicking the 'Open to Side' button in the Go to Symbol quick pick does nothing in the agents window.

## Best solution

The agents window needs its own GotoSymbolQuickAccess registration (analogous to `src/vs/workbench/contrib/codeEditor/browser/quickaccess/gotoSymbolQuickAccess.ts`) that overrides `gotoLocation` to handle the `forceSideBySide` case — likely opening the target in the session's side pane or a split editor within the agents layout.
