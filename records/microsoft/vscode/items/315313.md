# Issue #315313 — VS Code Copilot Quick Picker doesn't respect gitignore

[https://github.com/microsoft/vscode/issues/315313](https://github.com/microsoft/vscode/issues/315313)

## Summary

The report is too thin to reproduce — the chat `#` file completions use `searchFilesAndFolders` which respects `.gitignore` by default (via `disregardIgnoreFiles: false`). The reporter's gitignore pattern `./nx/cache` is unusual syntax (git treats `./` literally, so it would not match `.nx/cache`), which may explain the mismatch. Need their settings and a confirmed-valid gitignore pattern to proceed.

## Change summary

Typing `#exam` in Copilot chat suggests files from `.nx/cache` despite it being listed in `.gitignore`.

## Best solution

Ask the reporter to confirm: (1) whether their `.gitignore` entry is `./nx/cache` or `.nx/cache` (the leading `./` matters — git interprets it literally), (2) their `search.useIgnoreFiles` and `explorer.excludeGitIgnore` settings, and (3) whether the issue reproduces with a clean `.gitignore` entry like `.nx/cache` (no `./` prefix). Separately, there is a real inverted-logic bug in `src/vs/workbench/contrib/search/browser/searchChatContext.ts` line 245: when `explorer.excludeGitIgnore` is `true`, `disregardIgnoreFiles` is set to `true` (should be `!excludeGitIgnore`, matching the explorer code at `src/vs/workbench/contrib/files/browser/views/explorerViewer.ts:600`).
