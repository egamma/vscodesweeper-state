# Issue #199846 — Auto indent for Ruby language's block is not performed If a string include "#" is used as the method argument.

[https://github.com/microsoft/vscode/issues/199846](https://github.com/microsoft/vscode/issues/199846)

## Summary

Still reproduces on main: the Ruby increaseIndentPattern regex treats '#' inside a string literal as a comment marker, preventing the `do` keyword from triggering auto-indent. The `tree-sitter-autoindent-candidate` label is appropriate — regex alone cannot reliably solve this.

## Change summary

Ruby auto-indentation fails for lines like `method('#foo') do` because the `#` inside the string is mistaken for a comment by the indentation regex.

## Best solution

A proper fix requires tree-sitter-based indentation (already labeled). A partial regex workaround could try to skip quoted `#` but would remain fragile. Long-term, tree-sitter auto-indent for Ruby would resolve this and similar string-vs-comment ambiguity issues.
