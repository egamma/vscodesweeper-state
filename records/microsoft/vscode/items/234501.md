# Issue #234501 — The font of global search bar and search bar should be a monospace font.

[https://github.com/microsoft/vscode/issues/234501](https://github.com/microsoft/vscode/issues/234501)

## Summary

This is a feature request (mislabeled as bug) to use the editor's monospace font in search input boxes and results. The current behavior — using the workbench/system font — is a deliberate, consistent design choice across all VS Code input widgets. The request has merit (search content is code) but limited traction (2 upvotes in 600+ days).

## Change summary

Use a monospace (editor) font in the search bar inputs and search results instead of the default workbench font.

## Best solution

Reclassify as `feature-request`. If pursued, this would likely be a new setting (e.g. `search.useEditorFont`) that applies the editor font family to the search widget inputs and result highlights. The search view CSS lives in `src/vs/workbench/contrib/search/browser/media/searchview.css` and the find widget in `src/vs/editor/contrib/find/`. Given low community traction, let the community process decide its fate.
