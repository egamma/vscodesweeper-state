# Issue #312572 — Auto-detect language mode fails with LICENSE file without extension

[https://github.com/microsoft/vscode/issues/312572](https://github.com/microsoft/vscode/issues/312572)

## Summary

Valid bug — the ML-based language detection (via TensorFlow model in the language detection worker) can misclassify extensionless plain-text files like LICENSE as Markdown. Already properly triaged with correct area label and milestone.

## Change summary

Opening a LICENSE file without an extension incorrectly auto-detects the language mode as Markdown instead of plain text.

## Best solution

Add well-known extensionless filenames (LICENSE, LICENCE, COPYING, AUTHORS, etc.) to a plaintext allowlist in the language detection pipeline, or bias the ML model output away from Markdown for these filenames. The fix likely lives in `src/vs/workbench/services/languageDetection/browser/languageDetectionWorkerServiceImpl.ts` where biases are computed before calling `$detectLanguage`.
