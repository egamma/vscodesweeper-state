# Issue #318217 — [Error] unhandlederror-Model is disposed!

[https://github.com/microsoft/vscode/issues/318217](https://github.com/microsoft/vscode/issues/318217)

## Summary

Still reproduces on main: the `_inlineSuggestionItems` derived observable in `inlineCompletionsModel.ts` calls `completion.isVisible(this.textModel, …)` without checking whether the textModel is disposed, causing the 'Model is disposed!' error during observable recomputation after model disposal. An open PR (#318218) proposes the fix but has not yet merged.

## Change summary

Inline completions model accesses a disposed TextModel during observable recomputation, throwing 'Model is disposed!' — a 4.2x spike in error telemetry on 1.121.0.

## Best solution

Merge the guard in the open PR (https://github.com/microsoft/vscode/pull/318218) adding an early `if (this.textModel.isDisposed()) return undefined` at the top of the `_inlineSuggestionItems` derived computation, preventing visibility checks on a disposed model. @hediet should review for correctness within the observable lifecycle.
