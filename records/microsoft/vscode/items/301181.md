# Issue #301181 — Distribute Skills and Instructions from Organization Repositories

[https://github.com/microsoft/vscode/issues/301181](https://github.com/microsoft/vscode/issues/301181)

## Summary

Partially addressed: org instruction sync already exists (commit e8a3ddf7c72), but org-level skill distribution is not yet implemented. The infrastructure (PromptsType.skill exists, caching service is generic) is in place but the org sync path doesn't handle skills yet. Keeping open as a valid feature request for the remaining gap.

## Change summary

Request to distribute Skills (SKILL.md) and Instructions from organization repositories to local VS Code environments, similar to how agents are distributed.

## Best solution

Extend `githubOrgChatResourcesService.ts` to handle `PromptsType.skill` in `getCacheSubdirectory` and `isValidFile`, and add a `GitHubOrgSkillsProvider` analogous to `GitHubOrgInstructionsProvider`. The backend API (`getOrgCustomSkills` or similar) would also need to exist. This belongs in the Copilot extension (`extensions/copilot/src/extension/agents/vscode-node/`).
