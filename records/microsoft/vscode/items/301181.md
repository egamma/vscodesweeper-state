# Issue #301181 — Distribute Skills and Instructions from Organization Repositories

[https://github.com/microsoft/vscode/issues/301181](https://github.com/microsoft/vscode/issues/301181)

## Summary

Valid feature request for extending organization-level Copilot customization sync to include skills and instructions alongside agents. The codebase already has bundling infrastructure for skills/instructions (syncedCustomizationBundler.ts), but the org-level discovery/distribution mechanism is likely owned by the Copilot service or extension, not VS Code core.

## Change summary

Organizations should be able to centrally distribute Skills and Instructions (not just agents) from org repositories to all team members' VS Code environments automatically.

## Best solution

The org-level sync mechanism (likely in the Copilot Chat extension or service) should be extended to discover and distribute `.github/skills/` and `.github/instructions/` alongside `.github/agents/`. The VS Code-side bundling infrastructure in `syncedCustomizationBundler.ts` already supports all prompt types including skills and instructions, so the gap is on the discovery/distribution side. This may belong in the Copilot extension or GitHub service rather than VS Code core.
