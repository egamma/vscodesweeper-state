# Issue #326286 — Long commands are truncated

[https://github.com/microsoft/vscode/issues/326286](https://github.com/microsoft/vscode/issues/326286)

## Summary

The truncated command display in the terminal tool progress header is by design (`MAX_COMMAND_TITLE_LENGTH = 50` in `chatTerminalToolProgressPart.ts`), and the confirmation dialog already shows the full command text. However, the user's request to fully inspect commands — especially those running outside the sandbox — is a reasonable UX improvement. Routing to the chat-terminal area for consideration.

## Change summary

Long terminal commands displayed in the agent chat are truncated in the collapsible header, making it difficult to fully inspect the command the model intends to execute.

## Best solution

Consider adding a hover/tooltip on the truncated command header that shows the full command text, or allow the expanded state to reveal the full command in the title. The confirmation dialog already shows the full command, but post-confirmation visibility could be improved. The relevant constant is `MAX_COMMAND_TITLE_LENGTH` (line 78) and the truncation logic is at line 547 in `src/vs/workbench/contrib/chat/browser/widget/chatContentParts/toolInvocationParts/chatTerminalToolProgressPart.ts`.
