# Issue #317313 — Agents window: still prompted to commit and delegate even after selecting a new cloud GitHub repository

[https://github.com/microsoft/vscode/issues/317313](https://github.com/microsoft/vscode/issues/317313)

## Summary

Team-authored issue (MEMBER) — must keep open. The bug is real: when the user switches to an unrelated cloud GitHub repository in the Agents window, the uncommitted-changes prompt still fires because `promptForUncommittedChangesAction` checks the local repository unconditionally, without comparing it to the selected cloud target.

## Change summary

Switching to an unrelated cloud GitHub repository in the Agents window still triggers the commit-and-delegate prompt for uncommitted changes in the locally opened folder.

## Best solution

In `folderRepositoryManagerImpl.ts`, the worktree-setup path should skip the `promptForUncommittedChangesAction` call when the selected cloud repository is different from the local folder's repository — the uncommitted changes in the local repo are irrelevant to the cloud agent's target.
