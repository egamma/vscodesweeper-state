# Issue #151247 — SCM view doesn't maintain highlight after reload

[https://github.com/microsoft/vscode/issues/151247](https://github.com/microsoft/vscode/issues/151247)

## Summary

Team-authored issue (MEMBER); the SCM view's auto-reveal on reload likely still has a timing gap where the active editor change fires before SCM resource groups are populated. No fixing commit found on main.

## Change summary

After window reload, the SCM view does not highlight the file corresponding to the active editor until the user switches tabs.

## Best solution

In `src/vs/workbench/contrib/scm/browser/scmViewPane.ts`, ensure `onDidActiveEditorChange` is re-triggered after SCM repositories and their resource groups finish populating post-reload — the current call in `onDidChangeVisibleRepositories` (line 1847) fires before resources are loaded. The Explorer avoids this because its tree state is restored from storage.
