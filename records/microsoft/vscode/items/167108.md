# Issue #167108 — Enable debugging Code OSS with "break on caught exceptions" enabled

[https://github.com/microsoft/vscode/issues/167108](https://github.com/microsoft/vscode/issues/167108)

## Summary

Team-authored engineering debt issue. The `ExpectedError` class exists in `src/vs/base/common/errors.ts` with `isExpected = true`, but the broader systematic adoption (lint rules, converting existing expected-throw sites, debugger condition documentation) proposed here remains incomplete — only ~10 files use it.

## Change summary

Mark all intentionally-thrown errors with an `expected` property so that debugging with 'break on caught exceptions' becomes practical by filtering them out.

## Best solution

Incrementally adopt `ExpectedError` (or a lint-enforced base class hierarchy like `ExpectedError`/`BugIndicatingError`) across the codebase's known expected-throw sites, and document the recommended debugger condition `!(typeof error === 'object' && error && error.isExpected)` in the contributing guide.
