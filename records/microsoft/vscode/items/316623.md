# Issue #316623 — Hide minimap when holding hotkey for go-to-definition by mouse

[https://github.com/microsoft/vscode/issues/316623](https://github.com/microsoft/vscode/issues/316623)

## Summary

Keep open and route to the editor area as a focused minimap interaction feature request. The report describes a real usability gap with `editor.minimap.autohide: "mouseover"`, but there is no duplicate, maintainer decision, or concrete fixing commit in the provided context or current checkout.

## Change summary

Suppress the hover-autohide minimap while the go-to-definition mouse modifier is held so it does not cover edge-of-editor symbols before Ctrl/Cmd-click.

## Best solution

Have the editor/minimap owners decide whether modifier-key navigation should temporarily disable the minimap hover reveal, likely in the minimap view part or its interaction with the go-to-definition click gesture.

## Evidence

- **issue body:** The reporter uses `"editor.minimap.autohide": "mouseover"` and describes the minimap appearing under the cursor while Ctrl-clicking a symbol near the right edge, preventing the intended go-to-definition click.
- **current implementation:** `src/vs/editor/common/config/editorOptions.ts` defines `editor.minimap.autohide` as `none`, `mouseover`, or `scroll`; there is no option or documented behavior for suppressing hover reveal during navigation modifiers.
- **minimap behavior:** `src/vs/editor/browser/viewParts/minimap/minimap.css` reveals `.minimap-autohide-mouseover` on `:hover`, while `src/vs/editor/browser/viewParts/minimap/minimap.ts` applies that class from the minimap option.
- **navigation gesture:** `src/vs/editor/contrib/gotoSymbol/browser/link/clickLinkGesture.ts` separately tracks the Ctrl/Cmd/Alt trigger modifiers for go-to-definition, with no connection found to minimap visibility.
