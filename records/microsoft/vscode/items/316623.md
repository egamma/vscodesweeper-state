# Issue #316623 — Hide minimap when holding hotkey for go-to-definition by mouse

[https://github.com/microsoft/vscode/issues/316623](https://github.com/microsoft/vscode/issues/316623)

## Summary

Valid feature request: suppress minimap autohide reveal when a modifier key (Ctrl/Cmd) is held for go-to-definition. No existing logic addresses this — the autohide is purely CSS-based (:hover). Keeping open for community traction.

## Change summary

When holding Ctrl for go-to-definition, the autohiding minimap should stay hidden so it doesn't obscure clickable symbols near the right edge.

## Best solution

Add modifier-key awareness to the minimap autohide logic in `src/vs/editor/browser/viewParts/minimap/minimap.ts` — when the goto-definition modifier is held, suppress the hover-reveal by toggling the CSS class or adding a pointer-events gate. Alternatively, this could be generalized to suppress autohide whenever the editor detects a pending link-click gesture.
