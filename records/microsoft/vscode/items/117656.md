# Issue #117656 — Issue reporter stops me from posting issues if my auth session is invalid

[https://github.com/microsoft/vscode/issues/117656](https://github.com/microsoft/vscode/issues/117656)

## Summary

Still present on main: when a GitHub token becomes invalid mid-session, `submitToGitHub` silently returns `false` on 403 with only a console error — no user-facing feedback or retry mechanism. The issue reporter just reverts the button state without explaining the failure.

## Change summary

Issue reporter silently fails to post when the GitHub auth session becomes invalid, with no user-facing error or option to re-authenticate.

## Best solution

When `submitToGitHub` receives a 401/403, show a user-visible error notification and either prompt re-authentication or fall back to opening the issue in the browser (Preview on GitHub path). The fix belongs in `src/vs/workbench/contrib/issue/browser/baseIssueReporterService.ts` and the electron-browser counterpart.
