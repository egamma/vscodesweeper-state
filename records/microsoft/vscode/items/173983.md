# Issue #173983 — Selecting text highlighting in large files silently drops highlighting of occurrences

[https://github.com/microsoft/vscode/issues/173983](https://github.com/microsoft/vscode/issues/173983)

## Summary

The 999-match limit in `LIMIT_FIND_COUNT` is still present on main and intentional for performance. The real gap is lack of user feedback when highlights are silently truncated — a valid UX improvement request that hasn't been addressed.

## Change summary

Selection-based occurrence highlighting silently drops matches beyond 999 in large files with no user notification.

## Best solution

Add a visible indicator (e.g. status bar or inline hint like '999+ matches') when the selection-highlight limit is reached, similar to how Find shows match counts. The logic lives in `src/vs/editor/contrib/wordHighlighter/browser/` and the limit is defined in `src/vs/editor/common/model/textModel.ts`.
