# Issue #131062 — expose only the positions of those brackets colored by native bracket pair colorization

[https://github.com/microsoft/vscode/issues/131062](https://github.com/microsoft/vscode/issues/131062)

## Summary

This is an unimplemented feature request from a team member to expose bracket pair positions to extensions via an async API. The internal bracket-pair data exists but no extension API surfaces it; the request remains valid and unfulfilled.

## Change summary

Expose an extension API to query bracket pair positions (computed by the native bracket pair colorizer) within a given range.

## Best solution

Add a proposed extension API (e.g. `vscode.languages.getBracketPairs(document, range)`) backed by the existing `BracketPairsTextModelPart.getBracketPairsInRange` infrastructure. The async consistency problem hediet raised (text may change between request and response) would need a version/sequence-number guard. Related to https://github.com/microsoft/vscode/issues/177452 but narrower in scope — bracket positions only, not full AST tokens.
