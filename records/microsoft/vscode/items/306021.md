# Issue #306021 — Copilot Continue back to Local missing

[https://github.com/microsoft/vscode/issues/306021](https://github.com/microsoft/vscode/issues/306021)

## Summary

This is a feature request for a 'Continue back to Local' option after a CLI/remote agent session completes. The current 'Continue in...' infrastructure supports delegating from local to remote targets but not the reverse direction. Routing to the chat area for triage.

## Change summary

After a Copilot CLI agent session finishes (with a worktree), provide an option to continue the conversation back in the local editor chat.

## Best solution

The chat 'Continue in...' action infrastructure (chatContinueInAction.ts) would need a reverse-delegation path — from a completed remote/CLI session back to a local chat session, carrying over the conversation transcript. This is a product decision for the agent sessions team.
