# Issue #81029 — SCM Input - Display current line length for source control commit message input

[https://github.com/microsoft/vscode/issues/81029](https://github.com/microsoft/vscode/issues/81029)

## Summary

Valid, unimplemented feature request. The SCM input editor does not surface cursor position (Ln/Col) in the status bar because it is not an editor pane. Issue #211360 was closed as a duplicate of this canonical issue. Keeping open with its existing `help wanted` label.

## Change summary

Display current line/column position (or line length) when typing in the SCM commit message input to help users follow the 50/72 commit message convention.

## Best solution

Show the cursor position (Ln/Col) in the status bar or inline in the SCM view when the SCM input editor is focused, similar to how it works for regular editor tabs. Implementation would likely touch `src/vs/workbench/contrib/scm/browser/scmInput.ts` and possibly the editor status bar contribution. The `help wanted` label is appropriate — a community contributor could implement this.
