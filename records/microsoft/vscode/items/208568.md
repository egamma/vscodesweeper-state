# Issue #208568 — Clear automatically inserted repro steps

[https://github.com/microsoft/vscode/issues/208568](https://github.com/microsoft/vscode/issues/208568)

## Summary

The bug is still present on main: `applyExtensionIssueData` in `issueReporterOverlay.ts` appends new extension issue body text to the description textarea without clearing previously auto-inserted content. A team member (rzhao271) raised a valid UX concern about clearing user-typed text, so the fix needs care to distinguish auto-inserted from user-typed content. Keeping open as a valid, low-priority issue.

## Change summary

When switching the target extension in the issue reporter, previously auto-inserted reproduction steps are not cleared from the description, leaving stale/irrelevant content.

## Best solution

Track which portion of the description textarea was auto-inserted by the extension (vs. user-typed) and clear only the auto-inserted portion when the selected extension changes. The fix lives in `applyExtensionIssueData` and `updateSelectedExtension` in `src/vs/workbench/contrib/issue/browser/issueReporterOverlay.ts`. The UX concern raised in the comments (clearing user-typed text on accidental re-selection) must be addressed — only extension-provided `issueBody` content should be replaced, not free-form user input.
