# Issue #196587 — Git - Allow users to still force push, or drive them to the new force push settings

[https://github.com/microsoft/vscode/issues/196587](https://github.com/microsoft/vscode/issues/196587)

## Summary

The requested improvements — better error messaging for force-push-with-lease rejection and a per-push override button — have not been implemented on main. The error message at line 5598 of commands.ts still suggests 'Pull first', which is confusing when the user explicitly chose force push. The issue remains valid and actionable.

## Change summary

Improve the force-push-with-lease rejection dialog to explain why force push was blocked and offer a per-push override instead of only suggesting 'Pull'.

## Best solution

Update the `ForcePushWithLeaseRejected` error handler in `extensions/git/src/commands.ts` to (1) explain that `--force-if-includes` / `--force-with-lease` blocked the push because the remote tip changed, and (2) offer a button to retry with plain `--force` for that single push, rather than directing users to toggle a global setting.
