# Issue #252133 — RangeError [ERR_CHILD_PROCESS_STDIO_MAXBUFFER]: stdout maxBuffer length exceeded

[https://github.com/microsoft/vscode/issues/252133](https://github.com/microsoft/vscode/issues/252133)

## Summary

Fixed on main: the maxBuffer for `git show` in the hygiene pre-commit check was increased to `Math.max(stat.size * 2, 1024 * 1024)`, addressing the edge case where `stat.size` was insufficient. This can be closed as completed.

## Change summary

The hygiene pre-commit hook throws `ERR_CHILD_PROCESS_STDIO_MAXBUFFER` when running `git show` on large staged files because the buffer was set to exactly `stat.size`.

## Best solution

Keep the shipped fix in commit 638c99fdba0 — the doubled buffer with a 1 MB floor handles the mismatch between `stat.size` and the actual `git show` output. No further change needed.
