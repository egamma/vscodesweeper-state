# Issue #317702 — Sticky Scroll doesn't handle multi-line function headers

[https://github.com/microsoft/vscode/issues/317702](https://github.com/microsoft/vscode/issues/317702)

## Summary

Valid feature request in the community voting period (3/20 upvotes, started 2026-06-12). Sticky scroll currently renders only the start line of a scope; collapsing multi-line signatures into one line would be a new capability. No related fix or duplicate found on main.

## Change summary

Sticky scroll shows only the first line of a multi-line function signature, making it unhelpful for languages with Allman-style braces or long parameter lists.

## Best solution

Enhance the sticky scroll widget to detect multi-line declaration headers (via the outline/folding model) and render a collapsed single-line representation. Likely touches `stickyScrollWidget.ts` (rendering) and `stickyScrollModelProvider.ts` (range computation). Let the community voting process complete before investing effort.
