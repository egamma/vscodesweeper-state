# Issue #214070 — Standardize UI Navigation with `workbench.action.navigate`

[https://github.com/microsoft/vscode/issues/214070](https://github.com/microsoft/vscode/issues/214070)

## Summary

Valid feature request with `help wanted` label from a team member, asking for navigate commands to work within sidebar panes — not just between major parts. The navigation actions on main still only handle part-to-part movement, so the gap remains. Keeping open as accepted community contribution candidate.

## Change summary

Extend `workbench.action.navigateUp/Down` to move focus between individual view panes within sidebars, not just between major workbench parts.

## Best solution

Extend `BaseNavigationAction` in `src/vs/workbench/browser/actions/navigationActions.ts` to detect when focus is inside a sidebar and navigate between view panes (e.g., Explorer → Outline) using the pane composite's internal view list, rather than only delegating to `getVisibleNeighborPart`. The `help wanted` label invites a community contribution.
