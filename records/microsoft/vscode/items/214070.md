# Issue #214070 — Standardize UI Navigation with `workbench.action.navigate`

[https://github.com/microsoft/vscode/issues/214070](https://github.com/microsoft/vscode/issues/214070)

## Summary

Valid feature request: the existing `workbench.action.navigate*` commands only move between major workbench parts (editor, sidebar, panel, auxiliary bar) but do not navigate between individual views within a sidebar or between sub-elements within a pane. The gap remains on current main. Already triaged with `help wanted` in Backlog — keeping open.

## Change summary

Extend `workbench.action.navigate*` commands to move focus between individual views within the sidebar/panel and between sub-elements (e.g., input fields to result lists) within a pane.

## Best solution

Enhance `BaseNavigationAction` in `src/vs/workbench/browser/actions/navigationActions.ts` so that when focus is already inside a sidebar/panel part, Up/Down navigates between the individual view panes within that composite (and optionally into the activity bar). This likely involves integrating with the pane composite's view container layout to identify the next/previous focusable view. The `help wanted` label is appropriate — a community contributor could scope this incrementally.
