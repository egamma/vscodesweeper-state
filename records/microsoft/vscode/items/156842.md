# Issue #156842 — Restarting a Debug Adapter doesn't reset the Visual Studio Interface

[https://github.com/microsoft/vscode/issues/156842](https://github.com/microsoft/vscode/issues/156842)

## Summary

Still reproducible on main: when a debug adapter sets `supportsRestartRequest`, the restart path in `debugService.ts` sends the DAP restart request but never clears threads/variables, leaving stale UI in the Variables and Call Stack views.

## Change summary

Restarting a debug session via DAP's restart request does not clear the Variables and Call Stack views, leaving stale data from the previous run.

## Best solution

In the `supportsRestartRequest` branch of `restartSession()` in `debugService.ts`, call `session.clearThreads()` (or `model.clearThreads()`) before or after the restart request so the Variables and Call Stack views reset, matching the behavior of the non-restart path which terminates and relaunches the session.
