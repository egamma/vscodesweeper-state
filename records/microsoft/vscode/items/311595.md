# Issue #311595 — Add `requestTimeout` option for MCP server tool calls

[https://github.com/microsoft/vscode/issues/311595](https://github.com/microsoft/vscode/issues/311595)

## Summary

Valid feature request — no `requestTimeout` configuration exists in the MCP server implementation on current main. The tool call path (`mcpServerRequestHandler.ts` → `callTool`) has no per-server timeout mechanism; calls rely solely on user-initiated cancellation.

## Change summary

Allow configuring a per-server request timeout for MCP tool calls so slow/unresponsive servers fail fast instead of blocking indefinitely.

## Best solution

Add a `requestTimeout` (or similar) field to MCP server configuration in `mcp.json`, and wire it into `McpServerRequestHandler.callTool()` so the CancellationToken is automatically triggered after the configured duration. This is a reasonable core feature since it affects all MCP servers uniformly and doesn't require extension-API surface.
