# Issue #317827 — Agent window: other modes should be disabled in sub agent if it is not supported

[https://github.com/microsoft/vscode/issues/317827](https://github.com/microsoft/vscode/issues/317827)

## Summary

Team-authored issue — keeping open per policy. The mode picker in sub-agent sessions within the Agents window shows modes (Local, Claude) that aren't supported by the sub-agent's provider, leading to a no-op selection. The `ModePickerModel.getAvailableModes()` in `src/vs/sessions/contrib/providers/copilotChatSessions/browser/modePicker.ts` does not filter modes based on sub-agent provider capabilities.

## Change summary

Mode picker dropdown in sub-agent sessions shows unsupported modes (e.g. Local, Claude) that cannot be selected, instead of hiding or disabling them.

## Best solution

Filter or disable unsupported modes in the sub-agent session's mode picker. The fix likely belongs in `ModePickerModel.getAvailableModes()` (src/vs/sessions/contrib/providers/copilotChatSessions/browser/modePicker.ts), which should check the session's provider capabilities to exclude modes the sub-agent cannot use, or the picker UI should disable items that are unsupported for the current session context.
