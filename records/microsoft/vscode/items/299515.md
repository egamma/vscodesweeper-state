# Issue #299515 — Cmd+C with no selection leaks bare "c" into TUI input on 1.110.0 (selection path works)

[https://github.com/microsoft/vscode/issues/299515](https://github.com/microsoft/vscode/issues/299515)

## Summary

Fixed on main and shipped in 1.113.0. The kitty keyboard protocol was consuming Meta-modified keys via CSI u sequences; the fix intercepts Meta keybindings before xterm.js when kitty keyboard is active.

## Change summary

Cmd+C with no selection leaks a bare 'c' character into TUI input when the kitty keyboard protocol is enabled in the integrated terminal on macOS.

## Best solution

Keep the shipped fix (commit 7b17c65f78d, 1.113.0). The terminal custom key event handler now intercepts all Meta-modified keys that resolve to a VS Code command, preventing xterm.js from encoding them as CSI u sequences. No further change needed.
