# Issue #176665 — No diagnostics for the settings JSON split editor

[https://github.com/microsoft/vscode/issues/176665](https://github.com/microsoft/vscode/issues/176665)

## Summary

Still reproduces on main. The split settings JSON editor uses a SideBySideEditorInput with two different resources (default settings + user settings), so the Tab API in mainThreadEditorTabs.ts returns UnknownInput, preventing the JSON language server from discovering the document URI and pulling diagnostics. Team-authored issue on the On Deck milestone — keeping open.

## Change summary

When `workbench.settings.openDefaultSettings` is true, the settings JSON split editor shows no syntax-error squiggles because the Tab API cannot expose the document URI of a SideBySideEditorInput with mismatched resources.

## Best solution

Extend the SideBySideEditorInput handling in `src/vs/workbench/api/browser/mainThreadEditorTabs.ts` (around line 128) to expose both the primary and secondary resource URIs (or at least the editable primary) as a new `TabInputKind` (e.g. `TabInputSideBySide`), so extensions like the JSON language server can discover documents in split editors and pull diagnostics. The extension side (`vscode-json-languageservice` / `vscode.d.ts` Tab API types) would also need updating.
