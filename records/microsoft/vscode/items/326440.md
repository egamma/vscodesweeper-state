# Issue #326440 — AHP initialize failed: rpc error -32005: Client offered protocol versions [0.4.0], none of which are compatible with this server's version 0.6.0 (server accepts ^0.6.0)

[https://github.com/microsoft/vscode/issues/326440](https://github.com/microsoft/vscode/issues/326440)

## Summary

Real bug: the CLI client uses ahp crate 0.4.0 (protocol version 0.4.0) while the agent host server in the same release speaks protocol 0.6.0; the version mismatch causes `code agent ps` to fail. On current main, the server is at 0.5.1 and the CLI is still at 0.4.0 — the gap persists.

## Change summary

`code agent ps` fails with a protocol version incompatibility error because the CLI client offers protocol 0.4.0 but the agent host server requires ^0.6.0.

## Best solution

Bump the `ahp` and `ahp-types` crate dependencies in `cli/Cargo.toml` to match (or be compatible with) the protocol version the agent host server speaks, so the CLI and server shipped in the same release can negotiate successfully.
