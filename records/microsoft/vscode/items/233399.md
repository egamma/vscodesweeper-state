# Issue #233399 — Vscode does not open links in default browser

[https://github.com/microsoft/vscode/issues/233399](https://github.com/microsoft/vscode/issues/233399)

## Summary

The root cause is in the snap sandbox's interaction with xdg-open and Electron's shell.openExternal — VS Code already mitigates snap env vars (unsetSnapExportedVariables/restoreSnapExportedVariables in environmentMainService.ts) before calling shell.openExternal, but the snap-provided xdg-open still resolves desktop files differently than the host system. A team member (deepak1556) has engaged; keeping open as an upstream/snap confinement issue.

## Change summary

On Linux snap installs, Ctrl+clicking links opens the wrong browser instead of the user's configured default, because the snap sandbox affects how xdg-open resolves default applications.

## Best solution

Investigate whether the snap-provided xdg-open can be bypassed (e.g. calling the host xdg-open via a portal or using xdg-desktop-portal D-Bus interface directly). The existing mitigation in `src/vs/platform/environment/electron-main/environmentMainService.ts` (unsetting snap-exported env vars) is insufficient because the snap sandbox also confines filesystem access to desktop files. A complete fix likely requires upstream changes in snapd or Electron's xdg-open handling. Related to https://github.com/microsoft/vscode/issues/92902.
