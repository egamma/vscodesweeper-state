# Issue #303097 — Respect Per-Prompt `model` Declaration in Nested / Chained Prompt Calls

[https://github.com/microsoft/vscode/issues/303097](https://github.com/microsoft/vscode/issues/303097)

## Summary

Legitimate feature request to honour per-prompt `model` declarations during nested/chained prompt execution. The `model` field already exists in `.prompt.md` front-matter and is parsed by the prompt infrastructure, but the request is about runtime model routing for sub-prompts. No duplicates found; 0 upvotes after 121 days.

## Change summary

When a `.prompt.md` is invoked as a sub-prompt from another prompt or agent, its own `model` front-matter field should determine the model used for that sub-prompt's execution, rather than inheriting the parent's model.

## Best solution

Route to the chat/copilot customization area for product evaluation. The prompt infrastructure (`src/vs/workbench/contrib/chat/common/promptSyntax/`) already parses and stores the `model` field; the gap is in the execution path where the model selection for nested prompt calls should consult each prompt's declared model. This needs a product decision on the inheritance/override semantics before implementation.
