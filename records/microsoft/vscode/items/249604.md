# Issue #249604 — NES/Inline Completion - Get more information on show

[https://github.com/microsoft/vscode/issues/249604](https://github.com/microsoft/vscode/issues/249604)

## Summary

Valid feature request to enrich the inline completion `handleDidShowCompletionItem` callback with view context (view name, visible ranges, applied edits, cursor). The current proposed API only passes the item and updated insert text — this gap remains on main.

## Change summary

Enrich the inline completion show callback with view type, visible ranges, rendered edits, and cursor position so providers can distinguish meaningful visibility from incidental scroll.

## Best solution

Extend the proposed `inlineCompletionsAdditions` API to pass a structured info object (view kind, visible ranges, decoration edits, cursor) to `handleDidShowCompletionItem`. The work lives in `src/vscode-dts/vscode.proposed.inlineCompletionsAdditions.d.ts` and the corresponding ext-host adapter in `src/vs/workbench/api/common/extHostLanguageFeatures.ts`. The contributor offered to implement it — a maintainer decision on the shape of the info object is needed.
