# Issue #292266 — code freezes when trying to manage a repo folder 

[https://github.com/microsoft/vscode/issues/292266](https://github.com/microsoft/vscode/issues/292266)

## Summary

The reporter asks VS Code to only update repos with open files — this is already provided by the existing `git.autoRepositoryDetection` setting with the `openEditors` value. The issue reads as a support question rather than a missing feature or bug.

## Change summary

VS Code freezes and the SCM view is slow when a workspace contains ~200 git repositories; the user wants repo detection limited to repos with open files.

## Best solution

Point the reporter to `git.autoRepositoryDetection: "openEditors"`, which does exactly what they asked for — VS Code will only detect and track repositories that have files open in editors, ignoring the other ~200 repos. The `git.repositoryScanMaxDepth` setting (default 1) can also help limit discovery depth.
