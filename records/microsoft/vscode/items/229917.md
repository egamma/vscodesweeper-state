# Issue #229917 — SCM Graph - allow to multi select for e.g. cherry-pick

[https://github.com/microsoft/vscode/issues/229917](https://github.com/microsoft/vscode/issues/229917)

## Summary

Still a valid feature request. While multi-select infrastructure exists in the SCM graph (commit 0c0837460fc for view-commit range comparison) and in the repositories pane (commit d1068b5f94d for branch/tag/stash bulk operations), the graph cherry-pick command (`git.graph.cherryPick`) still accepts only a single history item — bulk cherry-pick from a multi-selection is not yet implemented.

## Change summary

Allow multi-selecting commits in the SCM Graph view to perform bulk operations like cherry-picking multiple commits at once.

## Best solution

Extend `git.graph.cherryPick` in `extensions/git/src/commands.ts` to accept an array of `SourceControlHistoryItem` (similar to how `d1068b5f94d` updated artifact commands), and wire the `HistoryItemActionRunner2` to pass all selected items. The graph's multi-select plumbing already exists via `HistoryItemActionRunner2` but currently only passes first+last; it would need to pass all selected items for cherry-pick.
