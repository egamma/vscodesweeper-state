# Issue #244211 — Spurious configuration addon overlay creating fresh launch.json

[https://github.com/microsoft/vscode/issues/244211](https://github.com/microsoft/vscode/issues/244211)

## Summary

Still reproduces on current main. When a user clicks 'create a launch.json file' from the debug welcome view, the command passes `{ addNew: true }`, which after creating the file immediately calls `addLaunchConfiguration()` — this triggers IntelliSense suggestions to add another debug configuration, overlaying the just-created file. The user considers this unsolicited and confusing.

## Change summary

After creating a fresh launch.json via the debug welcome view link, an IntelliSense overlay immediately appears prompting the user to add another debugger configuration, which is unsolicited and hides the newly created content.

## Best solution

Consider removing the `{ addNew: true }` argument from the welcome view's 'create a launch.json' link (in `src/vs/workbench/contrib/debug/browser/welcomeView.ts`), or skipping the `addLaunchConfiguration()` call when the file was freshly created (the `created` flag from `openConfigFile` is available). The file already contains an initial configuration from the selected debugger, so triggering suggest for a second one is arguably premature.
