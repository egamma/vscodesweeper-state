# Issue #289294 — Wrong diff is shown after git pull during session

[https://github.com/microsoft/vscode/issues/289294](https://github.com/microsoft/vscode/issues/289294)

## Summary

Team-authored issue reporting incorrect diff attribution after git pull during a chat agent session — must stay open per keep-open rules for MEMBER authors.

## Change summary

After a git pull during a chat editing session, the edit session diff incorrectly includes all changes from the pull as green lines mixed with the actual chat edits.

## Best solution

The diff baseline for agent edit sessions should be updated (or changes rebased) when external SCM operations like git pull modify files the agent has touched, so that only the agent's actual edits are attributed in the session diff.
