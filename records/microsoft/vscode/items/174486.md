# Issue #174486 — VS Code reports incorrect size of the terminal until the terminal is manually resized

[https://github.com/microsoft/vscode/issues/174486](https://github.com/microsoft/vscode/issues/174486)

## Summary

The bug is a race condition where terminal dimensions are computed before the container has its final layout on restore, causing the shell to receive incorrect COLUMNS/LINES values until a manual resize triggers a re-layout. The issue remains unfixed on main and is correctly categorized with `help wanted`.

## Change summary

Terminal reports incorrect COLUMNS/LINES to the shell on session restore until manually resized.

## Best solution

Fix the race in `terminalInstance.ts` by deferring the initial `createProcess` dimensions until a layout pass confirms the container has non-fallback dimensions, or by sending an explicit resize event after the first real layout completes. The `_initDimensions` fallback path (lines 697-704) and the `createProcess` call (line 1613) in `src/vs/workbench/contrib/terminal/browser/terminalInstance.ts` are the likely sites.
