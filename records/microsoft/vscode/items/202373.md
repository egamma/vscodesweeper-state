# Issue #202373 — Cursor position is incorrect in selection when cursorStyle is "Block"

[https://github.com/microsoft/vscode/issues/202373](https://github.com/microsoft/vscode/issues/202373)

## Summary

This is a long-standing request for proper modal-editor block cursor semantics during selections — previously tracked as a feature request in #97148 and #2997 (both closed via community upvote process). No fix has landed on main. The current issue reframes it as a bug but the behavior is unchanged by design; keeping open for the community process.

## Change summary

Block cursor visually sits one character past the selection end rather than on the last selected character, causing a mismatch with `selection.active`.

## Best solution

This is realistically a feature request for proper modal-editor cursor semantics (block cursor covering the last selected character). The canonical discussion is https://github.com/microsoft/vscode/issues/114851 (open, labeled `feature-request` + `VIM`). This issue could be consolidated there, or kept as a standalone feature request. The fix would involve the cursor rendering logic in `src/vs/editor/browser/viewParts/viewCursors/viewCursor.ts` and potentially cursor position model code, but it requires a product decision on whether to change the block cursor's relationship to selection boundaries.
