# Issue #50692 — QuickPick API is no longer type safe

[https://github.com/microsoft/vscode/issues/50692](https://github.com/microsoft/vscode/issues/50692)

## Summary

The type-safety issue is resolved on current main: `showQuickPick` has overloads using `QuickPickOptions & { canPickMany: true }` that return `T[]` when the literal `canPickMany: true` is passed, giving the correct return type at compile time.

## Change summary

The `showQuickPick` API did not narrow its return type to an array when `canPickMany: true` was specified in the options.

## Best solution

The shipped overloaded signatures (intersection type with `{ canPickMany: true }`) already solve the practical type-safety problem. The remaining edge case (pre-typed `QuickPickOptions` variable) is an inherent TypeScript narrowing limitation, not actionable debt. Close as completed.
