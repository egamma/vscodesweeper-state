GPU rendering corruption on Wayland/Intel — likely upstream (Chromium/Electron), but key diagnostics are missing: the reporter hasn't tested with --disable-gpu or confirmed --disable-extensions. Needs info before routing.

## Change summary

## Best solution

Confirm whether --disable-gpu resolves the corruption. If it does, this is an upstream Chromium/Electron GPU compositing issue on Wayland and should be labeled `upstream` + `gpu` + `linux`. The workaround is launching with --disable-gpu or setting `"disable-hardware-acceleration": true` in argv.json. If it persists without GPU acceleration, deeper investigation into the list/tree virtualization in `src/vs/base/browser/ui/list/` would be warranted.
