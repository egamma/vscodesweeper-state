# Issue #317203 — Agents App: Show what model the subagents are using when you hover on it

[https://github.com/microsoft/vscode/issues/317203](https://github.com/microsoft/vscode/issues/317203)

## Summary

Valid feature request: the Agents window's running-subagents chip (SessionRunningSubagentsControl) lacks the hover tooltip that the workbench's ChatSubagentContentPart already provides, showing the model name and credits. This is a real parity gap between the two surfaces.

## Change summary

In the Agents window, hovering over a running subagent chip does not show which AI model the subagent is using, unlike the normal VS Code chat view which displays model and credits in a tooltip.

## Best solution

Add hover tooltip support to `src/vs/sessions/contrib/chat/browser/sessionRunningSubagentsControl.ts` mirroring the pattern in `src/vs/workbench/contrib/chat/browser/widget/chatContentParts/chatSubagentContentPart.ts` (the `updateHover` method at line 666). The `IChat` model in the sessions layer would need to expose the model name, then `SessionRunningSubagentsControl` can wire up `IHoverService.setupDelayedHover` on each subagent button/menu item.
