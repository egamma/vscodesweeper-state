# Issue #277288 — Terminal autocomplete menu is partially offscreen when window is narrow

[https://github.com/microsoft/vscode/issues/277288](https://github.com/microsoft/vscode/issues/277288)

## Summary

Team-authored bug, still reproducible on main: the horizontal overflow logic flips the suggest widget left of the cursor but never clamps to ≥ 0, so it can extend offscreen. Keeping open per team-member rule.

## Change summary

Terminal suggest menu extends offscreen when the window is narrow or at high zoom, because horizontal positioning lacks a lower-bound clamp.

## Best solution

In `src/vs/workbench/services/suggest/browser/simpleSuggestWidget.ts` around the horizontal positioning logic (~line 816), clamp `anchorLeft` to `Math.max(0, anchorLeft)` after the overflow-right flip, and potentially cap `width` to `bodyBox.width` when the widget simply cannot fit.
