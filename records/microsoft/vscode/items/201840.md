# Issue #201840 — Changing selection of just after text insertion messes up the undo stack [jsx]

[https://github.com/microsoft/vscode/issues/201840](https://github.com/microsoft/vscode/issues/201840)

## Summary

Still reproduces on main: linked editing's `popUndoStop()` in `_syncRanges` corrupts the undo stack when a selection change occurs between the original edit and the linked sync edit, causing spurious text to appear on undo.

## Change summary

In JSX with linked editing enabled, undoing after a text edit + selection change leaves residual text from the linked editing sync in the editor.

## Best solution

The linked editing sync in `_syncRanges` (linkedEditing.ts ~line 243) uses `popUndoStop()` to merge its mirror edit with the user's edit. When an intervening selection change or external edit occurs, this merge corrupts the undo stack. The fix likely needs to either cancel the linked sync when an external selection change intervenes, or avoid the `popUndoStop()` in that scenario. The workaround of cancelling linked editing before undo (comment 3) confirms the root cause.
