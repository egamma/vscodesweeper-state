# Issue #60851 — Open next build error file+line based on regex parsing of build output.

[https://github.com/microsoft/vscode/issues/60851](https://github.com/microsoft/vscode/issues/60851)

## Summary

The core request — a keyboard shortcut to cycle through build errors identified by problem-matcher regex parsing of terminal output, independent of language extensions — remains unimplemented on current main. Low upvote count (2) but valid use case acknowledged by team members; keeping open in Backlog.

## Change summary

Add a keyboard-driven command to cycle through build errors parsed from terminal task output via problemMatcher regex, without requiring language support extensions.

## Best solution

Implement a 'Go to Next Build Error' command in the tasks contribution that navigates markers produced by problem matchers (via TaskProblemMonitor) in their terminal-output order, or make F8 respect a task-error-only filter. The TaskProblemMonitor in `src/vs/workbench/contrib/tasks/browser/taskProblemMonitor.ts` already collects error markers from problem matchers — the missing piece is a navigation command that cycles through those specifically.
