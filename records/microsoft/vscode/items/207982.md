# Issue #207982 — There should be a prompt when clicking on the codelens(command) in the read-only editor

[https://github.com/microsoft/vscode/issues/207982](https://github.com/microsoft/vscode/issues/207982)

## Summary

Valid feature request: the built-in merge-conflict extension silently fails when codelens commands attempt to edit a read-only file — it should show a warning message. Still unaddressed on current main.

## Change summary

Clicking merge conflict codelens (Accept Current/Incoming/Both) in a read-only editor silently does nothing instead of showing an error message.

## Best solution

Add a read-only check in `extensions/merge-conflict/src/commandHandler.ts` before calling `conflict.commitEdit()` — if the document is read-only, show a warning via `vscode.window.showWarningMessage` explaining the file cannot be modified.
