# Issue #207982 — There should be a prompt when clicking on the codelens(command) in the read-only editor

[https://github.com/microsoft/vscode/issues/207982](https://github.com/microsoft/vscode/issues/207982)

## Summary

Valid feature request: the merge-conflict extension's codelens commands (Accept Current/Incoming/Both) silently fail on read-only files with no feedback. The code in `extensions/merge-conflict/src/commandHandler.ts` does not check read-only state before calling `conflict.commitEdit()`. No fix has landed, no duplicates found, and the issue already carries the correct label. Keeping open in Backlog.

## Change summary

Clicking merge-conflict codelens commands in a read-only editor silently does nothing instead of showing an error message.

## Best solution

Add a read-only guard in `extensions/merge-conflict/src/commandHandler.ts` — in the `accept()` and `acceptAll()` methods, check `editor.document.isReadonly` (or attempt the edit and catch the rejection) and show a warning via `vscode.window.showWarningMessage` when the file is read-only, so the user understands why the action had no effect.
