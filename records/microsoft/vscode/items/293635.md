# Issue #293635 — Feature Request: Allow selective addition of error problems to Copilot context

[https://github.com/microsoft/vscode/issues/293635](https://github.com/microsoft/vscode/issues/293635)

## Summary

Valid feature request to add severity-level filtering (e.g., 'Errors Only', 'Warnings Only') when attaching problems to Copilot chat context. The current implementation in markersChatContext.ts offers 'All Problems' or individual problem picks, but no severity-based grouping. Routing to the markers area for triage.

## Change summary

Allow users to attach only problems of a specific severity (errors, warnings, info) to Copilot chat context, rather than all problems or individual items.

## Best solution

Add severity-level filter picks ('All Errors', 'All Warnings', 'All Info') alongside the existing 'All Problems' pick in `src/vs/workbench/contrib/markers/browser/markersChatContext.ts`, using the existing `MarkerSeverity` flags in `IDiagnosticVariableEntryFilterData.filterSeverity`.
