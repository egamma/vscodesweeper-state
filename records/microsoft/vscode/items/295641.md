# Issue #295641 — unable to add to use chat when i click on restore checkpoint

[https://github.com/microsoft/vscode/issues/295641](https://github.com/microsoft/vscode/issues/295641)

## Summary

Already fixed on main and shipped: commit 213b27d7c3e explicitly cancels the current request when restoring a checkpoint, which is exactly what this issue requests. Shipped in 1.115.0.

## Change summary

Restoring a checkpoint while a chat request is in progress does not cancel the running request, leaving the user unable to send new prompts.

## Best solution

Keep the shipped fix (commit 213b27d7c3e, 1.115.0+); no further change needed. The restore-checkpoint action now cancels any in-flight request before restoring.
