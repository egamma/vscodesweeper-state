# Issue #90611 — Git - Git does not show gutters unless file is opened from changed files tab

[https://github.com/microsoft/vscode/issues/90611](https://github.com/microsoft/vscode/issues/90611)

## Summary

This is a duplicate of #130082 which tracks the canonical-path resolution gap. The git extension now resolves repo roots via realpath, but files opened through symlinked/differently-cased paths still mismatch — the fix requires VS Code to always open files at their canonical path, which is exactly what #130082 requests.

## Change summary

Git gutters and dirty diff decorations do not appear when a file is opened through a symlinked or case-mismatched path because the editor URI differs from the canonical path git uses.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/130082 — implementing canonical-path resolution at the platform level (the 'always open files at their canonical path' option) would resolve this class of symlink/casing gutter issues. The git extension's rootRealPath work partially mitigates but doesn't fully solve it without platform support.
