# Issue #308887 — Chat/Configure Tools/Update Tools disables OK button, unless fold tool

[https://github.com/microsoft/vscode/issues/308887](https://github.com/microsoft/vscode/issues/308887)

## Summary

Real bug in the tool picker's accept handler: after 'Update Tools' rebuilds the tree, the activeItems check in onDidAccept still finds a callback item (or its replacement in the same position), causing OK to be silently swallowed. Collapsing the bucket clears that active state, unblocking acceptance.

## Change summary

The OK button in the Configure Tools picker becomes non-functional after using 'Update Tools' on an MCP server until the user manually collapses that server's bucket.

## Best solution

In `src/vs/workbench/contrib/chat/browser/actions/chatToolPicker.ts`, after `computeItems()` rebuilds the tree following a successful tool update (around line 363), clear or reset the active item state so the callback item is no longer in `activeItems`. Alternatively, the `onDidAccept` handler should only consider a callback item 'active' if it's still present in the current tree and explicitly focused, not merely retained from a stale active-items list.
