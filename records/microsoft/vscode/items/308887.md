# Issue #308887 — Chat/Configure Tools/Update Tools disables OK button, unless fold tool

[https://github.com/microsoft/vscode/issues/308887](https://github.com/microsoft/vscode/issues/308887)

## Summary

The reported bug appears plausible based on the code in chatToolPicker.ts — the onDidAccept handler checks activeItems for callback items, and the interaction between tree item rebuilding after 'Update Tools' and the active item tracking could cause the OK button to silently fail. However, full reproduction requires an MCP server with an outdated cache state, which cannot be verified from source alone.

## Change summary

After clicking 'Update Tools' in the Configure Tools picker, the OK button stops working until the user manually collapses the updated MCP server bucket.

## Best solution

Investigate the interaction between `computeItems` rebuilding the tree after an MCP tool update and the `onDidAccept` handler's `activeItems.find(isCallbackTreeItem)` check in `src/vs/workbench/contrib/chat/browser/actions/chatToolPicker.ts`. The issue likely stems from stale activeItems or focus state after `treePicker.setItemTree()` is called during rebuild.
