# Issue #206362 — `contextualTitle` not showing on view container

[https://github.com/microsoft/vscode/issues/206362](https://github.com/microsoft/vscode/issues/206362)

## Summary

Still reproduces on main — the panel title logic in `viewPaneContainer.ts` uses the container title when a single view is merged, ignoring the view's own name and contextualTitle. Milestone is 'On Deck', so keeping open.

## Change summary

When a contributed view is the only view in a panel container, only the container title is shown — the view's name and contextualTitle are not displayed.

## Best solution

When `isViewMergedWithContainer()` is true in `getTitle()`, the title should incorporate the view's name (or at least respect `contextualTitle`). The fix belongs in `src/vs/workbench/browser/parts/views/viewPaneContainer.ts` around line 547–561, and may also need adjustments to how header visibility works when only one view remains (per the comment about losing the context menu).
