# Issue #206362 — `contextualTitle` not showing on view container

[https://github.com/microsoft/vscode/issues/206362](https://github.com/microsoft/vscode/issues/206362)

## Summary

Valid bug still present on main. The issue has milestone 'On Deck' (a scheduling decision), so it must remain open regardless. The view title logic in viewPaneContainer.ts `getTitle()` does not properly surface the view name when a single-view container is merged in the panel.

## Change summary

When a contributed view container with a single view is in the panel, only the container title is shown — the individual view name is lost.

## Best solution

Fix the title computation in `src/vs/workbench/browser/parts/views/viewPaneContainer.ts` `getTitle()` (around line 547) so that when a view is merged with its container in the panel, the view name is included in the displayed title (e.g. 'Container: View Name'). The second commenter's issue about losing the right-click menu and drag target when only one view remains in a multi-view container may need separate attention in the panel header rendering.
