# Issue #316908 — Switching to Agent mode from Plan erases prompt

[https://github.com/microsoft/vscode/issues/316908](https://github.com/microsoft/vscode/issues/316908)

## Summary

Confirmed bug: switching modes while a prompt is typed in Plan mode erases the user's input and auto-sends "Start implementation" via the plan agent handoff mechanism. Multiple users reproduced across versions up to 1.127.0, and the code path is still present on current main.

## Change summary

Manually switching from Plan to Agent mode via the mode picker erases the user's typed prompt and auto-submits "Start implementation" instead.

## Best solution

The mode switch via the mode picker should preserve the user's typed input text and should not trigger the plan agent's handoff auto-send. The handoff `send: true` flow (in `chatWidget.ts` around line 1346–1354 and `executeHandoff` around line 1404) should only fire when the user explicitly clicks the "Start Implementation" handoff button, not when they manually switch modes via the picker. Issue https://github.com/microsoft/vscode/issues/325355 is a duplicate of this one and should be consolidated here.
