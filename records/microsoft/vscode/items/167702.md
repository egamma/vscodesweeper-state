# Issue #167702 — Proper fix for line endings

[https://github.com/microsoft/vscode/issues/167702](https://github.com/microsoft/vscode/issues/167702)

## Summary

Valid feature request at the 20-upvote threshold, now in Backlog. The text buffer intentionally treats CR and CRLF as separate line endings per standard conventions; the ask is to detect and normalize the malformed CR+CR+LF sequence produced by buggy FTP servers. No fix has landed on main.

## Change summary

Detect and normalize CR+CR+LF sequences (produced by broken FTP servers) to a single line ending when opening files.

## Best solution

Keep in Backlog. If pursued, the fix would live in the text buffer's EOL detection/normalization path (pieceTreeTextBuffer). However, adding heuristic correction for malformed line endings is a product-scope decision — it risks misinterpreting intentional sequences. Consider whether this belongs as an opt-in setting or an extension that normalizes on file open.
