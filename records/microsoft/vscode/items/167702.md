# Issue #167702 — Proper fix for line endings

[https://github.com/microsoft/vscode/issues/167702](https://github.com/microsoft/vscode/issues/167702)

## Summary

Keep open: this is an accepted Backlog feature request with sufficient community upvotes, and current text-buffer code still appears to treat CR+CR+LF as two line endings rather than a recoverable malformed CRLF sequence.

## Change summary

Detect malformed CR+CR+LF line endings from FTP/text-mode transfers and offer or apply a safer normalization instead of preserving the extra blank line behavior.

## Best solution

Leave this tracked in editor-textbuffer until the editor team decides whether the text buffer should add a targeted CR+CR+LF repair heuristic on file open/save, or whether this should remain an explicit user conversion/workaround outside core.
