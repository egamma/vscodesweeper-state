# Issue #200531 — Click scroll bar by page not smooth(Editor,Workbench list )

[https://github.com/microsoft/vscode/issues/200531](https://github.com/microsoft/vscode/issues/200531)

## Summary

Keep open: current main still appears to bypass smooth scrolling for scrollbar track clicks, including the page-scroll mode described here. The existing `bug`/`editor-scrollbar` routing is appropriate, and there is no concrete fixing commit to support closing this as completed.

## Change summary

Scrollbar track clicks with `editor.scrollbar.scrollByPage` or `workbench.list.scrollByPage` should respect the corresponding smooth-scrolling setting.

## Best solution

Handle scrollbar track page-clicks in the shared scrollbar path so editor and workbench lists use the configured smooth scroll duration, while keeping slider dragging immediate.

## Evidence

- **current code:** In `src/vs/base/browser/ui/scrollbar/abstractScrollbar.ts`, `_onPointerDown` computes `getDesiredScrollPositionFromOffsetPaged` when `scrollByPage` is enabled but then calls `_setDesiredScrollPositionNow`, which uses `this._scrollable.setScrollPositionNow(...)`.
- **editor wiring:** `src/vs/editor/browser/viewParts/editorScrollbar/editorScrollbar.ts` passes `scrollByPage: scrollbar.scrollByPage` into `SmoothScrollableElement`, and `src/vs/editor/common/viewLayout/viewLayout.ts` configures the editor `Scrollable` smooth duration from `EditorOption.smoothScrolling`.
- **list wiring:** `src/vs/base/browser/ui/list/listView.ts` creates a `Scrollable` with `smoothScrollDuration: (options.smoothScrolling ?? false) ? 125 : 0` and also passes `scrollByPage: options.scrollByPage` to `SmoothScrollableElement`.
- **history check:** A path-limited `git log --since=2023-12-11` over the shared scrollbar, editor scrollbar, list view, and scrollable files did not show a targeted fix for smooth scroll-by-page track clicks.
