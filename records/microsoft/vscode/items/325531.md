# Issue #325531 — Support explorer.filesSortOrderReverse and explorer.foldersSortOrderReverse

[https://github.com/microsoft/vscode/issues/325531](https://github.com/microsoft/vscode/issues/325531)

## Summary

Valid feature request for independent sort-reversal of files vs. folders in the Explorer. VS Code already has `explorer.sortOrderReverse` (a single boolean that reverses everything uniformly); this asks for two separate toggles so folders can stay A→Z while files sort Z→A. No existing issue covers the independent-reversal granularity. Routing to the file-explorer area.

## Change summary

Allow users to independently reverse the sort direction of files and folders in the File Explorer, rather than reversing both together.

## Best solution

Add two new boolean settings (`explorer.filesSortOrderReverse` / `explorer.foldersSortOrderReverse` or a single enum/object setting) and update the comparator in `src/vs/workbench/contrib/files/browser/views/explorerViewer.ts` (around line 1466) to apply the reverse swap conditionally based on whether the two items being compared are both directories, both files, or mixed. The existing `explorer.sortOrderReverse` could become a shorthand that sets both. Needs a product decision on setting shape before implementation.
