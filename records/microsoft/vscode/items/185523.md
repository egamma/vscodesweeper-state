# Issue #185523 — Assertion Failed

[https://github.com/microsoft/vscode/issues/185523](https://github.com/microsoft/vscode/issues/185523)

## Summary

Team-authored error-telemetry issue; the assertion in DocumentRangeMap constructor still exists on main and the race condition (focus during model update causing unordered rangeMappings) appears unfixed. Keeping open per team-author policy.

## Change summary

Merge editor throws 'Assertion Failed' when the editor receives focus while the model's diff mappings are being recomputed, because rangeMappings passed to DocumentRangeMap are momentarily unordered.

## Best solution

Guard the `translateBaseRangeToInput` path in `inputCodeEditorView.ts` so it does not read diffs while they are mid-update, or make DocumentRangeMap tolerate/sort unordered mappings gracefully instead of asserting.
