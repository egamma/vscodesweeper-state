# Issue #234433 — Editor GPU: Avoid invalidating lines and clearing render data when we can avoid it

[https://github.com/microsoft/vscode/issues/234433](https://github.com/microsoft/vscode/issues/234433)

## Summary

Still valid on current main: the file (now at src/vs/editor/browser/gpu/renderStrategy/fullFileRenderStrategy.ts) still calls _invalidateAllLines() in onDecorationsChanged, onZonesChanged, onLineMappingChanged, and onThemeChanged, and retains TODOs in onLinesDeleted/onLinesInserted about shifting data instead of wholesale invalidation. The optimization work hasn't landed yet.

## Change summary

Reduce unnecessary full-line cache invalidation in the GPU renderer's fullFileRenderStrategy to avoid clearing render data when only a subset of lines are affected.

## Best solution

Refine invalidation in `onDecorationsChanged` (scope to affected lines via the event), `onZonesChanged` (scope to shifted lines), and `onLinesDeleted`/`onLinesInserted` (shift cached data rather than clearing everything after the edit point). Work lives in `src/vs/editor/browser/gpu/renderStrategy/fullFileRenderStrategy.ts`.
