# Issue #72608 — [scss] Incorrect selector specificity in hover tooltip

[https://github.com/microsoft/vscode/issues/72608](https://github.com/microsoft/vscode/issues/72608)

## Summary

Still present on main — the specificity calculation in vscode-css-languageservice's selectorPrinting.js doesn't account for SCSS parent selectors (`&`). This is an upstream issue in the vscode-css-languageservice package.

## Change summary

SCSS hover tooltip shows incorrect selector specificity (0, 0, 0) for nested selectors using the `&` parent reference.

## Best solution

Fix the `calculateScore` function in vscode-css-languageservice (https://github.com/microsoft/vscode-css-languageservice) to resolve parent selectors and include their specificity contribution when computing the total. Then bump the dependency in VS Code.
