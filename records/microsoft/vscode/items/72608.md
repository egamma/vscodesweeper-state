# Issue #72608 — [scss] Incorrect selector specificity in hover tooltip

[https://github.com/microsoft/vscode/issues/72608](https://github.com/microsoft/vscode/issues/72608)

## Summary

Still present on main — the specificity calculation in vscode-css-languageservice doesn't account for SCSS parent-referencing selectors (`&`). The fix belongs in the upstream library's `calculateScore` function.

## Change summary

SCSS selectors using `&` (parent reference) show specificity (0, 0, 0) instead of the correct computed specificity.

## Best solution

Fix the `calculateScore` function in `microsoft/vscode-css-languageservice` (`src/services/selectorPrinting.ts`) to resolve the effective selector type when encountering a `SelectorCombinator` node from SCSS `&` references, then bump the dependency in VS Code.
