# Issue #297338 — Ctrl+scroll wheel zoom jumps around

[https://github.com/microsoft/vscode/issues/297338](https://github.com/microsoft/vscode/issues/297338)

## Summary

Still open on main — the editor mouse-wheel zoom handler in mouseHandler.ts does preventDefault/stopPropagation when zooming, but doesn't anchor the scroll position to the viewport center during layout reflow from font-size changes. No fix found in git history. A team member engaged but couldn't reproduce on macOS; the reporter provided clear Windows video evidence and a second user confirmed on macOS with Logitech hardware.

## Change summary

Ctrl+scroll wheel zoom causes the editor scroll position to jump erratically, especially when zooming out from a position deep in a large file.

## Best solution

The zoom handler at src/vs/editor/browser/controller/mouseHandler.ts:159-166 changes the zoom level without anchoring the viewport scroll position. The fix likely needs to record the scroll anchor (e.g. the line at the cursor or viewport center) before zooming and restore it after the layout reflow, similar to how font-size changes are handled elsewhere. This belongs to the editor-scrollbar / zoom area owners.
