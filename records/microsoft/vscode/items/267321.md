## Summary

The original socks5h proxy bug remains unconfirmed and uninvestigated — chrmarti noted proxy protocol shouldn't matter (implying it's a bug if it does), but no fix or further investigation has landed. The thread has accumulated many unrelated reports (org accounts, enterprise restrictions) that have separate root causes. The issue still needs team confirmation of the socks5h-specific behavior.

## Change summary

The 'Manage Models...' option disappears from the Copilot model picker when VS Code's proxy is set to a socks5h:// URL, but appears correctly with socks5://.

## Best solution

Investigate whether socks5h proxy prevents entitlement resolution (causing `shouldShowManageModelsAction` to return false due to an unresolved entitlement). The likely root cause is in the network/request layer — if socks5h connections fail silently, the entitlement check defaults to `Unknown`, which is not in the allowed list in `src/vs/workbench/contrib/chat/browser/widget/input/chatModelPicker.ts:416-427`. Consider making the 'Manage Models...' action resilient to entitlement resolution failures, or fix socks5h proxy handling upstream.
