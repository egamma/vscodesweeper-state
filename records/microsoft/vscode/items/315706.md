# Issue #315706 — Cannot re-enable AI features for a workspace after disabling

[https://github.com/microsoft/vscode/issues/315706](https://github.com/microsoft/vscode/issues/315706)

## Summary

Still reproduces on main — the EnableAIFeaturesInWorkspaceAction.update() returns early (disabled) when `chat.disableAIFeatures` merged value is `false`, but the Disable action only sets the extension to DisabledWorkspace without writing the setting, so the enable action is never activated. Author is MEMBER; keeping open.

## Change summary

After using 'Disable AI Features (Workspace)' via the extension context menu, the 'Enable AI Features (Workspace)' menu item is greyed out and cannot be clicked.

## Best solution

Fix `EnableAIFeaturesInWorkspaceAction.update()` in `src/vs/workbench/contrib/extensions/browser/extensionsActions.ts` to also enable itself when the extension's enablement state is `DisabledWorkspace` (or have `DisableAIFeaturesInWorkspaceAction.run()` also set `chat.disableAIFeatures` to `true` at workspace scope, mirroring how the enable action clears it).
