# Issue #168045 — Git - git.rebasewhensync setting attempts a rebase when there is nothing to pull

[https://github.com/microsoft/vscode/issues/168045](https://github.com/microsoft/vscode/issues/168045)

## Summary

Valid feature request to support `--rebase=merges` (preserve local merge commits) when `git.rebaseWhenSync` is enabled. The current code unconditionally passes `-r` to `git pull`, which drops merge commits. A git-level workaround exists (`git config pull.rebase merges`). Only 3 upvotes — keeping open for the community process.

## Change summary

When `git.rebaseWhenSync` is true, the sync command uses `git pull -r` which drops local merge commits; users want an option to use `--rebase=merges` instead to preserve them.

## Best solution

Add a setting (e.g. `git.rebaseMode` with values `true`/`merges`/`interactive`) controlling the `--rebase=` argument passed to `git pull` in `extensions/git/src/git.ts:~2467`. This would let users preserve merge commits during sync while still rebasing linear commits. The workaround of `git config pull.rebase merges` with `git.rebaseWhenSync: false` covers the gap for now.
