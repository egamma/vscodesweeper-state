# Issue #280829 — Copilot Code Review does not use `/instruction/**/NAME.instructions.md` files

[https://github.com/microsoft/vscode/issues/280829](https://github.com/microsoft/vscode/issues/280829)

## Summary

Confirmed gap: Copilot Code Review's instruction collection path (`ICustomInstructionsService.getAgentInstructions()`) only looks for `.github/copilot-instructions.md` and the personal instructions file, skipping `.github/instructions/**/NAME.instructions.md` entirely. The core chat path uses `ComputeAutomaticInstructions` via `IPromptsService` which handles the full instruction file system, but the review flow in the copilot extension bypasses that. A MEMBER confirmed this and reopened the issue as valid.

## Change summary

Copilot Code Review ignores path-specific instruction files (`.github/instructions/**/NAME.instructions.md`), only reading the top-level `.github/copilot-instructions.md`.

## Best solution

Extend the code review instruction collection in `extensions/copilot/src/extension/review/node/githubReviewAgent.ts` (the `loadCustomInstructions` function) to also discover and include `.github/instructions/**/NAME.instructions.md` files, ideally by reusing the same `IPromptsService`-based discovery that the core chat path uses in `ComputeAutomaticInstructions`. The `perf` label on this issue appears misplaced since the performance concern was split to https://github.com/microsoft/vscode/issues/279252.
