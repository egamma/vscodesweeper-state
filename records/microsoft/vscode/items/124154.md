# Issue #124154 — Calling `TextEditorEdit.replace` to replace a line of text introduces a partial selection of the new text

[https://github.com/microsoft/vscode/issues/124154](https://github.com/microsoft/vscode/issues/124154)

## Summary

Keep open because this is a team-authored issue from a VS Code member, which needs explicit maintainer judgment rather than cleanup closure. The report is also already routed as an API/editor API bug with a concrete extension-host repro and a member workaround comment.

## Change summary

Calling `TextEditorEdit.replace` on a line range leaves part of the replacement text selected instead of only moving the cursor to the end.

## Best solution

The editor API owners should decide whether `TextEditor.edit(...replace...)` should preserve/collapse selection differently after replacement; the likely implementation path runs through `src/vs/workbench/api/common/extHostTextEditor.ts` and `src/vs/workbench/api/browser/mainThreadEditor.ts`.
