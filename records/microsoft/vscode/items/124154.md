# Issue #124154 — Calling `TextEditorEdit.replace` to replace a line of text introduces a partial selection of the new text

[https://github.com/microsoft/vscode/issues/124154](https://github.com/microsoft/vscode/issues/124154)

## Summary

Still reproduces on main: `mainThreadEditor.ts#applyEdits` calls `executeEdits` without an `endCursorState` parameter, so the editor falls back to marker-based cursor recovery which creates a partial selection when replacement text differs in length from the original.

## Change summary

Calling `TextEditorEdit.replace` to replace text with a longer string introduces a spurious partial selection of the trailing portion of the new text.

## Best solution

Pass a cursor state computer to `executeEdits` in `mainThreadEditor.ts#applyEdits` (line ~506) that collapses selections to their anchor positions after the edit, preserving cursor positions without introducing selections. Alternatively, compute an `endCursorState` that maps each pre-edit selection through the edit operations.
