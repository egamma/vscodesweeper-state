## Summary

Team-authored issue (MEMBER); must stay open for maintainer judgment. The core window-state validation logic (`validateWindowState`) handles off-screen windows at startup, but the scenario of monitors waking at different speeds after login may expose a timing gap. A related RDP fix (commit 1f48d5756c8) added `display-added` tracking but is gated behind `enableRDPDisplayTracking` and may not cover physical monitor wake scenarios.

## Change summary

VS Code window ends up off-screen and inaccessible after logging in on a multi-monitor setup where monitors wake at different speeds.

## Best solution

The relevant code is in `src/vs/platform/windows/electron-main/windowImpl.ts` (lines 238–254) and `src/vs/platform/windows/electron-main/windows.ts` (`validateWindowState`).
