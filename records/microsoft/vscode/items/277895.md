# Issue #277895 — Accessibility Regression: NVDA mouse tracking broken due to Chromium 142 update (Upstream Issue)

[https://github.com/microsoft/vscode/issues/277895](https://github.com/microsoft/vscode/issues/277895)

## Summary

This is a well-investigated upstream issue at the intersection of Chromium's UIA provider changes and NVDA's heuristics. Current main is now on Chromium 148 (past the 144 fix), but the deeper NVDA/Chromium interop problem (NVDA not opting into UIA for newer Chromium) remains tracked upstream. The related VS Code issue https://github.com/microsoft/vscode/issues/224704 is still open with ongoing discussion as of April 2026. Keeping open as a tracked upstream issue.

## Change summary

NVDA mouse tracking stopped working in VS Code after the Chromium 142 update because NVDA defaults to the legacy IA2 accessibility path, which broke when Chromium enabled UIA by default.

## Best solution

The original Chromium hit-test regression (pre-144) is resolved — current main ships Chromium 148. The remaining gap is NVDA's default UIA opt-in heuristic for Electron apps, tracked in https://github.com/nvaccess/nvda/issues/19276. Keep this issue open as a tracked upstream reference alongside https://github.com/microsoft/vscode/issues/224704 until NVDA updates its defaults or VS Code can signal UIA preference. Consider consolidating with https://github.com/microsoft/vscode/issues/224704 if the remaining problem is identical.
