# Issue #199377 — Sparkle should render on new line

[https://github.com/microsoft/vscode/issues/199377](https://github.com/microsoft/vscode/issues/199377)

## Summary

Team-authored issue (MEMBER) about the sparkle/lightbulb rendering position when the cursor is at column 1. The lightbulb widget has seen significant rework since (gutter placement, inline chat affordance integration), but the core concern — sparkle appearing on a different line than the cursor — remains an open UX discussion. Keeping open per team-author policy and because this is still marked under-discussion.

## Change summary

The AI sparkle (lightbulb) renders on the line above the cursor instead of on the same line when the cursor is at column 1.

## Best solution

Revisit whether the sparkle/lightbulb positioning logic in `src/vs/editor/contrib/codeAction/browser/lightBulbWidget.ts` should place the indicator on the same line as the cursor (e.g. to the right) rather than on the line above, especially now that the widget has moved to the gutter in later iterations. The gutter rework (https://github.com/microsoft/vscode/commit/faf8c4f057a) and inline-chat affordance integration (https://github.com/microsoft/vscode/commit/c271611f6fe) may have already changed the dynamics — a maintainer should verify whether this specific complaint still applies.
