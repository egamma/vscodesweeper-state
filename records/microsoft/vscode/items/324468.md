# Issue #324468 — Icon for session context chip gone after reload

[https://github.com/microsoft/vscode/issues/324468](https://github.com/microsoft/vscode/issues/324468)

## Summary

Team-authored bug, keeping open. The session context chip icon is set when attaching via the picker but not restored after reload because `restoreSessionReferenceVariableEntryFromAttachment` omits the `icon` property.

## Change summary

The icon on a session context chip disappears after window reload because the restore path does not carry the icon forward.

## Best solution

In `restoreSessionReferenceVariableEntryFromAttachment` (src/vs/workbench/contrib/chat/browser/agentSessions/agentHost/agentHostSessionReferenceAttachment.ts), restore the `icon` property — either by persisting it alongside the attachment metadata (e.g. serializing the ThemeIcon id/color into `_meta`) or by re-deriving it from the session's provider type at restore time via `getAgentSessionProviderIcon`.
