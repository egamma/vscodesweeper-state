# Issue #308088 — It's not clear that starting a session with a remote repo will create a PR using whatever input you initially add

[https://github.com/microsoft/vscode/issues/308088](https://github.com/microsoft/vscode/issues/308088)

## Summary

Team-authored UX issue about cloud agent session clarity — kept open per policy. The team is actively pivoting the cloud session model away from PR-first, which may subsume or reshape this request.

## Change summary

Starting a cloud agent session with a remote repo silently creates a PR without confirming or communicating that to the user.

## Best solution

Address as part of the broader cloud-session UX pivot discussed in the comments: make cloud sessions behave like local sessions by default, with PR creation as an explicit opt-in action rather than an implicit side effect. This removes the confusion at its root rather than adding confirmation dialogs.
