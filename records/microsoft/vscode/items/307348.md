# Issue #307348 — Chat session permanently unloadable after VS Code killed mid-response (TypeError: Cannot read properties of undefined (reading 'response'))

[https://github.com/microsoft/vscode/issues/307348](https://github.com/microsoft/vscode/issues/307348)

## Summary

Still reproduces on current main: the deserialization loop at chatSessionStore.ts L676 has no guard for `request.response` being undefined, so an interrupted session (killed mid-stream) permanently crashes on load. Milestone is 'On Deck' — must stay open. An open PR (#309286) addresses the write-side but not this read-side guard.

## Change summary

A chat session becomes permanently unloadable when VS Code is killed during a streaming response, because the deserialization code does not guard against an undefined `request.response`.

## Best solution

Add a null/undefined guard in `chatSessionStore.ts` at L676 to skip or gracefully handle requests with no `response` property, so interrupted sessions degrade to showing completed turns rather than permanently crashing. PR https://github.com/microsoft/vscode/pull/309286 addresses the serializer-failure fallback but does not fix this read-side deserialization crash — both fixes are complementary.
