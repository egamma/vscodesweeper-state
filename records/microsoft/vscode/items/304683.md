# Issue #304683 — "Allow all files in this repository for this session" fail silently for external paths

[https://github.com/microsoft/vscode/issues/304683](https://github.com/microsoft/vscode/issues/304683)

## Summary

The core problem — 'Allow all files in this repository for this session' silently failing when git root detection fails — was fixed in https://github.com/microsoft/vscode/issues/307276. The fix (commit b2099b82ac1) makes the button fall back to allowing just the containing folder when no git root is found, eliminating the silent no-op. This issue is a duplicate of #307276.

## Change summary

Clicking 'Allow all files in this repository for this session' silently does nothing when the target path is not inside a detectable Git repository.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/307276, which is already closed with a fix. The remaining minor edge case (dirname starting from parent instead of the directory itself) is now harmless because the fallback prevents silent failure. If the dirname behavior matters, a narrow follow-up could be filed.
