# Issue #277454 — Wait for shell integration to be ready when opening new terminal via API

[https://github.com/microsoft/vscode/issues/277454](https://github.com/microsoft/vscode/issues/277454)

## Summary

The requested API already exists: `window.onDidChangeTerminalShellIntegration` fires when shell integration activates on a terminal, and `terminal.shellIntegration` exposes the current state. This was finalized in VS Code 1.93 (August 2024), well before the report.

## Change summary

Request for an event/API to detect when shell integration is ready after creating a terminal via the extension API.

## Best solution

Close as already provided. The `window.onDidChangeTerminalShellIntegration` event plus the `terminal.shellIntegration` property cover both asks (an event and a readiness check). The vscode.d.ts documentation includes an example of the exact pattern the reporter needs.

- **Proposed comment:**

  > Thanks for the report and the detailed description of the problem. The API you're looking for already exists since VS Code 1.93:
  > 
  > - **`window.onDidChangeTerminalShellIntegration`** — fires when shell integration activates on a terminal
  > - **`terminal.shellIntegration`** — check if shell integration is already active
  > 
  > The pattern is documented in the API itself ([vscode.d.ts example](https://github.com/microsoft/vscode/blob/main/src/vscode-dts/vscode.d.ts)):
  > 
  > ```typescript
  > const myTerm = window.createTerminal();
  > window.onDidChangeTerminalShellIntegration(({ terminal, shellIntegration }) => {
  >   if (terminal === myTerm) {
  >     const execution = shellIntegration.executeCommand('echo "Hello world"');
  >     // ...
  >   }
  > });
  > // Fallback if shell integration doesn't activate within 3 seconds
  > setTimeout(() => {
  >   if (!myTerm.shellIntegration) {
  >     myTerm.sendText('echo "Hello world"');
  >   }
  > }, 3000);
  > ```
  > 
  > Extensions like Claude Code / opencode should use this event to wait for readiness before executing commands. Closing as the capability is already available.
