# Issue #255975 — long Scrolling with physical trackpad on Android does not work properly

[https://github.com/microsoft/vscode/issues/255975](https://github.com/microsoft/vscode/issues/255975)

## Summary

Keep open: this is a specific vscode.dev Android/Chrome physical-trackpad scrolling bug, already routed to the web/Chromium/trackpad area, and there is no concrete duplicate or fixing commit in the available evidence.

## Change summary

Long physical-trackpad scroll gestures on Android Chrome stop immediately when fingers leave the trackpad instead of continuing momentum scrolling.

## Best solution

Investigate the editor wheel/scroll pipeline for Android Chrome physical trackpads, especially whether VS Code's custom wheel handling is suppressing browser momentum or whether this should be tracked as a Chromium Android input issue.
