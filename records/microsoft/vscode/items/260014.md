# Issue #260014 — Accidentally opened Desktop as a Git repo → Discard All Changes → files deleted.

[https://github.com/microsoft/vscode/issues/260014](https://github.com/microsoft/vscode/issues/260014)

## Summary

The current git extension already provides confirmation dialogs with explicit 'IRREVERSIBLE' / 'FOREVER LOST' warnings and defaults to moving untracked files to the Recycle Bin (via `git.discardUntrackedChangesToTrash`). A team member assessed this as expected behavior and reopened for community discussion. No upvotes or further input have arrived in nearly a year. Keeping open per the team member's intent to gather community input, though it may be closeable as `*as-designed`.

## Change summary

User accidentally initialized Desktop as a Git repo, then used 'Discard All Changes' which deleted untracked files, requesting safer guardrails.

## Best solution

If the team wants to close this, `*as-designed` is appropriate — the existing dialogs warn clearly, and `discardUntrackedChangesToTrash` (default true on Windows) should have sent files to the Recycle Bin. If the team wants to improve safety further, a possible enhancement would be detecting unusually large working trees (e.g., Desktop with hundreds of non-project files) and showing an extra-prominent warning. That would be a feature request in `extensions/git/src/commands.ts`.
