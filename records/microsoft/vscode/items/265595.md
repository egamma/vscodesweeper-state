# Issue #265595 — editor.inlineSuggest.suppressSuggestions is not working

[https://github.com/microsoft/vscode/issues/265595](https://github.com/microsoft/vscode/issues/265595)

## Summary

Still reproduces on main: the `editor.inlineSuggest.suppressSuggestions` setting is only consulted as a fallback when the inline suggestion is already visible AND the provider didn't set its own `suppressSuggestions` flag. A race condition means the suggest widget often appears before the inline completion arrives, bypassing the setting entirely. Milestoned to 'On Deck' — keeping open.

## Change summary

The `editor.inlineSuggest.suppressSuggestions` setting has no visible effect because the suggest widget triggers before inline completions are visible, and the suppression check early-exits when no inline suggestion is shown.

## Best solution

The `canShowQuickSuggest` check needs to either wait for inline completions to resolve (like the `offWhenInlineCompletions` path does) when `suppressSuggestions` is enabled, or dismiss the suggest widget when an inline suggestion subsequently appears. The fix likely lives in `src/vs/editor/contrib/suggest/browser/suggestModel.ts` — the `_doTriggerQuickSuggest` method should respect the setting proactively, not just reactively.

## Evidence

- **race condition in source:** In src/vs/editor/contrib/suggest/browser/suggestModel.ts line 111-113, `canShowQuickSuggest` returns `true` immediately when `inlineSuggestionVisible` is false — the setting at line 119 is never reached if the inline suggestion hasn't arrived yet.
- **waitForInlineCompletions not connected to setting:** The `waitForInlineCompletions` path (line 448) that handles the race is only activated when quickSuggestions is set to `offWhenInlineCompletions` (line 433), NOT when `editor.inlineSuggest.suppressSuggestions` is enabled.
- **provider override:** At src/vs/editor/contrib/inlineCompletions/browser/controller/inlineCompletionsController.ts:378, the context key is set from the provider's response — if Copilot doesn't set `suppressSuggestions: true` in its InlineCompletionList, the context key is `undefined` and only then does the user's setting apply.
