# Issue #149849 — Scrollbar Minimap and visible code are out of sync when CodeLens is used

[https://github.com/microsoft/vscode/issues/149849](https://github.com/microsoft/vscode/issues/149849)

## Summary

Still present on main: the minimap in fit/fill mode renders only text lines without accounting for CodeLens view zones, causing the slider position to drift from the actual viewport. No fix has been committed.

## Change summary

Minimap slider position is out of sync with the editor viewport when CodeLens is active and minimap size is set to 'fit' or 'fill'.

## Best solution

The minimap layout computation in fit/fill mode (src/vs/editor/browser/viewParts/minimap/minimap.ts) needs to account for whitespace introduced by view zones (CodeLens) when mapping viewport scroll position to slider position. Either render placeholder space for view zones in the minimap, or adjust the slider offset calculation to compensate for their height.
