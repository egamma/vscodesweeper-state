# Issue #149849 — Scrollbar Minimap and visible code are out of sync when CodeLens is used

[https://github.com/microsoft/vscode/issues/149849](https://github.com/microsoft/vscode/issues/149849)

## Summary

Keep open as an actionable editor minimap bug. The earlier https://github.com/microsoft/vscode/issues/111499 fix adjusted slider positioning, but current minimap rendering still operates on text line ranges and does not render CodeLens view-zone height as blank minimap space.

## Change summary

When CodeLens view zones are present, the minimap viewport/slider can diverge from the visible editor content for `editor.minimap.size` values `fit` and `fill`.

## Best solution

Fix the editor minimap path to account for CodeLens/view-zone whitespace consistently in fit/fill mode, likely in `src/vs/editor/browser/viewParts/minimap/minimap.ts`, either by rendering proportional blank minimap space for view zones or by otherwise aligning the minimap line-to-scroll mapping without jitter.
