# Issue #149849 — Scrollbar Minimap and visible code are out of sync when CodeLens is used

[https://github.com/microsoft/vscode/issues/149849](https://github.com/microsoft/vscode/issues/149849)

## Summary

Keep open as an actionable editor minimap bug: the report is specific, already routed to `editor-minimap`, and current code still shows CodeLens as editor view zones while minimap line rendering advances only by text model lines.

## Change summary

CodeLens view-zone height can make the minimap viewport highlight drift from the visible editor content when `editor.minimap.size` is `fit` or `fill`.

## Best solution

Handle CodeLens/view-zone whitespace in the minimap layout for fit/fill, likely in `src/vs/editor/browser/viewParts/minimap/minimap.ts`, either by reserving blank minimap rows for view zones or by mapping slider/content positions through view layout whitespace consistently.
