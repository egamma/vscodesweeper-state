# Issue #211305 — `editorError`, `editorWarning`, `editorInfo` and `editorHint` `background` properties aren't properly aligned

[https://github.com/microsoft/vscode/issues/211305](https://github.com/microsoft/vscode/issues/211305)

## Summary

Still reproducible on main: the `editorError.background` (and warning/info/hint) color is rendered via a `::before` pseudo-element inside the `.cdr` decoration div, and appears visually misaligned with the line background highlight. No fix has landed since the report, and a team member has engaged with it.

## Change summary

The background color set via `editorError.background` / `editorWarning.background` is not vertically aligned with the line's own background highlight.

## Best solution

Fix the CSS in `src/vs/editor/browser/widget/codeEditor/editor.css` so the `::before` pseudo-element on `.squiggly-error` / `.squiggly-warning` / `.squiggly-info` aligns flush with the line background. The rendering logic in `src/vs/editor/browser/viewParts/decorations/decorations.ts` may also need adjustment if the container positioning is the root cause.
