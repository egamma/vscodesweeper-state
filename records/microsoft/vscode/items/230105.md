# Issue #230105 — Conflict resolver worked not expected when I click accpect incoming or current

[https://github.com/microsoft/vscode/issues/230105](https://github.com/microsoft/vscode/issues/230105)

## Summary

Still reproduces on main: the `accept` method in the merge-conflict extension does not await `commitEdit`, so a rapid second accept uses stale conflict ranges. No fix has landed since the report.

## Change summary

Accepting two merge conflicts in quick succession causes the second resolution to apply against stale ranges, producing an incorrect edit that must be undone.

## Best solution

Await the `commitEdit` call in `commandHandler.ts#accept()` so subsequent conflict resolutions operate on the updated document state; alternatively, serialize accepts through a queue or re-scan conflicts after each edit lands.
