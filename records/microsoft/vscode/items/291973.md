# Issue #291973 — [Copilot BYOM] Allow 'parameterOverrides' in chatLanguageModels.json for API compatibility

[https://github.com/microsoft/vscode/issues/291973](https://github.com/microsoft/vscode/issues/291973)

## Summary

Valid feature request with 54 upvotes, already promoted to Backlog. The parameterOverrides capability does not exist on current main — the language model configuration schema has no mechanism for overriding or suppressing request parameters. A real gap remains for users connecting to strict OpenAI-compatible backends.

## Change summary

Allow users to override or suppress inference parameters (like top_p, temperature) per model in chatLanguageModels.json for compatibility with strict API backends.

## Best solution

Add a `parameterOverrides` (or similar) property to the per-model configuration schema in `src/vs/workbench/contrib/chat/common/languageModels.ts` and wire it into the request-building path so that null values suppress the parameter and non-null values override the default. This fits naturally into the existing BYOM configuration surface.
