# Issue #259145 — deleteWordRight handles 1 whitespace differently from 2 whitespace

[https://github.com/microsoft/vscode/issues/259145](https://github.com/microsoft/vscode/issues/259145)

## Summary

Confirmed bug still present on main: the whitespace heuristic in `_deleteWordRightWhitespace` uses a strict `<` comparison that requires 2+ whitespace chars, so a single leading space is not consumed by the heuristic and falls through to word deletion. Three community PRs are open but none merged.

## Change summary

`deleteWordRight` at the start of a line with exactly one leading space deletes the space plus the first word, whereas two or more leading spaces correctly delete only the whitespace.

## Best solution

Change the off-by-one guard in `_deleteWordRightWhitespace` (line 631 of `src/vs/editor/common/cursor/cursorWordOperations.ts`) from `startIndex + 1 < firstNonWhitespace` to `startIndex < firstNonWhitespace`, so a single whitespace character is also consumed by the heuristic. One of the open PRs (https://github.com/microsoft/vscode/pull/305680, https://github.com/microsoft/vscode/pull/306105, https://github.com/microsoft/vscode/pull/324211) likely addresses this; review and merge the best one.
