# Issue #228818 — Multiple cursors + Word wrap

[https://github.com/microsoft/vscode/issues/228818](https://github.com/microsoft/vscode/issues/228818)

## Summary

Keep open as a plausible editor-multicursor regression or uncovered path: the old fix for https://github.com/microsoft/vscode/issues/26393 only covered Insert Cursor Above/Below, while this report also points at word-wrapped multi-cursor selection/change-all behavior.

## Change summary

Multiple cursor placement and occurrence selection can target visual wrapped lines instead of logical model lines when word wrap is enabled.

## Best solution

Route to editor-multicursor and investigate the word-wrap interaction in the mouse/create-cursor and Change All Occurrences paths. The end state should either extend logical-line handling beyond Insert Cursor Above/Below or explicitly fold this into a broader command-argument/custom-keybinding design such as https://github.com/microsoft/vscode/issues/3130.
