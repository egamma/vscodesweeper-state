# Issue #318604 — AgentHost: Remote session stops updating after connection change

[https://github.com/microsoft/vscode/issues/318604](https://github.com/microsoft/vscode/issues/318604)

## Summary

Team-filed bug on the 'On Deck' milestone describing a real remote agent-host session staleness issue after network reconnection. No fix found on current main — must stay open for maintainer judgment.

## Change summary

Remote agent-host sessions stop updating after a network-triggered reconnection because a stale ChatModel is reused via requestInProgressKeepAlive, bypassing provideChatSessionContent.

## Best solution

Fix the reconnection path in agentHostSessionHandler.ts so that when a remote session reconnects, the stale ChatModel held by requestInProgressKeepAlive is invalidated or refreshed, and provideChatSessionContent is re-invoked to render the complete session content from the agent host.
