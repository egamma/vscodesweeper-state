# Issue #303254 — Repeated custom-agent rescans emit endless parse errors/diagnostics for cross-tool .agent.md files

[https://github.com/microsoft/vscode/issues/303254](https://github.com/microsoft/vscode/issues/303254)

## Summary

The repeated rescan behavior is traceable in the current source: CachedPromise invalidation on file-watcher events causes repeated re-parsing of agent files that always fail, with no memoization of previous parse failures. Still open on main.

## Change summary

Custom agent discovery repeatedly re-parses unchanged cross-tool .agent.md files on every file-watcher event, emitting repeated error logs and flashing diagnostics.

## Best solution

Add content-hash or error-memoization to `PromptsService.computeAgentDiscoveryInfo` (in `src/vs/workbench/contrib/chat/common/promptSyntax/service/promptsServiceImpl.ts`) so that files whose content hasn't changed since the last failed parse are not re-parsed and re-logged. Alternatively, reduce file-watcher noise for external folders (e.g. `~/.claude/agents`) by using correlated watchers with tighter filters.
