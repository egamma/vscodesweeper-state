# Issue #255613 — How many times do I have to reject a suggestion before it stops suggesting it in exactly the same place?

[https://github.com/microsoft/vscode/issues/255613](https://github.com/microsoft/vscode/issues/255613)

## Summary

Known NES issue where the rejection cache doesn't prevent re-suggestion of dismissed edits. Team members have engaged and acknowledged the problem — hediet confirmed a client-side rejection cache should exist but didn't work in this case. Properly labeled and routed; keeping open for the NES team to investigate the caching logic.

## Change summary

NES repeatedly re-suggests the same inline edit immediately after the user dismisses it with Escape, ignoring the rejection.

## Best solution

The rejection cache on the extension/provider side (Copilot) needs to reliably suppress re-suggestion of edits the user has explicitly dismissed. VS Code core already calls `handleRejection` on the provider (in `src/vs/editor/contrib/inlineCompletions/browser/model/provideInlineCompletions.ts`); the fix likely lives in the Copilot extension's caching logic that consumes that signal. The NES team (hediet, chrmarti) should investigate why the cache miss occurs.
