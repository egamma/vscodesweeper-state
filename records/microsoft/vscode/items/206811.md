# Issue #206811 — InlineCompletion Tab acceptance

[https://github.com/microsoft/vscode/issues/206811](https://github.com/microsoft/vscode/issues/206811)

## Summary

Valid feature request under active discussion by team members (hediet, isidorn). The related PR #221360 remains open. The indentation-prefix gating logic exists on main but no per-provider flag has been added yet — the gap remains.

## Change summary

Allow inline completion providers to opt out of the indentation-prefix Tab suppression, so Tab always accepts the completion regardless of leading whitespace.

## Best solution

Either implement the per-provider flag (`enableTabPrefixAcceptance` or similar on `InlineCompletionItem`) as proposed, or — per isidorn's suggestion — change the default behavior to always accept and add the flag only if complaints arise. The logic lives in `src/vs/editor/contrib/inlineCompletions/browser/controller/commands.ts` (keybinding when-clause) and `inlineCompletionsModel.ts` (`getIndentationInfo`). The open PR https://github.com/microsoft/vscode/pull/221360 is the related implementation attempt.
