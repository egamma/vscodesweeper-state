# Issue #278708 — Parallel MCP tools execution triggers multiple oauth logins

[https://github.com/microsoft/vscode/issues/278708](https://github.com/microsoft/vscode/issues/278708)

## Summary

Valid request — parallel MCP tool calls still independently trigger OAuth login prompts without deduplication. Maintainers acknowledged the issue and discussed solutions (serializing auth, or showing confirmation in chat) but no fix has landed.

## Change summary

When multiple MCP tools execute in parallel and each requires OAuth, multiple login popups appear instead of one shared auth flow.

## Best solution

Add auth-flow deduplication in `_getSessionForProvider` (or at the `loginPrompt` / `createSession` layer) in `src/vs/workbench/api/browser/mainThreadMcp.ts` so that concurrent requests for the same provider/scopes share a single interactive auth flow. TylerLeonhardt's suggestion of surfacing auth inline in Chat (comment 10) would also improve the UX.
