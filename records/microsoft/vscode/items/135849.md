# Issue #135849 — [html] Unexpected "Declaration or statement expected" in HTML code

[https://github.com/microsoft/vscode/issues/135849](https://github.com/microsoft/vscode/issues/135849)

## Summary

Still reproduces on main: when two script blocks are on the same line and the first ends with a `//` comment, the virtual JS document places both blocks' content on the same line, so the line comment bleeds into the second block causing spurious errors.

## Change summary

A `//` comment in a `<script>` block bleeds into the next `<script>` block when both are on the same line, causing false 'Declaration or statement expected' errors.

## Best solution

In `extensions/html-language-features/server/src/modes/embeddedSupport.ts`, the `substituteWithWhitespace` function (or a wrapper) should inject a newline between script regions that share the same line when the preceding region's content ends with a single-line comment (`//`). Alternatively, convert trailing `//` comments to `/* */` block comments in `updateContent`.
