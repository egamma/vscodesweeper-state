# Issue #135849 — [html] Unexpected "Declaration or statement expected" in HTML code

[https://github.com/microsoft/vscode/issues/135849](https://github.com/microsoft/vscode/issues/135849)

## Summary

Still reproduces on main: the `substituteWithWhitespace` function in embeddedSupport.ts only preserves actual newlines from the original content. When two script blocks are on the same line, a JS line comment (`//`) in the first script bleeds into the second script's embedded document because no newline separator is inserted between regions.

## Change summary

A single-line `//` comment in one `<script>` block causes false "Declaration or statement expected" errors in a subsequent `<script>` block when both are on the same line.

## Best solution

In `extensions/html-language-features/server/src/modes/embeddedSupport.ts`, the `substituteWithWhitespace` function (or the region stitching in `getEmbeddedDocument`) should insert a synthetic newline between consecutive script regions that share the same line, so a trailing line comment in one region cannot bleed into the next.
