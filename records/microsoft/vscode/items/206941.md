# Issue #206941 — Git - The staged file disappears from the git view

[https://github.com/microsoft/vscode/issues/206941](https://github.com/microsoft/vscode/issues/206941)

## Summary

Confirmed on current main: the staged-files (index group) switch in repository.ts does not handle the 'T' (type change) status code, so symlink→file type changes disappear from the staged view. The issue carries an 'On Deck' milestone, so it stays open.

## Change summary

Staging a file that replaced a symlink causes it to disappear from the Source Control staged changes view because the type-change status is unhandled.

## Best solution

Add a `case 'T'` to the index group switch in `extensions/git/src/repository.ts` (around line 3078) mapping to an appropriate status (e.g. `Status.TYPE_CHANGED` or a new `INDEX_TYPE_CHANGED`), mirroring the existing working-tree handling. Also handle 'T' in `parseGitChanges` and `parseGitChangesRaw` in `extensions/git/src/git.ts` which currently break out of parsing on unknown status letters.
