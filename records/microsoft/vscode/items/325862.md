## Change summary

All open VS Code windows crash with SIGTERM (code 15) when a Git worktree directory is deleted externally while VS Code has windows open for related worktrees.

## Best solution

Investigate what in the file-watcher shutdown path (parcelWatcher `onWatchedPathDeleted` or the shared-process cascade) sends SIGTERM to sibling windows. The watcher correctly detects the deleted path, but the process termination should be scoped to the affected window, not propagated to every window. Native crash logs (per the wiki guide) would help confirm whether this is an Electron/Chromium-level crash or a VS Code process-management issue. Reopen or reference https://github.com/microsoft/vscode/issues/307984 for prior context.
