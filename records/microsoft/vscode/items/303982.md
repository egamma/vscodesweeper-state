# Issue #303982 — copilot model thinking level selection not visible in WSL

[https://github.com/microsoft/vscode/issues/303982](https://github.com/microsoft/vscode/issues/303982)

## Summary

The thinking effort selector visibility is driven entirely by the Copilot Chat extension's model metadata (configurationSchema). VS Code core renders it when present — if WSL doesn't show it, the extension isn't providing that schema in that environment. This belongs to the Copilot Chat extension team, not VS Code core.

## Change summary

The thinking effort level selector in the model picker does not appear when using VS Code in WSL, though it sometimes appears on Windows.

## Best solution

File this against the Copilot Chat extension (microsoft/vscode-copilot-chat). The VS Code core model picker at `chatModelPicker.ts` renders thinking effort options when the extension provides a `configurationSchema` with a `navigation` group — the extension needs to ensure it registers models with this schema consistently across all remote environments including WSL.
