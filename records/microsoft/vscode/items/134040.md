# Issue #134040 — `acceptSuggestionOnCommitCharacter` with ghost text

[https://github.com/microsoft/vscode/issues/134040](https://github.com/microsoft/vscode/issues/134040)

## Summary

Team-authored feature request discussing whether commit characters (like `;`) should also accept inline completions (ghost text) alongside the regular completion item. Kept open per team-author policy.

## Change summary

When `acceptSuggestionOnCommitCharacter` triggers, also accept any visible inline completion (ghost text) so one keystroke commits the full suggested statement.

## Best solution

Decide whether commit characters should chain acceptance of both the completion item and the inline completion. If yes, implement in the suggest controller (`src/vs/editor/contrib/suggest/`) to forward acceptance to the inline completions controller when ghost text is visible, avoiding duplicate character insertion.
