# Issue #301028 — SCM repository visibility is not persisted across restarts

[https://github.com/microsoft/vscode/issues/301028](https://github.com/microsoft/vscode/issues/301028)

## Summary

Confirmed bug: the SCM visibility persistence breaks because isHidden repositories (e.g. Copilot worktrees) are excluded from the saved `all` array, so on restore they hit the `index === -1` fallback which makes all repos visible and resets the `didSelectRepository` flag. An open community PR addresses this.

## Change summary

SCM repository visibility selections are lost on restart because hidden repositories trigger a fallback path that resets all visibility state during restoration.

## Best solution

Review and land the open community PR (https://github.com/microsoft/vscode/pull/301029) which fixes the two bugs in `onDidAddRepository`: skipping restoration logic for `isHidden` repos and adding genuinely new repos without resetting existing visibility. The fix is localized to `src/vs/workbench/contrib/scm/browser/scmViewService.ts`.
