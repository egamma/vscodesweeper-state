# Issue #237842 — Investigate possible leak in code actions

[https://github.com/microsoft/vscode/issues/237842](https://github.com/microsoft/vscode/issues/237842)

## Summary

The reported disposable leak in `getCodeActions` has been fixed by multiple commits on main, most directly by 18afe2b9258 which properly tracks the `ManagedCodeActionSet` in the `DisposableStore`, and by 134b70c018f which fixes a race condition where cancelled code actions weren't disposed. Both are shipped.

## Change summary

The `DisposableStore` created in `getCodeActions` leaked because the `ManagedCodeActionSet` was not added to the store, and a cancellation race could leave provider results undisposed.

## Best solution

Keep the shipped fixes; no further change needed. The disposable tracking in `getCodeActions` now correctly adds the `ManagedCodeActionSet` to the store and disposes provider results on cancellation.
