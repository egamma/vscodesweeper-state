# Issue #267476 — Source Control: Cannot commit no-diff merge resolution when other modified tracked files are unstaged

[https://github.com/microsoft/vscode/issues/267476](https://github.com/microsoft/vscode/issues/267476)

## Summary

Real bug, confirmed in current source: the smartCommit logic in the git extension does not skip the 'no staged changes' prompt when a merge is in progress with a no-diff resolution. Milestone is 'On Deck', so keeping open.

## Change summary

When resolving a merge conflict to match HEAD (no-diff resolution) with other unstaged tracked files, the Source Control GUI incorrectly prompts to stage all changes instead of allowing the merge commit to proceed.

## Best solution

Add a `!repository.mergeInProgress` guard to the 'no staged changes' check at line ~2419 of `extensions/git/src/commands.ts`, mirroring the existing guard at line ~2484 for the empty-commit check. When a merge is in progress, the commit should proceed with only the merge state, leaving unstaged changes untouched.
