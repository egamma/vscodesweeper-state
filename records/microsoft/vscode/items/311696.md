# Issue #311696 — [Error] unhandlederror-startLineNumber 892 cannot be after endLineNumberExclusive 816

[https://github.com/microsoft/vscode/issues/311696](https://github.com/microsoft/vscode/issues/311696)

## Summary

Still reproduces on main: line 177 in `diffEditorViewModel.ts` uses the original decoration range's `length` for both original and modified sides when rebuilding `UnchangedRegion` from stale decorations. When the original model content diverges significantly (e.g. merge-base switch from PR #311026), the resulting hidden regions overlap and violate `LineRangeMapping.inverse()`'s sorted-input invariant. The June 2026 comment confirms ongoing hits in 1.124.0-insider.

## Change summary

The diff editor's `updateUnchangedRegions` throws when computing visible regions from stale decoration ranges whose lengths diverge between original and modified models.

## Best solution

In `diffEditorViewModel.ts` around line 177, use independent lengths from the original and modified decoration ranges (or at minimum clamp/validate the constructed `UnchangedRegion` so its hidden ranges never exceed model line counts) instead of assuming both sides share the same `lineCount`. The `LineRangeMapping.clip` at line 192 mitigates some cases but doesn't prevent overlaps that trip `inverse()`.
