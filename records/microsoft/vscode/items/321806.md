# Issue #321806 — Flexible "Centered Layout"

[https://github.com/microsoft/vscode/issues/321806](https://github.com/microsoft/vscode/issues/321806)

## Summary

The centered layout intentionally centers within the editor area (the space after sidebars/panels), not the full window. The CenteredViewLayout is a SplitView operating within the editor part's bounds — this is by design. The request for window-center and minimum-width behavior is a feature request against intentional architecture.

## Change summary

Request to change centered layout to center relative to the full window instead of the remaining editor area, and to add a minimum width constraint.

## Best solution

The current behavior is architecturally intentional — the centered layout operates within the editor part which sits alongside sidebars/panels. The existing `workbench.editor.centeredLayoutFixedWidth` setting partially addresses the width concern. A window-relative centering mode would require rethinking the layout hierarchy. If kept open, this belongs under `layout` as a feature request.
