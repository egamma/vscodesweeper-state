# Issue #316001 — Opção de colar arquivo desabilitada no menu de contexto

[https://github.com/microsoft/vscode/issues/316001](https://github.com/microsoft/vscode/issues/316001)

## Summary

Still reproduces on main: the explorer context menu paste is disabled when files are copied from an external OS file manager because `clipboardService.hasResources()` only checks for VS Code's internal MIME type, not native file entries. Ctrl+V works because the keybinding doesn't gate on `FileCopiedContext`.

## Change summary

Paste option in explorer context menu is disabled when pasting files copied from an external file manager (e.g. Windows Explorer), though Ctrl+V works.

## Best solution

Extend `hasResources()` in `src/vs/platform/clipboard/browser/clipboardService.ts` to also detect native file entries on the OS clipboard (or remove `FileCopiedContext` from the context menu precondition and rely on the paste handler to no-op when nothing is pasteable).
