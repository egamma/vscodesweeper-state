# Issue #86299 — [themes] IconTheme doesn't work with spaces in names

[https://github.com/microsoft/vscode/issues/86299](https://github.com/microsoft/vscode/issues/86299)

## Summary

Fixed on main — the `fileIconSelectorEscape` function (commit ae985a6b, PR #226006, August 2024 / VS Code 1.93) properly escapes spaces in icon theme folder/file names by replacing whitespace with `/`, used consistently on both CSS selector and DOM class sides.

## Change summary

Custom icon themes failed to match folder/file names containing spaces because the class name wasn't escaped consistently.

## Best solution

Keep the shipped fix (commit ae985a6b); no further change needed. The `fileIconSelectorEscape` function in `getIconClasses.ts` handles whitespace correctly on current main.
