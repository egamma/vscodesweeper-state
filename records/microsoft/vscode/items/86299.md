# Issue #86299 — [themes] IconTheme doesn't work with spaces in names

[https://github.com/microsoft/vscode/issues/86299](https://github.com/microsoft/vscode/issues/86299)

## Summary

Fixed on current main: the `fileIconSelectorEscape` function in `src/vs/editor/common/services/getIconClasses.ts` explicitly handles whitespace in file/folder names by replacing spaces with `/`, and this escape is applied consistently on both the CSS generation side (icon theme data) and the DOM class application side (file explorer). The mechanism works correctly for names with spaces.

## Change summary

Custom icon theme definitions with spaces in `folderNames` keys (e.g., "Some stuff") did not match the actual folder's CSS class, so custom icons were not applied.

## Best solution

Keep the shipped fix — `fileIconSelectorEscape` at `src/vs/editor/common/services/getIconClasses.ts:125` consistently escapes whitespace to `/` on both the theme CSS generation side (`fileIconThemeData.ts:497`) and the file class application side (`getIconClasses.ts:38`), combined with `CSS.escape` for selector safety. No further change needed.
