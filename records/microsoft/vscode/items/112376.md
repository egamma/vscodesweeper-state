# Issue #112376 — [json] Relative path wrongly resolved

[https://github.com/microsoft/vscode/issues/112376](https://github.com/microsoft/vscode/issues/112376)

## Summary

Confirmed upstream bug in vscode-json-languageservice's schema resolver — relative $ref paths are resolved against the wrong base URI when schemas are merged. The root cause was identified by the reporter in jsonSchemaService.ts but remains unfixed upstream.

## Change summary

JSON $ref relative paths are resolved against the wrong base URI, causing incorrect schema validation errors for multi-file JSON schema projects.

## Best solution

Fix the relative-path resolution logic in microsoft/vscode-json-languageservice's `jsonSchemaService.ts` (around the `merge`/`resolveRefs` interaction). The fix needs to ensure that when referenced schemas are merged, their relative $ref paths are resolved against the referenced schema's URI, not the root schema's URI. Once fixed upstream, bump the dependency in extensions/json-language-features/server/package.json.
