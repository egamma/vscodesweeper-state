# Issue #320818 — Regression: spaces render too wide with a custom monospaced font

[https://github.com/microsoft/vscode/issues/320818](https://github.com/microsoft/vscode/issues/320818)

## Summary

Confirmed upstream Chromium issue: VS Code renders spaces as U+00A0 (nbsp) to prevent HTML space-collapsing, and a Chromium rendering change causes some custom fonts to render U+00A0 wider than U+0020 even when the font declares identical metrics. Reproduces in plain Chrome on vscode.dev, so the root cause is in Chromium's text shaping.

## Change summary

Spaces render too wide with certain custom monospaced fonts after a Chromium update changed how U+00A0 is shaped.

## Best solution

Track this as an upstream Chromium text-shaping regression. The reporter should file a Chromium bug (crbug.com) with the font and a minimal HTML repro showing U+00A0 rendered wider than U+0020 despite identical font metrics. VS Code could potentially work around it by using a different space-preserving mechanism, but the proper fix belongs in Chromium.
