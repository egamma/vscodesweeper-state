# Issue #320818 — Regression: spaces render too wide with a custom monospaced font

[https://github.com/microsoft/vscode/issues/320818](https://github.com/microsoft/vscode/issues/320818)

## Summary

Keep this as info-needed rather than closing: the reporter reproduced the width problem in Chrome/vscode.dev, which points toward Chromium font rendering, but the custom font and a minimal repro are still missing.

## Change summary

A custom monospace font renders U+00A0/editor spaces wider than Latin monospace characters after upgrading VS Code, breaking column alignment.

## Best solution

Route to the editor area for a Chromium/Electron font-rendering investigation, with the reporter providing the font or a minimal HTML/font repro that compares U+0020 and U+00A0 widths. If the repro is plain Chrome-only, track it upstream rather than changing VS Code editor layout heuristics blindly.
