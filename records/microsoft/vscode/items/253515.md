# Issue #253515 — Support `content` instead of `url` for vscode:chat-* links

[https://github.com/microsoft/vscode/issues/253515](https://github.com/microsoft/vscode/issues/253515)

## Summary

Team-authored feature request to support inline `content` parameter on `vscode:chat-*` links as an alternative to `url`, avoiding auth/hosting requirements. Must keep open per policy.

## Change summary

Allow `vscode:chat-*` links to carry prompt content inline via a `content=` parameter instead of requiring a fetchable URL.

## Best solution

Extend the chat link handler (likely in `src/vs/workbench/contrib/chat/`) to accept a `content` query parameter as an alternative to `url`, with appropriate character-length limits, so prompts can be embedded directly without requiring accessible hosting.
