[https://github.com/microsoft/vscode/issues/253515](https://github.com/microsoft/vscode/issues/253515)

## Summary

Team-authored feature request to support inline `content` parameter on `vscode:chat-*` links as an alternative to `url`, avoiding auth/hosting requirements. Kept open per policy (MEMBER author).

## Change summary

Allow `vscode:chat-*` links to carry prompt content inline via a `content=` parameter instead of requiring a hosted URL.

## Best solution

Extend the chat link handler to accept a `content` query parameter as an alternative to `url`, with a reasonable character-length cap, so prompt content can be embedded directly in links without requiring externally hosted files.
