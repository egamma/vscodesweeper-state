# Issue #158471 — Command to substitute variables from launch config should also get workspaceFolder as an argument

[https://github.com/microsoft/vscode/issues/158471](https://github.com/microsoft/vscode/issues/158471)

## Summary

Valid, unimplemented API enhancement: command variables in launch configs still receive only the configuration object, not the workspace folder. The gap remains on current main (line 168 of baseConfigurationResolverService.ts passes `expr.toObject()` but not `folder`). Team members engaged constructively but no implementation followed.

## Change summary

Pass the workspace folder as an additional argument when executing command variables during debug configuration resolution, so extensions in multi-root workspaces can identify which folder the configuration belongs to.

## Best solution

Implement the folder-as-second-argument approach agreed on in the discussion: in `baseConfigurationResolverService.ts` line 168, pass `folder` as a second argument to `executeCommand`. This requires documenting the command variable contract (both the existing configuration argument and the new folder argument) as public API, per weinand's comment. The debug-specific substitution in `debugConfigurationManager.ts` may also need attention.
