# Issue #291311 — Agent mode does not reference instruction files after reading a file outside its initial context.

[https://github.com/microsoft/vscode/issues/291311](https://github.com/microsoft/vscode/issues/291311)

## Summary

Feature request for more deterministic applyTo instruction matching in agent mode. Current architecture relies on the model choosing to read instruction files from an index; the user wants open-editor or agent-read files to trigger matching automatically. Partially addressed by improved prompting in commit 51d818fec64, but the core determinism gap remains.

## Change summary

Instruction files with applyTo globs are not reliably loaded when the agent reads/modifies matching files — only files explicitly in the attached context trigger automatic matching.

## Best solution

Extend `_getContext` in `computeAutomaticInstructions.ts` to include files the agent reads or modifies during a turn (not just initially attached context), so applyTo matching works dynamically as the conversation progresses. Alternatively, include the active editor file in the initial context set. This belongs in `src/vs/workbench/contrib/chat/common/promptSyntax/computeAutomaticInstructions.ts` and the chat request variable pipeline.
