# Issue #291311 — Agent mode does not reference instruction files after reading a file outside its initial context.

[https://github.com/microsoft/vscode/issues/291311](https://github.com/microsoft/vscode/issues/291311)

## Summary

Valid feature request for more deterministic instruction file loading. The current design relies on the model choosing to read matching instructions (communicated via a customizations index), but users report models often don't. The team acknowledged the design but the gap remains.

## Change summary

Instruction files with `applyTo` globs are not deterministically applied when Agent mode reads or modifies matching files mid-conversation.

## Best solution

Enhance `ComputeAutomaticInstructions` (in `src/vs/workbench/contrib/chat/common/promptSyntax/computeAutomaticInstructions.ts`) to programmatically match and inject instruction files when the agent's tool calls read or create files matching an `applyTo` pattern, rather than relying solely on model behavior. Alternatively, strengthen the system prompt to be more imperative about loading matching instructions.
