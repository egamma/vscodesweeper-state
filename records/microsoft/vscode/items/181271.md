# Issue #181271 — Bracket Pair Colorization isn't properly disabled by configurationDefaults

[https://github.com/microsoft/vscode/issues/181271](https://github.com/microsoft/vscode/issues/181271)

## Summary

Still reproduces on main: when configurationDefaults change bracket pair colorization options on an already-open model, the decoration provider updates its internal state but never fires its `onDidChange` event, so the editor doesn't re-render — affecting the first file visible at startup before extensions finish activating.

## Change summary

Bracket pair colorization decorations are not refreshed when an extension's configurationDefaults disables them on an already-rendered editor at startup.

## Best solution

In `ColorizedBracketPairsDecorationProvider.handleDidChangeOptions`, detect whether `colorizationOptions` actually changed and fire `onDidChangeEmitter` when it did. This ensures the editor re-requests decorations and clears colorized brackets.
