# Issue #196641 — Installation trigger for Python extension when Python language is detected in an untitled file 

[https://github.com/microsoft/vscode/issues/196641](https://github.com/microsoft/vscode/issues/196641)

## Summary

Team-authored feature request (MEMBER) for prompting Python extension installation based on language detection in untitled files — distinct from #196640 (which requires the user to explicitly select the language mode). Keeping open per hard keep-open rule.

## Change summary

Prompt users to install the Python extension when language detection identifies Python code in an untitled file, without requiring the user to manually select the language mode or save with a .py extension.

## Best solution

Extend the extension recommendation system (likely in `src/vs/workbench/contrib/extensions/browser/extensionRecommendations*`) to trigger installation prompts based on automatic language detection events in untitled editors, not only on explicit language-mode selection or file-extension matching. The Python extension team and the extensions recommendations area owner should coordinate on the UX.
