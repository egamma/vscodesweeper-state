# Issue #89142 — [css] text-underline-position value completion is wrong

[https://github.com/microsoft/vscode/issues/89142](https://github.com/microsoft/vscode/issues/89142)

## Summary

Still reproduces: the `values` array for `text-underline-position` in `vscode-css-languageservice` (both 6.3.10 stable and 7.0.0-next.1) only lists `above`, `auto`, and `below` — missing `under`, `from-font`, `left`, and `right`. The `syntax` field is correct but completions are driven by the explicit `values` array. This is an upstream data issue in `@vscode/web-custom-data`.

## Change summary

CSS completions for `text-underline-position` are missing the `under` value (and others like `from-font`, `left`, `right`) despite the syntax being correct.

## Best solution

Fix the `values` array for `text-underline-position` in the `@vscode/web-custom-data` package (or its MDN data source) to include all valid values from the syntax definition (`auto`, `from-font`, `under`, `left`, `right`), then update `vscode-css-languageservice`.
