# Issue #135432 — Debugger: runInTerminal request fails if the path has "ł" character in it

[https://github.com/microsoft/vscode/issues/135432](https://github.com/microsoft/vscode/issues/135432)

## Summary

Still a valid open bug: Unicode characters in file paths are corrupted when VS Code sends debug runInTerminal commands to the integrated terminal on Windows with non-UTF-8 codepages (e.g. codepage 850). The DAP request contains correct Unicode, but the characters are dropped by the time they reach the shell. The related issue #235573 (also implicating PSReadline) was closed as a duplicate of this one, confirming this is the canonical tracker. VS Code now uses shell integration's executeCommand when available (commit 5451a4e69a3), which may mitigate the issue, but no targeted fix for the encoding corruption has been confirmed.

## Change summary

Debugging fails when the file path contains certain Unicode characters (e.g. Polish "ł") because the characters are dropped when VS Code sends the debug command to the integrated terminal on Windows.

## Best solution

Investigate whether the shell integration executeCommand path (added in 5451a4e69a3) resolves this for modern VS Code users, and if not, ensure the text sent to the terminal preserves Unicode regardless of the active Windows codepage or PSReadline behavior. The fix likely lives in the terminal text-sending pipeline or in how the debug terminal is configured (e.g. forcing UTF-8 codepage). Area: debug + terminal.
