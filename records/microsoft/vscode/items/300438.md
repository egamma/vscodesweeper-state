# Issue #300438 — ghost keypress (space, new lines)

[https://github.com/microsoft/vscode/issues/300438](https://github.com/microsoft/vscode/issues/300438)

## Summary

Root cause is a KDE Plasma virtual keyboard bug on Wayland, tracked at https://bugs.kde.org/show_bug.cgi?id=519186. Multiple commenters confirmed the workaround (disabling KDE Virtual Keyboard). This is upstream of VS Code — keeping open as a tracking point for affected users until the KDE fix ships.

## Change summary

Ghost keypresses (extra spaces, newlines) appear when switching tabs on Fedora/KDE Plasma with Wayland, caused by the KDE Plasma virtual keyboard leaking key events.

## Best solution

The fix belongs in KDE Plasma's virtual keyboard implementation (https://bugs.kde.org/show_bug.cgi?id=519186). Label as `upstream` with `linux` and `wayland` so it's tracked. Workaround for users: disable KDE Virtual Keyboard in System Settings or set it to 'None'/'iBus Wayland'. No VS Code code change is needed.
