# Issue #193349 — Bug: Links resolved incorrectly after editing - timing issue

[https://github.com/microsoft/vscode/issues/193349](https://github.com/microsoft/vscode/issues/193349)

## Summary

Still reproducible on main: after editing a link, the debounced async recomputation leaves stale link data in `currentOccurrences`, so a ctrl-click during that window opens the old URL. No fix has landed.

## Change summary

Clicking a link immediately after editing it opens the pre-edit URL because link detection is debounced and the stale occurrence is used.

## Best solution

When a link is clicked (`openLinkOccurrence`), if the link data is potentially stale (a pending `computeLinks` is scheduled), either force an immediate recompute or re-resolve the link text at the click position from the current buffer content before opening. The fix lives in `src/vs/editor/contrib/links/browser/links.ts`.
