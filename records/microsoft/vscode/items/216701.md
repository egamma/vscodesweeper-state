# Issue #216701 — Debug console doesn't appear on Restart (Ctrl+Shift+F5)

[https://github.com/microsoft/vscode/issues/216701](https://github.com/microsoft/vscode/issues/216701)

## Summary

Valid feature request acknowledged by the maintainer — `internalConsoleOptions` is not honored on debug restart, only on initial session start. The restart paths in `debugService.ts` skip the console-opening logic. No fix has landed since the maintainer's acknowledgment.

## Change summary

When restarting a debug session (Ctrl+Shift+F5), the Debug Console should reappear if `internalConsoleOptions` is set to `openOnSessionStart`, matching the behavior of initial launch.

## Best solution

Extend the restart paths in `src/vs/workbench/contrib/debug/browser/debugService.ts` (both the `supportsRestartRequest` branch at ~line 916 and the terminate-relaunch branch at ~line 929) to check `internalConsoleOptions` and open the REPL view, mirroring the logic in `startDebugging` at line 653–656. The maintainer suggested a new `internalConsoleOptions` enum value could also work, but the simplest fix is to honor the existing `openOnSessionStart` value on restart too.
