# Issue #160373 — completion on breakpoint remove all items

[https://github.com/microsoft/vscode/issues/160373](https://github.com/microsoft/vscode/issues/160373)

## Summary

Real architectural limitation in breakpoint condition completions, confirmed by team members — the widget fetches completions from the underlying editor at column 1 without incorporating the breakpoint expression text, so context-aware filtering after operators doesn't work. Keeping open as a documented gap with team discussion.

## Change summary

Breakpoint condition widget completions lose context after logical operators because the expression text is not passed to the completion provider.

## Best solution

The completion provider in `breakpointWidget.ts` (line ~426) would need to synthesize a virtual document combining the underlying editor context with the breakpoint expression text, so language services can provide context-aware completions (e.g. after `foo.` or after `&&`). This is a non-trivial architectural change that roblourens and dbaeumer discussed — LSP has no built-in support for this scenario, so it would likely require a VS Code-side workaround (virtual document injection or a new API).
