# Issue #275429 — Copy hover button does not show aria label on tab focus

[https://github.com/microsoft/vscode/issues/275429](https://github.com/microsoft/vscode/issues/275429)

## Summary

Still reproduces on main: the copy button in content hovers uses `setupDelayedHover` without `setupKeyboardEvents: true`, so tabbing to the button never triggers the tooltip. Author is a team member — keeping open.

## Change summary

The copy button in editor content hovers does not show its tooltip/aria label when focused via keyboard (Tab), only on mouse hover.

## Best solution

Pass `{ setupKeyboardEvents: true }` as `lifecycleOptions` in `SimpleButton`'s `setupDelayedHover` call (in `src/vs/editor/contrib/find/browser/findWidget.ts` around line 1422), or add a focus listener directly. The `_setupDelayedHover` implementation already supports keyboard events via the `setupKeyboardEvents` flag — it just needs to be enabled for `SimpleButton`.
