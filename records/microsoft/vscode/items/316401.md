# Issue #316401 — Agents App: Allow me to edit queued prompts

[https://github.com/microsoft/vscode/issues/316401](https://github.com/microsoft/vscode/issues/316401)

## Summary

Already triaged by the team with bug/ux/agents-window labels and Backlog milestone. The edit-pending-request action exists in shared chat contributions (available in sessions), but the user reports it doesn't work in the Agents window. Keeping open as a valid UX gap that needs maintainer investigation.

## Change summary

Users cannot edit queued/pending prompts in the Agents App, whereas the normal VS Code chat allows clicking on a queued prompt to edit it.

## Best solution

Investigate whether `ChatEditPendingRequestAction` and inline click-to-edit (controlled by `chat.editRequests` setting and `rendererOptions.editable`) work correctly in the sessions/Agents window context. The action is registered via `chat.shared.contribution.ts` and imported by `sessions.common.main.ts`, and `editable` defaults to true for non-inline/non-quick chat, so the gap may be in how pending requests are rendered or interacted with in the Agents window chat view (`src/vs/sessions/contrib/chat/browser/chatView.ts`).
