# Issue #292780 — "Add Dev Container Configuration Files" defaults to an image that does not exist

[https://github.com/microsoft/vscode/issues/292780](https://github.com/microsoft/vscode/issues/292780)

## Summary

The "Add Dev Container Configuration Files" command and its default image tags are provided by the Dev Containers extension (ms-vscode-remote.remote-containers), not VS Code core. The template definitions live in the devcontainers/templates repo. This should be reported to that extension or to the templates repository.

## Change summary

The default Node.js 24 dev container template references a non-existent Docker image tag (1-24-bookworm instead of 24-bookworm).

## Best solution

Report to the Dev Containers extension (https://github.com/microsoft/vscode-remote-release/issues) or to https://github.com/devcontainers/templates/issues where the image tags in the Node.js & TypeScript template are maintained.
