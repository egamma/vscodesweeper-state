# Issue #292780 — "Add Dev Container Configuration Files" defaults to an image that does not exist

[https://github.com/microsoft/vscode/issues/292780](https://github.com/microsoft/vscode/issues/292780)

## Summary

The Dev Containers extension (which owns the 'Add Dev Container Configuration Files' command and its template definitions) ships an incorrect default image tag. This is a Microsoft-owned extension with its own tracker — the issue should be moved to microsoft/vscode-remote-release.

## Change summary

The 'Add Dev Container Configuration Files' command defaults to the image tag `1-24-bookworm` which does not exist on the container registry.

## Best solution

Move this issue to microsoft/vscode-remote-release where the Dev Containers extension team maintains the templates and image tag defaults. The fix is updating the Node.js/TypeScript template to reference a valid tag (`24-bookworm` or `22-bookworm`).
