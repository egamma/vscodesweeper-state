# Issue #226471 — Cannot run dynamically-created shell tasks when a workspace folder is not opened

[https://github.com/microsoft/vscode/issues/226471](https://github.com/microsoft/vscode/issues/226471)

## Summary

Team-authored bug that is still present on main: when no workspace folder is open and no taskSystemInfo is registered, the task system falls back to Platform.platform for OS detection, which can yield the wrong shell (e.g. /bin/bash on Windows). Keeping open per team-member policy.

## Change summary

Dynamically-created shell tasks fail with the wrong shell executable when no workspace folder is open because the task system resolver returns undefined and the platform/OS fallback picks the wrong default.

## Best solution

When `_taskSystemInfoResolver` returns undefined (no workspace folder), the task system should still resolve the correct platform shell — either by registering a fallback taskSystemInfo for the empty-workspace case, or by using the terminal profile resolver's OS detection directly rather than relying on `Platform.platform` in the browser layer. Likely files: `src/vs/workbench/contrib/tasks/browser/terminalTaskSystem.ts` (line ~1520 platform fallback) and `src/vs/workbench/contrib/tasks/browser/abstractTaskService.ts` (line ~2264 resolver lambda).
