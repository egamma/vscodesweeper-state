# Issue #323173 — ENH: Explorer: Show the symlink dest in the tooltip

[https://github.com/microsoft/vscode/issues/323173](https://github.com/microsoft/vscode/issues/323173)

## Summary

Valid feature request to show symlink destination paths in explorer tooltips. Currently the tooltip only shows "Symbolic Link" via the decoration provider; the file service's IFileStat does not expose the symlink target at all, so this requires plumbing new data through the file service. Keeping open for the community upvote process (Backlog Candidates, 0 upvotes, 22 days old).

## Change summary

Show the symlink destination path in the VS Code Explorer tooltip when hovering over a symbolic link.

## Best solution

Extend `IFileStat` (in `src/vs/platform/files/common/files.ts`) to expose an optional `symlinkTarget` property, populate it in the disk file system provider (via `fs.promises.readlink`), propagate it through `ExplorerItem` in `src/vs/workbench/contrib/files/common/explorerModel.ts`, and interpolate it into the tooltip string in `src/vs/workbench/contrib/files/browser/views/explorerDecorationsProvider.ts`. This is a core enhancement since extensions don't control explorer decoration tooltips.
