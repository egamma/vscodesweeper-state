# Issue #53454 — [json] don't suggest top level snippet if there's already an object

[https://github.com/microsoft/vscode/issues/53454](https://github.com/microsoft/vscode/issues/53454)

## Summary

This is an old but still valid JSON/snippets feature request in the Backlog: avoid offering the top-level user-snippet object template once a snippet JSON file already has its root object. There is no strong duplicate or completed-fix evidence in the issue context, and Backlog is not a hard close condition.

## Change summary

User snippet JSON files should not suggest a full top-level snippet object template when inserting it would create a second root object and invalid JSON.

## Best solution

Keep this tracked in the JSON/snippet schema area and have the JSON completion/schema contribution suppress the top-level snippet-file template when the document already contains a root object, or convert the suggestion into an object-member snippet appropriate for insertion inside the existing root.
