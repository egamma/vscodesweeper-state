# Issue #53454 — [json] don't suggest top level snippet if there's already an object

[https://github.com/microsoft/vscode/issues/53454](https://github.com/microsoft/vscode/issues/53454)

## Summary

Still valid feature request — the top-level `defaultSnippets` in the snippet schema has no guard against suggesting itself when the file already contains a root object. The JSON language service offers the snippet unconditionally.

## Change summary

JSON snippet completion suggests a top-level wrapping object even when the file already has one, producing invalid JSON.

## Best solution

Suppress the top-level defaultSnippet suggestion in `vscode-json-languageservice` (or via a conditional in the schema contribution at `src/vs/workbench/contrib/snippets/browser/snippets.contribution.ts`) when the document already has a non-empty root object. This likely requires upstream work in the `vscode-json-languageservice` library to support conditional snippet suppression based on existing document content.
