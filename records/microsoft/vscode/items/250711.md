# Issue #250711 — Copilot overrides language server's preselect item in completion list

[https://github.com/microsoft/vscode/issues/250711](https://github.com/microsoft/vscode/issues/250711)

## Summary

Valid request for better collaboration between inline completions (Copilot) and the suggest widget's preselect behavior. A team member (hediet) is actively investigating this as part of broader IntelliSense/inline-completion collaboration work. Already triaged to Backlog with correct area labels.

## Change summary

Copilot inline completions override the language server's preselect item in the completion list, ignoring context-aware type-based prioritization.

## Best solution

The inline completion provider should respect the suggest widget's preselect semantics — either by deferring to language-server preselect items when confidence is low, or by filtering out completions that are type-incorrect. This is being explored as part of the broader suggest/inline-completion collaboration tracked in the internal backlog. The fix likely lives in `src/vs/editor/contrib/suggest/browser/suggestInlineCompletions.ts` and the inline completion integration layer.
