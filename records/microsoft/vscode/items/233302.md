# Issue #233302 — node debugger also connects to externally executed node scripts

[https://github.com/microsoft/vscode/issues/233302](https://github.com/microsoft/vscode/issues/233302)

## Summary

Valid feature request for controlling NODE_OPTIONS propagation to child processes during debugging. Assigned to 'On Deck' milestone — keeping open per policy. The underlying mechanism (autoAttachChildProcesses / NODE_OPTIONS injection) lives in the vscode-js-debug extension, though a VS Code-level toggle could be useful.

## Change summary

Request a VS Code setting to disable automatic debugger attachment to externally spawned Node.js child processes during debug sessions.

## Best solution

The js-debug extension already exposes `autoAttachChildProcesses` as a launch.json option (defaults to true). A more discoverable global setting or a clearer documentation path would address this — likely implemented in the vscode-js-debug extension (microsoft/vscode-js-debug) rather than VS Code core, unless a workbench-level toggle is desired.
