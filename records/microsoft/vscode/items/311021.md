# Issue #311021 — [Error] unhandlederror-ENOENT: no such file or directory, access '\github.copilot-chat-0.44.1'

[https://github.com/microsoft/vscode/issues/311021](https://github.com/microsoft/vscode/issues/311021)

## Summary

Still reproduces on main: `_createExtensionPathIndex` uses `Promise.all` without per-extension error handling, so a missing extension directory causes an unhandled ENOENT that breaks the entire path index. Affects 362K+ users in v1.117.0 and growing. Keeping open — no fix has been applied.

## Change summary

Extension host throws unhandled ENOENT when building the extension path index if a registered extension directory (e.g. github.copilot-chat) does not exist on disk.

## Best solution

Add a try/catch inside the `Promise.all` map in `_createExtensionPathIndex` (extHostExtensionService.ts) so that a single extension's missing directory is logged and skipped rather than rejecting the entire path index. Additionally investigate why the Copilot Chat extension directory is absent at startup on stable builds (packaging/deployment timing issue).
