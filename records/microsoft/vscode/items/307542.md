## Summary

Valid polish issue: the `github.copilot.chat.searchSubagent.model` setting is a free-form string with no enum, validation, or guidance on valid values, while sibling settings like `chat.exploreAgent.defaultModel` and `chat.planAgent.defaultModel` already use proper enum dropdowns with model IDs and labels. The setting's description references an internal model name (`vscode-agentic-search-router-a`) that users cannot discover from the UI. Still unaddressed on current main.

The same treatment should apply to `github.copilot.chat.exploreAgent.model` if it still exists as a free-form string.
