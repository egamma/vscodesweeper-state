Team-authored bug on the 'On Deck' milestone reporting an incorrect rename suggestion from NES (Next Edit Suggestions). Both the author association and the milestone are hard keep-open signals.

## Change summary

NES incorrectly suggests renaming a type reference (interface usage) when the user only intends to switch to a different interface, not rename the current one.

## Best solution

Refine the NES rename heuristic to distinguish type declarations from type references — avoid suggesting renames at usage sites in type-reference positions, as discussed in the comment by dbaeumer. The fix likely lives in the NES/completions pipeline that generates rename suggestions.
