# Issue #314092 — Scripts in package.json not checked for auto approve

[https://github.com/microsoft/vscode/issues/314092](https://github.com/microsoft/vscode/issues/314092)

## Summary

Team-authored bug that remains valid on main: the NpmScriptAutoApprover auto-approves any command wrapped in a package.json script without checking the script's actual content against the deny list. Keeping open per team-author policy.

## Change summary

Running a command directly (e.g. `exit 42`) prompts for approval, but running the same command via an npm script in package.json bypasses the approval check.

## Best solution

The `NpmScriptAutoApprover.isCommandAutoApproved` method should resolve the script content from package.json and pass it through the deny-list/allow-list rules in `CommandLineAutoApprover` before granting auto-approval. The fix lives in `src/vs/workbench/contrib/terminalContrib/chatAgentTools/browser/tools/commandLineAnalyzer/autoApprove/npmScriptAutoApprover.ts`.
