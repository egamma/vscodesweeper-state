# Issue #281507 — VS Code freezes when expanding a folder containing extremely large number of files (300M+ files)

[https://github.com/microsoft/vscode/issues/281507](https://github.com/microsoft/vscode/issues/281507)

## Summary

Real freeze bug acknowledged by a maintainer (benibenj). An open community PR (#320536) is actively attempting a fix with paged loading. Keep open while that PR is reviewed.

## Change summary

Expanding a folder with an extremely large number of files in the Explorer freezes the entire VS Code UI until the folder is collapsed.

## Best solution

Review and land (or iterate on) https://github.com/microsoft/vscode/pull/320536 which adds a configurable `explorer.fileLimit` setting with paged 'Load more' rendering. This also addresses the related https://github.com/microsoft/vscode/issues/237394. A follow-up streaming `readdir` provider API could further improve the OS-level enumeration cost.
