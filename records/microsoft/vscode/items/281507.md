# Issue #281507 — VS Code freezes when expanding a folder containing extremely large number of files (300M+ files)

[https://github.com/microsoft/vscode/issues/281507](https://github.com/microsoft/vscode/issues/281507)

## Summary

Real freeze bug with an open community PR (#320536) attempting a fix — keeping open so the maintainer can evaluate the PR and the related canonical issue #237394.

## Change summary

Expanding a folder with an extremely large number of files in the Explorer causes a full UI freeze because all entries are enumerated synchronously.

## Best solution

Evaluate the open PR https://github.com/microsoft/vscode/pull/320536 which adds paged loading for large folders. Also consider consolidating with https://github.com/microsoft/vscode/issues/237394 (same core problem: no safeguard against expanding huge folders). The fix likely lives in the Explorer data source and/or `asyncDataTree.ts` — either cap the initial expansion or implement incremental loading.
