[https://github.com/microsoft/vscode/issues/286087](https://github.com/microsoft/vscode/issues/286087)

## Summary

Valid extension-API feature request asking for a way to reuse VS Code's built-in Monaco editor instance from extensions (especially in webviews) instead of bundling a separate copy. Low traction so far (2 upvotes, no discussion), sitting in Backlog — keeping open for the community process.

## Change summary

Allow extensions to embed or reuse VS Code's own Monaco editor component rather than bundling a separate monaco-editor package.

## Best solution

This belongs in the extension API surface — either a dedicated webview editor component API or a mechanism to embed a lightweight editor panel from the host. The `editor` and `extensions` area owners should weigh whether exposing an embeddable editor primitive (beyond the existing CustomEditor/Webview APIs) is feasible without coupling extensions to internal Monaco internals.
