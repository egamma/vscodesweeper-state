# Issue #278364 — Monaco editor hijacks all input context menus

[https://github.com/microsoft/vscode/issues/278364](https://github.com/microsoft/vscode/issues/278364)

## Summary

Still reproduces on current main: TextInputActionsProvider registers its context menu listener on layoutService.mainContainer, which defaults to document.body, intercepting right-click on all input/textarea elements in the page — not just VS Code-owned ones. The issue carries milestone 'On Deck', so it stays open.

## Change summary

When mainContainer is document.body, the workbench's TextInputActionsProvider hijacks native context menus on non-VS-Code input elements embedded alongside Monaco editors.

## Best solution

Scope the context menu interception in TextInputActionsProvider so it only fires on inputs that are descendants of a VS Code-owned container (e.g. check whether the target input is inside a workbench part or Monaco editor DOM subtree), or provide an opt-out mechanism for embedders. The fix lives in src/vs/workbench/browser/actions/textInputActions.ts.
