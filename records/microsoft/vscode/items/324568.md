# Issue #324568 — Agents window detail panel: Why switch to Changes if no changes generated?

[https://github.com/microsoft/vscode/issues/324568](https://github.com/microsoft/vscode/issues/324568)

## Summary

Team-authored UX polish request for the Agents window detail panel: defer the switch to the Changes tab until changes actually exist. Keeping open for maintainer judgment.

## Change summary

The Agents window side panel switches to the Changes tab when a session starts, even when no changes have been generated yet, showing an empty view.

## Best solution

Conditionally switch to the Changes tab only when the session produces its first file change; until then, keep the panel on its current tab. The logic likely lives in `src/vs/sessions/` where the detail panel reacts to session state transitions.
