# Issue #127180 — Git - Inconsistent use of git.path variable if containing ${execPath}

[https://github.com/microsoft/vscode/issues/127180](https://github.com/microsoft/vscode/issues/127180)

## Summary

Still reproduces on main: the git extension reads `git.path` via `workspace.getConfiguration().get()` which does not resolve `${execPath}` or `${env:...}` variables — the `IConfigurationResolverService` is only used for tasks/launch configs, not general settings consumed by extensions. The bug is real but niche (portable mode with variable-based paths).

## Change summary

Setting `git.path` with `${execPath}` or `${env:...}` variables fails because the git extension does not resolve configuration variables.

## Best solution

The git extension should use `IConfigurationResolverService` (or a lighter-weight equivalent available to extensions) to resolve predefined variables in `git.path` before passing hints to `findGit()`. The fix would be in `extensions/git/src/main.ts` around line 42 where `pathValue` is read. Alternatively, this could be solved at the configuration API level by offering extensions an opt-in resolved-value getter.
