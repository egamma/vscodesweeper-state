# Issue #295212 — Setting to make side by side diff view the default for changes made by copilot

[https://github.com/microsoft/vscode/issues/295212](https://github.com/microsoft/vscode/issues/295212)

## Summary

Valid feature request with 43 upvotes, already promoted to Backlog. No implementation found on current main — the diff view mode for Copilot changes is still only togglable per-session via the button, with no persistent setting to default to side-by-side.

## Change summary

Add a setting to make side-by-side diff the default view mode when reviewing Copilot-generated changes.

## Best solution

Add a configuration (e.g. `chat.editing.defaultDiffViewMode`) that controls whether Copilot change diffs default to inline or side-by-side, wired into the chat editing diff editor creation in `src/vs/workbench/contrib/chat/browser/widget/chatContentParts/codeBlockPart.ts` and any multi-diff editor openings. The existing `diffEditor.renderSideBySide` could potentially be reused, or a chat-specific setting added.
