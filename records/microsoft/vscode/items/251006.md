# Issue #251006 — Feature Request:  tasks get inputs from nodejs or shell

[https://github.com/microsoft/vscode/issues/251006](https://github.com/microsoft/vscode/issues/251006)

## Summary

Valid feature request with 28 upvotes, promoted to Backlog. Requests a new `process` input type for tasks/launch configs that runs an external Node.js or shell script to dynamically generate pick-list options — distinct from the existing `command` type which only runs VS Code extension commands.

## Change summary

Allow tasks.json inputs to invoke an external process (Node.js/shell script) to dynamically generate candidate option lists at runtime.

## Best solution

Implement a `process` (or `shell`) input type in `src/vs/workbench/services/configurationResolver/` that spawns an external command and parses its stdout as pick-list options. This closes the gap between the static `pickString` and the extension-requiring `command` type, letting users get dynamic options without writing a full extension.
