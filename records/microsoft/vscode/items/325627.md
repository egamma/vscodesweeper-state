# Issue #325627 — Testing: Collapsible thinking close animation doesn't work during reasoning

[https://github.com/microsoft/vscode/issues/325627](https://github.com/microsoft/vscode/issues/325627)

## Summary

Team-authored issue — keeping open. The bug is traceable in the CSS: during streaming, the collapsed state uses different CSS paths (max-height clamp and display overrides in chatThinkingContent.css) that lack a smooth transition, whereas the completed state likely benefits from a proper collapse animation.

## Change summary

Collapsing the thinking section during active streaming snaps shut without an animation; the close animation only works after streaming completes.

## Best solution

Add a CSS transition (or a JS-driven animation) to the streaming-collapsed path in `chatThinkingContent.css` so that collapsing during active reasoning animates the same way it does after completion. The key rules are around lines 422–470 of `chatThinkingContent.css` and the general collapsed rule at line 2945 of `chat.css`.
