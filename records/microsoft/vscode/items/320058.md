# Issue #320058 — Per‑Session AI Credit Budget for Copilot Agent Sessions

[https://github.com/microsoft/vscode/issues/320058](https://github.com/microsoft/vscode/issues/320058)

## Summary

Valid feature request for a per-session AI credit budget. Not implemented on current main — the codebase has session cost display and monthly credit limits, but no per-session hard cap. Sitting in Backlog Candidates with the community upvote process running; keeping open for that process.

## Change summary

Add a per-session AI credit budget that pauses agent execution when a configurable credit threshold is reached, requiring explicit user approval to continue.

## Best solution

Implement a session-scoped credit budget as a setting (or policy for enterprise) that integrates with the existing session cost tracking in `src/vs/workbench/contrib/chat/browser/widgetHosts/viewPane/chatContextUsageWidget.ts` and the agent turn execution flow. The Copilot CLI/SDK already shipped this (July 2026 changelog), so parity with that implementation is the natural path. The existing `chat.agent.maxRequests` per-turn limit is not a substitute — a credit-denominated session cap is a distinct control.
