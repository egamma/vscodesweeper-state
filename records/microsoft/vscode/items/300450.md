# Issue #300450 — Main menu is inactive and inaccessible sometimes

[https://github.com/microsoft/vscode/issues/300450](https://github.com/microsoft/vscode/issues/300450)

## Summary

Legitimate bug: the custom titlebar's focus-state tracking (the `inactive` CSS class on the menubar and titlebar) can fall out of sync with actual window focus, causing the menu to appear disabled when focused or active when unfocused. The underlying `onDidChangeFocus` event path in `menubarControl.ts` lines 692–706 toggles the class, but rapid focus changes can produce a stale state. Still reproduces on main — no fix found. Related to but not identical to #241405.

## Change summary

The custom title bar and menu bar sometimes show the wrong active/inactive visual state, and menu dropdowns occasionally render behind other UI elements.

## Best solution

Investigate the race in focus-state tracking between `NativeHostService.onDidChangeFocus` (which uses `Event.latch` over focus/blur/activeWindow events) and the `inactive` class toggle in `CustomMenubarControl.onDidChangeWindowFocus` (`src/vs/workbench/browser/parts/titlebar/menubarControl.ts` ~line 692). A guard that re-checks `document.hasFocus()` after the class toggle, or debouncing the focus event, may resolve the desync. The z-index sub-issue (menu rendering behind content) may be a separate problem in `titlebarpart.css`.
