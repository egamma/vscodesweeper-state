# Issue #296426 — Rendering for extension info is broken when clicking show suggestion nudge (with many suggestions)

[https://github.com/microsoft/vscode/issues/296426](https://github.com/microsoft/vscode/issues/296426)

## Summary

Still reproduces on main: clicking "Show Recommendations" when multiple extensions are suggested opens each extension individually via a loop, and with modal editors enabled this creates stacked modal dialogs causing rendering damage.

## Change summary

Clicking the extension suggestion nudge with multiple recommended extensions causes stacked/overlapping dialogs with broken rendering.

## Best solution

The "Show Recommendations" handler in `extensionRecommendationNotificationService.ts` (lines 309-311) should either open only the first extension (or a summary view) instead of looping through all extensions, or should use the modal navigation pattern already built for `ExtensionsList.openExtension` (see `buildModalNavigationForPagedList` added in commit a9b50c8372a). The search view opened on line 312 already shows all recommendations — opening each extension editor individually is redundant and causes the stacking.
