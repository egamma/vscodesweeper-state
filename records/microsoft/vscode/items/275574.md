# Issue #275574 — Use reasoning level low for commit message generation

[https://github.com/microsoft/vscode/issues/275574](https://github.com/microsoft/vscode/issues/275574)

## Summary

The reasoning-level-low optimization was never implemented; instead, the model was switched back from gpt-5-mini to gpt-4.1 (copilot-utility-small) the day after this issue was filed (commit c6927a0975b), which likely addressed the speed concern. The feature request remains valid if the model is ever switched back to a reasoning-capable model, but may be moot for the current endpoint.

## Change summary

Pass reasoning level 'low' to the API when generating commit messages to reduce latency.

## Best solution

If the current copilot-utility-small model (gpt-4.1) is fast enough, this can be closed as moot — the speed problem was solved by switching models. If the team plans to move commit message generation back to a reasoning-capable model (like gpt-5-mini), then `reasoningEffort: 'low'` should be threaded through `OptionalChatRequestParams` in `extensions/copilot/src/platform/networking/common/fetch.ts` and passed in the `makeChatRequest` call in `extensions/copilot/src/extension/prompt/node/gitCommitMessageGenerator.ts`. The area owner (@lszomoru / @isidorn) should decide whether this is still needed given the model change.
