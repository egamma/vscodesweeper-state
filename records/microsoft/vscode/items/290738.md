# Issue #290738 — Github copilot default enables specific tools for agent mode.

[https://github.com/microsoft/vscode/issues/290738](https://github.com/microsoft/vscode/issues/290738)

## Summary

Still open and reproducible from source: the Copilot extension's tool deferral system marks certain tools as 'non-deferred' (always sent to the model) regardless of the user's tool selection state in the VS Code UI. Two team members have engaged but no fix has been committed.

## Change summary

Disabling all tools in the chat UI still results in certain tools (e.g. replace_string_in_file, runTests, tool_search_tool_regex) being sent to the model.

## Best solution

The platform's tool enablement state (from chatSelectedTools.ts) should be enforced in the Copilot extension's tool collection path, overriding the non-deferred classification when the user explicitly disables tools. Fix likely spans both `extensions/copilot/src/extension/tools/common/toolDeferralService.ts` and the VS Code platform's tool filtering in the chat service.
