# Issue #139340 — Long text always wrap

[https://github.com/microsoft/vscode/issues/139340](https://github.com/microsoft/vscode/issues/139340)

## Summary

Keep open: the reported behavior is the editor's defensive long-line wrapping, and maintainer discussion already identified a possible product change to expose control over it. I found related unmerged work, but current main still appears to force wrapping for dominated-by-long-lines models in the relevant path, so this is not a completed issue on main.

## Change summary

Allow users to disable or control automatic word wrapping for files dominated by very long lines when `editor.wordWrap` is otherwise off.

## Best solution

Add and ship an editor option in the editor-wrapping area that controls the defensive wrapping path for minified/very-long-line files, with tests around `isDominatedByLongLines` and `wrappingInfo` in `src/vs/editor/common/config/editorOptions.ts` and editor configuration tests.
