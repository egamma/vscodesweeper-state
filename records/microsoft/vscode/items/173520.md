# Issue #173520 — Please Let Debug Console Consistent With Editor

[https://github.com/microsoft/vscode/issues/173520](https://github.com/microsoft/vscode/issues/173520)

## Summary

Still valid: the debug console REPL input uses `getSimpleEditorOptions()` which hardcodes many editor behaviors and does not inherit user editor settings like `autoClosingBrackets`, `autoSurround`, `emptySelectionClipboard`, or `autoClosingQuotes`. The request for better editor-setting consistency remains unaddressed on main.

## Change summary

The debug console REPL input should inherit more editor settings (autoClosingBrackets, autoSurround, emptySelectionClipboard, autoClosingQuotes, etc.) from the user's editor configuration for a consistent typing experience.

## Best solution

Extend `getSimpleEditorOptions()` in `src/vs/workbench/contrib/codeEditor/browser/simpleEditorOptions.ts` (or override in `repl.ts` at the call site) to read additional editor settings from `IConfigurationService` — specifically `autoClosingBrackets`, `autoSurround`, `emptySelectionClipboard`, and `autoClosingQuotes`. This benefits all simple editor widgets (REPL, breakpoint condition editor, etc.) if done in the shared function, so scope carefully. A team member (roblourens) acknowledged the gap in the first comment.
