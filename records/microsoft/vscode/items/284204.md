## Summary

The agent worktree UX has improved substantially since this was filed (isolation picker, branch picker, apply-changes-to-parent action), but the specific requests — AI-generated descriptive branch names and a one-click commit→merge→delete flow — remain partially unaddressed. Keeping open as the area is under active iteration.

## Change summary

Improve worktree branch naming (AI-generated or user-defined) and add a one-click merge-back workflow for background agent sessions.

## Best solution

The naming could be improved by incorporating the session title or AI-generated slug (e.g. `copilot/implement-feature-x`) rather than `copilot-worktree-<hash>`. The merge flow exists via 'Apply Changes to Parent Repository' but could be streamlined into a single action that commits, merges, and optionally deletes the worktree. This belongs in the agents-window/copilot-chat-sessions area.
