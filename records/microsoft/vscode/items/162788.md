# Issue #162788 — Separated select and focus on delete in file explorer

Team-authored polish issue (MEMBER). No commits reference this issue and no obvious fix was found in the explorer delete/focus handling on current main. Keeping open for maintainer judgment.

## Change summary

After deleting a file from a multi-selection context menu in the explorer, focus and selection can end up on different files when the right-clicked file differs from the focused file.

## Best solution

Ensure that after a delete operation in the file explorer triggered via context menu on a multi-selection, the focus and selection converge on the same remaining file. The fix likely lives in the delete handler in `src/vs/workbench/contrib/files/browser/fileActions.ts` or the explorer view's response to item removal in `src/vs/workbench/contrib/files/browser/views/explorerView.ts`.
