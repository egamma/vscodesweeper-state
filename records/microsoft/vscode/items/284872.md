# Issue #284872 — VS Code saves Chinese-named PowerShell scripts as UTF‑8 BOM, causing PowerShell parser failures

[https://github.com/microsoft/vscode/issues/284872](https://github.com/microsoft/vscode/issues/284872)

## Summary

This is not a VS Code bug. VS Code defaults to UTF-8 without BOM (`files.encoding: utf8`), not UTF-8 with BOM. The reported parsing failures are in PowerShell's parser interacting with OneDrive metadata — neither component is owned by VS Code. The premise that VS Code automatically saves BOM for non-ASCII filenames is incorrect.

## Change summary

PowerShell parser fails on scripts with Chinese filenames stored under OneDrive when saved as UTF-8 BOM.

## Best solution

Close as a support question. The issue belongs in the PowerShell repo (https://github.com/PowerShell/PowerShell) if the parser bug is real, or with OneDrive if metadata streams corrupt file content. VS Code's default is already UTF-8 without BOM; the user can verify via the status bar encoding indicator.
