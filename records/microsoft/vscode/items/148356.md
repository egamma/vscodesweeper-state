## Evidence

- **current discussion:** A maintainer reported on 2024-09-19 that EditContext made the BS character stop appearing while typing, but noted it could still appear when copy-pasting and that VS Code may need to render the BS character differently.
- **reporter status:** The original reporter said on 2024-09-09 that they could no longer reproduce the issue on VS Code 1.93.0, even without `editor.experimentalEditContextEnabled`.
- **current main input path:** `src/vs/editor/common/config/editorOptions.ts` registers `editor.editContext` with default `true`, and `src/vs/editor/browser/view.ts` instantiates `NativeEditContext` when `EditorOption.effectiveEditContext` is enabled.
- **implementation history:** Commit 288596b9280 introduced the experimental EditContext editor input path, and commit 312284e7db0 migrated it to the non-experimental `editor.editContext` setting with a default of `true`.
- **remaining rendering path:** `src/vs/editor/common/viewLayout/viewLineRenderer.ts` renders control characters below char code 32 as Unicode control pictures when `editor.renderControlCharacters` is enabled, while that setting defaults to `true` in `src/vs/editor/common/config/editorOptions.ts`.
- **related item:** The thread references a Korean IME reproduction at https://github.com/microsoft/vscode/issues/201952, but unauthenticated GitHub access was unavailable in this read-only environment to confirm it as a canonical duplicate.
