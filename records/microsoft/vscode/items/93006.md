# Issue #93006 — [rename on type] strange behavior with auto indent in closing tag

[https://github.com/microsoft/vscode/issues/93006](https://github.com/microsoft/vscode/issues/93006)

## Summary

Still valid on main: the interaction between `AutoIndentOperation` (triggered by user typing at `autoIndent >= Full`) and linked editing's deferred `_syncRanges` (via `executeEdits`) is unguarded. Auto-indent can re-indent or shift the closing tag line before the linked editing sync mirrors the character to the opening tag, causing the described desync. No commit has addressed this.

## Change summary

When editing an HTML closing tag with linked editing enabled and `autoIndent: full`, the first typed character is lost in the opening tag and the closing tag's indentation jumps unexpectedly.

## Best solution

Guard the linked editing sync path so that auto-indent edits produced by `AutoIndentOperation.getEdits` during user typing do not interfere with the synced ranges. One approach: suppress auto-indent when typing inside a linked editing range (check `LinkedEditingContribution` decoration presence), or apply the synced edit atomically with the user's type so auto-indent sees the final state. Fix likely touches `src/vs/editor/contrib/linkedEditing/browser/linkedEditing.ts` and possibly `src/vs/editor/common/cursor/cursorTypeEditOperations.ts`.
