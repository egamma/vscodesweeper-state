# Issue #93006 — [rename on type] strange behavior with auto indent in closing tag

[https://github.com/microsoft/vscode/issues/93006](https://github.com/microsoft/vscode/issues/93006)

## Summary

Issue has milestone 'On Deck' (a scheduling milestone), so it must remain open. The linked editing contribution uses `executeEdits` without suppressing auto-indent for mirror edits, which is the likely root cause and appears unfixed on main.

## Change summary

Linked editing (rename on type) in HTML closing tags triggers auto-indent, causing the closing tag to jump to the wrong column and lose characters from the opening tag.

## Best solution

The linked editing mirror edit path in `src/vs/editor/contrib/linkedEditing/browser/linkedEditing.ts` should suppress auto-indent when applying synced edits to mirror ranges, likely by using an edit option that skips indentation adjustment.
