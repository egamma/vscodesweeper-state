# Issue #260871 — Explorer/Editor: Copy-pasting multiple files shows new file as selected but editor stays on original

[https://github.com/microsoft/vscode/issues/260871](https://github.com/microsoft/vscode/issues/260871)

## Summary

Confirmed in source: the paste handler intentionally opens the editor only when a single file is pasted (line 1272 guard), but always selects the first target in the explorer regardless of count — creating an inconsistent explorer-selection vs. editor-focus state on multi-file paste. Still present on current main.

## Change summary

Copy-pasting multiple files selects the first pasted file in the explorer but does not open it in the editor, unlike single-file paste which does both.

## Best solution

Extend the multi-file paste path in `src/vs/workbench/contrib/files/browser/fileActions.ts` (around line 1272) to also open the selected (first) target file in the editor when it is not a directory, matching the single-file behavior. This keeps the existing explorer selection and just adds the missing editor open call for multi-file pastes.
