# Issue #297472 — I am prompted again  to edit sensitive files after selecting "allow in session"

[https://github.com/microsoft/vscode/issues/297472](https://github.com/microsoft/vscode/issues/297472)

## Summary

Still appears unfixed on main — the 'Allow in this Session' approval for editing sensitive files (like package.json) is not being remembered across subsequent edit operations in the same Copilot CLI session. No commit references this issue, and the nearby b7527dded0f commit only changed button layout, not persistence logic.

## Change summary

Clicking 'Allow in this Session' when prompted about editing sensitive files does not prevent the same prompt from appearing again for subsequent edits to the same file in the same session.

## Best solution

The root cause likely lives in how the Copilot CLI permission flow (`extensions/copilot/src/extension/chatSessions/copilotcli/node/permissionHelpers.ts`) invokes the `CoreConfirmationTool` via `toolsService.invokeTool`. Each write permission request re-checks `requiresFileEditconfirmation` against the config patterns and re-invokes the confirmation tool, but the session-level approval stored by the confirmation service may not be reaching the `shouldAutoConfirm` check due to missing or mismatched session context. The fix should ensure that the session resource is properly threaded through the `invokeTool` call so that `shouldAutoConfirm` can find the stored session-level approval for `vscode_get_confirmation`.
