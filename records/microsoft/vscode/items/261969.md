## Summary

Already properly triaged as under-discussion with a team member engaged. The request is to streamline reverting staged changes (skip the unstage step). A maintainer (lszomoru) indicated the current multi-step behavior has always been the case and is by design given git's staging model. Keeping open for the team to decide whether a shortcut is warranted.

## Change summary

Reverting a staged git change requires unstaging first, then reverting — the user wants a single-action revert from the quick diff decorator on staged changes.

## Best solution

The git area owner (lszomoru) is already engaged. The decision is whether the quick diff gutter action on staged changes should offer a direct 'revert' that combines unstage+revert in one step, or whether the current explicit two-step flow is the right UX given overlapping staged/unstaged scenarios. This belongs in `src/vs/workbench/contrib/scm/` (quick diff contributions). Leave for the area owner to decide.
