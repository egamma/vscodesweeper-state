# Issue #228110 — SCM Graph - Hover does not escape markdown in commit message

[https://github.com/microsoft/vscode/issues/228110](https://github.com/microsoft/vscode/issues/228110)

## Summary

Team-authored issue, kept open. The bug is confirmed in current source: commit messages are injected via `appendMarkdown` without escaping markdown syntax tokens, so characters like `#` render as headings.

## Change summary

SCM Graph hover renders markdown syntax in commit messages (e.g. `# heading`) instead of displaying them as literal text.

## Best solution

In `extensions/git/src/hover.ts` line 125, the message is passed through `appendMarkdown` after only escaping image syntax (`![`). It should also escape other markdown tokens (e.g. `#`, `*`, `_`, etc.) — likely by using `escapeMarkdownSyntaxTokens` on the message before the `emojify` call, or by restructuring to use `appendText` for the message body while preserving emoji rendering separately.
