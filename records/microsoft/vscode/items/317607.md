# Issue #317607 — Copilot overwriting newer files when switching between nested workspaces

[https://github.com/microsoft/vscode/issues/317607](https://github.com/microsoft/vscode/issues/317607)

## Summary

The chat editing session restore path (`restoreFromSnapshot` in `chatEditingModifiedDocumentEntry.ts`) does not check whether the file has been externally modified since the session was active, causing silent data loss when switching between nested workspaces with pending Copilot edits. Still present on main.

## Change summary

Returning to a Copilot edit session silently overwrites a file that was modified externally (e.g. in a nested workspace), discarding the newer content.

## Best solution

Before restoring a snapshot in `ChatEditingModifiedDocumentEntry.restoreFromSnapshot`, compare the file's current disk content (or mtime) against the snapshot's expected original; if they diverge, show a conflict prompt instead of silently overwriting. The recent `af1ccce9ae4` commit added a confirmation for *undoing* requests but does not cover the session-restoration path.
