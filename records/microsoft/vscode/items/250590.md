# Issue #250590 — Extension host manager caches activation events of uninstalled extensions

[https://github.com/microsoft/vscode/issues/250590](https://github.com/microsoft/vscode/issues/250590)

## Summary

Keep open: the report is actionable and current main still appears to have the caching shape described, with no fixing commit or duplicate identified.

## Change summary

Activation events requested while an extension is uninstalled can be replayed and cached by a restarted extension host manager, preventing later activation after the extension is reinstalled.

## Best solution

Fix the extension-host activation replay path so activation events with no currently corresponding extension are not cached as completed by `ExtensionHostManager`, or are filtered before being passed as initial activation events. Add focused coverage around uninstalling an extension, restarting the extension host, then reinstalling/activating it by event.
