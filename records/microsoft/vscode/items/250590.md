# Issue #250590 — Extension host manager caches activation events of uninstalled extensions

[https://github.com/microsoft/vscode/issues/250590](https://github.com/microsoft/vscode/issues/250590)

## Summary

The root cause remains on main: ExtensionHostManager's _cachedActivationEvents map caches activation events for uninstalled extensions and replays them on EH restart, causing errors. PR #250551 was only a workaround in chatServiceImpl.ts.

## Change summary

Extension host manager replays cached activation events for extensions that have been uninstalled, causing errors when the EH restarts.

## Best solution

Filter or invalidate entries in `_cachedActivationEvents` and `_resolvedActivationEvents` when extensions are uninstalled, so stale activation events for removed extensions are not replayed on EH restart. The fix belongs in `src/vs/workbench/services/extensions/common/extensionHostManager.ts`.
