# Issue #205915 — [SUGGESTION] Test Explorer is able to be grouped

[https://github.com/microsoft/vscode/issues/205915](https://github.com/microsoft/vscode/issues/205915)

## Summary

Valid feature request with 24 upvotes, promoted to Backlog. The Test Explorer still only supports Tree/List view modes and sorting by location/status/duration — no 'group by tag/trait' capability exists on current main.

## Change summary

Allow Test Explorer to group tests by tag, trait, or category (similar to Visual Studio 2022's grouping options).

## Best solution

Implement a 'group by' option in the Test Explorer view that lets users group tests by tag (the existing tag infrastructure already supports filtering). This would live in `src/vs/workbench/contrib/testing/browser/testingExplorerView.ts` and the explorer projections, adding a new view grouping enum alongside the existing sorting options.
