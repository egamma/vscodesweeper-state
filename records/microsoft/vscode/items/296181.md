# Issue #296181 — Chat history synching between Claude Code and Github Copilot

[https://github.com/microsoft/vscode/issues/296181](https://github.com/microsoft/vscode/issues/296181)

## Summary

Valid feature request for a toggle to hide Claude Code sessions from Copilot Chat's session sidebar. The `claudeAgent.enabled` setting exists but reportedly does not gate the session reader, only the session provider's `when` clause. With 18 upvotes and active discussion, this has meaningful traction and should stay open for the Copilot team to address.

## Change summary

Claude Code chat sessions appear in the Copilot Chat sessions sidebar with no effective way to hide them, even when `github.copilot.chat.claudeAgent.enabled` is set to false.

## Best solution

Make the `github.copilot.chat.claudeAgent.enabled` setting actually gate the Claude session content provider in `extensions/copilot/src/extension/chatSessions/vscode-node/claudeChatSessionContentProvider.ts` so disabling it prevents Claude sessions from appearing in the sidebar. Alternatively, add a dedicated `sessionProvider.enabled` setting. Related issue https://github.com/microsoft/vscode/issues/319989 (deleting Claude sessions) addresses adjacent UX but is distinct.
