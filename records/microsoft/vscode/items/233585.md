# Issue #233585 — File comparer freezes in the background

[https://github.com/microsoft/vscode/issues/233585](https://github.com/microsoft/vscode/issues/233585)

## Summary

Fixed on main: when a diff editor tab is closed, model disposal now propagates into the web worker's diff algorithm loop via ModelDisposalTimeout, causing long-running computations to abort early and unblock subsequent diff requests.

## Change summary

Closing a diff editor with a heavy computation in progress blocks all subsequent diff comparisons until the first one finishes.

## Best solution

Keep the shipped fixes — c21846a1718 (model disposal checking in diff algorithm) and 817db05a564 (cancellation handling in diffEditorViewModel). No further change needed beyond confirming on the latest Insiders build.
