# Issue #322341 — SIGPIPE error in Remote SSH Extension Host on client disconnect

[https://github.com/microsoft/vscode/issues/322341](https://github.com/microsoft/vscode/issues/322341)

## Summary

Still reproduces on main: PersistentProtocol.sendDisconnect() flushes to a closed socket without catching EPIPE, triggering the SIGPIPE handler in remoteExtensionHostAgentServer.ts. An open PR (#322398) addresses it but has not merged.

## Change summary

Remote SSH server logs a spurious 'Unexpected SIGPIPE' error on every graceful client disconnect.

## Best solution

Merge or iterate on the approach in PR #322398 — wrap the flush() call in PersistentProtocol.sendDisconnect() to swallow EPIPE when the socket is already closed, matching the existing pattern in ProtocolWriter.dispose().
