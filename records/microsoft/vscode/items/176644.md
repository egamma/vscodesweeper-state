# Issue #176644 — balancedBracketScopes per bracket type

[https://github.com/microsoft/vscode/issues/176644](https://github.com/microsoft/vscode/issues/176644)

## Summary

Team-authored feature request for per-bracket-type scope filtering in bracket colorization; must remain open for maintainer judgment.

## Change summary

Allow `balancedBracketScopes` / `unbalancedBracketScopes` to be specified per bracket character (e.g. only for `<`/`>`) rather than applying globally to all bracket pairs.

## Best solution

Implement per-bracket-type scope configuration in the grammar contribution point so that angle brackets in TypeScript can use a whitelist (`punctuation.definition.typeparameters`) while other brackets remain unrestricted — eliminates false positives from operators and arrows without over-filtering.
