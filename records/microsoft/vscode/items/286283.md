# Issue #286283 — [Editor] Incorrect word wrap for monospace CJK fonts

[https://github.com/microsoft/vscode/issues/286283](https://github.com/microsoft/vscode/issues/286283)

## Summary

Still present on main: the fullwidth character width measurement uses U+FF4D (ｍ), which some monospace CJK fonts don't include, causing fallback to a different font and incorrect width calculation that breaks word wrap for CJK text.

## Change summary

Word wrap incorrectly calculates CJK line length when the font lacks fullwidth Latin characters (U+FF4D) used for width measurement.

## Best solution

Change the measurement character in `src/vs/editor/browser/config/fontMeasurements.ts` line 172 from U+FF4D (fullwidth Latin m) to a more universally available fullwidth/CJK character (e.g. U+4E2D '中' or another common CJK ideograph guaranteed present in CJK fonts).
