# Issue #193395 — Quick chat container can be focused and has focus border cut off

[https://github.com/microsoft/vscode/issues/193395](https://github.com/microsoft/vscode/issues/193395)

## Summary

Still present on main: the quick-input-widget container has tabIndex=-1, making it mouse-focusable with no dedicated outline handling for the quick chat host scenario. The focus border clipping remains unaddressed.

## Change summary

Clicking just above the textbox in quick chat focuses the container, showing a clipped focus border that serves no purpose.

## Best solution

Either remove focusability from the container when used as a quick chat host (set tabIndex to undefined or add `outline: none` for the quick chat context), or ensure the focus border renders fully. The fix likely touches `src/vs/platform/quickinput/browser/quickInputController.ts` (container tabIndex) or the quick-input-widget CSS.
