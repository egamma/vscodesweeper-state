# Issue #306961 — Sessions App: Cannot attach file from chosen branch

[https://github.com/microsoft/vscode/issues/306961](https://github.com/microsoft/vscode/issues/306961)

## Summary

Valid bug in the Sessions App file picker: when a user selects a branch for a new worktree-based session, the file picker still resolves against the currently checked-out branch rather than the selected branch, so files unique to the feature branch cannot be attached as context. Kept open per team-authored and milestone rules.

## Change summary

The Sessions App file picker does not update its search context when the user selects a different branch for a new session, preventing attachment of files that only exist in that branch.

## Best solution

When the user selects a branch in the new-session flow, update the file picker's resolution context to reflect that branch's file tree (e.g. via a git-tree lookup) rather than relying solely on the on-disk workspace folder. The fix likely lives in the sessions chat input / attachment picker integration under `src/vs/sessions/`.
