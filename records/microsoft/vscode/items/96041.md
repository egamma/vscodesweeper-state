# Issue #96041 — unable to input chinese character

[https://github.com/microsoft/vscode/issues/96041](https://github.com/microsoft/vscode/issues/96041)

## Summary

Still a valid open bug — the snap IME integration code in `resources/linux/snap/electron-launch` (lines 263–272) sets up GTK IM modules but does not fully resolve ibus/fcitx input for all host configurations. The issue is confirmed, carries `help wanted`, has 21 upvotes, and the underlying architectural limitation (containerized GTK communicating with host IM frameworks) remains unresolved on current main.

## Change summary

Chinese (and other CJK/non-Latin) character input via ibus or fcitx does not work in the VS Code snap package on Linux.

## Best solution

The snap's `electron-launch` script already attempts ibus/fcitx integration via GTK IM module caching, but the approach is incomplete for modern setups (Wayland, newer ibus protocols, portal-based IM). The best end state is either adopting the input-method portal (xdg-desktop-portal) for IME communication across the snap boundary, or migrating the snap to core22+ with the `desktop` extension that handles IM modules natively. This likely requires changes in `resources/linux/snap/electron-launch` and `resources/linux/snap/snapcraft.yaml`.
