[https://github.com/microsoft/vscode/issues/171030](https://github.com/microsoft/vscode/issues/171030)

## Summary

Valid feature request for context-dependent auto-closing pairs, still unimplemented on main. Only 2 upvotes and a team member flagged potential scope concerns, but it's in Backlog and the underlying need (context-aware auto-closing) is legitimate. Keeping open for the community process.

## Change summary

Allow auto-closing pairs to be conditionally disabled based on richer context (e.g. TextMate scopes or a tokenizer-agnostic mechanism) beyond just 'string' and 'comment'.

## Best solution

If pursued, add a tokenizer-agnostic context mechanism to `notIn` (avoiding a hard TextMate dependency, as hediet noted). The enum in `languageConfigurationExtensionPoint.ts` line 604 currently only allows `['string', 'comment']`; a richer discriminator would live there and in `src/vs/editor/common/languages/autoClosingPairs.ts`. Alternatively, the finalized `LanguageConfiguration.autoClosingPairs` extension API could expose this. Low community traction so far.
