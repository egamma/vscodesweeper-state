## Summary

Team-authored feature request for merge editor UX improvements — kept open per policy. The request is about surfacing awareness to users when the merge editor auto-resolves a conflict that git itself could not resolve, so users can verify correctness.

## Change summary

Design an awareness/notification strategy for when the merge editor resolves conflicts that git was unable to resolve automatically.

## Best solution

Implement a visual indicator or notification in the merge editor (likely in `src/vs/workbench/contrib/mergeEditor/`) that alerts users when a conflict resolution was performed by the editor's algorithm rather than git, so they can review and confirm the result. This is a UX design question that needs maintainer input on the right affordance (inline badge, notification, status bar item, etc.).
