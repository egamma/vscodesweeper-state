# Issue #306370 — Entire session lost when selecting workspace

[https://github.com/microsoft/vscode/issues/306370](https://github.com/microsoft/vscode/issues/306370)

## Summary

Verified duplicate of #319944 — both report the same flow: starting a Copilot Chat session in an empty (no-folder) window, then opening/adding a folder, which triggers an extension host restart that destroys the in-progress chat session.

## Change summary

Opening a folder from an empty-window workspace during a Copilot Chat session causes the extension host to restart, permanently losing the entire chat session.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/319944, which tracks the 0→1 folder transition restarting the extension host and killing in-progress extension state. Fixing that root cause (avoiding the extension host restart on first folder add) resolves both reports.
