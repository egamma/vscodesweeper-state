# Issue #148612 — Parenthesis and brace matching within RegExps

[https://github.com/microsoft/vscode/issues/148612](https://github.com/microsoft/vscode/issues/148612)

## Summary

Valid feature request for bracket matching inside regex literals. Not implemented on main — the bracket pairs engine has no regex-scope-aware matching. A contributor extension now exists (RedCMD's regex-syntax-highlighter), but core support remains unimplemented. Keeping open per backlog policy (9 upvotes, active discussion, team member reopened it).

## Change summary

When the cursor is on a parenthesis or bracket inside a regular expression, VS Code should highlight the matching counterpart, just as it does in normal code.

## Best solution

The bracket matching engine would need to consume TM grammar scope information to identify bracket pairs within regex-scoped tokens. The linked issue https://github.com/microsoft/vscode/issues/144736 (language-defined bracket pairs from TM scopes) is the likely prerequisite. Alternatively, this could remain extension territory — RedCMD's extension already provides this. A team member (hediet) reopened it as worth exploring, so it should stay in the backlog for future consideration.
