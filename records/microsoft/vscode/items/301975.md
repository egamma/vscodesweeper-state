# Issue #301975 — You will be asked to trust your folder, but you will also be asked to trust to repo root. Trust both.

[https://github.com/microsoft/vscode/issues/301975](https://github.com/microsoft/vscode/issues/301975)

## Summary

Valid polish request: the initial workspace trust dialog should be repo-aware so it can suggest trusting the git repo root directly, avoiding a second trust prompt from the git extension. A team member (aeschli) agreed this would be a good improvement. The request is still unaddressed on current main.

## Change summary

When opening a subfolder of a git repo, users face two separate trust prompts — the workspace trust dialog for the opened folder, then a notification toast from the git extension to trust the repo root — instead of one consolidated prompt.

## Best solution

Make the initial workspace trust dialog (in `src/vs/workbench/contrib/workspace/browser/workspace.contribution.ts`) aware of the git repo root when the opened folder is a subfolder of a repo, so it can offer to trust the repo root in a single step. This likely requires coordination between the workspace trust service and the git extension's repo-discovery logic (`extensions/git/src/model.ts` around line 674–679).
