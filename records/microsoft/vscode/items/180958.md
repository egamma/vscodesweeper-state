# Issue #180958 — Git - Use gitattributes encoding when opening a file

[https://github.com/microsoft/vscode/issues/180958](https://github.com/microsoft/vscode/issues/180958)

## Summary

Valid, community-promoted feature request (39 upvotes) to read encoding directives from .gitattributes and apply them when opening files. No implementation exists on current main — the git extension has no `check-attr` or gitattributes-encoding integration. Keeping open in the Backlog.

## Change summary

VS Code should read the `encoding` attribute from `.gitattributes` and automatically apply the correct file encoding when opening files.

## Best solution

Implement gitattributes encoding detection in the git extension (likely `extensions/git/src/git.ts` via `git check-attr encoding <path>`) and wire it into the file-open encoding resolution path in `src/vs/workbench/services/textfile/`. This bridges two subsystems — git attribute lookup and the editor's encoding resolver — and may benefit from an extension API surface for encoding providers.
