# Issue #113609 — Git - quick diff indicator does not work with global textconv

[https://github.com/microsoft/vscode/issues/113609](https://github.com/microsoft/vscode/issues/113609)

## Summary

Still reproduces on main: `Repository.buffer()` at git.ts:1573 passes `--textconv` when reading the quick-diff original, but the editor shows the raw working-tree content, creating phantom diffs when a global textconv is configured. A contributor has identified the root cause and offered a fix; the `help wanted` label is appropriate.

## Change summary

Quick diff shows phantom added lines on unmodified files when a global `textconv` filter is configured in `~/.gitconfig`, because the git: FS provider applies textconv to the original but the editor buffer is raw.

## Best solution

Skip `--textconv` when reading the quick-diff original in `Repository.buffer()` (extensions/git/src/git.ts:1573) so both sides compare raw content. The contributor @soreavis has offered to send a PR — accepting that would resolve this. Alternatively, a new parameter could let callers opt out of textconv selectively.
