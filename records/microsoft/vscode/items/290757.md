# Issue #290757 — Custom Agent: Model name should not require (Preview) syntax in the name

[https://github.com/microsoft/vscode/issues/290757](https://github.com/microsoft/vscode/issues/290757)

## Summary

Team-authored issue (MEMBER) about model name ergonomics in custom agent configuration — requiring '(Preview)' and showing '(copilot)' suffixes is confusing and fragile. Keeping open for maintainer judgment; active internal discussion is ongoing.

## Change summary

Custom agent model configuration requires exact model display names including transient suffixes like '(Preview)' and '(copilot)', making configurations fragile when models are renamed or promoted out of preview.

## Best solution

Use durable model IDs rather than display names for custom agent model configuration, and strip cosmetic suffixes like '(copilot)' from the suggest widget. The model picker/suggest should present friendly names but persist stable identifiers, so configurations survive model renames.
