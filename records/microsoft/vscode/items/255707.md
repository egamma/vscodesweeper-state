# Issue #255707 — Rendering faded out text very laggy

[https://github.com/microsoft/vscode/issues/255707](https://github.com/microsoft/vscode/issues/255707)

## Summary

Still reproducible on main: the unnecessary-code opacity is applied per-inline-span via CSS `opacity`, which forces expensive compositing when many tokens are faded. No fix has landed.

## Change summary

Scrolling becomes very laggy when many syntax tokens are rendered with `editorUnnecessaryCode.opacity` (faded unreachable code).

## Best solution

Optimize the rendering path for opacity decorations — consider batching faded spans under a single opacity container, using `will-change` hints, or applying opacity at the line level rather than per-token. The fix likely lives in the view-line rendering in `src/vs/editor/browser/view/` or the decoration application in `codeEditorWidget.ts`.
