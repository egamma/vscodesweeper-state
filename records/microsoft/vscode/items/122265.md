# Issue #122265 — link parsing should favor on screen urls over offscreen content for huge files

[https://github.com/microsoft/vscode/issues/122265](https://github.com/microsoft/vscode/issues/122265)

## Summary

Valid feature request for viewport-first link parsing in large files. No viewport prioritization has been implemented in the link detector — the code still computes all links for the full document. The issue sits in Backlog with 0 upvotes and no traction, but the gap remains real.

## Change summary

Request to prioritize link detection for the visible viewport area in large files rather than parsing the entire document before links become clickable.

## Best solution

Implement viewport-scoped link detection in `src/vs/editor/contrib/links/browser/links.ts` — compute links for a window around the visible range first, then lazily expand to the rest of the document. Similar to how tokenization already prioritizes the viewport.
