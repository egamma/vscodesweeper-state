# Issue #185563 — Async data tree doesn't maintain consistency on focus and selection events. 

[https://github.com/microsoft/vscode/issues/185563](https://github.com/microsoft/vscode/issues/185563)

## Summary

Team-authored issue (MEMBER). The consistency problem between focus/selection events in treeView.ts remains unresolved on current main — the same non-debounced pattern from the original report is still in place at lines 773–784.

## Change summary

WorkbenchAsyncDataTree fires focus and selection change events independently, so reading getFocus() during onDidChangeSelection (or getSelection() during onDidChangeFocus) may return stale values.

## Best solution

Implement the debounced event merging approach suggested in comment 4 (or a similar buffering strategy) in src/vs/workbench/browser/parts/views/treeView.ts so that the combined onDidChangeSelectionAndFocus event fires only after both focus and selection are settled. This would also unblock finalization of the TreeView API (issue #170248).
