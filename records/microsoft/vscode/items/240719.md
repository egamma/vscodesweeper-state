# Issue #240719 — Git - stage selected ranges in an untracked file

[https://github.com/microsoft/vscode/issues/240719](https://github.com/microsoft/vscode/issues/240719)

## Summary

Valid feature request with 'On Deck' milestone and maintainer acknowledgment — must stay open. The request to stage selected ranges in untracked files is a real gap since the 1.97 change to show untracked files in a single editor (no diff view), which removes the diff information needed by `stageSelectedRanges`.

## Change summary

Allow 'Stage Selected Ranges' to work on untracked (new) files, which currently open in a single editor without diff information after the 1.97 change.

## Best solution

Enable diff-based staging for untracked files by treating them as a diff against an empty document (all lines are additions). The implementation in `extensions/git/src/staging.ts` (`getWorkingTreeDiffInformation`) and the diff editor setup for untracked files need to synthesize diff information so `stageSelectedRanges` in `extensions/git/src/commands.ts` can operate. The maintainer (lszomoru) acknowledged the merit and placed it On Deck.
