# Issue #178138 — Not clear how to share attribute definitions between "attach" and "launch" configurations

[https://github.com/microsoft/vscode/issues/178138](https://github.com/microsoft/vscode/issues/178138)

## Summary

Still valid: no mechanism exists for extension authors to share property definitions between 'launch' and 'attach' in `configurationAttributes`. A team member acknowledged the gap and expressed interest in supporting it, but no implementation has landed.

## Change summary

Allow debug extension authors to share custom property definitions between 'launch' and 'attach' configurations in `configurationAttributes` without copy-pasting.

## Best solution

Support `$defs`/`$ref` (or a shared key like `"common"`) in extension-contributed `configurationAttributes` so authors can define properties once and reference them from both 'launch' and 'attach'. The likely implementation site is `getSchemaAttributes()` in `src/vs/workbench/contrib/debug/common/debugger.ts`, which already uses `$ref` for internal common properties — extending this to honor extension-supplied definitions would be a natural fit.
