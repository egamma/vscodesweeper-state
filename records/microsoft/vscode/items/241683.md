# Issue #241683 — Failure to use code.cmd --install-extension <valid extension id> behind authenticating proxy server

[https://github.com/microsoft/vscode/issues/241683](https://github.com/microsoft/vscode/issues/241683)

## Summary

Still reproduces on main: the CLI `--install-extension` path uses the Node.js `RequestService` which does not perform Kerberos/NEGOTIATE proxy authentication negotiation on 407 responses, unlike the Electron workbench path which handles this via `@vscode/proxy-agent` and the `proxyResolver.ts` callback. The gap is architectural and has not been addressed.

## Change summary

CLI `--install-extension` fails behind authenticating proxy servers (Kerberos/NEGOTIATE) because the Node.js request path lacks the 407-retry-with-Kerberos flow that the workbench path has.

## Best solution

Wire up a 407 retry loop in `RequestService.request()` (`src/vs/platform/request/node/requestService.ts`) that calls `lookupKerberosAuthorization()` when the proxy responds with 407 + `Negotiate`/`Kerberos` challenge, mirroring the logic in `src/vs/workbench/api/node/proxyResolver.ts` lines 463–480. The method already exists on the class; it just needs to be called in the request flow.
