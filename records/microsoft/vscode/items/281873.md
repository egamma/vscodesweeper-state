# Issue #281873 — Build failed: All tests fail during "Build client" step due to ENOENT for preload.js in electron-browser sandbox

[https://github.com/microsoft/vscode/issues/281873](https://github.com/microsoft/vscode/issues/281873)

## Summary

Three hard keep-open rules apply (MEMBER author, 'important' label, 'On Deck' milestone). The OOM sub-problem was addressed by PR #282079, but the silent-failure sub-problem — compilation crashes without propagating the error to the build step — remains unfixed on current main.

## Change summary

Build compilation can fail silently (e.g. from OOM), leaving the step green and causing downstream tasks to fail with misleading ENOENT errors for missing output files.

## Best solution

Ensure the gulp compilation task propagates fatal child-process errors (exit codes, signals) so the build step fails immediately and visibly. The likely fix area is `build/lib/task.ts` or the compilation gulp task definition, where process exit status needs to be checked and surfaced. PR https://github.com/microsoft/vscode/pull/282079 addressed the OOM trigger but not the silent-failure behavior.
