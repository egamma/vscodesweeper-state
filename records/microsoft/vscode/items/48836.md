Team-authored debt item (dbaeumer/MEMBER) requesting that the task API surface customized tasks properly — the original provided task should contain a pointer to its tasks.json customization, not the inverse. Hard keep-open applies; the issue remains a valid API-design debt item.

## Change summary

The tasks API should expose the original provided task with a pointer to its tasks.json customization, rather than the current inverted representation.

## Best solution

Refactor the task API so that `fetchTasks` returns the original provider-contributed task object with an attached reference to its tasks.json customization, aligning the public API shape with the intended design described in https://github.com/microsoft/vscode/issues/48831.
