# Issue #157096 — problemMatcher with multiple patterns doesn't work

[https://github.com/microsoft/vscode/issues/157096](https://github.com/microsoft/vscode/issues/157096)

## Summary

Confirmed bug still present on main: when multiple problemMatchers match the same output line, only the first match wins because `tryMatchers()` returns after the first successful match and clears the buffer — subsequent matchers never see the line. Already confirmed, labeled, and assigned; keeping open.

## Change summary

Multiple problemMatchers defined on the same task cannot each independently match the same output line; only the first matching one produces a diagnostic.

## Best solution

Fix `tryMatchers()` in `src/vs/workbench/contrib/tasks/common/problemCollectors.ts` to continue trying all candidate matchers against the buffer (collecting all matches) rather than returning after the first hit. The buffer-clear and `activeMatcher` assignment need to account for multiple simultaneous matches from independent matchers on the same line.
