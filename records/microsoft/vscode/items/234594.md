# Issue #234594 — Indicate that version property should not be used for reporting telemetry

[https://github.com/microsoft/vscode/issues/234594](https://github.com/microsoft/vscode/issues/234594)

## Summary

Team-authored debt issue requesting that the telemetry infrastructure warn or document that certain property names (like `version`) are reserved by the product's common properties. Still unfixed on main — no guard or warning exists in the telemetry layer. Keeping open per team-member policy.

## Change summary

Extensions can unknowingly use telemetry property names (e.g. `version`) that are silently overwritten by VS Code's common properties, with no warning or documentation.

## Best solution

Add a runtime warning or validation in `src/vs/platform/telemetry/common/telemetryUtils.ts` (or where extension telemetry events are merged with common properties) that flags when an extension-supplied property collides with a reserved common-property name. The reserved names are set in `src/vs/platform/telemetry/common/commonProperties.ts`. This is tracked alongside the internal backlog item referenced in the comments.
