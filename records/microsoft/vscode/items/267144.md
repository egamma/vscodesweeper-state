# Issue #267144 — CTRL+G regression is back for Screen Reader Users in Insiders build

[https://github.com/microsoft/vscode/issues/267144](https://github.com/microsoft/vscode/issues/267144)

## Summary

Still reproduces on main: the onDidChangeCursorPosition listener at line 100 of editorNavigationQuickAccess.ts updates the restore point on every cursor change (including preview navigations from the Go-to-Line dialog), so Escape restores to the wrong position. The related PR #271610 was closed without merging.

## Change summary

Pressing Ctrl+G then Escape with screen reader mode and EditContext enabled causes the cursor to jump to line 1 instead of staying at the original position.

## Best solution

Filter cursor-change events by source in the onDidChangeCursorPosition listener (line 100 of src/vs/editor/contrib/quickAccess/browser/editorNavigationQuickAccess.ts) to ignore preview/JUMP navigations, so lastKnownEditorViewState only updates for user-initiated cursor moves.
