# Issue #309821 — The View Tree Button in Source Control is missing

[https://github.com/microsoft/vscode/issues/309821](https://github.com/microsoft/vscode/issues/309821)

## Summary

The tree/list toggle buttons are intentionally hidden by default in the SCM toolbar (isHiddenByDefault: true) to reduce toolbar clutter. The user's core need — remembering tree view preference — is already served by the `scm.defaultViewMode` setting and workspace-scoped storage that persists the last choice.

## Change summary

The tree/list view toggle button is no longer visible by default in the Source Control toolbar.

## Best solution

This is as-designed: the buttons were hidden to declutter the toolbar. Users can right-click the SCM toolbar to re-show them, or set `scm.defaultViewMode` to `tree` in settings to always default to tree view. The preference is also remembered per-workspace via storage.
