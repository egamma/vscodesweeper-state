# Issue #184192 — order of search results swaps on notebook open

[https://github.com/microsoft/vscode/issues/184192](https://github.com/microsoft/vscode/issues/184192)

## Summary

Still open on main: the experimental setting `search.experimental.closedNotebookRichContentResults` that addresses this issue still defaults to `false`. The author (a team member) intends to close this when the setting is enabled by default, which has not yet happened.

## Change summary

Search results order swaps when opening a closed notebook because outputs appear before inputs in raw text.

## Best solution

Graduate `search.experimental.closedNotebookRichContentResults` from experimental to default-on, which would resolve the result-reordering problem for closed notebooks. Once enabled by default, this issue can be closed.
