# Issue #124165 — Workspace Trust - restricted mode banner accessibility improvements

[https://github.com/microsoft/vscode/issues/124165](https://github.com/microsoft/vscode/issues/124165)

## Summary

Team-authored accessibility bug with 2 of 4 checklist items still unresolved on current main. The banner has no Shift+Tab handler to return focus to the banner container, and pressing Escape while the banner container itself is focused does not close it. Keeping open per team-author policy and because remaining work is real.

## Change summary

The restricted mode banner has incomplete keyboard navigation: Shift+Tab from action links and Escape-to-close from the banner container don't work.

## Best solution

In `src/vs/workbench/browser/parts/banner/bannerPart.ts`, add a Shift+Tab keybinding (when `BannerFocused`) that calls `bannerService.focus()` to return to the banner container, and modify the Escape handler to distinguish between 'focused on an action link' (reset to banner) vs 'already focused on the banner container' (close the banner). The two unchecked items are small, localized changes in the banner keybinding registrations.
