# Issue #289001 — Copilot Hangs & Crashes With Obscure, Incorrect Error Message When Review Tool Is Used on Large File

[https://github.com/microsoft/vscode/issues/289001](https://github.com/microsoft/vscode/issues/289001)

## Summary

Legitimate report about poor error messaging in Copilot's inline Review feature when it fails on large selections, but reproduction details are too thin to act on — unclear file size, language, or whether this is a transient service-side timeout vs. a consistent failure.

## Change summary

Copilot inline Review hangs and then shows a raw `net::ERR_CONNECTION_CLOSED` error when used on large code selections, instead of a user-friendly message about size limitations or a retry suggestion.

## Best solution

The review feature in `extensions/copilot/src/extension/review/node/doReview.ts` and the networking layer should detect connection-closed errors during review and surface a friendlier message (e.g. 'Selection may be too large to review' or a retry prompt). Needs a reliable reproduction to determine the threshold and whether this is a server-side timeout or a client-side issue.
