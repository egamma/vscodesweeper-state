# Issue #307773 — `workbench.action.tasks.runTask` or `Terminal` > `Run Task...` > `Configure Task` will force `tasks.json` file to be formatted in `JSON` format will cause all comments in the `tasks: []` to disappear

[https://github.com/microsoft/vscode/issues/307773](https://github.com/microsoft/vscode/issues/307773)

## Summary

Verified duplicate of #29453 (closed *out-of-scope): the configuration service's `updateValue` replaces the entire `tasks` array value with parsed JSON that has already lost embedded comments — this is a known, intentional limitation.

## Change summary

Using 'Configure Task' writes the tasks array through `configurationService.updateValue`, which replaces the JSONC value with a comment-free parsed version, stripping inline comments.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/29453 which already tracked and closed this as out-of-scope. The limitation is architectural: the configuration writer replaces entire property values and cannot preserve comments embedded within.
