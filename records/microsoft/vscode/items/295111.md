# Issue #295111 — Output limit should be client configurable

[https://github.com/microsoft/vscode/issues/295111](https://github.com/microsoft/vscode/issues/295111)

## Summary

Valid feature request to make tool output limits client-configurable (default 32k) rather than hardcoded, to avoid sending excessive output to models like codex with large context windows. No evidence this has been implemented on main; routing to the agent-host area.

## Change summary

Make the tool output character/token limit a client-configurable setting with a 32k default, instead of allowing models to consume up to their full context window (e.g. 128K for codex).

## Best solution

Add a client-side configuration (likely in agent-host harness or chat session config) that caps tool output before it is sent to the model, defaulting to 32k. This keeps context budgets sane across models while letting power users raise the limit when needed.
