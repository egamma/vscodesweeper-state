# Issue #162283 — Search decoration "badge" term ambiguous 

[https://github.com/microsoft/vscode/issues/162283](https://github.com/microsoft/vscode/issues/162283)

## Summary

The setting description for `search.decorations.badges` is still vague on current main — it says "Controls whether search file decorations should use badges" without clarifying what "badges" means (the letter decorations like A/M/D, vs. the count badge most users associate with the word). This is a valid polish item that remains unaddressed.

## Change summary

The `search.decorations.badges` setting description is ambiguous — "badge" could mean the count number or the letter decorations (A/M/D), causing user confusion.

## Best solution

Clarify the description of `search.decorations.badges` (and potentially `explorer.decorations.badges`) to specify that "badges" refers to the letter-based file status decorations (e.g. M, A, D), not the match-count number. The fix is in `src/vs/workbench/contrib/search/browser/search.contribution.ts` line 294.
