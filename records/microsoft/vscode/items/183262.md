# Issue #183262 — Multiline problem-matcher with empty line breaks when re-run in same terminal

[https://github.com/microsoft/vscode/issues/183262](https://github.com/microsoft/vscode/issues/183262)

## Summary

Still appears unfixed on main. The multiline problem matcher buffer logic in `AbstractProblemCollector.tryFindMarker` does not account for spurious empty lines emitted by a reused terminal between task runs, breaking pattern matching on subsequent executions.

## Change summary

Multiline problem matchers with an empty-line pattern break when a task is re-run in the same terminal, because terminal separator/prompt output between runs confuses the matcher's buffer state.

## Best solution

When a StartStopProblemCollector is created for a re-run in a reused terminal, ensure the line data feed begins only after the new command's output starts (e.g., using shell integration command markers), or make the multiline matcher resilient to spurious empty lines injected between runs. The fix likely involves `src/vs/workbench/contrib/tasks/browser/terminalTaskSystem.ts` (the `onLineData` registration around line 1086) and possibly `src/vs/workbench/contrib/tasks/common/problemCollectors.ts` (the buffer/activeMatcher reset logic).
