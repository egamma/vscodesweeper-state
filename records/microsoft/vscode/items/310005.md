  > Thanks for reaching out. What you're seeing isn't a bug — when Git tracks a folder, it includes all files that aren't excluded by a `.gitignore` file. To fix this:
  > 
  > 1. Create a `.gitignore` file in your project root.
  > 2. Add entries for folders you don't want committed (e.g. `node_modules/`, `.vscode/extensions/`, etc.).
  > 3. If files were already tracked, run `git rm -r --cached .` then recommit.
  > 
  > You can find `.gitignore` templates at https://github.com/github/gitignore. For more help with Git workflows, https://stackoverflow.com or the [GitHub Community](https://github.com/orgs/community/discussions) are great resources.
