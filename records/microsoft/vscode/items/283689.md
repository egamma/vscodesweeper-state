# Issue #283689 — Problems -> View as Table shows `-` for Code when it exists in Tree view

[https://github.com/microsoft/vscode/issues/283689](https://github.com/microsoft/vscode/issues/283689)

## Summary

Still reproduces on main: the table view's Code column renderer requires both `source` AND `code` to be present (line 151, `&&`), while the tree view only requires either (`||` on line 389). Markers with a code but no source show `-` in table view.

## Change summary

Problems panel table view shows `-` in the Code column for markers that have a code but lack a source, even though the tree view shows the code correctly.

## Best solution

Change the condition in `markersTable.ts` line 151 from `element.marker.source && element.marker.code` to `element.marker.source || element.marker.code`, mirroring the tree view logic, and handle the case where source or code is independently missing (show code without source prefix, or source without code).
