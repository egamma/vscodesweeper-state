# Issue #50952 — onEnterRules should be multi-line aware

[https://github.com/microsoft/vscode/issues/50952](https://github.com/microsoft/vscode/issues/50952)

## Summary

Still a valid, unimplemented feature request. The onEnter infrastructure only exposes `previousLineText` (one line of context) to rules; multi-line awareness has not been added to current main.

## Change summary

onEnterRules should be able to consider multiple preceding lines of context to avoid false-positive matches (e.g., comment continuation rules firing outside multiline comments).

## Best solution

Extend the `OnEnterRule` interface in `src/vs/editor/common/languages/languageConfiguration.ts` and the evaluation logic in `src/vs/editor/common/languages/supports/onEnter.ts` to pass additional preceding-line context (or a scope/token-based signal) so rules can distinguish multiline comments from similar-looking non-comment lines. This is a core editor API change that would also need to surface through the extension API (`vscode.d.ts`).
