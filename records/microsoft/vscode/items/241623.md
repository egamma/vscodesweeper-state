# Issue #241623 — SCM - Terminal on sidebar loses focus when switching SCM working sets

[https://github.com/microsoft/vscode/issues/241623](https://github.com/microsoft/vscode/issues/241623)

## Summary

Still reproduces on main: the focus-preservation logic in workingSet.ts only checks `Parts.PANEL_PART`, so terminals moved to the sidebar or auxiliary bar lose focus when switching branches. The code comment explicitly acknowledges this gap.

## Change summary

Switching Git branches from a terminal in the sidebar causes the terminal to lose focus because the SCM working-set restore only preserves focus for the panel part.

## Best solution

Extend the `preserveFocus` check in `src/vs/workbench/contrib/scm/browser/workingSet.ts` (line 157) to also cover `Parts.SIDEBAR_PART` and `Parts.AUXILIARYBAR_PART` — or, more robustly, preserve focus whenever the editor part does not currently hold it (i.e. check `!this.layoutService.hasFocus(Parts.EDITOR_PART)`).
