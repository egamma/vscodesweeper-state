## Summary

Valid feature request acknowledged by a team member as desirable but blocked on tree widget support for pinning nodes. No progress on main yet; keeping open in Backlog.

## Change summary

Keep the SCM commit message input and commit button visible (sticky/pinned) outside the scrollable file list so users don't have to scroll back up to commit.

## Best solution

Implement a 'pinned nodes' capability in the base tree widget (src/vs/base/browser/ui/tree/), then adopt it in the SCM view to keep the input widget and action button fixed outside the scroll region. Alternatively, restructure the SCM view layout to separate the input/button from the tree's scroll container entirely.
