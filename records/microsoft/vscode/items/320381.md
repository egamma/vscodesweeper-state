## Summary

Team-authored accessibility bug about VoiceOver (Ctrl+Option+Arrow) navigation getting stuck on the 'Views and More Actions' toolbar button in the sidebar/panel header. Standard Tab navigation works; the issue is specific to VoiceOver's AX tree traversal. Keeping open per team-author policy.

## Change summary

VoiceOver navigation (Ctrl+Option+Arrow) gets stuck on the 'Views and More Actions' control in the Explorer/sidebar header, requiring excessive keystrokes or blocking forward navigation entirely.

## Best solution

Investigate the accessibility tree structure in the composite part header (`src/vs/workbench/browser/parts/compositePart.ts`) — the toolbar and action items may need ARIA grouping or role adjustments so VoiceOver can traverse the header controls linearly without getting trapped. A team member suggested downgrading to sev3; the discussion is still active between the reporter and the accessibility area owner.
