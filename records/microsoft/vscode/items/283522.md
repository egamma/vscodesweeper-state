# Issue #283522 — KDE's global menu won't work with latest 1.107.0 build

[https://github.com/microsoft/vscode/issues/283522](https://github.com/microsoft/vscode/issues/283522)

## Summary

Already correctly triaged as an upstream issue. KDE's global menu does not support non-Qt (GTK/Electron) applications under Wayland — this is a known KDE/Wayland-protocols limitation (KDE bug 424485), not a VS Code defect. The issue is well-labeled with `upstream`, `upstream-issue-linked`, and `electron-39-update`; the upstream Electron issue (electron/electron#34335) is open. No action needed from VS Code beyond tracking.

## Change summary

KDE's global menu integration broke for VS Code on Wayland starting with the Electron 39 update in 1.107.0.

## Best solution

Track upstream resolution in Electron (https://github.com/electron/electron/issues/34335) and the Wayland protocols MR (https://gitlab.freedesktop.org/wayland/wayland-protocols/-/merge_requests/52). VS Code currently ships Electron 42; when Electron gains Wayland-native global menu support (likely via a new Wayland protocol), VS Code will inherit the fix. Users can work around it today with `--ozone-platform=x11` or `window.titleBarStyle: native`.
