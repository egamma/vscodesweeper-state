# Issue #277754 — Default worktree location has no write permission in dev container

[https://github.com/microsoft/vscode/issues/277754](https://github.com/microsoft/vscode/issues/277754)

## Summary

Still reproduces on main: `getCopilotWorktreesRoot` computes the worktree directory as `<repoRoot>/../<repoName>.worktrees`, which is outside the mounted volume in dev containers and therefore not writable. The design discussion between team members is unresolved — no fix has landed.

## Change summary

In dev containers, the default worktree location (`/workspaces/<repo>.worktree`) falls outside the mounted volume and lacks write permission, causing worktree-isolated sessions to fail.

## Best solution

Resolve the design question raised by @chrmarti: either detect a containerized environment and place worktrees inside the repo (e.g. a `.worktrees` subfolder), mount the parent, or make the worktree root configurable. The fix lives in `getCopilotWorktreesRoot` in `src/vs/platform/agentHost/node/copilot/copilotAgent.ts` and likely needs coordination with the dev-containers extension for mount awareness.
