# Issue #235626 — VSCode does not open file paths while in a workspace

[https://github.com/microsoft/vscode/issues/235626](https://github.com/microsoft/vscode/issues/235626)

## Summary

Still reproduces on main: Quick Open's `getRelativePathFileResults` resolves the query relative to each workspace folder root without stripping a leading folder-name prefix, so typing `code/main.py` when `code/` is a workspace folder resolves to `code/code/main.py` (not found). No fix has landed.

## Change summary

Quick Open (Ctrl+P) fails to find files when the query includes the workspace folder's directory name as a prefix (e.g. `code/main.py` inside the `code` folder).

## Best solution

In `getRelativePathFileResults` (src/vs/workbench/contrib/search/browser/anythingQuickAccess.ts ~L779), after iterating workspace folders, also try stripping the folder's basename (and its custom display name in multi-root workspaces) from the query before resolving. TylerLeonhardt's comment outlines this approach.
