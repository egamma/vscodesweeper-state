# Issue #245055 — "CTRL + DEL" keyboard shortcut not working as expected and deleting comments

[https://github.com/microsoft/vscode/issues/245055](https://github.com/microsoft/vscode/issues/245055)

## Summary

Still reproduces on main: the whitespace heuristic in deleteWordRight requires >1 whitespace char, so with exactly 1 space before a word boundary it falls through to word-navigation and deletes the next token (e.g. `//`). An open PR (#324211) targets this but hasn't merged.

## Change summary

Ctrl+Del (deleteWordRight) with a single trailing space before a comment deletes the comment token instead of just the whitespace.

## Best solution

Fix the off-by-one in `_deleteWordRightWhitespace` (line 631 of `src/vs/editor/common/cursor/cursorWordOperations.ts`): change `startIndex + 1 < firstNonWhitespace` to `startIndex < firstNonWhitespace` so that a single whitespace character is also handled by the whitespace-only path. The open PR #324211 targets this.
