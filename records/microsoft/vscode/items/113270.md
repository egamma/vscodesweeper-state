# Issue #113270 — Support cancelation in getSession

[https://github.com/microsoft/vscode/issues/113270](https://github.com/microsoft/vscode/issues/113270)

## Summary

Team-authored feature request (hard keep-open). The `getSession` API still lacks a CancellationToken parameter on current main. The related issue #211406 fixed the 'stuck' symptom with a timeout in Microsoft auth flows, but the general-purpose cancellation token proposed here was never implemented.

## Change summary

Allow extensions to pass a CancellationToken to `authentication.getSession` so they can cancel in-progress authentication requests.

## Best solution

Add an optional `CancellationToken` parameter to the `authentication.getSession` API signature and propagate it through to auth providers, so extensions can cancel stuck or abandoned session requests. The timeout fix in https://github.com/microsoft/vscode/issues/211406 mitigates the worst symptom but doesn't give extensions programmatic cancellation control.
