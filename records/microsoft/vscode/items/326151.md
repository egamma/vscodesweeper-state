# Issue #326151 — Support writing to the local clipboard from Dev Containers and Remote SSH sessions

[https://github.com/microsoft/vscode/issues/326151](https://github.com/microsoft/vscode/issues/326151)

## Summary

A multi-faceted feature request for clipboard bridging from remote environments to the local host. Parts of this overlap with the open remote-release issue on OSC 52 forwarding, but the core ask — a `code --clipboard-write` CLI command or IPC bridge usable from Git hooks and non-interactive processes — is a distinct feature request that belongs in the terminal/remote area of VS Code core.

## Change summary

Allow scripts and Git hooks running inside Dev Containers or Remote SSH sessions to write to the local host clipboard, via a CLI command, reliable OSC 52 forwarding, or an IPC bridge.

## Best solution

The OSC 52 gap in Remote SSH terminals is already tracked at https://github.com/microsoft/vscode-remote-release/issues/11475. The broader ask here — a `code --clipboard-write` CLI command usable from non-interactive processes like Git hooks — is a genuinely new feature request that would require plumbing the clipboard service through the remote CLI tunnel. Route to the terminal and remote areas for evaluation; the CLI clipboard bridge is the most actionable proposal and would benefit any remote workflow.

- **No CLI clipboard support:** No `--clipboard-write` argument exists in VS Code's CLI (searched src/vs/code/node/cli* and src/vs/platform/environment/ — no hits).
- **Git hooks are non-terminal:** When a pre-commit hook runs via VS Code's SCM extension (not typed into the integrated terminal), its output does not flow through xterm.js, so OSC 52 sequences have no handler — this is a distinct gap from the terminal-based OSC 52 support.
