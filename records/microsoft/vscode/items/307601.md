# Issue #307601 — Daily terminal relaunch warning for Git auth provider on Windows

[https://github.com/microsoft/vscode/issues/307601](https://github.com/microsoft/vscode/issues/307601)

## Summary

Still reproduces on main: revived persistent terminals compare their serialized environment collection snapshot against the freshly-registered git auth variables on startup, producing a stale diff and the relaunch warning every time VS Code restarts.

## Change summary

Persistent terminals show a repeated relaunch-needed warning for the Git auth environment contribution on every VS Code startup.

## Best solution

When a terminal is revived from a persistent session, the environment variable collection comparison should account for the fact that extensions re-register the same variables on activation. Either suppress the stale diff when the effective env vars haven't meaningfully changed for a revived terminal, or update the stored collection baseline to the current one upon successful revival/attach.
