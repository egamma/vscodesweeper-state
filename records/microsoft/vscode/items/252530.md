# Issue #252530 — Make `security.promptForPromptProtocolHandling` a real setting and an allow list

[https://github.com/microsoft/vscode/issues/252530](https://github.com/microsoft/vscode/issues/252530)

## Summary

This feature request from a core team member asks for a new setting that does not yet exist on main. The setting `security.promptForPromptProtocolHandling` is absent from the codebase; only `security.promptForLocalFileProtocolHandling` and `security.promptForRemoteFileProtocolHandling` exist. No implementation work has landed.

## Change summary

Expose a `security.promptForPromptProtocolHandling` setting as a host allow-list (string[]) so users can review and edit prompt-protocol decisions after the fact.

## Best solution

Implement the setting as a `string[]` allow-list of hosts permitted without prompt, analogous to the existing `security.promptForLocalFileProtocolHandling` boolean but richer. This likely belongs near the existing security settings in `src/vs/workbench/electron-browser/desktop.contribution.ts` and the protocol-handling logic in `src/vs/code/electron-main/app.ts`.
