# Issue #185112 — Column Select / Multiple Cursors: Ability to not have cursor inserted in wrapped part of wrapped text, without need to disable line wrap

[https://github.com/microsoft/vscode/issues/185112](https://github.com/microsoft/vscode/issues/185112)

## Summary

Keep open: the older keyboard multi-cursor word-wrap fix does not appear to cover mouse column selection, and current main still routes mouse column selection through view-line coordinates.

## Change summary

Mouse column selection and multi-cursor placement should avoid adding cursors on wrapped continuations unless the user explicitly wants visual-line behavior.

## Best solution

Add a focused editor-multicursor fix for mouse-based column selection, likely around `src/vs/editor/browser/view/viewController.ts`, `src/vs/editor/browser/coreCommands.ts`, and `src/vs/editor/common/cursor/cursorColumnSelection.ts`, with a word-wrap regression test for mouse column selection.
