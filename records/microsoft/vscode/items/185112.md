# Issue #185112 — Column Select / Multiple Cursors: Ability to not have cursor inserted in wrapped part of wrapped text, without need to disable line wrap

[https://github.com/microsoft/vscode/issues/185112](https://github.com/microsoft/vscode/issues/185112)

## Summary

Still reproduces on main: the keyboard-driven 'Add Cursor Above/Below' was fixed to use logical lines (commit c0d32634, PR #135805), but mouse-driven column/box selection and Alt+Click multi-cursor still operate on view lines, inserting cursors on wrapped portions. No fix has been applied to `cursorColumnSelection.ts` or the mouse handler since the issue was filed.

## Change summary

Mouse-driven column selection and multi-cursor modifier click insert cursors on wrapped (visual) lines instead of only on logical lines when word wrap is enabled.

## Best solution

Extend the `logicalLine` treatment from the keyboard path (InsertCursorAbove/Below) to the mouse-driven column select and Alt+Click paths. The fix likely involves `src/vs/editor/common/cursor/cursorColumnSelection.ts` and the mouse handler in `src/vs/editor/browser/controller/mouseHandler.ts` — both need to collapse wrapped view lines to their parent model line when word wrap is active. Issue #198399 is the same underlying bug reported from the Alt+Shift+Click angle and could be consolidated here.
