# Issue #326391 — Confusing create file approval widget

## Summary

Team-authored issue (MEMBER) — keeping open. The file-creation approval widget uses past-tense wording ('Created file') while still waiting for user confirmation, which is confusing. A commenter notes edits have a similar issue and suggests 'will create' / 'will change' phrasing.

## Change summary

The tool confirmation widget for file creation shows past-tense 'Created file' in the title while simultaneously asking for approval to create it.

## Best solution

Update the confirmation title strings for create/edit tool invocations to use future-tense or intent phrasing ('Will create…' / 'Will change…'). The title flows from `state.confirmationMessages.title` into `ChatModifiedFilesConfirmationSubPart.getTitle()` (src/vs/workbench/contrib/chat/browser/widget/chatContentParts/toolInvocationParts/chatModifiedFilesConfirmationSubPart.ts); the fix likely lives in whichever code constructs the `confirmationMessages` for the agent-host tool invocations. This overlaps with the broader modernization work justschen references.
