# Issue #278104 — TypeError: items is not iterable

[https://github.com/microsoft/vscode/issues/278104](https://github.com/microsoft/vscode/issues/278104)

## Summary

Confirmed on current main: the ordering bug in `FromEventObservable.handleEvent` remains unfixed. The issue carries milestone 'On Deck' (a scheduling decision), so it stays open.

## Change summary

Observable `handleEvent` sets `_hasValue = true` after notifying observers, creating an inconsistent state if an observer unsubscribes during notification, leading to a TypeError crash.

## Best solution

Move `this._hasValue = true` before the observer notification loop (before line 108) in `src/vs/base/common/observableInternal/observables/observableFromEvent.ts`, so that if `onLastObserverRemoved` resets the state during notification, the subsequent `_hasValue = true` on line 126 doesn't leave an inconsistent cache. Alternatively, guard line 126 with a check that `this._subscription` still exists.
