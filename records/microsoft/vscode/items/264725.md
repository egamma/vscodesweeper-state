# Issue #264725 — Symbols/files not linkified in thinking markdown

[https://github.com/microsoft/vscode/issues/264725](https://github.com/microsoft/vscode/issues/264725)

## Summary

Team-authored issue, must keep open. The bug is still present on main: the thinking content part's body `renderMarkdown` method does not call `renderFileWidgets` after rendering, so symbols/files in backticks are not linkified in the thinking block body — only in the title detail.

## Change summary

Symbols and file references in backticks are not rendered as clickable links inside thinking/reasoning markdown blocks, unlike in normal chat output.

## Best solution

Add a `renderFileWidgets` call (with proper disposable management) after the markdown render in `ChatThinkingContentPart.renderMarkdown()` (around line 834), mirroring what is already done for the title detail at line 2291. The chatMarkdownAnchorService and instantiationService are already injected.
