# Issue #167757 — Run code on wayland mode result on broken ibus connection

[https://github.com/microsoft/vscode/issues/167757](https://github.com/microsoft/vscode/issues/167757)

## Summary

This is a confirmed upstream issue in Chromium/Electron: native Wayland IME support depends on the `text-input-v1`/`v3` protocol implementation in both the compositor and Electron. The issue is correctly labeled `upstream` and remains relevant — the upstream Chromium bug was closed as wontfix for GTK integration, but a Wayland-native solution is being pursued. Keeping open as the canonical tracker on the VS Code side.

## Change summary

IME input (ibus, fcitx5) does not work when VS Code runs in native Wayland mode, falling back to English-only input.

## Best solution

This depends on Chromium landing proper Wayland text-input protocol support, then Electron picking it up. VS Code should keep tracking the upstream progress and update Electron when a fix ships. The related open issue https://github.com/microsoft/vscode/issues/237061 tracks making `--enable-wayland-ime` configurable via `argv.json`. No actionable VS Code-side code change can fix the core problem.

## Evidence

- **upstream root cause:** The Chromium bug https://issues.chromium.org/issues/40801194 (GTK4 IME for Wayland) was closed as wontfix; Chromium is pursuing a Wayland-native solution instead (comment #24 links the design doc).
- **Electron limitation:** Electron does not support --gtk-version=4 (comment #4 by deepak1556, a VS Code team member), and only partially supports --enable-wayland-ime via text-input-v1, which not all compositors implement.
- **compositor dependency:** Per https://fcitx-im.org/wiki/Using_Fcitx_5_on_Wayland, --enable-wayland-ime only works with compositors supporting text-input-v1 (KDE Plasma 5.27+); GNOME does not support v1, only v3.
- **still active:** Latest comment (June 2025) confirms the double-character terminal issue with fcitx5 on Wayland persists. Multiple related open issues exist: https://github.com/microsoft/vscode/issues/277073, https://github.com/microsoft/vscode/issues/315649.
