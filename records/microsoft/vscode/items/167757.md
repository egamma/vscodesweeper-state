# Issue #167757 — Run code on wayland mode result on broken ibus connection

## Summary

This is a known upstream Electron/Chromium limitation — Wayland IME support depends on text-input protocol versions that vary by compositor, and Chromium's GTK4 integration path was closed as wontfix in favor of a native Wayland solution still in progress. Already correctly labeled as upstream.

## Change summary

IME input (ibus, fcitx5) does not work when VS Code runs in native Wayland mode due to missing text-input protocol support in Electron/Chromium.

## Best solution

Keep tracking upstream progress on Chromium's native Wayland IME solution (https://issues.chromium.org/issues/40801194 and the related design doc). Once Electron ships text-input-v3 support, VS Code can adopt it. No actionable VS Code-side change exists today.
