# Issue #130217 — Consolidate zindex for floating elements in the workbench

[https://github.com/microsoft/vscode/issues/130217](https://github.com/microsoft/vscode/issues/130217)

## Summary

The central z-index registry infrastructure was created (src/vs/platform/layout/browser/zIndexRegistry.ts) with a ZIndex enum and registerZIndex() helper, but adoption remains minimal — only notebook uses it. The broader consolidation of hardcoded z-index values across ~100+ CSS files is still incomplete, so this debt item remains valid.

## Change summary

Consolidate scattered hardcoded z-index values across the workbench into a central registry to prevent floating-element overlap bugs.

## Best solution

Continue migrating hardcoded z-index values in CSS files to use the existing registerZIndex() system and CSS custom properties (var(--z-index-*)). The registry in src/vs/platform/layout/browser/zIndexRegistry.ts is the right foundation; the remaining work is adopting it across the ~100+ CSS files that still use raw numeric z-index values.
