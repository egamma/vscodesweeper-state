# Issue #301159 — Accepting actions with `Ctrl + Enter` only works for the first action in a row

[https://github.com/microsoft/vscode/issues/301159](https://github.com/microsoft/vscode/issues/301159)

## Summary

Still reproduces on main: the `hasToolConfirmation` context key is only set in scoped context key services of individual confirmation sub-parts, so after the first Ctrl+Enter confirms a tool and returns focus to the chat input, the key is invisible from the input's context key chain — the keybinding's `when` clause never fires again. No fix or duplicate found.

## Change summary

Pressing Ctrl+Enter to accept tool confirmations only works for the first confirmation; subsequent presses do nothing because the keybinding's context key becomes invisible after focus moves to the chat input.

## Best solution

The `hasToolConfirmation` context key needs to be maintained at the chat widget level (or a common ancestor in the context key hierarchy visible from the chat input), not exclusively in individual confirmation sub-parts' scoped context key services. The fix likely touches `chatToolActions.ts` (the action's `when` clause or the `focusInput` call) and the confirmation sub-parts (`abstractToolConfirmationSubPart.ts`, `chatTerminalToolConfirmationSubPart.ts`, etc.) to bind the key at the right scope. Alternatively, the action could avoid `focusInput()` when more confirmations remain, keeping focus within the confirmation widget scope.
