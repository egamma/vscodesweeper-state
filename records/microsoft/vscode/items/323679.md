# Issue #323679 — Session file corrupted: negative data.tokensRemoved rejected by SDK ("Number must be greater than or equal to 0")

[https://github.com/microsoft/vscode/issues/323679](https://github.com/microsoft/vscode/issues/323679)

## Summary

Team-filed bug tracking a real data-loss scenario where the @github/copilot SDK's Zod schema rejects negative tokensRemoved values, making entire sessions unloadable. The root cause and fix both live in the SDK, not in VS Code's own source. Keeping open per team-member policy.

## Change summary

Sessions with negative tokensRemoved values (produced by the SDK during context compaction) are rejected by the SDK's own Zod loader, causing complete session history loss.

## Best solution

Fix in the @github/copilot SDK: either clamp tokensRemoved to >= 0 at serialization time, or make the Zod schema tolerant (.catch(0) / .transform) so a metadata accounting glitch cannot reject an entire session. This is the same fix pattern needed for the sibling issues #312793 and #319244.
