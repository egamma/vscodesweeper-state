# Issue #183353 — All dark color themes are broken after 1.78 update

[https://github.com/microsoft/vscode/issues/183353](https://github.com/microsoft/vscode/issues/183353)

## Summary

This is a confirmed upstream Chromium/GPU color-management bug affecting dark themes on macOS with external monitors using non-sRGB color profiles (custom ICC, P3). Already correctly labeled upstream/chromium/gpu/multi-monitor. The root cause lives in the GPU compositor's color space handling, not in VS Code's own code. Keeping open as a tracking issue for the upstream defect.

## Change summary

Dark color themes render with incorrect background colors (washed-out/mismatched) on macOS external monitors with non-sRGB color profiles when GPU acceleration is enabled.

## Best solution

This needs a fix in Chromium's GPU color management pipeline for non-sRGB display profiles. VS Code should continue tracking the upstream bug. The workaround (disable hardware acceleration via settings) is documented. If Chromium ships a fix, verify it lands via an Electron update and then close. Related open tracking issue https://github.com/microsoft/vscode/issues/165681 covers a similar GPU/multi-monitor symptom and may consolidate with this one once the upstream root cause is addressed.
