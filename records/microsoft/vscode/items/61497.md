# Issue #61497 — Task Manager: provide the ability to reuse custom problem matchers

[https://github.com/microsoft/vscode/issues/61497](https://github.com/microsoft/vscode/issues/61497)

## Summary

Still a valid, unimplemented feature request with 58 upvotes — well above the 20-vote backlog threshold. Users can only define reusable named problem matchers via extensions today; defining them directly in tasks.json remains unimplemented on main.

## Change summary

Allow defining custom named problem matchers in tasks.json that can be referenced by name across multiple tasks, without requiring an extension.

## Best solution

Add a top-level `problemMatchers` section to the tasks.json schema (similar to how `inputs` works) that registers named matchers into `ProblemMatcherRegistry` at task-config parse time, so `$my-matcher` references resolve from the workspace config. The likely files are `src/vs/workbench/contrib/tasks/common/jsonSchema_v2.ts` (schema), `src/vs/workbench/contrib/tasks/common/taskConfiguration.ts` (parsing), and `src/vs/workbench/contrib/tasks/common/problemMatcher.ts` (registry).
