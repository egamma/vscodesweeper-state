# Issue #29589 — Auto indent on pasting does not satisfy indentation for braces in JS/TS

[https://github.com/microsoft/vscode/issues/29589](https://github.com/microsoft/vscode/issues/29589)

## Summary

Still present on main: the paste auto-indent logic applies a uniform offset from line 2 onward rather than computing per-line indentation, so closing braces remain over-indented. The tree-sitter-autoindent-candidate label already tracks this for a potential better solution.

## Change summary

Pasting unindented JS/TS code with braces incorrectly indents closing braces because the algorithm only adjusts line 1 and applies a flat offset to subsequent lines.

## Best solution

Implement per-line indentation computation during paste (using indentation rules or tree-sitter) in `src/vs/editor/contrib/indentation/browser/indentation.ts`, so that lines like closing braces get correct dedentation. The `tree-sitter-autoindent-candidate` label already tracks the preferred path forward.
