# Issue #302990 — Source Control diff takes the file from disk and ignores .gitattributes filters (smudge / clean)

[https://github.com/microsoft/vscode/issues/302990](https://github.com/microsoft/vscode/issues/302990)

## Summary

Valid request: VS Code's SCM diff reads the working tree file directly from disk without applying git clean filters, so the diff doesn't match `git diff` when smudge/clean filters are configured in `.gitattributes`. This is distinct from the textconv support added in #24883 and remains unimplemented. Routing to the git area for triage.

## Change summary

VS Code's Source Control diff ignores `.gitattributes` smudge/clean filters, producing a diff that doesn't match `git diff` output.

## Best solution

To match `git diff` behavior, VS Code would need to pipe the working tree file content through the configured clean filter before diffing (or use `git diff` output directly). Related issue https://github.com/microsoft/vscode/issues/268239 covers the staging side of filter support. The fix would likely touch `extensions/git/src/fileSystemProvider.ts` (how the modified side is served) or require a new content-provider path that applies clean filters. This is a niche but valid gap for users of git-crypt, notebook output stripping, or code-formatting filters.
