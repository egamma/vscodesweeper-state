# Issue #316744 — Renderer crash (EXC_BREAKPOINT in v8::internal::ThreadIsolation::RegisterJitPage) on macOS 26.4.1 / Apple M5

[https://github.com/microsoft/vscode/issues/316744](https://github.com/microsoft/vscode/issues/316744)

## Summary

This is an upstream V8/Electron crash in the JIT page registration path (`v8::internal::ThreadIsolation::RegisterJitPage`) on macOS 26 / Apple Silicon. The crash stack is entirely within the Electron Framework with no VS Code frames — VS Code cannot fix this; it needs to be tracked and reported upstream against Electron or V8/Chromium. Two independent reporters (M5 and M4 Pro) confirm the pattern on macOS 26.x with Electron 42.5.0.

## Change summary

VS Code renderer crashes with EXC_BREAKPOINT in V8's JIT page registration on macOS 26 / Apple Silicon, causing the window to show a 'renderer process gone' recovery banner.

## Best solution

File or link an upstream bug against Electron (or Chromium/V8) for the `RegisterJitPage` crash on macOS 26 Apple Silicon. VS Code ships Electron 42.5.0 (Chromium 148 / V8 14.8); the fix will come from an Electron or Chromium update. In the meantime, the related external PR (archestra-ai/archestra#6177) suggests `--no-v8-untrusted-code-mitigations` as a potential workaround flag. Route to the Electron/platform area owner to track the upstream dependency.
