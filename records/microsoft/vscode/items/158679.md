# Issue #158679 — Git - no-verify option for git push command

[https://github.com/microsoft/vscode/issues/158679](https://github.com/microsoft/vscode/issues/158679)

## Summary

Still missing on current main: the `push()` method in the git extension has no `--no-verify` support. The `noVerify` flag exists only for commits (`git.allowNoVerifyCommit`), not for push or sync. The feature request is valid, has 48 upvotes and is in the Backlog milestone — it should stay open.

## Change summary

Add a `--no-verify` option for git push (and sync) commands in the built-in git extension, analogous to the existing `git.allowNoVerifyCommit` setting for commits.

## Best solution

Add a `git.allowNoVerifyPush` setting (and optionally `git.confirmNoVerifyPush`) mirroring the commit equivalents, wire a `noVerify` parameter through `Repository.push()` / `Repository.sync()` down to `git.ts:push()` where `args.push('--no-verify')` would be added, and expose matching 'Push (No Verify)' / 'Sync (No Verify)' commands.
