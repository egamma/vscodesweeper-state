# Issue #222922 — Add `Report Issue...` option to gearwheel menus of extension in Extension view

[https://github.com/microsoft/vscode/issues/222922](https://github.com/microsoft/vscode/issues/222922)

## Summary

Valid feature request: adding a 'Report Issue' entry to the Extensions view context/gear menu is not yet implemented. The existing ReportExtensionIssueAction is only surfaced in the Runtime Extensions editor, not in the Extensions sidebar. Low community traction (0 upvotes) but a reasonable convenience improvement — keeping open in the Backlog.

## Change summary

Add a 'Report Issue...' option to the gear/context menu of extensions in the Extensions view sidebar, so users can file issues against a specific extension without navigating through the Issue Reporter dropdowns.

## Best solution

Register the existing `ReportExtensionIssueAction` (or an equivalent command) into `MenuId.ExtensionContext` in `src/vs/workbench/contrib/extensions/browser/extensions.contribution.ts`, gated on the extension having a repository URL or being built-in. The action class already exists at `src/vs/workbench/contrib/extensions/common/reportExtensionIssueAction.ts` and calls `IWorkbenchIssueService.openReporter`.
