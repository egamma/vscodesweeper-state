# Issue #309368 — `TerminalSandboxService.checkForSandboxingPrereqs` re-runs on every command rewrite

[https://github.com/microsoft/vscode/issues/309368](https://github.com/microsoft/vscode/issues/309368)

## Summary

Valid debt item filed by a team member — the prereq check still runs uncached on every rewrite call on current main. No fix has landed.

## Change summary

Cache the result of `checkForSandboxingPrereqs` and invalidate only on configuration or workspace-folder changes, avoiding redundant binary probes on every `run_in_terminal` invocation.

## Best solution

Add a cached prereq result in `TerminalSandboxEngine` invalidated by `onDidChangeConfiguration` (sandbox settings) and `onDidChangeWorkspaceFolders`, so the binary probing and config reads run at most once per session until state actually changes.
