## Summary

Verified duplicate: this reports the same rename-doesn't-persist bug tracked by the canonical issue #283514 ("Allow to rename local chat sessions"), which is On Deck with an open fix PR. The root cause — `setSessionTitle` missing a `flushIndex()` call — is confirmed on current main.

## Change summary

Renaming a chat thread via right-click shows the new name briefly but reverts to the old name on navigation or reload.
