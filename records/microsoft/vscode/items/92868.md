# Issue #92868 — Terminal doesn't flush all line data before firing onExit

[https://github.com/microsoft/vscode/issues/92868](https://github.com/microsoft/vscode/issues/92868)

## Summary

Fixed on main since July 2020 — the terminal now awaits `_flushXtermData()` before firing the exit event, ensuring all xterm data is parsed before problem matchers are signaled. The reporter and maintainer both confirmed the fix in comments 8–9.

## Change summary

ProblemMatchers on CustomExecution tasks fail to process output because the terminal fires onExit before xterm finishes parsing buffered write data.

## Best solution

Keep the shipped fix (`_flushXtermData` in `_onProcessExit`); optionally remove the residual `setTimeout` hack at terminalTaskSystem.ts:1126 which is no longer needed but was never cleaned up.
