# Issue #291445 — Terminal timeout shouldn't count when waiting for user input

[https://github.com/microsoft/vscode/issues/291445](https://github.com/microsoft/vscode/issues/291445)

## Summary

Team-authored issue describing a real UX problem where terminal tool timeouts count wall-clock time instead of execution time, timing out while waiting for user input. No fix found on main.

## Change summary

Terminal tool timeout fires while the command is waiting for user input, preventing the user from responding.

## Best solution

Pause or exclude the timeout countdown when the terminal is in an 'awaiting user input' state, so only actual execution time counts. The fix likely belongs in the terminal tool execution layer (possibly in the agent host's terminal command harness or the chat terminal tool integration). The secondary issue about stale 'awaiting input' prompts after timeout should also be addressed.
