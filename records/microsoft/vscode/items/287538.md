# Issue #287538 — Default node_modules glob for worktrees does not copy .bin

[https://github.com/microsoft/vscode/issues/287538](https://github.com/microsoft/vscode/issues/287538)

## Summary

Still present on main: the worktree file-copy logic uses `workspace.findFiles2` which does not enumerate symlinks (like those in `node_modules/.bin`), so they are never included in the copy set. No fix has landed. The setting was marked experimental and its default reset to `[]` (commit 25c32afcbc9), but the underlying symlink gap remains.

## Change summary

When creating a git worktree with `git.worktreeIncludeFiles` set to include `node_modules`, symlinked entries like `node_modules/.bin` are silently skipped.

## Best solution

The `_getWorktreeIncludePaths` method in `extensions/git/src/repository.ts` relies on `workspace.findFiles2` to discover files, which skips symlinks. The fix should either use a file-enumeration method that includes symlinks, or explicitly handle symlinked directories (e.g. via `fs.readdir` with `withFileTypes`) alongside the glob-based discovery. The related issue https://github.com/microsoft/vscode/issues/287539 (copy errors) may share the same root cause and could be addressed together.
