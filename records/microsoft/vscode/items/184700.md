# Issue #184700 — Alignment is incorrectly replaced by tabs

[https://github.com/microsoft/vscode/issues/184700](https://github.com/microsoft/vscode/issues/184700)

## Summary

Still reproduces on main: `normalizeIndentation` in `cursorTypeEditOperations.ts` calls `_normalizeIndentationFromWhitespace` which converts all leading whitespace to max-tabs+remainder-spaces, destroying alignment spaces that follow indentation tabs. The bug is confirmed in the source; no fix has landed.

## Change summary

Pressing Enter replaces alignment spaces with tabs when using tabs-for-indentation, breaking the 'tabs for indent, spaces for alignment' style.

## Best solution

The autoindent logic in `cursorTypeEditOperations.ts` should distinguish between indentation (tabs) and alignment (trailing spaces after the last tab) when copying whitespace from the previous line, rather than flattening all leading whitespace through `normalizeIndentation`. The fix is localized to `src/vs/editor/common/cursor/cursorTypeEditOperations.ts` (lines ~541-565) where `config.normalizeIndentation(indentation)` is called on Enter.
