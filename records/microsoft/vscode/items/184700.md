# Issue #184700 — Alignment is incorrectly replaced by tabs

[https://github.com/microsoft/vscode/issues/184700](https://github.com/microsoft/vscode/issues/184700)

## Summary

Still reproducible on main: `normalizeIndentation` collapses all leading whitespace into tabs+remainder without distinguishing indentation from alignment spaces. The issue is valid, acknowledged by maintainers, and on the Backlog.

## Change summary

When pressing Enter with tabs-for-indentation, VS Code normalizes alignment spaces into tabs, breaking the 'tabs for indentation, spaces for alignment' style.

## Best solution

The auto-indent logic (in `getEnterAction` / `autoIndent.ts`) should preserve the raw whitespace prefix of the previous line rather than passing it through `normalizeIndentation` when the previous line uses a mixed tabs+spaces pattern for alignment. The fix likely involves `src/vs/editor/common/core/misc/indentation.ts` and whichever caller in `src/vs/editor/common/languages/autoIndent.ts` invokes `normalizeIndentation` on the inherited indent string.
