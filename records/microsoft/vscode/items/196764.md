# Issue #196764 — allow configuring the ghost text style

[https://github.com/microsoft/vscode/issues/196764](https://github.com/microsoft/vscode/issues/196764)

## Summary

Valid accessibility-motivated feature request from a team member. The ghost text font style (italic) is hardcoded in CSS with no user configuration; only color tokens exist. The request remains unimplemented on main.

## Change summary

Allow users to configure the ghost text style (e.g., italic vs normal) to improve readability and accessibility.

## Best solution

Add an editor setting (e.g., `editor.inlineSuggest.fontStyle`) that controls whether ghost text renders as italic, normal, or other styles. The change would be localized to `src/vs/editor/contrib/inlineCompletions/browser/view/ghostText/ghostTextView.ts` and its CSS, applying the configured style dynamically instead of the hardcoded `font-style: italic`.
