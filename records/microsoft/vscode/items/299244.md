# Issue #299244 — Git blame and gutter decorations broken for worktrees stored inside bare repositories

[https://github.com/microsoft/vscode/issues/299244](https://github.com/microsoft/vscode/issues/299244)

## Summary

Confirmed bug: the `isDescendant` check on line 30 of `extensions/git/src/quickDiffProvider.ts` uses `commonPath` (the bare repo root) to filter git-internal files, but when worktrees live inside the bare repo, every working-tree file matches — silently disabling quick diff, blame, and gutter decorations. An open PR (#299245) already targets the fix.

## Change summary

Git blame, gutter change decorations, and stage-hunk do not work when worktrees are stored inside a bare repository because `provideOriginalResource` incorrectly filters all working-tree files as git-internal.

## Best solution

Review and land https://github.com/microsoft/vscode/pull/299245, which fixes the overly broad `isDescendant` guard. The check needs to distinguish actual git metadata paths from working-tree paths that happen to be descendants of `commonPath` when a bare repo layout is used.
