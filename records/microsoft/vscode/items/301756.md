# Issue #301756 — Touchscreen issue with model selection.

[https://github.com/microsoft/vscode/issues/301756](https://github.com/microsoft/vscode/issues/301756)

## Summary

The model picker uses `addDisposableGenericMouseDownListener` which only upgrades to pointer events on iOS — Windows touchscreen users get `mousedown` only, which may not fire reliably from touch in all Electron/Chromium scenarios. No fix found on main, and no duplicate exists. However, the report lacks confirmation from other users and the reproduction path isn't fully clear (Chromium normally synthesizes mousedown from touch on Windows).

## Change summary

The Copilot model selector dropdown does not respond to touchscreen taps on Windows, while other Chat dropdowns work.

## Best solution

Investigate whether the `addDisposableGenericMouseDownListener` in `_registerButtonAction` (chatModelPicker.ts:1272) needs to use pointer events on Windows touchscreens, or whether another ancestor element is swallowing the touch event before synthesis. The fix likely belongs in `src/vs/base/browser/dom.ts` (broadening the pointer-event upgrade beyond iOS) or in the model picker's specific event setup.
