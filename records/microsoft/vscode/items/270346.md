# Issue #270346 — Do not highlight ambiguous characters in JSX/TSX

[https://github.com/microsoft/vscode/issues/270346](https://github.com/microsoft/vscode/issues/270346)

## Summary

Valid feature request: JSX/TSX text children use the scope `meta.jsx.children.tsx` (not `string.*`), so they aren't classified as StandardTokenType.String and the `includeStrings` setting has no effect on them. The request to suppress unicode highlighting in JSX children is reasonable but has no traction yet — keeping open for the community process.

## Change summary

Request to suppress ambiguous-character unicode highlighting inside JSX/TSX text children, treating them like strings.

## Best solution

Either extend the `hideInStringTokens` mechanism to also cover JSX children tokens (likely requires a token-type mapping change or a dedicated `hideInJsxChildrenTokens` decoration option), or add a new setting like `editor.unicodeHighlight.includeJsxText`. The relevant files are `src/vs/editor/contrib/unicodeHighlighter/browser/unicodeHighlighter.ts` and `src/vs/editor/common/viewModel/viewModelDecoration.ts`. Alternatively, the TextMate grammar for TSX could map JSX children to `string.*` scope, but that would have broader implications.
