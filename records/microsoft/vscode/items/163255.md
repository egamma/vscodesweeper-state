# Issue #163255 — WORKSPACE_CONTAINS_TIMEOUT should be configurable

[https://github.com/microsoft/vscode/issues/163255](https://github.com/microsoft/vscode/issues/163255)

## Summary

Still valid: WORKSPACE_CONTAINS_TIMEOUT remains hardcoded at 7000ms in current main with no user-facing configuration. The request has 31 upvotes and was promoted to Backlog — keeping open.

## Change summary

Make the 7-second workspaceContains glob-search timeout configurable so users with many extensions or large workspaces can increase it to avoid extensions failing to activate.

## Best solution

Expose the timeout as a setting (e.g. `extensions.workspaceContainsTimeout`) in `src/vs/workbench/services/extensions/common/workspaceContains.ts`, defaulting to 7000ms. This lets power users with large workspaces or slow disks trade startup speed for reliable activation.
