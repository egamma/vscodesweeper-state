# Issue #163255 — WORKSPACE_CONTAINS_TIMEOUT should be configurable

[https://github.com/microsoft/vscode/issues/163255](https://github.com/microsoft/vscode/issues/163255)

## Summary

Keep open: this is an accepted Backlog feature request with enough community upvotes, and current main still has a hard-coded workspaceContains timeout rather than a configurable value.

## Change summary

Allow users or extension hosts to configure the timeout used while resolving `workspaceContains` activation events.

## Best solution

Extension Host maintainers should either expose a carefully scoped setting for the `workspaceContains` search timeout or improve the activation search behavior so large extension/workspace setups do not fail activation after the fixed 7 second limit.
