# Issue #144211 — Debugger does not respect supportsDelayedStackTraceLoading false

[https://github.com/microsoft/vscode/issues/144211](https://github.com/microsoft/vscode/issues/144211)

## Summary

Still reproduces on current main: when `supportsDelayedStackTraceLoading` is false/missing, VS Code still sends `levels: 20` in the stackTrace request instead of omitting `startFrame` and `levels` to request all frames per the DAP spec.

## Change summary

The debugger sends `levels: 20` in the stackTrace request even when the debug adapter does not advertise `supportsDelayedStackTraceLoading`, causing adapters to return truncated stacks.

## Best solution

In `debugModel.ts`, when `supportsDelayedStackTraceLoading` is false, `fetchCallStack` should pass `levels: 0` (or omit it) so the raw session sends a stackTrace request without `startFrame`/`levels`, requesting all frames per the DAP spec. The non-delayed path in `refreshTopOfCallstack` (line 1676) already calls `fetchCallStack()` without arguments, but the default `levels = 20` on line 660 defeats this.
