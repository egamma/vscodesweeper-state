# Issue #234194 — Focus movement with Attached file pill should be improved

[https://github.com/microsoft/vscode/issues/234194](https://github.com/microsoft/vscode/issues/234194)

## Summary

Some of the original keyboard-focus issues with implicit context attachment pills appear partially addressed by the #248279 work (arrow navigation, Delete/Backspace for regular attachments, clear button no longer Tab-focusable), but the specific bugs — focus lost after toggling the eye/toggle button, and Escape not returning focus to the chat input — are not conclusively fixed in the current source. The issue was explicitly assigned to @justschen by @isidorn and may have remaining work.

## Change summary

Keyboard focus is lost when toggling the implicit context pill's enable/disable button, and pressing Escape on a pill does not return focus to the chat input.

## Best solution

Verify whether the remaining focus-management gaps (focus restoration after toggle re-render, Escape-to-input) are still present after the #248279 work landed. If they are, add focus restoration logic in `implicitContextAttachment.ts` after the toggle re-render and wire up an Escape keydown handler on attachment pill elements to call `focusInput()` on the chat widget.
