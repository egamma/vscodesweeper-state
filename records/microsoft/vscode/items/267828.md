# Issue #267828 — Dragging a prompt file in to chat behaves as if I invoked it

[https://github.com/microsoft/vscode/issues/267828](https://github.com/microsoft/vscode/issues/267828)

## Summary

Team-authored issue; the bug appears still present on main — prompt files attached via drag-and-drop are still treated as invocations rather than plain file references. The legacy behavior acknowledged by @aeschli has not been fully removed.

## Change summary

Dragging a .prompt file into the chat input treats it as an invocation (runs the prompt) instead of attaching it as a plain file reference.

## Best solution

Remove the legacy code path that auto-invokes prompt files when they are attached via drag-and-drop or 'Add File to Chat'. Only the `/foo` slash-command syntax should trigger prompt invocation. Likely touches `chatAttachmentModel.ts` (the special `isPromptFileVariableEntry` handling on attach) and the drag-drop / context-attach flows in `chatContext.ts` or `chatWidget.ts`.
