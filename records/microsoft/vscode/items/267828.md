# Issue #267828 — Dragging a prompt file in to chat behaves as if I invoked it

[https://github.com/microsoft/vscode/issues/267828](https://github.com/microsoft/vscode/issues/267828)

## Summary

Team-authored issue; the described behavior (prompt files invoked when dragged/attached to chat instead of being treated as plain file references) still appears present on main — prompt file attachments are still handled with special invoke semantics via PromptFileVariableKind.PromptFile.

## Change summary

Dragging a prompt file into chat invokes it (runs it as a slash command) rather than attaching it as a plain file reference.

## Best solution

Remove the automatic prompt-invocation behavior when a prompt file is attached via drag-and-drop or 'Add File to Chat'; only invoke prompts via the explicit `/foo` slash command syntax. The relevant handling is in the chat widget attachment pipeline (chatWidget.ts, chatAttachmentModel.ts, chatContinueInAction.ts) where `toPromptFileVariableEntry` with `PromptFileVariableKind.PromptFile` is used.
