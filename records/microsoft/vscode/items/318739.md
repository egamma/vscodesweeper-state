# Issue #318739 — Copilot inline suggestions override or slow down snippets

[https://github.com/microsoft/vscode/issues/318739](https://github.com/microsoft/vscode/issues/318739)

## Summary

Keep open for maintainer judgment: current main still contains the quick-suggest/inline-completions behavior that matches the report, and there is no concrete fixing commit for this issue.

## Change summary

Copilot inline completions can suppress or delay snippet and IntelliSense suggestions unless `editor.quickSuggestions.other` is set back to `on`.

## Best solution

Decide in the editor suggest/inline-completions area whether `offWhenInlineCompletions` should remain the default, or whether snippets should bypass the suppression/wait path so snippet completions stay responsive while inline completions are active.

## Evidence

- **current code:** In `src/vs/editor/common/config/editorOptions.ts`, current main defaults `editor.quickSuggestions.other` to `offWhenInlineCompletions`, matching the workaround suggested in the issue comments when users set it back to `on`.
- **suggest behavior:** `src/vs/editor/contrib/suggest/browser/suggestModel.ts` checks `offWhenInlineCompletions`, waits for inline completions when an inline provider exists, suppresses quick suggestions if an inline suggestion is already showing, and has a 750ms fallback timeout.
- **history:** Commit fb72b9be76b694b456c5b15903e1f63457848ec8 temporarily restored the default to `on` because Copilot inline completions suppressed snippet, IntelliSense, and Emmet completions, but commit f08b0cab721efb62b881aafccbadc031366fdcb6 reverted that restoration the same day.
- **maintainer context:** A VS Code member tagged `@alexdima` with “Another one”, and another member pointed users to the `editor.quickSuggestions` workaround rather than closing the issue as extension-caused or invalid.
