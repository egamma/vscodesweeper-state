# Issue #247468 — Open file/folder dialog isn't floating on Wayland

[https://github.com/microsoft/vscode/issues/247468](https://github.com/microsoft/vscode/issues/247468)

## Summary

The file dialog floating/transient behavior on Wayland is governed by Electron's ozone-wayland layer. VS Code's own code correctly passes the parent BrowserWindow to electron.dialog.showOpenDialog(). The previous fix (#203608) was an Electron 32 upgrade; the reporter claims regression on 1.99.3 with Hyprland. This is an upstream Electron/Chromium issue, likely compositor-specific (Hyprland).

## Change summary

On Wayland with Hyprland, the native Open File dialog is not opened as a floating/transient window — it tiles like a regular window instead of appearing as a modal popup.

## Best solution
