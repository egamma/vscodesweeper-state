# Issue #154348 — Unable to use outline with search editor

[https://github.com/microsoft/vscode/issues/154348](https://github.com/microsoft/vscode/issues/154348)

## Summary

Still reproduces on current main: clicking an outline item in a search editor tries to open the `search-editor:` model URI via `openCodeEditor`, which fails because that URI is not separately resolvable. A draft PR #317261 exists on branch `copilot/fix-invalid-outline-search-editor` but has not been merged.

## Change summary

Clicking an item in the Outline panel while a search editor is active opens a broken new search editor tab instead of navigating within the current one.

## Best solution

Review and land the draft PR https://github.com/microsoft/vscode/pull/317261, which guards the `reveal` method in `DocumentSymbolsOutline` to apply the selection in-place when the outline's model URI matches the current editor (avoiding the `openCodeEditor` call for non-openable URI schemes like `search-editor:`).
