# Issue #160644 — Git - Cannot add merge change if file is deleted

[https://github.com/microsoft/vscode/issues/160644](https://github.com/microsoft/vscode/issues/160644)

## Summary

Still reproducible on main: when a merge-conflicted file is deleted from disk, the stage ('+') button silently fails because `grep()` in `util.ts` throws ENOENT on the missing file, and the error propagates unhandled through `categorizeResourceByResolution` in `commands.ts`.

## Change summary

Clicking the '+' button on a merge-conflicted file that was deleted from disk does nothing — the stage action silently fails.

## Best solution

In `categorizeResourceByResolution` (extensions/git/src/commands.ts), handle the case where the file no longer exists on disk: catch ENOENT from the `grep` call and treat deleted files as deletion conflicts (or resolved), so `git add` can stage the deletion. The maintainer's open question about the expected UX (comment 4) should be resolved first.
