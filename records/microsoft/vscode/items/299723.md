## Summary

Confirmed on current main: InstructionBreakpointsRenderer lacks context key setup (breakpointSupportsCondition, breakpointItemType) and inline action bar rendering that all other breakpoint renderers provide, causing missing condition editing and inline buttons for instruction breakpoints.

## Change summary

Add the missing context key updates (breakpointSupportsCondition, breakpointItemType, breakpointHasMultipleModes) and inline action bar rendering to InstructionBreakpointsRenderer.renderInstructionBreakpoint(), mirroring DataBreakpointsRenderer and FunctionBreakpointsRenderer patterns in the same file.
