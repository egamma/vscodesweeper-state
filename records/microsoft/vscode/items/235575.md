# Issue #235575 — "Do you want to terminate the active terminal session?" on close for background terminals

[https://github.com/microsoft/vscode/issues/235575](https://github.com/microsoft/vscode/issues/235575)

## Summary

Still a valid open bug: a team member confirmed it reproduces after both PR #236588 and PR #240074 merged. The `help wanted` label is appropriate — the fix for `foregroundInstances` filtering is in place but doesn't cover all paths the Python extension uses to create background terminals.

## Change summary

Background terminals created by extensions (e.g. Python/Jedi) still trigger the 'Do you want to terminate?' confirmation dialog on window close.

## Best solution

Investigate why the Python extension's background terminal isn't classified as a backgrounded instance (likely not setting `hideFromUser: true` in its shell launch config, or using a different terminal creation path). The fix likely lives in `src/vs/workbench/contrib/terminal/browser/terminalService.ts` around the `confirmOnExit` logic or the terminal creation path that the Python extension uses.
