# Issue #196606 — Terminal quick fix should go away when a command is run

[https://github.com/microsoft/vscode/issues/196606](https://github.com/microsoft/vscode/issues/196606)

## Summary

Still reproduces on main: the quick fix addon only dismisses the decoration in `_resolveQuickFixes` (fired on `onCommandFinished`), so it stays visible while the next command is running. A listener on `onCommandStarted` or `onCommandExecuted` is needed to clear it earlier.

## Change summary

Terminal quick fix decoration remains visible while the next command is executing, instead of being dismissed when the command starts.

## Best solution

Add a listener on `commandDetection.onCommandStarted` (or `onCommandExecuted`) in `_registerCommandHandlers` that calls `_disposeQuickFix` for non-empty commands, mirroring the guard at line 199.
