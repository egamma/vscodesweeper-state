# Issue #253271 — Agent mode does not use correct path syntax for git bash on windows

[https://github.com/microsoft/vscode/issues/253271](https://github.com/microsoft/vscode/issues/253271)

## Summary

Still valid on main: the agent host terminal tool does not convert Windows-style absolute paths to Git Bash-style paths (/c/Users/...) when issuing cd or other commands. Path conversion utilities exist in terminalGitBashHelpers.ts but are only wired into the terminal completion service, not the chat agent's terminal execution path.

## Change summary

Agent mode uses Windows path syntax (C:\Users\...) when running terminal commands in Git Bash, which fails because Git Bash requires /c/Users/... style paths.

## Best solution

The agent host terminal contribution (or the terminal tool itself) should detect when the active shell is Git Bash and convert absolute paths using the existing `windowsToGitBashPath` helper before sending commands. Related work in https://github.com/microsoft/vscode/issues/313790 (piping profile args to agent host) partially addresses Git Bash integration but doesn't cover path syntax conversion in issued commands.
