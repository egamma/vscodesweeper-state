# Issue #189525 — HTML attribute quotes do not auto-close in PHP files

[https://github.com/microsoft/vscode/issues/189525](https://github.com/microsoft/vscode/issues/189525)

## Summary

Still reproducible on main: in PHP files, the tokenizer classifies the position after `=` in HTML attributes as a string-type token, causing `shouldAutoClose` to reject the quote due to the `notIn: ["string"]` rule — even with `autoClosingQuotes: always`.

## Change summary

Typing a double quote after `=` in an HTML attribute inside a PHP file does not auto-close the quote, even with `autoClosingQuotes` set to `always`.

## Best solution

The fix likely belongs in `cursorTypeEditOperations.ts` around line 262: when `autoCloseConfig === 'always'`, the `pair.shouldAutoClose()` token-type check should be bypassed (or relaxed) so that quotes still auto-close regardless of the token scope. Alternatively, the PHP grammar's tokenization of HTML attribute value positions could be adjusted, but changing the auto-closing logic is more robust.
