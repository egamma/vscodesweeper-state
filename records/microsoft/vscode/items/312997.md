# Issue #312997 — Out of memory: Cannot allocate Wasm memory for new instance

[https://github.com/microsoft/vscode/issues/312997](https://github.com/microsoft/vscode/issues/312997)

## Summary

The report shows the remote server's bundled Node runtime failing inside Node's internal undici WebAssembly path before VS Code server startup, but it lacks enough environment detail to tell whether this is a VS Code packaging issue, a Node/V8/undici runtime bug, or an HPC memory-limit configuration.

## Change summary

Connecting from Windows VS Code 1.117.0 to a Rocky Linux 8.10 HPC host fails when code-server runs because WebAssembly.instantiate cannot allocate memory in Node internal undici.

## Best solution

Ask for the Remote SSH/remote-server log, remote architecture, memory limits such as ulimit/cgroup constraints, whether the latest Insiders server still fails, and whether the failure reproduces by running the exact bundled server node binary directly on the Rocky host. If it reproduces outside VS Code server startup, track it as an upstream Node/V8/undici runtime issue.
