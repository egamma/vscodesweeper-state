# Issue #234568 — Git - Git blame editor decoration color

[https://github.com/microsoft/vscode/issues/234568](https://github.com/microsoft/vscode/issues/234568)

## Summary

The core concern (blame decoration color too bright, no customization) was addressed by PR #242665: the default was changed to the lighter `editorInlayHint.foreground` and a dedicated theme color `git.blame.editorDecorationForeground` was added for full customization. Shipped in 1.99.0.

## Change summary

Git blame editor decoration color was too bright and did not match theme comment colors; no dedicated theme color existed for customization.

## Best solution

Keep the shipped fix — the default is now `editorInlayHint.foreground` (lighter than the old `editorCodeLens.foreground`), and users who want a different color (e.g. matching their comment color) can set `git.blame.editorDecorationForeground` in `workbench.colorCustomizations`. No further change needed.
