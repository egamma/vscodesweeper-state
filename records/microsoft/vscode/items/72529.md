# Issue #72529 — Support restart of remote EH to avoid window reloads

[https://github.com/microsoft/vscode/issues/72529](https://github.com/microsoft/vscode/issues/72529)

## Summary

Still an open gap on current main: the `TODO@aeschli, workaround` at relauncher.contribution.ts:306 still does a full window reload when workspace folders change under a remote authority, rather than gracefully restarting the remote extension host. The local case was solved years ago but the remote case remains unaddressed.

## Change summary

Support graceful restart of the remote extension host when workspace folders change, instead of a full window reload.

## Best solution

Implement remote extension host restart (analogous to the local `stopExtensionHosts`/`startExtensionHosts` path at line 307–311) so that the `hostService.reload()` workaround on line 306 can be removed. The fix lives in `src/vs/workbench/contrib/relauncher/browser/relauncher.contribution.ts` and likely requires work in `src/vs/workbench/services/extensions/common/abstractExtensionService.ts` to support remote EH lifecycle management.
