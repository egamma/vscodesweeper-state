# Issue #120270 — dragging tabs to remote-ssh teminal window pastes paths with backslashes

[https://github.com/microsoft/vscode/issues/120270](https://github.com/microsoft/vscode/issues/120270)

## Summary

Fixed on main and shipped in 1.75.0 — the terminal's `sendPath` now converts path separators when the frontend and backend OS differ, which is exactly the reported problem. A commenter confirmed it was fixed in May 2023.

## Change summary

Dragging an editor tab into a remote-SSH terminal pastes the file path with Windows backslashes instead of the remote OS's forward slashes.

## Best solution

Keep the shipped fix from commit 5807d5f2c74 (1.75.0); no further change needed. The path separator conversion in `preparePathForShell` handles cross-OS remote scenarios correctly.
