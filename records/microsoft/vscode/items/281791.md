# Issue #281791 — GitHub Copilot agents should remember models on a per-agent basis.

[https://github.com/microsoft/vscode/issues/281791](https://github.com/microsoft/vscode/issues/281791)

## Summary

Valid feature request — per-agent model memory is not yet implemented. The model selection storage key is scoped by session type (e.g., agent-host vs local), not by individual agent/mode within the same session type. Already triaged into Backlog with correct labels.

## Change summary

When switching between custom agents (e.g., a 'Reviewer' agent and the default Agent mode), the model selection should be remembered independently per agent rather than sharing a single global selection.

## Best solution

Extend `getSelectedModelStorageKey()` in `chatInputPart.ts` to incorporate the active agent identifier (not just `sessionType`) so that each agent/mode gets its own persisted model selection. The current key format `chat.currentLanguageModel.${location}` would need an agent-scoped variant.
