# Issue #281791 — GitHub Copilot agents should remember models on a per-agent basis.

[https://github.com/microsoft/vscode/issues/281791](https://github.com/microsoft/vscode/issues/281791)

## Summary

The model-selection storage key is per-session-type (e.g. local vs agent-host), not per-mode/custom-agent. Recent work (commit 81d97ff0bd4) improved per-session-type persistence but doesn't address the per-mode model memory this issue requests. This remains a valid feature request.

## Change summary

When switching between chat modes (e.g. built-in Agent and a custom Reviewer agent), the previously-selected model should be remembered and restored per-mode rather than being shared across all local modes.

## Best solution

Extend `getSelectedModelStorageKey()` in `chatInputPart.ts` to incorporate the current mode/agent identifier into the key (e.g. `chat.currentLanguageModel.${location}.${sessionType}.${modeId}`), so each mode remembers its own last-used model independently.
