[https://github.com/microsoft/vscode/issues/154102](https://github.com/microsoft/vscode/issues/154102)

## Summary

Confirmed iPadOS-specific bug where the editor's hidden textarea doesn't regain input focus after interacting with a split-view/slide-over app via touch. Still appears unfixed on main — no commits reference this issue or address this specific iPad multitasking focus scenario.

## Change summary

On iPadOS, touching the vscode.dev editor after using split view or slide over does not move input focus back to the editor's hidden textarea.

## Best solution

The editor's textarea focus logic (likely in the TextAreaInput or related focus-handling code in src/vs/editor/browser/) needs to handle the case where iPadOS touch events from a multitasking transition don't trigger the same focus path as mouse clicks. The fix would ensure touch interactions restore focus to the hidden textarea the same way mouse clicks do.
