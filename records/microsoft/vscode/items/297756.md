# Issue #297756 — forking didn't submit the prompt

[https://github.com/microsoft/vscode/issues/297756](https://github.com/microsoft/vscode/issues/297756)

## Summary

Team-authored issue (MEMBER); the bug is confirmed in source — `/fork` discards any additional prompt text the user types after the command. Keeping open per hard keep-open rules.

## Change summary

Using `/fork some prompt text` forks the conversation but does not submit the trailing prompt text in the new session.

## Best solution

Pass the prompt text from the slash-command callback through to ForkConversationAction, and after the forked session opens, submit it as a new message. The fix touches `src/vs/workbench/contrib/chat/browser/chatSlashCommands.ts` (the `/fork` callback at ~line 206 that ignores `_prompt`) and `src/vs/workbench/contrib/chat/browser/actions/chatForkActions.ts` (to accept and forward the prompt into the newly opened session).
