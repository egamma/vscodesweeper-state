# Issue #315314 — Hydrating large chat session items block renderer during startup

[https://github.com/microsoft/vscode/issues/315314](https://github.com/microsoft/vscode/issues/315314)

## Summary

Real startup performance issue acknowledged and diagnosed by a team member (connor4312) as large chat session item hydration blocking the renderer, likely related to binary content. No fix has landed on main — both external PRs were closed without merge — so the issue remains valid.

## Change summary

Hydrating large chat session items (possibly binary content) on the main thread blocks the renderer for 1–2 seconds during startup.

## Best solution

Address the synchronous hydration cost in `src/vs/workbench/api/browser/mainThreadChatSessions.ts` — either defer large-item deserialization off the critical startup path, chunk the work across frames, or skip binary content during initial hydration and resolve it lazily via the existing `resolveChatSessionItem` pattern.
