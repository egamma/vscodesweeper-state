# Issue #137953 — Break all processes when one process breaks (mult-target debugging)

[https://github.com/microsoft/vscode/issues/137953](https://github.com/microsoft/vscode/issues/137953)

## Summary

Valid feature request with 26 upvotes, already in Backlog. The requested capability — automatically pausing all debug sessions when one hits a breakpoint during multi-target debugging — is not implemented on current main. The existing compound config `stopAll` property only controls termination, not pause-on-break propagation.

## Change summary

When one debug session hits a breakpoint during multi-target debugging, automatically pause (break) all other active debug sessions.

## Best solution

Implement a compound launch config option (e.g. `pauseAll` or `breakAll`) that, when a stopped event fires on one session, sends a DAP `pause` request to all sibling sessions in the compound. The logic would live near `src/vs/workbench/contrib/debug/browser/debugService.ts` (where `stopAll` is already handled for compound configs) and `src/vs/workbench/contrib/debug/browser/debugSession.ts` (where stopped events are processed). This requires a product decision on UX and scope.
