## Summary

Still present on main: only `*` is escaped at line 752 of fileActions.ts, so Markdown-active characters like `_` in filenames inject formatting into the error dialog. A team member already triaged this as a papercut (visual annoyance, not a security issue), which seems appropriate — keeping open for the hardening fix.

## Change summary

File validation error messages render unexpected Markdown formatting when invalid filenames contain unescaped Markdown characters like `_`, backticks, `[`, or `~`.
