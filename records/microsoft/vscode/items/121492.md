## Change summary

ALT+Scroll on Debian/XFCE triggers both VS Code file scroll and XFCE screen zoom simultaneously.

## Best solution

This is an XFCE window-manager keybinding conflict — the user needs to disable ALT+Scroll zoom in XFCE settings. VS Code cannot suppress WM-level key bindings. The existing `upstream` label is correct; no code change in VS Code would resolve this.
