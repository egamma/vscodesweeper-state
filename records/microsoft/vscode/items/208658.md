Valid feature request for responsive/saveable panel layouts and an extension API to get/set panel sizes. No existing capability covers this on current main, and it has 10 upvotes with recent community interest. Keeping open under discussion.

## Change summary

Request responsive panel layouts, layout save/restore, or an extension API to programmatically get and set panel widths.

## Best solution

The most pragmatic path would be exposing panel size get/set through a proposed extension API (in IWorkbenchLayoutService), letting the community build responsive-layout extensions. A built-in 'save/restore named layout' feature could follow if demand grows. The layout code lives in src/vs/workbench/browser/layout.ts and the service interface in src/vs/workbench/services/layout/browser/layoutService.ts.
