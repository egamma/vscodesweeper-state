# Issue #154977 — onDidChangeTextEditorVisibleRanges shows strange behaviour!

[https://github.com/microsoft/vscode/issues/154977](https://github.com/microsoft/vscode/issues/154977)

## Summary

Still present on main: the visible-ranges event fires inconsistently during editor tab restoration because the scroll/layout listeners emit intermediate states as the editor restores cursor position and then scroll position sequentially. No fix has landed.

## Change summary

onDidChangeTextEditorVisibleRanges fires 0, 1, or 2 times inconsistently when switching editor tabs or opening files, depending on the memorized cursor/scroll position.

## Best solution

The mainThreadEditor.ts visible-ranges event path (lines 313–326) should either batch/debounce the property updates during editor restoration, or suppress intermediate visible-range notifications until the editor has finished restoring its scroll state. The fix likely lives in `src/vs/workbench/api/browser/mainThreadEditor.ts` around the `_setCodeEditor` method.
