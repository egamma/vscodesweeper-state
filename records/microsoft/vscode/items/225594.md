# Issue #225594 — `Load More Stack Frames` renders poorly when horizontalScrolling enabled

[https://github.com/microsoft/vscode/issues/225594](https://github.com/microsoft/vscode/issues/225594)

## Summary

Still reproduces on main: the 'Load More Stack Frames' and 'Show More' labels use `text-align: center` in CSS, which centers relative to the full scrollable width rather than the visible viewport when `workbench.list.horizontalScrolling` is enabled, pushing them offscreen.

## Change summary

The 'Load More Stack Frames' / 'Show N More Stack Frames' labels in the call stack view are centered relative to the full scrollable content width, making them invisible when long frame names cause horizontal overflow.

## Best solution

Fix the `.load-all` and `.show-more` CSS rules in `debugViewlet.css` to use `position: sticky; left: 0` (or similar viewport-relative centering) so the label stays visible within the viewport when horizontal scrolling is active. The fix is small and localized to the debug call stack view CSS.
