# Issue #260851 — Support for MSYS2 Git breaks in version 1.103 (The folder currently open doesn't have a git repository)

[https://github.com/microsoft/vscode/issues/260851](https://github.com/microsoft/vscode/issues/260851)

## Summary

Confirmed regression in 1.103 caused by adding `--show-superproject-working-tree` to `rev-parse` (commit 6c8a25ae), which MSYS2 Git handles differently than standard Git for Windows. The issue is on the 'On Deck' milestone with an open fix PR — must stay open.

## Change summary

MSYS2 Git fails to open repositories in VS Code 1.103+ because the `--show-superproject-working-tree` flag causes `rev-parse` to return unexpected output.

## Best solution

Land https://github.com/microsoft/vscode/pull/317226 which adds a fallback when `--show-superproject-working-tree` fails. The fix is in `extensions/git/src/git.ts` around the `getRepositoryDotGit` method — the three-line split on stdout breaks when MSYS2 Git returns different output for the added flag.
