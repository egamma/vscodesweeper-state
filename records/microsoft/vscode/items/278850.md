# Issue #278850 — Prompts and instructions aren't syncing if non-default profile is used

[https://github.com/microsoft/vscode/issues/278850](https://github.com/microsoft/vscode/issues/278850)

## Summary

Valid feature request — prompts sync infrastructure exists (SyncResource.Prompts, useDefaultFlags.prompts) but the Profiles editor UI does not expose the 'Use data from default profile' toggle for prompts, and prompts sync is disabled by default. The gap remains on current main.

## Change summary

Prompts and instructions should be available for syncing and for 'Use data from default profile' in non-default profiles.

## Best solution

Add `ProfileResourceType.Prompts` to the resource list in `src/vs/workbench/contrib/userDataProfile/browser/userDataProfilesEditorModel.ts` (around line 252) and handle it in `userDataProfilesEditor.ts` so users can toggle 'Use data from default profile' for prompts. Also consider enabling prompts sync by default in `userDataSyncEnablementService.ts`.
