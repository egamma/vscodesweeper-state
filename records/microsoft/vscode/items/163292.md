# Issue #163292 — Consider supporting embedded language grammars when using semantic tokens

[https://github.com/microsoft/vscode/issues/163292](https://github.com/microsoft/vscode/issues/163292)

## Summary

Valid, community-backed feature request (27 upvotes, in Backlog). No implementation or progress toward reconciling semantic tokens with injected/embedded grammars found on current main — the gap remains.

## Change summary

When a language provides semantic tokens, injected embedded grammars (e.g. syntax highlighting inside strings) are overridden, with no way for the embedded grammar to layer over or selectively suppress semantic tokens.

## Best solution

Implement a mechanism in the tokenization pipeline (around `src/vs/editor/common/services/semanticTokensProviderStyling.ts` and the grammar injection logic) that allows injected grammars to layer their scopes over semantic tokens for specific token ranges, or lets the semantic token provider mark ranges as deferring to grammar-based tokenization. This is core editor infrastructure, not extension-candidate.
