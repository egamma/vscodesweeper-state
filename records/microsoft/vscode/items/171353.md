# Issue #171353 — Git - Add support for cancelling long commit operation (including its pre-commit script)

[https://github.com/microsoft/vscode/issues/171353](https://github.com/microsoft/vscode/issues/171353)

## Summary

Valid feature request with 69 upvotes, promoted to Backlog. Cancellation for commit operations (including pre-commit hooks) is still not implemented on current main — only sync/pull supports cancellation via `git.supportCancellation`. The gap is real and well-scoped.

## Change summary

Add the ability to cancel a long-running git commit operation (including its pre-commit hooks) from the Source Control panel.

## Best solution

Extend the existing `git.supportCancellation` pattern (currently sync-only) to commit operations in `extensions/git/src/repository.ts`. The `commit()` method at line ~1421 would need to accept a `CancellationToken`, surface a cancellable progress notification, and kill the spawned git process on cancellation — mirroring how sync already works around line 2410.
