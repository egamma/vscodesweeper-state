# Issue #232943 — Inline suggestion toolbar shows wrong keyboard shortcut

[https://github.com/microsoft/vscode/issues/232943](https://github.com/microsoft/vscode/issues/232943)

## Summary

Still present on main: the inline suggestion toolbar resolves and displays the keybinding at render time, but doesn't re-evaluate when context changes (e.g. suggest widget becomes visible), leading to a displayed shortcut that won't actually work.

## Change summary

The inline suggestion toolbar shows a keyboard shortcut (e.g. Tab or Cmd+Enter) that is inactive when the suggest widget is also visible, misleading users.

## Best solution

The toolbar's keybinding display should either reactively update when context keys change (re-call `lookupKeybinding` when `suggestWidgetVisible` changes), or suppress the keybinding display when the effective binding is shadowed. The fix likely lives in `src/vs/editor/contrib/inlineCompletions/browser/hintsWidget/inlineCompletionsHintsWidget.ts` around the `StatusBarViewItem.updateLabel()` and the `createCommandAction` tooltip resolution.
