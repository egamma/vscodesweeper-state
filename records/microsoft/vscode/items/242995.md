# Issue #242995 — [Git extension / Git API] Manage Git API dependency on Git / VS Code extension version

[https://github.com/microsoft/vscode/issues/242995](https://github.com/microsoft/vscode/issues/242995)

## Summary

Niche but valid feature request for formal API versioning in the built-in Git extension. The Git extension already has a basic `getAPI(version)` mechanism (only version 1 exists), but no build-time or runtime compatibility guards tied to `engines.vscode`. No traction after ~500 days, but it sits in the Backlog milestone, so it was accepted at some point — keeping open.

## Change summary

Add a versioning mechanism to the Git extension API so that extension authors get build-time type-checking and runtime safety when targeting older or newer VS Code versions.

## Best solution

If pursued, the Git extension API (`extensions/git/src/api/git.d.ts`) would need versioned type declarations (similar to how the core VS Code API handles `@since` tags and `engines.vscode`), plus runtime guards in `ApiImpl` / `GitExtensionImpl.getAPI()`. Given 0 community upvotes and no maintainer discussion, this likely stays parked unless demand grows — the existing single-version `getAPI(1)` pattern has been sufficient for the ecosystem so far.
