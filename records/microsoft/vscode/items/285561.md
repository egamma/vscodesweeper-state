# Issue #285561 — Scrolling chat can lead to significant lag

[https://github.com/microsoft/vscode/issues/285561](https://github.com/microsoft/vscode/issues/285561)

## Summary

Team-contributor-filed performance bug with trace evidence of 100ms+ main-thread locks during chat scrolling. Several layout-related improvements have landed since (deferred height updates, avoid-relayout-while-hidden, layout-shift fixes), but none directly targets the scroll-induced re-render path described here. Keeping open per contributor rule and because the core issue likely remains.

## Change summary

Scrolling up in the chat view causes significant main-thread blocking (100ms+), possibly from re-rendering or layout thrashing triggered by wheel events over certain chat content.

## Best solution

Profile the chat list scroll path to identify why wheel-up triggers expensive relayout/re-render cycles. The likely hot path is in the chat list renderer's item height recalculation or markdown re-rendering during scroll. Fixes like ff7d3ec85a8 (deferred height updates) and 0fc534223c4 (avoid relayout while hidden) address adjacent problems but not the scroll-specific jank shown in the traces.
