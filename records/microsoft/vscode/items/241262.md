# Issue #241262 — Debug Hover Widget Height Is Computed Inconsistently

[https://github.com/microsoft/vscode/issues/241262](https://github.com/microsoft/vscode/issues/241262)

## Summary

Still reproduces on main: the debug hover's `isUpdatingTree` flag suppresses layout during async `setInput`, but the `.finally()` block that clears it never triggers a compensating layout, so the tree can end up with a stale height.

## Change summary

Debug hover widget sometimes shows an incorrectly sized tree because the layout event fired during the suppressed window is never replayed.

## Best solution

In `src/vs/workbench/contrib/debug/browser/debugHover.ts`, call `this.layoutTreeAndContainer()` inside the `.finally()` block after clearing `isUpdatingTree`, so the tree always gets a correct layout once the async input completes.
