# Issue #255906 — Variable substitution in enum properties of `launch.json` causes un-suppressable warnings

[https://github.com/microsoft/vscode/issues/255906](https://github.com/microsoft/vscode/issues/255906)

## Summary

Still reproduces on main: the debug launch.json schema defines enum-constrained properties (e.g. configurations/type) without accommodating variable substitution patterns like ${command:...}, so the JSON language service produces unsuppressable validation warnings. No fix found in current main.

## Change summary

Using variable substitution (${command:...}, ${input:...}) in enum-constrained launch.json properties produces false JSON schema validation warnings that cannot be suppressed.

## Best solution

The JSON language service or the debug schema registration should recognize variable-substitution patterns (matching `\$\{...\}`) and skip enum validation for those values. Likely fix locations are `src/vs/workbench/contrib/debug/common/debugSchemas.ts` (to allow string patterns alongside enums) or the JSON validation layer in `extensions/json-language-features/` to recognize VS Code variable syntax.
