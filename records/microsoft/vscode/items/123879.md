# Issue #123879 — Debugger: Smart Step Into

[https://github.com/microsoft/vscode/issues/123879](https://github.com/microsoft/vscode/issues/123879)

## Summary

Team-authored feature request with 'On Deck' milestone — hard keep-open. The basic Step Into Target feature (context menu + cursor-position selection) is implemented, but the richer UX proposals (inline decorators on modifier-key hold, source-range highlighting during step-into selection) remain unimplemented.

## Change summary

Provide a richer 'Smart Step Into' UX with inline visual indicators and modifier-key-based target selection, beyond the existing context-menu approach.

## Best solution

The remaining gap is the inline-decorator or source-range-highlight UX that shows step-in targets directly in the editor (proposals 2 and 3 in the issue). The existing `StepIntoTargetsAction` in `src/vs/workbench/contrib/debug/browser/debugEditorActions.ts` already handles positional targets and context-menu fallback — extending it with inline decorations (using `IEditorDecorationsCollection`) when DAP targets include ranges would deliver the requested experience. Depends on debug adapters returning `line`/`column`/`endLine`/`endColumn` in `StepInTarget` responses.
