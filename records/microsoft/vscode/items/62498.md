## Summary

Keep open as an existing keybindings feature request: current main still formats Windows UI labels as Shift+Alt, and there is no verified fixing commit or confirmed canonical duplicate in the available evidence.

## Change summary

The request is to display Windows keybinding labels as Alt+Shift+... instead of Shift+Alt+... in the Keyboard Shortcuts UI and generated shortcut references.

## Best solution

If the team wants this convention change, it belongs in the shared keybinding label formatter and its tests, specifically the modifier ordering in src/vs/base/common/keybindingLabels.ts and the Windows/Linux expectations in src/vs/platform/keybinding/test/common/keybindingLabels.test.ts.
