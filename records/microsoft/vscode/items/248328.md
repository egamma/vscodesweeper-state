# Issue #248328 — Git extension API branch `upstream` is undefined when repo was cloned shallow

[https://github.com/microsoft/vscode/issues/248328](https://github.com/microsoft/vscode/issues/248328)

## Summary

Confirmed bug still present on main: `getBranch()` in `extensions/git/src/git.ts` relies solely on `git for-each-ref` with `%(upstream:*)` format fields, which return empty values in shallow clones even when tracking config exists in `.git/config`. An open community PR (#304980) proposes a `git config` fallback — that PR should be reviewed and landed (or an alternative fix applied).

## Change summary

In shallow clones (`git clone --depth 1`), the Git extension's `branch.upstream` property is always `undefined` because `git for-each-ref` upstream format fields return empty values, breaking status bar info, sync, and downstream extensions.

## Best solution

Review and land (or rework) https://github.com/microsoft/vscode/pull/304980, which adds a `git config --get branch.<name>.remote/merge` fallback in `getBranch()` when `for-each-ref` upstream fields are empty. The fix belongs in `extensions/git/src/git.ts` around line 3199–3209 where the upstream object is constructed.
