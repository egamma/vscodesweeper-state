# Issue #205555 — Add support to show the extension name from which inline suggestion is displayed in hover UI

[https://github.com/microsoft/vscode/issues/205555](https://github.com/microsoft/vscode/issues/205555)

## Summary

Still a valid gap: the classic inline-suggestion hover toolbar (prev/next/count) does not show which extension provided the current suggestion, though the newer inline-edits gutter indicator does display `provider.displayName`. The request remains relevant for users in 'onHover' toolbar mode.

## Change summary

Show the name of the extension that provided the current inline completion in the hover toolbar UI.

## Best solution

Add the provider's `displayName` (already available via `suggestion.source.provider.displayName`) to the `InlineSuggestionHintsContentWidget` in `src/vs/editor/contrib/inlineCompletions/browser/hintsWidget/inlineCompletionsHintsWidget.ts`, similar to how the gutter indicator menu already surfaces it.
