# Issue #11828 — [json] validation doesn't work offline

[https://github.com/microsoft/vscode/issues/11828](https://github.com/microsoft/vscode/issues/11828)

## Summary

Schema caching was partially implemented (conditional GETs, 48-hour cache for schemastore.org), but full offline fallback — serving any previously-cached schema when the network is unreachable — is still missing on current main.

## Change summary

JSON validation fails when offline because VS Code cannot fetch remote JSON schemas, even if they were previously downloaded.

## Best solution

In the `getContent` path of `extensions/json-language-features/client/src/node/jsonClientMain.ts`, add a fallback in the network-error catch block: when the XHR fails due to a network error (no HTTP status), attempt to serve the schema from the existing cache regardless of age or URI pattern. This keeps validation working offline for any schema that was previously fetched.
