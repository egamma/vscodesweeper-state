# Issue #11828 — [json] validation doesn't work offline

[https://github.com/microsoft/vscode/issues/11828](https://github.com/microsoft/vscode/issues/11828)

## Summary

JSON schema caching is now implemented on main via JSONSchemaCache — schemastore.org schemas are served from a local disk cache for up to 2 days without any network request, substantially addressing the offline validation scenario. The original 2016 request is largely fulfilled.

## Change summary

JSON validation fails offline because schemas are fetched from the network with no local caching.

## Best solution

The shipped JSONSchemaCache implementation (extensions/json-language-features/client/src/node/schemaCache.ts) addresses this for schemastore.org schemas with a 2-day TTL. If edge cases remain (non-schemastore URLs, cache expiration beyond 2 days while offline), they could be tracked in a new, more specific issue. This issue can be closed as implemented.
