# Issue #209023 — [json] Provide a way to disable the json-validation contribution point of a specific extension (e.g. ESLint)

[https://github.com/microsoft/vscode/issues/209023](https://github.com/microsoft/vscode/issues/209023)

## Summary

Valid feature request in Backlog with no implementation progress on main. The ability to selectively disable extension-contributed JSON schemas remains unimplemented. Keeping open as an accepted backlog item.

## Change summary

Users cannot disable a specific extension's jsonValidation contribution point without disabling the entire extension.

## Best solution

Add a mechanism (e.g. a setting like `json.schemas.disabled` listing glob+extension pairs, or per-extension contribution toggles) to let users suppress specific jsonValidation contributions. The implementation would likely touch `src/vs/workbench/api/common/jsonValidationExtensionPoint.ts` and the JSON language service schema resolution. Related to https://github.com/microsoft/vscode/issues/74943.
