# Issue #313471 — Prompted for SSH key usage every startup by Github Copilot

[https://github.com/microsoft/vscode/issues/313471](https://github.com/microsoft/vscode/issues/313471)

## Summary

Confirmed in source: the Copilot extension's repoTracker runs `git fetch --dry-run` with `BatchMode=yes` but does not suppress the SSH agent socket, so agents like 1Password still prompt on startup. The issue already carries `chat-ext-issue` and is correctly routed.

## Change summary

Copilot's startup git-fetch triggers SSH agent approval prompts (e.g. 1Password) because SSH_AUTH_SOCK is not suppressed.

## Best solution

In `extensions/copilot/src/platform/workspaceChunkSearch/node/codeSearch/repoTracker.ts` around line 388, add `SSH_AUTH_SOCK: ''` (or unset it) in the env passed to `execAsync` so that SSH agents are not contacted during this non-interactive probe. Alternatively, expose a user setting to disable the startup fetch entirely.
