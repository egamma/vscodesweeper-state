# Issue #164161 — Git - git reset --hard

[https://github.com/microsoft/vscode/issues/164161](https://github.com/microsoft/vscode/issues/164161)

## Summary

Still valid: the git extension has a `reset(treeish, hard)` API internally but never exposes a `git reset --hard` command in the UI. The only caller is `undoCommit` which does a soft reset to HEAD~. The request for a hard-reset GUI command — especially to reconcile force-pushed remote branches — remains unaddressed. Has 21 upvotes (above Backlog threshold) and is in the Backlog milestone.

## Change summary

Add a GUI command for `git reset --hard` so users can abandon local state and align with a force-pushed remote branch without the terminal.

## Best solution

Expose a new `git.resetHard` command in `extensions/git/src/commands.ts` that calls the existing `repository.reset(treeish, true)` with a confirmation dialog (destructive operation). A companion UX entry point (e.g. in the branch history or SCM title menu) would let non-CLI users resolve force-pushed remotes. The plumbing already exists in `extensions/git/src/git.ts:2317`.
