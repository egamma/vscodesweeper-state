# Issue #260035 — Search not working on in-memory file systems

[https://github.com/microsoft/vscode/issues/260035](https://github.com/microsoft/vscode/issues/260035)

## Summary

Still reproduces on main: when the only workspace scheme is a custom FS provider (e.g. `memfs:/`) without a dedicated search provider, `searchWithProviders` awaits a deferred promise that never resolves, hanging the search spinner indefinitely. An open PR (#310776) addresses this.

## Change summary

Search hangs forever on workspaces backed solely by a custom in-memory file system that doesn't register a search provider.

## Best solution

Review and land https://github.com/microsoft/vscode/pull/310776, which skips schemes with an FS provider but no search provider instead of awaiting a never-resolving deferred promise. The fix is localized to `searchWithProviders` in `searchService.ts`.
