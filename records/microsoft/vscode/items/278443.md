# Issue #278443 — linux mint issue

[https://github.com/microsoft/vscode/issues/278443](https://github.com/microsoft/vscode/issues/278443)

## Summary

This is a specific instance of the upstream Electron safeStorage detection problem already tracked by issue #185212. Linux Mint (Cinnamon) is misidentified as GNOME because Electron's safeStorage backend selection picks a gnome-* backend, and the VS Code notification then says "GNOME environment" — the root cause and fix both live in the canonical upstream issue.

## Change summary

On Linux Mint 22.2 (Cinnamon desktop), VS Code shows a recurring and misleading 'GNOME environment' keyring notification on every startup.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/185212 — resolving the Electron safeStorage detection for non-standard GNOME-derivative desktops (including Cinnamon) will fix this report. No separate VS Code change is needed beyond what that issue tracks.

- **canonical issue:** https://github.com/microsoft/vscode/issues/185212 is open with labels upstream, electron, keychain, and tracks exactly this: Electron's safeStorage API failing to detect or use the correct keyring when the desktop environment is not standard GNOME or KDE.
