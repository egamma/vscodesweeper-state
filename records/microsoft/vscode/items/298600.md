# Issue #298600 — Improve multiline text support in askQuestions tool in Copilot Chat

[https://github.com/microsoft/vscode/issues/298600](https://github.com/microsoft/vscode/issues/298600)

## Summary

Valid, well-documented feature request with a prototype PR. The text-question input still uses a single-line InputBox on current main, and the freeform textarea still has overflow-y: hidden and lacks Ctrl+Enter newline support. No duplicate found. Routing to the chat area.

## Change summary

The askQuestions tool's text-type questions use a single-line InputBox that flattens multiline paste, and the freeform textarea on select-type questions lacks Ctrl+Enter newline insertion and mouse wheel scrolling.

## Best solution

Replace the single-line InputBox in `renderTextInput` with a textarea (or a multiline-capable input) matching the existing freeform textarea pattern, fix the Ctrl+Enter key handler to allow newline insertion, and change `overflow-y: hidden` to `overflow-y: auto` with wheel-event isolation on the textarea in `chatQuestionCarousel.css`. The author's prototype at heguro/vscode#3 covers all three issues and could serve as a starting point.
