# Issue #305699 — Git extension throws "pathspec is beyond a symbolic link" error with pnpm workspace symlinks in node_modules

[https://github.com/microsoft/vscode/issues/305699](https://github.com/microsoft/vscode/issues/305699)

## Summary

Real bug: the git extension passes paths traversing symlinked parent directories (e.g. pnpm workspace symlinks in node_modules) to git commands like `check-ignore`, which fails with 'pathspec is beyond a symbolic link'. The quickDiffProvider checks if the file itself is a symlink but not if it's reached through a symlinked ancestor directory.

## Change summary

The built-in Git extension throws repeated errors when files inside symlinked directories (pnpm workspace node_modules) are encountered during operations like check-ignore or quick diff.

## Best solution

Before passing a file path to git commands that don't follow symlinks (like `check-ignore`), check whether any ancestor directory component is a symlink. The check in `quickDiffProvider.ts` line 37 (and similar paths in `decorationProvider.ts`) should be extended to detect symlinks in the path ancestry, not just on the file itself. Alternatively, catch and gracefully handle the 'beyond a symbolic link' stderr from git rather than surfacing it as an error.
