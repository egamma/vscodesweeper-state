# Issue #247611 — Image hover isn't buttery smooth

[https://github.com/microsoft/vscode/issues/247611](https://github.com/microsoft/vscode/issues/247611)

## Summary

Legitimate visual glitch where the hover pointer/arrow renders before the hover content is ready, acknowledged by a team member (justschen) who noted they'd investigate in a future iteration. No fix found on current main; the issue remains valid.

## Change summary

The hover widget's pointer (beak) appears slightly before the hover content finishes rendering when hovering over chat attachment pills, making the hover feel janky.

## Best solution

The timing mismatch likely lives in `src/vs/platform/hover/browser/hoverService.ts` or the hover widget rendering pipeline in `src/vs/base/browser/ui/hover/` — the pointer and content container should appear atomically. justschen's comment points to `setupDelayedHover` as the entry point; the fix would ensure the hover widget isn't shown until its content (including any async image load) is ready.
