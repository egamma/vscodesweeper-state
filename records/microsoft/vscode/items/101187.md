# Issue #101187 — app.clearRecentDocuments does not clear files from the recent entries list

[https://github.com/microsoft/vscode/issues/101187](https://github.com/microsoft/vscode/issues/101187)

## Summary

Confirmed upstream Electron bug: VS Code correctly calls `app.clearRecentDocuments()` when the user clears recent entries (line 190 of workspacesHistoryMainService.ts), but Electron does not honor it for files on the Windows Start menu. The issue is already correctly labeled `upstream` and `electron` with `confirmed`. Keeping open as the root cause is in the Electron dependency.

## Change summary

Clearing recently opened files in VS Code does not remove them from the Windows Start menu's recent entries list.

## Best solution

This requires a fix in Electron's `app.clearRecentDocuments()` implementation on Windows to properly clear files from the Start menu recent list. VS Code's code at `src/vs/platform/workspaces/electron-main/workspacesHistoryMainService.ts:190` is correct — it calls the API but Electron doesn't deliver. Track upstream in Electron and pick up the fix with a future Electron update.
