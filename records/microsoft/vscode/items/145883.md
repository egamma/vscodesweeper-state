# Issue #145883 — [html] don't show closing element proposal as first entry if element is already closed later

[https://github.com/microsoft/vscode/issues/145883](https://github.com/microsoft/vscode/issues/145883)

## Summary

Team-authored feature request (isidorn is a MEMBER) with maintainer acknowledgment from aeschli — must remain open per policy. No evidence of implementation on main.

## Change summary

HTML completion should not prioritize the closing-tag suggestion when the element is already closed later in the file.

## Best solution

In the HTML language service completion provider, look ahead in the document after the cursor position to detect whether the current element is already closed; if so, sort the closing-tag completion lower rather than presenting it first. This likely lives in vscode-html-languageservice (upstream dependency) rather than in the built-in extension itself.
