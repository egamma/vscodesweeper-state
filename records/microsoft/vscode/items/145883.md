# Issue #145883 — [html] don't show closing element proposal as first entry if element is already closed later

## Summary

Team-authored feature request for smarter HTML close-tag completion sorting. Must remain open for explicit maintainer judgment.

## Change summary

HTML completion should not show the closing tag as the first suggestion when the element is already closed later in the file.

## Best solution

In the HTML language service completion logic, look ahead in the document to check whether the current element is already closed; if so, sort the closing-tag suggestion lower rather than offering it first. The fix likely lives in the `html-language-features` extension or the underlying `vscode-html-languageservice` library.
