# Issue #161331 — Code action request from problems view includes additional diagnostics

[https://github.com/microsoft/vscode/issues/161331](https://github.com/microsoft/vscode/issues/161331)

## Summary

Still reproduces on main: the Problems view's code action request passes only the marker's range to `getCodeActions`, not the specific diagnostic, so all diagnostics sharing that range are included in the LSP context. An open community PR (#307596) is actively addressing this — keep open and let it land.

## Change summary

Code action requests from the Problems view include all diagnostics in the marker's range instead of only the selected diagnostic.

## Best solution

Review and land the open PR https://github.com/microsoft/vscode/pull/307596, which scopes the code action diagnostics to the specific marker. The fix likely needs changes in `src/vs/workbench/contrib/markers/browser/markersTreeViewer.ts` (to pass the specific marker diagnostic) and possibly `src/vs/editor/contrib/codeAction/browser/codeAction.ts` (to thread it into `CodeActionContext.diagnostics`).
