# Issue #42758 — [json] improve property suggestions with oneOf

[https://github.com/microsoft/vscode/issues/42758](https://github.com/microsoft/vscode/issues/42758)

## Summary

Still an open gap — the JSON language service excludes oneOf subschemas with unmet `required` fields from completions. No fix has landed in either the VS Code repo or the upstream vscode-json-languageservice package. With 15 upvotes it's close to (but below) the 20-vote Backlog promotion threshold.

## Change summary

JSON schema completions should suggest properties from all oneOf subschemas, not only those whose required fields are already satisfied.

## Best solution

The fix belongs in `microsoft/vscode-json-languageservice` (the `validate`/match logic in `jsonParser.ts` that gates subschema matching on `required` satisfaction). The completion provider should treat all oneOf branches as candidates when the object is incomplete, possibly with a relaxed matching mode that ignores missing required fields during suggestion collection.
