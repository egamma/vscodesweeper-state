# Issue #42758 — [json] improve property suggestions with oneOf

[https://github.com/microsoft/vscode/issues/42758](https://github.com/microsoft/vscode/issues/42758)

## Summary

This feature request for improved oneOf property suggestions remains valid. The underlying issue in vscode-json-languageservice's schema matching logic (validation-driven matching excludes subschemas with unmet required fields) has not been addressed on current main.

## Change summary

JSON completions should suggest properties from all oneOf subschemas, not just those whose required fields are already satisfied.

## Best solution

The fix belongs in the upstream vscode-json-languageservice repo (microsoft/vscode-json-languageservice). The `getMatchingSchemas` logic in the parser needs to be relaxed for completion purposes — either by splitting validation from schema matching, or by adding a completion-specific mode that includes subschemas regardless of required-field validation failures.
