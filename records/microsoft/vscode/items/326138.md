# Issue #326138 — "Run C/C++ File" fails in PowerShell if path has spaces (missing & call operator)

[https://github.com/microsoft/vscode/issues/326138](https://github.com/microsoft/vscode/issues/326138)

## Summary

The "Run C/C++ File" button is provided by the ms-vscode.cpptools extension, which lives outside the VS Code core repo. The command construction (missing `&` call operator for PowerShell) is entirely owned by that extension, not by VS Code's terminal or task infrastructure.

## Change summary

The C/C++ extension's "Run C/C++ File" command omits the PowerShell `&` call operator when the compiled binary path contains spaces, causing PowerShell to echo the path as a string instead of executing it.

## Best solution

File this against the C/C++ extension's tracker at https://github.com/microsoft/vscode-cpptools — the fix belongs in the extension's task/command construction logic, not in VS Code core.
