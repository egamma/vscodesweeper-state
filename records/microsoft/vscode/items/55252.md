# Issue #55252 — [folding] provide non-selection aware fold level command 

[https://github.com/microsoft/vscode/issues/55252](https://github.com/microsoft/vscode/issues/55252)

## Summary

Still a valid open feature request. The related PR #117991 was closed without merging, and no setting like `editor.foldLevelCommandsKeepSelectionVisible` exists on current main. With only 3 upvotes and no recent traction, it stays open as a backlog feature request.

## Change summary

Fold Level X commands exclude the region at the cursor position; users want an option to fold all regions of that level regardless of cursor.

## Best solution

Add an `editor.foldLevelCommandsKeepSelectionVisible` boolean setting (defaulting to `true` for backward compat) that, when disabled, lets Fold Level X fold the cursor's region too. The fix belongs in `src/vs/editor/contrib/folding/browser/folding.ts` in the fold-level action handlers.
