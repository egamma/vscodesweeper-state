# Issue #55252 — [folding] provide non-selection aware fold level command 

[https://github.com/microsoft/vscode/issues/55252](https://github.com/microsoft/vscode/issues/55252)

## Summary

The feature request is still unimplemented on main. The `setCollapseStateAtLevel` function in `foldingModel.ts` explicitly filters out regions containing cursor lines via `blockedLineNumbers`. No setting or alternative command exists to bypass this behavior.

## Change summary

Request a setting or command variant that folds ALL regions at a given level, including the one containing the cursor.

## Best solution

Add a boolean setting (e.g. `editor.foldLevelCommandsKeepSelectionVisible`) that, when false, passes an empty `blockedLineNumbers` array to `setCollapseStateAtLevel` in `FoldLevelAction.invoke()`. Alternatively, register a parallel set of commands ('Fold Level X (All)') that always pass `[]`. The change is localized to `src/vs/editor/contrib/folding/browser/folding.ts` line ~1043.
