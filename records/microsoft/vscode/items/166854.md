# Issue #166854 — Drag'n'drop code in Debug Watch list

[https://github.com/microsoft/vscode/issues/166854](https://github.com/microsoft/vscode/issues/166854)

## Summary

The feature is still unimplemented on main — the watch view's DnD handler only accepts internal reorder drags and rejects external text drops. Two related issues (#195548, #195547) were closed as duplicates of this one, confirming it as the canonical tracker. With 6 upvotes it hasn't hit the 20-upvote backlog threshold, but it's already in the Backlog milestone and has a concrete implementation sketch from a contributor — keeping open.

## Change summary

Allow dragging selected text from the editor (or other sources) into the Debug Watch panel to create a new watch expression.

## Best solution

Extend `WatchExpressionsDragAndDrop.onDragOver` and `drop` in `src/vs/workbench/contrib/debug/browser/watchExpressionsView.ts` to accept `text/plain` from the native `DataTransfer`, and update the editor's `DragAndDropController` (`src/vs/editor/contrib/dnd/browser/dnd.ts`) to set `text/plain` on the drag event's `dataTransfer` for selected text. The implementation sketch in comment #3 by @mjbvz outlines this approach.
