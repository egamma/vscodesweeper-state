# Issue #298865 — The 'Editor: Wrapping Indent' setting fails to take effect, possibly due to config synchronization issues in the remote environment.

[https://github.com/microsoft/vscode/issues/298865](https://github.com/microsoft/vscode/issues/298865)

## Summary

Confirmed source-reproducible on main: when accessibility support is enabled (the reporter has Screen Reader: yes), the editor forces wrappingIndent to None regardless of the user's setting. An open PR (#316384) directly addresses this.

## Change summary

The editor.wrappingIndent setting is silently overridden to 'none' when accessibility support is detected, ignoring the user's configured value.

## Best solution

Remove or relax the accessibility override in src/vs/editor/common/config/editorOptions.ts (WrappingIndentOption.compute) so the user's configured wrappingIndent is respected even with a screen reader active. PR https://github.com/microsoft/vscode/pull/316384 implements exactly this fix.
