# Issue #323202 — terminal.integrated.cwd:${fileDirname} confuses copilot

[https://github.com/microsoft/vscode/issues/323202](https://github.com/microsoft/vscode/issues/323202)

## Summary

Confirmed on current main: the copilot terminal creator does not set `ignoreConfigurationCwd: true` on the shell launch config, so the user's `terminal.integrated.cwd` setting (e.g. `${fileDirname}`) is applied to agent terminals, causing wrong cwd and broken relative paths.

## Change summary

When `terminal.integrated.cwd` is set to `${fileDirname}`, copilot agent terminals start in the wrong directory, causing the agent to pass incorrect relative paths to commands.

## Best solution

Set `ignoreConfigurationCwd: true` on the `IShellLaunchConfig` in `ToolTerminalCreator._createCopilotTerminal()` so agent terminals always start at the workspace root (or session working directory) regardless of the user's `terminal.integrated.cwd` setting. The cwd for the agent terminal is already determined separately in `runInTerminalTool.ts` lines 940–954 via `getCwdResource()` / session workingDirectory / workspace root — the configuration cwd should not override that.

## Evidence

- **root cause in source:** src/vs/workbench/contrib/terminalContrib/chatAgentTools/browser/toolTerminalCreator.ts lines 193–213: the `IShellLaunchConfig` sets `icon`, `hideFromUser`, `forcePersist`, and `env`, but neither `cwd` nor `ignoreConfigurationCwd`. This means the terminal falls through to `getCwd()` in src/vs/workbench/contrib/terminal/common/terminalEnvironment.ts (line 202) which applies `terminal.integrated.cwd` when `ignoreConfigurationCwd` is falsy.
- **cwd resolution path:** src/vs/workbench/contrib/terminal/common/terminalEnvironment.ts lines 186–221: `getCwd()` checks `shell.cwd` first, then if `!shell.ignoreConfigurationCwd && customCwd`, it resolves the config value (which includes variable substitution like `${fileDirname}`). Without `ignoreConfigurationCwd: true`, the user setting wins over the intended workspace root.
- **agent cwd is computed separately:** src/vs/workbench/contrib/terminalContrib/chatAgentTools/browser/tools/runInTerminalTool.ts lines 940–954: the run_in_terminal tool computes its own cwd from `instance.getCwdResource()`, session workingDirectory, or workspace root — but this is the *reported* cwd, not the *initial* cwd. The terminal's actual initial cwd comes from the shell launch config, creating a mismatch.
- **user report confirms impact:** Both the issue body and Comment 1 confirm consistent breakage: the agent passes wrong relative paths, and even when the agent tries to use `cd`, the run_in_terminal command simplifier strips it away, compounding the problem.
