# Issue #168195 — Remote SSH incorrectly strips empty but defined environment variables from shell environment

[https://github.com/microsoft/vscode/issues/168195](https://github.com/microsoft/vscode/issues/168195)

## Summary

This is a duplicate of #174330, which was fixed in commit b907d7875a8 (shipped in 1.77.0). The root cause was `mergeNonNullKeys` using a truthy check that stripped empty-string environment variables; the fix changed it to an explicit null/undefined check.

## Change summary

Environment variables defined as empty strings (e.g. `export TESTVAR3` or `export TESTVAR3=""`) are stripped from the terminal environment when spawning shells.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/174330 which is already closed-fixed. The fix in commit b907d7875a8 shipped in 1.77.0 and resolves this exact issue.
