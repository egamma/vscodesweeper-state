# Issue #267568 — VSCode Integrated Terminal IME Issue

[https://github.com/microsoft/vscode/issues/267568](https://github.com/microsoft/vscode/issues/267568)

## Summary

Already properly triaged as an upstream xterm.js issue. The terminal bypasses IME composition for certain Korean one-hand Hangul remapped keys; the root cause is in xterm.js's key event handling, not VS Code core. Maintainers have linked it to the upstream tracker and labeled it accordingly.

## Change summary

VS Code's integrated terminal ignores IME-composed output for remapped keys in macOS one-hand Hangul mode, showing raw Latin characters instead of the expected Hangul characters.

## Best solution

The fix belongs in xterm.js (https://github.com/xtermjs/xterm.js/issues/3836 or a related issue there). Once xterm.js resolves the IME composition bypass for these remapped keys, VS Code picks up the fix via its xterm.js dependency update. No VS Code core change needed.
