# Issue #95992 — Can't click to set/clear breakpoint when a gutter decoration exists

[https://github.com/microsoft/vscode/issues/95992](https://github.com/microsoft/vscode/issues/95992)

## Summary

The visual overlap was fixed in 1.78 via dedicated glyph margin lanes, but the click-to-set-breakpoint interaction is still intentionally blocked by `marginFreeFromNonDebugDecorations` when extension gutter decorations are present. The issue remains partially valid as a feature request for improved click behavior.

## Change summary

Allow clicking in the glyph margin to set a breakpoint even when an extension-contributed gutter decoration occupies that line.

## Best solution

Modify the `marginFreeFromNonDebugDecorations` check in `breakpointEditorContribution.ts` to allow breakpoint-setting clicks even when extension decorations exist (since decorations now render in separate lanes). The dedicated-lane system from 1.78 makes this safe — both icons coexist visually, so the click should work too.
