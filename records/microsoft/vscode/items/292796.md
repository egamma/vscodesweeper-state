## Summary

Team-authored issue (MEMBER) — must stay open for maintainer judgment. The bug involves repeated GHE auth prompts when connected to a non-GHE GitHub remote repo. A team member (alexr00) noted the modal may originate from GitHub Repositories rather than GHPR, and ownership between the in-repo `github-authentication` extension and the external GitHub Repositories/GHPR extensions is unclear.

## Change summary

Users with `github-enterprise.uri` configured get repeated GHE sign-in prompts when opening a private GitHub.com repo via the GitHub Repositories extension.

## Best solution

Clarify whether the spurious auth prompt originates in the in-repo `github-authentication` extension (extensions/github-authentication/) or in the external GitHub Repositories / GitHub Pull Requests extensions. If it's in `github-authentication`, the fix likely involves better scoping of when GHE auth is requested based on the active remote's host. If it's external, the issue should be moved to the appropriate tracker (microsoft/vscode-pull-request-github or microsoft/vscode-remote-repositories-github).
