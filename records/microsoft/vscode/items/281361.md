# Issue #281361 — Mismatch in background agent changed file count after migrating changes to worktree

[https://github.com/microsoft/vscode/issues/281361](https://github.com/microsoft/vscode/issues/281361)

## Summary

Team-authored bug report about inconsistent file-change counts between the Recent Sessions view and the chat output/input areas when migrating uncommitted changes to a background agent worktree. No fix found on main; keeping open per team-author policy.

## Change summary

When uncommitted local changes are migrated to a background agent's worktree, the chat output pill and changed-file count above the chat input box omit the migrated file, showing 1 instead of 2.

## Best solution

Ensure the chat-side file-change counters include files migrated to the worktree (not just files the agent itself modified), so all surfaces report the same total. The fix likely lives in the Copilot extension's session/worktree change-tracking code under `extensions/copilot/`.
