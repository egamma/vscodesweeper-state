# Issue #299493 — Source Control pannel says GIT while on SVN

[https://github.com/microsoft/vscode/issues/299493](https://github.com/microsoft/vscode/issues/299493)

## Summary

The "Scanning folder for Git repositories..." message is expected behavior from VS Code's built-in git extension, which always scans opened folders for git repos regardless of what other SCM systems are in use. SVN support is provided by the third-party svn-scm extension, not by VS Code core. This is as-designed.

## Change summary

Source Control panel shows a "Scanning for Git repositories" message when opening an SVN-only directory.

## Best solution

Keep the current behavior. The built-in git extension scans all opened folders for git repositories — this is independent of any other SCM provider. The message is accurate: it IS scanning for git repos. If the user wants to suppress this, they can set `git.autoRepositoryDetection` to `false`. Any SVN-specific messaging would be the responsibility of the svn-scm extension.
