# Issue #50836 — Support customisable alias for commands in command palette

[https://github.com/microsoft/vscode/issues/50836](https://github.com/microsoft/vscode/issues/50836)

## Summary

Valid backlog feature request with 34 upvotes. User-customizable command aliases are not implemented on main — the existing alias support is only for localized command titles (original vs. translated). The community PR #139670 was closed without merge, and an extension exists but the request is for core support.

## Change summary

Allow users to define custom aliases for commands so they appear and are searchable in the command palette under user-chosen names.

## Best solution

Either implement a `workbench.commandPalette.commandAliases` user setting that feeds into the quick-access command picker (touching `src/vs/workbench/contrib/quickaccess/browser/commandsQuickAccess.ts`), or formally decide this belongs in extension-land given the existing extension. The infrastructure for alias-based searching already exists for localization aliases — extending it to user-defined aliases would be a moderate addition.

## Vision fit

- **Fit:** unclear
- **Reason:** User-customizable command aliases are a power-user workflow optimization; the vision values lightweight-and-fast but also CLI/UI co-existence. It's not clearly aligned with current priorities nor rejected by guardrails.
