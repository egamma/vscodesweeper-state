# Issue #324907 — Copilot Agent built-in list_dir returns unbounded output for large directories and freezes/poisons the chat session

[https://github.com/microsoft/vscode/issues/324907](https://github.com/microsoft/vscode/issues/324907)

## Summary

Confirmed from source: the built-in `list_dir` tool reads the entire directory with no entry cap and passes all results into the chat context, which can freeze the session for directories with tens of thousands of files. No fix has been applied on current main.

## Change summary

The built-in `list_dir` tool returns unbounded directory listings for very large directories, injecting massive content into the chat context and freezing the session.

## Best solution

Add an entry limit (e.g. 1000–2000 entries) in the `ListDirTool.invoke` method in `extensions/copilot/src/extension/tools/node/listDirTool.tsx`, truncating the `contents` array from `readDirectory` and appending a note like '… and N more entries'. The `ListDirResult` prompt element could also use its `PromptSizing` budget to cap output. This is a small, localized fix.
