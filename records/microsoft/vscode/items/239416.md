# Issue #239416 — Git - submodule displayed with a file icon in the source control panel

[https://github.com/microsoft/vscode/issues/239416](https://github.com/microsoft/vscode/issues/239416)

## Summary

Still present on main: when a submodule appears as a modified resource in the parent repo's SCM change list, it shows with a generic file icon instead of a folder icon because the SourceControlResourceState's resourceUri is resolved by the file icon theme, which doesn't know the path is a directory. Recent work (commit 6c8a25ae475) added a `_repositoryKind` field and fixed the source control provider header icon but didn't address individual resource item icons.

## Change summary

Git submodule entries in the Source Control panel's change list display a file icon instead of a folder icon.

## Best solution

In `extensions/git/src/repository.ts`, the `Resource` class's `decorations` getter could explicitly set a folder-style `ThemeIcon` when `_repositoryKind === 'submodule'`, overriding the default file-icon-theme resolution for these directory-type entries. The maintainer for the `git` area (@lszomoru) is already aware of the report and may have a preferred approach.
