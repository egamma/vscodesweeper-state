# Issue #120460 — [Gutter] Git Decorations bounding box and appearing

[https://github.com/microsoft/vscode/issues/120460](https://github.com/microsoft/vscode/issues/120460)

## Summary

Keep open: current main still has an actionable SCM quick diff/editor gutter interaction around hover visibility and folding-adjacent hit testing. I did not find a concrete fixing commit for this issue, and the current code still contains a collision-specific TODO.

## Change summary

Git/SCM dirty diff gutter decorations can appear or open when the user is aiming for folding controls.

## Best solution

Adjust the SCM quick diff gutter behavior so hover visibility is scoped to the dirty diff lane rather than the whole editor margin, and ensure gutter click handling cannot steal folding-control clicks. This likely belongs around `src/vs/workbench/contrib/scm/browser/quickDiffDecorator.ts`, `quickDiffWidget.ts`, and editor gutter layout/hit testing.
