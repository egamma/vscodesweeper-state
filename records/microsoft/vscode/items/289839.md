# Issue #289839 — Find control is cut off and almost invisible when Chat has been opened in full screen mode and then closed

[https://github.com/microsoft/vscode/issues/289839](https://github.com/microsoft/vscode/issues/289839)

## Summary

Still reproduces on main. The find widget's user-resized inline `width` is not clamped when the editor shrinks (side bar maximized) and then restores, leaving the widget overflowing its container.

## Change summary

After resizing the find widget wider, maximizing the secondary side bar, and closing it, the find widget is cut off because its explicit inline width exceeds the editor bounds.

## Best solution

In `FindWidget._layoutWidgetInEditor` (around line 765–773 of `src/vs/editor/contrib/find/browser/findWidget.ts`), when the widget was user-resized, clamp `this._domNode.style.width` to the computed `maxWidth` — not just set `maxWidth` via CSS, since the explicit inline `width` overrides it.
