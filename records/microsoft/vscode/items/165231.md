# Issue #165231 — Use the Flyweight pattern to encode tokens

[https://github.com/microsoft/vscode/issues/165231](https://github.com/microsoft/vscode/issues/165231)

## Summary

Team-authored feature request proposing a flyweight encoding for tokenization metadata to reduce memory usage. Still valid — current main still uses the 4-byte-per-token encoding in the token store.

## Change summary

Use a flyweight/dictionary pattern to encode token metadata, reducing per-token storage from 4 bytes to ~1 byte by indexing into a small dictionary of unique token descriptors.

## Best solution

Implement a dictionary-based token encoding in the tokenization layer (likely `src/vs/editor/common/tokens/`) that maps the small set of unique token metadata values to compact indices, with a fallback for files exceeding the dictionary size. This is a performance/memory optimization for the editor core.
