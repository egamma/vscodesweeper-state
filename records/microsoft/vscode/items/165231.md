## Change summary

Use a flyweight/dictionary encoding for token metadata to reduce memory overhead of stored editor tokens.

## Best solution

Continue tracking this in the tokenization area until the editor team decides whether to prototype a compact token representation around LineTokens and ContiguousMultilineTokens, with measurements for memory savings and compatibility with existing token readers/editing code.
