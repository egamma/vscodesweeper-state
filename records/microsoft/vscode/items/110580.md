# Issue #110580 — [rename on type] stops when composing characters

[https://github.com/microsoft/vscode/issues/110580](https://github.com/microsoft/vscode/issues/110580)

## Summary

Still reproduces on main: the linked editing contribution (editor-synced-region) has no composition awareness — during IME composition, intermediate characters fail the wordPattern check in `_syncRanges`, causing `clearRanges()` to stop the session prematurely.

## Change summary

On-type rename (linked editing / synced regions) stops when composing characters via IME because intermediate composition text breaks the word pattern check.

## Best solution

In `src/vs/editor/contrib/linkedEditing/browser/linkedEditing.ts`, track composition state using the editor's `onDidCompositionStart`/`onDidCompositionEnd` events and suppress the word-pattern validation (or defer `_syncRanges`) while composing, similar to how `suggestModel.ts` handles composition.
