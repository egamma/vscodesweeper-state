# Issue #110580 — [rename on type] stops when composing characters

[https://github.com/microsoft/vscode/issues/110580](https://github.com/microsoft/vscode/issues/110580)

## Summary

Still reproduces on main: the linked editing contribution in `linkedEditing.ts` has no composition handling — intermediate IME values fail the word-pattern check in `_syncRanges`, clearing ranges and stopping the session.

## Change summary

Linked editing (rename on type) stops when the user composes characters via IME (e.g., macOS dead-key ¨ + u → ü).

## Best solution

Guard `_syncRanges` word-pattern validation during composition by listening to `editor.onDidCompositionStart`/`onDidCompositionEnd` and deferring sync until composition ends, similar to how other editor contributions handle IME.
