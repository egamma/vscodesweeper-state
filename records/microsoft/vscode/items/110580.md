# Issue #110580 — [rename on type] stops when composing characters

[https://github.com/microsoft/vscode/issues/110580](https://github.com/microsoft/vscode/issues/110580)

## Summary

Still reproduces on main: linkedEditing.ts never listens for composition events, so mid-composition content changes trigger the word-pattern check which clears the synced ranges.

## Change summary

Linked editing (rename-on-type) aborts when a user composes characters (e.g. ü via alt+u then u on Mac) because the intermediate composition value fails the word-pattern match.

## Best solution

In `src/vs/editor/contrib/linkedEditing/browser/linkedEditing.ts`, track composition state via `onDidCompositionStart`/`onDidCompositionEnd` and suppress `_syncRanges` (or at least the word-pattern bail-out in `clearRanges`) while composing. Resume sync after composition ends.
