# Issue #18232 — Allow to cycle through previous typed values in quick input picker

[https://github.com/microsoft/vscode/issues/18232](https://github.com/microsoft/vscode/issues/18232)

## Summary

This is the canonical feature request for input history cycling in quick input pickers. Multiple duplicates (#24180, #69843, #90114, #272846) were closed pointing here. The feature remains unimplemented — the quick input box (`quickInputBox.ts`) has no history navigation, unlike the `HistoryInputBox` used in find/replace.

## Change summary

Allow cycling through previously typed values in quick input pickers (e.g., Ctrl+G, Ctrl+P) using Alt+Up/Down, similar to search & replace history.

## Best solution

Integrate a `HistoryNavigator` (already used in `HistoryInputBox`) into the quick input widget at `src/vs/platform/quickinput/browser/quickInputBox.ts`, persisting typed values per-provider and allowing Alt+Up/Down navigation. This is the canonical tracker — keep it open for the community/backlog process.
