# Issue #294610 — Editor scroll happening too near the cursor initial position when moving text selection

[https://github.com/microsoft/vscode/issues/294610](https://github.com/microsoft/vscode/issues/294610)

## Summary

Still present on main: drag-scrolling triggers when the mouse exits the editor content area, with no margin/threshold. Larger font sizes reduce visible lines, making the boundary closer to the cursor and triggering scroll sooner than expected.

## Change summary

Editor drag-scroll triggers too eagerly with larger font sizes because the edge-detection uses the raw editor content boundary with no configurable threshold margin.

## Best solution

Add a small dead-zone threshold (in pixels or line-height units) before triggering drag-scroll in `_getPositionOutsideEditor` in `src/vs/editor/browser/controller/mouseHandler.ts`, so the scroll doesn't start the instant the cursor exits the content area. This could be a fixed margin or proportional to line height.
