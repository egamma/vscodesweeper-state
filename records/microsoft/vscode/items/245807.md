# Issue #245807 — `"editor.unicodeHighlight.allowedLocales": {"de": true }` should stop highlighting of Germany umlauts

[https://github.com/microsoft/vscode/issues/245807](https://github.com/microsoft/vscode/issues/245807)

## Summary

Confirmed bug on current main: `allowedLocales` only suppresses ambiguous-character highlighting, not non-basic-ASCII highlighting. German umlauts and € aren't in the ambiguous data at all — they're flagged under the `nonBasicASCII` path, which completely ignores locales.

## Change summary

Setting `editor.unicodeHighlight.allowedLocales` to include `"de": true` does not suppress highlighting of German umlauts (ä, ö, ü) or € in untrusted workspaces, despite the setting description implying it should.

## Best solution

Extend `shouldHighlightNonBasicASCII` in `src/vs/editor/common/services/unicodeTextModelHighlighter.ts` to consult a locale-aware allowlist of common characters (umlauts for `de`, accented chars for `fr`, etc.), or make the `allowedLocales` mechanism generate locale-specific `allowedCodePoints` that feed into the nonBasicASCII check. The locale→character mapping data would need to be added (possibly in `src/vs/base/common/strings.ts` alongside the existing confusable data).
