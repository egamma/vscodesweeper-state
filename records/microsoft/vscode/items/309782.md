# Issue #309782 — Infinite marker re-registration loop in clearAllMarkers causes UnresponsiveSampleError

[https://github.com/microsoft/vscode/issues/309782](https://github.com/microsoft/vscode/issues/309782)

## Summary

Confirmed on current main: the `setupRecreatingStartMarker` pattern in `strategyHelpers.ts` re-registers a marker inside an `onDispose` handler, which when triggered by xterm.js's `clearAllMarkers` (live-length iteration), creates an infinite loop that freezes the renderer. Both the VS Code listener and the xterm.js loop are still unfixed.

## Change summary

Exiting an alternate-screen-buffer program (e.g. `less`, `git log`) freezes the terminal in an infinite marker re-registration loop, killing the window.

## Best solution

Fix on the VS Code side in `src/vs/workbench/contrib/terminalContrib/chatAgentTools/browser/executeStrategy/strategyHelpers.ts`: defer the `recreateStartMarker` call (e.g. via `queueMicrotask` or a flag check) so it does not re-enter `addMarker` synchronously during xterm.js's `clearAllMarkers` loop. Additionally, file an upstream issue on xtermjs/xterm.js to snapshot `this.markers.length` before iterating in `clearAllMarkers` so that externally-added markers during disposal don't extend the loop.
