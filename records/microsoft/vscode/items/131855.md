# Issue #131855 — ctrl+z undo both auto-indentation and default behavior

[https://github.com/microsoft/vscode/issues/131855](https://github.com/microsoft/vscode/issues/131855)

## Summary

Still reproducible on main: TabOperation replaces the entire whitespace-only line with computed auto-indent as a single ReplaceCommand, so undo reverts to the empty line rather than to a simple-tab intermediate state. Related to but distinct from #30406 (paste variant). Labels are correct.

## Change summary

Pressing Tab on an empty line triggers auto-indentation; undoing removes all indentation instead of reverting to just a single tab's worth.

## Best solution

Split the Tab auto-indent operation into two undo-able steps: first a normal tab insertion, then the auto-indent adjustment — or insert an undo stop between them — so that Ctrl+Z reverts to the simple-tab state. The fix lives in `TabOperation.getCommands` in `src/vs/editor/common/cursor/cursorTypeEditOperations.ts` (lines 783–789).
