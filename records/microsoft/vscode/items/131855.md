## Change summary

Pressing Tab on an empty auto-indented JavaScript line jumps to the smart indentation level, but one undo returns to column 1 instead of leaving the normal single-tab indentation.

## Best solution

Editor core should decide the intended undo granularity for smart Tab indentation and add a focused regression test around `TabOperation`: either undo the smart auto-indent back to a normal tab stop, as requested here, or explicitly document/close as designed after maintainer review.
