# Issue #131855 — ctrl+z undo both auto-indentation and default behavior

[https://github.com/microsoft/vscode/issues/131855](https://github.com/microsoft/vscode/issues/131855)

## Summary

Keep open: the report describes a concrete editor undo/auto-indentation behavior, and the only maintainer signal is a related issue link rather than a confirmed duplicate or fix. Current editor code still has a distinct TabOperation path that can replace the whole whitespace-only line with computed auto indentation, so there is not enough evidence for a close proposal.

## Change summary

Pressing Tab on an auto-indented blank JavaScript line inserts the computed indentation, but Undo returns the cursor to line start instead of leaving one normal tab stop of indentation.

## Best solution

Editor core should decide the desired undo granularity for TabOperation auto indentation, likely around src/vs/editor/common/cursor/cursorTypeEditOperations.ts, and either preserve the user's explicit tab indentation on undo or intentionally fold this into the broader undo/auto-indent work tracked by https://github.com/microsoft/vscode/issues/30406 if it is the same product decision.
