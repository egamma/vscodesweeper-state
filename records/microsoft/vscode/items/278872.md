# Issue #278872 — Regression: TLS/remote devcontainer features broken in VS Code 1.106 (works in 1.105)

[https://github.com/microsoft/vscode/issues/278872](https://github.com/microsoft/vscode/issues/278872)

## Summary

This was a known regression in 1.106 caused by enabling Node.js system certificate loading by default. The feature was disabled by default in commit 16cd8a75cc7 and shipped in 1.107.0, resolving the TLS breakage.

## Change summary

TLS-dependent devcontainer features broke in VS Code 1.106 due to the new Node.js system certificate loading being enabled by default.

## Best solution

Keep the shipped fix (system certificate loading disabled by default since 1.107.0). The feature remains available as an opt-in setting (`http.systemCertificatesNode`), tracked by https://github.com/microsoft/vscode/issues/287739. No further change needed.
