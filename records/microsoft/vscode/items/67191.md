# Issue #67191 — [json] jsonValidation fileMatch is confusing when used with URIs

[https://github.com/microsoft/vscode/issues/67191](https://github.com/microsoft/vscode/issues/67191)

## Summary

Acknowledged design debt: jsonValidation fileMatch matches the full URI string (including query/fragment) rather than just the path. A team member agreed it's confusing and suggested a new property like `uriMatch`. No fix has landed on main.

## Change summary

jsonValidation fileMatch matches the entire URI string including query and fragment, rather than only the path component, which is surprising for custom scheme providers.

## Best solution

Introduce a URI-aware matching mode (or a new `uriMatch` property) in the JSON language service that allows matching against URI components separately, as aeschli suggested. The matching logic lives in vscode-json-languageservice (upstream npm package) with schema association wiring in `extensions/json-language-features/server/src/jsonServer.ts`.
