# Issue #306243 — Rate limit chat UI could use improvement

[https://github.com/microsoft/vscode/issues/306243](https://github.com/microsoft/vscode/issues/306243)

## Summary

Both feedback points remain valid on current main: the rate-limit time is rendered as a static string (not a live countdown), and the message doesn't explain which limit was hit or show current usage. The code in chatErrorMessages.ts computes `retryAfterString` once and never updates it. Worth keeping open as a polish item for the chat rate-limiting area.

## Change summary

The chat rate-limit error UI shows a static time string that doesn't count down and gives no visibility into what quota was consumed or how to monitor usage.

## Best solution

Make the rate-limit wait time a live countdown (either via periodic re-render of the error content part or a timer-driven update), and enrich the message to name the specific limit type (session, model, weekly) along with a link or summary of current usage. The rendering lives in `src/vs/workbench/contrib/chat/common/chatErrorMessages.ts` (message construction) and the chat content parts that display the error.
