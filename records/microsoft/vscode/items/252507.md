# Issue #252507 — Tab to accept inline (copilot) suggestion indents instead

[https://github.com/microsoft/vscode/issues/252507](https://github.com/microsoft/vscode/issues/252507)

## Summary

The Tab-indent-vs-accept tension is a known design area the team is actively iterating on (benibenj authored multiple related commits through Oct 2025). The `_tabShouldIndent` derived in the inline completions model intentionally returns true on empty lines, which is exactly the reporter's scenario. Labels are already appropriate; keeping open for the ongoing design discussion.

## Change summary

Pressing Tab on an empty line indents instead of accepting a visible inline Copilot suggestion that includes leading indentation.

## Best solution

The team should decide whether `_tabShouldIndent` should yield to a visible inline suggestion when the cursor is on an empty line and the suggestion includes indentation — the current logic at line ~787 of `inlineCompletionsModel.ts` unconditionally returns true for empty lines regardless of whether an inline suggestion is visible. The `_inAcceptFlow` guard partially addresses this but only triggers in specific accept-flow states.
