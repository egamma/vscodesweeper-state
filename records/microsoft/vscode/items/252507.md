# Issue #252507 — Tab to accept inline (copilot) suggestion indents instead

[https://github.com/microsoft/vscode/issues/252507](https://github.com/microsoft/vscode/issues/252507)

## Summary

The Tab-doesn't-accept behavior is traceable in source: `getIndentationInfo()` in `inlineCompletionsModel.ts` sets `startsWithIndentationLessThanTabSize = false` when the ghost text starts with indentation >= tabSize, disabling the Tab keybinding for acceptance. This is an intentional disambiguation design, but the reporter raises a valid UX concern. Already under discussion with a team member engaged.

## Change summary

Pressing Tab inserts indentation instead of accepting a visible Copilot inline suggestion when that suggestion's leading whitespace is >= tabSize.

## Best solution

The team should decide whether the Tab disambiguation logic in `getIndentationInfo()` (src/vs/editor/contrib/inlineCompletions/browser/model/inlineCompletionsModel.ts:283) should be relaxed — e.g., always allow Tab to accept when the inline suggestion is fully visible and the user is not in the suggest widget, or provide a clearer escape hatch. This is a product-design call between preventing accidental acceptance and not frustrating users who clearly want to accept.
