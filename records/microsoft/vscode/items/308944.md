# Issue #308944 — Unable to use Copilot/Local in multiple Chat Editors

[https://github.com/microsoft/vscode/issues/308944](https://github.com/microsoft/vscode/issues/308944)

## Summary

Team-authored bug, still reproduces on main. The `NewChatSessionInPlaceAction` targets `editorGroupService.activeGroup.activeEditor` instead of the editor that invoked it, so switching sessions in a non-active chat editor operates on the wrong one. A TODO on line 1548 confirms the issue is known but unfixed.

## Change summary

Starting a new Copilot/Local session in-place in a chat editor always targets the active editor group's active editor, not the editor where the action was triggered, breaking multi-editor chat workflows.

## Best solution

Pass the originating editor (or its group) through to `openChatSession` so the replace path in `ChatSessionPosition.Editor` uses the correct editor input instead of `editorGroupService.activeGroup.activeEditor`. The comment by mjbvz suggests focusing the chat editor on widget click as a simpler alternative. Fix is localized to `chatSessions.contribution.ts` around line 1547–1552.
