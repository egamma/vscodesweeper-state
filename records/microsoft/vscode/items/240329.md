# Issue #240329 — VScode snap fails to play audio

[https://github.com/microsoft/vscode/issues/240329](https://github.com/microsoft/vscode/issues/240329)

## Summary

This is a snap platform limitation, not a VS Code bug. Classic confinement in snap does not support interface plugs like `audio-playback`, as confirmed by the contributor comment linking to the upstream snapd bug. VS Code's snap uses classic confinement by design.

## Change summary

Audio playback does not work in the VS Code snap package because classic confinement prevents declaring the `audio-playback` interface plug.

## Best solution

This is blocked on snap upstream (https://launchpad.net/bugs/1655369) adding support for plugs/slots in classic confinement. VS Code cannot fix this without snap platform changes. Users who need audio playback should use the .deb package. The snap's `confinement: classic` in `resources/linux/snap/snapcraft.yaml` cannot declare an `audio-playback` plug until snapd supports it.
