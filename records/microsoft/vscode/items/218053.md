# Issue #218053 — [Error] Blocked script execution in...

[https://github.com/microsoft/vscode/issues/218053](https://github.com/microsoft/vscode/issues/218053)

## Summary

The bug appears to still exist on main: the extension editor creates webviews with empty contentOptions (no allowScripts), so the inner iframe in pre/index.html never gets 'allow-scripts' in its sandbox — Safari surfaces this as noisy console errors on scroll. No targeted fix was found in git history. Keeping open as it is a team-filed Safari/web-specific issue.

## Change summary

Scrolling extension pages on Safari/vscode.dev floods the error log with 'Blocked script execution' errors because the webview inner frame lacks the allow-scripts sandbox permission.

## Best solution

In `src/vs/workbench/contrib/extensions/browser/extensionEditor.ts` line 708, the webview is created with `contentOptions: {}` (allowScripts defaults to false). The inner iframe in `pre/index.html` then omits `allow-scripts` from its sandbox. Either the extension editor should set `allowScripts: true`, or `pre/index.html` should gracefully handle the no-scripts case without triggering Safari's error logging on scroll events.
