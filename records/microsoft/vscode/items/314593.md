# Issue #314593 — Feature Request: Language Models – Model Picker – Native ChatGPT Sign-In for Codex

[https://github.com/microsoft/vscode/issues/314593](https://github.com/microsoft/vscode/issues/314593)

## Summary

Reasonable feature request to add native ChatGPT/OpenAI OAuth sign-in as a model provider in the language model picker, bypassing the API-key-only BYOK path. With 11 upvotes and 73 days open, it hasn't hit the 20-upvote backlog threshold yet — keeping open for the community process.

## Change summary

Allow users to sign in with a ChatGPT account (OAuth) and expose subscription-backed OpenAI models (e.g. Codex) in the VS Code model picker without requiring an API key.

## Best solution

This is best addressed as an extension-provided authentication + model registration flow (the Codex extension or an OpenAI extension could register a `ChatGPTAuthenticationProvider` and populate models via the Language Model API). If a missing extension-API gap blocks that (e.g. subscription-vs-API-key billing metadata, or account-scoped provider registration), that gap would be a valid core request. As written, the bulk of this work — OAuth with OpenAI, model enumeration, billing labels — belongs in the extension layer, not VS Code core.
