# Issue #304396 — Accessible editor does not announce initial value unless focused

[https://github.com/microsoft/vscode/issues/304396](https://github.com/microsoft/vscode/issues/304396)

## Summary

Still reproduces on main: both screenReaderContentSimple.ts and screenReaderContentRich.ts guard updateScreenReaderContent with an early return when the editor isn't focused, preventing the textbox value from being set on initial load or model.setValue — so screen readers announce 'blank' when scanning without focus.

## Change summary

Screen reader users scanning the page without focusing the editor hear 'blank' even when the editor has content, because the accessible textbox value is only populated on focus.

## Best solution

Allow `updateScreenReaderContent` to populate the DOM node's textContent on initial render and on model value changes regardless of focus state, while still gating selection-range manipulation on focus. The fix is localized to the two files the reporter identified.
