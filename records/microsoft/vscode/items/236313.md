# Issue #236313 — In Terminal view, the 'Next/Previous Match' search function conflicts with the redirection triggered by Ctrl+Left Click.

[https://github.com/microsoft/vscode/issues/236313](https://github.com/microsoft/vscode/issues/236313)

## Summary

Still reproduces on main: Ctrl+Click link activation in the terminal moves the cursor/selection, and the xterm search addon's findNext/findPrevious searches relative to the current selection position, causing the search to jump away from the previous match position.

## Change summary

Terminal find 'Next Match' resets to the first match after Ctrl+Click opens a link, because link activation moves the cursor and the search addon uses cursor position as the search anchor.

## Best solution

Either preserve the search addon's internal match index across selection changes triggered by link clicks, or save/restore the search position in `terminalFindWidget.ts` when the selection changes due to a link activation. This likely requires coordination with the xterm search addon — the fix may need to land upstream in @xterm/addon-search if the addon resets its position on external selection changes, or VS Code could track the last match index and pass it back on the next findNext call.
