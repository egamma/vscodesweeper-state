# Issue #115022 — View actions disappear when clicking "..."

[https://github.com/microsoft/vscode/issues/115022](https://github.com/microsoft/vscode/issues/115022)

## Summary

Team-authored bug with 'On Deck' milestone — must keep open. The root cause was identified by sbatten: with native menus, focus is never placed on the toolbar button upon click, so the hover-dependent action visibility drops. No fix commit found on main.

## Change summary

View toolbar actions (e.g. '...') disappear when clicking them in smaller views because native menus don't place focus on the triggering button.

## Best solution

Fix the toolbar/action-bar focus handling so that when a native context menu is triggered from a view toolbar button, focus (or a focus-equivalent flag) is set on the button to keep it visible for the menu's lifetime. Likely in src/vs/base/browser/ui/toolbar/toolbar.ts or src/vs/base/browser/ui/actionbar/actionbar.ts.
