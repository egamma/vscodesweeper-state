# Issue #115022 — View actions disappear when clicking "..."

[https://github.com/microsoft/vscode/issues/115022](https://github.com/microsoft/vscode/issues/115022)

## Summary

Team-authored issue on the 'On Deck' milestone — both are hard keep-open conditions. The bug describes view toolbar actions disappearing when native context menus are triggered because focus is never placed on the button, so the hover-dependent visibility logic hides the toolbar.

## Change summary

View action toolbar items (e.g. '...' in Outline) disappear when clicked because native menus don't place focus on the triggering button, and visibility depends on hover or focus.

## Best solution

Ensure that when a native menu is triggered from a view toolbar action, focus (or an equivalent 'menu-open' state) is preserved on the action button so the toolbar remains visible for the duration of the menu interaction. The fix likely belongs in the toolbar/action-bar focus management code under `src/vs/base/browser/ui/actionbar/` or the view pane header logic.
