# Issue #322368 — left navigation bar > Source control > Git Changes are not visible

[https://github.com/microsoft/vscode/issues/322368](https://github.com/microsoft/vscode/issues/322368)

## Summary

This is a support question, not a bug — the described behavior is controlled by the existing `scm.repositories.visible` setting (default: 10). Setting it to `0` allows manual resizing of the repositories section, which is exactly what the reporter needs.

## Change summary

With ~70 repositories in a workspace, the Git Changes section in Source Control is too small to see changes because the repositories list takes up all the space.

## Best solution

The reporter should set `"scm.repositories.visible": 0` in their settings to enable manual resizing of the repositories section, or set it to a larger number. This setting already exists and is documented.
