# Issue #276896 — Search Editor tab labels don't reload correctly

[https://github.com/microsoft/vscode/issues/276896](https://github.com/microsoft/vscode/issues/276896)

## Summary

Still reproduces on main: SearchEditorInput.getName() depends on _cachedConfigurationModel which is only populated after async resolveModels(), so non-focused search editor tabs show just 'Search' after window reload. The closed PR #277059 was never merged and no other fix has landed.

## Change summary

After window reload, non-focused Search Editor tabs show only 'Search' instead of 'Search: <query>' because the configuration model hasn't been resolved yet for inactive tabs.

## Best solution

In SearchEditorInput.getName(), fall back to reading the query from the memento's stored searchConfig when _cachedConfigurationModel is not yet available. The memento already persists the config at line 164 of searchEditorInput.ts; getName() just needs to use it as a synchronous fallback.
