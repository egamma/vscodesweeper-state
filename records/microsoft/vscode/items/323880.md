# Issue #323880 — Allow vscode:// (file / folder open) URL to specify non-destructive window re-use options

[https://github.com/microsoft/vscode/issues/323880](https://github.com/microsoft/vscode/issues/323880)

## Summary

Valid feature request for new query parameters on vscode:// protocol URLs to control window reuse behavior (e.g. reuse matching workspace, never replace active window). No existing canonical issue tracks this; routing to the workbench-window area for triage.

## Change summary

Allow vscode:// URL emitters to specify window reuse intent via a query parameter so that folder URLs don't silently replace the user's active window context.

## Best solution

Add a query parameter (e.g. `windowBehavior`) to the vscode:// protocol handler in `src/vs/code/electron-main/app.ts` → `handleProtocolUrl()` that maps to the existing `forceNewWindow` / `forceReuseWindow` / workspace-match-or-new logic in `windowsMainService.open()`. The implementation would parse the parameter alongside the existing `windowId` handling and translate it into the appropriate `IOpenConfiguration` flags. This needs a product decision on which window-reuse modes to expose and their naming.
