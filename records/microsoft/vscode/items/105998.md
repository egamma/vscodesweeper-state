# Issue #105998 — [json] Format selection doesn't fix indentation

[https://github.com/microsoft/vscode/issues/105998](https://github.com/microsoft/vscode/issues/105998)

## Summary

Still open on main — the JSON range formatter delegates directly to vscode-json-languageservice's format with the selected range, which preserves original indentation rather than fixing it based on document context. No fix has landed.

## Change summary

JSON "Format Selection" should fix incorrect indentation of the selected range to match the document's structure.

## Best solution

The fix belongs in the upstream `vscode-json-languageservice` (or `jsonc-parser`) format function — it should compute correct indentation from document context when formatting a sub-range, rather than preserving the existing indent. Alternatively, the VS Code JSON server could format the whole document and filter edits to the selected range, as jrieken suggested.
