# Issue #322990 — Agent Host client tool hangs after active client removal; browser click invoked with empty args

[https://github.com/microsoft/vscode/issues/322990](https://github.com/microsoft/vscode/issues/322990)

## Summary

Team-authored bug, still present on main. The toolInput-to-`{}` fallback in agentHostSessionHandler.ts (line 2548) fires when the client disconnects mid-turn, and clickBrowserTool's prepareToolInvocation (line 74) does not guard against undefined pageId — causing the tool call to hang in Preparing.

## Change summary

Agent Host browser clickElement tool hangs permanently when the active client is removed mid-turn because it receives empty parameters and crashes during preparation.

## Best solution

Guard the toolInput fallback: do not invoke client tools with `{}` when the tool schema declares required arguments (or when `_meta.toolArguments` is available). Additionally, harden `ClickBrowserTool.prepareToolInvocation` to return an error result when `pageId` is missing, matching the guard already present in `invoke()` at line 98.
