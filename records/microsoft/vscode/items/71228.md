# Issue #71228 — http.systemCertificates requires window reload

[https://github.com/microsoft/vscode/issues/71228](https://github.com/microsoft/vscode/issues/71228)

## Summary

Team-authored issue; must keep open per policy. The underlying limitation (Node.js stores root CAs statically in native code, so toggling http.systemCertificates off requires a reload) likely still exists on current main — the proxy-agent code reads the setting dynamically but system certs loaded into Node's native TLS layer remain cached until the process restarts.

## Change summary

Disabling `http.systemCertificates` at runtime has no effect without a window reload because Node.js caches system certificates statically in native code.

## Best solution

Investigate whether `@vscode/proxy-agent`'s `loadSystemCertificates` path can be made revocable at runtime (e.g., by managing a custom SecureContext rather than relying on Node's static root store), or formally document the reload requirement and close as a known limitation.
