# Issue #205887 — Column selection is reset when selecting with arrow keys

[https://github.com/microsoft/vscode/issues/205887](https://github.com/microsoft/vscode/issues/205887)

## Summary

Still reproducible on main: word-selection commands (Ctrl+Shift+Left/Right) in column selection mode don't go through the ColumnSelectCommand path, so they clear `_columnSelectData` via `setStates`, breaking subsequent character-level column selection (Shift+Left/Right).

## Change summary

In column selection mode, using Ctrl+Shift+Arrow (word select) followed by Shift+Arrow (character select) loses the multi-line column selection.

## Best solution

Either register column-select–aware word selection keybindings (analogous to the existing `registerColumnSelection` for Shift+Arrow) when the `columnSelection` context is active, or preserve `_columnSelectData` across word-selection operations that don't explicitly leave column select mode. The fix lives in `src/vs/editor/browser/coreCommands.ts` (keybinding registration) and/or `src/vs/editor/contrib/wordOperations/browser/wordOperations.ts`.
