# Issue #205887 — Column selection is reset when selecting with arrow keys

[https://github.com/microsoft/vscode/issues/205887](https://github.com/microsoft/vscode/issues/205887)

## Summary

Still reproduces on main: Ctrl+Shift+Right (word select) in column selection mode doesn't go through the column select path, so it doesn't store columnSelectData. A subsequent Shift+Left uses the column select path but falls back to deriving state from the primary cursor only, losing the multi-line selection.

## Change summary

In column selection mode, using Ctrl+Shift+Right (word select) followed by Shift+Left (character deselect) resets the column selection to only the first line.

## Best solution

Register Ctrl+Shift+Left/Right column-select-word variants (similar to how Shift+Arrow is mapped to columnSelectLeft/Right) so that word selection in column mode stores proper columnSelectData, OR ensure the regular word-select path in column mode updates _columnSelectData so subsequent character-level column select commands work correctly.
