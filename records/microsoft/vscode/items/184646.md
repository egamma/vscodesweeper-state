# Issue #184646 — .vscode\cli\ ignores portable mode

[https://github.com/microsoft/vscode/issues/184646](https://github.com/microsoft/vscode/issues/184646)

## Summary

Issue has an assigned milestone (On Deck), which is a scheduling decision — must defer to maintainer judgment and keep open. The request is valid: the CLI component never implemented portable-mode awareness, confirmed by a team member.

## Change summary

The VS Code CLI creates ~/.vscode/cli/ in the user's home directory even when running in portable mode, breaking the portability guarantee.

## Best solution

Teach the CLI startup path to detect portable mode (presence of the `data` folder or `VSCODE_PORTABLE` env var) and redirect its data directory into the portable data folder (e.g. `data/cli/`), consistent with how the rest of VS Code handles portable mode. The workaround via `VSCODE_CLI_DATA_DIR` or `--cli-data-dir` exists but should become automatic.
