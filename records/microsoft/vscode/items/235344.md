# Issue #235344 — `Add line comment` should await the tokenization to get the correct language at the position

[https://github.com/microsoft/vscode/issues/235344](https://github.com/microsoft/vscode/issues/235344)

## Summary

Confirmed bug caused by async tokenization (commit 29be8e8665). Still reproduces on main — asyncTokenization defaults to true and the comment command doesn't await tokenization. An open PR (#244217) and investigation issue (#244225) track this; keeping open.

## Change summary

The 'Add line comment' command uses the wrong language (or no comment action) when tokenization hasn't completed for embedded languages like JS inside HTML script tags.

## Best solution

Either make the comment command await tokenization before determining the language at the cursor position, or resolve the broader async tokenization issues tracked in https://github.com/microsoft/vscode/issues/244225. The open PR https://github.com/microsoft/vscode/pull/244217 proposes turning off asyncTokenization by default as a stopgap.
