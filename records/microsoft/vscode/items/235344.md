# Issue #235344 — `Add line comment` should await the tokenization to get the correct language at the position

[https://github.com/microsoft/vscode/issues/235344](https://github.com/microsoft/vscode/issues/235344)

## Summary

Keep this open: the report has a maintainer-confirmed reproduction and current main still appears to make the line-comment decision synchronously without awaiting embedded-language tokenization. There is no fixing commit or confirmed canonical duplicate to support a cleanup close.

## Change summary

Line comment toggling in an HTML `<script>` block can run before embedded JavaScript tokenization is ready after reload, causing no comment or the wrong HTML comment behavior.

## Best solution

Update the editor comment contribution so line-comment commands wait for tokenization or embedded-language state before resolving the language configuration at the target position. If this was fixed elsewhere, link the exact commit or PR before closing as completed.
