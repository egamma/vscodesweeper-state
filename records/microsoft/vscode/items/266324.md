# Issue #266324 — Consistent crashes with "The window terminated unexpectedly (reason: 'crashed', code: '5')"

[https://github.com/microsoft/vscode/issues/266324](https://github.com/microsoft/vscode/issues/266324)

## Summary

Confirmed, reproduced crash where the debug console renders a very large string (e.g. a ~122 MB base64 source map) causing Chromium's HarfBuzz text shaper to OOM and terminate the renderer. A team member confirmed reproduction, the reporter bisected it and provided an lldb backtrace. No fix has landed on main.

## Change summary

Debugging apps with large inline source maps causes a renderer OOM crash (code 5) because the debug console tries to render the entire base64 string through HarfBuzz text shaping.

## Best solution

Truncate or virtualize very large string values before they reach the DOM in the debug console renderer. The fix likely belongs in `src/vs/workbench/contrib/debug/browser/` (REPL rendering or the debug session output pipeline) — cap the text length passed to DOM text nodes so HarfBuzz never receives a multi-hundred-megabyte string. Alternatively, the js-debug adapter could truncate large source-map-related output before sending it to the DAP client.
