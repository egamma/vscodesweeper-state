# Issue #320087 — scm.autoReveal does not reveal/select staged (index) files in the Source Control view

[https://github.com/microsoft/vscode/issues/320087](https://github.com/microsoft/vscode/issues/320087)

## Summary

Still reproduces on main: `onDidActiveEditorChange()` in scmViewPane.ts gets a `git:`-scheme URI for staged diffs but compares it against `sourceUri` which is always `file:`-scheme, so the match never succeeds for staged files.

## Change summary

`scm.autoReveal` does not reveal staged (index) files in the Source Control view because the URI scheme mismatch prevents the resource lookup from matching.

## Best solution

In `onDidActiveEditorChange()`, normalize the resolved URI before comparison — strip `git:` scheme URIs back to their underlying `file:` path (carried in the git URI's query/path) so they match `sourceUri`. The fix is localized to `src/vs/workbench/contrib/scm/browser/scmViewPane.ts` around lines 1759–1786.
