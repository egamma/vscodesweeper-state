# Issue #283460 — Numpad Delete key not working in version 1.107.0 (regression from 1.106.3)

[https://github.com/microsoft/vscode/issues/283460](https://github.com/microsoft/vscode/issues/283460)

## Summary

Confirmed upstream Chromium regression affecting Numpad Delete on Wayland (Chromium ≥140 / Electron ≥38). Already correctly labeled `upstream` + `electron` + `chromium`. A team member (deepak1556) has engaged. No fix on VS Code main; the root cause lives in Chromium's Wayland key event handling. Keep open to track upstream resolution.

## Change summary

Numpad Delete key sends `key: "Unidentified"` under Wayland in Chromium ≥140 / Electron ≥38, breaking delete functionality in the editor and input boxes.

## Best solution

The fix belongs in Chromium (https://issues.chromium.org/issues/429730008). VS Code should keep tracking this upstream bug. A possible interim workaround in VS Code's keybinding layer — mapping `NumpadDecimal` with `key: "Unidentified"` to Delete — could be considered, but Comment 12 shows it doesn't cover all surfaces (e.g. find/replace input boxes). The clean fix is the upstream Chromium patch.
