# Issue #101487 — Quick open: Include symbols should not block the search based on filename

[https://github.com/microsoft/vscode/issues/101487](https://github.com/microsoft/vscode/issues/101487)

## Summary

Still valid on main: `getAdditionalPicks` uses `Promise.all` to wait for both file and symbol picks before showing any results, so a slow symbol provider blocks file results from appearing. The request for non-blocking, incremental display of file results remains unimplemented. With 23 upvotes and Backlog status, this should stay open.

## Change summary

When `search.quickOpen.includeSymbols` is enabled, slow workspace symbol providers block file search results from appearing in Ctrl+P quick open.

## Best solution

Refactor `getAdditionalPicks` in `src/vs/workbench/contrib/search/browser/anythingQuickAccess.ts` to return file picks as soon as they resolve, then merge symbol picks incrementally when they arrive — likely by splitting the current `Promise.all` into two phases or using the existing `additionalPicks` merge-delay mechanism to stream file results first.
