# Issue #277667 — October 2025 release breaks GitHub Copilot `Ctrl+Y` "Keep" shortcut.

[https://github.com/microsoft/vscode/issues/277667](https://github.com/microsoft/vscode/issues/277667)

## Summary

The bug is acknowledged by the area owner (jrieken) — the `chatEdits.cursorInChangeRange` context key intermittently evaluates to `false` when the cursor is demonstrably inside a change range, preventing Ctrl+Y from dispatching `chatEditor.action.acceptHunk`. No fix has landed on main; the issue remains valid.

## Change summary

Ctrl+Y (Keep/accept hunk) intermittently fails because the `chatEdits.cursorInChangeRange` context key incorrectly evaluates to false even when the cursor is inside a chat-edit change hunk.

## Best solution

Investigate why `chatEdits.cursorInChangeRange` goes stale in `src/vs/workbench/contrib/chat/browser/chatEditing/chatEditingCodeEditorIntegration.ts` (lines 208–218). The intermittent nature suggests the `documentDiffInfo` observable or `enabledObs` may lose sync with the actual editor state — possibly a race where decorations are cleared/rebuilt between requests and the cursor-position autorun fires before diff data is rehydrated. A targeted repro (multi-file accept flow, cursor placed after cross-file navigation) and fix in the autorun's dependency tracking is likely needed.
