# Issue #214824 — Debugger UX should show stack for current thread instead of a treeview with all threads

[https://github.com/microsoft/vscode/issues/214824](https://github.com/microsoft/vscode/issues/214824)

## Summary

Team-authored feature request for a single-thread call stack view with a thread switcher in the debugger, distinct from the parallel/green-thread view in #116109. No implementation found on main; hard keep-open as the author is a MEMBER.

## Change summary

Show only the current thread's call stack in the debugger with a thread-switching dropdown, instead of a tree view listing all threads.

## Best solution

Implement a focused single-thread call stack mode in `src/vs/workbench/contrib/debug/browser/callStackView.ts` with a thread-picker dropdown, complementing the existing all-threads tree view. This is distinct from #116109 (high-scale green-thread/parallel-stacks view) and should be tracked separately as a UX improvement for traditional multi-thread debugging.
