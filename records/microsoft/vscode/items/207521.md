# Issue #207521 — If a URI contains an authority component, then the path component must either be empty or begin with a slash ("/") character

[https://github.com/microsoft/vscode/issues/207521](https://github.com/microsoft/vscode/issues/207521)

## Summary

The core reproduction path — splitting a terminal whose parent has a remote authority — was fixed on main; the URI.from call in _splitTerminal now prepends '/' to the path when an authority is present, preventing the UriError.

## Change summary

Creating or splitting a terminal in a multi-root or remote workspace sometimes throws a UriError because the path component lacks a leading slash when an authority is present.

## Best solution

Keep the shipped fix in commit da7155bb4f5; the _splitTerminal path in terminalService.ts now guards the URI path. No further change needed — close as completed.
