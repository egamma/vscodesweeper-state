# Issue #44056 — When A grammar injects to B, grammars embedding B should get A's injection

[https://github.com/microsoft/vscode/issues/44056](https://github.com/microsoft/vscode/issues/44056)

## Summary

Still an open gap — grammar injections target explicit scope names and do not transitively propagate into grammars that embed the injected scope. The `getInjections` callback in TMGrammarFactory resolves injections by direct scope-name prefix match, but doesn't account for grammars that embed the target scope via `include`.

## Change summary

When grammar A injects into scope B, grammars that embed B (e.g., Vue embedding source.js) should also receive A's injections in the embedded regions.

## Best solution

Extend the injection resolution in `src/vs/workbench/services/textMate/common/TMGrammarFactory.ts` (or upstream in vscode-textmate) so that `getInjections` propagates injections transitively through embedding relationships. Alternatively, document a workaround where injection grammars explicitly list all embedding scopes in their `injectTo` array.
