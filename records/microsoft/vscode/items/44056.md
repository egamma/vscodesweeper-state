# Issue #44056 — When A grammar injects to B, grammars embedding B should get A's injection

[https://github.com/microsoft/vscode/issues/44056](https://github.com/microsoft/vscode/issues/44056)

## Summary

Keep open as an old grammar feature request in the Backlog: it has not reached the 20-upvote backlog-candidate threshold, but there is no strong close reason or canonical duplicate in the provided context.

## Change summary

When one TextMate grammar injects into an embedded grammar scope, files that embed that grammar should also receive the injected grammar highlighting.

## Best solution

The grammar/TextMate owner should decide whether injection propagation across embedded languages belongs in VS Code's grammar registry or should remain the responsibility of individual embedding grammars/extensions such as Vetur.
