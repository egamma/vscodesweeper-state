# Issue #322566 — Renamed chat session title reverts to auto-generated name on reopen (1.126.0)

[https://github.com/microsoft/vscode/issues/322566](https://github.com/microsoft/vscode/issues/322566)

## Summary

Valid bug confirmed in current source: `ChatSessionStore.setSessionTitle` updates the in-memory index but never calls `flushIndex()`, so the renamed title is lost on reload. An open community PR (#323323) is already attempting a fix.

## Change summary

User-set chat session titles revert to the auto-generated name after reopening or reloading the window because the rename is not persisted to storage.

## Best solution

Review and land (or supersede) the open fix PR at https://github.com/microsoft/vscode/pull/323323. The core gap is that `ChatSessionStore.setSessionTitle` (src/vs/workbench/contrib/chat/common/model/chatSessionStore.ts, line 503) must call `flushIndex()` after mutating the index entry, and the full session data blob (which carries `customTitle`) should also be re-written so the two stores stay consistent.

## Evidence

- **missing flushIndex:** src/vs/workbench/contrib/chat/common/model/chatSessionStore.ts line 503–510: `setSessionTitle` mutates `index.entries[sessionId].title` but never calls `flushIndex()`. Every other index-mutating method (`storeSessions`, `clearAllSessions`, `deleteSession`, `storeSessionsMetadataOnly`) calls `flushIndex()` afterward.
- **customTitle serialization path:** ChatModel.toJSON() (src/vs/workbench/contrib/chat/common/model/chatModel.ts line 3022) serializes `_customTitle`, and the constructor restores it (line 2466). The model-level serialization is correct, but the index (which drives the sessions list UI) is not flushed on rename.
- **open fix PR:** https://github.com/microsoft/vscode/pull/323323 ('Fix: Session title rename not persisting after reload') is open and authored by tamuratak, targeting exactly this issue.
- **community confirmation:** A second user (bastabc) confirmed the bug in a comment on 2026-07-05: the title updates but reverts within a couple of minutes.
