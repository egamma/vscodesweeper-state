# Issue #285073 — Feature Request: Support defaultValue in l10n.t() to enable stable keys without side effects

[https://github.com/microsoft/vscode/issues/285073](https://github.com/microsoft/vscode/issues/285073)

## Summary

Valid feature request for a stable-key pattern in the extension l10n API. The current API has no defaultValue/fallback support, and this remains an unaddressed gap for extensions with large translation surfaces. Low upvote count (1) — leaving open for community traction.

## Change summary

Add a defaultValue/fallback parameter to vscode.l10n.t() so extension authors can use stable keys without the English text doubling as the lookup key.

## Best solution

The l10n extension API (declared in src/vscode-dts/vscode.d.ts and implemented via the @vscode/l10n package) would need a new overload or options-object field for defaultValue. Given this touches a cross-cutting API surface shared with vscode-l10n tooling, the right path is for the l10n-platform area owner to evaluate whether stable keys fit the l10n design philosophy or whether this belongs in userland wrappers.
