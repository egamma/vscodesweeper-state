# Issue #319512 — Hide Activity Bar icons in User/Workspace Settings

[https://github.com/microsoft/vscode/issues/319512](https://github.com/microsoft/vscode/issues/319512)

## Summary

Valid feature request to expose activity bar icon visibility as a user/workspace setting. Currently this state lives in storage (profile scope) via pinnedViewContainers, not in settings.json. The community process is active with 10 days remaining; 6/20 upvotes so far.

## Change summary

Allow hiding specific activity bar icons via a setting in settings.json rather than only through the context menu (which stores state in profile storage and can reset unexpectedly).

## Best solution

Add a configuration setting (e.g. `workbench.activityBar.hiddenViewContainers`) that accepts view container IDs, backed by the existing pinned/visible logic in `src/vs/workbench/browser/parts/paneCompositeBar.ts`. This would give users a durable, portable way to hide activity bar items across workspaces. Let the community process decide whether there's enough demand.
