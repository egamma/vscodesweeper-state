# Issue #211405 — onDidChangeTabGroups not fired when switching focus between floating window and main window

[https://github.com/microsoft/vscode/issues/211405](https://github.com/microsoft/vscode/issues/211405)

## Summary

Team-authored bug, still present on main. The `onDidChangeTabGroups` event is only fired via `GroupModelChangeKind.GROUP_ACTIVE` in `mainThreadEditorTabs.ts`, which doesn't trigger when window focus moves between the main window and a floating (auxiliary) window without an actual group activation change within the editor groups service.

## Change summary

The `onDidChangeTabGroups` extension API event is not fired when the user switches focus between a floating window and the main window.

## Best solution

In `src/vs/workbench/api/browser/mainThreadEditorTabs.ts`, listen for auxiliary window focus changes (or the equivalent `onDidChangeActiveGroup` across parts) and fire `$acceptTabGroupUpdate` when the active group effectively changes due to window focus switching. The fix likely also involves `src/vs/workbench/browser/parts/editor/editorGroupService.ts` to ensure the active group is updated across windows on focus.
