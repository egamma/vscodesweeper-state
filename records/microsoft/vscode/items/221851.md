# Issue #221851 — [wsl] Use curl instead of wget to download VS Code server

[https://github.com/microsoft/vscode/issues/221851](https://github.com/microsoft/vscode/issues/221851)

## Summary

Team-authored feature request for the WSL extension to use curl instead of wget. The WSL extension lives outside this repo (microsoft/vscode-remote-release or the bundled extension), so fixing it here is not straightforward, though the newer agentHost WSL helpers already use curl. Keeping open per team-author policy.

## Change summary

WSL extension's VS Code server download fails when wget is not installed, even though curl is available.

## Best solution

The WSL extension's server download script should prefer curl (or fall back to it) the same way Remote-SSH does. The newer agentHost WSL code in `src/vs/platform/agentHost/node/wslRemoteAgentHostHelpers.ts` already uses curl exclusively — once that path fully replaces the old WSL extension download flow, this issue is resolved.
