Firefox's Intl word segmentation differs from Chromium/WebKit for Japanese text. The failing tests are currently skipped on Firefox (commit 30a5144f2dd) but the underlying platform difference remains unresolved. This is an upstream browser behavior gap.

Either normalize word-boundary logic across browsers (potentially with a custom segmenter fallback for Firefox) or accept the platform difference and keep the tests skipped.
