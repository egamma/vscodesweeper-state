# Issue #105382 — Git - Use the git submodule's name in source control view

[https://github.com/microsoft/vscode/issues/105382](https://github.com/microsoft/vscode/issues/105382)

## Summary

Valid feature request: submodules still display using the last path segment of their filesystem path rather than their git-configured logical name. The related PR #106404 was closed without merging, and no subsequent fix has landed. Already labeled `help wanted` and in Backlog — keeping open.

## Change summary

Show git submodule's configured logical name (from `.gitmodules`) instead of the last path segment in the Source Control view.

## Best solution

In `extensions/git/src/repository.ts` around line 984, when creating the source control for a submodule (`repository.kind === 'submodule'`), look up the submodule's logical name (already parsed in `git.ts` line 899 as `submoduleSection.subSectionName`) and pass it to the SCM label or use it as a display override. The submodule data is available via `repository.submodules` on the parent. A setting to toggle between path and name display would address the 'optionally' qualifier in the request.
