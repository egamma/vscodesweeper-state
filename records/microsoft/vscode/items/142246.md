# Issue #142246 — Multi-line Highlight

[https://github.com/microsoft/vscode/issues/142246](https://github.com/microsoft/vscode/issues/142246)

## Summary

This is a feature request (not a bug) asking for an option to highlight only the primary cursor line rather than all cursor lines in multi-cursor mode. The current behavior is by design (introduced in 1.64). The issue carries the 'On Deck' milestone, which is a scheduling decision — keeping open.

## Change summary

Request an option to only highlight the primary cursor line instead of all cursor lines in multi-cursor scenarios.

## Best solution

Add a setting (e.g. `editor.renderLineHighlightForMultiCursors`) to control whether secondary cursor lines are highlighted. The change would be localized to `src/vs/editor/browser/viewParts/currentLineHighlight/currentLineHighlight.ts` in the `_readFromSelections` / `prepareRender` flow, gating `_cursorLineNumbers` to only the primary when the setting is off.
