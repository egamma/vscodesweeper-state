# Issue #319861 — Can't multi-select (for compare) in open editors

[https://github.com/microsoft/vscode/issues/319861](https://github.com/microsoft/vscode/issues/319861)

## Summary

Confirmed on current main: multi-selection for compare in Open Editors is effectively broken because two mechanisms reset selection to only the active editor, making it near-impossible to reach the 'Compare Selected' context menu item that IS registered for this view.

## Change summary

Ctrl+Right-Click in the Open Editors pane fails to maintain multi-selection, so the 'Compare Selected' context menu item never appears.

## Best solution

Fix the interaction between `focusActiveEditor()` in `openEditorsView.ts` and the base list's `onContextMenu` handler in `listWidget.ts` so that multi-selection is preserved during right-click. Two things fight against it: (1) the base list `onContextMenu` (line 750-757) unconditionally sets focus to only the right-clicked item; (2) clicking a file opens it, triggering `EDITOR_ACTIVE` → `focusActiveEditor()` which resets selection to just the active editor. The fix likely belongs in `openEditorsView.ts` — either suppress `focusActiveEditor` during context-menu interactions, or make right-click on an already-selected item preserve the multi-selection.
