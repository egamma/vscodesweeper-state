# Issue #324018 — In progress animation sometimes doesn't show in sessions list

[https://github.com/microsoft/vscode/issues/324018](https://github.com/microsoft/vscode/issues/324018)

## Summary

Still reproduces on main: the IntersectionObserver callback permanently unobserves a not-yet-connected spinner (line 87–89 of pixelSpinner.ts), so it stays stuck with the paused class. The draft fix PR #324028 was closed without merging. Team-authored — keeping open.

## Change summary

Pixel spinner in the sessions list sometimes stays paused because the IntersectionObserver unobserves it before it is appended to the DOM.

## Best solution

Guard the `unobserve` call in `src/vs/base/browser/ui/pixelSpinner/pixelSpinner.ts` so that only spinners that were connected at least once (genuine removals) are unobserved — a not-yet-inserted spinner should remain observed until it enters the DOM and the observer can correctly unpause it.
