# Issue #325149 — Agent Mode fails with GitHub Enterprise auth - "thinking" forever / TypeError: terminated

[https://github.com/microsoft/vscode/issues/325149](https://github.com/microsoft/vscode/issues/325149)

## Summary

Valid GHE-specific Agent Mode bug. The Copilot CLI agent issue was largely addressed by commit 62c7ad83b90 (July 7, 2026) which added GHE endpoint routing, but the Claude provider's proxy model fetch still lacks GHE-aware endpoint derivation on current main — the `_fetchProxyModels` path does not appear to route CAPI calls through the enterprise host. The user's VS Code 1.128.0 stable predates the fix entirely. Route to agent-host and claude-agent areas.

## Change summary

Agent Mode hangs or errors when using GitHub Enterprise authentication; the Copilot provider hangs on 'thinking' and the Claude provider fails with 'TypeError: fetch failed' during model refresh, while Chat mode works fine with the same GHE config.

## Best solution

The Copilot CLI agent path is likely resolved on main by 62c7ad83b90 (sets COPILOT_GH_HOST from the new endpoint service). The Claude agent's proxy model fetch (`_fetchProxyModels` in `src/vs/platform/agentHost/node/claude/claudeAgent.ts`) still needs investigation — it uses `IAgentHostGitHubEndpointService` for auth resources but may not be routing CAPI model-list calls through the enterprise endpoint. The workspace files sidebar ENOPRO error (comment 1) is a separate issue. Ask the reporter to test on the latest Insiders (post-July 7) for the Copilot provider fix, and file the Claude provider failure as a remaining gap.
