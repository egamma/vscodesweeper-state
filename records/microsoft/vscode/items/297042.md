# Issue #297042 — Quick pick session covers content when you click back

[https://github.com/microsoft/vscode/issues/297042](https://github.com/microsoft/vscode/issues/297042)

## Summary

Team-authored bug report — keeping open. The sessions picker (quick pick) stays visible and covers the previous session's content when the user clicks the title to open it, then clicks back, because `closeOnFocusLost` is disabled. No fix found on current main.

## Change summary

The sessions quick pick overlays previous session content when navigating back with `workbench.quickOpen.closeOnFocusLost` disabled.

## Best solution

The Go Back action (`sessions.goBack`) should dismiss any open quick pick before navigating, or the sessions picker should hook into the navigation lifecycle to auto-close. Likely files: `src/vs/sessions/contrib/sessions/browser/sessionsActions.ts` (both the picker and the GoBackAction) and possibly `src/vs/workbench/services/quickinput/browser/quickInputService.ts` for the `ignoreFocusOut` interaction.
