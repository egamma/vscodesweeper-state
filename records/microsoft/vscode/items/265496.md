# Issue #265496 — Add support in vscode for Agent Client Protocol (ACP)

[https://github.com/microsoft/vscode/issues/265496](https://github.com/microsoft/vscode/issues/265496)

## Summary

Well-supported feature request (324 upvotes, well above the 20-upvote Backlog threshold) for Agent Client Protocol integration. No ACP implementation exists on current main — the agent host infrastructure supports specific harnesses (Claude Code, Copilot CLI, Codex) but not the ACP standard. A team member acknowledged the request early on without committing. The existing labels (`under-discussion`, `chat-agent`) are correct; this should stay open for the community process and team evaluation.

## Change summary

Add support for the Agent Client Protocol (ACP) to allow third-party agent CLIs to integrate with VS Code's chat/agent infrastructure through a standardized protocol.

## Best solution

Evaluate ACP as a harness protocol alongside the existing Claude Code / Copilot CLI / Codex integrations in `src/vs/platform/agentHost/`. If the protocol matures sufficiently and the team decides to support it, it could be implemented as another agent harness. Alternatively, if a public extension API for agent harnesses is exposed (as the contributor suggested), the community could build ACP support as an extension. The decision depends on whether ACP reaches feature parity with the direct SDK integrations and whether a public agent-harness extension API is planned.
