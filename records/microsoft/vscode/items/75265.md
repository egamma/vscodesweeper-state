# Issue #75265 — Global regex search with "Not matching character" doesn't match newline

[https://github.com/microsoft/vscode/issues/75265](https://github.com/microsoft/vscode/issues/75265)

## Summary

Valid, acknowledged feature request for better multiline regex detection or an explicit multiline toggle in search. The current heuristic in `isMultilineRegexSource` only triggers multiline for `\n`, `\r`, `\W`, and literal newlines — negated character classes like `[^\]]` and `\s` are not detected. A team member (roblourens) acknowledged the gap and noted a future UX solution is needed. Still unfixed on main with 13 upvotes; keeping open.

## Change summary

Negated character classes (e.g. `[^\]]`) and `\s` in regex search do not match newlines because VS Code's multiline heuristic does not detect them as needing ripgrep's `--multiline` flag.

## Best solution

Either expand the `isMultilineRegexSource` heuristic in `src/vs/editor/common/model/textModelSearch.ts` to detect negated character classes and `\s`, or add an explicit multiline toggle button in the search widget UX — the latter was favored by the team member in the discussion. The `\n{0}` workaround should be documented in the interim.
