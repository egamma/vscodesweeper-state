# Issue #75265 — Global regex search with "Not matching character" doesn't match newline

[https://github.com/microsoft/vscode/issues/75265](https://github.com/microsoft/vscode/issues/75265)

## Summary

Still valid: the multiline heuristic in `isMultilineRegexSource` only detects `\n`, `\r`, `\W`, and literal newlines — it misses negated character classes like `[^\]]` and `\s` that semantically should match newlines. No fix or explicit multiline toggle has been added on main. Keeping open as an active feature request with community interest.

## Change summary

Regex search should enable ripgrep's `--multiline` flag when the pattern contains negated character classes or `\s`, or provide an explicit multiline toggle in the search UI.

## Best solution

Either expand the `isMultilineRegexSource` heuristic in `src/vs/editor/common/model/textModelSearch.ts` to detect negated character classes (`[^...]`) and `\s` as multiline indicators, or add an explicit multiline toggle button to the search widget UI. The latter is cleaner and avoids an ever-growing heuristic — roblourens acknowledged this as the likely long-term path in the discussion.
