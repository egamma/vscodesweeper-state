# Issue #234513 — Poor/pathological multi-cursor performance

[https://github.com/microsoft/vscode/issues/234513](https://github.com/microsoft/vscode/issues/234513)

## Summary

Keep open: the reported multi-cursor growth path still exists on current main, and there is no fixing commit, duplicate, or maintainer resolution in the provided discussion. The issue is already routed to the editor multi-cursor/performance area.

## Change summary

Repeated Add Cursor Below can generate duplicate cursor states and lead to pathological editor performance or freezes.

## Best solution

Have the editor-multicursor owner decide whether Add Cursor Above/Below should avoid constructing duplicate cursor states before `setCursorStates`, especially when overlapping cursor merging is disabled or when large intermediate cursor arrays hit `editor.multiCursorLimit`.
