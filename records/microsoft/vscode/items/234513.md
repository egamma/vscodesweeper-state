# Issue #234513 — Poor/pathological multi-cursor performance

[https://github.com/microsoft/vscode/issues/234513](https://github.com/microsoft/vscode/issues/234513)

## Summary

Still present on main: `addCursorDown` iterates all existing cursors and adds a new cursor below each one, creating O(2N) intermediate states before normalization merges duplicates. With many cursors this causes pathological DOM churn and freezes.

## Change summary

Using Add Cursor Below repeatedly causes exponential intermediate cursor growth and freezes VS Code after ~12 iterations.

## Best solution

Fix `addCursorDown`/`addCursorUp` in `src/vs/editor/common/cursor/cursorMoveCommands.ts` to only add a cursor below the last (most recently added) cursor rather than duplicating for every existing cursor, or deduplicate before returning. The normalize step handles merging but the intermediate array size and downstream DOM updates are the real performance issue.
