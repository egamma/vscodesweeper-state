# Issue #307922 — UI corruption when opening extension "Open to the Side" from Profiles → Contents → Extensions

[https://github.com/microsoft/vscode/issues/307922](https://github.com/microsoft/vscode/issues/307922)

## Summary

Confirmed reproducible layout bug: opening an extension 'to the side' from the Profiles editor causes UI corruption because the extension editor opens in SIDE_GROUP while the Profiles editor's SplitView layout doesn't account for the editor group split. No fix on main.

## Change summary

Opening an extension via 'Open to the Side' from Profiles → Contents → Extensions causes visual corruption with overlapping/duplicated UI elements.

## Best solution

Either open the extension editor in the same group (ACTIVE_GROUP) when triggered from the Profiles editor context, or ensure the Profiles editor's layout properly handles the resulting editor group split. The fix likely belongs in `src/vs/workbench/contrib/userDataProfile/browser/userDataProfilesEditorModel.ts` around line 326 where `extension.open` is called with `sideByside: true`.
