# Issue #324422 — Pseudoterminal plugins do not have a way to get the users cwd picked in a multi-root workspace

[https://github.com/microsoft/vscode/issues/324422](https://github.com/microsoft/vscode/issues/324422)

## Summary

Well-described API gap: the cwd chosen by the user in the multi-root workspace picker flows into $createContributedProfileTerminal but is never forwarded to provideTerminalProfile or exposed on Terminal.creationOptions. No duplicate found. Keeping open for the community process (Backlog Candidates, 0 upvotes, 15 days old).

## Change summary

Extension terminal profile providers cannot access the user-selected cwd from the multi-root workspace picker, because it is dropped before reaching provideTerminalProfile.

## Best solution

Extend the TerminalProfileProvider.provideTerminalProfile API to accept a context/options argument containing the resolved cwd/WorkspaceFolder, or expose it on Terminal.creationOptions for extension terminals. The fix touches src/vs/workbench/api/common/extHostTerminalService.ts ($createContributedProfileTerminal) and the vscode.d.ts API surface. This is a legitimate API addition for the terminal area team to evaluate.
