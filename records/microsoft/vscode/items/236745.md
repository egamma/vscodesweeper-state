# Issue #236745 — editorHoverVisible keybinding condition does not work

[https://github.com/microsoft/vscode/issues/236745](https://github.com/microsoft/vscode/issues/236745)

## Summary

Still reproduces on main: `editorHoverVisible` is bound to the editor-scoped context key service (created via `contextKeyService.createScoped(this._domElement)` in codeEditorWidget.ts:317), so it's invisible to keybinding resolution when focus is outside the editor DOM.

## Change summary

The `editorHoverVisible` context key is not visible to keybinding resolution when focus is outside the editor, even though the hover tooltip is still displayed.

## Best solution

Set `editorHoverVisible` on a broader (global or workbench-level) context key service rather than the editor-scoped one, or mirror it to a global key. The fix likely touches `src/vs/editor/contrib/hover/browser/contentHoverWidget.ts` to bind the key to a parent context key service that is visible regardless of focus location.
