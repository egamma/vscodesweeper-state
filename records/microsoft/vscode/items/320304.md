# Issue #320304 — Whitespace colors should be more customizable.

[https://github.com/microsoft/vscode/issues/320304](https://github.com/microsoft/vscode/issues/320304)

## Summary

Feature request for richer whitespace color customization (background color token, relative/theme-derived color functions in user settings). No existing support for either capability in user-facing colorCustomizations. Keeping open for the community process.

## Change summary

Allow users to set both foreground and background colors for whitespace characters, and support relative color adjustments (e.g. darker/lighter) referencing other theme colors in colorCustomizations.

## Best solution

The request bundles two distinct enhancements: (1) adding an `editorWhitespace.background` color token — straightforward, localized to `src/vs/editor/common/core/editorColorRegistry.ts` and the whitespace renderer; (2) exposing relative color functions (`darker`, `lighter`, referencing other tokens) in user-level `colorCustomizations` — a much larger design surface affecting the color registry, theme schema, and settings resolution. The first could be a standalone PR; the second is a broader theming infrastructure request that may warrant its own tracking issue.
