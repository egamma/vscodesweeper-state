# Issue #192110 — Drag file another remote machine differs from origin

[https://github.com/microsoft/vscode/issues/192110](https://github.com/microsoft/vscode/issues/192110)

## Summary

Still present on main: the explorer drag-and-drop logic in `explorerViewer.ts` never checks whether source and target files belong to different remote authorities, so dragging a file from remote A to an explorer rooted on remote B silently performs the operation using the wrong remote's file, potentially overwriting or misrepresenting content. The `help wanted` label is already applied.

## Change summary

Dragging a file between two Remote-SSH windows connected to different machines silently uses the wrong file when paths overlap, because the explorer doesn't validate remote authority during DnD.

## Best solution

Add an authority/scheme check in `FileDragAndDrop.handleDragOver` (and/or `handleExplorerDrop`) in `src/vs/workbench/contrib/files/browser/views/explorerViewer.ts` to reject or warn when source items' `resource.authority` differs from the target's, preventing silent cross-remote file operations.
