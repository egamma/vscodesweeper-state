## Summary

The rename from `FileSearchProviderOptions` to `FileSearchOptions` has not been done — the type still exists under the old name across the proposed API DTS, internal search types, and conversion types. The issue remains valid polish work.

## Change summary

Rename `FileSearchProviderOptions` to `FileSearchOptions` since it describes per-invocation options, not provider-level configuration.

## Best solution

Rename the type in `src/vscode-dts/vscode.proposed.fileSearchProvider2.d.ts`, `src/vs/workbench/services/search/common/searchExtTypes.ts`, and all internal consumers. Coordinate with the existing old-API `FileSearchOptions` in `searchExtConversionTypes.ts` (which already uses that name for the v1 API) to avoid a collision — likely the old-API type gets removed or aliased when the v1 provider is fully retired.
