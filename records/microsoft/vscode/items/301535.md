## Summary

Still reproducible on main: the 'Switch to Release Version' button is shown and enabled even when the extension has no release version, because TogglePreReleaseExtensionAction.update() does not check `hasReleaseVersion` — unlike the install action which correctly guards against this at line 486.

## Change summary

The 'Switch to Release Version' button appears and executes without effect when an extension has no release version available.

## Best solution

Add a `hasReleaseVersion` guard in `TogglePreReleaseExtensionAction.update()` (src/vs/workbench/contrib/extensions/browser/extensionsActions.ts ~line 1540) to hide or disable the button when no release version exists, matching the pattern already used by the install action.
