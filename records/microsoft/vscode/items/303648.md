# Issue #303648 — Chat renderer: subagent file edit indicator duplicated, replaces code blocks in main agent output

[https://github.com/microsoft/vscode/issues/303648](https://github.com/microsoft/vscode/issues/303648)

## Summary

Fixed on main and shipped in 1.118.0. The duplicate file-edit indicator was caused by the tool invocation widget remaining visible after the onFileEdits callback already rendered edit pills; commit e85baae4d67 hides the tool widget when file edits are present.

## Change summary

Subagent file edit indicators render twice — once correctly inside the subagent section, and again as a duplicate in the main agent output, replacing code blocks.

## Best solution

Keep the shipped fix from commit e85baae4d67 (1.118.0). No further change needed — the tool invocation widget is now hidden when file edits are shown via the separate onFileEdits path.
