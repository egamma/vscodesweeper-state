# Issue #323964 — .gitignore will prevent an extension from launching.

[https://github.com/microsoft/vscode/issues/323964](https://github.com/microsoft/vscode/issues/323964)

## Summary

Confirmed in source: `checkGlobFileExists` in `workspaceContains.ts` builds a file-search query without `disregardIgnoreFiles: true`, so the search service honors `.gitignore` and may fail to find activation-event target files. This is a real bug — activation events should not be subject to ignore-file filtering.

## Change summary

Extensions using `workspaceContains:**/folder/file` activation events fail to activate when the target path is covered by `.gitignore`, because the underlying file search respects ignore files.

## Best solution

Pass `disregardIgnoreFiles: true` (and likely `disregardGlobalIgnoreFiles: true`) in the `checkGlobFileExists` query-builder options so that `workspaceContains` glob searches always find files regardless of `.gitignore` patterns. The fix is in `src/vs/workbench/services/extensions/common/workspaceContains.ts`.
