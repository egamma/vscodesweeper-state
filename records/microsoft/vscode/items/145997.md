# Issue #145997 — Revisit storing entire search results in memento

[https://github.com/microsoft/vscode/issues/145997](https://github.com/microsoft/vscode/issues/145997)

## Summary

Partially addressed but still open debt: the worst offender (match ranges stored in memento) was fixed in c89ef733272, but SearchEditorInput still uses Memento/IStorageService for search config persistence (line 111, 164 of searchEditorInput.ts). The broader suggestion to move persistence to the file service remains unimplemented.

## Change summary

Move search editor persistence out of the workspace storage DB (Memento) and onto disk via the file service, as was done for test results.

## Best solution

Migrate the remaining Memento usage in `src/vs/workbench/contrib/searchEditor/browser/searchEditorInput.ts` (search config at line 164) to file-based persistence via IFileService, following the pattern established by test result storage (`src/vs/workbench/contrib/testing/common/testResultStorage.ts`). The search config stored per editor is small, so the urgency is lower than the original 1 MB match-ranges problem, but it's still using the storage service outside its intended simple key/value design.
