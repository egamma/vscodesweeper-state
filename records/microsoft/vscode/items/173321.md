# Issue #173321 — [Bug] VSCode.dev - Releasing the mouse outside of the editor area triggers a `click` event in Firefox

[https://github.com/microsoft/vscode/issues/173321](https://github.com/microsoft/vscode/issues/173321)

## Summary

This is a known Firefox browser behavior difference where `click` events fire on the mouseup target even when mousedown was elsewhere. It's an upstream quirk, not a VS Code-specific bug. No fix or workaround has been applied on main. Keeping open as upstream.

## Change summary

On Firefox, releasing the mouse button outside the editor area after starting a selection inside it triggers a spurious click event on the element under the cursor at mouseup.

## Best solution

This is upstream Firefox behavior (click dispatched to mouseup target regardless of mousedown target). A workaround could suppress clicks that follow a cross-element drag by tracking mousedown origin in `src/vs/base/browser/dom.ts` or the activity bar/sidebar, but it risks side effects. Alternatively, wait for Firefox to align with Chrome/Safari behavior.
