# Issue #152910 — Conflicting File Nesting Patterns

[https://github.com/microsoft/vscode/issues/152910](https://github.com/microsoft/vscode/issues/152910)

## Summary

Still reproduces on main: combining child patterns that overlap (e.g. `${basename}.*` and `${basename}*.${extname}`) under a `*` parent creates mutual parent-child cycles in the nesting trie, causing `findAllRootAncestors` cycle detection to collapse all files to root level with no nesting applied. No fix has been made to the trie logic since the report.

## Change summary

Combining `${basename}.*` and `${basename}*.${extname}` as child patterns under `*` in `explorer.fileNesting.patterns` breaks all file nesting — no patterns are applied.

## Best solution

Fix the cycle resolution in `ExplorerFileNestingTrie.nest()` in `src/vs/workbench/contrib/files/common/explorerFileNestingTrie.ts`. The `findAllRootAncestors` logic needs to handle mutual parent-child relationships more gracefully — e.g. by prioritizing the more specific pattern match or by breaking ties so that overlapping child patterns don't cause every file to collapse to a root.
