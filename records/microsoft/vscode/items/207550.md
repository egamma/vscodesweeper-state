# Issue #207550 — [Bug] MacOS: Multiline edit not functional in Safari, works well in Chrome

[https://github.com/microsoft/vscode/issues/207550](https://github.com/microsoft/vscode/issues/207550)

## Summary

This is an upstream Safari/WebKit limitation — Safari intercepts Cmd+Option+Arrow shortcuts at the browser level and does not dispatch them to web pages. The keybinding is correctly registered (Cmd+Alt+UpArrow for 'Add Cursor Above' in src/vs/editor/contrib/multicursor/browser/multicursor.ts:55), but Safari never delivers the keydown event. No code fix on VS Code's side can resolve this without a workaround (e.g. alternate keybinding for Safari).

## Change summary

Cmd+Option+Up/Down Arrow to add cursors above/below does not work in Safari because Safari intercepts the shortcut.

## Best solution

Either document the Safari limitation and suggest an alternative keybinding, or detect Safari and register a different default keybinding for 'Add Cursor Above/Below'. The root cause is WebKit not passing Cmd+Option+Arrow to the page.
