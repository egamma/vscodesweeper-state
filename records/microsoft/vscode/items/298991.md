# Issue #298991 — New Session based on worktree should let me start with at a remote tracking branch

[https://github.com/microsoft/vscode/issues/298991](https://github.com/microsoft/vscode/issues/298991)

## Summary

Valid feature request from a team member. The branch picker in the Sessions window only queries `refs/heads` (local branches), so remote tracking branches like `origin/main` are not available as starting points for worktree-based sessions. Keeping open per team-author rule.

## Change summary

Allow selecting remote tracking branches (e.g. `origin/main`) as the base branch when creating a worktree-based session in the Agents window.

## Best solution

Extend `_loadBranches` in `copilotChatSessionsProvider.ts` to also query `refs/remotes` (or at least offer them as an option in the branch picker). The branch picker UI in `branchPicker.ts` already supports arbitrary branch names, so the main change is in the data-fetching call at line ~400 of `copilotChatSessionsProvider.ts`. Consider whether remote refs should be shown in a separate group or merged with local branches.
