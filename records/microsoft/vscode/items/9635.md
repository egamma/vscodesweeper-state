# Issue #9635 — Problem matchers for error messages that span multiple lines

[https://github.com/microsoft/vscode/issues/9635](https://github.com/microsoft/vscode/issues/9635)

## Summary

Still a valid open feature request with 28 upvotes — variable-length multi-line message capture in problem matchers remains unimplemented on current main. The existing `appendProperty` mechanism only handles a fixed number of pattern lines, and the `loop` workaround creates separate problems per line. The programmatic problem matching API (issue #59337) is the acknowledged solution path but also remains open.

## Change summary

Allow problem matchers to capture error messages that span a variable number of lines into a single combined diagnostic message.

## Best solution

Implement either an `append` mode for pattern lines (accumulating into one message until a terminator pattern matches) or land the programmatic problem matching API (#59337) so extensions can handle arbitrary multi-line parsing in code. The relevant code is in `src/vs/workbench/contrib/tasks/common/problemMatcher.ts` around the `appendProperty`/`fillProblemData` logic and the pattern-stepping state machine.
