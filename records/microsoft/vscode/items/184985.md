# Issue #184985 — Manual linked editing doesn't seem to work

[https://github.com/microsoft/vscode/issues/184985](https://github.com/microsoft/vscode/issues/184985)

## Summary

Still reproduces on main: manual 'Start Linked Editing' command calls updateRanges(true) which shows decorations, but when the setting is off, reinitialize() never registers the onDidChangeModelContent listener needed to sync edits between linked ranges.

## Change summary

Running 'Start Linked Editing' manually shows linked range decorations but typing doesn't update the paired tag because the content-change listener is not registered when the setting is disabled.

## Best solution

When updateRanges is called with force=true (manual invocation), ensure the content-change and cursor-position listeners from reinitialize() are also registered in _localToDispose, even if the linkedEditing setting is off. The fix is localized to linkedEditing.ts — either reinitialize should accept a 'manual' flag that skips the setting check, or the action's run() should temporarily enable the listeners.
