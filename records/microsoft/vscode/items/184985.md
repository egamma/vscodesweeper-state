# Issue #184985 — Manual linked editing doesn't seem to work

[https://github.com/microsoft/vscode/issues/184985](https://github.com/microsoft/vscode/issues/184985)

## Summary

Still reproducible on main: when linked editing is manually triggered via the command, the content-change listeners that drive `_syncRanges` are never registered because `reinitialize()` only sets them up when the setting is enabled.

## Change summary

Manually invoking 'Start Linked Editing' shows decoration ranges but typing does not sync changes to the paired tag because the required listeners are gated on the editor setting.

## Best solution

When `updateRanges(force=true)` succeeds in establishing ranges, ensure the `onDidChangeModelContent` and `onDidChangeCursorPosition` listeners are also registered (even when the editor setting is off), so that the sync logic actually fires. The fix is localized to `linkedEditing.ts`.
