# Issue #197881 — [languages] contribution point `languages` provide the ability to extends

[https://github.com/microsoft/vscode/issues/197881](https://github.com/microsoft/vscode/issues/197881)

## Summary

This feature request for an `extend` property on the `languages` contribution point has not been implemented. It's in the Backlog milestone with no visible upvote traction, but it describes a real gap in language inheritance/reuse.

## Change summary

Allow a contributed language to inherit grammar, configuration, and tokenization from an existing language via an `extend` property.

## Best solution

Consider adding an `extend` field to `IRawLanguageExtensionPoint` in `src/vs/workbench/services/language/common/languageService.ts` that copies grammar/config from the referenced language. Alternatively, this could remain an extension-level concern (extensions already can register TextMate grammars for new language IDs pointing to existing scope names). Needs a product decision on whether core should own language inheritance.
