[https://github.com/microsoft/vscode/issues/197881](https://github.com/microsoft/vscode/issues/197881)

## Summary

Legitimate feature request for a language contribution point `extend` field, allowing a new language ID to inherit grammar/configuration from an existing one. Placed in Backlog by a maintainer, so it was accepted at some point; no community traction but no reason to close.

## Change summary

Allow the `languages` contribution point to declare a new language that inherits grammar and configuration from an existing language via an `extend` field.

## Best solution

If pursued, add an optional `extends` property to the language contribution point schema (in `src/vs/workbench/services/language/common/languageService.ts` and the extension manifest schema) that copies grammar, configuration, and folding from the referenced language. Alternatively, this could be considered `*out-of-scope` if aliases or file-association overrides are deemed sufficient — but that's a product decision for the area owner.
