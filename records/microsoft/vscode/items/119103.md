# Issue #119103 — Type ahead should turn off temporarily when it fails on a line

[https://github.com/microsoft/vscode/issues/119103](https://github.com/microsoft/vscode/issues/119103)

## Summary

Still valid on current main: the type-ahead addon has no per-line failure backoff — when a prediction fails, it continues predicting on the same line instead of temporarily disabling. The feature is now off by default (preview), reducing impact, but the bug remains for users who enable it.

## Change summary

Terminal type-ahead (local echo) should temporarily stop predicting on the current line after a prediction failure, instead of continuing to fail repeatedly.

## Best solution

Add per-line failure tracking to `TypeAheadAddon._onUserData` in `src/vs/workbench/contrib/terminalContrib/typeAhead/browser/terminalTypeAheadAddon.ts`: after `onPredictionFailed` fires, suppress new predictions for the remainder of that line (until the next newline/Enter). The `_lastRow` tracking already exists and could be extended with a `failedOnLine` flag.
