# Issue #249244 — Preserve Scroll Position When Zoom Level Changes

[https://github.com/microsoft/vscode/issues/249244](https://github.com/microsoft/vscode/issues/249244)

## Summary

Valid feature request for zoom-to-cursor behavior (preserving scroll position relative to mouse when zooming). No implementation exists on main. Parked in Backlog with zero upvotes and no community traction after 400+ days — keeping open for the community process.

## Change summary

When zooming in/out, center the zoom around the current mouse position so the content under the cursor stays visually stable.

## Best solution

This would require intercepting the zoom-level change in the workbench and computing a scroll offset adjustment to keep the content under the cursor stable. Since VS Code zoom is window-level (Electron webFrame), this is non-trivial — the entire page scales, not just the editor. A simpler scoped version could target only the editor viewport. Given zero community traction, this may naturally close via the community process if no votes arrive.
