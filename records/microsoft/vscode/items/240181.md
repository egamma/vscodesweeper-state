# Issue #240181 — Multi-file diff editor - support lazy text model loading

[https://github.com/microsoft/vscode/issues/240181](https://github.com/microsoft/vscode/issues/240181)

## Summary

Valid, team-acknowledged feature request for lazy text model loading in the multi-file diff editor. hediet confirmed the technical approach (~2-3 days) but it hasn't been implemented yet on main. Properly triaged and should stay open.

## Change summary

Multi-file diff editor loads all text models upfront, causing freezes when opening commits with thousands of files.

## Best solution

Implement lazy text model loading in `src/vs/editor/browser/widget/multiDiffEditor/` so collapsed or off-screen resources don't resolve their models until needed. hediet estimated 2-3 days; the work lives in the multi-diff-editor widget layer.
