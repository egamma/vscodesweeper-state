# Issue #233806 — [html] False error of inline Javascript in HTML

[https://github.com/microsoft/vscode/issues/233806](https://github.com/microsoft/vscode/issues/233806)

## Summary

Still reproduces on main: the embedded JS extraction naively replaces `<!--` with `/* ` regardless of context (regex, string), breaking the JS syntax sent to TypeScript for validation.

## Change summary

Inline JavaScript in HTML containing `<!--` inside a regex or string literal produces false syntax errors because the embedded content extraction corrupts the JS source.

## Best solution

Make `updateContent` in `extensions/html-language-features/server/src/modes/embeddedSupport.ts` context-aware so it only replaces `<!--`/`-->` when they appear as standalone HTML-comment wrappers (typically at the start/end of the script block), not when they occur inside string literals or regex patterns.
