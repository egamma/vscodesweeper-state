# Issue #233806 — [html] False error of inline Javascript in HTML

[https://github.com/microsoft/vscode/issues/233806](https://github.com/microsoft/vscode/issues/233806)

## Summary

Still reproduces on main: the embedded JS content transform in `embeddedSupport.ts` naively replaces `<!--` with `/* ` regardless of context (regex, strings), breaking syntax validation for code containing those characters in non-comment positions.

## Change summary

Inline JavaScript in HTML shows false syntax errors when a regex or string literal contains `<!--`, because the HTML language server's content transform blindly replaces it with a block-comment open.

## Best solution

Make the `<!--` → `/* ` replacement in `updateContent()` context-aware so it only fires when `<!--` appears as a standalone HTML-compat comment marker at the start of a line (the legacy pattern), not inside string/regex literals. Alternatively, only replace when preceded by whitespace/line-start. Related issue https://github.com/microsoft/vscode/issues/171153 tracks the same root cause with `-->` inside block comments.
