# Issue #218248 — Dragging text upwards triggers scrolling far too early

[https://github.com/microsoft/vscode/issues/218248](https://github.com/microsoft/vscode/issues/218248)

## Summary

Still reproduces on main: the DnD controller's `showAt` method calls `revealPosition` with source `'api'`, which applies `cursorSurroundingLines`/sticky-scroll padding when computing scroll thresholds — causing upward drag to trigger scrolling much earlier than downward drag (which follows a different path).

## Change summary

Dragging selected text upward in the editor triggers viewport scrolling far too early (around the midpoint) compared to dragging downward (which correctly waits until near the edge).

## Best solution

In `DragAndDropController.showAt` (src/vs/editor/contrib/dnd/browser/dnd.ts), avoid calling `revealPosition` for the vertical scroll — instead, replicate the scroll-on-edge logic used by the non-DnD selection-drag path, or pass a source/flag that suppresses the `cursorSurroundingLines` padding on the upward side. Alternatively, use `revealPositionInCenterIfOutsideViewport` or a minimal-reveal variant that only scrolls when the position is truly outside the viewport without padding.
