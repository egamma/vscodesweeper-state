# Issue #214277 — Disable notebook chat hint permanently when the user has triggered ctrl+i

[https://github.com/microsoft/vscode/issues/214277](https://github.com/microsoft/vscode/issues/214277)

## Summary

Milestone is 'On Deck' (a real scheduling milestone), so this must stay open per policy. The request — permanently dismiss the notebook cell editor hint after the user first triggers Ctrl+I — is a valid UX polish item that parallels the already-closed terminal equivalent (#213820).

## Change summary

Permanently hide the notebook empty-cell chat hint once the user has triggered inline chat (Ctrl+I) at least once.

## Best solution

In `src/vs/workbench/contrib/notebook/browser/contrib/editorHint/emptyCellEditorHint.ts`, wire the `onWillStartSession` handler to persist a 'hidden' state (via the existing `workbench.editor.empty.hint` setting or a notebook-scoped equivalent) so the hint never reappears after the user's first inline-chat invocation — mirroring the approach taken for the terminal hint in #213820.
