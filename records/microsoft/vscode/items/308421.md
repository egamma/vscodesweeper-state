# Issue #308421 — Black background after space in terminal

[https://github.com/microsoft/vscode/issues/308421](https://github.com/microsoft/vscode/issues/308421)

## Summary

The primary reported symptom (black background after spaces in PowerShell) is caused by an outdated PSReadLine module — multiple commenters confirmed that updating PSReadLine resolves it, and Microsoft's own Windows Terminal docs describe the same root cause. However, PR #312119 is open addressing a real secondary issue: the xterm composition view uses a hardcoded `#000` background instead of the terminal theme color, which is a valid VS Code bug worth fixing. Keeping open because the PR is in flight.

## Change summary
