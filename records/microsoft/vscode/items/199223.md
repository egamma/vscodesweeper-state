# Issue #199223 — Using 'function' in a Lua string causes wrong indentation

[https://github.com/microsoft/vscode/issues/199223](https://github.com/microsoft/vscode/issues/199223)

## Summary

Still reproduces on main: the Lua `increaseIndentPattern` regex doesn't skip keywords inside string literals, causing false-positive indentation. The fix PR #230632 was closed without merging. The `tree-sitter-autoindent-candidate` label suggests this may be addressed by future tree-sitter indentation.

## Change summary

Lua auto-indentation incorrectly triggers when keywords like `function`, `then`, `do` appear inside string literals.

## Best solution

Either improve the `increaseIndentPattern` regex in `extensions/lua/language-configuration.json` to consume quoted strings before matching keywords (as proposed in the comments), or wait for tree-sitter-based indentation to handle this properly. The comment thread has a well-tested regex solution ready to apply.
