# Issue #317167 — Reduce frequency of rate limit messages, they give me anxiety

[https://github.com/microsoft/vscode/issues/317167](https://github.com/microsoft/vscode/issues/317167)

## Summary

Valid feature request to reduce the noisiness of session/weekly rate-limit notifications. The current threshold-crossing logic (50/75/90/95%) already gates when a notification fires, but the previous-percent baseline is in-memory only — a window reload re-crosses every already-passed threshold, causing repeat notifications the user describes. The comment also suggests a time-aware trajectory heuristic for weekly/monthly limits.

## Change summary

Rate limit notification messages re-appear too frequently (e.g. on every window reload after crossing 50%), causing unnecessary anxiety.

## Best solution

Persist the last-acknowledged rate-limit threshold per window type (session/weekly) in storage — similar to the `QUOTA_EXHAUSTED_DISMISSED_STORAGE_KEY` pattern already used for quota-exhausted notifications — so that a window reload does not re-trigger a notification for a threshold the user has already seen and dismissed. The user's trajectory suggestion (only notify if usage pace exceeds the expected pace for the period) already has a partial analog in `_computeQuotaTrajectoryWarning` for monthly quota; extending that concept to weekly rate limits would further reduce noise. Work lives in `src/vs/workbench/contrib/chat/browser/chatQuotaNotification.ts`.
