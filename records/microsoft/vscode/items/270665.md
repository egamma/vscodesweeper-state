# Issue #270665 — Refactor noValidationMessage to be prompt

[https://github.com/microsoft/vscode/issues/270665](https://github.com/microsoft/vscode/issues/270665)

## Summary

Team-authored debt item requesting a rename of the internal `noValidationMessage` field to `prompt` in the QuickInput base class. The refactoring has not been done — the field still exists on current main, though `prompt` getter/setter accessors were added in PR #270497.

## Change summary

Rename the internal `noValidationMessage` field to `prompt` in the QuickInput base class and have each implementer handle the default appropriately.

## Best solution

Rename `noValidationMessage` to a `_prompt` backing field in `src/vs/platform/quickinput/browser/quickInput.ts`, remove the static `noPromptMessage` default from the base class, and let `InputBox` set its own default while `QuickPick`/`QuickTree`/`QuickWidget` leave it `undefined`.
