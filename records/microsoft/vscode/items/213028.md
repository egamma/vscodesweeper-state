# Issue #213028 — Git - Clone and open a repository using 'code' command 

[https://github.com/microsoft/vscode/issues/213028](https://github.com/microsoft/vscode/issues/213028)

## Summary

Valid backlog feature request with 21 upvotes. Not yet implemented — the CLI has no logic to detect a git URL argument and trigger clone+open. The protocol handler (`vscode://vscode.git/clone?url=...`) exists but doesn't cover the requested `code <url>` shorthand.

## Change summary

Allow `code <git-repo-url>` on the CLI to clone the repository and open the resulting folder.

## Best solution

Implement URL detection in the CLI argument parser (likely `src/vs/code/node/cliProcessMain.ts` or equivalent) that recognizes git-hosting URLs and delegates to `git.clone` with an auto-open flag. Alternatively, expose the existing protocol handler path more conveniently via `code --open-url 'vscode://vscode.git/clone?url=...'` and document it, though the user's ask is the simpler `code <url>` form.
