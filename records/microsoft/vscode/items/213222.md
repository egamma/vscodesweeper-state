# Issue #213222 — "Add selection to next find match" (cmd+d) is selecting also inlay hint

[https://github.com/microsoft/vscode/issues/213222](https://github.com/microsoft/vscode/issues/213222)

## Summary

Team-authored issue, still reproducible on main. Inlay hints are injected text decorations attached to the same range as the parameter; when Cmd+D selects the word, the selection highlight visually covers the injected text. No fix has landed.

## Change summary

Cmd+D (Add Selection to Next Find Match) visually selects/highlights the inlay hint text alongside the actual word when the cursor is on a parameter with a preceding inlay hint.

## Best solution

The selection highlight rendering in the view layer needs to exclude injected text from the visual selection rectangle. The fix likely involves changes in the selection rendering (`src/vs/editor/browser/viewParts/selections/`) or the view-line rendering to not extend the highlight background over injected text spans that aren't part of the model content.
