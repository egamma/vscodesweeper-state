# Issue #225317 — 1.92 broke my exension "The task '<task name>' is already active"

[https://github.com/microsoft/vscode/issues/225317](https://github.com/microsoft/vscode/issues/225317)

## Summary

Still present on main: concurrent tasks sharing the same taskDefinition type are blocked by the instance-check logic in terminalTaskSystem.ts. The area owner acknowledged the report and workaround exists, but the underlying behavior hasn't been fixed.

## Change summary

Running multiple tasks concurrently that share the same taskDefinition type incorrectly yields 'already active' error because instance matching uses task key (type-based) rather than a unique per-execution identity.

## Best solution

The `_getInstances` logic in `terminalTaskSystem.ts` (line ~441-444) matches tasks by `task.getKey()` which is based on the task definition type. For extensions that spawn child tasks with the same type, the instance limit blocks execution. Either the key comparison needs to account for distinct task instances more granularly, or the `instanceLimit` default/behavior should allow concurrent same-type tasks from different sources.
