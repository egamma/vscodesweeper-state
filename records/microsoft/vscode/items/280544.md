# Issue #280544 — Agent sessions: recent sessions list flashes on first open of chat view

[https://github.com/microsoft/vscode/issues/280544](https://github.com/microsoft/vscode/issues/280544)

## Summary

Still reproduces on main: the sessions list visibility check in `updateSessionsControlVisibility()` doesn't account for an in-progress session restoration, causing the list to flash while the widget is transiently empty during startup.

## Change summary

The recent sessions list flashes briefly on first open of the chat view because the widget appears empty while a session is being restored.

## Best solution

Guard the stacked-mode visibility check at line 775 of `chatViewPane.ts` with `!this.restoringSession` so the sessions control stays hidden while a session is actively being restored. The `restoringSession` promise already tracks this state but isn't consulted during visibility updates.
