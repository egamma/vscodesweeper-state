# Issue #237947 — Tooltip for Close Button in Dialog Box is Obscured by the Dialog Itself

[https://github.com/microsoft/vscode/issues/237947](https://github.com/microsoft/vscode/issues/237947)

## Summary

Still reproduces on main: the dialog close button's hover tooltip renders behind the dialog due to z-index stacking context. An open PR (#237952) exists but hasn't been merged; keeping open for maintainer review of that PR.

## Change summary

The tooltip for the close button in modal dialogs is obscured by the dialog's own z-index, making it invisible to users.

## Best solution

Review and land (or close) the open PR https://github.com/microsoft/vscode/pull/237952, which proposes a z-index fix in dialog.css. The core issue is that the hover's ContextView (z-index 2576) competes with the dialog modal block (z-index 2575) — the fix likely needs to ensure the hover container is placed outside the dialog's stacking context or raise its z-index further.
