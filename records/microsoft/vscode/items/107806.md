# Issue #107806 — New search results cleared after search-and-replace operation finishes

[https://github.com/microsoft/vscode/issues/107806](https://github.com/microsoft/vscode/issues/107806)

## Summary

Still reproduces on main: the `replaceAll` flow in `PlainTextSearchHeadingImpl` calls `this.clear()` asynchronously after the replace service finishes (textSearchHeading.ts:301), which wipes any new search results the user started during the replace. No fix has landed.

## Change summary

New search results are cleared when a prior search-and-replace-all operation finishes, because the replace completion callback clears the shared heading.

## Best solution

Either disable triggering a new search while a replace-all is in progress (as the maintainer suggested), or guard the `replaceAll` completion callback so it does not call `clear()` when a new search has already started on the same heading. The fix lives in `src/vs/workbench/contrib/search/browser/searchTreeModel/textSearchHeading.ts` (the `replaceAll` method) and possibly `src/vs/workbench/contrib/search/browser/searchView.ts` (to block new queries during replace).
