# Issue #107806 — New search results cleared after search-and-replace operation finishes

Issue carries the 'On Deck' milestone (a real scheduling milestone), so it must remain open per policy. A team member has confirmed it and suggested an approach.

## Change summary

New search results are cleared when a prior search-and-replace operation finishes saving files, forcing the user to re-run the search.

## Best solution

Implement the approach roblourens suggested: prevent initiating a new search while a replace operation is still saving files, or ensure that completion of a prior replace does not clear the results of a subsequently-initiated search. The fix likely lives in the search model/view interaction in `src/vs/workbench/contrib/search/`.
