## Summary

Valid terminal rendering bug with Iosevka on Linux/WebGL — already has info-needed applied and the right area label. The reporter couldn't confirm whether it's a regression; the issue likely lives in xterm.js's WebGL glyph atlas cell-width measurement, but keeping open for the terminal team to investigate.

## Change summary

Iosevka font glyphs overlap in the integrated terminal on Linux when using the WebGL renderer, despite correct monospace metrics.

## Best solution

The terminal team should investigate whether xterm.js's WebGL renderer is mis-measuring cell width for Iosevka on Linux (the font reports correct spacing metadata per fc-scan). This may need an upstream fix in xterm.js's glyph atlas or a VS Code-side override. The info-needed label is appropriate to gather regression data.
