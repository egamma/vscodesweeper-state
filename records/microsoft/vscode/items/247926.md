# Issue #247926 — Pass LanguageModelChat to tools

[https://github.com/microsoft/vscode/issues/247926](https://github.com/microsoft/vscode/issues/247926)

## Summary

Team-authored API finalization issue. The `model` property on `LanguageModelToolInvocationOptions` remains in the proposed `chatParticipantAdditions` API and has not been finalized into `vscode.d.ts`. Active discussion on naming continues; downstream teams (Java Upgrade) are waiting on it.

## Change summary

Finalize the `model?: LanguageModelChat` property on `LanguageModelToolInvocationOptions` so tools can access the chat model that invoked them without relying on proposed APIs.

## Best solution

Finalize the API by moving the `model` property from `vscode.proposed.chatParticipantAdditions.d.ts` into `vscode.d.ts`, resolving the naming question (whether to split into separate invoke-side vs. invokee-side interfaces). The comment thread suggests `LanguageModelToolInvokeOptions` for the tool-received options. Area owner (roblourens) should drive the final naming decision and finalization PR.
