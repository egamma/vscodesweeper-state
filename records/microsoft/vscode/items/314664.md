# Issue #314664 — Unsent Copilot Chat input is lost after using "Compact Conversation"

[https://github.com/microsoft/vscode/issues/314664](https://github.com/microsoft/vscode/issues/314664)

## Summary

Still reproduces on main: the Compact Conversation action calls `widget.acceptInput('/compact')` which replaces the user's unsent input with the slash command and then clears the editor — the draft is lost with no save/restore. Routing to the chat area for a fix.

## Change summary

Clicking 'Compact Conversation' while unsent text is in the chat input discards the user's draft with no warning or recovery.

## Best solution

The `CompactAgentHostConversationAction.run()` should save the current editor value before calling `acceptInput('/compact')` and restore it after the compact request is sent. This is a small change in `chatActions.ts` (the action) or `chatWidget.ts` (`_acceptInput` should skip clearing the input when the query is an internal command like `/compact`).
