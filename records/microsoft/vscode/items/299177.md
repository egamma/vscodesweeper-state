# Issue #299177 — renderMermaidDiagram in dark mode styles the diagram with white theme assumption

[https://github.com/microsoft/vscode/issues/299177](https://github.com/microsoft/vscode/issues/299177)

## Summary

Valid polish issue: the rendering infrastructure already adapts Mermaid diagrams to the active VS Code theme (dark/light) via `VsCodeMermaidThemeTracker` (commit 80afb576177), but the `modelDescription` for the `renderMermaidDiagram` tool doesn't instruct the LM to avoid hardcoded inline colors — so the LM may generate Mermaid markup with light-theme-friendly style directives that override the automatic theming.

## Change summary

Mermaid diagrams generated by the `renderMermaidDiagram` tool contain LM-chosen inline colors that clash with dark themes, because the tool's model description doesn't tell the LM that theming is applied automatically.

## Best solution

Update the `modelDescription` and/or the `markup` input `description` in `extensions/mermaid-markdown-features/package.json` to instruct the LM not to include hardcoded colors or style directives, since the renderer automatically applies theme-appropriate styling derived from the active VS Code color theme.
