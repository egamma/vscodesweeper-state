# Issue #201093 — Join Lines should not include the line after the last fully selected line

[https://github.com/microsoft/vscode/issues/201093](https://github.com/microsoft/vscode/issues/201093)

## Summary

Keep open: the reported behavior still appears to be present on current main, and there is no maintainer decision, duplicate, or fixing commit in the provided issue context.

## Change summary

Join Lines includes the line where a full-line selection ends at column 1, so selecting three full lines by Shift+Down also joins the following line.

## Best solution

Decide the intended editor-command semantics for full-line selections in `JoinLinesAction.run`; if the report is accepted, adjust `src/vs/editor/contrib/linesOperations/browser/linesOperations.ts` so a non-empty selection ending at column 1 stops at the previous line, and add a focused test in `src/vs/editor/contrib/linesOperations/test/browser/linesOperations.test.ts`.
