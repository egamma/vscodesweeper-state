# Issue #298314 — Integrated terminal loses PATH in tcsh when Copilot/Copilot Chat are forced to UI via `remote.extensionKind`

[https://github.com/microsoft/vscode/issues/298314](https://github.com/microsoft/vscode/issues/298314)

## Summary

The reporter is using an unsupported configuration (`remote.extensionKind: {"github.copilot-chat": ["ui"]}`). A VS Code team member stated in the closely related issue #321603 that forcing extension kind to UI is not supported and "many things will not work correctly." The report needs clarification on whether the issue reproduces without this override, and what the underlying need is (likely running Copilot without remote internet access).

## Change summary

Terminal PATH is minimal (missing /bin, /usr/bin, etc.) in tcsh on Remote SSH when Copilot Chat is forced to UI extension kind.

## Best solution

Clarify whether this reproduces without the `remote.extensionKind` override. If not, this is the same unsupported-configuration issue as https://github.com/microsoft/vscode/issues/321603. The underlying user need (running Copilot on a remote host without internet) is tracked at https://github.com/microsoft/vscode/issues/292758. If the PATH problem reproduces in a default configuration with tcsh, that would be a separate actionable terminal bug.
