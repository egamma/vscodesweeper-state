# Issue #313651 — Agents app keeps signing out when launching EDH for my VS Code extension

[https://github.com/microsoft/vscode/issues/313651](https://github.com/microsoft/vscode/issues/313651)

## Summary

Team-authored bug reporting that launching an Extension Development Host from the Agents window causes both windows to lose authentication state, blocking agent-assisted extension development. Hard keep-open: author is MEMBER.

## Change summary

Launching an EDH from the Agents app causes the parent window to sign out, and re-signing in fails until the EDH is closed.

## Best solution

Investigate how shared auth state (likely secret storage or token caching) is being corrupted or overwritten when a second VS Code window (EDH) starts from the Agents app. The fix likely lives in the auth/session management layer around `src/vs/workbench/services/authentication/` or the Agents window's session handling in `src/vs/sessions/`. Ensure EDH launch does not reset or compete for the parent window's auth tokens.
