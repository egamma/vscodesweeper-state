# Issue #191391 — Extension Contributed Menu Item: resourceFilename does not work as expected

[https://github.com/microsoft/vscode/issues/191391](https://github.com/microsoft/vscode/issues/191391)

## Summary

Team-authored issue (hediet/MEMBER) — must stay open. The bug appears to still exist on current main: `ResourceContextKey` is bound to the view's scoped context key service (explorerView.ts:223), but the context menu is shown with `this.tree.contextKeyService` (explorerView.ts:673), which is a separate child scope. This mismatch can cause `resourceFilename` to be intermittently unavailable when evaluating extension-contributed `when` clauses.

## Change summary

Extension-contributed context menu items using `resourceFilename` in their `when` clause intermittently fail to appear in the explorer context menu.

## Best solution

The `ResourceContextKey` should be bound to the tree's context key service (or the same scoped service passed to `showContextMenu`), so that `resourceFilename` is always visible during when-clause evaluation. The likely fix is in `src/vs/workbench/contrib/files/browser/views/explorerView.ts` — either rebind `resourceContext` after tree creation using the tree's context key service, or pass the view's context key service (where `resourceContext` lives) to `showContextMenu`.
