# Issue #92978 — Report task execution result in onDidEndTask

[https://github.com/microsoft/vscode/issues/92978](https://github.com/microsoft/vscode/issues/92978)

## Summary

Still a valid feature request — `TaskEndEvent` on current main only carries `execution`, with no exit code or success/failure status. The related proposed API (`taskProblemMatcherStatus`) partially addresses this but hasn't been finalized. Keeping open as an unresolved API enhancement.

## Change summary

Add task execution result (exit code or success/failure status) to the `TaskEndEvent` fired by `onDidEndTask`.

## Best solution

Either extend `TaskEndEvent` with an `exitCode` property (as team members discussed in comments 2–3), or finalize the proposed `taskProblemMatcherStatus` API which provides `hasErrors`. The implementation site is `src/vs/workbench/api/browser/mainThreadTask.ts` where the event is fired.
