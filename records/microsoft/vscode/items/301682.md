# Issue #301682 — Premium models not visible even if enabled

[https://github.com/microsoft/vscode/issues/301682](https://github.com/microsoft/vscode/issues/301682)

## Summary

The available-model list in VS Code's chat UI is populated by the Copilot Chat extension (via the GitHub API), not by VS Code core. Model visibility is controlled by the extension's entitlement/API layer, so this belongs against the Copilot extension, not the VS Code repo.

## Change summary

Some premium models enabled on the user's GitHub account do not appear in VS Code's Copilot model picker.

## Best solution

File against the Copilot Chat extension (https://github.com/microsoft/vscode-copilot-release) or contact GitHub support. VS Code core provides the model-picker UI (`src/vs/workbench/contrib/chat/browser/widget/input/chatModelPicker.ts`) but the set of registered language models comes entirely from the Copilot extension's API calls; core has no role in filtering or gating models.
