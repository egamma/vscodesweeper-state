# Issue #323796 — Escaping a suggestions popup loops infinitely

[https://github.com/microsoft/vscode/issues/323796](https://github.com/microsoft/vscode/issues/323796)

## Summary

Valid bug: dismissing the suggestions popup with Escape causes it to immediately reappear, likely due to a timing race between `quickSuggestionsDelay` and `inlineSuggest.minShowDelay`. Multiple users confirm. Not yet fixed on main.

## Change summary

Pressing Escape to dismiss the autocomplete suggestions popup causes it to loop back immediately, requiring multiple dismissals or a click elsewhere.

## Best solution

Fix the re-trigger path in `suggestModel.ts` so that when the suggest widget is explicitly dismissed via Escape, the subsequent inline-completions state change does not re-trigger quick suggestions. The interaction between `_waitForInlineCompletionsAndTrigger` and the widget-hide flow likely needs to respect an explicit-dismiss signal that suppresses re-triggering until the next user input.
