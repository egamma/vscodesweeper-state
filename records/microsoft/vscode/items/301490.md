# Issue #301490 — Plugin-installed skill resource files not accessible to search tools (grep_search, file_search, semantic_search)

[https://github.com/microsoft/vscode/issues/301490](https://github.com/microsoft/vscode/issues/301490)

## Summary

Valid feature request: plugin-installed skill directories should be automatically included in search tool scope. The `additionalReadAccessPaths` setting exists as a manual workaround but automatic inclusion for activated skills has not been implemented.

## Change summary

Search tools (grep_search, file_search, semantic_search) cannot access resource files from plugin-installed skills because those directories are outside workspace scope.

## Best solution

When a skill is activated via a plugin or user profile, automatically add its directory to the search scope used by workspace-scoped tools. This likely belongs in the Copilot extension's tool implementation layer (extensions/copilot) where `additionalReadAccessPaths` is already handled, extending that mechanism to auto-include resolved skill directories.
