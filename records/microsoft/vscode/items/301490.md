# Issue #301490 — Plugin-installed skill resource files not accessible to search tools (grep_search, file_search, semantic_search)

[https://github.com/microsoft/vscode/issues/301490](https://github.com/microsoft/vscode/issues/301490)

## Summary

Legitimate feature request in the Backlog Candidates community process (0 upvotes, 60-day window started Jul 2). Requesting that plugin-installed skill directories be included in workspace search scope for grep_search/file_search/semantic_search tools. Keeping open per community process rules.

## Change summary

Search tools (grep_search, file_search, semantic_search) cannot access resource files from plugin-installed skills because those directories are outside the workspace scope.

## Best solution

When a plugin-installed skill is activated, automatically include its directory in the search scope for workspace-scoped tools. This would live in the Copilot Chat extension's tool execution layer where search paths are resolved. The referenced issue https://github.com/microsoft/vscode/issues/293386 (additionalReadAccessPaths) provides a manual workaround but automatic inclusion based on skill activation is the cleaner UX.
