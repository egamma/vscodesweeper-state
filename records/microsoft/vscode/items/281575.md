# Issue #281575 — SCM: Stashes, branches and tags show multiple times when using worktrees

[https://github.com/microsoft/vscode/issues/281575](https://github.com/microsoft/vscode/issues/281575)

## Summary

Still reproduces on main: each worktree opens as a separate Repository with its own GitArtifactProvider, and since all worktrees share the same git object store, branches/stashes/tags are listed identically under every repository entry in the SCM Repositories view. The author is a MEMBER and the issue carries an 'On Deck' milestone — keeping open.

## Change summary

When using git worktrees, the SCM Repositories view shows duplicate entries for stashes, branches, and tags because each worktree's artifact provider independently lists the shared refs.

## Best solution

Deduplicate shared artifact groups (branches, stashes, tags) across worktree repositories in the SCM Repositories view — either by showing them only on the main worktree's artifact provider, or by filtering out groups already surfaced by a sibling worktree repository. The logic likely belongs in `extensions/git/src/artifactProvider.ts` (per-repository dedup) or in `extensions/git/src/model.ts` (model-level coordination of which repositories expose which artifact groups).
