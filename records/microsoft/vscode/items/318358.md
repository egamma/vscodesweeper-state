# Issue #318358 — Enable use of local agent in Agents Window

[https://github.com/microsoft/vscode/issues/318358](https://github.com/microsoft/vscode/issues/318358)

## Summary

Keep open: this is an active Backlog Candidates feature request, and maintainers confirmed the Agents Window local harness story is intentionally incomplete while the Agent Host Protocol and Copilot CLI integration mature.

## Change summary

Allow the Agents Window to use the local harness with the same local model and custom agent flexibility available in Copilot Chat.

## Best solution

Track this until the Agents Window can connect to local sessions through the Agent Host Protocol/Copilot CLI path with parity for local LLMs and custom agents, or split any narrower model/agent-picker gaps into implementation issues owned by the agents-window/local-harness area.
