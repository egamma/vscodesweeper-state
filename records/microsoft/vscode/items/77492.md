# Issue #77492 — If pasting over text containing TextEditorDecorations, they are retained

[https://github.com/microsoft/vscode/issues/77492](https://github.com/microsoft/vscode/issues/77492)

## Summary

Team-authored feature request (MEMBER) — keeping open per policy. The request is for TextEditorDecorations to be cleared when text they cover is replaced by a paste operation; currently labeled as feature-request and semantic-tokens.

## Change summary

TextEditorDecorations persist on newly pasted text when pasting over decorated content, rather than being cleared.

## Best solution

The decoration model in the editor could strip or truncate decorations whose original range is fully replaced by an edit (paste). This likely touches decoration tracking logic in `src/vs/editor/common/model/trackedRanges.ts` or how decorations respond to replace-edits. Needs maintainer judgment on whether this is the desired default behavior for all decoration types or only specific ones (e.g., semantic tokens).
