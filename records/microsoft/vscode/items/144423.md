# Issue #144423 — suggest list hover view shouldn't appear border

[https://github.com/microsoft/vscode/issues/144423](https://github.com/microsoft/vscode/issues/144423)

## Summary

Still reproducible on main: when the suggest widget pops up under the cursor, the sash's mouseenter fires and shows a hover border despite no intentional user hover. The 300ms hoverDelay mitigates but doesn't prevent this because the cursor is already stationary over the sash.

## Change summary

The suggest widget's resize sash shows a hover highlight border when the widget appears under the cursor without intentional user hovering.

## Best solution

In Sash.onMouseEnter, detect whether the mouse actually moved onto the sash (vs. the sash appearing under a stationary cursor) — e.g. by comparing the last known pointer position at the time the sash becomes visible and suppressing the hover feedback if the pointer didn't move. The fix would live in src/vs/base/browser/ui/sash/sash.ts.
