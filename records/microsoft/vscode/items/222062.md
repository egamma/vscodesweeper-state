# Issue #222062 — In code-server, disconnected processes eventually block forever on stdout/stderr PTY buffer

[https://github.com/microsoft/vscode/issues/222062](https://github.com/microsoft/vscode/issues/222062)

## Summary

Still present on main: the flow control mechanism in terminalProcess.ts pauses the PTY when unacknowledged chars exceed the high watermark, and when the browser disconnects in code-server there is no client to send acknowledgments, so the PTY stays paused indefinitely and the child process blocks on stdout/stderr. Confirmed by a team member on 1.96.0.

## Change summary

In code-server mode, disconnecting the browser while a process writes heavily to stdout/stderr causes the process to hang because the PTY flow control pauses with no client to acknowledge data.

## Best solution

When a persistent terminal session detects that the client has disconnected (no active replay listener), it should periodically drain or discard unacknowledged data and call `clearUnacknowledgedChars()` / `resume()` on the PTY so the child process is not blocked. This logic belongs in `PersistentTerminalProcess` in `src/vs/platform/terminal/node/ptyService.ts`, where the serializer already buffers scrollback — the unacknowledged char count should be reset or the PTY should remain unpaused while no client is connected.
