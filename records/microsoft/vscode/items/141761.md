# Issue #141761 — Some default shortcuts don't work with non-english layout: `cmd + .`, etc

[https://github.com/microsoft/vscode/issues/141761](https://github.com/microsoft/vscode/issues/141761)

## Summary

Keep open as an active keyboard-layout design discussion in Backlog; the request has known tradeoffs and no concrete fixing commit or canonical duplicate close signal. A maintainer explicitly rejected a blanket physical-key rewrite while suggesting targeted heuristics or layout-specific defaults as possible directions.

## Change summary

Make default keybindings such as Cmd+. and bracket shortcuts work reliably on non-English layouts without users adding per-layout overrides.

## Best solution

Resolve this in the keyboard-layout/keybinding mapper area by deciding between a targeted fallback for layouts with missing Latin/punctuation keys and the broader layout-specific defaults tracked at https://github.com/microsoft/vscode/issues/1240, while preserving expected German, French, and Dvorak shortcut behavior.
