# Issue #141761 — Some default shortcuts don't work with non-english layout: `cmd + .`, etc

[https://github.com/microsoft/vscode/issues/141761](https://github.com/microsoft/vscode/issues/141761)

## Summary

This is a known, complex keyboard-layout problem that a team member acknowledged and linked to the canonical tracking issue #1240. The issue remains under discussion with no shipped resolution; it should stay open as-is.

## Change summary

Default keybindings using character symbols (like `.`) don't fire on non-English layouts where the same physical key produces a different character.

## Best solution

This is partially tracked by https://github.com/microsoft/vscode/issues/1240 (keyboard-layout-specific default keybindings). The two issues overlap but aren't identical — #1240 is the broader framework request while this one highlights specific broken defaults. A team member's proposed heuristic (fallback scan codes when no key produces the expected character) would address the Russian-layout case described here. Keep open under `keyboard-layout` for the area owner to decide whether to absorb into #1240 or tackle independently.
