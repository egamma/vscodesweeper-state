# Issue #67566 — Revealing in the editor should be aware of how much space the command palette is using

[https://github.com/microsoft/vscode/issues/67566](https://github.com/microsoft/vscode/issues/67566)

## Summary

Still unimplemented on main — revealRangeInCenter does not account for the quick pick overlay height, so symbols revealed while navigating via the command palette can be hidden behind it. Keeping open as it's a valid UX improvement with team engagement.

## Change summary

When navigating symbols via the quick pick, the revealed line can be hidden behind the quick pick overlay because revealRange doesn't account for the overlay's height.

## Best solution

Modify the symbol-reveal path (e.g. in `anythingQuickAccess.ts` and the go-to-symbol quick access) to offset the reveal target below the quick pick's bottom edge, or introduce a general 'reveal padding top' concept in the editor that the quick pick can set while open.
