# Issue #161088 — Git - Stop git fsmonitor--daemon beforce closing / renaming / deleting a repo

[https://github.com/microsoft/vscode/issues/161088](https://github.com/microsoft/vscode/issues/161088)

## Summary

Still reproduces on main: the git extension's Repository.dispose() does not stop the fsmonitor daemon, so git-spawned fsmonitor processes linger after VS Code closes a workspace, blocking folder rename/delete on Windows. No fix has been committed.

## Change summary

VS Code should run `git fsmonitor--daemon stop` when closing or removing a repository to prevent lingering daemon processes that lock the repo directory.

## Best solution

In the built-in git extension's `Repository.dispose()` (extensions/git/src/repository.ts), call `git fsmonitor--daemon stop <repo-path>` before releasing the repository, so the daemon is cleaned up when VS Code closes a workspace or removes a repo from its model. The fix is small and localized to the dispose path.
