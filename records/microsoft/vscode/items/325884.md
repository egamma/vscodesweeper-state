# Issue #325884 — Agent Host: editing a steering message when it's in a pending state causes all versions to be sent

[https://github.com/microsoft/vscode/issues/325884](https://github.com/microsoft/vscode/issues/325884)

## Summary

Team-filed bug — keeping open. The issue is plausible from source: each edit of a pending steering message dispatches `ChatPendingMessageSet`, which triggers `_syncPendingMessages` → `setPendingMessages` → `sendSteering`. If each edit generates a new pending message ID, the dedup guard (`_steeringMessagesInFlight.has(id) || _pendingSteeringFlips.has(id)`) lets every revision through to the agent SDK.

## Change summary

Editing a pending steering message multiple times causes all revisions to be sent to the agent instead of only the final one.

## Best solution

The `sendSteering` path in `copilotAgentSession.ts` (and the analogous `setPendingMessages` in `codexAgent.ts` / `claudeAgent.ts`) should either coalesce rapid edits (debounce or replace-in-flight) or cancel a previously-buffered-but-unsent steering message when a new one arrives with a different ID. The dedup guard currently keys on `steeringMessage.id`, so if edits produce new IDs, each passes the guard independently.
