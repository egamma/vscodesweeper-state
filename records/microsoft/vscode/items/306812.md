# Issue #306812 — White flash when reloading window & using Dark theme

[https://github.com/microsoft/vscode/issues/306812](https://github.com/microsoft/vscode/issues/306812)

## Summary

Confirmed Electron limitation: BrowserWindow backgroundColor set at creation time briefly flashes on reload when the system theme has changed since window creation. Maintainer (aeschli) acknowledged this in comments. Keeping open as an upstream/polish issue.

## Change summary

Reloading the VS Code window after switching from light to dark system theme causes a white flash because the window was created with a light background.

## Best solution

This is an Electron limitation — the BrowserWindow backgroundColor set at creation persists through navigations/reloads. The best path is either (1) recreating the window on reload when the stored theme differs from the window's creation-time color, or (2) tracking upstream Electron progress on allowing dynamic backgroundColor updates during navigation. The code in `src/vs/platform/theme/electron-main/themeMainServiceImpl.ts` already calls `setBackgroundColor` on splash save, but the reload gap remains.
