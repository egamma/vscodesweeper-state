# Issue #103330 — Improve debug actions for multi-target debugging

[https://github.com/microsoft/vscode/issues/103330](https://github.com/microsoft/vscode/issues/103330)

## Summary

Legitimate feature request for bulk debug actions across multiple sessions in multi-target debugging. No implementation progress found on main, no canonical duplicate, and only 2 upvotes — keeping open in the Backlog for community traction or a maintainer decision on whether this belongs in core or an extension.

## Change summary

Add the ability to execute debug actions (continue, step, etc.) on all or a subset of debug sessions simultaneously during multi-target debugging.

## Best solution

A maintainer should decide whether this belongs in core (e.g. multiselect in the callstack view as roblourens suggested) or as an extension. The DAP protocol's `ContinueRequest` already supports `allThreadsContinued` within a session, but cross-session bulk actions would need new UX in `src/vs/workbench/contrib/debug/browser/` (callstack view or debug toolbar). If kept as a core feature, multiselect in the callstack view with bulk-action toolbar buttons is the natural path. Otherwise, the debug extension API may already be sufficient for an extension to implement this.
