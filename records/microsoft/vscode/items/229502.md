# Issue #229502 — SCM Graph - Picked repository is not remembered after restart

[https://github.com/microsoft/vscode/issues/229502](https://github.com/microsoft/vscode/issues/229502)

## Summary

Still reproduces on main: the selected repository in the SCM Graph view is stored only in an in-memory observable (`_selectedRepository`) and is never persisted to `IStorageService`, so it resets to 'auto' on every reload.

## Change summary

The SCM Graph view does not persist the user's chosen repository across window reloads, always reverting to 'auto'.

## Best solution

Persist `_selectedRepository` to `IStorageService` (workspace scope) alongside the existing `scm.graphView.viewMode` and `scm.graphView.referencesFilter` keys, restoring it on construction by matching the stored provider key against available repositories.
