# Issue #230089 — `editor.fontLigatuers`  doesn't wotk if the string is long

[https://github.com/microsoft/vscode/issues/230089](https://github.com/microsoft/vscode/issues/230089)

## Summary

This is an upstream Chromium limitation — the CSS parser silently ignores `font-feature-settings` values past an internal length limit (~1024 chars). VS Code's code passes the string through without truncation, so there is nothing to fix on the VS Code side.

## Change summary

Long `editor.fontLigatures` strings are silently truncated by Chromium's CSS parser, preventing font features from applying.

## Best solution

Classify as upstream (Chromium bug). VS Code cannot work around the browser's CSS parser limit for `font-feature-settings`. The user's workaround is to shorten the string or minimize whitespace. A potential VS Code mitigation could strip unnecessary whitespace before applying the CSS, but the real fix belongs in Chromium.
