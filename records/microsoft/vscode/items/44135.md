# Issue #44135 — [json] schema fileMatch for all *.json files in root folder

[https://github.com/microsoft/vscode/issues/44135](https://github.com/microsoft/vscode/issues/44135)

## Summary

Still a valid open feature request. The JSON schema fileMatch patterns still lack workspace-root-relative anchoring on current main — the folderUri is passed through but doesn't enable a glob like '/*.json' to mean 'only root-level files'.

## Change summary

Allow JSON schema fileMatch patterns to target only files in the workspace root folder, not recursively matching all subdirectories.

## Best solution

Support a workspace-folder-relative prefix (e.g. `${workspaceFolder}` or proper glob semantics where `/*.json` means root-only) in the `vscode-json-languageservice` matching logic. The fix would involve changes in both the VS Code JSON extension client (pattern normalization) and upstream in `vscode-json-languageservice` (pattern matching). The `help wanted` label is appropriate.
