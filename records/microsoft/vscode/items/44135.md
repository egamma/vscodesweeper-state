# Issue #44135 — [json] schema fileMatch for all *.json files in root folder

[https://github.com/microsoft/vscode/issues/44135](https://github.com/microsoft/vscode/issues/44135)

## Summary

Still a valid open feature request — fileMatch patterns in json.schemas cannot be anchored to the workspace root. Glob support was added in v1.56 but root-relative matching (e.g. via ${workspaceFolder}) remains unimplemented, as confirmed by maintainer @aeschli in related issues.

## Change summary

Users cannot restrict json.schemas fileMatch patterns to only files in the workspace root folder; patterns like `/*.json` still match files in subdirectories.

## Best solution

Add workspace-root-relative variable expansion (e.g. `${workspaceFolder}`) to fileMatch pattern resolution in the JSON language service client, so patterns can be anchored to the workspace root rather than matching against all file paths suffix-style.
