## Summary

Team-authored accessibility/keyboard-navigation bug in the multi diff editor — toolbar actions receive focus without being scrolled into view. No fix found on current main; hard keep-open applies (MEMBER author).

## Change summary

Focusing the Stage Changes toolbar action via Shift+Tab in the multi diff editor does not scroll the element into the visible viewport.

## Best solution

When focus moves to a toolbar action in `diffEditorItemTemplate.ts`, the header row should be scrolled into view. A `focusin` listener on the actions container that calls `scrollIntoView` (or the widget's `reveal` method) would fix this — likely in `src/vs/editor/browser/widget/multiDiffEditor/diffEditorItemTemplate.ts` or the widget impl's scroll management.
