# Issue #313251 — Collapse folders in Incoming Changes in tree view for many files

[https://github.com/microsoft/vscode/issues/313251](https://github.com/microsoft/vscode/issues/313251)

## Summary

Valid feature request: in the Source Control Graph view's tree mode, folder nodes within incoming changes are expanded by default (collapseByDefault returns false for resource tree nodes). The user wants them collapsed by default or a setting to control this, which is particularly painful in large monorepos.

## Change summary

Request to collapse folder nodes by default in the Incoming Changes tree view, or provide a setting to control this behavior.

## Best solution

Add a setting (e.g. `scm.graph.collapseChangeFolders`) or change the default in `scmHistoryViewPane.ts` so that resource tree nodes (folders) within incoming/outgoing change expansions start collapsed. The relevant logic is the `collapseByDefault` callback at line ~2012 of `src/vs/workbench/contrib/scm/browser/scmHistoryViewPane.ts`.
