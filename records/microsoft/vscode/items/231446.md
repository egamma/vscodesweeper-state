# Issue #231446 — Allow terminal links to be opened in a web browser

[https://github.com/microsoft/vscode/issues/231446](https://github.com/microsoft/vscode/issues/231446)

## Summary

Valid feature request — current code explicitly opens file:// URIs as local files in the editor (terminalLinkOpeners.ts _openFileSchemeLink), with no option to open in a browser. The related context-menu issue #216853 is also still open and would provide a natural UX for this. Keeping open as a legitimate gap.

## Change summary

Allow terminal links (especially file:// URIs to HTML files) to be opened in a web browser instead of the editor.

## Best solution

Implement an "Open in Browser" option in a link-specific context menu (as suggested in https://github.com/microsoft/vscode/issues/216853) — this avoids needing a regex-based settings mechanism and covers this use case naturally. The fix would live in `src/vs/workbench/contrib/terminalContrib/links/browser/terminalLinkOpeners.ts` and the link contribution.
