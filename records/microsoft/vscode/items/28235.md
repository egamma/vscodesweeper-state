# Issue #28235 — Provide Task Runner viewlet

[https://github.com/microsoft/vscode/issues/28235](https://github.com/microsoft/vscode/issues/28235)

## Summary

Team-authored feature request (dbaeumer is MEMBER) — hard keep-open applies. The requested dedicated task runner viewlet/panel has not been implemented on main; tasks still run in the integrated terminal without a dedicated sidebar or panel view.

## Change summary

Provide a dedicated Task Runner viewlet or panel in VS Code with task discovery, execution status, output folding, and in-place customization — beyond the current terminal-based task execution.

## Best solution

This is a long-standing design discussion with rich UX mockups from the team. The scope is broad (sidebar viewlet, task panel, MRU, problem matcher guidance, etc.). A maintainer should decide whether to break this into smaller trackable issues, close sub-parts that have been addressed (e.g. MRU in the task picker was implemented), or re-evaluate the overall direction given the current focus on agent experiences. Some sub-requests (task picker MRU, echo default) were shipped years ago; the core viewlet/panel request remains unaddressed.
