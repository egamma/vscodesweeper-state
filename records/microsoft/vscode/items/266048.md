# Issue #266048 — A large number of test output events cause hangs in the VS Code UI

[https://github.com/microsoft/vscode/issues/266048](https://github.com/microsoft/vscode/issues/266048)

## Summary

The initial fix (PR #266085, merged 2025-09-10) improved performance but the reporter confirmed on Insiders 1.105.0 that the UI still hangs/stutters significantly with high-volume test output. The issue remains valid with an 'On Deck' milestone — keeping open.

## Change summary

A large number of test output events causes VS Code UI to hang and the terminal to become unresponsive during test runs.

## Best solution

Further batching/debouncing of decoration updates triggered by test output in `src/vs/workbench/contrib/testing/` — the initial fix reduced severity but the remaining scripting burden (visible in the reporter's post-fix profile) still causes perceptible freezes with 50k+ output events.
