# Issue #209530 — Subword selection with shortcuts not working

[https://github.com/microsoft/vscode/issues/209530](https://github.com/microsoft/vscode/issues/209530)

## Summary

The `editor.smartSelect.selectSubwords` setting only applies to the Expand/Shrink Selection (Smart Select) feature, not to word-navigation commands like `cursorWordEndRightSelect`. The word operations code in `wordOperations.ts` never reads that setting — it uses `WordNavigationType.WordEnd` which navigates full words, not subwords. A team member recently engaged and assigned this, so it is actively under discussion.

## Change summary

User expects Ctrl+Shift+Right to select subwords (camelCase parts) when `editor.smartSelect.selectSubwords` is enabled, but word-navigation commands ignore that setting.

## Best solution

Either extend the word movement commands in `src/vs/editor/contrib/wordOperations/browser/wordOperations.ts` to optionally respect subword boundaries (reading the `smartSelect.selectSubwords` option or a new dedicated option), or document that the setting only applies to Expand/Shrink Selection. The team member assignment suggests this is being evaluated.
