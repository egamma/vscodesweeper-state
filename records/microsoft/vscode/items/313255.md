# Issue #313255 — [Regression] Cannot read properties of undefined (reading 'fsPath') — changesView.compare

[https://github.com/microsoft/vscode/issues/313255](https://github.com/microsoft/vscode/issues/313255)

## Summary

Still reproduces on current main: the `ChangesTreeSorter.compare()` method at line 1695 unsafely casts elements to `IChangesFileItem` without guarding against `IChangesRootItem` or `IResourceNode` elements that may lack a `uri` property, causing a TypeError when the tree is sorted during session open.

## Change summary

Opening a session in the Agents window crashes with 'Cannot read properties of undefined (reading fsPath)' when the changes view tree sorter encounters a non-file element.

## Best solution

Guard the `compare` method in `ChangesTreeSorter` (changesView.ts ~line 1695) using the existing `isChangesFileItem` type guard before accessing `.uri.fsPath`. When an element is not a file item, fall back to comparing by name or return a stable ordering (directories first, then by name).
