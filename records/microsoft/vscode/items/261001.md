# Issue #261001 — Search Editor: typing on "files to include" does not respect the "Search on Type" setting

[https://github.com/microsoft/vscode/issues/261001](https://github.com/microsoft/vscode/issues/261001)

## Summary

Confirmed in source: the search editor's includes/excludes pattern input handlers always trigger a search on every keystroke, ignoring the `search.searchOnType` setting. The main search widget correctly checks this setting before firing, but the search editor does not.

## Change summary

Typing in the Search Editor's "files to include" or "files to exclude" fields triggers a search on every keystroke regardless of the `search.searchOnType` setting.

## Best solution

Guard the `onSubmit` handlers for `inputPatternIncludes` and `inputPatternExcludes` in `src/vs/workbench/contrib/searchEditor/browser/searchEditor.ts` (lines ~204 and ~215) so that when `triggeredOnType` is `true`, the search is only triggered if `this.searchConfig.searchOnType` is enabled — mirroring the guard in `searchWidget.ts` line 656.
