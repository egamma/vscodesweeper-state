# Issue #261001 — Search Editor: typing on "files to include" does not respect the "Search on Type" setting

[https://github.com/microsoft/vscode/issues/261001](https://github.com/microsoft/vscode/issues/261001)

## Summary

Confirmed in source: the search editor's include/exclude pattern inputs always trigger a search on typing, ignoring the `search.searchOnType` setting. The bug is still present on current main.

## Change summary

Typing in the Search Editor's "files to include" or "files to exclude" fields triggers a search on every keystroke, even when `search.searchOnType` is disabled.

## Best solution

Guard the `onSubmit` handlers for `inputPatternIncludes` and `inputPatternExcludes` in `src/vs/workbench/contrib/searchEditor/browser/searchEditor.ts` (lines 204, 215) so that when `triggeredOnType` is `true`, the search is only triggered if `this.searchConfig.searchOnType` is also `true` — mirroring the guard in `searchWidget.ts` line 656.
