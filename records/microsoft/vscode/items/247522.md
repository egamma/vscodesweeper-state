# Issue #247522 — [Accessibility] Mouse echo don't work with NVDA and  JAWS

[https://github.com/microsoft/vscode/issues/247522](https://github.com/microsoft/vscode/issues/247522)

## Summary

Regression likely caused by the Electron 30 (Chromium 124) upgrade that landed in VS Code 1.92. The timing aligns exactly with the reporter's observation. This appears to be an upstream Chromium/Electron accessibility regression affecting how Windows screen readers (NVDA/JAWS) receive MSAA/UIA hit-test information for mouse echo. A related workaround PR (#320877) proposes exposing Chromium feature switches via argv.json to allow diagnostic/mitigation.

## Change summary

Mouse echo (screen reader announcing text under the mouse pointer) stopped working in VS Code's editor area starting from version 1.92.

## Best solution

Identify the specific Chromium accessibility change that broke MSAA/UIA hit-testing for mouse echo, file upstream if not already tracked, and either apply a Chromium feature-flag workaround (as PR #320877 proposes) or wait for an upstream Electron/Chromium fix. The `accessibility` area label is already correct.
