# Issue #247522 — [Accessibility] Mouse echo don't work with NVDA and  JAWS

[https://github.com/microsoft/vscode/issues/247522](https://github.com/microsoft/vscode/issues/247522)

## Summary

Verified duplicate of #277895, which identifies this as a Chromium/NVDA upstream interaction issue. The root cause is that newer Chromium builds changed UIA provider behavior and NVDA's heuristics don't yet recognize these builds as UIA-compatible — tracked upstream at nvaccess/nvda and chromium issues.

## Change summary

Screen reader mouse echo (NVDA/JAWS announcing text under the mouse pointer) stopped working in VS Code's editor starting around v1.92 due to Chromium accessibility provider changes.

## Best solution

Consolidate discussion on https://github.com/microsoft/vscode/issues/277895, which has the full root-cause analysis (Chromium UIA provider + NVDA heuristics mismatch), links to upstream NVDA and Chromium issues, and a workaround (forcing UIA in NVDA advanced settings or using the `--disable-features=UiaProvider` flag). PR https://github.com/microsoft/vscode/pull/320877 proposes exposing Chromium feature switches in argv.json as a persistent workaround path.
