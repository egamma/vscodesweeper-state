[https://github.com/microsoft/vscode/issues/315935](https://github.com/microsoft/vscode/issues/315935)

## Summary

Team-authored issue (MEMBER). The Windows taskbar uses a single `win32AppUserModelId` for the entire app, so both the main VS Code window and the Agents window share the same icon. No fix has landed on main — the issue remains valid.

## Change summary

On Windows, the VS Code and Agents windows share the same taskbar icon, making them indistinguishable.

## Best solution

Set a distinct `AppUserModelId` (or overlay icon) for the Agents window so Windows groups it separately in the taskbar. The relevant code is in `src/vs/code/electron-main/app.ts` where `app.setAppUserModelId` is called, plus likely per-`BrowserWindow` `appUserModelId` options. May also need a distinct `.ico` resource for the Agents window.
