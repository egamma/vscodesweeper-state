# Issue #255363 — Explorer context menu: Test actions should not show if file does not have any tests

[https://github.com/microsoft/vscode/issues/255363](https://github.com/microsoft/vscode/issues/255363)

## Summary

Team-authored polish issue, still unfixed on main. The related PR #255364 was closed without merging, and the `when` clause at line 1184 of testExplorerActions.ts only checks capability keys, not whether the file actually contains tests.

## Change summary

Explorer context menu shows Run/Debug/Coverage test actions on files that have no tests, cluttering the menu.

## Best solution

Add a context key (e.g. `explorerResourceHasTests`) that the testing service populates based on the test collection, and include it in the `when` clause of the `ExplorerContext` menu registration in `ExecuteTestsUnderUriAction`. The closed PR #255364 sketched this approach.
