# Issue #317985 — [Enhancement] Provide the user with an explanation of why VSC requests display output access upon first invocation.

[https://github.com/microsoft/vscode/issues/317985](https://github.com/microsoft/vscode/issues/317985)

## Summary

Already fixed on main and shipped in 1.122.1. The unwanted display-access portal dialog on Linux was caused by a desktopCapturer screen-source pre-warm on startup, which was removed in commit 8dc07ba0d48.

## Change summary

VS Code triggers an unexplained display-output permission dialog on first launch on Linux (Wayland/KDE), caused by desktopCapturer screen-source warmup at startup.

## Best solution

Keep the shipped fix in commit 8dc07ba0d48 (1.122.1) which removed the startup screen-source warmup; no further change needed. The display-access dialog now only appears when the user actually initiates screen recording via the issue reporter.
