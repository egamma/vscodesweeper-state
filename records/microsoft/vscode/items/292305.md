# Issue #292305 — VSCode doesn't use the default file manager to Open Containing Folder (tab context menu)

[https://github.com/microsoft/vscode/issues/292305](https://github.com/microsoft/vscode/issues/292305)

## Summary

Verified duplicate of the open canonical issue https://github.com/microsoft/vscode/issues/241354, which tracks the same problem — VS Code on Linux ignores the user's default file manager (configured via xdg-mime) when using 'Open Containing Folder'. Root cause is upstream in Electron's shell.showItemInFolder().

## Change summary

'Open Containing Folder' on Linux opens Thunar instead of the user's configured default file manager (pcmanfm).

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/241354, which tracks the same xdg-mime / default file manager issue on Linux. The root cause is Electron's shell.showItemInFolder() not respecting the desktop's configured default file manager — a fix or workaround (e.g. falling back to xdg-open) would need to land either upstream in Electron or as a VS Code-side override in nativeHostMainService.ts.
