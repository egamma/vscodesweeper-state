# Issue #251037 — Extension signature verification fails under Linux if file size limits are in effect

[https://github.com/microsoft/vscode/issues/251037](https://github.com/microsoft/vscode/issues/251037)

## Summary

The root cause is in the @vscode/vsce-sign native binary's dependency on .NET runtime's doublemapping.cpp, which ftruncates a memfd to 2TiB — an upstream issue. VS Code consumes this binary but doesn't own the fix. Keeping open as upstream.

## Change summary

Extension signature verification fails on Linux when file size ulimits below 2TiB are in effect, because the vsce-sign binary is killed by SIGXFSZ.

## Best solution

The fix belongs in the @vscode/vsce-sign package (or its .NET runtime dependency — tracked at https://github.com/dotnet/runtime/issues/117819). VS Code should track this upstream and update the vsce-sign dependency once a fixed version ships. In the meantime, documenting the ulimit workaround or providing a setting to skip signature verification could mitigate.
