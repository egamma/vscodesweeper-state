# Issue #116755 — SCM - improve focus movement

[https://github.com/microsoft/vscode/issues/116755](https://github.com/microsoft/vscode/issues/116755)

## Summary

This is a team-triaged, accepted accessibility bug in the SCM view's Tab/Shift+Tab focus order, labeled help-wanted and team-low-hanging. No custom Tab key handling exists in the SCM view pane, so the reported focus-order issues likely persist. Keeping open as an acknowledged contribution opportunity.

## Change summary

Tab and Shift+Tab focus navigation in the SCM view does not pair correctly — Shift+Tab after Tab jumps to the wrong element instead of returning to the original item.

## Best solution

Add proper tab-order management in `src/vs/workbench/contrib/scm/browser/scmViewPane.ts` so that Tab/Shift+Tab are paired symmetrically (Shift+Tab returns focus to the element Tab left). The `help wanted` and `team-low-hanging` labels indicate this is accepted and waiting for a community contribution.
