# Issue #326572 — VS Code Speech cannot stop playback for long GitHub Copilot Chat responses

[https://github.com/microsoft/vscode/issues/326572](https://github.com/microsoft/vscode/issues/326572)

## Summary

The VS Code core cancellation chain for text-to-speech is correctly wired: stop() cancels the token → mainThreadSpeech sends $cancelTextToSpeechSession → extHostSpeech cancels the session CTS passed to the speech provider extension. The actual audio stopping depends on the VS Code Speech extension (microsoft/vscode-speech) honoring the CancellationToken in its speech synthesis implementation. This likely belongs in that extension's repo, but since the workbench-voice label is already applied and the same team owns both, keeping open for the area owner to investigate the extension-side behavior.

## Change summary

Clicking stop or pressing Esc during text-to-speech playback of long Copilot Chat responses changes the UI icon but does not actually stop audio; multiple overlapping playbacks can result.

## Best solution

The root cause is likely in the VS Code Speech extension (microsoft/vscode-speech) not properly calling speechSynthesis.cancel() or equivalent when the CancellationToken fires. The area owner (bpasero, per the telemetry owner annotation) should investigate the extension's provideTextToSpeechSession implementation for incomplete cancellation handling, especially with long text and Japanese language. If the bug is confirmed extension-side, this issue should be transferred or a sibling filed in microsoft/vscode-speech.
