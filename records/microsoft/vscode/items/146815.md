# Issue #146815 — Can't type in Cmd+` in Keyboard Shortcut UI

[https://github.com/microsoft/vscode/issues/146815](https://github.com/microsoft/vscode/issues/146815)

## Summary

Keep open as an upstream/macOS menu dispatch limitation rather than closing: a maintainer identified that once this shortcut is reflected in the native macOS menu, the OS invokes the menu command before VS Code can capture the keybinding in the UI.

## Change summary

The Keyboard Shortcuts UI cannot record `cmd+\`` on macOS even though the binding can be configured manually in `keybindings.json` and then works for commands shown in the native menu.

## Best solution

Track this under keybindings/macOS as an upstream native menu accelerator issue; the useful end state is either an Electron/macOS API path that lets VS Code detect or capture these menu-dispatched accelerators, or a documented limitation/workaround in the keybindings UI.

- **reporter confirmation:** The reporter confirmed in https://github.com/microsoft/vscode/issues/146815#issuecomment-1089704589 that the JSON workaround makes the shortcut function, but the Keyboard Shortcuts UI still cannot enter it and either toggles Terminal or only registers `cmd`.
