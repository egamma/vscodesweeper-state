# Issue #146815 — Can't type in Cmd+` in Keyboard Shortcut UI

[https://github.com/microsoft/vscode/issues/146815](https://github.com/microsoft/vscode/issues/146815)

## Summary

Legitimate upstream macOS limitation: when a keybinding is displayed in the native macOS menu bar, the OS dispatches it directly, bypassing VS Code's keybinding recording widget. Already correctly labeled `upstream` by alexdima with a clear explanation of the root cause.

## Change summary

Cannot record Cmd+` in the Keyboard Shortcuts UI because macOS intercepts keybindings shown in native menus before they reach VS Code.

## Best solution

This requires either an Electron/macOS API to temporarily suppress native menu keybinding dispatch while recording, or a workaround in VS Code's keybinding recording widget (e.g. a text-input fallback). The `window.menuStyle` setting (added later) switching to custom menus would also sidestep this, but that's a user workaround, not a fix. Remains upstream until Electron exposes the needed control.
