# Issue #146815 — Can't type in Cmd+` in Keyboard Shortcut UI

[https://github.com/microsoft/vscode/issues/146815](https://github.com/microsoft/vscode/issues/146815)

## Summary

Keep open as an upstream/native macOS menu accelerator limitation rather than closing: the issue discussion identifies that macOS dispatches the configured menu accelerator before VS Code's keybinding recorder can capture it, and current main still has that native accelerator path.

## Change summary

The Keyboard Shortcuts UI cannot record Cmd+` on macOS, especially after it is configured for a menu-backed command such as Toggle Terminal.

## Best solution

Track this under keybindings/macOS until there is a viable Electron or macOS-native menu API approach, or a VS Code design change that lets the keybinding recorder bypass or temporarily suppress native menu accelerators while recording.

## Evidence

- **maintainer analysis:** In https://github.com/microsoft/vscode/issues/146815#issuecomment-1168795576, alexdima explains that once Cmd+` is displayed in the native macOS menu, the OS dispatches the menu action and VS Code does not receive the keybinding event for capture.
- **current code:** src/vs/platform/menubar/electron-main/menubar.ts still applies native menu accelerators from keybindings in withKeybinding(), setting options.accelerator = binding.label when binding.isNative is not false.
- **current code:** src/vs/platform/menubar/electron-main/menubar.ts createMenuItem() still treats event.triggeredByAccelerator as a native menu invocation and sends vscode:runKeybinding with the userSettingsLabel, consistent with the issue's explanation that the renderer recorder is bypassed.
