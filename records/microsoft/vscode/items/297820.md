# Issue #297820 — Chat Participant: pasted image references can have 0-byte payload

[https://github.com/microsoft/vscode/issues/297820](https://github.com/microsoft/vscode/issues/297820)

## Summary

Plausible bug in how pasted-image binary data is serialized across the extension-host IPC boundary for the `chatReferenceBinaryData` proposed API. No fix found on current main, but the report lacks enough detail to confirm — the proposed API gate, the reporter's extension configuration, and the exact access pattern are unclear. Keeping open for area routing.

## Change summary

Custom chat participants receive 0-byte image payloads when users paste images from the clipboard, making image-aware participants unable to process pasted images.

## Best solution

The area owner should verify whether `Uint8Array` image data survives IPC serialization intact when sent to extension-host chat participants via `IChatRequestVariableEntry.value`. The suspect code is in `src/vs/workbench/api/common/extHostTypeConverters.ts` (the `ChatPromptReference.to` converter, line ~3594) where `Object.values(variable.value)` reconstructs the bytes — if the `Uint8Array` is serialized to an empty object by the `Dto<>` transfer, bytes will be 0. Also confirm whether the reporter's extension correctly enables the `chatReferenceBinaryData` proposed API.
