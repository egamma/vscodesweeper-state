# Issue #318561 — Switched Merge Tooltip for Accept All Incoming/Current Changes

[https://github.com/microsoft/vscode/issues/318561](https://github.com/microsoft/vscode/issues/318561)

## Summary

Confirmed on current main: the AcceptAllInput1/AcceptAllInput2 command titles are hardcoded as 'Incoming'/'Current' but the actual input assignment depends on whether it's a merge or rebase — during a rebase, the tooltips are swapped relative to the panel content.

## Change summary

Merge Editor 'Accept All' button tooltips say 'Incoming' and 'Current' in the wrong order when the input1/input2 assignment differs from the hardcoded assumption (e.g. during a rebase).

## Best solution

Make the AcceptAllInput1/AcceptAllInput2 command titles generic ('Accept All Changes from Left' / 'Accept All Changes from Right') or dynamically derive them from the actual input titles, since the merge editor doesn't control whether input1 is 'Current' or 'Incoming' — that depends on the caller (git extension). The color mapping in colors.ts (input1→mergeCurrentHeaderBackground, input2→mergeIncomingHeaderBackground) should also be revisited for consistency.
