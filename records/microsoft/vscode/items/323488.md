# Issue #323488 — Explorer context menu command target depends on selection size

[https://github.com/microsoft/vscode/issues/323488](https://github.com/microsoft/vscode/issues/323488)

## Summary

The asymmetric behavior (focus wins with 1 selected item, selection wins with 2+) was deliberately introduced in PR #160342 (commit 92626b7a60de, Sep 2022). The reporter raises a legitimate UX consistency concern, but the current behavior is intentional — this is a design discussion for the explorer area owner, not a bug.

## Change summary

Explorer context menu targets the right-clicked file when 0–1 items are selected, but ignores the right-clicked file and acts on the existing multi-selection when 2+ items are selected — an inconsistency the reporter finds surprising.

## Best solution

Route to the explorer area owner to decide whether the single-vs-multi selection asymmetry in `getContext()` (src/vs/workbench/contrib/files/browser/views/explorerView.ts, line 103) should be revisited. A consistent rule — always clear the selection and target the right-clicked item when it's outside the selection — would match the reporter's expectation, but may break the multi-select workflow the PR #160342 was designed to support.
