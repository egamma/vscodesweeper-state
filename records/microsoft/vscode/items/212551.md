# Issue #212551 — redo command doesn't focus cursor

[https://github.com/microsoft/vscode/issues/212551](https://github.com/microsoft/vscode/issues/212551)

The report describes a plausible cursor-positioning issue after programmatic redo, traceable to the `_hasFocus` guard in `cursor.ts:260`. However, the report lacks a minimal reproduction (extension code) and no one has confirmed it, so requesting more info.

## Change summary

Running redo via extension API or command palette does not move the cursor to the point of the redo edit.

## Best solution

Request a minimal repro extension. The likely root cause is the `_hasFocus` guard in `src/vs/editor/common/cursor/cursor.ts` line 260 which skips `resultingSelection` restoration when the editor doesn't have DOM focus at the moment the model change event fires. A fix would need to distinguish programmatic redo-on-behalf-of-this-editor from redo-on-another-editor.
