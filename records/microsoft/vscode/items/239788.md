# Issue #239788 — PR Diff View Showing False Differences for Windows-1252 Encoded Files

[https://github.com/microsoft/vscode/issues/239788](https://github.com/microsoft/vscode/issues/239788)

## Summary

Still open on main. The quick diff model propagates encoding to original models (quickDiffModel.ts:396-401), but PR diff views provided by extensions (like GitHub Pull Requests) likely bypass this path — their content providers fetch raw bytes without applying workspace encoding settings. The bug sits at the boundary between core diff infrastructure and extension-provided content.

## Change summary

PR diff view shows false differences on lines with non-ASCII characters in Windows-1252 encoded files because the base file content is not decoded with the correct encoding.

## Best solution

Investigate whether the core diff infrastructure should propagate configured encoding to extension-provided content models, or whether the GitHub Pull Requests extension (v0.102.0) needs to apply encoding when fetching base content. The quickDiffModel.ts already handles this for inline diffs — a similar pattern may be needed for full-file diffs opened from PR extensions. May ultimately be a `*caused-by-extension` issue for the GitHub PR extension.
