# Issue #261561 — Git extension is very confused by `remote = .` and `merge = [anything not a branch]`

[https://github.com/microsoft/vscode/issues/261561](https://github.com/microsoft/vscode/issues/261561)

## Summary

Still reproduces on main: the Git extension assumes upstream refs always live under `refs/heads/`, so non-standard merge targets (like arbitrary refs with `remote = .`) produce garbled `UpstreamRef.name` values, breaking the history provider and graph view.

## Change summary

When a branch's upstream merge target is not under `refs/heads/`, the Git extension produces a corrupt upstream name and fails to show source control history.

## Best solution

Guard the upstream-name parsing in `getRefs()` (git.ts ~line 3203) to handle `upstreamRef` values that don't start with `refs/heads/` — either skip setting upstream or preserve the full ref. Downstream consumers in historyProvider.ts (line 126) that construct `refs/heads/${name}` for `remote = .` also need the same guard. The `UpstreamRef` API surface may need a documentation note or a future revision to clarify that `name` can be a non-branch ref.
