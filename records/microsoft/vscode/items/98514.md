# Issue #98514 — Search error: ENAMETOOLONG

[https://github.com/microsoft/vscode/issues/98514](https://github.com/microsoft/vscode/issues/98514)

## Summary

Still reproduces on current main: exclude patterns (from `files.exclude` or `onlyOpenEditors`) are passed as individual `-g` CLI args to ripgrep with no guard against OS command-line length limits, causing ENAMETOOLONG. The bug is already confirmed and correctly labeled; it should remain open.

## Change summary

Search and Quick Open fail with 'spawn ENAMETOOLONG' when too many exclude or include patterns produce a command line exceeding the OS limit.

## Best solution

Write exclude/include patterns to a temporary `.ignore` file or use ripgrep's `--ignore-file` flag instead of passing each pattern as a `-g` CLI argument, in `src/vs/workbench/services/search/node/ripgrepTextSearchEngine.ts` and `src/vs/workbench/services/search/node/ripgrepFileSearch.ts`. For the `onlyOpenEditors` path specifically (`src/vs/workbench/services/search/common/queryBuilder.ts#commonQueryFromFileList`), consider capping the number of include patterns or batching the ripgrep invocations.
