# Issue #98514 — Search error: ENAMETOOLONG

[https://github.com/microsoft/vscode/issues/98514](https://github.com/microsoft/vscode/issues/98514)

## Summary

Still reproduces on main: exclude/include patterns are passed as individual `-g` CLI arguments to ripgrep with no fallback when the total command length exceeds the OS ENAMETOOLONG limit. This is the canonical issue — multiple duplicates (109380, 178975, 184006, 221496) have been closed against it.

## Change summary

Search and Quick Open fail with ENAMETOOLONG when many exclude patterns or open-editor include patterns produce a ripgrep command line that exceeds OS limits.

## Best solution

Write exclude/include globs to a temporary ignore file and pass it via ripgrep's `--ignore-file` flag instead of individual `-g` arguments, in both `ripgrepTextSearchEngine.ts` (lines ~440–442) and `ripgrepFileSearch.ts` (lines ~39–57). This was the approach Rob Lourens suggested in https://github.com/microsoft/vscode/issues/109380.
