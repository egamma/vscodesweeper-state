# Issue #161969 — Merge UI: Show Accept code lenses in result

[https://github.com/microsoft/vscode/issues/161969](https://github.com/microsoft/vscode/issues/161969)

## Summary

Team-authored design discussion about adding actionable Accept codelenses to the merge editor result pane. The current code only shows status text ('No Changes Accepted') and Remove actions in the result — the Accept actions remain solely in the input editors. Still under discussion; keeping open.

## Change summary

Show actionable 'Accept Input 1/2' code lenses in the merge editor result pane instead of the non-actionable 'No Changes Accepted' status label.

## Best solution

Extend `resultItems` in `src/vs/workbench/contrib/mergeEditor/browser/view/conflictActions.ts` to include Accept commands (mirroring `getItemsInput` logic) when the base state has no inputs included, replacing the static status text with actionable lenses. Needs a design decision from the merge-editor owner on whether showing Accept in both input and result panes would confuse users.
