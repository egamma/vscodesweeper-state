# Issue #111616 — [theme icons] Include color class in ThemeIcon.asClassName()

[https://github.com/microsoft/vscode/issues/111616](https://github.com/microsoft/vscode/issues/111616)

## Summary

Still unimplemented on main: ThemeIcon.asClassNameArray() does not include a color-related class even though ThemeIcon has a `color` property. The issue is team-authored and in Backlog, so it stays open.

## Change summary

Request to include a CSS class representing the ThemeIcon color in the output of ThemeIcon.asClassName().

## Best solution

Extend `asClassNameArray` in `src/vs/base/common/themables.ts` to append a color class (e.g. derived from `icon.color.id`) when the color property is set, simplifying icon-color support across the workbench.
