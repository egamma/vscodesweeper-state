# Issue #110067 — Implement Debugger Modules View

[https://github.com/microsoft/vscode/issues/110067](https://github.com/microsoft/vscode/issues/110067)

## Summary

Valid feature request for a built-in Modules View in the debugger, leveraging the existing DAP ModulesRequest. No implementation exists on current main — only the protocol types are defined. With only 2 upvotes after nearly 5 years, community traction is low, but the request is technically sound and the debug area label is already applied.

## Change summary

Add a built-in Modules View to the debug sidebar that displays loaded modules via the DAP ModulesRequest, similar to Visual Studio's Modules window.

## Best solution

This could remain a feature request in the debug area backlog. The issue author notes it can already be done via an extension using a TreeDataProvider (as vscode-lldb demonstrates), so `*extension-candidate` is plausible — but since VS Code already ships a similar Loaded Scripts view and the DAP protocol types for modules are already in the codebase (`debugProtocol.d.ts`), a core implementation is also reasonable. Low traction suggests keeping it parked in Backlog for now; a maintainer could decide whether to promote or redirect to an extension.
