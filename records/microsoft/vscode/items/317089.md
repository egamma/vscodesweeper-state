## Summary

Team-authored bug about the Agents web app flashing a workspace picker before host discovery completes. No commit directly referencing this issue was found on main; keeping open for the team to address.

## Change summary

The Agents app on insiders.vscode.dev shows the 'start by picking a workspace' view during host discovery, then hides it when no host is found — it should not appear until a host is confirmed.

## Best solution

Gate the new-session workspace picker visibility on a 'host confirmed' state in the sessions chat view or new-chat widget, so that no input/picker UI renders until a host is definitively available. Likely touches `src/vs/sessions/browser/parts/chatView.ts` or `src/vs/sessions/contrib/chat/browser/newChatWidget.ts`.
