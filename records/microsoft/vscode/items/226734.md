# Issue #226734 — Consider to support streaming for file search results

[https://github.com/microsoft/vscode/issues/226734](https://github.com/microsoft/vscode/issues/226734)

## Summary

Valid, unimplemented feature request. The file search provider API (`FileSearchProvider2.provideFileSearchResults`) still returns `ProviderResult<Uri[]>` with no streaming support, while the text search provider already accepts a `Progress<>` parameter for streaming. No work has landed to add streaming to the file search path.

## Change summary

Add streaming support to the file search provider API and `findFiles2`, analogous to how the text search provider already streams results via a `Progress<>` callback.

## Best solution

Add a `Progress<Uri>` parameter to `FileSearchProvider2.provideFileSearchResults` (and possibly an `onFile` callback variant for `findFiles2`) so that large file search results can be consumed incrementally, matching the pattern already used in `TextSearchProvider2`. This touches the proposed API definitions in `src/vscode-dts/vscode.proposed.fileSearchProvider2.d.ts` and `src/vscode-dts/vscode.proposed.findFiles2.d.ts`, plus the backing implementation in `src/vs/workbench/api/common/extHostSearch.ts`.
