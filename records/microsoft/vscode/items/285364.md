# Issue #285364 — With minimap disabled, copilot keep/undo changes view bleeds into scrollbar

[https://github.com/microsoft/vscode/issues/285364](https://github.com/microsoft/vscode/issues/285364)

## Summary

Still reproduces on main: the overview ruler background is set to null (transparent) when minimap is disabled, allowing editor diff decorations to bleed through behind the scrollbar. When minimap is enabled on the right, the ruler gets a proper background that masks this.

## Change summary

When minimap is disabled, inline diff/edit decorations bleed through the transparent overview ruler into the scrollbar area, unlike when minimap is enabled.

## Best solution

In `decorationsOverviewRuler.ts`, when minimap is disabled, set the overview ruler background to `defaultBackground` (same as the minimap-enabled case) instead of `null`, so the scrollbar area is always opaque regardless of minimap state.
