# Issue #138664 — Setting `files.associations` is not observed/respected

[https://github.com/microsoft/vscode/issues/138664](https://github.com/microsoft/vscode/issues/138664)

## Summary

The reported behavior — language detection overriding files.associations — has been fixed on main through multiple improvements: the `!this.resourceHasExtension` guard prevents detection on files with extensions, and commit 7a82e67c8fe tracks language-change sources to prevent detection from stomping explicit assignments.

## Change summary

Language detection overrides explicitly configured `files.associations` settings, particularly when mapping file types to plaintext.

## Best solution

Keep the shipped fixes: the `resourceHasExtension` guard in `TextFileEditorModel.autoDetectLanguage()` and the language-change-source tracking from PR #159782. No further change needed — if a new edge case surfaces, it should be filed as a fresh issue with current reproduction steps.
