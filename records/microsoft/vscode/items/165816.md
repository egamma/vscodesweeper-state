# Issue #165816 — _sendLineData can freeze the renderer for seconds for very long lines

[https://github.com/microsoft/vscode/issues/165816](https://github.com/microsoft/vscode/issues/165816)

## Summary

Still reproduces on main: `_sendLineData` in lineDataEventAddon.ts has no length threshold — it concatenates all wrapped line segments without limit, which freezes the renderer for extremely long lines. No fix has landed.

## Change summary

The terminal renderer freezes for seconds when `_sendLineData` assembles very long wrapped lines via repeated `translateToString` and string concatenation.

## Best solution

Add a character-length threshold (e.g. 1024 chars) in `_sendLineData` that either truncates the assembled line or skips firing the event for lines exceeding the limit, balancing task-matcher needs against UI responsiveness.
