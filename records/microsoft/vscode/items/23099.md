# Issue #23099 — Add dropdown for search input history

[https://github.com/microsoft/vscode/issues/23099](https://github.com/microsoft/vscode/issues/23099)

## Summary

This is a well-established canonical feature request (59 upvotes, 14+ duplicates) for a visual dropdown to browse search history. The current implementation only supports sequential keyboard navigation via Alt+Up/Down — no dropdown UI exists on main. The issue serves as the canonical tracker and should remain open.

## Change summary

Add a visual dropdown list for search input history in the Find widget and Search view, rather than requiring sequential keyboard navigation.

## Best solution

Add a dropdown button or trigger (e.g. on a chevron icon or a keyboard shortcut) to the HistoryInputBox that renders the full history list in a context menu or quick-pick style overlay, allowing users to visually browse and select from prior searches. This would touch `src/vs/base/browser/ui/inputbox/inputBox.ts` (HistoryInputBox), `src/vs/base/browser/ui/findinput/findInput.ts`, and the search widget at `src/vs/workbench/contrib/search/browser/searchWidget.ts`.
