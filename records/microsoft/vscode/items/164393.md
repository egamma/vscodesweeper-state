# Issue #164393 — Git - diff editor cannot be opened while scanning for repositories

[https://github.com/microsoft/vscode/issues/164393](https://github.com/microsoft/vscode/issues/164393)

## Summary

Still reproducible on main: GitFileSystemProvider.stat() and readFile() both await model.isInitialized, which blocks diff editor opening until the full repository scan completes. The core issue — that diffs are gated on scanning unrelated subfolders — remains unfixed.

## Change summary

Opening a diff editor in the git extension is blocked until all subdirectory repository scanning finishes, even when the top-level repository is already discovered.

## Best solution

Allow the file system provider to serve diffs for already-discovered repositories without waiting for the full scan to complete — e.g., gate on the specific repository being ready rather than the global isInitialized promise, or resolve isInitialized earlier for the primary workspace repository and defer deep scanning.
