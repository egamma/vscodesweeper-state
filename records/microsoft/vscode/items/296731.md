# Issue #296731 — toggleQuickDiffWidgetFocus places cursor at (1,1) instead of at the change location

[https://github.com/microsoft/vscode/issues/296731](https://github.com/microsoft/vscode/issues/296731)

## Summary

Still reproduces on main: toggleFocus() at line 466 calls this.diffEditor.focus() without positioning the cursor at the change location, so the diff editor defaults to (1,1). The widget already tracks the current change and has a revealChange() helper, but toggleFocus() doesn't use either.

## Change summary

toggleQuickDiffWidgetFocus places the cursor at (1,1) instead of at the change location when focusing the diff editor.

## Best solution

Update QuickDiffWidget.toggleFocus() to call revealChange(this.change) (or set the cursor position using the change's modifiedStartLineNumber) after focusing the diff editor, similar to how the widget already reveals the change on initial layout.
