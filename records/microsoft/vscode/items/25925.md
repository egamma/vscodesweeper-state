# Issue #25925 — Improved search in "Go to file" panel

[https://github.com/microsoft/vscode/issues/25925](https://github.com/microsoft/vscode/issues/25925)

## Summary

Still a valid request — the fuzzy scorer still prefers filename matches over full-path matches (line 426 of fuzzyScorer.ts: `preferLabelMatches = !path || !query.containsPathSeparator`). Recent work (commit 7c3f359a5cb, Jun 2025) tweaked consecutive-match boosting but did not address the fundamental filename-over-path preference. The open siblings #27317 and #131431 track overlapping but not identical aspects; this issue's specific ask (path matches surfacing better) remains unresolved and has strong community signal (31 upvotes, active comments through Sep 2025).

## Change summary

Quick Open search results should rank full-path consecutive matches higher relative to filename-only scattered matches.

## Best solution

The algorithm in `src/vs/base/common/fuzzyScorer.ts` needs to relax its filename-over-path preference when the path produces a clearly stronger contiguous match. This is the same underlying work as #27317 and #131431 — consolidating these three into one canonical tracking issue would help, but this issue carries distinct community signal and concrete examples worth preserving. A maintainer should decide whether to merge into #27317 or keep as a separate actionable item.
