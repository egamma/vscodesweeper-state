# Issue #184766 — Cannot sign into GitHub when running elevated

[https://github.com/microsoft/vscode/issues/184766](https://github.com/microsoft/vscode/issues/184766)

## Summary

Still reproduces on main: the URL handler callback (`vscode://...`) cannot reach an elevated VS Code instance from a non-elevated browser process on Windows. The `isAdmin()` API exists but hasn't been wired into the authentication flow to use an alternative login path when elevated.

## Change summary

GitHub authentication fails when VS Code is running elevated (as administrator) on Windows because the URI callback from the browser cannot reach the elevated process.

## Best solution

Detect elevated mode via the existing `INativeHostService.isAdmin()` API in the GitHub authentication extension and fall back to a device-code or manual-token flow that doesn't rely on the URI handler. The auth extension (`extensions/github-authentication`) and possibly `src/vs/workbench/services/authentication/` would need the change.
