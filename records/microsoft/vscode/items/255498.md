# Issue #255498 — Unable to select New Window with Profile when windows are minimized

[https://github.com/microsoft/vscode/issues/255498](https://github.com/microsoft/vscode/issues/255498)

## Summary

Valid bug, still present on main. On macOS, when all windows are minimized, the native menubar disables items without fallback handlers — the profile submenu actions lack fallback handlers unlike 'New Window'.

## Change summary

File > New Window with Profile submenu items are greyed out on macOS when all VS Code windows are minimized.

## Best solution

Register fallback handlers in `src/vs/platform/menubar/electron-main/menubar.ts` for the profile open-window actions (pattern: `workbench.action.openProfile.*`) that call `windowsMainService.openEmptyWindow` with the `forceProfile` option, mirroring the existing `workbench.action.newWindow` fallback.
