# Issue #52700 — Git - Support submodule management

[https://github.com/microsoft/vscode/issues/52700](https://github.com/microsoft/vscode/issues/52700)

## Summary

Valid feature request with 24 upvotes (near the 20-vote backlog threshold), parked in Backlog with `help wanted`. Submodule management commands (add, init, update, remove) are still absent from the UI on current main — only an internal `submodule update` helper exists with no exposed command palette or menu entry. Keeping open as a legitimate community-driven request.

## Change summary

Add UI commands for git submodule add, init, update, and remove in VS Code's Source Control view.

## Best solution

Expose submodule management commands (add, update, remove) in the Source Control `...` menu and command palette, implemented in `extensions/git/src/commands.ts` backed by the git CLI wrappers in `extensions/git/src/git.ts`. The `help wanted` label is appropriate — this is a well-scoped contribution opportunity. Issue https://github.com/microsoft/vscode/issues/311662 tracks broader submodule behavior but is distinct (UX behavior vs. explicit management commands), so both should stay open.
