# Issue #182627 — Recent entry URL can get too large for Windows jumplist

[https://github.com/microsoft/vscode/issues/182627](https://github.com/microsoft/vscode/issues/182627)

## Summary

Still unfixed on main: the jumplist entry `args` field has no length guard, so long dev container URIs get cropped by Windows, producing invalid URIs that cause errors. The suggested validation (comment 4) was never implemented.

## Change summary

Dev container entries in the Windows jumplist get cropped when the folder-uri argument is too long, producing invalid URIs and error dialogs.

## Best solution

Add a length check on the `args` string in `workspacesHistoryMainService.ts` (around line 383) to skip entries whose URI exceeds the Windows jumplist arg limit (~260 chars), and add remote URI path validation in `windowsMainService.ts` (as bpasero suggested) to gracefully reject URIs with empty paths.
