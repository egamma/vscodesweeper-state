# Issue #182627 — Recent entry URL can get too large for Windows jumplist

[https://github.com/microsoft/vscode/issues/182627](https://github.com/microsoft/vscode/issues/182627)

## Summary

Still reproduces on main: the Windows jumplist args field has no length guard for long dev container URIs. The title/description are capped at 255 chars but args is unbounded, and Windows silently truncates it, resulting in an invalid URI that fails to open.

## Change summary

Dev container entries in the Windows jumplist have URIs too long for the OS limit, causing them to be silently cropped and fail to open.

## Best solution

Add a length check on `args` in `workspacesHistoryMainService.ts` (around line 383–394) and skip jumplist entries whose args exceed the Windows limit (~260–2048 chars depending on API). Alternatively, validate the URI path is non-empty when parsing jumplist args back (line 365–370). The upstream Electron/Windows limit is unlikely to change.
