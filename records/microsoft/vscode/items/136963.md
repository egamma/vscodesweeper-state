# Issue #136963 — [bug] Virtual Keyboard covers cursor and code on iOS/iPad if below keyboard waterline 

[https://github.com/microsoft/vscode/issues/136963](https://github.com/microsoft/vscode/issues/136963)

## Summary

Still reproduces on main: the iOS virtual keyboard resize handler in `window.ts` re-layouts the workbench and scrolls the window to (0,0), but never reveals the editor cursor into the reduced viewport. A team member confirmed it persists as of Dec 2024.

## Change summary

When tapping to edit text below the virtual keyboard waterline on iOS/iPad, the keyboard covers the cursor and code without scrolling it into view.

## Best solution

After the visualViewport resize on iOS, the active editor should reveal its current cursor position into view. The fix likely belongs in `src/vs/workbench/browser/window.ts` (the resize handler at ~line 263) or in the editor's layout cycle, calling something like `editor.revealPositionInCenterIfOutsideViewport` for the primary cursor after a keyboard-triggered resize.
