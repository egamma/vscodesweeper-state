# Issue #136963 — [bug] Virtual Keyboard covers cursor and code on iOS/iPad if below keyboard waterline 

[https://github.com/microsoft/vscode/issues/136963](https://github.com/microsoft/vscode/issues/136963)

## Summary

Still reproduces on main: the iOS virtual-keyboard resize handler re-layouts the workbench and scrolls to (0,0) but never calls revealPosition/revealRange on the editor to bring the cursor into view. A team member confirmed in Dec 2024 it persists.

## Change summary

On iOS/iPad, tapping to edit text below the keyboard waterline opens the virtual keyboard which covers the cursor and code — the editor doesn't scroll to reveal the cursor position.

## Best solution

In the iOS visualViewport resize handler (src/vs/workbench/browser/window.ts ~line 263), after the layout call, reveal the active editor's cursor position into view. The editor-core layer already has revealPosition/revealRangeInCenter APIs; the fix is wiring the resize event to a cursor-reveal call when the resize is due to a keyboard appearance.
