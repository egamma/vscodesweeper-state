# Issue #179031 — Git - File status shows "Deleted" even after file is restored

[https://github.com/microsoft/vscode/issues/179031](https://github.com/microsoft/vscode/issues/179031)

## Summary

Still reproduces on main: the diff editor tab title is computed once at open time via `getTitle()` in `ResourceCommandResolver.resolveChangeCommand()` and passed as a static string to `vscode.diff`. When the user discards changes (restoring a deleted file), the tab title remains stale. An open community PR (#181815) exists but has not been merged.

## Change summary

Clicking a deleted file in the SCM sidebar opens a diff editor titled '(Deleted)'; after discarding (restoring) the file, the tab title doesn't update.

## Best solution

Either close the stale diff editor when changes are discarded, or dynamically update the diff editor title when the underlying git resource status changes. The fix likely lives in `extensions/git/src/repository.ts` (the `getTitle` / `resolveChangeCommand` flow) and possibly the discard-changes handler in `extensions/git/src/commands.ts`. The open community PR https://github.com/microsoft/vscode/pull/181815 should be reviewed, updated, or superseded.
