# Issue #285363 — Migrating changes infinite loop

[https://github.com/microsoft/vscode/issues/285363](https://github.com/microsoft/vscode/issues/285363)

## Summary

Valid edge-case bug report from December 2025 (extension 0.35.2), but the background agent worktree and change-migration code has been extensively refactored since then — needs confirmation on a current build before action.

## Change summary

Background agent gets stuck in an infinite loop at "Migrating changes to worktree" when the user manually commits their changes via terminal before selecting "Copy Changes" in the agent prompt.

## Best solution

Ask the reporter to reproduce on the latest VS Code Insiders with current Copilot. The `moveOrCopyChangesToWorkTree` in `folderRepositoryManagerImpl.ts` already has a `hasUncommittedChanges` guard (line 795-800) that returns early when no changes exist, and the area was refactored in commits 876694e and 677ee01. If still reproducible, the issue likely lies in the git extension's `migrateChanges` flow or in state staleness between the prompt and the migration call.
