# Issue #198304 — Quick chat should not listen to main container changes only

[https://github.com/microsoft/vscode/issues/198304](https://github.com/microsoft/vscode/issues/198304)

## Summary

Still reproduces on current main: chatQuick.ts (now at widgetHosts/chatQuick.ts) still references mainContainerDimension and onDidLayoutMainContainer, ignoring auxiliary windows where quick chat may appear.

## Change summary

Quick chat layout calculations are hardcoded to the main window container, causing incorrect sizing when quick chat opens in an auxiliary (floating) window.

## Best solution

Replace `layoutService.mainContainerDimension` and `onDidLayoutMainContainer` in `src/vs/workbench/contrib/chat/browser/widgetHosts/chatQuick.ts` with container-aware equivalents that respect whichever window the quick chat is displayed in (e.g. `getContainerDimension`/`onDidLayoutContainer` scoped to the quick chat's actual container).
