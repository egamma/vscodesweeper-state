# Issue #235155 — Number-of-files-found Badge is hard to see on a selected list item

[https://github.com/microsoft/vscode/issues/235155](https://github.com/microsoft/vscode/issues/235155)

## Summary

Team-authored polish issue; the explorer find-match badge uses a fixed background color (`list.filterMatchBackground`) that doesn't adapt for selected/focused items, causing poor contrast in Light+. Still present on main.

## Change summary

The file-count badge in explorer find is hard to read when its parent item is selected, because the badge background doesn't change for the selected state.

## Best solution

Add a foreground color override (or adjust badge background) for the selected/focused state in `explorerViewer.ts` line ~1024 where the `CountBadge` is created, or register a dedicated color token for the badge-on-selected-item case. The fix is localized to the badge styling in `src/vs/workbench/contrib/files/browser/views/explorerViewer.ts`.
