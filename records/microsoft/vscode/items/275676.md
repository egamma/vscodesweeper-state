# Issue #275676 — bad auto-indenting

[https://github.com/microsoft/vscode/issues/275676](https://github.com/microsoft/vscode/issues/275676)

## Summary

This is a real auto-indentation bug that remains unfixed on main. The electric character / bracket-matching indentation logic re-indents an existing closing bracket to match a newly typed opening bracket, even though the user hasn't changed that closing bracket's scope. Already correctly labeled and under discussion.

## Change summary

Typing an opening bracket between an existing bracket pair on different lines causes the existing closing bracket to be re-indented incorrectly.

## Best solution

The electric character mechanism in `src/vs/editor/common/cursor/cursorTypeEditOperations.ts` (around line 470–494) needs to be refined so that when a closing bracket's indentation is adjusted via `findMatchingBracketUp`, it doesn't match against a newly-typed opening bracket that isn't the bracket's actual scope owner. The `editor-autoindent` area owner should investigate whether the bracket-pair matching needs additional context about which opening bracket truly owns the closing bracket.
