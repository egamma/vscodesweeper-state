## Summary

Team-authored error-telemetry issue tracking a performance sample error in the deltaDecorations path when extensions set many decorations. Hard keep-open (MEMBER author). Likely still relevant as an extension API perf concern.

## Change summary

Performance sample error fires when an extension sets a large number of decorations via the setDecorations API, triggering expensive deltaDecorations work on the renderer.

## Best solution

Either batch or throttle decoration updates from extensions in the `$trySetDecorations` path, or investigate whether extensions triggering this can be guided to use more efficient decoration strategies. The area owners (api + languages-diagnostics) should decide if a mitigation belongs in core or if this is inherently extension-caused.
