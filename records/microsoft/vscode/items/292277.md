# Issue #292277 — Content mismatch between Chat view and Accessible View (Alt+F2) with HTML and special characters:A11y_Visual Studio Code_Editor_Infor and relationship

[https://github.com/microsoft/vscode/issues/292277](https://github.com/microsoft/vscode/issues/292277)

## Summary

Team-authored accessibility bug — content mismatch between Chat view and Accessible View (Alt+F2) where HTML entities and special characters render differently. The accessible view pipeline in `chatResponseAccessibleView.ts` uses `renderAsPlaintext` which processes markdown through `marked` and then unescapes a limited set of HTML entities via `unescapeInfo`. Entities not in that map would pass through un-decoded, which could explain the reported mismatches. Keeping open per team-author policy.

## Change summary

The Accessible View (Alt+F2) for chat responses shows raw HTML entities, extra markdown syntax, and auto-inserted HTML tags instead of matching the rendered chat content.

## Best solution

Audit the `renderAsPlaintext` pipeline in `src/vs/base/browser/markdownRenderer.ts` — specifically expand the `unescapeInfo` map or use a general HTML entity decoder — and review `chatResponseAccessibleView.ts` to ensure all content part kinds produce output faithful to what the chat view renders. The area owner (accessibility) should verify with the reporter's specific prompts whether the many improvements since February 2026 (commits `96a5d8276cb`, `868d1ca73d3`, `430b0f928ea`, etc.) have already addressed the reported gaps.
