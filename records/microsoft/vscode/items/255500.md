# Issue #255500 — Reindent Lines doesn't work when the line has no initial indent

[https://github.com/microsoft/vscode/issues/255500](https://github.com/microsoft/vscode/issues/255500)

## Summary

Still reproduces on main: the `doesLineStartWithString` guard in `getReindentEditOperations` incorrectly skips lines that have zero indentation in JSON (and similar languages) because the first token at column 0 is a String token. When there's any indentation, whitespace precedes the string token and the line is processed normally.

## Change summary

Reindent Lines skips lines that start with a string token (like JSON property keys) when those lines have no leading whitespace, leaving them unindented.

## Best solution

Fix the `doesLineStartWithString` check in `src/vs/editor/contrib/indentation/common/indentation.ts` to only skip lines that are *entirely* within a multi-line string (the original intent of commit b085f37f6b8), rather than lines that merely happen to have a string token at position 0. The guard should check whether the line is inside a string context from the previous line, not whether the first visible token is a string.
