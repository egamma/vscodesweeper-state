# Issue #312775 — Claude: permit to write inside workspace folder

[https://github.com/microsoft/vscode/issues/312775](https://github.com/microsoft/vscode/issues/312775)

## Summary

Team-authored issue (MEMBER) requesting improved file-write permission granularity for Claude Code agent sessions — specifically, a workspace-scoped write-allow that doesn't extend to arbitrary locations, plus a way to revoke 'Allow in this workspace'. Keeping open per policy.

## Change summary

Claude Code's 'Ask before edits' mode lacks a permission tier that auto-allows writes inside the workspace folder while still blocking writes outside it, and the 'Allow in this workspace' grant cannot be revoked.

## Best solution

Add a workspace-scoped write permission tier for agent sessions (similar to the local-mode behavior) that auto-approves writes inside the workspace root but still prompts for external paths, and ensure the grant is revocable via the existing 'Reset permissions' flow or a visible toggle in the trust/permissions UI.
