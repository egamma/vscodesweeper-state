# Issue #316345 — Session records that cannot be deleted come from codex and claude code plug-ins.

[https://github.com/microsoft/vscode/issues/316345](https://github.com/microsoft/vscode/issues/316345)

## Summary

The Delete action for sessions in the Agents window is intentionally gated on `supportsDelete`, but only CopilotCLI sessions set this to `true` — Claude Code and Cloud (Codex) sessions do not, so the delete context menu never appears for them. The underlying `_deleteAgentSessions` already handles non-CLI sessions via `chatService.removeHistoryEntry`, so the capability gate is the only barrier.

## Change summary

Sessions created by Claude Code and Codex (cloud) agents cannot be deleted from the Agents window session list, even after those sessions are deleted in their respective clients.

## Best solution

Extend `_sessionTypeSupportsDelete` in `copilotChatSessionsProvider.ts` to return `true` for `AgentSessionProviders.Claude` and `CopilotCloudSessionType.id` as well, since the delete backend path already handles non-CLI sessions. Verify that `chatService.removeHistoryEntry` properly cleans up Claude and Cloud sessions end-to-end before enabling.
