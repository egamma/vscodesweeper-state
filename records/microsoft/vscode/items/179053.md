[https://github.com/microsoft/vscode/issues/179053](https://github.com/microsoft/vscode/issues/179053)

## Summary

The core request — a built-in git graph visualization — has been implemented on main and shipped since VS Code 1.92. The SCM history graph is enabled by default with commit graph rendering, filtering, multi-select, context menus, drag-and-drop, and ongoing polish through 2025–2026.

## Change summary

Request for a built-in git commit graph visualization in VS Code, similar to the Git Graph extension or WebStorm's git log.

## Best solution

Keep the shipped SCM history graph (enabled by default since 1.92.0). The implementation lives in `src/vs/workbench/contrib/scm/browser/scmHistory.ts` and `scmHistoryViewPane.ts`, with active ongoing polish. Specific sub-requests (e.g. squash UI from https://github.com/microsoft/vscode/issues/214809) are tracked separately.
