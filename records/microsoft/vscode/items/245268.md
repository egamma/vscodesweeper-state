# Issue #245268 — Trying to run as root (FreeBSD) or Admin (Windows) fails with an error

[https://github.com/microsoft/vscode/issues/245268](https://github.com/microsoft/vscode/issues/245268)

## Summary

Verified duplicate of the canonical sandbox-under-elevation issue. Running VS Code as admin/root triggers a Chromium sandbox failure (the sandbox cannot operate under elevated privileges), which is already tracked and linked upstream.

## Change summary

VS Code fails with 'launch-failed, code 18' when launched as administrator (Windows) or root (FreeBSD) due to Chromium sandbox incompatibility with elevated processes.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/122951, which tracks the upstream Chromium/Electron sandbox limitation under elevated privileges (linked to https://github.com/electron/electron/issues/49167). The workaround is launching with `--no-sandbox --disable-gpu-sandbox`.
