# Issue #274606 — HTTP MCP Server URL Configuration Fails to Preserve Percent-Encoded Slashes (%2F) in Path

[https://github.com/microsoft/vscode/issues/274606](https://github.com/microsoft/vscode/issues/274606)

## Summary

Still reproduces on main: URI.parse() decodes %2F to / in the path component, and then toString(true) emits the literal slash. The MCP HTTP transport uses this pipeline for all outbound requests, breaking services that require percent-encoded slashes in URL paths.

## Change summary

HTTP MCP server URLs with percent-encoded slashes (%2F) in the path are decoded to literal slashes before the request is sent, causing 404 errors from services like AWS Bedrock.

## Best solution

Store the original raw URL string from the mcp.json config and use it directly for HTTP requests in McpHTTPHandle, rather than round-tripping through URI.parse() → toString(true). The URI can still be used for display/matching but the raw string should drive the actual fetch calls. Likely files: the discovery adapters where URI.parse(server.url) is called, and extHostMcp.ts where uri.toString(true) is used for fetch.
