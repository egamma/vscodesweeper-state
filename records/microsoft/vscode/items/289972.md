# Issue #289972 — multi device folders in workspace

[https://github.com/microsoft/vscode/issues/289972](https://github.com/microsoft/vscode/issues/289972)

## Summary

Feature request to add folders from different tunneled/remote machines into a single multi-root workspace. This touches a fundamental architectural constraint — each VS Code window connects to a single remote authority. The request is underspecified and would benefit from clarification on the exact setup and workflow.

## Change summary

Allow workspace folders from multiple remote/tunneled devices in a single VS Code window.

## Best solution

This is a long-standing architectural limitation of VS Code's remote model (one remote authority per window). The request would need significant clarification — what tunnel type, what remote extensions are in use, what the expected workflow looks like. If the goal is AI context across codebases on different machines, MCP servers or workspace-level context files may be more practical paths. The reporter should clarify whether they are using Remote-SSH, Remote Tunnels, or another mechanism, and whether they've considered workarounds like mounting remote filesystems locally.
