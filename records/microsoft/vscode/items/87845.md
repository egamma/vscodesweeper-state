Team-authored feature request with 28 upvotes in Backlog. While shell integration was added to tasks (2022) improving output capture, and some error handling improvements landed, the core ask — explicitly surfacing which shell a task uses and providing richer failure diagnostics beyond exit codes — remains a valid gap. ## Change summary

Show which shell tasks run in and provide more diagnostic context than just an exit code when tasks fail.

## Best solution

Add a visible indicator (e.g. in the task terminal title or output preamble) showing which shell profile is being used for a task execution, and surface the actual error output (command-not-found, path issues) more prominently when a task exits with a non-zero code. The relevant code is in `src/vs/workbench/contrib/tasks/browser/terminalTaskSystem.ts` around line 1228 where the default profile is resolved.
