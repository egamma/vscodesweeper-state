# Issue #187046 — [wsl] Crash when launching from wsl with `--verbose`

[https://github.com/microsoft/vscode/issues/187046](https://github.com/microsoft/vscode/issues/187046)

## Summary

Still reproducible on main: `logToConsole` in the main process logger has no EPIPE error handling, so `--verbose` from WSL crashes when the stdout pipe breaks. The defect path is confirmed in source, but the scenario is narrow (WSLv1 + --verbose).

## Change summary

VS Code crashes with an unhandled EPIPE exception when launched from WSL with `--verbose` because console.log writes to a broken pipe.

## Best solution

Wrap `logToConsole` (or its callers in `ConsoleMainLogger`) with a try/catch that swallows EPIPE errors on write, or install a process-level handler for `uncaughtException` that ignores EPIPE on stdout/stderr. The fix is small and localized to `src/vs/platform/log/common/log.ts`.
