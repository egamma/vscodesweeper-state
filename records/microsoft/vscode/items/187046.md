# Issue #187046 — [wsl] Crash when launching from wsl with `--verbose`

[https://github.com/microsoft/vscode/issues/187046](https://github.com/microsoft/vscode/issues/187046)

## Summary

Still reproducible on main: ConsoleMainLogger's logToConsole does not catch EPIPE errors when the stdout pipe breaks (common in WSL v1 launches). No fix has landed.

## Change summary

VS Code crashes with EPIPE errors when launched from WSL with --verbose because console.log writes to a broken pipe.

## Best solution

Add an error handler for EPIPE on process.stdout/stderr in the main process startup (src/vs/code/electron-main/main.ts) or wrap logToConsole in a try/catch that silently swallows EPIPE, similar to how Node.js CLI tools handle broken pipes.
