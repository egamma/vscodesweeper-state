# Issue #322318 — Allow Git worktree entries in Source Control Repositories view to default collapsed

[https://github.com/microsoft/vscode/issues/322318](https://github.com/microsoft/vscode/issues/322318)

## Summary

Valid feature request with an open community PR (#322319) that addresses it. The non-explorer Repositories view currently hard-codes top-level repositories as always-expanded, ignoring saved collapsed state. Keeping open for the linked PR to be reviewed.

## Change summary

Allow Git worktree entries (and their parent repository nodes) to remain collapsed by default in the SCM Repositories view, preserving the user's collapsed state across sidebar reopens.

## Best solution

Review and land (or close with feedback) the author's open PR at https://github.com/microsoft/vscode/pull/322319, which makes the non-explorer Repositories view honor the saved `viewState.expanded` list for top-level repositories. The fix is localized to `collapseByDefault` in `src/vs/workbench/contrib/scm/browser/scmRepositoriesViewPane.ts`.
