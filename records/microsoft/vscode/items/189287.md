# Issue #189287 — Git - Problematic diff with nested repos

[https://github.com/microsoft/vscode/issues/189287](https://github.com/microsoft/vscode/issues/189287)

## Summary

Confirmed bug still present on main: the git extension's repository lookup picks the innermost nested repo for a file path without checking whether that repo actually tracks the file, causing incorrect diff content for files only tracked by an outer repo.

## Change summary

Git diff shows empty left-side content for files tracked by an outer repo when a nested (non-submodule) inner repo exists in the same directory tree.

## Best solution

Fix `getOpenRepository()` in `extensions/git/src/model.ts` (around line 982) to account for nested non-submodule repos — either by checking whether the candidate repo actually tracks the file, or by trying multiple matching repos when the deepest one returns empty content. The maintainer (lszomoru) already confirmed the root cause and intended to fix it.
