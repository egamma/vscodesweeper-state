# Issue #301539 — SCM view keeps deselecting my main repo

[https://github.com/microsoft/vscode/issues/301539](https://github.com/microsoft/vscode/issues/301539)

## Summary

Team-authored issue reporting that in a multi-repo worktree setup, the SCM view keeps reverting the selected repository. The activeRepository observable logic (driven by editor focus and focused-repository signals via `latestChangedValue`) likely resets the selection when agent activity triggers editor changes. Keeping open per team-member policy.

## Change summary

In a multi-repo worktree workspace with agents running, the SCM view repeatedly deselects the user's chosen repository and reverts to the default.

## Best solution

Investigate the interaction between `_activeEditorRepositoryObs` and `_focusedRepositoryObs` in `src/vs/workbench/contrib/scm/browser/scmViewService.ts` — when agent activity opens/modifies files in the other worktree, the active-editor signal likely wins over the user's manual selection. The pinned-repository mechanism (`_activeRepositoryPinnedObs`) should persist user selection, but it may not be set when the user right-clicks to reselect.
