# Issue #302043 — Terminal commands are always cropped after N character

[https://github.com/microsoft/vscode/issues/302043](https://github.com/microsoft/vscode/issues/302043)

## Summary

Still reproduces on main: terminal command titles in chat are hard-truncated at 50 characters (`MAX_COMMAND_TITLE_LENGTH`) regardless of available width. The team discussed CSS-based solutions (wrapping with line-clamp vs. responsive ellipsis) but hasn't reached consensus or shipped a fix.

## Change summary

Terminal command titles in the chat collapsible header are truncated at a fixed 50-character limit instead of using responsive CSS truncation.

## Best solution

Replace the hard-coded `MAX_COMMAND_TITLE_LENGTH = 50` JS truncation in `chatTerminalToolProgressPart.ts` with CSS-based responsive truncation — either `text-overflow: ellipsis` for single-line or `-webkit-line-clamp` for multi-line (as jrieken suggested). The team should settle the design question (pure ellipsis vs. wrap-then-clamp) first.
