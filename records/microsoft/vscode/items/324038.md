# Issue #324038 — Add overflow-y scrollbar to repo commit message hover-content, and option to move commit tag chips above commit message

[https://github.com/microsoft/vscode/issues/324038](https://github.com/microsoft/vscode/issues/324038)

## Summary

Two-part request: (1) the hover scrollbar already exists via the platform's DomScrollableElement with a default maxHeightRatio of 0.5, so long commit messages should already be scrollable — this may be a UX discoverability issue or the hover may be dismissed before scrolling can occur; (2) an option to reorder tag chips above the commit message is a new layout feature. Keeping open as a polish/feature request for the SCM area owner to evaluate.

## Change summary

Request to add a visible scrollbar in the SCM graph commit hover for long messages, and an option to show git tag chips above the commit message body.

## Best solution

The SCM area owner should evaluate: (1) whether the existing hover scrollbar is sufficient for long commit messages or if usability improvements are needed (e.g. the hover may dismiss on scroll attempts, or the max-height cap may not be applied correctly in the panel view), and (2) whether reordering tag chips above the commit body in the hover is desirable. The relevant code is in `src/vs/workbench/contrib/scm/browser/scmHistory.ts` (hover content construction) and `src/vs/workbench/contrib/scm/browser/scmHistoryViewPane.ts` (hover options).
