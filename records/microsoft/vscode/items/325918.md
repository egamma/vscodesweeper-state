## Summary

Valid feature request to make the macOS swipe gesture configurable — currently hardcoded to switch between recently used editors, but the user wants it to trigger Go Back/Forward (navigate through code locations). No duplicate found.

## Change summary

Allow the three-finger swipe gesture to navigate code locations (Go Back/Forward) instead of switching between recently used editors.

## Best solution

Add a setting (e.g. an enum on `workbench.editor.swipeToNavigate` or a new `workbench.editor.swipeNavigationAction` setting) that lets users choose between 'recentlyUsedEditors' and 'navigationHistory', then dispatch the appropriate command (`navigateBack`/`navigateForward` vs `openPreviousRecentlyUsedEditor`/`openNextRecentlyUsedEditor`) in `src/vs/platform/windows/electron-main/windowImpl.ts`.
