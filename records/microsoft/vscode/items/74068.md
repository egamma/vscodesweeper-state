# Issue #74068 — [css] Wrong selector specificity values with :nth-child(even) and :nth-child(odd)

[https://github.com/microsoft/vscode/issues/74068](https://github.com/microsoft/vscode/issues/74068)

## Summary

Fixed upstream in vscode-css-languageservice via PR microsoft/vscode-css-languageservice#369 (commit f55163f), and that fix is included in the v7.0.0-next.1 release which VS Code main currently depends on.

## Change summary

CSS selector specificity incorrectly counts `odd`/`even` keywords inside `:nth-child()` as element selectors, inflating specificity by (0,0,1).

## Best solution

Keep the shipped fix from vscode-css-languageservice. No further VS Code change needed — the dependency already includes the correction.
