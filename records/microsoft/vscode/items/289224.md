# Issue #289224 — vscode corrupts source code when toggling comments on folded regions

[https://github.com/microsoft/vscode/issues/289224](https://github.com/microsoft/vscode/issues/289224)

## Summary

A team member (aeschli) confirmed this already works correctly — the user needs to extend the selection to include the beginning of the next unfolded line to capture the folded block. The current behavior is intentional: selecting only the visible first line does not implicitly include hidden folded content.

## Change summary

Toggling comments on a folded region only comments the first visible line unless the selection explicitly spans past the fold.

## Best solution

Close as designed. The selection model intentionally treats a folded region as included only when the selection extends past it. The maintainer's video demonstrates the correct workflow.
