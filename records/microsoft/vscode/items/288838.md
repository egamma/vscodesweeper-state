# Issue #288838 — [Feature Request] Add frontmatter option to open .prompt.md files in a new chat session

[https://github.com/microsoft/vscode/issues/288838](https://github.com/microsoft/vscode/issues/288838)

## Summary

Valid feature request promoted to Backlog with 20+ upvotes. No `newChat` frontmatter option exists on current main — the prompt syntax service has no such property. The existing 'Run in New Chat' action requires a manual command/keybinding; the request is for a declarative frontmatter-driven approach.

## Change summary

Add a `newChat: true` frontmatter option to .prompt.md files so they automatically open in a fresh chat session instead of reusing the current one.

## Best solution

Extend the prompt frontmatter schema (likely in `src/vs/workbench/contrib/chat/common/promptSyntax/`) to support a `newChat` boolean, and wire it into `runPromptAction.ts` where the `inNewChat` parameter is already plumbed through. The infrastructure for running prompts in a new chat already exists (`RunCurrentPromptInNewChatAction`); this just needs a declarative trigger path.
