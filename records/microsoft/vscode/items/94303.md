# Issue #94303 — Timeline providers that don't need URI's should be able to render on empty workbench

[https://github.com/microsoft/vscode/issues/94303](https://github.com/microsoft/vscode/issues/94303)

## Summary

Still a valid, unimplemented feature request. The Timeline API's `provideTimeline` still requires a non-optional `URI` on current main, and the timeline pane still shows 'The active editor cannot provide timeline information' when no editor is open. The On Deck milestone is a holding bin and does not block closure, but there is no grounds to close — the gap is real, the request is clear, and a contributor expressed intent to work on it.

## Change summary

Allow Timeline providers to render workspace-level timeline items (e.g. git tags, sprints) without requiring an active text editor / URI.

## Best solution

Make the `uri` parameter optional in `TimelineProvider.provideTimeline` (both in the proposed extension API in `src/vscode-dts/vscode.proposed.timeline.d.ts` and the internal interface in `src/vs/workbench/contrib/timeline/common/timeline.ts`), and update `TimelinePane` in `src/vs/workbench/contrib/timeline/browser/timelinePane.ts` to query providers that opt into workspace-level timelines even when no editor is active. This is a moderate API design task that should be done before the Timeline API is finalized.
