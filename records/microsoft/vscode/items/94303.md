# Issue #94303 — Timeline providers that don't need URI's should be able to render on empty workbench

[https://github.com/microsoft/vscode/issues/94303](https://github.com/microsoft/vscode/issues/94303)

## Summary

Valid, unimplemented feature request. The Timeline API's `provideTimeline` still requires a non-optional `URI`, and the pane still early-returns when `uri` is undefined. Hard keep-open: milestone is 'On Deck' (a real scheduling milestone).

## Change summary

Allow Timeline providers to render entries (e.g. workspace-level events like git tags, sprints) when no active editor is open, by making the URI parameter optional.

## Best solution

Make the `uri` parameter optional on `TimelineProvider.provideTimeline` and on `ITimelineService.getTimeline`, update `TimelinePane` to query providers even when no editor is active, and add a UX toggle to switch between file-scoped and workspace-scoped timeline modes. The main files are `src/vs/workbench/contrib/timeline/common/timeline.ts` (API surface) and `src/vs/workbench/contrib/timeline/browser/timelinePane.ts` (pane logic). This also requires updating the extension API (`vscode.d.ts` `TimelineProvider`).
