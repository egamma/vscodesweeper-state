# Issue #189239 — Git - Source control picks up wrong git remote when using branch-level remote

[https://github.com/microsoft/vscode/issues/189239](https://github.com/microsoft/vscode/issues/189239)

## Summary

Still reproducible on main: when `gh pr checkout` sets `branch.<name>.remote` to a URL (not a named remote), VS Code's Git extension ignores it because `git for-each-ref %(upstream:remotename)` doesn't resolve URL-based remotes, causing push/sync to use the wrong remote.

## Change summary

Git source control pushes to the wrong remote when `branch.<name>.remote` is set to a URL (as done by `gh pr checkout`) instead of a named remote.

## Best solution

The `getBranch` method in `extensions/git/src/git.ts` should fall back to reading `branch.<name>.remote` and `branch.<name>.pushremote` via `git config` when `%(upstream:remotename)` returns empty, so that URL-based branch remotes are respected during push/sync operations.
