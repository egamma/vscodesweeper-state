# Issue #283516 — SCM repo tags Checkout bad dialog

[https://github.com/microsoft/vscode/issues/283516](https://github.com/microsoft/vscode/issues/283516)

## Summary

Confirmed bug, still present on main. The `_checkout` method's early-return path for a provided `treeish` (tag checkout) bypasses the dirty-worktree error handling that provides Stash/Migrate/Force options. An open PR (#304760) addresses this — keeping open for that PR to land.

## Change summary

Checking out a tag via SCM Repositories > Tags when the worktree is dirty shows a generic error dialog instead of the Stash & Checkout / Migrate Changes / Force Checkout options.

## Best solution

Land https://github.com/microsoft/vscode/pull/304760, which wraps the treeish-path checkout in `_checkout` with the same dirty-worktree error handling that the QuickPick-path already has.
