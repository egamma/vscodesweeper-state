# Issue #307824 — Git: Clone wizard skips destination folder selection step in VS Code Insiders

[https://github.com/microsoft/vscode/issues/307824](https://github.com/microsoft/vscode/issues/307824)

## Summary

Confirmed regression from commit a2a6d74b9fd ('Allow git clone command to also always use the cache'). When the repository cache has exactly one entry, `tryOpenExistingRepository` silently returns the cached path without offering the destination folder picker or the 'Clone again' option. An open PR (#309247) already addresses this — keeping open until it lands.

## Change summary

Git: Clone command skips the destination folder selection dialog when a cached clone of the repo already exists, silently reusing the cached path instead.

## Best solution

Land the open PR https://github.com/microsoft/vscode/pull/309247 which ensures the clone destination picker is always shown when a cached repo exists, giving users the choice between opening the existing clone or cloning again to a new location.
