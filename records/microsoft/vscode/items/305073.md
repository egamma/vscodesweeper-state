# Issue #305073 — [SCM] Staging multiple selected ranges fail and corrupts source

[https://github.com/microsoft/vscode/issues/305073](https://github.com/microsoft/vscode/issues/305073)

## Summary

Confirmed bug: staging multiple selected ranges at once corrupts the staged result because `applyLineChanges` in `extensions/git/src/staging.ts` constructs the patched content from the working-tree diff information, which becomes stale or is incorrectly applied when multiple disjoint intersections are fed in simultaneously. No fix has landed on main. Closely related to https://github.com/microsoft/vscode/issues/283148 (sequential staging), but this issue covers the distinct multi-select-at-once trigger path — both should be routed to the git area owner.

## Change summary

Staging multiple selected ranges at once via the SCM diff editor context menu fails to apply them correctly and can corrupt the staged content.

## Best solution

The root cause is in `extensions/git/src/staging.ts` (`applyLineChanges`) and the `stageSelectedChanges` method in `extensions/git/src/commands.ts` — the diff-to-patch logic doesn't correctly handle multiple disjoint selected ranges being applied in a single pass. The git area owner should fix the staging logic to correctly compose multiple partial hunks, and consolidate with https://github.com/microsoft/vscode/issues/283148 if they share the same root cause.
