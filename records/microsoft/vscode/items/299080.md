# Issue #299080 — JS: Incorrect indentation after comment line

[https://github.com/microsoft/vscode/issues/299080](https://github.com/microsoft/vscode/issues/299080)

## Summary

Valid bug with a clear reproduction path. An open fix PR (https://github.com/microsoft/vscode/pull/309259) exists but is not yet merged. The bracket-removal logic in `indentationLineProcessor.ts` should theoretically handle this, but the bug persists — likely due to tokenization timing or a code path that bypasses the processed-token logic. Keep open pending the fix PR.

## Change summary

Pressing Enter after a commented-out line containing an opening brace (e.g. `// const test = () => {`) incorrectly increases indentation on the next line.

## Best solution

Merge the fix in PR https://github.com/microsoft/vscode/pull/309259 (or an equivalent approach that ensures commented-out brackets don't trigger `increaseIndentPattern`). The root cause may be that the `increaseIndentPattern` regex in the JS/TS language configuration fires before the token-aware bracket-removal path can intervene, so a regex-level guard is the most robust fix.
