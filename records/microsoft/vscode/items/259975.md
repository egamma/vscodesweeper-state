# Issue #259975 — Custom Chat Agent Mode Reset on Edit

[https://github.com/microsoft/vscode/issues/259975](https://github.com/microsoft/vscode/issues/259975)

## Summary

Fixed on main and shipped in 1.108.0. The root cause was using `currentModeKind` (which collapses custom modes to their base kind) instead of `currentModeObs.get().id` (which preserves the custom mode identity) when entering and exiting edit mode. A commenter confirmed the fix in 1.108.0.

## Change summary

Editing a submitted chat prompt resets the chat mode from a custom agent to a built-in default mode, and canceling the edit also corrupts the mode selection for new messages.

## Best solution

Keep the shipped fix in commit 912018c9ea2 (1.108.0); no further change needed. The fix correctly propagates the full mode ID rather than just the mode kind during edit transitions.
