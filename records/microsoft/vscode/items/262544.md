# Issue #262544 — Task with process option prepends workspace directory

[https://github.com/microsoft/vscode/issues/262544](https://github.com/microsoft/vscode/issues/262544)

## Summary

Team-authored bug, still reproduces on main: `_resolveAndFindExecutable` calls `findExecutable` before checking `path.isAbsolute`, so absolute commands get the cwd prepended. An open PR exists to fix it.

## Change summary

Tasks with `"type": "process"` incorrectly prepend the workspace directory to absolute command paths.

## Best solution

Land (or close) PR https://github.com/microsoft/vscode/pull/262547 which moves the `path.isAbsolute` check before `findExecutable` in `src/vs/workbench/contrib/tasks/browser/terminalTaskSystem.ts`.
