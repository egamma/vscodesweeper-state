# Issue #301078 — Include remote execution context in Copilot's <environment_info>

[https://github.com/microsoft/vscode/issues/301078](https://github.com/microsoft/vscode/issues/301078)

## Summary

Valid feature request to surface VS Code's remote execution context (SSH, WSL, Dev Containers, Codespaces) in the <environment_info> block sent to Copilot's model. The info is already available via vscode.env.remoteName but not included in the prompt context.

## Change summary

Include the remote execution kind (local/ssh/wsl/devcontainer/codespaces/tunnel) in the <environment_info> block provided to Copilot's LLM context.

## Best solution

Extend the environment_info prompt construction in the Copilot extension to read `vscode.env.remoteName` (already exposed via `IEnvService.remoteName`) and include it as a field like `Remote context: ssh`. The change would likely live in the prompt-building code under `extensions/copilot/src/extension/prompts/` where the OS line is already emitted.
