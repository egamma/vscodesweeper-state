# Issue #88178 — Copying text from a Markdown document inserts non-breaking spaces instead of regular spaces

[https://github.com/microsoft/vscode/issues/88178](https://github.com/microsoft/vscode/issues/88178)

## Summary

Still reproduces on current main — the Windows-only nbsp insertion logic remains in `tokenizeLineToHTML`. Author is MEMBER, so keep-open is mandatory regardless.

## Change summary

Copying syntax-highlighted text from the editor on Windows inserts non-breaking spaces (alternating with regular spaces for consecutive whitespace) into the HTML clipboard, which breaks pasting into other applications.

## Best solution

Remove or replace the `platform.isWindows` nbsp workaround in `tokenizeLineToHTML` — the `white-space: pre-wrap` CSS already on the copied HTML root should prevent browser whitespace collapsing without needing nbsp characters. The fix is localized to `src/vs/editor/common/languages/textToHtmlTokenizer.ts` (the `useNbsp` logic) and the call site in `src/vs/editor/common/viewModel/viewModelImpl.ts` line 1129.
