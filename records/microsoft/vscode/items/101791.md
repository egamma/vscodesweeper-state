# Issue #101791 — Allow DAP progress notifications to be shown more prominently than the status bar

[https://github.com/microsoft/vscode/issues/101791](https://github.com/microsoft/vscode/issues/101791)

## Summary

Valid feature request with strong community support (51 upvotes). Current main shows all DAP progress as notifications uniformly, but there's no mechanism for debug adapters to indicate prominence level — the gap the issue describes remains.

## Change summary

Allow debug adapters to control how prominently DAP progress notifications are displayed, distinguishing background activity from user-initiated operations.

## Best solution

Extend the DAP ProgressStartEvent (or add a VS Code-specific extension) with a prominence/location hint so adapters can request notification-level vs. status-bar-level display. Implementation would touch `src/vs/workbench/contrib/debug/browser/debugProgress.ts` to read the hint and select the appropriate `ProgressLocation`. This likely also requires upstream DAP protocol discussion.
