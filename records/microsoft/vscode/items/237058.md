# Issue #237058 — "The window terminated unexpectedly (reason 'launch-failed' code '159')

[https://github.com/microsoft/vscode/issues/237058](https://github.com/microsoft/vscode/issues/237058)

## Summary

Upstream Electron/Chromium sandbox issue: exit code 159 (SIGSYS from seccomp-bpf) kills the renderer when loading a custom product icon theme font on certain Linux distros. A team member confirmed the Electron font-loading root cause. The issue is still open on main with no VS Code-side fix, and the crash path is in Electron's sandbox layer, not in VS Code's theme code.

## Change summary

VS Code crashes with exit code 159 on Fedora/Arch Linux when a product icon theme extension loads a custom font under the Chromium sandbox.

## Best solution

The root cause is in Electron/Chromium's seccomp sandbox disallowing a syscall triggered during font loading on newer Fedora/Arch configurations. VS Code could consider a defensive fallback (revert to default product icon theme on renderer crash) or file/track this upstream in the Electron issue tracker. The `--no-sandbox` workaround confirms the sandbox layer owns this.
