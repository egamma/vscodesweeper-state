# Issue #249924 — A way to distinguish inputfocus between rename file and find in explorer?

[https://github.com/microsoft/vscode/issues/249924](https://github.com/microsoft/vscode/issues/249924)

## Summary

Valid feature request: the explorer rename widget's dismiss is hardcoded to Escape with no command or context key, so users cannot rebind it. The issue is already labeled `under-discussion` with team engagement. Keeping open — no traction yet (0 upvotes) but it's a reasonable ergonomic gap.

## Change summary

Request a command and/or context key to dismiss the rename input in the file explorer, so it can be rebound independently of the find widget.

## Best solution

Expose the rename dismiss as a registered command (e.g. alongside `list.closeFind`) and add a context key like `treeRenameOpen` so users can bind a single key to either close rename or close find depending on which is active. The relevant code is in `src/vs/workbench/contrib/files/browser/views/explorerViewer.ts` around line 1137 where Escape is hardcoded.
