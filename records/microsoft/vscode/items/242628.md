# Issue #242628 — Feature Request: Persist Extension List Sorting in the Extensions Sidebar

[https://github.com/microsoft/vscode/issues/242628](https://github.com/microsoft/vscode/issues/242628)

## Summary

Valid feature request that passed the community upvote threshold (21 votes) and was promoted to Backlog. The sort order is not persisted on current main — the sort state lives only in the search box query string and context keys, with no storage or setting to remember it across sessions.

## Change summary

Persist the user's chosen extension list sort order across VS Code restarts, via a setting like `extensions.defaultSortOrder`.

## Best solution

Add persistence for the extensions sort selection — either a dedicated setting (`extensions.defaultSortOrder`) or viewlet-level storage that remembers the last-used sort. The implementation would touch `src/vs/workbench/contrib/extensions/browser/extensionsViewlet.ts` (where sort is applied to the search query) and possibly the extensions configuration registration in `extensions.contribution.ts`.
