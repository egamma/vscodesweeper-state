# Issue #241855 — Bash env doesn't update on first `unset`

[https://github.com/microsoft/vscode/issues/241855](https://github.com/microsoft/vscode/issues/241855)

## Summary

Still reproduces on current main: the bash shell integration script never sends `EnvSingleDelete` when a tracked variable is unset, so the first unset is silently ignored because the pending env still carries the variable from the copy. The bug is in the bash script's env reporting logic and the `ShellEnvDetectionCapability`'s reliance on delete signals that never arrive.

## Change summary

When a bash env variable is unset, the `onDidChangeTerminalShellIntegration` event does not fire on the first unset — only on a subsequent unset.

## Best solution

The bash shell integration script (`src/vs/workbench/contrib/terminal/common/scripts/shellIntegration-bash.sh`) needs to detect when a tracked variable is no longer set and either send `EnvSingleDelete` for it or use `EnvSingleStart` with `clear=1` and re-send only existing variables. The comments at lines 322 and 344 explicitly say 'Track missing env vars not needed for now' — removing that limitation would fix this bug.
