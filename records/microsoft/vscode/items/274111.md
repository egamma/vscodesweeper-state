# Issue #274111 — Switching between agent types breaks if images present in history

[https://github.com/microsoft/vscode/issues/274111](https://github.com/microsoft/vscode/issues/274111)

## Summary

Valid bug — switching to a non-vision model mid-conversation sends image attachments from earlier turns in the history, causing a 400 error. The UI warns about unsupported images on the current input, but does not filter images out of history turns before the request is sent. Already correctly labeled; keeping open.

## Change summary

Switching the chat agent to a model without vision support (e.g. Grok) after attaching images in an earlier turn causes a 400 Bad Request because images remain in the history payload.

## Best solution

When building the chat history for a request, filter out (or replace with a placeholder) image content parts when the selected model's `supportsVision` is false. The filtering logic likely belongs in `src/vs/workbench/contrib/chat/browser/chatService.ts` or in the Copilot extension's request-building path (where history turns are assembled for the API call). The existing `modelSupportsVision` helper in `chatAttachmentWidgets.ts` already exposes the needed capability check.
