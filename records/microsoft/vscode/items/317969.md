# Issue #317969 — Terminal renders as solid color blocks after switching macOS Spaces; text becomes visible only when selected

[https://github.com/microsoft/vscode/issues/317969](https://github.com/microsoft/vscode/issues/317969)

## Summary

Confirmed WebGL terminal rendering bug on macOS after Space switches or sleep — two reporters reproduce it on current builds (1.124.2). No fix found on main; the existing `onContextLoss` handler doesn't cover this case because the WebGL context remains valid while the texture atlas content becomes stale.

## Change summary

Terminal renders as solid color blocks (no glyphs) after switching macOS Spaces or waking from sleep; text reappears only when selected or scrolled.

## Best solution

Add a `visibilitychange` or Electron `browser-window-focus` listener in the terminal's WebGL renderer lifecycle that triggers `forceRedraw()` (which calls `clearTextureAtlas()`) when the window regains visibility/focus after an OS compositing pause. The fix belongs in `src/vs/workbench/contrib/terminal/browser/xterm/xtermTerminal.ts` near the existing WebGL addon setup.
