# Issue #298050 — Copilot extension chat history is suddenly mixed up with Codex extension chat history

[https://github.com/microsoft/vscode/issues/298050](https://github.com/microsoft/vscode/issues/298050)

## Summary

Valid report about Codex extension sessions appearing in Copilot's chat history picker, causing confusion and broken opening behavior. Several related issues exist under the codex-agent label but none is an exact duplicate — routing to the right area owners for triage.

## Change summary

Codex extension chat sessions appear mixed into Copilot Chat's history list, and selecting a Codex session from Copilot's history opens it as a raw file tab instead of in the Codex sidebar.

## Best solution

Route to the chat-agents-view / codex-agent owners. The chat session store and history UI should filter or group sessions by provider so that Copilot's history picker only shows Copilot-originated sessions. Related issues https://github.com/microsoft/vscode/issues/311165 and https://github.com/microsoft/vscode/issues/300191 track overlapping session-separation pain points and could be consolidated.
