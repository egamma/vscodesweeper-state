# Issue #324487 — macOS: show recent workspaces (incl. remote-SSH) in the dock menu, for parity with Windows JumpList

[https://github.com/microsoft/vscode/issues/324487](https://github.com/microsoft/vscode/issues/324487)

## Summary

Well-researched feature request with no existing duplicate. The macOS dock menu is hardcoded to a single 'New Window' item (menubar.ts L294-297), while the Windows JumpList already iterates all recent workspaces including remote URIs (workspacesHistoryMainService.ts L377). The parity gap is real and confirmed in the current source.

## Change summary

Populate the macOS dock menu with recent workspaces (including remote-SSH and other remote schemes) to match the existing Windows JumpList behavior.

## Best solution

Extend the dock menu construction in `src/vs/platform/menubar/electron-main/menubar.ts` (~L290) to populate recent workspace entries via `app.dock.setMenu()`, mirroring the pattern used by `updateWindowsJumpList()` in `workspacesHistoryMainService.ts`. The `updateMacOSRecentDocuments()` filter to `file://` is correct for the `app.addRecentDocument` API (macOS constraint), but `app.dock.setMenu()` is fully custom and can include remote URIs as click-handler items with `--folder-uri` args — no scheme restriction.
