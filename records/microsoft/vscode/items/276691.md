# Issue #276691 — Source Control GUI doesn't prompt for yubikey auth on push/pull

[https://github.com/microsoft/vscode/issues/276691](https://github.com/microsoft/vscode/issues/276691)

## Summary

Still reproduces on main: `handleSSHAskpass()` in the git extension only handles passphrase and host-authenticity prompts, falling through to the authenticity branch on keyboard-interactive challenges (e.g. YubiKey), which crashes or returns an empty response. An open PR (#299382) addresses this but has not merged.

## Change summary

SSH keyboard-interactive auth prompts (e.g. YubiKey) are not handled by the Source Control GUI's askpass helper, causing push/pull to fail with 'Permission denied (keyboard-interactive)'.

## Best solution

Review and land the open PR https://github.com/microsoft/vscode/pull/299382, which adds a keyboard-interactive case to `handleSSHAskpass()` in `extensions/git/src/askpass.ts`. The fix is localized to that function and the approach (forwarding the prompt via InputBox, detecting FIDO2 touch prompts) looks sound.
