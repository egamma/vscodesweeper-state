# Issue #260137 — Chat modes are not persistet when editing message

[https://github.com/microsoft/vscode/issues/260137](https://github.com/microsoft/vscode/issues/260137)

## Summary

Team-authored issue; the bug is confirmed in the current source. When editing a previous request, `clickedRequest()` sets the inline input's mode to the current input part's mode rather than the mode stored in the request's `modeInfo`, so custom modes are lost.

## Change summary

When editing a previously sent message, the chat mode resets to the current input's mode instead of preserving the mode that was originally used for that request.

## Best solution

In `clickedRequest()` in `chatWidget.ts`, restore the mode from `currentElement.modeInfo` (available on `IChatRequestModel`) rather than `this.inputPart.currentModeObs.get().id`. The model ID restoration at line 1778 already follows this pattern — mode restoration should parallel it. Both the inline-edit (`!isInput`) and input-part (`isInput`) paths need the fix. Recent commits `1100b7d8c29` and `4293b057ca2` fixed adjacent model-syncing issues but did not address mode restoration.
