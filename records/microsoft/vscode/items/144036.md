# Issue #144036 — Feature Request: Add way of getting all the color presentations in a specific range of a document in the vscode API

[https://github.com/microsoft/vscode/issues/144036](https://github.com/microsoft/vscode/issues/144036)

## Summary

The most common duplicate-color-picker scenario (built-in vs extension) was mitigated in late 2024 by the `editor.defaultColorDecorators` setting (auto/always/never), but the broader API request — letting extensions query existing color presentations at a range to avoid inter-extension duplicates — remains unimplemented.

## Change summary

Request for an extension API to retrieve all color presentations already registered at a given document range, so extensions can avoid showing duplicate color decorators.

## Best solution

The `editor.defaultColorDecorators: 'auto'` setting (shipped ~Nov 2024, commit bcd6c1dbc37a) solves the built-in-vs-extension conflict. For extension-vs-extension deduplication, a new API or enhancement to `vscode.executeDocumentColorProvider` that returns provider metadata per range would be needed. The team should decide whether that remaining gap justifies a new API surface or whether the existing mitigation is sufficient to close this.
