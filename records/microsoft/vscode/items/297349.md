# Issue #297349 — Chat panel should virtualize DOM for long conversations (performance degrades over time)

## Summary

The chat message list already uses WorkbenchObjectTree (a virtualized tree widget), so the core premise — 'keeps all messages in the DOM' — is incorrect. The real performance concern in long agent sessions is valid but stems from within-message complexity (code blocks, diff editors, tool outputs, webviews), not from lack of list-level virtualization. This is a valid performance improvement request but should be routed to the chat-perf area for proper investigation.
