## Summary

Legitimate feature request with 25 upvotes, promoted to Backlog by the community process. Keep open for prioritization.

## Change summary

Add a selective data masking/redaction layer to Copilot Chat that replaces sensitive filename and content spans with placeholders before sending prompts to the LLM, as a finer-grained alternative to .copilotignore.

## Best solution

The feature belongs in the Copilot extension's prompt-assembly pipeline (around `extensions/copilot/src/platform/ignore/`), where content exclusion already lives. A masking pass would intercept file references and content after context gathering but before LLM submission, applying user-defined regex/glob rules. This is a significant design effort touching trust, enterprise policy, and the prompt pipeline — it needs a product decision on scope (e.g. whether built-in patterns ship, how masking interacts with agent tool reads, and whether this is a Copilot extension or core platform concern).
