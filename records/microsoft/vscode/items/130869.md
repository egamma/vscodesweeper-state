# Issue #130869 — Adjust auto-highlighting of the cursor position to the setting at select.

[https://github.com/microsoft/vscode/issues/130869](https://github.com/microsoft/vscode/issues/130869)

## Summary

Keep open as a valid editor word-highlighting feature request; current main still has code paths that do not fully apply editor-configured word separators to textual occurrence highlighting.

## Change summary

Use `editor.wordSeparators` when auto-highlighting the word at the cursor so Japanese and other non-space-delimited text does not highlight an entire paragraph.

## Best solution

Route through the editor contribution owner and make textual occurrence highlighting honor the configured word separators consistently, especially in `src/vs/editor/contrib/wordHighlighter/browser/textualHighlightProvider.ts` and the word range checks in `wordHighlighter.ts`.
