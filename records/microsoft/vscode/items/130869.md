# Issue #130869 — Adjust auto-highlighting of the cursor position to the setting at select.

[https://github.com/microsoft/vscode/issues/130869](https://github.com/microsoft/vscode/issues/130869)

## Summary

Valid feature request: the word highlighter uses the language's wordDefinition regex (via model.getWordAtPosition) rather than the user-configurable editor.wordSeparators setting, which causes poor word boundary detection for CJK text. The gap remains on current main.

## Change summary

Word highlighting at cursor position should respect editor.wordSeparators so CJK users can configure proper word boundaries instead of highlighting entire paragraphs.

## Best solution

Modify the word highlighter's word-at-position logic (in _getCurrentWordRange and the textual highlight provider) to optionally use editor.wordSeparators for word boundary detection, or introduce a dedicated setting for highlight word boundaries. This lives in src/vs/editor/contrib/wordHighlighter/browser/wordHighlighter.ts and textualHighlightProvider.ts.
