# Issue #280266 — SELinux is preventing pasta from 'append' accesses to log file inside $HOME/.config/Code/logs/...

[https://github.com/microsoft/vscode/issues/280266](https://github.com/microsoft/vscode/issues/280266)

## Summary

This is an SELinux policy issue on Fedora, not a VS Code code defect. VS Code writes logs to `~/.config/Code/logs/` (standard XDG-compliant path via userDataPath); the root cause is that Fedora's SELinux policy for the `pasta_t` domain does not permit append access to `config_home_t` files. The fix belongs in Fedora's SELinux policy packages or Podman's pasta tool, not in VS Code.

## Change summary

SELinux on Fedora blocks Podman's `pasta` networking tool from appending to VS Code log files under `~/.config/Code/logs/`.

## Best solution

Report to Fedora's SELinux policy maintainers (the Fedora Bugzilla at https://bugzilla.redhat.com/show_bug.cgi?id=2374200 is already the right venue) to update the `pasta_t` policy to allow append access to `config_home_t` files, or to Podman upstream to prevent pasta from inheriting file descriptors to log files. VS Code's log path is standard and correct; changing it would not be appropriate.
