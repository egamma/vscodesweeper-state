# Issue #166944 — [css] Please add an option to switch to using line-by-line comments

[https://github.com/microsoft/vscode/issues/166944](https://github.com/microsoft/vscode/issues/166944)

## Summary

Legitimate feature request promoted to Backlog with 20+ upvotes. No implementation found on current main — no setting like `forceLineComment` or `preferLineComment` exists. A workaround (multi-cursor) and an extension (Toggle Line Comments) exist, but the core request for a built-in option remains unaddressed.

## Change summary

Request an option to use line-by-line `/* */` comments instead of a single block comment when toggling comments on multi-line CSS selections.

## Best solution

Either add a language-level configuration (e.g. `editor.commentStyle` or a CSS-language-specific override) that tells the comment command to produce per-line block comments instead of wrapping the selection, or decide this belongs in extension-land (the Toggle Line Comments extension already solves it). The comment command logic lives in `src/vs/editor/contrib/comment/browser/lineCommentCommand.ts`.
