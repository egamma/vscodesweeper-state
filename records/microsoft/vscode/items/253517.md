# Issue #253517 — More options when opening vscode:chat-prompt links in VS Code

[https://github.com/microsoft/vscode/issues/253517](https://github.com/microsoft/vscode/issues/253517)

## Summary

Team-authored feature request for richer options when opening vscode:chat-prompt links (open-only without saving, add-to-chat). Active discussion between members; needs explicit maintainer judgment.

## Change summary

Opening a vscode:chat-prompt link currently only allows saving; user wants lighter options like opening without persisting or inserting content directly into chat.

## Best solution

Add a picker or dialog when handling vscode:chat-prompt links that offers: (1) save as prompt file (current behavior), (2) open as untitled/temporary editor without persisting, (3) insert content into the active chat input box as a one-shot prompt. The UX details (especially option 3's interaction with slash commands) need design consensus from the chat-prompts area owners.
