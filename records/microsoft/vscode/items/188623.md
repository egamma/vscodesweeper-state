# Issue #188623 — [html] Links with fragment can't be Ctrl+clicked

[https://github.com/microsoft/vscode/issues/188623](https://github.com/microsoft/vscode/issues/188623)

## Summary

Still reproduces on main: `resolveReference` in `documentContext.ts` passes the raw href (including `#fragment`) to `Utils.resolvePath`, which encodes `#` as `%23` in the path instead of treating it as a fragment separator. The fix in vscode-html-languageservice#180 only strips fragments that are already correctly parsed, so it never triggers.

## Change summary

Ctrl+clicking an HTML link with a fragment identifier (e.g. `bar.html#baz`) fails because the fragment gets encoded into the file path.

## Best solution

In `extensions/html-language-features/server/src/utils/documentContext.ts`, split the fragment from `ref` before calling `Utils.resolvePath`, then re-attach it to the resolved URI. This ensures the URI is well-formed before `validateAndCleanURI` processes it.
