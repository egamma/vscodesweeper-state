# Issue #309548 — [Unhandled Error] Cannot resolve canonical URI

[https://github.com/microsoft/vscode/issues/309548](https://github.com/microsoft/vscode/issues/309548)

## Summary

Keep open: the reported telemetry error is still present in the current checkout and later stable-scan comments show continued spikes, so there is no strong basis for a cleanup close.

## Change summary

Canonical URI resolution throws `Cannot resolve canonical URI` when the local extension host path cannot provide a proxy or resolver.

## Best solution

Decide in the extension host/native extension service remote-authority flow whether missing or disposed local extension hosts should return `null`/fall back instead of throwing, and reduce this telemetry bucket once the intended behavior is explicit.
