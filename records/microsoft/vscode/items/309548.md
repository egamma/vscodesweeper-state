# Issue #309548 — [Unhandled Error] Cannot resolve canonical URI

[https://github.com/microsoft/vscode/issues/309548](https://github.com/microsoft/vscode/issues/309548)

## Summary

Still present on main: `extensionHostManager.ts:420` throws when the proxy is null instead of returning null gracefully. The caller at `nativeExtensionService.ts:335` already handles null returns correctly (lines 337-340 filter them), so this throw is unnecessary and hits thousands of users per stable release.

## Change summary

Extension host manager throws 'Cannot resolve canonical URI' when its proxy is null during remote authority resolution, instead of returning null.

## Best solution

Change `extensionHostManager.ts:420` to `return null;` instead of throwing — the caller (`nativeExtensionService.ts:335`) already iterates results and skips nulls. This mirrors the pattern already used by `start()` at line 427 of the same file, which returns early when proxy is null.
