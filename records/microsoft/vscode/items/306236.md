# Issue #306236 — Incorrect Minimap Vertical Size

[https://github.com/microsoft/vscode/issues/306236](https://github.com/microsoft/vscode/issues/306236)

## Summary

Team-authored bug report about minimap 'fit' mode showing incorrect vertical size when switching tabs. The layout logic in `editorOptions.ts` has a complex stabilization mechanism (the `memory.stableFitRemainingWidth` / `stableFitMaxMinimapScale` path) that may be persisting stale layout state across tab switches, causing the wrong scale to be applied initially.

## Change summary

Minimap in 'fit' mode briefly shows an incorrect intermediate size (between 'fit' and 'fill') when switching to a new file tab.

## Best solution

Investigate whether the `memory.stableFitRemainingWidth` / `stableFitMaxMinimapScale` stabilization path in `EditorLayoutInfoComputer` (around line 2793–2832 of `editorOptions.ts`) incorrectly carries over stale layout decisions when the active editor changes. The fix likely involves resetting the minimap layout memory when the editor model changes, or ensuring the first layout pass for a new tab doesn't inherit the previous tab's stabilization constraints.
