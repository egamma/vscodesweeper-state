# Issue #44612 — [razor] comment out razor code with @* *@

[https://github.com/microsoft/vscode/issues/44612](https://github.com/microsoft/vscode/issues/44612)

## Summary

Still a valid gap on current main: the built-in razor extension defines only `<!-- -->` as blockComment; `@* *@` Razor-specific commenting is not supported. Context-aware comment toggling in mixed-language files remains unimplemented.

## Change summary

Commenting in .cshtml/Razor files should use `@* *@` for Razor code regions instead of only `<!-- -->` or `//`.

## Best solution

Either update `extensions/razor/language-configuration.json` to use `@* *@` as the blockComment (simple but loses HTML comment support), or implement context-aware comment toggling that picks the right comment style based on embedded language scope. The latter is complex and may belong in the C# DevKit / Razor Language Server extension rather than VS Code core.
