# Issue #44612 — [razor] comment out razor code with @* *@

[https://github.com/microsoft/vscode/issues/44612](https://github.com/microsoft/vscode/issues/44612)

## Summary

The built-in Razor extension's language-configuration.json defines `blockComment` as `<!-- -->` but has no support for `@* *@` Razor comments. The request for context-aware comment toggling (Razor vs HTML) remains unaddressed on main. Low community traction (1 upvote over 8 years).

## Change summary

Commenting out Razor code in .cshtml files should use the Razor-specific `@* *@` block comment syntax instead of only HTML `<!-- -->`.

## Best solution

The built-in Razor extension (`extensions/razor/language-configuration.json`) could be updated to use `@* *@` as the blockComment, but true context-aware commenting (HTML regions use `<!-- -->`, Razor regions use `@* *@`) requires embedded-language comment support which VS Code's comment toggling doesn't natively handle per-region. The C# Dev Kit / Razor Language Server extension is the more appropriate place to implement this via a richer language service. Consider closing as `*extension-candidate` if the team decides this belongs in the C# extension rather than core.
