# Issue #303139 — When creating chat background session, copy node_modules

[https://github.com/microsoft/vscode/issues/303139](https://github.com/microsoft/vscode/issues/303139)

## Summary

Valid feature request to copy or symlink node_modules into worktrees created for background agent sessions, avoiding expensive npm install runs. No implementation exists on main. Routing to the copilot-cli-agent area.

## Change summary

When creating a chat background session worktree, copy (or symlink) node_modules from the source repository so npm tasks work immediately without a separate npm install.

## Best solution

In `ChatSessionWorktreeService._createWorktree` (extensions/copilot/src/extension/chatSessions/vscode-node/chatSessionWorktreeServiceImpl.ts), after the worktree is created, symlink or copy node_modules from the source repo root into the new worktree path. A symlink would be fastest and most resource-efficient; a full copy is the safe fallback. This is a product decision for the agents team — symlink vs copy, handling of monorepos with nested node_modules, and whether to make it opt-in.
