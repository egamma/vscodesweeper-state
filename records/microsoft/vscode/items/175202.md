# Issue #175202 — Debug with f5 never starts and caused extraneous extension activation 

[https://github.com/microsoft/vscode/issues/175202](https://github.com/microsoft/vscode/issues/175202)

## Summary

Still reproduces on main: resolving a named preLaunchTask (e.g. "npm: compile") falls back to activating ALL task providers because the type is unknown at that point, causing extraneous extension activation (Docker, Java, C++, etc.).

## Change summary

Pressing F5 with a named preLaunchTask activates unrelated extensions (Docker, Java, C++) because the task resolution path activates all task providers when it cannot find the task in workspace configuration.

## Best solution

When `getTask` falls back to `_getGroupedTasks` (line 986 of abstractTaskService.ts), infer the task type from the identifier string (e.g. "npm: compile" → type "npm") before activating providers, so only the relevant task provider is activated instead of all registered providers. The debug runner at `debugTaskRunner.ts:202` should also pass the type when it can be parsed from the task identifier.
