# Issue #154529 — Add option to display Selection Highlights across Split Editor panes

[https://github.com/microsoft/vscode/issues/154529](https://github.com/microsoft/vscode/issues/154529)

## Summary

Valid feature request with 59 upvotes (well above the 20-vote Backlog threshold). The existing `occurrencesHighlight: "multiFile"` setting only covers semantic/document-highlight-provider results, not the general selection highlight (arbitrary text selection matches). The `SelectionHighlighter` contribution in `multicursor.ts` still operates per-editor with no cross-pane logic on current main. The request remains unimplemented.

## Change summary

Extend the selection-highlight feature (arbitrary text match highlighting) to display matches across split editor panes showing the same or different files.

## Best solution

Add a multi-file mode to `SelectionHighlighter` (in `src/vs/editor/contrib/multicursor/browser/multicursor.ts`) analogous to the existing `occurrencesHighlight: "multiFile"` support in `WordHighlighter`. A new setting like `editor.selectionHighlight` accepting `"singleFile" | "multiFile"` (or a separate boolean) would gate the behavior. The implementation would query other visible editors' models for the same matches and set decorations on them, mirroring the pattern in `wordHighlighter.ts:getOtherModelsToHighlight`.
