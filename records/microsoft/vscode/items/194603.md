# Issue #194603 — Adopt dropdown action items to use sub menus

[https://github.com/microsoft/vscode/issues/194603](https://github.com/microsoft/vscode/issues/194603)

## Summary

Team-authored debt issue, partially addressed. The remote explorer has adopted the `isSelection` sub-menu pattern, but the debug viewlet (launch config dropdown), debug console (process selection), and terminal view (switch terminal dropdown) have not. Keeping open per team-authored rule and because work remains.

## Change summary

Dropdown actions in several views break when hidden to the overflow menu; they should use sub menus (the `isSelection` pattern) so they remain functional.

## Best solution

Adopt the `isSelection` sub-menu pattern (already used in output and remote explorer) in the remaining components: debug viewlet launch config dropdown (`src/vs/workbench/contrib/debug/browser/debugViewlet.ts`), debug console process dropdown (`src/vs/workbench/contrib/debug/browser/repl.ts`), and terminal view switch dropdown (`src/vs/workbench/contrib/terminal/browser/terminalView.ts`). The terminal case may be deprioritized given Tyriar's comment that the affected code path is effectively deprecated.
