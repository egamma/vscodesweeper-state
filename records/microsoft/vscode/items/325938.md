# Issue #325938 — Linux: Window WM_CLASS is "code" while code.desktop StartupWMClass is "Code", causing taskbar icon mismatch on Cinnamon

[https://github.com/microsoft/vscode/issues/325938](https://github.com/microsoft/vscode/issues/325938)

## Summary

Legitimate bug: the desktop file template sets StartupWMClass to `@@NAME_SHORT@@` (resolves to "Code" with uppercase C for the official build), but Electron derives WM_CLASS from the executable name ("code", lowercase). Case-sensitive desktop environments like Cinnamon cannot match the window to the launcher, producing a duplicate taskbar icon. No existing canonical issue tracks this for the official VS Code build.

## Change summary

On Linux Cinnamon, the VS Code window WM_CLASS ("code") does not match the desktop file's StartupWMClass ("Code") due to case difference, causing a second taskbar icon with a generic icon instead of the VS Code icon.

## Best solution

Align the WM_CLASS and StartupWMClass values. Either change `resources/linux/code.desktop` line 9 to use a placeholder that resolves to the lowercase application name matching what Electron sets as WM_CLASS, or set WM_CLASS on the Electron side to match `nameShort`. The 2020 fix for Code-OSS (#111327) addressed the same class of bug — apply the same reasoning to the official build. The fix also needs to account for Code-Insiders and any other product variants.
