# Issue #315497 — Integrated Terminal: Backspace fails to remove multibyte UTF-8 characters from input buffer in Docker (Full characters remain as ghost bytes)

[https://github.com/microsoft/vscode/issues/315497](https://github.com/microsoft/vscode/issues/315497)

## Summary

The reported behavior is likely caused by the Docker container's TTY lacking the IUTF8 flag, which governs multi-byte backspace at the kernel line-discipline level — not by VS Code's terminal. VS Code's terminal pass-through (`terminalProcess.ts:484`) writes keystrokes verbatim to node-pty without character-width processing, which is standard terminal-emulator behavior. However, the report is detailed enough to keep open for the terminal team to evaluate whether VS Code could/should work around this upstream TTY configuration gap.

## Change summary

Backspace in VS Code's integrated terminal only removes one byte (not one full multi-byte UTF-8 character) from the input buffer inside Docker containers, leaving ghost bytes.

## Best solution

The terminal team should evaluate whether this is purely a Docker IUTF8 configuration issue (same behavior in any terminal emulator) or whether VS Code can reasonably work around it. If upstream, close as `*as-designed` or `upstream` with guidance to run `stty iutf8` in the container. If VS Code can help (e.g. ensuring the IUTF8 flag propagates), that would be a terminal-input enhancement.
