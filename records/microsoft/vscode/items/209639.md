# Issue #209639 — Diff editor sides are not equally wide

[https://github.com/microsoft/vscode/issues/209639](https://github.com/microsoft/vscode/issues/209639)

## Summary

Still present on main: the default 0.5 split ratio places the sash at the visual midpoint, but the gutter (35px of action buttons) is subtracted only from the original (left) editor width, making it noticeably narrower. Confirmed by a commenter on 1.115.0.

## Change summary

Diff editor left (original) side is narrower than the right (modified) side because the gutter width is deducted only from the left.

## Best solution

Adjust `SashLayout._computeSashLeft` (or the default ratio) to account for the gutter width so that both editors end up with equal visible content width at the default position. The fix lives in `src/vs/editor/browser/widget/diffEditor/components/diffEditorSash.ts` and/or the layout block in `diffEditorWidget.ts` (lines ~140–150).
