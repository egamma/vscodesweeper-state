# Issue #149622 — Git - After VSCode is updated, git@github.com: Permission denied (publickey).

[https://github.com/microsoft/vscode/issues/149622](https://github.com/microsoft/vscode/issues/149622)

## Summary

Verified duplicate: the core problem — VS Code's integrated askpass overriding the user's ssh-agent for Git SSH operations — is the same bug tracked by the older canonical issue #57488, which has more discussion, workarounds, and covers both Linux and macOS.

## Change summary

VS Code's built-in Git operations fail with 'Permission denied (publickey)' when using SSH remotes because the integrated askpass bypasses the user's ssh-agent.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/57488 — both track the same underlying problem where `SSH_ASKPASS_REQUIRE=force` in `extensions/git/src/askpass.ts` overrides the user's ssh-agent. The fix belongs in the git extension's SSH environment handling. Users can work around it today by setting `git.useIntegratedAskPass: false`.
