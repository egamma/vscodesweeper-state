# Issue #249690 — [html] Rename Symbol doesn't work on attribute names

[https://github.com/microsoft/vscode/issues/249690](https://github.com/microsoft/vscode/issues/249690)

## Summary

The HTML language service's doRename only handles tag names; renaming attribute names is an unimplemented feature. Already correctly labeled as feature-request; stays open for potential upstream work.

## Change summary

Renaming an HTML attribute name via F2 yields "No result" because the rename provider only supports tag names.

## Best solution

Extend `doRename` in the upstream `microsoft/vscode-html-languageservice` (src/services/htmlRename.ts) to detect when the cursor is on an attribute name and produce edits for all occurrences of that attribute within the same tag or across matching tags. Until then, the feature-request stays open.
