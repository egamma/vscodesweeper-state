# Issue #249690 — [html] Rename Symbol doesn't work on attribute names

[https://github.com/microsoft/vscode/issues/249690](https://github.com/microsoft/vscode/issues/249690)

## Summary

Confirmed feature gap: the HTML language service's doRename only handles tag names. Attribute rename support was never implemented in vscode-html-languageservice. The issue is correctly labeled as a feature-request in the Backlog.

## Change summary

Renaming HTML attribute names via F2 should find all occurrences of that attribute in the document and rename them, but currently returns 'No result'.

## Best solution

Implement attribute rename support in the upstream vscode-html-languageservice package (in `services/htmlRename.ts`), then consume the updated package in the VS Code html-language-features extension. The rename should locate all uses of the same attribute name across the document and produce edits for each.
