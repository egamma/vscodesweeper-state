# Issue #105888 — Line height setting consistency

[https://github.com/microsoft/vscode/issues/105888](https://github.com/microsoft/vscode/issues/105888)

## Summary

Partially addressed: editor.lineHeight now supports multiplier mode (PR #125601), but the remaining asks — making editor.suggestLineHeight and debug.console.lineHeight support the same multiplier semantics, and decoupling the Output panel's line height from editor.lineHeight — are still valid and unimplemented on current main. Keeping open as a tracked feature request.

## Change summary

Make all line-height settings consistently support multiplier-of-font-size values instead of only pixels, and give the Output panel its own line-height setting.

## Best solution

Extend EditorLineHeight's multiplier logic (values 0–8 = multiplier, ≥8 = pixels) to editor.suggestLineHeight in src/vs/editor/common/config/editorOptions.ts and debug.console.lineHeight in src/vs/workbench/contrib/debug/browser/debug.contribution.ts. The Output panel line-height question is lower priority given the [Log] language-override workaround.
