# Issue #189924 — Git - VScode force changes the locale to en_US.UTF-8 for git commit hooks in dev-container

[https://github.com/microsoft/vscode/issues/189924](https://github.com/microsoft/vscode/issues/189924)

## Summary

Still reproduces on main: the git extension hardcodes LC_ALL/LANG to en_US.UTF-8 (line 692–693 of extensions/git/src/git.ts), which breaks git hooks in containers/systems without that locale installed. An open PR (#210506) proposes changing to C.UTF-8 but has not been merged.

## Change summary

VS Code's git extension forces en_US.UTF-8 locale on all spawned git processes, breaking commit hooks in environments where that locale is not installed.

## Best solution

Land https://github.com/microsoft/vscode/pull/210506 (or an equivalent change) switching LC_ALL/LANG from `en_US.UTF-8` to `C.UTF-8` in `extensions/git/src/git.ts` line 692–693. This preserves the English/parseable output intent while using a universally-available locale.
