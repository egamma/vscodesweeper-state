# Issue #210295 — Scrolling breaking semantic highlighting 

[https://github.com/microsoft/vscode/issues/210295](https://github.com/microsoft/vscode/issues/210295)

## Summary

Keep open: this is a maintainer-acknowledged semantic-token rendering bug with fresh evidence from September 2025, not a stale one-off report. It is not currently closeable as not-reproducible because the reporter reproduced it again and multiple commenters report similar viewport-dependent highlighting failures across languages.

## Change summary

Scrolling after semantic tokens are applied can cause semantic highlighting to disappear or change for parts of the visible editor, apparently related to viewport semantic-token requests and possibly folded ranges.

## Best solution

Investigate the editor semantic-token viewport path, especially how folded ranges split requests and how `SparseTokensStore` merges range responses; if the folding-range interaction in https://github.com/microsoft/vscode/issues/266597 is confirmed to be the same root cause, merge or explicitly link the issues rather than closing this one prematurely.
