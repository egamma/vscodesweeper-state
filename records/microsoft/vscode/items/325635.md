# Issue #325635 — `chat.tools.terminal.autoApprove` does not work for `npm exec -- ...` commands

[https://github.com/microsoft/vscode/issues/325635](https://github.com/microsoft/vscode/issues/325635)

## Summary

This is a specific instance of the PowerShell tree-sitter parser failing to extract sub-commands for commands with `--` (argument end marker), which is the same root cause tracked in the broader #324059. The reporter already cross-linked both issues. Proposing duplicate close to consolidate discussion.

## Change summary

The `chat.tools.terminal.autoApprove` setting does not work for `npm exec -- ...` commands because the PowerShell tree-sitter grammar returns no sub-commands for those command lines, blocking auto-approval evaluation.

## Best solution

Fix the PowerShell tree-sitter command parsing in `src/vs/workbench/contrib/terminalContrib/chatAgentTools/browser/treeSitterCommandParser.ts` to handle the `--` argument-end marker so that commands like `npm exec -- tsc` are correctly decomposed into sub-commands. The canonical issue https://github.com/microsoft/vscode/issues/324059 already tracks this class of parsing failures.
