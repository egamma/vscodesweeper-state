## Summary

Team-authored issue (isidorn, MEMBER) with milestone 'On Deck' — hard keep-open on both counts. The issue tracks overly aggressive Chromium accessibility detection on Windows, an upstream concern that remains relevant. A related screen-reader detection change (PR #194671) was later reverted (PR #196507, merged 2023-10-24), suggesting the problem space is still unsettled.

## Change summary

Chromium's accessibility detection on Windows fires too broadly, causing VS Code to enter screen-reader mode for users who have no assistive technology attached.

## Best solution

Continue tracking the upstream Chromium fix for Windows accessibility detection. If Chromium doesn't resolve it, the proposed approach (revert to old detection on Windows + add telemetry for the new detection and for deactivation events) should be evaluated against the current Electron version. The accessibility area team should re-assess whether the current Electron/Chromium version has improved detection fidelity.
