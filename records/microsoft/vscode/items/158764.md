# Issue #158764 — Parallel tasks with INPUT selection does not run all tasks.

[https://github.com/microsoft/vscode/issues/158764](https://github.com/microsoft/vscode/issues/158764)

## Summary

Verified duplicate of #87927, which was closed as *as-designed: parallel dependent tasks with input variables race on the quick pick, causing only one to resolve. The workaround is dependsOrder: sequence.

## Change summary

Running parallel dependent tasks that share an input variable causes only the last task to execute; earlier tasks are cancelled by the competing quick pick.

## Best solution

Close as duplicate of https://github.com/microsoft/vscode/issues/87927 — the behavior is as-designed because parallel execution races the input quick pick. The workaround is `"dependsOrder": "sequence"`, which properly shares the resolved input across tasks.
