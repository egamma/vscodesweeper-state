# Issue #135033 — Allow option to deprioritize Symbols in Quick Open

[https://github.com/microsoft/vscode/issues/135033](https://github.com/microsoft/vscode/issues/135033)

## Summary

Valid feature request to deprioritize symbols in Quick Open results. Not implemented on main — the current code in anythingQuickAccess.ts sorts file and symbol picks together purely by fuzzy score with no priority weighting. Low community traction (0 upvotes) but a team member (TylerLeonhardt) acknowledged the idea. Keeping open for the community process.

## Change summary

Add an option to deprioritize workspace symbols in Quick Open so file results appear above symbol matches.

## Best solution

Add a setting (e.g. `search.quickOpen.deprioritizeSymbols`) that applies a score penalty to symbol picks in `getAdditionalPicks()` in `src/vs/workbench/contrib/search/browser/anythingQuickAccess.ts`, similar to how snippets are deprioritized in completions. The existing `search.quickOpen.includeSymbols` setting already gates symbols on/off; a companion setting for priority would be a natural extension.
