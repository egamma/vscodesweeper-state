# Issue #192995 — Search path multiple patterns not working

[https://github.com/microsoft/vscode/issues/192995](https://github.com/microsoft/vscode/issues/192995)

## Summary

Confirmed bug still present on main: mixing glob patterns with search-path patterns (e.g. `*/Kconfig*,./virt`) in 'files to include' causes the search path to replace workspace folder queries, which narrows the search scope and drops results that would match the glob portion alone.

## Change summary

When 'files to include' combines a glob pattern with a relative-path search path (comma-separated), the glob pattern's results are lost because the search path replaces the workspace folder queries instead of unioning with them.

## Best solution

Fix `parseSearchPaths` / `commonQuery` in `src/vs/workbench/services/search/common/queryBuilder.ts` so that when include patterns contain both glob segments and search-path segments, the glob segments are applied across all workspace folders (not just the search-path-scoped folders). The ripgrep `-g !*` logic in `ripgrepTextSearchEngine.ts` (line 423) may also need adjustment so it doesn't suppress matches from the glob portion when a search path is present.
