# Issue #192995 — Search path multiple patterns not working

[https://github.com/microsoft/vscode/issues/192995](https://github.com/microsoft/vscode/issues/192995)

## Summary

Confirmed bug still present on main — the `-g !*` ripgrep argument added for workspace-folder-style includes (non-`**` patterns) excludes everything, breaking any `**`-prefixed globs in the same query. Milestone is 'On Deck', so keeping open.

## Change summary

Search 'files to include' field fails to return results when combining a glob pattern (e.g. `*/Kconfig*`) with a workspace folder path (e.g. `./virt`) separated by comma.

## Best solution

Fix the glob argument construction in `getRgArgs()` so that the `-g !*` exclusion does not suppress `doubleStarIncludes` patterns when both kinds are present; the `doubleStarIncludes` block at line 434 needs to also be covered by the re-inclusion logic, or the two categories need a unified approach that doesn't break each other.
