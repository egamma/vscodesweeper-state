# Issue #225179 — Python: Improved Bracket Pair Detection for strings that contain `{{` 

[https://github.com/microsoft/vscode/issues/225179](https://github.com/microsoft/vscode/issues/225179)

## Summary

Still unfixed on main: the Python extension lacks `unbalancedBracketScopes` for `constant.character.escape`, so escaped `{{`/`}}` in f-strings are incorrectly colored by bracket pair colorization. A concrete solution exists (comment 11) but hasn't been applied.

## Change summary

Escaped brackets `{{`/`}}` inside Python f-strings are incorrectly colored by bracket pair colorization instead of being treated as escape characters.

## Best solution

Add `"unbalancedBracketScopes": ["constant.character.escape"]` to the Python grammar entry in `extensions/python/package.json`, following the same pattern already used by latex, csharp, perl, typescript-basics, and others.
