# Issue #214274 — `baseUri` of `exclude` in `findTextInFiles` and `findFiles2` is ignored

[https://github.com/microsoft/vscode/issues/214274](https://github.com/microsoft/vscode/issues/214274)

## Summary

Team-authored issue — must stay open for maintainer judgment. The bug appears partially addressed: `findFiles2` and `findTextInFiles2` now use `globsToISearchPatternBuilder` which preserves the `baseUri`, but the legacy `findTextInFiles` still discards it (line ~880, extracting only `options.exclude.pattern`). The query builder does consume the `uri` field from `ISearchPatternBuilder`. Whether the legacy API still needs fixing or the issue is resolved by the newer APIs should be confirmed by the area owner.

## Change summary

The `baseUri` of a `RelativePattern` passed as `exclude` in `findTextInFiles` and `findFiles2` is silently dropped, making the exclusion pattern workspace-root-relative instead of relative to the specified base.

## Best solution

Verify whether the fix in `globsToISearchPatternBuilder` (used by `findFiles2` and `findTextInFiles2`) fully addresses the issue for the new APIs. If the legacy `findTextInFiles` also needs fixing, update its exclude-pattern extraction (around line 880 of `extHostWorkspace.ts`) to use `parseSearchExcludeInclude` and pass the folder through. Close the issue once the area owner confirms the new APIs work correctly.
