# Issue #189760 — Git commit (amend) context menu

[https://github.com/microsoft/vscode/issues/189760](https://github.com/microsoft/vscode/issues/189760)

## Summary

UX polish request to reorder the Git commit button dropdown so amend commands sit lower. The maintainer (lszomoru) intentionally kept it open to gather community feedback; the current menu structure already separates amend into its own group (group 3) below basic commit commands (group 1), which partially addresses the concern. Only 2 upvotes — not enough traction to promote, but the maintainer's explicit keep-open decision should be respected.

## Change summary

Move the "Commit (Amend)" entries lower in the SCM commit button dropdown menu so less-common and potentially destructive amend operations don't sit near frequently used commit actions.

## Best solution

The `git.commit` menu in `extensions/git/package.json` already groups amend commands in `3_amend` (after `1_commit` and `2_commit_noverify`). If the team decides further reordering is warranted (e.g., pushing amend below signoff), it's a small change to the group numbering in that file. The `git.rememberPostCommitCommand` setting already mitigates the accidental-click scenario for most users. Leave open for the community process; close if no further traction.
