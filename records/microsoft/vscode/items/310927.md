# Issue #310927 — Restart to Update not working on Mac

[https://github.com/microsoft/vscode/issues/310927](https://github.com/microsoft/vscode/issues/310927)

## Summary

This is an upstream bug in Squirrel.Mac / Electron's autoUpdater: after 100 ShipIt log files accumulate, the update helper fails silently. The root cause is tracked at https://github.com/electron/electron/issues/52284 and https://github.com/Squirrel/Squirrel.Mac/issues/322. VS Code cannot fix this in its own codebase — it needs to be resolved in Squirrel.Mac or Electron. Keeping open as an upstream tracking issue.

## Change summary

On macOS, clicking 'Update Now' closes VS Code but fails to apply the update or relaunch, requiring a manual reinstall.

## Best solution

The fix must land in Squirrel.Mac (https://github.com/Squirrel/Squirrel.Mac/issues/322) or Electron (https://github.com/electron/electron/issues/52284) — either by fixing the log-rotation crash at 100 files or by switching to a single log file. VS Code should track this upstream and pick up the fix once Electron ships it. A possible VS Code-side mitigation could be to periodically clean stale ShipIt logs, but the proper fix belongs upstream. Users can work around it now by deleting `~/Library/Caches/com.microsoft.VSCode.ShipIt/ShipIt_std*.log.*`.
