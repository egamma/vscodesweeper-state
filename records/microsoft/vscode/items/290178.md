# Issue #290178 — prompts-diagnostics-provider reports valid relative links as "File not found" in copilot-instructions.md

[https://github.com/microsoft/vscode/issues/290178](https://github.com/microsoft/vscode/issues/290178)

## Summary

Still reproduces on main: `resolveFilePath` in promptFileParser.ts does not strip URL fragments from markdown link targets before resolving, causing `fileService.exists()` to fail on valid paths. The plain-relative-path case (no anchor) may also fail in certain workspace configurations, but the anchor-fragment case is clearly traceable in source.

## Change summary

The prompts-diagnostics-provider reports false-positive 'File not found' warnings for valid relative markdown links in copilot-instructions files, especially when the link includes an anchor fragment.

## Best solution

Strip the URL fragment (everything after `#`) from the path in `resolveFilePath` (or in the validator before calling `fileService.exists`) so that anchor links like `../README.md#section` resolve to the correct file path. The fix belongs in `src/vs/workbench/contrib/chat/common/promptSyntax/promptFileParser.ts` around line 526.
