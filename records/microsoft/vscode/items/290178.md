# Issue #290178 — prompts-diagnostics-provider reports valid relative links as "File not found" in copilot-instructions.md

[https://github.com/microsoft/vscode/issues/290178](https://github.com/microsoft/vscode/issues/290178)

## Summary

The anchor-fragment sub-bug (comment 7) is confirmed on current main: `resolveFilePath` in `promptFileParser.ts` passes the raw link content (including `#fragment`) to `joinPath`, creating an invalid file path. The general relative-path false-positive (original report, comments 3/5) remains unclear — the maintainer couldn't reproduce it and no obvious defect is visible in source for plain relative paths.

## Change summary

The prompts-diagnostics-provider reports valid relative markdown links as 'File not found' in copilot-instructions.md, especially when the link contains a URL fragment (`#section`).

## Best solution

Strip the URL fragment (everything after `#`) from the link content before resolving the file path in `resolveFilePath` (or at the call site in `validateBody`). The fix belongs in `src/vs/workbench/contrib/chat/common/promptSyntax/promptFileParser.ts` or `promptValidator.ts`. A test should verify that `[text](../file.md#section)` does not produce a false-positive diagnostic.
