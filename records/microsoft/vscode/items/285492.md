# Issue #285492 — Feature Request: Add workspace-relative variant of ${resultsFiles} for search commands

[https://github.com/microsoft/vscode/issues/285492](https://github.com/microsoft/vscode/issues/285492)

## Summary

Valid feature request for cascading/iterative search support — the core ask is a workspace-relative path variant for search result files. Already promoted to Backlog with 27 upvotes. The `${resultsFiles}` variable itself comes from the third-party 'Find and Transform' extension, not VS Code core, but the underlying need (scoping a follow-up search to files from a prior search) is a reasonable core improvement. Needs the `search` area label for routing.

## Change summary

Add a workspace-relative variant of the search results file list (e.g. `${resultsFilesRelative}`) so follow-up searches can use result files in `filesToInclude`.

## Best solution

The search team should evaluate whether to add a built-in command variable (like `${resultsFilesRelative}`) that expands search result file URIs to workspace-relative paths, or alternatively provide a built-in 'search in found files' action. Issue #20530 ('Search: target particular set of files') overlaps broadly but is a superset — this issue is more specific and actionable. An extension (FreHu's vscode-fresh-file-explorer) already provides this workflow, which validates the use case but also shows it can be done via extension API today.
