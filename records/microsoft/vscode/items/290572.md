# Issue #290572 — KeyboardInterrupt seen in terminal when launching python debugger

[https://github.com/microsoft/vscode/issues/290572](https://github.com/microsoft/vscode/issues/290572)

## Summary

Team-authored issue (MEMBER); must keep open. The bug is confirmed by multiple users — a KeyboardInterrupt fires when the Python debugger sends commands to the terminal before the prompt input model is ready. The root cause appears to span VS Code terminal readiness signaling and the Python debugger/environments extensions.

## Change summary

Launching the Python debugger triggers a spurious KeyboardInterrupt (^C) in the terminal on the first attempt, because commands are sent before the terminal prompt is ready.

## Best solution

Coordinate between the terminal's prompt-readiness signal and extension `executeCommand` calls so that debug launch commands are not sent before the shell is ready to accept input. The fix likely involves both the VS Code terminal infrastructure (ensuring readiness is reliably signaled) and the Python debugger extension (waiting for that signal). Comment 2 points to the pattern already used in vscode-python-environments as a mitigation.
