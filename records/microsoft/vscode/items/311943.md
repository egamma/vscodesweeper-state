# Issue #311943 — Bug: Generated commit message is deleted when pressing the 'Up' arrow key

[https://github.com/microsoft/vscode/issues/311943](https://github.com/microsoft/vscode/issues/311943)

## Summary

Verified duplicate of #229970 which tracks the exact same problem — pressing Up arrow at the start of the SCM input navigates commit history, replacing the generated message. The canonical issue is open with 'scm' and 'under-discussion' labels in the Backlog milestone.

## Change summary

Generated commit message disappears when pressing Up arrow at the start of the SCM input box, because the keybinding navigates commit history.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/229970 which already tracks this exact behavior and is under discussion for how to resolve it (the current behavior is that Up arrow at position 1,1 navigates history; the message is saved and retrievable with Down, but users don't expect this).
