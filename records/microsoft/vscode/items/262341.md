# Issue #262341 — Update tool confirmation actions

[https://github.com/microsoft/vscode/issues/262341](https://github.com/microsoft/vscode/issues/262341)

## Summary

Team-authored feature request to redesign the tool confirmation UX for better discoverability of session/workspace-level approvals and a less disruptive 'No' path. Keeping open — authored by a team member and actively relevant to the agent experience.

## Change summary

Redesign tool-call confirmation actions so that 'Always allow for session' and 'suggest a different approach' options are more discoverable than the current dropdown hidden behind 'Continue'.

## Best solution

Redesign the tool confirmation widget actions (likely in `src/vs/workbench/contrib/chat/browser/`) to surface session-level approval as a primary action rather than a dropdown, and replace 'No' / cancel with a 'No, suggest a different approach' flow that focuses the chat input instead of cancelling. This is a UX design + implementation task owned by the chat-tools area.
