# Issue #322571 — [Error] unhandlederror-Invalid typed array length: -57045

[https://github.com/microsoft/vscode/issues/322571](https://github.com/microsoft/vscode/issues/322571)

## Summary

Still reproduces on main: line 344 of documentSemanticTokens.ts allocates a Uint32Array with an unchecked length that can go negative when semantic token edits have overly large deleteCount values. An open PR (#322575) addresses this but has not yet merged.

## Change summary

A RangeError crashes when applying semantic token delta edits whose deleteCount values produce a negative array length.

## Best solution

Validate that `srcData.length + deltaLength >= 0` (and that each edit's `deleteCount` doesn't exceed available data) before allocating the Uint32Array; bail out gracefully as is already done for invalid `edit.start`. The open PR https://github.com/microsoft/vscode/pull/322575 targets this fix — review and merge it.
