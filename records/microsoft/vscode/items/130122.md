# Issue #130122 — Line numbers are not restored if active window when exiting Zen mode is not a file.

[https://github.com/microsoft/vscode/issues/130122](https://github.com/microsoft/vscode/issues/130122)

## Summary

Still reproduces on main: the Zen mode exit path only restores line numbers for `visibleTextEditorControls`, so editors not visible at the moment of exit (because a non-text editor like Settings is focused) never get their line numbers restored.

## Change summary

Line numbers are not restored for file editors that are not visible when exiting Zen mode (e.g., when Settings is the active tab).

## Best solution

When exiting Zen mode, iterate over all open text editor controls (not just visible ones) to restore line numbers, or listen to editor visibility changes after exit to lazily restore them. The fix belongs in `src/vs/workbench/browser/layout.ts` around the `setLineNumbers()` helper at line 1448.
