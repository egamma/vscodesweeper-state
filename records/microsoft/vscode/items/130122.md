# Issue #130122 — Line numbers are not restored if active window when exiting Zen mode is not a file.

[https://github.com/microsoft/vscode/issues/130122](https://github.com/microsoft/vscode/issues/130122)

## Summary

Still reproduces on main: zen mode exit only restores line numbers for currently visible text editors via `visibleTextEditorControls`, missing editors hidden behind other tabs (like Settings). The known limitation acknowledged by isidorn in 2021 remains unresolved.

## Change summary

Line numbers are not restored for non-visible file editors when exiting Zen mode with a non-file editor (e.g., Settings) active.

## Best solution

On zen mode exit, additionally listen for `onDidVisibleEditorsChange` (temporarily) to restore line numbers for editors that become visible after the transition — similar to the listener already used during zen mode active state. The fix belongs in `src/vs/workbench/browser/layout.ts` around line 1589 where `setLineNumbers()` is called on exit.
