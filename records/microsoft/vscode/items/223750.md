## Summary

Team-authored issue on the On Deck milestone — must stay open. The bug is in Chromium/Electron's OSCrypt backend detection (`key_storage_linux.cc`), which selects GNOME_LIBSECRET even when libsecret is not installed. No VS Code-side fix exists; this is correctly labeled upstream/electron.

Electron/Chromium's OSCrypt backend detection incorrectly selects GNOME_LIBSECRET on fresh Debian installs where libsecret is not available, causing credential storage failures or hangs.
