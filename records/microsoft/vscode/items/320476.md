# Issue #320476 — mermaid block in Agents Window chat renders as `[Object object]`

[https://github.com/microsoft/vscode/issues/320476](https://github.com/microsoft/vscode/issues/320476)

## Summary

Team-authored bug with an 'On Deck' milestone — keeping open. The bug is still present on main: the Agents Window chat lacks the chat output renderer infrastructure that the main workbench chat uses to render mermaid code blocks via webview, causing the mermaid block's rendered object to be coerced to `[Object object]`.

## Change summary

Mermaid fenced code blocks in Agents Window chat render as `[Object object]` instead of the diagram or a plain code block fallback.

## Best solution

Ensure the Agents Window either registers the `ChatOutputRendererService` (so the mermaid extension's code block renderer works) or falls back gracefully to a plain code block when no output renderer is available. The likely files are the sessions workbench bootstrap (`src/vs/sessions/browser/workbench.ts`) for service registration and the chat markdown content part for the fallback path.
