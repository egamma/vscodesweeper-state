[https://github.com/microsoft/vscode/issues/260090](https://github.com/microsoft/vscode/issues/260090)

## Summary

Still present on main. The listener leak warning fires when a chat session has many requests (331+), because each ChatResponseModel registers listeners on the ChatModel's onDidChange emitter during deserialization. Roblourens has already identified the problematic pattern and assigned it — keeping open for the fix.

## Change summary

Chat sessions with many requests trigger a potential listener LEAK warning (331 listeners) due to each ChatResponseModel registering onDidChange listeners on the ChatModel emitter.

## Best solution

Refactor the listener registration in ChatModel (around line 2502-2515 in chatModel.ts) so that the autorun/observableFromEvent pattern on `onDidChange` does not accumulate one listener per request. Roblourens suggested in the comments that this could be done without registering all the listeners individually — e.g., by aggregating via a single forwarding listener or using the observable pattern that already tracks the last request.
