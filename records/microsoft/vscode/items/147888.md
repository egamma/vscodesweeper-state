# Issue #147888 — Prevent the blue warning strip at the top that keeps popping up

[https://github.com/microsoft/vscode/issues/147888](https://github.com/microsoft/vscode/issues/147888)

## Summary

Valid feature request: the unicode-highlight banner reappears on every model change because `_bannerClosed` resets to `false` (line 59 of unicodeHighlighter.ts). No persistent dismissal or configurable limit setting exists on current main. Keeping open as it has a milestone (Backlog) and is a reasonable UX improvement.

## Change summary

Request to persistently dismiss (or add a 'don't show again' option for) the unicode highlighting banner that reappears every time the user switches back to a file with many non-basic ASCII characters.

## Best solution

Add a setting (e.g. `editor.unicodeHighlight.bannerEnabled` or a higher configurable limit via `editor.unicodeHighlight.limit`) so users who deliberately enabled non-basic-ASCII highlighting can suppress the banner without disabling the highlighting itself. The fix would live in `src/vs/editor/contrib/unicodeHighlighter/browser/unicodeHighlighter.ts`.
