# Issue #143667 — Layout picker isn't accessible

[https://github.com/microsoft/vscode/issues/143667](https://github.com/microsoft/vscode/issues/143667)

## Summary

Still valid on main: the quick pick lacks support for radio-group semantics, so the layout picker's mutually exclusive items (Side Bar Position, Panel Alignment) aren't properly announced. PR #275954 attempted a fix but was closed without merging.

## Change summary

Layout picker's radio-like items (Side Bar Position, Panel Alignment) don't announce their checked/selected state to screen readers because the quick pick only supports 'option' or 'checkbox' roles, not 'radio'/'radiogroup'.

## Best solution

Add radio-group role support to the quick pick list (`src/vs/platform/quickinput/browser/quickInputList.ts`) so separator-grouped items can declare a `radio` role and announce checked state. The layout picker in `src/vs/workbench/browser/actions/layoutActions.ts` would then use this API for its mutually exclusive groups.
