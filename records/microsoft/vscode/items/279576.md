# Issue #279576 — bracket pair colorization incorrectly renders `>` in C# markdown code fence

[https://github.com/microsoft/vscode/issues/279576](https://github.com/microsoft/vscode/issues/279576)

## Summary

Valid bug with milestone 'On Deck' — keeping open. The `unbalancedBracketScopes` in markdown-basics doesn't suppress bracket-pair colorization of `>` inside embedded C# code fences, causing false red coloring. An open PR (#282153) exists but hasn't merged.

## Change summary

Bracket pair colorization incorrectly colors `>` as an unmatched bracket inside C# code fences in markdown files.

## Best solution

Merge or iterate on the open PR https://github.com/microsoft/vscode/pull/282153, which adds the relevant embedded-language scopes to `unbalancedBracketScopes` in `extensions/markdown-basics/package.json` so that operator `>` tokens inside fenced code blocks are excluded from bracket-pair colorization.
