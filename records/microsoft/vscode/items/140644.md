# Issue #140644 — Git - Add Default Commit Message 

[https://github.com/microsoft/vscode/issues/140644](https://github.com/microsoft/vscode/issues/140644)

## Summary

Valid feature request already promoted to Backlog by the community process. VS Code supports git's `commit.template` config but does not auto-generate default commit messages from changed files (e.g. "Update filename.ext") the way GitHub Desktop does. No implementation found on current main; the gap remains.

## Change summary

Auto-generate a default commit message (e.g. "Update file.txt") based on changed files, similar to GitHub Desktop's behavior.

## Best solution

Implement an optional auto-generated default commit message in the git extension (extensions/git) that populates the SCM input box when there's no user-typed message and no git commit.template — likely in `extensions/git/src/repository.ts` around the `commitTemplate` logic and `extensions/git/src/commands.ts` commit flow. A setting like `git.defaultCommitMessage` could control the format. Alternatively, this could be an `*extension-candidate` since extensions can already set `sourceControl.commitTemplate`, but given the Backlog status, it's been accepted for potential core work.
