# Issue #260556 — Configure tools... action should respect the tools I configured in my markdown file

[https://github.com/microsoft/vscode/issues/260556](https://github.com/microsoft/vscode/issues/260556)

## Summary

Still present on main: the 'Configure Tools...' code lens always rewrites the tools list even when no changes are made, reordering entries because the round-trip through ToolAndToolSetEnablementMap doesn't preserve the original file order.

## Change summary

The 'Configure Tools...' code lens rewrites and reorders the tools list in prompt markdown files even when the user confirms without making changes.

## Best solution

In `promptToolsCodeLensProvider.ts#updateTools`, compare the picker result to the original `selectedTools` before calling `rewriteTools`. If unchanged, skip the rewrite. Additionally, `rewriteTools` in `promptFileRewriter.ts` should preserve the original order of tools that remain selected.
