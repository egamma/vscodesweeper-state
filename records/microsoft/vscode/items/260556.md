# Issue #260556 — Configure tools... action should respect the tools I configured in my markdown file

[https://github.com/microsoft/vscode/issues/260556](https://github.com/microsoft/vscode/issues/260556)

## Summary

Partially fixed by commit 35c9204ff3c (range bug causing tools key to be eaten), but the 'Configure Tools...' action still rewrites and reorders the tools list even when the user clicks OK without making changes, because there's no no-op guard in the code path.

## Change summary

The 'Configure Tools...' code action modifies the prompt markdown file (reordering tools) even when the user confirms without making any changes.

## Best solution

Add a no-op guard in `promptToolsCodeLensProvider.ts` (around line 101) that compares the selected tools before/after the picker and skips calling `rewriteTools` when they are identical. Additionally, `rewriteTools` in `promptFileRewriter.ts` should preserve the original order of already-listed tools rather than re-sorting them.
