# Issue #260556 — Configure tools... action should respect the tools I configured in my markdown file

[https://github.com/microsoft/vscode/issues/260556](https://github.com/microsoft/vscode/issues/260556)

## Summary

Fixed on main by commit 35c9204ff3c which corrected the range passed to the rewriter from the full attribute range (including the 'tools:' key) to just the value range, preventing the file from being corrupted when using Configure Tools.

## Change summary

The 'Configure Tools...' code action in prompt markdown files rewrote the file incorrectly — removing tools, reordering them, and destroying the 'tools:' key — because it used the full attribute range instead of the value-only range.

## Best solution

Keep the shipped fix (commit 35c9204ff3c). The rewriter now targets `toolsAttr.value.range` and the picker pre-selects existing tools and returns early on cancel. No further change needed.
