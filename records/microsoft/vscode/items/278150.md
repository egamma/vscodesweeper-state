# Issue #278150 — Code server will detect the prerequisitions incorrectly when "\$ORIGIN" is in "$LD_LIBRARY_PATH"

[https://github.com/microsoft/vscode/issues/278150](https://github.com/microsoft/vscode/issues/278150)

## Summary

Still reproduces on main: the CLI prerequisite checker in `cli/src/util/prereqs.rs` spawns `ldconfig -p` inheriting the full environment. When `$ORIGIN` is in `LD_LIBRARY_PATH`, the spawned subprocess can malfunction, causing the glibcxx check to fail even though the library is present on the system.

## Change summary

VS Code Server prerequisite detection incorrectly fails when `$ORIGIN` (a legitimate dynamic linker token) is present in `$LD_LIBRARY_PATH`.

## Best solution

Sanitize `LD_LIBRARY_PATH` (strip `$ORIGIN` and other linker-special tokens like `$LIB`/`$PLATFORM`) before spawning the `ldconfig -p` subprocess in `check_glibcxx_version()`, or use `.env_remove("LD_LIBRARY_PATH")` on the child command since ldconfig doesn't need it.
