# Issue #158433 — Option to prevent quickpicks from dismissing other quickpicks

[https://github.com/microsoft/vscode/issues/158433](https://github.com/microsoft/vscode/issues/158433)

## Summary

Team-authored feature request exploring whether quickpicks should support a non-dismissing mode. Hard keep-open: author is a MEMBER.

## Change summary

Allow quickpicks to be shown without dismissing other currently-visible quickpicks, to avoid parallel workflows interrupting each other.

## Best solution

If revisited, add an option (e.g. `preventDismiss` or a stacking/queue mode) to the quickpick API so that a second quickpick doesn't auto-dismiss an existing one. The immediate launch-config scenario was worked around with a queue, but the general problem remains for extensions triggering concurrent inputs.
