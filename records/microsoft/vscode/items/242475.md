# Issue #242475 — Merge Editor does not recognize ActiveEditorWriteableInSession setting until reopened

[https://github.com/microsoft/vscode/issues/242475](https://github.com/microsoft/vscode/issues/242475)

## Summary

Valid feature request: the merge editor's result pane doesn't dynamically react to the 'Toggle Active Editor Read-only in Session' command because it only applies editor options at open time. No fix or related work found on current main. Keeping open as a niche backlog item.

## Change summary

The merge editor result pane ignores dynamic changes to the session-readonly toggle until the editor is reopened.

## Best solution

The merge editor's result view (in `src/vs/workbench/contrib/mergeEditor/browser/view/mergeEditor.ts`, `applyOptions`) should listen for changes to the session-readonly state from `IFilesConfigurationService` and dynamically update the result editor's `readOnly` option. This is a small enhancement localized to the merge editor view.
