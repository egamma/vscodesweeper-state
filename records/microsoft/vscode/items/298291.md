# Issue #298291 — Worktree created by Copilot doesn't show up in the list of worktrees in the side panel

[https://github.com/microsoft/vscode/issues/298291](https://github.com/microsoft/vscode/issues/298291)

## Summary

Team-authored issue (MEMBER) requesting that Copilot-created worktrees appear in the default multi-repo SCM view rather than being hidden under the single-repo Repository explorer. The worktree detection machinery exists in the git extension but the default multi-repo Changes view does not surface agent-created worktrees. Keeping open per hard keep-open rule and because a real UX gap remains.

## Change summary

Worktrees created by Copilot background agent sessions (prefixed `copilot-worktree-`) do not appear in the default multi-repo SCM side panel, requiring users to switch to single-repo mode and navigate deep into the Repository explorer to find them.

## Best solution

Surface agent-created worktrees in the default multi-repo SCM Changes view so users can review and commit changes without switching to single-repo mode. The worktree detection logic in `extensions/git/src/model.ts` (`checkForWorktrees`) already discovers worktrees; the gap is in the default multi-repo view's filtering/presentation layer. The SCM area owner should decide whether all worktrees (including agent-created ones) appear by default or via a new setting.
