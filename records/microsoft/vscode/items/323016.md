# Issue #323016 — Agent copilot terminal snapshots can be the only remaining execution record, and misreport edited commands

[https://github.com/microsoft/vscode/issues/323016](https://github.com/microsoft/vscode/issues/323016)

## Summary

Confirmed on current main: the chat transcript display chain prioritizes `forDisplay` over `userEdited`, so user edits to proposed terminal commands are invisible in the post-execution chat record despite being correctly executed. The bug is in the display priority order, not in execution.

## Change summary

When a user edits an agent-proposed terminal command before approving it, the chat transcript shows the original proposed command rather than the edited command that actually executed.

## Best solution

Fix the display priority order in `chatTerminalToolProgressPart.ts`, `chatModel.ts`, `chatWindowNotifier.ts`, and `agentSessionApprovalModel.ts` so that `userEdited` takes precedence over `forDisplay` (since `forDisplay` captures the original command's display form, not the user's edit). The execution path at `runInTerminalTool.ts:1895` already correctly uses `userEdited ?? toolEdited ?? original`; the display paths should mirror that priority.
