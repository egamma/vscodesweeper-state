# Issue #263504 — Restored integrated terminal sessions not using workspace profile

[https://github.com/microsoft/vscode/issues/263504](https://github.com/microsoft/vscode/issues/263504)

## Summary

Plausible bug where restored terminal sessions after a full VS Code restart fall back to the global default profile instead of the workspace-specific profile. The revival path in ptyService should preserve the serialized shellLaunchConfig (including executable and args), but the reporter's ps output shows bash launched with `-l` (the macOS default) rather than the workspace profile's `--rcfile` arg, suggesting either the revival didn't fire or the fallback new-process path didn't resolve the workspace profile. No fix commit found on main. The issue carries `confirmation-pending` and needs a maintainer to reproduce and validate.
