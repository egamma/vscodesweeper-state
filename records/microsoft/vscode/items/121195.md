# Issue #121195 — iPad editor renders double cursor

[https://github.com/microsoft/vscode/issues/121195](https://github.com/microsoft/vscode/issues/121195)

## Summary

Still reproduces on main: the textarea fallback path uses 1px width on Safari/iOS (line 111 of textAreaEditContext.ts sets `canUseZeroSizeTextarea = browser.isFirefox` only), causing a visible native cursor alongside the editor's painted cursor. Author is a team member — keeping open.

## Change summary

iPad Safari renders a double cursor because the editor's hidden textarea has 1px width, making its native caret visible.

## Best solution

Extend `canUseZeroSizeTextarea` to include Safari (`browser.isFirefox || browser.isSafari`) in `src/vs/editor/browser/controller/editContext/textArea/textAreaEditContext.ts`, after verifying IME (Chinese input) still works on iPad Safari with a zero-size textarea. Alternatively, add `-webkit-user-select: none` to `.monaco-editor .inputarea` in textAreaEditContext.css for the non-IME state.
