# Issue #275642 — make it easier to fix global mcp config

[https://github.com/microsoft/vscode/issues/275642](https://github.com/microsoft/vscode/issues/275642)

## Summary

Still open on main: when user mcp.json has a syntax error, adding a server throws a raw error without a notification offering to navigate to the malformed file. The related issue #278143 was closed as a duplicate of this one, confirming this is the canonical tracker.

## Change summary

When the user's global mcp.json has a JSON syntax error, attempting to add an MCP server fails without a helpful notification or option to open the broken file.

## Best solution

Catch the parse error in the MCP install flow (callers of `addMcpServers` in `mcpWorkbenchManagementService.ts` / `mcpManagementService.ts`) and surface a notification via `INotificationService.prompt` with an action button that opens the malformed `mcp.json` in the editor, similar to how `mcpServer.ts` already uses notification prompts for server-start failures.
