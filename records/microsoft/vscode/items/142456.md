# Issue #142456 — Inconsistent keyboard shortcuts between Windows and Linux

[https://github.com/microsoft/vscode/issues/142456](https://github.com/microsoft/vscode/issues/142456)

## Summary

Active feature request with 40 upvotes asking for a way to use Windows-style keybindings on Linux (or switch between platform layouts). No implementation exists on main; the community workaround is a third-party extension. Already correctly labeled `under-discussion`; should remain open for the community process.

## Change summary

Allow users to select or switch between platform keybinding layouts (e.g. use Windows defaults on Linux) without manually rebinding each shortcut.

## Best solution

Either ship a built-in keybinding-profile switcher (similar to how keymap extensions work but for platform defaults), or provide official keymap extensions for cross-platform defaults. The existing third-party extension ('default-keys-windows') validates demand but isn't a sustainable long-term answer given sync/profile interactions. The keybinding service in `src/vs/workbench/services/keybinding/` already separates platform mappers (`macLinuxKeyboardMapper`, `windowsKeyboardMapper`) — a profile mechanism could expose this choice to users.
