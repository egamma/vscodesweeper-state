# Issue #156305 — [Bug] onDidAttemptReadOnlyEdit is not triggered by commands that edit

[https://github.com/microsoft/vscode/issues/156305](https://github.com/microsoft/vscode/issues/156305)

## Summary

Still reproduces on main: commands like Tab have `precondition: EditorContextKeys.writable` which blocks them at the keybinding layer before reaching `_executeCursorEdit`, so `onDidAttemptReadOnlyEdit` never fires. The existing `under-discussion` label is appropriate as this is a design question about how read-only edit attempts should surface.

## Change summary

In read-only editors, pressing Tab (or other edit commands gated by `EditorContextKeys.writable`) does not trigger `onDidAttemptReadOnlyEdit`, so no read-only message is shown.

## Best solution

Either remove the `EditorContextKeys.writable` precondition from Tab/Outdent commands and let `_executeCursorEdit` handle the read-only guard (firing the event), or add explicit logic in the command precondition failure path to fire `onDidAttemptReadOnlyEdit`. The fix would touch `src/vs/editor/browser/coreCommands.ts` (preconditions) or potentially the keybinding dispatch layer.
