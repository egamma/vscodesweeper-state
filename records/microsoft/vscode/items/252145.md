# Issue #252145 — SCM - Quick Diff Decorations for Staged Changes Disappear When Adding Lines Before or After Staged Code

[https://github.com/microsoft/vscode/issues/252145](https://github.com/microsoft/vscode/issues/252145)

## Summary

Still reproduces on current main. The overlap-filtering logic in `quickDiffDecorator.ts` (line 150) uses `intersectsOrTouches` to suppress secondary (staged) decorations when they overlap primary (HEAD) changes — but when a user adds lines adjacent to a staged chunk, the primary diff expands to cover the staged region, incorrectly hiding the staged decorations. The issue also carries the 'On Deck' milestone, so it must stay open.

## Change summary

Quick diff gutter decorations for staged changes disappear when unstaged lines are added immediately before or after a staged chunk.

## Best solution

Refine the overlap logic in `quickDiffDecorator.ts` around line 150: instead of blanket-suppressing secondary decorations that merely touch a primary change, compute the non-overlapping sub-ranges of the secondary change and render decorations for those. The model-level dedup in `quickDiffModel.ts` (lines 292–317) already does content-aware matching — the decorator's `intersectsOrTouches` guard is the coarser, buggy layer.
