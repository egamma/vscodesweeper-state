# Issue #252371 — Copy and paste not working

[https://github.com/microsoft/vscode/issues/252371](https://github.com/microsoft/vscode/issues/252371)

## Summary

Active bug with team engagement. A workaround was merged (commit 884364539d1, Dec 2025) for the EditContext/Electron copy failure, but commenters still report intermittent issues — the root cause (Electron not firing clipboard events from execCommand) is not fully resolved. Already labeled `important` with significant upvotes; keeping open.

## Change summary

Copy/paste keyboard shortcuts intermittently stop working in the editor, especially with the EditContext API enabled, due to Electron not always firing clipboard events from execCommand('copy').

## Best solution

The workaround in commit 884364539d1 (falling back to electron's clipboard API when the copy event doesn't fire) addresses some cases, but the underlying Electron/Chromium bug with EditContext still manifests intermittently. The team should continue investigating the root cause in `src/vs/editor/browser/controller/editContext/native/nativeEditContext.ts` and `src/vs/editor/contrib/clipboard/browser/clipboard.ts` — potentially moving away from execCommand('copy') entirely when EditContext is active, or filing upstream with Electron/Chromium.
