# Issue #326174 — VS Code 1.129 crashes on launch with error code 135 on NVIDIA GPU + Wayland

[https://github.com/microsoft/vscode/issues/326174](https://github.com/microsoft/vscode/issues/326174)

## Summary

Regression crash (code 135) on Linux in VS Code 1.129, confirmed by multiple users across Wayland and X11, with and without NVIDIA GPUs. Likely caused by the Electron 42.5.0 bump (Chromium 148) landed on 2026-06-29. This is an upstream Electron/Chromium renderer crash — the root cause lives in the GPU process or compositor, not in VS Code's own code. Keeping open for the team to investigate the Electron/Chromium side.

## Change summary

VS Code 1.129 crashes immediately on launch with renderer crash code 135 on Linux, a regression from 1.128.

## Best solution

Investigate whether the Electron 42.5.0 bump (commit 4b6f5e55bb8, Chromium 148) introduced the renderer crash on Linux. File or find the upstream Electron/Chromium bug. In the interim, consider shipping a workaround (e.g. defaulting to `--ozone-platform=x11` on affected configurations, or a Chromium flag). Issue https://github.com/microsoft/vscode/issues/326177 and https://github.com/microsoft/vscode/issues/326113 may be the same root cause and should be investigated together.
