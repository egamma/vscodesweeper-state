# Issue #240296 — Resuming after OS suspend can cause sticky scroll to not work correctly until hovered

[https://github.com/microsoft/vscode/issues/240296](https://github.com/microsoft/vscode/issues/240296)

## Summary

Still reproduces on main: the sticky scroll overlay's WebGL addon lacks an `onContextLoss` handler, unlike the main terminal which disposes and recovers from GPU context loss after OS suspend.

## Change summary

After OS suspend/resume, the terminal sticky scroll overlay renders incorrectly (blank/corrupted) until hovered because its WebGL context is lost without recovery.

## Best solution

Add an `onContextLoss` handler to the sticky scroll overlay's WebGL addon (in `_refreshGpuAcceleration` in `terminalStickyScrollOverlay.ts`) that disposes and re-creates the WebGL renderer, mirroring the pattern at line 869 of `xtermTerminal.ts`.
