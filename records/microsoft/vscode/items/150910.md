# Issue #150910 — Debug Glyph overlaid with a block cursor doesn't render correctly.

[https://github.com/microsoft/vscode/issues/150910](https://github.com/microsoft/vscode/issues/150910)

## Summary

Still reproducible on main: the debug stack-frame inline decoration uses a codicon character via `before.content`, and the block cursor rendering does not handle non-standard font characters in inline decorations — the codicon glyph displays as garbled when the cursor overlaps it.

## Change summary

Block cursor overlapping the debug execution-position inline decoration (codicon arrow) renders a garbled character instead of the proper glyph.

## Best solution

The cursor rendering in `src/vs/editor/browser/viewParts/viewCursors/` needs to account for inline decorations that use non-editor fonts (codicons). Alternatively, the debug decoration could avoid relying on `before.content` in a way that conflicts with block cursor rendering. The fix likely touches both the cursor view part and possibly the decoration/view-line rendering interaction.
