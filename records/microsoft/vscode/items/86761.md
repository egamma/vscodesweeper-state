# Issue #86761 — Allow to configure symbol visibility in pickers that may show symbols

[https://github.com/microsoft/vscode/issues/86761](https://github.com/microsoft/vscode/issues/86761)

## Summary

Still a valid, unimplemented feature request with strong community support (54 upvotes). No symbol-kind filtering configuration exists in the quick-open pickers on current main; the Outline view has per-kind toggles but the pickers do not.

## Change summary

Allow users to configure which symbol kinds (variables, constants, etc.) are visible in the Go to Symbol and workspace symbol quick-open pickers, similar to the Outline view's per-kind visibility toggles.

## Best solution

Add per-symbol-kind visibility settings (or a filter mechanism) to the symbol quick-access providers in `src/vs/workbench/contrib/search/browser/symbolsQuickAccess.ts` and the editor-scoped goto-symbol provider, mirroring the Outline view's `outline.show*` settings. A unified approach across all three symbol-showing pickers (editor-scoped, workspace-scoped, and file+symbol when `search.quickOpen.includeSymbols` is on) would be ideal, as bpasero noted in the 2020 discussion.

## Vision fit

- **Fit:** aligned
- **Reason:** Consistent, uniform UI and user-configurable experiences are a core value; this extends an existing pattern (Outline filtering) to related surfaces without adding heavyweight new UI.
- **Evidence:** Value: "Simple, uniform UI and UI interactions." The Outline view already offers per-kind filtering; extending this to pickers unifies the experience.
