# Issue #195800 — VS Code Crashes from attempts to sign in

[https://github.com/microsoft/vscode/issues/195800](https://github.com/microsoft/vscode/issues/195800)

## Summary

Still reproduces on current builds (confirmed by multiple users through 2025). The root cause is an Electron/Chromium backend-detection bug tracked by the more specific issue https://github.com/microsoft/vscode/issues/223750, but this issue also captures the VS Code side — graceful degradation instead of hanging. Both remain valid.

## Change summary

VS Code freezes/hangs on Linux (Debian/Ubuntu) when attempting to sign in, because Electron selects GNOME_LIBSECRET as the secret storage backend even when libsecret is not available.

## Best solution

Two-pronged: (1) the upstream Electron bug (https://github.com/microsoft/vscode/issues/223750) needs Chromium's `key_storage_linux.cc` to stop selecting GNOME_LIBSECRET when libsecret is absent; (2) VS Code should detect the stall or missing backend and fall back gracefully (e.g. to basic text encryption with a user notification) rather than hanging indefinitely. The graceful-fallback work belongs in `src/vs/platform/encryption/electron-main/encryptionMainService.ts` or the app startup path.
