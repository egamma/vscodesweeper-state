# Issue #282433 — Integrated terminal on macOS freezes permanently after running a program that segfaults — Crashpad fatal error kills PTY host

[https://github.com/microsoft/vscode/issues/282433](https://github.com/microsoft/vscode/issues/282433)

## Summary

Root cause is Crashpad/Electron's Mach exception port inheritance on macOS — child process crashes in the PTY host trigger a fatal Crashpad error that kills the PTY host. A team member investigated and requested a CPU profile that was never provided, but a second reporter confirmed --disable-crash-reporter resolves it. Keeping open as an upstream issue.

## Change summary

Running a segfaulting native binary in the integrated terminal on macOS permanently freezes all terminals because Crashpad's Mach exception port handling kills the PTY host process.

## Best solution

The fix likely belongs in Electron or the utility-process layer: the PTY host's child processes should not inherit the Crashpad Mach exception port so that child-process crashes don't propagate a fatal Crashpad error to the PTY host. The workaround --disable-crash-reporter confirms the causal chain. A targeted Electron-side fix (or disclaiming/resetting exception ports for the PTY host utility process) would resolve this without disabling crash reporting entirely.
