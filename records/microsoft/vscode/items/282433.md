# Issue #282433 — Integrated terminal on macOS freezes permanently after running a program that segfaults — Crashpad fatal error kills PTY host

[https://github.com/microsoft/vscode/issues/282433](https://github.com/microsoft/vscode/issues/282433)

## Summary

Confirmed upstream Crashpad issue: Mach exception port inheritance from the Electron main process to the PTY host causes a fatal Crashpad error when a child process segfaults on macOS. A second reporter (comment 3) verified that `--disable-crash-reporter` resolves the freeze, confirming the root cause is in Chromium's Crashpad, not VS Code's terminal code. The `info-needed` label predates this confirmation.

## Change summary

Integrated terminal on macOS freezes permanently when a child process segfaults because Crashpad's inherited Mach exception port fatally errors and kills the PTY host.

## Best solution

Root cause is Chromium Crashpad inheriting its Mach exception port to utility/child processes. The fix belongs upstream (Electron/Chromium): either the PTY host utility process should disclaim the exception port or Crashpad should handle the `mach_port_request_notification` failure gracefully. VS Code could mitigate by launching the PTY host with crash reporter disabled or by resetting exception ports in the child, but the proper fix is upstream. The `info-needed` label can be removed since comment 3 confirmed the Crashpad link.
