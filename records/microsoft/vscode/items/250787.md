# Issue #250787 — Avoid individual indent guides in trees

[https://github.com/microsoft/vscode/issues/250787](https://github.com/microsoft/vscode/issues/250787)

## Summary

Team-authored performance/debt issue requesting optimization of indent guide rendering in tree views to avoid hundreds of individual DOM elements per row. Still open and valid on current main.

## Change summary

Tree view indent guides render hundreds of individual elements per row, causing long-task warnings and slow performance.

## Best solution

Implement a smarter rendering strategy for indent guides (e.g. a single background-image/gradient or CSS pseudo-elements) instead of individual DOM elements per tree row, in the tree widget rendering code under src/vs/base/browser/ui/tree/.
