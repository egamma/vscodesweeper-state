# Issue #250787 — Avoid individual indent guides in trees

[https://github.com/microsoft/vscode/issues/250787](https://github.com/microsoft/vscode/issues/250787)

## Summary

Team-authored performance/debt issue requesting smarter indent guide rendering in tree views to avoid hundreds of individual DOM elements per row. Hard keep-open rule applies (author is MEMBER).

## Change summary

Tree view indent guides render hundreds of individual elements per row, causing long-task warnings and poor performance.

## Best solution

Rearchitect indent guide rendering in tree rows to use a single composite element (e.g. CSS background-image gradients or a single canvas/SVG) instead of individual DOM elements per guide level, reducing DOM node count and layout cost.
