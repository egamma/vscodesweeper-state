# Issue #155597 — Allow users to choose which editor to use for editing variable binary data

[https://github.com/microsoft/vscode/issues/155597](https://github.com/microsoft/vscode/issues/155597)

## Summary

Verified duplicate: the request to let users choose a different editor for debug binary data is fully absorbed by the open canonical issue #197287 ("Investigate debug visualization extension points"), which is on the On Deck milestone with an api-proposal label and designs an extensible debug visualizer contribution point that would allow any extension to replace the hardcoded hex editor.

## Change summary

Allow users to choose an alternative extension (instead of the hardcoded Hex Editor) for viewing and editing binary variable data during debugging.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/197287 — the debug visualization extension point API (`registerDebugVisualizationProvider` + `debugVisualizers` contribution point) is already prototyped as a proposed API and would deliver exactly the extensibility this issue requests. The last commenter on this issue (the original author) already recognized #197287 as the path forward.
