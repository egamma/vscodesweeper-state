# Issue #174941 — Git - extension API gives repository remotes with aliases

[https://github.com/microsoft/vscode/issues/174941](https://github.com/microsoft/vscode/issues/174941)

## Summary

Still unfixed on main — the git extension API returns raw `insteadOf` aliases from config rather than resolved URLs. Author is a team member; keeping open.

## Change summary

The git extension API exposes unresolved URL aliases (e.g. `gh:user/private`) from `insteadOf` git config, instead of the resolved fetch/push URLs.

## Best solution

Modify `getRemotes()` in `extensions/git/src/git.ts` to resolve URLs via `git remote get-url <remote>` (for fetch) and `git remote get-url --push <remote>` (for push), so the API returns usable URLs. The closed PR https://github.com/microsoft/vscode/pull/238675 attempted this approach and could inform a fresh implementation. The downstream vscode-pull-request-github extension has already worked around this on its side (PRs #8245 and #8754), but the core API gap remains.
