# Issue #169498 — Debug output collapsing and expading

[https://github.com/microsoft/vscode/issues/169498](https://github.com/microsoft/vscode/issues/169498)

## Summary

Valid feature request with strong community support (33 upvotes, promoted to Backlog). VS Code already supports `console.group`/`console.groupEnd` collapsing (shipped in 1.43 via commit 454fad292f3), but this asks for a broader ability to collapse arbitrary debug output sections for readability — a distinct, unimplemented capability.

## Change summary

Add the ability to collapse and expand arbitrary output sections in the debug console for readability, beyond the existing `console.group` support.

## Best solution

Extend the debug console's REPL tree (`src/vs/workbench/contrib/debug/common/replModel.ts`, `replViewer.ts`) to support user-initiated or heuristic-based collapsing of output blocks, complementing the existing `console.group` and `collapseIdenticalLines` mechanisms. This is a meaningful UX improvement for verbose debug sessions and fits well in core rather than an extension.
