# Issue #307792 — workbench.editor.customLabels.patterns: support glob matching against URI authority for UNC/WSL/network paths

[https://github.com/microsoft/vscode/issues/307792](https://github.com/microsoft/vscode/issues/307792)

## Summary

Valid feature request: custom editor labels glob matching only considers URI path (line 147 of customEditorLabelService.ts), silently ignoring the authority component, making it impossible to target UNC/WSL files. No fix exists on main. Currently in Backlog Candidates with 0 upvotes and 50 days remaining in the community process.

## Change summary

Allow workbench.editor.customLabels.patterns to match against URI authority so UNC/WSL/network paths can be targeted.

## Best solution

Extend the glob matching in `src/vs/workbench/services/editor/common/customEditorLabelService.ts` (around line 147) to prepend the authority to the path (e.g. `//<authority>/<path>`) when an authority is present, enabling patterns like `//**/wsl.localhost/**` to match. This is backwards-compatible since no local file paths start with `//`. An `${authority}` template variable would also be useful.
