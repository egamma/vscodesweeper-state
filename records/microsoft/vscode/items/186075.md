The extension host failed to start after update when an installed extension package contained `"identifiers": {}`.

## Evidence

- **report:** The issue was filed against VS Code 1.79.2 on Windows with a stack in `ExtensionRunningLocationTracker.initializeRunningLocation`; the reporter later narrowed it to adding `"identifiers": {}` to an extension package.json.
- **current scanner path:** In `src/vs/platform/extensionManagement/common/extensionsScannerService.ts:721-733`, current main derives the scanned extension identifier from `manifest.publisher` and `manifest.name`, while keeping the package manifest as a separate `manifest` property.
- **description conversion:** `src/vs/platform/extensionManagement/common/extensionsScannerService.ts:1053-1067` constructs `IExtensionDescription.identifier` as `new ExtensionIdentifier(id)` before spreading the manifest; a missing or overridden singular `identifier` would still be dangerous, but the reported plural `identifiers` field is not referenced in the relevant scanner path.
- **running-location crash surface:** `src/vs/workbench/services/extensions/common/extensionHostKind.ts:109-110` and `src/vs/workbench/services/extensions/common/extensionRunningLocationTracker.ts:245-271` still depend on every extension description having a valid `identifier`, matching the reported stack shape if malformed data reaches that point.
