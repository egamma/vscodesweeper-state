# Issue #186075 — Error after update `Cannot ... of undefined (reading '_lower')` (ext host doesn't load)

[https://github.com/microsoft/vscode/issues/186075](https://github.com/microsoft/vscode/issues/186075)

## Summary

The reported crash in ExtensionRunningLocationTracker.toKey accessing `_lower` of undefined points to an extension providing a malformed identifier. The underlying spread pattern in `toExtensionDescription` (where `...extension.manifest` can overwrite `identifier`) still exists on main, but the reporter's stated reproduction (`"identifiers": {}` — plural) wouldn't trigger it. No other reports in 2+ years. Needs clarification to verify.

## Change summary

Extension host crashes with `Cannot read properties of undefined (reading '_lower')` when an extension's package.json contains certain non-standard fields.

## Best solution

If the root cause is confirmed as a malformed extension manifest overwriting `identifier` via the spread in `toExtensionDescription` (src/vs/platform/extensionManagement/common/extensionsScannerService.ts:1066), move `...extension.manifest` before the explicit properties so they always win. Alternatively, validate that extensions provide a proper `ExtensionIdentifier` before entering the running-location tracker.
