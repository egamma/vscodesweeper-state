# Issue #6829 — [css] show all variable definitions for Go To definition

[https://github.com/microsoft/vscode/issues/6829](https://github.com/microsoft/vscode/issues/6829)

## Summary

Still a valid feature request — the CSS language server currently calls `findDefinition` which returns a single Location. Supporting multiple definitions for CSS variables would require changes in the upstream `vscode-css-languageservice` package to return all matching declarations.

## Change summary

Go To Definition on CSS custom properties (variables) should show all definitions rather than jumping to just one.

## Best solution

The `vscode-css-languageservice` package's `findDefinition` should be updated to return all matching variable declarations (or a new `findDefinitions` plural API used). The server in `extensions/css-language-features/server/src/cssServer.ts` would then return multiple locations. This is an upstream change in the `vscode-css-languageservice` repo.
