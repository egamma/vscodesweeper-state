# Issue #6829 — [css] show all variable definitions for Go To definition

[https://github.com/microsoft/vscode/issues/6829](https://github.com/microsoft/vscode/issues/6829)

## Summary

Still valid feature request: CSS Go To Definition only returns a single definition for custom properties. The actual logic lives in the upstream vscode-css-languageservice package, so implementing this requires changes there first.

## Change summary

Go To Definition on CSS custom properties (variables) should show all definition sites, not just one.

## Best solution

Upstream the multi-definition support in vscode-css-languageservice's `findDefinition` to return all variable declarations across files. The VS Code CSS server already delegates to it, so once the language service supports it, it should work automatically.
