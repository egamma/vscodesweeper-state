# Issue #318896 — Add "Debug: On Errors"

[https://github.com/microsoft/vscode/issues/318896](https://github.com/microsoft/vscode/issues/318896)

## Summary

Legitimate feature request for a new `debug.onErrors` setting that checks Problems panel diagnostics before launching a debug session. Distinct from the existing `debug.onTaskErrors` (which only checks preLaunchTask exit codes). No duplicate found. Routing to the debug area for triage.

## Change summary

Add a setting to check for error-level diagnostics in the Problems panel before starting a debug session, with options to debug anyway, show errors, prompt, or abort.

## Best solution

Evaluate adding a `debug.onErrors` setting (or extending the existing `debug.onTaskErrors` mechanism) in `src/vs/workbench/contrib/debug/browser/debugTaskRunner.ts` to query the markers service (`IMarkerService`) for error-severity diagnostics before proceeding with a launch. The implementation would mirror the existing `onTaskErrors` flow but trigger on diagnostic markers rather than task exit codes. The default should preserve current behavior (debug anyway).
