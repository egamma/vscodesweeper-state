## Summary

The report lacks reproduction steps (which files, what settings are configured). Windows-1250 vs 1252 mis-detection is a known limitation of statistical encoding guessing; VS Code provides `files.candidateGuessEncodings` to constrain the guess. Need to confirm whether the reporter has `autoGuessEncoding` enabled and whether `candidateGuessEncodings` resolves their issue.

## Change summary

VS Code sometimes auto-detects Windows-1250 encoded files as Windows-1252, corrupting displayed content.

## Best solution

If the reporter has `files.autoGuessEncoding` enabled, the fix is to also set `files.candidateGuessEncodings` to `["windows1250"]` to constrain jschardet's detection. If the issue persists with that setting, a sample file would be needed to investigate whether jschardet or the detection pipeline has a bug. This may ultimately be an upstream jschardet limitation for statistically similar codepages.
