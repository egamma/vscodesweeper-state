# Issue #303402 — Remote SSH workspace paths are handled locally in Copilot Chat UI mode, causing repeated tool failures and renderer overload

[https://github.com/microsoft/vscode/issues/303402](https://github.com/microsoft/vscode/issues/303402)

## Summary

Copilot Chat's filesystem tools incorrectly resolve remote workspace paths against the local filesystem when running in UI mode over Remote SSH, causing repeated ENOENT failures and high renderer load from retries. Extension bisect confirmed the cause is Copilot Chat. Routing to the copilot area for investigation.

## Change summary

Copilot Chat tools attempt local filesystem access on remote workspace paths when running in UI/local mode, causing repeated failures and renderer overload.

## Best solution

The Copilot extension's tool implementations (likely in `extensions/copilot/src/extension/tools/`) need to detect when workspace paths are remote and either proxy requests through the remote filesystem provider (`vscode.workspace.fs`) or fail gracefully without aggressive retries. The retry/loop behavior that causes renderer overload should also be bounded.
