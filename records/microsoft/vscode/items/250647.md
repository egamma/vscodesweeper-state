# Issue #250647 — SCM - "Show Opened Editors" not change position

[https://github.com/microsoft/vscode/issues/250647](https://github.com/microsoft/vscode/issues/250647)

## Summary

Confirmed in the source: the git extension registers 'Stage Changes' in menu group `2_git` on `editor/title`, which sorts above the core 'Show Opened Editors' in group `3_open`. The issue has an 'On Deck' milestone — hard keep-open applies.

## Change summary

The editor title ellipsis menu shifts 'Show Opened Editors' down when a git-tracked file has unstaged changes, because 'Stage Changes' appears in an earlier menu group.

## Best solution

Fix the menu group ordering so git actions don't displace 'Show Opened Editors'. Either move the git `editor/title` contributions from group `2_git` to a group that sorts after `3_open` (e.g. `4_git`) in `extensions/git/package.json`, or move 'Show Opened Editors' to an earlier group in `src/vs/workbench/browser/parts/editor/editor.contribution.ts`.
