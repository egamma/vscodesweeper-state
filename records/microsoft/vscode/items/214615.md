# Issue #214615 — MSYS/Cygwin paths are not supported as terminal links

[https://github.com/microsoft/vscode/issues/214615](https://github.com/microsoft/vscode/issues/214615)

## Summary

Still unimplemented on main — no MSYS/Cygwin path translation exists in the terminal links code. The original PR was rejected for being too invasive (path.ts); a more scoped approach in the terminal link resolver would be needed. Keeping open as a valid niche feature request.

## Change summary

Support MSYS/Cygwin-style paths (e.g. `/d/Projects/foo.cpp` or `/cygdrive/d/Projects/foo.cpp`) as clickable terminal links on Windows.

## Best solution

Implement MSYS/Cygwin path detection and translation within the terminal link resolver (`src/vs/workbench/contrib/terminalContrib/links/`) rather than in the core `path.ts` module. Could translate `/X/...` or `/cygdrive/X/...` to `X:\...` at link-resolution time (when clicked or detected), keeping the change scoped to terminal links only.
