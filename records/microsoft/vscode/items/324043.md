# Issue #324043 — "explorer.sortOrder": "filesFirst" doesn't apply in "Source control" view with "View as Tree"

[https://github.com/microsoft/vscode/issues/324043](https://github.com/microsoft/vscode/issues/324043)

## Summary

The `explorer.sortOrder` setting is scoped exclusively to the File Explorer; the SCM tree view has its own independent sorting (path/name/status via `ViewSortKey`) and intentionally does not read it. This is a feature request for a "files first" sort option in the SCM tree view, not a bug in the current implementation.

## Change summary

Request for the Source Control tree view to support a "files before folders" sort order, consistent with the Explorer's `filesFirst` option.

## Best solution

Add a `filesFirst` sort option to the SCM tree view's `ViewSortKey` enum and sorting logic in `src/vs/workbench/contrib/scm/browser/scmViewPane.ts`, or introduce a dedicated `scm.sortOrder` setting. The Explorer's `explorer.sortOrder` is intentionally scoped to the Explorer, so a separate SCM-specific mechanism is the right approach.
