# Issue #227281 — Git - support for SHA256 repository

[https://github.com/microsoft/vscode/issues/227281](https://github.com/microsoft/vscode/issues/227281)

## Summary

Confirmed still broken on current main: at least 9 places in the git extension hardcode SHA-1's 40-character length, breaking SHA-256 repos. The issue carries the 'On Deck' milestone and has an open community PR (#317363), so it must stay open.

## Change summary

VS Code's built-in git extension assumes commit hashes are exactly 40 hex characters (SHA-1), causing SHA-256 repositories (64-char hashes) to show broken diffs, truncated commit IDs, and spurious branch rendering.

## Best solution

Review and land (or supersede) the open PR https://github.com/microsoft/vscode/pull/317363 which addresses the regex patterns, and audit all remaining `{40}` occurrences in `extensions/git/src/git.ts` and `extensions/git/src/blame.ts` — ideally centralizing the hash-matching pattern into a shared constant or helper to future-proof against further hash-length changes.
