# Issue #205762 — In debug mode `findfiles2` could print out why it skipped a particular file that matched the pattern

[https://github.com/microsoft/vscode/issues/205762](https://github.com/microsoft/vscode/issues/205762)

## Summary

Team-authored feature request for trace/debug logging in the file search pipeline so extension authors can understand why files are skipped (e.g. due to ignore files). Keeping open — requires explicit maintainer judgment.

## Change summary

Add trace or debug logging to the file search pipeline showing which settings (useIgnoreFiles, files.exclude, etc.) are active, so developers can diagnose why files are excluded from findFiles2 results.

## Best solution

Add trace-level logging in the file search provider that emits the resolved search configuration (which exclude/ignore settings are active and their values) when a search is initiated. The discussion converged on logging the effective settings rather than per-file skip reasons, which is pragmatic given ripgrep handles some exclusions opaquely. The search area owner should decide the verbosity level and output channel.
