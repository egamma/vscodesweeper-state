# Issue #205762 — In debug mode `findfiles2` could print out why it skipped a particular file that matched the pattern

[https://github.com/microsoft/vscode/issues/205762](https://github.com/microsoft/vscode/issues/205762)

## Summary

Valid feature request from a team member on the On Deck milestone — add trace/debug logging to findFiles2 showing why files were skipped (e.g. ignore-file settings resolution). Keep open for maintainer scheduling.

## Change summary

Add debug/trace logging to `findFiles2` that shows which settings (useIgnoreFiles, useGlobalIgnore, useParentIgnore, etc.) resolved to what values, helping extension authors understand why files are excluded.

## Best solution

Add trace-level logging to the file search path that emits which ignore/exclude settings were active and their resolved values, so extension developers debugging file-search behavior can see why files were skipped. The discussion converged on logging resolved settings rather than per-file skip reasons (which would require extra ripgrep calls).
