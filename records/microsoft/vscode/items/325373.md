# Issue #325373 — New Sidepane Layout: There is an unnecessary separator before layout actions in editor title bar

[https://github.com/microsoft/vscode/issues/325373](https://github.com/microsoft/vscode/issues/325373)

## Summary

Team-authored visual bug in the new sidepane layout: the separator before layout actions in the editor title bar is shown even when the editor-actions toolbar only contains invisible actions (e.g. a hidden 'changes' action). Keeping open for the author to resolve.

## Change summary

An unnecessary separator line appears before the layout actions in the editor title bar when using the new sidepane layout, because an invisible 'changes' action inflates the `editorActionsToolbarHasActions` flag.

## Best solution

Fix the separator visibility logic in `src/vs/workbench/browser/parts/editor/editorTabsControl.ts` (around line 348/370) so that `editorActionsToolbarHasActions` only counts actions that are actually visible/rendered — not actions that are present in the menu but visually hidden. Alternatively, fix the 'changes' contribution so it doesn't register into `MenuId.EditorTitle` when it shouldn't be visible.
