# Issue #93599 — Can't drag files from a remote vscode window to a local window

[https://github.com/microsoft/vscode/issues/93599](https://github.com/microsoft/vscode/issues/93599)

## Summary

Team-authored feature request on the 'On Deck' milestone with 40 community upvotes and active demand — multiple hard keep-open rules apply (MEMBER author, real milestone). The capability gap remains: dragging files from a remote window to a local window still doesn't work, and the earlier implementation attempt (PR #98817) was closed without merging.

## Change summary

Allow dragging files from a remote VS Code window's explorer to a local VS Code window's explorer to copy them across.

## Best solution

Implement cross-window remote-to-local file transfer on drag-and-drop, likely via IPC between windows through the electron-main process as outlined in the discussion (comment by bpasero). The earlier closed PR https://github.com/microsoft/vscode/pull/98817 explored this direction and could inform a fresh implementation. Key files likely in `src/vs/workbench/contrib/files/browser/views/explorerViewer.ts` and `src/vs/workbench/browser/dnd.ts`.
