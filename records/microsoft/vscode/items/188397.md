# Issue #188397 — VS Code not responding on launching

[https://github.com/microsoft/vscode/issues/188397](https://github.com/microsoft/vscode/issues/188397)

## Summary

The core keychain-hang scenario reported here — VS Code freezing at startup because the password store blocks on an inaccessible keyring — was addressed by defaulting to `basic` password-store (commit 1ec506bf0e9, July 2023) and subsequent migration work. The remaining open keychain/keyring startup-hang cases are tracked by https://github.com/microsoft/vscode/issues/187338 (upstream Chromium/Electron keyring detection) and https://github.com/microsoft/vscode/issues/200586 (lock-up when libsecret DB is locked). This issue is a grab-bag of reports that those canonical issues absorb; proposing duplicate close against #187338.

## Change summary

VS Code hangs indefinitely on startup on Linux when the system keyring (gnome-keyring / libsecret) is unavailable, misconfigured, or locked.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/187338, which is the canonical open tracking issue for Linux keyring detection and fallback behavior (labeled upstream/chromium). The specific lock-up-when-DB-is-locked variant is tracked separately at https://github.com/microsoft/vscode/issues/200586. The GPU/mesa-related reports in this thread are unrelated environment issues, not VS Code bugs.
