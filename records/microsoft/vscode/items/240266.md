# Issue #240266 — SCM - Binary File Diff does not show in Combined Working Diff, Combined Staged Diff or Commit Diff

[https://github.com/microsoft/vscode/issues/240266](https://github.com/microsoft/vscode/issues/240266)

## Summary

Still reproducible on main — the multi-diff editor silently drops binary files because the text-model creation error is caught and returns `undefined`, filtering them out. The issue carries the 'On Deck' milestone, which is a hard keep-open.

## Change summary

Binary files are excluded from combined working/staged diffs and commit diffs in SCM, showing 'No Changed Files' when only binary files changed.

## Best solution

In `src/vs/workbench/contrib/multiDiffEditor/browser/multiDiffEditorInput.ts` around line 201–206, instead of returning `undefined` when a binary file fails text-model creation, produce a valid `IDocumentDiffItemWithMultiDiffEditorItem` entry that renders as a binary-file placeholder (similar to how individual file diffs already handle binaries). This keeps binary files visible in combined/staged/commit diffs.
