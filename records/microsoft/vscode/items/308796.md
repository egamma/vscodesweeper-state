Valid bug confirmed in source: scmHistoryViewPane.ts line 565 uses the global `foreground` color instead of `historyItemHoverDefaultLabelForeground` for default-case badge text, making the dedicated color token ineffective. An open PR (#308802) already targets this fix.

## Change summary

SCM Graph history item badge labels use the global foreground color instead of the dedicated `scmGraph.historyItemHoverDefaultLabelForeground` token when no reference color is set, causing poor contrast in some themes.
