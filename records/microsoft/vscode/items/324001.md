# Issue #324001 — Pressing new terminal in new terminal window leads to terminal in wrong location

[https://github.com/microsoft/vscode/issues/324001](https://github.com/microsoft/vscode/issues/324001)

## Summary

Real bug, confirmed in source on current main. Author is a team member (self-assigned), so keeping open for their fix.

## Change summary

In a new terminal window opened via `newInNewWindow`, the dropdown's "New Terminal" action creates the terminal in the main panel instead of in the current auxiliary window.

## Best solution

The dropdown's "New Terminal" action at `src/vs/workbench/contrib/terminal/browser/terminalMenus.ts:801` calls `createAndFocusTerminal()` without a location. It should pass the current window's location context (similar to how the profile-based actions on line 892–896 use the `location` parameter, and how the plus button uses `CreateTerminalEditorSameGroup`). The fix is a small change to thread the location through to that action.
