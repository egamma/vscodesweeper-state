# Issue #65648 — Bidi text edit support

[https://github.com/microsoft/vscode/issues/65648](https://github.com/microsoft/vscode/issues/65648)

## Summary

Keep open: the issue is still an active Backlog feature request for practical RTL paragraph/editing support in VS Code, and the merged Monaco work is only a partial foundation. The recent discussion is still deciding whether core should provide default direction detection or expose enough API for extensions.

## Change summary

Support editing RTL paragraphs/lines in the editor, including right alignment and direction-aware cursor/selection behavior.

## Best solution

Build on the Monaco decoration-based text direction support from https://github.com/microsoft/vscode/pull/255455, then either expose it through the VS Code extension API or add a conservative built-in/default direction policy for RTL lines in the editor.
