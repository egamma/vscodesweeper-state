# Issue #238951 — Login state flaky across windows when triggered by multiple extensions

[https://github.com/microsoft/vscode/issues/238951](https://github.com/microsoft/vscode/issues/238951)

## Summary

Team-authored issue (MEMBER) discussing flaky authentication state across windows due to lazy extension activation. Active under-discussion with team members debating whether auth should be eager; no resolution reached yet.

## Change summary

Login state for multiple auth providers (GitHub, Microsoft) is inconsistent across window reloads because extensions activate lazily and don't re-request accounts at startup.

## Best solution

Resolve the under-discussion question: decide whether certain auth-consuming extensions (Cloud Changes, Azure Repos) should eagerly request sessions at startup, or whether the authentication layer should persist visible login state independently of extension activation. The fix likely touches authentication session restoration in `src/vs/workbench/services/authentication/` and possibly the lazy activation paths of the consuming extensions.
