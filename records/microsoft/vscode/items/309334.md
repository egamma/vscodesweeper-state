# Issue #309334 — VS Code 1.115 crashes with segfault on Linux VMware (SVGA3D) - WebGL software fallback deprecated

[https://github.com/microsoft/vscode/issues/309334](https://github.com/microsoft/vscode/issues/309334)

## Summary

Genuine upstream Chromium issue — the crash is caused by Chromium deprecating automatic software WebGL fallback, which breaks VMware SVGA3D virtual GPU environments. VS Code already carries the `chromium` label, and the workaround (`--disable-gpu`) confirms the root cause is in the GPU stack, not VS Code code. Keeping open as an upstream tracker.

## Change summary

VS Code 1.115+ crashes with segfault on Linux VMware guests using SVGA3D because Chromium no longer silently falls back to software WebGL.

## Best solution

Track this as an upstream Chromium/Electron issue. The team could consider detecting SVGA3D-class GPUs and automatically adding `--disable-gpu` or `--enable-unsafe-swiftshader`, but the proper fix lives in Electron's GPU blocklist or Chromium's fallback policy. Document the `--disable-gpu` workaround for VMware users in the meantime.
