# Issue #284014 — Files open in the same "floating" window as Chat

[https://github.com/microsoft/vscode/issues/284014](https://github.com/microsoft/vscode/issues/284014)

## Summary

Still present on main: the chat editor input (`chatEditorInput.ts`) does not use editor group locking, so when chat is opened in a floating (auxiliary) editor window, clicking file/function links in chat responses opens those files in the same floating window rather than the main editor window. Tyriar's suggestion to use editor group locking remains unimplemented.

## Change summary

Clicking file or function links in a floating chat window opens the file in the same floating window instead of the main editor window.

## Best solution

Implement editor group locking on the chat editor input (in `src/vs/workbench/contrib/chat/browser/widgetHosts/editor/chatEditorInput.ts`) so that non-chat editors opened from chat links are directed to the main window's editor group, similar to how the terminal handles this. The `confirmation-pending` label suggests a team member should confirm the behavior first.
