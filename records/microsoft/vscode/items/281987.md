# Issue #281987 — Git - chat should have access to git history of current workspace

[https://github.com/microsoft/vscode/issues/281987](https://github.com/microsoft/vscode/issues/281987)

## Summary

This feature request asks for Copilot chat to query and reason about git history in the workspace. VS Code has since added SCM history item attachments (users can attach commits/commit ranges to chat) and agent mode with terminal tools (which can run git commands). These partially address the request, but there's no dedicated git-history chat tool that proactively provides commit context without manual attachment. Keeping open as the gap — seamless, ask-a-question-about-git-history without manual attachment — still exists.

## Change summary

Let Copilot chat answer natural-language questions about the workspace's git history (latest commits, contributions, diffs) without requiring the user to manually attach commits.

## Best solution

The remaining gap is a dedicated git-history tool (or expanding existing SCM tools) that lets the agent autonomously query commit logs, diffs, and authorship. This could live in `src/vs/workbench/contrib/git/` as a language-model tool contribution registered alongside existing terminal tools, or as an extension-side tool in the Copilot extension. With agent mode's `run_in_terminal` tool, users can already ask the agent to run git commands — but a first-class tool would be more efficient and avoid terminal overhead. Reasonable to keep in the Backlog for the git area owner to evaluate.
