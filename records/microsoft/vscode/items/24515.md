# Issue #24515 — [folding] Folding HTML tags should hide the closing tag. 

[https://github.com/microsoft/vscode/issues/24515](https://github.com/microsoft/vscode/issues/24515)

## Summary

Long-standing feature request to hide the closing HTML tag when a range is folded. Still unimplemented on current main — the HTML language service deliberately sets the fold end line to the line before the closing tag, so it remains visible. Community interest is steady.

## Change summary

When folding an HTML tag, the closing tag line should be hidden so only the opening tag is visible.

## Best solution

The editor folding infrastructure would need to support a per-range option (or a language-service-level hint) to include the last line in the hidden region. This likely requires changes in `src/vs/editor/contrib/folding/browser/hiddenRangeModel.ts` (which currently hides `startLine+1` to `endLine`) and coordination with the HTML language service to report fold ranges that include the closing tag line. Alternatively, a new `FoldingRange` property (e.g., `collapsedText` or `includeEndLine`) in the LSP/VS Code API could let language servers opt in.
