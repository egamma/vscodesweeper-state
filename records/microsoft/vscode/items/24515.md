# Issue #24515 — [folding] Folding HTML tags should hide the closing tag. 

[https://github.com/microsoft/vscode/issues/24515](https://github.com/microsoft/vscode/issues/24515)

## Summary

Valid feature request with 35+ upvotes asking that folding HTML tags hides the closing tag line. Not yet implemented — current folding keeps the closing tag visible by design. Appropriately in Backlog with community interest above the 20-upvote threshold.

## Change summary

When folding an HTML tag, the closing tag line remains visible; users want it hidden so only the opening tag shows.

## Best solution

Implement an option (e.g. per-language or via the folding range provider API) to include the closing tag line in the hidden range when collapsing. This would require changes in the folding range sanitization logic in `src/vs/editor/contrib/folding/browser/` and possibly coordination with `vscode-html-languageservice` to signal that end lines should be consumed. The related prerequisite (language-aware folding, https://github.com/microsoft/vscode/issues/3422) has long been resolved.
