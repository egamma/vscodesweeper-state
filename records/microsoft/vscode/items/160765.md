# Issue #160765 — SCM - Shell integration ignores env overrides when opening from SCM panel?

[https://github.com/microsoft/vscode/issues/160765](https://github.com/microsoft/vscode/issues/160765)

## Summary

Still present on main. The 'Open in Integrated Terminal' command from the SCM panel passes `{ config: { cwd } }` to `createTerminal`, which bypasses the default profile resolution (including its `env` overrides). Milestone is 'On Deck', so keeping open per policy.

## Change summary

Opening a terminal from the SCM panel context menu ignores the default terminal profile's env overrides because `createTerminal` receives a partial config that prevents default-profile fallback.

## Best solution

In `src/vs/workbench/contrib/externalTerminal/browser/externalTerminal.contribution.ts` line 77, pass `cwd` as a top-level option (e.g. `createTerminal({ cwd })`) instead of inside `config`, so `createTerminal` falls through to `getDefaultProfile()` and merges the profile's env. Alternatively, resolve the default profile first and merge `cwd` into it before passing as `config`.
