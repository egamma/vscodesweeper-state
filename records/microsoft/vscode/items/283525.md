# Issue #283525 — No validation or logs on adding a private MCP registry (that fails)

[https://github.com/microsoft/vscode/issues/283525](https://github.com/microsoft/vscode/issues/283525)

## Summary

Milestoned 'On Deck' and acknowledged by a team member (sandy081) — hard keep-open applies. The trailing slash issue with private MCP registry policy enforcement is a valid security-adjacent bug where the 'registry only' access restriction fails silently.

## Change summary

A trailing slash in a policy-configured private MCP registry URL causes the 'registry only' access restriction to silently fall back to showing all servers from the default registry.

## Best solution

Ensure that `chat.mcp.gallery.serviceUrl` is normalized (trailing slash stripped) before comparing against the product gallery URL and that the 'registry only' access policy correctly blocks default registry servers when a private registry is configured but fails to load. The base `createMcpGalleryManifest` already strips trailing slashes (line 47 of `mcpGalleryManifestService.ts`), but the enforcement in `McpAccessValue.Registry` may need to fail-closed rather than silently falling back to the default registry.
