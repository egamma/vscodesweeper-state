# Issue #180629 — version `GLIBCXX_3.4.29' not found

[https://github.com/microsoft/vscode/issues/180629](https://github.com/microsoft/vscode/issues/180629)

## Summary

Still a valid open issue: the VS Code snap uses `base: core20` on current main, and core20's bundled libstdc++ only provides up to GLIBCXX_3.4.28 — newer host distros with libproxy compiled against 3.4.29+ trigger this mismatch warning. A dev branch (dev/dmitriv/build-snap-arm64) migrates to core22 but has not landed on main.

## Change summary

The VS Code snap's core20 base bundles an older libstdc++ missing GLIBCXX_3.4.29, causing warnings/failures when host libraries (e.g. libproxy) require it.

## Best solution

Land the snap base upgrade from core20 to core22 (or core24) in `resources/linux/snap/snapcraft.yaml` — core22 ships GLIBCXX_3.4.29+. Work toward this already exists on the `dev/dmitriv/build-snap-arm64` branch (commit 6e0f5d3577e). This would resolve the library version mismatch for all affected distros.
