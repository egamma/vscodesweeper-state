# Issue #256702 — Chat: Reflect multiple editor selections in current document bubble

[https://github.com/microsoft/vscode/issues/256702](https://github.com/microsoft/vscode/issues/256702)

## Summary

Still reproduces on current main: the implicit context code uses `getSelection()` (singular) and tracks only one selection, so multiple editor selections are never reflected in the chat context bubble. Active design discussion from team members on how to render multiple selections is ongoing.

## Change summary

The chat input's current-document bubble only tracks a single editor selection, ignoring additional multi-cursor selections and breaking when the first selection is removed.

## Best solution

Update `chatImplicitContext.ts` to use `getSelections()` (plural) and model multiple ranges. The UI rendering in `implicitContextAttachment.ts` needs a design decision (per comment 3's proposal of individual dismissible labels per selection). The design question from the team (comment 2) should be resolved before implementation.
