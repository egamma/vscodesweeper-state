# Issue #268111 — Add API / command for showing a specific local chat session

[https://github.com/microsoft/vscode/issues/268111](https://github.com/microsoft/vscode/issues/268111)

## Summary

Valid API feature request: extensions currently cannot create a chat session with a stable ID and later restore it. The capability gap still exists on current main — IChatViewOpenOptions has no sessionId parameter, and the chat sessions proposed API is provider-side only. A team member asked 'Still relevant?' with no reply, but the gap is real.

## Change summary

Allow extensions to create a new chat session with a unique ID via `workbench.action.chat.newChat` and later restore that same session via `workbench.action.chat.open` using that ID.

## Best solution

Extend IChatViewOpenOptions (or the proposed chatSessions API) with a session identifier so extensions can create and later reveal a specific chat session. This likely touches `src/vs/workbench/contrib/chat/browser/actions/chatActions.ts` (command args) and the chat widget/session service. The unanswered 'Still relevant?' from roblourens suggests the team should confirm whether this use case is still needed before investing.
