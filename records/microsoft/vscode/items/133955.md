# Issue #133955 — URI provided by didOpen does not conform to RFC3986

[https://github.com/microsoft/vscode/issues/133955](https://github.com/microsoft/vscode/issues/133955)

## Summary

Still reproducible on current main: the `+` character in URI authority is intentionally percent-encoded to `%2B` (commit 749b171dbfe from 2019), which violates RFC3986 §3.2.2. Team acknowledged the issue but no fix has landed; the `under-discussion` label remains appropriate as it involves backward-compatibility trade-offs.

## Change summary

URI.toString() percent-encodes the `+` character in the authority component as `%2B`, violating RFC3986 which allows sub-delimiters unencoded in reg-name.

## Best solution

The `encodeURIComponentFast` function in `src/vs/base/common/uri.ts` should treat `+` (0x2B) as an allowed sub-delimiter in the authority component rather than encoding it, but this requires careful analysis of downstream consumers (remote extensions, LSP clients) that may depend on the current encoding. A migration path or opt-in behavior may be needed.
