# Issue #163934 — Git - Sync Changes Button Should Not Pull When There Are No Incoming Changes

[https://github.com/microsoft/vscode/issues/163934](https://github.com/microsoft/vscode/issues/163934)

## Summary

Team-authored feature request that remains unimplemented on current main. The sync function in repository.ts already conditionally skips push when HEAD.ahead is 0 (line 2441), but does not apply the analogous skip for pull when HEAD.behind is 0 — the original gap persists.

## Change summary

The Git sync button always performs a `git pull` even when the extension already knows there are no incoming commits, risking unexpected auto-merges.

## Best solution

Add a `HEAD.behind === 0` guard in `_sync()` in `extensions/git/src/repository.ts` (around line 2418) to skip the pull call when there are no known incoming commits, mirroring the existing `HEAD.ahead` guard for push at line 2441.
