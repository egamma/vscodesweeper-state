# Issue #188746 — Selecting the text up to the end of the line shouldn't include test failure text

[https://github.com/microsoft/vscode/issues/188746](https://github.com/microsoft/vscode/issues/188746)

## Summary

This appears fixed on current main by the newer test failure badge UI: error messages are now rendered as content widgets instead of selectable inline decoration text. The original artifact from decoration selection no longer seems necessary to track separately.

## Change summary

Selecting to the end of a source line with a failed test should not visually include the inline test failure message.

## Best solution

Treat the current badge-based test failure presentation from commit cdd03ef9692 as the shipped fix for this selection artifact; keep any remaining decoration-selection limitations tracked only if a current repro exists.
