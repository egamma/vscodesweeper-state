# Issue #240202 — Illegal value for lineNumber

[https://github.com/microsoft/vscode/issues/240202](https://github.com/microsoft/vscode/issues/240202)

## Summary

Still reproduces on main: `ghostText.lineNumber` is passed to `textModel.getLineContent()` without validating it is within bounds. This is a race/glitch in the observable graph where ghost text data references a stale line number after the model shrinks.

## Change summary

Ghost text view throws 'Illegal value for lineNumber' when the text model is modified (lines removed) before the ghost text state updates.

## Best solution

Add a bounds check in `_state` derived (and in `computeGhostTextViewData`) to bail out (return `undefined`) when `ghostText.lineNumber > textModel.getLineCount()`. This guards against the observable glitch where stale ghost text references a deleted line.
