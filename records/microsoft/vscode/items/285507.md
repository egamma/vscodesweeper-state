# Issue #285507 — Detect adding files to git index made via the command line and auto keep

[https://github.com/microsoft/vscode/issues/285507](https://github.com/microsoft/vscode/issues/285507)

## Summary

Valid enhancement request with an assigned 'On Deck' milestone — hard keep-open. The SCM working set controller (`workingSet.ts`) currently only manages editor working sets on branch switches; it has no logic to auto-keep editors when files are staged via the CLI.

## Change summary

When files are added to the git index via the command line, the SCM working set should automatically keep/track those files, matching the behavior of the SCM panel's + button.

## Best solution

Extend `SCMWorkingSetController` in `src/vs/workbench/contrib/scm/browser/workingSet.ts` (and possibly the git extension's status detection) to listen for index changes and auto-keep editors for newly staged files, so CLI-driven `git add` parity with the SCM panel's stage action is achieved.
