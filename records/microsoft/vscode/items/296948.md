# Issue #296948 — Hovering in a debug session intermittently shows the language hover instead of the debug hover

[https://github.com/microsoft/vscode/issues/296948](https://github.com/microsoft/vscode/issues/296948)

## Summary

Real race condition in debug hover suppression. The editor's language hover can fire before `preventDefaultEditorHover()` takes effect, since the lockout is set asynchronously relative to the editor's own hover scheduling. No fix has landed on main.

## Change summary

During debug sessions, hovering a symbol intermittently shows the language hover instead of the debug hover due to a race between hover suppression and the editor's hover scheduling.

## Best solution

The fix likely needs to ensure `preventDefaultEditorHover()` synchronously cancels any in-flight editor hover computation (not just hides it), or that the editor hover controller checks a debug-session flag before starting. The relevant code is in `debugEditorContribution.ts` (`onEditorMouseMove` / `preventDefaultEditorHover`) and may also need coordination with `ContentHoverController`.
