# Issue #310436 — Restrict skills to specific agents

[https://github.com/microsoft/vscode/issues/310436](https://github.com/microsoft/vscode/issues/310436)

## Summary

Legitimate feature request for skill-to-agent binding. No existing implementation found on main — the codebase has `applyTo` patterns for file-path-based instruction filtering but no mechanism to restrict skills/tools to specific agents.

## Change summary

Request to add an `allowed_agents` (or similar) field to skill definitions so that skills can be scoped to specific agents rather than being globally available.

## Best solution

Add agent-scoping support in the prompt/skill metadata layer (likely in `src/vs/workbench/contrib/chat/common/promptSyntax/`) to allow skill definitions to declare which agents may invoke them, similar to the existing `applyTo` pattern for file paths. This is a product design decision that needs maintainer input on the right API shape.
