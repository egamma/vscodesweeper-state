# Issue #139538 — [json] support references in schema associations

[https://github.com/microsoft/vscode/issues/139538](https://github.com/microsoft/vscode/issues/139538)

## Summary

This is a confirmed feature request to support JSON pointer fragments (e.g. `#/$defs/inner_schema`) in schema association URLs. A maintainer acknowledged the limitation and pointed to the upstream vscode-json-languageservice as where the change belongs. No fix has landed on main.

## Change summary

JSON schema associations should support URL fragments (JSON pointers like `#/$defs/inner_schema`) to reference sub-schemas within a schema file.

## Best solution

Implement fragment/JSON-pointer resolution in the schema association path within `vscode-json-languageservice` (specifically around `jsonSchemaService.ts`). The VS Code JSON server passes the URI through without stripping or resolving fragments, so both layers may need attention. This is partly an upstream fix in `microsoft/vscode-json-languageservice`.
