# Issue #269731 — View shifts horizontally despite word wrap enabled, caused by copilot inline suggestion

[https://github.com/microsoft/vscode/issues/269731](https://github.com/microsoft/vscode/issues/269731)

## Summary

Confirmed bug (reproduced by hediet) where ghost text / inline decorations extend beyond the wrapped line boundary and cause unwanted horizontal scrolling. No fix found on current main.

## Change summary

Ghost text inline suggestions cause horizontal scroll shifts when word wrap is enabled and the suggestion extends past the viewport edge.

## Best solution

Ghost text and inline decorations that appear at the end of a wrapped line should not contribute to the editor's content width (scrollWidth) when word wrap is active — the scroll-width computation in `src/vs/editor/common/viewLayout/viewLayout.ts` (and whatever feeds `setMaxLineWidth`) needs to exclude ghost-text decoration widths in word-wrap mode.
