[https://github.com/microsoft/vscode/issues/139726](https://github.com/microsoft/vscode/issues/139726)

## Summary

Still unimplemented: the unicode highlight limit remains hard-coded at 1000 in current main with no user-facing setting to configure it. Valid feature request with low community traction (2 upvotes).

## Change summary

Allow users to configure the maximum number of unicode characters highlighted before VS Code gives up and shows a banner instead.

## Best solution

Expose a new setting like `editor.unicodeHighlight.maximumHighlights` that feeds into the `MAX_RESULT_LENGTH` constant in `src/vs/editor/common/services/unicodeTextModelHighlighter.ts`, and consider adding a checkbox/action in the banner to override the limit for the current file (as hediet suggested in the comments).
