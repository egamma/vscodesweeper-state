# Issue #197634 — Search: Support tree type navigation and find widget

[https://github.com/microsoft/vscode/issues/197634](https://github.com/microsoft/vscode/issues/197634)

## Summary

Still unimplemented on main — the Search tree does not set keyboardNavigationLabelProvider or enable the tree find widget. The 'On Deck' milestone also triggers a hard keep-open rule.

## Change summary

Enable tree type navigation and the tree find widget in the Search results tree so users can filter/find within large result sets.

## Best solution

Add a `keyboardNavigationLabelProvider` to the Search tree options in `src/vs/workbench/contrib/search/browser/searchView.ts` and ensure the find widget is not suppressed. The child-filtering issue noted in comment #1 (https://github.com/microsoft/vscode/issues/67397) needs resolution — children should remain visible when their parent matches the filter.
