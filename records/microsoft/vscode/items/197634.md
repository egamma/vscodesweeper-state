# Issue #197634 — Search: Support tree type navigation and find widget

[https://github.com/microsoft/vscode/issues/197634](https://github.com/microsoft/vscode/issues/197634)

## Summary

Still a valid open feature request — the Search tree on current main does not provide a `keyboardNavigationLabelProvider`, so type navigation and the tree find widget remain disabled. The issue is milestoned to On Deck and authored by a contributor (joaomoreno).

## Change summary

Enable type navigation and the tree find widget in the Search results tree so users can filter/navigate large result sets.

## Best solution

Implement a `keyboardNavigationLabelProvider` for the Search tree's `WorkbenchCompressibleAsyncDataTree` in `src/vs/workbench/contrib/search/browser/searchView.ts`, taking care that the find widget's filtering behavior preserves child matches even when their parent node doesn't match (the concern raised in comment 1 and linked issue https://github.com/microsoft/vscode/issues/67397).
