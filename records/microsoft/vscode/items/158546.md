[https://github.com/microsoft/vscode/issues/158546](https://github.com/microsoft/vscode/issues/158546)

## Summary

Valid feature request for folding/region support in search editor results. No folding provider is registered for the `search-result` language on current main, so the gap remains. The issue passed the 20-upvote threshold and is in Backlog — it should stay open.

## Change summary

Add region-based folding support to the search editor results so users can collapse per-file result blocks for call-graph or stack analysis.

## Best solution

Register a `FoldingRangeProvider` for the `search-result` language ID that emits folding ranges per file-result block in the search editor output. The implementation would likely live in `src/vs/workbench/contrib/searchEditor/browser/` and hook into the existing folding infrastructure in `src/vs/editor/contrib/folding/`.
