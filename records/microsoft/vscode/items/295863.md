# Issue #295863 — Welcome page and chat open one first open

## Summary

Team-authored bug (MEMBER). The related WIP PR #296138 was closed without merging. The current `AgentSessionsWelcomeRunnerContribution` hides the auxiliary bar on startup via `applyAuxiliaryBarHiddenOverride(true)` in layout.ts, and checks `this.editorService.activeEditor` before opening, which may partially mitigate the issue — but neither the welcome page nor the contribution explicitly closes the chat panel when both are visible, which was the agreed-upon fix direction. The welcome flow was refactored (commit 82008ac46f1 — `SessionsWelcomeService`) and there have been significant changes in this area, so the bug may or may not still reproduce. Keeping open per team-authored rule.

## Change summary

Opening VS Code with the agent sessions welcome page also opens the chat panel, creating a duplicate chat UI.

## Best solution

Verify whether the auxiliary-bar-hidden override in layout.ts (line ~3063) plus the `activeEditor` guard in the runner contribution now prevent the double-chat scenario. If not, implement the agreed-upon approach from the discussion: ensure the welcome view and chat panel are mutually exclusive — when the welcome page is visible, close/suppress the chat panel. The fix likely belongs in `src/vs/workbench/contrib/welcomeAgentSessions/browser/agentSessionsWelcome.contribution.ts` (the runner) or the welcome page itself.
