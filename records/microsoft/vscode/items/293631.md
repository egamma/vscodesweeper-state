# Issue #293631 — Add agent identifier to Copilot Chat hook parameters

[https://github.com/microsoft/vscode/issues/293631](https://github.com/microsoft/vscode/issues/293631)

## Summary

Valid feature request — the hooks system currently only passes agent identity (agent_id/agent_type) in SubagentStart and SubagentStop hook inputs, but not in PreToolUse, PostToolUse, SessionStart, UserPromptSubmit, or other hook types. The reporter's use case (per-agent tool restrictions) remains unaddressed. The per-agent hooks feature (PR #299029, comment 3) is related but orthogonal — it lets agents define their own hooks, not identify which agent triggered a shared hook.

## Change summary

Add agent identity (agentId/agentName) to the parameters passed to all Copilot Chat hook types so hook scripts can implement agent-specific policies.

## Best solution

Extend the hook input types in `extensions/copilot/src/platform/chat/common/chatHookService.ts` (and the corresponding stdin JSON contract in `hookCommandTypes.ts`) to include an `agent_id` and `agent_type` field for PreToolUse, PostToolUse, SessionStart, UserPromptSubmit, and other hook types — mirroring the pattern already used for SubagentStart/SubagentStop. The hook execution call sites would need to thread through the active agent identity.
