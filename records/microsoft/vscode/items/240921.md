# Issue #240921 — Word Wrap breaks grapheme clusters

[https://github.com/microsoft/vscode/issues/240921](https://github.com/microsoft/vscode/issues/240921)

## Summary

Keep open: a maintainer reproduced the original wrapping bug, and the reported workaround only avoids it by switching to the slower advanced wrapping strategy. There is not enough evidence for a duplicate, completed fix, or as-designed close.

## Change summary

Word wrap with the default simple wrapping strategy can split extended grapheme clusters such as family emoji sequences and wrap too early.

## Best solution

Fix or route within the editor wrapping implementation so the default/simple wrapping path does not choose break offsets inside extended grapheme clusters, or deliberately document and decide that complex grapheme-correct wrapping requires `editor.wrappingStrategy: advanced`.

## Evidence

- **maintainer reproduction:** In https://github.com/microsoft/vscode/issues/240921#issuecomment-3446714662, alexdima says the initial reported problem is reproducible and notes that it can be worked around with `"editor.wrappingStrategy": "advanced"`.
- **reporter confirmation:** In https://github.com/microsoft/vscode/issues/240921#issuecomment-3446821641, the reporter confirms that the advanced wrapping strategy solves the original issue.
- **current implementation:** src/vs/editor/common/config/editorOptions.ts defines `editor.wrappingStrategy` with default `simple`; the `advanced` option delegates wrapping point computation to the browser and is documented as working correctly in all cases.
- **wrapping path:** src/vs/editor/common/viewModel/viewModelLines.ts selects the DOM line-break computer only for `advanced`; otherwise it uses the monospace line-break computer.
- **likely gap:** src/vs/editor/common/viewModel/monospaceLineBreaksComputer.ts explicitly prevents breaking inside surrogate pairs, but the inspected simple wrapping loop does not show equivalent extended grapheme cluster handling for ZWJ emoji sequences like the reported family emoji.
