# Issue #240921 — Word Wrap breaks grapheme clusters

[https://github.com/microsoft/vscode/issues/240921](https://github.com/microsoft/vscode/issues/240921)

## Summary

Still reproduces on main: the default 'simple' wrapping strategy only respects surrogate pairs (2 code units) but not full extended grapheme clusters (ZWJ sequences). The workaround is `editor.wrappingStrategy: advanced`. A team member confirmed the bug; no fix has landed.

## Change summary

Word wrap with the default 'simple' strategy splits extended grapheme clusters (e.g. ZWJ family emoji) at code-point boundaries, breaking the visual representation.

## Best solution

The monospace line breaks computer in `src/vs/editor/common/viewModel/monospaceLineBreaksComputer.ts` needs to iterate by grapheme cluster boundaries (using `Intl.Segmenter` or the existing strings utilities) instead of only skipping surrogate pairs. Alternatively, the default wrapping strategy could be changed to 'advanced' when complex emoji are detected, though that has perf implications.
