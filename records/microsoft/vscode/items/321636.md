# Issue #321636 — [Error] unhandlederror-Unexpected reasons: unknown,applyEdits

[https://github.com/microsoft/vscode/issues/321636](https://github.com/microsoft/vscode/issues/321636)

## Summary

Still reproduces on main: the assertion at mainThreadDocuments.ts:100 fires a false-positive error when deferred content-change events merge during undo/redo, producing a multi-reason array. An open PR (#321642) fixes this but has not yet been merged.

## Change summary

Undo/redo of merged edits triggers a spurious 'Unexpected reasons: unknown,applyEdits' error in telemetry because the listener assumes exactly one detailed reason per content-change event.

## Best solution

Merge PR https://github.com/microsoft/vscode/pull/321642 (or an equivalent fix) which removes the invalid single-reason assertion in `src/vs/workbench/api/browser/mainThreadDocuments.ts` and adds optional chaining for the forwarded reason. The fix is small and localized; @hediet should confirm the forwarding-first-reason behavior is correct for the proposed API.
