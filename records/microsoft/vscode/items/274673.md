# Issue #274673 — Copilot Chat does not load/takes very long to load models on remote

[https://github.com/microsoft/vscode/issues/274673](https://github.com/microsoft/vscode/issues/274673)

## Summary

Still reproducible on main: Copilot Chat activation blocks on the experimentation service's `initialFetch` without any timeout, causing multi-minute delays on remote connections. The blocking `await` chain is unguarded in current code.

## Change summary

Copilot Chat takes 10–15 minutes to load models on remote servers because the experimentation service network fetch blocks extension activation indefinitely.

## Best solution

Add a timeout (e.g. 10–30 seconds) around the `await expService.hasTreatments()` call in the extension activation path (`extensions/copilot/src/extension/extension/vscode/extension.ts:70`) so that slow or unreachable experimentation endpoints on remote connections do not block the entire Copilot Chat experience. The experimentation data can be fetched lazily or retried in the background.
