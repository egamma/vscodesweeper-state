# Issue #299639 — let me `/fork` while a response is still coming in

[https://github.com/microsoft/vscode/issues/299639](https://github.com/microsoft/vscode/issues/299639)

## Summary

Team-authored feature request (hard keep-open). The ask is to allow `/fork` during an in-progress agent response without cancelling the streaming response — currently both 'steering' and 'stop and send' cancel the active response.

## Change summary

Allow `/fork` to branch a conversation while a chat/agent response is still streaming, without interrupting or cancelling the in-progress response.

## Best solution

The chat controller or fork-handling code needs to snapshot the conversation state at fork time and open a new chat session from that snapshot, while leaving the original session's streaming response intact. Likely involves the chat request lifecycle in `src/vs/workbench/contrib/chat/` — the fork action currently triggers a cancel-and-resend flow that should instead clone the session without disturbing the active request.
