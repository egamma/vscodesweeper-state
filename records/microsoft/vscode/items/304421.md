# Issue #304421 — Chromium Bug Report: NULL dereference in WaylandSurface::OnLeave / OnEnter

[https://github.com/microsoft/vscode/issues/304421](https://github.com/microsoft/vscode/issues/304421)

## Summary

This is a Chromium bug in the Ozone Wayland backend (WaylandSurface::OnLeave/OnEnter NULL dereference after wl_output user data is cleared). The root cause and fix live entirely in Chromium's code, not VS Code's. Keeping open as an upstream tracking issue.

## Change summary

Chromium crashes with SIGTRAP on Wayland when DPMS screen blank triggers a wl_output global_remove race, causing a NULL dereference in WaylandSurface::OnLeave.

## Best solution

The fix belongs in Chromium's `ui/ozone/platform/wayland/host/wayland_surface.cc` — adding a null check on the result of `wl_output_get_user_data()` in both OnLeave and OnEnter. The reporter has filed detailed reproduction steps and a proposed patch. VS Code should track this upstream and pick up the fix via a future Electron/Chromium update. The reporter should file at crbug.com if not already done.
