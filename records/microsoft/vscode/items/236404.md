# Issue #236404 — Devcontainers ignores terminal profile settings from settings.json when opening local integrated terminal

[https://github.com/microsoft/vscode/issues/236404](https://github.com/microsoft/vscode/issues/236404)

## Summary

Still reproduces on main: the 'Create New Integrated Terminal (Local)' command does not override profile resolution to use local OS settings — the profile service always resolves against the remote environment's platform key when connected remotely.

## Change summary

Terminal profile settings from workspace settings.json are ignored when opening a local terminal while connected to a Dev Container, because the profile service resolves profiles using the remote OS platform key.

## Best solution

The `TerminalCommandId.NewLocal` action in `terminalRemote.ts` should pass a local-context option (or explicit profile) to `createTerminal()`, and the profile resolver should support resolving against local OS settings rather than always deferring to `remoteAgentService.getEnvironment()` for the platform key.
