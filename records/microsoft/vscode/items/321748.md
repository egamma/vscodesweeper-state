# Issue #321748 — Terminal auto-approval fails for git diff -- <path> despite "git diff" allow rule

[https://github.com/microsoft/vscode/issues/321748](https://github.com/microsoft/vscode/issues/321748)

## Summary

Confirmed bug: the tree-sitter PowerShell grammar fails to parse commands containing `--` (the POSIX end-of-options / pathspec separator) as `(command)` nodes, so `extractSubCommands` returns an empty array. The `CommandLineAutoApproveAnalyzer` exits early with `isAutoApproveAllowed: false` before any auto-approve rules are checked.

## Change summary

Terminal auto-approval for `git diff -- <path>` (and other commands using `--`) fails in PowerShell because tree-sitter does not produce a `(command)` capture for the input.

## Best solution

Add a workaround in `TreeSitterCommandParser.extractSubCommands()` (similar to the existing `maskPwshFlagEquals` workaround for `--flag=value` in the same file) that masks or pre-processes bare `--` tokens so the PowerShell tree-sitter grammar can parse the command as a single `(command)` node. Alternatively, fix the upstream tree-sitter-powershell grammar to handle `--` correctly. The existing test in `commandLineAutoApproveAnalyzer.test.ts` (line 44-58, `rm -- file.txt`) would need updating once the workaround is in place.
