# Issue #253704 — Git - All lines in files with 400 lines and more are shown as modified in gutter while they are not

[https://github.com/microsoft/vscode/issues/253704](https://github.com/microsoft/vscode/issues/253704)

## Summary

Still reproduces on current main — the DP diff algorithm in `DynamicProgrammingDiffing.compute()` calls `timeout.isValid()` on every O(n²) iteration, and when the timeout fires it returns `trivialTimedOut()` (a single diff covering the entire file), which the quick diff model renders as all-lines-modified. The issue is on the 'On Deck' milestone with open fix PRs, so it must stay open.

## Change summary

Git gutter incorrectly marks every line as modified in files with ~400+ lines when the diff computation times out, even though `git status` is clean.

## Best solution

Land one of the open PRs (#314348 adding a `scm.diffDecorationsAlgorithm` setting for the faster legacy algorithm, or #303517 discarding timed-out results). The root fix is in the DP algorithm's tight timeout-check loop in `src/vs/editor/common/diff/defaultLinesDiffComputer/algorithms/dynamicProgrammingDiffing.ts` (line 30) and the quick diff model's lack of a `quitEarly` guard. Comment 17/18's clocksource analysis confirms `Date.now()` cost on non-TSC clocks exacerbates the timeout on Linux VMs.
