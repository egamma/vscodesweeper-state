# Issue #307818 — Source Control in devcontainers does not auto-refresh

[https://github.com/microsoft/vscode/issues/307818](https://github.com/microsoft/vscode/issues/307818)

## Summary

This is a known limitation of Docker bind mounts on Windows — inotify events do not propagate from the Windows host filesystem into the Linux container, so the file watcher inside the container never fires. The issue belongs in microsoft/vscode-remote-release, not in the core VS Code repo. A nearly identical issue was previously filed there (microsoft/vscode-remote-release#9399, closed as info-needed). Requesting more info and redirecting.

## Change summary

Source Control view does not auto-refresh when editing files in a devcontainer whose workspace is bind-mounted from the Windows filesystem.

## Best solution

Redirect to microsoft/vscode-remote-release where devcontainer-specific issues are tracked. The root cause is a Docker/infrastructure limitation: inotify doesn't work on bind mounts from Windows. The user can work around it by setting `files.watcherPollingInterval` or using the `CHOKIDAR_USEPOLLING=true` environment variable in their devcontainer, or by cloning into a container volume. A core VS Code change is unlikely to help here.
