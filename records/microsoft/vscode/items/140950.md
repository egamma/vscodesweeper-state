# Issue #140950 — Spacing out the buttons in action bar [Accessibility]

[https://github.com/microsoft/vscode/issues/140950](https://github.com/microsoft/vscode/issues/140950)

## Summary

The request for better-spaced inline action buttons in view list items remains partially unaddressed. The actionbar padding was bumped from 2px to 3px at some point, but the core concern — accidentally hitting revert instead of stage in tightly packed row actions — is still a valid accessibility gap. Keeping open as an acknowledged backlog item.

## Change summary

Increase spacing between inline action buttons in view list items to reduce accidental clicks on adjacent actions.

## Best solution

Increase padding/gap between action items in the compact list-item action bar (22px row height context), likely in `src/vs/base/browser/ui/actionbar/actionbar.css` or the views-specific CSS. The design team (miguelsolorio acknowledged the issue) should decide on the exact spacing that balances compactness with touch/click accuracy.
