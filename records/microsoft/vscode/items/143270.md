# Issue #143270 — Improve Performance Of Unicode Highlighting / Link Detection

[https://github.com/microsoft/vscode/issues/143270](https://github.com/microsoft/vscode/issues/143270)

## Summary

Team-authored engineering improvement idea for incremental decoration computation in unicode highlighting and link detection. Kept open per team-member authorship rule; the idea remains valid as a future optimization path.

## Change summary

Introduce an incremental text-matching mechanism so unicode highlighting and link detection avoid rescanning the entire document on every edit.

## Best solution

When the perf cost becomes material (larger files, more decorations), implement the proposed incremental matcher pattern in the editor worker layer, with delta-based decoration transport as alexdima suggested. Until then, this serves as a design note for future optimization work.
