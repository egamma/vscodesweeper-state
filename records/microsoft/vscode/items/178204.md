# Issue #178204 — vscode.dev: Support routes in PWA applications

[https://github.com/microsoft/vscode/issues/178204](https://github.com/microsoft/vscode/issues/178204)

## Summary

The PWA manifest in resources/server/manifest.json still has a static `"start_url": "/"` — no dynamic manifest support has been implemented. The issue carries a `keep` label from a maintainer and remains a valid, unaddressed feature request.

## Change summary

When adding vscode.dev/tunnel/<name> as a PWA shortcut, the app always opens at vscode.dev/ instead of the specific tunnel route because the manifest's start_url is hardcoded to "/".

## Best solution

Implement dynamic PWA manifest generation (or a service-worker-based redirect) so that the `start_url` in the manifest reflects the current route when a user installs the PWA. This likely involves server-side changes to serve a route-specific manifest.json or using the Web App Manifest's `start_url` with a query parameter approach.
