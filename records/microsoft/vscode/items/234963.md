## Summary

Still reproduces on main: the ghost text syntax highlighting depends on background tokenization state, which is non-deterministic — `tokenizeLinesAt` returns null when the start state for the target line hasn't been computed yet. An open PR (#269567) exists but hasn't merged.

## Change summary

Inline suggest preview shows inconsistent or broken syntax highlighting depending on background tokenization progress and user interactions.

## Best solution

Merge or iterate on PR https://github.com/microsoft/vscode/pull/269567 which addresses the inconsistency in `tokenizeLinesAt` for ghost text. The fix likely needs to ensure the tokenization state is available (or gracefully deferred) before rendering highlighted ghost text.
