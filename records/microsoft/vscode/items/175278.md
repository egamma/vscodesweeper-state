# Issue #175278 — Issue reporting fails silently when bugs url is invalid

[https://github.com/microsoft/vscode/issues/175278](https://github.com/microsoft/vscode/issues/175278)

## Summary

Team-authored issue (MEMBER); the underlying problem — silent failure when an extension's bugsUrl doesn't match a GitHub issues pattern — still has traces in current code. Keeping open per hard keep-open rule and because the gap persists.

## Change summary

Issue Reporter silently fails when an extension's bugs URL points to a non-standard GitHub path (e.g., Discussions) instead of a repo issues URL.

## Best solution

The newer issue reporter paths (`issueFormService.ts`, `issueReporterOverlay.ts`) have a fallback that returns the raw bugsUrl, but the legacy `baseIssueReporterService.ts` (line ~1291) returns `true` without opening it. Consolidate behavior: when the URL isn't a standard GitHub issues endpoint, either open it directly or show a clear message explaining the redirection — related open issue https://github.com/microsoft/vscode/issues/201650 tracks the same UX gap from a user perspective.
