# Issue #325352 — `terminalIsOpen` includes hidden extension terminals, making terminal keybindings unreliable

[https://github.com/microsoft/vscode/issues/325352](https://github.com/microsoft/vscode/issues/325352)

## Summary

Valid feature request: `terminalIsOpen` counts backgrounded/hidden extension terminals because it uses `this.instances.length` (which includes `_backgroundedTerminalInstances`), but no context key exposes the already-existing `foregroundInstances` getter. The gap is real and confirmed in source.

## Change summary

Request a context key that reflects only user-visible terminal instances, excluding hidden/backgrounded extension terminals.

## Best solution

Add a new context key (e.g. `terminalHasVisibleInstances` or similar) in `terminalContextKey.ts` and wire it in `_handleInstanceContextKeys()` in `terminalService.ts` using the existing `foregroundInstances` getter. This is a small, localized change — the infrastructure (`foregroundInstances`) already exists.
