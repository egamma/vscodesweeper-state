# Issue #216764 — VSCode "Open Editors" causing Logic Apps extension menu context issue

[https://github.com/microsoft/vscode/issues/216764](https://github.com/microsoft/vscode/issues/216764)

## Summary

Team-authored issue (MEMBER) — must keep open. The bug is still plausibly present on main: the Open Editors view sets ResourceContextKey only on focus-change events, so when another view (e.g. the designer) steals focus and the list loses it, resourceContext.reset() clears the keys and a subsequent right-click may not re-trigger the focus event to restore them.

## Change summary

Right-clicking a file in the Explorer loses its context menu items (e.g. 'Open Designer') when the Open Editors view is expanded and another view has taken focus.

## Best solution

The Open Editors view should ensure ResourceContextKey is set on the context menu event (or on selection), not only on focus change. The fix likely belongs in `src/vs/workbench/contrib/files/browser/views/openEditorsView.ts` around the list event handlers (lines 313-328). Alternatively, the list widget's context menu flow should scope the resource context key to the right-clicked element before the menu is built.
