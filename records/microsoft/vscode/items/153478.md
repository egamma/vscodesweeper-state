# Issue #153478 — Starting to debug should be snappier

[https://github.com/microsoft/vscode/issues/153478](https://github.com/microsoft/vscode/issues/153478)

## Summary

Valid polish request to make the debug startup sequence feel more immediate by parallelizing the visible UI transitions (toolbar, viewlet focus, dropdown, status bar). No evidence of a targeted fix for this specific perceived-lag concern on current main; the sequential visual updates likely remain.

## Change summary

Debug startup feels janky because the toolbar, viewlet focus, dropdown, and status bar color change appear in visibly staggered steps rather than all at once.

## Best solution

Audit the debug launch path in `src/vs/workbench/contrib/debug/browser/debugService.ts` and related UI components to batch or parallelize the visible state transitions (toolbar show, viewlet reveal, status bar update) so they appear in a single frame. The prelaunch-task lag is a separate contributor that has seen some improvement (e.g. https://github.com/microsoft/vscode/pull/224329), but the core UX sequencing request remains open.
