## Change summary

VS Code's built-in git extension should pass `--sparse` to `git add` (or run `git add --sparse`) when the repository uses sparse-checkout, so that new files outside the sparse definition can be staged and committed.

## Best solution

Review and land (or close with feedback) the open community PR https://github.com/microsoft/vscode/pull/233048, which adds `--sparse` support to the git add command. The fix likely touches `extensions/git/src/git.ts` where `git add` arguments are assembled. Ideally, detect sparse-checkout mode and conditionally pass `--sparse`.
