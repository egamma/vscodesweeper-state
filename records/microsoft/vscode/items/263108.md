# Issue #263108 — Surface the vscode://schemas/color-theme schema to allow for programmatic use

[https://github.com/microsoft/vscode/issues/263108](https://github.com/microsoft/vscode/issues/263108)

## Summary

Valid feature request with 20+ community upvotes, promoted to Backlog. The request for a static/exportable color-theme schema for external tooling is a real gap — the current schema is runtime-constructed and unavailable outside VS Code.

## Change summary

Provide the vscode://schemas/color-theme JSON schema in a form usable by external tools (CI/CD, linters) for theme validation without a running VS Code instance.

## Best solution

Export a baseline color-theme JSON schema (covering core-contributed colors) as a static artifact — either published to the repo, the docs site, or a dedicated npm package. Extensions that add colors would remain out of scope for the static export, but the core set would cover the majority of validation needs.
