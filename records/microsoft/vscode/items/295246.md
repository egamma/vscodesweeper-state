# Issue #295246 — Agent tries to run invalid request

[https://github.com/microsoft/vscode/issues/295246](https://github.com/microsoft/vscode/issues/295246)

## Summary

Fixed on main: image limit enforcement was added at the backend endpoint level (covering agent mode), with silent history-image filtering and a clear localized error when the current turn exceeds the model's limit.

## Change summary

Agent mode could submit more images than a model supports (e.g. 11 images to Gemini's 10-image limit), resulting in an opaque server error.

## Best solution

Keep the shipped fix — the endpoint-level `filterHistoryImages` guard in `chatEndpoint.ts` now silently drops excess history images and throws a clear error when the current turn alone exceeds the limit, covering all request paths including agent mode.
