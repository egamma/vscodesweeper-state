# Issue #297985 — Changed files list can be lost if agent uses `git stash`

[https://github.com/microsoft/vscode/issues/297985](https://github.com/microsoft/vscode/issues/297985)

## Summary

The changed files list in the chat editing session can get out of sync when an agent performs git stash/pop operations that externally modify the working tree. A related but inverse symptom is tracked by team-filed issue #315767. No fix has landed on main. The report lacks detailed reproduction steps, but the scenario is plausible given how the editing session tracks file changes independently of git state.

## Change summary

When a chat agent runs `git stash` and then `git stash pop`, the list of changed files in the editing session disappears and is not restored.

## Best solution

The chat editing session (likely in `src/vs/workbench/contrib/chat/browser/chatEditing/chatEditingSession.ts` and the response file changes service) needs to be resilient to external git operations that transiently remove and restore working tree changes. Consider reconciling the editing session's file change tracking with the actual working tree state after git operations complete, or snapshot the changed-files metadata independently of the git index. Issue https://github.com/microsoft/vscode/issues/315767 covers a related stash-sync problem in the agents window and may share a root cause — the two should be evaluated together.
