# Issue #326017 — Enabling (not using) Copilot Chat crash-loops the extension host with a V8 out-of-memory abort

[https://github.com/microsoft/vscode/issues/326017](https://github.com/microsoft/vscode/issues/326017)

## Summary

Well-documented crash-loop bug in the built-in Copilot extension's session checkpoint mechanism. The pathspec-based `git add -A` flow in `chatSessionWorkspaceFolderServiceImpl.ts` stages thousands of paths (including gitignored artifacts) into a temp index on activation, which floods the git decorations provider and drives the extension host to a V8 OOM abort (~5 GB in 12 seconds). Routing to the Copilot area for investigation.

## Change summary

Enabling the Copilot Chat extension in a large workspace causes the extension host to crash-loop with V8 OOM because the session checkpoint mechanism stages thousands of file paths (including gitignored build artifacts) via `git add -A --pathspec-from-file`, overwhelming memory.

## Best solution

The checkpoint/session diff code in `extensions/copilot/src/extension/chatSessions/vscode-node/` (multiple service impls) should guard against running on activation without an active session, and `getUncommittedFilePaths` in `extensions/copilot/src/platform/git/vscode-node/utils.ts` should be bounded or should exclude gitignored artifacts. Related OOM reports exist (https://github.com/microsoft/vscode/issues/294050, https://github.com/microsoft/vscode/issues/321441) — this issue adds a concrete root-cause trace that may explain those as well.

## Evidence

- **root cause code:** `getUncommittedFilePaths()` in `extensions/copilot/src/platform/git/vscode-node/utils.ts:98` collects all index, working-tree, and untracked changes; the result is written to a pathspec file and passed to `git add -A --pathspec-from-file=…` in `chatSessionWorkspaceFolderServiceImpl.ts:242`, `chatSessionWorktreeCheckpointServiceImpl.ts:275`, and `claudeWorkspaceFolderServiceImpl.ts:171`.
- **decorations overflow:** The staging of ~9,575 paths triggers `[Decorations] CAPPING events from decorations provider vscode.git 9575` (capping logic at `src/vs/workbench/api/common/extHostDecorations.ts:59`), indicating the git extension's decoration provider emits a massive event batch.
- **related issues:** https://github.com/microsoft/vscode/issues/294050 (Copilot Chat memory leak/OOM, open), https://github.com/microsoft/vscode/issues/321441 (error-telemetry Failed to allocate memory, copilot-chat), and https://github.com/microsoft/vscode/issues/324809 (Copilot Chat renderer OOM on idle Remote-SSH) report similar symptoms; this issue provides the most specific root-cause trace.
