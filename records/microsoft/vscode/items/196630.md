# Issue #196630 — Vscode cannot access extensions marketplace when http.proxy contains user credentials

[https://github.com/microsoft/vscode/issues/196630](https://github.com/microsoft/vscode/issues/196630)

## Summary

Fixed on main: commit c36c1716f8b strips user credentials from the proxy URL before passing it to Electron, then uses the credentials for basic auth separately. Shipped in 1.92.0.

## Change summary

VS Code cannot access the extensions marketplace when http.proxy contains user credentials (user:password@host).

## Best solution

Keep the shipped fix in commit c36c1716f8b (1.92.0); the proxy URL credentials are now stripped before passing to Electron's setProxy, and basic auth is handled separately. No further change needed.
