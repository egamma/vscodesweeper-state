# Issue #325204 — Navigating in multi-diff editor loses my place

[https://github.com/microsoft/vscode/issues/325204](https://github.com/microsoft/vscode/issues/325204)

## Summary

Team-authored bug report about multi-diff editor navigation in the Agents window changes view — keeping open for maintainer judgment.

## Change summary

Using Go to Definition in the multi-diff editor opens a new tab instead of navigating within the visible editor, and pressing Back resets scroll position.

## Best solution

The multi-diff editor's navigation/editor-opening logic should detect when the target is already visible in one of its embedded diff editors and move the cursor there instead of opening a new tab. The Back navigation should restore the previous scroll anchor. Likely involves the multi-diff editor contribution and its interaction with IEditorService open logic.
