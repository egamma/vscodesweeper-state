# Issue #298003 — Git integration not working with RemoteSSH and chroot

[https://github.com/microsoft/vscode/issues/298003](https://github.com/microsoft/vscode/issues/298003)

## Summary

Confirmed bug on current main: normalizePath("/") produces "." instead of "/", breaking git operations when the repo root is "/" (chroot). An open PR from the reporter exists but has not merged.

## Change summary

Git extension fails (discard, diff, stage) when the repository root is "/" because normalizePath strips the trailing separator from root, producing an invalid path.

## Best solution

Review and land https://github.com/microsoft/vscode/pull/317637, which guards the trailing-separator strip so that filesystem roots are preserved — consistent with the core removeTrailingPathSeparator() in src/vs/base/common/extpath.ts.
