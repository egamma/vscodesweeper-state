# Issue #281476 — Chat editors have mutating `editor.resource` property

[https://github.com/microsoft/vscode/issues/281476](https://github.com/microsoft/vscode/issues/281476)

## Summary

Still present on main: ChatEditorInput's effective identity mutates after construction — `_sessionResource` is reassigned in `resolve()` (lines 244, 270, 285) and `updateModel()` (line 313), while `matches()` uses that mutable field. The `On Deck` milestone and `important` label both trigger hard keep-open rules.

## Change summary

ChatEditorInput starts with a transient `vscode-chat-editor:` URI but its effective identity (`_sessionResource`) mutates during resolve, violating the editor part's assumption that `resource` is immutable.

## Best solution

Refactor ChatEditorInput so that the session resource is known at construction time (or the editor is constructed with the session resource directly), ensuring `matches()` and the editor part's resource-based lookups are stable. The fix likely lives in `src/vs/workbench/contrib/chat/browser/widgetHosts/editor/chatEditorInput.ts` and callers that create ChatEditorInput with a transient URI.
