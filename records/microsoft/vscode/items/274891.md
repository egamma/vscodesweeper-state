# Issue #274891 — Multiple `#selection` attachments only include the last selection

[https://github.com/microsoft/vscode/issues/274891](https://github.com/microsoft/vscode/issues/274891)

## Summary

Confirmed on current main: the `#selection` completion hardcodes `id: 'vscode.selection'` for every insertion, so `ChatRequestVariableSet.add()` deduplicates all but the first reference by ID. The bug is still present.

## Change summary

Using multiple `#selection` references in a single Copilot Chat message only sends the first selection's context because they all share the same static ID and get deduplicated.

## Best solution

Make the dynamic-variable ID unique per selection by incorporating the target URI and range into it (e.g. `vscode.selection:${uri}:${startLine}-${endLine}`). The fix is in `src/vs/workbench/contrib/chat/browser/widget/input/editor/chatInputCompletions.ts` around line 956.
