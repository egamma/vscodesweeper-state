## Summary

Team-authored issue (hard keep-open). Requests filtering out diagnostics from background agent session worktrees in the Problems view, so only the user's active workspace problems are shown.

## Change summary

Problems from background agent session worktrees appear in the main workbench Problems view, cluttering it with irrelevant diagnostics.

## Best solution

The markers/problems view should filter out diagnostics originating from worktrees that belong to background agent sessions. This likely involves changes in `src/vs/workbench/contrib/markers/` to exclude resources from session worktree paths, coordinating with the sessions service (`src/vs/sessions/`) to identify which folders are background session worktrees.
