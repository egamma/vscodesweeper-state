# Issue #251419 — Linux native context menu captures all mouse input and becomes unresponsive

[https://github.com/microsoft/vscode/issues/251419](https://github.com/microsoft/vscode/issues/251419)

## Summary

Likely an upstream Electron bug with the native system context menu pointer grab on Linux. The maintainer could not reproduce, and VS Code has since upgraded from Electron 35 to 42 — the reporter should re-test on the latest Insiders build to see if the issue persists.

## Change summary

Right-clicking the VS Code icon in the custom titlebar on Linux opens a native context menu that captures all mouse input, requiring Alt+Tab to recover.

## Best solution

If still reproducible on the latest build (Electron 42), file or link to an upstream Electron/Chromium bug for the pointer-grab leak on Linux system context menus. The VS Code-side handler in `src/vs/platform/windows/electron-main/windowImpl.ts` (lines 202–220) is minimal — the grab/ungrab lifecycle is owned by Electron/Chromium's native menu implementation.
