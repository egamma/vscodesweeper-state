# Issue #172233 — [css] missing colon causes error to be placed on next rule

[https://github.com/microsoft/vscode/issues/172233](https://github.com/microsoft/vscode/issues/172233)

## Summary

Team-authored issue (MEMBER); must remain open for maintainer judgment. The bug is a known CSS parser recovery limitation where a missing colon/semicolon causes error highlighting to bleed into the next rule.

## Change summary

When a CSS property is missing its colon, the error squiggle appears on the next rule rather than the malformed declaration.

## Best solution

Improve error recovery in the CSS parser (vscode-css-languageservice) so that diagnostics are anchored to the declaration missing the colon, not the subsequent rule. This likely lives in the upstream `vscode-css-languageservice` package rather than in-repo code.
