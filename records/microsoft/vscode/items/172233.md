# Issue #172233 — [css] missing colon causes error to be placed on next rule

[https://github.com/microsoft/vscode/issues/172233](https://github.com/microsoft/vscode/issues/172233)

## Summary

Team-authored issue describing a known CSS parser limitation where a missing colon/semicolon causes the error squiggle to appear on the next rule rather than the offending line. Kept open per policy (MEMBER author).

## Change summary

When a CSS property is missing a colon, the error diagnostic highlights the next rule instead of the line with the actual syntax error.

## Best solution

Improve error recovery in the CSS parser (vscode-css-languageservice) so that when a colon is missing in a declaration, the diagnostic range points at the offending declaration rather than bleeding into the next rule block.
