# Issue #151692 — Inconsistent rules for `.gitignore` and `files.exclude`

[https://github.com/microsoft/vscode/issues/151692](https://github.com/microsoft/vscode/issues/151692)

## Summary

Fixed across two commits: ab18a31ee83 fixed subdirectory .gitignore file discovery, and 32579e941f3 fixed the pattern-matching logic so patterns like `bin` are correctly treated as recursive (`**/bin`), matching Git's behavior. Both fixes shipped long ago.

## Change summary

`explorer.excludeGitIgnore` did not apply `.gitignore` patterns the same way Git does — e.g. `bin` only matched top-level instead of all `bin` directories recursively.

## Best solution

Keep the shipped fixes (ab18a31ee83 in 1.69.0 and 32579e941f3 in 1.72.0); no further change needed. The gitignore-to-glob conversion now correctly prepends `**/` for patterns without a path separator, matching Git semantics.
