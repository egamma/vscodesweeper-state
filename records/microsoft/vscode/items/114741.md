# Issue #114741 — Add an api for extensions to detect/observe Zen mode

[https://github.com/microsoft/vscode/issues/114741](https://github.com/microsoft/vscode/issues/114741)

## Summary

Valid API feature request: extensions still cannot observe zen mode state. The internal context key `inZenMode` exists but is not exposed through the extension API. Multiple extension authors have expressed need for this over 4+ years.

## Change summary

Expose an extension API to detect and observe whether VS Code is currently in Zen mode.

## Best solution

Expose `inZenMode` as a readable context key or add a property to the `window.state` API (or a new layout-related namespace) so extensions can query and observe zen mode. The internal plumbing (`InEditorZenModeContext` in `src/vs/workbench/common/contextkeys.ts`) already exists — the gap is purely the public API surface.
