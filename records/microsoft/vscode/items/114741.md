# Issue #114741 — Add an api for extensions to detect/observe Zen mode

[https://github.com/microsoft/vscode/issues/114741](https://github.com/microsoft/vscode/issues/114741)

## Summary

Valid, unimplemented feature request: no extension API exists to read or observe Zen mode state. The internal context key `inZenMode` exists but is not exposed to extensions. Keeping open as it has steady community interest and relates to but is distinct from #180056.

## Change summary

Add an extension API for detecting and observing whether Zen mode is active.

## Best solution

Expose the `inZenMode` state via the extension API — likely by adding a property to `vscode.window.state` (WindowState) or a new observable, and firing `onDidChangeWindowState` (or a new event) when it toggles. The implementation would wire up the existing `InEditorZenModeContext` context key to the API surface in `src/vs/workbench/api/browser/mainThreadWindow.ts` and `src/vs/workbench/api/common/extHostWindow.ts`.
