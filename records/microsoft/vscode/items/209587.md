## Summary

Still reproduces on main: the autoExpandElements loop in repl.ts calls isCollapsed on a ReplGroup element that may have been removed from the tree model between updateChildren and iteration, throwing 'Tree element not found'.

## Change summary

Debug REPL throws 'TreeError: Tree element not found' when iterating ReplGroup elements that were removed from the tree during a concurrent refresh.

## Best solution
