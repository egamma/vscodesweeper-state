# Issue #116989 — DND to download out of remote doesn't work

[https://github.com/microsoft/vscode/issues/116989](https://github.com/microsoft/vscode/issues/116989)

## Summary

Team-authored issue; the limitation is still present on main. A recent fix attempt (PR #320685) was merged then reverted (commit 26d36469b04, 2026-06-29). The underlying Chrome/browser limitation on downloadURL with non-file: schemes remains, and the `upstream` label is appropriate.

## Change summary

Dragging a file from the explorer to the native desktop doesn't work when connected to a remote (SSH, Codespaces) because the browser's downloadURL data transfer only supports file: URIs.

## Best solution

The reverted PR #320685 attempted a proxy-based approach to serve remote files over a local HTTP URL for the download transfer. The revert suggests that approach had issues. The next step is to investigate why that approach was reverted and either fix it or find an alternative (e.g., using a temporary local file or a different download mechanism). The core constraint is the browser's downloadURL API (Chrome-only, file: only) — a workaround at the Electron/main-process level is likely needed for the desktop case.
