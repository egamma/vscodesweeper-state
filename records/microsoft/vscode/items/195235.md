# Issue #195235 — diff editor can't display the deleted whitespace

[https://github.com/microsoft/vscode/issues/195235](https://github.com/microsoft/vscode/issues/195235)

## Summary

The bug appears still present on main — no whitespace-rendering fix has landed in the diff editor since filing. The inline diff view uses the modified editor's renderWhitespace setting for deleted-line rendering, which can hide whitespace-only deletions. Related issue #195267 is also open.

## Change summary

In the diff editor's inline view, deleted whitespace characters are not visually rendered, making whitespace-only deletions invisible unless the user selects the original side.

## Best solution

The diff editor's deleted-line rendering (in `renderLines.ts`) should force whitespace rendering to 'all' (or at least 'boundary') for inline decorations that represent deletions, so whitespace-only changes are visible. The fix likely lives in `src/vs/editor/browser/widget/diffEditor/components/diffEditorViewZones/renderLines.ts`.
