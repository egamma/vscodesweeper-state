# Issue #299197 — Checkpoint Restored sticks at the bottom of the chat

[https://github.com/microsoft/vscode/issues/299197](https://github.com/microsoft/vscode/issues/299197)

## Summary

Team-authored issue (MEMBER); the bug appears still present on main. The 'Checkpoint Restored' bar visibility is governed by a condition on line 969 of chatListRenderer.ts that checks whether the element is the last in the list and a checkpoint exists, but doesn't account for the bar needing to be dismissed after the user acts on it (e.g. pressing Implement).

## Change summary

After restoring a checkpoint and pressing Implement, the 'Checkpoint Restored' banner persists at the bottom of the chat instead of being dismissed.

## Best solution

The visibility condition in `chatListRenderer.ts` (line ~969) needs to also check whether the checkpoint-restore flow has been consumed/acted upon (e.g. after the user presses Implement). Likely needs a state flag on the view model that clears the checkpoint-restore indicator once the user proceeds.
