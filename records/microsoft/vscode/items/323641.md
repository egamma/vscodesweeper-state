# Issue #323641 — Agent-mode workspace directory tree is injected ahead of the cached prefix and breaks BYOK Anthropic prompt caching

[https://github.com/microsoft/vscode/issues/323641](https://github.com/microsoft/vscode/issues/323641)

## Summary

Confirmed on current main: the workspace directory tree injected by AgentMultirootWorkspaceStructure in GlobalAgentContext is rendered before the cacheBreakpoint, so any depth-≤2 directory change mid-session invalidates the entire Anthropic cached prefix. The tree walker in visualFileTree.ts uses shouldAlwaysIgnoreFile (a hardcoded exclusion list) and .copilotignore but does not consult files.exclude or search.exclude, confirming the reporter's observation.

## Change summary

BYOK Anthropic prompt cache is invalidated every time a shallow workspace directory changes because the regenerated directory tree sits ahead of the cache breakpoint in the agent prompt.

## Best solution

Two complementary fixes in the Copilot Chat extension: (1) move the workspace structure injection (AgentMultirootWorkspaceStructure) after the cacheBreakpoint in GlobalAgentContext so that tree changes don't invalidate the downstream cached prefix, and/or (2) have the visualFileTree walker also honor files.exclude / search.exclude / .copilotignore patterns for the injected tree, reducing churn from transient output directories. The tree is a hint — stability matters more than completeness here.
