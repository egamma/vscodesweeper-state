# Issue #284304 — Subagent spawned by subagent inherits root tools by default, not the spawner's tools

[https://github.com/microsoft/vscode/issues/284304](https://github.com/microsoft/vscode/issues/284304)

## Summary

Team-filed issue (roblourens); the nested subagent tool-inheritance gap is still present on main — when a named custom agent spawns an unnamed subagent, the grandchild resolves tools from the root request context rather than from its parent custom agent's declared toolset. Keeping open per team-author rule.

## Change summary

A subagent spawned by a custom-agent subagent inherits the root agent's tools instead of the spawning custom agent's tools.

## Best solution

In `runSubagentTool.ts`, when the unnamed-subagent branch executes (line ~230), propagate the parent custom agent's resolved `modeTools` into the nested invocation context rather than falling back to the root request's `userSelectedTools`. The `agentName` parameter could default to the spawning agent's own name so the grandchild runs as the same custom agent, as roblourens suggested.
