# Issue #211387 — Terminal animations break link detection in integrated terminal

[https://github.com/microsoft/vscode/issues/211387](https://github.com/microsoft/vscode/issues/211387)

## Summary

Team-authored issue (MEMBER), still unfixed on main. The bug is a known race between xterm.js viewport re-render events and link detection invalidation — no caching or debouncing fix has landed. Keep open per hard keep-open rules and because the defect persists.

## Change summary

Terminal animations that update lines cause link detection to break on those lines and nearby lines due to aggressive invalidation in xterm.js's Linkifier.

## Best solution

Implement link caching in xterm.js's Linkifier (or the VS Code adapter) so that if the chars under the cursor haven't changed, previously detected links remain valid across re-renders — as Tyriar suggested in the comments. The fix likely lives upstream in xterm.js's `src/browser/Linkifier.ts` with coordination in VS Code's `terminalLinkDetectorAdapter.ts`.
