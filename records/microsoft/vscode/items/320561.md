# Issue #320561 — Extension view: right bar doesn't scroll all the way down

[https://github.com/microsoft/vscode/issues/320561](https://github.com/microsoft/vscode/issues/320561)

## Summary

Legitimate layout bug: the extension editor's additional-details sidebar has conflicting scroll mechanics (DomScrollableElement wrapping content with native overflow-y: scroll and height: 100%), preventing full scroll to bottom. Not fixed on main.

## Change summary

The right-side metadata panel in the extension editor view cannot scroll to show all content when the window is small.

## Best solution

Fix the scroll interaction in the additional-details panel — likely remove `height: 100%` from `.additional-details-content` so the DomScrollableElement can properly detect overflow, or remove the DomScrollableElement and rely solely on native scrolling with correct height constraints. The fix lives in `src/vs/workbench/contrib/extensions/browser/media/extensionEditor.css` (lines 461-466) and `src/vs/workbench/contrib/extensions/browser/extensionEditor.ts` (lines 913-925).
