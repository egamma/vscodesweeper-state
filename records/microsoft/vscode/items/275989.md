# Issue #275989 — Cloning in VSCode sets absolute worktree path. It breaks git on WSL. How to prevent VSCode from doing this unnecessary behavior?

[https://github.com/microsoft/vscode/issues/275989](https://github.com/microsoft/vscode/issues/275989)

## Summary

Real bug, source-confirmed on main: the clone command passes an absolute `folderPath` to `git clone` (line 480 of `extensions/git/src/git.ts`), causing git to write the absolute Windows path as the worktree in `.git/config`, which breaks WSL interop. Milestone is 'On Deck' and an open PR exists — keeping open.

## Change summary

VS Code's 'Clone Git Repository' passes an absolute path as the git clone destination, causing git to record an absolute Windows worktree path in `.git/config` that is inaccessible from WSL.

## Best solution

Land https://github.com/microsoft/vscode/pull/300038 (or an equivalent fix) that passes a relative folder name instead of the absolute `folderPath` to the `git clone` command in `extensions/git/src/git.ts`, so git uses a relative worktree path. The fix is on line 480: use `folderName` instead of `folderPath` since `options.parentPath` is already the cwd for the exec call.
