# Issue #236857 — Git - Source control is not able to detect renaming a file

[https://github.com/microsoft/vscode/issues/236857](https://github.com/microsoft/vscode/issues/236857)

## Summary

Still reproduces on main: the timeline lists commits across renames via `--follow`, but clicking a pre-rename commit opens an empty diff because `resolveTimelineOpenDiffCommand` passes the current file URI to `toGitUri()` — git can't find the current path at the old commit. Issue #275318 describes the same root cause and could be closed as a duplicate of this one.

## Change summary

Clicking a timeline entry for a commit before a file rename shows an empty diff because the current file path is used instead of the path at that commit.

## Best solution

When building timeline items, capture the file's path at each commit (using `--name-status` or `--follow --name-only` output) so that `resolveTimelineOpenDiffCommand` can pass the correct historical path to `toGitUri()`. The fix touches `extensions/git/src/timelineProvider.ts` (to parse and store the old path) and `extensions/git/src/commands.ts` (to use it in the diff URI). Issue https://github.com/microsoft/vscode/issues/275318 is a duplicate of this and should be consolidated here.
