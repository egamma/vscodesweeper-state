# Issue #322796 — Changing from dark -> light with TUI CLI makes text difficult to see

[https://github.com/microsoft/vscode/issues/322796](https://github.com/microsoft/vscode/issues/322796)

## Summary

Team-authored bug (MEMBER) — terminal text from TUI apps becomes unreadable after a dark→light theme switch because already-rendered ANSI-colored content retains colors suited to the dark background. Keeping open for maintainer judgment.

## Change summary

Switching VS Code from a dark to a light theme while a TUI CLI is active in the terminal makes existing text difficult to read.

## Best solution

The terminal's `_updateTheme` in `src/vs/workbench/contrib/terminal/browser/xterm/xtermTerminal.ts` correctly updates xterm's palette on theme change, but text already written by TUI apps with explicit ANSI foreground colors (e.g. dark-on-dark that becomes dark-on-light) is not re-rendered. A potential approach is to signal xterm to redraw the viewport after a theme change, or to explore whether xterm can remap previously-rendered ANSI palette indices on theme switch. The area owner should decide whether this is tractable at the xterm/renderer level or is inherent to how TUI apps hardcode ANSI sequences.
