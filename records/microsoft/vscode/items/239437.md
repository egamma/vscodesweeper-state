# Issue #239437 — Keyring (seahorse) not detected on Cosmic-DE

[https://github.com/microsoft/vscode/issues/239437](https://github.com/microsoft/vscode/issues/239437)

## Summary

Verified duplicate: this is the same Chromium/Electron upstream issue where an unrecognized desktop environment (Cosmic-DE) causes os_crypt to fail keyring detection — already tracked by the canonical issue #187338, and a near-identical Pop!_OS report (#281008) was previously closed as a duplicate of the same.

## Change summary

Keyring (seahorse/gnome-keyring) is not detected on Cosmic-DE because Chromium's os_crypt reports the desktop environment as '(unknown)'.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/187338, which tracks the upstream Chromium/Electron limitation of not detecting keyrings when the desktop environment is unknown. The workaround of setting `"password-store": "gnome-libsecret"` in `argv.json` applies here. The fix ultimately lives upstream in Chromium's `key_storage_util_linux.cc`.
