# Issue #247564 — Refine alt text: Corrupts the image tag

[https://github.com/microsoft/vscode/issues/247564](https://github.com/microsoft/vscode/issues/247564)

## Summary

Team-filed bug still present on main — the 'Refine alt text' code action passes the user's selection as `initialRange` to inline chat, and the model replaces the entire image element instead of only the alt text within the brackets. No fix has landed since the issue was opened.

## Change summary

The 'Refine alt text' markdown code action corrupts the image tag by replacing the entire `![alt](url)` element with plain text instead of only updating the alt text inside the brackets.

## Best solution

Either constrain the `initialRange` passed to `vscode.editorChat.start` to cover only the alt-text span inside the brackets (so the model can only edit that portion), or add stronger system-prompt guardrails instructing the model to output only the replacement alt text and apply it programmatically. The fix lives in `extensions/copilot/src/extension/inlineChat/vscode-node/inlineChatCommands.ts` (the `doGenerateAltText` function) and possibly the code action provider in `inlineChatCodeActions.ts`.
