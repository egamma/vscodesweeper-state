# Issue #282790 — IME composition corrupts text during multi-cursor editing

[https://github.com/microsoft/vscode/issues/282790](https://github.com/microsoft/vscode/issues/282790)

## Summary

Still reproduces on main. The NativeEditContext computes replacePrevCharCnt/replaceNextCharCnt from the primary cursor's edit-context buffer offsets, then applies those same counts to all secondary cursors via CompositionOperation.getEdits — corrupting text when secondary cursors sit in different textual contexts.

## Change summary

IME composition with multi-cursor (Ctrl+D) replaces incorrect character ranges at secondary cursors, corrupting surrounding text.

## Best solution

The composition type path in `cursorTypeEditOperations.ts` needs per-cursor offset adjustment (or the edit context layer needs to account for multi-cursor). The fix likely touches `NativeEditContext._emitTypeEvent` (which only knows the primary selection) and/or `CompositionOperation._compositionType` (which blindly applies the same replacePrevCharCnt to every selection).
