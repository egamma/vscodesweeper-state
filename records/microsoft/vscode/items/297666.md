# Issue #297666 — Add Computer Use / Desktop Automation Support to GitHub Copilot

[https://github.com/microsoft/vscode/issues/297666](https://github.com/microsoft/vscode/issues/297666)

## Summary

Legitimate feature request with strong community support (45 upvotes, promoted to Backlog). VS Code is already investing in adjacent capabilities — integrated browser with agent-driven use, Codex harness computer-use protocol types, and screenshot context — but full desktop automation (clicking arbitrary OS UI, navigating non-browser applications) remains unimplemented. Keeping open as a valid Backlog item.

## Change summary

Add desktop automation / computer-use capabilities to Copilot agents, enabling GUI interaction, screen-based tasks, and OS-level operations from within VS Code.

## Best solution

The team is already building toward partial coverage via the integrated browser (agent-driven browser use) and agent host harnesses (Codex protocol includes `ComputerUseRequirements`). The remaining gap — full desktop automation beyond the browser — is a significant product decision about scope and security. This should stay in Backlog for the agent-experience team to evaluate whether full desktop computer use belongs in core (with sandboxing/trust implications) or should be scoped to the integrated browser surface.
