# Issue #275338 — `/savePrompt` should more clearly reveal created prompt

[https://github.com/microsoft/vscode/issues/275338](https://github.com/microsoft/vscode/issues/275338)

## Summary

Valid UX bug confirmed by a team member — the /savePrompt slash command creates a prompt file in a background tab without clearly surfacing it to the user. No evidence of a fix on main; the slash command implementation likely lives in the Copilot extension rather than core VS Code.

## Change summary

When /savePrompt creates a new prompt file, the editor tab opens in the background and the chat response lacks a clear link to it, making it easy to miss.

## Best solution

After /savePrompt creates the file, reveal the new editor tab (or open it focused) and include a clickable file link in the chat response body rather than only in the 'editing' section. The fix likely belongs in the Copilot extension's slash command handler for /savePrompt.
