# Issue #275338 — `/savePrompt` should more clearly reveal created prompt

[https://github.com/microsoft/vscode/issues/275338](https://github.com/microsoft/vscode/issues/275338)

## Summary

The UX issues with /savePrompt (background tab, no link in chat response) remain unaddressed on main. A team member agreed with the report. This is a valid polish item in the chat-prompts area.

## Change summary

The /savePrompt slash command creates the prompt file tab in the background with no clear link in the chat response, making it easy to miss.

## Best solution

When /savePrompt creates a prompt file, the resulting editor tab should be revealed (not opened in background), and the chat response should include a clickable link/reference to the created file rather than only showing it in the 'editing' section.
