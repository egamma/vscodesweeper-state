# Issue #319225 — GitHub Copilot "Restore Checkpoint" isn't persisted after switching sessions

[https://github.com/microsoft/vscode/issues/319225](https://github.com/microsoft/vscode/issues/319225)

## Summary

Real bug, still open: the checkpoint-restore state (disabled requests / timeline epoch) is serialized in both the chat model and the editing session timeline, but the save is only triggered on periodic `onWillSaveState` or shutdown — not on session switch — so switching sessions immediately after a restore checkpoint can lose the undone state. Already assigned to justschen and in Backlog; keeping open.

## Change summary

After restoring a checkpoint and switching to another session and back, the undone requests reappear because the checkpoint-restore state was not persisted before the session switch.

## Best solution

Ensure `storeState()` (or at minimum the chat model `saveState()`) is called when switching away from a session, so the checkpoint timeline's `currentEpoch` and the requests' `shouldBeRemovedOnSend` state are flushed before the model is unloaded. The fix likely touches `chatEditingServiceImpl.ts` (to trigger persistence on session deactivation) and possibly `chatServiceImpl.ts` (to flush the chat model synchronously on switch).
