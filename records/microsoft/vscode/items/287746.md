# Issue #287746 — Opening a codebase in Devcontainers on Windows leads to changes in line endings causing git diff to show changes across all source code files

[https://github.com/microsoft/vscode/issues/287746](https://github.com/microsoft/vscode/issues/287746)

## Summary

This is a git configuration issue (core.autocrlf) when using Dev Containers on Windows, not a bug in VS Code core. The Dev Containers extension (ms-vscode-remote.remote-containers) is the relevant component, and the reporter's own workarounds (setting core.autocrlf=input or adding .gitattributes) are the standard solutions. The request for VS Code to auto-detect and fix line endings belongs in the Dev Containers extension, not VS Code core.

## Change summary

Opening a Windows-cloned repo in a Dev Container causes git to show all files as changed due to CRLF/LF line ending differences.

## Best solution

Report this to the Dev Containers extension at https://github.com/microsoft/vscode-remote-release — the extension could propagate a safe `core.autocrlf=input` setting when forwarding git config into Linux containers from a Windows host, or warn users about line-ending mismatches. The workarounds the reporter listed (`.gitattributes` with `* text=auto`, or `core.autocrlf=input`) are the correct project-level fixes.

## Evidence

- **root cause:** The issue is caused by git's core.autocrlf=true default on Windows converting LF to CRLF on checkout. When mounted into a Linux Dev Container, the container's git sees CRLF files as modified. This is a git configuration issue, not a VS Code editor bug.
- **Dev Containers scope:** The Dev Containers extension (ms-vscode-remote.remote-containers) handles git config forwarding from host to container. The relevant repo for this issue is https://github.com/microsoft/vscode-remote-release, not microsoft/vscode.
- **reporter workarounds:** The reporter identified the standard fixes: setting core.autocrlf=input globally, or adding `* text=auto` to .gitattributes — both are standard git best practices for cross-platform projects.
- **related issue 147752:** https://github.com/microsoft/vscode/issues/147752 is about the diff editor's line-ending selector display, a different (UI-level) issue, not about the Dev Container line-ending propagation problem described here.
