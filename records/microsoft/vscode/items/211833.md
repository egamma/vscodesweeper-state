# Issue #211833 — Merge editor: expose commands for "accept left/right/combination" for current selection

[https://github.com/microsoft/vscode/issues/211833](https://github.com/microsoft/vscode/issues/211833)

## Summary

Partially addressed: per-conflict toggle commands exist for left/right (`merge.toggleActiveConflictInput1`/`2`), but a per-conflict "accept combination" command is still missing. The recently added `mergeEditor.acceptAllCombination` only operates on all conflicts at once. The gap remains for keyboard-driven per-conflict combination acceptance.

## Change summary

Expose a keyboard-bindable command to accept the smart combination for the currently active merge conflict.

## Best solution

Add a `merge.toggleActiveConflictCombination` command (or similar) in `src/vs/workbench/contrib/mergeEditor/browser/commands/commands.ts` that calls the combination logic on just the active conflict, mirroring how `toggleActiveConflict(1|2)` works for left/right.
