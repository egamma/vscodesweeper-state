# Issue #240709 — PATH variable in PowerShell profile missing paths

[https://github.com/microsoft/vscode/issues/240709](https://github.com/microsoft/vscode/issues/240709)

## Summary

Team-authored issue — keeping open per policy. The bug is about incomplete PATH resolution when VS Code is launched from macOS Finder/Dock (not Terminal), causing missing paths and broken tool discovery. The shell environment resolution in shellEnv.ts spawns a shell to capture env, but may not fully resolve PowerShell profile paths. Recent work in shellEnv.ts (commit f9a0a98a073, June 2025) reverted shell env resolution but did not specifically address this PowerShell-on-macOS scenario.

## Change summary

When VS Code is launched from macOS Finder/Dock, the PATH variable in the integrated terminal is missing paths that are present when launched from Terminal, breaking tool discovery for `code`, `dotnet`, and extensions.

## Best solution

Investigate whether the shell environment resolution (`src/vs/platform/shell/node/shellEnv.ts`) correctly sources the user's PowerShell profile on macOS when launched from Finder/Dock. The `getResolvedShellEnv` path spawns the system shell (likely zsh/bash) to resolve env, but if the user's default shell or PowerShell profile adds PATH entries, those may be lost. The fix likely involves ensuring the spawned shell resolution captures the full PATH, possibly by also running through `/etc/paths.d` or `path_helper`.
