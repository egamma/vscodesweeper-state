## Summary

Valid feature request to parse plain-text context-tag patterns (e.g. `#file:app.component.ts`) when pasting into the chat input and convert them into active variable chips. No duplicate found — the related closed issue #249523 only covers chip-to-chip copy/paste between chat inputs, not plain-text-to-chip conversion.

## Change summary

When pasting text containing `#file:…` or similar context-tag syntax into the chat input, recognize and convert those patterns into active context-tag chips instead of leaving them as plain text.

This would complement the existing chip-preserving paste from #249523 by covering the editor-to-chat-input scenario.
