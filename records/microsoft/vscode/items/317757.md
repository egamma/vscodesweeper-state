# Issue #317757 — Windows cursor sometimes becomes invisible in VS Code

[https://github.com/microsoft/vscode/issues/317757](https://github.com/microsoft/vscode/issues/317757)

## Summary

Verified duplicate: this is the same Windows cursor visibility issue already tracked by the open canonical issue #204103, which is an upstream Chromium/Windows compositor bug. The reporter themselves links to #217357 (which was closed as a dup of #204103) and the same Chromium bug.

## Change summary

Windows mouse cursor becomes invisible or hard to see in VS Code editable areas due to a Chromium/Windows DWM compositor conflict.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/204103 — it tracks the upstream Chromium bug with workarounds (disable direct composition, use pointer themes with shadows). A VS Code-specific cursor setting is not feasible since the cursor is rendered by the OS/compositor, not by the application.
