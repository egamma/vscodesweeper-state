# Issue #282902 — Remove "up/down arrow to navigate message history" feature from chat textarea

[https://github.com/microsoft/vscode/issues/282902](https://github.com/microsoft/vscode/issues/282902)

## Summary

Valid UX improvement request with 36 upvotes and strong community sentiment. The behavior still exists on current main — up/down arrow navigates chat history, which is unexpected for a multi-line text editor. The related PR #263943 was closed without merging. Already labeled under-discussion, which is appropriate while the team considers the right default.

## Change summary

Change or remove up/down arrow key history navigation in the chat input so users don't accidentally lose their typed prompt.

## Best solution

Change the default keybinding for chat history navigation from plain up/down arrow to a modified shortcut (e.g. Alt+Up/Down or Option+Up/Down), matching user expectations for a multi-line editor and aligning with competitors. The change lives in `src/vs/workbench/contrib/chat/browser/widget/input/chatInputPart.ts` (the `showPreviousValue`/`showNextValue` methods) and the history navigation context registration. A workaround via keybindings.json exists but the default should be less error-prone.
