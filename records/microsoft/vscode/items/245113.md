# Issue #245113 — All VS Code windows go to one display when reconnecting to RDP

[https://github.com/microsoft/vscode/issues/245113](https://github.com/microsoft/vscode/issues/245113)

## Summary

A workaround for this exact RDP multi-monitor window restoration issue was implemented in commit 1f48d5756c8 (PR #248537) and shipped in VS Code 1.101.0. The fix listens for `display-added` events and restores maximized windows to their saved positions on RDP reconnection, gated behind the `enable-rdp-display-tracking` flag in `argv.json`.

## Change summary

When reconnecting to an RDP session with multiple monitors, all VS Code windows collapse to a single display instead of being restored to their original monitors.

## Best solution

Keep the shipped fix (commit 1f48d5756c8, available since 1.101.0). Users enable it by setting `"enable-rdp-display-tracking": true` in their `argv.json`. No further core change is needed; the upstream Electron issue (electron/electron#47016) tracks a proper platform fix.
