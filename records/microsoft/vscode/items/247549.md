# Issue #247549 — Cannot interact with hover produced by Terminal warning

[https://github.com/microsoft/vscode/issues/247549](https://github.com/microsoft/vscode/issues/247549)

## Summary

The terminal environment variable warning hover cannot be interacted with (it disappears before the user can mouse over to its actions). Issue #300250 was closed as a duplicate of this one, confirming it's the canonical issue. No fix found on main.

## Change summary

The terminal warning hover (environment variable stale notification) dismisses before the user can interact with its actions or read its full content.

## Best solution

The hover produced by the terminal environment variable status should be interactive (sticky/lockable) so users can mouse over it and click the offered actions. The fix likely involves how the hover is triggered from the status icon in the terminal panel header — possibly switching from a tooltip-style hover to a properly interactive one via `showInstantHover` or ensuring the hover target geometry allows the mouse to reach the hover content.
