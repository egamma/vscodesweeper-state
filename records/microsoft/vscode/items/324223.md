# Issue #324223 — Performance loss in the VS Code `fs` abstraction that potentionally hits all extensions

[https://github.com/microsoft/vscode/issues/324223](https://github.com/microsoft/vscode/issues/324223)

## Summary

Confirmed on current main: every `workspace.fs` operation awaits `$ensureActivation` RPC to the renderer even when the provider is already registered locally, causing significant overhead for extensions doing many small FS operations. An open PR (#324070) addresses this but has not merged.

## Change summary

The `workspace.fs` API incurs an unnecessary per-operation RPC round-trip to the renderer process for `$ensureActivation`, severely degrading throughput for extensions performing many small file operations.

## Best solution

Cache the activation state per scheme in the extension host so `$ensureActivation` is only sent once (as PR #324070 proposes). The fix is localized to `src/vs/workbench/api/common/extHostFileSystemConsumer.ts`. Review and merge the existing PR.
