## Summary

Team-authored issue (hard keep-open). The keybinding registration at line 238 of accessibleViewActions.ts looks correct (Alt+K, WorkbenchContrib weight), but the precondition requires both `accessibilityHelpIsShown` and `accessibleViewHasUnassignedKeybindings` to be true — the latter context key may not be set in the scenario. The author asked for more repro details; the related WIP PR #296623 was closed without merging.

## Change summary

Alt+K no longer invokes the configure-keybindings command from the Accessibility Help dialog.

## Best solution

Investigate whether the `accessibleViewHasUnassignedKeybindings` context key is being correctly set when the Accessibility Help dialog opens — if that key is never true, the precondition blocks execution. The fix likely lives in `src/vs/workbench/contrib/accessibility/browser/accessibleView.ts` around line 131 where `_hasUnassignedKeybindings` is bound.
