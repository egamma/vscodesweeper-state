# Issue #318533 — Error with image limit in Microsoft PCF project submission

[https://github.com/microsoft/vscode/issues/318533](https://github.com/microsoft/vscode/issues/318533)

## Summary

The recursive directory image scanner in `_collectDirectoryImages` walks all subdirectories without excluding `node_modules`, `.git`, or `.gitignore`-d paths, so attaching any folder with dependencies triggers a false-positive image limit warning. The issue is already labeled `info-needed` with a maintainer engaged — keeping open.

## Change summary

Attaching a project folder to the Chat window warns about exceeding the image limit even when the project's own files contain no images, because the scanner counts images in `node_modules` and other dependency directories.

## Best solution

The `_collectDirectoryImages` method in `src/vs/workbench/contrib/chat/browser/attachments/chatAttachmentResolveService.ts` should exclude well-known non-user directories (`node_modules`, `.git`, `dist`, `out`, etc.) or respect the workspace's file-exclude / search-exclude patterns when counting images. This would eliminate false-positive warnings for projects whose dependencies contain images.
