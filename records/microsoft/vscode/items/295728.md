# Issue #295728 — Claude Opus 4.6 image support blocked

[https://github.com/microsoft/vscode/issues/295728](https://github.com/microsoft/vscode/issues/295728)

## Summary

Vision was gated behind a preview-features entitlement check that blocked enterprise users. That gate has been fully removed in the "GA vision" commit on main — the `isEditorPreviewFeaturesEnabled()` guard no longer exists, so all Claude models with vision capability (including Opus 4.6) will receive images.

## Change summary

Pasting an image into chat with Claude Opus 4.6 selected showed "images aren't supported" and did not send the image to the model, despite the model having vision capabilities.

## Best solution

Keep the shipped GA-vision change (commit f170fd1f23d); no further work needed. The preview-features gate that blocked enterprise vision has been removed, and vision is now generally available for all models that support it.
