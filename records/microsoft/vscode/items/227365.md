# Issue #227365 — Issue Reporter should include warning if VSCode is not pure

[https://github.com/microsoft/vscode/issues/227365](https://github.com/microsoft/vscode/issues/227365)

## Summary

Team-authored feature request to surface a warning in the Issue Reporter when the VS Code installation has been modified by extensions (not pure). Still valid — the integrity service exists but the Issue Reporter doesn't consume it yet. Keeping open per team-authored policy.

## Change summary

Add a warning in the Issue Reporter when the VS Code installation integrity check shows the build has been modified.

## Best solution

Wire the existing `IntegrityService.isPure()` check into the Issue Reporter UI (likely in `src/vs/workbench/contrib/issue/browser/`) so that a visible warning banner appears when the installation is corrupted, helping maintainers quickly identify extension-modified builds in bug reports.
