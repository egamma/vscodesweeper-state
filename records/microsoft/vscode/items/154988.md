# Issue #154988 — List widget - add api for a list item to specify not to receive focus

[https://github.com/microsoft/vscode/issues/154988](https://github.com/microsoft/vscode/issues/154988)

## Summary

Team-authored feature request for a list widget API to mark items as non-focusable. Not yet implemented on main — the IListVirtualDelegate and related interfaces have no such capability. Keeping open per team-member authorship rule.

## Change summary

Allow list widget items to declare they should never receive focus, useful for items like the SCM commit input field.

## Best solution

Add an optional method (e.g. `isFocusable?(element: T): boolean`) to IListVirtualDelegate or a navigation provider, and have the focus/keyboard-navigation logic in listWidget.ts skip items that return false. The SCM view would use this for its input-box and action-button rows.
