# Issue #295159 — Splitting a terminal window will open the terminal in the main workbench window, not the terminal window

[https://github.com/microsoft/vscode/issues/295159](https://github.com/microsoft/vscode/issues/295159)

## Summary

Real bug filed by the terminal maintainer, milestoned On Deck — the split action doesn't properly route the new terminal to the auxiliary window where the split was initiated. Hard keep-open due to milestone.

## Change summary

Splitting a terminal in an auxiliary (terminal) window incorrectly opens the split in the main workbench window instead of the same auxiliary window.

## Best solution

Fix the split terminal action to detect when invoked from an auxiliary window and route the created terminal instance to that same window, likely in `src/vs/workbench/contrib/terminal/browser/terminalActions.ts` around the Split command handler (line ~1067) by passing appropriate location options that respect the current window context.
