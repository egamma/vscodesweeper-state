# Issue #275318 — Timeline does not track renames

[https://github.com/microsoft/vscode/issues/275318](https://github.com/microsoft/vscode/issues/275318)

## Summary

Still reproduces on main: git log --follow correctly returns commits across renames, but the timeline provider always uses the current file URI when building the git show URI, so pre-rename commits fail with 'file not found'. The COMMIT_FORMAT lacks path information and no --name-status is passed.

## Change summary

Clicking a Timeline entry from before a file rename/move fails with 'file was not found' because the git URI uses the current path instead of the historical path.

## Best solution

Add --name-status (or --diff-filter=R with path output) to the logFile call, parse the historical file path from the output, store it on GitTimelineItem, and use it in resolveTimelineOpenDiffCommand and toGitUri when the path differs from the current one. The fix touches extensions/git/src/git.ts (logFile + parseGitCommits), extensions/git/src/timelineProvider.ts (GitTimelineItem + provideTimeline), and extensions/git/src/commands.ts (resolveTimelineOpenDiffCommand).
