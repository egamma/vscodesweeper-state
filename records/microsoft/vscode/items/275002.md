# Issue #275002 — Why does scm.repositories.selectionMode single exist?

[https://github.com/microsoft/vscode/issues/275002](https://github.com/microsoft/vscode/issues/275002)

## Summary

The default was already switched to 'multiple' for stable (commit 60bd43c8a20), addressing the core concern. However, 'single' mode was retained because it enables richer repository exploration (branches, tags sub-nodes) — this design discussion about whether to remove 'single' entirely remains open.

## Change summary

Question whether the 'single' option for scm.repositories.selectionMode should be removed, making 'multiple' the only behavior.

## Best solution

The default is now 'multiple' (the main ask). Whether to remove 'single' entirely depends on the repository explorer UI direction — the single-select mode enables showing child nodes (branches, tags) per repo, which multi-select doesn't support cleanly. Keep open for the SCM team to decide if that explorer direction is still planned.
