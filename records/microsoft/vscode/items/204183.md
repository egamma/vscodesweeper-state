# Issue #204183 — Add cursor below and above don't work in readonly files

[https://github.com/microsoft/vscode/issues/204183](https://github.com/microsoft/vscode/issues/204183)

## Summary

Still reproduces on main: InsertCursorAbove and InsertCursorBelow explicitly bail out when `cursorConfig.readOnly` is true (lines 82–84, 134–136 of multicursor.ts), even though adding cursors is a non-mutating operation that should work in readonly files — consistent with how Ctrl/Cmd+D (addSelectionToNextFindMatch) already works there.

## Change summary

Add Cursor Above/Below commands silently do nothing in readonly editors despite being non-mutating cursor operations.

## Best solution

Remove or relax the `readOnly` early-return guard in `InsertCursorAbove.run()` and `InsertCursorBelow.run()` — these only call `setCursorStates`, which does not mutate the document. The `pushStackElement` call before it can also be removed/guarded since no edit follows. This aligns with other cursor-only commands that already work in readonly editors.
