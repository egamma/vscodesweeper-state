# Issue #185578 — Git - VS Code doesn't act correctly when re-using a merged PR's branch

[https://github.com/microsoft/vscode/issues/185578](https://github.com/microsoft/vscode/issues/185578)

## Summary

The acknowledged bug (VS Code not detecting when a branch's upstream is removed) was fixed by the same maintainer who confirmed it. Two commits address this: one fixes the upstream state-change detection logic (wrong comparison operators), and the other switches the SCM UI to show "Publish" instead of "Push/Sync" when no upstream exists. Both shipped in 1.87.0.

## Change summary

After removing a branch's remote upstream (e.g. via `git fetch --prune` or `git branch --unset-upstream`), VS Code continued showing ahead/behind counts and sync actions instead of recognizing the branch as unpublished.

## Best solution

Keep the shipped fixes in commits d96a8b66cec and 45e2e2072fa (1.87.0). The upstream state detection and the publish-vs-sync UI action now correctly reflect when a local branch has no remote tracking branch. No further change needed.
