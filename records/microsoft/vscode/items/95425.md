# Issue #95425 — Improve rendering of find matches touching each other

[https://github.com/microsoft/vscode/issues/95425](https://github.com/microsoft/vscode/issues/95425)

## Summary

Valid feature request to improve visual differentiation of adjacent find matches by rendering individual outlines (à la Sublime Text). No evidence of implementation on main; the find match border rendering still uses simple CSS `border` per line, which merges adjacent matches visually.

## Change summary

When find matches touch each other (adjacent lines or multiline), they visually merge into one block, making it impossible to distinguish individual match boundaries.

## Best solution

Implement Sublime-style per-match outline rendering in the editor's find decoration layer so that adjacent matches each get a distinct visual border. This likely involves changes to the decoration rendering in `src/vs/editor/contrib/find/browser/findWidget.ts` and possibly the underlying decoration CSS strategy. Given the niche frequency of adjacent multiline matches, this is low priority but would improve safety for regex replace-all operations.
