# Issue #233151 — [WSL2] Use relative paths when linking to files

[https://github.com/microsoft/vscode/issues/233151](https://github.com/microsoft/vscode/issues/233151)

## Summary

Still reproduces on main: relative `file://./` link resolution in the editor links contribution guards on `modelUri.scheme === Schemas.file`, which is false in WSL2/remote contexts where the scheme is `vscode-remote`. The relative path logic is never reached.

## Change summary

Ctrl+clicking a relative `file://./bar.ts` link in the editor fails to open the file when working in WSL2 because the relative-path resolution only fires for local `file://` model URIs.

## Best solution

Relax the scheme guard in `src/vs/editor/contrib/links/browser/links.ts` (line 234) to also handle `vscode-remote` model URIs, resolving the relative path against the remote workspace root. This is a small, localized fix.
