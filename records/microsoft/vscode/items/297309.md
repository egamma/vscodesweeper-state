# Issue #297309 — CTRL-F freezes terminal with large history on initial search

[https://github.com/microsoft/vscode/issues/297309](https://github.com/microsoft/vscode/issues/297309)

## Summary

Verified duplicate of #242713 — both report the terminal find widget freezing/blocking the renderer when searching large scrollback buffers, with short search terms producing many matches. The root cause is xterm.js's synchronous search addon, tracked upstream at xtermjs/xterm.js#5176.

## Change summary

Terminal find (Ctrl+F) freezes or massively slows down when searching through large scrollback history, especially with short search terms that produce many matches.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/242713, which tracks the same renderer-blocking search with large scrollback and has the upstream xterm.js issue linked (xtermjs/xterm.js#5176). The fix requires the xterm search addon to run off the main thread or yield during search.
