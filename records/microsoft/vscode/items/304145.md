# Issue #304145 — Chat Input collapses model picker eagerly

[https://github.com/microsoft/vscode/issues/304145](https://github.com/microsoft/vscode/issues/304145)

## Summary

Team-authored bug (MEMBER) about the chat input toolbar collapsing the model picker behind overflow too eagerly on reload. Related toolbar/overflow fixes have landed since filing (e.g. toolbar collapse UX fix, per-action min-widths), but none reference this issue directly. Keeping open per hard keep-open rule for team issues.

## Change summary

After reload, the model picker in the chat input collapses entirely behind the overflow '...' menu even when it previously showed most of the model name.

## Best solution

The toolbar relayout logic in `chatInputPart.ts` (around the `_toolbarRelayoutScheduler` and `getToolbarsWidthCompact`) should preserve the model picker's prior expanded state across reloads rather than defaulting to a collapsed state. Recent commits like `1677f391f77` ('toolbar ux fix for context and collapse') and `ffe6e9afdd3` ('per-action min-widths') may have partially addressed this — the author should verify on current main.
