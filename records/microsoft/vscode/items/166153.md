# Issue #166153 — Change undo/redo to time based, AKA allow undo without erasing redo

[https://github.com/microsoft/vscode/issues/166153](https://github.com/microsoft/vscode/issues/166153)

## Summary

Feature request with an assigned 'On Deck' milestone — a scheduling decision by maintainers that blocks any close proposal. The request for non-destructive undo (preserving redo history on new edits) is a well-known UX pattern but would be a significant architectural change to the editor's undo system.

## Change summary

Change the undo/redo model so that making an edit after undoing does not erase the redo stack, instead treating history as a linear timeline.

## Best solution

This is a deliberate product decision about undo semantics that a maintainer placed On Deck. If pursued, the implementation lives in `src/vs/editor/common/model/editStack.ts` and related undo infrastructure. The linked 'klonk' approach (time-linear undo) is one design; another is branching undo trees. Either represents a significant change to core editor behavior and needs careful UX design to avoid confusing mainstream users.
