# Issue #277102 — Support array-based format for `github.copilot.chat.customOAIModels` to avoid key collisions

[https://github.com/microsoft/vscode/issues/277102](https://github.com/microsoft/vscode/issues/277102)

## Summary

Valid feature request with strong community support (30 upvotes, promoted to Backlog). The setting remains object-keyed (`Record<string, ...>`) on current main — no array format has been implemented. Keeping open for the team to design the migration path.

## Change summary

Change `github.copilot.chat.customOAIModels` from an object-keyed format to an array-based format so multiple providers can offer the same model name without JSON key collisions.

## Best solution

Migrate the setting schema to an array of entries (each with an explicit `model` field and a provider-scoped identity), with backward compatibility for the existing object format. The change would touch `extensions/copilot/src/platform/configuration/common/configurationService.ts` and all consumers of `CustomOAIModels`. A migration path reading both formats during a transition period is the safest approach.
