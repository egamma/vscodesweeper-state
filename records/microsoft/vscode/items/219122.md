# Issue #219122 — TreeError [FileExplorer] Data tree node not found: ExplorerItem: base

[https://github.com/microsoft/vscode/issues/219122](https://github.com/microsoft/vscode/issues/219122)

## Summary

Still reproducible on main: `itemsCopied` in `explorerView.ts` calls `tree.rerender(item)` on previously-cut items without a `hasNode` guard, which throws a TreeError when the item is part of a compressed node or otherwise absent from the tree's internal node map (e.g. when compact folders compresses a parent into a child).

## Change summary

Cutting a folder in the File Explorer and then cutting a child folder throws a TreeError because `rerender` is called on the previously-cut item without checking `hasNode`.

## Best solution

Guard the `rerender` calls in `itemsCopied` (explorerView.ts:912–914) with `this.tree.hasNode(item)` checks, consistent with the pattern already used elsewhere in the same file (lines 63, 65, 746, 935).
