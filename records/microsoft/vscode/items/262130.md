# Issue #262130 — Cannot `Tab` out of a focused chat request

[https://github.com/microsoft/vscode/issues/262130](https://github.com/microsoft/vscode/issues/262130)

## Summary

Still open on main — no fix has landed. A team member (justschen) acknowledged the issue and identified the root cause (tab indexes on the chat list rows) but no commit addresses it yet.

## Change summary

Pressing Tab while a chat request row is focused does not move focus out of the row, trapping keyboard-only users.

## Best solution

Update tab-index handling in the chat list renderer (likely `src/vs/workbench/contrib/chat/browser/chatListRenderer.ts` or the underlying list widget) so that Tab moves focus to the next focusable element outside the current row, consistent with standard keyboard navigation.
