# Issue #136390 — Giant, inaccurate mouse cursor on Wayland when using high DPI

[https://github.com/microsoft/vscode/issues/136390](https://github.com/microsoft/vscode/issues/136390)

## Summary

Still a live upstream Electron/Chromium issue — cursor scaling on Wayland with high-DPI and multi-monitor setups continues to be reported (most recently via issue #304308 on Ubuntu 26.04 with VS Code 1.109/Electron 42). The root cause is in Chromium's Wayland cursor handling, not in VS Code's own code. Keeping open as the established tracking issue with 33 upvotes.

## Change summary

Mouse cursor appears oversized and click position is inaccurate on Wayland when using high-DPI or multi-monitor setups with different scale factors.

## Best solution

Continue tracking upstream. The fix must come from Chromium/Electron's Wayland cursor scaling implementation. VS Code ships Electron 42 now (up from ~15 when filed), and while some configurations improved, fractional scaling and mixed-DPI multi-monitor setups still trigger the bug. Issue https://github.com/microsoft/vscode/issues/304308 is a recent duplicate that should be consolidated here. The `confirmation-pending` label is appropriate — a team member could ask reporters to re-test on the latest Insiders with Electron 42.
