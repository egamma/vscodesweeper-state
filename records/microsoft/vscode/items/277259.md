# Issue #277259 — diffEditor.removedTextBorder bug border overlaps a bug that displaces lines

[https://github.com/microsoft/vscode/issues/277259](https://github.com/microsoft/vscode/issues/277259)

## Summary

Still reproduces on main: the diff editor uses `border` for removed-text decorations which displaces lines, whereas `outline` (which doesn't affect layout) would fix it. Multiple open PRs propose the same solution but none have been merged.

## Change summary

Setting `diffEditor.removedTextBorder` adds a CSS border that shifts lines down/right, unlike `insertedTextBorder` which doesn't cause displacement.

## Best solution

Change `border` to `outline` for `.line-delete` and `.char-delete` in `src/vs/editor/browser/widget/diffEditor/style.css` (lines 266-267), matching the fix proposed in the open community PRs. One of those PRs should be reviewed and merged.
