# Issue #295844 — Error during update

[https://github.com/microsoft/vscode/issues/295844](https://github.com/microsoft/vscode/issues/295844)

## Summary

Still active: the initial fix (PR #313791) was merged then reverted (commit a6e058c0dbd) because it triggered anti-virus alerts via hidden PowerShell; the maintainer reopened this issue to track the remaining work on an alternative solution. Multiple users still report the problem as of July 2026.

## Change summary

VS Code update fails with 'Access is denied' on Windows because dllhost.exe (COM Surrogate hosting the Explorer context-menu DLL) holds a lock on the old installation directory.

## Best solution

The update service on Windows needs a non-PowerShell mechanism to release the COM surrogate lock on `code_explorer_command_x64.dll` before deleting old version directories. The team member (@dmitrivMS) is actively working on an alternative approach after the PowerShell-based fix was reverted. Keep open until that lands.
