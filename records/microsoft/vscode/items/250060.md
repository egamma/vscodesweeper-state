# Issue #250060 — Accessibility: Monaco Editor Progress Bar Violates "4.1.2 Name, Role, Value" Requirement

[https://github.com/microsoft/vscode/issues/250060](https://github.com/microsoft/vscode/issues/250060)

## Summary

Already fixed on main: commit 5ba9ca4ec1e added an aria-label to the progress bar (defaulting to a localized 'Progress' label and allowing overrides). Shipped in 1.110.

## Change summary

The Monaco Editor progress bar lacked an aria-label, making it inaccessible to screen readers.

## Best solution

Keep the shipped fix in commit 5ba9ca4ec1e (1.110); the progress bar now has role="progressbar", aria-valuemin, and aria-label attributes. No further change needed.
