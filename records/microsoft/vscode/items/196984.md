# Issue #196984 — Editor - use editor space when vertical scrollbar is hidden

[https://github.com/microsoft/vscode/issues/196984](https://github.com/microsoft/vscode/issues/196984)

## Summary

Team-authored feature request, still valid on main: the editor layout always reserves vertical scrollbar width even when the scrollbar is hidden. Keeping open per team-author policy.

## Change summary

When the vertical scrollbar is hidden, reclaim its reserved width for editor content so word wrapping uses the full available space.

## Best solution

In `src/vs/editor/common/config/editorOptions.ts` around line 2971, check whether `scrollbar.vertical === ScrollbarVisibility.Hidden` and, if so, use 0 instead of `verticalScrollbarWidth` when computing `viewportColumn`. The same adjustment may be needed for the minimap layout input at line 2955.
