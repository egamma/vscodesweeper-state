# Issue #184357 — [Accessibility]: Make syntax highlight accessible to screen reader users via a speech scheme

[https://github.com/microsoft/vscode/issues/184357](https://github.com/microsoft/vscode/issues/184357)

## Summary

Valid accessibility feature request already accepted into Backlog with active community discussion (most recent May 2025). No implementation exists on main; the work remains to be done.

## Change summary

Make syntax highlighting information accessible to screen reader users via speech scheme (different voices or CSS formatting attributes per syntax token type).

## Best solution

Implement a mechanism to expose syntax token types to screen readers — either via HTML `lang` attributes to trigger voice switching, or via standard CSS color/font attributes that screen readers like NVDA can query. This likely touches the editor's accessible view and tokenization rendering in `src/vs/editor/` and `src/vs/workbench/contrib/accessibility/`. Collaboration with screen reader vendors (NVDA, JAWS) would inform the best approach.
