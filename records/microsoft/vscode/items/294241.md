# Issue #294241 — Copilot Chat can't handle huge files to commit, even with small changes

[https://github.com/microsoft/vscode/issues/294241](https://github.com/microsoft/vscode/issues/294241)

## Summary

Fixed on main and shipped since 1.116.0. Commit 4a3ea92149d added diff size truncation (100KB per diff), untracked file size limits (1MB), file count caps (200 files), and retrieval timeouts to the SCM changes tool and GitDiffService, directly preventing the 'prompt is too long' error when large files like package-lock.json are in the changeset.

## Change summary

Copilot Chat fails with 'prompt is too long' when committing changes that include huge files (e.g. package-lock.json), even when actual code changes are small.

## Best solution

Keep the shipped fix in commit 4a3ea92149d (1.116.0); no further change is needed. The diff truncation and file-size limits in GitDiffService and scmChangesTool prevent oversized prompts from reaching the model.
