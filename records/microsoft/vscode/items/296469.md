# Issue #296469 — Dynamic Cross-Model Agent Delegation in Copilot: Let one LLM (e.g. Claude Opus) intelligently call sub-agents on other models (Gemini, GPT, etc.)

[https://github.com/microsoft/vscode/issues/296469](https://github.com/microsoft/vscode/issues/296469)

## Summary

A feature request for dynamic, agent-autonomous cross-model delegation during Copilot agent sessions. The request is thematically relevant to current agent infrastructure investments but has very low community traction (1 upvote in ~5 months). Keeping open for the community process.

## Change summary

Allow the orchestrating LLM in Copilot agent mode to dynamically spawn sub-agents on different model providers (Anthropic, Google, OpenAI) based on task characteristics, rather than requiring static model selection.

## Best solution

This is a product-direction decision for the Copilot agent orchestration layer. The `copilot` area team should evaluate whether dynamic model routing belongs in the agent host infrastructure or the Copilot service backend. Given only 1 upvote after 147 days, let the community process decide its fate. If the team is already exploring model routing internally (e.g. utility-model routing, sub-agent model selection), this could be absorbed into that work.
