# Issue #200734 — Diff Editor: Swapping Left & Right side is slow due to diff computation

[https://github.com/microsoft/vscode/issues/200734](https://github.com/microsoft/vscode/issues/200734)

## Summary

Team-authored performance issue about diff editor side-swap triggering a full recomputation. No fix has landed on main; the swap action (in commands.ts) still just swaps the models, triggering a fresh diff. Keeping open per team-author rule.

## Change summary

Swapping left/right sides in the diff editor is slow because it recomputes the entire diff instead of reusing or inverting the existing result.

## Best solution

When swapping sides, invert the existing diff result (swap each mapping's original/modified ranges) rather than recomputing from scratch. The optimization likely belongs in the diff editor widget layer (src/vs/editor/browser/widget/diffEditor/) where the swap action lives.
