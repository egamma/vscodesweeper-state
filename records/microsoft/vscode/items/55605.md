# Issue #55605 — Slow when editing a file with very long lines

[https://github.com/microsoft/vscode/issues/55605](https://github.com/microsoft/vscode/issues/55605)

## Summary

Mid-verification (carries `verification-found` label) — must not propose closing. The underlying issue (slow typing on long lines due to forced retokenization) was partially mitigated by the `CHEAP_TOKENIZATION_LENGTH_LIMIT = 2048` guard, but the core problem of retokenizing very long lines (~300ms per keystroke in HTML mode) remains on current main.

## Change summary

Editing a file with very long lines (>10k chars) in modes like HTML causes freezes because each keystroke triggers retokenization of the entire long line.

## Best solution

The remaining work is to either skip or debounce retokenization for lines exceeding the cheap limit during typing, or to split/chunk tokenization of long lines incrementally. The relevant code is in `src/vs/editor/common/model/textModelTokens.ts` (the `CHEAP_TOKENIZATION_LENGTH_LIMIT` constant and `isCheapToTokenize`) and the tokenization backends. This is mid-verification so a maintainer should confirm whether the current state is acceptable or further optimization is needed.
