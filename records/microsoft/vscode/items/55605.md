## Change summary

Editing very long lines, especially embedded/minified JSON in HTML with word wrap off, can be slow because typing triggers expensive tokenization or retokenization.

## Best solution

Keep this open for the tokenization/editor owners to finish or re-verify the long-line typing scenario; the likely end state is either a targeted tokenizer/editor performance fix for very long lines in embedded languages or explicit verification that current main no longer reproduces the HTML case.
