# Issue #155949 — Conflict and change traversal buttons have the same icons in the compare changes view.

[https://github.com/microsoft/vscode/issues/155949](https://github.com/microsoft/vscode/issues/155949)

## Summary

Still reproduces on main: the merge-conflict extension's 'Previous/Next Conflict' buttons use the same arrow-up/arrow-down icons as the diff editor's 'Previous/Next Change' buttons, making them visually indistinguishable when both appear in the editor title bar.

## Change summary

When viewing a merge conflict in the compare (diff) view, the conflict navigation and change navigation buttons in the editor title bar use identical icons with no distinguishing text.

## Best solution

Differentiate the merge-conflict navigation buttons from the diff-change navigation buttons — either by using distinct icons for conflict navigation (e.g. a warning/conflict-themed variant) in `extensions/merge-conflict/package.json`, or by conditionally hiding the diff-change buttons when merge-conflict buttons are shown. The fix touches `extensions/merge-conflict/package.json` (icon definitions) and possibly the `when` clauses or `editor.contribution.ts`.
