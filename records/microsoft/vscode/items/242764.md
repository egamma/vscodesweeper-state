# Issue #242764 — editor.setDecorations does not render all contentText correctly

[https://github.com/microsoft/vscode/issues/242764](https://github.com/microsoft/vscode/issues/242764)

## Summary

Still reproduces on main: per-decoration `renderOptions` are keyed by a 32-bit `hash()` which causes collisions when many unique `contentText` values are used, resulting in duplicate/wrong decorations.

## Change summary

When many decorations use per-instance `renderOptions.before.contentText`, hash collisions cause some decorations to display the wrong text.

## Best solution

Replace the 32-bit `hash()` keying of per-decoration renderOptions with a collision-resistant key (e.g. JSON.stringify or a longer hash) in `codeEditorWidget.ts`, so distinct `renderOptions` always get distinct decoration type registrations.
