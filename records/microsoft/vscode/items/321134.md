# Issue #321134 — Focus moves to static option list items but no visible focus indicator is present:A11y_VisualStudioCode_Improved UI for MCP elicitation_Keyboard

[https://github.com/microsoft/vscode/issues/321134](https://github.com/microsoft/vscode/issues/321134)

## Summary

Still reproduces on main: the fix PR #321147 was closed without merging, and current main still suppresses the focus outline on multi-select list items with `outline: none`. A team member confirmed reproduction on latest Insiders on July 20. Team-authored issue with an assigned milestone — keeping open.

## Change summary

Multi-select list items in chat elicitation forms receive DOM focus during keyboard navigation but show no visible focus indicator because `outline: none` is applied to `.chat-question-list-item:focus`.

## Best solution

Land the fix from the closed PR: split the `.chat-question-list-item:focus` and `.chat-question-list:focus` rules so that the list container keeps `outline: none` (it uses aria-activedescendant) while list items get `outline: 1px solid var(--vscode-focusBorder); outline-offset: -1px`. The change is in `src/vs/workbench/contrib/chat/browser/widget/chatContentParts/media/chatQuestionCarousel.css` around line 205.
