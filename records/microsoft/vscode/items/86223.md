# Issue #86223 — Proxy configuration doesn't allow URLs with port number AND pac files

[https://github.com/microsoft/vscode/issues/86223](https://github.com/microsoft/vscode/issues/86223)

## Summary

The `http.proxy` setting is designed for proxy server addresses, not PAC file URLs. The validation regex intentionally allows scheme + auth + host + port but not arbitrary paths. PAC file resolution is handled through the system's proxy auto-configuration (Electron's resolveProxy). This is working as designed.

## Change summary

Allow the `http.proxy` setting to accept URLs with both a port number and a path (e.g., for PAC file URLs).

## Best solution

Close as `*as-designed` — the `http.proxy` field is a proxy server endpoint, not a PAC file location. PAC-based proxy resolution is handled by the OS/Electron layer (via `resolveProxy`). Users needing PAC support should configure it at the OS level, which VS Code will respect automatically through `http.proxySupport`.
