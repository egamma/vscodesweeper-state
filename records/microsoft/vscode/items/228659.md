# Issue #228659 — The bottom border of Manage menu is cut

[https://github.com/microsoft/vscode/issues/228659](https://github.com/microsoft/vscode/issues/228659)

## Summary

Still reproduces on main: the menu's 1px outline is not accounted for in the viewport-clamping layout computation, so when positioned at the bottom edge (status bar hidden), the outline is clipped. An open PR exists but has not merged.

## Change summary

The Manage menu's bottom border (CSS outline) is cut off when the status bar is hidden and the menu renders at the viewport's bottom edge.

## Best solution

Land or close the open PR https://github.com/microsoft/vscode/pull/235711, which addresses the outline not being factored into the layout height. The fix lives in `src/vs/base/browser/ui/contextview/contextview.ts` (the `layout()` method's view height calculation needs to include the outline width).
