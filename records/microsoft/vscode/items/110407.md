# Issue #110407 — command tooltip and keyboard shortcut name should be the same.

[https://github.com/microsoft/vscode/issues/110407](https://github.com/microsoft/vscode/issues/110407)

## Summary

The issue has milestone 'On Deck' (a real scheduling milestone), so it must stay open. The underlying problem also persists on current main: search viewlet button tooltips ('Match Case', 'Match Whole Word', 'Use Regular Expression' in src/vs/base/browser/ui/findinput/findInputToggles.ts) still differ from the command titles shown in the keybinding editor ('Toggle Case Sensitive', 'Toggle Whole Word', 'Toggle Regex' in src/vs/workbench/contrib/search/browser/searchActionsNav.ts).

## Change summary

Search viewlet toggle button tooltips and their corresponding command titles in the keybinding editor use different wording, making it hard to find the right shortcut to customize.

## Best solution

Align the command titles registered in searchActionsNav.ts (e.g. 'Toggle Case Sensitive') with the tooltip labels in findInputToggles.ts (e.g. 'Match Case') — or vice versa — so users searching the keybinding editor find the same term they see in the tooltip. The fix touches both files and possibly the search editor equivalents.
