# Issue #277934 — GitHub Copilot fails to process workspace when multiple identical copilot-instructions.md files are present

[https://github.com/microsoft/vscode/issues/277934](https://github.com/microsoft/vscode/issues/277934)

## Summary

The issue is real and traceable in current source: `listAgentInstructions` collects copilot-instructions.md from every workspace folder without content-based deduplication, so identical files overwhelm the context window in multi-root workspaces. Still valid on main.

## Change summary

In a multi-root workspace with many repos containing identical copilot-instructions.md files, Copilot's context window is consumed by duplicate instruction content, making it unable to answer workspace-level questions.

## Best solution

Add content-hash-based deduplication in `listAgentInstructions` (src/vs/workbench/contrib/chat/common/promptSyntax/service/promptsServiceImpl.ts, around line 786–808) so that files with identical content are collapsed into a single instruction entry. The symlink dedup logic already there could be extended to compare content hashes.
