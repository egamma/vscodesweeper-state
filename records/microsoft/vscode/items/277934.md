# Issue #277934 — GitHub Copilot fails to process workspace when multiple identical copilot-instructions.md files are present

[https://github.com/microsoft/vscode/issues/277934](https://github.com/microsoft/vscode/issues/277934)

## Summary

Still valid on main: `listAgentInstructions` collects every `copilot-instructions.md` from each workspace folder without content-based deduplication, so a multi-root workspace with 13+ identical instruction files fills the LLM context with redundant text, degrading Copilot's ability to answer workspace questions.

## Change summary

In multi-root workspaces with many identical copilot-instructions.md files, Copilot floods its prompt context with redundant instruction content and becomes unable to answer workspace-level questions.

## Best solution

Add content-based deduplication in `listAgentInstructions` (or in `_addAgentInstructions`) so that files with identical content are included only once. The symlink dedup at promptsServiceImpl.ts:770-790 already handles path-based dedup; extend it with a content hash check.
