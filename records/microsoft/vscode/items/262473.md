# Issue #262473 — When the content of a text file is large, case conversion freezes

[https://github.com/microsoft/vscode/issues/262473](https://github.com/microsoft/vscode/issues/262473)

## Summary

Confirmed regression where case conversion on large files freezes due to synchronous DOM updates in NativeEditContext. Partially mitigated by commit a6967f74525 but PR #310901 (still open) proposes the full fix by deferring to render time. Keep open until the PR lands.

## Change summary

Case conversion (Transform to Upper/Lower Case) on large files causes a UI freeze because NativeEditContext synchronously updates the DOM for each content change.

## Best solution

Merge PR #310901 which defers NativeEditContext DOM refresh to render time, fully eliminating the synchronous DOM thrashing during bulk edits like case conversion.
