# Issue #227694 — Incorrect behavior of granting access for the extension to existing authentication session using Manage Trusted Extensions functionality

[https://github.com/microsoft/vscode/issues/227694](https://github.com/microsoft/vscode/issues/227694)

## Summary

Still reproduces on main: when access is granted via 'Manage Trusted Extensions', the 'Grant access' menu item is never cleaned up because `AuthenticationExtensionsService` doesn't listen to `onDidChangeExtensionSessionAccess` to remove stale access requests.

## Change summary

Granting access to an extension via the 'Manage Trusted Extensions' dialog does not remove the stale 'Grant access to {Provider} for {Extension}' menu item from the Accounts menu.

## Best solution

In `authenticationExtensionsService.ts`, listen to `IAuthenticationAccessService.onDidChangeExtensionSessionAccess` and call `removeAccessRequest` for extensions that have been granted access, similar to how it's done in the prompt-based grant flow (line 281).
