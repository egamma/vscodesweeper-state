# Issue #210899 — File Explorer Collapsed Items Focus Issue

[https://github.com/microsoft/vscode/issues/210899](https://github.com/microsoft/vscode/issues/210899)

## Summary

Team-authored issue, still reproduces on main. The CompressedNavigationController constructor always resets the active index to the last element on every re-render, so losing/regaining window focus (which triggers a re-render) snaps focus back to the deepest compressed label instead of preserving the user's selection.

## Change summary

Clicking a compressed folder in File Explorer resets the active label to the last segment when VS Code loses and regains focus, instead of preserving the previously selected segment.

## Best solution

Persist the active compressed-label index across re-renders — either store it in the template data (as benibenj suggested) or look it up from the previous CompressedNavigationController instance before disposing it. The fix is localized to `CompressedNavigationController` construction in `src/vs/workbench/contrib/files/browser/views/explorerViewer.ts`.
