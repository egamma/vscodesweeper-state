# Issue #299374 — Numpad keys not working in terminal TUI applications after 1.110 update (Electron 39 / Chromium 142 regression)

[https://github.com/microsoft/vscode/issues/299374](https://github.com/microsoft/vscode/issues/299374)

## Summary

Confirmed upstream Chromium 142 regression (numpad keys report `key: "Unidentified"`) that specifically affects terminal TUI apps using enhanced keyboard protocols (Kitty/CSI u). No fix has been merged on main; PR #299408 is still open. Already correctly labeled as upstream/chromium/electron-39-update — keep open and let the linked PR or an upstream Chromium fix resolve it.

## Change summary

Numpad keys are ignored by TUI applications in the integrated terminal because Chromium 142 reports them as `key: "Unidentified"`, which xterm.js passes through to enhanced-keyboard-mode apps.

## Best solution

The fix likely belongs either upstream in Chromium (issues 405793116 / 429730008) or as a workaround in xterm.js or VS Code's terminal keyboard handling (mapping `keyCode` back to proper key names when `key` is `"Unidentified"`). Review and land https://github.com/microsoft/vscode/pull/299408 if the approach is sound, or coordinate with the xterm.js maintainers for a fix at that layer. Related editor-level issue https://github.com/microsoft/vscode/issues/283460 tracks the same Chromium root cause.
