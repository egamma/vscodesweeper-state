# Issue #266366 — Prompts aren't updated when the prompt file updates

[https://github.com/microsoft/vscode/issues/266366](https://github.com/microsoft/vscode/issues/266366)

## Summary

Valid bug — no fix commit found referencing this issue, and the current welcome-view rendering path in chatWidget.ts does not appear to subscribe to prompt-file content changes. The welcome re-renders on mode changes, agent changes, and entitlement changes, but not when an individual prompt file's description is edited and saved. The area has been heavily refactored (AI Customization welcome page redesign), so the exact reproduction path may have shifted, but the underlying gap — no file-change listener triggering a welcome re-render — likely persists.

## Change summary

Editing and saving a prompt file's description does not update the chat welcome view until a full window reload.

## Best solution

Subscribe to prompt-file content-change events (e.g. via the prompts service's `onDidChange*` events) and trigger `renderWelcomeViewContentIfNeeded()` or the equivalent welcome-card refresh in the chat widget and/or the AI Customization welcome page. The likely files are in `src/vs/workbench/contrib/chat/browser/widget/chatWidget.ts` and `src/vs/workbench/contrib/chat/browser/aiCustomization/aiCustomizationWelcomePagePromptLaunchers.ts`.
