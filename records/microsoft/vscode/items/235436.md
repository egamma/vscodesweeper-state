# Issue #235436 — 1px gap to side of minimap lets code show through

[https://github.com/microsoft/vscode/issues/235436](https://github.com/microsoft/vscode/issues/235436)

## Summary

Still reproduces on main: the minimap container div never sets `overflow: hidden`, so overflowing canvas content (long lines) bleeds ~1px past the minimap boundary.

## Change summary

Long lines render a 1px strip of code visible to the right of the minimap boundary.

## Best solution

Add `overflow: hidden` to the `.monaco-editor .minimap` rule in `minimap.css` (or set it on the domNode in `minimap.ts`), so the canvas is clipped to the container width.
