# Issue #309458 — Worktrees not showing in Source Control

[https://github.com/microsoft/vscode/issues/309458](https://github.com/microsoft/vscode/issues/309458)

## Summary

The reported behavior is the result of an intentional default change in PR #306286 (git.detectWorktrees changed from true to false). A maintainer already pointed to the setting. With 24 upvotes and ongoing community pushback, this should stay open as a discussion about whether the default should be reverted.

## Change summary

Git worktrees stopped appearing in Source Control after the git.detectWorktrees setting default was changed from true to false in VS Code 1.115.0.

## Best solution

Evaluate whether the default for `git.detectWorktrees` should be reverted to `true`, or at minimum surface a more discoverable prompt/notification when worktrees are detected but the setting is off. The 24 upvotes and multiple related issues (https://github.com/microsoft/vscode/issues/314676, https://github.com/microsoft/vscode/issues/311189, https://github.com/microsoft/vscode/issues/317388) suggest the default change caused significant friction for worktree users.
