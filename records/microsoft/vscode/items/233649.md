# Issue #233649 — deleted files don't go to trash

[https://github.com/microsoft/vscode/issues/233649](https://github.com/microsoft/vscode/issues/233649)

## Summary

Confirmed snap-specific bug: the electron-launch script overrides XDG_DATA_HOME, redirecting trash to a snap-internal directory. An open PR (#292808) by a team member directly fixes this — keeping open pending that merge.

## Change summary

Files deleted in VS Code's explorer on the Linux snap build go to a snap-internal trash directory instead of the system trash, silently accumulating tens of gigabytes.

## Best solution

Land https://github.com/microsoft/vscode/pull/292808 which removes the XDG_DATA_HOME override from `resources/linux/snap/electron-launch`, restoring correct system trash behavior for snap users.

## Evidence

- **root cause in source:** `resources/linux/snap/electron-launch` exports `XDG_DATA_HOME="$SNAP_USER_DATA/.local/share"`, redirecting all XDG data (including trash) into the snap-private directory instead of the system location.
- **open fixing PR:** https://github.com/microsoft/vscode/pull/292808 by deepak1556 (team member) removes the XDG_DATA_HOME override; state is OPEN and MERGEABLE.
- **canonical sibling issue:** https://github.com/microsoft/vscode/issues/237608 tracks the broader XDG_DATA_HOME override problem (terminal history, fonts, trash) and is also fixed by the same PR.
- **user impact:** Multiple commenters report 35–200 GB of silently accumulated snap-internal trash; issue has 23 upvotes and external press coverage (The Register).
