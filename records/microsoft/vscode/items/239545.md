Team-authored GPU rendering bug with no fix on main; the canvas positioning issue when toggling line numbers in notebook cells remains open. Hard keep-open (MEMBER author).

## Change summary

Enabling line numbers in a notebook cell with GPU acceleration causes code/decoration overlap and incorrect positioning of the GPU canvas layer.

## Best solution

The GPU renderer needs to handle layout changes (line-number margin appearing/disappearing) by recalculating the canvas offset and re-rendering affected lines. Likely lives in `src/vs/editor/browser/gpu/` — the canvas positioning logic needs to respond to margin-width changes triggered by the line-number toggle.
