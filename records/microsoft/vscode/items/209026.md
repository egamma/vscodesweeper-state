# Issue #209026 — Quick chat resizes on scroll

[https://github.com/microsoft/vscode/issues/209026](https://github.com/microsoft/vscode/issues/209026)

## Summary

Active design disagreement between a contributor and the Quick Chat maintainer about whether scroll-to-resize is intuitive UX. The behavior is intentional and explained, but no consensus has been reached on whether it should change. Already correctly labeled under-discussion.

## Change summary

Quick Chat expands its container height when the user scrolls, rather than scrolling the content in place, which feels unintuitive to some users.

## Best solution

The Quick Chat team should make a final design call: either keep the current scroll-to-resize behavior (and close as *as-designed) or adopt a fixed-height-with-scroll model. The discussion needs a concrete alternative proposal or a firm acceptance of the current design.
