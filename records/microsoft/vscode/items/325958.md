# Issue #325958 — Extension Development Host fails with EADDRINUSE in WSL when using mirrored networking in WSL 2.9, works in 2.7

[https://github.com/microsoft/vscode/issues/325958](https://github.com/microsoft/vscode/issues/325958)

## Summary

This is an upstream WSL issue: EADDRINUSE on random ports only under WSL 2.9 mirrored networking, fixed by downgrading WSL to 2.7, and not reproducible by other mirrored-networking users — the root cause is in the WSL networking stack, not VS Code's extension host code.

## Change summary

Extension Development Host fails to launch in WSL 2.9 with mirrored networking due to EADDRINUSE on random ports.

## Best solution

File upstream with the WSL team (microsoft/WSL) since the regression is tied to WSL 2.9's mirrored networking port binding behavior. VS Code's extension host binds 127.0.0.1 with a port from Node's inspector (src/vs/workbench/services/extensions/electron-browser/localProcessExtensionHost.ts); no code change in VS Code would help — the WSL networking layer is reporting ports as in-use that aren't. A workaround for the reporter is switching to NAT mode or downgrading WSL.
