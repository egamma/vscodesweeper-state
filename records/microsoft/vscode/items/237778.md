# Issue #237778 — Native dialogs are broken, flickering and not readable

[https://github.com/microsoft/vscode/issues/237778](https://github.com/microsoft/vscode/issues/237778)

## Summary

This is a VirtualBox-specific GPU rendering issue on Wayland, identical to the closed #173727. The workaround (`--ozone-platform=wayland --disable-gpu`) was confirmed in that issue. The root cause is in VirtualBox's 3D acceleration / Chromium's rendering under the VM's virtual GPU, not in VS Code's own code. The reporter says the workaround from #173727 doesn't work for them, but hasn't provided detailed diagnostics, and this has sat 549 days with no comments or further engagement.

## Change summary

Native file dialogs flicker and are unreadable when running VS Code on Ubuntu in VirtualBox with 3D acceleration enabled.

## Best solution

This is a VirtualBox virtual GPU + Chromium compositor interaction — not actionable in VS Code's code. The known workaround is `--ozone-platform=wayland --disable-gpu`. The reporter should file with VirtualBox or try upgrading VirtualBox / guest additions. If the issue were to remain open, it would need the `upstream` and `virtual-machine` labels, but with no engagement and an existing workaround, closing as not-reproducible in VS Code's own code is the cleanest outcome.
