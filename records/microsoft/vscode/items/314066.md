# Issue #314066 — Terminal death recovery doesn't work

[https://github.com/microsoft/vscode/issues/314066](https://github.com/microsoft/vscode/issues/314066)

## Summary

Team-authored bug report about agent terminal death recovery failing — the agent doesn't detect when a hidden terminal is killed and continues to believe the process is running. Keeping open for maintainer judgment.

## Change summary

When the user kills a hidden terminal spawned by the chat agent, the agent fails to detect the terminal's death and incorrectly reports the process as still running.

## Best solution

The terminal death detection/recovery path in the chat-terminal integration needs to reliably propagate terminal exit events to the agent, so it can update its state and potentially re-spawn. Likely involves the terminal lifecycle signaling in the agent's tool execution layer.
