# Issue #229606 — ASCII characters should not be part of the editing undo/redo stack

[https://github.com/microsoft/vscode/issues/229606](https://github.com/microsoft/vscode/issues/229606)

## Summary

Still unfixed on main. The native EditContext composition path sends each intermediate ASCII keystroke as a separate compositionType edit, and the undo stack does not group them into a single unit. No fix commit found.

## Change summary

With EditContext and Chinese IME, undoing a composition reveals intermediate ASCII characters one by one instead of removing the entire composed text at once.

## Best solution

The compositionType path in nativeEditContext.ts / cursorTypeEditOperations.ts needs to suppress undo stack boundaries during active composition, so all intermediate edits collapse into one undo group — similar to how textarea handles it. Related to https://github.com/microsoft/vscode/issues/229647.
