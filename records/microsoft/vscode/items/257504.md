# Issue #257504 — NES: Word Wrap not supported

[https://github.com/microsoft/vscode/issues/257504](https://github.com/microsoft/vscode/issues/257504)

## Summary

Still present on main: the NES inline edits preview editor explicitly sets wordWrap to 'off', so inserted/modified lines never wrap even when the main editor wraps. This is a valid UX gap for users working with long lines (Markdown, LaTeX, etc.).

## Change summary

NES inline edit suggestions do not respect the editor's word wrap setting, making long-line changes difficult or impossible to read.

## Best solution

The NES side-by-side preview editor (inlineEditsSideBySideView.ts) should inherit or respect the parent editor's wordWrap setting rather than forcing 'off'. The long-distance preview editor has the same issue. This likely requires layout adjustments to handle variable-height wrapped content in the preview pane.
