# Issue #214187 — SCM - Allow SCM extension to opt out of `SourceControl.statusBarCommands` also appearing in SCM views

[https://github.com/microsoft/vscode/issues/214187](https://github.com/microsoft/vscode/issues/214187)

## Summary

Valid feature request: the SCM renderer still unconditionally prepends statusBarCommands into the SCM views toolbar (lines 175–178 of scmRepositoryRenderer.ts). No opt-out mechanism exists on current main, and no duplicate or canonical issue was found. Keeping open in the Backlog for the SCM area owner to evaluate.

## Change summary

Allow SCM extensions to opt out of having their SourceControl.statusBarCommands also appear in SCM view toolbars.

## Best solution

Add an optional flag (e.g. `statusBarCommandsVisibleInViews`) to the SourceControl API, or a menu contribution point, so extensions can control whether their statusBarCommands appear in the SCM views' toolbars. The change would touch `src/vs/workbench/contrib/scm/browser/scmRepositoryRenderer.ts` (the rendering logic) and the extension API surface in `src/vs/workbench/api/common/extHostSCM.ts`.
