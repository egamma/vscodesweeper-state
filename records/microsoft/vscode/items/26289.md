# Issue #26289 — [json] use the schemastore catalog

[https://github.com/microsoft/vscode/issues/26289](https://github.com/microsoft/vscode/issues/26289)

## Summary

Team-authored feature request still unimplemented on main. The JSON language server trusts and caches schemastore URLs but does not yet fetch the catalog to auto-associate files. Hard keep-open due to MEMBER authorship.

## Change summary

The JSON language server should automatically use the schemastore.org catalog to provide schema associations for files without explicit configuration.

## Best solution

Implement catalog fetching in `extensions/json-language-features/client/src/node/jsonClientMain.ts` or the server, with a `json.schemaStore.enable` setting (defaulting to true) and ensuring user/extension associations take precedence. The discussion in the thread already outlines the design constraints (fallback-only, user-disableable).
