# Issue #26289 — [json] use the schemastore catalog

[https://github.com/microsoft/vscode/issues/26289](https://github.com/microsoft/vscode/issues/26289)

## Summary

Team-authored feature request (MEMBER) with 23 upvotes, still unimplemented on main. The JSON server trusts schemastore.org for downloads but does not fetch the catalog to auto-associate schemas with files. Must keep open per policy.

## Change summary

VS Code's JSON language server should automatically use the SchemaStore catalog to provide schema associations for known config files without requiring manual configuration or extensions.

## Best solution

Implement fetching of http://schemastore.org/api/json/catalog.json in the JSON language server (extensions/json-language-features/server/src/jsonServer.ts) as a fallback source for schema associations, with a `json.schemaStore.enable` setting to disable it and ensuring user/extension associations take precedence.
