# Issue #154146 — The pseudo terminal's name cannot be changed for custom task provider

[https://github.com/microsoft/vscode/issues/154146](https://github.com/microsoft/vscode/issues/154146)

## Summary

Still reproduces on main — task terminals block process-sourced name changes (line 2406 of terminalInstance.ts). Commit 87de0978f3b made this behavior permanent by always preserving the task name. Keeping open per team-author rule; the fix likely needs the custom pty name change to use TitleEventSource.Api instead of Process for task terminals.

## Change summary

Custom task providers using Pseudoterminal.onDidChangeName cannot rename their terminal because task terminals block process-sourced title changes.

## Best solution

In `terminalInstance.ts`, when handling `ProcessPropertyType.Title` changes for terminals backed by a custom pty implementation (customPtyImplementation in shellLaunchConfig), use `TitleEventSource.Api` instead of `TitleEventSource.Process` so the task-terminal guard at line 2406 doesn't block the rename. Alternatively, refine the guard to allow name changes from custom ptys specifically.
