# Issue #283293 — Cannot read properties of undefined (reading 'getViewLineContent')

[https://github.com/microsoft/vscode/issues/283293](https://github.com/microsoft/vscode/issues/283293)

## Summary

The race condition is still present on main: `_constructLines` empties `modelLineProjections` at line 123 before rebuilding, so concurrent event dispatch can observe the incomplete array and crash. PR #285602 proposed a fix but was closed; PR #324235 is open with a different approach.

## Change summary

TypeError crash when `getViewLineContent` is called during `_constructLines` because `modelLineProjections` is temporarily empty/incomplete.

## Best solution

Apply the atomic-replacement pattern from PR #285602 (build into a local array, assign at the end) in `src/vs/editor/common/viewModel/viewModelLines.ts` to prevent partially-updated state from being observed by event handlers during reconstruction.
