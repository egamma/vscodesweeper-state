# Issue #298047 — Using gh Auth for authentication 

[https://github.com/microsoft/vscode/issues/298047](https://github.com/microsoft/vscode/issues/298047)

## Summary

Valid feature request with strong community support (36 upvotes), already promoted to Backlog. The GitHub authentication extension does not currently support reading tokens from `gh auth` or environment variables — the gap remains on current main.

## Change summary

Allow VS Code's GitHub/Copilot authentication to leverage existing `gh` CLI auth tokens or environment variables instead of requiring a separate OAuth flow.

## Best solution

Extend the built-in `github-authentication` extension (under `extensions/github-authentication/`) to detect and offer tokens from `gh auth token` or standard env vars (`GH_TOKEN`, `GITHUB_TOKEN`) as an authentication source, particularly for Copilot sessions. This reduces friction for users who already authenticate via the CLI.
