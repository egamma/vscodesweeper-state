# Issue #313109 — Error during VS Code Insiders installation: “Visual Studio Code - Insiders.app” is damaged and can’t be opened. You should move it to the Bin.

[https://github.com/microsoft/vscode/issues/313109](https://github.com/microsoft/vscode/issues/313109)

## Summary

Still reproduces: the macOS build pipeline notarizes the Insiders .app and .dmg but never staples the notarization ticket, causing Gatekeeper to reject the app as 'damaged' when the online lookup fails. An open PR (#313905) proposes the fix but has not merged. Keeping open.

## Change summary

VS Code Insiders macOS builds are reported as 'damaged' by Gatekeeper because the notarization ticket is not stapled onto the .app/.dmg.

## Best solution

Land https://github.com/microsoft/vscode/pull/313905 (or an equivalent change) to add an `xcrun stapler staple` step after notarization in `build/azure-pipelines/darwin/codesign.ts`, covering both the client .app and the .dmg. Issue https://github.com/microsoft/vscode/issues/313785 is the same underlying problem and should be closed as a duplicate of this one (or vice versa).
