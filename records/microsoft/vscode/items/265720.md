# Issue #265720 — Chat: New file tool calls lack name

[https://github.com/microsoft/vscode/issues/265720](https://github.com/microsoft/vscode/issues/265720)

## Summary

Team-authored bug — keeping open per policy. The issue reports that chat tool calls for file creation display the raw tool call ID instead of a meaningful file name, especially for non-file URI schemes. No fixing commit was found on current main.

## Change summary

Chat tool call UI for `create_file` shows the tool call ID instead of the file name/path.

## Best solution

The tool invocation rendering in `src/vs/workbench/contrib/chat/browser/widget/chatContentParts/toolInvocationParts/` should extract and display a human-readable file name from the tool call arguments (e.g. `filePath`) rather than falling back to the tool call ID. The `chatExternalEditContentPart.ts` already uses `ILabelService.getUriBasenameLabel` for external edits — a similar approach could be applied to the tool call progress part for `create_file` invocations, with special handling for non-file schemes like `vscode-vfs`.
