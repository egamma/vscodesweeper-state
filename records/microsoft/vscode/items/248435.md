# Issue #248435 — Debug on cwd path with exclamation mark and space will fail on second try (when terminal is CWD)

[https://github.com/microsoft/vscode/issues/248435](https://github.com/microsoft/vscode/issues/248435)

## Summary

Confirmed in source: the CMD quoting logic in `prepareCommand` escapes `!` to `^!` even inside double quotes, where the caret is literal — breaking `cd` on paths containing `!` when the debug terminal is reused (second run onward).

## Change summary

Debugging with `console: integratedTerminal` fails on the second run when the configured `cwd` contains an exclamation mark, because the CMD `cd` command receives an incorrectly escaped path.

## Best solution

Fix the `quote` function in `src/vs/workbench/contrib/debug/node/terminals.ts` (ShellType.cmd case) to not escape `!` with `^` when the string will be wrapped in double quotes. The `!` character is only special in CMD with delayed expansion enabled, and `^` is not an escape inside quotes. Alternatively, remove `!` from the regex `([><!^&|])` on the quoting path (the branch that wraps in `"..."`), since it doesn't need escaping there.
