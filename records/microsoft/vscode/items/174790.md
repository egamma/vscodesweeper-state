# Issue #174790 — Git - Support `git diftool -d`, AKA Git directory diff

[https://github.com/microsoft/vscode/issues/174790](https://github.com/microsoft/vscode/issues/174790)

## Summary

Still a valid, unimplemented feature request. The multi-diff editor now exists and could power this, but no directory-diff handling has been added to the CLI diff path on current main. With 16 upvotes and clear prior demand (closed predecessors #6898, #48437, #63548 had ~60 combined votes), it should stay open in the Backlog.

## Change summary

When VS Code is used as `git difftool -d`, it should open a multi-diff editor showing all changed files across two directory trees, rather than failing or opening nothing useful.

## Best solution

Extend the `--diff` CLI handling in `src/vs/workbench/electron-browser/window.ts` (around `onOpenFiles`/`openResources`) to detect when the two diff paths are directories, enumerate their files, and open an `IResourceMultiDiffEditorInput` in the multi-diff editor. This is a natural fit now that the multi-diff editor ships by default. A community contribution seems viable — commenter saierd outlined the approach in Comment 3.
