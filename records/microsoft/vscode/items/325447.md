# Issue #325447 — GitHub Copilot thinking effort purportedly used does not reflect thinking effort requested.

[https://github.com/microsoft/vscode/issues/325447](https://github.com/microsoft/vscode/issues/325447)

## Summary

Legitimate bug: model configuration (thinking effort) is not stored per-request in ChatRequestModel, so editing a submitted prompt shows/mutates the shared configuration store rather than reflecting what was actually used. Changes to thinking effort during an edit persist even if the edit is cancelled.

## Change summary

Editing a submitted prompt shows the wrong thinking effort level and mutates the shared model configuration store independently of what was actually submitted, including on cancel.

## Best solution

Store model configuration (thinking effort) per-request in ChatRequestModel alongside modelId, and restore it into the inline editor's ChatModelConfigurationStore when editing begins. On cancel, revert the configuration store to its pre-edit state. This parallels how modelId is already stored per-request and switched on edit (chatWidget.ts line 1778-1779).
