# Issue #287181 — Activate.ps1 messes up shell integration exit code

[https://github.com/microsoft/vscode/issues/287181](https://github.com/microsoft/vscode/issues/287181)

## Summary

Team-authored bug still present on main: when Python's Activate.ps1 runs via a terminal command (not shellStartup), the shell integration prompt function isn't updated to reflect the new prompt, causing exit code reporting to break. Keeping open per team-member rule.

## Change summary

Running Python's Activate.ps1 as a command in PowerShell corrupts the shell integration exit code tracking because the prompt function isn't re-captured.

## Best solution

Update shellIntegration.ps1 so that the VS Code prompt wrapper re-captures `OriginalPrompt` when it detects the prompt function has changed (e.g. after a Python venv activation via command). The fix is localized to the prompt function in shellIntegration.ps1 around line 84.
