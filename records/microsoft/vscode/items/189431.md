# Issue #189431 — Persistent background terminals (`hideFromUser = true`, `isTransient = false`) are killed on window reload

[https://github.com/microsoft/vscode/issues/189431](https://github.com/microsoft/vscode/issues/189431)

## Summary

Fixed on main: commit d0806c44a8c added `forcePersist` to `IShellLaunchConfig` and persists/revives background terminals marked with it across window reloads, shipping in 1.106.0.

## Change summary

Background terminals with `hideFromUser = true` and `isTransient = false` are killed 60 seconds after window reload because they are never reattached.

## Best solution

The fix (d0806c44a8c) introduces `forcePersist` on `IShellLaunchConfig` — extensions set this to `true` to ensure background terminals survive reloads. The layout state now saves and restores `forcePersist` background terminal IDs. Extensions should migrate from relying on `isTransient = false` alone to explicitly setting `forcePersist = true`.
