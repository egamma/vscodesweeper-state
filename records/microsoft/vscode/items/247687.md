# Issue #247687 — Some labels are backwards during a rebase merge conflict

[https://github.com/microsoft/vscode/issues/247687](https://github.com/microsoft/vscode/issues/247687)

## Summary

Still present on main: merge editor commands and accessibility labels hardcode 'Incoming' for left and 'Current' for right, which is backwards during a rebase. Team-authored issue; keeping open per policy.

## Change summary

During a rebase merge conflict the merge editor's screen-reader labels and 'Accept All' button tooltips say 'Incoming' and 'Current' on the wrong sides.

## Best solution

Replace the hardcoded 'Incoming'/'Current' strings in command titles and accessibility help with the actual input labels from the merge editor model (e.g. `input1.title` / `input2.title`), so they reflect whatever the consumer (git rebase, stash, etc.) provides. The fix touches `commands.ts` and `mergeEditorAccessibilityHelp.ts` in the merge-editor contribution.
