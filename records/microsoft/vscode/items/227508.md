# Issue #227508 — Tooltips Missing Name and Role Information

[https://github.com/microsoft/vscode/issues/227508](https://github.com/microsoft/vscode/issues/227508)

## Summary

Still reproduces on main: the hover widget sets role="tooltip" on the tooltip container but never links the triggering element via aria-describedby, so screen readers cannot associate the tooltip with its trigger. This is a valid accessibility gap in the workbench hover infrastructure.

## Change summary

Workbench hover tooltips lack aria-describedby on trigger elements, making them invisible to screen readers.

## Best solution

Add aria-describedby linkage in the hover service/widget infrastructure (likely src/vs/platform/hover/browser/ and src/vs/base/browser/ui/hover/hoverWidget.ts) so that when a hover is shown, the triggering element's aria-describedby references the tooltip's ID. The hover widget already has role="tooltip" — what's missing is the reverse link from the trigger.
