# Issue #227508 — Tooltips Missing Name and Role Information

[https://github.com/microsoft/vscode/issues/227508](https://github.com/microsoft/vscode/issues/227508)

## Summary

Still open on main: the hover widget sets `role="tooltip"` on itself, but the triggering controls (e.g. status bar items) never set `aria-describedby` pointing to the tooltip's ID, so screen readers cannot associate the trigger with the tooltip content.

## Change summary

Status bar tooltips lack `aria-describedby` on the trigger element, making them invisible to screen readers.

## Best solution

Add `aria-describedby` linking the trigger element to the hover widget's container ID in the hover service (likely `src/vs/platform/hover/browser/updatableHoverWidget.ts` or `hoverWidget.ts`), so that when a hover is shown, the trigger element references it. This is a platform-level fix that would benefit all workbench hovers.
