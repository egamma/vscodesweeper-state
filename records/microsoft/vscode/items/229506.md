# Issue #229506 — SCM Graph - Render code backticks

[https://github.com/microsoft/vscode/issues/229506](https://github.com/microsoft/vscode/issues/229506)

## Summary

Still valid: the SCM graph renders commit subjects as plain text via IconLabel.setLabel() with no backtick/inline-code rendering. The partial markdown label support added for extension tree views (bold/italic/strikethrough) doesn't cover inline code spans and isn't wired into the SCM history renderer.

## Change summary

Render code backticks (inline code spans) in SCM graph commit message labels instead of showing them as raw backtick characters.

## Best solution

Extend IconLabel or the SCM history item renderer to parse and render inline backtick code spans in commit subjects. The rendering lives in `src/vs/workbench/contrib/scm/browser/scmHistoryViewPane.ts` (line ~496, `setLabel` call). This could build on the partial markdown label work from https://github.com/microsoft/vscode/commit/70c31a49093 but would need inline syntax support rather than whole-string-wrapping only.
