# Issue #300749 — Model picker should support fuzzy searching

[https://github.com/microsoft/vscode/issues/300749](https://github.com/microsoft/vscode/issues/300749)

## Summary

Still reproduces on main: the model picker's filter in ActionListWidget uses simple substring matching (`label.includes(filterLower)`), so non-contiguous queries like 'cl op' for 'Claude Opus' fail. Milestone is 'On Deck', so keeping open.

## Change summary

Model picker search should support fuzzy/word-boundary matching so that typing partial words like 'cl op' finds 'Claude Opus'.

## Best solution

Replace or augment the `matchesFilter` logic in `src/vs/platform/actionWidget/browser/actionList.ts` (line ~992) with fuzzy scoring (e.g. `matchesFuzzy` from `src/vs/base/common/filters.ts`) or space-separated token matching, so each whitespace-delimited token is matched independently against the label.
