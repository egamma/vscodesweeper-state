# Issue #207118 — Disable Loop for Next Change

[https://github.com/microsoft/vscode/issues/207118](https://github.com/microsoft/vscode/issues/207118)

## Summary

Valid feature request: the diff editor's next/previous change navigation always wraps around (cycles) and there is no setting to disable this. The behavior is confirmed in current main — no option has been added.

## Change summary

Add a setting to disable the wrap-around (looping) behavior when navigating between changes in the diff editor.

## Best solution

Add a `diffEditor.navigation.loop` (or similar) boolean setting, defaulting to `true` for backward compatibility, and guard the fallback-to-first/last logic in `goToDiff()` in `src/vs/editor/browser/widget/diffEditor/diffEditorWidget.ts` (lines 649/651 and 654). This parallels the existing `editor.find.loop` setting for Find.
