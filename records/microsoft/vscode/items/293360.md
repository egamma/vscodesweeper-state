# Issue #293360 — Activity Bar hover tooltip persists after VS Code loses window focus

[https://github.com/microsoft/vscode/issues/293360](https://github.com/microsoft/vscode/issues/293360)

## Summary

Confirmed bug with an open PR (#305676) that adds a document-level mouseleave listener to dismiss hovers when the cursor exits the window. Keeping open pending PR review/merge.

## Change summary

Activity Bar hover tooltip persists when the mouse quickly moves from VS Code to an already-focused adjacent window, because no mouseleave event fires on the target element when the cursor exits the window boundary directly.

## Best solution

Review and land https://github.com/microsoft/vscode/pull/305676, which adds a document-level `mouseleave` handler in HoverService to dismiss active hovers when the cursor leaves the window — a small, localized fix in `src/vs/platform/hover/browser/hoverService.ts`.
