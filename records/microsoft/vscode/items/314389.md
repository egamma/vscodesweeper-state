# Issue #314389 — Open Worktree opens repo root, ignoring current subfolder

[https://github.com/microsoft/vscode/issues/314389](https://github.com/microsoft/vscode/issues/314389)

## Summary

Valid feature request: the Open Worktree commands always open at repository.root, ignoring any subfolder offset the user currently has open. The code path is clear and the request is well-motivated for monorepo workflows.

## Change summary

When opening a worktree, VS Code should preserve the current subfolder path relative to the repo root, rather than always opening the worktree's root.

## Best solution

In `extensions/git/src/commands.ts`, the `openWorktreeInCurrentWindow` and `openWorktreeInNewWindow` methods should compute the relative path from the current repo root to the current workspace folder, then append it to the target worktree root (falling back to the worktree root if the subfolder doesn't exist in the target).
