# Issue #325545 — Terminal Environment popup is clipped when content exceeds available height and action buttons become inaccessible

[https://github.com/microsoft/vscode/issues/325545](https://github.com/microsoft/vscode/issues/325545)

## Summary

Confirmed via source: the hover widget's action buttons (status bar) are appended outside the DomScrollableElement but inside the containerDomNode that receives maxHeight + overflow:hidden — when content is tall enough, the actions get clipped. The bug is real and still present on main.

## Change summary

Terminal environment hover actions (e.g. "Relaunch Terminal") become inaccessible when many extensions contribute environment variables, because the hover's status bar is clipped by overflow:hidden on the max-height-constrained container.

## Best solution

Fix the hover widget layout so that the status-bar row is always visible: either make `.monaco-hover` a flex column so the scrollable content shrinks to fit while the status bar retains its natural height, or set the maxHeight on the scrollable content area instead of the container. The fix lives in `src/vs/platform/hover/browser/hoverWidget.ts` (adjustHoverMaxHeight) and possibly `src/vs/base/browser/ui/hover/hoverWidget.css`. This affects all workbench hovers with actions, not just the terminal tooltip.

## Evidence

- **status bar outside scrollbar:** In src/vs/platform/hover/browser/hoverWidget.ts line 233, the statusBarElement is appended to this._hover.containerDomNode, which is outside the DomScrollableElement that wraps contentsDomNode (src/vs/base/browser/ui/hover/hoverWidget.ts lines 41-44).
- **maxHeight on container:** adjustHoverMaxHeight (line 604) sets maxHeight on containerDomNode, and hoverWidget.css line 9 sets overflow:hidden on .monaco-hover — this clips the status bar when content exceeds the budget.
- **no flex layout:** The containerDomNode has no flex layout, so the status bar doesn't get reserved space before the scrollable area fills the maxHeight.
- **details hidden by default mitigates:** Commit ca677df9f5b (March 2025, shipped ~1.99) hid terminal hover details by default, which reduces the content size. But toggling 'Show Details' or having enough extensions still triggers the bug.
