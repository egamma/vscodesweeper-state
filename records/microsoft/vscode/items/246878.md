# Issue #246878 — Accessible View for Copilot Agent: show file changes as one-liner instead of entire diff

[https://github.com/microsoft/vscode/issues/246878](https://github.com/microsoft/vscode/issues/246878)

## Summary

Still reproduces on main: the chat accessible view renders full code blocks from markdown content parts verbatim (line 297 of chatResponseAccessibleView.ts uses `includeCodeBlocksFences: true`), rather than the compact one-liner summary sighted users see for file changes.

## Change summary

The Accessible View for Copilot chat responses shows entire code diffs instead of the compact file-change summaries (e.g. "app.py +126, -1") that sighted users see.

## Best solution

In `chatResponseAccessibleView.ts`, detect code blocks that represent file edits (they have a `// filepath:` comment pattern) within `markdownContent` parts and render a condensed summary (filename + line counts) instead of the full diff. Alternatively, add a verbosity toggle setting to switch between summary and full content.
