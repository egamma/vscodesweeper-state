## Summary

Valid bug still open on main. The related fix PR (#312515) is still unmerged. The issue correctly identifies that detect-indentation overrides a manual tab-size change via the command palette.

## Change summary

After using 'Change Tab Display Size' to set tab size to 1, detect indentation overrides it back to the detected value (e.g. 4), making the manual setting ineffective.

## Best solution

Merge or iterate on the open PR https://github.com/microsoft/vscode/pull/312515 which keeps indentSize tied to tabSize after detect-indentation runs, so a user's explicit command-palette override is respected.
