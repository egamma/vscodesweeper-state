# Issue #191700 — Ask GitHub Copilot item updates slowly, making the command palette feel sluggish

[https://github.com/microsoft/vscode/issues/191700](https://github.com/microsoft/vscode/issues/191700)

## Summary

Still reproducible on main: the "Ask in Chat" item's label includes the current filter text but is rendered only after a 200ms debounce inside `getAdditionalCommandPicks`, so it visibly lags behind typing. The architecture hasn't changed.

## Change summary

The "Ask in Chat" command palette item updates its label with a visible delay after each keystroke, making the palette feel sluggish.

## Best solution

Render the "Ask in Chat" pick as a fast/static pick (outside the async `getAdditionalCommandPicks` path) so its label updates immediately with each keystroke, decoupled from the AI-related-info debounce. The fix lives in `src/vs/workbench/contrib/quickaccess/browser/commandsQuickAccess.ts`.
