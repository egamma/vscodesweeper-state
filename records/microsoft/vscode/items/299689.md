# Issue #299689 — High CPU after update: cpptools repeatedly retries LanguageModelProxy model resolution in Remote-WSL extension host

[https://github.com/microsoft/vscode/issues/299689](https://github.com/microsoft/vscode/issues/299689)

## Summary

Real bug confirmed by a team member (sean-mcmanus). An unmerged fix branch (aamunger/remoteModelResolution) addresses the infinite retry loop in `selectLanguageModels` → `getLanguageModelByIdentifier`, but it has not landed on main yet. Keep open until the fix merges.

## Change summary

Extensions calling `selectChatModels` trigger unbounded model-resolution retries in the extension host when models are unavailable, causing sustained high CPU.

## Best solution

Land the fix from branch `aamunger/remoteModelResolution` (commit 663b88e0e69) which adds a `resolveIfMissing` parameter to prevent re-resolution loops when `selectLanguageModels` results are already fresh. The core fix is in `src/vs/workbench/api/common/extHostLanguageModels.ts`.
