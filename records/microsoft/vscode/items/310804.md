# Issue #310804 — copilot mcp tools selection cross session

[https://github.com/microsoft/vscode/issues/310804](https://github.com/microsoft/vscode/issues/310804)

## Summary

Valid feature request: MCP tool selection is stored globally (StorageScope.PROFILE) by default and shared across sessions using the same mode. The code has session-scoped overrides keyed by mode ID, but not per chat thread, so changes in one session affect others.

## Change summary

MCP tool selection changes in one chat session affect other concurrent sessions because tool enablement defaults to global/profile-level storage.

## Best solution

Make tool selection per-session (per chat thread) by default, or clearly scope session-only changes to the individual chat session rather than the mode. The `_sessionStates` in `chatSelectedTools.ts` is keyed by mode ID — keying by session/thread ID would isolate sessions from each other.
