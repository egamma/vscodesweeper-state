# Issue #283155 — Opening a private repo should not open the auth modal directly until I use chat

[https://github.com/microsoft/vscode/issues/283155](https://github.com/microsoft/vscode/issues/283155)

## Summary

Team-authored issue (MEMBER) describing a UX polish bug where opening a private repo triggers an auth modal before the user interacts with chat. Hard keep-open rule applies.

## Change summary

Opening a private GitHub repo shows an auth modal immediately rather than deferring it until the user actually uses chat.

## Best solution

Defer the auth modal until the user initiates a chat interaction that requires the additional scopes, rather than prompting eagerly when a private repo is opened via Settings Sync or Cloud Changes.
