# Issue #283155 — Opening a private repo should not open the auth modal directly until I use chat

[https://github.com/microsoft/vscode/issues/283155](https://github.com/microsoft/vscode/issues/283155)

## Summary

Team-authored issue (MEMBER) — hard keep-open rule applies. The issue describes a narrow remaining case where a modal auth dialog appears prematurely when opening a private repo after signing in via Settings Sync/Cloud Changes without the full required scopes.

## Change summary

Opening a private GitHub repo after authenticating via Settings Sync (without repo scopes) shows an intrusive auth modal before the user interacts with chat.

## Best solution

Defer the auth modal until the user actually invokes a Copilot Chat interaction that requires the additional scopes, rather than prompting eagerly on repo open.
