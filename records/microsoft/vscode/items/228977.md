# Issue #228977 — [themes] support sematic colorization for 'excludedCode'

[https://github.com/microsoft/vscode/issues/228977](https://github.com/microsoft/vscode/issues/228977)

## Summary

Valid feature request: default themes lack a semanticTokenColors entry for the 'excludedCode' token type, so inactive preprocessor code is not visually dimmed. Still unaddressed on main.

## Change summary

C# code inside inactive preprocessor directives is not dimmed in default themes because they have no semantic token rule for 'excludedCode'.

## Best solution

Add a dimmed/faded foreground color rule for the `excludedCode` semantic token type in the default themes (dark_plus.json, light_plus.json, dark_vs.json, light_vs.json, hc_black.json) under their `semanticTokenColors` sections, e.g. `"excludedCode": { "foreground": "#808080" }`.
