# Issue #228977 — [themes] support sematic colorization for 'excludedCode'

[https://github.com/microsoft/vscode/issues/228977](https://github.com/microsoft/vscode/issues/228977)

## Summary

Valid feature request — no default theme includes a semanticTokenColors rule for `excludedCode`, so inactive preprocessor regions are not dimmed. The gap still exists on main. Low traction (0 upvotes, 652 days), but correctly classified and in Backlog.

## Change summary

Default themes should dim code inside inactive C# preprocessor directives via a semantic token color rule for `excludedCode`.

## Best solution

Add a `"excludedCode": { "foreground": "#..." }` entry (with reduced opacity or a muted color) to `semanticTokenColors` in the default themes (dark_modern.json, light_modern.json, dark_plus.json, light_plus.json). The C# language server already emits this semantic token type; the themes just need to style it.
