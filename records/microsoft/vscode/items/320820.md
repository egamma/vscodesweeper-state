# Issue #320820 — Voice input should mute mic routing to other apps (e.g. Teams) while active

[https://github.com/microsoft/vscode/issues/320820](https://github.com/microsoft/vscode/issues/320820)

## Summary

Exclusive microphone access that blocks other applications is not possible through web/Electron audio APIs — this is an OS-level audio routing concern outside VS Code's control. The feature belongs to the operating system's audio subsystem or a dedicated audio routing tool.

## Change summary

Request VS Code to take exclusive microphone access while Copilot voice is active, blocking other apps from receiving audio input.

## Best solution

This is not achievable at the application level on any major OS (Windows, macOS, Linux) through standard APIs available to Electron/web apps. Users should use their OS audio routing tools or manually mute in their calling app. The request is out of scope for VS Code core.

## Vision fit

- **Fit:** rejected
- **Reason:** OS-level audio routing is outside VS Code's application boundary; the product values a lightweight editor that delegates platform concerns to the OS.
- **Evidence:** Values: "Out of the box, VS Code is a text editor: lightweight and fast." Exclusive mic control is an OS audio subsystem responsibility, not an editor feature.

## Auto-fix candidate

- **Rationale:** OS-level audio routing is not achievable from within an Electron app — no code change in VS Code can deliver this.

## Evidence

- **no exclusive mic API:** Neither the Web Audio API, getUserMedia, nor Electron provide an 'exclusive microphone' mode that prevents other OS applications from accessing the mic simultaneously. This is an OS audio mixer concern.
- **speech implementation:** VS Code's speech contrib (src/vs/workbench/contrib/speech/) delegates to extension-provided speech services; no mic-muting or exclusive-access logic exists or could work at this layer.
- **OS limitation:** On Windows (WASAPI shared mode), macOS (CoreAudio), and Linux (PulseAudio/PipeWire), multiple applications share mic input by default; exclusive mode requires OS-level or driver-level intervention unavailable to Electron apps.
