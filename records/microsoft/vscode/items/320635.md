# Issue #320635 — Claude Sonnet 4.x Request Failed: 400 Tool names must be unique (bug, workaround)

[https://github.com/microsoft/vscode/issues/320635](https://github.com/microsoft/vscode/issues/320635)

## Summary

Valid bug: when an external MCP server exposes a tool whose final API name collides with a built-in Copilot tool (e.g. `memory`), VS Code sends duplicate tool names to the Anthropic API, which rejects the request with a 400 error. The reporter found the workaround (remove the external MCP server) but the underlying lack of deduplication or clear error messaging is a real gap. Issue https://github.com/microsoft/vscode/issues/316983 reports the same 'Tool names must be unique' error with a different trigger (VSCode tool enabled); both are untriaged and could be consolidated.

## Change summary

Claude Sonnet requests fail with 400 'Tool names must be unique' when an external MCP Memory Agent is installed alongside the built-in Copilot memory tool.

## Best solution

Deduplicate or conflict-detect tool names before sending them to the language model API — either by skipping/warning about the conflicting MCP tool or by surfacing a clear diagnostic. Consider consolidating with https://github.com/microsoft/vscode/issues/316983 which reports the same API error from a different trigger. The fix likely lives in the Copilot extension's tool assembly path (`extensions/copilot/src/extension/tools/`) and/or the MCP tool name prefixing logic in `src/vs/workbench/contrib/mcp/common/mcpServer.ts`.
