# Issue #40338 — [folding] configure initial collapse state

[https://github.com/microsoft/vscode/issues/40338](https://github.com/microsoft/vscode/issues/40338)

## Summary

Partially addressed (imports via `editor.foldingImportsByDefault`) but the broader request — auto-collapsing regions, comments, and other folding kinds on file open — remains unimplemented on current main. Still a valid feature request with community interest.

## Change summary

Allow configuring initial collapse state per folding-range kind (regions, comments, imports, custom) when a file is opened.

## Best solution

Generalize `editor.foldingImportsByDefault` into a per-kind setting (e.g. `editor.folding.initialState` accepting an object like `{"imports": true, "comment": true, "region": true}`) in `src/vs/editor/contrib/folding/browser/folding.ts`, leveraging the existing `FoldingRangeKind` infrastructure and `setCollapsedAllOfType` helper.
