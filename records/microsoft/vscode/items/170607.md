# Issue #170607 — set decoration's opacity is not working

[https://github.com/microsoft/vscode/issues/170607](https://github.com/microsoft/vscode/issues/170607)

## Summary

Keep open and route to the editor area: current main still has a plausible interaction where unused-code opacity is applied at the inline decoration level, which can dim extension decoration content rendered on that same text.

## Change summary

Extension-created editor decorations with their own opacity should not be dimmed by the editor's unused-code fading.

## Best solution

The editor team should decide whether decoration before/after content should be isolated from `MarkerTag.Unnecessary` fading, likely around marker inline decoration rendering and decoration CSS generation in `src/vs/editor/common/services/markerDecorationsService.ts`, `src/vs/editor/browser/widget/codeEditor/codeEditorWidget.ts`, and `src/vs/editor/browser/services/abstractCodeEditorService.ts`.
