# Issue #192309 — Support a special view in the quick pick optimized for displaying icons

[https://github.com/microsoft/vscode/issues/192309](https://github.com/microsoft/vscode/issues/192309)

## Summary

Legitimate feature request for a grid-based icon picker mode in the quick pick, still unimplemented on main. Team members have discussed design approaches and a member checked on status as recently as Jan 2026. Keeping open in Backlog.

## Change summary

Add a special grid/icon view mode to the quick pick for visually browsing and selecting icons (e.g. for terminal profiles, user profiles).

## Best solution

Implement a grid layout mode in the quick pick (src/vs/platform/quickinput/) that renders items as icons in a grid when no filter text is entered, reverting to list view on typing. This could later be exposed as extension API. The UX sync consensus was to revert to list view when filtering.
