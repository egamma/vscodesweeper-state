# Issue #245145 — Nonsensical "forbidden" cursor when dragging tabs to valid place

[https://github.com/microsoft/vscode/issues/245145](https://github.com/microsoft/vscode/issues/245145)

## Summary

Still reproduces on main: the dragover handler in multiEditorTabsControl.ts never sets `dropEffect = 'move'` for the local-editor-transfer case, so the browser falls back to a platform-default cursor (often 'forbidden' on Linux). An open PR (#296448) is in progress to fix this.

## Change summary

Dragging a tab within the tab bar shows a 'forbidden' cursor even though the drop is valid.

## Best solution

Land or iterate on the open PR https://github.com/microsoft/vscode/pull/296448 which adds proper `dropEffect = 'move'` feedback for local editor tab drags.
