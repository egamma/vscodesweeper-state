# Issue #245145 — Nonsensical "forbidden" cursor when dragging tabs to valid place

[https://github.com/microsoft/vscode/issues/245145](https://github.com/microsoft/vscode/issues/245145)

## Summary

Still reproduces on main: the dragover handler in multiEditorTabsControl.ts only sets dropEffect when hovering over the tabs container background, not over individual tabs — so the browser shows a forbidden cursor. An open PR (#296448) is addressing this.

## Change summary

Dragging editor tabs shows a 'forbidden' cursor icon even over valid drop targets where dropping works.

## Best solution

Land the open PR https://github.com/microsoft/vscode/pull/296448 which fixes the drag cursor feedback within editor tabs, or ensure the dragover handler sets `dropEffect` appropriately when hovering over individual tab elements.
