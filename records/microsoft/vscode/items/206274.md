# Issue #206274 — `explorer.upload` fails when a file is selected

[https://github.com/microsoft/vscode/issues/206274](https://github.com/microsoft/vscode/issues/206274)

## Summary

Team-authored issue, still reproducing as of April 2025. The upload handler uses the selected explorer item as the target directory without checking whether it's a file — when a file is selected, it incorrectly treats the file path as a folder. Keeping open per team-author policy.

## Change summary

The `explorer.upload` command fails with an error when a file (rather than a folder) is selected in the explorer, because it uses the selected file's path as the upload destination directory.

## Best solution

In `src/vs/workbench/contrib/files/browser/fileActions.ts` line 1103, when the selected element is a file (not a directory), resolve to its parent folder instead. A guard like `element.isDirectory ? element : element.parent` would fix this.
