# Issue #314622 — Introduce multi-tier difficulty labels to improve newcomer onboarding

[https://github.com/microsoft/vscode/issues/314622](https://github.com/microsoft/vscode/issues/314622)

## Summary

VS Code already uses both `good first issue` and `help wanted` labels with active issues tagged under them. The additional `intermediate` and `advanced` tiers are a triage-process proposal that the team has not adopted — label conventions are an internal maintainer decision, not an actionable product issue.

## Change summary

Introduce `intermediate` and `advanced` difficulty labels alongside the existing `good first issue` and `help wanted` labels to help newcomers find tasks matching their skill level.

## Best solution

The existing `good first issue` label already serves the newcomer onboarding purpose described in the proposal, and VS Code's triage conventions are maintained by the core team. Adding intermediate/advanced tiers is a process decision that doesn't belong as a GitHub issue — if the team wanted finer granularity, they would adopt it internally. Close as out-of-scope.
