# Issue #206864 — Moving cursor on the same word when hovering on package.json dependencies reloads hover

[https://github.com/microsoft/vscode/issues/206864](https://github.com/microsoft/vscode/issues/206864)

## Summary

Still reproduces on main: hover is recomputed on every character move within the same word because the anchor equality check fails per-character, even though the hover provider's range covers the entire word. The team acknowledged this needs a caching mechanism.

## Change summary

Moving the mouse within a word in package.json dependencies reloads the hover on every character, causing unnecessary npm process spawns and resource usage.

## Best solution

Add logic in `_startShowingOrUpdateHover` (contentHoverWidgetWrapper.ts) to skip restarting the hover operation when the new anchor falls within the range of an existing valid hover result — i.e., when `filter()` returns the same parts, treat the existing result as sufficient and don't call `_startHoverOperationIfNecessary`.
