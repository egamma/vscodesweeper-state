# Issue #291067 — Auto-approval information not viewable with screenreader

[https://github.com/microsoft/vscode/issues/291067](https://github.com/microsoft/vscode/issues/291067)

## Summary

Team-authored accessibility bug, still present on main: the auto-approval reason is rendered only as a hover tooltip (via IHoverService) and is not exposed to screen readers through ARIA labels or other accessible mechanisms.

## Change summary

Screen readers cannot read the auto-approval reason for tool invocations — the information is only available via mouse hover tooltip.

## Best solution

Include the hover message content in the aria-label of the collapse button element in chatCollapsibleContentPart.ts so screen readers announce the approval reason alongside the tool name.
