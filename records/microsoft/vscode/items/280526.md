# Issue #280526 — Todo response is sometimes just left as "Starting"

[https://github.com/microsoft/vscode/issues/280526](https://github.com/microsoft/vscode/issues/280526)

## Summary

Still an open bug with no fix committed. Team members discussed the root cause (tool call grouping/pinning logic in the chat list renderer) and a potential approach (allowing todos to opt out of pinned tool call groupings), but no fix has landed on main.

## Change summary

The todo tool response sometimes gets stuck showing "Starting" instead of its proper description like "Managed and tracked todo items for planning".

## Best solution

Implement the approach discussed in the comments: allow the todo/manageTodoList tool to opt out of the pinned tool-call grouping logic in `shouldPinPart()` in `src/vs/workbench/contrib/chat/browser/widget/chatListRenderer.ts`, so it renders its own progress label instead of being collapsed into a thinking container showing only "Starting".
