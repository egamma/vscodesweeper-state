# Issue #194573 — Search Files By Name Should Use Highlighted Text

[https://github.com/microsoft/vscode/issues/194573](https://github.com/microsoft/vscode/issues/194573)

## Summary

Valid feature request still unimplemented on main. The Go to File quick open does not pre-fill with editor selection text, while text search quick access already does. Two community PRs were closed without merge; the feature remains a gap.

## Change summary

Pre-fill the Go to File (Ctrl+P) quick picker with the currently selected/highlighted text from the active editor.

## Best solution

Implement selection seeding in `src/vs/workbench/browser/actions/quickAccessActions.ts` for the `workbench.action.quickOpen` action, following the pattern already used in `src/vs/workbench/contrib/search/browser/searchActionsTextQuickAccess.ts` (which calls `getSelectionTextFromEditor`). The layering concern from the earlier PR attempts (importing from `contrib/search`) can be resolved by extracting the helper to a shared location in `vs/workbench/browser` or `vs/platform`.
