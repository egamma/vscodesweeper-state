# Issue #288094 — When Copilot Chat Errors, it should RETRY instead of haltung

[https://github.com/microsoft/vscode/issues/288094](https://github.com/microsoft/vscode/issues/288094)

## Summary

The requested auto-retry behavior for agentic coding scenarios was implemented in March 2026 as part of the autopilot/autoApprove mode. In these modes, transient model errors are now automatically retried up to 3 times without stopping the flow — exactly the behavior this issue asks for.

## Change summary

When Copilot Chat encounters a model error during agentic coding, it should automatically retry instead of halting and asking the user to click retry.

## Best solution

Keep the shipped auto-retry behavior (up to 3 retries in autoApprove/autopilot modes, in `extensions/copilot/src/extension/intents/node/toolCallingLoop.ts`). The follow-up request to make the retry count configurable is already tracked by https://github.com/microsoft/vscode/issues/313485.
