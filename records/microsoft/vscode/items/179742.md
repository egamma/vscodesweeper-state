# Issue #179742 — Provide a global context menu to auto expand lazy variables

[https://github.com/microsoft/vscode/issues/179742](https://github.com/microsoft/vscode/issues/179742)

## Summary

Team-authored feature request for a global debug context menu to toggle auto-expand lazy variables. Internal discussion showed disagreement on whether a context menu should control a setting; keeping open for maintainer judgment.

## Change summary

Add a global context menu in the debug Variables view to toggle auto-expansion of lazy variables, improving discoverability of the existing `debug.autoExpandLazyVariables` setting.

## Best solution

The team needs to resolve the design question raised by roblourens — whether a context menu should toggle a setting (and at what scope). If the approach is accepted, implementation would live in `src/vs/workbench/contrib/debug/` (variables view context menu). Alternatively, improving discoverability of the existing setting through other UI affordances (e.g. a toolbar toggle or inline hint) could address the underlying problem without tying a context menu to a setting write.
