# Issue #185565 — Accessibility: Cannot turn off audio cues on a language level

[https://github.com/microsoft/vscode/issues/185565](https://github.com/microsoft/vscode/issues/185565)

## Summary

Still unimplemented: accessibility signal settings (formerly audio cues) are not registered with language-overridable scope, so per-language disabling is not possible. Low community traction (0 upvotes) but a valid accessibility feature request in the Backlog.

## Change summary

Allow accessibility signal settings (e.g. lineHasError) to be overridden on a per-language basis.

## Best solution

Register accessibility signal configuration properties with `LANGUAGE_OVERRIDABLE` scope in the signal service, and plumb the active editor's language into the signal evaluation path so that `[markdown]` overrides take effect. The likely files are `src/vs/platform/accessibilitySignal/browser/accessibilitySignalService.ts` and the configuration registration therein.
