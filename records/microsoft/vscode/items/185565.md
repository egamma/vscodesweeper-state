# Issue #185565 — Accessibility: Cannot turn off audio cues on a language level

[https://github.com/microsoft/vscode/issues/185565](https://github.com/microsoft/vscode/issues/185565)

## Summary

Valid feature request: accessibility signal settings are not registered with language-overridable scope, so per-language overrides (e.g., disabling lineHasError for markdown) have never worked. Still unimplemented on current main.

## Change summary

Allow accessibility signal settings (sounds/announcements) to be overridden on a per-language basis via language-scoped settings.

## Best solution

Register accessibility signal configuration properties with `ConfigurationScope.LANGUAGE_OVERRIDABLE` in `src/vs/workbench/contrib/accessibility/browser/accessibilityConfiguration.ts`, and update the signal service to read the setting with language-override-aware APIs (passing the active editor's language ID). This is a moderate change touching configuration registration and the signal enablement check.
