# Issue #41206 — [html] code completion replaces text after cursor

[https://github.com/microsoft/vscode/issues/41206](https://github.com/microsoft/vscode/issues/41206)

## Summary

Keep open as an old HTML completion behavior request rather than closing as a bug: a maintainer clarified that replacing the word after the cursor was intentional design, and the issue is already classified as a Backlog feature request.

## Change summary

HTML tag completion currently replaces following text after the cursor when accepting a suggestion before existing content, and the request is to avoid consuming that text in the new-tag case.

## Best solution

HTML completions should either keep the current replace-range behavior for editing existing tags or add a more precise insert/replace range distinction so accepting a new tag before text does not consume the following word. This belongs with the HTML language features completion provider.
