# Issue #41206 — [html] code completion replaces text after cursor

[https://github.com/microsoft/vscode/issues/41206](https://github.com/microsoft/vscode/issues/41206)

## Summary

A maintainer (aeschli) already confirmed in 2018 that the behavior — replacing the word after the cursor on completion — is intentional and commonly desired when modifying existing tags. The workaround is to add a space first. This has sat untouched for over 6 years with no community traction.

## Change summary

HTML completion replaces the word following the cursor when expanding a tag via autocomplete, consuming adjacent text.

## Best solution

Close as designed. The replace-word-after-cursor behavior is intentional for the common case of editing existing tag names. The workaround (insert a space before the adjacent text) is sufficient.
