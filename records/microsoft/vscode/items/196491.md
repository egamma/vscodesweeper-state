# Issue #196491 — [wrapped lines] inline chat zone shows on the wrong line

[https://github.com/microsoft/vscode/issues/196491](https://github.com/microsoft/vscode/issues/196491)

## Summary

Keep open: the report still maps to current inline chat and editor view-zone behavior, and there is no concrete fixing commit to justify closing it.

## Change summary

Inline chat opens its zone in the middle of a wrapped previous line when invoked from an empty line below wrapped text.

## Best solution

Fix the placement contract between inline chat and editor view zones: either inline chat should anchor the zone after the previous line's visual end for this empty-line case, or the editor view-zone API should provide a clear model/view-line placement mode for wrapped lines.
