# Issue #196491 — [wrapped lines] inline chat zone shows on the wrong line

[https://github.com/microsoft/vscode/issues/196491](https://github.com/microsoft/vscode/issues/196491)

## Summary

Still present on main: the ZoneWidget base class always sets `afterColumn` from the position column, causing view zones to appear mid-wrap instead of after the full wrapped line. This is a broader editor API issue with word-wrap and view zones, not just inline chat.

## Change summary

Inline chat zone widget appears in the middle of a wrapped line instead of below it when word wrap is enabled.

## Best solution

Fix `_computeWhitespaceProps` in `src/vs/editor/browser/viewParts/viewZones/viewZones.ts` or the `ZoneWidget._showImpl` logic in `src/vs/editor/contrib/zoneWidget/browser/zoneWidget.ts` to ensure that when a zone is placed after a line, it appears after the *last* view line of the wrap, not at the specified column's view line. Alternatively, inline chat could pass max-column when computing `initialPosition`.

- **initialPosition computation:** src/vs/workbench/contrib/inlineChat/browser/inlineChatSessionServiceImpl.ts line 133: `initialPosition: editor.getSelection().getStartPosition().delta(-1)` — preserves the original column (often 1) when moving one line up, so the zone lands at column 1 of the wrapped line above.
