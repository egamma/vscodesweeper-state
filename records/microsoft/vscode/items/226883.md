# Issue #226883 — Order of Compare Selected files by mouse

[https://github.com/microsoft/vscode/issues/226883](https://github.com/microsoft/vscode/issues/226883)

## Summary

Still valid on main: the 'Compare Selected' command places the right-clicked file on the left in Open Editors and Tabs views, but uses selection order (first-selected on the left) in the Folders explorer. The inconsistency is confirmed in the source and has not been addressed.

## Change summary

The ordering of files in the 'Compare Selected' diff editor is inconsistent between the Folders explorer (selection-order) and the Open Editors/Tabs views (right-click-target on left).

## Best solution

Unify the ordering logic in `getMultiSelectedResources` (src/vs/workbench/contrib/files/browser/files.ts) so that all three views use the same convention — either always selection-order or always right-click-target-first. The Explorer path (line 124) should apply the same 'move main resource to front' reordering already used by the Open Editors and Tabs branches.
