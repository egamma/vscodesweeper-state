# Issue #226883 — Order of Compare Selected files by mouse

[https://github.com/microsoft/vscode/issues/226883](https://github.com/microsoft/vscode/issues/226883)

## Summary

Still reproduces on main: the ordering of files in the Compare Selected diff editor is inconsistent between the Explorer folders view (selection order) and the Open Editors / tabs views (right-clicked element always goes left). Two other issues were closed as duplicates of this one, making it the canonical tracker. Properly labeled and in the Backlog — keep open.

## Change summary

Compare Selected command places files in inconsistent left/right order depending on whether the command is invoked from the Explorer folders view, Open Editors, or tabs.

## Best solution

Unify the ordering logic in `getMultiSelectedResources` (src/vs/workbench/contrib/files/browser/files.ts) so all three surfaces use the same convention for which file goes on the left of the diff editor — either always selection-order or always right-click-target-first.
