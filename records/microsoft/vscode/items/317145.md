# Issue #317145 — Quick access renders icon syntax from arbitrary sources

[https://github.com/microsoft/vscode/issues/317145](https://github.com/microsoft/vscode/issues/317145)

## Summary

Still reproduces on main: the quick pick list renders `IconLabel` with `supportIcons: true`, so any `$(...)` syntax in file names or editor titles from untrusted sources (e.g. HTML page titles, untitled file content) renders as codicons instead of literal text. No fix has been committed.

## Change summary

Quick access (Ctrl+P / Go to File) renders codicon syntax like `$(zap)` from arbitrary file/editor names as actual icons instead of showing the literal text.

## Best solution

The quick pick item rendering in `src/vs/platform/quickinput/browser/quickInputList.ts` uses `IconLabel` with `supportIcons: true` for all labels. Items whose labels come from untrusted sources (editor titles, file names) should escape codicon syntax before being set as the pick label, or the rendering path should distinguish trusted labels (commands with intentional icons) from untrusted ones (file/editor names). The fix likely belongs in `src/vs/workbench/contrib/search/browser/anythingQuickAccess.ts` where editor history picks are constructed, escaping codicon patterns in the label.
