# Issue #317832 — Agents: Don't attempt to start a terminal when opening a Done/worktree session

[https://github.com/microsoft/vscode/issues/317832](https://github.com/microsoft/vscode/issues/317832)

## Summary

Team-authored bug still present on main: `_onActiveSessionChanged` in the terminal contribution does not check whether the session is archived before calling `ensureTerminal`, causing a failed terminal launch dialog when the worktree directory no longer exists.

## Change summary

Opening a Done/archived worktree session triggers a terminal launch error because the deleted worktree CWD no longer exists.

## Best solution

Add an early return in `_onActiveSessionChanged` (and any other callers of `ensureTerminal`) when `session.isArchived.get()` is true and the session uses a worktree workspace, short-circuiting the terminal creation path. The fix belongs in `src/vs/sessions/contrib/terminal/browser/sessionsTerminalContribution.ts`.
