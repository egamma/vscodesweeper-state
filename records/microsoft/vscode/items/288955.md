# Issue #288955 — [BUG]: Virtual Keyboard Issue When Scrolling On Editor

[https://github.com/microsoft/vscode/issues/288955](https://github.com/microsoft/vscode/issues/288955)

## Summary

Still reproduces on main: the touch gesture handler in touch.ts dispatches a Tap event based solely on a 30px movement threshold, so small scroll gestures on Android trigger a tap → focus → virtual keyboard. PR #292179 attempted a fix but was closed without merging. No fix has landed on main.

## Change summary

Scrolling on the editor in vscode.dev on Android incorrectly triggers the virtual keyboard because small scroll gestures fire a Tap event.

## Best solution

Guard the Tap dispatch in `src/vs/base/browser/touch.ts` to suppress it when a Change (scroll) event has already fired during the same touch gesture, so that scroll gestures don't trigger focus and the virtual keyboard on mobile. The closed PR #292179 explored this approach and could inform a fresh attempt.
