# Issue #251076 — File protocol links in editor do not open in WSL

[https://github.com/microsoft/vscode/issues/251076](https://github.com/microsoft/vscode/issues/251076)

## Summary

Still present on main: the editor link detection passes file:// URIs directly to the opener service without translating them for the remote (WSL) context, unlike the terminal which has dedicated remote-path resolution logic.

## Change summary

Clicking file:// protocol links in editor content while connected to WSL attempts to open the path on the Windows host rather than in the remote filesystem.

## Best solution

The editor link opener in `src/vs/editor/contrib/links/browser/links.ts` should detect when the workspace is running in a remote context and rewrite `file://` URIs to `vscode-remote://` URIs before passing them to the opener service, similar to how the terminal link handling already does this.
