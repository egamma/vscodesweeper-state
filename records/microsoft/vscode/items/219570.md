# Issue #219570 — Git - git push hangs when pushing from remote

[https://github.com/microsoft/vscode/issues/219570](https://github.com/microsoft/vscode/issues/219570)

## Summary

Legitimate bug where git push hangs in the VS Code terminal when connected via Remote SSH to a Windows machine. The conversation narrowed it to a Remote SSH–specific scenario (works fine locally on Windows) but stalled without resolution. The askpass/SSH_ASKPASS environment variables VS Code injects into the terminal are the likely culprit, but no fix has been identified. Keeping open — the hard keep-open rule for Backlog items doesn't apply, but the bug is real and unresolved.

## Change summary

Git push hangs in VS Code's integrated terminal when connected to a remote Windows machine via SSH, while the same command works in a standalone PowerShell session on that machine.

## Best solution

Investigate the interaction between VS Code's SSH_ASKPASS/GIT_ASKPASS environment variables (set in `extensions/git/src/askpass.ts`) and the nested SSH session used by `git push`. The askpass IPC pipe may not be reachable or may block when the terminal is running on a remote host. A fix might involve detecting the remote context and adjusting or omitting the askpass env vars for terminal sessions. The Remote SSH extension team may need to be involved.
