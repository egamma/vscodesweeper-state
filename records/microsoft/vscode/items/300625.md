# Issue #300625 — Input/output confirmation cuts off text

[https://github.com/microsoft/vscode/issues/300625](https://github.com/microsoft/vscode/issues/300625)

## Summary

Still unfixed on main. The tool input/output confirmation widget truncates text without a proper multi-line ellipsis. The discussion identifies CSS `line-clamp` as the right fix, but it has not been applied to the confirmation widget CSS. Author is a team member, so keeping open per policy.

## Change summary

Tool input/output confirmation text is cut off abruptly without ellipsis when it exceeds the available height.

## Best solution

Apply CSS `line-clamp` (as discussed in the issue comments and demonstrated in `chatAgentFeedbackReviewConfirmation.css`) to the relevant truncated elements in the chat confirmation widget so overflow text shows a proper multi-line ellipsis instead of being cut off.
