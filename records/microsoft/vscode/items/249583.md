# Issue #249583 — Add an option to open the Timeline view from the file's context menu in the Explorer and/or from the Source Control panel. This would make it easier to discover and use the Timeline feature for viewing file history.

[https://github.com/microsoft/vscode/issues/249583](https://github.com/microsoft/vscode/issues/249583)

## Summary

The Explorer context menu part of this request already exists on main (the 'Open Timeline' action is registered in `MenuId.ExplorerContext`). The remaining ask — adding the same action to the Source Control panel's context menu — is a valid, small feature request that does not yet exist. Keeping open for area triage.

## Change summary

Request to add an 'Open Timeline' option to the Explorer context menu (already exists) and the Source Control panel context menu (does not exist yet).

## Best solution

The Explorer part is already shipped. For the SCM part, register the existing `files.openTimeline` command under `MenuId.SCMResourceContext` (and possibly `MenuId.SCMResourceGroupContext`) in `src/vs/workbench/contrib/timeline/browser/timeline.contribution.ts`, gated on `TimelineHasProviderContext`. This is a small, localized addition.
