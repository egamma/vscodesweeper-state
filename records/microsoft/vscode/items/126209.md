# Issue #126209 — Missed key presses and jumpy cursor in terminal when working remotely with high latency

[https://github.com/microsoft/vscode/issues/126209](https://github.com/microsoft/vscode/issues/126209)

## Summary

The immediate user pain was mitigated by turning local echo off by default (commit 5c5cb3dc1fc, shipped in 1.100.0), but the underlying reliability bug in the local echo/typeahead feature remains unfixed. This issue is the canonical tracker for making local echo actually work correctly under latency, so it should stay open.

## Change summary

Terminal local echo produces buffer corruption (missed characters, jumpy cursor, incorrect rendering) when working remotely with high latency.

## Best solution

The local echo typeahead system in `src/vs/workbench/contrib/terminalContrib/typeAhead/` needs an eventual-consistency mechanism that reconciles predicted text with actual server responses — healing mismatches rather than leaving corrupted output. Until that work is prioritized, the feature remains off-by-default (preview). This issue should remain open as the canonical tracker for that reliability work.
