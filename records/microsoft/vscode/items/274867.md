# Issue #274867 — Bracket Pair Colorization misinterprets double braces {{ }} in Twig templates

[https://github.com/microsoft/vscode/issues/274867](https://github.com/microsoft/vscode/issues/274867)

## Summary

The bracket pair colorization system already supports multi-character brackets (e.g. `<!--`/`-->` in HTML). The Twig extension just needs to declare `{{`/`}}` etc. in its language-configuration.json. Issue is labeled help-wanted with an open PR (#320527); keeping open for that PR to land.

## Change summary

Bracket pair colorization treats `{{ }}` as two separate pairs instead of one logical pair in Twig templates.

## Best solution

The existing mechanism works — the Twig grammar/extension needs to register multi-character brackets in its language-configuration.json. The open PR #320527 appears to address this. If that PR adds the bracket definitions to a built-in grammar that covers Twig, it resolves the issue; otherwise this is `*caused-by-extension` territory since a Twig extension owns its language configuration.
