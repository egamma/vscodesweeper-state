# Issue #293533 — MCP: Cannot authenticate to multiple Atlassian sites simultaneously - OAuth credentials shared across MCP server configurations

[https://github.com/microsoft/vscode/issues/293533](https://github.com/microsoft/vscode/issues/293533)

## Summary

Valid feature request for per-MCP-server credential isolation when multiple servers share the same OAuth authorization server origin. The maintainer (TylerLeonhardt) is engaged and aware of the design challenge; the issue is correctly categorized and in Backlog.

## Change summary

Allow multiple MCP server configurations sharing the same OAuth origin to maintain independent credential sessions rather than overwriting each other.

## Best solution

Incorporate the MCP server identity (server name from mcp.json or full configured URL) into the credential storage key so that the `providerId` differentiates per-server-config rather than per-origin. The fix would likely touch `src/vs/workbench/api/browser/mainThreadMcp.ts` (auth flow) and `src/vs/workbench/services/authentication/browser/authenticationService.ts` (dynamic provider creation/matching). The maintainer needs to decide how this maps to RFC 8414 semantics without special-casing individual MCP providers.
