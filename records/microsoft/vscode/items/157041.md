# Issue #157041 — Programmatically running a task in VS Code Web ignores `dependsOn` tasks

[https://github.com/microsoft/vscode/issues/157041](https://github.com/microsoft/vscode/issues/157041)

## Summary

Still reproduces on main: the `dependsOn` property is never serialized/deserialized through the task DTO layer, so programmatic `vscode.tasks.executeTask()` creates a task without dependency info when using a non-handle DTO path. The Command Palette path works because it fetches the full task model.

## Change summary

When executing a task programmatically via `vscode.tasks.executeTask()` in VS Code Web, the `dependsOn` chain is ignored because the DTO transfer layer doesn't include it.

## Best solution

Either serialize `dependsOn` through `ITaskDTO` (and handle it in both `TaskDTO.from`/`TaskDTO.to` in `mainThreadTask.ts`), or ensure that programmatic `executeTask` in the `else` branch resolves the full task (including dependencies) from the task service before running it — similar to what the `TaskHandleDTO` path does.
