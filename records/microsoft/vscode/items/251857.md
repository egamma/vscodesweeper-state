# Issue #251857 — Support arbitrary URI schemes in `workbench.action.chat.attachFile`

[https://github.com/microsoft/vscode/issues/251857](https://github.com/microsoft/vscode/issues/251857)

## Summary

The core request — allowing arbitrary URI schemes in `workbench.action.chat.attachFile` — remains unaddressed on main. The `_getResources` method in `AttachResourceAction` (line 110 of chatContextActions.ts) still uses a hardcoded allowlist of `file`, `vscodeRemote`, and `untitled`. A newer denylist-based `isSupportedChatFileScheme` function exists but is only used in the file-picker flow, not in the direct attach commands. A maintainer (roblourens) has engaged asking about the scenario, so this is actively under consideration.

## Change summary

Allow `workbench.action.chat.attachFile` to accept URIs from any registered file system provider, not just `file`, `vscodeRemote`, and `untitled` schemes.

## Best solution

Refactor `AttachResourceAction._getResources`, `AttachPinnedEditorsToChatAction`, and `AttachSelectionAction` to use the existing `isSupportedChatFileScheme` denylist helper instead of the hardcoded allowlist, unifying scheme filtering across all attach paths. The maintainer question about use-case (command-driven attach vs. tool contribution) should be resolved first.
