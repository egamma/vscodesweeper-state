# Issue #255303 — No window border on win 10

[https://github.com/microsoft/vscode/issues/255303](https://github.com/microsoft/vscode/issues/255303)

## Summary

The window border handling on Windows was significantly reworked after this issue was filed. The improved handling in commit 4edaa75440e (shipped in 1.104.0) plus subsequent refinements (accent color support, default/system/off modes) address this report. The current default setting is 'default' which respects theme and Windows system settings, providing visible window borders.

## Change summary

VS Code window on Windows 10 had no visible border when overlapping other windows, making it hard to distinguish window boundaries.

## Best solution

Keep the shipped improvements from the window border rework (commits 5c83cc9d836, 4edaa75440e, d7a3b79b914, 8b47079712e). The `window.border` setting now defaults to `default` which respects both theme and Windows system settings. No further change needed — the reporter should verify on current stable (1.104+).
