# Issue #209552 — Git - beyond a symbolic link error

[https://github.com/microsoft/vscode/issues/209552](https://github.com/microsoft/vscode/issues/209552)

## Summary

Team-authored issue — keeping open. The bug is still present on main: `git check-ignore` fails with exit code 128 when a path traverses a symbolic link, and the error handler in `checkIgnore()` doesn't gracefully handle this case, producing an unhandled promise rejection in the dev console.

## Change summary

Opening a repo with symlinked directories causes an unhandled `git check-ignore` error ('beyond a symbolic link') to appear in the dev console.

## Best solution

In `extensions/git/src/repository.ts`, the `checkIgnore` error handler (around line 2630) should detect the 'beyond a symbolic link' stderr pattern and either silently resolve (treat those paths as not-ignored) or filter them out before passing to `git check-ignore`. Similar to the existing `is in submodule` handling at line 2630.
