# Issue #295478 — Extensions editor uses a mix of native and custom scrollbar

[https://github.com/microsoft/vscode/issues/295478](https://github.com/microsoft/vscode/issues/295478)

## Summary

Still reproduces on main — the extension editor's markdown content renders in a webview with native scrollbars while the surrounding editor uses custom scrollbars. No fix has landed. Routing to the extensions area for triage.

## Change summary

The Extensions editor shows native browser scrollbars inside the markdown webview alongside VS Code's custom scrollbars elsewhere in the same view.

## Best solution

Style the webview scrollbar in the extension editor's markdown renderer to match VS Code's custom scrollbar appearance, or switch the markdown content area to use a custom scrollbar widget. The fix likely involves webview scrollbar CSS (possibly broken by Chromium updates per mjbvz's comment) in the webview layer, coordinated with the extensions editor.
