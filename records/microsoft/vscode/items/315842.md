# Issue #315842 — Testing: Open VS Code window + keybinding "broken" when multiple instances are open

[https://github.com/microsoft/vscode/issues/315842](https://github.com/microsoft/vscode/issues/315842)

## Summary

Team-authored feature request for improving the 'Open VS Code Window' keybinding behavior in the Agents window when multiple VS Code instances are open. Must stay open per hard keep-open rules (MEMBER author).

## Change summary

When triggering 'Open VS Code Window' from the Agents window with multiple VS Code instances open, focus should go to the most recently focused instance (or the one associated with the session) rather than always the same one.

## Best solution

Implement MRU-based window selection so the keybinding and command focus the most recently focused VS Code instance, with a secondary preference for the workspace associated with the active session. Likely touches the Agents window ↔ VS Code window communication layer.
