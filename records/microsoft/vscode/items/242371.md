# Issue #242371 — Support local hyperlink ansi escapes in the integrated terminal

[https://github.com/microsoft/vscode/issues/242371](https://github.com/microsoft/vscode/issues/242371)

## Summary

Fixed on main and shipped: OSC 8 file:// hyperlinks in the terminal now properly open files in the editor and folders in the explorer, as of commit ceb07bd32d4 (1.105.0).

## Change summary

Ctrl+Click on OSC 8 hyperlinks with file:// URLs in the integrated terminal did nothing — the URL link opener lacked file-scheme handling.

## Best solution

Keep the shipped fix; no further change needed. The file:// scheme was already in the default allowedLinkSchemes, and TerminalUrlLinkOpener now delegates file:// URIs to the appropriate local file/folder openers.
