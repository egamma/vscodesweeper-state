# Issue #84136 — The horizontal wheel of mouse is invalid

[https://github.com/microsoft/vscode/issues/84136](https://github.com/microsoft/vscode/issues/84136)

## Summary

The underlying Chromium bug (https://bugs.chromium.org/p/chromium/issues/detail?id=1011866) was fixed in Chromium 78.0.3904.126, confirmed working in exploration builds at the time. VS Code now ships Electron 42 (Chromium ~132), far past the fix. The remaining intermittent reports traced to Logitech Options driver interference, which is outside VS Code's control. No activity in over 4 years.

## Change summary

Horizontal mouse wheel scrolling (especially with Logitech mice on Windows) did not work in VS Code due to a Chromium bug in the version shipped at the time.

## Best solution

Close as not reproducible on current builds. The Chromium platform fix shipped years ago via Electron upgrades. Any remaining issues are driver-level (Logitech Options) and outside VS Code's scope — users experiencing driver conflicts should configure Logitech Options per Comment 30's suggestion (remove the Chrome application profile).
