# Issue #206840 — `selectHighlights` selects wrong value when search is set to regex

[https://github.com/microsoft/vscode/issues/206840](https://github.com/microsoft/vscode/issues/206840)

## Summary

Still reproduces on main: the selectAll method in MultiCursorSelectionController unconditionally defers to the find widget's search string when regex is enabled, ignoring whether the editor actually has focus.

## Change summary

`editor.action.selectHighlights` selects the find widget's regex search string instead of the editor's current selection when the find widget is open with regex enabled.

## Best solution

Fix the condition at line 622 of `src/vs/editor/contrib/multicursor/browser/multicursor.ts` to also check whether focus is NOT in the editor (as the comment already states). The guard should include something like `!this._editor.hasWidgetFocus()` so the find widget only owns the search when the user is actually interacting with it, not merely because it's visible with regex on.
