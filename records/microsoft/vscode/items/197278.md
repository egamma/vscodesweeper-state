# Issue #197278 — `replaceActive` context key is not set

[https://github.com/microsoft/vscode/issues/197278](https://github.com/microsoft/vscode/issues/197278)

## Summary

Still reproduces on main: the `replaceActive` context key is only set when `isReplaceShown() && replaceAllAction.enabled`, meaning it stays false whenever replace mode is open but no search results exist yet. This breaks user keybindings that depend on distinguishing find vs. replace mode.

## Change summary

The `replaceActive` context key should reflect whether the replace UI is shown, but it additionally requires search results to exist (via `replaceAllAction.enabled`).

## Best solution

Decouple the `replaceActive` context key from `replaceAllAction.enabled` in `searchWidget.ts#updateReplaceActiveState()` so it reflects purely whether the replace UI is visible (`isReplaceShown()`). This may require introducing a separate context key or condition for the replace-all button enablement if that distinction is needed elsewhere.
