# Issue #197278 — `replaceActive` context key is not set

[https://github.com/microsoft/vscode/issues/197278](https://github.com/microsoft/vscode/issues/197278)

## Summary

Still reproduces on main: `replaceActive` context key requires both the replace UI to be shown AND `replaceAllAction.enabled` (which needs search results), so it is never `true` when the user simply opens the replace view without results. Confirmed, labeled, and in Backlog — keeping open.

## Change summary

The `replaceActive` context key is not set to `true` when the search replace view is opened; it only becomes `true` when there are also search results, breaking user keybindings that distinguish find vs. replace mode.

## Best solution

In `src/vs/workbench/contrib/search/browser/searchWidget.ts`, `updateReplaceActiveState()` (line 626–634) should set `replaceActive` based solely on `isReplaceShown()`, not on `isReplaceShown() && replaceAllAction.enabled`. The `replaceAllAction.enabled` guard should only control the replace-all button's enabled state, not the context key. Care is needed to ensure no other consumers rely on the current combined semantics.
