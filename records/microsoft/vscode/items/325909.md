# Issue #325909 — Extensions Auto Update enterprise policy is overridden by user settings

[https://github.com/microsoft/vscode/issues/325909](https://github.com/microsoft/vscode/issues/325909)

## Summary

Valid bug: the ExtensionsAutoUpdate policy is registered and displayed as 'Managed by organization', but the per-extension auto-update toggle (pinning) and the 'Disable Auto Update for Extensions' action do not check whether the setting is policy-managed before allowing user overrides.

## Change summary

When ExtensionsAutoUpdate is set via enterprise policy, users can still override auto-update per-extension or disable it for all extensions via UI actions.

## Best solution

Guard `updateAutoUpdateForAllExtensions` and `setAutoUpdateFor` in extensionsWorkbenchService.ts to bail out (or show a policy-locked message) when `configurationService.inspect(AutoUpdateConfigurationKey).policyValue` is defined. Hide or disable the 'Disable Auto Update for Extensions' menu item and the per-extension auto-update checkbox when the policy is active.
