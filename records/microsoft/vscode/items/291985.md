# Issue #291985 — Bug: Enabled terminal.integrated.altClickMovesCursor alters entire Windows 11 system behavior for Ctrl-C hotkey

[https://github.com/microsoft/vscode/issues/291985](https://github.com/microsoft/vscode/issues/291985)

## Summary

The described behavior — a VS Code terminal setting affecting keyboard input in other Windows applications — is not technically possible: the setting only passes a boolean to xterm.js which handles events exclusively within its own DOM element. No global hooks or system-wide input interception exist in this code path.

## Change summary

Reporter claims `terminal.integrated.altClickMovesCursor` causes Ctrl-C to misbehave in other Windows applications (Excel) while VS Code runs in the background.

## Best solution

Close as not reproducible. The setting is purely internal to xterm.js's mouse event handling within the terminal panel. If the reporter still experiences cross-application input issues, they should investigate installed extensions that may register global keyboard hooks (e.g. Roo-Cline, Claude Dev) or a system-level input issue.
