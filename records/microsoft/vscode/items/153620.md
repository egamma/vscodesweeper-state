# Issue #153620 — Allow merging from right to left with diff arrows view

[https://github.com/microsoft/vscode/issues/153620](https://github.com/microsoft/vscode/issues/153620)

## Summary

Still unimplemented: the diff editor gutter only provides 'Revert Block' (right-to-left revert from modified to original state). There is no 'Apply Block' action in the opposite direction when both sides are editable. The infrastructure exists (originalEditable option, diffEditorOriginalWritable context key) but no corresponding apply-to-original action has been added.

## Change summary

Allow merging individual diff blocks from right to left (applying modified content to original) in the diff editor when both sides are editable.

## Best solution

Add a new 'Apply Block' action in the diff editor gutter (alongside 'Revert Block') that copies the original content to the modified side (or vice versa depending on perspective), gated on `diffEditorOriginalWritable`. The gutter toolbar infrastructure in `gutterFeature.ts` and `DiffEditorHunkToolbar` menu already supports multiple actions — this would add a second arrow pointing the opposite direction.
