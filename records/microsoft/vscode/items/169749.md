# Issue #169749 — Relative line numbers should change while folding (like in VIM). 

[https://github.com/microsoft/vscode/issues/169749](https://github.com/microsoft/vscode/issues/169749)

## Summary

Valid feature request with 21 upvotes (meets Backlog threshold). The behavior was implemented in #155970, reverted in #159236 due to breaking destructive-operation workflows, and the community is asking for it back behind a configuration option. No subsequent re-implementation found on main.

## Change summary

Relative line numbers should skip folded regions (counting folds as one line) to match Vim behavior, exposed as a configurable option.

## Best solution

Re-introduce the folded-region-skipping logic from commit 187ec0613a7 behind a new editor option (e.g. `editor.lineNumbers: 'relativeVisual'` or a separate boolean) so users who want Vim-style navigation get accurate relative numbers without breaking users who rely on the current counting for destructive operations. The relevant code lives in `src/vs/editor/common/viewModel/viewModelLines.ts` and `src/vs/editor/common/config/editorOptions.ts`.
