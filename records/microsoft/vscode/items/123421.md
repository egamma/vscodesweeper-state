# Issue #123421 — Couldn't open two folders with almost equal names, except for capitalization on Windows via SSH

[https://github.com/microsoft/vscode/issues/123421](https://github.com/microsoft/vscode/issues/123421)

## Summary

Still present on main: `extUriBiasedIgnorePathCase` in src/vs/base/common/resources.ts still returns `true` for non-file schemes, causing remote URIs to be compared case-insensitively. The agreed-upon fix (changing to `false`) was never applied.

## Change summary

Opening two remote folders differing only in capitalization redirects to the already-open window instead of opening a new one.

## Best solution

Apply the fix agreed upon in 2021: change line 343 of `src/vs/base/common/resources.ts` to return `false` for non-file schemes so remote URIs are compared case-sensitively. This matches the overwhelmingly common case (Linux remote) and the worst failure mode (two windows for the same folder on a case-insensitive remote) is benign.
