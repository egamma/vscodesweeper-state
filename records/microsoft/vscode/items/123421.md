# Issue #123421 — Couldn't open two folders with almost equal names, except for capitalization on Windows via SSH

[https://github.com/microsoft/vscode/issues/123421](https://github.com/microsoft/vscode/issues/123421)

## Summary

Still unfixed on main: `extUriBiasedIgnorePathCase` returns `true` for non-file schemes, making remote URIs case-insensitive. The agreed-upon fix from 2021 (return `false` for non-file schemes) was never committed.

## Change summary

Opening two remote folders whose names differ only in capitalization redirects to the already-open window instead of opening a second one.

## Best solution

Change `extUriBiasedIgnorePathCase` in `src/vs/base/common/resources.ts` to return `false` (case-sensitive) for non-file schemes, as agreed by @bpasero and @aeschli in the thread. This makes remote URIs case-sensitive by default, which is correct for the overwhelmingly common Linux remote scenario.
