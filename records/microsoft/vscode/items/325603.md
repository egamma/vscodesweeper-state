# Issue #325603 — Maximize/Unmaximize action moves when clicking it

[https://github.com/microsoft/vscode/issues/325603](https://github.com/microsoft/vscode/issues/325603)

## Summary

Team-filed bug about the Maximize/Unmaximize action in the agents window removing the toggle sidebar button (causing layout shift) instead of disabling it. Must stay open per team-author policy.

## Change summary

When maximizing a session in the agents window, the toggle sidebar action is removed from the toolbar rather than being disabled, causing the maximize button to shift position.

## Best solution

Change the maximize/unmaximize action's `when` clause for the sidebar toggle so it disables (via `enablement`) rather than hides the action, preventing toolbar layout shift. The fix likely lives in `src/vs/sessions/browser/layoutActions.ts` or `src/vs/sessions/contrib/editor/browser/editor.contribution.ts` where toolbar menu items are registered with `when` conditions that should become `enablement` conditions.
