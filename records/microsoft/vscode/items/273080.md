# Issue #273080 — Chat response: inconsistent rendering of file labels

[https://github.com/microsoft/vscode/issues/273080](https://github.com/microsoft/vscode/issues/273080)

## Summary

Team-authored issue (bpasero) reporting a real visual inconsistency in chat file label rendering that remains unfixed on main. The `renderFileWidgets` logic only renders pills for links with empty text or explicit `vscodeLinkType` metadata — when the LLM produces links with non-empty text and no metadata, they render as plain links instead of pills.

## Change summary

Chat responses render some file references as styled pill widgets and others as plain links, depending on whether the link text is empty or carries `vscodeLinkType` metadata.

## Best solution

Broaden `renderFileWidgets` in `src/vs/workbench/contrib/chat/browser/widget/chatContentParts/chatInlineAnchorWidget.ts` to detect file-scheme links (or links pointing to workspace resources) regardless of link text content, so all file references render consistently as pill widgets.
