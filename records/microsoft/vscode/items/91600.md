# Issue #91600 — [folding] pressing Enter in collapsed region should add new line outside region

[https://github.com/microsoft/vscode/issues/91600](https://github.com/microsoft/vscode/issues/91600)

## Summary

Valid feature request for improved Enter behavior in collapsed folding regions. No implementation exists on current main — the folding contribution has no special handling for newline insertion at the end of a collapsed region's header line. The issue has 9 upvotes and remains in Backlog; keeping open for community traction.

## Change summary

Pressing Enter at the end of a collapsed folding region's header line should insert a new line after the region (outside it), rather than expanding the region and inserting inside.

## Best solution

Implement logic in the editor's type/newline handling (likely around cursor operations or the folding contribution itself) so that when the cursor is at the visual end of a collapsed region header, Enter inserts after the folded range end line rather than at cursor's literal model position inside the range. Related to https://github.com/microsoft/vscode/issues/72422.
