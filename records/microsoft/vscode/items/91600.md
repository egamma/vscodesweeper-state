# Issue #91600 — [folding] pressing Enter in collapsed region should add new line outside region

[https://github.com/microsoft/vscode/issues/91600](https://github.com/microsoft/vscode/issues/91600)

## Summary

Still an open feature request — no implementation found on main. The current behavior (typing inside collapsed region expands it) is unchanged, and there's recent community interest.

## Change summary

Pressing Enter at the end of a collapsed folding region's header line should insert a new line after the region rather than expanding it and inserting inside.

## Best solution

Implement cursor/typing awareness of collapsed fold boundaries in the editor's type/enter handling (likely in `src/vs/editor/contrib/folding/browser/folding.ts` and the editor's cursor/type commands). When the cursor is at the visual end of a collapsed region, Enter should position the new line after the folded range. Related to https://github.com/microsoft/vscode/issues/72422.
