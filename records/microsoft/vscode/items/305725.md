# Issue #305725 — auto-deny output to dev null

[https://github.com/microsoft/vscode/issues/305725](https://github.com/microsoft/vscode/issues/305725)

## Summary

Valid request to auto-approve writes to `/dev/null` in the agent permission system. The current behavior requires manual approval because `/dev/null` is outside the workspace, but it's a harmless Unix null device that should be safe to auto-approve.

## Change summary

Agent tool permission system requires manual approval when commands write to `/dev/null`, which is a harmless Unix null device file.

## Best solution

Add `/dev/null` (and Windows `NUL`) as a safe path that is always auto-approved in the write permission handler (`extensions/copilot/src/extension/chatSessions/copilotcli/node/permissionHelpers.ts`), similar to how internal session resources are auto-approved. Alternatively, the shell permission handler could recognize output redirections to `/dev/null` as benign.
