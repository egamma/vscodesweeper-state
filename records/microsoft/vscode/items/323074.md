# Issue #323074 — Copilot Chat Autopilot switches UI to Agent mode but the model does not recognize Agent mode

[https://github.com/microsoft/vscode/issues/323074](https://github.com/microsoft/vscode/issues/323074)

## Summary

The report describes a state synchronization bug where Autopilot switches the UI to Agent mode but the model's execution context doesn't reflect the mode change. The mode transition logic spans both the Copilot extension (exitPlanModeHandler, planAgentProvider handoffs, toolCallingLoop) and core chat mode services. Recent commits (e.g. 79d34e6e519 aligning autopilot/assisted approvals, 6a4e80f425c adding autopilot goal mode) have improved the area but there's no clear fix for this specific symptom. The related issue #307385 is open in Backlog. The report lacks version info and concrete repro steps, but the described behavior is plausible given the complexity of the mode transition state machine.

## Change summary

When Autopilot switches from Plan to Agent mode, the model does not receive or recognize the updated mode context, requiring users to manually tell the model it is in Agent mode.

## Best solution

The mode transition path from Plan→Agent via Autopilot needs to ensure the model's system prompt or execution context includes the current mode state when the handoff occurs. The likely files are in the Copilot extension's exitPlanModeHandler.ts, planAgentProvider.ts (handoff configuration), and toolCallingLoop.ts (autopilot iteration). This may also relate to the open #307385 which covers Plan mode immediately starting implementation in Autopilot — the two reports suggest the Plan/Agent/Autopilot state machine has stale-transition issues that should be addressed together.
