# Issue #213300 — There is a bug with reporting bug here

[https://github.com/microsoft/vscode/issues/213300](https://github.com/microsoft/vscode/issues/213300)

## Summary

Still reproducible on main: extensions whose `bugsUrl` contains a path beyond `/issues` (e.g. `/issues/new/choose`) bypass the regex that normalizes GitHub URLs, so the issue reporter opens the raw URL without pre-filling title/body. The regex hasn't been updated to handle these patterns.

## Change summary

When reporting a bug against an extension (like Jupyter) whose bugs URL points to a GitHub issue template chooser (`/issues/new/choose`), the issue reporter redirects to that page without pre-filling system info, version, or extension data.

## Best solution

Widen the GitHub URL regex in the three issue reporter code paths (`baseIssueReporterService.ts`, `issueReporterOverlay.ts`, `issueFormService.ts`) to strip trailing `/issues/new/choose`, `/issues/new`, or similar suffixes back to `owner/repo`, then construct the pre-filled `/issues/new?title=...&body=...` URL as usual.
