# Issue #297308 — Chat: It's easy to accidentally enable editing all sensitive files

[https://github.com/microsoft/vscode/issues/297308](https://github.com/microsoft/vscode/issues/297308)

## Summary

Security-labeled, important-labeled issue on the On Deck milestone filed by a contributor — three independent hard keep-open rules apply. The concern is valid: the default 'Allow in Session' button for sensitive-file editing silently grants blanket permission for all sensitive files without adequate disclosure.

## Change summary

The 'Allow in Session' button for sensitive file edits in chat is the default action and grants blanket permission for all sensitive files without a disclaimer, making it too easy to accidentally allow unrestricted editing.

## Best solution

Revert the default prominent button to 'Allow Once' (per-file), move 'Allow in Session' to a secondary action (dropdown), and if 'Allow in Session' is kept, scope it to the specific file rather than all sensitive files — or at minimum add a clear disclaimer about its breadth, consistent with the /yolo experience's warning.
