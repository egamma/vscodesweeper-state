## Summary

Team-authored issue (MEMBER) — keeping open for maintainer judgment. The described behavior is likely by-design: VS Code's integrated terminal injects GIT_ASKPASS via the built-in git extension, enabling credential prompts, while a standalone WSL terminal lacks this setup. ## Change summary

Git push over HTTPS in a standalone WSL terminal fails without a credential prompt, while VS Code's integrated terminal succeeds because it injects GIT_ASKPASS.

## Best solution

This is likely *as-designed behavior: VS Code's git extension sets GIT_ASKPASS for its integrated terminal (extensions/git/src/askpass.ts), which is why credentials work there. The standalone WSL terminal needs its own credential helper configured (e.g. git-credential-manager). A maintainer should confirm whether this warrants documentation, a WSL-specific fix, or an *as-designed close.
