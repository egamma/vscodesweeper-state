# Issue #318565 — "Publish to GitHub" defaults to broken HTTPS remote instead of SSH

[https://github.com/microsoft/vscode/issues/318565](https://github.com/microsoft/vscode/issues/318565)

## Summary

Working as designed: the remote protocol is controlled by the `github.gitProtocol` setting (default: `https`). Setting it to `ssh` produces `git@github.com:…` remotes. HTTPS is a valid default because VS Code's built-in credential helper handles authentication transparently for most users.

## Change summary

"Publish to GitHub" uses an HTTPS remote URL by default; the reporter wants SSH by default.

## Best solution

Keep the current behavior. The `github.gitProtocol` setting already lets users opt into SSH remotes. HTTPS is the correct default for the majority of users who rely on VS Code's integrated credential manager.
