# Issue #226754 — SCM Graph - On refresh, do not lose the current selection

[https://github.com/microsoft/vscode/issues/226754](https://github.com/microsoft/vscode/issues/226754)

## Summary

Team-authored feature request on the 'On Deck' milestone — keep open per policy. The request to preserve SCM Graph selection across refresh is a valid UX improvement that also unblocks automatic refresh on fetch.

## Change summary

SCM Graph loses the currently selected commit when the view is refreshed; selection should be preserved and revealed.

## Best solution

In the SCM Graph view refresh path, capture the selected commit identity before re-rendering, then after the tree is repopulated, re-select and reveal that commit. This likely lives in `src/vs/workbench/contrib/scm/browser/` in the graph-related view or tree code. Once selection persistence works, automatic refresh on fetch detection becomes viable.
