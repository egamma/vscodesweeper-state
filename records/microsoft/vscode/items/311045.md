## Summary

Team-authored improvement request to enhance shell type detection for Codex (and similar agent shells) on Mac/Linux. Currently, non-Windows detection relies on matching the process title string against a static map (`generalShellTypeMap` / `posixShellTypeMap`), which is less robust than the Windows `WindowsShellHelper` process-tree walk. The gap is real and still present on main.

## Change summary

Improve Mac/Linux shell type detection for Codex and other agent CLI shells beyond simple process-title string matching.

## Best solution

Implement a lightweight Unix analogue to `WindowsShellHelper`'s process-tree walk ŌĆö for example, reading `/proc/<pid>/cmdline` on Linux and using `proc_pidpath` / `sysctl` on macOS ŌĆö so that agent shells like Codex, Claude Code, Copilot CLI, and Gemini CLI are reliably identified even when the PTY title doesn't match the executable name. The work lives in `src/vs/platform/terminal/node/terminalProcess.ts` (the non-Windows `shellType` getter at line 120) and likely a new helper analogous to `windowsShellHelper.ts`.
