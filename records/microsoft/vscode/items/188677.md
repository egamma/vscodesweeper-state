# Issue #188677 — Inconsistent column number due to unicode symbols between command palette and status bar (+ inconsistent with Chrome and Firefox Devtools)

[https://github.com/microsoft/vscode/issues/188677](https://github.com/microsoft/vscode/issues/188677)

## Summary

The inconsistency between status bar column (code-point-based via toStatusbarColumn) and command palette 'Character' (UTF-16-based model column) remains on current main. The issue is real but has low community traction and is already labeled under-discussion.

## Change summary

Status bar shows a code-point-based column number while the 'Go to Line' command palette shows a UTF-16-based character offset, creating confusion for users working with multi-code-unit Unicode characters like emoji flags.

## Best solution

Unify the column display so that the status bar and the Go-to-Line command palette agree on the same counting unit. The fix would likely touch `src/vs/editor/contrib/gotoLine/` to use the same `toStatusbarColumn` conversion, or document that 'Character' means something different. The broader question of whether to match JS engine column numbers (UTF-16) is a design decision that needs maintainer input.
