# Issue #188677 — Inconsistent column number due to unicode symbols between command palette and status bar (+ inconsistent with Chrome and Firefox Devtools)

[https://github.com/microsoft/vscode/issues/188677](https://github.com/microsoft/vscode/issues/188677)

## Summary

Keep open: the issue is already in Backlog and still describes a real editor-core policy gap around which column model VS Code should expose for Unicode text. Current main has partially changed Go to Line behavior, but the status bar still uses a separate code-point-based column calculation, so there is not enough evidence to close as fixed or as-designed.

## Change summary

Make column reporting and navigation for Unicode symbols consistent between the status bar, Go to Line/Column, and JavaScript/TypeScript stack trace positions.

## Best solution

Editor core should decide the intended column semantics for Unicode text and apply them consistently in `CursorColumns.toStatusbarColumn`, `CodeEditorWidget.getStatusbarColumn`, and `AbstractGotoLineQuickAccessProvider.parsePosition`, with tests covering regional-indicator flags and surrogate-pair emoji.
