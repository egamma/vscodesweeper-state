# Issue #307963 — Extremely slow search in Keyboard shortcut panel

[https://github.com/microsoft/vscode/issues/307963](https://github.com/microsoft/vscode/issues/307963)

## Summary

The keybindings editor search performs unindexed linear filtering over all items, creating a KeybindingItemMatches object (with multiple string-matching passes) per item on every keystroke. This is a real perf bug for broad queries like 'workbench.action.' that match many entries.

## Change summary

Searching in the Keyboard Shortcuts panel freezes the UI for several seconds when many results match.

## Best solution

Optimize filterByText in keybindingsEditorModel.ts — consider early-out when match count exceeds a threshold, batch rendering, or caching partial match results. The rendering splice of all results at once in renderKeybindingsEntries also contributes to the freeze.
