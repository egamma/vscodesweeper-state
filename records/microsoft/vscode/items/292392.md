# Issue #292392 — Open associated workspace / repo with recent session

[https://github.com/microsoft/vscode/issues/292392](https://github.com/microsoft/vscode/issues/292392)

## Summary

Team-authored feature request (MEMBER) to open the associated workspace/repo when clicking a recent agent session from the welcome page. Currently `openSessionInChat` only restores chat history without switching the workspace. Keeping open per policy.

## Change summary

Clicking a recent agent session from the welcome page should also open the workspace or repo associated with that session, not just restore the chat history.

## Best solution

When a recent session is clicked from the welcome page, look up the workspace/folder associated with that session (from the session metadata or agent host state) and open it alongside the chat history. The main touch points are `openSessionInChat` in `src/vs/workbench/contrib/welcomeAgentSessions/browser/agentSessionsWelcome.ts` and the session model in `src/vs/workbench/contrib/chat/browser/agentSessions/agentSessionsModel.ts` — the session likely needs to store and expose its associated workspace URI so the welcome page can call `IHostService.openWindow` or equivalent before/after restoring the session.
