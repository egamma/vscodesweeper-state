# Issue #300593 — Plan Mode "Start Implementation" button clears existing draft text in the chat input box

[https://github.com/microsoft/vscode/issues/300593](https://github.com/microsoft/vscode/issues/300593)

## Summary

Confirmed in source: `executeHandoff` in `chatWidget.ts` unconditionally calls `this.input.setValue(promptToUse, false)`, overwriting any draft text. No fix has been applied on main. This is a real UX bug — the handoff prompt replaces the user's in-progress input without warning.

## Change summary

Clicking 'Start Implementation' (or any handoff button) replaces draft text in the chat input box with the handoff prompt, discarding the user's in-progress work.

## Best solution

Guard the draft in `executeHandoff` — either stash and restore the user's input value after the handoff prompt is sent (when `send: true`), or append/prepend the handoff prompt rather than replacing. The fix is localized to `ChatWidget.executeHandoff` in `src/vs/workbench/contrib/chat/browser/widget/chatWidget.ts` around line 1429. When `send: true` the prompt is auto-submitted anyway, so the input could be restored to the draft afterward; when `send: false` the handoff prompt is placed for user review, and the draft should be preserved separately (e.g. via undo stack or prepend).
