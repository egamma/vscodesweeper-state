# Issue #238358 тАФ When window gets focus it changes the selected file in Explorer

## Summary

Real bug тАФ when the window regains focus, the Explorer auto-reveals the active editor's file (via `onDidActiveEditorChange` тЖТ `selectActiveFile` тЖТ `tree.reveal`), which scrolls the tree while a mousedown from the user's click is in flight, causing the wrong file to open. No fix found on current main; the code path is unchanged.

## Change summary

Clicking a file in the Explorer when the window doesn't have focus opens the wrong file because the tree scrolls to reveal the active editor's file during the focus-regain event, shifting items under the mouse.

## Best solution

In `explorerView.ts`, the `selectActiveFile()` call triggered by `onDidActiveEditorChange` should be suppressed (or deferred) when the reveal is caused by a window-focus-regain event that coincides with a mouse click on the tree. The fix likely lives in `src/vs/workbench/contrib/files/browser/views/explorerView.ts` around lines 318тАУ319 where `selectActiveFile()` is wired to `onDidActiveEditorChange`. One approach: detect that the focus change is from a window activation (not an editor tab switch) and skip or debounce the reveal in that case.
