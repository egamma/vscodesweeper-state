# Issue #275215 — Unclear that thinking did not make changes

## Summary

Team-authored plan-item reporting UX confusion when thinking headers suggest changes were made (e.g. 'Applying changes to listeners') when only read-only tool calls occurred. Significant rework landed in commit 49b5f6e36f6 and many follow-ups, but the core concern about clarity of thinking-vs-acting may still apply — keeping open for the area owner to assess.

## Change summary

Thinking section headers like 'Applying changes to listeners' misleadingly suggest the model made changes when it only performed read-only operations during thinking.

## Best solution

Ensure the thinking content part (src/vs/workbench/contrib/chat/browser/widget/chatContentParts/chatThinkingContentPart.ts) clearly distinguishes read-only thinking/tool use from actual code modifications, so users don't feel the need to expand every thinking block to verify nothing was changed.
