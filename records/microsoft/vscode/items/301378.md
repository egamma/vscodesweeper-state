# Issue #301378 — Terminal colours wrong from ANSI sequence

[https://github.com/microsoft/vscode/issues/301378](https://github.com/microsoft/vscode/issues/301378)

## Summary

This is the same as-designed behavior as #267565: the default `terminal.integrated.minimumContrastRatio` of 4.5 shifts foreground colors to meet WCAG AA contrast, which turns bright white into black on bright red. Setting it to 1 restores exact ANSI colors. The 24-bit color claim is likely the same mechanism at work.

## Change summary

Terminal shows black text instead of bright white on bright red backgrounds when using ANSI color sequences.

## Best solution

Close as as-designed — the workaround is `"terminal.integrated.minimumContrastRatio": 1`. This is the same behavior and resolution as https://github.com/microsoft/vscode/issues/267565.

- **Proposed comment:**

  > Thanks for the report. This is by design — VS Code enforces a minimum contrast ratio (WCAG AA, 4.5:1) by default, which shifts foreground colors to improve readability. On a bright red background, bright white gets shifted to black to meet the contrast threshold.
  > 
  > To restore exact ANSI colors, set:
  > 
  > ```json
  > "terminal.integrated.minimumContrastRatio": 1
  > ```
  > 
  > This also explains the 24-bit color issue — the colors are parsed correctly but then adjusted by the contrast algorithm.
  > 
  > See https://github.com/microsoft/vscode/issues/267565 and the [docs on minimum contrast ratio](https://code.visualstudio.com/docs/terminal/appearance#_minimum-contrast-ratio) for more details.
  > 
  > Closing as as-designed since the behavior is intentional and configurable.
