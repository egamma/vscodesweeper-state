# Issue #301378 — Terminal colours wrong from ANSI sequence

[https://github.com/microsoft/vscode/issues/301378](https://github.com/microsoft/vscode/issues/301378)

## Summary

The reported bright-white-to-black color change is the expected behavior of the `terminal.integrated.minimumContrastRatio` setting (default 4.5), which adjusts foreground colors to meet WCAG AA contrast against the background. Setting it to 1 disables the adjustment and restores raw ANSI colors.

## Change summary

Terminal shows black text instead of bright white on a bright-red background when using ANSI color sequences.

## Best solution

Close as designed — the `terminal.integrated.minimumContrastRatio` setting (default 4.5) intentionally adjusts foreground colors to meet accessibility contrast requirements. The user can set it to `1` to disable contrast enforcement and see raw ANSI colors.

- **Proposed comment:**

  > Thanks for the report. The behavior you're seeing — bright white becoming black on a bright-red background — is caused by the `terminal.integrated.minimumContrastRatio` setting, which defaults to `4.5` (WCAG AA). This setting adjusts the foreground color to meet the specified contrast ratio against the background, and in some cases that means flipping to black.
  > 
  > To restore raw ANSI colors without any contrast adjustment, set:
  > 
  > ```json
  > "terminal.integrated.minimumContrastRatio": 1
  > ```
  > 
  > Regarding 24-bit (truecolor) sequences: VS Code's terminal does support them, but the same contrast-ratio adjustment applies. Setting the value to `1` should resolve both issues you're seeing.
  > 
  > Closing this as the current behavior is intentional for accessibility, with a clear opt-out path.
