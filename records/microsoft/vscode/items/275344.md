# Issue #275344 — Can't actually save prompt from `/savePrompt` with a manual save

[https://github.com/microsoft/vscode/issues/275344](https://github.com/microsoft/vscode/issues/275344)

## Summary

Still open on main: the Save As Prompt File action is only wired to an EditorContent menu button, not to the standard file-save command path. The normal Cmd+S on an untitled prompt file does not delegate to the prompt-specific save flow.

## Change summary

Using Cmd+S / Ctrl+S on a prompt file created by /savePrompt errors instead of saving it as a prompt file.

## Best solution

Hook the standard save command for untitled files with the prompt language ID so it delegates to the `SaveAsPromptFileAction` flow (ask for folder/name), or register a save participant that intercepts save for these untitled prompt documents. The fix likely belongs in `src/vs/workbench/contrib/chat/browser/promptSyntax/saveAsPromptFileActions.ts` or a new save-participant file in that folder.
