# Issue #295255 — Option + left-mouse click causes screen to go blank in interactive programs (like pagers, e.g. less), making copy/paste tedious

[https://github.com/microsoft/vscode/issues/295255](https://github.com/microsoft/vscode/issues/295255)

## Summary

The existing setting `terminal.integrated.macOptionClickForcesSelection` directly addresses this use case — it forces Option+click to act as selection (bypassing mouse mode sent to programs like `less`). The default is `false`, which is intentional to preserve column-selection and alt-click-moves-cursor behavior for the common case.

## Change summary

Option+click in interactive terminal programs (like `less` or `man`) causes the screen to go blank instead of selecting text.

## Best solution

Close as as-designed. The setting `terminal.integrated.macOptionClickForcesSelection` exists to force selection on Option+click in exactly this scenario (programs with mouse mode enabled). The reporter can enable it to get the desired behavior.

- **Proposed comment:**

  > Thanks for the report. This behavior is controlled by the existing setting `terminal.integrated.macOptionClickForcesSelection`. When set to `true`, Option+click will force a regular text selection instead of sending the click to the program — which is exactly what you need for copying text from interactive programs like `less` or `man`.
  > 
  > You can enable it by adding this to your settings:
  > ```json
  > "terminal.integrated.macOptionClickForcesSelection": true
  > ```
  > 
  > The default is `false` because most users prefer the alt-click-moves-cursor and column-selection behavior, but for workflows that frequently use pagers/interactive programs, flipping this setting should resolve the blank-screen issue and make selection seamless.
