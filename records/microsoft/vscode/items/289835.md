# Issue #289835 — Remote SSH freezes with Workspace Trust/Workspace Trust is not remembered when using Remote SSH

[https://github.com/microsoft/vscode/issues/289835](https://github.com/microsoft/vscode/issues/289835)

## Summary

Confirmed bug: workspace trust entries for remote SSH workspaces can be stored without `fsPath`/`_sep` (due to lazy serialization in URI.toJSON) and with inconsistent path casing, causing the trust comparison to fail on reload. Still present on main — no fix found.

## Change summary

Workspace trust for Remote SSH connections is not persisted correctly, causing the trust dialog to reappear on every window reload.

## Best solution

Ensure that when trust URIs are stored, the path is canonicalized consistently (lowercase drive letter on Windows remotes). The fix likely belongs in `src/vs/workbench/services/workspaces/common/workspaceTrust.ts` in `doSetUrisTrust` — canonicalize the URI path before serializing. Alternatively, ensure the comparison in `doGetUriTrustInfo` uses case-insensitive matching for Windows remote paths.

- **comparison path:** src/vs/workbench/services/workspaces/common/workspaceTrust.ts:390-391 uses `uriIdentityService.extUri.isEqualOrParent` which for non-file schemes compares `base.path` vs `parentCandidate.path` — case sensitivity depends on the file system provider registration.
