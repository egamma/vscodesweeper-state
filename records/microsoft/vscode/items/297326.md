# Issue #297326 — Integrated terminal: flag emoji overlaps following space cell ("🇬🇧 to" looks like "🇬🇧to")

[https://github.com/microsoft/vscode/issues/297326](https://github.com/microsoft/vscode/issues/297326)

## Summary

This is an upstream xterm.js issue: VS Code uses the `@xterm/addon-unicode11` for character width, which doesn't properly handle regional indicator (flag emoji) grapheme clusters. The xterm.js project has a separate `addon-unicode-graphemes` that correctly handles RI pairs (their test suite verifies it), but VS Code doesn't use that addon.

## Change summary

Flag emoji (regional indicator pairs like 🇬🇧) visually overflow their allocated terminal cells, overlapping the following space character.

## Best solution

Evaluate adopting `@xterm/addon-unicode-graphemes` (which correctly computes width=2 for RI-pair clusters) as a replacement or supplement to `@xterm/addon-unicode11` in `src/vs/workbench/contrib/terminal/browser/xterm/xtermTerminal.ts`. Alternatively, file upstream on xtermjs/xterm.js to improve RI-pair handling in the unicode11 addon.
