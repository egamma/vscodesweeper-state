# Issue #274060 — Support content exclusions/ignore service in core

[https://github.com/microsoft/vscode/issues/274060](https://github.com/microsoft/vscode/issues/274060)

## Summary

Valid architectural request to move the content exclusion/ignore service from the Copilot extension into VS Code core so the terminal tool and other core components can respect content exclusions. Currently all content exclusion logic lives in `extensions/copilot/src/platform/ignore/`; nothing exists in `src/vs/`. Keeping open — this is a real gap that needs a product/architecture decision.

## Change summary

Move content exclusion/ignore service into VS Code core so the terminal tool and other core components can enforce content exclusions natively.

## Best solution

Design a core `IContentExclusionService` in `src/vs/platform/` (or similar) that the terminal tool and other core components can consume, then have the Copilot extension delegate to or extend it. This requires an architecture decision on the service boundary and what 'partially support' means for core vs extension responsibilities.
