# Issue #158309 — Git - Git error (Cannot fast-forward to multiple branches.) when syncing during autofetch

[https://github.com/microsoft/vscode/issues/158309](https://github.com/microsoft/vscode/issues/158309)

## Summary

Still reproduces on main — the autofetch and user-initiated sync/pull can run concurrently with no serialization, causing git ref conflicts. Milestone is 'On Deck' (a scheduling decision), so keeping open.

## Change summary

Git autofetch and user-initiated sync/pull can overlap, causing 'Cannot fast-forward to multiple branches' or 'incorrect old value provided' errors.

## Best solution

Serialize or single-flight git fetch operations in `extensions/git/src/repository.ts` so that a user-initiated pull/sync either waits for an in-flight autofetch to complete or skips the redundant fetch phase. The `run()` method (line ~2702) allows concurrent operations; adding a per-repo fetch lock or cancelling the autofetch when a user sync starts would fix this.
