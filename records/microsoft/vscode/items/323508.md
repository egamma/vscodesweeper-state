# Issue #323508 — Context window widget missing from AHP sessions

[https://github.com/microsoft/vscode/issues/323508](https://github.com/microsoft/vscode/issues/323508)

## Summary

Team-authored upstream bug — the Copilot SDK does not expose context-window size for models selected by 'auto' (outside the model picker), so the context widget cannot render in AHP sessions. Hard keep-open: author is MEMBER.

## Change summary

The context window widget does not appear in Agent Host sessions because the SDK omits model metadata for auto-selected models not in the model picker.

## Best solution

The upstream Copilot SDK needs to surface context-window size for all models it may select, including auto-picked ones like GPT 5.4 mini. Once the SDK exposes that metadata, VS Code can render the widget. The VS Code side should also consider a graceful fallback (show the widget with unknown/estimated capacity) so partial information is still surfaced.
