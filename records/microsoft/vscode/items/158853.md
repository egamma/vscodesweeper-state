# Issue #158853 — Tabs API does not allow to identify internal tabs

[https://github.com/microsoft/vscode/issues/158853](https://github.com/microsoft/vscode/issues/158853)

## Summary

Still valid: internal editors (Settings, Keyboard Shortcuts) fall through to `UnknownInput` in `_editorInputToDto`, producing `undefined` for `tab.input` in the extension API. With 55 upvotes (well above the 20 threshold) and Backlog milestone, this is an accepted feature request that remains unimplemented.

## Change summary

The Tabs API should expose a typed `TabInput` for built-in internal editors (Settings, Keyboard Shortcuts, etc.) so extensions can identify and programmatically interact with them.

## Best solution

Add new `TabInput` subclass(es) for internal editors — at minimum a `TabInputSettings` and/or a generic internal-editor input with a type identifier — in `src/vs/workbench/api/browser/mainThreadEditorTabs.ts` (`_editorInputToDto`) and the corresponding `vscode.d.ts` API surface, so that `SettingsEditor2Input`, `KeybindingsEditorInput`, and similar editors produce a typed input instead of `undefined`.
