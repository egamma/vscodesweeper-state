# Issue #253709 — SCM Graph - The height of the source control graph changes every time when some elements are enabled/disabled

[https://github.com/microsoft/vscode/issues/253709](https://github.com/microsoft/vscode/issues/253709)

## Summary

The SCM Graph view height resetting when toggling sub-views (like Repositories) or across restarts is a real bug in the view pane size persistence logic. Issue carries the 'On Deck' milestone, so it stays open for maintainer scheduling.

## Change summary

The Source Control Graph view's user-set height resets whenever a sub-view (e.g. Repositories) is toggled on/off, or when VS Code is restarted.

## Best solution

Fix the size save/restore cycle in `ViewPaneContainer`. When `onDidRemoveViewDescriptors` removes a pane, sizes of remaining panes (including the Graph) should be saved before the removal redistributes space. Currently `saveViewSizes` only runs on sash drag and layout, so toggling a sibling view loses the Graph's stored height. The fix likely belongs in `src/vs/workbench/browser/parts/views/viewPaneContainer.ts` around the `onDidRemoveViewDescriptors` / `onDidAddViewDescriptors` flow.
