# Issue #260005 — vscode should not inject a common Env name as `BROWSER`  in terminal when ssh connect.

[https://github.com/microsoft/vscode/issues/260005](https://github.com/microsoft/vscode/issues/260005)

## Summary

The BROWSER env var is intentionally set by VS Code's remote server to enable `open`-style commands in the remote terminal to open a browser on the local machine — this follows the standard Unix BROWSER convention. A `--without-browser-env-var` server flag already exists as an opt-out.

## Change summary

VS Code injects a `BROWSER` environment variable into the remote extension host environment, which can conflict with user scripts that check `process.env.BROWSER`.

## Best solution

Keep the current behavior — setting `BROWSER` follows the standard Unix convention and enables local browser opening from the remote. The existing `--without-browser-env-var` server flag provides an opt-out for users who experience conflicts. Renaming it to `__VSCODE__BROWSER` would break the convention that makes it useful.
