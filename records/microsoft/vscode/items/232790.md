# Issue #232790 — Toggling views/panels has inconsistent focus behaviour

[https://github.com/microsoft/vscode/issues/232790](https://github.com/microsoft/vscode/issues/232790)

## Summary

Active design issue on the 'On Deck' milestone — a fix was attempted and reverted due to breaking muscle memory; the team hasn't converged on the right approach. Must stay open per milestone policy.

## Change summary

Toggling view/panel visibility has inconsistent focus behavior: the focus call fires before the DOM is visible, so most views silently fail to receive focus while some (like SCM) work by accident.

## Best solution

Resolve the design tension (focus vs. no-focus on toggle) with an explicit, opt-in approach — e.g. separate the visibility toggle from the focus action, or move the grid visibility update before the focus call and pass `focus: false` for toggle commands to preserve muscle memory while fixing the underlying ordering bug.
