# Issue #232790 — Toggling views/panels has inconsistent focus behaviour

[https://github.com/microsoft/vscode/issues/232790](https://github.com/microsoft/vscode/issues/232790)

## Summary

Still open and unresolved on main — the fix attempt (PR #235622) was reverted (PR #236556) due to breaking terminal focus muscle memory. The issue carries milestone 'On Deck', which is a scheduling decision that blocks closing.

## Change summary

Toggling sidebar/panel visibility has inconsistent focus behavior: the focus call fires before the grid makes the container visible, so most views silently fail to receive focus.

## Best solution

The team needs to decide on the right semantics (focus vs. no-focus on toggle) and implement it in a way that respects muscle memory — likely by reordering grid visibility before the focus call, combined with explicit focus parameters on toggle actions. The discussion in comments 4–5 sketches the path: no focus on toggle-via-keybinding, but focus when triggered from layout controls.
