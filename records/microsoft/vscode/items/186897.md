# Issue #186897 — material-icon-theme slows down resizing file explorer

[https://github.com/microsoft/vscode/issues/186897](https://github.com/microsoft/vscode/issues/186897)

## Summary

Team-authored, confirmed bug still reproducing on current main — keeping open. The issue is a rendering performance regression on Windows with 4K displays when a third-party icon theme is active, likely rooted in Electron/Chromium rendering behavior.

## Change summary

Resizing the file explorer (and scrolling the editor) becomes very slow on Windows 4K displays when the material-icon-theme extension is installed.

## Best solution

Investigate the Electron/Chromium rendering path on Windows HiDPI that makes icon-theme-heavy tree views expensive to repaint. The root cause likely lives in how SVG/image icons are composited during resize on high-DPI Windows surfaces — profiling the GPU/compositor layer (not just JS) may be needed. If the bottleneck is in Chromium, file an upstream bug; if VS Code's tree virtualization or icon caching can mitigate it, the fix likely touches the file explorer tree rendering or the icon theme integration layer.
