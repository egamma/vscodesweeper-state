# Issue #186897 — material-icon-theme slows down resizing file explorer

[https://github.com/microsoft/vscode/issues/186897](https://github.com/microsoft/vscode/issues/186897)

## Summary

Team-authored and confirmed Electron rendering performance bug on Windows with 4K displays and custom icon themes. Still unfixed on main — no related commits found. Keeping open per team-member policy and because the root cause likely lives in Electron/Chromium's compositor.

## Change summary

Resizing the file explorer (or scrolling the editor) becomes very slow when a custom icon theme with many SVG icons is installed, on Windows with a 4K display.

## Best solution

Investigate whether Electron's GPU compositor or SVG rasterization pipeline is the bottleneck on Windows HiDPI; potentially file upstream with Electron/Chromium, or explore mitigations like caching rasterized icons at the VS Code layer. The `electron` label is already applied.
