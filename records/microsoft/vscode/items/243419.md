# Issue #243419 — Git - not obvious for will be commited (staged or changes)

[https://github.com/microsoft/vscode/issues/243419](https://github.com/microsoft/vscode/issues/243419)

## Summary

Valid feature request to make the Commit button label context-aware (e.g. 'Commit Staged') when both staged and unstaged changes exist. An open community PR (https://github.com/microsoft/vscode/pull/305711) already addresses this — it needs maintainer review.

## Change summary

When both staged and unstaged changes are present, the Commit button label and tooltip should clarify that only staged changes will be committed.

## Best solution

Review and land (or guide to completion) the open community PR https://github.com/microsoft/vscode/pull/305711, which adds a 'Commit Staged' label when staged and unstaged changes coexist. The smart-commit setting interaction noted by gjsjohnmurray should be considered during review.
