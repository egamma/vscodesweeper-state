# Issue #235453 — Opening a folder in `vscode.dev` with a domain (i.e. *.com) as its name through Remote Tunnels returns a “blocked” page

[https://github.com/microsoft/vscode/issues/235453](https://github.com/microsoft/vscode/issues/235453)

## Summary

Confirmed, reproduced bug with an identified root cause (WAF blocking paths containing domain-like strings). The issue is assigned to a team member and should remain open for resolution.

## Change summary

Opening a folder path containing a domain name (e.g. `sime.demo.com`) in vscode.dev via Remote Tunnels triggers a WAF block; adding a trailing slash works around it.

## Best solution

The fix is in the WAF (Web Application Firewall) configuration for vscode.dev — the rule that blocks domain-like patterns in URL paths needs to be refined to allow legitimate folder paths that happen to contain `.com`/`.net`/etc. This is an infrastructure change, not a codebase change.
