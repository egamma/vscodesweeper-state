# Issue #85564 — Git - Git should be able to automatically track new files

[https://github.com/microsoft/vscode/issues/85564](https://github.com/microsoft/vscode/issues/85564)

## Summary

Legitimate feature request with strong community support (104 upvotes) to automatically git-add newly created files. Already promoted to Backlog; extensions exist but the community wants a built-in option. Keeping open for the area owner to evaluate.

## Change summary

Add an option for VS Code to automatically run `git add` on newly created files, so they are tracked by Git without manual staging.

## Best solution

Evaluate adding an opt-in setting (e.g. `git.autoTrackNewFiles`) in the built-in Git extension that runs `git add` when a file is created inside a Git repository. The Git extension already watches workspace file events; hooking into file creation to stage new files would be a small, localized addition. Alternatively, if the team considers this too opinionated for core, formally route it as `*extension-candidate` — a community extension already exists (https://marketplace.visualstudio.com/items?itemName=unbirdev.git-track-new-file).
