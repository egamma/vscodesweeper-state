# Issue #280363 — Terminal Intellisense breaks with PowerShell when using Starship

[https://github.com/microsoft/vscode/issues/280363](https://github.com/microsoft/vscode/issues/280363)

## Summary

Confirmed bug: a specific Nerd Font character (U+E0B0) in Starship's prompt config breaks terminal suggest on Windows PowerShell. The prompt detection heuristics in commandDetectionCapability.ts don't handle this character correctly despite the single-character terminator fix (7af5d1ff412) shipping in 1.106.0. A maintainer is actively investigating.

## Change summary

Terminal Intellisense fails to show suggestions and insert completions when Starship uses U+E0B0 as the prompt character on Windows PowerShell.

## Best solution

Extend the prompt terminator detection in `commandDetectionCapability.ts` (around line 970-1003) and/or `shellIntegrationAddon.ts` `_updatePromptTerminator` to properly handle Nerd Font characters like U+E0B0 that may have non-standard widths or length calculations on Windows.
