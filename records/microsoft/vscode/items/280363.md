# Issue #280363 — Terminal Intellisense breaks with PowerShell when using Starship

[https://github.com/microsoft/vscode/issues/280363](https://github.com/microsoft/vscode/issues/280363)

## Summary

Still reproduces on main: the prompt detection fallback in commandDetectionCapability.ts only matches U+276F (❯) for custom prompts, so Starship configs using a different character glyph break prompt boundary detection, causing suggest to malfunction.

## Change summary

Terminal IntelliSense breaks when Starship uses a custom prompt character (Nerd Font glyph) instead of the default ❯ on Windows PowerShell.

## Best solution

Extend the fallback prompt detection at line 970-977 of commandDetectionCapability.ts to handle a broader set of common prompt-terminating characters, or improve the dynamic `promptTerminator` path so that PowerShell's shell integration correctly reports the prompt even when Starship overrides it.
