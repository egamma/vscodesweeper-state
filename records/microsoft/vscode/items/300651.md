# Issue #300651 — [Feature Request] Copilot Chat: "New chat with context" command to recover from corrupted conversations

[https://github.com/microsoft/vscode/issues/300651](https://github.com/microsoft/vscode/issues/300651)

## Summary

The core request — starting a new chat pre-populated with context from a corrupted conversation — is covered by the fork conversation feature shipped via https://github.com/microsoft/vscode/issues/291481. The `/fork` command and `Fork Conversation` action let users branch from any point in the conversation, preserving context. The specific 400-error cause (empty content blocks) has also been addressed in the Copilot extension's message API layer.

## Change summary

Add a command to start a new chat session pre-populated with sanitized context from a corrupted conversation, recovering from unrecoverable 400 errors caused by empty content blocks.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/291481 — the shipped `/fork` command (`ForkConversationAction`) lets users branch a conversation from any point, which is the recovery mechanism requested here. The empty-content-block 400-error root cause has also been mitigated in the Copilot extension's `messagesApi.ts`.
