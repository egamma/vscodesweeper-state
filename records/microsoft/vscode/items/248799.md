# Issue #248799 — mac / zsh: onDidEndTerminalShellExecution event never fires for second executeCommand

[https://github.com/microsoft/vscode/issues/248799](https://github.com/microsoft/vscode/issues/248799)

## Summary

Real bug with a second confirmation: `onDidEndTerminalShellExecution` doesn't fire for the second `executeCommand` call on zsh/macOS. No fix found on current main. The `confirmation-pending` label is already present but there is now a second confirming report from estruyf, so the bug should be treated as confirmed.

## Change summary

The `onDidEndTerminalShellExecution` event never fires after the second `terminal.shellIntegration.executeCommand` call in zsh on macOS, leaving extensions unable to chain sequential commands.

## Best solution

Investigate the interaction between `endShellExecution` (in `extHostTerminalShellIntegration.ts`) and the zsh shell integration command detection lifecycle — the `onCommandFinished` event from command detection likely doesn't fire for the second command because zsh's prompt/input state isn't properly transitioning back after the first programmatic execution. The `confirmation-pending` label can be upgraded to confirmed given estruyf's independent report. The fix area is `src/vs/workbench/api/common/extHostTerminalShellIntegration.ts` (the `endShellExecution`/`startShellExecution` flow) and possibly the zsh shell integration scripts or command detection capability.
