# Issue #201305 — Explorer: NPM when pasting a file repeatedly

[https://github.com/microsoft/vscode/issues/201305](https://github.com/microsoft/vscode/issues/201305)

## Summary

Still reproduces on main: the paste handler (`pasteFileHandler`) is an unguarded async function — rapid repeated Cmd+V invocations race through `findValidPasteFileTarget`, which checks the explorer model for a free name before the file is actually created, leading to conflicts and crashes. No concurrency guard has been added since the issue was filed.

## Change summary

Holding Cmd+V to paste a file repeatedly in the File Explorer causes a crash due to concurrent unguarded paste operations racing on file-name resolution.

## Best solution

Add a concurrency guard (e.g. a `Sequencer` or a boolean lock) around `pasteFileHandler` in `src/vs/workbench/contrib/files/browser/fileActions.ts` so that overlapping paste invocations are serialized or debounced, preventing the race in `findValidPasteFileTarget`.
