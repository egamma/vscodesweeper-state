# Issue #123987 — [html] allow to configure html formatter for other languages

[https://github.com/microsoft/vscode/issues/123987](https://github.com/microsoft/vscode/issues/123987)

## Summary

A triaged feature request in the Backlog milestone asking for a way to opt-in to running the built-in HTML formatter on related language modes (Django HTML, Jinja). A team member acknowledged the gap but suggested an extension is the proper path; the reporter pushed back noting the HTML formatter already partially supports Django via jsbeautify settings. No further activity since 2021, but it was intentionally placed in Backlog by a maintainer.

## Change summary

Allow users to configure the built-in HTML formatter to format files in related language modes like Django HTML or Jinja HTML.

## Best solution

Either expose an opt-in setting (e.g. `html.format.additionalLanguages`) that lets users explicitly enable the HTML formatter for extra language IDs at their own risk, or document clearly that a Django/Jinja extension should wrap the same beautify-js library to provide formatting for those modes. The `html` area team should decide which path fits the product.
