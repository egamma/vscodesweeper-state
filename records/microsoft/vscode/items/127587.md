# Issue #127587 — [remote] Prefer to open a file in an existing window even when that window is connected to a remote with that file opened

[https://github.com/microsoft/vscode/issues/127587](https://github.com/microsoft/vscode/issues/127587)

## Summary

Valid feature request with 29 upvotes, promoted to Backlog. The requested local-to-remote file path translation when opening files externally remains unimplemented on main — the existing canonicalUri infrastructure only supports the reverse mapping.

## Change summary

When opening a file from an external app (File Explorer, GitHub Desktop), VS Code should detect if that file exists in an open remote/devcontainer window and open it there instead of a new local window.

## Best solution

Implement a forward mapping from local file URIs to remote URIs in the window-opening flow, leveraging remote resolvers to advertise which local paths are mounted in their remote filesystem. This likely touches window routing logic in `src/vs/code/` and the remote authority resolver infrastructure.
