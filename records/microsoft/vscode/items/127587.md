# Issue #127587 — [remote] Prefer to open a file in an existing window even when that window is connected to a remote with that file opened

[https://github.com/microsoft/vscode/issues/127587](https://github.com/microsoft/vscode/issues/127587)

## Summary

Legitimate feature request promoted to Backlog via community upvotes. No evidence of implementation on main; the underlying challenge (mapping local URIs to remote URIs when opening files externally) remains unresolved.

## Change summary

When opening a file externally that exists in an already-open devcontainer/remote window, VS Code should detect the match and open it in the existing remote window instead of a new local window.

## Best solution

Implement a mechanism in the window-opening path (likely around `src/vs/platform/windows/` or `src/vs/code/electron-main/`) that consults remote resolvers for a reverse canonical-URI mapping, allowing local file paths to be matched to open remote windows. This likely requires cooperation from remote extensions (Dev Containers, WSL) to advertise which local paths they have mounted.
