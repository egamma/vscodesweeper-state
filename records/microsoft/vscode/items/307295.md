# Issue #307295 — Copilot actions appear for Python code when chat.disableAIFeatures is set to true

[https://github.com/microsoft/vscode/issues/307295](https://github.com/microsoft/vscode/issues/307295)

## Summary

The 'Convert to f-string with Copilot' and 'Convert to format() call with Copilot' code actions are provided by the Python/Pylance extension (ms-python), not VS Code core. VS Code's own AI code actions (in ChatCodeActionsProvider) are properly gated on the disabled-AI state. The extension needs to respect the `chat.disableAIFeatures` setting.

## Change summary

Python-related AI code actions ('Convert to f-string with Copilot', 'Convert to format() call with Copilot') appear even when `chat.disableAIFeatures` is enabled.

## Best solution

File this against the Python extension (https://github.com/microsoft/vscode-python) or Pylance (https://github.com/microsoft/pylance-release) so they gate their Copilot-branded code actions on the `chat.disableAIFeatures` setting or `ChatContextKeys.enabled`.
