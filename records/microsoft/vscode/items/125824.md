# Issue #125824 — The exitCode in onDidEndTaskProcess event callback is 0 when user hits Ctrl-C on the task.

[https://github.com/microsoft/vscode/issues/125824](https://github.com/microsoft/vscode/issues/125824)

## Summary

Still reproduces on main — the terminal's exit code for a Ctrl-C'd task comes from node-pty's onExit event, which reports 0 on Linux instead of 130. Team-authored and confirmed; keeping open.

## Change summary

The `onDidEndTaskProcess` event reports exitCode 0 when a user Ctrl-C's a running task, instead of 130 (SIGINT).

## Best solution

The fix likely involves either node-pty correctly reporting signal-based exit codes on Linux, or VS Code detecting SIGINT termination and synthesizing the expected exit code (128 + signal number). The terminal process layer in `src/vs/platform/terminal/node/terminalProcess.ts` (line ~407, `this._exitCode || 0`) and the node-pty dependency are the likely fix points.
