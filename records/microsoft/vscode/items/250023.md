# Issue #250023 — Building from Source Produces Several Errors

[https://github.com/microsoft/vscode/issues/250023](https://github.com/microsoft/vscode/issues/250023)

## Summary

The build errors are caused by using pnpm instead of npm. VS Code's build system is designed and documented around npm; the preinstall script explicitly rejects yarn but does not support pnpm either. The commenters themselves confirmed that `npm install` resolves the issue. This is a support question, not a VS Code bug.

## Change summary

Building VS Code from source with pnpm instead of npm produces type errors due to pnpm's stricter dependency hoisting.

## Best solution

Close as a support question — VS Code's build toolchain requires npm (documented in the wiki and enforced in the preinstall script). pnpm's hoisting model does not surface transitive type declarations (e.g. undici-types, node-pty types, 1DS SDK types) the same way npm does, which is expected. Users should follow the documented npm-based build instructions.
