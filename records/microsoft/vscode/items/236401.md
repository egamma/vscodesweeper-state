# Issue #236401 — remote-cli `code` command works in bash, but zsh or tmux

[https://github.com/microsoft/vscode/issues/236401](https://github.com/microsoft/vscode/issues/236401)

## Summary

Still unfixed on main: the remote-cli PATH is set in the extension host environment (extensionHostConnection.ts:54-60), but zsh login shells and tmux reset PATH from profile files, losing the remote-cli entry. No fix or workaround has been implemented in the codebase.

## Change summary

The `code` remote-cli command is unavailable in zsh and tmux terminals on remote SSH because those shells reset the inherited PATH.

## Best solution

The terminal process creation on the server side needs to ensure the remote-cli bin folder is injected into the terminal's PATH regardless of shell type — likely via the environment variable collection mechanism or shell integration scripts (src/vs/server/node/remoteTerminalChannel.ts or the shell integration scripts). The workaround of using VSCODE_GIT_ASKPASS_NODE to reconstruct the path in .zshrc works but shouldn't be required.
