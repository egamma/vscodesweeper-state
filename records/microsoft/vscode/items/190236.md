# Issue #190236 — Git - commit Msg cannot be edited while doing "Edit" during an interactive rebase

[https://github.com/microsoft/vscode/issues/190236](https://github.com/microsoft/vscode/issues/190236)

## Summary

Verified duplicate of #61877: both report that during an interactive rebase 'edit' stop, VS Code blocks commit-message editing with a warning and calls `rebase --continue` instead of `git commit`, preventing the intended amend/edit workflow.

## Change summary

During an interactive rebase 'edit' stop, VS Code prevents editing the commit message and uses `rebase --continue` instead of allowing a normal `git commit`.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/61877, which is the earlier canonical report with more community engagement (16 upvotes, 12 comments). The fix lives in `extensions/git/src/repository.ts`: the input validation at line ~1107 should allow message changes during a rebase edit stop, and the commit path at line ~1426 should use `git commit --amend` with the edited message rather than blindly calling `rebaseContinue()`.
