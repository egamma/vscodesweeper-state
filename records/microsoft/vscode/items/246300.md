# Issue #246300 — No remote SSH for Synology NAS servers: The remote host may not meet VS Code Server's prerequisites for glibc and libstdc+

[https://github.com/microsoft/vscode/issues/246300](https://github.com/microsoft/vscode/issues/246300)

## Summary

Still reproducible on main: `check-requirements-linux.sh` uses `getconf LONG_BIT` (line 48) without a fallback, which fails on Synology NAS and other custom Linux distros that lack `getconf`. With `set -e`, the missing command aborts the entire check, blocking Remote SSH even when glibc/libstdc++ requirements are met. Additionally, the error message the reporter sees about musl may originate from the Remote SSH extension's own prerequisite layer, but the core script's unguarded `getconf` call is a confirmed gap.

## Change summary

VS Code Server's `check-requirements-linux.sh` fails on Synology NAS (and other custom Linux distros missing `getconf`), blocking Remote SSH connections despite the system meeting glibc and libstdc++ requirements.

## Best solution

Add a fallback in `resources/server/bin/helpers/check-requirements-linux.sh` for when `getconf` is unavailable on arm64/aarch64 — e.g. default `BITNESS` to 64 when `getconf` is missing, or detect bitness via `uname -m` / reading ELF headers. This is a small, localized fix in the shell script. A coordinating fix may also be needed in the Remote SSH extension's own prerequisite messaging layer.
