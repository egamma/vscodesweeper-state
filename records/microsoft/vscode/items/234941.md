# Issue #234941 — Git - Built-in Git extension does not respect `includeIf` in a symbolic link directory

[https://github.com/microsoft/vscode/issues/234941](https://github.com/microsoft/vscode/issues/234941)

## Summary

Still reproduces on main: the git extension resolves symlinks via `git rev-parse --show-toplevel` and then uses the resolved real path as CWD for all subsequent git commands, which breaks `includeIf "gitdir:"` directives that match against the symlink path. No fix has been committed.

## Change summary

Git commits via the built-in git extension fail when `includeIf` conditions in `.gitconfig` reference a symlink directory path, because the extension resolves the symlink to the real path before executing git commands.

## Best solution

Preserve the original (symlink) path the user opened and pass it as CWD to git commands instead of the resolved real path from `--show-toplevel`. The fix would likely touch `extensions/git/src/git.ts` (the `exec` method and `Repository` class constructor) and possibly `extensions/git/src/model.ts` (the `openRepository` / `getRepositoryRoot` flow). The related closed issue https://github.com/microsoft/vscode/issues/229433 was about `includeIf` in devcontainers and expired as a feature request — this is a more concrete symlink-specific bug that warrants its own tracking.
