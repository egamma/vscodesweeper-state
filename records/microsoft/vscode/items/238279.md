# Issue #238279 — Hovering over an expression with a `!` (like `foo!.bar`) while debugging does not evaluate the expression

[https://github.com/microsoft/vscode/issues/238279](https://github.com/microsoft/vscode/issues/238279)

## Summary

Still reproduces on main: the regex in `getExactExpressionStartAndEnd` explicitly excludes `!`, so expressions like `a!.length` are broken up during debug hover evaluation. The related Copilot-authored PR #252116 was closed without merging, and a maintainer noted the fix is non-trivial because `!.` is TS-specific syntax requiring expression rewriting.

## Change summary

Hovering over an expression containing TypeScript's non-null assertion operator `!` (e.g. `a!.length`) during debugging fails to evaluate the full expression.

## Best solution

The fix lives in `getExactExpressionStartAndEnd` in `src/vs/workbench/contrib/debug/common/debugUtils.ts` (line 91). Simply removing `!` from the exclusion regex is insufficient — as roblourens noted, `!.` is TS syntax so the evaluated expression must differ from the source text (strip the `!`). A robust approach would handle the `!.` token similarly to how `...` spread is already post-processed (lines 107–113): include `!` in the match, then strip it before evaluation. Alternatively, languages could use the `EvaluatableExpressionProvider` API for language-specific handling.
