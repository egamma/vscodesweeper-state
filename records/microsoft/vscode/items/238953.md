# Issue #238953 — Save as keyboard shortcut does not work in Microsoft Edge when using the browser version

[https://github.com/microsoft/vscode/issues/238953](https://github.com/microsoft/vscode/issues/238953)

## Summary

This is an upstream Edge browser issue: Edge intercepts Ctrl+Shift+S for its built-in screen capture tool before the web page receives the keyboard event, so vscode.dev cannot handle the Save As shortcut. VS Code could potentially offer a browser-specific alternative keybinding, but the root cause is in Edge.

## Change summary

Ctrl+Shift+S (Save As) in vscode.dev is intercepted by Microsoft Edge's screen capture tool, preventing the Save As action.

## Best solution

Classify as `upstream` — the root cause is Edge reserving Ctrl+Shift+S for its screen capture feature. A possible workaround on the VS Code side would be to detect Edge and offer an alternative default keybinding for Save As in the web target, but the real fix belongs in Edge (allowing web apps to override or users to disable the shortcut). The reporter can work around it by rebinding Save As to a different shortcut in vscode.dev's keyboard shortcuts editor.
