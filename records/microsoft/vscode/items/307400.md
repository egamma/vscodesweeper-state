# Issue #307400 — "Terminal process failed to launch" error message is inaccurate

[https://github.com/microsoft/vscode/issues/307400](https://github.com/microsoft/vscode/issues/307400)

## Summary

Valid issue — the 500ms arbitrary threshold in ProcessConstants.ErrorLaunchThresholdDuration causes a misleading 'failed to launch' message for processes that complete quickly but with non-zero exit codes. An open PR (#309177) is attempting to fix this; keeping open pending that PR's review.

## Change summary

Terminal shows misleading 'failed to launch' error for processes that exit with non-zero code within 500ms of starting, even when the launch itself succeeded.

## Best solution

Land PR https://github.com/microsoft/vscode/pull/309177 (or an equivalent fix) that distinguishes genuine launch failures from fast-completing processes with non-zero exit codes — likely by checking launch success signals rather than relying on an arbitrary timeout in `terminalProcessManager.ts`.
