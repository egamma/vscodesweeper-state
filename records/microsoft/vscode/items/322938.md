# Issue #322938 — Copilot Chat renders Ollama streaming reasoning chunks as normal output after briefly showing Thinking UI

[https://github.com/microsoft/vscode/issues/322938](https://github.com/microsoft/vscode/issues/322938)

## Summary

Verified duplicate of the open BYOK reasoning-tokens issue. Both report that streaming `delta.reasoning` chunks from local OpenAI-compatible servers (Ollama, llama-cpp) are not rendered properly in the Thinking UI — consolidating on the earlier, already-labeled canonical issue.

## Change summary

BYOK models streaming reasoning via `choices[0].delta.reasoning` have their thinking text rendered as normal assistant output instead of in the Thinking UI.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/318211 which already carries `bug` and `model-byok` labels and tracks the same underlying defect: the Copilot extension's BYOK stream processing in `extensions/copilot/src/platform/networking/node/stream.ts` and `extensions/copilot/src/platform/thinking/common/thinkingUtils.ts` extracts the `reasoning` field from `choice.delta` but downstream rendering does not surface it correctly in the Thinking UI for BYOK models lacking explicit `thinking: true` configuration.
