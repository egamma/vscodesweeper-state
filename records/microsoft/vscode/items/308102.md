# Issue #308102 — "Go to Symbol in Workspace..." *still* strips all results if query contains a symbol

[https://github.com/microsoft/vscode/issues/308102](https://github.com/microsoft/vscode/issues/308102)

## Summary

Valid request: the normalizeQuery function in fuzzyScorer.ts only strips `*`, ellipsis, whitespace, quotes, and trailing `#` from workspace symbol queries before fuzzy scoring. Characters like `:` and `\` (used as qualifiers by rust-analyzer and PHP IntelliSense) are not stripped, causing returned results to be incorrectly filtered out. Already properly triaged with correct labels.

## Change summary

Workspace symbol search discards all results when the query contains language-server qualifier characters like `:` or `\` because those characters are not stripped before fuzzy scoring.

## Best solution

Extend the normalization regex in `normalizeQuery` (src/vs/base/common/fuzzyScorer.ts line ~909) to also strip characters commonly used as language-server qualifiers (at minimum `:` and `\`), or provide a mechanism for workspace symbol providers to signal which characters are qualifiers that should be ignored during client-side filtering.
