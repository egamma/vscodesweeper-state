# Issue #245421 — Git - timeline updates every second if "Autofetch period" is set to 1

[https://github.com/microsoft/vscode/issues/245421](https://github.com/microsoft/vscode/issues/245421)

## Summary

Still reproduces on main. The issue has milestone 'On Deck' (a scheduling milestone), so it must stay open regardless. The timeline provider refreshes on every successful fetch operation, and with a 1-second autofetch period this causes constant UI churn and focus loss.

## Change summary

Git timeline refreshes every second (losing hover focus) when `git.autofetchPeriod` is set to 1, because each autofetch triggers a non-debounced-enough timeline change event.

## Best solution

In `extensions/git/src/timelineProvider.ts`, `onRepositoryOperationRun` should either skip fetch operations (they don't change the local commit history) or use a longer debounce/deduplication window that suppresses redundant refreshes when the fetch result hasn't actually changed refs. The autofetch operation in `autofetch.ts` uses `silent: true`, so the timeline could check whether refs actually changed before firing.
