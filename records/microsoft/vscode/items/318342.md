# Issue #318342 — Change profile from custom to default caues a never-ending progress badge on explorer activity bar

[https://github.com/microsoft/vscode/issues/318342](https://github.com/microsoft/vscode/issues/318342)

## Summary

Valid bug report: the explorer progress badge gets stuck when switching from the default profile to a custom profile and back. The prior issue #214203 was closed as not-reproduced without a real fix, so this is a recurrence, not a duplicate.

## Change summary

Switching from default profile to a custom profile and back leaves a never-ending progress badge on the Explorer activity bar icon.

## Best solution

Investigate the explorer progress lifecycle during profile switches — likely `getCompleteWorkspace()` in `explorerViewlet.ts:55` or the tree `setInput` promise in `explorerView.ts:833` hangs when the workspace context is torn down/recreated during a profile switch. The fix should ensure progress promises are cancelled or resolved when the profile switch invalidates the underlying context.
