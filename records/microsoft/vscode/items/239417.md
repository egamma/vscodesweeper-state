# Issue #239417 — Inconsistent and confusing display of newlines in file paths

[https://github.com/microsoft/vscode/issues/239417](https://github.com/microsoft/vscode/issues/239417)

## Summary

Still present on main: VS Code doesn't sanitize or escape newline characters in file names for display, causing inconsistent rendering across Explorer, tooltips, and editor tabs. No fix has landed.

## Change summary

File names containing literal newline characters display inconsistently — truncated in the Explorer tree, shown over multiple lines in tooltips, and missing any newline indicator in editor tabs.

## Best solution

Normalize or escape newline characters (and potentially other control characters) when rendering file labels across the icon-label layer (`src/vs/base/browser/ui/iconLabel/`) or at the label-formatting level (`src/vs/base/common/labels.ts`), replacing them with a visible placeholder (e.g. Unicode ␊ or a similar indicator) so all surfaces show the full name on one line consistently.
