# Issue #164469 — Support breaking strings across lines (by extending onEnterRules to have more context?)

[https://github.com/microsoft/vscode/issues/164469](https://github.com/microsoft/vscode/issues/164469)

## Summary

Still an open gap on main: onEnterRules have no token-type awareness, so extensions cannot reliably scope rules to strings. The request has 26 upvotes and is properly in the Backlog.

## Change summary

Extend onEnterRules (or provide an alternative mechanism) so extensions can condition enter-key behavior on token type (e.g. only inside strings).

## Best solution

Add an optional token-type constraint (e.g. `tokenType?: StandardTokenType[]`) to the OnEnterRule interface and plumb it through the onEnter support in `src/vs/editor/common/languages/supports/onEnter.ts`, consulting the line's token information before applying the rule. This keeps the feature in the editor-core layer and unlocks string-splitting for language extensions like Dart.
