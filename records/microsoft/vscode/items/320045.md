# Issue #320045 — Error: VSCode Server fails, if update fails to download

[https://github.com/microsoft/vscode/issues/320045](https://github.com/microsoft/vscode/issues/320045)

## Summary

The report describes a real Remote WSL/server update failure mode, but the current thread does not have enough detail or current-main verification to close it. A maintainer already clarified that the old server cannot be reused with a newer client, so the actionable part is whether the update/install flow can avoid leaving the remote side unrecoverable when the new server download fails.

## Change summary

Remote WSL server update removes the existing server before the replacement is fully downloaded, leaving VS Code unable to reconnect after a failed download.

## Best solution

Keep this open for the remote server/WSL owners to decide whether the CLI/server install path should stage the new server download and checksum before deleting the previous install, or otherwise provide a recovery path when download fails. If the product decision is that rollback requires keeping the old client too, document the recovery path clearly and consider closing as designed only after that decision is explicit.
