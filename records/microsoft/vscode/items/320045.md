# Issue #320045 — Error: VSCode Server fails, if update fails to download

[https://github.com/microsoft/vscode/issues/320045](https://github.com/microsoft/vscode/issues/320045)

## Summary

This is a feature request for better resilience when server downloads fail after the client has already updated. A team member (aeschli) already engaged and explained the architectural constraint — the server must match the client version, so the old server cannot be reused. The download cache code already uses a staging-then-rename pattern (it does NOT rm -rf the old server before downloading), but the core issue is that once the client updates, a failed server download leaves the user stuck.

## Change summary

When VS Code updates its client and the remote server download subsequently fails, the connection is permanently broken with no rollback or retry path.

## Best solution

Implement a pre-download mechanism where the server binary for the new version is fetched and verified before the client fully commits to the update, or provide a client-side rollback that can fall back to the previous version pair (client+server) if the new server download fails. This likely belongs in the CLI tunnel infrastructure (`cli/src/tunnels/code_server.rs`) and possibly the WSL extension's bootstrap logic.
