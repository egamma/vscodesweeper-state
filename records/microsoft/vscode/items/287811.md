# Issue #287811 — edited file link widget opens a read-only file diff

[https://github.com/microsoft/vscode/issues/287811](https://github.com/microsoft/vscode/issues/287811)

## Summary

Team-authored polish issue — must stay open. The core UX gap (inline file-link in chat opens a read-only snapshot diff with no obvious path to the editable file) is partially mitigated by the new `chat.editing.openChangedFileInDiffEditor` setting (commit 12349653b12) but the inline-link flow itself is unchanged. The suggestion of an 'open file' toolbar button in the read-only diff has not been implemented.

## Change summary

Clicking an edited-file link in chat opens a read-only snapshot diff with no obvious way to navigate to the editable version of that file.

## Best solution

Add an 'Open File' action to the editor toolbar when viewing a chat snapshot diff, similar to how other read-only diff contexts provide an escape hatch. The setting from commit 12349653b12 is a workaround but doesn't help users who land in the read-only diff via the inline link and want to edit from there. The fix likely lives in `src/vs/workbench/contrib/chat/browser/chatContentParts/chatChangesSummaryPart.ts` or the snapshot diff editor contribution.
