# Issue #167643 — Setting icon and color in createTerminal API no longer works correctly

[https://github.com/microsoft/vscode/issues/167643](https://github.com/microsoft/vscode/issues/167643)

## Summary

Still reproduces on main per team member confirmation (Dec 2024): the `color` parameter passed via `createTerminal` API is silently ignored for terminal tab coloring. The code path exists but the color is not rendered correctly. Keeping open as a real, confirmed regression bug.

## Change summary

The `color` option in the `createTerminal` extension API does not apply the specified tab color to the terminal tab.

## Best solution

Debug the disconnect between `shellLaunchConfig.color` being set and the color CSS class not being applied to the terminal tab icon. The flow in `terminalInstance.ts` line 683 returns the color, and `getColorClass` in `terminalIcon.ts` should produce the CSS class — likely an issue with how/when the icon-changed event fires or the style element is created for the color. The related issue https://github.com/microsoft/vscode/issues/323231 tracks post-creation color changes but this issue covers initial creation.
