[https://github.com/microsoft/vscode/issues/323531](https://github.com/microsoft/vscode/issues/323531)

## Summary

Real regression confirmed in source: line 395 of inlineCompletionsModel.ts calls `reader.read()` inside a derived that can be re-entered via a transaction triggered by the suggest-widget adapter event path, violating the observables invariant. An open fix PR exists (#323549) but has not yet merged.

## Change summary

Accepting or hiding the suggest widget triggers a transaction that re-enters a derived's compute function, causing an illegal `reader.read()` outside its valid scope — a ~4.3× spike in error telemetry on stable 1.126.0.

## Best solution

Merge the open fix PR (https://github.com/microsoft/vscode/pull/323549) which avoids the reader use after the seeding side-effect, then verify the telemetry bucket drops in the next insiders cycle.
