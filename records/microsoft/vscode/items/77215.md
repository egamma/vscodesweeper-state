# Issue #77215 — Git - Support for gitdir

[https://github.com/microsoft/vscode/issues/77215](https://github.com/microsoft/vscode/issues/77215)

## Summary

Still a valid, unimplemented feature request with 33 upvotes (above the 20-upvote Backlog threshold). The git extension on current main has no support for custom `--git-dir` paths or a `git.gitdirs` setting; the worktree support shipped via #68038 is related but distinct. Keeping open — it's already correctly labeled `help wanted` + `feature-request` in the Backlog.

## Change summary

Allow VS Code's built-in git integration to discover and work with repositories that use a non-standard git directory (via `--git-dir` or a `.git` file pointing to a separate gitdir), and optionally expose a `git.gitdirs` setting for specifying multiple such directories.

## Best solution

Implement gitdir-aware repository discovery in `extensions/git/src/model.ts` (the scanning logic) and ensure `extensions/git/src/git.ts` properly resolves the worktree path from the gitdir config. A `git.gitdirs` workspace setting (similar to `git.scanRepositories`) would let users point at non-standard git directories. The `help wanted` label is appropriate — this is a well-scoped community contribution opportunity.
