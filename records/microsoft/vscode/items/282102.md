# Issue #282102 — Installing an MCP server from a web registry fails silently when a MCP registry policy is set

[https://github.com/microsoft/vscode/issues/282102](https://github.com/microsoft/vscode/issues/282102)

## Summary

Team-authored bug (MEMBER); hard keep-open applies. The issue describes a silent failure when installing an MCP server via deep link from a web registry that isn't in the configured policy. The code has received various error-handling improvements since filing, but the specific scenario of showing an error to the user when registry lookup returns nothing due to policy restrictions may still lack user-facing feedback.

## Change summary

Installing an MCP server from a web registry (github.com/mcp) via deep link fails silently when the server cannot be discovered from the configured registry policy.

## Best solution

Add a user-facing notification/error in the MCP gallery resolution path (around `getMcpServerByName` in `src/vs/platform/mcp/common/mcpGalleryService.ts` or its callers) when server lookup returns `undefined` after a deep-link-initiated install, so users understand why the installation did not proceed.
