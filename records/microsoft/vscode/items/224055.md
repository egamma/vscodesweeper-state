# Issue #224055 — [188] potential listener LEAK detected, having 175 listeners already. MOST frequent listener (1):

[https://github.com/microsoft/vscode/issues/224055](https://github.com/microsoft/vscode/issues/224055)

## Summary

Team-authored error-telemetry issue. The leak path through `observableSignalFromEvent('onMarkerChanged', ...)` in `editorTextPropertySignalsContribution.ts` still exists on main — each cursor-position-change autorun subscribes to the global `onMarkerChanged` event, and with multiple MarkerTextProperty instances the listener count can exceed the threshold before `timeouts.clear()` runs. The earlier fix (013b1861dc7) addressed a different leak (unregistered DisposableStore) but not this accumulation pattern.

## Change summary

Listener leak warning fires on `onMarkerChanged` due to accumulated subscriptions from accessibility signal autoruns created on each cursor move.

## Best solution

The autoruns created at line 148 in `editorTextPropertySignalsContribution.ts` each subscribe to the per-source `observableSignalFromEvent` observable. Consider sharing a single `observableSignalFromEvent` across all MarkerTextProperty instances (since they all listen to the same `onMarkerChanged` event), or raising the listener threshold for that specific event, to prevent the leak warning.
