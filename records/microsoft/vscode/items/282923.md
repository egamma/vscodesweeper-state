# Issue #282923 — Code Tab doesn't scroll up or down.

[https://github.com/microsoft/vscode/issues/282923](https://github.com/microsoft/vscode/issues/282923)

## Summary

Confirmed bug: drag-selecting text in a chat code block that extends beyond the viewport does not auto-scroll the chat view. A MEMBER (roblourens) reproduced it in Dec 2024 and noted it affects keyboard users. No fix found on current main.

## Change summary

Text selection by dragging in a chat code block does not auto-scroll the chat container when the code block extends beyond the visible viewport.

## Best solution

The code block editor has `scrollbar: { vertical: 'hidden' }` (in `src/vs/workbench/contrib/chat/browser/widget/chatContentParts/codeBlockPart.ts`), so auto-scroll during selection needs to be implemented at the chat container (list widget) level — when a mouse-drag selection in an embedded editor reaches the edge of the viewport, the chat scroll position should advance. This would also benefit keyboard selection (Shift+Arrow) as roblourens noted.
