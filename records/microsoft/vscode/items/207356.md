# Issue #207356 — Create Compound Debug Launches Dynamically

[https://github.com/microsoft/vscode/issues/207356](https://github.com/microsoft/vscode/issues/207356)

## Summary

Team-authored feature request for dynamically grouping debug sessions at runtime — kept open per the hard keep-open rule for MEMBER-authored issues. The request describes a real gap in the debug API.

## Change summary

Allow debug sessions to be dynamically grouped after launch so that pause, continue, stop, and terminate apply to the entire group simultaneously.

## Best solution

Extend the `vscode.debug` API with a session-grouping mechanism that lets extensions (or the user) add running debug sessions to a compound group post-launch, with synchronized pause/continue/stop/terminate. This belongs in `src/vs/workbench/contrib/debug/` and the extension API surface. The related issue https://github.com/microsoft/vscode/issues/137953 tracks the synchronized control aspect and should be considered together.
