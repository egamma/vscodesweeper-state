# Issue #125774 — Allow partial string replacement in package.json/package.nls.json, to ease translations of strings containing markdown

[https://github.com/microsoft/vscode/issues/125774](https://github.com/microsoft/vscode/issues/125774)

## Summary

Team-authored feature request for partial string replacement in package.nls.json to protect markdown syntax from translator breakage. Still open — the workaround (OneLoc locked-string syntax) is Microsoft-specific and doesn't solve the general tooling gap. Kept open per hard keep-open rule (MEMBER author).

## Change summary

Allow partial string replacement (placeholders) in package.json/package.nls.json so markdown-containing strings can be localized without corrupting their syntax.

## Best solution

Implement placeholder support (e.g. `{0}`, `{1}`) in the package.nls.json resolution logic, consistent with how `nls.localize` already handles placeholders at runtime. The l10n-platform area owner should decide whether this belongs in vscode-l10n tooling or in the core package.json resolver.
