# Issue #186866 — SIngle terminal tabs is re-rendered several times on start up

[https://github.com/microsoft/vscode/issues/186866](https://github.com/microsoft/vscode/issues/186866)

## Summary

The redundant re-creation of the single terminal tab action view item was addressed in commit 245b24a9587 which stopped caching via MutableDisposable (the source of 'multiple disposes' warnings) and the follow-up 8770675b06a which properly manages action item disposables in a map. Shipped in 1.98.0.

## Change summary

Single terminal tab action view item was re-created multiple times on startup due to repeated calls to getActionViewItem.

## Best solution

Keep the shipped fix — the single tab is no longer cached in a MutableDisposable that fought with repeated createActionViewItem calls; disposable management is now handled via a map. No further change needed.
