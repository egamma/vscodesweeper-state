# Issue #32840 — Cannot display emoji ✔️, 🙋‍♂️

[https://github.com/microsoft/vscode/issues/32840](https://github.com/microsoft/vscode/issues/32840)

## Summary

Keep open as an upstream font-rendering issue: maintainer investigation tied the behavior to Chromium/Electron rendering, and later comments show related emoji rendering gaps persisted across OSes after the initial Windows case improved.

## Change summary

Emoji sequences such as ✔️, 🙋‍♂️, ⚠️, ‼️, and 🎉 do not consistently render with the expected color emoji glyphs in VS Code editor and related UI surfaces unless users adjust font fallbacks.

## Best solution

Leave this tracked under the existing Chromium/font-rendering ownership until VS Code can either rely on an Electron/Chromium fix or make a deliberate editor font fallback decision without regressing custom text fonts. The known workaround is appending platform emoji fonts to `editor.fontFamily`, but that does not fully settle all UI surfaces mentioned in the thread.
