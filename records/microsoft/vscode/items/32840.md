# Issue #32840 — Cannot display emoji ✔️, 🙋‍♂️

[https://github.com/microsoft/vscode/issues/32840](https://github.com/microsoft/vscode/issues/32840)

## Summary

Still valid on current main: the default editor font families (Consolas/Menlo/Droid Sans Mono) do not include color emoji fonts in the fallback chain. The issue has 47 upvotes, a known workaround, and is correctly labeled upstream/chromium.

## Change summary

Emoji characters render as monochrome glyphs or broken symbols because VS Code's default font-family lacks color emoji font fallbacks.

## Best solution

Add platform-appropriate color emoji fonts ('Segoe UI Emoji', 'Apple Color Emoji', 'Noto Color Emoji') to the default font-family constants in `src/vs/editor/common/config/fontInfo.ts`. This is the fix commenters have proposed since 2017 and is what GitHub.com does in its own CSS. Alternatively, wait for Chromium/Electron to handle emoji fallback natively, but that hasn't happened in 9 years.
