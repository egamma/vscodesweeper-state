# Issue #306770 — Gituhb Copilot Chat: it's too easy to lose changes on ctrl+backspace

[https://github.com/microsoft/vscode/issues/306770](https://github.com/microsoft/vscode/issues/306770)

## Summary

Still reproduces on main: `Ctrl+Backspace` is bound to `chatEditing.discardAllFiles` when the chat input is empty, making it trivially easy to accidentally discard all edits while clearing text. The keybinding and its when-clause are unchanged on current main. A related issue https://github.com/microsoft/vscode/issues/317320 reports the same problem and should be consolidated here.

## Change summary

Ctrl+Backspace on an empty chat input triggers the discard-all-edits confirmation, which can be accidentally confirmed, causing loss of applied changes.

## Best solution

Change the keybinding for `chatEditing.discardAllFiles` from `Ctrl+Backspace` to a safer chord (e.g. `Ctrl+Shift+Backspace`) in `src/vs/workbench/contrib/chat/browser/chatEditing/chatEditingActions.ts` line 246, so that rapid text-clearing keystrokes cannot accidentally trigger the destructive action. Issue https://github.com/microsoft/vscode/issues/317320 is a duplicate and should be closed against this one.
