# Issue #105774 — Using `moveViews` command with multiple windows open can cause not all views to move

[https://github.com/microsoft/vscode/issues/105774](https://github.com/microsoft/vscode/issues/105774)

## Summary

The reported race condition in multi-window view movement via storage sync is plausible in the current code (the `onDidStorageChange` handler at line 575 of `viewDescriptorService.ts` still reacts to cross-window storage updates), but the storage handling has been significantly reworked since the report (batched moves, unified customizations storage). No reproduction steps or follow-up confirms this still occurs. Keeping open as a known potential issue in multi-window view sync.

## Change summary

Using the `moveViews` API command with multiple VS Code windows open can cause some view moves to be undone by a cross-window `onDidStorageChange` event.

## Best solution

Investigate whether the current `onDidStorageChange` guard in `viewDescriptorService.ts` (line 576) is sufficient to prevent cross-window undo of programmatic view moves, or whether the `moveViewsToContainer` path needs to suppress or debounce the storage-change listener during batch operations.
