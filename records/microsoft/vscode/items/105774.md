# Issue #105774 — Using `moveViews` command with multiple windows open can cause not all views to move

[https://github.com/microsoft/vscode/issues/105774](https://github.com/microsoft/vscode/issues/105774)

## Summary

Still plausible on current main: the `vscode.moveViews` command iterates views one-by-one, calling `saveViewCustomizations()` after each move; in a multi-window scenario, the other window's `onDidStorageChange` handler processes each intermediate state and may undo subsequent moves before they happen.

## Change summary

Using `vscode.moveViews` with multiple windows open can cause some views to not move because intermediate storage writes trigger `onDidStorageChange` in other windows that undo pending moves.

## Best solution

Batch the storage save in the `MoveViews` command loop — either move all views with `moveViewsWithoutSaving` and call `saveViewCustomizations()` once at the end, or add a suppression flag during bulk moves. The fix is localized to `src/vs/workbench/browser/parts/views/viewPaneContainer.ts` (the command loop) and possibly `src/vs/workbench/services/views/browser/viewDescriptorService.ts` (expose a batch-move API).
