# Issue #299267 — Allow filtering Git worktree list per window to show only the active worktree

[https://github.com/microsoft/vscode/issues/299267](https://github.com/microsoft/vscode/issues/299267)

## Summary

Valid feature request for a setting to limit worktree visibility in Source Control to the active worktree per window. The current `git.detectWorktrees` setting auto-opens all sibling worktrees, with no built-in way to scope per window. Community interest is growing (3 upvotes + 3 commenters confirming the pain). Not a duplicate — #313526 is related but focuses on "Close Other Repositories" being non-functional for worktrees, a distinct (complementary) problem.

## Change summary

Add a setting to filter the Source Control Repositories view to show only the worktree that is open in the current window, rather than all detected worktrees.

## Best solution

Add a setting like `git.detectWorktrees.scope` (or similar) in `extensions/git/` that limits auto-opened worktrees to only the one matching the current workspace folder. The detection logic in `extensions/git/src/model.ts` (`checkForWorktrees` around line 806) would need to be gated so it skips opening sibling worktrees when the setting is active. Alternatively, make the existing "Close Other Repositories" actually persist for worktrees (tracked separately in #313526). Both improvements would complement each other.
