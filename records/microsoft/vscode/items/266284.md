# Issue #266284 — Extremely large prompts are completely removed, along with the system prompt.

[https://github.com/microsoft/vscode/issues/266284](https://github.com/microsoft/vscode/issues/266284)

## Summary

Still appears valid on main: when a user pastes extremely large text directly into chat (not as an attachment), the prompt-tsx token budget pruning can strip both the user message and system prompt, producing an empty request. No targeted fix for this edge case has landed.

## Change summary

Pasting extremely large text directly into chat causes the entire prompt (including system prompt) to be removed instead of gracefully truncating the user input.

## Best solution

Guard the prompt construction so that the system prompt is never pruned regardless of user input size — either cap/truncate the raw query text before it enters the prompt-tsx renderer, or ensure the SystemMessage's priority=1000 is respected even when a single sibling element overflows the budget. The fix likely belongs in `extensions/copilot/src/extension/prompts/node/panel/panelChatBasePrompt.tsx` or in the `@vscode/prompt-tsx` library's pruning logic.
