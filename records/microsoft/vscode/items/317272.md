## Change summary

The uncommitted changes confirmation widget in the chat/agents flow uses overly technical language, lacks diff markers, doesn't remember the user's choice, and interrupts the prompt flow.

## Best solution

Improve the uncommitted changes widget along the three axes outlined: (1) replace technical jargon (commits, worktrees) with user-intent language and name the specific repository, (2) add diff markers and remember the user's choice, (3) explore moving the confirmation into the chat view itself rather than an inline blocking widget.
