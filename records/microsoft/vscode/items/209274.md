# Issue #209274 — Unicode character displayed as a question mark ?

[https://github.com/microsoft/vscode/issues/209274](https://github.com/microsoft/vscode/issues/209274)

## Summary

Confirmed Chromium upstream issue: fullwidth Unicode characters (e.g. U+FF02) fail to render on Linux due to Chromium's font fallback not finding an appropriate glyph. Reproduces in Chrome/Edge but not Firefox or native editors. Already labeled `chromium`; adding `upstream` for clarity. VS Code cannot fix this in its own code.

## Change summary

Fullwidth quotation mark U+FF02 (and similar characters) render as question marks or tofu boxes on Linux in VS Code.

## Best solution

Track the upstream Chromium font fallback bug (Chromium's text shaper on Linux does not fall back to fonts that contain fullwidth forms the way Firefox/GTK apps do). VS Code can't fix this — it depends on Electron picking up a Chromium fix. Consider linking to a Chromium bug tracker issue if one exists. In the meantime, users can work around it by installing a font that covers the fullwidth Unicode block (e.g. Noto Sans CJK) and setting it in `editor.fontFamily`.
