# Issue #324295 — Sessions from yesterday render "1hr ago"

[https://github.com/microsoft/vscode/issues/324295](https://github.com/microsoft/vscode/issues/324295)

## Summary

Team-authored bug — keep open. The `updatedAt` observable is initialized with `new Date()` (current time) in multiple session model classes, so until the real server timestamp arrives via `update()`, sessions display an incorrect relative time. Opening/loading a session likely bumps the displayed timestamp.

## Change summary

Sessions from yesterday incorrectly render as '1hr ago' because the updatedAt observable is initialized to the current time rather than the persisted timestamp.

## Best solution

Initialize `_updatedAt` in the session model classes (lines ~231, 559, 806 of `copilotChatSessionsProvider.ts`) with the actual persisted timestamp rather than `new Date()`, or ensure the real timestamp is set synchronously before the UI renders.
