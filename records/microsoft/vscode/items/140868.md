# Issue #140868 — [linked editing] stops when tag name gets empty and end tag is on same line

[https://github.com/microsoft/vscode/issues/140868](https://github.com/microsoft/vscode/issues/140868)

## Summary

Still reproduces on main: linked editing clears ranges when the tag name becomes empty because the word pattern check in _syncRanges rejects empty strings. The bug is in the pattern validation logic that doesn't allow empty intermediate states during editing.

## Change summary

Linked editing (synced regions) stops synchronizing opening/closing HTML tags when the tag name is fully deleted during editing, breaking the rename flow.

## Best solution

Allow empty reference values as a valid intermediate editing state in _syncRanges — skip the word-pattern check when referenceValue is empty, so linked editing remains active while the user types a new tag name.
