# Issue #82087 — Scrollbar in editor is off

[https://github.com/microsoft/vscode/issues/82087](https://github.com/microsoft/vscode/issues/82087)

## Summary

Still reproduces on main: the overview ruler decoration positions use a simple linear heightRatio that doesn't account for the scrollbar slider's artificial 20px minimum enlargement, causing decorations to misalign with scroll positions. Author is a team member and milestone is 'On Deck' — keeping open.

## Change summary

Clicking an error decoration in the scrollbar overview ruler navigates to the wrong position because the decoration and slider use different coordinate scales.

## Best solution

The `heightRatio` calculation in `decorationsOverviewRuler.ts` (line 388) needs to account for the slider's minimum 20px clamping from `scrollbarState.ts`. The decoration positions should be drawn using the same distorted coordinate system the scrollbar uses, or the slider state information needs to be exposed so the overview ruler can compensate.
