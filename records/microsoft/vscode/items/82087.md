# Issue #82087 — Scrollbar in editor is off

[https://github.com/microsoft/vscode/issues/82087](https://github.com/microsoft/vscode/issues/82087)

## Summary

Still reproduces on current main: the decorations overview ruler paints error/warning indicators on a different coordinate scale than the scrollbar slider, because the slider is clamped to a minimum 20px height but the decoration positions use a simple linear ratio that doesn't account for this clamping. The mismatch grows with file length.

## Change summary

Clicking an error decoration in the scrollbar overview ruler navigates to the wrong position because decoration indicators and the scrollbar slider are drawn on mismatched coordinate scales.

## Best solution

The decoration position calculation in `src/vs/editor/browser/viewParts/overviewRuler/decorationsOverviewRuler.ts` (line 388, `heightRatio = canvasHeight / outerHeight`) needs to account for the slider's artificial enlargement from `MINIMUM_SLIDER_SIZE` (20px) in `src/vs/base/browser/ui/scrollbar/scrollbarState.ts`. The decoration Y coordinate should map through the same coordinate transform the scrollbar uses, so that a decoration at a given line visually aligns with the slider position when that line is at the top of the viewport.
