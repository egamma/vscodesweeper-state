# Issue #82087 — Scrollbar in editor is off

[https://github.com/microsoft/vscode/issues/82087](https://github.com/microsoft/vscode/issues/82087)

## Summary

Keep open: this is a maintainer-triaged editor scrollbar bug with a real `On Deck` milestone, so the cleanup rules defer to the scheduling decision and do not propose closure.

## Change summary

Clicking an error marker in the editor overview ruler can scroll to a position where the diagnostic is not centered or visible because marker and scrollbar viewport positions use different scaling when the scrollbar thumb is clamped to a minimum size.

## Best solution

Editor scrollbar/overview ruler ownership should decide whether to align overview ruler decoration hit targets with the clamped scrollbar thumb geometry, likely around the editor view parts and scrollbar state plumbing already discussed in `src/vs/editor/browser/view/viewImpl.ts`.
