## Summary

Fixed on main: the fallback URI was changed from path-less `unknown://` to `unknown:///` (with root path `/`) so that downstream `joinPath` calls no longer throw. Continuing telemetry hits come from older Agents builds that predate the fix.

## Change summary

Sessions without a git repository used a path-less `unknown://` fallback URI for the workspace folder, causing `joinPath` to throw in multiple downstream consumers (extensionsConfig, MCP tool registration, debug, preferences, etc.).

## Best solution

Keep the shipped fix in commit `8c98219ee80` (1.118.0). No further change needed — the error will disappear as users update past the old Agents builds.
