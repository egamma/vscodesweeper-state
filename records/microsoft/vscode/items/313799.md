# Issue #313799 — GH Copilot Error - EM too many open files

[https://github.com/microsoft/vscode/issues/313799](https://github.com/microsoft/vscode/issues/313799)

## Summary

Root cause identified by the reporter: the Copilot CLI accumulates unbounded session-state files under ~/.copilot/session-state, exhausting the process file-descriptor limit. The fix belongs to Copilot CLI (github/copilot-cli#3892), not VS Code core. Keeping open — team-authored issue with active investigation.

## Change summary

VS Code crashes with EMFILE errors because the Copilot CLI never prunes ~/.copilot/session-state directories, exhausting the ~8,189 fd limit on Windows.

## Best solution

The real fix (session-state retention policy and bounded fd usage) belongs in the Copilot CLI / Copilot extension, tracked at https://github.com/github/copilot-cli/issues/3892. VS Code core has no action here beyond potentially surfacing a clearer error when EMFILE is hit. A maintainer should confirm the Copilot team owns the fix and consider re-labeling as `*caused-by-extension` once the upstream fix ships.
