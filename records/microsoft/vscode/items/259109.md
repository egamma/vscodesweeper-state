# Issue #259109 — Jump to line

[https://github.com/microsoft/vscode/issues/259109](https://github.com/microsoft/vscode/issues/259109)

## Summary

Valid feature request: chat code blocks that reference existing source files (via `codemapperUri`) lack a toolbar action to reveal/jump to that location in the editor. The infrastructure (`codemapperUri`) already exists but no 'Go to File' action is wired up for regular code blocks — only compare blocks have `OpenCompareBlockInDiffEditor` with `Codicon.goToFile`. Keeping open as a real gap.

## Change summary

Add a 'Go to File' / 'Jump to Line' toolbar action on chat code blocks so users can navigate directly to the referenced source location.

## Best solution

Add a new `ChatCodeBlockAction` (e.g. 'Go to File') in `src/vs/workbench/contrib/chat/browser/actions/chatCodeblockActions.ts` that opens the `codemapperUri` via `IEditorService` and reveals the relevant range. The action should appear in the code block toolbar (alongside Apply in Editor / Insert at Cursor) and be enabled only when `codemapperUri` is set. The plumbing already exists: the `ICodeBlockActionContext` carries `codemapperUri`, and `codeBlockOperations.ts` already demonstrates opening it.
