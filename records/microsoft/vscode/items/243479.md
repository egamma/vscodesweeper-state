# Issue #243479 — Git - Add support for cloning a repository from Azure Repos

[https://github.com/microsoft/vscode/issues/243479](https://github.com/microsoft/vscode/issues/243479)

## Summary

The 'Clone from GitHub' entry is contributed by the built-in GitHub extension via the `RemoteSourceProvider` API — any extension can register its own clone source the same way. Azure DevOps/Repos clone support belongs in an extension (Microsoft already ships the 'Azure Repos' extension), not in VS Code core.

## Change summary

Add a built-in 'Clone from Azure Repos' option alongside 'Clone from GitHub' in the Source Control clone picker.

## Best solution

Install Microsoft's Azure Repos extension (or a similar extension) which can register a `RemoteSourceProvider` via the Git extension API (`registerRemoteSourceProvider`) to add 'Clone from Azure Repos' to the clone picker. No core change needed — the extension point already exists and works.
