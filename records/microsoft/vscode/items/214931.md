# Issue #214931 — Quotes around paths with spaces are not honored when running tasks

[https://github.com/microsoft/vscode/issues/214931](https://github.com/microsoft/vscode/issues/214931)

## Summary

Still reproduces on main: when a task command variable resolves to a path with spaces that is already quoted, the `needsQuotes` function returns false, so `commandQuoted` stays false and the PowerShell `& ` call-operator prefix is never added — causing the shell to misinterpret the command. The bug is confirmed by recent comments (July–August 2025) and #246083 was closed as a duplicate of this issue.

## Change summary

Shell task execution fails when the resolved command path contains spaces because the quoting/escaping logic does not correctly handle already-quoted paths or ensure the PowerShell call operator is applied.

## Best solution

Fix `_buildShellCommandLine` in `src/vs/workbench/contrib/tasks/browser/terminalTaskSystem.ts` to detect when a resolved command contains spaces and is already quoted, and still prepend the `& ` call operator for PowerShell. The `needsQuotes` check needs to differentiate between 'already safe to pass as-is' and 'needs a call operator despite being quoted'. The Copilot agent branch `copilot/fix-265274` has related work that hasn't landed on `main`.
