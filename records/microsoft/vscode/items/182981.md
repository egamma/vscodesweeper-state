# Issue #182981 — Incorrectly placed diff scrollbar

[https://github.com/microsoft/vscode/issues/182981](https://github.com/microsoft/vscode/issues/182981)

## Summary

Still present on main: the diff editor's overview ruler is absolutely positioned at top:0 and doesn't offset for banners shown inside the child editors (e.g. the unicode highlight banner), causing it to overlap the banner's close button.

## Change summary

Diff editor scrollbar/overview ruler overlaps the 'Non ASCII Highlight' banner's close button because it doesn't account for banner height.

## Best solution

The `OverviewRulerFeature` in `src/vs/editor/browser/widget/diffEditor/features/overviewRulerFeature.ts` should read the banner height from the modified editor's layout info and offset its `top` style and ruler layout accordingly, similar to how the code editor internally accounts for banner height in its own layout.
