# Issue #170708 — Cannot paste full file paths after copying file from file explorer on Linux

[https://github.com/microsoft/vscode/issues/170708](https://github.com/microsoft/vscode/issues/170708)

## Summary

Keep open: current main has partial generic support for pasting `text/uri-list` as paths, but later reports show the issue changed rather than clearly resolved, and the terminal paste path is not covered by the editor paste providers.

## Change summary

Copying a file from the OS file manager and pasting into VS Code on Linux should paste the full absolute file path instead of only the name, a relative path, or a Markdown link.

## Best solution

Validate current Insiders on Linux file managers such as Nautilus and Dolphin, then fix the path through the editor paste data-transfer layer and terminal clipboard path so external file-manager copies consistently expose an absolute path option/default where appropriate.

## Evidence

- **linked prior issue:** The report explicitly says this is the same class of problem as https://github.com/microsoft/vscode/issues/146770, but maintainer comment says it was still partially broken for Linux despite that prior issue.
- **partial implementation on main:** Commit 9ab88c7abc4d90e67c94ac341bc0fee7679c5556 ('Add default uri-list paste providers') added default absolute and relative `text/uri-list` paste providers; current `src/vs/editor/contrib/dropOrPasteInto/browser/defaultProviders.ts` registers `PathProvider` and `RelativePathProvider` for generic editor paste.
- **post-fix reports:** Comments after that commit report continued or changed behavior in VS Code 1.83.1 and 1.87.0, including relative paths and `[text](../../example.txt)` being inserted, so the original full-path expectation is not clearly satisfied.
- **terminal gap:** The issue includes pasting into the VS terminal; current terminal code in `src/vs/workbench/contrib/terminal/browser/xterm/xtermTerminal.ts` wires xterm clipboard reads through `IClipboardService.readText`, not the editor `text/uri-list` paste providers.
