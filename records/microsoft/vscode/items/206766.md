[https://github.com/microsoft/vscode/issues/206766](https://github.com/microsoft/vscode/issues/206766)

## Summary

Valid feature request with 92 upvotes, promoted to Backlog. RISC-V support depends on upstream Node.js and Electron officially supporting the architecture — partial progress has been made (ripgrep-prebuilt PR merged), but core blockers remain. Keeping open as the tracking issue.

## Change summary

Add RISC-V (riscv64) architecture support for VS Code, particularly for remote server (vscode-reh) connections.

## Best solution

Continue tracking upstream dependencies: Node.js official riscv64 support (https://github.com/nodejs/build/issues/2876), Chromium riscv64 support, and Electron riscv64 support (https://github.com/electron/electron/issues/38892). The ripgrep-prebuilt dependency was resolved (PR merged June 2025). Once Node.js officially supports riscv64, VS Code server (REH) builds become viable; full desktop support additionally requires Electron/Chromium. This issue should remain open as the canonical tracker.
