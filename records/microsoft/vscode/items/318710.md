# Issue #318710 — File view still thinks it is on unknown although a folder is selected for the session

[https://github.com/microsoft/vscode/issues/318710](https://github.com/microsoft/vscode/issues/318710)

## Summary

Team-authored bug report (hard keep-open). The file view in the Agents window does not update from 'unknown' state when a workspace folder is selected for a session, likely a stale context key issue with `sessionHasWorkspace` or the workspace folder count context.

## Change summary

The file view in the sessions/agents window still displays an 'unknown' state after a folder has been selected for the session.

## Best solution

Investigate the context key binding for `SessionHasWorkspaceContext` and `WorkspaceFolderCountContext` — the file view visibility depends on these via the `when` clause in `src/vs/sessions/contrib/files/browser/files.contribution.ts` (lines 67, 79). The bug is likely that these context keys are not updated (or updated too late) when the workspace folder changes for a session. The fix would be in whatever service sets `SessionHasWorkspaceContext` to ensure it reacts to folder selection events.
