# Issue #11462 — Debug console architecture

[https://github.com/microsoft/vscode/issues/11462](https://github.com/microsoft/vscode/issues/11462)

## Summary

The debug console still uses WorkbenchAsyncDataTree on current main — the core architectural rethinking discussed here was never implemented, though many of the specific motivating issues (fonts, performance, selection) have been addressed individually over the years. With 16 upvotes, it sits just below the 20-upvote backlog threshold.

## Change summary

Rearchitect the debug console from a tree-based widget to an editor-based or hybrid approach, enabling better find, selection, word wrap, and performance for large outputs.

## Best solution

This is a long-standing architectural discussion that has been partially addressed through incremental improvements. The remaining value is in the hybrid editor+tree approach (Option 2) for better find/selection/copy across output. The debug team should assess whether the current tree-based architecture still has meaningful gaps that justify the investment, or whether this can be closed as sufficiently addressed by the years of incremental work. Related: https://github.com/microsoft/vscode/issues/197287 tracks debug visualization extension points which partially overlaps.
