# Issue #174195 — Git - Commit Staged (Amend) doesn't change the commit message if there are no staged changes but there are changes

[https://github.com/microsoft/vscode/issues/174195](https://github.com/microsoft/vscode/issues/174195)

## Summary

Fixed on main and shipped: commit 3c7cbd264e4 added an `!opts.amend` guard so that 'Commit Staged (Amend)' no longer bails when there are unstaged changes but no staged changes — the amend proceeds and updates the commit message as expected.

## Change summary

'Commit Staged (Amend)' silently did nothing when there were unstaged changes but no staged changes, instead of amending the commit message.

## Best solution

Keep the shipped fix (1.92.0); no further change is needed. The amend path now correctly bypasses the 'no staged changes' gates.
