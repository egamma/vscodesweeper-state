# Issue #199474 — Feature Request: Add setting to disable the default wrapping around navigation for workbench.action.compareEditor.nextChange

[https://github.com/microsoft/vscode/issues/199474](https://github.com/microsoft/vscode/issues/199474)

## Summary

Valid feature request from a team member, well above the 20-upvote threshold, and unimplemented on current main — the wrap-around is still hardcoded. Keeping open per MEMBER authorship and community traction.

## Change summary

Add a setting to disable wrap-around when navigating changes in the diff editor via nextChange/previousChange commands.

## Best solution

Add a `diffEditor.navigation.wrap` boolean setting (default true for back-compat). In `src/vs/editor/browser/widget/diffEditor/diffEditorWidget.ts` `goToDiff()`, when the setting is false, stop at the last/first diff instead of wrapping (replace the `?? diffs[0]` / `?? diffs[diffs.length - 1]` fallbacks with a no-op or a notification). Optionally surface a 'reached last change' notification when wrapping is disabled.
