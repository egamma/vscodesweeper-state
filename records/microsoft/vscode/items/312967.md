# Issue #312967 — Ollama BYOK agent conversation compaction appears to use incorrect token accounting/budget

[https://github.com/microsoft/vscode/issues/312967](https://github.com/microsoft/vscode/issues/312967)

## Summary

The reporter's analysis is correct: OpenAIEndpoint.cloneWithTokenOverride sets maxInputTokens on the top-level metadata object, but the parent ChatEndpoint constructor reads capabilities.limits.max_prompt_tokens — a different field — so the token budget override is a no-op for BYOK endpoints. This breaks conversation compaction triggering for Ollama and other BYOK providers.

## Change summary

BYOK/Ollama agent conversation compaction does not trigger at the correct point because the token budget override writes to a field that the endpoint constructor never reads.

## Best solution

Fix OpenAIEndpoint.cloneWithTokenOverride to update capabilities.limits.max_prompt_tokens (matching ChatEndpoint.cloneWithTokenOverride's pattern) instead of top-level maxInputTokens. The secondary issue of local tokenizer mismatch vs. server-reported prompt_eval_count is a deeper design gap that may warrant a separate tracking issue.
