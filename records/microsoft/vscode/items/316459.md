# Issue #316459 — Remote-WSL 0.104.3 fails with `navigator is now a global in nodejs` / `PendingMigrationError`, then WebSocket 1006 when connecting to WSL

[https://github.com/microsoft/vscode/issues/316459](https://github.com/microsoft/vscode/issues/316459)

## Summary

The failure is in the Remote - WSL extension, not VS Code core: the stack trace points into `ms-vscode-remote.remote-wsl-0.104.3\dist\node\extension.js`, and the `PendingMigrationError` is VS Code's extension-host guard for extensions still using the now-global Node.js `navigator`. This should be handled in the Remote - WSL / remote-release extension tracker rather than kept open in `microsoft/vscode`.

## Change summary

Remote - WSL 0.104.3 crashes during WSL authority resolution after reading `navigator`, leading to a failed WebSocket connection and no remote filesystem provider.

## Best solution

Fix or update the Remote - WSL extension so it no longer reads Node.js global `navigator` in the extension host, and track any remaining WSL connection failure in the Remote - WSL extension issue tracker.
