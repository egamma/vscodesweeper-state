# Issue #316459 — Remote-WSL 0.104.3 fails with `navigator is now a global in nodejs` / `PendingMigrationError`, then WebSocket 1006 when connecting to WSL

[https://github.com/microsoft/vscode/issues/316459](https://github.com/microsoft/vscode/issues/316459)

## Summary

The connection failure is in the Remote-WSL extension (ms-vscode-remote.remote-wsl-0.104.3), not VS Code core. The `navigator` PendingMigrationError is an intentional diagnostic warning in VS Code's extension host (not a crash), and the WebSocket 1006 failure originates from the extension's resolver logic. This belongs in the vscode-remote-release issue tracker.

## Change summary

Remote-WSL extension fails to establish a WebSocket connection to WSL, resulting in error code 1006 and inability to register the remote filesystem provider.

## Best solution

File this issue at https://github.com/microsoft/vscode-remote-release/issues where the Remote-WSL extension team can investigate the WebSocket 1006 failure in their resolver. The `navigator` warning is a red herring — it's an intentional VS Code diagnostic (see `src/vs/workbench/api/node/extensionHostProcess.ts`) that logs when extensions access `navigator` in Node.js, and does not cause the connectivity failure.
