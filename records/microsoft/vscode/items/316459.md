# Issue #316459 — Remote-WSL 0.104.3 fails with `navigator is now a global in nodejs` / `PendingMigrationError`, then WebSocket 1006 when connecting to WSL

[https://github.com/microsoft/vscode/issues/316459](https://github.com/microsoft/vscode/issues/316459)

## Summary

Keep open and route to the remote connection area; the report is detailed and points at a Remote-WSL extension-host compatibility problem, but there is not enough evidence to close it as fixed, duplicate, or caused by a third-party extension.

## Change summary

Remote-WSL 0.104.3 fails to establish the WSL remote extension host after a Node global `navigator` migration warning, ending in WebSocket 1006 and no remote filesystem provider.

## Best solution

Remote/WSL owners should determine whether `ms-vscode-remote.remote-wsl` still probes `navigator` in its local extension-host path and either update the extension or document a temporary `extensions.supportNodeGlobalNavigator` workaround if appropriate.

## Evidence

- **issue detail:** The report includes VS Code commit 0958016b2af9f09bb4257e0df4a95e2f90590f9f, Remote-WSL 0.104.3, Windows 11, Ubuntu-24.04 WSL, repeated WebSocket 1006 failures, and the stack path `ms-vscode-remote.remote-wsl-0.104.3\dist\node\extension.js`.
- **core behavior:** Current main still defines the migration guard in `src/vs/workbench/api/node/extensionHostProcess.ts`: when `--supportGlobalNavigator` is not passed, reading `globalThis.navigator` reports `PendingMigrationError` and returns `undefined`.
- **workaround/config path:** `src/vs/platform/extensions/electron-main/extensionHostStarter.ts` and `src/vs/server/node/extensionHostConnection.ts` pass `--supportGlobalNavigator` only when the `extensions.supportNodeGlobalNavigator` setting is enabled.
- **history:** Commit 7cc3d44323cd61f3368da74655d4ca1216665a3f introduced the Node `navigator` polyfill/migration behavior, and commit 0140ab3f1b3 updated built-in extensions to avoid using `navigator` for web detection; Remote-WSL is not present in this checkout, so no core fix for this specific extension can be confirmed.
