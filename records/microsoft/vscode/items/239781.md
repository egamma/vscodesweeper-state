# Issue #239781 — Runtime error when renaming an existing folder into a nested folder structure

[https://github.com/microsoft/vscode/issues/239781](https://github.com/microsoft/vscode/issues/239781)

## Summary

Still reproduces on main. Renaming folder `foo` to `foo/bar` constructs a target URI that is a child of the source, creating an impossible self-nesting move that throws a runtime error. No fix has landed.

## Change summary

Renaming an existing folder to include a path separator (e.g. `foo` → `foo/bar`) causes a runtime error because the resulting bulk edit tries to move the folder inside itself.

## Best solution

Add a guard in `validateFileName` (or in the `onFinish` callback of `renameHandler`) in `src/vs/workbench/contrib/files/browser/fileActions.ts` to detect when the target resource would be equal to or a child of the source resource, and either block it with a validation error or handle the nested rename as a mkdir + move.
