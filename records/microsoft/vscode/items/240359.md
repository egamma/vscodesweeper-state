# Issue #240359 — Support translations in Code OSS

[https://github.com/microsoft/vscode/issues/240359](https://github.com/microsoft/vscode/issues/240359)

## Summary

The gap remains on current main: web localization still requires `nlsCoreBaseUrl` (a product-specific unpkg endpoint) with no self-hostable alternative for Code OSS builds. A team member acknowledged this needs improvement.

## Change summary

Allow translations/language packs to work in web builds of Code OSS without depending on Microsoft's proprietary unpkg service.

## Best solution

Provide a documented, self-hostable mechanism for serving `nls.messages.js` from unpacked language packs in Code OSS web builds — likely a local file-based fallback in `webClientServer.ts` or a documented script to generate the NLS bundle from extension vsix files.
