# Issue #309170 — Provide an option to suppress GitHub access prompts from extensions

[https://github.com/microsoft/vscode/issues/309170](https://github.com/microsoft/vscode/issues/309170)

## Summary

Valid feature request. VS Code already has Allow/Deny per-extension in the auth prompt and a 'Manage Trusted Extensions' action, but the passive flow in mainThreadAuthentication.ts does not check whether access was previously denied before re-triggering the account badge — so the badge can reappear after restart even when the user already clicked 'Deny'. The centralized management UI also isn't easily discoverable.

## Change summary

Provide a way to permanently suppress GitHub authentication access prompts from extensions that a user has already denied.

## Best solution

In `src/vs/workbench/api/browser/mainThreadAuthentication.ts` around line 524-528, check `isAccessAllowed` for each session before calling `requestSessionAccess` — if all sessions return `false` (explicitly denied), skip the badge. Additionally, improve discoverability of the existing 'Manage Trusted Extensions' action in the Accounts view.
