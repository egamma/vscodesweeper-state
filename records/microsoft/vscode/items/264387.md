# Issue #264387 — Always runs 'compile' task in vscode-copilot-chat repo when other build tasks are running

[https://github.com/microsoft/vscode/issues/264387](https://github.com/microsoft/vscode/issues/264387)

## Summary

Team-authored issue requesting that the AI agent be smarter about detecting already-running build tasks instead of always launching the 'compile' task. Kept open per team-member rule.

## Change summary

The Copilot agent always runs the 'compile' task to verify its work even when watch-mode build tasks are already running, wasting time and resources.

## Best solution

Add custom instructions (similar to what was done for the vscode repo) to tell the agent to check running watch tasks via get_task_output rather than launching the compile task, or improve the agent's generic handling of the 'This task is currently running' annotations in workspace_info.
