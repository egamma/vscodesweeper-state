# Issue #298713 — Support loading agent skills from packages

[https://github.com/microsoft/vscode/issues/298713](https://github.com/microsoft/vscode/issues/298713)

## Summary

Valid feature request for glob support in `chat.agentSkillsLocations` to discover skills shipped inside packages. A team member (aeschli) expressed interest but noted it depends on whether this becomes a standard. Still under discussion with low community traction (2 upvotes).

## Change summary

Allow glob patterns (e.g. `node_modules/**/.agents/skills`) in the `chat.agentSkillsLocations` setting so agent skills distributed inside packages can be discovered.

## Best solution

Extend the path validation in `src/vs/workbench/contrib/chat/common/promptSyntax/utils/promptFilesLocator.ts` (specifically `VALID_PROMPT_FOLDER_PATTERN`) to accept glob characters, and update the skill discovery logic to expand globs when resolving skill folders. This depends on a product decision about whether package-distributed skills should be a supported pattern.
