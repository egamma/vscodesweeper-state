# Issue #298713 — Support loading agent skills from packages

[https://github.com/microsoft/vscode/issues/298713](https://github.com/microsoft/vscode/issues/298713)

## Summary

Valid feature request to support glob patterns in `chat.agentSkillsLocations` for discovering skills shipped inside packages (npm, pip). Not yet implemented — the setting's validation regex explicitly rejects glob characters. A team member expressed interest but noted it depends on whether package-distributed skills become a standard.

## Change summary

Allow `chat.agentSkillsLocations` to accept glob patterns (e.g. `node_modules/**/.agents/skills`) so that skills bundled in third-party packages can be discovered.

## Best solution

Either extend `VALID_PROMPT_FOLDER_PATTERN` and `resolveSkillConfigLocations` to support glob expansion, or introduce a separate mechanism for package-sourced skill discovery (e.g. a `package.json` contribution point). The decision depends on whether the ecosystem standardizes on shipping skills in packages — keep open under discussion until that becomes clearer.
