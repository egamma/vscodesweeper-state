# Issue #204258 — Configurable Position for Accessibility-Related Text Notifications in Code Editor

[https://github.com/microsoft/vscode/issues/204258](https://github.com/microsoft/vscode/issues/204258)

## Summary

Legitimate accessibility feature request still under discussion — a team member (meganrogge) explained the alert vs status tradeoff and CC'd accessibility experts, but no resolution or implementation followed. Keeping open for the accessibility team to decide.

## Change summary

Allow configuring whether accessibility text notifications (e.g. error on line) are read before or after line content by screen readers.

## Best solution

The accessibility team should decide whether a setting to control announcement order (alert-before vs status-after) is feasible without the content-override downside. If so, a verbosity-style setting in `accessibility.verbosity` or `accessibility.signals` could let users choose. Needs a product decision from the accessibility owners.
