## Summary

Confirmed duplicate of the known asyncTokenization issues tracked by #244225. The reporter verified that disabling `experimental.asyncTokenization` resolves it, and the symptom (bracket highlighting breaks until a character is typed) matches the known async tokenization race condition.

## Change summary

Bracket pair highlighting stops working after the first cursor move in a freshly-opened document until a character is typed, caused by async tokenization.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/244225 which tracks all asyncTokenization issues. The fix will land when either the race is resolved or asyncTokenization is turned off by default (PR https://github.com/microsoft/vscode/pull/244217 is open for the latter).
