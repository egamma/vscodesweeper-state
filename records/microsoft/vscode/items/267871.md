# Issue #267871 — [html] problem range in embedded content needs to be limited to embedded range.

[https://github.com/microsoft/vscode/issues/267871](https://github.com/microsoft/vscode/issues/267871)

## Summary

Still unfixed on main. The CSS mode's doValidation returns diagnostics without clamping their ranges to the embedded region boundaries, so a CSS error (like a missing closing brace) can produce a diagnostic range that extends past the </style> tag into unrelated HTML lines.

## Change summary

CSS diagnostics in embedded <style> blocks can report ranges outside the embedded region when an error spans to end-of-content (e.g., missing closing brace).

## Best solution

In `extensions/html-language-features/server/src/modes/cssMode.ts` doValidation, clamp each returned diagnostic's range to the embedded CSS region boundaries (available from the HTMLDocumentRegions). The JS mode doesn't need this because TypeScript diagnostics are bounded by token spans, but CSS errors like unterminated rules can extend to the document end.
