# Issue #267871 — [html] problem range in embedded content needs to be limited to embedded range.

[https://github.com/microsoft/vscode/issues/267871](https://github.com/microsoft/vscode/issues/267871)

## Summary

Confirmed bug still present on main: CSS diagnostics in embedded HTML are returned with ranges from the embedded document without being clamped to the embedded region's boundaries, causing incorrect error positions.

## Change summary

CSS validation errors in HTML files report wrong line/column because diagnostic ranges from the embedded CSS document are not remapped to the HTML document coordinates.

## Best solution

In `doValidation` in `extensions/html-language-features/server/src/modes/cssMode.ts`, clamp the diagnostic ranges returned by `cssLanguageService.doValidation` so they fall within the embedded CSS region's range in the original HTML document. The issue title already captures the fix: limit problem ranges to the embedded content range.
