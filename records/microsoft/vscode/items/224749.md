# Issue #224749 — It's not possible to set any Terminal Profile

[https://github.com/microsoft/vscode/issues/224749](https://github.com/microsoft/vscode/issues/224749)

## Summary

Still reproduces on main: the settings validation enum for `terminal.integrated.defaultProfile.<os>` is populated only from detected/auto-discovered profiles, so user-defined custom profile names trigger a spurious validation warning until the terminal backend refreshes the schema. Confirmed by a team member; no fix has landed.

## Change summary

Custom terminal profile names defined in `terminal.integrated.profiles.<os>` trigger a settings validation warning when used in `terminal.integrated.defaultProfile.<os>`.

## Best solution

The `registerTerminalDefaultProfileConfiguration` function in `src/vs/platform/terminal/common/terminalPlatformConfiguration.ts` should include user-configured profile names (from `terminal.integrated.profiles.<os>`) in the enum immediately — not only after the backend detects them. This likely means reading the configured profile keys from settings and merging them into `createProfileSchemaEnums` even before full profile detection completes.
