# Issue #275625 — run_in_terminal tool missed the first char in sending a command into git bash terminal

[https://github.com/microsoft/vscode/issues/275625](https://github.com/microsoft/vscode/issues/275625)

## Summary

Team-authored bug on 'On Deck' milestone — hard keep-open. The issue remains unresolved: no fixing commit found on main, and the conpty DLL 1.25 update did not resolve it for reporters. The root cause appears to involve Git Bash + terminal resize/focus interactions dropping the first character of input.

## Change summary

The run_in_terminal tool (and manual typing after resize) drops the first character when sending commands to a Git Bash terminal on Windows.

## Best solution

Investigate the interaction between terminal resize/focus events and early input in Git Bash specifically. The conpty DLL 1.25 upgrade (commit 262ba6d) did not fix it. The terminal input path in terminalProcessManager.ts and shell integration initialization timing are likely areas to investigate — possibly a race where input is sent before the shell is fully ready after a resize or new terminal creation.
