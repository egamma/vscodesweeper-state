# Issue #263436 — Find history does not work as a most recent stack

[https://github.com/microsoft/vscode/issues/263436](https://github.com/microsoft/vscode/issues/263436)

## Summary

Valid feature request with 24 community upvotes, promoted to Backlog. The related PR #265941 was closed without merging, and no MRU find history behavior exists on current main. The feature remains unimplemented.

## Change summary

Find history should use most-recently-used (MRU) ordering so re-searching a term moves it to the top of the history stack.

## Best solution

Implement an `editor.find.historyOrder` setting (or similar) in the find controller that optionally promotes reused search terms to the top of the history navigator. The change would be localized to `src/vs/editor/contrib/find/browser/findController.ts` and the history classes in that directory. The closed PR #265941 sketched a reasonable approach but was never merged — a fresh, maintainer-authored implementation following existing patterns would be appropriate.
