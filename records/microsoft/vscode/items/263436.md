# Issue #263436 — Find history does not work as a most recent stack

[https://github.com/microsoft/vscode/issues/263436](https://github.com/microsoft/vscode/issues/263436)

## Summary

Valid feature request for MRU-style find history ordering. The current `HistoryNavigator2.add()` does move duplicates to the tail, but the user's complaint likely stems from how history navigation (arrow keys) interacts with storage — selecting from history doesn't re-add to the tail unless a new search is committed. Has 24 upvotes, promoted to Backlog. The Copilot coding agent PR #265941 was closed without merging. No duplicate found.

## Change summary

Request to make the Find history behave as a most-recently-used stack so re-searching a term moves it to the top of the arrow-key navigation order.

## Best solution

Implement MRU promotion in the find history: when a term is re-used (either typed again or selected from history and searched), promote it to the most recent position. The change would likely touch `src/vs/base/browser/ui/inputbox/inputBox.ts` (the `showPreviousValue`/`showNextValue` flow) and possibly `src/vs/editor/contrib/find/browser/findWidget.ts` (the `_updateHistory` path). Consider a setting (`editor.find.historyOrder`) for users who prefer chronological order.
