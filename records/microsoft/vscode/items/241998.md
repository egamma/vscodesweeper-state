## Summary

Team-authored issue reporting an awkward warning message in the chat references list when a binary file is attached — the message exposes a raw file:/// URI and a technical 'Detail:' suffix. No fix found on current main; keeping open for the area owner.

## Change summary

The warning message for binary files in the chat references list includes a raw file:/// URI and a verbose technical detail string, unlike other reference warnings.

## Best solution

Improve the warning message formatting so that binary-file references show a human-friendly file name (e.g. just the basename) instead of the full `file:///` URI, and shorten or restructure the 'Detail: File seems to be binary…' suffix to match the tone of other reference warnings.
