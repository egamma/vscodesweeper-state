# Issue #273582 — Terminal suggest: Support path completions without the ./

[https://github.com/microsoft/vscode/issues/273582](https://github.com/microsoft/vscode/issues/273582)

## Summary

Still a valid open request: the `addPathRelativePrefix` function in `terminalCompletionService.ts` always adds `./` to relative paths. The code even has a comment noting this behavior may be changed in the future to match user input format.

## Change summary

Terminal suggest should offer path completions without the `./` prefix when the user has not typed `./`.

## Best solution

Modify `addPathRelativePrefix` in `src/vs/workbench/contrib/terminalContrib/suggest/browser/terminalCompletionService.ts` to respect the user's typed prefix: if the user typed a bare filename (no `./`), completions should omit the `./` prefix.
