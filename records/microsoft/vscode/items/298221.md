# Issue #298221 — Cannot reconnect in WSL window after hibernation

[https://github.com/microsoft/vscode/issues/298221](https://github.com/microsoft/vscode/issues/298221)

## Summary

The server-side reconnection grace timer (3 hours, wall-clock) expires during OS hibernation because setTimeout continues to count elapsed time. When the client wakes and attempts reconnection, the server has already disposed the connection. No fix exists on current main. Multiple users confirm this across various hardware.

## Change summary

WSL remote window shows 'Cannot reconnect' after long hibernation (>3h) because the server-side grace timer expires during sleep.

## Best solution

The VS Code server in WSL should account for system sleep/hibernation when computing grace-time expiry — either by using monotonic process-time instead of wall-clock time (the server already has `ProcessTimeRunOnceScheduler` in other paths), or by allowing the client to trigger a fresh reconnection token handshake when the server's old connection was cleaned up. The fix belongs in `src/vs/server/node/remoteExtensionManagement.ts` and potentially in the client reconnection loop (`src/vs/platform/remote/common/remoteAgentConnection.ts`).
