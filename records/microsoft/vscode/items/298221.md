# Issue #298221 — Cannot reconnect in WSL window after hibernation

[https://github.com/microsoft/vscode/issues/298221](https://github.com/microsoft/vscode/issues/298221)

## Summary

Real bug confirmed by multiple users and traceable in source: the server-side reconnection grace time timer in `remoteExtensionManagement.ts` fires based on wall-clock time, which includes hibernation time. After >3h hibernation, the server has already disposed the connection before the client even wakes up to attempt reconnection. Needs routing to remote-connection area.

## Change summary

WSL remote window cannot reconnect after PC hibernation exceeding the 3-hour reconnection grace time, because the server disposes the connection while the machine is hibernating.

## Best solution

The server-side grace time disposal logic in `src/vs/server/node/remoteExtensionManagement.ts` (and possibly the extension host's `ProcessTimeRunOnceScheduler`) should detect system resume and reset/extend the grace timer, or the client should be able to request a fresh connection rather than depending on the server having kept the old one alive. The `--reconnection-grace-time` CLI flag (commit 6cc2564bf9d) is a workaround but not a real fix since any finite timeout will fail for sufficiently long hibernation.
