[https://github.com/microsoft/vscode/issues/244591](https://github.com/microsoft/vscode/issues/244591)

## Summary

Still present on main: ExtensionsInput is created via createInstance and passed to openEditor without disposal tracking. The suggested fix (use editor resolver API) has not been applied.

## Change summary

ExtensionsInput is leaked because it's created directly via instantiationService.createInstance and not tracked for disposal.

## Best solution

Refactor ExtensionsInput opening to use the editor resolver API pattern (as bpasero suggested), so the workbench manages the editor input lifecycle. The fix is in `src/vs/workbench/contrib/extensions/browser/extensionsWorkbenchService.ts` around line 1708.
