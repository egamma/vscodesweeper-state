# Issue #260706 — Git - Make it easier to tell that the current window is a work tree

[https://github.com/microsoft/vscode/issues/260706](https://github.com/microsoft/vscode/issues/260706)

## Summary

Team-authored feature request for better worktree identification in the window title and SCM view — still unimplemented on main. Hard keep-open (MEMBER author).

## Change summary

Make it easier to identify which repository a worktree window belongs to, since worktree branch names are often meaningless.

## Best solution

Expose the parent repository name (not just the worktree name) in window title variables like `activeRepositoryName`, and surface worktree identity more prominently in the SCM view. The related issue https://github.com/microsoft/vscode/issues/306496 (headLabelOverride API) is a complementary piece but doesn't fully address this — a core UX pass on worktree identification in the git contribution and window title service is still needed.
