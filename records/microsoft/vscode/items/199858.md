# Issue #199858 — I want to set font file names or filepaths into `editor.fontFamily`

[https://github.com/microsoft/vscode/issues/199858](https://github.com/microsoft/vscode/issues/199858)

## Summary

Keep this open as an existing Backlog feature request under discussion; the current editor font pipeline still treats `editor.fontFamily` as a CSS font-family value, not as a font file name or path.

## Change summary

Allow `editor.fontFamily` to select a specific font file by file name or absolute path when installed fonts share the same metadata family name.

## Best solution

The workbench/editor fonts owners should decide whether core should support registering font files from paths for the editor, or explicitly document that `editor.fontFamily` only accepts CSS font family names. If accepted, the implementation likely belongs around editor font configuration and sanitization in `src/vs/editor/common/config` and rendering paths that currently reject path characters.
