# Issue #299757 — Agent Plugins: 404 error when fetching marketplace.json from awesome-copilot repo

[https://github.com/microsoft/vscode/issues/299757](https://github.com/microsoft/vscode/issues/299757)

## Summary

The 404 in the Developer Console is expected behavior — the plugin marketplace service probes multiple well-known paths (`marketplace.json`, `.plugin/marketplace.json`, `.github/plugin/marketplace.json`, `.claude-plugin/marketplace.json`) and gracefully handles 404s at the debug log level. The browser's DevTools network tab shows all HTTP requests including expected probing failures, but the service continues to the next path. This is as-designed.

## Change summary

A 404 network error appears in Developer Tools when VS Code probes `.claude-plugin/marketplace.json` on the awesome-copilot repository.

## Best solution

Close as designed. The probing pattern is intentional — each marketplace definition path is tried in order, and 404s on non-existent paths are handled gracefully. The 404 visible in DevTools is a browser-level network log, not an application error. The default configuration `github/awesome-copilot#marketplace` uses the `marketplace` branch ref, not `main`; the reporter's URL showing `main` suggests a possible misconfiguration or an older build.
