# Issue #120884 — Code screen will move up after write a unimported package method and undo

[https://github.com/microsoft/vscode/issues/120884](https://github.com/microsoft/vscode/issues/120884)

## Summary

Keep open: a maintainer reproduced the behavior and reframed it as an editor undo/redo feature request, and I did not find a concrete main-branch fix for recording editor scroll position in text undo elements.

## Change summary

Undoing an auto-import completion should restore the editor viewport rather than leaving the screen shifted after inserted import lines are removed.

## Best solution

If accepted, the editor undo/redo path should preserve enough view state, likely near the text edit stack/view model boundary, so undoing edits that insert or remove lines above the viewport restores the prior scroll position without relying on language-extension behavior.
