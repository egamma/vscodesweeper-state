# Issue #120884 — Code screen will move up after write a unimported package method and undo

[https://github.com/microsoft/vscode/issues/120884](https://github.com/microsoft/vscode/issues/120884)

## Summary

Still unimplemented on main: the edit undo stack (editStack.ts) only records cursor selections, not scroll position. The request to persist scroll state in the undo stack remains a valid gap.

## Change summary

After undoing an auto-import that inserted lines above the cursor, the viewport shifts because scroll position is not restored with undo.

## Best solution

Record and restore scrollTop/scrollLeft as part of SingleModelEditStackData in src/vs/editor/common/model/editStack.ts, and apply it on undo/redo in the editor widget. The cursorUndo contrib already tracks scroll — the pattern exists but isn't wired into the main undo stack.
