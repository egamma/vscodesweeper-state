# Issue #323413 — Side bar stays closed after window reload & resize

[https://github.com/microsoft/vscode/issues/323413](https://github.com/microsoft/vscode/issues/323413)

## Summary

Team-authored bug report from test plan verification of #323333. The responsive sidebar auto-collapse state (`_sidebarAutoHidden`) is a runtime flag that is not persisted across reloads, so after a window close/reopen the sidebar cannot be auto-restored when the window widens because the flag resets to `false`.

## Change summary

After the Agents window auto-collapses the sessions sidebar due to narrow width, closing and reopening the window loses the 'auto-hidden' state, so widening the window no longer auto-restores the sidebar.

## Best solution

Persist the `_sidebarAutoHidden` flag (or an equivalent 'collapsed-by-responsive-rule' marker) across reloads — likely in `desktopSessionLayoutController.ts` alongside the existing view-state persistence — so that on reload+widen the sidebar is auto-restored the same way it would be within a single session.
