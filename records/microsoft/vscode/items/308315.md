# Issue #308315 — High CPU usage with vscode/resources/app/node_modules/@vscode/ripgrep/bin/rg

[https://github.com/microsoft/vscode/issues/308315](https://github.com/microsoft/vscode/issues/308315)

## Summary

Ripgrep high CPU when following symlinks in a large multi-project workspace is a recurring report. The workaround (`search.followSymlinks: false`) resolves it, but the report lacks detail on the specific workspace layout that triggers the runaway traversal. Requesting more info to determine whether this is a new regression or the known large-workspace symlink traversal pattern.

## Change summary

Ripgrep (`rg`) bundled with VS Code consumes 100% CPU on startup, especially in large workspaces with symlinks.

## Best solution

Determine whether this is the known symlink-loop / large-workspace traversal pattern or a new regression. If the former, it may warrant a UX improvement — e.g., imposing traversal limits on ripgrep search or changing the default of `search.followSymlinks` to `false`. The search area team should evaluate whether the default is still appropriate given recurring reports. In the meantime, `search.followSymlinks: false` is the established workaround.
