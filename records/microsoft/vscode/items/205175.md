# Issue #205175 — (Ctrl + Shift) Multi-text Selection: Incorrect Selection Relative to Selections

[https://github.com/microsoft/vscode/issues/205175](https://github.com/microsoft/vscode/issues/205175)

## Summary

Still reproduces on main: the multi-cursor word-right fix (dc8c2d019c8) only addressed word-left. The word-right path in `moveWordRight` still doesn't use the `hasMulticursor` parameter, so multiple cursors can land at inconsistent positions when doing Ctrl+Shift+Right.

## Change summary

Multi-cursor Ctrl+Shift+Right selects inconsistently across cursors because word-right navigation doesn't account for multi-cursor behavior the way word-left now does.

## Best solution

Apply the same `hasMulticursor` guard to `moveWordRight` in `cursorWordOperations.ts` that was applied to `moveWordLeft` in commit dc8c2d019c8 — specifically, disable the single-separator skip in `WordEnd` navigation when multiple cursors are active, so all cursors behave uniformly.
