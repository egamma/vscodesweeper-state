# Issue #283149 — Unable to specify line numbers when providing file context in prompt files

[https://github.com/microsoft/vscode/issues/283149](https://github.com/microsoft/vscode/issues/283149)

## Summary

This is a valid feature request for line-range support in prompt file references. The current parser (`promptFileParser.ts`) treats the entire `#file:` content as a path with no line-number parsing. No implementation exists on main. Keeping open as it's in Backlog Candidates.

## Change summary

Allow specifying line numbers/ranges (e.g., `#file:path/to/file.ts:10-20`) in prompt file references to attach only a portion of a file as context.

## Best solution

Extend the prompt file reference parser in `src/vs/workbench/contrib/chat/common/promptSyntax/promptFileParser.ts` to recognize a trailing `:startLine-endLine` suffix, strip it from the path before resolution, and pass the line range downstream so only the specified lines are included as context. Should also be handled in `resolveFilePath` and the prompt attachment resolution layer.
