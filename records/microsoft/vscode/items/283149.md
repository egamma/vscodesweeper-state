# Issue #283149 — Unable to specify line numbers when providing file context in prompt files

[https://github.com/microsoft/vscode/issues/283149](https://github.com/microsoft/vscode/issues/283149)

## Summary

This is a feature request to support line-number ranges (e.g. `#file:path:10-20`) in prompt file references. Current `main` has no such support — the `#file:` regex captures the full path including any `:lineNum` suffix, and `resolveFilePath` treats it as a literal path with no line-range stripping.

## Change summary

Allow specifying line numbers/ranges when referencing files in prompt files so only a subset of the file is included as context.

## Best solution

Extend the prompt file `#file:` reference syntax to accept an optional line-range suffix (e.g. `#file:path.ts:10-20`), parse it in `promptFileParser.ts`, and pass the range through to the context-attachment layer so only the specified lines are included. This is a meaningful enhancement to prompt-file expressiveness.
