# Issue #256546 — Remote Window: "inputs" section in mcp.json is not registering

[https://github.com/microsoft/vscode/issues/256546](https://github.com/microsoft/vscode/issues/256546)

## Summary

Confirmed bug: RemoteUserConfiguration never reads the remote user-level mcp.json as a standalone configuration resource, so input variables are unresolvable in remote windows. An open community PR (#324124) addresses this but awaits review.

## Change summary

MCP server input variables defined in the remote user mcp.json (~/.vscode-server/data/User/mcp.json) fail to resolve in Remote SSH/WSL/devcontainer windows.

## Best solution

Review and land https://github.com/microsoft/vscode/pull/324124, which wires mcpResource into FileServiceBasedRemoteUserConfiguration (mirroring what UserConfiguration already does for local mcp.json). The fix lives in src/vs/workbench/services/configuration/browser/configuration.ts.
