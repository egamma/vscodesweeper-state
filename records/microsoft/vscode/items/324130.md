# Issue #324130 — Insiders regression: Extensions have no pre-release version after the release version is installed and reload window is done

[https://github.com/microsoft/vscode/issues/324130](https://github.com/microsoft/vscode/issues/324130)

## Summary

Team-authored Insiders regression report — pre-release version toggle disappears after installing release version and reloading. Kept open per team-member rule; the bug likely involves how `hasPreReleaseVersion` is resolved when the gallery extension data differs from the local installation state.

## Change summary

After installing the release version of an extension with a pre-release available and reloading, Insiders no longer shows the option to switch to pre-release.

## Best solution

Investigate how the `hasPreReleaseVersion` property in `extensionsWorkbenchService.ts` (line 400–401) resolves when a local extension is installed — when `this.gallery` is absent or stale after reload, the fallback `this.local?.hasPreReleaseVersion` may not carry the pre-release flag. The `hasPreReleaseForExtension` logic in `extensionGalleryService.ts` (line 361–363, 569) that checks `productService.extensionProperties` may also be relevant. Possibly regressed by commit 28af4cff058 ('Remove API version concept') which changed version compatibility filtering.
