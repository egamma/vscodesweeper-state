# Issue #322582 — Typed Text Doesn't Show Up in Separate Terminal Window When Main Window is Minimized

[https://github.com/microsoft/vscode/issues/322582](https://github.com/microsoft/vscode/issues/322582)

## Summary

Plausible rendering bug in an auxiliary terminal window when the main VS Code window is minimized on Windows, likely related to Electron/Chromium background-throttling of requestAnimationFrame or rendering pipelines. Cannot verify from source alone; requesting more info to confirm reproduction on latest builds.

## Change summary

Typed text is received by the terminal in a separate (auxiliary) window but not rendered on screen until the main VS Code window is un-minimized.

## Best solution

The terminal team should investigate whether xterm.js rendering in auxiliary windows is affected by the main window's visibility state — VS Code already works around setTimeout throttling across windows (src/vs/workbench/browser/window.ts enableMultiWindowAwareTimeout) but requestAnimationFrame and GPU-driven rendering may not be covered. If the root cause is Electron/Chromium throttling across BrowserWindows, it may need an upstream fix or an Electron-level workaround. Related (but distinct) issue https://github.com/microsoft/vscode/issues/317487 reports process-execution throttling in backgrounded windows.
