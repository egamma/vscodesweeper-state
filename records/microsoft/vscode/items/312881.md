# Issue #312881 — Truncation in sessions lists can hide deleted line counts

[https://github.com/microsoft/vscode/issues/312881](https://github.com/microsoft/vscode/issues/312881)

## Summary

Still reproduces on main: the `.session-diff` flex container in the sessions list has `overflow: hidden` without `flex-shrink: 0`, so when the sidebar is narrow the deletions count (`-N`) gets clipped with no visual indication that information is hidden.

## Change summary

When the sessions list sidebar is narrow, the deleted-lines count in session rows is truncated with no ellipsis or other indicator, making it look like only additions exist.

## Best solution

Add `flex-shrink: 0` to `.session-diff` (or its children `.session-diff-added` and `.session-diff-removed`) in `src/vs/sessions/contrib/sessions/browser/media/sessionsList.css` so the diff stats never partially clip. Alternatively, let the preceding elements (workspace badge, time) shrink/ellipsize first while keeping the compact diff stats intact.
