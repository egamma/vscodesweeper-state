# Issue #316371 — The floating text covers the button, and moving your mouse slightly to the left or right causes the button to flash.

[https://github.com/microsoft/vscode/issues/316371](https://github.com/microsoft/vscode/issues/316371)

## Summary

Confirmed visual bug: hover tooltip on small action buttons in the BREAKPOINTS view header overlaps the button, causing a mouseenter/mouseleave flicker loop. Reproduced by another user with a GIF. No fix found on current main.

## Change summary

Hovering over action buttons next to the BREAKPOINTS header causes the tooltip to cover the button, triggering a continuous flicker as the mouse alternates between the button and tooltip.

## Best solution

Fix the hover positioning for action buttons in view pane headers so the tooltip doesn't overlap the trigger element. The hover widget's position adjustment logic in `src/vs/platform/hover/browser/hoverWidget.ts` or the managed hover setup in `src/vs/base/browser/ui/actionbar/actionViewItems.ts` likely needs to ensure adequate clearance between the tooltip and the target button. This may also relate to the general hover service's mouse-tracking in `src/vs/platform/hover/browser/hoverService.ts`.
