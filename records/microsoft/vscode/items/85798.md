# Issue #85798 — Incorrect keyboard shortcut shown in menu.

[https://github.com/microsoft/vscode/issues/85798](https://github.com/microsoft/vscode/issues/85798)

## Summary

Still valid on current main for users with native menus on Linux. Chromium's accelerator parser ignores the Super/Meta modifier, so Electron renders e.g. 'Ctrl+Super+A' as just 'Ctrl+A'. VS Code has a non-native keybinding fallback, but it doesn't trigger because getElectronAccelerator() returns a non-null string. The default custom menu style bypasses this, but native-menu users are still affected. A team member (deepak1556) confirmed in 2023 that a VS Code-side workaround is the right path.

## Change summary

Keybindings involving the Super/Meta modifier on Linux display incorrectly in native Electron menus, with the Super modifier silently dropped.

## Best solution

Return `null` from `getElectronAcceleratorLabelForScanCodeChord` (in `src/vs/workbench/services/keybinding/common/macLinuxKeyboardMapper.ts`) when the chord includes the Meta modifier on Linux, so the existing `isNative: false` fallback in `src/vs/workbench/electron-browser/parts/titlebar/menubarControl.ts` renders the label correctly via the custom keybinding display path. This is the workaround deepak1556 recommended.

## Evidence

- **upstream root cause:** Chromium's accelerator parser at ui/base/accelerators/accelerator.cc (lines 396–405) does not handle the Super/Meta modifier; confirmed by deepak1556 (Electron team member) in comment 6.
- **VS Code workaround path:** src/vs/workbench/electron-browser/parts/titlebar/menubarControl.ts has a fallback for non-native keybindings (isNative: false) that renders keybinding labels in brackets, but getElectronAccelerator() returns a non-null string for Meta-containing chords, so the fallback never triggers.
- **ElectronAcceleratorLabelProvider:** src/vs/base/common/keybindingLabels.ts defines metaKey as 'Super' for Linux in the Electron accelerator label provider, which is the string Chromium cannot parse.
- **default mitigated:** Since window.menuStyle defaults to 'inherit' (following titleBarStyle, which defaults to 'custom'), most users now get custom menus that bypass Electron accelerators entirely. Only users explicitly using native menus on Linux are still affected.
- **team confirmation:** deepak1556 (MEMBER) commented on 2023-12-12: 'a workaround on our end would be better here' — confirming the VS Code-side fix approach.
