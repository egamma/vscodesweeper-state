# Issue #323164 — Make sure Python auto activation work with agent host

[https://github.com/microsoft/vscode/issues/323164](https://github.com/microsoft/vscode/issues/323164)

## Summary

Team-authored feature request for ensuring Python environment auto-activation works with the agent host's shell tool. Active cross-repo work is underway (SDK APIs for per-session shell init scripts). Keeping open per policy — team member issue requiring maintainer judgment.

## Change summary

Ensure Python virtual environment auto-activation works when the agent host's built-in shell tool spawns child processes, since those processes don't run VS Code's shell integration scripts or user rc files.

## Best solution

Coordinate with the Copilot SDK team on the per-session shell initialization API (github/copilot-sdk#1911, github/copilot-sdk#2040, and the merged copilot-agent-runtime#12710) to allow the Python extension to inject activation scripts into agent host shell sessions. The Python side likely lives in the vscode-python-environments repo; the VS Code side may need agent-host glue in src/vs/platform/agentHost to surface the SDK's init-script API to extensions.
