# Issue #237608 — Overriding XDG_DATA_HOME in snap leads to multiple problems

[https://github.com/microsoft/vscode/issues/237608](https://github.com/microsoft/vscode/issues/237608)

## Summary

Still reproduces on main: the snap launcher unconditionally overrides XDG_DATA_HOME, breaking many tools (trash, fish, uv, podman, atuin, mise, etc.) in the integrated terminal and VS Code subprocesses. An open PR (#292808) exists to fix it — keeping open pending that PR's review/merge.

## Change summary

The snap package overrides XDG_DATA_HOME to a snap-local path, causing all programs launched by VS Code to use wrong data directories.

## Best solution

Review and land https://github.com/microsoft/vscode/pull/292808 which removes the XDG_DATA_HOME override from the snap launcher script (`resources/linux/snap/electron-launch`). The original commit 2d28c0a79c87 added the override for mime DB generation — the fix should ensure mime DB still works without polluting the user's environment.
