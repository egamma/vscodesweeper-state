# Issue #249249 — Make it easier to distinguish & trigger chat completion types

[https://github.com/microsoft/vscode/issues/249249](https://github.com/microsoft/vscode/issues/249249)

## Summary

This feature request for scoped/typed chat completions (prefixes like `#file:`, `#tool:`, icons for completion types) is still unimplemented on main. The chat input completions code still explicitly uses `CompletionItemKind.Text` with comments noting 'icons are disabled here anyway,' and no scoped prefix completion system exists. The issue was discussed in May/June 2025 iteration plans and remains a valid design proposal worth keeping open.

## Change summary

Make chat `#`-completions distinguishable by type (files, tools, symbols) via scoped prefixes like `#file:` and/or completion icons, reducing noise when all types appear together.

## Best solution

Implement the two-level completion flow described in comment 1: typing `#` completes to category prefixes (`#file`, `#tool`, `#sym`), then a second completion scopes results to that category. This could extend `IChatContextPickService` in `src/vs/workbench/contrib/chat/browser/attachments/chatContextPickService.ts` and the completion providers in `src/vs/workbench/contrib/chat/browser/widget/input/editor/chatInputCompletions.ts`. Enabling icons (replacing `CompletionItemKind.Text` with appropriate kinds) is the lower-effort first step.
