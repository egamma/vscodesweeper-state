# Issue #318767 — Checks view appears in Agent window on main branch (not on a PR)

[https://github.com/microsoft/vscode/issues/318767](https://github.com/microsoft/vscode/issues/318767)

## Summary

Team-authored bug report — the Checks pane in the Changes view becomes visible when CI data is returned even without an active PR session context. The visibility is driven solely by `_checkCount > 0` without gating on whether the session actually has a PR association. Keeping open for maintainer attention.

## Change summary

The CI Checks section appears in the Agent window's Changes view when on the main branch with no active PR, confusing users.

## Best solution

Gate the CI checks pane visibility (or the upstream `activeSessionPullRequestCIObs` derivation) so it only fetches/shows data when the active session is explicitly associated with a pull request — not merely when the GitHub service can resolve CI status for the current branch's HEAD.
