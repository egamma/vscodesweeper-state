# Issue #216913 — vscode.dev: better focus for the git password entry field when git pushing from terminal

[https://github.com/microsoft/vscode/issues/216913](https://github.com/microsoft/vscode/issues/216913)

## Summary

Still a valid UX concern — the quick input appearing for password prompts in vscode.dev lacks sufficient visual prominence to draw attention from the terminal. Team members discussed it (daviddossett considering shadow/border adjustments, TylerLeonhardt suggesting it's a theme issue) but no targeted fix has landed. Keeping open as under-discussion.

## Change summary

The quick input password field triggered by git push in vscode.dev's terminal doesn't draw enough visual attention, making the operation appear to hang.

## Best solution

Improve visual prominence of the quick input when it appears over the terminal — either via theme-level shadow/border adjustments in `src/vs/platform/quickinput/browser/media/quickInput.css`, or a focus-drawing mechanism (animation, backdrop dimming). The shadow variable refactor (commit a6e427ed1f4) added `box-shadow: var(--vscode-shadow-xl)` which may partially help, but the team should evaluate whether it's sufficient for terminal-triggered password prompts in vscode.dev.
