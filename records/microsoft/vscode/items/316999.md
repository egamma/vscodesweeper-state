# Issue #316999 — [Bug] Incomplete Sanitization in snippetString.ts (Copilot): Unescaped Backslash Corrupts VS Code Snippet Grammar

[https://github.com/microsoft/vscode/issues/316999](https://github.com/microsoft/vscode/issues/316999)

## Summary

The incomplete backslash escaping in appendVariable is an intentional design choice, explicitly marked with a CodeQL suppression ('I do not want to escape backslashes here'). The _escape method (for appendText) escapes all three characters, but appendVariable's default value deliberately allows snippet escape sequences — this is by design.

## Change summary

appendVariable's sanitizer escapes $ and } but not backslash, potentially allowing backslash sequences to be interpreted as snippet grammar escapes.

## Best solution

Close as designed. The distinction is intentional: appendText fully escapes (including backslash) via _escape(), while appendVariable's default value intentionally permits snippet escape syntax. The CodeQL suppression was a deliberate team decision. The security impact is negligible (no RCE, attacker must already control extension snippet output).
