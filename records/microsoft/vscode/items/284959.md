# Issue #284959 — Copilot silently corrupts long attached files

[https://github.com/microsoft/vscode/issues/284959](https://github.com/microsoft/vscode/issues/284959)

## Summary

Hard keep-open: the issue carries the 'On Deck' milestone, indicating an active scheduling decision. The summarization behavior is confirmed in the source — `fileVariable.tsx` summarizes attached files that exceed a token budget and marks them `isSummarized="true"` without clear user-facing indication. A partial mitigation (the `omitFileAttachmentContents` experimental setting) has landed, and a related PR removed a problematic prompt instruction, but the core concern — silent context corruption of user-attached files — remains open and actively discussed by a team member.

## Change summary

Copilot silently summarizes/truncates long files attached to chat context, replacing content with omission comments, which corrupts the context the model actually receives without informing the user.

## Best solution

Complete the work started with the `omitFileAttachmentContents` setting: either stop including summarized content for agent-mode attachments (directing the agent to read files via tools instead), or give users clear visibility and control over when summarization occurs. The summarization logic lives in `extensions/copilot/src/extension/prompts/node/panel/fileVariable.tsx` and `extensions/copilot/src/extension/prompts/node/inline/summarizedDocument/`. The 'On Deck' milestone suggests this is already queued for attention.
