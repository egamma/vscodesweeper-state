# Issue #32693 — Monolithic structure, multiple project settings

[https://github.com/microsoft/vscode/issues/32693](https://github.com/microsoft/vscode/issues/32693)

## Summary

Highly requested feature (603 upvotes) for nested .vscode/settings.json support in monorepos remains unimplemented on current main. Acknowledged by a team member as a 'big plan item' — keeping open as a valid, well-supported backlog item.

## Change summary

Allow VS Code to discover and apply .vscode/settings.json files from subdirectories to their respective subtrees, enabling monorepo workflows without multi-root workspaces.

## Best solution

Implement hierarchical .vscode/settings.json resolution in the configuration service so that subfolder settings apply to their descendant files. This is a large architectural effort in `src/vs/workbench/services/configuration/` that requires design decisions around extension API surface, language server multiplexing, and settings precedence. The team acknowledged it needs a dedicated plan item.
