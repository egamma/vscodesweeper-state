# Issue #120163 — Debug: show welcome when while not debugging even if launch.json present

[https://github.com/microsoft/vscode/issues/120163](https://github.com/microsoft/vscode/issues/120163)

## Summary

Still unimplemented on main: the debug welcome view is hidden once a launch.json exists, even when not debugging. The request to show it while inactive (alongside the breakpoints view when breakpoints exist) remains a valid UX improvement tracked under `under-discussion`.

## Change summary

Show the debug welcome view when not actively debugging, even if a launch.json is present, so new users still see the 'Run and Debug' button.

## Best solution

Decouple the welcome-view visibility from the presence of debug configurations: change the `debugUx` context key logic in `src/vs/workbench/contrib/debug/browser/debugService.ts` so that the welcome view shows whenever `state === State.Inactive`, regardless of whether `configurationManager.getAllConfigurations().length > 0`. The breakpoints view should remain visible alongside the welcome view when breakpoints exist. This is a UX/design decision that needs maintainer sign-off before implementation.
