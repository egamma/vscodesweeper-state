# Issue #243017 — Cannot remove folders from workspace (Profiles view)

[https://github.com/microsoft/vscode/issues/243017](https://github.com/microsoft/vscode/issues/243017)

## Summary

Still unimplemented on main — the delete button in Profiles > Folders & Workspaces is intentionally disabled for the default profile (line 2222 of userDataProfilesEditor.ts). With 67 upvotes and Backlog milestone, this is a validated community request awaiting implementation.

## Change summary

Allow users to remove folder/workspace path associations from the Default profile in the Profiles editor.

## Best solution

Remove or relax the `getDefaultProfileToUse().id !== item.profileElement.profile.id` guard in `createDeleteAction` (src/vs/workbench/contrib/userDataProfile/browser/userDataProfilesEditor.ts) so that stale paths can be deleted from the default profile, possibly with a confirmation prompt.
