# Issue #249932 — Disassembly Viewer: disassemble by specified function/line

[https://github.com/microsoft/vscode/issues/249932](https://github.com/microsoft/vscode/issues/249932)

## Summary

Valid feature request — the disassembly view currently only navigates to the current PC (instruction pointer reference from the focused stack frame). No work on main addresses opening it at an arbitrary function or line. Keeping open in Backlog with 8 upvotes.

## Change summary

Allow the Disassembly View to navigate to a specified function or source line rather than only the current program counter.

## Best solution

Extend `OpenDisassemblyViewAction` to accept an optional `memoryReference` argument (e.g. resolved from the editor cursor position via the debug adapter), and pass it through to the disassembly view's initial address. The DAP `Disassemble` request already supports arbitrary memory references — the gap is in VS Code's UI command, which always uses the focused stack frame's `instructionPointerReference`. Implementation likely touches `src/vs/workbench/contrib/debug/browser/debugEditorActions.ts` and the disassembly view input/model.
