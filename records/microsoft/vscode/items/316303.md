# Issue #316303 — Agent Plugin Marketplace: support plugins that re-declare the same root path

[https://github.com/microsoft/vscode/issues/316303](https://github.com/microsoft/vscode/issues/316303)

## Summary

Still reproduces on main: plugins sharing `source: "./"` resolve to the same pluginUri, and addInstalledPlugin/removeInstalledPlugin dedup on pluginUri alone — installing one overwrites another. The name field was added for hydration but the install-path uniqueness key was not broadened.

## Change summary

Installing one marketplace plugin causes other plugins from the same marketplace with the same `source` path to disappear, because the installed-plugins store keys uniqueness on pluginUri alone.

## Best solution

Change the installed-plugins identity key from `pluginUri` alone to a composite of `pluginUri + name` (or derive a unique URI incorporating the plugin name). The addInstalledPlugin/removeInstalledPlugin methods and the _pluginMetadata map in pluginMarketplaceService.ts need to use this composite key. This aligns with Claude's marketplace format which supports this pattern.
