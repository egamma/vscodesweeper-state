# Issue #288926 — "Retry" removes attached context

[https://github.com/microsoft/vscode/issues/288926](https://github.com/microsoft/vscode/issues/288926)

## Summary

The retry mechanism was fundamentally broken at the time of the report (1.108.1) and was fixed in commit 623d5d85571 (shipped in 1.109.0). A further fix for attachments not showing in queued messages landed in 3c793749744 (1.110). Current main explicitly preserves attachedContext during resendRequest — this can be closed as completed.

## Change summary

Hitting the retry button after a failed agent call strips the pinned text context from the original request.

## Best solution

Keep the shipped fixes; no further change needed. The retry action (623d5d85571) and attachments preservation (3c793749744) address the reported behavior. If the reporter still sees the issue on the latest build, a new issue with updated reproduction steps would be appropriate.
