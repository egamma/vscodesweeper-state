# Issue #320760 — Main process doesn't exit on fatal startup errors in onReady

[https://github.com/microsoft/vscode/issues/320760](https://github.com/microsoft/vscode/issues/320760)

## Summary

The catch block in `onReady()` at src/main.ts:203–205 swallows fatal startup errors without terminating the process, leaving Electron alive in an invalid state. The defect is confirmed in the current source. An open PR (#320767) already addresses it — keeping open pending that PR's review.

## Change summary

When a fatal error occurs during `onReady()` initialization (e.g. NLS resolution or code-cache directory creation), the process silently continues instead of exiting.

## Best solution

Land the fix from https://github.com/microsoft/vscode/pull/320767 after review — adding `app.exit(1)` to the catch block in `onReady()` at src/main.ts:204, consistent with the existing fail-fast pattern already used elsewhere in the same file (lines 466, 474).
