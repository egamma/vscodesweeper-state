# Issue #318863 — getCwd() on macOS invokes lsof on every terminal command, generating many syscalls per invocation

[https://github.com/microsoft/vscode/issues/318863](https://github.com/microsoft/vscode/issues/318863)

## Summary

Still reproduces on main: getCwd() in terminalProcess.ts unconditionally spawns lsof on macOS without checking if the CWD is already known from shell integration or _properties.cwd. The guard at the call site (line 919) prevents calling when CwdDetection is active, but NaiveCwdDetection still triggers it.

## Change summary

On macOS, getCwd() spawns lsof (generating ~250k syscalls) on every terminal command even when shell integration has already reported the CWD via OSC 633.

## Best solution

Add an early return in getCwd() that uses `this._properties.cwd` when it's already set (shell integration reported it), falling back to lsof only when no CWD is known. This avoids the expensive process spawn while preserving correctness for terminals without shell integration.
