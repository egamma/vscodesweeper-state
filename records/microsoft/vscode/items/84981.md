# Issue #84981 — Support external diff algorithms in internal diff editor

[https://github.com/microsoft/vscode/issues/84981](https://github.com/microsoft/vscode/issues/84981)

## Summary

Valid, well-supported feature request (158 upvotes, in Backlog) for extensible diff algorithms. VS Code has improved its internal diff engine significantly (advanced, wasm variants) but does not yet expose an extension API for plugging in arbitrary algorithms like patience/histogram, which is the core ask here.

## Change summary

Allow external or alternative diff algorithms (patience, histogram, etc.) to be used in VS Code's internal diff editor, ideally via an extensibility point.

## Best solution

Expose a public extension API around `IDocumentDiffProvider` so extensions can contribute custom diff algorithms to the diff editor. The internal plumbing already delegates to an `IDocumentDiffProvider` (see `diffProviderFactoryService.ts` line 67–68), so the infrastructure is partially in place — the gap is a finalized extension-facing registration API.
