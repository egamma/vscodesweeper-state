# Issue #323884 — Recent Opens in New Window

[https://github.com/microsoft/vscode/issues/323884](https://github.com/microsoft/vscode/issues/323884)

## Summary

The core capability (open recent in new window) already exists via Ctrl/Cmd+Click on Welcome tab items and Ctrl+Enter in the Open Recent quick pick. This requests a dedicated visible icon/button for discoverability — a minor polish/UX enhancement. Keeping open as a low-traction feature request in Backlog.

## Change summary

Add a visible icon or button on recent items in the Welcome tab to open them in a new window, rather than relying solely on Ctrl/Cmd+Click.

## Best solution

The existing Ctrl/Cmd+Click behavior already covers the functional need. If the team wants to improve discoverability, a small 'open in new window' inline action (like the existing delete button) could be added to the recent items list in `src/vs/workbench/contrib/welcomeGettingStarted/browser/gettingStarted.ts`. However, given zero community upvotes and the existing modifier-key path, this is low priority and may be declined as unnecessary UI clutter.
