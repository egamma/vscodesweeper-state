# Issue #236999 — Inconsistency between “window.zoomLevel” and Minimap size

[https://github.com/microsoft/vscode/issues/236999](https://github.com/microsoft/vscode/issues/236999)

## Summary

Confirmed and diagnosed by a maintainer: a tiny zoomLevel change flips pixelRatio across the >=2 threshold, causing a dramatic minimap scale jump. The fix (allow scale >3) has been suggested but not yet landed.

## Change summary

Minimap size jumps dramatically with negligible window.zoomLevel changes due to a pixelRatio threshold at 2.

## Best solution

Increase the max allowed minimap scale beyond 3 at line 3564 of editorOptions.ts, and consider smoothing the pixelRatio-based doubling at line 2749 so that tiny zoom changes don't produce a binary jump. Maintainer alexdima already invited a community PR.
