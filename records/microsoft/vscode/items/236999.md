# Issue #236999 — Inconsistency between “window.zoomLevel” and Minimap size

[https://github.com/microsoft/vscode/issues/236999](https://github.com/microsoft/vscode/issues/236999)

## Summary

A maintainer has identified a plausible editor-minimap root cause in current code and suggested a concrete fix, so this should stay open for the editor team or a contributor rather than be closed as stale or by design.

## Change summary

Changing `window.zoomLevel` across a tiny threshold can sharply shrink a large proportional minimap because the minimap scale effectively changes with the pixel-ratio bucket.

## Best solution

Allow `editor.minimap.scale` values above 3 in the editor options schema/validation so users on high-DPI displays can compensate for the pixel-ratio-dependent minimap scaling behavior.
