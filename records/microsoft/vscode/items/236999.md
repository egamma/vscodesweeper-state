# Issue #236999 — Inconsistency between “window.zoomLevel” and Minimap size

[https://github.com/microsoft/vscode/issues/236999](https://github.com/microsoft/vscode/issues/236999)

## Summary

Keep open: the issue is already in the right editor minimap area, and current main still contains the pixel-ratio threshold and scale cap that plausibly explain the abrupt minimap size change.

## Change summary

Changing `window.zoomLevel` slightly can make a large proportional minimap collapse because minimap scaling changes discontinuously around the device pixel ratio threshold.

## Best solution

Adjust the minimap layout/settings behavior in `src/vs/editor/common/config/editorOptions.ts` so zoom-level changes do not cause an abrupt minimap size jump, likely by allowing larger `editor.minimap.scale` values or revisiting the pixel-ratio-dependent scaling.
