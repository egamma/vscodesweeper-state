# Issue #229011 — Git - vscode does not respect GIT_AUTHOR_NAME/EMAIL in pre-commit hook failures, setting git.requireGitUserConfig=false has no effect 

[https://github.com/microsoft/vscode/issues/229011](https://github.com/microsoft/vscode/issues/229011)

## Summary

Still reproduces on main: `handleCommitError` in git.ts unconditionally checks `git config --get-all user.name/email` regardless of `requireUserConfig` and regardless of GIT_AUTHOR_NAME/EMAIL env vars, masking the actual commit failure (e.g. pre-commit hook) with a misleading user.name/email error.

## Change summary

When a pre-commit hook fails and user.name/email are set via GIT_AUTHOR_NAME/EMAIL env vars instead of git config, VS Code shows a misleading 'configure user.name and user.email' error instead of the hook failure, even with `git.requireGitUserConfig` set to false.

## Best solution

Fix `handleCommitError` in `extensions/git/src/git.ts` to either skip the user.name/email config check when `requireUserConfig` is false, or also check for the GIT_AUTHOR_NAME/GIT_AUTHOR_EMAIL/GIT_COMMITTER_NAME/GIT_COMMITTER_EMAIL environment variables before concluding user identity is missing. The `requireUserConfig` option needs to be threaded into `handleCommitError`.
