# Issue #324169 — Support installing extensions from git URLs

Legitimate feature request to install extensions directly from git URLs. No duplicate found, no community traction yet (0 upvotes). Keeping open for the community process — this is a significant scope addition that touches extension trust, signing, and distribution infrastructure.

## Change summary

Allow VS Code to install extensions directly from git repository URLs (clone, build, install) as an alternative to the Marketplace or local VSIX files.

## Best solution

This is a substantial feature that intersects with extension signing/trust infrastructure and the existing Marketplace distribution model. It could potentially be addressed as an extension itself (a CLI tool or extension that clones, packages, and installs). The team should evaluate whether this belongs in core or as a companion tool/extension, considering the security implications of unsigned, unverified extensions from arbitrary git sources.
