# Issue #147447 — cmd+click behavior for Goto Definition and a Document Link

[https://github.com/microsoft/vscode/issues/147447](https://github.com/microsoft/vscode/issues/147447)

## Summary

Keep open: the original conflict is real and a previous fix was explicitly reverted because it caused https://github.com/microsoft/vscode/issues/169593. Current main still has independent document-link and goto-definition click handlers, so there is no shipped/current-main fix to close against.

## Change summary

When a document link and Go to Definition are both available on the same range, cmd/ctrl+click triggers both instead of arbitrating one action.

## Best solution

Implement a regression-safe arbitration between document links and goto-definition in the editor click-link path, likely spanning src/vs/editor/contrib/links/browser/links.ts and src/vs/editor/contrib/gotoSymbol/browser/link/goToDefinitionAtPosition.ts, with coverage for the regression from https://github.com/microsoft/vscode/issues/169593.
