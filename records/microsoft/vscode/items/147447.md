# Issue #147447 — cmd+click behavior for Goto Definition and a Document Link

[https://github.com/microsoft/vscode/issues/147447](https://github.com/microsoft/vscode/issues/147447)

## Summary

Still reproduces on main: when both a documentLink and a definition exist for the same range, cmd/ctrl+click fires both actions. An open PR (https://github.com/microsoft/vscode/pull/316840) addresses this but has not yet merged.

## Change summary

Cmd/ctrl+click triggers both Go to Definition and document link opening when both exist on the same text range, causing an unexpected browser window plus editor navigation.

## Best solution

Merge https://github.com/microsoft/vscode/pull/316840 (or an equivalent approach) which defers to the document link when both a link and a definition overlap, so only one action fires on cmd/ctrl+click.
