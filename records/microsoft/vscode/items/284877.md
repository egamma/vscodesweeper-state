# Issue #284877 — Terminal suggest making terminal input laggy

[https://github.com/microsoft/vscode/issues/284877](https://github.com/microsoft/vscode/issues/284877)

## Summary

Legitimate open performance bug filed by the terminal maintainer with profiling evidence; partial mitigations have landed (shared stat calls) but the core input lag with large pwsh 7 completion sets remains unaddressed on main.

## Change summary

Terminal suggest causes noticeable input lag when pwsh 7 provides a large number of completions, due to expensive widget creation and many file stat calls.

## Best solution

Optimize the terminal suggest widget for large completion sets: defer or batch file stats, lazy-initialize the widget, and consider throttling/virtualizing completions when the count is high. Work lives in `src/vs/workbench/contrib/terminalContrib/suggest/browser/`.
