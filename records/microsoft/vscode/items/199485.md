# Issue #199485 — Support scrolling at the end of the file at the top of the editor in Multi diff editor

[https://github.com/microsoft/vscode/issues/199485](https://github.com/microsoft/vscode/issues/199485)

## Summary

Valid feature request: the multi diff editor's outer scroll container has no extra bottom padding, so the last diff cannot scroll to the top of the viewport like a regular editor with scrollBeyondLastLine. Still unimplemented on main — the template explicitly sets scrollBeyondLastLine: false and there's no compensating padding on the outer container.

## Change summary

Allow the multi diff editor to scroll past the end so the last diff's final line can reach the top of the viewport.

## Best solution

Add configurable bottom padding to the multi diff editor's outer scroll container (in `multiDiffEditorWidgetImpl.ts` / the virtual list) proportional to the viewport height, mirroring the single-editor scrollBeyondLastLine behavior. Could be gated on the same `editor.scrollBeyondLastLine` setting.
