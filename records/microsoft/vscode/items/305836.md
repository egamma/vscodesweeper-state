# Issue #305836 — chatImplicitContext: chatContextService return values bypass ignore policy; setValues() lacks staleness guard

[https://github.com/microsoft/vscode/issues/305836](https://github.com/microsoft/vscode/issues/305836)

## Summary

Bug 1 is source-confirmed on current main: provider-supplied StringChatContextValue bypasses the ignoredFilesService check because line 263 explicitly maps it to undefined before the guard. providerContext is also passed to setValues without any ignore check. Bug 2 (staleness guard) is more of a polish/performance concern than a correctness bug — values are properly cleared, not accumulated.

## Change summary

Context returned by chatContextService.contextForResource() bypasses .copilotignore enforcement in updateImplicitContext(), allowing provider-supplied context for ignored files to enter the chat context window.

## Best solution

In updateImplicitContext(), check StringChatContextValue.resourceUri against ignoredFilesService.fileIsIgnored() for both the providerContext path (line 213) and the webview context path (lines 252-254) before forwarding to setValues(). A staleness guard in setValues() is a nice-to-have optimization but not the critical fix.
