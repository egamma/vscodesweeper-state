# Issue #269258 — Can't trigger "changed 1 file" button with keyboard

[https://github.com/microsoft/vscode/issues/269258](https://github.com/microsoft/vscode/issues/269258)

## Summary

Team-authored accessibility bug, still present on main. The 'View All File Changes' button is nested inside a <summary> element, so keyboard Enter/Space toggles the <details> disclosure instead of activating the button. Keep open per team-authored rule.

## Change summary

Pressing Enter or Space on the 'View All File Changes' button toggles the file list collapse instead of opening the multi-diff editor.

## Best solution

In `chatChangesSummaryPart.ts`, the button inside the `<summary>` needs to stop the keyboard event from reaching the `<details>` toggle. Either add a `keydown` listener on the button that calls `stopPropagation` for Enter/Space (and manually triggers click), or restructure the header so the button is outside the `<summary>` element. The focus-border-radius inconsistency on hover is a separate CSS issue in the same file's styling.
