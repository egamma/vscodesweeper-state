# Issue #110766 — Allow an output to go into full-screen mode

[https://github.com/microsoft/vscode/issues/110766](https://github.com/microsoft/vscode/issues/110766)

## Summary

Still a valid gap on current main — webview and notebook output iframes lack fullscreen permissions (no `allow-popups-to-escape-sandbox` or Fullscreen API delegation), so `requestFullscreen()` calls inside outputs are blocked. Multiple users and a Microsoft team (Q#) have expressed interest. Keeping open as a real feature request in the Backlog.

## Change summary

Allow notebook outputs (and webview content) to use the browser Fullscreen API so embedded videos and interactive widgets can go full-screen.

## Best solution

Add fullscreen support to the webview iframe pipeline: grant `allowfullscreen` on the outer iframe in `src/vs/workbench/contrib/webview/browser/webviewElement.ts` and propagate `allow-popups-to-escape-sandbox` or the Permissions-Policy `fullscreen` header through the nested iframe chain in `src/vs/workbench/contrib/webview/browser/pre/index.html`. For notebook outputs specifically, the back-layer webview's inner frames would also need the attribute. Security review is warranted since this crosses sandbox boundaries.
