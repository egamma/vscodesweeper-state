# Issue #167686 — Allow jump to bracket and select to bracket to stay within brackets

[https://github.com/microsoft/vscode/issues/167686](https://github.com/microsoft/vscode/issues/167686)

## Summary

Valid feature request with 23 upvotes, promoted to Backlog. The 'stay inside brackets' behavior for jumpToBracket is not implemented on current main — the cursor always lands at the start of the matching bracket (outside). selectToBracket already has a selectBrackets arg but jumpToBracket has no equivalent.

## Change summary

Allow jumpToBracket to place the cursor inside the brackets (at the end of an opening bracket or start of a closing bracket) rather than at the start of the bracket token.

## Best solution

Add an optional argument to `editor.action.jumpToBracket` (analogous to `selectBrackets` on `selectToBracket`) that controls whether the cursor lands inside or outside the matched bracket. The change lives in `src/vs/editor/contrib/bracketMatching/browser/bracketMatching.ts` in the `jumpToBracket()` method — when the option is set, use `getEndPosition()` for the opening bracket and `getStartPosition()` adjusted past the bracket for the closing one.
