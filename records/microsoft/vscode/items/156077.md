# Issue #156077 — explorer.excludeGitIgnore fails to show unignored files and directories

[https://github.com/microsoft/vscode/issues/156077](https://github.com/microsoft/vscode/issues/156077)

## Summary

Still reproduces on main — the merged PR #300613 partially fixed gitignore negation but directory traversal with negated patterns (e.g. `!d/`) remains broken. An open PR #316376 targets the remaining gap. Keeping open while that PR is in flight.

## Change summary

Explorer's `explorer.excludeGitIgnore` hides files and directories that are explicitly un-ignored via negation patterns in `.gitignore`, contradicting git's own behavior.

## Best solution

Land or iterate on the open PR https://github.com/microsoft/vscode/pull/316376, which targets the remaining negated-directory traversal logic in `src/vs/workbench/services/search/common/ignoreFile.ts`. The core issue is that `isPathIncludedInTraversal` short-circuits on intermediate directory segments without considering negation patterns that re-include them.
