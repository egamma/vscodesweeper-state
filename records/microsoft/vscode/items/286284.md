# Issue #286284 — 'fetch failed' with no 'try again'

[https://github.com/microsoft/vscode/issues/286284](https://github.com/microsoft/vscode/issues/286284)

## Summary

Team-filed issue (MEMBER); the gap between offline detection and user-facing messaging in chat is real and still present. The codebase has a `NetworkError` path with a generic message but the offline scenario doesn't reliably surface it — the user sees 'Language model unavailable' or raw 'fetch failed' instead of a specific offline state with a retry action.

## Change summary

When VS Code loses network connectivity, Copilot Chat shows a vague 'fetch failed' or 'Language model unavailable' error instead of a clear offline state with a retry/reconnect action.

## Best solution

Wire the existing offline detection (context key `github.copilot.offline`, `net::ERR_INTERNET_DISCONNECTED` signals) through to the chat error rendering so users see a specific 'You appear to be offline' message with a retry button. The `ChatFetchResponseType.NetworkError` path in `src/vs/workbench/contrib/chat/common/chatErrorMessages.ts` already exists but needs to be reliably triggered and its message improved to mention connectivity explicitly. The copilot extension's offline context key should feed a distinct UI state in the chat widget.
