# Issue #239296 — Console stacktraces when resizing window/pane using fish shell and tide prompt

[https://github.com/microsoft/vscode/issues/239296](https://github.com/microsoft/vscode/issues/239296)

## Summary

Still reproduces on main: the VS Code CLI process (invoked by fish shell integration or tide prompt during resize) crashes with an unhandled EPIPE when console.log writes to a broken pipe. No EPIPE error handling exists in src/vs/code/node/cli.ts for its stdout writes.

## Change summary

Terminal resize with fish shell + tide async prompt causes unhandled EPIPE errors from the VS Code CLI process's console.log calls.

## Best solution

Add an error handler for EPIPE on process.stdout in the CLI entry point (src/vs/code/node/cli.ts or its bootstrap), similar to how src/vs/base/parts/ipc/node/ipc.net.ts silences EPIPE. This would gracefully ignore the broken pipe instead of crashing with an unhandled error.
