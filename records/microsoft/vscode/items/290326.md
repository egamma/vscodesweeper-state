# Issue #290326 — [Unhandled Error] Cannot read properties of undefined (reading 'toString')

[https://github.com/microsoft/vscode/issues/290326](https://github.com/microsoft/vscode/issues/290326)

## Summary

Still reproduces on current main — the `.map(c => c.toString())` at line 251 of `inlineEditsLineReplacementView.ts` has no null guard; when the theme lacks the expected color, `observeColor` throws/returns undefined, and the `.map` callback crashes. 1.1M hits / 42.7K users. Milestone is 'On Deck' so keeping open.

## Change summary

The inline edits line replacement view crashes with 'Cannot read properties of undefined' when resolving a border color that the current theme doesn't define.

## Best solution

Add a null guard in the `.map` callback at line 251 (e.g. `c => c ? \`1px solid ${c.toString()}\` : 'none'`), or ensure `getEditorBlendedColor` returns a safe fallback Color when the theme lacks the identifier. The fix is localized to `inlineEditsLineReplacementView.ts` line 251 and possibly `theme.ts` line 193.
