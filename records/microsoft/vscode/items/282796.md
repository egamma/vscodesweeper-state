# Issue #282796 — Move chat into secondary side-bar doesn't work with VScode version version 1.107

[https://github.com/microsoft/vscode/issues/282796](https://github.com/microsoft/vscode/issues/282796)

## Summary

Still reproduces on main: imported chat sessions cannot be moved to the sidebar because `shouldStoreSession` explicitly excludes imported sessions (`!session.isImported`), so when the editor closes during the move, the session data is lost and the view pane cannot reload it.

## Change summary

Moving an imported chat session from the editor area to the sidebar silently fails because the session is not persisted on editor close.

## Best solution

Either persist imported sessions the same way as regular sessions (remove the `!session.isImported` exclusion in `shouldStoreSession`, or add a special persist path for move operations), or restructure `moveToSidebar` to acquire the session reference in the view before closing the editor, so the model stays alive across the transition. The fix likely touches `src/vs/workbench/contrib/chat/common/chatService/chatServiceImpl.ts` (the `shouldStoreSession` guard) and/or `src/vs/workbench/contrib/chat/browser/actions/chatMoveActions.ts` (the `moveToSidebar` flow).
