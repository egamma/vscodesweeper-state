# Issue #104008 — authentication.onDidChangeSessions should fire when access is revoked by a user

[https://github.com/microsoft/vscode/issues/104008](https://github.com/microsoft/vscode/issues/104008)

## Summary

Partially implemented: `onDidChangeSessions` now fires when access is revoked (via `onDidChangeExtensionSessionAccess` in authenticationService.ts), but it broadcasts to all extensions rather than targeting just the affected one with an `extensionIdFilter`. The last maintainer comment (Dec 2024) acknowledges this remaining gap and plans to add targeted filtering.

## Change summary

Fire `authentication.onDidChangeSessions` targeted to the specific extension when a user revokes that extension's access to an authentication session.

## Best solution

In `src/vs/workbench/services/authentication/browser/authenticationService.ts` (line ~124), propagate the affected extension ID through the `onDidChangeExtensionSessionAccess` event payload and pass it as an `extensionIdFilter` when firing `_onDidChangeSessions`, mirroring how `onDidChangeAccountPreference` already does this in `mainThreadAuthentication.ts` line 150–153.
