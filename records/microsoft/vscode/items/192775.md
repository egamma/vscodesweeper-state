# Issue #192775 — [folding] Imports are not collapsed added first

[https://github.com/microsoft/vscode/issues/192775](https://github.com/microsoft/vscode/issues/192775)

## Summary

Still reproduces on main: the `_currentModelHasFoldedImports` flag in the folding controller is a one-shot guard — once imports are folded on first load, newly added imports won't be re-collapsed because the flag blocks subsequent folds. The feature only auto-folds imports once per model, not continuously.

## Change summary

When `editor.foldingImportsByDefault` is enabled and a new import is auto-added above a collapsed import block, the new import appears unfolded because the folding controller only auto-collapses imports once per file open.

## Best solution

In `src/vs/editor/contrib/folding/browser/folding.ts`, the logic at line 323 should detect when new import lines have been added to the import folding range and re-collapse them, rather than using a simple boolean one-shot flag. This could check whether the import range has grown (new lines added) and re-fold in that case while still respecting manual user unfolds.
