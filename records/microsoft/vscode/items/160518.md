# Issue #160518 — Command palette and customize layout hide behind the WCO

[https://github.com/microsoft/vscode/issues/160518](https://github.com/microsoft/vscode/issues/160518)

## Summary

Still reproduces on main: the window controls container (z-index 3000) renders above the quick input widget (z-index 2550) on narrow windows, causing the command palette to hide behind the WCO. Overflow handling added for #303222 only hides the center-adjacent toolbar, not the quick input.

## Change summary

On narrow windows, the command palette and customize layout controls hide behind the window control overlay buttons.

## Best solution

Constrain the quick input widget's max-width or right margin to account for the window controls container width, or clip/reposition the widget so it never overlaps with WCO. The fix likely lives in `src/vs/platform/quickinput/browser/quickInputController.ts` (positioning logic) and `src/vs/platform/quickinput/browser/media/quickInput.css`.
