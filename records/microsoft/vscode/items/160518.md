# Issue #160518 — Command palette and customize layout hide behind the WCO

[https://github.com/microsoft/vscode/issues/160518](https://github.com/microsoft/vscode/issues/160518)

## Summary

Still a valid bug on main: the window controls container uses z-index 3000 and can overlap the command palette, context menus, and layout controls when the window is narrow. Multiple issues were closed as duplicates of this one, confirming it's the canonical tracker.

## Change summary

Command palette and customize layout controls hide behind the Window Controls Overlay (WCO) when the VS Code window is narrow.

## Best solution

Ensure titlebar items (command center, layout controls, account buttons) either reflow or clip before they overlap with the window controls container area. The fix likely lives in `src/vs/workbench/browser/parts/titlebar/titlebarPart.ts` and its CSS — the titlebar content area needs to respect the WCO reserved width so items never render beneath it.
