# Issue #304655 — Can't contribute a boolean?

[https://github.com/microsoft/vscode/issues/304655](https://github.com/microsoft/vscode/issues/304655)

## Summary

Team-authored issue reporting that contributing a boolean configuration point (with enum) requires `enumItemLabels`/`enumItemDescriptions` to render properly in the Settings UI — keeping open per team-member policy.

## Change summary

A boolean configuration point contributed via extension doesn't render in Settings UI unless `enumItemLabels`/`enumItemDescriptions` are also provided.

## Best solution

The settings tree enum renderer in `src/vs/workbench/contrib/preferences/browser/settingsTree.ts` should gracefully handle missing `enumItemLabels` for boolean enum settings, either by auto-generating labels from the enum values (e.g. 'true'/'false') or by falling back to the boolean checkbox renderer when no labels are provided.
