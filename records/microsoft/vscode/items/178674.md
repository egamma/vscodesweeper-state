# Issue #178674 — narrator and NVDA does not report process manager list item names in Visual Studio Code

[https://github.com/microsoft/vscode/issues/178674](https://github.com/microsoft/vscode/issues/178674)

## Summary

Fixed on main: the process explorer was rewritten as an editor-based tree widget with a proper ProcessAccessibilityProvider that returns element.name as the ARIA label, shipped in 1.101.0.

## Change summary

Screen readers (NVDA/Narrator) only reported 'list item' for process manager entries instead of the process name.

## Best solution

Keep the shipped fix from commit 708b6aa379c (1.101.0); the process explorer now uses a WorkbenchDataTree with a ProcessAccessibilityProvider that exposes element names to screen readers.
