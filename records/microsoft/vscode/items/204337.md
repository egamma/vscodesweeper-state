# Issue #204337 — Quick switch window don't allow multiple key presses to go to next window

[https://github.com/microsoft/vscode/issues/204337](https://github.com/microsoft/vscode/issues/204337)

## Summary

Valid feature request confirmed by a team member (TylerLeonhardt). The window picker lacks the quick-navigate-next/previous keybinding registration that the editor and view pickers have; the gap still exists on current main.

## Change summary

Allow repeated key presses (e.g. Tab) to cycle through items in the Quick Switch Window picker, matching the behavior of the editor and view pickers.

## Best solution

Register `quickOpenNavigateNextInWindowPicker` and `quickOpenNavigatePreviousInWindowPicker` commands with keybinding rules (analogous to the editor/view picker patterns in `editor.contribution.ts` and `quickAccess.contribution.ts`), wired to `getQuickNavigateHandler`. The likely location is near the window picker registration, possibly in a new contribution block in `src/vs/workbench/electron-browser/actions/windowActions.ts` or its contribution file.
