# Issue #321143 — Smoke Test Failure: sidebar has no accessibility violations

[https://github.com/microsoft/vscode/issues/321143](https://github.com/microsoft/vscode/issues/321143)

## Summary

Team-authored issue; still reproduces on main. The default `getAriaLevel` returns `node.depth` (0-based) for root-level tree items, producing an invalid `aria-level="0"` attribute that violates WAI-ARIA spec (minimum is 1).

## Change summary

Sidebar tree items at depth 0 emit `aria-level="0"`, which is an invalid ARIA attribute value (must be ≥ 1).

## Best solution

Change the default `getAriaLevel` in `src/vs/base/browser/ui/tree/abstractTree.ts` (line 209) from `return node.depth` to `return node.depth + 1` so root nodes get `aria-level="1"`. Verify no consumer relies on 0-based aria-level semantics.
