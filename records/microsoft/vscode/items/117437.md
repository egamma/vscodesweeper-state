# Issue #117437 — [product icon themes] No fallback for using icon contribution point

[https://github.com/microsoft/vscode/issues/117437](https://github.com/microsoft/vscode/issues/117437)

## Summary

The core request — a formal fallback mechanism when a referenced icon from another extension is unavailable — remains unimplemented. The simpler case of referencing built-in codicons as a default already works, but the cross-extension fallback scenario described in the original report has no dedicated support on current main.

## Change summary

Request a fallback icon (e.g. a codicon) when using the icon contribution point and the icon's source extension is not installed.

## Best solution

Add a `fallback` property to the icon contribution schema (or support the `$(icon|fallback)` inline syntax discussed in comment 2) so extensions can gracefully degrade when a referenced icon font extension is missing. Implementation would touch `src/vs/workbench/services/themes/common/iconExtensionPoint.ts` and `src/vs/platform/theme/common/iconRegistry.ts`.
