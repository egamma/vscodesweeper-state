# Issue #117437 — [product icon themes] No fallback for using icon contribution point

[https://github.com/microsoft/vscode/issues/117437](https://github.com/microsoft/vscode/issues/117437)

## Summary

The requested fallback mechanism for product icon contributions (e.g. inline `$(fluent-logo|rocket)` syntax or a `fallback` property in the icon declaration) has not been implemented on main. The issue remains a valid feature request in the Backlog.

## Change summary

Allow extensions to specify a fallback codicon when contributing product icons, so missing icon fonts degrade gracefully instead of showing nothing.

## Best solution

Implement a fallback property in the icon contribution point schema (in `src/vs/workbench/services/themes/common/iconExtensionPoint.ts` and `src/vs/platform/theme/common/iconRegistry.ts`) that resolves to a built-in codicon when the referenced icon font is unavailable. The inline `$(icon|fallback)` syntax discussed in the thread is another viable approach.
