# Issue #236724 — Document how to use VS Code Tunnels reliably for more than 2 or 3 days without disconnection

[https://github.com/microsoft/vscode/issues/236724](https://github.com/microsoft/vscode/issues/236724)

## Summary

The tunnel disconnection after 2–5 days is a service-side reliability concern in the dev tunnels API (tunnels.api.visualstudio.com), not a VS Code client code bug. The CLI already has a token refresh loop (fe318d50e0c, 2023). The prior issue #230058 was closed as *out-of-scope for the same reason. The title asks for documentation of workarounds, but the underlying problem is upstream service token/session management.

## Change summary

VS Code tunnels disconnect after 2–5 days due to token expiry/service-side issues, requiring manual restart.

## Best solution

The root cause lives in the dev tunnels service (tunnels.api.visualstudio.com) token lifetime and session management, not in VS Code's client code. The CLI already implements a refresh loop. Improvements belong in the Azure Dev Tunnels service team's backlog, not VS Code core. A documentation request alone doesn't create an actionable code change here.
