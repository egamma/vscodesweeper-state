# Issue #286272 — Deleted branch is still listed in repositories, breaking actions

[https://github.com/microsoft/vscode/issues/286272](https://github.com/microsoft/vscode/issues/286272)

## Summary

Team-authored bug with 'On Deck' milestone — must stay open. The defect is real: after deleting a worktree, the parent repository's branch list is not refreshed in other windows, leaving stale branch entries that produce git errors when selected.

## Change summary

Deleting a worktree does not refresh the branch list in other VS Code windows, so the deleted branch remains listed and selecting it errors out.

## Best solution

After `deleteWorktree` completes in `extensions/git/src/repository.ts`, trigger a branch/status refresh that propagates to other windows watching the same repository. The `model.ts` repository-open flow and the branch-picker quick-pick should also validate that a branch's backing ref still exists before offering it.
