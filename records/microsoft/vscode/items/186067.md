# Issue #186067 — Terminal revive is storing sessions where nothing occurs

[https://github.com/microsoft/vscode/issues/186067](https://github.com/microsoft/vscode/issues/186067)

## Summary

Still reproduces on main: `serializeTerminalState` persists terminals in `InteractionState.ReplayOnly` (revived but never typed into), because `hasWrittenData` checks `!= None` rather than requiring `Session`. The fix gap remains.

## Change summary

Terminal persistence stores revived sessions even when the user never typed anything into them, creating unnecessary data and a perpetual revive cycle.

## Best solution

Change the `hasWrittenData` getter (or the serialization guard in `serializeTerminalState`) to require `InteractionState.Session` — only serialize terminals that received actual user input, not merely replayed ones. Similarly update the `detach` guard at line 837. File: `src/vs/platform/terminal/node/ptyService.ts`.
