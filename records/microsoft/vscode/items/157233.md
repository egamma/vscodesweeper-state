## Change summary

Middle-click paste of X primary selection in the integrated terminal fails when clicking on an area that already contains text.

## Best solution

The middle-click handler at src/vs/workbench/contrib/terminal/browser/terminalInstance.ts needs to clear the terminal's internal selection (or prevent the terminal from intercepting the mousedown) so the X11 paste event reaches the terminal input layer.
