# Issue #317214 — Chat thinking output can render code blocks as blank-looking bordered blocks

[https://github.com/microsoft/vscode/issues/317214](https://github.com/microsoft/vscode/issues/317214)

## Summary

Still present on main: the thinking content part's sync code block renderer produces a visible bordered shell even when the code text is empty or not painted, because the CSS on `[data-code]` applies background/border unconditionally. No fix has been committed since the report.

## Change summary

Chat thinking/commentary output renders fenced code blocks as blank bordered rectangles when an extension-contributed model streams LanguageModelThinkingPart content containing code fences.

## Best solution

Guard `ChatThinkingContentPart._codeBlockRendererSync` (or the CSS rule for `.chat-thinking-item.markdown-content [data-code]`) so that empty or whitespace-only code blocks either collapse to zero height or are not rendered. The fix is localized to `src/vs/workbench/contrib/chat/browser/widget/chatContentParts/chatThinkingContentPart.ts` and its CSS.
