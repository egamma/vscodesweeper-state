# Issue #320237 — Dropping views into a single-view merged-header custom view container is hard to hit

[https://github.com/microsoft/vscode/issues/320237](https://github.com/microsoft/vscode/issues/320237)

## Summary

Legitimate UX bug: when a custom view container is in single-view merged-header state, the pane header is hidden and the webview content area (even with pointer-events:none during drag) makes the drop target unreliable. The reporter provided the requested GIF and the info-needed label can be removed.

## Change summary

Dropping views into a single-view merged-header custom view container is very difficult because the hidden pane header leaves no reliable drag-and-drop target area.

## Best solution

The drop target registration in `viewPaneContainer.ts` uses `pane.dropTargetElement` (the full pane element), but in practice the merged-header state hides the header and the webview iframe dominates the area. A fix could add a fallback drop zone on the container title area or expand the hit area for the merged-header state. Area: workbench views/viewPaneContainer.
