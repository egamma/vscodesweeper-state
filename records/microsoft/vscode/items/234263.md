# Issue #234263 — Light bulb shows up for move to new file?

[https://github.com/microsoft/vscode/issues/234263](https://github.com/microsoft/vscode/issues/234263)

## Summary

Still reproduces on main: the lightbulb widget has no filtering logic to suppress display when the only available code actions are 'Move to file' refactoring actions. The `computeLightBulbInfo` function in `lightBulbWidget.ts` and the `update` method show no action-kind-based filtering — any non-empty `validActions` set triggers the lightbulb.

## Change summary

The lightbulb icon appears when the only available code actions are 'Move to file' refactoring actions, which is noisy and unhelpful for the user.

## Best solution

Add action-kind filtering in `computeLightBulbInfo` or `LightBulbWidget.update` (in `src/vs/editor/contrib/codeAction/browser/lightBulbWidget.ts`) to suppress the lightbulb when the only available actions are move-to-file refactoring actions (CodeActionKind `refactor.move`). The TypeScript language service provides these via `_typescript.moveToFileRefactoring` in `extensions/typescript-language-features/src/languageFeatures/refactor.ts`.
