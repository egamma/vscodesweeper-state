# Issue #178075 — De-indentation breaks pair autoclosing

[https://github.com/microsoft/vscode/issues/178075](https://github.com/microsoft/vscode/issues/178075)

## Summary

Still reproduces on main: the auto-closing pair logic in AutoIndentOperation evaluates the line context before the de-indentation edit is applied, so when indentation is removed the auto-close check sees stale character context and may skip inserting the closing character.

## Change summary

Typing an auto-closing open character (like `{`) that triggers de-indentation fails to insert the matching close character.

## Best solution

In `AutoIndentOperation.getEdits()` (src/vs/editor/common/cursor/cursorTypeEditOperations.ts), the call to `getAutoClosingPairClose` should evaluate the line content *after* the indentation adjustment, not before. The `TypeWithIndentationAndAutoClosingCommand` class already handles emitting both edits, but the decision logic uses stale model state.
