# Issue #229511 — Chat file attachment - disambiguate when file name matches

[https://github.com/microsoft/vscode/issues/229511](https://github.com/microsoft/vscode/issues/229511)

## Summary

Team-authored issue, still unfixed on main. File attachment labels use `hidePath: true` with no disambiguation logic for same-named files — the bug remains.

## Change summary

When attaching multiple files with the same name from different directories, the chat input shows identical labels with no way to tell them apart except by hovering.

## Best solution

Add path disambiguation to file attachment labels in `chatAttachmentWidgets.ts` — when multiple attached files share the same `basename`, include enough parent directory segments to distinguish them (similar to how the editor tab title bar handles this via `ILabelService`). The related issue https://github.com/microsoft/vscode/issues/229572 covers the same problem in chat responses and could be addressed together.
