# Issue #120922 — The scroll bar  is not accessible through keyboard key present on the error pop-up

[https://github.com/microsoft/vscode/issues/120922](https://github.com/microsoft/vscode/issues/120922)

## Summary

Keep open: this is a real accessibility gap in the marker navigation/error widget, and I did not find evidence that current main has implemented keyboard access for the popup scrollbar. The bot-suggested issue is related to error popup scrolling but tracks mouse-wheel dismissal behavior, not keyboard scrollbar access.

## Change summary

The error/marker popup opened from Shift+F8 should allow keyboard users to scroll overflowing diagnostic text horizontally or vertically.

## Best solution

Make the marker navigation message area keyboard-scrollable, likely in `src/vs/editor/contrib/gotoError/browser/gotoErrorWidget.ts` and/or the shared `ScrollableElement` in `src/vs/base/browser/ui/scrollbar`, with appropriate focus and key handling for arrow/Page/Home/End navigation.

## Evidence

- **issue scope:** https://github.com/microsoft/vscode/issues/120922 reports Shift+F8 into the error popup, then trying to use keyboard keys to move the popup scrollbar.
- **current widget code:** `src/vs/editor/contrib/gotoError/browser/gotoErrorWidget.ts` creates a `ScrollableElement` for the diagnostic message and makes only the parent marker widget focusable with `tabIndex = 0`; there is no keydown handling for scrolling the message area.
- **scrollbar implementation:** `src/vs/base/browser/ui/scrollbar/abstractScrollbar.ts` marks scrollbar DOM nodes as `role='presentation'` and `aria-hidden='true'`, with pointer handling but no keyboard focus path.
- **related item checked:** The duplicate detector suggested https://github.com/microsoft/vscode/issues/116061, but that issue is about scrolling closing error popups and was closed as not reproducible; it does not cover keyboard access to the scrollbar.
- **maintainer routing:** A member comment on 2021-10-06 noted this likely belongs to the scrollbar widget owner, matching the existing `editor-scrollbar` label.
