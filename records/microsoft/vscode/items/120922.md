# Issue #120922 — The scroll bar  is not accessible through keyboard key present on the error pop-up

[https://github.com/microsoft/vscode/issues/120922](https://github.com/microsoft/vscode/issues/120922)

## Summary

Still reproduces on main: the ScrollableElement widget used in the marker navigation popup has no keyboard scroll support — it only responds to mouse events. The accessibility gap remains.

## Change summary

The scrollbar in the Shift+F8 error popup cannot be scrolled via keyboard when focus is inside the popup.

## Best solution

Add keyboard scroll support (arrow keys / Page Up/Down) to the marker navigation widget's scrollable container, likely in `src/vs/editor/contrib/gotoError/browser/gotoErrorWidget.ts` by handling keydown events on the scrollable element's DOM node. Alternatively, the base `ScrollableElement` could gain optional keyboard navigation support.
