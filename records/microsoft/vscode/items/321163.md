# Issue #321163 — Agents window: asking copilot to review a PR checks out a PR but there's no PR button

[https://github.com/microsoft/vscode/issues/321163](https://github.com/microsoft/vscode/issues/321163)

## Summary

Team-authored issue reporting a missing PR button in the Changes view after Copilot checks out a PR in the Agents window — must stay open for maintainer judgment.

## Change summary

When Copilot reviews a PR in the Agents window and checks it out, there is no PR button visible in the Changes view, leaving the user without a clear flow to interact with the PR.

## Best solution

Ensure that when an agent checks out a PR branch, the Changes view surfaces the PR association (a PR button or link) so the user can navigate to the PR context. The flow should also clarify whether 'Folder' vs 'Worktree' mode is appropriate for PR review sessions.
