# Issue #116840 — [html] handle HTML-escaped characters in links

[https://github.com/microsoft/vscode/issues/116840](https://github.com/microsoft/vscode/issues/116840)

## Summary

Still open. The bug lives in the upstream vscode-html-languageservice library (findDocumentLinks in src/services/htmlLinks.ts), which extracts href values without decoding HTML entities. No fix has landed in vscode-html-languageservice or in the VS Code repo.

## Change summary

Clickable links in HTML/XML documents fail to decode HTML entity escapes (e.g. &amp;), resulting in corrupted URLs.

## Best solution

Fix findDocumentLinks in microsoft/vscode-html-languageservice (src/services/htmlLinks.ts) to decode HTML entities in href attribute values before constructing the DocumentLink target URL.
