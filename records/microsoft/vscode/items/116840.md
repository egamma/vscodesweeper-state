# Issue #116840 — [html] handle HTML-escaped characters in links

[https://github.com/microsoft/vscode/issues/116840](https://github.com/microsoft/vscode/issues/116840)

## Summary

Still reproduces on main: the HTML extension delegates link extraction to `vscode-html-languageservice.findDocumentLinks` which does not decode HTML entities (like `&amp;`) in href values before constructing link targets. No fix found in git history or in the current server code.

## Change summary

Clickable links in HTML/XML documents containing HTML entity escapes (e.g. `&amp;`) navigate to a corrupted URL instead of decoding entities first.

## Best solution

Add HTML entity decoding for link targets either in the upstream `vscode-html-languageservice` package's `findDocumentLinks` implementation, or as a post-processing step in `extensions/html-language-features/server/src/modes/htmlMode.ts` before returning links.
