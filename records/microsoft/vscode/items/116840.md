# Issue #116840 — [html] handle HTML-escaped characters in links

[https://github.com/microsoft/vscode/issues/116840](https://github.com/microsoft/vscode/issues/116840)

## Summary

Still reproducible on main. The bug lives in the upstream `vscode-html-languageservice` package — `normalizeRef()` in `htmlLinks.js` strips quotes but never decodes HTML entities before constructing document links.

## Change summary

Clickable links in HTML/XML attributes containing HTML entities (e.g. `&amp;`) navigate to corrupted URLs because the entity is not decoded before link resolution.

## Best solution

Fix upstream in `microsoft/vscode-html-languageservice`: add HTML entity decoding (at minimum `&amp;`, `&lt;`, `&gt;`, `&quot;`, `&apos;`) in `normalizeRef()` or `createLink()` in `src/services/htmlLinks.ts`, then bump the dependency in VS Code's `extensions/html-language-features/server/package.json`.
