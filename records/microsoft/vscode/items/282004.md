# Issue #282004 â€” Dangerous keyboard shortcut in Copilot Chat: Delete triggers multi file undo from chat input

[https://github.com/microsoft/vscode/issues/282004](https://github.com/microsoft/vscode/issues/282004)

## Summary

Still present on main: the 'Undo Requests' action (`workbench.action.chat.undoEdits`) is bound to `Cmd+Backspace` (Mac) / `Delete` (Windows), which fires when focus is on the chat list (not in the input editor). Users who just submitted a message or clicked outside the input can accidentally trigger the multi-file undo dialog. Multiple duplicate reports exist (#306770, #317320), confirming this is a recurring UX hazard.

## Change summary

The Undo Requests keybinding (`Cmd+Backspace`/`Delete`) triggers too easily from within the chat panel, leading to accidental multi-file undo when the user intended to edit chat input text.

## Best solution

Harden the keybinding for `workbench.action.chat.undoEdits` â€” either require a more deliberate modifier combination (not a single key or common text-editing chord), or add a guard that requires the chat list item to be explicitly selected/focused rather than just having `inChatSession` context. The earlier closed issue #280443 noted this would be revisited if more reports came in â€” they have. Issues #306770 and #317320 are duplicates that should be consolidated here.
