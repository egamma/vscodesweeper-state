# Issue #91086 — [npm] load description/homepage from referenced node module version

[https://github.com/microsoft/vscode/issues/91086](https://github.com/microsoft/vscode/issues/91086)

## Summary

Still unimplemented on main. The npm extension's hover always queries the npm registry by package name, ignoring GitHub-style version specifiers. A maintainer welcomed a PR in 2020 but none was submitted.

## Change summary

When a package.json dependency uses a GitHub reference (e.g. `azer/indexeddb`), the hover should show the description/homepage from that GitHub repo, not the unrelated npm package of the same name.

## Best solution

In `extensions/npm/src/features/packageJSONContribution.ts`, detect GitHub-style version specifiers (`github:user/repo`, `user/repo`) and either skip the npm-registry lookup or resolve metadata from GitHub instead. The maintainer indicated willingness to accept a PR for this.
