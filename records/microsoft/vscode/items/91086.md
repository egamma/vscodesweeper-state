# Issue #91086 — [npm] load description/homepage from referenced node module version

[https://github.com/microsoft/vscode/issues/91086](https://github.com/microsoft/vscode/issues/91086)

## Summary

Still unimplemented on main. The npm extension's `npmView` always queries the registry by package name alone, ignoring GitHub-shorthand version specifiers like `azer/indexeddb`. A maintainer accepted the request and invited a PR, but none was submitted.

## Change summary

When a dependency uses a GitHub shorthand (e.g. `azer/indexeddb`), the hover tooltip should show the homepage/description of the referenced GitHub repo rather than the unrelated npm registry entry.

## Best solution

In `extensions/npm/src/features/packageJSONContribution.ts`, detect non-registry version specifiers (GitHub shorthands, `github:` prefixes, git URLs) in `fetchPackageInfo` and either skip the `npm view` call or pass the full specifier so the correct metadata is resolved. Given 0 upvotes and 5+ years of inactivity, this could also be closed via the community feature-request process if no contributor picks it up.
