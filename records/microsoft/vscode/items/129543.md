# Issue #129543 — Instruction breakpoints should maybe use a different breakpoint icon

[https://github.com/microsoft/vscode/issues/129543](https://github.com/microsoft/vscode/issues/129543)

## Summary

Still valid: instruction breakpoints use the generic breakpoint icon on current main. No dedicated icon set has been added to debugIcons.ts, and the getBreakpointMessageAndIcon function falls through to `icons.breakpoint` for InstructionBreakpoint instances. The design discussion (red square) happened but was never implemented.

## Change summary

Instruction breakpoints should use a visually distinct icon (e.g. a red square) instead of sharing the regular breakpoint's red circle.

## Best solution

Add a new `instructionBreakpoint` icon set (regular/disabled/unverified) to `src/vs/workbench/contrib/debug/browser/debugIcons.ts` with a distinct codicon (likely requiring a new codicon glyph), wire it into `getBreakpointMessageAndIcon` at line 1824 of `breakpointsView.ts`, and update the disassembly view gutter rendering in `disassemblyView.ts`. Needs design sign-off on the glyph shape (red square was proposed by @miguelsolorio).
