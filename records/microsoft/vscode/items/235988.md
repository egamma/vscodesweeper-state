# Issue #235988 — Terminal ligatures have limited length even when the font supports longer ligatures

[https://github.com/microsoft/vscode/issues/235988](https://github.com/microsoft/vscode/issues/235988)

## Summary

Properly triaged upstream issue with a partial fix already landed (xterm.js texture-size limit) and remaining work (cell-splitting for unlimited ligature width) still tracked. Maintainer explicitly kept it open.

## Change summary

Terminal ligatures have a limited rendering width even when the font supports longer ligatures, causing characters to vanish or render incorrectly beyond the texture-size limit.

## Best solution

The full fix lives in xterm.js: split ligature glyphs into cell-sized chunks so length is unlimited. Track upstream progress at https://github.com/xtermjs/xterm.js/issues/5246 and pull the xterm.js update once it lands.
