This is the canonical feature request for improving the missing Git identity UX. Another issue (#144738) was closed as a duplicate of this one. The issue carries `help wanted` and sits in `Backlog`, so it should remain open for community contribution.

## Change summary

When user.name or user.email is not configured, VS Code should offer an interactive flow to set them instead of only showing a static error notification.

## Best solution

Implement an interactive recovery flow in `extensions/git/src/commands.ts` at the `NoUserNameConfigured`/`NoUserEmailConfigured` error handler (lines 5623–5627): prompt for name and email via InputBox, let the user choose global vs local scope, write the config, and retry the failed command. The `help wanted` label makes this a good community contribution target.
