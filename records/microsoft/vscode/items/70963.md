# Issue #70963 — Git - .gitignore'd files sometimes not greyed-out

[https://github.com/microsoft/vscode/issues/70963](https://github.com/microsoft/vscode/issues/70963)

## Summary

Still a valid bug on current main — the GitIgnoreDecorationProvider only refreshes on .gitignore save or repo open/close events, missing other triggers (branch switches, external git operations, initial load races). 43 upvotes and marked help-wanted; should stay open.

## Change summary

Files listed in .gitignore intermittently fail to appear greyed out in the Explorer, requiring a manual .gitignore re-save to fix.

## Best solution

Broaden the event triggers in `GitIgnoreDecorationProvider` (extensions/git/src/decorationProvider.ts) to also fire decoration refreshes on repository status changes and branch switches, not only on .gitignore file saves. The debounced `checkIgnoreSoon` batching is fine, but the upstream events that invalidate the cache are too narrow. The `help wanted` label is appropriate given the intermittent nature.
