# Issue #244043 — Support `downloadExtensionsLocally` in bulk extension install APIs, when in remote

[https://github.com/microsoft/vscode/issues/244043](https://github.com/microsoft/vscode/issues/244043)

## Summary

Still open on main: the electron-browser `NativeRemoteExtensionManagementService` overrides `installFromGallery` with the `downloadExtensionsLocally` fallback but never overrides `installGalleryExtensions`, so the bulk API path skips local-download logic when the remote cannot reach the marketplace.

## Change summary

Bulk extension install APIs in remote scenarios don't respect the `downloadExtensionsLocally` fallback, unlike the single-extension `installFromGallery` API.

## Best solution

Override `installGalleryExtensions` in `NativeRemoteExtensionManagementService` (electron-browser) to apply the same `downloadExtensionsLocally` / error-fallback pattern that `doInstallFromGallery` uses, then simplify `remoteExtensionsInit.ts` to call the bulk API instead of looping single installs.
