# Issue #257648 — File explorer sticky folder UI bug

[https://github.com/microsoft/vscode/issues/257648](https://github.com/microsoft/vscode/issues/257648)

## Summary

Still reproducible on main: the sticky scroll container lacks `overflow: hidden`, so the last partially-overlapped sticky row can visually leak beyond the container bounds on hover. A team member (benibenj) is already engaged.

## Change summary

When the secondary sticky folder is about to be overlapped by the first sticky folder, hovering the first folder causes the overlapped portion of the secondary folder to visually reveal.

## Best solution

Add `overflow: hidden` to the `.monaco-tree-sticky-container` CSS rule in `src/vs/base/browser/ui/tree/media/tree.css` so child rows are clipped to the container's dynamically-set height.
