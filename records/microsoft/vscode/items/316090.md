# Issue #316090 — Multi process logging / perf tracing

[https://github.com/microsoft/vscode/issues/316090](https://github.com/microsoft/vscode/issues/316090)

## Summary

Valid feature request for unified multi-process logging and perf tracing across renderer, extension host, and agent host. Existing OTel infrastructure in the repo is narrowly scoped to the agent host; the broader cross-process correlation described here is not yet implemented. Keeping open — low traction so far but thematically aligned with performance investment.

## Change summary

Add a unified multi-process logging and tracing service that can correlate events across VS Code's renderer, extension host, and agent host processes, potentially using OpenTelemetry.

## Best solution

Evaluate extending the existing `src/vs/platform/otel/` infrastructure (currently agent-host-scoped) into a general cross-process tracing layer, or build a new lightweight service in `src/vs/platform/log/` that propagates trace/span IDs across IPC boundaries. The related internal backlog item (OTel Studio) suggests this is already under consideration.
