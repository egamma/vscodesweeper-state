# Issue #300819 — unable to customize the GIT `scm.providerCountBadge` color

[https://github.com/microsoft/vscode/issues/300819](https://github.com/microsoft/vscode/issues/300819)

## Summary

Feature request for an SCM-specific badge color token. The badge currently uses the global `badge.background`/`badge.foreground` theme colors (customizable via `workbench.colorCustomizations`), but there is no dedicated SCM-scoped override token.

## Change summary

Request a dedicated color customization token for the SCM provider count badge, separate from the global badge color.

## Best solution

The user can already change the badge color globally via `workbench.colorCustomizations` → `badge.background` / `badge.foreground`. A dedicated SCM-specific token (e.g. `scm.providerCountBadge.background`) would require registering new colors in the SCM contribution and wiring them through `getCountBadgeStyle` with an override. Whether this granularity is warranted is a product call for the SCM area owner.

## Vision fit

- **Fit:** unclear
- **Reason:** Fine-grained per-component color tokens are a borderline case between UI consistency (values favor uniform theming) and legitimate customization depth.
