# Issue #231128 — font ligatures that span several textmate scopes assume the font settings of last character (last component of the composite symbol)

[https://github.com/microsoft/vscode/issues/231128](https://github.com/microsoft/vscode/issues/231128)

## Summary

Verified duplicate of #43440 — the canonical tracking issue for ligatures rendering incorrectly across token/scope boundaries. A team member (alexdima) explicitly redirected a near-identical report (#278929) there.

## Change summary

Font ligatures that span multiple TextMate scopes inherit the color of the last span, causing incorrect colorization.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/43440, which is in the Backlog and tracks the same underlying problem: ligatures need to be broken (or suppressed) at token boundaries to preserve per-scope coloring.
