# Issue #294056 — MCPs that need to be started for a given prompt are auto-enabled

[https://github.com/microsoft/vscode/issues/294056](https://github.com/microsoft/vscode/issues/294056)

## Summary

Team-authored bug report about MCP servers being auto-enabled during prompt execution when they haven't been explicitly started — kept open per team-member policy.

## Change summary

Running a prompt file with `tools: []` auto-starts and enables unstarted MCP servers for the turn, bypassing the user's explicit enablement decision.

## Best solution

The prompt execution path should respect the user's MCP enablement state — tools from MCPs that haven't been explicitly started should not be auto-enabled just because a prompt triggers. The fix likely lives in the chat tool resolution or MCP session startup logic under `src/vs/workbench/contrib/chat/`.
