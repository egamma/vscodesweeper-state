# Issue #203897 — Should dispose code actions after auto-running AI action?

[https://github.com/microsoft/vscode/issues/203897](https://github.com/microsoft/vscode/issues/203897)

## Summary

Still valid on current main: `showCodeActionsFromLightbulb` at line 148–157 auto-runs a single AI action without disposing the `CodeActionSet`, unlike all other auto-run paths in the same file. The original fix (PR #203936) was reverted (PR #205508) due to issues, and the disposal gap remains. Team members are actively discussing ownership.

## Change summary

The CodeActionSet is not disposed after auto-running a single AI code action from the lightbulb, potentially leaking resources.

## Best solution

Add `actions.dispose()` in `showCodeActionsFromLightbulb` after the `applyCodeAction` call completes (line 156–157 in `codeActionController.ts`), mirroring the try/finally pattern used in the other auto-apply path at lines 240–245. The previous attempt was reverted because it disposed too eagerly in other paths — this time scope the fix narrowly to the AI lightbulb auto-run branch only.
