# Issue #158381 — editor.unicodeHighlight.ambiguousCharacters not accessible

[https://github.com/microsoft/vscode/issues/158381](https://github.com/microsoft/vscode/issues/158381)

## Summary

Team-authored accessibility gap that remains unaddressed on main — no aria announcement or audio cue is played when the cursor visits an ambiguous unicode character. Keeping open per hard keep-open rule (MEMBER author) and because it's a valid accessibility concern.

## Change summary

Screen readers do not announce anything special when the cursor lands on a character flagged as ambiguous by the unicode highlighter.

## Best solution

Add an aria-live announcement (e.g. "U+XXXX, ambiguous character") when the cursor lands on a decorated ambiguous character, and optionally an accessibility signal. The implementation likely belongs in `src/vs/editor/contrib/unicodeHighlighter/browser/unicodeHighlighter.ts`, listening to cursor position changes and checking decoration overlap.
