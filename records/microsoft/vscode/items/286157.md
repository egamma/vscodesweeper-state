# Issue #286157 — Default semantic token styles do not honour custom line heights or font sizes from TextMate themes

[https://github.com/microsoft/vscode/issues/286157](https://github.com/microsoft/vscode/issues/286157)

## Summary

Team-authored feature request discussing how semantic token styling inherits textMateRules (line height, font size) via scope resolution. Kept open per policy for team-authored issues; active discussion among members about whether the current behavior is as-designed or should be changed.

## Change summary

Semantic tokens that resolve to a TextMate scope inherit custom line height/font size rules from that scope, even when they weren't directly targeted by the user's textMateRules configuration.

## Best solution

Decide whether semantic token fallback styling should inherit all properties (including line height/font size) from resolved TextMate scopes, or only a subset (e.g. color/fontStyle). The fix would likely live in `src/vs/editor/common/services/semanticTokensProviderStyling.ts` and/or `src/vs/workbench/services/themes/common/colorThemeData.ts`, gating which style properties propagate through the scope-resolution path.
