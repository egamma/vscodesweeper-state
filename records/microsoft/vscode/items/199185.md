# Issue #199185 — Merge Editor: Current is not always HEAD for every merge conflict

[https://github.com/microsoft/vscode/issues/199185](https://github.com/microsoft/vscode/issues/199185)

## Summary

Team-authored issue (hard keep-open). The bug is still present on main: the merge editor uses HEAD/REBASE_HEAD commits for current/incoming URIs rather than git's stage markers (:1:/:2:/:3:), which is incorrect when rebasing multiple conflicting commits.

## Change summary

During rebase with multiple conflicting commits, the merge editor's 'Current' side doesn't always correspond to HEAD — it should use git's index stage content instead.

## Best solution

Replace the HEAD-based URI construction (lines ~895-905 in extensions/git/src/commands.ts) with git index stage references (:1: base, :2: ours, :3: theirs) so the merge editor shows the correct content regardless of how many commits are being rebased.
