# Issue #318095 — [feat] Allow Terminal in panel and Secondary Side Bar simultaneously

[https://github.com/microsoft/vscode/issues/318095](https://github.com/microsoft/vscode/issues/318095)

## Summary

Valid feature request for allowing terminal views in both the panel and a side bar simultaneously. The terminal is registered as a single view that can only exist in one container at a time — a genuine architectural limitation. This is the canonical issue; #318570 is a duplicate of it.

## Change summary

Allow independent terminal instances to exist in both the bottom panel and the secondary side bar simultaneously, rather than forcing a single location.

## Best solution

This requires workbench layout/view infrastructure changes to allow a view to exist in multiple containers, or a second terminal view registration (e.g. a 'Terminal (Secondary)' view). The fix likely spans `src/vs/workbench/contrib/terminal/browser/terminal.contribution.ts` and the view descriptor service. Given the AI-agent-terminal use case is growing, this is a reasonable request to keep open for the terminal and layout owners to evaluate.
