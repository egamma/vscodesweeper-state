# Issue #248759 — Sidebar "Run" Button Fails with command not found: npm on Linux (works on macOS)

[https://github.com/microsoft/vscode/issues/248759](https://github.com/microsoft/vscode/issues/248759)

## Summary

The report is well-written and describes a known class of issue (task ShellExecution not inheriting version-manager PATH on Linux when launched from GUI). The `confirmation-pending` label is appropriate — more diagnostic info would help narrow down whether VS Code's shell env resolution is failing entirely or the task terminal's non-interactive shell invocation is the root cause.

## Change summary

NPM Scripts sidebar 'Run' button fails with 'command not found: npm' on Linux when VS Code is launched from GUI, because the task shell doesn't inherit version-manager PATH modifications.

## Best solution

Determine whether the issue is in shell environment resolution at startup (check Developer: Log process env) or in the task terminal's shell invocation pattern (non-login `-c` flag vs the integrated terminal's interactive session). If the former, the fix is in `src/vs/platform/shell/node/shellEnv.ts`; if the latter, the task system in `src/vs/workbench/contrib/tasks/browser/terminalTaskSystem.ts` needs to ensure the resolved environment is propagated. The user workaround is launching VS Code from terminal.

## Evidence

- **task execution path:** The npm extension creates tasks via `new ShellExecution(scriptRunner, args, { cwd })` in `extensions/npm/src/tasks.ts:345`, which runs as `/bin/zsh -c 'npm run ...'` — a non-interactive, non-login shell invocation.
- **shell env resolution exists:** `src/vs/platform/shell/node/shellEnv.ts` does resolve shell env on Linux GUI launch by spawning a login shell, but this resolved env may not fully propagate to task terminals when version managers require interactive shell init.
- **Mac vs Linux difference:** In `terminalTaskSystem.ts:1305`, there's Mac-specific handling commented out for login shell flags (`-l`), but no equivalent Linux handling — both use just `-c` for command execution.
- **known issue class:** Issues #85753 and #78784 report the same symptom (nvm/fnm-managed npm not found in tasks) — both were closed (fixed for specific providers, or info-needed) without a general fix for Linux GUI-launched task shells.
