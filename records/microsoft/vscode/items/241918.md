# Issue #241918 — Warning when attaching image from localhost or private sites

[https://github.com/microsoft/vscode/issues/241918](https://github.com/microsoft/vscode/issues/241918)

## Summary

Still an open design question: when a user drags an image from localhost or a private/auth-gated site into chat, the model can't access the URL. The fallback to download-and-convert-to-base64 is not yet implemented. The discussion is active and the area owner acknowledged the gap — keeping open as under-discussion.

## Change summary

Images dragged from localhost or auth-gated sites into chat produce a warning because the model cannot access the URL, and no base64 fallback exists.

## Best solution

Implement the base64 fallback path in `chatDragAndDrop.ts`: when the image URL points to localhost or a private site, download the image locally and send it as inline data rather than a URL reference. The `downloadImageAsUint8Array` method already attempts this via `webContentExtractorService.readImage`, but it currently falls back to inserting the raw URL text on failure. The gap is making the local download succeed for localhost/private origins or using the browser's already-loaded image data from the drag event.
