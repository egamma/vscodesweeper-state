# Issue #174198 — Git - Cancel git Sync's pull (and the sync action itself) if behind the remote

[https://github.com/microsoft/vscode/issues/174198](https://github.com/microsoft/vscode/issues/174198)

## Summary

Valid feature request for smarter git Sync behavior. A team member (lszomoru) engaged constructively and suggested separate Pull/Push buttons as the better path forward, which overlaps with the broader request in #174088. The core ask — preventing an unreviewed merge+push — remains unaddressed on main. Keeping open as the maintainer's own suggestion has not yet been implemented.

## Change summary

The git Sync button should not automatically pull-merge-push when behind the remote; the user should get a chance to review before a merge commit is pushed.

## Best solution

Implement the approach lszomoru suggested: provide configurable separate Pull/Push action buttons in the SCM view (showing Pull when behind, Push when ahead, Fetch otherwise) as an alternative to the Sync button. This overlaps with https://github.com/microsoft/vscode/issues/174088 — consider whether one issue should absorb the other. The work lives in the SCM action button logic in `src/vs/workbench/contrib/scm/`.
