## Summary

Team-authored issue (MEMBER) reporting that Copilot's inline editing toolbar and 'View all edits' do not function for CustomTextEditorProvider-backed documents — keeping open for maintainer judgment.

## Change summary

The floating inline toolbar (Keep/Undo/Toggle Diff) is non-functional and 'View all edits' throws an error when Copilot edits are applied to files opened via a CustomTextEditorProvider.

## Best solution

Either wire the toolbar actions to operate on the underlying TextDocument model (bypassing the missing CodeEditorWidget) for custom editors, or suppress the toolbar when the active editor is webview-based. The 'View all edits' path in chatEditingEditorOverlay.ts needs a graceful fallback when getEditorIntegration() encounters a non-code editor.
