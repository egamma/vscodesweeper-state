# Issue #324992 — MCP tool names collide when multiple servers report the same serverInfo.name, producing unstable _2/_3 prefixes

[https://github.com/microsoft/vscode/issues/324992](https://github.com/microsoft/vscode/issues/324992)

## Summary

Team-authored issue; the defect is confirmed in current main — McpPrefixGenerator assigns collision suffixes by arrival order, making them unstable across reloads when multiple servers share the same serverInfo.name.

## Change summary

MCP tool name prefixes are derived from serverInfo.name with non-deterministic numeric suffixes when names collide, instead of using the stable unique mcp.json server key.

## Best solution

Derive the tool prefix from the mcp.json server key (definition.label) when multiple servers would collide on serverInfo.name, or always seed the prefix from the key. The fix is localized to McpPrefixGenerator.take() and the preferredName derivation in mcpServer.ts (~line 578).
