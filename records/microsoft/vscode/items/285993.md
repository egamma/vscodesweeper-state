# Issue #285993 — Azure Resources extension shows “Sign in to Azure…” even though I’m signed in to Azure ChinaCloud (previously worked)

[https://github.com/microsoft/vscode/issues/285993](https://github.com/microsoft/vscode/issues/285993)

## Summary

Confirmed upstream bug in MSAL library (microsoft-authentication-library-for-js) affecting sovereign cloud token cache lookups. Team member TylerLeonhardt reproduced it and confirmed with the MSAL team that a fix is in progress. Already correctly labeled as upstream; keeping open as a tracking issue until the MSAL fix lands.

## Change summary

Azure sovereign cloud (ChinaCloud/USGovCloud) sign-in succeeds but MSAL fails to find the refresh token in cache, leaving the Azure Resources view stuck on 'Sign in to Azure…'.

## Best solution

Wait for the MSAL team to ship their fix in microsoft-authentication-library-for-js, then bump the MSAL dependency in VS Code's microsoft-authentication extension. No VS Code core change needed — the root cause is entirely in the upstream library's token cache lookup for sovereign cloud scopes.
