# Issue #262495 — Remove the 'Keep' button in Chat Agent mode and use standard save behavior

[https://github.com/microsoft/vscode/issues/262495](https://github.com/microsoft/vscode/issues/262495)

## Summary

Team-authored feature request with 25 upvotes and strong internal agreement. The Keep/Discard UX is still fully present on current main (AcceptAction/'Keep' in chatEditingEditorActions.ts, ChatEditingSession infrastructure). No implementation toward removing or bypassing it was found. Keeping open — needs a product decision from the area owners.

## Change summary

Remove the intermediate 'Keep/Discard' state for agent edits and treat them as normal file edits governed by standard save/auto-save behavior.

## Best solution

Introduce a configurable mode (e.g. `chat.editing.mode: 'direct'`) that bypasses the ChatEditingSession snapshot/baseline machinery, letting agent edits land as ordinary workspace edits with git as the review/rollback mechanism. The bulk of the work lives in `src/vs/workbench/contrib/chat/browser/chatEditing/` (chatEditingSession.ts, chatEditingServiceImpl.ts, chatEditingEditorActions.ts). The related issue https://github.com/microsoft/vscode/issues/304539 captures additional concurrency-safety concerns and could be addressed in the same effort.
