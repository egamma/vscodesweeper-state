# Issue #212167 — Debugger: Collapse multiline values in variable pane preview

[https://github.com/microsoft/vscode/issues/212167](https://github.com/microsoft/vscode/issues/212167)

## Summary

Still present on main: the variable pane's `.expression .value` uses `white-space: pre` (debugViewlet.css:281), so multiline DAP values only show the first line and lack an ellipsis when truncation happens at a newline. The related PR #212168 was closed without merging. The maintainer (roblourens) agreed with the improvement but asked for newline indicators — nobody has followed up since August 2024.

## Change summary

Multiline debug variable values in the Variables pane only show the first line due to `white-space: pre`, with no ellipsis indicating hidden content.

## Best solution

Replace `white-space: pre` with `white-space: nowrap` (or remove it) on `.debug-pane .monaco-list-row .expression .value` in `src/vs/workbench/contrib/debug/browser/media/debugViewlet.css`, and programmatically replace newline characters with a visible glyph (e.g. ⏎) in the rendered value text in the expression renderer, as roblourens requested. The CSS-only part is trivial; the newline-indicator part requires a small change in the rendering logic.
