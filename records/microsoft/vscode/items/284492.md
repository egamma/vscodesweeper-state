# Issue #284492 — Copilot search input should be natural language and not text search properties

[https://github.com/microsoft/vscode/issues/284492](https://github.com/microsoft/vscode/issues/284492)

## Summary

Team-authored feature request that remains unaddressed — the search widget still exposes text-search toggles (match case, whole word, regex) alongside the AI search input on current main. Keeping open per team-member rule.

## Change summary

When AI search is active, the search input should switch to a natural-language mode that hides irrelevant text-search toggles (match case, whole word, regex).

## Best solution

When the user activates AI/semantic search, transform the search widget into a natural-language input that hides the match-case, whole-word, and regex toggles — these controls are meaningless for an AI query. The change would primarily affect `src/vs/workbench/contrib/search/browser/searchWidget.ts` (toggle visibility) and `searchView.ts` (mode switching logic).
