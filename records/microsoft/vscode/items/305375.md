# Issue #305375 — User prompt triggers "allow reading external file"

[https://github.com/microsoft/vscode/issues/305375](https://github.com/microsoft/vscode/issues/305375)

## Summary

Team-authored bug: the LLM strips the `vscode-userdata:/` scheme from the prompt file URI, causing the external-file-read confirmation to trigger unnecessarily. Keeping open for maintainer judgment.

## Change summary

User-defined prompts trigger an unexpected 'allow reading external file' confirmation because the LLM shortens the full `vscode-userdata:/` URI to a plain file path.

## Best solution

The external path confirmation logic in `src/vs/workbench/contrib/chat/common/tools/builtinTools/chatExternalPathConfirmation.ts` (and the tool invocation path) should recognize paths that originated from attached prompt files with `vscode-userdata:` scheme and skip the confirmation, or the prompt attachment should be presented to the LLM in a way that prevents it from producing a bare filesystem path in the tool call.
