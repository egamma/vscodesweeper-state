# Issue #305375 — User prompt triggers "allow reading external file"

[https://github.com/microsoft/vscode/issues/305375](https://github.com/microsoft/vscode/issues/305375)

## Summary

Team-authored issue (MEMBER) — must keep open for maintainer judgment. The bug is that the LLM strips the `vscode-userdata:/` scheme from prompt file URIs, causing an unnecessary external-file-read permission dialog.

## Change summary

When a user-defined prompt file is referenced via `vscode-userdata:` URI, the LLM's `read_file` tool call uses a bare filesystem path instead, triggering an unwanted 'allow reading external file' consent dialog.

## Best solution

Ensure that prompt file attachments passed to the LLM context preserve the `vscode-userdata:` scheme so the tool resolver recognizes them as already-trusted internal resources, or teach the `read_file` tool handler to map known userdata paths back to their trusted URI without prompting.
