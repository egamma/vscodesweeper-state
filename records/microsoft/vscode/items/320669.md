# Issue #320669 — Tasks cannot run MSIX-packaged pwsh

[https://github.com/microsoft/vscode/issues/320669](https://github.com/microsoft/vscode/issues/320669)

## Summary

Legitimate bug: the task system passes the configured shell executable name directly to the terminal without resolving MSIX execution aliases. The terminal profile resolver has MSIX-aware discovery via `src/vs/base/node/powershell.ts`, but the task runner's shell resolution path in `terminalTaskSystem.ts` doesn't use it when a bare `pwsh` is specified. Routing to the tasks area.

## Change summary

Shell-type tasks fail with 'file not found' when `pwsh` is installed via the MSIX package (now the default winget install method as of PowerShell 7.6.0), because the task runner doesn't resolve MSIX execution aliases.

## Best solution

The task system's shell executable resolution in `terminalTaskSystem.ts` should fall back to the PowerShell discovery logic in `src/vs/base/node/powershell.ts` (specifically `findPSCoreMsix`) when a bare `pwsh` executable fails to resolve directly. This is closely related to https://github.com/microsoft/vscode/issues/181302 which tracks custom task shell executable resolution more broadly.

## Evidence

- **MSIX discovery exists:** src/vs/base/node/powershell.ts already contains `findPSCoreMsix()` which resolves the MSIX pwsh path via LOCALAPPDATA/Microsoft/WindowsApps — the terminal profile system uses this, but the task runner does not.
- **task shell resolution:** In src/vs/workbench/contrib/tasks/browser/terminalTaskSystem.ts line 1259, `shellLaunchConfig.executable` is set directly from the user-configured value after variable resolution, without MSIX-aware path resolution.
- **related open issue:** https://github.com/microsoft/vscode/issues/181302 ('Custom task shell executable doesn't work') is open in Backlog with labels bug+tasks — it covers the broader problem of task shell executable resolution failures.
- **pwsh 7.6 default change:** PowerShell 7.6.0+ installs via MSIX by default through winget, making this a growing problem as users upgrade.
