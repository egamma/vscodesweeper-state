# Issue #324611 — Enterprise-managed plugin (extraKnownMarketplaces/enabledPlugins) marked installed but plugin files never synced to cache_path

[https://github.com/microsoft/vscode/issues/324611](https://github.com/microsoft/vscode/issues/324611)

## Summary

The root cause — plugin files never being synced to disk — lives in the Copilot CLI (agent host), not in VS Code's workbench code. VS Code resolves enterprise plugin IDs to `~/.copilot/installed-plugins/<marketplace>/<plugin>/` and gracefully skips missing directories. A related issue was filed and closed in github/copilot-cli. However, there may be a valid VS Code enhancement: surfacing a visible error when an enterprise-managed plugin's expected directory is empty.

## Change summary

Enterprise-managed plugins declared via `extraKnownMarketplaces`/`enabledPlugins` are marked installed in config but their content is never downloaded to disk, so agents/skills never load.

## Best solution

The file-sync bug belongs to the Copilot CLI (agent host) — the closed issue at github/copilot-cli#4039 likely tracks or resolved this. A valid VS Code follow-up would be to surface a user-visible warning in the Copilot plugin UI when an enterprise-managed plugin's directory is missing or empty, rather than silently skipping it (src/vs/workbench/contrib/chat/common/plugins/agentPluginServiceImpl.ts around line 650).
