# Issue #250604 — Consider to accept/keep changes on subsequent requests

[https://github.com/microsoft/vscode/issues/250604](https://github.com/microsoft/vscode/issues/250604)

## Summary

Issue is on the 'On Deck' milestone — a real scheduling decision by a maintainer — and is actively under discussion with maintainer engagement (jrieken). Must remain open for maintainer judgment.

## Change summary

When rejecting/undoing a Copilot Edits suggestion, all manual edits made since the last 'Accept/Keep' are also reverted, not just the last suggestion's changes.

## Best solution

The core UX gap is that manual edits between Accept/Keep points are not preserved when undoing a subsequent Copilot edit. The team should decide whether to introduce intermediate save points (auto-accept on manual edit, or a separate undo granularity) so that rejecting one suggestion does not discard unrelated manual work. This is already being tracked in the cross-file-editing / chat-agent-editing space.
