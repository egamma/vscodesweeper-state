# Issue #250604 — Consider to accept/keep changes on subsequent requests

[https://github.com/microsoft/vscode/issues/250604](https://github.com/microsoft/vscode/issues/250604)

## Summary

Keep open: this is already under maintainer discussion in the Copilot/agent editing area and has an assigned `On Deck` milestone, which is a hard keep-open condition. The remaining question is product behavior around when edit sessions create keep/undo checkpoints, especially after manual edits between Copilot requests.

## Change summary

Copilot Edits reject/undo can revert manual changes made since the last Keep checkpoint instead of only reverting the latest suggested changes.

## Best solution

Let the chat-agent-editing owners decide the checkpoint model for subsequent Copilot Edits requests; the likely end state is either an automatic checkpoint before a new request or clearer Keep/Undo semantics that preserve intervening manual edits.
