# Issue #316465 — Agent window: when starting a new session via worktree+branch, the remote branch is used, not the local branch

[https://github.com/microsoft/vscode/issues/316465](https://github.com/microsoft/vscode/issues/316465)

## Summary

The worktree start-point preference for origin/<branch> over the local branch is intentional (see code comment), but the user's request for local-first behavior in local workflows is a valid improvement. Routing to the agent host area for consideration.

## Change summary

When starting a worktree-based agent session targeting a local branch, the worktree uses origin/<branch> instead of the local branch, losing unpushed local commits.

## Best solution

Consider adding an option or heuristic in `_resolveRemoteTrackingBranch` / `addWorktree` (src/vs/platform/agentHost/node/agentHostGitService.ts) to prefer the local branch when it is ahead of the remote-tracking ref, or when the session is configured for local isolation. The current design intentionally prefers origin/ for freshness, but a local-first mode would serve users who merge locally before pushing.
