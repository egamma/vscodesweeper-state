# Issue #287054 — Issue with NODE_OPTIONS in the devconatiner when Debug Auto Attach is enabled

[https://github.com/microsoft/vscode/issues/287054](https://github.com/microsoft/vscode/issues/287054)

## Summary

The NODE_OPTIONS duplication and missing-space concatenation is caused by the js-debug extension's bootloader (ms-vscode.js-debug), which manages the --require injection for auto-attach. This belongs in the vscode-js-debug repository, not VS Code core.

## Change summary

When debug auto-attach is enabled in a devcontainer with a custom NODE_OPTIONS value, the js-debug bootloader duplicates and incorrectly concatenates the existing NODE_OPTIONS entries.

## Best solution

Report to https://github.com/microsoft/vscode-js-debug — the bootloader's NODE_OPTIONS string concatenation logic needs to properly handle pre-existing values with correct spacing and without duplication.
