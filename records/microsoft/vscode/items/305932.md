# Issue #305932 — Forever loop in chat, session list shows Failed.

[https://github.com/microsoft/vscode/issues/305932](https://github.com/microsoft/vscode/issues/305932)

## Summary

The looping behavior and 'Failed' status originate in the Copilot Chat extension's agent tool (`get_errors`), which mistakenly tries to open a directory URI as a document and likely retries on failure. VS Code core correctly rejects the invalid open; the retry/loop logic is extension-side. The reporter should file this against the Copilot Chat extension.

## Change summary

Copilot agent enters an infinite processing loop and shows 'Failed' when its get_errors tool tries to open a directory path as a file.

File this against the Copilot Chat extension (https://github.com/microsoft/vscode-copilot-release) — the agent's `get_errors` tool needs to guard against directory URIs or gracefully handle the rejection from `$tryOpenDocument` instead of looping.
