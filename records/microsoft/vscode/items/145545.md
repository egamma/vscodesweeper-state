# Issue #145545 — Consider incomplete vs complete results from inline completion provider

[https://github.com/microsoft/vscode/issues/145545](https://github.com/microsoft/vscode/issues/145545)

## Summary

Unimplemented feature request: the InlineCompletions interface still lacks an isIncomplete/complete concept. The issue is from a contributor (jrieken) and sits in Backlog with no upvotes or discussion — keeping open for future consideration.

## Change summary

Add an isIncomplete flag to InlineCompletionList so the editor knows whether to re-query the provider on further typing.

## Best solution

Add an optional `isIncomplete` property to the `InlineCompletions` interface in `src/vs/editor/common/languages.ts` and surface it in the public API's `InlineCompletionList` class, mirroring how `CompletionList.isIncomplete` works for suggest completions. This would let providers signal that the list is partial and should be re-fetched on further input.
