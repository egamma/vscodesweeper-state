# Issue #242393 — Sidebar does not empty when no views are present

[https://github.com/microsoft/vscode/issues/242393](https://github.com/microsoft/vscode/issues/242393)

## Summary

Still reproduces on main: when the active view container has all its views removed via a 'when' clause, it remains visible but empty because `shouldBeHidden` in `paneCompositeBar.ts` explicitly returns `false` for active containers even when they have no views. An open PR (#266219) addresses this.

## Change summary

Sidebar shows an empty panel instead of hiding or switching away when the active view container's only view is removed via a 'when' clause.

## Best solution

Land or iterate on the open PR https://github.com/microsoft/vscode/pull/266219 which adds show/hide logic based on visibility toggle for view containers — it explicitly lists this issue as a fix target.
