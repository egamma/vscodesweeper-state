# Issue #297774 — `chat.implicitContext.enabled` sounds like it would change context, but only changes visibility of toolbar?

[https://github.com/microsoft/vscode/issues/297774](https://github.com/microsoft/vscode/issues/297774)

## Summary

Team-authored issue reporting that the `chat.implicitContext.enabled` setting name and description are misleading — the setting controls both toolbar visibility and context attachment, but the UX makes it look like it only toggles UI. The `first` mode is particularly confusing. Keeping open for maintainer judgment.

## Change summary

The `chat.implicitContext.enabled` setting name suggests it changes what context is sent, but the observed behavior makes it appear to only toggle toolbar visibility; the `first` mode has unexpected disappearing-attachment behavior.

## Best solution

Clarify the setting's name and/or description to better reflect its actual behavior (toolbar visibility + context attachment), and review the `first` mode's UX where the attachment disappears after navigating to a different editor. The relevant code is in `src/vs/workbench/contrib/chat/browser/attachments/chatImplicitContext.ts` (the `updateImplicitContext` method, lines 282–293) and the setting registration in `src/vs/workbench/contrib/chat/browser/chat.shared.contribution.ts`.
