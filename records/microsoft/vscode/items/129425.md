# Issue #129425 — Chosen user-settings debug launch configuration not remembered

[https://github.com/microsoft/vscode/issues/129425](https://github.com/microsoft/vscode/issues/129425)

## Summary

Still reproducible on main: when a launch configuration change fires, `selectConfiguration(undefined)` re-resolves the launch to a folder-based launch via `historyService.getLastActiveWorkspaceRoot()`, and the user-settings config name isn't found in that folder launch's `getConfigurationNames()`, causing the selection to reset to the first folder config.

## Change summary

Selecting a debug launch configuration from user settings is not persisted across reload or configuration changes — it resets to the first folder-level config.

## Best solution

In `debugConfigurationManager.ts`, the `onDidChangeConfiguration` handler (line ~348) and the `selectConfiguration(undefined)` path should check whether the currently-selected configuration (stored in `this.selectedLaunch`/`this.selectedName`) still exists across all launches (including `UserLaunch`) before falling back to the first folder config. The fix is localized to the `selectConfiguration` method's fallback logic around lines 418–481.
