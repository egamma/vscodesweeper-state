# Issue #237095 — Support Line decoration editor offset retrieval

[https://github.com/microsoft/vscode/issues/237095](https://github.com/microsoft/vscode/issues/237095)

## Summary

Valid feature request for an editor API to retrieve horizontal offset of inline decorations. Not yet implemented on main; has 10 upvotes and is in the Backlog milestone. Keeping open for community process.

## Change summary

Add an API to retrieve the horizontal pixel offset of inline text decorations, so overlay widgets can position without overlapping them.

## Best solution

Expose a method on the editor view (or visible-line abstraction) that returns the right-edge pixel offset of the last inline decoration at a given column/className. This would live in the editor browser layer alongside the existing `getOffsetForColumn` family. Worth considering as part of broader overlay-widget positioning improvements.
