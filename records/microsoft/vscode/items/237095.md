# Issue #237095 — Support Line decoration editor offset retrieval

[https://github.com/microsoft/vscode/issues/237095](https://github.com/microsoft/vscode/issues/237095)

## Summary

Keep open: this is a Backlog feature request for an editor rendering API gap, and current main still exposes column offsets but not decoration offset/bounds retrieval. There is no linked duplicate or concrete fixing commit in the issue context.

## Change summary

Expose a way for editor contributions to retrieve the horizontal rendered offset of inline/text decorations so overlay widgets can avoid covering them.

## Best solution

If accepted, add an editor/view-lines API that returns rendered horizontal bounds for matching inline or injected-text decorations on a visible line, likely near the existing `getOffsetForColumn`/visible-range plumbing in `src/vs/editor/browser/view.ts` and `src/vs/editor/browser/viewParts/viewLines/viewLines.ts`.
