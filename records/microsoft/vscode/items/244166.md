# Issue #244166 — MCP: support server sessions connections per-edit session

[https://github.com/microsoft/vscode/issues/244166](https://github.com/microsoft/vscode/issues/244166)

## Summary

Team-authored feature request on the 'On Deck' milestone — hard keep-open. Partial infrastructure exists (chatSessionResource threading through the MCP gateway layer), but the full per-edit-session connection model is not yet complete on main.

## Change summary

MCP servers should support separate connections scoped to each edit/chat session so that stateful servers (e.g. execution sandboxes) get fresh instances per session.

## Best solution

Complete the per-session MCP connection lifecycle in the gateway service so that each chat/edit session creates and tears down its own MCP server connection. The existing `chatSessionResource` plumbing in `src/vs/workbench/contrib/mcp/` is the foundation; the remaining work is full lifecycle management and protocol-level session isolation.
