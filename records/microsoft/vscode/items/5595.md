# Issue #5595 — Allow to scope settings by platform

[https://github.com/microsoft/vscode/issues/5595](https://github.com/microsoft/vscode/issues/5595)

## Summary

This is the canonical tracking issue for platform-scoped settings (1146 upvotes, well above the 20-upvote Backlog threshold). The feature remains unimplemented on current main — no platform-override mechanism exists in the configuration system beyond machine-specific scopes. Multiple duplicates have been closed against this issue.

## Change summary

Allow users to scope settings (font size, paths, zoom level, etc.) by OS platform (Windows, macOS, Linux) so a single synced settings file works correctly across platforms.

## Best solution

Implement platform-specific overrides in the configuration model (similar to language-specific `[typescript]` overrides but for `[linux]`/`[windows]`/`[osx]`). The work would touch `src/vs/platform/configuration/common/configurationRegistry.ts` (add a platform-override scope), the configuration resolution pipeline, and the Settings Sync service to properly merge per-platform blocks.
