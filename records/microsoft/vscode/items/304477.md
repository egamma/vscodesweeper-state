# Issue #304477 — search.exclude does not work for "Go to Symbol in Workspace"

[https://github.com/microsoft/vscode/issues/304477](https://github.com/microsoft/vscode/issues/304477)

## Summary

The reporter asks for search.exclude to filter results in 'Go to Symbol in Workspace' (#). This filtering was added in commit 89906c073a8 (March 2020) but intentionally removed two weeks later in commit b0f248f60a6 ('symbols - remove exclude support for workspace symbols quick access', April 2020). Current main has no exclude filtering for workspace symbols. This is a feature request to re-add that capability.

## Change summary

Request to re-add search.exclude / files.exclude filtering for the 'Go to Symbol in Workspace' quick access picker.

## Best solution

Re-introduce resource-exclude filtering in SymbolsQuickAccessProvider (src/vs/workbench/contrib/search/browser/symbolsQuickAccess.ts) using the search.exclude and files.exclude glob matchers, as was done in commit 89906c073a8 before its removal. The removal may have been for performance reasons — investigate whether the exclude matcher can be applied efficiently against symbol URIs without regressing the picker's responsiveness.
