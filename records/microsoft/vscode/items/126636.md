# Issue #126636 — Workspace Trust - Restricted mode causes test failures

[https://github.com/microsoft/vscode/issues/126636](https://github.com/microsoft/vscode/issues/126636)

## Summary

Team-authored issue (MEMBER) — must keep open. The core ask is that extension integration tests fail in restricted mode with no easy programmatic workaround beyond `--disable-workspace-trust`. VS Code's own test harnesses already pass that flag, but extension authors need either better documentation or an API to grant trust to a specific folder during test runs.

## Change summary

Extension integration tests fail when the tested extension is disabled by Workspace Trust restricted mode, and there is no clean API to trust a folder programmatically during test execution.

## Best solution

Document `--disable-workspace-trust` as the recommended approach for extension test runs in the official testing docs, and consider exposing an API (or test-harness option in `@vscode/test-electron`) to grant trust to a specific folder at launch — the latter would let tests exercise trust-dependent code paths rather than bypassing trust entirely.
