# Issue #291741 — High CPU usage

[https://github.com/microsoft/vscode/issues/291741](https://github.com/microsoft/vscode/issues/291741)

## Summary

Confirmed upstream regression in Electron/Chromium GPU rendering on Linux (X11, NVIDIA). The issue appeared when VS Code moved from Electron 37 (1.106) to Electron 39 (1.109); the zygote/GPU compositor process spins CPU indefinitely. Two independent reporters confirm; `--disable-gpu` works around it. No VS Code-side fix exists — the root cause is in Chromium's GPU process. Should be routed to the upstream/GPU area and tracked against an Electron or Chromium bug.

## Change summary

The GPU compositor zygote process consumes 400–700% CPU on Linux (X11/XFCE, NVIDIA) after upgrading to VS Code 1.109+, making the system nearly unusable.

## Best solution

File or link an upstream Chromium/Electron bug for the GPU process CPU spin on Linux X11 with NVIDIA drivers. In the meantime, document `--disable-gpu` as the workaround. The fix must come from the Electron/Chromium GPU compositor layer, not from VS Code code.
