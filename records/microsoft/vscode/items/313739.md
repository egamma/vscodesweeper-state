# Issue #313739 — Recent files in `@` autocomplete are not workspace specific

[https://github.com/microsoft/vscode/issues/313739](https://github.com/microsoft/vscode/issues/313739)

## Summary

Team-authored issue (MEMBER); must keep open. The reported behavior — recent files in the `@` autocomplete not being scoped to the current workspace — is a legitimate bug in the Agents window file-picker UX.

## Change summary

The `@` autocomplete in the Agents window shows recent files from all workspaces instead of filtering to the current workspace context.

## Best solution

Scope the recent-files data source for the `@` autocomplete to the active workspace folder(s), likely in the chat attachment or file-picker contribution under `src/vs/sessions/` or `src/vs/workbench/contrib/chat/`. The history service backing recent files needs a workspace filter applied before surfacing results.
