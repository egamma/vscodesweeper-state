# Issue #189156 — [scss] Cannot open `.scss` files using `@use` import syntax

[https://github.com/microsoft/vscode/issues/189156](https://github.com/microsoft/vscode/issues/189156)

## Summary

This is a real, acknowledged bug in SCSS document link resolution for relative paths across directories. It's already triaged with `help wanted` and in the Backlog — no action needed beyond keeping it open.

## Change summary

Ctrl+click / Go to Definition on SCSS `@use` and `@import` paths fails with 'Unable to open' when referencing partials (underscore-prefixed files) via relative paths across directories.

## Best solution

The fix likely belongs in `vscode-css-languageservice` (upstream at https://github.com/microsoft/vscode-css-languageservice) in `SCSSNavigation.mapReference`/`toPathVariations` or in how `documentContext.resolveReference` interacts with case-sensitive path segments. Once fixed upstream, bump the dependency in `extensions/css-language-features/server/package.json`.
