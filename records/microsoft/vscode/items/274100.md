# Issue #274100 — Git - Partial stage should auto-accept agentic changes

[https://github.com/microsoft/vscode/issues/274100](https://github.com/microsoft/vscode/issues/274100)

## Summary

Valid feature request still unimplemented on main. The SCM integration with chat editing exists (quickDiffModel.ts uses IChatEditingService) but partial/hunk-level staging does not correlate with accepting the corresponding agentic edits. Keeping open as a Backlog item with low upvote count.

## Change summary

When a user partially stages agentic changes (hunk by hunk in the SCM diff view), the staged hunks should be auto-accepted as reviewed AI edits, matching the existing behavior where a full commit auto-accepts all agentic changes.

## Best solution

Implement hunk-level correlation between SCM staging and chat-editing acceptance in the SCM/chat-editing integration layer — likely in `src/vs/workbench/contrib/scm/browser/quickDiffModel.ts` and the chat editing session model. When a hunk is staged via the SCM diff, the corresponding agentic edit range should be marked as accepted. alexdima and lszomoru are already engaged in the discussion.
