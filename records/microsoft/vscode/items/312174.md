# Issue #312174 — Launch configuration with non-trivial env file hangs

[https://github.com/microsoft/vscode/issues/312174](https://github.com/microsoft/vscode/issues/312174)

## Summary

Known class of bug: `prepareCommand` in `terminals.ts` inlines all env vars into a single shell command string (e.g. `/usr/bin/env KEY=VALUE … command`). With a large env file, this exceeds terminal input buffer limits and causes a hang. A prior fix attempt (passing env via the terminal API, commit 6b6da7928db) was reverted (d748e6abca7) because it broke nvm (#209693). The predecessor issue https://github.com/microsoft/vscode/issues/198550 was closed but the underlying problem remains unresolved. Routing to `debug` for renewed attention.

## Change summary

Debug launch configurations with large `.env` files hang because the environment variables are serialized into a single overlong shell command.

## Best solution

Revisit the approach from the reverted commit 6b6da7928db — passing env vars via the terminal creation API instead of the command string — while handling the nvm edge case that prompted the revert. The fix lives in `src/vs/workbench/contrib/debug/node/terminals.ts` (`prepareCommand`) and `src/vs/workbench/api/node/extHostDebugService.ts` (`$runInTerminal`). Since the previous maintainer noted no known solution, this may need a fresh design pass.
