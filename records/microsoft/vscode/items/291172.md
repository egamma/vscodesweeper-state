# Issue #291172 — Testing: Subagent list items have visually clipped focus outline

[https://github.com/microsoft/vscode/issues/291172](https://github.com/microsoft/vscode/issues/291172)

## Summary

Team-authored visual bug with milestone 'On Deck' — hard keep-open. The clipped focus outline on chat list items remains unfixed on current main; the root cause is likely `overflow: hidden` on a parent container clipping the `outline-offset` of focused items.

## Change summary

Focus outlines on subagent list items (files, fetched links) in the chat widget appear visually clipped at the corners.

## Best solution

Fix the `overflow: hidden` on the containing element (likely `.chat-used-context-list` or a parent in `chat.css`) so it doesn't clip focus outlines. Common approaches: add `overflow: clip` with `overflow-clip-margin` for the outline width, use an inset `outline-offset`, or switch to a box-shadow-based focus indicator that isn't clipped by overflow.
