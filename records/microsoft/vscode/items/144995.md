# Issue #144995 — The value of "err" is out of range. It must be a negative integer. Received 1

[https://github.com/microsoft/vscode/issues/144995](https://github.com/microsoft/vscode/issues/144995)

## Summary

This is an upstream Node.js/libuv issue where `child_process.spawn` receives an unexpected positive error code (1) instead of a negative errno. As confirmed by a team member, the root cause is in libuv's error code handling. VS Code has since upgraded from Electron 17 (Node 16) to Electron 42 — the bug may be resolved upstream, but without platform/version data from affected users, it's unclear whether it still reproduces.

## Change summary

Spawning a git child process intermittently fails with a RangeError because libuv returns a positive error code (1) instead of a negative errno.

## Best solution

Verify whether this still reproduces on the current Electron 42 / Node.js version. If it does, file or link an upstream libuv/Node.js issue. If not, close as resolved by the Electron upgrade. Meanwhile, a defensive try/catch around `cp.spawn` in `extensions/git/src/git.ts` (line ~702) could prevent the unhandled error from surfacing to users.
