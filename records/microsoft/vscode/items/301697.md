# Issue #301697 — GitHub Copilot - Agent Yaml - Handoff Feature Requests

[https://github.com/microsoft/vscode/issues/301697](https://github.com/microsoft/vscode/issues/301697)

## Summary

Two reasonable feature requests for the agent handoffs system: file-based handoff definitions (for reusability and cleaner YAML) and script-based conditional handoffs (dynamic show/hide). The handoffs infrastructure exists in `promptFileParser.ts` and `chatModes.ts` but neither capability is currently supported. Keeping open for the area owner to evaluate.

## Change summary

Allow handoffs in agent prompt files to reference external prompt files instead of inline strings, and support script-based conditional handoffs that dynamically compute availability and content.

## Best solution

The file-reference request fits naturally into the existing prompt-file resolution infrastructure in `src/vs/workbench/contrib/chat/common/promptSyntax/` — a `path` key on a handoff entry could resolve through the same `IPromptsService` that already handles prompt file references. The script-based request is more complex and overlaps with the existing hooks system; the area owner should decide whether to extend hooks to cover conditional handoffs or add a separate mechanism. Route to the chat/copilot area for prioritization.
