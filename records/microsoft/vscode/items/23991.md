# Issue #23991 — Keyboard mappings with `setxkbmap` on Linux not working

[https://github.com/microsoft/vscode/issues/23991](https://github.com/microsoft/vscode/issues/23991)

## Summary

Keep open: this is a long-standing Linux keybinding bug already accepted into Backlog with maintainer analysis and a known but incomplete workaround. The discussion points to native-keymap/X11 scan-code mapping work rather than a duplicate, completed fix, or support question.

## Change summary

Linux `setxkbmap` remappings such as Caps Lock to Escape are not consistently respected by VS Code's default scan-code keyboard dispatch.

## Best solution

Fix Linux scan-code dispatch by teaching `node-native-keymap` to expose X11/XKB remappings from `setxkbmap`, then consume that data in VS Code's `macLinuxKeyboardMapper` so remapped keys work without requiring `"keyboard.dispatch": "keyCode"`.
