# Issue #23991 — Keyboard mappings with `setxkbmap` on Linux not working

[https://github.com/microsoft/vscode/issues/23991](https://github.com/microsoft/vscode/issues/23991)

## Summary

Still valid on current main: the default scan-code dispatch (`keyboard.dispatch: "code"`) fundamentally cannot see XKB-layer remappings like `caps:escape`. The workaround (`keyCode` dispatch) remains, but the underlying architectural gap in `node-native-keymap` is unresolved. With 286 upvotes and `help wanted`, this stays open.

## Change summary

VS Code's default scan-code-based keyboard dispatch ignores `setxkbmap` remappings (e.g. caps→escape) on Linux, breaking key bindings for users who customize their layout via XKB options.

## Best solution

Enhance `native-keymap` (now `@vscode/native-keymap`) to query XKB option remappings via X11/libxkbcommon APIs and expose them in `_GetKeyMap`, then consume those mappings in `macLinuxKeyboardMapper.ts` so the default `code` dispatch respects XKB option-level key swaps without requiring the `keyCode` fallback.
