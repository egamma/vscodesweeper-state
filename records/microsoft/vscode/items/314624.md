# Issue #314624 — Standardize the pull request template

[https://github.com/microsoft/vscode/issues/314624](https://github.com/microsoft/vscode/issues/314624)

## Summary

VS Code already has a PR template at `.github/pull_request_template.md` (created in commit 1cd8dce12c5 and refined over time). The request to 'add' one is based on the mistaken premise that none exists. The remaining ask — expanding it with more mandatory sections — is a process/governance change that the maintainers have intentionally kept lightweight, and is better characterized as a polish request on the existing template rather than a missing feature.

## Change summary

Standardize the pull request template by adding mandatory sections for description, motivation, related issue, testing, and a coding-guidelines checklist.

## Best solution

The existing PR template at `.github/pull_request_template.md` already covers the key points (associate an issue, describe changes, explain how to test). The team has intentionally kept it lightweight. If maintainers want to expand it, they can iterate on the existing file — no new issue is needed. This can be closed as the premise ('there is no equivalent formalized template for pull requests') is factually incorrect.
