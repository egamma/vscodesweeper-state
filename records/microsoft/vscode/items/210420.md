## Summary

Valid feature request with 26 upvotes, promoted to Backlog. Asks for expanding files in the Explorer to show their outline inline (like Visual Studio's Solution Explorer), enabling multi-file symbol browsing. No duplicate found — the existing Outline view only shows the active file. Keeping open as an accepted backlog item.

## Change summary

Allow expanding source files in the Explorer tree to reveal their outline (symbols), so users can browse classes and methods across multiple files without switching the active editor.

## Best solution

Implement inline outline expansion in the Explorer tree view, likely touching `src/vs/workbench/contrib/files/browser/` (Explorer) and integrating with the document symbol provider used by the Outline view (`src/vs/workbench/contrib/codeEditor/browser/outline/`). This is a meaningful UX enhancement over the current single-file Outline view and fits as a core feature rather than an extension.
