# Issue #57488 — Git - Linux ssh-agent/ssh-askpass problem

[https://github.com/microsoft/vscode/issues/57488](https://github.com/microsoft/vscode/issues/57488)

## Summary

The original root cause — VS Code not inheriting SSH agent environment variables when launched from non-shell environments (i3+rofi, dmenu, etc.) — has been substantially addressed by two later improvements: (1) VS Code's own SSH_ASKPASS handler (commit a6272c94161, Sep 2022) which sets SSH_ASKPASS and SSH_ASKPASS_REQUIRE in the git extension, and (2) the shell environment resolution in src/vs/platform/shell/node/shellEnv.ts which spawns the user's shell to pick up environment variables like SSH_AUTH_SOCK when VS Code is not launched from a CLI. The last community confirmation of the problem was January 2023; given the age and the improvements shipped since, confirmation on a current build is needed.

## Change summary

Git SSH operations fail when VS Code is launched from a non-shell environment (window manager launcher) because SSH_AUTH_SOCK is not inherited from the user's shell profile.

## Best solution

Ask whether this still reproduces on a recent VS Code build. If it does, the remaining gap is likely in shell environment resolution timing or specific WM/launcher combinations that prevent the shell env spawning from picking up SSH_AUTH_SOCK. If it no longer reproduces, close as completed. Issue https://github.com/microsoft/vscode/issues/242494 tracks a related but distinct kwallet/ksshaskpass integration problem.
