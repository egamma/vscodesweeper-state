# Issue #239898 — [Bug] `filesExplorer.paste` is not working when pasting from finder to vscode file explorer

[https://github.com/microsoft/vscode/issues/239898](https://github.com/microsoft/vscode/issues/239898)

## Summary

This is a platform limitation, not a VS Code bug. Pasting files from Finder requires the native browser/Electron paste event (triggered by Cmd+V), which provides the clipboard's FileList. A custom keybinding like `p` doesn't trigger a native paste event, so the OS clipboard file data is unavailable — the handler correctly falls back to VS Code's internal clipboard, which has no Finder files.

## Change summary

Custom keybinding bound to `filesExplorer.paste` doesn't paste files copied from Finder, while the default Cmd+V works.

## Best solution

Close as designed. The paste-from-Finder flow relies on the native DOM `paste` event to access `event.clipboardData.files`, which only fires on the platform's paste shortcut (Cmd+V). A custom keybinding invokes the command directly without a native event, so there is no mechanism to read the OS clipboard's file references. This is a fundamental browser/Electron platform constraint, not something VS Code can work around for arbitrary keybindings.

  > When you press `Cmd+V`, the browser/Electron fires a native `paste` event whose `clipboardData.files` contains the files you copied in Finder.   > 
  > This is a fundamental constraint of how clipboard access works in web/Electron — file data from the OS clipboard is only available during a native paste event. There's no workaround for arbitrary keybindings.
  > 
  > Closing as by design, but please reopen if you believe there's an approach I've missed.

## Evidence

- **keybinding registration:** In src/vs/workbench/contrib/files/browser/fileActions.contribution.ts:121-126, the keybinding uses `^filesExplorer.paste` (the `^` prefix prevents default bubble up) with Cmd+V, which intercepts the native paste event.
- **DOM paste listener:** In src/vs/workbench/contrib/files/browser/views/explorerView.ts:337-344, a DOM `paste` event listener reads `event.clipboardData?.files` and passes it to the command. This listener only fires on actual native paste events (Cmd+V), not on custom keybindings.
- **handler fallback:** In src/vs/workbench/contrib/files/browser/fileActions.ts:1123, `pasteFileHandler` receives an optional `fileList`. When undefined (custom keybinding), `getFilesToPaste` at line 1306-1320 falls back to `clipboardService.readResources()`, which reads VS Code's internal clipboard — not OS-level file references from Finder.
- **platform constraint:** The Web Clipboard API and Electron only expose file data from the OS clipboard during native paste events. There is no async API to read file paths from the OS clipboard outside a paste event handler.
