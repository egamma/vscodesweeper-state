# Issue #312110 — PendingMigrationError: navigator is now a global in nodejs thrown during module load on remote VS Code server,   causing repeated renderer crashes and Remote SSH disconnections

[https://github.com/microsoft/vscode/issues/312110](https://github.com/microsoft/vscode/issues/312110)

## Summary

Real, ongoing bug in the bundled Copilot extension: top-level navigator access triggers VS Code's PendingMigrationError during module load on Node.js v22, crashing the remote extension host. Multiple users confirm it persists through VS Code 1.129.1. Routing to the copilot area for a fix in the extension's module-level code.

## Change summary

The bundled Copilot extension accesses `navigator` at module load time, triggering a PendingMigrationError on Node.js v22 that crashes the remote extension host and terminates Remote SSH/WSL sessions.

## Best solution

The Copilot extension's bundled code (likely a Zod schema or similar library) must guard all top-level `navigator` accesses so they don't trigger VS Code's intentional PendingMigrationError getter trap during `require()`. The access in `extensions/copilot/src/util/vs/base/common/platform.ts` uses `typeof navigator` which still triggers the getter — consider checking `typeof globalThis.navigator` or restructuring to avoid the trap. Additionally, VS Code core could consider making the PendingMigrationError non-throwing during module load to prevent cascading crashes.
