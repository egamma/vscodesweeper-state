# Issue #246714 — Search Editor Progress Bar disappears when switching to another editor and back

[https://github.com/microsoft/vscode/issues/246714](https://github.com/microsoft/vscode/issues/246714)

## Summary

Confirmed bug on current main: EditorProgressIndicator unconditionally stops and hides the progress bar on active-editor change without tracking state for replay, so switching away from a search editor mid-search permanently hides its progress bar. The info-needed label should be removed.

## Change summary

The search editor's progress bar disappears when the user switches to another tab and back during a long-running search, giving no indication the search is still in progress.

## Best solution

EditorProgressIndicator in src/vs/workbench/services/progress/browser/progressIndicator.ts needs to track progress state (like ScopedProgressIndicator already does) and replay it when the editor becomes active again. The existing comment on line 26–29 acknowledges this gap: 'we do not track active editor progress and replay it later (yet)'. Adding state tracking and replay on re-activation would fix this for search editors and any other editor that uses LongRunningOperation.
