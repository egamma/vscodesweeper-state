# Issue #246714 — Search Editor Progress Bar disappears when switching to another editor and back

[https://github.com/microsoft/vscode/issues/246714](https://github.com/microsoft/vscode/issues/246714)

## Summary

Confirmed on current main: EditorProgressIndicator explicitly stops and hides the progress bar on any active-editor change, and never restores it when switching back. The code even acknowledges this limitation with a comment. The bug is real and unfixed.

## Change summary

The Search Editor's progress bar disappears when switching to another editor tab and back, leaving the user unable to tell whether a search is still running.

## Best solution

EditorProgressIndicator (src/vs/workbench/services/progress/browser/progressIndicator.ts) needs to track active editor progress and replay it when the editor regains focus — the existing comment on line 28–29 already notes this gap ('we do not track active editor progress and replay it later (yet)'). The fix should save the running-progress state per editor and re-show the progress bar when the editor becomes active again.
