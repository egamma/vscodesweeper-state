# Issue #232482 — Search does not find pattern in closed files

[https://github.com/microsoft/vscode/issues/232482](https://github.com/microsoft/vscode/issues/232482)

## Summary

The reported behavior — search skipping files with embedded binary/null-byte content — is the expected default behavior of ripgrep, which VS Code uses for text search. Ripgrep silently skips files it detects as binary (containing null bytes). The reporter confirmed in comment 5 that the root cause is files with mixed text and binary encoding. This is as-designed; a feature request to add an option to search binary files (#316571) was filed separately and closed through the community process.

## Change summary

VS Code search silently skips files containing binary/null-byte content (e.g. log files with embedded binary data), returning no results for text patterns that exist in the text portions of those files.

## Best solution

Close as as-designed — ripgrep intentionally skips binary files to avoid garbled output. The reporter's workaround is to open the file in the editor and use the in-editor find (Ctrl+F). A separate feature request to add a 'search binary files' option (#316571) was filed and closed via the community upvote process; if demand grows, a new request could be filed.

- **ripgrep default behavior:** ripgrep skips binary files (those containing null bytes) by default. VS Code's ripgrepTextSearchEngine.ts (getRgArgs function, line 406+) does not pass --binary flag to ripgrep, so binary files are silently excluded from search.
- **no --binary flag in args:** src/vs/workbench/services/search/node/ripgrepTextSearchEngine.ts constructs ripgrep arguments without any --binary flag; this is an intentional omission — searching binary content could produce garbled results.
