# Issue #232482 — Search does not find pattern in closed files

[https://github.com/microsoft/vscode/issues/232482](https://github.com/microsoft/vscode/issues/232482)

## Summary

Still a valid issue on main: VS Code's ripgrep-based text search skips files containing binary content (null bytes), which means log files or other text files with embedded binary data return no results. No fix or workaround has been added. The related issue #233679 was closed in favor of this one.

## Change summary

Search fails to find text patterns in files that contain mixed text and binary content (e.g. log files with embedded binary data).

## Best solution

Consider passing ripgrep's `--text` flag (or a more nuanced binary-detection override) for the text search engine in `src/vs/workbench/services/search/node/ripgrepTextSearchEngine.ts`, or expose a user-facing setting to control binary file handling. The trade-off is that `--text` can produce garbled output for truly binary files, so a scoped approach (e.g. per-search toggle or a file-size/extension heuristic) would be safer.
