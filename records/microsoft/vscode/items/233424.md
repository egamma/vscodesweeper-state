# Issue #233424 — Dialog "Copilot wants to sign-in" should not show

[https://github.com/microsoft/vscode/issues/233424](https://github.com/microsoft/vscode/issues/233424)

## Summary

Active design discussion between the issue author and the authentication team about suppressing the sign-in confirmation modal when the user explicitly triggered the flow. No resolution yet; the team is exploring API-level mechanisms to detect recent user interaction.

## Change summary

Skip the 'Copilot wants to sign-in' confirmation dialog when the user explicitly clicked a Sign In action moments before.

## Best solution

Implement a transparent mechanism in the authentication API stack that tracks whether the user interacted with the focused window moments before `getSession` was called, allowing the modal to be suppressed for user-initiated flows. This requires API enrichment in `src/vs/workbench/api/` (authentication area) rather than extension-side opt-in, as discussed in the comments.
