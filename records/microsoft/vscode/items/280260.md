# Issue #280260 — Copilot Agents Don't Install Dependencies via Proper Commands

[https://github.com/microsoft/vscode/issues/280260](https://github.com/microsoft/vscode/issues/280260)

## Summary

Valid feature request for the Copilot agent to prefer terminal-based dependency installation commands (npm install, cargo add, pip install) over manually editing manifest files. The current agent instructions in defaultAgentInstructions.tsx give weak, inconsistent guidance on this — mentioning 'npm install' only when the terminal tool is available but falling back to 'creating a requirements.txt' otherwise. A related issue was closed in microsoft/vscode-copilot (#13553), but the underlying prompt guidance on current main still appears incomplete. Keeping open as a legitimate improvement to agent behavior.

## Change summary

Request that the Copilot agent use proper CLI commands (npm install, cargo add, pip install, etc.) to install dependencies rather than manually editing manifest files like package.json or requirements.txt.

## Best solution

Strengthen the agent instructions in `extensions/copilot/src/extension/prompts/node/agent/defaultAgentInstructions.tsx` (around the `GenericEditingTips` component, line ~503) to explicitly prefer running install commands via the terminal tool when it is available, and to avoid hand-editing dependency manifest files unless explicitly asked. The related vscode-copilot issue #13553 was closed, so any remaining gap lives in the prompt wording here.
