# Issue #140647 — Activity bar activity indicator is `1px` but configured as `2px`

[https://github.com/microsoft/vscode/issues/140647](https://github.com/microsoft/vscode/issues/140647)

## Summary

Still reproduces on main: the activity bar indicator is CSS 2px but renders visually as 1px due to sub-pixel alignment with the window frame. The initial fix was reverted and the underlying issue persists, confirmed by a maintainer in Dec 2024.

## Change summary

The activity bar's active indicator renders as 1px despite being configured as 2px in CSS, due to sub-pixel rendering interaction with the window frame offset.

## Best solution

Fix the sub-pixel alignment of the active indicator border — likely needs the indicator element positioned to avoid landing on a half-pixel boundary caused by the frameless window's 1px offset. The fix lives in `src/vs/workbench/browser/parts/activitybar/media/activityaction.css` (line ~87, `border-left: 2px solid`) and possibly the layout code that positions the activity bar container relative to the window frame.
