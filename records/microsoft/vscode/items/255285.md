# Issue #255285 — Terminal viewport shifts left with Chinese IME in specific TUI(gemini cli) apps on Windows with ConPTY

[https://github.com/microsoft/vscode/issues/255285](https://github.com/microsoft/vscode/issues/255285)

## Summary

The root cause was an older OS-bundled ConPTY on Windows 10. VS Code now ships ConPTY 1.25 as a DLL and enables it by default (1.121.0), which delivers the same newer ConPTY that Windows Terminal used to handle this scenario gracefully — matching the maintainer's assessment that later ConPTY versions fix this.

## Change summary

Terminal viewport shifts left when composing Chinese IME input in gemini-cli on Windows with the older OS-bundled ConPTY.

## Best solution

Keep the shipped ConPTY DLL default. The reporter's own comparison showed Windows Terminal (newer ConPTY) handled this correctly; VS Code now ships the same newer ConPTY by default, resolving the upstream defect without OS upgrades.
