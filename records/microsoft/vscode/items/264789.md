# Issue #264789 — Agent can't find existing terminal running background watch task

[https://github.com/microsoft/vscode/issues/264789](https://github.com/microsoft/vscode/issues/264789)

## Summary

The underlying tool infrastructure (`runTaskTool`) already detects active tasks and reports 'already running,' but the agent's decision layer (model prompting/planning) doesn't reliably consult task state before proposing a raw `npm run compile` via `runInTerminal`. No fix commit found for this agent-planning gap; keeping open.

## Change summary

When a watch task is already running, the Copilot agent fails to detect it and proposes running `npm run compile` in a new terminal instead of reusing the existing task.

## Best solution

Improve the agent's system prompt or tool-selection logic so it queries active tasks (via `getTaskOutput` or `runTask`) before proposing compile/build commands through `runInTerminal`. The model description in `runTaskTool.ts` already hints at this ('Make sure that any appropriate build or watch task is running before trying to run tests'), but the agent doesn't always follow it. A stronger context injection of currently-running tasks into the agent's planning step would close the gap.
