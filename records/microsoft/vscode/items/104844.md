# Issue #104844 — Undo behavior: non-focused panel scrolls

[https://github.com/microsoft/vscode/issues/104844](https://github.com/microsoft/vscode/issues/104844)

## Summary

Keep open: the report has clear steps and is already labeled as an undo/redo bug, and I did not find a concrete fixing commit or duplicate to justify closure. Current main still stores undo cursor state on the text model undo stack, so the split-editor independence concern remains plausible without a verified fix.

## Change summary

Undo in one split editor for a shared file should not make the non-focused split lose its independent scroll position.

## Best solution

The undo/redo/editor cursor path should preserve each editor widget's independent view state when multiple editors share the same text model, likely around `src/vs/editor/common/model/editStack.ts`, `src/vs/editor/common/model/textModel.ts`, and `src/vs/editor/common/cursor/cursor.ts`. A maintainer should either add a focused regression test for split editors sharing a model or decide explicitly that model-level undo cursor recovery is intended.

## Evidence

- **issue detail:** https://github.com/microsoft/vscode/issues/104844 reports reproducible steps on VS Code 1.48.0 with extensions disabled: split one file, edit locations more than one screen apart, then undo in one pane and observe the other pane scrolling.
- **current code:** `src/vs/editor/common/model/editStack.ts` stores `beforeCursorState` and `afterCursorState` in `SingleModelEditStackData`, and `SingleModelEditStackElement.undo()` passes `data.beforeCursorState` into `TextModel._applyUndo(...)`.
- **current code:** `src/vs/editor/common/model/textModel.ts` forwards the undo `resultingSelection` through `_applyUndoRedoEdits(...)` and emits it with the model content change event.
- **current code:** `src/vs/editor/common/cursor/cursor.ts` only applies `e.resultingSelection` directly when `this._hasFocus` is true; unfocused editors recover from cursor markers, so the code still needs an explicit split-editor regression check before this can be considered fixed.
