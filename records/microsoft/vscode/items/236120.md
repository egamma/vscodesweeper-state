# Issue #236120 — Opening a `devcontainer.json` file in a codespace opens a prompt to install git locally

[https://github.com/microsoft/vscode/issues/236120](https://github.com/microsoft/vscode/issues/236120)

## Summary

Milestone is 'On Deck' (a scheduling milestone), so this must stay open for maintainer judgment. The bug is real and traceable in current source: the git extension's `findGitDarwin` runs `which git` and `xcode-select -p` locally even when VS Code is connected to a remote/codespace, triggering the macOS developer-tools install prompt for users without local git.

## Change summary

Opening a devcontainer.json in a Codespace triggers a macOS prompt to install Xcode command-line tools because the git extension probes for local git even during a remote session.

## Best solution

The git extension's local git discovery in `extensions/git/src/git.ts` (`findGitDarwin`) should either skip the local probe when the active workspace is remote-only, or the extension should be configured to not activate its local-git-finding logic when all open folders are remote. The maintainer assigned this to 'On Deck' and was actively investigating in the comments.
