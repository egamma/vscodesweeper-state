# Issue #279280 — CLI `--add-mcp` JSON argument loses quotes when called from PowerShell via `code.cmd`

[https://github.com/microsoft/vscode/issues/279280](https://github.com/microsoft/vscode/issues/279280)

## Summary

Still reproduces on main: `resources/win32/bin/code.cmd` passes arguments via `%*`, which causes `cmd.exe` to strip inner double quotes from JSON arguments when invoked from PowerShell. No fix has been committed.

## Change summary

The `code.cmd` wrapper mangles JSON double quotes passed from PowerShell because `cmd.exe`'s `%*` expansion strips them.

## Best solution

Rewrite `code.cmd` to properly quote-preserve arguments (e.g. using a loop with `%~1` and re-quoting each argument), or ship a `code.ps1` PowerShell wrapper alongside `code.cmd` that PowerShell would prefer, bypassing `cmd.exe` entirely. The fix lives in `resources/win32/bin/code.cmd` and `resources/win32/versioned/bin/code.cmd`.
