# Issue #324353 — lnk windows files

[https://github.com/microsoft/vscode/issues/324353](https://github.com/microsoft/vscode/issues/324353)

## Summary

This is a duplicate of https://github.com/microsoft/vscode/issues/89743, which was closed as *out-of-scope. VS Code does not resolve Windows .lnk shortcut files — they are a Windows shell concept, and VS Code treats them as opaque files. The same request has been declined twice before.

## Change summary

When opening a .lnk (Windows shortcut) file with VS Code, resolve the shortcut target instead of opening the .lnk file itself.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/89743 — the team has already evaluated and declined this as out-of-scope. If the reporter's workflow depends on shortcut resolution, the Windows shell's 'Open with' on the shortcut target (or using the actual folder path) is the workaround.

## Vision fit

- **Fit:** rejected
- **Reason:** Resolving Windows .lnk files is a platform-specific shell behavior that adds OS-specific complexity to the core editor, conflicting with the value of keeping VS Code lightweight and cross-platform.
- **Evidence:** Value: "Out of the box, VS Code is a text editor: lightweight and fast." Guardrail: lean default experience; platform-specific shell integration for a niche audience cuts against this.
