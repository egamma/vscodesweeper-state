# Issue #230033 — Git - GIT_ASKPASS fails on concurrent / interleaved username & password requests

[https://github.com/microsoft/vscode/issues/230033](https://github.com/microsoft/vscode/issues/230033)

## Summary

Still reproduces on main: the askpass credential cache is deleted on the first password request, so concurrent git processes sharing the same authority lose their cached password. The fix has not landed.

## Change summary

Concurrent GIT_ASKPASS invocations for the same authority fail because the credential cache is consumed and cleared by the first password request, leaving subsequent ones empty.

## Best solution

Change the cache eviction strategy in `extensions/git/src/askpass.ts` so that the cached credentials persist across multiple concurrent password requests for the same authority — e.g. use a reference count or a short TTL that outlives the concurrent window, rather than deleting on first consumption.
