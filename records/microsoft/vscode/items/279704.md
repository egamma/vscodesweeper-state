# Issue #279704 — Figure out a way to advertise MCP servers contributed by extensions without such a high cost of tokens

[https://github.com/microsoft/vscode/issues/279704](https://github.com/microsoft/vscode/issues/279704)

## Summary

Team-authored feature request for smarter MCP server tool advertisement to reduce token cost — kept open per policy (MEMBER author). The problem (all extension-contributed MCP tools dumped into every request) remains relevant to ongoing agent efficiency work.

## Change summary

Extension-contributed MCP servers currently auto-start and include all their tools in every LLM request, wasting tokens when the server has many tools.

## Best solution

Implement a lazy/on-demand tool advertisement mechanism for extension-contributed MCP servers — e.g. tool filtering, dynamic activation based on context, or a registration model that lets the LLM discover tools without paying full token cost upfront. Work likely lives in `src/vs/workbench/contrib/chat/` MCP integration code.
