# Issue #283457 — Restart extensions isn't properly updating extension configuration

[https://github.com/microsoft/vscode/issues/283457](https://github.com/microsoft/vscode/issues/283457)

## Summary

Confirmed bug — still unfixed on main. Two other issues (#227711, #287872) were closed as duplicates of this one. The configuration extension point handler processes added/removed extensions via deltaConfiguration, but newly added configuration defaults from an updated extension manifest are not properly applied after 'Restart Extensions' (only a full window reload picks them up).

## Change summary

When an extension adds a new configuration setting with a default in package.json and the user clicks 'Restart Extensions', the new default is not reflected in the configuration service until a full window reload.

## Best solution

Fix the restart flow in `src/vs/workbench/contrib/extensions/browser/extensionsWorkbenchService.ts` (`updateRunningExtensions`) and/or `src/vs/workbench/api/common/configurationExtensionPoint.ts` so that when an extension is removed and re-added, its full manifest (including any new configuration properties and defaults) is re-scanned and applied to the configuration registry. The `toExtensionDescription()` call should re-read the manifest from disk, and the configuration delta should include the new settings.
