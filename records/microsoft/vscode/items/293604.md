[https://github.com/microsoft/vscode/issues/293604](https://github.com/microsoft/vscode/issues/293604)

## Summary

The error and hang originate entirely within the Copilot Chat extension (github.copilot-chat v0.37.1), which attempts to register the `copilot-skill` filesystem provider a second time during activation in remote scenarios. The full stack trace is inside the extension's bundled code, not VS Code core. This is a `*caused-by-extension` issue — any fix belongs in the extension, which has received many updates since v0.37.1.

## Change summary

Copilot Chat hangs on remote SSH connections with 'Chat took too long to get ready' due to a duplicate filesystem provider registration error in the extension.

## Best solution

The root cause is in the Copilot Chat extension's activation path, not VS Code core. The extension (v0.37.1) tries to register a `copilot-skill` scheme provider twice when reconnecting on a remote host.
