# Issue #200011 — Git - compare the working tree with other revisions

[https://github.com/microsoft/vscode/issues/200011](https://github.com/microsoft/vscode/issues/200011)

## Summary

The request for an explorer-context-menu 'Compare with branch/tag/commit' workflow remains unimplemented. The built-in git extension has graph-level compare commands and SCM-view compare, but no file/folder-level 'compare with arbitrary revision' from the explorer. With only 6 upvotes and no comments, it hasn't reached community traction thresholds, but it's a valid feature request in a durable-theme area.

## Change summary

Add a file/folder right-click context menu option to compare working tree files against other git revisions (branches, tags, commits).

## Best solution

Keep open for the community process. The git extension already has `git.graph.compareRef` and `git.compareWithWorkspace` commands — extending these patterns to offer a file-explorer context menu entry ('Compare with Branch/Tag/Commit…') in `extensions/git` would address the request. Alternatively, this is well-served by extensions like GitLens, making it a borderline `*extension-candidate`.
