# Issue #137327 — Debugging on OSX with iTerm as external terminal creates a new window each time

[https://github.com/microsoft/vscode/issues/137327](https://github.com/microsoft/vscode/issues/137327)

## Summary

Valid feature request: the iTerm AppleScript helper (`iTermHelper.scpt`) still lacks session-reuse logic that the Terminal.app helper already implements. No fix has landed, no PR is open, and no canonical issue supersedes it. With only 5 upvotes and no recent traction, it stays in the backlog as-is.

## Change summary

When debugging with iTerm as the external terminal on macOS, a new window is created every time instead of reusing the previous session (unlike the built-in Terminal.app behavior).

## Best solution

Update `src/vs/workbench/contrib/externalTerminal/node/iTermHelper.scpt` to track the previous session and reuse it when idle, mirroring the `TerminalHelper.scpt` approach. The commenter's `is at shell prompt` strategy (requiring iTerm shell integration) is a reasonable path — consider showing a one-time notification about the shell-integration requirement.
