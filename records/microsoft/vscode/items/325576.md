# Issue #325576 — Work tree created for the prompt "List my sessions"

[https://github.com/microsoft/vscode/issues/325576](https://github.com/microsoft/vscode/issues/325576)

## Summary

Team-authored issue requesting smarter worktree isolation — skip worktree creation for read-only prompts that don't need file system changes. Keeping open for maintainer judgment.

## Change summary

A worktree is created even for read-only agent prompts like 'List my sessions' that don't require file system isolation.

## Best solution

Make worktree creation conditional on the prompt/tools actually needing file system writes, or defer worktree creation until a write-capable tool is invoked, so purely read-only sessions avoid the overhead.
