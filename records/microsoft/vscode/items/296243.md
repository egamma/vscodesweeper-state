# Issue #296243 — Handle rate limit errors more gracefully

[https://github.com/microsoft/vscode/issues/296243](https://github.com/microsoft/vscode/issues/296243)

## Summary

Valid polish request to add a one-click "Retry with Auto" inline button when rate-limit errors appear. The current code in chatQuotaNotification.ts shows rate-limit warnings with an empty actions array — no retry affordance exists on main. A team member (nhu-do) endorsed extending this to model degradation/deprecation messages too. Keeping open for the area owner.

## Change summary

Add an inline "Retry with Auto" action button to the rate-limit error notification so users can switch model and retry in one click.

## Best solution

Populate the `actions` array in `_showRateLimitWarning` (and potentially quota-exhausted paths) in `src/vs/workbench/contrib/chat/browser/chatQuotaNotification.ts` with a button that switches the selected model to Auto and re-sends the last message. As nhu-do suggested, the same pattern could cover model-degraded and model-deprecated notifications.
