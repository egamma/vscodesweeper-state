# Issue #232232 — Views drag and drop: allow to drop into the entire bar instead of splitting

[https://github.com/microsoft/vscode/issues/232232](https://github.com/microsoft/vscode/issues/232232)

## Summary

Active design discussion among team members about improving view DnD behavior — proposed but not yet implemented. Multiple maintainers support the direction; keeping open for implementation.

## Change summary

When dragging a view into a sidebar/panel, dropping in the main area should move it as a new container instead of splitting, aligning with editor group DnD behavior.

## Best solution

Implement the editor-group-style DnD zones for view containers: drop in the center moves/adds the view, drop in the edge splits. The shift-modifier proposal from sbatten offers a good escape hatch. Work belongs in the workbench DnD/layout area (src/vs/workbench/browser/parts/views/).
