# Issue #143991 — DAP - Terminate Request is not sent for "Attach" Configurations

[https://github.com/microsoft/vscode/issues/143991](https://github.com/microsoft/vscode/issues/143991)

## Summary

Still reproduces on current main: the terminate request is gated on `request === 'launch'` at line 438 of debugSession.ts, so attach configurations never receive the DAP terminate request even when the user explicitly clicks Stop. A maintainer (weinand) acknowledged the bug and agreed it should be fixed, but the fix was never applied.

## Change summary

DAP Terminate Request is not sent for 'attach' debug configurations when the user Alt-clicks to reveal and press the Stop button, because the code unconditionally requires `request === 'launch'`.

## Best solution

Remove the `&& this._configuration.resolved.request === 'launch'` guard on line 438 of `src/vs/workbench/contrib/debug/browser/debugSession.ts` so that `terminate` is called whenever `supportsTerminateRequest` is true and the user explicitly requests termination, regardless of launch vs attach. The `terminate()` method is only invoked when the user clicks Stop (not Disconnect), so the attach lifecycle semantics are preserved.
