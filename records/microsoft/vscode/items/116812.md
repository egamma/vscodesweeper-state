# Issue #116812 — Menubar access keys are not underlined in native titlebar mode

[https://github.com/microsoft/vscode/issues/116812](https://github.com/microsoft/vscode/issues/116812)

## Summary

Confirmed upstream issue in Electron/Chromium: native titlebar menus are handled by Electron, which does not honor the Windows setting for always-underlined access keys. VS Code already respects this setting in custom titlebar mode. Keeping open as upstream.

## Change summary

In native titlebar mode on Windows, menubar access keys are not underlined until Alt is pressed, ignoring the OS setting to always show access key underlines.

## Best solution

This requires a fix in Electron's native menu handling to respect the Windows `SPI_GETKEYBOARDCUES` setting. VS Code already implements this correctly for the custom titlebar (the code path they control). Track upstream and close once Electron ships the fix, or suggest users switch to the custom titlebar as a workaround.
