## Change summary

Switching from a corp GitHub account to a free GitHub account in Insiders can leave Chat broken with 'Failed to get a response' due to a 'No default agent registered' error.

## Best solution

The underlying race is between authentication state changes and Copilot extension activation/agent registration. The fix likely involves ensuring `activateDefaultAgent` in `src/vs/workbench/contrib/chat/common/chatService/chatServiceImpl.ts` retries or waits for the extension to re-register its agent after an account switch, or that `chatSetupProviders.ts` handles the transient auth-failure path more gracefully. The `confirmation-pending` label is appropriate — the bug is intermittent and needs a reliable reproduction path before a targeted fix.
