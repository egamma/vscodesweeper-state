# Issue #172460 — Setting trim_trailing_whitespace = true breaks undo history

[https://github.com/microsoft/vscode/issues/172460](https://github.com/microsoft/vscode/issues/172460)

## Summary

Keep open: the current save participant still performs trim-trailing-whitespace as a model edit, and there is no concrete fixing commit or regression test showing the reported redo-history loss is resolved.

## Change summary

Saving with `files.trimTrailingWhitespace` after undoing a user edit can clear the redo entry that would otherwise restore the undone text.

## Best solution

Add a focused regression test in `src/vs/workbench/contrib/codeEditor/test/browser/saveParticipant.test.ts` mirroring the existing trim-final-newlines undo test, then fix `TrimWhitespaceParticipant` in `src/vs/workbench/contrib/codeEditor/browser/saveParticipants.ts` so save-time whitespace trimming does not unnecessarily destroy redo history.
