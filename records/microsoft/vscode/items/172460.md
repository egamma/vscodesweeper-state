# Issue #172460 — Setting trim_trailing_whitespace = true breaks undo history

[https://github.com/microsoft/vscode/issues/172460](https://github.com/microsoft/vscode/issues/172460)

## Summary

Still reproduces on main: the trim-trailing-whitespace save participant uses `pushEditOperations` which inherently clears the redo stack when it edits the buffer after an undo. Maintainers have labeled this as a bug to track, even though #15458 was previously closed as *as-designed.

## Change summary

Enabling trim_trailing_whitespace causes redo history to be lost when a save triggers whitespace trimming after an undo operation.

## Best solution

The save participant's trim edits need to either not clear the redo stack (e.g. via a special edit mode that preserves redo) or the trimming should be applied only at the file-write layer without touching the in-memory model's undo history. This is architecturally non-trivial — it requires changes to how save participants interact with the undo/redo stack in `src/vs/editor/common/model/textModel.ts` and `src/vs/workbench/contrib/codeEditor/browser/saveParticipants.ts`.
