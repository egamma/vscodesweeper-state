## Summary

Root cause is ambiguous — the stack traces consistently point to `ms-python.vscode-python-envs` failing during disposal after extension host termination, not to Copilot code itself. The reporter's scenario (millions of files) likely causes resource exhaustion that terminates the extension host, with the disposal errors being a secondary symptom. An open community PR (#311739) references this issue, so it stays open for maintainer review.

## Change summary

## Best solution

A maintainer should review whether #311739's null-guard approach addresses the real problem or just papers over a symptom. The actual root cause appears to be extension host resource exhaustion with extremely large workspaces — the Copilot extension may add enough overhead to tip the system. The `outputChannelLogTarget.ts` already has a shutdown resilience fix (b4b65facc06). Consider whether the `ms-python.vscode-python-envs` disposal crash is the primary issue and whether this should be redirected to that extension's tracker.
