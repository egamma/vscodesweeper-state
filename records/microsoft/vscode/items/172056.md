# Issue #172056 — "Change All Occurrences" cursors get out of sync if one is moved with mouse

[https://github.com/microsoft/vscode/issues/172056](https://github.com/microsoft/vscode/issues/172056)

## Summary

Valid bug, confirmed by a team member with a video. Mouse click during 'Change All Occurrences' multi-cursor mode desynchronizes selections across cursors, causing inconsistent edits. No fix found on main.

## Change summary

Using the mouse to reposition one cursor during 'Change All Occurrences' desynchronizes selections across multi-cursors, so subsequent edits (e.g. backspace) behave differently at each cursor.

## Best solution

The mouse click handler during multi-cursor mode should either collapse all cursor selections consistently, or prevent repositioning a single cursor without affecting the others. Fix likely lives in `src/vs/editor/browser/controller/mouseHandler.ts` or the cursor state management in `src/vs/editor/common/cursor/`.
