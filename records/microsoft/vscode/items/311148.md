# Issue #311148 — Agents App: Support multi-root workspaces with cross-repo change tracking and worktrees

[https://github.com/microsoft/vscode/issues/311148](https://github.com/microsoft/vscode/issues/311148)

## Summary

Valid, well-documented feature request for multi-root workspace support in the Agents window. The current `SessionsWorkspaceContextService` manages workspace folders in-memory and doesn't support `.code-workspace` files. Several related open issues exist but none is a canonical duplicate that fully absorbs this scope. Keeping open for the community process — 16 upvotes, approaching the 20-vote backlog threshold.

## Change summary

Support opening `.code-workspace` files in the Agents window with cross-repo change tracking, per-session repository scoping, and per-repo worktree creation.

## Best solution

The Agents window needs to extend its workspace model (`SessionsWorkspaceContextService` in `src/vs/sessions/services/workspace/browser/workspaceContextService.ts`) to accept `.code-workspace` file input, discover child git repos, and expose a repository picker when starting sessions. The underlying agent-host multi-root support tracked in https://github.com/microsoft/vscode/issues/321651 is a prerequisite. This is a significant feature spanning workspace services, session creation UX, and the Changes panel — it needs product and design decisions, not just code.
