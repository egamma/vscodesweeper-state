# Issue #202942 — Wrong indent detected with PHP comments

[https://github.com/microsoft/vscode/issues/202942](https://github.com/microsoft/vscode/issues/202942)

## Summary

Still present on main: the indentation guesser in indentationGuesser.ts has no logic to skip or de-weight lines inside block comments, so comment content can skew the detected indent size.

## Change summary

Indentation detection incorrectly counts indentation inside PHP block comments, overriding the actual code indentation.

## Best solution

Teach the indentation guesser to skip or down-weight lines inside block comments (/* … */) so that example code in doc-blocks does not skew detection. The fix belongs in src/vs/editor/common/model/indentationGuesser.ts; however, doing this language-agnostically (without tokenization) is non-trivial and may require heuristics for common comment markers.
