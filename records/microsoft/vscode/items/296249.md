# Issue #296249 — [regression] Coloured brackets very laggy

[https://github.com/microsoft/vscode/issues/296249](https://github.com/microsoft/vscode/issues/296249)

## Summary

Still reproduces on main. The variable-fonts commit (474b70cbb72) switched whitespace overlay from a batch `getMinimapLinesRenderingData` call to per-line `getViewLineRenderingData`, which triggers `getDecorationsOnLine` (and therefore `collectBrackets`) for every visible line on every render. No subsequent commit has addressed this repeated-decoration-fetch overhead.

## Change summary

Colored bracket rendering became very slow after the variable-fonts change because whitespace overlay now fetches decorations (including bracket colorization) per-line instead of in batch.

## Best solution

Restore batch decoration fetching in `WhitespaceOverlay.prepareRender` (use `getMinimapLinesRenderingData` or a similar viewport-scoped batch call that avoids per-line `getDecorationsOnLine`), or filter out bracket-pair decorations from the whitespace overlay's decoration query since it doesn't need them.
