# Issue #296249 — [regression] Coloured brackets very laggy

[https://github.com/microsoft/vscode/issues/296249](https://github.com/microsoft/vscode/issues/296249)

## Summary

Confirmed regression from the variable fonts PR (#252308, commit 474b70cbb) — bracket pair colorization becomes very slow because getDecorationsInRange triggers collectBrackets many extra times per render when whitespace rendering is enabled. No fix has landed on main.

## Change summary

Colored bracket rendering became very slow after the variable fonts change when whitespace rendering is enabled, due to repeated bracket tree traversals per render frame.

## Best solution

The whitespace overlay or the rendering pipeline should avoid triggering redundant getDecorationsInRange calls that invoke the bracket pair tree. Either cache the bracket decorations per frame, reduce the number of getDecorationsInRange calls introduced by the variable fonts path, or add an early-out so the ColorizedBracketPairsDecorationProvider is not re-queried for whitespace-only decoration fetches.
