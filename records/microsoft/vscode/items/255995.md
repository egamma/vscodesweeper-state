# Issue #255995 — Profiles editor: it's possible to uncheck `Use for New Windows`

[https://github.com/microsoft/vscode/issues/255995](https://github.com/microsoft/vscode/issues/255995)

## Summary

Still reproduces on main: the checkbox can be unchecked without any UI indication of the fallback behavior (new windows will use the active window's profile). The maintainer acknowledged the issue and suggested adding clarifying text, but no fix has landed.

## Change summary

Unchecking 'Use for New Windows' in the Profiles editor leaves users with no indication of which profile new windows will use.

## Best solution

Either disable unchecking when only one profile has the checkbox (forcing at least one to always be set), or add descriptive text explaining the fallback behavior ('new windows will use the active window's profile') — as sandy081 suggested in the comment. Fix lives in `src/vs/workbench/contrib/userDataProfile/browser/userDataProfilesEditor.ts` around the `UseAsDefaultProfileRenderer`.
