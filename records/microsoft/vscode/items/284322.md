# Issue #284322 — Current line box thingy doesn't extend fully when switching between tabs

[https://github.com/microsoft/vscode/issues/284322](https://github.com/microsoft/vscode/issues/284322)

## Summary

Still open on main — the current line highlight width is computed from ctx.scrollWidth at render time, but when switching tabs with overflowing content, the scroll width may not yet reflect the new tab's content width, causing the highlight to appear truncated. Milestone is 'On Deck', so keeping open per scheduling rules.

## Change summary

Current line highlight box doesn't extend to full content width when switching between editor tabs containing overflowing text.

## Best solution

In `CurrentLineHighlightOverlay._renderOne`, the width uses `Math.max(ctx.scrollWidth, this._contentWidth)` — when a tab becomes visible, the rendering context's scrollWidth may be stale. The fix likely involves ensuring a re-render is triggered after the editor's layout/scroll dimensions are fully recalculated on tab switch, or responding to a tab-visibility event. The fix belongs in `src/vs/editor/browser/viewParts/currentLineHighlight/currentLineHighlight.ts` or in the editor view's tab-switch lifecycle.
