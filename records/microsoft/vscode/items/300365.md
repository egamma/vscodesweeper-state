# Issue #300365 — Crash when deleting any file on Windows unless files.enableTrash=false

[https://github.com/microsoft/vscode/issues/300365](https://github.com/microsoft/vscode/issues/300365)

## Summary

Verified duplicate of #284211 — the underlying problem is native OS dialogs crashing VS Code (an Electron/Chromium issue, most commonly on Wayland). The later comments on this issue confirm the same dialog-triggered crash pattern, and #284211 already has team engagement and crash dumps submitted.

## Change summary

VS Code crashes when any native dialog is triggered (file delete confirmation, save dialog, file picker), particularly on Wayland sessions.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/284211, which is the oldest report with team engagement (deepak1556 requested crash dumps). The fix likely lives in Electron's Wayland dialog handling. Workaround is `window.dialogStyle: custom`.
