# Issue #123517 — Relaunching VSCode forgets what macOS Desktop each window belongs to

[https://github.com/microsoft/vscode/issues/123517](https://github.com/microsoft/vscode/issues/123517)

## Summary

Valid, community-backed request (73 upvotes, in Backlog) with an open PR #308301 providing an AppKit-based fix via a native module. The issue is labeled upstream/Electron, but the PR proposes a workaround that doesn't require Electron changes. Keeping open — the PR needs maintainer review.

## Change summary

VS Code does not restore windows to their original macOS Spaces (virtual desktops) on restart, unlike native macOS apps.

## Best solution

Review and decide on https://github.com/microsoft/vscode/pull/308301, which adds `electron-osx-spaces` (AppKit state restoration) to persist and restore macOS Space assignment per window. The approach bypasses the upstream Electron gap (https://github.com/electron/electron/issues/37494). If the native-module dependency is acceptable, landing that PR resolves this and the related https://github.com/microsoft/vscode/issues/146915.
