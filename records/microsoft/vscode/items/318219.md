  > Thanks for the report. This isn't a VS Code bug — it's a configuration issue with your KDE desktop environment. VS Code relies on the OS keyring (kwallet on KDE) to securely store credentials.
  > 
  > To fix this, ensure kwallet is installed and running:
  > 1. Install `kwallet` and `kwalletmanager` if not already present.
  > 2. Make sure the KDE Wallet service is enabled in your system settings (System Settings → KDE Wallet).
  > 3. Restart VS Code after kwallet is running.
  > 
  > Alternatively, you can install `gnome-keyring` as a fallback secret store.
  > 
  > For more details, see the [VS Code Linux prerequisites](https://code.visualstudio.com/docs/setup/linux). Closing as a support question — please feel free to ask on [Stack Overflow](https://stackoverflow.com/questions/tagged/visual-studio-code) for further help with environment setup.
