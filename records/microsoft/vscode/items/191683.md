# Issue #191683 — Experience when having a lot of matches

[https://github.com/microsoft/vscode/issues/191683](https://github.com/microsoft/vscode/issues/191683)

## Summary

Team-authored feature request on the 'On Deck' milestone — keeping open. The quick text search already limits results (30 files, 10 matches/file) and shows a 'See More Files' link, but lacks explicit validation messaging warning users when results are too large and encouraging opening in the search view.

## Change summary

Add validation messaging in Quick Text Search when results are too large, encouraging the user to open results in the Search view.

## Best solution

Add a visible info/warning pick item at the top of the quick pick results when the result count hits the limit, indicating how many total matches were found and suggesting the user open the full Search view for better exploration. The implementation would live in `src/vs/workbench/contrib/search/browser/quickTextSearch/textSearchQuickAccess.ts`, likely near the existing limit-handling logic around line 246/378.
