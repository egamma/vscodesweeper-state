# Issue #191683 — Experience when having a lot of matches

[https://github.com/microsoft/vscode/issues/191683](https://github.com/microsoft/vscode/issues/191683)

## Summary

Team-authored feature request for improving Quick Text Search when there are too many matches. Current main has file/match limits and 'See More Files'/'Open in Search View' affordances, but still lacks a validation message or total match count summary as requested. Keeping open per team-author policy.

## Change summary

Add validation messaging and total match count display in Quick Text Search when results exceed practical limits, encouraging users to open the full Search view.

## Best solution

Add a summary line (e.g. '50 matches in 7 files') at the top of the quick pick results in `textSearchQuickAccess.ts`, and show a validation/warning message when the result set hits the limit, nudging users toward the Search view. The related suggestion from #191754 to reduce matches-per-file dynamically when the list is long could also be considered.
