# Issue #316243 — Copilot CLI agent panel: CWD silently inherited from active terminal, not from open workspace

[https://github.com/microsoft/vscode/issues/316243](https://github.com/microsoft/vscode/issues/316243)

## Summary

The VS Code core agent host working-directory resolution (`_resolveRequestedWorkingDirectory` in `agentHostSessionHandler.ts`) does NOT inherit from the active terminal — it falls back through explicit resolvers and then to the workspace folder. The described behavior may originate from an older Copilot Chat extension version (0.41) or a specific interaction path not reproducible on current main. More details are needed to confirm whether this still reproduces on the latest Insiders build.

## Change summary

The Copilot CLI agent panel silently picks up the active terminal's CWD instead of defaulting to the open workspace root, causing misrouted skills, wrong files panel, and invisible configuration issues.

## Best solution

Ask the reporter to reproduce on the latest VS Code Insiders + latest Copilot Chat extension. If confirmed, the root cause likely lives in the Copilot extension's session-launch path (`extensions/copilot/src/extension/chatSessions/`) rather than VS Code core's `_resolveRequestedWorkingDirectory` chain, which already defaults to workspace folders.
