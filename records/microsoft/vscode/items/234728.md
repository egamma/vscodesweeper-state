# Issue #234728 — [Bug] `"editor.glyphMargin": false` not working for diff editors

[https://github.com/microsoft/vscode/issues/234728](https://github.com/microsoft/vscode/issues/234728)

## Summary

Still reproduces on main: the left-hand diff editor unconditionally forces `glyphMargin = true` in side-by-side mode, ignoring the user's `editor.glyphMargin: false` setting.

## Change summary

Setting `editor.glyphMargin: false` has no effect on the original (left) side of a side-by-side diff editor.

## Best solution

Line 152 of `diffEditorEditors.ts` should respect the user setting — e.g. `result.glyphMargin = this._options.renderSideBySide.get() && (changedOptions.glyphMargin !== false)` — so the glyph margin is only forced on when the user hasn't explicitly disabled it.
