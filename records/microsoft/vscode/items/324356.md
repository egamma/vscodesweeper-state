# Issue #324356 — Using find references in the scm diff view doesn't currently work (possibly by design or bug)

[https://github.com/microsoft/vscode/issues/324356](https://github.com/microsoft/vscode/issues/324356)

## Summary

The `git` scheme is explicitly in the TypeScript extension's `disabledSchemes` set (since commit f694b07825a2, 2020), so language features like Find All References are intentionally disabled for SCM diff views. This is a well-researched request to enable them by unwrapping git URIs to their underlying file paths. Needs maintainer judgment on side effects (e.g., historical revisions vs working-copy files).

## Change summary

Find All References and other TypeScript language features do not work in the SCM diff viewer because the `git://` URI scheme is in the disabled schemes list.

## Best solution

The reporter's proposed approach — removing `git` from `disabledSchemes` and unwrapping git URIs to their real file paths in `toTsFilePath` — is sound but needs maintainer review for edge cases (historical revisions, merge conflicts). The fix lives in `extensions/typescript-language-features/src/configuration/fileSchemes.ts` and `extensions/typescript-language-features/src/typescriptServiceClient.ts`.
