# Issue #326585 — Terminal renders badly in light mode with gpu acceleration

[https://github.com/microsoft/vscode/issues/326585](https://github.com/microsoft/vscode/issues/326585)

## Summary

Valid terminal rendering bug on light themes with GPU acceleration (WebGL). The root cause likely lives in xterm.js's WebGL addon text rendering pipeline — how it handles alpha blending or subpixel antialiasing differs between light and dark backgrounds. Related to (but distinct from) editor GPU rendering issue #240774. Already labeled appropriately; keeping open for terminal team investigation.

## Change summary

Terminal text renders thin and jagged on light themes with GPU acceleration enabled, while dark themes render correctly.

## Best solution

Investigate whether the xterm.js WebGL addon's text atlas or glyph rendering handles light-background contrast/alpha differently from dark, and fix upstream in xterm.js if so. The workaround is `terminal.integrated.gpuAcceleration: off`. If the fix is in xterm.js, it would ship via an xterm.js version bump.
