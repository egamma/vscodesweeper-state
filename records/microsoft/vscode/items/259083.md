# Issue #259083 — Worktree within worktree not showing in UI

[https://github.com/microsoft/vscode/issues/259083](https://github.com/microsoft/vscode/issues/259083)

## Summary

Team-filed bug on the 'On Deck' milestone with an open PR (#267692) that addresses it. Multiple hard keep-open rules apply (MEMBER author, real milestone, open PR). The worktree discovery logic in `extensions/git/src/model.ts` only auto-detects worktrees from the 'main' repository, not from a worktree opened in a new window — the bug is traceable in source.

## Change summary

Creating a worktree from within an already-opened worktree (nested worktree creation) does not reflect the new worktree in the SCM UI.

## Best solution

Land the open PR https://github.com/microsoft/vscode/pull/267692, which adds worktree hierarchy detection from inside and outside the primary worktree. The fix touches the git extension's model/repository layer (`extensions/git/src/model.ts`, `extensions/git/src/repository.ts`).
