# Issue #20652 — [themes] different themes for different filetypes

[https://github.com/microsoft/vscode/issues/20652](https://github.com/microsoft/vscode/issues/20652)

## Summary

This feature request for per-filetype color themes remains unimplemented. The theming engine still supports only one active theme at a time. The issue is in the Backlog milestone with community interest, so it should stay open.

## Change summary

Allow users to assign different color themes to different file types/languages so the editor changes appearance based on the active file.

## Best solution

The theming engine in `src/vs/workbench/services/themes/` would need refactoring to support multiple simultaneous color themes scoped per-editor. This is a significant architectural change. Alternatively, an extension could approximate this by programmatically switching the global theme on active-editor change, though that affects the entire UI. The issue should remain in Backlog as a long-standing community request.
