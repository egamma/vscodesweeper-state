# Issue #241811 — MSAL Authentication: Error 0x80049dd3 in sign in dialog when using personal account

[https://github.com/microsoft/vscode/issues/241811](https://github.com/microsoft/vscode/issues/241811)

## Summary

Confirmed upstream issue in the Windows identity runtime (WAM). A VS Code maintainer has already identified the root cause, provided a workaround, and escalated to the Microsoft Identity team — keeping open to track resolution.

## Change summary

MSAL/WAM authentication returns error 0x80049dd3 when signing in with a personal Microsoft account not linked to the Windows account.

## Best solution

Await a fix from the Microsoft Identity/WAM team (TylerLeonhardt has escalated). The `"microsoft-authentication.implementation":"classic"` setting works around it in the meantime. Once the upstream fix ships, this can be closed.
