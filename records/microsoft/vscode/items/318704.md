# Issue #318704 — Error - Failed during initial scan: Could not find ripgrep binary in ZooCode extension after update vscode

[https://github.com/microsoft/vscode/issues/318704](https://github.com/microsoft/vscode/issues/318704)

## Summary

Third-party extensions (ZooCode, Roo Code, Todo Tree, Kilocode) were hardcoding the internal path to VS Code's bundled ripgrep binary. VS Code moved from @vscode/ripgrep to @vscode/ripgrep-universal in 1.122, which is an internal implementation detail — extensions should not rely on internal node_modules paths. The affected extensions have already shipped fixes.

## Change summary

After VS Code 1.122, third-party extensions that hardcoded the internal ripgrep binary path fail with 'Could not find ripgrep binary'.

## Best solution

Close as caused-by-extension. The internal module layout is not a public API contract; extensions that accessed it directly need to update. Most affected extensions (ZooCode 3.56+, Kilocode, Todo Tree) have already shipped fixes. Users should update their extensions.
