# Issue #236302 — Command Center should shrink when not enough space

[https://github.com/microsoft/vscode/issues/236302](https://github.com/microsoft/vscode/issues/236302)

## Summary

Still reproduces on main: the command center has a fixed `width: 38vw` with no flex-shrink, so it crowds out menu bar items on smaller windows or with longer-named locale menus. Team members acknowledged the issue and agreed the command center should shrink.

## Change summary

The command center's fixed width pushes menu bar items out of view when there isn't enough horizontal space (smaller screens, non-English locales with longer menu names).

## Best solution

Allow the command center to flex-shrink when the titlebar is space-constrained — likely by uncommenting/adjusting the `flex-shrink: 10` on `.titlebar-center` (line 88 of titlebarpart.css) and ensuring the command center's `width: 38vw` yields to a smaller min-width so the menu bar items remain visible.
