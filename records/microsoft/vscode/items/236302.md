# Issue #236302 — Command Center should shrink when not enough space

[https://github.com/microsoft/vscode/issues/236302](https://github.com/microsoft/vscode/issues/236302)

## Summary

Still present on main: the command center uses a fixed `width: 38vw` that doesn't shrink when the titlebar-left (menu bar) needs more space, causing menu items to overflow. Team members acknowledged and agreed it should shrink.

## Change summary

The Command Center's fixed viewport-relative width pushes menu bar items out of view when menu labels are long (e.g. non-English locales) or the window is narrow.

## Best solution

Allow the command-center-center element to shrink (add a reasonable `min-width` and let flex layout compress it) so it yields space to the menu bar. The fix lives in `src/vs/workbench/browser/parts/titlebar/media/titlebarpart.css` around line 178 (the `width: 38vw` rule) and the titlebar-center flex configuration at line 82–90.
