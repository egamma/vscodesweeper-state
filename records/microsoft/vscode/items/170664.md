# Issue #170664 — Workspace Trust - icon on the status bar cannot be pinned and always resets the state after a restart

[https://github.com/microsoft/vscode/issues/170664](https://github.com/microsoft/vscode/issues/170664)

## Summary

The Workspace Trust status bar entry is intentionally shown only in Restricted Mode (untrusted workspaces) and removed when the workspace is trusted. The right-click context menu toggle controls visibility of registered entries but cannot override the code that conditionally adds/removes the entry based on trust state — so the 'pin' doesn't survive a restart in a trusted workspace because the entry is never re-added.

## Change summary

User wants the Workspace Trust status bar icon to persist (be pinnable) across restarts even in trusted workspaces.

## Best solution

Close as designed. The restricted-mode indicator is intentionally conditional — it signals a non-default security state, not a permanent dashboard item. Keeping it visible in trusted workspaces would add noise to the status bar for most users.

## Evidence

- **source code:** In src/vs/workbench/contrib/workspace/browser/workspace.contribution.ts, `updateStatusbarEntry()` (lines 690-700) explicitly removes the entry when trusted (`statusbarEntryAccessor.clear()`) and only adds it when untrusted. The entry is dynamically registered, not a permanent status bar item.
- **design pattern:** The status bar hide/show model (statusbarModel.ts) controls visibility of *registered* entries. Since the workspace trust entry is never registered in a trusted workspace, the context menu toggle has no lasting effect — the entry simply doesn't exist to show.
- **entry ID:** Entry ID is `status.workspaceTrust` (line 289). It's added with `statusbarService.addEntry()` only in the untrusted branch and cleared in the trusted branch.
