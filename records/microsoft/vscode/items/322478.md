# Issue #322478 — Visible space glyphs with Yutapon font since 1.123

[https://github.com/microsoft/vscode/issues/322478](https://github.com/microsoft/vscode/issues/322478)

## Summary

Keep open and route to the editor rendering area; this is a recent, extension-independent regression with clear version bounds and a specific font family. There is not enough evidence for a close: no confirmed duplicate, no fixing commit, and no reproduction attempt against the affected Windows font stack.

## Change summary

Spaces render as visible glyph marks with the Yutapon font on Windows starting in VS Code 1.123.

## Best solution

Have the editor rendering owner investigate whether the DOM line renderer's use of non-breaking spaces for ordinary spaces now interacts badly with this font or with the Electron/Chromium version used from 1.123 onward; if the root cause is upstream, keep this tracking issue linked to the upstream fix.
