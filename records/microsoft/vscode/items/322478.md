# Issue #322478 — Visible space glyphs with Yutapon font since 1.123

[https://github.com/microsoft/vscode/issues/322478](https://github.com/microsoft/vscode/issues/322478)

## Summary

Likely an upstream Chromium/Electron font rendering regression introduced with Electron 42 (shipped in VS Code 1.123). The Yutapon font's space glyph is now rendered visibly, consistent with other font-rendering issues reported after the same Electron update. Routing to font-rendering area for triage alongside related issues.

## Change summary

Spaces display visible glyph marks when using the Yutapon font, starting from VS Code 1.123.

## Best solution

Investigate whether the Chromium update in Electron 42 changed how the space glyph (U+0020) is rendered for fonts like Yutapon that define a visible space character. If confirmed upstream, file a Chromium bug or apply a VS Code-level workaround in the editor text renderer. May consolidate with https://github.com/microsoft/vscode/issues/320369 if root cause is shared.
