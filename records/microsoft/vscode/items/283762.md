# Issue #283762 — VSCode loses connection with remote after "Developer: Restart Extension Host"

[https://github.com/microsoft/vscode/issues/283762](https://github.com/microsoft/vscode/issues/283762)

## Summary

Keep open: the report has multiple confirmations and the current command implementation still stops and restarts all extension hosts, including the remote host path that can surface the remote reconnection failure. A maintainer already identified a plausible design limitation and suggested redirecting the command to Reload Window in remote windows rather than closing as extension-caused or as-designed.

## Change summary

In a remote window, running Developer: Restart Extension Host drops the remote connection and shows "Cannot reconnect. Please reload the window."

## Best solution

Handle `workbench.action.restartExtensionHost` specially when `IWorkbenchEnvironmentService.remoteAuthority` is set, likely in `src/vs/workbench/services/extensions/electron-browser/nativeExtensionService.ts`, by using the normal reload flow or another remote-safe restart path instead of stopping the remote extension host underneath the active connection.
