# Issue #283762 — VSCode loses connection with remote after "Developer: Restart Extension Host"

[https://github.com/microsoft/vscode/issues/283762](https://github.com/microsoft/vscode/issues/283762)

## Summary

Confirmed design limitation: RestartExtensionHostAction stops all extension hosts including the one managing the remote connection, causing immediate disconnection. A team member acknowledged this and suggested redirecting to Reload Window in remote mode, but no fix has landed on main.

## Change summary

Running 'Developer: Restart Extension Host' while connected to a remote immediately drops the remote connection because the local extension host managing the connection is also stopped.

## Best solution

In `src/vs/workbench/services/extensions/electron-browser/nativeExtensionService.ts`, the `RestartExtensionHostAction` should detect when running in a remote context and either redirect to Reload Window (as aeschli suggested) or only restart non-remote extension hosts. The fix is localized to this action class.
