# Issue #88227 — Ability to drag and drop breakpoints

[https://github.com/microsoft/vscode/issues/88227](https://github.com/microsoft/vscode/issues/88227)

## Summary

Valid, well-supported feature request (40 upvotes, above the 20-upvote Backlog threshold) that has not been implemented on main. PR #114717 was closed without merging, and no breakpoint drag-and-drop code exists in the current codebase. Keeping open.

## Change summary

Allow drag-and-drop of breakpoints (including conditional breakpoints and logpoints) to move them between lines in the editor gutter.

## Best solution

Implement breakpoint drag-and-drop in the editor gutter, building on the GlyphWidgets infrastructure (https://github.com/microsoft/vscode/issues/114776, now closed/landed). The work would likely live in `src/vs/workbench/contrib/debug/browser/` (breakpoint decorations) and `src/vs/editor/browser/` (gutter drag support). Ctrl+drag to copy a breakpoint with its conditions would be a valuable addition.
