# Issue #178269 — Merge Editor not selecting the next unhandled conflict when using merge.toggleActiveConflictInput1/2

[https://github.com/microsoft/vscode/issues/178269](https://github.com/microsoft/vscode/issues/178269)

## Summary

Fixed on main: the root cause — manuallySetActiveModifiedBaseRange staying locked after a toggle via keyboard — was resolved by clearing the lastFocusedEditor cache in setState, shipped in 1.100.0.

## Change summary

After using merge.toggleActiveConflictInput1/2 via keyboard, goToNextUnhandledConflict fails to select the next conflict for toggling.

## Best solution

Keep the shipped fix (commit 1b401f6108e, VS Code 1.100.0). The clearCache call in setState ensures the active range re-derives from cursor position after a toggle, so the keyboard-driven workflow functions correctly.
