# Issue #119824 — Problems from main vscode build or extensions build can be shown, not both when a file is closed

[https://github.com/microsoft/vscode/issues/119824](https://github.com/microsoft/vscode/issues/119824)

## Summary

The race condition between parent and child onModelRemoved listeners in problemCollectors.ts remains structurally present on current main. The issue is filed by a team member and still represents a real gap. Keeping open.

## Change summary

When a file with errors from both the main VS Code build and extensions build tasks is closed, problems from only one build survive due to a race condition in problem collector event ordering.

## Best solution

Fix the listener ordering dependency in `src/vs/workbench/contrib/tasks/common/problemCollectors.ts`: ensure `AbstractProblemCollector.onModelRemoved` updates `openModels` before `WatchingProblemCollector.onModelRemoved` calls `shouldApplyMatch`. The simplest approach is to have the child call the parent's model-tracking logic synchronously before processing lines, rather than relying on independent event subscriptions.
