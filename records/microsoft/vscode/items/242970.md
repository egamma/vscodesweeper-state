# Issue #242970 — Regression: GDK_SCALE and GDK_DPI_SCALE environment variables are ignored

[https://github.com/microsoft/vscode/issues/242970](https://github.com/microsoft/vscode/issues/242970)

## Summary

This is an upstream Chromium/Electron regression: GDK_DPI_SCALE is handled by Chromium's GTK integration, not by VS Code. A commenter confirmed the same behavior in Chrome. The issue is already tagged electron-34-update and under-discussion; it should stay open as an upstream tracking item.

## Change summary

GDK_DPI_SCALE environment variable is ignored since Electron 34 update, causing oversized UI on Linux HiDPI setups using GDK_SCALE=2 with GDK_DPI_SCALE=0.5.

## Best solution

Track upstream in Chromium/Electron. The VS Code codebase has no code handling GDK_DPI_SCALE — it is consumed by Chromium's GTK layer. The workaround is --force-device-scale-factor=2. Electron issue https://github.com/electron/electron/issues/47289 is related. Once Chromium/Electron restores GDK_DPI_SCALE support, a future Electron bump will resolve this.
