# Issue #20488 — [html] prevent that embedded grammars can corrupt syntax highlighting of rest of document

[https://github.com/microsoft/vscode/issues/20488](https://github.com/microsoft/vscode/issues/20488)

## Summary

Long-standing upstream limitation of TextMate grammar semantics — embedded grammars can escape their intended bounds when the inner language has unclosed constructs. Team members confirmed this is inherent to TM design and VS Code intentionally maintains TM compatibility. Kept open as a collector issue.

## Change summary

Incomplete code in embedded language regions (e.g. CSS in <style>) causes TextMate tokenizer to corrupt syntax highlighting of the rest of the document.

## Best solution

A real fix requires either (a) a non-TM tokenization path for HTML-embedded languages (tree-sitter or similar structural parser), or (b) grammar authors adopting `while`-rule workarounds per-language. The issue serves as a tracking point for that longer-term architectural decision. No near-term core fix is viable without diverging from TM semantics.
