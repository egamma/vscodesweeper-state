# Issue #20488 — [html] prevent that embedded grammars can corrupt syntax highlighting of rest of document

[https://github.com/microsoft/vscode/issues/20488](https://github.com/microsoft/vscode/issues/20488)

## Summary

This is a well-understood fundamental limitation of TextMate grammar parsing semantics — embedded grammars can 'escape' their intended region when they encounter incomplete/invalid syntax. The VS Code team explicitly decided to maintain TextMate compatibility rather than fork parsing semantics. No tree-sitter or alternative tokenization system has been adopted on main to resolve this. The issue remains a valid tracking item for this long-standing upstream constraint.

## Change summary

Embedded grammars (CSS/JS in HTML) can corrupt syntax highlighting of the rest of the document when the embedded code is incomplete or invalid.

## Best solution

Ultimately requires moving beyond TextMate grammar semantics for embedded language tokenization — either adopting a tree-sitter-style parser with injection support, or implementing Sublime-style `embed`/`escape` constructs that give the end-pattern priority over nested rules. Until then, individual grammars can partially mitigate with `while` rules (as markdown does for fenced code blocks), but this doesn't fully solve the general case.
