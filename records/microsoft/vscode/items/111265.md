# Issue #111265 — Typing } changes the indentation (autoIndent "none" doesn't work)

[https://github.com/microsoft/vscode/issues/111265](https://github.com/microsoft/vscode/issues/111265)

## Summary

Still reproduces on main: `InterceptorElectricCharOperation` never checks `config.autoIndent`, so typing `}` re-indents the line even when `editor.autoIndent` is `"none"`. The issue is a real, unresolved bug.

## Change summary

Typing `}` changes indentation even when `editor.autoIndent` is set to `"none"`, because electric character handling bypasses the autoIndent setting.

## Best solution

Add an early-return guard in `InterceptorElectricCharOperation.getEdits()` (or `_isTypeInterceptorElectricChar`) that skips electric character reindentation when `config.autoIndent <= EditorAutoIndentStrategy.Keep`. This mirrors how `EnterOperation._enter` already guards on `autoIndent === None`.
