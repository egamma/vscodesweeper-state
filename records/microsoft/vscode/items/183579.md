# Issue #183579 — Aggressive rendering of imbalanced brackets in Markdown

[https://github.com/microsoft/vscode/issues/183579](https://github.com/microsoft/vscode/issues/183579)

## Summary

Still reproduces on main: the `collectBrackets` function in `bracketPairsTree.ts` unconditionally pushes `UnexpectedClosingBracket` nodes (line 362-364) without respecting the `onlyColorizedBrackets` flag, so Markdown's `colorizedBracketPairs: []` doesn't suppress the red 'unexpected bracket' decoration on orphan `)` characters.

## Change summary

In Markdown files, a closing parenthesis `)` in numbered list items (e.g. `1)`) is aggressively rendered in red as an 'unexpected bracket' despite `colorizedBracketPairs` being set to an empty array.

## Best solution

Guard the `AstNodeKind.UnexpectedClosingBracket` case in `collectBrackets` (bracketPairsTree.ts ~line 362) to check the `onlyColorizedBrackets` flag and skip pushing when the bracket type is not in the colorized set — mirroring how `AstNodeKind.Pair` already uses the `colorize` check.
