# Issue #175501 — Uncaught TypeError: Z.reject is not a function

[https://github.com/microsoft/vscode/issues/175501](https://github.com/microsoft/vscode/issues/175501)

## Summary

The root cause — `writer.reject` being set to `undefined!` after the executor settles — was fixed in commit d3e7ccc25f5, which replaces the `undefined!` assignments with no-op functions. The error can no longer occur on current main.

## Change summary

Calling `emitter.reject()` after the async iterable executor has settled throws `TypeError: Z.reject is not a function` because the writer's `reject` was set to `undefined!`.

## Best solution

Keep the shipped fix in commit d3e7ccc25f5; no further change needed. The `finally` block in `AsyncIterableObject` now assigns no-op functions instead of `undefined!`, so late cancellation calls are silently absorbed.
