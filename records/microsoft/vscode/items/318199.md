# Issue #318199 — Wrong GIT_EDITOR configuration breaks all tools

[https://github.com/microsoft/vscode/issues/318199](https://github.com/microsoft/vscode/issues/318199)

## Summary

The embedded quotes in GIT_EDITOR are intentional — git invokes the editor value via `sh -c`, so the quotes protect paths with spaces. The reporter's scripts consume the variable differently than git does, which is not a VS Code bug.

## Change summary

GIT_EDITOR env var contains embedded double quotes around the path, which breaks tools like cygpath that interpret the value as a literal path rather than a shell-evaluated string.

## Best solution

Close as designed. The quoting was deliberately added in commit 63b6b6adf2d to handle paths with spaces in the shell context that git uses. Scripts that need the raw path should strip the quotes themselves (e.g., `eval echo $GIT_EDITOR` or parameter expansion to remove quotes).
