# Issue #200865 — breakpoint decorations are not always hidden for folded regions

[https://github.com/microsoft/vscode/issues/200865](https://github.com/microsoft/vscode/issues/200865)

## Summary

Still reproduces on main: the fast-path heuristic in getDecorationsInRange incorrectly skips hidden-line filtering when word-wrapped lines inflate the view-line count beyond the folded model-line count.

## Change summary

Breakpoint decorations remain visible in folded regions when word wrap is enabled and a long wrapped line inflates the view-line count.

## Best solution

Fix the fast-path condition in `getDecorationsInRange` (viewModelLines.ts ~L927) to account for word-wrap-inflated view lines — e.g. by also checking whether any model line in the range is not visible, or by using projectedModelLineLineCounts to detect hidden areas accurately.
