# Issue #91108 — Add a command to file an issue (bug) with relevant info

[https://github.com/microsoft/vscode/issues/91108](https://github.com/microsoft/vscode/issues/91108)

## Summary

Valid niche feature request to bypass the issue reporter UI and open GitHub directly with system info. Still unimplemented on main — no command exists that skips the reporter form. Low traction (1 upvote) over 5+ years; leaving open for the community process.

## Change summary

Add a command/keybinding that opens a GitHub issue page directly with system info pre-filled, skipping the Issue Reporter form entirely.

## Best solution

If pursued, add a new command (e.g. `workbench.action.openNewGithubIssue`) in `src/vs/workbench/contrib/issue/common/issue.contribution.ts` that gathers system info and opens the GitHub new-issue URL with body pre-filled, accepting args like `includeExtensions` and `includeSystemInfo`. Given low demand, this could also be served by an extension using the existing `vscode.openIssueReporter` API plus the clipboard — consider whether it warrants core investment.
