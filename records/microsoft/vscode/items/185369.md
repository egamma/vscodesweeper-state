# Issue #185369 — Delayed pty host startup can cause terminal service connection state to stay connecting

[https://github.com/microsoft/vscode/issues/185369](https://github.com/microsoft/vscode/issues/185369)

## Summary

The bug remains unfixed on main. The focus-after-connection logic in terminalView.ts and terminalTabbedView.ts still guards on connectionState, and a terminal created during the Connecting phase may not get focused when toggled after connection completes. No commit addressing this specific scenario was found.

## Change summary

Toggle terminal (Ctrl+`) may not focus the terminal panel when the pty host had a delayed startup and the terminal was created during the connecting phase.

## Best solution

Fix the focus handoff in terminalView.ts and terminalTabbedView.ts so that terminals created while connectionState is Connecting receive focus once the state transitions to Connected — likely a small guard or deferred focus call in the `_setConnected` path or the toggle handler.
