# Issue #184324 — Git - View file diff sometimes takes 5 seconds longer than expect

[https://github.com/microsoft/vscode/issues/184324](https://github.com/microsoft/vscode/issues/184324)

## Summary

Still-open performance bug with active investigation by a team member. The intermittent ~5s delay when viewing git diffs has not been root-caused or fixed on main; multiple users confirm it persists through 2025. Keeping open for continued investigation.

## Change summary

Viewing file diffs from the Source Control panel intermittently takes exactly 5 seconds longer than expected, with the delay appearing in VS Code's process management rather than in git itself.

## Best solution

Root-cause the ~5s delay in the git extension's file-system provider flow. The reporter's traces show `git ls-tree` completes in milliseconds but VS Code reports ~5000ms — the suspicious `timeout(5000)` in `updateWhenIdleAndWait` (extensions/git/src/repository.ts ~line 3202) or a serialization bottleneck in operation queuing may be blocking the `getObjectDetails` call. A targeted CPU profile of the extension host during a slow diff open should pinpoint whether the delay is in process spawning, operation serialization, or the status-update throttle cycle.

## Evidence

- **git not the bottleneck:** Reporter's GIT_TRACE2 logs (comment 4) show git.exe completes in ~28ms, but VS Code reports 5189ms — the delay is in VS Code's process management, not git.
- **suspicious timeout:** extensions/git/src/repository.ts line 3202 has `await timeout(5000)` in `updateWhenIdleAndWait()` — exactly matching the ~5s delay pattern. This runs after status updates and could serialize/block subsequent operations.
- **regression window:** Reporter bisected to VS Code 1.42 (comment 12): the `git ls-tree` call was added alongside `git show` starting in 1.42, and the issue does not occur in 1.41.
- **ongoing reports:** Users continue reporting the issue in Nov 2024 and Jul 2025, confirming it persists on current builds.
- **no fix on main:** No commit in extensions/git/src/ addresses this issue; the code paths (fileSystemProvider.ts stat → getObjectDetails → lstree, and the 5000ms timeout in repository.ts) remain unchanged.
