# Issue #168720 — "Ctrl + d" doesn't respect "Match Case" and "Match Whole Word"

[https://github.com/microsoft/vscode/issues/168720](https://github.com/microsoft/vscode/issues/168720)

## Summary

The reported Ctrl+D behavior is intentional for the collapsed-selection path: VS Code deliberately enters a find-state disassociation mode and forces whole-word, case-sensitive matching for the word under the cursor.

## Change summary

Ctrl+D from a single collapsed cursor ignores the persisted Find widget Match Case and Match Whole Word settings and temporarily shows both enabled.

## Best solution

Keep the current collapsed-cursor multi-cursor behavior as designed; users who want Find-widget options to drive the match should make an explicit selection or start from the Find widget/search string path.

- **Proposed comment:**

  > Thanks for the report. This is the intended behavior for Ctrl+D when it starts from a single collapsed cursor: the editor expands to the word under the cursor and temporarily uses a disconnected multi-cursor find state with whole-word and case-sensitive matching. That path is covered by the existing Find state disassociation tests, so I think this should be closed as designed.

## Evidence

- **current code:** src/vs/editor/contrib/multicursor/browser/multicursor.ts:292-304 documents the collapsed-selection exception and sets wholeWord = true and matchCase = true when beginning with a single empty selection.
- **find widget override:** src/vs/editor/contrib/multicursor/browser/multicursor.ts:489-494 applies FindOptionOverride.True for wholeWord and matchCase while the disconnected multi-cursor session is active, matching the reporter's observation that the Find UI shows both toggles enabled.
- **test coverage:** src/vs/editor/contrib/multicursor/test/browser/multicursor.test.ts:505-536 has a 'Find state disassociation' test that expects Ctrl+D from a collapsed cursor on 'app' to select only whole-word, case-matching occurrences and skip 'apples', 'whatsapp', and 'App'.
- **history:** git blame attributes the collapsed-selection disassociation branch to commit d02a9cd78391205524889ce591e1a28d8928a2dd ('Introduce MultiCursorSession for easier state handling'), indicating this was an explicit design path rather than a recent regression.
