# Issue #193471 — "Go to definition" jumps to a hidden region in diff editor

[https://github.com/microsoft/vscode/issues/193471](https://github.com/microsoft/vscode/issues/193471)

## Summary

Fixed on main: commit c62947f67e4 (Dec 2023) added a cursor-position listener in the hide-unchanged-regions feature that automatically reveals hidden regions when the cursor lands inside them (e.g. via Go to Definition). A follow-up fix in 3363acb98bf (Feb 2024) addressed an edge case with the revealing logic.

## Change summary

"Go to Definition" in the diff editor jumps to the collapsed-region placeholder instead of revealing the hidden code containing the definition.

## Best solution

Keep the shipped fix (c62947f67e4, shipped in ~1.86); no further change needed. The `onDidChangeCursorPosition` handler in `hideUnchangedRegionsFeature.ts` now calls `ensureModifiedLineIsVisible`/`ensureOriginalLineIsVisible` so hidden regions expand when navigated into.
