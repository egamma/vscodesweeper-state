# Issue #202938 — [regression] `language-configuration` of embedded language not loading

[https://github.com/microsoft/vscode/issues/202938](https://github.com/microsoft/vscode/issues/202938)

## Summary

Still reproduces on main: asyncTokenization is still enabled by default, the fixing PR #244217 has not been merged, and the tracking issue #244225 remains open. The team is aware and investigating.

## Change summary

Embedded language bracket highlighting (and other language-configuration features) fail to load when async tokenization is enabled and the embedded region is off-screen at startup.

## Best solution

Resolve via the open PR https://github.com/microsoft/vscode/pull/244217 or by fixing the underlying race in the async tokenization pipeline tracked in https://github.com/microsoft/vscode/issues/244225. The root cause is that language-configuration data for embedded languages isn't propagated on initial async tokenization pass.
