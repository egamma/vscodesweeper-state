# Issue #268896 — Editor focus conflict after first layout

[https://github.com/microsoft/vscode/issues/268896](https://github.com/microsoft/vscode/issues/268896)

## Summary

Real bug, still open on main. The FocusTracker in nativeEditContextUtils.ts relies on blur events that never fire when two editors are focused in the same event loop iteration without window focus. PR #268897 proposes a fix but hasn't been merged.

## Change summary

When VS Code web loads with two editors in the default layout and the window lacks focus, both editors believe they are focused, causing keystrokes to go to the wrong editor.

## Best solution

Merge the approach from PR https://github.com/microsoft/vscode/pull/268897 — add a deferred re-check in FocusTracker (nativeEditContextUtils.ts) so that a second focus in the same tick causes the first tracker to realize it lost focus. The area owner (@aiday-mar) has been engaged and has asked for reproduction details, which the reporter provided.
