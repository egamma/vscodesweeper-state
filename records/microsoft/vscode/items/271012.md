# Issue #271012 — Copy and paste does not work between projects opened in dev containers

[https://github.com/microsoft/vscode/issues/271012](https://github.com/microsoft/vscode/issues/271012)

## Summary

Still reproduces on main: the explorer paste logic does `fileService.stat(fileToPaste)` on the source URI, which is bound to the originating container's remote connection and is unreachable from a second container window. No fix has landed. The issue is correctly labeled with `remote` and `file-explorer`; it sits at the boundary between core explorer clipboard handling and the remote/containers extension's file system provider.

## Change summary

Copy/paste files between two VS Code windows connected to different dev containers fails because the clipboard stores container-local URIs that are inaccessible from the other container's file system.

## Best solution

The fix likely involves either the explorer clipboard recording enough metadata (scheme, authority) to detect cross-remote-connection pastes and fall back to reading file contents through the originating connection, or the Dev Containers extension providing a cross-container file transfer mechanism. This may require coordination between the core explorer paste logic in `src/vs/workbench/contrib/files/browser/fileActions.ts` and the remote file system providers. Given the architectural complexity, this may ultimately be better addressed as a `*caused-by-extension` or upstream remote issue, but keeping open for a maintainer to decide.
