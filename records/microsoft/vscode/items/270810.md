# Issue #270810 — Terminal tool misuses double quotes in bash (eg. history expansion), especially when running python

[https://github.com/microsoft/vscode/issues/270810](https://github.com/microsoft/vscode/issues/270810)

## Summary

Still valid on main for bash: the zsh equivalent was fixed in bc33377584e (NO_BANG_HIST), but bash agent terminals have no `set +H` to disable history expansion — the reported `!` in double quotes issue persists for bash shells.

## Change summary

The terminal tool in Copilot Agent Mode triggers bash history expansion when commands contain `!` inside double quotes, causing errors and agent hangs.

## Best solution

Mirror the zsh fix for bash: add a `VSCODE_AGENT_BASH_FIXUPS` env var in `toolTerminalCreator.ts` when `isBash()`, and consume it in `shellIntegration-bash.sh` with `set +H` (disable history expansion). Additionally, add bash-specific steering text in `runInTerminalTool.ts` warning the model to prefer single quotes for strings containing `!`.
