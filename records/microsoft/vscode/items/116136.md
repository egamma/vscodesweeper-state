# Issue #116136 — Update close/dirty/pinned icons that are referenced by unicode characters

[https://github.com/microsoft/vscode/issues/116136](https://github.com/microsoft/vscode/issues/116136)

## Summary

Partially fixed on main — the original CSS offenders (tabs, quick access, select container, menubar) now use CSS variables for theming, but 3 TypeScript locations still hardcode the `\uea71` unicode character directly, including a TODO in decorationsService.ts referencing this issue.

## Change summary

Icons referenced by hardcoded unicode characters in CSS/TS are not replaceable by product icon themes.

## Best solution

Convert the 3 remaining hardcoded `\uea71` references in `decorationsService.ts`, `documentSymbolsTree.ts`, and `notebookOutline.ts` to use ThemeIcon or CSS variable patterns, completing the migration started for the CSS files.
