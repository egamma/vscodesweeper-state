# Issue #169923 — Merge conflict: save the file when choosing "accept all"

[https://github.com/microsoft/vscode/issues/169923](https://github.com/microsoft/vscode/issues/169923)

## Summary

Still reproduces on main: `acceptAllResources` in the merge-conflict extension applies edits via `WorkspaceEdit` but never saves the documents afterward, leaving them dirty and opened in tabs.

## Change summary

Multi-file 'Accept All Current/Incoming' from the SCM merge changes context menu resolves conflicts but leaves all files unsaved and opened in editor tabs.

## Best solution

After `vscode.workspace.applyEdit(edit)` resolves in `acceptAllResources`, save each modified document with `document.save()`. Optionally suppress opening them as visible tabs by not showing the text editors (the current code uses `openTextDocument` which doesn't show tabs — the tab opening likely comes from the SCM layer). The fix lives in `extensions/merge-conflict/src/commandHandler.ts`.
