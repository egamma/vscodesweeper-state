# Issue #269452 — Copilot Chat: get_changed_files sourceControlState ignored

[https://github.com/microsoft/vscode/issues/269452](https://github.com/microsoft/vscode/issues/269452)

## Summary

Valid bug with an open fix PR (https://github.com/microsoft/vscode/pull/324351). Multiple hard keep-open rules apply: team-authored (MEMBER) and milestone 'On Deck'. The root cause appears to be stale git repository state — the tool reads `repository.changes` without first refreshing the git index, so recently staged/unstaged files may not appear.

## Change summary

The `get_changed_files` tool returns no results when `sourceControlState` is set to specific values like `staged` or `unstaged`, because the git repository state is stale at invocation time.

## Best solution

Land the open PR https://github.com/microsoft/vscode/pull/324351, which refreshes the git repository state before reading changes. The filtering logic in `extensions/copilot/src/extension/tools/node/scmChangesTool.ts` (lines 82–104) correctly switches on `sourceControlState`, so the fix is ensuring `repository.changes` is up-to-date when the tool runs.
