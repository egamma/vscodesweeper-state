# Issue #305773 — Profiles: Infinite loop on "Profile already exists" dialog when clicking Replace or Cancel

[https://github.com/microsoft/vscode/issues/305773](https://github.com/microsoft/vscode/issues/305773)

## Summary

Still present on main: the profiles editor's extension-change event handler fires `_onDidChange` during background extension installs, which can re-trigger profile save/creation flows while a modal dialog is open, creating an inescapable loop. No fix commit found.

## Change summary

Creating a profile with an existing name while background extension tasks are running causes the 'Profile already exists' confirmation dialog to reappear in an infinite loop, freezing the UI.

## Best solution

Guard `getProfileToImport` (or its callers in the profiles editor model) against re-entrance during an active dialog/creation operation — likely a simple boolean flag in `userDataProfileImportExportService.ts` or `userDataProfilesEditorModel.ts` to prevent `_onDidChange`-driven `doSave()` from re-triggering `saveNewProfile` while a confirmation dialog is open.
