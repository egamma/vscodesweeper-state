# Issue #276577 — `New Window Tab` can result in swapped tab titles with macOS Native Tabs

[https://github.com/microsoft/vscode/issues/276577](https://github.com/microsoft/vscode/issues/276577)

## Summary

Still reproduces on main — the dual tab-addition code path (tabbingIdentifier + addTabbedWindow) is unchanged. The root cause involves Electron/macOS native tab behavior when both automatic (OS-level) and manual (addTabbedWindow) tab creation fire simultaneously, which is effectively undefined behavior. The existing `upstream` label is appropriate.

## Change summary

Using 'New Window Tab' with macOS native tabs enabled and 'Prefer tabs: Always' causes tab titles to swap between windows.

## Best solution

Avoid calling `addTabbedWindow()` when `tabbingIdentifier` is already set (the OS handles tab creation automatically). The guard at `src/vs/platform/windows/electron-main/windowsMainService.ts:1617` should check `!windowConfig?.nativeTabs` before calling `addTabbedWindow()`. The Positron fork's fix (https://github.com/posit-dev/positron/pull/11367) is a validated reference. An alternative is removing the `New Window Tab` command entirely and relying solely on the OS-level tab preference when `nativeTabs` is true. Either way the fix is in VS Code's code, but the underlying undefined behavior is in Electron's `addTabbedWindow` interacting with macOS's automatic tab management.
