# Issue #198841 — Diff editor loses view state and jumps around when I clear changes

[https://github.com/microsoft/vscode/issues/198841](https://github.com/microsoft/vscode/issues/198841)

## Summary

Team-authored bug report about the diff editor losing scroll position when reverting hunks. No fix commit referencing this issue was found on main; the issue appears still valid.

## Change summary

Diff editor jumps to the top and loses view state when reverting (clearing) changes, because the hidden-regions anchor resolves to line 1.

## Best solution

Fix the scroll anchor logic in hideUnchangedRegionsFeature.ts so that when a diff hunk is reverted and the unchanged region collapses, the editor preserves the current viewport position instead of snapping to line 1.
