## Change summary

Problem matchers should restore their diagnostics when a file is closed, distinguishing builder-reported problems from reconciler (live) problems to avoid duplication and timing issues.

## Best solution

Implement the reconciler/builder diagnostic distinction proposed in https://github.com/microsoft/vscode/issues/74524 — that API addition would let problem matchers safely restore diagnostics on file close without conflicting with live language-server diagnostics. The partial fix in df97bc32 helps but doesn't solve the fundamental coordination problem.
