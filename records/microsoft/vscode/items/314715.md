# Issue #314715 — OAuth state with literal `+` causes "State does not match" — VS Code emits raw `+` on /authorize but parses `+` as space on callback (asymmetric encoding)

[https://github.com/microsoft/vscode/issues/314715](https://github.com/microsoft/vscode/issues/314715)

## Summary

Confirmed on current main: `src/vs/workbench/api/node/loopbackServer.ts` generates OAuth state with standard base64 (includes `+`) but parses the callback with `URLSearchParams.get()` which decodes `+` as space, causing a mismatch ~50% of the time. The sibling GitHub auth extension already has a workaround for the same class of bug.

## Change summary

OAuth state with literal `+` characters fails round-trip because the loopback server generates standard base64 state but parses the callback URL using form-urlencoded semantics where `+` becomes space.

## Best solution

Change line 48 of `src/vs/workbench/api/node/loopbackServer.ts` from `randomBytes(16).toString('base64')` to `randomBytes(16).toString('base64url')` (Node's built-in base64url encoding, RFC 4648 §5). This eliminates `+` and `/` from the state, matching what PKCE code verifiers already use. Alternatively, apply the same `.replace(/ /g, '+')` workaround the GitHub authentication extension uses at line 106/118 of `extensions/github-authentication/src/node/authServer.ts`.
