# Issue #71813 — [css] highlighting not working for some transition properties

[https://github.com/microsoft/vscode/issues/71813](https://github.com/microsoft/vscode/issues/71813)

## Summary

Still reproduces on main: the CSS TextMate grammar has no rule to highlight property names when they appear as values of `transition:` or `transition-property:`. The grammar only matches them in a property-declaration context. Team member aeschli confirmed it needs a grammar fix.

## Change summary

CSS property names (e.g. `background-color`, `border-bottom`) are not syntax-highlighted when used as values in `transition:` shorthand declarations.

## Best solution

Add a value-context pattern in the CSS TextMate grammar (sourced from microsoft/vscode-css) that matches known animatable property names within `transition` and `transition-property` value positions, scoping them as `support.type.property-name.css` or a similar token.
