# Issue #71813 — [css] highlighting not working for some transition properties

[https://github.com/microsoft/vscode/issues/71813](https://github.com/microsoft/vscode/issues/71813)

## Summary

Still reproduces on current main — the CSS TextMate grammar does not highlight property names (e.g. `background-color`) when they appear as values inside `transition:`. A team member confirmed this belongs in the grammar and deferred to the upstream language-css repo.

## Change summary

CSS property names used as values in the `transition` shorthand are not syntax-highlighted.

## Best solution

The `property-keywords` repository rule in `extensions/css/syntaxes/css.tmLanguage.json` needs a new pattern (or the `property-values` rule needs a context-aware sub-pattern) that recognizes animatable CSS property names when they appear in value position after `transition:` or `transition-property:`. Alternatively, upstream at https://github.com/nicolo-ribaudo/language-css (which replaced octref/language-css) should add this pattern.
