# Issue #232298 — Large tooltip when not tokenized on long lines makes code hard to read

[https://github.com/microsoft/vscode/issues/232298](https://github.com/microsoft/vscode/issues/232298)

## Summary

Already addressed on main: the long-line tokenization warning hover now has a dedicated suppression setting and an inline Don't Show Again action, so this can be closed as completed.

## Change summary

The reporter wants the long-line tokenization warning hover to stop obscuring code, either by delaying it substantially or making it dismissible after one showing.

## Best solution

Keep the shipped `editor.hover.showLongLineWarning` setting and Don't Show Again command added for long-line warning hovers; no further core change is needed for this report unless users still want a separate hover-delay tuning request.
