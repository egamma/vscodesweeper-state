## Summary

Still reproduces on main: the explorer's `.gitignore` filter only discovers ignore files encountered during tree traversal inside the workspace root, so parent `.gitignore` files between the workspace folder and the git repo root are never loaded. The bug is already triaged with `bug` + `help wanted` and remains a valid open issue.

## Change summary

When `explorer.excludeGitIgnore` is enabled and the workspace root is a subdirectory of the git repo, `.gitignore` rules from ancestor directories (up to the repo root) are not applied â€” only `.gitignore` files within the workspace tree are processed.

## Best solution

Extend `FilesFilter` to walk up from each workspace folder to the git repo root on initialization, loading all ancestor `.gitignore` files into the `ignoreTreesPerRoot` tree. The fix belongs in `src/vs/workbench/contrib/files/browser/views/explorerViewer.ts` around the `processIgnoreFile` / configuration-change handler. Related issue https://github.com/microsoft/vscode/issues/172482 covers the same root cause for multi-root workspaces and could be fixed together.
