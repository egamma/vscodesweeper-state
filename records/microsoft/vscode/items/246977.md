# Issue #246977 — Selection to the left after double clicking

[https://github.com/microsoft/vscode/issues/246977](https://github.com/microsoft/vscode/issues/246977)

## Summary

Valid bug: after double-clicking a word, Shift+Alt+Left (cursorWordLeftSelect) deselects the word instead of extending leftward, because the command anchors at the word's start position rather than treating the entire double-clicked word as a sticky anchor unit — standard macOS behavior. Already correctly labeled.

## Change summary

Shift+Alt+Left after double-clicking a word deselects the word instead of extending the selection leftward as macOS native text editing does.

## Best solution

The cursorWordLeftSelect command (in src/vs/editor/contrib/wordOperations/browser/wordOperations.ts) should detect when the current selection was created by a word-select gesture and use the full word range as the anchor rather than just selectionStart. This likely requires the selection or cursor state to carry provenance information about how the selection was made (double-click vs keyboard). Alternatively, a setting could control anchor behavior.

## Evidence

- **source trace:** src/vs/editor/contrib/wordOperations/browser/wordOperations.ts lines 66-74: _moveTo() with inSelectionMode uses from.selectionStartColumn as anchor, which is the start of the double-clicked word. Moving position leftward past this anchor deselects the word.
- **macOS convention:** macOS native text editing treats a double-clicked word as an atomic anchor unit for subsequent shift-arrow extensions, keeping the word selected while extending in either direction.
