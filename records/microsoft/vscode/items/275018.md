# Issue #275018 — Changing default profile to bash on macOS does not update run_in_terminal prompt

[https://github.com/microsoft/vscode/issues/275018](https://github.com/microsoft/vscode/issues/275018)

## Summary

Still open on main — the run_in_terminal tool description is not re-registered when the default terminal profile changes, and a separate caching issue in vscode-copilot-chat prevents updated descriptions from being picked up. Team-authored, milestone On Deck; keeping open.

## Change summary

Changing the default terminal profile (e.g. zsh→bash) does not update the run_in_terminal tool's prompt description to reflect the new shell.

## Best solution

Two fixes needed: (1) in `terminal.chatAgentTools.contribution.ts`, add `terminal.integrated.defaultProfile.*` and `chat.tools.terminal.terminalProfile.*` to the config-change listener that triggers `_registerRunInTerminalTool()`; (2) on the extension side (vscode-copilot-chat), invalidate the tool cache when tool data changes — either via an `onDidChangeTools` event or by breaking object identity on update in `extHostLanguageModelTools.ts`.

## Evidence

- **missing config listener:** src/vs/workbench/contrib/terminalContrib/chatAgentTools/browser/terminal.chatAgentTools.contribution.ts lines 150-171 listen for sandbox config changes but do not include terminal profile settings, so changing the default profile does not trigger tool re-registration.
- **shell-specific descriptions:** src/vs/workbench/contrib/terminalContrib/chatAgentTools/browser/tools/runInTerminalTool.ts lines 400-408 select shell-specific descriptions (zsh/bash/fish/pwsh) based on the resolved profile at registration time; stale registration means stale description.
- **extension caching issue:** Comment by roblourens: vscode-copilot-chat caches tools by object identity (toolsService.ts line 25), and extHostLanguageModelTools.ts updates tool data without changing object identity (lines 56-58), so even if re-registered, the cache may not invalidate.
