# Issue #170353 — Incorrect prioritization of filename suggestions in quick open dialog (Ctrl+P)

[https://github.com/microsoft/vscode/issues/170353](https://github.com/microsoft/vscode/issues/170353)

## Summary

This is a specific instance of the long-standing fuzzy scorer ranking issue tracked by the open canonical issue #27317. The root cause — beginning-of-word bonus outweighing contiguous substring matches — is the same underlying problem. bpasero confirmed the mechanism and noted prior duplicates (#56667, #69648). Proposing a duplicate close to consolidate discussion.

## Change summary

Quick open (Ctrl+P) ranks scattered character matches above contiguous substring matches when the first scattered character hits the beginning of a path segment.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/27317 — any improvement to fuzzy scorer ranking that properly weights contiguous substring matches over scattered beginning-of-word hits would resolve this report.
