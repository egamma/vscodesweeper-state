# Issue #249477 — vscode.terminal scheme for lsp REPL should conform to all OS

[https://github.com/microsoft/vscode/issues/249477](https://github.com/microsoft/vscode/issues/249477)

## Summary

Team-authored debt tracking item for making the LSP REPL terminal URI scheme OS-generic once Python shell integration lands on Windows and broader language support is added. Keeping open per policy.

## Change summary

The vscode.terminal URI scheme used for LSP REPL virtual documents currently crafts paths assuming Mac/Linux; it should be made OS-agnostic for Windows support and multi-language generality.

## Best solution

When Python shell integration lands on Windows and the LSP REPL generalizes beyond Python, update the URI/path construction to use OS-appropriate separators or a platform-neutral scheme. The relevant code likely lives in the Python extension or the terminal suggest contribution.
