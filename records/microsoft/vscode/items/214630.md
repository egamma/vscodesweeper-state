# Issue #214630 — Node unconditionally uses AVX512 instructions  (illegal instruction starting Extension Host agent)

[https://github.com/microsoft/vscode/issues/214630](https://github.com/microsoft/vscode/issues/214630)

## Summary

The upstream root cause (aklomp/base64 not checking OS-level AVX512 support) has been fixed and merged upstream (aklomp/base64#143, nodejs/node#53426 both closed). VS Code has since upgraded from Electron 29 to Electron 42 (Node 24.15.0), which includes the corrected base64 library. This should be resolved.

## Change summary

Node.js unconditionally used AVX512 instructions via its vendored aklomp/base64 library even when AVX512 was disabled at the OS level, causing an illegal instruction crash in the extension host.

## Best solution

The fix shipped via the Electron 42 upgrade (Node 24.15.0). No further VS Code change is needed. Close as resolved upstream, and direct anyone still seeing AVX512 crashes on the latest stable to open a new issue.
