# Issue #257621 — Simplify wrapping an npm MCP server package in VS Code Extension

[https://github.com/microsoft/vscode/issues/257621](https://github.com/microsoft/vscode/issues/257621)

## Summary

Team-authored feature request for simplifying MCP server extension authoring — keeping open per policy. The request is for a streamlined API so extension authors can wrap an npm MCP server package without manual resolution boilerplate.

## Change summary

Allow extension authors to provide just an npm package name in the MCP Server Provider API and have VS Code auto-resolve dependencies, simplifying MCP server extension development.

## Best solution

Extend the MCP Server Provider API to accept an npm package reference and handle resolution/installation internally, reducing boilerplate for extension authors wrapping npm-based MCP servers. Related work tracked in https://github.com/microsoft/vscode/issues/262904.
