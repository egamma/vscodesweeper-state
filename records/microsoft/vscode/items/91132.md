# Issue #91132 — Cannot drag/drop file from VS Code to other apps due to invalid data object

[https://github.com/microsoft/vscode/issues/91132](https://github.com/microsoft/vscode/issues/91132)

## Summary

Team-authored issue on the On Deck milestone requesting proper native file drag/drop (CF_HDROP/FileDrop) to external apps. The gap remains on current main — VS Code sets custom and text data transfers but does not produce a valid native file drop payload. Hard keep-open: MEMBER author + real milestone.

## Change summary

Dragging files from VS Code's Explorer to other native apps (Visual Studio, Outlook, Excel, Finder targets) fails because the drag data object does not provide a valid CF_HDROP/FileDrop format.

## Best solution

Extend the drag data in `src/vs/workbench/browser/dnd.ts` (`fillEditorsDragData`) and/or the Electron layer to produce a proper native file-system drag payload (CF_HDROP on Windows, `NSFilenamesPboardType`/file URLs on macOS). This likely requires Electron-level support (e.g. `webContents.startDrag` or custom native module) since the web DataTransfer API alone cannot produce OS-native file drop formats. Coordinate with Electron upstream if needed.
