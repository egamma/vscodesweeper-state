# Issue #267416 — VS code should auto recover from previous failed start

[https://github.com/microsoft/vscode/issues/267416](https://github.com/microsoft/vscode/issues/267416)

## Summary

Team-authored issue (MEMBER) — must keep open for maintainer judgment. Reports repeated loss of `out/cli.js` on Windows requiring reinstallation, and requests automatic self-repair. The root cause (why the file disappears) is unclear and may be external (antivirus, disk corruption, update race), but the ask for resilience is reasonable.

## Change summary

VS Code fails to start when `out/cli.js` is missing and offers no self-repair; the user must reinstall.

## Best solution

Investigate why `cli.js` disappears repeatedly (antivirus exclusion? update issue?). Separately, consider a startup integrity check that detects missing critical files and triggers a repair/reinstall flow rather than a raw Node.js crash. The startup entry point lives in `src/vs/code/` (Electron main process); a file-existence guard before `require('cli.js')` could present a user-friendly error with a repair action.
