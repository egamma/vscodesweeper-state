# Issue #261886 — Git: Apply Stash picker, worktree is shown when closed

[https://github.com/microsoft/vscode/issues/261886](https://github.com/microsoft/vscode/issues/261886)

## Summary

Valid bug, still present on main: all stash commands use `{ repository: true }` without a `repositoryFilter`, so worktrees appear in the repository picker. Milestone is 'On Deck', so this stays open for the area owner.

## Change summary

Git stash commands (apply, pop, drop, etc.) show worktree repositories in the picker even when those worktrees are not relevant to the operation.

## Best solution

Add `repositoryFilter: ['repository', 'submodule']` to the `@command` decorators for all stash commands in `extensions/git/src/commands.ts` (lines ~4542–4692), mirroring the pattern already used by worktree-management commands like `git.createWorktree` and `git.migrateWorktreeChanges`.
