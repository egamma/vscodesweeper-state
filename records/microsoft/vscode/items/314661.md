# Issue #314661 — Final agent text hidden in tool call and double scrollbar in chat

[https://github.com/microsoft/vscode/issues/314661](https://github.com/microsoft/vscode/issues/314661)

## Summary

Team-authored bug reporting two distinct chat rendering issues: (1) conclusion text hidden inside a collapsed tool-call expando, and (2) nested scrollbar stealing scroll events. The assignee (justschen) has acknowledged the issue and landed several related fixes to the thinking content part since then, but neither problem appears fully resolved — the fixes target single-tool-call promotion rather than the conclusion-text grouping or double-scrollbar behavior described here.

## Change summary

After an agent completes a file edit, the final conclusion markdown is incorrectly grouped inside the tool-call expando (hidden behind a collapse), and the chat view shows a double scrollbar where the inner one steals wheel events.

## Best solution

Problem 1 likely lives in `chatThinkingContentPart.ts` — the content-part assembly needs to ensure conclusion markdown following a tool call is rendered at the top level, not nested inside the thinking/tool container. Problem 2 is a CSS/scroll-containment issue in the chat list or thinking container (`chatThinkingContent.css` / `chatListRenderer.ts`) — the inner scrollable region needs `overscroll-behavior: contain` or should not be independently scrollable when its content fits. justschen is already investigating; this should stay open for their resolution.
