# Issue #311784 — Agents Insiders cannot start Azure DevOps MCP server

[https://github.com/microsoft/vscode/issues/311784](https://github.com/microsoft/vscode/issues/311784)

## Summary

Team-authored issue; the MCP server startup path uses the configuration resolver which throws when encountering `${input:...}` variables outside debug/task configs. The inputs section in mcp.json is not wired into the resolver. No fix found on main.

## Change summary

MCP servers with `promptString` inputs in mcp.json cannot start because the configuration resolver does not support resolving input variables in that context.

## Best solution

Extend the MCP server launch path to resolve `${input:...}` variables via the configuration resolver service's interactive resolution, or implement a dedicated input-prompting flow for MCP server configs. Likely involves `src/vs/workbench/services/configurationResolver/browser/baseConfigurationResolverService.ts` and the MCP server startup code.
