# Issue #261630 — `npm.scriptRunner` auto-detection fails in subdirectories; defaults to npm instead of pnpm/yarn/bun

[https://github.com/microsoft/vscode/issues/261630](https://github.com/microsoft/vscode/issues/261630)

## Summary

Verified duplicate of #172865 — both report the same root cause: `getScriptRunner`/`getPackageManager` pass the workspace folder URI to `findPreferredPM` instead of the subdirectory containing the package.json, so lock-file detection only works at the workspace root.

## Change summary

When `npm.scriptRunner` or `npm.packageManager` is set to "auto", auto-detection checks only the workspace root for lock files, ignoring lock files in subdirectories where the actual package.json lives.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/172865 which owns the same underlying defect. The fix is to pass the package.json's directory (the `cwd` derived from `packageJsonUri`) to `detectPackageManager` instead of `folder.uri`. PR https://github.com/microsoft/vscode/pull/302916 is open attempting this fix.
