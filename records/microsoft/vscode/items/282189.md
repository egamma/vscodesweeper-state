# Issue #282189 — InlineCompletion Not Triggered on accepting when "command" option provided to InlineCompletionItem.

[https://github.com/microsoft/vscode/issues/282189](https://github.com/microsoft/vscode/issues/282189)

## Summary

Still reproduces on main: when an InlineCompletionItem carries a `command`, the accept flow calls `stop()` before executing the command (clearing the model), and `followUpTrigger` is only set for alternative actions — so no re-trigger occurs after the command. The workaround (setTimeout + explicit trigger) confirms the timing issue.

## Change summary

Accepting an InlineCompletionItem that has a `command` property does not re-trigger `provideInlineCompletionItems`, whereas accepting one without a command does.

## Best solution

In the `accept` method of `inlineCompletionsModel.ts`, after executing `completion.command`, call `this.trigger(undefined)` (similar to the `followUpTrigger` path on line 1005-1007) so that providers are re-invoked. The trigger should happen after the command completes, ensuring the text model is in its final state.
