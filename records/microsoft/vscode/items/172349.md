# Issue #172349 — Tabs do not align to tab stops when emoji are used

[https://github.com/microsoft/vscode/issues/172349](https://github.com/microsoft/vscode/issues/172349)

## Summary

Still reproduces on main: the view line renderer tracks visibleColumn using only isFullWidthCharacter but not isEmojiImprecise, causing tab stop miscalculation when emoji precede a tab character.

## Change summary

Tabs following emoji characters render with incorrect width because the renderer undercounts the visual column occupied by emoji.

## Best solution

In src/vs/editor/common/viewLayout/viewLineRenderer.ts, add isEmojiImprecise checks alongside the existing isFullWidthCharacter checks (lines ~865 and ~1141) so that emoji increment visibleColumn/tmpIndent by 2, matching the logic already in cursorColumns.ts _nextVisibleColumn.
