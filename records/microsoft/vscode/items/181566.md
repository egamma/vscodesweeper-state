# Issue #181566 — 'Switch Output' dropdown closes when a new item is added

[https://github.com/microsoft/vscode/issues/181566](https://github.com/microsoft/vscode/issues/181566)

## Summary

Still reproduces on main: the toolbar's `setActions` unconditionally calls `clear()`, which disposes all action view items — including any open dropdown — whenever composite actions change (e.g. a new output channel registers). No diffing or reuse logic has been added since the issue was filed.

## Change summary

The Output panel's 'Switch Output' dropdown closes whenever a new output channel is registered, because the toolbar disposes and recreates all action view items on every action-set update.

## Best solution

Introduce diffing in `Toolbar.setActions` (or `ActionBar.clear`/`push`) so that unchanged action view items are reused rather than disposed and recreated. The fix is primarily in `src/vs/base/browser/ui/toolbar/toolbar.ts` and `src/vs/base/browser/ui/actionbar/actionbar.ts`. As @joaomoreno noted, the actionbar needs to understand the diff in its action collection to minimize DOM operations.
