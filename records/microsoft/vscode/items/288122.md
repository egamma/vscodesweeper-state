# Issue #288122 — Session Initializing… message won't dissappear from VSC UI for abandoned job

[https://github.com/microsoft/vscode/issues/288122](https://github.com/microsoft/vscode/issues/288122)

## Summary

Team-authored issue (MEMBER) — must keep open. The bug is traceable in source: for v1 (Jobs API) sessions, when the backend still reports `in_progress`/`queued` but the PR has been closed, the filtering at line ~1393–1399 correctly removes CLOSED/MERGED PR-backed entries from the session list, but the active-session polling loop (line ~1270) may have already registered the session ID, and the `createResponseTurn` path (line ~360–362) renders a permanent 'Session is initializing...' placeholder for any `in_progress`/`queued` session without logs. The session state may never transition server-side for an abandoned job, leaving a ghost entry.

## Change summary

A Copilot cloud agent session whose PR was closed/abandoned continues showing 'Session Initializing…' in the Chat Sessions UI indefinitely.

## Best solution

The cloud sessions provider should detect when a session's associated PR is closed or the job is abandoned and either remove the entry from the UI or update its status to a terminal state. The fix likely touches `copilotCloudSessionsProvider.ts` (the polling loop and/or session-status derivation) and possibly `copilotCloudSessionContentBuilder.ts` (the placeholder logic). The server-side session state should also be checked — if the backend never transitions abandoned jobs, the client needs to reconcile PR state with session state.
