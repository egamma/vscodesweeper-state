# Issue #139845 — VS Code fails to open files from relative paths

[https://github.com/microsoft/vscode/issues/139845](https://github.com/microsoft/vscode/issues/139845)

## Summary

Still reproduces on main: the fuzzy scorer's normalizeQuery does not strip `..` or `.` path segments, so 'Go to File' fails to match files when the query contains relative directory traversals. The stat-based fallback in getRelativePathFileResults handles some cases (internal `../`) but not leading `../` that goes above the workspace root.

## Change summary

VS Code's 'Go to File' (Ctrl+P) and terminal output link handling fail to open files when paths contain `../` relative directory segments.

## Best solution

Add path normalization (strip `..` and `.` segments) in `normalizeQuery` in `src/vs/base/common/fuzzyScorer.ts` as bpasero suggested in comment 13. Additionally, strip leading `../` prefixes from the query before fuzzy matching (similar to the terminal link workaround). This would allow both the fuzzy scorer and the stat-based resolution to find the file.
