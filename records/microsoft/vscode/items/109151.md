# Issue #109151 — Text cursor indicator displays incorrect

[https://github.com/microsoft/vscode/issues/109151](https://github.com/microsoft/vscode/issues/109151)

## Summary

Partially addressed by commit 7cc154201cc (positions textarea on cursor when accessibility support is enabled), but the Windows text cursor indicator can be enabled independently of VS Code's screen reader mode — the bug likely persists for users who have the indicator on without enabling screen reader mode in VS Code. Keeping open as the team explicitly placed it in Backlog.

## Change summary

Windows text cursor indicator appears at the wrong position in the VS Code editor because the hidden textarea is not placed at the visual cursor location.

## Best solution

Extend the textarea-on-cursor positioning (currently gated on `platform.isMacintosh || accessibilitySupport === Enabled`) to also activate when the Windows text cursor indicator is detected or when on Windows generally, in `src/vs/editor/browser/controller/editContext/textArea/textAreaEditContext.ts` around line 793.
