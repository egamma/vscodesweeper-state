# Issue #312802 — [Error] unhandlederror-Timed out waiting for authentication provider 'microsoft' to register.

[https://github.com/microsoft/vscode/issues/312802](https://github.com/microsoft/vscode/issues/312802)

## Summary

Still reproduces on current main — the 5-second raceTimeout in tryActivateProvider (line 481) remains unchanged and the TODO acknowledges the architectural fragility. Team members have confirmed it's a real issue affecting 382+ users per stable release, not a user error.

## Change summary

Edit session resume throws an unhandled timeout error when the 'microsoft' auth provider takes >5 seconds to activate, causing a 383× spike in error telemetry on stable.

## Best solution

Replace the hard 5-second timeout with a more robust activation-completion signal (the existing TODO at line 479 acknowledges this). Alternatively, the editSessions caller should catch the timeout gracefully so it doesn't surface as an unhandled error in telemetry. The fix likely touches both `authenticationService.ts` (timeout design) and possibly `editSessionsStorageService.ts` (error handling).
