# Issue #64027 — Git - "Stage selected ranges" sometimes stages a revert of changes in the previous commit

[https://github.com/microsoft/vscode/issues/64027](https://github.com/microsoft/vscode/issues/64027)

## Summary

Confirmed bug with an open fix PR (#326397) — keeping open until that PR lands or is closed.

## Change summary

Staging selected ranges after a prior commit sometimes stages a revert of the previous changes because the index base is stale.

## Best solution

Land or iterate on https://github.com/microsoft/vscode/pull/326397 which re-reads the index base when staging/reverting/unstaging ranges, directly addressing the stale-index root cause joaomoreno identified.
