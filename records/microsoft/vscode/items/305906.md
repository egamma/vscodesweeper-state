# Issue #305906 — Debug adapter crashing extension host

[https://github.com/microsoft/vscode/issues/305906](https://github.com/microsoft/vscode/issues/305906)

## Summary

A real but low-impact race condition: when a debug adapter's extension host terminates mid-session, in-flight messages hit `$acceptDAMessage` which throws because the adapter handle is already gone. Connor4312 confirmed this is known raciness. The fix would be making `$acceptDAMessage` use the same safe-access pattern as `$acceptDAError`/`$acceptDAExit`.

## Change summary

Debug adapter in-flight messages crash the extension host log with 'Invalid debug adapter' errors when the adapter disconnects during a race window.

## Best solution

Apply the same defensive pattern used in `$acceptDAError` and `$acceptDAExit` (lines 394–401 of `mainThreadDebugService.ts`) to `$acceptDAMessage` at line 391 — use `this._debugAdapters.get(handle)?.acceptMessage(...)` instead of the throwing `getDebugAdapter(handle)`. This silences the noisy post-close errors without masking real bugs.
