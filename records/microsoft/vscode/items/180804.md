# Issue #180804 — Can we save a couple more pixels in the glyph margin

[https://github.com/microsoft/vscode/issues/180804](https://github.com/microsoft/vscode/issues/180804)

## Summary

Team-authored polish request still open on main — the glyph margin width is still computed as `lineHeight * laneCount` with no inter-lane pixel saving. Related commits (651bd249f9f, 2fa219d00c3, f5952f92c0f) improved glyph layout and multi-lane handling but did not reduce the spacing between two glyph margin icons.

## Change summary

When two glyph margin lanes are shown, the combined width is larger than necessary — shaving a couple of pixels between the icons would tighten the gutter.

## Best solution

Reduce the per-lane width slightly (or add a negative inter-lane gap) in `src/vs/editor/common/config/editorOptions.ts` around line 2921 so that two-lane glyph margins are narrower without affecting single-lane appearance.
