# Issue #264716 — Selecting text in a long thinking block changes the scroll position

[https://github.com/microsoft/vscode/issues/264716](https://github.com/microsoft/vscode/issues/264716)

Team-authored bug that remains open. Several thinking-scroll fixes have landed since filing (e.g. commits 04265b17f2d, 66fe91254e6, a3d8dbf9b70) but none specifically address the mousedown-triggers-scroll-to-top-of-element issue described here. The bug likely still reproduces.

## Change summary

Clicking to select text in a long thinking block scrolls the user to the top of that block, making text selection land in the wrong place.

## Best solution

The fix likely lives in the list/tree focus-on-click behavior interacting with the thinking content part. When a thinking block row receives focus via mousedown, the list widget's default reveal-on-focus scrolls to the element's top. A targeted fix would suppress the default focus-scroll for mousedown events inside the thinking container, or use `preventScroll` on the focus call. Likely files: `src/vs/workbench/contrib/chat/browser/widget/chatListWidget.ts` and `src/vs/workbench/contrib/chat/browser/widget/chatContentParts/chatThinkingContentPart.ts`.
