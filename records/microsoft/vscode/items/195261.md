## Summary

Still reproduces on main: the runtime extensions editor CSS lacks word-wrap/overflow-wrap on the `.msg` content area, causing text to overflow or be clipped rather than wrapping naturally.

## Change summary

The Running Extensions editor does not wrap long content lines, causing overflow or clipping instead of wrapping.

## Best solution

Add `overflow-wrap: break-word` or `word-wrap: break-word` to `.runtime-extensions-editor .extension .desc > .msg` in `src/vs/workbench/contrib/extensions/browser/media/runtimeExtensionsEditor.css`, and potentially adjust the header-container's `overflow: hidden` to allow wrapping.
