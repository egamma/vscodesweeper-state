# Issue #176260 — l is not iterable

[https://github.com/microsoft/vscode/issues/176260](https://github.com/microsoft/vscode/issues/176260)

## Summary

Still present on main: a race between visible-range computation and line-count changes can leave `indents[lineIndex]` as `undefined`, causing the 'not iterable' TypeError. Team-authored error-telemetry issue — keeping open.

## Change summary

TypeError 'l is not iterable' thrown in indent guides rendering when the document line count changes between visible range calculation and guide lookup.

## Best solution

Add a null/undefined guard for `indent` at line 130 in `src/vs/editor/browser/viewParts/indentGuides/indentGuides.ts` (e.g. `if (!indent) { continue; }`), preventing the crash when visible range and model line count are momentarily out of sync.
