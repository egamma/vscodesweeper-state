# Issue #201575 — vscode open recent <ctrl+r> command dont support fuzzy search path

[https://github.com/microsoft/vscode/issues/201575](https://github.com/microsoft/vscode/issues/201575)

## Summary

Real usability gap — the Open Recent quick pick uses `matchesFuzzyIconAware` which matches label and description independently, so a path-spanning query like "dir/hello" won't match. The more advanced Quick Open (Ctrl+P) scorer handles this via `scoreItemFuzzy` with path-aware matching. This is a polish improvement to bring path-aware scoring to the Open Recent picker.

## Change summary

Typing a path with separators (e.g. "dir/hello") in the Open Recent (Ctrl+R) picker fails to match because label and description are fuzzy-matched independently, not as a combined path.

## Best solution

Enhance the Open Recent quick pick's filter logic in `src/vs/platform/quickinput/browser/quickInputList.ts` (around line 1446) to concatenate label+description when the query contains a path separator, or switch to the path-aware `scoreItemFuzzy` from `src/vs/base/common/fuzzyScorer.ts`. Alternatively, consider this a limitation of the simple quick pick and recommend users use Quick Open (Ctrl+P) for path-based searches.
