# Issue #228421 — Make URI Handlers available for vscode.dev

[https://github.com/microsoft/vscode/issues/228421](https://github.com/microsoft/vscode/issues/228421)

## Summary

Valid feature request with strong community support (31 upvotes, promoted to Backlog). The browser URL service infrastructure exists in `src/vs/workbench/services/url/browser/urlService.ts` but requires an `IURLCallbackProvider` from the vscode.dev host — without it, `create()` returns `unsupported://`. No implementation of this provider for vscode.dev exists on current main.

## Change summary

Enable `window.registerUriHandler` to work in vscode.dev so extensions can handle custom URI links (e.g. `vscode.dev/redirect?...`) the same way desktop VS Code handles `vscode://` links.

## Best solution

Implement an `IURLCallbackProvider` for the vscode.dev hosting layer that translates browser-navigated URLs (e.g. via query parameters or hash fragments) into the `onCallback` event, and wire it into the `BrowserWorkbenchEnvironmentService` options. This likely involves both the vscode.dev server-side routing and the `BrowserURLService` consumer in `src/vs/workbench/services/url/browser/urlService.ts`.
