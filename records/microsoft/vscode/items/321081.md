# Issue #321081 — "Open Quick Access" panel directly into "commands" or add setting to choose default behavior

[https://github.com/microsoft/vscode/issues/321081](https://github.com/microsoft/vscode/issues/321081)

## Summary

Valid feature request: the unified quick access widget (agents/sessions experiment) hardcodes the Sessions tab as the first/default tab. The user wants a setting to choose which tab opens by default. No existing issue covers this specific request.

## Change summary

Add a setting to choose which tab (Sessions, Commands, or Files) the unified quick access panel defaults to when opened.

## Best solution

Add a configuration setting (e.g. `chat.unifiedQuickAccess.defaultTab`) that selects the initial tab in `UnifiedQuickAccess.show()`. The implementation is localized to `src/vs/workbench/contrib/chat/browser/agentSessions/experiments/unifiedQuickAccess.ts` — the `show()` method already accepts an `initialTabId` parameter; a setting would wire that up.
