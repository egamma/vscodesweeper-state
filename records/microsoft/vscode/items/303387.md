# Issue #303387 — Copilot Chat OOM crash (20+ GB memory leak) on large structured responses — reproducible across models

[https://github.com/microsoft/vscode/issues/303387](https://github.com/microsoft/vscode/issues/303387)

## Summary

Well-documented renderer OOM bug when Copilot Chat generates large structured responses. The issue is untriaged after 119 days and needs routing to the copilot/chat area. It is related to but distinct from the broader #294050 (different reproduction path: large tabular chat output vs. repeated keystrokes), so it should be routed rather than closed as a duplicate.

## Change summary

Copilot Chat causes the window renderer process to grow from ~350 MB to 20+ GB and OOM-crash when generating large structured tabular responses covering many files/directories.

## Best solution

Route to the copilot/chat team to investigate whether the chat markdown rendering pipeline has unbounded memory allocation when rendering large structured responses (tables, file lists). The fix in https://github.com/microsoft/vscode/issues/300339 addressed event listener leaks in tool invocation widgets, but this issue's reproduction path (large tabular output, no tool invocations) suggests a separate leak site — likely in the markdown/table rendering or DOM accumulation in the chat list renderer. The team should capture a heap snapshot mid-render of a large structured response to identify the retention path.
