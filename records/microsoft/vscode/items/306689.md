# Issue #306689 — Profile icon selection does not have development platforms and languages.

[https://github.com/microsoft/vscode/issues/306689](https://github.com/microsoft/vscode/issues/306689)

## Summary

Valid feature request to expand the built-in profile icon selection with more language/platform icons. The current list at `userDataProfileIcons.ts` has only 4 language icons (ruby, code, coffee, snake). No duplicate found — #194276 is about extension-contributed icons, which is related but distinct.

## Change summary

Add more programming language and platform icons (e.g. Java, Go, Rust, C++, etc.) to the profile icon picker.

## Best solution

The icon picker draws from codicons registered in VS Code. The practical constraint is that new language codicons would need to be added to the codicon font first, then referenced in `src/vs/workbench/services/userDataProfile/common/userDataProfileIcons.ts`. Alternatively, #194276 (extension-contributed icons in the picker) would solve this more broadly. A maintainer should decide whether to expand the built-in set or rely on the extension-icon approach.
