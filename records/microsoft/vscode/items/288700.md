# Issue #288700 — Double quote replacement in PS1 files can break the code in that file.

[https://github.com/microsoft/vscode/issues/288700](https://github.com/microsoft/vscode/issues/288700)

## Summary

This appears to be caused by the PowerShell extension formatter rather than VS Code core, so it should be routed to the PowerShell extension repository.

## Change summary

Formatting a PS1 file with PowerShell's constant-string conversion can replace double quotes with single quotes and break code containing typographical quotes.

## Best solution

Track and fix the formatter behavior in https://github.com/PowerShell/vscode-powershell, specifically around the `PowerShell: Code Formatting: Use Constant Strings` setting and smart-quote parsing.
