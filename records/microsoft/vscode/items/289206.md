## Summary

Team-authored issue discussing a real problem with runSubagent agent selection logic; active discussion between members on the right fix approach. Must stay open per policy.

## Change summary

The runSubagent tool sometimes picks an inappropriate subagent (e.g. Engineering agent without edit tools) instead of using the default agent with the caller's current toolset.

## Best solution

Either make `agentName` non-default with an explicit 'default' sentinel value so models don't feel compelled to fill it, or provide better context about available agents (including the built-in agent's capabilities) so the model picks correctly. The Engineering agent description may also need clarification or removal.
