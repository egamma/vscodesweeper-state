# Issue #325608 — Sessions with two chats only explains first chat

[https://github.com/microsoft/vscode/issues/325608](https://github.com/microsoft/vscode/issues/325608)

## Summary

Valid bug filed by a team member with an identified root cause and deferred to 1.130.0. Hard keep-open: team-authored and assigned to the On Deck milestone.

## Change summary

In a multi-chat session, asking AI to explain the previous session only surfaces the first chat's transcript, missing file edits from the second chat.

## Best solution

Fix `get_session_context` / `getChatContext` to aggregate all peer chats in a session rather than returning only the default chat's transcript. The root cause is identified by sandy081 and deferred to 1.130.0.
