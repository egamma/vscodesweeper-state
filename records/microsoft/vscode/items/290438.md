# Issue #290438 — Advanced setting for auth provider does not show in Settings UI after extension unification

[https://github.com/microsoft/vscode/issues/290438](https://github.com/microsoft/vscode/issues/290438)

## Summary

Team-authored issue; the `authProvider` setting for GitHub Enterprise authentication is not registered in the Copilot extension's package.json configuration contributions, so it doesn't appear in the Settings UI under extension unification. The setting is still respected programmatically but is invisible to users in the UI.

## Change summary

The `github.copilot.advanced` authProvider setting does not appear in the VS Code Settings UI after extension unification, though it is still functional when set manually in settings.json.

## Best solution

Register the `authProvider` setting (with `github-enterprise` as a valid enum value) in the Copilot extension's `package.json` configuration contributions so it surfaces in the Settings UI. Alternatively, if the comment from JamesMGreene is correct that it only reorders sign-in buttons, document that and close as low-priority polish.
