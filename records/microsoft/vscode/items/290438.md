## Summary

Team-authored issue; the bug is confirmed on current main — the `advanced.authProvider` setting is consumed in `configurationService.ts` but never declared in `package.json` contributes/configuration, so the Settings UI cannot render it. Keeping open per hard keep-open rule (MEMBER author).

## Change summary

The `github.copilot.advanced.authProvider` setting no longer appears in the Settings UI after extension unification because it is not registered in the Copilot extension's package.json configuration contribution.

## Best solution

Register `github.copilot.advanced.authProvider` (enum: `github`, `github-enterprise`) in the `contributes.configuration` section of `extensions/copilot/package.json` so it appears in the Settings UI, or decide it should remain a hidden/JSON-only setting and update the GitHub docs accordingly.
