# Issue #325172 — Changes panel does not update after history rewrites

[https://github.com/microsoft/vscode/issues/325172](https://github.com/microsoft/vscode/issues/325172)

## Summary

Team-authored issue (MEMBER) — must keep open for maintainer judgment. The Agents window Changes panel anchors its diff baseline at session-start HEAD and does not re-anchor after git history rewrites (rebase, reset, amend), causing phantom diffs in standard PR review workflows.

## Change summary

The Changes panel in the Agents window does not update its diff baseline after git history rewrites like rebase, reset, or amend, showing stale phantom diffs.

## Best solution

The changes diff infrastructure in `src/vs/sessions/contrib/changes/` needs to either (a) watch for history rewrites and re-compute the merge-base automatically, (b) re-anchor to the current merge-base with the branch's upstream on each refresh, or (c) expose a manual 'reset baseline' command. The `ISessionGitRepository` model (`src/vs/sessions/services/sessions/common/session.ts`) tracks `baseBranchName` and `upstreamBranchName` but appears to lack a mechanism to detect when the underlying commit graph has been rewritten. The `changesViewService` and `changesMultiDiffSourceResolver` likely need to invalidate their cached base when the branch tip's ancestry changes.
