# Issue #303827 — Copilot Chat shows +0/-0 and no diff for file edits, even though changes are applied

[https://github.com/microsoft/vscode/issues/303827](https://github.com/microsoft/vscode/issues/303827)

## Summary

Valid bug: the diff summary resets to +0/-0 after edits are accepted because `_doAccept()` clears the diff state before the UI can read it. A fix exists on branch `pierceboggan/auto-accept-agent-changes` (commit 051e8e35cd9) but has not merged to main.

## Change summary

Copilot Chat edit summary shows +0/-0 and empty diff view after edits are applied, because acceptance resets the diff state before the UI captures the counts.

## Best solution

Land the snapshot approach from commit 051e8e35cd9 (or equivalent) that preserves linesAdded/linesRemoved on the entry before `_doAccept()` resets the diff, so the UI always has valid fallback counts. The fix touches `chatEditingModifiedFileEntry.ts` and `chatInputPart.ts`.
