# Issue #214371 — VSCode forces file type associations on macos

[https://github.com/microsoft/vscode/issues/214371](https://github.com/microsoft/vscode/issues/214371)

## Summary

This is an ongoing discussion about VS Code's intentional macOS file type declarations in Info.plist. The behavior is by design (file types are explicitly configured in build/lib/electron.ts), but the user-experience complaint about macOS Launch Services overriding user preferences on install/update is legitimate and under discussion. Keeping open with its current `under-discussion` label.

## Change summary

VS Code on macOS registers CFBundleDocumentTypes for many file extensions, causing macOS to associate those files with VS Code and override user preferences on install or update.

## Best solution

The area owner should decide whether to reduce the set of declared file types, change the CFBundleTypeRole from 'Editor' to 'Viewer' (which avoids claiming default ownership on macOS), or document the workaround. The configuration lives in `build/lib/electron.ts` in the `darwinBundleDocumentTypes` array. The macOS Launch Services behavior that re-claims associations on app updates is partly an OS-level issue.

## Evidence

- **source location:** File type declarations are explicitly configured in `build/lib/electron.ts` in the `darwinBundleDocumentTypes` array, registering dozens of extensions (c, h, js, ts, py, go, html, json, md, etc.) with role 'Editor'.
- **prior issues:** https://github.com/microsoft/vscode/issues/8687 (closed, labeled bug+verified) and https://github.com/microsoft/vscode/issues/51252 (closed) cover the same complaint; both are locked.
- **macOS behavior:** macOS Launch Services re-evaluates default app associations when an app is installed or updated and it declares CFBundleDocumentTypes with role 'Editor'. This is partly OS-level behavior, not solely VS Code's doing.
- **user workaround:** Comment 4 provides a compact workaround: `plutil -remove CFBundleDocumentTypes` followed by re-signing, though this must be repeated after every update.
