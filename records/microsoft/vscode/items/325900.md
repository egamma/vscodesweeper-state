# Issue #325900 — Worktrees omit gitignored .claude folder, preventing CLAUDE settings/plugins from roaming

[https://github.com/microsoft/vscode/issues/325900](https://github.com/microsoft/vscode/issues/325900)

## Summary

Team-authored bug (hard keep-open). The agent host's `addWorktree` creates worktrees via `git worktree add`, which by design only checks out tracked files — gitignored `.claude/` customizations (rules, skills, plugins, settings) are silently lost in the new worktree. The worktree creation path needs to copy or symlink the `.claude/` folder from the main working tree.

## Change summary

Git worktrees created by the agent host omit the gitignored `.claude` folder, so CLAUDE customizations (CLAUDE.md, skills, plugins, settings) do not roam to agent worktree sessions.

## Best solution

After `git worktree add` in `AgentHostGitService.addWorktree`, copy (or symlink) the `.claude/` directory from the source repository root into the new worktree root, so that workspace-scoped Claude customizations discovered by `claudeSessionCustomizationDiscovery.ts` are present in worktree-based sessions. The same treatment may be needed for `.mcp.json` and root-level `CLAUDE.md`/`CLAUDE.local.md` files.
