# Issue #193210 — Problems filtering: multi-word text patterns and passing a few patterns at a time

[https://github.com/microsoft/vscode/issues/193210](https://github.com/microsoft/vscode/issues/193210)

## Summary

Valid feature request — the Problems panel text filter still only supports a single text pattern with one negate flag on current main. Multiple text patterns (combining include/exclude) remain unimplemented. Two duplicates (#196014, #247925) were closed pointing here, confirming this is the canonical issue. With 11 upvotes in the Backlog milestone, it should stay open for the community process.

## Change summary

Allow multiple text filter patterns (include and exclude) in the Problems panel filter bar, so users can exclude several unrelated diagnostic sources simultaneously.

## Best solution

Extend `FilterOptions` in `src/vs/workbench/contrib/markers/browser/markersFilterOptions.ts` to parse the text filter into multiple segments (space- or comma-separated), each independently negatable, and update the visibility logic in `markersTreeViewer.ts` and `markersTable.ts` to evaluate all segments. The `@source:` filter already supports this pattern and could serve as a model.
