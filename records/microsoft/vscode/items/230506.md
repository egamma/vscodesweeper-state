# Issue #230506 — Git - VSCode Git extension doesn't ask for remote user password, when trying to clone via ssh

[https://github.com/microsoft/vscode/issues/230506](https://github.com/microsoft/vscode/issues/230506)

## Summary

Confirmed on current main: the SSH askpass handler in the Git extension crashes on SSH password prompts because it only handles passphrase and host-authenticity flows — an SSH password prompt (e.g. `user@host's password:`) has only 4 argv elements, so `argv[6]` is undefined and `.replace()` throws.

## Change summary

When cloning or pushing via SSH with password authentication (no public key), VS Code's Git extension crashes in askpass-main.js instead of prompting the user for a password.

## Best solution

Add a third branch in `handleSSHAskpass` (extensions/git/src/askpass.ts) to detect SSH password prompts (where `request` matches /password/i but not /passphrase/i) and show an input box for the password, similar to the passphrase handling. The argv structure for password prompts is `[..., "user@host's", "password:"]` with only 4 elements.
