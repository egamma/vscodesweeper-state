# Issue #131410 — Allow F12 to be bound to both go to definition and open link

[https://github.com/microsoft/vscode/issues/131410](https://github.com/microsoft/vscode/issues/131410)

## Summary

Keep this open as a backlog editor feature request. The current issue asks for a keyboard-specific way to prefer opening an editor link when the cursor is on one, and the discussion explicitly distinguishes it from the mouse/ctrl-click overlap mentioned as a possible duplicate.

## Change summary

Allow F12 to work for both go-to-definition and opening editor links by exposing a context such as `linkAtCursor` or otherwise resolving the keybinding conflict.

## Best solution

If accepted, the editor links contribution should expose a cursor-sensitive context key or equivalent command/keybinding behavior so `editor.action.openLink` can be bound ahead of go-to-definition only when a link is present. The likely area is `src/vs/editor/contrib/links/browser/links.ts` together with editor context key plumbing.
