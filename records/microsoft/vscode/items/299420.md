# Issue #299420 — Terminal streaming output is clipped

[https://github.com/microsoft/vscode/issues/299420](https://github.com/microsoft/vscode/issues/299420)

## Summary

Team-authored bug report (MEMBER) — keeping open for maintainer judgment. The issue describes terminal streaming output being visually clipped, likely because the 'Executing' progress node inside the thinking/tool header displaces the terminal output area upward without the height calculation accounting for it.

## Change summary

Terminal streaming output in chat is visually clipped when the 'Executing' progress indicator is shown alongside it.

## Best solution

The height calculation in `ChatTerminalToolOutputSection._updateOutputHeight()` (around line 1580 of `chatTerminalToolProgressPart.ts`) likely needs to account for the space consumed by the 'Executing' thinking-content node rendered by `chatThinkingContentPart.ts`. The fix would adjust the available height or offset when the progress indicator is visible, or ensure the parent container's flex/overflow layout doesn't let the progress node push the terminal output out of view.
