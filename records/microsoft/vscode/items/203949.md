# Issue #203949 — Search: allow multi-select and open in editor

[https://github.com/microsoft/vscode/issues/203949](https://github.com/microsoft/vscode/issues/203949)

## Summary

Valid feature request to multi-select search results and open them all as editors. Not yet implemented on main — the search tree supports multi-selection but has no action to open all selected results. Filed by a core team member (bpasero) and parked in Backlog; keeping open.

## Change summary

Add the ability to multi-select search results in the Search view and open all selected files as editors.

## Best solution

Add a context-menu action and/or keybinding in the Search view that takes the current tree multi-selection (`tree.getSelection()`), extracts the unique file resources, and opens them via `IEditorService.openEditors()`. The implementation would live in `src/vs/workbench/contrib/search/browser/searchActionsNav.ts` or a new search actions file, registered with a `when` clause gated on multi-selection in the search tree.
