# Issue #297886 — Git prompts via askpass in Dev Container despite valid cached credentials

[https://github.com/microsoft/vscode/issues/297886](https://github.com/microsoft/vscode/issues/297886)

## Summary

Legitimate bug at the boundary between the git extension and the Dev Containers extension: the system's SSH_ASKPASS (gnome-ssh-askpass) fires alongside VS Code's own GIT_ASKPASS inside a Dev Container, producing unnecessary credential prompts. The git extension sets GIT_ASKPASS and SSH_ASKPASS for its own spawned processes, but the container environment's system askpass is not being suppressed. A related feature request exists in vscode-remote-release (#8883). Keeping open — the containers and git labels are correct, and the fix likely needs coordination between the two.

## Change summary

System askpass (gnome-ssh-askpass) prompts for credentials inside a Dev Container even though VS Code's git extension already provides valid credentials via its own GIT_ASKPASS.

## Best solution

The Dev Containers extension should ensure that GIT_ASKPASS and SSH_ASKPASS are set to VS Code's askpass scripts in the container environment, overriding any system-level askpass. Coordinate with https://github.com/microsoft/vscode-remote-release/issues/8883 which tracks the same gap. The git extension side (extensions/git/src/askpass.ts) already sets these vars for its own spawned processes, but the container-wide environment needs the same override.
