# Issue #306261 — git integration can fork wrong repository when there is a period in the repository name

[https://github.com/microsoft/vscode/issues/306261](https://github.com/microsoft/vscode/issues/306261)

## Summary

Confirmed on current main: the regex in pushErrorHandler.ts excludes dots from the repo-name capture group, causing repos with periods in their names (e.g. `test-fork.net`) to be truncated when forking. The correct pattern already exists in util.ts's `getRepositoryFromUrl`.

## Change summary

When pushing to a GitHub repo you don't own and the repo name contains a period (e.g. `test-fork.net`), VS Code forks the wrong repository because the name is truncated at the first dot.

## Best solution

Replace the regex on line 120 of `extensions/github/src/pushErrorHandler.ts` with one that allows dots in the repo name — ideally reuse `getRepositoryFromUrl` from `extensions/github/src/util.ts`, which already handles this correctly. This also fixes the related issue https://github.com/microsoft/vscode/issues/300647.
