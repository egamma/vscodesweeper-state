# Issue #318362 — debug configuration doesn't restore to last selected after Reload Window(Remote SSH, multi-repo .code-workspace)

[https://github.com/microsoft/vscode/issues/318362](https://github.com/microsoft/vscode/issues/318362)

## Summary

Confirmed by team member @roblourens as reproducing on Remote SSH but not locally. The restore logic in the ConfigurationManager constructor likely fails to match the stored launch URI in the remote case, causing it to fall back to the first launch alphabetically instead of the previously selected one. No fix found on main, no duplicate found.

## Change summary

After Reload Window in a Remote SSH multi-root workspace, the debug configuration reverts to the first folder's config alphabetically instead of restoring the last selected one.

## Best solution

Investigate why `previousSelectedLaunch` (line 90 of debugConfigurationManager.ts) is not found in the Remote SSH case — the string comparison of `l.uri.toString() === previousSelectedRoot` may fail due to URI encoding differences in remote URIs across reload. The `uriIdentityService.extUri.isEqual` method (already used in `getLaunch`) would be more robust than raw string comparison. Fix likely localized to the constructor restore logic in `src/vs/workbench/contrib/debug/browser/debugConfigurationManager.ts`.
