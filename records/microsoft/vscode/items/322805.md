# Issue #322805 — Agent customization for local continues to attempt to download old plugins from source after they have been uninstalled

[https://github.com/microsoft/vscode/issues/322805](https://github.com/microsoft/vscode/issues/322805)

## Summary

Confirmed bug: uninstalling a marketplace plugin removes it from the installed-plugins store but leaves the stale marketplace reference in `chat.plugins.marketplaces` config, causing errors on subsequent install attempts from that source.

## Change summary

Uninstalling a plugin from source does not remove its marketplace entry from settings, so subsequent installs attempt to download from stale/deleted branches.

## Best solution

In the `remove()` callback in `MarketplaceAgentPluginDiscovery._discoverPluginSources()` (agentPluginServiceImpl.ts ~line 785), after removing the installed plugin, also remove the marketplace config entry from `chat.plugins.marketplaces` when no other installed plugins reference that same marketplace. The existing logic in `chatPluginActions.ts` (the 'removeMarketplace' case) already shows how to filter the config value.
