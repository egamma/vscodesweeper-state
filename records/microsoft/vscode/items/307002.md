## Summary

The report describes new terminals being created for every command, but only with a third-party OpenRouter model. The terminal tool already has built-in foreground terminal reuse per chat session — new terminals are only created for background/async mode commands. The most likely cause is the third-party model sending `mode: "async"` or `isBackground: true` for every tool call, which is model behavior, not a VS Code defect. Needs confirmation on whether this reproduces with a standard Copilot model.
