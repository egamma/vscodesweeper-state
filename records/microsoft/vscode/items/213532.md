# Issue #213532 — Git - Git fails to switch to remote branch when same name local branch exists

[https://github.com/microsoft/vscode/issues/213532](https://github.com/microsoft/vscode/issues/213532)

## Summary

Still reproduces on current main: when checking out a remote branch whose default local name conflicts with an existing local branch, VS Code throws a raw git error instead of prompting for an alternative branch name. The defect path in `CheckoutRemoteHeadItem.run()` is unguarded.

## Change summary

Checking out a remote branch (e.g. `upstream/master`) via the VS Code branch picker fails with 'a branch named master already exists' when a local branch of the same name already exists, instead of prompting the user for an alternative local branch name.

## Best solution

In `CheckoutRemoteHeadItem.run()` (extensions/git/src/commands.ts), catch `BranchAlreadyExists` from `checkoutTracking()` and prompt the user for an alternative local branch name (with an input box and branch-name validation), then create the branch with `--track` pointing to the remote ref. A similar pattern exists in the branch-rename flow at line ~3363.
