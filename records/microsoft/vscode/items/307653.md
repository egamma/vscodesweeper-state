# Issue #307653 — Insiders failing to update, logs show "Failed to rename current to old: Access is denied."

[https://github.com/microsoft/vscode/issues/307653](https://github.com/microsoft/vscode/issues/307653)

## Summary

Team-authored issue (MEMBER); must keep open. The crashpad-handler process retains a handle on the old executable after VS Code closes, blocking the inno-updater rename step. No fix for this specific handle-retention issue found on current main.

## Change summary

VS Code Insiders update fails on Windows because the crashpad-handler process holds a file handle on the old executable, causing 'Access is denied' when inno-updater tries to rename it.

## Best solution

The inno-updater (build/win32/) needs to terminate the lingering crashpad-handler process before attempting the rename, or the crashpad-handler shutdown logic in the Electron main process needs to ensure the process is fully terminated before the updater runs. The relevant code paths are in `build/win32/` (inno_updater) and `src/vs/platform/update/electron-main/updateService.win32.ts`.
