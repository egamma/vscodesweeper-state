# Issue #319825 — Terminal detail popup missing vertical scrollbar, bottom content & hidden buttons cannot be clicked

[https://github.com/microsoft/vscode/issues/319825](https://github.com/microsoft/vscode/issues/319825)

## Summary

Confirmed in source: the hover widget sets maxHeight only on containerDomNode (line 604 of hoverWidget.ts), never on contentsDomNode, so the DomScrollableElement cannot detect content overflow and never shows a vertical scrollbar. Content is clipped by the container's overflow:hidden instead.

## Change summary

When hover popup content exceeds the available height, the vertical scrollbar does not appear and bottom content is clipped and unreachable.

## Best solution

Propagate the maxHeight constraint to contentsDomNode so the DomScrollableElement can detect overflow and show its scrollbar. The commenter's one-line fix in src/vs/platform/hover/browser/hoverWidget.ts (adding `contentsDomNode.style.maxHeight = maxHeight`) is on the right track. This affects all hover popups, not just the terminal info panel.
