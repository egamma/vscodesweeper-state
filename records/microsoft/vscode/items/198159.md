# Issue #198159 — Broken full-page search result navigation for multiple folder directories of the same name in a workspace

[https://github.com/microsoft/vscode/issues/198159](https://github.com/microsoft/vscode/issues/198159)

## Summary

Confirmed bug still present on main: the search-result extension's `relativePathToUri` resolves workspace folder by name only, picking the first match when multiple folders share the same base name, causing navigation to the wrong file.

## Change summary

Double-clicking a search editor result navigates to the wrong file (or shows file-not-found) when a multi-root workspace has multiple folders with the same base name.

## Best solution

Fix `relativePathToUri` in `extensions/search-result/src/extension.ts` (around line 156) so that when the `workspaceName` matches multiple folders, it disambiguates — for example by also matching the workspace folder URI against the serialized path, or by using the folder index. The serialization side in `searchEditorSerialization.ts` may also need to emit a disambiguating identifier (e.g. the full path) when folder names collide.
