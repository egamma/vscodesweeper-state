# Issue #198159 — Broken full-page search result navigation for multiple folder directories of the same name in a workspace

[https://github.com/microsoft/vscode/issues/198159](https://github.com/microsoft/vscode/issues/198159)

## Summary

Confirmed bug still reproduces on main: the search-result extension resolves file paths by workspace folder name, taking the first match when duplicates exist. The issue carries an 'On Deck' milestone, which mandates keep-open.

## Change summary

Double-clicking a search editor result navigates to the wrong file when multiple workspace folders share the same base name.

## Best solution

Fix `relativePathToUri` in `extensions/search-result/src/extension.ts` (line 156) to disambiguate folders with duplicate names — either by using the folder index or by making the serialization format unambiguous (e.g. include the full path prefix when names collide). The serialization side (`searchEditorSerialization.ts`) may also need adjustment to emit a disambiguated label.
