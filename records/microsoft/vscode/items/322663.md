# Issue #322663 — Extensions in custom --extensions-dir are unexpectedly deleted after VS Code updates

[https://github.com/microsoft/vscode/issues/322663](https://github.com/microsoft/vscode/issues/322663)

## Summary

Confirmed bug with an open fix PR (#324275) from a team member — keep open until the PR lands. The auto-update relaunch drops --extensions-dir, and Settings Sync then propagates the empty extension list, wiping the user's extensions.

## Change summary

After a VS Code update, the auto-relaunch drops the --extensions-dir CLI argument, causing extensions in the custom directory to be invisible and subsequently deleted via Settings Sync.

## Best solution

Land https://github.com/microsoft/vscode/pull/324275 which preserves relaunch arguments (including --extensions-dir) across updates. The Settings Sync half of the chain may also warrant a safeguard against syncing an empty extension list when the profile change is due to a missing CLI argument.
