# Issue #305570 — Sessions: Unmatched brackets are highlighted red

[https://github.com/microsoft/vscode/issues/305570](https://github.com/microsoft/vscode/issues/305570)

## Summary

Still reproduces on main — the Sessions chat input editor doesn't disable bracket pair colorization or matchBrackets, unlike the workbench chat input which explicitly sets `bracketPairColorization: { enabled: false }`. Milestone is 'On Deck', deferring to maintainer scheduling.

## Change summary

Unmatched brackets in the Sessions chat input are highlighted red because bracket pair colorization is not disabled for that editor.

## Best solution

Add `matchBrackets: 'never'` and `bracketPairColorization: { enabled: false }` to the editor options in `src/vs/sessions/contrib/chat/browser/newChatInput.ts`, mirroring how the workbench `chatInputPart.ts` disables them.
