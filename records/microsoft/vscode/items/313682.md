# Issue #313682 — SCM: Count Badge not working anymore

[https://github.com/microsoft/vscode/issues/313682](https://github.com/microsoft/vscode/issues/313682)

## Summary

Plausible regression from PR #300796 (commit e084e0729bc, shipped 1.113.0) which switched the 'all' count badge from counting non-hidden repositories to counting only 'visible' ones (requiring selectionIndex !== -1). The report is thin on reproduction details but the described behavior — 'all' showing only the focused repo count — is consistent with repos that are non-hidden but not 'selected' in the SCM view. Routing to the SCM area owner for evaluation.

## Change summary

The SCM activity bar count badge with setting 'all' only shows the focused repository's count instead of summing all repositories.

## Best solution

The SCM maintainer should evaluate whether `visibleRepositories` (which filters by `selectionIndex !== -1`) is the correct data source for the 'all' case, or whether it should revert to counting all non-hidden repos. The setting description says 'all Source Control Provider count badges' without mentioning visibility. If `visibleRepositories` is intentionally narrower, the setting description should be updated; otherwise the 'all' case should use a broader filter.
