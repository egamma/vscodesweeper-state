# Issue #284339 — Allow .json files to support comments when a specific comment line is at the top of the file.

[https://github.com/microsoft/vscode/issues/284339](https://github.com/microsoft/vscode/issues/284339)

## Summary

Legitimate feature request already triaged into Backlog with correct labels. Proposes auto-detecting JSON files as JSONC based on a magic comment line (per jsonc.org). No strong reason to close — it's a real gap, though niche.

## Change summary

Allow .json files to be treated as JSONC when a specific comment line is present at the top, following the jsonc.org convention.

## Best solution

If pursued, this would live in the JSON language service's mode detection or in the file association logic (likely `src/vs/workbench/contrib/files/` or `src/vs/workbench/services/languageDetection/`). Alternatively, this could remain as-is with `files.associations` as the recommended workaround, and the team may decide it's too niche for core — but that's a product call, not a sweep-close.
