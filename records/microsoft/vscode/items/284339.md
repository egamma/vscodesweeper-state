# Issue #284339 — Allow .json files to support comments when a specific comment line is at the top of the file.

[https://github.com/microsoft/vscode/issues/284339](https://github.com/microsoft/vscode/issues/284339)

## Summary

Legitimate feature request to auto-detect JSONC mode from a magic comment in .json files. No community traction (0 upvotes in ~6 months); VS Code already offers `files.associations` as a workaround. Keeping open as a low-priority feature request.

## Change summary

Request to treat .json files as JSONC when a specific comment line is present at the top of the file, following the jsonc.org proposed convention.

## Best solution

If pursued, the JSON language service could detect a magic comment and switch to JSONC mode automatically. However, VS Code already provides `files.associations` in settings to map specific .json files to jsonc. Given no traction, this likely stays parked unless the jsonc.org convention gains broader ecosystem adoption.
