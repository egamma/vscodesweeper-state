# Issue #180820 — SCM - highlight repositories with uncommitted changes

[https://github.com/microsoft/vscode/issues/180820](https://github.com/microsoft/vscode/issues/180820)

## Summary

Valid feature request for better visual differentiation of repositories with changes in the multi-repo SCM view. The current count badge and star icon exist but are subtle. In the Backlog milestone with 0 upvotes; keeping open for the community process.

## Change summary

Visually highlight SCM repositories that have uncommitted changes (e.g., with a different font color or stronger visual cue beyond the count badge).

## Best solution

Add a visual cue (e.g., a color change or bold label) to the repository renderer in `src/vs/workbench/contrib/scm/browser/scmRepositoryRenderer.ts` when `repository.provider.count` is non-zero. This could leverage the existing `data-count` attribute on the count container to also style the repository label via CSS. The related closed issue https://github.com/microsoft/vscode/issues/198941 asked for filtering (hiding unchanged repos) and was closed for lack of upvotes — this is a lighter ask (just visual emphasis).
