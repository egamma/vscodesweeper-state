# Issue #309450 — Open file action opens file in read only mode when invoked through keybinding

[https://github.com/microsoft/vscode/issues/309450](https://github.com/microsoft/vscode/issues/309450)

## Summary

Still reproduces on main: when `git.openFile` is invoked via keybinding without a Resource argument, the fallback path uses `window.activeTextEditor.document.uri` (a read-only git-scheme URI) instead of converting it to a file URI, causing the file to open in read-only mode.

## Change summary

The `git.openFile` command opens the file in read-only mode when invoked via keybinding because it uses the raw git-scheme URI of the active diff editor instead of converting to the file-scheme URI.

## Best solution

In the keybinding fallback path of `openFile` (extensions/git/src/commands.ts ~line 1319–1320), detect when `window.activeTextEditor.document.uri` is a git-scheme URI and convert it via `fromGitUri(...).path` to a file URI before opening, matching the behavior of the button-click code path.
