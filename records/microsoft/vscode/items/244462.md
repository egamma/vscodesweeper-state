# Issue #244462 — Use same hover group for all copilot actions near input

[https://github.com/microsoft/vscode/issues/244462](https://github.com/microsoft/vscode/issues/244462)

## Summary

Still valid on main: multiple chat input elements still use the deprecated `setupManagedHover` API instead of `setupDelayedHover` with a shared hover group, causing inconsistent hover timing across copilot actions near the input.

## Change summary

Hover timing is inconsistent across chat input toolbar icons — some show immediately (shared group) while others have a delay.

## Best solution

Migrate remaining `setupManagedHover` usages in `src/vs/workbench/contrib/chat/browser/widget/input/` (chatModelPicker.ts, modelPickerActionItem.ts, chatInputNotificationWidget.ts) to `setupDelayedHover` and place all chat input action hovers in a single hover group so they display instantly when moving between adjacent icons.
