## Change summary

When working in a WSL remote workspace, File > New Window opens a local (non-WSL) window instead of a new WSL window.

## Best solution

Either add a 'New Remote Window' / 'New WSL Window' command in the File menu when a WSL remote is active, or change 'New Window' to inherit the current remote context. This likely belongs in the WSL extension (`ms-vscode-remote.remote-wsl`) rather than VS Code core, since remote connection semantics are extension-owned.
