# Issue #285445 — Chat input field strips backslash before dot in Windows paths (e.g., 

[https://github.com/microsoft/vscode/issues/285445](https://github.com/microsoft/vscode/issues/285445)

## Summary

Still valid on main: user chat requests are rendered as markdown, which visually strips backslashes before dots in Windows paths. The maintainer confirmed the input is sent verbatim to the model — this is a display issue, not a data-loss issue. The underlying code in `chatListRenderer.ts` still renders request content via markdown, and no change has landed to address this.

## Change summary

Windows paths containing backslash-dot sequences (e.g. `\.ssh`) appear visually corrupted in the chat view because user requests are rendered as markdown.

## Best solution

Either escape markdown syntax characters in user request text before rendering, or disable markdown rendering for user request messages entirely (as roblourens suggested in the comment). The rendering logic lives in `chatListRenderer.ts` around the `renderMarkdown` method for request content parts.
