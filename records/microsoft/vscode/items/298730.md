# Issue #298730 — Inconsistent comment toggle in in `.ini` / `.properties` depending on which file is opened first

[https://github.com/microsoft/vscode/issues/298730](https://github.com/microsoft/vscode/issues/298730)

## Summary

Still reproduces on main: the ini and properties languages share the same TM grammar scope (`source.ini`), so `getLanguageIdAtPosition` returns whichever language was tokenized first, causing the wrong comment character to be used for the other language.

## Change summary

Toggle Line Comment uses the wrong comment character (`;` vs `#`) for `.ini` / `.properties` files depending on which file type was opened first in a session.

## Best solution

The fix likely belongs in the tokenization/language-resolution layer: when two language IDs share the same grammar scope, `getLanguageIdAtPosition` should return the language ID of the *model's own language* rather than whichever happened to register first. Alternatively, `_gatherPreflightCommentStrings` could use `model.getLanguageId()` (the document-level language) as a fallback when the token-level language is ambiguous due to shared scopes.
