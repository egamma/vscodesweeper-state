# Issue #314722 — Context Window indicator stays at 0% for third-party BYOK chat providers because ExtensionContributedChatEndpoint hardcodes usage to zero

[https://github.com/microsoft/vscode/issues/314722](https://github.com/microsoft/vscode/issues/314722)

## Summary

Already fixed on main: commit 2e3f347c80e added `CustomDataPartMimeTypes.Usage` handling in `extChatEndpoint.ts`, so third-party providers can now report token usage via a `LanguageModelDataPart` with mime type `'usage'`, and the Context Window widget reflects it. This can be closed as completed.

## Change summary

The Context Window indicator always showed 0% for models provided by third-party `vscode.lm.registerLanguageModelChatProvider` extensions because `ExtensionContributedChatEndpoint.makeChatRequest2` hardcoded usage to all zeros.

## Best solution

Keep the shipped fix in commit 2e3f347c80e; no further change is needed. The open PR https://github.com/microsoft/vscode/pull/314801 appears superseded by this commit and can likely be closed as well.
