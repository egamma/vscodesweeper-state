# Issue #160072 — Guarantee that TextDocument.version doesn't skip versions

[https://github.com/microsoft/vscode/issues/160072](https://github.com/microsoft/vscode/issues/160072)

## Summary

Valid request to strengthen the API documentation guarantee around TextDocument.version incrementing by exactly 1. The behavior holds in practice (confirmed by code inspection) but has never been formally guaranteed. Issue #223077 is a related but broader request (never reset on reopen) — not a duplicate since this issue is narrower (no-skip guarantee). Keeping open as under-discussion.

## Change summary

Guarantee in the API docs that TextDocument.version increments by exactly 1 per change (no skipped versions).

## Best solution

If the team decides to commit to the +1 increment contract, update the vscode-api TextDocument.version JSDoc to state the version increments by exactly 1. This is a documentation/contract decision that should be made alongside https://github.com/microsoft/vscode/issues/223077 (the broader monotonicity guarantee across close/reopen). The code already behaves this way (`_increaseVersionId` in `src/vs/editor/common/model/textModel.ts:783` does `+1`), so the question is whether the team wants to lock that in as a stable API contract.
