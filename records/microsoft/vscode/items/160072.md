# Issue #160072 — Guarantee that TextDocument.version doesn't skip versions

[https://github.com/microsoft/vscode/issues/160072](https://github.com/microsoft/vscode/issues/160072)

## Summary

This should remain open as an API contract discussion: the request is not merely to document current behavior, but to decide whether VS Code should guarantee contiguous `TextDocument.version` increments and behavior across close/reopen. A maintainer already confirmed part of the requested guarantee is not current behavior and asked for more eyes.

## Change summary

The issue asks VS Code to strengthen the `TextDocument.version` API contract from strictly increasing to non-skipping increments, including clearer semantics across document close and reopen during an LSP session.

## Best solution

Keep this in `editor-api`/`under-discussion` until the API owner decides the contract: either explicitly document that consumers may only rely on monotonic increase, or add tests and implementation guarantees for +1-per-content-change and define close/reopen version restoration semantics.
