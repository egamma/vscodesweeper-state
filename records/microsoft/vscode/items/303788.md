# Issue #303788 — Windows: First VS Code instance sometimes replaces PATH with full Visual Studio DevShell environment

[https://github.com/microsoft/vscode/issues/303788](https://github.com/microsoft/vscode/issues/303788)

## Summary

Most likely caused by an installed extension using the EnvironmentVariableCollection API to prepend Visual Studio paths to the terminal PATH. VS Code's own shell environment resolution is explicitly skipped on Windows (returns empty). The report contains a contradiction — `--disable-extensions` prevents the issue, yet extension bisect cannot identify the culprit — which needs clarification before triage can proceed.

## Change summary

When VS Code is launched as the first instance from the Start Menu on Windows, the integrated terminal's PATH is replaced with Visual Studio DevShell-style paths and user environment paths go missing.

## Best solution

Identify which installed extension is mutating PATH via the EnvironmentVariableCollection API. The user has C# DevKit, C#, C++ Tools, and .NET Runtime extensions, all of which interact with Visual Studio installations. The terminal's environment changes indicator (ⓘ icon on the terminal tab) should reveal which extensions are contributing. If it turns out to be C# DevKit or cpptools, the bug belongs on those extensions' issue trackers.
