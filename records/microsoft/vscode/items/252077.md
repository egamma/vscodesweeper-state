# Issue #252077 — navigator is now a global in nodejs

[https://github.com/microsoft/vscode/issues/252077](https://github.com/microsoft/vscode/issues/252077)

## Summary

Still valid: the `extensions.supportNodeGlobalNavigator` setting defaults to `false` on current `main`, so the `PendingMigrationError` shim continues to fire for any extension accessing `navigator` in the Node.js extension host. The debt to resolve the migration (change the default, remove the shim, or improve the handling) remains unaddressed. With 68 upvotes and broad impact across remote extensions and Copilot Chat, this should stay open.

## Change summary

The `PendingMigrationError` thrown when extensions access the Node.js `navigator` global crashes or disrupts remote extension hosts (SSH, WSL, Dev Containers) and Copilot Chat.

## Best solution

Flip the `extensions.supportNodeGlobalNavigator` default to `true` (or remove the shim entirely) once the major first-party extensions (remote-ssh, remote-wsl, copilot-chat) have been updated to handle `navigator` correctly. The shim lives in `src/vs/workbench/api/node/extensionHostProcess.ts` (line ~142) and the setting default in `src/vs/workbench/contrib/extensions/browser/extensions.contribution.ts` (line ~365). Alternatively, downgrade the error from `onUnexpectedExternalError` to a one-time console warning so it stops crashing extension hosts.
