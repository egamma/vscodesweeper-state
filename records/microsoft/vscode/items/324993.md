# Issue #324993 — MCP tools and vscode.lm.registerTool tools silently dropped in Agent mode on Remote SSH host (regression of #317992, still broken in 1.128)

[https://github.com/microsoft/vscode/issues/324993](https://github.com/microsoft/vscode/issues/324993)

## Summary

Legitimate bug with excellent reporter root-cause analysis: MCP and extension-registered tools from the remote extension host are silently excluded because they register after request.tools is serialized and lack the `extension_installed_by_tool` tag, causing the filter in getEnabledTools to drop them. Routing to the chat-mcp area.

## Change summary

MCP tools and vscode.lm.registerTool tools spawned on a Remote SSH extension host are silently absent from the agent session because they are not present in request.tools at serialization time.

## Best solution

Fix the race in `extensions/copilot/src/extension/tools/vscode-node/toolsService.ts` getEnabledTools (around line 347-358): tools that are present in the live tools list but absent from `requestToolsByName` (i.e. `toolPickerSelection === undefined`) should default to enabled when they are MCP or contributed tools — not silently excluded. The `extension_installed_by_tool` tag gate is too narrow for tools that registered late due to remote extension host latency.
