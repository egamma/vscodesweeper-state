# Issue #239851 — Force Update Git internals when Refreshing Explorer

[https://github.com/microsoft/vscode/issues/239851](https://github.com/microsoft/vscode/issues/239851)

## Summary

Kept open: the issue carries the 'On Deck' milestone (a scheduling decision), and a team member (lszomoru) is actively investigating. The underlying problem — GitIgnoreDecorationProvider caches check-ignore results on demand but never invalidates them when .gitignore changes — is still present on current main.

## Change summary

Explorer file decorations for git-ignored files don't update when .gitignore changes until the project is closed and reopened; 'Git: Refresh' does not clear the ignore cache.

## Best solution

The GitIgnoreDecorationProvider in extensions/git/src/decorationProvider.ts caches check-ignore results per-path but has no invalidation path tied to .gitignore file changes or the 'Git: Refresh' command. The fix should watch for .gitignore changes (or hook into onDidRunGitStatus) to clear the ignore decoration queue/cache and fire onDidChangeFileDecorations, and the 'Git: Refresh' command should trigger this invalidation as well. lszomoru is already investigating.
