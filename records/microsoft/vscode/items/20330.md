# Issue #20330 — Git - Support GitHub 2FA

[https://github.com/microsoft/vscode/issues/20330](https://github.com/microsoft/vscode/issues/20330)

## Summary

Resolved since VS Code 1.46.0: the built-in GitHub extension registers a git credential provider that supplies an OAuth session token as the password for github.com, so 2FA is handled transparently via the browser sign-in flow — no separate 2FA code dialog is needed.

## Change summary

Git push/sync to a GitHub repo with 2FA fails because VS Code only prompts for username/password with no way to enter a 2FA code.

## Best solution

Keep the shipped behavior — the `github.gitAuthentication` credential provider (commit a85a2e60446, shipped in 1.46.0) resolves this by using OAuth tokens instead of passwords. No further change needed.

  > This is resolved: since VS Code 1.46.0 (June 2020), the built-in GitHub extension registers a git credential provider (`extensions/github/src/credentialProvider.ts`, commit https://github.com/microsoft/vscode/commit/a85a2e60446) that supplies your OAuth session token as the git password for `github.com`. Two-factor authentication is handled transparently through the browser sign-in flow — there's no username/password box to stall on, and no need for a separate 2FA code dialog. The setting `github.gitAuthentication` controls this and defaults to `true`.
