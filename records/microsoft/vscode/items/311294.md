# Issue #311294 — C/C++ macros screw with bracket colouring

[https://github.com/microsoft/vscode/issues/311294](https://github.com/microsoft/vscode/issues/311294)

## Summary

This is a repeatedly reported, known limitation of bracket pair colorization: the algorithm operates on the token stream and has no understanding of C/C++ preprocessor conditionals. Issue https://github.com/microsoft/vscode/issues/222923 was closed as *as-designed with the explanation that the C++ grammar would need to mark one preprocessor branch as a comment for this to work. At least five other issues reporting the same behavior have been closed as duplicates or as-designed.

## Change summary

Bracket pair colorization mismatches brackets when they span C/C++ preprocessor `#ifdef`/`#else` branches.

## Best solution

Keep the current behavior as-designed. The bracket pair colorization algorithm works on the flat token stream and cannot reason about preprocessor conditional compilation branches. Fixing this would require the TextMate grammar to suppress one branch or a fundamentally different bracket-matching model — neither is planned. The reporter's suggestion of treating branches separately is interesting but would require significant architectural work in the bracket pair colorizer.
