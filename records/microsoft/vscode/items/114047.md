# Issue #114047 — Unexpected editor.selection in onDidChangeActiveTextEditor callback

[https://github.com/microsoft/vscode/issues/114047](https://github.com/microsoft/vscode/issues/114047)

## Summary

Still valid on main: the selection sent in the `onDidChangeActiveTextEditor` event is read before view state restoration occurs. The architecture (reading `codeEditor.getSelections()` at delta-construction time) remains unchanged.

## Change summary

When switching tabs in the same editor group, `onDidChangeActiveTextEditor` fires with selection at (1,1) because view state hasn't been restored yet.

## Best solution

The fix jrieken originally suggested — a `setModelAndSelection` atomic API at the editor level — would let the workbench set the model and restored selection before the ext-host delta is constructed. The change would live in `src/vs/workbench/api/browser/mainThreadDocumentsAndEditors.ts` (delta timing) and possibly `src/vs/editor/browser/widget/codeEditor/codeEditorWidget.ts` (atomic model+viewState setter).
