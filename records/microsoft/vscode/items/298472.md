# Issue #298472 — PreToolUse 'continue' blocks tool but doesn't terminate loop

[https://github.com/microsoft/vscode/issues/298472](https://github.com/microsoft/vscode/issues/298472)

## Summary

Team-filed bug with milestone 'On Deck' — must stay open per policy. The defect is traceable in the source: when a PreToolUse hook returns `continue: false` with a `stopReason`, `_handlePreToolUseDenial` blocks the tool and returns a denial `IToolResult`, but nothing propagates the stop signal to cancel the overall agent loop, so the model continues issuing tool calls.

## Change summary

A PreToolUse hook returning `continue: false` with a `stopReason` blocks the individual tool but does not terminate the agent's tool-calling loop as expected.

## Best solution

After `_handlePreToolUseDenial` returns in `invokeTool`, propagate the `stopReason` upward so the agent loop (likely in `chatServiceImpl.ts`) cancels further tool calls and stops the request. The `IExternalPreToolUseHookResult` or the denial result needs a signal the loop consumer recognizes as 'stop processing entirely'.
