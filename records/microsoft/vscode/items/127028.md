# Issue #127028 — Git - Adding a folder to `.gitignore` should not add every single file

[https://github.com/microsoft/vscode/issues/127028](https://github.com/microsoft/vscode/issues/127028)

## Summary

Still valid on main — the git.ignore command writes every individual file path rather than a folder pattern. An open PR exists, and the issue is milestoned 'On Deck', so it stays open.

## Change summary

When adding a folder to .gitignore from the SCM view, VS Code adds every individual file inside the folder instead of a single folder pattern entry.

## Best solution

Land or supersede the open PR https://github.com/microsoft/vscode/pull/138111 — the fix belongs in `extensions/git/src/repository.ts` in the `ignore()` method: detect when all files share a common directory prefix and write a single folder pattern (e.g. `node_modules/`) instead of enumerating every file. The SCM view's command handler in `extensions/git/src/commands.ts` may also need adjustment to pass folder-level info.
