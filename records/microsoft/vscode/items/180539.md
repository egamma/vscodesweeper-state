# Issue #180539 ŌĆö TreeError [SearchView] Unknown compressed tree node

[https://github.com/microsoft/vscode/issues/180539](https://github.com/microsoft/vscode/issues/180539)

## Summary

The 'Unknown compressed tree node' error in SearchView was fixed by adding a `hasNode` guard before updating individual tree elements, shipped in 1.95.0. The current code at `refreshTree` checks `event.elements.every(elem => this.searchView.getControl().hasNode(elem))` and falls back to a full tree refresh when elements are missing, preventing the stale-node crash.

## Change summary

Search view's `refreshTree` threw 'Unknown compressed tree node' when updating individual file matches that had already been removed from the compressed tree model.

## Best solution

Keep the shipped fix in commit 0729b0fb6f84 (1.95.0); no further change is needed. The `hasNode` guard in `RefreshTreeController.refreshTree` prevents `setChildren`/`updateChildren` from being called on stale nodes.
