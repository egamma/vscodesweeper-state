# Issue #137146 — "Don't show again" button does not work and is misleading

[https://github.com/microsoft/vscode/issues/137146](https://github.com/microsoft/vscode/issues/137146)

## Summary

Still present on current main: the git extension's "Don't Show Again" button for the too-many-changes warning writes `git.ignoreLimitWarning` at workspace scope (`false`), so it only suppresses the warning for that workspace. When a user opens a different folder, the warning reappears. The button label is misleading — it implies a permanent, global dismissal. Already labeled `under-discussion`; keeping open for a maintainer decision on whether to change the scope to global (`true`) or rename the button.

## Change summary

The "Don't Show Again" button on the git too-many-active-changes warning only suppresses the warning per-workspace, not globally, making the label misleading.

## Best solution

Change `config.update('ignoreLimitWarning', true, false)` to `config.update('ignoreLimitWarning', true, true)` in `extensions/git/src/repository.ts` to write the setting globally, or rename the button to something like "Dismiss for This Workspace" if per-workspace is intentional.
