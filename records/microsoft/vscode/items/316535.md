# Issue #316535 — Source Control Graph hover truncates commit metadata when commit message is long

[https://github.com/microsoft/vscode/issues/316535](https://github.com/microsoft/vscode/issues/316535)

## Summary

Valid feature request to reorder the SCM Graph hover tooltip so compact metadata (refs, author, hash) appears above the commit message, preventing long messages from pushing high-value info out of view. No existing issue owns this layout change.

## Change summary

Reorder the Source Control Graph commit hover to show metadata (refs, hash, author, date) before the commit message body, and truncate long messages.

## Best solution

Adjust the layout order in `extensions/git/src/hover.ts` (`getHistoryItemHover` / `appendContent`) and possibly the reference injection logic in `src/vs/workbench/contrib/scm/browser/scmHistory.ts` to place compact metadata (author, date, refs) first, then the commit message (with optional truncation or expand affordance).
