# Issue #260376 — New line is not being detected in prompt input model for Python shell integration

[https://github.com/microsoft/vscode/issues/260376](https://github.com/microsoft/vscode/issues/260376)

## Summary

Still open on main: Python shell integration does not report the ContinuationPrompt OSC sequence, so the PromptInputModel cannot distinguish newlines from wrapped lines, and the tab description lacks the ⏎ symbol shown for pwsh.

## Change summary

Python terminal tab description does not show the newline symbol (⏎) for multi-line input because the continuation prompt is never reported.

## Best solution

Python's shell integration (likely in the Python extension's REPL startup script) needs to emit the `ContinuationPrompt` OSC sequence (like pwsh does) so that `promptInputModel.ts` can detect newlines via `_lineContainsContinuationPrompt`. Alternatively, a heuristic for Python's known `... ` continuation pattern could be added in the model itself.
