Team-authored bug with an On Deck milestone — keeping open. The sessions list has sort logic (sortSessions, getSortKey) but search results may bypass the recency ordering. No fix commit found referencing this issue.

## Change summary

Searching for sessions in the Agents window does not preserve recency order — older sessions appear above recent ones in search results.

## Best solution

Ensure that when the sessions list is filtered by a search query, the resulting matches still pass through sortSessions with the user's chosen sorting (Updated/Created) before display. The fix likely belongs in src/vs/sessions/contrib/sessions/browser/views/sessionsList.ts where filtered results are assembled.
