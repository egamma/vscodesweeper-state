# Issue #316353 — MSAL Compliance issue with native broker on Windows Arm64 devices in our extension

[https://github.com/microsoft/vscode/issues/316353](https://github.com/microsoft/vscode/issues/316353)

## Summary

Still reproduces on main: `@azure/msal-node-runtime` is locked at 0.20.1 in `extensions/microsoft-authentication/package-lock.json`, and Windows Arm64 native broker support requires 0.20.4+. Two PRs to bump it (322475, 325386) were closed without merging.

## Change summary

Native authentication broker does not work on Windows Arm64 because the bundled `msal-node-runtime` (0.20.1) predates Arm64 Windows support (added in 0.20.4).

## Best solution

Bump `@azure/msal-node-runtime` to ≥0.20.4 in `extensions/microsoft-authentication/package-lock.json`. The closed PRs https://github.com/microsoft/vscode/pull/322475 and https://github.com/microsoft/vscode/pull/325386 attempted this — a new PR landing that bump would resolve the issue.
