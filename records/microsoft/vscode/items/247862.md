# Issue #247862 — Git "sync" fails when ssh config has "VisualHostKey=yes"

[https://github.com/microsoft/vscode/issues/247862](https://github.com/microsoft/vscode/issues/247862)

## Summary

Confirmed bug still present on main: three call sites in the git extension use non-empty stderr as a failure signal, which breaks when SSH's VisualHostKey outputs randomart to stderr. An open community PR (#309367) addresses exactly this; it should be reviewed and landed.

## Change summary

Git sync/push fails when the user's SSH config has `VisualHostKey=yes`, because informational stderr output from SSH is treated as a git command failure.

## Best solution

Review and land the open PR https://github.com/microsoft/vscode/pull/309367, which fixes the three affected call sites (`getDefaultBranch`, `revList`, `revParse`) to use exit code instead of stderr presence as a failure signal — a pattern already used elsewhere in `git.ts`.
