# Issue #126311 — Workspace Trust - Declared Untrusted Folders

[https://github.com/microsoft/vscode/issues/126311](https://github.com/microsoft/vscode/issues/126311)

## Summary

Valid feature request with 48 upvotes, in Backlog. The current trust model only stores trusted folders (allow-list); untrusting a folder merely removes it from the list rather than explicitly deny-listing it. The requested inverse workflow — trust everything except declared untrusted folders — is a real gap that no canonical issue owns.

## Change summary

Allow users to declare specific folders as always untrusted (deny-listed), so those folders automatically load in restricted mode regardless of other trust settings.

## Best solution

Extend the workspace trust model in `src/vs/workbench/services/workspaces/common/workspaceTrust.ts` to persist `trusted: false` entries via `doSetUrisTrust` (currently it only removes entries on untrust), and surface a deny-list UI in the trust editor (`workspaceTrustEditor.ts`). The longest-prefix-match logic at line ~390 already supports `trusted: false` entries — the gap is in storage and UX.
