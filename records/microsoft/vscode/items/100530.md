# Issue #100530 — Symbol occurrence highlighting overlaps occurrences of the selected text

[https://github.com/microsoft/vscode/issues/100530](https://github.com/microsoft/vscode/issues/100530)

## Summary

Partially addressed: commit c97f4be67d0 (Dec 2022) introduced a separate color for DocumentHighlightKind.Text highlights, but the core z-index layering problem — symbol occurrence highlights rendering on top of and obscuring the selection highlight — remains on current main. The issue is on On Deck milestone and should stay open.

## Change summary

Symbol occurrence highlights (document highlights) render on top of the selection highlight, making it impossible to see what text is selected when both overlap.

## Best solution

Set a lower z-index on word-highlight decorations (or a higher one on selection decorations) so the user's active selection always renders above document highlights. The fix likely lives in the decoration options in `src/vs/editor/contrib/wordHighlighter/browser/highlightDecorations.ts` or in the view-overlay rendering order in the editor.
