# Issue #100530 — Symbol occurrence highlighting overlaps occurrences of the selected text

[https://github.com/microsoft/vscode/issues/100530](https://github.com/microsoft/vscode/issues/100530)

## Summary

Keep open: the issue has an assigned `On Deck` milestone, which is an explicit scheduling signal and blocks cleanup closure. Current main has added separate colors for `DocumentHighlightKind.Text`, but that is not enough evidence to close this specific overlap/layering bug.

## Change summary

Selection highlighting can be visually obscured by symbol occurrence highlights from semantic document highlight providers such as C/C++.

## Best solution

Finish the editor-rendering/word-highlighter work so semantic document highlights do not obscure the primary selection, likely around `src/vs/editor/contrib/wordHighlighter/browser/highlightDecorations.ts` and `highlightDecorations.css`, while preserving the separate `editor.wordHighlightText*` color support already added.
