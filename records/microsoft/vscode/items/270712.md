# Issue #270712 — Selected model keeps switching back to GPT 4.1 / Sonnet 4

[https://github.com/microsoft/vscode/issues/270712](https://github.com/microsoft/vscode/issues/270712)

## Summary

The root cause was the `chat.defaultLanguageModel` A/B experiment treatment, which overrode the user's persisted model selection each time a new chat session was initialized. This experiment was removed in commit b409152750e (2025-10-23), and the model selection logic has since been extensively refactored with comprehensive tests. The bug is resolved.

## Change summary

The selected chat model kept reverting to GPT 4.1 or Sonnet 4 despite manually switching to Sonnet 4.5, even within the same workspace.

## Best solution

Keep the shipped fix — the `chat.defaultLanguageModel` experiment that was overriding persisted model selections was removed in b409152750e and shipped in 1.106.0. The model selection logic has been further hardened with a full refactor (98dc3fd3a4d) and additional fixes for session-type switching (2db203cbedb). No further change needed.
