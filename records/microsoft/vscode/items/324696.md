# Issue #324696 — Format Document shows "There is no formatter for 'fortios' files installed" even though it's installed and configured.

[https://github.com/microsoft/vscode/issues/324696](https://github.com/microsoft/vscode/issues/324696)

## Summary

The error correctly fires when no DocumentFormattingEditProvider is registered for the language. Setting `editor.defaultFormatter` only selects among registered providers — the extension must actually register one. This is a *caused-by-extension issue.

## Change summary

Format Document shows 'no formatter installed' for fortios files despite the extension being installed and defaultFormatter configured.

## Best solution

The FortiOS Configuration Helper extension (fortios-config-studio.fortios-configuration-helper) needs to register a DocumentFormattingEditProvider for the 'fortios' language via the VS Code extension API. The user should file a bug on that extension's repository.
