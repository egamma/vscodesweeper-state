# Issue #232984 — Editor GPU: Handle the case where glyphs do not fit into standard atlas page slabs

[https://github.com/microsoft/vscode/issues/232984](https://github.com/microsoft/vscode/issues/232984)

## Summary

Still present on main: the TODO referencing this issue remains in textureAtlasPage.ts, and the slab/shelf allocators still throw BugIndicatingError for oversized glyphs without graceful fallback. The bug is real and unfixed.

## Change summary

When glyphs are too large for standard atlas page slabs (e.g. at high zoom on HiDPI), rendering breaks entirely instead of falling back gracefully.

## Best solution

Implement a fallback path in the texture atlas allocator that handles oversized glyphs — either by allocating a dedicated oversized slab, falling back to DOM rendering for those glyphs, or dynamically resizing. The fix lives in `src/vs/editor/browser/gpu/atlas/`.
