# Issue #325601 — Github Copilot chat session not displaying final response

[https://github.com/microsoft/vscode/issues/325601](https://github.com/microsoft/vscode/issues/325601)

## Summary

Confirmed rendering bug where chat responses fail to display when the widget considers itself hidden (e.g. another window is on top). An open PR (#326497) is actively addressing the root cause — keeping open until it lands.

## Change summary

Chat final response content and follow-up UI are not rendered when the chat widget's visibility state is false, requiring the user to navigate away and back to see them.

## Best solution

Land or close https://github.com/microsoft/vscode/pull/326497, which fixes a content-array length mismatch between progressive and final rendering paths in `ChatListItemRenderer`. If that PR's approach doesn't cover the broader visibility-gating issue described in comment 3 (model changes silently dropped while `_visible` is false), a complementary fix in `chatWidget.ts` around line 1034 should mark the list dirty while hidden and flush on re-show.
