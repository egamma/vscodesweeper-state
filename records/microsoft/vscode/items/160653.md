# Issue #160653 — Git - should honor addIgnoredFile

[https://github.com/microsoft/vscode/issues/160653](https://github.com/microsoft/vscode/issues/160653)

## Summary

Still present on main: the git extension's generic error handler shows a popup whenever `git add` exits non-zero due to an ignored-path warning, regardless of the user's `advice.addIgnoredFile` git config. The `getGitErrorCode` function in `git.ts` has no case for the "following paths are ignored" stderr pattern, so it falls through to the default handler that displays the raw stderr.

## Change summary

VS Code's git extension shows a warning popup when staging a tracked file whose parent directory matches a .gitignore pattern, even when the user has set `git config advice.addIgnoredFile false` to suppress such warnings.

## Best solution

Add a recognized error code for the "following paths are ignored" stderr pattern in `getGitErrorCode()` (extensions/git/src/git.ts), and handle it in the error switch in `commands.ts` — either by suppressing the popup when `advice.addIgnoredFile` is false, or by providing a more actionable message (e.g. offering to force-add with `-f`). The fix is localized to the git extension's error pipeline.
