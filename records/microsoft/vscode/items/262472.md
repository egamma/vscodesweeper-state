# Issue #262472 — Perf: showing Chat view on startup is slow

[https://github.com/microsoft/vscode/issues/262472](https://github.com/microsoft/vscode/issues/262472)

## Summary

Active performance debt with multiple team members investigating and iterating. Several merged PRs have addressed parts (welcome re-rendering, suggest widget, markdown rendering), but the discussion through January 2026 identifies remaining optimization opportunities (markdown rendering cost, welcome view deferral, chat control creation overhead). This is ongoing work, not closeable.

## Change summary

Opening the Chat view in the secondary sidebar on startup adds 200–300ms due to expensive welcome-view rendering, markdown parsing, and chat control creation.

## Best solution

Continue the optimization work identified in the discussion: defer welcome-view rendering (requestIdleCallback + fade-in as connor4312 suggested), reduce markdown overhead (cache rendered HTML, use native sanitizer, or skip markdown for static welcome content), and profile the remaining ~55ms in createChatControl for further wins. The welcome view rendering and chat widget initialization in `src/vs/workbench/contrib/chat/browser/viewsWelcome/` and the chat widget are the key areas.
