# Issue #273868 — There is a 1px gap between the minimap and the scroll bar

[https://github.com/microsoft/vscode/issues/273868](https://github.com/microsoft/vscode/issues/273868)

## Summary

A visual 1px gap between minimap and scrollbar, likely caused by sub-pixel rounding in the canvas width calculation (`minimapCanvasOuterWidth = Math.floor(pixelRatio * minimapWidth) / pixelRatio`). No fix commit found. The issue is plausible but environment-specific (Linux/Wayland with a non-integer pixel ratio); keeping open.

## Change summary

A 1px gap appears between the minimap canvas and the vertical scrollbar when the minimap is enabled and content overflows horizontally.

## Best solution

The gap likely stems from the canvas outer width being slightly smaller than the minimap DOM container width due to `Math.floor(pixelRatio * minimapWidth) / pixelRatio` rounding. A fix in `src/vs/editor/common/config/editorOptions.ts` (the `_computeMinimapLayout` method) could ensure the minimap DOM width matches the canvas outer width, or set a background color on the minimap container to hide the sub-pixel gap.
