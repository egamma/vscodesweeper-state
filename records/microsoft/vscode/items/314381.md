# Issue #314381 — Integrated terminals can inherit env vars from a previously opened repository/session

[https://github.com/microsoft/vscode/issues/314381](https://github.com/microsoft/vscode/issues/314381)

## Summary

The report describes cross-workspace environment pollution via direnv when launching new VS Code windows from an already-active session. This is at least partially by design (VS Code inherits its parent process environment, and child windows share that process tree), but a clear reproduction against current main with specific variable names/values would help distinguish a genuine leak from expected process inheritance. Requesting more info.

## Change summary

Integrated terminals in a second VS Code window inherit direnv-set environment variables from the first window's workspace, contaminating unrelated projects.

## Best solution

Clarify whether the reporter has `terminal.integrated.inheritEnv` enabled (default: true) and whether the issue persists with it disabled. If confirmed as a real gap beyond process inheritance, this likely belongs in the terminal-process area as a feature request for per-workspace environment isolation, potentially related to https://github.com/microsoft/vscode/issues/47816.

## Evidence

- **inheritEnv setting:** The `terminal.integrated.inheritEnv` setting (src/vs/platform/terminal/common/terminalPlatformConfiguration.ts) controls whether new shells inherit from VS Code's resolved environment. Default is true.
- **process architecture:** When `code .` is run from a direnv-activated shell, the VS Code process inherits those env vars. New windows reuse the same app process tree, so terminals in subsequent windows inherit the original process environment.
- **related issue:** https://github.com/microsoft/vscode/issues/319200 is about terminal restoration using wrong CWD — different root cause (session restoration vs env inheritance).
- **partial overlap:** https://github.com/microsoft/vscode/issues/47816 requests fresh environment blocks for new terminals on Windows — related concept but different platform/mechanism.
