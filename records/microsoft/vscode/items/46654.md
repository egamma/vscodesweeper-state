[https://github.com/microsoft/vscode/issues/46654](https://github.com/microsoft/vscode/issues/46654)

## Summary

Keep open: the report still matches the current folding command flow, where every `FoldingAction` reveals the active selection after running, including `Fold All Block Comments`. The issue is already routed to `editor-folding` and needs an editor folding decision rather than cleanup closure.

## Change summary

Running `Fold All Block Comments` scrolls the editor back to the cursor even when the cursor was intentionally out of view.

## Best solution

Decide in the editor folding area whether fold-all style commands should preserve the previous scroll position, or only reveal the cursor when the command actually hides or changes the selected line.
