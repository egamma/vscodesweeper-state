# Issue #46654 — [folding] Fold block comments scrolls editor

[https://github.com/microsoft/vscode/issues/46654](https://github.com/microsoft/vscode/issues/46654)

## Summary

Still reproducible on main: `FoldingAction.runEditorCommand` unconditionally calls `foldingController.reveal(selection.getStartPosition())` after every folding action, scrolling the editor to the cursor even when it was intentionally out of view.

## Change summary

Folding block comments (or any fold action) scrolls the editor back to the cursor position even when the cursor was deliberately scrolled out of view.

## Best solution

Guard the `reveal` call in `FoldingAction.runEditorCommand` (line 564 of `src/vs/editor/contrib/folding/browser/folding.ts`) so it only reveals when the cursor is inside a newly-hidden (folded) range, not unconditionally after every fold action.
