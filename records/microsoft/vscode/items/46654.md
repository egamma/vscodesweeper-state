# Issue #46654 — [folding] Fold block comments scrolls editor

[https://github.com/microsoft/vscode/issues/46654](https://github.com/microsoft/vscode/issues/46654)

## Summary

Still reproduces on main: FoldingAction base class unconditionally reveals the cursor position after every invoke(), even for bulk actions like 'Fold All Block Comments' where the cursor isn't hidden by a fold.

## Change summary

Running 'Fold All Block Comments' scrolls the editor to reveal the cursor even when the cursor was intentionally scrolled out of view.

## Best solution

The `reveal()` call in `FoldingAction.runEditorCommand` (line 564 of src/vs/editor/contrib/folding/browser/folding.ts) should only fire when the cursor is actually inside a region that was just collapsed — not unconditionally after every fold action.
