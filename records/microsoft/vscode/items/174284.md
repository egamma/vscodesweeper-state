# Issue #174284 — Minimap on the side not updating correctly when using keyboard shortcuts to move cursor to top of file

[https://github.com/microsoft/vscode/issues/174284](https://github.com/microsoft/vscode/issues/174284)

## Summary

This appears fixed on current main by a concrete minimap slider calculation change; the reported top-of-file case matches the old scrollTop == paddingTop divide-by-zero path.

## Change summary

The minimap hover slider could stay visually at the bottom after keyboard navigation moved the editor viewport back to the top.

## Best solution

Treat this as completed by commit 6c8d3ea05da63f9769f0b548ec5244afc3011ba0, which shipped in VS Code 1.81.0; no additional triage label is needed.
