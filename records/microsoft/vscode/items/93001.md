# Issue #93001 — resolving multiple tasks with type npm and same script fails

[https://github.com/microsoft/vscode/issues/93001](https://github.com/microsoft/vscode/issues/93001)

## Summary

Still reproduces on main: `createTaskIdentifier` in tasks.ts builds the key from only the task definition's declared properties (for npm tasks: `type` and `script`), so two tasks with the same script but different labels, cwd, or options collide and only one survives. This is the canonical issue — all related reports (#142329, #227237, #205471) were closed as duplicates of it. No fix has been merged.

## Change summary

Multiple npm tasks sharing the same script but differing in label, cwd, or options are deduplicated to a single task because the task identifier key only includes the task definition's required/declared properties.

## Best solution

Include additional differentiating properties (at minimum `options.cwd`, and ideally the user-assigned `label`) in the key built by `KeyedTaskIdentifier.create` within `TaskDefinition.createTaskIdentifier` (src/vs/workbench/contrib/tasks/common/tasks.ts ~line 1400). The npm task definition's `path` property already partially addresses multi-folder scenarios, but the identifier must also account for tasks in the same folder with different options. Care is needed to avoid breaking task identity for persisted task state and `dependsOn` resolution.
