# Issue #93001 — resolving multiple tasks with type npm and same script fails

[https://github.com/microsoft/vscode/issues/93001](https://github.com/microsoft/vscode/issues/93001)

## Summary

Still reproducible on main — the task identifier key is built only from schema-defined properties (e.g. `script` for npm), so tasks differing only in `options`/`label` collide. Kept open per On Deck milestone.

## Change summary

Multiple npm tasks with the same script but different labels/options/cwd are deduplicated and only one is resolvable.

## Best solution

Include `options` (particularly `cwd` and `env`) and/or `label` in the task identity key computation in `src/vs/workbench/contrib/tasks/common/tasks.ts` (`KeyedTaskIdentifier.create` / `createTaskIdentifier`), or allow contributed task definitions to declare additional distinguishing properties. The fix is localized to the task identity logic in `src/vs/workbench/contrib/tasks/common/tasks.ts` and `taskConfiguration.ts`.
