# Issue #131494 — [scss] Hover hint doesn't support SCSS Interpolation syntax

[https://github.com/microsoft/vscode/issues/131494](https://github.com/microsoft/vscode/issues/131494)

## Summary

Valid feature request for better SCSS interpolation handling in selector hover hints. The logic lives in the upstream vscode-css-languageservice package (currently v7.0.0-next.1), and no fix has landed there or in the VS Code repo. Low community traction (1 comment) but the request is legitimate and already triaged to Backlog.

## Change summary

SCSS hover hints should resolve interpolation syntax (e.g. `#{&}__paragraph`) to the computed selector rather than showing the raw interpolation token.

## Best solution

Implement interpolation-aware selector resolution in vscode-css-languageservice's hover/selector-printing logic so that `#{&}` references are expanded when building the hover tooltip. The fix belongs upstream in https://github.com/microsoft/vscode-css-languageservice, not in this repo directly.
