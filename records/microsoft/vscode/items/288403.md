# Issue #288403 — Git Error with Submodule

[https://github.com/microsoft/vscode/issues/288403](https://github.com/microsoft/vscode/issues/288403)

## Summary

The report describes unhandled promise rejections from `checkIgnore` when files are in a git submodule — a known pattern (see two prior closed reports). The code partially handles this in `decorationProvider.ts` but `quickDiffProvider.ts` does not catch the `IsInSubmodule` error. However, the report lacks a VS Code version, exact reproduction steps, and confirmation on the latest Insiders build.

## Change summary

Opening files in a git submodule floods the developer console with unhandled `IsInSubmodule` Git errors from the built-in Git extension.

## Best solution

The `checkIgnore` callers in `extensions/git/src/quickDiffProvider.ts` and `extensions/git/src/decorationProvider.ts` should gracefully handle the `IsInSubmodule` rejection (resolve with an empty set or `undefined` instead of rejecting). The area label is `git`. Two prior reports (#197396, #208448) were closed without a fix; this one should collect repro confirmation on the latest Insiders build first.
