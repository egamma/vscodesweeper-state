# Issue #171033 — Trim Trailing Whitespace does not trim whitespace before cursor on autoSave

[https://github.com/microsoft/vscode/issues/171033](https://github.com/microsoft/vscode/issues/171033)

## Summary

This is intentional behavior: when autosave fires, VS Code deliberately skips trimming whitespace on cursor lines to prevent the cursor from jumping (fix for #18410). The reporter's request — trim on disk but keep in editor — would require the saved file to diverge from the editor model, which is a fundamental design boundary.

## Change summary

Trailing whitespace on the cursor's line is not trimmed during autosave, causing it to persist on disk until the cursor moves away.

## Best solution

Close as designed. The current behavior was an intentional fix (commit c936b0f6f59) to prevent cursor jumping during autosave. The proposed alternative (save-to-disk differs from editor state) conflicts with VS Code's model where the editor and disk stay in sync after a save. Users who need clean commits can use a pre-commit hook or formatOnSave with an explicit save.
