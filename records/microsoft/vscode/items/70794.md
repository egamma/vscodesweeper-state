This feature request for custom collapsed text in folding ranges is not yet implemented on main. The FoldingRange interface still only has start/end/kind — no collapsedText property. With 38 upvotes it exceeds the Backlog threshold and should remain open.

## Change summary

Allow FoldingRangeProvider to specify custom display text shown when a range is collapsed, instead of the default '...' placeholder.

## Best solution

Implement collapsedText support in the FoldingRange API (add optional string property) and wire it through foldingDecorations.ts to replace the CSS :after '…' with provider-supplied text. PR https://github.com/microsoft/vscode/pull/170447 is an existing attempt that could be reviewed/landed.
