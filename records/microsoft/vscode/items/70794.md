# Issue #70794 — [folding] custom folding text for folded ranges

[https://github.com/microsoft/vscode/issues/70794](https://github.com/microsoft/vscode/issues/70794)

## Summary

This feature request for custom collapsed text on folding ranges remains unimplemented on main. The FoldingRange API still only exposes start, end, and kind — no collapsedText or equivalent property exists. The issue is in the Backlog milestone with maintainer engagement and community interest.

## Change summary

Allow FoldingRangeProvider to specify custom display text shown when a range is collapsed, instead of the default '…' placeholder.

## Best solution

Extend the FoldingRange API (and internal FoldingRegion model) with an optional `collapsedText` property that providers can set. The folding decoration in `src/vs/editor/contrib/folding/browser/foldingDecorations.ts` would need to render this text in place of (or alongside) the CSS `:after` ellipsis. A proposed API stage is appropriate given the design question (replace whole line vs replace ellipsis only).
