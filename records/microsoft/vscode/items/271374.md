# Issue #271374 — Code Review: "Illegal value for `line`" error after CCR in SCM

[https://github.com/microsoft/vscode/issues/271374](https://github.com/microsoft/vscode/issues/271374)

## Summary

Team-authored issue; bug is still present on current main. The `createReviewComment` function does not validate/clamp the `line` number from the API response before calling `document.lineAt()`, causing a throw when the API returns a comment whose line is out of range (e.g. with `exclusion_reason: "no_matched_diff_hunk"`).

## Change summary

Running Copilot Code Review in SCM throws "Illegal value for `line`" when the API returns a comment referencing a line number outside the document's range.

## Best solution

Guard `createReviewComment` in `extensions/copilot/src/extension/review/node/githubReviewAgent.ts` to clamp `ghComment.data.line` to `[1, document.lineCount]` before calling `document.lineAt()`. Additionally, the `ExcludedComment.exclusion_reason` type union should be widened to include `'no_matched_diff_hunk'` (or use `string`) so the type matches what the API actually returns.
