# Issue #315739 — New session view should be in history

[https://github.com/microsoft/vscode/issues/315739](https://github.com/microsoft/vscode/issues/315739)

## Summary

Team-authored feature request for the Agents window: the 'new session' view should participate in navigation history so users can go back/forward to it. Keeping open per team-member authorship rule.

## Change summary

Starting a new session in the Agents window is not recorded in navigation history, so going 'Back' loses the in-progress prompt with no way to go 'Forward' to it.

## Best solution

Add the new-session view as a history entry in the Agents window navigation stack so that back/forward navigation includes it, likely in `src/vs/sessions/` history or navigation infrastructure.
