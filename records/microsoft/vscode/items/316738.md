## Summary

Valid issue: the repo uses git-lfs for files under extensions/copilot/test/simulation/cache/ but the devcontainer does not install git-lfs, so git operations (branching, checkout) can fail inside the container. An open PR (#316739) already addresses this.

## Change summary

The VS Code devcontainer is missing git-lfs, which is required since the repo tracks LFS objects (sqlite files in extensions/copilot).

## Best solution
