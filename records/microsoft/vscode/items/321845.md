# Issue #321845 — Source control tab showing incorrect branch (Capitalization)

[https://github.com/microsoft/vscode/issues/321845](https://github.com/microsoft/vscode/issues/321845)

## Summary

Real bug: when two branches differ only in case on a case-insensitive filesystem (Windows/macOS), `getBranch()` uses `--ignore-case` and picks the first result from `for-each-ref`, which may not match the actual HEAD. The source control panel then shows the wrong branch name.

## Change summary

Source control panel displays the wrong branch name when two branches exist that differ only in capitalization on Windows/macOS.

## Best solution

In `extensions/git/src/git.ts` `getBranch()`, when `--ignore-case` returns multiple results, prefer the one whose name exactly matches the input `name` parameter (from `getHEAD()`/`getHEADFS()`) before falling back to the first result. This preserves the case-insensitive fallback while fixing the ambiguous-match scenario.
