## Summary

Old report (Jan 2023, VS Code 1.74) of a ~10s delay when running a process with a 1GB stack reservation in the integrated terminal on Windows. Likely upstream conpty behavior. No confirmation on current builds, and the terminal/conpty layer has had significant changes since then — asking if it still reproduces.

## Change summary

Running a C++ binary compiled with a very large stack size (~1GB) in the integrated terminal on Windows causes a multi-second startup delay not seen in an external console.
