# Issue #169296 — Render quick input separators as individual items instead of applying border to list item

[https://github.com/microsoft/vscode/issues/169296](https://github.com/microsoft/vscode/issues/169296)

## Summary

Team-authored polish request to render quick input separators as distinct list items rather than borders on adjacent items, improving full-width rendering and spacing. Kept open per hard keep-open rule (author is MEMBER).

## Change summary

Quick pick separators should be rendered as standalone items in the list so they can span the full width and allow height/spacing adjustments.

## Best solution

Refactor the quick pick list rendering in `src/vs/platform/quickinput/browser/` to emit separators as their own list entries (non-selectable) instead of applying a CSS border-top to the following item. This would fix the padding gap, allow variable separator height, and eliminate the hover/focus collision with the border.
