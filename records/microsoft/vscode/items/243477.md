# Issue #243477 — Variables/Watch 'F2' for 'Set Value' is broken

[https://github.com/microsoft/vscode/issues/243477](https://github.com/microsoft/vscode/issues/243477)

## Summary

Still reproduces on current main. The shared `selectedExpression` slot in the debug view model combined with `lastFocusedList` resolution causes F2 to target stale or cross-view elements in the Variables and Watch panels.

## Change summary

Pressing F2 in the Variables or Watch debug views targets the wrong element, allows editing where it shouldn't, or silently sets state in the other view.

## Best solution

The F2 handlers in `debugCommands.ts` (lines ~833 and ~868) both use `listService.lastFocusedList` to resolve the target element, and both share a single `selectedExpression` slot in `debugViewModel.ts`. The fix should scope expression selection per-view (or clear stale selections when focus changes between views), and validate that `lastFocusedList` actually corresponds to the expected view before acting. Likely files: `debugCommands.ts`, `debugViewModel.ts`, and possibly `baseDebugView.ts`.
