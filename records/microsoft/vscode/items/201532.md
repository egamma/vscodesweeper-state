# Issue #201532 — context menu display on another monitor

[https://github.com/microsoft/vscode/issues/201532](https://github.com/microsoft/vscode/issues/201532)

## Summary

Confirmed upstream Electron/macOS issue with native context menu positioning on multi-monitor setups. The bug is still present on current main — VS Code's `menu.popup()` call in `src/vs/base/parts/contextmenu/electron-main/contextmenu.ts` does not pass a `window` parameter, and Electron/macOS misplaces the menu on the wrong screen. The issue is correctly labeled and should remain open.

## Change summary

Right-clicking in VS Code on a secondary monitor causes the native context menu to appear on the primary monitor instead.

## Best solution

The root cause is in Electron's `Menu.popup()` or macOS's native menu positioning when no explicit window is provided. A potential mitigation in VS Code would be to pass the `window` (BrowserWindow) parameter to `menu.popup()` in `src/vs/base/parts/contextmenu/electron-main/contextmenu.ts`, which could help Electron anchor the menu to the correct screen. However, the core fix belongs upstream in Electron. The `help wanted` label is appropriate — a contributor could investigate whether passing the window reference resolves the positioning.
