# Issue #30065 — Language services should be able to provide debug console IntelliSense

[https://github.com/microsoft/vscode/issues/30065](https://github.com/microsoft/vscode/issues/30065)

## Summary

Team-authored feature request (MEMBER). The debug console still only delegates completions to the debug adapter via `supportsCompletionsRequest`; there is no mechanism for a language service to provide completions instead. The requested capability remains unimplemented on main.

## Change summary

Allow language services (not just debug adapters) to provide IntelliSense in the debug console, enabling statically typed languages like C# to reuse their existing completion providers.

## Best solution

Add an opt-in mechanism so that when a language service declares support for debug console completions, VS Code routes the REPL completion request to the language service (with stack-frame context) instead of — or in addition to — the debug adapter. The completion provider registration in `src/vs/workbench/contrib/debug/browser/repl.ts` and possibly the debug protocol types in `debugProtocol.d.ts` would need extension. Needs a design discussion on the API shape between the debug and language-service teams.
