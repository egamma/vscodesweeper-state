# Issue #323359 — Opening a link from a staged .md file causes the wrong document to open

[https://github.com/microsoft/vscode/issues/323359](https://github.com/microsoft/vscode/issues/323359)

## Summary

Legitimate bug: when a markdown file is opened from Source Control's staged changes (git: scheme), clicking a relative link resolves with the git: scheme but inherits the original file's query parameters, causing the git file system provider to serve the markdown file's content instead of the linked file's content. Still present on main.

## Change summary

Opening a link from a staged markdown file (git: scheme) shows the markdown file's content in the linked file's tab instead of the linked file's actual content.

## Best solution

The markdown link resolution (in the markdown language service or the preview's link-click handler) needs to detect when `fromResource` uses the `git:` scheme and either (a) resolve the link target to a `file:` scheme URI instead of preserving the `git:` scheme, or (b) construct a proper git URI for the linked file with the correct query parameters (path pointing to the linked file, not the source markdown). The fix likely lives in `extensions/markdown-language-features/src/util/openDocumentLink.ts` or in the markdown language service's link resolution, and possibly `extensions/git/src/uri.ts`'s `toGitUri`.

- **no scheme handling:** extensions/markdown-language-features/src/util/openDocumentLink.ts and extensions/markdown-language-features/src/preview/preview.ts show no special handling for git: scheme URIs — the fromResource is passed as-is to the language service for link resolution.
