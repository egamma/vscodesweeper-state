# Issue #311917 — [Error] potential listener LEAK — textModel / codeBlockPart (chat code block rendering)

[https://github.com/microsoft/vscode/issues/311917](https://github.com/microsoft/vscode/issues/311917)

## Summary

Still present on main: the EditorPool/KeyedResourcePool has no cap on in-use CodeBlockPart instances, so chat sessions with many visible code blocks accumulate listeners on the shared `_languageConfigurationService.onDidChange` emitter past the warning threshold. An unmerged branch (`codex/chat-listener-leaks-upstream`) exists with partial work but has not landed.

## Change summary

Rendering many chat code blocks simultaneously causes a listener LEAK warning because each pooled CodeBlockPart creates a TextModel that subscribes to a shared global emitter with no in-use cap.

## Best solution

Cap or lazily manage in-use pool items so the total listener count on shared emitters stays below the threshold, similar to how PR #311747 deferred toolbar creation. The partial work on the `codex/chat-listener-leaks-upstream` branch should be completed and merged. Likely files: `chatCollections.ts` (pool logic), `codeBlockPart.ts` (TextModel creation), and possibly `chatContentCodePools.ts`.
