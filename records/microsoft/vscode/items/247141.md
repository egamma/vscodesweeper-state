# Issue #247141 — [Bug] “Transform to” actions don't function on text selected inside the Monaco F&R modal.

[https://github.com/microsoft/vscode/issues/247141](https://github.com/microsoft/vscode/issues/247141)

## Summary

Still reproduces on main — the transform-case actions operate on the main editor's selections, so they silently do nothing when focus/selection is in the Find widget input. An open PR (https://github.com/microsoft/vscode/pull/307247) is addressing this.

## Change summary

"Transform to Uppercase/Lowercase/etc." commands silently fail when text is selected inside the Find & Replace widget input rather than in the main editor.

## Best solution

Land the open PR https://github.com/microsoft/vscode/pull/307247 (or an equivalent fix) that teaches the transform-case actions to detect when the Find widget input is focused and apply the transformation there. The fix likely touches `src/vs/editor/contrib/linesOperations/browser/linesOperations.ts` (the `AbstractCaseAction.run` method) and possibly the find widget to expose its input selection.
