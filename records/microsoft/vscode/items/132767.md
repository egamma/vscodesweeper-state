# Issue #132767 — Support `dependsOn` programmatically in the vscode.Task API

[https://github.com/microsoft/vscode/issues/132767](https://github.com/microsoft/vscode/issues/132767)

## Summary

Still a valid gap: `dependsOn` and `dependsOrder` are not exposed on the `vscode.Task` API. The properties exist internally for tasks.json parsing but are absent from `vscode.d.ts` and all proposed API files. 31 upvotes place it comfortably in the Backlog.

## Change summary

Allow extensions to set `dependsOn` and `dependsOrder` on programmatically created tasks via the vscode.Task API.

## Best solution

Expose `dependsOn` (as an array of task references) and `dependsOrder` on the `Task` class in the extension API, wire the values through `extHostTask.ts` → `mainThreadTask.ts` into the existing task-dependency resolution in `terminalTaskSystem.ts`. Start as a proposed API in `src/vscode-dts/vscode.proposed.taskDependsOn.d.ts`.
