# Issue #203240 — Allow Extensions to contribute to DebugHoverContext/DebugWatchContext

[https://github.com/microsoft/vscode/issues/203240](https://github.com/microsoft/vscode/issues/203240)

## Summary

Team-authored feature request to expose DebugHoverContext (and possibly enhance DebugWatchContext) as extension contribution points. DebugHoverContext is not yet exposed in menusExtensionPoint.ts; DebugWatchContext is exposed but may lack full extension command contribution support. Keeping open per team-member rule.

## Change summary

Allow extensions to contribute commands to the debug hover and watch context menus.

## Best solution

Expose `DebugHoverContext` as an extension menu contribution point in `src/vs/workbench/services/actions/common/menusExtensionPoint.ts` (similar to how `DebugWatchContext` is already registered at `debug/watch/context`), and ensure extensions can register custom commands in both menus via the `contributes.menus` extension point.
