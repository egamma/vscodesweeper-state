# Issue #239567 — Closing editor scrolls to the open editor

[https://github.com/microsoft/vscode/issues/239567](https://github.com/microsoft/vscode/issues/239567)

## Summary

Verified duplicate of #201209, which describes the same problem — closing an editor in the Open Editors view scrolls the list to the active editor. The older issue is assigned and tracks the same root cause.

## Change summary

Closing a non-active editor in the Open Editors view scrolls the list to reveal the active editor, forcing the user to scroll back to continue closing other editors.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/201209, which is already assigned and tracks the same underlying bug: `focusActiveEditor()` in `openEditorsView.ts` unconditionally calls `list.reveal(index)` after the list refreshes on editor close.
