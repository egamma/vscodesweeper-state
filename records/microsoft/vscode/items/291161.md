# Issue #291161 — Agent sessions filtering is very annoying when filtering multiple providers

[https://github.com/microsoft/vscode/issues/291161](https://github.com/microsoft/vscode/issues/291161)

## Summary

Team-authored issue; the agent sessions filter uses a standard context menu (via MenuId) that closes on each selection, forcing users to reopen it repeatedly when toggling multiple provider filters. Still reproduces on current main — the filter mechanism hasn't changed.

## Change summary

Agent sessions filter dialog closes after each provider toggle, requiring repeated reopening to filter multiple providers.

## Best solution

Replace the context menu–based filter with a multi-select quickpick (as suggested in the comments, referencing https://github.com/microsoft/vscode/issues/281051) or modify the context menu to stay open on toggle actions. The relevant code is in `src/vs/workbench/contrib/chat/browser/agentSessions/agentSessionsFilter.ts` (provider action registration) and the sessions-window counterpart in `src/vs/sessions/contrib/sessions/browser/views/sessionsViewActions.ts`.
