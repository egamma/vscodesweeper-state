# Issue #213701 — Multi-select should fire one tabs API event

[https://github.com/microsoft/vscode/issues/213701](https://github.com/microsoft/vscode/issues/213701)

## Summary

Team-authored debt issue, still valid: `$acceptTabOperation` in `extHostEditorTabs.ts` fires one `onDidChangeTabs` event per tab operation with no batching, so a multi-select close fires N individual events instead of one aggregated event.

## Change summary

Multi-select tab operations (e.g. closing several tabs at once) should fire a single batched `onDidChangeTabs` event instead of one event per tab.

## Best solution

Buffer tab operations arriving in the same microtask/tick in `ExtHostEditorTabs.$acceptTabOperation` and coalesce them into a single `onDidChangeTabs` fire with the full arrays. The main-thread side (`mainThreadEditorTabs.ts`) may also need adjustment to send a batch IPC call rather than N individual `$acceptTabOperation` calls.
