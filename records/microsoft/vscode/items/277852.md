# Issue #277852 — Increase visual hierarchy of agent selection

[https://github.com/microsoft/vscode/issues/277852](https://github.com/microsoft/vscode/issues/277852)

## Summary

Team-authored UX feature request to add color differentiation to non-default mode/agent selections in the chat input area. Not yet implemented on main — the mode picker (`modePickerActionItem.ts`) and its base class (`chatInputPickerActionItem.ts`) show no color/foreground styling for non-default states. Keeping open per team-author policy.

## Change summary

Make non-default agent/mode selections in the chat input visually distinct with color-coded foregrounds (teal for Ask/Plan/Edit, purple for custom agents, grey for Agent).

## Best solution

Implement color-coded foreground styling in the mode picker action item (`src/vs/workbench/contrib/chat/browser/widget/input/modePickerActionItem.ts` and related CSS) to visually distinguish non-default mode selections, reducing accidental submissions to the wrong mode. Coordinate with the related prompt-area redesign tracked in https://github.com/microsoft/vscode/issues/272502.
