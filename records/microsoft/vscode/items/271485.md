Legitimate feature request that earned 20+ community upvotes and was promoted to Backlog. Despite a team member's earlier reluctance, the community process elevated it, so it should remain open for future consideration.

## Change summary

Add preferredDarkIconTheme / preferredLightIconTheme settings so icon themes auto-switch with system appearance, matching the existing color-theme behavior.

## Best solution

Add `workbench.preferredDarkIconTheme` and `workbench.preferredLightIconTheme` settings in the themes area, mirroring how `workbench.preferredDarkColorTheme` / `workbench.preferredLightColorTheme` already work. The implementation likely lives near `src/vs/workbench/services/themes/` where auto-detection and switching logic already exists. An existing PR (https://github.com/microsoft/vscode/pull/232336) may serve as a starting point.
