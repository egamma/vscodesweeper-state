## Evidence

- **binary conflict root cause:** The merge-conflict extension (extensions/merge-conflict/src/commandHandler.ts) resolves conflicts by parsing text-based conflict markers (`<<<<<<<`, `=======`, `>>>>>>>`). Binary files don't contain these markers, so `getConflicts()` returns nothing and the accept commands silently no-op.
- **auto-save/stage is by design:** The acceptAllResources method (commandHandler.ts:275) applies a WorkspaceEdit but intentionally does not save or stage — this is standard VS Code behavior where edits leave the file dirty for user review.
