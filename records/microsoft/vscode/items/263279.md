# Issue #263279 — Agent Mode: Disable tools by default besides core tools

[https://github.com/microsoft/vscode/issues/263279](https://github.com/microsoft/vscode/issues/263279)

## Summary

Team-authored feature request under active discussion among members and contributors. No implementation of default-disabled tools or per-MCP default configuration exists on main yet. The discussion highlights a real usability tension (tool discoverability vs. tool overload) that the team hasn't resolved — keep open for a product decision.

## Change summary

Request to disable non-built-in tools by default in agent mode, with an easy toggle and a tool-count indicator in the UI.

## Best solution

The team needs to resolve the usability tension raised by connor4312 (breaking existing MCP/extension setup assumptions) vs. the user-facing clutter problem. A configurable default — possibly via a setting or per-MCP `default: false` flag in mcp.json as commenters suggest — would let power users manage tool sprawl without breaking onboarding flows. The tool selection UI in `src/vs/workbench/contrib/chat/browser/actions/chatToolPicker.ts` and enablement logic in `src/vs/workbench/contrib/chat/browser/widget/input/chatSelectedTools.ts` are the likely implementation areas.
