# Issue #259720 — Unable to open file error when there are two same name workspace in a project & Open search result in editor

[https://github.com/microsoft/vscode/issues/259720](https://github.com/microsoft/vscode/issues/259720)

## Summary

Verified duplicate of the confirmed canonical issue #198159, which tracks the same search editor navigation bug with same-named workspace folders. The root cause is in `extensions/search-result/src/extension.ts` where `relativePathToUri` always picks the first workspace folder matching by name.

## Change summary

In a multi-root workspace with identically named folders, Ctrl+clicking a search editor result resolves to the wrong folder, producing an "Unable to open file" error.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/198159, which is confirmed and milestoned. The fix is in `extensions/search-result/src/extension.ts` — `relativePathToUri` must disambiguate same-named workspace folders (e.g. by matching against the full URI or using workspace folder index rather than just `wf.name`).
