# Issue #257673 — Long bg commands or tasks without begins/ends patterns might be considered idle when they're not

[https://github.com/microsoft/vscode/issues/257673](https://github.com/microsoft/vscode/issues/257673)

## Summary

Team-authored design discussion about improving idle-detection for long-running terminal commands without begin/end task patterns. Kept open per team-author rule; the discussion is active with concrete solution ideas.

## Change summary

Long-running background terminal commands (like `npm run compile`) may be prematurely detected as idle because there are gaps between output events, even though the command is still running.

## Best solution

Implement a timeout upper-bound (e.g. 2 minutes) after which the mechanism falls back to on-demand output retrieval, and suggest creating a task with problem matchers when this case is hit. The create-and-run-task tool already supports begin/end patterns; encouraging their use would solve the root cause for known commands.
