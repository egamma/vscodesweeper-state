# Issue #200270 — Do not autoplay videos in release notes

[https://github.com/microsoft/vscode/issues/200270](https://github.com/microsoft/vscode/issues/200270)

## Summary

Valid, well-supported feature request (35 upvotes) with an open PR in microsoft/vscode-docs. The fix lives in vscode-docs (HTML video attributes), not this repo's code, but the issue also affects the in-editor release notes webview. Keeping open as the tracking issue while the PR is reviewed.

## Change summary

Remove autoplay and loop attributes from video elements in release notes so videos don't play automatically when the page loads or scrolls.

## Best solution

Land the open PR in microsoft/vscode-docs (https://github.com/microsoft/vscode-docs/pull/9931) which removes the autoplay/loop attributes. The in-editor release notes consume the same markdown content, so both surfaces benefit. No code change needed in this repo — the fix is purely in the docs/content layer.
