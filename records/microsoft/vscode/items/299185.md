# Issue #299185 — Error when trying to save a plan

[https://github.com/microsoft/vscode/issues/299185](https://github.com/microsoft/vscode/issues/299185)

## Summary

The error originates from the GitHub Copilot extension's 'plan mode' feature (v0.37.9) writing a file to the root path `/plan-filterRefactor.prompt.md` instead of a valid workspace-relative location. This is a Copilot extension bug, not a VS Code core issue.

## Change summary

Saving a plan file created by Copilot's plan mode fails with EROFS because the extension targets the root filesystem path `/`.

## Best solution

This should be reported and tracked in the GitHub Copilot extension's issue tracker (https://github.com/microsoft/vscode-copilot-release/issues). The fix belongs in the extension's plan-file creation logic to use a proper workspace-relative or user-writable path.
