# Issue #299185 — Error when trying to save a plan

[https://github.com/microsoft/vscode/issues/299185](https://github.com/microsoft/vscode/issues/299185)

## Summary

This is a bug in the GitHub Copilot extension's plan agent, which creates an untitled file with a URI that resolves to the root filesystem `/` on save. The Copilot extension (v0.37.9) instructs the agent to use `untitled:plan-${camelCaseName}.prompt.md` which lacks a proper workspace-relative path, causing macOS's read-only root FS error.

## Change summary

Saving a plan file created by the Copilot extension's plan mode fails with EROFS because the untitled URI resolves to the root filesystem `/`.

## Best solution

The fix belongs in the Copilot extension's `planAgentProvider.ts` — it should use a workspace-relative path in the untitled URI (e.g., `untitled:${workspaceFolder}/plan-${name}.prompt.md`) so the save dialog defaults to a writable location. Report this to the Copilot extension team.
