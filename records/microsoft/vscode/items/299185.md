# Issue #299185 — Error when trying to save a plan

[https://github.com/microsoft/vscode/issues/299185](https://github.com/microsoft/vscode/issues/299185)

## Summary

The plan agent's "Open in Editor" handoff instructs the LLM to create a file at `untitled:plan-${camelCaseName}.prompt.md`, but the resulting file resolves its save path to the root filesystem (`/plan-filterRefactor.prompt.md`), producing an EROFS error on macOS. The handoff instruction is still present in current main (`planAgentProvider.ts` line 236), so the bug likely persists. Routing to the plan agent area.

## Change summary

Saving a plan file created via the "Open in Editor" handoff fails with EROFS because the path resolves to the root filesystem instead of the workspace or a user-chosen location.

## Best solution

Fix the `openInEditorHandoff` in `extensions/copilot/src/extension/agents/vscode-node/planAgentProvider.ts` so the `#createFile` instruction produces a properly-resolvable untitled URI or uses a workspace-relative path, ensuring the save operation targets a writable location (e.g. the workspace folder or triggers a Save As dialog).
