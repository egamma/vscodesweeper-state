# Issue #202860 — Zero width character highlighting broken for emojis containing ZWJs

[https://github.com/microsoft/vscode/issues/202860](https://github.com/microsoft/vscode/issues/202860)

## Summary

Still reproduces on main: the unicode highlighter flags ZWJ as an invisible character inside valid emoji sequences because it doesn't check whether the character is part of an emoji ZWJ sequence. A TODO comment in the code explicitly acknowledges this gap.

## Change summary

ZWJ characters inside valid emoji sequences (e.g. 🧑‍🦰) are incorrectly highlighted as invisible characters when followed by other content on the same line.

## Best solution

In `src/vs/editor/common/services/unicodeTextModelHighlighter.ts` around line 223–226, add a check that skips invisible-character highlighting when the ZWJ is part of a valid emoji sequence (using the existing `isEmojiImprecise` and grapheme-cluster utilities in `src/vs/base/common/strings.ts`). The existing `// TODO check for emojis` comment marks the exact insertion point.
