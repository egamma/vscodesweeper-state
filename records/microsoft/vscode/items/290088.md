# Issue #290088 — Hyprland DE is not being detected by os_crypt

[https://github.com/microsoft/vscode/issues/290088](https://github.com/microsoft/vscode/issues/290088)

## Summary

This is an upstream Chromium/Electron issue: the os_crypt component in Chromium (key_storage_util_linux.cc) does not recognize Hyprland as a desktop environment, so it cannot select the correct password storage backend. VS Code's own desktopEnvironmentInfo.ts also lacks Hyprland but is used only for telemetry, not credential storage. The fix belongs in Chromium upstream.

## Change summary

Credential storage fails on Hyprland because Chromium's os_crypt does not recognize it as a desktop environment.

## Best solution

The root cause is in Chromium's `components/os_crypt/sync/key_storage_util_linux.cc`, which needs to add Hyprland detection. A Chromium upstream bug should be filed (or linked if one exists). As a workaround, users can set `--password-store=gnome-libsecret` or `--password-store=basic` via the `password-store` CLI flag. VS Code could also consider adding Hyprland to its own `desktopEnvironmentInfo.ts` for telemetry purposes and potentially mapping it to an appropriate password store hint.
