# Issue #154156 — use "current working directory detection" in the implementation of "runInTerminal"

[https://github.com/microsoft/vscode/issues/154156](https://github.com/microsoft/vscode/issues/154156)

## Summary

Still unimplemented on main: `prepareCommand` in `terminals.ts` unconditionally emits `cd <cwd>` for every debug launch without checking whether the terminal's current working directory already matches. The debt item remains valid.

## Change summary

Skip the unnecessary `cd <cwd>` command in `runInTerminal` when the terminal's current working directory already matches the desired directory.

## Best solution

In the debug terminal launch path, query the terminal's detected cwd (via the terminal capability `cwd` or `initialCwd`) before calling `prepareCommand`, and pass `cwd` as `undefined` when it already matches the target. The change would touch `src/vs/workbench/contrib/debug/node/terminals.ts` (the `prepareCommand` caller) and potentially the `runInTerminal` handler in `src/vs/workbench/contrib/debug/browser/rawDebugSession.ts` or the debugger contribution that wires the terminal instance.
