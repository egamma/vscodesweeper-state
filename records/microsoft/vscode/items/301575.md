# Issue #301575 — Copilot Chat #classname #methodname intellisense not working in C# projects

[https://github.com/microsoft/vscode/issues/301575](https://github.com/microsoft/vscode/issues/301575)

## Summary

The `#` symbol completions in chat are sourced from cached outline models (symbols from open/recently-visited files), not a full workspace symbol search. The user expects all C# classes/methods to appear, but the current design intentionally scopes to cached outlines. Additionally, the quality of those symbols depends on C# DevKit's language server, making this partly an extension concern. Proposing close as caused-by-extension since the symbol provider is C# DevKit, and the limited scope is by design.

## Change summary

Typing `#` in Copilot Chat only shows a subset of C# classes and methods, not all symbols in the workspace.

## Best solution

The limited symbol list is a combination of VS Code's design (chat completions use cached outline models from open files, not workspace-wide symbol search — see `addSymbolEntries` in `chatInputCompletions.ts`) and C# DevKit's symbol provisioning. If the user wants broader coverage, they should open the relevant files first. A feature request for workspace-wide `#sym:` completions would be a separate, distinct ask. The current report is better directed at C# DevKit if specific open-file symbols are missing.
