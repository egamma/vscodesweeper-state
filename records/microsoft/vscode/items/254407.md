## Summary

Still open on main: the original fix (commit 2f09f40b0b7) was reverted (commit 35eddf193fd) due to a regression, and no re-landing has occurred. The issue carries the 'On Deck' milestone, which is a scheduling decision — keeping open per policy.

## Change summary

The `git.detectSubmodules` setting is not honored when opening a workspace from a workspace file, causing submodule repositories to appear in Source Control despite the setting being false.

## Best solution

Re-implement the fix from the reverted PR https://github.com/microsoft/vscode/pull/257813 in a way that avoids the regression that caused the revert. The setting check needs to be applied in the workspace-file code path in `extensions/git/src/model.ts`, likely around the `scanWorkspaceFolders` / `openRepository` flow, so that `detectSubmodules` is consulted before submodule repos are opened regardless of how the workspace was loaded.
