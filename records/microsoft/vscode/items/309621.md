# Issue #309621 — Find session flow is really not accessible

[https://github.com/microsoft/vscode/issues/309621](https://github.com/microsoft/vscode/issues/309621)

## Summary

Team-filed accessibility bug about the Agents window find-session flow: the search placeholder is vague ('Type to search entity'), no result count is announced to screen readers, and Ctrl+F doesn't work inside the list for keyboard-only users. No fix found on current main; keeping open per team-authored rule.

## Change summary

The find-session search flow in the Agents window lacks accessible labels, result count announcements, and keyboard search support.

## Best solution

An accessibility pass on the sessions list search/filter UI: replace the vague 'Type to search entity' placeholder with a descriptive label (e.g. 'Search sessions'), announce result counts via ARIA live regions when the filter changes, and wire Ctrl+F or an equivalent keybinding to focus the search input from within the list. Work likely belongs in `src/vs/sessions/contrib/sessions/browser/views/sessionsList.ts` and the sessions view's filter/search input components.
