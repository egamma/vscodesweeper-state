# Issue #290768 — Double rendering of same attachment when using background agent

[https://github.com/microsoft/vscode/issues/290768](https://github.com/microsoft/vscode/issues/290768)

## Summary

Team-authored bug, still unfixed on main. When a selection implicit context is pinned, `pinSelection()` adds it to `attachmentModel` as a regular file attachment but never disables or removes the implicit context entry, so both render — the implicit context widget and the regular attachment widget.

## Change summary

Pinning an implicit selection attachment in the background agent session type causes the same selection to render twice — once as the pinned regular attachment and once as the still-visible implicit context.

## Best solution

After `pinSelection()` adds the selection to `attachmentModel`, either disable/remove the corresponding implicit context entry so `isAttachmentAlreadyAttached` returns true on re-render, or call `render()` again after the attachment model update so the dedup check in `render()` (line 79) catches the newly-attached item. The fix lives in `src/vs/workbench/contrib/chat/browser/attachments/implicitContextAttachment.ts`.
