# Issue #125993 — Webview Service Worker Error in registration due to invalid document state

[https://github.com/microsoft/vscode/issues/125993](https://github.com/microsoft/vscode/issues/125993)

## Summary

Keeping open: this carries the 'On Deck' milestone (a scheduling decision), has 134 upvotes, and the underlying Chromium service-worker registration race remains unresolved in current main. The error path at index.html:284 still surfaces the exact reported message.

## Change summary

Webview service worker registration fails with 'InvalidStateError: The document is in an invalid state' — primarily on Linux/snap but also Windows — breaking all webview-based extensions until the user clears caches or reboots.

## Best solution

The root cause is a Chromium bug where `navigator.serviceWorker.register()` fails because the document enters an invalid state during navigation or sandbox attribute changes. The best end state is either an upstream Chromium fix or a VS Code workaround that retries registration or avoids the race (the registration logic lives in `src/vs/workbench/contrib/webview/browser/pre/index.html` around line 256). The 'On Deck' milestone suggests maintainers intend to address this.
