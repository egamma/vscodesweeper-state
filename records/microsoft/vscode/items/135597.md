# Issue #135597 — Add pagination support to `QuickPick` API

[https://github.com/microsoft/vscode/issues/135597](https://github.com/microsoft/vscode/issues/135597)

## Summary

Unimplemented feature request for QuickPick pagination — no `onDidScrollToEnd` event or similar API exists on current main. Low community traction (0 upvotes over nearly 5 years), but it's a real API gap for extensions handling large result sets. Keeping open in Backlog.

## Change summary

Add an `onDidScrollToEnd` event (or equivalent pagination support) to the QuickPick extension API so extensions can lazily load more items as the user scrolls.

## Best solution

If pursued, add an `onDidScrollToEnd` or `onDidTriggerLoadMore` event to `QuickPick` in `src/vscode.d.ts` and wire it through `src/vs/platform/quickinput/`. However, with zero community upvotes since 2021, this may not warrant investment — consider whether the community process should retire it. Alternatively, extensions can work around this with debounced `onDidChangeValue` filtering.
