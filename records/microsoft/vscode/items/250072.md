# Issue #250072 — Minimap overlaps content by the width of the vertical scrollbar

[https://github.com/microsoft/vscode/issues/250072](https://github.com/microsoft/vscode/issues/250072)

## Summary

Still reproduces on main. The minimap position calculation correctly accounts for scrollbar width (minimapLeft = outerWidth - minimapWidth - verticalScrollbarWidth), but contentWidth does not subtract the scrollbar width from the space allocated, causing a visual overlap of minimap over text content.

## Change summary

When the minimap is on the right side, it visually overlaps the editor text content by the width of the vertical scrollbar.

## Best solution

Adjust the content width calculation in EditorLayoutInfoComputer.computeLayout (src/vs/editor/common/config/editorOptions.ts around line 2968) so that contentWidth accounts for verticalScrollbarWidth when the minimap is on the right, preventing the overlap. The related PR #250315 attempted this but was closed without merge — a similar targeted fix is needed.
