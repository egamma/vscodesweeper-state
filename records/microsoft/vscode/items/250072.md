# Issue #250072 — Minimap overlaps content by the width of the vertical scrollbar

[https://github.com/microsoft/vscode/issues/250072](https://github.com/microsoft/vscode/issues/250072)

## Summary

Still reproduces on main: the content width calculation at line 2968 of editorOptions.ts doesn't subtract `verticalScrollbarWidth`, causing the content area to extend into the minimap region by exactly the scrollbar's width. The related PR #250315 was closed without merging.

## Change summary

When the minimap is on the right, the editor content area overlaps it by the vertical scrollbar's width because `contentWidth` doesn't account for the scrollbar.

## Best solution

Fix the layout arithmetic in `EditorLayoutInfoComputer.computeLayout` (src/vs/editor/common/config/editorOptions.ts ~line 2968) so that `contentWidth` properly accounts for `verticalScrollbarWidth` when the minimap is rendered on the right side, ensuring the content area ends where the minimap begins.
