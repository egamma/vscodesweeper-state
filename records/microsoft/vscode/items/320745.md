# Issue #320745 — Expose LM tool enabled or disabled status

[https://github.com/microsoft/vscode/issues/320745](https://github.com/microsoft/vscode/issues/320745)

## Summary

Valid feature request: the extension API's `vscode.lm.tools` returns all registered tools via `getAllToolsIncludingDisabled()` but does not expose whether each tool is currently enabled. The internal service already distinguishes enabled vs. disabled tools (`getTools` vs. `getAllToolsIncludingDisabled`), so surfacing an `enabled` property on `LanguageModelToolInformation` is feasible.

## Change summary

Expose an enabled/disabled status on `LanguageModelToolInformation` so Chat Participants can filter out disabled tools when building LLM requests.

## Best solution

Add an `isEnabled` (or similar) readonly boolean to `LanguageModelToolInformation` in `vscode.d.ts`, propagate the enablement state through `IToolDataDto`, and compute it in `mainThreadLanguageModelTools.ts` using the existing `getTools()`/`getAllToolsIncludingDisabled()` distinction. This is a proposed API addition that needs design input from the tools-service owner (roblourens).
