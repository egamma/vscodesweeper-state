# Issue #174430 — file explorer flash when startup

[https://github.com/microsoft/vscode/issues/174430](https://github.com/microsoft/vscode/issues/174430)

## Summary

Confirmed visual bug — a team member reproduced the explorer flash on startup under specific conditions (only two views in the explorer sidebar with local history disabled). No fix has landed; the related double-layout fix (8ab2e9793f7 for #164188) addressed a different trigger. Still worth tracking.

## Change summary

File explorer panel briefly flashes/flickers on startup when only the file explorer and timeline views are visible and local history is disabled.

## Best solution

Investigate the view pane layout restoration path in `src/vs/workbench/browser/parts/views/viewPaneContainer.ts` — the flash likely comes from the timeline view being registered, then immediately hidden (because its only source is local history, which is disabled), causing a relayout that redistributes the explorer's height. A fix would defer the initial layout until all view visibility is resolved, or suppress the intermediate resize.
