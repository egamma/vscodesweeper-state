# Issue #278301 — MCP: Encourage that MCP servers are installed via marketplace

[https://github.com/microsoft/vscode/issues/278301](https://github.com/microsoft/vscode/issues/278301)

## Summary

Team-authored feature request proposing to consolidate MCP server installation paths around the marketplace for consistent IDs and simpler UX. Must stay open per team-member policy.

## Change summary

Encourage marketplace as the primary MCP server installation path, hide advanced install methods, and prefix custom-installed server IDs with `local/`.

## Best solution

Design a unified MCP server identity scheme that works across installation methods (marketplace, extension-contributed, manual), surface the marketplace as the primary path while keeping extension-contributed servers viable for cases like the Dart extension's SDK-version-aware spawning. The `local/` prefix proposal and UX simplification need product discussion given the valid concerns raised about extension-contributed servers.
