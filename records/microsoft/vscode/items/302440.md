# Issue #302440 — Expose conversation/session metadata (for example UUID) to custom agents, prompts, and tools

[https://github.com/microsoft/vscode/issues/302440](https://github.com/microsoft/vscode/issues/302440)

## Summary

Legitimate feature request to expose stable conversation/session metadata (conversation ID, parent ID, turn ID, agent name, entrypoint type) to custom agents, prompts, and tools. Related issues exist for adjacent API surfaces but none cover this exact request for the custom agent/prompt/tool layer. Routing to chat-tools and chat-prompts for area ownership.

## Change summary

Custom agents, prompts, and tools have no built-in way to access a stable conversation/session identifier or metadata about the current agent run.

## Best solution

Design a small read-only metadata surface (conversation ID, root/parent conversation ID, turn ID, agent name) accessible from custom agents, prompts, and tools — likely as prompt variables (e.g. `${conversation.id}`) or a built-in tool. Coordinate with the related extension API efforts in https://github.com/microsoft/vscode/issues/305853 (chatSessionResource for LM providers) and https://github.com/microsoft/vscode/issues/318855 (session observation API) to ensure a consistent identity model across surfaces.
