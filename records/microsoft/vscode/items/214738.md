# Issue #214738 — Deleting itself

[https://github.com/microsoft/vscode/issues/214738](https://github.com/microsoft/vscode/issues/214738)

## Summary

Active diagnostic cycle: a team member (deepak1556) is engaged and requested updater logs in December 2025; the reporter responded but couldn't provide logs because the ShipIt cache folder was missing. Keeping open as the maintainer owns the next move. The info-needed label is already correctly applied.

## Change summary

VS Code's macOS auto-updater deletes the .app bundle instead of replacing it when restarting to apply an update, particularly when installed outside /Applications.

## Best solution

deepak1556 is actively investigating. The missing ShipIt cache folder strongly suggests a Squirrel/Electron updater misconfiguration when the app is outside /Applications. The next step is for the maintainer to determine whether non-/Applications installs are a supported update path or if this is as-designed behavior — the related open issue https://github.com/microsoft/vscode/issues/234708 tracks silent update failures for non-/Applications installs and may absorb this report. Issue https://github.com/microsoft/vscode/issues/310927 also describes a similar macOS update-and-restart failure.
