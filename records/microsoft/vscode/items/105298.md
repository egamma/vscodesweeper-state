# Issue #105298 — [html] Add value proposals for rel attribute 

[https://github.com/microsoft/vscode/issues/105298](https://github.com/microsoft/vscode/issues/105298)

## Summary

Valid feature request — the `rel` attribute in HTML still has no `valueSet` defined in the web custom data (confirmed in current vscode-html-languageservice@5.6.2), so no value completions are offered. The request has zero upvotes and no comments in nearly 6 years, but it's a reasonable enhancement that would improve consistency with other attributes that do offer value proposals.

## Change summary

Request to add autocomplete value proposals (stylesheet, noopener, noreferrer, etc.) for the HTML `rel` attribute.

## Best solution

Add a `valueSet` for the `rel` attribute in the upstream `@vscode/web-custom-data` package (which feeds `vscode-html-languageservice`). This would provide completions for standard link-type values like `stylesheet`, `noopener`, `noreferrer`, `icon`, `preload`, etc. on `<link>`, `<a>`, and `<area>` elements.
