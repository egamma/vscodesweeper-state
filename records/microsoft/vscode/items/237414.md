# Issue #237414 — Debug sessions become unresponsive after a failed 'restart' request

[https://github.com/microsoft/vscode/issues/237414](https://github.com/microsoft/vscode/issues/237414)

## Summary

Still reproduces on main: when a DAP `restart` request fails (adapter returns `success: false`), the `doRestart` finally block fires `_onDidEndSession` which removes the adapter handle from `_debugAdapters` in `mainThreadDebugService`, even though the adapter is still alive. All subsequent messages from the adapter then throw 'Invalid debug adapter', making the session completely unresponsive.

## Change summary

After a debug adapter returns `success: false` for a DAP `restart` request, the entire debug session becomes unresponsive and cannot be interacted with or terminated.

## Best solution

In the `supportsRestartRequest` branch of `restartSession` (debugService.ts ~line 916–926), avoid using the `doRestart` wrapper — or guard the `_onDidEndSession.fire` in its finally block — so that a failed in-place restart does not prematurely remove the adapter handle. The adapter was never terminated, so the session should remain interactive after a restart failure.
