# Issue #291457 — Welcome view closes my MRU picker

[https://github.com/microsoft/vscode/issues/291457](https://github.com/microsoft/vscode/issues/291457)

## Summary

Team-authored issue; bug is still present on main. The Welcome editor opening after LifecyclePhase.Restored can cause the quick input (MRU picker) to close via its blur handler, even though `shouldPreserveFocus()` tries to avoid stealing focus — the editor DOM creation itself can trigger a transient focus loss.

## Change summary

Opening the command palette MRU picker immediately after launching VS Code gets dismissed when the Welcome view opens asynchronously.

## Best solution

Either delay the Welcome editor open when a quick input is visible (check `IQuickInputService.currentQuickInput`), or set `ignoreFocusOut` transiently during the Welcome editor open in `startupPage.ts`. The fix is localized to `src/vs/workbench/contrib/welcomeGettingStarted/browser/startupPage.ts` and the interaction with the quick input blur handler in `src/vs/platform/quickinput/browser/quickInputController.ts`.
