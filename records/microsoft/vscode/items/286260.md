# Issue #286260 — Worktrees state is only reflecting latest commit, not pending changes

[https://github.com/microsoft/vscode/issues/286260](https://github.com/microsoft/vscode/issues/286260)

## Summary

Team-authored issue (MEMBER); the worktree artifact provider still lacks pending-changes modifiers and timestamps reflecting working-tree state — the gap remains on current main. The discussion between team members about icon design is ongoing.

## Change summary

Worktree indicators in the SCM view only show the last commit age, not pending/uncommitted changes, making the worktree state feel stale.

## Best solution

Add pending-changes modifiers (similar to branch modifiers) to worktree icons in `extensions/git/src/artifactProvider.ts`, and surface working-tree status (not just commit timestamp) in the worktree artifact description. Icon design iteration is tracked at https://github.com/microsoft/vscode-codicons/issues/374.
