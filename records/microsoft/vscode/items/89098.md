# Issue #89098 — Editor title being read out once suggestion is accepted

[https://github.com/microsoft/vscode/issues/89098](https://github.com/microsoft/vscode/issues/89098)

## Summary

Upstream issue still open — the NVDA bug (nvaccess/nvda#10887) that causes screen readers to re-read the editor aria-label when activedescendant is cleared remains unfixed, and VS Code has not implemented the discussed aria-controls workaround. Milestone is 'On Deck', so keeping open.

## Change summary

Screen readers (NVDA, VoiceOver) re-read the editor aria-label when a suggestion is accepted because activedescendant is cleared, returning virtual focus to the parent.

## Best solution

The upstream fix in NVDA (nvaccess/nvda#10887) would resolve this properly. On the VS Code side, the discussed approach of setting `aria-controls` on the textarea pointing to the suggest listbox could help screen readers understand the relationship and suppress re-reading. This would live in `src/vs/editor/contrib/suggest/` and the editor's textarea setup. The 'On Deck' milestone suggests a maintainer intends to revisit.
