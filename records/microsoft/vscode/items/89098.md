# Issue #89098 — Editor title being read out once suggestion is accepted

[https://github.com/microsoft/vscode/issues/89098](https://github.com/microsoft/vscode/issues/89098)

## Summary

Still reproduces on current main — the suggest widget clears `aria-activedescendant` on accept, causing NVDA/VoiceOver to re-read the editor's aria label. The root cause is screen-reader behavior (NVDA issue still open). The `upstream` label is correct; keeping open.

## Change summary

When a suggestion is accepted, screen readers re-read the editor title/aria label because clearing `aria-activedescendant` triggers a focus-change announcement.

## Best solution

The root cause is in NVDA/VoiceOver behavior when `aria-activedescendant` is cleared (tracked at https://github.com/nvaccess/nvda/issues/10887). VS Code could mitigate by implementing `aria-controls` on the textarea pointing to the suggest listbox (discussed in comment 20), or by temporarily clearing the editor aria label around suggestion acceptance. Keep open as upstream until screen readers improve their handling, or until a VS Code-side workaround is prioritized.
