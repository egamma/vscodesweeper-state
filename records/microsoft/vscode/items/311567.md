## Summary

Real bug with a team member already engaged and suggesting a likely root cause (subagents not allowed more tools than parent). The tool-passing logic exists in runSubagentTool.ts but may be filtered downstream. Keep open for continued triage.

## Change summary

Custom agent MCP tools declared in `tools:` frontmatter are not available when the agent is invoked as a sub-agent via the Task tool, though they work when the agent is used directly.

## Best solution

Investigate and fix the tool propagation path in `runSubagentTool.ts` — the `userSelectedTools` map is built from `subagent.tools` (lines 201–213) but MCP tools may be filtered out downstream if the parent agent doesn't also have them enabled. The fix likely belongs in the agent host's tool resolution layer that enforces parent-subset restrictions.
