# Issue #311567 — [Possible Regression] Custom agent MCP tools not available when invoked as sub-agent via Task tool

[https://github.com/microsoft/vscode/issues/311567](https://github.com/microsoft/vscode/issues/311567)

## Summary

Active bug report about MCP tools not being available to custom agents when invoked as sub-agents. A team member (aeschli) is already engaged and has identified a likely root cause: sub-agents are currently restricted to a subset of the parent agent's tools. Keeping open for the area owner to resolve.

## Change summary

Custom agents invoked as sub-agents via the Task tool cannot access MCP tools declared in their `tools:` frontmatter, despite those tools working when called directly by the parent agent.

## Best solution

The tool resolution in `runSubagentTool.ts` correctly reads `subagent.tools` and passes them as `userSelectedTools`, but the downstream extension tool-calling loop likely intersects these with the parent agent's available tools. The fix should allow sub-agents to access their declared MCP tools even when the parent agent doesn't explicitly have them enabled — likely in the Copilot extension's tool resolution logic or the `languageModelToolsService` filtering.
