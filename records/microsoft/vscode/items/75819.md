## Summary

Valid feature request for a cursor-aware 'Fold Current Level' command. Not implemented on main; the existing FoldLevelAction requires specifying a numeric level. Related to open issue #55252. Keeping open as an unresolved backlog feature request.

## Change summary

Request a command that folds all regions at the same nesting level as the cursor position, removing the need to manually count levels.

## Best solution

Either implement a new FoldingAction in `src/vs/editor/contrib/folding/browser/folding.ts` that determines the folding level of the cursor's current region and calls `setCollapseStateAtLevel` with it, or consolidate with https://github.com/microsoft/vscode/issues/55252 which requests a similar non-selection-aware fold level command.
