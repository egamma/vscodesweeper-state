# Issue #323732 — Allow me to fork the copilot chat conversation from the last chat

[https://github.com/microsoft/vscode/issues/323732](https://github.com/microsoft/vscode/issues/323732)

## Summary

Valid feature request to add a fork-conversation affordance at the end of the chat (after the last response), not just on checkpoint dividers between earlier requests. The fork action is currently gated to `ChatMessageCheckpoint` with `isRequest && !isFirstRequest`, so it only appears between turns — the user wants a way to fork the entire conversation as-is from the bottom.

## Change summary

Allow forking a Copilot chat conversation from the end of the conversation (after the last response), not only from intermediate checkpoints.

## Best solution

Add a fork entry point at the bottom of the chat widget (or on the last response's checkpoint) so users can duplicate the full conversation into a new session. The `/fork` slash command already does this when invoked without a target, but the UI affordance is missing. The likely files are `src/vs/workbench/contrib/chat/browser/actions/chatForkActions.ts` (menu registration `when` clause) and possibly `src/vs/workbench/contrib/chat/browser/widget/chatListRenderer.ts` (checkpoint rendering).
