# Issue #228286 — Disabling "Use Tab Stops" should be respected in more indentation contexts

[https://github.com/microsoft/vscode/issues/228286](https://github.com/microsoft/vscode/issues/228286)

## Summary

Legitimate feature request to extend 'Use Tab Stops' setting to also govern Tab key and auto-indentation behavior, not just the Indent Line command. No work has been done on main; the inconsistency remains.

## Change summary

When 'Use Tab Stops' is disabled, the Tab key and auto-indentation still snap to absolute tab-stop columns instead of inserting a fixed indent relative to the current line.

## Best solution

Extend the useTabStops check in auto-indent logic (cursorTypeEditOperations.ts) and Tab key handling (cursorTypeOperations.ts) so that when disabled, indentation is always relative (current column + tabSize) rather than snapping to the next absolute tab stop. This keeps Indent Line, Tab, and Enter consistent.
