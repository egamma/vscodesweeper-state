# Issue #176277 — tasks.json npm script does not work with arguments on Mac

[https://github.com/microsoft/vscode/issues/176277](https://github.com/microsoft/vscode/issues/176277)

## Summary

Team-authored issue — keeping open. The underlying quoting problem in the npm task provider's `escapeCommandLine` function is still present on main: a script string containing spaces (e.g. `build -- --sourcemap`) gets weakly-quoted as a single token, producing incorrect shell invocations on macOS/zsh. The proper fix is tracked by the open feature request #139937 to add an `arguments` property.

## Change summary

npm tasks in tasks.json that include arguments in the `script` field are incorrectly quoted on macOS, causing the shell to interpret the entire string as the script name.

## Best solution

Implement the `arguments` property for npm tasks (https://github.com/microsoft/vscode/issues/139937) so that script name and arguments are separate tokens. Short-term, the `escapeCommandLine` logic in `extensions/npm/src/tasks.ts` could split the script string on ` -- ` to separate the script name from its arguments before quoting.
