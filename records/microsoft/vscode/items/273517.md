# Issue #273517 — In a multi-folder project, if a folder name contains glob characters, it cannot be added to the Include/Exclude list.

[https://github.com/microsoft/vscode/issues/273517](https://github.com/microsoft/vscode/issues/273517)

## Summary

Still reproduces on main: folder names with glob-special characters (parentheses, brackets) are not escaped when building search include/exclude patterns, causing 'Find in Folder' to match wrong folders or fail entirely. An open PR exists but has not merged.

## Change summary

In a multi-folder workspace, 'Find in Folder' fails when the folder name contains glob metacharacters like parentheses.

## Best solution

Review and land the open PR https://github.com/microsoft/vscode/pull/273519, which modifies `src/vs/workbench/services/search/common/queryBuilder.ts` to escape glob characters in folder names when constructing search path patterns. The fix should ensure folder names are treated literally, not as glob syntax.
