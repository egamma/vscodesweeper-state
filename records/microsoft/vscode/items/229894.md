# Issue #229894 — With `editor.experimentalEditContextEnabled`, switching to another application during composition leads to a weird state

[https://github.com/microsoft/vscode/issues/229894](https://github.com/microsoft/vscode/issues/229894)

## Summary

This is an upstream Chromium/EditContext API issue where focus-switching during IME composition causes incorrect text update events. Filed as https://issues.chromium.org/issues/369883717 and awaiting Chromium team response. The issue remains valid and open pending upstream resolution.

## Change summary

Switching to another application during IME composition with EditContext enabled causes incorrect text insertion when returning to VS Code.

## Best solution

Await Chromium team's response on https://issues.chromium.org/issues/369883717. If they decline to fix, implement a workaround in `src/vs/editor/browser/controller/editContext/native/nativeEditContext.ts` that resets composition state on focus loss (e.g., force-ending the composition on blur or re-syncing the edit context selection on focus regain).
