# Issue #311234 — checkout error on agent plugin/marketplace using gitlfs

[https://github.com/microsoft/vscode/issues/311234](https://github.com/microsoft/vscode/issues/311234)

## Summary

Feature request for git-lfs support and sparse checkout in the agent plugin marketplace clone flow. The plugin install path currently does a plain git clone with no lfs or sparse-checkout handling; keeping open for the area owner to evaluate.

## Change summary

Agent plugin marketplace clone fails when the plugin repo uses git-lfs; user also requests sparse checkout to only fetch plugin-relevant folders.

## Best solution

Add git-lfs detection/invocation to the plugin clone flow in `pluginSources.ts` / `agentPluginRepositoryService.ts`, and consider sparse-checkout support for multi-plugin repos. The agent host git service already surfaces lfs-missing errors — the plugin install path should either require lfs or gracefully handle its absence.
