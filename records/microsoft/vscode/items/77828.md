# Issue #77828 — Git - Revert changes on a hunk sometimes corrupt entire files

[https://github.com/microsoft/vscode/issues/77828](https://github.com/microsoft/vscode/issues/77828)

## Summary

Rare but real race condition where stale diff hunks cause file corruption on revert. The code was significantly refactored (quickDiff model, diff information switch in 9cf5e81d4f5, Feb 2025), but no commit directly addresses stale-hunk revert corruption. Keeping open — labeled help wanted and in Backlog, still a valid concern.

## Change summary

Reverting a single hunk in the gutter sometimes applies stale diff data, corrupting the entire file by misaligning line ranges.

## Best solution

The root cause is likely stale `LineChange[]` data passed to `applyLineChanges` in `extensions/git/src/staging.ts` when the dirty diff model is out of sync with the editor content. A guard comparing the original document content hash or verifying line counts before applying would prevent silent corruption. The `help wanted` label is appropriate — this is a good candidate for a contributor who can reproduce the submodule/rebase scenario.
