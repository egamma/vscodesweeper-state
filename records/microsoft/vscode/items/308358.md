# Issue #308358 — BOM character (U+FEFF) in telemetry.devDeviceId causes 'Language model unavailable' — all API requests fail with ByteString error

[https://github.com/microsoft/vscode/issues/308358](https://github.com/microsoft/vscode/issues/308358)

## Summary

Already fixed on main and shipped in 1.121.0. The `resolveDevDeviceId` function and the underlying `getDevDeviceId` now both strip UTF-8 BOM via `stripUTF8BOM()`, preventing the corrupted value from reaching HTTP headers.

## Change summary

A BOM character (U+FEFF) in the persisted `telemetry.devDeviceId` value caused all Copilot API requests to fail with a ByteString error, showing 'Language model unavailable'.

## Best solution

Keep the shipped fix (commit 15c518a9b04, 1.121.0). Both the cached-value read path (`resolveDevDeviceId`) and the fresh-generation path (`getDevDeviceId`) now strip BOM. No further change needed.
