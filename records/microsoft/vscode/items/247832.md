# Issue #247832 — Timeline Hover Position Is Off

[https://github.com/microsoft/vscode/issues/247832](https://github.com/microsoft/vscode/issues/247832)

## Summary

Team-authored issue (hediet, MEMBER) — must keep open. The hover misposition with `workbench.list.horizontalScrolling: true` remains unaddressed on main; the related commit 394e0dfe10d only fixed panel-location hover behavior, not horizontal-scrolling width issues.

## Change summary

Timeline hover tooltip is positioned far from the hovered item when `workbench.list.horizontalScrolling` is enabled, because the underlying DOM element extends wider than the sidebar.

## Best solution

The hover target element likely needs to be constrained or a custom `hoverTarget` override supplied so the hover anchors to the visible row, not the full-width scrollable container. Fix likely lives in `src/vs/workbench/contrib/timeline/browser/timelinePane.ts` in the `TimelineTreeRenderer` hover delegate setup or in the `TimelineElementTemplate` where the hover is attached to the `IconLabel`.
