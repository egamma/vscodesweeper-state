# Issue #312129 — UI: Slash command dropdown truncates namespaced skills, hiding the command's purpose

[https://github.com/microsoft/vscode/issues/312129](https://github.com/microsoft/vscode/issues/312129)

## Summary

Valid polish request: the Monaco suggest widget truncates long slash-command labels from the left, hiding the distinctive command suffix for namespaced skills. The suggest widget uses a fixed 430px width with standard text-overflow: ellipsis, which is by-design for general completions but unhelpful for long namespace:command patterns. No fix has landed.

## Change summary

Slash command completions with long namespace prefixes (e.g. `mcp-apps:create-...`) truncate the unique command name, making it hard to distinguish between similar skills.

## Best solution

The chat slash-command completion providers in `chatInputCompletions.ts` could set a tooltip (via the `documentation` field, which some already do) or restructure the `label` to use middle-ellipsis or show the command portion more prominently. Alternatively, the suggest widget itself could be made resizable or show a title attribute on the label element. This is a polish item for the chat area owner to decide on approach.
