# Issue #234879 — SCM Graph - Add horizontal scrollbar for source control graph

[https://github.com/microsoft/vscode/issues/234879](https://github.com/microsoft/vscode/issues/234879)

## Summary

Valid feature request for horizontal scrolling in the SCM history graph. Horizontal scrolling is explicitly disabled (`horizontalScrolling: false`) in the tree widget — this is a deliberate design choice, not a bug. Keeping open as a feature request in the Backlog for community traction.

## Change summary

Add a horizontal scrollbar to the SCM history graph so wide graphs (many branches) can be scrolled horizontally.

## Best solution

The fix would be to change `horizontalScrolling: false` to `true` (or make it configurable) in `src/vs/workbench/contrib/scm/browser/scmHistoryViewPane.ts` around line 2016. However, this is a UX design decision — the SCM graph renders branch lanes inline, and enabling horizontal scrolling may conflict with how the graph columns are laid out. The item type should be corrected from `bug` to `feature-request`, and the area owner should decide whether horizontal scrolling is desirable for the graph view.
