# Issue #300390 — Inline suggestions violate context exclusion on VS Code start

[https://github.com/microsoft/vscode/issues/300390](https://github.com/microsoft/vscode/issues/300390)

## Summary

The race condition in content exclusion initialization is real and traced in the source. An open PR (https://github.com/microsoft/vscode/pull/308814) directly addresses the race but has not yet merged — keeping open until that PR lands.

## Change summary

Inline suggestions briefly appear for content-excluded files during the first seconds after VS Code starts, before the exclusion rules are fetched.

## Best solution

Land https://github.com/microsoft/vscode/pull/308814, which fixes the race in `remoteContentExclusion.ts` by preventing concurrent `isIgnored()` calls from caching stale 'not ignored' results during the initial fetch window.

## Evidence

- **source trace:** The fix commit on the feature branch (708647f0d83) modifies extensions/copilot/src/platform/ignore/node/remoteContentExclusion.ts to prevent concurrent isIgnored() callers from caching stale empty patterns during the initial exclusion-rule fetch.
- **root cause:** Concurrent isIgnored() calls could seed the pattern cache with empty rules while a fetch was in progress; callers matching against empty rules would return 'not ignored' and cache that stale result.
- **reporter version:** Reporter on VS Code 1.110.0; the fix has not shipped in any release tag (not in 1.122.0 through 1.126.0).
