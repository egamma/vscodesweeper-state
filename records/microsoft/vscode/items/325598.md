# Issue #325598 — Copilot session rate limit error gives no reset time or duration

[https://github.com/microsoft/vscode/issues/325598](https://github.com/microsoft/vscode/issues/325598)

## Summary

Still reproduces on main: the WebSocket CAPI error handlers in the Copilot extension hardcode `retryAfter: undefined`, so the core error-message code falls back to the vague "a moment" even though the server provides the exact retry-after seconds in the error message text.

## Change summary

The session rate limit error message says "wait a moment" instead of showing the actual reset time the server provides (e.g. "retry after 3572 seconds").

## Best solution

Parse the retry-after seconds from the WebSocket CAPI error message in both `_handleWebSocketCAPIError` and `_handleWebSocketError` in `extensions/copilot/src/extension/prompt/node/chatMLFetcher.ts`, and pass the extracted value as `retryAfter` instead of `undefined`. The core UI code in `src/vs/workbench/contrib/chat/common/chatErrorMessages.ts` already handles formatting it into a human-readable duration — it just needs the value.
