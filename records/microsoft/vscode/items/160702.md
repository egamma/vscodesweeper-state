# Issue #160702 — Fractional scaling makes cursor inaccurate on Chromebooks

[https://github.com/microsoft/vscode/issues/160702](https://github.com/microsoft/vscode/issues/160702)

Keep this open but ask for a current ChromeOS repro before further cleanup. The report is plausible and has an editor workaround, but it is based on VS Code 1.71.0 / ChromeOS 105 and I did not find a concrete post-report fix to justify closing it.

## Change summary

Fractional display scaling on ChromeOS makes the editor caret/click position drift several characters on long lines.

## Best solution

Have an affected user retest current Stable/Insiders or vscode.dev on ChromeOS or ChromeOS Flex with fractional scaling, including whether `editor.disableMonospaceOptimizations` still fixes it. If it still reproduces, the likely editor-rendering work is around font measurement / pixel-ratio invalidation in `src/vs/editor/browser/config/fontMeasurements.ts` and view-line monospace positioning.

## Evidence

- **report details:** The issue reports all extensions disabled, ChromeOS 105, VS Code 1.71.0, fractional/effective DPR values such as 1.375 or 1.25, and cursor drift of about 3-5 characters in github.dev/vscode.dev.
- **maintainer workaround:** https://github.com/microsoft/vscode/issues/160702#issuecomment-1262828356 says `editor.disableMonospaceOptimizations: true` is an immediate workaround because it makes the browser query cursor positions instead of relying on monospace optimizations.
- **additional affected user:** https://github.com/microsoft/vscode/issues/160702#issuecomment-1513610752 reports a similar issue on ChromeOS defaults and suggests ChromeOS Flex as the best VM-like repro path.
- **current code:** `src/vs/base/browser/pixelRatio.ts:18` monitors DPR changes with `matchMedia`, `src/vs/editor/common/config/fontInfo.ts:109` includes `pixelRatio` in the font-info cache key, and `src/vs/editor/browser/viewParts/viewLines/viewLineOptions.ts:36` still uses monospace optimizations unless the hidden setting disables them.
- **history check:** Local `git log --grep` searches for ChromeOS/Chromebook/DPR/devicePixelRatio/cursor did not reveal a concrete issue-specific fixing commit after this report, so an implemented-on-main close would be unsupported.
