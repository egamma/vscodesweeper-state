# Issue #160702 — Fractional scaling makes cursor inaccurate on Chromebooks

[https://github.com/microsoft/vscode/issues/160702](https://github.com/microsoft/vscode/issues/160702)

## Summary

This is the canonical issue for cursor position inaccuracy under fractional DPR on ChromeOS. Related issues #180223 and #165887 were closed as duplicates of this one. The workaround (`disableMonospaceOptimizations`) exists but the underlying monospace width assumption still applies on main — no fix commit found. Keeping open as it represents a real, unresolved platform-specific bug affecting ChromeOS users with fractional scaling.

## Change summary

Cursor position drifts from actual text position on ChromeOS when display scaling produces a non-integer devicePixelRatio, because the editor's monospace character width assumptions break under fractional scaling.

## Best solution

The editor's monospace width optimization (which assumes all characters have identical width) needs to account for sub-pixel rounding errors that accumulate under fractional DPR on ChromeOS. The fix likely lives in `src/vs/editor/browser/config/` (character width reading) or the view line rendering logic. Alternatively, detect fractional-DPR environments and automatically disable monospace optimizations. The issue should remain open as the canonical tracker — two other reports already point here.
