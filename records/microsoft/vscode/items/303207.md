# Issue #303207 — Copy Path feature returns string with Path Seperator for windows on Linux SSH only for staged files.

[https://github.com/microsoft/vscode/issues/303207](https://github.com/microsoft/vscode/issues/303207)

## Summary

Confirmed bug with an open fix PR (#309252) that has not yet merged. The defect is in `tildify()` returning the un-normalized path when it doesn't match the user home prefix. Keeping open until the PR lands.

## Change summary

Copy Path on a staged (git-scheme) file in a Remote SSH session to Linux returns Windows-style backslash separators instead of forward slashes.

## Best solution

Land the open PR https://github.com/microsoft/vscode/pull/309252, which changes `tildify()` in `src/vs/base/common/labels.ts` to return `normalizedPath` instead of the raw `path` on the non-tildified fallback path. The PR also adds a test covering cross-scheme URIs on remote SSH.
