# Issue #309418 — Confirmation Carousel: Displays empty dirname for some directories on Windows

[https://github.com/microsoft/vscode/issues/309418](https://github.com/microsoft/vscode/issues/309418)

## Summary

Team-authored visual bug — still present on main. When a tool confirmation targets a root-level Windows path (e.g. `C:\`), the dirname portion of the display is empty because the path formatting doesn't handle drive-root paths gracefully. Hard keep-open (MEMBER author).

## Change summary

The confirmation carousel shows an empty dirname label when a tool targets a root-level directory on Windows (e.g. `C:\`).

## Best solution

Guard the path display logic — likely in `formatPathAsMarkdownLink` in `src/vs/platform/agentHost/node/copilot/copilotToolDisplay.ts` or in how `toPlainText` strips markdown links in `chatToolConfirmationCarouselPart.ts` — so that a Windows drive root (`C:\`) renders as e.g. `C:\` rather than an empty string. The issue may also live in how the agent host builds `confirmationTitle` / `invocationMessage` for `list_dir` on a root path.
