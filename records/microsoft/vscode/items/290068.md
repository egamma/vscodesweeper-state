# Issue #290068 — Window won't focus under native Wayland

[https://github.com/microsoft/vscode/issues/290068](https://github.com/microsoft/vscode/issues/290068)

## Summary

This is a Wayland window-activation issue on GNOME where the compositor shows an 'Application is ready' notification instead of focusing the VS Code window. A prior instance (#225211) was fixed with the Electron 32 xdg-activation support, but the reporter confirms the issue persists on VS Code 1.108+. This is an upstream Electron/Chromium/GNOME Wayland interaction issue that needs investigation to determine whether the xdg-activation token is properly passed in all activation scenarios.

## Change summary

VS Code window does not receive focus under native Wayland on GNOME — the compositor shows an 'Application is ready' notification instead of raising the window.

## Best solution

Investigate whether the xdg-activation token is properly transferred in all window-activation paths (file explorer open, `code --reuse-window`, etc.) on current Electron. If the gap is in Electron/Chromium's Wayland support, file or link an upstream Electron issue. The `FocusMode.Transfer` path in `src/vs/platform/windows/electron-main/windowImpl.ts` calls `win.focus()` without the macOS-only `app.focus({ steal: true })` — on Wayland, Electron's `BrowserWindow.focus()` must use xdg-activation correctly; if it doesn't, the fix lives upstream.

However, there may be remaining gaps in GNOME/Mutter's handling or in how Electron passes the activation token across process boundaries (e.g., CLI -> existing window).
- **commenter confirms:** Comment by alk3pInjection (2026-02-26) confirms the issue persists on VS Code 1.109.5 and suspects an Electron-side issue.
- **window focus code:** src/vs/platform/windows/electron-main/windowImpl.ts:386-403 — `doFocusWindow()` calls `win.focus()` which relies on Electron's Wayland activation support. The `app.focus({ steal: true })` call is macOS-only (line 356-358).
- **Wayland design:** Wayland's security model intentionally prevents focus stealing. The 'Application is ready' notification from GNOME is the expected fallback when xdg-activation token transfer fails or is not supported for the activation path.
