# Issue #284550 — The ollama agent displays JSON when you make a request but the addin and/or VSCode does not react to the response it receives

[https://github.com/microsoft/vscode/issues/284550](https://github.com/microsoft/vscode/issues/284550)

## Summary

The Ollama extension's language model provider is not translating tool-call responses from the model into VS Code's structured tool-call API — it streams them as plain text instead. This is the extension's responsibility, not a VS Code core issue.

## Change summary

When using Ollama models in Agent mode, tool calls are displayed as raw JSON text instead of being executed.

## Best solution

File this against the Ollama VS Code extension (the extension's language model provider must parse tool-call responses from the Ollama API and surface them via the VS Code Language Model API's tool-call protocol). VS Code core correctly executes tools when they arrive through the structured API — the gap is in the extension's translation layer.

- **extension responsibility:** VS Code's agent/chat system consumes tool calls through the structured Language Model API (LanguageModelToolCallPart). The Ollama extension registers a language model provider and is responsible for parsing the model's raw output — including tool-call JSON — and converting it into the proper API types. When it fails to do so, the JSON appears as plain chat text.

- **confirmer diagnosis:** The July 2026 commenter states: 'the model is actually attempting to invoke tools, but VS Code is displaying the tool calls as plain text instead of executing them' — this is exactly what happens when tool calls are streamed as text content rather than structured LanguageModelToolCallPart objects.
- **works with other providers:** The original reporter notes 'a request that works fine with Anthropic' — Anthropic's provider correctly translates tool calls into the structured API, confirming VS Code's tool execution pipeline works.
