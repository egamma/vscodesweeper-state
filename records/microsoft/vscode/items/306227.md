# Issue #306227 — Copilot Agent Skill Validation Conflicts with "skill" Language ID (Cadence SKILL Programming Language)

[https://github.com/microsoft/vscode/issues/306227](https://github.com/microsoft/vscode/issues/306227)

## Summary

Confirmed on main: the prompt validator in `promptFileContributions.ts` triggers validation based solely on language ID `"skill"` (via `getPromptsTypeForLanguageId`), without checking the file path or filename. When a third-party extension or user file association assigns language ID `"skill"` to non-SKILL.md files, the validator erroneously applies agent skill validation to them.

## Change summary

Files with language ID "skill" (from Cadence SKILL extensions) receive false positive "Skill should provide a name" errors because the prompt validator matches on language ID alone without path filtering.

## Best solution

The validator in `promptFileContributions.ts` should additionally check that the file matches the expected SKILL.md filename pattern (via `isSkillFilename`) before applying skill validation, or the `prompt-basics` extension should use a more specific language ID (e.g., `copilot-skill`) to avoid colliding with legitimate third-party language IDs.
