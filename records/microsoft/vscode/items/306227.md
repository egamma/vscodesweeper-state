# Issue #306227 — Copilot Agent Skill Validation Conflicts with "skill" Language ID (Cadence SKILL Programming Language)

[https://github.com/microsoft/vscode/issues/306227](https://github.com/microsoft/vscode/issues/306227)

## Summary

Still reproduces on main: the prompt validator triggers on any open model with language ID 'skill' without verifying the file is actually a SKILL.md prompt file in a recognized skills folder. A third-party extension registering the same language ID causes false validation errors.

## Change summary

Files with language ID "skill" from third-party extensions (e.g., Cadence SKILL .il files) receive false "Skill must provide a name" validation errors from the Copilot prompt validator.

## Best solution

Add a path-based guard in `PromptValidatorContribution.acquire()` (and the `PromptModelTracker` listener in `promptsServiceImpl.ts`) so that a model with language ID 'skill' is only treated as a prompt skill file when its URI matches a recognized skill source folder or its filename is `SKILL.md`. This keeps the language ID mapping intact for real skill files while ignoring unrelated files that share the ID.
