# Issue #172902 — Extra selection getting added when you Alt + click on a multiple selection to remove the selection

[https://github.com/microsoft/vscode/issues/172902](https://github.com/microsoft/vscode/issues/172902)

## Summary

Still reproducible on main: when Alt+click removes a cursor (toggle behavior in CreateCursor), a subsequent tiny mouse drag triggers _lastCursorMoveToSelect which extends the last cursor into a new selection, because the drag path doesn't know the initial click removed rather than added a cursor.

## Change summary

Alt+clicking an existing cursor to remove it inadvertently creates a selection from the last cursor if the mouse moves even slightly during the click.

## Best solution

After CreateCursor removes a cursor (toggle path, line 933–943 in coreCommands.ts), the drag handler in viewController.ts (line 288–289) needs to skip _lastCursorMoveToSelect. A flag or return value from CreateCursor indicating removal occurred could suppress the subsequent drag selection.
