## Summary

Team-authored bug report — keeping open per policy. The issue describes prompt files (slash commands) not selecting the intended agent when invoked via 'run in new chat' (Ctrl+Shift+Enter).

## Change summary

Running a prompt file slash command with 'run in new chat' does not select the intended agent (e.g. 'engineering') in the new chat session.

## Best solution

Ensure that when a prompt file specifies an agent, invoking it via Ctrl+Shift+Enter (new chat) carries forward the agent selection into the freshly created chat session.
