[https://github.com/microsoft/vscode/issues/230764](https://github.com/microsoft/vscode/issues/230764)

## Summary

Legitimate upstream bug already triaged with the `upstream` label — a native macOS menu getting permanently stuck is almost certainly an Electron or macOS menu-management issue, not something VS Code's own code controls. No fix found on current main, and the niche reproduction path (accidental click outside a menu while holding mouse down) makes this hard to verify in source. Keeping open as-is.

## Change summary

On macOS, the Selection menu dropdown can become permanently displayed and cannot be dismissed by any user action short of restarting VS Code.

## Best solution

This likely needs an Electron or Chromium fix for native macOS menu lifecycle management. If it can be reproduced reliably, file upstream at https://github.com/electron/electron/issues with the spindump and screen recording. VS Code's own menu code (the native titlebar menu path on macOS) delegates to Electron's `Menu.setApplicationMenu` and has no control over dismissal of native dropdown popups.
