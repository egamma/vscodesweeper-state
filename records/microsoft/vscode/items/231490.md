# Issue #231490 — debug.onDidChangeActiveDebugSession not always fired when active session implicitly changes after old active session terminated

[https://github.com/microsoft/vscode/issues/231490](https://github.com/microsoft/vscode/issues/231490)

## Summary

The bug is plausible from source analysis: in `getStackFrameThreadAndSessionToFocus` (debugService.ts:1497), the fallback `sessions[0]` can return the avoidSession itself if the `find` clause fails, and there may be timing-dependent scenarios where `shouldEmitForSession` evaluates false despite an effective session change. No fix found on main.

## Change summary

When the active debug session is terminated and another session remains, `debug.onDidChangeActiveDebugSession` is not always fired to notify extensions of the implicit focus change.

## Best solution

Fix the fallback in `getStackFrameThreadAndSessionToFocus` (src/vs/workbench/contrib/debug/browser/debugService.ts:1497) to ensure `sessions[0]` never returns the `avoidSession`, and audit the session-end focus path (lines 782–785) for edge cases where the terminated session's reference identity could match the newly-focused session despite a logical change.
