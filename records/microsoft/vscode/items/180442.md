# Issue #180442 — Create different visuals on editor for different type of code actions

[https://github.com/microsoft/vscode/issues/180442](https://github.com/microsoft/vscode/issues/180442)

## Summary

The request to visually distinguish code action kinds (QuickFix vs Refactor) in the lightbulb indicator remains unimplemented. Current main differentiates autofix/AI-fix scenarios but not the underlying CodeActionKind. Keeping open as a valid feature request in the Backlog.

## Change summary

Show different lightbulb icons or visuals in the editor gutter depending on the code action kind (quick fix vs refactoring).

## Best solution

Extend `computeLightBulbInfo` in `src/vs/editor/contrib/codeAction/browser/lightBulbWidget.ts` to inspect `CodeActionKind` (QuickFix, Refactor, Source) and select distinct icons accordingly, building on the existing icon-selection logic that already differentiates autofix and AI-fix scenarios.
