# Issue #238128 — provide a way to collapse the extension info

## Summary

Valid accessibility feature request, acknowledged by team members (sandy081, meganrogge), not yet implemented on main. Keeping open in Backlog as an endorsed enhancement.

## Change summary

Provide a way to collapse the extension info header (logo, name, buttons) in the extension editor view so zoomed-in users can access the full description.

## Best solution

Add a collapsible section or toggle to the extension editor header area in `src/vs/workbench/contrib/extensions/browser/extensionEditor.ts` so visually impaired users at high zoom can reach the description without the header consuming the viewport.
