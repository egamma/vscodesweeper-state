# Issue #240067 — Git - Fixed alignment-column for git 'inline blame' annotations, regardless of line-length

[https://github.com/microsoft/vscode/issues/240067](https://github.com/microsoft/vscode/issues/240067)

## Summary

Valid feature request for a fixed-column alignment option for git inline blame decorations. Not yet implemented on main — the git blame editor decoration has no alignment/column setting. Keeping open in Backlog for the community process.

## Change summary

Request a configuration option to align git inline blame decorations to a fixed column instead of rendering them immediately after each line's content.

## Best solution

Add a setting like `git.blame.editorDecoration.startColumn` (numeric column value) in `extensions/git/src/blame.ts` and `extensions/git/package.json` so the decoration is padded to a minimum column. Prior art exists in the error-lens extension's `alignMessage` setting. Low upvote count (1) — let the community process decide priority.
