# Issue #154934 — Render of fullscreen stopped working correctly with auto hidden taskbar on windows

[https://github.com/microsoft/vscode/issues/154934](https://github.com/microsoft/vscode/issues/154934)

## Summary

Confirmed upstream Electron bug — the linked issue (electron/electron#36638) remains open and unresolved as of June 2026. The issue is already correctly labeled with `upstream` and `upstream-issue-linked`; it should stay open until the Electron fix lands and VS Code upgrades.

## Change summary

VS Code fullscreen mode renders incorrectly on Windows when the OS taskbar is set to auto-hide, requiring a window switch to fix the layout.

## Best solution

Wait for the upstream fix in Electron (https://github.com/electron/electron/issues/36638) to land, then verify the fix after VS Code upgrades to the patched Electron version. No workaround is practical on the VS Code side since, as bpasero noted, the workbench correctly believes it is in fullscreen mode — the window geometry from Electron is wrong.
