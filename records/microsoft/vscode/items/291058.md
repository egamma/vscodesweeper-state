# Issue #291058 — Showing this blocks any other hovers from showing up

[https://github.com/microsoft/vscode/issues/291058](https://github.com/microsoft/vscode/issues/291058)

## Summary

Team-authored issue; the hover service's single-hover-at-a-time constraint (locked hover blocks new hovers via the `isLocked` guard in `showDelayedHover`) causes the context window widget to prevent status bar hovers from appearing. Still present on main — no fix found for the blocking behavior itself.

## Change summary

While the context window hover widget is visible, hovering over status bar items shows no tooltip; dismissing the widget restores normal hover behavior.

## Best solution

Modify the hover service so that a new hover request from a different UI region (e.g. status bar) dismisses the currently locked hover rather than silently rejecting the new request. The fix likely lives in `src/vs/platform/hover/browser/hoverService.ts` around the `isLocked` guard in `showDelayedHover` (line ~144). As @TylerLeonhardt noted, any new hover should cancel previous hovers so the UI never feels broken.
