# Issue #326083 — Task problem matching inconsistency across task runs?

[https://github.com/microsoft/vscode/issues/326083](https://github.com/microsoft/vscode/issues/326083)

## Summary

Legitimate bug report with clear reproduction steps. On second task run, problem matchers appear to produce stale and incorrect markers — both extra entries (lines matched incorrectly) and a non-updating ValueError message. The issue is in the `tasks` area and was previously closed as `*question` (#325270), which was incorrect. No fix is present on current main; a related but distinct fix for stale markers across document state changes exists on an unmerged branch (`dbaeumer/issues/322730`).

## Change summary

Running a task with a problem matcher a second time produces stale/duplicate markers in the Problems panel instead of replacing the previous run's results.

## Best solution

Investigate the `StartStopProblemCollector` marker lifecycle during task rerun in `src/vs/workbench/contrib/tasks/common/problemCollectors.ts` and `src/vs/workbench/contrib/tasks/browser/terminalTaskSystem.ts`. The terminal is reused across runs and the old output may be re-fed to the new collector, or markers from the first run are not fully cleared before the second run delivers new ones. The `applyTo` filtering in `_cleanMarkers` may also prevent cleanup when the document's open/closed state doesn't match. Dirk Baeumer's in-progress `dbaeumer/issues/322730` branch addresses a related stale-marker scenario and may partially cover this.
