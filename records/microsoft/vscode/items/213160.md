# Issue #213160 — Tabs: clicking between active tabs results in 4 `redrawTabSelectedActiveAndDirty`

[https://github.com/microsoft/vscode/issues/213160](https://github.com/microsoft/vscode/issues/213160)

## Summary

Fixed by commit 4fb0b152981 (PR #219651), which adds an early return in `updateEditorSelections` when no selection was removed, eliminating the redundant redraws. Shipped in 1.92.0.

## Change summary

Clicking between active tabs triggered 4 redundant `redrawTabSelectedActiveAndDirty` calls due to unguarded selection change events.

## Best solution

Keep the shipped fix — the optimization in `updateEditorSelections` that skips redraws when selection count hasn't decreased resolves the unnecessary work.
