# Issue #192120 — For simple deletions, no words should be highlighted on the right side

[https://github.com/microsoft/vscode/issues/192120](https://github.com/microsoft/vscode/issues/192120)

## Summary

Team-authored feature request to improve diff editor word-level highlighting: for simple deletions within a line, words on the modified (right) side that weren't changed should not be highlighted. Still open on main — no fix found.

## Change summary

When text is deleted within a line, the diff editor incorrectly highlights unchanged words on the right (modified) side.

## Best solution

Improve the inner-changes computation (now in `@vscode/diff`) or post-process its output in `diffEditorDecorations.ts` so that for pure-deletion inner changes, no character-level decoration is applied on the modified side when the modified range represents unchanged text.
