# Issue #210303 — [html] Format does not work if script tag parameters do not have quotes

[https://github.com/microsoft/vscode/issues/210303](https://github.com/microsoft/vscode/issues/210303)

## Summary

Confirmed bug — a team member (aeschli) reproduced it. The root cause is in how the HTML language service (vscode-html-languageservice / js-beautify) handles unquoted attribute values in script tags. The code in embeddedSupport.ts line 60 already handles `token === 'module'` for the unquoted case, but the formatter itself (js-beautify) likely fails to recognize the embedded script region. Still valid on main with no evidence of a fix.

## Change summary

HTML Format Document fails when script tag attributes lack quotes (e.g. `<script type=module>`).

## Best solution

Fix upstream in vscode-html-languageservice's formatting integration with js-beautify to correctly identify embedded script regions when attribute values are unquoted. Alternatively, the pending Volar integration (https://github.com/microsoft/vscode/pull/171547) may resolve this by replacing the formatter entirely.
