# Issue #193875 — Nested for statements in JavaScript breaks in wrong indention

[https://github.com/microsoft/vscode/issues/193875](https://github.com/microsoft/vscode/issues/193875)

## Summary

Still reproduces on main. The regex-based `indentNextLinePattern` in TypeScript's language config cannot track nested bodyless control-flow statements; the `tree-sitter-autoindent-candidate` label correctly identifies the long-term fix path.

## Change summary

Typing a brace after nested bodyless `for` statements produces incorrect closing-brace indentation.

## Best solution

Resolve via tree-sitter-based auto-indentation, which can structurally understand nested statements without braces. The regex heuristic in `extensions/typescript-basics/language-configuration.json` (`indentNextLinePattern`) is fundamentally limited here — it only looks one line back.
