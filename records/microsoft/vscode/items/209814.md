[https://github.com/microsoft/vscode/issues/209814](https://github.com/microsoft/vscode/issues/209814)

## Summary

Valid but low-traction feature request to expose raw snippet text in SelectedSuggestionInfo for inline completion providers. Maintainer engaged but unconvinced by the approach; discussion stalled. Keeping open as under-discussion with only 1 upvote.

## Change summary

Expose the original snippet text (with placeholders) in InlineCompletionContext's selectedSuggestionInfo so providers can infer cursor position after suggest acceptance.

## Best solution

If the team decides this is worth pursuing, enrich `SelectedSuggestionInfo` in `src/vs/editor/common/languages.ts` with an optional `snippetText` field carrying the raw snippet string. However, hediet's concern about leaking suggest implementation details into the inline completion API is valid — an alternative design (e.g., exposing parsed placeholder positions rather than raw snippet syntax) may be preferable. Needs a design decision from the inline-completions area owner.
