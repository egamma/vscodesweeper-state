# Issue #166110 — 3-way merge: Have a hover to understand a change

[https://github.com/microsoft/vscode/issues/166110](https://github.com/microsoft/vscode/issues/166110)

## Summary

Team-authored polish request for the merge editor — still valid and unimplemented on main. The 3-way merge editor could benefit from hover tooltips summarizing per-side changes without requiring the base view.

## Change summary

Add a hover tooltip in the 3-way merge editor that summarizes what was added, changed, or removed per side, so users don't need to enable the base view to understand changes.

## Best solution

Implement a hover provider in the merge editor (`src/vs/workbench/contrib/mergeEditor/browser/`) that, when hovering a conflict region, shows a concise diff summary (lines added/removed/changed per side relative to the base) — reducing the cognitive load of understanding conflicts without toggling the base view.
