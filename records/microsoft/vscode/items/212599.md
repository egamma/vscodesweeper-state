# Issue #212599 — [folding] Remove manual folding range should only remove most inner folding range

[https://github.com/microsoft/vscode/issues/212599](https://github.com/microsoft/vscode/issues/212599)

## Summary

Valid feature request with `help wanted` label — the current behavior (removing all intersecting manual ranges) is confirmed in source on main. The request to remove only the innermost range is a reasonable UX improvement that remains unimplemented.

## Change summary

The 'Remove Manual Folding Ranges' command removes all manual folding ranges intersecting the selection instead of only the innermost one at the cursor.

## Best solution

Modify `removeManualRanges` in `src/vs/editor/contrib/folding/browser/foldingModel.ts` (or the action's `invoke` in `folding.ts`) to identify only the most-nested manual range at the cursor position and remove that one, rather than all intersecting ranges.
