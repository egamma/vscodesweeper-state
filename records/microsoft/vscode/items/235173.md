# Issue #235173 — Git - Don't show inline blame while dragging selection

[https://github.com/microsoft/vscode/issues/235173](https://github.com/microsoft/vscode/issues/235173)

## Summary

Still reproduces on main: the inline blame decoration updates on every `onDidChangeTextEditorSelection` event (line 321 of `extensions/git/src/blame.ts`) without distinguishing a completed click from an in-progress drag, so the blame tooltip follows the cursor during selection dragging.

## Change summary

Git inline blame decoration visually follows the cursor while the user is click-dragging to make a multi-line selection, which is distracting.

## Best solution

Debounce or delay the blame update for mouse-kind selection changes in `extensions/git/src/blame.ts` so the decoration only appears after the selection gesture is complete, not during drag. The `TextEditorSelectionChangeEvent.kind` (Mouse=2) can be used to gate the delay.
