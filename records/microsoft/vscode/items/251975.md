# Issue #251975 — Accept all ... button text is wrong in merge editor.

[https://github.com/microsoft/vscode/issues/251975](https://github.com/microsoft/vscode/issues/251975)

## Summary

Still present on main: the 'Accept All' button labels in the merge editor are swapped — Input 1 (left/current) says 'Incoming' and Input 2 (right/incoming) says 'Current'. An open community PR exists but hasn't been merged.

## Change summary

The merge editor's 'Accept All' buttons have their Current/Incoming labels swapped relative to standard Git terminology.

## Best solution

Swap the labels in `src/vs/workbench/contrib/mergeEditor/browser/commands/commands.ts` so Input 1 (left) reads 'Accept All Current Changes from Left' and Input 2 (right) reads 'Accept All Incoming Changes from Right'. The open PR https://github.com/microsoft/vscode/pull/278396 addresses this — review and merge it, or apply the same fix independently.
