# Issue #135275 — Snippets in sidebar search results visually blend with filenames

[https://github.com/microsoft/vscode/issues/135275](https://github.com/microsoft/vscode/issues/135275)

## Summary

Still valid on main: file match items (`.filematch`) and code snippet items (`.linematch`) in the search results tree share the same visual weight — no bold, no background, no color differentiation. The 2024 commit a081432cfa3 bolded the `textsearchresult` section header but did not address the per-file vs per-line distinction this issue requests. 16 upvotes, team-acknowledged (roblourens), milestone On Deck — keeping open.

## Change summary

In sidebar search results, code snippet lines visually blend with filename lines because they share identical styling (no bold, background, or color difference).

## Best solution

Add a visual differentiator (e.g. `font-weight: 500` or a subtle background) to `.search-view .filematch` / `.search-view .foldermatch` relative to `.search-view .linematch`, similar to how the peek view already distinguishes file headers. The change is localized to `src/vs/workbench/contrib/search/browser/media/searchview.css`.
