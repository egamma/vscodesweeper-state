## Summary

Still valid on current main: file name rows (`.filematch`) and code snippet rows (`.linematch`) in the search sidebar have no meaningful visual distinction (no bold, no background color difference). The issue is milestoned 'On Deck', so it stays open regardless.

## Change summary

File name rows in sidebar search results are visually indistinguishable from code snippet lines, making it hard to skim results.

## Best solution

Add a visual distinction to `.filematch` rows in `src/vs/workbench/contrib/search/browser/media/searchview.css` — either `font-weight: 500`/`600` or a subtle background, similar to how the peek view already differentiates file names (as roblourens noted). The commit `a081432cfa3` only bolded the 'Text Results' heading (`.textsearchresult`), not the individual file name rows.
