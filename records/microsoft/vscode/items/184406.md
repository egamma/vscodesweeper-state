# Issue #184406 — editor.renderFinalNewline is misleading

[https://github.com/microsoft/vscode/issues/184406](https://github.com/microsoft/vscode/issues/184406)

## Summary

Valid polish request: the setting name `editor.renderFinalNewline` suggests it controls whether the final newline character is rendered, but it actually controls the line number display for that trailing empty line. The current description in code is accurate ('Render last line number when the file ends with a newline'), but the setting name itself is misleading. Already labeled correctly as `polish` + `editor-core`.

## Change summary

The setting name `editor.renderFinalNewline` misleads users into thinking it hides the final newline itself, when it only controls line number rendering for the trailing empty line.

## Best solution

Rename the setting to something like `editor.renderFinalNewlineNumber` with a deprecation alias from the old name, or improve the setting description to make its actual behavior unmistakable. This is a minor naming/docs polish — low priority but valid.
