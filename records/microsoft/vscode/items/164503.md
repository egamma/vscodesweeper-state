# Issue #164503 — Unnecessary large screen update under Remote Desktop

[https://github.com/microsoft/vscode/issues/164503](https://github.com/microsoft/vscode/issues/164503)

## Summary

Team-authored upstream issue: Chromium produces unnecessarily large repaint regions when multiple small DOM updates are scattered across the editor. VS Code is only touching strictly necessary DOM elements (confirmed by alexdima's investigation). The root cause lives in Chromium's paint-region coalescing, and the issue is pending upstream investigation — keeping open.

## Change summary

Under Remote Desktop, pressing Enter in the editor causes Chromium to repaint nearly the entire editor area instead of just the changed regions, adding input latency.

## Best solution

File an upstream Chromium bug with a minimal reproduction and paint-flashing evidence. Cross-browser comparison (Edge vs Firefox via vscode.dev) would strengthen the report. Partial mitigations on the VS Code side (e.g., the related issues https://github.com/microsoft/vscode/issues/191423 and https://github.com/microsoft/vscode/issues/193422 reducing unnecessary scrollbar/minimap repaints) can shrink the coalesced region but cannot fix the root cause.
