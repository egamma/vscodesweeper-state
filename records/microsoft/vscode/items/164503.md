# Issue #164503 — Unnecessary large screen update under Remote Desktop

[https://github.com/microsoft/vscode/issues/164503](https://github.com/microsoft/vscode/issues/164503)

## Summary

Team-authored upstream issue about Chromium's paint-region coalescing under Remote Desktop. VS Code's DOM changes are logically minimal (confirmed by alexdima), but Chromium merges disjoint dirty rects into one large region. Keeping open — author is MEMBER and the root cause is upstream in Chromium.

## Change summary

Pressing Enter in the editor causes Chromium to repaint the entire editor area (not just the changed lines and scrollbar) under Remote Desktop, adding encoding latency.

## Best solution

File a Chromium bug with reproduction tooling (as alexdima suggested) to get their compositor to emit tighter dirty rects for disjoint DOM mutations. On the VS Code side, the partial fix in https://github.com/microsoft/vscode/issues/191423 (avoiding unnecessary scrollbar repaints) shipped, and https://github.com/microsoft/vscode/issues/193422 (minimap) remains open. Further VS Code-side mitigation is limited without Chromium changes.
