# Issue #160897 — Git - add submodule support to “git checkout”

[https://github.com/microsoft/vscode/issues/160897](https://github.com/microsoft/vscode/issues/160897)

## Summary

Still valid: the git checkout command in the built-in Git extension does not pass `--recurse-submodules`. The feature has 29 upvotes, is on the Backlog milestone, and the gap remains on current main.

## Change summary

Add `--recurse-submodules` support to the built-in Git extension's checkout command for projects with submodules.

## Best solution

Add a `--recurse-submodules` flag to the `checkout()` method in `extensions/git/src/git.ts` (around line 2032), gated on a user setting (e.g. `git.checkoutRecurseSubmodules`) or exposed as a checkbox in the checkout quick-pick UI. A maintainer-provided workaround exists (`git config submodule.recurse true`) for users who want this globally.
