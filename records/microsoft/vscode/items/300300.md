# Issue #300300 — VS Code Remote WSL extension host crashes repeatedly with Copilot Chat 0.39.0 due to OOM and inconsistent chat participant registration

[https://github.com/microsoft/vscode/issues/300300](https://github.com/microsoft/vscode/issues/300300)

## Summary

Both reported problems originate in the GitHub Copilot Chat extension, not VS Code core. The participant registration errors are thrown because the extension attempts to register chat participants (claude-code, copilot-cloud-agent, copilotcli) that aren't declared in its package.json — the validation in mainThreadChatAgents2.ts is working correctly. The OOM crash in the remote extension host is not attributed to any specific VS Code core code path and is most likely driven by the Copilot Chat extension's memory usage during long sessions.

## Change summary

Remote extension host crashes with OOM and Copilot Chat logs participant registration errors for undeclared chat participants.

## Best solution

File both issues against the GitHub Copilot Chat extension (https://github.com/microsoft/vscode-copilot-release). The participant registration errors belong to the extension's manifest/packaging, and the OOM needs profiling on the extension side. VS Code core's participant validation in `mainThreadChatAgents2.ts` is behaving as designed.
