# Issue #241201 — Github Authentication Fails When Using SOCKS5 Proxy in `http.proxy` (Invalid URL Protocol)

[https://github.com/microsoft/vscode/issues/241201](https://github.com/microsoft/vscode/issues/241201)

## Summary

Confirmed upstream issue: Node.js undici does not support SOCKS5 proxies, and VS Code's switch to Electron/Node fetch surfaces this gap. Already correctly labeled `upstream` and tracked against https://github.com/nodejs/undici/issues/2224. Keep open as a known limitation pending upstream resolution.

## Change summary

GitHub authentication (and other fetch-based requests) fails when `http.proxy` is set to a SOCKS5 URL because Node.js undici only supports HTTP/HTTPS proxy protocols.

## Best solution

Wait for upstream undici SOCKS5 support (nodejs/undici#2224) to land and propagate through Electron/Node updates. The workaround `"http.fetchAdditionalSupport": false` bypasses the affected code path. No core fix is possible without the upstream change.
