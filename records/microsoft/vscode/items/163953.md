# Issue #163953 — Actions to add paths to include/exclude in search results unclear wording

[https://github.com/microsoft/vscode/issues/163953](https://github.com/microsoft/vscode/issues/163953)

## Summary

Team-authored polish issue; the wording has not been updated on current main — 'Restrict Search to Folder' and 'Exclude Folder from Search' remain as-is in searchActionsFind.ts. Keeping open for maintainer judgment on the rename and the replace-vs-append behavior raised in the comments.

## Change summary

The context menu labels 'Restrict Search to Folder' and 'Exclude Folder from Search' don't clearly communicate that they set the include/exclude fields, and the replace-instead-of-append behavior is also questioned.

## Best solution

Rename the two actions in `src/vs/workbench/contrib/search/browser/searchActionsFind.ts` to something like 'Set Files to Include' / 'Set Files to Exclude' (or 'Add to…' if the behavior is changed to append). The comment from BillDenton also raises a valid UX question about appending vs. replacing — both the wording and the behavior should be decided together.
