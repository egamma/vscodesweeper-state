# Issue #284873 — Regression: Copilot Undo reverts git-committed changes (similar to #3492)

[https://github.com/microsoft/vscode/issues/284873](https://github.com/microsoft/vscode/issues/284873)

## Summary

Still reproduces on main: when git commits happen via terminal (as agents do), the chat editing session's Keep/Undo tracking is not cleared because only the SCM UI path calls `_chat.editSessions.accept`. Team members are actively discussing the API shape — keeping open.

## Change summary

Copilot's Undo operation reverts files that were already committed via terminal git commands, because terminal-based commits don't clear the chat editing session's working set.

## Best solution

Extend the git extension's terminal command detection (`extensions/git/src/terminal.ts`, which already listens to `onDidEndTerminalShellExecution` for SCM refresh) to also call `_chat.editSessions.accept` for committed resources. The challenge connor4312 raised — determining which files were affected by the terminal command — needs resolution first, likely by inspecting the post-commit diff or tracking the repository state transition.
