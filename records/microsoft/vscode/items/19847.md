# Issue #19847 — Explore using formatters for indentation adjustment when formatters are available

[https://github.com/microsoft/vscode/issues/19847](https://github.com/microsoft/vscode/issues/19847)

## Summary

Still unimplemented on main: the reindent actions use only regex-based indentation rules (ILanguageConfigurationService) and never consult registered formatters. The feature request remains valid.

## Change summary

Use registered document formatters as a fallback for indentation adjustment when a language lacks indentation rules.

## Best solution

Enhance `ReindentLinesAction` and `ReindentSelectedLinesAction` (in `src/vs/editor/contrib/indentation/browser/indentation.ts`) to optionally query a registered `DocumentRangeFormattingEditProvider` for indentation-only edits when the language has no indentation rules. Needs design work on how to extract only whitespace-leading edits from formatter output and handle the performance/correctness trade-offs raised in comment #3.
