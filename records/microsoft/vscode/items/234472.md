# Issue #234472 — Editor GPU: Ignore token foreground color when there is a decoration with a foreground color

[https://github.com/microsoft/vscode/issues/234472](https://github.com/microsoft/vscode/issues/234472)

## Summary

Still valid on main: tokenMetadata (including foreground color) is used as the full glyph cache key even when a decoration overrides the foreground color, causing unnecessary glyph duplication in the texture atlas. The optimization has not been implemented.

## Change summary

GPU renderer duplicates glyphs in the texture atlas when decorations override the token foreground color because the token's foreground is still part of the cache key.

## Best solution

Strip the foreground color portion from `tokenMetadata` before it's used as a cache key in `textureAtlasPage.ts` when `decorationStyleSet.color` is defined — similar to how the background is already handled in `textureAtlas.ts`. This keeps glyph deduplication correct under decoration overrides.
