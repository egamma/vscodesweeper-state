# Issue #263499 — Split individual chat response parts to list rows

[https://github.com/microsoft/vscode/issues/263499](https://github.com/microsoft/vscode/issues/263499)

## Summary

The requested architectural change — splitting individual chat response parts into tree children for incremental/virtualized rendering — has not been implemented on main. The chat list renderer still treats each response as a single flat tree item, so the core performance problem with long agent sessions remains.

## Change summary

Long chat responses render all content (including many code block editors) in one shot, causing half-second editor freezes.

## Best solution

Refactor the chat list renderer in `src/vs/workbench/contrib/chat/browser/widget/chatListRenderer.ts` so that response parts (tool calls, code blocks, markdown segments) are modeled as tree children of the response node, enabling the tree's virtualization to render only visible parts. This is the approach endorsed by both the reporter and the area owner in the discussion. Consider consolidating with https://github.com/microsoft/vscode/issues/263896 and https://github.com/microsoft/vscode/issues/290490 which track overlapping chat perf work.
