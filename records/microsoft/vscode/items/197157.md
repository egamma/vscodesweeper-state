# Issue #197157 — Renaming folders in the explorer go blank before getting renamed

[https://github.com/microsoft/vscode/issues/197157](https://github.com/microsoft/vscode/issues/197157)

## Summary

Still reproduces on current main. The input box is disposed (explorerViewer.ts:1086–1094) before the async rename operation completes (fileActions.ts:1021–1025), causing the folder to render blank during the rename. Author is a CONTRIBUTOR — keeping open per policy. Related issue https://github.com/microsoft/vscode/issues/152633 is also still open with the same root cause.

## Change summary

Renaming a folder in the explorer shows a blank item while the async rename operation is in progress.

## Best solution

Keep the rename input box visible (or show a progress indicator) until the async `applyBulkEdit` in `fileActions.ts` completes. The fix likely involves restructuring the `done` callback in `explorerViewer.ts` (~line 1086) to defer disposal until after `onFinish` resolves, or awaiting the rename before calling `setEditable(stat, null)` in `fileActions.ts` (~line 1032). This would also address https://github.com/microsoft/vscode/issues/152633.
