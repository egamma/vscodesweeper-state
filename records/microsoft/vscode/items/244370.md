# Issue #244370 — Cannot read properties of null (reading 'left')

[https://github.com/microsoft/vscode/issues/244370](https://github.com/microsoft/vscode/issues/244370)

## Summary

Still reproduces on current main: line 139 of indentGuides.ts uses a non-null assertion (`!.left`) on `ctx.visibleRangeForPosition()` which can return null, causing a TypeError. The same file already uses the safe `?.left ?? fallback` pattern on lines 132 and 148–150.

## Change summary

Editor crashes with TypeError when rendering indent guides because `ctx.visibleRangeForPosition()` returns null but is accessed with a non-null assertion.

## Best solution

Replace the non-null assertion on line 139 with optional chaining and a fallback (e.g. `?.left ?? leftOffset`), matching the existing safe pattern used on lines 132 and 148–150 in the same file.
