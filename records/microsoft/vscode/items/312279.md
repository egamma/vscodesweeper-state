# Issue #312279 — Let me / multiple skills in one message

[https://github.com/microsoft/vscode/issues/312279](https://github.com/microsoft/vscode/issues/312279)

## Summary

Keep open: this is a team-authored feature request in the Backlog, and the discussion is still a product/design question about how explicit skills should compose with slash commands and context references.

## Change summary

Allow a chat request to explicitly reference multiple skills in one message instead of treating each slash entry as a separate command.

## Best solution

The chat/agent owners should decide whether multiple skill references are modeled as slash commands, context attachments such as a future `#skill:foo`, or a separate trigger, then track the implementation under the chosen chat input and skill-resolution design.
