# Issue #282332 — No Copilot Models in Model List - Insiders

[https://github.com/microsoft/vscode/issues/282332](https://github.com/microsoft/vscode/issues/282332)

## Summary

The root cause — Node.js system certificate loading breaking TLS connections — is tracked by the canonical issue #287739 ("Track loading system certificates with Node.js built-in support"), which is on the On Deck milestone. The setting `http.systemCertificatesNode` was disabled by default in commit 2d19cfa394c (#277838), and remains `false` on current main. This report is a specific manifestation of that known problem.

## Change summary

Copilot model list fails to load in Insiders due to 'unable to get issuer certificate' TLS errors when `http.systemCertificatesNode` is enabled.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/287739 which tracks the full lifecycle of enabling Node.js system certificate loading by default. The setting defaults to `false` on current main, so new users are unaffected; the reporter may need to explicitly reset the setting if it was persisted.
