# Issue #70444 — TerminalOptions.waitOnExit API

[https://github.com/microsoft/vscode/issues/70444](https://github.com/microsoft/vscode/issues/70444)

## Summary

The waitOnExit capability exists internally but is not exposed in the public extension API. With 24 upvotes and active use cases from multiple extensions, this remains a valid API proposal worth keeping open.

## Change summary

Expose waitOnExit in the public TerminalOptions extension API so extensions can control whether the terminal waits after process exit.

## Best solution

Finalize the proposed API by exposing `waitOnExit` on `TerminalOptions` in `vscode.d.ts` — the internal plumbing already supports `boolean | string | ((exitCode: number) => string)` via `WaitOnExitValue` in `src/vs/platform/terminal/common/terminal.ts`. The extension host protocol currently only passes a boolean; widen it to match the internal type or a suitable subset (enum + callback).
