# Issue #145026 — Git - Request: don't save document on ranges revert (`git.revertSelectedRanges`)

[https://github.com/microsoft/vscode/issues/145026](https://github.com/microsoft/vscode/issues/145026)

## Summary

Still valid on main: `_revertChanges` unconditionally calls `modifiedDocument.save()` after applying the revert edit, triggering save participants like trimTrailingWhitespace. The request to either save without formatting or make saving optional is a reasonable enhancement that remains unaddressed.

## Change summary

The `git.revertSelectedRanges` command always saves the document after reverting, which triggers save participants (e.g. trim trailing whitespace) that can re-modify the reverted lines.

## Best solution

Add a setting (e.g. `git.revertSelectedRanges.autoSave`) or switch the save call in `_revertChanges` to use `workbench.action.files.saveWithoutFormatting` so save participants don't interfere. The fix is localized to `extensions/git/src/commands.ts` in the `_revertChanges` method.
