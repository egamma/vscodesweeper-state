# Issue #277536 — Share f5 keybinding for merge editor and editor with changes from agent

[https://github.com/microsoft/vscode/issues/277536](https://github.com/microsoft/vscode/issues/277536)

## Summary

Team-authored accessibility feature request to unify the Alt+F5 keybinding so it navigates changes in both the merge editor and the regular editor (e.g. reviewing agent diffs). Team members have discussed a solution approach and the merge-editor owner expressed willingness to align keybindings.

## Change summary

Alt+F5 (Go to Next Change) should work in both the merge editor and the regular diff/multi-diff editor so users don't lose navigation when switching contexts.

## Best solution

Adjust the `when` clause for the merge-editor's next-change keybinding so it fires for both `activeEditor == 'mergeEditor'` and `activeEditor == 'multiDiffEditor'`, as discussed in the comments. The relevant keybinding registration likely lives in `src/vs/workbench/contrib/mergeEditor/` keybinding contributions.
