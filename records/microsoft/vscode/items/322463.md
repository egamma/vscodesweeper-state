# Issue #322463 — Completions not sent after every character in debug console

[https://github.com/microsoft/vscode/issues/322463](https://github.com/microsoft/vscode/issues/322463)

## Summary

Still reproduces on main: the debug console completion provider returns a CompletionList without `incomplete: true`, so the suggest widget caches and locally filters results instead of re-querying the debug adapter on every keystroke.

## Change summary

Debug console completions are not re-fetched from the debug adapter after every character typed because the completion list is not marked incomplete.

## Best solution

Set `incomplete: true` on the returned CompletionList in `src/vs/workbench/contrib/debug/browser/repl.ts` (line ~324) so the editor re-invokes the provider on each keystroke, ensuring the debug adapter always receives a fresh completions request.
