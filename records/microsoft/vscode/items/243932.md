# Issue #243932 — `editor.action.clipboardCopyAction` command no longer works

[https://github.com/microsoft/vscode/issues/243932](https://github.com/microsoft/vscode/issues/243932)

## Summary

Still reproducible on main: the copy command's `hasTextFocus()` gate (line 223 of clipboard.ts) prevents execution when running in an automated test window without user interaction. The Electron workaround (884364539d1) addresses a different failure mode (execCommand not firing events), not the focus-gate issue.

## Change summary

Running `editor.action.clipboardCopyAction` via `vscode.commands.executeCommand` in extension tests fails because the test window lacks user activation/focus.

## Best solution

The copy command implementation at `src/vs/editor/contrib/clipboard/browser/clipboard.ts` line 223 gates on `hasTextFocus()`. For programmatic invocation via `executeCommand`, this gate could be relaxed or the command could fall through to a direct clipboard write (similar to the Electron bug workaround) when triggered programmatically rather than via keyboard shortcut. Alternatively, document this as a platform limitation for extension test authors.
