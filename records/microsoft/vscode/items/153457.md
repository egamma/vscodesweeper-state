# Issue #153457 — Profiles: Workbench startup scaffolding should take profile into account

[https://github.com/microsoft/vscode/issues/153457](https://github.com/microsoft/vscode/issues/153457)

## Summary

Still reproduces on main: the splash/startup scaffold stores theme data globally via IStateService (a single file) with no profile key, so switching profiles with different themes causes a brief color flash on startup.

## Change summary

Workbench startup scaffold shows wrong theme colors when opening a window in a profile that uses a different theme than the last-saved splash.

## Best solution

Store splash data per-profile (keyed by profile ID) in the state service or in a profile-aware storage layer, so that getWindowSplash in ThemeMainService can return the correct colors for the profile about to be loaded. Likely files: src/vs/platform/theme/electron-main/themeMainServiceImpl.ts (storage keys need profile disambiguation), src/vs/workbench/contrib/splash/browser/partsSplash.ts (saving), and the electron-browser workbench bootstrap (src/vs/code/electron-browser/workbench/workbench.ts) where splash data is read before profile services are available.
