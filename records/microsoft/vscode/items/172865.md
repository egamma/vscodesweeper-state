# Issue #172865 — npm.packageManager=auto ignores task's cwd

[https://github.com/microsoft/vscode/issues/172865](https://github.com/microsoft/vscode/issues/172865)

## Summary

Verified duplicate: this reports the same bug as #261630 — npm.packageManager/scriptRunner auto-detection ignores subdirectory paths and falls back to npm. The newer issue is more comprehensive and actively tracked.

## Change summary

When npm.packageManager is set to 'auto', tasks with a cwd pointing to a subdirectory still detect the package manager from the workspace root instead of the subdirectory containing the lockfile.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/261630 which covers this exact bug more comprehensively (all task invocation paths, multiple package managers). The fix lives in `extensions/npm/src/tasks.ts` — `getRunScriptCommand` and `resolveTask` need to pass the actual package.json directory (not the workspace folder root) to `detectPackageManager`.
