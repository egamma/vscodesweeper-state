# Issue #279327 — [Copilot] Agent always uses pwsh but grabs args list from default terminal profile.

[https://github.com/microsoft/vscode/issues/279327](https://github.com/microsoft/vscode/issues/279327)

## Summary

Fixed on main and shipped in 1.120.0. The agent host terminal shell handling was reworked to properly respect the user's default terminal profile (both executable and args), resolving the mismatch where pwsh was launched with another profile's args.

## Change summary

Copilot agent mode always used PowerShell as the shell but incorrectly grabbed args from the user's default terminal profile (e.g. cmd's /k), causing immediate exit.

## Best solution

Keep the shipped fix in commit 8b28f914f2a (1.120.0); no further change needed. The agent host now correctly resolves and uses the default terminal profile's shell and matching args.
