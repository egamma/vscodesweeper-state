# Issue #209116 — Problems tab errors claim background preLaunchTask has errors

[https://github.com/microsoft/vscode/issues/209116](https://github.com/microsoft/vscode/issues/209116)

## Summary

Still reproduces on main: debugTaskRunner.ts line 79 reads all global error markers instead of only those produced by the specific preLaunchTask, so unrelated Problems-tab errors trigger a false 'Error exists after running preLaunchTask' dialog for background tasks. Maintainers discussed a fix path but no change has landed.

## Change summary

Background preLaunchTasks falsely report errors when unrelated files have problems in the Problems tab, because the error check queries all markers globally instead of only those produced by the task.

## Best solution

Scope the marker check in `src/vs/workbench/contrib/debug/browser/debugTaskRunner.ts` (line 79) to only count markers owned by the specific task's problem matcher, using the per-task marker tracking already present in `AbstractProblemCollector.markers`. This requires exposing the task's own markers (or an error-count accessor) from the task system so `debugTaskRunner` can query task-scoped errors instead of the global `IMarkerService`.
