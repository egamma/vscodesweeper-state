## Change summary

The `runTests` agent tool returns 'No tests found' for .NET (and Python) projects whose tests are visible and runnable in the Test Explorer.

## Best solution

The `RunTestTool._getFileTestsToRun` in `src/vs/workbench/contrib/testing/common/testingChatAgentTool.ts` relies on `testsInFile` from the test collection; if the test provider hasn't discovered tests for the given URI (e.g. because C# Dev Kit indexes by DLL or solution, not by source file), the tool silently returns empty. The fix likely involves either triggering discovery before querying, or falling back to running all tests when file-scoped discovery yields nothing. connor4312 is the right owner to decide.
