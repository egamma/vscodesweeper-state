# Issue #124498 — Components that offer to open an editor should specify the `override` that makes most sense

[https://github.com/microsoft/vscode/issues/124498](https://github.com/microsoft/vscode/issues/124498)

## Summary

This debt issue remains partially unresolved. The centralized fix in codeEditorService was reverted (commit 8bcc883c6ab, fixing #128756), and the broader adoption across search, problems, and other editor-opening components has not been completed. The outline-specific partial fix (commit 1cbd1c6b651) was kept, but the core problem — components opening editors without specifying the right override — persists on current main.

## Change summary

UI components that open editors (search, problems, outline, custom views) should specify the correct editor override to avoid opening the wrong editor type when the user has configured a non-default editor.

## Best solution

Audit all components that open editors via IEditorService or ICodeEditorService and ensure they pass the appropriate `override` (e.g. `EditorResolution.EXCLUSIVE_ONLY` or a specific `editorId`) so that text-oriented UI surfaces (search results, problems, document symbol outline) open a text editor, and notebook-oriented surfaces open the correct notebook editor. The last commenter suggested using `editorId` (now available for all editor types) as a simpler approach than the reverted centralized fix in codeEditorService.ts.
