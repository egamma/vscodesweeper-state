## Summary

Real bug confirmed by multiple users running VS Code natively on WSL2 with WSLg. Already triaged with `help wanted` and `confirmation-pending`. No fix found on current main; the issue remains valid for community contribution. ## Change summary

VS Code's maximized window is shifted right and down on WSL2 Linux (WSLg), making parts of the window inaccessible and the cursor hotspot misaligned.

## Best solution

Investigate whether the custom titlebar or Electron's window bounds calculation on WSLg causes the offset. Since other Electron apps reportedly maximize correctly (comment 2), this may be a VS Code-specific window-management issue — possibly related to how VS Code configures `BrowserWindow` options or handles maximize on X11/Wayland under WSLg. The `help wanted` label is appropriate; a community contributor could bisect the WSL version and VS Code version to narrow the root cause.
