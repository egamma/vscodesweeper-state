# Issue #208350 — Tasks using command variables don't run in split terminals

[https://github.com/microsoft/vscode/issues/208350](https://github.com/microsoft/vscode/issues/208350)

## Summary

Still reproduces on main: command variables (e.g. `${command:workbench.action.terminal.new}`) used as task commands execute during variable resolution, creating terminals as a side effect that bypasses the task system's `presentation.group` split logic in `_doCreateTerminal`. No fix found in recent history.

## Change summary

Tasks using command variables as their command don't respect `presentation.group` for split terminal grouping.

## Best solution

The fix likely needs to detect when a command variable is the task's process command and either defer its execution until terminal creation respects the group, or handle the resulting terminal within the group-splitting flow in `_doCreateTerminal`. The relevant code is in `src/vs/workbench/contrib/tasks/browser/terminalTaskSystem.ts` around the variable resolution (`_resolveCommandAndArgs`) and terminal creation (`_doCreateTerminal`) paths.
