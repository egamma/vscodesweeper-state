# Issue #293109 — Display Context Window Information Immediately After Model Selection in GitHub Copilot

[https://github.com/microsoft/vscode/issues/293109](https://github.com/microsoft/vscode/issues/293109)

## Summary

Valid feature request to show context window information immediately after model selection, before any chat request is made. The current context-usage widget only renders after at least one response exists. Still an open gap on main — routing to the copilot area for triage.

## Change summary

Display context window size in the chat context-usage widget immediately after the user selects a model, without requiring a prior chat response.

## Best solution

Extend `ChatContextUsageWidget.setSelectedModel` to render a denominator-only state (showing the model's total context window) even when no response exists yet, so users can see the available context budget before sending their first message. The widget in `src/vs/workbench/contrib/chat/browser/widgetHosts/viewPane/chatContextUsageWidget.ts` currently guards rendering on `this._currentResponse`; removing that gate for the denominator display would address this. Related but distinct from https://github.com/microsoft/vscode/issues/323783, which asks for context-window info in the model picker hover.
