# Issue #111578 — Progress: cannot cancel

[https://github.com/microsoft/vscode/issues/111578](https://github.com/microsoft/vscode/issues/111578)

## Summary

Valid feature request that remains partially unaddressed: upload cancellation exists, but download cancellation is web-only (`cancellable: isWeb`), and bulk edit operations (copy/move/delete) still lack user-facing cancellation in the explorer. Keeping open as a legitimate backlog item.

## Change summary

Allow users to cancel file operations (download, upload, copy, move, delete) from the explorer progress UI.

## Best solution

Enable `cancellable: true` for desktop downloads in `src/vs/workbench/contrib/files/browser/fileImportExport.ts` (line 614, change `isWeb` to `true`), and investigate passing cancellation tokens through the BulkEditService for copy/move/delete operations as discussed by the team in the comments.
