# Issue #156351 — Confusing dialog for contributed launch configs when errors occur

[https://github.com/microsoft/vscode/issues/156351](https://github.com/microsoft/vscode/issues/156351)

## Summary

Still open: the VariableError infrastructure was added (PR #157788) but the debug service never adopted it to show friendlier error messages for contributed/dynamic launch configs. Author is a team member, so keeping open.

## Change summary

When a dynamic debug launch config using ${file} fails because no editor is open, the error dialog shows confusing variable-centric language and a misleading 'Open launch.json' button, even though the user never created a launch.json.

## Best solution

The debug service (src/vs/workbench/contrib/debug/browser/debugService.ts) should catch VariableError from the configuration resolver and present a user-friendly message tailored to contributed configs — e.g. 'A file must be open to run this configuration' instead of referencing ${file} variables. The VariableError/VariableKind infrastructure in src/vs/workbench/services/configurationResolver/common/ is already in place; the debug service just needs to consume it. PR #157049 already removed the launch.json button for dynamic configs, so the remaining gap is the error message text.
