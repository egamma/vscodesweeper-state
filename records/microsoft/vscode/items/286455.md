# Issue #286455 — documentSelector `json` for LSP-Connection will spam `textDocument/didOpen|didClose` request for all `package.json`

[https://github.com/microsoft/vscode/issues/286455](https://github.com/microsoft/vscode/issues/286455)

## Summary

Still reproduces on main: the npm extension's `getScripts()` at extensions/npm/src/tasks.ts:487 uses `workspace.openTextDocument()` for every package.json, triggering LSP didOpen/didClose spam for any extension registering a json document selector. Two fix PRs were closed without merging; the issue remains valid and needs a fix.

## Change summary

The built-in npm extension opens every package.json via the text document API during task detection, causing excessive LSP didOpen/didClose notifications to language servers registered for json files.

## Best solution

Replace `workspace.openTextDocument()` in `extensions/npm/src/tasks.ts:getScripts()` with `workspace.fs.readFile()` and parse the JSON directly, avoiding LSP notification side-effects. Additionally, consider whether the debug adapter manager's eager task prefetching (src/vs/workbench/contrib/debug/browser/debugAdapterManager.ts) should be deferred until a launch.json is actually opened.
