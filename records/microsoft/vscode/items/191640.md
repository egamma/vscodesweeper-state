Team-authored accessibility polish issue for the diff editor's moved-code blocks — tab order is unexpected, causing screen readers to announce the same element twice. Keeping open per policy (MEMBER author).

## Change summary

Tabbing through a moved code block in the diff editor focuses an intermediate invisible element before visibly focusing the 'Code moved with changes' decoration, causing double screen-reader announcements.

## Best solution
