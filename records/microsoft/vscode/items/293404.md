# Issue #293404 — "Failed to serialize MCP servers" error from private MCP registry

[https://github.com/microsoft/vscode/issues/293404](https://github.com/microsoft/vscode/issues/293404)

## Summary

Confirmed bug still present on main — the falsy check on `description` rejects empty strings returned by Azure API Center MCP registries. An open PR (#295563) addresses exactly this but has not yet merged.

## Change summary

MCP gallery serialization fails when a registry returns a server with an empty string description, because the validation treats empty strings as invalid.

## Best solution

Land or supersede the fix in https://github.com/microsoft/vscode/pull/295563 — change the falsy check `!from.server.description` to a null check `from.server.description == null` in `src/vs/platform/mcp/common/mcpGalleryService.ts` (line 595 and any parallel schema version).
