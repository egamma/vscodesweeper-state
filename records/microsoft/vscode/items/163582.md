# Issue #163582 — Git - Remote WSL2 git.path does not respect Windows Git.exe

[https://github.com/microsoft/vscode/issues/163582](https://github.com/microsoft/vscode/issues/163582)

## Summary

Still reproduces on main: the git extension passes Linux-style paths as cwd to Windows git.exe without WSL path translation, causing `rev-parse --show-toplevel` to fail. No fix or WSL-specific path handling has been added.

## Change summary

When `git.path` points to Windows git.exe in a WSL2 remote session, VS Code fails to detect git repositories because Linux paths are passed untranslated to the Windows binary.

## Best solution

The git extension's `exec`/`spawn` path in `extensions/git/src/git.ts` needs to detect when a Windows git.exe is being invoked from a Linux environment (WSL) and translate the `cwd` and relevant path arguments via `wslpath` or equivalent mapping (e.g. `/home/user/project` → `//wsl$/Ubuntu/home/user/project`). This is a localized change in the git spawning layer but requires careful cross-platform testing.
