# Issue #275007 — "files to include" doesn't work with folder names containing glob characters

[https://github.com/microsoft/vscode/issues/275007](https://github.com/microsoft/vscode/issues/275007)

## Summary

Still reproduces on main: user-typed input in the 'files to include' field has no working escape mechanism for glob characters like `[` and `]` in folder names. The `escapeGlobPattern` utility exists but is only used for programmatic paths (e.g., 'Find in Folder' from the explorer context menu), not for user-typed patterns.

## Change summary

Searching in the 'files to include' field fails for folders whose names contain glob metacharacters (`[`, `]`, `?`, `*`) because there is no user-facing escape mechanism.

## Best solution

Either support backslash-escaping of glob metacharacters in the user-typed 'files to include' / 'files to exclude' input (adjusting the glob parser in `src/vs/base/common/glob.ts` and the `splitGlobAware` function), or document and fix the `[[]` bracket-escape syntax so it works end-to-end through the query builder and ripgrep arguments. The programmatic `escapeGlobPattern` in `src/vs/workbench/services/search/common/queryBuilder.ts` already demonstrates the correct escaping — the gap is exposing a working mechanism to manual input.
