# Issue #221789 — An extension quick pick can break quick open

[https://github.com/microsoft/vscode/issues/221789](https://github.com/microsoft/vscode/issues/221789)

## Summary

Team-authored issue with a confirmed regression (Feb 2026). The original fix from PR #222248 is still in place but the bug has recurred through a likely different code path. Keeping open for maintainer investigation.

## Change summary

An extension quick pick that filters items before display can throw a TreeError, breaking the quick pick selection and subsequently breaking quick open (Ctrl+P).

## Best solution

Investigate the regression in `src/vs/platform/quickinput/browser/quickInputList.ts` — the hidden-item guard in `setFocusedElements` is present but the race between extension-supplied items and tree updates likely needs a broader fix, possibly in how `activeItems` are set when items haven't yet been rendered in the tree.
