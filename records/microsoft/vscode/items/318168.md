# Issue #318168 — VS Code 1.121.0: Open Chat (Plan) keybinding no longer works (`workbench.action.chat.openPlan`)

[https://github.com/microsoft/vscode/issues/318168](https://github.com/microsoft/vscode/issues/318168)

## Summary

Confirmed on current main: custom mode commands (including `workbench.action.chat.openPlan`) are dynamically registered only while a chat widget is focused, so keybindings, Command Palette entries, and chat-tip links that invoke them fail when no chat widget has focus. Telemetry confirms 20–24 users still hitting this on v1.130.0-insider.

## Change summary

The `workbench.action.chat.openPlan` command is not found when invoked via keybindings or chat tips because it is only registered while a chat widget has focus.

## Best solution

In `ChatAgentActionsContribution` (src/vs/workbench/contrib/chat/browser/chat.shared.contribution.ts), decouple the dynamic mode-action registration from focused-widget state so that custom mode commands remain in the command registry globally (or at least register a fallback that opens a new chat in the requested mode). The chat tip in chatTipCatalog.ts that renders a `command:workbench.action.chat.openPlan` link also needs the command to exist outside of focus.
