# Issue #111945 — Ligature incorrectly activated for 2 different tokens

[https://github.com/microsoft/vscode/issues/111945](https://github.com/microsoft/vscode/issues/111945)

## Summary

Keep open as a still-actionable editor rendering bug. Current main still renders adjacent token parts as neighboring spans without an explicit ligature break, and the 2026 comment provides a fresh reproduction plus a plausible renderer-level workaround.

## Change summary

Font ligatures can form across separate syntax token categories, so text like `>=?` may display the `>=` ligature even when `>` and `=` are different tokens.

## Best solution

Handle this in the editor line renderer, likely around `src/vs/editor/common/viewLayout/viewLineRenderer.ts`, by preventing ligature shaping across token or decoration boundaries when `editor.fontLigatures` is enabled while preserving DOM position mapping and existing long-token behavior.
