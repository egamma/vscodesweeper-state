# Issue #111945 — Ligature incorrectly activated for 2 different tokens

[https://github.com/microsoft/vscode/issues/111945](https://github.com/microsoft/vscode/issues/111945)

## Summary

Still reproduces on main: the view line renderer emits adjacent `<span>` elements for different tokens without any ligature-breaking mechanism, so fonts with ligatures (e.g. Fira Code) incorrectly merge glyphs across token boundaries (like `>` and `=` from different scopes).

## Change summary

Font ligatures activate across adjacent tokens that belong to different TextMate scopes, producing incorrect combined glyphs.

## Best solution

Insert a ligature-breaking boundary (e.g. `<wbr>` or zero-width space, or apply `unicode-bidi: isolate` on token spans when font ligatures are enabled) in `src/vs/editor/common/viewLayout/viewLineRenderer.ts` between spans whose combined characters form a known ligature sequence. The inlay-hints fix (commit 7a135a6acd7) used `unicode-bidi` for the same class of problem at a different boundary.
