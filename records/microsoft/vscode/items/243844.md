# Issue #243844 — Inserting backslash on Japanese romaji keyboard conflicts with keyboard shortcut for triggering inline suggestion

[https://github.com/microsoft/vscode/issues/243844](https://github.com/microsoft/vscode/issues/243844)

## Summary

Keep open: the report identifies a concrete default keybinding conflict on macOS Japanese romaji layouts, and the reporter confirmed it still happens after a maintainer follow-up. It is already routed to the IME/editor input area and needs owner judgment on whether the inline suggestion trigger should avoid this layout-specific text input chord.

## Change summary

On Japanese romaji keyboards on macOS, Option+Yen no longer inserts a backslash because it matches the default inline suggestion trigger keybinding.

## Best solution

Editor input and suggest owners should decide whether the default `editor.action.inlineSuggest.trigger` binding needs a macOS/layout-specific exclusion or replacement so text-producing Japanese keyboard chords are not consumed by inline suggestions.
