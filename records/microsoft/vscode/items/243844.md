# Issue #243844 — Inserting backslash on Japanese romaji keyboard conflicts with keyboard shortcut for triggering inline suggestion

[https://github.com/microsoft/vscode/issues/243844](https://github.com/microsoft/vscode/issues/243844)

## Summary

The `alt+\` keybinding for `editor.action.inlineSuggest.trigger` is not defined in VS Code core — it is contributed by the GitHub Copilot extension (marketplace version). The conflict with ⌥¥ on Japanese romaji keyboards is an extension issue, not a core VS Code bug.

## Change summary

On Japanese romaji keyboards, pressing ⌥¥ to insert a backslash triggers the Copilot inline suggestion instead, because the Copilot extension registers `alt+\` as a default keybinding.

## Best solution

Report this to the GitHub Copilot extension (https://github.com/microsoft/vscode-copilot-release/issues) so they can adjust or remove the `alt+\` default keybinding, or scope it to avoid conflict with Japanese keyboard layouts.
