# Issue #323231 — Let extensions change a terminal tab's icon and color after it is created

[https://github.com/microsoft/vscode/issues/323231](https://github.com/microsoft/vscode/issues/323231)

## Summary

Valid feature request for a missing extension API capability. The Terminal interface currently has no writable iconPath or color properties, and no commands exist to change them programmatically after creation. The internal machinery (TerminalInstance.changeColor/changeIcon) exists but is not exposed to extensions. At 9 upvotes in the Backlog Candidates process (needs 20), this should stay open for the community vote.

## Change summary

Allow extensions to change a terminal tab's icon and color after the terminal has been created, either via writable API properties or argument-taking commands.

## Best solution

Expose writable `iconPath` and `color` properties on the `Terminal` API (or add `workbench.action.terminal.changeColorWithArg` / `changeIconWithArg` commands following the `renameWithArg` pattern). The internal plumbing already exists in TerminalInstance — the gap is purely in the extension API surface. This is a reasonable core API addition rather than an extension-candidate, since it requires exposing internal terminal instance methods.
