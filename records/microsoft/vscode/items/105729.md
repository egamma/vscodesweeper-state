# Issue #105729 — Snap: Hardware acceleration is broken for wayland

[https://github.com/microsoft/vscode/issues/105729](https://github.com/microsoft/vscode/issues/105729)

## Summary

Still reproduces on current main: the snap launch script (`resources/linux/snap/electron-launch`) hardcodes `--ozone-platform=x11` on line 279, forcing XWayland even on native Wayland sessions. This prevents proper GPU/hardware acceleration under Wayland. Multiple fixes have landed (GL libraries in 3b80131054e, crash fix in c6bc1068180) but the fundamental X11 forcing remains.

## Change summary

Snap-packaged VS Code cannot use hardware-accelerated GPU rendering on Wayland because the snap launcher forces `--ozone-platform=x11`.

## Best solution

Enable native Wayland rendering in the snap by conditionally passing `--ozone-platform=wayland` (or `--ozone-platform=auto`) when a Wayland session is detected, similar to how the launch script already detects Wayland and sets `GDK_BACKEND=wayland`. This likely also requires upgrading the snap base from core20 to a newer base and ensuring the bundled GL/EGL libraries support Wayland compositing. The fix lives in `resources/linux/snap/electron-launch` and `resources/linux/snap/snapcraft.yaml`.
