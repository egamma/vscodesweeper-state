# Issue #308966 — One-Click Multi-Agent Code Review Loop

[https://github.com/microsoft/vscode/issues/308966](https://github.com/microsoft/vscode/issues/308966)

## Summary

This requests a highly opinionated, multi-agent orchestration workflow with specific UX (dedicated buttons, config panels, output panes) that is a natural fit for an extension built on top of the existing chat/agent APIs rather than baked into VS Code core.

## Change summary

Add a built-in one-click multi-agent review loop where an implementer agent and reviewer agent automatically negotiate code changes across multiple rounds.

## Best solution

Build this as a Marketplace extension leveraging the chat participants API and agent infrastructure — the specific review-loop workflow, role assignment, negotiation protocol, and round-based UX are specialized enough that they belong in extension territory, not core.

## Vision fit

- **Fit:** rejected
- **Reason:** While agent experiences are a priority, VS Code's core value is 'the power lives in extensions' — a self-contained, opinionated multi-agent workflow with its own UI surface is exactly the kind of specialized capability that belongs in an extension, not the default product.
- **Evidence:** Value: 'The power lives in extensions. Although the UI choices extension authors can make are limited, authors can write powerful extensions — up to turning VS Code into a full IDE.'
