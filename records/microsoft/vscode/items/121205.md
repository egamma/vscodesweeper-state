# Issue #121205 — Add input validation support to quick picks

[https://github.com/microsoft/vscode/issues/121205](https://github.com/microsoft/vscode/issues/121205)

## Summary

Valid feature request: `validationMessage` exists on the internal `IQuickPick` interface but is not exposed on the `QuickPick` extension API. The gap remains on current main — extensions mixing input entry with pickable choices still cannot validate.

## Change summary

Expose `validationMessage` (and severity) on the `QuickPick` extension API so extensions can validate quick-pick input the same way they can validate `InputBox` input.

## Best solution

Add `validationMessage: string | InputBoxValidationMessage | undefined` to the `QuickPick<T>` interface in `src/vscode-dts/vscode.d.ts` and wire it through `extHostQuickOpen.ts` (the internal plumbing in `src/vs/platform/quickinput/common/quickInput.ts` already supports it). This is a small API surface addition with severity support already landed via PR #187862.
