# Issue #182672 — Curly brace matching with Perl split command

[https://github.com/microsoft/vscode/issues/182672](https://github.com/microsoft/vscode/issues/182672)

## Summary

Still present on main: the Perl TextMate grammar (from textmate/perl.tmbundle) doesn't tokenize regex delimiters after `split`, so curly braces inside the regex are treated as brackets by the bracket matcher. The `help wanted` label is appropriate — this is a valid community-fixable grammar improvement.

## Change summary

Curly braces inside a Perl `split /regex/` pattern are incorrectly matched as bracket pairs by VS Code's bracket matching.

## Best solution

Fix the upstream Perl TextMate grammar (textmate/perl.tmbundle) to recognize regex context after `split` and other Perl operators that take regex arguments, then pull the updated grammar into VS Code via the periodic grammar update process. The `help wanted` label is correctly placed for a community contribution.
