# Issue #308350 — Agents: Default VSC chat input values initially rendered in Agents chat input?

[https://github.com/microsoft/vscode/issues/308350](https://github.com/microsoft/vscode/issues/308350)

## Summary

Team-authored polish issue about a visual flicker in the Agents window — chat input briefly renders default VS Code values (tip widget, 'Local' picker) before the restored session content loads. Keeping open per team-member rule.

## Change summary

When reopening an existing chat session in the Agents app, the chat input briefly flashes default VS Code chat values before loading the session's actual state.

## Best solution

Defer rendering the chat input (or hide it) until the session state is resolved, so the tip widget and default picker value never flash. The fix likely lives in the sessions chat widget initialization path under src/vs/sessions/.
