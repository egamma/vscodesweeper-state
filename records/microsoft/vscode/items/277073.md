# Issue #277073 — Japanese IME not working on Linux (Wayland + fcitx5)

[https://github.com/microsoft/vscode/issues/277073](https://github.com/microsoft/vscode/issues/277073)

## Summary

This is an upstream Electron/Chromium issue: on Wayland sessions with fcitx5, the GTK IM module is not auto-detected without explicitly setting GTK_IM_MODULE. VS Code has no control over how Electron discovers input method modules on Linux/Wayland.

## Change summary

Japanese IME via fcitx5 stopped working on Wayland without manually setting GTK_IM_MODULE=fcitx.

## Best solution

Track as upstream — the root cause is in Electron's Ozone/Wayland layer not auto-discovering the fcitx5 IM module. VS Code could potentially ship a workaround by detecting fcitx5 and setting GTK_IM_MODULE in its launch scripts, but the proper fix belongs in Electron/Chromium's input method broker for Wayland. Consider filing or linking an Electron issue.
