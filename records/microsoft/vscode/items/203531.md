# Issue #203531 — Inconsistent behavior of diff line border

[https://github.com/microsoft/vscode/issues/203531](https://github.com/microsoft/vscode/issues/203531)

## Summary

Still reproducible on main: in inline diff mode, deleted-line view zones get a border that only spans the text content width, while inserted lines (rendered as full-line editor decorations) get a border spanning the full editor width. The CSS applies the same border rule to both `.line-delete` and `.line-insert`, but the underlying DOM structure differs between view zones and editor decorations.

## Change summary

In the inline diff view, the `diffEditor.removedTextBorder` only wraps around the text content of deleted lines rather than extending the full editor width like `insertedTextBorder` does.

## Best solution

The fix likely lives in `src/vs/editor/browser/widget/diffEditor/components/diffEditorViewZones/diffEditorViewZones.ts` (around line 209–210) or in the CSS at `src/vs/editor/browser/widget/diffEditor/style.css`. The `deletedCodeDomNode` view zone container needs to be styled to stretch to full width so that the `.line-delete` border matches the full-line behavior of `.line-insert`.
