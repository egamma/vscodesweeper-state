# Issue #105935 — Interactive playground eats arrow keys

[https://github.com/microsoft/vscode/issues/105935](https://github.com/microsoft/vscode/issues/105935)

## Summary

The original reporter could not reproduce this intermittent context-key leak; no commenter has provided a reliable reproduction. After nearly five years with no confirmed repro steps, this needs info on whether it still occurs on current builds.

## Change summary

The interactive playground's `interactivePlaygroundFocus` context key sometimes remains true after navigating away, causing arrow-key bindings to be captured by the walkthrough instead of the active editor.

## Best solution

If still reproducible, the fix would be in `src/vs/workbench/contrib/welcomeWalkthrough/browser/walkThroughPart.ts` — ensuring `editorFocus.reset()` is called reliably when the WalkThroughPart loses visibility (not just on DOM blur). If no longer reproducible on current builds, close as not-reproducible.
