## Summary

Team-authored bug report: after undoing a chat-applied edit, the file still appears as modified in the SCM/dirty indicator. Keeping open per team-member policy.

## Change summary

A file remains marked as changed (dirty) after the user undoes the edit that was applied by chat, even though the file content matches the saved state.

## Best solution

The chat edit integration should properly reconcile undo state with the file's dirty tracking, so that undoing all chat-applied changes restores the file to a clean state. Likely involves the interaction between chat edit session tracking and the text model's version/alternative-version comparison in the working copy service.
