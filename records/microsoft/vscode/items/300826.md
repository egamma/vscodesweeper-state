# Issue #300826 — Accessibility: Only 2 of multiple VS Code windows appear in Alt+Tab on Windows 11

[https://github.com/microsoft/vscode/issues/300826](https://github.com/microsoft/vscode/issues/300826)

## Summary

Likely an upstream Electron issue — Windows exposes only one MainWindowHandle per Electron process, limiting Alt+Tab visibility. VS Code's source has no skipTaskbar usage; the behavior is governed by how Electron/Chromium manages window handles on Windows.

## Change summary

Only 2 of multiple VS Code windows appear in the Windows 11 Alt+Tab switcher, making additional windows inaccessible to screen reader users.

## Best solution

Route to workbench-window for triage. The root cause appears to be Electron's MainWindowHandle handling on Windows (referenced electron/electron#7850). The team should confirm whether this is purely upstream or whether VS Code can work around it via Electron BrowserWindow options, and if upstream, file or link the Electron tracking issue.
