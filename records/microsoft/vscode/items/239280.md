# Issue #239280 — Search progress goes forever after pasting a large string into search view

[https://github.com/microsoft/vscode/issues/239280](https://github.com/microsoft/vscode/issues/239280)

## Summary

Team-authored bug, must keep open. The search progress indicator runs forever when a very large string is pasted because the search promise in `doSearch` either hangs or errors silently without calling `progressComplete`. No fix commit found on current main.

## Change summary

Pasting a large string (e.g. full package.json contents) into the search view causes the progress indicator to spin indefinitely.

## Best solution

Add a length guard or timeout in `searchView.ts`'s `doSearch` or `_onQueryChanged` — either reject/skip queries above a reasonable length threshold (showing a user-facing message), or ensure `progressComplete` is always called even on abnormal search termination. The fix would be localized to `src/vs/workbench/contrib/search/browser/searchView.ts`.
