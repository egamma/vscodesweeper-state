# Issue #297386 — "Error: Illegal value for lineNumber" from tokenizeIfCheap when modifying documents with editor.edit()

[https://github.com/microsoft/vscode/issues/297386](https://github.com/microsoft/vscode/issues/297386)

## Summary

Fixed on main: the stale cursor position guard was added in commit e31de7959d3, preventing tokenizeIfCheap from receiving an invalid lineNumber when a non-user edit shortens the document. The fix shipped after the reported version (1.109.5).

## Change summary

tokenizeIfCheap throws 'Illegal value for lineNumber' when an extension replaces document content via editor.edit(), because the rename symbol tracker passes a stale cursor position to the tokenizer after the document shrinks.

## Best solution

Keep the shipped fix (commit e31de7959d3) which guards `updateCursorPosition` in ModelSymbolRenameTracker against stale lineNumbers exceeding the model's line count. No further change needed.
