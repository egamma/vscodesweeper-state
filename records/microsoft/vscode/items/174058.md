# Issue #174058 — Rendered whitespace characters non-uniform

[https://github.com/microsoft/vscode/issues/174058](https://github.com/microsoft/vscode/issues/174058)

## Summary

Keep open: the report has multiple confirmations, is scoped to VS Code's SVG whitespace renderer at negative zoom on Windows, and I did not find a concrete fixing commit or confirmed canonical duplicate. The current code still defaults `editor.experimentalWhitespaceRendering` to `svg`, so the reporter's workaround does not prove the bug is resolved.

## Change summary

Rendered whitespace dots become visually non-uniform when VS Code is zoomed out on Windows with SVG whitespace rendering enabled.

## Best solution

Investigate `src/vs/editor/browser/viewParts/whitespace/whitespace.ts` under negative `window.zoomLevel` and Windows display scaling, then either make the SVG circle positions/radii render uniformly at fractional zoom or fall back to the font/stable whitespace renderer for affected cases.

## Evidence

- **confirmed regression scope:** Reporter and commenters tied the issue to negative zoom levels after upgrading to 1.75, with Windows reports and `--disable-extensions --disable-gpu` not resolving it.
- **workaround, not resolution:** Comments confirm `"editor.experimentalWhitespaceRendering": "off"` or `"font"` fixes the visual issue, while `"svg"` and the default `"on"`/SVG path reproduce it.
- **current implementation:** `src/vs/editor/common/config/editorOptions.ts` still defaults `experimentalWhitespaceRendering` to `svg`, and `src/vs/editor/browser/viewParts/whitespace/whitespace.ts` still renders spaces as SVG `<circle>` elements using fractional `cx` and radius values.
- **history:** Commit 2c8d5bdc55861f28e2fbbec599ff1eee54f0f2f7 added SVG whitespace rendering; commit 210ea7f0cccc9ed79267d5562dfef482cc31a21b changed the default to SVG before the 1.75-era regression report.
