## Change summary

`editor.semanticTokenColorCustomizations.enabled: false` still allows semantic token customization rules such as `namespace` colors to be applied.

## Best solution

Have the semantic tokens/editor theming owner verify whether the `enabled` flag is still ignored on current main and, if so, fix the semantic token color customization resolution path so rules are skipped when the setting is false.
