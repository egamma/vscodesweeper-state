# Issue #204043 — Semantic colorization customization applied even if `editor.semanticTokenColorCustomizations.enabled` is false.

[https://github.com/microsoft/vscode/issues/204043](https://github.com/microsoft/vscode/issues/204043)

## Summary

Still reproduces on main: `setCustomSemanticTokenColors` unconditionally reads custom rules even when `enabled: false`, so if semantic highlighting is forced on by another path (e.g. extension language config or `editor.semanticHighlighting.enabled: true`), the custom color rules still apply. Team-authored issue — keeping open.

## Change summary

Custom semantic token color rules in `editor.semanticTokenColorCustomizations.rules` are applied even when `enabled` is explicitly set to `false` in the same setting block.

## Best solution

In `colorThemeData.ts#setCustomSemanticTokenColors`, gate the `readSemanticTokenRules` calls on `semanticTokenColors.enabled !== false` so that setting `enabled: false` suppresses both semantic highlighting and the custom rules.
