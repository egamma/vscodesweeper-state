# Issue #185173 — Unable to launch sandboxed process from when application is started from a UNC path

[https://github.com/microsoft/vscode/issues/185173](https://github.com/microsoft/vscode/issues/185173)

## Summary

Team-authored upstream tracking issue for a Chromium sandbox limitation when VS Code is launched from a UNC path. The issue remains valid as the upstream constraint has not been resolved, and it serves as the canonical tracker (issue #190669 was closed as a duplicate of this one). Hard keep-open: author is MEMBER.

## Change summary

VS Code cannot launch sandboxed processes (GPU, utility) when the application is started from a UNC path on Windows.

## Best solution

Continue tracking upstream Chromium sandbox support for UNC paths. If Chromium resolves the limitation, remove the workaround flags. Until then, this issue documents the constraint and the `--no-sandbox --disable-gpu-sandbox` workaround for affected users.
