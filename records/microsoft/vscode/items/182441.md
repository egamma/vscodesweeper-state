# Issue #182441 — Regression: TextEditorEdit.replace does not select empty selection anymore

[https://github.com/microsoft/vscode/issues/182441](https://github.com/microsoft/vscode/issues/182441)

## Summary

Still reproduces on main: `TextEditorEdit.replace` on an empty selection no longer selects the inserted text because `forceMoveMarkers: false` combined with no explicit `endCursorState` causes marker-based cursor recovery that doesn't produce a spanning selection.

## Change summary

Calling `editor.edit(e => e.replace(emptySelection, text))` no longer results in the inserted text being selected; the cursor collapses to a point instead.

## Best solution

The fix likely belongs in `src/vs/workbench/api/browser/mainThreadEditor.ts` — `applyEdits` should compute and pass an `endCursorState` that selects the replaced range when the original selection was empty, matching the behavior when a non-empty selection is replaced. Alternatively, the ext host side (`extHostTextEditor.ts`) could track edits applied to empty ranges and issue a follow-up `setSelections` call.
