# Issue #128912 — [folding] Allow to fold all method bodies

[https://github.com/microsoft/vscode/issues/128912](https://github.com/microsoft/vscode/issues/128912)

## Summary

This feature request for a new FoldingRangeKind (e.g. 'Body') and a 'Fold All Bodies' command remains unimplemented on current main. The FoldingRangeKind enum still only has Comment, Imports, and Region. Already triaged to Backlog with appropriate labels.

## Change summary

Request to add a 'body' FoldingRangeKind and a 'Fold All Method Bodies' command, allowing users to collapse all function/method bodies at once.

## Best solution

Add a new `FoldingRangeKind.Body` value in `src/vs/editor/common/languages.ts`, expose it via the extension API in `vscode.d.ts`, add a corresponding 'Fold All Method Bodies' editor action in `src/vs/editor/contrib/folding/browser/folding.ts`, and have language servers (e.g. TypeScript) report body ranges. This is a small API addition but requires language server cooperation to be useful.
