# Issue #256139 — major usability problems with your boarder-less windows

[https://github.com/microsoft/vscode/issues/256139](https://github.com/microsoft/vscode/issues/256139)

## Summary

This issue reports that VS Code's official Linux builds don't enable Electron's WaylandWindowDecorations feature flag at compile time, preventing server-side decorations (shadows, borders) under Wayland compositors like KDE/KWin. The root cause is in Electron's Wayland/Chromium layer, though VS Code's build configuration is the gating factor. A team member (bpasero) noted recent window-border work targets frameless windows only, not native titlebar mode. The issue remains valid and unresolved on current main — no compile-time or runtime change addresses Wayland SSD support in the official build.

## Change summary

Enable Wayland server-side decorations in VS Code's official Linux builds so that native window borders, shadows, and resize affordances appear under Wayland compositors.

## Best solution

Coordinate with Electron upstream (deepak1556 was cc'd) to determine whether enabling WaylandWindowDecorations in the official build is viable — the compile-time flag lives in Electron's build config, not VS Code's source. If feasible, enable it in the Linux build pipeline. The related issue https://github.com/microsoft/vscode/issues/225073 covers GNOME-side decoration loss and may benefit from the same fix. This is primarily an upstream/build-config decision rather than a VS Code source change.
