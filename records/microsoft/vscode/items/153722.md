# Issue #153722 — Provide speech feedback when commenting or uncommenting a line.

[https://github.com/microsoft/vscode/issues/153722](https://github.com/microsoft/vscode/issues/153722)

## Summary

Still unimplemented: no accessibility announcement exists in the comment/uncomment action on current main. A team member acknowledged it as sensible; keeping open as a valid accessibility feature request.

## Change summary

Add screen reader speech feedback when toggling line comments so users know the result without re-reading the line.

## Best solution

Add an ARIA announcement (via the accessibility signal or `ariaLiveMessage` pattern) in the toggle-line-comment action in `src/vs/editor/contrib/comment/browser/comment.ts` that reads the updated line content after commenting/uncommenting. This is a small, localized change but needs a product decision on the exact wording.
