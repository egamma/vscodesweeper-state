# Issue #153722 — Provide speech feedback when commenting or uncommenting a line.

[https://github.com/microsoft/vscode/issues/153722](https://github.com/microsoft/vscode/issues/153722)

## Summary

Still valid: no speech/aria feedback exists in the comment/uncomment actions on current main. The feature request is labeled under-discussion and lives in the Backlog milestone; keeping open for maintainer judgment on implementation approach.

## Change summary

Add screen reader speech feedback when toggling line comments via Ctrl+/ so users know the result without manually re-reading the line.

## Best solution

Add an ARIA announcement (via the accessibility signal or status service) in the comment toggle action in `src/vs/editor/contrib/comment/browser/comment.ts` that speaks the updated line content after commenting/uncommenting, gated on screen-reader-optimized mode.
