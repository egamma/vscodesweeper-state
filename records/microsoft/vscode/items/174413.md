# Issue #174413 — When the window is too small IHoverService doesn't take into account width of item

[https://github.com/microsoft/vscode/issues/174413](https://github.com/microsoft/vscode/issues/174413)

## Summary

Team-authored issue reporting that IHoverService doesn't account for hover width when the window is narrow, causing it to overlap the target instead of repositioning above/below. The underlying layout logic in contextview.ts still uses the same approach — no fix found on main.

## Change summary

When the window is too small, IHoverService renders the hover on top of the target element instead of repositioning it above or below.

## Best solution

Enhance the hover positioning logic (likely in `src/vs/base/browser/ui/contextview/contextview.ts` and the hover widget) to detect when the hover's width would cause overlap with the target anchor, and reposition vertically (above/below) rather than horizontally overlapping. The pointer direction challenge noted in the issue needs a design decision.
