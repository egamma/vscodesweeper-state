# Issue #88617 — onDebugInitialConfigurations activation event causes extensions to load when pressing F5 to run other projects

[https://github.com/microsoft/vscode/issues/88617](https://github.com/microsoft/vscode/issues/88617)

## Summary

Still reproducible on main: pressing F5 without a launch.json and without a language-matched editor causes `activateDebuggers('onDebugInitialConfigurations')` to fire, activating all extensions registered for that event — including unrelated language extensions that do expensive work at activation. No fix or mitigation has landed.

## Change summary

Pressing F5 to debug one language activates all debugger extensions registered for `onDebugInitialConfigurations`, causing unrelated extensions to perform expensive work (language server startup, SDK downloads).

## Best solution

The debug adapter manager in `src/vs/workbench/contrib/debug/browser/debugAdapterManager.ts` (line ~369) unconditionally activates all `onDebugInitialConfigurations` extensions when no language-matched debugger is found. A targeted fix could defer or scope this activation — e.g., only activate extensions whose debug type matches available project context, or support the activation-reason proposal (https://github.com/microsoft/vscode/issues/44711) so extensions can skip heavy work when activated for configuration discovery. The related issue https://github.com/microsoft/vscode/issues/175202 tracks a similar symptom and could be addressed together.
