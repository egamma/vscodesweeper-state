# Issue #126205 — cant select text with duble click

[https://github.com/microsoft/vscode/issues/126205](https://github.com/microsoft/vscode/issues/126205)

## Summary

Keep open as an editor RTL bug: the thread moved from a suspected GitLens problem to a maintainer-provided minimal reproduction using decoration `after` content on Persian text. I did not find a concrete fixing commit, so there is not enough evidence to close as completed or not reproducible.

## Change summary

Double-click word selection fails for Persian/RTL text when an editor decoration renders `after` content at the word boundary.

## Best solution

The editor team should either fix the mouse hit testing/word-selection interaction for RTL text with `afterContentClassName` decorations, or document a safe decoration pattern if extension-rendered `after` content cannot be supported there.
