# Issue #126205 — cant select text with duble click

[https://github.com/microsoft/vscode/issues/126205](https://github.com/microsoft/vscode/issues/126205)

## Summary

Keep open: the discussion narrowed this from a specific GitLens report to a reproducible core editor issue involving RTL/Persian word selection when an extension contributes an `after.contentText` decoration. There is no concrete fixing commit or canonical duplicate in the provided evidence, so a close proposal would be premature.

## Change summary

Double-click word selection can fail on Persian/RTL text after an extension adds an empty-range `after` decoration.

## Best solution

Fix the editor's RTL hit testing or word-selection mapping so injected decoration attachments do not interfere with double-click selection on the underlying Persian word. This belongs in the editor RTL/decoration rendering path rather than in GitLens specifically.
