# Issue #185061 — Expired auth notification is not dismissed after (re)sign-in

## Summary

Team-authored issue requesting that VS Code's authentication infrastructure natively handle the 'session expired' → 're-signed in' lifecycle so the stale notification is dismissed. Still open — no implementation found on main; the discussion concluded VS Code core (not extensions) should own this UX.

## Change summary

Expired-auth notification remains visible after the user successfully re-authenticates, because there is no mechanism to dismiss it.

## Best solution

Design a first-class authentication state-change UX in the authentication service layer (src/vs/workbench/services/authentication/) that replaces ad-hoc extension notifications with a VS Code–managed UI that automatically reflects session recovery — as jrieken suggested.
