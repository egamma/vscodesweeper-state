# Issue #94081 — Confusion about how EnvironmentVariableCollection handles multiple extensions changing the same variable 

[https://github.com/microsoft/vscode/issues/94081](https://github.com/microsoft/vscode/issues/94081)

## Summary

Still reproducible on main: the MergedEnvironmentVariableCollection constructor's ordering logic means a Replace from a persisted collection can block a subsequent Append from another extension, depending on Map iteration order. The team acknowledged this in the 2020 API call but it remains unfixed.

## Change summary

When multiple extensions mutate the same environment variable via EnvironmentVariableCollection, a persisted Replace from one extension can block an Append from another depending on collection iteration order, producing unexpected results.

## Best solution

Clarify and fix the merge priority in `src/vs/platform/terminal/common/environmentVariableCollection.ts`: Replace should set a base value, then Prepend/Append from other extensions should still apply on top of it regardless of iteration order. The constructor's skip logic at line 47-49 needs refinement to distinguish 'this extension's Replace wins over later Replaces' from 'Replace blocks all other mutators'.
