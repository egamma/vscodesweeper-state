# Issue #226092 — [Accessibility] Support audio cue for commented range

[https://github.com/microsoft/vscode/issues/226092](https://github.com/microsoft/vscode/issues/226092)

## Summary

Valid accessibility feature request still under discussion with a team member (meganrogge). No implementation exists on main — the accessibility signal registry has no comment-range signal. Keeping open as under-discussion.

## Change summary

Add an accessibility audio cue that plays when the cursor enters a commented range in the editor, so screen reader users can discover comments without purposeful navigation.

## Best solution

Implement a positional AccessibilitySignal (like errorAtPosition/warningAtPosition) that fires when the cursor enters a commenting range decoration. The signal registration belongs in `src/vs/platform/accessibilitySignal/browser/accessibilitySignalService.ts`, with the positional detection wired in the comments editor contribution (`src/vs/workbench/contrib/comments/browser/commentsEditorContribution.ts`). Should be disabled by default per the reporter's suggestion.
