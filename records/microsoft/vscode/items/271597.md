# Issue #271597 — Tooltip overlaps scrollbar/gutter when sidebars are hidden

[https://github.com/microsoft/vscode/issues/271597](https://github.com/microsoft/vscode/issues/271597)

## Summary

Still reproduces on main: the hover max-width calculation uses body width minus a fixed 14px padding, which doesn't account for the scrollbar/minimap when sidebars are hidden.

## Change summary

Editor hover tooltip overlaps the scrollbar/gutter on the right when all sidebars and the activity bar are hidden.

## Best solution

The `_findMaximumRenderingWidth()` method in `contentHoverWidget.ts` (line ~230) should constrain the hover width to the editor's layout width (or account for the scrollbar/minimap width) rather than using the raw body width minus a hardcoded 14px.
