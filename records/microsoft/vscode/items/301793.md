# Issue #301793 — Copilot Chat history is completely lost after saving an "Untitled Workspace" as a .code-workspace file

[https://github.com/microsoft/vscode/issues/301793](https://github.com/microsoft/vscode/issues/301793)

## Summary

Confirmed bug: the session index is lost during workspace transitions because `migrateSessionsToNewWorkspace` clears `indexCache` and then reads from the already-switched (empty) storage scope. An open community PR (#305109) addresses this with an `onWillSaveState` cache-preservation approach. Keeping open for the PR to land.

## Change summary

Saving an untitled workspace as a .code-workspace file loses all Copilot Chat history because the session index is rebuilt from the new empty storage scope instead of being preserved from the old one.

## Best solution

Land (or rework) https://github.com/microsoft/vscode/pull/305109, which preserves the index cache before the storage switch via `onWillSaveState`, then merges old entries into the new scope's index during migration. The fix is localized to `src/vs/workbench/contrib/chat/common/model/chatSessionStore.ts`.

## Evidence

- **source trace:** In `src/vs/workbench/contrib/chat/common/model/chatSessionStore.ts`, lines 172–175: `migrateSessionsToNewWorkspace` sets `this.indexCache = undefined` then calls `flushIndex()` → `internalGetIndex()` (line 541), which reads from `this.storageService.get(ChatIndexStorageKey, ...)`. By this point the storage service has already switched to the new workspace scope (empty DB), so it returns nothing and creates an empty index — all session metadata is lost.
- **prior fix (incomplete):** Commit 7393b0ec607 ('Fix chat session loss on workspace ID change', Feb 2026, PR https://github.com/microsoft/vscode/pull/289779) added the migration logic but introduced the index-clearing race that this issue reports. The fix shipped in 1.110 but the index bug persists on current main.
- **open PR:** https://github.com/microsoft/vscode/pull/305109 by w1nsid proposes preserving the index cache via `onWillSaveState` before the storage switch and merging old entries into the new scope. Includes a regression test.
- **community confirmation:** Two independent users confirmed the issue (comments by w1nsid and CXW-Kevin-Buchs), with a detailed manual recovery workaround documented.
