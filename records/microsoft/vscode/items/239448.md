# Issue #239448 — vscode 1.94 1.95 1.96 (for me at least) will do a "save as" to already existing file without issuing any warning first.

[https://github.com/microsoft/vscode/issues/239448](https://github.com/microsoft/vscode/issues/239448)

## Summary

This is an upstream issue: the native save dialog's overwrite confirmation is provided by the OS file dialog (GTK / xdg-desktop-portal on Fedora Linux), not by VS Code. VS Code delegates to Electron's `dialog.showSaveDialog()` which invokes the platform native dialog. The simple file dialog (VS Code's built-in) does have its own overwrite check, but the native path does not. The issue needs confirmation and likely belongs upstream (Electron or the GTK portal backend).

## Change summary

Native 'Save As' dialog on Fedora Linux silently overwrites existing files without an overwrite confirmation prompt.

## Best solution

Confirm whether this reproduces with the xdg-desktop-portal file chooser vs the legacy GTK dialog, and whether it is an Electron or portal regression. If it is a portal/GTK bug, file upstream and track here. Alternatively, VS Code could add its own overwrite confirmation after the native dialog returns a path that already exists (in `src/vs/workbench/services/dialogs/electron-browser/fileDialogService.ts`), similar to how the simple file dialog already does in `simpleFileDialog.ts`.
