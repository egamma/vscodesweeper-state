# Issue #266597 — editor.removeManualFoldingRanges should support empty selection

[https://github.com/microsoft/vscode/issues/266597](https://github.com/microsoft/vscode/issues/266597)

## Summary

Still open on main. The title was repurposed by @aeschli to track making `editor.removeManualFoldingRanges` useful with an empty selection (removing all manual ranges). An open community PR (#304793) exists but has not been merged.

## Change summary

When the cursor has no selection, `editor.removeManualFoldingRanges` should remove all manual folding ranges in the document so users can recover from broken/stale manual folds.

## Best solution

Merge or iterate on PR #304793 (or an equivalent fix in `src/vs/editor/contrib/folding/browser/folding.ts`) that detects an empty selection and falls back to removing all manual ranges document-wide.
