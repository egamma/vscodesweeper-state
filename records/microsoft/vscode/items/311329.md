# Issue #311329 — Custom subagent resolves to model outside its configured model list

[https://github.com/microsoft/vscode/issues/311329](https://github.com/microsoft/vscode/issues/311329)

## Summary

Legitimate bug: when the extension-layer endpoint resolution fails for a subagent's configured model, the fallback in executionSubagentToolCallingLoop.ts silently falls back to the main agent endpoint (which may resolve to a model outside the subagent's configured list). The core resolveSubagentModel correctly picks from the list, but the extension's network layer overrides that choice.

## Change summary

Custom subagent configured with an explicit model list sometimes resolves to claude-sonnet-4, a model not present in its configured allowlist, when the preferred model is transiently unavailable.

## Best solution

The fallback path in `extensions/copilot/src/extension/prompt/node/executionSubagentToolCallingLoop.ts` (lines 130-146) should iterate the subagent's remaining model list rather than falling back to the main agent endpoint. If no model in the list is available, it should surface an error rather than silently picking an unconfigured model.
