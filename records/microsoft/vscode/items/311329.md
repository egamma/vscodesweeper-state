# Issue #311329 — Custom subagent resolves to model outside its configured model list

[https://github.com/microsoft/vscode/issues/311329](https://github.com/microsoft/vscode/issues/311329)

## Summary

Legitimate bug report with detailed logs showing subagent model resolution ignoring the configured model list and falling back to claude-sonnet-4. The VS Code core resolution logic in runSubagentTool.ts appears correct, so the override likely happens in the Copilot extension or backend. Needs investigation by the chat-agent/copilot team.

## Change summary

Custom subagent configured with an explicit model list (Opus, GPT-5.4, GPT-5.3-Codex) sometimes resolves to claude-sonnet-4 which is not in the list, when the parent agent uses a higher cost-tier model.

## Best solution

Investigate whether the Copilot extension or backend overrides the `userSelectedModelId` passed by `runSubagentTool.ts`. The VS Code core model resolution at `src/vs/workbench/contrib/chat/common/tools/builtinTools/runSubagentTool.ts:531-551` correctly iterates the configured list, so the fallback to Sonnet likely occurs server-side or in the extension's request pipeline. Ensure the resolved model from the subagent's list is honored end-to-end.
