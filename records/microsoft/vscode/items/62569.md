# Issue #62569 — [scss] rename not working for placeholder selectors

[https://github.com/microsoft/vscode/issues/62569](https://github.com/microsoft/vscode/issues/62569)

## Summary

The rename logic lives in the upstream `vscode-css-languageservice` package. The `doRename` method in `cssNavigation.js` uses `findDocumentHighlights` to locate references and replaces ranges with the raw `newName`, without preserving the `%` prefix for SCSS placeholder selectors. This is still unaddressed on current main — the label and milestone are appropriate.

## Change summary

Renaming an SCSS placeholder selector (e.g. `%foo`) strips the leading `%` sign from all occurrences.

## Best solution

Fix in the upstream `vscode-css-languageservice` (`doRename` or `findDocumentHighlights` for SCSS) so that the `%` prefix is either excluded from the highlight range or preserved during the rename replacement. The fix belongs in https://github.com/microsoft/vscode-css-languageservice, not in the VS Code repo itself.
