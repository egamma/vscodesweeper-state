# Issue #62569 — [scss] rename not working for placeholder selectors

[https://github.com/microsoft/vscode/issues/62569](https://github.com/microsoft/vscode/issues/62569)

## Summary

The rename logic for SCSS placeholder selectors is handled entirely by the upstream `vscode-css-languageservice` package's `doRename` method. The issue remains valid but needs a fix in that library. Keeping open as a tracked feature request.

## Change summary

Renaming SCSS placeholder selectors (e.g. `%foo`) strips the leading `%` sign, breaking the selector semantics.

## Best solution

Fix `doRename` in [vscode-css-languageservice](https://github.com/microsoft/vscode-css-languageservice) to include the `%` prefix in the rename range for placeholder selectors. Once fixed upstream, updating the dependency in `extensions/css-language-features/server/package.json` would ship the fix.
