# Issue #312791 — [Error] unhandlederror-Throttler is disposed

[https://github.com/microsoft/vscode/issues/312791](https://github.com/microsoft/vscode/issues/312791)

## Summary

Still reproduces on current main: the async `onDidChangeBodyVisibility` handler in `SCMHistoryViewPane` has no disposal guard after its `await` points, so leaked observable subscriptions call `refresh()` on an already-disposed throttler. Thousands of stable users are affected across multiple releases (1.118–1.129).

## Change summary

The SCM history view pane throws 'Throttler is disposed' when observable callbacks fire after the view pane has been disposed, due to an unguarded async visibility handler.

## Best solution

Add a disposal check after each `await` in the `onDidChangeBodyVisibility` handler (lines 1744, 1747–1751) to bail out if `_visibilityDisposables` is already disposed, preventing leaked subscriptions from calling `refresh()` on the disposed throttler. Alternatively, guard `refresh()` itself to silently return when the throttler is disposed.
