# Issue #290962 — Terminal tool command is very horizontally cramped

[https://github.com/microsoft/vscode/issues/290962](https://github.com/microsoft/vscode/issues/290962)

## Summary

Team-authored UX polish issue about the terminal tool command header consuming too much horizontal space in sidebar chat. Several CSS refinements have landed since filing (intention labels, spinner alignment, border radius fixes), but none directly address the core complaint — the toolbar and margins still consume significant width. Keeping open per team-member rule and because the underlying layout concern remains.

## Change summary

The terminal tool command row in sidebar chat wastes too much horizontal space on indentation, decoration dot, toolbar, margins, and flex gap, leaving the actual command text cramped and wrapping.

## Best solution

Rework the `.chat-terminal-content-title` layout so the action-bar toolbar overlays on hover (or adopts the code-block toolbar pattern that doesn't reserve width), and tighten the 12 px gap and right padding. The relevant CSS is in `src/vs/workbench/contrib/chat/browser/widget/chatContentParts/media/chatTerminalToolProgressPart.css` and the TS part in `chatTerminalToolProgressPart.ts`. A design pass deciding the target density would unblock implementation.
