# Issue #149688 — The detail of QuickPick item is truncated by QuickPick separator

[https://github.com/microsoft/vscode/issues/149688](https://github.com/microsoft/vscode/issues/149688)

## Summary

Still reproduces on main: separators without buttons are rendered inline within the item's flex entry, so a long separator label consumes horizontal space and truncates the item's detail text. Filed by a team member — keep open.

## Change summary

A long separator label in QuickPick truncates the detail text of the item below it, even though they are on visually separate rows.

## Best solution

In `src/vs/platform/quickinput/browser/quickInputList.ts`, the inline separator element (`.quick-input-list-separator`) is a flex sibling of `.quick-input-list-label` in the same entry. The separator should not reduce the available width for the label/detail rows — either give separators without buttons their own tree row (like those with buttons), or change the CSS so the inline separator is positioned absolutely or doesn't consume flex space from the rows container.
