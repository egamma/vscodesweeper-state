# Issue #299850 — Allow workspace extensions to use proposed API

[https://github.com/microsoft/vscode/issues/299850](https://github.com/microsoft/vscode/issues/299850)

## Summary

Team-authored feature request to allow workspace extensions (.vscode/extensions/) to use proposed APIs in built VS Code. Kept open per policy — authored by a team member with a clear proposed fix.

## Change summary

Workspace extensions in .vscode/extensions/ have their enabledApiProposals stripped at runtime in built VS Code because they lack an isWorkspaceScoped flag on IExtensionDescription.

## Best solution

Propagate isWorkspaceScoped from the management layer (ILocalExtension) to the runtime layer (IExtensionDescription), then exempt workspace-scoped extensions from the proposed-API stripping in ExtensionsProposedApi.doUpdateEnabledApiProposals. The three files identified in the issue body are the right touch points.
