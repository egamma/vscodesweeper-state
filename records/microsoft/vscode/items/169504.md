# Issue #169504 — VSCode is sluggish and buggy on Fedora 37 with wayland + Nvidia drivers

[https://github.com/microsoft/vscode/issues/169504](https://github.com/microsoft/vscode/issues/169504)

## Summary

Genuine upstream Chromium/Electron issue affecting Wayland + Nvidia GPU users, already correctly triaged by a team member (deepak1556) and labeled `upstream`/`chromium`/`wayland`. With 32 upvotes and ongoing community reports through 2025, this remains a real pain point. The root cause lives in Chromium's Ozone/Wayland backend interacting with Nvidia's DRI/EGL stack — VS Code cannot fix it in its own code.

## Change summary

VS Code is sluggish, flickers, and has unresponsive UI elements when running on Wayland with Nvidia proprietary drivers on Linux.

## Best solution

Continue tracking upstream progress in the Chromium Ozone/Wayland + Nvidia pipeline (https://bugs.chromium.org/p/chromium/issues/detail?id=1378258). As Electron upgrades ship newer Chromium versions with improved Wayland/Nvidia support, the situation should improve. The workaround of falling back to XWayland (`--ozone-platform=X11`) remains valid. No actionable change exists in the VS Code codebase.
