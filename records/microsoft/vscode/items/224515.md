# Issue #224515 — Context menu gets too small depending on the cursor position

[https://github.com/microsoft/vscode/issues/224515](https://github.com/microsoft/vscode/issues/224515)

## Summary

The core user problem — oversized native context menus from Chromium's Refresh 2023 design, with no way to opt out without switching to a custom titlebar — has been resolved by the new `window.menuStyle` setting (commit 4e68b628ea4, shipped in 1.101.0). Users can now set `window.menuStyle` to `custom` independently of `window.titleBarStyle`, which was the most-requested workaround throughout this thread.

## Change summary

Native context menus became oversized after Electron 30 adopted Chromium's Refresh 2023 menu styling, and the only workaround required switching the entire titlebar to custom.

## Best solution

Keep the shipped `window.menuStyle` setting (1.101.0). The underlying Chromium menu sizing is upstream and was closed as not-planned by Electron; VS Code's mitigation — decoupling menu style from titlebar style — gives users a clean opt-out. No further change needed.
