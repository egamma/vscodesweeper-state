# Issue #162414 — Cannot paste according to universal cllipboard between Macos and iPados

[https://github.com/microsoft/vscode/issues/162414](https://github.com/microsoft/vscode/issues/162414)

## Summary

Still reproduces on main. The root cause is in how Electron/Chromium exposes macOS Universal Clipboard RTFD data — it arrives with files but no text/plain, triggering VS Code's filename-paste fallback. This is upstream (Electron/Chromium) but VS Code could add a workaround.

## Change summary

Pasting via Universal Clipboard from iPad/iPhone Safari into the VS Code editor inserts an RTFD filename instead of the copied text.

## Best solution

Either file an Electron/Chromium bug to ensure RTFD clipboard payloads expose text/plain, or add a VS Code workaround in `src/vs/editor/browser/controller/editContext/clipboardUtils.ts` to detect RTFD files in the clipboard and attempt to extract their text content before falling back to pasting filenames. The existing `copyPasteController.ts` drop/paste provider system could also handle this via a default provider that reads RTFD content.

## Evidence

- **no fix on main:** git log shows no commits addressing Universal Clipboard RTFD handling in this file or the editor paste path.
- **Electron limitation:** macOS Universal Clipboard from iOS Safari sends rich text as RTFD, and Electron/Chromium does not expose the plain-text representation in clipboardData.getData('text/plain'), only as a file entry.
