# Issue #276196 — # GitHub Copilot Chat Cannot Reference MCP Servers with Special Characters Using # Notation

[https://github.com/microsoft/vscode/issues/276196](https://github.com/microsoft/vscode/issues/276196)

## Summary

Valid bug from a team member — the `variableReg` regex in the chat request parser only allows `[\w_\-]+` after `#`, rejecting names with spaces, `@`, or `/`. The dynamic-variable fallback path may also have issues registering references with special characters from the picker. Keeping open per team-authored policy.

## Change summary

MCP servers with special characters (spaces, @, /) in their names cannot be referenced via `#` notation in Copilot Chat because the parser regex rejects them.

## Best solution

Either expand `variableReg` in `src/vs/workbench/contrib/chat/common/requestParser/chatRequestParser.ts` to handle special characters (possibly with quoting/escaping), or ensure that picker-selected MCP servers are always registered as dynamic variables whose position-based matching bypasses the regex. Alternatively, validate MCP server names at registration time to disallow characters that break parsing.
