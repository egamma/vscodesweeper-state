# Issue #226132 — Flicker'ish test tree

[https://github.com/microsoft/vscode/issues/226132](https://github.com/microsoft/vscode/issues/226132)

## Summary

Still open on main — the test tree spinner state transitions remain un-debounced, so the flicker between individual test runs persists. A maintainer suggested debouncing in the discussion but no fix was implemented.

## Change summary

Test tree progress spinners flicker rapidly when running multiple tests because the running state toggles off between each individual test execution.

## Best solution

Add a short debounce (lean toward showing 'running' longer) in the computed-state propagation path in `treeProjection.ts` or the tree element rendering, so that transient gaps between test executions don't toggle the spinner off and back on.
