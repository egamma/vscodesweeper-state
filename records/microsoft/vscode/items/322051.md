# Issue #322051 — No credits spent shown when a request is stopped in Auto mode

[https://github.com/microsoft/vscode/issues/322051](https://github.com/microsoft/vscode/issues/322051)

## Summary

Fixed on main: commit 0f0b5838655 changes the cancellation handler to resolve with the current turn (including accumulated usage) instead of `undefined`, so the response footer now shows the model and credits consumed before the interruption.

## Change summary

When a chat request is stopped in Auto mode, the response footer does not show which model was used or how many credits were spent.

## Best solution

Keep the shipped fix in commit 0f0b58386555a98f4c0d0f08406acf6f10e3c5e5; no further change needed. The fix preserves the turn's accumulated `usage` on cancellation so the footer renders model and credit details.
