# Issue #140698 — [html] Improve `html.autoCreateQuotes` to handle manually typed quotes

[https://github.com/microsoft/vscode/issues/140698](https://github.com/microsoft/vscode/issues/140698)

## Summary

Team-authored feature request to improve html.autoCreateQuotes so it swallows manually typed quotes instead of doubling them. Still open and valid — the underlying blocker (snippet undoStop behavior, #146171) appears unresolved, and no fix has landed.

## Change summary

When html.autoCreateQuotes inserts quotes automatically but the user also types a quote character, the result is doubled quotes with incorrect cursor placement.

## Best solution

Resolve the undoStop blocker (https://github.com/microsoft/vscode/issues/146171) so that the auto-inserted snippet can set undoStopBefore:false, then add quote-swallowing logic in the HTML language feature's autoInsertion handler (extensions/html-language-features/client/src/autoInsertion.ts).
