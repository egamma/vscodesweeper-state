# Issue #189166 — Cannot launch new terminals in multi-root workspace on a remote machine unless 'terminal.integrated.cwd' is unset

[https://github.com/microsoft/vscode/issues/189166](https://github.com/microsoft/vscode/issues/189166)

## Summary

This is a specific manifestation of the same multi-root terminal cwd bug tracked in #142520, which Tyriar explicitly grouped together with this issue. The canonical issue is older, still open, and covers the same root cause — consolidating there.

## Change summary

Setting `terminal.integrated.cwd` in a multi-root remote workspace fails because the path is not properly validated as a remote-side path before being used to launch the terminal process.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/142520 which tracks the broader multi-root terminal cwd issue. The fix likely involves properly resolving `terminal.integrated.cwd` relative to the remote filesystem in `remoteTerminalBackend.ts` and ensuring `getWorkspaceForTerminal` handles the cross-machine path scenario.
