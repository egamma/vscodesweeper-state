# Issue #179952 — RunTask command lists duplicate tasks when a filter is applied

[https://github.com/microsoft/vscode/issues/179952](https://github.com/microsoft/vscode/issues/179952)

## Summary

Still reproducible on main: when an npm task in tasks.json customizes a detected task but changes the `script` property, the key-matching in `_getCustomTaskPromises` fails, causing both the detected task and the configured task to appear in the filtered quick pick.

## Change summary

RunTask command with a type filter shows duplicate entries when a tasks.json customization changes the script name of a detected npm task.

## Best solution

Fix the deduplication logic in `_getCustomTaskPromises` (around line 2438) to handle the case where a configuring task's `_key` doesn't match a contributed task because the user changed the `script` property — the configured task should still replace (not supplement) the contributed task it customizes.
