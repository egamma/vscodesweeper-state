# Issue #132784 — [folding] Newlines created by Insert Line Above command get sucked into previous code fold

[https://github.com/microsoft/vscode/issues/132784](https://github.com/microsoft/vscode/issues/132784)

## Summary

Partially fixed on main (commit 453a142c65b added line-change tracking in hiddenRangeModel), but the full fix still depends on the unresolved #136867 (setHiddenAreas optimization skipping valid updates). The bug likely still reproduces for offside-rule languages.

## Change summary

Insert Line Above command causes the new line to be absorbed into the previous collapsed fold region instead of appearing outside/below it.

## Best solution

Complete the fix by resolving the setHiddenAreas optimization issue (https://github.com/microsoft/vscode/issues/136867) so that valid hidden-area updates are never skipped. The remaining bug is in `src/vs/editor/common/viewModel/viewModelImpl.ts` where the `mergedRanges === this.previousHiddenAreas` reference-equality check can suppress a legitimate update after line insertions near fold boundaries.
