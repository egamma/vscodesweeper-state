# Issue #169871 — Git - Message Box on git commit errors is too short

[https://github.com/microsoft/vscode/issues/169871](https://github.com/microsoft/vscode/issues/169871)

## Summary

Still reproduces on main: the default error handler at line 5653 of commands.ts shows only a single line from stderr/stdout, truncating multi-line git hook output. An open PR (#244052) exists but has not merged.

## Change summary

Git commit error dialog only shows the first (or last) line of hook output, hiding the actual failure reason.

## Best solution

Land or iterate on the open PR https://github.com/microsoft/vscode/pull/244052, which proposes showing multi-line error output. The fix is localized to the `default` case in `extensions/git/src/commands.ts` around line 5646, where `hintLines[0]` (or `hintLines[hintLines.length - 1]`) should be replaced with a multi-line excerpt or scrollable output.
