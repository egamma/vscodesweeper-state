# Issue #197970 — `"colorizedBracketPairs"` only supports lowercase, but matching is case insensitive

[https://github.com/microsoft/vscode/issues/197970](https://github.com/microsoft/vscode/issues/197970)

## Summary

Still present on main: the bracket tokenizer matches case-insensitively (regex 'gi' flag + toLowerCase lookup), but colorizedBracketPairs configuration entries are stored and looked up with exact case in the CachedFunction, causing a mismatch when casing differs from the brackets config.

## Change summary

colorizedBracketPairs only works when entries are lowercase, despite bracket matching being case-insensitive.

## Best solution

Normalize colorizedBracketPairs entries to lowercase before looking them up in the CachedFunction in languageBracketsConfiguration.ts (lines 58-59), or make the CachedFunction key case-insensitive, so that colorizedBracketPairs entries match regardless of case — consistent with the tokenizer's case-insensitive behavior.
