# Issue #201431 — [folding] Manual folding range not useful after copy-paste

[https://github.com/microsoft/vscode/issues/201431](https://github.com/microsoft/vscode/issues/201431)

## Summary

Still present on main: when content changes via paste, the folding model preserves previously-created manual/collapsed ranges via `_currentFoldedOrManualRanges` in `foldingModel.ts`, causing stale folding ranges to persist even when the underlying content no longer matches.

## Change summary

After copy-pasting content over a region that had manual folding ranges, the old folding ranges persist incorrectly because the folding model re-applies them without validating against the new content structure.

## Best solution

When `foldingModel.update()` re-applies preserved manual/collapsed ranges after a content change, validate that each range still corresponds to a valid indentation-based boundary in the new content. Invalid ranges should be discarded rather than forcibly reapplied. The fix belongs in `src/vs/editor/contrib/folding/browser/foldingModel.ts` around the `_currentFoldedOrManualRanges` / `sanitizeAndMerge` logic.
