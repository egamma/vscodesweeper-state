# Issue #313054 — Git SCM view broken when opening a file without workspace

[https://github.com/microsoft/vscode/issues/313054](https://github.com/microsoft/vscode/issues/313054)

## Summary

Confirmed regression introduced in 1.110 by commit 9caffaaa2aa (PR #292804), which intentionally hides Git repositories when workspace.workspaceFolders is undefined. This breaks the legitimate scenario of opening a single file via the OS explorer and using Source Control. The code comment acknowledges the approach as 'temporary.'

## Change summary

Git SCM view is non-functional (no commit box, no diff, no changes) when opening a file without a workspace, because the repository is marked as hidden.

## Best solution

Refine the hiding logic in `extensions/git/src/repository.ts` (line 979) to distinguish between the background-agent-session scenario (which motivated the hiding) and the legitimate single-file-open scenario. For example, only hide when the empty window was opened programmatically for agent sessions, or allow the user to unhide the repository via the Repositories view.
