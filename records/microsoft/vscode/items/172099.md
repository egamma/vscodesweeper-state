# Issue #172099 — Variables `relativeFile` and `relativeFileDirname` use backslashes in WSL when used in keyboard shortcuts

[https://github.com/microsoft/vscode/issues/172099](https://github.com/microsoft/vscode/issues/172099)

## Summary

Confirmed bug still present on main: `relativeFile` and `relativeFileDirname` are resolved using the local (Windows) path separator via `paths.relative()` in the renderer process, producing backslashes even when the terminal targets a Linux/WSL remote. An open PR (#310771) is attempting a fix.

## Change summary

Variables `relativeFile` and `relativeFileDirname` produce backslash-separated paths in WSL terminal `sendSequence` because variable resolution uses the Windows-native path module in the renderer.

## Best solution

Review and land https://github.com/microsoft/vscode/pull/310771, which addresses this by fixing separator handling for remote contexts. The core issue is that `configurationResolverService.resolveAsync` in the browser uses `isWindows`-based path logic; the fix should ensure remote-aware path resolution (either resolving in the extension host as alexr00 suggested, or normalizing separators based on the remote OS).
