# Issue #172099 — Variables `relativeFile` and `relativeFileDirname` use backslashes in WSL when used in keyboard shortcuts

[https://github.com/microsoft/vscode/issues/172099](https://github.com/microsoft/vscode/issues/172099)

## Summary

Still reproduces on main: `variableResolver.ts` uses the platform `path.relative()` (which produces backslashes on Windows) for `relativeFile`/`relativeFileDirname` even when the workspace is WSL-remote. An open PR (#310771) is attempting to fix this.

## Change summary

Variables `relativeFile` and `relativeFileDirname` produce backslash-separated paths in WSL terminal sendSequence because the variable resolver uses the local OS path separator instead of the remote OS separator.

## Best solution

Land or iterate on the open PR https://github.com/microsoft/vscode/pull/310771. The fix should ensure variable resolution for terminal sendSequence uses posix path operations when the terminal is connected to a non-Windows remote (WSL/SSH), as suggested by @alexr00's comment about resolving variables in the extension host.
