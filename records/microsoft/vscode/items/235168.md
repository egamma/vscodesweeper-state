# Issue #235168 — Debugger hover doesn't work with parenthesized expressions

[https://github.com/microsoft/vscode/issues/235168](https://github.com/microsoft/vscode/issues/235168)

## Summary

Still reproduces on main: the fallback expression regex in `getExactExpressionStartAndEnd` explicitly excludes parentheses, so `(a).prop` is not recognized as a single evaluatable expression during debug hover. The spread operator variant was fixed in 6edba302a91 but parenthesized expressions remain unhandled.

## Change summary

Debug hover fails to evaluate expressions wrapped in parentheses like `(a).isupper()` because the fallback expression-extraction regex excludes `()` characters.

## Best solution

Extend the expression extraction in `getExactExpressionStartAndEnd` (src/vs/workbench/contrib/debug/common/debugUtils.ts, line 91) to handle parenthesized sub-expressions, similar to the spread-operator fix. Alternatively, built-in language extensions (TypeScript) could register an `EvaluatableExpressionProvider` that correctly handles parenthesized expressions. The related `!` operator issue (https://github.com/microsoft/vscode/issues/238279) should be addressed together.
