[https://github.com/microsoft/vscode/issues/314044](https://github.com/microsoft/vscode/issues/314044)

## Summary

Still under discussion: the plan review widget's default action is determined by the `default` flag on actions sent from copilot CLI (line 117 of chatPlanReviewPart.ts). A team member was pinged for guidance on whether CLI or VS Code should own the ordering, with no resolution yet.

## Change summary

The 'Implement with Autopilot' action should not be the default choice in the plan review widget; 'Implement Plan' should be the default to avoid burning tokens unnecessarily.

## Best solution

Resolve whether copilot CLI should change the `default` flag it sends, or whether VS Code should override the ordering client-side in `chatPlanReviewPart.ts` (around line 117). The discussion with @dereklegenzoff is still pending. This likely belongs in the `plan-review` or `chat` area, and may require a cross-team decision between VS Code and copilot CLI.
