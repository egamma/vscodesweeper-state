# Issue #111760 — [html] handle HTML-escaped characters in JavaScript attribute properties

[https://github.com/microsoft/vscode/issues/111760](https://github.com/microsoft/vscode/issues/111760)

## Summary

Still reproduces on main: the `updateContent` function in embeddedSupport.ts decodes `&quot;`/`&#34;` for CSS attribute values but not for JavaScript attribute values (e.g. onclick handlers), so the JS language service sees raw HTML entities as syntax errors.

## Change summary

Using `&quot;` in JavaScript HTML attributes (like onclick) causes false syntax errors because HTML entities aren't decoded before JS validation.

## Best solution

Extend the `&quot;`/`&#34;` replacement logic in `updateContent()` (embeddedSupport.ts) to also cover the JavaScript-in-attribute case (`c.attributeValue && c.languageId === 'javascript'`), mirroring the existing CSS fix from commit 4685d759641.
