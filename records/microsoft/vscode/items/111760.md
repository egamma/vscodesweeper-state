# Issue #111760 — [html] handle HTML-escaped characters in JavaScript attribute properties

[https://github.com/microsoft/vscode/issues/111760](https://github.com/microsoft/vscode/issues/111760)

## Summary

Still reproduces on main: the `updateContent` function in embeddedSupport.ts decodes `&quot;`/`&#34;` for CSS attribute values but not for JavaScript attribute values, so the JS language service sees raw HTML entities and flags them as errors.

## Change summary

HTML-escaped quote entities (`&quot;`) in JavaScript event-handler attributes (e.g. `onclick`) are incorrectly reported as syntax errors.

## Best solution

Extend the `updateContent` function in `extensions/html-language-features/server/src/modes/embeddedSupport.ts` to apply the same `&quot;`/`&#34;` → `"` substitution for JavaScript attribute values (when `c.attributeValue && c.languageId === 'javascript'`), mirroring the existing CSS handling.
