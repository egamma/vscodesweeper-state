# Issue #232970 — Null editorGhostText.foreground color in default high contrast themes

[https://github.com/microsoft/vscode/issues/232970](https://github.com/microsoft/vscode/issues/232970)

## Summary

The null foreground in HC themes is intentional: ghost text uses a visible border (`editorGhostText.border`) instead of semi-transparent foreground for better accessibility contrast. Extensions consuming `editorGhostText.foreground` in HC themes should fall back to the border color or provide their own HC-aware styling.

## Change summary

The `editorGhostText.foreground` theme color resolves to null in High Contrast Dark/Light themes, making ghost text invisible for extensions relying solely on that color.

## Best solution

This is as-designed: in HC themes the ghost text border provides the visual distinction instead of a semi-transparent foreground. Extensions should use `editorGhostText.border` in HC contexts or define their own HC-aware fallback. If the team disagrees and wants a non-null HC foreground, it would be a one-line change in `src/vs/editor/common/core/editorColorRegistry.ts`.
