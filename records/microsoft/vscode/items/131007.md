# Issue #131007 — SCM Input - Expand SCM input validation support

[https://github.com/microsoft/vscode/issues/131007](https://github.com/microsoft/vscode/issues/131007)

## Summary

This is an active API-finalization tracking issue for the proposed `scmValidation` API, which still exists as `vscode.proposed.scmValidation.d.ts` and is actively consumed by the built-in git extension. The API has not been finalized — keeping open as the finalization decision is pending.

## Change summary

Finalize the proposed SCM input validation API (`showValidationMessage`, `validateInput`, `SourceControlInputBoxValidationType`) so third-party extensions can use it without opting into proposed APIs.

## Best solution

A maintainer should revisit the API shape (jrieken noted in 2021 the current shape wasn't ready to finalize) and either redesign and finalize it, or explicitly decide to drop it. The proposed declaration lives in `src/vscode-dts/vscode.proposed.scmValidation.d.ts` and the implementation in `src/vs/workbench/api/common/extHostSCM.ts`.
