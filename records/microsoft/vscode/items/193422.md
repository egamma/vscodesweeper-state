# Issue #193422 — Avoid unnecessary repaint in minimap during typing

[https://github.com/microsoft/vscode/issues/193422](https://github.com/microsoft/vscode/issues/193422)

## Summary

Keep open: this is a team-authored issue from a MEMBER, which the cleanup policy says must not be proposed for closure. The current minimap code still clears the full decorations canvas, matching the reported performance concern.

## Change summary

Avoid repainting the full minimap decorations layer while typing when only a small region changed.

## Best solution

Optimize minimap decorations rendering in src/vs/editor/browser/viewParts/minimap/minimap.ts so renderDecorations updates only dirty decoration regions, similar to the text-layer delta rendering that already uses putImageData.
