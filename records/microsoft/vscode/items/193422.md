# Issue #193422 — Avoid unnecessary repaint in minimap during typing

[https://github.com/microsoft/vscode/issues/193422](https://github.com/microsoft/vscode/issues/193422)

## Summary

Still reproduces on main: every keystroke triggers `onSelectionChanged()` which sets `_renderDecorations = true`, causing a full `clearRect` + redraw of the entire decorations canvas. No delta/incremental approach has been implemented for decoration rendering since this was filed.

## Change summary

Minimap repaints its entire decorations canvas on every keystroke because selection changes trigger a full clear-and-redraw cycle.

## Best solution

Implement a delta rendering approach for the minimap decorations canvas — either track dirty rectangles from the previous frame and only repaint changed regions, or skip the decorations repaint entirely when only the cursor position changed without affecting visible decorations. The fix belongs in `src/vs/editor/browser/viewParts/minimap/minimap.ts` in `renderDecorations()` and/or `onSelectionChanged()`.
