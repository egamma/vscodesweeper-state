# Issue #171040 — Automatic enter indentation in offside folding languages

[https://github.com/microsoft/vscode/issues/171040](https://github.com/microsoft/vscode/issues/171040)

## Summary

This is a feature request for improved auto-indentation in indentation-based languages (Python, etc.). The current auto-indent infrastructure relies on regex-based indentation rules and onEnter rules which don't adequately handle offside-rule languages. No implementation has landed on main.

## Change summary

Pressing Enter on an unindented line inside an indentation-based block (e.g. Python) should auto-indent to the current block level when subsequent lines indicate the block continues.

## Best solution

Enhance the editor's enter-indentation logic (src/vs/editor/common/languages/autoIndent.ts) to consider surrounding context — specifically, whether lines after the cursor maintain the same indentation level — so offside-rule languages get correct block-aware indentation on Enter. This likely also requires richer language configuration support or heuristics beyond existing indentationRules.
