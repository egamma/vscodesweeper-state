# Issue #245385 — C# language server doesn't start on MacOS 11

[https://github.com/microsoft/vscode/issues/245385](https://github.com/microsoft/vscode/issues/245385)

## Summary

The failure is in the .NET runtime downloaded by the ms-dotnettools.vscode-dotnet-runtime extension — its libhostfxr.dylib was built against macOS 12 and won't load on macOS 11. This is not a VS Code core issue; it belongs with the C# / .NET extension team.

## Change summary

C# language server fails to start on macOS 11 because the .NET runtime extension downloads a binary built for macOS 12+.

## Best solution

Report to the ms-dotnettools extensions (https://github.com/dotnet/vscode-csharp or https://github.com/dotnet/vscode-dotnet-runtime) — the .NET 9.0.3 arm64 runtime they bundle raised its minimum macOS version to 12. VS Code core has no involvement in the C# language server or its runtime provisioning.
