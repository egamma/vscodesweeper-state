# Issue #291640 — Debug disassembly view sometimes assumes a single contiguous memory space despite instructionReference changes

[https://github.com/microsoft/vscode/issues/291640](https://github.com/microsoft/vscode/issues/291640)

## Summary

Valid bug: the disassembly view caches instructions keyed by address without considering that a different instructionReference may refer to a different memory space. An open PR (#310782) is already attempting to fix this — keep open until it lands.

## Change summary

The disassembly view retains previously fetched instructions across instructionReference changes, showing stale data from a different process/memory region.

## Best solution

Review and land https://github.com/microsoft/vscode/pull/310782, which clears/reloads disassembly when the instructionReference targets a different memory region. The fix belongs in `src/vs/workbench/contrib/debug/browser/disassemblyView.ts`.
