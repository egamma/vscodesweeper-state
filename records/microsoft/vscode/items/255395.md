# Issue #255395 — Explorer component is blank when using a virtual drive

[https://github.com/microsoft/vscode/issues/255395](https://github.com/microsoft/vscode/issues/255395)

## Summary

This was fixed by a workaround for an upstream Node.js/Electron regression affecting `subst` drives. The fix landed on main three days after this issue was filed and shipped in 1.103.0, with a follow-up hardening commit in 1.104.0.

## Change summary

The File Explorer shows a blank view when a folder opened via `subst.exe` virtual drive is used as the workspace root.

## Best solution

Keep the shipped workaround (commit d9d3f3c6ec0, 1.103.0) and the follow-up hardening (commit 1197402466c); no further change needed. The upstream root cause is tracked in https://github.com/microsoft/vscode/issues/252361.
