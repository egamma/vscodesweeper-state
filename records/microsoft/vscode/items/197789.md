# Issue #197789 — Shouldn't "integrate" the first view into its container icon when in the secondary side bar

[https://github.com/microsoft/vscode/issues/197789](https://github.com/microsoft/vscode/issues/197789)

## Summary

The core request — not integrating a single view's title into the container icon in the secondary sidebar — remains partially unaddressed. Commit 6fd5ed4b706 (Feb 2025) added a `showLabels` setting that improves discoverability of view names, but the specific single-view integration behavior (collapsing the view header into the container) still appears to be the original UX concern. Team members engaged and agreed it should be unified with the primary sidebar behavior.

## Change summary

When the secondary sidebar has a single view, its title disappears into the container icon, losing context — the request is to render it like the multi-view case with a visible label/title.

## Best solution

Unify the single-view rendering in the secondary sidebar with the primary sidebar's activity-bar-on-top behavior, so a single view container always shows its title/label rather than integrating into an icon-only header. The `showLabels` default (6fd5ed4b706) helps but doesn't fully solve the single-view integration case described.
