# Issue #197789 — Shouldn't "integrate" the first view into its container icon when in the secondary side bar

[https://github.com/microsoft/vscode/issues/197789](https://github.com/microsoft/vscode/issues/197789)

## Summary

Still valid on current main: a single view in the secondary sidebar with the default activity bar position integrates into the container, losing its title context. The related PR #211794 was closed without merging, and subsequent label improvements (6fd5ed4b706) don't address this specific single-view integration behavior. A team member (sbatten) agreed it should unify with the primary sidebar's activity-bar-on-top behavior.

## Change summary

When the secondary sidebar has a single view with default activity bar position, the view title disappears because the view integrates into the container icon.

## Best solution

Unify the single-view rendering in the secondary sidebar with how the primary sidebar handles it when the activity bar is on top — show the view title bar rather than integrating into the icon. The fix would be in `src/vs/workbench/browser/parts/auxiliarybar/auxiliaryBarPart.ts`, likely adjusting `getCompositeBarPosition()` or how the composite bar renders for single-view containers, but needs care since comment #3 notes user friction with the chat view layout.
