# Issue #233131 — Terminal: failed link detection on WSL filesystem when wordSeparators contains slashes

[https://github.com/microsoft/vscode/issues/233131](https://github.com/microsoft/vscode/issues/233131)

## Summary

Still reproduces on main: the terminal word link detector splits file paths into individual words when wordSeparators includes slashes, causing link detection to fail on path components. No fix has landed since the report.

## Change summary

Terminal link detection fails for file paths on WSL filesystem when terminal.integrated.wordSeparators contains forward/back slashes, because the word link detector splits the path at separator characters.

## Best solution

The word link detector (terminalWordLinkDetector.ts) should not produce word links that overlap with higher-priority local file links already detected by the local link detector, or the link resolution logic in xterm should prefer the broader local-file link over the narrower word link when both cover the clicked position. The fix likely belongs in `terminalLinkDetectorAdapter.ts` or `terminalWordLinkDetector.ts`.
