# Issue #36893 — Git - Support to compare with arbitrary commit instead of the current HEAD

[https://github.com/microsoft/vscode/issues/36893](https://github.com/microsoft/vscode/issues/36893)

## Summary

Well-supported feature request (100 upvotes) for comparing gutter annotations against an arbitrary branch/commit instead of HEAD. The capability is not yet implemented — the built-in git quick diff provider still compares against HEAD only. Recent API work (proposed `createQuickDiffInformation`) is progressing but not finalized. Keeping open in Backlog.

## Change summary

Allow the editor's inline change annotations (gutter highlights, minimap markers) to diff against a user-chosen branch, tag, or commit rather than always comparing to HEAD.

## Best solution

The git extension's `GitQuickDiffProvider` (extensions/git/src/quickDiffProvider.ts) needs a way to resolve the original resource against a user-selected ref instead of HEAD. This depends on finalizing the proposed `textEditorDiffInformation` / `createQuickDiffInformation` API (https://github.com/microsoft/vscode/issues/169012) and then exposing a user-facing command in the git extension to select the comparison base. GitLens (comment 11) also plans to leverage this API once it lands.
