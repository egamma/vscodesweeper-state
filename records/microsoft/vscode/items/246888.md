# Issue #246888 — "Disable All Installed Extensions" not working because of language file

[https://github.com/microsoft/vscode/issues/246888](https://github.com/microsoft/vscode/issues/246888)

## Summary

Still reproduces on main: the 'Disable All Installed Extensions' command correctly filters language packs via `canChangeEnablement`, but `promptAndSetEnablement` → `getExtensionsRecursively` can re-introduce language pack extensions as pack members, which then throw in the enablement service. The error notification surfaces to the user and the operation fails.

## Change summary

'Disable All Installed Extensions' command fails with a notification error when a language pack extension is installed, because pack-dependency resolution re-adds it after the initial filter.

## Best solution

In `getExtensionsRecursively` (extensionsWorkbenchService.ts ~line 3179), add a `canChangeEnablement` guard so that extensions that cannot be disabled are not pulled in as pack members during bulk disable operations. Alternatively, catch and skip the error for extensions that fail `throwErrorIfCannotChangeEnablement` during bulk operations in `setEnablement`.
