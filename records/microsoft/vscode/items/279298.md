# Issue #279298 — Cannot read properties of null (reading 'domNode'

[https://github.com/microsoft/vscode/issues/279298](https://github.com/microsoft/vscode/issues/279298)

## Summary

Fixed on main and shipped: commit d68eb349b2a guards the chat list renderer against firing height-change events from disconnected templates, which was the root cause of the null `domNode` crash in `updateItemInDOM`. An additional defense was added in 319c95890d8 for the general listView path.

## Change summary

Opening a cloud session triggers a TypeError in `listView.updateItemInDOM` because `item.row` is null when the chat renderer fires a height-change event from a disconnected template.

## Best solution

Keep the shipped fixes — d68eb349b2a prevents the chat renderer from firing height updates on disconnected templates, and 319c95890d8 adds a general null guard in listView's insertItemInDOM. No further change needed.
