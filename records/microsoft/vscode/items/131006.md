# Issue #131006 — Ensure that scm.focus command works in all cases

[https://github.com/microsoft/vscode/issues/131006](https://github.com/microsoft/vscode/issues/131006)

## Summary

The core behavior still exists on main: `SCMViewPane.focus()` restores the last-focused tree element instead of always focusing the commit input. Dedicated `focusPreviousInput`/`focusNextInput` commands (added May 2024) partially address the use case but the default `Ctrl+Shift+G` / `workbench.view.scm` focus path remains inconsistent.

## Change summary

The `workbench.scm.focus` (now `workbench.view.scm`) command should always focus the commit message input, not restore focus to a previously selected file in the SCM tree.

## Best solution

Modify `SCMViewPane.focus()` in `src/vs/workbench/contrib/scm/browser/scmViewPane.ts` to always focus the commit input widget when the view is activated via its default keybinding, matching the behavior users expect from `Ctrl+Shift+G`. The existing `focusPreviousInput`/`focusNextInput` commands handle multi-repo cycling; the base `focus()` should handle the initial landing.
