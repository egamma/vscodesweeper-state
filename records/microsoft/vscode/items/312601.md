# Issue #312601 — Copilot - Allow user to override the base url of the BYOK model provider

[https://github.com/microsoft/vscode/issues/312601](https://github.com/microsoft/vscode/issues/312601)

## Summary

Valid feature request to expose a base URL override on the standard BYOK providers (OpenAI, Anthropic, etc.) so users don't have to fall back to the generic Custom Endpoint provider. The Custom Endpoint provider already covers this use case functionally, but the UX could be simpler for users connecting to an OpenAI-compatible third-party endpoint. Routing to the model-byok area for triage.

## Change summary

Allow users to override the base URL when adding a standard BYOK provider (OpenAI, Anthropic, etc.) instead of requiring them to use the generic Custom Endpoint provider for alternative-API-compatible services.

## Best solution

The Custom Endpoint provider (`customEndpointProvider.ts`) already supports arbitrary URLs with OpenAI-compatible and Anthropic Messages API types. The cleanest path is either (a) improving discoverability of the Custom Endpoint provider as the intended route for third-party compatible endpoints, or (b) adding an optional base-URL override to `AbstractOpenAICompatibleLMProvider.getModelsBaseUrl()` so standard providers can read it from configuration. Issue https://github.com/microsoft/vscode/issues/318101 is a related but distinct request (Azure-specific URL inference bugs). This request should be evaluated alongside the broader BYOK UX work tracked under the `model-byok` label.
