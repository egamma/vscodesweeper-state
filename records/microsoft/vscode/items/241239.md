# Issue #241239 — GitHub - "Open on GitHub" does not show for github private repo remote with auth params

[https://github.com/microsoft/vscode/issues/241239](https://github.com/microsoft/vscode/issues/241239)

## Summary

Still reproduces on main — the `getRepositoryFromUrl` regex in the GitHub extension does not account for credentials embedded in the remote URL (`USERNAME:PAT@`), so the extension fails to recognize the remote as a GitHub repo. Milestone is "On Deck", which triggers the hard keep-open rule.

## Change summary

The "Open on GitHub" action disappears when the git remote URL contains embedded authentication credentials (e.g. `https://user:token@github.com/owner/repo`).

## Best solution

Update the regex in `getRepositoryFromUrl` (`extensions/github/src/util.ts`) to optionally match an embedded `user(:pass)?@` segment before `github.com`, e.g. `https://([^@]+@)?github.com/…`. This is a small, localized fix.
