# Issue #134144 — Explorer does not refresh when previously removed root is coming back

[https://github.com/microsoft/vscode/issues/134144](https://github.com/microsoft/vscode/issues/134144)

## Summary

Team-authored issue (MEMBER); the bug is still present on main — the file-change handler in explorerService.ts does not detect re-creation of a root folder because the ADDED-event logic only checks children of resolved parents, not the roots themselves. Keeping open per policy and on merit.

## Change summary

Explorer does not auto-refresh when a previously deleted workspace root folder reappears while the window is unfocused.

## Best solution

In `explorerService.ts`, extend the ADDED-event handling (around line 84) to also check whether any root folder that was previously marked as non-existent has reappeared. This should trigger a refresh only for roots in a deleted/error state to avoid unnecessary full-tree refreshes, consistent with bpasero's comment.
