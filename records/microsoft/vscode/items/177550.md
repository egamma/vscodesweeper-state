# Issue #177550 — npm scripts do not run in integrated terminal when it has init script for cmd and powershell

[https://github.com/microsoft/vscode/issues/177550](https://github.com/microsoft/vscode/issues/177550)

## Summary

The behavior is as-designed: tasks use the default terminal profile unless `terminal.integrated.automationProfile` is configured. The reporter's own workaround (setting automationProfile to a plain shell) is the intended solution, and the maintainer acknowledged this pattern.

## Change summary

NPM scripts from package.json explorer don't complete when the default terminal profile has init scripts that block subsequent commands.

## Best solution

Close as designed — `terminal.integrated.automationProfile` exists precisely to give tasks a clean shell without init scripts. The reporter already discovered and documented the workaround in their first comment.
