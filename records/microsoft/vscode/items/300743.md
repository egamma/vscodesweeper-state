## Summary

This request for a `--no-recurse-submodules` option on git sync/pull is a specific case of the broader submodule management work tracked by the open canonical issue #52700 (Git - Support submodule management), which has 24+ upvotes, is in the Backlog milestone, and would absorb this request. Proposing duplicate close.

## Change summary

Git sync/pull in projects with many submodules is very slow because VS Code doesn't offer a `--no-recurse-submodules` flag.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/52700 — implementing proper submodule management there (including configuration for recursive submodule behavior during pull/sync) delivers exactly this request. The current `pull()` in `extensions/git/src/git.ts` builds the git args without any submodule recursion control; adding a `--no-recurse-submodules` option would be part of the #52700 work.
