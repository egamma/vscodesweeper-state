# Issue #194987 — Merge editor should not trigger on save events (such as formatting)

[https://github.com/microsoft/vscode/issues/194987](https://github.com/microsoft/vscode/issues/194987)

## Summary

Still valid on main: the merge editor's 'Complete Merge' action calls save without `skipSaveParticipants: true`, so format-on-save and other save participants fire and mutate the merged result. A team member (hediet) has already pinged another maintainer for input, indicating active interest.

## Change summary

Merge editor's 'Complete Merge' triggers format-on-save, adding unintended formatting changes to the staged merge result.

## Best solution

Pass `skipSaveParticipants: true` in the save options when the merge editor calls `accept()` — specifically in `TempFileMergeEditorInputModel.accept()` (line 165) and `WorkspaceMergeEditorInputModel.accept()` (line 420) in `src/vs/workbench/contrib/mergeEditor/browser/mergeEditorInputModel.ts`. This ensures the file is saved exactly as seen in the merge editor without triggering formatters or other save participants.
