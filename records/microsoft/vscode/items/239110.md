# Issue #239110 — Activating display pointer trails makes text editing pointer disappear

[https://github.com/microsoft/vscode/issues/239110](https://github.com/microsoft/vscode/issues/239110)

## Summary

This is an upstream Chromium/Electron issue: Windows 'Display pointer trails' interacts poorly with CSS text cursors on scaled external displays. VS Code has no custom pointer-trail handling — it relies on the browser engine's cursor rendering.

## Change summary

The CSS text-select cursor (I-beam) disappears when Windows pointer trails are enabled, specifically on scaled external monitors.

## Best solution

File upstream against Chromium (or check existing Chromium bugs for 'pointer trails CSS cursor') since VS Code's editor area uses standard CSS `cursor: text` and Electron/Chromium is responsible for rendering it. If a workaround is needed, VS Code could potentially offer an option to use a custom SVG cursor instead of the native text cursor, but that's heavyweight for a niche Windows setting.
