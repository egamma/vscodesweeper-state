# Issue #239110 — Activating display pointer trails makes text editing pointer disappear

[https://github.com/microsoft/vscode/issues/239110](https://github.com/microsoft/vscode/issues/239110)

## Summary

This is an upstream issue in how Windows pointer trails interact with the Chromium/Electron compositor's cursor rendering. VS Code only sets the standard CSS `cursor: text` — the trail rendering and cursor visibility is handled by the OS and Electron's graphics layer.

## Change summary

Enabling Windows 'Display pointer trails' causes the I-beam text cursor to disappear over the VS Code editor area on certain external displays.

## Best solution

Track as upstream (Electron/Chromium). The related open issue https://github.com/microsoft/vscode/issues/239283 describes the same symptom family. If this warrants a workaround, it would likely involve a custom SVG cursor in the CSS rather than relying on the OS-rendered `cursor: text`, but that is a significant tradeoff. Best path is an upstream Chromium/Electron bug report.
