# Issue #110794 — [json] no IntelliSense when all-optional branch matches

[https://github.com/microsoft/vscode/issues/110794](https://github.com/microsoft/vscode/issues/110794)

## Summary

Team-authored feature request for improved JSON IntelliSense when a oneOf contains an all-optional branch that greedily matches, suppressing proposals from other branches. Keeping open per team-member policy.

## Change summary

JSON IntelliSense fails to propose properties from other oneOf branches when one branch with no required properties matches the current (empty) object.

## Best solution

The JSON language service's completion logic should consider proposals from all matching oneOf branches (not just the first/best match) when the best-matching branch has only optional properties. This likely lives in the vscode-json-languageservice package (upstream dependency) rather than in the built-in extension itself.
