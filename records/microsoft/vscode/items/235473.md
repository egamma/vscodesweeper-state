# Issue #235473 — line numbers are not highlighted on the wrapped parts of long lines

[https://github.com/microsoft/vscode/issues/235473](https://github.com/microsoft/vscode/issues/235473)

## Summary

Already fixed on main by a targeted line-number rendering change, so this can be closed as completed rather than with a triage-reason label.

## Change summary

Active line-number highlighting did not follow the cursor when it was on a wrapped segment of a long line.

## Best solution

Keep the shipped fix from commit 503bace89cb48c7ef8cbee8109e262889bf99907, which tracks the active model line instead of the active view line for wrapped editor lines.

  > Thanks for the report. This was fixed by https://github.com/microsoft/vscode/pull/240029 in commit 503bace89cb48c7ef8cbee8109e262889bf99907 and is included starting with 1.101.0. The editor now tracks the active model line for line-number highlighting, so wrapped portions of the same long line get the active line-number styling as expected.

## Evidence

- **fix commit:** Commit 503bace89cb48c7ef8cbee8109e262889bf99907 is titled "Highlight active line number correctly regardless of word wrap (#240029)" and its message says the line number was previously only highlighted when the cursor was on the first part of a wrapped line.
- **code change:** src/vs/editor/browser/viewParts/lineNumbers/lineNumbers.ts now stores _activeModelLineNumber from convertViewPositionToModelPosition(...) and applies active-line-number when the rendered view line maps to that model line.
- **release:** git tag --contains shows the fix is included starting with tag 1.101.0.
