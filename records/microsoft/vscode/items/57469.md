# Issue #57469 — [html] Support SCSS in HTML with <style type="text/scss">

[https://github.com/microsoft/vscode/issues/57469](https://github.com/microsoft/vscode/issues/57469)

## Summary

Still unimplemented on main. The HTML language server hardcodes 'css' for all <style> blocks and does not inspect the type attribute to switch to SCSS/Less. This is a valid feature request in the Backlog.

## Change summary

Support SCSS language features (IntelliSense, validation) inside HTML <style type="text/scss"> blocks.

## Best solution

Extend the embedded language region detection in `extensions/html-language-features/server/src/modes/embeddedSupport.ts` to read the `type` attribute on `<style>` tags (similar to how `<script>` resolves `languageIdFromType`) and map `text/scss` → `scss` and `text/less` → `less`, then wire up the corresponding language services in cssMode.ts. Alternatively, this could be an extension-candidate if the upstream `vscode-html-languageservice` scanner doesn't expose the needed hooks.
