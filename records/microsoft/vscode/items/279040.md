# Issue #279040 — Allow to use `applyEdit` w/ AI-review UI from tools and also refactorings

[https://github.com/microsoft/vscode/issues/279040](https://github.com/microsoft/vscode/issues/279040)

## Summary

Valid feature request with team engagement (jrieken proposed a concrete design direction). Not yet implemented on main — the stable and proposed APIs still lack the ability to route `applyEdit` edits into the AI review UI via a tool invocation token. Keeping open as an active backlog item.

## Change summary

Allow extension tools to use the AI edit review UI (Keep/Undo) when making edits, either by passing a tool invocation token to `workspace.applyEdit` or by exposing the chat response stream to tool invocations.

## Best solution

Implement the approach jrieken outlined: extend `workspace.applyEdit` to accept a `ChatParticipantToolToken` so edits from tools are routed into the tracked-edits review UI. This avoids exposing the full response stream and keeps the API surface clean. The work belongs in the extension API layer (`src/vs/workbench/api/`) and the chat editing infrastructure.

- **Evidence:** Aligns with current agent-host and agent-security priorities — making the edit review flow available to third-party tools strengthens the agent ecosystem without sacrificing the trust/consent model.
