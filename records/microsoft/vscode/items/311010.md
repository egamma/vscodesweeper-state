# Issue #311010 — Copilot Chat crashes when repo remote has only pushurl (TypeError: Cannot read properties of undefined (reading 'trim'))

[https://github.com/microsoft/vscode/issues/311010](https://github.com/microsoft/vscode/issues/311010)

## Summary

Confirmed source-reproducible bug: when a git remote has only `pushurl` and no `url`, the VS Code git extension sets `fetchUrl` to `undefined`, which flows into the copilot extension's `remoteFetchUrls` array. The non-null assertion at `getOrderedRemoteUrlsFromContext` line 132 passes `undefined` into the Set, eventually reaching `parseRemoteUrl` which calls `.trim()` on it and crashes.

## Change summary

Copilot Chat crashes with TypeError when a git remote has only a `pushurl` and no `url` in `.git/config`.

## Best solution

Guard `getOrderedRemoteUrlsFromContext` in `extensions/copilot/src/platform/git/common/gitService.ts` to skip undefined entries (remove the `!` non-null assertion at line 132 and add a truthy check), and similarly guard line 521 in `gitServiceImpl.ts` or filter undefined values from `remoteFetchUrls`. A secondary defensive guard in `parseRemoteUrl` would also be prudent.
