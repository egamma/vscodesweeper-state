## Change summary

Allow users to toggle word wrapping on specific lines so that wide content like Markdown tables and lists remains unwrapped while the rest wraps normally.

## Best solution

This is closely related to https://github.com/microsoft/vscode/issues/164267 (language-aware word wrap) but differs in interaction model (manual toggle vs automatic detection). Both likely need the same underlying engine support in the editor wrapping logic (`src/vs/editor/common/viewModel/viewModelLines.ts` and related). A unified approach that exposes per-line wrap exclusion — usable both manually and via language-aware heuristics — would serve both issues. Keep open as a valid Backlog item.
