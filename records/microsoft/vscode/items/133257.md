# Issue #133257 — Adopt onDidCloseEditor context in terminal

[https://github.com/microsoft/vscode/issues/133257](https://github.com/microsoft/vscode/issues/133257)

## Summary

Still valid debt: the terminal's onDidCloseEditor handler (terminalEditorService.ts line 96) does not use EditorCloseContext to differentiate close vs. move — it removes the instance on any close event, including drag-and-drop moves. The issue remains actionable.

## Change summary

Use EditorCloseContext in the terminal's onDidCloseEditor handler to differentiate editor close from editor move, making terminal tab moves more robust.

## Best solution

In `src/vs/workbench/contrib/terminal/browser/terminalEditorService.ts`, check `e.context` (EditorCloseContext) in the onDidCloseEditor handler so the terminal instance is only removed on a true close, not on a move/drag. This would eliminate the need for workarounds when terminals are moved between groups.
