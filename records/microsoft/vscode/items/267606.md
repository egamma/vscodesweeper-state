# Issue #267606 — Git - better support for bare repositories

[https://github.com/microsoft/vscode/issues/267606](https://github.com/microsoft/vscode/issues/267606)

## Summary

Still reproduces on main: bare-repo worktree detection has multiple issues — worktrees opened inside bare repos aren't properly recognized, and the closedRepositories guard is bypassed when worktrees are re-scanned via `eventuallyScanPossibleGitRepository`. An open community PR (#267692) exists but hasn't merged. Keeping open with existing `help wanted` label.

## Change summary

VS Code doesn't properly detect worktree hierarchies in bare repositories, and closed worktree repositories reopen because the worktree scan path bypasses the closed-repository guard.

## Best solution

The worktree scan in `checkForWorktrees` (model.ts ~L822) calls `eventuallyScanPossibleGitRepository` which eventually calls `openRepository` with default `openIfClosed=false` — but the re-scan triggered by status changes may bypass this. The community PR https://github.com/microsoft/vscode/pull/267692 attempts to fix detection from inside/outside worktrees. A maintainer should review that PR or implement the fix in `extensions/git/src/model.ts` to properly handle bare-repo worktree hierarchies and respect `closedRepositories` on re-scan.
