# Issue #235973 — VSCode launch failed (code 43) (conflict with MacType?)

[https://github.com/microsoft/vscode/issues/235973](https://github.com/microsoft/vscode/issues/235973)

## Summary

Third-party DLL injection (MacType) disrupts Chromium's sandbox child-process launch. The issue is already labeled 'mitigated' with a working workaround (`disable-chromium-sandbox`), and a team member (deepak1556) has investigated. The root cause is in Chromium's sandbox layer — keeping open for the team to decide whether further action is needed.

## Change summary

VS Code fails to launch with exit code 43 when started indirectly (CLI, GitHub Desktop) on Windows with MacType installed, due to MacType's DLL injection conflicting with Chromium's sandbox process verification.

## Best solution

The `disable-chromium-sandbox` runtime argument (introduced in commits 7f9a404d08d and 35ddc72ddc0) already mitigates this. The underlying issue is in Chromium's sandbox (`target_process.cc` PEB base-address check) being disrupted by third-party font-rendering injection. Long-term, this may resolve upstream in Chromium or MacType. Consider whether documentation should mention known third-party software conflicts.
