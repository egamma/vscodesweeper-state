# Issue #247335 — QuickDiffProvider is not refreshable

[https://github.com/microsoft/vscode/issues/247335](https://github.com/microsoft/vscode/issues/247335)

## Summary

Valid feature request for an `onDidOriginalResourceChanged` event on the QuickDiffProvider interface and better null-handling in getOriginalResource. A maintainer has engaged and pointed to the existing FileSystemProvider workaround. The issue is already labeled `under-discussion` and in `Backlog`, which is appropriate — it should stay open for continued API design discussion.

## Change summary

The QuickDiffProvider interface lacks an event to notify VS Code when the original (base) resource has changed, forcing extension authors to use indirect FileSystemProvider workarounds.

## Best solution

Evaluate adding an `onDidOriginalResourceChanged` event to the `QuickDiffProvider` interface (or the proposed extended API at `vscode.proposed.quickDiffProvider.d.ts`) so SCM extensions can directly signal base-content changes without routing through a FileSystemProvider. The maintainer's FileSystemProvider pattern (extensions/git/src/fileSystemProvider.ts L66-79) is a viable documented workaround for now.
