# Issue #192376 — macOS: `tab.activeBorderTop` does not appear with native title bar and dark OS theme

[https://github.com/microsoft/vscode/issues/192376](https://github.com/microsoft/vscode/issues/192376)

## Summary

Confirmed upstream issue: macOS dark mode draws a native border (via Electron/Chromium) that covers the first pixel row of the workbench, hiding `tab.activeBorderTop` when using native title bars. The maintainer identified this as not under VS Code's control. Keeping open as an upstream/electron tracking issue.

## Change summary

The `tab.activeBorderTop` color is hidden on macOS when using native title bar with the OS dark theme, because a native Electron/Chromium border covers the first pixel line.

## Best solution

This is an Electron/Chromium compositing issue on macOS dark mode. A potential VS Code-side workaround would be to offset the top border by 1px when using native title bars on macOS dark mode, but the proper fix belongs in Electron. The issue is correctly labeled `electron` and `confirmed` and should remain open as an upstream tracker until Electron addresses the dark-mode native border rendering.
