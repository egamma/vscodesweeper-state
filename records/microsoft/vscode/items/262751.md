# Issue #262751 — Git - Support diff decorations relative to a particular commit

[https://github.com/microsoft/vscode/issues/262751](https://github.com/microsoft/vscode/issues/262751)

## Summary

Valid feature request — the git quick diff infrastructure currently only supports HEAD and staged (index) as diff bases; there is no mechanism to specify an arbitrary commit. The request remains unimplemented on current main. Keeping open in Backlog for community traction.

## Change summary

Allow choosing an arbitrary earlier commit (e.g. origin/main) as the base for quick diff gutter decorations, rather than only HEAD or the index.

## Best solution

Extend `GitQuickDiffProvider` in `extensions/git/src/quickDiffProvider.ts` to accept a configurable commit ref (instead of hardcoded empty string / 'HEAD') and expose a UI affordance (e.g. a picker or setting) to select the base commit. This likely also requires changes to the SCM QuickDiffProvider API surface in `src/vs/workbench/contrib/scm/` to support user-selectable bases beyond the current primary/secondary provider model.
