# Issue #307395 — github_repo tool in Copilot Chat does not index .yml files from repositories

[https://github.com/microsoft/vscode/issues/307395](https://github.com/microsoft/vscode/issues/307395)

## Summary

The github_repo tool's file-type coverage is determined by GitHub's server-side code search and embedding service, not by VS Code. The client code in the Copilot extension simply forwards the query to the remote semantic search API and returns whatever results the backend provides — no file-type filtering happens locally. This belongs in the GitHub Copilot service backend, not the VS Code repo.

## Change summary

The github_repo tool in Copilot Chat does not return results from .yml files when querying a repository.

## Best solution

File this against the GitHub Copilot backend service (the team that owns the code search / embedding indexing pipeline) — the VS Code extension is a passthrough here. The fix is to ensure the server-side indexer includes .yml/.yaml files in its embedding corpus.
