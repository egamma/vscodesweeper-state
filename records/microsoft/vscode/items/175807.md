# Issue #175807 — [html] Path completion isn’t working with the `srcset` attribute

[https://github.com/microsoft/vscode/issues/175807](https://github.com/microsoft/vscode/issues/175807)

## Summary

Still valid on current main: `srcset` is not recognized as a path attribute in vscode-html-languageservice, so path completion never triggers for it. The fix belongs upstream in the language service package.

## Change summary

Path completion (autocomplete for file paths) does not activate inside the `srcset` attribute on `<img>` and `<source>` elements.

## Best solution

Add `srcset` to the `PATH_TAG_AND_ATTR` map (or the `isPathAttribute` logic) in `vscode-html-languageservice` for `img` and `source` tags. Because `srcset` values contain path + descriptor pairs (e.g. `image.jpg 2x`), the path completion participant may also need adjustment to parse only the path portion before the descriptor.
