# Issue #321092 — all submenu items have in-focus border

[https://github.com/microsoft/vscode/issues/321092](https://github.com/microsoft/vscode/issues/321092)

## Summary

Valid visual bug: submenu items show a focus border on all items instead of only the focused one. Likely introduced or exposed by the selection border changes in commit a0efe49e539 (PR #319306). No fix has been applied since the report; the CSS rules for `:focus-visible` and the inline outline logic in `applyStyle()` are still in their June 1 state.

## Change summary

When hovering over an expandable menu item, the submenu appears with all menu items showing a rectangular focus border instead of only the focused item.

## Best solution

Investigate the interaction between the `:focus-visible` CSS suppression rule added in commit a0efe49e539 and the submenu rendering in `src/vs/base/browser/ui/menu/menu.ts`. The outline logic in `applyStyle()` (line ~707) and the CSS rules (lines ~1270–1282) may need adjustment to ensure only the truly focused submenu item receives the selection border, particularly when the submenu is opened from a keyboard-active menu context.
