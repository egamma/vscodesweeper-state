[https://github.com/microsoft/vscode/issues/286994](https://github.com/microsoft/vscode/issues/286994)

## Summary

Still reproduces on main — zoxide's PROMPT_COMMAND hook interferes with terminal command detection, causing #terminalLastCommand to report stale or incorrect data. A maintainer is actively triaging (asked for /troubleshoot output) and the bug is confirmed by a second reporter.

## Change summary

When zoxide is installed, its shell hook confuses terminal command detection so that #terminalLastCommand returns incorrect results.

## Best solution

The command detection layer (src/vs/platform/terminal/common/capabilities/commandDetectionCapability.ts) needs to filter or ignore output injected by PROMPT_COMMAND hooks between command end and the next prompt. The maintainer (meganrogge) is actively investigating with troubleshoot data from the reporter.
