# Issue #296258 — Copilot circumvents file exclusions

[https://github.com/microsoft/vscode/issues/296258](https://github.com/microsoft/vscode/issues/296258)

## Summary

Valid request for stricter file-exclusion enforcement in agent mode. A team member engaged but the reporter disagrees with the suggested workaround; no resolution yet. Keeping open as the desired behavior (hard-block excluded files across all tools including terminal) is a reasonable enhancement.

## Change summary

Copilot agent mode circumvents file exclusions by using the terminal tool (e.g. Get-Content) to read files that are excluded via .copilotignore or GitHub content exclusion settings.

## Best solution

Enforce file exclusion policies across all agent tool invocations, not just the file-read tool. The terminal tool's command-analysis heuristics (in `src/vs/workbench/contrib/terminalContrib/chatAgentTools/`) should detect and block file-access commands targeting excluded paths. Related to https://github.com/microsoft/vscode/issues/253884 (tool re-routing) but distinct — this is about exclusion enforcement, not routing optimization.
