# Issue #25986 — [theming] Separate workbench theme and syntax theme

[https://github.com/microsoft/vscode/issues/25986](https://github.com/microsoft/vscode/issues/25986)

## Summary

This feature request for separating workbench UI themes from syntax highlighting themes remains unimplemented. VS Code still uses a single color theme setting; workarounds exist via `workbench.colorCustomizations` and `editor.tokenColorCustomizations`, but no first-class UI separation has been built. The issue is in Backlog and has community interest spanning many years.

## Change summary

Allow users to independently select a workbench UI theme and a syntax highlighting theme, rather than bundling both in a single color theme.

## Best solution

Either keep as a long-term feature request in Backlog (community workarounds exist via color customization settings) or formally decline as `*as-designed` given the existing override mechanism. The most practical path may be documenting the override workflow more prominently rather than implementing full separation.
