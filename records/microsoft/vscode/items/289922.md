# Issue #289922 — Terminal being opened in wrong directory in multi-root workspace

[https://github.com/microsoft/vscode/issues/289922](https://github.com/microsoft/vscode/issues/289922)

## Summary

Legitimate bug report about terminal restoration in multi-root workspaces choosing the wrong workspace root for the cwd. The code's fallback via `historyService.getLastActiveWorkspaceRoot()` is documented as unreliable (a TODO in the source acknowledges this). Needs routing to the terminal area for further investigation.

## Change summary

Restored terminal in a multi-root workspace opens in the wrong workspace folder instead of the first (default) root.

## Best solution

The `getWorkspaceForTerminal` fallback in `src/vs/workbench/contrib/terminal/common/terminalEnvironment.ts` uses the unreliable `historyService.getLastActiveWorkspaceRoot()` when no explicit cwd is set. The fix should ensure that restored terminals default to the first workspace folder (or the folder that was active when the terminal was last used) rather than relying on a stale 'last active' root that may come from a different workspace.
