# Issue #179474 — [Linux Mint] Program icon doesn't match Linux Mint theme

[https://github.com/microsoft/vscode/issues/179474](https://github.com/microsoft/vscode/issues/179474)

## Summary

Still valid on current main: VS Code's Linux packaging only installs a single icon (resources/linux/code.png) without placing sized icons into the standard hicolor icon theme directories, so desktop environments like Linux Mint that look up icons via the freedesktop icon theme spec fall back to a generic icon. A maintainer already marked it help-wanted and invited a PR.

## Change summary

VS Code's icon doesn't integrate with Linux Mint's icon theme because sized icons aren't installed into hicolor theme directories.

## Best solution

Update the Linux .deb/.rpm packaging (build scripts and resources/linux/) to install appropriately sized PNG/SVG icons into /usr/share/icons/hicolor/{size}/apps/ so that freedesktop-compliant icon themes (Mint, GNOME, KDE) discover them. The existing `help wanted` label makes this a good community contribution target.
