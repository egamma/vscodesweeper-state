# Issue #241801 — WebView editor can dispatch keybinding when workbench does already

[https://github.com/microsoft/vscode/issues/241801](https://github.com/microsoft/vscode/issues/241801)

## Summary

Still reproduces on main for extension webviews and built-in webview editors: c74dbf449ec only blocks untrusted re-dispatch for notebook webviews but leaves extension/built-in webviews (image preview, Claude Code, etc.) exposed to the double-dispatch race between webview key forwarding and macOS menu accelerators.

## Change summary

Keybindings fire twice when a webview-based editor tab is focused on macOS — once from the webview's key-forwarding path and once from the Electron menu accelerator.

## Best solution

Extend the deduplication logic in `webviewElement.ts#handleKeyEvent` (or `abstractKeybindingService.ts#dispatchByUserSettingsLabel`) to detect when a keybinding has already been dispatched for the same physical keystroke, preventing the menu-accelerator path from firing a second time. The fix likely touches `src/vs/workbench/contrib/webview/browser/webviewElement.ts` and possibly `src/vs/platform/keybinding/common/abstractKeybindingService.ts`.
