# Issue #247218 — Index in model selector is skipping the first item of the 11 options

[https://github.com/microsoft/vscode/issues/247218](https://github.com/microsoft/vscode/issues/247218)

## Summary

Still reproduces on main: the ActionList widget does not provide custom getSetSize/getPosInSet implementations, so headers and separators are counted as items in the ARIA set-size/pos-in-set announcements, inflating the count and offsetting the index for screen reader users.

## Change summary

The model picker (and other ActionList-backed widgets) announces incorrect item index and count to screen readers because non-action items (headers, separators) are included in the ARIA set size.

## Best solution

Implement custom `getSetSize` and `getPosInSet` in the ActionList's accessibility provider (in `src/vs/platform/actionWidget/browser/actionList.ts`) that exclude `Header` and `Separator` kind items from the count and adjust positions accordingly. This would fix the issue for all ActionList consumers (model picker, code actions, permission picker).
