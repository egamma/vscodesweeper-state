# Issue #280104 — terminal profile can't resolve `${workspaceFolder}` in vscode remote workspaces

[https://github.com/microsoft/vscode/issues/280104](https://github.com/microsoft/vscode/issues/280104)

## Summary

Valid bug filed by a team member with an open fix PR (#280106) — the hardcoded `Schemas.file` in baseTerminalBackend.ts prevents variable resolution in remote workspaces. Keeping open pending PR merge.

## Change summary

Terminal profiles using `${workspaceFolder}` fail to resolve in remote workspaces because the variable resolver only looks for `file:` scheme workspace roots.

## Best solution

Land PR #280106 which should replace the `Schemas.file` filter with a scheme-agnostic lookup (or add the remote scheme) so `getLastActiveWorkspaceRoot` finds the workspace root regardless of connection type.
