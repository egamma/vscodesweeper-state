# Issue #116863 — SCM - Action "Move to Next Change" not working in Open Source Control tab

[https://github.com/microsoft/vscode/issues/116863](https://github.com/microsoft/vscode/issues/116863)

## Summary

Still reproduces on current main: the "Go to Next/Previous Change" actions are registered as `EditorAction` (requiring editor focus), so they silently no-op when invoked from the command palette while the SCM viewlet has focus. The milestone is "On Deck", so this stays open per scheduling rules.

## Change summary

"Go to Next Change" and "Go to Previous Change" commands do nothing when invoked from the command palette while focus is in the Source Control viewlet, because they are registered as EditorActions that require an active editor.

## Best solution

Migrate `GotoNextChangeAction` and `GotoPreviousChangeAction` (in `src/vs/workbench/contrib/scm/browser/quickDiffWidget.ts`) from `EditorAction` to a workbench-level `Action2` registration, so they can resolve the active editor from `ICodeEditorService` even when invoked without direct editor focus. The same applies to `ShowNextChangeAction` and `ShowPreviousChangeAction`.
