# Issue #205857 — Making refactors is stressful because I don't know when they will or will not show the refactor preview

[https://github.com/microsoft/vscode/issues/205857](https://github.com/microsoft/vscode/issues/205857)

## Summary

Team-authored polish request about making refactor preview availability more discoverable, especially for keyboard users. A maintainer (justschen) confirmed partial progress and is brainstorming further improvements — keeping open per team-author rule and because the UX gap for keyboard navigation remains.

## Change summary

Users cannot tell in advance whether a refactor action will show a preview or apply changes directly, especially when using keyboard navigation.

## Best solution

Surface preview availability inline in the code action widget (not just on mouse hover) so keyboard-only users know whether Cmd+Enter will offer a preview before committing to an action. Related brainstorming tracked alongside https://github.com/microsoft/vscode/issues/210171.
