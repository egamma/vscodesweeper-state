# Issue #273983 — Add setting to gradually hide or show copy button on different types of hovers

[https://github.com/microsoft/vscode/issues/273983](https://github.com/microsoft/vscode/issues/273983)

## Summary

Team-authored feature request for a setting to control hover copy button visibility. The copy button was added (PR #271990) and expanded to all hover parts (commit ab6ce975997), but no user-facing setting to hide/show it per hover type exists on current main. Keeping open per team-author policy.

## Change summary

Add a setting to let users control whether the copy button appears on different types of hovers (marker hovers, all hovers, or none).

## Best solution

Add an `editor.hover.copyButton` enum setting (e.g. `all`, `markers`, `none`) in `src/vs/editor/common/config/editorOptions.ts` and wire it through `src/vs/editor/contrib/hover/browser/contentHoverRendered.ts` to conditionally render the `HoverCopyButton`.
