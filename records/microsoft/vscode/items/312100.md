# Issue #312100 — Copilot chat crashes VS Code every 2 minutes with code 135 error

[https://github.com/microsoft/vscode/issues/312100](https://github.com/microsoft/vscode/issues/312100)

## Summary

Renderer crash (code 135) on Linux/Wayland with Intel Iris Xe GPU, where software rendering (`MESA_LOADER_DRIVER_OVERRIDE=softpipe`) works around it. The software-rendering workaround strongly implicates a GPU compositing path in Chromium/Electron or a Mesa driver regression rather than VS Code's own code. Keeping open as an upstream issue that needs triage with Electron/Chromium context.

## Change summary

VS Code renderer process crashes every ~2 minutes on Ubuntu 24.04 Wayland with Intel Iris Xe GPU, starting from version 1.115; forcing software rendering prevents the crash.

## Best solution

Investigate whether this is a known Chromium/Electron GPU compositing bug on Wayland with Intel Iris Xe (Alder Lake) and Mesa 25.x. If reproducible with a bare Electron app, file upstream against Electron/Chromium. The `--disable-gpu` flag alone didn't help, but `MESA_LOADER_DRIVER_OVERRIDE=softpipe` did — this narrows the fault to the Mesa hardware driver path. The Copilot Chat extension's high CPU usage may be a trigger (heavier rendering load) rather than the root cause. Consider adding `electron` label alongside `upstream`.
