# Issue #308101 — command-p "agent " cancels out of file search, pulls up agent commands

[https://github.com/microsoft/vscode/issues/308101](https://github.com/microsoft/vscode/issues/308101)

## Summary

Still reproduces on main: the quick access provider for agent sessions uses the prefix `'agent '` (a plain English word plus space), which collides with normal file search in Quick Open (Cmd+P) whenever a user types a multi-word query containing 'agent'. The author is a MEMBER — keeping open.

## Change summary

Typing 'agent ' in Quick Open (Go to File) hijacks file search and switches to the agent sessions picker, making it impossible to fuzzy-search for files with 'agent' in their name followed by another word.

## Best solution

Change the `AGENT_SESSIONS_QUICK_ACCESS_PREFIX` in `src/vs/workbench/contrib/chat/browser/agentSessions/agentSessionsQuickAccess.ts` to a symbol-prefixed string (e.g., `@agent ` or a dedicated sigil) that won't collide with natural-language file search queries, consistent with how other quick access providers use non-alphanumeric prefixes (`>`, `@`, `#`, `:`).
