# Issue #301760 — Autopilot approving askQuestions after being disabled

[https://github.com/microsoft/vscode/issues/301760](https://github.com/microsoft/vscode/issues/301760)

## Summary

Real bug: the askQuestions tool checks the request's original static permissionLevel rather than the live session permission level, so disabling autopilot mid-execution doesn't prevent auto-responding to subsequent askQuestions calls in the same request.

## Change summary

Disabling autopilot during an active chat session does not prevent the askQuestions tool from auto-approving questions that arrive after the toggle.

## Best solution

In `askQuestionsTool.ts` line 208, replace the static `request.modeInfo?.permissionLevel` check with the live session permission level, mirroring the pattern already used in `languageModelToolsService.ts` which combines the static check with `_isSessionLiveAutopilotLevel()`. This ensures mid-execution autopilot toggles are respected by the tool.
