# Issue #219855 — [json] allow to override & replace existing schemas

[https://github.com/microsoft/vscode/issues/219855](https://github.com/microsoft/vscode/issues/219855)

## Summary

Legitimate feature request already triaged to Backlog by a maintainer (aeschli). The user wants user-defined JSON schemas to override or replace extension-provided schemas rather than being combined additively via `allOf`. The maintainer acknowledged the current design but did not reject the request.

## Change summary

User-defined `json.schemas` entries should be able to override/replace extension-provided schemas for the same file pattern, rather than always being additive.

## Best solution

Add an option (e.g. a `replace` or `override` flag in `json.schemas` entries) that lets user-configured schemas suppress other matching schemas for the same file pattern, or define a priority model where workspace > user > extension schemas can selectively replace rather than combine.
