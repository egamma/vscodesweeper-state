# Issue #174877 — Can't keybind escape when renaming a file in explorer view

[https://github.com/microsoft/vscode/issues/174877](https://github.com/microsoft/vscode/issues/174877)

## Summary

Still reproduces on main. The explorer rename input's DOM keydown handler catches Escape (line 1137 of explorerViewer.ts) before the keybinding system processes it, so user-defined Escape keybindings with `!inputFocus` conditions fire incorrectly after the rename is cancelled and focus leaves the input. The linked PR #178639 was closed without merging. The issue is labeled `help wanted` and remains a valid bug.

## Change summary

User keybindings for Escape cannot respect `inputFocus` context during file rename in the Explorer because the input's DOM handler fires before the keybinding system.

## Best solution

The Escape handler in `renderInputBox` (explorerViewer.ts ~line 1137) should call `e.browserEvent.stopPropagation()` after calling `done(false, true)` to prevent the event from reaching the window-level keybinding listener. Alternatively, a more structural fix would route the Escape key through the keybinding system first (as discussed in comments 5–8). The `help wanted` label is appropriate — this is a good community contribution target.
