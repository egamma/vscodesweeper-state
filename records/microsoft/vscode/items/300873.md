# Issue #300873 — `Chat: Review` command silently does nothing when active file has not been pushed to remote

[https://github.com/microsoft/vscode/issues/300873](https://github.com/microsoft/vscode/issues/300873)

## Summary

Team-authored bug, kept open per policy. The silent failure is real and traceable in source: when the `chat:review` command runs with the `'selection'` group and no text selected, the `resolveSelection` method can return `undefined`, causing `review()` to silently return with zero user feedback.

## Change summary

The `chat:review` command silently does nothing when it cannot resolve a selection scope, providing no notification to the user about why the review didn't start.

## Best solution

Add a user-facing notification (e.g. via `notificationService.showInformationMessage`) in `ReviewSession.review()` at the early-return path (line 128–129 of `doReview.ts`) so users understand why the review didn't start, rather than getting silent no-ops. The fix is small and localized to `extensions/copilot/src/extension/review/node/doReview.ts`.
