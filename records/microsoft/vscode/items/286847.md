# Issue #286847 — Copilot setting focus to terminal ruins experience when terminal is also in right pane

[https://github.com/microsoft/vscode/issues/286847](https://github.com/microsoft/vscode/issues/286847)

## Summary

Still unfixed on main. The attempted PR #286862 was closed without merging. A second user confirmed the issue in the specific layout (terminal and chat as separate tabs in the same pane). A team member couldn't reproduce with a different sub-layout (split view within the pane). The bug appears layout-dependent and needs clearer reproduction steps for the exact tab arrangement.

## Change summary

When terminal and Copilot chat are tabs in the same pane, running a terminal command steals focus to the terminal and never returns it to chat.

## Best solution

The terminal tool invocation path (likely in `src/vs/workbench/contrib/terminalContrib/chat/`) should check whether the terminal is in the same view group as the active chat and either skip the focus call or use `preserveFocus: true` when revealing/running commands. The specific scenario is tabs (not split) within the same pane — reproduction requires that exact layout.
