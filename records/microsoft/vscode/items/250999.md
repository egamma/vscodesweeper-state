# Issue #250999 — Git - Unhelpful alert dialog when encountering pre-commit errors

[https://github.com/microsoft/vscode/issues/250999](https://github.com/microsoft/vscode/issues/250999)

## Summary

Still present on main: pre-commit hook failures fall through to the generic default error handler (commands.ts ~L5646) which strips context and shows only the last stderr line, with no indication that a hook caused the failure. Author is MEMBER and milestone is On Deck — keeping open.

## Change summary

When a git pre-commit hook fails, the error dialog shows a generic message with no indication that the failure came from a hook, making it hard to diagnose.

## Best solution

Add a `GitErrorCodes` entry for hook failures (detect via exit code 1 from `git commit` with stderr containing 'hook' patterns), and handle it in the error switch in `extensions/git/src/commands.ts` with a message that names the hook and suggests checking the Git output channel or using `--no-verify`.
