# Issue #295150 — Default model for compaction

[https://github.com/microsoft/vscode/issues/295150](https://github.com/microsoft/vscode/issues/295150)

## Summary

Valid feature request — compaction model selection is currently internal/experiment-gated, not user-configurable. The request for a user-facing setting to avoid burning premium model tokens on summarization is reasonable and has 6 upvotes. Keeping open; not yet at the 20-upvote backlog threshold.

## Change summary

Allow users to configure which language model is used for chat compaction/summarization, so expensive premium models aren't consumed for background summarization tasks.

## Best solution

Expose a user-facing setting (e.g. `chat.compaction.model`) that feeds into the existing `resolveCompactionEndpoint` logic in `extensions/copilot/src/extension/prompts/node/agent/compactionEndpoint.ts`. The internal `ConversationCompactionModel` experiment-based config already supports an override — surfacing it as a user setting in the Copilot extension's `package.json` contributions would address this. The commenter's concern about context-window mismatch (picking a model with a smaller window than the current chat) also needs consideration in the UX or validation logic.
