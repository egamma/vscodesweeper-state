# Issue #275761 — [Features]: Increase size of error indicator into vertical scrollbar

[https://github.com/microsoft/vscode/issues/275761](https://github.com/microsoft/vscode/issues/275761)

## Summary

Valid feature request that reached the 20-upvote threshold and is correctly in the Backlog. The overview ruler lane system currently divides decorations into lanes (left/center/right) with no user setting to force full-width rendering of error indicators.

## Change summary

Request to make error/warning indicators in the vertical scrollbar span the full scrollbar width (or be configurable) instead of occupying only a lane fraction.

## Best solution

Add a setting (e.g. `editor.overviewRulerLanes`) or adjust the lane logic in `src/vs/editor/browser/viewParts/overviewRuler/decorationsOverviewRuler.ts` so that when the scrollbar is narrow, error decorations use the full available width rather than being split into lanes. Alternatively, a minimum-pixel-width floor for decoration lanes would address the usability concern.
