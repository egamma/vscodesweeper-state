# Issue #204007 — Visual Studio Code should support 'Force Run to Cursor'

[https://github.com/microsoft/vscode/issues/204007](https://github.com/microsoft/vscode/issues/204007)

## Summary

Valid feature request from a team member for 'Force Run to Cursor' in the debugger — not yet implemented on main. Hard keep-open: author is MEMBER.

## Change summary

Add a 'Force Run to Cursor' debug command that temporarily disables all breakpoints/exception triggers, runs to the cursor, then restores them.

## Best solution

Implement as a new debug command in `src/vs/workbench/contrib/debug/` that composes existing primitives: disable all breakpoints, set a temporary breakpoint at the cursor line, continue, then re-enable breakpoints on hit. A community extension (https://github.com/volsa/vscode-inline-run-to-cursor) already demonstrates the approach; a native implementation could offer better UX with proper inline debug controls.
