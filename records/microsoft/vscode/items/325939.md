# Issue #325939 — winpty has been removed

[https://github.com/microsoft/vscode/issues/325939](https://github.com/microsoft/vscode/issues/325939)

## Summary

This is a support question, not a feature request or bug: the user cannot open the terminal on Windows 10 build 10240 after winpty was intentionally removed in VS Code 1.109. A team member has already responded with the recommended workaround (`terminal.integrated.windowsUseConptyDll`).

## Change summary

User on very old Windows 10 (build 10240) cannot open the integrated terminal because winpty support was removed.

## Best solution

The user should enable the `terminal.integrated.windowsUseConptyDll` setting as the team member already suggested; this ships a newer conpty DLL with VS Code that should work on older Windows builds. No product change is needed — the removal was intentional and the mitigation setting exists.
