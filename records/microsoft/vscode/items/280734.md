# Issue #280734 — Allow canceling fetch call on the tool call itself

[https://github.com/microsoft/vscode/issues/280734](https://github.com/microsoft/vscode/issues/280734)

## Summary

Valid feature request that two team members (TylerLeonhardt, roblourens) have endorsed as a generic improvement for all tool calls. The per-tool-call cancel UI has not been implemented on main — only request-level cancellation exists today via `cancelToolCallsForRequest`. Keeping open as a recognized backlog item.

## Change summary

Allow users to cancel an individual in-progress tool call from the chat UI (e.g. a stop button on the tool call widget) without canceling the entire turn.

## Best solution

Add a per-tool-call cancel affordance in the tool invocation progress UI (`chatToolProgressPart.ts`), likely a stop button that appears on hover over the spinner. This should be generic across all tool types, as roblourens suggested. The cancel should fire the existing `ChatToolInvocation` cancellation path (which already supports a `Cancelled` state kind) for a single tool call without aborting the whole request.
