# Issue #319291 — Changes are not immediately visible in the changes view

[https://github.com/microsoft/vscode/issues/319291](https://github.com/microsoft/vscode/issues/319291)

## Summary

Team-authored issue reporting that the Changes view defaults to 'Branch Changes' instead of 'Uncommitted Changes' after an agent generates code, requiring an extra click to see the actual output. Keeping open for maintainer judgment.

## Change summary

The Changes view defaults to showing Branch Changes instead of Uncommitted Changes after an agent generates code changes.

## Best solution

The Changes view should default to (or auto-switch to) 'Uncommitted Changes' when new uncommitted changes are generated by an agent session, so users see the relevant diff immediately without needing to switch the dropdown.
