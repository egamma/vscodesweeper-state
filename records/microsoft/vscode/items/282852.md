# Issue #282852 — rg heavy disk I/O usage, rg does not like brackets and is too greedy

[https://github.com/microsoft/vscode/issues/282852](https://github.com/microsoft/vscode/issues/282852)

## Summary

Verified duplicate of #286778, which tracks the same root cause (square brackets in folder paths cause ripgrep to search from the wrong directory) with more detailed diagnosis — including a MEMBER-identified root cause in the workspace.findFiles() API path and a workaround — and broader community engagement.

## Change summary

Square brackets in workspace folder paths cause ripgrep to interpret them as glob character classes, leading to searches from the wrong directory and excessive resource usage.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/286778 — the MEMBER diagnosis there (glob patterns passed to ripgrep don't escape brackets in the folder path, particularly via workspace.findFiles / RelativePattern) identifies the fix location. The unique evidence from this report (Samba/network drive amplifying the I/O impact, and the specific cwd observation) is carried over in the duplicate comment.
