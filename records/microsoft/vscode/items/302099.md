# Issue #302099 — Thinking collapse causes vertical jump/flicker

[https://github.com/microsoft/vscode/issues/302099](https://github.com/microsoft/vscode/issues/302099)

## Summary

Team-authored polish issue (MEMBER) about thinking-block collapse causing a vertical jump in the chat view. No fix found on current main; keeping open for maintainer judgment.

## Change summary

When a thinking block auto-collapses during streaming, empty space appears below the collapsed block, causing the chat content to visually jump downward until new text fills the gap.

## Best solution

Adjust the chat streaming/layout logic so that collapsing a thinking block does not leave empty vertical space — the subsequent content should immediately fill upward and only begin scrolling once it reaches the viewport bottom. Likely lives in the chat response rendering and scroll-anchor code under `src/vs/workbench/contrib/chat/browser/`.
