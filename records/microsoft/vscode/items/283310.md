## Summary

Team-authored issue (hediet is a MEMBER); the described behavior — inline completions appearing immediately after a Tab-to-indent — is still present on main with no fix found. Keeping open.

## Change summary

Inline completions reappear immediately after pressing Tab to indent, creating a disruptive loop where the user must dismiss completions repeatedly.

## Best solution

Add a short debounce (~1s as suggested) in the inline completions model or controller that suppresses fetching new completions when the preceding edit was a Tab-to-indent action. Likely in `src/vs/editor/contrib/inlineCompletions/browser/model/inlineCompletionsModel.ts` or the controller.
