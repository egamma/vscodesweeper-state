# Issue #201312 — Explorer: Expanding unmatched folders while filtering loses focus

[https://github.com/microsoft/vscode/issues/201312](https://github.com/microsoft/vscode/issues/201312)

## Summary

Still present on main: when Trait.onDidModelSplice falls back to getFirstViewElementWithTrait (line 2343–2351 in abstractTree.ts), it sets the focus silently via _set(nodes, true), which doesn't properly propagate DOM focus to the list widget — subsequent keyboard expand (RightArrow) fails on the visually-focused item.

## Change summary

In the Explorer's filtered tree, expanding an unmatched folder via RightArrow causes it to disappear; focus visually moves to the next item but keyboard navigation stops working on it.

## Best solution

In Trait.onDidModelSplice, when falling back to getFirstViewElementWithTrait after all trait nodes vanish, ensure the list widget's actual DOM focus is also updated (not just the silent trait set). The fix likely lives in src/vs/base/browser/ui/tree/abstractTree.ts around the fallback path at lines 2343–2351.
