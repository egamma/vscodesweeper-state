# Issue #148278 — vscode diff to allow regexp exclusions

[https://github.com/microsoft/vscode/issues/148278](https://github.com/microsoft/vscode/issues/148278)

## Summary

Legitimate feature request with 21 upvotes, already promoted to Backlog. No implementation exists on current main — the diff editor only supports `ignoreTrimWhitespace`, not regex-based line exclusions. Keeping open for the community process.

## Change summary

Allow configurable regexp patterns to exclude matching line portions from diff visualization, so reindexing noise doesn't obscure substantive changes.

## Best solution

Add a diff editor option (e.g. `diffEditor.ignorePatterns`) that accepts an array of regexps; before computing the diff, strip matching portions from both sides so they don't generate change regions. Implementation would likely touch `src/vs/editor/common/config/diffEditor.ts` (option definition) and the diff-computation pipeline. Alternatively, this could be explored as an extension via a custom content provider, but a first-class option would be more discoverable.
