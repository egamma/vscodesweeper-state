# Issue #312673 —  `chatagent` language mode reports false-positive "File not found" warnings for in-page Markdown anchor links

[https://github.com/microsoft/vscode/issues/312673](https://github.com/microsoft/vscode/issues/312673)

## Summary

Confirmed bug: the prompt file parser in `promptFileParser.ts` treats all markdown links — including fragment-only anchors like `[text](#heading)` — as file references, causing the validator to emit false 'File not found' warnings. An open external PR (#313804) already targets this fix.

## Change summary

Fragment-only Markdown anchor links (`[text](#anchor)`) in `.agent.md` files produce spurious 'File not found' diagnostics in the Problems panel.

## Best solution

Land (or iterate on) the open fix PR https://github.com/microsoft/vscode/pull/313804, which skips fragment-only links in the prompt parser/validator. The fix site is the markdown-link loop in `src/vs/workbench/contrib/chat/common/promptSyntax/promptFileParser.ts` (around line 476) — add a guard like `if (match[2].startsWith('#')) continue;` before pushing to `fileReferences`.

## Evidence

- **source trace:** In `src/vs/workbench/contrib/chat/common/promptSyntax/promptFileParser.ts`, lines 475–486: the regex `/\[(.*?)\]\((.+?)\)/g` captures all markdown link targets, including fragment-only `#heading` links, and pushes them into `fileReferences` without filtering.
- **validator emits false warning:** In `src/vs/workbench/contrib/chat/common/promptSyntax/languageProviders/promptValidator.ts`, line 181: `resolveFilePath('#details')` resolves as a relative path, the file doesn't exist, and the warning `File '#details' not found at '…/#details'` is emitted.
