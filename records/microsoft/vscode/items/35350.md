# Issue #35350 — File extension specific settings

[https://github.com/microsoft/vscode/issues/35350](https://github.com/microsoft/vscode/issues/35350)

## Summary

Valid, highly-upvoted feature request (180 👍) for file-extension/glob-based settings overrides. The configuration system on current main still only supports language-based overrides (`[language]`). No implementation exists; this remains the canonical tracking issue with multiple duplicates closed against it.

## Change summary

Allow settings overrides scoped to file extensions or glob patterns (e.g. `[*.csv]` or `[**/dist/**]`), not just language identifiers.

## Best solution

Extend the configuration override system in `src/vs/platform/configuration/common/configurationRegistry.ts` to accept glob/extension-based override keys alongside existing language overrides. This is a core platform change — the override identifier resolution, schema validation, and settings UI all need updates. Well above the 20-upvote promotion threshold; belongs in Backlog awaiting prioritization.
