# Issue #146993 — Mouse flicking back and forth causing vscode to hang

[https://github.com/microsoft/vscode/issues/146993](https://github.com/microsoft/vscode/issues/146993)

## Summary

Still-open Linux-specific bug where rapid mouse movement near the sash boundary triggers an infinite mouseenter/mouseleave feedback loop, hanging the renderer. A team member (deepak1556) investigated in 2022 but no fix was committed; the underlying sash hover/cursor interaction logic on `main` is unchanged. The `help wanted` label is appropriate — keeping open.

## Change summary

Rapidly clicking or moving the mouse near the sash/sidebar boundary on Linux causes an infinite cursor-style feedback loop that freezes all VS Code windows.

## Best solution

The likely root cause is a feedback loop in `src/vs/base/browser/ui/sash/sash.ts`: the sash hover CSS changes the element geometry, which repositions the cursor boundary, triggering new mouseenter/mouseleave events in a tight loop. A fix would debounce or coalesce the hover class toggling (lines 605–625) to break the cycle, or use a requestAnimationFrame guard on cursor-style mutations. The `help wanted` label already invites external contributions.
