# Issue #207033 — VS Code should default to Wayland when possible

[https://github.com/microsoft/vscode/issues/207033](https://github.com/microsoft/vscode/issues/207033)

## Summary

Valid, highly upvoted (222 👍) feature request that remains unimplemented on current main — VS Code still defaults to XWayland on Linux Wayland desktops. The snap launcher explicitly hardcodes `--ozone-platform=x11`, and no code sets `ELECTRON_OZONE_PLATFORM_HINT=auto` by default. This has strong community demand and should stay open.

## Change summary

VS Code should automatically use native Wayland rendering when running on a Wayland desktop, instead of requiring users to manually set flags or environment variables.

## Best solution

Set `ELECTRON_OZONE_PLATFORM_HINT=auto` (or equivalent `--ozone-platform=auto`) as the default on Linux, so VS Code uses native Wayland when available and falls back to X11 otherwise. Key locations: the snap launcher (`resources/linux/snap/electron-launch` currently hardcodes `--ozone-platform=x11`), the `.desktop` file exec line, and potentially the main electron bootstrap. This depends on Electron's Wayland stability being sufficient — the team should evaluate remaining Wayland-specific regressions (window decorations, keyboard shortcuts on non-US layouts, NVIDIA GPU issues) before flipping the default.

## Evidence

- **not implemented:** No code in the electron-main process or bootstrap sets `ELECTRON_OZONE_PLATFORM_HINT=auto` or `--ozone-platform=auto` by default. The `ozone-platform` arg is accepted but not defaulted.
- **snap hardcodes x11:** `resources/linux/snap/electron-launch` ends with `exec "$@" --ozone-platform=x11`, explicitly forcing X11 even on Wayland sessions.
- **telemetry only:** `src/vs/base/node/osDisplayProtocolInfo.ts` detects the display protocol but is only used for telemetry in `sharedProcessMain.ts`, not for setting defaults.
- **community demand:** 222 community upvotes on the issue body, with active discussion from users on multiple distros (Fedora, Ubuntu, openSUSE, Debian) all requesting this default.
- **upstream maturity:** Electron supports `ELECTRON_OZONE_PLATFORM_HINT=auto` which auto-detects Wayland vs X11, and multiple commenters confirm it works. Some edge cases remain (NVIDIA, non-US keyboards, window decorations).
