# Issue #309537 — `Failed to create worktree. Proceeding without isolation.`

[https://github.com/microsoft/vscode/issues/309537](https://github.com/microsoft/vscode/issues/309537)

## Summary

Team-authored issue, still active. The error message in `folderRepositoryManagerImpl.ts` still lacks detail on why worktree creation failed, and no retry/recovery option is offered. Multiple team members and community users have confirmed the issue.

## Change summary

When starting a new agent session with worktree isolation, worktree creation fails silently without explaining the root cause or offering recovery options.

## Best solution

Surface the underlying git error in the warning message (the catch block in `chatSessionWorktreeServiceImpl.ts` already does this on line 157, but the `undefined` return at line 155 swallows the actual reason). Offer a retry option or a "create without copying changes" fallback so users can still work in isolation.
