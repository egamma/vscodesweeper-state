[https://github.com/microsoft/vscode/issues/241548](https://github.com/microsoft/vscode/issues/241548)

## Summary

Confirmed bug still present on main: the `computeUnchangedMoves` function in `computeMovedLines.ts` lacks a lower-bound check when extending moves upward, causing an out-of-bounds array access that crashes the diff editor. PR #280535 has the fix but remains unmerged.

## Change summary

With `diffEditor.experimental.showMoves` enabled, certain file comparisons crash the diff editor due to an out-of-bounds array access in move detection.

## Best solution

Merge the open PR https://github.com/microsoft/vscode/pull/280535 which adds the missing lower-bound checks (`origLine < 1 || modLine < 1`) at lines 210 and 230 of `computeMovedLines.ts`, or land an equivalent guard.
