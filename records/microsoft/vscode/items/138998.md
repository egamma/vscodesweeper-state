# Issue #138998 — column number bug with emojis

[https://github.com/microsoft/vscode/issues/138998](https://github.com/microsoft/vscode/issues/138998)

## Summary

Keep open: current main still appears to compute the status bar column by code point, not by grapheme cluster, so emoji modifier/ZWJ sequences can still advance the reported column more than a user-perceived character. There is no duplicate, fixing commit, maintainer decision, or reproduction failure in the provided issue context to justify a close.

## Change summary

The status bar `Ln/Col` display reports unexpected column numbers after emoji sequences with modifiers or multiple code points.

## Best solution

For `workbench-status`, decide whether `CursorColumns.toStatusbarColumn` should count grapheme clusters, likely using the existing `GraphemeIterator`, while preserving tab-stop behavior; add focused tests for skin-tone and ZWJ emoji sequences in `src/vs/editor/test/common/core/cursorColumns.test.ts`.

## Evidence

- **issue context:** https://github.com/microsoft/vscode/issues/138998 is open with labels `bug` and `workbench-status`, no comments, no milestone, and reports Web/cross-platform status bar column mismatch for emoji sequences.
- **status bar path:** `src/vs/workbench/browser/parts/editor/editorStatus.ts` formats `Ln {0}, Col {1}` from `info.selections[0].positionColumn`; for a single selection it replaces that column with `editorWidget.getStatusbarColumn(editorPosition)`.
- **current implementation:** `src/vs/editor/browser/widget/codeEditor/codeEditorWidget.ts` delegates `getStatusbarColumn` to `CursorColumns.toStatusbarColumn(...)`, and `src/vs/editor/common/core/cursorColumns.ts` currently implements `toStatusbarColumn` with `strings.CodePointIterator`, not `strings.GraphemeIterator`.
- **unicode support nearby:** `src/vs/editor/common/core/cursorColumns.ts` already uses `strings.GraphemeIterator` for `visibleColumnFromColumn` and `columnFromVisibleColumn`, but not for `toStatusbarColumn`, leaving the reported status column distinct from grapheme-aware cursor-column logic.
- **tests:** `src/vs/editor/test/common/core/cursorColumns.test.ts` covers status bar columns for surrogate-pair code points such as `🎈`, but I found no focused test for skin-tone modifier or ZWJ emoji sequences matching this report.
