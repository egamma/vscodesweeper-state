# Issue #138998 — column number bug with emojis

[https://github.com/microsoft/vscode/issues/138998](https://github.com/microsoft/vscode/issues/138998)

## Summary

Still reproduces on main: `CursorColumns.toStatusbarColumn` uses `CodePointIterator` (counts code points) rather than `GraphemeIterator` (counts grapheme clusters), so multi-code-point emojis inflate the displayed column number.

## Change summary

Status bar column number is incorrect when the line contains multi-code-point emojis (ZWJ sequences, flags, skin-toned emojis).

## Best solution

Replace the `CodePointIterator` in `CursorColumns.toStatusbarColumn` with `GraphemeIterator` so that each visual grapheme cluster counts as one column, matching the approach already used in `columnFromVisibleColumn`.
