# Issue #272967 — Make behavior of "Show integrated terminal" more closely match "Create new terminal" in multi-root workspace

[https://github.com/microsoft/vscode/issues/272967](https://github.com/microsoft/vscode/issues/272967)

## Summary

The reported bug — wrong Python virtual environment activated when toggling the terminal in a multi-root workspace — is owned by the Python extension (ms-python.python), not VS Code core. The built-in terminal correctly sets the working directory to the focused file's root; venv activation logic lives entirely in the Python extension. This should be moved to the Python extension tracker.

## Change summary

In a multi-root workspace, toggling the terminal activates the Python venv from the first workspace folder instead of the one matching the focused file.

## Best solution

Move this issue to https://github.com/microsoft/vscode-python where the terminal environment activation logic is maintained. The VS Code core terminal correctly resolves the working directory; the venv activation is contributed by the Python extension via its terminal profile/environment API integration.
