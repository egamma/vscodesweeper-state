# Issue #173019 — SCM - add support for horizontal scrolling

[https://github.com/microsoft/vscode/issues/173019](https://github.com/microsoft/vscode/issues/173019)

## Summary

Valid feature request with low traction. The SCM view explicitly hardcodes `horizontalScrolling: false` across all three tree widgets (scmViewPane, scmHistoryViewPane, scmRepositoriesViewPane), overriding the user setting. This was intentionally disabled in 2019 by a maintainer, so re-enabling it requires a deliberate product decision.

## Change summary

Allow the SCM view's tree widgets to respect the `workbench.list.horizontalScrolling` user setting instead of hardcoding it to false.

## Best solution

Remove the `horizontalScrolling: false` overrides in the three SCM tree widget instantiation sites so they fall through to the user's `workbench.list.horizontalScrolling` setting. Needs a maintainer decision on whether the original 2019 rationale (commit 6906d7e3b26) still applies — the performance warning on the setting suggests it was a deliberate trade-off.
