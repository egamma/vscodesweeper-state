# Issue #194084 — `task.json` -> `options.env` not working of `${env:VAR}` expansion

[https://github.com/microsoft/vscode/issues/194084](https://github.com/microsoft/vscode/issues/194084)

## Summary

Still reproduces on main: `${env:VAR}` resolves against the OS process environment, not against `options.env` defined in tasks.json. The variable resolution in `variableResolver.ts` reads from `_envVariablesPromise` (the process env) with no awareness of task-level env overrides. This is the canonical issue (#246175 was closed as dup of this).

## Change summary

Environment variables defined in `options.env` of tasks.json cannot be referenced via `${env:VAR}` expansion in the same task's command or args.

## Best solution

The configuration resolver needs to be made aware of the task's `options.env` when resolving `${env:VAR}` references. The fix would involve passing `options.env` into the variable resolution context so that task-defined env vars are available for expansion alongside OS env vars. Likely requires changes in `terminalTaskSystem.ts` (where resolution is triggered) and potentially `variableResolver.ts` (where env lookup happens).
