# Issue #282752 — Shortcuts with dead keys broken in Linux since 1.107.0

[https://github.com/microsoft/vscode/issues/282752](https://github.com/microsoft/vscode/issues/282752)

## Summary

This is a confirmed upstream Electron bug affecting dead-key shortcuts under Wayland. The issue is correctly labeled `upstream` and `electron-39-update`. No fix has landed on current main (Electron 42.5.0), and the root cause remains in Electron's Wayland IME/composition handling. Keeping open to track upstream resolution.

## Change summary

Keyboard shortcuts using dead keys (e.g. Ctrl+Shift+backtick) are broken in Linux Wayland sessions because Electron's Wayland backend incorrectly reports composition state for dead-key keystrokes.

## Best solution

Track the upstream Electron fix (https://github.com/electron/electron/issues/46823 and https://github.com/electron/electron/issues/49806). Once Electron ships a fix for Wayland dead-key composition handling, bump the Electron dependency. A VS Code-side workaround in `src/vs/editor/browser/controller/editContext/native/nativeEditContext.ts` (around the compositionstart/compositionend listeners) may be possible but risky without understanding the full IME contract change.
