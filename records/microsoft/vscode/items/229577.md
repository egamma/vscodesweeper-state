## Summary

Confirmed upstream Chromium bug with the EditContext API — IME positioning is incorrect when the cursor is scrolled into view. Filed at Chromium and reproduced in a standalone demo outside VS Code. No VS Code-side fix is possible; awaiting upstream resolution.

IME composition editor appears at a wrong/random position when typing triggers a scroll to bring the cursor line into the viewport.
