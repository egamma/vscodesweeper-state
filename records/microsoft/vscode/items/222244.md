# Issue #222244 — Search in markdown preview panel not working in current main branch local build

[https://github.com/microsoft/vscode/issues/222244](https://github.com/microsoft/vscode/issues/222244)

## Summary

This is a known upstream Electron limitation: the `findInFrame` API on `WebFrameMain` is only available in Microsoft's custom Electron build, not in standard/OSS Electron. The issue is correctly labeled `upstream` and should remain open until the Electron PR (electron/electron#28274) lands or an alternative approach is implemented.

## Change summary

Search (Ctrl+F / Cmd+F) in webview-based panels (e.g. Markdown preview) does not work in OSS/local builds of VS Code because the custom `findInFrame` Electron API is unavailable.

## Best solution

The root fix is upstreaming the `findInFrame` / `FindInFrame` API into mainline Electron (electron/electron#28274, currently closed). Until that lands, this remains an OSS-build limitation. The issue should stay open to track the upstream dependency. An alternative approach could be implementing find via the webview's DOM (`window.find()`) as a fallback when the native API is unavailable, similar to how notebook find already works.
