# Issue #240243 — Do you trust the authors of the files in this folder? prompt blocks cmd-w from closing window

[https://github.com/microsoft/vscode/issues/240243](https://github.com/microsoft/vscode/issues/240243)

## Summary

A team member initially called this as-designed, but left it under-discussion. The later accessibility concern — that `disableCloseAction: true` on the trust dialog leaves no keyboard-only escape path — is substantive and unresolved. Keeping open for a maintainer decision on whether Escape or cmd-w should be allowed.

## Change summary

The workspace trust startup dialog blocks cmd-w (close window) and provides no keyboard dismissal (Escape is also disabled), trapping keyboard-only users.

## Best solution

Consider allowing Escape to dismiss the dialog (defaulting to restricted mode), or at minimum allowing cmd-w to close the window. The `disableCloseAction: true` in `src/vs/workbench/contrib/workspace/browser/workspace.contribution.ts` (line 479) controls this. Alternatively, make Escape equivalent to choosing 'No, I don't trust the authors' so there is always a keyboard exit path, which would address both the original report and the accessibility concern.
