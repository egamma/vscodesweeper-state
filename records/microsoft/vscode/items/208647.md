# Issue #208647 — [jsonl] Support `JSON` formatter in `JSONL`

[https://github.com/microsoft/vscode/issues/208647](https://github.com/microsoft/vscode/issues/208647)

## Summary

A maintainer (aeschli) already engaged and noted that full JSONL support in the JSON language server is non-trivial, suggesting a dedicated formatter extension as a simpler path. With only 2 upvotes after 830 days it hasn't gained community traction, but since the maintainer chose not to close it and it sits in Backlog, deferring to their judgment.

## Change summary

Request to register the built-in JSON formatter as a document formatter for JSONL files so users don't have to manually switch language modes to format.

## Best solution

A lightweight extension (community or built-in) that registers a JSONL document formatter using per-line JSON formatting. Alternatively, the vscode-json-languageservice could learn about JSONL, but aeschli noted that requires many small adoptions across all features. The extension route is the pragmatic path.
