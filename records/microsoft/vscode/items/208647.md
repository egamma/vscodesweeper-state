# Issue #208647 — [jsonl] Support `JSON` formatter in `JSONL`

[https://github.com/microsoft/vscode/issues/208647](https://github.com/microsoft/vscode/issues/208647)

## Summary

Feature request to add JSON formatting support for JSONL files remains valid and unimplemented. The JSON language features extension does not activate for the `jsonl` language ID, and a team member noted this requires non-trivial changes to the language server.

## Change summary

JSONL files should be formattable using the built-in JSON formatter without manually switching the language mode.

## Best solution

Either extend `json-language-features` activation to include `onLanguage:jsonl` with per-line formatting logic in the language server, or recommend/publish a lightweight extension that registers a document formatter for `jsonl` delegating to the JSON formatter line-by-line. The maintainer (aeschli) leaned toward the extension approach.
