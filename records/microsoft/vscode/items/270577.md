# Issue #270577 — CLI: Avoid asking the user to update server, if they specify a VS Code version in URL, in vscode.dev

[https://github.com/microsoft/vscode/issues/270577](https://github.com/microsoft/vscode/issues/270577)

## Summary

Valid bug: the CLI serve-web command still prompts users to update the server even when a specific VS Code version/commit is pinned in the URL. No fix found on main. Keeping open for the code-cli area owner.

## Change summary

When a specific VS Code commit is specified in the vscode.dev URL path, the CLI should not show an 'update server' notification since the version is intentionally pinned.

## Best solution

In the CLI serve-web update-check logic or the workbench notification path, suppress the update-available prompt when the client connected via a version-pinned URL (the commit is extracted by `get_release_from_path` in `cli/src/commands/serve_web.rs`). The server already skips periodic update checks when `args.commit_id` is set, but the version-from-URL path doesn't seem to suppress the notification to the client.
