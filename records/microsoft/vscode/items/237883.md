# Issue #237883 — extension get wrong resource in context key when right click a folder with subfolder

[https://github.com/microsoft/vscode/issues/237883](https://github.com/microsoft/vscode/issues/237883)

## Summary

Still reproduces on main. Team-authored issue (MEMBER) — keeping open. The `onContextMenu` handler in `explorerView.ts` calls `setContextKeys(stat)` where `stat.resource` is always the deepest element of the compact folder chain; it doesn't consult the compressed navigation controller's index to determine which folder was actually right-clicked, so the `resource` context key is wrong.

## Change summary

Right-clicking a parent folder in a compact-folder row sets the `resource` context key to the deepest subfolder instead of the clicked folder.

## Best solution

In `explorerView.ts` `onContextMenu`, when a compact folder is clicked by mouse, use the compressed navigation controller to resolve which folder label was clicked and pass that folder's resource to `setContextKeys` (or set `resourceContext` directly), similar to how line 665 already uses `compressedControllers[0].current.resource` for the menu arg.
