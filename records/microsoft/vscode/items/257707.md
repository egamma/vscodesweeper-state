# Issue #257707 — Tab order not working as expected in Copilot Chat for chat responses

[https://github.com/microsoft/vscode/issues/257707](https://github.com/microsoft/vscode/issues/257707)

## Summary

Confirmed accessibility regression: pressing Tab after Ctrl+Up to focus the latest chat response moves focus to the top of the chat list instead of to interactive elements within the focused message. Two team members (meganrogge, justschen) acknowledged the bug in comments, with meganrogge noting it used to work and justschen suspecting recently added tab indexes on the request bubble as the cause. No fix has landed on main.

## Change summary

Tab from a focused chat response should navigate to interactive elements within that message (references, code blocks) rather than jumping to the top of the chat list.

## Best solution

Investigate the tab index ordering in `src/vs/workbench/contrib/chat/browser/widget/chatListRenderer.ts` — specifically the `tabIndex = 0` assignments on username (line 650), footerDetailsContainer (line 676), and the request markdown part (line 3434). The recently added tabIndex on the request bubble (line 3434) likely disrupts the expected tab flow from a focused response. The fix should ensure that after Ctrl/Cmd+Up focuses a response, Tab traverses interactive children of that response before moving elsewhere.
