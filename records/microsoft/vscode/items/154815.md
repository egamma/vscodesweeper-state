# Issue #154815 — Git - VS Code does not respect the diff flag in .gitattributes

[https://github.com/microsoft/vscode/issues/154815](https://github.com/microsoft/vscode/issues/154815)

## Summary

Still reproduces on main: VS Code's SCM diff viewer never consults `.gitattributes` to check the `diff` attribute, so files marked `-diff` are still shown as text diffs instead of being treated as binary. No fix has been committed.

## Change summary

VS Code ignores the `-diff` flag in `.gitattributes`, showing plaintext diffs for files that `git diff` would treat as binary.

## Best solution

The git extension should call `git check-attr diff -- <path>` before opening a diff, and when the attribute is `unset` (i.e. `-diff`), treat the file as binary (show 'binary file differs' instead of a text diff). The likely implementation site is the diff content provider or resource opener in `extensions/git/src/`.
