# Issue #323274 — Copilot Chat: switching from Agent to Plan mode mid-conversation auto-injects \"Start implementation\" and begins executing

[https://github.com/microsoft/vscode/issues/323274](https://github.com/microsoft/vscode/issues/323274)

## Summary

Verified duplicate of #307385 — the same core problem where Plan mode auto-fires 'Start implementation' via the handoff's `send: true` flag when Autopilot is active, bypassing the user's intent to review a plan first.

## Change summary

Switching to Plan mode mid-conversation auto-injects a 'Start implementation' message and begins executing without user consent when Autopilot is enabled.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/307385 (Backlog). The fix belongs in the Copilot extension's plan agent provider (`extensions/copilot/src/extension/agents/vscode-node/planAgentProvider.ts`) or the handoff auto-send logic in `chatSuggestNextWidget.ts` — the `send: true` handoff should not auto-execute when the user explicitly chose Plan mode to review.
