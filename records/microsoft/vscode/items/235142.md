# Issue #235142 — Task command hangs and doesn't recognize completion of detached processes

[https://github.com/microsoft/vscode/issues/235142](https://github.com/microsoft/vscode/issues/235142)

## Summary

Still present on main — the task system resolves completion exclusively via `terminal.onExit`, and there's no timeout or alternative detection for commands like `docker compose up -d` that may hold briefly during container init before fully detaching. No fix has been committed.

## Change summary

Shell tasks hang indefinitely when a detached process (e.g. `docker compose up -d`) delays before fully releasing its process handle.

## Best solution

The task system in `src/vs/workbench/contrib/tasks/browser/terminalTaskSystem.ts` waits on `terminal.onExit` with no fallback. A possible fix is an optional task-level timeout or ensuring the terminal correctly signals exit once the shell command returns (the shell itself should exit promptly). The root interaction may also involve how the integrated terminal's PTY handles child-process detachment — investigating `terminal.onExit` reliability for commands that spawn and detach subprocesses would clarify whether this is a terminal-layer or task-layer issue.
