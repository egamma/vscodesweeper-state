# Issue #313072 — Agents window: "Delete..." silently fails for Claude (claude-code) sessions

[https://github.com/microsoft/vscode/issues/313072](https://github.com/microsoft/vscode/issues/313072)

## Summary

Still reproduces on main: `_deleteAgentSessions()` lacks a Claude special-case, so delete silently fails for claude-code sessions. A team member acknowledged the bug and noted an SDK limitation.

## Change summary

Deleting a Claude (claude-code) session in the Agents window silently fails — the session remains in the list with no error shown.

## Best solution

Either add a `github.copilot.claude.sessions.delete` extension command (mirroring the rename pattern at line 1930) once the SDK supports it, or hide the Delete menu item for Claude sessions as TylerLeonhardt suggested. The fix lives in `src/vs/sessions/contrib/providers/copilotChatSessions/browser/copilotChatSessionsProvider.ts` (line ~2014) and the Copilot extension's Claude session content provider.
