# Issue #322888 — Claude Agent fails with "Error during execution" when Claude Code is configured via Amazon Bedrock

[https://github.com/microsoft/vscode/issues/322888](https://github.com/microsoft/vscode/issues/322888)

## Summary

This is a support question about whether the Claude Agent in Copilot Chat supports enterprise Amazon Bedrock configurations. It does not — the Claude Agent routes model requests through the Copilot subscription proxy, not the user's own Bedrock endpoint. The user's `CLAUDE_CODE_USE_BEDROCK=1` in `~/.claude/settings.json` conflicts with this architecture, causing the execution error.

## Change summary

Claude Agent in Copilot Chat fails with 'Error during execution' when the user's Claude Code is configured to use Amazon Bedrock (`CLAUDE_CODE_USE_BEDROCK=1`).

## Best solution

Direct the user to the Claude Code / Anthropic support channels for enterprise Bedrock configurations. The VS Code Claude Agent integration explicitly sets `ANTHROPIC_BASE_URL` to its own local proxy server (line ~498 of `claudeCodeAgent.ts`), and is designed to use the Copilot subscription for model access — not user-provided Bedrock endpoints. If Bedrock support for the in-editor Claude Agent is desired, that would be a feature request to file separately.

## Evidence

- **proxy architecture:** In `extensions/copilot/src/extension/chatSessions/claude/node/claudeCodeAgent.ts` line ~497-499, the SDK is started with `ANTHROPIC_BASE_URL: http://localhost:${serverConfig.port}` and `ANTHROPIC_AUTH_TOKEN: ${serverConfig.nonce}.${this.sessionId}`, routing all model requests through the Copilot proxy — not to the user's Bedrock endpoint.
- **user confirmation:** The reporter confirmed in Comment 1 that setting `CLAUDE_CODE_USE_BEDROCK=0` resolves the issue (falls back to Copilot subscription), and that the error only occurs with `CLAUDE_CODE_USE_BEDROCK=1`.
- **error source:** The 'Error during execution' error is thrown at `extensions/copilot/src/extension/chatSessions/claude/common/claudeMessageDispatch.ts` line 793 when the SDK returns an `error_during_execution` result subtype — the SDK tried to use Bedrock (per user's `~/.claude/settings.json`) instead of the Copilot proxy and failed.
- **env override gap:** The `settings.env` passed to the SDK does not explicitly set `CLAUDE_CODE_USE_BEDROCK=0` to override the user's settings file, so the SDK picks up `CLAUDE_CODE_USE_BEDROCK=1` from `~/.claude/settings.json` and bypasses the proxy.
