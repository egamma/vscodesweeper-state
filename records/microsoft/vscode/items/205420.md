# Issue #205420 — Side Scrollbar does not match scroll position in document

[https://github.com/microsoft/vscode/issues/205420](https://github.com/microsoft/vscode/issues/205420)

## Summary

Still present on main: the diff editor's overview ruler zone positioning logic has not been fixed since the report. The overview ruler uses `heightInLines` to avoid view-zone inflation of individual zone heights, but the overall mapping from scroll position to ruler coordinates can drift on large files where alignment view zones significantly expand the scroll height.

## Change summary

Diff editor overview ruler markers (colored decorations) appear at incorrect vertical positions relative to the actual scroll location of the diff changes.

## Best solution

Fix the zone positioning in `src/vs/editor/browser/widget/diffEditor/features/overviewRulerFeature.ts` to correctly account for the relationship between view-zone-inflated scroll height and the overview ruler canvas, ensuring markers align with the viewport indicator (`diffViewport` element) position.
