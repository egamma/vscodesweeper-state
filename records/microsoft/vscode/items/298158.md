# Issue #298158 — Cannot drag and drop folders into prompt area in welcome page

[https://github.com/microsoft/vscode/issues/298158](https://github.com/microsoft/vscode/issues/298158)

## Summary

Team-authored issue (MEMBER) — must keep open. The welcome page's agent sessions chat widget does not wire up the ChatDragAndDrop handler, so folder drops fall through to the default browser/OS behavior (opening the folder) instead of attaching it as context to the prompt.

## Change summary

Dragging and dropping a folder into the chat prompt on the welcome page opens the folder instead of attaching it as chat context.

## Best solution

The welcome page's chat widget in `src/vs/workbench/contrib/welcomeAgentSessions/browser/agentSessionsWelcome.ts` needs to instantiate or wire the `ChatDragAndDrop` handler (from `src/vs/workbench/contrib/chat/browser/widget/chatDragAndDrop.ts`) on its chat input area, mirroring how the regular panel chat widget sets up drag-and-drop. This ensures folder drops are intercepted and converted to attachment context rather than falling through to the OS open-folder handler.
