# Issue #280623 — Github MCP server: Changing settings does not directly update tools

[https://github.com/microsoft/vscode/issues/280623](https://github.com/microsoft/vscode/issues/280623)

## Summary

Team-authored issue reporting a UX glitch in the GitHub MCP server tool configuration flow — tools list doesn't update immediately when settings change. Keeping open per team-author policy; a team member has acknowledged the 'Update Tools' placement bug and explained the lazy-start design.

## Change summary

Changing GitHub MCP server settings (e.g. enabling lockdown mode) does not immediately refresh the tools list in the Configure Tools picker, showing stale tools with an 'Update Tools' entry inline.

## Best solution

Either eagerly refresh the tool list when the MCP server settings change (if it can be done without startup cost), or at minimum hide stale tools and show only the 'Update Tools' action until the server is restarted — the inline placement bug connor4312 mentioned should also be fixed.
