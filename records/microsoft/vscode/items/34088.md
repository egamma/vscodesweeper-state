# Issue #34088 — Enable fuzzy matching for picker

[https://github.com/microsoft/vscode/issues/34088](https://github.com/microsoft/vscode/issues/34088)

## Summary

Still a valid gap: the quick pick's 'fuzzy' mode uses regex-based contiguous matching (`matchesFuzzy`), not true non-contiguous fuzzy scoring (`fuzzyScore`). The extension API does not expose `matchOnLabelMode` at all, so extensions cannot opt into even the existing behavior. Keeping open — well above the upvote threshold and the capability remains undelivered.

## Change summary

Enable true fuzzy matching (non-contiguous character matching) in the quick pick control, especially for extension-contributed pickers.

## Best solution

Expose a `matchOnLabelMode` option (or equivalent) in the public `QuickPick` extension API, and wire it to use `fuzzyScore`/`matchesFuzzy2` (the real non-contiguous scorer in `src/vs/base/common/filters.ts`) rather than the current `matchesFuzzy` regex match. This would let extensions opt into true fuzzy filtering. The internal quick pick list filter logic in `src/vs/platform/quickinput/browser/quickInputList.ts` already branches on `matchOnLabelMode` — the remaining work is switching the 'fuzzy' branch to use `fuzzyScore` and finalizing the API surface.

## Evidence

- **matchesFuzzy is not truly fuzzy:** src/vs/base/common/filters.ts line 459-474: `matchesFuzzy` uses `convertSimple2RegExpPattern` (line 190-192 of strings.ts) which only escapes special chars, producing a contiguous substring match — typing 'he_' searches for literal 'he_', not h…e…_.
