# Issue #137427 — Process explorer: update name for service worker process

[https://github.com/microsoft/vscode/issues/137427](https://github.com/microsoft/vscode/issues/137427)

## Summary

Team-authored issue (MEMBER) that remains unresolved — the upstream Electron/Chromium plumbing to pass `--renderer-type=service-worker` to worker renderer processes has not landed, and the process explorer still cannot distinguish service worker processes from workbench windows. Keeping open.

## Change summary

Service worker renderer processes spawned by webviews appear as unnamed 'window' entries in the Process Explorer, indistinguishable from real workbench windows.

## Best solution

The fix requires upstream Electron work: Chromium does not currently differentiate service-worker and shared-worker renderer processes via command-line arguments. Once Electron exposes something like `--renderer-type=service-worker`, the process explorer control (`src/vs/workbench/contrib/processExplorer/electron-browser/processExplorerControl.ts`) and the process-tree parser (`src/vs/base/node/ps.ts`) can use that argument to display a proper label.
