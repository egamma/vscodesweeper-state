# Issue #303654 — Repeated git errors

[https://github.com/microsoft/vscode/issues/303654](https://github.com/microsoft/vscode/issues/303654)

## Summary

Team-authored issue (hard keep-open). The git extension already has retry logic for CantLockRef errors during fetch (up to 10 retries with quadratic backoff), but the reporter still sees repeated failures — likely a race condition when multiple VS Code windows auto-fetch the same repo concurrently, or stale refs after sleep/resume.

## Change summary

Repeated 'cannot lock ref' / 'unable to update local ref' errors during git fetch across multiple VS Code windows.

## Best solution

Investigate whether the auto-fetch mechanism in `extensions/git/src/autofetch.ts` should coordinate across windows to avoid concurrent fetches on the same repository, or whether the retry logic in `extensions/git/src/repository.ts` (line ~2826) should run `git remote prune origin` or `git gc --prune=now` as a recovery step when CantLockRef persists after retries. The multi-window scenario with background auto-fetch is the likely trigger.
