# Issue #256761 — Install button should be first tab stop on extension page

[https://github.com/microsoft/vscode/issues/256761](https://github.com/microsoft/vscode/issues/256761)

## Summary

Team-authored accessibility bug that remains unfixed on main. The related PR #257353 was closed without merging, and no other fix has landed — the Install button is still not the first tab stop on the extension page.

## Change summary

The Install button on extension pages should be the first tab stop for keyboard/screen-reader users but currently is not.

## Best solution

Modify the action bar focus logic in `src/vs/workbench/contrib/extensions/browser/extensionEditor.ts` so that when `setFocusable(true)` is called, the Install action gets `tabIndex=0` preferentially over other enabled actions. The `focusOnlyEnabledItems` option and the existing enablement-change listener already provide the right hook points.
