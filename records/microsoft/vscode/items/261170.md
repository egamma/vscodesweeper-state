# Issue #261170 — Option to disable search highlighting from updating PRIMARY selection (Linux/X11 & Wayland)

[https://github.com/microsoft/vscode/issues/261170](https://github.com/microsoft/vscode/issues/261170)

## Summary

Verified duplicate of #130208, which tracks the same defect — Ctrl+F search highlighting overwrites the X11/Wayland PRIMARY selection buffer. The reporter themselves linked #130208 as a related issue.

## Change summary

On Linux, Find (Ctrl+F) highlighting a match updates the PRIMARY selection clipboard, overwriting whatever the user previously selected in another application.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/130208. The fix would be in `src/vs/workbench/contrib/codeEditor/electron-browser/selectionClipboard.ts`: the `onDidChangeCursorSelection` handler should filter out selection changes whose `source` originates from the find controller, so that programmatic find-match selections do not write to the PRIMARY clipboard.
