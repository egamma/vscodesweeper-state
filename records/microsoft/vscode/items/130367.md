# Issue #130367 — Enable consuming of ES modules in extensions

[https://github.com/microsoft/vscode/issues/130367](https://github.com/microsoft/vscode/issues/130367)

## Summary

Implemented on main for the Node extension host, with concrete commits and code paths showing ESM extension entry points are detected and loaded via dynamic import. The remaining web-worker limitation is explicit and was handled by hybrid-extension support, so the original desktop extension-host request can be closed as completed.

## Change summary

Allow VS Code extensions to consume ES modules instead of requiring CommonJS-only entry points and dependencies.

## Best solution

Treat the Node extension-host ESM support as shipped in commit 15260e72ae15426bf6093b1ad97e7f29bbc2397e and follow-up hybrid support in ea0483469b9f86986f35af2d436af65f37d969b9; track any separate web-worker ESM support request independently.

  > Thanks for the report and the follow-up context. This is now implemented for the Node extension host in commit 15260e72ae15426bf6093b1ad97e7f29bbc2397e via https://github.com/microsoft/vscode/pull/245214 and is contained in 1.100.0 and later. ## Evidence

- **implementation commit:** Commit 15260e72ae15426bf6093b1ad97e7f29bbc2397e, PR https://github.com/microsoft/vscode/pull/245214, is titled "ESM for extensions" and includes "allow import('vscode') from extensions" plus "support esm for extension tests".
- **current code:** src/vs/workbench/api/common/extHostExtensionService.ts detects ESM entry points with .mjs or package type "module"; src/vs/workbench/api/node/extHostExtensionService.ts loads ESM extension modules through import(module.toString(true)).
- **hybrid follow-up:** Commit ea0483469b9f86986f35af2d436af65f37d969b9, PR https://github.com/microsoft/vscode/pull/248286, says it enables extensions to use ESM for the NodeJS extension host and CJS for the web worker extension host, referencing https://github.com/microsoft/vscode/issues/130367.
- **release containment:** git tag --contains 15260e72ae15426bf6093b1ad97e7f29bbc2397e shows the implementation is contained in tag 1.100.0 and later.
- **remaining limitation:** src/vs/workbench/api/worker/extHostExtensionService.ts still throws "ESM modules are not supported in the web worker extension host", so any web-worker-only ESM request should be tracked separately.
