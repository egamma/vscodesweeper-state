# Issue #321274 — Staged Changes/Files don't allow "Go to Definition" [Working Tree Diff View]

[https://github.com/microsoft/vscode/issues/321274](https://github.com/microsoft/vscode/issues/321274)

## Summary

Verified duplicate: this is the same root-cause bug as #324356 — the `git` scheme is in `disabledSchemes` in the TypeScript language features extension, which disables Go to Definition, Find References, and other language features for staged files in the SCM diff view. #324356 has identified the cause and proposed a fix.

## Change summary

Go to Definition and other language features do not work on staged files in the SCM diff view because the `git` URI scheme is explicitly disabled in the TypeScript language features extension.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/324356 which already identifies the root cause (`git` in `disabledSchemes` at `extensions/typescript-language-features/src/configuration/fileSchemes.ts:50`) and proposes a concrete fix with a diff. Implementing that fix (unwrapping git URIs to their underlying file paths in `toTsFilePath`) would resolve both issues.
