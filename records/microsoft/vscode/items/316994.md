# Issue #316994 — Copilot Chat title rename reverts

[https://github.com/microsoft/vscode/issues/316994](https://github.com/microsoft/vscode/issues/316994)

## Summary

Plausible persistence bug: `ChatSessionStore.setSessionTitle()` updates the in-memory index but does not call `flushIndex()`, so the title change may not survive if the async `storeSessions()` write races with session unloading. A fix commit (e2f1c08d034) that added the missing `flushIndex()` exists but is NOT on main, suggesting it was lost or never landed. Routing to the chat area for investigation.

## Change summary

Renaming a Copilot Chat conversation via the context menu succeeds in the active session but the custom title reverts to the auto-generated one after navigating away and returning.

## Best solution

Land the missing `flushIndex()` call in `ChatSessionStore.setSessionTitle()` (as attempted in e2f1c08d034) and verify the async `storeSessions()` write in `saveState()` completes before the session model is unloaded. The fix is small and localized to `src/vs/workbench/contrib/chat/common/model/chatSessionStore.ts`.
