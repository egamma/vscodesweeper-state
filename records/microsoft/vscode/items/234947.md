# Issue #234947 — Checked Render whitespace is not shown when the Vscode is initialised at start

[https://github.com/microsoft/vscode/issues/234947](https://github.com/microsoft/vscode/issues/234947)

## Summary

Fixed on main by commit ea8cfb10157 (PR #306483), which replaced the misleading binary toggle with a submenu showing all render-whitespace modes, each correctly reflecting the current setting value. The old toggle showed 'checked' whenever the setting was not 'none', even if the default 'selection' meant no visible whitespace in many scenarios.

## Change summary

The View > Appearance > Render Whitespace menu item appeared checked on startup even when whitespace was not visibly rendered, because the toggle condition was 'not none' while the default setting is 'selection'.

## Best solution

Keep the shipped fix (commit ea8cfb10157). The submenu now correctly reflects each whitespace mode. This issue and its duplicate #305883 are resolved.
