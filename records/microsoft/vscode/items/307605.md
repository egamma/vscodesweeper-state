# Issue #307605 — # reference picker should prioritize the active document at the top of the suggestion list

[https://github.com/microsoft/vscode/issues/307605](https://github.com/microsoft/vscode/issues/307605)

## Summary

Confirmed regression: the active file ranks low in the `#` picker because commit 6dffc1fcfae changed `filterText` to include the full URI path, diluting the score relative to shorter tool/command names. On Deck milestone and an open community PR both block a close.

## Change summary

The active document appears near the bottom of the `#` reference suggestion list in Copilot Chat instead of at the top.

## Best solution

Review and land (or supersede) the open community PR https://github.com/microsoft/vscode/pull/308742, ensuring the active file's `sortText`/`filterText` keeps it above other completions when no filter pattern is typed. The regression lives in `chatInputCompletions.ts` around the `makeCompletionItem` helper.
