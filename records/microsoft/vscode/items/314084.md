# Issue #314084 — Multiline line commit messages

[https://github.com/microsoft/vscode/issues/314084](https://github.com/microsoft/vscode/issues/314084)

## Summary

VS Code faithfully passes the commit message text as-is to git via `--file -`. GitHub Desktop has a separate summary/description UI that auto-inserts a blank line between them (git convention). VS Code's single input box gives full control over formatting — this is by design, not a bug.

## Change summary

User wants VS Code to automatically separate commit subject and body lines like GitHub Desktop does.

## Best solution

This is as-designed: VS Code's SCM input box passes the message verbatim to git, giving the user full control. To get a separate subject/body in git, the user should insert a blank line between them (per git convention). GitHub Desktop's two-field UI is a design choice specific to that application.

- **Proposed comment:**

  > Thanks for the report. This is working as designed — VS Code's commit input passes your message to git exactly as you type it. The difference you're seeing comes from how GitHub Desktop structures its UI: it has separate "summary" and "description" fields and automatically inserts a blank line between them (which is what git uses to separate subject from body).
  > 
  > In VS Code, if you want a proper subject/body separation (so both apps display the commit identically), insert a **blank line** between your main message and the description:
  > 
  > ```
  > Main line
  > 
  > Sub line
  > ```
  > 
  > This follows the standard git commit message convention. Without that blank line, git treats both lines as a single paragraph, which is why GitHub Desktop displays them joined.
