# Issue #306539 — Copilot CLI: Make git errors more actionable. Currently just says "Failed to execute git"

[https://github.com/microsoft/vscode/issues/306539](https://github.com/microsoft/vscode/issues/306539)

## Summary

Team-authored polish request. The Copilot extension's `gitServiceImpl.ts` exec method already includes the specific git command and stderr in its error messages on current main. However, callers may still swallow details, and the built-in git extension's `git.ts` still uses a generic 'Failed to execute git' message. The request for more actionable errors remains valid as an improvement across the git error surfacing pipeline.

## Change summary

Git errors in the Copilot CLI show a generic 'Failed to execute git' without the specific command or stderr, making debugging difficult for users.

## Best solution

Audit the full error propagation chain from `GitServiceImpl.exec` through callers in the Copilot CLI agent to ensure the detailed error message (which already includes the git command and stderr) is surfaced to the user. Also consider improving the built-in git extension's `git.ts` (line 663) to include the command and stderr in the `GitError` message string, not just as structured fields.
