# Issue #249624 — [json] JSON Schema anyOf error reporting is misleading

[https://github.com/microsoft/vscode/issues/249624](https://github.com/microsoft/vscode/issues/249624)

## Summary

Already triaged and accepted as a feature request with `help wanted`. The JSON validator intentionally picks the best-matching alternative in `anyOf` when all fail equally, but this can mislead users. No action needed beyond keeping it open for a contributor.

## Change summary

When all `anyOf` alternatives fail equally, VS Code highlights only the first property's error rather than indicating the constraint applies to all alternatives.

## Best solution

Improve the `anyOf` error-reporting heuristic in the JSON language service (likely in `vscode-json-languageservice`) to surface a summary diagnostic spanning the validated object when all alternatives match equally, rather than arbitrarily picking one. The `help wanted` label invites an external contributor.
