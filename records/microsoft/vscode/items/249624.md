# Issue #249624 — [json] JSON Schema anyOf error reporting is misleading

[https://github.com/microsoft/vscode/issues/249624](https://github.com/microsoft/vscode/issues/249624)

## Summary

Already triaged by a team member (aeschli) and placed in Backlog with `help wanted`. The request for improved `anyOf` error reporting is valid and awaiting a contributor.

## Change summary

When all `anyOf` alternatives fail equally, VS Code arbitrarily picks one and reports only its error, misleading users about which property actually needs fixing.

## Best solution

Improve the `anyOf` matching heuristic in the JSON language service (likely in `vscode-json-languageservice`) to surface errors from all equally-matched alternatives, or at minimum indicate that multiple alternatives failed. The `help wanted` label is appropriate — this is a good external contribution target.
