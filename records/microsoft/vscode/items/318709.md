# Issue #318709 — Terminal local file links are mis-tokenized around CJK punctuation

[https://github.com/microsoft/vscode/issues/318709](https://github.com/microsoft/vscode/issues/318709)

## Summary

Confirmed source-reproducible on main: the local file link regex in terminalLinkParsing.ts does not exclude CJK sentence/separator punctuation from path characters, causing paths adjacent to these characters to be mis-tokenized. The reporter offers to contribute a PR.

## Change summary

Terminal local file links fail to resolve when paths are adjacent to CJK sentence/separator punctuation (，。：；！？、) because those characters are not excluded from the path-matching regex.

## Best solution

Add CJK sentence/separator punctuation (U+3001、U+3002、U+FF01、U+FF0C、U+FF1A、U+FF1B、U+FF1F) to the excluded character classes in `RegexPathConstants.ExcludedPathCharactersClause` and `WinExcludedPathCharactersClause` in `src/vs/workbench/contrib/terminalContrib/links/browser/terminalLinkParsing.ts`. The reporter has offered a PR — accepting that contribution is the best path.
