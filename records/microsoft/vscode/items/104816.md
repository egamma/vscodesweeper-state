# Issue #104816 — Not able to turn Settings Sync on after changing github password

[https://github.com/microsoft/vscode/issues/104816](https://github.com/microsoft/vscode/issues/104816)

## Summary

The core complaint — no graceful recovery when a GitHub token becomes invalid — is acknowledged by a team member (TylerLeonhardt, June 2024) as still relevant for token revocation scenarios. While VS Code now offers a manual Sign Out action, the automatic detection and re-auth prompt on 401 is still missing. Keeping open per team acknowledgement.

## Change summary

After changing a GitHub password or revoking OAuth permissions, Settings Sync gives a 401 error with no way to re-authenticate without manually signing out first.

## Best solution

The github-authentication extension should detect 401/invalid-token responses and automatically prompt the user to re-authenticate or at minimum surface a clear 'Sign out and sign in again' action, rather than silently failing. The sign-out capability now exists in `src/vs/workbench/contrib/authentication/browser/actions/signOutOfAccountAction.ts` but the graceful error-recovery path is missing in the token validation flow.
