# Issue #137013 — Support MS Edge's "edge-enhance-text-contrast" flag

[https://github.com/microsoft/vscode/issues/137013](https://github.com/microsoft/vscode/issues/137013)

## Summary

Legitimate, well-supported feature request (84 upvotes) in the Backlog. Despite VS Code now shipping Electron 42 (Chromium well past 124), the last commenter confirmed in July 2024 that the ClearType/font-contrast improvements were still not present — likely because VS Code's own GPU glyph rasterizer bypasses Chromium's native text rendering pipeline. The gap remains real.

## Change summary

Request to incorporate Microsoft Edge's enhanced text contrast / ClearType font rendering improvements into VS Code, particularly for non-HiDPI Windows displays.

## Best solution

Investigate whether VS Code's GPU glyph rasterizer (`src/vs/editor/browser/gpu/raster/glyphRasterizer.ts`) can respect Windows ClearType tuning the way Chromium 124+ does natively. The editor's canvas-based rasterization likely bypasses Chromium's Skia improvements; the fix may involve reading the system ClearType settings and applying them in the glyph rasterizer's subpixel anti-aliasing path. For non-editor UI (menus, sidebars, etc.), confirming that the Chromium-level improvements are active with the current Electron version would also help.
