# Issue #312848 — Extension Signature Verification Failed: SSH FS

[https://github.com/microsoft/vscode/issues/312848](https://github.com/microsoft/vscode/issues/312848)

## Summary

The user is running a community-packaged Code - OSS build (Void Linux xbps package, version 1.100.3) that does not bundle the proprietary `@vscode/vsce-sign` module. This causes signature verification to fail when installing extensions. This is a distro-packaging issue, not a VS Code core bug — the official Microsoft build includes the module. However, a team member has flagged it as a potential packaging concern, so requesting more info and updated reproduction on a current build is appropriate.

## Change summary

Extension installation fails with 'Signature verification was not executed' on a community-packaged Code - OSS build that lacks the `@vscode/vsce-sign` module.

## Best solution

The user should either use the official Microsoft VS Code build (which bundles `@vscode/vsce-sign`), or the Void Linux package maintainer should configure the build to disable signature verification for OSS builds that cannot include the proprietary signing module. If the VS Code team wants OSS builds to gracefully degrade when `@vscode/vsce-sign` is missing (i.e., skip verification instead of erroring), that would be a separate enhancement in `src/vs/platform/extensionManagement/node/extensionManagementService.ts`.
