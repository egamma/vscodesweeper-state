# Issue #219138 — Accessibility:  "Dark high contrast theme" cursor disappears and stops blinking after a time

[https://github.com/microsoft/vscode/issues/219138](https://github.com/microsoft/vscode/issues/219138)

## Summary

Valid accessibility feature request to default cursor style to 'block' in high contrast themes. Team members engaged and acknowledged it makes sense but kept it open for broader feedback. No implementation has landed on main.

## Change summary

Make the editor cursor style default to 'block' (or thicker) in high contrast themes for better visibility.

## Best solution

Implement theme-aware default settings so `editor.cursorStyle` defaults to `block` in high contrast themes, likely in `src/vs/editor/common/config/editorOptions.ts` or via a configuration-defaults overlay. The concept of 'default setting values based on theme' (noted by isidorn) would need to be introduced or approximated.
