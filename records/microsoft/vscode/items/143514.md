# Issue #143514 — Workspace root folder gets unnecessary nested in Explorer after VSC being restarted

[https://github.com/microsoft/vscode/issues/143514](https://github.com/microsoft/vscode/issues/143514)

## Summary

This is by design: a `.code-workspace` file opens with `WorkbenchState.WORKSPACE`, which always shows root folder names in the Explorer (even with a single folder), because workspaces can contain multiple roots. The initial session after 'Save Workspace As' briefly keeps the old FOLDER state, but on restart the correct WORKSPACE state applies.

## Change summary

Explorer hides the single root folder right after saving a workspace file, but shows it (nested) after restarting and reopening the same workspace.

## Best solution

Keep the current behavior — `.code-workspace` files intentionally show root folder names in the Explorer because they support multiple roots. The transient difference right after 'Save Workspace As' is a minor state-transition artifact, but the post-restart behavior is the correct steady state.

## Evidence
