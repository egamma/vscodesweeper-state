# Issue #309282 — Copy Line Up/Down commands do not preserve more than two multi-cursors on the same line

[https://github.com/microsoft/vscode/issues/309282](https://github.com/microsoft/vscode/issues/309282)

## Summary

Still reproduces on main: the deduplication logic in AbstractCopyLinesAction marks extra same-line cursors as 'ignored' (noop), but multiple noop commands targeting the same line's newline produce conflicting edits that lose cursors. An open PR (#312513) addresses this but has not merged.

## Change summary

Copy Line Up/Down drops all but two multi-cursors when three or more cursors share the same line.

## Best solution

Merge or iterate on https://github.com/microsoft/vscode/pull/312513, which preserves all multi-cursors through the copy-lines operation. The fix lives in `src/vs/editor/contrib/linesOperations/browser/linesOperations.ts` (the dedup loop in `AbstractCopyLinesAction.run`).
