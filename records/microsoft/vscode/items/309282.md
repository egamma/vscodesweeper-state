# Issue #309282 — Copy Line Up/Down commands do not preserve more than two multi-cursors on the same line

[https://github.com/microsoft/vscode/issues/309282](https://github.com/microsoft/vscode/issues/309282)

## Summary

Keep open as a likely still-valid editor multicursor bug. Current main still suppresses same-line copy-line commands after the winning selection, and I found no targeted fix or regression test for https://github.com/microsoft/vscode/issues/309282.

## Change summary

Copy Line Up/Down should preserve all cursors when three or more empty selections are on the same source line.

## Best solution

Fix the copy-lines multicursor path in `src/vs/editor/contrib/linesOperations/browser/linesOperations.ts` / `copyLinesCommand.ts` so the line text is inserted once while every same-line selection gets a correct tracked cursor state, then add regression coverage for three same-line cursors for both copy up and copy down.
