# Issue #175959 — Allow to fork utility process from a worker thread

[https://github.com/microsoft/vscode/issues/175959](https://github.com/microsoft/vscode/issues/175959)

## Summary

Team-authored upstream feature request for an Electron-level capability (forking utility processes from a worker thread). The original artificial delay in sharedProcessService.ts has been removed, but the underlying Electron limitation — synchronous main-process blocking during utility-process fork — remains unaddressed. Keeping open per team-author rule and because the upstream gap is still real.

## Change summary

Allow forking Electron utility processes from a Node.js worker thread to avoid blocking the main process during process spawning.

## Best solution

This depends on Electron exposing a worker-thread-compatible utility-process fork API. Track upstream progress in Electron and revisit when that capability lands. The original workaround (artificial spawn delay) appears to have been removed, so the practical impact may have shifted — a team member should reassess whether this is still a priority.
