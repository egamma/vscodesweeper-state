# Issue #209427 — Explorer range selection (shift + arrow) doesn't include focused item

[https://github.com/microsoft/vscode/issues/209427](https://github.com/microsoft/vscode/issues/209427)

## Summary

A team member confirmed this is by design due to disjoint focus and selection in VS Code's list/tree widgets. The request is to unify focus and selection for Shift+Arrow range selection to match platform conventions (Windows Explorer, Visual Studio). Keeping open as under-discussion since the member acknowledged exploring re-unification but noted it's a large undertaking.

## Change summary

Shift+Arrow range selection in the Explorer should include the initially focused item, matching platform-native behavior in Windows Explorer and Visual Studio.

## Best solution

The fix lives in `src/vs/base/browser/ui/list/listWidget.ts` in the `KeyboardController` class — specifically `onUpArrow`/`onDownArrow` (lines ~363–381), which call `focusPrevious`/`focusNext` and set the anchor but never seed the selection with the currently focused item when Shift is held. However, as the member noted, re-unifying focus and selection is a large cross-cutting change. A scoped approach might add a Shift check in the keyboard handler to include the anchor in the selection range, but the broader architectural implications need maintainer judgment.
