# Issue #299231 — Unable to open a specific conversation after clicking

[https://github.com/microsoft/vscode/issues/299231](https://github.com/microsoft/vscode/issues/299231)

## Summary

Valid, confirmed bug still present on main: when session data files (.json/.jsonl) go missing from workspace storage while the session index metadata persists, clicking the orphaned session in the Sessions panel silently fails and creates a new session instead of surfacing an error or cleaning up the stale entry.

## Change summary

Clicking a previous Copilot Chat session in the Sessions panel does nothing when the backing session files are missing from workspace storage, despite the session still appearing in the list.

## Best solution

Two-part fix in the chat session store and sessions panel: (1) when `readSession` returns undefined for a locally-indexed session, surface a user-visible notification that the session data is unavailable rather than silently creating a new session, and (2) consider removing or marking orphaned entries in the session index so they don't mislead users. The relevant code paths are in `chatSessionStore.ts` (readSessionFromLocation) and `chatServiceImpl.ts` (acquireOrRestoreLocalSession returning undefined).
