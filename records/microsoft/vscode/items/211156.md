# Issue #211156 — Use different templates for issue reporting vs performance vs features

[https://github.com/microsoft/vscode/issues/211156](https://github.com/microsoft/vscode/issues/211156)

## Summary

Team-authored feature request for per-issue-type templates in the issue reporter API. The API still only has a single `extensionTemplate` field — no per-type differentiation has been added. Keeping open as a team-member issue.

## Change summary

Allow extensions to provide different issue reporter templates for bugs, performance issues, and feature requests instead of a single shared template.

## Best solution

Extend `IssueReporterExtensionData` (in `src/vs/workbench/contrib/issue/common/issue.ts`) to accept per-`IssueType` templates (e.g. `extensionTemplates?: Partial<Record<IssueType, string>>`) and wire the issue reporter UI to select the matching template when the user picks an issue type. The corresponding extension API surface would need a similar update.
