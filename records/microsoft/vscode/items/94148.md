# Issue #94148 — Investigate setting Electron's proxy from user setting

[https://github.com/microsoft/vscode/issues/94148](https://github.com/microsoft/vscode/issues/94148)

## Summary

Fully implemented and shipped since 1.45.0. The `http.proxy` setting now configures Electron's session proxy via `session.setProxy()`, and the code remains active on current main with subsequent improvements (noProxy support, app-level proxy). The investigation is complete.

## Change summary

VS Code's `http.proxy` user setting was not applied to Electron's network layer, so extension marketplace browsing and updates ignored it.

## Best solution

Keep the shipped implementation (commit 1e960d5abb5 and follow-ups). The documentation gap noted in the last comment is tracked separately by https://github.com/microsoft/vscode/issues/121224.
