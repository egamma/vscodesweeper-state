# Issue #117327 — An extension spamming console.log will take a lot of CPU time in the render process

[https://github.com/microsoft/vscode/issues/117327](https://github.com/microsoft/vscode/issues/117327)

## Summary

Keep open: this is a team-authored issue, which the sweeper should not close, and the discussion identifies a real performance protection gap around excessive extension console output reaching the renderer.

## Change summary

An extension that writes verbose LSP logs through console.log can make the VS Code renderer consume high CPU and become slow or unresponsive.

## Best solution

Core should either keep tracking this under extension-host as renderer protection against abusive console traffic, or explicitly decide that extensions must avoid console logging and use OutputChannel instead.
