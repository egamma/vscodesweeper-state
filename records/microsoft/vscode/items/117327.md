# Issue #117327 — An extension spamming console.log will take a lot of CPU time in the render process

[https://github.com/microsoft/vscode/issues/117327](https://github.com/microsoft/vscode/issues/117327)

## Summary

Team-authored issue (hard keep-open). The protective throttling assigned to @alexdima was never implemented — current main still forwards every console.log from extensions to the renderer without any rate limiting.

## Change summary

An extension spamming console.log saturates the renderer's RPC channel and causes UI freezes because there is no throttling in the extension-host console forwarding path.

## Best solution

Add rate-limiting or batching in `AbstractExtHostConsoleForwarder._handleConsoleCall` (src/vs/workbench/api/common/extHostConsoleForwarder.ts) so that bursts of console messages are aggregated or dropped after a threshold, protecting the renderer from excessive IPC traffic regardless of extension behavior.
