# Issue #234754 — Empty code coverage (vscode development)

[https://github.com/microsoft/vscode/issues/234754](https://github.com/microsoft/vscode/issues/234754)

## Summary

Still valid on main: the coverage infrastructure (`test/unit/coverage.js`) uses an AMD loader `nodeInstrumenter` hook that is never invoked since the test runner migrated to ESM `import()`. The `--coverage` flag is parsed and `createReport` is called, but no instrumentation happens, so reports are empty.

## Change summary

Running `./scripts/test.sh --coverage` produces an empty coverage report because the Istanbul instrumentation hook is never called.

## Best solution

Either remove the outdated coverage instructions and `coverage.js` module (if coverage is no longer supported this way), or reimplement instrumentation for the ESM-based test runner — likely via V8's built-in coverage (`NODE_V8_COVERAGE`) or Electron's `Profiler.takePreciseCoverage` (which the per-test-coverage path already uses). The README should be updated to match whichever path is chosen.
