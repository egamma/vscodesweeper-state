# Issue #250617 — Find in selection finds no results with multipline lines selected

[https://github.com/microsoft/vscode/issues/250617](https://github.com/microsoft/vscode/issues/250617)

## Summary

Team-authored issue, still reproduces on main — the 'find in selection' scope locks to the selection at toggle time and ignores subsequent selection changes. No fix has landed. Keeping open per hard keep-open rule (MEMBER author).

## Change summary

Find in selection locks its search scope to the selection present when the toggle is activated, ignoring any subsequent selection changes by the user.

## Best solution

Update the find-in-selection logic in `src/vs/editor/contrib/find/browser/findController.ts` (and the find model) to track the active editor selection dynamically when the toggle is on, or at minimum re-evaluate the scope when the selection changes. Related issue https://github.com/microsoft/vscode/issues/284007 describes the same root cause and could be consolidated.
