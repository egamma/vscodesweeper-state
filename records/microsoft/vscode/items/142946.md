# Issue #142946 — [css] Allow extensions to choose to use whether to use CSS default data provider

[https://github.com/microsoft/vscode/issues/142946](https://github.com/microsoft/vscode/issues/142946)

## Summary

Accepted feature request (help wanted, team-endorsed) that has not been implemented on main. The `useDefaultDataProvider` option is not surfaced anywhere in the current css-language-features extension. Keeping open as a valid contribution opportunity.

## Change summary

Allow extensions to opt out of the CSS language service's default data provider via a `useDefaultDataProvider` toggle exposed alongside `css.customData`.

## Best solution

Add a property to the custom data contribution point (or a new setting) that lets an extension declare its custom data as a full replacement for the default provider, then pass `useDefaultDataProvider: false` to the language service when such a contribution is present. The change lives in `extensions/css-language-features/server/src/cssServer.ts` and the custom-data handling in `extensions/css-language-features/client/src/customData.ts`.
