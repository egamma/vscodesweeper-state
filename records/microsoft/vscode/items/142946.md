# Issue #142946 — [css] Allow extensions to choose to use whether to use CSS default data provider

[https://github.com/microsoft/vscode/issues/142946](https://github.com/microsoft/vscode/issues/142946)

## Summary

Valid feature request still unimplemented on main — `useDefaultDataProvider` remains hardcoded to `true` in `cssServer.ts`. A team member engaged positively and invited a contribution; the 'help wanted' label is appropriate.

## Change summary

Allow extensions to opt out of the default CSS data provider when contributing custom data, so they can fully replace the spec data for non-standard CSS dialects.

## Best solution

Add a property (e.g. `replaceDefaultDataProvider`) to the custom data file schema. In `extensions/css-language-features/server/src/cssServer.ts`, read that property and pass `false` for `useDefaultDataProvider` when an extension declares itself as a replacement. The 'help wanted' label already invites community contribution.
