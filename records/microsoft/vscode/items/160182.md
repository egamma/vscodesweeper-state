# Issue #160182 — Slightly de-clutter the merge editor UI

[https://github.com/microsoft/vscode/issues/160182](https://github.com/microsoft/vscode/issues/160182)

## Summary

Team-authored polish issue still partially open: the folding controller was removed from merge editor inputs (commit 831914170fa), but the gutter space reserved for folding indicators does not appear to be explicitly hidden via the `folding: false` editor option. The issue remains valid for reclaiming that horizontal space.

## Change summary

Hide the folding gutter area in merge editor input panes to reclaim horizontal space and reduce visual clutter.

## Best solution

Set `folding: false` in the editor options for the input code editor views in `src/vs/workbench/contrib/mergeEditor/browser/view/editors/codeEditorView.ts` so the gutter space is no longer reserved. The broader de-clutter discussion (auto-resolved conflict styling) was partially resolved by the move to CodeLens actions, but hiding the folding gutter remains unimplemented.
