# Issue #211501 — InlineCompletionItem Multiline Replace Range

[https://github.com/microsoft/vscode/issues/211501](https://github.com/microsoft/vscode/issues/211501)

## Summary

Valid feature request: the API documentation constraint ('Must begin and end on the same line') still exists on current main in vscode.d.ts line 5343. The underlying engine now handles multi-line replacements (via inline edits), but the InlineCompletionItem.range API contract hasn't been relaxed. Low community traction (0 upvotes) keeps this in Backlog without urgency.

## Change summary

Allow InlineCompletionItem.range to span multiple lines, removing the 'Must begin and end on the same line' API constraint.

## Best solution

Evaluate whether the existing inline edits infrastructure (which already handles multi-line replacements) can be surfaced through the InlineCompletionItem API by relaxing the single-line pre-condition. If the internal engine already supports it (no enforcement found in code), the change may be as simple as updating the API docs and ensuring the ghost text rendering handles multi-line replace ranges gracefully. Alternatively, point extension authors toward the inline edits API for multi-line scenarios.
