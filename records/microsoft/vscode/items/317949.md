# Issue #317949 — Welcome page cut off on small viewports

[https://github.com/microsoft/vscode/issues/317949](https://github.com/microsoft/vscode/issues/317949)

## Summary

Team-authored issue; confirmed in source — the welcome page's root container uses `overflow: hidden` with no scrollable inner wrapper, so content is clipped on small viewports. Keeping open for the area owner.

## Change summary

The Welcome/Getting Started page clips content on small viewports because the root container has `overflow: hidden` and no scroll fallback.

## Best solution

Change `overflow: hidden` on `.gettingStartedContainer` (line 15 of `gettingStarted.css`) to `overflow-y: auto` (or add a scrollable inner wrapper). The `height-constrained` / `width-constrained` grid variants already collapse the layout for small sizes, but the parent still clips. Verify that the slide transition animation (which likely motivated `overflow: hidden`) still works — consider scoping the hidden overflow to just the slide container.
