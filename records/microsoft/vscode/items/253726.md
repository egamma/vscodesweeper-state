# Issue #253726 — Hover badly positioned when size increases

[https://github.com/microsoft/vscode/issues/253726](https://github.com/microsoft/vscode/issues/253726)

## Summary

Still reproduces on main: when hover content grows (e.g. stopping a timer), the layout() repositioning logic can flip the hover position or misalign it relative to the target. No fix has landed since the issue was filed. Filed by a core team member (bpasero) while testing #253022.

## Change summary

Hover widget is badly positioned when its content size increases dynamically (e.g. when a timer stops).

## Best solution

The `layout()` method in `src/vs/platform/hover/browser/hoverWidget.ts` should preserve the hover's anchor position relative to the target when content grows, rather than re-running the full above/below flip logic which can cause jumps. The fix likely needs to track the original positioning intent and only flip when absolutely necessary.
