# Issue #168411 — local language specific setting priority is lower than user level language  specific setting

[https://github.com/microsoft/vscode/issues/168411](https://github.com/microsoft/vscode/issues/168411)

## Summary

Valid bug with milestone 'On Deck' and an open fixing PR (#285034) — must stay open. Multi-language workspace settings incorrectly lose to single-language user settings due to how the configuration model compares specificity.

## Change summary

Multi-language workspace-level language-specific settings (e.g. `[typescript][javascript]`) have lower priority than single-language user-level settings, violating the expected workspace > user precedence.

## Best solution

Land or iterate on the open PR https://github.com/microsoft/vscode/pull/285034 which addresses the specificity comparison in the configuration model so that workspace-level multi-language overrides correctly beat user-level single-language overrides.
