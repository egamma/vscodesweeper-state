# Issue #314781 — VS Code ignores my Freetype settings in all cases

[https://github.com/microsoft/vscode/issues/314781](https://github.com/microsoft/vscode/issues/314781)

## Summary

This is a duplicate of the open canonical issue tracking the Chromium Fontations font rendering regression on Linux. The root cause is upstream in Chromium — the Fontations font backend replaced FreeType and does not fully respect system fontconfig settings (hinting, subpixel antialiasing). VS Code previously had a workaround (`FontationsLinuxSystemFonts` disable flag) but it was removed when Electron 39 dropped that flag.

## Change summary

VS Code on Linux ignores system FreeType settings for subpixel antialiasing and font hinting, especially after switching from GNOME to KDE Plasma.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/260391 which is the canonical tracking issue for this Chromium Fontations regression on Linux, already labeled upstream-issue-linked and on the On Deck milestone.
