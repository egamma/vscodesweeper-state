# Issue #238804 — addBreakpoints not working for Instruction or Data breakpoints

[https://github.com/microsoft/vscode/issues/238804](https://github.com/microsoft/vscode/issues/238804)

## Summary

This issue requests addBreakpoints support for Data Breakpoints and Instruction Breakpoints. Both are already tracked by dedicated open canonical issues: #195151 (Data Breakpoints across sessions) and #253211 (Instruction Breakpoints in the API). Proposing duplicate close against #195151 as the primary, with #253211 covering the instruction breakpoint side.

## Change summary

The vscode.debug.addBreakpoints API should support DataBreakpoint and InstructionBreakpoint types, not just SourceBreakpoint and FunctionBreakpoint.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/195151 (Data Breakpoints) and https://github.com/microsoft/vscode/issues/253211 (Instruction Breakpoints). The ext host addBreakpoints method in extHostDebugService.ts needs to convert DataBreakpoint instances to IDataBreakpointDto (the main thread handler already accepts them), and InstructionBreakpoint support needs a new class and DTO path. Both canonical issues track this work.
