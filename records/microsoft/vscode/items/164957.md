# Issue #164957 — "Load More Stack Frames" is incorrectly displayed after debug adapter has sent the full stack

[https://github.com/microsoft/vscode/issues/164957](https://github.com/microsoft/vscode/issues/164957)

## Summary

Still reproduces on main: when the stale and current call stacks are identical (same frame IDs/names), `refreshTopOfCallstack` skips the `_onDidChangeCallStack.fire()` after fetching the full stack, so the UI never removes "Load More Stack Frames" even though `reachedEndOfCallStack` is now true.

## Change summary

"Load More Stack Frames" is incorrectly displayed after the debug adapter has sent all stack frames, when frames are identical across consecutive stops.

## Best solution

In `refreshTopOfCallstack` (debugModel.ts ~line 1655), fire `_onDidChangeCallStack` when `reachedEndOfCallStack` has changed even if the frame content is identical — the condition should be `if (bottomOfCallStackChanged || thread.reachedEndOfCallStack)` or similar, so the call stack view re-evaluates the "Load More" element.
