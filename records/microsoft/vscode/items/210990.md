# Issue #210990 — API to provide a terminal for use with `run-in-terminal` request

[https://github.com/microsoft/vscode/issues/210990](https://github.com/microsoft/vscode/issues/210990)

## Summary

Team-authored feature request on the 'On Deck' milestone — kept open per policy. The request for a debug terminal provider API remains unimplemented and is a valid extension-API gap for Python's environment activation needs.

## Change summary

Allow extensions to contribute an activated terminal for use with the DAP `runInTerminal` request, so debug sessions can run in properly environment-configured terminals.

## Best solution

Implement a `vscode.debug.registerDebugTerminalProvider` API (as sketched by connor4312) that lets extensions like Python provide a pre-activated terminal when `runInTerminal` fires for their debug type. The work likely lives in `src/vs/workbench/contrib/debug/` (terminal integration) and the extension API surface.
