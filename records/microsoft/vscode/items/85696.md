# Issue #85696 — Debug: call stack commands arguments 

[https://github.com/microsoft/vscode/issues/85696](https://github.com/microsoft/vscode/issues/85696)

## Summary

The internal commands now receive structured context (`sessionId`/`threadId`/`frameId`) via `getContext()`, but the extension-contributed commands still receive only a primitive (`string | number`) via `getContextForContributedActions()`. The comment on line 101 of callStackView.ts explicitly preserves this legacy shape. Documentation (point 3) also remains missing. The debt is partially paid but not fully resolved.

## Change summary

Pass structured objects (with sessionId/threadId) to extension-contributed call stack commands instead of bare threadId numbers or source URI strings, and document the contexts available.

## Best solution

Extend `getContextForContributedActions()` in `src/vs/workbench/contrib/debug/browser/callStackView.ts` to pass a structured object containing `sessionId` plus the existing primitive as an additional argument (preserving backward compat), and add documentation for the contributed command context shapes. The `CallStackContext` interface in `debugCommands.ts` already models the right shape.
