# Issue #232931 — Conditional breakpoint is not being hit with source-mapped TypeScript variables in VSCode

[https://github.com/microsoft/vscode/issues/232931](https://github.com/microsoft/vscode/issues/232931)

## Summary

Valid feature request for the JavaScript debugger to rewrite conditional breakpoint expressions using source-map name mappings. The work lives in vscode-js-debug, not VS Code core. Correctly labeled and routed; keeping open as a real gap.

## Change summary

Conditional breakpoint expressions should use source-mapped (TypeScript) variable names rather than the compiled JavaScript names.

## Best solution

The built-in JavaScript debugger (vscode-js-debug) would need to apply source-map name mappings to rewrite conditional breakpoint expressions before evaluating them in the runtime. This is related to the broader effort of supporting mapped variable names in debug expressions (watch, debug console, logpoints). The fix belongs in the vscode-js-debug extension's expression evaluation pipeline.
