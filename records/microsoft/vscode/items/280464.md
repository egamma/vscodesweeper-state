# Issue #280464 — VS Code Insiders blank screen on NVIDIA + Wayland after Electron 38 update (related to #230604)

[https://github.com/microsoft/vscode/issues/280464](https://github.com/microsoft/vscode/issues/280464)

## Summary

Upstream Electron/Chromium issue: Electron 38 changed `--ozone-platform-hint` default to `auto`, which triggers VA-API initialization failure on NVIDIA GPUs under Wayland, crashing the GPU process. A team member (deepak1556) is already engaged. Keeping open for the ongoing discussion.

## Change summary

VS Code Insiders shows a blank screen on NVIDIA + Wayland because Electron 38's new `auto` ozone-platform default triggers a VA-API crash in the GPU process.

## Best solution

The root cause is in Electron/Chromium's VA-API initialization on NVIDIA under Wayland. VS Code could mitigate by detecting NVIDIA + Wayland and falling back to X11 (the snap package already does this via `resources/linux/snap/electron-launch`). The upstream fix belongs in Electron's ozone platform auto-detection. deepak1556 is already investigating the snap-specific angle; this should stay open until a proper mitigation is landed, either upstream or as a VS Code-side fallback for non-snap packages.

## Evidence
