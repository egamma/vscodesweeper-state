# Issue #308463 — Warning about invalid MCP tool name

[https://github.com/microsoft/vscode/issues/308463](https://github.com/microsoft/vscode/issues/308463)

## Summary

Still reproduces on main: the tool name validation regex at line 144 of mcpServer.ts excludes dots, which the MCP specification explicitly allows. The code replaces dots with underscores and emits a spurious warning.

## Change summary

MCP tools with dots in their names trigger an incorrect 'invalid' warning and have dots replaced with underscores, despite dots being allowed by the MCP spec.

## Best solution

Update the `toolInvalidCharRe` regex in `src/vs/workbench/contrib/mcp/common/mcpServer.ts` (line 144) to allow dots: change `/[^a-z0-9_-]/gi` to `/[^a-z0-9_.-]/gi`. This aligns with the MCP specification's tool name rules.
