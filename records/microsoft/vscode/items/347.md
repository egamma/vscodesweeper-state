# Issue #347 — Support workspace extensions

[https://github.com/microsoft/vscode/issues/347](https://github.com/microsoft/vscode/issues/347)

## Summary

This is the canonical plan-item for workspace-scoped extension management with 76 upvotes and active use as the duplicate target (e.g. #322983 was duped here in July 2026). The core request — automatically enable/disable extensions per workspace via version-controlled config — remains unimplemented on main. Keep open.

## Change summary

Allow workspaces to define which extensions are enabled/disabled via a shareable config file, so teams get uniform setups and developers switching projects don't manually toggle extensions.

## Best solution

Implement a workspace-level extension activation config (e.g. in `.vscode/extensions.json` or workspace settings) that automatically enables/disables extensions when the workspace opens, shareable via version control. This is distinct from Profiles (personal, not repo-committed) and from recommendations (non-enforcing). The `plan-item` label indicates team intent to address this.
