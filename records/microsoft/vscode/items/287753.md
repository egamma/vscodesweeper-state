# Issue #287753 — memory leak related to `domStylesheets`

[https://github.com/microsoft/vscode/issues/287753](https://github.com/microsoft/vscode/issues/287753)

## Summary

Already fixed on current main by a concrete follow-up commit, so this can be closed as completed rather than with a `*` triage label.

## Change summary

Disposing stylesheets created from observables could leave `HTMLStyleElement` entries retained through `globalStylesheets`.

## Best solution

Treat the leak as fixed by commit a3dcfddb6ea1aee3033761d2528fe39cb29875db, which wires `WrappedStyleElement` disposal through `createStyleSheet`'s `DisposableStore` cleanup; no further action is needed unless a fresh leak is reported on a build containing that fix.

## Evidence

- **fix commit:** Commit a3dcfddb6ea1aee3033761d2528fe39cb29875db is titled `fix: memory leak in createStyleSheet2` and was merged via https://github.com/microsoft/vscode/pull/287754.
- **code change:** The fix changes `src/vs/base/browser/domStylesheets.ts` so `WrappedStyleElement` extends `Disposable` and calls `createStyleSheet(mainWindow.document.head, ..., this._store)`, letting disposal remove the style element and delete it from `globalStylesheets`.
- **current main:** Current `src/vs/base/browser/domStylesheets.ts` contains `disposableStore.add(toDisposable(() => style.remove()))` and `disposableStore.add(toDisposable(() => globalStylesheets.delete(style)))`; `createStyleSheetFromObservable` now stores a `WrappedStyleElement` in a `DisposableStore`.
- **release:** `git tag --contains a3dcfddb6ea1aee3033761d2528fe39cb29875db` includes `1.109.0` and later tags.
