# Issue #56719 — [npm] hover should show relevant latest version

[https://github.com/microsoft/vscode/issues/56719](https://github.com/microsoft/vscode/issues/56719)

## Summary

Keep open as a low-traction Backlog feature request: current main still shows only the package's absolute latest version in the npm package.json hover, not the latest version satisfying the user's declared range.

## Change summary

In package.json dependency hovers, show the latest version relevant to a range such as ^ or ~ in addition to the registry latest version.

## Best solution

If accepted, enhance the npm extension's package.json contribution to inspect the dependency value/range and registry version list, then add a range-aware latest version line to the existing hover. The existing Version Lens extension covers a related workflow, but that does not make the core feature request obsolete.
