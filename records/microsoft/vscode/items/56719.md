# Issue #56719 — [npm] hover should show relevant latest version

[https://github.com/microsoft/vscode/issues/56719](https://github.com/microsoft/vscode/issues/56719)

## Summary

The feature remains unimplemented on main. The hover now shows 'Latest version' and 'Installed version' (added in commit e42ae9ae255) but still does not show the latest version satisfying the declared semver range. The author noted an extension (Version Lens) covers this via CodeLens, making this also an extension-candidate, but it is a reasonable enhancement for the built-in npm extension.

## Change summary

When hovering a dependency version in package.json, show the latest version that satisfies the declared semver range (e.g. ^1.x or ~1.2.x), not just the absolute latest.

## Best solution

Enhance `getDocumentation` in `extensions/npm/src/features/packageJSONContribution.ts` to resolve and display the highest version satisfying the current semver range (using npm registry dist-tags or the versions list). Alternatively, this could remain an extension-candidate given Version Lens already covers it.
