# Issue #193143 — Git - After Rebase, recommend Push (Force)

[https://github.com/microsoft/vscode/issues/193143](https://github.com/microsoft/vscode/issues/193143)

## Summary

Still valid on main: after a completed rebase, VS Code's git action button and status bar show 'Sync Changes' (pull + push) with no awareness that a force-push is the correct next step. The 34 upvotes and steady community confirmation support keeping this open.

## Change summary

After rebasing a branch onto its upstream, VS Code recommends 'Sync Changes' instead of 'Push (Force)', which can mangle the branch history if followed.

## Best solution

Detect the post-rebase diverged state (both ahead and behind with no pending rebase) in `extensions/git/src/actionButton.ts` (`getSyncChangesActionButton`) and `extensions/git/src/statusbar.ts` (`SyncStatusBar`), and surface a 'Push (Force)' action instead of 'Sync Changes'. The `git.pushForce` command already exists; the gap is purely in the UI recommendation logic.
