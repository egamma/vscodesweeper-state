# Issue #319800 — VSCode + Fish Shell + MacOS: cmd+c & cmd+v not working for copy/paste

[https://github.com/microsoft/vscode/issues/319800](https://github.com/microsoft/vscode/issues/319800)

## Summary

The keybinding log shows cmd+c and cmd+v DO dispatch to the correct terminal commands (the 7b17c65f78d kitty keyboard fix is in 1.122.1), but the reporter says clipboard operations still fail. The spurious bare [KeyV] keydown 150ms after paste suggests a kitty-protocol key-release event leaking through. This is a real remaining bug with good detail, though needs confirmation from the terminal team.

## Change summary

cmd+c and cmd+v do not work for copy/paste in the integrated terminal when using Fish shell on macOS, despite the keybinding dispatching correctly.

## Best solution

The terminal team should investigate whether the kitty keyboard protocol's key-release events are generating spurious keydown events after meta-modified commands, and whether the terminal selection state is correctly propagated to the clipboard write path when kitty keyboard protocol is active. The extra [KeyV] at 09.157 in the keybinding log is the key clue — it appears after cmd+v paste completes.
