# Issue #763 — Hide empty folders after apply file exclusions (user settings)

[https://github.com/microsoft/vscode/issues/763](https://github.com/microsoft/vscode/issues/763)

## Summary

Still a valid, unimplemented feature request with 33 upvotes (above the 20-upvote backlog threshold). Compact folders (explorer.compactFolders) partially improved the UX but does not hide folders that become empty after files.exclude is applied. The underlying request remains unaddressed on current main.

## Change summary

Hide folders in the Explorer that become empty after files.exclude patterns are applied.

## Best solution

Introduce an opt-in setting (e.g. `explorer.hideEmptyExcludedFolders`) that filters out folders with no visible children after exclusions. The implementation would live in the `FilesFilter` in `src/vs/workbench/contrib/files/browser/views/explorerViewer.ts`, checking children recursively after exclusion filtering. The performance concern raised in comments (needing to resolve all children eagerly) is real and would need careful design — possibly deferred evaluation as folders are expanded.
