# Issue #173159 — Links aren't accessible in editors 

[https://github.com/microsoft/vscode/issues/173159](https://github.com/microsoft/vscode/issues/173159)

## Summary

Team-authored accessibility bug that remains unaddressed on main — no accessibility signal for links in editors has been implemented, and links remain undiscoverable to screen reader users via standard AT link-navigation commands.

## Change summary

Screen reader link-navigation commands (e.g. VoiceOver VO+Cmd+L) cannot discover links inside editors or markdown documents.

## Best solution

Add an accessibility signal (audio cue) when the cursor enters a link range, and surface the existing 'Open Link' command in the accessible help dialog so screen reader users have a discoverable path to activate links.
