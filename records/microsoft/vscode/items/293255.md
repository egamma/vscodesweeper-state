## Summary

Still reproduces on main: the model picker keybinding (Cmd+Alt+.) lacks the agent-mode filtering that the menu button has, so it shows all models even in Claude agent mode where only a subset should be visible. Team members are actively discussing the fix.

## Change summary

Keyboard shortcut to open model picker shows all models instead of the filtered set appropriate for the current agent mode (e.g. Claude).

## Best solution

Align the `OpenModelPickerAction` keybinding `when` clause with the menu visibility condition — add the `lockedToCodingAgent`/`chatSessionHasTargetedModels` guards from the menu to the keybinding, or have `openModelPicker()` itself respect the current session type's model constraints. A prior fix (524174e5a6f) was overwritten; the team (DonJayamanne, benibenj) is aware and discussing the right approach.
