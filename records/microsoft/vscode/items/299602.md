# Issue #299602 — `chat.tools.eligibleForAutoApproval` is ignored

[https://github.com/microsoft/vscode/issues/299602](https://github.com/microsoft/vscode/issues/299602)

## Summary

Team-authored issue, still reproduces on main. In `shouldAutoConfirm`, the session-level 'Bypass Approvals' check (line 1370) returns early before `isToolEligibleForAutoApproval` is consulted (line 1377), so `chat.tools.eligibleForAutoApproval` is effectively ignored when bypass mode is active.

## Change summary

The `chat.tools.eligibleForAutoApproval` setting is not honored when 'Bypass Approvals' mode is active — tools marked as not eligible still get auto-approved.

## Best solution

In `shouldAutoConfirm` in `src/vs/workbench/contrib/chat/browser/tools/languageModelToolsService.ts`, move the `isToolEligibleForAutoApproval` check before (or integrate it into) the session-level auto-approve early return at line 1370, so that tools explicitly marked `false` in `eligibleForAutoApproval` always require confirmation regardless of the session's approval level.
