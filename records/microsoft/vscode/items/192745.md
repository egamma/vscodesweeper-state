# Issue #192745 — Language specific settings are not language specific anymore when applied to all profiles

[https://github.com/microsoft/vscode/issues/192745](https://github.com/microsoft/vscode/issues/192745)

## Summary

Still reproduces on main: the `applyToAllProfiles` mechanism only handles flat setting keys and has no way to scope them to language-specific overrides. The gap remains unfixed.

## Change summary

Language-specific settings (e.g. `[python].editor.formatOnSave`) lose their language scope when the underlying key is added to `workbench.settings.applyToAllProfiles`, applying the setting globally across all languages in all profiles.

## Best solution

Extend the `applyToAllProfiles` mechanism to support language-override keys (e.g. `[python].editor.formatOnSave`) so users can share language-specific settings across profiles without losing the language scope. The fix likely lives in `src/vs/workbench/services/configuration/browser/configurationService.ts` (the `isSettingAppliedForAllProfiles` method and the parsing/filtering logic in `ApplicationConfiguration`).
