# Issue #247694 — No keyboard access to Cancel selection that is in progress

[https://github.com/microsoft/vscode/issues/247694](https://github.com/microsoft/vscode/issues/247694)

## Summary

The UI element selector feature has been entirely redesigned in the integrated browserView. The current implementation registers an Escape keybinding (when element selection is active) and Ctrl+Shift+C as a toggle, providing full keyboard access to cancel selection. The original Simple Browser contrib no longer exists.

## Change summary

No keyboard shortcut to cancel the in-progress element selection in the Simple Browser's UI element selector toolbar.

## Best solution

Keep the shipped behavior — the integrated browserView's `AddElementToChatAction` registers an Escape keybinding (gated on `CONTEXT_BROWSER_ELEMENT_SELECTION_ACTIVE`) and Ctrl+Shift+C as a toggle, so keyboard cancellation is fully supported. No further change needed.
