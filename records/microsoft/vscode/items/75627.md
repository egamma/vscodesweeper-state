# Issue #75627 — Extensions using the "type" command (for ex. Vim) have poor performance due to being single-threaded with other extensions

[https://github.com/microsoft/vscode/issues/75627](https://github.com/microsoft/vscode/issues/75627)

## Summary

Keep this open: it is an architectural feature request about latency-sensitive `type` command extensions sharing an extension host event loop, and the discussion shows the core team treated it as difficult but potentially improvable rather than resolved or out of scope.

## Change summary

Extensions such as Vim that intercept typing through the `type` command can become sluggish when unrelated extensions block the shared extension host.

## Best solution

Decide whether VS Code should provide a supported isolation path for latency-sensitive typing extensions, likely in the extension host/runtime allocation area, and whether the existing experimental affinity mechanism should be productized or extended for `type` command handlers.

## Evidence

- **maintainer discussion:** In https://github.com/microsoft/vscode/issues/75627, alexdima agreed with the general concern but explained that one extension host per extension has major architectural costs and that the current reality is extensions sharing a single event loop.
- **not resolved by policy:** The thread contains explicit discussion of potential improvements, including another extension host for Vim/type-handling extensions, so this is not a high-confidence *as-designed or *out-of-scope close.
- **current code:** Current main has an experimental affinity mechanism: src/vs/workbench/services/extensions/common/extensionRunningLocationTracker.ts reads `extensions.experimental.affinity` and assigns configured extensions to separate affinities, and src/vs/workbench/services/extensions/common/abstractExtensionService.ts starts local extension hosts for each affinity.
- **partial implementation:** Commit 3740be6c1eb087c2d87c37740a3844d67dd333cd introduced local process extension host affinity, but it is not an automatic or supported fix specifically for extensions that register `type`; commit c3992ef338c47b12f5fe336a51a1a4061e43c117 adds extension-declared affinity toward other extensions, which colocates rather than isolates them.
