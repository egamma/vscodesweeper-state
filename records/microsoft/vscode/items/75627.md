# Issue #75627 — Extensions using the "type" command (for ex. Vim) have poor performance due to being single-threaded with other extensions

[https://github.com/microsoft/vscode/issues/75627](https://github.com/microsoft/vscode/issues/75627)

## Summary

Still a valid feature request with 270 upvotes. The `extensions.experimental.affinity` setting provides a manual workaround, and a proposed `extensionAffinity` API (Jan 2026, commit c3992ef338c) lets extensions declare affinity in package.json — but the core ask (automatic isolation for type-intercepting extensions) remains unimplemented.

## Change summary

Extensions intercepting the `type` command (e.g. Vim) share a single-threaded extension host with all other extensions, causing input lag when any extension blocks the event loop.

## Best solution

Graduate the experimental affinity setting and proposed `extensionAffinity` API to stable, potentially with automatic affinity assignment for extensions that register the `type` command — so users don't need manual configuration. The work lives in `src/vs/workbench/services/extensions/common/extensionRunningLocationTracker.ts`.
