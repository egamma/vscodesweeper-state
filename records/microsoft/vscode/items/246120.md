# Issue #246120 — rounded corners are wrong radius on 2nd monitor

[https://github.com/microsoft/vscode/issues/246120](https://github.com/microsoft/vscode/issues/246120)

## Summary

This is likely an upstream Electron/Chromium issue. On Windows 11, rounded window corners are rendered by the OS DWM (Desktop Window Manager), not by VS Code's CSS. VS Code's custom title bar uses `frame: false` with `titleBarStyle: 'hidden'` — the corner radius is entirely determined by the OS and Electron's interaction with DWM. When moving between monitors with different DPI scaling, the OS-level corner radius doesn't update correctly. VS Code has no corner-radius CSS for Windows (only macOS).

## Change summary

Window rounded corners don't match native Windows style when moving VS Code from a high-DPI monitor (225% scaling) to a lower-DPI monitor (100% scaling).

## Best solution

Label as `upstream` and `windows`. The fix belongs in Electron's or Chromium's DWM integration for frameless windows on multi-DPI setups. Track Electron's progress on window frame handling (e.g. https://github.com/electron/electron/issues/51679 covers related frameless window DPI issues). Consider filing a targeted upstream issue if one doesn't exist yet.
