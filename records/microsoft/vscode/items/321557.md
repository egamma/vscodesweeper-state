# Issue #321557 — [Copilot] Display info about tokens spent on the concrete tasks and on the whole session

[https://github.com/microsoft/vscode/issues/321557](https://github.com/microsoft/vscode/issues/321557)

## Summary

Valid feature request for granular token usage visibility (input/output/cached token counts per turn and session totals) beyond the current credits-only display. Routes to billing and UBB area owners for triage.

## Change summary

Display detailed token usage breakdown (input, output, cached tokens) per task and per session, not just aggregate credit counts.

## Best solution

Expose per-turn token breakdown (input/output/cached) in the existing context-usage widget or a detail hover, and add a session-level summary. This depends on the Copilot service backend providing granular token data; coordinate with the billing/UBB team on data availability before designing the UI surface.
