# Issue #322207 — Feature Request/Regression: Extension update lifecycle causes aggressive cache invalidation and hardware resource exhaustion

[https://github.com/microsoft/vscode/issues/322207](https://github.com/microsoft/vscode/issues/322207)

## Summary

VS Code's globalStorageUri is keyed by extension identifier (not version) and is stable across updates — the platform already preserves extension global storage. The described cache invalidation is caused by the Kilo Code extension's own storage strategy, not by VS Code's update mechanism.

## Change summary

Extension updates allegedly invalidate local AI/LLM index caches stored via globalStorageUri, causing expensive re-indexing.

## Best solution

The extension (Kilo Code) should use ExtensionContext.globalStorageUri for its index databases — VS Code already guarantees this path is stable across version updates. The feature requests (resource-aware throttling, high-compute flags) are extension-candidate territory. The reporter should file against the Kilo Code extension.
