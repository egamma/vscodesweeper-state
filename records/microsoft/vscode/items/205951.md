# Issue #205951 — File drop should insert file paths, take 3

[https://github.com/microsoft/vscode/issues/205951](https://github.com/microsoft/vscode/issues/205951)

## Summary

Still reproducible on main: when dragging files from the explorer or tabs to external Windows apps (Command Prompt, PowerShell, Git Bash), VS Code sets only `text/uri-list` with `file:///` URIs and no `text/plain` fallback with the actual file path. Those apps paste the URI literally, which isn't a valid path. The issue is confirmed and labeled but has no fix or related PR.

## Change summary

Dragging files from VS Code to external Windows terminal apps pastes a file:// URI instead of a usable file path.

## Best solution

In `src/vs/workbench/browser/dnd.ts` (around the `fillEditorsDragData` function), also set `text/plain` on the `dataTransfer` with the platform file path (via `URI.fsPath`) for local `file://` resources, so that external apps that read `text/plain` get a usable path. This is a small, localized change in the drag data setup.
