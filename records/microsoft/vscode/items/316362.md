# Issue #316362 — Plugin-sourced MCP servers: no UI to view or control dedup state

[https://github.com/microsoft/vscode/issues/316362](https://github.com/microsoft/vscode/issues/316362)

## Summary

Legitimate polish request: the MCP collision/dedup logic works correctly (McpCollisionEnablementModel in mcpService.ts), but the Configure Tools picker and Customizations editor don't filter out or visually distinguish dedup-disabled plugin-sourced entries. Routing to the MCP management area.

## Change summary

Plugin-sourced MCP servers disabled by collision dedup still appear indistinguishably from active servers in Configure Tools and Customizations UI.

## Best solution

The Configure Tools picker (chatToolPicker.ts) and the AI Customization MCP view should either hide dedup-disabled plugin MCP entries or show a visual indicator (e.g. strikethrough, disabled badge, or source-plugin annotation). The enablement state is already available via McpCollisionEnablementModel — it just needs to flow into the UI rendering.
