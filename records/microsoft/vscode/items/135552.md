# Issue #135552 — [json editing] don't remove the comments of the next property node when removing first property

[https://github.com/microsoft/vscode/issues/135552](https://github.com/microsoft/vscode/issues/135552)

## Summary

Team-authored issue; the heuristic gap is still present on main. When removing the first property in a JSON object, the removal range extends to `next.offset`, consuming any comments that precede the next property.

## Change summary

Removing the first property in a JSONC object also removes comments that belong to the subsequent property.

## Best solution

Improve the heuristic in `src/vs/base/common/jsonEdit.ts` (around line 57-60) so that when removing the first property, the `removeEnd` stops before any comment tokens that precede the next property node, preserving them as belonging to the next property.
