# Issue #173134 — Support command variables in a terminal profile's env property

[https://github.com/microsoft/vscode/issues/173134](https://github.com/microsoft/vscode/issues/173134)

## Summary

The feature gap remains on current main: terminal profile `env` values are not passed through `_resolveVariables`, unlike `path` and `args`. The issue is correctly labeled `help wanted` and `feature-request` with milestone Backlog — it should stay open.

## Change summary

Support command variable substitution (e.g. `${command:python.interpreterPath}`) in a terminal profile's `env` property values.

## Best solution

Extend `_resolveProfile` in `src/vs/workbench/contrib/terminal/browser/terminalProfileResolverService.ts` to iterate over `profile.env` values and call `_resolveVariables` on each, mirroring the existing resolution for `path` and `args`. The `help wanted` label is appropriate for a community contributor to pick this up.
