# Issue #238775 — Dead Keys not working properly

[https://github.com/microsoft/vscode/issues/238775](https://github.com/microsoft/vscode/issues/238775)

## Summary

Dead key composition failure on Linux Wayland with Ozone platform is an upstream Electron/Chromium IME issue. VS Code receives composition events from Chromium — if those are broken in the Wayland backend, the fix belongs upstream. The existing editor-input-IME label is appropriate; keeping open as an upstream tracker.

## Change summary

Dead keys (e.g. circumflex) followed by a non-combining character produce no output when VS Code runs on Wayland with --ozone-platform=wayland on Fedora/GNOME.

## Best solution

Track this as an upstream Chromium/Electron Wayland IME bug. The composition event handling in Chromium's Ozone/Wayland backend needs to correctly emit the dead-key fallback character when composition is cancelled. VS Code should monitor upstream Electron issues (e.g. electron/electron#39679 and related) and update Electron once a fix ships.
