# Issue #239256 — Long descriptions in merge editor can push the action items off the screen.

[https://github.com/microsoft/vscode/issues/239256](https://github.com/microsoft/vscode/issues/239256)

## Summary

Still reproduces on main: the non-result merge editor header description has flex-shrink: 0, pushing toolbar actions off-screen when the column is narrow. The result header already has the correct flex-shrink: 1 for its description — the inconsistency is clear in the CSS.

## Change summary

Long descriptions in merge editor input headers push toolbar action buttons off-screen because the description element refuses to shrink.

## Best solution

Allow `.description` (and possibly `.title`) to shrink for non-result code-view headers, matching the existing pattern at line 36 of mergeEditor.css. Add `overflow: hidden; text-overflow: ellipsis` so truncated text is visually indicated.
