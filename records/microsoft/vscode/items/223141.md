Team-authored debt issue proposing an LRU cache optimization to reduce unnecessary extension activations during debugging/task runs. Keeping open per team-member policy — the optimization remains unimplemented.

## Change summary

Reduce extension activations triggered by debugging by caching task-label-to-extension mappings in workspace storage.

## Best solution

Implement the LRU cache (workspace storage, ~5 entries) mapping task labels to their providing extensions, so running a task doesn't activate all task-contributing extensions every time. Likely touches `src/vs/workbench/contrib/tasks/` resolution logic.
