# Issue #243122 — Bug: Arabic search fails to find words without diacritics (Tashkeel)

[https://github.com/microsoft/vscode/issues/243122](https://github.com/microsoft/vscode/issues/243122)

## Summary

This is a feature request for diacritic-insensitive search, not a bug — VS Code's text search performs exact text matching as designed. A team member acknowledged the limitation lives in ripgrep, which lacks a diacritic-ignore option. The same request was filed twice before (https://github.com/microsoft/vscode/issues/113461, https://github.com/microsoft/vscode/issues/303532) and both closed via the community upvote process. Keeping open given the member's engagement, but it should be reclassified as a feature-request.

## Change summary

VS Code text search should match Arabic words regardless of whether diacritics (Tashkeel/Harakat) are present in the search query or target text.

## Best solution

Reclassify as `feature-request`. The core limitation is in ripgrep, which has no diacritic-insensitive matching mode. A viable path would be either an upstream ripgrep feature request, or a VS Code-side normalization layer that strips combining marks before matching (similar to what quick access already does per the member comment). Given two prior closures for lack of upvotes, this needs community traction to reach the backlog.
