# Issue #208026 — Weird looking Output selector menu

[https://github.com/microsoft/vscode/issues/208026](https://github.com/microsoft/vscode/issues/208026)

## Summary

Already correctly triaged as upstream (Electron) by a maintainer. The Output selector menu's font size and separator rendering on macOS are governed by Electron's native menu implementation — VS Code cannot fix this without changes in Electron. The existing labels are accurate; keeping open for upstream tracking.

## Change summary

The Output view's channel selector menu on macOS renders with a too-small font (causing vertical misalignment) and uses dashes instead of native separators.

## Best solution

File or track this in Electron's issue tracker — the fix requires setting NSControlSize on macOS native menus and using proper native separators instead of dash strings. VS Code should adopt the fix once Electron ships it. No actionable change in the VS Code repo.
