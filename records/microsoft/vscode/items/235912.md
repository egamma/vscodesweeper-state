# Issue #235912 — Flicker of many color decorations when deleting

[https://github.com/microsoft/vscode/issues/235912](https://github.com/microsoft/vscode/issues/235912)

## Summary

Still reproduces on main: when a large block of color-decorated text is deleted, stale decorations briefly render at the collapsed position before the debounced recompute clears them. Two community PRs attempted fixes but were closed without merge.

## Change summary

Deleting a large block of text with color decorations causes a visible flicker frame where all color swatches stack at the deletion point.

## Best solution

In `ColorDetector.onDidChangeModelContent` (src/vs/editor/contrib/colorPicker/browser/colorDetector.ts), immediately clear or hide the visual color decorator decorations when a multi-line deletion is detected, before the debounced recompute fires. Alternatively, swap CSS class refs atomically in `updateColorDecorators` so old decorations aren't visible without styling.
