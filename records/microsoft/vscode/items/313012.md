# Issue #313012 — vscode-server cli 'code' tool does not report an error when called outside terminal

[https://github.com/microsoft/vscode/issues/313012](https://github.com/microsoft/vscode/issues/313012)

## Summary

Confirmed on current main: the remote CLI prints 'Command is only available in WSL or inside a Visual Studio Code terminal.' to stdout and returns with exit code 0, making it impossible to detect the error programmatically.

## Change summary

The `code` remote CLI tool exits with code 0 and prints to stdout (not stderr) when invoked outside a valid VS Code terminal, preventing scripts from detecting the error.

## Best solution

In `src/vs/server/node/server.cli.ts`, change the early-return path to write to stderr and set `process.exitCode = 1` before returning, so scripts can detect the invalid invocation.
