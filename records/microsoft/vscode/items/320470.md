# Issue #320470 — Explore an "efficiency mode" selection for chat

[https://github.com/microsoft/vscode/issues/320470](https://github.com/microsoft/vscode/issues/320470)

## Summary

Team-authored feature request exploring an efficiency/low-cost mode for chat — active internal discussion with design prototypes. Keeping open as a team-owned design exploration.

## Change summary

Add a toggleable 'efficiency mode' for chat that enforces lower reasoning effort, smaller context windows, and cost-optimized system prompts to reduce AI credit usage.

## Best solution

Continue the design exploration among the team. Key open questions from the discussion: whether this is a mode applied to any user-chosen model vs. a low-cost variant of Auto, whether it should influence model selection (token-thriftiness benchmarks), and whether surfacing cost/efficiency info per model is more compelling than a binary toggle. The CLI team is investigating a parallel concept (https://github.com/github/copilot-agent-runtime/issues/9962). Resolution rate vs. cost tradeoff data would inform the final design.
