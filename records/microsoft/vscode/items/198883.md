# Issue #198883 — Apparent lifecycle discrepancy between `ManagementPersistentConnection` and  `ExtensionHostPersistentConnection`

[https://github.com/microsoft/vscode/issues/198883](https://github.com/microsoft/vscode/issues/198883)

## Summary

The configurable reconnection grace time (commit 6cc2564bf9d, shipped ~1.96) addressed the original 3-hour hard-coded timeout, but the latest commenter (Feb 2026) reports premature extension host termination even with `reconnection-grace-time` set to 86400. The short-grace-time reduction path (`VSCODE_EXTHOST_IPC_REDUCE_GRACE_TIME`) could still trigger early shutdown when another client connects while the extension host is in a disconnected state — this remains a valid open bug.

## Change summary

Extension host terminates prematurely after long disconnections even when `reconnection-grace-time` is configured to a large value, leaving the management connection alive but extensions gone.

## Best solution

Investigate whether `VSCODE_EXTHOST_IPC_REDUCE_GRACE_TIME` messages are being sent to disconnected extension hosts unexpectedly (e.g. when a new unrelated connection arrives), causing the short grace timer to fire before the full configured grace period. The fix likely belongs in `src/vs/server/node/remoteExtensionHostAgentServer.ts` (the `shortenReconnectionGraceTimeIfNecessary` logic) or `src/vs/workbench/api/node/extensionHostProcess.ts`.
