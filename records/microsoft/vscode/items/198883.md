# Issue #198883 — Apparent lifecycle discrepancy between `ManagementPersistentConnection` and  `ExtensionHostPersistentConnection`

[https://github.com/microsoft/vscode/issues/198883](https://github.com/microsoft/vscode/issues/198883)

## Summary

Keep this open for remote-owner triage: the original hard-coded 3-hour timeout was made configurable, but current main still has short-grace paths that can terminate disconnected extension hosts, and a later report says the configured value is still not reliably honored.

## Change summary

Remote/web sessions can preserve workbench state while the extension host has terminated after a disconnected interval, leaving extensions gone after reconnect.

## Best solution

Clarify the intended contract for `--reconnection-grace-time` across management and extension-host connections, then either fix the short-grace/termination path in `src/vs/server/node/remoteExtensionHostAgentServer.ts`, `src/vs/server/node/remoteExtensionManagement.ts`, and `src/vs/server/node/extensionHostConnection.ts`, or document the cases where another client intentionally shortens the wait.

## Evidence

- **original report:** https://github.com/microsoft/vscode/issues/198883 reports `Extension host terminating: renderer disconnected for too long (1)` after more than 3 hours disconnected, followed by `Unknown reconnection token (seen before)` when reconnecting.
- **partial fix:** Commit 6cc2564bf9dfdb55cf62eebf7746cad0279819bd (`remote: configurable 'reconnection grace time'`, PR https://github.com/microsoft/vscode/pull/274910) added `--reconnection-grace-time`, plumbed it through the server environment, updated client reconnection grace time, and passed `VSCODE_RECONNECTION_GRACE_TIME` to the extension host.
- **current extension-host timeout:** `src/vs/workbench/api/node/extensionHostProcess.ts` currently reads `VSCODE_RECONNECTION_GRACE_TIME` and schedules `disconnectRunner1` to terminate with `renderer disconnected for too long (1)` after that grace time.
- **current short-grace path:** `src/vs/server/node/remoteExtensionHostAgentServer.ts` calls `shortenReconnectionGraceTimeIfNecessary()` on existing management and extension-host connections whenever a new connection arrives; `src/vs/workbench/api/node/extensionHostProcess.ts` then schedules `disconnectRunner2` for `renderer disconnected for too long (2)` using the short grace time.
- **still reported:** A 2026 comment on https://github.com/microsoft/vscode/issues/198883 says VS Code 1.109.5 with `reconnection-grace-time` set to `86400` still occasionally logs `Extension host terminating: renderer disconnected for too long` without a day-long disconnect.
