# Issue #295612 — Prevent distro from `release/xyz` pointing to a commit from `main`

[https://github.com/microsoft/vscode/issues/295612](https://github.com/microsoft/vscode/issues/295612)

## Summary

Hard keep-open: carries the `important` label. The issue remains substantively open — a `checkDistroCommit.ts` check exists but only warns (does not fail the build), and alexdima's unresolved comment about the branch-off timing case still needs a design answer.

## Change summary

Add a build-time check that fails when a release branch's `package.json#distro` points to a vscode-distro commit from `main` instead of the matching release branch.

## Best solution

Extend `build/azure-pipelines/common/checkDistroCommit.ts` to hard-fail (non-zero exit or `##vso[task.complete result=Failed]`) for release/* branches when the distro commit is not an ancestor of the release branch in vscode-distro, while allowing the legitimate just-branched-off grace period that alexdima flagged. The design question in the comment needs a maintainer answer first.
