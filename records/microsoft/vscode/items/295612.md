# Issue #295612 — Prevent distro from `release/xyz` pointing to a commit from `main`

[https://github.com/microsoft/vscode/issues/295612](https://github.com/microsoft/vscode/issues/295612)

## Summary

Valid engineering-debt request to add a build-time guard preventing release branches from referencing a distro commit on `main`. Carries the `important` label, so it stays open; the open design question from alexdima (how to distinguish the just-branched-off state from a mistaken update) still needs resolution.

## Change summary

Add a product-build check that fails when a `release/xyz` branch's `package.json#distro` points to a commit from `main` rather than the corresponding release branch in vscode-distro.

## Best solution

Implement a CI step in the product build pipeline that, when building from a `release/*` branch, verifies the referenced distro SHA belongs to the matching `release/*` branch in vscode-distro (e.g. via `git branch --contains <sha>` in the distro repo). The design challenge raised in the comment — distinguishing a fresh branch-off (where distro legitimately still points to `main`) from an accidental update — needs a policy decision first (e.g. a grace window, or requiring the distro branch to exist before the build is green).
