# Issue #68767 — Restart task fails if task already completed

[https://github.com/microsoft/vscode/issues/68767](https://github.com/microsoft/vscode/issues/68767)

## Summary

Still valid on current main: the `workbench.action.tasks.restartTask` command only operates on active (running) tasks, so completed tasks cannot be restarted through it. The terminal-context 'Rerun Task' action partially addresses the use case but doesn't cover the general restart command or the build-task shortcut flow.

## Change summary

Allow the 'Restart Task' command to also re-run tasks that have already completed, rather than only operating on currently active tasks.

## Best solution

Modify `_runRestartTaskCommand` in `src/vs/workbench/contrib/tasks/browser/abstractTaskService.ts` to also include recently-completed tasks (or all configured tasks) in the quick-pick when no active tasks match, so that 'restart' works regardless of task state — essentially making it a 'start or restart' command. The terminal 'Rerun Task' action already handles this for the focused terminal; the gap is in the general restart command.
