# Issue #68767 — Restart task fails if task already completed

[https://github.com/microsoft/vscode/issues/68767](https://github.com/microsoft/vscode/issues/68767)

## Summary

Still valid on main: the restart task command only operates on active tasks (`getActiveTasks()`), so it cannot restart a task that has already completed. The issue carries the 'On Deck' milestone, which is a scheduling decision that must be respected.

## Change summary

The 'Restart Running Task' command fails when the task has already completed, instead of unconditionally starting a new instance.

## Best solution

Modify `_runRestartTaskCommand` in `src/vs/workbench/contrib/tasks/browser/abstractTaskService.ts` to fall back to re-running the task (by looking up the task definition) when it's no longer in the active list, giving users a seamless 'start or restart' experience.
