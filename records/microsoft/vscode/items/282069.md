# Issue #282069 — GitHub Copilot checkpoint behavior for added and deleted files is neither expected nor well documented

[https://github.com/microsoft/vscode/issues/282069](https://github.com/microsoft/vscode/issues/282069)

## Summary

The checkpoint system has been substantially reworked since this was filed — file create/delete/rename operations are now tracked by `ChatEditingCheckpointTimelineImpl` and `ChatEditingDeletedFileEntry`. However, the user's specific scenario involved terminal-based `rm` (bypassing the editing API), and the documentation concerns remain valid. Keeping open because it's unclear whether all reported scenarios are now fully resolved on current main.

## Change summary

Checkpoint restore fails to recover files that the LLM created and then deleted (especially via terminal commands), and checkpoint behavior for added/deleted files is poorly documented.

## Best solution

Verify that the current checkpoint timeline implementation (which now tracks Create/Delete/Rename via `FileOperationType`) covers the reported scenarios — particularly when the agent deletes files through terminal tools rather than the workspace edit API. If terminal-based deletions are still untracked, the agent host snapshot controller or external-edit tracking needs to capture them. Separately, improve checkpoint documentation to clearly state what is and isn't restored.
