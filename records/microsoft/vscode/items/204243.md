# Issue #204243 — slow start up

[https://github.com/microsoft/vscode/issues/204243](https://github.com/microsoft/vscode/issues/204243)

## Summary

Correctly triaged upstream Chromium GPU-initialization issue causing slow startup when the Nvidia dGPU is disabled on Linux. The workaround (`--disable-gpu`) confirms the root cause lives in Chromium, and the existing labels (`upstream`, `gpu`, `chromium`) are accurate. Worth keeping open as a tracking point for this specific GPU path; a newer Nvidia driver (570) may resolve it for some users per the latest comment.

## Change summary

VS Code starts very slowly (15–30+ seconds of grey/unresponsive window) on Linux when the Nvidia discrete GPU is disabled, due to Chromium's GPU initialization path.

## Best solution

This is owned by Chromium's GPU probe logic. Keep tracking here until Electron ships a Chromium version that handles the missing-dGPU case gracefully. In the meantime the `--disable-gpu` flag or updating to nvidia-driver-570+ are effective workarounds. No VS Code core code change is actionable.
