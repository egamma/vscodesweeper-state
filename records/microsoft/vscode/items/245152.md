## Change summary

In the VS Code web version (github.dev), dragging to select text in the editor and releasing the mouse over the file tree sidebar causes the file under the cursor to open unexpectedly.

## Best solution

Investigate whether the list widget's click handler (`onViewPointer` in `src/vs/base/browser/ui/list/listWidget.ts`) should guard against click events where the mousedown originated outside the list (e.g. by tracking mousedown origin). This is likely a browser-specific behavior on Firefox where a `click` event fires even when `mousedown` started in a different element. A guard checking that a preceding mousedown was received on the same list would prevent the spurious file open.
