# Issue #269442 — Copying can replace new lines with another char

[https://github.com/microsoft/vscode/issues/269442](https://github.com/microsoft/vscode/issues/269442)

## Summary

Intermittent clipboard issue where newlines/whitespace are replaced with U+00A0 (nbsp) when copying, likely related to the editor's intentional use of nbsp for DOM whitespace rendering. Reporter cannot reproduce reliably and hasn't seen it in recent versions, but the hex dump proves it occurred at least once.

## Change summary

Copying text from VS Code (chat code blocks or editor) intermittently replaces newlines and whitespace with non-breaking space characters (U+00A0).

## Best solution

If it recurs, the fix likely involves ensuring clipboard serialization in the chat code block copy path strips nbsp back to regular whitespace. The rendering in `src/vs/editor/common/viewLayout/viewLineRenderer.ts` (lines 1069–1109) uses nbsp intentionally for DOM display, and the clipboard path should reverse this. Without a reliable repro, keep monitoring.
