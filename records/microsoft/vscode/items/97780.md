# Issue #97780 — Tab key when indenting with spaces does not insert correct number of spaces

[https://github.com/microsoft/vscode/issues/97780](https://github.com/microsoft/vscode/issues/97780)

## Summary

Valid feature request asking Tab within leading whitespace to indent the whole line rather than insert spaces at the cursor position. Already properly triaged with correct labels; low community traction (1 upvote in 6+ years).

## Change summary

When the cursor is within leading whitespace, pressing Tab should increase the line's total indentation to the next tab stop (like Xcode/TextMate) rather than inserting spaces relative to the cursor column.

## Best solution

Add an option (or modify `editor.useTabStops` behavior) so that Tab pressed within a line's leading whitespace triggers a full-line indent via `ShiftCommand` instead of `_replaceJumpToNextIndent`. The logic lives in `TabOperation.getCommands` in `src/vs/editor/common/cursor/cursorTypeEditOperations.ts` — a check for whether the cursor is within leading whitespace (before `firstNonWhitespaceIndex`) would branch to the indent path.
