# Issue #97780 — Tab key when indenting with spaces does not insert correct number of spaces

[https://github.com/microsoft/vscode/issues/97780](https://github.com/microsoft/vscode/issues/97780)

## Summary

Keep open as an accepted Backlog editor-commands feature request; current main still appears to insert spaces from the cursor position rather than reindenting the whole leading whitespace run when Tab is pressed inside indentation.

## Change summary

Pressing Tab anywhere within a line's leading spaces should advance the line indentation to the next full tab stop and move the cursor to the end of that indentation.

## Best solution

If the editor team accepts the behavior, update the Tab command path in src/vs/editor/common/cursor/cursorTypeEditOperations.ts so a cursor inside leading indentation replaces the leading whitespace with the next indentation level instead of inserting padding at the cursor position.
