# Issue #97780 — Tab key when indenting with spaces does not insert correct number of spaces

[https://github.com/microsoft/vscode/issues/97780](https://github.com/microsoft/vscode/issues/97780)

## Summary

This should stay open because it is a feature request already assigned to the Backlog milestone, which is a hard keep-open condition for this review. The report is a concrete editor indentation behavior request rather than a cleanup-close candidate.

## Change summary

When pressing Tab while the cursor is within leading spaces, VS Code should indent to the next full tab stop and move the cursor to the indentation boundary.

## Best solution

Keep this tracked in the editor command backlog and decide whether `editor.useTabStops` or the Tab indentation command should align leading-whitespace edits to the next configured tab stop.
