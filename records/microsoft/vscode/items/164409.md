# Issue #164409 — codicon under block cursor is not displayed properly

[https://github.com/microsoft/vscode/issues/164409](https://github.com/microsoft/vscode/issues/164409)

## Summary

Still reproducible on main: the block cursor renders plain text content over the cursor position, but codicons (inline decorations using icon fonts) are rendered in a separate layer and get visually obscured by the cursor's opaque background and text overlay.

## Change summary

Codicon inline decorations (e.g. debug stack frame icons) are visually hidden when a block cursor overlaps them because the cursor layer renders only the plain text character.

## Best solution

The block cursor rendering in `viewCursor.ts` should account for inline decorations at the cursor position — either by not rendering textContent when a codicon decoration is present, or by incorporating the decoration's visual into the cursor layer. This is a niche interaction (block cursor + inline codicon decoration) that affects debug scenarios.
