# Issue #190848 — search match case not working for notebooks

[https://github.com/microsoft/vscode/issues/190848](https://github.com/microsoft/vscode/issues/190848)

## Summary

Confirmed bug still present on main: the closed-notebook search path in CellSearchModel hardcodes case-insensitive, non-regex, non-whole-word search, ignoring the query flags. The open-notebook search path is correct. Related issue https://github.com/microsoft/vscode/issues/191571 shares the same root cause (whole-word flag also ignored).

## Change summary

Search 'match case' (and regex/whole-word) modifiers are ignored when searching inside .ipynb notebook cells via the closed-notebook search path.

## Best solution

Fix CellSearchModel.findInInputs() and findInOutputs() in src/vs/workbench/contrib/search/common/cellSearchModel.ts to accept and forward the query's caseSensitive, isRegExp, and wordSeparators options instead of hardcoding them to false/null. The caller in src/vs/workbench/api/common/extHostNotebook.ts ($searchInNotebooks) must also pass textQuery.contentPattern's flags through to CellSearchModel. This fix would also resolve the related issue https://github.com/microsoft/vscode/issues/191571.
