# Issue #182613 — respect Shebangs (#!) for langauge detection

[https://github.com/microsoft/vscode/issues/182613](https://github.com/microsoft/vscode/issues/182613)

## Summary

Still a valid issue on main: when typing a shebang incrementally, the ML-based language detection picks a language (e.g. Shell Script) partway through and doesn't re-evaluate against firstLine patterns as the line completes. The fix in ad980f0b210 (#287819) only added a new TypeScript firstLine pattern — it didn't address the re-evaluation behavior described here.

## Change summary

Typing a Python shebang incrementally in an untitled file causes language detection to stick on Shell Script instead of re-evaluating when the full shebang (with `python`) is present.

## Best solution

The `autoDetectLanguage` path in `BaseTextEditorModel` should re-check firstLine associations on content changes and allow them to override a previous ML detection result, so that a complete `#!/usr/bin/env python` shebang takes precedence over an earlier ML guess. Likely files: `src/vs/workbench/common/editor/textEditorModel.ts` and/or `src/vs/workbench/services/untitled/common/untitledTextEditorModel.ts`.
