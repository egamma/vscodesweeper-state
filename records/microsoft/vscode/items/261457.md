# Issue #261457 — NES: Debounce fetch based on `minShowDelay` setting

[https://github.com/microsoft/vscode/issues/261457](https://github.com/microsoft/vscode/issues/261457)

## Summary

Team-authored feature request for an optimization: use `minShowDelay` to also debounce the fetch itself (not just the display), giving providers more context-computation time. The `earliestShownDateTime` API exists and is passed to providers, but the fetch debounce is still separate. Keeping open per team-authored rule.

## Change summary

Delay the inline completions fetch request by `minShowDelay` duration so providers have more time to compute context and fewer requests are made.

## Best solution

In `inlineCompletionsSource.ts`, incorporate `minShowDelay` into the debounce logic before the fetch (around line 209) rather than only waiting after results arrive (line 372). This would reduce unnecessary requests when a large `minShowDelay` is configured.
