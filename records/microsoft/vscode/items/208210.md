# Issue #208210 — Exception with new Restart Extensions button

[https://github.com/microsoft/vscode/issues/208210](https://github.com/microsoft/vscode/issues/208210)

## Summary

This appears fixed on current main by the later cancellation-log suppression for extension-host shutdown; the reported stack is a cancellation thrown while running `setContext` during extension host termination.

## Change summary

Restarting extensions, reloading the window, or restarting the extension host produced noisy `Canceled: Canceled` errors in the Extension Host output.

## Best solution

Treat this as completed by commit 51a35659410cda847db2e56807e8ca2eb25e5e37, which stops logging cancellation errors from extension-host command execution during shutdown; no triage `*` label is needed.

- **current code:** `src/vs/workbench/api/common/extHostCommands.ts` now imports `isCancellationError` and wraps command error logging with `if (!isCancellationError(err))`, matching the issue's `Canceled: Canceled ... setContext` command stack.

- **scope match:** The reporter also reproduced with `Developer: Reload Window` and `Developer: Restart Extension Host`, which points to generic extension-host shutdown logging rather than only the Extensions view button.
