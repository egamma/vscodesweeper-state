# Issue #300583 — SubagentStart, UserPromptSubmit, and SubagentStop hooks don't have `transcript_path` in their input

[https://github.com/microsoft/vscode/issues/300583](https://github.com/microsoft/vscode/issues/300583)

## Summary

Confirmed on current main: SubagentStart, SubagentStop, and UserPromptSubmit (in subagent context) omit `transcript_path` from their hook input. The issue carries an 'On Deck' milestone, so it stays open for maintainer scheduling.

## Change summary

SubagentStart, SubagentStop, and subagent-context UserPromptSubmit hooks are missing `transcript_path` in their input, unlike other hooks (SessionStart, PreToolUse, PostToolUse) which include it.

## Best solution

The common input in `chatHookService.ts` already builds `transcript_path` from `sessionId`, but the subagent hooks appear to lack a transcript for their own session. Either the subagent session needs its own transcript path plumbed through, or the parent session's transcript path should be forwarded. The fix touches `extensions/copilot/src/extension/intents/node/toolCallingLoop.ts` (where SubagentStart/SubagentStop inputs are constructed) and possibly the `SubagentStartHookInput`/`SubagentStopHookInput` interfaces in `extensions/copilot/src/platform/chat/common/chatHookService.ts`.
