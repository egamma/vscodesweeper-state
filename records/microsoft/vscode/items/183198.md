# Issue #183198 — While draging to create mutipile-cursors the start point is where the cursor used to be rather than the position where I clicked.

[https://github.com/microsoft/vscode/issues/183198](https://github.com/microsoft/vscode/issues/183198)

## Summary

Still reproduces on main: Alt+Shift+drag column selection anchors from the existing cursor position rather than the mouse-down position, because the first click dispatches with `doColumnSelect: true` which reads `prevColumnSelectData` (defaults to cursor position when no prior column-select state exists).

## Change summary

Alt+Shift+drag to create multiple cursors anchors from the old cursor position instead of the click-start position.

## Best solution

In `viewController.ts` at line 284-285, the initial Alt+Shift mousedown should call `_columnSelect` with `doColumnSelect: false` (to anchor from the click position), and only subsequent drag events should use `true`. Alternatively, reset `_columnSelectData` to the mousedown position before the first `doColumnSelect: true` call. The fix lives in `src/vs/editor/browser/view/viewController.ts` and possibly `src/vs/editor/browser/controller/mouseHandler.ts`.
