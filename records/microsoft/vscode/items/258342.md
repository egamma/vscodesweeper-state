# Issue #258342 — `Chat: Show Chats` command doesn't respect Chat in Editor

[https://github.com/microsoft/vscode/issues/258342](https://github.com/microsoft/vscode/issues/258342)

## Summary

Still reproducible on main: the `PickAgentSessionAction` (`workbench.action.chat.history`) always opens selected sessions via `ChatViewPaneTarget` (sidebar), ignoring when chat is already open in an editor. The related PR #274268 was closed without merging.

## Change summary

The `Chat: Show Chats` command opens selected sessions in the sidebar even when the user is working in a chat editor, instead of opening them in the editor.

## Best solution

In `openSessionDefault` (src/vs/workbench/contrib/chat/browser/agentSessions/agentSessionsOpener.ts), detect when the active editor is a chat editor and use `ACTIVE_GROUP` as the target instead of `ChatViewPaneTarget`. Alternatively, the `PickAgentSessionAction.run` could pass context about the invocation source through to the picker/opener.
