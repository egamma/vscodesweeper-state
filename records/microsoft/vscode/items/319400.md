Team-authored bug report describing a stale sign-in dialog in the Agents window after authenticating in the editor window; keeping open for maintainer judgment.

## Change summary

The Agents window shows a sign-in dialog that does not dismiss after the user completes authentication via the editor window or the browser flow.

## Best solution

The authentication state change event from the editor window needs to propagate to the Agents window's sign-in dialog so it dismisses automatically. Likely involves the session/authentication service bridging between the two windows — investigate the agents-window auth flow in `src/vs/sessions/` and its consumption of `IAuthenticationService` events.
