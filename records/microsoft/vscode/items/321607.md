# Issue #321607 — Don' restore editors option not working

[https://github.com/microsoft/vscode/issues/321607](https://github.com/microsoft/vscode/issues/321607)

## Summary

Plausible bug report — the "Don't restore editors" crash-dialog option clears the editor-part layout memento, which should prevent non-dirty editors from reopening. However, if the offending files have unsaved changes, the hot-exit backup tracker independently reopens them. The report lacks the details needed to distinguish these cases or confirm against current main.

## Change summary

Selecting "Don't restore editors" in the crash/unresponsive dialog and clicking Reopen still restores the same editors, causing the crash to repeat.

## Best solution

Clarify whether the crash-causing files have unsaved content (hot-exit backups). If so, the fix is to also discard workspace backup metadata in `destroyWindow` when `skipRestoreEditors` is true (`src/vs/platform/windows/electron-main/windowImpl.ts`). If the files are clean, the storage deletion should already prevent restoration — investigate potential race between `workspaceStorage.close()` and the new window's storage init.
