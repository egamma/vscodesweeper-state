# Issue #308833 — Editing chats doesn't work properly

[https://github.com/microsoft/vscode/issues/308833](https://github.com/microsoft/vscode/issues/308833)

## Summary

Still reproduces on main: ChatSessionStore.setSessionTitle() updates the in-memory index entry but never calls flushIndex(), so the title change is lost on reload. A Copilot-authored fix PR (#308921) was opened but closed without merging.

## Change summary

Renaming a chat session from the session list (without opening it first) does not persist the updated title — it reverts on reload.

## Best solution

Add a `flushIndex()` call inside `ChatSessionStore.setSessionTitle()` (src/vs/workbench/contrib/chat/common/model/chatSessionStore.ts, line ~508) to persist the title change to disk, matching the pattern every other index-mutating method in that class already follows. An event should also be fired so the sessions list UI refreshes without requiring the session to be loaded.
