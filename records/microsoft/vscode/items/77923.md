# Issue #77923 — [html] auto close tags being inserted before its html content

[https://github.com/microsoft/vscode/issues/77923](https://github.com/microsoft/vscode/issues/77923)

## Summary

The auto-close insertion code in current main (and even back in 2019) has a document-version guard that should prevent inserting the closing tag when the user has typed further content. The issue is 6 years old, filed by a deleted user, has zero comments or confirmations, and the code path is well-guarded. Proposing close as not reproducible on current main.

## Change summary

HTML auto-close tag gets inserted before content the user typed during the round-trip latency in Remote-SSH.

## Best solution

Close as not reproducible. The version guard at `extensions/html-language-features/client/src/autoInsertion.ts:83` (`activeDocument.version === version`) already prevents stale insertions. If anyone still experiences this on a current build, they should open a new issue with detailed repro steps.

  > Thanks for the report. Looking at the current implementation of auto-close tag insertion (`extensions/html-language-features/client/src/autoInsertion.ts`), the code captures the document version when the `>` is typed and only inserts the closing tag if the version hasn't changed by the time the response arrives. This means if you continue typing during the network round-trip, the insertion is correctly skipped. Additionally, any keystroke within the first 100ms cancels the pending request entirely.
