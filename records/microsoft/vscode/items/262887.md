# Issue #262887 — javascript debugger local file in chrome: unable to connect port 9222

[https://github.com/microsoft/vscode/issues/262887](https://github.com/microsoft/vscode/issues/262887)

## Summary

The Chrome connection/retry logic lives in microsoft/vscode-js-debug (the bundled JS debugger extension), not in the VS Code core repo. The reporter's issue — Chrome not being ready on port 9222 before the debugger attempts connection — is owned by that extension's launch/attach flow.

## Change summary

Chrome debugging fails on first launch of a local HTML file because the debugger connects before Chrome finishes starting on port 9222.

## Best solution

This should be filed at https://github.com/microsoft/vscode-js-debug where the Chrome launch and connection-retry logic is implemented. The `timeout` option already exists (reporter has it set to 30000ms), so the real question is whether the extension's polling/retry within that window is sufficient — that's a js-debug concern.
