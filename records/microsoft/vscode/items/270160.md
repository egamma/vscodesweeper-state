# Issue #270160 — Add support for full-match patterns relative to workspace root for Copilot Instructions `applyTo` pattern

[https://github.com/microsoft/vscode/issues/270160](https://github.com/microsoft/vscode/issues/270160)

## Summary

Legitimate feature request with 26 upvotes, promoted to Backlog. The gap is confirmed on current main: patterns without a leading `/` or `**/` are auto-prefixed with `**/`, making workspace-root-relative matching impossible for users who expect anchored behavior.

## Change summary

Users want `applyTo` glob patterns to support full-match (workspace-root-anchored) semantics so `src/**/*.ts` only matches files under a top-level `src/` folder.

## Best solution

Document and/or implement the existing `/`-prefix escape as an anchor to workspace root (line 307 already skips the `**/` prefix for patterns starting with `/`), or introduce a new syntax. The fix lives in `src/vs/workbench/contrib/chat/common/promptSyntax/computeAutomaticInstructions.ts` around line 307.
