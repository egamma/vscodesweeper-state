# Issue #308311 — Display reason for why 'Add Models...' button in GitHub Copilot Chat is disabled ("greyed out")

[https://github.com/microsoft/vscode/issues/308311](https://github.com/microsoft/vscode/issues/308311)

## Summary

Valid polish request. The Add Models button already has a tooltip explaining the disabled state ('Adding models is managed by your organization'), but this asks for the reason to be displayed more prominently in the Language Models modal UI — not just on hover. Still a meaningful UX gap for enterprise users who may not think to hover.

## Change summary

When the 'Add Models' button is disabled due to enterprise policy, show the explanation text directly in the Language Models modal rather than only as a hover tooltip.

## Best solution

Add a visible inline message or description text near the disabled button in `chatModelsWidget.ts` (around line 1615) explaining the enterprise restriction, so users don't need to hover to discover why the button is greyed out. This is a small UX improvement in `src/vs/workbench/contrib/chat/browser/chatManagement/chatModelsWidget.ts`.
