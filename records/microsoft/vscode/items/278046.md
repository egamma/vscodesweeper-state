[https://github.com/microsoft/vscode/issues/278046](https://github.com/microsoft/vscode/issues/278046)

## Summary

Team-authored crash report — must stay open for maintainer judgment. The entire crash stack is in Electron/Chromium native drag-and-drop code (web_contents_view_cocoa.mm, render_widget_host_impl.cc), making this an upstream issue in the Electron layer.

## Change summary

VS Code crashes on macOS during app restart from an update when a stale dragging session is active in the Cocoa layer.

## Best solution

Investigate and fix upstream in Electron's WebContentsViewCocoa drag initiation path — the crash occurs in `-[NSDraggingSession _initWithPasteboard:draggingItems:clippingRect:source:]` triggered via Electron's `StartDrag` flow. A guard or cancellation of pending drag sessions before window teardown during update-restart may be needed at the Electron level.
