# Issue #324482 — Show timestamp for older conversations

[https://github.com/microsoft/vscode/issues/324482](https://github.com/microsoft/vscode/issues/324482)

## Summary

Valid feature request from a team member: the chat model already stores `lastMessageDate` and `creationDate` but the conversation view does not surface timestamps when reopening older sessions. Hard keep-open (MEMBER author).

## Change summary

When opening an older chat conversation, there is no visible timestamp indicating when the conversation took place.

## Best solution

Surface the conversation's `creationDate` or `lastMessageDate` (already stored in `ChatModel` and `IChatSessionEntryMetadata`) in the chat conversation header or session list UI. The rendering likely belongs in the chat widget header area (`src/vs/workbench/contrib/chat/browser/widget/`) or wherever the session title is displayed when restoring a previous conversation.
