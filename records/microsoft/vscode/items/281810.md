# Issue #281810 — Don't expand to show the same thing

[https://github.com/microsoft/vscode/issues/281810](https://github.com/microsoft/vscode/issues/281810)

## Summary

Team-authored (Tyriar, CONTRIBUTOR) polish request for collapsible thinking/collapsed blocks in chat — must keep open per hard keep-open rules. Active related work exists on single-item handling in the thinking content part.

## Change summary

Collapsible/thinking blocks that contain only a single item should merge or flatten their content so that expanding doesn't just reveal identical information.

## Best solution

Extend the single-item merging logic in `chatThinkingContentPart.ts` (specifically `shouldAllowExpansion()` and the `singleItemInfo` flow) to detect when expanding would reveal content identical to the collapsed title/summary, and flatten the display in that case. The existing `restoreSingleItemToOriginalPosition` pattern is a partial solution; this issue asks for the remaining cases where the expand is still redundant.
