# Issue #135179 — Nit: searching for markdown in language picker

[https://github.com/microsoft/vscode/issues/135179](https://github.com/microsoft/vscode/issues/135179)

## Summary

Still reproduces on main: the language picker sets a `meta` property (file extensions) on pick items, and `matchOnMeta` exists on the list, but the filter logic in `quickInputList.ts` never matches against `meta` — so typing "md" can't match Markdown's `.md` extension. Real but minor UX annoyance.

## Change summary

Searching "md" in the language picker does not prioritize Markdown because the `meta` (file-extension) field is set on items but never matched during filtering.

## Best solution

Wire `matchOnMeta` into the filter method in `quickInputList.ts` (around line 1440) so that items whose `meta` fuzzy-matches the query are kept visible — but without showing meta highlights unless there's an actual match. This was bpasero's suggestion in the original discussion.
