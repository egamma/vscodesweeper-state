# Issue #199829 — Handle markdown links better

[https://github.com/microsoft/vscode/issues/199829](https://github.com/microsoft/vscode/issues/199829)

## Summary

Team-authored debt issue requesting automatic locking of markdown links in localized strings to prevent translator breakage. Still valid — no automation for this exists on main yet.

## Change summary

Automatically lock markdown link syntax `[...](...)` in localize/l10n calls so translators cannot insert spaces that break links.

## Best solution

Add a build-time or lint-time rule (likely in `build/lib/i18n.ts` or the l10n tooling) that auto-detects markdown links in localized strings and either injects `{Locked}` comments or validates that they are present, removing the need for manual annotation.
