# Issue #236188 — Error: EPIPE: broken pipe, write

[https://github.com/microsoft/vscode/issues/236188](https://github.com/microsoft/vscode/issues/236188)

## Summary

Already correctly triaged as upstream. The EPIPE occurs because Vivado launches VS Code as a Windows GUI application without a console, so stdout/stderr pipes are broken when VS Code writes warnings. The root cause is in Vivado's launch mechanism, not VS Code.

## Change summary

Opening a .v file from Vivado via VS Code shows an EPIPE error dialog because stdout is unavailable.

## Best solution

Either Vivado fixes how it launches VS Code (as a console app), or VS Code could defensively catch EPIPE errors on console.warn calls in argvHelper.ts to suppress the dialog. The latter would be a resilience improvement in `src/vs/platform/environment/node/argvHelper.ts` or at the process level for stdout/stderr writes.
