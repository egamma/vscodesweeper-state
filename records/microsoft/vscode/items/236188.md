# Issue #236188 — Error: EPIPE: broken pipe, write

[https://github.com/microsoft/vscode/issues/236188](https://github.com/microsoft/vscode/issues/236188)

## Summary

The EPIPE error occurs because Vivado launches VS Code as a Windows GUI app without a valid stdout pipe, and VS Code's arg parser tries to console.warn about an unknown option (`-l3`). Already labeled `upstream` — the root cause is Vivado's launch method. VS Code could defensively handle EPIPE on console writes, but the issue is correctly classified.

## Change summary

Opening a .v file from Vivado triggers an EPIPE error dialog because Vivado launches VS Code without a console pipe, and VS Code writes a warning to stdout about an unknown CLI option.

## Best solution

VS Code could wrap `console.warn` calls in argvHelper.ts with a try/catch or suppress EPIPE errors on process.stdout to avoid the dialog, but the primary fix belongs to Vivado (launch VS Code as a console application). The current `upstream` label is appropriate; a defensive fix in `src/vs/platform/environment/node/argvHelper.ts` would be a minor polish.
