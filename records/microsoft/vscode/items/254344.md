# Issue #254344 — VSCode 1.90.x - 1.101.2 Crashes on Startup: bad_optional_access in -fno-exceptions Mode

[https://github.com/microsoft/vscode/issues/254344](https://github.com/microsoft/vscode/issues/254344)

## Summary

Confirmed upstream Chromium crash in the local font enumeration API (`queryLocalFonts`/FontAccess). A team member (deepak1556) identified the root cause and confirmed the workaround. The issue is correctly labeled `upstream`/`chromium` and should stay open pending an Electron/Chromium fix or a VS Code-side mitigation.

## Change summary

VS Code crashes on startup due to a `bad_optional_access` exception in Chromium's FontAccess/local font cache code on certain Windows systems.

## Best solution

Either wait for the upstream Chromium fix to land in Electron, or proactively guard `queryLocalFonts` usage in `src/vs/base/browser/fonts.ts` and consider whether FontAccess should be disabled by default in `src/main.ts` (lines 349–351) until the upstream fix ships — similar to how `FontMatchingCTMigration` and `StandardizedBrowserZoom` are already disabled there. A VS Code-side mitigation would unblock affected users without requiring a CLI flag.
