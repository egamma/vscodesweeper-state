# Issue #231076 — VS Code is ignoring the environment setting for node js --max-old-space-size

[https://github.com/microsoft/vscode/issues/231076](https://github.com/microsoft/vscode/issues/231076)

## Summary

Still reproduces on current main: the NODE_OPTIONS restoration workaround at line 279 of terminalEnvironment.ts is gated on `isMacintosh` only, so Windows integrated terminals never inherit the user's NODE_OPTIONS. Multiple community PRs are open to fix this — a maintainer needs to pick one and land it.

## Change summary

On Windows, the VS Code integrated terminal does not inherit the user's NODE_OPTIONS environment variable (e.g. --max-old-space-size) because the Electron workaround that restores it from VSCODE_NODE_OPTIONS is macOS-only.

## Best solution

Extend the `isMacintosh` guard at `src/vs/workbench/contrib/terminal/common/terminalEnvironment.ts:279` to also cover Windows (`isMacintosh || isWindows`). Several community PRs already propose this one-line change — review and merge one of the open PRs (e.g. https://github.com/microsoft/vscode/pull/305262 or https://github.com/microsoft/vscode/pull/320511).
