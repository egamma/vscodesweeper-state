# Issue #209374 — OSC hyperlinks are not restored properly on reconnect

[https://github.com/microsoft/vscode/issues/209374](https://github.com/microsoft/vscode/issues/209374)

## Summary

Still unfixed on main: OSC 8 hyperlink metadata (stored in xterm buffer cells) is not serialized or restored during terminal reconnection/reload, so the links appear as text but lose their associated URIs.

## Change summary

OSC 8 hyperlinks (escape-sequence-annotated links) lose their URL metadata after a window reload or terminal reconnection, rendering them unclickable.

## Best solution

Serialize OSC 8 hyperlink metadata (URI associations on buffer cells) as part of the terminal state that's persisted across reconnects, likely requiring xterm.js serialization support for link data and corresponding restoration logic in the terminal reconnection path.
