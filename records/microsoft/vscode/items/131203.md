# Issue #131203 — Configurable scroll padding

[https://github.com/microsoft/vscode/issues/131203](https://github.com/microsoft/vscode/issues/131203)

## Summary

Keep open: current main covers vertical cursor scroll padding through `editor.cursorSurroundingLines`, but the requested configurable horizontal cursor padding is still only partly covered by existing fixed/internal behavior.

## Change summary

Allow users to configure how much vertical and horizontal padding is kept around the cursor while scrolling.

## Best solution

Editor rendering should either expose a supported horizontal cursor reveal padding setting that pairs with `editor.cursorSurroundingLines`, or explicitly decide that the existing fixed horizontal reveal padding and `scrollBeyondLastColumn` are the intended product boundary.

## Evidence

- **current labels:** The issue is already classified as `feature-request` and routed to `editor-rendering`, with milestone `Backlog` rather than a release milestone.
- **vertical setting exists:** `src/vs/editor/common/config/editorOptions.ts` registers `editor.cursorSurroundingLines` with the description: controls the minimal number of visible leading and trailing lines surrounding the cursor.
- **horizontal behavior is fixed/adjacent:** `src/vs/editor/browser/viewParts/viewLines/viewLines.ts` still uses fixed `ViewLines.HORIZONTAL_EXTRA_PX = 30` for left reveal padding and `revealHorizontalRightPadding` default 15 for right padding; this is not the same as a documented character/line setting for both horizontal and vertical cursor scroll padding.
- **adjacent horizontal setting:** `src/vs/editor/common/config/editorOptions.ts` registers `editor.scrollBeyondLastColumn`, but its implementation in `src/vs/editor/common/viewLayout/viewLayout.ts` adds extra content width after the longest line, not configurable cursor-surrounding padding while revealing the caret.
