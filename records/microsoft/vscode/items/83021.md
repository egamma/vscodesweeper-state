# Issue #83021 — OS specific properties in Tasks.json to overwrite build shell doen't work

[https://github.com/microsoft/vscode/issues/83021](https://github.com/microsoft/vscode/issues/83021)

## Summary

The global-level OS-specific shell options (`windows.options.shell`) don't apply to contributed/typed tasks (e.g. `typescript`). A team member confirmed this limitation in 2019 and labeled it feature-request. No fix has landed on main; the gap remains, but has 0 upvotes and has been stale for 3+ years.

## Change summary

Global OS-specific shell configuration in tasks.json should apply to typed/contributed tasks (not just shell/process tasks).

## Best solution

Extend the task configuration merge logic in `src/vs/workbench/contrib/tasks/common/taskConfiguration.ts` so that global OS-specific shell options are applied to contributed task types, not just `shell`/`process` tasks. Given zero community traction this is low priority but remains a valid gap.
