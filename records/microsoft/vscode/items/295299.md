# Issue #295299 — Crashes when attempting to Open folder/Save new file/Navigate various menus, and causes VSCode install to become permanently unusable

[https://github.com/microsoft/vscode/issues/295299](https://github.com/microsoft/vscode/issues/295299)

## Summary

Untriaged SIGSEGV crash on EL8-based Linux distros (RHEL 8, Oracle Linux 8, Rocky Linux 8) and some other Linux systems, narrowed by a commenter to an Electron 42 regression (electron39 works, electron42 crashes). This is an upstream Electron/Chromium issue affecting users on older-but-supported glibc-2.28 platforms. Needs routing and upstream investigation.

## Change summary

VS Code crashes with SIGSEGV on RHEL 8 / Oracle Linux 8 / Rocky Linux 8 (and some ArchLinux configs) when opening folders, saving files, or accessing menus, starting with versions that shipped Electron upgrades beyond v35.

## Best solution

File or find the upstream Electron/Chromium bug for the SIGSEGV on glibc-2.28 platforms with Electron 42, and track it here. If the Electron team cannot fix the regression for these older platforms, VS Code may need to document updated minimum Linux requirements or carry a workaround. The issue needs the `linux` and `freeze-slow-crash-leak` area labels and a maintainer to investigate the Electron 42 native crash path.
