# Issue #242713 — Terminal search blocks renderer

[https://github.com/microsoft/vscode/issues/242713](https://github.com/microsoft/vscode/issues/242713)

## Summary

The upstream xterm.js search performance issue (xtermjs/xterm.js#5176) was fixed via decoration indexing by line and closed on 2026-05-26. VS Code adopted this fix by bumping to xterm.js 6.1.0-beta.288 in commit acc4a18ea6a — the terminal search renderer-blocking behavior should be resolved on current main.

## Change summary

Terminal search with high scrollback (50000 lines) blocks the renderer for ~300ms per keystroke due to slow decoration handling in xterm.js.

## Best solution

Keep the shipped xterm.js bump (6.1.0-beta.288) which includes the decoration-indexing fix from xtermjs/xterm.js#5902; no further VS Code change needed.
