# Issue #318327 — Add app and extension install/update history log

## Summary

Well-motivated feature request for extension install/update history logging to support supply-chain incident investigation. No existing implementation or duplicate found; routing to extensions area for triage.

## Change summary

VS Code should maintain a persistent log of which extension versions were installed and when, to aid supply-chain incident investigation.

## Best solution

Add a persistent, on-disk audit log in the extension management layer (src/vs/platform/extensionManagement) recording install/update/uninstall events with timestamps and version info, with a UI view or command to inspect it. This complements existing extension security efforts (signing, publisher verification) with retrospective forensics.
