# Issue #310844 — Atlassian MCP Server error "Error loading MCP App: MPC -32601: Method not found"

[https://github.com/microsoft/vscode/issues/310844](https://github.com/microsoft/vscode/issues/310844)

## Summary

The Atlassian MCP server advertises `_meta.ui.resourceUri` on its tools but doesn't implement the `resources/read` method — this is a server-side bug already tracked at atlassian/atlassian-mcp-server#186. VS Code correctly attempts to load the resource the server declared; the error is the server's responsibility.

## Change summary

VS Code calls `resources/read` on an MCP server that advertises a UI resource URI but doesn't support the resources capability, resulting in a -32601 error.

## Best solution

The fix belongs in the Atlassian MCP server — either stop advertising `_meta.ui.resourceUri` in tool responses, or implement `resources/read`. Already tracked at https://github.com/atlassian/atlassian-mcp-server/issues/186. VS Code could optionally add a defensive check against `capabilities.resources` before calling `readResource`, but that's a hardening improvement, not the root fix.
