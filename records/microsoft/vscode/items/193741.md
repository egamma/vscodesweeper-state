# Issue #193741 — gitlens doesn't set sideBarFocus, making sidbar keybindings difficult

[https://github.com/microsoft/vscode/issues/193741](https://github.com/microsoft/vscode/issues/193741)

## Summary

Valid bug acknowledged by the team (bpasero commented, team-applied labels, Backlog milestone). The sidebar focus context key may not update correctly for certain extension webview-based views. No fix found on current main; the trackFocus mechanism in paneCompositePart.ts still relies on DOM focusin/focusout which may not fire for iframe-hosted webview content.

## Change summary

The `sideBarFocus` context key is not set when focus moves into certain extension-contributed sidebar views (e.g. GitLens webview views), breaking keybindings that depend on `!sideBarFocus`.

## Best solution

Investigate whether the webview's focus notification (via `handleFocusChange`/`onDidFocus`) needs to explicitly refresh the sidebar's `paneFocusContextKey` when the webview is hosted in a sidebar pane — the fix likely lives in `src/vs/workbench/browser/parts/paneCompositePart.ts` or `src/vs/workbench/contrib/webviewView/browser/webviewViewPane.ts`.
