# Issue #193741 — gitlens doesn't set sideBarFocus, making sidbar keybindings difficult

[https://github.com/microsoft/vscode/issues/193741](https://github.com/microsoft/vscode/issues/193741)

## Summary

Still present on main: the sidebar's `sideBarFocus` context key is tracked via DOM focus events on the sidebar container element, but webview-based views (iframes) don't bubble focus events to the parent container, so `sideBarFocus` remains false when a webview view like GitLens has focus.

## Change summary

When a webview-based sidebar view (e.g. GitLens) receives focus, the `sideBarFocus` context key is not set to true, breaking keybindings that depend on it.

## Best solution

Bridge the webview's `onDidFocus`/`onDidBlur` events into the sidebar's focus tracking mechanism — either by calling `refreshState()` on the FocusTracker when a webview view gains focus, or by focusing a proxy element in the sidebar DOM when the webview gains focus. The fix likely belongs in `src/vs/workbench/contrib/webviewView/browser/webviewViewPane.ts` and/or `src/vs/workbench/browser/parts/paneCompositePart.ts`.
