## Summary

This should stay open: it is a reported editor RTL rendering bug tied to ligatures in non-default fonts, and the issue already has the focused `editor-RTL` area label. There is not enough evidence for a close because the reporter reproduced it with extensions disabled and with multiple ligature fonts, and no canonical duplicate or concrete fix commit is present in the supplied discussion.

## Change summary

Persian text inside a string literal is visually reversed when `editor.fontLigatures` is enabled with ligature-capable fonts.

## Best solution

Route/keep this with the editor RTL work so maintainers can decide whether the bidi/ligature interaction should be fixed in the editor text rendering layer or tracked upstream in Monaco/browser font shaping; a minimal repro with a specific font and current Insiders would help narrow ownership.
