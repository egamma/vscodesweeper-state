# Issue #271391 — Copilot cannot "see" contents of `Pseudoterminal`s created for `CustomExecution`

[https://github.com/microsoft/vscode/issues/271391](https://github.com/microsoft/vscode/issues/271391)

## Summary

Filed by a team member — keeping open. The bug is real: the chat terminal tools read output via xterm buffer (outputHelpers.ts), but CustomExecution Pseudoterminals may not populate that buffer in a way the tooling can capture, especially for the getTaskOutput tool path.

## Change summary

Copilot chat tools cannot read terminal output from tasks implemented via CustomExecution/Pseudoterminal, so the agent cannot help debug task failures.

## Best solution

The output collection in `src/vs/workbench/contrib/terminalContrib/chatAgentTools/browser/taskHelpers.ts` (line 246) and `outputHelpers.ts` (`getOutput`/`getRawOutput`) needs to handle the case where a CustomExecution Pseudoterminal writes output via its writeEmitter but the xterm buffer is not accessible or not populated at the time the tool reads it. The fix likely lives in ensuring the xterm instance is properly initialized for CustomExecution terminals before output collection, or adding a fallback output capture path.
