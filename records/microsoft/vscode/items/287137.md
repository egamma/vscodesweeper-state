# Issue #287137 — Pasting multiline in terminal when more than 1k characters in version 1.108.0 gets mangled

[https://github.com/microsoft/vscode/issues/287137](https://github.com/microsoft/vscode/issues/287137)

## Summary

Terminal paste mangling for multiline input exceeding ~1k characters is a reported regression from 1.108.0, confirmed by a second user in May 2026. No fix found on current main; the xterm.js update that landed in 1.108 (PR #285582) is a plausible culprit but unconfirmed. Routing to the terminal area for investigation.

## Change summary

Pasting multiline text longer than ~1000 characters with mixed brackets/quotes into the VS Code terminal garbles the output, a regression introduced in 1.108.0.

## Best solution

Investigate whether the xterm.js update in 1.108 or the bracketed-paste-mode handling introduces corruption at the ~1k boundary for shells like bash/zsh/psql on macOS. Likely involves the write path in terminalInstance.ts or xterm's internal paste handling. A targeted reproduction with a 1k+ multiline string containing quotes/brackets would confirm.
