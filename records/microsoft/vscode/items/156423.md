# Issue #156423 — Smooth scrolling has limited upper speed and causes skipping to make up

[https://github.com/microsoft/vscode/issues/156423](https://github.com/microsoft/vscode/issues/156423)

## Summary

Keep open: the report describes a specific smooth-scrolling lag/jump behavior, and current main still has fixed-duration smooth wheel animation logic that can trail a rapidly advancing target rather than proving the issue fixed.

## Change summary

Very fast physical mouse-wheel scrolling with editor smooth scrolling enabled can outrun the animation, then jump or skip when the wheel stops or reaches the file boundary.

## Best solution

Investigate the editor scrollbar smooth wheel path and adjust the smooth-scroll operation to keep up with high-rate wheel deltas, shorten/cancel smoothing when the target moves too far ahead, or otherwise avoid the catch-up jump. The likely code paths are src/vs/base/browser/ui/scrollbar/scrollableElement.ts and src/vs/base/common/scrollable.ts.

## Evidence

- **reporter confirmation:** In https://github.com/microsoft/vscode/issues/156423#issuecomment-1196736429 the reporter confirmed the problem only happens with "editor.smoothScrolling" enabled, and later stated it still happened after the requested information was provided.
- **current code:** src/vs/base/browser/ui/scrollbar/scrollableElement.ts computes wheel targets from getFutureScrollPosition() and calls setScrollPositionSmooth(desiredScrollPosition) for physical mouse wheels when mouseWheelSmoothScroll is enabled.
