# Issue #156423 — Smooth scrolling has limited upper speed and causes skipping to make up

[https://github.com/microsoft/vscode/issues/156423](https://github.com/microsoft/vscode/issues/156423)

## Summary

Still reproducible on main: the smooth scrolling animation uses a fixed 125ms duration with an ease-out-cubic curve, which inherently caps the maximum visual scroll speed. When rapid scroll events arrive faster than the animation can complete (flywheel mouse), the visual position accumulates lag and then jerks to catch up when scrolling stops.

## Change summary

Smooth scrolling with a fast-spinning flywheel mouse wheel caps visual scroll speed and then jerks to the final position when scrolling stops.

## Best solution

Adjust the smooth scrolling combine logic in `src/vs/base/common/scrollable.ts` to scale the animation duration (or skip animation entirely) when the delta between current visual position and target exceeds a threshold, so rapid scroll input doesn't accumulate unbounded lag.
