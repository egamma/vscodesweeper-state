# Issue #156423 — Smooth scrolling has limited upper speed and causes skipping to make up

[https://github.com/microsoft/vscode/issues/156423](https://github.com/microsoft/vscode/issues/156423)

## Summary

Keep open: this is a maintainer-acknowledged editor smooth-scrolling bug with a clear reporter confirmation that it only happens when `editor.smoothScrolling` is enabled, and the current issue context does not show a duplicate, fixing commit, or verification state.

## Change summary

Fast wheel input on a free-spinning Logitech mouse can outrun editor smooth scrolling, causing the visible scroll position to lag and then jump to catch up.

## Best solution

The editor scrolling implementation should either allow smooth-scroll animation to keep up with high-velocity wheel deltas or coalesce/cancel interpolation when the target scroll position gets too far ahead. This belongs with the editor scrollbar/smooth scrolling area rather than an extension.
