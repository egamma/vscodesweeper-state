# Issue #325456 — Copilot no longer explains terminal commands before approval

[https://github.com/microsoft/vscode/issues/325456](https://github.com/microsoft/vscode/issues/325456)

## Summary

The reporter wants Copilot Agent to explain terminal commands before the approval prompt. Whether the model provides explanatory prose before invoking the terminal tool is determined by model behavior and Copilot extension prompting — not a VS Code core UI regression. The confirmation widget currently shows the command with `message` as a hover tooltip, which is by design. Routing to the chat-terminal area for the team to decide whether to prompt the model to always explain commands or surface the message more prominently in the confirmation UI.

## Change summary

Copilot Agent stopped showing explanatory text alongside terminal command approval prompts, leaving users without context about what a command does before approving it.

## Best solution

The Copilot extension's agent prompts could instruct the model to always explain terminal commands in its response before calling `run_in_terminal`. Alternatively, the `confirmationMessages.message` content (currently shown only as a hover on the code editor in `chatTerminalToolConfirmationSubPart.ts`) could be rendered inline in the confirmation widget for better discoverability. The chat-terminal area owner should decide which approach fits the UX goals.
