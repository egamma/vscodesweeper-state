# Issue #170673 — Extra inlay hints appearing in minimap

[https://github.com/microsoft/vscode/issues/170673](https://github.com/microsoft/vscode/issues/170673)

## Summary

Still plausibly present on main: inlay hint decorations are placed as injected text in the view model for the full requested range (viewport + 30 lines above/below), and the minimap reads line content from the same view model — so hints outside the visible viewport appear only on the minimap. No fix found in git history.

## Change summary

Extra/phantom inlay hints appear on the minimap on lines outside the visible viewport when using `offUnlessPressed` mode.

## Best solution

The inlay hints controller (or the minimap rendering path) should ensure injected-text decorations created by inlay hints outside the current viewport don't render in the minimap. A fix likely lives in `src/vs/editor/contrib/inlayHints/browser/inlayHintsController.ts` or in how the minimap queries line content for sampled lines.
