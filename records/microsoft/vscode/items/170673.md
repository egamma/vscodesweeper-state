# Issue #170673 — Extra inlay hints appearing in minimap

[https://github.com/microsoft/vscode/issues/170673](https://github.com/microsoft/vscode/issues/170673)

## Summary

Keep open as a plausible editor-minimap/inlay-hints bug; I did not find a concrete fixing commit for https://github.com/microsoft/vscode/issues/170673, and current code still has a viewport-scoped inlay hint update path that could leave non-visible hint decorations observable by the minimap.

## Change summary

Inlay hints returned for viewport-based request ranges can appear as extra hint text in the minimap outside the visible editor viewport.

## Best solution

Decide in the editor area whether transient inlay hint decorations should be hidden from minimap rendering or whether old viewport-range hint decorations should be cleaned up more aggressively in src/vs/editor/contrib/inlayHints/browser/inlayHintsController.ts.
