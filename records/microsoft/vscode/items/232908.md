# Issue #232908 — workbench.files.action.createFileFromExplorer is not triggered via keybinding

[https://github.com/microsoft/vscode/issues/232908](https://github.com/microsoft/vscode/issues/232908)

## Summary

The report uses a single-letter keybinding ('a') which would be consumed by the tree's type-ahead keyboard navigation before it reaches the keybinding service — that is expected tree behavior. It is unclear whether the issue also reproduces with modifier keybindings (e.g. Ctrl+Shift+N). The precondition (`CanCreateContext`) should be satisfiable when an item is focused in the explorer, so a modifier-key binding ought to work. Need confirmation before closing or routing.

## Change summary

The command `workbench.files.action.createFileFromExplorer` does not fire when triggered via a user-configured keybinding while the file explorer has focus.

## Best solution

If the issue only reproduces with single-letter keys, this is likely as-designed (tree type-ahead consumes single keystrokes). If it also fails with modifier keybindings, the precondition or command dispatch for this ViewTitle-only action needs investigation in `src/vs/workbench/contrib/files/browser/views/explorerView.ts`. As a workaround, users should bind to `explorer.newFile` (inline new file) or `workbench.action.files.newFile` (save dialog new file) instead.
