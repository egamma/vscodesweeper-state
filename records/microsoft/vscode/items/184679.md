# Issue #184679 — Fuzzy search not working well in Recent Folders quickpick in 1.79.0

[https://github.com/microsoft/vscode/issues/184679](https://github.com/microsoft/vscode/issues/184679)

## Summary

Still reproduces on main: the quickpick filter matches the full query independently against label and description but never concatenates them, so typing 'foldername hostname' fails when the folder name is in the label and the host is in the description.

## Change summary

After PR #183740 moved remote host info from label to description in the Recent Folders picker, typing both folder name and host name together no longer filters correctly.

## Best solution

Either concatenate label+description for a combined fuzzy match in the quickpick filter (in `src/vs/platform/quickinput/browser/quickInputList.ts` around line 1450), or move the remote host suffix back into the label for recent workspace items so the existing single-field match covers it.
