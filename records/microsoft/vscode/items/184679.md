# Issue #184679 — Fuzzy search not working well in Recent Folders quickpick in 1.79.0

[https://github.com/microsoft/vscode/issues/184679](https://github.com/microsoft/vscode/issues/184679)

## Summary

This is a specific instance of the broader fuzzy cross-field matching limitation tracked by #194448. The Recent Folders picker already sets matchOnDescription: true, but the matching algorithm evaluates the full query against each field independently — it cannot match part of the query in the label and part in the description. Proposing duplicate close against the canonical issue.

## Change summary

Typing a query that spans folder name (label) and remote host (description) fails to match in the Recent Folders quickpick because fuzzy matching evaluates each field independently.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/194448 which tracks the general cross-field fuzzy matching improvement for all pickers. Implementing combined label+description matching there resolves this specific case.
