# Issue #244138 — Disabled and enabled (workspace) extension Disable button dropdown contains both "Disable" and "Disable (Workspace)" items

[https://github.com/microsoft/vscode/issues/244138](https://github.com/microsoft/vscode/issues/244138)

## Summary

Still reproduces on main: DisableGloballyAction enables itself when enablementState is EnabledWorkspace, causing a redundant 'Disable' option in the dropdown when the extension is already globally disabled but workspace-enabled.

## Change summary

When an extension is globally disabled but workspace-enabled, the Disable button dropdown incorrectly shows both 'Disable' and 'Disable (Workspace)' instead of just 'Disable (Workspace)'.

## Best solution

In DisableGloballyAction.update(), restrict the enabled condition to only EnablementState.EnabledGlobally (remove EnablementState.EnabledWorkspace from line 1783). This ensures 'Disable' (global) only appears when the extension is actually globally enabled. Multiple community PRs exist but none merged — a minimal one-line fix is all that's needed.
