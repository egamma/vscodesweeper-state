# Issue #197544 — Merge Editor: Accept Combination When Edits replace same range with same new text

[https://github.com/microsoft/vscode/issues/197544](https://github.com/microsoft/vscode/issues/197544)

## Summary

Team-authored feature request for smarter merge-editor combination logic — the smart-combine algorithm currently rejects overlapping edits even when both inputs produce the same replacement text. Still unimplemented on main; keeping open.

## Change summary

When both merge inputs replace the same base range with the same new text, the merge editor should auto-combine them instead of treating them as a conflict.

## Best solution

Extend the `smartCombineInputs` logic in `src/vs/workbench/contrib/mergeEditor/browser/model/modifiedBaseRange.ts` (around line 221) to recognize when overlapping `RangeEdit`s have identical replacement text and deduplicate them rather than returning `undefined`.
