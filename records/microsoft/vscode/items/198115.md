# Issue #198115 — `explorer.fileNesting.expand` ignored for items with `capture` and wildcard usage

[https://github.com/microsoft/vscode/issues/198115](https://github.com/microsoft/vscode/issues/198115)

## Summary

Verified duplicate of #149084: the root cause is `autoExpandSingleChildren: true` in the explorer tree, which overrides `fileNesting.expand: false` for folders with a single nested file stack. The refocus trigger in this report is just the refresh path that re-evaluates the collapse state.

## Change summary

File nesting items with `expand: false` auto-expand on window refocus when the nest has a single child, because `autoExpandSingleChildren` overrides the nesting expand setting.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/149084, which tracks the exact same root cause (`autoExpandSingleChildren` conflicting with `fileNesting.expand: false` for single-child nests). The fix would be to skip `autoExpandSingleChildren` for nested file items when `fileNesting.expand` is false.
