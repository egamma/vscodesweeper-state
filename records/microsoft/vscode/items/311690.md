# Issue #311690 — Chat copy button prioritizes selected text instead of the message

[https://github.com/microsoft/vscode/issues/311690](https://github.com/microsoft/vscode/issues/311690)

## Summary

Confirmed on current main: the copy-message button in chat checks for any native text selection scoped to the entire chat widget and copies that instead of the target message. An open PR already proposes a fix — keeping open for that PR to land.

## Change summary

Clicking the per-message copy button in Copilot Chat copies the current text selection instead of the message content when any text is selected anywhere in the chat widget.

## Best solution

Review and land https://github.com/microsoft/vscode/pull/311691 which removes the native-selection shortcut from `CopyItemAction`, so the copy button always copies its assigned message. The selection-aware copy behavior can remain available via the context menu or keyboard shortcut if desired.
