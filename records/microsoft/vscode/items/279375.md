# Issue #279375 — Use hover service not `title` for hovers in models and agent dropdown

[https://github.com/microsoft/vscode/issues/279375](https://github.com/microsoft/vscode/issues/279375)

## Summary

Still valid on current main: the action list and action view items still set native `title` attributes in several places instead of using the hover service. The discussion between contributors is still seeking a clean rework approach — keeping open.

## Change summary

Replace native `title` attribute hovers with the IHoverService-based managed hover in action list and action widget dropdown items for consistent hover UX.

## Best solution

Rework `ActionItemRenderer.renderElement` in `src/vs/platform/actionWidget/browser/actionList.ts` and the tooltip setup in `src/vs/base/browser/ui/actionbar/actionViewItems.ts` to use `setupManagedHover` / `IHoverService` instead of setting `element.title` directly. The `info-needed` label may no longer be appropriate since the technical investigation in the comments has identified the root cause (`ActionItemRenderer` setting `title`); re-labeling to `polish` would better reflect the state.
