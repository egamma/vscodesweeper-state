# Issue #275615 — SCM Graph - improve discoverability of commit expansion feature

[https://github.com/microsoft/vscode/issues/275615](https://github.com/microsoft/vscode/issues/275615)

## Summary

Valid UX polish request — still unaddressed on main. The SCM graph tree explicitly applies `force-no-twistie` to history item rows, so users with `doubleClick` expand mode have no visual cue that commits are expandable. Keeping open under discussion.

## Change summary

SCM Graph commit rows hide the twistie, making the expansion feature undiscoverable for users who set `workbench.tree.expandMode` to `doubleClick`.

## Best solution

Either conditionally show the twistie for commit rows when `workbench.tree.expandMode` is `doubleClick`, or add a tooltip/hover hint on the swimlane area indicating expandability. The relevant code is in `src/vs/workbench/contrib/scm/browser/scmHistoryViewPane.ts` around line 2018 where `force-no-twistie` is applied to `SCMHistoryItemViewModelTreeElement`.
