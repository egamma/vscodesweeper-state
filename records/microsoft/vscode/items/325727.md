# Issue #325727 — Folder sessions always show a merged PR icon

[https://github.com/microsoft/vscode/issues/325727](https://github.com/microsoft/vscode/issues/325727)

## Summary

Team-authored issue (MEMBER) — must stay open per policy. Reports that non-worktree folder sessions incorrectly display a merged-PR icon and unrelated git-pull diffs in per-turn changesets. The session's `IGitHubInfo.pullRequest` is populated from the branch's PR without distinguishing whether the session actually owns that PR. The issue describes a real UX bug in the Agents window session header.

## Change summary

Non-worktree folder sessions always show a merged PR icon and include unrelated git-pull changes in per-turn changesets, even when the session has nothing to do with that PR.

## Best solution

The session metadata population (likely in `sessionsService.ts` or wherever `IGitHubInfo.pullRequest` is set) should distinguish between sessions that own or are associated with a PR versus sessions that merely happen to be on a branch with a PR. For folder sessions not created from a worktree, the PR association should be suppressed or filtered. The per-turn changeset logic should similarly exclude upstream/pull changes unrelated to the session's own work.
