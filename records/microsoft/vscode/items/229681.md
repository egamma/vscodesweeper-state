# Issue #229681 ‚Äî Minimap folding marker names hide

[https://github.com/microsoft/vscode/issues/229681](https://github.com/microsoft/vscode/issues/229681)

## Summary

Keep open as an editor-minimap feature discussion: current VS Code already has all-or-nothing settings to hide these section header labels, but the report appears to be about unreadable/dense minimap labels rather than a proven fixed bug.

## Change summary

The request is to hide or reduce unreadable minimap folding/section marker names, especially for fenom `.tpl` files.

## Best solution

If this remains wanted, decide in the editor minimap area whether section headers should auto-suppress, merge, or otherwise avoid dense unreadable rendering. The existing workaround is to disable `editor.minimap.showRegionSectionHeaders` and/or `editor.minimap.showMarkSectionHeaders`.

- **settings wired to behavior:** src/vs/editor/contrib/sectionHeaders/browser/sectionHeaders.ts passes those minimap settings into `findRegionSectionHeaders` and `findMarkSectionHeaders`, and `findSectionHeaders()` returns without decorating when both are false.
- **rendering behavior:** src/vs/editor/browser/viewParts/minimap/minimap.ts renders each section header decoration in the viewport and fits long header text with an ellipsis, but I did not find logic that suppresses dense/overlapping section headers automatically.
