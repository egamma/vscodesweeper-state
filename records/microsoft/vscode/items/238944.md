# Issue #238944 — Should quick open remember it was sticked to a window side?

[https://github.com/microsoft/vscode/issues/238944](https://github.com/microsoft/vscode/issues/238944)

## Summary

Still reproduces on main: the DnD controller stores quick-open position as top/left ratios, so edge/corner placement drifts when restored in a different window size. The planned fix (moving to CSS-based positioning) hasn't landed.

## Change summary

Quick open position dragged to a window edge is not restored at the exact edge when reopened in a window of different size.

## Best solution

Refactor the DnD position storage in `quickInputController.ts` to detect and persist edge/corner 'sticky' alignment (like it already does for top/center snap), then restore via CSS constraints (e.g., `inset` anchoring) rather than ratio math, so edge placement survives window resizes.
