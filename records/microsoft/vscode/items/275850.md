# Issue #275850 — Switching to Insiders or back to stable in codespaces should handle chat extension properly

[https://github.com/microsoft/vscode/issues/275850](https://github.com/microsoft/vscode/issues/275850)

## Summary

Team-authored issue about chat extension incompatibility when switching between Stable and Insiders in Codespaces; must keep open per team-authored rule. The author reopened it after an initial duplicate close, indicating unresolved work remains.

## Change summary

Switching between VS Code Stable and Insiders when connected to the same Codespace shows a chat incompatibility message because both share the same extensions directory.

## Best solution

The extensions directory used by a Codespace should be scoped per connecting editor channel (Stable vs Insiders) so that chat extensions with channel-specific API versions don't conflict, or the incompatibility should be handled gracefully with a clear resolution path for the user.
