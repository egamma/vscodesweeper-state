# Issue #281871 — NES hidden behind the refactoring dialog

[https://github.com/microsoft/vscode/issues/281871](https://github.com/microsoft/vscode/issues/281871)

## Summary

Still reproduces on main: the inline completions/NES system does not check `CONTEXT_RENAME_INPUT_VISIBLE` before showing suggestions, so NES renders behind the rename dialog. Team-authored issue — keeping open for maintainer judgment.

## Change summary

NES (next edit suggestions) appear behind the rename refactoring dialog instead of being suppressed while the rename input is visible.

## Best solution

Suppress NES/inline edit suggestions when `CONTEXT_RENAME_INPUT_VISIBLE` is true, similar to how the system already suppresses inline completions in snippet mode (`_suppressInSnippetMode`). The fix likely belongs in `src/vs/editor/contrib/inlineCompletions/browser/model/inlineCompletionsModel.ts` alongside the existing snippet-mode suppression.
