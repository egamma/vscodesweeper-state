# Issue #253206 — Notification talks about inline suggestions

[https://github.com/microsoft/vscode/issues/253206](https://github.com/microsoft/vscode/issues/253206)

## Summary

Team-authored polish issue, still present on main: the notification at line 131 of commands.ts hardcodes 'No inline suggestion is available.' regardless of whether the user triggered the NES command or the inline suggestion command.

## Change summary

When 'Trigger Next Edit Suggestion' finds no result, the notification incorrectly says 'No inline suggestion is available' instead of mentioning NES.

## Best solution

Make the no-result notification message context-aware — pass the triggering command's display name (or a flag indicating NES vs inline suggestion) through the args so the message can say 'No next edit suggestion available' when triggered from NES. The fix is in `src/vs/editor/contrib/inlineCompletions/browser/controller/commands.ts` around line 131.
