## Summary

The crash occurs in the native Codex app-server binary bundled by the OpenAI ChatGPT extension (openai.chatgpt), not in VS Code itself. The stack trace shows V8 initialization failure inside the extension's own process — this is an extension-owned issue.

## Change summary

The OpenAI ChatGPT extension's bundled Codex native binary crashes with SIGTRAP during V8 initialization on Intel macOS Monterey.

## Best solution

Report this to the OpenAI ChatGPT extension team (openai.chatgpt) — the crash is in their bundled Codex app-server's V8 initialization (v8::Isolate::New → v8::base::OS::SetPermissions), likely a compatibility issue between their shipped V8 binary and older Intel/macOS Monterey. VS Code has no ownership of that native process.
