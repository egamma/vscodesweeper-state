# Issue #24043 — Neo keyboard layout: Some Keys stopped working

[https://github.com/microsoft/vscode/issues/24043](https://github.com/microsoft/vscode/issues/24043)

## Summary

Keep open: this is still a plausible Linux keyboard-layout/keybinding bug, and I found no concrete fixing commit or canonical duplicate. The issue is already in Backlog, which does not force closure, and later comments report remaining Neo/KOY/AltGr/Mod3/Mod4 failures even after the `keyboard.dispatch: keyCode` workaround.

## Change summary

Neo-style Linux layouts with M3/M4/AltGr layers fail to dispatch some editor navigation and modified shortcuts correctly.

## Best solution

Keep this with the keyboard-layout/Linux keybinding area and either fix the Linux mapper/native-keymap path to model ISO_Level3/ISO_Level5/Mod4 combinations correctly, or make an explicit maintainer decision if the limitation belongs upstream in native-keymap/Electron.
