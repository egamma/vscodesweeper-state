# Issue #24043 — Neo keyboard layout: Some Keys stopped working

[https://github.com/microsoft/vscode/issues/24043](https://github.com/microsoft/vscode/issues/24043)

## Summary

Still valid on current main: VS Code's keyboard layout model (`IKeyboardMapping`) only tracks 4 levels (value, shift, AltGr, Shift+AltGr) and cannot represent Neo's levels 3–6 via ISO_Level3_Shift/ISO_Level5_Shift, so higher-layer keys are lost. The `keyboard.dispatch: keyCode` workaround partially helps but doesn't cover Ctrl+layer4 combinations.

## Change summary

Neo keyboard layout layers 3+ (navigation, digits, special chars) don't work in VS Code on Linux because the layout model lacks support for ISO_Level3/5_Shift modifiers.

## Best solution

Extend `IKeyboardMapping` and the native-keymap consumer in `src/vs/platform/keyboardLayout/` to propagate levels 3–6 from the OS keyboard map (node-native-keymap already exposes ISO_Level3/5_Shift). The `help wanted` label is appropriate — this needs someone with a Neo/KOY layout to validate.

- **no Level3/Level5 references in src/:** grep for Level3_Shift, Level5_Shift, ISO_Level across src/vs/ returns zero results — these modifiers are not handled.
- **node-native-keymap PR merged:** https://github.com/microsoft/node-native-keymap/pull/19 added ISO_Level3_Shift/ISO_Level5_Shift support upstream, but VS Code's consumer has not been updated to use it.
- **workaround documented:** keyboard.dispatch: keyCode makes VS Code fall back to raw keyCodes, partially fixing layer 4 but not Ctrl+layer4 combinations (confirmed by multiple commenters).
