# Issue #191904 — `DebugSession` objects are leaked

[https://github.com/microsoft/vscode/issues/191904](https://github.com/microsoft/vscode/issues/191904)

## Summary

PR #191913 fixed some DebugSession leakage, but the core problem persists: `session.dispose()` is still commented out (TODO@roblourens) in debugService.ts:808, and the `DebugModel.sessions` array never removes ended sessions — they accumulate as inactive entries. The issue remains valid debt.

## Change summary

DebugSession objects leak memory because they are never disposed or removed from the sessions array after a debug session ends.

## Best solution

Uncomment and properly implement `session.dispose()` in `src/vs/workbench/contrib/debug/browser/debugService.ts` (line 808) and add removal of inactive sessions from `DebugModel.sessions` in `src/vs/workbench/contrib/debug/common/debugModel.ts`. The TODO comment suggests the dispose was intentionally deferred — the debug team should evaluate what blocks it (likely REPL history or session revival for EH debugging).
