# Issue #223405 — Only one changed variable gets announced by screen reader

[https://github.com/microsoft/vscode/issues/223405](https://github.com/microsoft/vscode/issues/223405)

## Summary

Team-authored accessibility bug still present on main — the screen reader alert fires per-expression but rapid successive alert() calls cause only the last to be announced. Keeping open per team-author policy.

## Change summary

When multiple watch variables change between breakpoints, only the last one is announced to the screen reader.

## Best solution

Batch the changed watch expression announcements into a single concatenated alert string (or use a short debounce to collect all changes and announce them together) in `src/vs/workbench/contrib/debug/common/debugAccessibilityAnnouncer.ts`.
