# Issue #156877 — Quick pick: allow to handle middle mouse button specifically

[https://github.com/microsoft/vscode/issues/156877](https://github.com/microsoft/vscode/issues/156877)

## Summary

Still unimplemented on main. The quick pick API does not expose middle mouse button state to callers — `IKeyMods` only covers ctrlCmd/alt/shift, and the middle-click handler in `quickInputList.ts` simply dismisses the picker. This is a valid, team-authored feature request that belongs in the backlog.

## Change summary

Expose middle mouse button clicks in the quick pick API so callers can handle them distinctly (e.g. open in new window).

## Best solution

Extend `IKeyMods` (or add a parallel `onMouseButton` callback) so quick pick clients can detect middle-click on accept and trigger secondary actions (e.g. open in new window). The change would touch `src/vs/platform/quickinput/common/quickInput.ts` (interface), `src/vs/platform/quickinput/browser/quickInputList.ts` (middle-click event propagation), and consumer sites like the recent-files picker in `windowActions.ts`.
