# Issue #291274 — Screen reader continuously announces terminal timing output when Copilot runs a build-like command in VS Code:A11y_Visuual studio code_Screen reader

[https://github.com/microsoft/vscode/issues/291274](https://github.com/microsoft/vscode/issues/291274)

## Summary

Team-authored accessibility bug: the integrated terminal's live region announces every line of rapid output to screen readers, making them unusable during build-like commands. No fix found on current main.

## Change summary

Screen readers continuously announce each terminal timing/progress update during rapid command output, overwhelming speech output.

## Best solution

Implement throttling or debouncing of aria-live announcements for rapid terminal output, or suppress intermediate progress lines from the screen reader live region. Likely involves the xterm.js accessibility layer or the VS Code terminal accessible buffer provider in `src/vs/workbench/contrib/terminalContrib/accessibility/`. The existing `chatTerminalToolProgressPart` work (commit cdf14551689) may provide a pattern for summarizing output rather than announcing it line-by-line.
