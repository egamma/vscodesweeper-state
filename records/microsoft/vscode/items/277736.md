# Issue #277736 — Support Right-to-Left (RTL) Text for Persian in Copilot Chat

[https://github.com/microsoft/vscode/issues/277736](https://github.com/microsoft/vscode/issues/277736)

## Summary

Keep open: this is an accepted Backlog feature request with a concrete chat rendering gap for mixed RTL/LTR text, and it is not stale-closeable under the community rules. Current main still appears to lack chat-specific bidi handling on the rendered markdown wrapper.

## Change summary

Copilot Chat should render Persian and other RTL chat text right-to-left while preserving correct mixed English/code layout.

## Best solution

Add chat-specific bidi support in the markdown/chat bubble rendering path, likely around src/vs/workbench/contrib/chat/browser/widget/chatContentMarkdownRenderer.ts, ChatMarkdownContentPart, and the relevant chat.css selectors; keep code blocks and tokenized code content explicitly LTR while applying per-block dir auto/plaintext-style behavior to natural-language message text.
