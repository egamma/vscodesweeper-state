# Issue #250942 — DEBUG CONSOLE - does not match file paths with ANSI escape sequences.

[https://github.com/microsoft/vscode/issues/250942](https://github.com/microsoft/vscode/issues/250942)

## Summary

Still reproduces on main: ANSI escape sequences within a file path cause the debug console link detector to split the path at the escape boundary, preventing full path+line+column matching. An attempted fix PR was closed without merging.

## Change summary

Debug console fails to detect clickable file path links when ANSI escape sequences appear between the filename and line/column suffix.

## Best solution

The fix in `debugANSIHandling.ts` needs to reassemble the stripped text across ANSI boundaries before passing to `linkDetector.linkify`, or linkify should receive the fully-stripped concatenation. The unmerged branch (commits 5ce182dd298, e84f245dafc) attempted this — reviewing and landing that approach (or a simpler variant) in `src/vs/workbench/contrib/debug/browser/debugANSIHandling.ts` would resolve it.
