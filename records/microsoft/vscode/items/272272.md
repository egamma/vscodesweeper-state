## Summary

Keeping open: the issue carries the 'On Deck' milestone, which is a scheduling decision that requires maintainer judgment. The file-locking problem in CI log publishing is an intermittent infrastructure issue that the team is actively tracking.

## Change summary

Azure DevOps pipeline fails to publish integration-test log files on Windows because a Git extension log file is still locked by another process during the publish step.

## Best solution

Fix the CI pipeline to either retry the publish with a delay, copy logs to a staging directory before publishing, or ensure the integration-test process fully exits (releasing file handles) before the publish step runs. The fix likely lives in the Azure Pipelines YAML under `build/azure-pipelines/`.
