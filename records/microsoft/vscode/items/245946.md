# Issue #245946 — Hover position is bogous

[https://github.com/microsoft/vscode/issues/245946](https://github.com/microsoft/vscode/issues/245946)

## Summary

The bogus hover positioning was fixed by changing the Quick Pick hover delegate from 'element' to 'mouse' placement — exactly the fix suggested in comment 1. Shipped in 1.105.0.

## Change summary

Quick Pick item hover tooltip renders at the wrong position when items have long multi-line descriptions.

## Best solution

Keep the shipped fix (commit 867ea4049c2, VS Code 1.105.0); the hover now positions at the mouse cursor instead of the overflowed element bounds. The secondary enhancement (container height growth instead of overflow) could be tracked separately if desired.
