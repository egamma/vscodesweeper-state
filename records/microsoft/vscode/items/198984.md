# Issue #198984 — Search view is default view when using remote

[https://github.com/microsoft/vscode/issues/198984](https://github.com/microsoft/vscode/issues/198984)

## Summary

Race condition where sidebar view restoration falls back to Search when the explorer view container isn't registered yet in remote scenarios. The closely related issue #245936 was closed as 'seems fixed' without a specific commit. No concrete fix was identified on current main for this specific race, but significant layout/restore work has occurred since the report (VS Code 1.84). The reporter confirmed it didn't reproduce in Insiders back in Nov 2023.

## Change summary

When reloading a remote (WSL) window, the sidebar switches to Search view instead of preserving the Explorer view.

## Best solution

Request confirmation on a current build. The reporter noted in comment 5 that Insiders didn't reproduce it (Nov 2023), and the related #245936 also self-resolved. If the race still exists, the fix likely belongs in the sidebar restore path in `src/vs/workbench/browser/layout.ts` (around line 793–813) or `paneCompositePart.ts`, ensuring the stored viewlet ID isn't discarded when the explorer view container hasn't registered yet during remote reconnection.
