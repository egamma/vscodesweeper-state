# Issue #286877 — Rendering code blocks that include `// filepath:` is buggy.

[https://github.com/microsoft/vscode/issues/286877](https://github.com/microsoft/vscode/issues/286877)

## Summary

The `// filepath:` annotation filtering in the code block processor strips content even when the LLM is asked to literally output such lines, causing content loss in rendered chat responses. No fix has been committed on main.

## Change summary

Code blocks containing `// filepath:` lines have their content incorrectly stripped from the rendered chat output, even when the user explicitly requested that content.

## Best solution

The fix belongs in `extensions/copilot/src/extension/codeBlocks/node/codeBlockProcessor.ts` — the `State.LineAfterFilePath` handling should not strip content when the filepath annotation wasn't injected by the system but came from the raw LLM response. Alternatively, the filtering should only apply when a `<vscode_codeblock_uri>` tag is present (system-annotated), not when the `// filepath:` comment is literally in the LLM output text.
