# Issue #211082 — Accessible view: pressing the "Menu" key breaks all context menus and accessible view

[https://github.com/microsoft/vscode/issues/211082](https://github.com/microsoft/vscode/issues/211082)

## Summary

Team-authored issue — keeping open. The underlying architecture conflict persists on current main: the accessible view is rendered via the singleton IContextViewService (line 318), and the context menu service also uses the same singleton (contextMenuService.ts:27). Pressing the Menu key can still trigger a context menu that overwrites the accessible view's context view, causing DOM errors.

## Change summary

Pressing the hardware Menu key in the accessible view causes it to close unexpectedly, breaks all context menus globally, and throws a removeChild DOM error until the window is reloaded.

## Best solution

The accessible view's blur handler (accessibleView.ts:719) should suppress hide when a context menu is opening, or the accessible view should not share the singleton context view service with context menus. The fix likely belongs in `src/vs/workbench/contrib/accessibility/browser/accessibleView.ts` — either by detecting the contextmenu event and preventing the blur-triggered hide, or by migrating the accessible view off IContextViewService to its own container.
