# Issue #275353 — Follow instructions file link tooltip shows full file path

[https://github.com/microsoft/vscode/issues/275353](https://github.com/microsoft/vscode/issues/275353)

## Summary

Still relevant on main — the tooltip for instruction file links in chat shows the full file:// path. For extension-provided prompts, displaying the extension name would be friendlier. The code at chatAttachmentWidgets.ts:902 still uses `labelService.getUriLabel(resource)` without special-casing extension-origin files.

## Change summary

Instruction file link tooltips in chat show the raw full file path instead of a user-friendly label like the extension name.

## Best solution

When rendering instruction file attachment tooltips in `src/vs/workbench/contrib/chat/browser/attachments/chatAttachmentWidgets.ts`, detect extension-contributed prompt files and show the extension display name instead of (or in addition to) the raw URI. The markdown link title syntax (`[text](path "Custom tooltip")`) could also be used upstream where the link markdown is constructed.
