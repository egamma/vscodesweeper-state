# Issue #124582 — User forwarded port has the first request TTFB too high

[https://github.com/microsoft/vscode/issues/124582](https://github.com/microsoft/vscode/issues/124582)

## Summary

Issue is on the 'On Deck' milestone (a real scheduling milestone), so it must remain open per policy. Beyond that, the problem is still confirmed by users as recently as May 2025 and no fix has landed — the bottleneck in remoteAgentConnection.ts (connectToRemoteExtensionHostAgent) remains on current main.

## Change summary

User-forwarded ports have excessive TTFB (~400–650ms) on initial and idle-timeout reconnections, whereas direct SSH port forwarding does not.

## Best solution

Investigate lazy/eager tunnel setup so the first request doesn't pay the full connection-establishment cost. The bottleneck is in `src/vs/platform/remote/common/remoteAgentConnection.ts` (the `connectToRemoteExtensionHostAgent` path for `ConnectionType.Tunnel`). A keep-alive or pre-warming strategy for user-forwarded ports could eliminate the idle-timeout re-establishment penalty.
