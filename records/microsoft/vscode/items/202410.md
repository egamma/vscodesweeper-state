# Issue #202410 — Aux window: xterm mouse tracking is broken in aux windows

[https://github.com/microsoft/vscode/issues/202410](https://github.com/microsoft/vscode/issues/202410)

## Summary

Still appears unfixed on main — no commit addresses xterm mouse tracking (SGR mode 1003/1006) in auxiliary windows. The bug is confirmed by the reporter on Insiders and reconfirmed by a second commenter in 2026, but the author association is NONE so the issue stays open on its merits.

## Change summary

xterm mouse tracking sequences (drag/mouseup) are not emitted when a terminal is moved to an auxiliary window.

## Best solution

Fix the mouse event routing so xterm.js receives document-level mousemove/mouseup events in the auxiliary window's document context, likely in the terminal instance's mouse-listener setup (terminalInstance.ts) or in how xterm.js attaches its CoreMouseService listeners to the owning window/document.
