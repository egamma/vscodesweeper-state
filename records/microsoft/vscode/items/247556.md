# Issue #247556 — 'An unknown error occurred' when using 'Refine alt text'

[https://github.com/microsoft/vscode/issues/247556](https://github.com/microsoft/vscode/issues/247556)

## Summary

Team-authored bug, still open. The 'Refine alt text' code action fails for relative image paths in markdown. The root cause (confirmed by justschen) is that the inline chat attachment pipeline cannot handle a resolved local file path when the image URI was originally relative. The code in `inlineChatCodeActions.ts` resolves the path (lines 151/162) but the downstream `vscode.editorChat.start` with the attached `URI.file(resolvedPath)` still triggers a fileService error. No fix has landed on main.

## Change summary

The 'Refine alt text' markdown code action errors with 'An unknown error occurred' when the image reference uses a relative path.

## Best solution

Fix the image URI resolution in the alt-text code action so that relative markdown image paths are correctly resolved to absolute file URIs before being passed as attachments to `vscode.editorChat.start`. The fix likely lives in `extensions/copilot/src/extension/inlineChat/vscode-node/inlineChatCodeActions.ts` (lines ~150-170) and possibly in the inline chat attachment handling in `src/vs/workbench/contrib/inlineChat/`.
