# Issue #284831 — INI Language Detection Overly Aggressive with Non-English Characters

[https://github.com/microsoft/vscode/issues/284831](https://github.com/microsoft/vscode/issues/284831)

## Summary

Legitimate, well-documented bug in the neural language detection model that over-triggers INI detection for non-English text. No fix has landed on main. Already correctly labeled and in Backlog.

## Change summary

Non-English text (21+ consecutive characters) is incorrectly auto-detected as INI format, forcing users to manually switch language mode.

## Best solution

The root cause is in the TensorFlow-based neural model (`@vscode/vscode-languagedetection` v1.0.21) which likely weights non-ASCII byte n-grams toward INI. Fix paths: retrain the model with better non-English plain-text samples, or add a heuristic guard that requires positive INI syntax markers (brackets, key=value) before accepting INI classification for short text. Relates to the broader https://github.com/microsoft/vscode/issues/131450 tracking issue.
