# Issue #237000 — Inline completions with multi cursor

[https://github.com/microsoft/vscode/issues/237000](https://github.com/microsoft/vscode/issues/237000)

## Summary

Valid feature request with 23 upvotes, promoted to Backlog. The InlineCompletionContext interface still has no multi-cursor information on current main — the gap remains.

## Change summary

Expose all cursor positions in the InlineCompletionContext so inline completion providers can generate context-aware completions for each cursor in a multi-cursor session.

## Best solution

Extend InlineCompletionContext (src/vs/editor/common/languages.ts) with an optional array of secondary cursor positions, and pass all selections when invoking provideInlineCompletions. The extension API (vscode.d.ts InlineCompletionContext) would surface the same field.
