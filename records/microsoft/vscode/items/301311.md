# Issue #301311 — Copilot Chat responded in Korean to a question written in Chinese

[https://github.com/microsoft/vscode/issues/301311](https://github.com/microsoft/vscode/issues/301311)

## Summary

The Copilot extension's ResponseTranslationRules correctly instructs the model to respond in the user's configured locale, but the underlying LLM occasionally ignores this instruction — this is intermittent model behavior, not a VS Code or extension code defect. Routing as *caused-by-extension since the Copilot extension/service owns the model interaction, and the reporter confirms it is not reliably reproducible.

## Change summary

Copilot Chat occasionally responds in Korean instead of Chinese when the user writes in Chinese.

## Best solution

This is an inherent LLM compliance issue — the system prompt correctly requests the user's locale (see `extensions/copilot/src/extension/prompts/node/base/responseTranslationRules.tsx`), but models sometimes produce responses in a wrong language. The Copilot service team could investigate whether stronger locale enforcement in the system prompt or post-processing helps, but this is not actionable as a VS Code core bug. The reporter could also try setting `github.copilot.chat.localeOverride` explicitly to `zh-CN` to reinforce the locale instruction.
