# Issue #316870 — VS Code Installer Option Add "Open with Code" action to Windows Explorer directory context menu failed

[https://github.com/microsoft/vscode/issues/316870](https://github.com/microsoft/vscode/issues/316870)

## Summary

The reporter's analysis is accurate: on Windows 11, the installer tasks `addcontextmenufiles`/`addcontextmenufolders` no longer control the directory context menu via legacy registry entries — folder context menus are now delivered via an AppX package. The checkboxes are misleading on Windows 11 since the AppX is installed unconditionally. If the AppX isn't registering correctly, that's a separate regression. Keeping open for the installer team to evaluate whether the task checkboxes need updated behavior or clearer UX on Windows 11.

## Change summary

The 'Add Open with Code to directory context menu' installer checkbox does not create the directory context menu entry on Windows 11 because the legacy registry path was replaced by an AppX-based mechanism that ignores the task selection.

## Best solution

The installer team should decide whether: (a) the task checkboxes on Windows 11 should also control the AppX-based context menu registration (currently AppX is unconditional), (b) the checkboxes should be hidden or relabeled on Windows 11 to avoid confusion, or (c) this is covered by the broader AppX context menu registration issue tracked in https://github.com/microsoft/vscode/issues/303103. The fix likely lives in `build/win32/code.iss` in the `ShouldInstallLegacyFolderContextMenu` and AppX installation logic.
