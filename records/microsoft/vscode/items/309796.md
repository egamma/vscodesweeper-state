# Issue #309796 — Terminal approval UI is missing slashes (on windows)

[https://github.com/microsoft/vscode/issues/309796](https://github.com/microsoft/vscode/issues/309796)

## Summary

Still reproduces on current main: the `toPlainText` regex on line 564 of `chatToolConfirmationCarouselPart.ts` strips backslashes along with markdown characters, eating Windows path separators. Team-authored issue with a release milestone — keeping open.

## Change summary

Terminal approval UI strips backslash characters from Windows file paths when converting markdown to plain text for the collapsed title.

## Best solution

Fix the regex in `toPlainText()` at line 564 to stop stripping `\` — it should only strip the markdown-significant characters (`*`, `_`, `#`, `>`). The fixture added in PR #315226 should then verify the fix.
