# Issue #145200 — CTRL delete/backspace navigation is different from CTRL arrow navigation

[https://github.com/microsoft/vscode/issues/145200](https://github.com/microsoft/vscode/issues/145200)

## Summary

The behavioral difference between Ctrl+Left (WordStartFast, skips single-char separators) and Ctrl+Backspace (WordStart, stops at every word boundary) is intentional — two different WordNavigationType enum values are used. Whether they should be unified is a design discussion that remains unresolved.

## Change summary

Ctrl+Left arrow skips single-character punctuation separators while Ctrl+Backspace stops at each word boundary, creating inconsistent word-navigation behavior.

## Best solution

A maintainer should decide whether `cursorWordLeft` (which uses `WordStartFast`) and `deleteWordLeft` (which uses `WordStart`) should share the same word-boundary logic, or whether the current asymmetry is desirable. The relevant code is in `src/vs/editor/common/cursor/cursorWordOperations.ts` lines 228–241 (the fast-skip logic). If unified, either the delete should adopt `WordStartFast` or navigation should drop the separator-skipping heuristic.
