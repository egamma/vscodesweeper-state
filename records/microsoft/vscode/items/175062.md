# Issue #175062 — Git - Clone of ssh-style URL fails b/c path does not strip username/hostname components

[https://github.com/microsoft/vscode/issues/175062](https://github.com/microsoft/vscode/issues/175062)

## Summary

Still reproduces on main: the SCP-style SSH URL folder-name derivation in `extensions/git/src/git.ts` line 437 does not handle the `user@host:repo` format, producing an invalid folder path. Two open PRs exist but neither has merged.

## Change summary

Cloning an SCP-style SSH URL (e.g. `git@host:repo`) fails because the derived folder name includes the `user@host:` prefix, creating an invalid path.

## Best solution

Land one of the two open PRs (https://github.com/microsoft/vscode/pull/297582 or https://github.com/microsoft/vscode/pull/318518) that fix the folder-name regex in `extensions/git/src/git.ts` to strip the `user@host:` prefix from SCP-style URLs before deriving the clone target directory.
