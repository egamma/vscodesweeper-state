# Issue #296255 — Chat Handoff should not change when switching mode

[https://github.com/microsoft/vscode/issues/296255](https://github.com/microsoft/vscode/issues/296255)

## Summary

Team-authored bug report about chat handoff actions disappearing when switching the active agent/mode. Must remain open per policy (author is MEMBER).

## Change summary

Chat handoff actions disappear when switching the agent after a plan is created, because they are based on the active agent rather than the agent from the last turn.

## Best solution

The handoff actions should be derived from the agent associated with the last chat turn, not the currently selected agent. The fix likely lives in the chat handoff/mode-switching contribution area.
