# Issue #296255 — Chat Handoff should not change when switching mode

[https://github.com/microsoft/vscode/issues/296255](https://github.com/microsoft/vscode/issues/296255)

## Summary

Team-authored bug report about chat handoff actions disappearing when switching the active agent/mode after a plan is created. Kept open per policy (MEMBER author).

## Change summary

Chat handoff actions disappear when the user switches the agent after a plan turn, because they are tied to the active agent rather than the agent from the last turn.

## Best solution

Make the handoff action visibility derive from the agent associated with the last chat turn rather than the currently selected agent, likely in the chat handoff contribution area.
