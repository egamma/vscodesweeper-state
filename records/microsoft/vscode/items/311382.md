# Issue #311382 — Expose a dedicated environment variable for agents

[https://github.com/microsoft/vscode/issues/311382](https://github.com/microsoft/vscode/issues/311382)

## Summary

Implemented on main and shipped: agent terminals now set `AI_AGENT=github_copilot_vscode_agent` (cross-vendor standard) and `COPILOT_AGENT=1` (back-compat), enabling exactly the use case described.

## Change summary

Request a dedicated environment variable in agent terminal sessions so shell scripts can distinguish agent from interactive use.

## Best solution

Keep the shipped behavior (AI_AGENT + COPILOT_AGENT env vars in agent terminals). The remaining expansion work (setting these in all agent-mode sessions, not just runInTerminal) is tracked by https://github.com/microsoft/vscode/issues/326069.
