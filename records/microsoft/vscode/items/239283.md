# Issue #239283 — Mouse Cursor Hides When Moving from Explorer Pane to Code Editor

[https://github.com/microsoft/vscode/issues/239283](https://github.com/microsoft/vscode/issues/239283)

## Summary

This is a known Windows 11 24H2 / Chromium bug where the mouse cursor disappears in Chromium-based apps, not a VS Code issue. The commenter already identified the root cause and linked the closed duplicate #236807 which confirmed the same. Workarounds exist at the OS level (change mouse pointer scheme or increase pointer size).

## Change summary

Mouse cursor disappears when moving from the Explorer pane to the Code Editor on Windows 11 24H2.

## Best solution

This is a Windows 11 24H2 / Chromium rendering bug. The workaround is to change the mouse pointer scheme or increase pointer size in Windows settings. The fix belongs in Windows or Chromium, not VS Code. See also https://github.com/microsoft/vscode/issues/239110 for the related pointer trails variant that remains open under discussion.
