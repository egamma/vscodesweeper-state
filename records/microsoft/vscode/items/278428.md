# Issue #278428 — On smaller screens vertical space is impacted when you have a lot of context

[https://github.com/microsoft/vscode/issues/278428](https://github.com/microsoft/vscode/issues/278428)

## Summary

Team-authored UX polish issue about chat context attachments consuming excessive vertical space on small screens. The chat input area has seen significant rework since filing (attachment tiles, compact icon variants, image carousel with max-height budgeting), but a deliberate compact/collapsed mode for context items on constrained viewports has not been explicitly addressed. Keeping open per team-member rule.

## Change summary

On smaller screens, adding multiple context attachments to the chat input consumes too much vertical space, leaving little room for the conversation.

## Best solution

Implement a collapsible or summary-style rendering for context attachments when vertical space is constrained — e.g. a scrollable single-row chip strip or a "+N more" overflow indicator. The relevant code lives in `src/vs/workbench/contrib/chat/browser/widget/input/chatInputPart.ts` and the attachment widgets under `src/vs/workbench/contrib/chat/browser/attachments/`. Recent work on compact tile rendering (commits `88df8ebeaea`, `d892fa747f4`, `7030aa9d697`) may partially address this, but a viewport-aware collapse strategy for the input area's context section would fully resolve the request.
