# Issue #298746 — Add deleteFile tool

[https://github.com/microsoft/vscode/issues/298746](https://github.com/microsoft/vscode/issues/298746)

## Summary

The request for a standalone deleteFile tool remains valid. While apply_patch gained delete support (commit 2b4e4592d67 for GPT models), there is no dedicated deleteFile/deleteFolder tool that integrates with the auto-approve permission model for replace_string models or users with terminal execution disabled.

## Change summary

Add a standalone deleteFile tool that respects workspace boundaries and integrates with the chat tool permission/auto-approve model.

## Best solution

Implement a dedicated deleteFile (and possibly deleteFolder) tool in extensions/copilot/src/extension/tools/node/ that verifies the target is within the workspace, respects files.exclude, and participates in the edit-tool permission model — enabling safe auto-approve for workspace-scoped file deletions without requiring terminal execution.

## Vision fit

- **Fit:** aligned
- **Reason:** A workspace-scoped delete tool aligns with the agent-security priority of sandboxing agent operations and reducing the need for blanket terminal approval.
- **Evidence:** Aligns with current agent security priorities around sandboxing local agent operations so approvals are less painful without sacrificing security.
