# Issue #221362 — BUG: `Move line down` command doesnt preserve indentation

[https://github.com/microsoft/vscode/issues/221362](https://github.com/microsoft/vscode/issues/221362)

## Summary

Still open and actively investigated by a team member. The indentation logic in matchEnterRuleMovingDown uses the wrong context line when moving lines back up, causing formatting loss. No fix has shipped.

## Change summary

Move Line Up/Down commands lose indentation when moving lines past multi-line import/block statements.

## Best solution

Fix the indentation calculation in `matchEnterRuleMovingDown` in `moveLinesCommand.ts` to consider the moved line's context rather than only the previous valid line. The `tree-sitter-autoindent-candidate` label suggests this may be better solved with tree-sitter-based indent computation.
