# Issue #273020 — Menu Bar Compact ，The menu check status is inconsistent, and the status will be consistent when it is opened the second time

[https://github.com/microsoft/vscode/issues/273020](https://github.com/microsoft/vscode/issues/273020)

## Summary

Confirmed bug still present on main: the `focusInsideMenubar` guard in menubarControl.ts suppresses menu-item state updates while the menubar has focus, causing stale checked states on the first open after a toggle action. The issue is already confirmed by a maintainer and appropriately labeled.

## Change summary

In compact menu bar mode, toggled menu items (like Word Wrap) show stale checked/unchecked state on first open; the correct state appears only on the second open.

## Best solution

Fix the update logic in `src/vs/workbench/browser/parts/titlebar/menubarControl.ts` so that when `focusInsideMenubar` becomes false (menu closes), any pending `onDidChange` updates are flushed — or update the actions when the menu is about to open rather than relying solely on the change event while the menu is closed.
