# Issue #273020 — Menu Bar Compact ，The menu check status is inconsistent, and the status will be consistent when it is opened the second time

[https://github.com/microsoft/vscode/issues/273020](https://github.com/microsoft/vscode/issues/273020)

## Summary

Confirmed bug still present on main: when a toggled menu action executes in compact mode, the checked state is stale on the next open because the onDidChange handler at line 604 skips updates while focusInsideMenubar is true, and the timing of focus-out vs. menu-change events causes the update to be missed.

## Change summary

In compact menu bar mode, toggled menu items (like Word Wrap) show stale checked state on first reopen after executing the action; correct state only appears on second open.

## Best solution

After the menubar loses focus (or after an action executes and the menu closes), force a re-evaluation of checked states for the affected menu's actions. The fix likely belongs in `src/vs/workbench/browser/parts/titlebar/menubarControl.ts` around the focus-out / onFocusTrackerBlur handler — queue a deferred updateActions call when focusInsideMenubar transitions from true to false and a change was missed.
