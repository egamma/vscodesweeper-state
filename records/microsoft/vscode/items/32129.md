# Issue #32129 — SCM - Support altCommand for SCM resources

[https://github.com/microsoft/vscode/issues/32129](https://github.com/microsoft/vscode/issues/32129)

## Summary

Still unimplemented on main: the SCM resource state API does not expose an altCommand property, so Alt+Click alternative actions on SCM resources are not supported. The platform menu system supports `alt` commands, but the SCM extension API and view never wires it. Keeping open as a valid, labeled feature request with the `help wanted` tag.

## Change summary

Support an alternative command (Alt+Click) on SCM resource states so extensions can offer 'Open File' as the alt action to the default 'Open Change'.

## Best solution

Extend the SCM resource state API (`SourceControlResourceState.command`) with an `altCommand` property, wire it through `extHostSCM.ts` and the SCM view pane so that Alt+Click triggers the alternative command. The platform's `MenuItemAction.alt` pattern already exists as a model.
