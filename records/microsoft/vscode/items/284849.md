# Issue #284849 — Support for Custom Attributes in `.prompt.md` YAML Front Matter

[https://github.com/microsoft/vscode/issues/284849](https://github.com/microsoft/vscode/issues/284849)

## Summary

Valid feature request to allow custom/unknown YAML front matter attributes in .prompt.md files without triggering validation hints. The current validator in promptValidator.ts intentionally flags unknown attributes. Only 3 upvotes so far — well below the 20-upvote backlog threshold. Keep open for community traction.

## Change summary

Custom YAML front matter attributes in .prompt.md files trigger 'not supported' validation hints, preventing teams from adding organizational metadata without warnings.

## Best solution

Either suppress validation for unknown attributes (possibly behind a setting), or introduce a defined 'custom'/'metadata' namespace in the front matter schema that passes through without warnings. The change would be localized to `src/vs/workbench/contrib/chat/common/promptSyntax/languageProviders/promptValidator.ts` (the unknown-attribute check at ~line 293–327).
