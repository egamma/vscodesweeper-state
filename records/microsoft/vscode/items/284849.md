# Issue #284849 — Support for Custom Attributes in `.prompt.md` YAML Front Matter

[https://github.com/microsoft/vscode/issues/284849](https://github.com/microsoft/vscode/issues/284849)

## Summary

Valid feature request to allow custom YAML front matter attributes in .prompt.md files without triggering validation hints. Currently in Backlog with the chat-prompts label, properly triaged.

## Change summary

Custom/unknown YAML front matter attributes in .prompt.md files trigger 'not supported' validation hints, preventing clean use of organizational metadata.

## Best solution

Either suppress validation hints for unknown top-level attributes entirely (making the schema open-ended), or introduce a designated namespace/prefix (e.g. `x-` or `custom:`) that the validator explicitly skips. The change would be localized to `checkForInvalidArguments` in `src/vs/workbench/contrib/chat/common/promptSyntax/languageProviders/promptValidator.ts`.
