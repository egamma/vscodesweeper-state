# Issue #189161 — Opening the quick pick with a <path>@<symbol> pattern does not enter the symbol navigation mode

[https://github.com/microsoft/vscode/issues/189161](https://github.com/microsoft/vscode/issues/189161)

## Summary

Team-authored issue (CONTRIBUTOR); must keep open. The behavior was intentionally disabled to support files with `@` in paths (per bpasero's comment citing #93845). The request is to re-enable symbol navigation when the path portion matches a real file, which requires disambiguation logic.

## Change summary

Opening quick pick with a `<path>@<symbol>` pattern should enter symbol navigation mode when the path matches a file on disk.

## Best solution

Implement file-existence check in the quick-pick path parser: when the prefix before `@` resolves to a real file, switch to symbol navigation mode; otherwise treat the full string as a filename. Likely lives in `src/vs/workbench/contrib/search/browser/` or the quick-open file picker area. Needs product decision on disambiguation UX.
