# Issue #266421 — MCP.json no error checking in user config, works in local vscode/mcp.json

[https://github.com/microsoft/vscode/issues/266421](https://github.com/microsoft/vscode/issues/266421)

## Summary

Confirmed on current main: the user-level mcp.json lacks a JSON schema fileMatch entry, so it gets no validation squiggles. The workspace .vscode/mcp.json is covered but the user profile one is not.

## Change summary

User-level mcp.json does not receive JSON schema validation (error squiggles), while workspace .vscode/mcp.json does.

## Best solution

Add `{ "fileMatch": "%APP_SETTINGS_HOME%/mcp.json", "url": "vscode://schemas/mcp" }` (and the profiles variant) to `extensions/configuration-editing/package.json`'s `jsonValidation` array, mirroring how tasks.json is handled for the user location.
