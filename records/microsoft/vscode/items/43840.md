# Issue #43840 — [folding] Code folding does not respect multi-cursor

[https://github.com/microsoft/vscode/issues/43840](https://github.com/microsoft/vscode/issues/43840)

## Summary

The fold action already respects all cursors/selections on current main. The `FoldAction.invoke` calls `getSelectedLines` which maps all editor selections to their start line numbers and passes them to `setCollapseStateUp`. This multi-cursor support was introduced in commit 51f31b0d3ff (2016) and remains intact.

## Change summary

Code folding (Ctrl+Shift+[) should fold at all cursor positions when multiple cursors are active, not just the primary cursor.

## Best solution

Close as completed — the fold action already iterates over all selections via `getSelectedLines()` in `src/vs/editor/contrib/folding/browser/folding.ts` and applies the fold at each cursor position.
