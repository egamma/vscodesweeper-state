# Issue #284820 — Terminal suggest works with `docker-compose` but not `docker compose`

[https://github.com/microsoft/vscode/issues/284820](https://github.com/microsoft/vscode/issues/284820)

## Summary

Still open on main: the docker compose spec uses `loadSpec: "docker-compose"` but loadSpec support is intentionally commented out in the Fig parser integration. This is a known gap, not a bug — the TODO acknowledges it needs investigation.

## Change summary

Terminal suggest does not provide subcommand completions for `docker compose` because `loadSpec` resolution is disabled in the Fig parser integration.

## Best solution

Implement `loadSpec` resolution in `extensions/terminal-suggest/src/fig/autocomplete-parser/parseArguments.ts` so that subcommands like `docker compose` can delegate to another spec (here `docker-compose`). Alternatively, inline the docker-compose subcommands directly into the docker spec's `compose` entry.
