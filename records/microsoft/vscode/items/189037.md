# Issue #189037 — Allow extension link detection to run on exception popups

[https://github.com/microsoft/vscode/issues/189037](https://github.com/microsoft/vscode/issues/189037)

## Summary

Valid feature request: the debug exception widget's LinkDetector only handles file-path patterns and standard URIs — it has no integration point for extension-provided link providers (like terminal link providers) to resolve custom URI schemes such as `package:`. The gap still exists on current main. Low traction (0 upvotes over ~3 years), but the request is coherent and the area label is correct.

## Change summary

Allow extensions to contribute link detection for custom URI schemes (e.g. Dart's `package:`) in the debug exception popup widget.

## Best solution

Expose a link-provider extension point (or reuse the terminal link provider mechanism) in the debug exception widget so that extensions like Dart can resolve custom URI schemes to file paths. The implementation would touch `src/vs/workbench/contrib/debug/browser/exceptionWidget.ts` and `src/vs/workbench/contrib/debug/browser/linkDetector.ts`, and likely require a new or extended debug API surface. Given the low community demand, this could remain in Backlog until more upvotes arrive or a broader 'debug view link providers' initiative picks it up.
