# Issue #257972 — CLI upgrade requires manual restart of system service

[https://github.com/microsoft/vscode/issues/257972](https://github.com/microsoft/vscode/issues/257972)

## Summary

Both reported problems — the manual restart requirement and the non-persisting 'always update' prompt — are superseded by CLI-driven server upgrades shipped in 1.121.0. The CLI now exposes a control socket that the client uses to trigger an upgrade and automatic restart without manual intervention.

## Change summary

After a remote tunnel CLI update, VS Code cannot connect to the new version without a manual `systemctl restart` of the service.

## Best solution

Keep the shipped fix (commit 56fe5495850, 1.121.0): the CLI-driven upgrade flow handles download, restart, and reconnection automatically. No further change needed.
