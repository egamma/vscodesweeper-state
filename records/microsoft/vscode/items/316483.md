# Issue #316483 — Agent terminal history does not respect checkpoints

[https://github.com/microsoft/vscode/issues/316483](https://github.com/microsoft/vscode/issues/316483)

## Summary

The checkpoint restore implementation for the Claude agent host (Phase 6.7, merged June 26 2026 in https://github.com/microsoft/vscode/commit/cd6804d7d8b) was landed after this issue was filed and handles conversation history truncation via the SDK's `resumeSessionAt` mechanism. However, the issue specifically describes terminal history in the agent's `<context>` block not being pruned on restore — it's unclear whether the new truncation fully addresses the context-block terminal history the reporter describes. The reporter confirmed the issue on the latest stable at the time but hasn't been asked to retest after the checkpoint restore implementation shipped. Keeping open pending confirmation.

## Change summary

After restoring to a checkpoint, the agent's context block still contains terminal command history from turns past the checkpoint, causing the agent to act on stale state.

## Best solution

Ask the reporter to retest on the latest Insiders/stable build (1.123+) which includes the checkpoint restore implementation (https://github.com/microsoft/vscode/commit/cd6804d7d8b). If the `<context>` block terminal history is now correctly truncated, close as fixed. If the context block still leaks post-checkpoint terminal history, the fix likely needs to be in the Copilot extension's context-building logic (`extensions/copilot/src/extension/chatSessions/`) to respect the truncation boundary when assembling the `<context>` block.
