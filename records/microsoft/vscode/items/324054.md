# Issue #324054 — Copilot agent continues after file edit before user selects Keep or Undo

[https://github.com/microsoft/vscode/issues/324054](https://github.com/microsoft/vscode/issues/324054)

## Summary

This is a feature request, not a bug — the current editing architecture intentionally applies edits optimistically and presents Keep/Undo for post-hoc review while the agent continues working. The reporter is requesting a blocking-confirmation mode where the agent pauses until the user resolves each edit.

## Change summary

Request the Copilot agent to pause after each file edit until the user selects Keep or Undo, rather than continuing to run while edits are pending review.

## Best solution

If the team wants to support this, a setting or mode could gate agent continuation on the user resolving pending Keep/Undo states. The editing session architecture (`chatEditingModifiedFileEntry.ts`, `chatEditingSession.ts`) already tracks `ModifiedFileEntryState` (Modified/Accepted/Rejected), so a blocking gate could check that state before signaling the agent host to proceed. However, this would fundamentally change the agent's throughput model — the current design (apply edits, let the user review after) is deliberate and allows the agent to complete multi-file tasks efficiently. Routing to the copilot area for a product decision.
