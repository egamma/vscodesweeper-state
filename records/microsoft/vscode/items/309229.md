# Issue #309229 — Allow capturing stderr from DebugAdapterExecutable and routing it to an OutputChannel

[https://github.com/microsoft/vscode/issues/309229](https://github.com/microsoft/vscode/issues/309229)

## Summary

Valid feature request: stderr from DebugAdapterExecutable is explicitly discarded on current main (line 271 of debugAdapter.ts calls `.resume()` with no capture). The prior issues were closed without implementing this capability. The issue is in Backlog Candidates with 0 upvotes and 60 days for the community process — let it run.

## Change summary

Allow extensions to optionally capture stderr from a DebugAdapterExecutable process and route it to an OutputChannel.

## Best solution

Add an optional `stderr` property to `DebugAdapterExecutableOptions` (in `vscode.d.ts`) accepting an `OutputChannel`, and wire it in `ExecutableDebugAdapter.startSession()` (src/vs/workbench/contrib/debug/node/debugAdapter.ts) to pipe stderr to the provided channel instead of discarding it. This is an API addition that needs a proposal cycle.
