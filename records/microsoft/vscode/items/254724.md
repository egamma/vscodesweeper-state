## Summary

Valid open bug — shell integration's OSC-633 sequence parsing fails to detect command completion for subshell/parenthesized command patterns; not fixed on current main, already triaged with assignee and labels.

## Change summary

Shell integration hangs or returns no output when commands are wrapped in subshell parentheses like `(echo a ; sleep 1)`.

## Best solution

Fix the OSC-633 parser in the shell integration addon (likely `shellIntegrationAddon.ts` and the capability handler) to correctly handle command boundaries when subshells or parenthesized groups are used.
