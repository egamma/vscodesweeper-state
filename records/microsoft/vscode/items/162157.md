# Issue #162157 — Search tree - selected item should be kept in the viewport

[https://github.com/microsoft/vscode/issues/162157](https://github.com/microsoft/vscode/issues/162157)

## Summary

Still reproduces on main: `setTreeView` in searchView.ts toggles the layout and refreshes the tree but never reveals the focused/selected element afterward. Kept open — author is a team member.

## Change summary

When switching search results from list view to tree view, the currently selected item is not scrolled into the viewport.

## Best solution

After `refreshTreeController.queue()` resolves in `setTreeView`, retrieve the tree's current selection/focus and call `tree.reveal()` on the first selected element so it scrolls into view. The fix is localized to `src/vs/workbench/contrib/search/browser/searchView.ts`.
