## Change summary

When switching search results from list view to tree view, the previously selected item is not scrolled into the viewport.

## Best solution

When the search view switches between list and tree representations, call `tree.reveal()` on the first selected element so it remains visible in the viewport. The fix likely lives in the search view's view-mode toggle handler in `src/vs/workbench/contrib/search/browser/`.
