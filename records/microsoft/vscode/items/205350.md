# Issue #205350 — Git - Warn Unsaved Files Before Typing Commit Message

[https://github.com/microsoft/vscode/issues/205350](https://github.com/microsoft/vscode/issues/205350)

## Summary

This is a feature request (filed via the feature-request template) asking for a proactive unsaved-files indicator before the user types a commit message, rather than a modal warning at commit time. Currently labeled `bug` but should be `feature-request`. The existing code in `smartCommit` already checks unsaved files before consuming the commit message, so the SCM input box text is preserved if the user cancels — the real ask is for earlier, pre-typing feedback.

## Change summary

Show an unsaved-files warning or indicator when the user starts typing a commit message in the SCM input box, rather than only at commit time.

## Best solution

Reclassify as `feature-request`. A reasonable implementation would show an inline warning decoration or status in the SCM input area when dirty documents exist in the repository, likely in `extensions/git/src/repository.ts` or the SCM view contribution. The comment about autosave not triggering on focus-switch to the commit input is a separate (arguably more impactful) concern that could be addressed independently. Low engagement (2 upvotes) suggests this can remain in the backlog for now.
