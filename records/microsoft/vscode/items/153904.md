# Issue #153904 — Searching `.` in the command palette matches `:`, searching for multiple `.` shifts the words around

[https://github.com/microsoft/vscode/issues/153904](https://github.com/microsoft/vscode/issues/153904)

## Summary

Still reproduces on main: `.` is treated as equivalent to all other word separators (`:`, space, etc.) in the fuzzy filter, so searching `.` in the command palette matches commands with `:` or spaces. The related PR #154320 was closed without merging. The behavior is intentional for word-boundary scoring but the equivalence-class treatment causes confusing filtering.

## Change summary

Typing `.` in the command palette matches `:` and other separators instead of literal dots, and highlighting shifts words when typing `...`.

## Best solution

Revisit `charactersMatch` in `src/vs/base/common/filters.ts` (line 158–160) to decide whether `.` should remain in the separator equivalence class for the quick-open filter, or whether literal matching should take priority when the pattern character is itself a separator. The highlight-trimming aspect (comment 2) also needs a fix in label rendering.
