# Issue #317345 — Bad hiding behavior of pickers in toolbar

[https://github.com/microsoft/vscode/issues/317345](https://github.com/microsoft/vscode/issues/317345)

## Summary

Team-filed visual bug: the agent-host permission picker in `ChatInputSecondary` disappears too early on sidebar resize, likely due to the responsive overflow logic treating it differently from other pickers despite sharing the same toolbar. Still open on main — no fix found.

## Change summary

When resizing the sidebar, the agent-host permission picker in the chat input toolbar hides prematurely while other pickers resize correctly.

## Best solution

Investigate the responsive overflow behavior in `chatInputPart.ts` for the `ChatInputSecondary` toolbar (around line 3536). The `agentHostShortPickerMinWidths` map and `getActionMinWidth` callback likely need tuning for the permission picker — it may be missing from the map or reporting the wrong min-width, causing the responsive layout to overflow it earlier than peers. The fix is localized to the toolbar overflow config and/or `permissionPickerActionItem.ts`.
