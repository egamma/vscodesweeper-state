# Issue #289902 — Reviewing TypeScript changes from a Cloud sessions mixes up the language server

[https://github.com/microsoft/vscode/issues/289902](https://github.com/microsoft/vscode/issues/289902)

## Summary

Still open and unfixed on main. The TypeScript language server serves stale on-disk content instead of the cloud session's modified content when reviewing changes via the Working Set, causing incorrect word highlighting and likely other language features. No related fix commit found.

## Change summary

TypeScript language server uses disk file content instead of cloud session modifications when reviewing changes, producing wrong word highlights.

## Best solution

The TS language service (likely `bufferSyncSupport.ts` in `extensions/typescript-language-features/`) needs to be aware of the virtual/modified document content provided by cloud sessions when the Working Set diff view is active, so it serves language features against the correct file version. Needs investigation by the TypeScript extension and cloud-changes area owners.
