# Issue #318701 — chat.instructionsFilesLocations: applyTo frontmatter changes not reflected until VS Code restart (stale cache)

[https://github.com/microsoft/vscode/issues/318701](https://github.com/microsoft/vscode/issues/318701)

## Summary

The report describes a plausible stale-cache bug in custom chat instructions: external instruction files are discovered through `chat.instructionsFilesLocations`, but updated frontmatter is not reflected in new chat sessions until restart. Keep open for chat area investigation rather than closing as extension-caused, because the settings and prompt/instructions file cache are implemented in VS Code core.

## Change summary

Changing an instruction file's `applyTo` frontmatter in a configured external instructions location is not reflected in new chat sessions until VS Code is restarted.

## Best solution

The chat prompt/instructions file service should invalidate or refresh cached metadata when `.instructions.md` files under configured `chat.instructionsFilesLocations` change, including external user locations, so new chat sessions use updated `applyTo` frontmatter without restart.
