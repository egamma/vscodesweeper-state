# Issue #307149 — Skill triggers external file allow dialog

[https://github.com/microsoft/vscode/issues/307149](https://github.com/microsoft/vscode/issues/307149)

## Summary

Team-authored issue (MEMBER) — must keep open per policy. Reports that skill file reads trigger an external file allow dialog unexpectedly when referencing files outside the workspace.

## Change summary

When a skill reads files outside the workspace (e.g. from ~/.copilot/skills/), the second and subsequent file reads trigger an external file access confirmation dialog.

## Best solution

The file access trust/allowlist logic should recognize skill-referenced files (or the ~/.copilot/skills/ path) as implicitly trusted when triggered by a skill invocation, avoiding repeated confirmation dialogs for each file reference within a skill.
