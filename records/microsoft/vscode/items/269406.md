# Issue #269406 — Automatic synchronization with Windows High Contrast mode

[https://github.com/microsoft/vscode/issues/269406](https://github.com/microsoft/vscode/issues/269406)

## Summary

The bug is confirmed on current main: the `onDidChangeColorScheme` listener in `workbenchThemeService.ts` only triggers theme restoration when `autoDetectColorScheme` is enabled, but ignores the separate `autoDetectHighContrast` setting, so toggling Windows HC mode at runtime doesn't switch the theme.

## Change summary

VS Code does not automatically switch to high contrast theme when Windows High Contrast mode is toggled at runtime via Alt+Shift+PrintScreen.

## Best solution

Fix `installPreferredSchemeListener` in `workbenchThemeService.ts` to also call `restoreColorTheme()` when `window.autoDetectHighContrast` is enabled and the host color scheme changes to/from high contrast. The condition at line 453 should be widened to `if (this.settings.isDetectingColorScheme() || this.settings.getPreferredColorScheme() !== undefined)`  or a similar check that covers the HC path.
