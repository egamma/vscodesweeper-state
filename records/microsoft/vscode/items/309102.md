# Issue #309102 — agents app: Changes view should be able to consistently compute against remote default branch's

[https://github.com/microsoft/vscode/issues/309102](https://github.com/microsoft/vscode/issues/309102)

## Summary

Team-authored issue (MEMBER) — must keep open. The Changes view in the agents app computes diffs via `git diff --merge-base` against the local feature branch, which produces wildly inflated change counts after a rebase against origin/main. The issue requests comparing against the remote default branch instead. The changeset infrastructure in `agentHostSessionChangesets.ts` surfaces changesets from the agent host but does not appear to have logic to rebase the diff base against the remote default branch.

## Change summary

After rebasing a feature branch in an agent session, the Changes view shows inflated diff counts because it computes against the local branch's merge-base rather than the remote default branch.

## Best solution

The changeset computation (likely in the agent host protocol layer or `agentHostSessionChangesets.ts` / `agentHostDiffs.ts`) should offer a mode — or default — that diffs the session's HEAD against the remote repository's default branch (available via `ISessionGitHubRepository.defaultBranch`), matching what GitHub shows on the PR. This may require fetching `origin/<defaultBranch>` as the diff base when constructing the 'Branch Changes' changeset.
