# Issue #189802 — [json] validation not working for a large enough number

[https://github.com/microsoft/vscode/issues/189802](https://github.com/microsoft/vscode/issues/189802)

## Summary

Still reproduces on main. The root cause is in vscode-json-languageservice: JavaScript loses precision for numbers beyond Number.MAX_SAFE_INTEGER (~16 digits), so the multipleOf validation silently stops working. The maintainer acknowledged this and suggested a warning, but no fix has landed.

## Change summary

JSON schema multipleOf validation silently stops working for numbers exceeding 16 digits due to JavaScript floating-point precision limits.

## Best solution

Implement a warning in vscode-json-languageservice when a number exceeds Number.MAX_SAFE_INTEGER, informing the user that validation may be unreliable. The fix belongs in the upstream vscode-json-languageservice package (microsoft/vscode-json-languageservice), then consumed here.
