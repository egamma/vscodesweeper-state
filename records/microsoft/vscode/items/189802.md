# Issue #189802 — [json] validation not working for a large enough number

[https://github.com/microsoft/vscode/issues/189802](https://github.com/microsoft/vscode/issues/189802)

## Summary

Still open — no fix or warning was implemented. The JSON validation silently produces incorrect results for numbers exceeding JavaScript's safe integer range (2^53). The team acknowledged it and suggested a warning, but no change landed.

## Change summary

JSON schema validation (e.g. `multipleOf`) silently stops working for numbers with 16+ digits due to JavaScript floating-point precision limits.

## Best solution

Implement a diagnostic warning in `vscode-json-languageservice` (upstream) when a JSON number exceeds `Number.MAX_SAFE_INTEGER`, so users know the validation result may be unreliable. The fix belongs in the upstream library (https://github.com/microsoft/vscode-json-languageservice), not in the VS Code repo itself.
