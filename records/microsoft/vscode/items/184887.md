# Issue #184887 — Does the `require("fs")` in the VSCode extension code turn into `require("graceful-fs")` when using SSH remote?

[https://github.com/microsoft/vscode/issues/184887](https://github.com/microsoft/vscode/issues/184887)

## Summary

This appears fixed by the remote dependency update that landed shortly after the report; current main no longer carries the old remote graceful-fs version implicated by the issue.

## Change summary

Remote extension hosts running prettier-vscode failed when graceful-fs mishandled fs.statSync with throwIfNoEntry: false.

## Best solution

Treat this as completed by commit 329cfd399977b30a36d79bb0031b37a2426db694, which updated the remote graceful-fs dependency to 4.2.11 and shipped starting with 1.81.0.
