# Issue #152250 — SCM - Add drag and drop support within the Source Control views

[https://github.com/microsoft/vscode/issues/152250](https://github.com/microsoft/vscode/issues/152250)

## Summary

Still valid: the SCM view's drag-and-drop handler (`SCMTreeDragAndDrop.drop()`) is an empty no-op on current main, and no extension API exposes SCM drag-and-drop to providers. The feature has strong community support (45 upvotes) and is correctly in the Backlog.

## Change summary

Allow dragging files between SCM resource groups (e.g. staged ↔ unstaged, or between revisions for extension-contributed SCM providers) via drag and drop in the Source Control view.

## Best solution

Implement the `SCMTreeDragAndDrop.drop()` handler in `src/vs/workbench/contrib/scm/browser/scmViewPane.ts` to support intra-view drops on resource groups, and expose a corresponding extension API (e.g. `SourceControlResourceGroup.onDrop`) so SCM providers like Perforce can handle cross-group moves. The built-in Git provider could wire this to stage/unstage operations.
