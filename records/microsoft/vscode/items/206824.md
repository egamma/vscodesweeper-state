## Summary

The underlying HOMESHARE/network-home issue is a niche Windows enterprise configuration problem that hasn't been confirmed to still reproduce on a current build, and needs clarification on whether setting the HOME env var resolves it.

## Change summary

On Windows with a network home drive (HOMESHARE), VS Code's git integration fails to find .gitconfig and .ssh from the network path, falling back to USERPROFILE instead.

## Best solution

The reporter should test on a current VS Code version (the askpass crash is fixed). If the HOMESHARE resolution issue persists, the fix would likely be setting `HOME` in the git spawn environment to match what Git for Windows expects, or documenting a workaround (set HOME env var system-wide). This may also be *as-designed* since git's home resolution is git's responsibility and VS Code passes the environment through.
