# Issue #153095 — Support multline details in quickpick items

[https://github.com/microsoft/vscode/issues/153095](https://github.com/microsoft/vscode/issues/153095)

## Summary

This feature request for multiline detail text in quick pick items remains unimplemented on current main — the detail field is still rendered as a single truncated line. It crossed the 20-upvote threshold, was accepted into Backlog, and a maintainer welcomed a PR; keeping it open.

## Change summary

Allow line breaks in the `detail` field of quick pick items so extensions can surface more than two lines of information.

## Best solution

Implement multiline detail support in `src/vs/platform/quickinput/browser/quickInputList.ts` (rendering) and the associated CSS in `src/vs/platform/quickinput/browser/media/quickInput.css`, likely by allowing `white-space: pre-wrap` on the detail element and adjusting the virtual-list row height calculation. The extension API type in `vscode.d.ts` (`QuickPickItem.detail`) already accepts a string — it just needs the renderer to honor embedded newlines. A maintainer welcomed a PR in 2022; this is ready for implementation.
