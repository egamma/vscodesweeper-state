# Issue #208289 — MultiProvider is not supported for registerDebugConfigurationProvider Dynamic

[https://github.com/microsoft/vscode/issues/208289](https://github.com/microsoft/vscode/issues/208289)

## Summary

Confirmed on current main: `getDynamicProviders()` (line 238) and `getDynamicConfigurationsByType()` (line 295) both use `.find()` instead of `.filter()` for Dynamic providers, so only the first registered provider per debug type takes effect. The Initial provider path (line 193) correctly uses `.filter()`, making this an inconsistency/bug.

## Change summary

Multiple Dynamic DebugConfigurationProviders registered for the same debug type are silently discarded — only the first one is used.

## Best solution

Change the two `.find()` calls at lines 238 and 295 in `src/vs/workbench/contrib/debug/browser/debugConfigurationManager.ts` to `.filter()`, and aggregate the results from all matching providers (as the Initial path already does at line 193). The `getDynamicProviders` return shape and callers may need adjustment to handle multiple providers per type.
