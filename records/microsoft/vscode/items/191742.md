# Issue #191742 — Listening music in VS Code: "Chromium" in the volume app

[https://github.com/microsoft/vscode/issues/191742](https://github.com/microsoft/vscode/issues/191742)

## Summary

The upstream Electron issue (electron/electron#27581) that caused audio streams to show as 'Chromium' in the volume mixer was closed as completed in March 2026. VS Code now ships Electron 42.5.0, which includes the fix. This issue can be closed as the upstream root cause has been resolved.

## Change summary

Audio played through VS Code (e.g. via extensions) shows as 'Chromium' instead of 'Visual Studio Code' in the OS volume mixer on Linux.

## Best solution

No VS Code change needed — the fix landed upstream in Electron. The current Electron 42.5.0 bundled with VS Code should reflect the correct application name in PulseAudio/PipeWire. Users on the latest VS Code should see 'Visual Studio Code' in their volume mixer.
