# Issue #291796 — Changing Close File Behaviour - UX Issue

[https://github.com/microsoft/vscode/issues/291796](https://github.com/microsoft/vscode/issues/291796)

## Summary

The requested behavior is already available via the existing `explorer.autoReveal` setting — setting it to `focusNoScroll` (or `false`) prevents the Explorer from scrolling to reveal the newly-focused file when a tab is closed.

## Change summary

When closing a file tab, the Explorer scrolls to reveal the next active file, losing the user's scroll position in large projects.

## Best solution

This is addressed by the existing `explorer.autoReveal` setting. Setting it to `"focusNoScroll"` keeps the Explorer from scrolling when the active editor changes (including on tab close), while still focusing the file. Setting it to `false` disables auto-reveal entirely. No code change is needed.
