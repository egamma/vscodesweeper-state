# Issue #292114 — Copilot Chat workbench.action.chat.retry shortcut key doesn't work

[https://github.com/microsoft/vscode/issues/292114](https://github.com/microsoft/vscode/issues/292114)

## Summary

The `workbench.action.chat.retry` command requires a response view model as its first argument (passed by the menu context), but when invoked via keyboard shortcut or programmatically with no arguments, it silently returns at the `isResponseVM(item)` guard. This is a real bug traceable in the current source.

## Change summary

Invoking `workbench.action.chat.retry` via a keyboard shortcut or programmatically does nothing because the command handler expects a response VM argument that only the menu button provides.

## Best solution

Add a fallback in `RetryChatAction.run()` (in `src/vs/workbench/contrib/chat/browser/actions/chatTitleActions.ts`) so that when no argument is provided, it resolves the last response from the currently focused chat widget (via `chatWidgetService.lastFocusedWidget?.viewModel?.getItems().at(-1)`). The third comment about `nextUserPrompt` may be a separate issue — that command's implementation already uses `lastFocusedWidget` and should work; it may need a separate investigation.
