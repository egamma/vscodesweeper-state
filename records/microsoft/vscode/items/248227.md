# Issue #248227 — PHP Autoindent is stupid

[https://github.com/microsoft/vscode/issues/248227](https://github.com/microsoft/vscode/issues/248227)

## Summary

Still reproduces on main. The current PHP indentation rules (simplified in commit 8d2287f6f7d for #248229) removed brace/bracket logic but still cannot detect that a semicolon terminates a multi-line expression requiring de-indent. The `tree-sitter-autoindent-candidate` label is appropriate — regex rules alone cannot solve this.

## Change summary

After a multi-line string concatenation ending with `;`, pressing Enter keeps the continuation indent instead of returning to the base indent level.

## Best solution

This needs tree-sitter-based autoindent (already labeled `tree-sitter-autoindent-candidate`). A regex `decreaseIndentPattern` cannot reliably detect that `;` terminates a multi-line expression started at a lower indent. Keep open until tree-sitter autoindent ships for PHP.
