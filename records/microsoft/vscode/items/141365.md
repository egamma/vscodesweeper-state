# Issue #141365 — Wrong keybind is used with colemak & russian layout

[https://github.com/microsoft/vscode/issues/141365](https://github.com/microsoft/vscode/issues/141365)

## Summary

Keep this open: it is a Backlog keyboard-layout bug with later community confirmations and no concrete fixing commit or canonical duplicate found. The current code still contains the Linux/macOS fallback path that maps missing Latin letters to QWERTY scan codes, which is directly relevant to the reported Colemak plus Russian shortcut mismatch.

## Change summary

Ctrl shortcuts on Colemak can resolve as QWERTY shortcuts after switching to Russian or some other non-Latin layouts, causing the wrong command to run.

## Best solution

Keep this in the keyboard layout backlog for the keybinding owner to decide whether Linux layout fallback should preserve the user's configured Latin layout when switching to affected non-Latin layouts. Any fix should include regression coverage around `src/vs/workbench/services/keybinding/common/macLinuxKeyboardMapper.ts` and layout-change handling in `src/vs/workbench/services/keybinding/electron-browser/nativeKeyboardLayoutService.ts`.
