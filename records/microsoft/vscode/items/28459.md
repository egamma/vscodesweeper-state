# Issue #28459 — [css] Use CSS custom properties imported via @import

[https://github.com/microsoft/vscode/issues/28459](https://github.com/microsoft/vscode/issues/28459)

## Summary

This long-standing feature request for cross-file CSS custom property intellisense remains unimplemented. It has strong community interest and existing extension workarounds, and sits in the Backlog milestone appropriately.

## Change summary

VS Code's built-in CSS language service does not provide intellisense for CSS custom properties declared in files imported via @import.

## Best solution

Implement cross-file CSS custom property resolution in the upstream vscode-css-languageservice (https://github.com/microsoft/vscode-css-languageservice/issues/156) so that @import-ed files contribute their custom properties to completion and hover. Until then, extensions like vscode-css-variables fill the gap.
