# Issue #302155 — Confusing names/slash commands

[https://github.com/microsoft/vscode/issues/302155](https://github.com/microsoft/vscode/issues/302155)

## Summary

Team-authored polish issue; the inconsistent naming and redundant slash commands (`/yolo`, `/autoApprove`, `/disableYolo`, `/disableAutoApprove`, `/autopilot`) remain on current `main` in `chatSlashCommands.ts`. Hard keep-open due to MEMBER authorship.

## Change summary

The approval-bypass slash commands use multiple inconsistent names (`/yolo`, `/autoApprove`) and redundant disable variants, making the feature confusing to discover and use.

## Best solution

Consolidate the auto-approve slash commands in `src/vs/workbench/contrib/chat/browser/chatSlashCommands.ts` to a single canonical name (e.g. `/autoApprove` and `/disableAutoApprove`), keeping `/yolo` as a hidden alias at most. Also gate `/disableYolo`/`/disableAutoApprove` visibility on whether auto-approve mode is actually active, as the author noted.
