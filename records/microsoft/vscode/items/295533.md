# Issue #295533 — `"chat.editing.explainChanges.enabled": true,` the tooltips still visible after i click on keep button.

[https://github.com/microsoft/vscode/issues/295533](https://github.com/microsoft/vscode/issues/295533)

## Summary

Still reproduces on main: the `accept()` method in `chatEditingSession.ts` does not call `clearExplanations()`, so explanation tooltips persist after clicking Keep. The bug is real, localized, and unfixed.

## Change summary

Explanation tooltips from the experimental `chat.editing.explainChanges.enabled` feature persist after the user clicks Keep to accept changes, and stale tooltips reappear in unrelated files on subsequent edits.

## Best solution

Add a `this.clearExplanations()` call inside the `accept()` method (and likely `reject()` too) in `src/vs/workbench/contrib/chat/browser/chatEditing/chatEditingSession.ts`, so that explanations are dismissed when all entries are resolved. Also clear per-file explanation state in the explanation model manager to prevent stale tooltips from reappearing in subsequent edits.
