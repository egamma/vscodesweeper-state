# Issue #181392 — Git - Doesn't show changed symbol for folder in Git when selected

[https://github.com/microsoft/vscode/issues/181392](https://github.com/microsoft/vscode/issues/181392)

## Summary

Valid polish issue: git decoration colors on folders become invisible when the item is focused+selected in the Explorer, because `list.activeSelectionForeground` (white) overrides the decoration foreground. Still present on current main — the selection foreground override is unchanged.

## Change summary

Git modification decoration on folders is not visible when the folder is selected in the Explorer due to the selection foreground color overriding the decoration color.

## Best solution

Preserve file-decoration foreground colors on focused+selected list items instead of forcing them all to `list.activeSelectionForeground`. The fix would likely live in the list/tree widget CSS or the decoration style computation in `src/vs/workbench/services/decorations/browser/decorationsService.ts` and the list-widget styling in `src/vs/platform/theme/common/colors/listColors.ts`. Alternatively, the existing `list.activeSelectionForeground` could be made to not override decoration-specific foreground, similar to how icon foreground is handled separately.
