# Issue #279707 — SCM Graph - Control / Shift to select multiple commits.

[https://github.com/microsoft/vscode/issues/279707](https://github.com/microsoft/vscode/issues/279707)

## Summary

Valid feature request — the SCM history graph tree explicitly disables multi-selection (`multipleSelectionSupport: false`). The request to Ctrl/Shift-select multiple commits for drag-and-drop into chat is reasonable and not yet implemented. Keeping open in the Backlog for the area owner to prioritize.

## Change summary

Allow Ctrl/Shift multi-selection of commits in the SCM Graph view so multiple commits can be dragged into chat at once.

## Best solution

Enable `multipleSelectionSupport` in the SCM history tree (line ~2017 in `scmHistoryViewPane.ts`) and wire up the drag-and-drop provider (`SCMHistoryTreeDragAndDrop`) to handle multi-selection payloads. The area owner should decide whether multi-select should be on by default or gated behind the existing `selectionModeConfig` pattern used in `scmRepositoriesViewPane.ts`.
