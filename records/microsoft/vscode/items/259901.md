# Issue #259901 — Assisted MCP server installation should support custom package sources

[https://github.com/microsoft/vscode/issues/259901](https://github.com/microsoft/vscode/issues/259901)

## Summary

Team-authored feature request with active exploration already underway (PR #262558 for NPM). Must keep open per hard keep-open rule (MEMBER author) and because the work is in progress.

## Change summary

MCP server assisted installation should use native package manager CLIs and respect custom package sources (Azure Artifacts, GitHub Package Registry, etc.) rather than hardcoding public registry HTTP calls.

## Best solution

Continue the approach started in https://github.com/microsoft/vscode/pull/262558 — replace direct HTTP registry calls with native CLI invocations (npm, dotnet, etc.) across all supported package ecosystems, and default MCP server cwd to the workspace directory so local package config (.npmrc, NuGet.config) is respected.
