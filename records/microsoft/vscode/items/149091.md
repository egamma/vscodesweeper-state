# Issue #149091 — SCM - Sort by path does not work as expected for submodules

[https://github.com/microsoft/vscode/issues/149091](https://github.com/microsoft/vscode/issues/149091)

## Summary

Still reproduces on current main: `comparePaths` in `comparers.ts` short-circuits on path-length differences before comparing the current segment, so shorter directory paths always sort before longer ones regardless of alphabetical order. SCM sort-by-path for repositories with submodules is broken.

## Change summary

SCM 'sort by path' for repositories mistreats submodule directory paths because `comparePaths` gives shorter paths priority over alphabetical segment order.

## Best solution

The SCM repository sort (in `scmViewService.ts` line 529) should not use `comparePaths` as-is, since that function is designed for file-explorer paths where files sort before subdirectories. Either add a `directoryOnly` mode to `comparePaths` that compares the current segment before falling back to length, or use a simple segment-by-segment string comparison for repository root URIs.
