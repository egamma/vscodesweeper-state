# Issue #313339 — Agents Mobile: "Use AI Features" Button Shown in Accounts Menu When Already Signed In

[https://github.com/microsoft/vscode/issues/313339](https://github.com/microsoft/vscode/issues/313339)

## Summary

Team-authored bug report (MEMBER) — must keep open. The "Use AI Features" button in the Accounts menu has a `when` clause gating on `ChatContextKeys.Entitlement.signedOut`, but on the mobile Agents surface the entitlement state may not resolve correctly when already signed in, causing the button to appear spuriously.

## Change summary

The Accounts menu shows a "Use AI Features" button even when the user is already signed in to GitHub and using AI features, specifically in the Agents Mobile experience.

## Best solution

Investigate why the `signedOut` context key (set in `chatEntitlementService.ts` line ~1465, based on `ChatEntitlement.Unknown`) evaluates as true in the Agents Mobile context despite the user being authenticated. The `ChatSetupFromAccountsAction` when-clause at `chatSetupContributions.ts:371-377` should correctly hide the button. The fix likely involves ensuring entitlement resolution completes on the mobile surface before the accounts menu renders, or adjusting the context key evaluation for the sessions/mobile app shell.
