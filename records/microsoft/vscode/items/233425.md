# Issue #233425 — Issue reporter fails without telling the user why if content is too large

[https://github.com/microsoft/vscode/issues/233425](https://github.com/microsoft/vscode/issues/233425)

## Summary

Still reproduces on main: the electron-browser issue reporter silently returns false with only a console.error when the issue body exceeds MAX_GITHUB_API_LENGTH and there's no extension data to strip. The user gets no feedback about why submission failed.

## Change summary

Issue reporter fails silently (no user-facing message) when the issue body is too large to submit to GitHub and there is no extension data to remove.

## Best solution

Show a user-facing notification (dialog or inline error) in the else branch at line 150 of issueReporterService.ts explaining that the content is too large and suggesting the user reduce it. The web path in issueFormService.ts already handles this more gracefully with clipboard fallback — consider bringing similar UX to the electron path.
