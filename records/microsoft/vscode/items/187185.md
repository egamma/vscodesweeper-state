# Issue #187185 — Shell integration breaks prompt when using Fish shell with Tide prompt

[https://github.com/microsoft/vscode/issues/187185](https://github.com/microsoft/vscode/issues/187185)

## Summary

Still reproduces on current main: the `functions --copy fish_prompt __vsc_fish_prompt` approach (line 157 of shellIntegration.fish) still loses file-scoped variables and event handlers, breaking Tide, Hydro, and other multi-file Fish prompts. The `help wanted` label is appropriate — this needs a contributor.

## Change summary

VS Code's Fish shell integration copies the user's `fish_prompt` function via `functions --copy`, which discards file-local variables and event bindings, breaking prompts like Tide and Hydro.

## Best solution

Replace the `functions --copy` approach with one that preserves the original function's execution context — e.g. calling the original `fish_prompt` in-place via `source` or wrapping with event hooks rather than copying the function body. The fix lives in `src/vs/workbench/contrib/terminal/common/scripts/shellIntegration.fish` around line 150–250.
