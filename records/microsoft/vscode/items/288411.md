# Issue #288411 — Skipped tool status needs to be represented in thinking blocks

[https://github.com/microsoft/vscode/issues/288411](https://github.com/microsoft/vscode/issues/288411)

## Summary

Still present on main: the thinking content part (chatThinkingContentPart.ts) does not check for ToolConfirmKind.Skipped when rendering tool status. Skipped tools display their pastTenseMessage (e.g. "Ran") instead of a "Skipped" label with a circleSlash icon, unlike the standalone tool invocation sub-part which handles this correctly.

## Change summary

When a tool invocation is skipped inside a thinking block, the UI incorrectly shows "Ran" instead of indicating the tool was skipped.

## Best solution

Add skipped-state handling in chatThinkingContentPart.ts mirroring what chatToolInvocationSubPart.ts already does: check IChatToolInvocation.executionConfirmedOrDenied for ToolConfirmKind.Skipped, update the icon to Codicon.circleSlash, and display a "Skipped" label instead of the pastTenseMessage.
