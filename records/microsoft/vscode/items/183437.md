# Issue #183437 — Mouse wheel scroll does not respect windows mouse settings

[https://github.com/microsoft/vscode/issues/183437](https://github.com/microsoft/vscode/issues/183437)

## Summary

Keep this open as a real Windows scrolling bug: the report has a second-user confirmation with version details, and current main still appears to normalize browser wheel deltas through VS Code's scroll handling rather than explicitly honoring Windows mouse-wheel preferences. There is no concrete fixing commit, duplicate, or reproduction attempt against current Windows that would support closing it.

## Change summary

Mouse wheel scrolling in VS Code on Windows should honor the system setting for multiple-lines-per-notch or one-screen-at-a-time scrolling.

## Best solution

Keep this in the scroll/input area and reproduce on current Windows with both 'multiple lines at a time' and 'one screen at a time' configured. If Chromium/Electron already emits OS-adjusted deltas, fix VS Code's handling in the StandardWheelEvent/DomScrollableElement path; otherwise track the upstream Electron/Chromium gap explicitly.
