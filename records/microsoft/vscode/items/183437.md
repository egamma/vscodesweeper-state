# Issue #183437 — Mouse wheel scroll does not respect windows mouse settings

[https://github.com/microsoft/vscode/issues/183437](https://github.com/microsoft/vscode/issues/183437)

## Summary

This is an upstream issue: Chromium/Electron normalizes wheel events without fully preserving the Windows system 'lines to scroll' setting. A maintainer on the related #263245 confirmed this would require Windows API/registry watching, making it likely out of scope for VS Code core without Electron-level changes.

## Change summary

Mouse wheel scrolling in VS Code ignores the Windows system setting for 'Lines to scroll at a time'.

## Best solution

This depends on Electron/Chromium properly propagating Windows scroll-line settings through wheel events. VS Code's `mouseEvent.ts` normalizes `wheelDeltaY / 120` which strips the OS multiplier. A fix would either require Electron to expose the system scroll-lines value, or VS Code to read it from the Windows registry — both are upstream or out-of-scope per maintainer assessment. The `editor.mouseWheelScrollSensitivity` setting serves as a manual workaround.
