# Issue #251039 — Second generation profile does not inherit "Default" keybindings of parent

[https://github.com/microsoft/vscode/issues/251039](https://github.com/microsoft/vscode/issues/251039)

## Summary

Still reproducible on main: when Profile A uses Default's keybindings via `useDefaultFlags.keybindings = true`, copying from Profile A to create Profile B fails to propagate either the flag or the content, leaving Profile B with broken keybindings.

## Change summary

Creating a profile by copying from a parent that inherits keybindings from Default results in the new profile having no valid keybindings.

## Best solution

In `createFromProfile`, when the source profile has `useDefaultFlags` set for a resource, the copy-from flow should either propagate that flag to the new profile or correctly read the default profile's content. The fix likely belongs in `src/vs/workbench/services/userDataProfile/browser/userDataProfileImportExportService.ts` around the `createFromProfile` method or in `src/vs/workbench/contrib/userDataProfile/browser/userDataProfilesEditorModel.ts` in the new-profile creation UI flow.
