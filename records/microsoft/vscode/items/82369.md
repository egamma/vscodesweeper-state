## Summary

The proxy infrastructure for extensions has been completely reworked since this 2019 report — VS Code now uses `@vscode/proxy-agent` (v0.43.0) which delegates to `https-proxy-agent` and `http-proxy-agent`, both of which handle HTTPS-over-HTTP CONNECT tunneling properly. However, without a single fixing commit to cite and no reporter confirmation, requesting verification on a current build before closing.

## Change summary

Extensions making HTTPS requests behind an HTTP proxy (HTTPS_PROXY=http://...) fail with 'socket hang up' because VS Code's extension proxy support did not tunnel HTTPS through HTTP.
