# Issue #321912 — Corporate Controls: Whitelist/Block Extensions by Tag

[https://github.com/microsoft/vscode/issues/321912](https://github.com/microsoft/vscode/issues/321912)

## Summary

Valid feature request to extend the `extensions.allowed` policy to support tag-based filtering. The current implementation in `allowedExtensionsService.ts` only matches by extension ID and publisher — no tag/category dimension exists. No canonical issue already owns this work.

## Change summary

Allow corporate administrators to whitelist or block extensions by marketplace tag (e.g. `language-models`) in the `extensions.allowed` policy, rather than only by individual extension ID or publisher.

## Best solution

Extend `AllowedExtensionsService.isAllowed()` in `src/vs/platform/extensionManagement/common/allowedExtensionsService.ts` to accept tag-based entries (e.g. `"tag:language-models": false`) alongside existing ID/publisher entries, and expose this as a policy-capable configuration. This would let enterprise admins block entire extension categories without maintaining individual extension lists.
