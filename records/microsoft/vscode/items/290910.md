This is a duplicate of the older issue #284211 — both report the same root cause: native dialog operations crash VS Code on Wayland. The crash is in Electron/Chromium's native dialog handling under Wayland, and the workaround (--ozone-platform=x11 or window.dialogStyle=custom) is the same for both.
