Valid polish suggestion: the 'View changes' button in the working set is still a one-way action (not a toggle). The chat editing UI has evolved with the Agents/Sessions architecture but this toggle behavior hasn't been implemented. Worth keeping open as a polish item for the area owner to evaluate against the current UI state.

Make the 'Show all changes' / per-file 'Show changes' buttons in the working set act as toggles — clicking again should dismiss the diff and show the original file.
