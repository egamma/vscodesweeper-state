# Issue #314121 — Copilot subagent model-selection blocked when reviewer model cost tier exceeds parent session (Claude Opus 4.7 vs GPT-5.5)

[https://github.com/microsoft/vscode/issues/314121](https://github.com/microsoft/vscode/issues/314121)

## Summary

The cost-tier capping of subagent model invocations is intentional and explicitly implemented in the codebase. Subagents are deliberately prevented from exceeding the parent session's cost tier. This is as-designed behavior, not a bug.

## Change summary

Copilot subagent model invocation is blocked when the requested model's cost tier exceeds the parent chat session's effective cost tier.

## Best solution

Close as designed. The multiplier constraint in `runSubagentTool.ts` intentionally caps subagent models to the parent session's tier. The workaround is to start the parent session with a model at or above the desired subagent's tier. A separate feature request could be filed for filtering tier-blocked models from the picker or for a 'dynamic tier elevation' capability, but the current enforcement is a deliberate cost-control guardrail.
