# Issue #208378 — Align review comments UX and diff buttons UX

[https://github.com/microsoft/vscode/issues/208378](https://github.com/microsoft/vscode/issues/208378)

## Summary

The UX alignment between comment gutter decorations and diff editor buttons remains unresolved on main. The border aspect was deliberately declined per #209627, but the animation/transition alignment was never completed due to a technical limitation (DOM removal prevents fade-out). The issue is still valid as a polish item.

## Change summary

Align the visual feedback (border width, hover animation) between review comment gutter buttons and diff editor gutter buttons.

## Best solution

The remaining gap is the hover fade-in/fade-out animation on comment gutter decorations. A CSS approach using `:hover` on the parent line element (rather than adding/removing DOM nodes) could unblock the transition. The fix would live in `src/vs/workbench/contrib/comments/browser/media/review.css` and possibly the decoration logic in the comment glyph widget. The border question was already resolved (no border).
