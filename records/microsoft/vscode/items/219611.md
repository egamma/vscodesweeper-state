# Issue #219611 — Git - Provide way to open the merge editor even if the "source control" pane doesn't load

[https://github.com/microsoft/vscode/issues/219611](https://github.com/microsoft/vscode/issues/219611)

## Summary

Team-authored feature request (MEMBER) — hard keep-open. The core ask (open the merge editor when the SCM pane can't load due to statusLimit) is still unaddressed on main; no code was added to detect conflicting files independently of git status or to offer the merge editor button based on conflict markers alone.

## Change summary

Provide a way to open the merge editor for conflicted files even when the Source Control pane fails to load due to exceeding the git.statusLimit threshold.

## Best solution

Either surface the 'Open in Merge Editor' codelens/button based on conflict markers detected in the editor (independent of the SCM pane), or use a lightweight git command like `git diff --name-only --diff-filter=U` to enumerate conflicting files when full `git status` is too expensive. The workaround (`git.statusLimit: 0`) exists but the decoupling would make the merge editor more robust for large merges.
