# Issue #285481 — code cli command create a new log folder each time when it launched

[https://github.com/microsoft/vscode/issues/285481](https://github.com/microsoft/vscode/issues/285481)

## Summary

Still reproduces on main: each CLI invocation (including background ones from terminal-suggest) eagerly creates a log folder via LoggerService in cliProcessMain.ts. An open PR (#288408) proposes delaying folder creation but has not yet merged.

## Change summary

Running the `code` CLI command creates a new empty log folder each time, cluttering the logs directory and rotating out useful logs prematurely.

## Best solution

Land or iterate on PR https://github.com/microsoft/vscode/pull/288408 which delays log folder creation until the primary instance is confirmed, preventing secondary/CLI instances from creating empty folders.
