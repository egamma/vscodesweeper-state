# Issue #171153 — [html] validation error on JavaScript block comment with arrow in HTML script

[https://github.com/microsoft/vscode/issues/171153](https://github.com/microsoft/vscode/issues/171153)

## Summary

Still reproduces on main: the `updateContent` function in `embeddedSupport.ts` naively replaces `-->` with ` */` even when it appears inside a JS block comment, producing invalid JS that triggers spurious diagnostics.

## Change summary

JavaScript block comments containing `-->` inside HTML `<script>` tags produce spurious 'Expression expected' and 'Unterminated regular expression' errors.

## Best solution

Replace the naive `content.replace('<!--', '/* ').replace('-->', ' */')` at line 200 of `embeddedSupport.ts` with logic that blanks out (whitespace-fills) only the actual HTML comment delimiters at script boundaries, rather than replacing all occurrences of those character sequences. The fix should handle multiple occurrences and avoid mangling `-->` that appears inside JS comments or regex literals.
