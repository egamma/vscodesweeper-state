# Issue #171153 — [html] validation error on JavaScript block comment with arrow in HTML script

[https://github.com/microsoft/vscode/issues/171153](https://github.com/microsoft/vscode/issues/171153)

## Summary

Still reproduces on main — the naive `replace('<!--', '/* ').replace('-->', ' */')` in embeddedSupport.ts produces invalid JS when `-->` appears inside a JS comment or regex. PR #240932 is open but unmerged. Milestone is 'On Deck', so keeping open.

## Change summary

JavaScript block comments containing `-->` inside HTML `<script>` tags produce spurious validation errors because the HTML-to-JS comment substitution is too simplistic.

## Best solution

Replace the naive string substitution in `extensions/html-language-features/server/src/modes/embeddedSupport.ts` (the `updateContent` function) with whitespace-blanking of `<!--` and `-->` tokens rather than converting them to `/* */`, avoiding interference with existing JS comments and regex literals. PR https://github.com/microsoft/vscode/pull/240932 is the active effort.
