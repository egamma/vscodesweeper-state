## Summary

Team-authored feature request for modifier+click to trigger step-in-target during debugging. Not yet implemented on main — the step-in-target flow is command/menu-driven only. Keeping open per hard keep-open rule (MEMBER author).

## Change summary

Add a modifier+click gesture in the editor during debugging to invoke step-in-target on the clicked call site.

## Best solution

Implement a modifier+click handler (likely in `src/vs/workbench/contrib/debug/browser/`) that, during an active debug session paused on a line, detects the clicked expression, resolves available step-in targets, and steps into the matching one. The existing `stepInTargets` session API in `debugSession.ts` and the target-selection UI in `debugEditorActions.ts` provide the foundation.
