# Issue #302102 — Sessions: File tree is not visible or discoverable on first use

[https://github.com/microsoft/vscode/issues/302102](https://github.com/microsoft/vscode/issues/302102)

## Summary

Team-authored UX issue from a user study — hard keep-open applies. Significant layout work has landed since filing (Files-vs-Changes default logic, responsive sidebar, session files widget), but the core discoverability concern (file tree not obvious on first use after clone) may still need explicit onboarding or layout defaults to be fully addressed.

## Change summary

In the Agents window, the file tree is not visible or discoverable after cloning a repository, blocking new users from verifying clone success or browsing project structure.

## Best solution

Ensure the file tree (or a clear pointer to it) is visible by default immediately after a clone completes in the Agents window — e.g. auto-reveal the Files sidebar or show a confirmation with a link to the file view. Recent commits (3a99d2cf92f, 18f683e0a77) improved the Files/Changes default logic but focus on *created* sessions; the first-use/post-clone moment may still need explicit treatment in the layout controller (`src/vs/sessions/contrib/layout/`).
