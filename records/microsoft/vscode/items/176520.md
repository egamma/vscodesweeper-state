# Issue #176520 — Support debug adapter server over WebSockets (similar to DebugAdapterServer)

[https://github.com/microsoft/vscode/issues/176520](https://github.com/microsoft/vscode/issues/176520)

## Summary

Valid but low-traction feature request for a built-in WebSocket debug adapter descriptor. A workable inline-implementation workaround exists (confirmed in comments), and no new API surface has been added on main. Keeping open per the team member's (roblourens) explicit decision to leave it open, but community traction is minimal (1 upvote).

## Change summary

Add a built-in DebugAdapterWebSocketServer descriptor (or stream-based variant) so debug adapters can connect over WebSockets without manually wrapping DebugAdapterInlineImplementation.

## Best solution

If demand grows, add a `DebugAdapterWebSocketServer` class alongside the existing `DebugAdapterServer` in `src/vs/workbench/api/common/extHostTypes.ts` and wire it through the debug adapter descriptor factory. Until then, the `DebugAdapterInlineImplementation` workaround shown in the comments is a viable path for extension authors.
