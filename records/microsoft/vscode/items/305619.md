# Issue #305619 — Copilot doesn't see python test discovery errors (pytest)

[https://github.com/microsoft/vscode/issues/305619](https://github.com/microsoft/vscode/issues/305619)

## Summary

Still valid on current main: the `RunTestTool` returns a generic 'No tests found' message when discovery fails (e.g. due to import errors), without surfacing the underlying discovery error to the agent. The tool has no mechanism to relay test-framework-level discovery diagnostics.

## Change summary

When pytest fails to discover tests due to import errors, the Copilot agent receives only 'No tests found' with no diagnostic about the discovery failure cause.

## Best solution

Enhance `RunTestTool` in `src/vs/workbench/contrib/testing/common/testingChatAgentTool.ts` to detect when discovery produced errors (e.g. via test item error states or diagnostics from the test controller) and include those error messages in the tool result so the agent can reason about broken imports.
