# Issue #319856 — Default Approvals do not prompt for subsequent send_to_terminal inputs after starting an interactive shell, allowing destructive commands without confirmation

[https://github.com/microsoft/vscode/issues/319856](https://github.com/microsoft/vscode/issues/319856)

## Summary

The reported behavior contradicts current main code, which gates send_to_terminal behind confirmation in Default Approvals mode. The reporter may have inadvertently enabled session auto-approval or have terminal auto-approve enabled in settings. Requesting clarification.

## Change summary

Under Default Approvals, send_to_terminal commands into an interactive shell bypass confirmation after the initial shell launch is approved.

## Best solution

Ask the reporter to verify their `chat.agent.terminal.enableAutoApprove` setting and whether they might have clicked 'Auto-approve for this session' in a confirmation dialog. On current main, send_to_terminal requires confirmation in Default Approvals mode unless session auto-approval is explicitly enabled or a question carousel match skips it.
