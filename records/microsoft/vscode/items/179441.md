# Issue #179441 — SCM - Tab refresh on every keypress on editor

[https://github.com/microsoft/vscode/issues/179441](https://github.com/microsoft/vscode/issues/179441)

## Summary

Still valid on main: `provideOriginalResource` is called on every content change (throttled to 200ms) with no caching. The caching PR #188506 was merged then reverted due to stale quick-diff bases after commits. The maintainer-proposed fix — an `onDidChange` event on `QuickDiffProvider` so extensions can signal base changes and enable safe caching — has not been implemented. Keeping open as a real backlog item.

## Change summary

Avoid calling `provideOriginalResource` on every editor keypress by caching the result and letting extensions signal when the quick-diff base changes.

## Best solution

Add an `onDidChange` event to the `QuickDiffProvider` API (as alexr00 proposed in comment 14), then cache the `provideOriginalResource` result and invalidate only when the provider fires `onDidChange`. This requires API review and a proposed-API addition in `src/vscode-dts/vscode.proposed.quickDiffProvider.d.ts`, with the caching logic in `src/vs/workbench/contrib/scm/browser/quickDiffModel.ts`.
