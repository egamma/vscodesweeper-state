# Issue #317019 — Minimap improvement - click & drag anywhere

[https://github.com/microsoft/vscode/issues/317019](https://github.com/microsoft/vscode/issues/317019)

## Summary

Valid feature request: in proportional mode (the default), clicking the minimap reveals the line but does not initiate a drag — only clicking directly on the slider starts a drag. The scrollbar already supports click-and-drag anywhere. No existing issue or implementation covers this gap.

## Change summary

Allow click-and-drag anywhere on the minimap (proportional mode) to scroll, matching the scrollbar's behavior.

## Best solution

Extend the pointer-down handler in `src/vs/editor/browser/viewParts/minimap/minimap.ts` (around line 1260) so that, in proportional mode, a left-click outside the slider also initiates `_startSliderDragging` (centering the slider on the click point first, then allowing drag) — similar to how non-proportional mode already does at line 1250–1257.
