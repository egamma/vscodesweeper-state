# Issue #115519 — Guide users to 'extension bisect' from the issue reporter dialog

[https://github.com/microsoft/vscode/issues/115519](https://github.com/microsoft/vscode/issues/115519)

## Summary

Team-authored feature request (hard keep-open). The issue reporter dialog still does not directly link to extension bisect when a user selects 'An Extension' as the issue source. The separate 'Troubleshoot Issue' command exists but is not surfaced inline in the reporter UI as requested.

## Change summary

Show a link/hint in the issue reporter dialog guiding users to the extension bisect command when they report an extension-related issue.

## Best solution

Add a visible hint (e.g. 'Don't know which extension? Use Extension Bisect to find out.') in the issue reporter overlay when the user selects 'An Extension' as the source, likely in `src/vs/workbench/contrib/issue/browser/issueReporterOverlay.ts` near the extension-source selection UI. Could also link to the newer 'Troubleshoot Issue' command.
