# Issue #289258 — GitHub authentication browser does not open on KDE Wayland

[https://github.com/microsoft/vscode/issues/289258](https://github.com/microsoft/vscode/issues/289258)

## Summary

Verified duplicate of #224574 — both report that Electron's shell.openExternal fails to open a browser on KDE/Plasma Wayland, breaking GitHub (and Microsoft) sign-in. The canonical issue is already labeled upstream/electron.

## Change summary

GitHub authentication browser window never opens on KDE Wayland because Electron's shell.openExternal does not invoke xdg-open correctly under Wayland.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/224574, which already tracks the upstream Electron shell.openExternal failure on Plasma/KDE Wayland. The fix belongs in Electron; VS Code cannot work around it except via the --ozone-platform=x11 flag the reporter already found.
