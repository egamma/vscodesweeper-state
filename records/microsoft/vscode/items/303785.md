Team-authored bug report about terminal tool output rendering in duplicate/confusing locations during agent sessions — must keep open per policy. The discussion suggests a real rendering issue where a shared terminal's output appears in multiple tool-output views simultaneously.

## Change summary

When an agent runs multiple terminal commands, later command output incorrectly appears in the output view of earlier tool invocations, causing confusing duplicate scrolling.

## Best solution

Fix the terminal tool output routing so each tool invocation's rendered output view only shows content from its own command, not from subsequent commands sharing the same terminal. The fix likely lives in the chat-terminal contribution area, possibly in how output ranges are tracked per tool call.
