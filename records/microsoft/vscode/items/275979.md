# Issue #275979 — Prompt file type detection should be case-insensitive

[https://github.com/microsoft/vscode/issues/275979](https://github.com/microsoft/vscode/issues/275979)

## Summary

Team-authored feature request; a maintainer (aeschli) explicitly suggested waiting for user demand before acting. Keeping open per policy.

## Change summary

Prompt file type detection uses case-sensitive endsWith() checks, which could miss files with non-lowercase extensions on case-insensitive file systems.

## Best solution

Wait for user reports demonstrating real-world pain, then make the extension matching in `promptFileLocations.ts` case-insensitive if warranted. The team comment suggests defining the canonical casing as lowercase `.md`/`.x.md` is also a valid resolution.
