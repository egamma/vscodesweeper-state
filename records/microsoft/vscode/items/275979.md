# Issue #275979 — Prompt file type detection should be case-insensitive

[https://github.com/microsoft/vscode/issues/275979](https://github.com/microsoft/vscode/issues/275979)

## Summary

Team-authored feature request; still unfixed on main. A maintainer suggested waiting for user demand before acting — keeping open per hard keep-open rule for MEMBER-authored issues.

## Change summary

Prompt file type detection uses case-sensitive `endsWith()` checks, failing to recognize files with non-lowercase extensions like `.Prompt.md` or `.MD`.

## Best solution

If user demand materializes, lowercase the filename before matching extensions in `getPromptFileType()` and `getCleanPromptName()` in `src/vs/workbench/contrib/chat/common/promptSyntax/config/promptFileLocations.ts` — similar to the existing `.json` check on line 284 that already uses `.toLowerCase()`.
