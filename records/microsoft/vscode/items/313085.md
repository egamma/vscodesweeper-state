# Issue #313085 — `files.exclude` cannot match filenames with trailing CR (macOS `Icon\r`) because `parsePattern` calls `.trim()`

[https://github.com/microsoft/vscode/issues/313085](https://github.com/microsoft/vscode/issues/313085)

## Summary

Confirmed on current main: `parsePattern` in `glob.ts` line 351 calls `pattern.trim()`, which per ECMA-262 strips CR (U+000D) from the pattern ends, making it impossible to match filenames ending in CR (like macOS `Icon\r`) via `files.exclude`. The workaround (`Icon[\r]`) exists but the underlying trim is overly aggressive.

## Change summary

`files.exclude` patterns cannot match filenames with trailing CR because `parsePattern` uses `.trim()` which strips CR characters.

## Best solution

Replace `pattern.trim()` at `src/vs/base/common/glob.ts:351` with a narrower trim that strips only spaces and tabs (`/^[ \t]+|[ \t]+$/g`), preserving control characters like CR/LF. This is a one-line, low-risk change that preserves the original intent (forgiving accidental spaces from the settings UI) without stripping characters that may be meaningful in filenames.
