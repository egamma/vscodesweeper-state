# Issue #313564 — Agents: Copy chat markdown should retain correct links, not `vscode-file`

[https://github.com/microsoft/vscode/issues/313564](https://github.com/microsoft/vscode/issues/313564)

## Summary

Team-authored bug (hard keep-open). The copy-markdown action in the Agents window produces `vscode-file://` protocol URLs instead of the original link targets. The `stringifyItem` function serializes the model's markdown, but rendered links in the DOM resolve to `vscode-file://` scheme — native browser copy captures those resolved hrefs rather than the source markdown URLs. Still unfixed on current main.

## Change summary

Copying chat markdown from the Agents window replaces original URLs with internal `vscode-file://vscode-app/…` protocol links.

## Best solution

The copy path (native selection copy and the `stringifyItem`-based copy action) needs to either restore original hrefs before copying or intercept the clipboard write to rewrite `vscode-file://` URLs back to their source targets. The fix likely lives in `src/vs/workbench/contrib/chat/browser/actions/chatCopyActions.ts` (for the action-based copy) and possibly in the markdown rendering layer that resolves links for the Agents window (`src/vs/sessions/`).
