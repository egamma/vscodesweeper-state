# Issue #187027 — [wsl] support undo after deleting directory

[https://github.com/microsoft/vscode/issues/187027](https://github.com/microsoft/vscode/issues/187027)

## Summary

Valid feature request for VS Code to provide its own undo-delete mechanism for directories in WSL, since the platform lacks OS-level trash. Already triaged to Backlog by a maintainer; no progress on main yet.

## Change summary

Deleting a directory in WSL cannot be undone because WSL lacks an OS-level trash/recycle bin, and VS Code relies on that mechanism.

## Best solution

Either implement a soft-delete (move-to-temp) undo layer in the remote file service for environments without native trash support, or document the limitation clearly. The WSL remote file system provider (`src/vs/platform/files/common/`) would need to signal trash unsupported and an alternative undo path would need to be built. This is a non-trivial cross-cutting change.
