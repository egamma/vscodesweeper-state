# Issue #315841 — Deadlock in extension host activation blocks tunnels auth

[https://github.com/microsoft/vscode/issues/315841](https://github.com/microsoft/vscode/issues/315841)

## Summary

Keep open for maintainer handling: this is a team-authored issue, carries the `important` label, and is assigned to the `On Deck` milestone, all of which block cleanup close proposals. The report also has a recent member confirmation that the tunnel connection hang still reproduced on Insiders.

## Change summary

Remote tunnel auth can deadlock because immediate activation of auth extensions causes the webworker extension host to wait on extension registry readiness that depends on the remote being connected.

## Best solution

Fix the extension-host activation path so auth-provider activation for remote tunnels can complete before remote installed extensions are ready, likely by aligning the webworker extension host registry/init-data path with the local process extension host scanning behavior noted in the report.
