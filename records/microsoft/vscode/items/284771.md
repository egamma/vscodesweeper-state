## Summary

Valid feature request with 21 upvotes, promoted to Backlog. Syntax highlighting in the Disassembly view is not implemented on current main — the view renders instructions without tokenization. Keeping open as a backlog item with community support.

## Change summary

Add syntax highlighting support in the built-in Disassembly view for assembly instructions read from memory during debugging.

## Best solution

Integrate a tokenization/colorization pass in `src/vs/workbench/contrib/debug/browser/disassemblyView.ts` for rendered instruction lines, likely leveraging existing TextMate grammars for assembly languages or a lightweight inline tokenizer. Could also expose a contribution point so language extensions can provide tokenization for disassembled output.
