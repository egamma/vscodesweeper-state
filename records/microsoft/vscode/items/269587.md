# Issue #269587 — Terminal menu would benefit from separators

[https://github.com/microsoft/vscode/issues/269587](https://github.com/microsoft/vscode/issues/269587)

## Summary

The terminal menu separator issue remains unresolved. The initial fix (PR #269596) was reverted because it broke toggle-visibility for non-navigation groups, and the platform feature needed (#275119, duped to #201782) was closed as not-planned. The issue still tracks a valid polish gap.

## Change summary

Terminal toolbar/context menu items from different logical groups (e.g. voice actions alongside edit actions) appear without separators, making the menu harder to scan.

## Best solution

Either revisit the platform-level `isHiddenByDefault` support for non-navigation groups (closed as not-planned in https://github.com/microsoft/vscode/issues/201782), or find an alternative approach to place voice actions in a separate menu group while preserving toggle-visibility. The fix lives in `src/vs/workbench/contrib/terminal/browser/terminalMenus.ts` and the voice contribution.
