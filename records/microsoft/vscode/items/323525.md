# Issue #323525 — overly aggressive matching for searching chat sessions

[https://github.com/microsoft/vscode/issues/323525](https://github.com/microsoft/vscode/issues/323525)

## Summary

Team-filed bug about overly aggressive fuzzy matching in the chat sessions tree filter — the workbench tree's default FuzzyScore highlights every character individually. Keeping open for maintainer judgment.

## Change summary

Typing in the chat sessions list filter matches every character individually, producing unhelpful results where single characters are highlighted across the session title.

## Best solution

The agent sessions tree in `src/vs/workbench/contrib/chat/browser/agentSessions/agentSessionsControl.ts` uses the default workbench tree fuzzy filter. Either switch to a stricter matching algorithm (e.g. contiguous substring or word-boundary fuzzy) for the sessions tree, or adjust the filter threshold to reject very sparse matches. The `AgentSessionsKeyboardNavigationLabelProvider` in `agentSessionsViewer.ts` provides the labels; the matching itself is in the base tree filter logic.
