# Issue #221407 — sudo reboot now shows the characters I typed in zsh for the password via remote SSH

[https://github.com/microsoft/vscode/issues/221407](https://github.com/microsoft/vscode/issues/221407)

## Summary

The root cause — local echo displaying typed characters (including passwords) during disconnection — was addressed by turning local echo off by default in commit 5c5cb3dc1fc, shipped in 1.98.0. New installs no longer exhibit this behavior without explicit opt-in.

## Change summary

Terminal local echo reveals password characters typed during SSH disconnection/reconnection.

## Best solution

Keep the shipped mitigation (local echo off by default since 1.98.0). The deeper fix of respecting ECHO termios mode remains tracked in https://github.com/microsoft/vscode/issues/130821, but this report's scenario no longer reproduces on defaults.
