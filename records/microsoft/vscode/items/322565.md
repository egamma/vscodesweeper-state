# Issue #322565 — Tunnels stuck in "Starting Remote tunnel" and not installed as service

[https://github.com/microsoft/vscode/issues/322565](https://github.com/microsoft/vscode/issues/322565)

## Summary

Duplicate of #322248, which reports the same symptom (tunnel works but "Starting remote tunnel" notification never resolves). Root cause is that commit c9dcf2c3170 changed the CLI output format from 'Open this link in your browser {url}' to a new banner format, but the regex in remoteTunnelService.ts was never updated to match.

## Change summary

The "Starting remote tunnel" progress notification never dismisses after the tunnel successfully connects, because the node tunnel service regex no longer matches the CLI's new output format.

## Best solution

Update the regex in src/vs/platform/remoteTunnel/node/remoteTunnelService.ts line 412 to match the new CLI output format introduced in c9dcf2c3170 — consolidate discussion on https://github.com/microsoft/vscode/issues/322248 which already has maintainer attention.
