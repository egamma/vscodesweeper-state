# Issue #177319 — TimelineItem command callback values are now useless for "foreign" extensions

[https://github.com/microsoft/vscode/issues/177319](https://github.com/microsoft/vscode/issues/177319)

## Summary

This is a specific aspect of the unfinalized Timeline API — providing cloned public properties to foreign extensions in command callbacks. The canonical tracking issue https://github.com/microsoft/vscode/issues/84297 owns this work, and jrieken's comment confirms the plan is to provide 3rd-party extensions a clone of public API properties once the timeline API is finalized.

## Change summary

TimelineItem command callbacks no longer provide the original item's properties (like git SHA) to extensions that didn't create the item, after PR #169621 stopped passing live extension objects.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/84297 — finalizing the Timeline API includes providing 3rd-party extensions a clone of the item's public properties in command callbacks, which is exactly what this issue requests.
