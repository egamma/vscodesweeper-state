# Issue #311247 — Can't resize or maximize the window on monitors with below 100% display scaling (Wayland)

[https://github.com/microsoft/vscode/issues/311247](https://github.com/microsoft/vscode/issues/311247)

## Summary

This is an upstream Electron/Chromium bug in the Wayland fractional scaling protocol implementation when display scale is below 100%. VS Code has no fractional scaling code of its own — the defect lives in Chromium's `WaylandFractionalScaleV1` feature. Should be labeled upstream and kept open until the Electron/Chromium fix lands.

## Change summary

Window cannot be resized or maximized on Wayland monitors with display scaling below 100%, because Chromium's fractional scale protocol mishandles sub-unity scale factors.

## Best solution

Track upstream: Electron issue https://github.com/electron/electron/issues/42200 (closed as stale but confirmed still broken in Electron 41). A new upstream issue should be filed against the current Electron version. In the meantime, the workaround is `--disable-features=WaylandFractionalScaleV1`. Label as `upstream` + `linux` + `wayland` so it's visible for the next Electron upgrade evaluation.
