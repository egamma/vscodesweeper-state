# Issue #224728 â€” BUG: `editor.cursorBlinking`: Cursor stops blinking after 10 blinkings (seconds) and remains visible

[https://github.com/microsoft/vscode/issues/224728](https://github.com/microsoft/vscode/issues/224728)

## Summary

The reported 10-second stop is intentional for the non-flat cursor animations: current main still limits `smooth`, `phase`, and `expand` to 20 half-second alternates, after which the cursor remains visible.

## Change summary

The editor cursor stops animating after about 10 seconds when using a non-solid `editor.cursorBlinking` style.

## Best solution

Keep the current behavior for fancy cursor animations as designed; users who want continuous blinking should use the plain `blink` style, which is handled by the JS interval path.

  > Thanks for the report. This is currently by design for the fancy cursor animations: commit 25888d5fcc810ab4f6b7aba0d191ccc7b9024ddd intentionally limited `smooth`, `phase`, and `expand` to blink 10 times after the cursor becomes idle, and current main still implements those styles as 20 half-second alternates. ## Evidence

- **intentional change:** Commit 25888d5fcc810ab4f6b7aba0d191ccc7b9024ddd is titled "Limit fancy blinking animations to blink 10 times after cursor idle" and changed `smooth`, `phase`, and `expand` from infinite animation to `20 alternate`: https://github.com/microsoft/vscode/commit/25888d5fcc810ab4f6b7aba0d191ccc7b9024ddd
- **current code:** `src/vs/editor/browser/viewParts/viewCursors/viewCursors.css` still defines `.cursor-smooth`, `.cursor-phase`, and `.cursor-expand > .cursor` as `animation: ... 0.5s ... 20 alternate`, which is exactly 10 seconds of animation.
- **continuous blink path:** `src/vs/editor/browser/viewParts/viewCursors/viewCursors.ts` handles `TextEditorCursorBlinkingStyle.Blink` with `_cursorFlatBlinkInterval.cancelAndSet(...)`, so the finite CSS animation limit applies to the fancy styles rather than the plain `blink` style.
