# Issue #157112 — Add support for running custom tasks in VS Code Web

[https://github.com/microsoft/vscode/issues/157112](https://github.com/microsoft/vscode/issues/157112)

## Summary

This is a valid feature request to officially support custom tasks in VS Code Web. The 'On Deck' milestone indicates scheduling intent, so it stays open. Significant progress has been made (context keys, execution registration), but the UX for surfacing task commands when only custom execution is available in serverless web still has gaps.

## Change summary

Officially support and expose custom task execution in VS Code Web (vscode.dev) when no remote authority is available.

## Best solution

Complete the work to show task menu items on serverless web when a custom execution task provider is contributed — the infrastructure (`CustomExecutionSupportedContext`, `ServerlessWebContext`, `TaskExecutionSupportedContext`) is largely in place since commits aad3b4be485 and 071b61c1845, but the end-to-end UX needs final verification and polish to consider this officially supported.
