# Issue #323596 — chat.disableAIFeatures change value to true without user interaction

[https://github.com/microsoft/vscode/issues/323596](https://github.com/microsoft/vscode/issues/323596)

## Summary

Still reproducible on main: `ChatTeardownContribution` at line 818-833 of `chatSetupContributions.ts` unconditionally sets `chat.disableAIFeatures` to `false` when the default chat extension fires an onChange event while enabled — which happens during profile creation/extension installation. A fix exists on branch `dev/dmitriv/fix-disable-ai-features` but has not yet merged to main.

## Change summary

Creating a new profile or adding an extension resets `chat.disableAIFeatures` from `true` to `false` without user consent.

## Best solution

Land the pending fix on `dev/dmitriv/fix-disable-ai-features` (commit f18797eaaca) which introduces `chatAIDisabledHelpers` to guard against unwanted flips of the setting. The problematic unconditional `updateValue(ChatConfiguration.AIDisabled, false)` at line 830 of `chatSetupContributions.ts` needs the guard that branch adds.
