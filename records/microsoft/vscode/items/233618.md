# Issue #233618 — VSCode trying to update WSL on launch for literally no reason

[https://github.com/microsoft/vscode/issues/233618](https://github.com/microsoft/vscode/issues/233618)

## Summary

Still reproduces on main: the WSLContextKeyInitializer in remote.contribution.ts unconditionally calls hasWSLFeatureInstalled() on startup, which runs wsl.exe --status regardless of the useWslProfiles setting, triggering the WSL update/install prompt for users who uninstalled WSL. PR #248438 only fixed the terminal profile detection path, not the context key initializer path.

## Change summary

VS Code runs wsl.exe on every launch to detect whether WSL is installed, triggering an unwanted WSL update/install prompt for users who don't have or have uninstalled WSL.

## Best solution

Guard the WSLContextKeyInitializer in src/vs/workbench/contrib/remote/electron-browser/remote.contribution.ts to check the useWslProfiles setting before calling hasWSLFeatureInstalled(), similar to the unmerged fix on the copilot/fix-257269 branch. Alternatively, the hasWSLFeatureInstalled() function in src/vs/platform/remote/node/wsl.ts could suppress the update prompt by using a different detection method (e.g. checking for the lxss registry key or the wsl.exe file existence rather than executing it).
