# Issue #289469 — Allow me to redirect a URI to another URI in the Content Provider

[https://github.com/microsoft/vscode/issues/289469](https://github.com/microsoft/vscode/issues/289469)

## Summary

Team-authored feature request for URI redirection in ChatSessionContentProvider — must keep open per policy. The request is for the content provider to support returning a replacement URI when an untitled session URI is resolved to a real session.

## Change summary

Allow ChatSessionContentProvider to redirect an initial untitled URI to a meaningful session URI after the backing session is created.

## Best solution

Add a `uri` property (or a redirect mechanism) to the ChatSession interface returned by the content provider, so that when an untitled session URI is opened, the provider can signal that it should resolve to a different canonical URI. This belongs in the chat session content provider API surface, likely in `src/vs/workbench/contrib/chat/` or the extension API layer.
