# Issue #111143 — Ctrl+click on `file:` link in Debug Console in devcontainer opens host filesystem instead

[https://github.com/microsoft/vscode/issues/111143](https://github.com/microsoft/vscode/issues/111143)

## Summary

Still reproduces on main: the debug console link detector constructs a local-only file URI (linkDetector.ts:248) without attaching the remote authority, so `file:` links open on the host instead of the devcontainer filesystem. An open PR (#307244) is already addressing this — keeping open for that PR to land.

## Change summary

Ctrl+clicking a `file:` link in the Debug Console while in a devcontainer opens the file on the host filesystem instead of the remote container filesystem.

## Best solution

Land or iterate on PR https://github.com/microsoft/vscode/pull/307244, which should attach the remote authority to the constructed URI (similar to how the terminal link manager already handles this). The fix lives in `src/vs/workbench/contrib/debug/browser/linkDetector.ts` around line 248 where `URI.parse(fileUrl)` needs to incorporate the remote authority from `IWorkbenchEnvironmentService.remoteAuthority`.
