# Issue #111016 — [html] move mdn link to the right

[https://github.com/microsoft/vscode/issues/111016](https://github.com/microsoft/vscode/issues/111016)

## Summary

This is a UX polish request to reposition the MDN reference link in HTML/CSS hovers to reduce accidental clicks. The existing `html.hover.references` setting only toggles the link on/off but doesn't address placement. The issue remains valid and unresolved on current main.

## Change summary

Move the MDN Reference link in HTML/CSS hover tooltips so it's less likely to be accidentally clicked when navigating code.

## Best solution

The MDN link content is generated by the upstream `vscode-html-languageservice` and `vscode-css-languageservice` packages. The best fix would be either repositioning the link (e.g., to the right or behind a modifier-click) in those language services, or adding a core hover rendering option that places documentation links less intrusively. This likely requires changes in `microsoft/vscode-html-languageservice` and `microsoft/vscode-css-languageservice`.
