# Issue #111016 — [html] move mdn link to the right

[https://github.com/microsoft/vscode/issues/111016](https://github.com/microsoft/vscode/issues/111016)

## Summary

Valid UX improvement request to reposition the MDN reference link in hover tooltips to reduce accidental clicks. No upvote traction, but the pain point is confirmed by a second commenter. Keeping open as an unscheduled feature request in its current area.

## Change summary

Move the MDN Reference link away from the bottom of hover tooltips (or require modifier-click) to prevent accidental navigation when hovering CSS/HTML properties.

## Best solution

The MDN link rendering lives in the external `vscode-html-languageservice` / `vscode-css-languageservice` packages. A fix could either reposition the link (e.g. right-aligned, or in a separate hover section) or require Cmd/Ctrl+click to follow it. Alternatively, the existing `html.hover.references` / `css.hover.references` settings already let users disable the link entirely, which is a partial workaround.
