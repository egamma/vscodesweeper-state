# Issue #173437 — Git - Changes are not staged if multiple parts are selected separately

[https://github.com/microsoft/vscode/issues/173437](https://github.com/microsoft/vscode/issues/173437)

## Summary

This bug is confirmed still present on current VS Code (v1.114+). Sequential 'Stage Selected Ranges' operations can silently clobber earlier staged ranges because the diff information used by the second staging operation is stale — the index changed after the first stage but the editor's diff hasn't refreshed yet. Multiple recent reports corroborate this. Keeping open as the canonical oldest report.

## Change summary

Staging multiple selected ranges sequentially causes earlier staged ranges to be silently lost, because each stage operation rebuilds the index content from stale diff information.

## Best solution

Fix the race between staging and diff-information refresh in `extensions/git/src/commands.ts` (`_stageChanges`). After the first stage, the working-tree diff and the `~` original document must reflect the updated index before a second stage operation reads them. Alternatively, serialize stage operations so each waits for the diff refresh before proceeding. Related newer reports (https://github.com/microsoft/vscode/issues/283148, https://github.com/microsoft/vscode/issues/264028) should be consolidated onto this issue once fixed.

## Evidence

- **still reproduces:** https://github.com/microsoft/vscode/issues/283148 reports the identical bug on v1.107.0 (Dec 2025), with confirmations on v1.111.0 and v1.114.0.
- **root cause:** In extensions/git/src/commands.ts, `_stageChanges` reads `toGitUri(modifiedUri, '~')` as the original and uses `getWorkingTreeDiffInformation` for line changes. After a stage, the index changes but the editor's diff information hasn't refreshed, so the next stage overwrites the index with HEAD + only the second change.
- **related issue:** https://github.com/microsoft/vscode/issues/161916 tracks the underlying diff-view-not-refreshing problem that enables this data-loss scenario.
- **newer duplicate:** https://github.com/microsoft/vscode/issues/283148 is a newer report of the same bug with active community engagement confirming it persists.
- **code path:** extensions/git/src/commands.ts lines 1769–1800 (stageSelectedChanges) and 1932–1946 (_stageChanges); extensions/git/src/staging.ts lines 16–58 (applyLineChanges).
