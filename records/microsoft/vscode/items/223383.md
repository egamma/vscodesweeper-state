# Issue #223383 — Debug console sticky scroll is much larger than its content

[https://github.com/microsoft/vscode/issues/223383](https://github.com/microsoft/vscode/issues/223383)

## Summary

Still reproduces on main: sticky scroll uses the delegate's static height for node sizing without accounting for dynamic heights (e.g., word-wrap in REPL). No fix has landed.

## Change summary

Debug console sticky scroll renders much larger than its content when elements use dynamic heights (word wrap).

## Best solution

Support dynamic heights in the sticky scroll widget — either measure the actual rendered height of sticky nodes after rendering, or signal the renderer/delegate to render sticky nodes without word-wrap (as connor4312 suggested). The fix lives in `src/vs/base/browser/ui/tree/abstractTree.ts` in the `StickyScrollWidget` and `StickyScrollController` classes.

- **no dynamic height support:** Grepping for 'dynamicHeight' or 'onDidChangeHeight' combined with 'sticky' in abstractTree.ts returns no results — sticky scroll has no mechanism to update node height post-render.
