# Issue #223383 — Debug console sticky scroll is much larger than its content

[https://github.com/microsoft/vscode/issues/223383](https://github.com/microsoft/vscode/issues/223383)

## Summary

Still reproduces on main: the sticky scroll controller uses the full dynamic height of REPL elements (which can be very large with word-wrap on arrays), without any clamping. A fix exists on a feature branch but has not merged.

## Change summary

Debug console sticky scroll takes the full dynamic height of a large array element, making the sticky header much taller than useful.

## Best solution

Clamp sticky scroll node height in the tree's StickyScrollController (as prototyped on branch `justin/scolipede` with a `stickyScrollMaxNodeHeight` option), then have the REPL tree configure that option. Alternatively, render sticky nodes without word-wrap as connor4312 suggested.
