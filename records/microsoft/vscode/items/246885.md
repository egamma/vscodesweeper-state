# Issue #246885 — auto-clean old versions remote ssh server

[https://github.com/microsoft/vscode/issues/246885](https://github.com/microsoft/vscode/issues/246885)

## Summary

Valid feature request for auto-cleaning old remote server versions from ~/.vscode-server/cli/servers/. This is the canonical issue — #249065 was closed as a duplicate of it. No implementation exists on main; the server code only cleans up old logs, not old installation binaries. With 13 upvotes and clear community demand, it should remain open.

## Change summary

Old VS Code remote server versions accumulate in ~/.vscode-server/cli/servers/ without any automatic cleanup mechanism, consuming disk space on remote hosts.

## Best solution

Implement a retention policy in the remote server CLI that removes old server versions after a new one is successfully installed — likely in the VS Code CLI (Rust) or the server bootstrap path. This belongs in core (not an extension) since it manages the server's own installation lifecycle. The `remote` label is correct; this also touches the CLI team's domain.
