# Issue #308766 — Unsupported MCP protocol version: 2025-03-26

[https://github.com/microsoft/vscode/issues/308766](https://github.com/microsoft/vscode/issues/308766)

## Summary

Confirmed bug still present on main: VS Code's MCP HTTP client does not send the `MCP-Protocol-Version` header on subsequent requests after `initialize`, violating the MCP spec. A maintainer acknowledged the issue (connor4312: 'Will fix for the next release') but the fix has not landed.

## Change summary

VS Code's MCP HTTP client omits the required `MCP-Protocol-Version` header on post-initialization HTTP requests to MCP servers, causing servers that validate the header to reject requests.

## Best solution

In `src/vs/workbench/api/common/extHostMcp.ts`, add `'MCP-Protocol-Version': MCP.LATEST_PROTOCOL_VERSION` to the headers object in `_sendStreamableHttp` (and the legacy SSE and GET reconnect paths) so all subsequent HTTP requests include the negotiated protocol version as required by the MCP transport spec.
