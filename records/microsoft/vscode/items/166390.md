# Issue #166390 — Sluggish scrolling in search view when there are many results in one file

[https://github.com/microsoft/vscode/issues/166390](https://github.com/microsoft/vscode/issues/166390)

## Summary

Hard keep-open: the issue carries the 'On Deck' milestone, which is a scheduling decision that defers to maintainer judgment. The bug is confirmed and describes a real performance problem in the search view's tree rendering.

## Change summary

Scrolling in the search results view becomes sluggish/blocking when a single file has many matches, due to expensive tree item creation/disposal and highlight label generation without rendering optimization.

## Best solution

Optimize the search view's tree rendering to defer or batch DOM updates during scroll (e.g. render on next animation frame, virtualize highlight computation). The issue diagnosis points to highlight label generation taking ~200ms per viewport update — a throttle or incremental render in the search result tree renderer would address this.
