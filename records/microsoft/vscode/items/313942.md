# Issue #313942 — New Window opens with title bar or status bar outside visible area on 1080p displays with 125% DPI scaling (regression from #262266)

[https://github.com/microsoft/vscode/issues/313942](https://github.com/microsoft/vscode/issues/313942)

## Summary

Confirmed regression: commit 3f44d800597 set the default workspace window size to 1440×900 without clamping to the display work area, so on screens with a logical work area smaller than 900px the new window overflows. An open fix PR (#313944) exists but has not merged — keeping open for the PR author.

## Change summary

New Window opens partially off-screen on displays with work area height < 900 logical pixels because the default 1440×900 size is not clamped to the available work area.

## Best solution

Land the open PR https://github.com/microsoft/vscode/pull/313944 (or an equivalent fix) that clamps the default window dimensions to the display work area in `defaultWindowState` in `src/vs/platform/window/electron-main/window.ts`, mirroring the clamping already done in `defaultAuxWindowState`.
