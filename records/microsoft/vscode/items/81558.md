There is not enough concrete evidence here for a close reason, and no canonical duplicate or fixing commit is identified in the discussion.

## Change summary

Allow semantic highlighting providers or language servers to identify ranges as semantically belonging to another embedded language, so themes/colorization can treat those ranges like that language.

## Best solution

Leave this with the semantic tokens/API discussion until the editor and language API owners decide whether semantic tokens should carry embedded-language identity, or whether this belongs in a different embedded-language service model. If accepted, the end state would likely be an API/design change around semantic token metadata rather than a Razor-specific fix.
