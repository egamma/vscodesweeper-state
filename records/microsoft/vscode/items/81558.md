# Issue #81558 — Expanding semantic theming to support semantically embedded languages for colorization

[https://github.com/microsoft/vscode/issues/81558](https://github.com/microsoft/vscode/issues/81558)

## Summary

This should stay open as an active design gap: the issue requests semantic-token support for dynamically marking embedded language regions, and current VS Code APIs still model semantic tokens as token type/modifier data rather than language handoff. It is already labeled `under-discussion` and has no hard-close evidence such as a duplicate or fixing commit.

## Change summary

Allow language servers/extensions to mark ranges within a document as semantically belonging to another language for colorization or embedded-language processing.

## Best solution

Continue design discussion in the semantic token/theme API area and decide whether embedded-language handoff belongs in semantic tokens, token-to-TextMate scope mapping, or a separate embedded-language service API. A concrete proposal should cover dynamic embedded ranges, host/embedded language boundaries, and how theme rules consume the embedded language identity.
