# Issue #107016 — Slow reponse to scroll and typing

[https://github.com/microsoft/vscode/issues/107016](https://github.com/microsoft/vscode/issues/107016)

## Summary

Long-running upstream performance issue rooted in Chromium/Electron accessibility-mode overhead on Windows. Partially mitigated by PR #114957 (event limiter reduction) and the `--disable-renderer-accessibility` workaround, but the underlying Chromium perf problem persists. The 'mitigated' label is accurate; the issue still collects reports (most recently May 2025) and has 69 upvotes, so it remains a valid tracking point.

## Change summary

Scrolling, typing, and general editor interaction are sluggish on Windows, traced to Chromium's accessibility tree (AXObject) computations firing even when no screen reader is active.

## Best solution

Continue tracking as an upstream Chromium/Electron issue. The core perf regression lives in Blink's accessibility tree recomputation (`blink::AXObject::UpdateCachedAttributeValuesIfNeeded`). VS Code has applied the mitigations it can (event limiter in PR #114957, `--disable-renderer-accessibility` flag). Further progress depends on Electron/Chromium improving accessibility-mode performance or detection. Recent 2025 comments also implicate GPU driver settings (Nvidia adaptive power), which is a separate user-environment concern.
