# Issue #219811 — Minimap scroll bar stops highlighting all the selected word occurrences in HTML

[https://github.com/microsoft/vscode/issues/219811](https://github.com/microsoft/vscode/issues/219811)

## Summary

Same behavior as #101552 (closed *as-designed): when a language provides a DocumentHighlightProvider (HTML highlights matching open/close tags), it takes priority over the textual word-occurrence fallback. The minimap therefore only shows semantic highlights, not all textual occurrences of the word.

## Change summary

Minimap does not highlight all textual occurrences of a selected word in HTML files — only matching open/close tag pairs are highlighted.

## Best solution

This is the same design decision documented in https://github.com/microsoft/vscode/issues/101552: a language's DocumentHighlightProvider is authoritative and does not fall back to textual word matching. Closing as *as-designed keeps the triage consistent. If users want textual highlights regardless of the language provider, that would be a separate feature request for a new occurrencesHighlight mode.
