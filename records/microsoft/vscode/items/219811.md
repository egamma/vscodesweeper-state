# Issue #219811 — Minimap scroll bar stops highlighting all the selected word occurrences in HTML

[https://github.com/microsoft/vscode/issues/219811](https://github.com/microsoft/vscode/issues/219811)

## Summary

Keep open: the report matches a real current code path where HTML has a semantic document highlight provider, so selection-highlight decorations are suppressed from the minimap while the HTML provider only contributes semantic/tag highlights. I found no fixing commit on current main and no confirmed duplicate in the provided discussion.

## Change summary

Selecting or placing the cursor on words in HTML should continue showing all word occurrences in the minimap, not only matching HTML tags.

## Best solution

The editor word/selection highlight path should preserve minimap markers when a language document highlight provider gives partial or empty results, or the built-in HTML highlight provider should return textual occurrences where appropriate. The likely implementation area is the editor word highlighter/selection highlighter interaction, with HTML language highlights as the reproducing provider.

## Evidence

- **current labels:** The issue is already classified as `bug` and `editor-minimap`, has no milestone, and has no hard keep-open or close-forcing labels.
- **HTML provider:** `extensions/html-language-features/server/src/modes/htmlMode.ts` delegates document highlights to `htmlLanguageService.findDocumentHighlights(...)`; the issue body says only opening/closing tags are highlighted in HTML.
- **fallback behavior:** `src/vs/editor/contrib/wordHighlighter/browser/wordHighlighter.ts` uses the first non-null document highlight provider result and treats an empty result as valid, so the registered textual fallback provider is not necessarily used after the HTML provider responds.
- **minimap suppression:** `src/vs/editor/contrib/multicursor/browser/multicursor.ts` sets `hasSemanticHighlights` when any document highlight provider exists, and `src/vs/editor/contrib/wordHighlighter/browser/highlightDecorations.ts` then uses selection-highlight options without overview/minimap decorations.
- **related issue reference:** The reporter linked similar behavior at https://github.com/microsoft/vscode/issues/101552, but the provided context has no comments or timeline confirming this issue as a duplicate.
