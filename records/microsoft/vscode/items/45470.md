# Issue #45470 — Add a setting to not promote workspace folders to top level if they are children of another top level folder

[https://github.com/microsoft/vscode/issues/45470](https://github.com/microsoft/vscode/issues/45470)

## Summary

Well-supported feature request (107 upvotes) for hiding workspace folders that are children of another top-level folder in the explorer. No implementation exists on current main; the request remains valid and has strong community backing well above the 20-upvote Backlog threshold.

## Change summary

Add a setting so that workspace folders that are sub-folders of another open workspace folder are not promoted to the top level in the file explorer, avoiding duplication.

## Best solution

Implement an explorer setting (e.g. `explorer.hideNestedWorkspaceFolders`) that suppresses top-level promotion of workspace folders already visible as children of another root. The logic would live in the explorer view/model layer under `src/vs/workbench/contrib/files/browser/`. The workaround via `files.exclude` is fragile and breaks extensions that inherit those excludes, so a first-class setting is warranted given the sustained demand.
