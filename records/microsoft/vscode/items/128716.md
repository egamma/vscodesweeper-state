# Issue #128716 — [html] propose `charset="UTF-8"` `target="_blank"` and `rel="stylesheet"`

[https://github.com/microsoft/vscode/issues/128716](https://github.com/microsoft/vscode/issues/128716)

## Summary

Partially addressed on current main: `target` attribute now offers value completions (`_self`, `_blank`, etc.), but `charset` and `rel` still lack value suggestions. The data lives upstream in `vscode-html-languageservice`'s `webCustomData`, so the remaining gap is an upstream enhancement in that package.

## Change summary

HTML attribute value completions are missing for `charset` (e.g. UTF-8) and `rel` (e.g. stylesheet) in the built-in HTML language features.

## Best solution

Add valueSets for `charset` and `rel` attributes in the upstream `vscode-html-languageservice` web custom data (https://github.com/microsoft/vscode-html-languageservice). Once added there, updating the dependency in VS Code picks them up automatically. `target` is already done.

## Evidence

- **charset has no valueSet:** The `charset` attribute on `<meta>` has no `valueSet` or `values` property — verified by inspecting the webCustomData at runtime.
- **rel has no valueSet:** The `rel` attribute on `<link>` has only `name` and `description` keys — no completions for 'stylesheet', 'icon', etc.
- **upstream ownership:** HTML attribute data is maintained in https://github.com/microsoft/vscode-html-languageservice; VS Code consumes it as a dependency (currently v5.3.3).
