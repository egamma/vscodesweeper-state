# Issue #195005 — Auto-suggested '-->' doesn't overwrite when manually typing full HTML comment

[https://github.com/microsoft/vscode/issues/195005](https://github.com/microsoft/vscode/issues/195005)

## Summary

Valid feature request: the auto-closing overtype mechanism only handles single-character closing pairs (line 983 of cursorTypeEditOperations.ts checks `autoClosingPairsCloseSingleChar`). Multi-character closings like `-->` are not overtyped. Already labeled and triaged correctly as a feature-request in the Backlog.

## Change summary

When manually typing `-->` to close an HTML comment that was auto-closed, the auto-inserted `-->` should be overwritten rather than duplicated.

## Best solution

Extend the auto-closing overtype logic in `src/vs/editor/common/cursor/cursorTypeEditOperations.ts` to support multi-character closing sequences. This would require tracking multi-char auto-closed ranges and implementing sequential overtype (consuming one character at a time through the closing sequence). The existing `autoClosedCharacters` tracking could be extended to cover multi-char pairs.
