# Issue #238073 — terminal.integrated.defaultLocation: editor makes it impossible to open a normal 'pane' terminal as well

[https://github.com/microsoft/vscode/issues/238073](https://github.com/microsoft/vscode/issues/238073)

## Summary

Confirmed and still reproducible on current main: the panel's [+] button and its dropdown 'New Terminal' entry invoke createTerminal/createAndFocusTerminal without specifying location, so they fall back to defaultLocation (editor) instead of always targeting the panel.

## Change summary

When `terminal.integrated.defaultLocation` is `editor`, the panel's [+] button opens a terminal in the editor area instead of in the panel.

## Best solution

The panel toolbar's primary action (line 801 in terminalMenus.ts) and the TerminalCommandId.New handler (terminalActions.ts) should pass `location: TerminalLocation.Panel` when invoked from the panel context, overriding the defaultLocation setting for that specific interaction.
