# Issue #272563 — Build Failure: 1ES PT artifact collection error - missing .build/logs and .build/crashes directories

[https://github.com/microsoft/vscode/issues/272563](https://github.com/microsoft/vscode/issues/272563)

## Summary

Team-authored infrastructure debt with milestone 'On Deck' — kept open. The issue is still present on main: the darwin CI test steps create `.build/crashes` but never `.build/logs`, while the 1ES PT artifact collection in `product-build-darwin-ci.yml` expects both directories to exist.

## Change summary

MacOS CI Browser Tests fail at artifact collection because `.build/logs` directory is never created before 1ES PT tries to collect it.

## Best solution

Add `mkdir -p .build/logs` alongside the existing `mkdir -p .build/crashes` in `build/azure-pipelines/darwin/steps/product-build-darwin-test.yml` (and the equivalent linux/win32 test steps if they share the pattern). The flaky-smoke YAMLs already do `mkdir -p .build/crashes .build/logs` — the main CI steps should match.
