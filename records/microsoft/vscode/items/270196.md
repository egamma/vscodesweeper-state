# Issue #270196 — VS Code can't identify my desktop environment

[https://github.com/microsoft/vscode/issues/270196](https://github.com/microsoft/vscode/issues/270196)

## Summary

This is a specific instance (Hyprland on Arch) of the upstream Chromium desktop-environment detection problem already tracked by the canonical issue #187338, which is open with upstream-issue-linked. Proposing duplicate close to consolidate discussion.

## Change summary

Chromium's os_crypt component does not recognize Hyprland as a desktop environment, causing keyring/password storage detection to fail.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/187338, which tracks the upstream Chromium os_crypt desktop-environment detection gap for unrecognized Linux DEs and already has the upstream issue linked. The fix lives in Chromium's key_storage_util_linux.cc, not in VS Code.
