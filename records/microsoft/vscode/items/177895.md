# Issue #177895 — Test: Does not detect `orca` screen reader for Linux Mint or Ubuntu

[https://github.com/microsoft/vscode/issues/177895](https://github.com/microsoft/vscode/issues/177895)

## Summary

This is a team-authored issue (MEMBER) reporting that Electron/Chromium's AT-SPI screen reader detection does not pick up Orca on Linux Mint/Ubuntu. The detection relies on Electron's `accessibility-support-changed` event (src/vs/code/electron-main/app.ts:466), which is upstream of VS Code. Keeping open per the hard keep-open rule for team-authored issues, and because the upstream dependency means this needs explicit maintainer judgment.

## Change summary

VS Code does not detect the Orca screen reader on Linux Mint/Ubuntu, so accessibility mode is not automatically enabled.

## Best solution

The root cause is in Electron/Chromium's AT-SPI integration for Linux screen reader detection. VS Code listens for Electron's `accessibility-support-changed` event (src/vs/code/electron-main/app.ts:466) and has no independent detection. The fix belongs upstream in Electron/Chromium's AT-SPI bridge. In the meantime, users can set `"editor.accessibilitySupport": "on"` as a workaround. The `confirmation-pending` label suggests it still needs verification on a current Electron version — if Electron has since fixed AT-SPI detection, this may be resolved.
