# Issue #326276 — Copilot Chat: mimo-v2.5 tool-calling loop fails with 400 —  not set when tool results contain images

[https://github.com/microsoft/vscode/issues/326276](https://github.com/microsoft/vscode/issues/326276)

## Summary

Plausible bug in the built-in Copilot extension's tool-calling loop: when tool results contain only image parts (no text), the follow-up request to mimo-v2.5 omits the required `text` field, causing a 400. The code lives in extensions/copilot/ (languageModelAccess.ts / toolCallingLoop.ts). Routing to chat-tools and chat-vision for the relevant area owners to investigate.

## Change summary

The Copilot extension's tool-calling loop sends a 400-triggering request to mimo-v2.5 when tool results contain image content but no text content, because the request builder does not set a `text` field for image-only tool result messages.

## Best solution

The request construction in `extensions/copilot/src/extension/conversation/vscode-node/languageModelAccess.ts` (or the prompt/message assembly in the tool-calling loop at `extensions/copilot/src/extension/intents/node/toolCallingLoop.ts`) should ensure a non-empty `text` field is present when composing follow-up messages to vision models that require it, even when tool results contain only image parts. The area owners should verify the mimo-v2.5 API contract and confirm whether this is a client-side missing field or a server-side model limitation.
