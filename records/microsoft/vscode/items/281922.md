## Summary

Team-authored feature request for editor hover timing behavior — showing delay should take precedence over hiding delay. Must remain open per team-member policy.

## Change summary

When hovering a new item while a hiding delay is active, the showing delay should override the hiding delay so the new hover appears predictably.

## Best solution

Implement precedence logic in the hover scheduling code so that when a new hover target is detected during an active hiding delay, the showing delay governs the transition timing. Likely lives in `src/vs/editor/contrib/hover/browser/contentHoverController.ts` or the hover scheduling layer.
