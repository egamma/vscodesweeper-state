# Issue #184519 — Can't resolve conflict of submodule checksum file

[https://github.com/microsoft/vscode/issues/184519](https://github.com/microsoft/vscode/issues/184519)

## Summary

Still reproduces on main: submodule merge conflicts are categorized into the merge group but lack special resolution handling — the merge editor expects text conflict markers which submodules (commit-hash references) don't have, making accept current/incoming/stage non-functional for submodule conflicts.

## Change summary

When a submodule has a merge conflict, VS Code's merge conflict UI shows 0 conflicts and none of the resolution options (accept current/incoming, stage) work.

## Best solution

The git extension needs special-case handling for submodule merge conflicts in the resolution commands (accept current/incoming/stage). For submodules, resolution means choosing which commit ref to use, not merging text content. The fix likely belongs in `extensions/git/src/commands.ts` (stage/accept commands) and `extensions/git/src/repository.ts` (resource handling), possibly adding a `git checkout --ours/--theirs` or `git add` path for submodule resources in the merge group.
