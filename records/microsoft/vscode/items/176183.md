# Issue #176183 — Add option to persist search UI settings 

[https://github.com/microsoft/vscode/issues/176183](https://github.com/microsoft/vscode/issues/176183)

## Summary

Valid feature request with 26 upvotes in the Backlog. The search UI state (replace toggle, details panel, regex mode) is persisted per-workspace via StorageScope.WORKSPACE memento, but no user-level setting exists to set defaults across workspaces — the central ask remains unimplemented.

## Change summary

Add user-level settings to control the default state of search UI toggles (replace, details, regex, case-sensitive) across all workspaces.

## Best solution

Add configuration settings like `search.defaultShowReplace`, `search.defaultShowDetails`, and `search.defaultUseRegex` that override the per-workspace memento defaults, in `src/vs/workbench/contrib/search/browser/search.contribution.ts` and consumed in `searchView.ts`.
