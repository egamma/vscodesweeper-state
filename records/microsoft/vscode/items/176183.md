# Issue #176183 — Add option to persist search UI settings 

[https://github.com/microsoft/vscode/issues/176183](https://github.com/microsoft/vscode/issues/176183)

## Summary

Valid feature request with 26 upvotes (promoted to Backlog). Search UI state (replace visibility, details expansion, regex toggle) is persisted per-workspace via `StorageScope.WORKSPACE` memento, but there is no user-facing setting to control the defaults across workspaces. The gap remains on current main.

## Change summary

Add settings to control the default state of search UI toggles (replace input, search details, regex/case/whole-word) so they persist across workspaces without requiring per-workspace state.

## Best solution

Add settings like `search.defaultShowReplace`, `search.defaultExpandDetails`, and possibly `search.defaultRegex` / `search.defaultCaseSensitive` in `search.common.contribution.ts`. The search view (`searchView.ts`) would read these settings as fallbacks when no per-workspace memento state exists (line ~664 for `showReplace`, line ~500 for `queryDetailsExpanded`, line ~658 for `isRegex`). Alternatively, change `StorageScope.WORKSPACE` to `StorageScope.PROFILE` for these UI-state bits, though a setting-based approach gives more control.
