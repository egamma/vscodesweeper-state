# Issue #236606 — Automatically adjust minimap scale according to display scale

[https://github.com/microsoft/vscode/issues/236606](https://github.com/microsoft/vscode/issues/236606)

## Summary

Keep open: current main still appears to have a real minimap scaling gap for fractional display scale, and there is no concrete duplicate or fixing commit in the provided issue context or local history.

## Change summary

Minimap content should stay visually consistent when a VS Code window moves between displays with different OS display scales.

## Best solution

Update the editor minimap layout to derive an effective minimap scale from the current window pixelRatio, including fractional DPR below 2, or intentionally support non-integer/per-display minimap scale in the editor minimap configuration. The likely ownership is editor minimap layout in src/vs/editor/common/config/editorOptions.ts plus the minimap renderer.
