# Issue #236606 — Automatically adjust minimap scale according to display scale

[https://github.com/microsoft/vscode/issues/236606](https://github.com/microsoft/vscode/issues/236606)

## Summary

This is a feature request for the minimap to dynamically adjust its rendering scale when a window moves between displays with different DPI/scaling factors. The current implementation has a coarse pixelRatio-based adjustment (doubling at ≥2) but doesn't support finer-grained or per-monitor adaptive scaling. Keeping open as under-discussion.

## Change summary

Request for the minimap to automatically adapt its scale when moving between monitors with different display scaling (e.g. 175% on 4K vs 100% on 1080p).

## Best solution

Enhance `EditorLayoutInfoComputer.computeLayout` in `src/vs/editor/common/config/editorOptions.ts` to support finer-grained pixelRatio-based minimap scale adjustment, possibly by supporting non-integer scales (blocked by #84168) or by rounding to nearest integer scale per monitor. This is a meaningful UX improvement for multi-monitor setups with mixed DPI but requires design decisions around non-integer scale support.
