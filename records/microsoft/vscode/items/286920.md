# Issue #286920 — GitHub Copilot doesn't reflect usage of custom prompt

[https://github.com/microsoft/vscode/issues/286920](https://github.com/microsoft/vscode/issues/286920)

## Summary

This is a UX improvement request for the Copilot chat extension to include invoked custom prompt files (.prompt.md) in the 'Used N references' content references it reports. The VS Code core UI renders whatever references the extension provides — the gap is in what the extension chooses to report.

## Change summary

When a custom prompt is invoked via /foo syntax, the 'Used N references' indicator does not reflect that the custom prompt file was used.

## Best solution

The Copilot extension (or the chat participant handling the request) should include the resolved .prompt.md file as a content reference in its response, so the 'Used N references' UI reflects its usage. This likely belongs in the Copilot extension's request handling code rather than VS Code core.
