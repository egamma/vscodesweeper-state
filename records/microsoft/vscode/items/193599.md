# Issue #193599 — Highlight confusable Unicode characters for file explorer

[https://github.com/microsoft/vscode/issues/193599](https://github.com/microsoft/vscode/issues/193599)

## Summary

Valid feature request from a team member. The existing `UnicodeTextModelHighlighter` in the editor has not been extended to the file explorer's resource labels. No implementation found on current main. Hard keep-open: author is MEMBER.

## Change summary

Highlight confusable Unicode characters (e.g. Cyrillic mixed with Latin) in file/folder names shown in the explorer tree.

## Best solution

Reuse `UnicodeTextModelHighlighter` (or its underlying confusable-detection logic in `src/vs/editor/common/services/unicodeTextModelHighlighter.ts`) to detect mixed-script names in explorer resource labels and apply a visual indicator. The integration point would be the `ResourceLabels` / icon-label rendering in `src/vs/workbench/contrib/files/browser/views/explorerViewer.ts`. As @hediet noted, the detection logic is already available — it needs to be wired into the explorer's label rendering path.

## Vision fit

- **Fit:** aligned
- **Reason:** Detecting confusable Unicode in filenames is a security-relevant feature that aligns with VS Code's durable security theme and the value of a simple, trustworthy UI.
- **Evidence:** Durable theme: Security — workspace trust, process sandboxing, safe supply chain. Confusable file names are a known attack vector (homoglyph attacks).
