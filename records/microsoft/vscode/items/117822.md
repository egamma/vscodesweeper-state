## Summary

Legitimate feature request still unimplemented on main. The showMessage APIs still accept a plain string with no theme-icon or markdown support. Parked in Backlog with no recent activity but no reason to close.

## Change summary

Allow theme icons (`$(<name>)` syntax) or MarkdownString in `vscode.window.show(Information|Warning|Error)Message` notifications.

## Best solution

Extend the notification message parameter to support MarkdownString (or at minimum theme-icon interpolation), with a fallback for native/OS modal dialogs that cannot render markdown. This touches the ext-host message service and the notification renderer.
