# Issue #117822 — [product icon themes] theme icons support for vscode.show(Information|Warning|Error)Message

[https://github.com/microsoft/vscode/issues/117822](https://github.com/microsoft/vscode/issues/117822)

## Summary

Feature not implemented — the showMessage API still passes a plain string with no theme icon or markdown processing on current main. Low traction (0 upvotes) but a valid feature request in the Backlog.

## Change summary

Allow theme icon `$(<name>)` syntax (or MarkdownStrings) in `vscode.window.show(Information|Warning|Error)Message` so extension authors can render codicons in notification messages.

## Best solution

Support remains unimplemented. Given zero community upvotes over 4+ years, this could be closed by the community feature-request process. If pursued, the change would need to flow through `extHostMessageService.ts` → `mainThreadMessageService.ts` → the notification renderer, with a decision on how modal (OS-native) dialogs handle unsupported markup (as noted by team members in comments).
