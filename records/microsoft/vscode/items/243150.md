## Summary

Legitimate polish request for better visual distinction in the diff editor tab when comparing files with the same name. The code already enforces showing descriptions when names match (via `forceDescription` in `diffEditorInput.ts`), but the user wants the presentation to be clearer. Already triaged with appropriate labels (`ux`, `polish`, `diff-editor`); keeping open.

## Change summary

When comparing two files with the same filename via 'Select for Compare', the diff editor tab title makes it hard to distinguish the left and right file paths.

## Best solution

Improve the diff editor tab title/description rendering when both sides share the same filename — e.g. by using the `shorten` utility more aggressively or showing distinguishing path segments more prominently. The relevant code is in `src/vs/workbench/common/editor/diffEditorInput.ts` (the `computeLabels` method). The existing `forceDescription` path already fires; the improvement would be in how that description is displayed in the tab.
