# Issue #118892 — Word wrapping incorrect for variable width font with formatting that changes the width of characters

[https://github.com/microsoft/vscode/issues/118892](https://github.com/microsoft/vscode/issues/118892)

## Summary

Still reproduces on main: the DOM line breaks computer does not account for inline decorations (bold, heading styles) when measuring wrapped line widths with variable-width fonts. A fix existed on a feature branch but was never merged.

## Change summary

Word wrapping with variable-width fonts ignores decoration-induced width changes (bold/heading styles), causing incorrect wrap positions and horizontal scroll.

## Best solution

Extend `ILineBreaksComputerContext` to supply inline decorations (CSS classes) so the DOM measurement in `domLineBreaksComputer.ts` renders text with correct styling when computing break offsets. A partial implementation exists in commit 03425a941ab on the `font-size-3` branch that could be adapted.
