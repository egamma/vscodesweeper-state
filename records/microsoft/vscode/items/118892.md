# Issue #118892 — Word wrapping incorrect for variable width font with formatting that changes the width of characters

[https://github.com/microsoft/vscode/issues/118892](https://github.com/microsoft/vscode/issues/118892)

## Summary

Keep open as a plausible editor wrapping bug: current `advanced` wrapping still computes line breaks from a temporary DOM that uses the base editor font, not the markdown-rendered bold/token styling that changes glyph widths. I did not find a concrete fixing commit or confirmed duplicate, so there is not enough evidence for a close proposal.

## Change summary

With a proportional editor font and markdown bold/header formatting, word wrap is calculated as if the text were not bold, causing horizontal scroll.

## Best solution

Route to editor wrapping ownership to decide whether `DOMLineBreaksComputer` should account for rendered token/decorator font weight/style, or whether the `advanced` wrapping documentation should explicitly limit support for style changes that alter glyph width.

## Evidence

- **issue context:** https://github.com/microsoft/vscode/issues/118892 reports VS Code 1.55.0-insider with extensions disabled, `editor.wordWrap: on`, `editor.wrappingStrategy: advanced`, Verdana, and markdown bold/header styling causing horizontal scroll.
- **current code:** `src/vs/editor/common/viewModel/viewModelLines.ts` still selects `_domLineBreaksComputerFactory` only when `wrappingStrategy === 'advanced'`, so this path remains owned by the editor wrapping machinery.
- **current code:** `src/vs/editor/browser/view/domLineBreaksComputer.ts` builds measurement HTML with `applyFontInfo(containerDomNode, fontInfo)` and plain `<span>` content in `renderLine(...)`; it does not apply token/decorator font-weight or markdown styling while measuring break points.
- **setting contract:** `src/vs/editor/common/config/editorOptions.ts` describes `editor.wrappingStrategy: advanced` as delegating wrapping points to the browser and working correctly in all cases, which makes the reported mismatch actionable rather than clearly as-designed.
