# Issue #226741 — Inline completions floating meu does not show the number of total completions when intellisense is open at the same time

[https://github.com/microsoft/vscode/issues/226741](https://github.com/microsoft/vscode/issues/226741)

## Summary

Team-authored bug, still reproducible on main. The inline completions count is gated on `lastTriggerKind === Explicit` in the model, so when completions are triggered automatically (e.g. alongside the suggest widget), the count returns `undefined` and the toolbar omits it.

## Change summary

The inline completions floating toolbar does not display the completion count (e.g. "1/1") when the suggest widget is simultaneously visible.

## Best solution

Adjust the `inlineCompletionsCount` derived in `src/vs/editor/contrib/inlineCompletions/browser/model/inlineCompletionsModel.ts` (line ~578) to also return a count when triggered automatically (or at least when the toolbar is being shown on hover), so the count is visible regardless of how the inline completion was triggered.
