# Issue #291986 — GitHub Copilot imported chat is not kept

[https://github.com/microsoft/vscode/issues/291986](https://github.com/microsoft/vscode/issues/291986)

## Summary

Still reproduces on main: the import action opens the chat as a transient editor tab (via `ChatEditorInput.getNewEditorUri()`) without persisting it to the session store, so closing the tab loses it and it never appears in the sessions list. The `chatViewPane` path also immediately disposes the model reference. No fix has landed.

## Change summary

Importing a chat session does not persist it to the session store; it opens as a disposable editor tab, disappears from the sessions list, and is lost when closed.

## Best solution

The import action in `src/vs/workbench/contrib/chat/browser/actions/chatImportExport.ts` should persist the imported session to the local session store (via `loadSessionFromData` + keeping the reference alive) so it appears in the sessions list and survives tab close. The `chatViewPane` path's immediate `modelRef.dispose()` also needs to be reconsidered — the session widget host should own that lifetime.
