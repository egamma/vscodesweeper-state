# Issue #97949 — [html] don't let '/' trigger a suggest in embedded JavaScript

[https://github.com/microsoft/vscode/issues/97949](https://github.com/microsoft/vscode/issues/97949)

## Summary

Still present on main: the HTML language server registers '/' as a trigger character globally (for closing tags), which also fires in embedded JavaScript regions. The Volar integration PR that might restructure this is still open/unmerged.

## Change summary

Typing '/' inside embedded JavaScript in an HTML file triggers IntelliSense unexpectedly because the HTML completion provider registers '/' as a global trigger character.

## Best solution

The HTML language server should suppress completion triggers (specifically '/') when the cursor is inside an embedded JavaScript region. This could be addressed in `extensions/html-language-features/server/src/htmlServer.ts` by scoping trigger characters per embedded language region, or deferred until the Volar integration (https://github.com/microsoft/vscode/pull/171547) lands which would restructure completion handling.
