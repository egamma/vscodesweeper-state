# Issue #97949 — [html] don't let '/' trigger a suggest in embedded JavaScript

[https://github.com/microsoft/vscode/issues/97949](https://github.com/microsoft/vscode/issues/97949)

## Summary

Still present on main: the HTML completion provider registers '/' as a global trigger character (for closing tags), which fires inappropriately inside embedded JavaScript blocks. The maintainer acknowledged this in 2020 but no fix has landed.

## Change summary

Typing '/' inside embedded JavaScript in an HTML file triggers HTML IntelliSense suggestions, which doesn't happen in standalone .js files.

## Best solution

Suppress the '/' trigger character completions when the cursor is inside an embedded JavaScript region. This likely requires the HTML language server to check the document region/mode before responding to '/' triggers, in `extensions/html-language-features/server/src/htmlServer.ts` or the completion logic downstream.
