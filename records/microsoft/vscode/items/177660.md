# Issue #177660 — Git - Prompt to stash or rebase uncommitted changes on checkout

[https://github.com/microsoft/vscode/issues/177660](https://github.com/microsoft/vscode/issues/177660)

## Summary

Valid feature request with 22 upvotes, in the Backlog milestone. VS Code already prompts reactively when checkout fails due to dirty-tree conflicts ("Stash & Checkout" / "Migrate Changes" / "Force Checkout"), but this asks for a proactive prompt before any checkout when uncommitted changes exist — similar to GitHub Desktop — which remains unimplemented.

## Change summary

Proactively prompt users to stash, migrate, or leave uncommitted changes when switching branches, even when git wouldn't fail.

## Best solution

Add an optional proactive check in the checkout flow (`extensions/git/src/commands.ts`, around the `checkout` / `_checkout` methods) that detects uncommitted changes before attempting `git checkout` and presents a dialog offering to stash, migrate, or leave them — gated by a setting like `git.promptForStashOnCheckout` so users who prefer silent carry-over aren't disrupted.
