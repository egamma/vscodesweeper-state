## Summary

The 'labels' part was shipped in 1.43 (commit 7b315c80be3), but the core request — object-typed values with sub-property access like ${input:image.tag} — remains unimplemented. The feature request is still valid and sits in the Backlog with 8 upvotes, below the 20-upvote promotion threshold.

## Change summary

Allow pickString input options to carry object values with named sub-properties, accessible via dot notation in variable substitution.
