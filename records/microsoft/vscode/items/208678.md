# Issue #208678 — Download Code from Currently Opened Folder in VSCode Workspace

[https://github.com/microsoft/vscode/issues/208678](https://github.com/microsoft/vscode/issues/208678)

## Summary

Team-authored feature request to expose `explorer.download` programmatically with URI arguments so extensions can trigger downloads without requiring user interaction in the explorer. Still unimplemented on main — the command handler only reads from explorer selection context, not from passed arguments.

## Change summary

Allow the `explorer.download` command to accept a list of URIs so extensions can programmatically trigger file downloads in vscode.dev.

## Best solution

Extend the `downloadFileHandler` in `src/vs/workbench/contrib/files/browser/fileActions.ts` to accept an optional array of URIs as command arguments, falling back to explorer context when none are provided. This keeps backward compatibility while enabling the extension scenario Azure Functions needs.
