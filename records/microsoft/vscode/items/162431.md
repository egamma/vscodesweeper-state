[https://github.com/microsoft/vscode/issues/162431](https://github.com/microsoft/vscode/issues/162431)

## Summary

Team-authored feature request (hard keep-open rule). The issue tracks improving merge editor labels for rebase, stash, and pull conflicts — labels still use generic 'Current'/'Incoming' on main with no context-specific optimization. Valid ongoing design discussion with 19 upvotes.

## Change summary

Optimize merge editor labels to reflect the actual git operation (merge, rebase, stash pop, pull) instead of always showing generic 'Current'/'Incoming'.

## Best solution

Implement context-aware labels in the merge editor that detect the type of git operation (merge, rebase, stash, pull) and display appropriate, less confusing titles — possibly with tooltips explaining the relationship (e.g. 'commit being rebased' vs 'new parent'). The related issue https://github.com/microsoft/vscode/issues/247687 about reversed labels during rebase is a specific instance of this broader design work.
