# Issue #164791 — Variable substitution in tasks.json doesn't work for custom tasks provided by extension

[https://github.com/microsoft/vscode/issues/164791](https://github.com/microsoft/vscode/issues/164791)

## Summary

Team-authored issue; the bug appears still present on main — variable substitution in custom task definition properties relies on the `defines` object which may not include undeclared `args` fields from tasks.json. Keeping open per team-authored rule.

## Change summary

Variable substitution (e.g. `${workspaceFolder}`) doesn't resolve in `args` fields of extension-provided custom task definitions when using CustomExecution.

## Best solution

Ensure that all user-specified properties in tasks.json (including `args`) are included in the resolved definition passed to the CustomExecution callback, not just the properties declared in the extension's `taskDefinitions` contribution. The fix likely lives in how `task.defines` is populated during task configuration merging in `src/vs/workbench/contrib/tasks/browser/terminalTaskSystem.ts` and/or the task parsing in `src/vs/workbench/contrib/tasks/common/taskConfiguration.ts`.
