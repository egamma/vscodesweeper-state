# Issue #288027 — fcitx5 IME composition underline not visible in 1.107+ (regression from 1.106.3) on Wayland

[https://github.com/microsoft/vscode/issues/288027](https://github.com/microsoft/vscode/issues/288027)

## Summary

This is an upstream issue in Chromium/Electron caused by the Wayland text-input-v3 protocol deliberately omitting preedit styling. VS Code's EditContext handler in nativeEditContext.ts correctly processes textformatupdate events, but Chromium on GTK4/Wayland never fires them with underline data because the protocol doesn't provide it.

## Change summary

fcitx5 IME composition underline is invisible on Wayland in VS Code 1.107+ due to Chromium's GTK4 switch removing preedit styling information.

## Best solution

Track upstream: this requires either Chromium to fall back to text-input-v1 when preedit styling is needed, or the Wayland text-input protocol to re-add styling support. VS Code's code at src/vs/editor/browser/controller/editContext/native/nativeEditContext.ts (lines 505–534) already correctly renders composition decorations when format events arrive — the gap is that Chromium never sends them on Wayland/GTK4. Consider filing or linking an upstream Chromium/Electron issue.
