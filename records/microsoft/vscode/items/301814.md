# Issue #301814 — Primary/secondary sidebar change

[https://github.com/microsoft/vscode/issues/301814](https://github.com/microsoft/vscode/issues/301814)

## Summary

The primary and secondary sidebars are intentionally placed on opposite sides by design. However, the user's core need — reading AI chat on the left — may already be solvable by dragging the Chat view into the primary sidebar, or by using the panel (bottom). Asking the reporter to clarify whether these workarounds address their need before triaging further.

## Change summary

Request to place both primary and secondary sidebars on the same side of the workbench, rather than enforcing opposite-side placement.

## Best solution

The opposite-side layout is by design (`workbench.sideBar.location` description explicitly states this). The user's accessibility concern about reading on the right side could likely be addressed by dragging the Chat view into the primary (left) sidebar, or moving it to the bottom panel. If neither workaround suffices, this becomes a feature request for independent sidebar positioning — which would be a significant layout architecture change. Clarify with the reporter first.
