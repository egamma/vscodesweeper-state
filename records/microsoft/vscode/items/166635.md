# Issue #166635 — Undo search/replace leaves 'replace all' button greyed

[https://github.com/microsoft/vscode/issues/166635](https://github.com/microsoft/vscode/issues/166635)

## Summary

Still reproduces on main: after Replace All clears search results and disables the button, undoing the replacements does not re-run the search or re-enable the Replace All button. The issue is confirmed and correctly triaged in Backlog.

## Change summary

After performing Replace All in the search view and then undoing, the search results remain blank and the Replace All button stays greyed out until the user manually re-runs the search.

## Best solution

After Replace All completes and the user undoes, the search view should detect that file contents have changed back and either automatically re-run the search or restore the previous search results. The fix likely involves listening for undo operations on the affected files in `searchView.ts` and re-triggering the search query, or preserving the search result state so it can be restored on undo.
