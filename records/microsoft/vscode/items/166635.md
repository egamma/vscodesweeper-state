# Issue #166635 — Undo search/replace leaves 'replace all' button greyed

[https://github.com/microsoft/vscode/issues/166635](https://github.com/microsoft/vscode/issues/166635)

## Summary

Still present on main: after Replace All in the search view, undoing the file changes does not re-trigger the search, leaving the results pane empty and the Replace All button greyed out. No undo-service listener exists in the search contribution to refresh results.

## Change summary

Undoing a Replace All in the search view leaves the search results cleared and the Replace All button disabled, requiring the user to manually re-run the search.

## Best solution

After a Replace All, listen for undo events (via IUndoRedoService) affecting the replaced files and automatically re-execute the search query so results repopulate. The fix likely touches the search view's replace-all flow in `src/vs/workbench/contrib/search/browser/searchView.ts` or the search model to restore results state on undo.
