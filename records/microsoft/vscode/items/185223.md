## Summary

Team-authored issue that remains unresolved. The flickering occurs because renderHoverParts is called multiple times when hovering over marker diagnostics, and the async code action fetch in _renderMarkerStatusbar causes visible flashing. A partial mitigation (recentMarkerCodeActionsInfo caching and &nbsp; placeholder) exists but doesn't fully solve the problem.

## Change summary

Quick fix text in the marker hover statusbar flashes/flickers when moving the mouse over it, because renderHoverParts is re-invoked and the async code action fetch restarts.

## Best solution

Prevent re-rendering of the marker statusbar when the hover anchor hasn't meaningfully changed (same marker range), or ensure the code action fetch result is properly cached and reused without re-creating DOM elements. The fix likely belongs in `src/vs/editor/contrib/hover/browser/markerHoverParticipant.ts` (_renderMarkerStatusbar) and possibly in the hover operation's anchor-equality logic.
