# Issue #259978 — start must be before end

[https://github.com/microsoft/vscode/issues/259978](https://github.com/microsoft/vscode/issues/259978)

## Summary

Still reproduces on main — the `renderForScreenReader` path in `ghostText.ts` can pass unsorted positions to `rangeFromPositions` when ghost text parts have columns that, after capping the line text, produce an edit sequence where `lastEditEnd` exceeds the next `editStart`. No fix has landed. Milestone is 'On Deck', so keeping open.

## Change summary

Inline completions screen reader rendering throws 'start must be before end' when ghost text part columns exceed the capped line text length.

## Best solution

Guard the edit construction in `GhostText.renderForScreenReader` so that parts whose columns exceed `cappedLineText` length are clamped or skipped, preventing the `rangeFromPositions` assertion. The fix belongs in `src/vs/editor/contrib/inlineCompletions/browser/model/ghostText.ts`.
