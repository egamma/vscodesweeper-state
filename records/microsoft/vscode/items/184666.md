# Issue #184666 — Toggle line comment doesn't work for multiple lines in CSS

[https://github.com/microsoft/vscode/issues/184666](https://github.com/microsoft/vscode/issues/184666)

## Summary

Still reproduces on main: toggling line comment on multiple already-commented CSS lines wraps the whole selection in a single `/* */`, stripping the inner comment markers and producing invalid CSS. The fallback in `_executeBlockComment` doesn't handle nested block-comment tokens.

## Change summary

Toggle Line Comment on multi-line CSS selections containing existing `/* */` comments corrupts the code by removing inner comment markers.

## Best solution

When falling back to block-comment mode (language has no line-comment token), the toggle should wrap each line individually in `/* … */` rather than wrapping the whole selection. This avoids nesting issues and matches DevTools behavior. Fix belongs in `_executeBlockComment` in `src/vs/editor/contrib/comment/browser/lineCommentCommand.ts`. Related feature request https://github.com/microsoft/vscode/issues/166944 asks for the same per-line approach as an option.
