# Issue #184666 — Toggle line comment doesn't work for multiple lines in CSS

[https://github.com/microsoft/vscode/issues/184666](https://github.com/microsoft/vscode/issues/184666)

## Summary

Still reproduces on main: CSS has no line comment token, so toggle-line-comment falls back to a single block comment around the entire selection. When the selection already contains `/* */` pairs, `_attemptRemoveBlockComment` only strips the outermost pair, corrupting the intermediate tokens. No fix has landed.

## Change summary

Toggle line comment on multiple already-commented CSS lines corrupts the code by stripping only the outer comment delimiters while leaving inner ones.

## Best solution

The block-comment fallback in `_executeBlockComment` should handle each line individually when the selection spans multiple lines that already contain block comment tokens — essentially a per-line toggle. Related feature request https://github.com/microsoft/vscode/issues/166944 asks for the same underlying change. The fix belongs in `src/vs/editor/contrib/comment/browser/lineCommentCommand.ts`.
