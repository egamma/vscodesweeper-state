## Summary

Team-authored polish request with an 'On Deck' milestone — keeping open. The worktree list in the git extension still lacks any visual indication of the currently active worktree, and context menu actions don't adapt to it. No fix found on main.

## Change summary

The git worktrees list does not highlight which worktree is currently active, and offers redundant 'Open' actions for the already-open worktree.

## Best solution

Add a visual indicator (e.g. a description suffix or decoration) to the active worktree in the worktree quick-pick/tree view, and suppress or relabel the 'Open' / 'Open in New Window' actions when the worktree is already the current workspace. The logic lives in `extensions/git/src/commands.ts` (worktree list rendering) and would compare worktree paths against the current workspace folder.
