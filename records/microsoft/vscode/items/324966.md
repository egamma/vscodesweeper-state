# Issue #324966 — GitHub Copilot Chat is not available as an Activity Bar entry. Right-clicking the Activity Bar shows Explorer, Search, Source Control, Run and Debug, Extensions, Testing, Python, and Accounts, but no Chat/Copilot entry.

[https://github.com/microsoft/vscode/issues/324966](https://github.com/microsoft/vscode/issues/324966)

## Summary

Chat is intentionally registered in the Secondary Side Bar (Auxiliary Bar) by default, not the primary Activity Bar. Users can drag it to the sidebar/Activity Bar via standard view drag-and-drop. The team has separately been experimenting with an Activity Bar shortcut icon, but the current default placement is as-designed.

## Change summary

Request for Copilot Chat to appear as a pinnable entry in the primary Activity Bar by default.

## Best solution

Close as as-designed. The Chat view container lives in the Secondary Side Bar by default; users can drag it to the primary sidebar (Activity Bar section) at any time. The team is separately running an experiment (`chat.showInActivityBar`) to test an Activity Bar icon for discoverability — that work is tracked internally and does not need a community issue to drive it.

  > Thanks for the report. Chat is intentionally placed in the **Secondary Side Bar** (the right-side panel) by default — it doesn't appear in the primary Activity Bar context menu because it's registered in a different view location.
  > 
  > You can move it to the primary sidebar yourself: open Chat (Ctrl+Shift+I or the command palette → "Chat: Focus on Chat View"), then drag its icon from the Secondary Side Bar header to the primary Activity Bar on the left. It will stay pinned there across sessions.
  > 
  > The team is also exploring making Chat more discoverable in the Activity Bar natively, but the current behavior is by design. Closing as as-designed since a supported customization path exists.
