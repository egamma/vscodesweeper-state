# Issue #325000 — Dropping a URL into the editor from Firefox requires holding shift

[https://github.com/microsoft/vscode/issues/325000](https://github.com/microsoft/vscode/issues/325000)

## Summary

Confirmed bug on current main: Firefox on Windows packages URL drag events with a Files type, triggering the DropOverlay's shift-required gate. An open PR (#325015) proposes a fix by detecting the text/x-moz-url MIME type — keeping open pending that PR's review.

## Change summary

Dragging a URL from Firefox's address bar into a VS Code editor on Windows requires holding Shift, unlike Chrome which inserts directly.

## Best solution

Review and land the open PR https://github.com/microsoft/vscode/pull/325015, which adds text/x-moz-url detection to isDragIntoEditorEvent in editorDropTarget.ts so Firefox URL drags bypass the shift gate.
