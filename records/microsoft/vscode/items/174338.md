# Issue #174338 — Diff / Compare does not honor ignore trailing/leading whitespace settings

[https://github.com/microsoft/vscode/issues/174338](https://github.com/microsoft/vscode/issues/174338)

## Summary

The original regression (toggle completely non-functional) was fixed around 1.90, but a remaining bug persists on current main: the toolbar icon's checked state uses the global config context key (`config.diffEditor.ignoreTrimWhitespace`) rather than the language-specific effective value, causing a mismatch for offside languages like Python/YAML where the default is `false`. Still a valid open bug.

## Change summary

The diff editor's 'ignore trim whitespace' toggle icon state doesn't reflect language-specific settings, showing the wrong checked state for Python/YAML files.

## Best solution

Fix the toggled state in `src/vs/workbench/browser/parts/editor/editor.contribution.ts` to reflect the effective (language-specific) config value for the active diff editor's modified file, rather than the global `config.diffEditor.ignoreTrimWhitespace` context key. Alternatively, use a dedicated context key that the toggle command also updates when it writes the language-specific value.
