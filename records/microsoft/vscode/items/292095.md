# Issue #292095 — Permission to read prompt file already attached?

[https://github.com/microsoft/vscode/issues/292095](https://github.com/microsoft/vscode/issues/292095)

## Summary

Still an open design question with no fix on main. Two team members discussed but no resolution was reached on how attached file permissions should propagate to subagents.

## Change summary

A prompt file already attached as a reference gets blocked as a sensitive file when a subagent attempts to read it.

## Best solution

The session permission system (likely in `src/vs/platform/agentHost/node/sessionPermissions.ts` or the agent host file system provider) should automatically grant read access to resources already attached in the parent conversation context, so subagents don't redundantly prompt for permission.
