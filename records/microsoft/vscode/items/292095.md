# Issue #292095 — Permission to read prompt file already attached?

[https://github.com/microsoft/vscode/issues/292095](https://github.com/microsoft/vscode/issues/292095)

## Summary

Real bug with active team discussion (aeschli, roblourens) about how to resolve it — no fix landed yet. The external-path confirmation system does not exempt files already attached as references, and the subagent context propagation question is still open. Keeping open for the team to resolve.

## Change summary

A prompt file already attached as a chat reference is blocked as a sensitive file when the agent (or subagent) tries to read it via the read_file tool.

## Best solution

The external-path confirmation logic in `chatExternalPathConfirmation.ts` should check whether the requested path is already present in the chat context attachments and skip the confirmation for those files. The subagent context-propagation question (comment by aeschli) also needs a decision: should subagents inherit the parent's file-access grants or the full attachment list?
