# Issue #252610 — expose the `jsonSchemaRegistry` in the extension API

[https://github.com/microsoft/vscode/issues/252610](https://github.com/microsoft/vscode/issues/252610)

## Summary

Valid feature request for dynamic JSON schema association in the extension API. A team member (aeschli) acknowledged this as a long-standing gap. No API surface has been added on current main. Keeping open as a legitimate backlog item.

## Change summary

Extensions cannot dynamically associate JSON schemas with files at runtime — only static glob patterns via contributes.jsonValidation are supported.

## Best solution

Expose an extension API (e.g. `vscode.languages.registerJsonSchemaAssociation`) that allows extensions to dynamically register and unregister JSON schema associations for specific documents, complementing the static `contributes.jsonValidation` mechanism. This would live in the JSON language features extension API surface.
