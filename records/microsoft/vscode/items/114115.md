# Issue #114115 — [html] support imports in embedded JavaScript

[https://github.com/microsoft/vscode/issues/114115](https://github.com/microsoft/vscode/issues/114115)

## Summary

Valid, distinct feature request for JavaScript module import resolution in embedded HTML script tags. Team confirmed the limitation is still present (Jan 2025). Related to but not a direct duplicate of #47331 (which focuses on CSS/HTML cross-file class/id support). Keeping open as an active backlog item with 15 upvotes and ongoing community interest.

## Change summary

JavaScript imports inside `<script type="module">` in HTML files lack IntelliSense, go-to-definition, and module resolution that works in standalone .js files.

## Best solution

Implement module resolution for embedded JS in HTML, likely via the Volar integration effort (PR #171547) or by creating virtual JS documents at the HTML file's location and forwarding TS/JS language features. The fix belongs in `extensions/html-language-features/` and the embedded language support layer.
