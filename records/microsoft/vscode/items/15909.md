# Issue #15909 — Add ability to extend from other settings files

[https://github.com/microsoft/vscode/issues/15909](https://github.com/microsoft/vscode/issues/15909)

## Summary

Highly popular feature request (1350 upvotes) for settings inheritance/extends that remains unimplemented on main. The attempted PR #299164 was closed without merging. Well above the 20-upvote backlog threshold — should stay open.

## Change summary

Allow .vscode/settings.json to extend/inherit from other settings files, similar to tsconfig.json extends.

## Best solution

Implement an `extends` property in workspace/folder settings.json that resolves and merges a referenced base settings file, enabling shared configuration across repos without duplication. The configuration resolution logic lives in `src/vs/platform/configuration/`. PR #299164 was an attempt but was closed — a fresh implementation effort is needed with maintainer design input on scope (folder settings only vs. all levels, node_modules resolution, etc.).
