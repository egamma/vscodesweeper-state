# Issue #320033 — Unable to use GPG

[https://github.com/microsoft/vscode/issues/320033](https://github.com/microsoft/vscode/issues/320033)

## Summary

This is a GPG/pinentry configuration issue in WSL, not a VS Code bug. The error 'Inappropriate ioctl for device' occurs because GPG is configured to use curses-based pinentry which requires a TTY, but VS Code's git integration spawns git as a non-interactive child process. The fix is a user-side GPG configuration change.

## Change summary

GPG commit signing fails with 'Inappropriate ioctl for device' when VS Code invokes git in WSL because pinentry-curses cannot open a TTY.

## Best solution

This is a known GPG configuration issue in non-TTY environments. The user should configure `pinentry-program` in `~/.gnupg/gpg-agent.conf` to use a GUI-based pinentry (e.g. pinentry-gnome3, pinentry-qt, or pinentry-wsl-ps1 for WSL), or set `export GPG_TTY=$(tty)` in their shell profile. This is not actionable as a VS Code code change.

  > Thanks for the report. The error "Inappropriate ioctl for device" is a GPG/pinentry configuration issue rather than a VS Code bug. When VS Code invokes `git commit`, it runs git as a non-interactive process without a TTY, so curses-based pinentry (which your log shows) cannot prompt for a passphrase.
  > 
  > To fix this in WSL, configure a pinentry program that doesn't need a TTY. Add this to `~/.gnupg/gpg-agent.conf`:
  > 
  > ```
  > pinentry-program /usr/bin/pinentry-gnome3
  > ```
  > 
  > (or `pinentry-qt`, or for WSL specifically, [pinentry-wsl-ps1](https://github.com/psmay/pinentry-wsl-ps1))
  > 
  > Then restart the agent: `gpgconf --kill gpg-agent`
  > 
  > Alternatively, if you use `gpg-preset-passphrase` or cache the passphrase via the agent, signing will work without an interactive prompt.
  >
