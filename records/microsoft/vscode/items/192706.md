# Issue #192706 — Large contents in Editor causes VoiceOver to lag significantly while typing

[https://github.com/microsoft/vscode/issues/192706](https://github.com/microsoft/vscode/issues/192706)

## Summary

A real accessibility/performance issue on macOS VoiceOver that remains unresolved on current main. The existing mitigations (accessibilityPageSize, LIMIT_CHARS=500) cap text exposed to the screen reader, but VoiceOver's behavior when re-focusing the editor textarea in large files may still trigger excessive reading. The under-discussion label is appropriate — the root cause may be partially upstream (Chromium accessibility layer) and needs further macOS-specific investigation.

## Change summary

VoiceOver on macOS lags significantly when editing large files because re-focusing the editor textarea causes it to read too much content.

## Best solution

Investigate whether the native EditContext path (src/vs/editor/browser/controller/editContext/native/) improves the VoiceOver interaction by reducing the amount of text exposed on focus transitions, or whether a macOS-specific aria-live/role optimization can limit what VoiceOver attempts to read when IntelliSense dismisses and focus returns to the editor. If the root cause is Chromium's accessibility tree, file an upstream Chromium bug and track as upstream.
