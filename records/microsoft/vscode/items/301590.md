# Issue #301590 — Request for folder settings interface in the .code-workspace file

[https://github.com/microsoft/vscode/issues/301590](https://github.com/microsoft/vscode/issues/301590)

## Summary

Valid feature request to allow per-folder settings inline in the .code-workspace file (currently only supported via .vscode/settings.json per folder). No existing canonical issue covers this exact ask. Routes to the config area for triage.

## Change summary

Allow specifying per-folder settings directly in the `folders` section of a .code-workspace file, eliminating the need for separate .vscode/settings.json files in each folder.

## Best solution

Extend `IRawFileWorkspaceFolder`/`IRawUriWorkspaceFolder` to support an optional `settings` property, and teach the workspace configuration service to merge those inline settings at folder scope. The config area team should evaluate whether this adds enough value over the existing .vscode/settings.json mechanism to justify the added complexity.
