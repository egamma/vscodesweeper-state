# Issue #268095 — Use @mode_name to specify your mode.

[https://github.com/microsoft/vscode/issues/268095](https://github.com/microsoft/vscode/issues/268095)

## Summary

Team-authored feature request for @mentioning chat modes in prompts. The capability does not exist on current main — modes are only selectable via a dropdown picker or the /agents slash command. Keeping open per team-member policy; the internal discussion (comment 2) leans toward slash commands or quick access rather than @mentions.

## Change summary

Allow users to @mention chat modes in the chat input to switch or chain modes, instead of only using the dropdown picker.

## Best solution

Decide on the UX direction flagged in comment 2 — slash commands for modes vs. @mention participants — and implement the chosen approach in the chat input completions layer (`src/vs/workbench/contrib/chat/browser/widget/input/editor/chatInputCompletions.ts` and `src/vs/workbench/contrib/chat/browser/chatSlashCommands.ts`). The related issue https://github.com/microsoft/vscode/issues/268096 (chaining mode tasks) should be resolved alongside.
