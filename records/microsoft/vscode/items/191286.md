# Issue #191286 — DocumentSemanticTokensProvider fails for the first untitled editor created with showTextDocument since v1.76

[https://github.com/microsoft/vscode/issues/191286](https://github.com/microsoft/vscode/issues/191286)

## Summary

Fixed on main and shipped in 1.86.0: commit 838acd0f20f added iteration over existing models at DocumentSemanticTokensFeature initialization, so models created before the onModelAdded listener was registered now get semantic tokens. This directly addresses the reported race.

## Change summary

DocumentSemanticTokensProvider.provideDocumentSemanticTokens is never called for the first untitled document created via showTextDocument when no other editors are open.

## Best solution

Keep the shipped fix (commit 838acd0f20f, VS Code 1.86.0). The root cause — models created before the `onModelAdded` listener were ignored — is resolved by the `modelService.getModels().forEach(...)` loop at initialization. No further change needed.
