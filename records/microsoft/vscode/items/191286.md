# Issue #191286 — DocumentSemanticTokensProvider fails for the first untitled editor created with showTextDocument since v1.76

[https://github.com/microsoft/vscode/issues/191286](https://github.com/microsoft/vscode/issues/191286)

## Summary

Keep open: this is a detailed extension-host/editor semantic-token bug report with a repro and a second similar report, and I did not find a concrete fixing commit or targeted regression test on current main.

## Change summary

DocumentSemanticTokensProvider is not invoked for the first untitled editor opened via openTextDocument/showTextDocument after starting with no editor tabs.

## Best solution

Add a regression test for first untitled documents opened through the extension API and fix the semantic-token scheduling/attachment ordering in the editor semantic-token feature or the workbench API document-opening path.
