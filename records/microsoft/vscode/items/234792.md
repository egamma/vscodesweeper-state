# Issue #234792 — Editor GPU: Canvas stretches on resize

[https://github.com/microsoft/vscode/issues/234792](https://github.com/microsoft/vscode/issues/234792)

## Summary

Still reproduces on main: the GPU canvas stretches during resize because CSS dimensions update immediately while the canvas backing store (width/height) updates asynchronously via ResizeObserver — no fix has been committed for this issue.

## Change summary

GPU-accelerated editor canvas visibly stretches during window resize until the debounced/async resize observer updates the canvas dimensions.

## Best solution

Eliminate the stretch by either (a) synchronously updating the canvas width/height in the same frame the CSS size changes (e.g. hooking into the layout path before paint), or (b) using CSS `image-rendering: pixelated` or setting the canvas to `display:none` briefly during resize to avoid the stretch interpolation. The fix lives in `src/vs/editor/browser/gpu/viewGpuContext.ts` and/or `gpuUtils.ts`.
