# Issue #272233 — Powershell 7 Ctrl + G from Terminal fails for paths with space

[https://github.com/microsoft/vscode/issues/272233](https://github.com/microsoft/vscode/issues/272233)

## Summary

Still reproduces on main: the Go To Recent Directory feature (Ctrl+G) uses `preparePathForShell` which prepends `& ` for PowerShell paths with spaces, resulting in invalid `cd & 'path'` commands. dmitrivMS's analysis in the comments is accurate and no fix has landed.

## Change summary

Ctrl+G (Go to Recent Directory) in PowerShell produces invalid `cd & 'path with spaces'` because the path formatting function is designed for command execution, not CWD arguments.

## Best solution

In `terminalRunRecentQuickPick.ts` line 355, replace `preparePathForShell` with either `getUriLabelForShell` plus appropriate quoting for the CWD scenario, or introduce a dedicated CWD path formatter that quotes without the `& ` prefix — as dmitrivMS suggested in the issue comments.
