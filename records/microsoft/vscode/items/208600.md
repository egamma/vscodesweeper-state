# Issue #208600 — Multi-file diff: Actions are hidden when on last line of small file

[https://github.com/microsoft/vscode/issues/208600](https://github.com/microsoft/vscode/issues/208600)

## Summary

Team-authored issue (joaomoreno/MEMBER) — must keep open per policy. The bug reports that action buttons in the multi-file diff editor are clipped/hidden when the cursor is on the last line of a small file. While there was a toolbar cleanup in commit a126becb4a7 (Jan 2026), no commit explicitly addresses this overflow/clipping behavior for small files.

## Change summary

In the multi-file diff editor, inline actions (accept/reject buttons) are visually hidden when the cursor is on the last line of a small file because the editor area clips them.

## Best solution

Ensure the diff editor item template allocates enough vertical space (or allows overflow) so that floating action widgets remain visible even when the cursor is on the last line of a short file. The fix likely lives in `src/vs/editor/browser/widget/multiDiffEditor/diffEditorItemTemplate.ts` or the associated `style.css` (the `.editorParent` has `overflow: hidden` which may clip the actions).
