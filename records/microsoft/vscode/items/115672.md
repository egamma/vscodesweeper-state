# Issue #115672 — Twisties appear only after the view has loaded for the view

[https://github.com/microsoft/vscode/issues/115672](https://github.com/microsoft/vscode/issues/115672)

## Summary

The twistie width is now reserved via CSS (`width: 16px; flex-shrink: 0`), which may mitigate the layout shift from font loading. However, the deeper issue — twisties not rendering until `hasChildren` is resolved by the async data source — likely still exists for the explorer's initial load. Keeping open as it remains a valid polish bug without a clear fixing commit.

## Change summary

File explorer twisties appear only after the tree data loads, causing labels to shift right.

## Best solution

Ensure the explorer tree reserves twistie space (or pre-renders a placeholder) before `hasChildren` is resolved by the async data source, likely in `src/vs/base/browser/ui/tree/abstractTree.ts` or the explorer's `AsyncDataTree` setup. Alternatively, the explorer could optimistically assume folders have children on first render.
