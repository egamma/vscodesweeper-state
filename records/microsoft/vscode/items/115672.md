# Issue #115672 — Twisties appear only after the view has loaded for the view

[https://github.com/microsoft/vscode/issues/115672](https://github.com/microsoft/vscode/issues/115672)

## Summary

Visual polish bug about codicon font loading timing causing twisties to pop in after the explorer tree renders. No evidence of a targeted fix on main; the tree CSS and codicon loading mechanism are essentially unchanged. Keeping open as a valid low-priority polish item.

## Change summary

File explorer twistie icons appear only after the codicon font finishes loading, causing a visual pop-in and label shift.

## Best solution

Ensure the codicon font is loaded (or a placeholder of the same dimensions is shown) before the explorer tree first paints, likely by blocking initial tree render on font readiness or by using an inline SVG fallback for the twistie. The fix would touch `src/vs/base/browser/ui/tree/abstractTree.ts` or the codicon loading in `src/vs/base/browser/ui/codicons/`.
