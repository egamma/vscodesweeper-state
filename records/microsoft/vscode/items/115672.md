# Issue #115672 — Twisties appear only after the view has loaded for the view

[https://github.com/microsoft/vscode/issues/115672](https://github.com/microsoft/vscode/issues/115672)

## Summary

A visual polish issue about twistie icons appearing late in the explorer, causing a layout shift. The `font-display: block` change (commit c8209aff4c8, Aug 2021) may have mitigated the codicon-font-loading aspect, but the async-data-resolution path — where collapsible state is unknown until children resolve — could still produce a brief shift. Keeping open as a real polish item in Backlog; insufficient evidence to close as fixed without a reproduction check on current main.

## Change summary

Twistie expand/collapse icons in the file explorer appear only after the view finishes loading, shifting labels to the right.

## Best solution

Investigate whether the async data tree can reserve twistie space (or assume collapsible) for items whose children haven't resolved yet, avoiding the layout shift. The fix would likely touch `src/vs/base/browser/ui/tree/abstractTree.ts` (twistie rendering) and possibly the explorer's data source in `src/vs/workbench/contrib/files/browser/views/explorerViewer.ts`. A maintainer should first verify whether this still reproduces on current builds — the `font-display: block` fix for codicons (https://github.com/microsoft/vscode/issues/131396) may have resolved the most visible symptom.

## Evidence

- **font-display fix:** Commit c8209aff4c8 (2021-08-23) added `font-display: block` to the codicon font-face, which blocks rendering until the font loads — this addresses the codicon-loading aspect sandy081 identified in the comments.
- **twistie CSS reserves space:** `.monaco-tl-twistie` in src/vs/base/browser/ui/tree/media/tree.css has `width: 16px` and is always created in the DOM (abstractTree.ts line 429), so space is reserved even when not collapsible.
- **async collapsible state:** The explorer uses WorkbenchCompressibleAsyncDataTree; items may initially render as non-collapsible (no twistie chevron) until children resolve, at which point the chevron appears — this is the likely remaining cause of the shift bpasero demonstrated.
