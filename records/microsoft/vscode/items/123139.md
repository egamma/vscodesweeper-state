# Issue #123139 — Git - Add ability to have ignored repository wildcards

[https://github.com/microsoft/vscode/issues/123139](https://github.com/microsoft/vscode/issues/123139)

## Summary

The feature request is still valid — `git.ignoredRepositories` only supports exact path matching on current main, with no glob or wildcard support. It has 27 upvotes and sits in the Backlog milestone, so it should stay open.

## Change summary

Add glob/wildcard pattern support to the `git.ignoredRepositories` setting so users can ignore dynamically-named repository directories (e.g. Bazel caches, pre-commit checkouts).

## Best solution

Extend `shouldRepositoryBeIgnored()` in `extensions/git/src/model.ts` to match each `ignoredRepositories` entry as a glob pattern (via `minimatch` or a similar matcher already available in the extension) instead of only doing `pathEquals`. This would let entries like `**/bazel-*` or `~/.cache/pre-commit/repo*` work as expected.
