# Issue #145768 — Provide support for separate aria label for the customButton on quickpick component

[https://github.com/microsoft/vscode/issues/145768](https://github.com/microsoft/vscode/issues/145768)

## Summary

Valid accessibility feature request: the QuickPick customButton still lacks a dedicated ariaLabel property on current main. An initial implementation was committed and then reverted in March 2022; the gap remains.

## Change summary

Expose a separate aria-label property for the QuickPick's custom button so screen readers announce it distinctly from the input box.

## Best solution

Add a `customButtonAriaLabel` property to the IQuickPick interface (in `src/vs/platform/quickinput/common/quickInput.ts`) and wire it through the implementation class to call `this.ui.customButton.setAriaLabel(...)` during the update pass in `src/vs/platform/quickinput/browser/quickInput.ts`. The Button class already supports `setAriaLabel`.
