# Issue #265184 — Support for Uploading and Analyzing More File Formats in Copilot Chat Agent (e.g., Excel, PDF)

[https://github.com/microsoft/vscode/issues/265184](https://github.com/microsoft/vscode/issues/265184)

## Summary

Valid feature request acknowledged by a team member (roblourens) who agreed the attachment system should be expanded beyond images now that agents can process richer file types. Still unimplemented on current main — the paste provider only accepts image MIME types and non-image binary attachments trigger a 'does not support this file type' warning. Keeping open in the Backlog.

## Change summary

Expand Copilot Chat file attachment support to accept non-image binary formats like Excel, PDF, and DOCX so the agent can analyze them.

## Best solution

Extend the attachment pipeline (chatPasteProviders.ts, chatAttachmentWidgets.ts, and the model-capability checks) to allow binary file types beyond images — at minimum PDF, XLSX, CSV, DOCX — and pass them through to the agent for analysis. The team member's comment suggests this is a known gap from the pre-agent attachment design that should be straightforward to address.
