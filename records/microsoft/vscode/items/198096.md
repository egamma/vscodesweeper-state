## Summary

Team-authored feature request for per-token-type fontFamily in the editor — hard keep-open applies. Still valid and unimplemented on main; 12 upvotes show community interest.

## Change summary

Allow configuring different font families (and potentially fontVariations/fontLigatures) per semantic token type in the editor.

## Best solution

Extend the editor's font rendering pipeline to support per-token fontFamily overrides, likely by enhancing ViewLineRenderingData and the token-to-style mapping in src/vs/editor/browser/viewParts/lines/. The semantic-token customization schema would also need expansion to accept fontFamily alongside the existing foreground/bold/italic/underline/strikethrough properties.
