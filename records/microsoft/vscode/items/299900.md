# Issue #299900 — Inconsistent ctrl+` (Toggle Terminal) shortcut on macOS French keyboard

[https://github.com/microsoft/vscode/issues/299900](https://github.com/microsoft/vscode/issues/299900)

## Summary

Legitimate keyboard layout bug on macOS with French keyboards: the toggle terminal shortcut is registered as ctrl+[Backslash] (where backtick lives on AZERTY), but the physical key sends ctrl+[Backquote] via normal keydown, so the shortcut only works intermittently when macOS routes it through the menu accelerator path instead. Well-diagnosed by the reporter, no fix on current main.

## Change summary

The ctrl+` shortcut to toggle the terminal does not work reliably on macOS with a French keyboard because the scan code mapping differs between the normal keydown path and the menu accelerator path.

## Best solution

Investigate the scan code mismatch in the macOS keybinding resolution: on French keyboards, backtick is on the Backslash scan code (per `src/vs/workbench/services/keybinding/browser/keyboardLayouts/fr.darwin.ts` line 58), but the physical key event arrives as Backquote. The native keyboard layout service (`src/vs/workbench/services/keybinding/electron-browser/nativeKeyboardLayoutService.ts`) and the macLinuxKeyboardMapper need to reconcile this for ISO keyboards. May relate to the broader IntlBackslash/Backquote swap tracked in https://github.com/microsoft/vscode/issues/163432 (upstream for web, but Electron native path may have its own variant).

## Evidence

- **Related upstream issue:** https://github.com/microsoft/vscode/issues/163432 tracks IntlBackslash/Backquote swap on ISO keyboards for Chrome/Firefox (web). The Electron native path may have a similar mapping issue for ISO keyboards on macOS.

- **Keybinding registration:** src/vs/workbench/contrib/terminal/browser/terminal.contribution.ts line 129 registers the toggle terminal shortcut as KeyMod.WinCtrl | KeyCode.Backquote for macOS.
