# Issue #32813 — [theming] Access theme's colors programmatically 

[https://github.com/microsoft/vscode/issues/32813](https://github.com/microsoft/vscode/issues/32813)

## Summary

The core request — programmatic access to resolved color values (workbench colors and token colors) — remains unimplemented. The stable API only exposes `ColorTheme.kind` (light/dark/HC). The `getColor()` method proposed in 2019 was never finalized, and token color access was never added. This is a valid open feature request in the Backlog.

## Change summary

Extensions want to read resolved theme color values (both workbench colors and token/scope colors) programmatically at runtime, beyond just knowing light/dark/HC kind.

## Best solution

Finalize the `ColorTheme.getColor(colorIdentifier)` API for workbench colors (proposed in https://github.com/microsoft/vscode/issues/32813#issuecomment-558440351) and consider a separate or combined API for token/scope colors. The partial solution (theme kind + ThemeColor references in decorations) covers some use cases but not computed-color or SVG-rendering scenarios.
