## Summary

Keep open: this is a long-standing, highly upvoted API proposal in the Backlog, and the remaining request is broader than the already-available `ColorTheme.kind` API. The discussion and later related issues show active extension demand for theme-resolved token/workbench colors rather than a solved or duplicate request.

## Change summary

Expose a supported extension API for accessing the active theme's resolved colors, especially token colors, instead of only referencing colors by id in selected APIs.

## Best solution

Continue tracking this as the canonical API proposal for theme-resolved color access, likely owned by the themes/API area, and decide whether to expose resolved workbench colors, token colors by scope/category, or a narrower token-color API covered by related newer proposals.
