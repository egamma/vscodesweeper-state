# Issue #194923 — Consider re-ranking TF-IDF results

[https://github.com/microsoft/vscode/issues/194923](https://github.com/microsoft/vscode/issues/194923)

## Summary

Team-authored feature request to improve Command Palette TF-IDF ranking with a reranking pass; no implementation has landed and the author is a MEMBER — keeping open.

## Change summary

TF-IDF scoring in the Command Palette over-weights repeated common words, causing unintuitive ranking (e.g. 'File: New File' outranks 'File Utils: Move…' for the query 'move file').

## Best solution

Investigate a secondary reranking pass (or score normalization) in `src/vs/base/common/tfIdf.ts` or at the quick-access layer to penalize frequency inflation from repeated common terms, so that relevance to the full query is weighted more heavily than raw term frequency.
