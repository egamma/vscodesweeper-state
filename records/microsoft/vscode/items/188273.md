# Issue #188273 — Files in .gitignore missing from search when Exclude Git Ignore is off

[https://github.com/microsoft/vscode/issues/188273](https://github.com/microsoft/vscode/issues/188273)

## Summary

The ambiguous setting descriptions for `search.useIgnoreFiles`, `search.useGlobalIgnoreFiles`, and `search.useParentIgnoreFiles` remain unchanged on current main. The reporter's proposed rewording was acknowledged by a maintainer who invited a PR. This is a valid polish item worth keeping open.

## Change summary

The `search.useIgnoreFiles` setting description is ambiguous — 'use .gitignore files when searching' can be read as either 'include those files in results' or 'apply their exclusion rules', leading users to believe gitignored files should appear in search.

## Best solution

Update the `markdownDescription` strings for `search.useIgnoreFiles`, `search.useGlobalIgnoreFiles`, and `search.useParentIgnoreFiles` in `src/vs/workbench/contrib/search/browser/search.contribution.ts` to clarify that enabling these settings *excludes* matching files from search results. The reporter's proposed wording (comment 5) is a good starting point. This is a small, self-contained text change.
