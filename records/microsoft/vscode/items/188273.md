# Issue #188273 — Files in .gitignore missing from search when Exclude Git Ignore is off

## Summary

The ambiguous description for `search.useIgnoreFiles` remains unfixed on main — it still reads 'Controls whether to use .gitignore and .ignore files when searching for files' without clarifying that enabling it *excludes* matched files from results. The polish request is still valid.

## Change summary

Clarify the descriptions of `search.useIgnoreFiles` and related settings so users understand that enabling them excludes gitignored files from search results.

## Best solution

Update the `markdownDescription` for `search.useIgnoreFiles` in `src/vs/workbench/contrib/search/browser/search.contribution.ts` (line ~156) to explicitly say 'When enabled, files and folders listed in .gitignore and .ignore files are excluded from search results.' Similar clarity pass on `search.useGlobalIgnoreFiles` and `search.useParentIgnoreFiles`.
