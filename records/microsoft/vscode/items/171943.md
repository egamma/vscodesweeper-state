# Issue #171943 — Using npm as the preferred package manager. Found multiple lockfiles

[https://github.com/microsoft/vscode/issues/171943](https://github.com/microsoft/vscode/issues/171943)

## Summary

Still reproducible on main: the lockfile-detection logic in `isPNPMPreferred` uses `findUp` which traverses parent directories, potentially counting a lockfile from a parent folder alongside the project's own lockfile, triggering a false 'multiple lockfiles' warning.

## Change summary

VS Code shows a spurious 'Found multiple lockfiles' notification even when only one lockfile exists in the project directory.

## Best solution

Restrict the lockfile counting in `findPreferredPM` to consider only lockfiles in the project directory itself (not those found via `findUp` in parent directories) when computing `multipleLockFilesDetected`. The `findUp` traversal is useful for resolving which PM to use but should not inflate the lockfile count that drives the warning.
