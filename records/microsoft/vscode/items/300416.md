# Issue #300416 — Support passing a skip reason on TestRun.skipped()

[https://github.com/microsoft/vscode/issues/300416](https://github.com/microsoft/vscode/issues/300416)

## Summary

Valid, well-scoped API enhancement request for the testing framework — adding an optional message parameter to TestRun.skipped() for skip reasons. Not yet implemented on main; already triaged into Backlog with the correct area label.

## Change summary

Allow TestRun.skipped() to accept an optional TestMessage parameter so extensions can surface a skip reason in the UI.

## Best solution

Add an optional `message?: TestMessage | readonly TestMessage[]` parameter to `TestRun.skipped()` in the stable API, wire it through `extHostTesting.ts` to the test result state, and surface it in the hover/peek UI for skipped tests. A commit (6260bf587c4) exists on a side branch addressing this but has not landed on main.
