# Issue #188307 — ExtensionHostManager: Trying to add a disposable to a DisposableStore that has already been disposed of

[https://github.com/microsoft/vscode/issues/188307](https://github.com/microsoft/vscode/issues/188307)

## Summary

Still present on main: the `.then()` callback in the ExtensionHostManager constructor calls `this._register()` without a disposal guard, causing a leak warning when the manager is disposed before the promise resolves (e.g., EH startup failure/cancellation).

## Change summary

ExtensionHostManager logs a DisposableStore leak error when the extension host fails to start and the manager is disposed before the startup promise resolves.

## Best solution

Guard the `this._proxy.then()` callback at line 155 of `extensionHostManager.ts` with a `this._store.isDisposed` check before calling `this._register()`, preventing the leak warning on a race between disposal and async startup completion.
