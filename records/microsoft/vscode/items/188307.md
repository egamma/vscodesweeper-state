# Issue #188307 — ExtensionHostManager: Trying to add a disposable to a DisposableStore that has already been disposed of

[https://github.com/microsoft/vscode/issues/188307](https://github.com/microsoft/vscode/issues/188307)

## Summary

Keep open: the original CI failure is intermittent and was not locally reproduced, but current main still has the same async pattern that can register a disposable after ExtensionHostManager disposal.

## Change summary

A LocalWebWorker extension host startup failure can race with disposal and cause ExtensionHostManager to add a disposable to an already disposed DisposableStore.

## Best solution

Harden `src/vs/workbench/services/extensions/common/extensionHostManager.ts` so the `_proxy.then` startup continuation does not call `_register` after the manager has been disposed, and add focused coverage for disposing the manager before extension host startup settles.
