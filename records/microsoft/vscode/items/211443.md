# Issue #211443 — OSC 8 terminal hyperlinks don't work properly in Remote - WSL

[https://github.com/microsoft/vscode/issues/211443](https://github.com/microsoft/vscode/issues/211443)

## Summary

Still reproduces on main: `TerminalUrlLinkOpener._openFileSchemeLink` does not convert `file://` URIs to `vscodeRemote://` in remote contexts (WSL, SSH), so `fileService.stat` fails silently and the link does nothing. The sibling `TerminalSearchLinkOpener` already handles this correctly.

## Change summary

OSC 8 terminal hyperlinks with `file://` scheme silently fail when connected to a remote (WSL or SSH) because the URI is not converted to the remote scheme before resolution.

## Best solution

In `TerminalUrlLinkOpener._openFileSchemeLink`, when `this._isRemote` is true, convert the incoming `file://` URI to a `vscodeRemote://` URI (using `remoteAuthority` from the environment service) before calling `fileService.stat` and delegating to the sub-openers — mirroring the pattern already used in `TerminalSearchLinkOpener` at lines 201–206.
