# Issue #200777 — Preserve terminal cwd when reviving process

[https://github.com/microsoft/vscode/issues/200777](https://github.com/microsoft/vscode/issues/200777)

## Summary

Confirmed bug still present on main: when a terminal process is revived after VS Code restart, the CwdDetectionCapability is not restored with the last known cwd, so the terminal resets to the workspace root. Team-authored, milestone Backlog, confirmed by multiple members including a root-cause analysis.

## Change summary

Terminal cwd resets to workspace root on VS Code restart instead of preserving the last shell-integration-detected cwd.

## Best solution

Serialize the CwdDetectionCapability's last-known cwd into the persistent terminal state (attachPersistentProcess) and restore it on reconnection — likely in shellIntegrationAddon.ts (deserialize cwd into the CwdDetection capability) and the terminal persistence layer that populates attachPersistentProcess.cwd with the shell-integration cwd rather than the initial cwd.
