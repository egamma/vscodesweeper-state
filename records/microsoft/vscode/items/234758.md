# Issue #234758 — Right-clicking injected text caused funny selection

[https://github.com/microsoft/vscode/issues/234758](https://github.com/microsoft/vscode/issues/234758)

## Summary

Still present on main: injected text elements (ghost text, inlay hints) lack explicit user-select protection against native DOM selection on right-click; no fix commit found.

## Change summary

Right-clicking on injected text (ghost text or inlay hints) in the editor causes a native DOM selection highlight instead of the expected editor context menu behavior.

## Best solution

Add `user-select: none` (and `-webkit-user-select: none`) to the CSS for injected text elements — likely in `ghostTextView.css` and the inlay hints rendering — or ensure the view-line rule cascades properly to those children. Alternatively, suppress DOM selection via a `selectstart` or `contextmenu` event handler on those elements.
