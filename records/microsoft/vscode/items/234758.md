# Issue #234758 — Right-clicking injected text caused funny selection

[https://github.com/microsoft/vscode/issues/234758](https://github.com/microsoft/vscode/issues/234758)

## Summary

Still appears unfixed on main. No commits since the report address native DOM selection on injected text during right-click. The existing `user-select: none` on `.view-line` doesn't prevent this browser behavior on injected text spans.

## Change summary

Right-clicking on injected text (ghost text, inlay hints) triggers a native DOM selection on the element instead of only the editor's own selection.

## Best solution

Add CSS `user-select: none` (or prevent default selection via JS on contextmenu event) specifically for injected text spans rendered in the editor view lines, likely in `src/vs/editor/browser/viewParts/viewLines/viewLines.css` or in the inline completions/inlay hints rendering code.
