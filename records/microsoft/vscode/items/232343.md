# Issue #232343 — Comments broken in markdown codeblock

[https://github.com/microsoft/vscode/issues/232343](https://github.com/microsoft/vscode/issues/232343)

## Summary

Still reproduces on main: `_gatherPreflightCommentStrings` detects the language at column 1, which returns `markdown` for indented embedded code blocks instead of the actual embedded language (e.g. TypeScript). The fix should check the language at the first non-whitespace column.

## Change summary

Toggle Line Comment in markdown indented code blocks (e.g. inside a bulleted list) applies HTML comments instead of the embedded language's comment style.

## Best solution

Fix the three `getLanguageIdAtPosition(lineNumber, 1)` calls in `lineCommentCommand.ts` to use the first non-whitespace column instead, so embedded languages inside indented blocks are correctly identified.
