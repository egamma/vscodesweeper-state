# Issue #261897 — Find and Search/Find in Files with selection from Terminal doesn't do regex escaping

[https://github.com/microsoft/vscode/issues/261897](https://github.com/microsoft/vscode/issues/261897)

## Summary

Confirmed on current main: two code paths pass the raw terminal selection into find inputs without regex escaping. The editor's equivalent paths correctly call `strings.escapeRegExpCharacters()`. A maintainer has acknowledged the bug (comment 7).

## Change summary

When regex mode is on, selecting text with regex-special characters in the terminal and pressing Ctrl+F or Ctrl+Shift+F populates the find input with unescaped text, producing an 'Invalid regular expression' error — unlike the editor, which escapes properly.

## Best solution

Add `escapeRegExpCharacters` calls in two places: (1) `terminalFindWidget.ts` line 157, escape `initialInput` when `_getRegexValue()` is true before passing to `super.show()`; (2) `terminal.find.contribution.ts` line 251, escape `activeInstance.selection` when the search view's regex toggle is on before passing it as `query` to `findInFilesCommand`.
