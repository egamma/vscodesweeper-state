# Issue #184107 — VSCode Isn't completely respecting "editor.insertSpaces". It inserts spaces, but deletes them four at a time (editor.tabSize).

[https://github.com/microsoft/vscode/issues/184107](https://github.com/microsoft/vscode/issues/184107)

## Summary

This is as-designed behavior controlled by the `editor.useTabStops` setting (default: true). Setting it to false gives single-space deletion, which is exactly what the reporter wants.

## Change summary

Backspace deletes spaces in tab-stop increments rather than one at a time when using spaces for indentation.

## Best solution

Close as designed — the `editor.useTabStops` setting already provides the requested behavior when set to `false`.
