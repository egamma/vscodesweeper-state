# Issue #252403 — Display blanks when sharing a VSCode window via Google Meet (MacOS 26)

[https://github.com/microsoft/vscode/issues/252403](https://github.com/microsoft/vscode/issues/252403)

## Summary

This is a Chromium/Electron rendering issue triggered by macOS 26's window capture APIs during Google Meet window sharing. The report is from macOS 26 beta 1–2 on Electron 34 (VS Code 1.101.1); VS Code has since moved to Electron 42.5.0 with many Chromium rendering fixes. Needs confirmation on the latest build before any action.

## Change summary

Sharing a VS Code window via Google Meet on macOS 26 causes the editor area to blank out on scroll, persisting until restart.

## Best solution

This is an upstream Chromium/Electron rendering issue with macOS window-capture APIs — VS Code doesn't control how Electron handles window-level screen sharing. The reporter should retest on the latest VS Code stable (now on Electron 42) with macOS 26 release; if it still reproduces, file upstream at https://github.com/nicedoc/electron or the Chromium bug tracker.
