# Issue #163649 — [less] support `each` in keyframes

[https://github.com/microsoft/vscode/issues/163649](https://github.com/microsoft/vscode/issues/163649)

## Summary

Still reproduces on current main: the LESS parser in vscode-css-languageservice does not recognize `each()` function calls inside `@keyframes` bodies — the keyframe selector parser only handles detached ruleset mixins and standard selectors, not function invocations like `each()`.

## Change summary

VS Code's LESS parser incorrectly reports a syntax error when the LESS `each()` function is used inside a `@keyframes` block, though this is valid LESS since v3.9.0.

## Best solution

Extend `_parseKeyframeSelector()` (or the keyframe body parsing) in vscode-css-languageservice's `lessParser.ts` to accept function calls like `each()` as valid keyframe body statements. The fix lives in the upstream `microsoft/vscode-css-languageservice` repo and would flow into VS Code via a service update.
