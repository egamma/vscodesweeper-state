# Issue #239770 — Alter an expression breakpoint deletes log message

[https://github.com/microsoft/vscode/issues/239770](https://github.com/microsoft/vscode/issues/239770)

## Summary

Confirmed bug still present on main: when editing a breakpoint in one context (e.g. Expression), properties from other contexts (e.g. logMessage) are overwritten with undefined because the update data object always includes all keys and Breakpoint.update() uses hasOwnProperty checks. An open community PR exists to address this.

## Change summary

Editing a breakpoint's expression (or hit count) clears any previously set log message because the update data includes all breakpoint properties as undefined, and the update method overwrites on hasOwnProperty.

## Best solution

Review and land (or close with feedback) the open community PR https://github.com/microsoft/vscode/pull/243851, which addresses this by ensuring only explicitly-provided properties are passed in the update data. The fix is localized to `src/vs/workbench/contrib/debug/browser/breakpointWidget.ts` (how update data is constructed) or `src/vs/workbench/contrib/debug/common/debugModel.ts` (how `Breakpoint.update()` handles undefined values).
