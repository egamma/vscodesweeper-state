# Issue #313966 — Indentation still being detected/changed after setting is disabled

[https://github.com/microsoft/vscode/issues/313966](https://github.com/microsoft/vscode/issues/313966)

## Summary

This should stay open and be routed to the editor area: the report has concrete repro steps, and current main still has a code path that can run indentation detection after language assignment without checking `editor.detectIndentation`.

## Change summary

Opening C files can reset tab size from the configured 2 spaces to detected 8 spaces even when `editor.detectIndentation` is disabled.

## Best solution

The editor open/language-detection path should honor `editor.detectIndentation: false`, likely by guarding the `textModel.detectIndentation(...)` call in `src/vs/workbench/browser/parts/editor/textResourceEditor.ts` with the resolved creation option.

- **expected guarded behavior elsewhere:** `src/vs/editor/common/services/modelService.ts:254-265` only calls `model.detectIndentation(...)` when `newOptions.detectIndentation` is true; otherwise it applies configured indentation options directly.
