# Issue #230230 — Markdown should not split up words in backticks that end in `/` when wrapping

[https://github.com/microsoft/vscode/issues/230230](https://github.com/microsoft/vscode/issues/230230)

## Summary

Still reproduces on main: `/` is in the default `wordWrapBreakAfterCharacters` (editorOptions.ts line 6801), and the monospace line breaks computer has no awareness of Markdown backtick spans — it treats `/` as a break-after character regardless of context. No fix has landed.

## Change summary

Words in backtick-delimited code spans containing `/` are split at the slash when the editor wraps lines, breaking path readability.

## Best solution

The monospace line breaks computer (`src/vs/editor/common/viewModel/monospaceLineBreaksComputer.ts`) would need language-aware context — or the Markdown extension could contribute language-specific wrapping hints — to suppress break-after-slash inside inline code spans. Alternatively, this could be handled at the Markdown renderer level if the issue is about preview rendering rather than the editor itself. Needs design input on where the fix belongs.
