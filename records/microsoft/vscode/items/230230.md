# Issue #230230 — Markdown should not split up words in backticks that end in `/` when wrapping

[https://github.com/microsoft/vscode/issues/230230](https://github.com/microsoft/vscode/issues/230230)

## Summary

Keep open as a small markdown preview wrapping polish issue; the report is actionable and already routed to the relevant markdown/editor wrapping areas, and I did not find evidence that current main has implemented this edge case.

## Change summary

Inline code spans containing folder paths ending in `/` should prefer not to wrap by splitting the path awkwardly after the slash.

## Best solution

Handle this in the built-in Markdown preview styling, likely in `extensions/markdown-language-features/media/markdown.css`, by refining wrapping behavior for inline `code` path-like text without regressing long-token overflow handling.
