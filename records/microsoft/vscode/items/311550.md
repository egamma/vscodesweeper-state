# Issue #311550 — [Error] potential listener LEAK — menuService

[https://github.com/microsoft/vscode/issues/311550](https://github.com/microsoft/vscode/issues/311550)

## Summary

Still reproduces on current main: the lazy-update toolbar fix (957e119f8a1) was reverted (4c1d2477856), so each tree row template in changesViewRenderer.ts still creates a MenuWorkbenchToolBar that subscribes to the shared PersistedMenuHideState._onDidChange emitter, exceeding the listener threshold with large working sets.

## Change summary

The Agents changes tree view triggers a 'potential listener LEAK' warning because each rendered row template subscribes a separate MenuWorkbenchToolBar to the shared PersistedMenuHideState._onDidChange emitter.

## Best solution

Either defer the menu subscription in MenuWorkbenchToolBar until the toolbar is actually visible (the reverted approach in 957e119f8a1, needs a non-regressing implementation), or pool/share the menu instance across row templates in changesViewRenderer.ts so only one subscription exists regardless of row count.
