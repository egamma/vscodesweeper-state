# Issue #213356 — Operation would exceed heap memory limits

[https://github.com/microsoft/vscode/issues/213356](https://github.com/microsoft/vscode/issues/213356)

## Summary

Team-authored error-telemetry issue; the defect is still present on current main — `getLinesContent()` in the tokenizer controller has no size guard beyond the factory's 50MB check, and files near that limit can still OOM when the lines array is allocated.

## Change summary

Opening a very large file causes an OOM crash in the TextMate background tokenizer when it calls `getLinesContent()` to send the full document to the worker.

## Best solution

Add a size guard in `TextMateWorkerTokenizerController` (or tighten the factory guard) so that `getLinesContent()` is never called for documents whose buffer size approaches heap limits — either skip async tokenization entirely for such files or stream chunks to the worker instead of sending a full `string[]`.
