# Issue #225945 — Error: `No view is registered with id` might be caused by a race condition

[https://github.com/microsoft/vscode/issues/225945](https://github.com/microsoft/vscode/issues/225945)

## Summary

The race condition is still present on current main: `$registerTreeViewDataProvider` waits on `whenInstalledExtensionsRegistered()` then calls `getTreeView()` which looks up the ViewsRegistry, but during extension host restart after an update, the view descriptor may not yet be re-registered — the error path at line 70 fires. No fix has landed.

## Change summary

Extension restart after update triggers 'No view is registered with id' error due to a race between view contribution registration and tree data provider registration.

## Best solution

In `mainThreadTreeViews.ts#$registerTreeViewDataProvider`, either await the specific view descriptor appearing in the ViewsRegistry (with a timeout), or queue data-provider registration until the view is available, rather than immediately erroring when `getTreeView` returns null during a restart cycle.
