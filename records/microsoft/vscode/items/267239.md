# Issue #267239 — Wrong active branch being used in repoContext

[https://github.com/microsoft/vscode/issues/267239](https://github.com/microsoft/vscode/issues/267239)

## Summary

Still reproduces on main: the `_activeEditorRepositoryObs` observable in `scmViewService.ts` only re-evaluates when the active editor changes, not when SCM repositories are added/removed, so the branch context can be stale. Author is a team contributor and an open PR exists — keeping open.

## Change summary

The chat agent's repo context reports the wrong active branch because the SCM active-editor-repository observable doesn't re-evaluate when repositories change.

## Best solution

Land or refine the open PR https://github.com/microsoft/vscode/pull/308929, which adds `onDidAddRepository`/`onDidRemoveRepository` as dependencies to the `_activeEditorRepositoryObs` derived observable in `src/vs/workbench/contrib/scm/browser/scmViewService.ts`.
