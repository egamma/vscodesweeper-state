# Issue #324297 — Search-based file-path autocomplete for the terminal

[https://github.com/microsoft/vscode/issues/324297](https://github.com/microsoft/vscode/issues/324297)

## Summary

VS Code's terminal already ships built-in file-path autocomplete via the terminal suggest feature (`terminal.integrated.suggest.enabled`, on by default). The existing implementation resolves files and folders relative to the current directory, supports quick suggestions while typing, and works with PowerShell, zsh, bash, and fish — which is exactly what the screenshot from Copilot shows.

## Change summary

Request for search-based file-path autocomplete in the VS Code integrated terminal.

## Best solution

This is already shipped — `terminal.integrated.suggest.enabled` (default `true`) provides file and folder path completions in the terminal via the terminal IntelliSense system (`src/vs/workbench/contrib/terminalContrib/suggest/`). The reporter likely has shell integration disabled or is using an unsupported shell. Close as already available and point them to the setting and docs.

## Evidence

- **existing feature:** Terminal suggest with file/folder path completions is implemented in `src/vs/workbench/contrib/terminalContrib/suggest/browser/terminalCompletionService.ts` — the `resolveResources` method resolves files and folders from the filesystem and presents them as completion items.
- **enabled by default:** `terminal.integrated.suggest.enabled` defaults to `true` (see `src/vs/workbench/contrib/terminalContrib/suggest/common/terminalSuggestConfiguration.ts` line 88).
- **quick suggestions:** `terminal.integrated.suggest.quickSuggestions` supports automatic suggestions for commands, arguments, and unknown contexts — including file/folder fallback.
- **supported shells:** Works with PowerShell, zsh, bash, and fish when shell integration is enabled.
