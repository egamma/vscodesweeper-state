# Issue #239872 — Error when installing extensions when using HTTP proxy

[https://github.com/microsoft/vscode/issues/239872](https://github.com/microsoft/vscode/issues/239872)

## Summary

Still reproducible on main: setting `http.proxyAuthorization` to a raw credential value (e.g. `user:pass`) is passed verbatim as a `Proxy-Authorization` header to Electron's SimpleURLLoaderWrapper, which rejects it with `ERR_INVALID_ARGUMENT`. No fix or validation has been applied. The team member chrmarti is engaged, and a new report (Jan 2026) confirms the issue persists.

## Change summary

Setting `http.proxyAuthorization` causes `net::ERR_INVALID_ARGUMENT` in Electron when installing extensions or performing network requests through the proxy.

## Best solution

Either validate/encode the `http.proxyAuthorization` value before passing it as a header (e.g. wrap raw credentials in `Basic <base64>`), or deprecate the setting in favor of Electron's built-in proxy authentication flow. The fix likely touches `src/vs/platform/request/node/requestService.ts` (line 111-116) and `src/vs/base/parts/request/common/requestImpl.ts` (line 80-82).
