# Issue #314323 — Confusing UI descriptions for `bypass approvals` vs `autopilot`

[https://github.com/microsoft/vscode/issues/314323](https://github.com/microsoft/vscode/issues/314323)

## Summary

Valid UX polish request about confusing permission-level descriptions. Team members are actively discussing improved wording in the comments, and the strings on current main haven't materially changed. Keeping open for the team to land on final copy.

## Change summary

The UI descriptions for 'Bypass Approvals' and 'Autopilot' permission levels are too similar and don't clearly convey that Autopilot is a superset of Bypass Approvals.

## Best solution

Update the detail/description strings in `permissionPickerActionItem.ts` to clearly differentiate: Bypass Approvals auto-approves tool calls, while Autopilot auto-approves AND autonomously continues iterating until the task is complete. The team discussion suggests copy like 'Auto-approves all tool calls' vs 'Auto-approves all tool calls and runs without check-ins'.
