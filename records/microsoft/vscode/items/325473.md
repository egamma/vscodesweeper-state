# Issue #325473 — Terminal file links should resolve relative paths using the terminal's current working directory

[https://github.com/microsoft/vscode/issues/325473](https://github.com/microsoft/vscode/issues/325473)

## Summary

The requested behavior — resolving relative terminal file links against the terminal's current working directory — is already implemented when shell integration with command detection is active. The code in `terminalLocalLinkDetector.ts` calls `updateLinkWithRelativeCwd` which uses `getCwdForLine(y)` to resolve relative paths against the per-command cwd, exactly as the reporter suggests.

## Change summary

Resolve relative file links in the terminal against the shell's current working directory instead of doing a workspace-wide search.

## Best solution

Close as already implemented. The terminal link resolver already uses the per-command cwd from shell integration (`updateLinkWithRelativeCwd` in `terminalLinkHelpers.ts`, called from `terminalLocalLinkDetector.ts` and `terminalLinkOpeners.ts`). If the reporter's MINGW64 setup doesn't resolve correctly, it's likely a shell integration activation issue — they should verify shell integration is active (look for the shell integration indicator in the terminal tab) and that `terminal.integrated.shellIntegration.enabled` is `true`.
