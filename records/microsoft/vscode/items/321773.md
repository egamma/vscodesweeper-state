# Issue #321773 — symbol-* codicons specified via `iconPath` aren't colored in quickpicks created by extensions

[https://github.com/microsoft/vscode/issues/321773](https://github.com/microsoft/vscode/issues/321773)

## Summary

Confirmed on current main: when extensions set `iconPath` to a ThemeIcon in quickpick items, the `codicon-colored` CSS class is never added, so symbol-* codicons lose their foreground color. The same gap was already fixed for tree views (#320391) and Go to Symbol (#299753), but the quickpick path remains unpatched.

## Change summary

symbol-* codicons specified via `iconPath` in extension-created quickpicks render without their designated color.

## Best solution

Add `codicon-colored` to the icon class in `src/vs/workbench/api/browser/mainThreadQuickOpen.ts` (the `expandIconPath` ThemeIcon branch, ~line 298) or in `src/vs/platform/quickinput/browser/quickInputList.ts` (the `iconClass` rendering path, ~line 487) — mirroring the approach from #320391 and #299753.
