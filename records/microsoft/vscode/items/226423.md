# Issue #226423 — Allow more than 9 items in the title bar

[https://github.com/microsoft/vscode/issues/226423](https://github.com/microsoft/vscode/issues/226423)

## Summary

The hard-coded `maxItems: 9` in the editor title toolbar overflow behavior is still present on current main. A team member acknowledged this as a valid improvement worth exploring. Keeping open as an accepted feature request in the Backlog.

## Change summary

Allow more than 9 action items in the editor title bar by making the limit dynamic or configurable.

## Best solution

Replace the hard-coded `maxItems: 9` in `src/vs/workbench/browser/parts/editor/editorTabsControl.ts` with a width-based approach (as benibenj suggested) that fills as many actions as fit in the available toolbar space, falling back to overflow for the rest.
