# Issue #273003 — Chat: Resolve Merge Conflict doesn't reset tools that it needs

[https://github.com/microsoft/vscode/issues/273003](https://github.com/microsoft/vscode/issues/273003)

## Summary

Team-authored bug on the 'On Deck' milestone — keep open. The defect is confirmed in source: `resolveMergeConflicts` opens a chat with `mode: 'agent'` but does not specify or reset the tool set, so it inherits whatever tools the current session has (which may exclude the file-editing tools the merge-conflict flow requires).

## Change summary

The 'Resolve conflicts with AI' command reuses the current chat session's tool set instead of ensuring the merge-conflict-required tools are available.

## Best solution

When `resolveMergeConflicts` calls `workbench.action.chat.open`, it should either force a new session with the default/full tool set, or explicitly pass the required tools (e.g. file-editing tools) in the open options so the agent flow doesn't fail when invoked from a session with a restricted tool configuration.
