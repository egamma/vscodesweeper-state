# Issue #145984 — Timeline API has no way to signal an entry should be removed

[https://github.com/microsoft/vscode/issues/145984](https://github.com/microsoft/vscode/issues/145984)

## Summary

The Timeline API's `TimelineChangeEvent` still only supports `uri` and `reset` — no granular entry-removal signal exists on current main. This is a valid API gap that remains unaddressed, filed by a core contributor.

## Change summary

The Timeline API provides no way to signal that specific entries were removed; the only option is `reset: true`, which forces a full refresh.

## Best solution

Extend `TimelineChangeEvent` (in `src/vs/workbench/contrib/timeline/common/timeline.ts` and the proposed API `src/vscode-dts/vscode.proposed.timeline.d.ts`) with a field to express removed entry IDs or timestamps, so providers can signal fine-grained removals without a full reset. Since the Timeline API is still proposed, this can be added without a finalization concern.
