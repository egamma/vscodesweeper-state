# Issue #187369 — Option-key navigation between words separated by a non-breaking space on a Mac

[https://github.com/microsoft/vscode/issues/187369](https://github.com/microsoft/vscode/issues/187369)

## Summary

Still reproduces on main: WordCharacterClassifier only classifies Space and Tab as Whitespace; non-breaking space (U+00A0) defaults to Regular, so word navigation treats NBSP-separated words as one token.

## Change summary

Option+arrow word navigation on Mac skips over non-breaking spaces as if they were word characters, jumping across multiple words at once.

## Best solution

Add `this.set(CharCode.NoBreakSpace, WordCharacterClass.Whitespace)` in `WordCharacterClassifier` constructor (src/vs/editor/common/core/wordCharacterClassifier.ts, line ~39) so NBSP is treated as a word boundary like regular space.
