# Issue #188296 — Handle bad auth state better in the Issue Reporter

[https://github.com/microsoft/vscode/issues/188296](https://github.com/microsoft/vscode/issues/188296)

## Summary

Team-authored issue describing a real gap in the Issue Reporter's auth error handling. Keeping open per policy; the problem likely still exists on main.

## Change summary

The Issue Reporter silently fails when the stored GitHub auth token is invalid, giving no feedback to the user.

## Best solution

When the Issue Reporter detects an invalid/expired token (e.g. a 401 from GitHub), fall back to the 'preview on GitHub' browser flow so the user can still file the issue. The fix likely lives in `src/vs/workbench/contrib/issue/browser/` (issueReporterService or related).
