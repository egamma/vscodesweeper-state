# Issue #260025 — Setting {"explorer.enableUndo": false} not respected when Undo file renaming

[https://github.com/microsoft/vscode/issues/260025](https://github.com/microsoft/vscode/issues/260025)

## Summary

Still reproduces on main: when `explorer.enableUndo` is `false`, file operations (rename, etc.) are still pushed onto the undo/redo stack via `applyBulkEdit`. The `enableUndo` check only gates the explorer-focused undo handler (priority 110 in `files.contribution.ts`), so when focus is in the text editor, pressing Ctrl+Z surfaces the file operation and shows the confirmation dialog despite the setting.

## Change summary

Setting `explorer.enableUndo: false` does not prevent file rename operations from appearing in the undo stack, so Ctrl+Z in the editor still prompts to undo the rename.

## Best solution

The fix should prevent file operations from being registered on the undo stack when `enableUndo` is `false`. In `explorerService.ts` `applyBulkEdit`, check `explorer.enableUndo` and either skip the `undoRedoSource` parameter or pass a flag to the bulk edit service that prevents undo registration. This keeps the operation non-undoable regardless of where focus is.
