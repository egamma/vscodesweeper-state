## Summary

Valid bug report about configurationSchema interactions causing duplicate models and incorrect config values in the model picker. No fix found on main. Already labeled `model-picker` which is the right area.

## Change summary

When a LanguageModelChatProvider extension provides models with configurationSchema and users change per-model settings via the secondary menu, duplicate models appear in the picker and the toggled configuration value is not applied to the request.

## Best solution

Investigate the interaction between `setModelConfiguration` in `src/vs/workbench/contrib/chat/common/languageModels.ts` and the model resolution/display pipeline. The duplicate-model bug likely involves the storage cache in `getAllMergedModels` (chatInputPart.ts:1879) or the group-based re-resolution when settings change. The config-not-applied bug is in how `_resolveModelConfigurationWithDefaults` merges user config with schema defaults.
