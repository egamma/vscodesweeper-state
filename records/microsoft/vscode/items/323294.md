# Issue #323294 — Flaky unit test: McpStdioStateHandler sigterm after grace

[https://github.com/microsoft/vscode/issues/323294](https://github.com/microsoft/vscode/issues/323294)

## Summary

Flaky test still reproduces in CI as of July 2026 despite prior fix attempts (d64abc5368b, a19ae5a9fc5). The race condition between stdin close causing process exit and SIGTERM delivery remains unresolved. Keeping open — team-authored issue tracking a real CI stability problem.

## Change summary

The 'sigterm after grace' unit test intermittently fails because the child process exits on stdin end before the SIGTERM signal is delivered.

## Best solution

The child script's `stdin 'end'` handler should not allow the process to exit before SIGTERM arrives — e.g. keep the process alive explicitly (the setInterval already does this, but Node may still exit if stdin is the only active handle after `resume()`). A more robust fix: have the child script not call `process.exit` on stdin end, or add an explicit keepalive ref that only clears after SIGTERM. The fix belongs in `src/vs/workbench/contrib/mcp/test/node/mcpStdioStateHandler.test.ts`.
