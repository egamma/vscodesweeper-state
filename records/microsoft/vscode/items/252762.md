# Issue #252762 — Cannot copy code that the AI wants to remove

[https://github.com/microsoft/vscode/issues/252762](https://github.com/microsoft/vscode/issues/252762)

## Summary

Active team discussion: connor4312 asked for a screen recording, the reporter provided one on 2026-05-09 showing the issue persists in 1.119.0. The chat editing overlay does not use the inline-diff copySelection support (commit 2b00e85c864, shipped in 1.106.0) — that fix only covers the regular diff editor view zones, not the chat editing hunk overlay. Awaiting team follow-up.

## Change summary

Users cannot select or copy text from deleted lines in the AI chat editing overlay, preventing them from rescuing comments the AI removes.

## Best solution

Extend the copy-selection capability (from `src/vs/editor/browser/widget/diffEditor/diffEditorViewZones/copySelection.ts`) to the chat editing overlay in `src/vs/workbench/contrib/chat/browser/chatEditing/chatEditingEditorOverlay.ts`, or allow partial accept/reject of individual deletions within a hunk.
