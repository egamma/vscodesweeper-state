# Issue #284159 — Copilot Agent adds Cyrillic 'с' prefix to commands when chatting in Ukrainian/Russian

[https://github.com/microsoft/vscode/issues/284159](https://github.com/microsoft/vscode/issues/284159)

## Summary

Real bug where the Copilot agent's LLM output confuses Cyrillic homoglyphs (U+0441 'с') with Latin 'c' when generating terminal commands in a Ukrainian/Russian conversation context. Already triaged to chat-terminal with confirmation-pending; one confirming comment exists. No fix found on main.

## Change summary

Copilot Agent prepends Cyrillic 'с' (U+0441) to terminal commands when the chat conversation is in Ukrainian or Russian, causing command execution failures.

## Best solution

Add Unicode homoglyph sanitization for terminal commands generated by the agent — either in the Copilot extension's terminal tool output path or in VS Code's terminal command execution layer (e.g. near `chatCodeblockActions.ts` or `chatTerminalToolConfirmationSubPart.ts`). Alternatively, improve model prompting to enforce Latin-only output for shell commands.
