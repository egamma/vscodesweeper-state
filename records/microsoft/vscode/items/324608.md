# Issue #324608 — Copilot chat pane not rendering final response, yet shown in debugger

[https://github.com/microsoft/vscode/issues/324608](https://github.com/microsoft/vscode/issues/324608)

## Summary

Team-authored bug report with an open fix PR (#326497) that hasn't merged yet — keeping open. The root cause is a content-array length mismatch between progressive and final rendering paths in ChatListItemRenderer, causing the final markdown to not render until a scroll or tab switch forces a re-render.

## Change summary

Copilot chat's final response markdown fails to render during streaming when the response includes thinking and tool invocation parts, requiring a scroll or tab switch to appear.

## Best solution

Land https://github.com/microsoft/vscode/pull/326497, which introduces IChatTailPlaceholder sentinels to keep progressive and final content arrays at identical lengths so the diff algorithm matches slots correctly.
