# Issue #246932 — Add config to control alpha/contrast of syntax highlighted completion suggestions (big copilot blocks can be jarring)

[https://github.com/microsoft/vscode/issues/246932](https://github.com/microsoft/vscode/issues/246932)

## Summary

Valid feature request: syntax-highlighted ghost text uses a hardcoded `opacity: 0.7` in CSS with no user-configurable setting. The issue is already labeled `under-discussion` in the Backlog — keeping it open for the community process.

## Change summary

Allow users to configure the opacity/contrast of syntax-highlighted inline completion suggestions instead of the hardcoded 0.7 value.

## Best solution

Add a configurable opacity setting (e.g. `editor.inlineSuggest.syntaxHighlightingOpacity`) or a dedicated color token (like `editorGhostText.syntaxHighlightedForeground`) so users can tune how much the syntax-colored ghost text is dimmed. The fix would touch `src/vs/editor/contrib/inlineCompletions/browser/view/ghostText/ghostTextView.css` and the editor options in `src/vs/editor/common/config/editorOptions.ts`.
