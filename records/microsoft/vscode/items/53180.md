## Summary

Still an open, unimplemented feature request. Source/code actions remain editor-only on current main — no explorer context menu integration exists. Low community traction (1 upvote) but carries `help wanted`, so it stays open for a potential contributor.

## Change summary

Allow triggering source code actions (e.g. organize imports) from the file explorer context menu, not just from an open editor.

## Best solution

This likely belongs as an extension-contributed context menu item rather than core, as the author noted. If pursued in core, it would need a new activation event and a way to run code actions without an active editor — touching `src/vs/workbench/contrib/files/browser/fileActions.contribution.ts` and the code action infrastructure. Given low traction, keeping it in Backlog with `help wanted` is reasonable.
