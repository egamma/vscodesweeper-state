# Issue #283514 — Allow to rename local chat sessions

[https://github.com/microsoft/vscode/issues/283514](https://github.com/microsoft/vscode/issues/283514)

## Summary

Rename for chat sessions still does not persist reliably — multiple users confirm on current stable (1.117–1.118), and an open PR #323323 directly targets this issue with a fix that has not yet merged.

## Change summary

Renaming a chat session via the sidebar does not persist across reload or reverts to the auto-generated title, especially when the session is not the currently active one.

## Best solution

Land PR https://github.com/microsoft/vscode/pull/323323 (or an equivalent fix in chatSessionStore.ts and localChatSessionsProvider.ts) that flushes the renamed title to the session data file immediately and propagates the change event to the sessions framework. The prior commit e2f1c08d034 partially addressed this but users still reproduce; the open PR addresses the remaining persistence gap.
