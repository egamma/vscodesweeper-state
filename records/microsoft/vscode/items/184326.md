# Issue #184326 — Clipboard access is blocked when not via a user gesture

[https://github.com/microsoft/vscode/issues/184326](https://github.com/microsoft/vscode/issues/184326)

## Summary

Team-authored issue (CONTRIBUTOR) — hard keep-open. The gap remains: the extension Clipboard API still only exposes readText/writeText with no multi-mime read(), and the webview user-gesture requirement for navigator.clipboard is still in place on current main.

## Change summary

Webview clipboard access requires a user gesture that cannot be mapped from extension commands, and the extension API lacks a multi-mime clipboard.read() equivalent.

## Best solution

Either relax the user-gesture requirement for clipboard access inside webviews (since extensions are trusted code, not arbitrary web pages), or add a `clipboard.read()` API that returns ClipboardItems with multiple mime types — paralleling the browser's navigator.clipboard.read(). The webview clipboard plumbing lives in src/vs/workbench/contrib/webview/ and the extension clipboard API is defined in src/vscode-dts/vscode.d.ts (Clipboard interface).
