# Issue #21122 — When using Run to Cursor, then there is no inline instruction pointer decoration

[https://github.com/microsoft/vscode/issues/21122](https://github.com/microsoft/vscode/issues/21122)

## Summary

Team-authored feature request (MEMBER) — must keep open. The inline column decoration heuristic has been simplified since 2017 (now shows when startColumn > 1), but the broader request to use DAP `breakpointLocations` to intelligently decide when to show the inline instruction pointer remains unimplemented.

## Change summary

Improve the heuristic for showing the inline instruction pointer decoration by leveraging DAP's `breakpointLocations` to determine whether a line has multiple statement locations.

## Best solution

Use the now-available DAP `breakpointLocations` request (the successor to `getPossibleBreakpoints` discussed in the thread) to determine whether a line contains multiple breakpoint-able locations, and only show the inline column decoration when the stop is not at the first location. This would replace the simple `startColumn > 1` check in `createDecorationsForStackFrame` in `src/vs/workbench/contrib/debug/browser/callStackEditorContribution.ts` with a smarter heuristic. The last comment (2019) explicitly asked to reconsider now that the API is available.
