# Issue #285428 — Opening new terminals in the editor area seemingly cannot figure out ${fileDirname}

[https://github.com/microsoft/vscode/issues/285428](https://github.com/microsoft/vscode/issues/285428)

## Summary

Still reproduces on main: when a terminal opens in the editor area, it becomes the active editor before CWD variable resolution occurs, causing `${fileDirname}` to resolve to undefined (falling back to home). The panel path works because it doesn't change the active editor.

## Change summary

Opening a terminal in the editor area with `terminal.integrated.cwd` set to `${fileDirname}` resolves to home directory instead of the active file's directory.

## Best solution

The variable resolver in `baseConfigurationResolverService.ts` (line 72) should capture or receive the active editor resource at the time the terminal creation is initiated, rather than reading `editorService.activeEditor` at resolution time when the terminal editor may already be active. Alternatively, the terminal creation path could snapshot the active file URI before opening the editor terminal and pass it through.
