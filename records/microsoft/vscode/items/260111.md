# Issue #260111 — Seeing translation errors in notebook snapshot in Window log

[https://github.com/microsoft/vscode/issues/260111](https://github.com/microsoft/vscode/issues/260111)

## Summary

Team-authored issue; the error is triggered by VS Code's test discovery evaluating copilot-chat extension code where `vscode.l10n.t` is not yet initialized. Connor confirmed the cause is test discovery. Keeping open for the team to decide ownership (test discovery vs. extension resilience).

## Change summary

Test discovery triggers a TypeError in the copilot-chat extension's `vscodeTypes.ts` because `vscode.l10n.t` is undefined when the module is evaluated outside a fully initialized extension host context.

## Best solution

Either the copilot-chat extension should guard `vscode.l10n.t` access to tolerate evaluation during test discovery, or the test discovery mechanism should ensure the l10n API is stubbed/available when evaluating extension code. The fix likely lives in the copilot-chat repo (`src/vscodeTypes.ts`) rather than VS Code core.
