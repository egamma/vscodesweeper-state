# Issue #258610 — Chat changed files view formats full paths incorrectly

[https://github.com/microsoft/vscode/issues/258610](https://github.com/microsoft/vscode/issues/258610)

## Summary

Still present on main: the chat resource group widget uses `part.uri.path` (always forward slashes) instead of `ILabelService.getUriLabel()` for the full-name display, producing incorrect path separators on Windows.

## Change summary

Chat changed files view shows full paths with forward slashes on Windows instead of platform-appropriate backslashes.

## Best solution

Inject `ILabelService` into the `ChatResourceGroupWidget` constructor and replace the three `part.uri.path` usages on lines 82, 89, and 95 of `chatResourceGroupWidget.ts` with `this._labelService.getUriLabel(part.uri)` to render platform-correct paths.
