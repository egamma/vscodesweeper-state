# Issue #283598 — Git - Branch Protection Rules Should Use Upstream Branch name

[https://github.com/microsoft/vscode/issues/283598](https://github.com/microsoft/vscode/issues/283598)

## Summary

Team-authored feature request that remains unimplemented. The branch protection check in `isBranchProtected()` only matches against the local branch name (`branch.name`), never the upstream tracking branch name (`branch.upstream.name`). The related merged PR #283606 only added a workspace settings glob — it did not change the matching logic.

## Change summary

Git branch protection rules should match against the upstream/remote tracking branch name, not just the local branch name, so that a local branch named 'main2' tracking 'origin/main' is still protected.

## Best solution

In `extensions/git/src/repository.ts`, `isBranchProtected()` should additionally check `branch.upstream.name` against the protection patterns (lines 3368 and 3377 currently only check `branch.name`). This is a small, localized change in the git extension.
