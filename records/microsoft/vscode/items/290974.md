# Issue #290974 — Can't exit Claude session properly after disabling Claude Agent setting

[https://github.com/microsoft/vscode/issues/290974](https://github.com/microsoft/vscode/issues/290974)

## Summary

Team-authored bug report (MEMBER): disabling the Claude agent setting mid-session leaves the user stuck with no back button or way to create a new session. Keeping open for maintainer judgment.

## Change summary

Disabling `github.copilot.chat.claudeAgent.enabled` while a Claude session is open removes the back button and session title, trapping the user in the current view.

## Best solution

The sessions UI should handle the setting being toggled off gracefully — either navigate back to the sessions list automatically, or retain navigation controls so the user can exit the now-defunct session. Likely involves the sessions window contribution layer (`src/vs/sessions/`) and possibly the copilot chat sessions provider (`src/vs/sessions/contrib/providers/copilotChatSessions/`).
