# Issue #283221 — Changing the GitHub Copilot model in one workspace affects another, even in a different context.

[https://github.com/microsoft/vscode/issues/283221](https://github.com/microsoft/vscode/issues/283221)

## Summary

Still valid on main: model selection is persisted at StorageScope.APPLICATION, so changing the model in one workspace/instance immediately affects all others. PR #320036 improved session-type isolation but did not address cross-workspace scoping.

## Change summary

Changing the Copilot model in one workspace or VS Code instance changes it in all other open workspaces/instances because the selection is stored at the application scope.

## Best solution

Scope model selection storage to the workspace (StorageScope.WORKSPACE) instead of StorageScope.APPLICATION in chatInputPart.ts, at least for the persisted model key. This would let users pick different models per workspace without cross-contamination. The change is localized to the storage key reads/writes in chatInputPart.ts (lines ~1034, 1035, 1815, 1816).
