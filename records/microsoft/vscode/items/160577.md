# Issue #160577 — Regex parse error: missing terminating ] for character class

[https://github.com/microsoft/vscode/issues/160577](https://github.com/microsoft/vscode/issues/160577)

## Summary

Team-authored error-telemetry issue — must keep open. The defect is still present on main: `SearchError` extends plain `Error` (not `ErrorNoTelemetry`), so regex parse errors from ripgrep bubble up as unhandled errors and hit telemetry. No fix has landed.

## Change summary

Regex parse errors from the search service (e.g. invalid user regex) are reported to error telemetry as unhandled errors instead of being caught or wrapped in `ErrorNoTelemetry`.

## Best solution

Either make `SearchError` extend `ErrorNoTelemetry` (so user-caused regex errors don't pollute telemetry), or catch/handle the error at the call site in `searchService.ts` around the `throw searchError` path (line ~340) so it is shown to the user but not re-thrown as an unhandled error. The comment from roblourens already outlines both approaches.
