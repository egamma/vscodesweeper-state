# Issue #160577 — Regex parse error: missing terminating ] for character class

[https://github.com/microsoft/vscode/issues/160577](https://github.com/microsoft/vscode/issues/160577)

## Summary

Team-authored error-telemetry issue that remains open on current main. The SearchError from ripgrep regex parse failures still extends plain Error (not ErrorNoTelemetry), and while the search view catches these errors to display them, the searchModel's onSearchError re-throws non-cancellation errors, letting them reach the global error handler and telemetry. Keeping open per team-authored rule and because the defect path still exists.

## Change summary

Ripgrep regex parse errors (e.g. missing terminating ] for character class) bubble up as unhandled errors to VS Code error telemetry instead of being caught and shown to the user without telemetry noise.

## Best solution

Either make SearchError extend ErrorNoTelemetry so regex parse errors never hit telemetry, or ensure that the re-throw in searchModel.ts onSearchError is removed so the view-level catch is the terminal handler. The fix is small and localized to src/vs/workbench/services/search/common/search.ts (SearchError class) or src/vs/workbench/contrib/search/browser/searchTreeModel/searchModel.ts (onSearchError re-throw).
