# Issue #261186 — Shortcuts inside Quick Open does not work for editor picker. Shortcuts inside editor picker?

[https://github.com/microsoft/vscode/issues/261186](https://github.com/microsoft/vscode/issues/261186)

## Summary

Valid feature request: the editor picker ("edt" prefix) ignores keyMods in its accept handler, so Ctrl+Enter doesn't open in a split editor the way the file picker does. Already correctly labeled and in Backlog.

## Change summary

Allow keyboard shortcuts like Ctrl+Enter (open to side) to work inside the editor picker, matching the behavior of the standard Quick Open file picker.

## Best solution

Extend the `accept` handler in `src/vs/workbench/browser/parts/editor/editorQuickAccess.ts` to respect `keyMods.ctrlCmd`/`keyMods.alt` for opening in a side group (like `anythingQuickAccess.ts` does), bridging the gap between the file picker and editor picker shortcuts.
