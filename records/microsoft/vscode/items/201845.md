# Issue #201845 — Cannot reliably get editors associated with diff editor tab

[https://github.com/microsoft/vscode/issues/201845](https://github.com/microsoft/vscode/issues/201845)

## Summary

Still present on main: the extension API provides no way to associate a TextEditor inside a diff editor with its owning Tab, because diff-embedded editors have viewColumn === undefined and TabInputTextDiff only exposes URIs. When the same diff is open in multiple groups, extensions cannot disambiguate.

## Change summary

Extensions cannot reliably determine which TextEditor belongs to a specific diff editor tab when the same diff is open in multiple editor groups.

## Best solution

Expose an association between TextEditor and its parent Tab (or at minimum, set viewColumn on diff-embedded editors to match the group). The fix likely lives in `src/vs/workbench/api/common/extHostTextEditors.ts` (viewColumn propagation) and/or the tab API surface in `src/vscode-dts/vscode.d.ts` (TabInputTextDiff or TextEditor). This is an API gap that should be discussed by the API team.
