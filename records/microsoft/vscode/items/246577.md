# Issue #246577 — Cant open md with Code.exe

[https://github.com/microsoft/vscode/issues/246577](https://github.com/microsoft/vscode/issues/246577)

## Summary

Portable mode by design does not register VS Code in the Windows application registry, so OS-level file associations (double-click, default-app settings) cannot route to it. A VS Code maintainer confirmed this is expected behavior in an earlier identical report (#73650). Drag-and-drop works because it bypasses OS file association entirely.

## Change summary

User cannot open .md files via double-click or Windows default-app settings when running VS Code in portable mode on Windows.

## Best solution

Close as designed. Portable mode intentionally avoids modifying the system registry, which means Windows file associations cannot discover the portable Code.exe. Users who need file-association support should use the standard installer, which registers VS Code properly. Alternatively, users can manually create the registry entries under HKEY_CLASSES_ROOT\Applications\Code.exe as described in #73650.

  > This is expected behavior for VS Code's portable mode. Portable mode is designed to be fully self-contained and does not register itself in the Windows application registry. Because of this, Windows can't discover it for file-association purposes (double-click or the default-app picker). This is the same behavior discussed in https://github.com/microsoft/vscode/issues/73650.
  > 
  > The options are:
  > - **Use the standard installer** — it registers VS Code with Windows so file associations work out of the box.
  > - **Manually create registry entries** — you can add the portable Code.exe path under `HKEY_CLASSES_ROOT\Applications\Code.exe\shell\open\command` so Windows can find it.
  > - **Drag and drop** — as you discovered, this works because it bypasses file associations entirely.
  > 
  > Closing this as by-design, but feel free to open a feature request if you'd like portable mode to optionally register file associations.
