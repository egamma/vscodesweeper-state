# Issue #187829 — vscode does not automatically open closed repos when a file is opened

[https://github.com/microsoft/vscode/issues/187829](https://github.com/microsoft/vscode/issues/187829)

## Summary

Valid feature request for a setting to control whether manually closed SCM repositories auto-reopen when a file in that repo is opened. The current close-stays-closed behavior is intentional (driven by #107398), but the discussion shows ongoing community demand for a toggle. With 15 upvotes and active discussion, it should remain open under the community process.

## Change summary

Request a setting to automatically reopen a closed git repository in Source Control when a file belonging to that repository is opened.

## Best solution

Add an optional setting (e.g. `git.reopenClosedRepositoryOnFileOpen`) in `extensions/git/src/model.ts` that, when enabled, passes `openIfClosed: true` in the repository-open path triggered by file opens. The `openRepository` method already accepts the `openIfClosed` parameter — the work is wiring a user-facing setting to it. Keep the current default (closed stays closed) for backward compatibility with #107398.
