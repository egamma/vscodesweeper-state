# Issue #325292 — Markdown test failure messages clip long unbreakable text in Test Results view — cannot wrap, scroll, or configure

[https://github.com/microsoft/vscode/issues/325292](https://github.com/microsoft/vscode/issues/325292)

## Summary

Confirmed on current main: `.test-output-peek-message-container .preview-text` lacks `overflow-wrap` and its parent has `overflow: hidden`, so long unbreakable tokens (stack traces) in Markdown test messages are clipped with no way to read them.

## Change summary

Long unbreakable text in Markdown-rendered test failure messages is clipped in the Test Results view because the CSS lacks overflow-wrap.

## Best solution

Add `overflow-wrap: anywhere;` to `.test-output-peek-message-container .preview-text` in `src/vs/workbench/contrib/testing/browser/media/testing.css`. This is a one-line CSS fix the reporter has already verified locally.
