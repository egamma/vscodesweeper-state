# Issue #317766 — git-commit editor syntax highlighting should respect git.inputValidationSubjectLength

[https://github.com/microsoft/vscode/issues/317766](https://github.com/microsoft/vscode/issues/317766)

## Summary

Valid feature request: the git-commit TextMate grammar hardcodes 50/72 char boundaries while git.inputValidationSubjectLength only applies to the SCM input box diagnostics. Bridging these requires runtime-aware highlighting, which TextMate grammars cannot do statically.

## Change summary

Make COMMIT_EDITMSG editor syntax highlighting respect the git.inputValidationSubjectLength setting instead of hardcoding 50/72 characters.

## Best solution

The git extension could add a DocumentSemanticTokensProvider or diagnostic provider for COMMIT_EDITMSG files that reads git.inputValidationSubjectLength at runtime, similar to how extensions/git/src/diagnostics.ts already does for the SCM input. The static TextMate grammar in git-base would either be left as a fallback or have its line-length pattern removed.
