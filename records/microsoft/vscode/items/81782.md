# Issue #81782 — [json] allow $ref in confiuguration schemas

[https://github.com/microsoft/vscode/issues/81782](https://github.com/microsoft/vscode/issues/81782)

## Summary

Still an open feature request — the configuration extension point does not support `$ref` or `definitions`/`$defs` in contributed schemas. Related issue #92042 (JSON `$id` references in the language service) shipped in Nov 2021, but that's a different scope than allowing `$ref` in `contributes.configuration`.

## Change summary

Allow extensions to use JSON Schema `$ref` and `definitions` in their `contributes.configuration` schemas so complex settings can be DRY.

## Best solution

Add `$ref`/`definitions` resolution to the configuration extension point schema processor (likely in `src/vs/workbench/api/common/configurationExtensionPoint.ts` and `src/vs/platform/configuration/common/configurationRegistry.ts`). The JSON language service already supports `$ref` in schema files; the gap is at the configuration contribution layer.
