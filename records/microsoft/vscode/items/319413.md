Team-authored bug still open on main — chat agent repeatedly polls a long-running async terminal task, adding unnecessary status-check entries. No fix commit found. Keeping open per team-member policy.

## Change summary

When Copilot kicks off a long-running async terminal task, the chat agent repeatedly spins up to check its status, flooding the conversation with unnecessary entries.

## Best solution

The terminal tool's async-mode handling should detect that a task is still running and avoid re-querying unless the user explicitly asks for a status update, or implement a backoff/cooldown. Likely lives in the chat terminal tool integration layer (possibly model-prompt-level guidance about not re-checking async tasks).
