# Issue #208942 — File explorer only supports drag and drop of a single item

[https://github.com/microsoft/vscode/issues/208942](https://github.com/microsoft/vscode/issues/208942)

## Summary

The single-file limitation when dragging from the explorer to the OS desktop is a known Chromium/Electron constraint on the `DownloadURL` data transfer type, which only supports one file. The code explicitly documents this at `src/vs/workbench/browser/dnd.ts:244-248`. This is an upstream limitation, not a VS Code bug per se, but VS Code could potentially work around it using Electron's native drag APIs.

## Change summary

Dragging multiple selected files from VS Code's file explorer to the OS desktop or file manager only copies a single file instead of all selected files.

## Best solution

Investigate using Electron's `webContents.startDrag` API (which supports multiple file paths) as an alternative to the Chromium `DownloadURL` data transfer for file-scheme resources. This would bypass the single-file Chromium limitation. The fix would live in `src/vs/workbench/browser/dnd.ts` (the `fillEditorsDragData` function) and possibly `src/vs/workbench/contrib/files/browser/views/explorerViewer.ts` (`onDragStart`). Alternatively, track the upstream Chromium limitation and wait for multi-file `DownloadURL` support.
