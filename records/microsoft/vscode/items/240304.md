# Issue #240304 — Can't sign into Azure from VS Code with the new authentication. VPN blocking native auth flow

[https://github.com/microsoft/vscode/issues/240304](https://github.com/microsoft/vscode/issues/240304)

## Summary

Upstream issue in the WAM/native broker library (backgroundtaskhost.exe, Microsoft.AAD.BrokerPlugin.exe) not routing through VPNs. VS Code has mitigated this with the permanent `msal-no-broker` setting, and the maintainer confirmed it resolves the scenario. Keeping open as the upstream tracking item.

## Change summary

Native auth broker (WAM) used by MSAL authentication doesn't route through VPN tunnels, preventing sign-in for users whose Azure access requires VPN.

## Best solution

The `microsoft-authentication.implementation: msal-no-broker` setting already ships on main as the permanent mitigation. The underlying fix belongs in Microsoft's WAM/broker library (routing broker traffic through the system VPN). This issue can serve as the upstream tracker until WAM resolves VPN routing, or be closed once the team decides the `msal-no-broker` escape hatch is sufficient.
