# Issue #58315 — [html] Automatically delete HTML closing tag when converting to self-closing tag

[https://github.com/microsoft/vscode/issues/58315](https://github.com/microsoft/vscode/issues/58315)

## Summary

Valid feature request in the Backlog with 36 upvotes; not yet implemented on main. The HTML auto-insertion logic only handles inserting closing tags, not removing them when a tag becomes self-closing.

## Change summary

When a user converts an HTML/JSX opening tag to a self-closing tag (adding `/>`), the paired closing tag should be automatically deleted.

## Best solution

Extend the auto-insertion logic in `extensions/html-language-features/client/src/autoInsertion.ts` (or linked editing in the HTML language server) to detect when a tag becomes self-closing and remove the corresponding closing tag. This likely also involves the HTML language server's `htmlMode.ts` for the actual document edit computation.
