# Issue #58315 — [html] Automatically delete HTML closing tag when converting to self-closing tag

[https://github.com/microsoft/vscode/issues/58315](https://github.com/microsoft/vscode/issues/58315)

## Summary

This is a valid, community-supported feature request (20+ upvotes, moved to Backlog in 2020) that has not been implemented on current main. The HTML language features extension has auto-close and linked editing but not the specific self-closing conversion behavior requested.

## Change summary

Automatically delete the HTML/JSX closing tag when the opening tag is converted to a self-closing tag (e.g., typing `/>`).

## Best solution

Implement in `extensions/html-language-features` (likely `client/src/autoInsertion.ts` or via the language server's `htmlMode.ts`) a handler that detects when an opening tag becomes self-closing and removes the paired closing tag. The linked editing infrastructure could be leveraged to identify the matching close tag.
