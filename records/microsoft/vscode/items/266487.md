# Issue #266487 — Running task terminal left open after extension host reload

[https://github.com/microsoft/vscode/issues/266487](https://github.com/microsoft/vscode/issues/266487)

## Summary

Still reproducible on main: CustomExecution tasks lose their extension-host-side pseudoterminal on extension host restart, leaving a zombie terminal that blocks re-running the task. No fix has landed for this specific lifecycle gap.

## Change summary

After extension host reload, CustomExecution task terminals remain open in a frozen state and block re-running the same task without manual terminal cleanup.

## Best solution

When the extension host restarts, CustomExecution task terminals whose backing TerminalProcessExtHostProxy is disconnected should be marked as ended (dispose or show exit status) so the task system no longer considers them active. The fix likely belongs in src/vs/workbench/contrib/tasks/browser/terminalTaskSystem.ts around the reconnection path and the extension host lifecycle listener.
