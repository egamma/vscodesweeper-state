# Issue #279415 — Scrolling through long single line files is terrible.

[https://github.com/microsoft/vscode/issues/279415](https://github.com/microsoft/vscode/issues/279415)

## Summary

Valid UX improvement request for single-line files. The current OUTSIDE_EDITOR detection in mouseHandler.ts has no special tolerance for files with very few lines, causing selections to jump to line boundaries with minimal vertical mouse drift. No fix has been made on main. Already triaged as feature-request in Backlog with 0 upvotes.

## Change summary

Mouse selection in long single-line files jumps to line start/end when the cursor drifts slightly above or below the text line.

## Best solution

Increase the vertical tolerance for OUTSIDE_EDITOR detection in `src/vs/editor/browser/controller/mouseHandler.ts` when the file has very few lines (e.g. 1–2), so small vertical mouse drift during horizontal selection doesn't trigger the above/below jump behavior.
