# Issue #316051 — Copilot Chat Agent uses wrong session/file context when multiple chat sessions are active

[https://github.com/microsoft/vscode/issues/316051](https://github.com/microsoft/vscode/issues/316051)

## Summary

The editorContext injection in Agent mode uses the globally active editor rather than anything session-scoped, so follow-up prompts in one chat session can silently target the active file from another session. The defect path is still present on main — `inferDocumentContext` in `documentContext.ts` falls through to `activeEditor` without session awareness, and the setting `chat.agent.currentEditorContext.enabled` merely toggles the feature wholesale rather than scoping it per session. Keeping open as a real bug that needs a design fix.

## Change summary

When multiple chat sessions are open, implicit editorContext injection picks up the globally active editor rather than the session's own file context, causing the agent to edit the wrong file.

## Best solution

Scope the editor context injection per chat session: either track which editor/file the session was last interacting with and inject that, or (at minimum) suppress implicit editorContext when the session history already references a different file. The fix likely touches `extensions/copilot/src/extension/prompt/node/documentContext.ts` (the `inferDocumentContext` function) and wherever the `CurrentEditorAgentContext` setting is consumed to build the agent prompt. Issue https://github.com/microsoft/vscode/issues/305251 tracks a related-but-narrower stale-context variant and could be addressed in the same pass.
