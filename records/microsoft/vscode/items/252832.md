# Issue #252832 — Diff:Switch sides does not announce which side is active to screen reader (JAWS)

[https://github.com/microsoft/vscode/issues/252832](https://github.com/microsoft/vscode/issues/252832)

## Summary

Still reproduces on main: the `switchSide()` method in `diffEditorWidget.ts` calls `destination.focus()` without any ARIA announcement, so screen reader users have no indication of which side (original vs modified) became active.

## Change summary

The diff editor's Switch Side command does not announce to screen readers which side is now focused.

## Best solution

After `destination.focus()` in `switchSide()`, announce which editor gained focus (e.g. 'Original editor' or 'Modified editor') via an ARIA alert or the accessibility signal service, similar to how other editor navigation actions announce state changes.
