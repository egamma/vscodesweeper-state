# Issue #305685 — Using "Toggle between Light/Dark Themes" resets theme

[https://github.com/microsoft/vscode/issues/305685](https://github.com/microsoft/vscode/issues/305685)

## Summary

The toggle command uses `workbench.preferredDarkColorTheme` / `workbench.preferredLightColorTheme` settings by design; the defaults changed in 1.113 to 'VSCode Dark'/'VSCode Light'. A maintainer already explained the behavior and provided a workaround. The reporter's follow-up is a separate enhancement suggestion (auto-syncing preferredDark/LightColorTheme when the user changes colorTheme).

## Change summary

Toggle between Light/Dark Themes uses the new 1.113 default themes instead of the user's previously active theme.

## Best solution

Close as designed. The reporter's suggestion to auto-sync `preferredDarkColorTheme`/`preferredLightColorTheme` when `colorTheme` changes is a valid separate feature request — if worth pursuing, it should be filed as a new issue.
