# Issue #305685 — Using "Toggle between Light/Dark Themes" resets theme

[https://github.com/microsoft/vscode/issues/305685](https://github.com/microsoft/vscode/issues/305685)

## Summary

Valid enhancement request: syncing preferredDarkColorTheme/preferredLightColorTheme when the user changes colorTheme. The behavior is as-designed per the maintainer's explanation, but the reporter's suggestion to auto-sync is a reasonable UX improvement that hasn't been implemented yet.

## Change summary

When a user changes their color theme, the corresponding preferred dark/light theme setting should update automatically so that Toggle Light/Dark reflects the user's actual preference.

## Best solution

Update the theme configuration service in `src/vs/workbench/services/themes/common/workbenchThemeService.ts` to detect when `workbench.colorTheme` changes and auto-set the matching `preferredDarkColorTheme` or `preferredLightColorTheme` based on the new theme's type (dark/light). The closed PR #309182 attempted this approach.
