# Issue #291735 — Delegate button shows background/cloud options when not in a git repository

[https://github.com/microsoft/vscode/issues/291735](https://github.com/microsoft/vscode/issues/291735)

## Summary

Still reproduces on main: the delegation picker in the regular VS Code window (non-sessions-window) shows Background and Cloud options even without a git repository. The git-gating fix (commit 6edd5669785, PR #302979) only applied to the sessions window; the core VS Code chat delegate picker is unguarded.

## Change summary

The delegate button in the regular VS Code Copilot Chat shows Background and Cloud agent options in non-git workspaces, even though those options require git to function.

## Best solution

Extend the git-repository guard in `_isSessionTypeEnabled` (line 93–96 of `delegationSessionPickerActionItem.ts`) to also apply in the regular VS Code window, not just the sessions window. This mirrors the pattern already established by PR #302979 for the sessions window. Consider also gating the Background option on git, since background agents also require git worktrees.
