# Issue #291735 — Delegate button shows background/cloud options when not in a git repository

[https://github.com/microsoft/vscode/issues/291735](https://github.com/microsoft/vscode/issues/291735)

## Summary

The core ask — preventing confusing delegation options in non-git workspaces — was addressed: Cloud delegation is now disabled (greyed out) when no git repository is detected, both in the main window's Continue In action and the sessions window's delegation picker. Background delegation remains available by design, as background agents handle their own git setup.

## Change summary

The delegate button showed Background and Cloud agent options as fully enabled even in non-git workspaces, making them appear usable when they would fail.

## Best solution

Keep the shipped fix: Cloud delegation is disabled with a 'Requires a Git repository' message in non-git workspaces (commit 6edd5669785, shipped in 1.113.0, with follow-up refinements in eb8922169d4). No further change needed.
