# Issue #215988 — "Cannot set property activeDebugSession of #<Object> which has only a getter" when trying to change active debug session

[https://github.com/microsoft/vscode/issues/215988](https://github.com/microsoft/vscode/issues/215988)

## Summary

Valid feature request: extensions cannot programmatically set the active debug session despite `vscode.d.ts` declaring it as `let`. The getter-only implementation in `extHostDebugService.ts` confirms no setter exists. The use case (extension-managed debug session selection UI) is reasonable and unaddressed. Keeping open in Backlog.

## Change summary

Allow extensions to programmatically set `vscode.debug.activeDebugSession` to change the focused debug session.

## Best solution

Add a setter for `activeDebugSession` in `ExtHostDebugService` that calls back to the main thread's debug service to focus the specified session, or expose a dedicated command/API like `vscode.debug.setActiveDebugSession(session)`. The API declaration in `vscode.d.ts` (line 17292) already uses `let`, so adding write support would fulfill the implied contract. Related issue https://github.com/microsoft/vscode/issues/216555 about normalizing `let`/`const` should coordinate — if this feature is accepted, keep it as `let`; otherwise change to `const`.
