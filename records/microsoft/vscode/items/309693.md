## Summary

Real bug confirmed by a team member (Giuspepe), with an assigned 'On Deck' milestone — must stay open. Multiple agent sessions sharing a worktree cause button duplication and misfiring across sessions.

## Change summary

When multiple agent sessions share the same worktree, action buttons (like 'Sync Pull Requests') duplicate and fire in the wrong session.

## Best solution

The Agents window session contributions (buttons/actions) need to be scoped per-session so that shared worktree state doesn't cause cross-session leakage of UI contributions. Likely in the sessions layer's contribution or toolbar registration logic under `src/vs/sessions/`.
