# Issue #324903 — UX: Inability to switch between Microsoft and GitHub accounts without first turning off Settings Sync is highly unintuitive

[https://github.com/microsoft/vscode/issues/324903](https://github.com/microsoft/vscode/issues/324903)

## Summary

Valid feature request for a more intuitive account-provider switching flow in Settings Sync. No existing open canonical issue owns this exact UX gap. The related issues cited were all closed as info-needed, so this is a fresh articulation of a recurring pain point worth routing to the settings-sync area.

## Change summary

Users cannot switch between Microsoft and GitHub accounts for Settings Sync without the undiscoverable step of running 'Settings Sync: Turn Off' from the command palette first.

## Best solution

Add a 'Switch Account' or 'Change Provider' affordance in the Accounts menu or Settings Sync UI that re-presents the provider selection without requiring users to fully turn off sync. This belongs in the settings-sync area (likely `src/vs/workbench/services/userDataSync/browser/`).
