# Issue #215617 — Long prompt text with shell integration causes "ghost" text display in the VSCode bash terminal

[https://github.com/microsoft/vscode/issues/215617](https://github.com/microsoft/vscode/issues/215617)

## Summary

The issue is on the 'On Deck' milestone (a scheduling decision), so it must stay open. The bug remains valid: long PS1 prompts in bash with shell integration cause ghost text due to the shell integration script wrapping PS1 with escape sequences that push the total length past terminal column width, causing line-wrap artifacts.

## Change summary

Long bash PS1 prompts with shell integration enabled display ghost text (escaped sequences/partial prompt fragments) before each prompt line.

## Best solution

The shell integration bash script wraps PS1 with additional escape sequences (visible in the `\[\]` bookends around the echoed PS1). When the total wrapped prompt exceeds terminal width, the invisible characters are miscounted for line-wrap purposes, causing raw escape residue to render. The fix likely belongs in `src/vs/workbench/contrib/terminal/common/scripts/shellIntegration-bash.sh` — ensure the injected markers don't contribute to visible prompt length calculations, or use a different injection strategy for long prompts.
