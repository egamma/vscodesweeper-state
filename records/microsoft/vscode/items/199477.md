# Issue #199477 — Clicking inside editor and releasing the mouse button on a link outside activates the link (web/monaco)

[https://github.com/microsoft/vscode/issues/199477](https://github.com/microsoft/vscode/issues/199477)

## Summary

Keep open as a still-actionable editor-core/Monaco web bug. The issue was acknowledged by an editor maintainer as likely fixable in the mouse handler, and current main does not show a concrete fixing commit or code change that would close the Firefox click-through behavior.

## Change summary

Dragging from inside the editor and releasing over an external link/button can activate that external element in Firefox/web, even though the original mouse down started in Monaco.

## Best solution

Handle the browser event in the editor mouse/pointer release path so a drag or selection that starts in Monaco cannot turn into an external click activation, likely around `src/vs/editor/browser/controller/mouseHandler.ts` and `src/vs/base/browser/globalPointerMoveMonitor.ts`, with a Firefox/web regression test or manual verification.
