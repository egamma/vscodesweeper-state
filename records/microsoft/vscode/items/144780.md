# Issue #144780 — Git - repository auto-detection not working for junctions on windows

[https://github.com/microsoft/vscode/issues/144780](https://github.com/microsoft/vscode/issues/144780)

## Summary

Still reproduces on main: `traverseWorkspaceFolder` in the git extension filters subdirectories with `dirent.isDirectory()` only, which does not follow Windows NTFS junctions — repositories behind junctions are never scanned. The issue also carries the 'On Deck' milestone, which is a hard keep-open.

## Change summary

Git repositories located inside NTFS junctions are not auto-detected by the Source Control panel; the user must manually open a file from each repository to trigger detection.

## Best solution

In `extensions/git/src/model.ts`, the `traverseWorkspaceFolder` method's directory filter (line ~432) should include `dirent.isSymbolicLink()` alongside `dirent.isDirectory()` so that junctions (and symlinks) are traversed during workspace scanning. A `realpath` call on the resolved path may also be needed to normalize paths for duplicate-repository detection.
