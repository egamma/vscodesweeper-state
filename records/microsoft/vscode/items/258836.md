# Issue #258836 — Should secret storage allow using empty key?

[https://github.com/microsoft/vscode/issues/258836](https://github.com/microsoft/vscode/issues/258836)

## Summary

Active design discussion about whether to add input validation (rejecting empty keys) to the SecretStorage API. A team member engaged and raised the parallel to Mementos; no resolution yet.

## Change summary

SecretStorage.store() currently accepts an empty string as a key, which is likely unintentional and could confuse extension authors.

## Best solution

Decide whether to throw or warn on empty keys in `ExtensionSecrets.store()`/`get()`/`delete()` (in `src/vs/workbench/api/common/extHostSecrets.ts`), and whether the same validation should apply to Mementos. If yes, add an early guard with a clear error message.
