# Issue #169922 — "Reveal in Explorer View" misbehaves with out-of-workspace files

[https://github.com/microsoft/vscode/issues/169922](https://github.com/microsoft/vscode/issues/169922)

## Summary

Still reproduces on main: the 'Reveal in Explorer View' command for non-workspace files expands the Open Editors view but does not select the specific file, so it doesn't actually 'reveal' anything. A team member acknowledged the issue and proposed either disabling the entry or fixing it, but no decision was reached.

## Change summary

"Reveal in Explorer View" on a non-workspace file opens the Open Editors view without selecting the specific file, defeating the purpose of the command.

## Best solution

Either (a) select/reveal the specific file in the Open Editors view tree when it's outside the workspace, or (b) hide the 'Reveal in Explorer View' context menu entry for non-workspace files as lramos15 suggested. The fix lives in `src/vs/workbench/contrib/files/browser/fileCommands.ts` lines 338–345. A maintainer decision is needed on which approach to take.
