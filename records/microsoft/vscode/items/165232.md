# Issue #165232 — Incorrect highlighting of tsx code inside Markdown code blocks (bracket token exclusion rules of embedded grammars are not considered)

[https://github.com/microsoft/vscode/issues/165232](https://github.com/microsoft/vscode/issues/165232)

## Summary

Still present on main: bracket pair colorization only uses the root grammar's bracket token exclusion rules, ignoring those of embedded grammars. The fix proposed by hediet (per-bracket-type scopes) was never implemented.

## Change summary

Bracket pair colorization incorrectly highlights TSX/TS code in Markdown code blocks because embedded grammar bracket exclusion rules are not applied.

## Best solution

Implement per-bracket-type scope filtering (the `bracketScopes` proposal from hediet's comment 4) so that embedded grammars like TypeScript can specify that `<`/`>` are only brackets in specific token scopes, while `()`, `[]`, `{}` remain universally matched. This lives in the vscode-textmate integration layer and the bracket pair colorization engine.
