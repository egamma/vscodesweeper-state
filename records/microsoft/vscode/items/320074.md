# Issue #320074 — IME: Duplicate text appears after converting Japanese input when moving caret before conversion

[https://github.com/microsoft/vscode/issues/320074](https://github.com/microsoft/vscode/issues/320074)

## Summary

Active IME bug with Japanese input where arrow-key caret movement during composition causes duplicate text on conversion. A team member is engaged and the reporter provided detailed repro steps. No fix found on main.

## Change summary

Japanese IME input produces duplicate text when the caret is moved with arrow keys before confirming conversion, specifically when IME predictive text is disabled.

## Best solution

Investigate the composition handling in `src/vs/editor/browser/controller/editContext/native/nativeEditContext.ts` and the textarea fallback path — the arrow-key movement during an active composition likely leaves stale uncommitted text that isn't cleaned up on final conversion confirm. The fix likely needs to handle caret movement as an implicit composition cancel or ensure the pre-edit text is replaced (not appended) on confirm.
