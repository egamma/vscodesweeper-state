# Issue #173739 — Copilot hover covers parameter hint

[https://github.com/microsoft/vscode/issues/173739](https://github.com/microsoft/vscode/issues/173739)

## Summary

Still present on main: the parameter hints content widget positioning does not account for ghost text (inline completions), so they can visually overlap. No fix found in git history. Labeled as feature-request already; keeping open.

## Change summary

The parameter hints widget overlaps with inline completion ghost text instead of being rendered next to it.

## Best solution

The content widget positioning logic in `src/vs/editor/browser/viewParts/contentWidgets/contentWidgets.ts` or the parameter hints widget in `src/vs/editor/contrib/parameterHints/browser/parameterHintsWidget.ts` should account for the width of rendered ghost text when computing widget placement, so the two don't overlap.
