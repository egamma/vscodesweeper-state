## Change summary

Ctrl+clicking a subfolder name in Git Bash terminal navigates to the wrong folder in File Explorer when identically-named folders exist at different directory levels.

## Best solution

If confirmed on latest, the fix would be in `terminalLinkOpeners.ts` / `terminalLinkHelpers.ts` — the `_getExactMatch` fallback uses `_initialCwd` when command detection is unavailable (common in Git Bash), which resolves relative paths against the wrong directory. Enabling shell integration in Git Bash or improving the cwd fallback would address this.
