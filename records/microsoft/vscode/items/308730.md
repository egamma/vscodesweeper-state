# Issue #308730 — GIthub Copilot Chat session cannot be reopened from Sessions list due to malformed chatSessions JSONL (missing base record / BOM parse error)

[https://github.com/microsoft/vscode/issues/308730](https://github.com/microsoft/vscode/issues/308730)

## Summary

The root write-corruption path was fixed in #309292 (5e9baab3da4), but already-corrupted files remain permanently unrecoverable and the BOM variant is unhandled. An open PR #309286 would add fallback resilience but has not merged. The issue remains partially valid.

## Change summary

Copilot Chat sessions saved as JSONL can become permanently unreadable if the file lacks a base record (kind:0) or contains a BOM, with no recovery or graceful fallback.

## Best solution

Land the open PR https://github.com/microsoft/vscode/pull/309286 (fall back to full snapshot on serializer failure) and add BOM-stripping in the JSONL read path in `src/vs/workbench/contrib/chat/common/model/chatSessionStore.ts` so corrupted-but-salvageable sessions can be recovered rather than silently failing.
