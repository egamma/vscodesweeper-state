# Issue #43365 — [css] propose ids used in other selectors

[https://github.com/microsoft/vscode/issues/43365](https://github.com/microsoft/vscode/issues/43365)

## Summary

Still valid on current main: the CSS language service's `getCompletionsForSelector` only collects `.class` selectors, not `#id` selectors. The broader issue #47331 tracks cross-file CSS/HTML support including IDs, but same-file ID completion is a simpler subset that remains unimplemented. Keeping open as a valid feature request with the On Deck milestone.

## Change summary

CSS files suggest previously-used class selectors (`.foo`) but not ID selectors (`#bar`) when typing a new selector in the same file.

## Best solution

The fix belongs in `vscode-css-languageservice` (upstream dependency): extend the selector visitor in `getCompletionsForSelector` to also collect `#`-prefixed selectors alongside `.`-prefixed ones. Alternatively, this could be absorbed by the broader https://github.com/microsoft/vscode/issues/47331 if that lands first. The milestone is On Deck so a maintainer should decide whether to pursue the narrow same-file fix independently or wait for the broader cross-file work.
