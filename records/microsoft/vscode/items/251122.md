# Issue #251122 — Respect explorer.excludeGitIgnore in Multi-root Workspace Folders

[https://github.com/microsoft/vscode/issues/251122](https://github.com/microsoft/vscode/issues/251122)

## Summary

Verified duplicate: the root cause is the same as #172482 — the explorer's gitignore tree is built per workspace-folder root, so `.gitignore` files above a workspace folder are never discovered or applied. Both issues ask for the same fix.

## Change summary

In a multi-root workspace, `explorer.excludeGitIgnore` does not hide files matched by a `.gitignore` that lives above a workspace folder (e.g. at the git repo root).

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/172482, which is the canonical issue for this limitation. Fixing `FilesFilter` to walk up to the git repo root when building the ignore tree would resolve both issues.

## Evidence

- **root cause in source:** src/vs/workbench/contrib/files/browser/views/explorerViewer.ts — `ignoreTreesPerRoot` (line 1238) and `ignoreFileResourcesPerRoot` (line 1234) are keyed per workspace folder URI. The `filter` method (line 1386) only discovers `.gitignore` files encountered during tree traversal within each workspace root. A `.gitignore` above the workspace folder root is never traversed or processed.
- **canonical issue:** https://github.com/microsoft/vscode/issues/172482 (open, filed 2023-08-08) describes the identical scenario: multi-root workspace whose folders are subfolders of a git repo, and `.gitignore` at the repo root is not respected for those workspace folders.
- **related issue:** https://github.com/microsoft/vscode/issues/189951 (open) is also about the same limitation — not walking up to the git repo root for `.gitignore` discovery — and is cross-referenced from #172482.
- **no fix on main:** git log on explorerViewer.ts since the issue was filed shows no commit addressing multi-root gitignore parent traversal. The limitation remains on current main.
