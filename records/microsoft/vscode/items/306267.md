# Issue #306267 — Stale task output captured switching configurations with terminals reuse

[https://github.com/microsoft/vscode/issues/306267](https://github.com/microsoft/vscode/issues/306267)

## Summary

Team-authored bug report — keeping open. The defect is source-reproducible: `_capturedTaskVariables` in `TerminalTaskSystem` is a single shared map that is never cleared between task runs, so stale values from a previous task are resolved when a terminal is reused with a different configuration.

## Change summary

When switching launch configurations that reuse background task terminals, `taskVar` variables resolve to stale values from the previous task run instead of the current one.

## Best solution

Clear (or scope per-task) the `_capturedTaskVariables` map in `TerminalTaskSystem` before a reused terminal begins a new task execution, so the variable resolver waits for fresh captured output rather than returning stale values.
