# Issue #283335 — Terminal Sticky Scroll causes incorrect screen size detection for scrolling

[https://github.com/microsoft/vscode/issues/283335](https://github.com/microsoft/vscode/issues/283335)

## Summary

Still reproducible on main: terminal sticky scroll is a visual overlay that does not adjust the PTY-reported rows, causing programs like pagers and full-screen TUI apps to miscalculate available screen space. The existing pager detection (lines 300-313 in terminalStickyScrollOverlay.ts) is a partial workaround for `less`/`git log` at-bottom states only, but doesn't cover the general case of scrolling or screen-filling output.

## Change summary

Terminal sticky scroll overlay visually obscures top terminal rows without adjusting the PTY row count, causing programs to render content behind the overlay and making scroll/page navigation miss lines.

## Best solution

Either reduce the reported PTY rows when sticky scroll is visible (complex — involves resize events and potential re-rendering by the application) or expand the existing pager-detection heuristic to hide sticky scroll in more cases where it would obscure content. A third option: offset the terminal viewport rendering downward by the sticky scroll height so no content is hidden. The fix likely involves both terminalStickyScrollOverlay.ts and potentially the terminal dimension calculation in terminalInstance.ts.
