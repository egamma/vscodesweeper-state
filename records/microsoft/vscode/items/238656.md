# Issue #238656 — Allow customization of editor tab height and editor tab bar scrollbar height

[https://github.com/microsoft/vscode/issues/238656](https://github.com/microsoft/vscode/issues/238656)

## Summary

The existing `window.density.editorTabHeight` setting only offers `default` vs `compact` (two preset heights), not freeform numeric customization. The request for pixel-level tab height and scrollbar height control remains unimplemented. Low upvote count (1) means this stays in the community process.

## Change summary

Allow users to set editor tab height and tab-bar scrollbar height to arbitrary pixel values rather than only two presets.

## Best solution

Extend `window.density.editorTabHeight` to accept a numeric pixel value (or add a separate numeric setting) in `src/vs/workbench/browser/parts/editor/editorTabsControl.ts` and the contribution in `workbench.contribution.ts`. Scrollbar height for the tab bar would need a separate CSS variable or setting. However, with only 1 upvote this is unlikely to gain traction soon — let the community process decide.
