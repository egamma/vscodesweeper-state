# Issue #198116 — Explore quick fixes and command decoration being attached to the previous command (with help of sticky scroll)

[https://github.com/microsoft/vscode/issues/198116](https://github.com/microsoft/vscode/issues/198116)

## Summary

This is a known terminal UX issue filed by the terminal maintainer, tracking a design problem where sticky scroll causes decorations and quick fixes to attach to the wrong command. The attempted fix PR #263297 was closed without merging, so the problem remains open and valid.

## Change summary

Terminal quick fixes and command decorations incorrectly attach to the previous command when sticky scroll is active.

## Best solution

The terminal team needs to revisit how command boundary detection interacts with sticky scroll overlay positioning, likely in `terminalStickyScrollOverlay.ts` and `quickFixAddon.ts`. The closed PR #263297 attempted one approach; a different design is needed.
