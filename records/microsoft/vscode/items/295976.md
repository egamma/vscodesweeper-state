# Issue #295976 — "Proceed from plan" section missing when session is reopened

[https://github.com/microsoft/vscode/issues/295976](https://github.com/microsoft/vscode/issues/295976)

## Summary

Still reproduces on main: when a session with a plan review is deserialized, all `planReview` parts have `isUsed` set to `true` (chatModel.ts line 2648), which hides the 'proceed from plan' buttons. The workaround of switching modes confirms the state is recoverable but the restore path doesn't handle it.

## Change summary

The 'Proceed from plan' section with its action buttons disappears when a plan-mode session is reopened, because deserialization marks all planReview parts as used.

## Best solution

When restoring a session that was in plan mode and the last planReview was not yet acted upon, either re-register the plan review listener so the buttons remain functional, or show a clear 'plan expired — regenerate' state instead of silently hiding the section. The fix likely involves `chatModel.ts` deserialization (around line 2648) and potentially `chatPlanReviewPart.ts`.
