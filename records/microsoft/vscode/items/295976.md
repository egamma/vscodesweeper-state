# Issue #295976 — "Proceed from plan" section missing when session is reopened

[https://github.com/microsoft/vscode/issues/295976](https://github.com/microsoft/vscode/issues/295976)

## Summary

Still reproduces on main: the deserialization path in chatModel.ts explicitly marks planReview parts as `isUsed = true` on session restore (line 2640), which intentionally hides the 'proceed from plan' UI. The issue is a real gap — the team has triaged it as a bug in Backlog.

## Change summary

When reopening a chat session containing a plan, the 'Proceed from plan' section with the 'Start Implementation' button is missing because the plan review widget is marked as used during deserialization.

## Best solution

Re-establish the plan review interaction capability on session restore rather than unconditionally marking it as `isUsed`. The fix likely involves `src/vs/workbench/contrib/chat/common/model/chatModel.ts` (line ~2640) and may require the plan review tool to support re-registration of its completion promise after deserialization, or a UX that re-creates the plan review part when plan mode is re-entered.
