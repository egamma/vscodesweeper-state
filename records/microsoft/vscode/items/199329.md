# Issue #199329 — The sash gets away from the mouse when using scroll wheel

[https://github.com/microsoft/vscode/issues/199329](https://github.com/microsoft/vscode/issues/199329)

## Summary

Team-authored bug, still unfixed on main. The sash drag handler doesn't account for scroll-wheel changes during a drag operation, causing the sash position to drift from the cursor.

## Change summary

Dragging a sash in the diff editor while using the scroll wheel causes the sash to lose alignment with the mouse cursor.

## Best solution

Update the sash drag logic in `src/vs/base/browser/ui/sash/sash.ts` to account for scroll offset changes during an active drag, or re-anchor the sash position on wheel events. The diff-editor's scroll-synced sash in `src/vs/editor/browser/widget/diffEditor/` may also need adjustment.
