# Issue #248110 — SCM - scm.workingSets.enabled steals focus from integrated terminal

[https://github.com/microsoft/vscode/issues/248110](https://github.com/microsoft/vscode/issues/248110)

## Summary

Still reproduces on main. The earlier fix (commit bbb3b1c06b4) added a `preserveFocus` guard that only checks `Parts.PANEL_PART`, but the `applyState` calls inside `editorParts.ts#applyWorkingSet` (lines 612/616) can themselves shift focus during editor open/close operations before the `preserveFocus` gate at line 619 is even reached. The issue is milestoned On Deck, so it stays open regardless.

## Change summary

Switching git branches with `scm.workingSets.enabled` steals focus from the integrated terminal, despite an earlier fix that attempted to preserve panel focus.

## Best solution

The focus-preservation logic in `workingSet.ts` line 157 needs to be broader: (1) capture the currently focused element/part before _any_ editor state changes, not just check `PANEL_PART`, and (2) ensure the `applyState` calls in `editorParts.ts` lines 612–616 propagate `preserveFocus` so intermediate editor open/close operations don't grab focus. The fix likely touches `workingSet.ts` and `editorParts.ts` / `editorPart.ts`.
