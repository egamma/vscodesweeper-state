# Issue #260188 — Copilot Chat: chat summarization makes broken and bad edits

[https://github.com/microsoft/vscode/issues/260188](https://github.com/microsoft/vscode/issues/260188)

## Summary

Team-authored bug with multiple MEMBER confirmations describing context loss after chat summarization, leading to broken edits. Significant ongoing work on summarization/compaction has landed since filing (custom compaction models, /compact fixes, telemetry improvements), but the core issue — the agent losing critical file context post-summarization and making incorrect edits — remains an active area of investment. Keeping open per team-authored rule and because the fundamental problem persists.

## Change summary

After chat conversation summarization, the agent loses context of files being edited (attachments, prompt files, tool results) and makes incorrect edits with fabricated content.

## Best solution

Improve summarization to preserve critical editing context — ensure files actively being edited remain attached or are re-read post-summarization, and strengthen the post-summary system prompt to instruct the model to re-read files before editing. The compaction pipeline in `extensions/copilot` is the right place; recent work (custom trajectory compaction model in commit 43d80e3ba38, /compact fixes) addresses parts of this but the core context-preservation gap remains.
