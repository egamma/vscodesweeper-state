# Issue #289346 — Chat markdown rendering of numbered lists is misleading

[https://github.com/microsoft/vscode/issues/289346](https://github.com/microsoft/vscode/issues/289346)

## Summary

Still reproduces on main: the chat markdown renderer uses standard marked.js ordered-list rendering which renumbers list items sequentially, ignoring the user's original numbering. Author is a team member who offered to contribute a fix — keeping open.

## Change summary

User-typed numbered lists in chat messages (e.g. `2.` then `4.`) are rendered with renumbered sequential indices (2, 3), making it look like the message was altered.

## Best solution

Add a custom `renderer.list` / `renderer.listitem` override in `src/vs/workbench/contrib/chat/browser/widget/chatContentMarkdownRenderer.ts` that preserves the original list-item numbers from the parsed tokens (marked.js tokens include the original number). The author offered to contribute this fix.
