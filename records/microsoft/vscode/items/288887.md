# Issue #288887 — Keeps reverting to tabs for input when Code gets focus

[https://github.com/microsoft/vscode/issues/288887](https://github.com/microsoft/vscode/issues/288887)

## Summary

The revert is as-designed: language-specific settings (`[shellscript].editor.insertSpaces: false`) override transient model options set via the status bar. The reporter confirmed this in their own follow-up comment.

## Change summary

Status bar indentation changes revert when the window regains focus because language-specific settings re-apply.

## Best solution

Close as designed. Language-specific settings are authoritative over transient per-model overrides made via the status bar. A possible UX polish (warning the user that a language setting will override their choice) could be filed separately as a feature request.

This is working as designed: the status bar indentation change sets a transient per-model option, but your language-specific setting (`[shellscript].editor.insertSpaces: false`) is authoritative and re-applies when the editor re-evaluates configuration (e.g. on focus).
