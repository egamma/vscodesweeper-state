# Issue #288887 — Keeps reverting to tabs for input when Code gets focus

[https://github.com/microsoft/vscode/issues/288887](https://github.com/microsoft/vscode/issues/288887)

## Summary

This should stay open and be routed to the editor area: the reporter identified a language-specific `editor.insertSpaces: false` override, but the current UI still allows a per-file indentation change that is later overwritten, which is at least a confusing editor/settings interaction.

## Change summary

Changing a shell script from tabs to spaces through the status bar only persists until VS Code regains focus when a `[shellscript]` setting forces `editor.insertSpaces: false`.

## Best solution

Editor indentation actions/status should either make clear that the change is only a current-model override and can be superseded by language-specific settings, or avoid offering an apparently persistent override when `[language]` settings will reapply. The likely ownership is the editor indentation/status path around `src/vs/editor/contrib/indentation/browser/indentation.ts` and `src/vs/workbench/browser/parts/editor/editorStatus.ts`.

This looks like an editor/settings interaction rather than an extension-only report: the status bar action changes the current model indentation, while the `[shellscript]` override can later reapply `editor.insertSpaces: false`. ## Evidence

- **reporter follow-up:** The reporter found `[shellscript]` settings with `editor.detectIndentation: false` and `editor.insertSpaces: false`, then noted VS Code allowed changing to spaces but later reverted to tabs.
- **indent action behavior:** `src/vs/editor/contrib/indentation/browser/indentation.ts` shows `IndentUsingSpaces` ultimately calls `model.updateOptions({ tabSize, indentSize, insertSpaces: true })`; it changes the current text model rather than updating user or language-specific settings.
- **configuration behavior:** `src/vs/editor/common/services/modelService.ts` reads language/resource-specific `editor.insertSpaces` in `getCreationOptions`, and `_setModelOptionsForModel` reapplies configured `insertSpaces`, `tabSize`, and `indentSize` to the model when indentation detection is disabled.
- **status display:** `src/vs/workbench/browser/parts/editor/editorStatus.ts` displays the current model indentation state as `Spaces: {0}` or `Tab Size: {0}`, which explains why the user sees the status bar switch back when the model options are reapplied.
