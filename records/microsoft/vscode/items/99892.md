# Issue #99892 — Ellipsis in search viewlet shows more UI, not commands

[https://github.com/microsoft/vscode/issues/99892](https://github.com/microsoft/vscode/issues/99892)

## Summary

Still present on main: the search viewlet uses `Codicon.ellipsis` to toggle the include/exclude input boxes, which is an inconsistent use of the ellipsis pattern. Team-authored issue (MEMBER) — keeping open.

## Change summary

The search viewlet's ellipsis button toggles persistent UI (include/exclude boxes) instead of showing a menu or temporary commands, inconsistent with ellipsis conventions elsewhere.

## Best solution

Replace the ellipsis icon in `src/vs/workbench/contrib/search/browser/searchIcons.ts` (the `searchDetailsIcon` registration) with a chevron or other expand/collapse icon that better conveys toggling UI sections, consistent with how similar toggles work in the panel and elsewhere in the workbench. The interaction itself (toggling query details) is fine — only the icon is wrong.
