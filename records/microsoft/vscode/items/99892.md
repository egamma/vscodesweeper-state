# Issue #99892 — Ellipsis in search viewlet shows more UI, not commands

[https://github.com/microsoft/vscode/issues/99892](https://github.com/microsoft/vscode/issues/99892)

## Summary

Team-authored UX debt issue; the ellipsis icon is still used on current main (Codicon.ellipsis in searchIcons.ts). The inconsistency remains unaddressed — keeping open per policy for team-authored issues and because the debt is real.

## Change summary

The search viewlet uses an ellipsis icon to toggle the include/exclude input boxes, which is inconsistent with the intended use of ellipsis (temporary overflow menus, not permanent UI toggles).

## Best solution

Replace the ellipsis icon in `src/vs/workbench/contrib/search/browser/searchIcons.ts` (line 10, `Codicon.ellipsis`) with a chevron or other expand/collapse icon that better communicates toggling persistent UI. The same icon is also used in the quick text search access (`textSearchQuickAccess.ts`). A broader pass on ellipsis usage across the product (as stevencl's comment 4 suggests) would address the wider inconsistency.
