# Issue #50638 — Cannot exclude root folders while searching

[https://github.com/microsoft/vscode/issues/50638](https://github.com/microsoft/vscode/issues/50638)

## Summary

Team-authored feature request (MEMBER) with 20 upvotes, On Deck milestone, and 'help wanted' label — hard keep-open. No evidence of a fix on main; the gap remains: glob matching in search exclude starts under the search folder, so there is no syntax to exclude a root folder in a multi-root workspace.

## Change summary

Support excluding root folders (e.g. via `./name` syntax) in the search exclude pattern for multi-root workspaces.

## Best solution

Extend the search glob-matching to support `./name` in the 'files to exclude' field (it already works in 'files to include'), so users can exclude specific root folders in multi-root workspaces. The fix likely lives in the glob/pattern resolution layer in `src/vs/workbench/services/search/`. The issue is already labeled `help wanted` and is a good community contribution candidate.
