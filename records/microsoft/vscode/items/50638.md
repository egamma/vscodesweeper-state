# Issue #50638 — Cannot exclude root folders while searching

[https://github.com/microsoft/vscode/issues/50638](https://github.com/microsoft/vscode/issues/50638)

## Summary

Team-authored feature request on the 'On Deck' milestone with 20 upvotes and 'help wanted' — both the member authorship and the scheduling milestone trigger hard keep-open rules. The gap remains: glob matching in search exclude starts under each search folder, so there is no syntax to exclude a root folder itself in a multi-root workspace.

## Change summary

In a multi-root workspace, there is no way to exclude an entire root folder from search results because glob patterns start matching under each root, not at the root level.

## Best solution

Support `./name` syntax (or a similar mechanism) in the search 'files to exclude' field so that root-level folders can be targeted, mirroring how `./name` already works in 'files to include'. The fix likely lives in the search glob-matching layer under `src/vs/workbench/contrib/search/` and possibly in `src/vs/workbench/services/search/`. The 'help wanted' label suggests this is ready for a community contribution.
