# Issue #170345 — iPad, Web: Double context menu in editor area

[https://github.com/microsoft/vscode/issues/170345](https://github.com/microsoft/vscode/issues/170345)

## Summary

Confirmed upstream WebKit bug — iPadOS Safari ignores `event.preventDefault()` on the `contextmenu` event when a trackpad tap-to-click is used with a hardware keyboard attached. The fix lives in WebKit, not VS Code.

## Change summary

Right-clicking in the editor on iPad with a pointer device shows both the OS context menu and VS Code's context menu simultaneously.

## Best solution

Wait for the WebKit fix (https://bugs.webkit.org/show_bug.cgi?id=260722) to land in a shipping iPadOS Safari build. No viable workaround exists in VS Code since `preventDefault()` is correctly called but ignored by the platform. Keep tracking until Apple resolves it.

## Evidence

- **Apple feedback:** Reporter filed FB11920676 with Apple's Feedback Assistant.
- **root cause:** Comment 7 explains: two-finger-tap on multi-touch trackpads triggers both the browser contextmenu event and the OS context menu; a full haptic click does not. This is a platform-level dispatch bug.
- **still open:** As of iPadOS 17.2 (Jan 2024) and the last comment (Feb 2025), the WebKit bug remains unresolved.
