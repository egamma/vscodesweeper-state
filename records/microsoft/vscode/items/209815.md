# Issue #209815 — Can't sign in to settings sync with GitHub if using non-default profile and `--user-data-dir`

[https://github.com/microsoft/vscode/issues/209815](https://github.com/microsoft/vscode/issues/209815)

## Summary

Still reproduces on main: the `vscode://` protocol handler routes auth callbacks to the default instance, not the one launched with a custom `--user-data-dir`. A workaround exists via the `preferDeviceCodeFlow` setting (added Oct 2025), but the underlying routing bug remains unfixed.

## Change summary

GitHub sign-in for settings sync fails when VS Code is launched with a non-default `--user-data-dir` because the protocol handler redirects the auth callback to the wrong instance.

## Best solution

Either make the protocol handler routing aware of custom `--user-data-dir` instances (in `electronUrlListener.ts` / window matching logic), or detect non-default user-data-dir in the auth flow and skip the protocol-handler redirect (similar to the portable-mode fix in `ec08a89b569`). The `preferDeviceCodeFlow` setting is a viable user workaround but the default flow should work correctly.
