# Issue #322103 — VS Code allows incompatible local extension installation into Linux Codespace, leading to platform mismatch and "bad image format" errors

[https://github.com/microsoft/vscode/issues/322103](https://github.com/microsoft/vscode/issues/322103)

## Summary

Valid feature request: the VSIX install path does not validate the extension's targetPlatform against the remote environment's platform. The gallery-based install validates platform compatibility, but the local VSIX install only checks engine version — no platform mismatch guard exists for the manual install workflow.

## Change summary

When installing a VSIX locally into a remote (Codespace/SSH/container), VS Code should validate the extension's target platform against the remote and warn or block on mismatch.

## Best solution

Add a targetPlatform validation check in the VSIX install path (around `src/vs/platform/extensionManagement/node/extensionManagementService.ts:145–165`) that reads the extension manifest's `targetPlatform` field and compares it against `getTargetPlatform()` of the remote server, similar to the gallery install path's `IncompatibleTargetPlatform` error. The remote extension management service (`src/vs/workbench/services/extensionManagement/electron-browser/remoteExtensionManagementService.ts`) should surface this as a user-facing warning or block.
