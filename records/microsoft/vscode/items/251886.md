# Issue #251886 — github.copilot-chat/panel.action.vote should include model

[https://github.com/microsoft/vscode/issues/251886](https://github.com/microsoft/vscode/issues/251886)

## Summary

Valid and unimplemented: the `panel.action.vote` telemetry event still lacks a `model` property on current main. The requested enhancement would avoid expensive joins when measuring upvotes/downvotes per model.

## Change summary

Add the resolved model identifier to the `panel.action.vote` telemetry event so vote quality can be sliced per model without a join.

## Best solution

Add a `modelId` string property (using `resolveModelIdForTelemetry` from the same file's existing pattern) to the `panel.action.vote` call in `extensions/copilot/src/extension/conversation/vscode-node/userActions.ts`, along with the corresponding GDPR annotation. The pattern is already established by commit 82b1b81b994 for neighboring events.
