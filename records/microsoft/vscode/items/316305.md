# Issue #316305 — Copilot suggestions override standard VS Code and extension autocompletion

[https://github.com/microsoft/vscode/issues/316305](https://github.com/microsoft/vscode/issues/316305)

## Summary

Current main intentionally suppresses automatically triggered quick suggestions while inline completions are showing by default, and the requested control already exists through editor settings.

## Change summary

Allow Copilot inline completions to avoid blocking or deprioritizing normal completion-list suggestions from language extensions.

## Best solution

Keep the current inline-completion and quick-suggestion behavior, and direct users who prefer the old completion-list behavior to configure `editor.quickSuggestions` to `on` or disable automatic inline suggestions via `editor.inlineSuggest.enabled`.

- **Proposed comment:**

  > Thanks for the report. This is the intended default behavior on current main: quick suggestions use `offWhenInlineCompletions` by default, so automatically triggered completion-list suggestions are suppressed while inline completions are showing. Users who prefer the completion list to continue appearing can configure `editor.quickSuggestions` to `on`; users who want no automatic inline completions can set `editor.inlineSuggest.enabled` to `false` and still invoke normal suggestions with Ctrl+Space.
