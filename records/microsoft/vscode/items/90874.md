# Issue #90874 — Parameter hints should not use aria.alert

[https://github.com/microsoft/vscode/issues/90874](https://github.com/microsoft/vscode/issues/90874)

## Summary

Still valid on current main: parameterHintsWidget.ts line 261 still calls aria.alert(). The fix PR #93606 was closed unmerged because it triggers the still-open NVDA regression (#89098). The debt remains.

## Change summary

Parameter hints use aria.alert instead of activeDescendant, causing poor screen reader behavior (repeated messages, no pause between params, no scrollability).

## Best solution

Resolve the blocking NVDA issue (#89098 — editor title re-read on widget close) first, then adopt the activeDescendant approach from the closed PR #93606 in src/vs/editor/contrib/parameterHints/browser/parameterHintsWidget.ts.
