# Issue #166161 — Issue with "reopen editor with" -> Jupyter notebook

[https://github.com/microsoft/vscode/issues/166161](https://github.com/microsoft/vscode/issues/166161)

## Summary

Still open and valid: the untitled notebook cell URI lacks the notebook-type query string, so when the debugger tries to open a paused location by that cell URI, the editor service cannot resolve it back to the notebook editor. Author is MEMBER and milestone is On Deck — hard keep-open.

## Change summary

When using 'Reopen Editor With' -> Jupyter Notebook on an untitled file and then debugging (Run By Line), the debugger opens a plain text editor instead of revealing the notebook cell.

## Best solution

Add an untitled-notebook fallback in the notebook editor resolution path (as discussed in comment 11): when a `vscode-notebook-cell:` URI arrives without a notebook-type query string, look up open notebook models to find which provider owns the cell, so the editor service can reuse the existing notebook editor. The fix likely lives in `src/vs/workbench/contrib/notebook/browser/services/notebookServiceImpl.ts` around the editor input factory registration.
