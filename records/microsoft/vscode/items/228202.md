# Issue #228202 — Azure DevOps - Enable seamless Azure DevOps repository cloning in VSCode similar to GitHub

[https://github.com/microsoft/vscode/issues/228202](https://github.com/microsoft/vscode/issues/228202)

## Summary

Legitimate feature request with community backing (21 upvotes, promoted to Backlog). The GitHub extension provides a credential provider bridge (extensions/github/src/credentialProvider.ts) that handles github.com hosts via OAuth; no equivalent exists for Azure DevOps (dev.azure.com / *.visualstudio.com). No duplicate found.

## Change summary

Add seamless OAuth-based git credential management for Azure DevOps repositories, analogous to the existing GitHub credential provider integration.

## Best solution

Implement an Azure DevOps credential provider that mirrors the GitHub pattern in extensions/github/src/credentialProvider.ts — register a CredentialsProvider for dev.azure.com and *.visualstudio.com hosts, backed by the existing microsoft-authentication extension's Microsoft auth provider. This could live in a new built-in extension or be added to an existing one.
