# Issue #322374 — Allow a custom/arbitrary sort order for files in the Source Control 'Changes' list

[https://github.com/microsoft/vscode/issues/322374](https://github.com/microsoft/vscode/issues/322374)

## Summary

Well-researched feature request for custom/arbitrary sort ordering in the SCM Changes list. The prior related issue #235493 was closed as not-planned, but this request is more specific — asking for an extension API or explicit custom-order mode rather than just matching Explorer sort. Routing to the SCM area for a product decision.

## Change summary

Allow custom or arbitrary sort ordering of files within the Source Control Changes list, beyond the existing name/path/status modes.

## Best solution

Evaluate whether to add an extension API surface (e.g. a sort priority on `SourceControlResourceState` or a custom comparator on `SourceControlResourceGroup`) that lets extensions control resource ordering in the SCM view. The simplest path would be a new `ViewSortKey` mode that preserves the provider's `resourceStates` array order, bypassing `SCMTreeSorter`'s forced sort. The extension API route is the most powerful and aligns with 'the power lives in extensions.' Changes would touch `src/vs/workbench/contrib/scm/browser/scmViewPane.ts` (SCMTreeSorter) and the SCM extension API types.
