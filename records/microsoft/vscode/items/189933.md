# Issue #189933 — Focusing another window while dragging a file from the FileExplorer causes a TreeError

[https://github.com/microsoft/vscode/issues/189933](https://github.com/microsoft/vscode/issues/189933)

## Summary

Still plausibly reproduces on main: when the window loses focus mid-drag, the drag sequence can complete against stale tree state, hitting the unguarded `getDataNode` throw in `asyncDataTree.ts`. No fix has been committed for this edge case.

## Change summary

Focusing another window while dragging a file in the File Explorer causes a TreeError because the tree node lookup fails after the drag state goes stale.

## Best solution

Guard the drop/dragEnd path in `src/vs/base/browser/ui/list/listView.ts` (or the async data tree's DnD delegate) so that a blur during drag either cancels the operation cleanly or skips the `getDataNode` call when the node is no longer present. A defensive `hasNode` check before `getDataNode` in the explorer's drop handler would prevent the throw.
