# Issue #324786 — Subagent spawn with a pinned non-default model intermittently fails with `key is missing`, and the broken session poisons subsequent chats

[https://github.com/microsoft/vscode/issues/324786](https://github.com/microsoft/vscode/issues/324786)

## Summary

Legitimate race condition in copilot token acquisition during subagent model switches. A team member (aeschli) has already identified the failing code path. Keeping open for the team to address.

## Change summary

When a subagent pins a different model than the parent session, the first request intermittently fails with 'key is missing' because the copilot token is not yet available during request assembly, and the failure state leaks into subsequent sessions.

## Best solution

Fix the token acquisition race in `extensions/copilot/src/extension/prompt/node/chatMLFetcher.ts` — either ensure `getCopilotToken()` is retried/awaited properly when a model switch triggers a new endpoint resolution, or add a transparent retry at the request-assembly layer before surfacing the error. The session-poisoning symptom likely needs a separate fix to isolate failed request state.
