# Issue #314777 — Accessibility: Notification Dialogue with toolbar violates  WCAG 4.1.2 Name , Role, Value  "SC 4.1.2 Name, Role, Value (Level A)"

[https://github.com/microsoft/vscode/issues/314777](https://github.com/microsoft/vscode/issues/314777)

## Summary

Two of three reported WCAG 4.1.2 violations remain on current main: the notification list container has role='list' (not a widget role) while being tabbable, and list items with role='dialog' receive invalid ARIA attributes (aria-setsize, aria-posinset, aria-selected). Issue #1 was fixed in 1.126.

## Change summary

Notification dialog list violates WCAG 4.1.2 — tabbable element has non-widget role 'list', and dialog-role items carry invalid ARIA attributes.

## Best solution

In the NotificationAccessibilityProvider in src/vs/workbench/browser/parts/notifications/notificationsList.ts, add a getWidgetRole() override returning an appropriate widget role (e.g. 'listbox' or removing the tabbable behavior), and suppress aria-setsize/aria-posinset/aria-selected on items whose getRole() returns 'dialog'. The list widget in src/vs/base/browser/ui/list/listWidget.ts and listView.ts may need to conditionally omit those ARIA attributes based on the item role.
