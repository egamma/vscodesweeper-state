## Summary

VS Code's extension API already provides exactly this capability — extensions can declare their own configuration sections (including AI-related ones) via `contributes.configuration` in their `package.json`, and the settings infrastructure is fully extensible. There is no missing core mechanism here; this belongs in an extension.

## Change summary

## Best solution

This is already achievable via the extension API: any extension can contribute a `configuration/ai` (or any named) settings section and use it to integrate with external AI services. No core change is needed — build this as an extension.
