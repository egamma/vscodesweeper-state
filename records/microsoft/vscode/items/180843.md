# Issue #180843 — VS Code crashes when opening a window on Ubuntu

[https://github.com/microsoft/vscode/issues/180843](https://github.com/microsoft/vscode/issues/180843)

## Summary

Team-authored upstream issue — V8 incorrectly uses AVX instructions when AVX CPU flags are absent (Parallels VM on Intel Mac). Already correctly labeled `upstream` with `upstream-issue-linked` to the Chromium bug. Must stay open per keep-open rules (MEMBER author).

## Change summary

VS Code crashes on launch in Linux VMs where the hypervisor does not expose AVX CPU flags, because V8's SIMD search path assumes AVX availability.

## Best solution

The fix lives in Chromium/V8 (https://bugs.chromium.org/p/chromium/issues/detail?id=1412373). Monitor that upstream bug and verify the fix lands via a future Electron update. Related issue https://github.com/microsoft/vscode/issues/214630 tracks a similar AVX512 problem in Node. Once the Electron version shipping with VS Code includes the V8 fix, this can be closed.
