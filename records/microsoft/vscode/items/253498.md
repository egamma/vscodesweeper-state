# Issue #253498 — Using the same link again should do file validation before I accept it

[https://github.com/microsoft/vscode/issues/253498](https://github.com/microsoft/vscode/issues/253498)

## Summary

Team-authored feature request (MEMBER) — keeping open per policy. Requests that chat prompt link suggestions validate whether the linked file already exists before showing the acceptance UI.

## Change summary

When a chat prompt suggests reusing an existing file link, it should validate the file's existence before prompting the user to accept.

## Best solution

In the chat-prompts link suggestion flow, add a file-existence check (via the file service) before presenting the accept UI, so users aren't asked to confirm a link to a file that already exists at that path.
