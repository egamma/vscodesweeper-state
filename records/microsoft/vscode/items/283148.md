# Issue #283148 — Stage Selected Range the second time wipes out the first Staged Selected Range which was never shown as staged.

[https://github.com/microsoft/vscode/issues/283148](https://github.com/microsoft/vscode/issues/283148)

## Summary

Verified duplicate of #305073, which tracks the same race condition in staging multiple selected ranges — the second stage operation overwrites the first because the diff base and quick-diff state aren't refreshed between operations. #305073 is assigned to @lszomoru and is the canonical issue.

## Change summary

Staging a second selected range in the SCM diff editor wipes out the previously staged first range because the staging logic reads stale diff state.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/305073 which is assigned and tracks the same root cause (staging multiple selected ranges fails). The fix likely needs `_stageChanges` to either serialize operations with a refreshed diff base, or batch multiple range selections into a single apply-and-stage operation.
