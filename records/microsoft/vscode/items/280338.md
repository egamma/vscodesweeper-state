# Issue #280338 — Chat response height is not adjusted for thinking labels

[https://github.com/microsoft/vscode/issues/280338](https://github.com/microsoft/vscode/issues/280338)

## Summary

Team-authored bug (roblourens) — must keep open for maintainer judgment. Several related height/cutoff fixes have landed on main since this was filed (e.g. 725d69a9a85 'fix cutoff chat requests', 719c62b60e8 'improve thinking layout thrashing'), but none explicitly addresses the thinking-label-wrap re-measurement gap described here. The bug may or may not be resolved as a side effect.

## Change summary

When thinking labels load and wrap to a second line, the chat response row height is not re-measured, causing the bottom of the response to be cut off.

## Best solution

Verify whether the cutoff fix in commit 725d69a9a85 (chatListRenderer.ts — schedules a height-change event on first render via rAF) and the thinking layout thrashing fix in 719c62b60e8 resolve this specific scenario. If not, add a re-measurement trigger in the thinking-label rendering path in chatThinkingContentPart.ts or chatListRenderer.ts when the label element wraps and changes height.
