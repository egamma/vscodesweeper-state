# Issue #262711 — MCP view leaks

[https://github.com/microsoft/vscode/issues/262711](https://github.com/microsoft/vscode/issues/262711)

## Summary

The disposable leak in DefaultBrowseMcpServersView is still present on current main — the race between consecutive `show()` calls during rapid AI-state toggling leaks the DisposableStore created in `query()`. An open PR (#266174) targets this. Keeping open given the 'On Deck' milestone and the open PR.

## Change summary

DefaultBrowseMcpServersView leaks DisposableStore instances when the MCP view is rapidly shown/hidden during AI feature toggling.

## Best solution

Land the fix in the open PR (https://github.com/microsoft/vscode/pull/266174) or an equivalent that uses a MutableDisposable / cancellation token to ensure only the latest `query()` result's disposables survive, and prior ones are disposed before reassignment.
