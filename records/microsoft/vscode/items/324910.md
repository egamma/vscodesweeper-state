# Issue #324910 — Terminal title overlaps with action buttons when panel is moved to the side and tabs are collapsed

[https://github.com/microsoft/vscode/issues/324910](https://github.com/microsoft/vscode/issues/324910)

## Summary

Still reproduces on main: the `.single-terminal-tab` flex container lacks `overflow: hidden` / `text-overflow: ellipsis` / `min-width: 0`, so long terminal titles overflow into the action buttons. An open community PR (#325078) addresses this but has not merged yet.

## Change summary

Terminal title text overflows into action buttons when the panel is docked to the side and tabs are collapsed to single-tab mode with a long title.

## Best solution

Review and land the open PR https://github.com/microsoft/vscode/pull/325078, which wraps the title in a constrained span and adds the missing CSS truncation properties to `.single-terminal-tab`. The fix is small and localized to `terminalView.ts` and `terminal.css`.

## Evidence

- **Title rendering path:** In src/vs/workbench/contrib/terminal/browser/terminalView.ts, `dom.reset(label, ...renderLabelWithIcons(getSingleTabLabel(...)))` inserts text nodes directly into the flex container with no wrapper element to constrain overflow.
