## Change summary

Typing `./` or `../` in Quick Open (Ctrl+P) should list files relative to the currently active editor's directory, not just the workspace root.

## Best solution

Extend `getRelativePathFileResults` in `src/vs/workbench/contrib/search/browser/anythingQuickAccess.ts` to detect `./` or `../` prefixes and resolve them against the active editor's URI directory (via `IEditorService`) instead of only workspace folder roots. This keeps the existing workspace-root relative path resolution as a fallback.
