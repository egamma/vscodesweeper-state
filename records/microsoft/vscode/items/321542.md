# Issue #321542 — Feature Request: Native Azure DevOps Pull Request and Inline Comments Support

[https://github.com/microsoft/vscode/issues/321542](https://github.com/microsoft/vscode/issues/321542)

## Summary

Azure DevOps PR support belongs in an extension, not VS Code core. The GitHub Pull Requests experience the author references is itself a Marketplace extension (not built-in), and the same model applies here — Microsoft already ships the Azure Repos extension for basic integration.

## Change summary

Request for built-in Azure DevOps pull request management and inline review comments in VS Code core.

## Best solution

Ship this as a first-party Microsoft extension on the Marketplace (like GitHub Pull Requests and Issues). The VS Code core already provides the extension API surface (SCM, comments, authentication providers) needed to build this without core changes. The existing Azure Repos extension or a new dedicated extension is the right vehicle.

## Vision fit

- **Fit:** rejected
- **Reason:** VS Code's values explicitly state 'the power lives in extensions' and the product should remain a lightweight text editor out of the box; a full PR management workflow for a specific hosting provider is exactly the kind of specialized capability that belongs in an extension.
- **Evidence:** Value: 'Out of the box, VS Code is a text editor: lightweight and fast. Users may install extensions that impact the appearance and nature of VS Code.' Value: 'The power lives in extensions.'
