# Issue #286359 — SCM Checkout remote doesn't work properly when local branch exists

[https://github.com/microsoft/vscode/issues/286359](https://github.com/microsoft/vscode/issues/286359)

## Summary

The behavior is intentional (checkout local tracking branch instead of detached remote), but the UX is misleading — the user expects to land on the remote ref they clicked. The existing `git.pullBeforeCheckout` setting mitigates but doesn't fully address the issue. Keeping open as under-discussion for a UX improvement.

## Change summary

Checking out a remote branch in the SCM graph silently checks out the (potentially stale) local tracking branch instead of updating it or warning the user.

## Best solution

Improve the checkout flow in `extensions/git/src/commands.ts` (around line 2831) to prompt the user when the local tracking branch is behind the remote — offering to fast-forward, checkout detached, or cancel. The `git.pullBeforeCheckout` setting partially addresses this but defaults to off and doesn't cover all cases (e.g. diverged branches).
