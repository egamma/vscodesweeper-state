# Issue #300659 — [Bug] Garbled text appears in the right-click context menu after update

[https://github.com/microsoft/vscode/issues/300659](https://github.com/microsoft/vscode/issues/300659)

## Summary

The garbled text in the Windows right-click context menu is likely a localization issue in the Win11 context menu integration (AppX sparse package / explorer command DLL) introduced when the feature was enabled for stable in ~v1.109. The reporter is on a Chinese locale system, and the DLL may not be rendering its localized strings correctly. This should be routed to the system-context-menu area for investigation.

## Change summary

After updating VS Code (around v1.109+), a garbled/mojibake entry appears in the Windows 11 right-click context menu on a Chinese locale system, even when the user did not opt into the context menu option during installation.

## Best solution

Investigate whether the `vscode-explorer-command` DLL (v8.0.0-398351) properly handles CJK locale string rendering for the Win11 context menu entry. The AppxManifest.xml declares `zh-cn` as a supported resource language, so the issue may be in the DLL's string table or in how the registry `Title` value at `Software\Classes\{RegValueName}ContextMenu` is encoded. Also investigate why the entry appears even when the user unchecked the context menu option — possibly a leftover from a previous installation or the appx package auto-registering.
