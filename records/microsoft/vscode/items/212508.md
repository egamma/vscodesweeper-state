# Issue #212508 — After updating to version 1.89 my $HOME path is incorrect. And not able to change it.

[https://github.com/microsoft/vscode/issues/212508](https://github.com/microsoft/vscode/issues/212508)

## Summary

Still reproduces on current main: the shell integration bash script uses `echo -e` to decode environment variable values but the TypeScript encoder (`_encodeColons` in environmentVariableCollection.ts) does not encode backslashes, so Windows paths containing `\t`, `\n`, `\U` etc. get interpreted as escape sequences, corrupting $HOME and other path variables in Git Bash.

## Change summary

Git Bash terminal $HOME path is corrupted when shell integration is enabled because backslashes in environment variable values are not escaped before being passed through `echo -e`.

## Best solution

Land the open PR https://github.com/microsoft/vscode/pull/240614 (or an equivalent fix) that encodes backslashes as `\x5c` in `_encodeColons` (rename to a general escaping method) in `src/vs/platform/terminal/common/environmentVariableCollection.ts`, so the bash script's `echo -e` decoding round-trips Windows paths correctly.

## Evidence

- **root cause in source:** src/vs/platform/terminal/common/environmentVariableCollection.ts line 109-111: `_encodeColons` only escapes `:` as `\x3a` but does NOT escape backslashes. The value is later decoded in shellIntegration-bash.sh (line 76/85/94) via `echo -e`, which interprets `\t`, `\n`, `\U` etc. as escape sequences.
- **open fix PR:** https://github.com/microsoft/vscode/pull/240614 is open and targets this exact encoding gap.
- **closed alternative PR:** https://github.com/microsoft/vscode/pull/255176 was closed without merge on 2025-07-10.
- **user confirmations:** Multiple users (comments 3-13) confirm the issue reproduces with shell integration enabled on Git Bash with Windows paths; disabling shell integration is the workaround.
