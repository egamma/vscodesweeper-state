# Issue #305602 — Bring back the 'Used references' indication for instructions

[https://github.com/microsoft/vscode/issues/305602](https://github.com/microsoft/vscode/issues/305602)

## Summary

Team-authored request to restore the 'Used references' indication for instruction files in agent mode. The indication was intentionally hidden in agent mode by commit 724656efa2c (Feb 2026); this issue asks to bring it back specifically for instructions so users can verify which instruction files were incorporated. Keeping open — team member issue requiring maintainer judgment.

## Change summary

After commit 724656efa2c hid the 'Used references' button in agent mode, users lost visibility into which instruction files were attached to a chat conversation, making it impossible to verify context.

## Best solution

Restore the 'Used references' indication selectively for instruction-type references in agent mode. The hiding logic lives in `src/vs/workbench/contrib/chat/browser/widget/chatListRenderer.ts` around line 2502, where `renderNoContent` is returned for agent-mode responses. A targeted fix would filter the suppression so instruction references still surface while other reference types remain hidden.
