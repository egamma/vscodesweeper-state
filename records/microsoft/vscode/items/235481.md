# Issue #235481 — Show Primary Side Bar in new empty window when Activity Bar position not Default

[https://github.com/microsoft/vscode/issues/235481](https://github.com/microsoft/vscode/issues/235481)

## Summary

Valid feature request already acknowledged by a team member (benibenj) who noted they'll consider adding a setting for the primary side bar if more users ask for it. Keeping open in Backlog for community traction.

## Change summary

When opening a new empty window with a non-default Activity Bar position, the Primary Side Bar (and thus Explorer viewsWelcome contributions) is not shown.

## Best solution

Add a setting (analogous to the secondary side bar setting added for https://github.com/microsoft/vscode/issues/191202) that controls whether the primary side bar is shown in new empty windows regardless of Activity Bar position. The fix would likely live in the layout restoration logic in `src/vs/workbench/browser/layout.ts` or related layout service code.
