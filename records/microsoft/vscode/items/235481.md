## Summary

Valid feature request for showing the Primary Side Bar (Explorer) in new empty windows when the Activity Bar isn't in its default position. The original fix was reverted due to regressions and no replacement has landed. A maintainer acknowledged it and is waiting for more community demand.

## Change summary

Show the Explorer view in the Primary Side Bar when opening a new empty window, even when the Activity Bar position is non-default.

## Best solution

Implement a user setting (analogous to the secondary sidebar setting added for that use case) that controls whether the primary sidebar is shown in new empty windows regardless of activity bar position. The fix likely lives in `src/vs/workbench/browser/layout.ts` and needs to avoid the regression that caused the revert of #227677 (sidebar appearing when restoring a workspace that had it hidden).
