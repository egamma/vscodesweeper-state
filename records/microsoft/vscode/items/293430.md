# Issue #293430 — GitHub Copilot `file_search` glob with `\**\` (backslash double-star) produces partial or no results where `/**/` succeeds

[https://github.com/microsoft/vscode/issues/293430](https://github.com/microsoft/vscode/issues/293430)

## Summary

The VS Code glob engine splits only on `/` (`GLOB_SPLIT = '/'`), so `\**\` is never recognized as a globstar — confirmed in `src/vs/base/common/glob.ts`. The maintainer (roblourens) shipped a fix for the principal issue (#293428) in `vscode-copilot-chat` PR #4287 and said to re-check whether it also resolves this secondary issue. The issue needs routing and follow-up verification.

## Change summary

Copilot's `file_search` tool fails when the LLM produces backslash-separated `**` glob patterns on Windows (`alpha\**\file.ext`), because the glob engine only recognizes `/**/`.

## Best solution

Verify whether the fix that shipped for https://github.com/microsoft/vscode/issues/293428 (vscode-copilot-chat PR #4287) also normalizes `\**\` to `/**/` before glob evaluation. If not, add backslash-to-forward-slash normalization in the Copilot extension's search tool layer (`extensions/copilot/src/platform/search/`) before the pattern reaches `vscode.workspace.findFiles2`.
