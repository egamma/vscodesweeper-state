# Issue #187440 — F5 starts debugging when inactive, but refreshes the page when active

[https://github.com/microsoft/vscode/issues/187440](https://github.com/microsoft/vscode/issues/187440)

## Summary

Still reproduces on main: F5 is only bound for debugState 'inactive' (start) and 'stopped' (continue) — when debugState is 'running' or 'initializing', the keybinding falls through to the browser, which refreshes the page on Linux/Windows web, destructively killing the debug session. A maintainer (roblourens) already acknowledged and sketched a fix.

## Change summary

On web, pressing F5 while a debug session is running (but not stopped at a breakpoint) triggers a browser page refresh instead of being intercepted by VS Code, destroying the debug session.

## Best solution

Add a no-op keybinding for F5 in `src/vs/workbench/contrib/debug/browser/debugCommands.ts` that claims the key when `debugState` is neither 'inactive' nor 'stopped' (i.e. 'running' or 'initializing'), preventing browser fallthrough. This is exactly what roblourens suggested in comment #6 — a dummy command to fill the gap and block the browser default.
