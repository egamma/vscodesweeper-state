# Issue #226469 — Behavioral change in quick pick with selected items

[https://github.com/microsoft/vscode/issues/226469](https://github.com/microsoft/vscode/issues/226469)

## Summary

Still reproduces on main: reassigning `quickPick.items` in a multi-select quick pick fires `onDidChangeSelection` with an empty array because `setElements` creates fresh unchecked elements, then the tree-model-change listener re-emits checked state. Team-authored issue; keeping open.

## Change summary

Setting `quickPick.items` on a multi-select quick pick spuriously fires `onDidChangeSelection` with no items, clearing all checked selections.

## Best solution

Guard `_registerOnTreeModelChanged` (quickInputList.ts:947-953) so it does not emit a checked-elements change while `setElements` is in progress (or preserve checked state across item replacement by matching on identity). The fix is localized to `src/vs/platform/quickinput/browser/quickInputList.ts`.
