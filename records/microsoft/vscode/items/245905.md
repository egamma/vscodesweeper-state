# Issue #245905 — MCP Server Configuration: predefined variables (workspaceFolder, workspaceFolderBasename, selectedText) fail to resolve

[https://github.com/microsoft/vscode/issues/245905](https://github.com/microsoft/vscode/issues/245905)

## Summary

Still reproduces on main: user-level MCP server configs pass `undefined` as the folder to `IConfigurationResolverService.resolveAsync`, causing `${workspaceFolder}` and `${workspaceFolderBasename}` to throw even when a workspace is open. The inconsistency with `${relativeFile}` (which resolves from the active editor path) is confirmed in the source.

## Change summary

Predefined variables like `${workspaceFolder}` and `${workspaceFolderBasename}` fail to resolve in user-level MCP server configurations, while `${relativeFile}` succeeds — an inconsistency in how the folder context is provided to the variable resolver.

## Best solution

When `variableReplacement.folder` is undefined (user-level config) but a workspace is open, fall back to the first workspace folder (or prompt for multi-root) before calling `resolveAsync`. The fix belongs in `src/vs/workbench/contrib/mcp/common/mcpRegistry.ts` around the `_replaceVariablesInLaunch` method, or in how `installedMcpServersDiscovery.ts` populates `variableReplacement.folder` for user-scoped configs.
