# Issue #326006 — BranchPickerWidget uses sessions-specific CSS class names in workbench core

[https://github.com/microsoft/vscode/issues/326006](https://github.com/microsoft/vscode/issues/326006)

## Summary

Team-authored debt item: sessions-specific CSS class names leaked into workbench core during the PR #325712 migration. The issue is well-scoped with a concrete fix plan; keeping open for the author or area owner to address.

## Change summary

BranchPickerWidget and related workbench-layer files reference `.sessions-chat-picker-slot` and `.sessions-chat-dropdown-label` CSS classes whose styles are defined only in the sessions layer.

## Best solution

Rename the CSS classes to drop the `sessions-` prefix across the ~8 picker TS files and ~4 CSS files, extract base picker-slot styles into a new CSS file in the workbench layer next to the picker widgets, and keep sessions-specific overrides in sessions CSS. This is a clean follow-up to the migration in PR #325712.
