# Issue #300838 — VS Code doesn't remember anymore the repos which were manually closed ...

[https://github.com/microsoft/vscode/issues/300838](https://github.com/microsoft/vscode/issues/300838)

## Summary

The SCM view service's state restoration logic in `scmViewService.ts` can fail to keep manually-closed repos closed across restarts. When a repository not in the saved state triggers the 'select all' branch (line 364–386), it resets `didSelectRepository = false`, causing subsequently loaded repos that should remain invisible to be made visible instead. No fix has landed on main.

## Change summary

Manually closed (hidden) SCM repositories reopen on VS Code restart when a folder contains multiple git repositories.

## Best solution

Fix the state restoration logic in `src/vs/workbench/contrib/scm/browser/scmViewService.ts` so that the 'unknown repo' branch (index === -1) does not reset `didSelectRepository` in a way that causes previously-invisible repos to become visible. The interaction between the early-return at line 386 and the `didSelectRepository` flag needs to be reworked so that repos explicitly marked not-visible in `previousState` stay invisible regardless of new repo arrivals.
