# Issue #314923 — extensions.allowed should support version range

[https://github.com/microsoft/vscode/issues/314923](https://github.com/microsoft/vscode/issues/314923)

## Summary

Valid feature request still in the Backlog Candidates community process (10/20 upvotes, ~45 days remaining). Version range support is not implemented on main — current `extensions.allowed` only accepts exact version strings and platform-specific versions.

## Change summary

Support version range specifiers (e.g., `>=2025.8.1`, `<4.2.3`) in the `extensions.allowed` setting for more flexible enterprise extension management.

## Best solution

Implement semver range parsing in the extension allowlist evaluation logic (likely in the service that calls `isAllowed`). This would let enterprises respond to CVEs by allowing only patched versions without enumerating each one. The schema in `src/vs/platform/extensionManagement/common/extensionManagement.ts` would need to document range syntax, and the version-matching logic would need a semver comparator.
