# Issue #251735 — Screen doesn't automatically scroll when using mouse to select text if the status bar is disabled.

[https://github.com/microsoft/vscode/issues/251735](https://github.com/microsoft/vscode/issues/251735)

## Summary

Confirmed bug, still unfixed on main. The bottom-edge check in `_getPositionOutsideEditor` requires the mouse to exceed the editor height, which is physically impossible when the editor fills the screen (status bar hidden + maximized window). PR #313488 is open with a proposed fix.

## Change summary

Mouse-drag text selection doesn't auto-scroll downward when the status bar is hidden and the window is maximized, because the cursor can never physically exceed the editor's bottom edge.

## Best solution

Add a small bottom-edge padding to the boundary check in `src/vs/editor/browser/controller/mouseHandler.ts` at line 554, so the cursor is considered 'outside' when within a few pixels of the editor bottom. PR https://github.com/microsoft/vscode/pull/313488 is already pursuing this approach and should be reviewed/merged.
