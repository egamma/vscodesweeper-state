# Issue #326363 — No way to "uninstall" local plugin folder

[https://github.com/microsoft/vscode/issues/326363](https://github.com/microsoft/vscode/issues/326363)

## Summary

Confirmed in source: the plugin list context menu only shows 'Uninstall' for marketplace-installed plugins (`plugin.fromMarketplace`), excluding local folder plugins that do have a valid `remove` callback. Team-authored issue — keeping open.

## Change summary

Local folder plugins added via `chat.pluginLocations` cannot be uninstalled from the Customizations UI — only disabled.

## Best solution

In `getInstalledPluginContextMenuActions` (src/vs/workbench/contrib/chat/browser/agentPluginActions.ts line 127), broaden the condition from `if (plugin.fromMarketplace)` to `if (plugin.fromMarketplace || plugin.remove)` so that any plugin with a removal mechanism surfaces the Uninstall action.
