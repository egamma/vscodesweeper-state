# Issue #309036 — [Unhandled Error] Cannot read properties of undefined (reading 'toString')

[https://github.com/microsoft/vscode/issues/309036](https://github.com/microsoft/vscode/issues/309036)

## Summary

Still reproduces on current main: line 251 of inlineEditsLineReplacementView.ts calls `.map(c => c.toString())` on the result of `getEditorBlendedColor()` without guarding against undefined. The related fix PR (#309156) remains open/unmerged.

## Change summary

Calling `.toString()` on an undefined color value in the inline edits line replacement view causes an unhandled TypeError.

## Best solution

Merge or land the null guard from PR https://github.com/microsoft/vscode/pull/309156 (or equivalent) so that `getEditorBlendedColor(...).map(c => c.toString())` handles the case where the color is undefined. The fix is small and localized to the inline completions view layer.
