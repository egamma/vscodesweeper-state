# Issue #115157 — Modify issue reporter description text when signed in

[https://github.com/microsoft/vscode/issues/115157](https://github.com/microsoft/vscode/issues/115157)

## Summary

Still reproduces on main: the description subtitle always says 'when we preview it on GitHub' even when the user is signed in and the button reads 'Create on GitHub' (direct creation, no preview step). The text is misleading.

## Change summary

Issue reporter description text mentions 'preview on GitHub' even when the user is signed in and the action is 'Create on GitHub' (direct submission).

## Best solution

Make the description subtitle text conditional on auth state — when `githubAccessToken` is present, update the wording to reflect that the issue will be created directly (e.g. 'you can edit the created issue to add screenshots'). The fix is localized to `src/vs/workbench/contrib/issue/browser/baseIssueReporterService.ts` around lines 1196–1216.
