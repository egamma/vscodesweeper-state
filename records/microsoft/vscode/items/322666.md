# Issue #322666 — The follow-link copy button copies the `Follow link (ctrl + click)` text literally

[https://github.com/microsoft/vscode/issues/322666](https://github.com/microsoft/vscode/issues/322666)

## Summary

Confirmed source-reproducible: the hover copy button uses `getAccessibleContent()` which returns `innerText` of the rendered hover element. For link hovers, the markdown is rendered as `<a href="URL">Follow link</a> (ctrl + click)`, so `innerText` yields the display text, not the URL. The copy button feature was recently added in ab6ce975997.

## Change summary

The hover copy button on link hovers copies the literal display text "Follow link (ctrl + click)" instead of the target URL.

## Best solution

Either (a) have the MarkdownHoverParticipant's `getAccessibleContent` extract link href targets when the hover contains a single markdown link, or (b) set `hideCopyButton: true` on the MarkdownHoverParticipant for link-type hovers where the copy content would be misleading. The fix likely touches `src/vs/editor/contrib/hover/browser/markdownHoverParticipant.ts` (the `getAccessibleContent` method around line 409).
