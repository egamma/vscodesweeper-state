# Issue #269161 — Not sure how to instruct the model to use the reference

[https://github.com/microsoft/vscode/issues/269161](https://github.com/microsoft/vscode/issues/269161)

## Summary

Active design discussion about how git history file references should attach context in chat — whether to include the diff, the file contents, or both. A team member (lszomoru) identified the root cause and is considering the right UX gesture. No resolution or implementation found on main.

## Change summary

When a file from git history is referenced in chat, the attached context includes only the file contents at that commit, not the diff, causing the model to misunderstand the user's intent.

## Best solution

Decide the UX for attaching git history context in chat: either provide two gestures (attach file contents vs. attach diff), or have the single gesture attach both. Implementation would likely touch the SCM/git graph chat attachment code and the chat context resolution layer.
