# Issue #185224 — Code Actions - Fix All Support

[https://github.com/microsoft/vscode/issues/185224](https://github.com/microsoft/vscode/issues/185224)

## Summary

Milestone is "On Deck" (a scheduling decision), so this is a hard keep-open. The requested scoped Fix All UX (document/project/solution) and the corresponding LSP protocol additions do not exist on current main — only the existing `source.fixAll` code-action kind is present, which is a different, simpler mechanism.

## Change summary

Add scoped Fix All support for code actions (document, project, solution scope) with new LSP protocol, VS Code API, and UI — similar to Visual Studio's Fix All experience.

## Best solution

Implement scoped Fix All as a coordinated effort: new LSP spec additions (`codeAction/fixAll` request, `FixAllScope` enum), new VS Code extension API surface, and a condensed code-action-list UX that avoids flattening per-scope variants. This needs product and LSP spec alignment before implementation.
