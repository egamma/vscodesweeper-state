# Issue #281885 — SCM - add agent tool for fetching repository history

[https://github.com/microsoft/vscode/issues/281885](https://github.com/microsoft/vscode/issues/281885)

## Summary

Team-authored feature request for an SCM agent tool to fetch repository history. Not yet implemented — the existing SCM tool (`scmChangesTool.ts` / `get_changed_files`) only provides changed-file diffs, not commit history. Keeping open per hard keep-open rule (MEMBER author) and because the work remains undone.

## Change summary

Add an agent tool that lets chat/agent sessions query git commit history from the SCM layer.

## Best solution

Implement a new language-model tool (alongside `scmChangesTool.ts` in `extensions/copilot/src/extension/tools/node/`) that exposes `git log` data — commit messages, SHAs, authors, dates — to the agent context. The SCM history infrastructure already exists in `src/vs/workbench/contrib/scm/browser/scmHistory.ts` and `extensions/git/src/historyProvider.ts`; the tool would surface that data to the chat layer.
