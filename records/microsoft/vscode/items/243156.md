# Issue #243156 — SCM Graph - auto should include default branch when checked out detached

[https://github.com/microsoft/vscode/issues/243156](https://github.com/microsoft/vscode/issues/243156)

## Summary

Team-authored feature request — must stay open for maintainer judgment. The enhancement is still unimplemented on current main: in detached HEAD mode, the auto branch filter only includes the detached commit ref because historyItemRemoteRef and historyItemBaseRef are set to undefined.

## Change summary

When in detached HEAD mode, the SCM Graph 'auto' filter drops the default branch (and upstream), making it hard to navigate back.

## Best solution

In the git extension's history provider (extensions/git/src/historyProvider.ts, around the detached-commit branch at line 170), resolve the repository's default branch and set it as the base ref so that auto mode includes it. The SCM-side auto filter logic in src/vs/workbench/contrib/scm/browser/scmHistoryViewPane.ts (lines 1402-1407) would then pick it up automatically via historyItemBaseRef.
