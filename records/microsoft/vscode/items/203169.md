# Issue #203169 — Undocumented feature "Turn on Cloud Changes…" is visible

[https://github.com/microsoft/vscode/issues/203169](https://github.com/microsoft/vscode/issues/203169)

## Summary

Legitimate transparency and documentation request for the Cloud Changes feature, which is still present on main with the same 'Turn on Cloud Changes…' menu entry in the Accounts context menu. A contributor partially addressed questions in August 2024, but community concerns about privacy policy, encryption at rest, and data-handling documentation remain unanswered. With 38 upvotes and active discussion, this should stay open.

## Change summary

The 'Turn on Cloud Changes…' entry in the Accounts menu lacks adequate documentation about what the feature does, where data is stored, what privacy policy applies, and how stored changes are protected.

## Best solution

Add comprehensive user-facing documentation for Cloud Changes covering: (1) what it does and how it relates to Continue Working On, (2) where data is stored (same service as Settings Sync), (3) the applicable privacy policy, (4) encryption and data protection details, and (5) how to view/manage/delete stored changes. The menu entry itself in `src/vs/workbench/contrib/editSessions/browser/editSessionsStorageService.ts` could also benefit from a more descriptive label or a learn-more link.
