# Issue #203169 — Undocumented feature "Turn on Cloud Changes…" is visible

[https://github.com/microsoft/vscode/issues/203169](https://github.com/microsoft/vscode/issues/203169)

## Summary

The core ask — clear documentation and privacy transparency for Cloud Changes — has been partially addressed (joyceerhl added docs in Sept 2024, and commit 13d2a615cb9 hides the action when unconfigured), but the community continues to ask about data protection, privacy policy, and encryption at rest. With 38 upvotes and active discussion, this should stay open.

## Change summary

The 'Turn on Cloud Changes' menu item in the Accounts menu lacks documentation explaining what it does, where data is stored, and what privacy policy applies.

## Best solution

Complete the documentation to explicitly address the remaining community questions: what privacy/usage policy governs synchronized code, what encryption at rest protects the data, and whether stored changes could be used for model training. The docs page joyceerhl referenced should be expanded with a privacy/security section, and ideally the 'Turn on Cloud Changes' entry itself should link to it.
