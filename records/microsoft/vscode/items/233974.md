# Issue #233974 — Issue Reporter is counter productive

[https://github.com/microsoft/vscode/issues/233974](https://github.com/microsoft/vscode/issues/233974)

## Summary

The issue reporter has been substantially reworked since this was filed: a wizard-style multi-step UI was introduced, 'Create by default' replaced the old Preview-only flow, and the 'Preview on GitHub' URL was fixed to open the new-issue form instead of the repo landing page. The specific complaints are addressed by these changes.

## Change summary

The old issue reporter forced users to fill in title/steps before Preview was enabled, did not allow copying collected info, and Preview on GitHub opened the template-choice page instead of creating an issue.

## Best solution

Keep the shipped improvements: the wizard UI (c0a8d36b4ed), create-by-default with Preview as dropdown (164ebb939d5), and the Preview URL fix (4b24afbf129). The commenter's specific complaint about Preview opening the 'choose template' page was directly fixed. No further change needed beyond confirming on latest Insiders.
