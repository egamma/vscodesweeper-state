# Issue #248369 — Git - discard changes of a new untracked file fails when using a subst drive 

[https://github.com/microsoft/vscode/issues/248369](https://github.com/microsoft/vscode/issues/248369)

## Summary

Still reproduces on main — the `_clean` method in `extensions/git/src/repository.ts` uses `workspace.fs.delete(Uri.file(fsPath))` where the fsPath originates from the subst drive, but the repository root resolves to the real path, causing a mismatch when discarding untracked files via trash. An open community PR (https://github.com/microsoft/vscode/pull/248552) exists to fix this.

## Change summary

Discarding changes on a new untracked file fails when the VS Code workspace is opened via a Windows `subst` drive, because the file path from the subst drive doesn't resolve correctly against the repository root.

## Best solution

Review and land (or close) the open community PR https://github.com/microsoft/vscode/pull/248552 which addresses the subst-drive path mismatch in the git extension's `_clean` method. The fix is localized to `extensions/git/src/repository.ts`.
