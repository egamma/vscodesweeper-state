# Issue #194177 — F7 does not work consistently in diff view when there are comments

[https://github.com/microsoft/vscode/issues/194177](https://github.com/microsoft/vscode/issues/194177)

## Summary

Team-authored issue; the F7 keybinding conflict between wordHighlight.next and accessibleDiffViewer.next still exists on current main — both register at the same weight (EditorContrib), causing inconsistent behavior when word highlights are active in a diff editor.

## Change summary

F7 in a diff editor inconsistently triggers 'Go to Next Symbol Highlight' instead of 'Go to Next Difference' when word highlights are active.

## Best solution

Scope the accessibleDiffViewer.next keybinding to a higher weight (e.g. EditorContrib + 1) when in a diff editor, or add `&& !hasWordHighlights` to the wordHighlight.next when clause when `isInDiffEditor` is true, so F7 reliably navigates diffs. The fix touches `src/vs/editor/browser/widget/diffEditor/commands.ts` (bump weight or add when-clause) or `src/vs/editor/contrib/wordHighlighter/browser/wordHighlighter.ts` (restrict when-clause).
