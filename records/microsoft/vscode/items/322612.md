# Issue #322612 — Maximized secondary windows are not restored properly when main window reloads or app relaunches

[https://github.com/microsoft/vscode/issues/322612](https://github.com/microsoft/vscode/issues/322612)

## Summary

Legitimate bug with auxiliary (secondary) window restore when maximized. The state-save path correctly captures maximized mode, but the restore path appears to have an issue on reload or relaunch. No fix found on main, no duplicate identified. A second user confirms the behavior. Already labeled correctly with `bug` and `layout`.

## Change summary

Maximized secondary (auxiliary) windows are not properly restored after a window reload or app relaunch — they either disappear or lose their maximized state.

## Best solution

Investigate the restore flow in `src/vs/workbench/browser/parts/editor/editorParts.ts` (`restoreState`) and `src/vs/platform/auxiliaryWindow/electron-main/auxiliaryWindowsMainService.ts` to determine why the `window-maximized` feature flag doesn't result in a visible maximized window after reload. Likely a timing issue between window creation and maximize call, or a bounds problem where the saved bounds of a maximized window are invalid for a normal restore.
