# Issue #185762 — macOS tabs don't work correctly in fullscreen

[https://github.com/microsoft/vscode/issues/185762](https://github.com/microsoft/vscode/issues/185762)

## Summary

Already correctly triaged as upstream — the `newWindowTab` command delegates to Electron's native macOS tab API (`forceNewTabbedWindow`), and the tab-ordering issue in fullscreen is an Electron/macOS platform behavior. VS Code's own code has no logic that controls which project a new native tab inherits. Keeping open as an upstream tracker.

## Change summary

When using macOS native window tabs in fullscreen, creating a new tab from one project's tab incorrectly opens it in a different project's context.

## Best solution

This is an Electron/macOS platform issue. VS Code's `newWindowTab` implementation in `src/vs/platform/native/electron-main/nativeHostMainService.ts` simply passes `forceNewTabbedWindow: true` to the window-open path — no VS Code-side fix is feasible. The fix would need to come from Electron's `addTabbedWindow` handling. Consider filing or linking an upstream Electron issue if one doesn't already exist.
