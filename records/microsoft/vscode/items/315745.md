# Issue #315745 — Two scrollbars in plan widget

[https://github.com/microsoft/vscode/issues/315745](https://github.com/microsoft/vscode/issues/315745)

## Summary

Team-filed bug (MEMBER author) — keeping open. The plan review widget wraps its body in a DomScrollableElement (`_messageScrollable`) while also embedding a CodeEditorWidget that has its own `scrollbar: { vertical: 'auto' }`, producing two nested vertical scrollbars. No fix commit found on current main.

## Change summary

The plan review widget shows two overlapping vertical scrollbars when the plan content overflows.

## Best solution

Either disable the outer `_messageScrollable` vertical scrollbar when the inline editor is active, or set the editor's own vertical scrollbar to `hidden` and let the outer scrollable handle overflow. The fix belongs in `src/vs/workbench/contrib/chat/browser/widget/chatContentParts/chatPlanReviewPart.ts`.
