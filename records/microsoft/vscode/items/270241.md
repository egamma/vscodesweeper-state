# Issue #270241 — Clicking (+) on the current file attachment doesn't get rid of the ghost/italics one

[https://github.com/microsoft/vscode/issues/270241](https://github.com/microsoft/vscode/issues/270241)

## Summary

Team-authored bug (MEMBER) — the implicit context attachment widget doesn't re-render after converting a ghost attachment to a regular one via the (+) button, leaving both visible. Keeping open per team-author policy.

## Change summary

Clicking (+) on a ghost/implicit file attachment converts it to a regular attachment but doesn't hide the ghost version, resulting in duplicate display.

## Best solution

After `convertToRegularAttachment` completes and the attachment model is updated, the implicit context widget needs to re-render (call `render()` or the parent needs to rebuild the widget) so that `isAttachmentAlreadyAttached` can filter out the now-attached context. The fix likely lives in `implicitContextAttachment.ts` — either by subscribing to attachment-model changes to trigger a re-render, or by immediately re-rendering after `convertToRegularAttachment`. The notebook-specific angle (per roblourens's comment) may mean the (+) widget shouldn't appear at all for notebook cells in certain conditions.
