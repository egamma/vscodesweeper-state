# Issue #239849 — Searching `c` no longer brings up the `C` extension first

[https://github.com/microsoft/vscode/issues/239849](https://github.com/microsoft/vscode/issues/239849)

## Summary

Still reproduces on main: the local preferred-extensions matching in `getPreferredExtensions` uses a naive `indexOf` with no relevance scoring, so any installed extension containing the search text in name/displayName/description gets surfaced equally — burying short exact-name matches like 'C' below 154 others.

## Change summary

Searching 'c' in the extensions view no longer surfaces the C/C++ extension first because the local matching has no relevance ranking.

## Best solution

Add relevance scoring to `getPreferredExtensions` in `extensionsViews.ts` — prioritize exact displayName matches, then prefix matches, then substring matches, and sort by score before returning. This would bring local search quality closer to the marketplace's relevance algorithm.
