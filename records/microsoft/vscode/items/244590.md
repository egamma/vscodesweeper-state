# Issue #244590 — Git - Bad git diff handling for UTF-16 LE file

[https://github.com/microsoft/vscode/issues/244590](https://github.com/microsoft/vscode/issues/244590)

## Summary

Confirmed bug with 'On Deck' milestone — must stay open per scheduling rules. The issue is traceable in the source: `git show --textconv` returns raw UTF-16 LE bytes, and the diff pipeline doesn't properly handle this encoding, causing garbled display and false 'modified' status.

## Change summary

UTF-16 LE encoded files show garbage characters in the git diff view and appear falsely modified.

## Best solution

The git extension's `buffer()` method (extensions/git/src/git.ts ~L1571) retrieves file content via `git show --textconv` as raw bytes. For UTF-16 LE files without a .gitattributes `diff` filter, this returns raw UTF-16 bytes that are then misinterpreted. The fix likely needs to detect UTF-16 encoding (the `detectUnicodeEncoding` utility already exists in util.ts) and transcode the buffer, or ensure the diff view's encoding detection aligns with the working-tree file's encoding so they compare correctly.
