# Issue #267927 — SCM Graph item hover position incorrect after Issue Reporter opened and moved to second display

[https://github.com/microsoft/vscode/issues/267927](https://github.com/microsoft/vscode/issues/267927)

## Summary

Confirmed multi-window hover positioning bug — hover coordinates become stale after an auxiliary window (Issue Reporter) is opened and moved to a second display. No fix found on current main; keeping open.

## Change summary

SCM Graph hover widget position is incorrect after the Issue Reporter auxiliary window is opened and moved to a second display, until the main window regains focus.

## Best solution

The hover positioning logic in `src/vs/platform/hover/browser/hoverWidget.ts` likely needs to account for window coordinate offsets when the main window hasn't received a focus event after an auxiliary window is moved. The fix would ensure hover target coordinates are recalculated relative to the correct window bounds even without a focus change.
