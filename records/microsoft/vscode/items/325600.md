# Issue #325600 — Third-party extensions cannot configure TLS CA verification — structural disparity with BYOK

[https://github.com/microsoft/vscode/issues/325600](https://github.com/microsoft/vscode/issues/325600)

## Summary

Valid feature request about TLS CA parity for third-party extensions, though the existing `http.electronFetch` setting already provides a workaround by routing `globalThis.fetch()` through Electron's network stack (OS cert store). The request for better defaults and documentation remains actionable.

## Change summary

Third-party extensions using `globalThis.fetch()` in the extension host cannot leverage the OS certificate store for TLS verification, unlike first-party features that use `electron.net.fetch`.

## Best solution

The `http.electronFetch` setting (src/vs/platform/request/common/request.ts:257) already exists and routes extension `globalThis.fetch()` through `electron.net.fetch` on desktop, solving the incomplete-chain problem. Consider either enabling it by default (with appropriate testing) or documenting it as the recommended workaround for enterprise environments with custom CAs. The `http.systemCertificates` and `http.systemCertificatesNode` settings also load additional certs into the proxy-agent path.
