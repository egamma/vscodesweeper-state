# Issue #263722 — Code-Insiders blocks sleep and there is no way to disable this feature

[https://github.com/microsoft/vscode/issues/263722](https://github.com/microsoft/vscode/issues/263722)

## Summary

The report says VS Code Insiders blocks sleep on Linux, but the root cause is unclear — it could be the built-in Copilot extension's power-save blocker (which activates during chat streaming, gated by experiment), the Remote Tunnel feature's optional prevent-sleep setting, or Electron's own behavior. More info is needed to identify the source.

## Change summary

VS Code Insiders registers a systemd sleep inhibitor on Linux with no user-facing way to disable it.

## Best solution

Determine whether the sleep inhibitor comes from the Copilot extension's `PowerService` (in `extensions/copilot/src/extension/power/`), the Remote Tunnel's `remote.tunnelAccess.preventSleep` setting, or Electron itself. If it's the Copilot extension, the fix belongs in the Copilot extension (expose a setting or respect a config to disable the power-save blocker). If it's Electron core, it's upstream.
