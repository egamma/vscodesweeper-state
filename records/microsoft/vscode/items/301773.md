# Issue #301773 — Search: Mode reuseEditor doesn't take you to the correct line in the source code editor after you make changes in the source code editor

[https://github.com/microsoft/vscode/issues/301773](https://github.com/microsoft/vscode/issues/301773)

## Summary

Real bug: the search editor's definition provider (in extensions/search-result/src/extension.ts) computes target line numbers from the static search-result text at parse time, so after the user edits the source file (adds/deletes lines), double-clicking a result navigates to a stale line number. No fix has landed on main.

## Change summary

When using Search mode reuseEditor, double-clicking a search result navigates to the wrong line after the user has added or deleted lines in the source file.

## Best solution

The search-result definition provider should either track document edits to adjust target line numbers, or the search editor should re-run the search / invalidate stale results when the source file changes. The fix would touch extensions/search-result/src/extension.ts (the definition provider's targetSelectionRange computation) and possibly the search editor's result-tracking in src/vs/workbench/contrib/searchEditor/.
