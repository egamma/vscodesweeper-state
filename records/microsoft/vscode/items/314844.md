# Issue #314844 — [Proposal] Risk scoring for pull requests — prioritize reviews & CI

[https://github.com/microsoft/vscode/issues/314844](https://github.com/microsoft/vscode/issues/314844)

## Summary

PR risk scoring is a specialized DevOps/CI capability that fits squarely in the extension model rather than VS Code core — it needs no new core API surface and can be built today with the SCM extension API, GitHub APIs, and git tooling.

## Change summary

Add a built-in risk-scoring system for pull requests to prioritize reviews and CI resources.

## Best solution

Build this as a Marketplace extension (or GitHub Action). The SCM API, `git` CLI, and GitHub REST/GraphQL APIs already provide all the signals described (diff size, path mapping, churn history, test coverage). No core editor change is needed.
