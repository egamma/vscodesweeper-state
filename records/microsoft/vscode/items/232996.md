# Issue #232996 — Editor GPU: Lines that get decorations after initial render should be hidden in the canvas

[https://github.com/microsoft/vscode/issues/232996](https://github.com/microsoft/vscode/issues/232996)

## Summary

Still present on main: when a line gains decorations that make it non-GPU-renderable after initial render, the viewport render strategy skips writing to the GPU buffer (line 250 continue) but does not clear previously rendered content for that line, causing a visual doubling with the DOM fallback layer.

## Change summary

Lines initially rendered by the GPU renderer flash bold/doubled when decorations arrive that force DOM fallback, because the canvas pixels are not cleared.

## Best solution

In `viewportRenderStrategy.ts`, when `canRender` returns false for a previously-GPU-rendered line, actively clear that line's cells in the GPU buffer (write empty/zero data) so the canvas no longer shows stale content underneath the DOM fallback render.
