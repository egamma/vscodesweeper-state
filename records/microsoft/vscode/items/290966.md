# Issue #290966 — Spinner moves down while thinking text loads

[https://github.com/microsoft/vscode/issues/290966](https://github.com/microsoft/vscode/issues/290966)

## Summary

Team-authored polish item about whether the in-thinking spinner should be hidden while thinking text is actively streaming (matching the convention used for the main response spinner). The thinking content part has been heavily reworked since filing but the spinner-while-streaming behavior still exists in `updateWorkingSpinnerVisibility` — the spinner is only removed for running terminal tools, not when thinking text is streaming. Needs a UX/design decision from the team.

## Change summary

The spinner inside the thinking container moves down as thinking text loads, rather than being hidden while text is streaming.

## Best solution

Decide whether the in-thinking spinner should follow the same convention as the main response spinner (hidden while text streams). If yes, update `updateWorkingSpinnerVisibility` in `chatThinkingContentPart.ts` to also hide the spinner when thinking markdown content is actively streaming. The comment at line 576 already documents the current design rationale.
