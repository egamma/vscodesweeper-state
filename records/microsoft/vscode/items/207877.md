# Issue #207877 — Testing API > Tags Feedback

[https://github.com/microsoft/vscode/issues/207877](https://github.com/microsoft/vscode/issues/207877)

## Summary

Both requested enhancements remain unimplemented on current main — the filter still requires the full `@ctrlId:tagId` format with no shorthand for a single-provider case, and uses the controller ID rather than its label. Keeping open as a valid feature request in its area.

## Change summary

Allow filtering tests by tag without requiring the test controller ID prefix, and show the controller label instead of its internal ID in the filter UI.

## Best solution

In `src/vs/workbench/contrib/testing/common/testExplorerFilterState.ts`, accept a bare `@tagId` (without the `ctrlId:` prefix) when only one controller is active, resolving ambiguity by matching across all controllers. In the suggestion provider (`testingExplorerFilter.ts` line ~87–93), display the controller's user-facing label rather than its internal ID. Low upvote count — let the community process decide priority.
