# Issue #244491 — Terminal QuickFix Provider and Command with arguments fail.

[https://github.com/microsoft/vscode/issues/244491](https://github.com/microsoft/vscode/issues/244491)

## Summary

Confirmed on current main: the terminal quick fix addon never passes command arguments to `executeCommand`. The `ITerminalQuickFixCommandAction` interface lacks an `arguments` property, and the `run` closure in `quickFixAddon.ts` ignores them entirely.

## Change summary

When a terminal quick fix provider returns a command action with arguments, applying the fix executes the command without its arguments, causing a runtime error.

## Best solution

Add an `arguments?: unknown[]` field to `ITerminalQuickFixCommandAction` in `quickFix.ts`, and update the `run` closure in `quickFixAddon.ts` line 465 to spread those arguments into `commandService.executeCommand(fix.id, ...(fix.arguments ?? []))`.
