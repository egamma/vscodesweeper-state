# Issue #314918 — Chat: Run command approval prompt truncates command

[https://github.com/microsoft/vscode/issues/314918](https://github.com/microsoft/vscode/issues/314918)

## Summary

The terminal command confirmation UI was significantly reworked after this issue was filed (PR #315215 and follow-ups), replacing the truncated text with a word-wrapped Monaco editor that can be expanded. The worst of the truncation is fixed, but the collapsed max-height (200px/30vh) may still hide very long commands until the user expands — so the polish concern remains valid for extremely long commands.

## Change summary

The run-command approval prompt truncates long terminal commands, preventing users from reviewing what they're approving.

## Best solution

The current editor-based confirmation (with word wrap and expand) largely addresses this. Remaining polish: consider increasing the default collapsed max-height for the terminal editor area, or auto-sizing it to the command length (capped at a reasonable limit), so users don't need to manually expand to review typical multi-line commands. The relevant CSS is in `chatToolConfirmationCarousel.css` (`.chat-confirmation-widget-message { max-height: min(200px, 30vh) }`).
