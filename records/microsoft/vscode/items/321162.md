# Issue #321162 — Feature request: Option to disable "Keep/Undo" confirmation popup when discarding Copilot Agent changes

[https://github.com/microsoft/vscode/issues/321162](https://github.com/microsoft/vscode/issues/321162)

## Summary

Valid feature request: the 'Undo all edits' confirmation dialog in chat editing sessions has no opt-out setting, unlike the similar confirmEditRequestRemoval and confirmEditRequestRetry settings. Routing to the chat-agent-editing area.

## Change summary

Add a setting to disable the confirmation dialog shown when discarding/undoing all Copilot chat editing changes.

## Best solution

Add a `chat.editing.confirmDiscardAll` boolean setting (defaulting true) gating the dialog in `discardAllEditsWithConfirmation()` in `src/vs/workbench/contrib/chat/browser/chatEditing/chatEditingActions.ts`, mirroring the existing pattern used by `chat.editing.confirmEditRequestRemoval`.
