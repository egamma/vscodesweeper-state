# Issue #251459 — Allow Extensions to Register In-Process MCP Servers via In-Memory Transport

[https://github.com/microsoft/vscode/issues/251459](https://github.com/microsoft/vscode/issues/251459)

## Summary

Valid feature request on the 'On Deck' milestone — a scheduling decision that blocks automated closure. The request for in-process MCP server registration via in-memory transport is well-articulated and aligns with the current MCP investment.

## Change summary

Allow extensions to register an MCP server implemented directly in-process via an in-memory transport, avoiding the need for external processes or networked interfaces.

## Best solution

Expose a `registerInMemoryMcpServer(transport)` API (or equivalent) in the MCP extension API surface that wires an extension-hosted MCP server to VS Code's internal MCP client via the existing MCP SDK transport abstraction. This complements the existing `registerMcpServerDefinitionProvider` for external servers and the tool registration API.
