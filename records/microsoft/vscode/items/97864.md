# Issue #97864 — Cursor changes to arrow when switching windows (macOS)

[https://github.com/microsoft/vscode/issues/97864](https://github.com/microsoft/vscode/issues/97864)

## Summary

This is a confirmed upstream Electron/macOS bug where the cursor gets stuck as an arrow after context menus or window switches. A VS Code team member (alexdima) reproduced and confirmed it as an Electron issue in 2021, demonstrating it does not occur in Chrome. The issue already carries the correct labels (upstream, electron, upstream-issue-pending). Related issue https://github.com/microsoft/vscode/issues/238796 was closed and consolidated back to this one. No fix has landed in Electron or VS Code, and the bug is still reported as of May 2026.

## Change summary

On macOS, the cursor gets stuck as an arrow instead of the I-beam after switching VS Code windows or dismissing context menus, requiring a mouse-leave/enter cycle to recover.

## Best solution

The fix belongs in Electron — after a native context menu is dismissed or a window switch occurs on macOS, Electron should refresh the cursor style to reflect the element under the pointer. VS Code should keep tracking this until Electron resolves the underlying platform behavior. If an Electron issue number is identified, link it via the upstream-issue-linked label.
