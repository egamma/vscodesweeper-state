# Issue #201842 — Cache the list of files for "Go to file" / "Quick open" / "Search files by name"

[https://github.com/microsoft/vscode/issues/201842](https://github.com/microsoft/vscode/issues/201842)

## Summary

VS Code already has a per-session file query cache (`FileQueryCacheState`) that reuses results across picker invocations, but the user is asking for a persistent cross-session cache and/or file-watcher-driven incremental updates for very large repos on Windows where initial file enumeration is slow (~2s for 100k files). The existing cache doesn't fully address this — a real gap remains for the first-open cold-start case, especially on Windows.

## Change summary

Cache the file list used by Quick Open (Ctrl+P) across sessions so that opening the picker doesn't re-enumerate the filesystem each time, especially on Windows where file enumeration is slow.

## Best solution

The existing `FileQueryCacheState` already caches results within a session. The remaining gap is cold-start latency on large repos, especially on Windows. A persistent or file-watcher-driven cache could help, but involves significant complexity (cache invalidation, memory cost). With only 3 upvotes and being in Backlog Candidates, this should stay open for the community process to decide traction. The `search` area label is already correct.
