# Issue #201842 — Cache the list of files for "Go to file" / "Quick open" / "Search files by name"

[https://github.com/microsoft/vscode/issues/201842](https://github.com/microsoft/vscode/issues/201842)

## Summary

Legitimate feature request for persistent cross-session file-list caching in Quick Open. VS Code already has per-session caching (FileQueryCacheState + rawSearchService cache map), but it is rebuilt each time Quick Open opens. The request for a persistent cache with file-watching invalidation remains unimplemented. Low upvote count (3) but the request is technically sound; keeping open for the community process.

## Change summary

Cache the file enumeration results across Quick Open sessions so the file picker doesn't re-walk the filesystem every time it opens.

## Best solution

The existing per-session cache in `src/vs/workbench/contrib/search/common/cacheState.ts` and `src/vs/workbench/services/search/node/rawSearchService.ts` could be extended to persist across Quick Open invocations, invalidated via the existing file-watcher infrastructure. This would primarily benefit large repos on Windows where filesystem enumeration is slow. An opt-in setting would be appropriate given the memory trade-off. Alternatively, this may be partially addressed if the upstream Windows filesystem enumeration speed improves.
