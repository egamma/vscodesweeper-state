# Issue #232700 — Git Source Control: Support keyboard-only staging

[https://github.com/microsoft/vscode/issues/232700](https://github.com/microsoft/vscode/issues/232700)

## Summary

Valid feature request for keyboard-only SCM staging. The maintainer acknowledged the gap and has been incrementally improving keyboard support for Git commands (git.stageFile, git.diff.stageHunk), but the core ask — staging a focused-but-not-opened file in the SCM list via a keybinding — remains unaddressed on main.

## Change summary

Allow staging the currently focused file in the SCM list view via a keyboard shortcut, without needing to open the file first or use the context menu.

## Best solution

Extend `git.stage` (or add a dedicated command) so that when invoked without arguments and no editor is active, it resolves the focused resource from the SCM list view. The maintainer (lszomoru) already did similar fallback work for `git.stageFile` in commit 61deced4744 — the same pattern could apply to `git.stage` by querying the focused SCM resource state.
