# Issue #30406 — Autoindent inserts extra undo item, requires double undo after paste

[https://github.com/microsoft/vscode/issues/30406](https://github.com/microsoft/vscode/issues/30406)

## Summary

The extra undo stop is still present on main (pushUndoStop calls at lines 528/531 in indentation.ts). The new `autoIndentOnPaste` setting provides a workaround (disable auto-indent entirely) but doesn't resolve the undo-grouping behavior when auto-indent is enabled. The discussion about the ideal UX remains unresolved.

## Change summary

Pasting with auto-indent enabled creates a separate undo stop for the indentation adjustment, requiring two undos to fully reverse a paste.

## Best solution

Either group the auto-indent edit into the same undo stop as the paste (removing the pushUndoStop pair around the AutoIndentOnPasteCommand in `src/vs/editor/contrib/indentation/browser/indentation.ts`), or add a dedicated setting to control undo grouping. The community is split: some users rely on single-undo to strip formatting. A setting or the existing `autoIndentOnPaste` toggle may be sufficient, but the under-discussion status should remain until the team decides on the default behavior.
