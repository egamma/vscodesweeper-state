# Issue #319193 — Unable to change "debug.activeStackItem"

[https://github.com/microsoft/vscode/issues/319193](https://github.com/microsoft/vscode/issues/319193)

## Summary

Valid feature request for a writable `debug.activeStackItem` extension API. Currently the property is read-only (`export const`). No existing issue tracks making it settable, and the referenced parallel-callstacks issue (https://github.com/microsoft/vscode/issues/272887) is a separate feature. Keeping open for the community process.

## Change summary

Allow extensions to programmatically set the active debug stack frame/thread via the `debug.activeStackItem` API, so that custom debug views can change the debugger's focused frame.

## Best solution

Add a setter or `debug.setActiveStackItem(item)` method to the extension API, backed by `IDebugService.focusStackFrame` in `src/vs/workbench/contrib/debug/browser/debugService.ts`. This is a small, well-scoped API addition in the debug area. Should go through the proposed-API process.
