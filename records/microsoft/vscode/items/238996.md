# Issue #238996 — Filtering of output channels should run asynchronously and avoid locking up the renderer

[https://github.com/microsoft/vscode/issues/238996](https://github.com/microsoft/vscode/issues/238996)

## Summary

Still reproduces on main: the output channel filter runs synchronously over all lines/log entries in `compute()` without yielding, blocking the renderer for large channels. No async chunking has been added.

## Change summary

Output channel filtering blocks the renderer for large logs because it processes all lines synchronously in one frame.

## Best solution

Chunk the `compute()` loop in `src/vs/workbench/contrib/output/browser/outputView.ts` using an async yielding mechanism (e.g. `IdleTaskQueue` from `src/vs/editor/browser/gpu/taskQueue.ts` or `runWhenGlobalIdle`) so the filtering yields back to the event loop periodically and avoids locking the renderer.
