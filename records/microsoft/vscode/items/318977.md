# Issue #318977 — Chinese IME Compatibility Issues in Model Picker Search

[https://github.com/microsoft/vscode/issues/318977](https://github.com/microsoft/vscode/issues/318977)

## Summary

Still reproduces on main: the action list's filter input in the model picker has no IME composition handling — the keydown forwarder at line 922 of actionList.ts doesn't check `e.isComposing`, breaking Chinese (and other) IME input.

## Change summary

Chinese IME input in the model picker search box produces garbled/incorrect results because keydown events during composition are intercepted and forwarded prematurely.

## Best solution

Add `e.isComposing` guards to the keydown handlers in `src/vs/platform/actionWidget/browser/actionList.ts` (the character-forwarding handler at ~line 922 and the ArrowRight submenu handler at ~line 902), matching the pattern used elsewhere in the codebase for IME-safe keyboard handling.
