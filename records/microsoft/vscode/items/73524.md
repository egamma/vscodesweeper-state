# Issue #73524 — Stabilize FileSearchProvider API

[https://github.com/microsoft/vscode/issues/73524](https://github.com/microsoft/vscode/issues/73524)

## Summary

The FileSearchProvider API remains proposed (in `vscode.proposed.fileSearchProvider2.d.ts`) and has not been finalized into the stable API surface (`vscode.d.ts`). With 42 upvotes and active community demand, this is a valid, unfinished work item that should stay open.

## Change summary

Finalize the FileSearchProvider API from proposed to stable so extensions can use file search without enabling proposed APIs.

## Best solution

A search-area owner should pick up where @andreamah left off: the v2 shape (`FileSearchProvider2`, `FileSearchProviderOptions`) in `vscode.proposed.fileSearchProvider2.d.ts` looks close to final — review it, resolve any remaining design questions (the old v1 shape in `vscode.proposed.fileSearchProvider.d.ts` should be deprecated), and move the types into `vscode.d.ts`. The sibling issue https://github.com/microsoft/vscode/issues/59921 (TextSearchProvider) should ideally be finalized together, as @andreamah planned.

## Vision fit

- **Fit:** aligned
- **Reason:** Stabilizing the FileSearchProvider API strengthens the extension ecosystem — a durable theme — and enables virtual filesystem extensions (remote development) to provide proper search, which aligns with both the extension-ecosystem and remote-development themes.
- **Evidence:** Durable theme: "Extension ecosystem — API hygiene (finalize or clean out proposed APIs)." Also supports remote development scenarios where FileSystemProvider implementations need search.
