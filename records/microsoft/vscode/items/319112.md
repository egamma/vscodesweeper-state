# Issue #319112 — workbench.action.findInFiles: accept file URIs to bypass argv-length limits

[https://github.com/microsoft/vscode/issues/319112](https://github.com/microsoft/vscode/issues/319112)

## Summary

Valid feature request: the internal `commonQueryFromFileList` path already exists for the SCM toggle but is not exposed through `IFindInFilesArgs`. No duplicate found, no existing capability covers this for extensions.

## Change summary

Expose a `filesToIncludeUris` (or URI[] overload) on the `findInFiles` / search editor commands so extensions can bypass the argv-length limit that the string-based `filesToInclude` hits on Windows.

## Best solution

Add a `filesToIncludeUris?: URI[]` field to `IFindInFilesArgs` in `src/vs/workbench/contrib/search/browser/searchActionsBase.ts` and route it through the existing `QueryBuilder.commonQueryFromFileList` path. This reuses the same infrastructure the built-in SCM toggle already uses, keeping the implementation small and well-tested.
