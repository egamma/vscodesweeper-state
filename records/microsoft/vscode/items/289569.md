## Summary

Still reproduces on main: `convertViewPositionToModelPosition` at viewModelLines.ts:837 accesses `modelLineProjections[index]` without a bounds/undefined guard, causing the reported crash. Affects 1.5M+ users across all platforms.

## Change summary

Unguarded array access in `convertViewPositionToModelPosition` crashes when `modelLineProjections` element is undefined during cursor state changes.

## Best solution

Add a defensive bounds check in `convertViewPositionToModelPosition` (and potentially other unguarded access sites in `ViewModelLinesFromProjectedModel`) before dereferencing `modelLineProjections[index]`, returning a safe fallback position when the projection is unavailable.
