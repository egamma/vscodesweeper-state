# Issue #200692 — Ctrl+w not deleting word backward in Command Palette

[https://github.com/microsoft/vscode/issues/200692](https://github.com/microsoft/vscode/issues/200692)

## Summary

The QuickPick input remains a plain HTML FindInput, not a Monaco editor, so editor commands like deleteWordLeft cannot fire there. This is a valid feature request to support richer editing keybindings in the QuickPick input — still unaddressed on main.

## Change summary

Ctrl+W (deleteWordLeft) doesn't work in the Command Palette input because it's a plain HTML input, not a Monaco editor.

## Best solution

Either convert the QuickPick input to a Monaco instance (as Tyler suggested) or wire native-input word-deletion keybindings (Ctrl+W mapped to the browser's word-delete behavior) into the QuickInputBox. The work lives in `src/vs/platform/quickinput/browser/quickInputBox.ts` and potentially `quickInputActions.ts`.
