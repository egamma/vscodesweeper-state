# Issue #125422 — Setting service should return default value when json scheme is invalid

[https://github.com/microsoft/vscode/issues/125422](https://github.com/microsoft/vscode/issues/125422)

## Summary

Team-authored debt item that remains unimplemented — `getValue<T>` still returns the generic `T` without validation, and no schema-based sanitization has been added to the configuration service. The latest discussion (Nov 2023, comment 13) outlined a concrete plan for validation in the service layer, but it hasn't landed. Keeping open per team-author policy and because the underlying work is still valid.

## Change summary

The configuration service should validate user-provided setting values against their registered JSON schema and return the default value when the stored value is invalid.

## Best solution

Implement the plan from comment 13: add validation in the configuration service (not consumers) for non-default settings, using the registered schema; allow optional validator callbacks for complex settings. The likely entry point is `ConfigurationModel.getValue` in `src/vs/platform/configuration/common/configurationModels.ts` and the registration layer in `src/vs/platform/configuration/common/configurationRegistry.ts`. The separate but related `getValue<unknown>` migration (~850 errors per comment 10) could be a follow-up engineering item.
