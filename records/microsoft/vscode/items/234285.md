# Issue #234285 — Quick search does not retain search term when moving to view

[https://github.com/microsoft/vscode/issues/234285](https://github.com/microsoft/vscode/issues/234285)

## Summary

Still reproduces on main. The initial fix (ffd85e7fe07) was merged but doesn't handle the race condition when searching quickly — the async refresh promise gets canceled before the search model's query is populated, so `refreshInputs()` reads an empty pattern. No subsequent fix has landed.

## Change summary

Quick search does not carry over the search term to the Search view when the user pastes and navigates to the view very quickly.

## Best solution

Fix the race condition in `moveToSearchViewlet` or `replaceSearchModel`: either guard against the canceled `refreshPromise` in `asyncDataTree.ts` (as sahin52 identified in Comment 5), or ensure the search query text is passed directly to the search widget rather than relying solely on `searchResult.query` which may not be populated yet due to the cancellation. The fix is localized to `searchView.ts` and/or `textSearchQuickAccess.ts`.
