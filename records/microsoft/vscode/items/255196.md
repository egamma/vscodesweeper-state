# Issue #255196 — Copilot instructions in multi-root should apply per folder

[https://github.com/microsoft/vscode/issues/255196](https://github.com/microsoft/vscode/issues/255196)

## Summary

Valid feature request with active team discussion and no resolution. The existing `applyTo` mechanism is glob-pattern-based, not workspace-folder-scoped. The requested per-folder scoping for multi-root workspaces has not been implemented.

## Change summary

In multi-root workspaces, copilot instruction files from all workspace folders are applied to every agentic session regardless of which folder the work targets.

## Best solution

Implement folder-aware instruction scoping in `src/vs/workbench/contrib/chat/common/promptSyntax/computeAutomaticInstructions.ts` so that when files are generated in a specific workspace folder, only that folder's instruction files are automatically applied (unless an instruction's `applyTo` pattern explicitly crosses folders). Team should first resolve the design disagreement (all-inclusive vs per-folder) visible in comments.
