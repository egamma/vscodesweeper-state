# Issue #226047 — SCM Graph - HTML tags are omitted from pop-ups for git commit messages

[https://github.com/microsoft/vscode/issues/226047](https://github.com/microsoft/vscode/issues/226047)

## Summary

Still reproduces on main: commit messages containing HTML tags have those tags stripped in SCM Graph and Timeline hover tooltips because the message is injected via `appendMarkdown()` without escaping HTML syntax.

## Change summary

HTML tags in git commit messages are silently stripped from hover tooltips in the SCM Graph and Timeline views.

## Best solution

In `extensions/git/src/hover.ts` line 125, the commit message is passed to `appendMarkdown()` raw — HTML-like content (e.g. `<div>`) is interpreted as HTML and stripped by the markdown renderer. The fix is to escape `<` and `>` in the message before appending as markdown (similar to how `![` is already escaped to `&#33;&#91;`), or restructure the call to use `appendText()` for the message body while preserving emoji and newline handling.
