# Issue #252197 — Popup style is off on wayland

[https://github.com/microsoft/vscode/issues/252197](https://github.com/microsoft/vscode/issues/252197)

## Summary

This is a Wayland-specific window-decoration issue — non-rounded corners and wrong title-bar text color on popup windows — rooted in how Electron/Chromium renders client-side decorations under Wayland compositors. VS Code's inner dialog CSS already uses proper border-radius tokens; the problem is at the native window frame level, which Electron controls. Keeping open as an upstream issue.

## Change summary

Popup windows on Wayland with custom title bar style show non-rounded corners and black title text on dark backgrounds.

## Best solution

Track as an upstream Electron/Chromium Wayland CSD (client-side decoration) issue. The fix needs to come from Electron properly communicating dark-theme and corner-radius hints to the Wayland compositor. A workaround exists (switching to native title bar style), but that triggers the related #224515 menu-sizing problem. The area owner should confirm whether an upstream Electron issue exists and link it.

## Evidence

- **upstream root cause:** Window-frame rounded corners and title-bar text color on Wayland are controlled by Electron/Chromium's CSD interaction with the compositor, not by VS Code's CSS. VS Code's dialog CSS already applies border-radius via var(--vscode-cornerRadius-xLarge) in src/vs/base/browser/ui/dialog/dialog.css, but that only affects the inner rendered dialog, not the native window chrome.
- **related open issue:** https://github.com/microsoft/vscode/issues/224515 — 'Context menu gets too small depending on the cursor position' is the issue the reporter references as the alternative breakage when switching to native title bar style; it is also labeled upstream-issue-linked and electron.
- **reporter workaround:** Setting window.titleBarStyle to native avoids the popup styling issue but triggers #224515 menu-sizing problems — both paths are broken on Wayland.
