# Issue #326421 — Provide manual model selection for AI-generated git change suggestions

[https://github.com/microsoft/vscode/issues/326421](https://github.com/microsoft/vscode/issues/326421)

## Summary

Reasonable UX polish request: the user wants a quick-pick model selector for Git AI suggestions (commit messages, etc.) similar to the chat model picker, rather than editing JSON settings. The underlying settings (`chat.byokUtilityModelDefault`, `chat.utilityModel`) already exist but lack an accessible in-context UI. Keeping open for community traction and area routing.

## Change summary

Add an inline model-picker UI for Git AI-generated suggestions (commit messages, etc.) instead of requiring users to edit JSON settings to change the utility model.

## Best solution

Route to the `git` and `chat` area owners. A model picker affordance in the SCM input (similar to the chat model dropdown) could let users quickly switch the utility model for commit message generation. Alternatively, this may be addressed by a broader effort to surface utility model selection more prominently in the UI. The existing `chat.utilityModel` / `chat.byokUtilityModelDefault` settings already control the behavior — this is about surfacing them more conveniently.
