# Issue #289129 — MCP HTTP Client Encoding Query String Parameters

[https://github.com/microsoft/vscode/issues/289129](https://github.com/microsoft/vscode/issues/289129)

## Summary

Still reproduces on main: URI.parse decodes percent-encoded path segments (%3A, %2F) and toString(true) does not re-encode them, breaking URLs where specific encoding is semantically required (e.g. AWS ARN-based paths). The prior fix (#247111/fc5d4141f9b) addressed query-parameter encoding by switching to toString(true), but it doesn't help when the path contains intentionally percent-encoded reserved characters.

## Change summary

MCP HTTP client loses percent-encoding in URL path segments (e.g. %3A, %2F) because URI.parse normalizes them, causing 404s with servers like AWS Bedrock AgentCore that require exact encoding.

## Best solution

Store the raw URL string alongside or instead of the URI object in McpServerTransportHTTP, and pass it directly to fetch without round-tripping through URI.parse/toString. The discovery adapters (nativeMcpDiscoveryAdapters.ts, installedMcpServersDiscovery.ts, pluginMcpDiscovery.ts) should preserve the original URL string for HTTP transport, and extHostMcp.ts should use it for requests.
