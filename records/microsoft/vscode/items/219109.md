# Issue #219109 — Strings in editor build use unicode escape sequences.

[https://github.com/microsoft/vscode/issues/219109](https://github.com/microsoft/vscode/issues/219109)

## Summary

Team-authored engineering debt issue still relevant on main — both build/lib/nls.ts and build/next/nls-plugin.ts emit nls.messages.js via plain JSON.stringify, which produces bloated unicode escape sequences for non-ASCII strings instead of raw UTF-8.

## Change summary

The monaco-editor-core minified NLS output uses \uXXXX unicode escape sequences instead of raw UTF-8 characters, bloating file size.

## Best solution

Replace plain `JSON.stringify(allMessages)` in the nls.messages.js emission paths (build/lib/nls.ts:71 and build/next/nls-plugin.ts:141) with a variant that preserves raw UTF-8, or use a post-processing step to decode the escapes — the file is served with a UTF-8 charset anyway.
