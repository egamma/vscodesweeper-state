# Issue #225579 — Illegal value for lineNumber

[https://github.com/microsoft/vscode/issues/225579](https://github.com/microsoft/vscode/issues/225579)

## Summary

Still reproduces on main: when a notebook cell's editor model changes (setModel), the ghost text derived can read a stale ghostText.lineNumber against the new (smaller) model, throwing 'Illegal value for lineNumber'. No guard exists in the current code.

## Change summary

Ghost text widget throws 'Illegal value for lineNumber' when the editor model changes while inline completions data references a now-invalid line number.

## Best solution

Add a validation guard in the `_state` derived (ghostTextView.ts ~line 241) that checks `ghostText.lineNumber <= textModel.getLineCount()` and returns `undefined` when out of range, similar to how `_isDisposed` is already guarded. Also guard `computeGhostTextViewData` at line 406.
