# Issue #303549 — GitHub Copilot 1.112 severely reduces usefulness of context window meter.

[https://github.com/microsoft/vscode/issues/303549](https://github.com/microsoft/vscode/issues/303549)

## Summary

The context window meter was deliberately changed (commit f3680f6a81e) to show only actual token usage, with the output buffer rendered separately in the details panel. The reporter wants the meter to include the reserved-for-response portion so it better predicts when compaction will occur. This is a design-feedback feature request, not a bug — the current behavior is intentional per the code comment "Pie chart shows actual usage percentage only."

## Change summary

Include the 'reserved for response' output buffer in the context window meter's percentage so compaction doesn't occur well before the meter shows full.

## Best solution

Consider adding a combined 'effective usage' mode or color threshold that accounts for the output buffer, so the meter's warning/error states fire before compaction rather than after. The relevant code is in `src/vs/workbench/contrib/chat/browser/widgetHosts/viewPane/chatContextUsageWidget.ts` (line ~437 percentage calculation and lines ~468-474 warning thresholds). This needs a product/UX decision on whether the meter should reflect prompt-only usage or effective capacity remaining.
