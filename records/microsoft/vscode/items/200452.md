# Issue #200452 — Aux window workspace file explorer selects wrong file

[https://github.com/microsoft/vscode/issues/200452](https://github.com/microsoft/vscode/issues/200452)

## Summary

The explorer's auto-reveal feature intentionally scrolls the tree to track the active editor; when switching between windows, the resulting scroll can shift the click target. Two team members (bpasero and lramos15) identified this as inherent to auto-reveal behavior with `explorer.autoReveal` as the existing escape hatch. Proposing as-designed.

## Change summary

Clicking a file in the explorer tree while using auxiliary windows causes the tree to scroll mid-click due to auto-reveal, selecting the wrong file.

## Best solution

Keep the current behavior — auto-reveal is working as intended. Users who find this disruptive in multi-window setups can set `explorer.autoReveal` to `false` or `focusNoScroll`. A smarter fix (e.g., suppressing auto-reveal when the explorer tree itself is the click target) would be complex and the scenario is niche.
