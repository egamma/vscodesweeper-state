# Issue #102857 — Git - Bulk merge conflict resolving issue on large files

[https://github.com/microsoft/vscode/issues/102857](https://github.com/microsoft/vscode/issues/102857)

## Summary

Valid feature request for performant bulk merge-conflict resolution on large files. The current approach applies text edits via the editor, which doesn't scale. A git-native approach (e.g. `git checkout --theirs/--ours`) would bypass editor limitations. Only 2 upvotes after 5 years, but the gap remains real — keeping open in the Backlog.

## Change summary

Bulk 'Accept All Incoming/Current' merge conflict resolution fails or stalls on large files because it uses editor text edits instead of a git-native path.

## Best solution

For large files (or when invoked from the SCM view on resources), use `git checkout --theirs`/`--ours` instead of parsing conflict markers and applying text edits through the editor. This would live in `extensions/git/src/` as new commands, complementing the existing `extensions/merge-conflict/src/commandHandler.ts` approach. The `acceptAllResources` path at line 275 of `commandHandler.ts` could also delegate to git when the file exceeds a size threshold.
