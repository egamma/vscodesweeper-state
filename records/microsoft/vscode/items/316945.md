# Issue #316945 — powershell tool assumes PowerShell 6+ (pwsh.exe) — fails silently on machines with only Windows PowerShell 5.1

[https://github.com/microsoft/vscode/issues/316945](https://github.com/microsoft/vscode/issues/316945)

## Summary

The agent host's shell resolution code (both at 1.120.0 and current main) enumerates PowerShell installations via getFirstAvailablePowerShellInstallation() which falls back to Windows PowerShell 5.1 (powershell.exe) when pwsh.exe isn't installed. The reported error suggests a different code path or the AI model itself generating pwsh.exe commands — needs clarification to reproduce.

## Change summary

The powershell tool in Copilot agent mode fails with 'pwsh.exe is not recognized' on machines with only Windows PowerShell 5.1 installed.

## Best solution

Clarify whether the error comes from the VS Code agent host spawning the shell (unlikely given the fallback logic in src/vs/base/node/powershell.ts) or from the model generating commands that invoke pwsh.exe as a subprocess within the terminal. If the latter, it's a model-prompting issue in the PowerShell tool description, which already handles this case on main (isWindowsPowerShell detection in copilotShellTools.ts).
