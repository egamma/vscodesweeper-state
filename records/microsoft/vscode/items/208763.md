# Issue #208763 — Request json/sort failed with message: Range#create

[https://github.com/microsoft/vscode/issues/208763](https://github.com/microsoft/vscode/issues/208763)

## Summary

Still reproduces on current main — the bug lives in vscode-json-languageservice's sort implementation (installed v5.4.4), which mishandles trailing commas in nested objects. The upstream fix has not landed yet (microsoft/vscode-json-languageservice#266 is still open).

## Change summary

JSON sort command crashes with 'Cannot read properties of undefined (reading addChildProperty)' when the document contains trailing commas inside nested objects.

## Best solution

Fix the sort scanner logic in microsoft/vscode-json-languageservice (the CommaToken handler navigates `currentProperty` to `parent` after a trailing comma in a nested object, leaving it undefined for the next token). Once fixed upstream, bump the dependency in extensions/json-language-features/server/package.json.
