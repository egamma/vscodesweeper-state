# Issue #177905 — Cannot drag file from one window to another

[https://github.com/microsoft/vscode/issues/177905](https://github.com/microsoft/vscode/issues/177905)

## Summary

Still reproducible on main: the null error from `webkitGetAsEntry()` returning `null` is unguarded in `BrowserFileUpload.doUpload()`. The cross-window drag limitation is acknowledged but the error dialog is a real bug.

## Change summary

Dragging a file from one vscode.dev window to another (or from an OS file manager to a remote-connected vscode.dev) produces a 'Cannot read properties of null (reading name)' error dialog.

## Best solution

Add a null guard when collecting entries from `webkitGetAsEntry()` in `src/vs/workbench/contrib/files/browser/fileImportExport.ts` (line 142) — filter out null entries so the error dialog is suppressed. The cross-window drag feature itself may remain unsupported, but the null crash should not surface to users.
