# Issue #133229 — Webview editors are not registered editors?

[https://github.com/microsoft/vscode/issues/133229](https://github.com/microsoft/vscode/issues/133229)

## Summary

Still valid debt on current main: webview editors remain unregistered in the IEditorResolverService, and the hardcoded AUTO_LOCK_EXTRA_EDITORS workaround in editorConfiguration.ts is still present. The issue is authored by a CONTRIBUTOR (not MEMBER), but the debt is real and unresolved.

## Change summary

Webview editors (e.g. Markdown Preview, Simple Browser) are not registered in the IEditorResolverService, requiring a hardcoded workaround list for auto-lock and editor configuration features.

## Best solution

Register webview editor types in the IEditorResolverService (possibly as lightweight, non-pickable entries), either on first use or by caching known view types per extension. This would eliminate the hardcoded AUTO_LOCK_EXTRA_EDITORS list in src/vs/workbench/browser/parts/editor/editorConfiguration.ts and allow new webview-based editors to automatically appear in configuration surfaces.
