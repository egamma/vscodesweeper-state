# Issue #176192 — XWayland is still required to run as a native Wayland client

## Summary

This is a confirmed upstream issue in Electron/Chromium's Ozone Wayland backend. The linked Electron issue (electron/electron#39732) was closed as stale — not fixed — so the root cause may still exist. VS Code is now on Electron 42; the issue's current reproducibility on that version is unknown. Keeping open as an upstream tracker.

VS Code running in native Wayland mode still launches and depends on XWayland, crashing if XWayland is killed.

If it's resolved in current Electron, this can be closed as completed.
