# Issue #304880 — Individual project theming

[https://github.com/microsoft/vscode/issues/304880](https://github.com/microsoft/vscode/issues/304880)

## Summary

Feature request for a local-only, per-workspace theming scope that doesn't require committing settings to the repo or switching profiles. Partially addressed by extensions like Peacock and existing `.vscode/settings.json`, but the user's core ask (a non-committed personal workspace scope) remains unmet.

## Change summary

Allow users to set a per-project color theme locally without committing workspace settings to version control or switching profiles.

## Best solution

This is largely addressed by the Peacock extension today (which stores settings in `.vscode/settings.json` or user settings). The deeper ask — a new 'local workspace settings' scope that sits between user and workspace settings without being committed — is a broader settings architecture request. Classify as `feature-request` with area `themes` and let community traction determine priority.
