# Issue #304880 — Individual project theming

[https://github.com/microsoft/vscode/issues/304880](https://github.com/microsoft/vscode/issues/304880)

## Summary

Feature request for per-project theming without committing to repo or switching profiles. The Peacock extension partially addresses this, and the deeper ask (a new non-committed folder-level settings scope) is a significant architectural request. Keeping open as under-discussion with 0 upvotes.

## Change summary

Allow different color themes per project without committing settings to the repo or switching profiles.

## Best solution

The core request has two layers: (1) the surface-level 'pick a theme per window' UX is already served by the Peacock extension and workspace `workbench.colorCustomizations`; (2) the deeper ask — a new non-repo-committed, per-folder settings scope — is a broader settings architecture feature. If the team sees value in a local-only workspace settings layer, it belongs in the settings/configuration area. Otherwise this remains well-served by extensions like Peacock.
