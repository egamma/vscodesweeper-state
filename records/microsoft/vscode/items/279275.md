# Issue #279275 — Inconsistent Sidebar Width Reset Behavior

[https://github.com/microsoft/vscode/issues/279275](https://github.com/microsoft/vscode/issues/279275)

## Summary

The inconsistent reset widths are by design: each sidebar view defines its own `getOptimalWidth()` (Explorer computes from labels, SCM/Extensions return 400px), and the sidebar's `preferredWidth` delegates to the active view. Whether to unify these is a design decision, not a bug fix.

## Change summary

Double-clicking the sidebar sash resets to different widths depending on which panel is active (Explorer ~300px vs SCM/Extensions 400px).

## Best solution

If the team wants consistent reset width, introduce a shared constant or setting that overrides per-view `getOptimalWidth()` for the double-click-reset case in `sidebarPart.ts`. Alternatively, clarify this as intentional (each view picks its own optimal width) and close as `*as-designed`. The `under-discussion` label already on this issue is appropriate until a design decision is made.
