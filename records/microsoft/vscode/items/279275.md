# Issue #279275 — Inconsistent Sidebar Width Reset Behavior

[https://github.com/microsoft/vscode/issues/279275](https://github.com/microsoft/vscode/issues/279275)

## Summary

The inconsistent reset widths are intentional: each panel defines its own optimal width via `getOptimalWidth()` (Explorer = dynamic/file-based, SCM = 400, Extensions = 400), and the sidebar's `preferredWidth` delegates to the active panel. This is by design to give each panel its appropriate content width.

## Change summary

Double-clicking the sidebar sash resets to different widths depending on which panel is active (Explorer ~300px vs SCM/Extensions 400px).

## Best solution

Keep the current design where each view container reports its own optimal width. The sidebar's `preferredWidth` (in `sidebarPart.ts:52`) intentionally delegates to `viewlet.getOptimalWidth()` so each panel gets its content-appropriate reset width. A uniform reset would degrade the experience for content-rich panels.
