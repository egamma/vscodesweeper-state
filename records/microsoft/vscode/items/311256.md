# Issue #311256 — Archiving and unarchiving session breaks it

[https://github.com/microsoft/vscode/issues/311256](https://github.com/microsoft/vscode/issues/311256)

## Summary

Team-filed bug: archiving an active session (via the 'done' button) and then restoring it loses the worktree and leaves the agent stuck. The archive/unarchive code in copilotChatSessionsProvider.ts toggles the `archived` flag but does not appear to handle worktree preservation or re-creation on restore. Keeping open per team-author policy.

## Change summary

Archiving and then unarchiving (restoring) an actively running session in the agents window destroys the worktree and leaves the agent in a broken state.

## Best solution

The unarchive path in copilotChatSessionsProvider.ts needs to either preserve the worktree across archive/unarchive or re-create it on restore; additionally, archiving an actively running session should either warn or gracefully stop the agent before tearing down the worktree.
