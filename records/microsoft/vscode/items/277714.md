# Issue #277714 — Global search doesn't always look in open editors anymore

[https://github.com/microsoft/vscode/issues/277714](https://github.com/microsoft/vscode/issues/277714)

## Summary

The report describes a plausible edge case where global search fails to find matches in out-of-workspace open editors unless their tab is focused or results also exist in workspace files. The extraFileResources mechanism that should handle this is still in place on current main, but the described conditional behavior suggests a real gap — possibly related to model loading state or result deduplication logic. No fix found; keeping open for investigation.

## Change summary

Global search does not consistently include out-of-workspace open editors in results unless their tab is focused or workspace files also contain the search term.

## Best solution

Investigate the interaction between `getOpenEditorResults` (which searches only loaded text models) and the `extraFileResources` disk-based search path in `src/vs/workbench/services/search/common/searchService.ts`. The sync open-editor path may be marking unfocused editors as 'null' (no match) because no model is loaded, while simultaneously the deduplication at line 144 (`!openEditorResults.results.has(progress.resource)`) suppresses async results for those same resources. The fix likely belongs in the search service's result-merging logic.
