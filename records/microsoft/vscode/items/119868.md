# Issue #119868 — Npm Scripts view not available if package.json located only in nested folder

[https://github.com/microsoft/vscode/issues/119868](https://github.com/microsoft/vscode/issues/119868)

## Summary

Valid feature request with 32 upvotes, promoted to Backlog. The npm extension still activates only on root-level `workspaceContains:package.json`; nested-only workspaces remain unserved. No implementation has landed.

## Change summary

NPM Scripts view should be discoverable and activatable when package.json exists only in nested folders, not just at the workspace root.

## Best solution

Change the npm extension's activation to use a glob pattern (`workspaceContains:**/package.json`) or add a user setting (e.g. `npm.packageJsonPath`) that forces activation and deep search. The previously-blocking issue #55299 was closed as out-of-scope, but the glob-based activation approach doesn't depend on it — it just needs acceptance of the search cost tradeoff, possibly behind a setting.
