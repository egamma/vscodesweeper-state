# Issue #251048 — Switching back and forth between windows and apps brings focus to wrong window

[https://github.com/microsoft/vscode/issues/251048](https://github.com/microsoft/vscode/issues/251048)

## Summary

macOS-specific window focus issue: right-clicking a non-frontmost VS Code window while returning from another app raises the wrong window. Likely rooted in Electron/macOS app activation ordering. Needs area routing to the window management team and potentially an upstream Electron investigation.

## Change summary

Right-clicking a VS Code window after switching from another app brings the previously-focused VS Code window to the foreground instead of the clicked one.

## Best solution

Investigate whether the macOS `app.on('activate')` event (src/vs/code/electron-main/app.ts:471) or Electron's BrowserWindow focus/ordering interacts badly with the right-click context menu flow. A related issue (https://github.com/microsoft/vscode/issues/292877) describes the same symptom — right-clicking an unfocused window fails to give it proper focus. These may share a root cause in how Electron handles window z-ordering on macOS reactivation. If the fix belongs in Electron, classify as upstream and file an Electron issue.
