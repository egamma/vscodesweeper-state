# Issue #324316 — Listener leak: AgentHostSessionAdapter subscribes to shared rootState.onDidChange per session

[https://github.com/microsoft/vscode/issues/324316](https://github.com/microsoft/vscode/issues/324316)

## Summary

Still unfixed on main — the per-session subscription pattern on the shared rootState emitter remains at line 724 of baseAgentHostSessionsProvider.ts. Author is a team member, so this stays open for maintainer judgment.

## Change summary

Each AgentHostSessionAdapter subscribes independently to a shared rootState emitter, causing O(N) listener fan-out and a leak warning at ~175 sessions.

## Best solution

Hoist a single shared root-state observable to the provider or connection level (rebuilt on connection swap) so the emitter gets one listener instead of N; the existing `observableFromSubscription` helper or a provider-scoped `observableFromEvent` would work.
