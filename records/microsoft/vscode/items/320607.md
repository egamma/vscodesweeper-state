# Issue #320607 — VS Code does not respond when I save a file

[https://github.com/microsoft/vscode/issues/320607](https://github.com/microsoft/vscode/issues/320607)

## Summary

The delay on save is caused by five extensions providing code actions on save (SonarQube, Amazon Q, ESLint, Auto Import, GitHub Copilot Chat). VS Code is working as designed — it shows a progress message while waiting for extensions to complete. The user can configure `editor.codeActionsOnSave` to disable unneeded actions or adjust `editor.codeActionsOnSaveTimeout` to limit wait time.

## Change summary

VS Code becomes unresponsive during save while waiting for code actions from multiple installed extensions.

## Best solution

The user should configure `editor.codeActionsOnSave` to disable code actions they don't need on every save, or reduce `editor.codeActionsOnSaveTimeout` to limit how long VS Code waits. This is an extension configuration issue, not a VS Code core defect.
