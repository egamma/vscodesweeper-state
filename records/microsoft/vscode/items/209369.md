# Issue #209369 — Disassembly window cannot be themed.

[https://github.com/microsoft/vscode/issues/209369](https://github.com/microsoft/vscode/issues/209369)

## Summary

Still valid: the disassembly view on current main has no syntax highlighting or assembly-language theming — it renders plain white text. The request has 21 upvotes and was promoted to Backlog; it should stay open.

## Change summary

Add syntax highlighting and theming support to the built-in disassembly view so assembly instructions can be colored by token type.

## Best solution

Add tokenization or a lightweight syntax-coloring pass to `src/vs/workbench/contrib/debug/browser/disassemblyView.ts` so that mnemonics, registers, immediates, and addresses render with distinct theme colors. Ideally leverage a TextMate grammar (e.g. a generic assembly grammar) or register custom color tokens so the view responds to the active color theme. The commenter's screenshot shows a clear before/after of the value.
