# Issue #295523 — Unresponsive window with problem matcher errors

[https://github.com/microsoft/vscode/issues/295523](https://github.com/microsoft/vscode/issues/295523)

## Summary

Team-authored issue with `important` label — must stay open. The specific regex trigger was fixed (PR #296295), and trace logging was added (#295961), but the broader systemic issue remains: the task framework has no guardrails against slow/pathological regex patterns causing window freezes. The reporter's last comment (Feb 26) confirms it still occurs and requests better log-level granularity.

## Change summary

Problem matcher regex execution in the task framework can make the renderer unresponsive for several seconds.

## Best solution

The task framework should detect slow regex patterns at runtime (e.g. measure per-match time, skip or abort patterns exceeding a threshold) rather than relying on well-written regexes. This belongs in `src/vs/workbench/contrib/tasks/common/problemMatcher.ts` — add a timeout/budget mechanism around `RegExp.exec` calls in the line-processing loop, and promote the slow-regex logging from trace to info/warn level so it's useful without full trace mode.
