# Issue #308438 — subagent description leaking instructions?

[https://github.com/microsoft/vscode/issues/308438](https://github.com/microsoft/vscode/issues/308438)

## Summary

Team-authored bug report (hard keep-open). The subagent content part renders whatever `description` string arrives in `toolSpecificData` without sanitizing or truncating instruction-like text — if the upstream CLI sends an internal system prompt as the description, it leaks into the UI title.

## Change summary

Subagent descriptions in the chat UI display raw upstream instruction text instead of a clean user-facing summary.

## Best solution

Sanitize or validate the `description` field in `ChatSubagentContentPart.extractSubagentInfo()` (src/vs/workbench/contrib/chat/browser/widget/chatContentParts/chatSubagentContentPart.ts) — either truncate overly long descriptions, strip instruction-like content, or fall back to the default description when the upstream value looks like a system prompt rather than a human-readable label. Coordinate with the CLI team on what shape the description field should have.
