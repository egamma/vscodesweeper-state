# Issue #196009 — DebugAdapterTracker not working in web extensions

[https://github.com/microsoft/vscode/issues/196009](https://github.com/microsoft/vscode/issues/196009)

## Summary

Still valid on current main: DebugAdapterTracker factories are local to the extension host instance that owns the debug adapter, so web extensions (or any extension in a different EH) never get their trackers called. Maintainers have confirmed this and #233184 was closed as a duplicate of this issue, making this the canonical tracker. The fix requires significant architectural changes to how debug adapters interact across extension hosts.

## Change summary

DebugAdapterTrackerFactory registered by a web extension is never invoked because tracker factories are stored locally in the extension host and only consulted in the same EH where the debug adapter runs.

## Best solution

Make debug adapter tracker factories work across extension hosts. This requires plumbing tracker registrations through the main thread debug service so that tracker factories in any extension host (including web) can observe debug adapter messages from adapters running in a different host. The main files involved are `src/vs/workbench/api/common/extHostDebugService.ts` (tracker registration and invocation) and the main-thread debug service proxy. Maintainers have noted this needs 'big changes to how debug adapters work in the EH.'
