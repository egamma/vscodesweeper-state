# Issue #204841 — saving the .code-workspace resets the currently selected launch config from that file

[https://github.com/microsoft/vscode/issues/204841](https://github.com/microsoft/vscode/issues/204841)

## Summary

Still reproduces on main: when `.code-workspace` is saved, `onDidChangeConfiguration('launch')` calls `selectConfiguration(undefined)` which resolves the launch via the last active workspace root (a folder), discarding the workspace-level launch selection and resetting to the first folder-level config.

## Change summary

Saving the .code-workspace file resets the currently selected debug launch configuration when that configuration comes from the workspace file rather than a folder-level launch.json.

## Best solution

In `debugConfigurationManager.ts` line ~348, the `onDidChangeConfiguration` handler should preserve the current `selectedLaunch` and `selectedName` when they are still valid after the config change, rather than unconditionally calling `selectConfiguration(undefined)`. The comment on line 350 already states this intent ('do not change the selection') but the code doesn't implement it.
