# Issue #151319 — [html] `html.autoCreateQuotes` is not instant

[https://github.com/microsoft/vscode/issues/151319](https://github.com/microsoft/vscode/issues/151319)

## Summary

The 100ms debounce delay in autoCreateQuotes is still present on current main (line 95 of autoInsertion.ts). The request to make it instant or configurable remains valid and unaddressed.

## Change summary

The html.autoCreateQuotes feature has a hardcoded 100ms delay that causes quote insertion to be missed when typing quickly.

## Best solution

Either remove the 100ms timeout in `extensions/html-language-features/client/src/autoInsertion.ts` (making auto-quote insertion instant like bracket pairs), or add a configurable delay setting. The timeout was originally added to allow cancellation if the user keeps typing, but auto-close brackets don't need this — parity would improve UX for fast typists.
