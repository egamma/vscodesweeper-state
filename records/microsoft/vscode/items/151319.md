# Issue #151319 — [html] `html.autoCreateQuotes` is not instant

[https://github.com/microsoft/vscode/issues/151319](https://github.com/microsoft/vscode/issues/151319)

## Summary

The 100ms debounce in `autoInsertion.ts` (line 95) is still present on main. This is a feature request to remove or reduce it so fast typists aren't interrupted. Zero upvotes and low traction, but still a valid request parked in Backlog.

## Change summary

Typing `=` inside an HTML tag attribute triggers a 100ms debounce before auto-inserting quotes, causing missed insertions for fast typists.

## Best solution

Remove or significantly reduce the 100ms timeout in `extensions/html-language-features/client/src/autoInsertion.ts` (line 76/95), or switch to a version-check approach that cancels only if the document changed in the interim rather than using a fixed delay. Alternatively, expose a setting to configure the delay.
