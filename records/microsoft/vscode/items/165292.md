# Issue #165292 — Incorrect errors shown when launching in remote container (Codespaces)

## Summary

Team-authored issue (MEMBER) — must stay open for maintainer judgment. The race condition between extension activation and task validation in Codespaces may still exist; no fixing commit was found on main.

## Change summary

VS Code shows false-positive task-type validation errors and spurious extension recommendations when extensions specified in devcontainer.json haven't finished installing during Codespaces startup.

## Best solution

The task validation in `src/vs/workbench/contrib/tasks/` should defer validation until remote-installed extensions have activated, or suppress unrecognized-task-type errors during the initial extension activation window. The related issue #168494 was closed as a duplicate of this one, confirming this is the canonical tracker.
