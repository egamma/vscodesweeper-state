[https://github.com/microsoft/vscode/issues/203014](https://github.com/microsoft/vscode/issues/203014)

## Summary

The request is to scope the 'Search Feature Requests' URL to include `label:vscode.dev` when running on vscode.dev, so users land on web-relevant feature requests. The URL comes from `productService.requestFeatureUrl` — a product configuration value set outside this repo (in the proprietary vscode.dev product config). No code change has been made, and the request remains valid. Keeping open for a maintainer to decide whether to adjust the vscode.dev product configuration.

## Change summary

When running on vscode.dev, the Help > Search Feature Requests link should filter GitHub issues to include `label:vscode.dev` so results are scoped to web-relevant requests.

## Best solution

Update the `requestFeatureUrl` in the vscode.dev product configuration to append `+label%3Avscode.dev` to the GitHub search query. Alternatively, the action in `src/vs/workbench/browser/actions/helpActions.ts` could be enhanced to dynamically append the vscode.dev label when `isWeb` is true, keeping the product config simple.
