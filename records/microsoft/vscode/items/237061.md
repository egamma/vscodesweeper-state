## Change summary

Support `--enable-wayland-ime` (or generically `--enable-features`/`--disable-features`) in `~/.vscode/argv.json` so Wayland IME users can configure the flag without shell aliases.

## Best solution

Add `enable-features` and `disable-features` to the `SUPPORTED_ELECTRON_SWITCHES` array in `src/main.ts` (around line 224), which would generically solve this request and future Chromium feature-flag needs. The maintainer discussion in comment 3 already converged on this approach. If the narrower fix is preferred, add `enable-wayland-ime` to the same array gated on `process.platform === 'linux'`.
