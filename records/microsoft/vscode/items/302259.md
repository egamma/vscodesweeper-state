# Issue #302259 — Confusing UX when you accidentally add args to `/yolo`

[https://github.com/microsoft/vscode/issues/302259](https://github.com/microsoft/vscode/issues/302259)

## Summary

Team-authored polish issue, still unfixed on main. The `executeImmediately`/`silent` slash commands like `/yolo` and `/models` don't prevent trailing text and give no visual feedback when typed manually with arguments, leading to a confusing dead-end UX.

## Change summary

Typing `/yolo say hi` and pressing Enter produces no visible response, confusing users who accidentally add arguments to argument-less slash commands.

## Best solution

Either strip/ignore trailing text after `executeImmediately` commands and still execute them (with a transient notification that args were ignored), or prevent typing after the slash command token in the input editor. The fix likely lives in `src/vs/workbench/contrib/chat/browser/widget/input/editor/chatInputCompletions.ts` (completion insertion) and the submit action path that parses slash commands from manually typed input.
