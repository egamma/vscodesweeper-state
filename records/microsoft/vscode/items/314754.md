# Issue #314754 — Git blame inline annotations do not show properly when refocused in side by side view and the other file isn't from the git repository

[https://github.com/microsoft/vscode/issues/314754](https://github.com/microsoft/vscode/issues/314754)

## Summary

Confirmed on current main: the early-return paths in `_updateTextEditorBlameInformation` (lines 382–383 and 391–393) do not clear the cached blame information, so when focus returns to the tracked editor on the same line, the setter's equality check suppresses the event and decorations are never redrawn.

## Change summary

Git blame inline annotations disappear and don't reappear when refocusing a tracked file on the same line after focusing a non-repository file in side-by-side view.

## Best solution

Clear `textEditorBlameInformation` (set to `undefined`) in the early-return paths at lines 382–383 (no `diffInformation` or not active editor) and 391–393 (no repository) of `extensions/git/src/blame.ts`, so the cached state resets when blame doesn't apply, and the change event fires correctly on refocus.
