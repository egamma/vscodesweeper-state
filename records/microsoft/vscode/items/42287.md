# Issue #42287 — [folding] Allow folding block comment that starts in the middle of the line

[https://github.com/microsoft/vscode/issues/42287](https://github.com/microsoft/vscode/issues/42287)

## Summary

Still unfixed on main. The 'Fold All Block Comments' command uses a regex anchored to line-start (`^\s*` + token), so it only matches block comments that begin a line. The request to also detect mid-line block comment starts remains unaddressed.

## Change summary

The 'Fold All Block Comments' command (Cmd+K Cmd+/) does not detect block comments that start after code on the same line.

## Best solution

Relax the regex in `FoldAllBlockCommentsAction.invoke` (src/vs/editor/contrib/folding/browser/folding.ts, line 880) to also match lines where the block comment token appears after non-whitespace content, or handle the case via the language-aware folding path. Since folding only supports whole lines, the fold range would start at the line containing the mid-line `/*`.
