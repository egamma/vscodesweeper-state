# Issue #295560 — Approval UI needs to be friction-less to use

[https://github.com/microsoft/vscode/issues/295560](https://github.com/microsoft/vscode/issues/295560)

## Summary

Team-authored UX improvement request for the agent tool approval flow. The current UI uses a dropdown (moreActions) for session/workspace-scoped approvals; the request is to show all options inline with dedicated keyboard shortcuts. The approval UI has evolved since filing but the core dropdown pattern persists — keeping open for maintainer judgment.

## Change summary

Show all approval options (Allow Once, Allow in Session, Allow in Workspace) inline with individual keyboard shortcuts instead of behind a dropdown.

## Best solution

Redesign the confirmation widget in `abstractToolConfirmationSubPart.ts` and `chatToolConfirmationSubPart.ts` to render session/workspace/profile-scoped actions as inline buttons with distinct keybindings (e.g. 1/2/3), similar to the question-ask tool pattern, rather than collapsing them behind `moreActions`.
