# Issue #313372 — [Agents] Forked sessions do not use the default approval settings

[https://github.com/microsoft/vscode/issues/313372](https://github.com/microsoft/vscode/issues/313372)

## Summary

Team-authored bug that is still present on main: the `forkChat` method in `baseAgentHostSessionsProvider.ts` does not propagate or seed approval/permission settings from the source session or the user's configured defaults — it only passes `model` and `fork` to `createChat`, leaving the forked chat at the default approval level.

## Change summary

Forking an agent session creates a new tab that ignores the user's configured default approval level (e.g. Bypass Approvals) and does not inherit the originating session's approval settings.

## Best solution

In `baseAgentHostSessionsProvider.ts`, the `forkChat` method should either inherit the source session's approval/permission config or apply the user's configured defaults (via `_initialNewSessionConfig()` or equivalent logic) when creating the forked chat. The fix is localized to the `forkChat` method and the `createChat` call site around line 3061.
