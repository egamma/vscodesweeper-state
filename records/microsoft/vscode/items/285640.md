# Issue #285640 — Terminal suggest doesn't get content from the actual terminal session.

[https://github.com/microsoft/vscode/issues/285640](https://github.com/microsoft/vscode/issues/285640)

## Summary

Valid feature request with 21 upvotes, promoted to Backlog. The gap remains on current main — terminal suggest still uses static completions rather than dynamically fetching session-defined functions. A maintainer (Tyriar) acknowledged the limitation and discussed potential approaches.

## Change summary

Terminal suggest should include functions/commands defined during the current terminal session (e.g. PowerShell functions created via F8 or inline).

## Best solution

Implement a mechanism to periodically or on-demand diff shell-defined functions and feed them into the terminal completion service, as Tyriar suggested. The work would likely touch `src/vs/workbench/contrib/terminalContrib/suggest/browser/terminalCompletionService.ts` and the shell integration scripts.
