# Issue #241245 — Wrong time format when using English language localization

[https://github.com/microsoft/vscode/issues/241245](https://github.com/microsoft/vscode/issues/241245)

## Summary

This is a specific manifestation of the broader internationalization settings gap tracked by the open canonical issue #191094. VS Code delegates time formatting to JavaScript's Intl APIs using the display-language locale, which for en_US defaults to 12-hour time regardless of the OS preference. The canonical issue would absorb this work; proposing a duplicate close.

## Change summary

VS Code displays timestamps in 12-hour format when the display language is English (US) even though macOS is configured for 24-hour time.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/191094, which tracks international format settings in VS Code — implementing OS time-format awareness or a user-facing setting there delivers exactly what this reporter needs.
