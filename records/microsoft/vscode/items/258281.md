# Issue #258281 — *Destructive* shortcut keys override standard text file operations (Ctrl+N rejects changes)

[https://github.com/microsoft/vscode/issues/258281](https://github.com/microsoft/vscode/issues/258281)

## Summary

Still a live issue on current main. PR #274361 scoped Ctrl+N (undoHunk) to require cursor-in-hunk, but Ctrl+Shift+N (undo all edits in file) remains bound with only `EditorContextKeys.focus` — no hunk-proximity guard — so users still lose work. Multiple team members have been personally bitten and the discussion on the right UX approach (removing default keybindings, requiring an explicit review mode, or showing keybinding labels) is ongoing.

## Change summary

Ctrl+N and Ctrl+Shift+N override universally-expected shortcuts (new file / new window) with destructive chat-edit undo actions, causing silent data loss.

## Best solution

The team discussion (comments 6–7, 13, 19) converges on either (a) removing the default keybindings entirely and letting users opt in, or (b) gating them behind an explicit review mode with visible UI indicators. The immediate remaining gap is `Ctrl+Shift+N` ('Undo Chat Edits in this File') at line 200 of `chatEditingEditorActions.ts`, which lacks the `ctxCursorInChangeRange` guard that was added for the per-hunk action and still fires whenever any editor has focus during a chat editing session.
