Still valid: current main only auto-expands the first non-expensive scope (line 111 of variablesView.ts). The request for expanding all block scopes or providing an expand-all option remains unimplemented. Two team members expressed interest in the comments.

## Change summary

Request to auto-expand all (or more) scope groups in the debug variables panel, or provide an expand-all button, instead of only expanding the first non-expensive scope.

## Best solution

Add a setting (e.g. `debug.autoExpandScopes`) to control how many non-expensive scopes are auto-expanded in the variables view, or expand all non-expensive scopes by default. The logic lives in `src/vs/workbench/contrib/debug/browser/variablesView.ts` around line 109–118 where `scopes.find(s => !s.expensive)` would become `scopes.filter(s => !s.expensive)` with a loop. Team members connor4312 and roblourens already expressed interest in revisiting this.
