# Issue #265482 — Copilot Suggestion Struggles With non-ASCII Characters

[https://github.com/microsoft/vscode/issues/265482](https://github.com/microsoft/vscode/issues/265482)

## Summary

Still present on main: the partial-accept (accept-next-word) logic in inlineCompletionsModel.ts uses JavaScript string indices without surrogate-pair awareness, which can corrupt multi-code-unit characters. Issue is on the 'On Deck' milestone — keeping open.

## Change summary

Accepting an inline completion word-by-word corrupts non-ASCII characters that use surrogate pairs or multi-byte sequences.

## Best solution

The `acceptNextWord` path in `src/vs/editor/contrib/inlineCompletions/browser/model/inlineCompletionsModel.ts` needs to compute `acceptUntilIndexExclusive` in a surrogate-pair-safe manner — ensure substring boundaries never land between code units of a multi-code-unit character. The word regex match index and length already handle this for BMP characters, but supplementary-plane characters (surrogate pairs) need an explicit grapheme/code-point boundary check before slicing.
