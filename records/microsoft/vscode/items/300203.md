# Issue #300203 — Prompt rendering overflows

[https://github.com/microsoft/vscode/issues/300203](https://github.com/microsoft/vscode/issues/300203)

## Summary

The reporter is on VS Code 0.39.0 (circa 2016), far too old to diagnose against current behavior. Significant chat rendering and KaTeX overflow CSS changes have landed since. Without a reproduction on a current build, this is not actionable.

## Change summary

User prompt text containing `${}` patterns is parsed as inline LaTeX math, producing a KaTeX-rendered formula that overflows the chat sidebar.

## Best solution

Ask the reporter to reproduce on the latest stable (1.111+). If the overflow persists, the fix would be adding `overflow-x: auto` or `overflow: hidden` to inline `.katex` elements in `src/vs/workbench/contrib/chat/browser/widget/chatContentParts/media/chatMarkdownPart.css`, which currently only guards `.katex-display .katex`.
