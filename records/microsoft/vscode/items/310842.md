# Issue #310842 — Region movement issue

[https://github.com/microsoft/vscode/issues/310842](https://github.com/microsoft/vscode/issues/310842)

## Summary

Confirmed by maintainer aeschli — moving a collapsed/folded region upward with Alt+Up causes lines above to disappear temporarily. No fix has landed on main; the move-lines command doesn't account for folded ranges when shifting up.

## Change summary

Moving a collapsed folding region upward with Alt+Up temporarily hides the lines above it, inconsistent with moving downward.

## Best solution

The move-lines command (`src/vs/editor/contrib/linesOperations/browser/moveLinesCommand.ts`) needs to account for collapsed folding ranges when computing the block to shift upward, mirroring the logic that already works for downward movement. The folding model interaction likely also lives in `src/vs/editor/contrib/folding/`.
