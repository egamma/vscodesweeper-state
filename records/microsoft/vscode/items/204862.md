# Issue #204862 — `window.titleBarStyle` should be `native` by default for SR users

[https://github.com/microsoft/vscode/issues/204862](https://github.com/microsoft/vscode/issues/204862)

## Summary

Team-authored accessibility feature request. The default for `window.titleBarStyle` is still `'custom'` on current main with no screen-reader-aware override; the requested behavior has not been implemented. The technical challenge (detecting SR status before window creation) remains an open design question per comment #5.

## Change summary

Default `window.titleBarStyle` to `native` when the user is a screen reader user, so the native OS menu with accelerator keys is available out of the box.

## Best solution

Implement a startup-time check (e.g. persisted `editor.accessibilitySupport` value from the previous session) to default `window.titleBarStyle` to `native` for SR users, resolving the detection-timing question raised in the thread. The change would live in `src/vs/workbench/electron-browser/desktop.contribution.ts` and possibly the window-creation path.
