# Issue #204862 — `window.titleBarStyle` should be `native` by default for SR users

[https://github.com/microsoft/vscode/issues/204862](https://github.com/microsoft/vscode/issues/204862)

## Summary

Team-authored accessibility feature request to default `window.titleBarStyle` to `native` for screen reader users; still open with unresolved implementation questions about detecting SR users at startup. Keeping open per team-authored rule.

## Change summary

Default `window.titleBarStyle` to `native` when a screen reader is detected, so SR users get accessible native menus with accelerator keys.

## Best solution

Implement startup-time SR detection (via stored `editor.accessibilitySupport` setting or platform accessibility APIs) in `src/vs/workbench/electron-sandbox/desktop.contribution.ts` to conditionally default the title bar style to native. The open question from benibenj about first-launch detection needs a product decision.
