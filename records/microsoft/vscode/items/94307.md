# Issue #94307 — Better support for case sensitive macOS

[https://github.com/microsoft/vscode/issues/94307](https://github.com/microsoft/vscode/issues/94307)

## Summary

Still valid and unfixed on main. The disk file system provider at `diskFileSystemProvider.ts:61` hardcodes `PathCaseSensitive` only for Linux — macOS case-sensitive APFS and Windows per-directory case sensitivity remain unsupported. With 56 upvotes and Backlog milestone, this is the canonical tracking issue for the work.

## Change summary

VS Code assumes macOS and Windows file systems are always case-insensitive, causing the file explorer to merge directories that differ only by case and creating rename/open issues on case-sensitive volumes.

## Best solution

Detect actual file-system case sensitivity at runtime (e.g. by probing the volume or reading OS flags) instead of relying solely on `isLinux` in `src/vs/platform/files/node/diskFileSystemProvider.ts`. A user/workspace setting override (as suggested in the thread) would be a pragmatic first step. The fix is cross-cutting — every `isLinux`-as-proxy-for-case-sensitivity check across the codebase needs auditing.
