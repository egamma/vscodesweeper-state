# Issue #286505 — Fish integration `__vsc_nonce` is broken

[https://github.com/microsoft/vscode/issues/286505](https://github.com/microsoft/vscode/issues/286505)

## Summary

Verified duplicate of #183972 — both report the same root cause: `set -l __vsc_nonce` in the fish shell integration script scopes the variable locally to the `if` block, making it unavailable when referenced later in functions.

## Change summary

The `__vsc_nonce` variable in fish shell integration is set with local scope (`set -l`) inside an `if` block, so it's empty when used later in `__vsc_cmd_executed` and `__vsc_cmd_clear`.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/183972 which is already confirmed and labeled `help wanted`. The fix is changing `set -l` to `set -g` on line 106 of `shellIntegration.fish`.
