# Issue #195768 — Terminal profile default discovery isn't working as expected

[https://github.com/microsoft/vscode/issues/195768](https://github.com/microsoft/vscode/issues/195768)

## Summary

Still reproduces on main: when `defaultProfileName` is null (default setting), no fallback to `$SHELL` exists to mark the system default shell as the default profile. The bug Tyriar diagnosed in December 2023 remains unfixed.

## Change summary

Terminal profile discovery does not use the system's `$SHELL` environment variable to determine which auto-detected profile should be marked as default when the user hasn't explicitly configured `defaultProfile.osx`.

## Best solution

In `detectAvailableUnixProfiles` (or `validateProfilePaths`), when `defaultProfileName` is falsy, fall back to comparing each profile's path (or `basename(path)`) against `shellEnv.SHELL` to set `isDefault: true`. The fix is localized to `src/vs/platform/terminal/node/terminalProfiles.ts`.
