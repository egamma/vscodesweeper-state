# Issue #134785 — Git - merge-conflict cannot resolve binary files

[https://github.com/microsoft/vscode/issues/134785](https://github.com/microsoft/vscode/issues/134785)

## Summary

Still reproduces on main. The merge-conflict extension and the git extension's SCM context menu actions for resolving merge conflicts rely entirely on text-based conflict markers (`<<<<<<<`/`=======`/`>>>>>>>`), which git never inserts into binary files. Accepting current/incoming on a binary conflict silently does nothing.

## Change summary

Right-clicking a binary file in the Merge Changes section and choosing 'Accept All Incoming' or 'Accept All Current' silently fails because the merge-conflict resolver cannot find text-based conflict markers in binary files.

## Best solution

For binary merge conflicts (files in the merge group that lack text conflict markers), the resolution should fall back to `git checkout --ours <path>` or `git checkout --theirs <path>` instead of attempting text-marker-based replacement. This likely requires changes in `extensions/merge-conflict/src/commandHandler.ts` (the `acceptAllResources` path) and possibly a new code path in `extensions/git/src/commands.ts` that detects binary conflicts and delegates to the git CLI.
