# Issue #211299 — a11y issue on tooltips

[https://github.com/microsoft/vscode/issues/211299](https://github.com/microsoft/vscode/issues/211299)

## Summary

Valid accessibility/UX request: tooltips placed below elements get obscured by the mouse cursor, especially with larger cursors. The specific testCoverageBars case still uses 'element' placement, and the broader problem persists across many workbench icon buttons. Issue #265015 tracks the same broader concern as a bug.

## Change summary

Tooltips on icon buttons appear below the mouse cursor, obscuring tooltip text — especially problematic with larger cursors or on high-DPI displays.

## Best solution

Consider whether #265015 (which is labeled `bug` and describes the same core problem more broadly) should absorb this issue, or whether this should remain as the original feature-request tracking the desired behavior change. The fix involves either changing the default hover placement for icon buttons to 'mouse' or adding cursor-aware positioning logic in the hover service.
