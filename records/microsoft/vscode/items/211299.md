# Issue #211299 — a11y issue on tooltips

[https://github.com/microsoft/vscode/issues/211299](https://github.com/microsoft/vscode/issues/211299)

## Summary

Still valid: tooltips positioned below the mouse cursor remain an issue on current main. The test coverage bars still use `getDefaultHoverDelegate('element')` which places hovers below the target, and the broader request for smarter hover positioning (avoiding cursor occlusion) has not been addressed.

## Change summary

Tooltips on icon buttons and coverage pills appear below/under the mouse cursor, making them hard to read especially with larger cursors.

## Best solution

Improve the hover positioning logic in `src/vs/platform/hover/browser/hoverService.ts` to account for cursor size and prefer placement that avoids occlusion — or change the default for action-label/icon-button hovers to position above or to the side. The specific coverage bars case in `testCoverageBars.ts` could switch from `'element'` to `'mouse'` placement as a targeted fix, but the broader request is for smarter global positioning.
