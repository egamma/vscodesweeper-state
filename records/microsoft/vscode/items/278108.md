# Issue #278108 — Debugger completion filters out matching items

[https://github.com/microsoft/vscode/issues/278108](https://github.com/microsoft/vscode/issues/278108)

## Summary

Confirmed bug on current main: the filterText/range logic at repl.ts:303 uses a truthy check on `item.start` which fails when start is 0. An open PR (#308014) already targets this fix — keeping open until it merges.

## Change summary

Debug console completion items from DAP adapters are incorrectly filtered out while typing when the adapter omits or sets start/length to 0.

## Best solution

Land the open PR https://github.com/microsoft/vscode/pull/308014 which fixes the falsy check on `item.start` and computes a default overlap length when start/length are omitted, so completions remain visible while typing.
