# Issue #312256 — bug: Custom agents duplicated in agent picker when same name exists in both org-synced and local paths

[https://github.com/microsoft/vscode/issues/312256](https://github.com/microsoft/vscode/issues/312256)

## Summary

The bug is confirmed in current source: agent discovery lacks the name-based deduplication that skill discovery already implements. Agents from different source paths with the same name both appear because chatModes.ts deduplicates by URI, not by name.

## Change summary

Custom agents with the same name from org-synced and local paths both appear in the agent picker instead of applying the documented preference/override order.

## Best solution

Add name-based deduplication with priority ordering to `computeAgentDiscoveryInfo` in `promptsServiceImpl.ts`, mirroring the existing pattern from `computeSkillDiscoveryInfo` (sort by source priority, keep first occurrence of each name). The `chatModes.ts` refresh loop should also deduplicate by agent name rather than only by URI.
