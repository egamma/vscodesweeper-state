# Issue #248992 — error in processing .gitignore file

[https://github.com/microsoft/vscode/issues/248992](https://github.com/microsoft/vscode/issues/248992)

## Summary

Still reproduces on main: the `FilesFilter.isVisible()` method does not skip gitignore checks for workspace root folders, so in a multi-root workspace a `.gitignore` pattern matching the root folder name hides the entire root. Single-root 'Open Folder' is unaffected because the root is never filtered as a tree item.

## Change summary

In a multi-root workspace, a `.gitignore` entry matching the workspace root folder name causes the entire folder to disappear from the explorer.

## Best solution

Guard `FilesFilter.isVisible()` (or `filter()`) to skip gitignore/exclude checks when `stat.isRoot` is true — workspace root folders should never be hidden by gitignore patterns. The fix is a one-line guard in `src/vs/workbench/contrib/files/browser/views/explorerViewer.ts`.

## Evidence

- **source trace:** In `src/vs/workbench/contrib/files/browser/views/explorerViewer.ts`, `FilesFilter.filter()` calls `isVisible()` for every tree item including workspace roots. `isVisible()` at line 1410 calls `this.isIgnored(stat.resource, stat.root.resource, stat.isDirectory)` without checking `stat.isRoot`, so a gitignore pattern like `XYZ_component` (converted to `**/XYZ_component`) matches the root folder path and hides it.
- **single-root unaffected:** In single-root ('Open Folder') mode, the opened folder is the tree root and its children are filtered — the root itself never goes through `filter()`. In multi-root mode, each workspace folder is a child of a virtual root and does go through filtering, triggering the bug.
- **no fix on main:** Checked `git log` on `explorerViewer.ts` and `ignoreFile.ts` — no commit addresses this root-folder filtering issue. The isRoot guard is absent on current main.
