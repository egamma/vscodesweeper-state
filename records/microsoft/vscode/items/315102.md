# Issue #315102 — Copilot Chat rich text copy/paste loses formatting for tables and code blocks

[https://github.com/microsoft/vscode/issues/315102](https://github.com/microsoft/vscode/issues/315102)

## Summary

Valid feature request: the Copy button in Copilot Chat writes only plain Markdown text via `clipboardService.writeText()`, so tables and code blocks lose formatting when pasted into rich-text apps. No rich-text copy path exists on current main.

## Change summary

Request a 'Copy as rich text / HTML' option for Copilot Chat responses so that tables and code blocks paste correctly into apps like Teams, Outlook, and Word.

## Best solution

Add a rich-text copy path (likely in `chatCopyActions.ts`) that renders the Markdown response to HTML and writes it to the clipboard as `text/html` alongside the plain-text Markdown. This would require extending `IClipboardService` with an HTML write method (or using the native clipboard API directly). The chat widget already renders Markdown to HTML for display, so the conversion infrastructure exists — the gap is wiring it into the copy action.
