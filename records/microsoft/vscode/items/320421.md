# Issue #320421 — Issue Re-open: Repo name is not displayed in Source Control sidebar

[https://github.com/microsoft/vscode/issues/320421](https://github.com/microsoft/vscode/issues/320421)

## Summary

Likely regression or incomplete fix for the repo-name-hidden-by-branch-name problem originally addressed in #261388. The current CSS gives the label `min-width: 0` which allows it to shrink to nothing when the panel is narrow, while the responsive toolbar still reserves space for branch-name actions. The reporter supplied version, OS, screenshots, and steps — the `info-needed` label can be removed.

## Change summary

When the Source Control sidebar is narrow, the repository name is pushed out by the branch name and becomes invisible, making it impossible to distinguish repositories in a multi-repo workspace.

## Best solution

Ensure the `.scm-provider > .monaco-icon-label` has a reasonable `min-width` (or a `flex-shrink` lower than the actions) so the repository name remains partially visible even when the panel is narrow, similar to how the original fix in e4954800639 intended. The fix lives in `src/vs/workbench/contrib/scm/browser/media/scm.css` (lines 53–56) and possibly the responsive toolbar configuration in `scmRepositoryRenderer.ts`.
