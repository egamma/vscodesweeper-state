# Issue #138323 — Move telemetry cleansing into shared process

[https://github.com/microsoft/vscode/issues/138323](https://github.com/microsoft/vscode/issues/138323)

## Summary

The telemetry cleansing (cleanData/removePropertiesWithPossibleUserInfo) still runs inside TelemetryService at the call site (including the renderer), not centralized in the shared process. The request remains valid. Additionally, the 'On Deck' milestone is a scheduling decision that requires maintainer judgment — keeping open.

## Change summary

Move telemetry PII cleansing (regex-based scrubbing) from the renderer process into the shared process to avoid impacting startup performance.

## Best solution

Refactor so that cleanData() runs in the shared process telemetry pipeline rather than at each TelemetryService call site. The cleansing regexes and cleanupPatterns would need to be forwarded over IPC alongside the raw telemetry data. Key files: src/vs/platform/telemetry/common/telemetryService.ts (where cleanData is called today) and src/vs/platform/telemetry/common/telemetryUtils.ts (where cleanData/removePropertiesWithPossibleUserInfo live). PR https://github.com/microsoft/vscode/pull/164815 mitigated the cost with a fast-path check but did not relocate the work.
