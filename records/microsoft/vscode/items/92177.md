# Issue #92177 — API for reading current word-wrap state (and toggling it) on a TextDocument

[https://github.com/microsoft/vscode/issues/92177](https://github.com/microsoft/vscode/issues/92177)

## Summary

Current `vscode.d.ts` still does not expose word-wrap state or a word-wrap show/override option to extensions. Since this is a Backlog API feature request with extension-author demand and no confirmed duplicate or rejection evidence, it should stay open for API owner discussion.

## Change summary

Expose extension API to read the effective word-wrap state for a text editor/document and optionally disable or toggle wrapping while an extension feature is active.

## Best solution

If accepted, add a narrow API under `TextEditor`/`TextEditorOptions` or `TextDocumentShowOptions` for effective word-wrap state and a scoped override, backed by the existing editor word-wrap override plumbing; otherwise leave the request for API owner discussion rather than closing it as stale.

## Evidence

- **current labels:** The issue is open with `feature-request` and `api`, and has milestone `Backlog`, which is an unscheduled holding bin rather than a release milestone that blocks evaluation.
- **API surface:** In `src/vscode-dts/vscode.d.ts`, `TextEditorOptions` currently exposes `tabSize`, `indentSize`, `insertSpaces`, `cursorStyle`, and `lineNumbers`, but no `wordWrap` or effective wrapping property.
- **show options:** In `src/vscode-dts/vscode.d.ts`, `TextDocumentShowOptions` currently contains `viewColumn`, `preserveFocus`, `preview`, and `selection`, but no word-wrap override option as suggested in the comment.
- **existing implementation boundary:** Core editor internals have word-wrap override options such as `wordWrapOverride1` and `wordWrapOverride2` in `src/vs/editor/common/config/editorOptions.ts`, but those are not exposed through the extension API.
- **extension demand:** The issue body cites GitLens decoration/blame use cases, and a later comment adds Wallaby.js and Quokka.js as extensions that would use the same API.
