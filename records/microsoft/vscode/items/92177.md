# Issue #92177 — API for reading current word-wrap state (and toggling it) on a TextDocument

[https://github.com/microsoft/vscode/issues/92177](https://github.com/microsoft/vscode/issues/92177)

## Summary

Still a valid open request — no extension API for reading or toggling per-editor word-wrap state exists on current main. Low community traction (1 upvote) but from a notable extension author; keeping open for the community process.

## Change summary

Expose an extension API to read the effective word-wrap state on a text editor and toggle it programmatically.

## Best solution

Add a per-editor word-wrap read/toggle capability to the TextEditor or TextEditorOptions API surface. This would let extensions like GitLens conditionally adjust decorations when word-wrap is active and temporarily disable wrapping for inline blame annotations. Likely touches `src/vscode-dts/vscode.d.ts` (TextEditorOptions) and the ext-host editor options proxy.
