# Issue #129538 — Disassembly view: support keyboard navigation

[https://github.com/microsoft/vscode/issues/129538](https://github.com/microsoft/vscode/issues/129538)

## Summary

Still unimplemented on main: the disassembly view does not set a keyboardNavigationLabelProvider, so type-to-navigate by address is not supported. Every other debug tree view already has one. Keeping open as a valid enhancement.

## Change summary

Add keyboard navigation (type-to-search by address) support to the disassembly view.

## Best solution

Add a `keyboardNavigationLabelProvider` to the disassembly view's tree options in `src/vs/workbench/contrib/debug/browser/disassemblyView.ts`, returning the instruction address as the navigation label — following the pattern already used in variablesView, callStackView, breakpointsView, and other debug trees.
