# Issue #215390 — Cursor appears before decoration on empty lines

[https://github.com/microsoft/vscode/issues/215390](https://github.com/microsoft/vscode/issues/215390)

## Summary

Still reproduces on main: the cursor is placed before `::before` decorations on empty lines because the CharacterMapping length check in `_readPixelOffset` is bypassed when renderViewLine creates a mapping of length 1 instead of 0 for empty decorated lines.

## Change summary

Clicking on an empty line with a `before` decoration places the cursor before the decoration text instead of after it.

## Best solution

In `src/vs/editor/common/viewLayout/viewLineRenderer.ts` around line 403, change the CharacterMapping creation for empty lines with decorations so that `_readPixelOffset` correctly enters the foreign-element handling branch. The reporter's suggested fix (using length 0 and removing the `setColumnInfo` call) needs validation against other consumers of CharacterMapping, but the direction is correct.
