# Issue #215390 — Cursor appears before decoration on empty lines

[https://github.com/microsoft/vscode/issues/215390](https://github.com/microsoft/vscode/issues/215390)

## Summary

This still appears to be a valid editor rendering bug rather than a stale or duplicate cleanup candidate. Current main retains the code shape described in the report, so there is not enough evidence to close it.

## Change summary

Before decorations on empty editor lines cause cursor hit-testing to place the cursor before the decoration instead of after it.

## Best solution

Fix the empty-line before-decoration cursor offset path in the editor view line renderer/hit-testing code, likely around src/vs/editor/common/viewLayout/viewLineRenderer.ts and src/vs/editor/browser/viewParts/viewLines/viewLine.ts, and add a regression test that covers cursor offset behavior for empty lines with before decorations.
