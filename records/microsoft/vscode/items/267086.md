# Issue #267086 — Support cursor rules

[https://github.com/microsoft/vscode/issues/267086](https://github.com/microsoft/vscode/issues/267086)

## Summary

Valid feature request for cross-tool compatibility — VS Code's instruction files system currently only recognizes `.instructions.md` files, not `.mdc` files from `.cursor/rules`. Already labeled and in Backlog; no implementation has landed on main.

## Change summary

Allow `chat.instructionsFilesLocations` to pick up `.mdc` files (Cursor rules format) from configured directories alongside `.instructions.md` files.

## Best solution

Extend the prompt file discovery in `src/vs/workbench/contrib/chat/common/promptSyntax/config/promptFileLocations.ts` to recognize `.mdc` as an additional instruction file extension when scanning configured locations, or document that users can use `.instructions.md` files alongside their `.mdc` files. Alternatively, the broader issue https://github.com/microsoft/vscode/issues/266120 (full support for instruction file formats) may subsume this.
