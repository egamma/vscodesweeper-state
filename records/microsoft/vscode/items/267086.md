# Issue #267086 — Support cursor rules

[https://github.com/microsoft/vscode/issues/267086](https://github.com/microsoft/vscode/issues/267086)

## Summary

Valid feature gap — custom instruction file locations configured via `chat.instructionsFilesLocations` only pick up `*.instructions.md` files, not plain `.md` or `.mdc` files. The underlying `getPromptFileType()` filter rejects them unless they're in well-known folders like `.claude/rules/`. Keep open as an unresolved feature request.

## Change summary

User wants `.md` and `.mdc` files in custom-configured instruction directories (e.g. `.cursor/rules`) to be recognized as instruction files.

## Best solution

Relax the `getPromptFileType()` filter in `src/vs/workbench/contrib/chat/common/promptSyntax/config/promptFileLocations.ts` so that all `.md` (and optionally `.mdc`) files found in user-configured instruction locations are treated as instructions, similar to how `.claude/rules/` already works. This would let teams with cross-tool rule directories share instructions without renaming files.
