# Issue #230705 — Can't create Breakpoint with custom mode from extension

[https://github.com/microsoft/vscode/issues/230705](https://github.com/microsoft/vscode/issues/230705)

## Summary

Valid gap: the extension API passes `mode` but not `modeLabel` when adding breakpoints, so the Breakpoints View cannot render the mode badge. The issue is still present on current main — `mainThreadDebugService.ts` line 222 only extracts `mode` from the DTO, not `modeLabel`.

## Change summary

When creating breakpoints via the extension API with a custom mode, the mode label is not propagated so the Breakpoints View shows no mode badge.

## Best solution

Extend the breakpoint DTO in `extHostDebugService.ts` to include `modeLabel` alongside `mode`, and pass it through in `mainThreadDebugService.ts` when constructing the raw breakpoint data. This likely requires finalizing/extending the proposed breakpoint mode API surface (`SourceBreakpoint.mode` → also expose `modeLabel`).
