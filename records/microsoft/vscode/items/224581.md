# Issue #224581 — [json] colorize string literals based on language information coming from schemas

## Summary

Valid community-promoted feature request with team engagement, 'help wanted' label, and no implementation yet. Should remain open for contribution.

## Change summary

Colorize JSON string literals as embedded languages (e.g. shellscript in package.json scripts) using language information from JSON schemas.

## Best solution

Implement semantic highlighting in the built-in json-language-features extension that reads schema annotations (e.g. `x-intellij-language-injection`) and applies token-based coloring to matched string values. The community extension by RedCMD demonstrates the TextMate injection approach as a reference.
