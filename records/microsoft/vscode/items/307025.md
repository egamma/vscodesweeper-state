# Issue #307025 — plugin marketplace refresh causes error popups

[https://github.com/microsoft/vscode/issues/307025](https://github.com/microsoft/vscode/issues/307025)

## Summary

Still partially reproducible on main: transient git 'cannot lock ref' errors during marketplace refresh still surface as error popups. PR #318270 added recovery actions (purge & reclone) and handles fast-forward divergence, but the ref-lock transient error isn't caught by the retry logic.

## Change summary

Plugin marketplace refresh shows error popups for transient git ref-lock failures that should be handled gracefully with a retry.

## Best solution

Extend `LocalGitService._isFastForwardPullFailure` (or add a sibling transient-error check) to recognize 'cannot lock ref' / 'unable to update local ref' as a retryable condition, so the existing retry-then-hard-reset path handles it silently. Alternatively, add a retry with `--prune` to the `fetch()` path in `localGitService.ts`.
