# Issue #177569 — Git - add support for `git restore`

[https://github.com/microsoft/vscode/issues/177569](https://github.com/microsoft/vscode/issues/177569)

## Summary

The core `git restore` infrastructure exists in the codebase (added for stash operations), but no dedicated 'Git: Restore' command palette command is exposed to users. The existing 'Discard Changes' commands still use `git checkout` under the hood. The feature request for an explicit command remains valid.

## Change summary

Add a 'Git: Restore' command to the command palette that invokes `git restore` to discard local file changes, complementing the existing 'Git: Stage Changes' command.

## Best solution

Wire the existing `repository.restore()` method (which already calls `git restore`) into a new `git.restore` command registered in `extensions/git/src/commands.ts` and `extensions/git/package.json`, giving users an explicit 'Git: Restore' command palette entry. Additionally, consider migrating the existing `clean` method from `git checkout` to `git restore` for tracked files. The `help wanted` label is already applied, making this a good community contribution target.
