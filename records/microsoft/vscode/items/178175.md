# Issue #178175 — Git - pre-push and pre-commit git hooks are handled differently when errors occur

[https://github.com/microsoft/vscode/issues/178175](https://github.com/microsoft/vscode/issues/178175)

## Summary

Still present on main: pre-push hook failures are misclassified as PushRejected, causing VS Code to show the hardcoded 'Try running Pull' message instead of the hook's actual error output. Pre-commit hooks correctly surface hook output via the default error path.

## Change summary

When a pre-push git hook fails, VS Code shows a misleading 'Can't push refs to remote. Try running Pull first' message instead of the hook's actual error output, unlike pre-commit hooks which correctly display hook stderr.

## Best solution

In `extensions/git/src/git.ts` push error handling (~line 2557), distinguish pre-push hook failures from genuine remote rejections (e.g. check stderr for hook-related patterns like `hook declined` or absence of `[rejected]`). Either add a new `GitErrorCodes.PrePushHookRejected` that surfaces the hook output in `commands.ts`, or let hook failures fall through to the default handler that already extracts stderr lines.
