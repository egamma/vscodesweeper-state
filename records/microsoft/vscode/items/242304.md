# Issue #242304 — "Reveal in File Explorer" does not work as intended

[https://github.com/microsoft/vscode/issues/242304](https://github.com/microsoft/vscode/issues/242304)

## Summary

Still reproduces on main: `revealResourcesInOS` in fileCommands.ts calls `showItemInFolder` sequentially for each selected resource via `sequence()`, causing the OS file explorer to open/scroll through each file one by one. A team member engaged but no fix has landed. Keeping open as it's a valid papercut with an acknowledged design question (reveal parent folder vs. first file vs. current behavior).

## Change summary

When revealing multiple selected files in the OS file explorer, VS Code calls `showItemInFolder` sequentially for each file, causing a disorienting scrolling effect instead of revealing them all at once or just the parent folder.

## Best solution

Since Electron's `shell.showItemInFolder` only supports a single path, the fix is to detect multi-selection and either (a) reveal only the parent folder with no file selected, or (b) reveal only the first/last selected file. The decision between these was raised by the team member in the comments and should be resolved before implementing. The fix is localized to `revealResourcesInOS` in `src/vs/workbench/contrib/files/electron-browser/fileCommands.ts`.
