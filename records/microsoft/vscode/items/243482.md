# Issue #243482 — Support re-using HTTP connections with `keepAlive` in Proxy agent

[https://github.com/microsoft/vscode/issues/243482](https://github.com/microsoft/vscode/issues/243482)

## Summary

Still an open issue — the `keepAlive` flag remains commented out/skipped in the current `@vscode/proxy-agent` on main (v0.43.0), and the community fix PR (vscode-proxy-agent#68) is still open and unmerged. A maintainer has engaged with guidance, so this is actively in progress.

## Change summary

HTTP connection reuse via `keepAlive` is globally disabled in the proxy agent, preventing clients from opting in to keep-alive even when their target servers support it.

## Best solution

Review and land https://github.com/microsoft/vscode-proxy-agent/pull/68 (which makes `keepAlive` opt-in from the original agent rather than the global agent), publish a new `@vscode/proxy-agent` version, then bump the dependency in VS Code. The maintainer guidance from chrmarti is already in the comments.
