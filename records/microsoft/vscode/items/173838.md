# Issue #173838 — Exception when moving views around

[https://github.com/microsoft/vscode/issues/173838](https://github.com/microsoft/vscode/issues/173838)

## Summary

Still reproduces on main: the fix PR #198526 was reverted, and the unguarded access to `this.panes[this.panes.length - 1].element` in `getOverlayBounds` still crashes when the panes array is empty after all views are moved out. Milestone is 'On Deck' — keeping open.

## Change summary

Dragging views between containers throws TypeError when the source container's panes array becomes empty.

## Best solution

Guard the `getOverlayBounds` function in `viewPaneContainer.ts` (line ~405) to handle an empty `this.panes` array — either return the full container bounds or bail out of the drag overlay logic when no panes remain. The reverted PR #198526 tried deregistering the container; a simpler approach is a bounds fallback.
