# Issue #173838 — Exception when moving views around

[https://github.com/microsoft/vscode/issues/173838](https://github.com/microsoft/vscode/issues/173838)

## Summary

Still reproduces on main: getOverlayBounds accesses the last pane's element without guarding against an empty panes array, causing a TypeError when all views have been moved out of a container.

## Change summary

Moving all views out of a view container and then dragging views back causes a TypeError because the empty panes array is accessed without a guard.

## Best solution

Guard the `this.panes[this.panes.length - 1]` access in `getOverlayBounds` (viewPaneContainer.ts:405) to handle the empty-panes case — either return a fallback bounding rect from the parent container, or skip registering the drag target when no panes remain. The reverted PR #198526 tried deregistering the container entirely but caused regressions; a simpler null-guard in the bounds computation would be safer.
