# Issue #321918 — Gemma models return reasoning in `<thought>` tags – UI does not format/hide them

[https://github.com/microsoft/vscode/issues/321918](https://github.com/microsoft/vscode/issues/321918)

## Summary

Valid feature request: the Copilot extension's BYOK/custom-endpoint pipeline handles thinking data via structured API fields (reasoning_content, cot_summary, thinking, etc. in thinkingUtils.ts) but does not parse `<thought>` XML tags embedded in text content. Gemma models via Google AI API send reasoning this way, and the extension has no code to recognize it. This is a real gap in the BYOK thinking-tag support.

## Change summary

Gemma models accessed via custom endpoints send reasoning in `<thought>...</thought>` text tags, but the Copilot extension only recognizes structured API thinking fields — the `<thought>` content renders as raw text in chat.

## Best solution

Extend the BYOK streaming pipeline (likely in the OpenAI-compatible endpoint layer or the response-conversion layer in `extensions/copilot/src/platform/networking/common/responseConvert.ts` and `extensions/copilot/src/platform/thinking/common/thinkingUtils.ts`) to recognize `<thought>` tags in text content and emit them as thinking parts, similar to how `<think>` tags are stripped in `xtabNextCursorPredictor.ts`. This would be a Copilot extension change, not a VS Code core change.
