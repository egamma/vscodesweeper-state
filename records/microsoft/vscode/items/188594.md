# Issue #188594 — [themes] support non-HEX formats in color customizations

[https://github.com/microsoft/vscode/issues/188594](https://github.com/microsoft/vscode/issues/188594)

## Summary

Valid feature request; color customizations in settings.json still only accept HEX/HEXA via Color.fromHex(). Not implemented on main, low community traction (2 upvotes in ~3 years). Keeping open per backlog-candidate rules.

## Change summary

Allow non-HEX color formats (HSL, OKLCH, named colors, etc.) in workbench.colorCustomizations and related theme settings.

## Best solution

Extend the color parsing in `src/vs/workbench/services/themes/common/colorThemeData.ts` (and possibly `src/vs/base/common/color.ts`) to accept CSS color strings beyond HEX — at minimum `hsl()`/`hsla()` and named colors. The HSLA class already exists internally; exposing it as an accepted input format would be the smallest lift.
