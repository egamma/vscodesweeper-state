# Issue #213628 — [html] Formatting Breaks with Space Between Commented Lines in Script Tag

[https://github.com/microsoft/vscode/issues/213628](https://github.com/microsoft/vscode/issues/213628)

## Summary

Still present on main — the HTML formatter delegates to vscode-html-languageservice (which uses js-beautify), and the formatting logic in formatting.ts has not changed since this was filed. The bug is in the upstream js-beautify library's handling of script tag content with whitespace-only lines between comments.

## Change summary

HTML Format Document fails silently when a script tag contains commented lines separated by a line with only whitespace.

## Best solution

This is rooted in the js-beautify library used by vscode-html-languageservice. The issue is correctly labeled `help wanted` and in Backlog. The long-term fix is the Volar integration (PR https://github.com/microsoft/vscode/pull/171547) which would replace the js-beautify formatter, or an upstream fix in js-beautify itself.
