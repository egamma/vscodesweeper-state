# Issue #213628 — [html] Formatting Breaks with Space Between Commented Lines in Script Tag

[https://github.com/microsoft/vscode/issues/213628](https://github.com/microsoft/vscode/issues/213628)

## Summary

This is a real bug but the root cause is in js-beautify (v1.15.4), which is vendored into vscode-html-languageservice. The fix needs to happen upstream in js-beautify or in how vscode-html-languageservice consumes it. Keeping open as an upstream issue.

## Change summary

HTML Format Document fails to format when a script tag contains commented lines separated by a whitespace-only line.

## Best solution

The fix belongs in js-beautify (the JavaScript beautifier used by vscode-html-languageservice). Either file/track the bug upstream at https://github.com/beautifier/js-beautify, or work around it in vscode-html-languageservice's htmlFormatter.ts. Once fixed upstream, update the vendored copy in vscode-html-languageservice.
