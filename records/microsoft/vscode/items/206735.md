# Issue #206735 — `Vscode.Window.onDidCloseTerminal` not invoked even terminal is halted in a specific case

[https://github.com/microsoft/vscode/issues/206735](https://github.com/microsoft/vscode/issues/206735)

## Summary

Still reproduces on main: when a terminal editor tab is moved (dragged between editor groups), the `onDisposed` listener is dropped and never re-registered, so `onDidCloseTerminal` never fires to the extension host when the terminal is subsequently closed.

## Change summary

`vscode.window.onDidCloseTerminal` does not fire after a terminal editor tab is moved/dragged to a different editor group and then closed.

## Best solution

In `terminalEditorService.ts`, the `onDidVisibleEditorsChange` handler at lines 88-90 re-adds a moved terminal to `_editorInputs` and `instances` but skips `_registerInstance`, so the `onDisposed` listener is never re-attached. Fix by calling `_registerInstance` (or at minimum re-registering `_instanceDisposables` including the `onDisposed` forwarder) instead of directly pushing to `instances`.

## Evidence

- **missing listener re-registration:** src/vs/workbench/contrib/terminal/browser/terminalEditorService.ts lines 88-90: `onDidVisibleEditorsChange` handler adds moved terminal editors back to `_editorInputs` and `instances` directly, but does NOT call `_registerInstance`, so the `onDisposed` listener (line 189) that forwards to `_onDidDisposeInstance` is never re-attached.
- **listener disposal on move:** src/vs/workbench/contrib/terminal/browser/terminalEditorService.ts lines 96-107 and 196-209: `onDidCloseEditor` calls `_removeInstance`, which disposes `_instanceDisposables` including the `onDisposed` listener. This fires during drag-and-drop before the editor re-opens in the new group.
- **event chain:** The `onDidDisposeInstance` event from the editor service is forwarded by `terminalService.ts` line 349 to `_onDidDisposeInstance`, which `mainThreadTerminalService.ts` line 88 listens to in order to call `$acceptTerminalClosed` on the ext host, ultimately firing `onDidCloseTerminal`.
