## Summary

Valid feature request to show which extension triggered a dialog/quick pick. Not yet implemented on main — mainThreadQuickOpen.ts does not pass extension identity to the quick input UI. Issue #247541 was closed as a duplicate of this one, confirming it's the canonical tracker. Only 1 upvote, so it stays in Backlog Candidates without promotion.

## Change summary

## Best solution

Propagate extension identity from the ext-host API call through to the quick input / dialog UI layer, likely in `src/vs/workbench/api/browser/mainThreadQuickOpen.ts` and the quick input service, so the title or a subtitle indicates which extension opened the prompt.
