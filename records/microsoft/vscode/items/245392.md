# Issue #245392 — Explorer jumps position when focusing from a seperate window in the same workspace

[https://github.com/microsoft/vscode/issues/245392](https://github.com/microsoft/vscode/issues/245392)

## Summary

Still reproducible on main: when two windows share a workspace, focusing the Explorer in one window triggers `selectActiveFile()` and `focus()` → `tree.reveal()`, which scrolls to the other window's active editor position before registering the user's click. No fix has landed.

## Change summary

Explorer scrolls/jumps to the wrong file position when clicking it in a second window of the same workspace, because auto-reveal fires for the other window's active editor.

## Best solution

Guard `selectActiveFile()` and/or the `focus()` reveal path so they only act when the triggering editor change originates from the *same* window. The fix likely belongs in `src/vs/workbench/contrib/files/browser/views/explorerView.ts` — either by checking the window identity in `onDidActiveEditorChange` or by suppressing the reveal in `focus()` when the focus event comes from a cross-window switch.
