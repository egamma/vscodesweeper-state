# Issue #243417 — [Shell Integration Error] bash: vsc_aa_env: bad array subscript

[https://github.com/microsoft/vscode/issues/243417](https://github.com/microsoft/vscode/issues/243417)

## Summary

The underlying bug remains on main — the bash shell integration script still doesn't handle multi-line exported functions from `env` output. Currently masked by `scopedDownShellEnvs` limiting which env vars are reported. A team member explicitly asked to keep this open until a general fix lands.

## Change summary

Bash shell integration errors (`bad array subscript`) when exported multi-line bash functions appear in `env` output.

## Best solution

When `scopedDownShellEnvs` is eventually removed or broadened, the env-parsing logic in `shellIntegration-bash.sh` needs to use `env -0` (NUL-delimited) output or otherwise sanitize multi-line values to avoid treating function body lines as key/value pairs.
