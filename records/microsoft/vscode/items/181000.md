# Issue #181000 — Search & Replace is behaving strange for readonly files

[https://github.com/microsoft/vscode/issues/181000](https://github.com/microsoft/vscode/issues/181000)

## Summary

Still reproduces on main: the Search & Replace path does not check file-level readonly status (from filesystem providers or `files.readonlyInclude`), only notebook webview match-level readonly. The edits are silently applied to the buffer and then lost on close, misleading the user.

## Change summary

Search & Replace applies edits to readonly files without blocking or warning, then the changes are silently lost when the editor is closed and reopened.

## Best solution

The replace service (`src/vs/workbench/contrib/search/browser/replaceService.ts`) and/or the replace actions should check the file-level readonly status (via `IFilesConfigurationService` or the text file model's readonly state) before applying edits. Readonly files should either be skipped entirely or surfaced with a clear warning. The `hasOnlyReadOnlyMatches()` / `hasReadonlyMatches()` methods on `FileMatch` currently only reflect notebook webview readonly — they should also account for file-level readonly.
