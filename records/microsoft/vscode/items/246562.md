# Issue #246562 — Git - improve error handling for publishing an empty repository

[https://github.com/microsoft/vscode/issues/246562](https://github.com/microsoft/vscode/issues/246562)

## Summary

Still reproduces on main — the `publish` command in the git extension does not guard against an empty repository (no commits). The issue has an 'On Deck' milestone and an open community PR, so it should stay open.

## Change summary

Publishing an empty git repository (no commits) fails with an unhelpful error instead of a clear warning.

## Best solution

Review and land the open community PR https://github.com/microsoft/vscode/pull/304796 (or an equivalent fix) that adds a guard in `extensions/git/src/commands.ts` at the `publish` method to detect when `repository.HEAD` has no commit ref and show a user-friendly warning before attempting the push.
