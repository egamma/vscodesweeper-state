# Issue #309556 — MCP stdio server config cached after editing .vscode/mcp.json — restart uses old config

[https://github.com/microsoft/vscode/issues/309556](https://github.com/microsoft/vscode/issues/309556)

## Summary

Still reproducible on main: the multi-layered propagation chain from file watcher → mcpManagementService → mcpWorkbenchService → InstalledMcpServersDiscovery → McpService involves several throttles/queues, so a user clicking 'Restart' shortly after editing .vscode/mcp.json can hit a race where the old definition is still live in the registry.

## Change summary

Restarting an MCP stdio server after editing .vscode/mcp.json uses the stale (pre-edit) command/args until a full window reload.

## Best solution

The restart action (RestartServerAction.run or resolveConnection) should await completion of any pending config-file-change propagation before launching, or McpServer.start should force a synchronous re-read of the current server definitions from the registry. The propagation chain spans src/vs/platform/mcp/common/mcpManagementService.ts (file watcher → reloadConfigurationScheduler), src/vs/workbench/services/mcp/common/mcpWorkbenchManagementService.ts (workspace event relay), src/vs/workbench/contrib/mcp/browser/mcpWorkbenchService.ts (onChange relay), and src/vs/workbench/contrib/mcp/common/discovery/installedMcpServersDiscovery.ts (sync throttler).
