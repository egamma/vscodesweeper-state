# Issue #307385 — Chat: Plan Mode immediately starts implementation when in Autopilot mode

## Summary

The plan mode + autopilot interaction has been substantially rearchitected since this was filed (plan mode now takes precedence on the mode axis, and the agent host surfaces an explicit plan-review UI before implementation begins), but the issue remains under active community discussion with 15 upvotes and no team resolution. Keeping open for a maintainer to confirm whether the current behavior fully addresses the original concern across all harnesses.

## Change summary

When in Autopilot mode with Plan mode selected, Copilot automatically sends 'Start Implementation' instead of letting the user review the plan first.

A team member should verify whether the current agent host plan-review flow (the `exit_plan_mode` RPC with explicit user-facing action picker) and the legacy migration (`plan` taking precedence over `autopilot` on the mode axis) fully resolve this for all harnesses (Copilot CLI, Claude, Codex).
