# Issue #76505 — [json] format on save should remove last trailing comma in JSON with json-language-features

[https://github.com/microsoft/vscode/issues/76505](https://github.com/microsoft/vscode/issues/76505)

## Summary

Still a valid open feature request. The built-in JSON formatter (via vscode-json-languageservice) only reformats syntactically valid documents and does not fix trailing commas. No implementation found on main.

## Change summary

The built-in JSON formatter should remove trailing commas in strict JSON and add missing commas during format-on-save.

## Best solution

Enhance the vscode-json-languageservice formatter (upstream in microsoft/vscode-json-languageservice) to support a 'fix trailing commas' mode for strict JSON documents, or expose a code action from the JSON language server that auto-fixes comma diagnostics. Alternatively, this could remain a Prettier/extension concern since it involves modifying invalid syntax rather than reformatting valid documents.
