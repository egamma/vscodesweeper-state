# Issue #326119 — Stuttering due to missing commit information.

[https://github.com/microsoft/vscode/issues/326119](https://github.com/microsoft/vscode/issues/326119)

## Summary

This is a support question, not a bug. The reported 'freeze' is the expected behavior when `git.useEditorAsCommitInput` is enabled (default): VS Code opens a full editor tab for the commit message and waits for the user to type a message, save, and close the tab. The user appears unfamiliar with this workflow.

## Change summary

User perceives a freeze when committing without a commit message, but VS Code is waiting for input in the commit message editor tab.

## Best solution

Close as a question. The behavior is by design: when `git.useEditorAsCommitInput` is `true` (the default), clicking commit without a message opens an editor tab where the user writes the commit message. After typing the message, closing the editor tab completes the commit. Alternatively, the user can type the message directly in the SCM input box before clicking commit.

  > Thanks for the report! What you're seeing isn't actually a freeze — it's the expected behavior.   >
