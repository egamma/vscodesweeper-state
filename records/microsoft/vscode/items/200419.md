# Issue #200419 — Git - input validation got broken in 1.85.0

[https://github.com/microsoft/vscode/issues/200419](https://github.com/microsoft/vscode/issues/200419)

## Summary

The default for `git.inputValidation` was intentionally changed to `false` in 1.85.0 and remains `false` on current main. The team member acknowledged the default should be reverted (comment 26) but noted outstanding issues. With 43 upvotes and active maintainer discussion, this should stay open as under-discussion.

## Change summary

Restore the default for `git.inputValidation` to enabled (previously `warn`, now `false`) and refine the diagnostics-based validation UI based on user feedback.

## Best solution

The maintainer (@lszomoru) should decide when to flip `git.inputValidation` default back to `true` in `extensions/git/package.json`, having already committed to doing so once remaining UI issues are sorted. The new diagnostics-based validation (in `extensions/git/src/diagnostics.ts`) is shipped but the default still suppresses it — the remaining work is a one-line default change plus any UI polish (e.g. squiggle only on overflowing characters, not the whole line).
