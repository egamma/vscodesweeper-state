# Issue #176107 — Same file nested under two different files cannot be renamed

[https://github.com/microsoft/vscode/issues/176107](https://github.com/microsoft/vscode/issues/176107)

## Summary

Still reproducible on main: when a file nesting pattern with a wildcard (e.g. `.prettierignore*`) matches multiple parent files, the same child file appears nested under each matching parent, and rename fails because the single ExplorerItem instance is shared across multiple tree positions.

## Change summary

A file nested under two different parent files via wildcard nesting patterns appears duplicated in the explorer and cannot be renamed.

## Best solution

The nesting algorithm in `explorerFileNestingTrie.ts` intentionally assigns a child to all matching ancestors, but `explorerModel.ts` reuses the same `ExplorerItem` object across parents (overwriting `nestedParent`). Either the nesting trie should pick only the first/best matching parent, or `explorerModel.ts` should handle multi-parent nesting gracefully — e.g. by showing the child only under its first matching parent, or by creating wrapper nodes. The rename handler in `fileActions.ts` relies on `stat.parent` which would also need to be correct for the focused instance.
