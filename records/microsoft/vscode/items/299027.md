# Issue #299027 — View all edits throws Unexpected type when a CustomTextEditorProvider is the active editor

[https://github.com/microsoft/vscode/issues/299027](https://github.com/microsoft/vscode/issues/299027)

## Summary

Still reproduces on main: `_createEditorIntegration` in chatEditingModifiedDocumentEntry.ts calls `assertType(codeEditor)` which throws 'Unexpected type' when the active editor is a CustomTextEditorProvider (webview-backed, no code editor). Team-authored issue, keeping open.

## Change summary

Clicking 'View all edits' throws 'Unexpected type' when the active editor is a CustomTextEditorProvider because getCodeEditor() returns null for webview editors.

## Best solution

Guard the `_createEditorIntegration` path in `chatEditingModifiedDocumentEntry.ts` (line ~307) to handle the case where `getCodeEditor()` returns null for CustomTextEditorProvider panes — either return a no-op integration or fall back to opening a text diff editor.
