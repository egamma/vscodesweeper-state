# Issue #255091 — Add option to exclude search with git submodules

[https://github.com/microsoft/vscode/issues/255091](https://github.com/microsoft/vscode/issues/255091)

## Summary

Valid feature request that passed the community upvote threshold (21 👍) and was promoted to Backlog. No implementation exists on current main — the search infrastructure has no submodule-awareness. The request fills a real gap for projects with many submodules.

## Change summary

Add an option to exclude git submodules from VS Code's search, analogous to the existing 'use ignore files' toggle.

## Best solution

Add submodule-awareness to the search file-walking layer: parse `.gitmodules` to discover submodule paths, then expose a toggle (similar to 'Use Ignore Files') in the search view and a corresponding `search.useIgnoreSubmodules` setting. Implementation would touch the search file provider and the search view UI. This is a core-appropriate feature since it relies on git integration already built into VS Code.
