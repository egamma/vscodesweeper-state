# Issue #146671 — [scss] interpolation validation issues

[https://github.com/microsoft/vscode/issues/146671](https://github.com/microsoft/vscode/issues/146671)

## Summary

This is an upstream issue in vscode-css-languageservice. The SCSS parser's interpolation handling has known gaps (multiple open upstream issues), and VS Code simply delegates validation to that library. The bug persists on current main.

## Change summary

SCSS interpolation syntax (e.g. `#{$var}(...)`) triggers false validation errors like `css-semicolonexpected` and `css-ruleorselectorexpected`.

## Best solution

The fix belongs in microsoft/vscode-css-languageservice's SCSS parser (src/parser/scssParser.ts). Multiple related issues are open upstream (e.g. https://github.com/microsoft/vscode-css-languageservice/issues/269, https://github.com/microsoft/vscode-css-languageservice/issues/412). Once fixed upstream, bumping the dependency version in VS Code will resolve this.
