# Issue #146671 — [scss] interpolation validation issues

[https://github.com/microsoft/vscode/issues/146671](https://github.com/microsoft/vscode/issues/146671)

## Summary

Still reproduces on current main — the SCSS parser in vscode-css-languageservice (v6.3.4) does not handle interpolation in certain value/term positions (e.g. `#{$var}(...)` as a function call, or `#{$var}` in @keyframes names). The fix belongs in the upstream language service.

## Change summary

SCSS interpolation like `#{$function}(0)` in property values or `#{$name}` in @keyframes triggers false validation errors.

## Best solution

Fix the SCSS parser in microsoft/vscode-css-languageservice to recognize interpolation in `_parseTermExpression` and in `@keyframes` name positions, then bump the dependency in VS Code's css-language-features extension.
