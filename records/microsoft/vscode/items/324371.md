# Issue #324371 — Cannot read properties of undefined (reading 'length') when built-in search tool returns 0 matches

[https://github.com/microsoft/vscode/issues/324371](https://github.com/microsoft/vscode/issues/324371)

## Summary

Real, confirmed bug in the Copilot Chat webview renderer: the React component tree crashes with a null/undefined `.length` access when rendering a built-in search tool result that returned 0 matches. A second commenter confirms reproduction on VS Code 1.128.0. The tool-side code correctly handles 0 matches (returns `status:ok`), so the fix belongs in the webview rendering path. Not fixed on current main.

## Change summary

The chat panel shows a red error (`Cannot read properties of undefined (reading 'length')`) instead of a normal '0 matches' message when a built-in search tool returns no results.

## Best solution

Guard the webview's prompt-tsx node renderer against null/undefined `children` and `references` arrays in the serialized tree. The crash is in the minified React component chain (`Fkt > iEt > uEt > Jv`) that processes the serialized `PromptElementJSON` node tree; the 0-match branch emits a node with `children: null` or no `references` array, and the renderer unconditionally accesses `.length`. The fix is a null-coalesce (`?? []`) or optional-chain (`?.length`) at the point where the renderer iterates over children/references. The relevant source is in `extensions/copilot/` — likely the webview rendering code that deserializes and renders `LanguageModelPromptTsxPart` results.
