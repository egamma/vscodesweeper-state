# Issue #276898 — Unexpected interactions with faster NES provider

[https://github.com/microsoft/vscode/issues/276898](https://github.com/microsoft/vscode/issues/276898)

## Summary

Team-authored bug (alexdima, MEMBER) describing two visual glitches in the inline-completions/NES system when the provider responds faster than default debounce. Kept open per hard keep-open rule for team-authored issues.

## Change summary

When NES debounce/cache delays are set to 0, inline edit suggestions flicker and change unexpectedly while the user types text that matches the suggestion, and ghost text disappears/reappears when navigating with arrow keys.

## Best solution

Fix the inline-completions controller to properly detect when the user is typing characters that match the current suggestion (avoid replacing it with a new one), and stabilize ghost text rendering during cursor navigation. Likely in `src/vs/editor/contrib/inlineCompletions/browser/` — the suggestion-tracking and cursor-movement handling paths.
