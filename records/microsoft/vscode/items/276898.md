# Issue #276898 — Unexpected interactions with faster NES provider

[https://github.com/microsoft/vscode/issues/276898](https://github.com/microsoft/vscode/issues/276898)

## Summary

Team-authored bug report (alexdima) documenting two visual glitches in inline completions when the NES provider responds faster than default debounce settings: suggestion content instability while typing, and ghost text flickering on cursor movement. Keeping open per team-member policy.

## Change summary

Inline edit suggestions flicker or change unexpectedly when NES debounce/cache delays are set to 0, causing visual instability during typing and cursor navigation.

## Best solution

Fix the inline-completions controller to properly stabilize suggestions when the user is typing text that matches the current suggestion, and suppress ghost text re-rendering on cursor movement when the suggestion content hasn't changed. Likely in the inline completions widget/model reconciliation logic.
