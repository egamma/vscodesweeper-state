## Summary

Team-authored confirmed bug about task identity mismatch when launched from folder navigation vs. recent tasks — must stay open for maintainer judgment.

## Change summary

Running a task by navigating to its folder creates a new task instance instead of recognizing the already-running task and offering to terminate/re-run it.

## Best solution

Fix task identity/key computation so that tasks launched from folder navigation produce the same key as tasks launched from recent runs, ensuring the duplicate-detection logic correctly identifies them as the same task.
