## Summary

Still reproducible on main: the autorun at line 230 of hideUnchangedRegionsFeature.ts calls setHiddenAreas without preserving scroll position when the feature is toggled. No fix has been committed.

## Change summary

Toggling 'hide unchanged regions' in the diff editor resets the scroll position instead of preserving the user's current view.

## Best solution
