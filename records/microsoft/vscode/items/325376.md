# Issue #325376 — "reveal in file explorer" does nothing on deletes, the diff tab is useless too

[https://github.com/microsoft/vscode/issues/325376](https://github.com/microsoft/vscode/issues/325376)

## Summary

Team-authored issue — keeping open. The bug is real and traceable in source: 'Reveal in File Explorer' for a git-deleted file silently fails because the explorer cannot select a file that no longer exists on disk; the diff editor breadcrumb for deleted files also has no interaction handlers.

## Change summary

When a file is deleted in git, 'Reveal in File Explorer' from the SCM panel does nothing, and the diff tab's breadcrumb path segments appear interactive but have no click/context-menu behavior.

## Best solution

For deleted files, the reveal command should fall back to revealing the parent folder (which still exists) in the explorer. The logic is in `src/vs/workbench/contrib/files/browser/fileCommands.ts` around line 326 — when `explorerService.select()` fails (file not found), it should walk up to the nearest existing ancestor and select that. The diff editor breadcrumb for deleted resources likely needs attention in the SCM diff editor integration to either disable the segments or wire up a parent-folder reveal.
