# Issue #291562 — Chat took too long to get ready

[https://github.com/microsoft/vscode/issues/291562](https://github.com/microsoft/vscode/issues/291562)

## Summary

The slow startup is caused by network connectivity failures (repeated `net::ERR_CONNECTION_CLOSED` and `fetch failed` errors) during GitHub Copilot Chat extension activation, not by VS Code core. The extension's activation blockers waited ~76 seconds due to failed network requests. This belongs with the Copilot Chat extension, not VS Code.

## Change summary

GitHub Copilot Chat extension took ~76 seconds to activate due to network connection failures during token and model fetching.

## Best solution

The user's network environment is dropping connections to Copilot services, causing the extension to retry and eventually fall back. The user should check their network/proxy/firewall settings. If there's a genuine extension bug in timeout handling, it should be filed at https://github.com/microsoft/vscode-copilot-release.
