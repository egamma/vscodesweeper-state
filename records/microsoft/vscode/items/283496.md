# Issue #283496 — Better granularity/configuration options for the "Terminal integrated Suggest" feature

[https://github.com/microsoft/vscode/issues/283496](https://github.com/microsoft/vscode/issues/283496)

## Summary

Valid feature request for per-command terminal suggest configuration. Significant granularity has been added since filing (quickSuggestions object with commands/arguments/unknown, provider toggles, selectionMode), but the specific per-command opt-in/opt-out Tyriar endorsed in comment #5 remains unimplemented.

## Change summary

Allow users to enable or disable terminal IntelliSense suggestions on a per-command basis (e.g., disable for `cd` but keep for `git`).

## Best solution

Implement per-command suggest configuration as Tyriar suggested — a setting like `terminal.integrated.suggest.commandOverrides` mapping command names to enable/disable. The existing `quickSuggestions` object schema (commands/arguments/unknown) partially addresses the concern but doesn't offer per-command granularity.
