# Issue #320375 — Linux Pasting file from Nautilus only shows filename or "/" instead of full path

[https://github.com/microsoft/vscode/issues/320375](https://github.com/microsoft/vscode/issues/320375)

## Summary

Verified duplicate of the open canonical issue #170708, which tracks the exact same Linux clipboard problem — Nautilus copies put the full path in text/uri-list but VS Code reads text/plain, yielding only the filename. Two earlier issues (#147285, #155453) were already closed as duplicates of #170708.

## Change summary

Pasting a file copied from Nautilus on Linux inserts only the filename instead of the full file path, because VS Code reads text/plain rather than text/uri-list from the clipboard.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/170708 — the fix belongs in VS Code's Electron clipboard integration or the paste provider pipeline (src/vs/editor/contrib/dropOrPasteInto/browser/defaultProviders.ts and src/vs/editor/browser/dataTransfer.ts), ensuring text/uri-list is surfaced for external clipboard pastes on Linux so the PathProvider can decode file:// URIs into full paths.
