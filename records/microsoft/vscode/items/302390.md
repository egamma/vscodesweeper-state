# Issue #302390 — Debug Console: text selection is lost when scrolling up (virtualized list resets selection)

[https://github.com/microsoft/vscode/issues/302390](https://github.com/microsoft/vscode/issues/302390)

## Summary

The user selection feature in the virtualized list (added in dd86be1a954, fixed in eb9dbc75474) tracks selection bounds to keep selected rows rendered, but when dragging upward past the viewport, the browser's DOM selection cannot extend into rows that haven't been rendered yet — the selection collapses. This is a real limitation of the current implementation that remains on main.

## Change summary

Text selection in the Debug Console is lost when scrolling upward because the virtualized list recycles DOM elements outside the viewport, breaking the browser's native selection.

## Best solution

Extend the `userSelection` logic in `src/vs/base/browser/ui/list/listView.ts` to pre-render rows above the viewport as the user drags upward during a selection (similar to how `currentSelectionBounds` already extends the render range downward). The `selectionchange` handler and `getRenderRange` expansion need to anticipate the direction of the drag and proactively expand the render range ahead of the mouse. This is a non-trivial enhancement to the list-widget's user-selection support.
