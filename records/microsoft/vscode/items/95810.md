# Issue #95810 — Git - VSCode locks git index in AFP repo

[https://github.com/microsoft/vscode/issues/95810](https://github.com/microsoft/vscode/issues/95810)

## Summary

Legitimate but very niche AFP-specific bug that maintainers already triaged to Backlog with help-wanted. The reporter confirmed it does not reproduce on local (non-AFP) repos, and AFP is now deprecated by Apple in favor of SMB. No fix or recent confirmation exists, but the existing maintainer triage decision is reasonable — keeping open for a community contributor.

## Change summary

VS Code's git integration locks the git index on AFP (Apple Filing Protocol) network volumes, preventing command-line git operations until the VS Code window is closed.

## Best solution

Investigate whether VS Code's file watcher or the built-in git extension holds file handles open under `.git/` in a way that triggers AFP's locking semantics. The fix likely lives in `extensions/git/src/` (possibly the repository status polling or ref watching) or in the file-watcher layer. Given AFP's deprecation, it would also be worth checking whether the issue reproduces on SMB volumes — if not, this may be closeable as an upstream AFP quirk. The `help wanted` label is appropriate for a community contributor to pick up.
