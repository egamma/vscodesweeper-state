## Summary

This request targets the GitHub Copilot cloud service architecture (data retention policies, encryption at rest, audit logging, tenant isolation, threat models) rather than VS Code the editor. Nearly all of the requested capabilities — scoped context retention, ephemeral memory with expiration, IAM-integrated admin policies, telemetry/audit logs for context lifecycle — are Copilot service-side concerns outside VS Code core's scope.

## Change summary

Add a security-first, policy-enforced context retention and session memory architecture for GitHub Copilot Chat with enterprise audit, encryption, and zero-trust controls.

VS Code core owns the editor surface and local extension host, not the Copilot cloud service's data architecture.
