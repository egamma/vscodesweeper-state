# Issue #165266 — [folidng] python ranges unfold when the folding ranges are coming late

[https://github.com/microsoft/vscode/issues/165266](https://github.com/microsoft/vscode/issues/165266)

## Summary

Still reproduces on main. The sanitizeAndMerge logic in foldingRanges.ts unfolds a recovered range when the language server provides ranges with different end lines than the indent-based fallback, which happens every time a language server loads late.

## Change summary

Folded Python regions unfold when the language server (Pylance) provides folding ranges with different end lines than the indent-based fallback used during startup.

## Best solution

Relax the condition at foldingRanges.ts:339 so that when a recovered/previously-collapsed range matches a new provider range by start line, the collapsed state is preserved even if the end line differs slightly (e.g. within a tolerance or when the source was FoldSource.recovered). This avoids interpreting the provider switch as a user edit.
