# Issue #165266 — [folidng] python ranges unfold when the folding ranges are coming late

[https://github.com/microsoft/vscode/issues/165266](https://github.com/microsoft/vscode/issues/165266)

## Summary

Still reproducible on main: when the syntax provider (Pylance) arrives late with ranges whose end lines differ from the indent provider's initial ranges, `sanitizeAndMerge` drops the collapsed state because it treats the end-line mismatch as a structural change.

## Change summary

Folded Python regions unfold automatically when the language server provides folding ranges that differ from the initial indent-based ranges.

## Best solution

Guard the collapse-state transfer in `FoldingRegions.sanitizeAndMerge` (src/vs/editor/contrib/folding/browser/foldingRanges.ts, line ~339) so that when ranges transition from indent provider to syntax provider with only an end-line difference, the collapsed state is preserved rather than dropped. The fix should distinguish a provider-switch update from a user edit.
