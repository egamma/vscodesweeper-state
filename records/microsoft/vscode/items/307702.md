# Issue #307702 — Copy and paste from Explorer to Chat view should attach the image

[https://github.com/microsoft/vscode/issues/307702](https://github.com/microsoft/vscode/issues/307702)

## Summary

Team-authored issue confirmed by a second member (meganrogge). The paste providers in `chatPasteProviders.ts` handle image clipboard data (`PasteImageProvider` via `image/*` mime) and code text (`PasteTextProvider` via `text/plain`), but when a file is copied from the Explorer, the clipboard carries a file path as text — there is no provider that intercepts Explorer-copied file URIs and converts them into chat context attachments the way drag-and-drop does. Keeping open per the team-author rule.

## Change summary

Copying a file in the Explorer and pasting into the Chat input inserts the file path as text instead of attaching the file as context (drag-and-drop works correctly).

## Best solution

Add a paste provider (or extend `PasteTextProvider`) in `chatPasteProviders.ts` that detects Explorer-originated file URIs on the clipboard and converts them into context attachments — mirroring the drag-and-drop attachment path. For image files, this should create an image attachment; for other files, a file-context attachment.
