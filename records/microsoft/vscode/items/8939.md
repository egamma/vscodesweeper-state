# Issue #8939 — Show quick open results as preview while navigating through them

[https://github.com/microsoft/vscode/issues/8939](https://github.com/microsoft/vscode/issues/8939)

## Summary

This highly-upvoted (247 👍) feature request for previewing files while navigating Quick Open results remains unimplemented on current main. The onDidChangeActive handler only decorates symbols in the already-active editor — no preview editor is opened for file results during navigation. The request is well above the 20-upvote backlog threshold and has been in the Backlog milestone since 2016.

## Change summary

Show a preview editor of the focused file while navigating through Quick Open (Ctrl+P) results, restoring the previous state on cancel.

## Best solution

Implement preview-on-navigate in `src/vs/workbench/contrib/search/browser/anythingQuickAccess.ts` by opening a transient preview editor in the `onDidChangeActive` handler for file picks (similar to how `gotoSymbolQuickAccess.ts` previews symbols), with state restoration on cancel. The `enablePreviewFromQuickOpen` setting could gate this behavior.
