# Issue #236365 — "Undo" history lost after "Save as".

[https://github.com/microsoft/vscode/issues/236365](https://github.com/microsoft/vscode/issues/236365)

## Summary

Still reproduces on main: the Save As flow creates a fresh target working copy and reverts the source, discarding all undo history. No fix has landed.

## Change summary

Save As discards the undo/redo history because the target file gets a new text model while the source model is reverted.

## Best solution

Preserve the undo/redo stack when performing Save As by transferring the text model (or its undo elements) from the source working copy to the target, rather than creating a blank model and copying content into it. The fix would likely touch `src/vs/workbench/services/workingCopy/common/fileWorkingCopyManager.ts` (the `doSaveAs` method) and possibly the undo/redo service to support model identity transfer.
