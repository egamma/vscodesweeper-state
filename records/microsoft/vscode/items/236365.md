# Issue #236365 — "Undo" history lost after "Save as".

[https://github.com/microsoft/vscode/issues/236365](https://github.com/microsoft/vscode/issues/236365)

## Summary

Keep open: current main still appears to rebuild the Save As target as a new edit rather than preserving the source edit history, which matches the reported undo behavior. I did not find a fixing commit for https://github.com/microsoft/vscode/issues/236365, so there is not enough evidence for a completed close.

## Change summary

After Save As, the editor should preserve useful undo history instead of leaving only a blank-to-final-state undo step.

## Best solution

The undo-redo/text-file owners should decide whether Save As should transfer or preserve the source model undo stack when replacing the editor with the target resource, and add coverage for Save As with an existing edit stack. The large-file path also needs an explicit decision because closed-file undo preservation is capped at 20 MB in the model service.
