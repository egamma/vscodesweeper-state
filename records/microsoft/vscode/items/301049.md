# Issue #301049 — Trailing comma breaks fonts in editor.fontFamily, terminal.integrated.fontFamily

[https://github.com/microsoft/vscode/issues/301049](https://github.com/microsoft/vscode/issues/301049)

## Summary

Still reproducible on main: a trailing comma in fontFamily settings produces invalid CSS font-family values because neither the editor nor terminal code strips trailing commas before applying the value.

## Change summary

A trailing comma in `editor.fontFamily` or `terminal.integrated.fontFamily` causes fonts to break because the raw value (with trailing comma) is passed directly to CSS.

## Best solution

Sanitize the fontFamily value by trimming trailing commas (and optional whitespace) before it is used in CSS. The editor path is in `src/vs/editor/common/config/fontInfo.ts` (`getMassagedFontFamily` / `_wrapInQuotes`); the terminal path flows through `src/vs/workbench/contrib/terminal/browser/terminalConfigurationService.ts`. A small `.replace(/,\s*$/, '')` guard in both paths would resolve it.
