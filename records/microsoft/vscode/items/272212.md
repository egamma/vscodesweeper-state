[https://github.com/microsoft/vscode/issues/272212](https://github.com/microsoft/vscode/issues/272212)

## Summary

Valid feature request — the terminal multi-line link detector currently handles ripgrep, eslint, and git-diff formats but does not recognize Claude Code or Codex diff output. No work has landed since this was filed.

## Change summary

Support terminal link detection for Claude Code and Codex CLI diff output formats, allowing click-to-jump to file and line number.

## Best solution

Extend the `lineNumberPrefixMatchers` or add new matchers in `src/vs/workbench/contrib/terminalContrib/links/browser/terminalMultiLineLinkDetector.ts` to recognize the Claude Code (`⏺ Update(path)` with numbered lines) and Codex (`• Edited path (+N -M)` with numbered lines) output patterns, mapping them to file+line links.
