# Issue #303204 — Searching for file using copy/pasted Excel cell text

[https://github.com/microsoft/vscode/issues/303204](https://github.com/microsoft/vscode/issues/303204)

## Summary

Still reproduces on main: the explorer's paste listener at line 337 of explorerView.ts fires on any paste within the explorer container — including the file-finder input — without checking whether the event target is an editable element, so clipboard data containing files (e.g. Excel's image representation of a copied cell) triggers the file-paste dialog even when the user intended to paste text into the filter.

## Change summary

Pasting Excel cell text into the Explorer file finder triggers an unwanted 'paste image.png' confirmation dialog because Excel puts both text and image data on the clipboard.

## Best solution

Guard the paste listener in `src/vs/workbench/contrib/files/browser/views/explorerView.ts` (line 337) with a `DOM.isEditableElement(event.target)` check — the same pattern already used at line 633 for the keydown handler — so that paste events targeting text inputs are left to the browser's default handling and only non-input pastes trigger the file-paste flow.
