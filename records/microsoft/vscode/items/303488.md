## Summary

Valid bug — WebGL context loss on macOS causes a brief white flash before the fallback renderer activates. The context loss handler exists (disposes webgl addon) but doesn't prevent the momentary visual artifact. No fix has landed.

## Change summary

Terminal briefly shows as white with a broken-image icon when switching back to VS Code after macOS GPU context loss.

## Best solution

Improve the WebGL context loss recovery path in `src/vs/workbench/contrib/terminal/browser/xterm/xtermTerminal.ts` to either pre-render a placeholder or trigger the DOM fallback renderer faster to avoid the visible flash. This may also need coordination with the @xterm/addon-webgl package for smoother context restoration.
