# Issue #323444 — 'Branch Changes' still offers to commit changes when there are no uncommitted changes

[https://github.com/microsoft/vscode/issues/323444](https://github.com/microsoft/vscode/issues/323444)

## Summary

Team-authored bug: the 'Commit Changes' action in the Agents window's Branch Changes view is visible even when `sessions.hasUncommittedChanges` is false. Keeping open per team-member policy.

## Change summary

The 'Commit Changes' button in the Agents window's Branch Changes header remains visible/enabled when there are no uncommitted changes, offering an action that does nothing.

## Best solution

Gate the commit action's visibility (its `when` clause in the menu contribution or the `_getButtonConfiguration` logic in `changesView.ts`) on the `sessions.hasUncommittedChanges` context key so the button is hidden or disabled when uncommitted changes count is 0.
