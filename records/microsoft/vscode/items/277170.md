# Issue #277170 — [Accessibility]: Screen reader do not announce when user is in the "Search workspace" dialog.

[https://github.com/microsoft/vscode/issues/277170](https://github.com/microsoft/vscode/issues/277170)

## Summary

Still reproduces on main: the quick pick container has no ARIA dialog role or aria-labelledby, so screen readers don't announce the context change. An open PR (#277172) exists but hasn't been merged; keeping open pending maintainer review of that PR.

## Change summary

Screen readers do not announce when the quick pick ("Search workspace") overlay opens because the container lacks an appropriate ARIA role.

## Best solution

Review and land (or close with guidance) the linked PR https://github.com/microsoft/vscode/pull/277172 which adds `role="dialog"` and `aria-labelledby` to the quick pick widget. The team discussion (Comment 4) about whether `dialog` is the correct role for a non-modal overlay should be resolved first.
