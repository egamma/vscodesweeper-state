# Issue #281808 — Image attachment disappears when sending to new chat

[https://github.com/microsoft/vscode/issues/281808](https://github.com/microsoft/vscode/issues/281808)

## Summary

Still reproduces on main: the SendToNewChatAction only saves the text input (via getInput(), which returns a string) before clearing the widget and sending to a new chat — image and other attachments are lost. Filed by a team contributor, so keeping open for maintainer judgment.

## Change summary

Image attachments pasted into the chat input are lost when using Ctrl+Shift+Enter (Send to New Chat).

## Best solution

In SendToNewChatAction.run(), save the current attachments (from widget.input.attachmentModel) alongside the text before calling setInput(''), then re-attach them to the new session's attachment model before calling acceptInput(). The fix is localized to chatExecuteActions.ts.
