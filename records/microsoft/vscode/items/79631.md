# Issue #79631 — [html] don't validate custom event handlers as JavaScript

[https://github.com/microsoft/vscode/issues/79631](https://github.com/microsoft/vscode/issues/79631)

## Summary

Still reproduces on main: `getAttributeLanguage` in the HTML language server treats any `on*` attribute value as JavaScript via a blanket `/^(on\w+)$/i` regex, causing false JS diagnostics for custom (non-event) attributes.

## Change summary

Custom HTML attributes prefixed with "on" (e.g. `onInvalidAttribute`) have their values incorrectly validated as JavaScript, producing spurious "; expected" errors.

## Best solution

Restrict the `on*` regex in `getAttributeLanguage` (extensions/html-language-features/server/src/modes/embeddedSupport.ts) to only match known DOM event handler names, or add a setting to opt out of JS validation for attribute values. This keeps existing event-handler intelligence intact while eliminating false positives for custom attributes.
