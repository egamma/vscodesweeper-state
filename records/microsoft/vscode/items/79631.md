# Issue #79631 — [html] don't validate custom event handlers as JavaScript

[https://github.com/microsoft/vscode/issues/79631](https://github.com/microsoft/vscode/issues/79631)

## Summary

Still reproduces on current main: `getAttributeLanguage` in embeddedSupport.ts uses `/^(on\w+)$/i` to classify any on-prefixed attribute as JavaScript, causing false diagnostics for custom/non-standard attributes like `onInvalidAttribute`.

## Change summary

Custom HTML attributes prefixed with "on" are incorrectly validated as JavaScript event handlers, producing spurious ';' expected errors.

## Best solution

Restrict the `on*` → JavaScript mapping in `extensions/html-language-features/server/src/modes/embeddedSupport.ts` to known DOM event handler attributes (e.g. using an allowlist from the HTML spec or vscode-html-languageservice data), so that custom `on`-prefixed attributes are not treated as embedded JavaScript.
