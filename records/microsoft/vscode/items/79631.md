# Issue #79631 — [html] don't validate custom event handlers as JavaScript

[https://github.com/microsoft/vscode/issues/79631](https://github.com/microsoft/vscode/issues/79631)

## Summary

Still reproduces on main: the HTML extension treats any attribute starting with 'on' as JavaScript via a broad regex, causing false validation errors for custom non-event attributes.

## Change summary

Custom HTML attributes prefixed with 'on' (e.g. onInvalidAttribute) are incorrectly validated as JavaScript, producing spurious ';' expected errors.

## Best solution

Restrict the regex in `getAttributeLanguage()` to only match known HTML event handler names (onclick, onload, etc.) rather than any `on\w+` pattern, or use the HTML data providers to determine valid event attributes. The fix is localized to `extensions/html-language-features/server/src/modes/embeddedSupport.ts` line 246.
