# Issue #189407 — Git - staging file does not show any progress indicator

[https://github.com/microsoft/vscode/issues/189407](https://github.com/microsoft/vscode/issues/189407)

## Summary

Reporter confirmed the issue no longer reproduces and explicitly requested closure. The maintainer previously explained the as-designed behavior: when `git.optimisticUpdate` is enabled (the default), progress indicators are intentionally skipped because the UI updates immediately. The underlying delay was environmental (slow git operations on Windows).

## Change summary

No progress indicator shown when staging/unstaging files in the Source Control view.

## Best solution

Close as not reproducible per the reporter's own confirmation. The design is intentional — optimistic updates trade progress indicators for a snappier feel. Users with slow git environments can disable `git.optimisticUpdate` to restore progress indicators.
