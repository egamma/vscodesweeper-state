# Issue #238365 — Wrong Keyboard Layout and ignored settings - happens on `vscode.dev`

[https://github.com/microsoft/vscode/issues/238365](https://github.com/microsoft/vscode/issues/238365)

## Summary

Still reproduces on main: the `keyboard.layout` setting is ignored on page reload in vscode.dev due to a race condition — the setting is applied before the async keymap data has loaded.

## Change summary

On vscode.dev, configuring `keyboard.layout` to a non-US layout and reloading the page resets the status bar to US because the setting is consumed before the keyboard layout data has been asynchronously loaded.

## Best solution

In `BrowserKeyboardMapperFactory`, after the async `import()` of keyboard layout data resolves and `_initialized` is set to `true`, check whether a non-autodetect `keyboard.layout` setting is configured and, if so, call `setKeyboardLayout(layout)` instead of (or after) `setLayoutFromBrowserAPI()`. The fix is localized to `src/vs/workbench/services/keybinding/browser/keyboardLayoutService.ts` around lines 463–469.
