# Issue #229530 — Semantic tokens types do not override "standard token type" and "font style"

[https://github.com/microsoft/vscode/issues/229530](https://github.com/microsoft/vscode/issues/229530)

## Summary

Still present on main: the semantic token merge logic in sparseTokensStore.ts only overrides font style and foreground/background from semantic tokens, but never overrides the standard token type (Comment/String/RegExp). This causes TextMate-assigned comment type to persist even when semantic tokens reclassify the token, and the font style (italic) leaks through because the theme's comment styling remains active via the standard token type.

## Change summary

Semantic tokens cannot override the TextMate-assigned 'standard token type' (Comment/String/RegExp), causing incorrect styling (e.g., italic from comment themes) to persist even when semantic tokens reclassify a token.

## Best solution

Add a `SEMANTIC_USE_TOKEN_TYPE` flag to allow semantic tokens to override the standard token type bits during the merge in `src/vs/editor/common/tokens/sparseTokensStore.ts`. This requires also computing and encoding the appropriate token type in `semanticTokensProviderStyling.ts`. The scope inspector's strikethrough display for overridden font style (when it still applies) is a secondary polish issue.
