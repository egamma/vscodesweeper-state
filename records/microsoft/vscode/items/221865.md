# Issue #221865 — Illegal value for lineNumber

[https://github.com/microsoft/vscode/issues/221865](https://github.com/microsoft/vscode/issues/221865)

## Summary

Team-authored error-telemetry issue; this is the canonical tracker for an 'Illegal value for lineNumber' crash in the merge editor's `editsToLineRangeEdit` function. Two duplicates (#210995, #202006) were closed pointing here. The bug remains present on main — no fix commit targets this code path.

## Change summary

The merge editor's `editsToLineRangeEdit` calls `textModel.getLineMaxColumn()` with a lineNumber that can be out of bounds when ranges are stale relative to the text model state.

## Best solution

Add a bounds check in `editsToLineRangeEdit` (src/vs/workbench/contrib/mergeEditor/browser/model/modifiedBaseRange.ts) before calling `getLineMaxColumn`, or ensure the derived observable in `conflictActions.ts` invalidates properly when the underlying text model changes line count.
