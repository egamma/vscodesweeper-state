## Summary

Team-authored issue (MEMBER); the suggested validation fix (reject scopes containing spaces in `isAuthorizationServerMetadata`/`isAuthorizationProtectedResourceMetadata`) has not been implemented on current main. Keeping open.

## Change summary

MCP OAuth sign-in loops repeatedly when the auth server metadata contains scopes with spaces instead of a proper array.

## Best solution

Add validation in `src/vs/base/common/oauth.ts` inside `isAuthorizationServerMetadata` and `isAuthorizationProtectedResourceMetadata` to check each entry in `scopes_supported` for spaces, and surface a clear error message to the user rather than silently looping auth.
