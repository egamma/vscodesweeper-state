# Issue #282418 — Regression in vscode.workspaceConfiguration.Update: version 1.106.3 throws CodeExpectedError

[https://github.com/microsoft/vscode/issues/282418](https://github.com/microsoft/vscode/issues/282418)

## Summary

Team-authored regression report. The bug is source-traceable: when writing a standalone configuration key (like `launch`) as a whole object, `operation.key` becomes `undefined` (empty jsonPath access), and `inspect(undefined)` now returns a policyValue after the Policy improvements commit landed between 1.105 and 1.106, triggering the spurious policy error.

## Change summary

Calling `workspaceConfiguration.update('launch', {...}, WorkspaceFolder)` throws 'Unable to write undefined because it is configured in system policy' in 1.106.3, whereas it succeeded in 1.105.1.

## Best solution

The policy check in `configurationEditing.ts` line 512 should skip when `operation.key` is `undefined` (standalone configuration written in its entirety), or the `getConfigurationEditOperation` method should not produce an undefined key. The likely regression cause is commit bbf25583ab7 ('Policy improvements') changing inspect behavior for undefined/empty keys.
