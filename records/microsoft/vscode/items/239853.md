# Issue #239853 — ligatures starting with a pipe in the terminal only render correctly if selected

[https://github.com/microsoft/vscode/issues/239853](https://github.com/microsoft/vscode/issues/239853)

## Summary

Still reproduces as of 1.125.0 per the reporter's latest confirmation. The root cause is likely in the `font-ligatures` library (used by `@xterm/addon-ligatures`) failing to detect ligature ranges that begin with the pipe character `|`, or in how xterm.js renders those character-joiner ranges. The merged PR #318992 addressed config change detection, not the underlying rendering defect. This is an upstream issue in the xterm.js ligatures stack.

## Change summary

Terminal ligatures beginning with a pipe character (`|==`, `|===`, `|---`, `|->`) do not render correctly unless the text is selected.

## Best solution

The defect is in the xterm.js ligatures addon or the `font-ligatures` library (v1.4.x) — specifically in how `font.findLigatureRanges()` handles sequences starting with `|`. The fix should be pursued upstream in `xtermjs/xterm.js` (addon-ligatures) or `princjef/font-ligatures`. If the font-ligatures library cannot parse pipe-starting ligatures from the font tables, the fallback ligatures list in `LigaturesAddon` (which already includes `<|`, `<|>`, `|>`) could be extended with `|==`, `|===`, `|---`, `|->` as a workaround.
