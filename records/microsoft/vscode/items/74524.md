# Issue #74524 — Add support to distinguish between diagnostics created by a reconciler and a builder

[https://github.com/microsoft/vscode/issues/74524](https://github.com/microsoft/vscode/issues/74524)

## Summary

Team-authored feature request for a diagnostics API enhancement to distinguish builder vs. reconciler origins. Hard keep-open rule applies (author is MEMBER). Still a valid open design question with no implementation on main.

## Change summary

Add API support to distinguish diagnostics created by a reconciler from those created by a builder, preventing stale/duplicate problems.

## Best solution

Design and finalize a `DiagnosticTag` value or a dedicated property (e.g. `creator`) on `Diagnostic` to mark the source as builder or reconciler, enabling smarter merging in the Problems panel. This belongs in the extensions API surface (`src/vscode-dts/`) and the diagnostics collection machinery.
