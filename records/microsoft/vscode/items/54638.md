# Issue #54638 — [scss] Add "Go to or peek defintion" for imported mixins and variables for SCSS files

[https://github.com/microsoft/vscode/issues/54638](https://github.com/microsoft/vscode/issues/54638)

## Summary

Still a valid feature request. Go to Definition for cross-file SCSS symbols (imported mixins/variables) is not implemented — the built-in CSS language service's `findDefinition` only resolves symbols within the current document. The upstream language service tracks this as microsoft/vscode-css-languageservice#162.

## Change summary

Request cross-file 'Go to Definition' support for SCSS imported mixins and variables.

## Best solution

Implement cross-file symbol resolution in `vscode-css-languageservice` (tracked at https://github.com/microsoft/vscode-css-languageservice/issues/162). Once the upstream service supports it, bump the dependency in `extensions/css-language-features/server/`. Alternatively, this could be marked as a duplicate of an existing VS Code issue if one exists for the same scope.
