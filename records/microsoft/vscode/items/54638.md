## Summary

This requests cross-file Go to Definition for SCSS imported mixins/variables. The CSS language server already has a definition provider, but cross-file resolution quality depends on the upstream vscode-css-languageservice. Only 1 upvote, parked in Backlog — keep open as a valid feature request.

## Change summary

Request for Go to Definition / Peek Definition to resolve SCSS mixins and variables imported from other files.

## Best solution

Improve cross-file symbol resolution in the upstream `vscode-css-languageservice` library (which already provides `findDefinition`). The VS Code CSS extension server at `extensions/css-language-features/server/src/cssServer.ts` delegates directly to it, so the fix belongs upstream in that npm package rather than in core VS Code.
