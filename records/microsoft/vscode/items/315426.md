# Issue #315426 — The handling of characters with ambiguous widths

[https://github.com/microsoft/vscode/issues/315426](https://github.com/microsoft/vscode/issues/315426)

## Summary

Valid feature request for an editor setting to treat East Asian Ambiguous width characters as full-width (equivalent to vim's `ambiwidth=double`). The current `isFullWidthCharacter` implementation in `src/vs/base/common/strings.ts` does not cover Unicode East_Asian_Width=Ambiguous characters (like α, ↑), causing column misalignment with CJK monospace fonts.

## Change summary

Add an option to treat Unicode East Asian Ambiguous width characters as double-width for column calculation, so tab stops and alignment work correctly with CJK monospace fonts.

## Best solution

Add an editor option (e.g. `editor.ambiguousCharacterWidth: 'single' | 'double'`) that feeds into `cursorColumns.ts` column calculation, so users of CJK monospace fonts can opt into treating ambiguous-width characters as full-width. The change would touch `src/vs/base/common/strings.ts` (add an ambiguous-width character check) and `src/vs/editor/common/core/cursorColumns.ts` (consume the setting).
