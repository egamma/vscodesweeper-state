# Issue #243771 — sometimes when the terminal is wrapped, suggestions aren't provided

[https://github.com/microsoft/vscode/issues/243771](https://github.com/microsoft/vscode/issues/243771)

## Summary

Team-authored bug that remains unfixed on main — the `getCommand` call in `figInterface.ts` still returns null when the terminal input wraps (cursor index exceeds line length due to newline). Keeping open per team-member rule; the author plans to address it with custom command parsing.

## Change summary

Terminal suggestions fail to appear when the terminal input line wraps across multiple lines because cursor index accounting breaks in the shell parser.

## Best solution

Fix the cursor-index calculation in `extensions/terminal-suggest/src/fig/shell-parser/command.ts` (or the planned replacement parser) to handle newline characters from wrapped input, so `getCommand` no longer returns null when the prompt wraps.
