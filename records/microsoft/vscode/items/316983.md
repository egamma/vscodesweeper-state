# Issue #316983 — Bug: "tools: Tool names must be unique" Error when using Claude Sonnet 4.5 or 4.6 in Plan, Ask and Agent modes with the VSCode tool enabled

[https://github.com/microsoft/vscode/issues/316983](https://github.com/microsoft/vscode/issues/316983)

## Summary

Real bug in the Copilot extension's tool deduplication logic: `VirtualToolGrouper.deduplicateGroups` computes a unique name for duplicate `LanguageModelToolInformation` items but never applies the rename — it falls through to the else branch and pushes the original item with its duplicate name. This causes the Anthropic API to reject the request. Still present on current main. Routing to chat-tools for triage.

## Change summary

Claude Sonnet 4.5/4.6 requests in Agent/Plan/Ask modes fail with 'Tool names must be unique' when the VSCode tool is enabled, because duplicate tool definitions are sent to the Anthropic API.

## Best solution

Fix the `deduplicateGroups` method in `extensions/copilot/src/extension/tools/common/virtualTools/virtualToolGrouper.ts` (around line 246) to handle the `LanguageModelToolInformation` branch — when a non-VirtualTool item has a duplicate name, the computed unique name is discarded and the original duplicate name is pushed into the result. The fix should either clone/wrap the LanguageModelToolInformation with the new name, or prevent duplicate registrations upstream. Issue https://github.com/microsoft/vscode/issues/320635 appears to be a duplicate of this one and should be consolidated.
