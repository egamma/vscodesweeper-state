# Issue #301692 — After renaming a file from the directory tree, blank lines are continuously inserted in open file

[https://github.com/microsoft/vscode/issues/301692](https://github.com/microsoft/vscode/issues/301692)

## Summary

Likely a Wayland/KDE/Electron platform issue where the Enter keyup event is not properly delivered during the focus transition from the rename input box to the editor, causing repeated newline insertions. All confirmed reporters are on Fedora + KDE Plasma + Wayland. No VS Code fix commit exists on main; one commenter suggests it may have been resolved by an upstream Fedora/Wayland change. Issue #305132 is a duplicate of this one.

## Change summary

After renaming a file via F2 in the explorer and pressing Enter, blank lines are continuously inserted in the editor, as if the Enter key is stuck.

## Best solution

Investigate whether Electron/Chromium's Wayland IME or key-repeat handling drops the Enter keyup during the rename→editor focus transition. If reproducible, a defensive keyup-flush or focus-gate in the explorer rename completion path (`src/vs/workbench/contrib/files/browser/views/explorerViewer.ts`) could work around it. Close #305132 as a duplicate of this issue. If the upstream Wayland/Electron fix is confirmed, close both as upstream-resolved.
