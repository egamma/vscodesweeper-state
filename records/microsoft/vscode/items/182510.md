# Issue #182510 — Git - Support the "push.default=current" strategy

[https://github.com/microsoft/vscode/issues/182510](https://github.com/microsoft/vscode/issues/182510)

## Summary

Still reproduces on main: the Git extension's push logic constructs an explicit refspec (`HEAD.name:HEAD.upstream.name`) that bypasses Git's native `push.default` resolution. An open PR (#323650) addresses this — keeping open for that PR to land.

## Change summary

VS Code's Git push/sync ignores `push.default=current` and always pushes to the upstream branch, even when Git CLI would push to a same-named remote branch.

## Best solution

Review and land (or iterate on) https://github.com/microsoft/vscode/pull/323650, which fixes the push path to stop constructing an explicit refspec for default push/sync, letting Git decide the push target. This would also resolve the related issues #137686 and #158644.
