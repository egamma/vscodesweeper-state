## Best solution

Implement a 'Clear Task Problems' command in the tasks contribution, likely requiring changes to the marker service to distinguish task-originated markers from language-server markers that share the same owner. The marker service double-indexing (owner + resource) discussed in the thread needs to support a sub-owner or source tag so task markers can be selectively removed.
