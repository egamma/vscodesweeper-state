## Change summary

Add a command to clear all problems/markers that originated from task problem matchers.

## Best solution

Implement a command (and optionally a task-name parameter variant) that identifies markers contributed by task problem matchers and removes them via the marker service. The main challenge noted in the discussion is that task markers share owners with language servers, so a sub-owner or secondary filtering property on markers would be needed to disambiguate.
