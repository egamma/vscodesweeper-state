# Issue #280834 — Git Warning Message Incorrectly Styled as Error

[https://github.com/microsoft/vscode/issues/280834](https://github.com/microsoft/vscode/issues/280834)

## Summary

Still reproduces on main: SSH informational warnings on stderr are surfaced with error-level styling because the git extension treats all stderr content from a failed command as error text without distinguishing warning-level lines. Multiple community PRs are open but none have been merged.

## Change summary

Git/SSH informational warnings (e.g. 'warning: permanently added ... to the list of known hosts') are displayed with error styling (red icon) instead of warning/info styling.

## Best solution

The fix should filter or reclassify known informational SSH warning lines from stderr before surfacing them to the user. Several open PRs attempt this — a maintainer should review and land one of the open PRs (e.g. https://github.com/microsoft/vscode/pull/309186 or https://github.com/microsoft/vscode/pull/301090). The likely files are `extensions/git/src/git.ts` (stderr handling around line 661) and possibly the notification display code.
