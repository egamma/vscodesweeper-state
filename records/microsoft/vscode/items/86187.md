## Summary

Valid feature request for SCSS at-rule hover information. The hover logic lives in the upstream vscode-css-languageservice; no implementation has landed on main. The 'On Deck' milestone does not block a close, but the request is still valid and unaddressed — keeping open.

## Change summary

Hovering over SCSS at-rules (e.g. @mixin, @include, @use) shows no hover information, even though completion items for them display descriptions.

## Best solution

Implement at-rule hover descriptions in `vscode-css-languageservice` (the hover service there needs to return documentation for at-directive nodes the same way it does for CSS properties), then consume the updated package in VS Code. Related issue https://github.com/microsoft/vscode/issues/86488 covers references for SASS at-rules and could share groundwork.
