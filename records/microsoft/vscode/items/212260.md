# Issue #212260 — Extension buttons are cropped

[https://github.com/microsoft/vscode/issues/212260](https://github.com/microsoft/vscode/issues/212260)

## Summary

Still reproduces on current main: the extension list footer uses `overflow: hidden` (extension.css line 188) which clips buttons that exceed the container width rather than gracefully degrading with ellipsis or hiding overflow items. A team member confirmed it still occurs in April 2025, and the recent 'restart required' button addition makes this more likely.

## Change summary

Extension view buttons (Install, Restart Extensions, etc.) get visually cropped when the extensions panel is narrow, instead of ellipsizing or using an overflow menu.

## Best solution

Add `text-overflow: ellipsis` or an overflow-menu pattern to the footer's action bar in `src/vs/workbench/contrib/extensions/browser/media/extension.css` so that buttons that cannot fit at full width either truncate with ellipsis or collapse into an overflow indicator, rather than being hard-clipped by the parent's `overflow: hidden`.
