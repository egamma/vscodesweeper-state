# Issue #170937 — Support `runOn` with `folderOpen` when adding folder to workspace

[https://github.com/microsoft/vscode/issues/170937](https://github.com/microsoft/vscode/issues/170937)

## Summary

Still reproduces on main: RunAutomaticTasks sets a _hasRunTasks flag after the initial run, preventing folderOpen tasks from being triggered when a new folder is added to the workspace. The abstractTaskService does re-fetch tasks on folder change, but RunAutomaticTasks never re-evaluates.

## Change summary

Tasks with `runOn: folderOpen` are not triggered when a new folder is added to an already-open multi-root workspace.

## Best solution

RunAutomaticTasks should listen to onDidChangeWorkspaceFolders, diff the newly added folders against already-processed ones, and run any folderOpen tasks scoped to the new folders (respecting permissions). The fix belongs in src/vs/workbench/contrib/tasks/browser/runAutomaticTasks.ts.
