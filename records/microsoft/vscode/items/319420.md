# Issue #319420 — Model only visible when response completes in auto mode

[https://github.com/microsoft/vscode/issues/319420](https://github.com/microsoft/vscode/issues/319420)

## Summary

Implemented on main: auto-mode model routing now streams a 'Routed to {model}' collapsible content part as soon as the routing decision is known, before the response completes. This addresses the core request to see the chosen model during generation.

## Change summary

In auto model selection mode, the resolved model name was only visible after the response fully completed; the user wants it shown as soon as the model is chosen.

## Best solution

Keep the shipped implementation: the ChatAutoModeResolutionContentPart (commit 04b756ba9f0) streams 'Routed to {model}' with confidence/label details inline in the response as soon as the routing decision arrives, and the model picker hover (commit e0b6f9fabd2) shows auto-mode description. No further change needed.
