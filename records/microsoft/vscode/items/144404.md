# Issue #144404 — File explorer mask won't disappear when creating file/folder in a nested folder

[https://github.com/microsoft/vscode/issues/144404](https://github.com/microsoft/vscode/issues/144404)

## Summary

Still reproducible on current main: `openExplorerAndCreate` in `fileActions.ts` does not cancel/finish an existing editable item before starting a new one, so the `highlight` CSS class on the tree container is orphaned when the second create command overwrites `this.editable` without calling the first item's `onFinish`.

## Change summary

Clicking "New File" then "New Folder" (or vice versa) in the explorer without completing the first input leaves a semi-transparent overlay (the `highlight` class) stuck on the tree.

## Best solution

Add an early guard in `openExplorerAndCreate` (src/vs/workbench/contrib/files/browser/fileActions.ts ~line 930) that cancels any in-progress editable item before proceeding — similar to the `getEditable()` guard already present in `pasteFileHandler` (~line 1179). The guard should call `editable.data.onFinish('', false)` to trigger proper cleanup (child removal, `setEditable(null)`, `highlight` class removal).
