# Issue #258689 — canceled chats have checkmarks

[https://github.com/microsoft/vscode/issues/258689](https://github.com/microsoft/vscode/issues/258689)

## Summary

Team-authored polish issue: canceled chat responses show a checkmark icon in the history, which misleadingly signals success. The `ChatProgressContentPart` defaults to `Codicon.check` when the spinner stops, regardless of whether the response was canceled or completed. Still present on main — no fix found.

## Change summary

Canceled chats display a checkmark icon in the history, making them indistinguishable from successfully completed chats.

## Best solution

When rendering completed progress steps in `chatProgressContentPart.ts` and the response row rendering in `chatListRenderer.ts`, check `element.isCanceled` and use a distinct icon (e.g. `Codicon.close` or `Codicon.circle-slash`) for canceled responses instead of the default `Codicon.check`. The `ResponseModelState.Cancelled` state is already tracked in the model.
