# Issue #324996 — `list.find` doesn't match by file's directory path in reference view

[https://github.com/microsoft/vscode/issues/324996](https://github.com/microsoft/vscode/issues/324996)

## Summary

Valid request: the references view's keyboard navigation label provider only returns basename(uri) for file nodes, so list.find cannot match by directory path. An open PR (#325016) is attempting a fix.

## Change summary

list.find in the reference view should match file entries by their directory path, not just the filename basename.

## Best solution

Update `StringRepresentationProvider.getKeyboardNavigationLabel` in `src/vs/editor/contrib/gotoSymbol/browser/peek/referencesTree.ts` to include the relative directory path for FileReferences nodes (returning an array or combined string). Review and land the open PR https://github.com/microsoft/vscode/pull/325016 which attempts this.
