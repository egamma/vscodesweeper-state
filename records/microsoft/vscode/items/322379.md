# Issue #322379 — Spurious PR detections in Agents window

[https://github.com/microsoft/vscode/issues/322379](https://github.com/microsoft/vscode/issues/322379)

## Summary

Team-authored bug — the PR detection pipeline in the Agents window incorrectly resolves a PR for sessions that don't own a PR branch (e.g. a sidebar chat opened on `main`). The `SessionGitHubInfoResolver` queries `findPullRequestNumberByHeadBranch` for whatever branch the workspace is on, even for non-agent-host sessions that should not display PR status.

## Change summary

Sidebar chat sessions opened from a repo on `main` spuriously show PR links and CI status in the Agents window because the PR lookup pipeline runs on the workspace's current branch regardless of session type.

## Best solution

Guard the PR-number lookup path so it only activates for agent-host sessions that actually created/own a branch (sessions with an explicit PR association from the agent host git state), not for arbitrary sidebar chats that inherit the workspace's current branch. The fix likely lives in `src/vs/sessions/contrib/providers/agentHost/browser/sessionGitHubInfo.ts` (the `_coords` derivation or `_computeGitHubInfo`) or in how the local-chat-sessions provider populates `gitHubInfo`.
