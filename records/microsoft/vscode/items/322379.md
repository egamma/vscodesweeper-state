# Issue #322379 — Spurious PR detections in Agents window

[https://github.com/microsoft/vscode/issues/322379](https://github.com/microsoft/vscode/issues/322379)

## Summary

Team-authored bug about sidebar chat sessions inheriting workspace git state and falsely showing PR links/CI checks in the Agents window. The root cause is likely in how the Agents window resolves GitHub info for sessions that originated from sidebar chat — the `SessionGitHubInfoResolver` runs a PR-number lookup against the workspace branch (e.g. `main`), which matches unrelated PRs.

## Change summary

Sidebar chat sessions opened in the Agents window spuriously display a PR link and failing CI checks derived from the workspace's current git branch.

## Best solution

Either skip `SessionGitHubInfoResolver` for sessions that originated from sidebar chat (local chat sessions), or filter out PR lookups when the branch is a default branch like `main`/`master`. The `localChatSessionsProvider.ts` already sets `gitHubInfo: constObservable(undefined)`, but the Agents window may re-wrap sessions with its own `SessionGitHubInfoResolver` based on the workspace git state rather than honoring the provider's intent.
