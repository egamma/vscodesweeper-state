# Issue #251105 — Select Default Profile (terminal) doesn't work for projects with terminal specified

[https://github.com/microsoft/vscode/issues/251105](https://github.com/microsoft/vscode/issues/251105)

## Summary

The planned fix (workspace-scope picker step) from comment 8 is still unimplemented on main — the quickpick always writes to ConfigurationTarget.USER, ignoring workspace-level overrides. The closed PR #282054 never merged. Keeping open as a valid feature request with a clear plan from the terminal team.

## Change summary

Select Default Profile ignores workspace-level terminal.integrated.defaultProfile settings and always writes to user scope, making the picker ineffective for projects that define their own default shell.

## Best solution

Implement the plan from comment 8: add a second step to the default-profile picker that lets the user choose User/Workspace scope, and surface workspace-defined profiles in a separate section. The fix lives in `src/vs/workbench/contrib/terminal/browser/terminalProfileQuickpick.ts`.
