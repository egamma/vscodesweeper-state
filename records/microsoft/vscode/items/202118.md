# Issue #202118 — Parsing of .gitignore files doesn't match git, causing cmd+p to not find any files

[https://github.com/microsoft/vscode/issues/202118](https://github.com/microsoft/vscode/issues/202118)

## Summary

Confirmed upstream ripgrep bug (BurntSushi/ripgrep#2745) — bracket escape handling in .gitignore patterns diverges from git. The upstream issue remains open; VS Code correctly tracks this as an upstream dependency.

## Change summary

VS Code's file finder (Cmd+P) excludes all files when a .gitignore contains bracket-escaped characters like `*[\<\>\:\"\/\\\|\?\*]*`, because ripgrep misinterprets the `/` inside brackets as a path separator.

## Best solution

Wait for the upstream fix in ripgrep's `ignore` crate (BurntSushi/ripgrep#2745), then update VS Code's bundled @vscode/ripgrep-universal dependency. No core code change needed.
