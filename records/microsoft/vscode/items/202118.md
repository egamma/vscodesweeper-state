# Issue #202118 — Parsing of .gitignore files doesn't match git, causing cmd+p to not find any files

[https://github.com/microsoft/vscode/issues/202118](https://github.com/microsoft/vscode/issues/202118)

## Summary

Confirmed upstream ripgrep bug — the root cause lives in ripgrep's .gitignore glob parsing (BurntSushi/ripgrep#2745), which is still open. The issue is correctly labeled `upstream` and should stay open until the upstream fix lands and VS Code updates its bundled ripgrep.

## Change summary

VS Code's file search (Cmd+P) ignores all files when a .gitignore contains certain escaped bracket patterns like `*[\<\>\:\"\/\\\|\?\*]*`, because ripgrep misinterprets the `/` inside `[]` as a path separator.

## Best solution

Wait for the upstream fix in https://github.com/BurntSushi/ripgrep/issues/2745 to land, then update the bundled ripgrep version. No workaround is available in VS Code's own code since the glob parsing happens entirely inside ripgrep.
