# Issue #299549 — Clarify why someone would ever want to use `Bypass Approvals` vs `Autopilot`

[https://github.com/microsoft/vscode/issues/299549](https://github.com/microsoft/vscode/issues/299549)

## Summary

Team-authored issue requesting UX clarity on the distinction between two orthogonal session config axes (Bypass Approvals on the autoApprove axis vs Autopilot on the mode axis). The two-axis split exists in agentHostSchema.ts but the issue asks for better user-facing explanation of when you'd want one without the other. Keeping open per team-author rule.

## Change summary

The UI doesn't clearly explain why a user would choose Bypass Approvals (auto-approve all tool calls) independently of Autopilot (agent runs autonomously start to finish), making the two options seem redundant.

## Best solution

Improve the enum descriptions in `src/vs/platform/agentHost/common/agentHostSchema.ts` and/or the picker UI in `src/vs/sessions/contrib/providers/agentHost/browser/agentHostModePicker.ts` to clarify the orthogonality: Bypass Approvals controls tool-call consent (skip approval prompts), while Autopilot controls agent autonomy (no check-ins or questions). A user might want Bypass Approvals in Interactive mode to skip tool prompts but still collaborate step-by-step, or Autopilot with Default Approvals to let the agent drive but still gate dangerous operations.
