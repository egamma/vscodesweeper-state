# Issue #296643 — Model picker renders in top left corner when clicking "Other models"

[https://github.com/microsoft/vscode/issues/296643](https://github.com/microsoft/vscode/issues/296643)

## Summary

Team-authored bug with 'On Deck' milestone — keeping open. The model picker occasionally renders unanchored in the top-left corner when clicking 'Other models'. No fix commit found on main; the related issue #299589 was closed as a duplicate of this one, confirming it's the canonical tracker.

## Change summary

The model picker popup sometimes renders at position (0,0) instead of anchored to the button when clicking 'Other models'.

## Best solution

Fix the anchor element lookup in `src/vs/workbench/contrib/chat/browser/widget/input/chatModelPicker.ts` — the `show()` method falls back to `this._domNode` when no anchor is passed, but that element may not be in the DOM or may have zero dimensions during certain timing conditions (e.g. streaming completion). A guard ensuring the anchor has valid layout before positioning the action widget would resolve this.
