# Issue #308890 — Incorrect command line argument quoting/escaping in `extHostTerminalShellIntegration.ts`

[https://github.com/microsoft/vscode/issues/308890](https://github.com/microsoft/vscode/issues/308890)

## Summary

Team-authored issue; the naive quoting logic is still present on current main and the linked PR #314642 (addressing MCP/debug adapter paths) is still open. Keeping open for maintainer judgment on the broader fix strategy.

## Change summary

The `executeCommand(executable, args)` API overload uses a shell-agnostic, incomplete quoting heuristic that fails for arguments containing quotes, shell metacharacters, or executables with spaces.

## Best solution

Implement shell-aware escaping in the `executeCommand` overload (and audit the related locations listed in comments), or explicitly document the API as best-effort. The open PR https://github.com/microsoft/vscode/pull/314642 covers the MCP/debug-adapter subset but not the terminal shell integration path itself.
