# Issue #234294 — Feature Request: Expose findWidget Actions State via API

[https://github.com/microsoft/vscode/issues/234294](https://github.com/microsoft/vscode/issues/234294)

## Summary

Genuine API gap — the Find Widget's toggle states (case sensitive, whole word, regex) are not exposed to extensions. No existing setting, command argument, or proposed API covers reading or explicitly setting these states. The request has 0 upvotes after 8 months, so it stays in Backlog Candidates for the community process.

## Change summary

Expose the Find Widget's toggle states (case sensitive, whole word, regex) through the extension API so extensions can read or explicitly set them.

## Best solution

Add a proposed API surface — either on the text editor or as parameterized command variants — that lets extensions query and explicitly set the find widget's case-sensitive, whole-word, and regex toggle states. This would live in the editor API layer (`src/vs/workbench/api/`) and bridge to the find model's state in `src/vs/editor/contrib/find/`. Given low community traction, leave in Backlog Candidates until demand materializes.
