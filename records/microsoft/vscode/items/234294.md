[https://github.com/microsoft/vscode/issues/234294](https://github.com/microsoft/vscode/issues/234294)

## Summary

Valid feature request for a genuine API gap — the Find Widget's toggle states (case sensitive, whole word, regex) are not exposed via the extension API. Already triaged into Backlog with appropriate labels. No duplicate found, but low community traction (0 upvotes over 8 months).

## Change summary

Expose the Find Widget's toggle states (case sensitive, whole word, regex) through the VS Code extension API so extensions can read or set them programmatically.

## Best solution

This is a real API gap: `FindReplaceState` in `src/vs/editor/contrib/find/browser/findState.ts` tracks these toggles internally but nothing bridges them to `vscode.d.ts`. The extension API would need a new surface — either on `TextEditor` or via command arguments — to expose `isRegex`, `matchCase`, and `matchWholeWord`. Keep open for the API team to evaluate priority and design; the use case (extensions coordinating find behavior for multi-cursor workflows) is legitimate but niche.
