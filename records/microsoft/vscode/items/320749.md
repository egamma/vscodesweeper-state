# Issue #320749 — Git: detect newly added worktrees without requiring window reload

[https://github.com/microsoft/vscode/issues/320749](https://github.com/microsoft/vscode/issues/320749)

## Summary

Valid feature request to watch `.git/worktrees` for changes and auto-refresh the Source Control Repositories list. Includes a well-diagnosed secondary bug where prunable worktree entries inflate the count past `detectWorktreesLimit`, permanently disposing the status listener. No existing canonical issue covers this specific ask.

## Change summary

VS Code should detect newly created Git worktrees without requiring a window reload, by watching the `.git/worktrees` directory for changes and re-running worktree discovery.

## Best solution

Two changes in the git extension: (1) In `extensions/git/src/git.ts` `getWorktreesFS()`, filter out prunable/stale worktree entries (those whose `gitdir` target path no longer exists) so they don't count toward `detectWorktreesLimit`. (2) In `extensions/git/src/repository.ts`, add a file watcher on `.git/worktrees/` (the common git dir's worktrees folder) that triggers `checkForWorktrees()` when entries are added or removed, so new worktrees are discovered without a reload. The secondary bug (prunable entries inflating the limit) is a concrete defect worth fixing independently.

## Evidence

- **worktree discovery code:** `extensions/git/src/model.ts` lines 806–828: `checkForWorktrees()` only runs on `onDidRunGitStatus`, so new worktrees created externally are only discovered during a periodic status refresh — and only if the status listener hasn't been disposed.
- **prunable worktree bug:** `extensions/git/src/git.ts` lines 3026–3082: `getWorktreesFS()` reads all `.git/worktrees/*` entries without checking if the referenced path exists. Prunable/stale entries inflate the count past `detectWorktreesLimit` (default 50), causing the status listener to be disposed (model.ts line 818–819), permanently stopping worktree discovery.
- **no existing watcher:** The root watcher in `extensions/git/src/repository.ts` line 470 explicitly excludes `.git/worktrees/*/index.lock` but does not watch for new directory entries in `.git/worktrees/`.
- **community extension:** The reporter created a workaround extension: https://marketplace.visualstudio.com/items?itemName=Relequestual.git-worktree-refresh
- **related issue:** https://github.com/microsoft/vscode/issues/311189 reports the same symptom (worktrees created by agents not appearing) but is less detailed and unlabeled.
