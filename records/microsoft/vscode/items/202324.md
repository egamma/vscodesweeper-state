# Issue #202324 — scroll behavior in editor is different compared to native scrolling on macOS or other websites on chrome

[https://github.com/microsoft/vscode/issues/202324](https://github.com/microsoft/vscode/issues/202324)

## Summary

Team-authored tracking issue for macOS trackpad scroll acceleration feeling non-native. The underlying scroll curve mismatch remains unaddressed on current main, and recent upstream Electron/Chrome support (scrollBounce) has not been adopted yet.

## Change summary

Editor scroll acceleration on macOS trackpad feels off compared to native scrolling — the delta curve forces a tradeoff between precision and speed.

## Best solution

Investigate adopting Electron's `scrollBounce` (available since Chrome 145) and adjusting the custom scroll delta calculation in `src/vs/base/browser/ui/scrollbar/scrollableElement.ts` to better match the macOS native acceleration curve. The `overscroll-behavior: none` in workbench styles should be reconsidered per-element once native bounce is enabled.
