# Issue #234461 — Make "Reopen editor with" have a higher priority than the "editor.defaultBinaryEditor" setting.

[https://github.com/microsoft/vscode/issues/234461](https://github.com/microsoft/vscode/issues/234461)

## Summary

Still reproduces on main: the `openAsBinary` method in `textFileEditor.ts` unconditionally redirects to the `defaultBinaryEditor` without checking whether the user explicitly chose an editor via "Reopen editor with" (i.e., `options.override` is already set). The user's explicit editor choice should take priority.

## Change summary

When "Reopen editor with" is used on a binary-detected file, the `defaultBinaryEditor` setting overrides the user's explicit editor choice, making it impossible to reopen the file in a different editor without clearing the setting.

## Best solution

In `openAsBinary` (src/vs/workbench/contrib/files/browser/editors/textFileEditor.ts ~line 242), check whether `options?.override` is already set before falling back to `defaultBinaryEditor`. If `override` is present, honor it instead of redirecting to the configured default. This is a small, localized guard.
