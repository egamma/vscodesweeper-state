# Issue #315918 — Model picker: cost dots don't help users choose the right model for a task

[https://github.com/microsoft/vscode/issues/315918](https://github.com/microsoft/vscode/issues/315918)

## Summary

Substantive feature request for task-aware model recommendations in the model picker, going beyond cost indicators to help users pick the right model for their context. Not a duplicate of the completed model picker enhancements (#293875, #315892), which addressed cost display but not task-fit scoring. Routes to the chat/model-picker area for product evaluation.

## Change summary

Add task-aware model recommendations or nudges in the model picker so users can choose models based on task fit, not just cost.

## Best solution

Evaluate whether a lightweight task-fit signal (e.g. a 'recommended' badge or sorting by capability match) belongs in the model picker UX alongside the existing cost indicators. The detailed design proposal in the issue comment outlines a phased IModelRecommendationService approach — the product team should decide how much of that complexity is warranted vs. simpler heuristics like surfacing model capability tags.
