# Issue #283151 — Terminal Shell Integration requires update for newer PowerShell 7.4+ ConstrainedLanguage AUDIT mode

[https://github.com/microsoft/vscode/issues/283151](https://github.com/microsoft/vscode/issues/283151)

## Summary

Still reproduces on main: the PowerShell shell integration guard at line 12 of shellIntegration.ps1 blocks all non-FullLanguage modes, including the newer audit-only ConstrainedLanguage mode that is functionally equivalent to FullLanguage. Two open PRs exist but neither has merged.

## Change summary

Terminal shell integration fails to initialize under PowerShell 7.4+ ConstrainedLanguage AUDIT mode because the guard treats all non-FullLanguage modes as restricted.

## Best solution

Land one of the two open PRs (https://github.com/microsoft/vscode/pull/283178 or https://github.com/microsoft/vscode/pull/319252) that refine the language-mode guard in `src/vs/workbench/contrib/terminal/common/scripts/shellIntegration.ps1` to allow audit-mode ConstrainedLanguage while still blocking enforced ConstrainedLanguage. Review should ensure the `[System.Management.Automation.Security.SystemPolicy]::GetSystemLockdownPolicy()` check is robust and doesn't regress enforced CLM scenarios.
