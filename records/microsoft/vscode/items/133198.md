## Summary

Still valid: the TestRun.skipped() API on current main takes only a TestItem with no message parameter. The requested capability — attaching a reason/message to a skipped test — remains unimplemented.

## Change summary

Allow the TestRun.skipped() method to accept an optional TestMessage explaining why a test was skipped.
