# Issue #299786 — paste is slow with copilot

[https://github.com/microsoft/vscode/issues/299786](https://github.com/microsoft/vscode/issues/299786)

## Summary

Active performance regression: pasting text triggers a flood of inline completion requests from Copilot, causing 5–6 second freezes. Multiple users confirm across versions 1.110–1.119; disabling Copilot resolves it. No fix found on current main.

## Change summary

Pasting text into the editor triggers excessive inline completion requests from Copilot, causing multi-second UI freezes.

## Best solution

The inline completions infrastructure should debounce or suppress fetch requests triggered by paste operations (multi-character edits from a single paste source). The fix likely belongs in `src/vs/editor/contrib/inlineCompletions/browser/model/inlineCompletionsSource.ts` or in the Copilot extension's request handling — needs investigation whether the redundant requests originate from VS Code triggering per-character notifications or the extension not cancelling stale requests.
