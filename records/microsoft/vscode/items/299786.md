# Issue #299786 — paste is slow with copilot

[https://github.com/microsoft/vscode/issues/299786](https://github.com/microsoft/vscode/issues/299786)

## Summary

Still reproduces on main: paste triggers inline completion requests via `model.trigger()` with no paste-specific throttling or cancellation, causing multiple redundant Copilot requests and multi-second UI freezes. No fix has landed since the report.

## Change summary

Pasting text into the editor with Copilot enabled causes 5–6 second delays as inline completions are triggered repeatedly without debouncing.

## Best solution

Add paste-aware throttling or cancellation in the inline completions controller — either skip/debounce triggers originating from `onDidPaste` when the paste is multi-character, or cancel in-flight requests more aggressively when a new trigger arrives from a paste. The fix likely lives in `src/vs/editor/contrib/inlineCompletions/browser/controller/inlineCompletionsController.ts` (the paste handler at line ~215) and possibly the fetch/cancellation logic in `inlineCompletionsModel.ts`.
