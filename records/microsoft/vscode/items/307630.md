# Issue #307630 — Feature request: per-agent skill scoping (skills property in .agent.md frontmatter)

[https://github.com/microsoft/vscode/issues/307630](https://github.com/microsoft/vscode/issues/307630)

## Summary

Valid feature request: the standard `.agent.md` frontmatter supports `tools` scoping but has no equivalent `skills` property. A `skills` attribute already exists for Claude subagent definitions but not for the standard custom-agent format. Routing to the chat-prompts area for product consideration.

## Change summary

Allow `.agent.md` files to declare a `skills` frontmatter property that restricts which SKILL.md skills are visible to that agent, analogous to the existing `tools` property.

## Best solution

Extend `customAgentAttributes` in `src/vs/workbench/contrib/chat/common/promptSyntax/languageProviders/promptFileAttributes.ts` with a `skills` entry (similar to the existing Claude subagent `skills` attribute at line 1280), and wire up runtime filtering in the skill discovery/auto-loading path so that when an agent declares `skills`, only matching skills appear in the `/` menu and auto-loading. The Claude subagent path already has the attribute definition — the work is adding it to the standard agent format and implementing the filtering.
