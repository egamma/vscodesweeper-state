# Issue #326790 — Opening edit pills from the chat UI shows really long paths

[https://github.com/microsoft/vscode/issues/326790](https://github.com/microsoft/vscode/issues/326790)

## Summary

Team-authored bug report about edit pills in the chat UI opening editors with excessively long tab names. No fix found on current main; keeping open for maintainer attention.

## Change summary

Opening edit file pills from the chat UI results in editor tabs with very long paths, making them difficult to close.

## Best solution

The editor tab label shown when opening a file from a chat edit pill should use the compact/short resource label (basename or disambiguated short path) rather than the full absolute path. The fix likely lives in how the pill's click handler opens the editor — either in `chatEditPillElement.ts` or in the caller that constructs the editor input with a label override.
