[https://github.com/microsoft/vscode/issues/299967](https://github.com/microsoft/vscode/issues/299967)

## Summary

The crash is entirely within the Copilot Chat extension's own code — its chat-history parser calls `.match()` on an undefined command string. The full stack trace lives in `github.copilot-chat-*/dist/extension.js`, not in VS Code core. This belongs on the extension's issue tracker.

## Change summary

## Best solution

File this against https://github.com/microsoft/vscode-copilot-release (the Copilot Chat extension's issue tracker). The fix is a null guard in the extension's `yqi` function before calling `.match()` on the command string, as the commenter identified.
