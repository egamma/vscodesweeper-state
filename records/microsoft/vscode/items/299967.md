# Issue #299967 — Failed to open chat session: Cannot read properties of undefined (reading 'match')

[https://github.com/microsoft/vscode/issues/299967](https://github.com/microsoft/vscode/issues/299967)

## Summary

Still reproduces on main: `formatShellInvocationCompleted` in `copilotCLITools.ts` passes `toolCall.arguments.command` directly to `getCdPresentationOverrides` → `extractCdPrefix` without a null guard, crashing on `.match()` when a shell tool call has an undefined command. The sibling function `formatShellInvocation` already guards with `?? ''`.

## Change summary

Opening a Copilot CLI chat session crashes with 'Cannot read properties of undefined (reading match)' when a shell tool call in the history has an undefined command string.

## Best solution

Add a null guard for `toolCall.arguments.command` in `formatShellInvocationCompleted` (and defensively in `extractCdPrefix` itself), matching the `?? ''` pattern already used in the sibling `formatShellInvocation` function.
