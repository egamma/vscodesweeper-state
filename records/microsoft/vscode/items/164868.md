# Issue #164868 — GitHub - for VSCode not properly unlinked until VSCode restart

[https://github.com/microsoft/vscode/issues/164868](https://github.com/microsoft/vscode/issues/164868)

## Summary

Still reproduces on main: GithubRemoteSourceProvider caches user repos but never clears the cache when the authentication session changes. An open community PR (https://github.com/microsoft/vscode/pull/308927) already targets this fix.

## Change summary

After unlinking a GitHub account, the clone dialog still shows repositories from the unlinked account until VS Code is restarted, because the cached repo list is never invalidated on session change.

## Best solution

Review and land (or close) the open community PR https://github.com/microsoft/vscode/pull/308927, which adds an `onDidChangeSessions` listener to `GithubRemoteSourceProvider` to clear `userReposCache` on sign-out/sign-in. The fix is small and localized to `extensions/github/src/remoteSourceProvider.ts`.

- **existing pattern:** Other components in the same extension already listen to session changes — e.g. extensions/github/src/branchProtection.ts:131 and extensions/github/src/historyItemDetailsProvider.ts:92 both subscribe to `onDidChangeSessions` to reset their state.
- **open PR:** https://github.com/microsoft/vscode/pull/308927 (opened 2026-04-10, still open) by yogeshwaran-c adds the missing listener and cache invalidation. It references `Fixes #164868`.
- **maintainer confirmation:** Team member @TylerLeonhardt confirmed in comment 2 (2022-10-31) that the GitHub extension needs to listen to the session-change event, and that this is an issue in the GitHub extension, not the auth extension.
