# Issue #165034 — Improve telemetry ergonomics/perf

[https://github.com/microsoft/vscode/issues/165034](https://github.com/microsoft/vscode/issues/165034)

## Summary

All three proposed improvements (stricter event-name types, static-string lint rule, in-place payload cleansing) remain unimplemented on current main. The telemetry interface still types eventName as plain `string`, no telemetry-specific ESLint rule exists, and there is no in-place mutation spec. Worth keeping open as a valid polish item with team interest.

## Change summary

Improve telemetry API ergonomics by restricting event-name types, adding a lint rule for static-only strings, and specifying in-place payload cleansing for performance.

## Best solution

Implement the three improvements incrementally: (1) use a TypeScript template literal type for eventName to reject numbers/uppercase, (2) add an ESLint rule in `build/lib/eslint/` to flag string interpolation in publicLog calls, (3) document and implement in-place data cleansing in `src/vs/platform/telemetry/common/telemetryService.ts` to avoid payload copying. The number-in-property-name issue hediet reported (comment 3) may have been addressed by the data team, but the API-level guardrails proposed here are still missing.
