# Issue #229020 — `xdg-open <folder>` opens folder in VSCode instead of file browser since Ubuntu 24

[https://github.com/microsoft/vscode/issues/229020](https://github.com/microsoft/vscode/issues/229020)

## Summary

The root cause — VS Code's desktop file registering inode/directory as a MimeType — was fixed in commit f1c007109d0 (shipped in 1.89.0). The reporter's snap installation on 1.93.1 likely retained stale mime associations from a pre-fix version. This is the same underlying issue as #114425, which was closed as fixed.

## Change summary

Running `xdg-open <folder>` in VS Code's integrated terminal on Ubuntu 24 (snap) opens the folder in VS Code instead of the system file manager.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/114425 — the fix (removing `inode/directory` and `text/plain` from the desktop file's MimeType) already shipped in 1.89.0. Users with stale snap installations can fix it by running `xdg-mime default nautilus.desktop inode/directory` or reinstalling the snap to regenerate mime caches.
