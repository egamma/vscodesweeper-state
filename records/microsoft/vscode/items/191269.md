# Issue #191269 — Terminal Toggle Size to Content Width can stretch the canvas beyond the max texture size

[https://github.com/microsoft/vscode/issues/191269](https://github.com/microsoft/vscode/issues/191269)

## Summary

Already correctly triaged as upstream — the canvas texture-size limit is an xterm.js issue (xtermjs/xterm.js#4186) that VS Code cannot fix in core. Keeping open to track the upstream dependency.

## Change summary

Terminal canvas stretches beyond max GPU texture size when word wrap is toggled off with very long lines, causing rendering corruption.

## Best solution

Wait for the upstream fix in xterm.js (https://github.com/xtermjs/xterm.js/issues/4186) and pick it up in a future xterm dependency update. The workaround (Terminal: Set Fixed Dimensions) is already documented in the thread.
