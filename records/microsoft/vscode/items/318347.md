# Issue #318347 — chat.tools.terminal.autoApprove (and related terminal tool auto-approval) does not match inner commands when the command line uses common wrappers (timeout, bash -c, env, sudo time, etc.)

[https://github.com/microsoft/vscode/issues/318347](https://github.com/microsoft/vscode/issues/318347)

## Summary

Still reproduces on main: the auto-approve path uses `extractSubCommands` which only captures outermost tree-sitter `(command)` nodes, never unwrapping into inner commands behind wrappers like `timeout`, `bash -c`, `env`, etc. The wrapper-stripping logic in `terminalCommandParser.ts` exists but is not wired into the auto-approve flow.

## Change summary

Terminal auto-approval rules for inner commands (e.g. `git`) do not fire when the command line uses common wrappers (`timeout`, `bash -c`, `env`, `sudo`).

## Best solution

Extend `CommandLineAutoApproveAnalyzer.analyze` to also feed the wrapper-stripped effective commands from `terminalCommandParser.ts`'s `stripPrefixesAndWrappers` logic into the rule matcher, while preserving the `neverAutoApproveCommands` deny list. The security-critical path (deny before allow) must still evaluate both the outer and inner commands.
