# Issue #293400 — Discussion: Interest in cross-IDE WebAssembly extension standard?

[https://github.com/microsoft/vscode/issues/293400](https://github.com/microsoft/vscode/issues/293400)

## Summary

This is a solicitation to adopt an external cross-IDE WebAssembly extension standard (v0.1 draft from an unrelated project). VS Code's extension API is a core architectural pillar designed and maintained by the team; adopting an external standard would cede control of this surface. This is out of scope for VS Code core.

## Change summary

Adopt/align with a third-party cross-IDE WebAssembly extension standard for extension portability across editors.

## Best solution

Close as out-of-scope. VS Code's extension model is intentionally proprietary and tightly integrated with the workbench; cross-IDE portability of extensions is not a goal. The WASM work VS Code has done serves its own extension API, not an external standard. The external project is free to implement VS Code API compatibility on their side if they wish.
