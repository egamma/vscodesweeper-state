# Issue #199684 — Inline completion showing 1/0 or not showing 1/1 at all

[https://github.com/microsoft/vscode/issues/199684](https://github.com/microsoft/vscode/issues/199684)

## Summary

Team-authored bug, still reproducible from source. The hints widget displays `1/0` when `inlineCompletionsCount` is 0 because the label is unconditionally rendered as `${idx+1}/${count}` without guarding against an empty completions list.

## Change summary

The inline completions hints widget shows '1/0' when triggered explicitly with zero completions, and may not show '1/1' when exactly one completion exists.

## Best solution

Guard the count label in `inlineCompletionsHintsWidget.ts` (line ~267–269): when `suggestionCount === 0`, either hide the label or skip rendering the widget entirely. The index/count display should only render when `suggestionCount >= 1`.
