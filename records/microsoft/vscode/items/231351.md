# Issue #231351 — Configurable fast scroll to keys other than Alt

[https://github.com/microsoft/vscode/issues/231351](https://github.com/microsoft/vscode/issues/231351)

## Summary

Valid feature request to make the fast-scroll modifier key configurable instead of hardcoded to Alt. Still unimplemented on current main — the Alt check remains hardcoded in scrollableElement.ts. Only 7 upvotes, so not yet at the 20-vote backlog threshold; keeping open for community traction.

## Change summary

Allow users to configure which modifier key triggers fast scrolling (currently hardcoded to Alt).

## Best solution

Add a setting (e.g. `editor.fastScrollModifier`) accepting values like `alt`, `ctrl`, `shift`, or `none`, then use that in `src/vs/base/browser/ui/scrollbar/scrollableElement.ts` instead of the hardcoded `altKey` check. This mirrors the pattern used by `editor.multiCursorModifier`.
