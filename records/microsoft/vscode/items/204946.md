# Issue #204946 — Do not show complete content after reload if output is cleared 

[https://github.com/microsoft/vscode/issues/204946](https://github.com/microsoft/vscode/issues/204946)

## Summary

Still reproduces on main: clearing a LogOutputChannel only adjusts an in-memory offset that is lost on reload; the underlying log file is never truncated and no cleared-offset is persisted, so cleared content reappears after window reload.

## Change summary

Clearing a LogOutputChannel does not persist the clear state, causing previously-cleared logs to reappear after window reload.

## Best solution

Persist the clear offset (e.g. in workspace storage or as metadata alongside the log file) so that on reload, the FileContentProvider starts reading from the cleared position rather than offset 0. The fix likely touches `src/vs/workbench/contrib/output/common/outputChannelModel.ts` (FileOutputChannelModel.clear and OutputChannelBackedByFile) and possibly the output service initialization path.
