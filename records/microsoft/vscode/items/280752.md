# Issue #280752 — [Copilot] Editing message in "in progress" planning mode and accepting the changed message drops image attachements

[https://github.com/microsoft/vscode/issues/280752](https://github.com/microsoft/vscode/issues/280752)

## Summary

Team-authored bug — hard keep-open. The editing flow in `clickedRequest` collects attached context from the original request but may not properly restore image/binary attachments pasted from the clipboard, causing them to be silently dropped when the edited message is resubmitted. The mode also flips from planning to agent, suggesting a separate mode-preservation issue.

## Change summary

Editing a message in planning mode and resubmitting drops clipboard-pasted image attachments and switches to agent mode.

## Best solution

Fix the `clickedRequest` editing flow in `src/vs/workbench/contrib/chat/browser/widget/chatWidget.ts` to ensure binary/image variable entries from the original request's `attachedContext` are properly restored into the attachment model, and that the planning mode is preserved through the edit-resubmit cycle.

## Evidence

Image/binary attachments from clipboard paste may not survive this round-trip if they aren't serialized into `attachedContext` or if `addContext` doesn't handle binary entries.
