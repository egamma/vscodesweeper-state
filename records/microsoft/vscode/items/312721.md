# Issue #312721 — Disabled workspace trust isn't reflected in agents app

[https://github.com/microsoft/vscode/issues/312721](https://github.com/microsoft/vscode/issues/312721)

## Summary

Still appears unfixed on main: no commit directly addresses the scenario where `security.workspace.trust.enabled: false` should suppress the trust dialog in the agents window. The trust rework commits from June 2026 added trust gating but did not target this disabled-trust bypass. Keeping open as a real bug filed by a team contributor during testing.

## Change summary

When workspace trust is disabled in VS Code settings, the agents window still shows the trust dialog when opening a folder.

## Best solution

The agents window's `_onWorkspaceSelected` in `src/vs/sessions/contrib/chat/browser/newChatWidget.ts` should check `IWorkspaceTrustEnablementService.isWorkspaceTrustEnabled()` before calling `_requestFolderTrust`, or ensure the sessions window's `WorkspaceTrustEnablementService` correctly reads the user's `security.workspace.trust.enabled` setting at initialization time. Investigate whether the sessions-specific `ConfigurationService` loads user settings before the trust enablement check runs.
