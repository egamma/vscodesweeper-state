# Issue #292267 — code freezes when trying to manage a repo folder 

[https://github.com/microsoft/vscode/issues/292267](https://github.com/microsoft/vscode/issues/292267)

## Summary

The user asks for VS Code to only track repos with open files — this is already available via the `git.autoRepositoryDetection` setting set to `"openEditors"`. This is a support question, not a bug or missing feature.

## Change summary

VS Code freezes when opening a folder containing ~200 git repositories because all repos are discovered and tracked by default.

## Best solution

The user should set `"git.autoRepositoryDetection": "openEditors"` to limit repo discovery to only repos containing currently open files, which is the exact behavior they requested.
