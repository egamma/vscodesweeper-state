# Issue #96120 — [json] support setting schema associations at runtime

[https://github.com/microsoft/vscode/issues/96120](https://github.com/microsoft/vscode/issues/96120)

## Summary

This is a valid, canonical feature request for a public Extension API to register JSON schema associations at runtime. Internal machinery exists (registerSchemaAssociation in jsonContributionRegistry, the vscode://schemas-associations/ filesystem provider), but no public API is exposed to extensions. Issue #197035 was closed as a duplicate of this one, confirming it as the canonical tracker.

## Change summary

Extensions cannot programmatically set JSON schema associations at runtime — only the static `jsonValidation` contribution point in package.json is available.

## Best solution

Expose a public Extension API (e.g. on the `vscode.languages` or `vscode.workspace` namespace) that allows extensions to register/unregister JSON schema associations at runtime, leveraging the existing internal `registerSchemaAssociation` infrastructure in `src/vs/platform/jsonschemas/common/jsonContributionRegistry.ts` and the `vscode://schemas-associations/` update flow.
