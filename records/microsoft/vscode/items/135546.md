# Issue #135546 — Jittery ssh host input box

[https://github.com/microsoft/vscode/issues/135546](https://github.com/microsoft/vscode/issues/135546)

## Summary

The core timing issue persists on current main: in QuickPick.doSetValue(), the list is filtered/highlighted before the onDidChangeValue event fires to extensions, so extensions always see stale highlights for one frame before they can update items. This is still a valid visual bug in the quick pick API.

## Change summary

Quick pick items visually jitter on each keypress because filtering/highlighting runs before extensions can update item labels via onDidChangeValue.

## Best solution

Defer filtering until after the onDidChangeValue event fires (or batch the filter+item-update into a single render pass) in src/vs/platform/quickinput/browser/quickInput.ts, around the doSetValue method. This would let extensions update items before highlighting recalculates, eliminating the one-frame stale-highlight flash.
