# Issue #240139 — Terminal suggest: fig generator completions often complain about the executable not existing

[https://github.com/microsoft/vscode/issues/240139](https://github.com/microsoft/vscode/issues/240139)

## Summary

Still reproduces on current main — the ENOENT error when fig generators try to execute commands (e.g. `npm global remove`) on Windows has no targeted fix yet. The cwd/env fallback fix (ed6dac518c2, Aug 2025) addressed a related issue but not this one; author confirmed it persists after that fix.

## Change summary

Fig generator completions on Windows throw ENOENT errors when the target executable isn't found in the expected path during suggestion generation.

## Best solution

The fig execute layer in `extensions/terminal-suggest/src/fig/execute.ts` needs to either resolve executables via PATH before spawning (especially on Windows where `.cmd`/`.exe` resolution matters), or gracefully handle ENOENT from the spawned process without surfacing an error to the user — likely in the generator runner in `extensions/terminal-suggest/src/fig/autocomplete/state/generators.ts`.
