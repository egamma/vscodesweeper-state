# Issue #262670 — Give time for file saves before refreshing git

[https://github.com/microsoft/vscode/issues/262670](https://github.com/microsoft/vscode/issues/262670)

## Summary

Niche feature request for triggering auto-save when the cursor enters the sidebar (or delaying git refresh). A team member engaged and suggested existing workarounds (`files.autoSave: afterDelay`). Already labeled `under-discussion` with the `git` area — the current state is appropriate.

## Change summary

Request to auto-save files when the mouse enters the sidebar, or delay the git refresh button, so that focus-out-based auto-save completes before the SCM view refreshes.

## Best solution

The existing `files.autoSave: afterDelay` setting covers most of this workflow. If the team decides to act, it would be a new `files.autoSave` trigger (e.g. on sidebar focus) in `src/vs/workbench/services/filesConfiguration/`. Given 0 upvotes and an available workaround, leaving it in `under-discussion` for the community process is the right call.
