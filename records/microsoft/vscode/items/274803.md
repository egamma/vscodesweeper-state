# Issue #274803 — 1.106.x - Github Copilot run_in_terminal - fails to capture output when the terminal uses some fancy plugins or PS1 (e.g. zsh and aws plugin) (worked in VS Code 1.103)

[https://github.com/microsoft/vscode/issues/274803](https://github.com/microsoft/vscode/issues/274803)

## Summary

Valid open bug — the AWS zsh plugin's RPROMPT interferes with run_in_terminal output capture. Reporter confirmed still reproducible in Dec 2025. Recent zsh fixes (bc33377584e, May 2026) address different issues (bang history/interactive comments), not the RPROMPT output detection conflict. The `confirmation-pending` label is stale since the reporter already re-confirmed the bug.

## Change summary

Copilot's run_in_terminal tool fails to capture terminal output when zsh's RPROMPT is set by plugins like the ohmyzsh AWS plugin.

## Best solution

Fix the command output detection in the shell integration layer to handle RPROMPT content from plugins like ohmyzsh/aws that inject region info. The right prompt wrapping logic in `shellIntegration-rc.zsh` (lines 271–327) already exists but doesn't prevent the output capture failure. The chat terminal profile workaround (disabling the plugin via env var) works but shouldn't be necessary. Likely needs investigation in the interaction between RPROMPT sequences and the command detection/output boundaries in `shellIntegration-rc.zsh` and the terminal command detection code.
