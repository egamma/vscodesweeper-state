# Issue #95021 — Task terminal truncates long/quick output.

[https://github.com/microsoft/vscode/issues/95021](https://github.com/microsoft/vscode/issues/95021)

## Summary

Still a valid open bug — the 250ms DataFlushTimeout workaround in terminalProcess.ts does not fully prevent output truncation when a process emits large amounts of data quickly before exiting. The root cause (node-pty not guaranteeing all data is flushed before onExit) remains unresolved on current main.

## Change summary

Task terminal output is truncated when commands produce large or fast output because the pty process fires exit before all data is delivered.

## Best solution

The fix likely lives in node-pty itself (microsoft/node-pty#72) — a reliable 'all data flushed' signal before exit. The VS Code workaround (DataFlushTimeout in src/vs/platform/terminal/node/terminalProcess.ts) could be made more robust, but the real solution is upstream. Consider consolidating with https://github.com/microsoft/vscode/issues/92868 which tracks the same root cause from the CustomExecution/PseudoTerminal angle.
