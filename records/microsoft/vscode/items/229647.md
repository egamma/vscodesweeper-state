# Issue #229647 — Undo stacks are not granular enough for Korean

[https://github.com/microsoft/vscode/issues/229647](https://github.com/microsoft/vscode/issues/229647)

## Summary

Already triaged feature request with 'help wanted' label, team endorsement, and Backlog milestone — properly routed and awaiting a community contribution.

## Change summary

Undo in Korean IME removes the entire line (or word on Windows) instead of the last consonant/vowel or last fully formed character.

## Best solution

Improve undo granularity for Korean (and potentially other composing IME languages) in the editor input handling layer, likely in the EditContext or TextAreaInput composition handling, so undo removes the last composed syllable component rather than the entire composition session.
