# Issue #206511 — Search pane: F4 after editing should visit the next search entry

[https://github.com/microsoft/vscode/issues/206511](https://github.com/microsoft/vscode/issues/206511)

## Summary

Confirmed bug still present on main: after editing a file, the search tree loses its selection, so F4 (selectNextMatch) restarts from the top instead of continuing from the previous position. An open community PR (#209228) exists but has not been merged.

## Change summary

F4 in the search pane restarts navigation from the first result after any edit, instead of continuing from the previously selected match.

## Best solution

Review and land (or close with feedback) the open community PR https://github.com/microsoft/vscode/pull/209228 which targets exactly this issue. The fix likely belongs in `src/vs/workbench/contrib/search/browser/searchView.ts` and/or `searchModel.ts` — when results refresh due to a model change, the tree should preserve or shift selection to the nearest surviving match so that `selectNextMatch` resumes from the right position.

## Evidence

- **bug confirmed in source:** In src/vs/workbench/contrib/search/browser/searchView.ts line 1134–1143, `selectNextMatch` calls `this.tree.navigate(selected)` where `selected` comes from `this.tree.getSelection()`. After an edit triggers a result refresh, the tree selection is cleared, so `selected` is undefined and navigation restarts from the top.
- **open PR:** https://github.com/microsoft/vscode/pull/209228 is still open (not merged) and targets this exact issue. It was authored by a community contributor in March 2024.
- **maintainer confirmation:** Team member @andreamah confirmed the bug in comment 4, identifying the root cause: result refresh on model change clears the tree selection, and selectNextMatch requires a selected element to navigate from.
- **duplicates closed against this:** Issues https://github.com/microsoft/vscode/issues/221840 and https://github.com/microsoft/vscode/issues/224113 were closed as duplicates of this issue, confirming it is the canonical tracker.
