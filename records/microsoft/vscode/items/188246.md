# Issue #188246 — incorrect indentation detection

[https://github.com/microsoft/vscode/issues/188246](https://github.com/microsoft/vscode/issues/188246)

## Summary

The indentation guesser is intentionally language-agnostic and considers all lines. The closely related issue #244494 (which explicitly references this issue) was closed as *as-designed by alexdima, with the same underlying cause: HTML formatter continuation lines produce 8-space indentation that the guesser picks up as a valid diff.

## Change summary

After formatting an HTML file with 4-space indentation, reopening detects 8-space indentation because the guesser considers HTML continuation lines indented at 8 spaces.

## Best solution

Close as *as-designed consistent with the team's decision on https://github.com/microsoft/vscode/issues/244494. Users who need deterministic indentation should use `.editorconfig` or explicit workspace settings.
