# Issue #188246 — incorrect indentation detection

[https://github.com/microsoft/vscode/issues/188246](https://github.com/microsoft/vscode/issues/188246)

## Summary

Still reproducible on main: the indentation guesser counts more 8-space diffs than 4-space diffs for HTML files with continuation-indented wrapped inline content, incorrectly detecting tabSize=8 instead of 4.

## Change summary

After formatting an HTML file with 4-space indentation, reopening it causes VS Code to detect indentation as Spaces: 8 because wrapped inline element content produces 8-space diffs that outnumber the 4-space diffs.

## Best solution

Add a heuristic in the indentation guesser (similar to the existing 2-vs-4 preference) that recognizes 8-space diffs as likely being 2 levels of 4-space indentation. A ratio-based check (e.g., if tabSize guessed 8 and spacesDiffCount[4] > 0, prefer 4 when 4 is a divisor of 8) would address this class of false positives.
