# Issue #156656 — [html] Duplicate double quotes are inserted for HTML attributes when working over SSH

[https://github.com/microsoft/vscode/issues/156656](https://github.com/microsoft/vscode/issues/156656)

## Summary

The auto-quote insertion race condition over SSH is still present on main. The version check in autoInsertion.ts can pass in edge timing windows with high-latency remote connections, causing duplicate quotes.

## Change summary

When typing `=` followed quickly by `"` in an HTML attribute over SSH, duplicate double quotes are inserted due to a race between user input and the auto-quote provider response.

## Best solution

The auto-insert logic in `extensions/html-language-features/client/src/autoInsertion.ts` should verify that the character immediately before the cursor is still `=` (not already `"`) before inserting the auto-quote snippet, or cancel the pending auto-insert when a subsequent content change arrives that would conflict.
