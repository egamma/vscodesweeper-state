# Issue #156656 — [html] Duplicate double quotes are inserted for HTML attributes when working over SSH

[https://github.com/microsoft/vscode/issues/156656](https://github.com/microsoft/vscode/issues/156656)

## Summary

Still present on main: the HTML auto-quote insertion in autoInsertion.ts has a race condition over high-latency connections where the version check doesn't protect against user input delayed by SSH latency.

## Change summary

Typing `=` followed quickly by `"` in an HTML attribute over SSH causes duplicate double quotes because the auto-quote insertion races with the user's own quote keystroke.

## Best solution

The auto-quote logic in `extensions/html-language-features/client/src/autoInsertion.ts` should cancel the pending auto-insert when the user types a `"` immediately after `=`, or re-check the document state after the provider responds to detect that a quote was already typed by the user.
