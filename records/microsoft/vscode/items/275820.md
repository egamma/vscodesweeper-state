# Issue #275820 — SCM Graph - incoming changes sometimes show whole file instead of diff

[https://github.com/microsoft/vscode/issues/275820](https://github.com/microsoft/vscode/issues/275820)

## Summary

The rendering sub-issue (icon display) was fixed in PR #276055 and tracked separately in #276064, but the core report — incoming changes sometimes showing the whole file instead of a diff — remains unaddressed and intermittently reproducible. Keeping open as a valid Backlog bug that needs better reproduction steps.

## Change summary

Clicking 'Open Changes' on the SCM Graph incoming changes node sometimes shows the entire file instead of a diff view.

## Best solution

The likely root cause is in `src/vs/workbench/contrib/scm/browser/scmHistoryViewPane.ts` where the merge-base resolution for incoming changes may intermittently fail or return an unexpected value, causing the multi-diff editor to fall back to showing the full file. A maintainer who can reproduce this should trace the `resolveHistoryItemRefsCommonAncestor` path and verify the `historyItemParentId` is correctly computed in all merge/sync states.
