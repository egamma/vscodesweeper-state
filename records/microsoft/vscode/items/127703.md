# Issue #127703 — Add cursor stops around inlay hints

[https://github.com/microsoft/vscode/issues/127703](https://github.com/microsoft/vscode/issues/127703)

## Summary

Valid open feature request: the injected-text engine already supports `InjectedTextCursorStops.Both`, but the inlay hints controller hardcodes `Right`/`None` — the ask is to expose configurable cursor-stop behavior. Team has discussed but not shipped a resolution; 12 upvotes, active community interest.

## Change summary

Allow cursor stops on both sides of inlay hints (or make the behavior configurable) so navigation around hints feels natural.

## Best solution

Add a setting (e.g. `editor.inlayHints.cursorStops`) that controls whether cursor stops appear on both sides, only start, only end, or directionally — the plumbing already exists in `InjectedTextCursorStops`. The fix lives in `src/vs/editor/contrib/inlayHints/browser/inlayHintsController.ts` where `InjectedTextCursorStops.Right`/`None` are currently hardcoded.
