# Issue #307695 — Source Control Graph shows repository not visible in Source Control view

[https://github.com/microsoft/vscode/issues/307695](https://github.com/microsoft/vscode/issues/307695)

## Summary

Team-authored issue; confirmed in source. The SCM Graph view model resolves its repository from `scmService.repositories`/`onDidAddRepository` (lines 1152–1156 of scmHistoryViewPane.ts), bypassing the `visibleRepositories` filtering that the Repositories and Changes sections respect. The bug is still present on current main — no fix commit found.

## Change summary

The Source Control Graph section displays a repository registered by an extension even when that repository is not visible in the SCM view's Repositories/Changes sections.

## Best solution

In `SCMHistoryViewModel` (scmHistoryViewPane.ts), the `firstRepository` and fallback logic should filter through `scmViewService.visibleRepositories` (or equivalent) rather than reading directly from `scmService.repositories`/`onDidAddRepository`, so the Graph section stays consistent with the rest of the Source Control view.
