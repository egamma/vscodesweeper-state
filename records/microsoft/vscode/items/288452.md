# Issue #288452 — Commit & Push pushes to the tracked branch rather than the checked out branch

[https://github.com/microsoft/vscode/issues/288452](https://github.com/microsoft/vscode/issues/288452)

## Summary

Team-authored issue — keeping open for maintainer judgment. The reported behavior is traceable in the current source: `repository.push()` constructs the refspec as `localBranch:upstreamName`, which pushes to the upstream tracking branch rather than a same-named remote branch. This diverges from `git push origin HEAD` semantics when the local branch name differs from its upstream.

## Change summary

Commit & Push pushes to the upstream tracking branch name on the remote rather than a remote branch matching the local branch name, which differs from `git push origin HEAD` behavior.

## Best solution

Decide whether VS Code's push should honor `push.default` / support `current`-style push semantics (push to same-named remote branch) alongside the current upstream-tracking behavior. The fix site is `extensions/git/src/repository.ts` in the `push()` method (line ~2355–2358) where the refspec `${head.name}:${head.upstream.name}` is constructed. At minimum, the UI should warn when the upstream branch name differs from the local branch name.
