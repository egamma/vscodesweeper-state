# Issue #157633 — Extensions list is empty on a disconnected window bug report screen

[https://github.com/microsoft/vscode/issues/157633](https://github.com/microsoft/vscode/issues/157633)

## Summary

Still reproduces on main: the issue reporter uses `IExtensionManagementService.getInstalled()` which calls `Promise.all` across all servers (including remote) without error handling — when the remote connection is dead, it throws and the extensions list is empty. The fix sandy081 offered in comments (an arg to not bail when a server fails) was never implemented.

## Change summary

The issue reporter's extension dropdown is empty when a remote SSH connection has died, because `getInstalled()` throws on a dead remote server.

## Best solution

Make `ExtensionManagementService.getInstalled()` (in `src/vs/workbench/services/extensionManagement/common/extensionManagementService.ts`) resilient to individual server failures — use `Promise.allSettled` or add a flag to skip failed servers, as sandy081 suggested in the comments. The issue reporter callers already have try/catch but get nothing because the entire call fails; the fix belongs in the management service so all callers benefit.
