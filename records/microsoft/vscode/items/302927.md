# Issue #302927 — To Do Tool items show unchecked on reload until you expand thinking block

[https://github.com/microsoft/vscode/issues/302927](https://github.com/microsoft/vscode/issues/302927)

## Summary

Team-authored bug (hard keep-open). The todo list status update is coupled to tool-invocation part rendering inside thinking blocks — on reload, collapsed thinking blocks defer rendering, so `setTodos()` never fires until the user expands the block.

## Change summary

To Do Tool items show as unchecked after VS Code reload until the collapsed thinking block containing the status-update tool invocation is expanded.

## Best solution

Decouple the todo-list state hydration from DOM rendering. On session restore, the todo state should be read from persisted storage or eagerly evaluated from deserialized tool-invocation data, rather than relying on `ChatToolInvocationPart` constructor side-effects in `chatToolInvocationPart.ts` (line ~81-93). The `ChatTodoListStorage` already persists via `Memento`, so either (a) ensure the storage round-trip preserves the last-written state correctly across reload, or (b) eagerly walk deserialized tool invocations at model-restore time to replay todo updates before any rendering occurs.
