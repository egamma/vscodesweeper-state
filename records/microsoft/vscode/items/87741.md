# Issue #87741 — Auto save & new line prevents undo from working

[https://github.com/microsoft/vscode/issues/87741](https://github.com/microsoft/vscode/issues/87741)

## Summary

This should stay open as a real undo/save-participant bug: the reporter corrected the setting to `files.insertFinalNewline`, clarified that the problem is the save participant entering the undo stack, and later confirmed it still reproduced in VS Code 1.51.0. There is not enough concrete evidence in the provided context to close it as fixed or duplicate, and the current labels already route it to undo/redo.

## Change summary

With auto save and `files.insertFinalNewline`, undo at EOF removes the automatically inserted final newline instead of undoing the user's last edit or paste.

## Best solution

The undo stack should not expose the automatic final-newline save participant as the next user undo step; either merge it with the triggering edit or exclude it from undo history in the same spirit as the save-participant undo fixes discussed for https://github.com/microsoft/vscode/issues/90100. Keep this tracked under the undo/save participant area until current behavior is verified or a concrete fixing commit is identified.

- **related issue:** The reporter linked https://github.com/microsoft/vscode/issues/90100 as recognizing automatic on-save formatting actions entering the undo stack as a bug, while noting that `files.insertFinalNewline` apparently was not covered by that fix.
