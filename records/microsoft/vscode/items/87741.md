# Issue #87741 — Auto save & new line prevents undo from working

[https://github.com/microsoft/vscode/issues/87741](https://github.com/microsoft/vscode/issues/87741)

## Summary

Keep open as a real undo/save-participant bug: current code still records the `files.insertFinalNewline` save participant as an edit separated by undo stops. The later throttling fix for https://github.com/microsoft/vscode/issues/102542 helps the immediate re-save loop, but it does not make the inserted final newline undo together with the user's edit.

## Change summary

With `files.autoSave` and `files.insertFinalNewline`, undo first removes the save participant's final newline instead of undoing the user's last edit.

## Best solution

Adjust the text save participant/undo interaction so `FinalNewLineParticipant` changes do not become the blocking top-level undo step after auto save, likely in `src/vs/workbench/contrib/codeEditor/browser/saveParticipants.ts` or the working-copy save participant undo-stop handling.

## Evidence

- **current implementation:** `src/vs/workbench/contrib/codeEditor/browser/saveParticipants.ts:121` still runs `FinalNewLineParticipant` for `files.insertFinalNewline`, and lines 135-140 apply the newline through `editor.executeEdits('insertFinalNewLine', ...)` or `model.pushEditOperations(...)`, both undo-stack-aware paths.
- **undo stops:** `src/vs/workbench/services/workingCopy/common/storedFileWorkingCopySaveParticipant.ts:40` and `:87` push undo stops before and after save participation, so the save participant edit can remain a distinct undo step.
- **related fix is partial:** Commit 69fd7afc56b4c4f793981d6b0cfbeb06cc1ae613 fixed https://github.com/microsoft/vscode/issues/102542 by throttling auto-save participants after undo/redo; current main carries that logic at `src/vs/workbench/services/workingCopy/common/storedFileWorkingCopy.ts:741` and `:964`, but it does not combine or suppress the final-newline edit in the undo stack.
- **reporter follow-up:** The reporter clarified that the intended behavior of inserting a final newline is fine; the bug is that the automatic newline becomes the last undo action. They also noted the issue persisted in VS Code 1.51.0 after the earlier https://github.com/microsoft/vscode/issues/102542 fix had shipped.
