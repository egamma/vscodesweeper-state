# Issue #87741 — Auto save & new line prevents undo from working

[https://github.com/microsoft/vscode/issues/87741](https://github.com/microsoft/vscode/issues/87741)

## Summary

Still reproducible on main: FinalNewLineParticipant uses editor.executeEdits() which pushes undo stops, making the auto-inserted newline a separate undo entry that immediately re-appears on save. The fix for #90100 addressed format-on-save but not this save participant.

## Change summary

When `files.insertFinalNewline` and `files.autoSave: afterDelay` are both enabled, undoing a user edit at the end of the file undoes the auto-inserted newline instead, which is then immediately re-added on save — making the real user edit effectively un-undoable.

## Best solution

Change `FinalNewLineParticipant.doInsertFinalNewLine` to use `model.pushEditOperations` (like the trim-whitespace participant) instead of `editor.executeEdits`, so the inserted newline doesn't create a separate undo stop. This matches the pattern established by the #90100 fix for format-on-save.
