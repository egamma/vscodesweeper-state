# Issue #287857 — New background agent sessions defaulting to workspace

[https://github.com/microsoft/vscode/issues/287857](https://github.com/microsoft/vscode/issues/287857)

## Summary

Team-authored issue — must stay open for maintainer judgment. Reports that new background agent sessions default to 'Workspace' instead of 'Worktree' in a git-backed repo where worktrees are already in use.

## Change summary

New background agent chat sessions default to Workspace isolation instead of Worktree, even in repos that already use git worktrees for other background agents.

## Best solution

The agent session creation flow should detect when the repo uses git worktrees (or when the user has previously used worktrees for background agents in the same repo) and default to Worktree isolation accordingly. Likely lives in the agents window / chat session initialization code.
