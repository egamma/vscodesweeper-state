# Issue #185135 — "self signed certificate in certificate chain" while attempting to install an extension via command line

[https://github.com/microsoft/vscode/issues/185135](https://github.com/microsoft/vscode/issues/185135)

## Summary

Valid, maintainer-acknowledged feature request to auto-load system CA certificates for the CLI extension-install path. Work has progressed (http.systemCertificatesNode setting exists since Nov 2025) but it defaults to false, and the CLI doesn't auto-detect system cert stores. chrmarti confirmed tracking it here.

## Change summary

CLI extension installation via `--install-extension` fails with 'self signed certificate in certificate chain' behind corporate proxies because the Node.js request path doesn't automatically load system CA certificates the way Electron does.

## Best solution

Enable automatic system certificate loading for the CLI code path (src/vs/code/node/cliProcessMain.ts → RequestService) — either default `http.systemCertificatesNode` to `true` or wire system cert loading into the CLI's request pipeline independently of user settings. The proxy-agent infrastructure already supports this via `loadSystemCertificates`.
