# Issue #238361 — `vscode.workspace.fs.copy` does not copy folders properly on github.dev

[https://github.com/microsoft/vscode/issues/238361](https://github.com/microsoft/vscode/issues/238361)

## Summary

Legitimate bug: cross-provider copy in fileService.ts correctly handles directory traversal (lines 843–850), but on github.dev the source provider for extension URIs (served via CDN/unpkg) likely does not report directories as `isDirectory: true` in its `stat` implementation, causing the copy to treat the folder as a file and write the CDN's JSON directory listing as file content. No fix found on main.

## Change summary

When using `vscode.workspace.fs.copy` to copy a folder from extension assets to the workspace on github.dev, the folder is treated as a file and its JSON directory listing is written instead of recursively copying folder contents.

## Best solution

The fix likely belongs in the web extension filesystem provider that handles extension resource URIs on github.dev — its `stat` implementation needs to correctly identify directories so the cross-provider copy path in `src/vs/platform/files/common/fileService.ts` (line 844–846) enters the `doCopyFolder` branch. The extension resource provider may live outside this repo (e.g. in the vscode-remote-web infrastructure), in which case this issue should be routed to that team.
