# Issue #240711 — Missing border under window buttons when the application window is maximized

[https://github.com/microsoft/vscode/issues/240711](https://github.com/microsoft/vscode/issues/240711)

## Summary

Three team members on different Windows builds were unable to reproduce this titlebar border issue at non-100% text scaling. The reporter's last comment (May 2025) added useful detail about specific scaling percentages but no one has confirmed the bug since, and both Windows and Electron have been updated substantially. Requesting confirmation on the latest Insiders build.

## Change summary

A 1px border under the minimize/maximize/close window buttons disappears when the window is maximized and certain Windows text-scaling percentages (103%, 104%, 106%, 107%) are active.

## Best solution

This is likely a Chromium compositor subpixel-rounding artifact at fractional DPI values — the pattern of some percentages working and others not is characteristic. Ask the reporter to retest on the latest Insiders (which ships a newer Electron/Chromium) with the same Windows scaling; if it persists, file upstream against Chromium's DWM/compositor layer and track here with the `upstream` label.
