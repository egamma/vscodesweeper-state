# Issue #229357 — [html] Vscode detect wrong scope for script with type=module

[https://github.com/microsoft/vscode/issues/229357](https://github.com/microsoft/vscode/issues/229357)

## Summary

Still reproduces on main: the HTML extension's embedded document generation treats `<script type="module">` identically to regular `<script>`, concatenating all script contents into a single virtual JS document without scope isolation. An open community PR (#308027) proposes a fix but has not been merged.

## Change summary

Variables declared in `<script type="module">` tags incorrectly trigger 'Cannot redeclare block-scoped variable' errors because VS Code's HTML language server doesn't isolate module scripts into their own scope in the virtual JS document.

## Best solution

In `extensions/html-language-features/server/src/modes/embeddedSupport.ts`, track whether a region came from a `type="module"` script, and in the virtual document generation (`getEmbeddedDocument`), wrap module-script content in block-scope delimiters so TypeScript's checker treats each module script independently. PR https://github.com/microsoft/vscode/pull/308027 takes this approach.
