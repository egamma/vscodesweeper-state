# Issue #229357 — [html] Vscode detect wrong scope for script with type=module

[https://github.com/microsoft/vscode/issues/229357](https://github.com/microsoft/vscode/issues/229357)

## Summary

Still reproduces on main: all <script> blocks are concatenated into a single virtual JS document in embeddedSupport.ts regardless of type="module", so TypeScript sees shared scope and reports false redeclaration errors.

## Change summary

VS Code's HTML language server incorrectly reports 'Cannot redeclare block-scoped variable' when the same variable is declared in a regular <script> and a <script type="module">, because module scripts should have their own scope.

## Best solution

The HTML language server's embedded document extraction (extensions/html-language-features/server/src/modes/embeddedSupport.ts) needs to treat <script type="module"> blocks as separate virtual documents (or mark them as ES modules for TypeScript), so each module script gets its own scope and doesn't share declarations with other script blocks.
