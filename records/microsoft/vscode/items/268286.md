# Issue #268286 — MCP ‘mcp.servers’ config in .code-workspace is shown as Unknown Configuration Setting / grayed out

[https://github.com/microsoft/vscode/issues/268286](https://github.com/microsoft/vscode/issues/268286)

## Summary

Still reproduces on main: `WorkspaceConfigurationModelParser` parses and merges `launch` and `tasks` models from the workspace file top-level keys, but never creates or merges an `mcp` model — so `mcp.servers` is invisible to the configuration system and shows as unknown in the settings editor.

## Change summary

MCP server configuration in `.code-workspace` top-level `mcp` key is not recognized by the settings editor (shown as 'Unknown Configuration Setting' / grayed out).

## Best solution

Add `_mcpModel` to `WorkspaceConfigurationModelParser` (analogous to `_launchModel`/`_tasksModel`) and merge it into `workspaceSettings` in the workspace configuration classes, so the configuration system surfaces `mcp.*` values from the workspace file.
