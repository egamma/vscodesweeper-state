# Issue #296151 — [Bug] “Word wrap” lacks `overflow-wrap: break-word`, for ligatures whose widths exceed that of the viewport.

[https://github.com/microsoft/vscode/issues/296151](https://github.com/microsoft/vscode/issues/296151)

## Summary

Confirmed bug with team engagement (alexdima) and an open fix PR (#309258). The DOM line-breaks computer uses `overflow-wrap: break-word` with `word-break: inherit`, which doesn't break wide ligatures; `word-break: break-all` is the identified fix. Keep open pending PR review.

## Change summary

Word wrap fails to break lines within ligatures (e.g., repeated Tamil ௌ U+0BCC) whose rendered width exceeds the viewport, causing horizontal overflow.

## Best solution

Merge the open PR https://github.com/microsoft/vscode/pull/309258 which changes `word-break` from `inherit` to `break-all` in the DOM line-breaks measurement container, allowing browsers to find break points within wide ligature glyphs.
