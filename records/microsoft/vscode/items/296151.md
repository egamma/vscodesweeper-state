# Issue #296151 — [Bug] “Word wrap” lacks `overflow-wrap: break-word`, for ligatures whose widths exceed that of the viewport.

[https://github.com/microsoft/vscode/issues/296151](https://github.com/microsoft/vscode/issues/296151)

## Summary

Keep open: the literal title is partly inaccurate because current main already applies `overflow-wrap: break-word`, but the reported gap remains because the advanced wrapping DOM probe does not use the reporter's suggested `word-break: break-all`. There is not enough evidence for a close, duplicate, extension cause, or upstream-only handoff.

## Change summary

A very wide Tamil ligature sequence can exceed the editor viewport without wrapping, even with word wrap and `editor.wrappingStrategy: advanced` enabled.

## Best solution

Editor wrapping owners should decide whether `src/vs/editor/browser/view/domLineBreaksComputer.ts` can safely use `word-break: break-all` for this case, or add a narrower fallback/test for over-wide grapheme or ligature runs without regressing normal CJK word-break behavior.
