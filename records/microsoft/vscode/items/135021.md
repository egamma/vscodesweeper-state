# Issue #135021 — Extensions keeping the extension host busy lead to disconnection

[https://github.com/microsoft/vscode/issues/135021](https://github.com/microsoft/vscode/issues/135021)

## Summary

This appears to have been fixed by the persistent protocol change that prevents reconnects when the event loop is busy rather than truly disconnected. The issue matches the fixed canonical scenario in https://github.com/microsoft/vscode/issues/83672, and current main still contains that mitigation.

## Change summary

Remote windows could show false disconnection/reconnection prompts when tokenization or an extension kept the remote extension host busy for long enough to miss protocol acknowledgements.

## Best solution

Treat this as completed by commit 45c70c29e572de3e9ca24d993f0616fd31f05029, which avoids declaring the persistent socket timed out while the process is under high event-loop load; any remaining modern repro should be filed with fresh logs against current Insiders.
