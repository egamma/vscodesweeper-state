# Issue #135021 — Extensions keeping the extension host busy lead to disconnection

[https://github.com/microsoft/vscode/issues/135021](https://github.com/microsoft/vscode/issues/135021)

## Summary

Keep open: the report describes a real remote-connection failure mode where a busy remote extension host can still look like a dropped connection, and I do not see a specific fixing commit for this issue. Related timeout behavior has changed since the report, but the current code still treats long-unacknowledged protocol messages as a socket timeout unless the local process is under high load.

## Change summary

Remote SSH/WSL sessions show reconnect/disconnect dialogs when extensions or large-file processing keep the remote extension host busy.

## Best solution

Remote connection handling should distinguish a busy remote extension host from a dead transport, ideally in the persistent protocol/remote extension host connection path, and avoid tearing down port forwards or prompting reconnection for extension-host stalls.
