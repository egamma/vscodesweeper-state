# Issue #107246 — VSCode's FileDialog overwrites the file managers (Nautilus) "Show hidden files" setting

[https://github.com/microsoft/vscode/issues/107246](https://github.com/microsoft/vscode/issues/107246)

## Summary

This is a confirmed upstream GTK3/Electron bug where the file chooser dialog overwrites the system's 'show hidden files' dconf setting. The upstream Electron issues (electron#34706 and electron#46817) were both closed by stale bots without a fix. VS Code forces GTK3 dialogs via `xdg-portal-required-version`, which triggers this behavior. The issue is correctly labeled upstream and should remain open as a tracking issue.

## Change summary

Opening VS Code's file dialog on Linux/GNOME resets the system-wide 'show hidden files' setting in Nautilus/GTK file chooser.

## Best solution

The root cause is in GTK3's `gtk_file_chooser_set_show_hidden()` being called by Electron without respecting the existing dconf setting. Best path forward: re-open or file a new Electron issue (electron#46817 was closed by stale bot, not fixed), or investigate whether VS Code can pass `showHiddenFiles` in dialog properties to preserve the user's system setting. The VS Code-side workaround would be reading `org.gtk.Settings.FileChooser show-hidden` via dconf before opening dialogs and passing it through Electron's `properties` array.
