# Issue #201045 — Replace in files global command should move focus

[https://github.com/microsoft/vscode/issues/201045](https://github.com/microsoft/vscode/issues/201045)

## Summary

Still reproduces on main: `findOrReplaceInFiles` passes `updatedText` as the `focusReplace` parameter instead of properly focusing the replace input when the replace widget is already shown and the command is re-invoked.

## Change summary

Running `workbench.action.replaceInFiles` when the search view is already visible opens the replace input but does not focus it.

## Best solution

In `findOrReplaceInFiles` (src/vs/workbench/contrib/search/browser/searchActionsNav.ts), when `expandSearchReplaceWidget` is true and the replace widget is already shown, pass `true` for the `focusReplace` parameter — similar to how the editor find widget toggles focus between search and replace inputs on repeated invocations.
