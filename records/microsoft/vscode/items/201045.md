# Issue #201045 — Replace in files global command should move focus

[https://github.com/microsoft/vscode/issues/201045](https://github.com/microsoft/vscode/issues/201045)

## Summary

Still reproduces on main: the `findOrReplaceInFiles` helper always focuses the search input (or conditionally based on `updatedText`), never toggling focus to the replace input when it is already visible. The issue is confirmed and labeled, sitting in On Deck.

## Change summary

Running `workbench.action.replaceInFiles` when the search view is already open with the replace widget visible does not move focus to the replace input.

## Best solution

In `findOrReplaceInFiles` in `src/vs/workbench/contrib/search/browser/searchActionsNav.ts`, when `expandSearchReplaceWidget` is true and the replace widget is already shown, pass `focusReplace: true` to `searchAndReplaceWidget.focus()` — mirroring how the editor find widget toggles focus between search and replace on repeated Ctrl+H invocations.
