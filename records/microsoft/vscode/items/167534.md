# Issue #167534 — A full absolute path does not work

[https://github.com/microsoft/vscode/issues/167534](https://github.com/microsoft/vscode/issues/167534)

## Summary

Fixed on main: absolute path support was added to ResourceGlobMatcher in commit 05e3b001da1 (May 2023), which handles the exact scenario described — full absolute paths in glob-based settings like explorer.autoRevealExclude now match correctly.

## Change summary

Full absolute paths (e.g. /Users/.../file.ts) did not work as patterns in explorer.autoRevealExclude — only relative paths matched.

## Best solution

Keep the shipped fix in commit 05e3b001da1 (1.79.0). The ResourceGlobMatcher now detects absolute paths in configured expressions and falls back to matching against the resource's full filesystem path when relative matching fails. No further change needed.
