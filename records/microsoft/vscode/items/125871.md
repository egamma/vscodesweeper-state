# Issue #125871 — Resolve file with same name as folder as index

[https://github.com/microsoft/vscode/issues/125871](https://github.com/microsoft/vscode/issues/125871)

## Summary

The Custom Editor Labels feature (`workbench.editor.label.patterns`) partially addresses the tab/open-editors aspect with manual configuration, but the broader request — better display of `index.ts`/`__init__.py` files in search results and quick-open without manual setup — remains unaddressed. Keeping open as the request still has a meaningful gap, particularly for search and quick-open contexts.

## Change summary

Improve display of commonly-named files (index.ts, __init__.py) across tabs, search results, open editors, and quick-open by showing parent directory context more prominently.

## Best solution

The tabs/open-editors aspect is partially served by `workbench.editor.label.patterns` (users can set `**/index.ts: ${dirname}` today). The remaining gap is in search results and quick-open where `index.ts` files lack directory context. This overlaps heavily with https://github.com/microsoft/vscode/issues/72514 which focuses on quick-open ranking; a maintainer should consider whether this issue adds distinct value beyond #72514 + the existing custom labels feature, or whether it should be consolidated.
