# Issue #125871 — Resolve file with same name as folder as index

[https://github.com/microsoft/vscode/issues/125871](https://github.com/microsoft/vscode/issues/125871)

## Summary

The core ask — better display of index.ts/index.tsx files — is largely addressed by the shipped `workbench.editor.customLabels.patterns` setting (covers tabs, open editors, and quick open). However, search results still don't use custom labels, so a residual gap remains. Keeping open for that remaining surface.

## Change summary

Improve display of identically-named files (index.ts, __init__.py) across tabs, open editors, search results, and quick open so users see meaningful context instead of many 'index' entries.

## Best solution

The shipped `workbench.editor.customLabels.patterns` setting (e.g. `"**/index.tsx": "${dirname}.tsx"`) already covers tabs, open editors, and quick open. The remaining gap is search results — `src/vs/workbench/contrib/search/` does not consume `ICustomEditorLabelService`. Extending custom labels to search result file names would close this issue entirely. Alternatively, this could be consolidated with https://github.com/microsoft/vscode/issues/72514 which covers a related quick-open ranking concern.
