# Issue #125871 — Resolve file with same name as folder as index

[https://github.com/microsoft/vscode/issues/125871](https://github.com/microsoft/vscode/issues/125871)

## Summary

The core UX pain (tabs/open-editors/quick-open flooded with identical `index.ts` names) is now largely addressable via `workbench.editor.customLabels.patterns` (e.g. `"**/index.{ts,tsx}": "${dirname}"`), which applies to tabs, open editors, and the quick-open picker. Some residual value remains — making this a smarter default or extending custom labels to the search results view — so keeping open in the Backlog is reasonable.

## Change summary

Show contextual parent-directory names instead of generic `index.ts` across tabs, open editors, search results, and quick open when many files share the same filename.

## Best solution

The `workbench.editor.customLabels.patterns` setting already solves the tab/open-editor/quick-open portion of this request. The remaining gap is (a) whether VS Code should ship a built-in default pattern for `index.*` files and (b) whether the search results view should also honor custom labels. A maintainer should decide if the residual scope justifies keeping this open or whether to close as largely addressed and track any remaining work in https://github.com/microsoft/vscode/issues/72514 (file picker ranking for index files) or https://github.com/microsoft/vscode/issues/41909 (tab label API, On Deck).
