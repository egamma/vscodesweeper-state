# Issue #197865 — Feature Request: Quick Open dialogue shows exact matches while loading other results

[https://github.com/microsoft/vscode/issues/197865](https://github.com/microsoft/vscode/issues/197865)

## Summary

Valid feature request with team interest (MEMBER comment). The quick open already uses a fast/slow picks pattern but exact path matches are still in the slow path — the requested optimization is not yet implemented on main.

## Change summary

Show exact file path matches immediately in Quick Open while broader fuzzy search results continue loading.

## Best solution

Move exact/relative path resolution (currently in `getFilePicks` → `getAbsolutePathFileResult`/`getRelativePathFileResults`) into the fast-picks phase of `doGetPicks` in `src/vs/workbench/contrib/search/browser/anythingQuickAccess.ts`, so users typing an exact path see results without waiting for the full file index search.
