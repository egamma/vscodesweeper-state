# Issue #245932 — WARN Via 'product.json#extensionEnabledApiProposals' extension wants API proposal

[https://github.com/microsoft/vscode/issues/245932](https://github.com/microsoft/vscode/issues/245932)

## Summary

Still valid debt: the shipped product.json (in the distro) contains stale API proposal entries for extensions, generating console warnings at startup. A build-time validation was added in Jan 2026 to prevent new mismatches, but existing stale entries and the noisy runtime warnings remain.

## Change summary

Stale entries in product.json's extensionEnabledApiProposals list cause console warnings for non-existent API proposals at startup.

## Best solution

Clean up stale entries from the distro product.json (the OSS product.json doesn't carry them), and consider downgrading the runtime log level from `warn` to `debug`/`trace` in `src/vs/workbench/services/extensions/common/extensionsProposedApi.ts` so finalized proposals don't pollute the console for users who happen to open DevTools or run extension tests.
