# Issue #170529 — Search with regular expression ignores match whole word

[https://github.com/microsoft/vscode/issues/170529](https://github.com/microsoft/vscode/issues/170529)

## Summary

Confirmed bug, still present on main. The `createRegExp` whole-word logic checks the literal last character of a regex pattern (e.g. `.`) rather than understanding it as a metacharacter, so `\b` is not appended when it should be, producing incorrect search results.

## Change summary

Search view with regex + match whole word enabled produces incorrect results because word boundaries are not properly added around regex metacharacters like `.`.

## Best solution

Fix `createRegExp` in `src/vs/base/common/strings.ts` so that when `isRegex` is true and `wholeWord` is requested, word boundaries are always wrapped around the entire pattern (e.g. `\b(?:fa.)\b`) rather than conditionally added based on the literal first/last characters. The ripgrep arg construction in `src/vs/workbench/services/search/node/ripgrepTextSearchEngine.ts` would then emit the correct pattern.

For pattern `fa.`, the last char `.` is a non-word character, so `\B` matches it and `\b` is NOT appended — producing `\bfa.` instead of a properly bounded pattern.
