# Issue #187338 — Linux: An OS keyring couldn't be identified for storing the encryption related data in your current desktop environment

[https://github.com/microsoft/vscode/issues/187338](https://github.com/microsoft/vscode/issues/187338)

## Summary

Verified duplicate of the canonical upstream tracking issue. The root cause — Chromium/Electron failing to auto-detect the OS keyring on non-standard Linux desktop environments — is owned by #185212, which a VS Code maintainer explicitly linked in comment 14.

## Change summary

VS Code fails to detect the OS keyring for secret storage on Linux desktop environments not recognized by Chromium's os_crypt backend (i3, Sway, Hyprland, bspwm, XMonad, qtile, etc.).

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/185212, which tracks the upstream Electron/Chromium fix to auto-discover the secret provider via D-Bus when the desktop environment is unknown. The `password-store` argv.json workaround remains the mitigation until the upstream fix lands.
