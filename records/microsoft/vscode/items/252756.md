## Evidence

- **search lifecycle:** When a file is closed, `onModelWillDispose` in `src/vs/workbench/contrib/search/browser/searchTreeModel/fileMatch.ts:145` calls `updateMatchesForModel` which clears `_textMatches` and repopulates from `model.findMatches`. - **regex engine split:** Open files use `model.findMatches` (JS regex, supports lookbehinds natively). Closed files use ripgrep with `--engine auto` (falls back to PCRE2 for lookbehinds) and `fixRegexNewline` transforms `[?!\s\t\n]` to `(?:[?!\s\t]|\r?\n)` — see `src/vs/workbench/services/search/node/ripgrepTextSearchEngine.ts:630-634`.
- **multiline detection:** `isMultilineRegexSource` in `src/vs/editor/common/model/textModelSearch.ts:69` detects `\n` in the pattern and sets multiline mode, adding `--multiline` to ripgrep args.
