# Issue #220048 — debugDescriptionSymbol should not be used for snapshots

[https://github.com/microsoft/vscode/issues/220048](https://github.com/microsoft/vscode/issues/220048)

## Summary

Team-authored debt issue requesting that snapshot formatting not use debugDescriptionSymbol, so that debug-description changes don't break snapshot tests. Still open and valid — the separation hasn't been implemented yet.

## Change summary

The snapshot utility's `formatValue` uses `debugDescriptionSymbol` to render objects, coupling debug descriptions to test assertions.

## Best solution

Introduce a separate `snapshotDescriptionSymbol` (or inline formatting helper) for snapshot rendering in `src/vs/base/test/common/snapshot.ts`, duplicating the useful representations for URI/Range/Position without coupling to the debug utility. This keeps `debugDescriptionSymbol` free to change without breaking tests.
