# Issue #282726 — Agent session says it's in progress when it's not

[https://github.com/microsoft/vscode/issues/282726](https://github.com/microsoft/vscode/issues/282726)

## Summary

The root cause — sessions cached as in-progress surviving a restart — was fixed in commit 61c1b25b464, which coerces in-progress status to Completed during serialization. Shipped in 1.112.0.

## Change summary

Agent session list shows a session as "in progress" after restarting VS Code even though the session has completed.

## Best solution

Keep the shipped fix (commit 61c1b25b464); no further change needed. The serialization guard ensures sessions are never persisted with in-progress status.
