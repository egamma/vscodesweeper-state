# Issue #213825 — Run Task command silently does nothing in serverless

[https://github.com/microsoft/vscode/issues/213825](https://github.com/microsoft/vscode/issues/213825)

## Summary

Still reproduces on main: in serverless web, `_trust()` returns false with no user feedback when `TaskExecutionSupportedContext` is not set, causing the Run Task command to silently do nothing. Team-authored issue, keeping open.

## Change summary

The 'Tasks: Run Task' command silently does nothing in serverless vscode.dev — no empty list, no message indicating tasks are unavailable.

## Best solution

When `_trust()` returns false due to serverless context (line ~3098 of abstractTaskService.ts), show an informational message or an empty quick pick indicating no tasks are available, rather than silently returning. As meganrogge noted, extension-provided custom tasks can still run in web, so the fix should show the (possibly empty) task list rather than blocking entirely.
