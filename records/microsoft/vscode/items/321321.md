# Issue #321321 — Toggle commrnting keyboard shortcut conflict with Agent window

[https://github.com/microsoft/vscode/issues/321321](https://github.com/microsoft/vscode/issues/321321)

## Summary

Confirmed keybinding conflict on current main: OpenAgentsWindowAction binds Ctrl+Shift+A at WorkbenchContrib weight (200) without an editor-focus exclusion, overriding Toggle Block Comment at EditorContrib weight (100). Team members acknowledged the fix (add editor-focus when-clause) in comments and assigned to 1.127.0, but the fix has not landed yet.

## Change summary

Ctrl+Shift+A opens the Agents Window instead of toggling a block comment when the editor has focus.

## Best solution

Add `EditorContextKeys.editorTextFocus.toNegated()` (or equivalent) to the `when` clause of the OpenAgentsWindowAction keybinding in `src/vs/workbench/contrib/chat/electron-browser/agentSessions/agentSessionsActions.ts` so the editor's Toggle Block Comment takes precedence when the editor is focused, as sandy081 proposed in the comments.
