# Issue #167818 — If I set focus back to Input 1, ctrl+z does nothing:

[https://github.com/microsoft/vscode/issues/167818](https://github.com/microsoft/vscode/issues/167818)

## Summary

Team-authored issue (MEMBER) requesting that undo works in merge editor input panes after clicking a codelens, not only in the result editor. Keeping open per policy.

## Change summary

In the merge editor, pressing Ctrl+Z in an input pane after clicking a codelens does nothing — undo only works when focus is in the result editor.

## Best solution

Extend the merge editor's undo stack so that actions triggered from input-pane codelenses can be undone while focus remains in that input pane, or route undo commands to the result editor's undo stack regardless of which pane has focus.
