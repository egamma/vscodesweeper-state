# Issue #285931 — Workbench hover appears when focused after quick pick closes

[https://github.com/microsoft/vscode/issues/285931](https://github.com/microsoft/vscode/issues/285931)

## Summary

Still reproduces on main: the hover service's onFocus handler (hoverService.ts:653) doesn't distinguish programmatic focus restoration (after QuickPick closes) from intentional keyboard focus, so a hover appears spuriously. No fix has landed.

## Change summary

Status bar hover tooltip appears when focus returns programmatically after a QuickPick closes, despite no user hover or keyboard navigation.

## Best solution

Teach the onFocus handler in `src/vs/platform/hover/browser/hoverService.ts` to detect programmatic focus restoration (e.g., via a flag set by the QuickInput service or a general programmatic-focus signal) and suppress the hover in that case, similar to the existing `fromHover` and `!e.relatedTarget` guards.
