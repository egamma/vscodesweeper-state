# Issue #325956 — Linux/X11: restored window height is clamped to 1080 on a 4K display in a mixed-resolution layout

[https://github.com/microsoft/vscode/issues/325956](https://github.com/microsoft/vscode/issues/325956)

## Summary

Confirmed in source: the post-construction setBounds() workaround in applyState() (windowImpl.ts:280) is gated to macOS and Windows only, excluding Linux. On multi-display Linux/X11 setups with mixed resolutions, Electron's BrowserWindow constructor clamps the height to the smaller display's dimension, and the missing re-application leaves the wrong bounds. A real gap remains.

## Change summary

Restored window height on Linux/X11 is deterministically clamped to the smallest display's height (e.g. 1080) on a mixed-resolution multi-monitor layout, losing the saved tall geometry.

## Best solution

Extend the existing post-construction setBounds() workaround at src/vs/platform/windows/electron-main/windowImpl.ts:280 to include Linux (isLinux is already imported). This is the same pattern that fixed the equivalent macOS (#74872) and Windows (#146499) regressions.
