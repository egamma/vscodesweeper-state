# Issue #287158 — Git - surface additional actions for worktrees

[https://github.com/microsoft/vscode/issues/287158](https://github.com/microsoft/vscode/issues/287158)

## Summary

Team-authored feature request with an 'On Deck' milestone — hard keep-open applies. The request to surface lock/unlock/prune/repair worktree actions is a natural extension of the existing git worktree support.

## Change summary

Expose git worktree lock, unlock, prune, and repair commands as actions in VS Code's built-in Git integration.

## Best solution

Implement the four worktree subcommands (lock, unlock, prune, repair) as registered Git commands in `extensions/git/`, alongside the existing worktree support. The lock/unlock commands would need UI for the optional --reason string and visual state for locked worktrees.
