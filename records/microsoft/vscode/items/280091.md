# Issue #280091 — Support for `${userHome}` variable in `chat.promptFilesLocations` setting

[https://github.com/microsoft/vscode/issues/280091](https://github.com/microsoft/vscode/issues/280091)

## Summary

The user's core need (referencing home directory in prompt file locations) is already addressed by tilde (`~/`) path support on current main, but the specific `${userHome}` variable substitution syntax they requested is not implemented. Keeping open as a feature request for broader variable support, though the workaround exists.

## Change summary

Support `${userHome}` and other VS Code predefined variables in the `chat.promptFilesLocations` setting.

## Best solution

The tilde path (`~/`) already covers the primary use case. If broader variable substitution (e.g., `${userHome}`, `${workspaceFolder}`) is desired for consistency with other settings, it could be added to the path resolution in `src/vs/workbench/contrib/chat/common/promptSyntax/utils/promptFilesLocator.ts`. However, this may be low priority given that `~/` works today. A maintainer should decide whether full variable substitution is worth the complexity or whether documenting `~/` support is sufficient.
