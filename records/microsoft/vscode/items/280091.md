# Issue #280091 — Support for `${userHome}` variable in `chat.promptFilesLocations` setting

[https://github.com/microsoft/vscode/issues/280091](https://github.com/microsoft/vscode/issues/280091)

## Summary

The setting currently supports `~` (tilde) expansion for the user's home directory but does not support VS Code's `${userHome}` variable syntax. The request for full variable substitution remains unimplemented.

## Change summary

Allow `${userHome}` (and other VS Code variables) in `chat.promptFilesLocations` so prompt file paths can reference the home directory portably.

## Best solution

The `~` workaround already covers the primary use case (home-relative paths). Full `${userHome}` support would require integrating `IConfigurationResolverService` into the prompt files locator path resolution in `src/vs/workbench/contrib/chat/common/promptSyntax/utils/promptFilesLocator.ts`. This is a nice-to-have polish — the tilde path already works for the stated scenario.
