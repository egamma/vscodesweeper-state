# Issue #325107 — Semantic search / workspace indexing returns "Unauthorized" for Azure DevOps repo after successful Microsoft account sign-in

[https://github.com/microsoft/vscode/issues/325107](https://github.com/microsoft/vscode/issues/325107)

## Summary

The ADO semantic search auth flow in the built-in Copilot extension obtains a Microsoft/AAD session token and sends it as Basic auth to the ADO code search endpoint, but the endpoint rejects it with 401/403. Multiple users confirm this on both macOS and Windows. The issue lives in the Copilot extension's `AdoCodeSearchService`.

## Change summary

Semantic search workspace indexing fails with 'Unauthorized' for Azure DevOps repos despite successful Microsoft account sign-in.

## Best solution

Investigate whether the ADO auth token scope or format obtained in `getAdoAccessTokenBase64` (via `getAnyAdoSession`) is correct for the semantic search ALM status endpoint — the token may need different scopes (e.g. `vso.code` or `vso.search`) or the endpoint URL construction in `getAdoAlmStatusUrl` may be wrong. The fix likely lives in `extensions/copilot/src/platform/remoteCodeSearch/common/adoCodeSearchService.ts` and/or `extensions/copilot/src/platform/authentication/vscode-node/authenticationService.ts`.
