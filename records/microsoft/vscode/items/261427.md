# Issue #261427 — VSCode crashes when pressing Ctrl + Alt + Backspace

[https://github.com/microsoft/vscode/issues/261427](https://github.com/microsoft/vscode/issues/261427)

## Summary

The crash is in the Electron/Chromium layer — Ctrl+Alt+Backspace is a system-level Linux shortcut that terminates the X server (or XWayland session), severing the display connection for all GUI apps. The fatal error at `electron/shell/browser/electron_browser_main_parts.cc:501` and the X connection error confirm this is upstream, not a VS Code bug. A team member has engaged and it sits in Backlog; keeping open for the team to decide whether to track graceful-shutdown handling upstream.

## Change summary

VS Code process terminates when pressing Ctrl+Alt+Backspace on Linux, because the OS kills the X/XWayland display server.

## Best solution

File or link to an Electron issue requesting graceful handling of abrupt X connection loss during shutdown, so the process exits cleanly rather than crashing. Alternatively, classify as `*as-designed` since the OS-level shortcut intentionally kills the display server and all connected clients. The `upstream` label would be more accurate than `bug` here.
