# Issue #204576 — Tree Sticky Scroll Compression: Show most important parts

[https://github.com/microsoft/vscode/issues/204576](https://github.com/microsoft/vscode/issues/204576)

## Summary

Valid feature request to show the most important (deepest) segments of compressed sticky scroll nodes rather than the top-most. The related bug (#204613) was fixed, but this UX improvement remains unimplemented on main. Low community traction (0 upvotes) so keeping open for the community process.

## Change summary

When tree sticky scroll compresses nodes, show the last (deepest) folder segments with leading ellipsis instead of the first segments.

## Best solution

Update the sticky scroll compressed-node rendering in `src/vs/base/browser/ui/tree/abstractTree.ts` (StickyScrollController) to truncate from the left and show trailing segments, e.g. '.../folderN-1/folderN'. This is a polish-level UX improvement in the tree-sticky-scroll area.
