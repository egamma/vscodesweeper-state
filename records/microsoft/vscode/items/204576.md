# Issue #204576 — Tree Sticky Scroll Compression: Show most important parts

[https://github.com/microsoft/vscode/issues/204576](https://github.com/microsoft/vscode/issues/204576)

## Summary

Valid feature request acknowledged by the area owner (benibenj). The compressed sticky scroll still shows elements from the top, not the most relevant deepest folders. No implementation found on main.

## Change summary

When tree sticky scroll compresses nodes, show the deepest (most important) folders with ellipsis at the start rather than truncating from the end.

## Best solution

Modify the compressed sticky scroll rendering to prioritize the deepest path segments (e.g. '.../folderN-1/folderN') when the full path overflows, as the area owner suggested in the comments. The rendering logic lives in `src/vs/base/browser/ui/tree/objectTree.ts` (CompressibleStickyScrollDelegate) and the tree renderers.
