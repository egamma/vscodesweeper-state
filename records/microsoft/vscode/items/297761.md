# Issue #297761 — "Restore Checkpoint" in Github Copilot in VS Code v1.109.5 is not working as intended

[https://github.com/microsoft/vscode/issues/297761](https://github.com/microsoft/vscode/issues/297761)

## Summary

The described 'Restore Checkpoint' behavior — pre-filling the prompt and visually blocking subsequent turns rather than deleting them — is the intended design on current main. The checkpoint system marks later requests as blocked (excluded from context) while preserving them, and populates the input with the original prompt text for re-submission.

## Change summary

Reporter expects 'Restore Checkpoint' to delete subsequent chat turns, but the current design blocks them visually and pre-fills the prompt instead.

## Best solution

Close as designed. The checkpoint mechanism in `chatModel.ts` (`setCheckpoint`) intentionally blocks subsequent requests from context without deleting them, and the action in `chatEditingActions.ts` pre-fills the input. This is the shipped behavior since the feature was reworked. If the reporter wants deletion, that would be a separate feature request.
