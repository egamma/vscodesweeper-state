# Issue #176467 — Allow VS-code to respect native titlebar decorations configuration on GNOME/Linux

[https://github.com/microsoft/vscode/issues/176467](https://github.com/microsoft/vscode/issues/176467)

## Summary

Partially addressed — WCO on Linux now draws native-styled window controls (shipped ~1.93), but the core request to respect the system's button-layout configuration (which buttons appear, left/right positioning) remains unresolved and depends on Electron/Chromium WCO improvements. Keeping open with 276 upvotes in Backlog.

## Change summary

VS Code's custom titlebar on Linux should read the system's window-decoration configuration (button visibility, ordering, and position) so it matches native GNOME/GTK apps.

## Best solution

The remaining gap — reading `org.gnome.desktop.wm.preferences button-layout` and reflecting it in WCO — lives primarily in Electron's WCO implementation. Track upstream progress on Electron's Linux WCO (https://github.com/electron/electron/pull/41769 and follow-ups); once Electron exposes the layout, VS Code can wire it through. Keep this issue as the canonical tracker for the full native-decoration-parity request.
