# Issue #251118 — Chat: Inline context doesn't work inside parens

[https://github.com/microsoft/vscode/issues/251118](https://github.com/microsoft/vscode/issues/251118)

## Summary

Still reproduces on main. The `computeCompletionRanges` function in `chatInputCompletionUtils.ts` explicitly requires a space before the `#` trigger character — parentheses (and other non-space, non-word characters) cause the completion to bail out. Team-authored issue; keeping open.

## Change summary

Typing `#` to invoke inline chat context references fails when the `#` immediately follows a non-space character like `(`.

## Best solution

Relax the guard in `computeCompletionRanges` (lines 28–32 of `chatInputCompletionUtils.ts`) so that non-word punctuation characters like `(`, `[`, `{` are treated the same as whitespace for the purpose of allowing completions to trigger. The same logic in `isAtTriggerCharacterToken` (line 74) also uses whitespace-only token boundary detection and would need a similar adjustment.
