# Issue #122951 — Unable to run sandboxed process when run as elevated under applocker

[https://github.com/microsoft/vscode/issues/122951](https://github.com/microsoft/vscode/issues/122951)

## Summary

Long-standing upstream Chromium/Electron sandbox issue when VS Code runs as administrator — the upstream Electron issue (electron/electron#49167) remains open, new reports continue to arrive (e.g. #318538), and the issue has 98 community upvotes. Should remain open as the canonical VS Code tracker for this upstream defect.

## Change summary

Webviews fail to load when VS Code runs as administrator because Chromium's sandboxed renderer process cannot start under elevated privileges with AppLocker or similar policies.

## Best solution

Resolution depends on the upstream Electron fix at https://github.com/electron/electron/issues/49167. VS Code already documents `--no-sandbox` as a workaround. Keep this issue as the canonical tracker; new reports (#318538, #318139) should be closed as duplicates of this one. Once Electron ships a fix and VS Code upgrades, verify and close.
