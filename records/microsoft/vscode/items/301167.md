# Issue #301167 — chat.pluginLocations: absolute paths and tilde (~/) paths not resolved in Remote WSL

[https://github.com/microsoft/vscode/issues/301167](https://github.com/microsoft/vscode/issues/301167)

## Summary

Verified duplicate of #293768 — same root cause (URI.file() used instead of remote-scheme-aware URI for absolute/tilde paths) affects `chat.pluginLocations` in the same way it affects `chat.agentFilesLocations` et al. The canonical issue already tracks the shared fix needed across all chat location settings in remote contexts.

## Change summary

`chat.pluginLocations` with absolute or tilde paths silently fails in Remote WSL because `_resolvePluginPath` uses `URI.file()` instead of a remote-scheme-aware URI.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/293768 — a single fix replacing `URI.file(path)` with a remote-authority-aware URI construction in both `agentPluginServiceImpl.ts` and the `promptSyntax` locator resolves all affected settings together.
