# Issue #147916 — Warn when auth extension is disabled

[https://github.com/microsoft/vscode/issues/147916](https://github.com/microsoft/vscode/issues/147916)

## Summary

Still valid on main: when an auth provider extension is disabled and a user initiates sign-in, VS Code shows a generic timeout/registration error instead of explaining that the required extension is disabled. The gap remains in `tryActivateProvider`.

## Change summary

When an authentication provider extension is disabled, sign-in attempts produce a cryptic timeout error instead of a clear warning that the extension needs to be enabled.

## Best solution

In `tryActivateProvider` (authenticationService.ts), after the timeout fires, check the extension enablement state via `IExtensionManagementService` or `IExtensionsWorkbenchService` — if the provider's contributing extension is disabled, surface a specific error/notification offering to enable it, rather than a generic 'timed out waiting for provider' message.
