# Issue #110281 — Ubuntu `lxc profile show default > default.yml'` on zsh fails to redirect

[https://github.com/microsoft/vscode/issues/110281](https://github.com/microsoft/vscode/issues/110281)

## Summary

This is a snap confinement issue, not a VS Code code bug. The snap sandbox restricts child processes from accessing the LXD Unix socket, causing `lxc` to produce empty output. The reporter confirmed it works with the deb package and fails only with the snap install. Keeping open as it carries `help wanted` and is an upstream/packaging concern.

## Change summary

Running `lxc` commands in VS Code's integrated terminal produces empty output when VS Code is installed as a snap, because snap confinement blocks access to the LXD daemon socket.

## Best solution

This requires a snap packaging fix — either adding the appropriate snap interface/plug (e.g. `lxd` interface) to VS Code's `snapcraft.yaml`, or documenting the limitation. The root cause is snap confinement restricting socket access for child processes, not anything in VS Code's terminal implementation. The fix likely lives in `build/` snap packaging files or in a snap store configuration change.
