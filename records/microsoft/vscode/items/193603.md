# Issue #193603 — [wsl] Can't run `code .` when offline and VS Code got updated

[https://github.com/microsoft/vscode/issues/193603](https://github.com/microsoft/vscode/issues/193603)

## Summary

The reported issue involves the WSL server bootstrap script removing the existing installation before verifying download availability. The bootstrap logic may live in the WSL extension rather than core VS Code, but the WSL label and Backlog milestone suggest a team decision was already made to track it here. Keeping open for maintainer judgment.

## Change summary

Running `code .` in WSL while offline after a VS Code update deletes the existing server before attempting download, leaving the user unable to use VS Code until reconnected.

## Best solution

The bootstrap script should verify network/download availability (or download to a temp location first) before removing the previous server installation. If this logic lives in the WSL extension rather than core, the issue should be transferred to microsoft/vscode-remote-release. If it's in core's CLI server setup, guard the removal behind a successful download.
