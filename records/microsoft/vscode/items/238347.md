# Issue #238347 — Cannot discard merge changes in merge editor (with auto save on)

[https://github.com/microsoft/vscode/issues/238347](https://github.com/microsoft/vscode/issues/238347)

## Summary

This issue describes the same underlying problem as #251581 — the merge editor result file gets auto-saved with conflict markers stripped, preventing recovery of the original state. Both request a way to preserve or restore the original file. Proposing duplicate close to consolidate discussion.

## Change summary

When auto-save is enabled, closing the merge editor with 'Close with Conflicts' leaves the file in a partially-merged state with conflict markers removed, with no way to recover the original.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/251581, which proposes disabling auto-save for the merge editor result file — solving the same root cause. The fix would likely involve either suppressing auto-save for the result buffer or restoring the original content (with markers) on close/discard.
