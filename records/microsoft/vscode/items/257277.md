# Issue #257277 — MCP oauth: support client_secret_post and client_secret_basic

[https://github.com/microsoft/vscode/issues/257277](https://github.com/microsoft/vscode/issues/257277)

## Summary

Still valid on main: the MCP OAuth flow always registers with `token_endpoint_auth_method: 'none'` and only supports `client_secret_post` (body parameter) when a secret is returned. Full `client_secret_basic` (Authorization header) support is not yet implemented. An open PR exists but has not merged.

## Change summary

Support `client_secret_basic` and `client_secret_post` token endpoint authentication methods in the MCP OAuth flow.

## Best solution

Land (or supersede) PR https://github.com/microsoft/vscode/pull/257283 to add `client_secret_basic` support (Base64 Authorization header), and update the dynamic client registration to negotiate supported auth methods rather than always requesting `none`.
