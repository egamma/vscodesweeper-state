# Issue #299357 — VS Code v1.110 and Copilot Chat v0.38.0 issue - GitHub Copilot Chat is stuck in "Ask" mode

[https://github.com/microsoft/vscode/issues/299357](https://github.com/microsoft/vscode/issues/299357)

## Summary

Root cause identified by the reporter and confirmed by another user: the setting `chat.agent.enabled` was set to `false`, which by design hides agent and edit modes from the picker. This is working as intended — the setting controls whether agent mode is available.

## Change summary

Chat mode picker only shows 'Ask' and does not offer Agent or Edit modes.

## Best solution

Close as designed. The `chat.agent.enabled` setting (default `true`) intentionally controls whether agent/edit modes appear in the picker. The reporter's own troubleshooting (comment 5) and an independent confirmation (comment 6) both identified setting the value to `true` (or removing the `false` override) as the fix.
