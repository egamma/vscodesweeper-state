# Issue #224115 — When selecting a specific font (MonoLisa), weird symbols instead of characters on latest insiders build

[https://github.com/microsoft/vscode/issues/224115](https://github.com/microsoft/vscode/issues/224115)

## Summary

Upstream Chromium bug in the FontMatchingCTMigration Blink feature, already triaged and labeled `upstream-issue-fixed` by a team member. The issue was on Chromium 124 (Electron 30.1.2); current VS Code ships Electron 42.5.0 (~Chromium 134), well past the upstream fix. No longer reproducible on current builds.

## Change summary

MonoLisa font rendered garbled symbols instead of characters on macOS due to a Chromium FontMatchingCTMigration regression.

## Best solution

No VS Code change needed — the fix was in Chromium's font matching code and has shipped through multiple Electron upgrades since. Close as not reproducible on current builds.
