# Issue #323726 — Agents, /code-review command conflicts with Claude Agent SDK

[https://github.com/microsoft/vscode/issues/323726](https://github.com/microsoft/vscode/issues/323726)

## Summary

Genuine slash-command routing conflict: VS Code's built-in 'Run Code Review' action sends `/code-review` to the chat widget, which collides with the Claude Agent SDK's own `/code-review` built-in slash command. The autocomplete popup shows the correct Claude SDK entry, but execution routes through VS Code's code review feature instead. This is a real bug in how slash commands are dispatched in the Agents window when an agent-host harness provides a command with the same name as VS Code's built-in action.

## Change summary

Typing `/code-review` in the Agents window with Claude Agent SDK selected executes VS Code's built-in code review instead of the Claude SDK's `/code-review` slash command.

## Best solution

The slash command dispatcher in the Agents window (likely in `chatSlashCommands.ts` or the chat widget's `acceptInput` path) needs to give priority to the active agent-host harness's registered slash commands over VS Code's built-in actions when resolving `/code-review`. Alternatively, the `RunSessionCodeReviewAction` in `src/vs/sessions/contrib/codeReview/browser/codeReview.contributions.ts` could be made aware of the active harness's command set and defer to it when a collision exists.
