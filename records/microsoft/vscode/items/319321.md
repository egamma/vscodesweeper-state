# Issue #319321 — Feature Request: Presaved MCP Toolsets / Named Presets for Context-Aware Tool Activation

[https://github.com/microsoft/vscode/issues/319321](https://github.com/microsoft/vscode/issues/319321)

## Summary

VS Code already ships a 'tool sets' feature (`.toolsets.jsonc` files, 'Configure Tool Sets...' command) that covers much of what's requested — named groups of tools including MCP tools. However, the issue's specific ask for a global 'activate toolset = disable everything else' workflow (rather than per-request tool attachment) and server-level toggling is a nuanced gap that may still have value as a feature request.

## Change summary

Allow users to define named presets of MCP servers/tools that can be quickly activated or deactivated as a unit, disabling non-included servers.

## Best solution

Classify as feature-request and route to the chat/tools area. The existing tool sets feature covers named groupings and per-request tool selection; the remaining gap is a 'globally activate this toolset and disable everything else' toggle mode. The maintainer should decide whether this gap warrants additional work or whether the existing tool sets + per-request selection is sufficient.
