# Issue #304550 — SCM is not updating when agent makes changes

[https://github.com/microsoft/vscode/issues/304550](https://github.com/microsoft/vscode/issues/304550)

## Summary

Team-authored bug with `important` label — hard keep-open. The SCM view fails to refresh when an agent modifies files, even though `git status` in the terminal reflects the changes correctly.

## Change summary

The SCM view does not update to reflect file changes made by the agent, despite git status showing the correct state.

## Best solution

Investigate whether the agent's file modifications bypass the file watcher or the SCM service's refresh trigger. Likely the agent host writes files through a path that doesn't fire the expected `FileChangeEvent` to the Git extension's status scanner. Fix in the SCM/git contrib or ensure the agent host's file operations go through `IFileService` or explicitly trigger a refresh.
