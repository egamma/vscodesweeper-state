# Issue #278311 — Auto-indent mangles JSON files due to URL misinterpretation

[https://github.com/microsoft/vscode/issues/278311](https://github.com/microsoft/vscode/issues/278311)

## Summary

Fixed on main — the onEnterRule regex for JSON comment continuation was tightened to require whitespace-separated slashes, preventing URLs from triggering comment insertion.

## Change summary

Pressing Enter after a line containing a URL with `//` in a JSON file incorrectly inserted `// ` as if continuing a comment.

## Best solution

Keep the shipped fix in commit 9b43a71f039; no further change needed.
