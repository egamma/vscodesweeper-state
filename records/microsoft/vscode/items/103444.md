# Issue #103444 — Git - gutters don't show on filenames that have changed case

[https://github.com/microsoft/vscode/issues/103444](https://github.com/microsoft/vscode/issues/103444)

## Summary

Genuine bug with milestone 'On Deck' — must stay open. The root cause (IUriIdentityService caching the old-cased URI after a case-only rename, which then mismatches git's case-sensitive path) is well-understood by maintainers but still unfixed on current main.

## Change summary

Git gutter decorations (quick diff) fail to appear after a file is renamed with only a case change (e.g. counter.jsx → Counter.jsx).

## Best solution

Fix the URI identity / text-model caching layer so that after a case-only file rename, the editor model's URI reflects the new casing. The quick diff pipeline in `quickDiffModel.ts` passes `this._model.resource` to `getQuickDiffs`, and on a case-insensitive filesystem the cached URI retains old casing while git uses the new casing, breaking the diff lookup. A fix likely involves either invalidating the URI identity cache on file-rename events that change case, or normalizing paths to git's known casing before the diff comparison. The git extension's `provideOriginalResource` in `extensions/git/src/quickDiffProvider.ts` and the workbench's `quickDiffModel.ts` / `quickDiffDecorator.ts` are the key files.
