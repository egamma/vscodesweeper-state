# Issue #130415 — Git - Git: gpg failed to sign the data 

[https://github.com/microsoft/vscode/issues/130415](https://github.com/microsoft/vscode/issues/130415)

## Summary

Verified duplicate of the canonical GPG passphrase prompting issue #43809. The root cause across all reports in this thread is that VS Code's git extension invokes `git commit -S` without a mechanism to prompt for the GPG passphrase, which is the exact capability tracked by #43809.

## Change summary

Git commit signing with GPG fails in VS Code because there is no mechanism to prompt for the GPG passphrase when the agent cache has expired.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/43809, which tracks adding GPG passphrase prompting support to the git extension. Implementing that feature resolves this report and the many duplicates in this thread.
