# Issue #117255 — Move Line Up/Down alters selection

[https://github.com/microsoft/vscode/issues/117255](https://github.com/microsoft/vscode/issues/117255)

## Summary

Still reproducible on main: the autoIndent logic in MoveLinesCommand alters selection boundaries when indentation is adjusted during move-down, and the adjustment isn't undone on move-up. The existing `_moveEndLineSelectionShrink` logic handles only a subset of the problem.

## Change summary

Moving lines down then up with autoIndent enabled alters the selection end position by one character.

## Best solution

Fix the selection tracking in `src/vs/editor/contrib/linesOperations/browser/moveLinesCommand.ts` to properly account for indent edits when computing the cursor state. The `getIndentEditsOfMovingBlock` indent adjustments (line ~380-407) expand/shrink line content, but the tracked selection doesn't compensate symmetrically on round-trip moves.
