Team-authored bug report (bpasero is a CONTRIBUTOR). The X button to remove chat context attachments is a no-op in a floating/auxiliary window until the chat input field receives focus. No fix found on current main. Keeping open for maintainer judgment.

## Change summary

In a floating chat window, clicking the X button to remove a context attachment does nothing until the chat input field is focused first.

## Best solution

The click handler for the context-removal button likely depends on the chat input widget having focus (or the correct scoped context-key service from the main window).
