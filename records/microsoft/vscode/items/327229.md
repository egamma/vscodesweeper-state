# Issue #327229 — Terminal fails to launch due to outdated libnode.so dependency on Linux

[https://github.com/microsoft/vscode/issues/327229](https://github.com/microsoft/vscode/issues/327229)

## Summary

This is a distro-packaging issue, not a VS Code core defect. The reporter is using a Debian-packaged 'code-oss' build that links against system libnode.so instead of bundling its own Electron/Node.js. Official VS Code and the upstream OSS build bundle their own runtime, so no system libnode dependency exists. The libnode.so version mismatch is caused by the distro packager's build configuration.

## Change summary

Integrated terminal fails to launch on a distro-packaged code-oss build because the system's libnode.so version doesn't match the version the package was compiled against.

## Best solution

This should be reported to the Debian code-oss package maintainer — they need to rebuild the package against the current system libnode. Alternatively, the reporter can use the official VS Code .deb from https://code.visualstudio.com/ which bundles its own Node.js runtime and has no system libnode dependency.

## Evidence

- **distro packaging:** The reporter uses 'code-oss (latest)' on Debian, which is a distro-repackaged build that links against system libraries. Official VS Code bundles Electron (which includes Node.js/libnode), so there is no system libnode.so dependency.
- **error analysis:** The error 'libnode.so.115: cannot open shared object file' is a native shared-library mismatch — the distro package was compiled against Node.js 20.x (ABI 115) but the system was upgraded to a newer Node.js (ABI 120). This is a packaging rebuild issue, not a VS Code source code bug.
- **node-pty architecture:** node_modules/node-pty/lib/unixTerminal.js loads a native pty.node addon. In the official build, this addon is compiled and shipped with the bundled Electron. In distro builds, it links against system libnode, creating the version coupling.
- **no hard-coded version in VS Code source:** The reporter claims unixTerminal.js has a 'hard-coded dependency on a specific libnode version' — this is incorrect. The libnode ABI version is determined at compile time by the distro packager, not by VS Code's source code.
