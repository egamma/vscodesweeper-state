## Summary

The branch dropdown being disabled on the new session page is likely still present on current main — the refactoring in ae95cb2c056 improved session resolution for pickers but did not change the underlying logic that disables the picker when branches haven't loaded. The Agents window is under active development and this remains a valid UX issue.

## Change summary

Branch dropdown on the Agents window new session page is greyed out/disabled and switching branches via Command Palette doesn't update it.

## Best solution

Ensure the new-session page either loads branches eagerly when a workspace folder has a git repo, or shows a loading state rather than a permanently disabled dropdown. The fix likely lives in the initialization flow of CopilotChatSession in copilotChatSessionsProvider.ts and the disabled logic in branchPicker.ts.
