# Issue #247968 — MCP client should validate tool descriptions for prompt injections and freeze them.

[https://github.com/microsoft/vscode/issues/247968](https://github.com/microsoft/vscode/issues/247968)

## Summary

Team-authored security-adjacent feature request for MCP tool description validation and freezing. Still under discussion with no implementation on main — keeping open per team-member rule and because the security concern remains valid.

## Change summary

MCP client should validate tool descriptions for prompt injections and allow freezing the tool list to detect rug-pull attacks.

## Best solution

Design a mechanism to hash/freeze MCP tool descriptions after initial approval, notify users on changes, and optionally run prompt-injection detection on tool descriptions. The discussion between connor4312 and pelikhan identified that automation (not user prompts) is the right UX for re-validation, possibly with a lock icon for explicit freeze intent.
