# Issue #305729 — [Unhandled Error] Make sure the ref is set before accessing the element. Maybe wrong initialization order?

[https://github.com/microsoft/vscode/issues/305729](https://github.com/microsoft/vscode/issues/305729)

## Summary

Still reproduces on current main — the initialization-order bug is unpatched. An open PR (#315078) exists but has not been merged. High-volume error (362K users on 1.127.0).

## Change summary

The InlineEditsOnboardingExperience constructor triggers a gutter animation before the DOM ref is mounted, causing an unhandled error for hundreds of thousands of users per release.

## Best solution

Merge or iterate on PR https://github.com/microsoft/vscode/pull/315078 which gates `triggerAnimation()` on icon mount state. The guard belongs in `gutterIndicatorView.ts` (line ~214) or in `inlineEditsNewUsers.ts` (line ~96/102) — either check `_iconRef` is set before calling `.element`, or defer the animation trigger until after render.
