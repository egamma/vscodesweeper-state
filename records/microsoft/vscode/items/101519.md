# Issue #101519 — Git - File watcher events should respect `.gitignore`

[https://github.com/microsoft/vscode/issues/101519](https://github.com/microsoft/vscode/issues/101519)

## Summary

Valid, accepted feature request (has `help wanted`) that remains unimplemented on current main — the git extension's file watcher at `extensions/git/src/repository.ts:920` watches `**` with no gitignore filtering. Keeping open as an accepted backlog item.

## Change summary

The built-in git extension's file watcher triggers `git status` on changes to gitignored files, causing unnecessary polling in repos with active gitignored directories (build outputs, database temp files).

## Best solution

Filter `onRepositoryWorkingTreeFileChange` events through the repository's gitignore rules before triggering `git status` in `extensions/git/src/repository.ts`. The git extension already has `checkIgnore` capability (via `git check-ignore`) that could be leveraged, though batching/caching would be needed for performance. The `help wanted` label is appropriate — this is a good community contribution target.
