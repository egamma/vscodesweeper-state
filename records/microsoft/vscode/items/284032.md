# Issue #284032 — Webview Developer tools - Select element on page always selects entire page

[https://github.com/microsoft/vscode/issues/284032](https://github.com/microsoft/vscode/issues/284032)

## Summary

This is an Electron/Chromium limitation: the 'Open Webview Developer Tools' command opens DevTools for the main BrowserWindow, but webviews live inside sandboxed iframes with separate browsing contexts. The 'Select element on page' tool in DevTools cannot pierce into cross-origin iframe contents — it can only select the top-level iframe element. Keeping open as an upstream constraint.

## Change summary

The 'Select element on page' tool in Webview Developer Tools always selects the entire iframe rather than individual elements inside the webview.

## Best solution

The root cause is that `OpenWebviewDeveloperToolsAction` calls `nativeHostService.openDevTools()` on the main window, but webview content lives in sandboxed iframes. A potential improvement would be to open DevTools directly on the webview's `webContents` (Electron's `webContents.openDevTools()` on the iframe's web contents) rather than the parent window's DevTools. This would require plumbing in the electron-sandbox layer to identify the active webview's web contents ID and open dedicated DevTools for it. Alternatively, this may need upstream Electron support for inspecting into OOP iframes from the parent DevTools.
