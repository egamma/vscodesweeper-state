# Issue #300828 — Don't show Apply button when a Copilot CLI session works on the workspace

[https://github.com/microsoft/vscode/issues/300828](https://github.com/microsoft/vscode/issues/300828)

## Summary

Team-authored issue — keeping open. The Apply submenu in the changes toolbar is registered without checking isolation mode, so it appears even for workspace-isolated sessions where changes are already applied directly.

## Change summary

The Apply button appears in the changes toolbar for Copilot CLI sessions using workspace isolation, even though changes are already applied to the workspace.

## Best solution

Add an `ActiveSessionContextKeys.IsolationMode` guard (e.g. `isEqualTo('worktree')`) to the `AgentsChangesPrimaryActionSubMenu` registration on `AgentsChangesToolbar` in `applyChangesToParentRepo.ts` so the Apply submenu only appears when the session uses worktree isolation.
