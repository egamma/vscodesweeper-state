# Issue #112099 — Downloading files and folders should preserve executable bit

[https://github.com/microsoft/vscode/issues/112099](https://github.com/microsoft/vscode/issues/112099)

## Summary

Still a valid gap: the download path in `fileImportExport.ts` does not read or write executable/permission metadata, even though `FilePermission.Executable` now exists in the file system provider API. The native download path could feasibly preserve permissions, but the browser path is limited by DOM APIs. The broader permissions API issue (https://github.com/microsoft/vscode/issues/48659) remains open and partially overlaps but is a broader scope.

## Change summary

Downloading files from a remote (e.g. Codespaces) strips the executable bit; the user must manually re-apply it after download.

## Best solution

The native download path in `src/vs/workbench/contrib/files/browser/fileImportExport.ts` (`doDownloadNative`) should read `FilePermission.Executable` from the source stat and apply it to the downloaded file via the local file system provider. The browser download path cannot preserve permissions due to DOM API limitations — document that gap or use a tar/zip wrapper. This is a concrete, scoped improvement that could land independently of the broader https://github.com/microsoft/vscode/issues/48659 permissions API work.
