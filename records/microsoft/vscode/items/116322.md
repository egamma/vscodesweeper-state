# Issue #116322 — Git - Adopt `execInstallFolder` for askpass

[https://github.com/microsoft/vscode/issues/116322](https://github.com/microsoft/vscode/issues/116322)

## Summary

The debt item is partially addressed — askpassManager.ts (commit e37fdc9118b, Jan 2026) now copies askpass scripts to content-addressed user storage on Windows, solving the path-stability-across-updates problem. However, `__dirname` is still used as the source directory throughout the git extension (main.ts line 75, askpassManager.ts, gitEditor.ts), so the original ask to adopt `execInstallFolder` has not been completed. Keeping open as valid remaining debt.

## Change summary

Replace `__dirname`-based askpass script paths with `execInstallFolder` for more robust path resolution, especially in remote scenarios.

## Best solution

Evaluate whether `execInstallFolder` is still the right abstraction now that askpassManager handles Windows path stability via content-addressed copies. If the remote-development motivation (vscode-remote-release#4308) is fully addressed by current remote extension host resolution, this debt item may be closeable as superseded. Otherwise, adopt `execInstallFolder` in `extensions/git/src/main.ts` and `extensions/git/src/askpassManager.ts` where `__dirname` is passed as `sourceDir`.
