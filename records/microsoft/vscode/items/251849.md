# Issue #251849 — Windows: unable to bring window to front even when trying to steal focus

[https://github.com/microsoft/vscode/issues/251849](https://github.com/microsoft/vscode/issues/251849)

## Summary

Confirmed upstream Electron limitation on Windows: the OS foreground-lock prevents VS Code from stealing focus, and Electron's `app.focus()` / `BrowserWindow.focus()` do not work around it. A maintainer has engaged and the issue is correctly labeled `upstream` + `help wanted`. Keeping open.

## Change summary

On Windows, VS Code cannot bring its window to the foreground when a debugged child process holds focus; the taskbar icon flashes instead.

## Best solution

The fix needs Electron to expose `AllowSetForegroundWindow` or implement the minimize→show→setAlwaysOnTop→focus workaround internally. Track progress on https://github.com/electron/electron/issues/2867. If Electron adds support, update `doFocusWindow()` in `src/vs/platform/windows/electron-main/windowImpl.ts` to use it. The `help wanted` label invites a contributor to prototype one of the suggested workarounds (e.g. calling `AllowSetForegroundWindow` from the focused process, or the minimize-show-alwaysOnTop-focus sequence) as an interim measure.
