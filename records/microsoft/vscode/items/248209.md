# Issue #248209 — Search Replace: Cursor does not move to next match when replace string starts with a newline followed by search string

[https://github.com/microsoft/vscode/issues/248209](https://github.com/microsoft/vscode/issues/248209)

## Summary

Still reproduces on main: after a replace, the cursor position is computed incorrectly for multi-line replace strings because `setStartPosition` adds the full replace-string length to the column without accounting for newlines, so the cursor stays on the original line and the same match gets replaced repeatedly.

## Change summary

Editor find-and-replace cursor does not advance to the next match when the replace string contains a newline followed by the search string.

## Best solution

Fix the post-replace start position calculation in `findModel.ts` line 498 to account for newlines in the replace string — compute the correct line number and column offset from the replace string's last line, rather than adding the full string length to the start column.

- **Fix prompt:**

  > In src/vs/editor/contrib/find/browser/findModel.ts, the `replace()` method at line 498 computes the post-replace cursor start position as `new Position(selection.startLineNumber, selection.startColumn + replaceString.length)`. This is wrong for multi-line replace strings: it stays on the original line and adds the entire string length (including newline characters) as a column offset. The fix: split `replaceString` on newlines; if it contains newlines, advance `startLineNumber` by the number of newlines, and set the column to the length of the last line segment (plus 1 for 1-based columns); if no newlines, keep the current arithmetic. Do not change `replaceAll()` or any other method. Validate with a test in src/vs/editor/contrib/find/test/browser/findModel.test.ts: set up a document with 'text text text', search for 'text', replace with '\ntext1' (newline + text1), call replace three times, and assert the document content matches the expected result with each 'text' replaced sequentially rather than the first one being replaced repeatedly.
