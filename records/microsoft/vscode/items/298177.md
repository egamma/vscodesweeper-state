# Issue #298177 — Cancel git clone does not kill the git process

[https://github.com/microsoft/vscode/issues/298177](https://github.com/microsoft/vscode/issues/298177)

## Summary

Confirmed bug on current main: the cancellation handler at line 252 of `extensions/git/src/git.ts` calls `child.kill()` which only terminates the immediate git process, leaving child processes (e.g. `git-remote-https`) running. An open PR (#309248) is already addressing this — keeping open for that PR to land.

## Change summary

Cancelling a git clone operation leaves orphaned child processes (e.g. git-remote-https) running in the background, holding file locks.

## Best solution

Land https://github.com/microsoft/vscode/pull/309248 (or an equivalent fix) that replaces `child.kill()` with process-tree termination (`taskkill /T /F` on Windows, `process.kill(-pid)` on Unix) in the cancellation handler in `extensions/git/src/git.ts`.
