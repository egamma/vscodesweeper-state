## Summary

Team-authored bug (MEMBER) — must keep open. The issue reports that file-changed state leaks across chat sessions (sidebar and editor), and #276539 was closed as a duplicate of this one, confirming this is the canonical tracker. No specific fixing commit was found.

## Change summary

Editing a file in one chat session causes the 'files changed' indicator to appear in a different, unrelated chat session.

## Best solution

Scope the file-changed state display per chat editing session so each chat widget only shows changes from its own session. The relevant code is likely in `src/vs/workbench/contrib/chat/browser/widget/chatContentParts/chatChangesSummaryPart.ts` and `src/vs/workbench/contrib/chat/browser/chatEditing/chatEditingServiceImpl.ts` — the editing session is already keyed by `chatSessionResource`, so the UI layer may just need to filter accordingly.
