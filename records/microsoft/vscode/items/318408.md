# Issue #318408 — Selected model in a session isn't always preserved on Reload

[https://github.com/microsoft/vscode/issues/318408](https://github.com/microsoft/vscode/issues/318408)

## Summary

Team-authored bug report — model selection per session is not reliably restored on reload. The model picker restore logic in `ModelPicker._initModel()` relies on both the provider-side `session.modelId` observable and the per-provider storage fallback, but a timing race between session restoration and model-vendor resolution can cause the wrong model to appear. Keeping open for the team to address.

## Change summary

The model selected in an individual agents-window session (e.g. Haiku) reverts to a different model (e.g. Sonnet) after a window reload.

## Best solution

Fix the restore path in `ModelPicker._initModel()` (and possibly the agent-host provider's cached `modelSelection` / `modelId` round-trip) so that the per-session model identity survives reload. The session cache in `baseAgentHostSessionsProvider.ts` stores `modelSelection` but the picker's `_initModel` may resolve the fallback model before the provider has pushed the restored `modelId` observable — the guard at line 261 (`_hasResolvedSessionModelVendor`) was added to address this, but the race window appears to remain.
