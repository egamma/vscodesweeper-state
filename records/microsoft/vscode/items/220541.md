# Issue #220541 — Git - Stage Selected Range Bug with Path Capitalisation

[https://github.com/microsoft/vscode/issues/220541](https://github.com/microsoft/vscode/issues/220541)

## Summary

Still reproduces on main: when staging selected ranges on a case-insensitive filesystem, `git update-index --cacheinfo` receives the path from the file URI (filesystem casing) rather than the casing in git's index, creating a duplicate entry with wrong capitalization. The bug is in the `stage()` method in `extensions/git/src/git.ts`. Keeping open as it remains unfixed.

## Change summary

Staging selected ranges on Windows/macOS can introduce a duplicate file entry with wrong path capitalization when the filesystem casing differs from the git index casing.

## Best solution

In `extensions/git/src/git.ts` `stage()`, before calling `update-index --cacheinfo`, look up the existing path casing from the git index (e.g. via `git ls-files` with case-insensitive matching) and use the index's casing for `update-index` instead of the filesystem/URI casing. This preserves git's case-sensitive path tracking on case-insensitive filesystems.

## Evidence

- **root cause in source:** `extensions/git/src/git.ts` line 2029: `await this.exec(['update-index', add, '--cacheinfo', mode, hash, relativePath])` — `relativePath` comes from the file URI, preserving filesystem casing which can differ from the git index on case-insensitive OS.
- **path derivation:** `_stageChanges()` in `extensions/git/src/commands.ts` line 1934 gets `modifiedUri` from `textEditor.document.uri`, passing it through `runByRepository` to `repository.stage(resource.fsPath, ...)`. On Windows, `fsPath` may have different casing than git's index.
- **casing-aware comparison but casing-preserving extraction:** `relativePathWithNoFallback()` in `extensions/git/src/util.ts` line 365 uses case-insensitive `isDescendant()` check but then does `to.substring(from.length)` preserving the `to` path's original casing — correct for general use, but wrong when the casing must match git's index.
