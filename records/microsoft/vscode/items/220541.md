# Issue #220541 — Git - Stage Selected Range Bug with Path Capitalisation

[https://github.com/microsoft/vscode/issues/220541](https://github.com/microsoft/vscode/issues/220541)

## Summary

Still reproduces on main: the stage-selected-ranges flow uses the filesystem's path capitalization for `git update-index --cacheinfo`, which on case-insensitive filesystems (Windows/macOS) can create a duplicate index entry with wrong case when the directory was previously renamed.

## Change summary

Staging selected ranges on Windows creates a duplicate file entry in the git index with incorrect path capitalization when a parent folder's case was previously changed.

## Best solution

In the `stage` method in `extensions/git/src/git.ts`, use `getGitFilePath` (which does case-insensitive matching against the git index/tree) to resolve the correct git-tracked path before passing it to `update-index --cacheinfo`. This ensures the staged content uses git's canonical path rather than the filesystem's potentially mismatched capitalization.
