[https://github.com/microsoft/vscode/issues/327238](https://github.com/microsoft/vscode/issues/327238)

## Summary

The xterm.js parser on current main correctly handles BEL (0x07) as an OSC string terminator without triggering the bell callback (line 129 of EscapeSequenceParser.ts). The reporter is on VS Code 1.126.0 (current stable is 1.130.0), so this may already be resolved. Needs confirmation on a current build and more detail about the sound heard.

## Change summary

Terminal bell sound plays on every prompt in the integrated terminal when zsh shell integration is enabled, apparently triggered by OSC 633 sequences using BEL as the string terminator.

If it still reproduces on latest, we need the exact accessibility signal and terminal bell settings, and whether the sound is the OS bell or a VS Code accessibility signal.
