# Issue #184081 — Disable global replace all if all files are readonly

[https://github.com/microsoft/vscode/issues/184081](https://github.com/microsoft/vscode/issues/184081)

## Summary

Still reproduces on main: the global Replace All button in the search widget is enabled based solely on whether results are non-empty, ignoring whether all matches are in readonly files. The per-item readonly check exists but is not applied to the global button.

## Change summary

The global Replace All button in the search view should be disabled when all search result files are readonly.

## Best solution

In `searchView.ts`, the `refreshAndUpdateCount` method (line 792) should check whether ALL results are readonly (via existing `hasOnlyReadOnlyMatches()`) in addition to the `isEmpty()` check before enabling the global Replace All button. The infrastructure already exists — `FileMatch` and `FolderMatch` both have `hasOnlyReadOnlyMatches()`; the search result root just needs a similar aggregate check passed to `setReplaceAllActionState`.
