# Issue #184081 — Disable global replace all if all files are readonly

[https://github.com/microsoft/vscode/issues/184081](https://github.com/microsoft/vscode/issues/184081)

## Summary

Team-authored issue on the 'On Deck' milestone; the bug is still present on main — Replace All is enabled purely based on non-empty results with no readonly check. Keeping open per policy.

## Change summary

The global Replace All button in search remains enabled even when all matched files are readonly, misleading users into thinking a replace succeeded.

## Best solution

In `searchView.ts`, extend `refreshAndUpdateCount` (and related callers of `setReplaceAllActionState`) to also check whether any matched file actually supports replace (is writable). The per-file replace button already handles this via #177808; the global button needs the same guard.
