# Issue #247828 — Merge Editor: "The editor could not be opened because the file was not found" error after reload

[https://github.com/microsoft/vscode/issues/247828](https://github.com/microsoft/vscode/issues/247828)

## Summary

Team-authored issue, must keep open. The bug is still present on main: MergeEditorSerializer.canSerialize() unconditionally returns true and serializes git-scheme URIs that reference ephemeral merge state (MERGE_HEAD, index conflict entries). On reload, those URIs are unresolvable because the git merge context is gone.

## Change summary

Merge editor fails to reopen after VS Code reload because serialized git-scheme URIs referencing transient merge state become unresolvable.

## Best solution

Fix `canSerialize()` in `src/vs/workbench/contrib/mergeEditor/browser/mergeEditorSerializer.ts` to return `false` when the merge editor's input URIs use a transient scheme (like `git:`), or alternatively have the deserializer gracefully handle unresolvable URIs instead of showing an error. The git extension's merge editor integration should either avoid serialization of ephemeral state or re-establish the merge context on reload.
