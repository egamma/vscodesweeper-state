# Issue #267866 — Hide the 'Keep' 'Undo' and diff buttons on the working set until changes are complete

[https://github.com/microsoft/vscode/issues/267866](https://github.com/microsoft/vscode/issues/267866)

## Summary

Team-authored UX polish request that is still valid. The working set toolbar buttons (Keep All, Undo All, View Changes) and per-file buttons do not currently gate on request-in-progress, so they remain visible (disabled) during generation. The editor-content-level Keep/Undo buttons already hide via `ctxHasRequestInProgress.negate()`, but the working set equivalents do not.

## Change summary

Hide the Keep, Undo, and diff buttons in the chat editing working set toolbar while code generation is in progress, showing them only when changes are ready for review.

## Best solution

Add a `ChatContextKeys.requestInProgress.negate()` (or equivalent) condition to the `when` clauses for Keep All, Undo All, and View Changes on `MenuId.ChatEditingWidgetToolbar`, and for per-file Keep/Undo on `MenuId.ChatEditingWidgetModifiedFilesToolbar` in `src/vs/workbench/contrib/chat/browser/chatEditing/chatEditingActions.ts`. The editor-content buttons in `chatEditingEditorActions.ts` already do this (line 206) and can serve as a pattern. Needs a UX decision on whether the per-file diff button (`OpenFileInDiffAction`) should also be hidden.
