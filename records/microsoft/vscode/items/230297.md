# Issue #230297 — Editor: cannot Ctrl+click to goto definition in a link with a method name included

[https://github.com/microsoft/vscode/issues/230297](https://github.com/microsoft/vscode/issues/230297)

## Summary

Still reproduces on main: when a method call sits inside a URL (e.g. template literal), Ctrl+click always opens the link because the link detector claims the position first. Commit 5a1859bf44d (May 2026) even reinforced this by explicitly deferring goto-definition when a link is detected. Team-authored issue; keeping open.

## Change summary

Ctrl+click on a method name embedded in a URL opens the link instead of navigating to the method's definition.

## Best solution

When both a link and a definition exist at a position, allow go-to-definition to win when the click target is specifically a word/identifier (i.e. has a definition provider match), rather than always deferring to the link. The logic in `goToDefinitionAtPosition.ts` needs to be refined so that the link only wins when the click is on non-identifier portions of the link.
