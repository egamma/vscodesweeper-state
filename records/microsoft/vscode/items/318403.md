# Issue #318403 — Source Control accordion panels are not resizable like other accordions

[https://github.com/microsoft/vscode/issues/318403](https://github.com/microsoft/vscode/issues/318403)

## Summary

Feature request to make the SCM resource group sections (Changes, Staged Changes, etc.) individually resizable. The SCM view renders these as tree nodes in a single tree rather than as separate resizable accordion panes. No prior open issue owns this exact request.

## Change summary

Allow resizing the accordion-like sections (Changes, Staged Changes, etc.) within the Source Control view, similar to resizable panes in other sidebar views.

## Best solution

Route to the SCM area owner for evaluation. The resource groups are rendered as tree sections inside a single WorkbenchAsyncDataTree in `src/vs/workbench/contrib/scm/browser/scmViewPane.ts` — making them resizable would require either splitting them into separate view panes or adding a custom sash/splitter within the tree. The `scm.repositories.visible` setting addresses the Repositories pane height but not intra-tree section sizes.
