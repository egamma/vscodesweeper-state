# Issue #227477 — textsearchprovider - clarify that `includes` are relative to `folder`

[https://github.com/microsoft/vscode/issues/227477](https://github.com/microsoft/vscode/issues/227477)

## Summary

Team-authored polish issue requesting JSDoc clarification on the textSearchProvider2 API. The asymmetry between `includes: string[]` and `excludes: GlobPattern[]` is still undocumented on current main — the clarification has not been added.

## Change summary

Clarify in the textSearchProvider API docs that `includes` patterns are relative to `folder` (hence `string[]`) while `excludes` can reference a different `baseUri` (hence `GlobPattern[]`).

## Best solution

Add a JSDoc comment on `includes` in `src/vscode-dts/vscode.proposed.textSearchProvider2.d.ts` (line ~70) explaining that include patterns are resolved relative to `folder` and therefore use plain strings rather than `GlobPattern`. A one-line doc addition.
