# Issue #293314 — Copy Paste Not Working with Keyboard

[https://github.com/microsoft/vscode/issues/293314](https://github.com/microsoft/vscode/issues/293314)

## Summary

Intermittent focus-loss bug where the editor cursor is visible but keyboard shortcuts (Ctrl+C/V) don't fire — right-click restores focus. Reporter suspects Copilot inline suggestions may steal focus. Not enough data for a definitive root cause or fix, but the report is plausible and has multiple confirmations.

## Change summary

Ctrl+C/V intermittently stops working in the editor while right-click paste continues to work; right-clicking restores functionality.

## Best solution

Investigate whether inline completions (Copilot ghost text) or the EditContext implementation can leave the editor in a state where it appears focused (cursor visible) but does not receive keyboard events. The keybinding service logs in comment 7 ('Keyboard event cannot be dispatched') suggest the event reaches VS Code but the editor's input handler isn't the active target. Likely area: `src/vs/editor/browser/controller/editContext/` focus management.
