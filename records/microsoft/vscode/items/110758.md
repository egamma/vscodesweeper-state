# Issue #110758 — Provide an error code for "User did not consent to login" error

[https://github.com/microsoft/vscode/issues/110758](https://github.com/microsoft/vscode/issues/110758)

## Summary

Still unimplemented on main: the 'User did not consent to login' path still throws a plain Error with no error code or typed AuthenticationError in the public API. This is a team-authored issue in the Backlog with an api-proposal label, so it stays open.

## Change summary

Expose a typed error or error code from `authentication.getSession` when the user declines the login consent prompt, so extensions can detect the refusal without string-matching.

## Best solution

Introduce an `AuthenticationError` type (or error code property) in the proposed authentication API surface (`src/vscode-dts/`), analogous to `FileSystemError`, so extensions can programmatically detect 'user did not consent'. The error is thrown in `src/vs/workbench/api/browser/mainThreadAuthentication.ts`.
