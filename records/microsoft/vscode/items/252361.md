# Issue #252361 — Latest update breaks File Explorer view when folder openned is the root of a subst

[https://github.com/microsoft/vscode/issues/252361](https://github.com/microsoft/vscode/issues/252361)

## Summary

The workaround for this upstream Node.js bug was merged and shipped in 1.103.0. Both the reporter and another user confirmed the fix works in Insiders. This can be closed as completed.

## Change summary

File Explorer shows no files when a subst drive root is opened as a workspace folder, due to an upstream Node.js 22 regression in `fs.readdir`.

## Best solution

Keep the shipped workaround in commit d9d3f3c6ec0 (retry `fs.readdir` without trailing backslash on drive-letter roots). The upstream Node.js fix (nodejs/node#58970) will eventually remove the need for the workaround.
