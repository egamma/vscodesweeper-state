# Issue #136187 — Extension provided tasks are not recognized in workspace settings file only in .vscode/launch.json (for instance)

[https://github.com/microsoft/vscode/issues/136187](https://github.com/microsoft/vscode/issues/136187)

## Summary

Still reproduces on main: when a launch config lives in a `.code-workspace` file, the workspace root passed to `getTask` is keyed by the workspace-configuration URI, which never matches extension-provided tasks keyed by workspace-folder URIs, so the task is not found. Keeping open — the bug path is clear in the source.

## Change summary

Extension-provided preLaunchTask references in `.code-workspace` launch configurations fail with "Could not find the task" because the task lookup key mismatches between workspace-configuration URI and workspace-folder URI.

## Best solution

Fix the key mismatch in `abstractTaskService.ts#getTask` (and/or `debugService.ts` callers): when the `folder` parameter is an `IWorkspace`, the lookup should search across all workspace folders rather than matching only on the workspace-configuration URI. The `_findWorkspaceTasks` first pass at line 967–973 correctly iterates per-folder, but the `requestedFolder` guard at line 969 rejects every folder because `TaskMap.getKey(IWorkspace)` returns the `.code-workspace` URI. Similarly, the fallback `map.get(folder)` at line 987 returns nothing for the same reason.
