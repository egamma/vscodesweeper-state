# Issue #324531 — Uncommitted changes are in just "Changes view"

[https://github.com/microsoft/vscode/issues/324531](https://github.com/microsoft/vscode/issues/324531)

## Summary

Team-authored testing issue (MEMBER): uncommitted changes appear under the 'Changes' section instead of the expected 'Uncommitted changes' section in the SCM view when working in a folder (not a worktree). Keeping open per team-authored policy.

## Change summary

Uncommitted (not staged, not committed) files show in the 'Changes' group with a 'Reviewed' action instead of an 'Uncommitted changes' group in the SCM view.

## Best solution

The SCM view should categorize files that are neither staged nor committed into an 'Uncommitted changes' section (as introduced by the referenced PR #324394), rather than the generic 'Changes' section. The fix likely lives in the SCM view model or the git extension's repository grouping logic under `extensions/git/` and `src/vs/workbench/contrib/scm/`.
