# Issue #260220 — Copilot Chat: Search excludes files and folders that are in .gitignore

[https://github.com/microsoft/vscode/issues/260220](https://github.com/microsoft/vscode/issues/260220)

## Summary

Team-authored feature request with 42 upvotes. The `includeIgnoredFiles` tool parameter shipped in 1.107 partially addresses this, but the core ask — customizable Copilot-specific file exclusions (e.g. `.copilotignore`, per-tool ignore settings independent of `.gitignore`) — remains unimplemented. Keeping open.

## Change summary

Allow users to configure which files/folders Copilot search tools exclude, independently of `.gitignore` and workspace exclusions.

## Best solution

Implement a dedicated Copilot ignore mechanism (e.g. a `.copilotignore` file or a VS Code setting like `chat.search.exclude`) that lets users control which files are excluded from `file_search`, `grep_search`, and `semantic_search` independently of `.gitignore`. The `includeIgnoredFiles` parameter on `findTextInFilesTool` (in `extensions/copilot/src/extension/tools/node/findTextInFilesTool.tsx`) is a partial workaround but requires the model to opt in per-call rather than giving users declarative control.
