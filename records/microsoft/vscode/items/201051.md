# Issue #201051 — Use inline diff view for added and removed files by default

[https://github.com/microsoft/vscode/issues/201051](https://github.com/microsoft/vscode/issues/201051)

## Summary

Valid feature request: automatically use inline diff view for fully-added or fully-removed files. Not implemented on current main — the diff editor only switches to inline based on viewport width, not on file addition/removal status. Low upvote count but in Backlog milestone; keeping open.

## Change summary

When diffing a fully-added or fully-removed file, automatically use the inline diff view instead of side-by-side (which wastes half the viewport showing an empty pane).

## Best solution

Add logic in the diff editor options or the multi-diff editor to detect when one side is entirely empty (added/removed file) and default to inline view in that case. The relevant area is `src/vs/editor/browser/widget/diffEditor/diffEditorOptions.ts` where `renderSideBySide` is resolved.
