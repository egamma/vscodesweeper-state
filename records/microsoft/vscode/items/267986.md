# Issue #267986 — VS Code Microsoft account sync unexpectedly registers account in Windows

[https://github.com/microsoft/vscode/issues/267986](https://github.com/microsoft/vscode/issues/267986)

## Summary

Already correctly triaged as upstream — the Microsoft auth library (MSAL/OneAuth) registers accounts system-wide by design, and a team member confirmed VS Code cannot control this behavior. No action needed beyond what's already done.

## Change summary

Signing into VS Code with a Microsoft account registers that account system-wide in Windows (visible to OneDrive and other apps) due to the underlying Microsoft auth library's behavior.

## Best solution

This would need to be addressed in the Microsoft authentication library (MSAL/OneAuth) itself — either by offering a scoped/app-local token cache mode, or by the VS Code auth integration opting into such a mode if one becomes available upstream. No VS Code-side fix exists today.
