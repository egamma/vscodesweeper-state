# Issue #206471 — The file to paste has been deleted or moved. No actually, the target already exists.

[https://github.com/microsoft/vscode/issues/206471](https://github.com/microsoft/vscode/issues/206471)

## Summary

Still reproduces on main: the catch block at line 1280 of fileActions.ts wraps every paste error — including 'target already exists' — with the misleading 'deleted or moved' prefix. The related WIP PR #288377 was closed without merging. A team member (lramos15) acknowledged the bug and it has 14 upvotes.

## Change summary

Cut-pasting a file into a folder where a same-named file already exists shows a contradictory error message claiming the source was 'deleted or moved' when the real problem is a name conflict at the destination.

## Best solution

The generic catch block in `src/vs/workbench/contrib/files/browser/fileActions.ts` (line ~1279) should either detect the 'already exists' error and show the overwrite-confirmation dialog (like drag-and-drop does), or at minimum stop prefixing every error with the 'deleted or moved' text. The ideal fix would offer the same replace/skip dialog that DnD uses.
