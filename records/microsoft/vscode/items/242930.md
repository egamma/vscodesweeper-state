# Issue #242930 — Powershell extension activated for a chat terminal command

[https://github.com/microsoft/vscode/issues/242930](https://github.com/microsoft/vscode/issues/242930)

## Summary

Team-authored bug that remains open: PR #255578 fixed tool confirmation editors with `isSimpleWidget`, but connor4312 confirmed in comment 4 that the PowerShell extension still activates from terminal command code blocks. The terminal tool confirmation/progress parts in `toolInvocationParts/` do not use `isSimpleWidget`, and code blocks created via the `EditorPool` for terminal commands may still sync models to the extension host, triggering unwanted extension activation.

## Change summary

Chat terminal command code blocks trigger PowerShell extension activation because their editor models are synced to the extension host.

## Best solution

Ensure the code block editors used for chat terminal commands (in `chatTerminalToolConfirmationSubPart.ts` and the `EditorPool`/`CodeBlockPart` usage) pass `isSimpleWidget: true` so their models are not synced to the extension host, preventing spurious extension activation. The fix likely lives in the `codeBlockPart.ts` pool key path or in how the terminal confirmation sub-part requests editors.
