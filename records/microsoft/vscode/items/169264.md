# Issue #169264 — zsh HISTFILE Hardcoded after #168396 fix

[https://github.com/microsoft/vscode/issues/169264](https://github.com/microsoft/vscode/issues/169264)

## Summary

Still reproduces on main: the zsh shell integration script unconditionally overwrites HISTFILE before sourcing .zshrc, ignoring any custom value set via terminal.integrated.env or .zshenv. An open PR (#314615) is addressing this.

## Change summary

VS Code's zsh shell integration hardcodes HISTFILE to the default location, overriding user-configured custom history file paths.

## Best solution

Land PR https://github.com/microsoft/vscode/pull/314615 which preserves a custom HISTFILE when set via terminal.integrated.env or the user's shell init files. The fix belongs in `src/vs/workbench/contrib/terminal/common/scripts/shellIntegration-rc.zsh` around line 21-22.
