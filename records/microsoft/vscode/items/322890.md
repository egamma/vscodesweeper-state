# Issue #322890 — New Chat Prompt Lost

[https://github.com/microsoft/vscode/issues/322890](https://github.com/microsoft/vscode/issues/322890)

## Summary

Valid bug: draft text typed in a new (unsent) chat session is lost when switching to a previous conversation and back. Active work on draft persistence exists on main (commit b2dd4c84d70, June 2026), but the specific new/untitled-session scenario appears to still have gaps. A near-identical report exists at #324149.

## Change summary

Unsent prompt text in a new Copilot Chat conversation is discarded when the user switches to another conversation and returns.

## Best solution

Extend the draft input persistence in `src/vs/workbench/contrib/chat/browser/widget/input/chatInputPart.ts` and `chatInputStatePersistence.ts` to cover untitled/new chat sessions that have not yet sent a message. Consolidate with https://github.com/microsoft/vscode/issues/324149 which describes the same gap more precisely.
