# Issue #74094 — TextEditorDecoration applied to incorrect range if racing with user edits

[https://github.com/microsoft/vscode/issues/74094](https://github.com/microsoft/vscode/issues/74094)

## Summary

Team-authored feature request with 32 upvotes and an open community PR (#172672) — must stay open. The race between setDecorations and user edits is a real gap; no fix has landed on main.

## Change summary

TextEditorDecoration ranges are applied against a stale document version when user edits arrive between the extension capturing the document state and calling setDecorations, causing decorations to appear on the wrong text.

## Best solution

Review and land (or close) the open PR https://github.com/microsoft/vscode/pull/172672, which adds version-aware decoration application to reject or adjust ranges when the document version has advanced. The fix touches `extHostTextEditor.ts` and `mainThreadEditor.ts`.
