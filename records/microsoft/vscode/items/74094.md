# Issue #74094 — TextEditorDecoration applied to incorrect range if racing with user edits

[https://github.com/microsoft/vscode/issues/74094](https://github.com/microsoft/vscode/issues/74094)

## Summary

This should stay open for maintainer judgment: it was authored by a VS Code member and is assigned to the On Deck milestone, both of which are hard keep-open signals under the cleanup rules. The report describes a real API race around decoration ranges and pending document edits, with later discussion pointing toward an API/versioning design rather than a stale support issue.

## Change summary

TextEditor.setDecorations can apply ranges to the wrong text when an extension computes a range during didChangeTextDocument while the user makes additional edits before the handler returns.

## Best solution

Keep this in the editor API/editor decoration backlog for an explicit maintainer decision on whether setDecorations should capture or accept a document version and reject/remap stale ranges. The likely owning areas remain editor-core and semantic-tokens/API-facing decoration behavior.
