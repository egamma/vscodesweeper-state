# Issue #320565 — Chromebook - High Density is not working properly

[https://github.com/microsoft/vscode/issues/320565](https://github.com/microsoft/vscode/issues/320565)

## Summary

Verified duplicate of the open canonical issue for fractional scaling cursor inaccuracy on Chromebooks — the reporter describes the same click-offset and rendering symptoms when using ChromeOS display scaling at non-100% values.

## Change summary

On Chromebooks with fractional display scaling (e.g. 70% / high density), clicks register at the wrong position and the window does not maximize properly.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/160702 which tracks fractional-scaling cursor inaccuracy on Chromebooks. The root cause is the same: non-integer device pixel ratios on ChromeOS cause coordinate mismatches in Electron/Chromium rendering. The workaround noted there (`"editor.disableMonospaceOptimizations": true`) may help with editor cursor drift specifically.
