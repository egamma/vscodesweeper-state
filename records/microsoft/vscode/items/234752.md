# Issue #234752 — Staging Lines Changes EOL of Whole File

[https://github.com/microsoft/vscode/issues/234752](https://github.com/microsoft/vscode/issues/234752)

## Summary

Still reproducible on main: `applyLineChanges` in `extensions/git/src/staging.ts` builds the staged content from `TextDocument.getText()` calls which return text with VS Code's internal LF line endings, discarding the original CRLF. No EOL preservation logic exists in this path.

## Change summary

Staging individual lines/blocks via the diff editor gutter '+' button converts the entire file's EOL from CRLF to LF on Windows.

## Best solution

The `applyLineChanges` function (or the `repository.stage` call site in `_stageChanges`) should detect the original file's EOL and normalize the assembled result string to match before passing it to git. The fix is localized to `extensions/git/src/staging.ts` and/or `extensions/git/src/commands.ts`.
