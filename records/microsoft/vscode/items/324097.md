# Issue #324097 — minimum release age should not show badging for deferred updates

[https://github.com/microsoft/vscode/issues/324097](https://github.com/microsoft/vscode/issues/324097)

## Summary

The activity bar badge already excludes auto-update-delayed extensions (since 1.124.0), but the per-extension 'Update' button and the @outdated list still show them without indicating the deferral context — the remaining ask is UX polish for that surface.

## Change summary

Extensions with deferred auto-updates still appear in the outdated list with an 'Update' button, without clear indication they are intentionally held back by the delay setting.

## Best solution

The UpdateAction in extensionsActions.ts could either hide or relabel itself for delayed extensions (e.g. 'Update available — auto-installing in Xh'), and the @outdated view could separate or annotate delayed items so users can distinguish safely-deferred updates from ones requiring manual action.
