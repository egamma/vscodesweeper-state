# Issue #232538 — Give more control over syncing rapidly updating documents with extensions

[https://github.com/microsoft/vscode/issues/232538](https://github.com/microsoft/vscode/issues/232538)

## Summary

Keep open as a real API design request: current extension document sync still forwards model changes to the extension host without an obvious throttling/suspension control, and there is no linked duplicate or fixing commit in the issue discussion.

## Change summary

Provide extension/API control over synchronization for rapidly changing text models such as streaming chat code blocks.

## Best solution

API and inline chat owners should decide whether rapidly streaming virtual/chat documents need a core sync primitive, such as coalescing, suspension, or a completion signal, likely around the MainThreadDocuments to ExtHostDocuments sync path.
