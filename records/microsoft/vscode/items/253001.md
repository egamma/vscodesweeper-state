# Issue #253001 — Pickers that require a button to confirm should not close on focus lost

[https://github.com/microsoft/vscode/issues/253001](https://github.com/microsoft/vscode/issues/253001)

## Summary

Active design discussion among team members about whether quick picks with confirm buttons should ignore focus-out. The participants agreed to handle it case-by-case; the broader question remains open for future feedback.

## Change summary

Quick pickers that require a button to confirm (e.g. multi-select tools picker) dismiss on focus loss, discarding the user's selection.

## Best solution

Keep as a design-discussion tracking issue. Individual pickers (like the tools picker) can be fixed ad-hoc by setting `ignoreFocusOut: true`; whether to enforce this globally for all button-confirmed pickers remains an open UX/API question given the extension API implications Tyler raised.
