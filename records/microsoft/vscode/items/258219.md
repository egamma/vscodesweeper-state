# Issue #258219 — Toggle chat mode command works when dropdown is disabled

[https://github.com/microsoft/vscode/issues/258219](https://github.com/microsoft/vscode/issues/258219)

## Summary

Still reproduces on current main: the OpenModePickerAction (Ctrl+.) lacks the requestInProgress guard that was on the ToggleChatModeAction before the keybinding was moved. The mode picker can be opened via keyboard while a request is in progress, even though the dropdown UI is disabled.

## Change summary

The Ctrl+. keybinding opens the chat mode picker even while a request is in progress, bypassing the disabled state of the dropdown.

## Best solution

Add `ChatContextKeys.requestInProgress.negate()` to the `OpenModePickerAction` precondition and/or keybinding `when` clause in `src/vs/workbench/contrib/chat/browser/actions/chatExecuteActions.ts`, matching the guard that exists on `ToggleChatModeAction`.
