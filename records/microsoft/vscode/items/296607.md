# Issue #296607 — Navigate to next/previous changed file when reaching end of changes in current file (like IntelliJ)

[https://github.com/microsoft/vscode/issues/296607](https://github.com/microsoft/vscode/issues/296607)

## Summary

Valid feature request for cross-file change navigation in the single-file SCM diff editor. The multi-diff editor already has this (navigates across files at change boundaries), but the regular diff/quick-diff view does not. No duplicate found — keeping open for the area owner.

## Change summary

When using next/previous change navigation in a single-file diff editor, allow jumping to the next/previous changed file when reaching the boundary of changes in the current file.

## Best solution

Extend the single-file diff change navigation (`workbench.action.editor.nextChange` / `previousChange` in `src/vs/workbench/contrib/scm/browser/quickDiffWidget.ts`) to optionally open the next/previous changed file from the SCM provider when the user is at the last/first change. The multi-diff editor already implements cross-file navigation in `src/vs/editor/browser/widget/multiDiffEditor/multiDiffEditorWidgetImpl.ts` — the SCM diff view could follow a similar pattern, consulting the SCM change list for the next file.
