## Change summary

Improve merge editor to diff input1 vs input2 (not just each vs base) so that identical edits on both sides are handled more intelligently, matching git's `merge` conflict style rather than `diff3`.

## Best solution

Implement an input1-vs-input2 diff pass in the merge editor's conflict detection logic (likely in `src/vs/workbench/contrib/mergeEditor/`) to auto-resolve or better present conflicts where both sides made identical or overlapping edits relative to base.
