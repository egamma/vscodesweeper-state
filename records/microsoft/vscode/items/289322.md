## Summary

The reported 1–2 pixel transparent gap on non-maximized windows is a well-known Windows platform behavior: non-maximized windows retain invisible resize borders even when snapped to fill the screen. This is expected OS behavior and likely upstream (Electron/Windows). The report lacks enough detail to confirm whether it deviates from normal Electron window handling or is specific to FancyZones interaction. Requesting more info.

## Change summary

A 1–2 pixel transparent gap appears on all four sides of the VS Code window when snapped to fill the screen (non-maximized) using FancyZones on Windows.
