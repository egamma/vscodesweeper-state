# Issue #166927 — When cursor moves horizontally, it shows two different behaviors when passing `inlayHints`

[https://github.com/microsoft/vscode/issues/166927](https://github.com/microsoft/vscode/issues/166927)

## Summary

Still a valid open discussion. The issue reports asymmetric line-cursor behavior around inlay hints (left-to-right vs right-to-left). The recent commit a2f1c104a1a only addresses the block cursor case (#209417), leaving the original line-cursor UX concern unresolved. A team member (jrieken) agreed it's confusing and worth considering an option.

## Change summary

When moving a line cursor horizontally past inlay hints, left-to-right and right-to-left behave asymmetrically — the cursor appears to skip a real character in one direction but not the other.

## Best solution

Decide whether to add a setting (or change the default) so that the line cursor treats inlay hints as non-blocking in both directions, making horizontal movement symmetric. The fix would likely live in `src/vs/editor/common/cursor/cursorMoveOperations.ts` alongside the existing block-cursor logic.
