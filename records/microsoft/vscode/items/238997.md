# Issue #238997 — Pressing up on quick pick with checkboxes goes to the first item, not the last

[https://github.com/microsoft/vscode/issues/238997](https://github.com/microsoft/vscode/issues/238997)

## Summary

Still present on main: `shouldLoop` is explicitly set to `false` when `canSelectMany` is true. A team member agrees it should loop by default. Valid enhancement request.

## Change summary

Quick pick with checkboxes (canSelectMany) does not wrap around when pressing Up on the first item, unlike the normal command palette.

## Best solution

Change the looping behavior in `src/vs/platform/quickinput/browser/quickInput.ts` (line 1093) so that arrow navigation wraps even when `canSelectMany` is true, or decouple the loop setting from checkbox mode while preserving the space-bar toggle ergonomics.
