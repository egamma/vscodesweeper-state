# Issue #307942 — Contents in extra windows can be rendered out-of-view

[https://github.com/microsoft/vscode/issues/307942](https://github.com/microsoft/vscode/issues/307942)

## Summary

Legitimate layout bug: the extension editor header clips action buttons in auxiliary windows because the installed-extension layout (list-layout class) increases content height beyond the header's overflow:hidden boundary. No fix found on current main.

## Change summary

Extension editor action buttons (Disable, Uninstall, Settings) are clipped/hidden when the editor is moved to an auxiliary window.

## Best solution

The header's `overflow: hidden` in `extensionEditor.css` (line 24) clips the taller installed-extension layout in auxiliary windows. The fix likely involves ensuring the header height accommodates the `list-layout` variant or adjusting overflow behavior. The related issue #271260 (inline chat buttons out-of-view in separate windows) was closed as obsolete for its specific case, but this extension editor instance remains unfixed.
