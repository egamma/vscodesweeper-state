# Issue #320928 — Webview resource cache grows unboundedly: media-preview cache-busts with Date.now(), service worker never evicts (21+ GB observed)

[https://github.com/microsoft/vscode/issues/320928](https://github.com/microsoft/vscode/issues/320928)

## Summary

Confirmed on current main: media-preview cache-busts with Date.now() on every render (line 181 of imagePreview/index.ts) and the service worker caches every response with no eviction (service-worker.js has no caches.delete, no LRU, no old-VERSION cleanup in its activate handler). An open PR (#321358) addresses the media-preview half; the service-worker eviction gap remains unaddressed.

## Change summary

The webview service worker caches every resource response keyed by full URL, and media-preview appends a unique Date.now() query on every render, causing unbounded disk growth (21+ GB observed).

## Best solution

Land the open PR https://github.com/microsoft/vscode/pull/321358 to stabilize the cache key (mtime-based instead of Date.now()). Separately, add eviction logic to service-worker.js: delete previous-VERSION caches in the activate handler, and consider an LRU or size cap on vscode-resource-cache entries to prevent unbounded growth from any future cache-busting source.
