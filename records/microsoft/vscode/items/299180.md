# Issue #299180 — Session name should be the last user prompt in the session

[https://github.com/microsoft/vscode/issues/299180](https://github.com/microsoft/vscode/issues/299180)

## Summary

Feature request to dynamically update chat session titles based on the most recent user prompt instead of LLM-generated titles. This is a UX design decision for the chat/sessions area — no duplicate found, but it has zero community traction so far.

## Change summary

Replace LLM-generated chat session titles with the most recent user prompt text, keeping session names contextually relevant as conversations evolve.

## Best solution

Route to the chat/sessions area owner for a product decision. The current LLM-generated title approach is intentional — switching to raw last-prompt text would lose semantic summarization. A middle ground (e.g. re-generating the title periodically as the conversation evolves, or offering a user setting) may be more appropriate. Related request https://github.com/microsoft/vscode/issues/313971 asks for an on-demand 'regenerate title' action, which partially overlaps.
