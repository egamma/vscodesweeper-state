# Issue #272927 — Unreliable middle-mouse-button paste in integrated terminal on Linux

[https://github.com/microsoft/vscode/issues/272927](https://github.com/microsoft/vscode/issues/272927)

## Summary

Still reproduces on main — the middle-click handler just calls `focus()` and relies on the browser to paste, but xterm.js intercepts the mouse event over words (for word-selection) before the browser can perform primary-selection paste. An open PR #307884 addresses this by explicitly pasting from the primary selection clipboard on Linux.

## Change summary

Middle-mouse-button paste in the integrated terminal on Linux fails when clicking over non-blank characters because xterm.js intercepts the mouse event for word selection.

## Best solution

Land the open PR https://github.com/microsoft/vscode/pull/307884 which explicitly pastes the primary selection clipboard on Linux when `middleClickBehavior` is `default`, bypassing the xterm.js word-selection interception.
