# Issue #199141 — Double-click on scrollbar creates a new file, and it should not ...

[https://github.com/microsoft/vscode/issues/199141](https://github.com/microsoft/vscode/issues/199141)

## Summary

Still reproduces on main: double-clicking the scrollbar in the explorer fires the 'New File' action because the scrollbar does not stop propagation of `dblclick` events, and the explorer's `onMouseDblClick` handler treats any double-click on empty space as a new-file trigger. An open PR (#199334) exists but has not been merged.

## Change summary

Double-clicking the file explorer scrollbar erroneously triggers the 'New File' action.

## Best solution

Review and land the open PR https://github.com/microsoft/vscode/pull/199334 (or an equivalent fix) that stops propagation of `dblclick` from the scrollbar, so the explorer's empty-space handler in `explorerView.ts:581–588` is never reached when the scrollbar is the click target.

## Evidence

- **source trace:** src/vs/workbench/contrib/files/browser/views/explorerView.ts:581–587 fires `NEW_FILE_COMMAND_ID` on any `onMouseDblClick` with `e.element === null` (empty space). The scrollbar's `dblclick` bubbles up because src/vs/base/browser/ui/scrollbar/abstractScrollbar.ts only stops propagation of `click`, not `dblclick`.
- **list view event source:** src/vs/base/browser/ui/list/listView.ts:1122 listens for `dblclick` on the list's domNode, which includes the scrollbar — any unhandled dblclick bubbles into this.
- **open PR:** https://github.com/microsoft/vscode/pull/199334 is still open and unmerged; it proposes stopping dblclick propagation from the scrollbar.
- **no fix on main:** git log on src/vs/base/browser/ui/scrollbar/ shows no commit addressing dblclick propagation since the issue was filed.
