# Issue #108757 — Git - Removing a folder from multi-root workspace does not remove repo

[https://github.com/microsoft/vscode/issues/108757](https://github.com/microsoft/vscode/issues/108757)

## Summary

Still reproduces on main — the activeRepositories guard in model.ts prevents repo disposal when an editor is open on a file from a removed workspace folder. An open PR (#284139) addresses this, and the issue carries the 'On Deck' milestone, so it should stay open.

## Change summary

Removing a folder from a multi-root workspace does not remove its Git repository from the SCM pane when a file from that repo is open in an editor.

## Best solution

Review and land https://github.com/microsoft/vscode/pull/284139, which removes the activeRepositories guard so repository disposal is driven purely by workspace folder membership. The fix touches `extensions/git/src/model.ts` in `onDidChangeWorkspaceFolders`. Consider whether the open-editor case needs any user-facing prompt or just silent disposal.
