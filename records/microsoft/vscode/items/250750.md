# Issue #250750 — Search not always searching in open files outside of work folder

[https://github.com/microsoft/vscode/issues/250750](https://github.com/microsoft/vscode/issues/250750)

## Summary

Plausible bug: after restart, search does not include open files outside the workspace until each tab is visited. The mechanism (`getOutOfWorkspaceEditorResources` → `extraFileResources`) is intact on main with no related fix, but the report lacks confirmation from other users and needs reproduction to pin down whether the issue is in editor restoration timing, search provider indexing, or the `fileService.hasProvider` check for lazily-restored editors.

## Change summary

After restarting VS Code, the search view does not search in pinned files that are outside the opened workspace folder until the user manually switches to each tab.

## Best solution

Investigate whether lazily-restored editors outside the workspace are correctly reported by `getOutOfWorkspaceEditorResources` (in `src/vs/workbench/contrib/search/common/search.ts`) immediately after restart. The likely root cause is either that `EditorResourceAccessor.getOriginalUri` returns undefined for not-yet-resolved editors, or that the text search provider cannot read those files until their models are loaded. A fix would ensure the URI is available from the serialized editor input even before resolution, or that the search falls back to disk for `extraFileResources`.
