# Issue #321881 — VS Code can become stuck reopening the Agent login window and prevent access to the normal editor

[https://github.com/microsoft/vscode/issues/321881](https://github.com/microsoft/vscode/issues/321881)

## Summary

Keep open as a plausible agents-window launch/restore bug: the reporter retested on 1.125.0, another user confirmed, and the current code still has a distinct agents-window startup path that can bypass the normal editor window flow.

## Change summary

Launching VS Code after closing the normal editor before the Agent login window can reopen only the Agent login experience and leave the user without access to the normal editor.

## Best solution

Agents window startup/restoration should ensure a normal desktop launch or explicit folder/file open cannot be trapped in the agents login surface, likely in the Electron startup/open flow around src/vs/code/electron-main/app.ts and src/vs/platform/windows/electron-main/windowsMainService.ts.
