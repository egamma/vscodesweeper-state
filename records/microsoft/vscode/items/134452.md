# Issue #134452 — Support global evaluation that requires a file for context

[https://github.com/microsoft/vscode/issues/134452](https://github.com/microsoft/vscode/issues/134452)

## Summary

This feature request for passing the active editor's source file as context in DAP global evaluation (no frameId) remains unimplemented on current main. The discussion between the issue author and a VS Code maintainer (weinand) produced a concrete proposal (option 2: pass active editor as a Source attribute on the evaluate request) but work was never finalized. Keeping open as a valid, discussed-but-unimplemented debug feature request.

## Change summary

When evaluating expressions in the debug console without a stopped stack frame, VS Code should pass the active editor's file as a Source attribute in the DAP evaluate request to provide context for global evaluation.

## Best solution

Finalize the proposal from the 2021–2022 discussion: add an optional `source` (or `activeSource`) attribute to the DAP `EvaluateArguments` carrying the active editor's file, and have VS Code populate it when `frameId` is absent. The implementation would touch `src/vs/workbench/contrib/debug/browser/debugSession.ts` (the `evaluate` method) and `src/vs/workbench/contrib/debug/common/debugModel.ts` (the `evaluateExpression` call site). A DAP spec change would also be needed upstream. The DebugAdapterTracker workaround exists for extensions in the interim.
