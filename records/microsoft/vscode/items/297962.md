# Issue #297962 — Provide a way for custom test sessions to be re-run with commands like "Rerun last session"

[https://github.com/microsoft/vscode/issues/297962](https://github.com/microsoft/vscode/issues/297962)

## Summary

Valid feature request with milestone 'On Deck' — must keep open. The capability to attach rerun metadata to extension-created test runs does not exist on current main; the gap remains real.

## Change summary

Allow extensions to attach metadata to a TestRun so that 'Rerun last session' can reproduce custom/external test sessions.

## Best solution

Add a metadata/opaque-data field to the TestRun creation API that VS Code stores and passes back when the user invokes 'Rerun last run', enabling the extension to reconstruct the original launch configuration. Implementation would touch `src/vs/workbench/api/common/extHostTesting.ts` (createTestRun), the TestRunRequest DTO, and the rerun action in `testExplorerActions.ts`.
