# Issue #193164 — Editor is broken when opening files with >120M lines

[https://github.com/microsoft/vscode/issues/193164](https://github.com/microsoft/vscode/issues/193164)

## Summary

Keep open: the issue is in Backlog, has maintainer discussion, and current main still contains the 32-bit integer coercions called out in the report. There is no concrete fixing commit or canonical duplicate in the provided context or local history.

## Change summary

Opening files with more than roughly 120 million lines can overflow editor scroll height math and break scrolling/rendering.

## Best solution

The editor should either fail gracefully for files whose rendered height exceeds the 32-bit scroll math limit, or intentionally replace the affected SMI/int32 coercions in the editor scroll/layout path after measuring the performance impact.
