# Issue #184047 — Display disabled tasks in Run Build Task... menu

[https://github.com/microsoft/vscode/issues/184047](https://github.com/microsoft/vscode/issues/184047)

## Summary

Valid feature request with 20 upvotes, promoted to Backlog. No implementation exists on current main — the Task API has no disabled/unavailable concept. Keeping open as an accepted backlog item.

## Change summary

Add a way to display tasks as disabled/greyed-out in the Run Build Task menu when they are temporarily unavailable.

## Best solution

Add a `disabled` (or similar) property to `vscode.Task` in the extension API that task providers can set in `provideTasks` to mark tasks as visible but not actionable. The quick-pick UI would render them greyed out. Likely touches `src/vscode-dts/vscode.d.ts` (API surface), `src/vs/workbench/api/common/extHostTypes.ts` (ext host Task class), and `src/vs/workbench/contrib/tasks/browser/abstractTaskService.ts` (quick-pick rendering).
