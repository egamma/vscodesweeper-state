# Issue #233525 — Git diff does not work over a Samba share

[https://github.com/microsoft/vscode/issues/233525](https://github.com/microsoft/vscode/issues/233525)

## Summary

The report describes a real class of path-resolution bugs with GVfs-mounted Samba shares, but lacks the diagnostic detail needed to pinpoint the root cause — specifically what git repository root VS Code discovers vs. the GVfs mount path, and what the Git extension output log shows.

## Change summary

VS Code's git diff shows files as entirely new (empty left pane) when working on a repository accessed via a GVfs Samba mount, while `git diff` in the terminal works correctly.

## Best solution

Ask the reporter for the Git extension output log (`Git` output channel) and the result of `git rev-parse --show-toplevel` run from the GVfs path. The likely root cause is a path mismatch in `sanitizeRelativePath` in `extensions/git/src/git.ts` — the GVfs mount path doesn't match `repositoryRoot` or `repositoryRootRealPath`, so `git show ref:path` receives a broken relative path. A fix would need to handle the GVfs path normalization, but the exact mismatch needs to be diagnosed first.
