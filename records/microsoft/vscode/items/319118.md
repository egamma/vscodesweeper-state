# Issue #319118 — Auto-completion disabled for languages with ID >= 256 due to LanguageID bits overlapping with TokenType bits

[https://github.com/microsoft/vscode/issues/319118](https://github.com/microsoft/vscode/issues/319118)

## Summary

Keep open: the report describes a concrete failure mode and current main still has the same 8-bit token metadata layout with unmasked language IDs flowing into adjacent token-type bits. There is not enough evidence for a close; this needs an editor/tokenization owner decision or fix.

## Change summary

Auto-triggered quick suggestions can be disabled for languages whose encoded language ID is 256 or higher because token metadata bit fields overlap.

## Best solution

Preserve token metadata field integrity in the editor tokenization path, either by masking encoded fields at construction sites or by enforcing/documenting the language ID capacity before metadata is created, with a regression test for LanguageId >= 256 and quick suggestion token classification.

## Evidence

- **current construction path:** src/vs/editor/common/tokens/contiguousTokensStore.ts:51 and :247-256 still pass the encoded language ID into getDefaultMetadata and shift it without applying LANGUAGEID_MASK.
- **quick suggestions dependency:** src/vs/editor/contrib/suggest/browser/suggestModel.ts:423-431 reads lineTokens.getStandardTokenType(...) before deciding whether quick suggestions should auto-trigger; src/vs/editor/contrib/suggest/browser/suggest.ts:464-469 maps Comment/String differently from Other.
