# Issue #319118 — Auto-completion disabled for languages with ID >= 256 due to LanguageID bits overlapping with TokenType bits

[https://github.com/microsoft/vscode/issues/319118](https://github.com/microsoft/vscode/issues/319118)

## Summary

Confirmed on current main: `getDefaultMetadata` in `contiguousTokensStore.ts` does not mask `topLevelLanguageId` before shifting, so language IDs >= 256 overflow into the TokenType bits, silently breaking quickSuggestions. The related fix PR #319144 was closed without merging.

## Change summary

Languages with numeric IDs >= 256 have their default token metadata corrupted because the language ID overflows into TokenType bits, disabling auto-completion.

## Best solution

Mask `topLevelLanguageId` with `& 0xFF` (the `LANGUAGEID_MASK`) before the shift in `getDefaultMetadata` at `src/vs/editor/common/tokens/contiguousTokensStore.ts:249`. This prevents overflow into adjacent bit fields. A similar guard may be needed in other metadata-packing call sites.
