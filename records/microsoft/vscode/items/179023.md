# Issue #179023 — Feature Request: `enablement` property on `MessageItem`s and `QuickPickItem`s

[https://github.com/microsoft/vscode/issues/179023](https://github.com/microsoft/vscode/issues/179023)

## Summary

Team-authored feature request for an `enablement` property on QuickPickItem/MessageItem to allow graying out items instead of hiding them. Acknowledged by another team member as sensible. Hard keep-open (MEMBER author).

## Change summary

Add an `enablement` property to `vscode.MessageItem` and `vscode.QuickPickItem` so extensions can gray out and disable items based on context conditions rather than hiding them entirely.

## Best solution

Implement an `enabled`/`disabled` property on QuickPickItem (and optionally MessageItem) in the extension API, rendering disabled items as grayed-out and non-selectable. The work would touch `src/vs/workbench/api/common/extHostQuickOpen.ts`, the QuickPick widget in `src/vs/base/parts/quickinput/`, and the `vscode.d.ts` typings. A workaround exists (separator + no-op on accept), but the native property would be cleaner for extensions supporting multiple environments.
