# Issue #249508 — Can't copy the whole line if there is a null character in it. When trying to copy the entire line, only the part before the null character is copied.

[https://github.com/microsoft/vscode/issues/249508](https://github.com/microsoft/vscode/issues/249508)

## Summary

Still reproduces on main: clipboard copy does not strip or escape NUL (\u0000) characters, causing the Windows clipboard API to truncate text at the first NUL. An open PR (#305684) addresses this but has not been merged.

## Change summary

Copying a line containing a NUL character only copies text up to the first NUL, because the clipboard API treats NUL as a string terminator.

## Best solution

Strip or replace NUL characters (\u0000) from text before writing to the clipboard in the editor clipboard contribution (`src/vs/editor/contrib/clipboard/browser/clipboard.ts`) or in the platform clipboard service. PR https://github.com/microsoft/vscode/pull/305684 is in progress.
