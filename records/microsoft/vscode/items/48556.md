# Issue #48556 — [folding] fold all but comments

[https://github.com/microsoft/vscode/issues/48556](https://github.com/microsoft/vscode/issues/48556)

## Summary

Still a valid, unimplemented feature request. The existing `editor.foldAllBlockComments` folds comments themselves; the inverse — fold everything except comments — does not exist on current main.

## Change summary

Request a command to fold all code regions except comments, so that comments remain visible for a quick file overview.

## Best solution

Add a new editor action (e.g. `editor.foldAllExceptComments`) in `src/vs/editor/contrib/folding/browser/folding.ts` that calls `setCollapseStateForType` to collapse all ranges whose kind is NOT `FoldingRangeKind.Comment`. This mirrors the existing `FoldAllBlockCommentsAction` logic but inverts the type filter.
