# Issue #194902 — Substitution performed when accepting a Kanji conversion of Japanese input method

[https://github.com/microsoft/vscode/issues/194902](https://github.com/microsoft/vscode/issues/194902)

## Summary

This appears fixed on current main by the IME composition guard added to the editor find/replace Enter keybinding. The fix directly matches the reported failure mode: Enter in the replace input no longer invokes Replace One while composition is active.

## Change summary

Accepting a Japanese IME Kanji conversion with Enter in the editor Find/Replace widget also triggered the replacement action.

## Best solution

Treat this as completed by commit abd3802747f015b3822493424cd40c0914f969cc, which makes Replace One on Enter require `!isComposing`; no additional core behavior change is needed unless a fresh reproduction is found on a build containing that commit.
