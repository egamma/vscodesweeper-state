# Issue #211851 — Task doesn't execute bash commands in background (with &)

[https://github.com/microsoft/vscode/issues/211851](https://github.com/microsoft/vscode/issues/211851)

## Summary

This is by-design behavior: VS Code's task system monitors the shell process exit to determine task completion. When a script backgrounds a subprocess with `&`, the shell exits immediately, and the task terminal lifecycle may terminate the child process. The task `isBackground` property controls VS Code's own background-task concept (problem-matcher-driven watching), not shell-level process backgrounding.

## Change summary

Running a bash script that uses `&` to background a subprocess from a VS Code task results in the backgrounded command not completing its work.

## Best solution

Close as designed. VS Code tasks monitor shell process exit; shell-level backgrounding with `&` causes the shell to return immediately. Users should either use `wait` in their scripts, avoid `&` in task scripts, or use the task's `isBackground: true` with a problem matcher for long-running watch-style tasks. The workaround of removing `&` (as the reporter noted works) is the correct approach for task-driven scripts.
