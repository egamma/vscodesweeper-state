# Issue #243686 — Ubuntu: Unable to move/drag around native popup windows independently from the main electron app

[https://github.com/microsoft/vscode/issues/243686](https://github.com/microsoft/vscode/issues/243686)

## Summary

This is an upstream Electron/Chromium Wayland issue. VS Code uses standard Electron dialog APIs (electron.dialog.showOpenDialog with parent window), and the inability to independently drag modal dialogs on Wayland is a platform/compositor behavior outside VS Code's control. Should be labeled as upstream and tracked against Electron/Chromium's Ozone Wayland backend.

## Change summary

On Ubuntu with Wayland, native file-open dialogs cannot be dragged independently from the main VS Code window — dragging the dialog moves the entire app.

## Best solution

Label as `upstream` and `linux`. The fix belongs in Electron or Chromium's Ozone/Wayland backend, which controls how modal child windows (xdg_toplevel with parent) interact with Wayland compositors. VS Code's dialog code in `src/vs/platform/dialogs/electron-main/dialogMainService.ts` correctly uses `electron.dialog.showOpenDialog(window, options)` — the standard Electron API. Filing or linking an upstream Electron issue would help track resolution.
