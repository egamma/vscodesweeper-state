# Issue #303257 — Timeline: operator precedence bug in pageSize calculation causes ~24x over-fetch

[https://github.com/microsoft/vscode/issues/303257](https://github.com/microsoft/vscode/issues/303257)

## Summary

Confirmed on current main: the operator precedence bug is still present at line 332 of timelinePane.ts, and an open PR (#303258) already proposes the fix. Keeping open pending that PR.

## Change summary

Operator precedence causes timeline pageSize to use raw pixel height (~500) instead of dividing by ItemHeight (22), resulting in ~24x over-fetch on every timeline load.

## Best solution

Land PR https://github.com/microsoft/vscode/pull/303258 which adds parentheses around `(this.tree?.renderHeight ?? 0)` to enforce division by ItemHeight. The fix is a one-character-class change in `src/vs/workbench/contrib/timeline/browser/timelinePane.ts`.
