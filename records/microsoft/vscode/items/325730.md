# Issue #325730 — workbench.action.chat.nextUserPrompt/previousUserPrompt reveal the target prompt at inconsistent viewport positions

[https://github.com/microsoft/vscode/issues/325730](https://github.com/microsoft/vscode/issues/325730)

## Summary

Confirmed on current main: the prompt-navigation commands call `widget.reveal(targetPrompt)` without a `relativeTop` argument, producing direction-dependent viewport alignment. A small, well-scoped fix.

## Change summary

The next/previous user-prompt navigation commands reveal the target prompt at inconsistent viewport positions (top vs bottom) depending on scroll direction.

## Best solution

Pass a fixed `relativeTop` (e.g. `0` for top-aligned) to `widget.reveal(targetPrompt, 0)` in `chatPromptNavigationActions.ts` line 119 so both directions land the prompt at the same viewport position.
