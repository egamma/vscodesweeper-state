[https://github.com/microsoft/vscode/issues/278102](https://github.com/microsoft/vscode/issues/278102)

## Summary

Piping to `code -` in WSL remains broken per recent reports (1.114.0). The server CLI stdin path exists but something prevents it from working in WSL. The linked #300383 shows mixed results even on Windows. This is an active bug, not a feature request.

## Change summary

Piping output to `code -` from a WSL terminal does not open the piped content in the editor.

## Best solution

Investigate why `hasStdinWithoutTty()` or the stdin data flow in `src/vs/server/node/server.cli.ts` fails in the WSL remote CLI path. The fix likely lives in the remote CLI stdin handling (lines ~189–210 of server.cli.ts) or in how the WSL helper script forwards stdin. Should also be reconciled with the Windows-side issue https://github.com/microsoft/vscode/issues/300383 which reports the same symptom.
