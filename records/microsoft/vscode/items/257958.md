# Issue #257958 — Cursor's shown position and where it actually point are different in fullscreen

[https://github.com/microsoft/vscode/issues/257958](https://github.com/microsoft/vscode/issues/257958)

## Summary

This is almost certainly an upstream Electron/Chromium Wayland compositor issue: the OS-level hardware cursor is rendered at a different position/scale than what the application reports, which is why screenshots (which capture the application's internal state) show the correct position while the physical display does not. VS Code's code does not control system cursor rendering. Should be routed to the linux/wayland/upstream area for tracking.

## Change summary

Mouse cursor appears offset and enlarged in fullscreen mode on Linux Wayland with software GPU rendering; screenshots show the correct position.

## Best solution

Classify as upstream (Electron/Chromium on Wayland) and track against the relevant Electron or Chromium bug. The root cause is in how Chromium's Ozone/Wayland backend handles cursor compositing in fullscreen with software rendering — VS Code cannot fix this in its own code. A workaround the user could try is launching with `--ozone-platform=x11` or enabling hardware acceleration if their GPU supports it.
