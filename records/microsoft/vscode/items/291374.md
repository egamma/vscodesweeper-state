## Summary

The crash occurs entirely within the Copilot Chat extension's own bundle code (setItems, init in extension.js), not in VS Code core. The extension is responsible for migrating its own persisted session data and implementing the Iterable protocol correctly. This should be tracked against the github.copilot-chat extension, not VS Code.

## Change summary

VS Code core's ChatSessionItemCollection API is just the interface contract; the extension's setItems implementation is where the crash lives.
