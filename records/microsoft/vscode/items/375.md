## Summary

Still a valid unimplemented feature request for context-aware HTML completions. The HTML language service does not yet filter suggestions by parent tag or element context. Low upvote count (4) but parked in Backlog.

## Change summary

HTML intellisense should filter suggested tags and attributes based on the parent element context (e.g., only suggest <li> inside <ol>/<ul>).

## Best solution

Implement parent-tag-aware completion filtering in the upstream vscode-html-languageservice (which owns HTML completion logic), then consume it in the built-in HTML extension. This is a substantial enhancement to the data model and completion provider.
