# Issue #211577 — Add selection to next find match (CTRD + D) does not check current selection when changing focus back from editor to find widget

[https://github.com/microsoft/vscode/issues/211577](https://github.com/microsoft/vscode/issues/211577)

## Summary

Keep open: the current main code still has the reported path where the find widget owns the search text, but Add Selection to Next Find Match preserves the existing nonmatching editor selection. The issue is already typed and routed as `bug` / `editor-multicursor`.

## Change summary

Ctrl/Cmd+D from the find widget adds the next find match while leaving a nonmatching editor selection in the multicursor group.

## Best solution

Fix `src/vs/editor/contrib/multicursor/browser/multicursor.ts` so the find-widget-owned session replaces or drops the current nonmatching editor selection when adding the next match, and add a regression test in `src/vs/editor/contrib/multicursor/test/browser/multicursor.test.ts` for focus in the find widget with a stale editor selection.

- **selection preservation:** `src/vs/editor/contrib/multicursor/browser/multicursor.ts:343-349` gets the next find match and returns `allSelections.concat(nextMatch)`, so an existing editor selection that does not match the find criteria is retained.
