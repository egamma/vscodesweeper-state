# Issue #211577 — Add selection to next find match (CTRD + D) does not check current selection when changing focus back from editor to find widget

[https://github.com/microsoft/vscode/issues/211577](https://github.com/microsoft/vscode/issues/211577)

## Summary

Still reproduces on main: when the find widget owns the search (focus in find widget), `addSelectionToNextFindMatch` concatenates the next match onto all existing selections without filtering out selections that don't match the search criteria.

## Change summary

Ctrl+D with focus in the find widget preserves non-matching selections from the editor instead of dismissing them.

## Best solution

In `MultiCursorSession.addSelectionToNextFindMatch()` (or at session creation in the find-widget-owns-search path), filter existing selections to only those whose text matches `this.searchText` before concatenating the next match. The fix lives in `src/vs/editor/contrib/multicursor/browser/multicursor.ts`.
