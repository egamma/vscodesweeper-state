# Issue #128744 — DAP ContinuedEvent with allThreadsContinued=true causes selected debug session to change

[https://github.com/microsoft/vscode/issues/128744](https://github.com/microsoft/vscode/issues/128744)

## Summary

Still reproducible on main: the `passFocusScheduler` in `debugSession.ts` unconditionally passes focus away from the current session on a `ContinuedEvent` with `allThreadsContinued=true`, even during a restart where focus should stay. The issue is assigned to @roblourens and marked `under-discussion` — keeping open for a product decision on the intended behavior.

## Change summary

When a debug adapter sends a `ContinuedEvent` with `allThreadsContinued=true` during a restart, VS Code shifts focus to a different debug session instead of keeping it on the restarted one.

## Best solution

Guard the `passFocusScheduler` logic in `src/vs/workbench/contrib/debug/browser/debugSession.ts` (around line 164–180) so that when the `ContinuedEvent` originates from a restart flow, focus is not passed to another session. Alternatively, only pass focus when the user explicitly continued (not when the adapter sent a `ContinuedEvent` as a side effect of restart). This needs a debug-area owner decision on the right semantics.
