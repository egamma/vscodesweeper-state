# Issue #304693 — VS Code does not work properly with multiple monitors

[https://github.com/microsoft/vscode/issues/304693](https://github.com/microsoft/vscode/issues/304693)

## Summary

Team-authored issue (MEMBER) — hard keep-open. The rendering-stale behavior when a VS Code window spans two monitors on Windows is confirmed by multiple reporters and is very likely a Chromium/Electron GPU compositing issue. Issue #300404 describes the same symptom but also lacks triage; neither is clearly canonical yet.

## Change summary

When VS Code spans multiple monitors on Windows, content on the secondary monitor does not repaint (scroll, search dialog updates) until the window is moved or resized.

## Best solution

Investigate whether this is a known Chromium compositor bug with cross-monitor rendering (likely related to DirectComposition or GPU rasterization across display boundaries). If confirmed upstream, file or link the Chromium bug, label both this and https://github.com/microsoft/vscode/issues/300404 as `upstream`, and consider a workaround (e.g. forcing a
 
repaint on display-change events in `src/vs/platform/windows/electron-main/windowImpl.ts`). One of these two issues should become the canonical tracker and the other closed as duplicate.
