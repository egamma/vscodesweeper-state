# Issue #314898 — Diff editor loses word wrap after inline -> side-by-side transition

[https://github.com/microsoft/vscode/issues/314898](https://github.com/microsoft/vscode/issues/314898)

## Summary

Confirmed on current main: when the diff editor transitions back to side-by-side mode, wordWrapOverride2 is never cleared, so its stale 'off' overrides the restored wordWrapOverride1. An open fix PR exists but hasn't merged.

## Change summary

Diff editor loses word wrap when transitioning from inline to side-by-side mode because wordWrapOverride2 is not reset.

## Best solution

In `_adjustOptionsForLeftHandSide` (diffEditorEditors.ts), the `else` branch (side-by-side) should explicitly reset `wordWrapOverride2` to `'inherit'` (or remove the override) so it no longer suppresses `wordWrapOverride1`. PR https://github.com/microsoft/vscode/pull/314903 addresses this.
