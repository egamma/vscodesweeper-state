# Issue #295430 — Toggle Maximize Editor Group moves the window seperation in split view when in zen mode

[https://github.com/microsoft/vscode/issues/295430](https://github.com/microsoft/vscode/issues/295430)

## Summary

Legitimate layout bug: exiting maximized view in zen mode does not perfectly restore the original split proportions. The defect path is traceable in the gridview/splitview restore logic on current main.

## Change summary

Toggle Maximize Editor Group in zen mode with split view shifts the divider position when unmaximizing, instead of restoring the original proportions.

## Best solution

The fix likely belongs in `src/vs/base/browser/ui/grid/gridview.ts` — `exitMaximizedView` should save and restore exact split proportions rather than relying on cached visible sizes and `distributeEmptySpace`, which can drift when the container size differs (zen mode hides panels/sidebars, changing available space). A proportional save/restore approach would be more robust.
