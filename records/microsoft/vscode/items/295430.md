# Issue #295430 — Toggle Maximize Editor Group moves the window seperation in split view when in zen mode

[https://github.com/microsoft/vscode/issues/295430](https://github.com/microsoft/vscode/issues/295430)

## Summary

Legitimate bug: when Toggle Maximize Editor Group is used in zen mode, the editor container resizes (sidebars/panels hidden), and the split view's cached absolute pixel sizes for hidden groups become stale relative to the new container size, causing incorrect proportions on restore.

## Change summary

Toggle Maximize Editor Group in zen mode with split view does not restore the original split proportions when unmaximizing.

## Best solution

When exiting maximized view, the restore logic should account for container size changes that occurred while the view was hidden — either by caching proportions (not absolute sizes) or by scaling the cached size relative to the container size delta. The fix likely belongs in `src/vs/base/browser/ui/splitview/splitview.ts` (the `setVisible(true)` path at line 233–234) or in `src/vs/base/browser/ui/grid/gridview.ts` `exitMaximizedView()`.
