# Issue #237307 — When pasting multiline text into a comment, it should keep all lines commented out

[https://github.com/microsoft/vscode/issues/237307](https://github.com/microsoft/vscode/issues/237307)

## Summary

Valid feature request with 23 upvotes, promoted to Backlog. No implementation found on current main — the paste logic in the editor does not extend line comments to pasted lines.

## Change summary

When pasting multiline text into a line comment, the pasted lines should each receive the comment prefix so they remain commented out.

## Best solution

Extend the paste handler in the editor clipboard contribution (src/vs/editor/contrib/clipboard/browser/clipboard.ts) to detect when the cursor is inside a line comment and prepend the comment marker to each pasted line. The comment-detection logic can leverage the language configuration's comment tokens.
