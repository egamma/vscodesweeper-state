# Issue #311894 — Chat editors are not affected by `accessibility.dimUnfocused.enabled`

[https://github.com/microsoft/vscode/issues/311894](https://github.com/microsoft/vscode/issues/311894)

## Summary

Verified duplicate: the dim-unfocused feature uses an explicit CSS allowlist in `unfocusedViewDimmingContribution.ts` that does not include chat editor selectors. This is exactly the class of gap tracked by the canonical issue #236576, which was created to consolidate all missing-editor-type reports for this feature.

## Change summary

Chat editors opened in the editor area are not dimmed when unfocused, unlike text editors and terminals, when `accessibility.dimUnfocused.enabled` is on.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/236576, which tracks making the dim-unfocused feature comprehensive across all editor types. A fix would add a CSS rule like `.monaco-workbench .editor-instance:not(:focus-within) .interactive-session { filter: opacity(...) }` to `unfocusedViewDimmingContribution.ts`, but the canonical issue may pursue a more robust approach than the current per-editor-type allowlist.
