# Issue #178392 — Cannot move a directory to its parent with the same name

[https://github.com/microsoft/vscode/issues/178392](https://github.com/microsoft/vscode/issues/178392)

## Summary

Still reproduces on main: the safety check in fileService.ts at line 946–950 blocks moving a child directory to its parent when they share the same name, even though native file managers handle this correctly. No fix has been committed.

## Change summary

Moving a directory to its parent directory fails with an error when the child and parent share the same name (e.g., `folder/folder` → `folder`).

## Best solution

The check in `src/vs/platform/files/common/fileService.ts` (lines 946–950) correctly identifies that deleting the target would delete the source, but the operation could be supported by first moving the source to a temporary location, then replacing the target — similar to how native file managers handle this case. The fix belongs in the file service's move/copy validation logic.
