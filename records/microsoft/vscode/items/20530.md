# Issue #20530 — Search: target particular set of files

[https://github.com/microsoft/vscode/issues/20530](https://github.com/microsoft/vscode/issues/20530)

## Summary

Two of the major sub-requests (search in open editors, search in SCM changed files) have shipped, but the broader ask — user-defined filter presets, extensible @-syntax in the include/exclude fields, and other scoped file sets — remains unimplemented and is still a valid, high-demand feature request with 357 upvotes. Keeping open.

## Change summary

Allow searching in user-defined subsets of files (open editors, git-modified files, custom named filter sets) via the search include/exclude fields.

## Best solution

The remaining gap is the extensible/user-defined filter preset system (the `@open`, `@modified`, `@custom` syntax in include/exclude fields, plus user-configurable named glob sets). This is a design-heavy feature that needs UX decisions tracked in the closed investigation issue https://github.com/microsoft/vscode/issues/106790. The issue should stay open as the canonical tracker for the broader custom-filter-set feature, with partial progress noted.
