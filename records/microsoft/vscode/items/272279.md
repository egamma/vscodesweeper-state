# Issue #272279 — NES proposal on backspacing

[https://github.com/microsoft/vscode/issues/272279](https://github.com/microsoft/vscode/issues/272279)

## Summary

Team-authored issue (MEMBER) — must keep open. The bug is a real UX problem where NES ghost text proposes reverting the user's intentional backspace (restoring a typo the user is actively deleting), causing confusion. Team discussion indicates the behavior stems from how completions handle local cache hits within single-word edits, and alexdima notes it aligns with longstanding completion behavior — but the experience is still jarring.

## Change summary

NES ghost text proposes restoring a typo immediately after the user backspaces to fix it, making the text appear to not get shorter.

## Best solution

Tighten the local-cache-hit heuristic for NES so that when the user's edit is a backspace (deletion), the system does not propose ghost text that simply restores the deleted characters. chrmarti's comment suggests condition (2) — 'user's change is within a single word' — may be too lenient. The fix likely lives in the inline completions / NES suggestion filtering logic under `src/vs/editor/contrib/inlineCompletions/`.
