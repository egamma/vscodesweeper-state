# Issue #145202 — Timeline: support multi select commands

[https://github.com/microsoft/vscode/issues/145202](https://github.com/microsoft/vscode/issues/145202)

## Summary

Still a valid gap: the timeline tree's context menu actions only receive the focused item, not the full multi-selection. No implementation has landed on main. Keeping open in the Backlog for a maintainer to prioritize.

## Change summary

Timeline context menu commands should receive all multi-selected items, enabling bulk operations like deleting multiple local history entries.

## Best solution

Wire multi-selection through the timeline pane's context menu invocation in `src/vs/workbench/contrib/timeline/browser/timelinePane.ts` — the `getItemActions`/`getItemContextActions` methods currently resolve actions for a single element; they'd need to pass the full `tree.getSelection()` as command arguments, consistent with how other tree views (e.g. Explorer) handle multi-select commands.
