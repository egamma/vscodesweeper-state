# Issue #158254 — Allow not automatically saving files when running a test

[https://github.com/microsoft/vscode/issues/158254](https://github.com/microsoft/vscode/issues/158254)

## Summary

The request for a per-TestRunProfile API opt-out of auto-save remains unimplemented. A global user setting (`testing.saveBeforeTest`) exists but doesn't give extension authors programmatic control. An open WIP PR (#190427) references this but hasn't merged.

## Change summary

Allow test run profiles to declare they don't need files saved before running, via the TestRunProfile API.

## Best solution

Land the per-profile `saveBeforeTest` opt-out on the TestRunProfile API (the WIP PR #190427 is a starting point). This lets LSP-backed test runners skip the save without requiring users to change a global setting.
