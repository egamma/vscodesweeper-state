# Issue #287195 — Copilot Chat - all files are "sensitive", apparently

[https://github.com/microsoft/vscode/issues/287195](https://github.com/microsoft/vscode/issues/287195)

## Summary

Still reproducible on main: when a workspace lives inside a home-directory dotfile folder (e.g. `~/.devprojects/myapp`), every file matches `HOME_DOTFILE_PATTERNS` in `sessionPermissions.ts` and is flagged as platform-restricted, causing the agent to prompt for confirmation on every edit. Unlike `PLATFORM_RESTRICTED_DIRS`, the dotfile check has no working-directory carve-out.

## Change summary

When a workspace is nested inside a dot-prefixed folder under the user's home directory, the Copilot agent marks every file as 'sensitive' and requires manual confirmation for all edits.

## Best solution

Add a working-directory carve-out to the `HOME_DOTFILE_PATTERNS` check in `_isPlatformRestrictedPath` (analogous to the existing carve-out for `PLATFORM_RESTRICTED_DIRS`): when the working directory itself lives inside the dotfile directory, auto-approve edits within that tree. This preserves the security intent for ad-hoc edits to `~/.ssh` or `~/.aws` while allowing normal development in a workspace that happens to be under a dot-prefixed home subdirectory.
