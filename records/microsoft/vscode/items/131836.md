# Issue #131836 — Self-signed certificate error when installing Python support in WSL in spite of custom root certificate correctly installed in WSL

[https://github.com/microsoft/vscode/issues/131836](https://github.com/microsoft/vscode/issues/131836)

## Summary

Self-signed certificate errors in WSL were partially addressed in VS Code 1.85 (automatic remote OS certificate loading) and further by Node system certificate support, but 2024 comments still reported failures on 1.87.2. Significant certificate infrastructure work has landed since (systemCertificatesNode setting, loadSystemCertificates rework) — requesting confirmation on whether this still reproduces on the latest build before keeping it active.

## Change summary

VS Code's remote server in WSL does not pick up custom root CA certificates installed via update-ca-certificates, causing 'self signed certificate in certificate chain' errors when installing extensions.

## Best solution

If this still reproduces on current VS Code, the fix likely belongs in the `@vscode/proxy-agent` certificate loading path (src/vs/workbench/api/node/proxyResolver.ts, loadAdditionalCertificates) to ensure WSL system certificate stores are fully covered. If it no longer reproduces after the 1.85+ and Node system certificate changes, close as resolved.
