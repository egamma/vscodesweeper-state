# Issue #315866 — Cmake shows quick pick when you open any C++ project in the agents window

[https://github.com/microsoft/vscode/issues/315866](https://github.com/microsoft/vscode/issues/315866)

## Summary

Valid bug discovered during agents-window extension testing: CMake Tools shows an unwanted quick pick immediately on project open. The issue sits at the boundary between the agents window needing to suppress or gate extension prompts, and CMake Tools not detecting the agents-window context. Keeping open for maintainer routing.

## Change summary

CMake Tools extension shows an unwanted quick pick prompt immediately when opening a C++ project with a CMakeLists.txt in the agents window.

## Best solution

Either the agents window should suppress extension-contributed quick picks that fire automatically on workspace open, or CMake Tools should check whether it's running in the agents window context and skip auto-prompts. The agents-window team should decide which side owns this — likely a combination of both (agents window providing an API/signal, and extensions respecting it).
