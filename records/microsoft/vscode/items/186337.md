# Issue #186337 — Diff Editor - code move is not shown

[https://github.com/microsoft/vscode/issues/186337](https://github.com/microsoft/vscode/issues/186337)

## Summary

Team-authored issue; the move-detection threshold is still character/line-count based (≥15 chars and ≥2 lines with ≥2 chars each) which can miss short moves. The underlying behavior persists on main. Keeping open per team-authored policy.

## Change summary

Diff editor fails to detect code moves when the moved block is short (e.g. 2 lines with few characters).

## Best solution

Adjust the move-detection heuristic in `src/vs/editor/common/diff/defaultLinesDiffComputer/computeMovedLines.ts` (the filter around line 39) to lower or rethink the threshold — possibly counting meaningful characters rather than raw line count, as hediet suggested.
