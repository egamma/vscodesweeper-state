# Issue #311046 — Add more AI CLIs as shell type

[https://github.com/microsoft/vscode/issues/311046](https://github.com/microsoft/vscode/issues/311046)

## Summary

Valid feature request from a team member to expand the recognized AI CLI shell types. The current `GeneralShellType` enum in `src/vs/platform/terminal/common/terminal.ts` lists only Claude, Codex, Copilot, and Gemini — the requested CLIs (pi, cursor, droid, amp) are not yet present. Keeping open per the hard keep-open rule for team-authored issues.

## Change summary

Add pi (pi-mono), cursor, droid (Factory.ai), and amp as recognized AI CLI shell types in VS Code's terminal so they benefit from `allowAgentCliTitle` and proper shell type detection.

## Best solution

Add new entries to `GeneralShellType` in `src/vs/platform/terminal/common/terminal.ts`, add detection patterns in `windowsShellHelper.ts`, add OSC title patterns in `terminalInstance.ts` (`agentCliTitlePatterns`), and register them in `agentCliShellTypes`. Each new CLI needs its executable-name mapping for Windows and its title regex for cross-platform detection.
