# Issue #226412 — Watch expressions should allow LSP or debugger to remove inner whitespaces

[https://github.com/microsoft/vscode/issues/226412](https://github.com/microsoft/vscode/issues/226412)

## Summary

Legitimate feature request with constructive team engagement (roblourens acknowledged the problem and suggested trimming/escaping newlines). No implementation has landed on main — the watch expression is stored as-is with no whitespace normalization. Keeping open for further design work.

## Change summary

When a multi-line code selection is added to the debug watch list, inner whitespace and newlines are preserved verbatim, making the expression hard to read.

## Best solution

At minimum, apply `.trim()` and collapse/escape internal newlines when adding watch expressions (as roblourens suggested). A richer solution could involve a DAP or LSP hook for expression normalization (stripping comments, trailing semicolons), but the simple trim in `debugModel.ts:addWatchExpression` and the watch expression renderer would already improve the experience significantly.
