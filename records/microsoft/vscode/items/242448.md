# Issue #242448 — cursor moves to the wrong spot when `editor.quickSuggestions.other` is set to `"inline"` and completion includes a text edit that adds an import

[https://github.com/microsoft/vscode/issues/242448](https://github.com/microsoft/vscode/issues/242448)

## Summary

Fixed on main by PR #249554 (commit d5d3945bf19) which corrects cursor positioning after inline completion acceptance — the cursor now moves to the end of the actual edit, not the end of the range provided by the extension. Shipped in 1.101.0.

## Change summary

When `editor.quickSuggestions.other` is set to `"inline"` and an inline completion includes additional text edits (e.g. adding an import), the cursor jumps to the wrong location after accepting.

## Best solution

Keep the shipped fix (commit d5d3945bf19, VS Code 1.101.0). The inline completions model now strips common suffix/prefix from ghost-text edits before computing cursor position, so the cursor lands at the end of the meaningful insertion. No further change needed.
