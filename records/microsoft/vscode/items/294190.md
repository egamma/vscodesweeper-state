# Issue #294190 — Reverse lines should preserve single-line selection

[https://github.com/microsoft/vscode/issues/294190](https://github.com/microsoft/vscode/issues/294190)

## Summary

Still reproduces on main: when a single-line selection triggers whole-document reversal, the resulting selection is incorrectly transformed rather than preserved. PR #284988 addresses this but remains open/unmerged.

## Change summary

Running 'Reverse Lines' with a single-line selection expands the selection to the entire document instead of preserving the original selection.

## Best solution

Merge or rework PR https://github.com/microsoft/vscode/pull/284988 — the fix is to preserve `originalSelections[0]` as-is (adjusting only the line number via `updateLineNumber`) when the single-line expansion path is taken, so the user's cursor/selection stays in place.
