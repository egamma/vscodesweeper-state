# Issue #317483 — vscode.FileDecorationProvider in git extension is too greedy in applying file decorations to URIs

[https://github.com/microsoft/vscode/issues/317483](https://github.com/microsoft/vscode/issues/317483)

## Summary

Valid bug: the git extension's GitIgnoreDecorationProvider applies decorations to URIs with non-file schemes because model.getRepository uses uri.fsPath without checking the scheme. Still present on main.

## Change summary

The built-in git extension's file decoration provider incorrectly decorates URIs with custom (non-file) schemes when their fsPath matches a path inside a git repository.

## Best solution

Add a scheme guard in GitIgnoreDecorationProvider.provideFileDecoration (and potentially in Model.getOpenRepository) to bail out early for non-file/non-git URIs, preventing false-positive git decorations on custom-scheme tree items.
