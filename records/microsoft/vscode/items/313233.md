# Issue #313233 — ## Bug Report: Command Prefix `/explain` Not Recognized

[https://github.com/microsoft/vscode/issues/313233](https://github.com/microsoft/vscode/issues/313233)

## Summary

The report describes the Copilot extension's `/explain` slash command in agent mode executing a code change instead of providing an explanation. The intent routing code in `extensions/copilot/src/extension/common/constants.ts` correctly maps `/explain` to `Intent.Explain` for agent mode, so the routing itself looks correct. The issue may be a model-level prompt problem (Claude Sonnet 4.5 interpreting the imperative linting text as an instruction despite the explain intent). The report is from VS Code 1.117.0 (April 2026) and may not reproduce on current builds — requesting confirmation on latest Insiders.

## Change summary

Using the `/explain` quick action on a linting diagnostic in agent mode causes the Copilot agent to execute a code change instead of providing a read-only explanation.

## Best solution

The Copilot team should verify whether the explain intent's prompt scaffolding in agent mode (via the tool-calling loop) sufficiently constrains the model to read-only responses. If the model is receiving the `/explain` intent but the agent-mode prompt doesn't convey the read-only constraint, the fix would be in the explain intent's prompt construction in `extensions/copilot/src/extension/intents/node/explainIntent.ts` or the agent-mode routing layer.
