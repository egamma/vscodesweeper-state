# Issue #290063 — MCP: structuredContent in tool result overrides content[].text sent to model

[https://github.com/microsoft/vscode/issues/290063](https://github.com/microsoft/vscode/issues/290063)

## Summary

Team-authored issue; the described behavior (structuredContent suppressing content[].text) is still present on current main. A maintainer comment notes structuredContent should not go to the model at all per spec discussion, but the code hasn't been updated to reflect either interpretation correctly.

## Change summary

When an MCP tool returns both content[].text and structuredContent, VS Code drops content and only sends stringified structuredContent to the model.

## Best solution

Align with the MCP spec clarification from https://github.com/modelcontextprotocol/modelcontextprotocol/issues/1624 — if structuredContent is not meant for the model, stop sending it to the model and restore sending content[].text. The fix is localized to src/vs/workbench/contrib/mcp/common/mcpLanguageModelToolContribution.ts around lines 333 and 389-391.
