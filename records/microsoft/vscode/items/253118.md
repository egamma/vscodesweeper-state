# Issue #253118 — Undo should delete generated files

[https://github.com/microsoft/vscode/issues/253118](https://github.com/microsoft/vscode/issues/253118)

## Summary

Team-authored issue (MEMBER) — hard keep-open. The request is valid and confirmed by multiple community members: undoing agent-generated file creations does not delete those files from disk. No implementation found on current main.

## Change summary

When a user undoes agent-mode edits that created new files, those files remain on disk instead of being deleted.

## Best solution

Extend the chat/agent undo infrastructure to track file-creation edits and delete the created files (and directories) from disk when the user triggers undo. This likely involves the chat editing session or bulk edit service that manages agent-mode changes, ensuring file-creation operations are reversible. The thread history sent to the model should also be updated to exclude the undone creations.
