# Issue #127521 — [linked editing] does not work right after enter

[https://github.com/microsoft/vscode/issues/127521](https://github.com/microsoft/vscode/issues/127521)

## Summary

Still appears unfixed on main. The linked editing contribution clears ranges and re-fetches via a debounced async call after Enter changes the model; during that window the feature appears broken. Confirmed by a team member and in the Backlog — should stay open.

## Change summary

Linked editing (synced tag renaming) fails to activate immediately after pressing Enter, requiring a cursor move or extra keystroke to re-engage.

## Best solution

In `src/vs/editor/contrib/linkedEditing/browser/linkedEditing.ts`, handle the content-change-then-cursor-move sequence after Enter more gracefully — either reduce/eliminate the debounce for cases where the cursor lands inside a known linked range, or re-query the provider immediately when the model version changes but the cursor remains in a tag position. A test asserting linked editing activates after an Enter inside an HTML tag would validate the fix.
