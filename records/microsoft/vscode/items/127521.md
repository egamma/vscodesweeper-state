# Issue #127521 — [linked editing] does not work right after enter

[https://github.com/microsoft/vscode/issues/127521](https://github.com/microsoft/vscode/issues/127521)

## Summary

Still reproducible on main: pressing Enter inside a tag breaks the linked editing reference range, triggering a debounced recompute during which linked editing is inactive. No fix has addressed this specific timing gap.

## Change summary

Linked editing for HTML tags does not work immediately after pressing Enter because the ranges are cleared and recomputed with a debounce delay.

## Best solution

In `src/vs/editor/contrib/linkedEditing/browser/linkedEditing.ts`, when Enter is pressed and the tag structure is still intact (closing tag on a new line), the recompute should fire immediately (zero debounce) or the existing ranges should be preserved/updated in place rather than cleared and recomputed asynchronously.
