# Issue #205737 — Adopt Editor Hover for "Revert Change" action

[https://github.com/microsoft/vscode/issues/205737](https://github.com/microsoft/vscode/issues/205737)

## Summary

Team-authored issue requesting the dirty-diff 'Revert Change' action adopt the editor hover widget instead of a standalone hover. Not yet implemented on main — the quick-diff decorator has no glyphMarginHoverMessage or editor-hover integration. Keeping open per hard keep-open rule (MEMBER author).

## Change summary

Replace the standalone hover for the 'Revert Change' gutter action with the editor's glyph-margin hover widget to avoid double-hover conflicts with breakpoints.

## Best solution

Refactor the dirty-diff revert action in `src/vs/workbench/contrib/scm/browser/quickDiffDecorator.ts` (or `quickDiffWidget.ts`) to contribute its hover content via `glyphMarginHoverMessage` on the model decoration, following the pattern in `breakpointEditorContribution.ts`. This ensures a single editor hover instance handles both breakpoints and revert actions.
