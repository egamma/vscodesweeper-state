# Issue #205737 — Adopt Editor Hover for "Revert Change" action

[https://github.com/microsoft/vscode/issues/205737](https://github.com/microsoft/vscode/issues/205737)

## Summary

Team-authored feature request to integrate the dirty-diff 'Revert Change' action into the editor hover system instead of showing a separate hover. Not yet implemented on main — the quick diff widget still uses PeekViewWidget, not the glyph margin hover infrastructure.

## Change summary

Hovering over the dirty-diff gutter glyph shows both a breakpoint hover and a revert-change hover simultaneously; the revert action should be adopted into the editor's margin hover to avoid double-hover.

## Best solution

Refactor the dirty-diff gutter decoration in `src/vs/workbench/contrib/scm/browser/quickDiffWidget.ts` (and possibly `quickDiffDecorator.ts`) to use `glyphMarginHoverMessage` on the model decoration, integrating the revert action into the editor's built-in margin hover system as suggested in the comments.
