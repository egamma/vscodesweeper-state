# Issue #248124 — Cannot drag a file back into VS Code after deleting it

[https://github.com/microsoft/vscode/issues/248124](https://github.com/microsoft/vscode/issues/248124)

## Summary

This is likely an Electron/Chromium limitation: the Windows Recycle Bin is a shell virtual namespace, and dragging items from it does not provide standard file paths via the DataTransfer API that Electron exposes through `webUtils.getPathForFile()`. VS Code relies on this Electron API for native file drag-and-drop, so it cannot receive usable paths from virtual shell folders.

## Change summary

Dragging a deleted file from the Windows Recycle Bin back into VS Code's explorer does nothing — the file is not restored or imported.

## Best solution

This requires Electron/Chromium to support drag-and-drop from Windows shell virtual namespaces (like the Recycle Bin) by translating virtual item paths into real file system paths or supporting the shell's 'move' semantics. VS Code's DnD pipeline in `src/vs/platform/dnd/browser/dnd.ts` relies on `Electron.webUtils.getPathForFile()` (line 517), which likely returns no usable path for Recycle Bin items. Filing an upstream Electron issue would be the appropriate next step. Alternatively, if Electron does return a valid internal `$Recycle.Bin` path, VS Code could potentially detect and handle it, but that would be fragile and platform-specific.
