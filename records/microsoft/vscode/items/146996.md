# Issue #146996 — Reuse tokens when a new line is inserted

[https://github.com/microsoft/vscode/issues/146996](https://github.com/microsoft/vscode/issues/146996)

## Summary

Team-authored enhancement request still valid on current main — the token-reuse optimization for newline insertion has not been implemented. Keeping open per hard keep-open rule (MEMBER author).

## Change summary

When pressing Enter to insert a new line, reuse/split existing tokens for the new line instead of discarding them, avoiding a brief flash of unstyled text.

## Best solution

In `src/vs/editor/common/tokens/contiguousTokensStore.ts`, the `acceptEdit` method (around line 206) should split the deleted-ending tokens from the original line and assign them to the newly inserted line instead of inserting `null`. This eliminates the flash between the edit and the next tokenization pass.
