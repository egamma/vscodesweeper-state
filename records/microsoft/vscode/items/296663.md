# Issue #296663 — `ChatTitleProvider#provideChatTitle` which passes in `context: vscode.ChatContext,` does not include the `chatSessionItem`

[https://github.com/microsoft/vscode/issues/296663](https://github.com/microsoft/vscode/issues/296663)

## Summary

Team-authored issue (MEMBER); still reproduces on main. The `$provideChatTitle` method omits `chatSessionContext` from the `ChatContext` it passes to the title provider, making it impossible for the extension to know the session identity for persisting the title.

## Change summary

The `ChatTitleProvider#provideChatTitle` receives a `ChatContext` that lacks `chatSessionContext` (and its `chatSessionItem`), preventing extensions from identifying which session the title belongs to.

## Best solution

In `src/vs/workbench/api/common/extHostChatAgents2.ts`, the `$provideChatTitle` method (and likely `$provideChatSummary`) should resolve and include `chatSessionContext` in the `ChatContext` it constructs, mirroring how the main request handler at line 1017-1038 does it.
