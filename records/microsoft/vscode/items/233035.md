# Issue #233035 — Validate tool input in vscode

[https://github.com/microsoft/vscode/issues/233035](https://github.com/microsoft/vscode/issues/233035)

## Summary

Team-authored feature request, partially addressed but not fully adopted. The `json.validate` command (PR #244784) landed and is used to validate `updatedInput` from preToolUse hooks, but the broader goal — validating LLM-generated tool call parameters before invocation and implementing a retry loop for invalid calls — remains unimplemented. Keeping open per hard keep-open rule (MEMBER author) and because the remaining work is real.

## Change summary

Validate LLM-generated tool call parameters against the tool's declared JSON schema before invocation, with a retry loop for invalid inputs.

## Best solution

Extend `invokeTool` in `src/vs/workbench/contrib/chat/browser/tools/languageModelToolsService.ts` to validate `dto.parameters` against the tool's `inputSchema` using the existing `json.validate` command before dispatching to the tool implementation, and implement a retry loop that feeds validation errors back to the model for correction.
