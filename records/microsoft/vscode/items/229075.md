# Issue #229075 — Settings : `editor.cursorSurroundingLines` misleading default

[https://github.com/microsoft/vscode/issues/229075](https://github.com/microsoft/vscode/issues/229075)

## Summary

The behavior is by design — sticky scroll forces a minimum surrounding-lines value equal to its maxLineCount (default 5) so the cursor never hides behind sticky headers. But the setting description doesn't explain this interaction, which is legitimately confusing. Valid polish request to improve the description.

## Change summary

The `editor.cursorSurroundingLines` setting description doesn't mention that sticky scroll overrides its effective value, making values 0–4 appear ignored.

## Best solution

Update the `cursorSurroundingLines` setting description in `src/vs/editor/common/config/editorOptions.ts` to mention that when sticky scroll is enabled, the effective minimum is `editor.stickyScroll.maxLineCount`. This makes the interaction transparent without changing behavior.
