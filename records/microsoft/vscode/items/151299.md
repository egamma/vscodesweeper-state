# Issue #151299 — Change the "Run and Debug" View's "Play" icon to a "Connect" icon when an Attach Mode Configuration is Selected

[https://github.com/microsoft/vscode/issues/151299](https://github.com/microsoft/vscode/issues/151299)

## Summary

Valid, well-argued feature request with 35 upvotes (above the 20 threshold, already in Backlog). Not implemented on current main — the start button in debugActionViewItems.ts always renders `debugStart` regardless of attach/launch mode. Team discussion shows no consensus but genuine interest; keeping open for continued consideration.

## Change summary

Show a 'connect' icon instead of the 'play' icon on the debug start button when an attach-mode configuration is selected.

## Best solution

In `src/vs/workbench/contrib/debug/browser/debugActionViewItems.ts`, make the start button icon dynamic: observe `CONTEXT_FOCUSED_SESSION_IS_ATTACH` (or the selected configuration's request type) and swap the icon from `debugStart` to a connect-style codicon. A prerequisite is adding a suitable 'connect' codicon to vscode-codicons (tracked in https://github.com/microsoft/vscode-codicons/issues/112). The debug toolbar already swaps stop/disconnect icons based on attach state, so the pattern exists.
