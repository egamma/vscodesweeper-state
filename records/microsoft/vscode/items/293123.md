# Issue #293123 — VS Code switches to wrong instance to commit code in multi-root workspace

[https://github.com/microsoft/vscode/issues/293123](https://github.com/microsoft/vscode/issues/293123)

## Summary

Still reproduces on main: `findWindowOnFile` in `windowsFinder.ts` returns the first multi-root workspace window whose folders contain the file path, with no preference for a more specific single-root workspace or the window where the terminal command originated. The bug is confirmed by a second commenter who reports the same behavior with git worktrees.

## Change summary

Running `git commit` in a single-root VS Code instance opens the commit message editor in a different multi-root instance that also contains the same folder.

## Best solution

Fix `findWindowOnFile` in `src/vs/platform/windows/electron-main/windowsFinder.ts` to prefer the most specific window match (single-root over multi-root when both contain the file's folder), or prefer the window that originated the CLI invocation. Related issue https://github.com/microsoft/vscode/issues/299540 reports the same root cause for git worktrees — both should be addressed together.
