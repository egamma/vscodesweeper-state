# Issue #301517 — Allow opening `file://` URI externaly on Ctrl+Click in the terminal

[https://github.com/microsoft/vscode/issues/301517](https://github.com/microsoft/vscode/issues/301517)

## Summary

Valid feature request: terminal file:// URI links currently always open in the editor; the user wants an option (or heuristic) to open them externally in the browser instead, especially for HTML report files.

## Change summary

Request an option to open file:// URIs externally (in the browser) when Ctrl+clicking them in the integrated terminal, rather than always opening them in the editor.

## Best solution

Add a setting (e.g. `terminal.integrated.links.fileUriAction`) that controls whether file:// URIs open in the editor or externally, or use a file-extension heuristic (e.g. .html → external). The fix would live in `src/vs/workbench/contrib/terminalContrib/links/browser/terminalLinkOpeners.ts` in `_openFileSchemeLink`.
