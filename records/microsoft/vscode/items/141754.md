# Issue #141754 — Clone from Github doesn't rank org repos highly

[https://github.com/microsoft/vscode/issues/141754](https://github.com/microsoft/vscode/issues/141754)

## Summary

Still reproduces on main: in `getRemoteSources`, the Map is populated with query (public search) results before user/org repos, so public repos appear first in the picker. Three open PRs are attempting to fix this — keep open for the maintainer to land one.

## Change summary

When cloning from GitHub, public search results are ranked above the user's own and org repos in the picker, making personal/org repos hard to find.

## Best solution

Land one of the open fix PRs (e.g. https://github.com/microsoft/vscode/pull/310787). The fix is to iterate user/org repos into the Map first (or reverse the `all` array iteration) so they appear at the top of the picker. The code change is small and localized to `extensions/github/src/remoteSourceProvider.ts`.
