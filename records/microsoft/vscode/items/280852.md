# Issue #280852 — `ctrl` activates hover on macOS when hover is set to `onKeyboardModifier`

[https://github.com/microsoft/vscode/issues/280852](https://github.com/microsoft/vscode/issues/280852)

## Summary

Team-authored issue with a confirmed source-level bug and an open community PR — keeping open for maintainer judgment on the PR.

## Change summary

On macOS with hover set to `onKeyboardModifier`, pressing Ctrl triggers the hover when only Cmd should (to match `multiCursorModifier` behavior).

## Best solution

Review and land (or adapt) the open PR https://github.com/microsoft/vscode/pull/309138 which fixes `isTriggerModifierPressed` in `src/vs/editor/contrib/hover/browser/hoverUtils.ts` to be platform-aware, using only `metaKey` on macOS instead of `ctrlKey || metaKey`.
