# Issue #236016 — SCM - The version-control comment is not DevContainer-specific

[https://github.com/microsoft/vscode/issues/236016](https://github.com/microsoft/vscode/issues/236016)

## Summary

Still a valid open bug. The team identified the root cause in December 2024 — the SCM input history migration code assumes `file:` URIs via `URI.file(rootPath)`, dropping the authority that distinguishes dev containers — but no fix has landed on main. The main storage path now uses `StorageScope.WORKSPACE` (since commit f7c6e1a7a13, Sept 2023), which should scope new entries per workspace, but the migration from old APPLICATION-scope keys with `URI.file()` can re-introduce cross-container leakage.

## Change summary

SCM commit message input text is shared across all dev containers instead of being scoped per container.

## Best solution

Fix the migration code in `src/vs/workbench/contrib/scm/common/scmService.ts` around line 332 to preserve the full URI (including authority) instead of using `URI.file(rootPath)`. The TODO at line 316 also suggests the migration code should eventually be simplified to just remove old `scm/input:` keys. @lszomoru was asked to take a stab at this in comment 11.
