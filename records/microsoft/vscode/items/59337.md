# Issue #59337 — Extend tasks API to allow programmatic problem matching

[https://github.com/microsoft/vscode/issues/59337](https://github.com/microsoft/vscode/issues/59337)

## Summary

Hard keep-open: milestone is 'On Deck' (a scheduling decision). The feature — accepting programmatic problem matcher definitions (not just registered names) in the Task API — remains unimplemented on current main; `Task.problemMatchers` is still `string[]`.

## Change summary

Allow extensions to pass problem matcher definitions programmatically to the Task constructor/property, rather than only referencing pre-registered matcher names by string.

## Best solution

Extend the `Task` API so that `problemMatchers` accepts inline matcher definitions (objects) in addition to registered names. This likely touches `src/vscode-dts/vscode.d.ts` (the Task class), the task execution pipeline in `src/vs/workbench/contrib/tasks/`, and the extension host task handling. The 'On Deck' milestone signals team intent to address this.
