# Issue #59337 — Extend tasks API to allow programmatic problem matching

[https://github.com/microsoft/vscode/issues/59337](https://github.com/microsoft/vscode/issues/59337)

## Summary

The core request — accepting programmatic problem matcher definitions (not just names) in the Task API — remains unimplemented on current main. The `problemMatchers` property is still `string[]`. With 25 upvotes and two duplicates closed against it (#51957, #137221), this is the canonical issue and should stay open.

## Change summary

Allow extensions to pass inline problem matcher definitions to `vscode.Task` instead of only referencing pre-registered matcher names by string.

## Best solution

Extend the `problemMatchers` type on `vscode.Task` (and constructor parameter) to accept `ProblemMatcher` definition objects alongside the existing string names. This likely touches `src/vscode-dts/vscode.d.ts` (the stable API surface), `src/vs/workbench/api/common/extHostTypes.ts`, and the task resolution pipeline in `src/vs/workbench/api/common/extHostTask.ts` / `src/vs/workbench/contrib/tasks/`. Consider proposing the API shape first via `vscode.proposed.*.d.ts`.
