# Issue #280025 — Task terminal sticky scroll bug after being cleared

[https://github.com/microsoft/vscode/issues/280025](https://github.com/microsoft/vscode/issues/280025)

## Summary

Confirmed bug still present on main — getCommandForLine returns disposed commands after clearBuffer because it doesn't check for disposed markers (line === -1). An open PR (#282888) exists targeting this fix, so the issue should stay open until that PR lands.

## Change summary

Terminal sticky scroll shows stale/disposed command after the terminal is cleared (e.g. re-running a task with presentation.clear), because getCommandForLine returns commands whose markers have been disposed.

## Best solution

Land or rework the fix in PR https://github.com/microsoft/vscode/pull/282888 — add a disposed-marker guard (marker.line === -1) in `getCommandForLine()` in `src/vs/platform/terminal/common/capabilities/commandDetectionCapability.ts` so disposed commands are skipped after clearBuffer.
