# Issue #239718 — Git - The .gitignore file does not respect exclusions when opening a remote repository

[https://github.com/microsoft/vscode/issues/239718](https://github.com/microsoft/vscode/issues/239718)

## Summary

The 'Open Remote Repository' command is provided by the GitHub Repositories extension (ms-vscode.remote-repositories), which is not part of VS Code core. Any .gitignore handling in that virtual file system context is the extension's responsibility, so this issue belongs on that extension's repo.

## Change summary

When using 'Open Remote Repository', .gitignore negation patterns (e.g. `!/path/to/file`) are not respected, causing excluded files to not appear as tracked.

## Best solution

Close as caused-by-extension and direct the reporter to file against the GitHub Repositories extension (ms-vscode.remote-repositories). The 'Open Remote Repository' feature, its virtual file system, and any .gitignore interpretation in that context are entirely owned by that extension — VS Code core and the built-in git extension have no remote-repository code paths.
