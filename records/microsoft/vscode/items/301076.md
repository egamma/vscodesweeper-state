# Issue #301076 — Auto-run project initialisation on first session start instead of requiring /init manually

[https://github.com/microsoft/vscode/issues/301076](https://github.com/microsoft/vscode/issues/301076)

## Summary

Valid feature request to auto-run project initialization for AI agents on first session start. VS Code already surfaces `/init` via a tip system (`tip.init` in chatTipCatalog.ts) but does not auto-execute it. The gap is real — discoverability of `/init` depends on tips the user may not notice. Keeping open for product decision on the right UX.

## Change summary

Automatically scan and offer to save project configuration for AI agents on first session start, instead of requiring the user to manually run `/init`.

## Best solution

Consider auto-detecting when no workspace instructions exist on the first agent session and prompting the user (not auto-running) to generate them. The existing `tip.init` in `src/vs/workbench/contrib/chat/browser/chatTipCatalog.ts` already nudges users but is passive; a more active prompt (e.g. a notification or inline suggestion on first agent message) would bridge the discoverability gap without surprising users. This belongs in the chat/AI customization area.
