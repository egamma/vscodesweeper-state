# Issue #96578 — QuickPick in web with VoiceOver does not read anything

[https://github.com/microsoft/vscode/issues/96578](https://github.com/microsoft/vscode/issues/96578)

## Summary

The original dual aria-activedescendant conflict has been substantially addressed by multiple commits since 2021 (guarding activedescendant to the focused element, removing it on keystrokes, etc.), but no commit directly references this issue, and the last comment narrowed the problem to Safari/VoiceOver only. Requesting confirmation on the latest build before closing.

## Change summary

QuickPick in VS Code for the Web does not announce items via VoiceOver when navigating with the keyboard.

## Best solution

The aria-activedescendant handling has been heavily reworked since 2021 (e.g. commit 092bbc8d6fa guards it to only fire when the input box has focus). If the reporter or a community member can confirm VoiceOver now reads QuickPick items in Safari on vscode.dev, this can be closed as fixed. If the Safari-specific regression persists, a targeted fix in the quickInput controller or list widget is needed.
