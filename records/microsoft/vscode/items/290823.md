## Summary

Team-authored polish issue about the Agent Sessions Welcome Page skeleton UI showing 3 rows that then collapse to 1, causing visual jumpiness. Must stay open per team-authored rule. The skeleton CSS classes exist in the stylesheet but appear unused in current TypeScript code, suggesting the loading skeleton implementation may have changed since filing — worth verifying whether the jumpiness still reproduces.

## Change summary

The welcome page loading skeleton initially renders 3 rows before collapsing to 1, causing a visible UI jump.

## Best solution

Reduce the default skeleton row count to 1 (or match it to the expected final count) in the welcome page's loading state so the layout doesn't jump when real sessions load. The relevant CSS is in `src/vs/workbench/contrib/welcomeAgentSessions/browser/media/agentSessionsWelcome.css` (the `.agentSessionsWelcome-sessionsLoading` grid), and the layout logic is in `agentSessionsWelcome.ts` (`layoutSessionsControl`). The skeleton CSS classes appear orphaned from the TypeScript — the skeleton rendering path may need to be re-examined or cleaned up.
