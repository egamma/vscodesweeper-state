# Issue #324093 — Add a confirmation dialog with option to revert instead to "Git: Undo Last Commit" when last commit has been pushed

[https://github.com/microsoft/vscode/issues/324093](https://github.com/microsoft/vscode/issues/324093)

## Summary

Reasonable feature request to improve the 'Undo Last Commit' command by detecting whether the commit has been pushed and offering a revert instead, preventing users from accidentally needing a force-push. The prior issue #76107 asked only for a generic confirmation and was closed as-designed; this request is more specific and actionable — detect pushed state and suggest the appropriate undo strategy.

## Change summary

The 'Git: Undo Last Commit' command should detect if the last commit has been pushed and offer a revert instead of a soft reset to avoid requiring a force-push.

## Best solution

In `extensions/git/src/commands.ts` (`undoCommit` method around line 2779), check whether HEAD has been pushed to the tracking branch before performing the reset. If pushed, show a dialog offering to revert the commit instead of resetting, or at minimum warn the user that a force-push will be required. The git extension already has repository/tracking info available to make this determination.
