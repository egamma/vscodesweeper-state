# Issue #23633 — showInputBox should have the ability to have an array of quickpick options as well

[https://github.com/microsoft/vscode/issues/23633](https://github.com/microsoft/vscode/issues/23633)

## Summary

The request for a combined input-box + quick-pick remains valid. While `createQuickPick` enables workarounds via `value` and `onDidAccept`, no first-class ergonomic API (e.g. an `allowCustomValue` flag on `showQuickPick`) has been added. With 46 upvotes, this exceeds the backlog threshold and should stay open.

## Change summary

Allow `showQuickPick` (or a new convenience overload) to accept free-form user input in addition to pre-defined items, instead of returning `undefined` for non-matching input.

## Best solution

Add an `allowCustomValue` option to `showQuickPick` (or a dedicated option on `QuickPickOptions`) so extensions can accept typed text that doesn't match any item without hand-rolling the `createQuickPick` event-wiring pattern. The work belongs in `src/vscode-dts/vscode.d.ts` (API surface) and `src/vs/workbench/api/common/extHostQuickOpen.ts` (implementation).
