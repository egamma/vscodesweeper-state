## Change summary

Opening a subfolder of a large git repository causes 100% CPU on multiple cores for minutes as VS Code's git integration enumerates parent repository changes.

## Best solution

The `info-needed` label should be removed since the samply profile was provided. The team member who requested the profile (deepak1556) should analyze it to determine where the bottleneck is — likely in the git status/diff enumeration path in `extensions/git/src/model.ts` or in the file watcher setup. A possible mitigation is to cap or defer parent repository scanning when the repo is very large, or to make the existing `git.openRepositoryInParentFolders` prompt happen before the expensive enumeration rather than during or after it.
