# Issue #319673 — [Regression in 1.121.0] MCP auth broken: microsoft-authentication extension (MSAL+WAM broker) ships device-bound (cnf) tokens as plain Bearer -> 401

[https://github.com/microsoft/vscode/issues/319673](https://github.com/microsoft/vscode/issues/319673)

## Summary

Team-authored regression report with extensive telemetry evidence showing MCP auth broken since 1.121.0 due to WAM broker emitting device-bound (cnf) tokens sent as plain Bearer. Hard keep-open: author is MEMBER.

## Change summary

Since VS Code 1.121.0, the microsoft-authentication extension's WAM broker path emits sender-constrained (cnf) access tokens for MCP servers, but sends them as plain Bearer without proof-of-possession, causing 401s or AADSTS530084 rejections on Entra-joined Windows devices.

## Best solution

The fix likely lives in `extensions/microsoft-authentication/` — either request unbound tokens for MCP OAuth flows (skip the broker's PRT-binding path), or implement PoP/SHR token presentation when the broker returns a cnf-stamped AT. Bisect the MSAL Node version bump between 1.120.0 and 1.121.0 to identify the behavioral change.
