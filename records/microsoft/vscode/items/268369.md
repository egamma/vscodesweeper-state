## Summary

Valid feature request for a variable that dynamically resolves to the workspace root of the active file in multi-root workspaces. No existing variable provides this, and no duplicate tracks it. Low community signal so far (0 upvotes), but it's already triaged to Backlog with the debug label.

## Change summary

Request a new variable (e.g. ${workspaceFolder:$current}) that resolves to the workspace root containing the currently active file, avoiding per-root duplication of debug configs in .code-workspace files.

## Best solution

If pursued, add a new variable (or special token like `${workspaceFolder:$current}`) in `src/vs/workbench/services/configurationResolver/common/variableResolver.ts` that resolves to the workspace folder containing the active editor's file. This is a niche multi-root debug scenario; could also be addressed by an extension contributing a variable.
