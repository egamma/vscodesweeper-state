# Issue #323765 — "Try Again" in Chat does not allow selection of new model

[https://github.com/microsoft/vscode/issues/323765](https://github.com/microsoft/vscode/issues/323765)

## Summary

Confirmed from source: several resend/retry/confirmation paths pass `userSelectedModelId` without the corresponding `userSelectedModelConfiguration`, causing per-model settings to be dropped on follow-up requests.

## Change summary

Retry, confirmation, and rerun-with-agent paths in Chat do not forward the editor-scoped model configuration alongside the selected model ID, so per-model settings (e.g. reasoning effort) are lost on resends.

## Best solution

Add `userSelectedModelConfiguration` (via `widget.input.getModelConfiguration(...)`) to the request options in `chatTitleActions.ts`, `chatListWidget.ts`, `chatConfirmationContentPart.ts`, and `chatErrorConfirmationPart.ts` — mirroring the pattern already used in `chatWidget.ts` `acceptInput` and `rerunLastRequest`.
