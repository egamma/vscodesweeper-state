# Issue #277490 — I can crash the entire VS code by referencing a large XML file

[https://github.com/microsoft/vscode/issues/277490](https://github.com/microsoft/vscode/issues/277490)

## Summary

Valid OOM crash when referencing a multi-gigabyte file via `#file` in Copilot Chat. A team member (deepak1556) is actively investigating and has requested crash dumps, which the reporter provided. Keep open for ongoing investigation.

## Change summary

Referencing a very large XML file (e.g. a multi-GB Wikipedia dump) via `#filename` in Copilot Chat causes the renderer process to crash with an out-of-memory error (exit code 5).

## Best solution

Add a file-size guard in the chat file-reference resolution path (likely in the attachment/variable resolution code under `src/vs/workbench/contrib/chat/browser/attachments/`) to reject or truncate files above a safe threshold before reading them into the renderer process. Image attachments already have a 30 MB guard and repo-info reads have a 1 MB limit, but text file references via `#` appear unguarded for size. The crash dump analysis requested by deepak1556 should confirm the exact allocation site.
