# Issue #102574 — User settings for language configurations

[https://github.com/microsoft/vscode/issues/102574](https://github.com/microsoft/vscode/issues/102574)

## Summary

This is a valid, open feature request for user-configurable language configurations (autoClosingPairs, surroundingPairs, folding) via settings.json. The underlying asks are partially tracked by #38352 and #36002 but this issue has a broader framing (all language-configuration properties). With 14 upvotes it hasn't hit the 20-upvote backlog threshold but remains relevant.

## Change summary

Allow users to override language-configuration.json properties (autoClosingPairs, surroundingPairs, folding markers) through user settings rather than only via extensions.

## Best solution

The broadest version of this (exposing all language-configuration.json properties in settings.json, as alexdima suggested in 2021) would subsume #38352 and #36002. The right end state is a settings-based override layer in the language configuration registry (`src/vs/editor/common/languages/languageConfigurationRegistry.ts`) so users can tweak pairs/folding without authoring an extension. Until that ships, this issue captures the umbrella ask distinctly from the per-property issues.
