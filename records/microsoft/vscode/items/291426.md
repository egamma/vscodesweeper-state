# Issue #291426 — Codex sessions still appear in sessions list after uninstalling Codex

[https://github.com/microsoft/vscode/issues/291426](https://github.com/microsoft/vscode/issues/291426)

## Summary

Team-authored bug that remains unfixed on main. The sessions list filtering code does not check whether a session's provider is still registered, so cached sessions from uninstalled extensions persist in the list with broken resolution.

## Change summary

Sessions from uninstalled extension providers still appear in the sessions list and show resolution errors when clicked.

## Best solution

Add a filter in the sessions list (src/vs/sessions/contrib/sessions/browser/views/sessionsList.ts) that excludes sessions whose `providerId` does not match any currently registered provider in `ISessionsProvidersService`, or surface a graceful fallback when resolution fails for an unknown provider.
