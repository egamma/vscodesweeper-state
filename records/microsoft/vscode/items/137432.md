# Issue #137432 — Git - git.autoRepositoryDetection does not honour git.ignoredRepositories nor files.exclude

[https://github.com/microsoft/vscode/issues/137432](https://github.com/microsoft/vscode/issues/137432)

## Summary

The code at `extensions/git/src/model.ts:626` does check `git.ignoredRepositories` during auto-detection via `openRepository`, so the setting is honored — but only when configured with the exact repository root path. The `files.exclude` setting is never consulted by git detection code. The report likely reflects a misconfiguration or a desire for `files.exclude` to also suppress detection, which would be a feature request.

## Change summary

Git auto-repository detection reportedly ignores `git.ignoredRepositories` and `files.exclude` settings when discovering repos from opened editors.

## Best solution

Clarify whether the reporter's `ignoredRepositories` value matched the exact repository root (the check exists at model.ts:626). The `files.exclude` part is genuinely not implemented — if that's desired, it would be a small enhancement in `openRepository` to skip paths matching the exclude globs. Keep open pending clarification.
