# Issue #239285 — Mouse not changing to line cursor on text hover.

[https://github.com/microsoft/vscode/issues/239285](https://github.com/microsoft/vscode/issues/239285)

## Summary

Confirmed bug where the mouse cursor gets stuck as a pointer (instead of changing to the text/I-beam cursor) when hovering over editor text, especially after drag-from-explorer or screen recording. No fix found on current main.

## Change summary

Mouse cursor does not change to the text/I-beam cursor when hovering over editor content, remaining stuck as a default pointer until an external event resets it.

## Best solution

Investigate where the editor's CSS cursor state gets stuck after drag operations or other interactions. The fix likely involves explicitly resetting `cursor: text` on the editor content area after drag-end or drop events in `src/vs/editor/browser/controller/mouseHandler.ts` or the drag-and-drop controller. May also be an Electron/Chromium compositor issue on certain platforms (Wayland, macOS).
