# Issue #250392 — Accessible Diff Viewer not focusable with keyboard shortcuts after switching to different editor group.

[https://github.com/microsoft/vscode/issues/250392](https://github.com/microsoft/vscode/issues/250392)

## Summary

The bug describes a real focus-management issue in the Accessible Diff Viewer when switching editor groups via keyboard. No fix has landed on main for this specific scenario. The report is reasonably detailed but no commenter has confirmed reproduction, and the issue is in the accessibility domain — keeping open.

## Change summary

Accessible Diff Viewer in a diff editor group becomes unfocusable via keyboard shortcuts (Cmd+1/Cmd+2) after switching away and back.

## Best solution

Fix focus restoration logic in the diff editor widget or the editor group focus handler so that when an editor group containing a visible Accessible Diff Viewer regains focus, the viewer (or its parent diff editor) properly receives focus. Likely involves `src/vs/editor/browser/widget/diffEditor/components/accessibleDiffViewer.ts` and possibly the editor group focus transfer code in `src/vs/workbench/browser/parts/editor/`.
