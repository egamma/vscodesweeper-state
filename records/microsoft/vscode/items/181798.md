# Issue #181798 — punctuation echo issues under MacOS

[https://github.com/microsoft/vscode/issues/181798](https://github.com/microsoft/vscode/issues/181798)

## Summary

Still-open accessibility bug: VoiceOver on macOS doesn't announce certain punctuation (brackets, quotes) as they're typed in the editor. The discussion confirmed it's not Electron-specific (reproduces on vscode.dev) and not a plain textarea issue. No fixing commit found on main, but the report is from 2023 with macOS 13 — the current state with the EditContext rewrite is unknown.

## Change summary

VoiceOver does not echo certain punctuation characters (speech marks, curly braces, brackets) when typed in the VS Code editor, though a plain textarea echoes them correctly.

## Best solution

Verify whether the issue persists with the current EditContext-based input handling (which replaced the old textarea approach). If it does, investigate how the screen reader content update in `src/vs/editor/browser/controller/editContext/` handles auto-closing pairs — the auto-inserted closing bracket may interfere with VoiceOver's character echo. If it no longer reproduces with modern Electron/Chromium, close as not-reproducible.
