# Issue #134311 — Language detection server retains 5Mb if not cleaned up

[https://github.com/microsoft/vscode/issues/134311](https://github.com/microsoft/vscode/issues/134311)

## Summary

The language detection worker is still created once and never stopped — no lifecycle management has been added. The issue remains valid as a performance improvement opportunity.

## Change summary

The language detection worker retains ~5MB of model data after use because it is never stopped when idle.

## Best solution

Add idle-timeout lifecycle management to LanguageDetectionWorkerClient: after detection completes and a timeout elapses without further requests, terminate the worker and null out the reference, recreating it lazily on the next call. The fix is localized to `src/vs/workbench/services/languageDetection/browser/languageDetectionWorkerServiceImpl.ts`.
