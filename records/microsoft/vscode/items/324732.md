# Issue #324732 — Menu items in Explore section are missing indexing information for screen reader users:A11y_Visual Studio Code Client_Editor_Screen Reader

[https://github.com/microsoft/vscode/issues/324732](https://github.com/microsoft/vscode/issues/324732)

## Summary

Team-authored accessibility issue — keeping open for maintainer judgment. The Explorer toolbar's action bar (`ActionBar`) does not set `aria-posinset`/`aria-setsize` on its items, while dropdown menus (`Menu`) already do. The gap is in the toolbar surface.

## Change summary

Explorer toolbar buttons (New File, New Folder, etc.) lack `aria-posinset`/`aria-setsize` attributes, so screen readers cannot announce positional indexing like "item 1 of X".

## Best solution

Add `aria-posinset` and `aria-setsize` attributes to `ActionBar` items in `src/vs/base/browser/ui/actionbar/actionbar.ts`, mirroring the pattern already used in `src/vs/base/browser/ui/menu/menu.ts:300-302` where `updatePositionInSet` is called on non-separator items. This would propagate to all workbench toolbars, not just the Explorer.
