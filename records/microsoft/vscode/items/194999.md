# Issue #194999 — "Delete Permanently" has misguiding description

[https://github.com/microsoft/vscode/issues/194999](https://github.com/microsoft/vscode/issues/194999)

## Summary

Still reproduces on main: when permanently deleting a file (Shift+Delete), the confirmation dialog says 'You can restore this file using the Undo command,' which contradicts the 'permanently' framing and is factually wrong for files exceeding 5MB. The folder path correctly says 'This action is irreversible!' but the file path does not.

## Change summary

The 'Delete Permanently' confirmation dialog for files misleadingly says the action can be undone, contradicting the 'permanently' label and failing for large files.

## Best solution

The detail message for permanent file deletion (line 162-163 of `src/vs/workbench/contrib/files/browser/fileActions.ts`) should use 'This action is irreversible!' for files too — or at minimum warn that undo only works for small files. The `MAX_UNDO_FILE_SIZE` (5MB) threshold means the current message is factually wrong for large files, and semantically misleading for all files given the 'permanently' framing.
