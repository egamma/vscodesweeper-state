# Issue #249184 — SCM - Allow configuration via settings(.json) of quick diff decoration toggle state

[https://github.com/microsoft/vscode/issues/249184](https://github.com/microsoft/vscode/issues/249184)

## Summary

Valid feature request: the quick diff provider visibility toggle (e.g. for staged changes) is stored via IStorageService, not as a user-facing setting. Exposing it in settings.json would improve discoverability and allow per-workspace/profile configuration. Keeping open — 7 upvotes, under the 20-vote threshold but a reasonable ask in the SCM area.

## Change summary

Allow configuring which quick diff decoration providers (e.g. staged changes) are visible via settings.json instead of only through a gutter context menu toggle.

## Best solution

Migrate the hidden-provider state from `IStorageService` (`workbench.scm.quickDiffProviders.hidden` in `src/vs/workbench/contrib/scm/common/quickDiffService.ts`) to a proper configuration setting (e.g. `scm.diffDecorationsQuickDiffProviders`) registered in `src/vs/workbench/contrib/scm/browser/scm.contribution.ts`, so it appears in settings.json and the Settings UI. The commenter's suggestion of a command-palette toggle command is also worth considering for discoverability.
