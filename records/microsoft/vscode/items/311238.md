The postToolUse hook receives an empty `tool_response` field for the `fetch_webpage` tool in agent mode. Already assigned to TylerLeonhardt; routing to chat-hooks area for visibility.

## Change summary

The `tool_response` field in the postToolUse hook stdin JSON is empty, making it impossible to inspect tool output in hook scripts.

## Best solution

Fix the serialization so that the tool output text is available in the hook input.
