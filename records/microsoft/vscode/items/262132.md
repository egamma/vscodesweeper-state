# Issue #262132 — ObservableMap/Set API should be improved

[https://github.com/microsoft/vscode/issues/262132](https://github.com/microsoft/vscode/issues/262132)

## Summary

Team-authored debt issue requesting API improvements to ObservableMap/Set to make it harder to accidentally read values without tracking dependencies. Keeping open for maintainer judgment.

## Change summary

ObservableMap/Set API makes it too easy to access data without calling `read()`, leading to missed dependency tracking.

## Best solution

Redesign the ObservableMap/Set API surface so that accessing values without a reader is either impossible or requires an explicit opt-out (similar to how `observableValue` forces `.read(reader)` vs `.get()`), reducing accidental untracked reads like the one in PR #261793.
