# Issue #262916 — 3 Way Merge Editor issue with git subtree

[https://github.com/microsoft/vscode/issues/262916](https://github.com/microsoft/vscode/issues/262916)

## Summary

Still reproduces on main: the git extension constructs merge editor URIs using the workspace-relative path with the subtree prefix, but the ref points to an upstream commit where the file lives without that prefix — so the git content provider can't resolve it. No fix found in recent history.

## Change summary

The 3-way merge editor fails to open files during git subtree merge conflicts because the git URI path doesn't match the file's path at the referenced upstream commit.

## Best solution

The git extension's merge conflict URI construction (extensions/git/src/uri.ts) needs to handle subtree scenarios where the file path at a given ref differs from the workspace-relative path. The fix likely involves detecting subtree merges and adjusting the path in the git URI to strip the subtree prefix when resolving content at the upstream ref.
