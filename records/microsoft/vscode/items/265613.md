# Issue #265613 — Better integration of pre-commit hooks with Python virtual environments.

[https://github.com/microsoft/vscode/issues/265613](https://github.com/microsoft/vscode/issues/265613)

## Summary

Valid feature request with `help wanted` already applied. The git extension spawns git (and thus hooks) with VS Code's own process environment, not the Python venv PATH. This is a cross-extension integration gap that remains unaddressed on main. Low upvote count (2) means it hasn't reached the community threshold, but it's already triaged in Backlog with `help wanted`.

## Change summary

Git hooks run by VS Code's git extension don't inherit the Python virtual environment PATH, so pre-commit hooks can't find venv-installed tools unless VS Code is launched from an activated venv.

## Best solution

The git extension could expose a mechanism for other extensions (like Python) to contribute environment variables to the git spawn environment — similar to how terminal environment providers work via `context.environmentVariableCollection`. The fix area is `extensions/git/src/git.ts` around line 689 where `options.env` is assembled. Alternatively, the Python extension could be enhanced to contribute venv PATH to git operations. Given the `help wanted` label, this is positioned for a community contribution.
