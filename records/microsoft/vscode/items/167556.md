[https://github.com/microsoft/vscode/issues/167556](https://github.com/microsoft/vscode/issues/167556)

## Summary

Legitimate upstream tracking issue for a Chromium/Electron/Windows RDP freeze affecting multiple apps. A partial Chromium fix landed in m124 (https://chromium-review.googlesource.com/c/chromium/src/+/5365680), and current VS Code ships Electron 42 (Chromium ~128) which includes it, but there's no confirmation the full freeze scenario is resolved — keeping open as the upstream tracker with 55 upvotes.

## Change summary

VS Code window hangs and becomes unresponsive after switching between local login and Remote Desktop sessions on Windows, particularly with differing display scaling/resolution.

## Best solution

Continue tracking upstream progress. The partial Chromium fix from m124 is now included via Electron 42 (Chromium ~128). A maintainer should verify whether the full RDP freeze scenario (not just IME) is resolved on current stable, and if so close as fixed upstream.
