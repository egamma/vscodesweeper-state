# Issue #102236 — Add support for SteppingGranularity in the UI

[https://github.com/microsoft/vscode/issues/102236](https://github.com/microsoft/vscode/issues/102236)

## Summary

Still unimplemented: VS Code only passes SteppingGranularity as 'instruction' when the disassembly view is focused, with no UI for users to toggle between statement/line/instruction stepping. With 91 upvotes this well exceeds the backlog threshold and should remain open.

## Change summary

Add a UI control allowing users to select the DAP SteppingGranularity (statement, line, or instruction) for debug step commands.

## Best solution

Add a toggle or dropdown in the debug toolbar (or a setting) that lets users select the stepping granularity. The plumbing already exists — `IThread.next/stepIn/stepOut/stepBack` all accept a `granularity` parameter, and `debugSession.ts` forwards it to the DAP adapter. The UI just needs a way to set it outside of the disassembly-view-focus context check in `debugCommands.ts`.

## Evidence

- **no UI implementation:** In src/vs/workbench/contrib/debug/browser/debugCommands.ts, the step commands (STEP_OVER_ID, STEP_INTO_ID, STEP_OUT_ID) only pass 'instruction' granularity when CONTEXT_DISASSEMBLY_VIEW_FOCUS is true, and undefined otherwise. There is no user-facing toggle or setting for choosing granularity.
- **plumbing exists:** The DAP protocol types (debugProtocol.d.ts:2276), session methods (debugSession.ts:736-772), and thread interface (debug.ts:541-544) all accept SteppingGranularity — only the UI affordance to select it is missing.
- **community demand:** 91 community upvotes, well above the 20-upvote backlog promotion threshold. Multiple commenters from different debug adapter ecosystems (Julia, TLA+, Dart, C/C++) have expressed interest.
- **supportsSteppingGranularity capability:** debugProtocol.d.ts:1836 defines a `supportsSteppingGranularity` capability that adapters can advertise, which could gate the UI toggle.
