# Issue #303391 — Error during settings sync

[https://github.com/microsoft/vscode/issues/303391](https://github.com/microsoft/vscode/issues/303391)

## Summary

The user has 13 local profiles (under the 20-profile limit) but the sync check at line 181 of userDataProfilesManifestSync.ts counts remoteProfiles from the server state, which may still contain stale entries from previously-deleted profiles. This is a plausible bug in remote state cleanup, but needs more information (e.g. what the remote profile count actually is).

## Change summary

Settings Sync throws 'Too many profiles' error even though local profile count (13) is well below the 20-profile limit, likely due to stale remote sync state.

## Best solution

The settings-sync team should investigate whether the remote profile state is being properly cleaned up when profiles are deleted locally. The check on line 181 of `userDataProfilesManifestSync.ts` validates `remoteProfiles.length` — if old remote entries aren't purged, users who previously exceeded 20 profiles and then reduced will stay stuck. A workaround (clearing remote sync data) or a code fix (reconciling stale remote profiles) is needed.
