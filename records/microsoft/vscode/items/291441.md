# Issue #291441 — Keep new users on the welcome page after chat if not signed in

[https://github.com/microsoft/vscode/issues/291441](https://github.com/microsoft/vscode/issues/291441)

## Summary

Team-authored issue (MEMBER) describing a UX polish gap in the welcome-page-to-chat transition for unauthenticated users — keeping open for maintainer judgment.

## Change summary

New users who submit a chat request from the welcome view are transitioned to the maximized chat view even when they hit the sign-in dialog and haven't authenticated, leaving them in a confusing state away from the welcome actions.

## Best solution

When a chat request from the welcome view triggers the sign-in dialog and the user is not yet authenticated, keep them on the welcome page so they retain access to the other onboarding actions. The fix likely lives in the welcome view or chat widget transition logic that handles the post-submit navigation.
