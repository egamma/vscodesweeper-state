# Issue #275981 — [Feedback] [Plan Mode] Confusing UX for "Open in Editor" option and .prompt.md file extension

[https://github.com/microsoft/vscode/issues/275981](https://github.com/microsoft/vscode/issues/275981)

## Summary

Valid UX feedback on plan mode handoff button labeling and file extension. A team member (aeschli) agreed that .md would be better. The handoff labels come from mode/prompt definitions and haven't been renamed yet on main. Keeping open as an accepted feature request in Backlog.

## Change summary

The 'Open in Editor' handoff button label is ambiguous, and the resulting .prompt.md file extension is confusing to users who expect a plain .md file.

## Best solution

Rename the plan mode's 'Open in Editor' handoff label to something like 'Save Plan' or 'Export Plan to File' in the plan mode definition, and default to .md instead of .prompt.md for the saved plan file. The fix likely lives in the Copilot extension's plan mode prompt definition (where handoffs are declared), not in VS Code core.
