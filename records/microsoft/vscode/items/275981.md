Valid UX feedback on Plan Mode's 'Open in Editor' button label and .prompt.md file extension, acknowledged by a team member (aeschli). Belongs in chat-plan-agent area and should stay open for the feature owner to address.

## Change summary

The 'Open in Editor' button in Plan Mode is confusing — users don't know if it opens code files or saves the plan, and the .prompt.md extension is unexpected.

## Best solution

Rename the button to something like 'Save Plan to File' or 'Export Plan,' simplify the default extension to .md (or explain .prompt.md's purpose), and auto-focus the newly created file tab. The chat-plan-agent area owner (@digitarald, as tagged by aeschli) should decide on final wording.
