# Issue #308524 — Update fails with OS error 32 when renaming Code.exe (file locked by child process)

[https://github.com/microsoft/vscode/issues/308524](https://github.com/microsoft/vscode/issues/308524)

## Summary

Real bug in the inno-updater's rename path (no retry on OS error 32), with an open fix PR at microsoft/inno-updater#69 that hasn't merged yet. The reporter notes the issue hasn't recurred since the update button was moved, but the underlying code path is still unguarded. Keeping open — the linked PR is the right fix lane and a human owns it.

## Change summary

The inno-updater's `perform_three_way_rename` fails immediately with OS error 32 when Code.exe is locked by a child process, instead of retrying like other file operations in the same codebase.

## Best solution

Land https://github.com/microsoft/inno-updater/pull/69 (retry logic for the rename step), then bump the inno-updater dependency in VS Code. The fix lives in the external `microsoft/inno-updater` repo, not in this repository. Once merged, this issue and the related https://github.com/microsoft/vscode/issues/296791 can likely be closed.
