# Issue #308524 — Update fails with OS error 32 when renaming Code.exe (file locked by child process)

[https://github.com/microsoft/vscode/issues/308524](https://github.com/microsoft/vscode/issues/308524)

## Summary

Keep this open: the report is a plausible Windows updater bug, and the implicated rename logic lives in the external `inno_updater.exe` component rather than VS Code TypeScript code. I found no concrete fix or binary bump on current `main` after the report date.

## Change summary

Windows updates can fail when `inno_updater.exe` tries once to rename `Code.exe` while another process still holds a lock, producing OS error 32.

## Best solution

Track this against the Inno updater executable replacement path and either add retry/backoff around the three-way executable rename there or confirm a separate lock source with a reproducible `code` command scenario.
