# Issue #308524 — Update fails with OS error 32 when renaming Code.exe (file locked by child process)

[https://github.com/microsoft/vscode/issues/308524](https://github.com/microsoft/vscode/issues/308524)

## Summary

The bug lives in the inno-updater rename logic (no retry on OS error 32). There's an open PR in microsoft/inno-updater adding retry logic, and VS Code main already added COM surrogate termination before updates. The reporter notes it hasn't recurred since the update button was moved. Keeping open as the inno-updater PR hasn't merged yet.

## Change summary

The Windows updater fails immediately with OS error 32 when renaming Code.exe because a child process still holds a file lock, with no retry logic.

## Best solution

Merge https://github.com/microsoft/inno-updater/pull/69 (retry logic for the rename step), then bump the inno-updater dependency in VS Code's `build/win32/Cargo.toml`. The COM surrogate fix in c2a42605f78 already addresses one source of locks; the retry covers the remaining transient cases.
