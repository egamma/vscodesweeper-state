# Issue #207221 — can't access module `original-fs` in remote machine

[https://github.com/microsoft/vscode/issues/207221](https://github.com/microsoft/vscode/issues/207221)

## Summary

The `original-fs` module is intentionally only available in Electron-based (local) extension hosts. Remote extension hosts run in a pure Node.js environment where no ASAR patching occurs, so `original-fs` has no meaning there — this is by design, as confirmed by the maintainer comment.

## Change summary

Extension cannot require the `original-fs` module when running in a remote extension host.

## Best solution

Close as designed. Remote extension hosts are pure Node.js — `fs` already behaves as the "original" fs there (no ASAR interception). Extensions needing `original-fs` should conditionally require it only in local hosts, or simply use `fs` in remote contexts.
