# Issue #207221 — can't access module `original-fs` in remote machine

[https://github.com/microsoft/vscode/issues/207221](https://github.com/microsoft/vscode/issues/207221)

## Summary

The `original-fs` module is an Electron-only concept that bypasses ASAR archive interception — it intentionally does not exist in remote extension hosts, which run pure Node.js where `fs` is already the "original" fs. bpasero confirmed this in the discussion. Proposing close as designed.

## Change summary

Extension cannot `require('original-fs')` in a remote extension host because the module does not exist there.

## Best solution

Close as designed. In remote extension hosts (pure Node.js), `fs` is already the real filesystem module — there is no ASAR interception to bypass. Extensions that need to work in both local and remote should use `fs` and handle the `original-fs` import conditionally (e.g. try/catch fallback to `fs`).
