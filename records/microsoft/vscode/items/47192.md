## Summary

Feature request to show HTML 4 attributes (cellPadding, cellSpacing, valign, align) in autocomplete remains unimplemented on main, but a workaround exists via HTML custom data. Keeping open as it's in the Backlog milestone with maintainer interest.

## Change summary

HTML autocomplete does not suggest obsolete HTML 4 attributes like cellPadding, cellSpacing, valign, and align, which are still needed for email templates.

## Best solution

Either add deprecated HTML 4 attributes to the built-in HTML data (possibly with a deprecation marker), or document the HTML custom data workaround more prominently and close as *as-designed*. The HTML language service data lives upstream in vscode-html-languageservice; the fix would likely belong there rather than in vscode core.
