# Issue #140719 — C symbol names are not displayed in the Disassembly view

[https://github.com/microsoft/vscode/issues/140719](https://github.com/microsoft/vscode/issues/140719)

## Summary

Still valid on current main: the Disassembly view's InstructionRenderer never reads or renders the `symbol` field from DAP's DisassembledInstruction. The feature request to display symbol names remains unimplemented.

## Change summary

Display the `symbol` field from DisassembledInstruction objects in the Disassembly view so users can see function/symbol names alongside disassembled instructions.

## Best solution

Add rendering of the `symbol` field in `InstructionRenderer.renderElementInner` (src/vs/workbench/contrib/debug/browser/disassemblyView.ts), likely as a label displayed before the first instruction at a new symbol boundary — similar to how Eclipse shows symbol names as section headers in its disassembly view.
