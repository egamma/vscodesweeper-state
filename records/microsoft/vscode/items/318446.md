## Summary

The reported layout shift when switching sessions — terminal output block positioned too low and clipped — appears to be addressed by commit b7f08a3026d on main, which publishes fresh dynamic row measurements to the delegate cache so chat can size the last response correctly before the initial ResizeObserver report. The fix landed after the issue was filed and is main-only.

Terminal output block gets clipped at the bottom of the chat scroll pane when switching to a session with a running command; resizing the window fixes the layout.
