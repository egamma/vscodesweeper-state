# Issue #321116 — Git: "Fetch (Prune)" should also prune local tags deleted from remote (--prune-tags)

[https://github.com/microsoft/vscode/issues/321116](https://github.com/microsoft/vscode/issues/321116)

## Summary

Valid feature request — the git extension's fetch-prune path currently only passes `--prune` and has no support for `--prune-tags`. The feature does not exist on current main.

## Change summary

Request to add `--prune-tags` support to the Git Fetch (Prune) command so stale local tags deleted from the remote are also cleaned up.

## Best solution

Add a `git.pruneTagsOnFetch` boolean setting (default false) and pass `--prune-tags` alongside `--prune` in `extensions/git/src/git.ts` when enabled. Alternatively, expose a separate 'Fetch (Prune Tags)' command. This is a small, self-contained change in the built-in git extension.
