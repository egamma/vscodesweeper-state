# Issue #187232 — VSCode UI: wierd white line in certain window height

[https://github.com/microsoft/vscode/issues/187232](https://github.com/microsoft/vscode/issues/187232)

## Summary

This is a longstanding subpixel rendering artifact in Chromium/Electron that surfaces at non-integer device-pixel ratios or specific window heights. A nearly identical issue (#23968) was previously closed as upstream. Keeping open since the `layout` label is already applied and a commenter confirmed it at 125%/175% zoom, but this is likely an Electron/Chromium compositor issue rather than something fixable in VS Code's layout code.

## Change summary

A 1px white line appears at the bottom of the status bar area at certain window heights or non-standard zoom levels, caused by subpixel rendering gaps.

## Best solution

Reclassify as `upstream` — this is a Chromium compositor subpixel rounding artifact at fractional device-pixel ratios. The previous instance (https://github.com/microsoft/vscode/issues/23968) was closed under that label. If it persists on current Electron versions, a workaround in the workbench layout (floor/ceil on computed part heights) could be explored, but the root fix belongs in Chromium.
