# Issue #174722 — Deleted files move to another directory, not to trash

[https://github.com/microsoft/vscode/issues/174722](https://github.com/microsoft/vscode/issues/174722)

## Summary

Correctly triaged upstream issue: VS Code calls Electron's standard `shell.trashItem()` API, which in turn calls the correct macOS API, but OneDrive's file-provider implementation mishandles the trash operation on CloudStorage-mounted directories. Nothing actionable on the VS Code side; the bug lives in OneDrive's macOS integration.

## Change summary

Deleting files in OneDrive/CloudStorage folders via VS Code moves them to the OneDrive root directory instead of the macOS Trash.

## Best solution

The fix belongs in OneDrive's macOS file-provider implementation. Since both VS Code and OneDrive are Microsoft products, the most productive path is an internal escalation to the OneDrive team referencing https://github.com/electron/electron/issues/38541. VS Code's `shell.trashItem()` call in `src/vs/platform/files/electron-main/diskFileSystemProviderServer.ts` is correct and should not be worked around.
