# Issue #66801 — Input variables only work with one configuration in a compound launch config

[https://github.com/microsoft/vscode/issues/66801](https://github.com/microsoft/vscode/issues/66801)

## Summary

Still reproduces on current main: compound debug launch uses `Promise.all` to start all configurations concurrently, and each calls `substituteVariables` which shows a quickpick for input variables. Concurrent quickpicks dismiss each other, so only the last configuration successfully resolves its input. Two fix attempts in 2022 were both reverted due to a deadlock regression.

## Change summary

When a compound debug configuration references input variables, only the last configuration in the compound is launched because concurrent quickpick prompts cancel each other.

## Best solution

Resolve input variables for all compound configurations sequentially (or pre-resolve shared inputs once) before launching sessions in parallel. The root blocker is https://github.com/microsoft/vscode/issues/158433 (quickpicks dismissing each other). A targeted fix in `debugService.ts` around line 400 could resolve inputs before the `Promise.all`, or the compound launch loop could be changed from parallel to sequential for the variable-resolution phase. Previous fix attempts (commits f1c5243126a and 0dddb856fd8) were reverted due to deadlocks — any new approach must avoid re-introducing that.
