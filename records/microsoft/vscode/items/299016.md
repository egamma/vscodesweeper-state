# Issue #299016 — Provide optional notification when Git auto-fetch detects remote changes

[https://github.com/microsoft/vscode/issues/299016](https://github.com/microsoft/vscode/issues/299016)

## Summary

Reasonable feature request for an optional notification when auto-fetch detects the local branch is behind the remote. The status bar already shows behind counts, but there is no proactive pop-up notification. No duplicate found. Routing to the git area for consideration.

## Change summary

Add an optional notification (pop-up or enhanced status bar indicator) when git auto-fetch detects new remote commits on the tracked branch.

## Best solution

Add an opt-in setting (e.g. `git.autofetchNotify`) that, when enabled, fires a notification after auto-fetch detects `HEAD.behind > 0` increasing. Implementation would live in `extensions/git/src/autofetch.ts` (the `run()` loop) and compare before/after behind counts. The existing status bar indicator in `extensions/git/src/statusbar.ts` already shows behind counts passively, so the notification should be additive and disabled by default to avoid notification fatigue.
