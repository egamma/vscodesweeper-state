# Issue #231837 — gif/video support in chat

[https://github.com/microsoft/vscode/issues/231837](https://github.com/microsoft/vscode/issues/231837)

## Summary

Team-authored feature request for gif/video support in chat — still unimplemented on main. GIF images are accepted but only the first frame is sent to the model; no video (mp4/webm/mov) support exists. Keeping open per team-member policy.

## Change summary

Allow users to attach gifs (with animation) and videos to chat, so models can process screen recordings and animated content.

## Best solution

Extend the chat attachment pipeline to extract frames from gifs/videos (following the OpenAI cookbook pattern justschen cited) and send them as a sequence of images. This likely touches `src/vs/workbench/contrib/chat/browser/chatImageUtils.ts` (frame extraction), `src/vs/workbench/contrib/chat/browser/widget/input/editor/chatPasteProviders.ts` (accepted MIME types), and `src/vs/workbench/contrib/chat/common/languageModels.ts` (content part types). Needs a product decision on frame limits and token budget.
