# Issue #99159 — Partial cross-file undo if a file is changed in the meantime

[https://github.com/microsoft/vscode/issues/99159](https://github.com/microsoft/vscode/issues/99159)

## Summary

Team-authored feature request exploring partial cross-file undo when some files have been modified. The discussion includes nuanced design considerations from alexdima about predictability vs. partial application. Kept open per team-author rule.

## Change summary

When a cross-file undo (e.g. after rename) fails because one file changed, allow undoing the unaffected files rather than failing entirely.

## Best solution

Design decision needed on whether partial cross-file undo is acceptable UX (alexdima raised valid concerns about predictability). If pursued, the undo/redo infrastructure in `src/vs/workbench/common/editor/` would need to support partial element application with user notification of which files were skipped. Related to https://github.com/microsoft/vscode/issues/99295 (random-access undos).
