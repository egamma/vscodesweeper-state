# Issue #121799 — [remote][connection] Allow offline editing in Remote-WSL

[https://github.com/microsoft/vscode/issues/121799](https://github.com/microsoft/vscode/issues/121799)

## Summary

Issue has milestone 'On Deck' (a scheduling decision), so it must stay open. Additionally, team member alexdima confirmed reproduction and created a dedicated repro at https://github.com/alexdima/wsl-socket-sample — this is a real, investigated bug with active team interest.

## Change summary

Remote-WSL WebSocket connections are dropped by Chromium/OS when a network adapter re-enables after a disconnect, causing a ~30s reconnection loop that blocks editing.

## Best solution

Investigate whether the WSL connection can use a more resilient transport (e.g. a Node.js socket proxy instead of browser WebSocket) that survives network adapter state changes, or implement a grace period / offline-edit mode that keeps the editor usable during short reconnection windows. The related issues in vscode-remote-release (#5894, #5960) about a wsl.exe proxy may address this.
