# Issue #121799 — [remote][connection] Allow offline editing in Remote-WSL

[https://github.com/microsoft/vscode/issues/121799](https://github.com/microsoft/vscode/issues/121799)

## Summary

Issue is on the 'On Deck' milestone (a scheduling decision), so it must remain open. Additionally, the bug was actively reproduced and investigated by team members with a concrete repro repo.

## Change summary

Remote-WSL reconnection modal blocks editing when network connectivity is unstable, because regaining WiFi/Ethernet causes Chromium to close WebSockets, triggering a ~30s reconnection loop.

## Best solution

Improve the reconnection experience so that local editing remains possible during transient disconnects in WSL — either by buffering edits locally during the reconnection window, suppressing the blocking modal for short outages, or investigating whether the WebSocket teardown on network-regain is a Chromium/WSL issue that can be worked around at the connection layer.
