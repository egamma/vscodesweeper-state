# Issue #175282 — [Accessibility] Do not use title attribute when labeling buttons

[https://github.com/microsoft/vscode/issues/175282](https://github.com/microsoft/vscode/issues/175282)

## Summary

Hard keep-open: milestone is 'On Deck'. The core ActionBar now sets aria-label alongside title (lines 250/428 in actionViewItems.ts), but the issue tracks a broader initiative to audit all buttons — it remains valid as a tracking item for the accessibility team.

## Change summary

JAWS screen reader does not announce buttons that rely solely on `title` attribute for their accessible name; `aria-label` should be used instead.

## Best solution

Audit remaining button surfaces (beyond ActionBar) that still rely on `title` alone for labeling, and ensure `aria-label` is consistently set. The ActionBar base class already does this; the remaining gap is likely in one-off buttons or custom widgets outside that path.
