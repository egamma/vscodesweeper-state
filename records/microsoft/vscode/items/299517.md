# Issue #299517 — Not enough contrast in permission dropdown

[https://github.com/microsoft/vscode/issues/299517](https://github.com/microsoft/vscode/issues/299517)

## Summary

Legitimate visual contrast bug in the permission picker dropdown when using a light theme. The issue is in VS Code core's action widget CSS (not an extension). No fix has been committed since the report, and the styling relies on standard theme variables that may not provide sufficient contrast for the selected/checked item in light mode. Routing to the chat area for owner review.

## Change summary

The permission dropdown in the chat input area has insufficient text contrast when viewed in a light color theme (e.g. GitHub Light).

## Best solution

Audit the contrast of the checked/selected item in `src/vs/platform/actionWidget/browser/actionWidget.css` under light themes. The `.action-widget .monaco-list .monaco-list-row.action.focused` styling and description foreground colors may need a light-theme-specific adjustment or a better fallback for `--vscode-descriptionForeground` in the action widget context.
