# Issue #176666 — Split View: Allow Snap To Content Size

[https://github.com/microsoft/vscode/issues/176666](https://github.com/microsoft/vscode/issues/176666)

## Summary

Team-authored feature request for split view snap-to-content-size behavior — hard keep-open rule applies (author is MEMBER). The feature remains unimplemented on main.

## Change summary

Allow split view panes to snap their size to their content height, binding the pane size to content size until the splitter is manually moved.

## Best solution

Implement an optional 'snap size' property and corresponding change event in the SplitView widget, allowing views to declare a preferred content size that sashes lock to. The layout algorithm would preserve this size until the user manually repositions the sash. Likely touches `src/vs/base/browser/ui/splitview/splitview.ts` and view container contributions in the debug pane.
