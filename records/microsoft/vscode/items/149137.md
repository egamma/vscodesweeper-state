# Issue #149137 — Problem matcher network path bug

[https://github.com/microsoft/vscode/issues/149137](https://github.com/microsoft/vscode/issues/149137)

## Summary

The bug is still present on main: `getResource` in `problemMatcher.ts` processes UNC paths through `normalize` and backslash-to-slash replacement before calling `URI.file()`, but depending on the `fileLocation` mode and how the compiler output interacts with `filePrefix`, UNC authority can be lost or a UriError thrown. No fix commit found.

## Change summary

Running a task with a problem matcher on a UNC-path workspace produces a UriError instead of populating the Problems panel.

## Best solution

Fix the UNC-path handling in `getResource()` (`src/vs/workbench/contrib/tasks/common/problemMatcher.ts` lines 196–246) so that paths starting with `\\` or `//` are correctly handed to `URI.file()` regardless of `fileLocation` mode; add a unit test with a UNC `filePrefix`.
