# Issue #152907 — GitHub - Unable to push to existing fork with codespaces

[https://github.com/microsoft/vscode/issues/152907](https://github.com/microsoft/vscode/issues/152907)

## Summary

Still reproduces on main: the push error handler only offers 'Create Fork' when push is denied, but doesn't handle the case where a fork already exists. The code itself acknowledges this gap with a TODO comment.

## Change summary

Pushing to an existing fork from a Codespace (or locally) fails because the push error handler only offers to create a new fork, which errors out when one already exists.

## Best solution

Update `handlePermissionDeniedError` in `extensions/github/src/pushErrorHandler.ts` to detect an existing fork (via `octokit.repos.get` or `octokit.repos.listForks`) and add it as a remote instead of always calling `createFork`. The TODO comment at line 192 already flags this exact gap.
