# Issue #204608 — The Context does not set hasSearchResult when searching directly in the SearchEditor

[https://github.com/microsoft/vscode/issues/204608](https://github.com/microsoft/vscode/issues/204608)

## Summary

Confirmed bug still present on main: the SearchEditor never sets the `hasSearchResult` context key after a search completes — only the Search panel view does.

## Change summary

The `hasSearchResult` context key is never set to `true` when performing a search directly in the SearchEditor.

## Best solution

Bind and update `Constants.SearchContext.HasSearchResults` in the SearchEditor's `triggerSearch` completion path (in `src/vs/workbench/contrib/searchEditor/browser/searchEditor.ts`) based on whether the search model produced results, mirroring what `searchView.ts` does at line ~2063.
