# Issue #204608 — The Context does not set hasSearchResult when searching directly in the SearchEditor

[https://github.com/microsoft/vscode/issues/204608](https://github.com/microsoft/vscode/issues/204608)

## Summary

Confirmed bug still present on main: the `hasSearchResult` context key is only bound and set in SearchView, never in SearchEditor — so commands gated on that key don't work correctly when using the SearchEditor.

## Change summary

The `hasSearchResult` context key is not set to `true` when a search in the SearchEditor produces results.

## Best solution

In `src/vs/workbench/contrib/searchEditor/browser/searchEditor.ts`, bind `SearchContext.HasSearchResults` to the SearchEditor's scoped context key service and set it to `true` after a successful search completes (around the `triggerSearch` / result rendering path). This mirrors what `searchView.ts` does at line 2063.
