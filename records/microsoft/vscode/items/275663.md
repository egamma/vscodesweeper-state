# Issue #275663 — Model picker needs to react to changes in model visibility in other windows

[https://github.com/microsoft/vscode/issues/275663](https://github.com/microsoft/vscode/issues/275663)

## Summary

Team-authored bug, still present on main. The model visibility state is stored in profile-scoped storage but the service never listens for cross-window storage changes, so other windows don't pick up visibility updates.

## Change summary

Changing visible models in one VS Code window does not propagate to other open windows; a reload is required.

## Best solution

Add an `IStorageService.onDidChangeValue` listener for the `chatModelVisibility` key (and likely `chatModelPinned` / `chatModelRecentlyUsed` too) in `LanguageModelsService` to call `_readVisibility()` and fire `_onDidChangeModelVisibility` when another window writes the key. The fix is localized to `src/vs/workbench/contrib/chat/common/languageModels.ts`.
