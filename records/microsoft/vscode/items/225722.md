# Issue #225722 — Editor GPU: Persist texture atlas to disk

[https://github.com/microsoft/vscode/issues/225722](https://github.com/microsoft/vscode/issues/225722)

## Summary

Valid optimization request for the GPU renderer — persist the texture atlas to disk to avoid cold-start rasterization. Not yet implemented on main; tracked as a plan-item by the GPU renderer maintainer.

## Change summary

Persist the GPU renderer's texture atlas to disk so it survives restarts, invalidating on theme/font changes.

## Best solution

Implement a serialization/deserialization path in `src/vs/editor/browser/gpu/atlas/textureAtlas.ts` that writes the atlas pages and glyph map to a cache file (keyed by theme + font config), loading on startup when valid. This is a performance optimization for the editor-gpu area.
