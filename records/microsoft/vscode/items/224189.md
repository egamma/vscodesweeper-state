# Issue #224189 — Diffs panels are not splitted equally when Word Wrap option is enabled

[https://github.com/microsoft/vscode/issues/224189](https://github.com/microsoft/vscode/issues/224189)

## Summary

Valid feature request: when word wrap is enabled in the diff editor, the 50/50 split doesn't account for the different visual heights caused by wrapping, leaving a grey/empty area on the side with fewer wrapped lines. The splitViewDefaultRatio logic in diffEditorSash.ts is purely width-based and doesn't consider content height. Already triaged into Backlog as a feature-request with the diff-editor label — keeping open.

## Change summary

Diff editor panels should split more intelligently (or allow content-aware sizing) when word wrap causes unequal visual line heights between the two sides.

## Best solution

The diff editor sash positioning (in `src/vs/editor/browser/widget/diffEditor/components/diffEditorSash.ts`) could be enhanced to consider the wrapped content height of both sides, or the diff editor could synchronize the vertical padding/folding so both sides visually align. This is a non-trivial layout change in the diff editor widget.
