# Issue #109221 — SCM Input - suggest widget is behind editor

[https://github.com/microsoft/vscode/issues/109221](https://github.com/microsoft/vscode/issues/109221)

## Summary

Safari-specific z-index stacking issue where the SCM input suggest widget renders behind the editor. The milestone is 'On Deck' (a scheduling milestone), so this must stay open. The bug was re-confirmed by a team member in December 2024 with a video showing flickering and the widget initially appearing behind the editor.

## Change summary

In Safari, the suggest widget in the SCM input renders behind the main editor area due to a browser-specific stacking context issue.

## Best solution

Investigate Safari's stacking context handling for the `.scm-overflow-widgets-container` (which already has `z-index: 5000` in `src/vs/workbench/contrib/scm/browser/media/scm.css`). The fix likely requires either restructuring the DOM to avoid a conflicting stacking context created by the workbench editor area in Safari, or applying a Safari-specific workaround (e.g. `transform: translateZ(0)` to force GPU compositing). The `help wanted` label makes this a good community contribution candidate.
