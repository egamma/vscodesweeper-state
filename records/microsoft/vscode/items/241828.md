# Issue #241828 — Accepting a completion should trigger suggest automatically

[https://github.com/microsoft/vscode/issues/241828](https://github.com/microsoft/vscode/issues/241828)

## Summary

Partially addressed: re-triggering now works for folder completions and when insertTrailingSpace appends a space (commit e0af9ccaffa), but the general request — re-trigger after any accepted completion — remains unimplemented. Keeping open as a valid enhancement in the terminal-suggest area.

## Change summary

After accepting a terminal suggest completion, automatically show the next set of suggestions so the user can tab-flow through building a command.

## Best solution

Extend the acceptance logic in `terminalSuggestAddon.ts` so that after any completion acceptance (not just trailing-space or folder cases), completions are re-requested. The `lastAcceptedCompletionTimestamp` guard at line 284 currently blocks this; the fix would generalize the `_requestCompletionsOnNextSync` pattern or relax the guard conditionally based on a setting.
