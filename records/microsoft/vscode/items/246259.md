# Issue #246259 ÔÇö Discard changes of an Untracked file raise an error if no recycle bin option

[https://github.com/microsoft/vscode/issues/246259](https://github.com/microsoft/vscode/issues/246259)

## Summary

Still reproduces on main: the git extension's `_clean` method checks only `git.discardUntrackedChangesToTrash` but ignores `files.enableTrash`, so when the user disables the recycle bin globally, untracked-file discard still attempts `useTrash: true` and errors. An open PR (#305686) addresses this. Milestone is 'On Deck' ÔÇö keeping open.

## Change summary

Discarding an untracked file errors when `files.enableTrash` is disabled because the git extension unconditionally attempts trash-based deletion.

## Best solution

Land or review https://github.com/microsoft/vscode/pull/305686 which adds a `files.enableTrash` check. The fix is in `extensions/git/src/repository.ts` (`_clean` method) and possibly `extensions/git/src/commands.ts` (dialog text logic). Both sites should respect `files.enableTrash` alongside `git.discardUntrackedChangesToTrash`.
