# Issue #241563 — Sticky scroll doesn't work when function result is destructured

[https://github.com/microsoft/vscode/issues/241563](https://github.com/microsoft/vscode/issues/241563)

## Summary

Correctly triaged as upstream — the root cause is in TypeScript's navigation tree (navtree) which omits object-literal properties when the result is destructured. Already linked to the upstream TypeScript issue; nothing for VS Code to fix.

## Change summary

Sticky scroll fails to show inner elements when a function call result is destructured, because TypeScript's navtree omits them in that pattern.

## Best solution

Wait for TypeScript to fix the navtree inconsistency in `src/services/navigationBar.ts` (https://github.com/microsoft/TypeScript/issues/63551). No VS Code-side change is needed — once TypeScript emits the correct outline, sticky scroll will work automatically.
