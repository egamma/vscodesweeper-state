# Issue #319160 — Inconsistent tool list syntax between "Configure Tools..." CodeLens and tools icon for custom agents

[https://github.com/microsoft/vscode/issues/319160](https://github.com/microsoft/vscode/issues/319160)

## Summary

Valid inconsistency report: both the CodeLens and tools icon share the same `showToolsPicker` → `toFullReferenceNames` pipeline, but the tree-to-map collection logic may handle MCP bucket-level checked state differently depending on how the picker was entered. This is a real UX polish gap worth keeping open.

## Change summary

When adding an entire MCP server's tools via the chat tools icon, individual tools are listed instead of using the wildcard `server/*` syntax that the CodeLens produces.

## Best solution

Ensure `collectResults` in `chatToolPicker.ts` consistently marks MCP toolsets as enabled when all member tools are checked (regardless of entry point), so `toFullReferenceNames` emits wildcard references. The fix likely belongs in `src/vs/workbench/contrib/chat/browser/actions/chatToolPicker.ts` around the MCP bucket handling (lines 577–582).
