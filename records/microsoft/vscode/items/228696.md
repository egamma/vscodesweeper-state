# Issue #228696 — Change the global `fetch` available in local extension hosts to be the one from Electron

[https://github.com/microsoft/vscode/issues/228696](https://github.com/microsoft/vscode/issues/228696)

## Summary

Team-authored tracking issue for switching the global `fetch` in local extension hosts to Electron's `net.fetch`. The infrastructure is in place but disabled by default (`useElectronFetchDefault = false`) due to Electron limitations (manual redirects, unsafe headers, integrity). The author explicitly put this on hold pending evaluation of Node.js' own fetch with proxy support. Keeping open per team-authored rule and because the work is genuinely incomplete.

## Change summary

Replace the global `fetch` in local extension hosts with Electron's `net.fetch` to gain automatic proxy and certificate support without extra patching.

## Best solution

The plumbing in `src/vs/workbench/api/node/proxyResolver.ts` already patches `globalThis.fetch` to optionally route through `electron.net.fetch` (gated by `http.electronFetch`, default `false`). The remaining work is resolving Electron limitations (manual redirects per electron/electron#43715, unsupported headers, integrity/blob/data URLs) and deciding whether Node.js' native fetch with proxy support (tracked in https://github.com/microsoft/vscode/issues/228697, now closed) is sufficient instead. A maintainer decision is needed on which path to pursue before enabling by default.
