# Issue #24783 — vscode reads AltGr key as cursor-left move through X11 connections

[https://github.com/microsoft/vscode/issues/24783](https://github.com/microsoft/vscode/issues/24783)

## Summary

This should stay open: multiple recent reports say the AltGr/cursor-left behavior still reproduces in remote X11/RDP-style Linux sessions, and the known `keyboard.dispatch: keyCode` workaround is incomplete because it breaks other shortcuts such as Ctrl+/ on some layouts.

## Change summary

AltGr input over remote Linux desktop/X11 connections emits the intended character and also triggers cursor-left navigation in VS Code.

## Best solution

Keep this tracked in the keybindings/Linux area until the keyboard event handling can distinguish AltGr composition from cursor-left in remote desktop/X11 forwarding scenarios, or until maintainers document a supported per-environment workaround that does not regress layout-dependent shortcuts.
