# Issue #24783 — vscode reads AltGr key as cursor-left move through X11 connections

[https://github.com/microsoft/vscode/issues/24783](https://github.com/microsoft/vscode/issues/24783)

## Summary

Still reproduces on current main — no fix has landed for the AltGr/X11 forwarding misbehavior. The keyboard.dispatch workaround exists but breaks other shortcuts. 21 upvotes and active confirmations through 2026 show continued community impact. Keeping open as a valid, unfixed bug.

## Change summary

AltGr key is misinterpreted as cursor-left when VS Code runs over X11 forwarding or remote desktop to Linux.

## Best solution

The keyboard layout detection in `src/vs/platform/keyboardLayout/` and `src/vs/workbench/services/keybinding/` needs to correctly handle AltGr key events coming through X11 forwarding without requiring the `keyCode` dispatch mode that breaks other shortcuts like Ctrl+/. This likely requires distinguishing X11-forwarded AltGr from a real Left arrow at the native keyboard layout layer.
