## Summary

Tooltips in the file explorer and editor tabs persist after the mouse leaves in Firefox on vscode.dev. The hover system uses standard `mouseleave` events via `CompositeMouseTracker` in `src/vs/platform/hover/browser/hoverWidget.ts`, but contains no Firefox-specific handling. No fix exists on current main. The issue needs proper area routing for investigation.

## Change summary

In the web version of VS Code on Firefox, tooltips on file explorer items and editor tabs do not disappear when the mouse moves away.

## Best solution

Investigate why `mouseleave` events are not firing reliably for hover targets in Firefox's web context. The likely fix is in `CompositeMouseTracker` (`src/vs/platform/hover/browser/hoverWidget.ts`) — possibly adding a fallback dismissal mechanism (e.g. `mouseout` or periodic pointer-position checks) when running in Firefox.
