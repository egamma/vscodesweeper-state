# Issue #149323 — Support for updatable/async menus and submenus

[https://github.com/microsoft/vscode/issues/149323](https://github.com/microsoft/vscode/issues/149323)

## Summary

Team-authored upstream feature request blocked on Electron. The upstream Electron issue (electron/electron#528) was closed as wontfix in Jan 2025, so an alternative approach within VS Code is needed if this capability is still desired. Keeping open for maintainer judgment.

## Change summary

Support dynamically populating or updating native context menu submenus (e.g. Refactor, Source Action) while they are displayed, to improve discoverability of code actions.

## Best solution

With Electron's wontfix on dynamic menu updates, the team should decide whether to pursue a non-native menu approach (custom HTML menus with async loading support) or accept the current two-click UX. The related #238095 QuickPick-proximity exploration is closed and didn't directly address this. If still desired, the fix would likely live in the custom context menu renderer (`src/vs/base/browser/ui/menu/`) rather than Electron's native menus.
