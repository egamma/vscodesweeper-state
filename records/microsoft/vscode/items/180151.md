# Issue #180151 — Git - Git smart commit commits also sub-repositories, which are normally ignored

[https://github.com/microsoft/vscode/issues/180151](https://github.com/microsoft/vscode/issues/180151)

## Summary

Still reproduces on main: smart commit with `smartCommitChanges: 'all'` runs `git add -A -- .` which stages nested sub-repositories as gitlinks, even though the Source Control view never shows them as changes. The discrepancy between what the UI shows and what gets committed is a real bug.

## Change summary

Git smart commit with `smartCommitChanges: all` silently commits nested sub-repositories (as submodule entries) that are not shown as changes in the Source Control view.

## Best solution

When smart-committing with `all`, instead of running a blanket `git add -A -- .`, only add the specific resources that VS Code's working tree group actually shows — i.e., pass the known `workingTreeGroup.resourceStates` paths to `repository.add()` rather than an empty array. This aligns the commit with what the UI displays. The fix would be localized to `extensions/git/src/repository.ts` in the commit method.
