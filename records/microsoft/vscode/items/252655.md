# Issue #252655 — VS Code does not appear in Windows “Default apps” list (ms-settings:defaultapps)

[https://github.com/microsoft/vscode/issues/252655](https://github.com/microsoft/vscode/issues/252655)

## Summary

VS Code's Inno Setup installer (`build/win32/code.iss`) registers per-extension `OpenWithProgids` entries but never creates the `RegisteredApplications` + `Capabilities\FileAssociations` registry keys that Windows requires for an app to appear in `ms-settings:defaultapps`. This is the same root cause as the already-tracked issue https://github.com/microsoft/vscode/issues/191782 (Backlog, labeled `install-update` + `windows`), which covers the installer's failure to properly register file associations — proposing a duplicate close to consolidate.

## Change summary

VS Code (user setup) does not appear in the Windows 'Default apps' settings list, making it impossible to set file associations through the main Default Apps UI.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/191782, which tracks the installer's file-association registration gaps on Windows. The fix for both involves adding `RegisteredApplications` and `Software\<AppName>\Capabilities\FileAssociations` registry entries to `build/win32/code.iss` (for both user-setup and system-wide variants), following the Windows Default Programs API contract.

## Evidence

- **missing RegisteredApplications key:** Searched `build/win32/code.iss` for `RegisteredApplications` and `Capabilities` — neither appears. The installer only sets `OpenWithProgids` per extension (lines 128+) and `App Paths` (line 1305), which is insufficient for `ms-settings:defaultapps` visibility.
- **canonical issue:** https://github.com/microsoft/vscode/issues/191782 ('Visual Studio Code installer fails to register file associations for supported file types') is open in Backlog with labels `bug`, `help wanted`, `install-update`, `windows`. It describes the same underlying installer gap — missing Windows file-association infrastructure — and the commenter in #252655 confirms the same symptom (only 'Just once', no 'Always' option).
- **Windows API requirement:** To appear in Default Apps, an app must register under `HKCU\Software\RegisteredApplications` (user install) or `HKLM\...` (system), pointing to a `Capabilities` key with `FileAssociations`. VS Code's installer does neither.
