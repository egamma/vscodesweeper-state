# Issue #124280 — Provide a smart run/debug command for loose files

[https://github.com/microsoft/vscode/issues/124280](https://github.com/microsoft/vscode/issues/124280)

## Summary

Team-authored feature request; the described run/debug-TS-from-a-loose-file experience has not been implemented — the TypeScript extension still does not contribute a debug configuration provider or editor title run button. Keeping open per hard keep-open rule (MEMBER author) and because the gap remains.

## Change summary

Provide a smart run/debug command for loose TypeScript files so beginners can execute a single .ts file without configuring a project.

## Best solution

The TypeScript language features extension should contribute a DebugConfigurationProvider and an `editor/title/run` menu item for .ts files (transpiling to a tmpdir and forwarding to js-debug). The 2021 discussion converged on this approach but work never landed. A motivated contributor could pick this up following the plan in Comment 6.
