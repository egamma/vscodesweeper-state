The issue reports a very low error rate (0.027%) of decode_failed responses from the auto-intent-service backend, likely caused by edge-case request serialization or mid-flight cancellation in the client. A contributor has already engaged and assigned it for investigation — keeping open as-is.

## Change summary

The auto-intent-service returns HTTP 400 decode_failed for ~257 requests/day when the VS Code Copilot extension sends a malformed or empty JSON body to the router prediction endpoint.
