# Issue #286273 — Offer to open worktree

[https://github.com/microsoft/vscode/issues/286273](https://github.com/microsoft/vscode/issues/286273)

## Summary

Team-authored issue with an 'On Deck' milestone — keeping open. The bug is confirmed in source: the happy-path worktree creation flow in `_createWorktree` never offers to open the new worktree after it's created, unlike the error/conflict handlers which do.

## Change summary

After successfully creating a git worktree, VS Code should offer to open it in the current or a new window instead of silently returning.

## Best solution

After the successful `repository.createWorktree()` call in `_createWorktree` (extensions/git/src/commands.ts ~line 3548), open the new worktree repository via `this.model.openRepository()` and then present the same 'Open Worktree in Current Window / New Window' choice that the conflict handlers already use via `handleWorktreeConflict`. This keeps the UX consistent.
