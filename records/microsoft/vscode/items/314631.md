# Issue #314631 — Repo broke: fatal: bad object refs/sessions/...

[https://github.com/microsoft/vscode/issues/314631](https://github.com/microsoft/vscode/issues/314631)

## Summary

Team-authored issue describing a real bug: refs/sessions/ checkpoint refs created by the Agents/Copilot worktree system are not protected from git prune, leaving dangling refs that break all subsequent git operations. No fix found on main.

## Change summary

Running `git prune` deletes objects referenced by `refs/sessions/` checkpoint refs, breaking git operations with 'fatal: bad object' errors.

## Best solution

The checkpoint service (extensions/copilot/src/extension/chatSessions/vscode-node/chatSessionWorktreeCheckpointServiceImpl.ts) should either ensure checkpoint objects are reachable from a packed/protected ref, or clean up stale session refs before they become dangling. Alternatively, register a git hook or use keep-alive packs for session objects.
