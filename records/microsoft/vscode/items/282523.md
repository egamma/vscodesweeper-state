# Issue #282523 — Wrong error message extraction when Git hooks fail

[https://github.com/microsoft/vscode/issues/282523](https://github.com/microsoft/vscode/issues/282523)

## Summary

Valid bug with source-reproducible root cause. The error message extraction in the default case (lines 5646–5658 of commands.ts) naively picks the first non-empty line from stderr, which for multi-hook scenarios may be an informational line from a passing hook rather than the actual failure. An open PR (#282524) from the reporter already proposes a fix — defer to that.

## Change summary

When Git hooks fail, VS Code shows the first line of hook output as the error, which may be a success message from a passing hook rather than the actual failure reason.

## Best solution

Review and land the open PR https://github.com/microsoft/vscode/pull/282524, which improves the heuristic in the default error-extraction branch at `extensions/git/src/commands.ts` (around line 5646) to prioritize failure-indicating lines over informational ones. The issue is on the On Deck milestone, so it already has maintainer attention.
