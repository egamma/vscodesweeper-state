# Issue #318366 — "Open Recent" / "Select to open" (CTRL+R) shows incorrect results - it doesn't consider the file/dir title

[https://github.com/microsoft/vscode/issues/318366](https://github.com/microsoft/vscode/issues/318366)

## Summary

Valid regression: PR #272651 added `sortByLabel: false` to the Open Recent picker to preserve grouping, but this also disables the `compareEntries` logic that prioritizes label-matches over description-only-matches when filtering — a side effect the reporter correctly identified.

## Change summary

When typing in the Open Recent (Ctrl+R) picker, items that only match on the file path/description appear above items matching on the basename/label, because `sortByLabel: false` disables all match-quality sorting.

## Best solution

In `src/vs/platform/quickinput/browser/quickInputList.ts` (line ~784), decouple the match-quality sorting from the `sortByLabel` flag: when there's a query, still call `compareEntries` to prioritize label-matches over description-only matches, even when `sortByLabel` is `false`. This preserves the grouping fix from #272651 while restoring match-quality ranking during filtering.
