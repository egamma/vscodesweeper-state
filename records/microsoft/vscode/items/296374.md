# Issue #296374 — Feature Request: Add settings to disable "Upload" and "Download" in File Explorer context menu

[https://github.com/microsoft/vscode/issues/296374](https://github.com/microsoft/vscode/issues/296374)

## Summary

Valid feature request for enterprise/DLP use cases — no settings to disable Upload/Download exist on current main. The menu items are registered unconditionally (gated only on web/remote context keys) in fileActions.contribution.ts with no user-configurable toggle. Routing to the explorer area for triage.

## Change summary

Add machine-level settings (e.g. explorer.enableDownload, explorer.enableUpload) to hide the Download and Upload context menu items in the File Explorer, enabling administrators to prevent file transfer in sandboxed remote environments.

## Best solution

Add two boolean settings (`explorer.enableDownload` and `explorer.enableUpload`, defaulting to `true`) in `src/vs/workbench/contrib/files/browser/fileActions.contribution.ts` and gate the existing menu registrations on them via `ContextKeyExpr.has(...)`. These could also be policy-backed for enterprise deployment. The implementation is localized to the menu registration code and the explorer configuration schema.
