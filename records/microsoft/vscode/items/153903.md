# Issue #153903 — Automatically Run task does not wait for postCreateCommand in a Dev Container

[https://github.com/microsoft/vscode/issues/153903](https://github.com/microsoft/vscode/issues/153903)

## Summary

Valid feature request for coordination between Dev Container lifecycle commands and VS Code's auto-run tasks. No implementation exists on main; the tasks system has no awareness of devcontainer lifecycle events. Keeping open — under 20 upvotes, in Backlog, community process applies.

## Change summary

Tasks configured with `runOn: folderOpen` execute before the Dev Container's `postCreateCommand` finishes, causing failures when tasks depend on packages installed by that command.

## Best solution

Introduce an event or barrier in the tasks auto-run system (`src/vs/workbench/contrib/tasks/browser/runAutomaticTasks.ts`) that can wait for a 'container ready' signal from the Dev Containers extension before firing folderOpen tasks. This likely requires a new extension API or a coordination contract between the remote-containers extension and the task runner. Alternatively, the Dev Containers extension could expose a 'ready' event that the task system subscribes to.
