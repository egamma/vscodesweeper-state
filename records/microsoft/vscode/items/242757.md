# Issue #242757 — "Find all" with "occurrences to exclude"

[https://github.com/microsoft/vscode/issues/242757](https://github.com/microsoft/vscode/issues/242757)

## Summary

Valid feature request with 25 upvotes, promoted to Backlog by the community process. Asks for content-level exclusion in search results — a capability that does not exist today (current exclude filters operate at the file level, not the match/line level). No duplicate found. Keeping open.

## Change summary

Add a way to exclude search result matches whose line contains a specified word or phrase, complementing the existing file-level include/exclude filters.

## Best solution

Add an 'occurrences to exclude' (or 'context filter') input to the search widget that filters out result lines matching a secondary pattern. Implementation would likely touch `src/vs/workbench/contrib/search/browser/searchWidget.ts` (UI) and the result-filtering logic in the search view/model. Regex already provides this capability for power users, but a plain-text exclude field would lower the barrier. An alternative is a post-filter on rendered results rather than a query-time filter.
