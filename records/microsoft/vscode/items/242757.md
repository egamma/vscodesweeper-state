# Issue #242757 — "Find all" with "occurrences to exclude"

[https://github.com/microsoft/vscode/issues/242757](https://github.com/microsoft/vscode/issues/242757)

## Summary

Legitimate feature request for content-level filtering of search results (exclude matches containing a word/phrase). It passed the 20-upvote threshold and was promoted to Backlog by the community process — it should stay open.

## Change summary

Add a UI field in the global search view to exclude search results whose matching line contains a specified word or phrase.

## Best solution

Implement a content-level exclusion filter in the search widget (alongside the existing files-to-include/exclude fields) that removes matches whose context line contains the exclusion term. The search widget UI lives in `src/vs/workbench/contrib/search/browser/searchWidget.ts` and the result filtering logic in `src/vs/workbench/contrib/search/browser/searchView.ts`. Regex negative lookahead (`(?!...)`) technically works today but the request is for a discoverable, non-regex UI affordance — a valid UX gap.
