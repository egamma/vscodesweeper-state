# Issue #324145 — Wrong tool used in chat input when multiple MCP servers expose the same tool name

[https://github.com/microsoft/vscode/issues/324145](https://github.com/microsoft/vscode/issues/324145)

## Summary

Confirmed on current main: MCP tool `referenceName` is set to the bare tool name without the server prefix, so two servers exposing the same tool name produce identical `toolReferenceName` values, causing the chat input picker/parser to conflate them.

## Change summary

When two MCP servers expose tools with the same name, the chat input picker inserts an unqualified reference and the parser resolves it to an arbitrary one, ignoring the user's selection.

## Best solution

Include the server prefix in `McpTool.referenceName` (or in the `toolReferenceName` passed to the tools service at registration time in `mcpLanguageModelToolContribution.ts`) so that each tool is uniquely addressable in the chat input. The picker completion and parser already key on `toolReferenceName`, so making it unique would fix resolution. The `id` field already includes the prefix — `referenceName` should too.
