# Issue #207365 — Rendering paused for long line for performance reasons. This can be configured via `editor.stopRenderingLineAfter`.

[https://github.com/microsoft/vscode/issues/207365](https://github.com/microsoft/vscode/issues/207365)

## Summary

Keep open: the report is still actionable against current main because the long-line hover is still triggered from line length and hover column, not from whether rendering is actually paused in the visible wrapped line.

## Change summary

The rendering-paused long-line hover appears with word wrap on when the line is visibly rendered, and does not appear in the non-wrapped case where rendering is actually truncated.

## Best solution

Update the editor hover logic so the `Rendering paused for long line` warning is shown only when the hovered content is actually affected by `editor.stopRenderingLineAfter`, likely in `src/vs/editor/contrib/hover/browser/markdownHoverParticipant.ts`, with coverage for wrapped and unwrapped long lines.

## Evidence

- **current code:** On current main, `src/vs/editor/contrib/hover/browser/markdownHoverParticipant.ts` still sets `stopRenderingMessage` when `lineLength > stopRenderingLineAfter` and `anchor.range.startColumn >= stopRenderingLineAfter`; the condition does not check word wrap or whether the rendered view line is actually truncated.
- **rendering cutoff:** `src/vs/editor/common/viewLayout/viewLineRenderer.ts` applies the `stopRenderingLineAfter` cutoff when rendering line content, so the hover condition should align with the view-rendering state rather than only the model line length.
- **related change is not a fix:** Commit c3e06383df1a12da82c45ba454dc77b999cf5214 added `editor.hover.showLongLineWarning` and a dismiss action for long-line hovers, but it fixed https://github.com/microsoft/vscode/issues/172713 and did not change the reported word-wrap/inversion behavior.
- **issue discussion:** The reporter clarified that changing `editor.stopRenderingLineAfter` only moves where the hover appears with word wrap on, and suggested the hover should appear only when the end of the long line is in view.
