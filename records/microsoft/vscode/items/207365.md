# Issue #207365 — Rendering paused for long line for performance reasons. This can be configured via `editor.stopRenderingLineAfter`.

[https://github.com/microsoft/vscode/issues/207365](https://github.com/microsoft/vscode/issues/207365)

## Summary

Still reproduces on main: the long-line hover warning triggers based solely on column position vs stopRenderingLineAfter, without considering whether word wrap makes the content visible. The recent c3e06383df1 commit added a suppression setting but did not fix the inverted logic.

## Change summary

The 'Rendering paused for long line' hover appears when word wrap is ON (where content is visible) but not when word wrap is OFF (where rendering actually stops).

## Best solution

The condition in `markdownHoverParticipant.ts` line 120 should account for whether word wrap is enabled — when word wrap is on and the line is fully rendered (wrapped), the warning hover should not appear. The check needs to incorporate the editor's wrapping state.
