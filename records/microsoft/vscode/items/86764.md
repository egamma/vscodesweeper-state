# Issue #86764 — [css] hover for property values

[https://github.com/microsoft/vscode/issues/86764](https://github.com/microsoft/vscode/issues/86764)

## Summary

Still unimplemented on current main. The CSS hover service handles property names, at-rules, and pseudo-selectors but does not provide hover information for property values. The work belongs in the upstream vscode-css-languageservice package.

## Change summary

CSS property values (e.g. `block`, `flex`, `auto`) should show documentation on hover, similar to how completion details already shows descriptions for them.

## Best solution

Implement value-level hover in vscode-css-languageservice (cssHover.ts) by looking up the hovered value token against the property's known value entries from the CSS data manager and returning `getEntryDescription` content. The VS Code side just needs to consume the updated service version.
