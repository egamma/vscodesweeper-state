# Issue #86764 — [css] hover for property values

[https://github.com/microsoft/vscode/issues/86764](https://github.com/microsoft/vscode/issues/86764)

## Summary

Valid feature request for CSS property value hover info. The hover logic is delegated to the vscode-css-languageservice package; the data exists (shown in completions) but isn't surfaced on hover. Still unimplemented on main, no upvotes, parked in Backlog.

## Change summary

Hovering a CSS property value (e.g. `absolute` for `position`) shows no documentation, even though completion details already provide it.

## Best solution

Implement property-value hover in the vscode-css-languageservice library (https://github.com/microsoft/vscode-css-languageservice) so that the same MDN descriptions shown in completions are also returned from `doHover` when the cursor is on a value token. No core VS Code change needed beyond a dependency bump.
