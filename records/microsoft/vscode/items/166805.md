# Issue #166805 — [Bug] Diff Editors are not in the back/forward history

[https://github.com/microsoft/vscode/issues/166805](https://github.com/microsoft/vscode/issues/166805)

## Summary

Still valid on current main: diff editors stored as typed EditorInputs in the navigation stack are removed when disposed (tab closed), so back/forward cannot restore them. No fix has been committed.

## Change summary

Diff editors are not preserved in the back/forward navigation history after their tab is closed, because they cannot be serialized to an untyped resource input that survives disposal.

## Best solution

Extend `preferResourceEditorInput` (or the navigation stack entry logic) in `src/vs/workbench/services/history/browser/historyService.ts` to handle `IResourceDiffEditorInput` as a durable, serializable entry — similar to how single-resource editors are converted to untyped inputs that survive disposal. The diff editor's `toUntyped()` already produces a usable `IResourceDiffEditorInput`; the history service just needs to accept and later re-open that shape.
