# Issue #313317 — [Agents] Forking a conversation includes the last sent messages

[https://github.com/microsoft/vscode/issues/313317](https://github.com/microsoft/vscode/issues/313317)

## Summary

Team-authored bug report about conversation forking in the Agents window including the most recent user prompt when it should fork from just before it. Keeping open per team-member policy.

## Change summary

Forking a conversation in the Agents window incorrectly includes the latest user prompt below the fork button instead of forking from the state before that prompt.

## Best solution

Fix the fork logic in the Agents window chat widget to capture conversation state up to (but not including) the user message below the fork point. Likely in the sessions/chat fork handling code under `src/vs/sessions/`.
