# Issue #225747 — Invalid line width

[https://github.com/microsoft/vscode/issues/225747](https://github.com/microsoft/vscode/issues/225747)

## Summary

Font-rendering issue where Unicode card suit symbols break monospace alignment in certain fonts (Noto Sans Mono, Droid Sans Mono). Alex explained this is Chromium's font kerning behavior — effectively upstream. The reporter confirmed it's font-dependent and noted VS Code desktop handles some fonts better than the browser playground.

## Change summary

Lines with equal character counts render at different visual widths when using Unicode card suit symbols in certain fonts.

## Best solution

This is upstream Chromium text-shaping behavior for certain Unicode characters in specific fonts. VS Code could potentially force monospace grid rendering for known-monospace fonts regardless of kerning tables, but the root cause lives in Chromium's text layout engine. Keeping open as an upstream tracking issue; suggest users switch to a stricter monospace font (Consolas, Inconsolata) as a workaround.
