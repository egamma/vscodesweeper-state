# Issue #129571 — Instruction breakpoints can not be disabled

[https://github.com/microsoft/vscode/issues/129571](https://github.com/microsoft/vscode/issues/129571)

## Summary

Implemented on main: the disassembly view now tracks and renders instruction breakpoint enabled/disabled state, responding to changes from the breakpoints view. Shipped in 1.83.0.

## Change summary

Instruction breakpoints in the disassembly view did not reflect their enabled/disabled state when toggled from the breakpoints view.

## Best solution

Keep the shipped implementation from commit 530f01336e2 (1.83.0); no further change is needed. The disassembly view now tracks `isBreakpointEnabled`, handles `bpEvent.changed` to update it, and renders distinct enabled/disabled breakpoint icons.
