# Issue #118627 — Git - Apply encoding to all sides of diff

[https://github.com/microsoft/vscode/issues/118627](https://github.com/microsoft/vscode/issues/118627)

## Summary

Still valid on main: the git diff view's left side (git: scheme) does not inherit the workspace-scoped `files.encoding` setting because `textResourceConfigurationService.getValue` receives a `git:` URI that doesn't match workspace folder configuration scope. No fix has landed. The issue is confirmed by multiple team members and #194986 was closed as a duplicate of this one.

## Change summary

Git diff view does not apply `files.encoding` to the committed (left) side, causing garbled text for non-UTF-8 files like GBK, Shift_JIS, or Windows-1252.

## Best solution

The encoding resolution path in `src/vs/workbench/services/textfile/browser/textFileService.ts` (around `getUnvalidatedEncodingForResource`) needs to map `git:` (and similar SCM-scheme) URIs back to their associated `file:` resource when looking up `files.encoding` from configuration. Alternatively, the git extension's content provider could propagate the associated file's encoding when providing text document content. This is a cross-cutting concern between the git extension and the text file service.
