# Issue #322164 — special paste hotkey conflict within chat input

[https://github.com/microsoft/vscode/issues/322164](https://github.com/microsoft/vscode/issues/322164)

## Summary

Team-authored bug, keep open. The agent picker (OpenModePickerAction) and the special paste dropdown (editor.changePasteType) both bind Ctrl+. at the same keybinding weight inside the chat input editor, causing the agent picker to intercept the keystroke when the paste widget is visible.

## Change summary

Pressing Ctrl+. in the chat input after pasting opens the agent picker instead of the special paste dropdown.

## Best solution

Resolve the keybinding conflict so that `editor.changePasteType` wins when `pasteWidgetVisibleCtx` is true inside the chat input. The simplest fix is to add `pasteWidgetVisibleCtx.negate()` to the agent picker's `when` clause in `src/vs/workbench/contrib/chat/browser/actions/chatExecuteActions.ts` (around line 515), or raise the paste widget command's weight above `EditorContrib` when the paste widget is visible.
