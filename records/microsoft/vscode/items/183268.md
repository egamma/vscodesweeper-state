# Issue #183268 — Some Mac keyboard shortcuts does not work in the new Terminal Editor after being moved with `workbench.action.moveEditorTo<Side>Group` until the terminal editor has been unfocused and focused again

[https://github.com/microsoft/vscode/issues/183268](https://github.com/microsoft/vscode/issues/183268)

## Summary

Fixed on main by commit 4bc47d4316e which updates the terminal instance's parent context key service when the editor is moved to a new group, so keybindings evaluate correctly without needing a re-focus cycle.

## Change summary

Mac keyboard shortcuts stop working in terminal editors after moving them to a new editor group until unfocused and refocused.

## Best solution

Keep the shipped fix (commit 4bc47d4316e, 1.86.0); no further change needed. The terminal editor now updates its context key service parent on group change.
