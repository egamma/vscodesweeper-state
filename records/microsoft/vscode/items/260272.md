# Issue #260272 — When deleting a file the file explorer resets vertical position and goes to bottom

[https://github.com/microsoft/vscode/issues/260272](https://github.com/microsoft/vscode/issues/260272)

## Summary

Still reproduces on main: deleting a file triggers an active-editor change, which fires `selectActiveFile()` with `autoReveal: true`, scrolling the explorer away from the deletion site to wherever the now-active editor's file sits. The delete handler in `explorerService.ts` already calls `focusNext()` to keep the view near the deleted item, but the subsequent `autoReveal` scroll overrides it.

## Change summary

Deleting a file in the explorer causes the tree to scroll away to the active editor's file, losing the user's position near the deleted item.

## Best solution

After a delete operation completes in `explorerService.ts`, temporarily suppress `autoReveal` in the explorer view so the `onDidActiveEditorChange` handler does not scroll the tree. The fix would likely be in `explorerService.ts` (around the DELETE handler at line 422) coordinating with `explorerView.ts` (around the `selectActiveFile` handler at line 318). A workaround exists: setting `explorer.autoReveal` to `focusNoScroll`.
