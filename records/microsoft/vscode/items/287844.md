# Issue #287844 — Copilot Agent create_file fails on NTFS data drive (D:) but works on C:\ and Dev Drive (ReFS)

[https://github.com/microsoft/vscode/issues/287844](https://github.com/microsoft/vscode/issues/287844)

## Summary

Team-authored issue (MEMBER) — must stay open. The bug lives in the Copilot extension's content-exclusion/ignore logic (`extensions/copilot/src/platform/ignore/`), where `RemoteContentExclusion` fetches server-side exclusion rules based on git remote URLs. When the workspace has a remote pointing to an internal repo with content-exclusion policies, those policies incorrectly block local file operations even on unrelated drives. The reporter confirmed removing the git remote fixed the issue, and rolling back Copilot Chat to v0.33.5 also resolved it, pointing to a regression in the ignore service. A second user (comment 4-5) reports the same symptom in a different context.

## Change summary

Copilot Agent's create_file and readFile tools incorrectly reject files as 'configured to be ignored by Copilot' when the workspace git remote has content-exclusion policies, even for files outside the excluded scope.

## Best solution

Investigate the `RemoteContentExclusion` logic in `extensions/copilot/src/platform/ignore/node/remoteContentExclusion.ts` — the glob/minimatch matching of content-exclusion rules fetched from the GitHub API appears too broad, catching files that should not be excluded. The fix likely involves tightening how exclusion patterns are scoped to the repository root, so files on other drives or outside the repo are not matched. The `assertFileNotContentExcluded` call in `extensions/copilot/src/extension/tools/node/toolUtils.ts:275` is the gate that surfaces the error.
