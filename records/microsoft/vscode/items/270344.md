# Issue #270344 — "Compare selected" on two fairly small text files never completes computing a comparison

[https://github.com/microsoft/vscode/issues/270344](https://github.com/microsoft/vscode/issues/270344)

## Summary

Still reproducible on main: character-level diff refinement via Myers algorithm on very long lines (100KB+) with `maxComputationTime: 0` (infinite timeout) effectively never completes, freezing the diff editor and bricking SCM.

## Change summary

Comparing files with very long lines (100KB+) with infinite computation timeout hangs VS Code's diff editor indefinitely.

## Best solution

Guard `refineDiff` against extremely long character sequences — either cap the per-line character diff length (bail out to a whole-line diff when `slice1.length + slice2.length` exceeds a threshold, regardless of timeout setting), or propagate the newly adopted `@vscode/diff` WASM computer as the default which likely handles this more gracefully. The fix belongs in `src/vs/editor/common/diff/defaultLinesDiffComputer/defaultLinesDiffComputer.ts` around the `refineDiff` method.
