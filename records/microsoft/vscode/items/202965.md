# Issue #202965 — Fails to launch on Ubuntu 24.04+ Linux kernel 6.6+

[https://github.com/microsoft/vscode/issues/202965](https://github.com/microsoft/vscode/issues/202965)

## Summary

Upstream issue — the root cause is Electron/Chromium's use of unprivileged user namespaces combined with Ubuntu 24.04+'s AppArmor restriction. The upstream Electron issue (electron/electron#41066) is now closed, with the fix handled via Electron Builder/Forge shipping AppArmor profiles in deb packages. The VS Code-specific concern (the .desktop file pointing to `/usr/share/code/code` rather than the wrapper script covered by the AppArmor profile) may still be relevant for the VS Code deb packaging.

## Change summary

VS Code fails to launch on Ubuntu 24.04+ because the default AppArmor profile restricts unprivileged user namespaces, and the shipped AppArmor profile only covers the wrapper script path, not the binary path used in the .desktop file.

## Best solution

Verify that the current VS Code deb package's AppArmor profile covers both `/usr/share/code/bin/code` and `/usr/share/code/code` (the path used in the .desktop file). If the upstream Electron Builder fix has been incorporated and the profile now covers both paths, this can be closed as resolved. Otherwise, update the deb packaging to align the .desktop Exec path with the AppArmor profile or extend the profile.
