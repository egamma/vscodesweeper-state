# Issue #319000 — [Error] [GitHub.copilot-chat] unhandlederror-command 'github.copilot.chat.sendCompletionFeedback' already exists  Error: command '...

[https://github.com/microsoft/vscode/issues/319000](https://github.com/microsoft/vscode/issues/319000)

## Summary

Confirmed re-entrancy bug in getOrCreateProvider() — the _provider field isn't assigned until after construction completes, so re-entrant autorun evaluations triggered during provider instantiation cause duplicate command registration. An open fix PR exists but hasn't merged yet.

## Change summary

Authentication change events can trigger re-entrant getOrCreateProvider() calls that register the 'sendCompletionFeedback' command twice, throwing an error.

## Best solution

Merge the open fix PR (https://github.com/microsoft/vscode/pull/319016) which adds re-entrancy protection to getOrCreateProvider(), or apply an equivalent guard (set a sentinel before instantiation) in extensions/copilot/src/extension/completions/vscode-node/copilotInlineCompletionItemProviderService.ts.
