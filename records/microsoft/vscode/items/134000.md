# Issue #134000 — `activeItems` not preserved even if keepScrollPosition = true

[https://github.com/microsoft/vscode/issues/134000](https://github.com/microsoft/vscode/issues/134000)

## Summary

Still reproduces on main: when `keepScrollPosition` is true and items are updated, scroll position is restored but the focused/active item is reset to the first item. The `update()` method's `itemsUpdated` block unconditionally calls `trySelectFirst()` without considering `keepScrollPosition`.

## Change summary

QuickPick with `keepScrollPosition = true` loses the focused/active item when `.items` is reassigned, even though scroll position is preserved.

## Best solution

In the `update()` method's `itemsUpdated` block, when `keepScrollPosition` is true and `_itemActivation` is `FIRST` (default), save the currently focused item(s) before `setElements()` and restore them afterward (matching by identity or a stable ID), mirroring how trees preserve focus across data refreshes.
