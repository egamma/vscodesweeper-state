# Issue #321551 — Bug: Prompt cache expires during active agent sessions, causing 10x cost spikes on gaps > 5 min

[https://github.com/microsoft/vscode/issues/321551](https://github.com/microsoft/vscode/issues/321551)

## Summary

This is a feature request for cache-keepalive mitigation and cache-status UI, not a bug — the prompt-cache TTL is set by the model provider (Anthropic ~5 min) and is working as designed at that layer. A team member confirmed 'some of this is expected.' Related upstream work (cacheExpiresAt tracking in copilot-agent-runtime) is already merged, and the issue has team assignees actively managing it.

## Change summary

Request for VS Code Copilot to send keepalive pings to prevent model-provider prompt-cache expiration during active agent sessions, and to surface cache-status indicators in the chat UI.

## Best solution

The keepalive-ping mechanism and cache-status UI belong in the Copilot extension and/or copilot-agent-runtime — the upstream PR (copilot-agent-runtime#12632 exposing `cacheExpiresAt`) is a prerequisite that has already merged. The next step is for the assigned team members to decide whether a keepalive strategy is cost-effective at scale and whether cache-status indicators fit the chat UX. The `/compact` workaround the reporter discovered is a reasonable interim mitigation.
