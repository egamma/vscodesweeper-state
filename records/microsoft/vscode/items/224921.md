# Issue #224921 — Vimium or Vim easymotion like keyboard navigation for the whole UI

[https://github.com/microsoft/vscode/issues/224921](https://github.com/microsoft/vscode/issues/224921)

## Summary

Legitimate community-promoted feature request (35 upvotes, already in Backlog) for Vimium/easymotion-style keyboard hint overlays across the entire VS Code UI. No duplicate exists; the capability gap remains on current main.

## Change summary

Add overlay-based keyboard navigation hints (like Vimium's link hints) for the entire VS Code UI so users can click any element without a mouse.

## Best solution

This is a substantial UX feature that would benefit from design exploration. It could be pursued as a core accessibility contribution (since it improves mouseless workflows) or as an extension-API surface that lets extensions provide hint overlays. The most practical near-term path may be ensuring all UI elements are keyboard-reachable via existing focus management and documenting gaps, while a hint-overlay system could be prototyped as an extension first.
