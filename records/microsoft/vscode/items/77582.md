# Issue #77582 — [css][html] Publish language servers modules on npm

[https://github.com/microsoft/vscode/issues/77582](https://github.com/microsoft/vscode/issues/77582)

## Summary

The request to publish CSS and HTML language servers as standalone npm packages remains unaddressed — only the language *services* (vscode-css-languageservice, vscode-html-languageservice) are published, not the full LSP-speaking server wrappers. The package.json files exist in the repo with names like `vscode-css-languageserver` and `vscode-html-languageserver` but are not published to npm by Microsoft.

## Change summary

Publish the CSS and HTML language server modules (the LSP-speaking wrappers, not just the language services) to npm for easier reuse by third-party editors and tools.

## Best solution

Publish `vscode-css-languageserver` and `vscode-html-languageserver` packages to npm (mirroring what was done for `vscode-json-languageserver`), or document that consumers should use the language service packages directly with their own LSP transport layer.
