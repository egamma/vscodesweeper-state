# Issue #263827 — Make installing a specific version disable auto update until update is clicked

[https://github.com/microsoft/vscode/issues/263827](https://github.com/microsoft/vscode/issues/263827)

## Summary

Team-authored feature request for extension auto-update behavior — keeping open per policy. The request is clear and actionable: when a user manually installs a specific extension version, auto-update should be suppressed until the user explicitly clicks update.

## Change summary

When a user installs a specific version of an extension, auto-update should be disabled for that extension until the user manually triggers an update.

## Best solution

Implement a per-extension pinned-version state in the extension management service (likely in `src/vs/workbench/contrib/extensions/browser/extensionsWorkbenchService.ts` or the gallery service) that suppresses auto-update when the user explicitly installed a specific version, and clears that pin when the user clicks update.
