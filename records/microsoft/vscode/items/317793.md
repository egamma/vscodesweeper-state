## Summary

Real regression in 1.121.0: deserialized chat model state can have undefined `isDefaultForLocation`, causing a TypeError in `setCurrentLanguageModel`. An open PR (#317802) addresses this; keeping open pending its review and merge.

## Change summary

Chat widget crashes with 'Cannot read properties of undefined (reading panel)' when restoring a session persisted before the `isDefaultForLocation` field was added to model metadata.

## Best solution

Review and land the open fix PR (https://github.com/microsoft/vscode/pull/317802), which adds a `?? {}` fallback during deserialization in `chatModel.ts` — the producer-side fix. The existing migration in `chatInputPart.ts:299` covers the untitled-input storage path but the chat-model deserialization path was missed.
