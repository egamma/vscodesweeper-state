# Issue #247340 — Some active breakpoints can be removed only from the Breakpoints menu

[https://github.com/microsoft/vscode/issues/247340](https://github.com/microsoft/vscode/issues/247340)

## Summary

Still present on main: the CSS `::after` pseudo-element rules that show a breakpoint behind the stack-frame arrow only cover `breakpoint.regular` — conditional, log, data, and function breakpoints are not handled, so their glyph disappears when the current instruction pointer lands on the same line.

## Change summary

Breakpoints of non-regular types (conditional, logpoint, data, function) lose their gutter glyph when the debugger's instruction pointer (stack frame) is on the same line, making them invisible and only removable from the Breakpoints panel.

## Best solution

Extend the CSS `::after` rules in `breakpointEditorContribution.ts` (around lines 816–817) to cover all breakpoint types in `allBreakpoints`, not just `breakpoint.regular`. Each type's `.regular` icon combined with `debugStackframe` and `debugStackframeFocused` needs its own `::after` rule so the breakpoint indicator remains visible behind the stack-frame arrow.
