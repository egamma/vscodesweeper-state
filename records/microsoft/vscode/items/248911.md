# Issue #248911 — UI Flicker Observed During Image Rendering in VS Code Copilot Extension

[https://github.com/microsoft/vscode/issues/248911](https://github.com/microsoft/vscode/issues/248911)

## Summary

Still reproducible on main: the streaming markdown repair logic (`fillInIncompleteTokens`) handles backticks, bold, italic, links, and tables but has no guard for incomplete HTML tags like `<img src="data:...`. Partial HTML is rendered mid-stream causing visible flicker. The team has acknowledged the problem (roblourens comment) but no fix has landed.

## Change summary

Streaming markdown containing HTML image tags (e.g. base64 `<img>`) flickers during progressive rendering because partial HTML is rendered before the tag is complete.

## Best solution

Extend `fillInIncompleteTokens` (or add a pre-render guard in `ChatContentMarkdownRenderer`) to detect and suppress rendering of incomplete HTML tags — specifically an unterminated `<img` or similar self-closing tag. The backtick-flicker fix (commit 2e9dc7f2fd2) and link-repair fix (commit 07c97749857) show the established pattern for this kind of streaming repair.
