# Issue #182679 — GitHub - improving git error message for fatal error in commit_refs

[https://github.com/microsoft/vscode/issues/182679](https://github.com/microsoft/vscode/issues/182679)

## Summary

Still reproduces on main: the push error handler in commands.ts unconditionally shows 'Try running Pull first' for all PushRejected errors, including remote server failures like 'fatal error in commit_refs'. The issue is valid and has an open PR (#301115) attempting to fix it.

## Change summary

When git push fails due to a remote server error ('fatal error in commit_refs'), VS Code misleadingly suggests running Pull, when the actual cause is a server-side failure.

## Best solution

Review and land the open PR https://github.com/microsoft/vscode/pull/301115, which adds detection for 'fatal error in commit_refs' in the push stderr and shows a more appropriate error message pointing to the remote service status. The fix touches `extensions/git/src/git.ts`, `extensions/git/src/commands.ts`, and `extensions/git/src/api/git.d.ts`.
