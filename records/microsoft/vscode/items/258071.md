# Issue #258071 — "m" appears when trying to run a command while another is in progress in powershell

[https://github.com/microsoft/vscode/issues/258071](https://github.com/microsoft/vscode/issues/258071)

## Summary

Known papercut bug filed by the terminal maintainer — the stray 'm' likely comes from an incomplete ANSI escape sequence (SGR reset `\e[m`) being printed as literal text when input races with command execution. No fix found on main; keeping open.

## Change summary

Typing a command (including Enter) while another command is still running in PowerShell causes a literal 'm' to appear at the end of the line instead of executing the new command.

## Best solution

Investigate the race between shell integration prompt rendering and user input in the PowerShell shell integration script (`shellIntegration.ps1`) or terminal input handling — the 'm' is almost certainly the trailing character of an SGR reset sequence (`\e[0m` or `\e[m`) that gets split when input arrives mid-sequence. Fix likely lives in `src/vs/workbench/contrib/terminal/common/scripts/shellIntegration.ps1` or the terminal's input/sequence handling layer.
