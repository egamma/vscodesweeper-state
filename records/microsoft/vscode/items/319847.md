## Change summary

Request to allow different color themes per panel (Explorer, Chat, Editor) instead of a single global workbench.colorTheme.

## Best solution

This is architecturally very complex — VS Code's theming applies CSS custom properties at the workbench root. The narrower request in https://github.com/microsoft/vscode/issues/25986 (separating workbench from syntax theme, 193 upvotes) is the more tractable canonical ask. Per-panel scoping would require CSS variable isolation per view container, which is a major architectural change. If community traction grows, it could build on top of #25986's separation work.
