# Issue #154417 — update the language pack and then click reload then the shown language is changed to English

[https://github.com/microsoft/vscode/issues/154417](https://github.com/microsoft/vscode/issues/154417)

## Summary

Still valid on current main: the localization contribution only handles `InstallOperation.Install` (not `Update`), so when a language pack updates and the user reloads, the language pack cache may not reflect the updated extension paths, causing a fallback to English. Multiple users have confirmed this across versions.

## Change summary

After updating a language pack extension and clicking reload, the display language reverts to English even though settings still report the non-English locale.

## Best solution

Fix the race between extension update, language pack cache refresh, and window reload. The `onDidInstallExtensions` handler in `src/vs/workbench/contrib/localization/electron-browser/localization.contribution.ts` filters on `InstallOperation.Install` only (line 45); the `postInstallExtension` participant in `src/vs/platform/languagePacks/node/languagePacks.ts` does run on updates, but the cache may not be flushed before the reload occurs. Ensure the language pack cache is synchronously updated before reload completes.
