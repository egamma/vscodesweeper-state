# Issue #288860 — .copilotignore pattern **/copilot-instructions.md not working in multi-folder workspaces - All repository attachments sent despite ignore rules

[https://github.com/microsoft/vscode/issues/288860](https://github.com/microsoft/vscode/issues/288860)

## Summary

The `.copilotignore` implementation has a real multi-folder workspace limitation: `addWorkspace` discovers `.copilotignore` files per workspace folder, and `isIgnored` scopes pattern matching to descendants of the directory containing the ignore file — so a `.copilotignore` in one workspace folder cannot affect files in sibling workspace folders. Confirmed by two independent commenters and traceable in the source. Keeping open as a real bug in the Copilot extension's ignore service.

## Change summary

`.copilotignore` patterns are not applied correctly in multi-folder workspaces — ignore rules from one workspace folder do not affect files in sibling folders, and repository metadata attachments are not filtered by `.copilotignore` at all.

## Best solution

Fix the ignore service in `extensions/copilot/src/platform/ignore/` to properly resolve `.copilotignore` patterns across all folders in a multi-folder workspace. The `isIgnored` method in `ignoreFile.ts` skips files outside the `.copilotignore` directory's subtree (`rel.startsWith('..')` guard at line 107), which is correct for single-folder but breaks cross-folder patterns in multi-root workspaces. This is closely related to https://github.com/microsoft/vscode/issues/260344 (making `.copilotignore` official) but is a distinct bug worth tracking separately.
