# Issue #70136 — Git - Pull and Sync  fails when name of tracking branch differs from name of tracked branch on remote

[https://github.com/microsoft/vscode/issues/70136](https://github.com/microsoft/vscode/issues/70136)

## Summary

Fixed in 1.67.0: the upstream branch name is now resolved from %(upstream:remoteref) instead of the tracking branch short name, so pull/sync correctly references the remote branch even when its path differs from the local tracking branch name.

## Change summary

Git pull and sync fail with 'couldn't find remote ref' when the local tracking branch name differs from the actual remote branch path.

## Best solution

Keep the shipped fix (commit cb1851d2dc7, VS Code 1.67.0); no further change needed. The upstream.name field now correctly uses %(upstream:remoteref) to get the real remote branch name.
