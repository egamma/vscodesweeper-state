# Issue #275754 — Deleting a tab character using backspace doesn't work.

[https://github.com/microsoft/vscode/issues/275754](https://github.com/microsoft/vscode/issues/275754)

## Summary

Still reproduces on main: when tabSize > indentSize and a tab character is in the indentation, the backspace unindent logic computes a zero-width deletion range because `columnFromVisibleColumn` rounds back to the same column. PR #300059 is open to fix this.

## Change summary

Pressing Backspace on a tab character in indentation does nothing when tabSize is larger than indentSize (e.g. tabSize=8, indentSize=2).

## Best solution

Merge or iterate on https://github.com/microsoft/vscode/pull/300059 which reworks the backspace-in-indentation path to handle the tabSize ≠ indentSize case correctly.
