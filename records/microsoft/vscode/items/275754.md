# Issue #275754 — Deleting a tab character using backspace doesn't work.

[https://github.com/microsoft/vscode/issues/275754](https://github.com/microsoft/vscode/issues/275754)

## Summary

Still reproduces on main: backspace in indentation with a literal tab character fails when indentSize < tabSize because the unindent logic computes a zero-width deletion range. PR #300059 addresses this but has not been merged.

## Change summary

Pressing Backspace on a tab character in indentation does nothing when indentSize (2) is less than tabSize (8).

## Best solution

Merge or iterate on PR https://github.com/microsoft/vscode/pull/300059 which treats backspace in indentation as an unshift, ensuring literal tab characters are properly deleted even when indentSize < tabSize.
