# Issue #275754 — Deleting a tab character using backspace doesn't work.

[https://github.com/microsoft/vscode/issues/275754](https://github.com/microsoft/vscode/issues/275754)

## Summary

Keep open: the report has concrete settings and the current delete-left code still has a plausible empty-range path for a real tab when `indentSize` is smaller than `tabSize`. There is not enough evidence for `*not-reproducible`, especially after the reporter supplied a second video.

## Change summary

Backspace can fail to delete an actual tab character when indentation uses 2 spaces but tab display size is 8.

## Best solution

Add a regression test in `src/vs/editor/test/browser/controller/cursor.test.ts` for a lone `\t` with `tabSize: 8`, `indentSize: 2`, `insertSpaces: true`, and `useTabStops: true`, then adjust `DeleteOperations.getDeleteLeftRange` so the computed range deletes the tab character instead of collapsing to the cursor position.

## Evidence

- **reported reproduction:** The issue gives specific settings: Indent Using Spaces = 2, Change Tab Display Size = 8, then Backspace before/after a literal tab does not remove it; Delete and Ctrl+Backspace still work.
- **reporter follow-up:** After a member said they could not reproduce in latest code/Insiders, the reporter replied on 2025-11-17 that they still saw the error and attached another video.
- **current code path:** `src/vs/editor/common/cursor/cursorDeleteOperations.ts` computes delete-left inside indentation using `CursorColumns.prevIndentTabStop(fromVisibleColumn, config.indentSize)` and then maps that visible column back with `config.columnFromVisibleColumn`.
- **plausible root cause:** For a line containing only `\t`, cursor column 2, `tabSize: 8`, and `indentSize: 2`, the visible column is 8 and the previous indent stop is 6; `src/vs/editor/common/core/cursorColumns.ts` maps visible column 6 back to column 2 because it is closer to the end of the rendered tab, yielding an empty delete range.
