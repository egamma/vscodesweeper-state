# Issue #174008 — Task problemMatcher with fileLocation method set to "search" only seems to work for background task

[https://github.com/microsoft/vscode/issues/174008](https://github.com/microsoft/vscode/issues/174008)

## Summary

Still reproduces on main: the `done()` method in `AbstractProblemCollector` does not await `this.tail` (the async processing chain), so when `search` fileLocation triggers async file-system lookups, markers are reported before resolution completes. Background tasks work because `WatchingProblemCollector` delivers markers on `endsPattern` matches after lines are individually processed.

## Change summary

Task problemMatcher with `fileLocation: "search"` fails to produce diagnostics for non-background (start/stop) tasks because marker reporting fires before async file search resolves.

## Best solution

Make `done()` await `this.tail` before calling `reportMarkers()` — either by making `done()` async or by chaining `reportMarkers` onto the tail promise. The call site in `terminalTaskSystem.ts` (line ~1129) would need to await the result. This is a small, localized fix in the problem collector lifecycle.
