## Summary

Still valid: the Quick Pick uses HTML input elements (via FindInput in quickInputBox.ts) and Windows 11's native text suggestions cannot be suppressed with known HTML attributes. The team acknowledged the issue and labeled it help-wanted; no fix has landed on main.

Windows 11 native text suggestion overlay obscures VS Code's command palette results when typing.

## Best solution

Either find a Windows API/attribute that suppresses OS-level text suggestions on input elements (upstream Windows fix), or migrate the Quick Pick input to a Monaco editor-based textarea (as TylerLeonhardt suggested), which doesn't trigger the Windows suggestions. The `help wanted` label is appropriate since this requires platform-specific investigation.
