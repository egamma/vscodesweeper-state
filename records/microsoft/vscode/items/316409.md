# Issue #316409 — Unneeded warnings

[https://github.com/microsoft/vscode/issues/316409](https://github.com/microsoft/vscode/issues/316409)

## Summary

The quota/rate-limit notifications are intentional and only fire at specific thresholds (50%, 75%, 90%, 95%) — not every 1% as reported. They are dismissible and auto-dismiss on the next message. This is working as designed.

## Change summary

User finds Copilot quota usage notifications annoying and wants them reduced or removed.

## Best solution

Keep the current notification behavior: notifications fire only at defined thresholds (50, 75, 90, 95%), are dismissible, and auto-dismiss on message send. A setting to suppress them could be considered as a separate feature request if demand grows.

## Evidence

- **threshold design:** src/vs/workbench/contrib/chat/browser/chatQuotaNotification.ts line 28 defines `const THRESHOLDS = [50, 75, 90, 95]` — notifications only fire when crossing these specific thresholds, not every 1%.
- **crossing detection:** The `_findCrossedThreshold` method (line 385-396) only returns a threshold when `previous < threshold && current >= threshold`, preventing repeated notifications at the same level.
- **dismissible:** All quota notifications set `dismissible: true` and `autoDismissOnMessage: true` (e.g. line 488-489), meaning they disappear on dismiss or on the next sent message.
- **rate limit same logic:** Session and weekly rate limits use the same threshold-crossing logic via `_checkRateLimitCrossing` (line 513-528), so they also only notify at 50/75/90/95%.
