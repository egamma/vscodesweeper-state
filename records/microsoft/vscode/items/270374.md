# Issue #270374 — Linkified path in Test Results panel converts to windows path in WSL

[https://github.com/microsoft/vscode/issues/270374](https://github.com/microsoft/vscode/issues/270374)

## Summary

Still reproduces on main: `TerminalMessagePeek.updateCwd()` sets `terminalCwd` to `wf.uri.fsPath`, which yields a Windows UNC path in WSL remoting. The detached terminal's link detection then resolves relative paths against this invalid CWD, producing broken Windows-style paths when clicked.

## Change summary

Clicking a linkified path in the Test Results panel while using WSL remoting opens the command palette with a Windows-style path instead of the correct Linux path.

## Best solution

In `TerminalMessagePeek.updateCwd()`, use the workspace folder's URI path (`wf.uri.path`) instead of `fsPath` when connected to a remote (WSL), mirroring the approach used for terminal CWD detection. The `processInfo.initialCwd` should similarly use the remote-aware path.
