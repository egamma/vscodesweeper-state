# Issue #263309 — Git - When cloning repo should specify that password can be a token

[https://github.com/microsoft/vscode/issues/263309](https://github.com/microsoft/vscode/issues/263309)

## Summary

Valid UX polish request: when Git's HTTPS askpass prompts for a password during clone (or other operations), the input box could hint that many hosts now require a personal access token rather than an account password. The current code at `extensions/git/src/askpass.ts:107-114` passes through Git's raw prompt with no additional guidance. No fix or duplicate found on main; keeping open.

## Change summary

Improve the Git HTTPS password prompt in VS Code to hint that a personal access token (PAT) may be required instead of an account password.

## Best solution

Add a brief hint to the password input box prompt in `extensions/git/src/askpass.ts` (around line 107–114) — e.g. updating the `prompt` or adding a description like "Some hosts require a personal access token instead of a password." This is a small, self-contained UX improvement in the built-in git extension. The current `bug` label should be changed to `feature-request` or `polish` since the behavior is not broken — it's a clarity improvement.
