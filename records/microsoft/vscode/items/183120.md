# Issue #183120 — Debugging indicators can mess with tab spacing

[https://github.com/microsoft/vscode/issues/183120](https://github.com/microsoft/vscode/issues/183120)

## Summary

Still reproduces on main: the debug column indicator (inline `before` decoration with content '\uEB8B') shifts tab stop alignment because it introduces a foreign element that offsets the visual column calculation without compensating for tab width.

## Change summary

When debugging stops on a line with leading tabs, the debug arrow indicator shifts the tab's rendered width by one character, breaking visual alignment.

## Best solution

The line renderer in `viewLineRenderer.ts` needs to exclude the width of `before` foreign elements from the tab stop calculation, or the debug decoration in `callStackEditorContribution.ts` needs to use a different approach (e.g., absolute-positioned overlay) that doesn't affect inline flow.
