# Issue #321556 — "Relaunch Terminal" and other actions are hidden when notification message is too long

[https://github.com/microsoft/vscode/issues/321556](https://github.com/microsoft/vscode/issues/321556)

## Summary

Valid bug: when an expanded notification toast has a very long message, the details row (containing action buttons like 'Relaunch Terminal') is pushed below the visible viewport because there's no max-height or scroll constraint on the toast content.

## Change summary

Notification toast action buttons become inaccessible when the notification message is very long and overflows the available viewport height.

## Best solution

Cap the expanded message height within a toast notification so it doesn't exceed the available viewport, and add a scrollbar to the message region (or truncate with an expand-in-center affordance). The fix likely belongs in `notificationsViewer.ts` (`getHeight`) and/or the toast CSS, ensuring the details row with action buttons is always reachable.
