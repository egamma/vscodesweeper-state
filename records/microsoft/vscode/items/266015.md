# Issue #266015 — Pre-commit Problems detection comes up with false positives

[https://github.com/microsoft/vscode/issues/266015](https://github.com/microsoft/vscode/issues/266015)

## Summary

Still reproduces on main — the diagnostics commit hook reads stale diagnostics because it doesn't wait for the async diagnostic update after a file is saved and staged. Author is a team member, so this is a hard keep-open. The attempted Copilot-agent fix PR was closed without merging; a maintainer added telemetry to investigate further but the underlying race remains.

## Change summary

The git diagnostics commit hook shows false-positive warnings when a user fixes a problem, stages, and commits quickly — stale diagnostics from before the fix are still returned by `languages.getDiagnostics()`.

## Best solution

Add a short delay or wait for a diagnostics-changed event in `evaluateDiagnosticsCommitHook` (extensions/git/src/commands.ts) before reading diagnostics for open documents, so that the debounced diagnostic pipeline has time to flush. The telemetry added in commit 24dc95ef647 may provide data to inform the right approach.
