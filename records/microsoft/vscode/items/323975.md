# Issue #323975 — shellIntegration.ps1 Script digital signatures do not match file hashes

[https://github.com/microsoft/vscode/issues/323975](https://github.com/microsoft/vscode/issues/323975)

## Summary

The report describes a known Windows behavior where ZIP-extracted (portable mode) files inherit Zone.Identifier marks, causing PowerShell signature verification to fail. The script IS signed since #174021, and the try/catch wrapper silently handles this. The report lacks detail on execution policy settings and whether unblocking the ZIP before extraction was attempted — requesting more info.

## Change summary

Shell integration fails to activate in portable mode because the ZIP-extracted shellIntegration.ps1 inherits a Windows Zone.Identifier, causing PowerShell signature verification to fail despite the file being signed.

## Best solution

This is a known limitation of Windows Zone.Identifier inheritance on ZIP-extracted files, discussed extensively in https://github.com/microsoft/vscode/issues/174021. The workaround is to unblock the ZIP file before extraction (right-click → Properties → Unblock) or use Unblock-File. If the team wants to go further, they could explore stripping Zone.Identifier marks during extraction or adding guidance to the portable mode docs.
