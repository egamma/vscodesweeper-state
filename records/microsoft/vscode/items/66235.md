# Issue #66235 — Allow more powerful onEnterRules for cursor alignment

[https://github.com/microsoft/vscode/issues/66235](https://github.com/microsoft/vscode/issues/66235)

## Summary

Team-authored feature request with 48 upvotes (well above the 20-upvote Backlog threshold) requesting more powerful onEnterRules for variable-length cursor alignment. Still unimplemented on main — no related fix found.

## Change summary

Allow onEnterRules to produce variable-length indentation (e.g. aligning to opening parenthesis) rather than only static appendText or fixed indent multiples.

## Best solution

Implement capture-group support in onEnterRules (as sketched in https://github.com/microsoft/vscode/issues/17281 and comment #3 here) so that `appendText` can reference regex captures with transformations like `${1/./ /g}`. This keeps the rules declarative/serializable while enabling the Python alignment use case. Alternatively, a dedicated indentation provider API (discussed in comment #9) would be more powerful but introduces async latency on Enter.
