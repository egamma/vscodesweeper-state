Team-authored feature request for a telemetry metric capturing when chat response rendering finishes; not yet implemented on main. Keeping open per hard keep-open rule (MEMBER author).

## Change summary

Add a telemetry metric that records when the chat response finishes rendering to the user, distinct from when the response data is fully received.

## Best solution

Instrument the progressive rendering pipeline in `chatListRenderer.ts` to emit a telemetry event (or extend the existing Copilot chat response measures bag) when all content parts have finished rendering — likely where `doNextProgressiveRender` returns `true` with `element.isComplete`. This would capture the gap between response-complete and render-complete that the author observed.
