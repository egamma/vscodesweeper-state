# Issue #189285 — Copy/Paste files in Explorer pastes in wrong folder

[https://github.com/microsoft/vscode/issues/189285](https://github.com/microsoft/vscode/issues/189285)

## Summary

This report describes a specific instance of the broader explorer context-menu targeting inconsistency tracked by the open canonical issue https://github.com/microsoft/vscode/issues/323488. When multiple files are selected and the user right-clicks a different folder to paste, the context menu command target depends on selection size rather than the clicked item — the same root cause. Proposing duplicate close.

## Change summary

Pasting files via the Explorer context menu targets the original folder (of the selected files) instead of the right-clicked destination folder when multiple files are selected.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/323488, which describes the general explorer context-menu targeting inconsistency. Fixing the context resolution in `getContext()` in `src/vs/workbench/contrib/files/browser/views/explorerView.ts` to always respect the right-clicked item over the existing multi-selection would resolve this paste scenario as well.
