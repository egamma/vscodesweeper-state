# Issue #311305 — Missing Bearer token error when connecting to MCP behind OAuth

[https://github.com/microsoft/vscode/issues/311305](https://github.com/microsoft/vscode/issues/311305)

## Summary

Real bug with a credible root cause identified by a commenter: the dynamic auth provider's scope-matching logic in extHostAuthentication.ts overwrites token scopes to empty string when no scopes are explicitly requested, breaking the session lookup and causing the Bearer token to be dropped. The scope overwrite at line 608-610 remains on current main.

## Change summary

MCP OAuth fails with a TypeError during metadata parsing and a subsequent 401 Unauthorized because the Bearer token is never attached — caused by scope-mismatch overwrite logic that sets token scopes to empty string when no scopes were requested.

## Best solution

Fix the scope overwrite logic in `src/vs/workbench/api/common/extHostAuthentication.ts` (around line 608) so that when the requested scopes are empty (no explicit scopes), the token's original scopes from the authorization server response are preserved rather than being overwritten to empty string. This aligns with RFC 6749 §3.3 which allows servers to assign default scopes when none are requested.
