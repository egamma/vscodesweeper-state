# Issue #234358 — Task API for `ShellExecutionOptions.env` type does not support `undefined` values

[https://github.com/microsoft/vscode/issues/234358](https://github.com/microsoft/vscode/issues/234358)

## Summary

Still unfixed on main: `ShellExecutionOptions.env` is typed as `{ [key: string]: string }` while `TerminalOptions.env` supports `string | null | undefined`. Team-authored issue; keeping open.

## Change summary

The Task API's `ShellExecutionOptions.env` does not allow `undefined` values, unlike `TerminalOptions.env` which accepts `string | null | undefined`.

## Best solution

Update the `env` property type in `ShellExecutionOptions` (and likely `ProcessExecutionOptions`) in `src/vscode-dts/vscode.d.ts` from `{ [key: string]: string }` to `{ [key: string]: string | null | undefined }` to match `TerminalOptions.env`, and update the corresponding implementation in `src/vs/workbench/api/common/extHostTypes.ts`.
