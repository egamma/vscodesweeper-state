# Issue #324621 — 3 subagents kicked off, first one to finish still shows running in primary chat

[https://github.com/microsoft/vscode/issues/324621](https://github.com/microsoft/vscode/issues/324621)

## Summary

Team-authored bug report: inline pill spinner doesn't stop when a subagent finishes while other subagents are still running. The `isExternallyActive` / `markAsInactive` logic in `chatSubagentContentPart.ts` may not fire correctly when the first of multiple concurrent subagents completes. No fix has landed since filing.

## Change summary

When multiple subagents run concurrently and the first one finishes, its inline pill in the primary chat still shows a running spinner.

## Best solution

Fix the per-invocation completion path in `src/vs/workbench/contrib/chat/browser/widget/chatContentParts/chatSubagentContentPart.ts` so `markAsInactive()` fires for each subagent independently when its tool invocation completes, regardless of sibling subagent state.
