# Issue #321648 — Context window indicator shows higher fill immediately after compacting conversation than actual

[https://github.com/microsoft/vscode/issues/321648](https://github.com/microsoft/vscode/issues/321648)

## Summary

Real visual bug: the context usage widget always displays the last response's token usage, so immediately after a compact the indicator shows the compact turn's own (large) prompt tokens rather than the reduced post-compact context size. The value self-corrects on the next normal turn.

## Change summary

Context window indicator shows inflated usage immediately after compacting because it reflects the compact turn's own token counts rather than the reduced post-compact state.

## Best solution

After a compact turn completes, the widget should either use the compact response's usage to reflect the actual reduced context, or skip the compact turn's own usage and wait for the next turn's usage. The fix likely involves `chatContextUsageWidget.ts` (`updateFromResponse`) and possibly `chatInputPart.ts` (`updateContextUsageWidget`), e.g. detecting that the just-completed request was a compact and using a post-compact context estimate instead of the raw compact-turn token counts.

- **update trigger:** src/vs/workbench/contrib/chat/browser/widget/input/chatInputPart.ts:2962–2964 calls `contextUsageWidget.update(model.lastRequest, …)` on `completedRequest`, so when the compact turn completes, the widget renders the compact turn's own (inflated) token counts.
- **compact action:** src/vs/workbench/contrib/chat/browser/actions/chatActions.ts:1118 shows compact is just `widget.acceptInput('/compact')` — it's a normal chat turn, so its response usage is treated identically to any other turn by the widget.
