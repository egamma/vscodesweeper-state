# Issue #313352 — Agents Mobile: Long Press on Session List Opens Activity Bar Configuration Menu Instead of Context Menu

[https://github.com/microsoft/vscode/issues/313352](https://github.com/microsoft/vscode/issues/313352)

## Summary

Team-filed mobile bug: the sessions list tree relies on the tree widget's built-in `onContextMenu` for right-click/long-press, but on mobile the long-press gesture is intercepted by the activity bar's own context menu handler instead of reaching the session list's handler. The `installLongPress` utility exists but is not wired into the sessions list. Keeping open as a team-authored issue.

## Change summary

Long-pressing a session in the Agents window session list on mobile opens the activity bar configuration menu instead of the session context menu.

## Best solution

Wire `installLongPress` from `src/vs/sessions/browser/parts/mobile/longPress.ts` into the session list tree rows (in `src/vs/sessions/contrib/sessions/browser/views/sessionsList.ts`) so a long-press synthesises a contextmenu event on the correct element, or add `e.preventDefault()` / `e.stopPropagation()` on the session list container to prevent the activity bar from capturing the touch gesture.
