# Issue #224056 — Cannot read properties of undefined (reading 'size')

[https://github.com/microsoft/vscode/issues/224056](https://github.com/microsoft/vscode/issues/224056)

## Summary

Team-authored error-telemetry issue. The crash is a race condition where `this.items[index]` is undefined when `elementHeight` is called during a reveal triggered by keyboard navigation — the item array can change between the bounds check in `reveal()` and the actual access. Still present on current main with no guard.

## Change summary

TypeError when `listView.elementHeight(index)` accesses `.size` on an undefined item, triggered during keyboard-navigation-driven reveal.

## Best solution

Add a defensive guard in `elementHeight` (or at the call site in `reveal`) to handle the case where `this.items[index]` is undefined — either return 0 or bail out of the reveal. The fix belongs in `src/vs/base/browser/ui/list/listView.ts` around line 858.
