# Issue #317938 — Agent window: `@` mention does not trigger file/symbol reference picker in new sessions

[https://github.com/microsoft/vscode/issues/317938](https://github.com/microsoft/vscode/issues/317938)

## Summary

Real bug, traceable on current main: the `AgentHostInputCompletionHandler` in the Agents window only registers its Monaco completion provider when `activeSession` is non-null, but in a brand-new session (before the first message) `activeSession` is `undefined`, so `@` trigger completions are never offered. The `_resolveContext` method also early-returns `undefined` when there is no active session. The issue is well-described with clear repro steps and remains unfixed.

## Change summary

Typing `@` in the Agents window input box does not trigger the file/symbol reference picker when the session is new (no prior messages).

## Best solution

The `AgentHostInputCompletionHandler` constructor's autorun (line ~185 of `src/vs/sessions/contrib/chat/browser/agentHostInputCompletions.ts`) and `_resolveContext` (line ~225) should fall back to the default/pending session type scheme when `activeSession` is `undefined` — for example by consulting the new-session draft from `ISessionsManagementService.newSession` or deriving the scheme from the default provider. This ensures trigger characters are registered and completions resolve even before the first message creates a concrete session.
