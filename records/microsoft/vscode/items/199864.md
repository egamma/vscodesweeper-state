# Issue #199864 — No Feedback when Issue Reporter's GitHub API Call Fails

[https://github.com/microsoft/vscode/issues/199864](https://github.com/microsoft/vscode/issues/199864)

## Summary

Still reproduces on main: `submitToGitHub` in baseIssueReporterService.ts silently returns `false` on API failure with no user-visible feedback. The bug remains unfixed and is a valid, localized defect.

## Change summary

When the GitHub API call fails during issue submission, the Issue Reporter shows no error to the user — it just silently resets the button state.

## Best solution

Show a user-visible error notification (or inline message) in baseIssueReporterService.ts when `submitToGitHub` returns false or throws, and optionally fall back to the browser-based submission flow. The related duplicate https://github.com/microsoft/vscode/issues/201650 should be closed against this one.
