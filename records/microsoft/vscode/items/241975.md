# Issue #241975 — Very large set of details in terminal suggestions for `Where-object`

[https://github.com/microsoft/vscode/issues/241975](https://github.com/microsoft/vscode/issues/241975)

## Summary

Valid open feature request: terminal suggest should truncate or consolidate overwhelming parameter set lists (like PowerShell's 32-set Where-Object). A VS Code team member (Tyriar) acknowledged the problem and outlined a plan to truncate, but no implementation exists on main yet.

## Change summary

Terminal suggestions show an overwhelming number of parameter set variations for PowerShell commands like Where-Object (32 entries), making the suggest widget hard to use.

## Best solution

Implement truncation/consolidation of repetitious detail entries in the terminal suggest widget (e.g., show first 3 parameter sets with an ellipsis), likely in `src/vs/workbench/contrib/terminalContrib/suggest/browser/terminalSuggestAddon.ts` or the completion model. Tyriar's plan from the discussion (comment 8) outlines the approach.
