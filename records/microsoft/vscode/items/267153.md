# Issue #267153 — Cannot accept suggestion from accessible view

[https://github.com/microsoft/vscode/issues/267153](https://github.com/microsoft/vscode/issues/267153)

## Summary

Source-reproducible on current main and assigned to the On Deck milestone. The root cause is that the accessible view's own InlineCompletionsController erroneously stops all other controllers' models when its editor gains focus, so by the time the Accept action runs, the main editor's model state is null.

## Change summary

Accepting an inline suggestion from the accessible view toolbar does nothing because the suggestion model is already stopped.

## Best solution

Guard the 'stop other models' branch in InlineCompletionsController (line ~270) so that a readonly editor (whose model is undefined) does not stop other controllers. An additional guard `if (!model) { return; }` before the stop loop would prevent the accessible view's controller from clearing the main editor's completion state.
