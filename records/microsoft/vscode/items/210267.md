# Issue #210267 — Git - Add an option to cancel a git pull/push request if it takes too long

[https://github.com/microsoft/vscode/issues/210267](https://github.com/microsoft/vscode/issues/210267)

## Summary

Valid feature request: `git.supportCancellation` only covers sync, not standalone push/pull. The gap remains on current main — keeping open.

## Change summary

Extend cancellation support to standalone git push and git pull operations, not just sync.

## Best solution

Extend the `supportCancellation` logic in `extensions/git/src/repository.ts` to wrap `pullFrom()`, `push()`, `pushTo()`, `pushFollowTags()`, and `pushTags()` with a cancellable progress notification, mirroring the existing pattern in `sync()`. This is a localized change in the git extension, not a core workbench change.
