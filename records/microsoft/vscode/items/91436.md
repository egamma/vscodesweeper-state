# Issue #91436 — Task instanceLimit doesn't work when depending on long running tasks

[https://github.com/microsoft/vscode/issues/91436](https://github.com/microsoft/vscode/issues/91436)

## Summary

Fixed on main by PR #181656 (commit 9b2ce6996ee) which rewrote instanceLimit tracking to work correctly with dependent tasks, shipped in 1.79.0. The related issue #200429 was also closed as duplicate. This can be closed as completed.

## Change summary

Task `instanceLimit` is ignored when the task depends on a long-running task — all queued instances launch at once when the dependency finishes.

## Best solution

Keep the shipped fix from commit 9b2ce6996ee (1.79.0); no further change needed. The fix replaced the broken `_instances` map with proper counting via `_activeTasks`, so instanceLimit is now enforced for dependent tasks.
