## Summary

The requested work-schedule-aware credit burn-rate dashboard is a GitHub/Copilot billing concern, not a VS Code editor feature — the credit quota, reset cycle, and consumption data all live in the Copilot service, not in the editor. VS Code only surfaces what the service reports; building a configurable work-calendar and trend engine in the editor would add always-on complexity for a niche audience.
