# Issue #323888 — Support `.worktreeinclude`

[https://github.com/microsoft/vscode/issues/323888](https://github.com/microsoft/vscode/issues/323888)

## Summary

Valid feature request to support the `.worktreeinclude` file convention (used by Claude Code, Codex, etc.) as an alternative to the VS Code-specific `git.worktreeIncludeFiles` setting. No duplicate found; the existing `git.worktreeIncludeFiles` setting covers the same goal but requires VS Code-specific configuration rather than a cross-tool convention file.

## Change summary

Support the `.worktreeinclude` file as a convention for specifying files to include when creating git worktrees, instead of requiring the VS Code-specific `git.worktreeIncludeFiles` setting.

## Best solution

Extend the git extension's `_getWorktreeIncludePaths` logic in `extensions/git/src/repository.ts` to also read patterns from a `.worktreeinclude` file in the repository root, merging them with the existing `git.worktreeIncludeFiles` setting. Additionally, register `.worktreeinclude` as a git-related file type for language/icon association. This aligns with the cross-tool convention emerging around agent workflows.
