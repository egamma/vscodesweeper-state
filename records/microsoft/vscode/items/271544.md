# Issue #271544 — Searching across files with regex has incorrect and inconsistent handling of CRLF line endings

[https://github.com/microsoft/vscode/issues/271544](https://github.com/microsoft/vscode/issues/271544)

## Summary

Valid bug — regex search produces inconsistent results between CRLF and LF files because `fixRegexNewline` rewrites `\n` but does not account for `.` matching `\r` in CRLF files. An open PR (#310798) is already attempting a fix; keeping open pending that PR's review.

## Change summary

Regex search across files gives different results for CRLF vs LF files because `.` matches `\r` in CRLF content, and `$` anchoring behaves inconsistently.

## Best solution

Review and land https://github.com/microsoft/vscode/pull/310798 (or an equivalent fix) that ensures `.` does not match `\r` in CRLF files during ripgrep regex search, so results are consistent regardless of line endings. The fix belongs in `src/vs/workbench/services/search/node/ripgrepTextSearchEngine.ts` in or near `fixRegexNewline`.

- **ripgrep --crlf flag:** Line 486 passes `--crlf` to ripgrep, which makes `$` match before `\r\n`, but `.` still matches `\r` — the mismatch causes the inconsistency the reporter describes.
- **open PR:** https://github.com/microsoft/vscode/pull/310798 is an open PR by maruthang that proposes rewriting `.` to exclude `\r` for consistent results across LF and CRLF files.
