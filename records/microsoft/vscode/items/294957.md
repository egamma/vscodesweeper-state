# Issue #294957 — MCP refresh showing when I don't have any enabled?

## Summary

Legitimate UX polish concern about the MCP tools refresh button being visible when no MCP servers are enabled. Already under discussion with a team member explanation — needs a product decision on whether to hide the button or gate its visibility on known/enabled servers.

## Change summary

The MCP tools refresh button appears even when the user has no MCP servers enabled and `chat.extensionTools.enabled` is false.

## Best solution

Decide whether the refresh button should be hidden when no MCP servers are configured/enabled, or whether it needs to remain because extension-contributed servers haven't been discovered yet. If the latter, consider gating visibility on whether any extensions that *could* contribute MCP servers are installed.
