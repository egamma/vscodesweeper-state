# Issue #141272 — Testing API: allow extension to control sort-order of `TestController`s it contributes

[https://github.com/microsoft/vscode/issues/141272](https://github.com/microsoft/vscode/issues/141272)

## Summary

Still a valid, unaddressed request. Issue #130882 added `sortText` to `TestItem` but the TestController interface itself still has no sort control — controllers are ordered by label with no extension-controlled override.

## Change summary

Allow extensions to control the sort order of TestController entries in the test explorer when multiple controllers are registered.

## Best solution

Add a `sortText` (or similar) property to the `TestController` interface so extensions with multiple controllers can sequence them independently of the display label. The change would touch `src/vscode-dts/vscode.d.ts` (API surface), `src/vs/workbench/contrib/testing/common/testTypes.ts`, and the tree sorter in `testingExplorerView.ts`.
