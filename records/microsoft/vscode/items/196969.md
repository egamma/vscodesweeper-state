# Issue #196969 — Activation `onCommand` sometimes never returns when there are multiple extension hosts

[https://github.com/microsoft/vscode/issues/196969](https://github.com/microsoft/vscode/issues/196969)

## Summary

Team-authored issue with `important` label, reopened after the initial fix (PR #197440) proved insufficient. The underlying race — a terminated web worker extension host blocking all subsequent activation events — can still occur in multi-extension-host environments like Codespaces.

## Change summary

When the web worker extension host terminates unexpectedly, `activateByEvent` never resolves, blocking commands that depend on `onCommand` activation in multi-extension-host setups.

## Best solution

The extension host manager needs to detect a terminated/defunct extension host and resolve (or reject) pending activation promises rather than waiting indefinitely. The skip-if-no-interested-extensions fix in PR #197440 helped the common case but doesn't cover the scenario where the host dies mid-activation. A proper fix would add lifecycle tracking in `ExtensionHostManager._activateByEvent` (around line 338–355) to fail fast when the host connection is lost.
