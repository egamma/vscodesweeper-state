# Issue #196969 — Activation `onCommand` sometimes never returns when there are multiple extension hosts

[https://github.com/microsoft/vscode/issues/196969](https://github.com/microsoft/vscode/issues/196969)

## Summary

Keep open: this is a team-authored issue and it carries the `important` label, both of which are hard keep-open signals. The later maintainer comment also says it was seen again after an attempted verification, so there is no strong completed/fixed basis to close it.

## Change summary

Activation for `onCommand` can hang in Codespaces when a web worker extension host has terminated and later activation events still try to communicate with it.

## Best solution

The extension host service should detect a terminated web worker extension host and fail, remove, or recover its pending activation path so `activateByEvent` cannot block command activation indefinitely. The likely area remains `src/vs/workbench/services/extensions/common/extensionHostManager.ts` and `src/vs/workbench/services/extensions/common/abstractExtensionService.ts`.
