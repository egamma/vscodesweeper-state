# Issue #257528 — Another instance of Code is already running as Administrator

[https://github.com/microsoft/vscode/issues/257528](https://github.com/microsoft/vscode/issues/257528)

## Summary

This is a known Windows limitation where mixing admin and non-admin VS Code instances fails with EPERM on the IPC named pipe. The error dialog at src/vs/code/electron-main/main.ts:383-389 handles this intentionally, but the behavior is a real workflow blocker for users who work with elevated permissions. Needs area routing to the right owner.

## Change summary

Opening a second VS Code window fails with 'Another instance of Code is already running as Administrator' when there is an elevation mismatch between the running instance and the new one on Windows.

## Best solution

Route to the `workbench-run-as-admin` area owner. The IPC elevation-mismatch guard in `src/vs/code/electron-main/main.ts` (line 383) catches EPERM and shows the dialog. A prior feature request to allow mixed-elevation windows (https://github.com/microsoft/vscode/issues/230511) was closed for insufficient upvotes. The area owner should decide whether to revisit this limitation or close as `*as-designed`. Issue https://github.com/microsoft/vscode/issues/274859 is a duplicate of this one and should be consolidated.
