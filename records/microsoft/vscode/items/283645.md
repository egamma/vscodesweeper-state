# Issue #283645 — Support Scoped permissions to Singular Organization or Repo

[https://github.com/microsoft/vscode/issues/283645](https://github.com/microsoft/vscode/issues/283645)

## Summary

This is a valid, actively discussed concern about OAuth scope granularity for Copilot authentication. A team member engaged early explaining the GitHub OAuth App limitation. With 20 upvotes it meets the backlog threshold. The issue sits at the intersection of GitHub platform constraints and the Copilot extension's auth model — it remains under discussion and should stay open.

## Change summary

Copilot requests overly broad `repo` scope (full read/write to all repos) when only search API access is needed, with no way to scope permissions to a single org or repo.

## Best solution

The ideal end state is either migrating Copilot auth to a GitHub App (which supports per-repo granularity), or GitHub adding a read-only repo scope to OAuth Apps. This is primarily a GitHub platform decision and Copilot extension architecture choice — VS Code core's auth service passes through whatever scopes extensions request. The fix lives in the Copilot extension and/or GitHub's OAuth infrastructure, not in VS Code core.
