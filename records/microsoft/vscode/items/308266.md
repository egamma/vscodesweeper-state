This is a duplicate of the canonical feature request for a default-mode setting. The described behavior — new conversations resetting to the default Agent mode — is by design, and the setting `chat.newSession.defaultMode` now exists on main to let users configure which agent/mode new sessions start with (including custom agents by name).
