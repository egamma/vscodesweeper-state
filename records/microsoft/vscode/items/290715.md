# Issue #290715 — Zen Mode is not using space well while center mode.

[https://github.com/microsoft/vscode/issues/290715](https://github.com/microsoft/vscode/issues/290715)

## Summary

Feature request to improve Zen Mode's centered layout UX: a quick toggle to disable center layout while in Zen Mode, and making the sidebar use the empty center-layout margins rather than pushing them aside. The existing `zenMode.centerLayout` setting lets the user disable it permanently but doesn't address the dynamic toggle or sidebar overlap use case.

## Change summary

When Zen Mode is active with centered layout, the sidebar doesn't reclaim the empty margin space, and there's no quick way to toggle center layout off without leaving Zen Mode.

## Best solution

Consider adding a visible toggle (button or action) in Zen Mode to quickly enable/disable centered layout, and improve layout so that when the sidebar opens in Zen Mode, it overlaps the centered-layout margins rather than compressing the editor further. The layout logic lives in `src/vs/workbench/browser/workbench.zenMode.contribution.ts` and the centered layout implementation in the editor part.
