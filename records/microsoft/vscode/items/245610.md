# Issue #245610 — Alt Down Arrow to move */ comment is reformatting code

[https://github.com/microsoft/vscode/issues/245610](https://github.com/microsoft/vscode/issues/245610)

## Summary

Still reproduces on main. The causing commit (359ec5270a9c) added indent/deindent rules to C++ language-configuration.json which interfere with move-lines auto-indentation. Active fix work exists on a feature branch but has not yet landed on main.

## Change summary

Alt+Down Arrow to move a `*/` comment closing line past other code causes incorrect reformatting/reindentation of subsequent lines.

## Best solution

The fix is being developed on a feature branch (commits bf65dbaa04e, 2f076614ea8) that modifies auto-indent calculation in moveLinesCommand.ts to use onEnter rules. This should land once the branch is merged. The root cause is the C++ indentationRules added in commit 359ec5270a9c interacting poorly with the move-lines auto-indent logic.
