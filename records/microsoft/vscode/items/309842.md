# Issue #309842 — Custom models in business plan results in 500 error

[https://github.com/microsoft/vscode/issues/309842](https://github.com/microsoft/vscode/issues/309842)

## Summary

This is a support question about how GitHub Copilot's custom-models infrastructure works, not a VS Code bug. The 500 error originates from GitHub's Copilot API proxy (CAPI), which cannot reach the user's local-only ollama server — the custom model endpoint must be reachable from GitHub's servers.

## Change summary

User receives a 500 error when using a custom model endpoint (local ollama) configured via GitHub org custom-models settings in Copilot Chat.

## Best solution

Direct the user to GitHub Copilot support or the GitHub Community forum for custom-models configuration help. The core issue is that CAPI proxies requests server-side, so the base URL must be publicly reachable — a localhost-only endpoint will always fail from the proxy. The CLI may use a different routing path. This is not actionable in the VS Code codebase.
