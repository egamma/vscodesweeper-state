# Issue #164905 — Starving extension host can block file navigation

[https://github.com/microsoft/vscode/issues/164905](https://github.com/microsoft/vscode/issues/164905)

## Summary

A team member (osortega) tried to reproduce this in December 2024 and could not, and the reporter has not responded. The underlying file search in the quick pick (AnythingQuickAccessProvider) uses ISearchService which delegates to ripgrep in the main process for local workspaces, not through the extension host — so it's unclear how TS server initialization would block it today. Marking info-needed to see if this is still relevant.

## Change summary

Request to isolate the search extension into its own extension host via affinity so that a starving extension host (e.g. during TS server initialization) cannot block file navigation in quick pick.

## Best solution

If the reporter confirms this is still happening on a current build, investigate whether any extension-provided FileSearchProvider is in the critical path for quick pick file search and whether extension affinity would help. Otherwise, let the info-needed process retire it.
