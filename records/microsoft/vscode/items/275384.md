# Issue #275384 — Undoing the edit by `/savePrompt` leaves behind blank untitled file

[https://github.com/microsoft/vscode/issues/275384](https://github.com/microsoft/vscode/issues/275384)

## Summary

Still reproducible on current main: when rejecting/undoing a chat edit on an untitled prompt file, `_doReject()` tries to delete the untitled URI via `fileService.del()`, which fails silently (EROFS), leaving an empty untitled file open. No fix has landed.

## Change summary

Undoing a `/savePrompt` edit in the chat view leaves behind an empty untitled file and logs an EROFS error.

## Best solution

In `chatEditingModifiedDocumentEntry.ts` `_doReject()`, when the file is an untitled scheme, instead of calling `fileService.del()`, revert and dispose the text model and close the associated editor tab (similar to how other untitled file dismissals work in the workbench).
