# Issue #268397 — Test results show incorrect validation state after repeated continuous runs

[https://github.com/microsoft/vscode/issues/268397](https://github.com/microsoft/vscode/issues/268397)

## Summary

Real bug: LiveTestResult.updateState never clears the `retired` flag, so after `markTestRetired` is called during a continuous run, subsequent state updates in the same live result still show dull (retired) icons.

## Change summary

Test results show dull/retired icons after a second continuous run trigger because the retired flag is never cleared when new results arrive.

## Best solution

In `LiveTestResult.updateState` (or `fireUpdateAndRefresh`), reset `entry.retired = false` when a new terminal state is applied. This ensures that re-execution within the same live continuous-run result clears the stale retirement flag.
