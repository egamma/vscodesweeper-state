# Issue #206769 — Advanced diff algorithm hangs when dealing with large diff 

[https://github.com/microsoft/vscode/issues/206769](https://github.com/microsoft/vscode/issues/206769)

## Summary

Still valid on main: the advanced diff algorithm can hang on large diffs. The timeout-passing bug (dccb86cc946) was fixed, but users who set maxComputationTime to 0 get InfiniteTimeout, and the new @vscode/diff external computer explicitly lacks timeout support. The recent comment confirms this remains problematic with large files like package-lock.json.

## Change summary

The advanced diff algorithm hangs (infinite computation) when diffing files with thousands of changed lines, leaving the editor unresponsive.

## Best solution

Implement timeout support in the @vscode/diff external computer (currently commented out at src/vs/editor/common/diff/externalLinesDiffComputer.ts:62), and consider adding a cancellation/progress mechanism so that large diffs can be aborted by the user or by tab close. The character-level refinement in defaultLinesDiffComputer.ts may also need a per-block size guard to skip refinement for very large diff blocks.
