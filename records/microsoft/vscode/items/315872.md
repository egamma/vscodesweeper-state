# Issue #315872 — Copilot chat stalling when invoking an agent defined in a web extension

[https://github.com/microsoft/vscode/issues/315872](https://github.com/microsoft/vscode/issues/315872)

## Summary

Team-authored bug report — must stay open for maintainer judgment. The issue describes chat agent handlers never being invoked when the contributing extension runs as a web extension, leading to a stalled request and eventual OOM crash. No fix found on current main.

## Change summary

Chat agent handlers registered by web extensions are never invoked in desktop mode; requests stall indefinitely and eventually crash the extension development host with an OOM error.

## Best solution

Investigate the RPC proxy flow between the web worker extension host and `MainThreadChatAgents2` — the `$invokeAgent` call may be routed to the wrong extension host instance, or the handle mapping may not reach the web worker's `ExtHostChatAgents2`. The likely files are `src/vs/workbench/api/browser/mainThreadChatAgents2.ts` and `src/vs/workbench/api/common/extHostChatAgents2.ts`. The OOM suggests an unbounded retry or message accumulation when the invoke never resolves.
