# Issue #178705 — workbench.action.toggleSidebarVisibility forgets selected file

[https://github.com/microsoft/vscode/issues/178705](https://github.com/microsoft/vscode/issues/178705)

## Summary

Confirmed bug still present on main: when the sidebar is hidden and re-shown, the explorer's previous file selection is lost because the visibility handler unconditionally calls `selectActiveFile(true)`, overriding the prior selection state. The maintainer confirmed this in the discussion.

## Change summary

Toggling the sidebar off and on causes the file explorer to lose its previous file selection, resetting focus to the active editor's file instead.

## Best solution

In `src/vs/workbench/contrib/files/browser/views/explorerView.ts`, the `onDidChangeBodyVisibility` handler (line ~325) should preserve the tree's existing selection/focus state across hide/show cycles rather than unconditionally calling `selectActiveFile(true)`. A flag or saved selection state before hide could be restored on re-show, falling back to `selectActiveFile` only when no prior state exists.
