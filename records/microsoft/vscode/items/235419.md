# Issue #235419 — `Search Service` does not use `asCanonicalUri` 

[https://github.com/microsoft/vscode/issues/235419](https://github.com/microsoft/vscode/issues/235419)

## Summary

Team-authored debt issue. The search service still does not use `asCanonicalUri` on current main — no commit addresses it. Keeping open for the area owner.

## Change summary

The SearchService should use `IUriIdentityService.asCanonicalUri` to normalize URIs returned by ripgrep so they are in the correct canonical format.

## Best solution

Add `IUriIdentityService` to the search service layer and call `asCanonicalUri` on URIs constructed from ripgrep results (in `ripgrepTextSearchEngine.ts` and the file-search path) to ensure canonical drive-letter casing and consistent URI formatting.
