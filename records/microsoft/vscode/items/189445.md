# Issue #189445 — Cursor moves to breakpoint when Debug: Focus Editor On Break is turned off

[https://github.com/microsoft/vscode/issues/189445](https://github.com/microsoft/vscode/issues/189445)

## Summary

Still valid on main: the `debug.focusEditorOnBreak` setting only controls editor focus via `preserveFocus`, but the cursor/selection is still moved to the breakpoint line. A MEMBER acknowledged this and suggested expanding the setting to an enum with finer-grained options. Issue #210811 was closed as a duplicate of this one, making this the canonical tracker.

## Change summary

When `debug.focusEditorOnBreak` is disabled, the cursor still jumps to the breakpoint line — the setting only prevents editor focus, not cursor movement.

## Best solution

Expand `debug.focusEditorOnBreak` from a boolean to an enum (as roblourens suggested) with options like: always focus, focus only if editor not already focused, never focus and never move cursor. The logic lives in `src/vs/workbench/contrib/debug/browser/debugSession.ts` (line ~1387) and `src/vs/workbench/contrib/debug/browser/debugService.ts` (`focusStackFrame`). The enum would need to also guard the `openInEditor` call or pass a flag to skip cursor repositioning.
