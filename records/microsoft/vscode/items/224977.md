# Issue #224977 — [css] allow to overtype the semicolon

[https://github.com/microsoft/vscode/issues/224977](https://github.com/microsoft/vscode/issues/224977)

## Summary

Valid feature request to allow overtyping the semicolon inserted by snippet/Emmet completion in CSS. The semicolon comes from snippet expansion, not auto-closing pairs, so overtype logic doesn't apply. Already labeled correctly and in Backlog.

## Change summary

Typing `;` after a CSS snippet completion inserts a duplicate semicolon instead of overtyping the existing one.

## Best solution

Extend the editor's overtype tracking so that characters inserted by snippet tabstop exits (like the trailing `;` in CSS property snippets) are eligible for overtype. This likely involves `src/vs/editor/common/cursor/cursorTypeEditOperations.ts` and snippet session tracking in `src/vs/editor/contrib/snippet/`.
