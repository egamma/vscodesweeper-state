# Issue #241757 — Allow me to drag and drop a diff editor into chat/edits

[https://github.com/microsoft/vscode/issues/241757](https://github.com/microsoft/vscode/issues/241757)

## Summary

Still a valid gap: the chat drag-and-drop path resolves editor drops as plain file resources with no diff-aware semantics. Dragging a diff editor tab into chat does not attach the diff content or both sides — the feature remains unimplemented.

## Change summary

Allow dragging a diff editor tab into the chat input to attach diff content (git diff or both files for arbitrary comparisons).

## Best solution

Extend `ChatDragAndDrop.resolveAttachContext` and `resolveEditorAttachContext` to detect `DiffEditorInput` and produce either a git-diff attachment (for SCM diffs) or attach both modified/original resources (for arbitrary file comparisons). The logic lives in `src/vs/workbench/contrib/chat/browser/widget/chatDragAndDrop.ts` and `src/vs/workbench/contrib/chat/browser/attachments/chatAttachmentResolveService.ts`.
