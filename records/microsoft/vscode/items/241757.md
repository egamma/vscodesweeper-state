# Issue #241757 — Allow me to drag and drop a diff editor into chat/edits

[https://github.com/microsoft/vscode/issues/241757](https://github.com/microsoft/vscode/issues/241757)

## Summary

Still a valid gap on current main: the chat drag-and-drop pipeline (chatDragAndDrop.ts → chatAttachmentResolveService.ts) has no diff-editor-aware logic — dropping a diff editor into chat treats it as a regular file resource, discarding the diff context. The feature request remains unimplemented.

## Change summary

Allow dragging a diff editor tab into the chat input to attach the git diff or both sides of the comparison as context.

## Best solution

Add diff-editor detection in `resolveEditorAttachContext` (chatAttachmentResolveService.ts): when the dropped editor is a `DiffEditorInput`, attach the diff content (for git diffs, the unified diff; for arbitrary file comparisons, both files). The `ChatDragAndDrop` and `extractEditorsDropData` pipeline already passes editor data through — the missing piece is recognizing and handling the diff case in the resolve service.
