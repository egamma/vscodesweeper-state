# Issue #196780 — Cannot get compare result when files size >= 2M

[https://github.com/microsoft/vscode/issues/196780](https://github.com/microsoft/vscode/issues/196780)

## Summary

Still relevant on main: the `diffEditor.maxFileSize` setting is defined (default 50 MB) but never enforced in the current diff editor — large-file diffs simply run until the 5-second `maxComputationTime` expires, potentially freezing the UI. The performance report for 2 MB files is a real gap.

## Change summary

Comparing two large files (≥ 2 MB) in the diff editor blocks and is extremely slow, with no effective size guard.

## Best solution

Enforce `diffEditor.maxFileSize` in `diffEditorViewModel.ts` before calling `computeDiff` — check both models' `getValueLength()` against the configured limit and bail out early with a user-facing message (similar to how the old `DiffEditorWidget` handled it). This would also surface a clear UX when files exceed the limit rather than silently timing out.
