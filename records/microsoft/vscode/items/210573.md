# Issue #210573 — Resizing Primary Side Bar opens terminal Panel

[https://github.com/microsoft/vscode/issues/210573](https://github.com/microsoft/vscode/issues/210573)

## Summary

Confirmed and still present on main: the SplitView snap logic causes a hidden panel to snap open when resizing the primary sidebar, when both occupy the same axis (panel positioned left/right). No fix has been committed.

## Change summary

Resizing the primary side bar inadvertently snaps the hidden panel open when the panel is positioned on the same side.

## Best solution

The fix belongs in the grid/splitview snap logic in `src/vs/base/browser/ui/splitview/splitview.ts` or `src/vs/workbench/browser/layout.ts` — when a user is resizing one part, the snap threshold for an adjacent hidden part on the same axis should not activate. The `paneCompositePart.ts` snap getter or the grid's resize handling could be made context-aware to suppress snapping during an unrelated sash drag.
