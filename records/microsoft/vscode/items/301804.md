# Issue #301804 — Copilot Agent run_in_terminal fails with ENOPRO: No file system provider found in GitHub Codespaces

[https://github.com/microsoft/vscode/issues/301804](https://github.com/microsoft/vscode/issues/301804)

## Summary

Fixed on main and shipped in 1.124.0. Commit 0712539c435 adds a canHandleResource() guard in getCwdResource() so that the missing file:// provider in VS Code web no longer throws ENOPRO during agent terminal tool calls.

## Change summary

Copilot Agent's run_in_terminal tool fails with ENOPRO in VS Code web (Codespaces, tunnels) on browsers lacking the File System Access API (Safari, Firefox, Brave).

## Best solution

Keep the shipped fix (commit 0712539c435, released in 1.124.0). No further change needed — the canHandleResource() guard in terminalInstance.ts prevents the ENOPRO path. Users should update to 1.124+.
