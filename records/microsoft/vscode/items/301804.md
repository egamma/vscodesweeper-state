# Issue #301804 — Copilot Agent run_in_terminal fails with ENOPRO: No file system provider found in GitHub Codespaces

[https://github.com/microsoft/vscode/issues/301804](https://github.com/microsoft/vscode/issues/301804)

## Summary

Fixed on main and shipped in 1.124.0: commit 0712539c435 guards getCwdResource() with canHandleResource() to prevent the ENOPRO error when the file:// provider is absent in VS Code web.

## Change summary

Copilot Agent's run_in_terminal tool throws ENOPRO when used in VS Code Web (Codespaces) on Safari, Firefox, and Brave because getCwdResource() calls fileService.exists() on a file:// URI with no registered provider.

## Best solution

Keep the shipped fix in commit 0712539c435 (1.124.0). No further change needed — reporters should confirm the fix on the latest Codespaces build.
