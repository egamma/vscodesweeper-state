# Issue #193363 — vscode seems don't handle newline and tab correctly in snippet string provided by inlineCompletionProvider

[https://github.com/microsoft/vscode/issues/193363](https://github.com/microsoft/vscode/issues/193363)

## Summary

Still reproduces on main: when an inline completion provider returns a SnippetString with multiline text that already includes indentation, accepting it via SnippetController2.insert() with the default adjustWhitespace:true doubles the indentation on each line after the first.

## Change summary

Accepting a multiline snippet from an inlineCompletionProvider corrupts indentation by adding extra tabs/spaces to each line.

## Best solution

Pass `adjustWhitespace: false` when calling `SnippetController2.insert()` during inline completion acceptance (line ~961 of inlineCompletionsModel.ts), since inline completion snippets are expected to include their own indentation context — or document that providers must omit base indentation from their snippet text.
