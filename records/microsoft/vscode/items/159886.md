# Issue #159886 — `DebugService` has high cost

[https://github.com/microsoft/vscode/issues/159886](https://github.com/microsoft/vscode/issues/159886)

## Summary

The critical startup-path issue (editor contributions force-creating DebugService) was fixed by multiple merged PRs, but the broader debt goal of making the service itself lighter (splitting ConfigurationManager, AdapterManager, etc. into separate services, further lazy-loading) remains partially unaddressed. Keeping open as a valid debt/perf tracking item.

## Change summary

DebugService construction is expensive and was being force-created at editor startup via CallStackEditorContribution, causing ~30% of editor creation time to be spent instantiating the debug infrastructure.

## Best solution

The immediate startup regression is resolved. The remaining value is further decomposing DebugService (ConfigurationManager lazy config loading, splitting into DebugModelService/DebugViewService/DebugConfigurationService as discussed in the comments). A maintainer should decide whether that further refactoring is still worth tracking here or should be closed as sufficiently addressed.
