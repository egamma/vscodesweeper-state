# Issue #158996 — "Follow link" hover is not linkified on markdown links

[https://github.com/microsoft/vscode/issues/158996](https://github.com/microsoft/vscode/issues/158996)

## Summary

Still reproduces on main: when a link provider lazily resolves targets (link.url is null until resolve is called), the hover message falls into the plain-text branch and shows 'Follow link' without a clickable hyperlink. Team-authored issue on the On Deck milestone — keeping open.

## Change summary

The 'Follow link' hover on markdown document links is not rendered as a clickable link because link resolution happens lazily and the hover is computed before resolve() is called.

## Best solution

Call `link.resolve()` before or during hover message computation in `src/vs/editor/contrib/links/browser/links.ts`, so that `link.url` is available when `getHoverMessage` builds the MarkdownString. The hover should use the `appendLink` path (line 389–392) instead of the plain-text fallback (line 394).
