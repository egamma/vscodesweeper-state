# Issue #306680 — Built-in Source Control does not properly work with a worktree in WSL

[https://github.com/microsoft/vscode/issues/306680](https://github.com/microsoft/vscode/issues/306680)

## Summary

Legitimate bug: VS Code's git extension has no WSL-to-Windows path translation when reading worktree gitdir files. When a worktree is created inside WSL, the `.git` file contains a WSL-style path (e.g. `/mnt/c/...`) that native Windows VS Code cannot resolve, so repository detection fails entirely.

## Change summary

Opening a WSL-created git worktree from native Windows VS Code fails to detect the git repository because the `.git` file contains a WSL-format path that isn't translated to a Windows path.

## Best solution

The git extension's repository discovery path (particularly `getRepositoryRoot` and `getRepositoryDotGit` in `extensions/git/src/git.ts`) would need WSL path translation when running on Windows and encountering `/mnt/` style paths. This is related to but distinct from https://github.com/microsoft/vscode/issues/275989 (the reverse direction — Windows paths breaking git in WSL). A holistic fix for WSL/Windows git path interop would address both.
