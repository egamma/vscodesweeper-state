# Issue #172189 — Inline suggest hover insufficient contrast when hovering actions

[https://github.com/microsoft/vscode/issues/172189](https://github.com/microsoft/vscode/issues/172189)

## Summary

The merged PR #172526 only bumped keybinding opacity from 0.5 to 0.6 — a partial fix. The agreed-upon solution (using proper keybinding widget styling with background color, matching the command palette) remains unimplemented on current main.

## Change summary

Inline suggestion hints hover should use proper keybinding widget styling (with background color) for sufficient contrast, matching the command palette and keybinding shortcuts UI.

## Best solution

Restyle the `.inlineSuggestionsHints .keybinding` elements to use the same keybinding widget appearance (background color, border) as the command palette, removing the opacity hack. Likely touches `src/vs/editor/contrib/inlineCompletions/browser/hintsWidget/inlineCompletionsHintsWidget.css` and possibly the widget TypeScript to use the standard `KeybindingLabel` renderer.
