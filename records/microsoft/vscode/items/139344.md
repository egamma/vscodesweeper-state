# Issue #139344 — User customizable ambiguous characters for Unicode Highlight

[https://github.com/microsoft/vscode/issues/139344](https://github.com/microsoft/vscode/issues/139344)

## Summary

Valid feature request for a setting to add custom characters to the unicode highlight list. The existing `allowedCharacters` setting only exempts characters; there is no inverse setting to flag additional ones. Low community traction (1 upvote) and in Backlog Candidates territory — keeping open for community process.

## Change summary

Allow users to define additional characters that should be highlighted by the Unicode Highlight feature, beyond the built-in ambiguous character set.

## Best solution

Add an `editor.unicodeHighlight.additionalCharacters` setting (a Record<string, true> like `allowedCharacters`) that marks extra characters for highlighting. Implementation would live in `src/vs/editor/common/config/editorOptions.ts` (config) and `src/vs/editor/contrib/unicodeHighlighter/browser/unicodeHighlighter.ts` (logic).
