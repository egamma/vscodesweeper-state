# Issue #325742 — Hook-denied (PreToolUse deny) tool calls are invisible in the chat UI — add a way to surface them

[https://github.com/microsoft/vscode/issues/325742](https://github.com/microsoft/vscode/issues/325742)

## Summary

Well-substantiated feature request: hook-denied tool invocations are intentionally hidden via ToolInvocationPresentation.Hidden in _handlePreToolUseDenial, with no setting to override. The code confirms the described behavior. No duplicate found. Worth routing to the chat area for consideration.

## Change summary

When a PreToolUse hook denies a tool call, the blocked invocation is completely hidden from the chat UI with no way to surface it.

## Best solution

Add an opt-in setting (e.g. `chat.tools.showBlockedInvocations`) or render denied tool calls as a collapsed/dimmed entry in the chat showing the tool name and denial reason, likely touching `_handlePreToolUseDenial` in `src/vs/workbench/contrib/chat/browser/tools/languageModelToolsService.ts` and the rendering logic in `chatToolInvocationPart.ts`. This would make hook authoring and debugging significantly more transparent.
