# Issue #300647 — Incorrect GitHub repository is targeted when user is attempting to publish branch to a non-writable repository with a name containing period/dot

[https://github.com/microsoft/vscode/issues/300647](https://github.com/microsoft/vscode/issues/300647)

## Summary

Confirmed bug on current main: the regex in pushErrorHandler.ts excludes dots from the repo name capture group, causing repos like `name.github.io` to be truncated to `name`. The fix is straightforward — reuse the existing `getRepositoryFromUrl` utility or fix the regex.

## Change summary

When pushing to a non-writable GitHub repository whose name contains a period (e.g. `user.github.io`), VS Code targets the wrong repository because the repo name is truncated at the first dot.

## Best solution

Replace the inline regex on line 120 of `extensions/github/src/pushErrorHandler.ts` with a call to the existing `getRepositoryFromUrl()` from `util.ts`, which already handles dots correctly via the `([^/]+?)` capture group. This also eliminates the code duplication.
