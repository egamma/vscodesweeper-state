## Summary

Valid feature request: the chat panel's tree-based list widget does not implement drag-to-scroll during text selection. No fix or implementation exists on main. The issue is in the Backlog milestone with only 1 upvote, so it should stay open for the community process.

## Change summary

Chat panel does not auto-scroll when dragging a text selection beyond the visible viewport, limiting custom text selection to only the visible area.

## Best solution

Implement drag-to-auto-scroll in the chat list widget (likely in `src/vs/workbench/contrib/chat/browser/widget/chatListWidget.ts` or the underlying tree/list infrastructure). This would require detecting mouse-drag near viewport edges during text selection and programmatically scrolling. The chat area team should evaluate whether this belongs in the base list/tree widget or as chat-specific behavior.
