# Issue #203627 — Merge Editor creates empty file instead of deleting file when accepting Incoming delete (Only Incoming, no Current)

[https://github.com/microsoft/vscode/issues/203627](https://github.com/microsoft/vscode/issues/203627)

## Summary

Still present on main: the merge editor's accept flow only saves the result text model content to disk — it has no code path to delete the file when accepting an incoming deletion, resulting in an empty file instead.

## Change summary

Accepting an incoming file-deletion in the merge editor produces a zero-byte file instead of actually deleting the file.

## Best solution

The merge editor's `accept()` method (in `mergeEditorInputModel.ts`) needs to detect when the accepted merge result represents a file deletion and call the file service's delete rather than saving an empty buffer. This likely requires the merge editor model to track whether the conflict involves a full-file deletion (from git's modify/delete conflict type) and propagate that to the accept flow.
