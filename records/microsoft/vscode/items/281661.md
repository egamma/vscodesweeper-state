# Issue #281661 — Word wrap doesn't work for long lines by default (unless disabling large file optimizations)

[https://github.com/microsoft/vscode/issues/281661](https://github.com/microsoft/vscode/issues/281661)

## Summary

Large file optimizations intentionally disable wrapping for very large models, and current main already shows a notification explaining that wrapping is turned off with an action to force-enable features. This should be closed as designed rather than tracked as an editor wrapping regression.

## Change summary

Alt+Z does not enable word wrap for long lines in files opened with large file optimizations enabled.

## Best solution

Keep wrapping disabled by default under large file optimizations to avoid memory and responsiveness problems; any further work should be a separate focused UX issue if the existing large-file notification is not visible enough.

  > Thanks for the report. Large file optimizations intentionally disable wrapping for very large files to avoid memory and responsiveness problems. Given that, I think the current behavior is as designed.

## Evidence

- **intentional large-file behavior:** src/vs/workbench/contrib/codeEditor/browser/largeFileOptimizations.ts shows an info prompt when model.isTooLargeForTokenization() is true: "tokenization, wrapping, folding, codelens, word highlighting and sticky scroll have been turned off" for the large file.
- **existing workaround UI:** The same contribution offers a "Forcefully Enable Features" action that updates editor.largeFileOptimizations to false and tells the user to reopen the file.
- **model-level constraint:** src/vs/editor/common/model/textModel.ts explicitly treats models that are too large for tokenization as unable to word wrap, with the code comment "Cannot word wrap huge files anyways".
- **history:** git blame attributes the notification text mentioning disabled wrapping to commit 17f32eac9dead2e23b6941852be46d504a9ffd6f from 2023-09-13, so the explanatory message is not missing on current main.
