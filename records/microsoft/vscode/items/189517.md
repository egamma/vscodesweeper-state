## Summary

Team-authored issue (MEMBER) — keeping open per policy. The report describes a crash when `autoClosingPairs` entries in a language-configuration.json mix array-of-strings and object formats; the extension-point parser on current main already normalizes both, but the backward-compat `__characterPairSupport` path in mainThreadLanguageFeatures.ts may still pass raw arrays without normalization.
