## Summary

Team-authored issue (CONTRIBUTOR); hard keep-open rule applies. The specific request — delay showing ghost text while completion providers are still in progress — does not appear to be fully addressed by the general flickering fix in 848883ca9cd, which focuses on preserving the current item across result updates rather than suppressing initial display.

## Change summary

Ghost text flickers briefly before suggestion providers finish, causing visual noise when typing triggers both inline completions and the suggest widget.

## Best solution

Add logic in the inline completions model to suppress initial ghost text rendering while completion providers are still loading (the `loading` observable already tracks this state in `inlineCompletionsSource.ts`). The recent flickering fix (848883ca9cd) reduces flicker when results update but doesn't gate the initial display on provider readiness.
