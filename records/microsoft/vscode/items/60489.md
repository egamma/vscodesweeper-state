# Issue #60489 — [html] proper support of XHTML

[https://github.com/microsoft/vscode/issues/60489](https://github.com/microsoft/vscode/issues/60489)

## Summary

This is a longstanding feature request for proper XML-based XHTML parsing. VS Code currently treats .xhtml files as HTML, associating them with the HTML language mode but not providing XML-strict parsing semantics. No fix or implementation has landed on main.

## Change summary

XHTML files should be parsed with XML rules (strict self-closing tags, required attribute quoting, no implicit void elements) rather than HTML5 parsing rules.

## Best solution

Either add an XHTML-aware parsing mode to the HTML language service (likely upstream in vscode-html-languageservice), or treat .xhtml files as XML and leverage the existing XML extension ecosystem. Given complexity, this may be better served by a dedicated XHTML extension or by contributing XML-mode support to the html-language-features server.
