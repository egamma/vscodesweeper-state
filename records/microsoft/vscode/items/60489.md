# Issue #60489 — [html] proper support of XHTML

[https://github.com/microsoft/vscode/issues/60489](https://github.com/microsoft/vscode/issues/60489)

## Summary

Valid feature request for proper XHTML/XML-mode parsing in the HTML language service. No implementation progress found on main; 6 upvotes is below the 20-upvote backlog promotion threshold, but it's already in Backlog so it stays open.

## Change summary

VS Code's HTML extension parses XHTML as HTML rather than XML, causing incorrect handling of self-closing tags, void elements, and attribute quoting rules.

## Best solution

The HTML language service (vscode-html-languageservice) would need an XHTML mode that switches from HTML5 parsing rules to XML-strict parsing when the document declares an XHTML doctype or uses .xhtml extension. This likely belongs upstream in the html-language-service library rather than in this repo directly.
