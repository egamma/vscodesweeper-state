# Issue #251609 — Add option for disabling middle mouse button scroll-lock

[https://github.com/microsoft/vscode/issues/251609](https://github.com/microsoft/vscode/issues/251609)

## Summary

Valid feature request for a hold-to-scroll mode (vs. the current click-lock behavior) — already under active team discussion with the `under-discussion` label; keeping open for maintainer judgment.

## Change summary

Add a setting to allow middle-mouse scrolling only while the button is held, instead of the current click-lock (click to start, click to stop) behavior.

## Best solution

Add an enum or boolean setting (e.g. `editor.scrollOnMiddleClick.mode` with values like `scrollLock` and `holdToScroll`) in `src/vs/editor/common/config/editorOptions.ts`, and branch on it in `src/vs/editor/contrib/middleScroll/browser/middleScrollController.ts` — the mouseUp handler at line 77 already conditionally ends the session when `didScroll` is true; the hold-to-scroll mode would unconditionally end it on release regardless of `didScroll`.
