# Issue #67068 — Find in files in non-existent folder highlights wrong input field on error

[https://github.com/microsoft/vscode/issues/67068](https://github.com/microsoft/vscode/issues/67068)

## Summary

Still reproduces on current main: the validation error from a non-existent folder in 'files to include' is displayed on the search input box instead of the include-pattern input, because `onQueryValidationError` unconditionally calls `searchWidget.searchInput.showMessage()` regardless of which input caused the error.

## Change summary

When a non-existent folder is entered in the 'files to include' field, the red error highlight and message appear on the search expression field instead.

## Best solution

In `searchView.ts`, the `onQueryValidationError` handler (around line 1687) should distinguish validation errors originating from `validateQuery` (folder path errors) and show the message on `this.inputPatternIncludes.inputBox` instead of `this.searchWidget.searchInput`. The query-builder catch (line 1695) can keep showing on `searchInput` since those are search-expression errors.
