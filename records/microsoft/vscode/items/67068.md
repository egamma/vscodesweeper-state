# Issue #67068 — Find in files in non-existent folder highlights wrong input field on error

[https://github.com/microsoft/vscode/issues/67068](https://github.com/microsoft/vscode/issues/67068)

## Summary

Still reproduces on main: the `onQueryValidationError` handler at line 1688 of `searchView.ts` always shows the error on `searchWidget.searchInput` (the search expression field), even when the error is a search-path-not-found validation from the 'files to include' input. Confirmed by a team member in comment 5. A real, small bug that belongs in the search area.

## Change summary

When a non-existent folder is entered in the 'files to include' field during Find in Files, the red error highlight appears on the wrong input (the search expression box) instead of the include-pattern input.

## Best solution

The `onQueryValidationError` callback in `searchView.ts` (line 1687–1690) should distinguish validation errors originating from the include-pattern input and show the error message on `inputPatternIncludes` rather than always on `searchWidget.searchInput`. The `validateQuery` rejection (line 1728) could carry metadata indicating the error source, or the handler could route path-not-found errors to the include input directly.
