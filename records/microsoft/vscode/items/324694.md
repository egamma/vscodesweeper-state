# Issue #324694 — Failed task terminal title is not red when active/focus

[https://github.com/microsoft/vscode/issues/324694](https://github.com/microsoft/vscode/issues/324694)

## Summary

Confirmed visual bug on current main: the generic `.monaco-list:focus .selected .monaco-icon-label { color: inherit !important }` rule in iconlabel.css overrides the error-status foreground color on terminal tabs when focused+selected. An open PR (#324698) already proposes a fix.

## Change summary

Failed task terminal tab title loses its red error color when the tab is focused/selected, reverting to the default foreground.

## Best solution

Review and land the open PR https://github.com/microsoft/vscode/pull/324698, which adds CSS specificity overrides for error/warning status colors in the terminal tabs list so they survive the focused+selected state.
