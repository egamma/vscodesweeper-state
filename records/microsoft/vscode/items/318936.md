# Issue #318936 — Feature Request: Agent mode should support working across multiple workspace folders simultaneously

[https://github.com/microsoft/vscode/issues/318936](https://github.com/microsoft/vscode/issues/318936)

## Summary

Valid feature request with 28 upvotes, already in Backlog. The Agents window currently scopes each session to a single workspace folder (confirmed via `sessionWorkspacePicker.ts` using `folders[0]`); multi-root support remains unimplemented on main.

## Change summary

Agent mode sessions should be able to operate across multiple workspace folders simultaneously, matching the cross-folder context that the normal Copilot Chat panel already supports.

## Best solution

Extend the Agents window's session workspace model (`SessionsWorkspaceContextService`, `sessionWorkspacePicker`, and agent host session creation) to support multiple workspace folders per session, similar to how the standard workbench chat already resolves context across all folders in a multi-root workspace.
