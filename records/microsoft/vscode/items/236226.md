## Summary

Team-authored polish issue on milestone 'On Deck' — hard keep-open. The cleanup is still relevant: several views (Chat, Search, Testing, Comments, Extensions Popular) still set canToggleVisibility to false on current main, and the discussion about which ones should keep that restriction is unresolved.

## Change summary

Audit which views should set canToggleVisibility to false and remove the restriction where it's unnecessary.

## Best solution

Each view owner should evaluate whether their view genuinely needs canToggleVisibility: false (like Explorer does for discoverability reasons) and remove the restriction for views where hiding is safe — likely Chat, Search, Testing, and Comments. The relevant files are chatParticipant.contribution.ts, search.contribution.ts, testing.contribution.ts, mainThreadComments.ts, and extensionsViewlet.ts.
