# Issue #176671 — Translations not getting picked up when extension is loaded via `install extension from location`

[https://github.com/microsoft/vscode/issues/176671](https://github.com/microsoft/vscode/issues/176671)

## Summary

Team-authored feature request for best-effort l10n bundle loading when extensions are installed via 'install extension from location'. No fix has landed; the gap remains on current main.

## Change summary

Extension translations (l10n bundles) are not loaded when an extension is sideloaded via the 'install extension from location' command, only when published to the marketplace.

## Best solution

Add best-effort l10n bundle resolution in `src/vs/workbench/api/common/extHostLocalizationService.ts` for extensions loaded from a local location, falling back to the extension's own `l10n` folder relative to `extensionLocation`. This would improve the developer inner loop for extension localization testing.
