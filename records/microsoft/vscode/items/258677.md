# Issue #258677 — Code in repl suggestion doesn't use monospace font

[https://github.com/microsoft/vscode/issues/258677](https://github.com/microsoft/vscode/issues/258677)

## Summary

Still present on main: the CSS for the simple suggest widget details only applies monospace font to `p code` elements but not to `pre code` blocks. The terminal suggest widget correctly passes font info for the type header, but the docs/details markdown body relies on a CSS rule that misses multi-line code blocks.

## Change summary

Code blocks in the terminal REPL suggest details panel render in the system UI font instead of a monospace font.

## Best solution

Broaden the CSS selector in `src/vs/workbench/services/suggest/browser/media/suggest.css` from `.workbench-suggest-widget .suggest-details p code` to also cover `pre code` and standalone `code` elements, matching the pattern used elsewhere in the workbench. Alternatively, apply the monospace font-family to `.workbench-suggest-widget .suggest-details code` (line 430) which already targets all code elements but only sets border-radius and padding.
