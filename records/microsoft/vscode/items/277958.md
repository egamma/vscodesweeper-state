# Issue #277958 — GitHub.copilot-chat incorrectly contributing Windows PATH components to Linux terminal (via Remote-SSH)

[https://github.com/microsoft/vscode/issues/277958](https://github.com/microsoft/vscode/issues/277958)

## Summary

Still reproduces on main: the Copilot Chat terminal PATH contribution uses the host OS's `path.delimiter` and `globalStorageUri.fsPath` (Windows-style `;` separator and `\` paths) even when the terminal targets a Linux remote via Remote-SSH, corrupting the remote shell's PATH and breaking commands like `code`.

## Change summary

When using Remote-SSH from Windows to Linux, the Copilot Chat extension contributes Windows-style PATH components (semicolons, backslashes) to the Linux terminal, corrupting PATH and breaking CLI commands.

## Best solution

The `updateEnvironmentPath()` method in `extensions/copilot/src/platform/terminal/vscode/terminalServiceImpl.ts` (lines 164–172) must detect whether the terminal targets a remote (non-Windows) environment and use the appropriate path delimiter (`:` vs `;`) and path format. Similarly, the storage locations computed in `copilotCLITerminalIntegration.ts` and `copilotDebugCommandContribution.ts` using `globalStorageUri.fsPath` need to produce POSIX paths when the terminal is remote-Linux. The workaround of running as a workspace extension is not universally available per user reports.
