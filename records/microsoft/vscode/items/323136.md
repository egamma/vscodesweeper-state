## Summary

This is a transient CDN content issue — the embeddings cache resources for the reporter's VS Code version (v1.126) hadn't been published to the CDN yet, or there was a temporary deployment gap. The extension code correctly constructs the URL and gracefully handles 404s. No code bug exists.

## Change summary

Copilot's copilot_searchCodebase tool errors when the remote embeddings CDN returns 404 for the expected cache files.

## Best solution

This is a transient service-side issue, not a code defect. The CDN paths are populated by a deployment pipeline; a 404 means the cache for that specific version/model combination wasn't yet available. The extension already handles this gracefully (logs and returns undefined). If it persists, it's a CDN deployment issue to investigate internally, not a code fix in this repo.
