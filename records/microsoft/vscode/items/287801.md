# Issue #287801 — MCP prompt arguments cannot reliably accept binary file data

[https://github.com/microsoft/vscode/issues/287801](https://github.com/microsoft/vscode/issues/287801)

## Summary

Team-authored issue; the defective code path (UTF-8 decode of binary files with no guard) is confirmed present on current main at line 225 of mcpPromptArgumentPick.ts. Keeping open per team-member rule.

## Change summary

Selecting a binary file for an MCP prompt argument silently corrupts the content via unconditional UTF-8 decoding.

## Best solution

In `src/vs/workbench/contrib/mcp/browser/mcpPromptArgumentPick.ts`, the `case 'file'` branch should either pass the file URI/path as a string (preferred per the issue), or detect non-text files and fall back to the URI. This avoids silent corruption while staying within the MCP protocol's string-only argument constraint.
