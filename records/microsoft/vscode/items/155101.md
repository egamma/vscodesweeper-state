## Summary

Valid feature request for richer JSON array display in the Outline view. Still unimplemented on main — the document symbol logic delegates to vscode-json-languageservice. Low community traction (0 upvotes, one +1 comment after 3 years).

## Change summary

JSON arrays in the Outline view show only numeric indices (0, 1, 2) instead of meaningful value previews.

## Best solution

Enhance the document symbol provider in the upstream `vscode-json-languageservice` to include a value preview (or first property key) for array items, then surface that in the VS Code JSON extension's outline. Alternatively, this could be handled in the VS Code JSON extension layer by post-processing symbols before returning them.
