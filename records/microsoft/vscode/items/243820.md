# Issue #243820 — Force language auto-detect to keep checking a new document when typing a first line even if it finds another language first (looking at you, HTML).

[https://github.com/microsoft/vscode/issues/243820](https://github.com/microsoft/vscode/issues/243820)

## Summary

Valid feature request for continuous firstLine-based language re-evaluation as the user types in untitled documents. The current behavior (firstLine matching runs once at model creation; subsequent ML-based detection may agree with the initial guess) is by design, not a bug. The request asks for an improvement to keep re-evaluating firstLine regex patterns on each keystroke.

## Change summary

Re-run firstLine-based language matching on every first-line edit in untitled documents, so a more specific match (e.g. `<!DOCTYPE stml>`) can override an earlier partial match (e.g. HTML from `<!DOCTYPE`).

## Best solution

Enhance `doAutoDetectLanguage` in `src/vs/workbench/common/editor/textEditorModel.ts` (or the untitled model's content-change handler) to also re-run firstLine regex matching — not just ML-based detection — when the first line changes in an untitled document. This would allow a more specific `firstLine` pattern to supersede an earlier partial match. The `languages-guessing` label is correct; reclassify as `feature-request`.
