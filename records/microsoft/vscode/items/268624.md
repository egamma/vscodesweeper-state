# Issue #268624 — Left-side panel size

[https://github.com/microsoft/vscode/issues/268624](https://github.com/microsoft/vscode/issues/268624)

## Summary

Confirmed bug: multiple users reproduce sidebar size not persisting across folder reopens on current versions (1.118.x). Issue #267466 was closed as a duplicate of this one, making this the canonical issue. No fix has landed on main.

## Change summary

Sidebar panel width resets to default when reopening a folder/workspace instead of restoring the user's previously set size.

## Best solution

Investigate the interaction between the `InitializationStateKey` save/restore flow for `SIDEBAR_SIZE` (StorageScope.PROFILE) and the `applyOverrides` logic at layout.ts:3082 that resets sizes for 'new workspaces'. The override introduced in fd280cf7cfc (Aug 2025) may be too aggressive — it fires on `isNew[StorageScope.WORKSPACE]` but overwrites profile-scoped sizing. A fix likely lives in `src/vs/workbench/browser/layout.ts` in the `LayoutStateModel.applyOverrides` method.
