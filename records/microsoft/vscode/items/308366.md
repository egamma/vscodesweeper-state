# Issue #308366 — applyPatchTool: original content leaks through when trailing newline counts differ

[https://github.com/microsoft/vscode/issues/308366](https://github.com/microsoft/vscode/issues/308366)

## Summary

Confirmed on current main: the three-operation edit logic in `generateUpdateTextDocumentEdit` leaves a gap when `originalTrailing > newTrailing`, leaking original lines. An open PR (vscode-copilot-chat#4668) already addresses this.

## Change summary

When the AI shortens a file and trailing newline counts differ, original content leaks into the result because the replace range uses `lines.length` (new content) instead of covering the full original document.

## Best solution

Land the fix from https://github.com/microsoft/vscode-copilot-chat/pull/4668, which replaces the three-operation approach with a single full-document replace in `extensions/copilot/src/extension/tools/node/applyPatchTool.tsx`.
