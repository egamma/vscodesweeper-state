# Issue #156811 — Activation event for a particular folder opened

[https://github.com/microsoft/vscode/issues/156811](https://github.com/microsoft/vscode/issues/156811)

## Summary

Team-authored feature request for a new activation event that triggers on workspace/folder name patterns, allowing extensions to avoid eager activation via onStartupFinished. Keeping open per team-author rule.

## Change summary

Add an activation event that fires based on the opened folder's name (e.g. a glob/pattern match), so extensions don't need onStartupFinished just to check folder naming conventions.

## Best solution

Extend `workspaceContains` or add a new `workspaceName` activation event in the extension host activation machinery (src/vs/workbench/api/common/extHostExtensionActivator.ts and related). This would let extensions like GistPad activate only when the folder name matches a pattern, avoiding unnecessary startup cost.
