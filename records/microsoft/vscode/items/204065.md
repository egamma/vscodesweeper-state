# Issue #204065 — Syntax highlighting error in PHP function parameter when string literal contains block comment

[https://github.com/microsoft/vscode/issues/204065](https://github.com/microsoft/vscode/issues/204065)

## Summary

Still reproduces on main: the PHP grammar's regex-double-quoted rule matches strings starting with `/*` as regex patterns, breaking highlighting. Two open PRs exist but neither has merged.

## Change summary

PHP syntax highlighting breaks when a string literal contains `/*` because the grammar's regex-double-quoted rule incorrectly matches it as a regex pattern.

## Best solution

Merge a fix to the regex-double-quoted begin pattern in `extensions/php/syntaxes/php.tmLanguage.json` (e.g. adding a negative lookahead `(?!\*)` after the `/`), ideally upstreamed to https://github.com/KapitanOczywisty/language-php as the commenter suggested. PR #317058 is the active attempt.
