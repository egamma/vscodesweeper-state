# Issue #317845 — GitHub Copilot stopping GPT and resubmitting prompt shows ongoing mid-response output from previous submission.

[https://github.com/microsoft/vscode/issues/317845](https://github.com/microsoft/vscode/issues/317845)

## Summary

Acknowledged bug on the On Deck milestone — team members have already triaged and routed it. The milestone is a scheduling decision, so this stays open for the owning team.

## Change summary

Stopping a GPT chat response mid-stream and resubmitting the prompt often shows a continuation of the cancelled output instead of a fresh response.

## Best solution

Investigate whether the cancel-and-resubmit flow properly clears the previous response stream before issuing the new request. The recent fix in commit 3d87c046caa (CancellationToken propagation on chat stop) is related but appears to address error messages, not stale content continuation. The chat response pipeline likely needs to ensure the old stream is fully drained/discarded before the new request's tokens are rendered.
