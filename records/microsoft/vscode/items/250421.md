# Issue #250421 — Reconsider `languageModelToolSets` validation Notifications

[https://github.com/microsoft/vscode/issues/250421](https://github.com/microsoft/vscode/issues/250421)

## Summary

Team-authored feature request discussing UX improvements to languageModelToolSets validation — must remain open for maintainer judgment.

## Change summary

When an extension contributes a tool set referencing tools from an uninstalled extension, the validation notification fires on every VS Code launch rather than gracefully handling the partial availability.

## Best solution

Instead of showing a repeated notification for missing tools from an optional extension dependency, display the tool set in the tools picker as partially available with missing tools listed inline. The chat-tools area owns this decision.
