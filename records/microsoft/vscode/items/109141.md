# Issue #109141 — [json] symlink prevents JSON schema from updating validation

[https://github.com/microsoft/vscode/issues/109141](https://github.com/microsoft/vscode/issues/109141)

## Summary

Still a valid bug — the JSON schema file watcher in the extension client uses `workspace.createFileSystemWatcher` on the symlinked path, and there's no evidence of a fix for symlink-traversal watching on current main. The issue remains relevant.

## Change summary

JSON schema validation does not update when the schema file (accessed via a symlink/junction) is modified, requiring a window reload.

## Best solution

The JSON language features client (`extensions/json-language-features/client/src/jsonClient.ts`) should resolve symlinks to their real paths before setting up file watchers, or the watcher infrastructure should be configured to follow symlinks. Alternatively, the per-schema watcher at line ~476 could resolve the URI to its real filesystem path before creating the `RelativePattern`.
