# Issue #109141 — [json] symlink prevents JSON schema from updating validation

[https://github.com/microsoft/vscode/issues/109141](https://github.com/microsoft/vscode/issues/109141)

## Summary

The JSON schema file watcher uses workspace.createFileSystemWatcher with a glob pattern and RelativePattern on the symlink path, which may not detect changes to the symlink target. The bug is still present on main — no symlink-aware fix has been applied to the JSON extension's schema watching logic.

## Change summary

JSON schema validation does not refresh when the schema file is modified if the schema is referenced through a symlink.

## Best solution

The `ensureFilesystemWatcherInstalled` function in `extensions/json-language-features/client/src/jsonClient.ts` should resolve the real path of the schema URI before installing the file system watcher, so changes to the actual target file trigger the schema change notification.
