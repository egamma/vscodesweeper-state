# Issue #151105 — Some shells don't remember history

[https://github.com/microsoft/vscode/issues/151105](https://github.com/microsoft/vscode/issues/151105)

## Summary

Still valid: VS Code's terminal process shutdown kills shells before they can persist history to disk. The `-i` flag fix (245c0c781c9) was necessary but insufficient; the root cause in node-pty's process-kill logic remains unaddressed.

## Change summary

Shells (Git Bash, zsh, bash) don't save command history when VS Code closes because the terminal processes are killed without graceful shutdown.

## Best solution

Modify node-pty's process shutdown logic to allow shells a brief grace period to flush history before forcibly killing them — likely in the WindowsPtyAgent kill path and equivalent Unix handling. The workaround (`PROMPT_COMMAND='history -a'` for bash, `SHARE_HISTORY` for zsh) is documented but the default experience should work without it.
