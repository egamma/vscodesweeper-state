# Issue #297842 — `/yolo <prompt>` does not disable auto approve

[https://github.com/microsoft/vscode/issues/297842](https://github.com/microsoft/vscode/issues/297842)

## Summary

Team-authored testplan bug — must stay open for maintainer judgment. The `/yolo` slash command is registered with `silent: true` and ignores the `_prompt` parameter, so `/yolo <prompt>` enables auto-approve but never submits the prompt as a chat request. The implementation has evolved significantly since filing (now switches permission modes), but the prompt-forwarding gap appears to persist on current main.

## Change summary

Typing `/yolo <prompt>` should enable auto-approve and then submit the trailing prompt as a chat request, but currently the prompt is silently discarded.

## Best solution

Update the `/yolo` handler in `src/vs/workbench/contrib/chat/browser/chatSlashCommands.ts` to forward the trailing prompt text as a chat request after switching the permission mode, or decide that `/yolo <prompt>` is no longer a supported flow given the command's evolution toward a simple mode toggle.
