# Issue #136890 — Support Bitbucket.org repositories in vscode.dev

[https://github.com/microsoft/vscode/issues/136890](https://github.com/microsoft/vscode/issues/136890)

## Summary

Still a valid, community-backed feature request (58 upvotes, in Backlog). No implementation of Bitbucket repository support in vscode.dev exists on current main — the remote-repository integration is handled by separate extensions (e.g. vscode-remote-repositories-github), and no Bitbucket equivalent has been added. Keeping open.

## Change summary

Allow opening Bitbucket.org repositories in vscode.dev via the 'Open Remote Repository' flow, similar to existing GitHub support.

## Best solution

Implement a Bitbucket remote-repository provider extension (analogous to the GitHub remote-repositories extension) that enables browsing and editing Bitbucket repos in vscode.dev. This likely requires extensibility in the RemoteHub layer (referenced by the closed internal backlog item https://github.com/microsoft/vscode-internalbacklog/issues/2465) to support additional hosting providers. Whether this ships as a first-party extension or is enabled for third-party extensions to build is a product decision.
