# Issue #323076 — "Indentations" to replace tabs and spaces

[https://github.com/microsoft/vscode/issues/323076](https://github.com/microsoft/vscode/issues/323076)

## Summary

This is a valid editor feature request, not a close candidate: it asks for display-only indentation blocks for space-indented files, which current indentation settings do not provide.

## Change summary

Allow users to render full leading-space indentation levels at a preferred visual width without changing the file contents on disk.

## Best solution

Route to the editor area for a design decision on whether the core editor should support a display-only indentation width for leading space runs, likely touching view rendering and cursor-column mapping rather than formatting alone.

- **Triage action:** route-to-area

- **Proposed comment:**

  > Thanks for the context. This looks like a feature request for the core editor: preserving space indentation on disk while rendering complete leading indentation levels at a user-selected visual width. VS Code has related settings for `tabSize`, `indentSize`, and `stickyTabStops`, but they do not currently provide this display-only behavior for existing spaces.

## Evidence

- **request clarified:** The reporter clarified that they want an indent concept that behaves like tab stops while preserving the actual space indentation on disk.
- **current model options:** src/vs/editor/common/model/textModel.ts tracks tabSize, indentSize, insertSpaces, and detectIndentation for model editing/formatting, but detectIndentation currently maps the detected indentation size back into tabSize/indentSize.
- **existing adjacent behavior:** src/vs/editor/common/config/editorOptions.ts has stickyTabStops, described as emulating selection behavior of tab characters when using spaces for indentation, but that is selection behavior rather than a display-only rendering width.
- **rendering/cursor implication:** src/vs/editor/common/core/cursorColumns.ts advances tabs to render tab stops but advances normal spaces by one visible column, so the requested behavior would require editor rendering/cursor-column work rather than only changing indentation formatting settings.
