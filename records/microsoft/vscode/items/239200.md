# Issue #239200 — Character under block cursor within a selection results in that character missing from sticky scroll

[https://github.com/microsoft/vscode/issues/239200](https://github.com/microsoft/vscode/issues/239200)

## Summary

Still reproduces on main. The sticky scroll widget renders all inline decorations from the view line, including cursor-related decorations (block cursor foreground styling). When a block cursor sits within a selection, the cursor's inline decoration hides the character in the sticky scroll header. No fix has been committed since the report.

## Change summary

A character under a block cursor within a selection disappears from the sticky scroll header because cursor inline decorations are rendered there unfiltered.

## Best solution

Filter out cursor-related inline decorations when rendering lines in the sticky scroll widget (`src/vs/editor/contrib/stickyScroll/browser/stickyScrollWidget.ts`), since cursor state is irrelevant in the sticky scroll context.
