# Issue #320249 — Files changed via the copilot cli in terminal in another session pop up when a different session is selected

[https://github.com/microsoft/vscode/issues/320249](https://github.com/microsoft/vscode/issues/320249)

## Summary

Team-authored issue on the On Deck milestone describing a real cross-session editor routing problem in the Agents window; two hard keep-open rules apply (MEMBER author + On Deck milestone). The discussion is active with design considerations around scoping editor opens to sessions.

## Change summary

Files changed by a background agent session (via copilot CLI in terminal) incorrectly pop up as diffs in a different, focused session in the Agents window.

## Best solution

Implement session-scoped editor routing in the Agents window so that file changes triggered by an agent in one session only surface editors within that session's context, while preserving the ability for user-initiated file opens (e.g. from desktop) to appear in the active window. The design discussion in the comments (how to attribute which session triggered an editor open) needs resolution first.
