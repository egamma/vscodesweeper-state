# Issue #320249 — Files changed via the copilot cli in terminal in another session pop up when a different session is selected

[https://github.com/microsoft/vscode/issues/320249](https://github.com/microsoft/vscode/issues/320249)

## Summary

Team-authored issue on a real milestone (On Deck) describing a cross-session editor leak in the Agents window — must stay open for maintainer judgment.

## Change summary

File diffs opened by a background agent session (via copilot CLI in terminal) incorrectly surface in the currently focused/different session.

## Best solution

The editor-opening path in the Agents window needs session affinity: when a file change is detected (e.g. via a terminal-driven edit), it should route to the originating session's editor group rather than the active one. As benibenj notes, this requires a way to attribute editor-open requests to a session, while still allowing user-initiated file opens from outside to land in the focused session.
