# Issue #12787 — [html] closing </script> tag is not proposed

[https://github.com/microsoft/vscode/issues/12787](https://github.com/microsoft/vscode/issues/12787)

## Summary

Team-authored issue requesting closing tag completion (e.g. `</script>`) when typing `</` inside a `<script>` block. The auto-close-on-`>` feature exists, but this is about completion suggestions for manually typed `</`. Must keep open per team-author rule.

## Change summary

Typing `</` inside a `<script>` tag should offer `</script>` as a completion suggestion.

## Best solution

The HTML language service's `doComplete2` (via vscode-html-languageservice) should propose the matching closing tag when the cursor is after `</` inside an unclosed element like `<script>`. This likely needs enhancement in the upstream `vscode-html-languageservice` package's completion provider to handle this case within embedded content regions.
