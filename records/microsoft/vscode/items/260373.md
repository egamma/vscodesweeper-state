# Issue #260373 — Continuation prompt not handled properly in python shell integration script

[https://github.com/microsoft/vscode/issues/260373](https://github.com/microsoft/vscode/issues/260373)

## Summary

Team-filed bug: Python's continuation prompt (`...`) leaks into the terminal tab description because the prompt input model doesn't handle Python's `sys.ps2` the way it handles bash/zsh/pwsh continuation prompts. The issue is still open on main — bash/zsh/pwsh report their continuation prompt via OSC 633;P;ContinuationPrompt, but Python REPL sessions do not, so the prompt input model cannot strip it.

## Change summary

Python's `...` continuation prompt appears verbatim in the terminal tab title/description because VS Code's shell integration doesn't receive or handle it for Python shells.

## Best solution

Extend the terminal prompt input model or Python shell detection to recognize and strip Python's `...` continuation prompt from the command line value shown in the tab description — likely in `src/vs/platform/terminal/common/capabilities/commandDetection/promptInputModel.ts` where continuation prompt trimming logic lives.
