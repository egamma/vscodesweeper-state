# Issue #325546 — Integrated terminal requires manual restart on every VS Code startup

[https://github.com/microsoft/vscode/issues/325546](https://github.com/microsoft/vscode/issues/325546)

## Summary

Verified duplicate of #307601 — both report the same root cause: restored/persistent terminals skip the automatic environment-changes relaunch (guarded by `!attachPersistentProcess` in terminalInstance.ts:2335), so the manual-restart warning reappears on every startup.

## Change summary

Integrated terminal shows a manual-restart warning for extension environment contributions on every VS Code startup because restored terminals are excluded from auto-relaunch.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/307601 which already tracks the repeated relaunch warning for restored terminals. The fix belongs in `src/vs/workbench/contrib/terminal/browser/terminalInstance.ts` — the `attachPersistentProcess` exclusion in `_refreshEnvironmentVariableInfoWidgetState` needs revisiting so that restored terminals can also be auto-relaunched when uninteracted.
