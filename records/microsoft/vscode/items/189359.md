# Issue #189359 — search results return much less than search.maxResults with sibling clause

[https://github.com/microsoft/vscode/issues/189359](https://github.com/microsoft/vscode/issues/189359)

## Summary

Team-authored confirmed bug, still present on main. The sibling-clause filtering happens asynchronously after ripgrep results are counted toward maxResults, causing premature search termination. No fix commit found.

## Change summary

Search results return far fewer than search.maxResults when the query involves sibling clauses (e.g. multi-root workspace with path-prefixed include globs), because results filtered out by async sibling resolution are still counted toward the limit.

## Best solution

Defer the maxResults limit check until after sibling-clause filtering completes, or exclude sibling-filtered results from the running count. The fix likely touches `src/vs/workbench/services/search/common/textSearchManager.ts` (the `progress.report` callback around line 140–170) and possibly `src/vs/workbench/services/search/node/ripgrepTextSearchEngine.ts` where the result limit is enforced.
