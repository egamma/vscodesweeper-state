# Issue #189359 — search results return much less than search.maxResults with sibling clause

[https://github.com/microsoft/vscode/issues/189359](https://github.com/microsoft/vscode/issues/189359)

## Summary

Still reproduces on main: the ripgrep engine counts raw results toward maxResults before async sibling-clause filtering removes non-matching ones, so the user sees far fewer results than the limit. Team-authored issue, keeping open.

## Change summary

Search results return far fewer than search.maxResults when the include pattern triggers async sibling-clause evaluation, because ripgrep hits the limit before filtering.

## Best solution

The fix needs to either defer the maxResults limit check until after sibling-clause filtering, or over-fetch from ripgrep when sibling clauses are active and trim afterward. The relevant code path is in `src/vs/workbench/services/search/common/textSearchManager.ts` (the `onResult` callback at ~line 66 that counts and cancels) and `src/vs/workbench/services/search/node/ripgrepTextSearchEngine.ts` (the `RipgrepParser` numResults limit at ~line 326).
