## Change summary

The working set UI inside chatInputPart has structural debt: DOM teardown/recreation on session change, querySelector usage, type-bending in the edit list model, and an oversized render method.

## Best solution

Refactor the working set rendering in chatInputPart: hold DOM references instead of querySelector, introduce a dedicated working set entry type (eliminating IChatReferenceListItem's state property hack), extract renderChatEditingSessionState into a proper component, and avoid full DOM teardown on session transitions.
