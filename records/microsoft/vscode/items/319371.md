# Issue #319371 — cursoring down through Arabic letters in parenthesis is odd

[https://github.com/microsoft/vscode/issues/319371](https://github.com/microsoft/vscode/issues/319371)

## Summary

Legitimate RTL cursor navigation bug when moving through Arabic text containing parentheses — no fix found on current main. The editor's cursor movement logic doesn't properly handle neutral BiDi characters (parentheses) embedded in RTL text, causing the cursor to jump unexpectedly.

## Change summary

Cursor up/down/left/right navigation behaves erratically when moving through Arabic text that contains parentheses.

## Best solution

Fix cursor movement in `src/vs/editor/common/cursor/cursorMoveOperations.ts` (and possibly the view-model cursor mapping) to correctly handle BiDi neutral characters like parentheses within RTL runs. This is a long-standing complexity area; the fix likely requires respecting the resolved BiDi level of neutral characters when computing visual cursor positions.
