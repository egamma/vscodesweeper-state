# Issue #318381 — Add a setting to not resolve shell configuration and not inherit environment from cli

[https://github.com/microsoft/vscode/issues/318381](https://github.com/microsoft/vscode/issues/318381)

## Summary

Valid feature request: a user-facing setting to disable shell environment resolution exists only as an undocumented CLI flag (`--force-disable-user-env`) but has no counterpart in VS Code settings. Routing to the workbench area for consideration.

## Change summary

Add a user-facing setting to completely disable shell environment resolution, complementing the existing `--force-disable-user-env` CLI flag and `application.shellEnvironmentResolutionTimeout` timeout setting.

## Best solution

Expose the existing `--force-disable-user-env` behavior as a boolean application-scope setting (e.g. `application.shellEnvironmentResolution` with an option to disable) in `src/vs/workbench/electron-browser/desktop.contribution.ts`, and have `getResolvedShellEnv` in `src/vs/platform/shell/node/shellEnv.ts` check it alongside the CLI flag. The `terminal.integrated.inheritEnv` setting already handles terminal-specific env inheritance separately.
