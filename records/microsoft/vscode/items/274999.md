# Issue #274999 — Make zero-based offsets the default?

[https://github.com/microsoft/vscode/issues/274999](https://github.com/microsoft/vscode/issues/274999)

## Summary

Active design discussion between team contributors about whether the quick-open offset feature should default to zero-based or one-based indexing. TylerLeonhardt gave a rationale for one-based (symmetry with -1 for last line); jrieken countered with String#at precedent. Leaving open as under-discussion pending further community signal.

## Change summary

Change the quick-open offset-based revealing to use zero-based offsets by default instead of one-based.

## Best solution

Wait for community feedback on whether zero-based offsets feel more natural to users coming from debug sessions or tooling output. If consensus forms, the change would be in the quick-open reveal logic. The current `under-discussion` label is appropriate.
