# Issue #317682 — [Memory Leak] dwm.exe memory spikes to 400MB+ and remains orphaned after closing VS Code (AMD RDNA1)

[https://github.com/microsoft/vscode/issues/317682](https://github.com/microsoft/vscode/issues/317682)

## Summary

The dwm.exe memory leak is an Electron/Chromium GPU-compositor issue specific to AMD RDNA1 on Windows, not a VS Code bug. VS Code 1.121 shipped Electron 39.8.8; the reporter confirms Insiders (which carried Electron 42) does not reproduce. Since main now ships Electron 42.5.0 (from 1.123.0 onward), the fix has already shipped in Stable.

## Change summary

After closing VS Code on an AMD RDNA1 GPU, dwm.exe retains ~400MB of leaked GPU memory indefinitely.

## Best solution

This was resolved by the Electron 42 upgrade (commit 8748be1f1a8, shipped in 1.123.0). The reporter should update to VS Code 1.123+ to pick up the fix. If the leak persists on the latest Stable, a new issue with updated diagnostics would be appropriate. No VS Code code change is needed — the root cause was in Chromium's GPU resource cleanup.
