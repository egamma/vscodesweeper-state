# Issue #324602 — Automatic marking as unreviewed on edits is very slow

[https://github.com/microsoft/vscode/issues/324602](https://github.com/microsoft/vscode/issues/324602)

## Summary

Team-filed bug reporting slow 'unreviewed' state updates on manual edits; the changeset recompute pipeline uses a 5-second debounce which likely explains the delay. Keeping open as a MEMBER-authored issue.

## Change summary

Manually editing a reviewed file in the Agents Changes view takes several seconds to unset the 'reviewed' state, where it should be instant.

## Best solution

The root cause is likely `AgentHostChangesetService._DIFF_DEBOUNCE_MS = 5000` in `src/vs/platform/agentHost/node/agentHostChangesetService.ts`, which debounces diff recomputation by 5 seconds. For user-initiated edits (as opposed to mid-turn tool edits), the changeset refresh path should either bypass the debounce or use a much shorter delay, so the reviewed-state context key updates near-instantly. Alternatively, the client-side optimistic update in `changesActions.ts` could detect local saves and eagerly remove the file from `clientReviewedFilesObs` without waiting for the server round-trip.
