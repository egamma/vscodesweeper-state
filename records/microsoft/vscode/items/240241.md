# Issue #240241 — Terminal suggest: Support lazy loading of fig specs via an index

[https://github.com/microsoft/vscode/issues/240241](https://github.com/microsoft/vscode/issues/240241)

## Summary

Still unimplemented on main: all fig specs are eagerly loaded via `upstreamSpecs.ts` static re-exports into `availableSpecs` at module load time. The requested lazy-loading index pattern has not been introduced. Team-authored issue — keeping open.

## Change summary

Replace eager import of all fig specs with a lightweight index (name+description only) and load full specs on demand via dynamic import when argument completions are requested.

## Best solution

Generate an index file mapping command names to descriptions (for the initial completions list) and use dynamic `import()` in `terminalSuggestMain.ts` to load full specs only when subcommand/argument completions are needed. This reduces activation cost and memory as the spec count grows.
