# Issue #300442 — Auto reveal scrolls to wrong file in multi window context

[https://github.com/microsoft/vscode/issues/300442](https://github.com/microsoft/vscode/issues/300442)

## Summary

Verified duplicate of #200452 — both report that explorer auto-reveal scrolls to the wrong file in a multi-window setup when focus shifts between windows. The canonical issue is open and under discussion by maintainers.

## Change summary

Explorer auto-reveal scrolls to the visible-in-viewport file of the main window when interacting with the file explorer after focusing an auxiliary window, disrupting the user's browsing.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/200452, which tracks the same auto-reveal-in-multi-window behavior. The fix would likely guard `selectActiveFile()` in `explorerView.ts` to skip reveals triggered by cross-window focus changes, or scope `onDidActiveEditorChange` to same-window events.
