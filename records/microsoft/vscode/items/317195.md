# Issue #317195 — Notify when Copilot agent sessions need attention or finish

[https://github.com/microsoft/vscode/issues/317195](https://github.com/microsoft/vscode/issues/317195)

## Summary

Team-authored feature request with an open PR (https://github.com/microsoft/vscode/pull/322839) actively implementing it. Current main already has partial notification support via `ChatWindowNotifier` (needs-input and response-received notifications with `chat.notifyWindowOnConfirmation` / `chat.notifyWindowOnResponseReceived` settings), but the request covers broader session lifecycle notifications (finish, failure, blocked). Keeping open — team member issue with active implementation work.

## Change summary

Add OS notifications for Copilot agent session state changes (needs input, finished, failed/blocked) with click-through to the correct window and session.

## Best solution

Land the open PR https://github.com/microsoft/vscode/pull/322839 which adds notification hooks for chat sessions. The existing `ChatWindowNotifier` in `src/vs/workbench/contrib/chat/browser/chatWindowNotifier.ts` already handles needs-input notifications; the PR extends coverage to completion and failure states. The author should verify the PR covers all requested triggers (success, failure, blocked) and the click-through routing.
