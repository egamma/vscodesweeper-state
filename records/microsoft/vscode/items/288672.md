# Issue #288672 — macOS system file dialog always appends `.text` extension when saving new untitled files

[https://github.com/microsoft/vscode/issues/288672](https://github.com/microsoft/vscode/issues/288672)

## Summary

Correctly triaged as upstream. The VS Code dialog code falls back to the Plain Text filter (`.txt`) for untitled files without an extension, and macOS's NSSavePanel enforces that filter's extension on save. The existing labels (`upstream`, `electron`, `macos`, `native-file-dialog`) are accurate — this is an interaction between VS Code's filter construction and macOS/Electron's NSSavePanel behavior. Keep open for tracking.

## Change summary

macOS native save dialog forcibly appends `.text` when saving new untitled files because VS Code defaults the file-type filter to Plain Text for extensionless files.

## Best solution

The root fix likely belongs in how Electron maps the filter list to NSSavePanel's `allowedContentTypes` / `allowedFileTypes`. On the VS Code side, a possible workaround would be to not default to the Plain Text filter when the file has no extension, or to ensure 'All Files' is the active (selected) filter rather than just the first in the list — but this risks regressions against https://github.com/microsoft/vscode/issues/102713 and https://github.com/microsoft/vscode/issues/115860. The upstream Electron issue should be filed or tracked if not already.
