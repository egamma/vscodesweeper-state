The discussion identifies a real editor clipboard feature request: preserve tabs in the HTML clipboard format so Word can paste syntax-colored code without converting indentation to spaces.

## Change summary

Add a setting or behavior change so VS Code can keep tab characters in HTML clipboard content instead of expanding them to spaces.

## Best solution

The implementation should preserve current alignment behavior by default unless the editor team intentionally changes compatibility with rich-text paste targets such as Word.
