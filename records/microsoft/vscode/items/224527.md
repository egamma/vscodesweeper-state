# Issue #224527 — Git - During repo initialization a gitignore file should be created

[https://github.com/microsoft/vscode/issues/224527](https://github.com/microsoft/vscode/issues/224527)

## Summary

Valid feature request — the git.init command on current main (extensions/git/src/commands.ts) performs a bare `git init` with no .gitignore creation or offer. The capability does not exist yet, and the request has low traction (2 upvotes, no comments in ~2 years). Keeping open for the community process.

## Change summary

When initializing a new git repository via Source Control, VS Code should offer to create a .gitignore file if one does not already exist.

## Best solution

Add a post-init step in `extensions/git/src/commands.ts` (the `git.init` command, ~line 1179) that checks for a .gitignore and optionally creates one — either from a user-selected template (GitHub's gitignore templates API) or a configurable default via a `git.defaultGitignore` setting. This is a reasonable enhancement to the built-in git extension rather than an extension-candidate, since it augments the existing init flow.
