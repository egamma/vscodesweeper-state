# Issue #296237 — Error 'Unable to lauch browser: "Unable to attach to browser"' after every update

[https://github.com/microsoft/vscode/issues/296237](https://github.com/microsoft/vscode/issues/296237)

## Summary

Confirmed recurring bug in the bundled js-debug extension: browser debug launch fails on the first session after a VS Code update on Windows, requiring a second restart. Multiple users reproduce it consistently. The root cause lives in ms-vscode.js-debug and is tracked upstream at microsoft/vscode-js-debug#2249.

## Change summary

Browser debug launch (msedge/chrome) fails with 'Unable to attach to browser' on the first run after a VS Code update, requiring an additional restart to resolve.

## Best solution

The fix belongs in the bundled ms-vscode.js-debug extension. The open upstream issue https://github.com/microsoft/vscode-js-debug/issues/2249 tracks this. The debug team should investigate whether the js-debug adapter's browser-attach path fails when the extension binary was just updated in-place (possibly a stale path or lock issue on Windows with non-admin installs). Coordinate with the js-debug maintainers to land a fix there.
