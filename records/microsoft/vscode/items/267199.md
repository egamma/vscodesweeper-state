## Summary

Intermittent ERR_HTTP2_PROTOCOL_ERROR during Copilot chat requests is tracked internally (vscode-internalbacklog#5985) with significant infrastructure work already done (fetcher fallback, retry improvements). Keeping open as it still affects users and has 8+ upvotes, though the root cause spans server-side infrastructure and client-side retry behavior.

## Change summary

Copilot chat requests intermittently fail with net::ERR_HTTP2_PROTOCOL_ERROR, with excessively long retry delays (~10 minutes) before fallback.

## Best solution

The internal tracking issue (microsoft/vscode-internalbacklog#5985) owns this work. Client-side improvements (faster retry/fallback via FetcherService, node-fetch fallback on network process crash) have shipped. Remaining gap is the long stall before retry on HTTP2 protocol errors specifically — the fetcher should detect and retry/fallback faster for this error class. A maintainer should confirm whether recent fetcher changes (d4f9754abda, b2590e0b22b) already address the timeout concern raised in comment 2.
