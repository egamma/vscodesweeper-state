# Issue #242620 — Git - keyboard shortcut of `git.stage` can't add unstaged image files to staged area

[https://github.com/microsoft/vscode/issues/242620](https://github.com/microsoft/vscode/issues/242620)

## Summary

Confirmed real bug on current main: `git.stage` via keyboard shortcut silently fails for image files because the fallback `getSCMResource()` relies on `window.activeTextEditor` (null for non-text files), and additionally `getSCMResource()` doesn't search `untrackedGroup`. Issue has milestone 'On Deck', so keeping open per policy.

## Change summary

The `git.stage` keyboard shortcut cannot stage image files because the command's fallback to resolve the target resource uses `window.activeTextEditor`, which is null for non-text file previews.

## Best solution

Two fixes needed in `extensions/git/src/commands.ts`: (1) Add `untrackedGroup` to the resource search in `getSCMResource()` (~line 5728). (2) When `window.activeTextEditor` is null, fall back to the SCM view's currently focused/selected resource state (e.g., via `window.tabGroups` active tab URI or the SCM view selection API) so non-text files like images can be staged via keyboard shortcut.
