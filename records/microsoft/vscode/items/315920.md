# Issue #315920 — Upgrade and top-up prompts lack usage-based reasoning

[https://github.com/microsoft/vscode/issues/315920](https://github.com/microsoft/vscode/issues/315920)

## Summary

Valid feature request to add usage-based reasoning to upgrade prompts and detect repeated top-ups as a signal to recommend plan changes. Not implemented on current main — the upgrade path is a simple button without usage context.

## Change summary

Upgrade and top-up prompts should show usage history and detect repeated purchases to proactively recommend a better plan with cost comparison.

## Best solution

Enhance the quota-exceeded and upgrade UX in `src/vs/workbench/contrib/chat/browser/chatQuotaNotification.ts` and `chatQuotaExceededPart.ts` to surface usage patterns (e.g. cumulative spend, top-up frequency) and recommend plan changes when repeated top-ups are detected. Requires integration with entitlement/billing data from the Copilot API.
