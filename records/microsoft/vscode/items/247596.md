# Issue #247596 — http/sse server should not should use stdio/process language

[https://github.com/microsoft/vscode/issues/247596](https://github.com/microsoft/vscode/issues/247596)

## Summary

Still present on main: the MCP code lens provider in mcpLanguageFeatures.ts uses the same process-oriented language (Running/Stop/Restart) for all server types without distinguishing HTTP/SSE transport, which should use connection-oriented language (Connected/Disconnect/Reconnect).

## Change summary

HTTP/SSE MCP servers show process-oriented code lens labels (Running/Stop/Restart) instead of connection-oriented ones (Connected/Disconnect/Reconnect).

## Best solution

In `src/vs/workbench/contrib/mcp/browser/mcpLanguageFeatures.ts`, check the server's transport type and use connection-oriented labels (Connected/Disconnect/Reconnect) for HTTP/SSE servers while keeping the current labels for stdio servers.
