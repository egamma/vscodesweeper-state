Team-authored perf bug that has seen significant iterative improvement (layout thrashing fixes, lazy rendering, height caching, leak fixes) but remains open — the underlying incremental markdown rendering issue (#275671) is still unresolved, and the assignee confirmed in March 2026 that 'still has a few issues' remain. ## Change summary

Excessive updateThinking calls during chat thinking-section streaming cause redundant HTML sanitization, string parsing, and style recalculation, degrading renderer performance.

## Best solution

Continue the work in `chatThinkingContentPart.ts` toward true incremental/batched markdown rendering so that streaming thinking tokens avoid O(n²) re-rendering. The broader fix depends on https://github.com/microsoft/vscode/issues/275671 (incremental markdown in chat), which is still open.
