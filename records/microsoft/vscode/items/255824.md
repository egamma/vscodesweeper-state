# Issue #255824 — Titlebar overlay is not enabled

[https://github.com/microsoft/vscode/issues/255824](https://github.com/microsoft/vscode/issues/255824)

## Summary

Still reproducible on main: `updateWindowControls` calls `win.setTitleBarOverlay()` without a try/catch, and the configuration-level check (`useWindowControlsOverlay`) can be true even when a specific guest/auxiliary BrowserWindow wasn't created with the `titleBarOverlay` option — Electron throws in that case.

## Change summary

Electron throws "Titlebar overlay is not enabled" when `setTitleBarOverlay()` is called on a guest window that wasn't created with `titleBarOverlay` enabled.

## Best solution

Guard the `win.setTitleBarOverlay()` call in `BaseWindow.updateWindowControls` (src/vs/platform/windows/electron-main/windowImpl.ts:441) with a try/catch, or check whether the window actually has overlay enabled before calling. The root cause is that `useWindowControlsOverlay(configurationService)` is a global config check but doesn't reflect whether the specific BrowserWindow instance was created with the overlay option.
