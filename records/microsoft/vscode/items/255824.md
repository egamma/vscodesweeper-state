# Issue #255824 — Titlebar overlay is not enabled

[https://github.com/microsoft/vscode/issues/255824](https://github.com/microsoft/vscode/issues/255824)

## Summary

Still reproducible from source on main: when an auxiliary (guest) window is opened without the `titleBarOverlay` BrowserWindow option but the user's configuration enables window controls overlay, `setTitleBarOverlay()` throws. The guard checks the user setting, not whether the specific window was created with the option.

## Change summary

Electron throws "Titlebar overlay is not enabled" when `setTitleBarOverlay()` is called on an auxiliary/guest window that was not created with the `titleBarOverlay` option.

## Best solution

Add a try/catch or a capability check in `updateWindowControls` (around line 441 in `windowImpl.ts`) that catches the Electron error when `setTitleBarOverlay` fails, or verify from the window's creation options that `titleBarOverlay` was actually set before calling the API. The Positron cross-reference confirms this hits Linux/Wayland users.
