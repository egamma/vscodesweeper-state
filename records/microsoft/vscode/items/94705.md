# Issue #94705 — Search does not respect `workbench.editor.revealIfOpen`

[https://github.com/microsoft/vscode/issues/94705](https://github.com/microsoft/vscode/issues/94705)

## Summary

Still reproduces on main: the search view hardcodes `revealIfVisible: true` when opening editors, overriding the user's `workbench.editor.revealIfOpen` setting. Confirmed by team members, with duplicates closed against this issue. Needs a product decision on the right approach (new setting vs. conditional behavior).

## Change summary

Clicking a search result reveals the file in an unfocused editor group instead of opening a new editor in the focused group, ignoring the `workbench.editor.revealIfOpen: false` setting.

## Best solution

Make the `revealIfVisible: true` in `src/vs/workbench/contrib/search/browser/searchView.ts` conditional on the user's `workbench.editor.revealIfOpen` setting, or introduce a new setting as discussed in the thread. The team discussion (roblourens, sandy081, bpasero) shows this needs a deliberate product decision about whether Search should override user preferences — the fix is localized but the design question is broader (Problems view, Quick Open, etc. have the same override).
