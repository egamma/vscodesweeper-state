# Issue #94705 — Search does not respect `workbench.editor.revealIfOpen`

[https://github.com/microsoft/vscode/issues/94705](https://github.com/microsoft/vscode/issues/94705)

## Summary

Still reproduces on main: the search view hardcodes `revealIfVisible: true` when opening editors, which overrides the user's `workbench.editor.revealIfOpen: false` setting. An open (but stale) PR #134525 exists that attempted to address the broader problem.

## Change summary

Clicking a search result reveals the file in an unfocused editor group even when `workbench.editor.revealIfOpen` is set to false.

## Best solution

Remove or conditionalize the hardcoded `revealIfVisible: true` in `src/vs/workbench/contrib/search/browser/searchView.ts` (line ~2249) so that the search view respects the user's `workbench.editor.revealIfOpen` setting. The same pattern exists in `openEditorWithMultiCursor`. Note PR https://github.com/microsoft/vscode/pull/134525 is open and attempts a broader fix — a maintainer should decide whether to land that PR, close it in favor of a narrower search-only fix, or introduce a new setting.
