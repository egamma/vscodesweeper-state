# Issue #191649 — Moved lines aren't indicated to screen reader users

[https://github.com/microsoft/vscode/issues/191649](https://github.com/microsoft/vscode/issues/191649)

## Summary

Team-authored accessibility issue for diff editor moved-range indicators; no screen-reader announcement has been implemented on current main — the gap remains.

## Change summary

Screen reader users receive no indication when their cursor enters a moved code range in the diff editor.

## Best solution

When a user navigates into a moved range (especially via 'Go to Next/Previous Change'), emit an ARIA alert such as 'Code moved from line X to Y' and expose a keybinding for the compare action. Implementation likely lives in `src/vs/editor/browser/widget/diffEditor/features/movedBlocksLinesFeature.ts` and/or the diff editor's accessible view contribution.
