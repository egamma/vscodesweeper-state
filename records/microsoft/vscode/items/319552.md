# Issue #319552 — Windows: PowerShell Store detection throws ENOENT when scanning %LOCALAPPDATA%\Microsoft\WindowsApps

[https://github.com/microsoft/vscode/issues/319552](https://github.com/microsoft/vscode/issues/319552)

## Summary

Still reproduces on main: `findPSCoreMsix()` in `src/vs/base/node/powershell.ts` has a TOCTOU race — `existsDirectory` passes but the subsequent unguarded `readdir` throws ENOENT if the WindowsApps directory is absent or inaccessible at read time. The error bubbles up and aborts shell detection.

## Change summary

PowerShell Store detection throws ENOENT on `readdir` of `%LOCALAPPDATA%\Microsoft\WindowsApps` when that directory doesn't exist or becomes inaccessible after the existence check.

## Best solution

Wrap the `pfs.Promises.readdir(msixAppDir)` call on line 212 in a try/catch that returns `null` on ENOENT/EACCES, so detection falls through to subsequent PowerShell candidates gracefully.
