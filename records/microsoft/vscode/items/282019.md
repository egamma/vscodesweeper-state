# Issue #282019 — NES Disable "import package" Feature with Far Away Edits Turned On

[https://github.com/microsoft/vscode/issues/282019](https://github.com/microsoft/vscode/issues/282019)

## Summary

Active UX discussion about how LLM-based NES (Cursor Jump) and heuristic-based import NES should coexist — two team members are engaged with concrete ideas (NRS-style rendering, priority ordering). Keeping open as under-discussion.

## Change summary

When far-away edits (Cursor Jump) are enabled, heuristic import suggestions conflict with LLM NES suggestions, creating a confusing UX where the user must accept a false-positive import to reach the correct far-away edit.

## Best solution

Design a priority/coexistence model for LLM NES and import NES — e.g. render both with NRS-style shift-tab disambiguation, or suppress import NES when LLM NES covers the same symbol. The NES-ux area owns this.
