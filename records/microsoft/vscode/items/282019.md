# Issue #282019 — NES Disable "import package" Feature with Far Away Edits Turned On

[https://github.com/microsoft/vscode/issues/282019](https://github.com/microsoft/vscode/issues/282019)

## Summary

Active UX discussion between the reporter and two team members about how NES should prioritize LLM-based suggestions over heuristic import suggestions; already correctly labeled under-discussion with a stated plan to revisit in a future iteration.

## Change summary

When far-away edits are enabled, heuristic-based import NES can conflict with or obscure the stronger LLM-based NES suggestion, requiring users to accept an irrelevant import to reach the actual far-away edit.

## Best solution

Resolve the prioritization design: when both an LLM NES and an import NES target the same context, either suppress the import suggestion or render both with a secondary accept gesture (like shift-tab, similar to NRS). The NES-ux area owns this decision.
