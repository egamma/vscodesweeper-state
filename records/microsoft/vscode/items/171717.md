# Issue #171717 — Bug blocking texts with ctrl+shift+arrow

[https://github.com/microsoft/vscode/issues/171717](https://github.com/microsoft/vscode/issues/171717)

## Summary

Confirmed bug: word-extending selection (Ctrl+Shift+Arrow) behaves incorrectly when accessibility mode is enabled on Windows — the selection jumps back to the original anchor instead of extending from the current position. Team reproduced and identified the accessibility word navigation path as the cause but no fix has landed.

## Change summary

When accessibility mode is on, extending a character-by-character selection with Ctrl+Shift+Arrow resets the selection to the initial cursor instead of continuing from the current position.

## Best solution

Fix the accessibility word navigation path so that extending an existing selection with Ctrl+Shift+Arrow properly continues from the current selection end rather than resetting. The fix likely involves how the accessibility word commands interact with the native textarea selection state in `src/vs/editor/contrib/wordOperations/browser/wordOperations.ts` and possibly the editor's accessibility input handling.
