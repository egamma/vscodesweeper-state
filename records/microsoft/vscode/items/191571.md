# Issue #191571 — Match Whole Words modifier has no effect for ipynb

[https://github.com/microsoft/vscode/issues/191571](https://github.com/microsoft/vscode/issues/191571)

## Summary

Still reproduces on main: the closed-notebook search path in CellSearchModel hardcodes all search modifiers to off, ignoring wholeWord/matchCase/isRegex from the query. Team-authored issue — keeping open.

## Change summary

The "Match Whole Words" modifier in workspace search has no effect when searching closed notebook files, returning too many results.

## Best solution

Propagate the full set of search modifiers (isWordMatch, isCaseSensitive, isRegExp) from the ITextQuery through to CellSearchModel.findInInputs/findInOutputs instead of hardcoding them to false/null. The fix touches src/vs/workbench/contrib/search/common/cellSearchModel.ts (method signatures and SearchParams construction) and src/vs/workbench/api/common/extHostNotebook.ts (the caller that currently only passes `pattern`).
