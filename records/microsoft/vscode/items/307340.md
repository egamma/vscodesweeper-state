# Issue #307340 — [microsoft-authentication] Sovereign cloud (ChinaCloud) tokens not persisted in msal-no-broker mode — shared PCA cache between microsoft and microsoft-sovereign-cloud providers

[https://github.com/microsoft/vscode/issues/307340](https://github.com/microsoft/vscode/issues/307340)

## Summary

Confirmed in source: both microsoft and microsoft-sovereign-cloud providers share a single PCA instance keyed only by clientId (no authority differentiation), causing silent token acquisition to fail for sovereign clouds in msal-no-broker mode. No fix has landed on main. The maintainer acknowledged the context but hasn't closed it.

## Change summary

Sovereign cloud tokens are not persisted in msal-no-broker mode because global and sovereign providers share the same PCA cache entry, causing acquireTokenSilent to fail with authority mismatch.

## Best solution

Differentiate PCA cache keys by authority/environment (e.g. `pca:{clientId}:{authorityHost}`) so sovereign-cloud tokens are stored and retrieved independently from global tokens. The fix lives in `extensions/microsoft-authentication/src/node/cachedPublicClientApplication.ts` (constructor) and `extensions/microsoft-authentication/src/node/publicClientCache.ts` (the Map key and `getOrCreate` signature).
