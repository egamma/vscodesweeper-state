# Issue #271526 — RFC: TabInputCustomDiff

[https://github.com/microsoft/vscode/issues/271526](https://github.com/microsoft/vscode/issues/271526)

## Summary

Legitimate API gap: custom editors lack a diff-aware TabInput type, unlike text and notebook editors which both have diff counterparts. The feature has not been implemented on main. A team member has engaged but no resolution yet — keeping open for further discussion.

## Change summary

Add a TabInputCustomDiff API so custom editors can detect when they are displayed in a diff view, analogous to TabInputTextDiff for text editors.

## Best solution

Design and propose a `CustomEditorDiffTabInput` (or equivalent) in the extension API alongside the existing `CustomEditorTabInput` in `src/vs/workbench/api/common/extHostTypes.ts` and the corresponding mapping in `extHost.api.impl.ts`. This would follow the established pattern of TextDiffTabInput and NotebookDiffEditorTabInput. The API proposal should be discussed with the custom-editor API owners to decide whether the diff scheme approach (as used in hex editor) is sufficient or whether first-class API support is warranted.
