# Issue #212034 — Maximizing the current editor while in zen mode causes current editor group size toggle state to be lost

[https://github.com/microsoft/vscode/issues/212034](https://github.com/microsoft/vscode/issues/212034)

## Summary

Confirmed bug still present on main: no fix has been committed. The grid widget's exitMaximizedView restores view visibility without accounting for zen mode's centered layout state, causing the editor group sizes toggle state to be lost.

## Change summary

Unmaximizing an editor group while in zen mode causes all other editor groups to collapse to minimum size, requiring two 'Toggle Editor Group Sizes' invocations to restore the original layout.

## Best solution

The fix likely belongs in the interaction between `exitMaximizedView` (src/vs/base/browser/ui/grid/gridview.ts) and the zen mode / centered layout logic (src/vs/workbench/browser/layout.ts). The unmaximize path needs to properly restore the pre-maximize sash sizes when centered layout is active. Related issue #221985 was closed as a duplicate of this family of bugs, confirming this as the canonical tracker.
