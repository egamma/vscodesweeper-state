# Issue #301688 — feat: display agent name in chat list (overview)

[https://github.com/microsoft/vscode/issues/301688](https://github.com/microsoft/vscode/issues/301688)

## Summary

Valid feature request to surface the agent/provider name in the sessions list rows. Currently the sessions list shows title, workspace badge, diff stats, and time — the agent/provider name only appears in the hover tooltip. The request is thin on detail (no mockup, no specific surface named) but reasonable; keeping open for the community process and routing to the agents-window area.

## Change summary

Show the agent or provider name inline in the sessions/chats list so users can identify which agent was used without hovering.

## Best solution

Add an inline agent-name badge or label to `SessionItemRenderer.renderSession()` in `src/vs/sessions/contrib/sessions/browser/views/sessionsList.ts`, drawing from the provider label already surfaced in the hover (`sessionHoverContent.ts`). The broader UX exploration in https://github.com/microsoft/vscode/issues/322934 (communicating session/chat model through clear UI structure) partly overlaps — a maintainer should decide whether this request is absorbed by that design exploration or warrants a standalone change.
