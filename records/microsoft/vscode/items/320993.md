# Issue #320993 — Agents Window: Custom agents disappear from dropdown after sending a message

[https://github.com/microsoft/vscode/issues/320993](https://github.com/microsoft/vscode/issues/320993)

## Summary

Confirmed open bug: custom agents disappear from the Agents Window dropdown after sending a message. A MEMBER corroborated similar behavior. The root cause likely involves the session state lifecycle — when a session transitions from Untitled to running, `getCustomAgents()` depends on `_lastSessionStates` which may not yet be populated for the new session ID, causing the picker to see an empty agent list.

## Change summary

Custom agents defined in `.github/agents/` disappear from the Agents Window dropdown after the first message is sent, though they persist correctly in the Chat view.

## Best solution

Fix the agent state handoff during session start: ensure `_lastSessionStates` is seeded (or the previous untitled state carried forward) before the picker re-evaluates after status transitions. The fix likely lives in `baseAgentHostSessionsProvider.ts` (around the `_applySessionStateUpdate` / session lifecycle) and `agentHostAgentPicker.ts` (`_initAgent` logic when `isUntitled` flips to false).
