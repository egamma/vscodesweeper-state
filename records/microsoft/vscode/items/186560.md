# Issue #186560 — "Go to file" time-to-first-result is pathological (~15s) when quickOpen.includeSymbols enabled

[https://github.com/microsoft/vscode/issues/186560](https://github.com/microsoft/vscode/issues/186560)

## Summary

The core request — showing file results incrementally without waiting for slow workspace/symbol providers — remains unaddressed on main. The `getAdditionalPicks` method still bundles file and symbol results behind a single `Promise.all`, so a slow LSP blocks file results from appearing. Valid feature request in the quick-open area.

## Change summary

When `search.quickOpen.includeSymbols` is enabled, quick open blocks file results from displaying until all workspace/symbol providers respond, causing multi-second delays with slow LSPs.

## Best solution

Refactor `getAdditionalPicks` in `src/vs/workbench/contrib/search/browser/anythingQuickAccess.ts` to stream file picks independently of symbol picks — e.g. return file results as soon as they resolve and merge symbol results when they arrive later, rather than gating both on `Promise.all`. The existing `FastAndSlowPicks` pattern could be extended to support a third tier, or file picks could move into the fast lane with a short delay.
