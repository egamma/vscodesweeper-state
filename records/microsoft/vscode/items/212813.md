# Issue #212813 — VSCode started with `--extensionDevelopmentPath` crashes after the computer enters in sleep mode

[https://github.com/microsoft/vscode/issues/212813](https://github.com/microsoft/vscode/issues/212813)

## Summary

Still reproduces on main: the extension dev host socket-close handler at line 140-143 of remoteExtensionHost.ts immediately terminates instead of attempting reconnection after sleep. Alexdima diagnosed the root cause in June 2024 but no fix has landed.

## Change summary

VS Code window started with `--extensionDevelopmentPath` crashes (terminates) after sleep/wake when connected to a WSL remote, because reconnection is skipped for extension dev hosts.

## Best solution

Remove or gate the early bail-out in `onSocketClose` (line 140-143 of `src/vs/workbench/services/extensions/common/remoteExtensionHost.ts`) so that extension dev hosts attempt reconnection on socket close, similar to non-dev hosts. Alexdima's comment suggests the original reason for skipping reconnection may no longer apply.
