# Issue #212813 — VSCode started with `--extensionDevelopmentPath` crashes after the computer enters in sleep mode

[https://github.com/microsoft/vscode/issues/212813](https://github.com/microsoft/vscode/issues/212813)

## Summary

Keep open: a maintainer identified a real remote extension-host reconnection gap for extension development/debug hosts, and current main still contains the same special-case exit path. I did not find a fixing commit or duplicate that would make this safe to close.

## Change summary

VS Code launched with `--extensionDevelopmentPath` against a WSL folder exits after the machine sleeps instead of reconnecting the remote extension host.

## Best solution

Decide whether remote extension development hosts should participate in the normal remote reconnect/restart flow after WSL socket loss, then adjust the special handling in `remoteExtensionHost.ts` / `abstractExtensionService.ts` with coverage for the remote extension-development case.

- **current code:** `src/vs/workbench/services/extensions/common/remoteExtensionHost.ts:140` still calls `_onExtHostConnectionLost(reconnectionToken)` immediately on `protocol.onSocketClose()` when `_isExtensionDevHost` is true.

- **history check:** `git log --all --grep=212813` found no commit referencing this issue; the later related-looking commit `7e805145f76dea04d774cb14b7bc85366c02e79d` only added graceful remote extension-host disconnect during shutdown and did not change the socket-close reconnection behavior.
