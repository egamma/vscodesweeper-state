# Issue #280911 — Command Palette closes unexpectedly with VoiceOver enabled on VS Code Web

[https://github.com/microsoft/vscode/issues/280911](https://github.com/microsoft/vscode/issues/280911)

## Summary

Still reproducible on main: the quick input blur handler (quickInputController.ts:332-334) hides the picker unconditionally when focus leaves the container, and VoiceOver/Narrator move focus to `body` during typing, triggering the hide. No fix has landed for this specific screen-reader interaction. Keeping open as a valid accessibility bug.

## Change summary

Command Palette closes (or loses typing) when a screen reader (VoiceOver/Narrator) is active on VS Code Web because the screen reader moves focus to body during typing.

## Best solution

The blur handler in `src/vs/platform/quickinput/browser/quickInputController.ts` (line ~332) should detect when focus moves to `document.body` (rather than to another meaningful element) and treat it as a transient screen-reader focus shift rather than a genuine blur — either by adding a small delay to re-check focus or by checking `document.activeElement === document.body` before hiding.
