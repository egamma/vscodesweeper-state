# Issue #245849 — [folding] folding gets messed up after Copilot edits a file

[https://github.com/microsoft/vscode/issues/245849](https://github.com/microsoft/vscode/issues/245849)

## Summary

Still unfixed on main — the folding region update logic after content changes has not been patched. Multiple users confirm across platforms and recent VS Code versions (up to Feb 2026).

## Change summary

Folding regions become incorrect (fold in wrong places) after Copilot or other AI tools apply multi-line edits to a file.

## Best solution

Investigate how `_currentFoldedOrManualRanges` in `src/vs/editor/contrib/folding/browser/foldingModel.ts` reconciles decoration-tracked ranges with the new folding provider output after large edits. The decoration ranges may drift when edits span collapsed regions, causing stale collapse state to be merged back via `sanitizeAndMerge`. A targeted fix in the update path (lines 146–169) to better validate decoration-range consistency after multi-line edits would likely resolve this.
