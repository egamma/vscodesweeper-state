# Issue #245849 — [folding] folding gets messed up after Copilot edits a file

[https://github.com/microsoft/vscode/issues/245849](https://github.com/microsoft/vscode/issues/245849)

## Summary

The issue is still open and reproducible per multiple reporters (latest confirmation Feb 2026). No targeted fix for folding corruption after bulk edits has landed on main — only debt/leak fixes. Keeping open as a valid bug in the editor-folding area.

## Change summary

Folding ranges become corrupted (fold in wrong places) after Copilot or other AI tools apply multi-region edits to a file.

## Best solution

Investigate how `FoldingModel.update()` preserves/restores collapsed state after large content changes — likely the `hiddenRangeModel.notifyChangeModelContent` or the range-update logic in `foldingModel.ts` doesn't correctly reconcile ranges when edits span or overlap existing folded regions. The fix likely lives in `src/vs/editor/contrib/folding/browser/foldingModel.ts` and/or `hiddenRangeModel.ts`.
