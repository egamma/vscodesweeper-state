# Issue #251581 — Disable Autosave for Merge Editor Result File

[https://github.com/microsoft/vscode/issues/251581](https://github.com/microsoft/vscode/issues/251581)

## Summary

Team-authored feature request to disable autosave for the merge editor result file, preventing unintended automatic application of changes. Keeping open per team-member policy.

## Change summary

When autosave is enabled, the merge editor result file gets auto-saved, which automatically applies the current merge state — confusing users who haven't finished resolving conflicts.

## Best solution

Implement an autosave exclusion for the merge editor result document so that saving (and thus applying) merge results remains an explicit user action, regardless of the global autosave setting. The fix likely lives in the merge editor contribution area.
