# Issue #163603 — GitHub - repo quick pick is sorts jumpy and unexpected

[https://github.com/microsoft/vscode/issues/163603](https://github.com/microsoft/vscode/issues/163603)

## Summary

Team-authored issue; the sorting instability is still present on main (the Map-merge in `getRemoteSources` has no stable ordering), and an open community PR (#312496) is already addressing it. Keeping open.

## Change summary

The GitHub repo quick pick jumbles results when async search results arrive and overwrite the initial user-repo ordering via an unordered Map merge.

## Best solution

Review and land (or close) the open PR https://github.com/microsoft/vscode/pull/312496, which adds stable ordering to the repository quick pick results in `extensions/github/src/remoteSourceProvider.ts`.
