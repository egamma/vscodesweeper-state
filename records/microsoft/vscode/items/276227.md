# Issue #276227 — Weird Prompt file linting error: No link definition found

[https://github.com/microsoft/vscode/issues/276227](https://github.com/microsoft/vscode/issues/276227)

## Summary

Team-authored issue, still unfixed on main. The markdown validator has no frontmatter-awareness in its link diagnostics, so YAML arrays in prompt file frontmatter are flagged as broken link references.

## Change summary

Markdown link validation incorrectly reports errors for YAML array items in prompt file frontmatter (e.g. `tools: ['runCommands', ...]`).

## Best solution

The markdown extension's diagnostic pipeline should skip or suppress link validation within frontmatter blocks. The fix likely belongs in the markdown language server's document-diagnostics logic (under `extensions/markdown-language-features/`) — either by parsing frontmatter boundaries before validation, or by adding a configuration knob to opt out of link checks in frontmatter regions.
