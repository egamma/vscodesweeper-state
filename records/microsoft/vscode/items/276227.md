# Issue #276227 — Weird Prompt file linting error: No link definition found

[https://github.com/microsoft/vscode/issues/276227](https://github.com/microsoft/vscode/issues/276227)

## Summary

Team-authored issue, still reproduces on main. The markdown link validator does not exclude YAML frontmatter content, so `tools:` arrays in prompt files produce spurious 'no link definition found' diagnostics. No fix has landed.

## Change summary

Markdown validation incorrectly flags YAML frontmatter array items (e.g. `tools: ['runCommands', ...]`) as missing link references.

## Best solution

The markdown diagnostics pipeline should skip validation inside frontmatter blocks. The `yamlPreamble` markdown-it rule already parses and identifies frontmatter token ranges; the diagnostic code in `extensions/markdown-language-features/src/languageFeatures/diagnostics.ts` needs to exclude those ranges from link-reference validation.
