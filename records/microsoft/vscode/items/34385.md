# Issue #34385 — XHR Failed on trying to install plugins via Visual Studio code

[https://github.com/microsoft/vscode/issues/34385](https://github.com/microsoft/vscode/issues/34385)

## Summary

The specific XHR-based networking infrastructure that produced the reported 'XHR failed' error was completely replaced years ago with a fetch-based request layer and @vscode/proxy-agent. The exact error is no longer possible in current VS Code. Remaining proxy-related issues are tracked by more specific, focused issues.

## Change summary

Installing extensions behind a proxy produced an 'XHR failed' error because the old XHR-based request layer did not properly handle proxy configurations.

## Best solution

Close as not reproducible — the XHR request infrastructure was removed and replaced with fetch + @vscode/proxy-agent. Specific remaining proxy issues (certificate handling, SOCKS5, proxy bypass, etc.) are tracked individually by more recent, focused issues like https://github.com/microsoft/vscode/issues/44941, https://github.com/microsoft/vscode/issues/12588, and others under the `proxy` label.
