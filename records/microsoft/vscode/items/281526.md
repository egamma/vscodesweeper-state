# Issue #281526 — Cropped text should have ellipises

[https://github.com/microsoft/vscode/issues/281526](https://github.com/microsoft/vscode/issues/281526)

## Summary

Still relevant on main: while inner elements (outline breadcrumbs) have text-overflow ellipsis, the bar container and the 'go-to-label' element lack overflow handling, so the widget text gets cropped without ellipsis when narrow.

## Change summary

When the NES long distance hint widget is resized narrow, cropped text should show ellipsis instead of being abruptly clipped.

## Best solution

Add `overflow: 'hidden'` and `textOverflow: 'ellipsis'` to the `.bar` container div and ensure child elements (especially the go-to-label) degrade gracefully when the widget is too narrow, in `src/vs/editor/contrib/inlineCompletions/browser/view/inlineEdits/inlineEditsViews/longDistanceHint/inlineEditsLongDistanceHint.ts`.
