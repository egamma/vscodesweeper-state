# Issue #293768 — `chat.agentFilesLocations`, `chat.promptFilesLocations`, and `chat.agentSkillsLocations` resolve custom paths on remote host instead of local host during Remote SSH sessions

[https://github.com/microsoft/vscode/issues/293768](https://github.com/microsoft/vscode/issues/293768)

## Summary

Confirmed still present on main: the promptFilesLocator explicitly converts absolute paths to vscodeRemote URIs when a remoteAuthority is present (line 588-592), causing user-configured custom locations to resolve on the remote host instead of locally. The author is a team member, so keeping open regardless.

## Change summary

Custom paths in `chat.agentFilesLocations`, `chat.promptFilesLocations`, and `chat.agentSkillsLocations` resolve on the remote host instead of the local machine during Remote SSH sessions.

## Best solution

The locator in `src/vs/workbench/contrib/chat/common/promptSyntax/utils/promptFilesLocator.ts` (lines 586-592) should distinguish user-storage locations from workspace-storage locations when a remote authority is present: user-storage absolute paths should remain as local `file://` URIs (or use `vscode-local://` scheme) so they resolve on the local machine, matching how the built-in User Data path already works.
