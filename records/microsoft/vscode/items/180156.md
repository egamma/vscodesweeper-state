# Issue #180156 — [linked editing] Linked editing stops syncing on quick edits

[https://github.com/microsoft/vscode/issues/180156](https://github.com/microsoft/vscode/issues/180156)

## Summary

Still present on main — the race between the debounced range-update provider request and rapid model-version increments causes linked editing to drop its decorations on fast typing.

## Change summary

Linked editing stops syncing tag pairs when the user types quickly, because the async provider response is discarded when the model version advances during the request.

## Best solution

The `updateRanges` flow should not clear existing decorations or discard a valid response solely because the model version advanced mid-request. Instead, it should keep the current sync ranges active while a new provider request is in-flight, or re-validate ranges against the current state rather than bailing out. The fix belongs in `src/vs/editor/contrib/linkedEditing/browser/linkedEditing.ts`.
