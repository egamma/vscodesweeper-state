# Issue #232159 — Devcontainer setup ad-hoc parsing fails on Rocky 8.5

[https://github.com/microsoft/vscode/issues/232159](https://github.com/microsoft/vscode/issues/232159)

## Summary

Still reproduces on main: line 33 of check-requirements-linux.sh uses a regex that explicitly excludes double-quoted ID values, which Rocky 8.5 (and potentially other distros) produce. Multiple open PRs exist but none have been merged.

## Change summary

The devcontainer setup script fails to parse OS_ID from /etc/os-release when the ID value is double-quoted (e.g., ID="rocky" on Rocky Linux 8.5), causing a non-zero exit code.

## Best solution

Land one of the open PRs (e.g., https://github.com/microsoft/vscode/pull/286456) that strips quotes from the ID value in `resources/server/bin/helpers/check-requirements-linux.sh` line 33. The reporter's suggested `source /etc/os-release && echo $ID` approach or a `tr -d '"'` pipe would both work. A maintainer should pick one PR, review for security (sourcing arbitrary files), and merge.
