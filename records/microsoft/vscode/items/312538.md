# Issue #312538 — Pressing Esc in extensions search should clear the search term

[https://github.com/microsoft/vscode/issues/312538](https://github.com/microsoft/vscode/issues/312538)

## Summary

Valid feature request for Escape to clear extensions search, consistent with existing Settings search behavior (shipped via #17442). No default Escape keybinding or precise focus context key exists today; the command exists but users cannot bind it precisely.

## Change summary

Pressing Escape in the Extensions search input should clear the search term, matching Settings search behavior.

## Best solution

Add a default Escape keybinding for `workbench.extensions.action.clearExtensionsSearchResults` gated on `extensionSearchHasText` plus the extensions viewlet being focused (e.g. `focusedView == 'workbench.view.extensions'`). Consider also adding a dedicated `inExtensionsSearch` context key for the search input focus state, for parity with `inSettingsSearch`.
