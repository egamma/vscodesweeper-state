# Issue #282456 — Feature Request: OAuth Authentication Support for MCP Registry Queries

[https://github.com/microsoft/vscode/issues/282456](https://github.com/microsoft/vscode/issues/282456)

## Summary

Valid feature request: OAuth authentication for MCP registry queries is not yet implemented on main. The MCP spec dependency appears unblocked (registry auth PR merged upstream), a team member expressed conditional agreement, and multiple enterprise users are asking for it. Keeping open for the team to schedule.

## Change summary

Add RFC 9728 OAuth authentication support when querying MCP server registries, enabling private/authenticated enterprise registries.

## Best solution

Extend the MCP gallery/registry query path (`src/vs/platform/mcp/common/mcpGalleryManifestService.ts` and `mcpGalleryService.ts`) to handle 401 responses by performing OAuth Protected Resource Metadata discovery (the plumbing already exists in `src/vs/base/common/oauth.ts`) and retrying with a bearer token. The team should decide scope and timing given the spec's current state.
