# Issue #286207 — terminal integrated suggest inserts wrong text when tab is pressed due to latency

[https://github.com/microsoft/vscode/issues/286207](https://github.com/microsoft/vscode/issues/286207)

## Summary

The race condition between keystrokes and tab-accept in terminal suggest is still plausible on current main. The acceptance logic uses `_currentPromptInputState` but this can be stale when async completion resolution and keystroke processing overlap, especially over SSH remote with latency. The reporter confirmed reproduction after the referenced fix (#282458, path validation).

## Change summary

Terminal integrated suggest inserts wrong/mangled text when tab is pressed quickly after typing, due to a race between keystroke processing and completion acceptance.

## Best solution

The acceptance logic in `terminalSuggestAddon.ts` needs to validate that the completion's replacement range is still consistent with the current prompt state at the moment of acceptance — if keystrokes arrived between when completions were computed and when tab is pressed, the widget should either re-filter or dismiss rather than inserting stale text. The `info-needed` label should be removed since the reporter already confirmed reproduction on Insiders.
