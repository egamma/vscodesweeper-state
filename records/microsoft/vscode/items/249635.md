# Issue #249635 — TextModel got disposed before DiffEditorWidget model got reset

[https://github.com/microsoft/vscode/issues/249635](https://github.com/microsoft/vscode/issues/249635)

## Summary

Still reproduces on current main: the disposal race between the search replace preview model and the diff editor widget is unfixed. The defective path at replaceService.ts:84 directly disposes the text model while the diff editor still holds a reference, triggering the BugIndicatingError. Two other issues (#240190, #260081) were closed as duplicates of this one; #312382 is a newer open duplicate. A PR (#313937) that proposed downgrading the error to a warning was closed without merging. The real fix, as hediet diagnosed, is to ensure the model isn't disposed while the diff editor still references it.

## Change summary

When a search file match is disposed, the replace preview text model is directly disposed before the diff editor widget releases its reference, producing a BugIndicatingError in telemetry.

## Best solution

Fix the disposal ordering in `src/vs/workbench/contrib/search/browser/replaceService.ts` (line 84): instead of directly disposing the replacePreviewModel on fileMatch disposal, ensure the diff editor's model reference is cleared first. This likely means the ReplacePreviewContentProvider should coordinate with the editor service to close/reset any diff editor showing the preview before disposing the underlying model. The diff editor's recovery path (setModel(null)) already works, but the proper fix is to not trigger it at all.
