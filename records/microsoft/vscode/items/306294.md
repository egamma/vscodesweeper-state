## Summary

A reasonable feature request to streamline the 'find in selection' workflow with a single action that applies the current editor selection and enables the mode. Related to the UX confusion reported in #284007 but proposes a distinct interaction improvement rather than being a direct duplicate.

## Change summary

Request a one-click action to apply the current editor selection to the find widget's 'find in selection' scope, replacing the old selection and toggling the mode on/off.

## Best solution

Route to the editor-find area owner. The request complements the discussion in https://github.com/microsoft/vscode/issues/284007 about confusing find-in-selection behavior. Implementation would likely touch `src/vs/editor/contrib/find/browser/findWidget.ts` and related controller logic.
