# Issue #315209 — $env:ComSpec = 'pwsh.exe' broke startup

[https://github.com/microsoft/vscode/issues/315209](https://github.com/microsoft/vscode/issues/315209)

## Summary

Setting ComSpec to pwsh.exe breaks the Windows system contract that ComSpec points to cmd.exe. VS Code's CLI launcher (code.cmd) is a standard batch file that Windows routes through %ComSpec%; when that points to PowerShell instead of cmd.exe, the batch syntax is misinterpreted. This is as-designed — VS Code correctly relies on the documented Windows convention.

## Change summary

Running `code .` fails when the user overrides $env:ComSpec to point to pwsh.exe instead of cmd.exe.

## Best solution

Close as as-designed. ComSpec is a Windows system variable that must point to cmd.exe — this is a documented Windows convention that all .cmd/.bat launchers depend on. The user should not override ComSpec to PowerShell; instead, they should set their default shell via other mechanisms (Windows Terminal default profile, VS Code's terminal.integrated.defaultProfile.windows, etc.).
