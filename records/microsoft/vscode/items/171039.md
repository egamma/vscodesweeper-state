# Issue #171039 — Git - click Stage Changes should trigger files.autoSave first 

[https://github.com/microsoft/vscode/issues/171039](https://github.com/microsoft/vscode/issues/171039)

## Summary

Still reproduces on main: the git.stage command in extensions/git/src/commands.ts does not trigger files.autoSave before staging, so users with onFocusChange get a stale-file dialog. The commit flow has promptToSaveFilesBeforeCommit but staging has no equivalent save step.

## Change summary

Clicking Stage Changes should trigger files.autoSave (e.g. onFocusChange) before staging, so unsaved edits are included and the stale-file warning is avoided.

## Best solution

Add a save step (similar to the existing promptToSaveFilesBeforeCommit pattern) at the top of the stage/stageAll commands in extensions/git/src/commands.ts, saving dirty documents that are about to be staged before running git add.
