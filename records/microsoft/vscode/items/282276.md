# Issue #282276 — In Firefox, an extra keystroke is required when replacing text in the editor

[https://github.com/microsoft/vscode/issues/282276](https://github.com/microsoft/vscode/issues/282276)

## Summary

Still reproduces on main. When accessibility support is 'auto' (Unknown) in Firefox, the textarea is populated with screen reader content via `SimplePagedScreenReaderStrategy`, and keyboard selection triggers content writes that swallow the next keystroke. The workaround (setting accessibility to 'off') confirms the root cause path.

## Change summary

In Firefox on vscode.dev, selecting text with Shift+Arrow requires an extra keystroke before typed text replaces the selection.

## Best solution

When accessibility support is Unknown (auto-detected) in Firefox web, the textarea content strategy should avoid populating the full page of screen reader content after keyboard selection, or the input handler should properly handle the state transition so the next keystroke is not lost. The fix likely lives in `src/vs/editor/browser/controller/editContext/textArea/textAreaEditContext.ts` around the `getScreenReaderContent` logic or in `textAreaEditContextInput.ts` where selection/input state is managed.
