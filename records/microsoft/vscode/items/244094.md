# Issue #244094 — Local server auth does not work when connected to remote

[https://github.com/microsoft/vscode/issues/244094](https://github.com/microsoft/vscode/issues/244094)

## Summary

Confirmed bug — TylerLeonhardt reproduced it and the underlying issue (env.openExternal opening localhost URL doesn't work correctly in a remote session despite the extension running client-side) remains unfixed on current main.

## Change summary

Local loopback server auth flow for GitHub sign-in fails when VS Code is connected to a remote (e.g. Docker) — the localhost URL never opens in the browser.

## Best solution

Investigate why `env.openExternal` on line 362 of `extensions/github-authentication/src/flows.ts` fails to open the `http://127.0.0.1:{port}/signin` URL when connected to a remote despite the extension running on the client. The `supportsRemoteExtensionHost: false` guard at line 299 may not trigger because the extension is a UI extension (local host), yet something in the remote session's URL-opening pipeline still interferes. Fix should ensure the loopback flow either works correctly in remote sessions or is properly disabled when it cannot.
