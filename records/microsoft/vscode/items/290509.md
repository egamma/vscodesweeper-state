# Issue #290509 — Git: Stash editor redirects to wrong stash when new stashes are created

[https://github.com/microsoft/vscode/issues/290509](https://github.com/microsoft/vscode/issues/290509)

## Summary

Confirmed bug with an open fix PR and an 'On Deck' milestone — keeping open. The stash editor identifies stashes by their shifting index rather than their immutable commit hash, causing wrong-stash operations when new stashes are created.

## Change summary

Git stash editor uses the mutable stash index as its identifier, so opening or operating on a stash targets the wrong entry after new stashes shift the indices.

## Best solution

Land the open PR https://github.com/microsoft/vscode/pull/288616 which switches the stash editor identity from index to commit hash. The fix touches `extensions/git/src/commands.ts` (particularly `getStashFromUri` around line 4738) and the URI construction. Review the PR for correctness around the editor-title side effect the author noted.

- **open PR:** https://github.com/microsoft/vscode/pull/288616 is still open (not merged), proposing to use the stash commit hash instead of the index as the editor identity.
