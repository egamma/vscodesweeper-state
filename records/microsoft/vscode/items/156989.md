# Issue #156989 — Ambiguous characters warn about box drawing \ but not / or |

[https://github.com/microsoft/vscode/issues/156989](https://github.com/microsoft/vscode/issues/156989)

## Summary

Still reproduces on main: U+2572 (╲), U+2502 (│), and U+2503 (┃) are absent from the ambiguous character data in strings.ts, while the visually similar U+2571 (╱→/) and U+2573 (╳→X) are present. The inconsistency is confirmed in the current source.

## Change summary

Box drawing characters U+2572 (backslash-like), U+2502 and U+2503 (pipe-like) are not flagged as ambiguous despite being visually confusable with ASCII \, |.

## Best solution

Update the ambiguous character data generated by https://github.com/hediet/vscode-unicode-data to include U+2572→92 (backslash), U+2502→124 (pipe), and U+2503→124 (pipe) in the `_common` set in `src/vs/base/common/strings.ts`.
