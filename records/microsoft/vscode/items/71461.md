# Issue #71461 — Task input parameters from showOpenDialog

[https://github.com/microsoft/vscode/issues/71461](https://github.com/microsoft/vscode/issues/71461)

## Summary

Valid feature request with 22 upvotes (above the 20-upvote threshold) asking for a file/folder picker input variable type for tasks and launch configs. Not yet implemented on main — the schema still only has promptString, pickString, and command. An extension workaround exists but the core gap remains.

## Change summary

Add a file/folder open-dialog input variable type (wrapping showOpenDialog) for tasks and launch configurations.

## Best solution

Keep in Backlog. The `command` input type already provides a workaround via extensions like Command Variable, but a built-in `openDialog` input type would be a natural addition to `src/vs/workbench/services/configurationResolver/`. With 22+ upvotes this has crossed the community threshold for Backlog promotion.
