# Issue #869 — Exclude all files except for...

[https://github.com/microsoft/vscode/issues/869](https://github.com/microsoft/vscode/issues/869)

## Summary

Highly popular, unimplemented feature request (648 upvotes) for negated/exception patterns in files.exclude. No code changes have landed; the only attempt (PR #152225) was closed without merging. The request remains valid and well above the community threshold.

## Change summary

Support negated glob patterns or exception overrides in files.exclude so users can exclude everything except specific files or folders.

## Best solution

Implement exception support in the glob matching engine (src/vs/base/common/glob.ts) and the file-tree walker so that a more-specific `false` entry overrides a broader `true` pattern. The main engineering challenge — noted by @bpasero in the comments — is that the tree walker short-circuits excluded directories, so an exception nested inside an excluded folder requires the walker to descend into it selectively. This likely touches glob.ts, the file explorer data source, and the search walker.
