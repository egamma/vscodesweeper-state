# Issue #869 — Exclude all files except for...

[https://github.com/microsoft/vscode/issues/869](https://github.com/microsoft/vscode/issues/869)

## Summary

This is a highly-upvoted (648 👍) feature request for negated/override patterns in `files.exclude`, still unimplemented on current main. It carries the 'On Deck' milestone — a real scheduling milestone that triggers a hard keep-open. The feature gap is genuine and well-documented.

## Change summary

Allow `files.exclude` entries set to `false` to override broader `true` patterns, enabling users to hide everything except specific files or folders.

## Best solution

Implement override semantics in the glob-matching infrastructure (`src/vs/base/common/glob.ts`) so that a more-specific `false` entry in `files.exclude` takes precedence over a broader `true` pattern. The tree-walking optimization challenge noted by @bpasero in comment 26 (skipping excluded directories entirely prevents seeing overridden children) needs an architectural solution — likely a pre-analysis pass that identifies excluded folders with deeper overrides, walking only the override paths. PR https://github.com/microsoft/vscode/pull/152225 attempted this but was closed without merging; its approach could inform a new attempt.
