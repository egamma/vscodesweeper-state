# Issue #270454 — SCM - vscode doesn't remember collapsed state of `staged changes`

[https://github.com/microsoft/vscode/issues/270454](https://github.com/microsoft/vscode/issues/270454)

## Summary

Hard keep-open: author is a team MEMBER, milestone is 'On Deck', and an open PR (#305267) already targets the fix. The bug is confirmed in current source — `onDidActiveEditorChange` unconditionally calls `tree.expandTo()` which expands collapsed groups to reveal the active editor's resource.

## Change summary

SCM view expands user-collapsed groups (e.g. 'Staged Changes') when auto-revealing the active editor's file, losing the user's collapsed state.

## Best solution

The fix in the open PR https://github.com/microsoft/vscode/pull/305267 is the right lane — it should check whether a parent group is user-collapsed before expanding it during auto-reveal. The logic lives in `onDidActiveEditorChange` in `src/vs/workbench/contrib/scm/browser/scmViewPane.ts` around line 1789. Review and land that PR (or an equivalent fix).
