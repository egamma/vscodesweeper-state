# Issue #202964 — Dynamic softwrap margin whilst retaining horizontal scroll.

[https://github.com/microsoft/vscode/issues/202964](https://github.com/microsoft/vscode/issues/202964)

## Summary

Niche feature request for a hybrid soft-wrap mode that wraps at a dynamic margin while retaining horizontal scroll for overflowing content. No traction (0 upvotes in 2.5 years), but it's a valid request that doesn't conflict with existing behavior.

## Change summary

Allow soft word wrapping at a dynamic margin (e.g. viewport-relative column) while preserving horizontal scrolling for content that exceeds that margin.

## Best solution

This sits in the editor word-wrap machinery (`src/vs/editor/common/config/editorOptions.ts`, wrapping logic). VS Code already supports `wordWrap: bounded` which wraps at min(viewport, column), but a truly dynamic margin that still allows horizontal scroll is a different interaction model. Given zero community traction, leave open for the community process; if demand grows it could be explored as a new `wordWrap` mode value.
