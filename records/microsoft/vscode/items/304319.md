# Issue #304319 — Language Model selector retains removed API provider models after deletion

[https://github.com/microsoft/vscode/issues/304319](https://github.com/microsoft/vscode/issues/304319)

## Summary

Fixed on main and shipped: commit ec46e336fb6 (2026-05-05) directly addresses stale BYOK models persisting in the model picker after API key removal, which is exactly this report. Shipped in 1.120.0.

## Change summary

After deleting a BYOK API key/provider, its models still appeared in the chat model selector and silently fell back to Copilot Auto.

## Best solution

Keep the shipped fix (ec46e336fb6, 1.120.0). The `mergeModelsWithCache` function now drops cache for resolved vendors with no live models, and `shouldResetOnModelListChange` resets the selected model when it disappears. No further work needed.
