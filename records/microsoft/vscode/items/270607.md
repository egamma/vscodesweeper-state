# Issue #270607 — Can't attach an SVG to have the agent edit it

[https://github.com/microsoft/vscode/issues/270607](https://github.com/microsoft/vscode/issues/270607)

## Summary

Still reproduces on main: SVG files are explicitly omitted when attached to chat. The attempted fix PR #270608 was closed without merging. Author is a team member, so keeping open for maintainer judgment.

## Change summary

Dragging an SVG file into chat input marks it as unsupported because the attachment resolver treats SVGs as omitted images rather than text/code files.

## Best solution

Remove or reclassify the SVG exclusion in `chatAttachmentResolveService.ts` so SVGs are attached as text/code content (like any other XML-based file) rather than rejected as unsupported images. Connor4312's comment confirms SVGs should upload as text/code. A security review of SVG content in chat context may be warranted before landing.
