# Issue #169779 — GitHub - Handle GitHub's branch protection push errors

[https://github.com/microsoft/vscode/issues/169779](https://github.com/microsoft/vscode/issues/169779)

## Summary

Team-authored feature request (MEMBER). The push error handler in the GitHub extension currently handles PermissionDenied (fork+PR) and GH009 push protection (secrets), but does not yet handle branch-protection-rule push rejections gracefully — those still fall through with a raw error. The request for better UX here remains valid.

## Change summary

When GitHub branch protection rules reject a push, VS Code should surface a helpful message and offer actions (e.g. create a PR) instead of showing a raw git error.

## Best solution

Extend `GithubPushErrorHandler.handlePushError()` in `extensions/github/src/pushErrorHandler.ts` to detect branch-protection-specific stderr patterns (e.g. 'protected branch', 'required status check', 'required pull request reviews') and surface a user-friendly dialog with actionable options like 'Create PR' — similar to the existing PermissionDenied flow. The built-in branch protection provider in `extensions/git/src/branchProtection.ts` and `extensions/github/src/branchProtection.ts` already tracks protected branches and could inform the UX.
