# Issue #190749 — Restored terminals in editor area don't restore the current working directory from the previous session.

[https://github.com/microsoft/vscode/issues/190749](https://github.com/microsoft/vscode/issues/190749)

## Summary

Still reproduces on main: the terminal editor serializer hardcodes `cwd: ''` instead of persisting the instance's actual working directory, so restored editor terminals lose their cwd.

## Change summary

Restored terminals in editor area don't restore the current working directory from the previous session.

## Best solution

In `terminalEditorSerializer.ts`, replace the hardcoded `cwd: ''` in `_toJson()` with the instance's actual cwd (via `instance.cwd` or the shell integration cwd). The panel terminals already handle this via reconnection; the editor serializer simply never wired it up.
