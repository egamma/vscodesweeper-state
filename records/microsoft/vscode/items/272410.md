# Issue #272410 — Exception Breakpoints don't honor BreakpointMode

[https://github.com/microsoft/vscode/issues/272410](https://github.com/microsoft/vscode/issues/272410)

## Summary

Confirmed on current main: sendExceptionBreakpoints builds filterOptions without including the mode property, despite the UI allowing mode selection and the DAP ExceptionFilterOptions supporting it. Small localized fix needed.

## Change summary

Exception breakpoint mode (set via 'Edit Mode...' context menu) is never sent to the debug adapter in the filterOptions of SetExceptionBreakpointsArguments.

## Best solution

Add `mode: exb.mode` to the filterOptions mapping in `debugSession.ts#sendExceptionBreakpoints()`, mirroring how SourceBreakpoint and InstructionBreakpoint already pass mode in their toDAP() methods.
