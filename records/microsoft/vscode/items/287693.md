# Issue #287693 — VSCode doesn't stop a diff operation on a large file when the editor window is closed.

[https://github.com/microsoft/vscode/issues/287693](https://github.com/microsoft/vscode/issues/287693)

## Summary

Still reproduces on main: the diff worker runs synchronously and cannot be interrupted once started. The related PR #287726 was closed without merging, and the new @vscode/diff adoption (commit 5fc781f799f) still has maxComputationTimeMs commented out. The issue remains valid and acknowledged by a maintainer.

## Change summary

Closing a diff editor while computing diffs on a large file (with maxComputationTime=0) does not stop the worker, wasting CPU resources.

## Best solution

Make the diff algorithm cancellable from the worker — either by killing the worker when the diff editor is disposed, or by making the algorithm resumable with periodic setTimeout breaks (as @hediet suggested in the comments). The timeout/cancellation mechanism in `defaultLinesDiffComputer.ts` (ITimeout) is the natural extension point.
