# Issue #127776 — Call stack decoration: consider using background color and not a decoration

[https://github.com/microsoft/vscode/issues/127776](https://github.com/microsoft/vscode/issues/127776)

## Summary

Still an open gap on main: the inline arrow decoration (`\uEB8B` in `makeStackFrameColumnDecoration`) still pushes text when the breakpoint is mid-line. The requested switch to a background-color-only approach (like Chrome DevTools) has not been implemented. Valid feature request, keeping open.

## Change summary

Replace the inline arrow decoration for the current debug stack frame position with a background-color highlight to avoid pushing/indenting the source line.

## Best solution

Refactor `callStackEditorContribution.ts` to drop (or conditionally suppress) the inline `before` content decoration in `makeStackFrameColumnDecoration` and instead rely on a column-range background highlight (similar to Chrome DevTools), preserving the glyph-margin icon and whole-line highlight. The work lives in `src/vs/workbench/contrib/debug/browser/callStackEditorContribution.ts` and its CSS.
