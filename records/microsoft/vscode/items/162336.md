# Issue #162336 — Terminal hover positioning is unfriendly

[https://github.com/microsoft/vscode/issues/162336](https://github.com/microsoft/vscode/issues/162336)

## Summary

Still valid: the terminal link hover positions at the start of the link range, ignoring the mouse x-coordinate. The editor case was fixed (PR #163970) but the terminal case was explicitly left open and remains unfixed on current main.

## Change summary

Terminal link hover should position near the mouse cursor rather than left-aligning to the start of the link range.

## Best solution

Update `CellHoverTarget` in `src/vs/workbench/contrib/terminal/browser/widgets/terminalHoverWidget.ts` to accept and use the mouse x-coordinate when constructing the hover target, similar to how the editor hover was improved in PR #163970. The hover service already supports positioning near a target element — the terminal just needs to pass a more mouse-aware target.
