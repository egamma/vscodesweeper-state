# Issue #202072 — VSCode Snap doesn't start when using Wayland mode

[https://github.com/microsoft/vscode/issues/202072](https://github.com/microsoft/vscode/issues/202072)

## Summary

The snap crash on Wayland startup has been addressed: commit c6bc1068180 forces `--ozone-platform=x11` in snapcraft.yaml, and the electron-launch script also appends `--ozone-platform=x11`. This prevents the GLib-GIO-ERROR crash in Wayland sessions. The fix shipped in 1.107.0.

## Change summary

The VS Code Snap package crashes on startup in Wayland sessions due to a GLib settings schema error ('antialiasing' key not found).

## Best solution

Keep the shipped fix in commit c6bc1068180 which forces X11 mode for the snap to avoid the Wayland GLib crash. The snap now starts reliably. Native Wayland support in the snap is a separate concern tracked upstream.
