# Issue #254751 — Visual Studio Code reopens after closing

[https://github.com/microsoft/vscode/issues/254751](https://github.com/microsoft/vscode/issues/254751)

## Summary

This is the same report as #88626 (closed wont-fix): the Windows Inno Setup installer relaunches VS Code after applying a pending update, even when the user was closing the app. The behavior comes from the `/mergetasks=runcode` flag passed to the installer. A nearly identical open issue (#52855) also tracks the broader update-on-shutdown experience.

## Change summary

VS Code reopens after the user closes it because the Windows background updater relaunches the app after applying a pending update.

## Best solution

Close as duplicate of https://github.com/microsoft/vscode/issues/88626, which was resolved as wont-fix — the relaunch is an intentional part of the Windows update flow via the `runcode` Inno Setup merge task. The open issue https://github.com/microsoft/vscode/issues/52855 tracks broader update-on-shutdown concerns if the team revisits this area.
