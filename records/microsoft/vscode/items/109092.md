# Issue #109092 — [html] Unecessary warning when script contains quotes on the same line as an onFoo attribute using the same quotes

[https://github.com/microsoft/vscode/issues/109092](https://github.com/microsoft/vscode/issues/109092)

## Summary

The bug likely lives in the vscode-html-languageservice scanner (upstream), which misparses quotes inside <script> content when on the same line as an on* event attribute. The Volar migration (PR #171547) is still open, so the current scanner-based architecture remains in place.

## Change summary

False warning appears when a script tag containing quotes is on the same line as an event handler attribute using the same quote character.

## Best solution

Fix the tokenizer in https://github.com/microsoft/vscode-html-languageservice to correctly handle quote boundaries between <script> content and subsequent on* attribute values on the same line. Alternatively, the in-progress Volar integration (PR #171547) may replace this scanner entirely.
