# Issue #109092 — [html] Unecessary warning when script contains quotes on the same line as an onFoo attribute using the same quotes

[https://github.com/microsoft/vscode/issues/109092](https://github.com/microsoft/vscode/issues/109092)

## Summary

Still reproduces on main. The embedded JS document construction in embeddedSupport.ts combines <script> block content and inline event handler attribute values into one virtual document; when both are on the same line, the whitespace substitution produces invalid JS that triggers a false TypeScript diagnostic.

## Change summary

False JavaScript warning when a <script> tag containing quotes appears on the same line as an onFoo attribute using the same quote character.

## Best solution

Fix the virtual-document construction in `extensions/html-language-features/server/src/modes/embeddedSupport.ts` so that the substituteWithWhitespace logic correctly handles the gap between a script-block region and an attribute-value region on the same line, or suppress diagnostics that span cross-region boundaries.
