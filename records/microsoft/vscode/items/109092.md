# Issue #109092 — [html] Unecessary warning when script contains quotes on the same line as an onFoo attribute using the same quotes

[https://github.com/microsoft/vscode/issues/109092](https://github.com/microsoft/vscode/issues/109092)

## Summary

Still a valid bug — the HTML scanner in vscode-html-languageservice incorrectly tokenizes quotes inside <script> when on the same line as on* event attributes using the same quote style. No fix found on current main.

## Change summary

False JavaScript diagnostic when a <script> tag containing quotes appears on the same line as an event handler attribute using the same quote character.

## Best solution

Fix the scanner in vscode-html-languageservice (upstream dependency) to correctly handle quote boundaries between <script> content and subsequent inline event attributes on the same line. The embedded region extraction in `extensions/html-language-features/server/src/modes/embeddedSupport.ts` would then get correct token boundaries without needing local changes.
