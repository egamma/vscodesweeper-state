# Issue #218252 — Missing theme variables for test coverage highlight background

[https://github.com/microsoft/vscode/issues/218252](https://github.com/microsoft/vscode/issues/218252)

## Summary

VS Code already provides themeable coverage colors (testing.coveredBackground, testing.uncoveredBackground, etc.) via the built-in test coverage API. The reporter is using the Jest extension's own coverage rendering (Jest: Toggle Coverage), which predates the built-in API and uses its own decorations — theming support depends on the extension adopting VS Code's native coverage API.

## Change summary

Theme authors cannot customize the highlight background colors shown by test coverage in the editor.

## Best solution

The built-in coverage theme colors exist and work for extensions using the native Test Coverage API. The Jest extension should adopt vscode's TestCoverage API to inherit these theme colors; this is an extension-side adoption issue, not a missing VS Code capability.

  > Thanks for the report. VS Code does provide themeable coverage colors — `testing.coveredBackground`, `testing.uncoveredBackground`, `testing.coveredGutterBackground`, `testing.uncoveredGutterBackground`, and their border counterparts — all registered in the testing theme module. These work for extensions that use VS Code's built-in [Test Coverage API](https://code.visualstudio.com/api/extension-guides/testing#test-coverage).
