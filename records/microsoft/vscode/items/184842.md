# Issue #184842 — workbench.action.navigateBack/Forward incorrect after adding/removing lines

[https://github.com/microsoft/vscode/issues/184842](https://github.com/microsoft/vscode/issues/184842)

## Summary

The navigation history stores static line numbers that don't adjust when content changes — this is a known architectural limitation, but the report is from an old version (1.74) with no further engagement. Requesting confirmation on a current build before investing triage effort.

## Change summary

workbench.action.navigateBack/Forward should adjust stored positions when lines are added or removed in the document.

## Best solution

This is better classified as a feature request: the `TextEditorPaneSelection` in `src/vs/workbench/browser/parts/editor/textEditor.ts` stores a fixed `Selection` and `EditorNavigationStack` in `historyService.ts` never adjusts entries for content changes. A proper fix would use tracked ranges (like `ITextModel.deltaDecorations`) to keep history positions accurate. Given the complexity (cross-cutting architectural change), this could remain in backlog until demand grows.
