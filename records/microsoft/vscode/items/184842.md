# Issue #184842 — workbench.action.navigateBack/Forward incorrect after adding/removing lines

[https://github.com/microsoft/vscode/issues/184842](https://github.com/microsoft/vscode/issues/184842)

## Summary

Keep open and route to the editor/workbench history area: current main still appears to restore navigation history from stored line-number selections, so the central report is not shown as fixed.

## Change summary

Go Back/Go Forward should return to the logical prior location after lines are inserted or deleted before that location.

## Best solution

Track text editor navigation entries through document edits, likely in `src/vs/workbench/services/history/browser/historyService.ts` and `src/vs/workbench/browser/parts/editor/textEditor.ts`, using an edit-aware range/marker rather than restoring only the original `Selection` line numbers. Add a history service or text editor pane test that inserts/removes many lines before a saved navigation stop and verifies the restored location follows the text.
