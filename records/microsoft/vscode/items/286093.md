# Issue #286093 — Agent sessions - recent list update before session is loaded

[https://github.com/microsoft/vscode/issues/286093](https://github.com/microsoft/vscode/issues/286093)

## Summary

Team-authored bug report about a timing issue where the recent sessions list updates (session disappears or moves) before the clicked cloud agent session finishes loading. No fixing commit found on main; the discussion identifies root causes (timestamp changes on activation, read-marking clearing the slot) but no resolution yet. Keeping open per team-member rule.

## Change summary

Clicking an unread cloud agent session causes the recent sessions list to visually update before the session is actually loaded, causing a jarring UX.

## Best solution

Address both root causes identified in the discussion: (1) prevent session property changes (like timestamps) from re-sorting the list while a session is being opened, and (2) defer the read-marking (which frees a slot for another unread session) until the session is visibly loaded. The related work in https://github.com/microsoft/vscode/issues/281961 may also improve this once recent sessions UX is reworked.
