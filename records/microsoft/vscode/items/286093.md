# Issue #286093 — Agent sessions - recent list update before session is loaded

[https://github.com/microsoft/vscode/issues/286093](https://github.com/microsoft/vscode/issues/286093)

## Summary

Team-filed bug with video reproduction showing the recent sessions list updating (and the clicked session disappearing) before the session finishes loading. No fix found on main; team discussion identifies root causes (property changes on session activation, read-marking evicting unread sessions). Keeping open for the team to address.

## Change summary

Clicking an unread cloud agent session causes the recent session list to visually update (session disappears) before the session content is loaded, creating a jarring UX.

## Best solution

Two fixes discussed by the team: (1) prevent session property changes on activation from causing re-sorting that makes the session disappear, and (2) defer read-marking or list update until the session is actually displayed. The related UX improvement in https://github.com/microsoft/vscode/issues/281961 (recent list rework) would also help by not evicting sessions when unread count exceeds the visible limit.
