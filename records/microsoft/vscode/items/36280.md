# Issue #36280 — [html] JSON edit support inside HTML document / script tag with type="application/json"

[https://github.com/microsoft/vscode/issues/36280](https://github.com/microsoft/vscode/issues/36280)

## Summary

Feature request with exactly 20 upvotes (Backlog threshold). Not implemented on main — the HTML language server only supports CSS and JavaScript as embedded languages. A community extension provides syntax highlighting via grammar injection, but full JSON language service integration remains unimplemented.

## Change summary

Request for JSON editing support (syntax highlighting, validation, intellisense) inside HTML script tags with type="application/json" or "application/ld+json".

## Best solution

Either integrate JSON as an embedded language in the HTML language server (alongside CSS/JS in `extensions/html-language-features/server/src/modes/`), or accept the community grammar-injection extension as sufficient and close as `*extension-candidate`. Given the 20 upvotes and widespread use of ld+json/import maps, keeping open as a valid Backlog item is reasonable; the Volar integration PR (https://github.com/microsoft/vscode/pull/171547) may eventually subsume this.
