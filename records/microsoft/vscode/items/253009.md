# Issue #253009 — Caption button show 2 tooltips on Windows

[https://github.com/microsoft/vscode/issues/253009](https://github.com/microsoft/vscode/issues/253009)

## Summary

Already triaged as upstream (Electron/Chromium). The duplicate tooltip on Windows caption buttons is caused by the OS rendering a native tooltip alongside the application-drawn one. Current main still lacks a workaround. Keeping open as an upstream tracking issue.

## Change summary

Windows caption buttons (minimize, maximize, close) show two tooltips — one from the OS and one from the application — when hovered.

## Best solution

Track the upstream Electron/Chromium fix for native tooltip suppression on custom-drawn caption controls. A possible VS Code-side workaround is adding explicit empty `title=""` attributes to the window control elements in `src/vs/workbench/electron-browser/parts/titlebar/titlebarPart.ts` (lines ~185–205) to suppress the browser-generated native tooltip, but the root fix belongs upstream.
