# Issue #313003 — Add an option to disable ghost inline auto complete for GitHub Copilot for comments specifically

[https://github.com/microsoft/vscode/issues/313003](https://github.com/microsoft/vscode/issues/313003)

## Summary

This is a feature request for the GitHub Copilot extension to suppress inline completions in comments. The behavior is controlled by the extension (which decides when/what to suggest), not by VS Code's editor core. The existing `editor.quickSuggestions` setting only governs the suggest widget, not inline completions — as confirmed by the MEMBER commenter. The appropriate venue is the Copilot extension's own issue tracker.

## Change summary

Add an option to suppress GitHub Copilot ghost/inline completions specifically when the cursor is inside a comment.

## Best solution

File this against the GitHub Copilot extension (https://github.com/community/community or the Copilot feedback channel). The extension itself decides what completions to show; VS Code core's inline completions API does not currently gate suggestions by token type. If a core API change were needed (e.g. exposing comment-context to providers so they can self-suppress), that would be a separate, narrower API proposal.
