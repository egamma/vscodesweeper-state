# Issue #164929 — Search & Replace exclude deleted files by default

[https://github.com/microsoft/vscode/issues/164929](https://github.com/microsoft/vscode/issues/164929)

## Summary

Confirmed bug, still unfixed on main. The search system has no guard against orphaned (deleted-from-disk) files — they appear in results and replace-all can silently recreate them. The `confirmed` label and `Backlog` milestone indicate the team accepted this; it should stay open.

## Change summary

Search & Replace includes orphaned (deleted) files in results, allowing accidental file recreation via replace-all.

## Best solution

When building search results or before applying replacements, filter out files whose text model is in orphan state (`TextFileEditorModelState.ORPHAN` / `inOrphanMode`). The fix likely touches `src/vs/workbench/contrib/search/browser/searchView.ts` (result filtering) and possibly the replace-all flow in the search result model.
