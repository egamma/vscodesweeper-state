# Issue #164929 — Search & Replace exclude deleted files by default

[https://github.com/microsoft/vscode/issues/164929](https://github.com/microsoft/vscode/issues/164929)

## Summary

Still reproducible on main: search does not exclude files that git marks as deleted (present on disk from a branch switch but not tracked on the current branch). The recent 'search in changed files' toggle (commit 5da06da4844) is a partial workaround but does not address the default behavior. The issue is confirmed, labeled, and belongs in the backlog.

## Change summary

Search & Replace includes files marked as deleted by git (from branch switching), risking accidental re-addition of those files to the working tree.

## Best solution

When building the search file set, consult SCM status and exclude resources whose git status is 'deleted' (not tracked on the current branch) by default — likely in `src/vs/workbench/contrib/search/browser/searchView.ts` where the query is constructed, integrating with `IScmService` to filter out deleted-status URIs. A setting to control this behavior would also be appropriate.
