# Issue #319745 — VS Code crashes on startup with TypeError when http.noProxy is a string

[https://github.com/microsoft/vscode/issues/319745](https://github.com/microsoft/vscode/issues/319745)

## Summary

Confirmed on current main: line 1132 of windowImpl.ts calls `.map()` on the raw config value with only `|| []` as a guard, which does not protect against a string. A misconfigured `http.noProxy` (string instead of array) crashes the main process.

## Change summary

VS Code crashes on startup when `http.noProxy` is set to a plain string instead of the expected array, because `.map()` is called without type validation.

## Best solution

Add a type guard in `windowImpl.ts` at line 1132 to ensure the value is an array before calling `.map()` (e.g. `Array.isArray(val) ? val : []`). The same pattern should be checked in the other noProxy consumers (`proxyResolver.ts`, `agentHostProxyResolver.ts`).
