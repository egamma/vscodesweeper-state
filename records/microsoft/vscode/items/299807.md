## Change summary

VS Code Insiders update on Windows fails with 'Access is denied' when the inno-updater tries to rename the current executable, requiring a machine restart to unblock.

## Best solution

Investigate whether the inno-updater can retry with a backoff, or detect and report which process holds the lock on the executable. The root cause is likely a lingering VS Code or extension-host process holding a handle on `Code - Insiders.exe`. The updater code lives in the `vscode-inno-updater` repo — any robustness improvement would land there, not in this repo directly.
