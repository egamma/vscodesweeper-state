# Issue #291290 — Roll the line numbers into the read file link, highlight range

[https://github.com/microsoft/vscode/issues/291290](https://github.com/microsoft/vscode/issues/291290)

## Summary

Still unfixed on main: copilotToolDisplay.ts generates 'Read [file.ts](uri), lines x to y' with the range as plain text outside the link, so the line numbers aren't part of the clickable anchor and clicking doesn't highlight the range. The inline anchor widget already supports range display and selection when range data is encoded in the link — the tool display just doesn't use that path.

## Change summary

File links from the read-file tool show line numbers as separate plain text outside the clickable link, and clicking the link doesn't highlight the referenced range.

## Best solution

Update `formatPathAsMarkdownLink` in `copilotToolDisplay.ts` to encode the view_range as a URI fragment (the inline anchor widget in `chatInlineAnchorWidget.ts` already handles `:x-y` suffix display and passes `location.range` as the selection on click). The 'lines x to y' text should be removed from the localized strings and the range rolled into the link URL instead.
