# Issue #320063 — "Open New External Terminal" does not correctly work with GNOME's default terminal emulator `ptyxis` . Fails to open window  that is navigated to the directory

[https://github.com/microsoft/vscode/issues/320063](https://github.com/microsoft/vscode/issues/320063)

## Summary

Confirmed in source: `spawnTerminal` in LinuxExternalTerminalService only passes `--working-directory` for ghostty; for ptyxis it relies on `cwd` in spawn options, which ptyxis ignores. A small fix adding ptyxis-specific args (like was done for ghostty) would resolve this.

## Change summary

"Open in External Terminal" with ptyxis does not open a window navigated to the target directory because VS Code lacks ptyxis-specific argument handling.

## Best solution

Add a ptyxis case in `LinuxExternalTerminalService.spawnTerminal` (src/vs/platform/externalTerminal/node/externalTerminalService.ts, around line 328) that passes `--new-window -d <cwd>` arguments, mirroring the existing ghostty special case. Also add ptyxis handling in `runInTerminal` for the debug adapter path.
