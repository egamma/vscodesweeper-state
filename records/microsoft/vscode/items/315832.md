# Issue #315832 — `Run task` not available for local sessions in agents windows

[https://github.com/microsoft/vscode/issues/315832](https://github.com/microsoft/vscode/issues/315832)

## Summary

Current main has concrete task-runner support for agent-host sessions, including local agent-host sessions, so the original “Run task not available” gap appears completed. This is not a `*`-label close; it should be closed as fixed/completed with the implementation commits cited.

## Change summary

Make the Agents window Run Task action available for local sessions instead of reporting that the session type is unsupported.

## Best solution

Keep the task-runner registry and `AgentHostSessionTaskRunner` implementation on main; reopen or file a focused follow-up only if a latest Insiders build still lacks Run Task for a specific local agent provider.

## Evidence

- **maintainer context:** A MEMBER comment on https://github.com/microsoft/vscode/issues/315832 said local support was experimental and missing features, and assigned the area owner rather than treating the report as invalid.
- **implementation commit:** Commit 85c156cccfd8155408da9cf62a4d338ac4b0eff6, merged via https://github.com/microsoft/vscode/pull/317186, added `ISessionTaskRunnerRegistry`, `WorkbenchSessionTaskRunner`, and `AgentHostSessionTaskRunner` so session tasks can dispatch through the owning runtime.
- **local agent-host support:** `src/vs/sessions/contrib/terminal/browser/agentHostSessionTaskRunner.ts` now describes the runner as for agent-host sessions “local or remote”; `canRun` accepts agent-host providers and maps providers without `remoteAddress` to the local sentinel `__local__`.
- **registration path:** `src/vs/sessions/contrib/terminal/browser/sessionsTerminalContribution.ts` registers `AgentHostSessionTaskRunner` with `ISessionTaskRunnerRegistry`, and `src/vs/sessions/contrib/chat/browser/sessionsTasksService.ts` dispatches `runTask` through the selected runner.
- **local follow-ups:** Commit d1a5e5ecf2bb52e0f1a3ec3abfb7c135541a6a26 (https://github.com/microsoft/vscode/pull/322336) explicitly fixed OS-specific task overrides for local agent-host tasks, and 27725b9a6380582f63485556f3484c0ba4c4347f (https://github.com/microsoft/vscode/pull/322629) fixed `${workspaceFolder}` resolution in agent-host tasks.
