# Issue #312886 — Agents files change list captures horizontal scrolls

[https://github.com/microsoft/vscode/issues/312886](https://github.com/microsoft/vscode/issues/312886)

## Summary

Still present on main: the files change list in the Agents window uses a monaco-list/tree widget that captures horizontal wheel events even when it has no horizontal scrollbar, preventing the parent container from receiving them. No fix commit found.

## Change summary

The Agents window files change list captures horizontal scroll events even though it is not horizontally scrollable, blocking the outer window scroll.

## Best solution

Configure the scrollable element in the changes file list (likely in `src/vs/sessions/contrib/changes/browser/`) to pass through horizontal wheel events when horizontal scrolling is disabled, or set `alwaysConsumeMouseWheel: false` on the horizontal axis. Alternatively, add `overscroll-behavior-x: none` or configure the list's ScrollableElementCreationOptions to not consume horizontal deltaX.
