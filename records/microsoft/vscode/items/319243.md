# Issue #319243 — Custom models disappear when opening a Dev Container

[https://github.com/microsoft/vscode/issues/319243](https://github.com/microsoft/vscode/issues/319243)

## Summary

This is a feature request for BYOK/OpenRouter model configurations to persist across VS Code profiles and Dev Containers. The storage uses profile-scoped `globalState`, so models disappearing on profile switch is by design — the reporter is asking for global (cross-profile) persistence of model registrations.

## Change summary

Custom OpenRouter/BYOK models disappear when switching profiles or reopening in a Dev Container because model configs are stored in profile-scoped extension globalState.

## Best solution

The Copilot extension's BYOK storage service (`extensions/copilot/src/extension/byok/vscode-node/byokStorageService.ts`) stores model configs in `extensionContext.globalState` which is profile-scoped. Making model registrations profile-independent would require either using a non-profile-scoped storage mechanism or syncing BYOK configs across profiles. This is a product decision for the Copilot team.
