# Issue #173259 — Incorrect Keyboard Shortcuts on Web Version of VSCode on Samsung Galaxy Tab S8 Ultra

[https://github.com/microsoft/vscode/issues/173259](https://github.com/microsoft/vscode/issues/173259)

## Summary

This should stay open: it is a device/browser-specific keyboard layout bug in VS Code for Web, and the discussion contains confirmations on Samsung keyboard covers plus a partial workaround, not evidence that the core issue was fixed or duplicated. The Backlog milestone is a holding bin and does not by itself block cleanup, but there is not enough concrete fix or duplicate evidence to close it.

## Change summary

VS Code for Web misinterprets German-layout shortcuts and AltGr combinations on Samsung Galaxy Tab keyboard covers, making Ctrl+Z and characters like `[` behave incorrectly.

## Best solution

Keep tracking this under the web keyboard-layout area until maintainers can verify current behavior on Android tablet hardware and decide whether the browser `KeyboardEvent` handling in VS Code for Web can normalize AltGr/layout behavior or whether this needs an upstream browser-specific follow-up.

## Evidence

- **issue report:** The reporter describes VS Code for Web on Samsung Galaxy Tab S8 Ultra with German layout interpreting Ctrl+Z as Ctrl+Y and AltGr+8 as Alt+8, while the system layout works elsewhere.
- **additional confirmation:** A later commenter reports the same AltGr failure on Galaxy Tab S9+ with the Samsung keyboard cover, including both editor and terminal impact.
- **browser-specific signal:** The same commenter notes the behavior does not work in Brave but does work in Firefox, suggesting a web/browser keyboard event path rather than a resolved VS Code-only issue.
- **workaround only:** A workaround was posted using Shift+AltGr on Galaxy Tab S9+/Brave, but no comment or timeline entry says the underlying incorrect shortcut interpretation was fixed.
