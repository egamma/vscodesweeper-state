# Issue #173259 — Incorrect Keyboard Shortcuts on Web Version of VSCode on Samsung Galaxy Tab S8 Ultra

[https://github.com/microsoft/vscode/issues/173259](https://github.com/microsoft/vscode/issues/173259)

## Summary

Keyboard layout detection on web VS Code relies on the browser's `navigator.keyboard.getLayoutMap()` API, which has limited/broken support on Android (Chrome on Samsung tablets). The `getLayoutMap` result only populates the `value` field — `withAltGr` and `withShiftAltGr` remain empty strings — so AltGr combinations cannot be resolved correctly. This is fundamentally a platform/browser limitation on Android.

## Change summary

Keyboard shortcuts (Ctrl+Z, AltGr+8) are misinterpreted on web VS Code when using a physical German keyboard on Samsung Galaxy tablets.

## Best solution

The root cause is that Android Chrome's Keyboard API does not correctly expose non-US keyboard layout mappings (especially AltGr layers). A partial mitigation in VS Code could improve the fallback path in `keyboardLayoutService.ts` to better detect AltGr key events on Android, but a full fix requires upstream Chromium work on the Keyboard Layout API for Android. Track upstream progress and consider Android-specific heuristics in the fallback mapper.

## Evidence

- **partial fix in 2024:** Commit e0baf3af8eb (March 2024) added a fallback when getLayoutMap is not allowed, but does not address the Android-specific AltGr/layout issue.
- **browser limitation:** The Keyboard API (navigator.keyboard) has limited support on Android Chrome; the layout map may not be populated correctly for non-US layouts on mobile platforms.
- **workaround:** Comment by JMLX42: using Shift+AltGr instead of AltGr alone works, and the issue does not reproduce in Firefox (different keyboard API implementation).
