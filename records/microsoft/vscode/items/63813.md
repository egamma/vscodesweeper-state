# Issue #63813 — terminal stays open after process exit

[https://github.com/microsoft/vscode/issues/63813](https://github.com/microsoft/vscode/issues/63813)

## Summary

Valid feature request on the 'On Deck' milestone — terminal accumulation after debug sessions remains unresolved on current main. The related PR #72080 was closed without merging and no alternative fix has landed. Hard keep-open rule applies (assigned 'On Deck' milestone).

## Change summary

Debug terminals stay open with 'Press any key to continue...' after the debugged process exits, accumulating zombie terminals across sessions.

## Best solution

Implement an option (likely in the DAP `runInTerminal` request or as a debug configuration property) to auto-close or reuse debug terminals when the session ends. The maintainer-suggested approach (comment #17) was adding an `autoTerminate` hint to `runInTerminal` so debug extensions can opt in. This would live in `src/vs/workbench/contrib/debug/browser/rawDebugSession.ts` and the terminal integration layer.
