# Issue #63813 — terminal stays open after process exit

[https://github.com/microsoft/vscode/issues/63813](https://github.com/microsoft/vscode/issues/63813)

## Summary

Valid feature request: the 'Press any key to continue...' behavior in external debug terminals is hardcoded with no user-facing setting to disable it. The request for an opt-out remains unaddressed on current main.

## Change summary

Add a setting to control whether external debug terminals wait for a keypress after the debugged process exits, to prevent accumulation of zombie terminals.

## Best solution

Add a debug setting (e.g. `debug.terminal.waitOnEnd`) that controls whether external terminals append the 'Press any key to continue...' / `--wait-after-command` / `/wait` logic. The implementation lives in `src/vs/platform/externalTerminal/node/externalTerminalService.ts` where the wait behavior is hardcoded per platform (line 268 for Linux `read -p`, line 56/113 for Windows `/wait`, line 249 for Ghostty `--wait-after-command`). A contributor comment (#10 by weinand) suggested an `autoTerminate` hint on `runInTerminal` as a possible DAP-level approach.
