# Issue #292302 — Pull request view flickers on startup

[https://github.com/microsoft/vscode/issues/292302](https://github.com/microsoft/vscode/issues/292302)

## Summary

Still open and acknowledged by team members — the view restore mechanism doesn't capture enough state to prevent showing a view that should no longer be visible (e.g. a merged PR view). sandy081 concluded core needs to capture the view state; no fix has landed.

## Change summary

Pull request view briefly flickers on startup when the associated PR was merged, because the view restore shows it before the extension can determine it should be hidden.

## Best solution

The view restore infrastructure in core (src/vs/workbench/browser/parts/views/) should capture and respect extension-provided view state so that views whose preconditions no longer hold are not restored. This likely involves persisting a when-context snapshot or a 'should restore' flag alongside the view layout state.
