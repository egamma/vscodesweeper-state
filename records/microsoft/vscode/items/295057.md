# Issue #295057 — `git diff` commands in built-in git extension don't use `--no-ext-diff`

[https://github.com/microsoft/vscode/issues/295057](https://github.com/microsoft/vscode/issues/295057)

## Summary

Valid bug filed by a team member: all `git diff` invocations in the built-in git extension lack `--no-ext-diff`, so users with an external diff tool configured (e.g. difftastic) get broken output parsing. An open PR (https://github.com/microsoft/vscode/pull/295262) already addresses this — keeping open pending that PR's review and merge.

## Change summary

Git diff invocations in the built-in git extension do not pass `--no-ext-diff`, causing failures when users have an external diff tool configured.

## Best solution

Land https://github.com/microsoft/vscode/pull/295262, which adds `--no-ext-diff` to all diff/diff-tree/log-p callsites in `extensions/git/src/git.ts`. The fix is straightforward and well-scoped.
