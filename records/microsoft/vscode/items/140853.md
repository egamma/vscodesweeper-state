# Issue #140853 — File nesting: collapsing with nested file selected selects parent file's next sibling

[https://github.com/microsoft/vscode/issues/140853](https://github.com/microsoft/vscode/issues/140853)

## Summary

Still unfixed on main: the original fix commit (78b7c963b31) never landed, and commit e2dc70e3936 only partially mitigates the twistie-click path without addressing the underlying selection-index invalidation on collapse.

## Change summary

Collapsing a tree node while a nested child is selected causes the selection to jump to the wrong item instead of the parent.

## Best solution

The tree's selection/focus trait needs to move selection to the collapsed parent when all selected children become hidden. The fix should live in `src/vs/base/browser/ui/tree/abstractTree.ts` in the `TreeNodeListTrait.onDidModelSplice` method (around line 2343) where the fallback logic currently fails to select the parent node. This is a general tree-widget fix, not file-nesting-specific.
