# Issue #1214 — Support regex in files.exclude => when

[https://github.com/microsoft/vscode/issues/1214](https://github.com/microsoft/vscode/issues/1214)

## Summary

Valid feature request with 56 upvotes (well above the 20-upvote backlog threshold). The `when` clause in `files.exclude` still only supports simple `$(basename)` string replacement with no regex or multi-pattern support — the gap remains on current main. This is the canonical issue; #78652 was explicitly closed as a duplicate of it.

## Change summary

Support regex (or at least glob alternation like `tsx?`) in the `when` clause of `files.exclude` so users can hide derived files matching multiple sibling patterns (e.g. hide `.js` when either `.ts` or `.tsx` exists).

## Best solution

Extend the sibling-clause matching in `src/vs/base/common/glob.ts` (around line 772, `parseExpressionPattern`) to support glob alternation or regex in the `when` value, so a single rule can match multiple sibling extensions. The workaround of two separate glob patterns works in Explorer but reportedly fails in Search (comment 12), adding practical urgency. With 56 upvotes and canonical status, this belongs in the backlog for the file-glob/file-explorer area owners to schedule.
