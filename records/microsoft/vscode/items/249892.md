# Issue #249892 — Extension Signature Verification Failed: Python

[https://github.com/microsoft/vscode/issues/249892](https://github.com/microsoft/vscode/issues/249892)

## Summary

Signature verification fails with UnknownError (exit code 1) on certain remote Linux hosts, preventing extension installation. The issue is still present on current main for linux-x64 (only linux-armhf is skipped). A Feb 2026 commenter confirms it persists on 1.109.4.

## Change summary

Extension signature verification fails with UnknownError on remote Linux hosts, blocking extension install/upgrade via SSH.

## Best solution

Investigate whether the vscode-signtool binary has compatibility issues on older/certain Linux distributions (RHEL 7, certain Ubuntu VMs). Consider gracefully handling UnknownError (exit code 1) similarly to how linux-armhf is handled, or improve error messaging to guide users toward the `extensions.verifySignature` setting as a workaround.
