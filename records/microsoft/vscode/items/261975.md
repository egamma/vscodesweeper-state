# Issue #261975 — Terminals sessions should not be deleted when starting a new conversation

[https://github.com/microsoft/vscode/issues/261975](https://github.com/microsoft/vscode/issues/261975)

## Summary

This is the canonical tracking issue for terminal persistence across chat sessions. Significant progress has landed on main — user-revealed terminals now survive session switches (68f213e0bec), terminals with outputLocation=terminal persist (b1889b0104a), and Agents window terminals survive archival (e76364f9169) — but the broader request for full configurability (e.g. never auto-close, let users choose which terminal to use) remains open and under discussion.

## Change summary

Terminals created by Copilot chat sessions are disposed when switching to a new chat session, losing command output and killing long-running processes.

## Best solution

Keep as the tracking issue for remaining terminal-persistence configurability. The core disposal-on-session-switch behavior has been substantially improved (user-revealed terminals, outputLocation=terminal, Agents window terminals all persist now). The remaining gap is the broader configurability the reporter and commenters requested — a setting like 'auto-close: never/when-hidden/always' and the ability to direct the agent to use an existing terminal. These are product decisions best made by the terminal/chat area owners.
