# Issue #283290 — Cannot read properties of undefined (reading 'normalizePosition')

[https://github.com/microsoft/vscode/issues/283290](https://github.com/microsoft/vscode/issues/283290)

## Summary

Still reproduces on current main: `normalizePosition` in viewModelLines.ts (line 998) accesses `this.modelLineProjections[info.modelLineNumber - 1]` without a null/bounds guard, causing a crash during race conditions when the model is being updated. A prior PR (#285602) attempted a fix but was closed without merging, and subsequent guard commits (7f75b4bddc5) addressed neighboring methods but missed `normalizePosition`.

## Change summary

Editor crashes with TypeError when `normalizePosition` is called during a race condition where `modelLineProjections` array is incomplete or has undefined entries.

## Best solution

Add the same defensive null/bounds check pattern used in neighboring methods (getViewLineContent, getMinColumnOfViewLine, etc.) to `normalizePosition` in `src/vs/editor/common/viewModel/viewModelLines.ts` at line 998, returning the input position unchanged when the projection is unavailable.
