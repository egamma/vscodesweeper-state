# Issue #264528 — Accepting an ML completion while in references view switches focus to main editor

[https://github.com/microsoft/vscode/issues/264528](https://github.com/microsoft/vscode/issues/264528)

## Summary

Still reproducible on main: the inline completions accept path uses `getOuterEditor` which resolves embedded editors (peek/references view) to the parent editor, causing the completion to target the wrong editor. On Deck milestone requires keep-open.

## Change summary

Accepting an ML/NES inline completion while editing in a references peek view incorrectly applies the completion to the outer main editor instead of the embedded peek editor.

## Best solution

The accept action in `inlineCompletionsController.ts` should use the focused code editor directly (from `ICodeEditorService.getFocusedCodeEditor()`) rather than unwrapping via `getOuterEditor`, or the controller instance lookup should get the controller from the embedded editor itself rather than the parent.
