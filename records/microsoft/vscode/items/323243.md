# Issue #323243 — Allow extensions to create terminals adjacent to a parent terminal without splitting

[https://github.com/microsoft/vscode/issues/323243](https://github.com/microsoft/vscode/issues/323243)

## Summary

Well-scoped feature request for a creation-time placement hint in the terminal API. Currently in the Backlog Candidates community voting process with 0 upvotes and 36 days remaining — keeping open for that process to run.

## Change summary

Allow extensions to create a new terminal adjacent to a related terminal in the tab list without forcing a split pane layout.

## Best solution

If the community process promotes this, add an optional `insertAfter` (or similar) property to `TerminalOptions` that places a new terminal tab adjacent to the referenced terminal without splitting. The implementation would touch `src/vscode-dts/vscode.d.ts` (API surface), the terminal service's tab management, and the extension host API shim. The request is intentionally narrower than the closed general-reordering proposals (#253020, #309068) and could be a small, self-contained API addition.
