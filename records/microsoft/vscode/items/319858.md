# Issue #319858 — Agent tool `read/problems` on Windows is sensitive to drive letter case. Not stable enough.

[https://github.com/microsoft/vscode/issues/319858](https://github.com/microsoft/vscode/issues/319858)

## Summary

Confirmed bug: the `GetErrorsTool` uses case-sensitive `isEqualOrParent` (from `extUri` with `() => false`) to compare URIs, so a lowercase drive letter from the agent doesn't match the uppercase drive letter in diagnostics. An open PR (#320053) already addresses this.

## Change summary

The `read/problems` agent tool on Windows fails to find diagnostics when the agent generates a lowercase drive letter, because the URI comparison is case-sensitive.

## Best solution

Land the open PR https://github.com/microsoft/vscode/pull/320053, which updates `getErrorsTool.tsx` to use case-insensitive path comparison (likely switching from `extUri.isEqualOrParent` to `extUriBiasedIgnorePathCase.isEqualOrParent`). No further work needed beyond reviewing that PR.
