# Issue #198518 — Cannot create new profile via copying default one

[https://github.com/microsoft/vscode/issues/198518](https://github.com/microsoft/vscode/issues/198518)

## Summary

Still reproduces on main: `copyExtensions` in `extensionManagementService.ts` unconditionally throws 'Not Supported' when a remote extension management server is present, blocking profile creation with extensions in any remote window (WSL, SSH, containers).

## Change summary

Creating a new profile by copying the Default profile fails with 'Not Supported' when VS Code is connected to a remote window because `copyExtensions` throws instead of delegating to the local server.

## Best solution

Fix `copyExtensions` in `src/vs/workbench/services/extensionManagement/common/extensionManagementService.ts` to delegate to the local extension management server when a remote server is present, rather than throwing. The local server handles profile extension storage; the remote server's presence shouldn't block the operation.
