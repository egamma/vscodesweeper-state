# Issue #243325 — Folder Auto-Collapses After Window Switch in VS Code

[https://github.com/microsoft/vscode/issues/243325](https://github.com/microsoft/vscode/issues/243325)

## Summary

Confirmed bug, acknowledged by a team member: compact folders are not properly re-rendered after the explorer refreshes on window focus. The refresh path (`forgetChildren` + `view.refresh`) loses the compact-folder rendering state, showing the parent as a non-compact collapsed folder. No fix has landed on main.

## Change summary

When switching away from VS Code and returning, folders that should be rendered as compact (e.g. `a/b`) instead render as separate collapsed folders.

## Best solution

Fix the explorer refresh path so that compact-folder compression state is preserved (or correctly recomputed) after a window-focus-triggered refresh. The likely files are `explorerService.ts` (the `onDidChangeFocus` refresh at line 132-137 and the `refresh()` method) and `explorerView.ts` (the tree refresh and compression options). The `collapseByDefault` callback returns `true` for newly loaded children, which combined with `forgetChildren()` causes the tree to lose expanded+compressed state.
