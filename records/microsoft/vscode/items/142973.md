# Issue #142973 — Language Detection crashes language server due to misdetection.

[https://github.com/microsoft/vscode/issues/142973](https://github.com/microsoft/vscode/issues/142973)

## Summary

Valid feature request to suppress ML language detection when a file has an unrecognized shebang line. The behavior persists on current main — `autoDetectLanguage` in `textFileEditorModel.ts` still runs for any extension-less plaintext file without checking for shebang lines. Low community traction (0 upvotes) but a real gap for users working with non-standard interpreters.

## Change summary

When an extension-less file has a shebang line (`#!...`) that no registered `firstLine` regex matches, VS Code should treat the file as plaintext and skip ML language detection rather than misidentifying it.

## Best solution

Add a shebang-presence check in `textFileEditorModel.ts#autoDetectLanguage` (or in the base `doAutoDetectLanguage`): if the file's first line starts with `#!` and no `firstLine` regex matched it to a known language, skip detection and leave it as plaintext. This keeps detection for untitled/new files intact while respecting the shebang as an authoritative signal.
