# Issue #180049 — [Accessibility]: Support filtering symbol types in Document Sylbol View

[https://github.com/microsoft/vscode/issues/180049](https://github.com/microsoft/vscode/issues/180049)

## Summary

Valid accessibility feature request for symbol-type filtering in the Document Symbol accessible view. The Outline panel already supports `outline.show*` settings to hide symbol kinds, but the accessible view layer may not expose equivalent filtering. Keeping open as an accessibility enhancement in the Backlog.

## Change summary

Allow filtering document symbols by type (e.g. hide variables, show only functions) in the Document Symbol accessible view for screen reader users.

## Best solution

Expose the existing `outline.show*` kind-filtering settings (or an equivalent accessible UI) within the accessible document-symbol navigation so screen reader users can narrow symbols by type without leaving the accessible view. The work likely lives in the outline accessible-view provider and the outline pane's filter integration.
