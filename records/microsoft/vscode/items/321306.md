# Issue #321306 — Copilot Chat destroys edits to files done by other chats, and those reverts are not recorded in edit history (timeline)

[https://github.com/microsoft/vscode/issues/321306](https://github.com/microsoft/vscode/issues/321306)

## Summary

Real architectural bug: each ChatEditingSession maintains an independent checkpoint timeline, so navigating back to a checkpoint in session-1 restores files to that session's recorded baseline — silently overwriting edits applied by session-2. The reporter demonstrates reproducible data-loss across dozens of files with no undo path.

## Change summary

When a chat editing session restores a checkpoint or starts a new request, it blindly overwrites file contents from its own timeline baseline, discarding edits made by other concurrent chat sessions.

## Best solution

The checkpoint-restore path in `ChatEditingCheckpointTimelineImpl._navigateToEpoch` / `_reconstructAllFileContents` needs awareness of cross-session edits. Likely fix: before restoring, compare the current disk/model content against the session's last-known state; if they diverge (another session touched the file), either conflict-mark, warn, or scope the restore to only the session's own files. The `multi-chat-sessions` area owns the design decision.
