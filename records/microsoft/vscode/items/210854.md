# Issue #210854 â€” Changing lettercase of directory causes unbound breakpoints as VSCode does not refresh path

[https://github.com/microsoft/vscode/issues/210854](https://github.com/microsoft/vscode/issues/210854)

## Summary

Still reproduces on main: breakpoint URIs are never updated when a parent directory is renamed (including case-only renames on macOS). The `onFileChanges` handler in debugService only removes breakpoints on DELETED events but does not relocate them on renames. The `IBreakpointUpdateData` interface does not include a `uri` field, and the `Breakpoint._uri` is immutable after construction.

## Change summary

Renaming a directory (especially changing letter casing on macOS) causes existing breakpoints to retain the old path, making them unbound.

## Best solution

Add a rename/move handler in `debugService.onFileChanges` (or via `IWorkingCopyFileService.onDidRunWorkingCopyFileOperation` for MOVE events) that updates `Breakpoint._uri` when a parent directory or the file itself is renamed. The `IBreakpointUpdateData` interface should be extended to include `uri`, or a dedicated `moveBreakpoints` method added to the debug model. Key files: `src/vs/workbench/contrib/debug/browser/debugService.ts` and `src/vs/workbench/contrib/debug/common/debugModel.ts`.
