[https://github.com/microsoft/vscode/issues/320027](https://github.com/microsoft/vscode/issues/320027)

## Summary

Feature request for better visual indication of cost-tier boundaries on the context usage meter. The specific motivating scenario (Claude Sonnet 4.6 long-context pricing) was resolved as an incorrect pricing error, but the general concept could apply to future models with tiered pricing. A team member engaged and tagged a colleague for input, so keeping open for team consideration.

## Change summary

Show cost-tier boundaries (e.g. a tick mark or colored bands) on the Copilot chat context usage meter so users can see at a glance when they cross from standard to long-context pricing.

## Best solution

The immediate motivation was resolved (Anthropic's long-context surcharge was a pricing error and has been corrected). If other models retain tiered context pricing, the Copilot extension's context meter could add a visual threshold indicator. The meter UI likely lives in the Copilot extension rather than VS Code core; this should be evaluated there if tiered pricing remains relevant for any model.
