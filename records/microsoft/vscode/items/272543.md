# Issue #272543 — `vscode.lm.selectChatModels` returns `undefined` in dev container

[https://github.com/microsoft/vscode/issues/272543](https://github.com/microsoft/vscode/issues/272543)

## Summary

Team-authored issue describing a race condition in dev containers where `selectChatModels` resolves before the Copilot provider has fully registered. The timing-sensitive code path in `_resolveAllLanguageModels` is visible on current main — the `activateByEvent` call may resolve before the provider is actually registered in a remote/dev-container scenario. Keeping open per team-authored policy.

## Change summary

In dev containers, `vscode.lm.selectChatModels({ vendor: 'copilot' })` sometimes returns an empty result because the provider hasn't fully registered by the time the activation event resolves.

## Best solution

The race is in `LanguageModelsService._resolveAllLanguageModels` (`src/vs/workbench/contrib/chat/common/languageModels.ts`): after `activateByEvent` resolves, the provider may still not be registered in a remote extension host scenario. A fix could add a brief wait-for-provider pattern (e.g., waiting on a barrier/event that fires when the provider for the vendor is actually registered) rather than assuming the provider is immediately available after activation. The related pyrx issue (https://github.com/microsoft/pyrx/issues/7187) confirms downstream impact.

## Evidence

- **race condition path:** In `src/vs/workbench/contrib/chat/common/languageModels.ts` lines 1040–1061, `_resolveAllLanguageModels` calls `activateByEvent('onLanguageModelChatProvider:copilot')` and then immediately checks `this._providers.get(vendorId)`. In dev containers, the remote extension host may not have completed provider registration by the time the activation event resolves, causing the method to return early with no models.
- **downstream impact:** https://github.com/microsoft/pyrx/issues/7187 — Pylance's AI Hover feature fails in Codespaces due to this same timing issue.
- **no fix on main:** Current main still has the same pattern: activate-then-check with no wait-for-registration barrier. The race remains.
