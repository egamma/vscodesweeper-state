## Change summary

The third-party terminal application awrit (an in-terminal browser using kitty graphics protocol) fails to load in VS Code's integrated terminal.

## Best solution

Investigate whether this is a gap in VS Code's kitty graphics protocol implementation (in xterm.js) or a missing feature that awrit depends on (e.g. relative placements, animations, or other unimplemented KGP specs). The xterm.js discussion at https://github.com/xtermjs/xterm.js/discussions/5683 tracks implementation coverage. The terminal team should determine whether awrit's requirements fall within the supported KGP subset and file upstream xterm.js issues for any missing pieces.
