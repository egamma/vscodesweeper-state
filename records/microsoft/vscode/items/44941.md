# Issue #44941 — Proxy parameters are ignored and user-settings are missing a proxy-bypass option

[https://github.com/microsoft/vscode/issues/44941](https://github.com/microsoft/vscode/issues/44941)

## Summary

Both core requests are now implemented: the `http.noProxy` setting was added in 1.91.0 (commit e3e15e15e82) providing a proxy-bypass option in user settings, and `--proxy-bypass-list` was added as a supported argv argument (commit 8993f76c9fc). The `http.proxy` and `http.proxyStrictSSL` settings have long covered the CLI-param equivalents for the Node.js networking layer. This can be closed as completed.

## Change summary

CLI proxy parameters (--proxy-server, --proxy-bypass-list) were ignored by Node.js-based networking in VS Code, and there was no user-settings option to configure a proxy bypass list.

## Best solution

Keep the shipped `http.noProxy` setting (1.91.0) and `--proxy-bypass-list` argv support; no further change needed. The related open issue https://github.com/microsoft/vscode/issues/200464 can also be closed as the `http.noProxy` setting addresses its request.
