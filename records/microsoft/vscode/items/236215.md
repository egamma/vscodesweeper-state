# Issue #236215 — [css] `:after' suggested inside comment

[https://github.com/microsoft/vscode/issues/236215](https://github.com/microsoft/vscode/issues/236215)

## Summary

Still unfixed on main: the CSS language service completion code (in the external vscode-css-languageservice repo) does not check whether the cursor offset is inside a comment before offering completions. The fix belongs in the upstream dependency, not in this repo directly.

## Change summary

CSS completions (e.g. `:after`) are incorrectly triggered inside comments when `:` is typed.

## Best solution

Add a comment-offset check in `vscode-css-languageservice/src/services/cssCompletion.ts` (as aeschli outlined) so completions bail out when the scanner reports the offset is inside a comment, then update the dependency in VS Code.
