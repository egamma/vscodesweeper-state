# Issue #236215 — [css] `:after' suggested inside comment

[https://github.com/microsoft/vscode/issues/236215](https://github.com/microsoft/vscode/issues/236215)

## Summary

Still unfixed — CSS completions inside comments remain unguarded. Both attempted PRs were closed without merge, and the fix belongs in the upstream vscode-css-languageservice repo where it has not yet landed.

## Change summary

Typing `:` inside a CSS comment triggers pseudo-element completions (e.g. `:after`) even when `editor.quickSuggestions.comments` is off, because the CSS language service doesn't check whether the cursor is inside a comment before offering trigger-character completions.

## Best solution

Fix in https://github.com/microsoft/vscode-css-languageservice in `src/services/cssCompletion.ts` — use the scanner to detect whether the offset is inside a comment and bail out before computing completions. Then bump the dependency version in VS Code.
