# Issue #210850 — `Searn only in Opened Editors` and `files to include` doesn't support filename with special characters

[https://github.com/microsoft/vscode/issues/210850](https://github.com/microsoft/vscode/issues/210850)

## Summary

The primary bug (scenario 1: 'Search only in Opened Editors' failing for files with square brackets) was fixed in commit 4e518fed162 and shipped in 1.106.0. Scenario 2 ('files to include' treating `[a].js` as a glob character class) is as-designed — the field accepts glob patterns, and `[[]a[]].js` is the correct escape, as confirmed by a commenter.

## Change summary

Searching in opened editors or via 'files to include' fails to match filenames containing square brackets like `[a].js`.

## Best solution

Keep the shipped fix from commit 4e518fed162 (1.106.0). The 'files to include' field intentionally interprets glob syntax, so `[a]` matching the character `a` is correct behavior; users escape with `[[]a[]].js`.
