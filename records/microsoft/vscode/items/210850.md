# Issue #210850 — `Searn only in Opened Editors` and `files to include` doesn't support filename with special characters

## Summary

The 'Search only in Opened Editors' part (steps 2–3) was fixed in commit 4e518fed162 (shipped in 1.106.0). However, the issue carries the 'On Deck' milestone, so it must stay open for maintainer judgment. The 'files to include' part (step 4) may still be relevant — glob input fields interpret brackets as glob syntax by design, but users may expect literal matching.

## Change summary

Searching in opened editors fails to find content in files whose names contain glob special characters like `[` and `]`.

## Best solution

The opened-editors scenario (steps 2–3) is fixed by commit 4e518fed162 (1.106.0). If step 4 is accepted as-designed, close as completed.
