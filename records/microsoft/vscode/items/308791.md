# Issue #308791 — chat.pluginLocations: Support parent directory resolution to discover all plugins under a common root

[https://github.com/microsoft/vscode/issues/308791](https://github.com/microsoft/vscode/issues/308791)

## Summary

Valid feature request — the current implementation in `ConfiguredAgentPluginDiscovery._addPluginSource` treats each `chat.pluginLocations` entry as a single plugin directory with no parent-directory/recursive discovery. The capability does not exist on main.

## Change summary

Allow `chat.pluginLocations` entries pointing to a parent directory to automatically discover all valid plugin subdirectories within it.

## Best solution

Extend `ConfiguredAgentPluginDiscovery._addPluginSource` (in `src/vs/workbench/contrib/chat/common/plugins/agentPluginServiceImpl.ts`) to detect when a resolved directory does not itself contain a plugin manifest, and in that case enumerate its immediate children for valid plugin directories. Keep it shallow (one level) by default to avoid unintended traversal.
