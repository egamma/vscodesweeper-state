# Issue #325539 — Extend `chat.tools.terminal.autoApprove` to deny tool calls

[https://github.com/microsoft/vscode/issues/325539](https://github.com/microsoft/vscode/issues/325539)

## Summary

Reasonable feature request to extend the terminal auto-approve setting with an auto-deny mode that rejects tool calls without user interaction and optionally sends a steering message back to the agent. No existing mechanism covers this — `false` only forces manual approval, it does not auto-deny or provide agent feedback.

## Change summary

Allow `chat.tools.terminal.autoApprove` entries to automatically deny matching commands (without prompting the user) and optionally return a steering message to the agent.

## Best solution

Extend `chat.tools.terminal.autoApprove` to accept a third approval state (e.g. `"deny"` or an object `{ approve: "deny", message: "Use X instead" }`) that auto-rejects the tool call and feeds the optional message back as tool output so the agent can self-correct. The implementation would touch `commandLineAutoApprover.ts` and the tool confirmation flow in `runInTerminalTool.ts`. This is a well-scoped enhancement to the existing terminal tool approval pipeline.
