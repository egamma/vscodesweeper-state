# Issue #191706 — Dismissing a message will shrink the quick question UI

[https://github.com/microsoft/vscode/issues/191706](https://github.com/microsoft/vscode/issues/191706)

## Summary

The dynamic layout logic in quick chat still recalculates height from only the last 2 messages when any item height changes, which would shrink the viewport on message dismissal even when more content is available above. The bug pattern remains on current main.

## Change summary

Dismissing a message in quick chat history causes the UI to shrink despite having enough content to fill the expanded viewport.

## Best solution

Guard `layoutDynamicChatTreeItemMode` (in `src/vs/workbench/contrib/chat/browser/widget/chatWidget.ts`) so that when the user has scrolled up or manually expanded the view, dismissing an item does not shrink the layout below the current viewport height if sufficient content remains.
