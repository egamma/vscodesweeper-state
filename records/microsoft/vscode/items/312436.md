# Issue #312436 — Codex login fails with ERR_CONNECTION_REFUSED on localhost:1455 after VS Code update (arm64 1.117.0)

[https://github.com/microsoft/vscode/issues/312436](https://github.com/microsoft/vscode/issues/312436)

## Summary

The OAuth callback server on port 1455 is owned by the third-party "chatgpt" extension (by "ope"), not by VS Code core. The extension is responsible for starting the localhost listener that handles the OAuth redirect — VS Code itself does not manage port 1455 or this authentication flow.

## Change summary

Third-party Codex/OpenAI extension fails to start its local OAuth callback server on port 1455 when connected via Remote SSH.

## Best solution

Close as caused-by-extension. The user should file this against the third-party "chatgpt" extension (by openai/ope on the Marketplace). The likely root cause is that the extension starts the OAuth callback server on the remote host while the browser opens locally — this is an extension-level bug in how it handles Remote SSH scenarios, not a VS Code regression.
