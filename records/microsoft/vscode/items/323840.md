# Issue #323840 — Expose themeVariables of the built-in mermaid-markdown-features extension in user settings

[https://github.com/microsoft/vscode/issues/323840](https://github.com/microsoft/vscode/issues/323840)

## Summary

This is a valid feature request for the built-in Mermaid markdown extension: current settings expose theme selection but not arbitrary Mermaid `themeVariables`. It should stay open and be routed to the Markdown/Mermaid extension owners for a product decision.

## Change summary

Expose Mermaid `themeVariables` as a user setting for the built-in `mermaid-markdown-features` extension.

## Best solution

Decide whether the built-in Mermaid renderer should accept user-provided `themeVariables`; if accepted, add a typed setting to `extensions/mermaid-markdown-features/package.json`, carry it through `MermaidExtensionConfig`, and merge it in `buildMermaidConfig` without breaking the existing VS Code-derived theme variables.

## Evidence

- **current settings:** `extensions/mermaid-markdown-features/package.json` contributes `markdown-mermaid.lightModeTheme`, `darkModeTheme`, `languages`, `maxTextSize`, navigation, controls, resize, and max height settings, but no `themeVariables` setting.
- **current theme implementation:** `extensions/mermaid-markdown-features/preview-src/shared/vsCodeTheme.ts` computes Mermaid `themeVariables` from VS Code theme colors for the special `vscode` theme, but does not read user-specified Mermaid theme variables.
- **config shape:** `extensions/mermaid-markdown-features/preview-src/shared/config.ts` defines `MermaidExtensionConfig` without a `themeVariables` field, and `buildMermaidConfig` currently only applies the theme returned by `resolveMermaidTheme`.
