# Issue #199873 — "Reveal in File Explorer" not working after opening file with incomplete path via Ctrl+P

[https://github.com/microsoft/vscode/issues/199873](https://github.com/microsoft/vscode/issues/199873)

## Summary

Still reproducible on main: on Windows, typing a drive-relative path like `\Bug\file` in Quick Open creates a URI without the drive letter, so `isInsideWorkspace()` returns false and 'Reveal in File Explorer' silently does nothing. The code path in `anythingQuickAccess.ts` (line ~745) treats the path as absolute and builds a URI via `pathService.fileURI()`, but the resulting URI lacks the drive prefix that the workspace folder URI has, breaking the workspace-membership check in the reveal command (`fileCommands.ts` line 326).

## Change summary

On Windows, opening a file via Ctrl+P with a drive-relative path (e.g., `\Tools\file` instead of `C:\Tools\file`) causes 'Reveal in File Explorer' to silently fail because the resulting URI doesn't match the workspace.

## Best solution

Either normalize the URI in `getAbsolutePathFileResult` to include the current drive letter when the path is drive-relative (Windows only), or teach the reveal command to resolve drive-relative URIs before the `isInsideWorkspace` check. The fix likely touches `src/vs/workbench/contrib/search/browser/anythingQuickAccess.ts` (the `getAbsolutePathFileResult` method around line 745) or `src/vs/workbench/contrib/files/browser/fileCommands.ts` (the reveal command around line 326).
