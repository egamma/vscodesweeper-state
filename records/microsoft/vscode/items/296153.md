# Issue #296153 — Unclear why I need to take action when archiving background agent

[https://github.com/microsoft/vscode/issues/296153](https://github.com/microsoft/vscode/issues/296153)

## Summary

Team-authored UX polish request — the archive confirmation dialog is confusing after edits have been applied via the 'Apply' button. The dialog still warns about 'pending changes' and offers 'Keep & Continue' / 'Undo & Continue' options that don't map to the user's mental model. The file count discrepancy (says 1 file, but 2 were changed) is a separate bug in the count logic. Hard keep-open: author is MEMBER.

## Change summary

Archiving a background agent session after applying its changes shows a confusing confirmation dialog about 'pending changes' with unclear actions and an incorrect file count.

## Best solution

Improve `showClearEditingSessionConfirmation` in `src/vs/workbench/contrib/chat/browser/widgetHosts/editor/chatEditorInput.ts` — either skip the dialog entirely when all edits have already been applied (i.e., their state is no longer `ModifiedFileEntryState.Modified`), or reword the dialog to clearly explain what 'Keep' vs 'Undo' means in the archive context. Also fix the file count in `shouldShowClearEditingSessionConfirmation` which filters for `Modified` state entries — the count mismatch (1 vs 2) suggests one file transitioned state but another didn't.
