# Issue #303074 — "Allow All Commands" not work in chat session

[https://github.com/microsoft/vscode/issues/303074](https://github.com/microsoft/vscode/issues/303074)

## Summary

Team-authored bug report — must keep open for maintainer judgment. The issue describes 'Allow All Commands in this Session' failing to persist across subsequent tool invocations in a chat session. The session auto-approval is stored in `_sessionAutoApprovalEnabled` (ResourceMap keyed by session URI) and checked via `hasChatSessionAutoApproval`, but the auto-approve analyzer at `commandLineAutoApproveAnalyzer.ts:54-55` gates it behind `isAutoApproveEnabledInSettings`. The `curl` command also triggers prompt-injection warnings (line 27) which may interact with the approval flow. No fix commit found on `main` since the report.

## Change summary

'Allow All Commands in this Session' does not persist — the next command in the same chat session still requires confirmation.

## Best solution

Debug the interaction between `hasChatSessionAutoApproval` and the command-line analyzers in `src/vs/workbench/contrib/terminalContrib/chatAgentTools/browser/tools/commandLineAnalyzer/commandLineAutoApproveAnalyzer.ts`. The session auto-approval check at line 55 should bypass subsequent confirmation for all commands. Likely the session resource URI is changing between turns, or the `requiresAllowNetworkConfirmation`/`requiresUnsandboxConfirmation` path overrides the session approval (see `runInTerminalTool.ts:1259`).

- **auto-approve analyzer gate:** `CommandLineAutoApproveAnalyzer.analyze()` at `commandLineAutoApproveAnalyzer.ts:54-55` checks `isAutoApproveEnabledInSettings && hasChatSessionAutoApproval(...)`. If the setting is on (default: true) and session approval is set, it returns `isAutoApproved: true`. But something prevents this from working on the second invocation.
- **curl prompt injection warning:** `curl` is in `promptInjectionWarningCommandsLower` (line 26-28), which adds a disclaimer but should not block auto-approval when `isAutoApproved` is true (the disclaimer is gated on `!isAutoApproved` at line 177).
- **shouldShowConfirmation logic:** At `runInTerminalTool.ts:1259`, `shouldShowConfirmation` depends on `!isFinalAutoApproved && (!isSessionAutoApproved || requiresAllowNetworkConfirmation)`. If `requiresAllowNetworkConfirmation` is true (possible for curl), the session-level auto-approve is overridden.
