# Issue #199944 — Color picker is far too wide and does not remember when I resize it smaller

[https://github.com/microsoft/vscode/issues/199944](https://github.com/microsoft/vscode/issues/199944)

## Summary

The hover max-width is still computed as 66% of editor width (min 750px) on current main, and no color picker width cap was merged. The issue remains valid as a feature request for better default sizing and persistence of user-resized dimensions.

## Change summary

The color picker hover is excessively wide (scales with editor width at 66%) and does not persist user resize to a smaller size across different hover invocations.

## Best solution

Cap the color picker width independently of the hover content width (the approach discussed in the thread), and consider lowering the hover max-width default or adding a user setting for max natural hover dimensions (as proposed in the related https://github.com/microsoft/vscode/issues/200356). The relevant code is in `src/vs/editor/contrib/hover/browser/contentHoverWidget.ts` line 309.
