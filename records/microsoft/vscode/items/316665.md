# Issue #316665 — Right-click in an unfocused window opens the context menu over the focused window

[https://github.com/microsoft/vscode/issues/316665](https://github.com/microsoft/vscode/issues/316665)

## Summary

Keep open and route as a workbench auxiliary-window/context-menu bug. The report has concrete steps and current code still has paths that can anchor context menus to the active/focused window rather than the window that received the right-click.

## Change summary

Right-clicking an unfocused auxiliary or main window should open the context menu in the clicked window, not over the currently focused window.

## Best solution

Route to the workbench auxiliary window owners and make both native and custom context menu paths carry/use the target event or anchor window instead of defaulting to the active container/focused BrowserWindow. Add coverage for context menus opened from an unfocused auxiliary window, ideally for both HTML and native menu styles.
