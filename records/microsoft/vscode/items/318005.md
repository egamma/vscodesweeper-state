# Issue #318005 — Customizations loader doesn't validate path or available file system provider

[https://github.com/microsoft/vscode/issues/318005](https://github.com/microsoft/vscode/issues/318005)

## Summary

Keep open for maintainer handling: this is a team-authored issue, which the cleanup policy says not to close automatically, and it reports a plausible missing guard in the customizations loading path for unknown workspaces or unsupported URI schemes.

## Change summary

Clicking an old agent session with an `unknown` workspace causes many errors because the customizations loader still checks paths without validating the workspace scheme or file system provider availability.

## Best solution

Add or confirm a guard in the customizations loading path so it skips customization checks when the workspace is unknown or when no file system provider is registered for the URI scheme; the agents/customizations owner should decide the exact product boundary.
