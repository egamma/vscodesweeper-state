# Issue #318005 — Customizations loader doesn't validate path or available file system provider

[https://github.com/microsoft/vscode/issues/318005](https://github.com/microsoft/vscode/issues/318005)

## Summary

Team-authored issue (MEMBER) — requires explicit maintainer judgment. The report describes a validation gap in the customizations loader when no file system provider matches the workspace scheme.

## Change summary

Opening an old session in the agents window with an unknown workspace triggers errors because the customizations loader doesn't validate the path or check for an available file system provider.

## Best solution

Add a guard in the customizations loader to skip loading when the workspace is unknown or when no file system provider is registered for the URI scheme, likely in `src/vs/workbench/contrib/chat/browser/aiCustomization/` area.
