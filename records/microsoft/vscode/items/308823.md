# Issue #308823 — Copilot Chat: SSE stream inactivity timeout (60s) should be configurable — causing mid-stream failures under backend congestion

[https://github.com/microsoft/vscode/issues/308823](https://github.com/microsoft/vscode/issues/308823)

## Summary

Valid feature request: the SSE stream inactivity timeout (60s) in the Copilot networking layer is still hardcoded on current main with no user-facing setting. The team has been actively working on stream idle handling (an idle watchdog was added and reverted in June 2026), indicating awareness of the problem space, but no configurable timeout has landed yet.

## Change summary

Request to make the Copilot Chat SSE stream inactivity timeout configurable (currently hardcoded at 60s in nodeFetcher.ts), as backend congestion can cause mid-stream pauses exceeding the limit.

## Best solution

Expose the inactivity timeout in `extensions/copilot/src/platform/networking/node/nodeFetcher.ts` (line 119) as a VS Code advanced setting (e.g. `github.copilot.advanced.streamIdleTimeoutMs`) with a reasonable default (60–120s). The recent idle-watchdog work on Anthropic streaming suggests the team is already investing in this area; this request extends configurability to the base fetcher layer.
