# Issue #296038 — When /compact fails for auth token refresh, the error says otherwise.

[https://github.com/microsoft/vscode/issues/296038](https://github.com/microsoft/vscode/issues/296038)

## Summary

The report describes a misleading error message when /compact hits a 401 auth-token-refresh race, but the issue is 150 days old with significant compaction and auth changes since. The error handling path still surfaces raw error text, but the underlying auth refresh timing may have improved. Requesting confirmation on the latest build.

## Change summary

When /compact fails due to an expired auth token, the error message shown to the user is misleading and lacks the retry UI that other chat flows provide.

## Best solution

The error handler in `handleSummarizeCommand` (extensions/copilot/src/extension/intents/node/agentIntent.ts, line ~609–617) catches errors and shows the raw message string. Ideally, 401/auth errors should either be retried transparently (as other request paths do) or surface a user-friendly message with a retry action. If this still reproduces, the fix belongs in the Copilot extension's compaction error path — either by integrating the same auth-retry logic used by the main chat request flow, or by catching auth-specific errors and presenting the standard retry UI.
