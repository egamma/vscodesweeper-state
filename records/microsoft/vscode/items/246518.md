# Issue #246518 — MCP: Input preview can be confused with looking like in-file configuration

[https://github.com/microsoft/vscode/issues/246518](https://github.com/microsoft/vscode/issues/246518)

## Summary

Team-authored UX polish issue that remains valid — the MCP input inlay hints still use a plain '= <value>' label with no distinguishing prefix or formatting, and the confusion reported has not been addressed on main.

## Change summary

MCP input value inlay hints in .mcp.json look like hardcoded in-file configuration, confusing developers into thinking secrets are written to the file.

## Best solution

Add a distinguishing prefix (e.g. 'saved:', 'input:') or visual differentiation to the inlay hint label in `src/vs/workbench/contrib/mcp/browser/mcpLanguageFeatures.ts` so users can immediately tell these are transient saved-input previews, not persisted file content.
