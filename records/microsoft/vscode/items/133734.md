# Issue #133734 — Extend `vscode.extensions.getExtension` to return extensions from other extension host

[https://github.com/microsoft/vscode/issues/133734](https://github.com/microsoft/vscode/issues/133734)

## Summary

Keep open: current main has a proposed `extensionsAny` API that covers the metadata lookup case, but the stable API still exposes only the one-argument `vscode.extensions.getExtension`. This should not be closed as completed unless maintainers decide the proposed API is sufficient or graduate it.

## Change summary

Allow `vscode.extensions.getExtension` to return metadata for extensions running in a different extension host without enabling cross-host extension API use.

## Best solution

Continue tracking this under `api` and `extension-host` until the `extensionsAny` proposed API is either finalized or explicitly accepted as the solution; keep cross-host `exports` and activation unavailable as designed.

- **Proposed comment:**

  > Thanks for the context. This is now partially covered by the proposed `extensionsAny` API: extensions can opt in to query metadata across extension hosts, while `exports` remains unavailable for foreign extensions. Since the stable API still only exposes the one-argument `getExtension`, I would keep this open until we either finalize that proposed API or explicitly decide that the proposed API is the intended endpoint.

## Evidence

- **current implementation:** src/vs/workbench/api/common/extHost.api.impl.ts implements `getExtension(extensionId, includeFromDifferentExtensionHosts?)`; when the proposed `extensionsAny` API is enabled it looks in `extensionInfo.all` and returns a foreign extension wrapper.
- **metadata-only behavior:** src/vs/workbench/api/common/extHostExtensionService.ts marks `Extension.isFromDifferentExtensionHost` and returns `undefined` from `exports`, matching jrieken's 2024-08-06 clarification that this is metadata-only and not cross-host API use.
- **proposed API:** src/vscode-dts/vscode.proposed.extensionsAny.d.ts declares `getExtension(extensionId, includeDifferentExtensionHosts)` and `allAcrossExtensionHosts`; src/vscode-dts/vscode.d.ts still declares only `getExtension(extensionId)`.
- **history:** Commit ad4470522ecd858cfaf53a87c2702d7a40946ba1 introduced the proposed `extensionsAny` API with the message `propose extensions.allAcrossExtensionHosts and related APIs` and references https://github.com/microsoft/vscode/issues/145307, not this issue directly.
