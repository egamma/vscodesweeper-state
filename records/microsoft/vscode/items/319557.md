# Issue #319557 — Prevent Escape key from accidentally closing the askQuestions dialog

[https://github.com/microsoft/vscode/issues/319557](https://github.com/microsoft/vscode/issues/319557)

## Summary

Valid feature request: the askQuestions carousel intentionally dismisses on a single Escape press (line 234 of chatQuestionCarouselPart.ts). No existing issue tracks making this more accident-proof. Keeping open for area-owner consideration.

## Change summary

Request to prevent accidental Escape keypresses from immediately dismissing the askQuestions carousel, losing in-progress answers.

## Best solution

Add a double-Escape or confirmation guard before calling `ignore()` in `src/vs/workbench/contrib/chat/browser/widget/chatContentParts/chatQuestionCarouselPart.ts` (line 234). A first Escape could defocus/blur the carousel, with a second Escape within a short timeout window actually dismissing it — similar to patterns used elsewhere in VS Code for destructive dismiss actions.
