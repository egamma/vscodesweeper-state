## Summary

The title wrapping bug was traced to `.title.modified` lacking `overflow: hidden` and `text-overflow: ellipsis` in the multi-diff editor CSS. A fix landed on main today (af32fc9c27a) that extends overflow protection to all `.title` elements — this very likely resolves the reported wrapping, but it was just committed and doesn't reference this issue, so a maintainer should verify before closing.

- **root cause:** At time of filing, only `.title.original` had `min-width: 0` and `text-overflow: ellipsis` in src/vs/editor/browser/widget/multiDiffEditor/style.css; `.title.modified` (the primary path shown first) had no overflow protection, causing wrapping.
- **likely fix commit:** Commit af32fc9c27ab82bc4a004b8a475c6e8c6c5ff412 ('Editor - resource label should use text-overflow', PR https://github.com/microsoft/vscode/pull/324819) moved `min-width: 0`, `overflow: hidden`, and `text-overflow: ellipsis` to all `.title` elements — landed on main 2026-07-08.
- **icon label provides nowrap:** src/vs/base/browser/ui/iconLabel/iconlabel.css uses `white-space: pre` on labels, so combined with the new overflow rules, titles will correctly truncate with ellipsis.
