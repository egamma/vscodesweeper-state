# Issue #136792 — Allow Run To Cursor to start debug session

[https://github.com/microsoft/vscode/issues/136792](https://github.com/microsoft/vscode/issues/136792)

## Summary

This feature request has been implemented on main and shipped since VS Code 1.76.0. The `runTo` method in `debugService.ts` now handles the inactive-debugger case by starting a debug session with a temporary breakpoint at the cursor.

## Change summary

Allow the 'Run to Cursor' debug action to start a new debug session when no session is active, similar to Visual Studio behavior.

## Best solution

Keep the shipped implementation — `debugService.runTo()` already starts a debug session when `state === State.Inactive`, sets a temporary breakpoint at the cursor line, and cleans it up on stop. No further change needed.
