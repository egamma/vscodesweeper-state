# Issue #279299 — Leaked disposables in chat renderers

[https://github.com/microsoft/vscode/issues/279299](https://github.com/microsoft/vscode/issues/279299)

## Summary

All three leaked disposable sources from the stack traces have been fixed on main across three commits between Nov 27 and Dec 16, 2025, and shipped in 1.107.0. The current code properly manages disposable lifetimes for tool invocation parts, markdown hover stores, and autorun reader stores.

## Change summary

Chat renderers leaked disposables in ChatToolInvocationPart, ChatContentMarkdownRenderer.attachCustomHover, and ChatToolProgressSubPart autorun stores.

## Best solution

Keep the shipped fixes; no further change needed. The three fixing commits addressed each leak source: tool invocation parts now use addDisposable, the custom hover DisposableStore is composed into the returned IRenderedMarkdown.dispose(), and the autorun uses reader.store.add() correctly.
