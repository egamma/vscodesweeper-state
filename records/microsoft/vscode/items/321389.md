# Issue #321389 — UI for cancelling queued messages is broken

[https://github.com/microsoft/vscode/issues/321389](https://github.com/microsoft/vscode/issues/321389)

## Summary

Team-authored bug report (MEMBER) — the cancel/remove-from-queue toolbar on queued messages in the Agents window is mispositioned or invisible on hover. The `.request-hover` toolbar is absolutely positioned and the pending-request CSS override appears to cause layout issues. Keeping open per team-author policy.

## Change summary

Hovering a queued message in the Agents window shows the cancel toolbar either not at all or pushed to the far right, making it unusable.

## Best solution

Fix the positioning of the `.request-hover` toolbar for `.pending-request` items — likely in `src/vs/workbench/contrib/chat/browser/widget/media/chat.css` (around line 4288–4294) and possibly in the `chatListRenderer.ts` hover/toolbar setup. The `top: -17px !important` override and the `right: 20px` absolute positioning may need adjustment for the pending-request layout context.
