# Issue #125261 — Leading whitespace before bracket/brace should not be automatically removed

[https://github.com/microsoft/vscode/issues/125261](https://github.com/microsoft/vscode/issues/125261)

## Summary

Still reproduces on main: the electric character operation adjusts indentation on closing brackets regardless of the autoIndent setting, because InterceptorElectricCharOperation never checks config.autoIndent.

## Change summary

Typing a closing bracket/brace removes intentional leading whitespace even when editor.autoIndent is set to "keep".

## Best solution

Guard InterceptorElectricCharOperation._typeInterceptorElectricChar to bail out (return null) when config.autoIndent < EditorAutoIndentStrategy.Brackets, similar to how AutoIndentOperation checks config.autoIndent < Full. The Brackets level is the appropriate threshold since the electric character feature is fundamentally about bracket-matched indentation.
