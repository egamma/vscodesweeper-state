# Issue #221873 — Add css variables with editor font settings

[https://github.com/microsoft/vscode/issues/221873](https://github.com/microsoft/vscode/issues/221873)

## Summary

Keep open and route to editor: the central cleanup is still only partially addressed on current main. There is a root editor font-size variable now, but widgets such as parameter hints still manually read `EditorOption.fontInfo` and maintain their own font-related styles/listeners.

## Change summary

Add shared editor-scoped CSS variables for editor font settings so editor UI can render code without duplicating `fontInfo` reads and configuration listeners.

## Best solution

Have editor code publish stable CSS variables for the editor font settings on the editor root, then migrate editor UX widgets such as parameter hints, hover, suggest, and similar code-rendering surfaces to consume those variables where layout metrics are not required.
