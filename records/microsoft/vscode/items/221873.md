# Issue #221873 — Add css variables with editor font settings

## Summary

Partial progress has been made — `--editor-font-size` is now set on the editor DOM element and some consumers use CSS variables — but the full systematic rollout (font-family, font-weight, line-height as broadly-available CSS variables plus migrating all consumers) is not complete. ## Change summary

Expose editor font settings (size, family, weight, line-height) as CSS variables so UX code can use them instead of reading EditorOption.fontInfo directly.

## Best solution
