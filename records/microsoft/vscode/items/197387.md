# Issue #197387 — terminal confirm on exit dialog appears when no child process running after reload

[https://github.com/microsoft/vscode/issues/197387](https://github.com/microsoft/vscode/issues/197387)

## Summary

Confirmed bug still present on main: the `hasChildProcesses` getter short-circuits on a stale persistence snapshot, causing a spurious confirm dialog after reload. An open PR (#311860) addresses this.

## Change summary

Terminal 'confirm on exit' dialog incorrectly fires after window reload when the child process has already exited, because the persisted snapshot is never updated with the live process state.

## Best solution

Land PR https://github.com/microsoft/vscode/pull/311860, which updates `attachPersistentProcess.hasChildProcesses` from the live `HasChildProcesses` property event so the stale snapshot no longer masks the real state.
