# Issue #198399 — Mutilple cursor also includes wrapped lines

[https://github.com/microsoft/vscode/issues/198399](https://github.com/microsoft/vscode/issues/198399)

## Summary

Still reproduces on main: Alt+Shift+Click uses the column-select code path which operates on view lines (including wrapped lines), placing cursors on each visual line. The 2021 keyboard fix (InsertCursorAbove/Below) did not extend to the mouse path.

## Change summary

Alt+Shift+Click to create multi-cursors across lines erroneously places cursors on wrapped (virtual) lines in addition to real lines.

## Best solution

The mouse-triggered column select path in `viewController.ts` (line 284–285) should either skip wrapped view lines when creating cursors, or use model line numbers instead of view line numbers when the intent is multi-cursor creation (not box/column selection). The fix likely involves `src/vs/editor/browser/view/viewController.ts` and possibly `src/vs/editor/common/cursor/cursorColumnSelection.ts`.
