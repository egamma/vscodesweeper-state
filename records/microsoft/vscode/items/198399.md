# Issue #198399 — Mutilple cursor also includes wrapped lines

[https://github.com/microsoft/vscode/issues/198399](https://github.com/microsoft/vscode/issues/198399)

## Summary

Keep open as a plausible editor multi-cursor bug: the earlier #26393 fix covered keyboard insert-cursor commands, while the mouse column-selection path still operates over view line numbers, which can include wrapped visual lines.

## Change summary

Alt+Shift mouse column selection creates an extra cursor on a wrapped continuation line instead of only on the two real text lines.

## Best solution

Fix the mouse column-selection path in `src/vs/editor/browser/view/viewController.ts` / `src/vs/editor/browser/coreCommands.ts` so wrapped continuation view lines are not treated as separate target rows for this interaction, and add a regression test analogous to the existing #26393 keyboard test.
