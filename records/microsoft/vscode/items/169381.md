# Issue #169381 — Input variables quoted incorrectly

[https://github.com/microsoft/vscode/issues/169381](https://github.com/microsoft/vscode/issues/169381)

## Summary

Still reproduces on main: the `quote` function in terminalTaskSystem.ts naively wraps a value with shell quote characters without escaping embedded quotes, causing values like `';'` to produce broken bash output `'';''`.

## Change summary

Task input variables containing single quotes are incorrectly quoted when using bash shell's strong or escape quoting, producing broken command lines.

## Best solution

The `quote` function in `src/vs/workbench/contrib/tasks/browser/terminalTaskSystem.ts` (line ~1660) needs to handle embedded quote characters — for bash strong quoting this requires the `'foo'\''bar'` concatenation pattern since backslash has no meaning inside single quotes. The escape quoting path should already work (it escapes `'` with `\`), so the primary gap is strong quoting with embedded single quotes.
