# Issue #271711 — Failed to apply code block: Response contained no choices.

[https://github.com/microsoft/vscode/issues/271711](https://github.com/microsoft/vscode/issues/271711)

## Summary

Still a live issue on main: the instant-apply code path surfaces a raw 'Response contained no choices' error from the Copilot backend with no graceful recovery, leaving the editor stuck. A team member (aeschli) engaged asking about improving the error handling. 16 upvotes and reports across all platforms confirm broad impact — keeping open.

## Change summary

Applying a chat code block via the instant-apply feature fails with 'Response contained no choices' and leaves the editor in a stuck state requiring a restart.

## Best solution

Two improvements needed: (1) the Copilot extension's instant-apply request path should handle a no-choices response gracefully (retry or fall back to direct insertion) rather than throwing — similar to the fix in b9df6378549 for the xtab/NES path; (2) `src/vs/workbench/contrib/chat/browser/actions/codeBlockOperations.ts` should recover from this error class instead of leaving the editor locked. The backend model reliability (copilot-api side, tracked at https://github.com/github/copilot-api/issues/23163) is the upstream root cause.
