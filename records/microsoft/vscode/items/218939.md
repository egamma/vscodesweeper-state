# Issue #218939 — Inline diff is at wrong position

[https://github.com/microsoft/vscode/issues/218939](https://github.com/microsoft/vscode/issues/218939)

## Summary

Still reproduces on main: the inline diff deleted-code view zone doesn't set `afterColumn`, so with word wrap it positions mid-wrapped-line instead of after the full line content.

## Change summary

Inline diff deleted-code indicator appears in the middle of a word-wrapped line instead of after the last visual line of the wrap.

## Best solution

Set `afterColumn: Constants.MAX_SAFE_SMALL_INTEGER` on the deleted-code view zone (line ~278 in `diffEditorViewZones.ts`) so it renders after all wrapped content, consistent with how codelens zones handle this.
