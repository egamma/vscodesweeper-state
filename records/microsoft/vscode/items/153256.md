## Evidence

- **requested behavior:** The issue asks for ESM files in `monaco-editor-core` to pass through the NLS transformation instead of being built only by plain `tsc`.
- **fix commit:** Commit 91f0cb7b907fc1d5b869388ca5d55f0af11bc7f0 is titled `Add nls info to monaco-editor-core and remove left-over AMD tasks` and changes `build/gulpfile.editor.js` to pipe `compile-editor-esm` through `i18n.processNlsFiles(...)`; see https://github.com/microsoft/vscode/commit/91f0cb7b907fc1d5b869388ca5d55f0af11bc7f0.
- **current main:** Current `build/gulpfile.editor.ts` lines 71-88 compile `out-editor-src` with `compilation.createCompile(... preserveEnglish: true ...)`, then pipe through `i18n.processNlsFiles({ out: 'out-monaco-editor-core/esm', ... })` before writing to `out-monaco-editor-core/esm`.
- **nls generation:** Current `build/lib/compilation.ts` pipes build output through `nls.nls({ preserveEnglish })`, and `build/lib/i18n.ts` generates `nls.messages.<language>.js` when it sees `nls.keys.json`, matching the requested localization support for ESM output.
- **release:** `git tag --contains 91f0cb7b907fc1d5b869388ca5d55f0af11bc7f0` includes `1.99.0`, so the implementation is present in VS Code 1.99.0 and later.
