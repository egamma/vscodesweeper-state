High-demand feature request (1049 upvotes) with an open community PR (#214051) still unmerged. The Diagnostic API on current main remains string-only — no markdown support exists. Keeping open as the work is not yet done.

## Change summary

Allow diagnostic messages to contain markdown or formatted text (bold, links, indentation) for richer error display in the editor hover and Problems panel.

## Best solution

Land a markdown-capable diagnostic message API (e.g. an optional `MarkdownString` property on `Diagnostic`) with appropriate rendering in both the editor hover and the Problems panel.
