# Issue #48810 — Does the watch view support auto completions?

[https://github.com/microsoft/vscode/issues/48810](https://github.com/microsoft/vscode/issues/48810)

## Summary

Still unimplemented on main: the watch expression input uses a plain InputBox with no completion support. With 33 community upvotes (well above the 20-upvote backlog threshold), this is a well-supported feature request that should remain open.

## Change summary

Add auto-completion support to the debug watch view's expression input, similar to what the debug console already provides.

## Best solution

Replace the plain InputBox in the watch expression renderer (src/vs/workbench/contrib/debug/browser/baseDebugView.ts) with a CodeEditorWidget (or add a completion provider to the input) that can leverage the debug adapter's completions request — the same mechanism the debug console (replView) already uses. This would require plumbing the debug session's completions provider into the watch input flow.
