# Issue #276603 — Clicking Try Again on a failed request should clear the error and resend the last network request to the model

[https://github.com/microsoft/vscode/issues/276603](https://github.com/microsoft/vscode/issues/276603)

## Summary

Team-authored feature request (MEMBER) — keeping open. The ask is for the error-state "Try Again" button to clear the error and resend the last network request rather than posting a new chat message like "User selected 'Try again'".

## Change summary

The "Try Again" button on a failed chat response should dismiss the error UI and resend the original request to the model instead of sending a new message.

## Best solution

In the error confirmation button handler (ChatErrorConfirmationContentPart), replace the `chatService.sendRequest` call with `chatService.resendRequest` targeting the failed request — the latter already removes the prior request and re-sends. This would clear the error from the UI and retry the original network call. Likely files: `src/vs/workbench/contrib/chat/browser/widget/chatContentParts/chatErrorConfirmationPart.ts` and the error rendering in `chatListRenderer.ts`.
