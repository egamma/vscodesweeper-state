# Issue #276326 — Search: line number should be wider for coherent preview

[https://github.com/microsoft/vscode/issues/276326](https://github.com/microsoft/vscode/issues/276326)

## Summary

Still reproduces on main: trailing context lines in the search editor output are not padded to align with match lines, causing misaligned text when line numbers have different digit counts.

## Change summary

Search editor result preview misaligns text on trailing context lines because their line numbers are not padded to match the width of the longest line number.

## Best solution

Fix the trailing-context formatting in `src/vs/workbench/contrib/searchEditor/browser/searchEditorSerialization.ts` line 111 to use the same `longestLineNumber`-based padding as line 95. The two context-line emission paths should use a shared helper.
