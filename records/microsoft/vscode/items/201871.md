# Issue #201871 — Unnecessary calls to 'codeAction/resolve' on mouse hovers

[https://github.com/microsoft/vscode/issues/201871](https://github.com/microsoft/vscode/issues/201871)

## Summary

Already fixed: the unnecessary `action.resolve()` call on hover was removed in commit 228f3ffc595 (Dec 2023) and shipped in 1.86.0. The current `onHover` delegate no longer triggers code action resolution — it only checks the action kind and existing edit property to determine preview eligibility.

## Change summary

Hovering over code actions in the lightbulb menu triggered costly `codeAction/resolve` LSP calls for each item, degrading performance.

## Best solution

Keep the shipped fix from commit 228f3ffc595. No further change needed — the `onHover` handler in `codeActionController.ts` no longer calls `action.resolve()`. The second commenter's concern about distinguishing hover-triggered vs. selection-triggered resolves is also moot since resolve is no longer called on hover at all.
