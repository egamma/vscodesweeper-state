# Issue #301178 — Copilot slow when delete code

[https://github.com/microsoft/vscode/issues/301178](https://github.com/microsoft/vscode/issues/301178)

## Summary

The user confirmed the typing slowness disappears when the GitHub Copilot Chat extension is disabled, and two maintainers found nothing abnormal in the CPU profile from the VS Code core side. This is caused by the Copilot Chat extension on a resource-constrained system (0.65 GB free RAM, high load averages).

## Change summary

Deleting code becomes slow/blocked while Copilot inline completions are loading, making the editor feel unresponsive.

## Best solution

Close as caused-by-extension and direct the user to report the performance issue against the Copilot extension (https://github.com/microsoft/vscode-copilot-release). The related open issue https://github.com/microsoft/vscode/issues/299786 already tracks a similar Copilot-induced typing performance problem with the `inline-completions` and `perf` labels — any core-side improvements would land there.
