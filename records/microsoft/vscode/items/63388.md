# Issue #63388 — Indent/Outdent with tab key does not honor editor.autoIndent=false

[https://github.com/microsoft/vscode/issues/63388](https://github.com/microsoft/vscode/issues/63388)

## Summary

Still reproduces on main: the ShiftCommand's useTabStops path computes each line's indentation independently by snapping to absolute tab stops, which destroys relative alignment within a multi-line selection. A team member confirmed the bug in March 2024 and reopened it; it carries `tree-sitter-autoindent-candidate`, suggesting it may be addressed as part of that work.

## Change summary

Indenting/outdenting a multi-line selection breaks relative indentation between lines because each line is independently snapped to the nearest tab stop.

## Best solution

In `src/vs/editor/common/commands/shiftCommand.ts`, the `useTabStops` path should compute the indent delta from the first (or least-indented) line and apply the same relative shift to all lines in the selection, preserving intra-selection alignment. The `tree-sitter-autoindent-candidate` label suggests this may be addressed as part of tree-sitter autoindent work.
