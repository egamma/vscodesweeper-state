# Issue #63388 — Indent/Outdent with tab key does not honor editor.autoIndent=false

[https://github.com/microsoft/vscode/issues/63388](https://github.com/microsoft/vscode/issues/63388)

## Summary

Still unfixed on main — indent/outdent does not preserve relative indentation of lines within a selection when some lines have sub-tab-stop offsets. Issue is on the 'On Deck' milestone, confirming maintainer intent to address it.

## Change summary

Indenting or outdenting a multi-line selection collapses lines with sub-tab-stop relative indentation to the nearest tab stop, breaking alignment.

## Best solution

Modify the shift (indent/outdent) command in `src/vs/editor/common/commands/shiftCommand.ts` to preserve relative indentation offsets between lines in a selection, rather than snapping each line independently to the next tab stop. The `tree-sitter-autoindent-candidate` label suggests this may also be addressed via tree-sitter-based indentation logic.
