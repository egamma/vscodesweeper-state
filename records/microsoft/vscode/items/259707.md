# Issue #259707 — Consolidate File Changes response part

[https://github.com/microsoft/vscode/issues/259707](https://github.com/microsoft/vscode/issues/259707)

## Summary

Valid internal debt: the two file-changes response parts (ChatCheckpointFileChangesSummaryContentPart in chatChangesSummaryPart.ts and the multi-diff part in chatMultiDiffContentPart.ts) remain separate implementations on current main. Author is a contributor, so this stays open for maintainer judgment.

## Change summary

Consolidate the 'File Changes' chat response part (from PR #257906) with the existing Checkpoint File Changes summary part, which has richer presentation (e.g., changed-line counts).

## Best solution

Unify chatChangesSummaryPart.ts and chatMultiDiffContentPart.ts into a single file-changes content part that supports both checkpoint-sourced diffs and extension-contributed diffs, sharing the richer changed-lines presentation. The IChatResponseFileChangesService already provides a provider abstraction that could feed both use cases.
