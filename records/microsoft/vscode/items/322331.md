# Issue #322331 — files.exclude glob pattern: negation does not work with the path separator /

[https://github.com/microsoft/vscode/issues/322331](https://github.com/microsoft/vscode/issues/322331)

## Summary

The behavior is intentional: VS Code's glob implementation follows the POSIX glob(7) specification, which explicitly prohibits matching `/` inside bracket expressions. The code at `src/vs/base/common/glob.ts:175-179` silently strips `/` from character classes by design, with a comment citing the man page.

## Change summary

The glob pattern `[^M/]` in `files.exclude` silently strips the `/` from the bracket expression, so `Source[^M/]*` unexpectedly matches paths under `Source/`.

## Best solution

Close as designed — the POSIX glob spec prohibits `/` inside bracket expressions, and VS Code intentionally follows this. The reporter's intent (exclude everything except `Source/...` and `SourceM...`) would require a different pattern structure, such as separate include/exclude entries. The broader "exclude all except" capability is tracked in https://github.com/microsoft/vscode/issues/869.
