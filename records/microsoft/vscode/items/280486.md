# Issue #280486 — chat will always error after attaching an image that exceeds the limit

[https://github.com/microsoft/vscode/issues/280486](https://github.com/microsoft/vscode/issues/280486)

## Summary

The specific 8000px dimension error is likely mitigated by the image resize pipeline (images are now capped at 2048px before sending), but the broader UX issue — that a failed attachment persists in chat history and poisons all subsequent requests — remains a real gap. A team member acknowledged the behavior; keeping open for the history-cleanup aspect.

## Change summary

After attaching an image that exceeds the API size limit, the image remains in chat history and causes all subsequent chat requests to fail with the same error, even after restoring checkpoints.

## Best solution

The resize pipeline in `chatImageUtils.ts` now caps images at 2048px, preventing the 8000px error. The remaining UX gap is that when a request fails due to an attachment, users have no clear way to remove the offending attachment from history. Ideally, failed requests with attachment errors should either auto-strip the problematic attachment or surface a clear 'remove attachment' action so the conversation can continue.
