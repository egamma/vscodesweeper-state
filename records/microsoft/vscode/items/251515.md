# Issue #251515 — Support Storing Toolset Files Directly Within the Workspace

[https://github.com/microsoft/vscode/issues/251515](https://github.com/microsoft/vscode/issues/251515)

## Summary

Still a valid open request. Toolsets are currently only loaded from the user profile's promptsHome directory, not from workspace folders. The feature to store .toolsets.jsonc files within the workspace (e.g. .github/prompts/ or .vscode/) has not been implemented on main.

## Change summary

Allow .toolsets.jsonc files to be stored in and loaded from workspace folders for version-controlled, team-shared tool configuration.

## Best solution

Extend `toolSetsContribution.ts` to scan workspace folders (likely `.github/prompts/` alongside prompt files) for `.toolsets.jsonc` files, similar to how prompt files are already discovered from workspace roots. This keeps tool configuration version-controlled and team-shareable.
