# Issue #251515 — Support Storing Toolset Files Directly Within the Workspace

[https://github.com/microsoft/vscode/issues/251515](https://github.com/microsoft/vscode/issues/251515)

## Summary

Still a valid unimplemented feature request. Toolset files are currently only read from the user profile's promptsHome folder, not from workspace directories. The issue has strong community support (44 upvotes) and team endorsement.

## Change summary

Allow `.toolsets.jsonc` files to be stored directly in the workspace (e.g., `.vscode/` or `.github/`) for version control and team sharing.

## Best solution

Extend `UserToolSetsContributions._initToolSets()` in `src/vs/workbench/contrib/chat/browser/tools/toolSetsContribution.ts` to also scan workspace folders (e.g., `.vscode/` or `.github/`) for `.toolsets.jsonc` files, in addition to the existing user profile `promptsHome` folder. This aligns with how other workspace-scoped config (settings, tasks, launch) already works.
