# Issue #326519 — Silent Git auto-fetch can still open Git Credential Manager dialogs

[https://github.com/microsoft/vscode/issues/326519](https://github.com/microsoft/vscode/issues/326519)

## Summary

Valid bug, confirmed in source: silent auto-fetch only suppresses VS Code's own askpass but not GCM's interactive prompts. The gap is in VS Code's git extension, not solely in GCM. No fix on main.

## Change summary

Silent Git auto-fetch (`silent: true`) does not set `GCM_INTERACTIVE=0`, so Git Credential Manager can still open interactive authentication dialogs during background fetches.

## Best solution

In `extensions/git/src/git.ts` around line 2412, when `options.silent` is true, also set `GCM_INTERACTIVE=0` (and possibly `GIT_TERMINAL_PROMPT=0`) in `spawnOptions.env` so that credential managers respect the non-interactive intent. This is a small, localized change.
