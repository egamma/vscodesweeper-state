# Issue #278130 — Pressing `F5` doesn't capture the active file for `${file}` immediately sometimes resulting in launching the wrong file

[https://github.com/microsoft/vscode/issues/278130](https://github.com/microsoft/vscode/issues/278130)

## Summary

Valid bug with an open fix PR (#310790). The `${file}` variable is resolved lazily from `editorService.activeEditor` after async work (extension activation, save-all) completes, creating a race window where the active editor can change. Keeping open pending the PR.

## Change summary

Pressing F5 with `${file}` in a launch config can resolve to the wrong file if the user switches editors during the async debug startup sequence.

## Best solution

Snapshot the active editor at the moment F5 is pressed and pass that snapshot through to variable resolution. PR https://github.com/microsoft/vscode/pull/310790 attempts this — review and land it, or implement the snapshot in `debugService.ts` before the async calls begin and thread it to `baseConfigurationResolverService.ts`'s `getFilePath`.
