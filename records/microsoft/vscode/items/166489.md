# Issue #166489 — Support alt+enter in terminal quick fixes to add a command but not execute it

[https://github.com/microsoft/vscode/issues/166489](https://github.com/microsoft/vscode/issues/166489)

## Summary

Still unimplemented on main — the relevant code in quickFix.ts contains a TODO comment for exactly this feature. Issue has an 'On Deck' milestone, so it stays open for maintainer scheduling.

## Change summary

Support alt+enter in terminal quick fixes to paste a command into the terminal without executing it, matching the behavior of the 'run recent command' feature.

## Best solution

Implement alt+enter handling in the terminal quick fix code action menu so that selecting a quick fix with alt held writes the command to the terminal input without executing. The likely change site is `src/vs/workbench/contrib/terminalContrib/quickFix/browser/quickFix.ts` (the existing TODO on line 71) and the quickFixAddon that dispatches the action. This also requires the code action menu to surface an alt-key modifier to the terminal quick fix handler.
