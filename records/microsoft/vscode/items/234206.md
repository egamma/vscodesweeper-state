# Issue #234206 — github.dev search options no longer work

[https://github.com/microsoft/vscode/issues/234206](https://github.com/microsoft/vscode/issues/234206)

## Summary

Legitimate bug: search options (Match Case, Match Whole Word, Use Regular Expression) are ignored in github.dev when the repository content has not been fully downloaded. The commenter confirmed the options work after downloading the full index. Already triaged and assigned to osortega in Backlog — keeping open.

## Change summary

In github.dev, the search view's Match Case, Match Whole Word, and Use Regular Expression toggle buttons have no effect on results until the full repository content is downloaded.

## Best solution

The TextSearchProvider registered by the remote-repositories-github extension likely ignores the query flags (isCaseSensitive, isWordMatch, isRegExp) when searching via the GitHub Code Search API. Either forward those flags to the API where supported, or surface a clear UX indication that these options require a full content download. The fix likely lives in microsoft/vscode-remote-repositories-github, not in this repo's search service — the assignee should coordinate with that extension's codebase.
