# Issue #234206 — github.dev search options no longer work

[https://github.com/microsoft/vscode/issues/234206](https://github.com/microsoft/vscode/issues/234206)

## Summary

The search options (case, whole word, regex) are correctly passed by VS Code core to extension-provided TextSearchProviders — the bug is in the remote repositories extension (microsoft/vscode-remote-repositories-github) which powers github.dev search and apparently ignores these flags when operating on its partial index. This is a Microsoft-owned extension with its own tracker, so the issue should be moved there.

## Change summary

In github.dev, search options (Match Case, Match Whole Word, Use Regular Expression) have no effect when the full repository content has not been downloaded.

## Best solution

Move this issue to microsoft/vscode-remote-repositories-github. The remote repositories extension's TextSearchProvider needs to honor the query flags (isCaseSensitive, isRegExp, isWordMatch) even when serving results from its partial/cached index rather than the full downloaded content.
