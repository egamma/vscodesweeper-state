The requested open-source localization tooling was delivered via the microsoft/vscode-l10n repository (created August 2022), which provides @vscode/l10n-dev for extracting and working with XLF files and @vscode/l10n for loading translations — exactly the pipeline the issue asked to be open-sourced. ## Change summary

Request to open-source the translation pipeline so third-party VS Code extensions (like Python) can be built from source with translations enabled.

## Best solution

The work is done: microsoft/vscode-l10n ships the open-source l10n tooling (extraction, XLF handling, runtime loading) and the vscode.l10n.t API (since VS Code 1.73) provides the in-extension translation surface. No further core change is needed.
