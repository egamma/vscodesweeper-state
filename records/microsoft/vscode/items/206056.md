# Issue #206056 — CSS for debug icons applies to TreeItem inline commands

[https://github.com/microsoft/vscode/issues/206056](https://github.com/microsoft/vscode/issues/206056)

## Summary

Still reproduces on main: most debug icon CSS selectors in debugColors.ts use overly broad `.monaco-workbench .codicon-debug-*` patterns that color-bleed into extension TreeItem inline commands and other non-debug contexts. Only the disconnect icon was scoped after #208089; all other debug icons remain unscoped.

## Change summary

Debug icon CSS colors (step-back, step-over, pause, stop, etc.) bleed into extension TreeItem inline commands because the selectors are too broad.

## Best solution

Scope all debug icon CSS rules in `src/vs/workbench/contrib/debug/browser/debugColors.ts` (lines 304–349) the same way the disconnect icon was scoped in commit 5d8491d7130 — restrict them to `.debug-view-content`, `.debug-toolbar`, `.command-center-center`, and the title-actions bar, instead of matching any `.codicon-debug-*` anywhere in the workbench.
