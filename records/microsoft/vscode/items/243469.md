# Issue #243469 — Test Peek Output Window and Test Results Tab don't support file links.

[https://github.com/microsoft/vscode/issues/243469](https://github.com/microsoft/vscode/issues/243469)

## Summary

Still reproduces on main: the test results output view uses a detached terminal whose external link providers are explicitly excluded and whose built-in file link detection has CWD timing issues. A related issue #318548 confirms the same gap from the extension link provider angle.

## Change summary

File paths in the Test Peek Output Window and Test Results output tab are not clickable links, unlike in the integrated terminal.

## Best solution

Fix link detection in the detached terminal used by test output: (1) enable extension-provided link providers for detached terminals in `terminal.links.contribution.ts` (currently gated by `isDetachedTerminalInstance` on line 72), and (2) ensure the `initialCwd` passed to `DetachedProcessInfo` is set before the terminal link manager resolves paths. Consider consolidating with https://github.com/microsoft/vscode/issues/318548 which reports the same surface from the extension link provider angle.
