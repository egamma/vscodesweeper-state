# Issue #95828 — vscode.workspace.fs.readDirectory() does not get all files (reparse points) under %USERPROFILE%\AppData\Local\Microsoft\WindowsApps

[https://github.com/microsoft/vscode/issues/95828](https://github.com/microsoft/vscode/issues/95828)

## Summary

Team-authored upstream issue tracking a Node.js limitation with AppExecLink reparse points on Windows. The canonical Node.js issue (nodejs/node#36790) remains open, so no fix has landed upstream yet. Must keep open per team-author policy.

## Change summary

vscode.workspace.fs.readDirectory() fails to enumerate AppExecLink reparse point files (e.g. Windows Store app executables) because Node.js fs.stat/lstat throws UNKNOWN errors on them.

## Best solution

This is blocked on the upstream Node.js fix at https://github.com/nodejs/node/issues/36790. Keep tracking until Node.js resolves AppExecLink reparse point handling in its fs APIs, then verify the fix flows through Electron into VS Code.
