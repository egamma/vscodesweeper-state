# Issue #179086 — GTK+ module libcanberra-gtk-module.so cannot be loaded

[https://github.com/microsoft/vscode/issues/179086](https://github.com/microsoft/vscode/issues/179086)

## Summary

Mid-verification (`verification-found`): the initial fix in commit 42457bdb9f4 (PR #185834) addressed some snap env leaking but did not fully resolve GTK_PATH/GIO_MODULE_DIR leaking into the integrated terminal. A team member (deepak1556) confirmed in comment #17 that this remains non-trivial. The issue is still open with `help wanted` and `confirmed`.

## Change summary

Snap-installed VS Code leaks its own GTK_PATH, GTK_EXE_PREFIX, and GIO_MODULE_DIR environment variables into the integrated terminal, causing GTK-based applications launched from the terminal to fail to load modules.

## Best solution

The `electron-launch` script (resources/linux/snap/electron-launch) saves originals into `*_VSCODE_SNAP_ORIG` vars and the environment service (src/vs/platform/environment/electron-main/environmentMainService.ts) restores them for child processes, but this restoration does not fully cover the terminal's inherited environment. The fix likely needs the pty host starter (src/vs/platform/terminal/node/electron-main/electronPtyHostStarter.ts) or the terminal environment resolution to more completely strip or restore all snap-modified GTK/GIO variables before spawning terminal shells.
