# Issue #181050 — editor.selectionClipboard option has no effect on vscode.dev

[https://github.com/microsoft/vscode/issues/181050](https://github.com/microsoft/vscode/issues/181050)

## Summary

Still reproduces on main: the paste-prevention logic (`LinuxSelectionClipboardPastePreventer`) is only registered in `electron-browser/`, so it never runs on vscode.dev. The maintainer already diagnosed the fix path (move to an `IEditorFeature` or `browser/` layer). Milestone is 'On Deck' — keeping open.

## Change summary

Setting `editor.selectionClipboard` to false has no effect on vscode.dev because the paste-prevention code only runs in the Electron layer.

## Best solution

Move `LinuxSelectionClipboardPastePreventer` (or its relevant logic) from `electron-browser/selectionClipboard.ts` to the `browser/` layer so it also activates on the web, guarded by platform detection. alexdima suggested converting it to an `IEditorFeature`.
