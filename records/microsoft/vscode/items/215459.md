# Issue #215459 — Feature: add View: Balance Panes function

[https://github.com/microsoft/vscode/issues/215459](https://github.com/microsoft/vscode/issues/215459)

## Summary

Valid feature request with 22 upvotes, promoted to Backlog. The existing 'Reset Editor Group Sizes' command only equalizes editor groups, not the panel/editor/sidebar proportions the user describes. The requested capability does not exist on main.

## Change summary

Add a command to balance (equalize) the sizes of the main workbench parts — editor area, panel, sidebar — not just editor groups.

## Best solution

Implement a 'View: Balance Panes' (or 'View: Reset Part Sizes') command in `src/vs/workbench/browser/actions/layoutActions.ts` that calls `distributeViewSizes()` on the workbench grid (`workbenchGrid` in `layout.ts`), equalizing editor area, panel, and optionally sidebar proportions. The toggle behavior (remembering previous layout) would need state tracking in the layout service.
