# Issue #215459 — Feature: add View: Balance Panes function

[https://github.com/microsoft/vscode/issues/215459](https://github.com/microsoft/vscode/issues/215459)

## Summary

Valid feature request with 22 upvotes (promoted to Backlog). Existing commands only balance editor groups within the editor area — no command exists to balance workbench parts (editor vs. panel/terminal height). The gap is real.

## Change summary

Add a command to toggle between the current workbench part sizes and an evenly-distributed layout (especially editor vs. panel/terminal).

## Best solution

Implement a 'View: Balance Panes' command in `src/vs/workbench/browser/actions/layoutActions.ts` that calls `distributeViewSizes()` on the workbench grid (or toggles between current and even sizes). The underlying grid infrastructure (`Grid.distributeViewSizes()`) already exists in `src/vs/base/browser/ui/grid/grid.ts`; the work is wiring a workbench-level command that applies it to the top-level parts layout, with a toggle to restore the previous sizes.
