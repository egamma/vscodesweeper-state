# Issue #24389 — SCM - Support workspace diff navigation

[https://github.com/microsoft/vscode/issues/24389](https://github.com/microsoft/vscode/issues/24389)

## Summary

Well-supported feature request (113 upvotes, well above the 20-upvote backlog threshold) for workspace-level SCM diff navigation. The core ask — keyboard-driven cycling through changed files — remains unimplemented on main. Several related open issues track narrower slices (#145009, #296607, #24916), but this umbrella issue has accumulated the most community signal and discussion.

## Change summary

Add keyboard-driven navigation to cycle through all SCM changes across files in the workspace, similar to IntelliJ's F7 behavior.

## Best solution

Implement a command (and default keybinding) that advances through diffs across changed files — when the last change in the current file is reached, automatically open the next changed file's diff. The logic likely belongs in `src/vs/workbench/contrib/scm/` coordinating with the compare-editor navigation in `src/vs/workbench/contrib/codeEditor/`. Consider consolidating the narrower tracking issues (#145009, #296607) as duplicates of this one or of #24916, since the community discussion lives here.
