## Summary

The root cause is unclear — likely a Windows registry virtualization or process architecture issue when shell tasks wrap commands through PowerShell/cmd.exe. No team member reproduced it, the reporter found a workaround (`type: process`), and it's been inactive since late 2022. Requesting confirmation on current VS Code.

## Change summary

The `reg import` command reports success but silently fails to import registry keys when run via a VS Code shell task, while working correctly in VS2022 or when using `type: process`.
