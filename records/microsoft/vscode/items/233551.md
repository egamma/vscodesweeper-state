## Summary

Keep open: the report is actionable and already routed to editor wrapping, but I did not find a concrete fixing commit or a current-main reproduction attempt sufficient to close it. Current configuration code supports compound language overrides generally, yet there is no focused evidence that the tab-switching `editor.wordWrap: "bounded"` case is fixed.

## Change summary

`editor.wordWrap: "bounded"` inside a compound language override such as `[markdown][plaintext]` stops behaving like bounded wrapping after switching away from and back to the editor tab.

## Best solution

Add or verify a focused regression test for editor configuration recomputation after model/tab switches when `editor.wordWrap` comes from a compound language override, likely around `TextResourceConfigurationService` and `TextEditor.updateEditorConfiguration`; then fix the editor option refresh path if it still drops the bounded setting.
