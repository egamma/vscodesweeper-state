# Issue #225545 — [html] Relative links don't work with <base> element 

[https://github.com/microsoft/vscode/issues/225545](https://github.com/microsoft/vscode/issues/225545)

## Summary

Bug is confirmed by maintainer aeschli — the base URI resolution strips the last segment incorrectly. The fix likely spans both vscode-html-languageservice (upstream) and possibly the documentContext.ts resolve logic in this repo. Keeping open as it's already triaged to Backlog with a known root cause.

## Change summary

Clicking relative links in HTML files that use a `<base href>` element resolves to the wrong path because the URI resolver strips the last path segment of the base.

## Best solution

Fix the base href resolution in `vscode-html-languageservice` (where the base is constructed before being passed to `resolveReference`), or adjust `extensions/html-language-features/server/src/utils/documentContext.ts` line 38 to preserve trailing-slash semantics when the base comes from a `<base>` element.
