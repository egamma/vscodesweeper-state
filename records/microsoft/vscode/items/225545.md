# Issue #225545 — [html] Relative links don't work with <base> element 

[https://github.com/microsoft/vscode/issues/225545](https://github.com/microsoft/vscode/issues/225545)

## Summary

Still reproduces on current main. The bug is in `resolveReference` in documentContext.ts: `Utils.resolvePath` strips the trailing slash from `images/`, so subsequent link resolution treats it as a file rather than a directory, resolving `skibidi.jpg` against the parent folder instead of `images/`.

## Change summary

Ctrl+clicking a relative link in HTML ignores the `<base href>` directory path because the trailing slash is stripped during resolution.

## Best solution

Fix `resolveReference` in `extensions/html-language-features/server/src/utils/documentContext.ts` to preserve the trailing slash when the original ref ended with `/`. Alternatively, fix this upstream in `vscode-html-languageservice`'s `htmlLinks.ts` where it stores the resolved base.
