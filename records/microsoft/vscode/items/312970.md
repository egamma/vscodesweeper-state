# Issue #312970 — terminal progress display overflow

[https://github.com/microsoft/vscode/issues/312970](https://github.com/microsoft/vscode/issues/312970)

## Summary

Still reproduces on main: the `.single-terminal-tab` CSS in terminal.css has no overflow/text-overflow constraints, so long progress or description text overflows a narrow panel. Multiple open PRs are attempting to fix this.

## Change summary

Terminal progress/description text overflows the panel header when the terminal is in a narrow side pane with a single tab displayed.

## Best solution

Land one of the open fix PRs (e.g. https://github.com/microsoft/vscode/pull/322041 or https://github.com/microsoft/vscode/pull/316228) that adds `overflow: hidden` and `text-overflow: ellipsis` to the single-tab label container in `src/vs/workbench/contrib/terminal/browser/media/terminal.css`.
