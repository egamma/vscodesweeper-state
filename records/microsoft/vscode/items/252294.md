# Issue #252294 — Handle 'removed' BreakpointEvent for instruction breakpoints

[https://github.com/microsoft/vscode/issues/252294](https://github.com/microsoft/vscode/issues/252294)

## Summary

The gap is confirmed on current main: the `onDidBreakpoint` handler in `debugSession.ts` queries source, function, data, and exception breakpoints but never looks up or acts on instruction breakpoints for 'removed' or 'changed' events. The feature request remains valid and unimplemented.

## Change summary

Handle the DAP BreakpointEvent for instruction breakpoints — specifically 'removed' and 'changed' reasons — so that adapter-initiated instruction breakpoint changes are reflected in VS Code's UI.

## Best solution

In `src/vs/workbench/contrib/debug/browser/debugSession.ts` around line 1248, add a lookup via `this.model.getInstructionBreakpoints()` by adapter ID, then handle 'removed' (call `removeInstructionBreakpoints`) and 'changed' (update session data) for the matched instruction breakpoint, mirroring the existing patterns for data/function breakpoints.
