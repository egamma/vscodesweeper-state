# Issue #203580 — No merge conflict codelenses in vscode.dev

[https://github.com/microsoft/vscode/issues/203580](https://github.com/microsoft/vscode/issues/203580)

## Summary

Team-authored issue (roblourens/MEMBER) — must keep open. The merge conflict codelens provider registers only for schemes `file`, `vscode-vfs`, `untitled`, and `vscode-userdata`; if vscode.dev opens files under a different scheme (e.g. `vscode-github`), the codelenses would not appear. No fix has landed on main since the report.

## Change summary

Merge conflict codelenses (Accept Current/Incoming/Both, Compare) do not appear on vscode.dev when editing a file with merge conflict markers.

## Best solution

Investigate which URI scheme vscode.dev uses for the affected files and add it to the `registerCodeLensProvider` document selector in `extensions/merge-conflict/src/codelensProvider.ts` (line 101–106). Alternatively, register the codelens provider with a wildcard `*` scheme or use `{ pattern: '**' }` to cover all schemes, mirroring how the decorator (`mergeDecorator.ts`) already applies to all visible editors regardless of scheme.
