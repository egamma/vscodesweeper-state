# Issue #325434 — Title: Allow users to hide unavailable “Upgrade” models from the Chat model picker

[https://github.com/microsoft/vscode/issues/325434](https://github.com/microsoft/vscode/issues/325434)

## Summary

Valid feature request in Backlog Candidates with 0 upvotes and 53 days remaining in the community process. The gap is real: the Language Models editor exposes hide/show toggles only for models the user already has access to, while unavailable 'Upgrade' entries from the control manifest are unconditionally rendered in the model picker. Keeping open for the community upvote process.

## Change summary

Allow users to hide unavailable 'Upgrade' models from the Chat model picker, the same way available models can already be hidden via the Language Models editor.

## Best solution

Extend the hidden-model infrastructure (`isModelHidden`/`setModelHidden` in `ILanguageModelsService`) to cover control-manifest entries that have no local registration (i.e. unavailable models). In `buildModelPickerItems` (src/vs/workbench/contrib/chat/browser/widget/input/chatModelPicker.ts), filter out unavailable `controlModels` entries whose id is in the hidden set, mirroring how `recentModelIds` and `pinnedModelIds` are already filtered at lines 1376-1377. The Language Models editor (chatModelsWidget.ts / chatModelsViewModel.ts) would need to surface these unavailable entries with a visibility toggle as well.
