# Issue #276982 — Give context into what helper is being installed or updated

[https://github.com/microsoft/vscode/issues/276982](https://github.com/microsoft/vscode/issues/276982)

## Summary

The 'helper tool' password prompt is a macOS system dialog triggered by Electron's Squirrel auto-updater when VS Code is installed in a non-user-writable location. The dialog text is controlled by macOS, not VS Code. The root cause is upstream (Electron/Squirrel), though VS Code could potentially improve the experience by providing context before the system dialog appears. Two team members have already engaged with this issue.

## Change summary

On macOS, VS Code's auto-updater triggers a vague system dialog asking for a password to 'add a new helper tool' with no indication of what is being installed or why.

## Best solution

The ideal end state is either (1) Electron/Squirrel improves the helper tool dialog to include the app name and purpose, or (2) VS Code intercepts the update flow to show its own explanatory notification before the macOS system prompt appears. Since team members are already engaged (deepak1556 was pinged, dmitrivMS acknowledged), this should stay open for a maintainer decision on whether to pursue an Electron upstream fix or a VS Code-side workaround.

## Evidence

- **upstream root cause:** The 'Code is trying to add a new helper tool' dialog is a macOS Authorization Services prompt triggered by Electron's Squirrel updater (ShipIt helper) when the app location requires elevated privileges. The string is controlled by macOS, not VS Code.
- **Electron issue:** https://github.com/electron/electron/issues/51387 — 'Unexpected helper tool install prompt on macOS' (closed). Electron team confirmed this is expected behavior when the app bundle is in a non-user-writable location.
- **community confirmation:** Comment by grahamr confirms the issue across enterprise environments, citing compliance concerns and noting it has been reported for 5+ years.
- **team engagement:** Team member joshspicer pinged deepak1556 about whether this is part of the updater. dmitrivMS acknowledged the UX concern with a suggestion from the duplicate about providing context.
- **related closed issues:** https://github.com/microsoft/vscode/issues/95534 (closed as upstream/fixed-in-electron-13) and https://github.com/microsoft/vscode/issues/115805 (closed as info-needed) describe similar problems.
- **VS Code update code:** src/vs/platform/update/electron-main/updateService.darwin.ts delegates to Electron's autoUpdater, which triggers the macOS system prompt. VS Code's own osascript usage (in nativeHostMainService.ts) is for the 'Install code command in PATH' feature, not the updater.
