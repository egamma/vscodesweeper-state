# Issue #169017 — `search.maxResults` does not affect how many results are found/highlighted

[https://github.com/microsoft/vscode/issues/169017](https://github.com/microsoft/vscode/issues/169017)

## Summary

Team-authored, confirmed bug still present on main: the editor find widget uses a hardcoded MATCHES_LIMIT = 19999 in findModel.ts and ignores the `search.maxResults` setting entirely. Keeping open per team-author rule and because the defect remains.

## Change summary

The `search.maxResults` setting has no effect on the editor's Ctrl+F find widget match limit, which is hardcoded to 19999.

## Best solution

Either wire `search.maxResults` into the editor find widget (replacing the hardcoded `MATCHES_LIMIT` in `src/vs/editor/contrib/find/browser/findModel.ts`), or introduce a separate `editor.find.maxResults` setting — the choice depends on whether the team wants a unified setting or separate controls for workspace search vs. in-file find.
