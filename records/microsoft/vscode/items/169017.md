# Issue #169017 — `search.maxResults` does not affect how many results are found/highlighted

[https://github.com/microsoft/vscode/issues/169017](https://github.com/microsoft/vscode/issues/169017)

## Summary

Still valid on main: the editor find widget uses a hardcoded MATCHES_LIMIT of 19999 in findModel.ts, completely ignoring the `search.maxResults` setting. Team-authored issue — keeping open for maintainer judgment.

## Change summary

The `search.maxResults` setting has no effect on the editor's Ctrl+F find widget; its match limit is hardcoded at 19999.

## Best solution

Wire the `search.maxResults` configuration into the editor find widget's match-counting logic (src/vs/editor/contrib/find/browser/findModel.ts, where `MATCHES_LIMIT = 19999` is hardcoded), so users can raise or lower the in-editor find limit. Alternatively, introduce a separate `editor.find.maxResults` setting if decoupling from the sidebar search is preferred.
