# Issue #296642 — Different cloud and background options when delegating from local vs new empty chat

[https://github.com/microsoft/vscode/issues/296642](https://github.com/microsoft/vscode/issues/296642)

## Summary

Team-authored issue (MEMBER) tracking UX inconsistency between delegation and new-session flows for agent session targets — active under-discussion with multiple team members, must stay open for maintainer judgment.

## Change summary

When delegating from an existing local chat to a cloud/background agent, the available options (model picker, Configure Tools button, agent provider dropdown) differ from what appears in a new empty chat for the same session target.

## Best solution

Unify the input-box options so that delegation into a session target offers the same controls (model picker, Configure Tools, provider selection) as creating a new session of that target type directly. The work likely spans the chat input rendering and delegation flow in `src/vs/workbench/contrib/chat/` and the agent-session contribution points. The team discussion (comments 4–6) is still refining scope — Tyler noted Claude doesn't support delegation yet, and the Configure Tools question for Copilot CLI remains open.
