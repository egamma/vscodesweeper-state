# Issue #307696 — User confirmation feedback audio is sometimes not heard in terminal

[https://github.com/microsoft/vscode/issues/307696](https://github.com/microsoft/vscode/issues/307696)

## Summary

Team-authored accessibility bug — keeping open. The issue is likely rooted in Chromium's autoplay policy (audio play() requires a recent user gesture), which the codebase already silently catches. The upstream constraint makes a pure code fix non-trivial, but the impact on screen reader users is real.

## Change summary

Audio feedback for terminal confirmation prompts is sometimes silently dropped, making prompts invisible to screen reader users.

## Best solution

Investigate whether the accessibility signal service can work around the Chromium autoplay restriction for confirmation prompts — e.g. by pre-warming the AudioContext on a prior user gesture, or by falling back to an ARIA alert announcement when audio playback fails. The silent catch at `accessibilitySignalService.ts:211` already tracks this as https://github.com/microsoft/vscode/issues/178642. A fix likely touches `src/vs/platform/accessibilitySignal/browser/accessibilitySignalService.ts` and possibly terminal-side code to ensure the announcement fallback fires reliably.
