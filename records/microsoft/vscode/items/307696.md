# Issue #307696 — User confirmation feedback audio is sometimes not heard in terminal

## Summary

Team-authored accessibility bug about intermittent audio feedback loss for terminal confirmation prompts — kept open per policy (MEMBER author). Root cause likely Chromium autoplay restrictions; remains an open problem.

## Change summary

Audio feedback for terminal user-confirmation prompts is sometimes not played, leaving screen reader users unaware a prompt appeared.

## Best solution

Investigate workarounds for Chromium's autoplay-without-interaction restriction — e.g. ensuring a recent user gesture is always in scope before playing the signal, or falling back to an ARIA live-region alert that doesn't rely on media playback. The fix likely touches the accessibility signal service's play path and/or the terminal's confirmation-detection layer.
