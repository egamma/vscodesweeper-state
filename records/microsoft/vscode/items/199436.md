# Issue #199436 — Git - Handle rebase when pulling & syncing

[https://github.com/microsoft/vscode/issues/199436](https://github.com/microsoft/vscode/issues/199436)

## Summary

The core request — better UX safeguards when pulling with rebase and uncommitted changes — remains valid. The existing `git.autoStash` setting plus commit a2a0272687a (which passes `--autostash` to native git) provides a workaround, but the issue asks for smarter out-of-the-box behavior rather than requiring users to discover that setting. Keeping open as under-discussion.

## Change summary

Provide better safeguards (auto-stash or guided prompts) when pulling or syncing with rebase while the index contains uncommitted changes.

## Best solution

The simplest improvement would be to detect the "cannot pull with rebase: uncommitted changes" error in `extensions/git/src/repository.ts` (around the `_pullAndHandleTagConflict` error handling) and offer the user a prompt to stash changes and retry, or to surface the `git.autoStash` setting. A more ambitious approach is the proposed `git.postCommitCommandWhenUncommittedChanges` setting. Either way this is a UX/polish decision for the git area owner.
