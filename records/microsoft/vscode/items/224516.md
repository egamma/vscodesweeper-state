# Issue #224516 — LogOutputChannel does not clear all characters of Log Message

[https://github.com/microsoft/vscode/issues/224516](https://github.com/microsoft/vscode/issues/224516)

## Summary

Still reproduces on current main: the ext host tracks raw message byte length as the offset for clear(), but the logger writes formatted output (with timestamps/level prefixes) to the file, so the byte offset is too small and leftover characters remain after clear.

## Change summary

LogOutputChannel.clear() leaves residual characters from the previous log message because the byte offset sent to the renderer doesn't account for log formatting added by the logger.

## Best solution

Fix the offset tracking in `src/vs/workbench/api/common/extHostOutput.ts` so it accounts for the formatted output byte length (including timestamp and level prefix), or have the renderer-side clear ignore the `till` parameter for log channels and always reset to the current file end position.
