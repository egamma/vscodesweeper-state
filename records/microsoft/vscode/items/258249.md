# Issue #258249 — sticky apply button

[https://github.com/microsoft/vscode/issues/258249](https://github.com/microsoft/vscode/issues/258249)

## Summary

Team-authored polish request — the code block toolbar (including the Apply button) uses `position: absolute` at the top of the block and does not follow the user when scrolling a long code block. The feature remains unimplemented on current main.

## Change summary

Make the chat code-block action buttons (Apply, Copy, etc.) stick to the top of the visible area so they remain accessible when scrolling through a long code block.

## Best solution

Change the code-block toolbar positioning from `position: absolute` to `position: sticky` (or an equivalent scroll-tracking mechanism) in `src/vs/workbench/contrib/chat/browser/widget/chatContentParts/media/codeBlockPart.css` and potentially its TypeScript counterpart, so the toolbar stays visible as users scroll through long code responses. The chat/editor area team owns this.
