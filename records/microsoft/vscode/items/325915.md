# Issue #325915 — Feature: first-class Grok Build CLI agent in Copilot Chat (SuperGrok `grok login`, not API-key-only)

[https://github.com/microsoft/vscode/issues/325915](https://github.com/microsoft/vscode/issues/325915)

## Summary

This asks VS Code core to build and maintain first-party integration for a specific third-party AI agent (xAI Grok Build). VS Code's architecture delegates vendor-specific agent integrations to extensions via ACP/extension API — the request itself notes a community extension already provides this capability. This belongs in an extension, not core.

## Change summary

Add native Grok Build CLI agent integration into Copilot Chat with SuperGrok subscription auth, session management, and terminal-chat sync.

## Best solution

Ship this as an extension (or improve the existing community extension at phuryn/grok-build-vscode). VS Code's ACP framework already enables external agent providers to integrate into the agent UX — the right path is for xAI or the community to build a first-class extension, not for VS Code core to hardcode support for a specific vendor's CLI. If the existing extension API or ACP surface is missing capabilities needed for parity, those gaps would be valid core feature requests.

## Vision fit

- **Fit:** rejected
- **Reason:** VS Code's values explicitly state 'the power lives in extensions' and the product keeps core lightweight; hardcoding integration for a specific third-party vendor's agent CLI conflicts with this. The ACP framework exists precisely so external agents can integrate via extensions.
- **Evidence:** Value: 'Out of the box, VS Code is a text editor: lightweight and fast. Users may install extensions that impact the appearance and nature of VS Code.' Value: 'The power lives in extensions. Although the UI choices extension authors can make are limited, authors can write powerful extensions.'
