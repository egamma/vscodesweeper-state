# Issue #311324 — Terminal title changes in unexpected ways

[https://github.com/microsoft/vscode/issues/311324](https://github.com/microsoft/vscode/issues/311324)

## Summary

Still reproduces on main: when `allowAgentCliTitle` is false, the code falls back to `tabs.title` (default `${sequence}`), so agent CLI escape sequences still leak into the title — the setting doesn't achieve its stated purpose.

## Change summary

Setting `terminal.integrated.tabs.allowAgentCliTitle` to false does not prevent agent CLIs from changing the terminal tab title because the default title template `${sequence}` still renders their escape-sequence output.

## Best solution

When `allowAgentCliTitle` is false and the shell type is an agent CLI, the title template fallback in `refreshLabel` should ignore `${sequence}` — either substitute `${process}` for agent CLI shell types in that path, or filter sequence values when the setting is disabled. Fix is localized to `TerminalLabelComputer.refreshLabel` in `src/vs/workbench/contrib/terminal/browser/terminalInstance.ts`.
