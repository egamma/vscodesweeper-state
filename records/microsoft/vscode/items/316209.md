# Issue #316209 — No port forwarding for "code serve-web" when accessed from another machine

[https://github.com/microsoft/vscode/issues/316209](https://github.com/microsoft/vscode/issues/316209)

## Summary

Legitimate feature gap: `code serve-web` doesn't configure a tunnelProvider in the web workbench, so `vscode.env.asExternalUri()` cannot map localhost URIs to the remote host when accessed from another machine. No fix or prior tracking found on main.

## Change summary

Extensions calling `vscode.env.asExternalUri()` with localhost URIs get no port forwarding or URL rewriting in the `code serve-web` scenario when accessed from another machine.

## Best solution

Implement a tunnelProvider for the `serve-web` workbench that rewrites localhost URIs to the server's host/port, similar to how Remote-SSH or Codespaces provide theirs via `IWorkbenchConstructionOptions.tunnelProvider`. The work likely belongs in `src/vs/code/browser/workbench/workbench.ts` or a new dedicated tunnel factory for serve-web.
