# Issue #316209 — No port forwarding for "code serve-web" when accessed from another machine

[https://github.com/microsoft/vscode/issues/316209](https://github.com/microsoft/vscode/issues/316209)

## Summary

This is a valid feature request for the serve-web remote/web environment: `asExternalUri()` does not currently provide localhost port forwarding or URI rewriting for browser clients connecting from another machine. It should stay open for the remote/web owners to decide whether this belongs in core and whether it should be gated behind a CLI/server option.

## Change summary

`code serve-web --host 0.0.0.0` should optionally forward or rewrite localhost URIs returned by extension calls to `vscode.env.asExternalUri()` so webviews work from a different client machine.

## Best solution

Decide in the remote/web port forwarding area whether `code serve-web` should expose a tunnel/forwarding endpoint for `localhost` targets used by extensions, likely alongside the existing `asExternalUri` remote authority handling rather than treating this as an extension bug.
