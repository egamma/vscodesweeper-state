# Issue #64311 — Search: find, replace input boxes miss scrollbar

[https://github.com/microsoft/vscode/issues/64311](https://github.com/microsoft/vscode/issues/64311)

## Summary

The issue is on the 'On Deck' milestone, which is a scheduling decision — keeping open per policy. The underlying request is still valid: the search view CSS explicitly hides scrollbars on the multi-line input boxes, so tall or wide content has no visible scroll affordance.

## Change summary

Multi-line search find/replace input boxes lack visible scrollbars when content overflows vertically or horizontally.

## Best solution

The InputBox already wraps its textarea in a ScrollableElement (src/vs/base/browser/ui/inputbox/inputBox.ts:161), but the search view CSS (src/vs/workbench/contrib/search/browser/media/searchview.css:71–82) forces scrollbar opacity to 0 and hides native scrollbars. The fix is to remove or relax these CSS overrides so the ScrollableElement's scrollbar is visible when the input content overflows the max height.
