# Issue #64311 — Search: find, replace input boxes miss scrollbar

[https://github.com/microsoft/vscode/issues/64311](https://github.com/microsoft/vscode/issues/64311)

## Summary

Still valid: both the editor find widget and the search view explicitly hide scrollbars on multi-line input boxes via CSS (`opacity: 0` / `display: none`). No fix has landed on main, and no canonical issue absorbs this work.

## Change summary

Add scrollbars to the multi-line find/replace input boxes in both the editor find widget and the search view so tall or wide content can be scrolled visually.

## Best solution

Remove or conditionally show the scrollbar-hiding CSS rules in `src/vs/editor/contrib/find/browser/findWidget.css` (line 110–113) and `src/vs/workbench/contrib/search/browser/media/searchview.css` (lines 71–82) so the existing `ScrollableElement` vertical scrollbar becomes visible when the input content overflows. A design pass is needed to decide when scrollbars appear (hover? always?) and whether horizontal scrolling is also needed.
