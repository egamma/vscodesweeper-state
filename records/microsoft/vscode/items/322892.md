# Issue #322892 — 🚀 Feature Request: Include response duration in Copilot Chat statistics

[https://github.com/microsoft/vscode/issues/322892](https://github.com/microsoft/vscode/issues/322892)

## Summary

Valid feature request to display response generation duration in the chat stats line. The model layer already tracks `elapsedMs` on each response (confirmation-wait-adjusted); this is purely a UI surfacing gap. Routing to the chat area for triage.

## Change summary

Show the total response generation duration (e.g. "3.4s") in the per-turn statistics line alongside model name and credits.

## Best solution

Surface `ChatResponseModel.elapsedMs` in the turn footer stats line (the area that currently shows model name and credit cost). The data is already computed and serialized — a small UI addition in the chat list renderer or turn pills would deliver this. The feature could also be gated behind a setting if the team prefers an opt-in.
