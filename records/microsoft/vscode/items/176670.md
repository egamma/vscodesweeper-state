# Issue #176670 — Error message complaining about build task

[https://github.com/microsoft/vscode/issues/176670](https://github.com/microsoft/vscode/issues/176670)

## Summary

Still reproduces on main: the error message fires because `customTask.command.name` is undefined for tasks created via extension API with ShellExecution — the fix in commit afe7f56bc4d was never merged. The issue is correctly labeled confirmed/bug/tasks.

## Change summary

Extensions creating tasks programmatically via the Task API with ShellExecution get a spurious error in the Tasks Output: "neither specifies a command nor a dependsOn property".

## Best solution

Merge the approach from the abandoned fix (afe7f56bc4d / PR #200074): in `CustomTask.fillGlobals` in `src/vs/workbench/contrib/tasks/common/taskConfiguration.ts`, set `task.command.name` from `task.configurationProperties.name` so that extension-created tasks with a ShellExecution pass the command-presence check at line 1785. Alternatively, the check itself could be made aware of tasks with an execution object already set.
