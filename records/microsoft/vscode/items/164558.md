# Issue #164558 — Fresh profile leaks auth info from original profile

[https://github.com/microsoft/vscode/issues/164558](https://github.com/microsoft/vscode/issues/164558)

## Summary

The bug likely still exists: auth usage/access data is stored at StorageScope.APPLICATION (shared across profiles), and no auth service listens to onDidChangeCurrentProfile. However, the exact trigger path from 2022 may have shifted due to significant auth refactoring, so reproduction status is unclear without testing.

## Change summary

A fresh VS Code profile shows authentication sign-in prompts for extensions that are only installed in the original profile.

## Best solution

The auth services (authenticationExtensionsService, authenticationUsageService, authenticationAccessService) should either scope their data per-profile or filter sign-in requests/access prompts to only extensions installed in the current profile. The fix likely belongs in src/vs/workbench/services/authentication/browser/authenticationExtensionsService.ts — clear _signInRequestItems and _sessionAccessRequestItems on profile change, or gate requestNewSession/requestSessionAccess against installed extensions.
