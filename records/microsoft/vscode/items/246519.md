# Issue #246519 — Remote SSH - Github extensions sign-in fails - [error] HttpError: getaddrinfo EBUSY api.github.com

[https://github.com/microsoft/vscode/issues/246519](https://github.com/microsoft/vscode/issues/246519)

## Summary

The `getaddrinfo EBUSY` error is a known Node.js DNS resolution issue (linked by a team member), surfacing on a very old Ubuntu 16.04 remote host. This is an upstream Node.js/Electron concern rather than a VS Code bug. Routing to the relevant area labels for visibility.

## Change summary

GitHub extension sign-in fails over Remote SSH on Ubuntu 16.04 with `getaddrinfo EBUSY api.github.com`, despite DNS working from the terminal.

## Best solution

Track as upstream: the root cause is in Node.js's DNS resolver (see https://github.com/nodejs/help/issues/2390). The regression between 1.97 and 1.99 likely corresponds to an Electron/Node.js version bump. A workaround may be possible via `--dns-result-order=ipv4first` or similar Node.js DNS options, but the fix belongs in Node.js. Consider whether the bundled Node.js version on the remote server can be configured differently, and whether newer Electron versions have resolved this.
