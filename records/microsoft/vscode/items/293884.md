# Issue #293884 — Configure default directory/pattern for git worktrees

[https://github.com/microsoft/vscode/issues/293884](https://github.com/microsoft/vscode/issues/293884)

## Summary

Valid feature request for a configurable worktree path template in the built-in git extension. The current code hardcodes a default pattern or remembers the last-used location, but offers no user setting for a template. No existing issue fully owns this work.

## Change summary

Allow users to configure a default directory pattern/template for where git worktrees are created, rather than relying on the hardcoded default or last-used location.

## Best solution

Add a `git.worktreeDefaultPath` setting (with template variable support like `${repo}`, `${branch}`) in `extensions/git/package.json` and consume it in `getWorktreePath()` in `extensions/git/src/commands.ts` to provide a user-configurable default worktree location.
