# Issue #192689 — {sequence} in terminal title doesn't result in same title as in other terminals

[https://github.com/microsoft/vscode/issues/192689](https://github.com/microsoft/vscode/issues/192689)

## Summary

Already fixed and shipped in 1.83.0. The bug was that on Windows, when the escape-sequence title didn't match a Windows absolute path pattern, `_sequence` was incorrectly set to `undefined`, causing `${sequence}` to be empty.

## Change summary

Setting `terminal.integrated.tabs.title` to `${sequence}` showed only the shell name instead of the dynamic title set by tools like oh-my-posh via escape sequences.

## Best solution

Keep the shipped fix in commit 257869baca2 (1.83.0); no further change needed. The `_sequence` property now correctly preserves the title from escape sequences on all platforms.
