# Issue #324556 — Navigation shortcut conflicts between the sessions list and the chat tabs list

[https://github.com/microsoft/vscode/issues/324556](https://github.com/microsoft/vscode/issues/324556)

## Summary

Team-filed bug confirmed in source: the chat-tab navigation keybinding (weight SessionsContrib+10) shadows the session-list navigation keybinding (weight SessionsContrib) when SessionHasMultipleOpenChatsContext is true, trapping the user in one session. Keeping open per team-author rule.

## Change summary

Cmd+Shift+[/] for navigating sessions is captured by the chat-tabs navigation action when the current session has multiple chats open, making it impossible to switch sessions with those shortcuts.

## Best solution

Disambiguate the two navigation levels — either use different keybindings for chat-tab vs session navigation, or add a focus-based context key (e.g. sessions list focused vs chat area focused) so the same shortcut routes to the correct action depending on focus. The fix lives in `src/vs/sessions/contrib/sessions/browser/sessionsActions.ts` (chat tab keybindings) and `src/vs/sessions/contrib/sessions/browser/views/sessionsViewActions.ts` (session list keybindings).
