# Issue #125306 — iPad: improve browsing experience when no real keyboard is available

[https://github.com/microsoft/vscode/issues/125306](https://github.com/microsoft/vscode/issues/125306)

## Summary

Valid feature request for touch-friendly navigation in the main workbench on iPad. Some mobile/touch gesture work exists in the new Agents/Sessions window layer, but the main workbench has no equivalent touch gesture system for opening the command palette or replacing the menubar. Keeping open — low traction but still a real gap for iPad users without keyboards.

## Change summary

Add touch-friendly gestures (e.g. 3-finger swipe) and a larger touch-based overlay to access frequent actions like the Command Palette on iPad when no keyboard is attached.

## Best solution

A touch gesture service in the main workbench (possibly adapting patterns from `src/vs/sessions/browser/parts/mobile/mobileEdgeSwipe.ts`) that maps configurable gestures to commands, starting with opening the quick pick. Alternatively, a persistent touch-accessible button in the activity bar or title bar for the command palette would be a simpler first step. Low community demand suggests this can remain in the backlog until iPad/mobile usage grows.
