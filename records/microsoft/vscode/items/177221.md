# Issue #177221 — Differentiate breakpoint types more clearly

[https://github.com/microsoft/vscode/issues/177221](https://github.com/microsoft/vscode/issues/177221)

## Summary

Valid polish/accessibility request to make breakpoint type icons more visually distinguishable. No changes have been made to the breakpoint icon shapes on current main; the existing codicon shapes remain. Low upvote count but touches accessibility, so keeping open.

## Change summary

Conditional breakpoints (and data breakpoints) use shapes too similar to regular breakpoints, making them hard to distinguish visually.

## Best solution

Redesign the conditional and/or data breakpoint codicons to use more distinct shapes (e.g. a sector or semicircle for conditional, a more angular shape for data). This is a codicons + debugIcons.ts change. Related issue https://github.com/microsoft/vscode/issues/129543 covers instruction breakpoint icons specifically — both could be addressed in a broader breakpoint icon refresh.
