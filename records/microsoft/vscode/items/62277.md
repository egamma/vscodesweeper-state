# Issue #62277 — Surface the 'origin and 'presentationHint' attributes of a Source in the Loaded Scripts Explorer

[https://github.com/microsoft/vscode/issues/62277](https://github.com/microsoft/vscode/issues/62277)

## Summary

Still a valid gap on current main: the Loaded Scripts view renderer in `loadedScriptsView.ts` does not consume `Source.presentationHint` or `Source.origin`, even though both properties are available on the `Source` model class. The request is small and well-scoped but has zero upvotes and no discussion, so it sits quietly in Backlog.

## Change summary

Surface the DAP Source `origin` and `presentationHint` attributes in the Loaded Scripts Explorer, matching how the Call Stack view already uses them.

## Best solution

In `src/vs/workbench/contrib/debug/browser/loadedScriptsView.ts`, extend the `render` method of `LoadedScriptsRenderer` to (1) apply CSS classes based on `presentationHint` (mirroring `callStackView.ts` line 752–753) and (2) include `origin` in the label description or hover tooltip. The `Source` class already exposes both via `.presentationHint` and `.origin`.
