# Issue #283480 — Suggesting switching to GPT-4.1 on high demand error

[https://github.com/microsoft/vscode/issues/283480](https://github.com/microsoft/vscode/issues/283480)

## Summary

The VS Code client-side rate-limit messages on current main already suggest 'switch to Auto' rather than naming a specific model. However, the GPT-4.1 suggestion comes from the server (`capiError.message`) as confirmed by a team member, making this an upstream issue that needs server-side coordination. Keeping open since the upstream fix is still needed.

## Change summary

When a model hits high demand, the error message suggests switching to the outdated GPT-4.1 instead of a newer or evergreen model recommendation.

## Best solution

Coordinate with the Copilot server team to make the suggested-model string in the CAPI error response either evergreen (e.g. 'switch to another model' or 'switch to Auto') or dynamically point to a current model. The VS Code client already handles its own messages well; the gap is the server-provided `capiError.message` passed through at lines 160–164 of `chatErrorMessages.ts`.
