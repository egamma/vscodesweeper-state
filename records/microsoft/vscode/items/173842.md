# Issue #173842 — Task with empty command does not terminate

[https://github.com/microsoft/vscode/issues/173842](https://github.com/microsoft/vscode/issues/173842)

## Summary

Confirmed bug with a clear root cause in `_isTaskEmpty`: an empty-string command passes the `name !== undefined` check, so the task enters normal terminal execution but never terminates. An open PR (#251115) already targets the fix.

## Change summary

A task with an empty string command (`"command": ""`) is not detected as empty and spawns a terminal that never terminates.

## Best solution

Land or refine PR https://github.com/microsoft/vscode/pull/251115, which improves the `_isTaskEmpty` logic in `terminalTaskSystem.ts` to treat empty-string commands as empty tasks so they terminate immediately with exit code 0.
