# Issue #102867 — CodeLens messes up page scrolling via cursorPageUp / cursorPageDown

[https://github.com/microsoft/vscode/issues/102867](https://github.com/microsoft/vscode/issues/102867)

## Summary

Still reproduces on main: pageSize is computed as `floor(height / lineHeight) - 2` in cursorCommon.ts, which doesn't account for CodeLens-injected vertical space. The issue is already labeled feature-request by the team and sits in Backlog — keeping open.

## Change summary

cursorPageUp/cursorPageDown scrolls too far (or not far enough) when CodeLens decorations add extra vertical height between lines.

## Best solution

The page-size calculation in `src/vs/editor/common/cursorCommon.ts` (line 129) should account for variable-height lines caused by CodeLens (and potentially other view zones). This would require querying the view layout for the actual number of fully visible lines rather than dividing viewport height by a uniform lineHeight.
