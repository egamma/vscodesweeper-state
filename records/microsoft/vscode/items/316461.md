## Summary

Valid feature request for GitHub Issues integration in the Agents window. No such capability exists today — the sessions/github contrib only covers Pull Requests. Keeping open for community traction and product judgment; 0 upvotes after 66 days so not yet at the community promotion threshold.

## Change summary

Surface GitHub Issues browsing, `#issue` mentions in prompts, auto-linking, and issue lifecycle management inside the VS Code Agents window.

## Best solution

This is a product-direction decision for the Agents window team. The request aligns with making the agent window a complete task-execution environment. If pursued, it would likely live in `src/vs/sessions/contrib/github/` alongside the existing PR integration. However, much of this (browse/search issues, issue picker) could also be served by the existing GitHub Pull Requests & Issues extension — the team should decide what belongs in core vs. extension.
