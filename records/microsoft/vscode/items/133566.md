# Issue #133566 — Git - Source Control Tab "Discard Changes" on a file called '*' discards all untracked changes

[https://github.com/microsoft/vscode/issues/133566](https://github.com/microsoft/vscode/issues/133566)

## Summary

Still reproduces on main: `git clean` passes filenames as pathspecs without `--literal-pathspecs`, so metacharacters like `*` glob-expand and delete unintended files. Milestone is 'On Deck' (a scheduling milestone), so keeping open. An open PR (#324378) already addresses this with the correct approach.

## Change summary

Discarding changes on a file named `*` in Source Control expands the glob and deletes all untracked files instead of just the target file.

## Best solution

Review and land https://github.com/microsoft/vscode/pull/324378, which passes `--literal-pathspecs` to targeted `git clean` operations — the correct fix after the earlier shell-escaping approach (#236849) was reverted for breaking paths with spaces.
