# Issue #289448 — Chat editor unexpectedly closes when switching git branches with SCM Working Sets enabled

[https://github.com/microsoft/vscode/issues/289448](https://github.com/microsoft/vscode/issues/289448)

## Summary

Team-authored issue; the bug is still present on main. The `shouldShowClearEditingSessionConfirmation` function bypasses confirmation when `willKeepAlive` is true, so SCM working-set restore silently closes chat editors with pending edits.

## Change summary

Chat editor windows with active editing sessions close without confirmation when switching git branches with SCM Working Sets enabled.

## Best solution

Fix the `showConfirm()` path in `chatEditorInput.ts` so that the presence of undecided edits in an editing session is checked independently of `willKeepAlive`. The `willKeepAlive` flag should suppress the confirmation for background keep-alive disposal but not for explicit close-all-editors calls from working-set restore. Likely touch: `src/vs/workbench/contrib/chat/browser/widgetHosts/editor/chatEditorInput.ts` lines 93–94 and 480–483.
