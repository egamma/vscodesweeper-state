## Change summary

Opening the welcome view after a chat session causes multiple misbehaviors: wrong session type, missing folder/model defaults, unnecessary trust prompts, and agent running despite denied trust.

## Best solution

The remaining issue is that dropdowns lose their defaults when the welcome screen is re-opened (particularly the folder picker falling back to root). Comment 8 from DonJayamanne points to https://github.com/microsoft/vscode/issues/290989 as the underlying core issue. The fix should ensure the welcome view properly inherits or restores folder/model/session-type selections when re-opened, and block send until a valid repository is selected. This likely lives in the agents window / chat welcome view contribution code.
