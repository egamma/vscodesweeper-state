# Issue #320900 — Git meesage opens in agent window

[https://github.com/microsoft/vscode/issues/320900](https://github.com/microsoft/vscode/issues/320900)

## Summary

Team-authored issue (dbaeumer). The agents window should never receive git commit-message files opened via `code --wait`. Despite guards at line 1522 and 774–775 in `windowsMainService.ts` that skip the sessions window, there appear to be code paths where the sessions window still wins — likely when it is the only window or when contextWindowId routes to it before the guard fires.

## Change summary

When VS Code is the default git editor and the agents window is open, git commit messages (COMMIT_EDITMSG) incorrectly open in the agents window instead of a regular editor window.

## Best solution

Audit the `--wait` file-opening path in `windowsMainService.ts` to ensure the sessions window is never selected as the target for CLI-opened files. The `openInBrowserWindow` guard at line 1522 already skips it, but the resolution logic in `open()` / `doOpenEmpty()` / `doOpenFolderOrWorkspace()` may need an additional check when no non-sessions window exists — in that case, a new editor window should be created rather than falling back to the sessions window.
