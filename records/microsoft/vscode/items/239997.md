Team-authored accessibility feature request for a progressive audio cue during edit application — kept open for maintainer judgment.

## Change summary

Add a progressive sound/audio cue that conveys the percentage of edits applied, giving non-sighted users parity with the visual progress indicator.

Implement a progressive accessibility signal (rising tone or incremental cue) in the edit-application flow, likely in the accessibility signal service, so screen-reader users get real-time feedback on edit progress without relying on the visual percentage.
