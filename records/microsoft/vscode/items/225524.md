# Issue #225524 — Forward/Back buttons on mouse do not work in fullscreen/maximized window on macOS Sonoma

[https://github.com/microsoft/vscode/issues/225524](https://github.com/microsoft/vscode/issues/225524)

## Summary

Already correctly triaged as an upstream Chromium bug by a team member. The same behavior is reported in Microsoft Edge and other Chromium-based apps on macOS fullscreen, confirming the root cause is in Chromium's macOS event handling, not VS Code's code. Keeping open as an upstream tracker.

## Change summary

Mouse back/forward buttons (buttons 3/4) stop working when VS Code is in macOS fullscreen mode, affecting Logitech and other mice.

## Best solution

The fix belongs in Chromium's macOS event pipeline for handling auxiliary mouse button events in fullscreen/native-tab mode. VS Code should keep this open as an upstream tracker and link the Chromium bug if one is filed. No workaround is feasible in VS Code's own code — the events never reach the renderer process.
