# Issue #227697 — LSP DocumentColor Refresh Request

[https://github.com/microsoft/vscode/issues/227697](https://github.com/microsoft/vscode/issues/227697)

## Summary

Team-authored feature request; no implementation of a documentColor refresh mechanism exists on current main. The feature is still needed.

## Change summary

LSP servers need a way to trigger documentColor refresh requests in VS Code (workspace/documentColor/refresh), similar to existing refresh requests for folding ranges, inlay hints, etc.

## Best solution

Implement a documentColor refresh provider event (analogous to the existing folding-range/inlay-hint refresh patterns) in the extension host API and mainThread language features, then wire it through to the color picker contribution so LSP servers can signal when colors should be re-fetched.
