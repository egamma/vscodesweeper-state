# Issue #213197 — Custom labels: support full file name variable

[https://github.com/microsoft/vscode/issues/213197](https://github.com/microsoft/vscode/issues/213197)

## Summary

The request for a `${fullFileName}` (i.e. basename) variable in custom labels is still unimplemented on main. The reporter found a workaround using multiple glob patterns, but the ergonomic single-variable solution has not been added. Keeping open as a valid enhancement in the Backlog.

## Change summary

Add a full file name variable (e.g. `${basename}`) to custom editor labels so users don't need multiple glob patterns to handle dotfiles and extensionless files.

## Best solution

Add a `${basename}` variable to the `_parsedTemplateExpression` regex in `src/vs/workbench/services/editor/common/customEditorLabelService.ts` that returns `parsedPath.base` directly. Small, localized change.
