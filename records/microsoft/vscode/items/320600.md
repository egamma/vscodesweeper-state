# Issue #320600 — Copilot Chat activation delayed 80-120s in WSL remote due to `onDidChangeSessions` IPC proxy latency

[https://github.com/microsoft/vscode/issues/320600](https://github.com/microsoft/vscode/issues/320600)

## Summary

Legitimate performance bug: Copilot Chat activation is blocked 80–120s in remote (WSL/SSH) workspaces because it passively waits for `onDidChangeSessions` IPC events that are slow to arrive from the local extension host. Confirmed by a second user on SSH remote. No fix on current main — the event proxy path in `extHostAuthentication.ts` and `mainThreadAuthentication.ts` remains reactive with no eager session forwarding.

## Change summary

Copilot Chat activation is delayed 80–120 seconds in remote workspaces (WSL, SSH) because authentication session change events take that long to be proxied from the local to the remote extension host.

## Best solution

Two complementary fixes: (1) In the Copilot extension, proactively call `vscode.authentication.getSession('github', scopes, { silent: true })` at activation rather than passively waiting for `onDidChangeSessions`. (2) In the platform (`src/vs/workbench/api/browser/mainThreadAuthentication.ts`), eagerly forward cached authentication sessions to remote extension hosts during the initialisation handshake so extensions don't depend on the event being proxied after startup.
