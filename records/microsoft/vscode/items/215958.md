# Issue #215958 — [folding] pasting over collapsed blocks leaves random lines collapsed

[https://github.com/microsoft/vscode/issues/215958](https://github.com/microsoft/vscode/issues/215958)

## Summary

The bug remains plausible on current main. When pasting over collapsed regions, decoration ranges shift but their collapsed state persists, causing stale folding. No fix commit was found.

## Change summary

Pasting text that replaces or overlaps collapsed folding regions leaves random lines collapsed because decoration-tracked collapse state survives the edit with shifted ranges.

## Best solution

In `_currentFoldedOrManualRanges` (src/vs/editor/contrib/folding/browser/foldingModel.ts), detect when a decoration range has been invalidated by content changes (e.g. its content no longer matches the original folding region) and discard or uncollapse it, rather than blindly re-applying the collapsed state to the shifted range.
