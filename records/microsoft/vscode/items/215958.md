# Issue #215958 — [folding] pasting over collapsed blocks leaves random lines collapsed

[https://github.com/microsoft/vscode/issues/215958](https://github.com/microsoft/vscode/issues/215958)

## Summary

Still reproducible on main. When pasting over collapsed regions, the selection-based uncollapse logic only expands ranges where the cursor ends up, but shifted decorations from other collapsed regions can survive the paste, leaving random lines hidden.

## Change summary

Pasting text over collapsed folding blocks leaves some lines incorrectly collapsed/hidden.

## Best solution

In `_currentFoldedOrManualRanges` (src/vs/editor/contrib/folding/browser/foldingModel.ts), the uncollapse check should consider the full edited range (not just cursor position) — any collapsed range whose decoration overlaps or was displaced by the edit should be uncollapsed. Alternatively, `notifyChangeModelContent` in hiddenRangeModel.ts could track the affected line ranges and force-expand any collapsed region that intersects the replaced span.
