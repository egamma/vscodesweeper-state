# Issue #324409 — chat: make maximum persisted sessions a user-configurable setting

[https://github.com/microsoft/vscode/issues/324409](https://github.com/microsoft/vscode/issues/324409)

## Summary

Valid feature request — the 50-session limit is still hardcoded on current main with no user-facing setting. The issue is in Backlog Candidates with 11 upvotes (needs 20 for promotion); keeping open for the community process.

## Change summary

Expose the hardcoded 50-session chat history limit as a user-configurable setting so heavy Copilot Chat users can retain more sessions.

## Best solution

Add a `chat.session.maxPersistedSessions` setting (default 50, min ~10) and have `trimEntries()` in `src/vs/workbench/contrib/chat/common/model/chatSessionStore.ts` read from `IConfigurationService` instead of the hardcoded constant. Small, localized change.
