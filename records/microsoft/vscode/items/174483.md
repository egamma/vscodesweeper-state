# Issue #174483 — Git - Git checkout for remote branch does not work

[https://github.com/microsoft/vscode/issues/174483](https://github.com/microsoft/vscode/issues/174483)

## Summary

Still valid on current main: the checkout error message at line 2964 of commands.ts says 'Your local changes would be overwritten by checkout' without listing the affected files. A maintainer acknowledged this should be improved but no fix has landed for the standard checkout path.

## Change summary

Git checkout error message does not list which files would be overwritten, making it hard for the user to know what to stash or commit.

## Best solution

Extract the file list from `err.stderr` (git's error output includes the affected filenames) and include it in the warning message at `extensions/git/src/commands.ts:2964`, similar to how the worktree migration path at line 2779 of `repository.ts` already lists files.
