# Issue #147105 — Clicking in the middle of a selected range, does not deselect the text sometimes

[https://github.com/microsoft/vscode/issues/147105](https://github.com/microsoft/vscode/issues/147105)

## Summary

Still reproducible on main: the cursor hit-testing logic in `_hitTestViewCursor` uses last-rendered cursor position/width to intercept clicks, which can incorrectly snap a click to the cursor position instead of the actual text position. Issue #164484 already identifies this as the root cause but remains open.

## Change summary

Clicking inside a selected range occasionally fails to deselect the text, particularly with wider cursor widths.

## Best solution

Remove or constrain the positional cursor hit-testing in `_hitTestViewCursor` (lines 640–670 of `src/vs/editor/browser/controller/mouseTarget.ts`), as proposed in https://github.com/microsoft/vscode/issues/164484. The Edge workaround is stale and the hit-test width grows with `editor.cursorWidth`, causing false positives.
