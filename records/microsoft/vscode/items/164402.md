# Issue #164402 — Git - operations using SSH with a FIDO/U2F key trigger a host authentication dialog

[https://github.com/microsoft/vscode/issues/164402](https://github.com/microsoft/vscode/issues/164402)

## Summary

Still reproducible on main: the SSH askpass handler in the git extension doesn't recognize FIDO/U2F 'Confirm user presence' prompts, misrouting them to the host-key authenticity dialog with an undefined fingerprint. The bug is real, has 26 upvotes, and has no merged fix.

## Change summary

SSH operations with FIDO/U2F keys (ecdsa-sk, ed25519-sk) incorrectly trigger a host authenticity dialog instead of a user-presence confirmation prompt.

## Best solution

Add a branch in `extensions/git/src/askpass.ts` `handleSSHAskpass()` (around line 117) to detect 'Confirm user presence' prompts and show an appropriate notification (or return empty string to let SSH proceed), rather than falling through to the host-key authenticity handler. Also handle the PIN entry prompt for verify-required keys. The closed PR https://github.com/microsoft/vscode/pull/165976 attempted this but was never merged — it could serve as a starting point.
