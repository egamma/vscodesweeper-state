# Issue #170407 — Two extensions with conflicting custom semantic tokens cause vscode to crash

[https://github.com/microsoft/vscode/issues/170407](https://github.com/microsoft/vscode/issues/170407)

## Summary

Keep open: the report describes a core crash path from extension contributions, and current main still appears to accept conflicting semantic token supertype registrations without cycle protection. This is already routed to the right area and needs a product/code fix rather than a cleanup close.

## Change summary

Prevent conflicting custom semantic token type hierarchies from creating an infinite loop and crashing VS Code.

## Best solution

Add defensive validation in the semantic token contribution/registry path so duplicate or cyclic token type hierarchies are rejected with an extension contribution error, while the workbench continues running. The likely code path is `src/vs/workbench/services/themes/common/tokenClassificationExtensionPoint.ts` and `src/vs/platform/theme/common/tokenClassificationRegistry.ts`.
