# Issue #182641 — Multicursor on duplicated lines disappears

[https://github.com/microsoft/vscode/issues/182641](https://github.com/microsoft/vscode/issues/182641)

## Summary

Still reproduces on main: when multiple cursors exist on the same line and the user duplicates the line, the dedup logic in AbstractCopyLinesAction marks all but one cursor as noop, resulting in cursor loss on the duplicated line.

## Change summary

Duplicating a line that has multiple cursors (from Ctrl+D) drops most cursors, leaving only one or two instead of preserving all.

## Best solution

The dedup logic in `AbstractCopyLinesAction.run()` (src/vs/editor/contrib/linesOperations/browser/linesOperations.ts, lines 50-65) needs to preserve multiple cursors on the same line when duplicating. The noop path in `CopyLinesCommand` should still track and restore all original cursor positions on the newly created line rather than collapsing them.
