# Issue #182641 — Multicursor on duplicated lines disappears

[https://github.com/microsoft/vscode/issues/182641](https://github.com/microsoft/vscode/issues/182641)

## Summary

Keep open: the report is already routed to editor multicursor and I do not see strong evidence for a duplicate, extension cause, or shipped fix. Current main still has copy-line selection handling that could plausibly collapse same-line multicursor selections, and the existing tests do not cover this scenario.

## Change summary

Copying or duplicating a line after selecting multiple occurrences on that same line should preserve all resulting cursors/selections on the duplicated line.

## Best solution

Add a focused regression test in src/vs/editor/contrib/linesOperations/test/browser/copyLinesCommand.test.ts for multiple non-empty selections on one line, then fix src/vs/editor/contrib/linesOperations/browser/linesOperations.ts or copyLinesCommand.ts so Copy Line Down preserves every expected selection after deduplicating the line edit.
