# Issue #250402 — Chat continues progress in background when sending empty message.

[https://github.com/microsoft/vscode/issues/250402](https://github.com/microsoft/vscode/issues/250402)

## Summary

Fixed on main: the chat submit action now requires sendable content as a precondition, and the Enter key is blocked on empty input — empty messages can no longer be submitted, so progress signals won't fire indefinitely.

## Change summary

Sending an empty message in chat triggers indefinite progress signals (sound and announcements) because the request proceeds with no content.

## Best solution

Keep the shipped fix (PR #270117, merged Oct 2025, shipped 1.106.0): the submit action's precondition gates on `inputHasSendableContent` and the Enter key is intercepted when input is empty. No further change needed.
