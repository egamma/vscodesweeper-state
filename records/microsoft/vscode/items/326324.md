# Issue #326324 — SSH askpass shows yes/no host-fingerprint prompt instead of PIN input for FIDO2/security-key authentication

[https://github.com/microsoft/vscode/issues/326324](https://github.com/microsoft/vscode/issues/326324)

## Summary

Verified duplicate of the open canonical issue tracking FIDO/U2F SSH key handling in VS Code's askpass. The root cause — the `/passphrase/i` regex in `handleSSHAskpass()` not matching FIDO2 PIN prompts — is the same defect described in the canonical issue, which already has extensive community discussion and workarounds.

## Change summary

VS Code's SSH askpass handler misclassifies FIDO2 PIN prompts as host-key verification prompts, showing a yes/no dialog instead of a password input box.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/164402, which tracks the full scope of FIDO/U2F key handling in the git extension's askpass. Fixing the regex in `extensions/git/src/askpass.ts` to also match PIN and user-presence prompts resolves both issues.
