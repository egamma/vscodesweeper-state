# Issue #243110 — "Insert At Cursor" should be disabled when no text editor is opened

[https://github.com/microsoft/vscode/issues/243110](https://github.com/microsoft/vscode/issues/243110)

## Summary

Still reproduces on main: the InsertAtCursor action's precondition only checks `ChatContextKeys.enabled` with no guard for an active text editor, so it remains enabled even when no editor is open.

## Change summary

The 'Insert At Cursor' code block action in chat is enabled even when no text editor is visible, causing an error dialog on use.

## Best solution

Add a context key guard (e.g. `ActiveEditorContext` or a custom 'hasTextEditor' key) to the `precondition` of `InsertAtCursorAction` in `chatCodeblockActions.ts` so the action is disabled when no text editor is open.
