# Issue #312772 — Commands are auto approved even when some subcommands are not correctly parsed

[https://github.com/microsoft/vscode/issues/312772](https://github.com/microsoft/vscode/issues/312772)

## Summary

Valid security-relevant bug: the CommandAutoApprover never checks tree.rootNode.hasError() after parsing, so partially-parsed compound commands can be auto-approved based only on the successfully-captured subcommands while destructive subcommands hiding in error nodes are ignored.

## Change summary

Compound terminal commands with parse errors (e.g. PowerShell end-of-parameters token not recognized by the bash grammar) get auto-approved based on whichever subcommand happens to parse successfully, skipping unparsed destructive subcommands.

## Best solution

Add a `tree.rootNode.hasError()` check in `_extractSubCommands` (src/vs/platform/agentHost/node/commandAutoApprover.ts ~line 258) and return `undefined` when the parse contains error nodes, falling through to `noMatch` so the user is prompted for confirmation rather than partially auto-approving.
