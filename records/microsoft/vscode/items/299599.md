# Issue #299599 — MCP and C# Dev Kit sign-in fails repeatedly when MSAL broker is enabled — auth prompt loops or silently drops session

[https://github.com/microsoft/vscode/issues/299599](https://github.com/microsoft/vscode/issues/299599)

## Summary

Team-authored issue; the described race condition is confirmed present on current main — `setAllowedAccess` does not update `this.value` in-memory before `_update()` filters against it. Keeping open per hard keep-open rule (MEMBER author) and because the bug remains unfixed.

## Change summary

MSAL broker auth flows fail because `ScopedAccountAccess.setAllowedAccess()` does not synchronously update its in-memory cache before the subsequent `_update()` call filters the account out.

## Best solution

Add `this.value = [...this.value, account.homeAccountId]` (or the removal equivalent) synchronously inside `setAllowedAccess()` before the SecretStorage write, so that `isAllowedAccess()` returns the correct value immediately. The fix is localized to `extensions/microsoft-authentication/src/common/accountAccess.ts`.
