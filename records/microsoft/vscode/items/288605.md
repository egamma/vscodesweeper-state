# Issue #288605 — VSCode resizes its window every time it starts

[https://github.com/microsoft/vscode/issues/288605](https://github.com/microsoft/vscode/issues/288605)

## Summary

Correctly labeled as an upstream Electron issue. Window bounds drift on restart when Windows text scaling is set to a non-default value — the root cause is in Electron's bounds get/set round-tripping under non-standard DPI/text-scale configurations. The existing labels (`upstream`, `electron`, `workbench-window`) are appropriate; keeping open to track the upstream fix.

## Change summary

VS Code window progressively resizes and shifts position on each restart when Windows text scaling is set to a non-100% value.

## Best solution

Track and push for an Electron fix for bounds round-tripping under Windows text scaling. VS Code's `getBounds()` in `windowImpl.ts` uses `getPosition()`/`getSize()` which Electron reports in logical coordinates affected by text scaling — the mismatch accumulates across save/restore cycles. A workaround in VS Code (e.g. snapping saved bounds to nearest pixel) could mitigate, but the proper fix belongs in Electron's coordinate handling.
