# Issue #94375 — vscode://file/ URI not working from chromium

[https://github.com/microsoft/vscode/issues/94375](https://github.com/microsoft/vscode/issues/94375)

## Summary

Confirmed upstream snap-confinement issue preventing vscode:// protocol handler from working between snap-packaged Chromium and snap-packaged VS Code. A contributor acknowledged this as a known snap limitation in 2020; the root cause lives in snap's sandboxing, not in VS Code's code. Keeping open per upstream policy.

## Change summary

The vscode://file/ custom protocol URI does not work when both VS Code and Chromium are installed as snap packages on Linux, due to snap confinement restrictions.

## Best solution

This requires a fix in the snap platform itself (snap's inter-application protocol handler forwarding). VS Code cannot resolve this unilaterally. The issue serves as a tracking point for when snap improves its confinement model to allow custom protocol handlers across snap boundaries. Users can work around it by installing either VS Code or Chromium via non-snap channels (e.g. .deb package).
