# Issue #100225 — Data corruptted when pasting large JSON in the terminal (or via sendText)

[https://github.com/microsoft/vscode/issues/100225](https://github.com/microsoft/vscode/issues/100225)

## Summary

Still reproduces on current main — the single unbuffered `ptyProcess.write()` in the terminal input path corrupts multi-line pastes into shells without bracketed paste on macOS. A chunking fix was attempted (commit eb7e1c7cb89) but reverted (commit f144e450250). The bug remains open and valid.

## Change summary

Pasting multi-line text into the integrated terminal on macOS corrupts data when the shell lacks bracketed paste support, because the PTY write is not chunked.

## Best solution

Implement chunked writes in `TerminalProcess.input()` for multi-line data on macOS (`src/vs/platform/terminal/node/terminalProcess.ts`), avoiding whatever regression caused the first chunking attempt (#298993) to be reverted (#300933). The fix needs to respect flow control so readline-based shells can keep up. The `sendText` path is already fixed; only the paste/input path remains.

- **Fix prompt:**

  > The `input()` method in `src/vs/platform/terminal/node/terminalProcess.ts` (lines 484–492) writes the entire paste payload to `this._ptyProcess!.write(data)` in a single call. On macOS, when pasting multi-line text (≥2 KB) into a shell without bracketed paste (e.g. `/bin/bash` 3.2), the kernel's TTY input buffer (1024 bytes, `MAX_INPUT`) overflows and garbles the data — dropped chars, duplicated fragments, eaten lines. The fix: chunk the write into segments that stay under the TTY buffer limit (e.g. 512 bytes) with a small delay between chunks (e.g. 5–10 ms), but ONLY on macOS and ONLY for non-binary input — binary writes and non-macOS platforms must remain single-write. A prior attempt (commit eb7e1c7cb89, PR #298993) was reverted (commit f144e450250, PR #300933) — examine that diff to understand what went wrong (likely the chunking interfered with normal typing or flow control). The new fix should be more conservative: chunk only when `data.length` exceeds a threshold AND `process.platform === 'darwin'` AND `!isBinary`. Do not restructure the rest of the input/output pipeline. Add a unit test that sends a ≥2 KB multi-line string through the input path and verifies the PTY receives it intact. See also https://github.com/microsoft/vscode/issues/100225.
