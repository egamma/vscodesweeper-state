# Issue #237517 — Incorrect TerminalShellExecution exit code

[https://github.com/microsoft/vscode/issues/237517](https://github.com/microsoft/vscode/issues/237517)

## Summary

Still reproduces on main: the exit-code hack at line 394–404 of commandDetectionCapability.ts blindly assigns the previous exit code when the current one is undefined and the command string matches, without detecting an interrupt. An open PR (#307256) addresses this but has not merged.

## Change summary

Interrupted (^C) commands incorrectly inherit the previous command's exit code when the command text matches.

## Best solution

Land (or iterate on) the open PR https://github.com/microsoft/vscode/pull/307256 which adds interrupt detection to the exit-code hack in commandDetectionCapability.ts. A maintainer already approved approach 2 (detect ^C and skip the hack).
