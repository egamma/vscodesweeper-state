# Issue #278353 — Word Wrap turns off when switching files in Explorer panel

[https://github.com/microsoft/vscode/issues/278353](https://github.com/microsoft/vscode/issues/278353)

## Summary

The View > Toggle Word Wrap action stores a transient per-model override that is intentionally cleared when the model is disposed (as happens in Explorer preview/single-click mode). This is as-designed; the global setting `editor.wordWrap` is the correct way to enable persistent word wrap.

## Change summary

Word wrap toggled via the View menu resets when switching files in the Explorer panel (preview mode), because the transient state is per-model and lost on model disposal.

## Best solution

Close as designed. The toggle action is intentionally transient and per-file. Users wanting persistent word wrap should set `"editor.wordWrap": "on"` in their settings.

- **Proposed comment:**

  > Thanks for the report. The "View: Toggle Word Wrap" action is designed to apply a **transient per-file override** — it's stored on the text model and intentionally lost when that model is disposed. In Explorer preview mode (single-click), files share one editor slot, so the old model is disposed and the override is cleared.
  > 
  > If you want word wrap to persist across all files, the right approach is to set `"editor.wordWrap": "on"` in your user settings (open Settings with `Ctrl+,` and search for "word wrap").
  > 
  > Closing this as the current behavior is by design.
