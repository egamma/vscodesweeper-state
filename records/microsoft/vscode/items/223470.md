# Issue #223470 — Scroll momentum transferred between different scrollables

[https://github.com/microsoft/vscode/issues/223470](https://github.com/microsoft/vscode/issues/223470)

## Summary

This should stay open as a real scrollable-widget bug: a VS Code member confirmed the behavior appears to affect all internal Scrollable Elements, not only editor tabs. There is no linked duplicate, fixing PR, commit, or verification evidence in the issue context that would support a close proposal.

## Change summary

Fast inertial scrolling in an editor can continue under the pointer after moving to another scrollable VS Code control, causing unintended scrolling such as horizontal tab-list movement.

## Best solution

Route this through the scroll infrastructure owners and investigate whether VS Code's shared ScrollableElement handling can suppress or isolate momentum events when pointer focus moves between scroll containers; the likely implementation area is shared scrolling behavior rather than editor tabs alone.
