# Issue #322010 — editor.insertSnippet() still destroys active snippet session tabstops (regression from #214757, misclosed)

[https://github.com/microsoft/vscode/issues/322010](https://github.com/microsoft/vscode/issues/322010)

## Summary

Confirmed regression on current main: `mainThreadEditor.ts` routes extension `insertSnippet()` through `snippetController.apply(edits[])`, which passes an array to `_doInsert()` and triggers `this.cancel()` at line 149 of `snippetController2.ts`, destroying the active snippet session. The merge path (for strings) works correctly but is no longer used by the API bridge.

## Change summary

Extension API `editor.insertSnippet()` destroys the active snippet session's remaining tabstops because the API bridge uses the `apply()` (array) path instead of the `insert()` (string/merge) path.

## Best solution

Route single-edit `insertSnippet()` calls through `snippetController.insert()` (the merge-capable string path) in `src/vs/workbench/api/browser/mainThreadEditor.ts`, falling back to `apply()` only for multi-range edits. The author's open PR #322008 addresses this directly.
