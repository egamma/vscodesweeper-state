# Issue #249519 — Make `IChatRequestVariableEntry` more OOP like

[https://github.com/microsoft/vscode/issues/249519](https://github.com/microsoft/vscode/issues/249519)

## Summary

The requested OOP refactoring of IChatRequestVariableEntry has not been done. The type remains a union of interfaces with namespace utility functions (toExport/fromExport, toUri) rather than classes with toJSON/fromJSON and narrowed subtypes. This is a valid, actionable debt item that should stay open.

## Change summary

Refactor IChatRequestVariableEntry from open data + utility functions to OOP classes with toJSON/fromJSON, concrete subtypes narrowing the value type, and better encapsulation.

## Best solution

Convert IChatRequestVariableEntry and its variants into proper OOP classes with serialization (toJSON/fromJSON) for IPC and persistence, concrete subtypes that narrow the `value` type, and revisit all consumers of `value` to use typed access. Primary file: src/vs/workbench/contrib/chat/common/attachments/chatVariableEntries.ts, with ripple effects across ~20 consumers.
