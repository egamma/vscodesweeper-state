# Issue #309828 — Option to disable auto “Jump to next edit” after approving a Copilot/AI edit (not file-based)

[https://github.com/microsoft/vscode/issues/309828](https://github.com/microsoft/vscode/issues/309828)

## Summary

Valid, distinct feature request. The existing `chat.editing.revealNextChangeOnResolve` setting (added for #298434) only controls cross-file navigation after keep/undo. The in-file jump to the next hunk (unconditional `this.next(true)` in `acceptNearestChange`) has no setting to disable it. This issue correctly identifies that gap.

## Change summary

After accepting/rejecting a single hunk in chat edits, the cursor unconditionally jumps to the next hunk within the same file; the user wants a setting to disable this in-file jump.

## Best solution

Extend `chat.editing.revealNextChangeOnResolve` (or add a sibling setting) to also gate the `this.next(true)` call in `chatEditingCodeEditorIntegration.ts:acceptNearestChange` so that hunk approval can leave the cursor in place. The fix is small and localized to that method.
