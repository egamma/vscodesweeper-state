# Issue #107090 — Add selection to next Find match does not work with Regex Find Mode

[https://github.com/microsoft/vscode/issues/107090](https://github.com/microsoft/vscode/issues/107090)

## Summary

Confirmed on current main: MultiCursorSession never stores or passes isRegex to findNextMatch/findPreviousMatch, so Ctrl+D always does a literal search even when the find widget has regex enabled. The selectAll path correctly handles regex, but the per-match path does not.

## Change summary

Ctrl+D (addSelectionToNextFindMatch) ignores the regex flag from the find widget, treating the search string as a literal even when regex mode is active.

## Best solution

Store `isRegex` in `MultiCursorSession` (from `findState.isRegex`) and pass it through to `findNextMatch`/`findPreviousMatch`/`findMatches` in `_getNextMatch`, `_getPreviousMatch`, and `selectAll`. Also handle the disconnected case (line 494) where regex should be forced off. The fix is small and localized to `src/vs/editor/contrib/multicursor/browser/multicursor.ts`.
