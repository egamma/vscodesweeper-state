## Summary

The issue should remain open: it is still labeled as an accepted bug in the Backlog with recent contributor activity, and current main still appears to omit regex state from the multi-cursor find session. There is not enough evidence for a duplicate, completed fix, or non-reproducible close.

## Change summary

`Add selection to next Find match` should honor the Find widget's regex mode when adding the next match to the selection.

## Best solution

Update the editor multi-cursor find path so `MultiCursorSession` carries the current regex find state into `findNextMatch`, `findPreviousMatch`, and `findMatches`, then add focused editor tests for Ctrl+D / add-selection-next-match with regex enabled.
