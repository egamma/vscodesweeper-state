# Issue #158064 — Turn quick access prefixes into blocks so that they can be easily cleared

[https://github.com/microsoft/vscode/issues/158064](https://github.com/microsoft/vscode/issues/158064)

## Summary

Still unimplemented on main: the quick access prefix input box has no chip/block rendering. Team members expressed interest in 2022 but no implementation followed. Valid feature request with 12 upvotes, below the 20-upvote Backlog promotion threshold but with clear team engagement.

## Change summary

Render quick access prefixes (>, ?, @, #, etc.) as deletable chip/block UI elements for easier clearing and discoverability.

## Best solution

Implement chip-style prefix rendering in `src/vs/platform/quickinput/browser/quickInputBox.ts` — similar to how chat slash commands render as blocks (PR #187620) but for quick access mode prefixes. Needs UX decision on whether to show symbolic prefixes (>) or named labels (Commands).
