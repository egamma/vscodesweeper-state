# Issue #170868 — Adapt the bash of busybox.

[https://github.com/microsoft/vscode/issues/170868](https://github.com/microsoft/vscode/issues/170868)

## Summary

Still reproduces on main: the bash shell integration script uses bash-specific constructs (builtin, BASH_VERSINFO, arrays) without checking whether /bin/bash is actually busybox ash. Three open PRs attempt to fix this but none have merged.

## Change summary

VS Code's bash shell integration script fails with errors on systems where /bin/bash is actually busybox ash (e.g. Alpine Linux).

## Best solution

Land one of the open PRs (https://github.com/microsoft/vscode/pull/295061, https://github.com/microsoft/vscode/pull/300900, or https://github.com/microsoft/vscode/pull/320540) that adds an early-exit guard in shellIntegration-bash.sh when busybox is detected. The maintainer-suggested approach (comment 5) is a simple `if` at the top of the script to exit early when the shell is busybox.
