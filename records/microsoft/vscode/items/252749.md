# Issue #252749 — Setting `workbench.panel.opensMaximized` behaves strangely

[https://github.com/microsoft/vscode/issues/252749](https://github.com/microsoft/vscode/issues/252749)

## Summary

Still reproduces on current main: the `preserve` mode of `workbench.panel.opensMaximized` fails because hiding a maximized panel calls `toggleMaximizedPanel()` before saving `PANEL_WAS_LAST_MAXIMIZED`, likely corrupting the remembered state. A team member is actively investigating.

## Change summary

The `workbench.panel.opensMaximized` setting with `preserve` value does not restore the panel to maximized after hide/show when it was maximized before hiding.

## Best solution

Fix the ordering or state-save logic in `setPanelHidden()` in `src/vs/workbench/browser/layout.ts` so that the `PANEL_WAS_LAST_MAXIMIZED` runtime value is correctly preserved when the panel is unmaximized during the hide sequence. bpasero noted commenting out the unmaximize code alone doesn't fix it, so additional state interactions need investigation.
