## Change summary

Enter, Backspace, and Ctrl keys are not forwarded to the shell when the integrated terminal is in raw mode on macOS 26, breaking interactive CLI prompts.

## Best solution

Now that a second reporter has provided concrete reproduction steps (`brew install gum && gum confirm "test"`), the team should retry reproduction on macOS 26 with that workflow. If reproducible, the root cause likely lives in xterm.js's renderer initialization path (the dimensions getter returns undefined before the renderer is ready, corrupting terminal state). The fix would be either upstream in xterm.js or a guard in VS Code's terminal key dispatch.
