# Issue #107265 — Support tab completion / custom key handlers in QuickPick

[https://github.com/microsoft/vscode/issues/107265](https://github.com/microsoft/vscode/issues/107265)

## Summary

Valid feature request for custom key handlers in the QuickPick extension API. Not yet implemented on main — no proposed or stable API exists for this. Low traction (7 upvotes) but a reasonable extensibility gap; keep open in Backlog for the community process.

## Change summary

Allow extensions to register custom key event handlers (e.g. Tab) on the QuickPick input box to implement behaviors like path tab-completion.

## Best solution

Expose an `onDidTriggerKey` or similar event on the QuickPick extension API (in `src/vs/platform/quickinput/common/quickInput.ts` and `src/vscode-dts/vscode.d.ts`) that lets extensions intercept specific key presses in the input box. A keybinding-based workaround exists but is fragile. The community process should decide whether to promote this based on demand.
