# Issue #143956 — Better path presentation in quick open window

[https://github.com/microsoft/vscode/issues/143956](https://github.com/microsoft/vscode/issues/143956)

## Summary

Valid, well-upvoted feature request (37 upvotes, above the 20-vote Backlog threshold) with multiple open PRs attempting implementation but none yet merged. Keeping open for the area owner to shepherd one of the PRs to completion.

## Change summary

When file paths are truncated in Quick Open, the ellipsis appears at the end, hiding the distinguishing directory segments — users want left-truncation so the unique tail of the path remains visible.

## Best solution

Land one of the open PRs (https://github.com/microsoft/vscode/pull/310761 looks most recent and focused). The implementation should left-truncate the description path at a path-separator boundary using CSS `direction: rtl` or a TS-based approach, preserving fuzzy-match highlights. A setting to control truncation direction would be ideal.
