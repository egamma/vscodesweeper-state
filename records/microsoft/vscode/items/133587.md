# Issue #133587 — Allow pinning hover popovers (datatips)

[https://github.com/microsoft/vscode/issues/133587](https://github.com/microsoft/vscode/issues/133587)

## Summary

Valid feature request with 31 upvotes, properly in Backlog. The requested persistent pinning of hover popovers is not implemented — only the existing `editor.hover.sticky` setting exists, which merely keeps the hover alive while the mouse is over it, not a true pin.

## Change summary

Allow users to pin hover popovers so they remain visible while continuing to edit code, similar to Atom's datatips or Visual Studio's pinned DataTips.

## Best solution

Implement a pin action on the editor hover widget that keeps it visible independently of mouse/focus state, possibly with drag-to-reposition support. This would likely involve changes to `src/vs/editor/contrib/hover/browser/contentHoverWidgetWrapper.ts` and the hover controller, plus an extension API surface (given the `api` label) for language extensions to participate.
