# Issue #290708 — If multiple instances of VSCode are opened for different git worktrees, the git editor is opened in the main worktree VSCode instance

[https://github.com/microsoft/vscode/issues/290708](https://github.com/microsoft/vscode/issues/290708)

## Summary

Valid bug report about the git editor opening in the wrong VS Code window when using worktrees. The maintainer (lszomoru) engaged with a clarifying question, and a closely related issue (https://github.com/microsoft/vscode/issues/304579) tracks a broader worktree-dispatch problem with the same assignee. No fix has been applied to the git editor IPC path; keeping open for the area owner.

## Change summary

When multiple VS Code instances are open for different git worktrees, git editor requests (commit messages, rebase todo) open in the main worktree's VS Code window instead of the one where the git command was run.

## Best solution

The root cause likely lives in the IPC handle derivation in `extensions/git/src/ipc/ipcServer.ts` or the terminal environment propagation in `extensions/git/src/main.ts`. The area owner (lszomoru) should investigate whether `context.storagePath` collides across worktree windows of the same repo, or whether the `VSCODE_GIT_IPC_HANDLE` env var in the terminal isn't correctly scoped per window. This is closely related to https://github.com/microsoft/vscode/issues/304579 (wrong-worktree dispatch causing data loss) and may share a root cause, but this issue is narrower (editor routing only) and was filed first — the area owner should decide whether to consolidate.
