# Issue #326092 — VSCode crashes as soon as it starts with codex extension pane

[https://github.com/microsoft/vscode/issues/326092](https://github.com/microsoft/vscode/issues/326092)

## Summary

Confirmed crash regression on Linux in 1.129.0, reproduced by a team member (deepak1556), with two open fix PRs in progress. Hard keep-open: carries `candidate` label and milestone `1.129.1`.

## Change summary

VS Code 1.129.0 crashes on startup on Linux when webview-heavy extensions (e.g. Codex) restore their panes, with clean-exit code 0 or SIGSEGV/SIGBUS renderer crashes.

## Best solution

Land the fix from https://github.com/microsoft/vscode/pull/326272 (or its release-branch variant https://github.com/microsoft/vscode/pull/326278) which mitigates Linux webview renderer FD exhaustion during resource loading, and ship in 1.129.1. The Electron 42.6.2 upgrade mentioned in the thread may also help with related renderer crashes.
