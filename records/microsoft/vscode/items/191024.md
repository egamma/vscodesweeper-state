# Issue #191024 — Consider moving `NotebookSearchService` to be called with `ISearchService` or related service

[https://github.com/microsoft/vscode/issues/191024](https://github.com/microsoft/vscode/issues/191024)

## Summary

Team-authored architectural debt issue. The NotebookSearchService still exists as a separate service on current main — callers must still independently invoke it alongside ISearchService. The requested unification has not been done.

## Change summary

Unify notebook search into ISearchService (or a central function) so callers don't need to separately orchestrate notebook and text search results.

## Best solution

Refactor the search contribution so that ISearchService (or a new unified entry point) internally dispatches to NotebookSearchService, removing the need for callers like SearchModel to manually coordinate both services. The work lives in `src/vs/workbench/contrib/search/browser/notebookSearch/` and `src/vs/workbench/contrib/search/browser/searchTreeModel/searchModel.ts`.
