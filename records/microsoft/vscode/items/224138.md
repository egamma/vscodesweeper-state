# Issue #224138 — Catch-all check for deletion of nameless folder

[https://github.com/microsoft/vscode/issues/224138](https://github.com/microsoft/vscode/issues/224138)

## Summary

Valid data-loss concern: no guard exists in the delete path against resources with empty basenames. The exact SSH-disconnect reproduction is hard to replicate, but the defensive check the reporter requests is reasonable. A team member (lramos15) expressed interest. Keeping open for investigation — the `confirmation-pending` label is appropriate.

## Change summary

Deleting a 'ghost' folder (created during an SSH disconnect with no name) recursively deleted the parent directory, causing significant data loss.

## Best solution

Add a defensive guard in `deleteFiles` (src/vs/workbench/contrib/files/browser/fileActions.ts) or `doValidateDelete` (src/vs/platform/files/common/fileService.ts) that refuses to delete a resource whose basename resolves to empty string, preventing accidental parent-directory deletion regardless of root cause. The lower-level fileService guard would be more robust.
