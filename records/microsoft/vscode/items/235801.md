# Issue #235801 — Report issue fails with HTTP 401

[https://github.com/microsoft/vscode/issues/235801](https://github.com/microsoft/vscode/issues/235801)

## Summary

Still reproducible on main: the `submitToGitHub` method in `baseIssueReporterService.ts` silently swallows HTTP 401 errors with a misleading 'Invalid GitHub URL provided' console message and no user-visible feedback — the button appears to do nothing. The scope-filtering logic could also pick up a stale or under-scoped token. Keeping open as a real UX bug.

## Change summary

When the issue reporter's 'Create on GitHub' button gets an HTTP 401 from the GitHub API, no user-visible error is shown — the button silently fails.

## Best solution

Improve error handling in `submitToGitHub` (`baseIssueReporterService.ts` and `issueReporterService.ts` electron variant): distinguish 401/403 from other failures, show a user-visible notification suggesting re-authentication, and fall back to the 'Preview on GitHub' flow so the user can still file the issue. The misleading 'Invalid GitHub URL provided' log should be replaced with a status-specific message.
