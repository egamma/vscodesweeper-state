# Issue #223307 — SCM Graph - View commit action on multiple commits is showing no changed files

[https://github.com/microsoft/vscode/issues/223307](https://github.com/microsoft/vscode/issues/223307)

## Summary

Team-authored issue (sandy081) — keeping open per policy. The bug is still present on current main: the viewChanges action only processes the first selected history item, ignoring multi-selection, and commit ordering can produce empty diffs.

## Change summary

Selecting multiple commits in the SCM Graph and choosing 'View Changes' shows no changed files, because the action only processes the first selected commit and ignores the rest.

## Best solution

The `workbench.scm.action.graph.viewChanges` action in `scmHistoryViewPane.ts` needs to handle multi-selection properly — either computing a combined diff range across the selected commits, or showing a meaningful error/fallback when multiple non-adjacent commits are selected. The commenter's observation about selection order producing empty diffs (reversed parent/child) should also be addressed.
