# Issue #224043 — "Notebook Find Filters" button shows in search when no notebook is active

[https://github.com/microsoft/vscode/issues/224043](https://github.com/microsoft/vscode/issues/224043)

## Summary

Still reproduces on main: the notebook find filters button visibility is based on whether any notebook editor is open (including non-visible ones), not whether one is actually visible. An open PR (#309137) exists to fix this.

## Change summary

The 'Notebook Find Filters' toggle in search shows when a notebook is open but not visible, confusing users who don't see any notebook.

## Best solution

Land or finish the open PR https://github.com/microsoft/vscode/pull/309137, which changes the visibility check from any-open to visible-notebooks-only. The fix is in `src/vs/workbench/contrib/search/browser/searchWidget.ts` — `_hasNotebookOpen()` needs to check visible editors rather than all open editors.
