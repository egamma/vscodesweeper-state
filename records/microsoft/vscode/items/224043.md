# Issue #224043 — "Notebook Find Filters" button shows in search when no notebook is active

[https://github.com/microsoft/vscode/issues/224043](https://github.com/microsoft/vscode/issues/224043)

## Summary

Still reproduces on main: `_hasNotebookOpen()` in searchWidget.ts checks whether any notebook editor is open (including background tabs), not whether one is visible — so the notebook filter button appears even when no notebook is actively visible. An open community PR exists to fix this.

## Change summary

The 'Notebook Find Filters' toggle in the search widget appears when a notebook is open in any tab, even if it is not visible to the user.

## Best solution

Review and land https://github.com/microsoft/vscode/pull/309137 (or a similar fix) that changes `_hasNotebookOpen()` in `src/vs/workbench/contrib/search/browser/searchWidget.ts` to check for *visible* notebook editors rather than all open editors. The fix should use `editorService.visibleEditors` (or an equivalent visibility check) instead of `editorService.editors`.
