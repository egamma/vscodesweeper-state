# Issue #275003 — SCM hover behavior isn't instant if the up/down change text is hovered

[https://github.com/microsoft/vscode/issues/275003](https://github.com/microsoft/vscode/issues/275003)

## Summary

Still reproduces on main: the SCM repository toolbar is created without a hoverDelegate, causing non-instant hover transitions on the up/down change counts. The broader migration tracked by #270314 is still open and unresolved.

## Change summary

SCM hover on incoming/outgoing change counts is not instant like neighboring toolbar items because the WorkbenchToolBar lacks a hoverDelegate.

## Best solution

Either pass a hoverDelegate to the WorkbenchToolBar in scmRepositoryRenderer.ts as a short-term fix, or wait for the broader setupManagedHover removal (https://github.com/microsoft/vscode/issues/270314) to resolve this systemically. The SCM area owner should decide which path to take.
