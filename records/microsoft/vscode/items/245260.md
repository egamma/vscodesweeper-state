# Issue #245260 — ShellIntegration-bash is not properly escaping windows paths

[https://github.com/microsoft/vscode/issues/245260](https://github.com/microsoft/vscode/issues/245260)

## Summary

Still reproduces on main: the `echo -e` on lines 76/85/94 of shellIntegration-bash.sh interprets escape sequences in Windows paths (e.g. `\t` → tab, `\n` → newline). The attempted fix (PR #245264) was reverted due to side effects, so the bug persists.

## Change summary

Windows paths containing backslash sequences like `\t` or `\n` are corrupted by `echo -e` in the shell integration env-var collection code when using Git Bash.

## Best solution

Replace the `echo -e "$ITEM"` calls on lines 76, 85, 94 with a mechanism that does not interpret escape sequences — e.g. `printf '%s' "$ITEM"` or bash parameter expansion (`${ITEM#*=}`) — while avoiding the side effects that caused the original fix (#245264) to be reverted.
