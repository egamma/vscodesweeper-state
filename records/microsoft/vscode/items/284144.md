# Issue #284144 — Window Maximizing is not working since `1.107.0`

[https://github.com/microsoft/vscode/issues/284144](https://github.com/microsoft/vscode/issues/284144)

## Summary

This is a legitimate Electron 39 regression affecting macOS window maximize behavior with third-party tiling tools. It is already correctly labeled as upstream/electron/regression/electron-39-update and should stay open until the upstream Electron/Chromium fix lands.

## Change summary

Third-party macOS window tiling tools (e.g. BetterSnapTool) can no longer maximize the VS Code window, regressed in 1.107.0 with the Electron 39 update.

## Best solution

Track this as an Electron/Chromium upstream regression. The fix will come from a future Electron update that restores proper macOS window management API behavior for third-party tools. Other electron-39-update maximize issues exist (#284296 on Windows, #282956 white bar) but describe different symptoms on different platforms, so this macOS-specific report is distinct.
