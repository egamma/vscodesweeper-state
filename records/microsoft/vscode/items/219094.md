# Issue #219094 — Extension Profiling Path PII-Masking Hides Relevant Information

[https://github.com/microsoft/vscode/issues/219094](https://github.com/microsoft/vscode/issues/219094)

## Summary

Still unfixed on main: the PII-masking in `rewriteAbsolutePaths` strips paths to basename only, losing the extension identifier. The request for a setting or smarter masking remains valid.

## Change summary

Extension host CPU profile PII masking removes the extension name from paths, making profiling output useless for identifying which extension causes slowdowns.

## Best solution

Improve `rewriteAbsolutePaths` in `src/vs/platform/profiling/common/profiling.ts` to preserve the extension identifier (e.g. keep the `publisher.name` segment from the extensions path) while still stripping the user-home prefix. Alternatively, add a hidden setting to disable masking entirely. The comment suggestion of keeping the extension name is the cleanest approach.
