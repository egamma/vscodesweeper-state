# Issue #219094 — Extension Profiling Path PII-Masking Hides Relevant Information

[https://github.com/microsoft/vscode/issues/219094](https://github.com/microsoft/vscode/issues/219094)

## Summary

Keep open: this is a MEMBER-authored feature request, which requires explicit maintainer judgment, and current main still appears to mask extension profile paths down to only the basename. The comments add user impact and a workaround, but not a resolution.

## Change summary

Extension host CPU profiles saved from built VS Code replace absolute script URLs with `piiRemoved/<basename>`, hiding which extension owns common files like `extension.js`.

## Best solution

Preserve enough non-PII extension identity in saved extension host profiles, likely by refining `Utils.rewriteAbsolutePaths` in `src/vs/platform/profiling/common/profiling.ts` and its built-save use in `src/vs/workbench/contrib/extensions/electron-browser/runtimeExtensionsEditor.ts`, while keeping user path components redacted.
