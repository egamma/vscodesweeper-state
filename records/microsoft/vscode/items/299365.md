# Issue #299365 — "$$n" doesn't replace by literal "$n" in Regex replace (Workspace search only)

[https://github.com/microsoft/vscode/issues/299365](https://github.com/microsoft/vscode/issues/299365)

## Summary

Confirmed bug on current main: the workspace search `parseReplaceString` has no handler for `$$`, so it never collapses to a literal `$`. An open PR (#326451) addresses this but has not merged.

## Change summary

Workspace search regex replace treats `$$0` as literal `$$0` instead of collapsing `$$` to `$` like the in-editor replace does.

## Best solution

Land the open PR https://github.com/microsoft/vscode/pull/326451 which adds a `$$` case to `parseReplaceString` in `src/vs/workbench/services/search/common/replace.ts`, bringing workspace search in line with the editor's `replacePattern.ts` behavior.
