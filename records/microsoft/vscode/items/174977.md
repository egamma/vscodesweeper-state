# Issue #174977 — Add option to not focus next search result

[https://github.com/microsoft/vscode/issues/174977](https://github.com/microsoft/vscode/issues/174977)

## Summary

Valid feature request from a team member; the behavior (auto-focusing next search result after replace) is still present on main with no option to disable it. Keep open for maintainer judgment.

## Change summary

After replacing a search result, the editor auto-navigates to the next match (potentially in a different file), which is disorienting — request an option to disable this.

## Best solution

Add a setting (e.g. `search.focusNextAfterReplace`) that, when disabled, skips the reveal/focus/open logic in `performReplace` (in `src/vs/workbench/contrib/search/browser/searchActionsRemoveReplace.ts` around line 305). The tree focus update in the search view could still happen without opening the next file in the editor.
