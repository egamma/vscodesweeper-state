# Issue #174977 — Add option to not focus next search result

[https://github.com/microsoft/vscode/issues/174977](https://github.com/microsoft/vscode/issues/174977)

## Summary

Team-authored feature request, still unimplemented on main. The `performReplace` function in search unconditionally focuses and opens the next match after replacing — no setting or option exists to suppress this behavior.

## Change summary

After accepting a search-and-replace result, VS Code auto-focuses the next match and may swap the active editor, which can be disorienting.

## Best solution

Add a setting (e.g. `search.focusNextResultAfterReplace`) that, when disabled, skips the focus/reveal/open logic in `performReplace` (lines ~300–319 of `src/vs/workbench/contrib/search/browser/searchActionsRemoveReplace.ts`). The replace still removes the matched element; only the auto-navigation to the next result is suppressed.
