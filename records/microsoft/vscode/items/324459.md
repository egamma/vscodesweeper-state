# Issue #324459 — Chat asks to read a file after I've explicitly attached a chat session

[https://github.com/microsoft/vscode/issues/324459](https://github.com/microsoft/vscode/issues/324459)

## Summary

Team-authored issue (MEMBER) — must keep open. Reports that attaching a prior chat session still prompts for file-read permission with an opaque file path instead of either granting implicit access or referencing 'the attached session'. Filed during TPI verification of #324418.

## Change summary

When a chat session is explicitly attached as context, the agent still asks permission to read a file (using an internal path) rather than having implicit access or describing it as the attached session.

## Best solution

Two improvements: (1) granting implicit read permission for explicitly-attached session content so the agent doesn't prompt, or (2) if prompting is required, surfacing a user-friendly description ('the session you attached') instead of an opaque file path. Likely involves the agent host session reference attachment handling in `src/vs/workbench/contrib/chat/browser/agentSessions/agentHost/agentHostSessionReferenceAttachment.ts` and possibly the permissions/tools layer.
