# Issue #129980 — the default value of `editor.autoClosingDelete` should be "always"

[https://github.com/microsoft/vscode/issues/129980](https://github.com/microsoft/vscode/issues/129980)

## Summary

Keep open as an editor default-setting discussion. The current main branch still defaults `editor.autoClosingDelete` to `auto`, and the maintainer response explicitly framed this as feedback to wait on rather than a bug or duplicate.

## Change summary

Change the default value of `editor.autoClosingDelete` from `auto` to `always` so paired quote/bracket deletion behaves consistently even when the pair is not tracked as auto-inserted.

## Best solution

The editor team should make an explicit product decision for `editor.autoClosingDelete` in `src/vs/editor/common/config/editorOptions.ts`: either keep `auto` as the less-aggressive default or change the registered default to `always` with release-note coverage for the behavior change.
