# Issue #306984 — Starting from version 1.108.2 and later, the focus outline no longer appears when clicking with the mouse or when the focus is placed on "Explore."

[https://github.com/microsoft/vscode/issues/306984](https://github.com/microsoft/vscode/issues/306984)

## Summary

The focus outline regression is caused by commit d4a8aec2b7c which switched from `:focus` to `:focus-visible` in the list widget. This is a valid accessibility concern — a team member is already engaged and the reporter confirmed they rely on keyboard-driven focus indicators.

## Change summary

Focus outline no longer appears on Explorer items when clicked or focused via keyboard shortcuts, after the switch from `:focus` to `:focus-visible` in list widget styling.

## Best solution

The team should decide whether `:focus-visible` adequately covers all keyboard navigation scenarios, or whether certain focus paths (e.g. command-initiated focus) need the outline restored. The fix likely lives in `src/vs/base/browser/ui/list/listWidget.ts` — potentially adding back `:focus` selectors for accessibility mode or ensuring programmatic focus triggers `:focus-visible`.
