# Issue #310739 — Custom added model will still show in model picker

[https://github.com/microsoft/vscode/issues/310739](https://github.com/microsoft/vscode/issues/310739)

## Summary

Still reproduces on main: removed custom models persist in the model picker because the cached-models storage (`chat.cachedLanguageModels.v2`) and recently-used storage (`chatModelRecentlyUsed`) are not cleaned up when a model/provider is removed. The `mergeModelsWithCache` function retains cached models for vendors that haven't resolved yet, and the Copilot vendor is explicitly exempted from eviction.

## Change summary

After removing a custom-added model (e.g. via OpenRouter BYOK), the model continues to appear in the chat model picker.

## Best solution

When a model or provider configuration is removed, proactively purge matching entries from both the `chatModelRecentlyUsed` and `chat.cachedLanguageModels.v2` storage keys. The eviction logic in `mergeModelsWithCache` should also handle the case where a vendor's models were removed by the user (not just where the vendor resolved to empty).
