# Issue #246388 — Git: deletion marker in the gutter disappears when folding adjacent regions

[https://github.com/microsoft/vscode/issues/246388](https://github.com/microsoft/vscode/issues/246388)

## Summary

Still reproduces on main. The deletion marker decoration is placed on the last line of the preceding folding region; when that region is folded the decoration is hidden because it lives on a now-invisible line.

## Change summary

Git deletion gutter marker disappears when an adjacent folding region above it is collapsed.

## Best solution

When a ChangeType.Delete decoration falls on a line that is the last line of a foldable region, the decoration should be moved to the next visible line (or the folded placeholder line). The fix likely belongs in `src/vs/workbench/contrib/scm/browser/quickDiffDecorator.ts` around line 171–180 where the delete decoration range is computed, possibly consulting the folding model to adjust placement, or by setting a decoration option (like `collapseOnReplaceEdit` or an equivalent) that keeps gutter decorations visible on fold boundaries.
