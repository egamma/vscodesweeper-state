# Issue #311557 — [Error] potential listener LEAK — terminal/decorationAddon

[https://github.com/microsoft/vscode/issues/311557](https://github.com/microsoft/vscode/issues/311557)

## Summary

Still reproduces on main: the fix in 8379a047a31 addressed listener leaks in XtermTerminal for detached terminals but did not address the `lifecycleService.onWillShutdown` listener registered unconditionally in DecorationAddon (line 89). Each detached terminal still accumulates a listener on this global singleton.

## Change summary

Creating many detached terminals (e.g. in the Agents product) causes a listener leak on `lifecycleService.onWillShutdown` because each `DecorationAddon` instance registers its own listener unconditionally.

## Best solution

Either skip the `onWillShutdown` listener in `DecorationAddon` when created for a detached terminal (similar to how XtermTerminal skips other global listeners), or centralize the shutdown cleanup in the detached-terminal coordination logic in `terminalService.ts`. The `DecorationAddon` constructor needs a `detached` option to conditionally skip the listener, or disposable cleanup should be handled at the `DetachedTerminal` level instead.
