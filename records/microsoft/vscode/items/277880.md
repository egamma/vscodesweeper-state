# Issue #277880 — editor.find.loop set to false not respected in larger files

[https://github.com/microsoft/vscode/issues/277880](https://github.com/microsoft/vscode/issues/277880)

## Summary

Confirmed on main: `canNavigateInLoop()` in findState.ts unconditionally returns true when matches >= MATCHES_LIMIT (19999), overriding the user's `editor.find.loop` setting. An open PR #307998 addresses this but has not merged.

## Change summary

The `editor.find.loop` setting is ignored when the number of find matches in a file reaches or exceeds the internal MATCHES_LIMIT (19999), causing find navigation to always wrap.

## Best solution

Merge the fix from PR #307998 (or equivalent): remove the `matchesCount >= MATCHES_LIMIT` override from `canNavigateInLoop()` and add wrap-around detection in the large-file navigation paths in `findModel.ts`.
