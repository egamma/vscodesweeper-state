# Issue #277880 — editor.find.loop set to false not respected in larger files

[https://github.com/microsoft/vscode/issues/277880](https://github.com/microsoft/vscode/issues/277880)

## Summary

Confirmed bug: `canNavigateInLoop()` in findState.ts unconditionally returns `true` when matches >= MATCHES_LIMIT (19999), ignoring the user's `editor.find.loop: false` setting. An open community PR (#307998) addresses this but has not yet been reviewed or merged.

## Change summary

When a file has >= 19999 find matches, `editor.find.loop` set to `false` is ignored and search wraps around.

## Best solution

Review and land https://github.com/microsoft/vscode/pull/307998 (or an equivalent fix): remove the `MATCHES_LIMIT` override from `canNavigateInLoop()` in `src/vs/editor/contrib/find/browser/findState.ts` and ensure the large-file navigation paths in `findModel.ts` respect the loop setting.
