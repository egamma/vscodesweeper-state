## Summary

Plausible bug: plugin auto-update likely triggers onDidChangeAvailability which refreshes the session-type picker without preserving the currently selected custom agent. No fix found on main. More info needed on exact reproduction timing.

## Change summary

Custom agent selection in the Copilot Chat picker resets to default 'Agent' when plugin auto-update fires, because the availability-change event refreshes the list without preserving the active selection.

## Best solution

The session-target picker's `_updateAgentSessionItems()` (triggered by `onDidChangeAvailability`) should preserve the currently selected session type if it remains valid after the refresh. The fix likely belongs in `src/vs/workbench/contrib/chat/browser/widget/input/sessionTargetPickerActionItem.ts` or in the chat sessions contribution to avoid firing availability changes for no-op re-registrations during plugin updates.
