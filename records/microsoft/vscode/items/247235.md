# Issue #247235 — Deleting JSX tag name and then typing a different tag name doesn't update closing tag

[https://github.com/microsoft/vscode/issues/247235](https://github.com/microsoft/vscode/issues/247235)

## Summary

Still reproduces on main. The core issue is that when all tag-name characters are deleted, `updateRanges` re-queries the linked editing provider which returns nothing (syntax is broken), causing decorations to be cleared. PR #242993 (fbefb06cae2) fixed a related desync but not this specific empty-tag-name scenario.

## Change summary

Deleting an entire JSX/HTML tag name and retyping a new one fails to update the closing tag because linked editing decorations are lost when the provider can't parse the temporarily invalid syntax.

## Best solution

In `linkedEditing.ts`, when the reference range collapses to zero-width (all characters deleted) and the cursor is still at that collapsed position, preserve the existing decoration pair instead of re-querying the provider. This keeps the sync active so subsequent typing at the same position still propagates to the mirror range. The fix is localized to `src/vs/editor/contrib/linkedEditing/browser/linkedEditing.ts`.
