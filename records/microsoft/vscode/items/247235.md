# Issue #247235 — Deleting JSX tag name and then typing a different tag name doesn't update closing tag

[https://github.com/microsoft/vscode/issues/247235](https://github.com/microsoft/vscode/issues/247235)

## Summary

Still reproducible on main — when a tag name is fully deleted, the linked editing provider returns no ranges (since there's no word at cursor), clearing the session. An open PR (#304957) addresses this but has not been merged.

## Change summary

Deleting an entire JSX/HTML tag name (especially when the tag has attributes) breaks the linked editing session, so typing a new name no longer updates the closing tag.

## Best solution

Preserve the linked editing decoration ranges when the reference range collapses to empty, so that subsequent typing re-engages the sync without requiring a fresh provider response. The open PR https://github.com/microsoft/vscode/pull/304957 targets exactly this in `src/vs/editor/contrib/linkedEditing/browser/linkedEditing.ts`.
