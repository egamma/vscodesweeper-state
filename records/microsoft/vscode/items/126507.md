# Issue #126507 — [scss] map value override => identifier expected

[https://github.com/microsoft/vscode/issues/126507](https://github.com/microsoft/vscode/issues/126507)

## Summary

Still reproduces on main — the vscode-css-languageservice SCSS parser does not handle namespaced variable assignments (e.g. `test.$var: value`). The issue is on the 'On Deck' milestone and remains valid.

## Change summary

SCSS parser incorrectly reports 'identifier expected' for valid namespaced variable overrides like `test.$test: (…)`.

## Best solution

Fix the SCSS parser in the upstream `vscode-css-languageservice` (microsoft/vscode-css-languageservice) to recognize `namespace.$variable: value` as a valid assignment. The parser currently fails to handle the dot-prefixed identifier after a namespace reference in assignment position.
