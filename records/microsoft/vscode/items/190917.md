# Issue #190917 — Hot reload in extension development - automatically reload the Extension Development Host window

[https://github.com/microsoft/vscode/issues/190917](https://github.com/microsoft/vscode/issues/190917)

## Summary

Valid, well-supported feature request (80 upvotes) for automatic hot reload in extension development. Not yet implemented on main — the existing hot reload infrastructure is for VS Code's own internal dev loop, not for extension authors. Keeping open in Backlog.

## Change summary

Automatically reload the Extension Development Host window when extension source files change, removing the manual Ctrl+Shift+F5 step.

## Best solution

Add a file-watcher-driven automatic restart of the extension host during extension development, likely in the debug/extension-host launch flow. Could also be an opt-in launch.json option (e.g. `watchFiles` trigger). The webview hot-reload part may be better addressed via extension-side tooling (Vite HMR) rather than core.
