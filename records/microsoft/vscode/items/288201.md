# Issue #288201 — "add ___ to chat" doesn't work when chat hasn't been previously opened

[https://github.com/microsoft/vscode/issues/288201](https://github.com/microsoft/vscode/issues/288201)

## Summary

Team-authored bug (MEMBER) — must keep open per policy. The report describes a real race condition where chat attachments are lost when chat hasn't been previously opened in the session.

## Change summary

Using 'add to chat' before the chat panel has been opened in a session silently drops the attachment due to a model synchronization issue.

## Best solution

Fix the initialization/synchronization order so that attachments added before the chat widget is fully initialized are preserved when the model syncs. The author suspects this is related to https://github.com/microsoft/vscode/issues/278683 and the model synchronization clearing pre-added attachments.
