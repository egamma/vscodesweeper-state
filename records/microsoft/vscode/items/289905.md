# Issue #289905 — Allow the user to open the PR from the agent list entry context menu

[https://github.com/microsoft/vscode/issues/289905](https://github.com/microsoft/vscode/issues/289905)

## Summary

Valid feature request to improve the agent session list context menu by adding PR-specific actions (Open PR in VS Code, Open PR in External Browser) and grouping them with Close Pull Request. None of this is implemented on current main — the OpenPullRequestAction only appears in the session header meta row, not in the session list item context menu. Keeping open as a legitimate UX improvement for the Agents window.

## Change summary

Add PR-related actions (Open PR, Open PR in External Browser, Close Pull Request) to the agent session list item context menu, grouped separately from session-related actions.

## Best solution

Add the OpenPullRequestAction (and a new 'Open PR in External Browser' variant) to SessionItemContextMenuId in a dedicated menu group alongside a Close Pull Request action. The relevant files are `src/vs/sessions/contrib/github/browser/pullRequestActions.ts` (existing Open PR action to extend with new menu registrations) and `src/vs/sessions/contrib/sessions/browser/views/sessionsViewActions.ts` (session list context menu actions). The 'Open in Browser' label should be disambiguated to clarify it opens the PR on github.com, not the chat session.
