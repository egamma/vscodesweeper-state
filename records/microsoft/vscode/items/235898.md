# Issue #235898 — Git - add a delay before showing git blame information

[https://github.com/microsoft/vscode/issues/235898](https://github.com/microsoft/vscode/issues/235898)

## Summary

Valid feature request for a configurable delay before showing git blame inline decoration. No such setting exists on current main — the decoration renders immediately on cursor move, throttled only by async serialization, not by a user-facing delay.

## Change summary

Add a configurable delay before the git blame editor decoration appears after cursor movement.

## Best solution

Add a `git.blame.editorDecoration.delay` setting (in ms, default 0) and debounce the `_updateTextEditorBlameInformation` call in `extensions/git/src/blame.ts` by that value on selection-change events. This is a small, self-contained change in the git extension.
