# Issue #288048 — Live Subagents Panel

[https://github.com/microsoft/vscode/issues/288048](https://github.com/microsoft/vscode/issues/288048)

## Summary

Valid feature request with 28 upvotes, already promoted to Backlog. Current main has partial subagent visibility (inline subagent content parts, a running-subagents status chip, and an open-subagent-chat action) but no dedicated live panel with a full hierarchical tree view, interaction affordances, or per-agent context/skill/file-access summaries as described.

## Change summary

Add a dedicated live panel showing currently active subagents as a tree with their status, output, loaded context/skills, and files read/updated, with the ability to interact with agents that need attention.

## Best solution

The existing `SessionRunningSubagentsControl` (status chip) and `chatSubagentContentPart` (inline rendering) provide partial visibility into subagent activity, but a richer dedicated panel — likely as a new view contribution in `src/vs/sessions/contrib/` or `src/vs/workbench/contrib/chat/browser/` — would require product design decisions around scope, tree depth, interaction model, and how much internal agent state to expose. This aligns with the Agents window investment and should be evaluated by the chat/agents team.

## Evidence

- **inline subagent rendering:** src/vs/workbench/contrib/chat/browser/widget/chatContentParts/chatSubagentContentPart.ts renders inline subagent blocks within the chat transcript with collapsible tool invocations.
- **open subagent action:** src/vs/sessions/contrib/providers/agentHost/browser/openSubagentChat.ts contributes an action to open a subagent's chat as a peer tab in the Agents window.
- **debug flow graph:** src/vs/workbench/contrib/chat/browser/chatDebug/chatDebugFlowGraph.ts and chatDebugLogsView.ts provide a debug-oriented flow/tree visualization of chat events, but this is a developer debug tool, not a user-facing live subagents panel.
