# Issue #289899 — Don't ask to enable workspace trust in system folders and temp directories

[https://github.com/microsoft/vscode/issues/289899](https://github.com/microsoft/vscode/issues/289899)

## Summary

Valid feature request from a team member (Tyriar) to suppress the workspace trust dialog for system folders and temp directories where the trust decision is meaningless. Not yet implemented on main — the workspace trust service has no path-based auto-trust logic. Related to https://github.com/microsoft/vscode/issues/291933 but distinct in scope (suppressing the dialog vs. pre-trusting home folders).

## Change summary

Suppress the workspace trust dialog for system folders (e.g. `/`, `~/`, `C:\`) and temp directories (e.g. `/tmp/`) where the trust prompt adds friction without value.

## Best solution

Add path-based heuristics to the workspace trust dialog flow in `src/vs/workbench/services/workspaces/common/workspaceTrust.ts` or `src/vs/workbench/contrib/workspace/browser/workspace.contribution.ts` to auto-trust or skip the prompt for well-known system roots and temp directories. Consider consolidating with https://github.com/microsoft/vscode/issues/291933 (default trusted folders) since they share implementation surface.
