# Issue #283650 — Improve agent behavior when certain tools are disabled

[https://github.com/microsoft/vscode/issues/283650](https://github.com/microsoft/vscode/issues/283650)

## Summary

Team-authored feature request to improve agent prompting when tools are disabled — the agent should gracefully inform the user instead of looping or invoking nonexistent tools. Keeping open for maintainer judgment.

## Change summary

When a tool is disabled for the agent, the agent attempts random actions instead of stopping and telling the user it cannot complete the task.

## Best solution

Revise the agent's system prompt to explicitly instruct it to stop and inform the user when a required tool is disabled, and write evals to verify the behavior. The work likely lives in the Copilot extension's prompt-construction and tool-registration layers.
