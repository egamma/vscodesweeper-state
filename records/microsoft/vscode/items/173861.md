# Issue #173861 — Proxy agent patch prevents extensions from re-using HTTP connections

[https://github.com/microsoft/vscode/issues/173861](https://github.com/microsoft/vscode/issues/173861)

## Summary

Still reproducible on main: the proxy agent's keepAlive passthrough is explicitly commented out in the current @vscode/proxy-agent code due to the regression in #228872, so extensions still cannot reuse HTTP connections through the proxy layer. The bug remains valid and needs a proper fix.

## Change summary

VS Code's proxy agent patch replaces extensions' HTTP agents but does not pass through keepAlive, preventing connection reuse and degrading performance.

## Best solution

Re-implement keepAlive passthrough in @vscode/proxy-agent in a way that avoids the regression that caused the revert (https://github.com/microsoft/vscode/issues/228872). The fix lives in the `microsoft/vscode-proxy-agent` repo — the commented-out line at `node_modules/@vscode/proxy-agent/out/index.js:352` shows where it needs to go. The regression involved fetch failures, so the re-implementation likely needs more careful socket lifecycle handling.
