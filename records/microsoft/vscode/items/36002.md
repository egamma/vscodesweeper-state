# Issue #36002 — [folding] allow defining folding regions in user settings and/or extensions

[https://github.com/microsoft/vscode/issues/36002](https://github.com/microsoft/vscode/issues/36002)

## Summary

Feature request remains valid — folding markers are still only configurable through language-configuration.json files provided by extensions, not via user settings. Extensions like zokugun.explicit-folding fill the gap but a native setting doesn't exist yet.

## Change summary

Allow users to define custom folding region markers (start/end patterns) in user settings per language, without requiring a language extension to provide them.

## Best solution

Expose folding marker configuration as a user-facing setting (e.g. `editor.foldingMarkers` keyed by language ID) in `src/vs/editor/contrib/folding/`, feeding into the existing `FoldingMarkers` infrastructure in `languageConfiguration.ts`. This would let users override or supplement extension-provided folding markers without needing a dedicated extension.
