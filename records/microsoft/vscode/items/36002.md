# Issue #36002 — [folding] allow defining folding regions in user settings and/or extensions

[https://github.com/microsoft/vscode/issues/36002](https://github.com/microsoft/vscode/issues/36002)

## Summary

Valid, well-supported feature request (53 upvotes) for user-configurable folding markers. Still unimplemented on main — language-configuration folding markers exist but are not exposed in user settings. Belongs in Backlog with continued community interest.

## Change summary

Allow users to define custom folding region markers (open/close tags) per language via user settings, without depending on language extensions to ship them.

## Best solution

Expose folding marker configuration in user settings (alongside the existing language-configuration mechanism), potentially as part of the broader effort in https://github.com/microsoft/vscode/issues/102574 to allow user settings for language configurations. Extensions like zokugun.explicit-folding provide workarounds but native support remains the ask.
