# Issue #319728 — git stopped working immediatetely after opening vscode or reloading

[https://github.com/microsoft/vscode/issues/319728](https://github.com/microsoft/vscode/issues/319728)

## Summary

This reports git operations queued behind a blocked extension host at startup ('Activating Extensions...' in status bar). The root cause — Copilot Chat blocking the extension host for 60–135s — was tracked and fixed in #321901 (verified) and the related #319710 (completed). The reporter's version (1.122.1) predates the fix.

## Change summary

Git stage/unstage/discard actions are non-functional for up to several minutes after opening or reloading VS Code in WSL, with status bar showing 'Activating Extensions...'.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/321901 which tracked the extension host unresponsiveness regression and was verified fixed. The reporter should update to 1.125.0+ where the fix shipped.
