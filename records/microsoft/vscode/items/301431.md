# Issue #301431 — Improve Chat Read-Aloud (Text-to-Speech) Controls in VS Code Chat: Start Position, Navigation, Speed Control, and Visible-Text-Only Reading

[https://github.com/microsoft/vscode/issues/301431](https://github.com/microsoft/vscode/issues/301431)

## Summary

Legitimate accessibility feature request for richer TTS playback controls in chat. Currently in the Backlog Candidates community voting process (2 of 20 upvotes, ~50 days remaining). The existing implementation supports start/stop and code-block filtering but lacks pause/resume, positional navigation, speed control, and selection-based reading. Keeping open for the community process.

## Change summary

Add granular playback controls (pause, resume, skip, start-from-selection), speed adjustment, and visible-text-only filtering to the chat read-aloud (TTS) feature.

## Best solution

If the community process promotes this, the work belongs in `src/vs/workbench/contrib/chat/electron-browser/actions/voiceChatActions.ts` (the `ChatSynthesizerSessions` class and related actions) and `src/vs/workbench/contrib/speech/common/speechService.ts` (the `ITextToSpeechSession` interface). The request bundles several independent improvements — pause/resume, positional playback, speed presets, and filtering of hidden reasoning content — that could be scoped individually. Speed control and pause/resume are the most impactful and self-contained; filtering hidden content (reasoning traces, subagent output) requires coordination with the chat response model to distinguish visible from internal parts.
