# Issue #190549 — Incorrect indentation when passing snippet string as argument in inline suggestions

[https://github.com/microsoft/vscode/issues/190549](https://github.com/microsoft/vscode/issues/190549)

## Summary

Still reproduces on main: when inline completions accept a snippet (snippetInfo path), the SnippetController2.insert call uses default adjustWhitespace:true, causing re-indentation of multi-line snippets with tab stops. Team-authored issue; keeping open.

## Change summary

Inline suggestions with snippet strings (containing tab stops) get extra indentation on commit because adjustWhitespace defaults to true during snippet insertion.

## Best solution

Pass `adjustWhitespace: false` when inserting snippets from inline completions (line 961 of inlineCompletionsModel.ts), since the inline suggestion text is already formatted for its target position. Alternatively, pre-adjust the snippet before display so the WYSIWYG rendering matches the committed result.
