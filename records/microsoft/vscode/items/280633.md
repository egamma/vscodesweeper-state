## Summary

Team-authored issue (MEMBER) — must keep open. Partial progress: UBB quota messages now include the reset date, but the non-UBB quota paths (e.g. free-tier `chatError.quota.free`) still omit it, so the original request remains partially unaddressed.

## Change summary

Include the quota reset date in the chat error message shown when a user exceeds their chat limit, rather than just saying 'wait for your allowance to renew.'

## Best solution

Extend the non-UBB branches of `getQuotaMessageForPlan` in `src/vs/workbench/contrib/chat/common/chatErrorMessages.ts` (lines 209–223) to accept and display the `quotaResetDate` the same way the UBB branches already do. The `quotaResetDate` field already exists in the payload interface, so the plumbing is in place — the remaining work is adding date-aware localized strings for each non-UBB plan tier.
