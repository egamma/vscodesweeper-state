# Issue #214547 — Windows: WSL prompt option only shows in terminal add menu if you add another terminal 

[https://github.com/microsoft/vscode/issues/214547](https://github.com/microsoft/vscode/issues/214547)

## Summary

Still reproduces on main: WSL profile detection uses a 1-second timeout on `wsl.exe -l -q` which can fail on a cold boot before WSL is initialized; the profile only appears after a subsequent refresh triggered by opening another terminal.

## Change summary

WSL terminal profile does not appear in the new-terminal dropdown until a different terminal is opened first, because the initial detection times out before WSL responds.

## Best solution

Increase the `wsl.exe -l -q` timeout (or add a retry/backoff) in `getWslProfiles` in `src/vs/platform/terminal/node/terminalProfiles.ts`, or schedule a delayed re-detection if the initial WSL probe fails or times out.
