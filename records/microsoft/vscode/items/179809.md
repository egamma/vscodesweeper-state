# Issue #179809 — Git - merge is canceled when removing an untracked file

[https://github.com/microsoft/vscode/issues/179809](https://github.com/microsoft/vscode/issues/179809)

## Summary

Fixed on main and shipped in 1.99.0. The root cause was that `repository.checkout('', [])` was called with an empty paths array when only untracked files were discarded, which ran a bare `git checkout -q` and canceled the merge state. The fix guards checkout (and clean/submodule-update) with length checks.

## Change summary

Discarding an untracked file during a Git merge via VS Code's Source Control view cancels the merge state by running a bare `git checkout -q`.

## Best solution

Keep the shipped fix in commit 3ea87919641d (1.99.0); no further change needed.
