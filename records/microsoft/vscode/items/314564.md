# Issue #314564 — Copilot

[https://github.com/microsoft/vscode/issues/314564](https://github.com/microsoft/vscode/issues/314564)

## Summary

The terminal command editor is intentionally read-only when presentationOverrides is active (e.g., when the displayed command is a simplified version with the cd prefix stripped). This is by design — editing a presentation that differs from the actual executed command would be misleading.

## Change summary

User wants to edit terminal commands in the Copilot agent mode confirmation box, but the editor is read-only when commands use presentationOverrides (simplified display).

## Best solution

Keep the current as-designed behavior. The read-only state prevents editing a presentation-only view that differs from the actual command. If users need to edit, they can disable the sandbox setting so commands run without presentationOverrides; alternatively, a future enhancement could allow editing the underlying command while showing the simplified presentation, but that would be a separate feature request.
