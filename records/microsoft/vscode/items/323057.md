# Issue #323057 — Searching files by exact name is way too slow

[https://github.com/microsoft/vscode/issues/323057](https://github.com/microsoft/vscode/issues/323057)

## Summary

Valid enhancement request: when the user types an exact relative path in Quick Open, the result should surface immediately (as a 'fast pick') rather than waiting for the full file search index. The code already resolves relative paths via stat() in `getRelativePathFileResults`, but it's grouped in the slow-picks pipeline. No fix or duplicate found on main.

## Change summary

Quick Open (Cmd+P) should immediately display an exact relative path match while the full file index loads, rather than waiting for the entire search to complete.

## Best solution

Move the `getRelativePathFileResults` stat-based check (anythingQuickAccess.ts line ~769) into the fast-picks path of `doGetPicks`, so exact relative path matches appear instantly alongside editor history while the full file search runs in the background.
