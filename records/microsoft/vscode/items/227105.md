# Issue #227105 — Editor GPU: Make first glyph in first page 0x0 without wasting space in the texture

[https://github.com/microsoft/vscode/issues/227105](https://github.com/microsoft/vscode/issues/227105)

## Summary

Valid tech-debt item: the TODO and wasteful first-slab hack are still present on current main (line 92 of textureAtlas.ts). The issue correctly tracks the optimization work.

## Change summary

The first slab in the GPU texture atlas is entirely wasted because the null glyph reserves it for 0x0 glyphs instead of using a zero-size rectangle that doesn't consume slab space.

## Best solution

Implement a special-case glyph entry in the atlas allocator that points to a 0-width, 0-height rectangle without claiming a slab, then remove the null-rasterizer hack in `_initFirstPage()`. Work lives in `src/vs/editor/browser/gpu/atlas/textureAtlas.ts` and likely `textureAtlasPage.ts`.
