# Issue #169253 — Git - Untracked Folders

[https://github.com/microsoft/vscode/issues/169253](https://github.com/microsoft/vscode/issues/169253)

## Summary

Team-authored feature request (MEMBER). The proposed optimization — using plain `git status` for untracked folders then selectively expanding — has not been implemented on main. The code still unconditionally uses `-uall` or `-uno`. Keeping open per hard keep-open rule and because the perf gap remains real for large monorepos.

## Change summary

Optimize git status for large repos by showing untracked folders instead of always enumerating all untracked files with `-uall`.

## Best solution

Implement the two-pass approach from comment 2: run `git status` (default, shows folders), parse untracked folder entries, then selectively run `git status -uall <folder>` to expand them. The relevant code is in `extensions/git/src/git.ts` around line 2748 and the status parser in `extensions/git/src/repository.ts`. The earlier API concerns (comment 3) about an 'Untracked Folders' group may have been mitigated by subsequent SCM API improvements.
