# Issue #155196 — Git - Missing context menu command to stage deleted hunk in Git

[https://github.com/microsoft/vscode/issues/155196](https://github.com/microsoft/vscode/issues/155196)

## Summary

Still reproduces on main: the editor context menu's 'Stage Selected Ranges' command is gated on `isInDiffRightEditor`, so purely-deleted hunks (visible only on the left/original side) cannot be staged via context menu. The gutter stage button partially mitigates this, but the context menu gap remains.

## Change summary

Right-clicking a deleted hunk in the diff view's left (original) editor does not show 'Stage Selected Ranges' because the when-clause requires the right editor.

## Best solution

Extend the editor context menu contributions for `git.stageSelectedRanges` (and potentially `git.revertSelectedRanges`) to also appear when `isInDiffEditor && !isInDiffRightEditor && resourceScheme == git` (left side), or add a dedicated left-side staging flow. The `diffStageHunkOrSelection` path already handles the data model correctly via `originalWithModifiedChanges`; the fix is primarily about menu visibility and wiring the command to work from the original-side editor context.
