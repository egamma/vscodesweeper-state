# Issue #193268 — Tasks should add a new line before the VS Code format message if it's not at the start of a line

[https://github.com/microsoft/vscode/issues/193268](https://github.com/microsoft/vscode/issues/193268)

## Summary

Still reproduces on main: the `taskShellIntegrationWaitOnExitSequence` function writes its message without ensuring the cursor is at column 0, so after Ctrl+C the reuse message can appear mid-line. No fix commit found.

## Change summary

When a task is cancelled with Ctrl+C, the 'Terminal will be reused by tasks' message can appear on the same line as existing output instead of on a new line.

## Best solution

Add a `\r\n` prefix to the message in `taskShellIntegrationWaitOnExitSequence` (or use `formatMessageForTerminal` which already handles leading newlines) in `src/vs/workbench/contrib/tasks/browser/terminalTaskSystem.ts`.
