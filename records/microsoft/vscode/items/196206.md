# Issue #196206 — 'Disconnect' option doesn't show in Call-Stack Session in debugger

[https://github.com/microsoft/vscode/issues/196206](https://github.com/microsoft/vscode/issues/196206)

## Summary

Fixed on main and shipped: commit 7b2cc73cf27 set `menuAsChild: false` on the call stack dropdown, preventing the menu from rendering inside the shadow DOM and appearing behind the editor. Shipped in 1.89.0.

## Change summary

The debug call stack session dropdown menu (Disconnect/Stop) renders behind the editor area due to shadow DOM z-index issues.

## Best solution

Keep the shipped fix (commit 7b2cc73cf27, 1.89.0); no further change needed. The `menuAsChild: false` option ensures the dropdown renders in the top-level DOM.
