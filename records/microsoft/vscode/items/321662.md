# Issue #321662 — MCP tool input validation rejects anyOf schemas - blocks Union-typed parameters

[https://github.com/microsoft/vscode/issues/321662](https://github.com/microsoft/vscode/issues/321662)

## Summary

Still reproduces on main: the Ajv instance in BaseToolsService uses `coerceTypes: true` without `allowUnionTypes`, and the recovery logic in `ajvValidateForTool` only handles string→object coercion — not `anyOf` aggregation failures. MCP tools with union-typed parameters are blocked.

## Change summary

MCP tool calls with `anyOf` JSON Schema parameters (e.g. Python Union types) are rejected client-side before reaching the server, because Ajv's `coerceTypes` mode corrupts data across `anyOf` branches.

## Best solution

Fix `ajvValidateForTool` in `extensions/copilot/src/extension/tools/common/toolsService.ts` to handle `anyOf` validation failures — either by adding `allowUnionTypes: true` to the Ajv instance, deep-cloning input before validation so coercion doesn't corrupt across branches, or adding an anyOf-aware recovery path similar to the existing string→object coercion.
