# Issue #209132 — QuickPick gets in a bad state if `canSelectMany` is set after it is visible and has items

[https://github.com/microsoft/vscode/issues/209132](https://github.com/microsoft/vscode/issues/209132)

## Summary

Still reproduces on main: when `canSelectMany` is set after items are visible, the list items already rendered don't get re-rendered with checkboxes because `setElements` is only called when `itemsUpdated` is true, leaving pre-existing rows without checkbox UI until they're scrolled out and recycled.

## Change summary

Setting `canSelectMany = true` on a QuickPick after items are already displayed leaves checkboxes missing on visible items until they scroll out of view and back.

## Best solution

In `QuickPick.update()`, detect when the `show-checkboxes` class toggled and force a re-render of existing list items (e.g. re-call `setElements` or invalidate the tree rows). The fix belongs in `src/vs/platform/quickinput/browser/quickInput.ts` around the existing `if (this.ui.container.classList.contains('show-checkboxes') !== !!this.canSelectMany)` guard at line 1113.
