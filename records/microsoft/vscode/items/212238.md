## Summary

Fixed on main: the hardcoded `pinned: true` in the `vscode.openWith` command was removed in commit ba026a3849d, so the caller's options (including `pinned`) are now respected.

`vscode.openWith` ignored the `pinned` option because it was hardcoded to `true` before the caller's options were spread.
