# Issue #140261 — terminal.integrated.defaultProfile and terminal.integrated.automationProfile behaves differently

[https://github.com/microsoft/vscode/issues/140261](https://github.com/microsoft/vscode/issues/140261)

## Summary

The core request — allowing automationProfile to accept a profile name string like defaultProfile does — remains unimplemented on main. The schema now supports args/env in inline objects, but the string-reference pattern (already implemented for agentHostProfile) has not been applied here. 28 upvotes and in the Backlog; keeping open.

## Change summary

Allow terminal.integrated.automationProfile to accept a profile name string (referencing terminal.integrated.profiles), matching the defaultProfile behavior.

## Best solution

Apply the same string-name resolution pattern used by agentHostProfile (terminalProfileResolverService.ts line 293) to _getUnresolvedAutomationShellProfile — accept a string, look it up in availableProfiles, and return the matching profile. Update the schema in terminalPlatformConfiguration.ts to allow type ['object', 'string', 'null'].
