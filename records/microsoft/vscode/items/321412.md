# Issue #321412 — Agent Window session list should show icon for sessions

[https://github.com/microsoft/vscode/issues/321412](https://github.com/microsoft/vscode/issues/321412)

## Summary

Still reproduces on main: the session list's icon column renders only status indicators (spinner/codicon) via SessionStatusIcon but never incorporates the session's agent-type icon (ISession.icon / ISessionType.icon), making it impossible to visually distinguish agent providers in the list.

## Change summary

The Agents Window session list does not display the session/agent type icon, so users cannot tell which agent (Claude, Copilot CLI, Codex) a session belongs to.

## Best solution

Incorporate the session's type icon into the session list row — either by extending the SessionStatusIcon widget to overlay/replace with the agent icon when idle, or by adding a separate icon element to the session item template that renders `element.icon` (the ThemeIcon from ISession). The data is already on the model; only the list renderer needs updating.
