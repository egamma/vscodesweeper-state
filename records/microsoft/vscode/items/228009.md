# Issue #228009 — Editor goes berserk and start scrolling on mouse movements

[https://github.com/microsoft/vscode/issues/228009](https://github.com/microsoft/vscode/issues/228009)

## Summary

Still present on main: the editor's `onDragOver` handler calls `revealPosition` on every dragover event, so if the platform fails to fire `dragleave`/`dragend`/`drop` (a known KDE/Wayland quirk), the editor continuously scrolls to follow the mouse. 26 upvotes meet the backlog threshold.

## Change summary

Editor gets stuck in a drag-scroll state where mouse movement causes continuous scrolling, requiring a restart or a manual drop to escape.

## Best solution

Make the editor resilient to stuck drag states — e.g., add an escape hatch in `showDropIndicatorAt` (check mouse buttons, add a timeout, or listen for a pointermove-without-buttons event to cancel the drag state). The relevant code is `src/vs/editor/browser/widget/codeEditor/codeEditorWidget.ts` lines 372–405 (DragAndDropObserver) and the `showDropIndicatorAt`/`removeDropIndicator` pair at lines 2054–2066.
