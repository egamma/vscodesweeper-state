# Issue #273097 — Terminal link hover stopped being a link

[https://github.com/microsoft/vscode/issues/273097](https://github.com/microsoft/vscode/issues/273097)

## Summary

Still reproduces on current main: terminal link hovers lack a pointer cursor (and thus don't appear clickable) because the markdown renderer clears `href` to `""` for security, and no CSS rule restores `cursor: pointer` on rendered hover links. Two community PRs attempted a fix but neither was merged.

## Change summary

Terminal link hover text no longer displays as a clickable link (no pointer cursor) after the markdown renderer started clearing `href` attributes for security sanitization.

## Best solution

Add `cursor: pointer` to `.monaco-hover.workbench-hover a` in `src/vs/platform/hover/browser/hover.css` — the same one-line fix proposed (but never merged) in PR #283081. This restores the visual affordance without undoing the security sanitization that moves the real href to `data-href`.
