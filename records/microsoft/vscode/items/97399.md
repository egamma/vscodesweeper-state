# Issue #97399 — Extension Development: ensure package.json contributions match the implementation

[https://github.com/microsoft/vscode/issues/97399](https://github.com/microsoft/vscode/issues/97399)

## Summary

Team-authored feature request for type-safe extension contributions that prevent string-key mismatches between package.json and extension code. Still relevant — no implementation has landed. Kept open per team-author policy.

## Change summary

Provide tooling or type-safe mechanisms to ensure extension package.json contribution keys match the strings used in extension code at registration time.

## Best solution

Ship a `@vscode/typed-contributions` package (as discussed by connor4312 and hediet) that augments vscode API interfaces with opaque branded string types, exports package.json contribution typings, and provides a CLI to sync a `package.ts` manifest into package.json. This keeps the core lightweight while giving extension authors type-checked, go-to-definition-friendly contribution declarations.
