# Issue #264338 — Can't unstage changes of edited file in new git repository

[https://github.com/microsoft/vscode/issues/264338](https://github.com/microsoft/vscode/issues/264338)

## Summary

Confirmed bug on current main: `git rm --cached -r` at line 2328 of `extensions/git/src/git.ts` lacks the `-f` flag, causing unstage to fail in repos with no commits when staged content differs from the working tree. An open PR (#304783) addresses this but has not merged.

## Change summary

Unstaging a file in a newly initialized git repository (no commits) fails when the staged content differs from the working tree copy.

## Best solution

Review and land the open PR https://github.com/microsoft/vscode/pull/304783, which adds `-f` to the `git rm --cached -r` fallback path in `extensions/git/src/git.ts`. The flag is safe because `--cached` never touches the working tree.
