# Issue #117106 — Advanced wrapping strategy causes hangs for larger files

[https://github.com/microsoft/vscode/issues/117106](https://github.com/microsoft/vscode/issues/117106)

## Summary

Keep open: the report identifies a concrete editor wrapping performance hang with the opt-in advanced strategy, and current main still exposes advanced wrapping as a slow DOM-based path rather than an auto-degraded large-file mode.

## Change summary

Advanced word wrapping can make opening and resizing large text files hang, and users are asking for VS Code to automatically avoid that path for large files.

## Best solution

The editor wrapping owner should decide whether large-file handling should force or suggest the simple wrapping strategy, add an auto mode, or introduce a timeout/threshold for advanced wrapping. The likely implementation area is the editor wrapping configuration and view-model line break computation paths.

## Evidence

- **maintainer reproduction context:** In https://github.com/microsoft/vscode/issues/117106#issuecomment-783504178, alexdima could not reproduce with a generated 80,000-line file and specifically pointed to `editor.wrappingStrategy: advanced`, whose setting text warns it is slow and may freeze large files.
- **reporter confirmation:** The reporter confirmed in later comments that disabling word wrap or switching away from advanced wrapping made the file open instantly or much faster, narrowing the issue to advanced wrapping rather than extensions.
- **current main code:** `src/vs/editor/common/config/editorOptions.ts` still defines only `simple` and `advanced` wrapping strategies, with `advanced` documented as delegating to the browser and possibly freezing for large files; no `auto` strategy or large-file threshold is present there.
- **current main implementation:** `src/vs/editor/common/viewModel/viewModelLines.ts` still chooses `_domLineBreaksComputerFactory` whenever `wrappingStrategy === 'advanced'`, so the central behavior described by the issue appears still relevant on main.
