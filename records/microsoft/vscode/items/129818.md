# Issue #129818 — Git - Git changes panel shows mid-checkout changes (is not atomic)

[https://github.com/microsoft/vscode/issues/129818](https://github.com/microsoft/vscode/issues/129818)

## Summary

Still valid on main. The existing guard in `onFileChange` (line 3185 of repository.ts) only suppresses file-system events during VS Code-initiated git operations — terminal-initiated checkouts bypass the operation manager, so mid-checkout file changes still trigger SCM view refreshes. A team member acknowledged this in December 2024 and proposed a solution, but no fix has landed.

## Change summary

Git changes panel shows intermediate/incomplete working directory state during a terminal-initiated `git checkout` on large repos because file-system events are not suppressed for externally-initiated git operations.

## Best solution

Implement the approach lszomoru proposed: detect external write operations (e.g. via index.lock presence or a post-checkout hook) and suppress file-system-event-driven refreshes while they run, deferring to a single refresh on completion. The fix lives in `extensions/git/src/repository.ts` around the `onFileChange` / `eventuallyUpdateWhenIdleAndWait` path.
