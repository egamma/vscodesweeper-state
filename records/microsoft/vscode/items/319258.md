# Issue #319258 — Copliot Chat, path rendering configuration options

[https://github.com/microsoft/vscode/issues/319258](https://github.com/microsoft/vscode/issues/319258)

## Summary

Valid feature request for configurable path display depth in chat inline file anchors. The current behavior (`getUriBasenameLabel`) shows only the last path segment, which is unhelpful in monorepos with recurring directory structures. No existing setting or duplicate found.

## Change summary

Chat inline file anchors strip paths to the last segment only, losing context in monorepos with repeated directory names like `api`.

## Best solution

Add a setting (e.g. `chat.inlineAnchor.pathSegments`) to control how many parent directory segments are shown in `InlineAnchorWidget`. The fix would live in `src/vs/workbench/contrib/chat/browser/widget/chatContentParts/chatInlineAnchorWidget.ts` around line 188, replacing the bare `getUriBasenameLabel` call with a configurable path-shortening strategy. The hover on line 288–289 already shows the relative path, so the full path is available — this is about the inline label only.
