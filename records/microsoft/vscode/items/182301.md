# Issue #182301 — Cursor Movement by Option+Click inside folded tmux line on Remote SSH makes vscode server hang

[https://github.com/microsoft/vscode/issues/182301](https://github.com/microsoft/vscode/issues/182301)

## Summary

The issue describes a real interaction problem where xterm.js's alt+click cursor positioning sends a flood of arrow-key escape sequences when tmux has wrapped a long line, overwhelming the remote server. No fix was found on current main. The bug sits at the xterm.js/tmux boundary and remains valid.

## Change summary

Option+Click to reposition cursor in a tmux session with wrapped lines sends excessive escape sequences, hanging the VS Code remote server.

## Best solution

Investigate whether xterm.js's altClickMovesCursor logic can be capped or made tmux-aware to avoid sending an unbounded number of cursor-movement sequences. This likely requires upstream xterm.js changes or a VS Code-side guard limiting the number of sequences sent in a single alt+click action.
