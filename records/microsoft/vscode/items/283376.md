## Summary

Team-authored debt issue with an 'On Deck' milestone — kept open per policy. The work (decoupling Copilot extension activation from GitHub auth) was attempted, reverted, and remains unfinished on current main.

## Change summary

Decouple the Copilot Chat extension's activation path from requiring GitHub authentication upfront, so the extension can activate and show UI without blocking on auth.

## Best solution

Complete the decoupling in `extensions/copilot/` so that extension activation, chat view rendering, and basic UI are not gated on a resolved auth session — auth should be lazy, triggered only when an actual request needs it. The earlier revert (PR #1860) documents the edge cases that need handling.
