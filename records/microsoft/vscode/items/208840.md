# Issue #208840 — File quick open does not match exact file paths in multi-root workspace

[https://github.com/microsoft/vscode/issues/208840](https://github.com/microsoft/vscode/issues/208840)

## Summary

Still reproduces on main: the Quick Open relative-path resolver prepends the query verbatim to each workspace folder URI without stripping a leading folder-name segment, so pasting paths like `workspace_1/test_file.txt` yields no results. An open PR (#318517) addresses this — keep open until it lands.

## Change summary

Quick Open file picker does not find files when the pasted relative path starts with a workspace folder name in a multi-root workspace.

## Best solution

Review and land PR https://github.com/microsoft/vscode/pull/318517, which adds workspace-folder-prefix stripping to `getRelativePathFileResults` in `src/vs/workbench/contrib/search/browser/anythingQuickAccess.ts`. The fix is localized and includes unit tests.
