# Issue #180579 — Quick pick label tooltips

[https://github.com/microsoft/vscode/issues/180579](https://github.com/microsoft/vscode/issues/180579)

## Summary

Still reproduces on main: when a quick pick item has no explicit tooltip, the label text gets no hover/title fallback, so truncated labels cannot be read in full. The description and detail paths have fallback titles, but the label path does not.

## Change summary

Quick pick items with labels longer than the quick pick width cannot be read in full because no tooltip/hover is shown for the label text.

## Best solution

In `src/vs/platform/quickinput/browser/quickInputList.ts`, add a `title` property to the `IIconLabelValueOptions` object (analogous to `descriptionTitle`) that falls back to `element.saneLabel` when no explicit `saneTooltip` is set, so truncated labels show a hover with the full text.
