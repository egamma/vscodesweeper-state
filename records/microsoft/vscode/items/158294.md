# Issue #158294 — Multiselect selecting extra elements

[https://github.com/microsoft/vscode/issues/158294](https://github.com/microsoft/vscode/issues/158294)

## Summary

Team-authored confirmed bug still present on main: shift+click range selection in the search view can use a stale focus index as the anchor after a Replace operation, selecting more elements than intended. Hard keep-open (MEMBER author).

## Change summary

After replacing a search result and then shift+clicking another result, the range selection includes extra elements because the anchor falls back to the focused index rather than the selected index.

## Best solution

In the search view's list interaction (or in the base list widget's `changeSelection` at `src/vs/base/browser/ui/list/listWidget.ts` ~line 818), ensure that after a Replace operation the anchor/focus are kept in sync with the actual selection, so shift+click computes the correct range. The fix likely belongs in how the search view updates focus after replacing a result.
