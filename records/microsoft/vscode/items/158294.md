# Issue #158294 — Multiselect selecting extra elements

[https://github.com/microsoft/vscode/issues/158294](https://github.com/microsoft/vscode/issues/158294)

## Summary

Team-authored confirmed bug: after Replace removes a search result, the list anchor/focus isn't updated, so shift+click range selection uses a stale focus index and selects extra elements. Still appears present on main.

## Change summary

Shift+click multiselection in search results selects extra elements after using Replace, due to stale focus/anchor state.

## Best solution

After a Replace operation removes search results from the list, the list's anchor (or focus fallback used as anchor) should be cleared or updated to reflect the new selection state. The fix likely belongs in the search view's splice/replace handling or in how `listWidget.ts` reconciles anchor after element removal.
