# Issue #326392 — create_chat-created chats: multi-chat pill is non-responsive

[https://github.com/microsoft/vscode/issues/326392](https://github.com/microsoft/vscode/issues/326392)

## Summary

Team-authored bug about the create_chat tool producing a Chat object that lacks Session semantics, causing the multi-chat pill to be non-responsive. Keeping open per policy (MEMBER author).

## Change summary

The create_chat MCP tool creates a Chat rather than a Session, so the multi-chat pill rendered on that chat is non-functional.

## Best solution

Either the create_chat tool should create a Session (which supports multi-chats), or the UI should suppress the multi-chat pill on standalone Chats that cannot host them. The fix likely lives in the agent host MCP server tool registration or the chat UI contribution that renders the pill.
