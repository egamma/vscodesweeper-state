# Issue #135981 — VScode copy to Onenote missing blank line

[https://github.com/microsoft/vscode/issues/135981](https://github.com/microsoft/vscode/issues/135981)

## Summary

The blank-line loss is caused by OneNote's HTML rendering stripping one <br> per sequence, not by VS Code producing incorrect HTML. The issue is nearly 5 years old with no upvotes and a known workaround. Requesting confirmation on latest build before further action.

## Change summary

Copying text with blank lines from VS Code and pasting into OneNote loses one blank line per sequence because OneNote misinterprets the <br> HTML VS Code places on the clipboard.

## Best solution

This is most likely an upstream OneNote rendering bug — VS Code's `<br>` for empty lines is standard HTML. The workaround is `editor.copyWithSyntaxHighlighting: false`. If the reporter still needs syntax-highlighted copy with correct blank lines in OneNote, the fix belongs in OneNote. Could alternatively be closed as `*as-designed` since VS Code's HTML output is correct.
