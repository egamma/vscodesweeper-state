# Issue #135981 — VScode copy to Onenote missing blank line

[https://github.com/microsoft/vscode/issues/135981](https://github.com/microsoft/vscode/issues/135981)

## Summary

The report appears to be a paste/rendering interoperability problem with OneNote rather than a clear VS Code editor bug, but there is not enough concrete maintainer evidence here to close it as designed or not reproducible. Keep it open only if the editor-clipboard area wants to track compatibility improvements to VS Code's generated HTML clipboard payload.

## Change summary

Copying Markdown text with syntax highlighting from VS Code to OneNote preserves colors but OneNote drops blank lines from the formatted paste.

## Best solution

The ideal end state is either a deliberate editor-clipboard decision that VS Code's generated HTML should remain unchanged, or a targeted compatibility adjustment in the copy-with-syntax-highlighting HTML if maintainers decide OneNote's handling is important enough to accommodate. The known user workaround is setting `editor.copyWithSyntaxHighlighting` to `false`, but that loses formatting.

## Evidence

- **issue context:** The issue is an external paste target scenario: copying Markdown from VS Code 1.61.2 on macOS into OneNote removes blank lines only in the formatted paste.
- **comment analysis:** https://github.com/microsoft/vscode/issues/135981#issuecomment-964273230 reports that VS Code puts both raw text and an HTML representation on the clipboard, and OneNote drops one `<br>` per blank-line sequence when rendering the HTML.
- **workaround:** https://github.com/microsoft/vscode/issues/135981#issuecomment-970650006 identifies `editor.copyWithSyntaxHighlighting=false` as a workaround that makes OneNote receive plain text instead of formatted HTML, but the reporter explicitly wants to keep colors and syntax highlighting.
