# Issue #135981 — VScode copy to Onenote missing blank line

[https://github.com/microsoft/vscode/issues/135981](https://github.com/microsoft/vscode/issues/135981)

## Summary

This appears to be a OneNote paste/rendering compatibility issue rather than a VS Code clipboard bug. VS Code intentionally places both plain text and syntax-highlighted HTML on the clipboard, and the report/comment thread identifies OneNote as dropping one blank line from the HTML representation.

## Change summary

Copying syntax-highlighted Markdown from VS Code into OneNote loses blank rows while preserving colors.

- **Triage action:** propose-close

- **Proposed comment:**

  > Thanks for the report. VS Code intentionally writes both plain text and syntax-highlighted HTML when `editor.copyWithSyntaxHighlighting` is enabled; current main still does that in `src/vs/editor/browser/controller/editContext/clipboardUtils.ts`, and blank editor lines in the rich HTML are represented as `<br>` from `src/vs/editor/common/viewModel/viewModelImpl.ts`. Based on the investigation above, OneNote is choosing the HTML clipboard format and dropping one blank line from those `<br>` sequences when rendering the paste. Since preserving syntax highlighting while changing OneNote's paste interpretation would require an app-specific workaround for OneNote's HTML renderer, I think this is out of scope for VS Code. The available VS Code-side workaround remains setting `editor.copyWithSyntaxHighlighting` to `false`, which copies plain text only.

## Evidence

- **current implementation:** src/vs/editor/browser/controller/editContext/clipboardUtils.ts:46 builds clipboard data with plain text always present and HTML added only when copyWithSyntaxHighlighting is enabled.
- **current implementation:** src/vs/editor/common/viewModel/viewModelImpl.ts:1092-1105 generates syntax-highlighted HTML with CSS including white-space: pre, and src/vs/editor/common/viewModel/viewModelImpl.ts:1126-1128 represents empty editor lines as <br>.
- **setting:** src/vs/editor/common/config/editorOptions.ts:6167-6169 defines editor.copyWithSyntaxHighlighting as enabled by default and documents that it controls whether syntax highlighting is copied into the clipboard.
- **discussion:** In https://github.com/microsoft/vscode/issues/135981, dev-manuel investigated the clipboard contents and concluded OneNote removes one <br> per blank-line sequence; they also identified editor.copyWithSyntaxHighlighting=false as a plain-text workaround.
