# Issue #135981 — VScode copy to Onenote missing blank line

[https://github.com/microsoft/vscode/issues/135981](https://github.com/microsoft/vscode/issues/135981)

## Summary

Keep open for maintainer judgment in the editor clipboard area: current code still emits rich HTML for syntax-highlighted copies, with empty editor lines represented as `<br>`, and the report is specifically about OneNote losing those blank lines. There is evidence this may be a OneNote rendering issue, but not enough maintainer-confirmed evidence to close it as out of scope.

## Change summary

Copying syntax-highlighted Markdown from VS Code to OneNote drops blank lines in the pasted rich text.

## Best solution

If VS Code wants to support this paste target, adjust the rich clipboard HTML generated by the editor clipboard path so blank lines survive in OneNote while preserving syntax highlighting; otherwise a maintainer should explicitly decide this is target-application behavior and document the `editor.copyWithSyntaxHighlighting: false` workaround.
