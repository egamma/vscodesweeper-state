# Issue #316412 — Built-in Copilot extension force-opens worktrees via public Git API, bypassing closed-list and git.detectWorktrees

[https://github.com/microsoft/vscode/issues/316412](https://github.com/microsoft/vscode/issues/316412)

## Summary

Confirmed on current main: the public git API's `openRepository` hardcodes `openIfClosed=true`, allowing any extension (here the bundled Copilot extension) to bypass the user's closed-repository list. The resulting mass re-open of worktrees causes severe performance degradation. An open PR (#313612) partially addresses re-opening from the git side but does not fix the public API bypass.

## Change summary

The public `vscode.git` extension API method `openRepository` unconditionally overrides the user's closed-repository list, causing extensions like Copilot Chat to force-open all linked worktrees on every load.

## Best solution

The public `openRepository` API in `extensions/git/src/api/api1.ts` should respect the closed-repository list (pass `openIfClosed=false`), or introduce a separate parameter/option to let callers express intent. The Copilot extension side should also avoid calling `openRepository` speculatively. Coordinate with PR #313612 which tackles the `checkForWorktrees` angle.
