# Issue #273203 — Auto complete doesn't realize we are using React 19

[https://github.com/microsoft/vscode/issues/273203](https://github.com/microsoft/vscode/issues/273203)

## Summary

Valid NES quality issue — suggestions don't account for the project's React version, leading to outdated forwardRef patterns. A team member (dbaeumer) has already identified a solution direction (adding library version traits). Properly labeled and awaiting implementation.

## Change summary

NES suggests React 18-style forwardRef wrapping in projects using React 19 (where forwardRef is unnecessary), ignoring the installed @types/react version.

## Best solution

Implement library version traits for NES context, as dbaeumer suggested — when standard libraries contribute to the global namespace, their version should inform suggestion generation so version-specific patterns (like React 19's ref-as-prop) are preferred over deprecated ones.
