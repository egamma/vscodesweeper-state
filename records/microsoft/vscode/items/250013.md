# Issue #250013 — Provide a command to Restart Extensions

[https://github.com/microsoft/vscode/issues/250013](https://github.com/microsoft/vscode/issues/250013)

## Summary

Valid feature request: there is no standalone command or global button to invoke `updateRunningExtensions` (the logic behind per-extension 'Restart Extensions' buttons). The existing 'Developer: Restart Extension Host' does not perform the same operation, as confirmed in the discussion. The request has 3 upvotes, is in Backlog, and the gap is real on current main.

## Change summary

Provide a global 'Restart Extensions' command (and optionally a button in the extensions view header) that applies pending extension updates, enables, and disables — equivalent to what the per-extension restart button does.

## Best solution

Register a new command (e.g. 'Extensions: Restart Extensions') that calls `IExtensionsWorkbenchService.updateRunningExtensions()`, visible in the command palette and optionally as a view action in the extensions viewlet header when extensions have pending runtime state changes. The logic already exists in `extensionsWorkbenchService.ts`; it just needs a command-palette entry point and optional UI surface.
