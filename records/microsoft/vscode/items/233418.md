# Issue #233418 — File Search/Replace replacing multiple times (incorrectly) when double-clicking

[https://github.com/microsoft/vscode/issues/233418](https://github.com/microsoft/vscode/issues/233418)

## Summary

Still reproducible on main: the search replace action (`performReplace`) is async but has no concurrency guard, so rapid clicks can trigger multiple replace operations on the same match before the first completes and removes the element from the tree.

## Change summary

Clicking the replace button rapidly in search results can replace the same match multiple times, corrupting the file.

## Best solution

Add a concurrency guard (e.g. a `Sequencer` or a boolean lock) in `performReplace` in `searchActionsRemoveReplace.ts` to prevent re-entrant execution while a replace is already in progress, or disable the replace action's UI affordance while the async operation is pending.
