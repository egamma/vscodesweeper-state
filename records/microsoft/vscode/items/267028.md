# Issue #267028 — Shell integration leaks variables

[https://github.com/microsoft/vscode/issues/267028](https://github.com/microsoft/vscode/issues/267028)

## Summary

Confirmed on current main: the five reported variables (vsc_env_keys, vsc_env_values, use_associative_array, bash_major_version, envVarsToReport) are declared at the top of shellIntegration-bash.sh (lines 13–22) and never unset, leaking into the user's shell session. The issue is correctly labeled as debt and belongs in terminal-shell-integration.

## Change summary

Shell integration bash script leaks internal variables into the user's shell environment because they are never unset after use.

## Best solution

Add `unset` calls for all five variables after their last use in the script (around the end of the env-reporting block). Other VS Code env vars in the same file are already cleaned up with `builtin unset`; these should follow the same pattern. The fix is in `src/vs/workbench/contrib/terminal/common/scripts/shellIntegration-bash.sh`.
