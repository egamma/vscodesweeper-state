# Issue #204103 — mouse text selection cursor turned white on light themes

[https://github.com/microsoft/vscode/issues/204103](https://github.com/microsoft/vscode/issues/204103)

## Summary

Keep this open as an upstream Chromium/GPU issue that is already labeled and mitigated, not as a VS Code-core bug to close. The discussion shows the issue still reproduced in Insiders for at least some users and is mitigated by disabling Direct Composition/MPO-related paths, so the best maintainer action is to continue tracking the upstream fix/workaround status.

## Change summary

On Windows dark system theme with VS Code light themes, the text-selection mouse cursor can become white and hard to see, especially on AMD GPU setups.

## Best solution

Track this under the existing Windows/GPU/Chromium upstream labels until the Chromium/Electron rendering path no longer produces the white I-beam cursor, while keeping `--disable_direct_composition=1` documented as the VS Code-side mitigation if needed.
