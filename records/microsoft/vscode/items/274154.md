# Issue #274154 — copy text, delete text, pastes different text that is now in the location where old text was

[https://github.com/microsoft/vscode/issues/274154](https://github.com/microsoft/vscode/issues/274154)

## Summary

Valid bug on Linux/KDE when 'Keep the selection and clipboard the same' is enabled. VS Code's SelectionClipboard feature writes to the selection clipboard on every cursor selection change, and KDE's sync propagates that back to the system clipboard, overwriting what was explicitly Ctrl+C'd. Other editors (Kate, LibreOffice) handle this correctly. No fix found on current main.

## Change summary

Ctrl+C then Delete then Ctrl+V pastes wrong text on Linux/KDE when KDE clipboard sync is active.

## Best solution

The SelectionClipboard contribution (`src/vs/workbench/contrib/codeEditor/electron-browser/selectionClipboard.ts`) should avoid writing to the selection clipboard in scenarios where it would conflict with an explicit user copy (Ctrl+C). A debounce or guard that skips selection-clipboard writes immediately after a programmatic clipboard write could prevent the KDE sync from clobbering the user's intended paste content.
