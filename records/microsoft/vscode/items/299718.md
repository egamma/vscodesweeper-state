# Issue #299718 — 100% CPU time during startup by copilot extension in WSL

[https://github.com/microsoft/vscode/issues/299718](https://github.com/microsoft/vscode/issues/299718)

## Summary

Still reproduces on main: the built-in Copilot extension's workspace file indexer uses expensive per-file regex glob matching in `isCopilotIgnored` and a concurrency limiter of 20 that saturates the extension host CPU in large workspaces. No performance fix has landed since the report.

## Change summary

The Copilot extension's workspace indexing causes 100% CPU and blocks the extension host during startup in large WSL workspaces due to expensive glob pattern matching in `isCopilotIgnored`.

## Best solution

Optimize the ignore-pattern matching in `extensions/copilot/src/platform/ignore/node/ignoreServiceImpl.ts` — pre-compile patterns, use map-based fast paths for common globs (extension, basename, prefix), and consider reducing the `_fileReadLimiter` concurrency or yielding to the event loop during indexing in `extensions/copilot/src/platform/workspaceChunkSearch/node/workspaceFileIndex.ts`.
