# Issue #313105 — Agents: Recent project remains in new session dropdown after X

[https://github.com/microsoft/vscode/issues/313105](https://github.com/microsoft/vscode/issues/313105)

## Summary

Team-authored bug with milestone 'On Deck' — hard keep-open. The workspace picker's remove action for VS Code recents calls `workspacesService.removeRecentlyOpened` but the picker likely re-reads the full recents list on next open without filtering out previously dismissed items.

## Change summary

Dismissing a recent project from the new session dropdown does not persist — the project reappears when the picker is reopened.

## Best solution

Ensure `_removeVSCodeRecentWorkspace` (or `_removeRecentWorkspace` depending on which path applies) actually prevents the item from reappearing on the next picker open. If the issue is the VS Code recents path, the picker needs to maintain a persistent exclusion list or the `workspacesService.removeRecentlyOpened` call needs to be verified as taking effect before the picker re-queries.
