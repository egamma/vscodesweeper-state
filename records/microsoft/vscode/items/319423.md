# Issue #319423 — Extension host crash loop on Stable 1.122.1: built-in Copilot Chat 0.50.1 registers undeclared "claude-code" chatParticipant → TypeError: e is not iterable

[https://github.com/microsoft/vscode/issues/319423](https://github.com/microsoft/vscode/issues/319423)

## Summary

The fatal crash loop (exit code 5) is fixed in current Stable builds (confirmed by comment 2 on 1.127.0/0.55.0), and on main the claude-code session type is properly declared via the chatSessions contribution point. However, non-fatal TypeError errors still fire on Stable per comments 2–3, and a team member provided a workaround (Agent Host settings). Keeping open as the team is actively rolling out the full fix.

## Change summary

Built-in Copilot Chat registers an undeclared 'claude-code' chatParticipant, causing an unhandled TypeError that crash-loops the extension host on Stable.

## Best solution

Complete the Agent Host rollout so the legacy claude-code chatParticipant registration path is no longer exercised on Stable builds, eliminating both the crash and the residual non-fatal errors. The architecture on main already uses chatSessions instead of chatParticipants for claude-code.
