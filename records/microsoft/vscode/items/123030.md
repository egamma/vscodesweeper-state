# Issue #123030 — Fails To Copy Large Text

[https://github.com/microsoft/vscode/issues/123030](https://github.com/microsoft/vscode/issues/123030)

## Summary

Still an open gap: the native clipboard write path (`clipboard.writeText` in Electron) has no size guard or chunking on Linux, so large copies fail silently. Tracked as a sub-item of the Clipboard API parity issue #141704.

## Change summary

Copying very large text (2M+ chars) on Linux silently fails — nothing is written to the system clipboard.

## Best solution

Either implement a workaround in VS Code (e.g. fallback to xclip/xsel for large payloads, or chunk the write), or track upstream in Electron/Chromium. The parent issue https://github.com/microsoft/vscode/issues/141704 already owns this item; this issue provides the specific reproduction.
