# Issue #290901 — Can't scroll over expanded thinking container

[https://github.com/microsoft/vscode/issues/290901](https://github.com/microsoft/vscode/issues/290901)

## Summary

Team-filed bug — the DomScrollableElement wrapping the thinking container still exists on main and likely still intercepts trackpad scroll events, preventing the parent chat list from scrolling. Keeping open per team-authored policy.

## Change summary

Trackpad scrolling over an expanded thinking container in the chat view is intercepted by the thinking box's inner scrollable, preventing the outer chat list from scrolling.

## Best solution

The DomScrollableElement in chatThinkingContentPart.ts should propagate scroll events to the parent list when the inner container has reached its scroll boundary (top or bottom), or the scroll interception should be disabled when the thinking content fits within the viewport. The fix likely lives in src/vs/workbench/contrib/chat/browser/widget/chatContentParts/chatThinkingContentPart.ts around the scrollable element setup (~line 595).
