# Issue #250074 — Can't use both env and envFile for MCP server configuration

[https://github.com/microsoft/vscode/issues/250074](https://github.com/microsoft/vscode/issues/250074)

## Summary

Team-authored issue requesting that `${env:VARNAME}` in MCP server config can resolve variables from the `envFile`, or at minimum better documentation of the current order-of-operations. Keeping open — MEMBER author, valid UX gap.

## Change summary

Allow `${env:VARNAME}` in MCP server `env` config to reference variables defined in `envFile`, or clearly document the resolution order.

## Best solution

Either (a) resolve `envFile` before `${env:}` substitution so users can reference envFile vars in the `env` block, or (b) clearly document that `${env:}` resolves from the process environment only and `envFile` is applied independently at spawn time. Option (a) is the better UX. The relevant code is in `src/vs/workbench/api/node/extHostMcpNode.ts` (env merging) and variable resolution likely in the MCP config resolver.
