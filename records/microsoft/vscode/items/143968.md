# Issue #143968 — Unable to rename/create a file when window has no focus.

[https://github.com/microsoft/vscode/issues/143968](https://github.com/microsoft/vscode/issues/143968)

## Summary

Still reproduces on current stable macOS builds. The root cause is well understood: native macOS context menus (the default `window.menuStyle` on stable) do not grant window focus, so rename/new-file input fields appear without keyboard focus and are dismissed when the user clicks to focus the window. A workaround exists (`"window.menuStyle": "custom"`), but the default experience is broken. 54 upvotes and 17+ closed duplicates confirm broad impact. A team member engaged recently but could not reproduce because Insiders defaults to custom menus.

## Change summary

Right-clicking the file explorer to rename or create a file when VS Code is unfocused on macOS creates an input field that cannot receive keyboard input; focusing the window dismisses it.

## Best solution

When a rename/new-file inline input is created via context menu, the window should programmatically request focus (e.g. via `BrowserWindow.focus()` from the main process) before showing the input. The fix likely lives in the explorer's file-creation/rename flow or the context-menu handling in the Electron main process. Alternatively, the macOS stable default for `window.menuStyle` could be changed from `native` to `inherit` or `custom`, though that has broader UX implications. The TODO comment on line 284 of `desktop.contribution.ts` suggests this default is already under review.
