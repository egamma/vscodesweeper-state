# Issue #134370 — Git - Merge Conflict - Accept All Incoming/Current - Multiple Repositories

[https://github.com/microsoft/vscode/issues/134370](https://github.com/microsoft/vscode/issues/134370)

## Summary

Still reproduces on main: 'Merge Conflict: Accept All Incoming/Current' invoked from the command palette in a multi-repo workspace silently fails because the command handler relies on `vscode.window.activeTextEditor` and operates only on that single document — no repository picker is shown. Maintainer discussion (joaomoreno, lszomoru) confirmed this is a real bug and that a quick pick for repo selection should appear but doesn't.

## Change summary

'Merge Conflict: Accept All Incoming/Current' command does nothing when invoked from the command palette in a workspace with multiple git repositories.

## Best solution

When the command palette invokes 'Accept All Current/Incoming' without resource args, the command handler in `extensions/merge-conflict/src/commandHandler.ts` should either show a quick pick to select the repository (as joaomoreno suggested) or operate across all repos. The `registerTextEditorCommand` wrapper (line 43–50) needs a fallback path when no active editor has conflicts — likely by querying all open repos for conflicted files and prompting.
