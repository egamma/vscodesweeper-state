# Issue #158644 — Git - Push (and Sync) does not respect remote.pushdefault configuration

[https://github.com/microsoft/vscode/issues/158644](https://github.com/microsoft/vscode/issues/158644)

## Summary

Still reproduces on main: the git extension has no code to read `remote.pushDefault` or `branch.<name>.pushRemote` when determining the push target. An open PR (#323650) is attempting to fix this — keeping open for that PR to land.

## Change summary

Git Push and Sync commands ignore `remote.pushDefault` and `branch.<name>.pushRemote` configuration, always pushing to the branch's tracking remote instead.

## Best solution

Review and land the open PR https://github.com/microsoft/vscode/pull/323650 which adds push-target resolution respecting git's push configuration hierarchy (`branch.<name>.pushRemote` > `remote.pushDefault` > branch remote).
