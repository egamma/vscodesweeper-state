# Issue #324989 — Feature Request: Add an interactive Inline Command History Search/Filter UI for Integrated Terminal

[https://github.com/microsoft/vscode/issues/324989](https://github.com/microsoft/vscode/issues/324989)

## Summary

The requested feature already exists: the built-in 'Terminal: Run Recent Command...' command (Ctrl+Alt+R / Ctrl+Cmd+R) provides a Quick Pick UI with fuzzy search over terminal command history, with options to paste or execute the selected command. This covers every aspect of the request.

## Change summary

Request for an interactive Quick Pick–style command history search/filter UI in the integrated terminal.

## Best solution

Close as already available. The built-in command `workbench.action.terminal.runRecentCommand` (Terminal: Run Recent Command…) provides exactly the requested Quick Pick with fuzzy search, session-aware history, and paste/execute on Enter. The companion `workbench.action.terminal.goToRecentDirectory` covers directory history. The history size is configurable via `terminal.integrated.shellIntegration.history`.

  > Thanks for the detailed feature request! This capability actually already exists in VS Code as the **Terminal: Run Recent Command…** command (`workbench.action.terminal.runRecentCommand`). You can trigger it with:
  > 
  > - **Ctrl+Alt+R** (Windows/Linux) or **Ctrl+Option+R** (macOS)
  > - Or via the Command Palette: `Terminal: Run Recent Command…`
  > 
  > It opens a Quick Pick with fuzzy-searchable command history from the current session, previous sessions, and your shell's history file. Selecting a command pastes it into the terminal (or executes it on Enter). There's also a companion **Terminal: Go to Recent Directory…** command.
  > 
  > The number of remembered commands is configurable via `terminal.integrated.shellIntegration.history` (default: 100).
  > 
  > Closing this as the feature is already available — please reopen if you see a gap the existing command doesn't cover.
