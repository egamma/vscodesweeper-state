# Issue #315250 — Moving multiple tabs reverses their order

[https://github.com/microsoft/vscode/issues/315250](https://github.com/microsoft/vscode/issues/315250)

## Summary

Confirmed bug, still present on main: prepareMoveCopyEditors always places the active editor first, breaking the original visual order when moving multiple tabs across groups. Team member acknowledged the behavior.

## Change summary

Moving multiple selected editor tabs to another group reorders them by placing the active tab first instead of preserving original visual order.

## Best solution

Fix `prepareMoveCopyEditors` in `src/vs/workbench/browser/parts/editor/editor.ts` to preserve the original visual order of all editors (including the active one) while still opening the active editor as the focused/active tab in the target group. The active editor should retain its relative position among the moved set rather than always being placed first.
