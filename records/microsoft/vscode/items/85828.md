# Issue #85828 — [css] Support for SVG 2 CSS Properties

[https://github.com/microsoft/vscode/issues/85828](https://github.com/microsoft/vscode/issues/85828)

## Summary

Valid feature request with 23 upvotes, already in Backlog with 'help wanted'. The CSS language service still lacks SVG 2 CSS properties; no fix has landed on main. The upstream vscode-css-languageservice owns the property data.

## Change summary

VS Code's CSS language service does not recognize SVG 2 CSS properties (cx, cy, r, x, y, rx, ry, d, dominant-baseline, vector-effect, etc.), causing false 'unknown property' warnings.

## Best solution

Add the missing SVG 2 presentation properties to the property data in `microsoft/vscode-css-languageservice`. The workaround of `css.lint.validProperties` exists but a first-class fix should land upstream. The 'help wanted' label is appropriate — a community PR to vscode-css-languageservice adding these properties would resolve it.
