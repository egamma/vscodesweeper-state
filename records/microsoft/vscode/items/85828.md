## Summary

Most SVG 2 CSS properties requested here (cx, cy, rx, ry, r, x, y, d, dominant-baseline, vector-effect) are now recognized on current main via the updated vscode-css-languageservice data. Only `alignment-baseline` remains missing. The issue is largely addressed but not fully complete.

## Change summary

VS Code's CSS linter flags SVG 2 CSS properties (cx, cy, r, d, dominant-baseline, etc.) as unknown, producing false warnings.

## Best solution

The remaining gap is `alignment-baseline` — it should be added to the property data in the upstream `vscode-css-languageservice` repo (https://github.com/microsoft/vscode-css-languageservice). Once that ships, this issue can be closed as complete. A maintainer could also close this now since the vast majority of the request is fulfilled and file a focused follow-up for the one missing property.
