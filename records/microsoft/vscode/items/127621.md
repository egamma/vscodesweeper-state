# Issue #127621 — Explore improving search performance to support high `maxResults`

[https://github.com/microsoft/vscode/issues/127621](https://github.com/microsoft/vscode/issues/127621)

## Summary

Still valid on current main: the original fix (commit 07a0575b6cb) was reverted (commits 2a0410598a5, 1fc68f3cf27) because it broke smoke tests, and no replacement fix has landed. The `refreshAndUpdateCount` path in searchView.ts still runs a full tree refresh via `updateChildren(undefined)` on every batch, which scales poorly at very high result counts. The related issue #164927 (adopt diffIdentityProvider for search results) is also still open and unfixed.

## Change summary

Search view becomes unusably slow when `search.maxResults` is set very high, because each incremental result batch triggers a full tree refresh.

## Best solution

Adopt `diffIdentityProvider` for the search results tree (tracked by https://github.com/microsoft/vscode/issues/164927) and throttle or skip intermediate tree refreshes when result counts are very large. The fix area is `src/vs/workbench/contrib/search/browser/searchView.ts` (the `RefreshTreeController` and `_updateResults` paths).
