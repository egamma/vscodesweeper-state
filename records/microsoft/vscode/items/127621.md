# Issue #127621 — Explore improving search performance to support high `maxResults`

[https://github.com/microsoft/vscode/issues/127621](https://github.com/microsoft/vscode/issues/127621)

## Summary

Still valid: the attempted diffIdentityProvider fix was reverted (broke smoke tests), PR #163903 was closed without merging, and issue #164927 tracking adoption remains open. The refreshAndUpdateCount/refreshTree code path still does full tree refreshes with no identity-based diffing, so the slowdown with very high maxResults persists on current main.

## Change summary

Search becomes unusably slow when `search.maxResults` is set very high because repeated full tree refreshes scale poorly with hundreds of thousands of results.

## Best solution

Adopt a diffIdentityProvider for the search results tree (tracked in https://github.com/microsoft/vscode/issues/164927) so that incremental updates diff by identity rather than rebuilding the entire tree. Additionally, throttle or skip rendering entirely when result counts exceed a practical viewing threshold, since nobody scrolls through 300k results — keep them in the model for replace but avoid rendering overhead.
