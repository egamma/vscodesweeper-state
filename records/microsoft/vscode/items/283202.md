# Issue #283202 — enhancement: enable extensions (spell checker) access in copilot prompt window

[https://github.com/microsoft/vscode/issues/283202](https://github.com/microsoft/vscode/issues/283202)

## Summary

Valid feature request for enabling extension contributions (specifically spell checking) in the chat input widget. The prior issue #258940 was incorrectly closed as a duplicate of an unrelated offline-spell-checker issue; this is the surviving request. Low upvote count (4), no milestone — route to the chat area for visibility.

## Change summary

Allow extensions like spell checkers to provide diagnostics and squiggles in the Copilot chat/prompt input editor.

## Best solution

The chat input widget would need to expose a text model that participates in the standard diagnostics/language-features pipeline so extension-contributed providers (spell check, grammar, etc.) can operate on it. This likely involves the chat input part (`src/vs/workbench/contrib/chat/browser/widget/input/chatInputPart.ts`) and how its editor model is registered. Whether this belongs in core or should be enabled via an extension-API surface is a product decision for the chat team.
