# Issue #313041 — chat not taking input from letter "n"

[https://github.com/microsoft/vscode/issues/313041](https://github.com/microsoft/vscode/issues/313041)

## Summary

The reporter's own description strongly indicates the Vim extension is intercepting the 'n' key (which is 'find next' in Vim normal mode). The issue resolves when forcing Vim out of its modal state, and only affects users with that extension installed.

## Change summary

The letter 'n' stops being accepted in the chat input and other areas, requiring ESC presses or a restart to recover.

## Best solution

This belongs in the Vim extension's issue tracker (https://github.com/VSCodeVim/Vim). The Vim extension occasionally leaks its normal-mode keybinding context, causing 'n' (find-next in Vim) to be captured globally. The user should report to the extension and can work around it by disabling the Vim extension or checking its keybinding configuration.
