# Issue #236602 — Link parsing stoped where comments include a single quote

[https://github.com/microsoft/vscode/issues/236602](https://github.com/microsoft/vscode/issues/236602)

## Summary

Keep open and route to editor/link detection: current main still has the quote-termination behavior that cuts off this URL, and the prior #151631 test was explicitly removed after the #156875 tradeoff.

## Change summary

Editor link detection stops early when an unwrapped URL contains a literal single quote in the fragment.

## Best solution

Have the editor link detection owner decide whether `src/vs/editor/common/languages/linkComputer.ts` can distinguish valid URL-internal quotes from surrounding quotes without regressing the #156875 case, then add focused `linkComputer.test.ts` coverage for the regexper URL.
