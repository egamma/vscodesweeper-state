# Issue #236602 — Link parsing stoped where comments include a single quote

[https://github.com/microsoft/vscode/issues/236602](https://github.com/microsoft/vscode/issues/236602)

## Summary

Still reproduces on main: the link parser terminates on a single quote inside URLs when the link isn't wrapped by a quote character. The test for #151631 was commented out when #156875's fix landed, and the community PR #237126 was closed without merge.

## Change summary

URLs containing literal single quotes (e.g. in regex fragments) are truncated at the quote when parsed as links in code comments.

## Best solution

Revise the quote-termination logic in `src/vs/editor/common/languages/linkComputer.ts` (lines 264–273) to allow percent-encoded or context-appropriate single quotes inside URLs that aren't quote-wrapped. The commented-out test at line 266 of the test file provides the expected assertion. A nuanced heuristic — e.g. allowing quotes when preceded by `%` or inside a `#fragment` — would avoid regressing #156875.
