# Issue #312289 — chat/subagent: Provide API to control default expanded state of subagent content parts

[https://github.com/microsoft/vscode/issues/312289](https://github.com/microsoft/vscode/issues/312289)

## Summary

Team-authored feature request for a new API property on subagent invocations to control the default expanded/collapsed state. No such API surface exists today — the subagent content part is hardcoded to start collapsed. Keeping open per team-author policy.

## Change summary

Extensions invoking subagents via `runSubagent` cannot control whether the subagent content section starts expanded or collapsed; it is always collapsed, hiding real-time progress from users.

## Best solution

Add an optional `defaultExpanded` (or similar) property to `ChatSubagentToolInvocationData` in the proposed `chatParticipantAdditions` API, and wire it through the internal model to `ChatSubagentContentPart` so it initializes `_isExpanded` from the data rather than always `false`. The rendering side lives in `src/vs/workbench/contrib/chat/browser/widget/chatContentParts/chatSubagentContentPart.ts`; the API surface is in `src/vscode-dts/vscode.proposed.chatParticipantAdditions.d.ts`.
