# Issue #322570 — [Error] unhandlederror-Uncaught TypeError: Cannot set properties of undefined (setting 'isWrapped')

[https://github.com/microsoft/vscode/issues/322570](https://github.com/microsoft/vscode/issues/322570)

## Summary

Active upstream xterm.js bug — the entire crash stack is inside `@xterm/xterm` with no VS Code frames, and the error persists across multiple stable releases (1.125.0 through 1.129.1 with growing hit counts). The fix belongs in the xterm.js library; keeping open to track resolution.

## Change summary

xterm.js throws `Cannot set properties of undefined (setting 'isWrapped')` during `lineFeed`/`print` when an internal buffer lookup returns `undefined`, crashing the terminal renderer for ~2,000+ users across stable releases.

## Best solution

File or track the fix upstream in `xtermjs/xterm.js` — the `lineFeed` and `print` code paths need a guard (or a fix for the buffer desync) when `buffer.lines.get(row)` returns `undefined`. The xterm.js version on current main (6.1.0-beta.288) still ships the vulnerable code. Once the upstream fix lands, bump `@xterm/xterm` in VS Code's `package.json`.
