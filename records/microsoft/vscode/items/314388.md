# Issue #314388 — Runtime errors when prompt/agent references model or agent that doesn't exist

[https://github.com/microsoft/vscode/issues/314388](https://github.com/microsoft/vscode/issues/314388)

## Summary

Legitimate feature request: the prompt validator already surfaces unknown model/agent references as editor diagnostics at edit time, but the user wants a runtime error or warning before execution begins so they don't silently fall back to the wrong model and waste tokens.

## Change summary

When a prompt or custom agent references a model or agent name that doesn't exist, VS Code should produce a visible runtime error rather than silently falling back to the currently selected model.

## Best solution

At request submission time (in chatServiceImpl or the prompt resolution path), check whether the resolved model/agent references are valid and surface a user-visible warning or error in the chat response area before sending tokens to the backend. The edit-time diagnostics in `promptValidator.ts` already perform this lookup — the runtime path should reuse the same resolution logic.
