# Issue #314388 — Runtime errors when prompt/agent references model or agent that doesn't exist

[https://github.com/microsoft/vscode/issues/314388](https://github.com/microsoft/vscode/issues/314388)

## Summary

This is a valid feature request for stricter validation/erroring in chat prompt and custom-agent metadata, not a cleanup close. It should be routed to the Chat area because the current prompt syntax defines `model`/`agent` metadata and appears to intentionally fall back or choose available entries rather than fail hard.

## Change summary

Add a runtime or pre-run error when a prompt/custom agent references an unavailable model or agent instead of silently using the currently selected/default fallback.

## Best solution

Have the Chat prompt/customization owners decide whether unresolved `model` or `agent` frontmatter should produce diagnostics, a pre-run warning, or a hard runtime error, likely in the prompt syntax/service path that parses and applies prompt metadata.
