# Issue #239271 — Potential leak for WatchingProblemCollector

[https://github.com/microsoft/vscode/issues/239271](https://github.com/microsoft/vscode/issues/239271)

## Summary

Fixed on main: the listener leak in WatchingProblemCollector was caused by calling this._register() inside the onModelRemoved callback (which fires per model removal), accumulating disposables. Fixed by removing the register call and relying on the existing setTimeout-based disposal.

## Change summary

WatchingProblemCollector leaks listeners because a debounced event subscription is registered via this._register() inside a repeatedly-fired callback.

## Best solution

Keep the shipped fix in commit a95abce2b35 (1.110.1); no further change needed.
