# Issue #144767 — [scss] variables in @media parsing

[https://github.com/microsoft/vscode/issues/144767](https://github.com/microsoft/vscode/issues/144767)

## Summary

Still reproduces on main. The SCSS parser's `_parseMediaCondition` override only handles interpolation at the top level of a media condition, not when it appears as a subsequent `<media-in-parens>` after `and` within the CSS parser's while loop. The bug lives in the upstream `vscode-css-languageservice` package.

## Change summary

SCSS interpolation (`#{...}`) in a compound `@media` query (e.g. `(max-width: 735px) and #{breakpoint(medium)}`) produces a spurious `css-lparentexpected` error.

## Best solution

Fix in `vscode-css-languageservice`: the SCSS parser needs to override or hook into the inner loop of `_parseMediaCondition` (in `cssParser.ts`) so that each `<media-in-parens>` position also tries `_parseInterpolation()` before requiring a `(`. Track upstream at https://github.com/microsoft/vscode-css-languageservice/issues/157.
