# Issue #144767 — [scss] variables in @media parsing

[https://github.com/microsoft/vscode/issues/144767](https://github.com/microsoft/vscode/issues/144767)

## Summary

This is a known limitation in the vscode-css-languageservice SCSS parser around interpolation in @media queries. The upstream fix from 2019 (issue #157) didn't cover all edge cases, and multiple related interpolation issues remain open upstream (#412, #375, #269, #419). The bug is correctly labeled and placed in the Backlog.

## Change summary

SCSS `#{...}` interpolation after `and` in `@media` queries produces a false `css-lparentexpected` parsing error.

## Best solution

Fix should be made in `microsoft/vscode-css-languageservice` SCSS parser to handle interpolation expressions in additional @media query positions. Once fixed upstream, the dependency bump in `extensions/css-language-features/server/package.json` brings it to VS Code.
