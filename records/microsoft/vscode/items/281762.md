# Issue #281762 — Language model settings from GitHub.com not syncing to VS Code

[https://github.com/microsoft/vscode/issues/281762](https://github.com/microsoft/vscode/issues/281762)

## Summary

The model-availability syncing from GitHub.com is handled entirely by the GitHub Copilot extension's communication with GitHub's backend API — VS Code core provides only the language-model registration infrastructure (`ILanguageModelsService`) and the 'Manage Language Model Access' UI shell. The second commenter confirmed the issue self-resolved after 12–15 hours, pointing to a backend propagation delay on GitHub's side rather than a VS Code core defect.

## Change summary

Language models enabled/disabled on GitHub.com do not immediately appear in VS Code's Copilot Chat model picker; syncing takes hours and varies between users.

## Best solution

This belongs on the Copilot extension or GitHub backend side — the VS Code core `ILanguageModelsService` only surfaces models that extensions register. If the delay persists, a bug should be filed against the Copilot extension or GitHub's Copilot entitlement API, not VS Code core.
