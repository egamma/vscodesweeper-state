# Issue #251934 — Splitting primary/secondary sidebars by dragging the tool's icon is impossible in certain situations

[https://github.com/microsoft/vscode/issues/251934](https://github.com/microsoft/vscode/issues/251934)

## Summary

Valid inconsistency in sidebar DnD splitting behavior — the drop overlay for splitting a pane appears only when the target container accepts added views, but extension-contributed containers that should accept them sometimes don't show the overlay. Already triaged in Backlog with correct area label.

## Change summary

Dragging a tool icon onto certain view containers in the sidebar does not show the split drop overlay, even though those containers don't explicitly reject added views.

## Best solution

Investigate why some extension-contributed containers (like Roo, Cline) don't show the ViewPaneDropOverlay during drag despite not having `rejectAddedViews: true`. The issue likely lives in `src/vs/workbench/browser/parts/views/viewPaneContainer.ts` around the drop target registration or in how single-view webview containers expose their pane drop targets.
