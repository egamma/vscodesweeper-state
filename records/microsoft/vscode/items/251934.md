# Issue #251934 — Splitting primary/secondary sidebars by dragging the tool's icon is impossible in certain situations

[https://github.com/microsoft/vscode/issues/251934](https://github.com/microsoft/vscode/issues/251934)

## Summary

Valid bug: the `rejectAddedViews` guard on view containers prevents the drop overlay from showing at all, which blocks sidebar splitting even when the intent is to create a new adjacent container rather than add views into the target. Already triaged to Backlog with correct area label.

## Change summary

Dragging a view onto a view container with `rejectAddedViews: true` blocks sidebar splitting entirely, even though creating a new adjacent container doesn't violate the reject-added-views constraint.

## Best solution

Separate the 'add views into this container' guard from the 'split the sidebar area by creating a new container next to this one' operation in `src/vs/workbench/browser/parts/views/viewPaneContainer.ts`. The drop overlay should still appear for sidebar-level splitting even when the target container has `rejectAddedViews: true`.
