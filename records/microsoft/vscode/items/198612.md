# Issue #198612 — Code cannot be read and edited using a screen reader in Android web version of VS code.

[https://github.com/microsoft/vscode/issues/198612](https://github.com/microsoft/vscode/issues/198612)

## Summary

TalkBack (Android screen reader) cannot read or edit code in VS Code for Web — a known unsupported scenario per maintainer comment, but kept open for feedback and future consideration as an accessibility gap.

## Change summary

Screen reader users on Android (TalkBack) cannot read or edit code in the web version of VS Code because the custom editor input does not expose content to TalkBack.

## Best solution

Keep open as a long-term accessibility tracking item. The editor's custom textarea/EditContext input mechanism doesn't interoperate with TalkBack; a fix would require significant investment in mobile screen reader support in the editor input layer (src/vs/editor/browser/controller/editContext/). A maintainer already decided to keep this open for community signal.
