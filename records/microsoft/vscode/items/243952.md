## Summary

Team-authored crash report (MEMBER) related to the Electron 34 update — pty.node throws after the JS environment has shut down during utility process cleanup. Hard keep-open: team member issue requiring explicit maintainer judgment.

## Change summary

pty.node crashes by throwing a JavaScript exception after the Node/JS environment has already been torn down during utility process shutdown.

## Best solution

The fix likely lives in node-pty's native addon layer: the ThreadSafeFunction callback must check whether the JS environment is still alive before calling Napi::Error::ThrowAsJavaScriptException(). This may need coordination with the node-pty upstream or a guard in the Electron/Node bindings shutdown sequence, especially in the context of the Electron 34 update.
