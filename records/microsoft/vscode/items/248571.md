# Issue #248571 — Bottom Panel toolbar doesn't collapse to maintain rightmost actions

[https://github.com/microsoft/vscode/issues/248571](https://github.com/microsoft/vscode/issues/248571)

## Summary

Team-authored bug that remains valid on main: the bottom panel toolbar does not use the responsive toolbar infrastructure (added in commits 6470f9b8835 and 16cfd3b6d4a) to collapse gracefully, so actions still overflow when the panel is narrowed.

## Change summary

Bottom panel toolbar actions overflow past the toolbar width when the viewport is narrow, instead of collapsing into an overflow menu while preserving rightmost actions.

## Best solution

Apply the `responsiveBehavior` option (with appropriate `minItems` to preserve Hide Panel / Clear Console) to the panel's toolbar in `src/vs/workbench/browser/parts/compositePart.ts` or `paneCompositePart.ts`, similar to how the SCM toolbar uses it.
