## Summary

Active regression still hitting 50 users in stable 1.126.0. The core `isExpectedError` suppression mechanism landed (commit 3727d0cef92), but the Copilot extension apparently hasn't marked this auth error as expected yet, so it continues appearing in telemetry. The root cause (account policy gate from PR #311487) may also be causing legitimate auth failures during transient states.

## Change summary

Users see 'Authorization failed. Please sign into GitHub and try again' errors in Agents/Copilot chat sessions, regressing after the Account Policy Gate feature landed.

## Best solution

The Copilot extension should either (1) mark this auth error as a `ChatExpectedError` so it's suppressed from telemetry (if it's a user-actionable sign-in prompt), or (2) fix the underlying race where the policy gate disrupts auth state during transient boot conditions. The mechanism is ready in `src/vs/workbench/api/browser/mainThreadChatAgents2.ts`; the extension-side fix belongs in `copilotCLIChatSessionsContribution.ts`.
