# Issue #305758 — Model picker does nothing when opened from the ellipsis menu

[https://github.com/microsoft/vscode/issues/305758](https://github.com/microsoft/vscode/issues/305758)

## Summary

Still open and unfixed on main. The issue has milestone 'On Deck' (hard keep-open rule). The bug appears to stem from how `OpenModelPickerAction` resolves the widget via `lastFocusedWidget` — when triggered from the overflow/ellipsis menu, the model selection may not commit to the correct widget instance.

## Change summary

Selecting a model via the chat panel's ellipsis overflow menu does not apply the selection; the previously active model remains unchanged.

## Best solution

Fix the model selection commit path when the picker is invoked from the toolbar overflow menu. The likely root cause is in `OpenModelPickerAction.run()` (src/vs/workbench/contrib/chat/browser/actions/chatExecuteActions.ts) or how the model widget's `setModel` callback connects when rendered in the overflow context. Ensure the picker's delegate properly commits when opened indirectly.
