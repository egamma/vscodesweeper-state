# Issue #286974 — Terminal tabs description/title using cwdFolder shows folder incorrectly in WSL

[https://github.com/microsoft/vscode/issues/286974](https://github.com/microsoft/vscode/issues/286974)

## Summary

Still reproduces on main: the cwdFolder comparison uses `path.resolve()` which applies the frontend's path module (Windows backslash) when the backend is WSL (Linux forward-slash), causing a false mismatch and showing the folder name when it shouldn't. The fix PR was closed without merging.

## Change summary

Terminal tab description incorrectly shows cwdFolder in WSL because the cwd comparison uses the frontend path module instead of comparing backend-native paths directly.

## Best solution

Land the fix from the closed PR #286985: compare `templateProperties.cwd` against `instance.workspaceFolder.uri.path` directly instead of constructing a URI with `path.resolve()`, avoiding the frontend/backend path-format mismatch. The change is in `src/vs/workbench/contrib/terminal/browser/terminalInstance.ts`.
