# Issue #316790 — Random chartacter on each command

[https://github.com/microsoft/vscode/issues/316790](https://github.com/microsoft/vscode/issues/316790)

## Summary

The ^U character is intentionally sent by the terminal agent execute strategies to clear pending input, but PowerShell on Windows echoes it as a literal character rather than interpreting it as a kill-line control sequence, causing commands to fail.

## Change summary

Copilot agent mode prepends a visible ^U character to terminal commands on PowerShell/Windows, causing command execution failures.

## Best solution

Guard the Ctrl+U send in noneExecuteStrategy.ts and basicExecuteStrategy.ts to use a PowerShell-appropriate clear mechanism (e.g. Escape or Home+Shift+End+Delete) when the shell is PowerShell, or skip it entirely when shell integration confirms a clean prompt.
