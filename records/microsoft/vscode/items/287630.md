# Issue #287630 — Chat took too long to get ready

[https://github.com/microsoft/vscode/issues/287630](https://github.com/microsoft/vscode/issues/287630)

## Summary

The slow chat readiness is caused by network connectivity timeouts in the GitHub Copilot Chat extension — the logs show ConnectTimeoutError to api.githubcopilot.com:443, causing ~35s activation delays. This is a user-side network issue, not a VS Code core defect.

## Change summary

Chat panel took ~35 seconds to become ready due to network timeouts connecting to the Copilot API.

## Best solution

The user's network is timing out reaching api.githubcopilot.com. They should check firewall/proxy settings or report to the Copilot Chat extension repo (https://github.com/microsoft/vscode-copilot-release) if they believe the extension's timeout/retry behavior should be improved.
