# Issue #169117 — [html] "Cannot redeclare block-scoped variable" in script type module

[https://github.com/microsoft/vscode/issues/169117](https://github.com/microsoft/vscode/issues/169117)

## Summary

Still reproduces on main: the embedded JS document builder concatenates all script regions into a single scope, so `type="module"` scripts incorrectly share declarations. Maintainer confirmed the fix direction (wrap module script regions in `{`/`}`).

## Change summary

Multiple `<script type="module">` blocks erroneously report "Cannot redeclare block-scoped variable" because the HTML language server treats them as one shared scope.

## Best solution

In `extensions/html-language-features/server/src/modes/embeddedSupport.ts`, track whether an `EmbeddedRegion` came from a `type="module"` script tag, then emit `{` / `}` as prefix/suffix in `getPrefix`/`getSuffix` for those regions so the synthesized JS document gives each module block its own scope.
