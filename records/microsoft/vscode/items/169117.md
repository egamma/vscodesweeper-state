# Issue #169117 — [html] "Cannot redeclare block-scoped variable" in script type module

[https://github.com/microsoft/vscode/issues/169117](https://github.com/microsoft/vscode/issues/169117)

## Summary

Still reproduces on main: all script regions are concatenated into one virtual JS document without scope isolation, so TypeScript sees duplicate declarations across module scripts. The fix location is identified and a maintainer has confirmed the approach.

## Change summary

Multiple `<script type="module">` blocks in HTML incorrectly share scope, causing false 'Cannot redeclare block-scoped variable' errors.

## Best solution

Track `isModule` on `EmbeddedRegion` when `type="module"` is detected, then emit `{`/`}` wrappers in `getPrefix`/`getSuffix` for module script regions so TypeScript treats each as a separate block scope. The fix is in `extensions/html-language-features/server/src/modes/embeddedSupport.ts`.
