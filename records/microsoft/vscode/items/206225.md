# Issue #206225 — SCM - Tree view versioned directories decorations ignored

[https://github.com/microsoft/vscode/issues/206225](https://github.com/microsoft/vscode/issues/206225)

## Summary

Still reproduces on main. The `renderCompressedElements` method in `ResourceRenderer` unconditionally strips all decorations (faded, strike-through, decoration icon) from folder nodes without checking whether the folder node carries an `element` (a real `SourceControlResourceState`). Already triaged with `help wanted`; the fix is localized.

## Change summary

In SCM tree view, directory-type `SourceControlResourceState` elements lose their decorations (icon, faded, strikethrough) because the tree renderer unconditionally clears decorations on folder nodes.

## Best solution

In `ResourceRenderer.renderCompressedElements` (src/vs/workbench/contrib/scm/browser/scmViewPane.ts ~line 582), check whether the last compressed element's `folder.element` is defined and, if so, apply its decorations (icon, faded, strikethrough) the same way `renderElement` does at lines 544-549. The `help wanted` label is appropriate — this is a good community contribution candidate.
