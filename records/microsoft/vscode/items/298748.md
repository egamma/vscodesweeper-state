# Issue #298748 — tunnel service does not persist after reboot on MacOS

[https://github.com/microsoft/vscode/issues/298748](https://github.com/microsoft/vscode/issues/298748)

## Summary

Still reproduces on main: the macOS tunnel service plist is written to the home directory root (~/) instead of ~/Library/LaunchAgents/, so launchd does not reload it after reboot. No fix has been committed.

## Change summary

The tunnel service installed via `code tunnel service install` does not persist after macOS reboot because the LaunchAgent plist is placed in the wrong directory.

## Best solution

Fix `get_service_file_path()` in `cli/src/tunnels/service_macos.rs` to write the plist to `~/Library/LaunchAgents/` (creating the directory if needed) instead of the home directory root. This is the standard location macOS scans at user login for LaunchAgents.
