# Issue #256181 — Feature Request: Mobile-Friendly Layout for vscode.dev

[https://github.com/microsoft/vscode/issues/256181](https://github.com/microsoft/vscode/issues/256181)

## Summary

Legitimate feature request with strong community support (41 upvotes, promoted to Backlog). Asks for mobile/touch-optimized layout in vscode.dev — no existing capability covers this; the codebase has no responsive/mobile layout mode. Keeping open as a valid backlog item.

## Change summary

Make vscode.dev usable on phones and tablets with larger touch targets, swipe-based navigation, resizable panels, and an optional minimal layout for small screens.

## Best solution

Invest in responsive layout breakpoints and touch-friendly interaction in the web workbench (src/vs/workbench/browser/layout/). A lightweight 'mobile mode' activated by viewport size detection on vscode.dev would address the core ask. Given current agent-experience priorities, a focused mobile supervisor view (as described in https://github.com/microsoft/vscode/issues/302363) could be a pragmatic first step.
