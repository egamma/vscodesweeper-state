# Issue #309735 — Copilot Chat: flood of ENOPRO errors after agentic edit session ends (chat-editing-snapshot-text-model provider not cleaned up)

[https://github.com/microsoft/vscode/issues/309735](https://github.com/microsoft/vscode/issues/309735)

## Summary

Legitimate unfixed bug: after a chat editing session ends, stale URIs with `chat-editing-snapshot-text-model` and `chat-editing-text-model` schemes are resolved via IFileService (which has no provider for them) rather than ITextModelService (which does). The text model content providers are properly registered and return null for missing sessions, but something is routing resolution through the wrong service path, producing a flood of ENOPRO errors that loads the extension host.

## Change summary

After a Copilot Chat agentic edit session completes, repeated ENOPRO file system errors flood the extension host for stale chat-editing URIs, contributing to renderer freezes.

## Best solution

Investigate what holds references to `chat-editing-*` URIs after session disposal (likely undo/redo stack or document reference tracking) and either: (1) ensure those references are cleaned up on session end, or (2) register a no-op file system provider for these schemes that returns gracefully instead of throwing. The fix likely touches `src/vs/workbench/contrib/chat/browser/chatEditing/chatEditingSession.ts` (disposal/cleanup) or the undo stack integration.
