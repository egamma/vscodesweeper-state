# Issue #137504 — Bracket Pair Colorization for HTML tags

[https://github.com/microsoft/vscode/issues/137504](https://github.com/microsoft/vscode/issues/137504)

## Summary

Well-supported feature request (79 upvotes, well above the 20-upvote Backlog threshold) for HTML tag bracket pair colorization. Not implemented on main — the bracket pair system still doesn't natively handle HTML angle brackets as paired brackets with nesting awareness. Keeping open for the community process.

## Change summary

Colorize HTML/XML angle brackets (`<`, `</`, `/>`) by nesting level, similar to how curly braces and parentheses are colorized by the built-in bracket pair colorization.

## Best solution

Extend the bracket pair colorization engine to understand HTML/XML tag structure — recognizing `<tag>...</tag>` as bracket pairs and colorizing the angle brackets/slash by nesting depth. This likely requires language-aware bracket pair detection beyond the current token-based matching, touching `src/vs/editor/common/model/bracketPairsTextModelPart/`. The workaround of adding `["<", "</"]` to `colorizedBracketPairs` is brittle and doesn't handle self-closing or void elements correctly.
