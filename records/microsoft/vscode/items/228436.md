# Issue #228436 — Editor scrolls horizontally with wrapping enabled

[https://github.com/microsoft/vscode/issues/228436](https://github.com/microsoft/vscode/issues/228436)

## Summary

The bug is plausible and still present on main: when viewport wrapping is enabled, the scroll-width computation in `viewLayout.ts` can still produce a `contentWidth` that exceeds the viewport width — especially when the minimap is on the right — triggering an unwanted horizontal scrollbar. No fix has landed since the report.

## Change summary

When word wrap is enabled, an unwanted horizontal scrollbar appears due to the vertical scrollbar width and/or minimap width not being fully accounted for in the scroll-width computation.

## Best solution

The `getContentWidth()` method in `src/vs/editor/common/viewLayout/viewLayout.ts` (lines 313–334) should never return a value exceeding the viewport width when viewport wrapping is active and no line genuinely overflows (the current guard at line 321 adds scrollbar width when a line exceeds content width, but the base return at line 328 can still exceed the viewport). The fix likely also involves the wrapping-column calculation in `editorOptions.ts` line 2971 to ensure the scrollbar/minimap gap is harmonized with the content-width signal.

## Evidence

- **wrapping column calc:** src/vs/editor/common/config/editorOptions.ts line 2971: viewportColumn = (contentWidth - verticalScrollbarWidth - 2) / typicalHalfwidthCharacterWidth; this subtraction of verticalScrollbarWidth should prevent overflow, but the contentWidth used here (line 2968) already excludes the minimap, so the arithmetic can still leave a small overflow when the minimap is on the right.
