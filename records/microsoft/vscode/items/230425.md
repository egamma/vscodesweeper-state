# Issue #230425 — Cannot set working directory when opening a new terminal when using vscode remotes (WSL and SSH)

[https://github.com/microsoft/vscode/issues/230425](https://github.com/microsoft/vscode/issues/230425)

## Summary

Still reproduces per the reporter's January 2026 comment (v1.108.2). The terminal cwd folder picker selection is ignored when connected to WSL or SSH remotes in a multi-root workspace — the terminal opens in the home directory instead. No fix found on current main. Keeping open as a confirmed, active bug with the existing area labels.

## Change summary

In a multi-root workspace over WSL/SSH remotes, selecting a working directory from the folder picker when creating a new terminal is ignored — the terminal opens in the home directory instead.

## Best solution

Investigate why the cwd URI from `pickTerminalCwd` (in `terminalActions.ts`) is not being honored by the remote terminal backend when creating a new terminal in a WSL/SSH remote session. The flow through `_resolveCwd` → `shellLaunchConfig.cwd` looks correct on the client side; the issue likely lies in how the cwd is transmitted to or applied by the remote pty host. Related but distinct from https://github.com/microsoft/vscode/issues/142520 (which involves `terminal.integrated.cwd` setting concatenation).
