# Issue #283714 — Past chat sessions can't be opened after transitioning to a multi-root workspace

[https://github.com/microsoft/vscode/issues/283714](https://github.com/microsoft/vscode/issues/283714)

## Summary

A fix for this landed in commit 7393b0ec607 (shipped in 1.110.1), which migrates chat session data when the workspace ID changes (e.g. single-folder → multi-root transition). However, the author is a MEMBER, so this stays open for explicit maintainer judgment — likely ready to close as completed.

## Change summary

Chat sessions saved in a single-folder workspace become inaccessible after transitioning to a multi-root workspace because the chat index migrates but the on-disk content does not.

## Best solution

The fix in commit 7393b0ec607 added workspace-ID migration logic to ChatSessionStore so sessions carry over when the workspace identity changes. This shipped in 1.110.1. A maintainer should verify the fix covers the scenario and close as completed.
