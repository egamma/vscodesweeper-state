# Issue #242891 — Killing tasks does not actually kill them

[https://github.com/microsoft/vscode/issues/242891](https://github.com/microsoft/vscode/issues/242891)

## Summary

The root cause — the legacy conpty path not killing child processes — was resolved by enabling `terminal.integrated.windowsUseConptyDll` by default in commit cfebe6aeafb (shipped in 1.121.0). Multiple commenters confirmed the setting fixed the issue; the team then shipped it as the default.

## Change summary

Killing tasks on Windows does not actually terminate the underlying process, leaving ports occupied.

## Best solution

Keep the shipped default (`windowsUseConptyDll: true` since 1.121.0); this resolves the Windows kill-task regression. The macOS report in comment 27 may be a separate issue and should be filed independently if it persists.
