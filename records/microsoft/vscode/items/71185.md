# Issue #71185 — Accessibility service not firing onDidChangeAccessibilitySupport

[https://github.com/microsoft/vscode/issues/71185](https://github.com/microsoft/vscode/issues/71185)

## Summary

Team-authored upstream issue: Chromium does not fire an event when assistive technology is disabled on Windows. VS Code still relies on this Electron/Chromium event (app.ts:466), so the limitation persists. Keeping open as it requires an upstream fix.

## Change summary

Turning off a screen reader on Windows does not revert VS Code's accessibility mode until a full restart, because Chromium never fires the disablement event.

## Best solution

Upstream fix in Chromium (https://bugs.chromium.org/p/chromium/issues/detail?id=1015906) to detect assistive technology disablement on Windows. Alternatively, VS Code could implement a polling fallback or a manual toggle, but the canonical fix lives in Chromium/Electron.
