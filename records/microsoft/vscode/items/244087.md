# Issue #244087 — Browser Cursor and VSCode Cursor Overlapping Issue

[https://github.com/microsoft/vscode/issues/244087](https://github.com/microsoft/vscode/issues/244087)

## Summary

Still reproduces on main: the editor textarea lacks `caret-color: transparent` in its base state, so iPad Safari renders its native caret alongside VS Code's custom cursor. Two community PRs attempted fixes but neither was merged.

## Change summary

On iPad Safari, both the browser's native text caret and VS Code's custom cursor are visible simultaneously in the editor.

## Best solution

Add `caret-color: transparent` to the `.monaco-editor .inputarea` base rule in `src/vs/editor/browser/controller/editContext/textArea/textAreaEditContext.css`, keeping the existing `caret-color: var(--vscode-editorCursor-foreground)` override in the `.ime-input` state so IME composition still shows a caret.
