# Issue #283871 — Regression: requesterUsername and requesterAvatarIconUri missing from exported chat JSON since VS Code 1.107.0

[https://github.com/microsoft/vscode/issues/283871](https://github.com/microsoft/vscode/issues/283871)

## Summary

The removal of `requesterUsername` and `requesterAvatarIconUri` from exported chat JSON was an intentional change in PR #277233 ('Remove requester info from API'), not a regression. These fields were deliberately stripped from `IExportableChatData` and the `toExport()` method as part of an API cleanup that shipped in 1.107.0.

## Change summary

Exported chat JSON no longer includes `requesterUsername` and `requesterAvatarIconUri` fields after the intentional removal in VS Code 1.107.0.

## Best solution

Keep the current behavior. The requester info removal was a deliberate API cleanup (commit 13b1a4e793e, PR https://github.com/microsoft/vscode/pull/277233). External tools consuming exported chat JSON should adapt to the new schema — the requester identity is no longer part of the export contract.
