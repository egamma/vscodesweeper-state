# Issue #130100 — Git - Commit does not work if there are git hooks from husky and lint-staged

[https://github.com/microsoft/vscode/issues/130100](https://github.com/microsoft/vscode/issues/130100)

## Summary

Still a valid issue: when VS Code is launched from a desktop launcher (not a terminal), the git extension inherits a stripped-down process.env that lacks user shell customizations (nvm, yarn paths), causing husky/lint-staged hooks to fail with 'command not found'. VS Code resolves the shell environment at startup (src/vs/code/electron-main/app.ts:1546) but the git extension simply uses `process.env` (extensions/git/src/git.ts:689) and the resolved shell env may not fully propagate. Low upvote count but common scenario across platforms.

## Change summary

Git commits fail when husky/lint-staged hooks require tools (node, yarn, npx) that are set up via shell init files (nvm, etc.) not present in VS Code's process environment.

## Best solution

Ensure the git extension's child-process environment includes the resolved shell environment from the shell environment service, particularly PATH enrichment from user shell init files. The fix would be in `extensions/git/src/git.ts` around line 689 where options.env is assembled, ensuring it benefits from the resolved shell environment rather than only raw `process.env`. Alternatively, improve the error message to surface the actual stderr from the hook so users can diagnose the PATH issue themselves (the reporter's original suggestion in comment 5).
