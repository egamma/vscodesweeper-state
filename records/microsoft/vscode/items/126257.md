# Issue #126257 — function "setBreakPointsRequest" in the Debug Adapter is triggered to much

[https://github.com/microsoft/vscode/issues/126257](https://github.com/microsoft/vscode/issues/126257)

## Summary

Still reproduces on main: `$unregisterBreakpoints` in mainThreadDebugService.ts calls `removeBreakpoints(id)` individually per breakpoint ID instead of passing the array, causing one `setBreakpointsRequest` per breakpoint per source file rather than one per source file total.

## Change summary

Calling `vscode.debug.removeBreakpoints()` with multiple breakpoints fires one `setBreakpointsRequest` per breakpoint instead of batching them by source file.

## Best solution

Change `mainThreadDebugService.ts` line 249 to call `this.debugService.removeBreakpoints(breakpointIds)` once with the full array (the method already accepts `string[]`), and do the same for function/data breakpoints. This collapses N calls into one per source file.
