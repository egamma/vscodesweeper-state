# Issue #61877 — Git - VS Code no longer supports 'edit' in 'git rebase' flow

[https://github.com/microsoft/vscode/issues/61877](https://github.com/microsoft/vscode/issues/61877)

## Summary

Still reproduces on main: when a rebase is in `edit` mode, VS Code unconditionally routes commits through `rebaseContinue()` instead of allowing a normal `git commit`. The author is a MEMBER, so this stays open for maintainer judgment.

## Change summary

During `git rebase -i` with `edit` stops, VS Code's Source Control pane forces `git rebase --continue` instead of allowing normal `git commit`, blocking the intended edit workflow.

## Best solution

The `commit()` method in `extensions/git/src/repository.ts` (line ~1426) needs to distinguish between a rebase waiting for conflict resolution (where `rebase --continue` is correct) and an `edit` stop (where the user should be able to make normal commits). Detect the `edit` state via `.git/rebase-merge/amend` or `git status` output and allow normal commits in that case, offering a separate 'Continue Rebase' action.
