# Issue #193108 — Git - "Git: connect ENOENT" error 

[https://github.com/microsoft/vscode/issues/193108](https://github.com/microsoft/vscode/issues/193108)

## Summary

Still reproducible on main: the git extension's IPC server creates a Unix socket once at activation and never monitors its health or recreates it. When the socket file is removed (OS temp cleanup, sleep/wake cycles, extension host restart), subsequent git editor operations fail with ENOENT. No fix has been committed.

## Change summary

Git commit operations (especially amend) fail with 'connect ENOENT …vscode-git-*.sock' when the IPC socket file becomes stale after sleep/wake, temp cleanup, or extension host restarts.

## Best solution

Add socket health monitoring to `IPCServer` in `extensions/git/src/ipc/ipcServer.ts` — either periodically verify the socket file still exists and recreate it, or detect the ENOENT in the client (`ipcClient.ts`) and trigger a server-side socket refresh. The environment variables (`VSCODE_GIT_IPC_HANDLE`) propagated to terminals also need updating when the socket path changes. This is a localized fix in the git extension's IPC layer.
