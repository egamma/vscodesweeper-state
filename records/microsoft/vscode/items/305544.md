# Issue #305544 — 1.113.0 regression: unbounded ripgrep processes freeze macOS when worktrees exist

[https://github.com/microsoft/vscode/issues/305544](https://github.com/microsoft/vscode/issues/305544)

## Summary

Team-authored issue; the merged PR #316349 added an experimental opt-in setting but the underlying default behavior (--no-ignore in findFiles) is unchanged, and the setting remains experimental — the issue is not fully resolved. Keeping open per policy and pending finalization of the fix.

## Change summary

Unbounded ripgrep processes spawn and freeze macOS when workspaces contain git worktrees, due to findFiles API defaulting to --no-ignore.

## Best solution

Finalize `search.experimental.useIgnoreFilesInFindFiles` into a stable setting (remove the `experimental` prefix) and decide on the correct default (likely `true`) so extensions honoring .gitignore is the standard behavior rather than opt-in. The core issue in `src/vs/workbench/api/common/extHostWorkspace.ts` needs the default flipped or the setting graduated.
