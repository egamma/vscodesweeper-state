# Issue #288779 — FileLoggerService logger can't increase verbosity

[https://github.com/microsoft/vscode/issues/288779](https://github.com/microsoft/vscode/issues/288779)

## Summary

Confirmed on current main: BufferLogger.setLevel() does not propagate log level changes down to its delegate FileLogger, so increasing verbosity at runtime has no effect on the actual file logger output.

## Change summary

Changing a file-backed output channel's log level to trace at runtime doesn't take effect because BufferLogger only propagates level changes upward (delegate→buffer) but not downward (buffer→delegate).

## Best solution

Override `setLevel` in `BufferLogger` (src/vs/platform/log/common/bufferLog.ts) to also call `this._logger?.setLevel(level)` when the delegate is already set, mirroring the existing upward propagation added in commit 2c08afede60.
