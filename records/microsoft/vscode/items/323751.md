# Issue #323751 — Copilot agent session worktree creation silently fails when repository is in detached HEAD state

[https://github.com/microsoft/vscode/issues/323751](https://github.com/microsoft/vscode/issues/323751)

## Summary

Confirmed on current main: the overly strict guard at line 115 of chatSessionWorktreeServiceImpl.ts discards a successfully created worktree when in detached HEAD state because headBranchName is undefined, even though it is only needed as a fallback.

## Change summary

Copilot agent session worktree creation silently fails in detached HEAD state because the success guard requires headBranchName, which is undefined when HEAD is detached.

## Best solution

Relax the guard on line 115 of `extensions/copilot/src/extension/chatSessions/vscode-node/chatSessionWorktreeServiceImpl.ts` so that `headBranchName` is only required when `baseBranch` was not provided — e.g. `if (worktreePath && activeRepository.headCommitHash && (baseBranch || activeRepository.headBranchName))`. This preserves the fallback logic on line 116 while allowing detached-HEAD worktrees to succeed when an explicit baseBranch is supplied.
