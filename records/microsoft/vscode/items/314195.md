# Issue #314195 — Make fishtank activation button stop wiggling after I've clicked on it

[https://github.com/microsoft/vscode/issues/314195](https://github.com/microsoft/vscode/issues/314195)

## Summary

Team-authored polish request (hard keep-open). The aquarium toggle button's wiggle animation repeats indefinitely via CSS whenever the button is in the not-active state; the request is to suppress it once the user has clicked the button at least once, so it only serves as a discovery hint.

## Change summary

The fishtank/aquarium activation button wiggles on a 12-second CSS animation loop even after the user has already discovered and clicked it; the wiggle should stop after the first interaction.

## Best solution

Persist a 'has interacted' flag (e.g. in StorageService under application scope) and add a CSS class like `discovered` to the toggle button after the first click, then scope the wiggle animation to `.agents-aquarium-toggle:not(.active):not(.discovered)` so it only plays for users who haven't yet found the button.
