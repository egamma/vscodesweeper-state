# Issue #203983 — SCM Input - steals focus

[https://github.com/microsoft/vscode/issues/203983](https://github.com/microsoft/vscode/issues/203983)

## Summary

Still reproduces on main — the SCM input unconditionally steals focus via `input.onDidChangeFocus(() => this.focus())` in scmInput.ts:453. Team-authored issue on milestone 'On Deck'; keeping open.

## Change summary

After a commit with a slow hook, the SCM input editor steals focus from whatever the user switched to (e.g. the command palette).

## Best solution

Guard the `onDidChangeFocus` handler in `scmInput.ts` so it only calls `this.focus()` when the SCM view already has focus (or the user explicitly requested it), rather than unconditionally stealing focus from other UI surfaces. The fix is localized to `src/vs/workbench/contrib/scm/browser/scmInput.ts` around line 453.
