[https://github.com/microsoft/vscode/issues/138546](https://github.com/microsoft/vscode/issues/138546)

## Summary

Feature request for Go to Definition from JSON properties to their schema definitions. Not implemented on current main; the JSON extension still has no DefinitionProvider. With only 5 upvotes it hasn't reached the 20-upvote promotion threshold, but a team member endorsed the idea — keep open as a valid backlog item.

## Change summary

Add a DefinitionProvider to the JSON language extension that navigates from a JSON property to its corresponding definition in the associated JSON schema file.

## Best solution

Implement a DefinitionProvider in `extensions/json-language-features` that resolves the schema location for the property under the cursor and returns a Location pointing into the schema file. The vscode-json-languageservice would need to expose schema-node mapping for this to work cleanly.
