# Issue #115214 — `context.secrets.onDidChange` fires without change

[https://github.com/microsoft/vscode/issues/115214](https://github.com/microsoft/vscode/issues/115214)

## Summary

Still reproducible on main: BaseSecretStorageService.set() does not compare the new plaintext value against the existing one before encrypting and writing, so non-deterministic encryption produces different ciphertext each time, bypassing the storage-layer dedup and firing a spurious change event.

## Change summary

Storing the same secret value twice via `context.secrets.store()` fires `onDidChange` on the second call even though the plaintext hasn't changed.

## Best solution

In `BaseSecretStorageService.set()`, read and decrypt the current value first; skip the write (and therefore the event) when the existing plaintext matches the new value. The fix is small and localized to `src/vs/platform/secrets/common/secrets.ts`.
