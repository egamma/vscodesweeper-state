# Issue #306460 — Git - Commit message template is trimmed of comments and whitespace before it is shown to the user.

[https://github.com/microsoft/vscode/issues/306460](https://github.com/microsoft/vscode/issues/306460)

## Summary

The comment stripping is intentional: VS Code's SCM input box strips `#`-prefixed lines from commit templates because it has no concept of git comment lines — the user would have to manually remove them before committing. This is a feature request to preserve the raw template, which would require the SCM input to understand git's `--cleanup` semantics.

## Change summary

Git commit message templates have `#`-comment lines and surrounding blank lines stripped before being shown in the SCM input box.

## Best solution

The current behavior was deliberately introduced in commit `f4f00621685` to avoid users accidentally committing comment lines. If the template comments are meant as instructions the user should see, this becomes a feature request to either preserve them and strip at commit time (respecting git's `commit.cleanup` config), or to render them as non-editable hints. Issue https://github.com/microsoft/vscode/issues/133770 is a related open request about `core.commentChar` support. The `*as-designed` close is appropriate, with a pointer to file a feature request if the user wants a behavioral change.
