# Issue #317578 — Delayed Python environment auto-activation can corrupt active terminal input

[https://github.com/microsoft/vscode/issues/317578](https://github.com/microsoft/vscode/issues/317578)

## Summary

The delayed terminal activation behavior is caused by the ms-python.python extension (microsoft/vscode-python), which is external to this repository. The same issue was already reported and resolved in microsoft/vscode-python#25868 by switching to the python-envs extension with shellStartup activation type.

## Change summary

Python extension's delayed virtual environment activation via sendText can inject activation commands into the user's active terminal input.

## Best solution

This belongs in the Python extension's tracker (microsoft/vscode-python). The fix is to use the newer python-envs extension with `python-envs.terminal.autoActivationType` set to `shellStartup`, which activates the environment before the shell becomes interactive. The reporter should file at https://github.com/microsoft/vscode-python/issues if the workaround doesn't resolve their case.
