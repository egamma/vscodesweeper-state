# Issue #134461 — VSCode requests bluetooth access on startup

[https://github.com/microsoft/vscode/issues/134461](https://github.com/microsoft/vscode/issues/134461)

## Summary

Still a valid upstream issue — Electron connects to BlueZ on Linux startup, and no fix has landed on VS Code's main. An open PR (https://github.com/microsoft/vscode/pull/322212) proposes a workaround via the `--disable-bluetooth` Chromium switch; a human should review that PR.

## Change summary

VS Code triggers a BlueZ/Bluetooth D-Bus activation on Linux startup because Electron unconditionally connects to the Bluetooth service.

## Best solution

Review and land (or close) the open PR https://github.com/microsoft/vscode/pull/322212, which adds `--disable-bluetooth` as a Chromium switch in `src/main.ts`. Alternatively, wait for an upstream Electron fix that makes this configurable. The VS Code-side workaround is small and localized.
