# Issue #137401 — "Keybinding" shown is misleading for Ctrl-modified keyboard layouts (like Dvorak+QWERTY)

[https://github.com/microsoft/vscode/issues/137401](https://github.com/microsoft/vscode/issues/137401)

## Summary

Valid bug for a rare keyboard layout class (Dvorak+QWERTY where Ctrl reverts to QWERTY). No fix found on main; the keybinding label and search logic doesn't account for modifier-dependent character remapping. However, reproduction is infeasible without the specific layout, and no other reporters have confirmed it — keeping open as a known niche layout edge case.

## Change summary

Keybinding labels and search results are incorrect/confusing on keyboard layouts where holding Ctrl changes the character mapping (e.g., Dvorak+QWERTY).

## Best solution

The keybinding resolver and label renderer in `src/vs/workbench/services/keybinding/` need to account for layouts where the character produced changes based on modifier state. This is a complex fix touching the keyboard layout abstraction — the layout info structures assume a static scan-code-to-character map. Likely needs a product decision on whether to support modifier-dependent layouts at the platform level.
