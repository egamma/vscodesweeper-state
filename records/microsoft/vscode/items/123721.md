# Issue #123721 — Git conflict resolution Accept Current Change "ignores" addition of a single empty line

[https://github.com/microsoft/vscode/issues/123721](https://github.com/microsoft/vscode/issues/123721)

## Summary

Confirmed on current main: the `isNewlineOnly` guard in the merge-conflict extension explicitly replaces a single-newline content with an empty string, discarding the user's intended empty line when accepting current or incoming changes.

## Change summary

Accepting current or incoming change in a git merge conflict drops a single empty line instead of preserving it.

## Best solution

Remove or rework the `isNewlineOnly` check in `extensions/merge-conflict/src/documentMergeConflict.ts` so that a single-newline content is preserved as actual content rather than collapsed to empty string. The original intent was likely to avoid leaving a trailing newline artifact, but it incorrectly eats legitimate single-empty-line diffs.
