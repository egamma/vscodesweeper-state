# Issue #156445 — Allow to merge to a resulting file that does not exist

[https://github.com/microsoft/vscode/issues/156445](https://github.com/microsoft/vscode/issues/156445)

## Summary

Still unimplemented on main: the merge editor's result file resolution (mergeEditorInputModel.ts line 305) has no fallback for non-existent files, unlike the base file which gets an empty model. The request remains valid for external merge tool integrations (e.g. Plastic SCM).

## Change summary

Allow the merge editor to accept a result file path that does not yet exist on disk, auto-creating it instead of erroring.

## Best solution

Add a catch/fallback on the result file's `createModelReference` call in `src/vs/workbench/contrib/mergeEditor/browser/mergeEditorInputModel.ts` (similar to the base-file pattern at lines 300–316) that creates the file or opens an untitled model with the associated path when the result URI doesn't resolve. The CLI-side tracking is in https://github.com/microsoft/vscode/issues/156456.
