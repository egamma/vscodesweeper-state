# Issue #274731 — Expand Thinking section pushes chat layout upwards

[https://github.com/microsoft/vscode/issues/274731](https://github.com/microsoft/vscode/issues/274731)

## Summary

Still reproduces on main: expanding a collapsed thinking section triggers _updateElementHeight → _withPersistedAutoScroll → scrollToEnd, which pushes the expanded content upward instead of growing it downward. PR #310966 attempted a fix but was closed without merging.

## Change summary

Expanding a collapsed Thinking section in chat scrolls the viewport to the bottom instead of keeping the expanded content visible in place.

## Best solution

Distinguish user-initiated collapsible toggles from programmatic/streaming height changes in ChatListWidget so that _withPersistedAutoScroll skips the scroll-to-bottom when the user manually expands a section. The closed PR #310966 proposed a CustomEvent-based approach; a simpler alternative would be a suppressAutoScroll flag set around the click handler in ChatCollapsibleContentPart. Likely files: chatListWidget.ts and the collapsible content part base class.
