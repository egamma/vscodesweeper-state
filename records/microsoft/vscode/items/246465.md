# Issue #246465 — Cannot resize or move window with Custom Title bar on Linux via MobaXterm

[https://github.com/microsoft/vscode/issues/246465](https://github.com/microsoft/vscode/issues/246465)

## Summary

The custom title bar's drag/resize relies on Electron's frameless window APIs (`-webkit-app-region: drag` and native resize handles), which don't work under X11 forwarding via MobaXterm. A team member already identified this as likely an Electron issue. Keeping open and suggesting upstream + linux labels so it's properly tracked.

## Change summary

Custom title bar cannot be dragged or resized when VS Code runs on a remote Linux host with X11 forwarded through MobaXterm.

## Best solution

File an upstream Electron issue for frameless window drag/resize under X11 forwarding, then link it here with `upstream-issue-linked`. The workaround (native title bar) is the right interim answer. VS Code's own code delegates drag/resize entirely to Electron via `-webkit-app-region: drag` CSS (in `src/vs/workbench/browser/parts/titlebar/media/titlebarpart.css`) and Electron's `BrowserWindow` frameless mode — there is nothing VS Code can fix in its own layer.
