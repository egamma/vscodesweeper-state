[https://github.com/microsoft/vscode/issues/312382](https://github.com/microsoft/vscode/issues/312382)

## Summary

Still present on main: the BugIndicatingError at diffEditorWidget.ts:406 fires when a TextModel is disposed before the DiffEditorWidget resets its model. The related fix PR #313937 was closed without merging. The milestone is 'On Deck', so this stays open for maintainer scheduling.

## Change summary

A lifecycle race causes an unhandled error when a TextModel is disposed while a DiffEditorWidget still references it.

## Best solution

Fix the disposal ordering race in the DiffEditorWidget — either by unsubscribing from the model's onWillDispose before the model can be externally disposed, or by converting the BugIndicatingError into a graceful recovery (the widget already calls setModel(null) as cleanup). The closed PR #313937 attempted the latter approach and could inform a new fix.
