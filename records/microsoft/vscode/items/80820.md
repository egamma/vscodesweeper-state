# Issue #80820 — "editor font zoom" does not persist

[https://github.com/microsoft/vscode/issues/80820](https://github.com/microsoft/vscode/issues/80820)

## Summary

Still valid: EditorZoom is a plain in-memory singleton (`src/vs/editor/common/config/editorZoom.ts`) with no persistence — the zoom level resets to 0 on restart. No work has landed to persist it, and no other open issue covers this request. The team acknowledged it (accessibility label, On Deck milestone, isidorn engagement), so it should stay open.

## Change summary

The "Increase/Decrease Editor Font Size" commands modify an in-memory zoom level that resets on restart, unlike window zoom which persists via the `window.zoomLevel` setting.

## Best solution

Persist the editor font zoom level — either by writing it to a setting (analogous to `window.zoomLevel`) or to workbench storage — so it survives a restart. The change would touch `src/vs/editor/common/config/editorZoom.ts` (make it a service or wire it to storage) and `src/vs/editor/contrib/fontZoom/browser/fontZoom.ts` (inject the storage dependency into the actions).
