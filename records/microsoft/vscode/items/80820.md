# Issue #80820 — "editor font zoom" does not persist

[https://github.com/microsoft/vscode/issues/80820](https://github.com/microsoft/vscode/issues/80820)

## Summary

Still valid on main: EditorZoom stores the zoom level in a plain in-memory field with no persistence. The issue has an 'On Deck' milestone, so it stays open per scheduling policy.

## Change summary

Editor font zoom (increase/decrease font size commands) does not persist across restarts, unlike the window zoom level which does.

## Best solution

Persist the editor font zoom level to storage (e.g. via IStorageService) so it survives restarts, consistent with how window.zoomLevel is persisted. The fix would touch `src/vs/editor/common/config/editorZoom.ts` and likely a workbench contribution that wires it to storage.
