# Issue #80820 — "editor font zoom" does not persist

[https://github.com/microsoft/vscode/issues/80820](https://github.com/microsoft/vscode/issues/80820)

## Summary

Still valid on main — editor font zoom is stored only in-memory (EditorZoom._zoomLevel) with no persistence layer. The issue has an 'On Deck' milestone assigned, so it must remain open for maintainer judgment.

## Change summary

Editor font zoom level should persist across restarts, mirroring how window zoom persists.

## Best solution

Persist EditorZoom level to user settings (or storage) on change, and restore it on startup. The fix would touch src/vs/editor/common/config/editorZoom.ts (add persistence) and src/vs/editor/contrib/fontZoom/browser/fontZoom.ts (inject a storage/config service). Accessibility label is appropriate given the use case.
