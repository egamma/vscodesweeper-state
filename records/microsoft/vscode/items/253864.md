# Issue #253864 — We should automatically accept Chat changes in certain scenarios

[https://github.com/microsoft/vscode/issues/253864](https://github.com/microsoft/vscode/issues/253864)

## Summary

Active feature request with team discussion and partial implementation. The confirmation dialog for pending chat edits still exists in the codebase (`showClearEditingSessionConfirmation`). Scenario 2 (SCM commit) was addressed by the closed https://github.com/microsoft/vscode/issues/255432, but the broader goal of auto-accepting in all listed scenarios and eliminating the modal dialog remains unimplemented.

## Change summary

Automatically accept (keep) pending chat edits when starting a new session, committing in SCM, or switching to edit mode, instead of showing a scary confirmation dialog.

## Best solution

Implement auto-accept for the remaining scenarios (new session start, edit mode switch) in `showClearEditingSessionConfirmation` / `shouldShowClearEditingSessionConfirmation` in `src/vs/workbench/contrib/chat/browser/widgetHosts/editor/chatEditorInput.ts`, gated behind the `willKeepAlive` / background-session refactoring connor4312 described — untying document modification state from 1:1 chat sessions to support global pending-change lifecycle.
