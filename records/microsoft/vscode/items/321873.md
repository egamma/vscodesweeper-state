# Issue #321873 — Internal Stop hook forces agent to continue indefinitely ("Keep working autonomously") — no setting to disable

[https://github.com/microsoft/vscode/issues/321873](https://github.com/microsoft/vscode/issues/321873)

## Summary

The reporter is describing the autopilot continuation mechanism in the Copilot Chat tool-calling loop, which nudges the agent to keep working until the task is done. This is working as designed for agent/autopilot mode, but the request for a setting to relax or disable it — especially for non-implementation conversations — is a reasonable feature request.

## Change summary

Request a user-facing setting to control or disable the internal autopilot continuation nudge that forces the agent to keep working when it tries to stop in agent mode.

## Best solution

Consider adding a configuration option (e.g. under `chat.agent.*` or `chat.autopilot.*`) to control how aggressively the autopilot continuation mechanism nudges the model. The relevant code is in `extensions/copilot/src/extension/intents/node/toolCallingLoop.ts` — the `shouldAutopilotContinue` and `advancedAutopilotContinue` methods. The standard autopilot already exits gracefully on text-only responses (line 443), but the advanced autopilot (`chat.autopilot.advanced.enabled`) delegates to a goal classifier that may not handle pure Q&A conversations well.
