# Issue #324126 — Standard (non-AI) suggestions menu doesn't trigger unless AI completions are snoozed

[https://github.com/microsoft/vscode/issues/324126](https://github.com/microsoft/vscode/issues/324126)

## Summary

This appears to be a duplicate of #318380, which tracks the same symptom: when Copilot inline suggestions are active, the standard suggest widget (completions popup) is suppressed. The underlying mechanism is `canShowQuickSuggest` in suggestModel.ts blocking quick suggestions when `inlineSuggestionVisible` is true and the Copilot extension sets `suppressSuggestions` on its inline completions.

## Change summary

The standard completions popup (suggest widget) does not appear while Copilot inline suggestions are active, only appearing when Copilot is snoozed.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/318380, which tracks the same interaction where Copilot inline suggestions suppress the suggest widget. The fix likely involves either the Copilot extension not setting `suppressSuggestions: true`, or the editor providing a more nuanced policy that allows the suggest widget to coexist with ghost text.
