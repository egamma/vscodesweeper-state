# Issue #242351 — Terminal zoom with mouse wheel conflicts with scrolling

[https://github.com/microsoft/vscode/issues/242351](https://github.com/microsoft/vscode/issues/242351)

## Summary

Confirmed bug still present on main — the terminal zoom contribution never feeds the MouseWheelClassifier before querying it, causing physical wheel events to be misclassified and routed to scroll instead of zoom. An open PR (#310769) addresses it.

## Change summary

Ctrl+mouse wheel in the terminal scrolls instead of zooming when the buffer has scrollback content, because the MouseWheelClassifier is queried without being fed the incoming event first.

## Best solution

Land or iterate on the open PR https://github.com/microsoft/vscode/pull/310769, which feeds the classifier before querying it and fixes the double-accumulation and unnormalized-delta bugs in `terminal.zoom.contribution.ts`.
