# Issue #291933 — Default Trusted folders on various platforms

[https://github.com/microsoft/vscode/issues/291933](https://github.com/microsoft/vscode/issues/291933)

## Summary

Feature request to add platform-specific default trusted folders (e.g. user home directory). Current behavior is *as-designed* — trust is always an explicit user action for security reasons — but there's a reasonable UX argument for auto-trusting the user's own home directory. Keeping open for the community process given no upvotes yet.

## Change summary

Add default trusted folders (e.g. user home directory) so personal files don't require an explicit trust grant.

## Best solution

This borders on *as-designed given the security-first intent of workspace trust (trusting the home directory transitively trusts everything under it, including cloned repos from untrusted sources). If the team wanted a middle ground, a setting like `security.workspace.trust.defaultTrustedFolders` could exist, but it may undermine the feature's purpose. Most likely outcome is closure as *as-designed or keeping it for community discussion.

## Vision fit

- **Fit:** rejected
- **Reason:** Auto-trusting broad folder trees by default conflicts with VS Code's explicit security guardrails around workspace trust; the current explicit-grant model is intentional.
- **Evidence:** Value: security — workspace trust, process sandboxing. Trusting /home/User by default would implicitly trust every cloned repository and downloaded project underneath it, weakening the security boundary workspace trust was designed to provide.
