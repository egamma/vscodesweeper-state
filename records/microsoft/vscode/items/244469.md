# Issue #244469 — Repair out-of-bound line number variables / other inconsistent state, but log and notify

[https://github.com/microsoft/vscode/issues/244469](https://github.com/microsoft/vscode/issues/244469)

## Summary

Team-authored architectural debt proposal (hediet, MEMBER) on the 'On Deck' milestone — describes a strategy for repairing inconsistent internal state with logging/telemetry instead of throwing. Both the author association and the milestone require keeping this open.

## Change summary

Establish a pattern for repairing out-of-bound or inconsistent internal state (e.g. line numbers) with error telemetry and notifications instead of hard throws.

## Best solution

Implement the proposed pattern: repair inconsistent state where feasible, emit error telemetry via onUnexpectedError, log with console.error for dev visibility, and show notifications to Insiders users (rate-limited, snoozable). The example PR (https://github.com/microsoft/vscode/pull/244627) demonstrates the direction; broader adoption across editor-core APIs like Range.fromPositions is the next step.
