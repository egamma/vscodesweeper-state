# Issue #251172 — Git extension: rebase -i skips commit message prompt

[https://github.com/microsoft/vscode/issues/251172](https://github.com/microsoft/vscode/issues/251172)

## Summary

Confirmed bug: `rebaseContinue()` sets `GIT_EDITOR: 'true'`, which suppresses the squash commit message editor during interactive rebase. The defect is still present on current main. Keeping open — this is a real gap in the git extension's rebase flow.

## Change summary

When using the Git extension's 'Continue' button during an interactive rebase with squash, git skips the commit message prompt because the extension overrides GIT_EDITOR with the no-op `true` command.

## Best solution

Change `rebaseContinue()` in `extensions/git/src/git.ts` to use the VS Code git editor integration (the same `GIT_EDITOR` env from `GitEditor.env`) instead of `'true'`, so that when git needs a commit message editor during rebase (e.g. for squash), VS Code opens the message file for editing. The tricky part is that the current `'true'` suppression was intentional for the simple rebase case (no squash) to avoid an unnecessary editor popup — a conditional approach or accepting the editor for all rebase-continue cases may be needed.

## Evidence

- **intentional suppression:** Commit f19251a4caa (PR #155748) deliberately set GIT_EDITOR to 'true' to suppress the editor during rebase continue, but this has the side effect of also suppressing the squash message editor.
- **git editor integration exists:** extensions/git/src/gitEditor.ts provides a proper VS Code-integrated GIT_EDITOR via IPC that opens commit message files in the editor — this is what the terminal uses, but rebaseContinue() bypasses it.
