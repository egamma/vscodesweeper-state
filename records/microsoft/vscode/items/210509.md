# Issue #210509 — [Testing] Coverage API Issues: Monorepo & Accumulation

[https://github.com/microsoft/vscode/issues/210509](https://github.com/microsoft/vscode/issues/210509)

## Summary

The core request — accumulating coverage across multiple controllers — remains unimplemented on main. The coverage service stores a single selected coverage and replaces it on each run. This is a valid enhancement with active community interest, best kept open for the area owner.

## Change summary

Coverage data from multiple test controllers should accumulate rather than replace each other, and running a non-coverage test should not dismiss the coverage panel.

## Best solution

Redesign TestCoverageService to support a collection of coverage results (one per controller/task) and merge them for display. The coverage view and decorations would need to aggregate across results. Key files: src/vs/workbench/contrib/testing/common/testCoverageService.ts, src/vs/workbench/contrib/testing/browser/testCoverageView.ts.
