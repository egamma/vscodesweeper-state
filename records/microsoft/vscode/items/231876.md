# Issue #231876 — Multiline cursor using ALT-Up and Down on number pad causes character to display after releasing keys

[https://github.com/microsoft/vscode/issues/231876](https://github.com/microsoft/vscode/issues/231876)

## Summary

Keep open for editor/input investigation. The report is specific to Windows Alt+numpad-arrow input and there is no confirmed duplicate or fixing commit in the local history.

## Change summary

Using Alt+Up/Down on the numeric keypad for multicursor expansion inserts unexpected Alt-code characters after the keys are released.

## Best solution

Verify on current Windows Insiders/main with NumLock off and a keybinding that maps Alt+numpad Up/Down to multicursor commands; if reproduced, suppress the generated text input after a handled editor keybinding or route upstream only if Chromium/Electron emits unavoidable Alt-code input.
