# Issue #316603 — committerDate as variable for Git blame decorations

[https://github.com/microsoft/vscode/issues/316603](https://github.com/microsoft/vscode/issues/316603)

## Summary

Valid feature request — the git blame template currently only exposes author-date tokens; committer-date information is not parsed or surfaced, so this remains an unimplemented gap.

## Change summary

Add committerDate and committerDateAgo variables to the git blame decoration/status bar template system.

## Best solution

Extend `BlameInformation` in `extensions/git/src/git.ts` to include a `committerDate` field (parsed from `git blame --porcelain`), add `committerDate`/`committerDateAgo` to `BlameInformationTemplateTokens` and `formatBlameInformationMessage` in `extensions/git/src/blame.ts`, and document the new tokens in `package.json`/`package.nls.json`.
