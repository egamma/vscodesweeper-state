# Issue #30874 — Regex Find can cause VS Code to Freeze

[https://github.com/microsoft/vscode/issues/30874](https://github.com/microsoft/vscode/issues/30874)

## Summary

Still reproduces on main — the editor find uses JavaScript's native RegExp with no timeout or cancellation guard against catastrophic backtracking. Correctly labeled upstream (V8 regex engine limitation) and remains a valid tracking issue for the problem.

## Change summary

Regex patterns with catastrophic backtracking (e.g. nested quantifiers) freeze the VS Code editor window during find operations.

## Best solution

Either adopt V8's non-backtracking regexp mode (the `l` flag / linear-time engine) when available in the target Electron version, or add a Web Worker / timeout-based cancellation around regex execution in `src/vs/editor/common/model/textModelSearch.ts` so the main thread is never blocked indefinitely by a user-supplied pattern.
