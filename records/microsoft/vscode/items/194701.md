# Issue #194701 — Search fails in folders having curly brackets groups in folder name (ie. '/{{cookiecutter.package_name}}')

[https://github.com/microsoft/vscode/issues/194701](https://github.com/microsoft/vscode/issues/194701)

## Summary

Still reproduces on main: `escapeGlobPattern` in queryBuilder.ts escapes `?*[]` but not `{}`, so folder names with curly braces (like cookiecutter templates) produce invalid glob patterns when used as search include patterns.

## Change summary

Search fails with 'nested alternate groups' error when using 'Find in Folder' on directories whose names contain curly braces.

## Best solution

Extend `escapeGlobPattern` in `src/vs/workbench/services/search/common/queryBuilder.ts` to also escape `{` and `}` (e.g. `[{]` and `[}]`), matching the existing bracket-escaping approach. Verify the ripgrep include-pattern path in `ripgrepTextSearchEngine.ts` also handles escaped braces correctly.
