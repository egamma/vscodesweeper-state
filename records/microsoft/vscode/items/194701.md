# Issue #194701 — Search fails in folders having curly brackets groups in folder name (ie. '/{{cookiecutter.package_name}}')

[https://github.com/microsoft/vscode/issues/194701](https://github.com/microsoft/vscode/issues/194701)

## Summary

Still reproduces on main: `escapeGlobPattern` in `queryBuilder.ts` escapes `?*[]` but not `{}`, so folder names containing curly braces (e.g. `{{cookiecutter.package_name}}`) are misinterpreted as glob alternation groups, breaking 'Find in Folder'.

## Change summary

Search fails with 'nested alternate groups' error when using 'Find in Folder' on directories whose names contain curly braces.

## Best solution

Extend `escapeGlobPattern` in `src/vs/workbench/services/search/common/queryBuilder.ts` to also escape `{` and `}` characters, and verify that ripgrep's glob handling (in `ripgrepTextSearchEngine.ts`) correctly passes through escaped braces. The same pattern used for square brackets (`[{]` / `[}]`) should work.
