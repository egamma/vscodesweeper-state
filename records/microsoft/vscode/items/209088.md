# Issue #209088 — SCM - find a git repository by search in the SCM view

[https://github.com/microsoft/vscode/issues/209088](https://github.com/microsoft/vscode/issues/209088)

## Summary

Valid feature request — the SCM tree's keyboard navigation label provider explicitly returns `undefined` for repository nodes (line 809 of scmViewPane.ts), so the built-in 'Find in tree' feature cannot filter by repository name. The gap remains on current main. Keeping open in the Backlog milestone.

## Change summary

Allow filtering/searching for git repositories by name in the SCM view's tree filter, which currently only matches files and resource groups but not repository nodes.

## Best solution

Return the repository provider label from `SCMTreeKeyboardNavigationLabelProvider.getKeyboardNavigationLabel` when the element is an `ISCMRepository`, so the existing 'Find in tree' (Ctrl+F) feature surfaces repository nodes. The fix is localized to `src/vs/workbench/contrib/scm/browser/scmViewPane.ts` around line 809.
