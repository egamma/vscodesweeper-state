# Issue #246049 — MCP: Expose current git remote as root

[https://github.com/microsoft/vscode/issues/246049](https://github.com/microsoft/vscode/issues/246049)

## Summary

Team-authored feature request exploring whether MCP roots should expose the git remote URL alongside the file:// root. Active member discussion shows interest but no consensus yet; the MCP spec itself currently only allows file:// roots, which is a blocker.

## Change summary

Expose the current git remote as an additional MCP root so MCP servers can identify the repository without extra tool calls.

## Best solution

Wait for the MCP spec to evolve beyond file:// roots (tracked upstream at modelcontextprotocol/modelcontextprotocol), then implement git-remote root exposure in the VS Code MCP client. In the interim, MCP servers can use file access to read .git/config for remote info.
