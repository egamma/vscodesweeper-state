# Issue #284176 — Hallucinates response when attaching an unsupported file type

[https://github.com/microsoft/vscode/issues/284176](https://github.com/microsoft/vscode/issues/284176)

## Summary

Team-authored bug report about chat hallucinating when an unsupported file type is attached (possibly due to an empty `<attachment>` tag in the prompt). Keeping open per team-member policy — needs owner assignment and further investigation.

## Change summary

Copilot chat produces hallucinated/random responses when a user attaches an unsupported file type, potentially caused by an empty `<attachment>` element in the prompt.

## Best solution

Investigate how unsupported file attachments are serialized into the prompt — if an empty `<attachment>` tag is sent, either strip it or replace it with an explicit 'unsupported file type' message so the model doesn't hallucinate around the gap. The fix likely lives in the Copilot extension's attachment handling (vscode-copilot repo) rather than VS Code core.
