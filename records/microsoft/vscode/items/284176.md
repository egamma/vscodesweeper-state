# Issue #284176 — Hallucinates response when attaching an unsupported file type

[https://github.com/microsoft/vscode/issues/284176](https://github.com/microsoft/vscode/issues/284176)

## Summary

Team-authored bug report about Copilot Chat hallucinating responses when an unsupported file type is attached (possibly due to an empty <attachment> tag being sent). Keeping open per team-member policy.

## Change summary

Copilot Chat produces hallucinated/random responses when a user attaches an unsupported file type, possibly influenced by an empty attachment element in the prompt.

## Best solution

Investigate how unsupported file attachments are serialized into the prompt — if they produce an empty `<attachment>` tag, either strip it entirely or replace it with a clear 'unsupported file type' message so the model doesn't hallucinate around it. Likely in the chat attachment/context-building code under `extensions/copilot/`.
