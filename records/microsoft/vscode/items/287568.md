# Issue #287568 — Incorrect parsing and highlighting of GitHub Copilot terminal command with heredoc

[https://github.com/microsoft/vscode/issues/287568](https://github.com/microsoft/vscode/issues/287568)

## Summary

The heredoc highlighting bug is in the upstream shellscript TextMate grammar (jeff-hykin/better-shell-syntax), which VS Code consumes. The grammar has heredoc support but apparently mishandles certain backtick/single-quote interactions inside heredoc content. A previous similar issue (#246283) was closed as upstream.

## Change summary

Shell heredoc content containing backticks and single quotes is incorrectly tokenized, causing misleading red highlighting in the Copilot terminal command confirmation dialog.

## Best solution

File an issue upstream at jeff-hykin/better-shell-syntax for the heredoc + backtick/quote tokenization edge case. Once fixed upstream, pull the updated grammar into VS Code via the grammar update process. Label as `grammar` and `upstream-issue-linked` once the upstream issue is filed.
