# Issue #239878 — Git - Open Git commit online in any hosting site, not just GitHub

[https://github.com/microsoft/vscode/issues/239878](https://github.com/microsoft/vscode/issues/239878)

## Summary

Valid feature request with an open community PR (https://github.com/microsoft/vscode/pull/282970) that directly targets it. The PR modifies the built-in GitHub extension to derive commit/branch links from the remote's base URL rather than hardcoding github.com, which would support GitLab, GitHub Enterprise, and other hosting sites. Keeping open so a maintainer can review and land (or close) the PR.

## Change summary

The built-in Git blame 'Open on GitHub' action only works for github.com repositories; users on GitLab, GitHub Enterprise, or other Git hosting sites cannot open commits in their browser.

## Best solution

Review and land (or iterate on) the open PR https://github.com/microsoft/vscode/pull/282970, which extends the GitHub extension's URL parsing in `extensions/github/src/util.ts` and `extensions/github/src/links.ts` to derive commit/branch links from the remote's actual base URL rather than assuming github.com. The approach is sound — the label and links update to reflect the actual host.
