# Issue #311695 — [Error] unhandlederror-Cannot read properties of undefined (reading 'fsPath')

[https://github.com/microsoft/vscode/issues/311695](https://github.com/microsoft/vscode/issues/311695)

## Summary

Still reproduces on current main: `ChangesTreeSorter.compare()` unsafely accesses `.uri.fsPath` without a null guard, and `toIChangesFileItem()` can produce items with `uri: undefined` for deletions when the `isIChatSessionFileChange2` type guard fails on deserialized objects.

## Change summary

The Changes view tree crashes with 'Cannot read properties of undefined (reading fsPath)' when sorting items that have an undefined URI (e.g., deleted files from deserialized IPC data).

## Best solution

Guard `toIChangesFileItem` to use `change.modifiedUri ?? change.uri` as a safe fallback (restoring the pre-#307836 behavior), or filter out entries with no valid URI before inserting into the tree. Additionally, add a defensive check in `ChangesTreeSorter.compare()` at line 1698.
