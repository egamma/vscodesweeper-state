# Issue #238363 — Git - Opening diff from blame should reveal previously focused line

[https://github.com/microsoft/vscode/issues/238363](https://github.com/microsoft/vscode/issues/238363)

## Summary

Milestone 'On Deck' means this stays open for maintainer scheduling. The gap is real and confirmed in the source: the blame decoration click invokes `git.viewCommit` which reveals the correct file in the multi-diff editor but does not pass a line/range, so the diff opens scrolled to the top.

## Change summary

When clicking the git blame inline decoration, the multi-file diff editor should scroll to the line that was focused, not to the top of the file.

## Best solution

Thread the cursor line number from the blame click through `git.viewCommit` into the `reveal` object's `range` field (which the multi-diff editor already supports). Likely touches `extensions/git/src/blame.ts` (pass line number in command arguments) and `extensions/git/src/commands.ts` (`viewCommit` method, around line 5129, to include `range` in the reveal).
