# Issue #224714 — Smooth scroll not working if window.zoomLevel==1

[https://github.com/microsoft/vscode/issues/224714](https://github.com/microsoft/vscode/issues/224714)

## Summary

This appears to be fixed on current main by a targeted wheel-classifier tolerance change for `window.zoomLevel = 1`; the issue can be closed as completed without a `*` triage label.

## Change summary

Editor smooth mouse-wheel scrolling stopped animating when `window.zoomLevel` was exactly `1`.

## Best solution

Keep the shipped wheel-classifier fix from commit e9df78f747311117bbe0678feefdd278e0f21ff7, which prevents zoom-factor floating point noise from disabling the physical-mouse-wheel path that enables smooth scrolling.

- **release:** `git tag --contains e9df78f747311117bbe0678feefdd278e0f21ff7` shows the fix is included starting with the 1.103.0 tag; it is also present on current `main`.
