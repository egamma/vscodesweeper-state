# Issue #149641 — Unify inline edit experience

[https://github.com/microsoft/vscode/issues/149641](https://github.com/microsoft/vscode/issues/149641)

## Summary

Team-authored debt issue requesting a unified inline-rename widget across Explorer, Timeline, and Terminal tabs. The inconsistency is still visible in the codebase (each view implements its own rename UX). Hard keep-open: author is a MEMBER.

## Change summary

Unify the different inline edit/rename experiences (Explorer, Timeline, Terminal tabs) into a single reusable composable widget.

## Best solution

Implement a composable tree renderer (as joaomoreno suggested) that provides a consistent inline-rename experience, then adopt it in Explorer, Timeline, Terminal tabs, and potentially Outline. This lives in the base list/tree layer (`src/vs/base/browser/ui/list/` or `src/vs/base/browser/ui/tree/`) and would be consumed by each view's renderer.
