# Issue #299075 — Copilot not respecting excludeAgent flag in instructions file

[https://github.com/microsoft/vscode/issues/299075](https://github.com/microsoft/vscode/issues/299075)

## Summary

The reported behavior is expected: `excludeAgent: "code-review"` means 'exclude this instruction FROM the code-review agent,' not 'use this instruction only for code-review.' Regular Copilot Chat correctly sees the instruction because it is not excluded. This is a usage question, not a bug.

## Change summary

User expects `excludeAgent: "code-review"` to restrict an instruction file to only the code-review agent, but the attribute actually excludes the named agent from receiving the instruction.

## Best solution

Close as a question. The user needs to restructure their setup: to have an instruction apply only to the code-review agent, they would exclude all other agents instead, or use a different scoping mechanism. The `excludeAgent` attribute works as its name implies — it excludes the listed agents.
