# Issue #280999 — Message reappears after checkpoint

[https://github.com/microsoft/vscode/issues/280999](https://github.com/microsoft/vscode/issues/280999)

## Summary

Team-authored bug that still reproduces on main. The checkpoint restore marks messages as 'blocked' via `setCheckpoint` without removing them, and the retry flow (`resendRequest`) only removes the retried request — blocked messages remain in `_requests` and can resurface in incorrect order.

## Change summary

After restoring a checkpoint and retrying a message, previously-removed messages reappear in the wrong position in the chat history.

## Best solution

The `resendRequest` path in `chatServiceImpl.ts` (line ~975) should also remove or permanently discard requests that were blocked by the checkpoint before re-sending, rather than only removing the single retried request. The fix likely touches `ChatModel.removeRequest` or `resendRequest` to cascade removal of checkpoint-blocked requests.
