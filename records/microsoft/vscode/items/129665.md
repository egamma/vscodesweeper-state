# Issue #129665 — [Mac OSX 11.5] Cannot launch code after build from source on a case sensitive partition

[https://github.com/microsoft/vscode/issues/129665](https://github.com/microsoft/vscode/issues/129665)

## Summary

The original bug — case-sensitive icon file paths breaking the build on case-sensitive macOS volumes — was fixed in commit bfb7af52e543 (shipped in 1.91.0), which added `.toLowerCase()` to the `darwinBundleDocumentType` function. The remaining plural helper already receives only lowercase icon args, so the entire surface is case-safe on current main.

## Change summary

Building VS Code from source on a case-sensitive macOS partition failed because `darwinBundleDocumentType` referenced icon filenames with mixed case (e.g. `Bower.icns`) while the actual files were all lowercase.

## Best solution

Keep the shipped fix (commit bfb7af52e543, 1.91.0). The two open PRs (#306089, #307865) are now redundant and can be closed. No further code change is needed.
