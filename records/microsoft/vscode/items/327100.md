# Issue #327100 — When running C++ code, VS Code automatically navigates to the MinGW bin directory to run it

## Summary

Both reported behaviors — the C++ run navigating to the MinGW bin directory and the unresponsive Python run button — are provided by extensions (C/C++ and Python), not VS Code core. A commenter already identified this and pointed to the correct tracker.

## Change summary

Running C++ code navigates to the MinGW bin directory, and the Python run button is unresponsive.

## Best solution

The C++ run behavior belongs to the C/C++ extension (microsoft/vscode-cpptools) and the Python run behavior belongs to the Python extension (microsoft/vscode-python). These are the correct trackers for investigation. Since the issue spans two separate extensions, moving it to one tracker would lose the other half — better to close here and note both trackers.
