# Issue #227924 — Persisted files associated with a test run

[https://github.com/microsoft/vscode/issues/227924](https://github.com/microsoft/vscode/issues/227924)

## Summary

Valid feature request with team buy-in (connor4312 proposed a storage URI per TestResult) but not yet implemented on main. Low upvote count (2) but has meaningful engagement; keeping open in Backlog.

## Change summary

Provide a managed storage URI per TestResult so extensions can persist files (profiles, coverage) with test-run lifecycle.

## Best solution

Implement a `storageUri` on `TestResult` that VS Code manages — cleaning up associated files when the result is discarded. This lives in `src/vs/workbench/contrib/testing/` (result lifecycle) and the extension API surface (`vscode.d.ts` TestRun/TestRunResult). connor4312's comment outlines the approach.
