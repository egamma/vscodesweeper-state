# Issue #302394 — MCP Streamable HTTP client does not send cookies, breaking ARR affinity on multi-instance Azure App Service

[https://github.com/microsoft/vscode/issues/302394](https://github.com/microsoft/vscode/issues/302394)

## Summary

Team-authored bug, still unfixed on main. The MCP Streamable HTTP transport does not pass credentials or handle cookies, breaking session affinity behind load balancers. No fix commit found.

## Change summary

MCP Streamable HTTP client omits cookies on subsequent requests, causing session-not-found errors on multi-instance Azure App Service deployments with ARR affinity.

## Best solution

The MCP gateway session or transport construction code (likely in `src/vs/platform/mcp/node/mcpGatewaySession.ts` or the SDK integration layer) should pass `credentials: "include"` via `requestInit` when constructing the StreamableHTTPClientTransport, or implement a cookie jar for Node-side fetch calls. This may also require coordination with the MCP TypeScript SDK upstream.
