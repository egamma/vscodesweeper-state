## Change summary

Stopping a debug session while a pending `restartRequest` hangs leaves the debug state stuck, preventing F5 from starting a new session.

## Best solution

In `debugService.ts`, the `doRestart` path for `supportsRestartRequest` (line ~916-926) calls `session.restart()` with no timeout or cancellation. When the user stops the session, the pending restart promise in `rawDebugSession.send()` should be cancelled or rejected so that `doRestart` can clean up `restartingSessions` and emit the proper end-session event. Adding a cancellation token or timeout to the restart send, or making `stopSession` force-reject pending restart promises, would fix the state leak.
