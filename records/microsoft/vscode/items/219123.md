# Issue #219123 — Cut node fade applies to all inline folders

[https://github.com/microsoft/vscode/issues/219123](https://github.com/microsoft/vscode/issues/219123)

## Summary

Still reproduces on main: cutting a folder inside a compact/compressed path fades the entire inline label because the `cut` CSS class applies opacity to the whole `.explorer-item` container rather than individual path segments.

## Change summary

Cutting a folder that is part of a compact/inline folder path in the explorer fades all segments of the compressed label, not just the cut folder.

## Best solution

The fix should apply the `cut` fade per-segment in compressed labels. In `renderCompressedElements` / `renderStat`, each element in the compressed node array should be checked via `isCut()`, and the resulting cut styling should target individual `.label-name` segments rather than the entire `.explorer-item`. This likely requires changes in `explorerViewer.ts` (the renderer) and possibly the `IconLabel` component to support per-segment extra classes.
