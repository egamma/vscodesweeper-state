# Issue #73559 — Cannot set embedded language indentation rules

[https://github.com/microsoft/vscode/issues/73559](https://github.com/microsoft/vscode/issues/73559)

## Summary

This is the canonical tracking issue for embedded language indentation rules. Issue #176034 was closed as completed and redirected here, and #207725 was closed by the community upvote process pointing back here. The gap remains: while `autoIndent.ts` does call `getLanguageIdAtPosition` for some flows, the full indentation experience (move line up/down, on-type indent) doesn't consistently honor embedded language configurations set via `setLanguageConfiguration`.

## Change summary

Extensions cannot reliably set indentation rules for embedded languages — only the outer document language's rules are applied in operations like move-line-up/down.

## Best solution

Keep as the canonical tracker for embedded-language indentation. The fix would involve ensuring all indentation code paths in `src/vs/editor/common/languages/autoIndent.ts` and related cursor/move-lines logic consistently resolve and apply the embedded language's `indentationRules` rather than only the top-level document language.
