# Issue #151948 — Allow extensions to contribute Group Policy

[https://github.com/microsoft/vscode/issues/151948](https://github.com/microsoft/vscode/issues/151948)

## Summary

The request for extensions to freely declare Group Policy settings via their package.json contribution remains unimplemented. Current main has a product-level `extensionConfigurationPolicy` mechanism (configured in product.json by the VS Code team), but no open extension API for arbitrary extensions to promote their settings to policies. Keeping open as a valid feature request in the Backlog.

## Change summary

Allow extensions to declare their configuration settings as Group Policy entries through their package.json contribution point.

## Best solution

Expose policy declaration in the extension configuration contribution point schema (package.json `contributes.configuration`) so extensions can mark settings as policy-eligible, then wire the policy service to respect those declarations. This likely touches `src/vs/workbench/api/common/configurationExtensionPoint.ts` and the configuration registry's policy handling.
