# Issue #310917 — Git - Add support for unstaging a range of deleted lines

[https://github.com/microsoft/vscode/issues/310917](https://github.com/microsoft/vscode/issues/310917)

## Summary

Valid feature request: allow unstaging selected ranges of deleted lines in the staged diff editor. Currently the 'Unstage Selected Ranges' command only works on added/modified lines because deleted lines in the modified side of the diff are not selectable. No existing fix or duplicate found.

## Change summary

Allow 'Unstage Selected Ranges' to apply to deleted lines in the staged diff editor, which currently cannot be selected or right-clicked.

## Best solution

The git extension's staging logic in `extensions/git/src/staging.ts` already handles deletions in `intersectDiffWithRange` (line 117-118 returns the full diff when `modifiedEndLineNumber === 0`). The gap is in the diff editor UI — deleted lines on the modified side collapse to a zero-width range (per `getModifiedRange`), making them un-selectable. The fix likely involves making the gutter/context-menu available on deletion markers, or allowing selection of the adjacent anchor line to target the deletion hunk.
