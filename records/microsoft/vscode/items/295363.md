# Issue #295363 — Reasoning summaries LOVE being blocked with no actual mention in the content

[https://github.com/microsoft/vscode/issues/295363](https://github.com/microsoft/vscode/issues/295363)

## Summary

Team-authored issue (MEMBER). The reasoning-summary LLM prompt already includes instructions to avoid 'blocked' when hookCount is 0, but the model still hallucinates the word. The prompt design and mitigation suggestions from the comment remain unaddressed on current main.

## Change summary

Reasoning/thinking summaries in the chat UI frequently include the word 'blocked' even when no hooks or blocking actually occurred, confusing users.

## Best solution

Restructure the summarization prompt in `chatThinkingContentPart.ts` per roblourens' suggestions: move instructions to the system prompt, simplify the prompt for single-thinking-message cases, reduce the instruction surface that mentions 'blocked' (which ironically primes the model), and write evals to regression-test the summaries.
