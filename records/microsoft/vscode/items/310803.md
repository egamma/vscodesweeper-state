# Issue #310803 — copilot mcp tools selection cross session

[https://github.com/microsoft/vscode/issues/310803](https://github.com/microsoft/vscode/issues/310803)

## Summary

Still valid: tool selection state in `chatSelectedTools.ts` is keyed by mode ID, not per-chat-session, so enabling/disabling tools in one session's Agent mode affects all concurrent Agent-mode sessions. The architecture needs per-session tool selection scoping.

## Change summary

MCP tool selection changes in one chat session bleed into other concurrent sessions using the same chat mode, causing task failures.

## Best solution

Refactor `ChatSelectedToolsModel._sessionStates` to key by the individual chat session ID (not mode ID) so that tool enablement is isolated per session. The global state should remain as the default, with per-session overrides staying local to the session that set them. The fix belongs in `src/vs/workbench/contrib/chat/browser/widget/input/chatSelectedTools.ts`.
