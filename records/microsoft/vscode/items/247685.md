# Issue #247685 — How do I get from the inline merge conflicts view to the merge editor?

[https://github.com/microsoft/vscode/issues/247685](https://github.com/microsoft/vscode/issues/247685)

## Summary

Still open on main — no fix has landed. The inline merge-conflict 'resolve in merge editor' button has no keyboard/screen-reader discoverability path. Multiple hard keep-open rules apply: team-authored (MEMBER), milestone 'On Deck'.

## Change summary

The inline merge-conflict resolution button in the editor is not reachable via keyboard or discoverable by screen readers.

## Best solution

Per the discussion, expose a global command (e.g. 'Open Merge Editor') in the Command Palette via the Git extension, and also make the inline button a tab stop in 'tab moves focus' mode. The command registration belongs in `extensions/git/src/commands.ts` (where `git.openMergeEditor` already lives); the tab-stop work likely touches the merge-conflict decoration code in `extensions/merge-conflict/` or the editor's content-widget layer.
