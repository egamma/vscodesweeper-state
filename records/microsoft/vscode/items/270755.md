# Issue #270755 — Handle optional arguments properly installing/starting an MCP Server

[https://github.com/microsoft/vscode/issues/270755](https://github.com/microsoft/vscode/issues/270755)

## Summary

Team-authored bug, still reproducible on main. When an optional input variable is skipped (Escape/empty), the literal `${input:...}` placeholder is passed to the MCP server instead of being omitted or replaced with empty string.

## Change summary

Optional MCP server arguments that the user skips are passed as literal `${input:...}` placeholders instead of being omitted.

## Best solution

In `_replaceVariablesInLaunch` (src/vs/workbench/contrib/mcp/common/mcpRegistry.ts), after resolution, strip or empty-out any args that still contain unresolved `${input:...}` placeholders for optional inputs. Alternatively, handle `undefined` in `ConfigurationResolverExpression._resolveAtLocation` by removing the arg from the array rather than leaving the placeholder intact.
