# Issue #293274 — Multi-select on worktrees in SCM -> delete only deletes one at a time, despite multi select

[https://github.com/microsoft/vscode/issues/293274](https://github.com/microsoft/vscode/issues/293274)

## Summary

Confirmed on current main: the `git.repositories.deleteWorktree` command accepts only a single `SourceControlArtifact` parameter, so multi-select in the SCM worktree view only deletes one item. Team-authored issue — keeping open.

## Change summary

Multi-selecting worktrees in the SCM view and deleting only removes one worktree instead of all selected.

## Best solution

Update `artifactDeleteWorktree` in `extensions/git/src/commands.ts` to accept an array of `SourceControlArtifact` items (matching the multi-select pattern used by other SCM commands) and iterate over each, calling `repository.deleteWorktree()` for each selected worktree.
