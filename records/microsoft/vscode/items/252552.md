Discussion between team members is ongoing about the best format (header field vs. stripped markdown comments).

## Change summary

Custom chat mode files should support human-facing comments (not sent to the LLM) to provide usage tips or documentation to readers.

## Best solution

Add comment syntax support (e.g. `//`-prefixed lines or a dedicated frontmatter field) to the prompt file parser in `src/vs/workbench/contrib/chat/common/promptSyntax/promptFileParser.ts`, stripped before the body content is sent to the model. The team discussion leans toward either a `comments` header attribute or stripping `//`-prefixed lines from the body.
