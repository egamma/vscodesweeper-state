# Issue #96392 — NVDA doesn't read the status line

[https://github.com/microsoft/vscode/issues/96392](https://github.com/microsoft/vscode/issues/96392)

## Summary

Team-authored upstream issue tracking an NVDA bug where hit-testing fails to locate the VS Code status bar. The root cause is in NVDA's pixel-based status bar detection and Chromium's async hit-test caching. Already correctly labeled upstream with a linked NVDA issue.

## Change summary

NVDA's 'read status line' command fails to find VS Code's status bar because NVDA uses pixel hit-testing rather than looking for the ARIA status role in the accessibility tree.

## Best solution

Track progress on https://github.com/nvaccess/nvda/issues/11064 — NVDA needs to use the accessibility tree (role=status) instead of pixel hit-testing for Chromium-based apps. VS Code already sets the correct ARIA role; no core change needed.
