# Issue #276413 — Git - changes tool should avoid context blowout

[https://github.com/microsoft/vscode/issues/276413](https://github.com/microsoft/vscode/issues/276413)

## Summary

Team-authored feature request on the 'On Deck' milestone requesting smarter context management for the Git changes tool — must stay open per both rules (MEMBER author and real milestone).

## Change summary

The `#changes` tool dumps entire diffs (50k–80k tokens) into the conversation context instead of progressively disclosing details, degrading agent performance on large changesets.

## Best solution

Implement progressive disclosure in the changes tool: return a file-level summary first (like the agent naturally does via git CLI), then let the agent request per-file diffs on demand — or persist the full diff to a `content.txt`-style file reference so the model can parse it without context blowout. Likely lives in the git tool implementation under `extensions/git/` or the chat tool infrastructure.
