# Issue #210062 — #accessibility in Github Copilot chat: Screenreader announces all copilot's path references and links as URL encoded file paths

[https://github.com/microsoft/vscode/issues/210062](https://github.com/microsoft/vscode/issues/210062)

## Summary

Fixed on main: the chat accessible view now uses `useLinkFormatter` to render command-URI links as clean file names/paths instead of raw URL-encoded strings, and inline references are explicitly formatted with human-readable paths.

## Change summary

Screen reader announces raw URL-encoded command URIs for Copilot file path references instead of clean file names.

## Best solution

Keep the shipped fix — `renderAsPlaintext` with `useLinkFormatter: true` strips command URIs to their link text/basename, and the `inlineReference` case provides clean `name (path)` formatting. No further change needed.
