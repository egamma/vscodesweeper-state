Team-authored bug (MEMBER) — hard keep-open. No fix has been committed.

## Change summary

Clicking 'Show More' in the CLI subagent file-read permission confirmation widget focuses the confirmation container instead of expanding the truncated message.

## Best solution

Debug the click-event propagation chain around the `.see-more` element in `chatToolConfirmationSubPart.ts`. The confirmation widget container's focus handling (or a `mousedown`/`pointerdown` listener on a parent) is likely swallowing the event before the `click` handler on `elements.showMore` (line 303) fires. Adding `e.stopPropagation()` or switching to `mousedown` may fix it. The secondary issue — the label says 'read file(s)' when listing a directory — likely lives in the tool's confirmation-message generation in the agent host client tools layer.
