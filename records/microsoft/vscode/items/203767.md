# Issue #203767 — click to expect the hover display information to be dynamically updated without refreshing the hover.

[https://github.com/microsoft/vscode/issues/203767](https://github.com/microsoft/vscode/issues/203767)

## Summary

Feature request for in-place hover content refresh (without close/reopen flicker) remains unimplemented on main. No `refreshHover` command or `onDidChangeHover` provider event exists. Low upvote count but has clear use cases from multiple extension authors.

## Change summary

Extension authors want a way to dynamically update hover content in place (e.g. via a command link inside the hover) without the hover widget closing and reopening.

## Best solution

Introduce an `editor.action.refreshHover` command (or an `onDidChangeHover` event on `HoverProvider`) that re-fetches and re-renders hover content in the existing hover widget without hiding it first. The implementation would live in `src/vs/editor/contrib/hover/browser/contentHoverController.ts` and the proposed API in `src/vscode-dts/`. This complements the work tracked in #195394.
