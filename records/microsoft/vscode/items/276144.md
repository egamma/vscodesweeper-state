# Issue #276144 — Mangler enabled in hyenie/CI part of pipeline?

[https://github.com/microsoft/vscode/issues/276144](https://github.com/microsoft/vscode/issues/276144)

## Summary

Team-authored engineering discussion about whether enabling the mangler in the hygiene/CI step actually catches meaningful failures or just slows PR merges. Both participants are team members; this needs an explicit maintainer decision on the CI pipeline trade-off.

## Change summary

The mangler is enabled in the PR hygiene/compile CI step, adding build time without catching the real failures (which only surface when tests run on mangled code).

## Best solution

The team should decide whether to (a) remove the mangler from the hygiene step to speed up PR merges, (b) add a post-mangle test run to actually catch private-access failures, or (c) accept the current cost as a partial safety net. This lives in `build/` CI pipeline configuration.
