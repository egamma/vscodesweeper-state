# Issue #304064 — Sessions: PR created on the wrong remote

[https://github.com/microsoft/vscode/issues/304064](https://github.com/microsoft/vscode/issues/304064)

## Summary

Valid bug: agent sessions pushed a PR branch to the wrong git remote (`private` instead of `origin`) when multiple remotes are configured. No fix found on main. The remote-selection logic likely lives in the agent host process rather than in VS Code's sessions UI code, but the issue is correctly filed here as the orchestration layer.

## Change summary

When an agent session creates a PR in a repository with multiple git remotes, it may push to the wrong remote instead of `origin`.

## Best solution

The agent host's git-push logic (or the session provider that instructs it) needs to select the correct remote — likely defaulting to `origin` or the remote that matches the upstream of the current branch. The relevant coordination code is in `src/vs/sessions/contrib/providers/agentHost/browser/` and the agent host SDK. A fix should ensure the remote is explicitly chosen based on the fork/upstream topology rather than an arbitrary or alphabetical pick.
