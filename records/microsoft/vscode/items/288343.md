# Issue #288343 — Pressing no on periodically run git fetch doesn't do anything

[https://github.com/microsoft/vscode/issues/288343](https://github.com/microsoft/vscode/issues/288343)

## Summary

Team-authored issue (Tyriar). The bug is real and traceable in source: pressing 'No' on the autofetch prompt writes to extension globalState (Memento), which doesn't sync across machines — so the prompt reappears on a new device despite Settings Sync. Keeping open for maintainer judgment.

## Change summary

Pressing 'No' on the periodic git fetch prompt stores the decision in local globalState instead of user settings, so it isn't synced across machines.

## Best solution

When the user presses 'No', explicitly set `git.autofetch` to `false` in `ConfigurationTarget.Global` (user settings) — mirroring how 'Yes' already sets it to `true` — so the decision syncs via Settings Sync. The fix is a small change in `extensions/git/src/autofetch.ts` around line 58–63.
