Still unimplemented on main — no generic pathMapping infrastructure exists in the debug platform layer. The issue is on the 'On Deck' milestone, which is a scheduling decision that requires maintainer judgment; keeping open.

## Change summary

Introduce a generalized 'pathMapping' mechanism in the debug adapter layer to replace the limited remoteRoot/localRoot model.

## Best solution

Implement a generic path-mapping transform layer in `src/vs/workbench/contrib/debug/` that debug adapters can leverage instead of each adapter reimplementing its own mapping logic. This would live alongside the existing source-mapping infrastructure and support multiple glob-based path pairs.
