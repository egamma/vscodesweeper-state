# Issue #33861 — introduce "pathMapping" as a more powerful replacement for "remoteRoot/localRoot"

[https://github.com/microsoft/vscode/issues/33861](https://github.com/microsoft/vscode/issues/33861)

## Summary

Still a valid, unimplemented feature request. The VS Code debug infrastructure has no generic path-mapping layer — each debug adapter (js-debug, PHP, etc.) implements its own. The request for a core transform layer remains open with low but real community interest.

## Change summary

Introduce a generic "pathMapping" mechanism in the VS Code debug infrastructure to replace per-adapter remoteRoot/localRoot with a multi-mapping, glob-capable path transform.

## Best solution

Implement a generic path-mapping transform layer in the debug adapter protocol handling (likely in `src/vs/workbench/contrib/debug/`) that intercepts paths between VS Code and debug adapters, supporting multiple mappings with glob patterns. This would eliminate the need for each debug adapter to implement its own path mapping. The feature remains relevant for remote debugging and container scenarios.
