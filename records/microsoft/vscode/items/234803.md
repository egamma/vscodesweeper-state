# Issue #234803 — Invalid outline in search editor

[https://github.com/microsoft/vscode/issues/234803](https://github.com/microsoft/vscode/issues/234803)

## Summary

Still reproduces on main: the outline reveal path calls `openCodeEditor` with the search editor's `search-editor:` scheme URI, which the editor service cannot resolve. An open PR (#317261) addresses this but has not merged yet.

## Change summary

Clicking an outline entry in a search editor fails with 'Unable to resolve resource search-editor:…' because the outline tries to open the non-resolvable search-editor URI via the code editor service.

## Best solution

Review and land https://github.com/microsoft/vscode/pull/317261, which guards the reveal path to navigate in-place when the outline model belongs to the current editor — avoiding the unresolvable URI round-trip for search editors and similar non-openable schemes.
