# Issue #129999 — datalist popup isn't positioned correctly when used within a webview

[https://github.com/microsoft/vscode/issues/129999](https://github.com/microsoft/vscode/issues/129999)

## Summary

Confirmed Electron upstream bug: datalist popups are mispositioned in webviews loaded via custom protocol schemes. The root cause is in Electron's custom protocol handler, not VS Code's code. The vscode-webview custom protocol is still in use on current main (webviewProtocolProvider.ts), and no Electron fix has landed. Keeping open as upstream tracking.

## Change summary

The HTML datalist popup renders at an incorrect position when used inside a VS Code webview, because Electron mispositions popups in iframes loaded via custom protocol schemes.

## Best solution

This needs a fix in Electron's popup positioning for custom-protocol iframes. Track the upstream Electron bug and consume the fix once it ships. The March 2026 comment by mku11 provides a minimal Electron repro — if not already filed upstream, an Electron issue with that repro would help drive a fix. VS Code currently uses Electron 42.5.0; the fix would arrive via an Electron version bump.
