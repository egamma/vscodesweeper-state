# Issue #316963 — Signing in... stays when I cancel the auth flow

[https://github.com/microsoft/vscode/issues/316963](https://github.com/microsoft/vscode/issues/316963)

## Summary

Team-authored bug report from testing #316711 — the 'Signing in…' dialog in the Agents web workbench persists when the user cancels or denies the auth flow. Still unfixed on main. Keeping open per team-author rule.

## Change summary

Canceling or denying the OAuth sign-in popup in the Agents web workbench leaves the 'Signing in…' dialog stuck on screen instead of returning to a clean state.

## Best solution

In `sessionsSetUpService.ts`, the `onSignInStarted` callback creates a 'Signing in…' dialog that is disposed at line 334 after `triggerSetup` resolves. The likely root cause is that the auth provider's promise (in `chatSetupController.setupWithProvider`) never resolves when the user cancels the OAuth popup in the browser flow. The fix should ensure that canceling the OAuth flow properly rejects or resolves the promise so `signingInDialogRef.dispose()` runs. Investigate the GitHub authentication extension's `flows.ts` browser-tab flow handling and how cancellation propagates back through `chatSetupController`.
