# Issue #318029 — Setup gets stuck due to dllhost.exe blocking inno_updater.exe from finishing cleanup

[https://github.com/microsoft/vscode/issues/318029](https://github.com/microsoft/vscode/issues/318029)

## Summary

Verified duplicate of the still-open #294546 — the reporter describes the exact same dllhost.exe/inno_updater hang, and provides useful new evidence that the PowerShell quoting in the KillContextMenuComSurrogate fix (commit c2a42605f78) is broken (exits with code 1). Consolidating discussion on the canonical issue.

## Change summary

VS Code setup hangs during update because dllhost.exe (COM surrogate for the context-menu shell extension) holds a lock on the old commit folder, and the PowerShell command intended to kill it fails due to a quoting error.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/294546 and fix the PowerShell quoting bug in `KillContextMenuComSurrogate` in `build/win32/code.iss` — the `""` escaping for the WMI `-Filter` argument doesn't survive the ShellExec→PowerShell command-line parsing chain. Consider using `-EncodedCommand` with a Base64-encoded script block or a taskkill/wmic fallback instead of nested PowerShell quoting.
