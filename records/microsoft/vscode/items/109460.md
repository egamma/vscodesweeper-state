# Issue #109460 — Support theme icon colour in all API places

[https://github.com/microsoft/vscode/issues/109460](https://github.com/microsoft/vscode/issues/109460)

## Summary

Still unresolved on main: `ThemeIcon.asClassNameArray()` does not emit a color class, so `QuickInputButton` (and any consumer using `asClassName`) cannot honour `ThemeIcon.color`. The dependency issue https://github.com/microsoft/vscode/issues/111616 is also still open. Keeping open as real remaining debt.

## Change summary

Extend ThemeIcon color support beyond custom tree views to all API surfaces that accept ThemeIcons (notably QuickInputButton).

## Best solution

Land the color-class support in `ThemeIcon.asClassNameArray()` (tracked by https://github.com/microsoft/vscode/issues/111616), then wire it through the quick-input action bar so `QuickInputButton` icons honour `ThemeIcon.color`. The unchecked item in this issue and the open dependency both point to the same gap in `src/vs/base/common/themables.ts` and `src/vs/base/browser/ui/iconLabel/iconLabels.ts`.
