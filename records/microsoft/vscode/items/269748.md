# Issue #269748 — spawn xterm ENOENT

[https://github.com/microsoft/vscode/issues/269748](https://github.com/microsoft/vscode/issues/269748)

## Summary

Still reproduces on main: the Linux external terminal fallback logic uses `$COLORTERM` and `$TERM` values (e.g. 'truecolor', 'xterm') as executable names to spawn, which fails with ENOENT when those values aren't installed programs.

## Change summary

Spawning an external terminal on Linux fails with ENOENT when $COLORTERM is 'truecolor' or $TERM is 'xterm' because the code uses those environment variable values as executable paths.

## Best solution

Fix the fallback logic in `LinuxExternalTerminalService.getDefaultTerminalLinuxReady()` to validate that the resolved terminal name is an actual executable (e.g. via `which`) before using it, or filter out known non-executable values like 'truecolor'. The COLORTERM fallback at line 309 is particularly bad since VS Code itself sets COLORTERM='truecolor' in terminalEnvironment.ts.
