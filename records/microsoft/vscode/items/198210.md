# Issue #198210 — Git - error while synching changes

[https://github.com/microsoft/vscode/issues/198210](https://github.com/microsoft/vscode/issues/198210)

## Summary

Team-authored issue, must stay open. The bug is real and still present on main: the sync command does not pass an explicit merge strategy flag to `git pull`, so git ≥2.39 errors out on divergent branches when no `pull.rebase` default is configured. An open community PR (https://github.com/microsoft/vscode/pull/307253) is attempting a fix.

## Change summary

Clicking 'Sync Changes' fails with a confusing git error about divergent branches when `git.rebaseWhenSync` is false and no global `pull.rebase` is set.

## Best solution

The sync/pull path in `extensions/git/src/commands.ts` (around line 4300) should pass `--no-rebase` (or `--rebase` when configured) explicitly so git never falls back to an unconfigured default. Alternatively, catch the specific 'divergent branches' error and surface a helpful VS Code notification guiding the user. The open PR https://github.com/microsoft/vscode/pull/307253 takes the latter approach — reviewing and landing (or closing) that PR is the next step.
