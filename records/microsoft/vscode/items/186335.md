# Issue #186335 — Diff Editor - editor side context key issue

[https://github.com/microsoft/vscode/issues/186335](https://github.com/microsoft/vscode/issues/186335)

## Summary

Still present on main: the original fix (854108f7d2a) was reverted in 538412ba9de and the context keys for diff editor sides still use focus/blur events rather than a scoped context key service, reproducing the macOS-only right-click context menu issue.

## Change summary

Right-clicking the left side of the diff editor loses the Stage/Unstage/Revert Selected Ranges context menu commands on the second invocation (macOS only) because the editor-side context key is driven by focus/blur rather than DOM-scoped context.

## Best solution

Replace the focus/blur-driven `isInDiffLeftEditor` / `isInDiffRightEditor` context keys with a context key service scoped to each editor's DOM element, so the key reflects which side the context menu targets regardless of focus timing. The fix lives in `src/vs/editor/browser/widget/diffEditor/components/diffEditorEditors.ts`; the previous attempt (PR #186997) was reverted due to side effects — the next attempt needs to avoid whatever regression caused the revert.
