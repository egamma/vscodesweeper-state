# Issue #199604 — Diff chunks not minimal if last line of added block equals last line of previous block, but differs by indentation

[https://github.com/microsoft/vscode/issues/199604](https://github.com/microsoft/vscode/issues/199604)

## Summary

Still reproduces on main: the left-shift pass in `joinSequenceDiffsByShifting` uses trimmed hashes (`getElement`) rather than full-line comparison (`isStronglyEqual`), allowing diff chunks to shift through lines that differ only by indentation, producing non-minimal results.

## Change summary

Diff chunks include unchanged lines when an added block's last line matches a nearby line after trimming whitespace.

## Best solution

The left-shift pass in `heuristicSequenceOptimizations.ts` (lines 50-56) should use `isStronglyEqual` (or an equivalent full-line comparison) instead of `getElement` when checking whether a diff can be shifted, matching the behavior of the right-shift pass. This would prevent merging through lines that differ only by indentation.
