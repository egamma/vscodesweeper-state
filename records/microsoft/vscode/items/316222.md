# Issue #316222 — Run in terminal sync mode returns stale `^C` in Remote-SSH Windows/PowerShell

[https://github.com/microsoft/vscode/issues/316222](https://github.com/microsoft/vscode/issues/316222)

## Summary

Still reproduces on main: the pre-command Ctrl+C generates a spurious shell integration command lifecycle that consumes the `_nextCommandId` and wins the `onCommandFinished` race before the real command is sent. No fix has landed for this specific ordering issue.

## Change summary

In Remote-SSH Windows/PowerShell, the run_in_terminal sync mode returns stale `^C` output instead of the actual command's stdout because the cleanup Ctrl+C triggers an empty command-finished event that resolves the completion race early.

## Best solution

Fix the ordering in `terminalInstance.ts#runCommand`: either defer `setNextCommandId` until after the Ctrl+C cleanup completes (so the empty command doesn't consume the id), or add a guard in `RichExecuteStrategy` to also filter out command-finished events that fire before `runCommand` actually sends the command text. The `_ensureCurrentCommandId` in `commandDetectionCapability.ts` should also validate command-line content before consuming `_nextCommandId`.
