# Issue #316222 — Run in terminal sync mode returns stale `^C` in Remote-SSH Windows/PowerShell

[https://github.com/microsoft/vscode/issues/316222](https://github.com/microsoft/vscode/issues/316222)

## Summary

Still reproduces on main: the Ctrl+C cleanup in `runCommand` can produce a full shell-integration command lifecycle on Windows/PowerShell that wins the `onCommandFinished` race in RichExecuteStrategy before the real command is sent. No fix has been merged.

## Change summary

In sync mode via Remote-SSH to Windows/PowerShell, `run_in_terminal` returns stale `^C` output instead of the actual command output because the pre-command Ctrl+C triggers an empty `onCommandFinished` event that resolves the completion race early.

## Best solution

Move `setNextCommandId` to after the Ctrl+C cleanup (or defer the `onCommandFinished` subscription until after the real command is sent), so the empty Ctrl+C command cannot consume the command ID or win the completion race. The fix touches `src/vs/workbench/contrib/terminal/browser/terminalInstance.ts` (reorder lines 1028-1041) and possibly the stale-marker filter in `richExecuteStrategy.ts`.
