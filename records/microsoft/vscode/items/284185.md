# Issue #284185 — Worktrees not preserving last selected location

[https://github.com/microsoft/vscode/issues/284185](https://github.com/microsoft/vscode/issues/284185)

## Summary

Confirmed bug: the worktree root storage is never saved on the first custom worktree creation because of a faulty guard condition. Issue is on the 'On Deck' milestone, so it stays open.

## Change summary

When creating a worktree in a custom directory, the next worktree creation does not remember the last-used location and reverts to the default path.

## Best solution

In `extensions/git/src/repository.ts` line 1971, change the condition from `if (defaultWorktreeRoot && !pathEquals(...))` to `if (!defaultWorktreeRoot || !pathEquals(...))` so that the worktree root is persisted on the first custom creation as well as on subsequent changes.

- **buggy condition:** extensions/git/src/repository.ts line 1971: `if (defaultWorktreeRoot && !pathEquals(newWorktreeRoot, defaultWorktreeRoot))` — when `defaultWorktreeRoot` is undefined (first custom worktree), the condition short-circuits to false and the new root is never saved to global state.
- **two read sites:** extensions/git/src/commands.ts lines 3627 and 3663 read `WORKTREE_ROOT_STORAGE_KEY` from global state to populate the default path in the input box, but only extensions/git/src/repository.ts line 1972 writes it — and only when it was already set.
