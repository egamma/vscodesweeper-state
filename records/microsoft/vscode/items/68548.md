# Issue #68548 — [html] [custom data] allow to give custom HTML tags/attributes a higher completion rank

[https://github.com/microsoft/vscode/issues/68548](https://github.com/microsoft/vscode/issues/68548)

## Summary

Still unimplemented on main. CSS custom data gained a `relevance` field (per the member comment), but the HTML language service has no equivalent — the feature request remains valid.

## Change summary

Allow custom HTML tags/attributes to specify a completion rank so they appear higher in IntelliSense results.

## Best solution

Add a `relevance` (or equivalent sort priority) field to the HTML custom data schema in vscode-html-languageservice (ITagData/IAttributeData), mirroring what was done for CSS in microsoft/vscode-css-languageservice@2588430. The completion provider should then use that field to set `sortText` on completion items.
