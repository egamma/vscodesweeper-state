# Issue #323808 — Telemetry: `agentHost.turnCompleted.permissionLevel` is provider-specific and shouldn't live on the generic turn event

[https://github.com/microsoft/vscode/issues/323808](https://github.com/microsoft/vscode/issues/323808)

## Summary

Team-authored issue (MEMBER) requesting telemetry refactoring — keeping open for maintainer judgment. The ask is well-scoped: move a provider-specific field off a generic event.

## Change summary

The shared `agentHost.turnCompleted` telemetry event carries a `permissionLevel` field that only the Copilot provider populates, making it misleadingly empty for Claude and Codex harnesses.

## Best solution

Remove `permissionLevel` from the shared turn-completed event in `agentHostTelemetryReporter.ts` and `agentHostTurnTracker.ts`, then emit provider-specific permission/config telemetry from each harness's own namespaced event (e.g. `agentHost.copilot.*`). Also address the timing warm-up discrepancy noted in comments.
