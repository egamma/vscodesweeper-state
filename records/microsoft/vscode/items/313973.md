# Issue #313973 — Failed to inline attachments: getting signed attachment URLs for images

[https://github.com/microsoft/vscode/issues/313973](https://github.com/microsoft/vscode/issues/313973)

## Summary

Team-authored upstream bug: the error originates in the GitHub Copilot backend's attachment-signing service (twirp errors 'not_found' and 'permission_denied'), not in VS Code client code. Multiple team members confirm it is recurring and a top error for Anthropic models. Keeping open per hard keep-open rule (MEMBER author) and because the upstream fix is still outstanding.

## Change summary

Image attachments in Copilot chat sessions fail with server-side 'attachment not found' or 'permission_denied' errors after images expire or become inaccessible, breaking multi-turn conversations that reference images.

## Best solution

The immediate fix lives upstream in the GitHub Copilot backend's attachment-signing service. On the VS Code side, the broader mitigation is tracked by https://github.com/microsoft/vscode/issues/314556 — moving image persistence from inline base64 / remote-URL-dependent references to a local session-scoped resource store so that image availability does not depend on remote signed URLs. Until the backend fix lands, the client-side persistence redesign in #314556 would also reduce exposure to this class of failure.
