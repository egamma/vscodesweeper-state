# Issue #207977 — Split View does not account for 1px border

[https://github.com/microsoft/vscode/issues/207977](https://github.com/microsoft/vscode/issues/207977)

## Summary

Still present on main: the split-view separator border is drawn as an absolutely-positioned ::before pseudo-element inside the view's bounds without reserving space, so view content (e.g. tabs) renders underneath it. Confirmed by benibenj as a real SplitView problem.

## Change summary

The 1px separator border in split views overlaps view content because it doesn't reserve layout space.

## Best solution

Account for the 1px border in the split-view layout calculation (either offset the view content by 1px via padding/margin on the second child, or adjust the positioning logic in splitview.ts so the border doesn't overlap the content area).
