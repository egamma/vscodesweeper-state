# Issue #243119 — alert when code action fails

[https://github.com/microsoft/vscode/issues/243119](https://github.com/microsoft/vscode/issues/243119)

## Summary

Team-authored issue requesting that code-action resolve failures surface an error notification to the user instead of being silently swallowed. The code on current main still silently catches the error via `onUnexpectedExternalError` (which logs but never alerts the user). No fix has landed; keeping open per team-authored policy.

## Change summary

When `resolveCodeAction` throws, the error is silently caught and the user is never notified, preventing them from reporting the failure.

## Best solution

In `CodeActionItem.resolve()` (`src/vs/editor/contrib/codeAction/common/types.ts`), after catching the error, surface a user-visible notification (e.g. via `INotificationService`) so users can see and report code-action resolution failures instead of experiencing silent no-ops.
