# Issue #232070 — Git - diff view does not work for PDF files

[https://github.com/microsoft/vscode/issues/232070](https://github.com/microsoft/vscode/issues/232070)

## Summary

Still present on main: the git file system provider returns binary content via `git show --textconv` which is then interpreted as text by the editor, causing incorrect display of older revisions of binary files (PDFs). No fix has landed.

## Change summary

Git diff view shows incorrect content (single character) for non-HEAD revisions of PDF/binary files because binary data is interpreted as text.

## Best solution

The git FileSystemProvider's `readFile` path (`extensions/git/src/fileSystemProvider.ts`) returns binary Buffer content that the text editor misinterprets. The fix likely involves either detecting binary content and signaling it appropriately, or ensuring the editor layer handles non-text content from the git scheme correctly. Related to but distinct from #154815 (respecting -diff attribute).
