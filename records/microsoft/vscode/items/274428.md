# Issue #274428 — Chat response: revealing the last output sometimes stuck

[https://github.com/microsoft/vscode/issues/274428](https://github.com/microsoft/vscode/issues/274428)

## Summary

Valid, team-acknowledged chat scrolling bug that remains unfixed on main. A fix attempt (PR #284652) was closed without merging. The root cause — codeblock layout updates falling out of sync with the renderer's reveal/autoscroll — is confirmed by a maintainer and still present in the current code.

## Change summary

Chat response auto-scrolling stops prematurely when a code block editor appears during streaming, leaving the user stranded mid-response.

## Best solution

Implement reliable autoscroll in the chat ListView that survives asynchronous codeblock layout changes. The closed PR #284652 attempted this at the ListView level (`src/vs/base/browser/ui/list/listView.ts`) — that approach or a variant should be revisited and landed. The chat widget scroll logic lives in `src/vs/workbench/contrib/chat/browser/widget/chatWidget.ts`.
