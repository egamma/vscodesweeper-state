[https://github.com/microsoft/vscode/issues/301920](https://github.com/microsoft/vscode/issues/301920)

## Summary

Plausible crash scenario — bulk submodule deletion likely overwhelms the git extension with parallel file-watcher events and git operations — but the report lacks crash logs, process monitor output, or a minimal reproduction repo, making it unverifiable as filed.

## Change summary

VS Code crashes or requires a window reload when many git submodules are deleted at once via `git rm -rf` while the Git panel is open.

## Best solution

Ask the reporter for crash logs (Help → Open Crash Report Folder) and a minimal reproduction repo. The likely root cause is the file-watcher storm in `extensions/git/src/repository.ts` triggering parallel `updateModelState` calls for each disappearing submodule repository, combined with the `NotAGitRepository` → `RepositoryState.Disposed` path. A fix would involve debouncing or batching the disposal cascade when many submodule repositories disappear simultaneously.
