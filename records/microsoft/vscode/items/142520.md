# Issue #142520 — Opening a terminal in Multi-root workspace appends cwd option for new terminal. 

[https://github.com/microsoft/vscode/issues/142520](https://github.com/microsoft/vscode/issues/142520)

## Summary

The bug remains: the `newWithProfile` terminal action (Ctrl+Shift+') uses `PICK_WORKSPACE_FOLDER_COMMAND_ID` which doesn't account for `terminal.integrated.cwd`, unlike the `TerminalCommandId.New` action which uses the config-aware `pickTerminalCwd`. A maintainer previously grouped this with related issues without closing as duplicate.

## Change summary

In multi-root workspaces, creating a new terminal via workspace folder picker ignores or misapplies the `terminal.integrated.cwd` setting, either concatenating paths or always overriding the user's folder selection.

## Best solution

The `newWithProfile` action (line ~1617 in terminalActions.ts) should use the config-aware `pickTerminalCwd` function (or its resolved cwd) instead of the raw `PICK_WORKSPACE_FOLDER_COMMAND_ID`, so that `terminal.integrated.cwd` interacts correctly with the user's workspace folder choice rather than blindly overriding or concatenating.
