# Issue #324292 — Prompt Caching Expiration shown in Session/Chat

[https://github.com/microsoft/vscode/issues/324292](https://github.com/microsoft/vscode/issues/324292)

## Summary

Valid feature request for surfacing prompt-cache expiration and hit/miss info directly in the session/chat UI. VS Code already has an internal Cache Explorer in the debug panel, but nothing user-facing. The meta billing issue #319403 and observability issue #293225 are related but neither specifically owns this ask — keeping open for routing.

## Change summary

Show prompt caching expiration timing and cache-hit status per response in the sessions list and chat UI so users can prioritize sessions for cost efficiency.

## Best solution

Route to the chat-billing / agent-observability area owners. The existing Cache Explorer infrastructure in `src/vs/workbench/contrib/chat/browser/chatDebug/` already computes cache expiration insights — the work would be surfacing a subset of that data (cache hit rate, estimated TTL remaining) in the user-facing session list and per-response details. Could also be addressed as part of https://github.com/microsoft/vscode/issues/319403 (usage-based billing UX) or https://github.com/microsoft/vscode/issues/293225 (agent observability).
