# Issue #39152 — Git - Support git subtree

[https://github.com/microsoft/vscode/issues/39152](https://github.com/microsoft/vscode/issues/39152)

## Summary

Legitimate, well-upvoted feature request (88 👍) for git subtree command support in the built-in git extension. No implementation exists on current main; the request remains valid and is properly parked in the Backlog milestone.

## Change summary

Add git subtree commands (add, push, pull) to VS Code's built-in git extension.

## Best solution

Implement git subtree commands in `extensions/git/` or, given the niche audience relative to core git operations, consider whether an `*extension-candidate` verdict is more appropriate — a dedicated marketplace extension could serve subtree users without expanding the built-in git surface. If kept in core, the work would live in `extensions/git/src/commands.ts` and related files, adding subtree add/push/pull alongside existing submodule support.
