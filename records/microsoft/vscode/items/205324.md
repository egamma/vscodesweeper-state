# Issue #205324 — Scrolling bug with trackpad & smooth scrolling enabled

[https://github.com/microsoft/vscode/issues/205324](https://github.com/microsoft/vscode/issues/205324)

## Summary

This is a plausible editor scrolling bug, but it is hardware/driver-specific, was reported on VS Code 1.86.2, and has no follow-up confirming whether it still reproduces on current builds. Ask for a current repro before deciding whether this is VS Code, the Windows input stack, or the Magic Utilities driver.

## Change summary

With an Apple Magic Trackpad using Magic Utilities on Windows 11, smooth editor scrolling drifts horizontally while scrolling vertically.

## Best solution

Keep the issue in the Windows/trackpad scrolling area until the reporter confirms current behavior on the latest Stable or Insiders, including whether `editor.scrollPredominantAxis` is enabled and whether the same wheel deltas reproduce outside VS Code. If it still reproduces, investigate `src/vs/base/browser/ui/scrollbar/scrollableElement.ts` smooth scrolling and predominant-axis handling.

## Evidence

- **report details:** The issue was filed against VS Code 1.86.2 on Windows 11 with an Apple Magic Trackpad using the third-party Magic Utilities driver; the reporter could not test another trackpad.
- **workarounds:** The reporter says the drift disappears when VS Code smooth scrolling is disabled, and also disappears when Magic Utilities' own 'scroll only vertical or horizontal' option is enabled.
- **current code:** Current main has `editor.scrollPredominantAxis` defaulting to true in `src/vs/editor/common/config/editorOptions.ts`, described as preventing horizontal drift on a trackpad, and `src/vs/base/browser/ui/scrollbar/scrollableElement.ts` zeros the non-predominant wheel delta when that option is enabled.
