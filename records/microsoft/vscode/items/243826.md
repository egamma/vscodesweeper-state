# Issue #243826 — theme-seti overrides langauge extension icons by file extension, not by language

[https://github.com/microsoft/vscode/issues/243826](https://github.com/microsoft/vscode/issues/243826)

## Summary

Valid bug with a team-authored fix PR (#244872) still open. The seti theme's generated mapping assigns CSS icon to `.sss` files, overriding language extension icons. The fix adds an ignore list to the update script but hasn't merged yet.

## Change summary

theme-seti assigns CSS icon to `.sss` (and potentially other) file extensions, overriding language-contributed icons because file-extension match has higher precedence than language match.

## Best solution

Merge PR #244872 which adds an ignore list to `extensions/theme-seti/build/update-icon-theme.js` so that extensions like `.sss` that no longer meaningfully map to CSS are excluded from the generated theme, letting language-contributed icons take effect.
