# Issue #243826 — theme-seti overrides langauge extension icons by file extension, not by language

[https://github.com/microsoft/vscode/issues/243826](https://github.com/microsoft/vscode/issues/243826)

## Summary

Still reproduces on main: `.sss` is mapped to the CSS icon in theme-seti. The maintainer's fix PR (https://github.com/microsoft/vscode/pull/244872) is still open and unmerged after 15+ months.

## Change summary

theme-seti overrides a language extension's custom icon for `.sss` files because the update script maps that extension to the CSS icon, taking precedence over the language-defined icon.

## Best solution

Merge or revive PR https://github.com/microsoft/vscode/pull/244872, which adds `.sss` to the `ignoreExtAssociation` list in `extensions/theme-seti/build/update-icon-theme.js` and regenerates the icon theme JSON. The broader design (file-extension precedence over language match) is intentional and won't change.
