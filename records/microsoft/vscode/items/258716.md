## Summary

Team-authored issue identifying a real activation-event gap in the Python extension: terminal completions require a Python file to be open because there's no activation event for starting a Python REPL. Keeping open per policy.

## Change summary

Python terminal completions don't appear until a Python file is opened because the Python extension lacks an activation event for Python REPL start.

## Best solution

Add an activation event in the Python extension (microsoft/vscode-python) that triggers on Python REPL launch in the terminal, so shell integration completions work without needing a .py file open. The fix likely lives in the Python extension's package.json activation events and pythonrc.py.
