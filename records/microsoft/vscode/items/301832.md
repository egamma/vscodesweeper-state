# Issue #301832 — Search pane compacts folders despite "explorer.compactFolders": false in settings

[https://github.com/microsoft/vscode/issues/301832](https://github.com/microsoft/vscode/issues/301832)

## Summary

Confirmed bug: the search results tree never wires up `explorer.compactFolders`, so folder compression is always on regardless of the setting. An open PR (not yet merged) addresses it.

## Change summary

The Search pane's tree view ignores the `explorer.compactFolders: false` setting and always compacts folders.

## Best solution

Land the open PR https://github.com/microsoft/vscode/pull/325654 which wires the `explorer.compactFolders` setting into the search results tree's `compressionEnabled` option, with live updates when the setting changes.
