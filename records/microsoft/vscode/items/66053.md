# Issue #66053 — [css][html] SVG language support

[https://github.com/microsoft/vscode/issues/66053](https://github.com/microsoft/vscode/issues/66053)

## Summary

Valid, well-upvoted feature request (44 👍) for SVG language support in HTML/CSS. No implementation exists on current main — the HTML and CSS language feature extensions contain zero SVG-specific logic. Should remain open in the Backlog.

## Change summary

VS Code lacks SVG-specific auto-completion, syntax highlighting, and Emmet support in HTML and CSS files.

## Best solution

Implement SVG tag/attribute data via custom data providers in vscode-html-languageservice and vscode-css-languageservice (the custom data mechanism referenced in the issue already exists). Alternatively, endorse a community extension like jock.svg for standalone .svg files while adding inline SVG support to the built-in HTML language features.
