# Issue #66053 — [css][html] SVG language support

[https://github.com/microsoft/vscode/issues/66053](https://github.com/microsoft/vscode/issues/66053)

## Summary

SVG language support (completions, syntax highlighting in HTML/CSS) has not been implemented on main. The HTML/CSS language features extensions have no SVG-specific custom data or tag/attribute definitions. This remains a valid, unaddressed feature request.

## Change summary

Request for SVG tag/attribute completions, syntax highlighting, and emmet support in HTML and CSS files.

## Best solution

Provide SVG custom data (tag/attribute definitions) via the existing HTML custom data provider mechanism in `extensions/html-language-features/`, or treat this as an `*extension-candidate` since the custom data API already allows extensions like jock.svg to provide this. Given the infrastructure exists for custom data providers, an official SVG data package could be added to the built-in HTML extension or published as a first-party extension.
