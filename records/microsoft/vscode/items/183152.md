# Issue #183152 — Git - after putting password in the cursor should go back to the terminal/cli window

[https://github.com/microsoft/vscode/issues/183152](https://github.com/microsoft/vscode/issues/183152)

## Summary

Still reproducible on main: the git askpass handler uses `window.showInputBox()` without any mechanism to restore focus to the originating terminal after dismissal. No fix found in git log for this file or the terminal focus code.

## Change summary

After entering git credentials via VS Code's askpass input box (triggered from terminal), focus goes to the last active editor instead of returning to the terminal.

## Best solution

After the askpass `showInputBox` resolves in `extensions/git/src/askpass.ts`, the terminal that triggered the credential prompt should be re-focused. This likely requires the askpass handler to track which terminal initiated the git operation and call `terminal.show()` or equivalent after the input box closes.
