# Issue #250393 — `Done` versus `Cancel` when selection mode is chosen

[https://github.com/microsoft/vscode/issues/250393](https://github.com/microsoft/vscode/issues/250393)

## Summary

Valid polish issue filed by a team member: the integrated browser's selection toolbar shows 'Done' when continuous selection is active but 'Cancel' when single-element selection is active — the labels should be consistent. Hard keep-open rule applies (author is MEMBER).

## Change summary

The integrated browser's selection mode toolbar button inconsistently labels itself 'Done' (continuous selection) vs 'Cancel' (select an element) for what is essentially the same stop-selection action.

## Best solution

Unify the toggled button label for both selection modes in the browser editor chat features — likely in the `AddElementToChatAction` and `AddAreaScreenshotToChatAction` action definitions in `src/vs/workbench/contrib/browserView/electron-browser/features/browserEditorChatFeatures.ts`, or in the split-button submenu rendering. Both should show the same label (likely 'Done' or 'Cancel') when their respective selection mode is active.
