# Issue #112730 — Offer to open a workspace from empty explorer

[https://github.com/microsoft/vscode/issues/112730](https://github.com/microsoft/vscode/issues/112730)

## Summary

Still unimplemented on main — the empty explorer welcome view uses single-command link buttons with no dropdown/split-button support. The issue carries the On Deck milestone, which is a scheduling decision that blocks a close.

## Change summary

Add a split/dropdown button to the empty explorer welcome view so users can also pick 'Open Workspace' (and extensions can contribute additional actions) without cluttering the view with extra buttons.

## Best solution

Implement the split-button pattern in the welcome-view infrastructure (likely `src/vs/workbench/contrib/files/browser/explorerViewlet.ts` welcome content registration and the underlying welcome-view rendering in `src/vs/workbench/browser/parts/views/`) so the 'Open Folder' button gains a dropdown with 'Open Workspace' and extensible secondary actions. Comment 19 by JacksonKearl proposed a concrete markdown syntax (`[link1](cmd:...) [link2](cmd:...)` on one line → dropdown) worth revisiting.
