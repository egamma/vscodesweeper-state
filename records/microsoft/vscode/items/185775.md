# Issue #185775 — [wsl] Automatically give vscode wsl sh scripts +x

[https://github.com/microsoft/vscode/issues/185775](https://github.com/microsoft/vscode/issues/185775)

## Summary

Legitimate feature request explicitly placed in Backlog by a maintainer after the community process ran. The scenario (wsl.conf fmask settings breaking VS Code shell script permissions on update) is real but niche. Keeping open as accepted-but-unscheduled work.

## Change summary

VS Code WSL update/install scripts lose the executable bit when wsl.conf sets restrictive fmask mount options, requiring manual chmod after every update.

## Best solution

The VS Code WSL server bootstrap or update script should explicitly `chmod +x` the relevant shell scripts after extracting/copying them, ensuring they work regardless of the user's wsl.conf automount fmask settings. This likely belongs in the remote WSL extension or the server install script.
