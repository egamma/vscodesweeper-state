# Issue #206284 — Quick search: flicker when typing

[https://github.com/microsoft/vscode/issues/206284](https://github.com/microsoft/vscode/issues/206284)

## Summary

The flicker is still present on current main: each keystroke in Quick Search triggers a full new search via `doSearch()` which clears previous results before populating new ones, rather than filtering existing results incrementally. The debounce commit 4c742de8cf1 addressed editor preview flicker but not the results list flicker. Author is a team contributor, so this stays open regardless.

## Change summary

Quick Search results list flickers on each keystroke because existing results are cleared and replaced rather than filtered incrementally.

## Best solution

When the user types additional characters (narrowing the query), filter the already-displayed results client-side while the new search runs in the background, similar to how file search (quick open) works. The change would be in `_getPicks` / `doSearch` in `src/vs/workbench/contrib/search/browser/quickTextSearch/textSearchQuickAccess.ts` — retain the previous pick items as a baseline and merge incrementally with incoming results rather than clearing first.
