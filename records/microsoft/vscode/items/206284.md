# Issue #206284 — Quick search: flicker when typing

[https://github.com/microsoft/vscode/issues/206284](https://github.com/microsoft/vscode/issues/206284)

## Summary

The flicker remains on current main — when typing in Quick Search, the picker clears existing results before async results arrive because `_getPicks` starts fresh on each keystroke. The `mergeDelay` anti-flicker pattern exists in the base class but isn't fully addressing this case. The comment thread has a viable approach (filter existing results instead of clearing).

## Change summary

Quick Search results flicker (disappear and reappear) when typing because existing picks are replaced on each keystroke before new async results arrive.

## Best solution

Implement client-side filtering of existing results while the new search runs, similar to how file search (quick open) handles incremental typing — likely in `textSearchQuickAccess.ts` by retaining previous picks until new ones are ready, or using the `mergeDelay` option in `FastAndSlowPicks`.
