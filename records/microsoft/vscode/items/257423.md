# Issue #257423 — Git:Autofetch failing for private Github repositories

[https://github.com/microsoft/vscode/issues/257423](https://github.com/microsoft/vscode/issues/257423)

## Summary

Still reproduces on main: autofetch passes `silent: true`, which sets `VSCODE_GIT_FETCH_SILENT`, causing `askpass-main.ts` to exit before requesting credentials. Private repos that need credential helper interaction always fail on autofetch while manual fetch works. The reporter provided full trace logs confirming the behavior; the maintainer conversation stalled after the initial info exchange.

## Change summary

Git autofetch fails for private GitHub repositories because silent fetches skip the askpass credential flow entirely, while manual fetch succeeds.

## Best solution

The silent fetch path in `askpass-main.ts` (line 28–29) needs a more nuanced approach — instead of skipping askpass entirely for silent fetches, it should attempt to retrieve cached/stored credentials without prompting the user (e.g., check the credential helper non-interactively). If no cached credential is available, then skip silently as today. This keeps the no-prompt guarantee while allowing repos with stored tokens to autofetch. The fix touches `extensions/git/src/askpass-main.ts` and possibly `extensions/git/src/askpass.ts`.
