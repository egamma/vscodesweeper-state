# Issue #183906 — Git - Remove all local branches which origin is deleted (Git Fetch Prune)

[https://github.com/microsoft/vscode/issues/183906](https://github.com/microsoft/vscode/issues/183906)

## Summary

Well-supported feature request (159 upvotes, well above the 20-vote threshold) with an open PR. The built-in git extension already has `Git: Fetch (Prune)` but lacks the second step — automatically deleting local branches whose upstream is gone. The open PR #205031 targets exactly this gap; keep the issue open until that PR is resolved.

## Change summary

After `git fetch --prune`, automatically delete local branches whose remote tracking branch is gone, instead of requiring manual one-by-one deletion.

## Best solution

Review and land (or close with feedback) the open PR https://github.com/microsoft/vscode/pull/205031, which adds a combined fetch-prune-and-delete command to the git extension. The implementation would live in `extensions/git/src/commands.ts` and `extensions/git/src/repository.ts`, detecting branches marked `[gone]` after a prune fetch and offering to delete them.
