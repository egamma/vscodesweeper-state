# Issue #172108 — Document Content Providers are being passed URI-unescaped paths when invoked through the DAP

[https://github.com/microsoft/vscode/issues/172108](https://github.com/microsoft/vscode/issues/172108)

## Summary

Still present on main: `URI.parse` at debugSource.ts:143 decodes percent-encoded characters in DAP source paths before passing them to document content providers. The maintainer discussion (comments 7–14) acknowledged the problem but concluded it's entangled with VS Code's long-standing URI semantics and no safe fix path was identified. Author is a MEMBER — keeping open for maintainer judgment.

## Change summary

Document content providers receive URI-unescaped paths when invoked through DAP, breaking filenames that contain characters requiring percent-encoding.

## Best solution

The fix lives in `src/vs/workbench/contrib/debug/common/debugSource.ts` line 143 — the `URI.parse(path)` call needs to preserve percent-encoded characters. The team discussion explored using `new URL` + `URI.from` or an opt-in `preserveEncodedURICharacters` flag, but both carry regression risk due to VS Code's internal URI normalization. A targeted solution for the DAP path (e.g., parsing via `URL` then constructing with `URI.from`) is the most promising approach, scoped narrowly to `getUriFromSource`.
