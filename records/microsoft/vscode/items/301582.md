# Issue #301582 — create_file tool corrupts file content for Go files

## Summary

The create_file corruption (duplicated content, chunk reversal, tab→newline substitution) was caused by the tool streaming content line-by-line instead of writing it in a single batch. This was fixed on main and shipped in 1.116.0.

## Change summary

The create_file tool corrupted Go file content with duplicate declarations, reversed chunks, and tab→newline substitution when writing files.

Keep the shipped fix — commit f6e4a3243e9 replaced the line-by-line streaming approach with a single-batch insert, and 641cbea24b3 closed a remaining duplication edge case. No further change needed.
