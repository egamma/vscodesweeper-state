# Issue #285334 — Opus doesn't support SVGs?

[https://github.com/microsoft/vscode/issues/285334](https://github.com/microsoft/vscode/issues/285334)

## Summary

Team-authored issue (Tyriar is a contributor) reporting that SVG files cannot be used as text context in chat with Opus. Current code in `chatAttachmentResolveService.ts` fully omits SVG attachments (`OmittedState.Full`). Labeled `upstream` — the root cause is model-level SVG/image support. Keeping open per team-author policy.

## Change summary

SVG files attached to chat are omitted or treated as images instead of being sent as text/XML content to the model.

## Best solution

Either treat SVGs as text files (XML) when attaching to chat context — removing the `OmittedState.Full` override in `chatAttachmentResolveService.ts` line 124–126 — or wait for upstream model support for SVG images. The VS Code side fix is small (stop omitting `.svg` and let it be sent as text), but the issue is labeled upstream, suggesting the intent was to send it as an image attachment and have the model understand it.
