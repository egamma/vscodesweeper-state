## Summary

Valid bug with a confirmed reproduction path (activity bar at top + view moved to secondary sidebar causes wrong tab on reload). An open PR (#288264) already proposes a fix — keeping open until it lands.

## Change summary

When activity bar is at top and a view like Outline is moved to the secondary sidebar, reloading selects the wrong activity tab (e.g. Search instead of Explorer).

## Best solution

Review and land the fix in https://github.com/microsoft/vscode/pull/288264, which removes a premature `!viewContainer` guard in `shouldBeHidden()` so that builtin containers cached as visible aren't incorrectly hidden during initialization.
