# Issue #233536 — Can’t use Cmd-W to close editors/windows on iPad

[https://github.com/microsoft/vscode/issues/233536](https://github.com/microsoft/vscode/issues/233536)

## Summary

Confirmed upstream iPadOS limitation: the OS intercepts Cmd-W (and other system shortcuts) before they reach web apps, so vscode.dev never receives the keypress. No fix is possible on the VS Code side. Keeping open as an upstream tracker since it carries the Backlog milestone.

## Change summary

Cmd-W keyboard shortcut is intercepted by iPadOS and never delivered to vscode.dev, preventing editor/window close.

## Best solution

Reclassify as `upstream` — VS Code cannot work around iPadOS intercepting system shortcuts before they reach Safari/WebKit web apps. The existing workaround (rebinding to a different shortcut like Cmd-Shift-W) is the best users can do until Apple changes behavior. Consider documenting this limitation in the iPadOS known-issues wiki.
