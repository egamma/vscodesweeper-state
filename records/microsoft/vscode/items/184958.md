# Issue #184958 — Fix a browser sniff for Firefox support of async clipboard

[https://github.com/microsoft/vscode/issues/184958](https://github.com/microsoft/vscode/issues/184958)

## Summary

Still reproduces on main: the `isFirefox` sniff at line 34 of clipboard.ts forces the deprecated `queryCommandSupported('paste')` path even though Firefox now ships the async clipboard API, disabling paste functionality in vscode.dev on Firefox.

## Change summary

Firefox users on vscode.dev cannot paste because a stale browser sniff bypasses the async clipboard API detection.

## Best solution

Remove the `browser.isFirefox` guard from the `supportsPaste` check in `src/vs/editor/contrib/clipboard/browser/clipboard.ts` (line 34) and rely on `'readText' in navigator.clipboard` or simply treat clipboard presence as sufficient, matching the non-Firefox path. The outdated MDN comment above it should also be removed.
