# Issue #92902 — Hyperlinks don't open on Arch Linux running KDE Plasma 5

[https://github.com/microsoft/vscode/issues/92902](https://github.com/microsoft/vscode/issues/92902)

## Summary

Confirmed snap-specific issue where xdg-open gets orphaned (ppid=1) due to snap confinement interfering with the Electron shell.openExternal flow. Already correctly labeled upstream/snap/electron. The non-snap installation works fine (Comment 7). Keeping open as the upstream labels are appropriate and the snap package is still distributed.

## Change summary

Hyperlinks fail to open a browser when VS Code is installed via snap on Arch Linux with KDE Plasma, because snap's confinement orphans the xdg-open child process.

## Best solution

This is a snap confinement issue, not a VS Code core bug. The fix belongs in the snap package's confinement configuration (allowing proper xdg-open propagation) or in Electron's child-process handling under snap. The workaround is to use the non-snap installation (e.g. AUR or .deb). If the snap team resolves the confinement issue, this can be closed.
