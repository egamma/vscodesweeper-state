# Issue #138727 — support theme icons in menus in the shadow dom

[https://github.com/microsoft/vscode/issues/138727](https://github.com/microsoft/vscode/issues/138727)

## Summary

Team-authored tech-debt issue still unresolved: the hardcoded `formatRule` calls for menu icons in shadow DOM remain on current main (lines 1029–1030 of menu.ts), preventing product icon themes from customizing menu icons there.

## Change summary

Menu icons in shadow DOM contexts ignore product icon theme overrides because codicon rules are hardcoded rather than sourced from the icon style sheet.

## Best solution

Pass icon definitions (font family + character code) from the platform theme service into the shadow DOM menu widget CSS, removing the hardcoded `formatRule` calls in `src/vs/base/browser/ui/menu/menu.ts` and allowing product icon themes to apply to menus rendered inside shadow DOMs.
