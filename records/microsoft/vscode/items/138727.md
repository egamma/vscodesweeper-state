# Issue #138727 — support theme icons in menus in the shadow dom

[https://github.com/microsoft/vscode/issues/138727](https://github.com/microsoft/vscode/issues/138727)

## Summary

Team-authored debt issue. The hardcoded formatRule lines at menu.ts:1029-1030 are still present on current main — the shadow DOM menu icon theming gap remains unresolved.

## Change summary

Product icon themes cannot override menu icons (selection checkmark, submenu arrow) when menus render inside a shadow DOM because the icon rules are hardcoded rather than using the dynamic icon style sheet.

## Best solution

Pass dynamic icon definitions (fontFamilyId + character code) into the shadow DOM menu styler so product icon themes can replace the menu selection and submenu icons. The fix lives in src/vs/base/browser/ui/menu/menu.ts (the formatRule calls) and likely requires coordination with src/vs/platform/theme/browser/iconsStyleSheet.ts.
