# Issue #174972 — Run to Cursor and Add to Watch available in JSON settings file context menu 

[https://github.com/microsoft/vscode/issues/174972](https://github.com/microsoft/vscode/issues/174972)

## Summary

Still reproduces on main: the debug context-menu items (Run to Cursor, Add to Watch, Evaluate in Debug Console, Add Inline Breakpoint) appear in any editor during a debug session, including JSON settings files, because their `contextMenuOpts.when` clauses only check `CONTEXT_IN_DEBUG_MODE` / `CONTEXT_DEBUGGERS_AVAILABLE` and never gate on whether the editor's language supports breakpoints.

## Change summary

Debug context menu items (Run to Cursor, Add to Watch, etc.) appear in settings.json and other files where breakpoints are explicitly disabled.

## Best solution

Introduce a per-editor context key (e.g. `editorLangSupportsBreakpoints`) that tracks `canSetBreakpointsIn(model)` and add it to the `when` / `contextMenuOpts.when` clauses of RunToCursorAction, SelectionToWatchExpressionsAction, SelectionToReplAction, and the inline breakpoint menu registration in `debugCommands.ts`. The key should update on active-editor and language-mode changes. The fix is localized to `src/vs/workbench/contrib/debug/browser/debugEditorActions.ts` and `src/vs/workbench/contrib/debug/browser/debugCommands.ts`, with the context key likely managed in `breakpointEditorContribution.ts`.
