# Issue #312925 — VS Code crashes (code 5) with Remote SSH + Copilot Chat codebase indexing on M1

[https://github.com/microsoft/vscode/issues/312925](https://github.com/microsoft/vscode/issues/312925)

## Summary

Confirmed crash (SIGSEGV, code 5) in the built-in Copilot Chat extension's semantic indexing on macOS Apple Silicon with Remote SSH. Multiple users reproduce it; the root cause appears to be in the WorkspaceChunkSearchService's native SQLite usage. Needs routing to the Copilot semantic search team for investigation.

## Change summary

VS Code window crashes with SIGSEGV ~40 seconds after connecting via Remote SSH when Copilot Chat's codebase semantic indexing starts on macOS Apple Silicon.

## Best solution

The crash is in `extensions/copilot/src/platform/workspaceChunkSearch/` — likely a native SQLite module issue on arm64 in a remote extension host context. The Copilot semantic search team should investigate the crash path triggered by the embedding type `metis-1024-I16-Binary` during remote indexing, and either fix the native module or add a graceful fallback when the native layer faults.
