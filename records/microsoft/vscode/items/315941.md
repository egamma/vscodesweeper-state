# Issue #315941 — Very high cpu usage from renderer/gpu process with ongoing chat

[https://github.com/microsoft/vscode/issues/315941](https://github.com/microsoft/vscode/issues/315941)

## Summary

Team-authored perf bug still present on main: the scrollbar fade CSS transition (`opacity 800ms` in scrollbars.css) forces the GPU compositor to re-rasterize the entire workbench layer on every frame during chat scrolling. Keeping open — team member issue requiring maintainer judgment.

## Change summary

Long-running chat with hardware acceleration disabled causes very high CPU in the renderer/GPU process due to scrollbar fade animation re-rasterizing the entire workbench compositing layer.

## Best solution

Isolate the scrollbar slider into its own compositing layer (e.g. `will-change: opacity` or `contain: paint` on the scrollbar element) so the fade transition only re-rasterizes the scrollbar, not the full workbench. The fix likely lives in `src/vs/base/browser/ui/scrollbar/media/scrollbars.css` and possibly the scrollbar widget TypeScript that manages visibility classes.
