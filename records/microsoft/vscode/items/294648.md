## Summary

Team-authored feature request for model-switching within a single Copilot Chat conversation — kept open per hard keep-open rule (MEMBER author). The request is for instructing the agent to use a different model mid-workflow (e.g. plan with one model, review with another).

## Change summary

Allow users to instruct Copilot Chat to switch to a different model for a specific step within a multi-step workflow (e.g. 'use model ABC to review the changes').

## Best solution

Implement model-switching support in the agent/chat orchestration layer so that a user prompt can request a different model for a sub-task (e.g. review pass). This likely touches the chat request routing and model selection logic in `src/vs/workbench/contrib/chat/` and the Copilot extension's agent harness. Needs product design on how model directives are parsed, which models are available, and how costs/permissions interact.
