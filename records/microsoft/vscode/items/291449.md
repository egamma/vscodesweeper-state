# Issue #291449 — Transition from chat workspace dropdown for signed out users

[https://github.com/microsoft/vscode/issues/291449](https://github.com/microsoft/vscode/issues/291449)

## Summary

Team-authored UX consistency issue: the chat workspace dropdown immediately transitions signed-out users to a new window but doesn't do the same for signed-in users. Keeping open — team member issue requires explicit maintainer judgment.

## Change summary

Make the chat workspace dropdown transition behavior consistent between signed-out and signed-in users.

## Best solution

Align the workspace picker behavior so signed-in users get the same immediate transition (or signed-out users get the same non-immediate flow) — likely involves changes in `src/vs/workbench/contrib/chat/browser/widget/input/workspacePickerActionItem.ts` and the delegate's `setSelectedWorkspace` logic. The team should decide which direction is correct (immediate transition for all, or deferred for all).
