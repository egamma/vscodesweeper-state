# Issue #325418 — Custom style for titleBar.inactiveBackground not applied when window was never active

[https://github.com/microsoft/vscode/issues/325418](https://github.com/microsoft/vscode/issues/325418)

## Summary

Confirmed source-reproducible: `isInactive` is hardcoded to `false` at construction, and no initial focus-state check is performed — so a window that was never activated renders with active title bar colors until its first blur/focus cycle.

## Change summary

Custom `titleBar.inactiveBackground` color is not applied on startup for windows that were never focused, because the title bar part assumes active state by default.

## Best solution

Initialize `isInactive` from the host service's actual focus state (e.g., check `hostService.hasFocus` or compare `onDidChangeActiveWindow` at construction time) so that windows restored in the background correctly render inactive colors from the start.
