# Issue #186160 — Custom snippets not prioritized when quick suggestions are inline 

[https://github.com/microsoft/vscode/issues/186160](https://github.com/microsoft/vscode/issues/186160)

## Summary

Still appears valid on current main: when quick suggestions are set to inline, the inline completion order is driven by suggest-memory selection (line 69 of suggestInlineCompletions.ts), which doesn't account for snippet priority settings or the fact that a user snippet is the best prefix match.

## Change summary

Custom snippets do not appear as the first inline suggestion nor receive initial focus in the suggestion list when `editor.quickSuggestions` is set to inline mode.

## Best solution

In `src/vs/editor/contrib/suggest/browser/suggestInlineCompletions.ts`, the `items` getter should respect the `snippetSuggestions` sort order (top/bottom/inline) when determining which item is shown first as the inline completion, rather than relying solely on suggest-memory. The focus issue in the suggestions list when triggered via Ctrl+Space may also need attention in the suggest widget adapter.
