# Issue #238646 — New file using keyboard only in file explorer not working on macbook

[https://github.com/microsoft/vscode/issues/238646](https://github.com/microsoft/vscode/issues/238646)

## Summary

Still reproduces on main: selecting a context menu action via Enter in the file explorer causes the keyup event to trigger rename on the previously focused item. An open PR (#288498) addresses this but has not yet merged.

## Change summary

Using keyboard-only workflow (Shift+F10, then Enter on 'New File/Folder') in the file explorer starts renaming instead of creating a new file/folder.

## Best solution

Review and land https://github.com/microsoft/vscode/pull/288498, which adds a keyup suppression listener in `contextMenuHandler.ts` to consume the Enter keyup after a context menu action is triggered via keyboard, preventing it from reaching the explorer's rename handler.
