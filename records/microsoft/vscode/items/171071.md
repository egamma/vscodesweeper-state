# Issue #171071 — Move the vertical ruler (the column limit) correctly when the inlay hints are used.

[https://github.com/microsoft/vscode/issues/171071](https://github.com/microsoft/vscode/issues/171071)

## Summary

Valid feature request to offset vertical rulers on lines with inlay hints. No implementation exists on main; the issue is in Backlog with 7 upvotes (below the 20 community threshold). Team member hediet acknowledged it as a legitimate but tricky feature request.

## Change summary

Vertical rulers should account for inlay hint widths so the ruler accurately indicates the source-code column limit on lines containing inlay hints.

## Best solution

Implement per-line ruler offset in `src/vs/editor/browser/viewParts/rulers/rulers.ts` (and the GPU variant) that queries inlay-hint decorations to shift the ruler position on affected lines. This is a non-trivial rendering change since rulers are currently drawn as straight vertical lines; a design decision is needed on whether to offset per-line (jagged ruler) or use another visual approach.
