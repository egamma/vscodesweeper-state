# Issue #235009 — Git - git blame info is not provided to screen reader users

[https://github.com/microsoft/vscode/issues/235009](https://github.com/microsoft/vscode/issues/235009)

## Summary

Still reproduces on main: git blame editor decorations use CSS `::after` pseudo-elements via `renderOptions.after.contentText`, which are invisible to screen readers. No accessibility fix has landed since the issue was filed.

## Change summary

Git blame inline editor decorations are not announced to screen reader users because the content is rendered via CSS pseudo-elements with no ARIA alternative.

## Best solution

The fix likely requires either (1) adding an `accessibilityInformation` field to the `DecorationOptions` API so extensions can provide screen-reader labels for decoration content, or (2) a git-extension-side workaround such as an accessible view provider or a dedicated command that reads the blame info aloud. The extension-side approach in `extensions/git/src/blame.ts` (the `GitBlameEditorDecoration` class) could register an `AccessibleContentProvider` so Alt+F1 surfaces blame info, but the deeper fix is a platform-level decoration accessibility API.
