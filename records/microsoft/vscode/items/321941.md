# Issue #321941 — Agents: Showing irrelevant changes when created a branch from non main branch

[https://github.com/microsoft/vscode/issues/321941](https://github.com/microsoft/vscode/issues/321941)

## Summary

Verified duplicate: the Changes view in the Agents window diffs against `refs/remotes/origin/HEAD` (the remote default branch), so branching from a non-default branch shows all the commits between those two branches. This is already tracked and assigned in the canonical issue.

## Change summary

The Agents window Changes view shows irrelevant file changes when a branch is created from a non-default branch (e.g. release) because it diffs against the remote's default branch rather than the actual parent branch.

## Best solution
