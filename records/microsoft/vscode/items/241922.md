# Issue #241922 — Can't dnd image from built-in preview

[https://github.com/microsoft/vscode/issues/241922](https://github.com/microsoft/vscode/issues/241922)

## Summary

Still present on main: dragging an image from the built-in preview into chat fails because the webview-internal `https://` URI is treated as a regular URL and fetched directly, producing a 400 error. The code has no path to resolve webview resource URIs back to workspace file URIs.

## Change summary

Dragging an image from VS Code's built-in image or markdown preview into chat does not attach correctly and returns a 400 error.

## Best solution

In `src/vs/workbench/contrib/chat/browser/widget/chatDragAndDrop.ts`, detect webview-origin URIs in `getImageTransferDataFromUrl` (or in `extractUrlsFromDragEvent`) and resolve them back to local workspace file URIs before attempting to read the image data, rather than trying to HTTP-fetch the webview-internal URL.
