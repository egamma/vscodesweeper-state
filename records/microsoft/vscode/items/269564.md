## Summary

Valid feature request acknowledged by a team member. The prompt URL handler on current main still performs a plain unauthenticated GET request with no credential forwarding or domain allow-listing, so the gap remains.

## Change summary

The vscode:chat-prompt/install URL handler cannot fetch prompt files from URLs that require authentication or are on non-public hosts.

## Best solution

Add authentication support (e.g., credential/token forwarding or domain allow-listing) to the prompt URL handler in `src/vs/workbench/contrib/chat/browser/promptSyntax/promptUrlHandler.ts`, so that organizations can host internal prompt repos accessible via vscode: links.
