# Issue #317917 — Add setting to suppress terminal context menu when mouse reporting is active

[https://github.com/microsoft/vscode/issues/317917](https://github.com/microsoft/vscode/issues/317917)

## Summary

Valid feature request for a conditional context-menu suppression when xterm mouse reporting is active. No existing setting covers this — `rightClickBehavior: "nothing"` is unconditional. The issue is well-specified with prior art references and a clear proposed shape; it's in the Backlog Candidates community process with 2 upvotes so far.

## Change summary

Add a setting to conditionally suppress the terminal context menu and pass right-click to the pty only when the terminal application has enabled mouse reporting mode.

## Best solution

Implement a new terminal setting (e.g. `terminal.integrated.rightClickBehavior` gaining a new enum value, or a separate boolean) that checks xterm's mouse tracking mode before deciding whether to show the VS Code context menu or pass the event through. The logic would live in `src/vs/workbench/contrib/terminal/browser/terminalInstance.ts` around the right-click handling, querying the xterm instance for active mouse mode. Alternatively, this could be an additional enum value on the existing `rightClickBehavior` setting.
