# Issue #318744 — Configure Inline Suggestions not working at all

[https://github.com/microsoft/vscode/issues/318744](https://github.com/microsoft/vscode/issues/318744)

## Summary

The `github.copilot.toggleStatusMenu` command is registered conditionally (requires `extensionUnification` AND completions provider instantiation); if the Copilot extension fails to fully activate, the command is never registered but `product.json` still references it. The reporter's environment (local AI model, possible auth issues) may prevent full activation. Already labeled info-needed; maintainer asked for steps but the reporter's additional info hasn't yet yielded a clean reproduction.

## Change summary

Clicking 'Configure Inline Suggestions' does nothing and logs `command 'github.copilot.toggleStatusMenu' not found` when the Copilot completions provider hasn't fully activated.

## Best solution

Either register the `github.copilot.toggleStatusMenu` command unconditionally (with a graceful fallback when the completions service isn't ready), or display a user-facing error/notification instead of silently failing. The fix likely lives in `extensions/copilot/src/extension/completions-core/vscode-node/completionsServiceBridges.ts` and possibly `extensions/copilot/src/extension/completions/vscode-node/completionsCoreContribution.ts`.
