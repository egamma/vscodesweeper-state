# Issue #306830 — Display changed files more prominently (Source Control, Changes, Font, Color, Colour)

[https://github.com/microsoft/vscode/issues/306830](https://github.com/microsoft/vscode/issues/306830)

## Summary

Feature request for more prominent visual treatment of changed files in the Source Control panel. VS Code already colors modified files via gitDecoration theme colors and shows status badges, but the user wants bolder fonts, stronger colors, or larger icons. This is a reasonable polish/UX request with no community traction yet — keeping open for the community process.

## Change summary

Make changed files in the Source Control panel more visually prominent through bolder fonts, stronger colors, or larger icons.

## Best solution

VS Code already provides customizable file decoration colors (`gitDecoration.modifiedResourceForeground` etc.) and status letter badges in the SCM view. The user may not be aware of these existing customization points. A maintainer could consider whether this is best addressed by documentation/discoverability improvements, or whether the default theme colors for SCM decorations should be revisited for better contrast. Adding font-weight or icon-size changes would be a broader UX decision for the SCM area owner.
