# Issue #157195 — Poor sort order for 'go to symbol'

[https://github.com/microsoft/vscode/issues/157195](https://github.com/microsoft/vscode/issues/157195)

## Summary

Still valid: go-to-symbol scoring uses `scoreFuzzy2` without full-match boosting — exact matches like `Tab` are not promoted over fuzzy matches like `tabSize`. The discussion reached consensus on a fix approach (use `fuzzyScore` with `boostFullMatch`) but it was never implemented. Keeping open as a real gap with a clear path forward.

## Change summary

Exact/full word matches should rank higher than partial fuzzy matches in go-to-symbol quick access.

## Best solution

Switch go-to-symbol scoring in `src/vs/editor/contrib/quickAccess/browser/gotoSymbolQuickAccess.ts` from `scoreFuzzy2` to `fuzzyScore` with `boostFullMatch: true` (as jrieken suggested in the discussion), or add equivalent full-match boosting to the existing scorer. This would promote exact matches like `Tab` above partial matches like `tabSize`.
