# Issue #321432 — Agent turn hangs indefinitely when a streaming response stalls (no idle timeout in messagesApi/responsesApi SSE read loop)

[https://github.com/microsoft/vscode/issues/321432](https://github.com/microsoft/vscode/issues/321432)

## Summary

Team-authored issue (MEMBER); the core defect — missing idle timeout in SSE read loops — remains open. Merged PR #321671 adds telemetry instrumentation for the stall but does not implement the proposed idle-timeout abort or cancellation-token passthrough that would actually recover from a hung stream.

## Change summary

Agent turns hang indefinitely when an upstream streaming response stalls because the SSE read loop in messagesApi.ts and responsesApi.ts has no inactivity timeout.

## Best solution

Implement a streaming idle watchdog in `extensions/copilot/src/platform/endpoint/node/messagesApi.ts` (and the parallel Responses path) that destroys the body after ~120s of silence, rejects the feed, and lets retry logic recover. Wire the existing `CancellationToken` through so user/turn cancellation can abort a stuck read.
