# Issue #246105 — Inline git blame is rendered before code when deleting a line

[https://github.com/microsoft/vscode/issues/246105](https://github.com/microsoft/vscode/issues/246105)

## Summary

Confirmed bug with inline git blame editor decorations rendering incorrectly (appearing before code or on wrong lines) when deleting lines. Reproduced on Insiders by the reporter, confirmed with extensions disabled, and assigned to @hediet by a team member. No fix found on main.

## Change summary

Inline git blame decorations render in the wrong position (before code content) when lines are deleted, and persist incorrectly after undo.

## Best solution

Fix the decoration positioning in the editor core or the blame extension's decoration update logic — likely in how `setDecorations` interacts with line deletions. The `GitBlameEditorDecoration._onDidChangeBlameInformation` in `extensions/git/src/blame.ts` applies decorations based on `lineNumber` from blame info, but the issue may be in how the editor's decoration rendering handles `after` content during line-deletion view updates. @hediet was assigned to investigate.
