# Issue #160891 — Incorrect Command Variable Resolution in tasks.json if task type is 'process'

[https://github.com/microsoft/vscode/issues/160891](https://github.com/microsoft/vscode/issues/160891)

## Summary

Confirmed on current main: command variables in process-type task commands get the cwd prepended before they can be resolved, producing an invalid path. An open PR (#316369) exists to fix this — keeping open until it lands.

## Change summary

Process-type tasks with command variables (e.g. `${command:python.interpreterPath}`) produce an invalid path because the cwd is prepended before the variable is resolved.

## Best solution

Land or iterate on the open PR https://github.com/microsoft/vscode/pull/316369 which preserves unresolved variables in the process command path, deferring cwd-prepend until after full variable resolution in `terminalTaskSystem.ts`.
