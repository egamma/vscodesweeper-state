# Issue #72945 — [themes] Support for hue, saturation, lightness (HSL) color codes for theming

[https://github.com/microsoft/vscode/issues/72945](https://github.com/microsoft/vscode/issues/72945)

## Summary

HSL color parsing is not supported in theme color settings on current main. The `Color.fromHex` path used for user-customized theme colors only accepts hex, and the CSS parser explicitly notes `// TODO: Support more formats as needed` without handling HSL. This remains a valid open feature request.

## Change summary

Allow HSL/HSLA color strings in theme color customization settings (e.g. `editor.background: "hsla(301, 46%, 9%, 1)"`).

## Best solution

Extend `Color.Format.CSS.parse()` in `src/vs/base/common/color.ts` to handle `hsl(…)` and `hsla(…)` strings, and update `colorThemeData.ts` to use `CSS.parse()` (or a broader parser) instead of `Color.fromHex()` when reading user color overrides. The HSLA model class already exists in the codebase.
