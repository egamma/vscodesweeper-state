# Issue #210171 — CodeAction previews don't work on "Source" actions

[https://github.com/microsoft/vscode/issues/210171](https://github.com/microsoft/vscode/issues/210171)

## Summary

The label-visibility part was fixed (commit df0a6ebd8b1 added Source to the previewable kinds list), but the core problem remains: command-based CodeActions still cannot show previews because VS Code has no mechanism to associate a command's resulting edits with the preview flow. PR #210175 (still open) is a step toward a fix. Keeping open — the underlying gap is real and unresolved.

## Change summary

CodeAction previews do not work for Source actions or any command-based CodeActions because VS Code cannot preview edits it doesn't know about until a command executes.

## Best solution

Land or iterate on the open PR https://github.com/microsoft/vscode/pull/210175 (or a successor) to let command-based CodeActions opt into the preview flow. The `onHover` logic in `src/vs/editor/contrib/codeAction/browser/codeActionController.ts` already marks Source kinds as previewable, but the actual preview path requires edits on the action — a 'command that returns edits' concept is needed. Related issue https://github.com/microsoft/vscode/issues/205857 tracks the broader UX confusion around preview availability.
