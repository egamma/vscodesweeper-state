# Issue #203939 — Cannot undo changes after opening an active file in Hex Editor

[https://github.com/microsoft/vscode/issues/203939](https://github.com/microsoft/vscode/issues/203939)

## Summary

Still reproducible on main: when a custom editor (e.g. Hex Editor) is closed/disposed, it unconditionally removes all undo/redo elements for that URI, wiping the text editor's undo history for the same file.

## Change summary

Opening and closing a custom editor on the same file destroys the text editor's undo history because the custom editor's dispose clears all undo elements for that resource URI.

## Best solution

The `removeElements` call in `mainThreadCustomEditors.ts` dispose should only remove undo elements that belong to the custom editor (its own edits), not wipe the entire URI's undo stack. The fix likely needs to scope removal to the custom editor's undo group or only call `removeElements` when no other editor holds undo state for that resource.
