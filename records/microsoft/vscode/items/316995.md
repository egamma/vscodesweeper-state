# Issue #316995 — [Bug] Improper Encoding in Copilot Git Commit Message Sanitizer — Double-Escaping Allows Shell Argument Injection

[https://github.com/microsoft/vscode/issues/316995](https://github.com/microsoft/vscode/issues/316995)

## Summary

Security-sensitive issue describing a real double-escaping bug in the Copilot commit-message sanitizer — must stay open for human/MSRC handling. The vulnerable code is confirmed present on current main at line 299 of conversationFeature.ts.

## Change summary

The commit-message sanitizer escapes double-quotes before backslashes, producing a double-escape (`\"` → `\\"`) that allows a crafted quote to break out of the shell argument in `git commit -m "..."`, enabling argument injection.

## Best solution

Swap the replace order so backslashes are escaped first, then double-quotes, then dollar signs — or better, avoid shell-quoting entirely by using the git extension's programmatic commit API instead of `sendText`. Security team (MSRC) should evaluate the practical exploitability given that the input comes from model output influenced by repository content.
