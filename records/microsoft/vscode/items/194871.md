# Issue #194871 — Default context menu on IPad shows on top of the VSCode context menu

[https://github.com/microsoft/vscode/issues/194871](https://github.com/microsoft/vscode/issues/194871)

## Summary

Still a valid bug: iPadOS Safari's native context menu overlaps VS Code's custom context menu when using an external mouse/trackpad. No fix found on main. Multiple users confirm. Keeping open as a known iPad/Safari platform interaction issue.

## Change summary

On iPad with external mouse in Safari, right-clicking shows the iPadOS system context menu on top of VS Code's custom context menu.

## Best solution

Suppress the native context menu on iPad/Safari by calling preventDefault() on the contextmenu event in the browser context menu handler, or detect iPad/Safari and use a platform-specific workaround. The fix likely belongs in src/vs/base/browser/dom.ts or the contextmenu event handling layer. This may also be partially an upstream Safari/WebKit behavior.
