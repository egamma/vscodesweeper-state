# Issue #40115 — indent back to line start after enter newline and press up

[https://github.com/microsoft/vscode/issues/40115](https://github.com/microsoft/vscode/issues/40115)

## Summary

Well-established feature request with 45 upvotes (above the 20-vote Backlog threshold), already properly labeled and milestoned. The `tree-sitter-autoindent-candidate` label indicates it's being considered for future implementation. The existing `editor.trimAutoWhitespace` workaround doesn't fully address the core request.

## Change summary

When pressing Enter to create a new indented line, then moving the cursor away and back, the cursor returns to column 0 instead of maintaining the expected indentation level.

## Best solution

Implement virtual/implicit indentation on empty lines so the cursor respects the contextual indent level without requiring trailing whitespace to persist. The `tree-sitter-autoindent-candidate` label suggests this may be addressed as part of the tree-sitter autoindent work in the editor. Related issue https://github.com/microsoft/vscode/issues/209939 tracks the complementary 'trim only on save' approach.
