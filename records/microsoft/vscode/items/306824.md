# Issue #306824 — Automatically starting plan implementation without approval

[https://github.com/microsoft/vscode/issues/306824](https://github.com/microsoft/vscode/issues/306824)

## Summary

The Autopilot auto-send handoff in `renderChatSuggestNextWidget()` lacks a guard against re-triggering on subsequent renders (e.g., when the user navigates back to a plan chat). This causes unintended auto-implementation when returning to a completed plan while Autopilot permission is active.

## Change summary

Returning to a completed Plan-mode chat with Autopilot enabled auto-triggers 'Start Implementation' without user clicking the button.

## Best solution

Add a one-shot guard in `renderChatSuggestNextWidget()` (around line 1346 of chatWidget.ts) so the Autopilot auto-send fires only once per response completion, not on every re-render. A flag on the response viewmodel or a deduplicated call would prevent the re-trigger on chat re-focus.
