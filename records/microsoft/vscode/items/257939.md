# Issue #257939 — Repeated, periodic crash when editing text

[https://github.com/microsoft/vscode/issues/257939](https://github.com/microsoft/vscode/issues/257939)

## Summary

The original OOM crash was fixed (confirmed by reporter in comment 5), but excessive CPU usage when editing Git-tracked files remains and was re-confirmed by the reporter in December 2025. The issue is assigned to lszomoru and should stay open for the remaining CPU regression.

## Change summary

Editing Git-tracked files causes ~6-7x higher CPU usage compared to untracked files, with the git dirty-diff mechanism suspected as the cause.

## Best solution

Investigate the CPU overhead in the SCM/dirty-diff path that fires on each keystroke for Git-tracked files. The reporter's controlled experiment (same file, tracked vs untracked, 7% vs 1% CPU) isolates the issue to the Git integration layer. Likely involves throttling or debouncing dirty-diff computations in `src/vs/workbench/contrib/scm/browser/dirtydiff*.ts` or the git extension's status update path.
