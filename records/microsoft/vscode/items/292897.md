# Issue #292897 — [object Object] in network error message

[https://github.com/microsoft/vscode/issues/292897](https://github.com/microsoft/vscode/issues/292897)

## Summary

Team-authored bug still present on main: when a network error's `reason` field arrives as a non-string object from the agent host, the `??` null-coalescing on line 280 of chatErrorMessages.ts does not coerce it, so `[object Object]` leaks into the user-facing error message. Keeping open per team-member policy.

## Change summary

Network error messages in the chat UI display `[object Object]` instead of a human-readable reason string.

## Best solution

Guard `fetchError.reason` with a `typeof` check or `String()` coercion on line 280 of `src/vs/workbench/contrib/chat/common/chatErrorMessages.ts` so that non-string payloads (objects forwarded from agent host harnesses) are safely stringified before interpolation into the localized error message. A similar guard may be warranted for `requestId` and other fields interpolated into user-facing strings.
