## Summary

This is a support question about Copilot Chat connectivity on remote machines with restricted networks. The user already found the correct workaround — forcing extensions to run locally via `remote.extensionKind` — which is the documented approach for this network topology.

## Change summary

Copilot Chat fails to initialize on a Remote SSH host when the remote machine's network cannot fully reach Copilot services.

## Best solution

The `remote.extensionKind` setting to force Copilot to run as a UI extension is the supported configuration for this network topology. If the user wants Copilot to automatically detect and handle restricted remote networks, that would be a separate feature request with clearer requirements.
