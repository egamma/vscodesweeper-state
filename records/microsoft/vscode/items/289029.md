# Issue #289029 — A temporary solution to solve "Chat took too long to get ready", and hope this can be fixed.

[https://github.com/microsoft/vscode/issues/289029](https://github.com/microsoft/vscode/issues/289029)

## Summary

The reported problem — Copilot Chat not loading models and showing 'Chat took too long to get ready' on a network-restricted remote host — is a GitHub Copilot extension issue, not a VS Code core bug. The workaround of forcing `extensionKind` to `ui` confirms the extension's remote-side network handling is the root cause. Proposing close as caused-by-extension, with a pointer to the Copilot extension's own issue tracker and the related feature request #292758.

## Change summary

Copilot Chat fails to initialize and load models when running on a Remote SSH host with restricted network access (e.g., behind a Chinese firewall).

## Best solution

This belongs in the GitHub Copilot extension's issue tracker (https://github.com/microsoft/vscode-copilot-release/issues) — the extension needs to handle restricted-network remote environments gracefully, either by proxying through the local client or by improving its network fallback behavior. The related VS Code feature request https://github.com/microsoft/vscode/issues/292758 tracks the broader ask for Copilot on no-internet remotes.
