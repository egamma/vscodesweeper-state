# Issue #129607 — Automatic classification: more aggressive debounce and perf considerations

[https://github.com/microsoft/vscode/issues/129607](https://github.com/microsoft/vscode/issues/129607)

## Summary

Two of the four proposed optimizations (items 3 and 4) remain unimplemented on current main. The language detection throttle is 600ms and the code still runs detection on every content change without tracking failed attempts or adapting after a successful detection. Keeping open as a valid perf improvement request.

## Change summary

Add smarter heuristics to language auto-detection so it backs off after repeated failures and becomes less aggressive once a language is successfully detected.

## Best solution

Implement the remaining two optimizations in `BaseTextEditorModel.doAutoDetectLanguage` (src/vs/workbench/common/editor/textEditorModel.ts): (1) track consecutive failed detection attempts and stop retrying after N failures unless a large content change occurs, and (2) reduce detection frequency once a language has been successfully auto-detected, re-triggering only on substantial edits.
