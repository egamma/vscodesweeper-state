# Issue #205244 — Cannot view changes from timeline before file renamed

[https://github.com/microsoft/vscode/issues/205244](https://github.com/microsoft/vscode/issues/205244)

## Summary

Still reproduces on current main: the timeline diff command uses the current file URI for both sides of the comparison, ignoring that `--follow` may have surfaced commits where the file had a different name. The result is a blank diff for any commit before a rename.

## Change summary

Timeline entries from before a file rename show a blank diff because the diff command uses the current (post-rename) filename to resolve both sides of the comparison, instead of the filename the file had at each commit.

## Best solution

Extend `logFile` (or its output) to capture the file path at each commit when `--follow` is active (e.g. via `--name-status` or `--diff-filter`), propagate it through the `Commit`/`GitTimelineItem` chain, and use the per-commit path in `resolveTimelineOpenDiffCommand` when constructing the `toGitUri` calls. The fix touches `extensions/git/src/git.ts` (logFile + Commit interface), `extensions/git/src/timelineProvider.ts` (GitTimelineItem), and `extensions/git/src/commands.ts` (resolveTimelineOpenDiffCommand).

## Evidence

- **root cause in source:** In `extensions/git/src/commands.ts` line 4816, `resolveTimelineOpenDiffCommand` passes the same `uri` (the current file path) to both `toGitUri(uri, item.previousRef)` and `toGitUri(uri, item.ref)`. For commits before a rename, the file had a different path, so `git show <ref>:<current-path>` returns nothing — producing a blank diff.
- **missing data path:** The `Commit` interface (`extensions/git/src/git.ts` line 760) has no field for the file path at each commit. `logFile` uses `--follow` but does not use `--name-status` or similar to capture the old filename, so the rename information from `git log --follow` is discarded.
- **comment confirmation:** Comment by @byronwall (2025-07-03) independently traced the same root cause: `toGitUri` is called with a common `uri` that is not correct for files renamed between commits.
