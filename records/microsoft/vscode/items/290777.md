# Issue #290777 — Show disabled code actions in lightbulb/refactor menu

[https://github.com/microsoft/vscode/issues/290777](https://github.com/microsoft/vscode/issues/290777)

## Summary

Valid feature request to let extensions opt-in to showing disabled code actions (with their disabled reason) in the lightbulb and refactor menus. The current behavior intentionally hides them from the lightbulb and only shows them faded in specific filtered contexts. No duplicate found, and the request has clear UX value for extension authors like the Dart/Flutter LSP.

## Change summary

Allow disabled code actions to appear (greyed out with their disabled reason) in the lightbulb and refactor menus, rather than being hidden entirely.

## Best solution

Extend the code action menu logic in `src/vs/editor/contrib/codeAction/browser/codeActionController.ts` (around line 325 where `actionsToShow` is computed) and `codeActionMenu.ts` to include disabled actions in the lightbulb/refactor menu by default (or via an extension-settable flag), showing them greyed out with the `disabled` reason as a tooltip or secondary label. The comment about the missing tooltip for the disabled reason (comment #1) should also be addressed in the menu rendering.

## Evidence

- **current behavior:** In `src/vs/editor/contrib/codeAction/browser/codeAction.ts` line 82, `validActions` filters out all disabled actions. The lightbulb widget (lightBulbWidget.ts line 66) only checks `validActions.length`, so disabled actions never trigger the lightbulb.
- **menu logic:** In `codeActionController.ts` line 325, disabled actions are only included when `includeDisabledActions` is set AND either `_showDisabled` is true or there are zero valid actions — meaning they are hidden in most normal invocations.
- **disabled reason tooltip missing:** Comment #1 notes the disabled reason doesn't appear in the tooltip when disabled actions do show up (codeActionMenu.ts line 48 uses the disabled reason as the label text, but no tooltip is set).
