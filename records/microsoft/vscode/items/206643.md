# Issue #206643 — Git - Source Control tab doesn't respect global .gitignore for Git repositories

[https://github.com/microsoft/vscode/issues/206643](https://github.com/microsoft/vscode/issues/206643)

## Summary

Hard keep-open: the issue carries the 'On Deck' milestone, which is a real scheduling milestone. The reported behavior — Source Control tab not reflecting files excluded by the global `core.excludesfile` git config — is a legitimate bug that remains unresolved on main. The git extension spawns `git status -z -uall` and inherits `process.env`, so the global config should be honored; the root cause may be a caching/refresh issue or a Windows-specific environment subtlety.

## Change summary

The Source Control tab shows files that should be excluded by the global .gitignore (`core.excludesfile` in `~/.gitconfig`), even though `git status` on the command line correctly ignores them.

## Best solution

Investigate whether the git extension's process environment on Windows properly resolves `core.excludesfile` (especially with tilde paths), and whether a FileSystemWatcher or manual refresh is needed when the global gitignore file is changed. The relevant code is in `extensions/git/src/git.ts` (the `spawn` method at line ~676 and `getStatus` at line ~2738) and `extensions/git/src/repository.ts`.
