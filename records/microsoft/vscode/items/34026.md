# Issue #34026 — Debug console link detection issues

[https://github.com/microsoft/vscode/issues/34026](https://github.com/microsoft/vscode/issues/34026)

## Summary

This long-standing tracking issue covers two concerns: (1) improving debug console link detection regex patterns (significantly improved over the years but still imperfect) and (2) an extension API for contributing link detection rules (never implemented). With 21 upvotes and active community interest across many languages (Python, C#, Dart/Flutter, PHP), the remaining gap — especially the extension contribution API — is still valid and unaddressed.

## Change summary

Improve debug console link detection for various stack trace formats and allow extensions to contribute custom link detection rules.

## Best solution

The regex-based link detection in `src/vs/workbench/contrib/debug/browser/linkDetector.ts` has seen substantial improvement (C# format, ANSI handling, permissive path matching, line/column formats), but the core request for an extension API to contribute link detection rules remains unimplemented. The best end state would be a proposed API (similar to `TerminalLinkProvider`) that lets debug adapters register custom link patterns/handlers, consolidating the many language-specific requests that keep surfacing.
