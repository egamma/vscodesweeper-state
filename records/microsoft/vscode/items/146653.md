# Issue #146653 — Source Control view ignores filesystem scheme

[https://github.com/microsoft/vscode/issues/146653](https://github.com/microsoft/vscode/issues/146653)

## Summary

Still reproduces on main: a partial fix in commit 01f32e6a44f (1.76.0) added a scheme filter only to `isRepositoryOutsideWorkspace`, but the primary scanning paths in `scanWorkspaceFolders`, `onDidChangeWorkspaceFolders`, and `onDidChangeConfiguration` still use `folder.uri.fsPath` on non-file-scheme workspace folders, causing git to scan the local filesystem at the matching path.

## Change summary

The git extension's Source Control view treats non-file-scheme workspace folders (e.g. `myFS:/root`) as local file paths, scanning the real filesystem at that path for git repositories.

## Best solution

Add a `folder.uri.scheme === 'file'` filter to the workspace-folder iteration in `scanWorkspaceFolders` (line 370), `onDidChangeWorkspaceFolders` (line 472), and `onDidChangeConfiguration` (line 517) in `extensions/git/src/model.ts`, matching the existing guard in `isRepositoryOutsideWorkspace` (line 1126).
