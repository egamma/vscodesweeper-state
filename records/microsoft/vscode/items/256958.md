# Issue #256958 — Diff indicators cannot be used in some places

[https://github.com/microsoft/vscode/issues/256958](https://github.com/microsoft/vscode/issues/256958)

## Summary

Still reproduces on main: the gutter click handler in quickDiffWidget.ts hardcodes a ±3px hit-test (line 708), but the cursor changes to a pointer over a wider area (the full line-decoration zone), creating a mismatch where the hand cursor appears but clicking does nothing.

## Change summary

Clicking a few pixels to the right of the dirty diff gutter bar shows a pointer cursor but does not open the diff widget.

## Best solution

Widen the click hit-test in `onEditorMouseDown` (quickDiffWidget.ts:708) to match the actual cursor-change area, or narrow the cursor-change zone to match the hit-test. The configurable width (`scm.diffDecorationsGutterWidth`, up to 5px) should also be respected instead of hardcoding 3px.
