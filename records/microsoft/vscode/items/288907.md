# Issue #288907 — Do not hide away thinking output if user is reading it

[https://github.com/microsoft/vscode/issues/288907](https://github.com/microsoft/vscode/issues/288907)

## Summary

Team-authored polish request about thinking output UX — the related PR #288908 was closed without merging, and no mouse-hover-based auto-hide prevention has been implemented on current main. Active team discussion with no consensus yet; keeping open for the team to decide on the right UX pattern.

## Change summary

Thinking output in chat is auto-hidden while the user is actively reading it, with no way to prevent the collapse when hovering/interacting with the widget.

## Best solution

The team should settle on a UX pattern — options discussed include: (1) pausing auto-hide when mouse is over the thinking widget (Twitch-chat pattern), (2) keeping thinking collapsed by default but not auto-hiding once user expands it, (3) adding a subtle 'Hide' button instead of auto-collapsing. Implementation would be in `src/vs/workbench/contrib/chat/browser/widget/chatContentParts/chatThinkingContentPart.ts`. The `collapsed` thinkingStyle was tried but felt too opaque per comment #7.
