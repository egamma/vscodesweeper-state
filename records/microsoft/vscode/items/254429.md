# Issue #254429 — Attach a terminal as context

[https://github.com/microsoft/vscode/issues/254429](https://github.com/microsoft/vscode/issues/254429)

## Summary

Team-authored feature request for attaching user-created terminals as explicit chat context. Partially addressed by task awareness (#252162, closed) and agent-created terminal tracking, but the core ask — giving agents real-time read access to arbitrary user-started terminals — remains unimplemented.

## Change summary

Allow users to attach a running terminal instance as context in chat so the agent sees what's already running and avoids redundant commands.

## Best solution

Extend the chat attachment/context mechanism to allow selecting any active terminal instance (not just agent-created ones) as an implicit context provider, likely in `terminalContrib/chatAgentTools` and the chat input attachment UI. The agent would then receive terminal output/state before deciding whether to spawn new processes.
