# Issue #319199 — Chat: add a setting to suppress or delay the sessions list hover preview

[https://github.com/microsoft/vscode/issues/319199](https://github.com/microsoft/vscode/issues/319199)

## Summary

Valid feature request: the agent sessions hover preview is implemented in core (agentSessionsViewer.ts) with an internal `disableHover` flag but no user-facing setting to control delay or disable it. No existing setting covers this.

## Change summary

Request a user-facing setting to suppress or delay the agent sessions list hover preview in the Chat view.

## Best solution

Expose a configuration setting (e.g. `chat.sessions.hoverPreview.enabled` or a delay option) in the agent sessions viewer that reads from `IConfigurationService` and gates the `renderHover` call in `src/vs/workbench/contrib/chat/browser/agentSessions/agentSessionsViewer.ts`. The internal `disableHover` plumbing already exists.
