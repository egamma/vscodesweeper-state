# Issue #236348 — `vscode://vscode-remote/wsl+<distro>/path/to/file` des not use already open window 

[https://github.com/microsoft/vscode/issues/236348](https://github.com/microsoft/vscode/issues/236348)

## Summary

Confirmed and reproduced by a team member (aeschli). The bug is still present on main — no fix has been committed. The protocol handler in `getWindowOpenableFromProtocolUrl` defaults remote file paths to `folderUri`, causing a new window instead of reusing the existing WSL window.

## Change summary

Opening a file via `vscode://vscode-remote/wsl+<distro>/path/to/file` opens a new window instead of reusing the already-open WSL window.

## Best solution

Fix `getWindowOpenableFromProtocolUrl` in `src/vs/code/electron-main/app.ts` to better distinguish file vs folder for remote URIs (e.g. check for file extension or add a `type=file` query parameter), so the open call uses `fileUri` and the window-matching logic can reuse the existing remote window.
