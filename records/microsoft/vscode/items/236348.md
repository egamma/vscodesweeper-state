# Issue #236348 — `vscode://vscode-remote/wsl+<distro>/path/to/file` des not use already open window 

[https://github.com/microsoft/vscode/issues/236348](https://github.com/microsoft/vscode/issues/236348)

## Summary

Still open on main. The confirmed bug is that `vscode://vscode-remote/wsl+<distro>/path/to/file` opens a new window instead of reusing an already-open WSL window. A team member (aeschli) reproduced and confirmed it. No fix has landed.

## Change summary

Opening a file via `vscode://vscode-remote/wsl+<distro>/path/to/file` protocol link creates a new window instead of reusing the existing WSL window.

## Best solution

Fix the protocol handler in `src/vs/code/electron-main/app.ts` (`getWindowOpenableFromProtocolUrl`) so that remote file paths (not folders/workspaces) are returned as `fileUri` rather than `folderUri`, enabling the existing window-matching logic in `windowsMainService.ts` to find and reuse the open WSL window.
