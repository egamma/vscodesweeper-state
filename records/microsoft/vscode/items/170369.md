# Issue #170369 — 3+ elements for "editor.language.colorizedBracketPairs"

[https://github.com/microsoft/vscode/issues/170369](https://github.com/microsoft/vscode/issues/170369)

## Summary

Valid feature request to support multi-keyword bracket groups (3+ elements) in colorized bracket pairs. The current implementation is strictly typed as `[string, string][]` pairs, and a related issue (#159586 for C/C++ preprocessor directives) was closed as `*out-of-scope`. However, this request is broader and slightly different — it asks for general multi-token bracket support rather than just preprocessor-specific highlighting. Low community traction (1 upvote) suggests it should remain open in Backlog Candidates but doesn't warrant priority.

## Change summary

Allow `editor.language.colorizedBracketPairs` to accept 3+ element tuples (e.g. `["if", "then", "else", "end"]`) for languages with multi-keyword bracket structures.

## Best solution

This is architecturally non-trivial: the bracket-pair colorizer's core matching algorithm (`languageBracketsConfiguration.ts`, the bracket-pair AST in `src/vs/editor/common/model/bracketPairsTextModelPart/`) is built around binary open/close pairs. Supporting n-ary groups would require rethinking the matching and nesting model. The related #159586 was closed as out-of-scope. Given low traction, this should remain parked in Backlog unless community demand grows — an extension using decorations or a TextMate grammar with scopes is the pragmatic workaround today.
