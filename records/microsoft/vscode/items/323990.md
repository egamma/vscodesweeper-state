# Issue #323990 — Copilot Chat: first invocation of a remote MCP tool fails with "Cannot read properties of undefined (reading 'invoke')"

[https://github.com/microsoft/vscode/issues/323990](https://github.com/microsoft/vscode/issues/323990)

## Summary

Credible race condition report: MCP tool invocation fails on first call after session activation because the invocation handler is not yet wired up when the tool list is already populated. No fix found on current main. Routing to the MCP/chat tools area for investigation.

## Change summary

First invocation of a remote MCP tool after window/session activation fails with TypeError because the tool's invoke handler is undefined, while subsequent calls succeed.

## Best solution

Guard `invokeTool` to await MCP server readiness (or retry the connection) when `tool.impl` is present but the underlying MCP transport is still connecting. The fix likely belongs in `src/vs/workbench/contrib/chat/browser/tools/languageModelToolsService.ts` (around line 714) or in the MCP tool contribution's registration timing in `src/vs/workbench/contrib/mcp/common/mcpLanguageModelToolContribution.ts`.
