# Issue #293380 — Merge conflict UI incorrectly shown for SEARCH/REPLACE text patterns

[https://github.com/microsoft/vscode/issues/293380](https://github.com/microsoft/vscode/issues/293380)

## Summary

The merge-conflict extension uses simple prefix matching (`startsWith('<<<<<<<')`) without verifying git merge state, so any file containing text resembling merge markers (e.g. SEARCH/REPLACE diff patterns) triggers the merge conflict UI. An open PR by the reporter exists but has not been merged; keeping open for area-owner review.

## Change summary

Merge conflict decorations and codelens actions appear on files that merely contain merge-marker-like text patterns but are not part of any active Git merge.

## Best solution

Review PR https://github.com/microsoft/vscode/pull/293381 which adds a git-state check via `repository.state.mergeChanges` before displaying conflict UI. The fix lives in `extensions/merge-conflict/src/mergeConflictParser.ts` and `extensions/merge-conflict/src/documentTracker.ts`. Consider whether the fallback (show markers when git extension is unavailable) is the right trade-off, and whether rebase/cherry-pick conflict states also need coverage.
