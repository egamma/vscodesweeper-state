# Issue #172504 — Terminal DOM renderer clips the top of the cell, cutting off tall glyphs like U+23A1 ⎡

[https://github.com/microsoft/vscode/issues/172504](https://github.com/microsoft/vscode/issues/172504)

## Summary

Confirmed upstream issue in xterm.js — the DOM renderer clips tall glyphs that extend above the cell baseline (e.g. U+23A1). The upstream issue remains open; no fix has landed in xterm.js yet.

## Change summary

Terminal DOM renderer clips the top of tall Unicode glyphs (like U+23A1 ⎡) when using fonts like Consolas that vertically offset them.

## Best solution

Wait for the upstream fix in xterm.js (https://github.com/xtermjs/xterm.js/issues/5240) to land, then pick it up via a version bump of the @xterm packages. No VS Code-side fix is needed beyond tracking.
