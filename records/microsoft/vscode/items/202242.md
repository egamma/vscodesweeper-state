# Issue #202242 — "The diff algorithm was stopped early" when removing a large chunk (2k lines) of code

[https://github.com/microsoft/vscode/issues/202242](https://github.com/microsoft/vscode/issues/202242)

## Summary

Still reproducible on main: the advanced diff algorithm times out on large deletions (~2k lines), showing a disruptive notification. The timeout mechanism works as designed but the UX — a repeated warning prompt — is a real pain point, especially with AI agent workflows generating large diffs. The underlying performance of the advanced algorithm on asymmetric diffs remains unimproved.

## Change summary

The advanced diff algorithm times out and shows a disruptive repeated notification when comparing files with large (2k+ line) deletions.

## Best solution

Either improve the advanced algorithm's performance on asymmetric diffs (large deletion + small insertion), or make the timeout notification less intrusive (e.g., show it once per session or use a less-invasive status bar indicator). The new `@vscode/diff` WASM computer currently has `maxComputationTimeMs` commented out — implementing timeout support there may also help if it's faster for these cases.
