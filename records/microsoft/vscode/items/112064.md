# Issue #112064 — Pressing `shift-left/right` will exit multi-caret mode

[https://github.com/microsoft/vscode/issues/112064](https://github.com/microsoft/vscode/issues/112064)

## Summary

This should stay open: a VS Code maintainer reproduced the issue, it is already routed to the column selection area, and later discussion still describes the same multi-cursor truncation behavior. There is no concrete fixing commit or canonical duplicate in the provided context, so a cleanup close would be premature.

## Change summary

Shift-left/right and cursor column selection commands can collapse multi-cursor or column-selection state to the primary cursor instead of extending all aligned selections.

## Best solution

Keep this tracked under `editor-columnselect` and have the editor owner decide whether `ColumnSelectCommand` should operate across all current cursors/selections or intentionally collapse to a box-selection primary range; if fixed, add regression coverage around `cursorColumnSelectLeft/Right` and normal `CursorLeftSelect` after column selections.
