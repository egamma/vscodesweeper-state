# Issue #112064 — Pressing `shift-left/right` will exit multi-caret mode

[https://github.com/microsoft/vscode/issues/112064](https://github.com/microsoft/vscode/issues/112064)

## Summary

Still reproduces on main: `ColumnSelectCommand` in `coreCommands.ts` only uses the primary cursor state (`viewModel.getPrimaryCursorState()`) when computing column select results, so shift+left/right in column selection mode discards all secondary cursors. No fix has landed since the 2020 report.

## Change summary

Pressing shift+left/right in column selection mode with multiple cursors discards all secondary cursors instead of extending/contracting selections on each cursor.

## Best solution

Make `ColumnSelectCommand.runCoreEditorCommand` multi-cursor aware: when existing multi-cursor state is present and cursors are column-aligned, compute the column select result spanning all cursor lines rather than falling back to only the primary cursor's position. The fix lives in `src/vs/editor/browser/coreCommands.ts` around line 420–434.
