# Issue #314079 — Install agent plugin specific version or provide branch name

[https://github.com/microsoft/vscode/issues/314079](https://github.com/microsoft/vscode/issues/314079)

## Summary

Valid feature request: the agent plugin data model already supports ref/version pinning in source descriptors, but the UI lacks a version-selection experience analogous to extensions' 'Install Another Version' flow. Routing to the chat/AI area for prioritization.

## Change summary

Allow users to install a specific version (or branch/tag) of an agent plugin from the UI, similar to the extension version picker.

## Best solution

Add a version/ref picker in the agent plugin detail view (likely in `src/vs/workbench/contrib/chat/browser/agentPluginEditor/`) that lets users browse available versions/tags from the marketplace and install a pinned version. The plumbing (`ref`, `version` fields in `IPluginSourceDescriptor`) already exists — this is primarily a UI/UX gap.

## Evidence

- **data model supports versions:** src/vs/workbench/contrib/chat/common/plugins/pluginMarketplaceService.ts defines `ref?: string` on IGitHubPluginSource/IGitUrlPluginSource and `version?: string` on INpmPluginSource/IPipPluginSource — the pinning plumbing exists.
- **no version picker UI:** src/vs/workbench/contrib/chat/browser/agentPluginEditor/ has no version-selection UI; the install flow in pluginInstallService.ts hard-codes `version: ''` when installing from source.
- **git checkout support exists:** pluginGitService.ts exposes `checkout(repoDir, treeish, detached)` which would allow checking out a specific tag/branch once the UI passes it through.
