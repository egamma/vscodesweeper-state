# Issue #41024 — macOS: alt+i always inserts '^' character

[https://github.com/microsoft/vscode/issues/41024](https://github.com/microsoft/vscode/issues/41024)

## Summary

Keep open as an upstream Chromium/macOS keyboard handling issue; the VS Code maintainer discussion already confirms reproduction and links the owning Chromium bug. There is no strong evidence in the issue context that VS Code has implemented an independent core workaround.

## Change summary

Option-based macOS keybindings such as Alt+I can both trigger a VS Code command and insert the corresponding dead-key character.

## Best solution

Continue tracking this under keybindings/macOS/Chromium until Chromium fixes https://bugs.chromium.org/p/chromium/issues/detail?id=887519 or VS Code identifies a reliable workbench-level workaround for dead-key Option shortcuts.
