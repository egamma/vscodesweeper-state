# Issue #41024 — macOS: alt+i always inserts '^' character

[https://github.com/microsoft/vscode/issues/41024](https://github.com/microsoft/vscode/issues/41024)

## Summary

Confirmed upstream Chromium bug (crbug.com/887519) where e.preventDefault() fails to suppress dead-key character insertion on macOS for alt+key combinations. Properly triaged and labeled; should remain open as a tracking issue until Chromium resolves the root cause.

## Change summary

On macOS, using alt+key combinations as VS Code shortcuts inserts dead-key characters (e.g. ˆ, ˜, ¨) into the editor because Chromium does not honor preventDefault() for these events.

## Best solution

This remains blocked on Chromium fixing https://bugs.chromium.org/p/chromium/issues/detail?id=887519. Keep open as an upstream tracker. If Chromium ever lands a fix, VS Code will inherit it via the next Electron update. No VS Code-side workaround is feasible without native-level interception that Electron/Chromium must provide.
