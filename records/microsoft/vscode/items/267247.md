# Issue #267247 — Git Gutter Fails Half The Time

[https://github.com/microsoft/vscode/issues/267247](https://github.com/microsoft/vscode/issues/267247)

## Summary

Plausible symlink path-resolution bug: `getOpenRepository` in `extensions/git/src/model.ts` only checks `repository.root` (symlink path) against the editor resource URI (which uses the resolved real path), so `isDescendant` fails and `provideOriginalResource` bails out. The maintainer is actively investigating and the `info-needed` label is appropriate while awaiting complete trace logs.

## Change summary

Git gutter diff decorations fail to appear when a workspace folder is accessed through a directory symlink, because the repository root path and the editor resource path use different path forms.

## Best solution

In `extensions/git/src/model.ts`, `getOpenRepository` (line 984) should additionally check `liveRepository.repository.rootRealPath` via `isDescendant` when the initial `root` check fails, similar to how `getRepositoryExact` already handles both paths. The maintainer (lszomoru) is actively investigating — the trace logs requested in the last comment would confirm this theory.

- **path mismatch in getOpenRepository:** extensions/git/src/model.ts line 984: `isDescendant(liveRepository.repository.root, resourcePath)` only checks the symlink path (e.g. /home/yo/code/slide) against the real path of the resource (e.g. /media/struct1/code/slide/src/songs.js). These differ, so the repository is not found.
