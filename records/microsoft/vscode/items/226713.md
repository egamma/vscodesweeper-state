# Issue #226713 — Not fully clear why `export type TextSearchResultNew = TextSearchMatchNew | TextSearchContextNew;`

[https://github.com/microsoft/vscode/issues/226713](https://github.com/microsoft/vscode/issues/226713)

## Summary

Still a valid open design discussion: the proposed `textSearchProvider2` API keeps context lines as a flat union with matches rather than nesting them. The API is still proposed and unchanged; the design question raised by bpasero and echoed by jrieken remains unresolved.

## Change summary

The `TextSearchResult2` union type streams context lines separately from matches, making it unclear for consumers how to associate context with its surrounding match.

## Best solution

Resolve the API design before finalizing `textSearchProvider2`: either restructure `TextSearchResult2` so context is nested under matches (as bpasero and jrieken suggest benefits consumers), or document the streaming order contract so consumers know how to reassemble context around matches. The search area owner (andreamah) should drive the decision.
