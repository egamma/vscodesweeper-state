# Issue #226713 — Not fully clear why `export type TextSearchResultNew = TextSearchMatchNew | TextSearchContextNew;`

[https://github.com/microsoft/vscode/issues/226713](https://github.com/microsoft/vscode/issues/226713)

## Summary

This is an open API design discussion about whether TextSearchContext should be a separate union member or referenced from TextSearchMatch in the proposed textSearchProvider2 API. The types were renamed (New→2) but the union design is unchanged — the discussion remains unresolved and relevant.

## Change summary

Question whether TextSearchResult2's union of TextSearchMatch2 | TextSearchContext2 is the right API shape, versus having context referenced directly from matches.

## Best solution

The search team (area owner for `search`) should decide whether the current union-type design for `TextSearchResult2` in `src/vscode-dts/vscode.proposed.textSearchProvider2.d.ts` is the right shape before the API finalizes. The tradeoff is provider simplicity (streaming matches and context separately, as ripgrep does) vs. consumer ergonomics (consumers must reassemble context-to-match associations). If the API is close to finalization, this design point needs explicit resolution.
