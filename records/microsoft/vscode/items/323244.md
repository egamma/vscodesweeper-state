[https://github.com/microsoft/vscode/issues/323244](https://github.com/microsoft/vscode/issues/323244)

## Summary

The report likely conflates the absolute latest version number (from a raw marketplace API query that ignores target platform) with what VS Code correctly installs — the latest version *available for the user's platform*. Needs confirmation whether the affected extension versions actually have linux-x64 assets published, or whether this is a marketplace CDN staleness issue.

## Change summary

If the extension's latest version does publish a universal/linux-x64 package and VS Code still installs the older one, this likely belongs to the marketplace service team (microsoft/vsmarketplace), not the VS Code client.
