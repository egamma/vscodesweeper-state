# Issue #268239 — Staging file in git through VSCode doesn't trigger filter

[https://github.com/microsoft/vscode/issues/268239](https://github.com/microsoft/vscode/issues/268239)

## Summary

Valid bug report: staging a file via VS Code's SCM panel does not trigger the git clean filter defined in .gitattributes, whereas `git add` from the CLI correctly runs the filter. The report is detailed with clear reproduction steps, but no commenter has confirmed and no fix exists on main. The `git.stage` command calls `git add`, which should trigger filters — the root cause likely involves the subprocess environment (e.g., `astyle` not on PATH for VS Code's git child process) or a code path difference. Keeping open as a real, unfixed bug.

## Change summary

Git clean filters configured via .gitattributes are bypassed when staging files through VS Code's Source Control panel, causing changes that would be normalized away by the filter to be staged.

## Best solution

Investigate whether VS Code's git subprocess environment includes the user's PATH so filter executables (like `astyle`) are found. The fix likely lives in `extensions/git/src/git.ts` around how `exec`/`stream` sets up the child process environment, or potentially in how `git add` is invoked. A test with a custom clean filter in .gitattributes would validate the fix.
