# Issue #145009 — SCM - Keyboard shortcut for going to next changed file

[https://github.com/microsoft/vscode/issues/145009](https://github.com/microsoft/vscode/issues/145009)

## Summary

Still a valid, unimplemented feature request with strong community support (65 upvotes, in Backlog). Existing SCM keyboard commands navigate between resource groups, not individual changed files. The multi-diff editor's Alt+F5 navigation only works inside that editor, not from the SCM sidebar tree. Related #24389 is broader and doesn't subsume this specific ask.

## Change summary

Add keyboard shortcuts to navigate between individual changed files in the SCM sidebar tree view.

## Best solution

Add commands (e.g. `workbench.scm.action.focusNextResource` / `focusPreviousResource`) with default keybindings that cycle through individual changed-file entries in the SCM tree, analogous to the existing `focusNextResourceGroup` / `focusPreviousResourceGroup` but at the file level. Implementation would live in `src/vs/workbench/contrib/scm/browser/scmViewPane.ts` and be registered in `scm.contribution.ts`.
