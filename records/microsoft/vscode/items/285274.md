# Issue #285274 — VSCode API Range intersection does not differentiate between inclusive and exclusive

[https://github.com/microsoft/vscode/issues/285274](https://github.com/microsoft/vscode/issues/285274)

## Summary

Feature request to change Range.intersection semantics so touching (but not overlapping) ranges return undefined instead of an empty range. A team member has been pinged for input; keeping open for discussion.

## Change summary

Range.intersection() returns an empty range for adjacent selections that merely touch at a boundary, rather than returning undefined.

## Best solution

This is an API semantics question best resolved by the extensions API team. The current behavior (returning an empty range when ranges touch) is consistent with how ranges are modeled as position pairs without inclusive/exclusive semantics. Changing it would be a breaking API change. A possible alternative: add an option parameter or a separate method that treats empty intersections as non-overlapping. The team member ping suggests this is under consideration.
