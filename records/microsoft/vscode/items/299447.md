# Issue #299447 — SCM review ignores .github/instructions/ files

[https://github.com/microsoft/vscode/issues/299447](https://github.com/microsoft/vscode/issues/299447)

## Summary

Still reproduces on main: the SCM review agent's `loadCustomInstructions` only calls `customInstructionsService.getAgentInstructions()`, which discovers `.github/copilot-instructions.md` and `~/.copilot/instructions.md` but never searches `.github/instructions/`. The VS Code core side has the discovery path (`INSTRUCTIONS_DEFAULT_SOURCE_FOLDER` in `promptFileLocations.ts`) but the review agent does not use it.

## Change summary

SCM code review ignores `.github/instructions/*.instructions.md` files, creating an inconsistency with Chat panel interactions that do discover and load them.

## Best solution

Update `loadCustomInstructions` in `extensions/copilot/src/extension/review/node/githubReviewAgent.ts` (or `getAgentInstructions` in `customInstructionsService.ts`) to also discover `.instructions.md` files from `.github/instructions/` (recursively), matching the pattern used by the Chat panel via the prompts service. The `applyTo` front-matter matching should be applied against the files under review.
