# Issue #45207 — [json] External schemas referenced from schema not updated

[https://github.com/microsoft/vscode/issues/45207](https://github.com/microsoft/vscode/issues/45207)

## Summary

Still a valid bug on main. The JSON language service has transitive dependency invalidation logic and a file watcher for `**/*.json`, but referenced schemas loaded via `$ref` still don't reliably trigger revalidation when changed — likely a URI normalization mismatch between file watcher events and the schema dependency graph.

## Change summary

Editing a JSON schema file referenced via `$ref` from another schema does not update validation in files using the parent schema until VS Code is restarted.

## Best solution

Fix the URI matching between `didChangeWatchedFiles` events and the schema dependency tracking in `vscode-json-languageservice`'s `onResourceChange`. The server's `jsonSchemaService.js` stores dependencies at line 361 (`parentHandle.dependencies.add(uri)`) and invalidates at line 188; the mismatch likely stems from how relative `$ref` URIs are resolved vs. how the file watcher reports absolute URIs. The fix spans both `extensions/json-language-features/server/src/jsonServer.ts` and upstream `vscode-json-languageservice`.
