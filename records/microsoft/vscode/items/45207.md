# Issue #45207 — [json] External schemas referenced from schema not updated

[https://github.com/microsoft/vscode/issues/45207](https://github.com/microsoft/vscode/issues/45207)

## Summary

Still present on main: when an externally-referenced schema file changes, only that file's schema is invalidated via resetSchema — parent schemas that $ref it remain stale in cache until manually cleared or restarted.

## Change summary

Changes to JSON schema files referenced via $ref from another schema are not picked up without restarting VS Code or clearing the schema cache.

## Best solution

The JSON language service's `resetSchema` should transitively invalidate all schemas that reference the changed schema URI. This likely requires tracking reverse-references (dependents) in `vscode-json-languageservice` so that when a schema is reset, all schemas that $ref it are also evicted from cache.
