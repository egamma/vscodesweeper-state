# Issue #201703 — Redo action does not reveal content that was added by accepting inlay hint

[https://github.com/microsoft/vscode/issues/201703](https://github.com/microsoft/vscode/issues/201703)

## Summary

Still reproducible on main: the inlay-hint accept path calls `executeEdits` without an `endCursorState`, so redo doesn't set the cursor at the edit site and the viewport isn't revealed.

## Change summary

Redo after undoing an accepted inlay hint does not scroll to or reveal the re-applied edit.

## Best solution

Pass a cursor-state computer (or explicit Selection at the edit range) as the third argument to `this._editor.executeEdits` in `inlayHintsController.ts` so that undo/redo restores the cursor position and triggers viewport reveal.
