# Issue #312549 — Linux (Snap) + Copilot Chat cannot type Chinese with IBus (can paste Chinese only)

[https://github.com/microsoft/vscode/issues/312549](https://github.com/microsoft/vscode/issues/312549)

## Summary

Confirmed on main: the Snap `electron-launch` script unconditionally appends `--ozone-platform=x11` (line 279), overriding any user-supplied Wayland flags and breaking IME composition for CJK input in Wayland sessions. The fix is localized to one line in the launcher script.

## Change summary

IME composition for CJK languages fails in the Snap package on Wayland because the launcher forces X11 mode unconditionally.

## Best solution

Conditionally set `--ozone-platform` in `resources/linux/snap/electron-launch` based on the already-detected `wayland_available` variable (lines 148-166), e.g. use `--ozone-platform=wayland --enable-wayland-ime` when Wayland is detected and fall back to `--ozone-platform=x11` otherwise. This is related to but narrower than https://github.com/microsoft/vscode/issues/207033 (defaulting to Wayland broadly).
