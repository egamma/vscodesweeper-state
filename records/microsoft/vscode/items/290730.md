# Issue #290730 — No indication in subagent that intervention is needed

[https://github.com/microsoft/vscode/issues/290730](https://github.com/microsoft/vscode/issues/290730)

## Summary

Still valid on main: when a subagent is collapsed and waiting for user confirmation, the header shows the same spinner icon as when actively working, providing no visual cue that intervention is needed. Author is a team member — keeping open.

## Change summary

Replace the spinner icon on a collapsed subagent header with a distinct indicator (e.g. warning/exclamation icon) when the subagent is waiting for user confirmation.

## Best solution

In `chatSubagentContentPart.ts`, update the autorun at ~line 326 (and the `trackToolState` logic) so that when `toolsWaitingForConfirmation > 0`, the collapsed header icon switches from `Codicon.circleFilled` to something like `Codicon.warning` or a dedicated attention icon, making it clear the subagent is blocked on user input rather than actively working.
