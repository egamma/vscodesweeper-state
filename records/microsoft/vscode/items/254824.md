# Issue #254824 — Code CLI script fails with "Unable to determine app path from symlink" with bash 5.3

[https://github.com/microsoft/vscode/issues/254824](https://github.com/microsoft/vscode/issues/254824)

## Summary

Still reproduces on main: the `app_realpath` function in `resources/darwin/bin/code.sh` uses nested parameter expansion `${SOURCE_DIR%%${SOURCE_DIR#*.app}}` that behaves differently under bash 5.3 (Homebrew) vs macOS's native bash 3.2. An open PR (#255558) exists but has not merged.

## Change summary

The `code` CLI launcher script on macOS fails with 'Unable to determine app path from symlink' when the user's default bash is version 5.3 (e.g. installed via Homebrew).

## Best solution

Review and land https://github.com/microsoft/vscode/pull/255558 (or an equivalent fix) that replaces the non-portable nested parameter expansion in `resources/darwin/bin/code.sh` line 23 with a portable alternative such as `grep -o '.*\.app'`. The fix is small and localized to that one shell function.
