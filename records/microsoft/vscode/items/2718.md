# Issue #2718 — Git - Automatically insert line breaks in git commit messages

[https://github.com/microsoft/vscode/issues/2718](https://github.com/microsoft/vscode/issues/2718)

## Summary

The core request — automatically inserting hard line breaks in user-typed commit messages before committing — remains unimplemented. With 111 upvotes and `help wanted`, this is a well-supported Backlog feature request that should stay open.

## Change summary

Automatically insert hard line breaks at a configurable column (e.g. 72) in user-typed git commit messages before the actual commit is made.

## Best solution

Implement an opt-in setting (e.g. `git.commitMessageAutoWrap`) that hard-wraps the commit message body at a configurable column before committing. The `help wanted` label invites community contribution; the wrapping logic from the removed `wrapCommitMessage` helper (commit 4cd1f6ce840) could serve as a starting point, extended to run on user-typed messages at commit time.
