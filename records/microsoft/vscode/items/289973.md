## Best solution

The worktree cleanup path in `extensions/copilot/src/extension/chatSessions/vscode-node/chatSessionWorktreeServiceImpl.ts` (around line 486, `cleanupWorktreeOnArchive`) should be hardened: if the initial stash-based migration failed ('Illegal argument'), the worktree still held the user's changes — cleanup should refuse to delete a worktree whose migration was never confirmed successful. A transactional guard ensuring the original workspace is untouched when migration fails would prevent this class of data loss. The maintainer's ask for git output channel logs showing the exact failing `git stash` commands is the right next step to identify the root cause of the 'Illegal argument' failure.
