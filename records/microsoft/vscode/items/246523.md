# Issue #246523 — scroll behavior is not smooth sometimes

[https://github.com/microsoft/vscode/issues/246523](https://github.com/microsoft/vscode/issues/246523)

## Summary

Team-authored issue (MEMBER); must stay open per policy. The underlying scroll-jump problem in the chat list has been partially mitigated (comments 8–10 confirm improvement in v1.108) but comment 11 documents remaining edge cases with large responses causing layout recalculations that jerk the scroll position. Multiple related fixes have landed but the root cause — async height measurement in the ListView causing scroll-anchor drift when tall off-screen items are measured — remains.

## Change summary

Chat scroll position jumps erratically in long conversations when lazily-rendered items with large content cause sudden height recalculations in the ListView.

## Best solution

The remaining problem is in the chat list's height-update path (src/vs/workbench/contrib/chat/browser/widget/chatWidget.ts and the underlying ListView's updateElementHeight). When a tall item scrolls into/out of view and its measured height changes, the scroll anchor needs to compensate. The fix likely lives in the list view's scroll-anchor logic or in chat-specific height-update scheduling. Related open issue https://github.com/microsoft/vscode/issues/287163 covers a related symptom (output stops scrolling during generation) but this issue's edge case (scroll jumps when scrolling *up* past large responses) is distinct and still needs a targeted fix.
