# Issue #300389 — LaTeX Support for Chat Window

[https://github.com/microsoft/vscode/issues/300389](https://github.com/microsoft/vscode/issues/300389)

## Summary

VS Code already supports math rendering in chat via KaTeX (enabled by default with `chat.experimental.mathEnabled`). The real issue is the model not wrapping equations in `$`/`$$` delimiters — as comment 4 showed, telling the model to use those delimiters works. The remaining ask (adjusting the system prompt) is a model-behavior concern, not a VS Code rendering gap.

## Change summary

Request for LaTeX/math rendering in the chat window when Copilot outputs equations.

## Best solution

The rendering capability already exists — KaTeX support is built in and on by default. The model sometimes outputs raw LaTeX without `$`/`$$` delimiters; users can prompt the model to use them. Any system-prompt adjustment to default to math delimiters is a Copilot model/prompt concern, not a VS Code core feature gap.
