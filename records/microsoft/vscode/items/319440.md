# Issue #319440 — Add fixed OAuth callback port option for MCP servers in Copilot

[https://github.com/microsoft/vscode/issues/319440](https://github.com/microsoft/vscode/issues/319440)

## Summary

Valid, well-articulated feature request for a user-configurable OAuth callback port in MCP server config. Not yet implemented — the current code uses a hardcoded DEFAULT_AUTH_FLOW_PORT (33418) with fallback to random, but no per-server override exists. A team member has confirmed the need.

## Change summary

Allow users to specify a fixed OAuth callback port per MCP server so that identity providers requiring exact redirect URI matching can be pre-registered.

## Best solution

Add an optional `callbackPort` field to `McpServerTransportHTTPOAuth` in `src/vs/workbench/contrib/mcp/common/mcpTypes.ts`, thread it through the MCP auth flow in `src/vs/workbench/api/node/loopbackServer.ts` (replacing the hardcoded `DEFAULT_AUTH_FLOW_PORT` when configured), and document the new option in mcp.json schema.
