# Issue #302116 — Sessions: Conversation appears to alter the user's original prompt

[https://github.com/microsoft/vscode/issues/302116](https://github.com/microsoft/vscode/issues/302116)

## Summary

Team-authored UX issue from a user study — the original prompt in an Agents window session appears summarized/truncated when scrolling back, undermining user confidence. No evidence of a fix on main; keeping open for maintainer judgment.

## Change summary

In the Agents window, scrolling back to the original session prompt shows a summarized version rather than the full user-authored text, making it seem like the agent altered the prompt.

## Best solution

Ensure the full original prompt text is always preserved and displayed in the conversation history, with the summarized version used only for the session title. Users should be able to copy/reuse the exact prompt. The issue was observed specifically with cloud sessions — the cloud session protocol or its rendering in the agents window chat widget likely needs to retain and surface the original prompt verbatim.
