# Issue #302116 — Sessions: Conversation appears to alter the user's original prompt

[https://github.com/microsoft/vscode/issues/302116](https://github.com/microsoft/vscode/issues/302116)

## Summary

Team-authored issue from a UX study — the Agents window summarizes/truncates the user's original prompt when scrolling back in the conversation, undermining trust. No fix found on main; keeping open for the owning team.

## Change summary

After an agent session progresses, scrolling back to the original prompt shows a summarized version instead of the full text, with no way to retrieve the original.

## Best solution

Ensure the full original prompt text is always preserved and displayed in the conversation history (even if a summary is used for the session title). Likely involves the chat widget or session model in `src/vs/sessions/`. The cloud-sessions backend may also need to preserve the unmodified user input. Needs coordination between the Agents window team and the cloud sessions team to determine where the summarization occurs.
