# Issue #169813 — Compound launch configuration: add abiliity to ignore preLaunchTask of launch configurations.

[https://github.com/microsoft/vscode/issues/169813](https://github.com/microsoft/vscode/issues/169813)

## Summary

Valid feature request that reached 20+ upvotes and was promoted to Backlog. The ICompound interface on current main (src/vs/workbench/contrib/debug/common/debug.ts:875) has no property to suppress individual configurations' preLaunchTasks — the feature remains unimplemented.

## Change summary

Add an option to compound launch configurations to skip preLaunchTask of individual configurations, avoiding duplicate config entries when a compound-level preLaunchTask handles all builds.

## Best solution

Add a boolean property (e.g. `ignoreConfigurationPreLaunchTasks`) to `ICompound` in `src/vs/workbench/contrib/debug/common/debug.ts`, update the compound schema in `debugSchemas.ts`, and skip individual `preLaunchTask` execution in `debugService.ts` when the flag is set. This is a localized change in the debug contribution.
