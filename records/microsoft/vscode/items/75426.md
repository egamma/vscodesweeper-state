# Issue #75426 — Breakpoint moves with copy line up/down action

[https://github.com/microsoft/vscode/issues/75426](https://github.com/microsoft/vscode/issues/75426)

## Summary

This should stay open as an intentional but contested editor behavior already classified as under discussion; member comments explain the current implementation and the tradeoff rather than identifying a clear duplicate or fixed bug.

## Change summary

Copy Line Up/Down currently transfers breakpoint and line-associated state to the copied/moved line, while the reporter expects the original line to keep that state.

## Best solution

Decide the desired editor semantics for line-associated state during Copy Line Up/Down in the editor contribution that implements these commands, or document the current insertion-based behavior as the accepted design if the team does not want to change it.
