# Issue #75426 — Breakpoint moves with copy line up/down action

[https://github.com/microsoft/vscode/issues/75426](https://github.com/microsoft/vscode/issues/75426)

## Summary

Still reproduces on current main — the copy-down action inserts text above the original line, pushing it (and its decorations/breakpoints) down. Team members disagreed on whether this is as-designed or a bug; hediet marked it under-discussion. Keeping open for the community process.

## Change summary

Copy Line Down/Up moves breakpoints and decorations away from the original line instead of keeping them in place.

## Best solution

Decide whether the insert should happen such that the original line keeps its position (and thus its decorations), or whether the copy should inherit them. The fix would be in `src/vs/editor/contrib/linesOperations/browser/copyLinesCommand.ts` — reversing the insertion point for copy-down (insert below instead of above) would preserve original-line decorations. Needs a product decision first.
