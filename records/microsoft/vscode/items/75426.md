## Change summary

Copy Line Up/Down moves the existing breakpoint/decorations with the original line instead of leaving them on the original location or copying them to the duplicated line.

## Best solution

Leave this as an editor contribution design decision for maintainers to resolve: either document/keep the current sticky-decoration behavior or define a targeted editor/debug API behavior for line-associated data during duplicate-line commands.
