# Issue #137938 — [html] "Auto Closing Tags" adds unnecessary tags in some cases with prettier-vscode 

[https://github.com/microsoft/vscode/issues/137938](https://github.com/microsoft/vscode/issues/137938)

## Summary

Still potentially relevant. The Sep 2024 fix (f0cdaab75f0) added a `isSingleLine` guard that mitigates many formatter-triggered auto-close insertions, but the underlying issue — no `TextDocumentChangeReason.Format` exists to filter format operations — means single-line format edits ending with `>` could still trigger spurious auto-close. A team member confirmed the bug in Dec 2022; the partial mitigation shipped later but a full fix is uncertain.

## Change summary

HTML auto-closing tags feature fires during prettier formatting operations, inserting unwanted duplicate closing tags.

## Best solution

Add explicit filtering in `autoInsertion.ts` to suppress auto-close when changes originate from a format provider (possibly by checking if multiple edits arrive in the same event batch, or by coordinating with the formatting request lifecycle). Alternatively, upstream a `TextDocumentChangeReason.Format` variant in the VS Code API so extensions can distinguish formatter edits from user typing.
