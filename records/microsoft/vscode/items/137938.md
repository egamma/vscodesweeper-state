# Issue #137938 — [html] "Auto Closing Tags" adds unnecessary tags in some cases with prettier-vscode 

[https://github.com/microsoft/vscode/issues/137938](https://github.com/microsoft/vscode/issues/137938)

## Summary

Still present on main: the auto-closing tag logic in autoInsertion.ts has no mechanism to distinguish formatting edits from user typing, so when prettier (or any formatter) produces changes ending with `>`, a spurious closing tag is inserted. A team member (aeschli) confirmed in Dec 2022.

## Change summary

HTML auto-closing tags fires during document formatting edits (e.g. prettier), inserting duplicate closing tags.

## Best solution

The auto-insertion handler needs to detect and skip formatting-triggered document changes. Since `TextDocumentChangeReason` lacks a `Format` value, a workaround could track whether a format operation is in progress (e.g. via `onWillSaveTextDocument` or a formatting-provider wrapper) and suppress auto-insert during that window. The fix lives in `extensions/html-language-features/client/src/autoInsertion.ts`.
