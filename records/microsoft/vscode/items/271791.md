## Change summary

Git gutter diff decorations (added/modified/deleted bars) stopped appearing in VS Code 1.105 on Windows 11 build 26200+, and opening working-tree diffs errors with 'file was not found'.

## Best solution

The area owner should follow up on the trace logs the reporter provided — the 'file was not found' error when opening diffs on drive E: strongly suggests a path/URI mismatch in the git extension's dirty-diff pipeline, possibly related to drive-letter normalization. Check whether the path.sep and separator-handling fixes (commits b276d35cd60, d0cb4a16a56) already address this, and if not, investigate the URI construction in `extensions/git/src/` for non-C-drive paths on recent Windows 11 builds.
