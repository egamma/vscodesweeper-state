# Issue #262216 — Not respecting per monitor subpixel rendering cleartype config

[https://github.com/microsoft/vscode/issues/262216](https://github.com/microsoft/vscode/issues/262216)

## Summary

This is the same underlying problem as the closed issue #110864 — per-monitor ClearType subpixel order (RGB vs BGR) is not respected because font rendering is controlled by the Electron/Chromium runtime, not by VS Code's own code. The prior issue was closed as upstream/out-of-scope for this reason.

## Change summary

VS Code does not respect per-monitor ClearType subpixel rendering configuration (e.g. BGR on an external monitor), causing blurry or incorrect text rendering.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/110864, which already tracked this exact problem (subpixel order not following system settings) and was closed as upstream/out-of-scope — the fix belongs in Chromium/Electron's DirectWrite integration, not in VS Code core. The workaround is to disable subpixel rendering via `"disable-lcd-text": true` in `argv.json`.
