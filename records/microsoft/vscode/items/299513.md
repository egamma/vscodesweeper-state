# Issue #299513 — issue with claude code in github copilot

[https://github.com/microsoft/vscode/issues/299513](https://github.com/microsoft/vscode/issues/299513)

## Summary

The error comes from the Claude Code CLI binary refusing to run under root/sudo — this is a security restriction enforced by the Claude Code SDK itself (not VS Code core). The Copilot Chat extension spawns that process; the fix belongs in the extension or in Claude Code's upstream policy, not in VS Code core.

## Change summary

Claude Code agent fails with exit code 1 when VS Code Remote SSH session runs as root, because the CLI refuses root execution.

## Best solution

Report to the Copilot Chat extension team (github.copilot-chat) or the Claude Code CLI team. The extension could either detect the root scenario and provide a clearer error message, or the Claude Code CLI could relax its root restriction when `--dangerously-skip-permissions` is not passed. Users can work around it by connecting as a non-root user.
