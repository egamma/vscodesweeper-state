# Issue #202880 — SCM Input - Allow contributing editor context menus

[https://github.com/microsoft/vscode/issues/202880](https://github.com/microsoft/vscode/issues/202880)

## Summary

Valid, unimplemented feature request. The SCM input editor uses `isSimpleWidget: true`, which maps its context menu to `MenuId.SimpleEditorContext` — a menu not exposed to extensions. There is no dedicated `scm/inputBox/context` contribution point either, so extensions currently cannot add context menu items to the SCM input editor. The gap remains on current main.

## Change summary

Allow extensions to contribute context menu items to the editor used in the SCM input box, similar to how `editor/context` works for regular editors.

## Best solution

Expose a new `scm/inputBox/context` menu contribution point (or override `contextMenuId` on the SCM input's `CodeEditorWidget` to a dedicated `MenuId`) so extensions can register context menu actions in the SCM input editor. The likely files are `src/vs/workbench/services/actions/common/menusExtensionPoint.ts` (add the contribution point), `src/vs/platform/actions/common/actions.ts` (add the MenuId), and `src/vs/workbench/contrib/scm/browser/scmInput.ts` (set `contextMenuId` in `codeEditorWidgetOptions`).
