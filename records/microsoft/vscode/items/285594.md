# Issue #285594 — Language configuration `lineComment` supports both string and object values

[https://github.com/microsoft/vscode/issues/285594](https://github.com/microsoft/vscode/issues/285594)

## Summary

Still present on main — the JSON schema for `lineComment` only declares `type: 'object'`, producing a spurious warning when extension authors use the valid string shorthand. Two open PRs exist but neither has merged.

## Change summary

The JSON schema for `lineComment` in language-configuration.json warns on string values even though the runtime accepts them.

## Best solution

Update the `lineComment` schema entry in `src/vs/workbench/contrib/codeEditor/common/languageConfigurationExtensionPoint.ts` to use `oneOf` accepting both a plain string and the current object shape. One of the two open PRs (#312588 or #319349) should be reviewed and merged.
