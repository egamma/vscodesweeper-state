# Issue #147190 — [Accessibility] Audio Cues doesn't work in web editor

[https://github.com/microsoft/vscode/issues/147190](https://github.com/microsoft/vscode/issues/147190)

## Summary

Resolved through multiple fixes: the switch from opus to mp3 (PR #158732, shipped 1.71.0) addressed browser codec compatibility, and CSP fixes in vscode-dev (PRs #606 and #879) allowed media loading. Current main has mp3 audio files and proper `media-src 'self'` CSP. The feature now ships as 'accessibility signals' and works on web.

## Change summary

Audio cues (now accessibility signals) did not play in the web editor due to CSP blocking media and opus codec incompatibility in some browsers.

## Best solution

Keep the shipped fixes (mp3 migration in 1.71.0, CSP corrections in vscode-dev). The issue is fully resolved — audio signals play on vscode.dev and github.dev. Close as completed.
