# Issue #255286 — Language Model API doesn't allow system prompt

[https://github.com/microsoft/vscode/issues/255286](https://github.com/microsoft/vscode/issues/255286)

## Summary

The System role exists as a proposed API (`languageModelSystem`) but has not yet been finalized into the stable `vscode.d.ts`. The issue carries the `On Deck` milestone and `api-finalization` label, indicating active team tracking — must stay open per milestone rules.

## Change summary

The Language Model API's `LanguageModelChatMessageRole` enum lacks a `System` role in the stable API, forcing extension authors to pass system prompts as user messages.

## Best solution

Finalize the `languageModelSystem` proposed API by promoting `LanguageModelChatMessageRole.System = 3` from `src/vscode-dts/vscode.proposed.languageModelSystem.d.ts` into the stable `src/vscode-dts/vscode.d.ts` enum. The internal plumbing (type converters, extHostTypes, extHostLanguageModels gate) is already in place; the remaining work is the finalization decision and removing the proposed-API gate in `extHostLanguageModels.ts`.
