# Issue #200634 — Question mark disappears when the next colon is highlighted in high contrast themes

[https://github.com/microsoft/vscode/issues/200634](https://github.com/microsoft/vscode/issues/200634)

## Summary

Keep open and route to the editor rendering area. The report has a concrete font/theme-specific reproduction, and I found no linked duplicate or concrete fixing commit that would justify closing it.

## Change summary

In high contrast themes with JetBrains Mono, selecting the colon after a question mark can make the adjacent question mark disappear.

## Best solution

Investigate the editor view-line selection rendering path so font ligatures do not visually merge selected and unselected text in a way that hides adjacent glyphs in high contrast themes, likely around src/vs/editor/browser/viewParts/viewLines/viewLine.ts and src/vs/editor/browser/viewParts/selections/selections.ts.

- **current code path:** src/vs/editor/browser/viewParts/viewLines/viewLine.ts still adds an inline-selected-text decoration for selected ranges in high contrast themes, and src/vs/editor/browser/viewParts/selections/selections.ts applies editor.selectionForeground to .inline-selected-text.
- **theme colors:** src/vs/platform/theme/common/colors/editorColors.ts currently defines high contrast selection foreground/background values, so selected text is intentionally recolored in the same rendering path implicated by the report.
