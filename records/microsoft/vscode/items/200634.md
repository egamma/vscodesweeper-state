# Issue #200634 — Question mark disappears when the next colon is highlighted in high contrast themes

[https://github.com/microsoft/vscode/issues/200634](https://github.com/microsoft/vscode/issues/200634)

## Summary

Still reproduces on main: in high contrast themes, selected text gets an `inline-selected-text` span decoration that splits ligature glyphs across element boundaries, causing partial glyph disappearance with fonts like JetBrains Mono.

## Change summary

Question mark disappears when the adjacent colon is selected in high contrast themes with a ligature font (JetBrains Mono).

## Best solution

The fix likely lives in `src/vs/editor/browser/viewParts/viewLines/viewLine.ts` where the `inline-selected-text` decoration is applied. The selection foreground span should avoid splitting ligature sequences — either by expanding the decoration to cover the full ligature cluster, or by using an alternative CSS technique (e.g. `::selection` pseudo-element) that doesn't break glyph boundaries.
