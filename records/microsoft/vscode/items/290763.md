# Issue #290763 — uvx not found when trying to start MCP server although available in the terminal

[https://github.com/microsoft/vscode/issues/290763](https://github.com/microsoft/vscode/issues/290763)

## Summary

Team-authored issue (MEMBER); keeping open. The bug is confirmed by multiple commenters: MCP server processes under WSL (and macOS) don't inherit the user's full PATH, so tools like `uvx` installed in user-local directories aren't found.

## Change summary

MCP stdio servers fail to find user-installed commands (e.g. `uvx`) because the spawned process inherits a limited PATH that lacks user-local directories.

## Best solution

The MCP stdio launch path in `src/vs/workbench/api/node/extHostMcpNode.ts` starts from `process.env` but under WSL/remote scenarios the extension host process may not have a fully resolved login shell environment. A shell-environment resolution step (similar to what the terminal does) before spawning the MCP process would fix this. Alternatively, document or support explicit PATH configuration in MCP server definitions.
