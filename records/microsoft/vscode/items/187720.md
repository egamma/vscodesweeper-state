# Issue #187720 — Ability to overtype angle brackets (similar to normal parentheses)

[https://github.com/microsoft/vscode/issues/187720](https://github.com/microsoft/vscode/issues/187720)

## Summary

Valid feature request for overtyping angle brackets inserted by completions/snippets without requiring them to be declared as autoClosingPairs. The existing `autoClosingOvertype` mechanism only handles pairs registered in `autoClosingPairs`, which languages like Rust intentionally omit for `<>` due to operator ambiguity. This gap remains on current main.

## Change summary

Allow overtyping of closing angle brackets that were auto-inserted by completions or snippets, even when `<>` is not declared as an autoClosingPair in the language configuration.

## Best solution

Extend the overtype logic in `src/vs/editor/common/cursor/cursorTypeEditOperations.ts` to also track and allow overtyping of closing characters inserted by completions/snippets (not just those from auto-closing pairs). This could involve a new `overtypablePairs` category in language configuration, or extending the `autoClosedCharacters` tracking to include snippet-inserted brackets. The discussion in comments suggests this is the right approach.
