# Issue #202502 — Firefox: buttons/links trigger on `click` when dragged from editor

[https://github.com/microsoft/vscode/issues/202502](https://github.com/microsoft/vscode/issues/202502)

## Summary

Still present on main: the button click handler in button.ts has no guard against drag-originated click events, which Firefox on macOS fires when a drag ends over the button. Confirmed by a team member; no fix has landed.

## Change summary

In VS Code web on Firefox/macOS, dragging a text selection from the editor onto a sidebar button/link triggers a spurious click on that button.

## Best solution

Add a guard in the button click handler (or globally in the editor's drag handling) to suppress click events that follow a drag — e.g. track whether a drag was in progress and ignore the subsequent click if the mousedown didn't originate on the button. The fix likely lives in `src/vs/base/browser/ui/button/button.ts` (line ~169) or as a `preventDefault` on the editor's drag end.
