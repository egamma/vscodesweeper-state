# Issue #149912 — applyTo: closedDocuments and a custom owner caused errors to leak/persist

[https://github.com/microsoft/vscode/issues/149912](https://github.com/microsoft/vscode/issues/149912)

## Summary

Still reproduces on main: the `closedDocuments` applyTo logic in WatchingProblemCollector never removes markers when a file is opened, and `_cleanMarkers` skips open files for `closedDocuments` owners — so markers set while the file was closed persist indefinitely once the file is reopened.

## Change summary

Problem matcher markers with `applyTo: closedDocuments` and a custom owner persist after the file is opened or the error is fixed, because the cleanup logic skips open files.

## Best solution

In `AbstractProblemCollector`, the `onModelAdded` listener should remove markers from `closedDocuments` owners for the newly-opened resource. Additionally, `_cleanMarkers` should unconditionally remove markers that were recorded for cleaning regardless of the current open/closed state (the `applyTo` filter already gates recording, so cleanup should be unconditional).
