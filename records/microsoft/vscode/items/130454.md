# Issue #130454 — [Bug] `"merge-conflict.diffViewPosition": "Below"` opens multiple diff editors

[https://github.com/microsoft/vscode/issues/130454](https://github.com/microsoft/vscode/issues/130454)

## Summary

Still reproduces on main: the "Below" diff view position path unconditionally creates a new editor group on every compare, whereas "Beside" reuses the existing side column via ViewColumn.Beside. The bug is a missing reuse check in the built-in merge-conflict extension.

## Change summary

Setting `merge-conflict.diffViewPosition` to `"Below"` opens a new diff editor group on every compare instead of reusing the existing one.

## Best solution

Guard the "Below" path in `extensions/merge-conflict/src/commandHandler.ts` (around line 143) so it only creates a new group below when one doesn't already exist, or reuse an existing below group for subsequent diffs — mirroring the reuse behavior of the "Beside" path.
