# Issue #144730 — [html] allow to turn off closing tags when not needed

[https://github.com/microsoft/vscode/issues/144730](https://github.com/microsoft/vscode/issues/144730)

## Summary

Feature request for smarter HTML auto-closing tag behavior (Sublime-style). The basic on/off toggle `html.autoClosingTags` already exists, but the user and maintainer discussion centers on more context-aware closing — e.g., not inserting a closing tag when wrapping existing content. Keeping open as an accepted feature request in Backlog.

## Change summary

User wants more intelligent HTML auto-closing tags that avoid inserting unnecessary closing tags, similar to Sublime Text's behavior.

## Best solution

Enhance the auto-close logic in `extensions/html-language-features/client/src/autoInsertion.ts` to be context-aware — e.g., suppress the closing tag when the cursor is above existing content that would be wrapped. The existing `html.autoClosingTags` boolean could be extended to a multi-value setting (e.g., `always`, `never`, `smart`) or a separate `html.autoClosingTags.mode` setting could be introduced.
