# Issue #321482 — Claude shows twice in Agents window dropdown

[https://github.com/microsoft/vscode/issues/321482](https://github.com/microsoft/vscode/issues/321482)

## Summary

Team-filed UX polish issue: the Agents window dropdown shows 'Claude' twice because two providers (Copilot Chat and Agent Host) both advertise it. The code intentionally lists both with disambiguation headers, but the author finds the distinction confusing. Keeping open per team-authored policy.

## Change summary

The Agents window new-session dropdown shows 'Claude' twice — once per provider — which is confusing to users who don't understand the provider distinction.

## Best solution

Either deduplicate same-label session types across providers in the picker (picking the preferred/default provider automatically), or make the disambiguation headers clearer so users understand what each entry means. The logic lives in `src/vs/sessions/contrib/chat/browser/sessionTypePicker.ts` around the `hasDuplicateLabels` / `showSectionHeaders` path.
