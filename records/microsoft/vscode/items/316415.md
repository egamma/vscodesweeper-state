# Issue #316415 — ARIA attributes "aria-label" are not valid for the element <body> with implicit ARIA role "generic" violates WCAG 2.1 SC 4.1.2 Name, Role, Value (Level A)

[https://github.com/microsoft/vscode/issues/316415](https://github.com/microsoft/vscode/issues/316415)

## Summary

The `<body aria-label="">` in multiple HTML templates is technically a WCAG violation — `aria-label` is not a valid attribute for elements with the implicit `generic` role. The attribute is present but empty and never updated dynamically. The real workbench landmark structure lives in the `mainContainer` div (with `role="application"`).

## Change summary

The `<body>` element carries an empty `aria-label` attribute that is invalid per ARIA spec for elements with the `generic` role, triggering automated accessibility test failures.

## Best solution

Remove the `aria-label=""` attribute from the `<body>` elements in all HTML templates (`src/vs/code/electron-browser/workbench/workbench.html`, `src/vs/code/browser/workbench/workbench.html`, `src/vs/code/electron-browser/workbench/workbench-dev.html`, `src/vs/code/browser/workbench/workbench-dev.html`, and the sessions and build variants). If a label is needed on the body, assign `role="document"` or `role="application"` first, which supports `aria-label`. The empty string suggests it was vestigial rather than intentional.
