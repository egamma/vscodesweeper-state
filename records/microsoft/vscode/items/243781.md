# Issue #243781 — Command palette can be triggered from issue reporter

[https://github.com/microsoft/vscode/issues/243781](https://github.com/microsoft/vscode/issues/243781)

## Summary

Team-authored bug that remains unfixed on main. The issue reporter (legacy aux window path) does not suppress the command palette keybindings (F1 / Cmd+Shift+P), allowing the command palette to appear in a context where it doesn't belong; dismissing it with Escape then hides the reporter window.

## Change summary

Command palette shortcuts (F1, Cmd+Shift+P) are not suppressed in the issue reporter window, causing unexpected behavior when dismissed.

## Best solution

Suppress or ignore the quick-access / command-palette keybindings when the issue reporter auxiliary window is focused. The fix likely belongs in the legacy aux-window issue reporter path in `src/vs/workbench/contrib/issue/electron-browser/issueReporterService.ts` or in the keybinding handling in `nativeIssueFormService.ts`. With the newer wizard-based editor pane path becoming default, this may also be worth verifying there.
