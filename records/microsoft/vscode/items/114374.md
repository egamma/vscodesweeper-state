# Issue #114374 — Git - Discarding changes on open file error

[https://github.com/microsoft/vscode/issues/114374](https://github.com/microsoft/vscode/issues/114374)

## Summary

Still valid on main: the `getGitErrorCode` function in `extensions/git/src/git.ts` does not recognize the "failed to remove ... Invalid argument" stderr pattern from git, so the raw git error is surfaced instead of a user-friendly message. A maintainer acknowledged the issue and a contributor offered to submit a PR but none has landed.

## Change summary

Discarding changes on a file locked by another application (e.g., a CSV open in Excel) shows a raw git error ("Invalid argument") instead of a user-friendly message explaining the file is in use.

## Best solution

Add a new case in `getGitErrorCode()` in `extensions/git/src/git.ts` to match the `failed to remove` stderr pattern and map it to a descriptive error code, then surface a user-friendly message in the discard-changes flow (likely in `extensions/git/src/commands.ts`). The `help wanted` label is appropriate — @LeonardoBraga offered to work on it in Dec 2024.
