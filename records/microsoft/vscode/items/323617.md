# Issue #323617 — Hiding chats inside a session doesn't make sense

[https://github.com/microsoft/vscode/issues/323617](https://github.com/microsoft/vscode/issues/323617)

## Summary

Team-authored UX feedback on the Agents window session management: hiding chats within a session is confusing (looks like switching, and hidden running chats continue silently). Keeping open for maintainer judgment.

## Change summary

The hide-chats UI inside an Agents window session is misleading — it looks like a switcher, and hiding running chats lets them continue invisibly.

## Best solution

Rethink the multi-chat visibility controls in the Agents window session UI: either remove in-session hide (only allow switching), or surface clear state indicators when hidden chats are still running. Likely lives in `src/vs/sessions/` or the agents-window chat contribution layer.
