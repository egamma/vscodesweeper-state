# Issue #295547 — Do not scroll chat too much such that my full message is not visible

[https://github.com/microsoft/vscode/issues/295547](https://github.com/microsoft/vscode/issues/295547)

## Summary

Still reproduces on current main. When a long user message is sent, `scrollToEnd()` is called unconditionally, which reveals the bottom of the last element rather than the top of the user's message. The request to reveal the top of the user message instead is a valid UX polish item.

## Change summary

After sending a long chat message, the chat view scrolls too far down so the user's own message is not fully visible; it should reveal the top of the user message instead.

## Best solution

When handling an `addRequest` event in `chatWidget.ts` (around line 2244), instead of calling `scrollToEnd()`, reveal the newly added request element with `relativeTop: 0` so the top of the user message is visible. This keeps the user oriented after submitting a long prompt.
