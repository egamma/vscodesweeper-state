# Issue #161438 — Use new code action widget in problems view

[https://github.com/microsoft/vscode/issues/161438](https://github.com/microsoft/vscode/issues/161438)

## Summary

Still valid: the problems panel's quick-fix UI uses the old ActionBar/context-menu pattern (`QuickFixActionViewItem` in `markersViewActions.ts`), not the `IActionWidgetService` code action widget used in the editor. The migration hasn't happened.

## Change summary

Adopt the `IActionWidgetService` code action widget for lightbulb quick fixes in the Problems panel, replacing the current context-menu-based `QuickFixActionViewItem`.

## Best solution

Refactor `QuickFixActionViewItem` in `src/vs/workbench/contrib/markers/browser/markersViewActions.ts` (and its consumers in `markersTreeViewer.ts` and `markersTable.ts`) to present code actions through `IActionWidgetService` instead of a plain context menu, mirroring how `src/vs/editor/contrib/codeAction/browser/codeActionController.ts` invokes the widget.
