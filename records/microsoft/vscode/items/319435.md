# Issue #319435 — VSCode instances all open in the same Windows Desktop after an update/restart

[https://github.com/microsoft/vscode/issues/319435](https://github.com/microsoft/vscode/issues/319435)

## Summary

Team-authored feature request for Windows virtual desktop persistence across restarts; hard keep-open rule applies (MEMBER author). The capability does not exist in current VS Code — window restoration does not track virtual desktop placement.

## Change summary

VS Code windows should remember which Windows virtual desktop they were on before a reboot/update and restore to that desktop on relaunch.

## Best solution

Investigate whether the Windows API exposes virtual desktop IDs that can be persisted alongside window state (position/size) in the window state storage, and restore windows to their prior desktop on relaunch. This would likely touch `src/vs/platform/windows/` state serialization and Electron's native window management layer. May require Electron-level support or native module work to read/write virtual desktop associations.
