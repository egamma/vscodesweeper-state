# Issue #312686 — Copilot Chat auto-steers In Progress on new message instead of queueing to To Do

[https://github.com/microsoft/vscode/issues/312686](https://github.com/microsoft/vscode/issues/312686)

## Summary

The requested queue-first behavior already exists via the `chat.requestQueuing.defaultAction` setting — changing it from the default `steer` to `queue` gives exactly the behavior described. This is as-designed.

## Change summary

User wants new chat messages to be queued as To Do instead of auto-steering the in-progress request.

## Best solution

No code change needed. The user should set `chat.requestQueuing.defaultAction` to `"queue"` in their settings. The current default of `"steer"` is intentional, and the setting already provides the exact opt-in the user requests.
