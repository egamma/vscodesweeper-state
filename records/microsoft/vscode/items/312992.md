# Issue #312992 — Can't change Ollama's cloud API KEY

[https://github.com/microsoft/vscode/issues/312992](https://github.com/microsoft/vscode/issues/312992)

## Summary

This is about the Copilot extension's BYOK (Bring Your Own Key) flow for Ollama cloud — specifically, not being able to change an API key once set. The `chat-ext-issue` label already routes it correctly, and the API key management lives in the Copilot extension, not VS Code core. However, since the BYOK flow straddles core chat management UI and the Copilot extension, and reproduction details are thin, this should be kept open for the extension team to evaluate.

## Change summary

After adding an Ollama cloud API key for BYOK in Copilot, there is no way to change or update the stored API key.

## Best solution

The BYOK API key management (store/update/delete) is owned by the Copilot extension's authentication layer and the core chat management widget (`src/vs/workbench/contrib/chat/browser/chatManagement/chatModelsWidget.ts`). The extension team should verify whether a 'change API key' action exists and surface it in the model management UI. If the credential is stored via VS Code's SecretStorage API, an update or delete path needs to be exposed.
