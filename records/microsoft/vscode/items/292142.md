# Issue #292142 — Support for extensions to control tool invocation collapse behavior in Chat

[https://github.com/microsoft/vscode/issues/292142](https://github.com/microsoft/vscode/issues/292142)

## Summary

Valid feature request for an extension API to control tool invocation collapse/expand behavior in Chat. An open PR (#292143) by the author exists, so the issue should stay open pending maintainer review of that PR and the proposed API surface.

## Change summary

Allow extensions contributing Chat Tools to declare whether their tool invocations should be expanded or collapsed, instead of relying on hardcoded logic in shouldPinPart().

## Best solution

Review and land (or iterate on) the open PR at https://github.com/microsoft/vscode/pull/292143. The `shouldPinPart` method in `src/vs/workbench/contrib/chat/browser/widget/chatListRenderer.ts` currently hardcodes tool-specific pin decisions by tool ID (MCP, Mermaid, copilot_askQuestions, etc.); exposing a declarative `displayStyle` in the `languageModelTools` contribution point and/or a runtime `presentation` property would let extensions own this decision. A maintainer should evaluate whether the proposed API shape is the right one and whether the related issue https://github.com/microsoft/vscode/issues/292119 (user-facing collapse setting) should be considered together.
