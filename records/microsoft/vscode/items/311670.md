# Issue #311670 — %LOCALAPPDATA% appears in working dir

[https://github.com/microsoft/vscode/issues/311670](https://github.com/microsoft/vscode/issues/311670)

## Summary

The reporter conclusively traced this to Windows UE-V (User Experience Virtualization) intercepting process spawns from the GitHub Copilot Chat extension — UE-V writes its sync data with un-expanded environment variables into the working directory. This is not a VS Code core defect; it's an interaction between the Copilot Chat extension and UE-V's WindowsCommandPrompt template.

## Change summary

The GitHub Copilot Chat extension triggers Windows UE-V to create a literal `%LOCALAPPDATA%` folder tree in the workspace when a git repository is present.

## Best solution

File this against the GitHub Copilot Chat extension (or report to the UE-V/Windows team). The extension likely spawns cmd.exe in the workspace directory, which triggers UE-V's WindowsCommandPrompt template to sync profile data using un-expanded variables. The workaround is to rename the UE-V template file as the reporter discovered.
