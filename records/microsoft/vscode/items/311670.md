# Issue #311670 — %LOCALAPPDATA% appears in working dir

[https://github.com/microsoft/vscode/issues/311670](https://github.com/microsoft/vscode/issues/311670)

## Summary

Not reproducible on current Insiders/main where Copilot is built-in. The root cause is UE-V (Windows enterprise feature) intercepting process spawning by the old separate Copilot Chat extension; the built-in version does not trigger this behavior.

## Change summary

The old separate Copilot Chat extension triggers Windows UE-V to create a literal %LOCALAPPDATA% folder in the workspace when a git repo is present.

## Best solution

Close as not reproducible on current builds. The transition from the separate Copilot Chat extension to built-in Copilot resolved whatever process-spawning pattern triggered UE-V. Users on affected enterprise environments should upgrade to a version with built-in Copilot.
