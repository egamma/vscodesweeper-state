# Issue #303695 — MCP: "Failed to parse message: ''" warning from empty SSE priming frames

[https://github.com/microsoft/vscode/issues/303695](https://github.com/microsoft/vscode/issues/303695)

## Summary

Still present on main: two SSE handlers in extHostMcp.ts lack the `&& event.data` guard that the streamable backchannel handler already has, causing spurious 'Failed to parse message' warnings from empty SSE priming frames.

## Change summary

Empty SSE priming frames from MCP servers cause a repeated 'Failed to parse message: ""' warning in the MCP output log.

## Best solution

Add the same `&& event.data` guard already present at line 607 of `src/vs/workbench/api/common/extHostMcp.ts` to the two other SSE event handlers (POST SSE at line 521 and legacy SSE at line 655). Alternatively, guard in `pushMessage` in `mainThreadMcp.ts` to early-return on empty strings.
