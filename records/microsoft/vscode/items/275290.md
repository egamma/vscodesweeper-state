# Issue #275290 — Agent gets distracted by extra terminal context

[https://github.com/microsoft/vscode/issues/275290](https://github.com/microsoft/vscode/issues/275290)

## Summary

Team-authored issue reporting that terminal context (recent commands) distracts the agent from answering about an explicitly attached command. Keeping open per policy; the reporter and area owner have acknowledged it as a real but low-priority polish item contingent on telemetry showing the attachment action is popular.

## Change summary

When a terminal command is attached to a chat prompt and a newer command exists in the same terminal, the agent answers about the newer command instead of the attachment.

## Best solution

Improve the terminal context prompt or reorder attached-command context so attachments are given higher priority than ambient recent-terminal output; add vscbench evals for attachment scenarios to catch regressions.
