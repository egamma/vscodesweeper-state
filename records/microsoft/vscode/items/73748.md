# Issue #73748 — Be able disable task detection from tasks.json or override tasks execution

[https://github.com/microsoft/vscode/issues/73748](https://github.com/microsoft/vscode/issues/73748)

## Summary

Team-authored feature request for scheme-aware task detection control. The capability remains unimplemented on current main — task detection in abstractTaskService.ts reads tasks.json without scheme-based filtering. Keeping open per team-member authorship rule.

## Change summary

Allow extensions to disable or override task detection from tasks.json based on file scheme, so remote/non-local workspaces don't surface inapplicable tasks.

## Best solution

Add an API or configuration point in the task system (src/vs/workbench/contrib/tasks/browser/abstractTaskService.ts) that lets extensions opt in or opt out of tasks.json detection per URI scheme, or restrict default detection to the `file:` scheme with an extension opt-in for others. The referenced issue #33523 (resolveTask) is closed, so this needs its own implementation path.
