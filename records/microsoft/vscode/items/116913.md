# Issue #116913 — Git - VSC never sign the merge commit even if I turned `Enable Commit Signing` on

[https://github.com/microsoft/vscode/issues/116913](https://github.com/microsoft/vscode/issues/116913)

## Summary

Still reproduces on main: the `merge()` method in the built-in Git extension never passes GPG signing flags, so the `enableCommitSigning` setting is ignored for merge commits. The defect is localized and confirmed in current source.

## Change summary

Git merge commits created via VS Code's UI are never GPG-signed, even when the `git.enableCommitSigning` setting is enabled.

## Best solution

Read the `git.enableCommitSigning` config in the merge code path and pass `-S` to `git merge` when enabled, mirroring what the `commit()` method already does. The fix belongs in `extensions/git/src/git.ts` (`merge()` method) and possibly `extensions/git/src/repository.ts` to thread the option through. The same gap exists for `cherryPick()`.
