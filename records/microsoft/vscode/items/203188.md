# Issue #203188 — SCM - Stash viewer shows a deletion after popping the stash

[https://github.com/microsoft/vscode/issues/203188](https://github.com/microsoft/vscode/issues/203188)

## Summary

Team-authored issue (MEMBER) reporting confusing stash viewer behavior when a stash is popped externally. An open community PR (#288616) addresses the underlying stash identity problem. Keeping open per team-author policy.

## Change summary

When a stash is popped externally while its diff editor is open, the editor flips the diff sides (showing a deletion instead of an addition), which is confusing.

## Best solution

Review and land PR https://github.com/microsoft/vscode/pull/288616 which switches stash editor identity from index-based to hash-based, addressing both this issue and the related misdirection bug https://github.com/microsoft/vscode/issues/290509. An additional improvement would be to close or invalidate the stash diff editor when the stash no longer exists.
