# Issue #274816 — Terminal keeps scrolling up when typing, leaving a mostly blank panel.

[https://github.com/microsoft/vscode/issues/274816](https://github.com/microsoft/vscode/issues/274816)

## Summary

Confirmed terminal scroll/resize bug on Windows (Git Bash/pwsh) when maximizing the panel — the terminal viewport doesn't properly reflow to the new height, causing unwanted scroll-to-bottom on input. No fix found on current main; keeping open in Backlog.

## Change summary

After maximizing the terminal panel, typing any key scrolls history up and leaves blank space below, because the terminal viewport doesn't properly account for the enlarged panel height.

## Best solution

The terminal resize/reflow logic (likely in the xterm terminal adapter or the panel layout change handler) needs to correctly update the terminal's viewport rows when the panel is maximized, so that subsequent input doesn't trigger a scroll-to-bottom against a stale smaller viewport size. The fix likely involves ensuring the terminal dimensions are fully recalculated after `toggleMaximizedPanel`, particularly on Windows with conpty. May involve xterm.js reflow behavior or the `scrollToBottom` call in `terminalInstance.ts` line 1395.
