# Issue #266750 — "inherit" instructions in profiles

[https://github.com/microsoft/vscode/issues/266750](https://github.com/microsoft/vscode/issues/266750)

## Summary

The underlying infrastructure for prompts inheritance exists (ProfileResourceType.Prompts and useDefaultFlags.prompts), but the Profiles editor UI does not expose it — the resource type list at line 252 omits Prompts. This is a valid gap that needs UI work to surface the option.

## Change summary

Allow users to inherit/share instruction files (prompts) across profiles via the 'use default profile' toggle in the Profiles editor UI.

## Best solution

Add `ProfileResourceType.Prompts` to the resource types array in `src/vs/workbench/contrib/userDataProfile/browser/userDataProfilesEditorModel.ts` (line ~257), add a title string in `getResourceTypeTitle`, and wire up copy flags — the backend already handles `useDefaultFlags.prompts`.
