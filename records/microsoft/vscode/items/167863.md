# Issue #167863 — Column number is incorrect when using tabs.

[https://github.com/microsoft/vscode/issues/167863](https://github.com/microsoft/vscode/issues/167863)

## Summary

Valid feature request in Backlog with 18 upvotes. The status bar column display still expands tabs via `CursorColumns.toStatusbarColumn`; no option for a raw character-offset mode exists on current main.

## Change summary

Add an option to show the 1-based character offset (counting each tab as 1) in the status bar instead of the visual column.

## Best solution

Add a setting (e.g. `editor.statusBarColumn: 'visual' | 'character'`) that controls whether the status bar reports the render column or the raw character position. The change would touch `src/vs/workbench/browser/parts/editor/editorStatus.ts` (where the status bar text is built) and possibly `src/vs/editor/common/core/cursorColumns.ts`.
