# Issue #296785 — When switching tabs, it starts typing spaces

[https://github.com/microsoft/vscode/issues/296785](https://github.com/microsoft/vscode/issues/296785)

## Summary

This is an upstream Electron + Wayland key-handling bug affecting Fedora 43 with KDE (and GNOME via mutter). The workaround is disabling the virtual keyboard in KDE settings. VS Code cannot fix this in its own codebase; the fix belongs in Electron or the compositor.

## Change summary

When switching tabs on Wayland (Fedora 43, KDE/GNOME), keyup events are lost causing ghost key repeats (spaces/newlines).

## Best solution

Classify as `upstream` and track as an Electron/Chromium Wayland input bug. Issue https://github.com/microsoft/vscode/issues/300438 is the same report filed later; one should be duped into the other. An upstream Electron bug should be filed (or linked) tracking the fix in Chromium's Wayland key handling. The KDE virtual-keyboard workaround should be documented in the meantime.
