## Summary

Team-authored bug (MEMBER) — must stay open. The agent-sessions welcome view shows an indefinite loading skeleton when offline because the session model's `resolved` event never fires without network connectivity, leaving the skeleton visible forever.

## Change summary

The agent-sessions welcome view skeleton loading indicator hangs indefinitely when there is no internet connection.

## Best solution

Add a timeout or error-handling path in the welcome view's session loading flow so that when the session model fails to resolve (e.g., no network), the skeleton is replaced with an appropriate offline/empty state. The fix likely lives in `src/vs/workbench/contrib/welcomeAgentSessions/browser/agentSessionsWelcome.ts` around the `onDidResolve` listener and the `model.resolved` check. The `b83d48d126a` commit ("Use cache for welcome view") partially addresses this by using cached data, but doesn't handle the truly-offline-no-cache scenario.
