## Change summary

Include chat history as additional context when generating commit messages via Copilot to produce more meaningful messages that capture the 'why' behind changes.

## Best solution

This likely belongs in the Copilot extension (which owns the `generateCommitMessageCommand` implementation) rather than VS Code core. The extension would need to pull recent chat turns — either from the current session or since branch checkout — and include them in the prompt context. If a VS Code API surface is needed to expose chat history to the SCM commit-message flow, that would be a core contribution.
