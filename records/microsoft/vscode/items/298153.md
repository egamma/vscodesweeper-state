# Issue #298153 — Unexpected Focus Shift To Other Chat's AskQuestions Area

[https://github.com/microsoft/vscode/issues/298153](https://github.com/microsoft/vscode/issues/298153)

## Summary

The focus-stealing behavior was fixed in commit 3ef7eb46a8b (shipped in 1.125.0). The carousel now only auto-focuses when the user's focus is already within that specific chat widget, preventing one chat's askQuestions from stealing focus from another chat or any other VS Code UI.

## Change summary

When a chat's askQuestions dialog appears, it steals focus from whatever the user is currently doing — including typing an answer in a different chat's askQuestions area.

## Best solution

Keep the shipped fix in commit 3ef7eb46a8b (1.125.0). The `shouldAutoFocus` guard now checks `dom.isAncestorOfActiveElement(widget.domNode)` so only the actively-focused chat widget's carousel auto-focuses. No further change needed.
