# Issue #148974 — [Feature Request] Better alignment for file nesting

[https://github.com/microsoft/vscode/issues/148974](https://github.com/microsoft/vscode/issues/148974)

## Summary

Valid feature request with strong community support (66 upvotes). The alignment problem when icon themes use `hidesExplorerArrows: true` remains on current main — nested file parents are indented past their siblings due to the twistie arrow space. No fix has landed; keeping open in the Backlog.

## Change summary

File nesting parents are indented past their siblings when the icon theme hides explorer arrows, making it hard to visually parse which files are at the same directory level.

## Best solution

Rework the twistie/indentation logic in the explorer tree so that nesting parents align with their siblings even when the icon theme sets `hidesExplorerArrows: true`. The relevant code is in `src/vs/workbench/contrib/files/browser/views/explorerViewer.ts` (around the `force-twistie` and `themeIsUnhappyWithNesting` logic) and potentially `src/vs/base/browser/ui/tree/abstractTree.ts`. A configuration option (similar to Visual Studio's behavior) to control nesting arrow placement could satisfy the request without breaking themes that show arrows.
