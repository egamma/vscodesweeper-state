# Issue #230930 — Modified image attachments don't update the thumbnail/hover

[https://github.com/microsoft/vscode/issues/230930](https://github.com/microsoft/vscode/issues/230930)

## Summary

Still reproduces on main: the file watcher for chat image attachments only handles deletion, not modification — so editing and saving an attached image never refreshes the thumbnail or hover preview.

## Change summary

Modifying an attached image file does not update the chat attachment thumbnail or hover preview.

## Best solution

Extend `_watchAttachment` in `chatAttachmentModel.ts` to handle `FileChangeType.UPDATED` in addition to `DELETED` — on update, re-read the image bytes (via `asImageVariableEntry` or `resolveImageEditorAttachContext`), invalidate the thumbnail cache entry, and fire an `onDidChange` event with the attachment in the `updated` array so the widget re-renders.
