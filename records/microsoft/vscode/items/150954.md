# Issue #150954 — Double dialog when installing new language pack

[https://github.com/microsoft/vscode/issues/150954](https://github.com/microsoft/vscode/issues/150954)

## Summary

Still reproduces on main: installing a language pack via 'Configure Display Language' triggers both a modal restart dialog (from setLocale) and a notification prompt (from the onDidInstallExtensions listener), because nothing suppresses the notification when the install originates from the setLocale flow.

## Change summary

When installing a new language pack through the Configure Display Language command, both a modal dialog and a notification prompt appear asking the user to restart.

## Best solution

Suppress the notification in onDidInstallExtension when the install was triggered by the setLocale flow — e.g. pass context metadata through the install call so the event listener knows to skip the prompt, or track an in-progress locale-change flag in the contribution.
