# Issue #127441 — Git - incorrectly highlight changed lines

[https://github.com/microsoft/vscode/issues/127441](https://github.com/microsoft/vscode/issues/127441)

## Summary

Still reproduces on main: the git extension's quick diff provider fetches blob content via `git show --textconv` which returns data in git's internal encoding, but the editor renders the file in the `working-tree-encoding`. This encoding mismatch produces phantom changed-line decorations. No fix has landed.

## Change summary

Git gutter decorations incorrectly show changed lines when `working-tree-encoding` is set in `.gitattributes`, because the quick diff original-content fetch ignores the encoding attribute.

## Best solution

The `buffer()` method in `extensions/git/src/git.ts` (line ~1571) returns raw bytes from `git show --textconv`. When `working-tree-encoding` is configured, the quick diff pipeline needs to re-encode (or decode) those bytes to match the working-tree encoding before VS Code diffs them against the editor content. The fix likely involves reading the `.gitattributes` `working-tree-encoding` value for the file and transcoding the blob content in `GitFileSystemProvider.readFile()` or in `Repository.buffer()`.
