# Issue #71091 — [html] bracket matching in strings

[https://github.com/microsoft/vscode/issues/71091](https://github.com/microsoft/vscode/issues/71091)

## Summary

Still present on main. The tokenTypes override (`meta.tag string.quoted` → `other`) in extensions/html/package.json intentionally breaks bracket matching in attribute values to enable quick suggest (per #46145). The trade-off remains unresolved.

## Change summary

Bracket matching incorrectly treats `<` and `>` inside HTML attribute value strings as tag brackets, breaking matching and smart select.

## Best solution

Decouple the quick-suggest trigger from the bracket-matching token classification so that attribute value strings can be marked as 'other' for completion purposes without affecting bracket matching. This likely requires changes to how the editor's bracket matcher consults token types, or a more granular tokenTypes mapping in extensions/html/package.json.
