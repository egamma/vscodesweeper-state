# Issue #71091 — [html] bracket matching in strings

[https://github.com/microsoft/vscode/issues/71091](https://github.com/microsoft/vscode/issues/71091)

## Summary

Still reproduces on main: HTML attribute values are explicitly overridden to StandardTokenType.Other (to enable quick suggest per #46145), so bracket matching does not skip `<`/`>` inside quoted attribute strings. The trade-off remains unresolved.

## Change summary

Angle brackets inside HTML attribute value strings (e.g. `<=`, `>`) are incorrectly matched as tag brackets, breaking bracket highlighting and smart select.

## Best solution

Implement a more granular solution that preserves quick-suggest in attribute values while still suppressing bracket matching — e.g. a new token type or a bracket-matching-specific override in `extensions/html/package.json`. The fix would touch `extensions/html/package.json` (tokenTypes) and possibly the bracket matching infrastructure to support finer-grained token classification.
