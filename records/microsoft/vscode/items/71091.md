# Issue #71091 — [html] bracket matching in strings

[https://github.com/microsoft/vscode/issues/71091](https://github.com/microsoft/vscode/issues/71091)

## Summary

Still reproduces on main: the `tokenTypes` override marking `meta.tag string.quoted` as `other` (added for quick suggest, per #46145) causes bracket matching to misinterpret `<`/`>` inside HTML attribute values as tag delimiters. No fix has been applied.

## Change summary

Bracket matching incorrectly treats `<` and `>` inside HTML attribute strings as tag open/close brackets.

## Best solution

The `tokenTypes` override in `extensions/html/package.json` needs a more nuanced approach that preserves quick suggest in attribute values while still suppressing bracket matching for `<`/`>` inside quoted strings. This likely requires changes to how the bracket matching engine respects token types, or a different tokenType strategy for the HTML grammar.
