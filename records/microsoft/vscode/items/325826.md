# Issue #325826 — Auto-approve in Copilot multi-chat is session-scoped and applies across chats

[https://github.com/microsoft/vscode/issues/325826](https://github.com/microsoft/vscode/issues/325826)

## Summary

Team-authored bug with a clear protocol-level root cause and an upstream fix already filed; keeping open per hard keep-open rules for MEMBER-authored issues.

## Change summary

Auto-approve/permission-mode config is session-scoped in multi-chat, so enabling it in one chat unintentionally grants it to all chats in the same session.

## Best solution

Land the per-chat config concept (ChatConfig) in the agent-host-protocol (microsoft/agent-host-protocol#335), then flag `permissionMode`/`autoApprove` as `chatScoped` on the VS Code side and route the picker to the active chat channel.
