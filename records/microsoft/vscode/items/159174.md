[https://github.com/microsoft/vscode/issues/159174](https://github.com/microsoft/vscode/issues/159174)

## Summary

Multi-part accessibility feature request for the find widget (moveable position, bigger size, persistent tree find state). None of these have been implemented on current main. The requests are reasonable but bundled and have no community traction (0 upvotes). Keeping open as it's in Backlog with valid accessibility motivation.

## Change summary

Allow the editor find widget to be moved/centered, made bigger independently of zoom, and preserve tree find filter state across sessions.

## Best solution

The moveable find widget request (point 1) is the most actionable and distinct item. The sizing request partially overlaps with https://github.com/microsoft/vscode/issues/101064 (199 upvotes). Consider splitting into separate tracked issues: one for find widget repositioning (editor-find), one for persistent tree find state (tree-widget). The sizing concern could consolidate into #101064.
