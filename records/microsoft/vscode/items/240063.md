# Issue #240063 — Please add HTTPS support for serve-web

[https://github.com/microsoft/vscode/issues/240063](https://github.com/microsoft/vscode/issues/240063)

## Summary

Valid feature request — serve-web has no HTTPS support on current main. The server creates only plain HTTP connections with no TLS/certificate options. Keeping open as a low-traction feature request in Backlog.

## Change summary

Request to add HTTPS support (with custom/temporary certificates) to the `serve-web` command so the web UI can be accessed securely over a LAN.

## Best solution

Add optional `--cert` and `--cert-key` flags to `serve-web` (similar to what code-server provides) and use `https.createServer` when they are supplied, allowing secure-context APIs like `crypto.randomUUID` to work in browsers on LAN access.
