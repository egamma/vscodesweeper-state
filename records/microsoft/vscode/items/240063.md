# Issue #240063 — Please add HTTPS support for serve-web

[https://github.com/microsoft/vscode/issues/240063](https://github.com/microsoft/vscode/issues/240063)

## Summary

Valid feature request — serve-web has no HTTPS support on current main. The server only creates plain HTTP listeners with no --cert/--key/--https options. The referenced issue #240334 was a different bug (JS runtime error) that has since been fixed, but the underlying need for HTTPS (secure context APIs on non-localhost) remains unaddressed.

## Change summary

Add HTTPS support (custom or self-signed certificates) to the `code serve-web` server so browsers treat non-localhost connections as secure contexts.

## Best solution

Implement --cert and --key CLI flags in `src/vs/server/node/remoteExtensionHostAgentServer.ts` (the `createServer` path) to allow `code serve-web` to bind an HTTPS listener with user-supplied certificates, similar to how other dev servers handle this.
