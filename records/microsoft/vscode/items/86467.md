# Issue #86467 — Trailing comma-aware move line up/down

[https://github.com/microsoft/vscode/issues/86467](https://github.com/microsoft/vscode/issues/86467)

## Summary

Valid feature request with 28 upvotes, promoted to Backlog. No implementation exists on main — move-line operations in src/vs/editor/contrib/linesOperations/ have no comma-adjustment logic. Still a real gap.

## Change summary

Moving a line up/down in JSON (or other no-trailing-comma languages) should automatically adjust commas so the file remains valid.

## Best solution

Implement comma-aware adjustment in the move-lines command (src/vs/editor/contrib/linesOperations/), likely via a language-configuration hook or a new provider API (as suggested in comment 4). Alternatively, a dedicated MoveLinesProvider extension API could let language extensions handle post-move fixups including comma management.
