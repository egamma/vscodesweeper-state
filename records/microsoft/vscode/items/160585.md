# Issue #160585 — [html] Extensions using htmlLanguageParticipants don't activate vscode.html-language-features

[https://github.com/microsoft/vscode/issues/160585](https://github.com/microsoft/vscode/issues/160585)

## Summary

Still an open gap on main: html-language-features only activates on `onLanguage:html` and `onLanguage:handlebars`, so third-party extensions contributing `htmlLanguageParticipants` must manually activate it. The maintainer-proposed fix (auto-activate on participant languages) was never implemented.

## Change summary

Extensions contributing `htmlLanguageParticipants` must manually activate `vscode.html-language-features` because no automatic activation is triggered for participant languages.

## Best solution

Auto-generate activation events for languages declared in `htmlLanguageParticipants` contributions (either at extension scan time in the extension host, or by having `html-language-features` activate on `onStartupFinished` and listen for participant language documents). The fix would touch `extensions/html-language-features/package.json` and possibly `extensions/html-language-features/client/src/htmlClient.ts` or the extension host activation logic.
