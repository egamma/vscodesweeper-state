# Issue #235765 — Missing support for asymmetric visibility when using PHP constructor property promotion

[https://github.com/microsoft/vscode/issues/235765](https://github.com/microsoft/vscode/issues/235765)

## Summary

Valid grammar gap — PHP 8.4 asymmetric visibility in constructor property promotion isn't recognized. An upstream PR exists but hasn't been merged and pulled into VS Code yet. The issue already carries correct labels (`grammar`, `php`, `help wanted`) and milestone (`Backlog`); keeping open until the upstream fix lands.

## Change summary

PHP 8.4 asymmetric visibility modifiers (e.g. `private(set)`) in constructor property promotion are not tokenized as `storage.modifier` by the PHP grammar.

## Best solution

Merge upstream PR https://github.com/KapitanOczywisty/language-php/pull/28 in the `language-php` fork, then pull the updated grammar into `extensions/php/syntaxes/php.tmLanguage.json` via the regular grammar-update process (`npm run update-grammars` or equivalent).
