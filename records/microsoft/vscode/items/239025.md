# Issue #239025 — Delay to scroll to bottom when auto-scroll is enabled

[https://github.com/microsoft/vscode/issues/239025](https://github.com/microsoft/vscode/issues/239025)

## Summary

Team-filed bug that remains open — the auto-scroll logic in the output view triggers revealLastLine only on model content changes, so when filter categories are re-enabled (bulk content appears), there's a visible delay before the scroll fires. Still present on main.

## Change summary

When auto-scroll is enabled in the Output panel and a filter category is re-checked, there's a noticeable delay before the view scrolls to the bottom.

## Best solution

In `outputView.ts`, when the output channel or filter state changes such that new content becomes visible, trigger `revealLastLine()` synchronously on the filter-change path (not only on `onDidChangeModelContent`). The fix likely belongs in the filter-change handler or in the `setInput` flow that fires when categories are toggled.
