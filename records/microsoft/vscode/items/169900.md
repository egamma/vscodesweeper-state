# Issue #169900 — [scss] Display a wrong selector preview when using sibling combinators in SCSS, Less

[https://github.com/microsoft/vscode/issues/169900](https://github.com/microsoft/vscode/issues/169900)

## Summary

Still present on main. The bug lives in the upstream vscode-css-languageservice selectorPrinting.ts, which hasn't received a fix for this case. The issue is labeled help-wanted and the fix location was identified by a maintainer.

## Change summary

SCSS/Less selector preview incorrectly renders sibling combinators (`+`/`~`) as parent-child relationships when combined with the `&` parent selector in nested rules.

## Best solution

Fix in microsoft/vscode-css-languageservice `src/services/selectorPrinting.ts` — the `toElement` function's `SelectorCombinator` branch with `cloneWithParent` doesn't preserve sibling relationships from the parent selector. A contributor volunteered but no PR was opened; the help-wanted label is appropriate.
