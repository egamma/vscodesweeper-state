# Issue #169900 — [scss] Display a wrong selector preview when using sibling combinators in SCSS, Less

[https://github.com/microsoft/vscode/issues/169900](https://github.com/microsoft/vscode/issues/169900)

## Summary

Still reproduces on current main: the selector preview incorrectly renders the HTML element hierarchy when sibling combinators (`+`/`~`) interact with SCSS/Less parent references (`&`). The bug lives in vscode-css-languageservice's selectorPrinting.ts, confirmed unpatched. The issue already carries `help wanted` and a maintainer directed contributors to the right files.

## Change summary

Selector hover preview shows wrong element hierarchy when SCSS/Less nesting uses `&` inside a ruleset whose parent selector contains sibling combinators (`+` or `~`).

## Best solution

Fix `SelectorElementBuilder.processSelector` in https://github.com/microsoft/vscode-css-languageservice `src/services/selectorPrinting.ts` to correctly reconstruct the element tree when `toElement` clones a parent that was assembled via sibling combinators. Add test cases in `src/test/css/selectorPrinting.test.ts` for `.foo + .bar { .baz & { } }` and the `~` equivalent.
