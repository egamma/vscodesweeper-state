# Issue #309543 — Custom agents should be allowed to exceed the current cost tier

[https://github.com/microsoft/vscode/issues/309543](https://github.com/microsoft/vscode/issues/309543)

## Summary

Team-authored feature request (MEMBER) for allowing custom subagents to exceed the invoking agent's cost tier when the user has explicitly configured a higher-cost model. Keeping open per hard keep-open rule for team-authored issues.

## Change summary

Custom subagents with a user-specified model that exceeds the caller's cost tier are currently blocked; they should be allowed to run (possibly with user approval).

## Best solution

Allow custom agent definitions to override the cost-tier ceiling when the model is explicitly user-configured, gated behind user consent or a trust prompt. The fix likely lives in the chat subagent invocation path where cost-tier validation occurs.
