# Issue #215972 — Unable to scroll text in hover dialogs

[https://github.com/microsoft/vscode/issues/215972](https://github.com/microsoft/vscode/issues/215972)

## Summary

The hover widget has scrollbar infrastructure but terminal tab hovers can still clip content when tall; an open PR (#312646) is actively addressing this. Keeping open pending that PR's review.

## Change summary

Terminal tab hover dialogs clip content when it overflows the visible area, with no way to scroll to hidden elements.

## Best solution

Review and land https://github.com/microsoft/vscode/pull/312646 which enables scrolling in the workbench hover widget when content overflows, or ensure the terminal hover properly uses the existing `maxHeightRatio` + scrollbar infrastructure in `src/vs/platform/hover/browser/hoverWidget.ts`.
