# Issue #296319 — Persistent suggestions about already-committed changes from different branches

[https://github.com/microsoft/vscode/issues/296319](https://github.com/microsoft/vscode/issues/296319)

## Summary

Valid bug: chat editing sessions persist their pending-changes state (Keep/Undo UI) across git branch switches, with no mechanism to clear or invalidate stale session state. The editing service has no branch-awareness. Keeping open — the reporter resolved it manually but the underlying lifecycle gap remains.

## Change summary

Chat editing session shows Keep/Undo prompts for changes that were already committed on a different branch, persisting across restarts and branch switches.

## Best solution

The chat editing service (src/vs/workbench/contrib/chat/common/editing/chatEditingService.ts) should listen for git branch-change events and either auto-dismiss or warn about pending edits that no longer apply to the current working tree. Related to #292211 but distinct — that tracks data loss from accepting stale sessions; this is about the stale UI persisting at all.
