# Issue #306383 — session unrecoverable from "image dimensions exceed max size" error

[https://github.com/microsoft/vscode/issues/306383](https://github.com/microsoft/vscode/issues/306383)

## Summary

The root cause (images from tool calls not being resized) was fixed on main before filing (commit ba560e4eca1, March 26), but the broader concern — a session becoming permanently unrecoverable after any API error — appears unaddressed and is still a valid resilience bug. A team member already engaged.

## Change summary

A single API error (oversized image in a tool-call screenshot) makes the entire chat session permanently unrecoverable, losing all prior work.

## Best solution

The image-resize root cause is fixed (ba560e4eca1). The remaining work is ensuring session resilience: when an API request fails due to an invalid image or similar transient error, the session should allow retrying or continuing without the offending content, rather than becoming permanently stuck. This likely belongs in the copilot extension's error-handling/session-recovery logic.
