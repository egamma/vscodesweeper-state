# Issue #312320 — Switching sessions looses attached context in "New sesssion" composer

[https://github.com/microsoft/vscode/issues/312320](https://github.com/microsoft/vscode/issues/312320)

## Summary

Valid UX improvement request: the chat view pane does not persist draft input (text or attachments) for the "New Session" composer when the user temporarily switches to another session. A team member has engaged and it's already labeled under-discussion — keeping it open for the product discussion to conclude.

## Change summary

Switching away from the "New Session" composer and back discards the draft message and attachments, forcing the user to re-compose from scratch.

## Best solution

Persist the draft input state (text and attachments) for the new-session composer across session switches in the chat view pane. The likely implementation area is `src/vs/workbench/contrib/chat/browser/widgetHosts/viewPane/chatViewPane.ts` and `src/vs/workbench/contrib/chat/browser/widget/input/chatInputPart.ts`, saving/restoring the composer state when navigating between sessions. A team member is already engaged, so the design direction should come from that discussion.
