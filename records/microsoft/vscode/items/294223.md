# Issue #294223 — Markdown code blocks don't render correctly in Test Results pane

[https://github.com/microsoft/vscode/issues/294223](https://github.com/microsoft/vscode/issues/294223)

## Summary

Still reproduces on main: MarkdownTestMessagePeek lacks an onDidContentSizeChange event, so code blocks that render asynchronously get their height clipped until a manual resize triggers re-layout.

## Change summary

Markdown code blocks in the Test Results pane are truncated on initial display because the layout height is captured before the content fully renders.

## Best solution

Add a ResizeObserver (or MutationObserver) on the rendered markdown element in MarkdownTestMessagePeek and expose it as onDidContentSizeChange, so the call-stack widget re-measures after the code block finishes painting — similar to how DiffContentProvider and PlainTextMessagePeek already emit content-size changes.
