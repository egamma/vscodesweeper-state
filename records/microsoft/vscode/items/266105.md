Team-authored feature request on the 'On Deck' milestone — keeping open per policy. The request is to change MCP server configurations from inline CLI args to a named options map, enabling safe version upgrades and cleaner variable handling.

## Change summary

MCP server configurations should use a named options map (config object) instead of raw CLI args, so servers can evolve their parameters across versions without breaking user setups.

## Best solution

Implement the config-map approach in the MCP server settings schema so that registry-defined named inputs replace positional CLI arguments, allowing graceful server upgrades and standard variable substitution. Work lives in the chat-mcp-management area.
