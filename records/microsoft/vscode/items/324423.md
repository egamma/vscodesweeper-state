# Issue #324423 — Copilot changes diff do not honor word wrap

[https://github.com/microsoft/vscode/issues/324423](https://github.com/microsoft/vscode/issues/324423)

## Summary

Confirmed on main: the chat editing inline diff renders deleted (red) lines without word wrap because it passes an empty lineBreakData array to the LineSource constructor, unlike the standard diff editor which computes proper break offsets.

## Change summary

Copilot agent edit diffs show deleted (red) lines extending horizontally without word wrap, while added (green) lines respect the editor's word wrap setting.

## Best solution

In `src/vs/workbench/contrib/chat/browser/chatEditing/chatEditingCodeEditorIntegration.ts` around line 367, compute word-wrap line break data for the original lines (similar to how `src/vs/editor/browser/widget/diffEditor/components/diffEditorViewZones/diffEditorViewZones.ts` line 220 does) and pass it to the `LineSource` constructor instead of `[]`.
