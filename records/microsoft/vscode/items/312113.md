# Issue #312113 — Remote Tunnels: 401 Unauthorized with Microsoft Account - "Anonymous has scopes []"

[https://github.com/microsoft/vscode/issues/312113](https://github.com/microsoft/vscode/issues/312113)

## Summary

Still a valid open issue for the CLI path. The workbench UI removed Microsoft auth as an option in commit a73c604649f (shipped 1.121.0), acknowledging Microsoft auth is not supported end-to-end for tunnels. However, the CLI (`code tunnel user login --provider microsoft`) still accepts Microsoft auth, and the underlying service-side token/scope issue remains unresolved. A team MEMBER has already assigned this to relevant engineers.

## Change summary

The `code tunnel` CLI returns 401 Unauthorized when authenticating with a Microsoft Account because the tunnel API receives a token with empty scopes, treating the user as Anonymous.

## Best solution

Either fix the Microsoft Account token flow end-to-end in the dev tunnels service so scopes propagate correctly, or align the CLI with the workbench fix by removing/deprecating `--provider microsoft` for tunnel login in `cli/src/auth.rs` and documenting GitHub as the only supported auth provider for tunnels.
