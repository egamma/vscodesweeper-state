# Issue #274918 — Instruction ApplyTo Not Applied - Open Editor Files Not Auto-Attached as Context

[https://github.com/microsoft/vscode/issues/274918](https://github.com/microsoft/vscode/issues/274918)

## Summary

Still reproduces on main: the `InstructionsContextComputer._getContext` method only considers explicitly attached variables, not open editor files. The `applyTo` pattern matching therefore cannot fire unless files are manually attached. Author is a MEMBER — keeping open for maintainer judgment.

## Change summary

Instruction files with `applyTo` patterns are not automatically applied based on open editor files; users must manually attach files for pattern matching to work.

## Best solution

Extend `InstructionsContextComputer._getContext` (or its caller `collect`) to also include URIs of open/visible editor tabs (via `IEditorService`) in the `files` set passed to `_matches`, so `applyTo` patterns can match against open files without manual attachment. This likely touches `src/vs/workbench/contrib/chat/common/promptSyntax/computeAutomaticInstructions.ts` and possibly the chat request variable resolution pipeline.
