# Issue #243455 — Change default keyboard shortcut for `editor.action.inlineSuggest.acceptNextWord` on Mac OS

[https://github.com/microsoft/vscode/issues/243455](https://github.com/microsoft/vscode/issues/243455)

## Summary

Valid feature request with 25 upvotes (exceeds the 20-vote threshold for backlog promotion). The keybinding is still `Cmd+Right` on current main, unchanged from the report. A team member engaged and solicited input — this needs a product decision.

## Change summary

Change macOS default keybinding for `editor.action.inlineSuggest.acceptNextWord` from Cmd+Right to Option+Right, freeing Cmd+Right for `acceptNextLine`.

## Best solution

A team member should decide the macOS keybinding for partial accept commands (`acceptNextWord` and `acceptNextLine`). The current `CtrlCmd+Right` binding in `src/vs/editor/contrib/inlineCompletions/browser/controller/commands.ts` conflicts with the system 'jump to end of line' shortcut on macOS; a mac-specific `Alt+Right` override for acceptNextWord (and optionally assigning `Cmd+Right` to acceptNextLine) would resolve the conflict.
