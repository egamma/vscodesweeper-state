# Issue #309032 — Integrated terminal on Windows can start with stale PATH and fail to resolve npm while standalone PowerShell works

[https://github.com/microsoft/vscode/issues/309032](https://github.com/microsoft/vscode/issues/309032)

## Summary

Well-diagnosed bug: persistent terminal sessions on Windows can revive with a stale PATH that's missing entries added after the session was first created (e.g. after installing Node.js). Disabling persistent sessions is a confirmed workaround. A team member has asked a follow-up question but no response yet. Routing to the terminal-persistence area.

## Change summary

Revived persistent terminal sessions on Windows can start with a stale PATH that lacks entries added since the session was originally created, causing commands like npm to fail.

## Best solution

When reviving a persistent terminal session, the environment (especially PATH) should be refreshed from the current system/user environment variables rather than reusing the environment snapshot from when the session was first created. The fix likely belongs in the terminal persistence/revive path in `src/vs/workbench/contrib/terminal/browser/terminalProcessManager.ts` or the backend revive logic.
