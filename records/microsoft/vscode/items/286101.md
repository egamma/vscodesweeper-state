# Issue #286101 — Close Other Repositories on a top-level worktree node closes all

[https://github.com/microsoft/vscode/issues/286101](https://github.com/microsoft/vscode/issues/286101)

## Summary

Team-authored issue (chrmarti) with milestone On Deck — keeping open. The bug is source-reproducible: closeOtherRepositories in extensions/git/src/commands.ts closes every non-selected repository with no worktree relationship awareness, so invoking it on a worktree node also closes the parent repo (and vice versa). Related issue #313526 reports the inverse problem (worktrees reappear after being closed).

## Change summary

"Close Other Repositories" on a worktree node also closes its parent repository, breaking the worktree.

## Best solution

In `extensions/git/src/commands.ts` around line 1287, the close loop should skip repositories that are worktree-related to the selected ones (parent or sibling worktrees). The git Model likely needs to expose the worktree↔parent relationship so the command can filter correctly. Issue https://github.com/microsoft/vscode/issues/313526 is the complementary bug and both should be addressed together.
