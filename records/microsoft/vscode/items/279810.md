# Issue #279810 — Inline completions are too aggressive on empty lines

[https://github.com/microsoft/vscode/issues/279810](https://github.com/microsoft/vscode/issues/279810)

## Summary

Valid UX bug from a core team contributor — inline completions re-appear too eagerly on empty lines after dismissal, and trigger too quickly. No targeted fix found on current main; the debounce/dismiss logic in the inline completions model has not been adjusted for this specific empty-line scenario.

## Change summary

Inline completions re-trigger too quickly on empty lines and ignore repeated dismissals, causing annoyance during normal editing.

## Best solution

Adjust the inline completions triggering logic to either increase the debounce delay on empty lines, or respect dismissal state so completions don't re-trigger on the same empty line without a meaningful edit. The relevant logic lives in `src/vs/editor/contrib/inlineCompletions/browser/model/inlineCompletionsModel.ts` and the source/debounce infrastructure.
