# Issue #279810 — Inline completions are too aggressive on empty lines

[https://github.com/microsoft/vscode/issues/279810](https://github.com/microsoft/vscode/issues/279810)

## Summary

Valid UX bug from a team contributor — inline completions reappear too eagerly on empty lines after dismissal. No fix found on current main targeting this specific behavior. Keeping open for the inline-completions area owner.

## Change summary

Inline completions show too aggressively on empty lines — they reappear quickly after dismissal and trigger too fast.

## Best solution

Add suppression logic so that after a user dismisses an inline completion on an empty line, subsequent triggers are delayed or suppressed for a cooldown period. The debounce/suppress logic in `src/vs/editor/contrib/inlineCompletions/browser/model/inlineCompletionsModel.ts` is the likely place to implement empty-line-aware throttling.
