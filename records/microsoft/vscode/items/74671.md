# Issue #74671 — Allow debugger contributions to specify filenames as well as languages to be default debugger

[https://github.com/microsoft/vscode/issues/74671](https://github.com/microsoft/vscode/issues/74671)

## Summary

Still unimplemented on main — the debugger contribution point only accepts a plain string array of language IDs, with no support for filename patterns or DocumentFilter-style objects. Low community signal (0 upvotes) but parked in Backlog by a maintainer and referenced as a prereq for the now-closed #146338.

## Change summary

Allow debugger contributions to specify filename glob patterns (not just language IDs) so the right debugger is selected when the active file matches a pattern like `**/pubspec.yaml` or `**/package.json`.

## Best solution

Extend the `languages` array in the `contributes.debuggers` schema (in `src/vs/workbench/contrib/debug/common/debugSchemas.ts`) and the debugger-selection logic to accept DocumentFilter-like objects with a `pattern` field alongside plain language strings. Low upvote count suggests this can stay in the backlog until there is more community demand or a related UX improvement revisits the debugger-selection flow.
