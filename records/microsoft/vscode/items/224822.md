[https://github.com/microsoft/vscode/issues/224822](https://github.com/microsoft/vscode/issues/224822)

## Summary

Team-authored feature request to stabilize the cursor line when code lenses are inserted, avoiding visual jumping. Kept open per hard keep-open rule (MEMBER author) and because it remains a valid, unresolved UX improvement in the editor core.

## Change summary

Keep the cursor line visually stable when code lenses appear or disappear, preventing the editor from jumping.

## Best solution

Implement a scroll-anchor mechanism specific to code lenses in the editor viewport layout so that the line under the cursor stays at the same vertical position when lenses are inserted/removed. The earlier generic PR (#221031) was rejected because it affected other elements; the fix needs to be scoped to code-lens insertions only.
