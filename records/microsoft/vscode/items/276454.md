# Issue #276454 — fetch fails with InvalidArgumentError when proxy is set to socks://... in settings

[https://github.com/microsoft/vscode/issues/276454](https://github.com/microsoft/vscode/issues/276454)

## Summary

Already correctly triaged as upstream by a team member. The root cause is that undici's fetch dispatcher does not support SOCKS proxy URLs, so VS Code's proxy-agent cannot route fetch calls through a SOCKS proxy even though the proxy-agent itself has SOCKS handling code. This is blocked on upstream undici work.

## Change summary

Node.js fetch fails with InvalidArgumentError when VS Code's http.proxy setting uses a socks:// URL, because undici does not support SOCKS protocol dispatchers.

## Best solution

Blocked on https://github.com/nodejs/undici/issues/2224 adding SOCKS proxy support to undici's ProxyAgent. Once undici ships that, @vscode/proxy-agent can wire it through. No actionable VS Code core change until then.
