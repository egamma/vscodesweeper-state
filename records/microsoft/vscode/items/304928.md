# Issue #304928 — Commit messages are getting corrupted

[https://github.com/microsoft/vscode/issues/304928](https://github.com/microsoft/vscode/issues/304928)

## Summary

The report describes commit message corruption when the Copilot agent sends long `git commit -m` commands via the terminal, likely caused by PTY column-width wrapping. The issue straddles the Copilot extension's `run_in_terminal` tool and VS Code's terminal PTY infrastructure, but the report lacks precise reproduction steps, version/OS details beyond the screenshot, and confirmation on whether this persists on the latest Insiders build.

## Change summary

Long git commit messages sent via the Copilot agent's `run_in_terminal` tool get corrupted due to PTY line-wrapping when the command exceeds the terminal's column width.

## Best solution

The agent's `run_in_terminal` tool (in the Copilot extension or the VS Code terminal agent tools layer at `src/vs/workbench/contrib/terminalContrib/chatAgentTools/`) should write long commit messages to a temp file and use `git commit -F <file>` instead of inlining the full message with `-m`. This avoids PTY wrapping entirely. The fix likely belongs in the execute strategy code or the Copilot extension's terminal tool invocation path. Needs reproduction confirmation first.
