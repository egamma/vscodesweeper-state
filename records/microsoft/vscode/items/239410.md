## Summary

Still reproducible on main: `normalizeRangeMapping` in diffEditorViewModel.ts calls `getLineMaxColumn` without validating the line number is within model bounds. The issue also carries the 'On Deck' milestone, which is a hard keep-open.

## Change summary

The diff editor's `normalizeRangeMapping` throws 'Illegal value for lineNumber' when the diff result contains stale line numbers from an async computation against an edited model.

## Best solution

Add bounds validation in `normalizeRangeMapping` (src/vs/editor/browser/widget/diffEditor/diffEditorViewModel.ts, around line 397) before calling `getLineMaxColumn`, guarding against stale line numbers that exceed the current model line count.
