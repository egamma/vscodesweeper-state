Confirmed bug with an open community PR (#310896) addressing it — keep open pending PR review and merge.

## Change summary

Renaming a file to a path where the first segment matches the file name (e.g. 'hello' → 'hello/world') silently fails instead of either succeeding or showing a validation error.
