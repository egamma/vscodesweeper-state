# Issue #316619 — The character U+2013 "–" could be confused with the ASCII character U+002d "-", which is more common in source code. Adjust settings doesn't work if item is near top of view

[https://github.com/microsoft/vscode/issues/316619](https://github.com/microsoft/vscode/issues/316619)

## Summary

Real bug: clicking 'adjust settings' in the unicode highlight hover fails when the highlighted character is near the top of the viewport. The hover positioning logic flips the widget below the line when there's insufficient space above, and this position-dependent interaction with the command link (which opens a quick pick) causes the hover to dismiss before the quick pick can appear. Not fixed on current main.

## Change summary

The 'adjust settings' command link in the unicode highlight hover doesn't work when the highlighted character is near the top of the editor viewport.

## Best solution

The hover's command-link click handler needs to prevent the hover from dismissing when it triggers a quick pick. The fix likely belongs in the hover widget's link-click handling (around `contentHoverWidgetWrapper.ts` or `contentHoverController.ts`) — similar to how `ec055d97395` prevented dismissal for quick-fix code action widgets, this needs equivalent protection for quick picks opened from hover command links.
