# Issue #171755 — Code lens is not accessible via screen reader

[https://github.com/microsoft/vscode/issues/171755](https://github.com/microsoft/vscode/issues/171755)

## Summary

Code lens accessibility gaps identified in the team discussion remain unaddressed on current main: no accessibility signal for lines with lenses, no next/prev lens navigation commands, and no screen-reader-friendly announcement of lens presence. The existing 'Show CodeLens Commands for Current Line' command helps but the discovery and navigation UX is still missing.

## Change summary

Screen reader users cannot discover or navigate to code lenses; there is no audio/accessibility signal when the cursor lands on a line with a code lens, and no next/prev code lens navigation command.

## Best solution

Implement (1) an accessibility signal when the cursor is on a line with a code lens, (2) next/prev code lens navigation commands scoped to accessibility mode, and (3) an accessible view or announcement for the lens content. The work lives in `src/vs/editor/contrib/codelens/browser/codelensController.ts` and the accessibility signal infrastructure.
