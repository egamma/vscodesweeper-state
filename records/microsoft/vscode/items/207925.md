# Issue #207925 — Git - Can't stage changing in parent repository

[https://github.com/microsoft/vscode/issues/207925](https://github.com/microsoft/vscode/issues/207925)

## Summary

Still reproduces on main: when staging a file in a parent repo that also belongs to a nested child repo, `runByRepository` resolves the URI to the child repo (deepest match), silently running `git add` against the wrong repository. The bug is in the URI-based repository lookup, which ignores the SCM group context the Resource came from.

## Change summary

Clicking '+' to stage a file in the parent repository's SCM group does nothing when the file also belongs to a nested child repository.

## Best solution

Fix `runByRepository` in `extensions/git/src/commands.ts` to use the repository that owns the `Resource`'s source-control group rather than resolving the repository from the URI alone. The `Resource` objects passed to `stage()` carry a `resourceGroupType` and come from a specific repository's SCM groups — that association should be preserved through to the `add` call.
