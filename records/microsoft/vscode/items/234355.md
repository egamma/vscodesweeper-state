# Issue #234355 — [Remote-SSH] Extension Host with Copilot Fails on VS Code 1.95.3 but Works on 1.93.0

[https://github.com/microsoft/vscode/issues/234355](https://github.com/microsoft/vscode/issues/234355)

## Summary

Already well-triaged as an upstream V8/Node.js issue with a working mitigation. The root cause — V8 heap setup OOM when RLIMIT_AS is constrained — lives in Node.js/V8, not VS Code. Keeping open as the upstream tracker; labels are correct.

## Change summary

Extension host crashes on HPC login nodes with address-space limits (RLIMIT_AS) because V8 heap setup allocates large virtual memory regions that exceed the configured 32 GB cap.

## Best solution

The fix belongs in Node.js/V8 (respecting RLIMIT_AS during heap/code-range reservation). VS Code's role is to track the upstream dependency. The current mitigation (raising RLIMIT_AS from 32 GB to 40 GB) works. No VS Code code change is needed.
