# Issue #144120 — Can't read deeply nested files in narrow explorer

[https://github.com/microsoft/vscode/issues/144120](https://github.com/microsoft/vscode/issues/144120)

## Summary

A 4-year-old feature request for smarter indentation handling when deeply nested files are visible in a narrow explorer. The existing `workbench.list.horizontalScrolling` setting addresses the core pain point (content cut off at depth), though the specific request — auto-shifting content when no shallow items are visible — remains unimplemented. Zero community upvotes; #149136 was closed as a duplicate of this issue, so it serves as the canonical tracker. Keeping open for the community process.

## Change summary

Auto-shift explorer tree contents leftward when no shallow-depth items are visible, so deeply nested file names are readable without horizontal scrolling.

## Best solution

The existing `workbench.list.horizontalScrolling` setting is the practical workaround. The requested auto-shift behavior would require tree-widget changes in `src/vs/base/browser/ui/tree/` to detect visible-depth range and adjust indentation dynamically — a nontrivial UX change with performance implications. If this gains community traction (20+ upvotes), it could be evaluated as a tree-widget enhancement; otherwise let the community process decide its fate.
