# Issue #313029 — Support for agent plugin repos without the `.git` preffix

[https://github.com/microsoft/vscode/issues/313029](https://github.com/microsoft/vscode/issues/313029)

## Summary

Team-authored issue — hard keep-open. The `.git` suffix requirement is still enforced on current `main` (line 1043 of pluginMarketplaceService.ts), so the bug is confirmed and valid. Azure DevOps URLs that omit `.git` are rejected, breaking plugin marketplace use on that platform.

## Change summary

Agent plugin marketplace rejects git URLs that don't end with `.git`, which breaks Azure DevOps repo URLs.

## Best solution

Relax or remove the `.endsWith('.git')` guard in `parsePluginSource` (src/vs/workbench/contrib/chat/common/plugins/pluginMarketplaceService.ts, ~line 1043) so that `url`-source plugins with valid git-hosting URLs that omit the `.git` suffix are accepted. The clone operation itself should handle URL normalization. Update the corresponding test in `pluginMarketplaceService.test.ts` (line ~910) which currently asserts `undefined` for URLs not ending in `.git`.
