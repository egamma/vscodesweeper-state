# Issue #104154 — Allow blocked environment variables through if they're included explicitly in terminal launch configs

[https://github.com/microsoft/vscode/issues/104154](https://github.com/microsoft/vscode/issues/104154)

## Summary

The original GOOGLE_API_KEY blocking was fixed in commit 1248ddadb8c (shipped 1.53). The broader request — allowing blocked env vars through when explicitly set in terminal config — is implemented by the current code ordering: sanitization runs first, then terminal profile env and shellLaunchConfig.env are merged on top, effectively re-adding any explicitly configured variable.

## Change summary

Allow environment variables that match the sanitization blocklist to pass through to the terminal when explicitly set in terminal launch configuration.

## Best solution

Close as implemented. The specific GOOGLE_API_KEY pattern was removed in 1248ddadb8c (1.53), and the code architecture in terminalEnvironment.ts already merges `terminal.integrated.env.*` and profile `env` settings after sanitization, so explicitly configured blocked vars are restored. The `remoteEnv` case belongs to the Dev Containers extension, not VS Code core.
