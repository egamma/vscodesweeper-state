# Issue #241870 — Disable shell env via `terminal.integrated.shellIntegration.environmentReporting` does apply to existing terminal

[https://github.com/microsoft/vscode/issues/241870](https://github.com/microsoft/vscode/issues/241870)

## Summary

Still reproduces on main: the `shellIntegrationEnvironmentReporting` setting is only read at terminal construction time (line 421 of terminalInstance.ts) and never re-evaluated on config change. The team discussion between jrieken and Tyriar on whether to fix this at the shell-script level or the API level is unresolved. Keeping open as debt.

## Change summary

Changing the `terminal.integrated.shellIntegration.environmentReporting` setting does not take effect in already-running terminals.

## Best solution

Either (a) listen for configuration changes in `terminalInstance.ts` and toggle the API-level acceptance of environment sequences at runtime, as jrieken suggested, or (b) update the setting description to note that a terminal restart is required, as Tyriar suggested. The team should decide which approach to take.
