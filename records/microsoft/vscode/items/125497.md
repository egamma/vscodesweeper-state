# Issue #125497 — Search editor isRegex argument

[https://github.com/microsoft/vscode/issues/125497](https://github.com/microsoft/vscode/issues/125497)

## Summary

The argument naming inconsistency across search commands (`isRegex` vs `isRegexp`, `query` vs `searchString`, `replace` vs `replaceString`) still exists on current main. Valid API polish request; keeping open.

## Change summary

Unify the argument names for regex, query, and replace across findInFiles, search editor, and editor find commands.

## Best solution

Normalize the command argument names: add legacy-name translation (like the existing `translateLegacyConfig` in `searchEditor.contribution.ts`) so old names keep working, but standardize on one canonical name per concept (`isRegex`, `query`, `replace`). Touch `src/vs/workbench/contrib/searchEditor/browser/searchEditor.contribution.ts`, `src/vs/workbench/contrib/search/browser/searchActionsFind.ts`, and `src/vs/editor/contrib/find/browser/findModel.ts`.
