# Issue #125497 — Search editor isRegex argument

[https://github.com/microsoft/vscode/issues/125497](https://github.com/microsoft/vscode/issues/125497)

## Summary

The argument naming inconsistencies across search commands still exist on current main — the search editor uses `isRegexp` while findInFiles uses `isRegex`, and `editor.actions.findWithArgs` uses `searchString`/`replaceString` instead of `query`/`replace`. The issue is milestoned On Deck and the inconsistencies remain unaddressed.

## Change summary

Harmonize the argument names (`isRegex`/`isRegexp`, `query`/`searchString`, `replace`/`replaceString`) across `workbench.action.findInFiles`, `search.action.openNewEditor`, and `editor.actions.findWithArgs` commands.

## Best solution

Standardize all three commands on one set of argument names (likely `isRegex`, `query`, `replace` to match the most common pattern), adding legacy-name translation (like the existing `translateLegacyConfig`) for backward compatibility. The search editor contribution already has a legacy translation layer that could serve as a model.
