# Issue #96115 — Moving a view into panel does not distribute size evenly

[https://github.com/microsoft/vscode/issues/96115](https://github.com/microsoft/vscode/issues/96115)

## Summary

Still present on main: moveViewsWithoutSaving calls addViews without redistributing sizes evenly among existing panes. The partial fix (expanding collapsed views on drop) shipped, but even distribution (problem 2 from sbatten's comment) was never implemented.

## Change summary

When a view is drag-dropped into the panel, existing views and the new view should be resized to share the space evenly, matching the drop feedback overlay.

## Best solution

After a view is added via drag-drop in viewPaneContainer.ts, trigger an even-size redistribution across all visible panes in the target container, similar to how editors call distributeSize on a three-way split.
