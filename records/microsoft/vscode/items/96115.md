# Issue #96115 — Moving a view into panel does not distribute size evenly

[https://github.com/microsoft/vscode/issues/96115](https://github.com/microsoft/vscode/issues/96115)

## Summary

The core sizing problem (views not distributing evenly on DnD move) remains unresolved on main. The partial fix b314697b4d0 only addressed collapsed views being expanded on drop; the even-distribution logic was never implemented.

## Change summary

Moving a view into the panel via drag-and-drop does not distribute sizes evenly despite the drop feedback suggesting it will.

## Best solution

When a view is added via DnD (`'dnd'` source in `moveViewsToContainer`), the target container should redistribute all pane sizes evenly (similar to how editors handle a third side-by-side split), likely in `onDidAddViewDescriptors` or `addPanes` in `viewPaneContainer.ts`.
