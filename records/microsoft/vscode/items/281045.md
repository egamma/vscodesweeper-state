# Issue #281045 — Built-in git blame editor decorations are not applied when diff editor is opened via third-party extension (e.g. Git Graph / GitLens)

[https://github.com/microsoft/vscode/issues/281045](https://github.com/microsoft/vscode/issues/281045)

## Summary

Confirmed bug: `isResourceSchemeSupported` in `extensions/git/src/blame.ts` (line 68) only accepts `file` and `git` URI schemes, so blame decorations are skipped for diff editors opened by third-party extensions that use custom URI schemes. Two open community PRs target this fix — keeping open for a maintainer to review/land one of them.

## Change summary

Built-in git blame decorations do not appear in diff editors opened by third-party extensions because the URI scheme check rejects non-`file`/`git` schemes.

## Best solution

Review and land one of the open PRs (https://github.com/microsoft/vscode/pull/287965 or https://github.com/microsoft/vscode/pull/305524). The fix likely involves relaxing `isResourceSchemeSupported` or resolving the underlying `file` path from third-party URIs before checking blame eligibility.
