# Issue #85898 — Incorrect scaling of native file picker

[https://github.com/microsoft/vscode/issues/85898](https://github.com/microsoft/vscode/issues/85898)

## Summary

This is a confirmed upstream Electron/Windows issue about native file picker dialogs using incorrect DPI scaling on multi-monitor setups with mixed DPI. The upstream Electron issue (electron/electron#26622) was closed as stale without a fix. VS Code is now on Electron 42.5.0 — whether the issue persists with recent Electron/Chromium versions is unclear, but the root cause remains in Electron's per-monitor DPI awareness handling for native dialogs.

## Change summary

Native file open/save dialogs display at wrong scale (too large or too small) on multi-monitor Windows setups with different DPI settings per monitor.

## Best solution

Re-open or file a new upstream issue against Electron's current major version (42.x) if the problem persists. VS Code has no practical workaround in its own code — the dialogs are rendered by the OS via Electron's `dialog.showOpenDialog`/`dialog.showSaveDialog`. A user-side workaround exists: setting the Code.exe compatibility High DPI scaling override to 'System (Enhanced)'. If modern Electron has fixed this, the issue can be closed as resolved upstream.
