# Issue #325919 — Agents renderer OOM: unbounded workspaceFileCount telemetry search materializes every file

[https://github.com/microsoft/vscode/issues/325919](https://github.com/microsoft/vscode/issues/325919)

## Summary

Confirmed on current main: `_computeWorkspaceFileCount` performs an unbounded `fileSearch` with no `maxResults`, materializing every matching URI in the renderer. For large workspaces (e.g. $HOME) this OOMs the renderer. No fix has landed.

## Change summary

Sending a message in the Agents window triggers an unbounded file search for telemetry that OOMs the renderer on large workspaces.

## Best solution

Add `maxResults` (e.g. 100 000) to the file search query in `_computeWorkspaceFileCount`, report a capped/bucketed count, and gate the entire collection on the telemetry level being sufficient. Fix the cache so repeated foreground sends don't re-scan.
