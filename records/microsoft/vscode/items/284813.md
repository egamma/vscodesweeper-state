# Issue #284813 — [vscode cli]Cant use serve-web and tunnel after updated snap package

[https://github.com/microsoft/vscode/issues/284813](https://github.com/microsoft/vscode/issues/284813)

## Summary

Confirmed regression on current main: the snap electron-launch script appends --ozone-platform=x11 after all arguments, which breaks CLI subcommands (serve-web, tunnel) that don't accept Electron flags. The fix from fae32c4c21f introduced this; no correction has landed yet.

## Change summary

Running `code serve-web` or `code tunnel` from the snap package fails because `--ozone-platform=x11` is appended as a trailing argument to the CLI binary, which rejects it as unexpected.

## Best solution

Fix the argument order in `resources/linux/snap/electron-launch` so `--ozone-platform=x11` is inserted between the binary (first arg) and the remaining arguments, e.g. `arg1="$1"; shift; exec "$arg1" --ozone-platform=x11 "$@"`. Alternatively, detect when the target is the CLI tunnel binary and skip the flag entirely.
