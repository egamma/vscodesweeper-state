# Issue #12588 — Extension proxy support

[https://github.com/microsoft/vscode/issues/12588](https://github.com/microsoft/vscode/issues/12588)

## Summary

The core request — transparent proxy support for extensions — was implemented via @vscode/proxy-agent which patches http/https/fetch/tls/net/WebSocket in the extension host. Extensions automatically inherit proxy settings without custom code. Remaining edge cases (http2, CLI) are tracked by separate, more specific issues.

## Change summary

Extensions need a way to make HTTP requests through the user's configured proxy without reimplementing proxy logic themselves.

## Best solution

Close as implemented: the @vscode/proxy-agent module in the extension host transparently routes all extension network calls through the configured proxy. Remaining gaps (http2.connect paths in #310124, CLI proxy in #29910) are tracked by focused issues.
