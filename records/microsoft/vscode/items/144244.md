## Summary

The incorrect hover tooltip for multiple `&` in LESS is an upstream bug in vscode-css-languageservice's SelectorPrinting logic, not in VS Code core. The issue is valid but the fix belongs in the upstream dependency.

## Change summary

LESS hover tooltip incorrectly shows `.link > .link` instead of `.link.link` when using `&&` (multiple parent selectors).
