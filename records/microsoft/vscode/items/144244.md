# Issue #144244 — [less] Build-in LESS syntax hint error, multiple '&'

[https://github.com/microsoft/vscode/issues/144244](https://github.com/microsoft/vscode/issues/144244)

## Summary

Still reproduces on main — the selector hover tooltip incorrectly renders `&&` (multiple parent selectors in LESS) as a descendant relationship instead of concatenation. The bug is in the upstream vscode-css-languageservice's `selectorPrinting` logic.

## Change summary

Hovering over `&&` in a LESS file shows an incorrect selector interpretation (descendant instead of concatenation).

## Best solution

Fix the `toElement` function in `vscode-css-languageservice/src/services/selectorPrinting.ts` to handle adjacent `&` references by concatenating the parent selector classes rather than nesting them as descendants. The fix belongs upstream in microsoft/vscode-css-languageservice, then a dependency bump brings it into VS Code.
