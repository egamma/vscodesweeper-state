# Issue #292553 — ‘code --diff file1 file2‘ is very slow/ not working for bigger files

[https://github.com/microsoft/vscode/issues/292553](https://github.com/microsoft/vscode/issues/292553)

## Summary

This is a duplicate of the open, Backlog-milestoned issue #216953, which tracks the same problem: the advanced diff algorithm is too slow (or hangs) on larger files. The reporter's scenario (20k-line files via `code --diff`) is a specific instance of that known defect.

## Change summary

Running `code --diff` on two large files (~20k lines) is extremely slow or never completes because the advanced diff algorithm doesn't fall back to a faster alternative.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/216953, which proposes falling back to the legacy diff algorithm when the advanced one exceeds its time budget. The recent adoption of `@vscode/diff` (commit 5fc781f799f) may also address this in future iterations.
