# Issue #300675 — Alt+Shift+C: Cannot get current workspace folder path

[https://github.com/microsoft/vscode/issues/300675](https://github.com/microsoft/vscode/issues/300675)

## Summary

Valid bug, still present on main. When no Explorer item is selected (clicking blank area), Alt+Shift+C does nothing because `getMultiSelectedResources` returns an empty array — the fallback to the workspace root is missing. An open PR (#306248) addresses this; keeping open until it lands.

## Change summary

Pressing Alt+Shift+C (Copy Path) in the Explorer with no item selected copies nothing instead of the workspace folder path.

## Best solution

Review and land the open PR https://github.com/microsoft/vscode/pull/306248 which adds a workspace-root fallback to `getMultiSelectedResources` in `src/vs/workbench/contrib/files/browser/files.ts`.
