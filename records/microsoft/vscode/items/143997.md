# Issue #143997 — Cannot read property 'scheme' of null

[https://github.com/microsoft/vscode/issues/143997](https://github.com/microsoft/vscode/issues/143997)

## Summary

Still reproducible on main: the `API_OPEN_EDITOR_COMMAND_ID` handler at editorCommands.ts:463 accesses `.scheme` on a null `resourceArg` without any guard. The defect remains unfixed — keeping open as an actionable debt/bug.

## Change summary

When the SCM view invokes `vscode.open` with a null resource argument, `URI.from(null)` throws `TypeError: Cannot read property 'scheme' of null`.

## Best solution

Add a null guard for `resourceArg` at the top of the `API_OPEN_EDITOR_COMMAND_ID` handler in `src/vs/workbench/browser/parts/editor/editorCommands.ts` (around line 462), returning early when the argument is nullish. This prevents the crash regardless of which caller passes bad data.
