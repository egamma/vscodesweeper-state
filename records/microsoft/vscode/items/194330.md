# Issue #194330 — Consider using experiment friendly name when using `Report Issue` command

[https://github.com/microsoft/vscode/issues/194330](https://github.com/microsoft/vscode/issues/194330)

## Summary

Still valid: the issue reporter's experiment section shows encoded assignment-context IDs (e.g. `pythonnosmt12:30797651`) rather than human-readable feature-gate names. No change has landed on main to address this; the `getCurrentExperiments()` method still returns raw assignment context strings.

## Change summary

Show human-readable experiment/feature-gate names instead of encoded IDs in the Report Issue dialog's A/B experiments section.

## Best solution

In `WorkbenchAssignmentService.getCurrentExperiments()` (or at the issue-reporter consumption site in `issueService.ts`), map the encoded assignment-context entries to their friendly feature-gate names. Implement the fallback Tyler suggested: use friendly names when they fit the URL character budget, fall back to encoded IDs otherwise.
