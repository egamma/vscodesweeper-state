## Summary

Still unimplemented on main — the TODO comments in onLinesInserted/onLinesDeleted confirm the optimization (shift line data instead of invalidating everything below) has not been done yet. Keeping open as a valid plan-item.

## Change summary

GPU renderer invalidates all cached line data below an insert/delete point instead of shifting the still-valid entries up or down.

## Best solution

Implement the shift logic in `_invalidateLinesFrom` within `fullFileRenderStrategy.ts` so that on insert/delete, entries in `_upToDateLines` are renumbered rather than discarded, preserving already-rendered line data and reducing GPU buffer writes.
