# Issue #127932 — Custom keyboard layouts does not works properly on ozone wayland only mode

[https://github.com/microsoft/vscode/issues/127932](https://github.com/microsoft/vscode/issues/127932)

## Summary

Keep this open: the report is still a valid native Wayland keyboard-layout problem with recent confirmations, and the Backlog milestone does not require closure. The likely root is VS Code's native keymap dependency/path not getting the active Wayland layout, not an extension or a resolved VS Code behavior.

## Change summary

Native Wayland sessions resolve non-US/custom Linux keyboard layouts incorrectly for keybindings, causing shortcuts to dispatch as the wrong keys.

## Best solution

Track this with the Linux/Wayland keyboard-layout work until VS Code or `native-keymap` can reliably obtain the active Wayland keymap; the relevant integration is `src/vs/platform/keyboardLayout/electron-main/keyboardLayoutMainService.ts`, which currently forwards `native-keymap.getKeyMap()` and `getCurrentKeyboardLayout()` into the keybinding service.
