# Issue #127932 — Custom keyboard layouts does not works properly on ozone wayland only mode

[https://github.com/microsoft/vscode/issues/127932](https://github.com/microsoft/vscode/issues/127932)

## Summary

Still reproduces on current main: the keyboard layout service reads layout data exclusively via `native-keymap` which depends on X11/xkb APIs that are unavailable in Wayland-only mode, resulting in incorrect key mappings for non-QWERTY layouts. No Wayland-aware path exists in the codebase.

## Change summary

Custom keyboard layouts (Dvorak, Neo2, German with dead keys, ABNT2) produce incorrect keybindings when VS Code runs in Wayland-only mode because layout detection falls back to a default US layout.

## Best solution

The `native-keymap` module (microsoft/node-native-keymap) needs Wayland-native keyboard layout detection — likely via libxkbcommon or the wl_keyboard protocol's keymap event. The fix lives in that native module; once it reports correct mappings on Wayland, `src/vs/platform/keyboardLayout/electron-main/keyboardLayoutMainService.ts` will propagate them correctly without further changes.
