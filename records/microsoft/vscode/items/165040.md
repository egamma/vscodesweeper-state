# Issue #165040 — Set profile to have custom number of terminals at the same time by default

[https://github.com/microsoft/vscode/issues/165040](https://github.com/microsoft/vscode/issues/165040)

## Summary

Valid feature request in the Backlog with 24 upvotes. Requests the ability to define a default terminal layout (multiple terminals with preset names, colors, icons) that opens automatically on workspace start. No existing setting or canonical issue covers this; the gap remains on current main.

## Change summary

Allow users to configure a default set of terminals (with profile, color, name, icon) that automatically open when a workspace starts.

## Best solution

Implement a workspace-level setting (e.g. `terminal.integrated.defaultLayout`) that declares an array of terminal instances to create on startup, each referencing a profile and optional overrides for color/icon/name. This belongs in the terminal contrib area (`src/vs/workbench/contrib/terminal/`). Alternatively, tasks.json workarounds exist today but are clunky — a first-class setting would be the clean solution.
