# Issue #303668 — Terminal: Run Active File In Active Terminal fails when folder name contains a "~"

[https://github.com/microsoft/vscode/issues/303668](https://github.com/microsoft/vscode/issues/303668)

## Summary

Confirmed on current main: the `escapeNonWindowsPath` function strips `~` from paths via a banned-characters regex, breaking paths like `com~apple~CloudDocs` even though the path is subsequently single-quoted (where `~` is inert).

## Change summary

Running a file in the terminal fails when the file path contains a literal `~` character because the path-escaping logic removes it.

## Best solution

Remove `~` from the `bannedChars` regex in `escapeNonWindowsPath` (src/vs/platform/terminal/common/terminalEnvironment.ts line 69). The character is harmless inside single quotes, which is where the path ends up for all POSIX shells.
