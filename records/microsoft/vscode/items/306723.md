# Issue #306723 — The terminal title for a workspace-scoped task in a multi-root workspace shows a misleading folder suffix

[https://github.com/microsoft/vscode/issues/306723](https://github.com/microsoft/vscode/issues/306723)

## Summary

Confirmed on current main: `getQualifiedLabel()` appends the task's scoped workspace-folder name to the terminal title, which can be misleading when the task's actual `cwd` resolves to a different directory. The behavior is by-design for disambiguation but produces incorrect user expectations in multi-root workspaces with workspace-level tasks and dynamic `cwd`.

## Change summary

In a multi-root workspace, a task terminal's title suffix shows the task's scope folder rather than the actual execution directory, misleading users about where the task runs.

## Best solution

Modify `_createTerminalName` in `terminalTaskSystem.ts` (or `getQualifiedLabel` in `tasks.ts`) so that when the resolved `cwd` differs from the scoped workspace folder, the suffix reflects the actual execution directory — or omit it entirely for workspace-scoped tasks where the folder is ambiguous. A simpler alternative: add a `${taskFolder}` variable to the terminal title template so users can control this.
