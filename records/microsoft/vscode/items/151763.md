# Issue #151763 — when hovers are not sticky, alt/opt key should make them temporarily sticky

[https://github.com/microsoft/vscode/issues/151763](https://github.com/microsoft/vscode/issues/151763)

## Summary

The specific request — pressing Alt/Opt to make a non-sticky hover temporarily sticky so it can be interacted with — remains unimplemented on main. PR #236356 was the direct fix but was reverted due to regression. PR #274001 adds a related but different feature (hover only appears on modifier key press, rather than making a visible hover sticky on-demand).

## Change summary

When editor.hover.sticky is false, pressing Alt/Opt during an active hover should make it temporarily sticky so the user can click links or copy text.

## Best solution

Re-implement the reverted PR #236356 logic: when `editor.hover.sticky` is false and the hover is visible, pressing Alt/Opt should prevent the hover from dismissing, allowing interaction. The fix likely touches `src/vs/editor/contrib/hover/browser/contentHoverController.ts` (the sticky/hiding logic around line 190–248). The new `onKeyboardModifier` mode from PR #274001 is complementary but doesn't replace this.
