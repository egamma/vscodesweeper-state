# Issue #151763 — when hovers are not sticky, alt/opt key should make them temporarily sticky

[https://github.com/microsoft/vscode/issues/151763](https://github.com/microsoft/vscode/issues/151763)

## Summary

Still a valid open request. The user wants Alt/Opt to temporarily make editor hovers sticky when `editor.hover.sticky` is false. PR #236356 implemented this but was reverted due to regressions. PR #274001 (merged Nov 2025) added the related `onKeyboardModifier` mode for `editor.hover.enabled`, but that controls whether the hover *appears*, not whether a non-sticky hover becomes temporarily sticky — the core ask remains unimplemented.

## Change summary

When `editor.hover.sticky` is false, pressing Alt/Opt should temporarily make the hover sticky so users can interact with links and text in it.

## Best solution

Re-implement the reverted alt-makes-hover-temporarily-sticky logic in `src/vs/editor/contrib/hover/browser/contentHoverController.ts`, guarding against the regression that caused the revert (https://github.com/microsoft/vscode/issues/236583). The `isTriggerModifierPressed` helper in `hoverUtils.ts` can likely be reused. This is a code change in the editor hover contribution, not an extension-candidate.
