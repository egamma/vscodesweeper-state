# Issue #259650 — Search and replace across files scrolls poorly

[https://github.com/microsoft/vscode/issues/259650](https://github.com/microsoft/vscode/issues/259650)

## Summary

Still present on main: after a file-level Replace All in the search view, `viewer.reveal(nextFocusElement)` scrolls the next result into view but doesn't anchor it near the top of the pane, leaving the user with collapsed/hidden lines and poor orientation. No fix has landed.

## Change summary

After applying Replace All for a file in the search results pane, the next focused result often appears at the bottom of the viewport or is obscured by collapsed context lines, making it hard to see what will be replaced next.

## Best solution

In `searchActionsRemoveReplace.ts`, the `viewer.reveal(nextFocusElement)` calls (lines ~137 and ~306) should pass a scroll-position hint (e.g. `TreeViewVisibility.Top` or equivalent) to ensure the next focused element appears near the top of the search results pane after a replacement, rather than using the default minimal-scroll reveal.
