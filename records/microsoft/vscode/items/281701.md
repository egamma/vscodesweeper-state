# Issue #281701 — VSCode not detect Laravel Blade file as Blade instead of HTML

[https://github.com/microsoft/vscode/issues/281701](https://github.com/microsoft/vscode/issues/281701)

## Summary

Laravel Blade is a framework-specific templating language; detecting it belongs in an extension (several exist on the Marketplace), not in VS Code core's built-in language detection.

## Change summary

Detect `.blade.php` files as Blade language instead of HTML by default.

## Best solution

Install a Blade extension from the Marketplace (e.g. "Laravel Blade Snippets" or "Laravel Blade formatter") which registers the `.blade.php` file association and provides syntax highlighting, completions, and formatting. This is not something VS Code core should bundle.

## Vision fit

- **Fit:** rejected
- **Reason:** Adding built-in detection for a framework-specific template language cuts against the value of keeping VS Code a lightweight text editor out of the box, with specialized capabilities living in extensions.
- **Evidence:** Out of the box, VS Code is a text editor: lightweight and fast. Users may install extensions that impact the appearance and nature of VS Code. The power lives in extensions.
