# Issue #232021 — Dragging a view or tab could show grabbing hand

[https://github.com/microsoft/vscode/issues/232021](https://github.com/microsoft/vscode/issues/232021)

## Summary

Valid polish request to show a grabbing cursor during tab/view drag operations. Not yet implemented on main — the workbench DnD code has no grabbing cursor logic for tabs or views. A team member engaged with implementation questions, so this is actively considered.

## Change summary

Show a CSS `grabbing` cursor when dragging editor tabs or workbench views.

## Best solution

Add a CSS class (e.g. `.dragging`) to the dragged tab/view element on drag-start that sets `cursor: grabbing`, applied via the existing DnD handling in the workbench editor tabs and views infrastructure. Needs cross-platform testing (Windows cursor rendering noted as a concern).
