# Issue #254179 — Window resize lag when a webview is visible

[https://github.com/microsoft/vscode/issues/254179](https://github.com/microsoft/vscode/issues/254179)

## Summary

This is an upstream Electron/Chromium regression introduced with the Electron 35 upgrade and confirmed to persist through Electron 37 and likely Electron 39. The reporter bisected it precisely and the root cause is in how Chromium composites webview/iframe layers during window resize. No VS Code-side fix has been applied, and the issue remains valid.

## Change summary

Window resize becomes laggy when any webview (e.g. extension page) is visible, a regression introduced with the Electron 35 upgrade.

## Best solution

File or track an upstream Chromium/Electron bug for webview compositing lag during window resize. In the interim, VS Code could explore throttling webview relayout during resize or deferring webview repaints until resize ends, but a real fix belongs in Electron's compositor. The `upstream` label would be appropriate if it exists.
