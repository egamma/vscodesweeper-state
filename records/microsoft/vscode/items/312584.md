# Issue #312584 — Agent Plugins: marketplace plugin detail view renders blank body for valid README

[https://github.com/microsoft/vscode/issues/312584](https://github.com/microsoft/vscode/issues/312584)

## Summary

Still reproduces on main: the README detail pipeline in AgentPluginEditor silently renders a blank body when `fetchReadme` returns an empty string (due to a missing `readmeUri` or a silently caught file-read error), because the fallback message only triggers on exception — not on empty content.

## Change summary

Agent Plugins marketplace detail view renders a blank body instead of the README content (or a fallback message) for valid plugins.

## Best solution

Guard the `openMarkdown`/`renderMarkdown` flow in `agentPluginEditor.ts` to treat an empty-string result from `fetchReadme` the same as an exception — show the 'No README available.' fallback. Additionally, investigate why `fetchReadme` returns empty for plugins whose README exists on disk (likely the `readmeUri` is not being propagated correctly from the marketplace service when `repoDir` is set).
