# Issue #185870 — Editor rulers disappear randomly in large files

[https://github.com/microsoft/vscode/issues/185870](https://github.com/microsoft/vscode/issues/185870)

## Summary

Still reproduces on the DOM renderer path: rulers use a tall CSS element capped at 1M px but Chromium's GPU compositor can still produce rendering gaps at certain scroll positions in large files. The GPU renderer works around this with a different approach, but the DOM fallback remains affected.

## Change summary

Editor rulers randomly disappear (render with gaps) when scrolling large files due to Chromium GPU texture limits on tall DOM elements.

## Best solution

Apply the virtualizing approach the author suggested for the DOM path — reposition the ruler element relative to the viewport on each scroll rather than spanning the full scroll height. Alternatively, promote the GPU renderer as the default path, which already uses `Number.MAX_SAFE_INTEGER` without the DOM texture-limit issue.

## Evidence
