# Issue #312145 — Remote SSH: Copilot/agent plugin discovery fails with PendingMigrationError: navigator is now a global in nodejs on VS Code 1.117.0

[https://github.com/microsoft/vscode/issues/312145](https://github.com/microsoft/vscode/issues/312145)

## Summary

The PendingMigrationError is an intentional VS Code compatibility shim that warns extensions accessing `navigator` in Node.js — the root cause is the bundled Copilot extension accessing `navigator` in the remote extension host. This is a Copilot extension bug, not a VS Code core issue.

## Change summary

Copilot extension activation in Remote SSH triggers PendingMigrationError when it accesses the `navigator` global in a Node.js extension host, causing incomplete agent plugin discovery.

## Best solution

The bundled Copilot extension needs to avoid accessing `navigator` when running in a Node.js extension host (Remote SSH). The user can work around it by setting `"extensions.supportNodeGlobalNavigator": true` to expose the real Node.js navigator global. The fix belongs in the Copilot extension, not VS Code core.
