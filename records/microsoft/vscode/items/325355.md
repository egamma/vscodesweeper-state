# Issue #325355 — Switching from Autopilot to Plan erases draft and automatically sends “Start implementation”

[https://github.com/microsoft/vscode/issues/325355](https://github.com/microsoft/vscode/issues/325355)

## Summary

The Plan mode (contributed by the Copilot extension via `planAgentProvider.ts`) registers a 'Start Implementation' handoff with `send: true`, targeting the Agent mode. When the user manually switches from Autopilot to Plan via the mode picker, the mode switch may trigger session clearing (due to kind change) and the handoff prompt appears to auto-fire, erasing the user's draft. This is a real bug involving both core mode-switching and the Copilot extension's Plan mode configuration.

## Change summary

Switching from Autopilot to Plan mode via the mode picker erases the user's draft text and automatically sends 'Start implementation' to the agent.

## Best solution

The mode-switch flow should preserve the user's draft input across mode changes. The Plan mode's 'Start Implementation' handoff (with `send: true`) should only fire when the user explicitly clicks the handoff button in the suggest-next widget, never when the user manually switches modes via the picker. The fix likely involves `chatWidget.ts` (ensuring `executeHandoff` is not triggered by mode picker changes) and possibly `chatInputPart.ts` (preserving draft text across mode/session transitions). The Copilot extension's `planAgentProvider.ts` handoff configuration itself is correct — the bug is in when/how the handoff gets triggered.
