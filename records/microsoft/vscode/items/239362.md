# Issue #239362 — [Accessibility] Cannot read multi diff content in Source Control Graph View

[https://github.com/microsoft/vscode/issues/239362](https://github.com/microsoft/vscode/issues/239362)

## Summary

Confirmed and root-caused by a team member (lszomoru): the multi-diff editor's active document observable is undefined when focus() is called from the SCM Graph view, so keyboard focus never lands in the editor. The issue remains unfixed on main.

## Change summary

Screen readers cannot read multi-diff editor content opened from the SCM Graph view because the editor never receives keyboard focus.

## Best solution

Fix the timing issue in `src/vs/editor/browser/widget/multiDiffEditor/multiDiffEditorWidgetImpl.ts` so that the `_navigateToChange` / focus logic waits for the view model's documents to be loaded before attempting to focus. The existing `preserveFocus` infrastructure (lines 275–281) already gates focus — the fix likely involves ensuring documents are populated before the autorun fires, or deferring focus until the first document materializes.
