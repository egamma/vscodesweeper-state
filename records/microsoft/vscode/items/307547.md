# Issue #307547 — `runSubagent` tool not available to custom agents invoked as subagents (nested subagent invocation broken)

[https://github.com/microsoft/vscode/issues/307547](https://github.com/microsoft/vscode/issues/307547)

## Summary

The VS Code core nesting logic in runSubagentTool.ts appears functional on current main — it checks the config, tracks depth, and conditionally enables RunSubagentTool.Id in spawned agent modeTools. However, a team member acknowledged the report, one commenter says it works for them, and the root cause may lie in how the Copilot extension resolves the `agent` tool alias for custom agents at depth-2. Keeping open for the area owner to confirm whether the extension-side tool resolution is the gap.

## Change summary

Custom agents invoked as subagents (depth-2) do not receive the `runSubagent` tool even when nesting is enabled via the config setting.

## Best solution

The area owner should verify whether the Copilot extension's tool-set resolution properly threads the `agent` tool alias through to depth-2 custom agent contexts. The VS Code core depth-gating logic in `src/vs/workbench/contrib/chat/common/tools/builtinTools/runSubagentTool.ts` (lines 294–309) looks correct; the gap is likely in extension-side tool alias mapping. Also note the setting was renamed from `chat.subagents.allowInvocationsFromSubagents` to `chat.subagents.allowSubagentInvocation` (commit e82718d8ba1, 2026-03-24) — the reporter may have been using the old name.
