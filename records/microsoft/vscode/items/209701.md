# Issue #209701 — New line support in labels and `workbench.editor.customLabels.patterns` setting

[https://github.com/microsoft/vscode/issues/209701](https://github.com/microsoft/vscode/issues/209701)

## Summary

Valid feature request with 23 upvotes (near the 20-upvote Backlog threshold) asking for multi-line or description support in custom editor labels. Not yet implemented on main — the custom label service only supports a single template string per pattern with no description/subtitle field.

## Change summary

Allow custom editor tab labels to display a second line (description/subtitle) to reduce horizontal space usage when many similarly-named files are open.

## Best solution

Extend `workbench.editor.customLabels.patterns` to support a description template (rendered as a subtitle below the label, like the existing `..\tree` description shown in tab tooltips/breadcrumbs). This would live in `src/vs/workbench/services/editor/common/customEditorLabelService.ts` and the tab rendering code. A maintainer comment from @benibenj already explored the description approach — the next step is a design decision on the schema (e.g. `{ label: "...", description: "..." }` vs. a separator character).
