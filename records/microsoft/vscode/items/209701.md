# Issue #209701 — New line support in labels and `workbench.editor.customLabels.patterns` setting

[https://github.com/microsoft/vscode/issues/209701](https://github.com/microsoft/vscode/issues/209701)

## Summary

Valid feature request with 23 upvotes asking for multiline/description support in custom tab labels. The underlying tab infrastructure already supports descriptions (a second line), but the `customLabels.patterns` setting doesn't expose a way to populate it. Not yet implemented on main; should remain open in the Backlog for future consideration.

## Change summary

Allow `workbench.editor.customLabels.patterns` to specify a description (second line) for editor tabs, enabling compact two-line labels.

## Best solution

Extend the custom editor label service (`src/vs/workbench/services/editor/common/customEditorLabelService.ts`) to support a description pattern alongside the existing name pattern, and wire it into the tab rendering in `multiEditorTabsControl.ts`. The tab infrastructure already renders descriptions — the gap is in the custom-label configuration surface.
