# Issue #146799 — Make test message be able to toggle between 'verbose' and 'simple' mode.

[https://github.com/microsoft/vscode/issues/146799](https://github.com/microsoft/vscode/issues/146799)

## Summary

Team-authored feature request for a verbose/simple toggle in the test message peek view; still open and valid — the capability does not exist on current main.

## Change summary

Allow the test message peek view to accept two message sets (verbose and simple) with a toggle for users to switch between them.

## Best solution

Add an API surface in the test results peek view (likely in `src/vs/workbench/contrib/testing/browser/testingOutputPeek.ts` and related files) that accepts both a verbose and simple message from test extensions, with a UI toggle similar to IntelliJ/Eclipse. The Test Controller API would need a way for extensions to supply both message variants.
