Current code still disables VS Code's editor scroll animation when `editor.smoothScrolling` is false, so the remaining smoothness may be native OS/compositor wheel behavior as the maintainer comment suggested.

## Change summary

Mouse wheel scrolling in the text editor still appears animated on Linux even with `--disable-smooth-scrolling` and `editor.smoothScrolling: false`.

If it reproduces on current Insiders with native smooth scrolling disabled, investigate the wheel path in `src/vs/base/browser/ui/scrollbar/scrollableElement.ts` and the editor duration wiring in `src/vs/editor/common/viewLayout/viewLayout.ts`.
