# Issue #150010 — Testing: regarding to refreshHandler 

[https://github.com/microsoft/vscode/issues/150010](https://github.com/microsoft/vscode/issues/150010)

## Summary

The requested capability — a global busy/progress state on TestController that toggles the refresh→stop button during automatic (non-user-triggered) rediscovery — is still missing on current main. The existing workaround (withProgress) only shows the progress bar, not the stop button.

## Change summary

Allow test extensions to signal global busy state (progress bar + stop button) during automatic test rediscovery when no TestItem exists yet.

## Best solution

Add a `busy` property or method on TestController that, when set, toggles the refresh button to a cancel/stop button and shows the progress indicator — mirroring what refreshHandler does but available for extension-initiated (automatic) discovery. The relevant files are `src/vs/workbench/api/common/extHostTesting.ts` and `src/vs/workbench/contrib/testing/common/testServiceImpl.ts`.
