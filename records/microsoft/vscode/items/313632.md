# Issue #313632 — Copilot Chat Agent file changes removed lines do not respect word-wrap

[https://github.com/microsoft/vscode/issues/313632](https://github.com/microsoft/vscode/issues/313632)

## Summary

Still reproduces on main: in the chat's inline diff view, the diff editor explicitly disables word wrap for the original (left/removed) side when not rendering side-by-side, so removed lines don't wrap even when `wordWrap: 'on'` is set via chat editor options.

## Change summary

In Copilot Chat agent mode, removed lines in inline file-change diffs do not respect word-wrap, causing horizontal overflow while added lines wrap correctly.

## Best solution

The fix likely belongs in `src/vs/editor/browser/widget/diffEditor/components/diffEditorEditors.ts` — when `useTrueInlineView` is active (both sides are visible even in inline mode), the original editor should not force `wordWrapOverride1: 'off'`. Alternatively, the chat's `CodeCompareBlockPart` could explicitly pass `diffWordWrap: 'on'` and coordinate with a diff-editor-level fix.
