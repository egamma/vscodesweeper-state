# Issue #250857 — git merge conflict: if you right click on a file and accept all incoming changes, or accept all current changes, the file should be staged

[https://github.com/microsoft/vscode/issues/250857](https://github.com/microsoft/vscode/issues/250857)

## Summary

Valid feature request: the right-click 'Accept All Incoming/Current' commands on merge-conflict files in the SCM view resolve conflict markers in the text but don't stage the file afterward, requiring an extra manual step. This is a workflow gap between the inline merge-conflict extension and the merge editor (which does stage on close). Keeping open for a product decision on whether auto-staging is the right behavior.

## Change summary

After right-clicking a merge-conflict file in the SCM view and choosing 'Accept All Incoming Changes' or 'Accept All Current Changes', the file should be automatically staged.

## Best solution

After `acceptAllResources` in `extensions/merge-conflict/src/commandHandler.ts` finishes applying the workspace edit, call the git extension's stage API (e.g. `git.stage` command) on the resolved files. Alternatively, the git extension could listen for conflict-marker removal in merge-group files and auto-stage. Note that https://github.com/microsoft/vscode/issues/260382 is a near-duplicate that should be consolidated with this issue.
