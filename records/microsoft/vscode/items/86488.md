# Issue #86488 — [scss] references for all SASS functions/at-rules

[https://github.com/microsoft/vscode/issues/86488](https://github.com/microsoft/vscode/issues/86488)

## Summary

Partially addressed — some SASS at-rules and module functions now have documentation references in vscode-css-languageservice, but many at-rules (@mixin, @include, @extend, @if, @for, @each, @while, @function, @at-root, @debug, @warn, @error) still lack them, and the data isn't in a standalone JSON file as requested.

## Change summary

Add links to official SASS documentation for all SASS functions and at-rules in completion/hover results.

## Best solution

Complete the reference coverage in the vscode-css-languageservice package (upstream) by adding sass-lang.com documentation URLs for the remaining at-rules (@mixin, @include, @extend, @if, @for, @each, @while, @function, @at-root, @debug, @warn, @error) and consider extracting all SCSS data into a JSON file for maintainability.
