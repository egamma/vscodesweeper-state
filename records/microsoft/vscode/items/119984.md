# Issue #119984 — EnvironmentVariableCollection in remote vs local terminals

[https://github.com/microsoft/vscode/issues/119984](https://github.com/microsoft/vscode/issues/119984)

## Summary

Team-authored feature request; the EnvironmentVariableScope API currently only supports workspaceFolder scoping, not remote/local terminal scoping. The requested capability remains unimplemented on main.

## Change summary

Allow EnvironmentVariableCollection mutations to be scoped to remote-only or local-only terminals.

## Best solution

Extend `EnvironmentVariableScope` (in `src/vscode-dts/vscode.d.ts`) with a property indicating whether the collection applies to remote terminals, local terminals, or both. The implementation in `src/vs/platform/terminal/common/environmentVariable.ts` and `src/vs/workbench/api/common/extHostTerminalService.ts` would need to filter mutations based on the terminal's locality.
