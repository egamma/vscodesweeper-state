# Issue #101925 — Zooming with Ctrl+MouseWheel doesn't update configuration

[https://github.com/microsoft/vscode/issues/101925](https://github.com/microsoft/vscode/issues/101925)

## Summary

This should stay open because it has an assigned milestone (`On Deck`), which is a hard keep-open condition for this review. The report describes an extension-facing gap around editor mouse-wheel font zoom state/events rather than a closeable defect with current evidence.

## Change summary

Ctrl+MouseWheel editor font zoom changes the rendered editor font size/line height without updating configuration or exposing an extension-observable zoom event.

## Best solution

Keep tracking this under editor-core and decide whether the editor should expose the effective font zoom level or a dedicated event/API for extensions that depend on rendered font metrics, rather than mutating user settings during transient mouse-wheel zoom.
