# Issue #101925 — Zooming with Ctrl+MouseWheel doesn't update configuration

[https://github.com/microsoft/vscode/issues/101925](https://github.com/microsoft/vscode/issues/101925)

## Summary

The issue has an assigned milestone (On Deck), so it must stay open per policy. The underlying request — exposing the editor font zoom level to extensions — remains unaddressed on current main: EditorZoom is an in-memory singleton with no configuration writeback and no extension API surface.

## Change summary

Ctrl+MouseWheel editor zoom changes are not reflected in configuration or exposed to extensions, breaking decorations that depend on font size.

## Best solution

Either expose `EditorZoom.getZoomLevel()` / `onDidChangeZoomLevel` through the extension API (e.g. as a read-only property + event on `vscode.window`), or write the zoom level back to a configuration key so extensions can observe it via `onDidChangeConfiguration`. The relevant code is in `src/vs/editor/common/config/editorZoom.ts` and the mouse-wheel handler in `src/vs/editor/browser/controller/mouseHandler.ts`.
