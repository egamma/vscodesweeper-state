# Issue #323620 — Autopilot/Bypass silently skips user-requested high-risk actions with no opt-out

[https://github.com/microsoft/vscode/issues/323620](https://github.com/microsoft/vscode/issues/323620)

## Summary

Legitimate feature request: the Autopilot risk gate (`_maybeApplyAutopilotRiskGate`) is intentionally designed to silently skip high-risk tool calls with no user-facing opt-out. The reporter wants either a way to disable this secondary gate or a confirmation prompt instead of a silent skip. This is a deliberate design choice, but the gap between the documented promise ("bypass all approvals including destructive ones") and the actual behavior is a real UX issue worth tracking.

## Change summary

Autopilot mode silently skips tool calls assessed as high-risk by an LLM classifier, with no setting to disable or override this behavior, contradicting the documented "bypass all approvals" promise.

## Best solution

Either expose a setting (e.g. under `chat.autopilot.advanced.*`) to let users opt out of the risk gate for auto-approved calls, or fall back to a confirmation prompt instead of silently skipping. The risk gate lives in `_maybeApplyAutopilotRiskGate` in `languageModelToolsService.ts` and only fires when `chat.autopilot.advanced.enabled` is `true` — so the scope is narrower than the reporter thinks (requires Advanced Autopilot). The docs should also be updated to clarify the risk gate layer.

## Evidence

- **risk gate code:** The autopilot risk gate is in `src/vs/workbench/contrib/chat/browser/tools/languageModelToolsService.ts` at `_maybeApplyAutopilotRiskGate()` (line ~945). It only activates when `chat.autopilot.advanced.enabled` is `true` (line 981), the session is Autopilot-level, and the tool would otherwise show a confirmation or is a self-approving tool (terminal/fetch).
- **gate behavior:** When the LLM risk classifier returns `ToolRiskLevel.Red`, the tool call is silently skipped (line 1001–1005) and an info note is appended to the chat (line 650–652). There is no setting to disable this classification layer.
- **advanced autopilot prerequisite:** The risk gate requires `chat.autopilot.advanced.enabled = true` (default `false`, experimental). The reporter's config may have this enabled. Without it, the gate never fires.
- **related closed issue:** https://github.com/microsoft/vscode/issues/303174 ('autopilot mode should allow curl in terminal') was a similar complaint and was closed as a bug fix — the `chat.tools.terminal.ignoreDefaultAutoApproveRules` setting was added. This issue is about a different, deeper layer.
- **design intent:** The code comments explicitly describe this as a safety net: 'a risky command or a prompt-injected URL would run unclassified' — the gate is intentional, but the lack of an opt-out is a gap.
