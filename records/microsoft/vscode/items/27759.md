# Issue #27759 — Align explorer sorting with platform sorting

[https://github.com/microsoft/vscode/issues/27759](https://github.com/microsoft/vscode/issues/27759)

## Summary

Significant progress was made — the `explorer.sortOrderLexicographicOptions` setting (upper/lower/unicode) and edge-case fixes address most concrete examples — but the original ask of *automatic* platform-aligned sorting (detecting the OS collation behavior) remains unimplemented. With 39 upvotes and a real remaining gap, this should stay open.

## Change summary

Explorer file sorting should automatically match the host platform's native sort behavior (case grouping, underscore/dot ordering) instead of requiring manual settings.

## Best solution

The remaining gap is auto-detection of platform collation (e.g. reading `LC_COLLATE` on Linux, matching Finder behavior on macOS). A maintainer should decide whether that auto-alignment is still worth pursuing or whether the existing manual `explorer.sortOrderLexicographicOptions` setting is sufficient. If auto-detection is out of scope, this could be closed as addressed-enough by the existing settings, but that's a product call given 39 upvotes.
