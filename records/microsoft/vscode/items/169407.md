# Issue #169407 — Exit code isn't handled correctly in fish shell integration

[https://github.com/microsoft/vscode/issues/169407](https://github.com/microsoft/vscode/issues/169407)

## Summary

Still reproduces on main: in the no-fish_mode_prompt path, `__vsc_fish_prompt_start` is called before `__vsc_fish_prompt`, overwriting `$status`/`$pipestatus` before the user's prompt function can read them. The related fix PR #245109 was closed without merging. Issue is labeled `help wanted` and remains valid.

## Change summary

Fish shell integration overwrites `$status` and `$pipestatus` by calling `__vsc_fish_prompt_start` before the user's prompt function, breaking exit-code display in custom prompts.

## Best solution

In `src/vs/workbench/contrib/terminal/common/scripts/shellIntegration.fish`, the no-mode-prompt `fish_prompt` wrapper (lines 243–248) should capture the user prompt output first (preserving `$status`/`$pipestatus`), then emit the prompt-start sequence, then print the captured output — matching the fix pattern the reporter suggested. The mode-prompt path (lines 233–241) likely needs the same treatment.
