# Issue #299953 — Agent mode Keep/undo Edit diff should allow copying from the deleted (red) part

[https://github.com/microsoft/vscode/issues/299953](https://github.com/microsoft/vscode/issues/299953)

## Summary

Valid request: the chat editing inline diff renders deleted text in a view zone without the `line-delete-selectable` class, so it cannot be selected or copied. The diff editor proper already supports this via PR #267991 — it just needs to be propagated to the chat editing integration.

## Change summary

Allow selecting and copying text from the deleted (red) portion of agent mode keep/undo edit diffs.

## Best solution

Add the `line-delete-selectable` class to the view zone DOM node at `src/vs/workbench/contrib/chat/browser/chatEditing/chatEditingCodeEditorIntegration.ts` line 426, and wire up the copy-selection logic from `src/vs/editor/browser/widget/diffEditor/components/diffEditorViewZones/copySelection.ts` to make deleted text selectable and copyable in agent mode diffs.
