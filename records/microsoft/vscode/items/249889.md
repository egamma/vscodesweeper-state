# Issue #249889 — Debug hover tooltip too small

[https://github.com/microsoft/vscode/issues/249889](https://github.com/microsoft/vscode/issues/249889)

## Summary

Still present on main: the debug hover tooltip width is hard-clamped to 550px in `layoutTree()`, making it too narrow for long identifiers (e.g. C++ namespaced symbols). The constraint has not been relaxed.

## Change summary

Debug hover tooltip is too small for long identifiers because the tree width is clamped to a maximum of 550px.

## Best solution

Relax the width clamp in `layoutTree()` (line 360 of `debugHover.ts`) so that `treeWidth` scales with editor width or window width instead of being hard-capped at 550px. Consider making the hover resizable or at least allowing a wider maximum proportional to the editor layout width.
