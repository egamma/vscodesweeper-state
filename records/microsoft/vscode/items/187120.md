# Issue #187120 — The configured editorOverviewRuler.selectionHighlightForeground property is invalid

[https://github.com/microsoft/vscode/issues/187120](https://github.com/microsoft/vscode/issues/187120)

## Summary

The language-dependent difference is intentional: when a document highlight provider supplies semantic word highlights, the overview ruler uses the word-highlight color keys rather than `editorOverviewRuler.selectionHighlightForeground`.

## Change summary

The reporter wants `editorOverviewRuler.selectionHighlightForeground` to control occurrence markers in HTML, JS, and CSS the same way it appears to in JSON.

## Best solution

Keep the existing split between selection highlights and semantic word highlights. Users who want the same ruler color in HTML/JS/CSS should also customize `editorOverviewRuler.wordHighlightForeground`, `editorOverviewRuler.wordHighlightStrongForeground`, and where applicable `editorOverviewRuler.wordHighlightTextForeground`.

  > Thanks for the report. This looks as designed: `editorOverviewRuler.selectionHighlightForeground` applies to the plain selection-highlight overview ruler marker, but when a language provides semantic document highlights, VS Code uses the word-highlight ruler colors instead. For HTML/JS/CSS, please configure `editorOverviewRuler.wordHighlightForeground`, `editorOverviewRuler.wordHighlightStrongForeground`, and where applicable `editorOverviewRuler.wordHighlightTextForeground` to get the same ruler color.

## Evidence

- **selection highlight behavior:** In `src/vs/editor/contrib/wordHighlighter/browser/highlightDecorations.ts`, `getSelectionHighlightDecorationOptions` explicitly returns the no-overview-ruler decoration when semantic highlights are present, with the comment: `Show in overviewRuler only if model has no semantic highlighting`.
- **semantic highlight colors:** `src/vs/editor/contrib/wordHighlighter/browser/highlightDecorations.ts` registers and uses separate overview-ruler colors for semantic word highlights: `editorOverviewRuler.wordHighlightForeground`, `editorOverviewRuler.wordHighlightStrongForeground`, and `editorOverviewRuler.wordHighlightTextForeground`.
- **reported file types:** The affected HTML, JS, and CSS cases are languages that commonly provide document highlights, so the code path uses `getHighlightDecorationOptions(highlight.kind)` in `src/vs/editor/contrib/wordHighlighter/browser/wordHighlighter.ts` rather than the plain selection-highlight overview ruler decoration.
