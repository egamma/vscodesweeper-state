# Issue #250087 — node debug clobbers breakpoints if a transpiled asset already exists

## Summary

Active investigation by a team member (connor4312) who requested and received trace logs; the reporter provided detailed reproduction evidence with screenshots. Keeping open pending maintainer analysis of the logs.

## Change summary

Node debugger maps breakpoints to incorrect source locations when a previously-transpiled output file already exists on disk, even though the source hasn't changed.

## Best solution

Investigate the source-map resolution logic in vscode-js-debug — the second session reuses a stale source-map association from the existing compiled file rather than re-resolving from the fresh build. The trace logs provided should pinpoint where the mapping diverges between first run (no dist) and second run (dist exists).
