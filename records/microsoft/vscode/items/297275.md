# Issue #297275 — Git diff fails for modified files when opening drive root as workspace on Windows (regression in 1.105)

[https://github.com/microsoft/vscode/issues/297275](https://github.com/microsoft/vscode/issues/297275)

## Summary

Confirmed regression on current main: normalizePath() in the git extension strips the trailing separator from drive-root paths (e.g. `d:\` → `d:`), which path.normalize() turns into `d:.`, breaking isDescendant() and causing GitFileSystemProvider to throw FileNotFound for modified files.

## Change summary

Git diff fails for modified tracked files when a drive root (e.g. `D:\`) is opened as the VS Code workspace on Windows.

## Best solution

Guard the trailing-separator removal in `normalizePath()` (extensions/git/src/util.ts, line 316) so it does not strip when the path is a bare Windows drive root (e.g. `d:\`). The fix is a small conditional — the same class of bug was fixed once before for issue #142842 and was reintroduced by commit b276d35cd60.
