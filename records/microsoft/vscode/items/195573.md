# Issue #195573 — Allow commands to have optional descriptions to be used for similar commands feature

[https://github.com/microsoft/vscode/issues/195573](https://github.com/microsoft/vscode/issues/195573)

## Summary

Valid feature request — the internal TF-IDF similar-commands machinery already consumes `commandDescription` for scoring, but the `commands` extension contribution point in package.json has no `description` field, so extensions cannot feed keywords into the similarity index.

## Change summary

Allow extensions to contribute an optional description for commands (via package.json) that would be used by the TF-IDF similar-commands index in the Command Palette.

## Best solution

Add an optional `description` (or `keywords`) property to `IUserFriendlyCommand` / `schema.commandType` in `src/vs/workbench/services/actions/common/menusExtensionPoint.ts`, plumb it through to `ICommandMetadata.description` so the existing TF-IDF chunk computation in `src/vs/platform/quickinput/browser/commandsQuickAccess.ts` picks it up automatically.
