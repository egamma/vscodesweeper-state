# Issue #285587 — explorer.autoReveal false and focusNoScroll is broken.

[https://github.com/microsoft/vscode/issues/285587](https://github.com/microsoft/vscode/issues/285587)

## Summary

Source-traceable bug for the `focusNoScroll` case: when the explorer becomes visible on restore, `selectActiveFile(true)` at line 332 hardcodes `reveal=true`, bypassing the user's `focusNoScroll` setting. Additionally, the `focus()` method at line 353 calls `this.tree.reveal()` whenever `_autoReveal` is truthy, which includes `'focusNoScroll'`. The `autoReveal: false` case is less clear — the setting is respected in `selectActiveFile` but the tree's own view-state restoration may scroll to the previously focused item.

## Change summary

Explorer scrolls to the active file on restore even when `explorer.autoReveal` is set to `false` or `focusNoScroll`.

## Best solution

Fix `selectActiveFile` call at line 332 of `explorerView.ts` to pass the actual `_autoReveal` value instead of hardcoded `true`. Fix the `focus()` method to skip `tree.reveal()` when `_autoReveal === 'focusNoScroll'`. The `autoReveal: false` report may need separate investigation into tree view-state restoration behavior.
