# Issue #287800 — ⛔ 100% CPU on Windows when a watched file is deleted

[https://github.com/microsoft/vscode/issues/287800](https://github.com/microsoft/vscode/issues/287800)

## Summary

Confirmed upstream Node.js bug (fs.watch infinite loop on Windows when watched path is deleted). The VS Code fix PR #288003 was merged then reverted because the root cause is in libuv/Node.js. The upstream issue (nodejs/node#61398) remains open. A team member reopened this to track the fix landing. Keep open.

## Change summary

On Windows, deleting a watched directory causes the Node.js file watcher utility process to spin at 100% CPU in an infinite ReadDirectoryChangesW loop.

## Best solution

Wait for the upstream fix in nodejs/node#61398 to land, then pick it up via a Node.js/Electron update. Alternatively, add a VS Code-side workaround in `src/vs/platform/files/node/watcher/nodejs/nodejsWatcherLib.ts` that detects the deleted-path spin (e.g., check for ENOENT on the watched path when receiving rapid events and close the watcher). The issue is correctly labeled `upstream` and should stay open until resolved.
