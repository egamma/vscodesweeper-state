# Issue #166024 — Cursor selection scroll down not working when status bar is hidden and in full screen mode

[https://github.com/microsoft/vscode/issues/166024](https://github.com/microsoft/vscode/issues/166024)

## Summary

Still present on main. The editor's drag-scroll-down detection requires `e.posy > editorContent.y + editorContent.height`, which is physically impossible when the editor fills the screen bottom (fullscreen + hidden status bar). The milestone is 'On Deck', so keeping open regardless.

## Change summary

Selection drag-scroll-down fails when the editor extends to the very bottom of the OS screen (fullscreen + status bar hidden), because the mouse cursor can never exceed the editor's bottom boundary.

## Best solution

In `_getPositionOutsideEditor` (src/vs/editor/browser/controller/mouseHandler.ts ~line 554), add a special case: when the editor's bottom aligns with the screen bottom, treat `e.posy >= editorContent.y + editorContent.height - 1` (or some small tolerance) as 'below' to trigger drag scrolling at the OS-clamped screen edge.
