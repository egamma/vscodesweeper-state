# Issue #268692 — An OS keyring couldn't be identified

[https://github.com/microsoft/vscode/issues/268692](https://github.com/microsoft/vscode/issues/268692)

## Summary

Verified duplicate of the canonical upstream-keyring-detection tracking issue. The root cause is Chromium's desktop environment detection in `xdg_util.cc` not recognizing COSMIC/Budgie, which is already tracked with upstream Chromium links.

## Change summary

Chromium's desktop environment detection fails on newer Linux DEs (COSMIC, Budgie), causing VS Code's encryption/secret storage to fall back to in-memory storage.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/187338, which is the canonical tracking issue already labeled `upstream, chromium, upstream-issue-linked`. The fix belongs in Chromium (https://issues.chromium.org/issues/464482033); the workaround is setting `"password-store"` in `~/.vscode/argv.json`.
