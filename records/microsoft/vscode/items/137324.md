# Issue #137324 — Merge Conflict incorrectly removing conflict markers

[https://github.com/microsoft/vscode/issues/137324](https://github.com/microsoft/vscode/issues/137324)

## Summary

Verified duplicate of #107821 — both describe the same race condition where stale conflict ranges from codelens arguments are applied after a prior edit shifts line positions. The canonical issue is older, better-trafficked, and fully absorbs this report.

## Change summary

Clicking 'Accept' on a merge conflict codelens while a prior accept edit is still processing applies the second edit at stale line positions, corrupting the document.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/107821 which tracks the same stale-range race condition with more community signal. The fix likely involves either re-resolving conflict positions at edit time in `commandHandler.ts` (checking the conflict still exists at the stored range before applying), or disabling/invalidating other codelens items immediately when one is clicked.
