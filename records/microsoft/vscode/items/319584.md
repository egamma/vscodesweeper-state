# Issue #319584 — Incorrect indent line highlighted when selected line has children

[https://github.com/microsoft/vscode/issues/319584](https://github.com/microsoft/vscode/issues/319584)

## Summary

The reported behavior is intentional — introduced in commit b0b2a0cf160 (fixing #76276) to highlight an expanded node's own indent guide. However, the resulting inconsistency between siblings (expanded vs. leaf) is a valid polish concern worth routing to the tree/list area.

## Change summary

Selecting a tree item with expanded children highlights a different indent guide than selecting a sibling leaf at the same depth, creating visual inconsistency.

## Best solution

Revisit the heuristic in `_onDidChangeActiveNodes` (src/vs/base/browser/ui/tree/abstractTree.ts ~line 574) to either always highlight the parent's guide for all siblings at the same level, or find a middle ground that preserves the expanded-folder scope hint without confusing the indent-level signal. The area owner should decide which guide semantics are most useful.
