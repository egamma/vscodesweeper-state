# Issue #172072 — Source control buttons should not shift after calculating errors/problems

[https://github.com/microsoft/vscode/issues/172072](https://github.com/microsoft/vscode/issues/172072)

## Summary

Still a valid UX concern on current main — async file decorations in the SCM view can still cause action button layout shifts. The `ResourceRenderer` in `scmViewPane.ts` appends the `decorationIcon` element and file-label decorations without reserving fixed space, so late-arriving decorations push the toolbar. Team members discussed solutions (reserving space, hiding decorations on hover) but no fix was landed. Keeping open as an unresolved polish item under discussion.

## Change summary

Source control action buttons shift position when async file decorations (problem counts) load, causing accidental misclicks.

## Best solution

In `src/vs/workbench/contrib/scm/browser/scmViewPane.ts`, either reserve fixed space for decorations in the `ResourceRenderer` template, or adopt the Search view pattern mentioned by joaomoreno — hide decorations when showing inline actions on hover. The latter avoids reserving variable-width space.
