# Issue #228781 — `code serve-web` ignores the `--enable-proposed-api` flag

[https://github.com/microsoft/vscode/issues/228781](https://github.com/microsoft/vscode/issues/228781)

## Summary

Still a valid gap on main: the server CLI accepts `--enable-proposed-api` but `webClientServer.ts` never forwards it to the browser workbench environment, so the flag is silently ignored in `serve-web` mode. The issue is correctly triaged with `help wanted`.

## Change summary

The `--enable-proposed-api` CLI flag is accepted by `code serve-web` but never forwarded to the browser workbench, so extensions cannot use proposed APIs in the web IDE.

## Best solution

Wire `--enable-proposed-api` from `ServerParsedArgs` through `webClientServer.ts` into the workbench construction options or the URL payload so that the browser environment service can read it and populate `extensionEnabledProposedApi`. Likely files: `src/vs/server/node/webClientServer.ts` and `src/vs/workbench/services/environment/browser/environmentService.ts`.
