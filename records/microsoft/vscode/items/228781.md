# Issue #228781 — `code serve-web` ignores the `--enable-proposed-api` flag

[https://github.com/microsoft/vscode/issues/228781](https://github.com/microsoft/vscode/issues/228781)

## Summary

Still reproduces on main: the `--enable-proposed-api` flag is accepted by `serve-web` (declared in serverEnvironmentService.ts) but never wired through to the browser workbench environment service or the server-side extension scanner. An open PR (#310207) addresses this but has not been merged.

## Change summary

The `code serve-web` command accepts `--enable-proposed-api` but does not propagate it to the workbench, so extensions with proposed API declarations fail to activate in the web client.

## Best solution

Wire the parsed `enable-proposed-api` args from the server environment through to `IWorkbenchConstructionOptions` (in webClientServer.ts) and into the server extension scanner so extensions' `enabledApiProposals` are preserved — matching desktop behavior. PR https://github.com/microsoft/vscode/pull/310207 implements this approach.
