## Summary

Valid feature request with strong community support (52 upvotes, already promoted to Backlog). The 100KB limit is enforced server-side (HTTP 413); increasing it requires a backend change to the Settings Sync service, not just a client code modification.

## Change summary

Increase the Settings Sync file size limit for keybindings.json from 100KB to 200KB to accommodate power users with large keybinding configurations from modal editing extensions.

## Best solution

Coordinate with the Settings Sync service team to raise the per-resource payload limit (currently 100KB, triggering HTTP 413) for keybindings.json. The client-side error handling is in `src/vs/workbench/contrib/userDataSync/browser/userDataSync.ts` and `src/vs/platform/userDataSync/common/userDataSyncStoreService.ts`, but the actual enforcement is server-side.
