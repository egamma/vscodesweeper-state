# Issue #167414 — Git - pull rebase is stuck after merge conflict using submodules

[https://github.com/microsoft/vscode/issues/167414](https://github.com/microsoft/vscode/issues/167414)

## Summary

Still reproduces on main: getRebaseCommit, isMergeInProgress, and isCherryPickInProgress all hardcode `path.join(root, '.git', ...)` instead of using the already-available `this.repository.dotGit.path`, which correctly resolves submodule gitdir paths. This breaks rebase/merge/cherry-pick detection for submodule repositories.

## Change summary

Git pull rebase gets stuck after a merge conflict in submodules because rebase/merge/cherry-pick state files are looked up at the wrong path.

## Best solution

Replace the hardcoded `path.join(this.repository.root, '.git', ...)` in `getRebaseCommit`, `isMergeInProgress`, and `isCherryPickInProgress` with `this.repository.dotGit.path` (or `this.dotGit.path`), which already handles submodule gitdir resolution via `IDotGit`. The `dotGit` property is available on the Repository class at line 873.
