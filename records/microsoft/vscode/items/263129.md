# Issue #263129 — Easy "#selection" addition in Chat

[https://github.com/microsoft/vscode/issues/263129](https://github.com/microsoft/vscode/issues/263129)

## Summary

Valid UX polish request for easier selection attachment in chat. A team member has already responded with a workaround (right-click → add selection to chat) and noted ongoing implicit-context improvements. The implicit context system on main already auto-tracks editor selections (`src/vs/workbench/contrib/chat/browser/attachments/chatImplicitContext.ts`), which partially addresses the friction. Keeping open as a polish item for the chat team to evaluate alongside their context-flow work.

## Change summary

Make it easier to attach editor selections as chat context, avoiding the cumbersome `#selection` autocomplete and repeated switching between selection and text input.

## Best solution

Evaluate whether the current implicit-context auto-attachment of selections (already on main) and the right-click action sufficiently address this, and if not, improve the `#selection` variable autocomplete ranking or add a dedicated toolbar button in the chat input. The chat team's ongoing implicit-context work may fully subsume this.
