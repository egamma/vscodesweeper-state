# Issue #287097 — Search does not work in workspaces having names

[https://github.com/microsoft/vscode/issues/287097](https://github.com/microsoft/vscode/issues/287097)

## Summary

Confirmed on current main: `resolveResourcesForSearchIncludes` uses the workspace folder's display `name` (which can be a user-customized string with commas, parens, etc.) to construct the 'files to include' path, and `splitGlobPattern` then splits that path on commas, breaking the search. The bug is still present.

## Change summary

"Search in folder" in multi-root workspaces uses the folder's custom display name instead of the actual directory name, causing search to fail when the name contains commas or other special characters.

## Best solution

In `resolveResourcesForSearchIncludes` (src/vs/workbench/services/search/common/queryBuilder.ts, lines 730–738), replace `owningFolder.name` with the URI basename (`path.basename(owningFolder.uri.path)`) for constructing the search include path. The uniqueness check on line 732 should compare the URI basenames as well. `expandOneSearchPath` (line 519) already matches by `folder.name`, so it would also need to be updated to match by URI basename, or the include paths need commas escaped in a way `splitGlobAware` respects (e.g. wrapping in `{}`). Alternatively, pass the resolved URIs directly instead of stringified paths.
