# Issue #149048 — White bar appears when using external keyboard on iPad

[https://github.com/microsoft/vscode/issues/149048](https://github.com/microsoft/vscode/issues/149048)

## Summary

Confirmed upstream WebKit bug — the root cause is WebKit's incorrect `visualViewport` reporting on iPadOS with external keyboards; the team filed https://bugs.webkit.org/show_bug.cgi?id=247410 and is waiting on Apple. Issue is correctly labeled and triaged, should stay open as the tracking item on the VS Code side.

## Change summary

A white bar appears at the bottom of the VS Code web UI on iPad when an external keyboard is connected, caused by WebKit's incorrect visualViewport height calculation.

## Best solution

The fix belongs in WebKit (bug 247410). VS Code uses `window.visualViewport.height` on iOS which returns an incorrect value when the floating keyboard bar is shown with an external keyboard. Until Apple ships a fix, this remains a known limitation. A possible interim mitigation would be matching the bar color to the workbench theme background, as Tyler suggested in comment 15, but the real resolution is upstream.
