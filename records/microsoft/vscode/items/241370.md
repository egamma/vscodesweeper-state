# Issue #241370 — Incorrect indentation after C multi-line comment blocks

[https://github.com/microsoft/vscode/issues/241370](https://github.com/microsoft/vscode/issues/241370)

## Summary

Confirmed bug acknowledged by maintainer aiday-mar. The pasting aspect was partially fixed (confirmed by users on 1.104.x), but the typing/Enter-key indentation issue after multi-line C comment blocks persists on main. The issue is labeled tree-sitter-autoindent-candidate, indicating it's tracked for a broader fix.

## Change summary

Pressing Enter after a C multi-line comment block (`/* ... */`) incorrectly indents the new line by one space instead of returning to column 0.

## Best solution

Fix the indentation logic in `src/vs/editor/common/languages/autoIndent.ts` to not inherit the leading whitespace from the closing `*/` line of a multi-line comment when computing indent for the next line. The tree-sitter-based auto-indent approach (as tagged) may be the long-term fix path.
