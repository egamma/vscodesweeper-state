# Issue #136645 — [json] Explorer Outline view shows wrong int values when viewing JSON document

[https://github.com/microsoft/vscode/issues/136645](https://github.com/microsoft/vscode/issues/136645)

## Summary

Still unfixed on main — the JSON language service converts large integers to JavaScript Number (losing precision beyond MAX_SAFE_INTEGER) when building document symbol labels for the Outline view. Team member acknowledged the bug and suggested a fix path, but no fix has landed.

## Change summary

JSON Outline view displays incorrect values for integers exceeding Number.MAX_SAFE_INTEGER because the language service round-trips them through JavaScript Number.

## Best solution

Fix in vscode-json-languageservice (microsoft/vscode-json-languageservice) in `src/services/jsonDocumentSymbols.ts`: use the raw text from the document for number node labels instead of parsing via `getNodeValue()` which goes through `JSON.parse`/Number conversion. Then bump the dependency in extensions/json-language-features/server/package.json.
