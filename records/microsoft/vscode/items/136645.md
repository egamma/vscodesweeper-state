# Issue #136645 — [json] Explorer Outline view shows wrong int values when viewing JSON document

[https://github.com/microsoft/vscode/issues/136645](https://github.com/microsoft/vscode/issues/136645)

## Summary

Still reproduces on main. The upstream `vscode-json-languageservice` converts large integer values through JavaScript `number` (losing precision beyond MAX_SAFE_INTEGER) before displaying them in the Outline. A team member acknowledged the fix approach in 2021 but it hasn't been implemented.

## Change summary

JSON Outline view displays incorrect values for integers exceeding Number.MAX_SAFE_INTEGER because the language service parses them as JS numbers then re-stringifies.

## Best solution

Fix in `vscode-json-languageservice` (`src/services/jsonDocumentSymbols.ts`): in `getDetail()`, when `node.type === 'number'`, extract the raw text from the document (using offset/length) instead of `String(node.value)`. Same for `getName()` which calls `Parser.getNodeValue`. Then bump the dependency in VS Code.
