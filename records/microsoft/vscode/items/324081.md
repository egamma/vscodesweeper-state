# Issue #324081 — Agent window SSH does not work well with keys

[https://github.com/microsoft/vscode/issues/324081](https://github.com/microsoft/vscode/issues/324081)

## Summary

Team-authored issue (MEMBER) — must stay open. SSH key-based authentication fails in the Agents window while working fine in normal VS Code remote SSH. The shared-process logs show only keyboard-interactive being tried, suggesting publickey/agent auth attempts are either not being built or failing silently before being logged. No fixing commit found on main.

## Change summary

SSH connections in the Agents window fail when using SSH keys, even though normal VS Code SSH remoting works with the same keys and config.

## Best solution

Debug why `_buildAuthAttempts` in `src/vs/platform/agentHost/node/sshRemoteAgentHostService.ts` produces only a `keyboard-interactive` attempt instead of the expected `agent` + `publickey` attempts. The log line at L1048 shows what was built — the reporter's log says only keyboard-interactive was produced, which means either `config.authMethod` isn't `SSHAuthMethod.Agent`, the agent socket lookup failed silently, or all key file reads returned nothing. Adding trace-level logging for the agent socket and key-file resolution paths on Windows would help diagnose.
