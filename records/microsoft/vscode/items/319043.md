# Issue #319043 — [Copilot Sessions] Claude Code CLI sessions incorrectly marked as "Failed" after successful completion

[https://github.com/microsoft/vscode/issues/319043](https://github.com/microsoft/vscode/issues/319043)

## Summary

Confirmed the defect path in current source: when Claude Code's handleRequest throws (even for recoverable SDK errors), it returns errorDetails which causes chatSessions.contribution.ts to unconditionally map the session to Failed status. No fix found on main.

## Change summary

Claude Code sessions are marked as 'Failed' in the Sessions panel even when the underlying Claude process completed successfully, because recoverable internal errors surface errorDetails that the status mapping treats as terminal failures.

## Best solution

The fix likely belongs in two places: (1) `extensions/copilot/src/extension/chatSessions/claude/node/claudeCodeAgent.ts` should distinguish recoverable errors from true session failures, only returning `errorDetails` for non-recoverable cases; (2) optionally, the status derivation in `src/vs/workbench/contrib/chat/browser/chatSessions/chatSessions.contribution.ts` (line ~1678) could consider an `errorDetails` severity or recoverable flag rather than treating all errorDetails as Failed.
