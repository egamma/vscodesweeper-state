# Issue #280819 — No vertical scrollbar for longer terminal outputs in panel chat

[https://github.com/microsoft/vscode/issues/280819](https://github.com/microsoft/vscode/issues/280819)

## Summary

Still reproduces on main: the xterm.js vertical scrollbar is explicitly hidden via CSS (`display: none` at chatTerminalToolProgressPart.css:289-291) due to a technical limitation where the scrollbar mispositions during horizontal scroll. The issue is team-authored and must stay open for maintainer judgment.

## Change summary

Terminal output in panel chat lacks a vertical scrollbar, making it hard to know scroll position or extent of output.

## Best solution

Requires upstream xterm.js API to control the scrollable viewport size independently of rows/cols (as Tyriar noted). Once that API lands, remove the `display: none` on `.xterm-scrollbar` and wire up the vertical scrollbar properly. Alternatively, a custom scrollbar overlay could be implemented outside xterm's own scrollbar.
