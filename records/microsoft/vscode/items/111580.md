# Issue #111580 — Align upload & download progress between desktop and web

[https://github.com/microsoft/vscode/issues/111580](https://github.com/microsoft/vscode/issues/111580)

## Summary

The gap is real and unresolved on current main: native (desktop) download/upload progress still only reports the file name, with no byte-level or percentage progress. The issue carries the 'On Deck' milestone, which is a scheduling decision — keeping open per policy.

## Change summary

Desktop upload/download progress in the file explorer shows only a file name, while the web version shows detailed byte-level progress with percentages.

## Best solution

Add byte-level progress reporting to the file service layer (likely `doCopyFile`/pipe helpers in `src/vs/platform/files/common/fileService.ts`) so that `doDownloadNative` and the native upload path in `src/vs/workbench/contrib/files/browser/fileImportExport.ts` can report transferred bytes against total size, matching the web experience. The cancel button for uploads should also be wired up (currently non-functional in remote scenarios). Community contributor JalinWang left detailed architecture notes in the comments that could guide implementation.
