# Issue #326244 — Agent window: steering messages pollute other chat forks

[https://github.com/microsoft/vscode/issues/326244](https://github.com/microsoft/vscode/issues/326244)

## Summary

Team-authored bug report — keeping open per policy. Steering messages (e.g. 'stop') sent in one chat fork incorrectly affect sibling forks in the Agents window.

## Change summary

Steering messages submitted in one forked chat session bleed into other forks, causing unintended interruption of parallel agent runs.

## Best solution

The steering/cancellation signal path in the agent host needs to be scoped to the specific fork (session/conversation branch) rather than broadcast to all active forks sharing the same root. Likely in the agent-host session routing or the chat widget's fork-management layer under src/vs/sessions or src/vs/platform/agentHost.
