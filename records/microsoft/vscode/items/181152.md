# Issue #181152 — Can't Highlight Text By Draggin Mouse

[https://github.com/microsoft/vscode/issues/181152](https://github.com/microsoft/vscode/issues/181152)

## Summary

Root cause identified by reporters as a hardware/driver issue: an Audio Technica AT2020usb+ microphone spams MediaTrackPrevious key events every 50ms, which disrupts text selection in all Chromium-based apps (not just VS Code). VS Code has no specific binding for MediaTrackPrevious that would deselect text — the behavior is at the OS/Chromium layer. The workaround is disabling the mic's USB Input Device.

## Change summary

Cannot highlight text by dragging mouse due to external USB microphone spamming MediaTrackPrevious key events every 50ms.

## Best solution

Close as not reproducible in VS Code itself — the root cause is a hardware driver issue (Audio Technica AT2020usb+ entering a bad state). VS Code does not bind MediaTrackPrevious to any selection-clearing action; the interference happens at the Chromium/OS input layer. The commenter's question about whether MediaTrackPrevious *should* deselect text is a Chromium/Electron concern, not a VS Code editor bug.
