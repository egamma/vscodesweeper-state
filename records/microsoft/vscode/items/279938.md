# Issue #279938 — Copying a chat response includes tool calls

[https://github.com/microsoft/vscode/issues/279938](https://github.com/microsoft/vscode/issues/279938)

## Summary

Still present on main: the `toString()` method of chat responses includes raw markdown from tool invocation messages (e.g. `Read [](file:///…)`), which leaks into copy/paste, voice readout, and previews. The merged PR #279965 was only a workaround for agent session descriptions, not a fix for the core `toString()` path.

## Change summary

Copying a chat response (or using it in voice/preview features) includes raw markdown link syntax from tool invocation messages instead of plain text.

## Best solution

In `getToolInvocationText` (src/vs/workbench/contrib/chat/common/model/chatModel.ts ~line 703), strip markdown link syntax from `invocationMessage`/`pastTenseMessage` values before including them in the text representation — e.g. convert `Read [](file:///path)` to `Read path` or just the label text. This affects copy, voice, and preview surfaces.
