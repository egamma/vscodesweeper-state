# Issue #267911 — Allow customizing "Sync Changes" behavior to exclude git push

[https://github.com/microsoft/vscode/issues/267911](https://github.com/microsoft/vscode/issues/267911)

## Summary

Valid feature request to make the Sync Changes button configurable (pull-only vs pull+push). The current sync implementation in `extensions/git/src/repository.ts` always does pull then push. Low community traction (1 upvote) — keeping open for the community process.

## Change summary

Allow users to configure the Sync Changes button to perform only pull (without push), instead of the hardcoded pull+push sequence.

## Best solution

Add a `git.syncAction` setting (or extend the existing sync button configuration) in `extensions/git` to let users choose between pull-only, fetch+pull, or pull+push (default). The implementation would touch `extensions/git/src/repository.ts` (_sync method) and `extensions/git/src/actionButton.ts` (button label/command). The commenter's suggestion of a dropdown menu (like the Commit button) is also worth considering as an alternative UX.
