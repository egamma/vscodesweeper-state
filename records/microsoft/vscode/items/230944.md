# Issue #230944 — `files.restoreUndoStack` should take effect immediately

[https://github.com/microsoft/vscode/issues/230944](https://github.com/microsoft/vscode/issues/230944)

## Summary

Team-authored feature request (MEMBER). The setting is read at model-dispose time but toggling it off doesn't clear already-stored undo stacks — the request is for immediate effect (purge stored stacks on change). Still valid on current main; no fix found.

## Change summary

Changing `files.restoreUndoStack` to false should immediately discard any already-stored undo/redo stacks rather than only taking effect on future file closes.

## Best solution

Listen for configuration changes on `files.restoreUndoStack` in `ModelService` and, when toggled to `false`, iterate over stored undo/redo elements for disposed models and remove them. The work lives in `src/vs/editor/common/services/modelService.ts`.
