# Issue #281154 — Unexpected conversion to `string` from `markdown` for `label`

[https://github.com/microsoft/vscode/issues/281154](https://github.com/microsoft/vscode/issues/281154)

## Summary

Still open on main: `IChatSessionListItem.label` is typed `string` but the viewer at line 329 wraps it in `new MarkdownString(...)` then calls `renderAsPlaintext()`, suggesting `label` can actually be markdown. The type should be `string | IMarkdownString` to match `description`/`tooltip`/`badge`, and the same plaintext-conversion guard should apply to `description`.

## Change summary

The `label` field on `IChatSessionListItem` is typed `string` but handled as if it could be markdown; there is also a missing markdown-to-plaintext guard for `description`.

## Best solution

Widen `IChatSessionListItem.label` to `string | IMarkdownString` (matching `description`, `tooltip`, `badge` on the same interface), and ensure every consumer that renders `label` or `description` as plain text uses `renderAsPlaintext` when the value is `IMarkdownString`. The changes are localized to `chatSessionsService.ts` (interface) and `agentSessionsViewer.ts` (consumers).
