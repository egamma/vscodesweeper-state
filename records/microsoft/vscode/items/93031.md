# Issue #93031 — Replace in search Editor

[https://github.com/microsoft/vscode/issues/93031](https://github.com/microsoft/vscode/issues/93031)

## Summary

This is a well-supported feature request (118 upvotes) for replace functionality in the Search Editor, already promoted to the Backlog. The feature remains unimplemented — the search editor widget explicitly sets `_hideReplaceToggle: true`. It should stay open.

## Change summary

Add replace-in-files functionality to the Search Editor so users can perform replacements directly from search results.

## Best solution

Implement replace support in the Search Editor (`src/vs/workbench/contrib/searchEditor/browser/searchEditor.ts`) by removing the `_hideReplaceToggle: true` constraint and wiring up replace actions that apply text changes back to the source files. This is a non-trivial feature involving diff tracking and multi-file writes — the existing extension (search-editor-apply-changes) demonstrates feasibility but also the edge cases involved.
