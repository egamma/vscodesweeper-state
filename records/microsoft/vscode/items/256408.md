## Summary

Team member deepak1556 confirmed this is a known issue — the ExternalLocation in the AppX installer hasn't been adjusted, producing incorrect logo paths in the manifest. He stated it will be addressed as part of #254613, which tracks the broader package-identity work and is still open.

## Change summary

The AppX manifest's logo paths resolve incorrectly because the `-ExternalLocation` points to the `appx` subfolder while the logo files live one level up.
