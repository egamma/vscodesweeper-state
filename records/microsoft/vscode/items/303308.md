# Issue #303308 — Test results panel vertical/horizontal splitting

[https://github.com/microsoft/vscode/issues/303308](https://github.com/microsoft/vscode/issues/303308)

## Summary

Valid feature request with 24 upvotes, promoted to Backlog. The test results panel's SplitView is hardcoded to Orientation.HORIZONTAL (side-by-side) with no vertical (top-bottom) option — the requested capability remains unimplemented on current main.

## Change summary

Allow the test results panel split to be toggled between horizontal (side-by-side) and vertical (top-bottom) orientation, especially useful when the panel is in a narrow sidebar.

## Best solution

Extend `TestingResultsViewLayout` in `src/vs/workbench/contrib/testing/common/configuration.ts` to include vertical orientation options, and update `testResultsViewContent.ts` to pass `Orientation.VERTICAL` to the SplitView when configured. The existing toggle action could cycle through the new layouts or a separate toggle could be added.
