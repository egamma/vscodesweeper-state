## Summary

Still reproduces on current main: `onDidRemoveRepository` is an async handler that calls `updateChildrenThrottler.queue()` without checking whether the view pane (and thus the throttler) is already disposed. No fix has landed.

## Change summary

SCM Repositories view throws 'Throttler is disposed' when a repository removal event fires after the view pane is disposed.

## Best solution

Add a disposal guard in `updateChildren()` (or `onDidRemoveRepository`) to bail out early when `this._store.isDisposed` is true, preventing the throttler from being queued post-disposal. The fix is a one-line guard in `scmRepositoriesViewPane.ts`.
