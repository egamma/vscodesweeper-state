## Summary

Team-authored issue (MEMBER) reporting a UX conflict where both an MCP app card and a tool approval prompt appear simultaneously when filing a GitHub issue from chat. Keeping open for maintainer judgment.

## Change summary

When creating a GitHub issue from chat, both the MCP app UI and a separate tool approval prompt for `GitHub-issue_write` are shown at the same time, creating a confusing duplicate flow.

## Best solution

The tool approval prompt should be suppressed or consolidated when an MCP app card is already handling the user-facing confirmation for the same action. The fix likely lives in the chat tool-approval orchestration layer — either the MCP app flow should implicitly satisfy the tool approval gate, or the approval prompt should detect that an MCP app card is already providing the confirmation UI and skip rendering.
