# Issue #116303 — Search all files (ctrl+shift+f) doesn't find results in files with very long filepaths unless opened in the editor

[https://github.com/microsoft/vscode/issues/116303](https://github.com/microsoft/vscode/issues/116303)

## Summary

Still a valid upstream issue: ripgrep does not handle Windows paths exceeding MAX_PATH (260 chars), and VS Code's search service has no workaround. The issue is correctly labeled upstream/confirmed/help wanted and sits in Backlog — it should stay open.

## Change summary

Search across files (Ctrl+Shift+F) fails to find results in files whose full path exceeds 260 characters on Windows, unless the file is already open in the editor.

## Best solution

The root fix belongs in ripgrep — it needs to use the extended-length path prefix (`\\?\`) on Windows to bypass the MAX_PATH limit. A VS Code-side workaround could prepend `\\?\` to paths passed to ripgrep, but this would live in `src/vs/workbench/services/search/node/ripgrepTextSearchEngine.ts`. The `help wanted` label is appropriate; a community PR addressing either the ripgrep upstream or the VS Code path-prefix workaround would resolve this.
