# Issue #116303 — Search all files (ctrl+shift+f) doesn't find results in files with very long filepaths unless opened in the editor

[https://github.com/microsoft/vscode/issues/116303](https://github.com/microsoft/vscode/issues/116303)

## Summary

Confirmed upstream bug in ripgrep's Windows long-path handling, already well-triaged with correct labels. The upstream ripgrep fix (BurntSushi/ripgrep#364) landed in ripgrep's source but it's unclear whether the version bundled via @vscode/ripgrep includes it. Keeping open until the fix is confirmed shipped.

## Change summary

Search across files (Ctrl+Shift+F) fails to find results in files whose full path exceeds Windows' MAX_PATH (260 chars), unless the file is already open in the editor.

## Best solution

Verify whether the current @vscode/ripgrep package (v1.15.11, wrapping ripgrep-universal v1.18.0) includes the long-path fix from https://github.com/BurntSushi/ripgrep/issues/364. If not, update the bundled ripgrep binary to a version that includes it. If the fix is already present, request a community member to re-test on a current Insiders build and close if resolved.
