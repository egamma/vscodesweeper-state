# Issue #311849 — VS Code Integrated Terminal Fails to Launch (ConPTY / winpty error in corporate VM)

[https://github.com/microsoft/vscode/issues/311849](https://github.com/microsoft/vscode/issues/311849)

## Summary

The report is missing critical information (VS Code version, Windows OS version/build number) needed to triage this ConPTY failure. The error is a known consequence of winpty removal in VS Code 1.109+ on older Windows builds (e.g. 10.0.14393 / Server 2016) where ConPTY doesn't function properly. Without the OS version, it's impossible to confirm whether this is a duplicate of the cluster of existing ConPTY issues on older Windows, or a distinct problem on a supported OS version with restrictive corporate policies.

## Change summary

VS Code integrated terminal fails to launch with 'Cannot launch conpty / Winpty has been removed' error on a corporate-managed Windows VM.

## Best solution

Request the reporter's exact Windows build number (`winver` or `[System.Environment]::OSVersion`) and VS Code version. If the Windows build is older than 17763 (Windows 10 1809), this is a known limitation after winpty removal — the reporter needs to either update Windows or use the `terminal.integrated.windowsUseConptyDll` setting (which the reporter claims to have tried). If on a supported Windows build, this may indicate a corporate policy blocking ConPTY — that would be a distinct investigation for the terminal team.
