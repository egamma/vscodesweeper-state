# Issue #252509 — Search only in Open Editors issue

[https://github.com/microsoft/vscode/issues/252509](https://github.com/microsoft/vscode/issues/252509)

## Summary

Still present on main: when 'Search only in Open Editors' collects editor URIs, it includes cross-OS files like settings.json opened from Windows in WSL. The URI's fsPath (`/C:/Users/...`) passes `isAbsolutePath` but doesn't exist on the Linux filesystem, causing ripgrep to fail with ENOENT and aborting the entire search.

## Change summary

'Search only in Open Editors' fails with ENOENT in WSL when settings.json (a Windows-side file) is open, because the invalid cross-OS path is passed to ripgrep.

## Best solution

In `QueryBuilder.commonQueryFromFileList()`, filter out URIs whose scheme is not `file` (or whose fsPath doesn't exist on the remote OS) before constructing folder queries for ripgrep. This prevents cross-OS paths from poisoning the search. Alternatively, the search service should gracefully handle ENOENT from ripgrep per-file rather than aborting the entire search.
