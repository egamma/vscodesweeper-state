# Issue #252509 — Search only in Open Editors issue

[https://github.com/microsoft/vscode/issues/252509](https://github.com/microsoft/vscode/issues/252509)

## Summary

Real bug: 'Search only in Open Editors' fails with ENOENT in WSL when settings.json (a vscode-userdata URI with a Windows path) is open — the query builder includes URIs with non-file schemes whose paths don't exist on the remote filesystem.

## Change summary

'Search only in Open Editors' fails with ENOENT in WSL when a vscode-userdata file (like settings.json) is open, because its Windows path is passed to ripgrep running on Linux.

## Best solution

In `commonQueryFromFileList` (src/vs/workbench/services/search/common/queryBuilder.ts, ~line 308), filter out URIs whose scheme isn't backed by a searchable filesystem provider in the current context — at minimum skip non-file, non-vscode-remote schemes the way `walkThrough` is already skipped, or check `fileService.hasProvider` before adding the folder query. The `getOpenEditorResults` path in searchService.ts already does this filtering (line 534) for the in-memory search, but the ripgrep folder-query path does not.
