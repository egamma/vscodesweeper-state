# Issue #307534 — Fix: Windows 11 Modern Context Menu Entry Missing due to 32-bit/64-bit Architecture Mismatch

[https://github.com/microsoft/vscode/issues/307534](https://github.com/microsoft/vscode/issues/307534)

## Summary

Verified duplicate of #304214, which is the canonical open issue tracking the missing 'Open with Code' Windows 11 context menu entry. The root cause (x64 DLL actually compiled as x86) is confirmed upstream in microsoft/vscode-explorer-command#26, where a maintainer has acknowledged it. Consolidating discussion there.

## Change summary

The 'Open with Code' entry is missing from the Windows 11 modern context menu because code_explorer_command_x64.dll is compiled as x86, which 64-bit explorer.exe cannot load.

## Best solution

Fix the build configuration in the microsoft/vscode-explorer-command repo (the TargetMachine msvs_settings placement issue identified in vscode-explorer-command#26), publish a corrected x64 DLL release, then update the checksums in build/checksums/explorer-dll.txt and the version tag in build/win32/explorer-dll-fetcher.ts. Track this through https://github.com/microsoft/vscode/issues/304214.
