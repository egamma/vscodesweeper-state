# Issue #277186 — Instructions keyboard shortcut doesn't work on windows

[https://github.com/microsoft/vscode/issues/277186](https://github.com/microsoft/vscode/issues/277186)

## Summary

Team-authored issue — must stay open. The attach context picker shows a keyboard shortcut for the Instructions item via `commandId`, but the referenced action (`workbench.action.chat.attach.instructions`) is never registered, so the shortcut either shows incorrectly or doesn't work. The fix is to either remove the `commandId` from `ChatInstructionsPickerPick` or stop displaying the keybinding label for unregistered commands.

## Change summary

The Instructions entry in the attach context picker displays a non-functional keyboard shortcut on Windows.

## Best solution

Remove the `commandId` property from `ChatInstructionsPickerPick` in `src/vs/workbench/contrib/chat/browser/promptSyntax/attachInstructionsAction.ts` (line 90) so the context picker no longer renders a keybinding label for it, or alternatively guard the keybinding display in `chatContextActions.ts` to skip unregistered commands.
