# Issue #295949 — Retain user model selection

[https://github.com/microsoft/vscode/issues/295949](https://github.com/microsoft/vscode/issues/295949)

## Summary

The core model-selection persistence mechanism exists and has received multiple edge-case fixes (PR #320036, commit c53722ab694), but reporters still describe cases where the selection reverts to Auto. With 29 upvotes, continued community reports, and a maintainer acknowledging this is a real bug (comment 10: 'any cases that lead to it swapping and racking up usage is a bug'), this should stay open for the team to investigate remaining edge cases.

## Change summary

User-selected model in Copilot Chat reverts to Auto mode after VS Code restart or under certain runtime conditions, forcing users to re-select their preferred model.

## Best solution

Investigate remaining edge cases where model persistence fails — the storage-backed restore logic in `chatInputPart.ts` (`shouldRestorePersistedModel`) can fail when models aren't available at restore time or when extensions alter the endpoint list. The late-arriving model path exists but may not cover all scenarios (e.g. extension-provided models loading after the fallback fires). The `chat` / `model-picker` area team should trace the remaining failure paths.
