# Issue #246719 — Copying with empty selection clears the clipboard when editor.emptySelectionClipboard=false in Web mode

[https://github.com/microsoft/vscode/issues/246719](https://github.com/microsoft/vscode/issues/246719)

## Summary

Still reproduces on main: when `emptySelectionClipboard=false` and the selection is empty, the web copy event handler unconditionally calls `ensureClipboardGetsEditorData()`, which `preventDefault()`s and writes empty text to the clipboard, clearing it. Desktop avoids this because its copy action has an early-return guard.

## Change summary

Copying with an empty selection in web mode clears the clipboard instead of leaving it unchanged when `editor.emptySelectionClipboard` is false.

## Best solution

Guard the copy event handlers in both `textAreaEditContextInput.ts` and `nativeEditContext.ts` to skip `ensureClipboardGetsEditorData()` (and not call `e.preventDefault()`) when `emptySelectionClipboard` is false and the selection is empty — matching the desktop behavior where the action returns early.
