# Issue #273768 — missing required Copilot-Vision-Request

[https://github.com/microsoft/vscode/issues/273768](https://github.com/microsoft/vscode/issues/273768)

## Summary

This is a feature request for more granular enterprise policy controls — separating vision/image upload capability from broader Editor Preview settings that also enable remote MCP servers. The underlying error is by design (vision requires the header, which requires the preview feature), so the real ask is decoupling these capabilities. Multiple commenters confirm they hit this limitation. Issue #253136 (the meta request-failed tracker) is not a true canonical for this specific enterprise-policy granularity concern, and #273766 was closed as not-planned by the bot based on that mismatch.

## Change summary

MCP tools that upload images (Playwright, Figma) fail with 'missing required Copilot-Vision-Request header' unless Editor Preview options are enabled, but enabling those also opens remote MCP access that enterprise admins want to block.

## Best solution

Introduce finer-grained enterprise policy controls that allow enabling vision/image uploads independently of remote MCP server access, likely in the Copilot extension's configuration surface under `chat-vision` or enterprise policy settings. This aligns with the unified settings and enterprise management work.
