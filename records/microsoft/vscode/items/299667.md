# Issue #299667 — "bypass approvals" should be per-session

[https://github.com/microsoft/vscode/issues/299667](https://github.com/microsoft/vscode/issues/299667)

## Summary

Team-authored issue (MEMBER) — must stay open for maintainer judgment. The report identifies that the 'bypass approvals' dropdown state is global to the chat widget rather than scoped per-session, so switching sessions silently inherits approval bypass from the previous session.

## Change summary

The 'bypass approvals' mode leaks across sessions in the same chat widget because the dropdown controlling it is global rather than per-session.

## Best solution

Make the approval-bypass state per-session so that switching sessions in the chat widget resets or restores the correct approval mode for each session. The comment from justschen indicates partial work is already in place (new sessions default correctly when the global setting is off), but the dropdown UI state still bleeds across session switches.
