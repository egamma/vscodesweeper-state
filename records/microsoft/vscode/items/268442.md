# Issue #268442 — Weird bug when closing the auto quotation double quotes in any HTML file

[https://github.com/microsoft/vscode/issues/268442](https://github.com/microsoft/vscode/issues/268442)

## Summary

Still reproduces on main: quotes inserted by completion items are not tracked in `autoClosedCharacters`, so `autoClosingOvertype: 'auto'` fails to overtype them. The overtype logic at line 1004–1016 of cursorTypeEditOperations.ts only checks ranges that were auto-closed by the editor, not by completions.

## Change summary

Typing a closing double-quote inside quotes inserted by an HTML attribute completion adds a duplicate quote instead of moving the cursor past the existing one.

## Best solution

Extend the auto-closed character tracking so that quotes inserted by completion items (e.g. `lang=""`) are registered in `autoClosedCharacters`, or teach the overtype logic to handle completion-inserted pairs. The fix is in `src/vs/editor/common/cursor/cursorTypeEditOperations.ts` (the `isAutoClosingOvertype` function) and possibly the completion accept path that inserts snippet/pair characters.
