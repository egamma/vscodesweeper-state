# Issue #307704 — Cannot navigate to full list of deprecated extensions via Tab key

[https://github.com/microsoft/vscode/issues/307704](https://github.com/microsoft/vscode/issues/307704)

## Summary

Team-authored accessibility bug about Tab key navigation in the deprecated extensions section. A MEMBER commenter suggests arrow keys are the expected pattern here, which may indicate this is as-designed list/tree behavior — but the author hasn't responded yet, so keeping open for clarification.

## Change summary

Tab key only focuses the first deprecated extension in the extensions view; subsequent Tab presses skip remaining deprecated extensions.

## Best solution

Clarify whether the deprecated extensions section uses a list/tree widget (where arrow keys navigate items and Tab moves between sections) or whether Tab navigation within the group is actually expected. If it's a tree, this may be as-designed per standard workbench list accessibility patterns. The relevant code is in `src/vs/workbench/contrib/extensions/browser/extensionsViews.ts` and `src/vs/workbench/contrib/extensions/browser/extensionsList.ts`.
