# Issue #257396 — Multiple Repos with git worktrees in worspace

[https://github.com/microsoft/vscode/issues/257396](https://github.com/microsoft/vscode/issues/257396)

## Summary

Fixed on main by commit a99073234c6 which resolves relative git worktree paths correctly; the old code assumed absolute paths in the gitdir content, causing the ENOENT errors reported here.

## Change summary

Git worktrees created with relative paths (--relative-paths) are not detected by VS Code's SCM view, producing ENOENT errors on realpath.

## Best solution

Keep the shipped fix in commit a99073234c6 — `parseGitWorktreePath` now resolves relative gitdir content against the gitdir file's directory before stripping the `.git` suffix. No further change needed.
