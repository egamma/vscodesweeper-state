# Issue #294596 — [FR] Allow callStackDown and callStackUp keyboard shortcuts to move through virtual environment

[https://github.com/microsoft/vscode/issues/294596](https://github.com/microsoft/vscode/issues/294596)

## Summary

Valid bug: `callStackUp`/`callStackDown` skip deemphasized frames via `findNextVisibleFrame` (line 315 of debugCommands.ts), but clicking those same frames in the call stack view works. When `justMyCode` is false, library frames are still marked `deemphasize` by the Python debug adapter and thus skipped by keyboard navigation — an inconsistency with mouse behavior.

## Change summary

Debug call stack keyboard shortcuts skip library/virtual-environment frames even when `justMyCode` is false, while mouse clicks on the same frames work.

## Best solution

Either make `findNextVisibleFrame` in `src/vs/workbench/contrib/debug/browser/debugCommands.ts` stop skipping deemphasized frames (possibly gated on a setting or only skip when the frame source is truly hidden), or coordinate with the Python debug adapter to not mark frames as deemphasized when `justMyCode` is false. The VS Code side fix would be in `findNextVisibleFrame` around line 315.
