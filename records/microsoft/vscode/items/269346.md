# Issue #269346 — [html] Incorrect onclick attribute Parsing

[https://github.com/microsoft/vscode/issues/269346](https://github.com/microsoft/vscode/issues/269346)

## Summary

Still reproduces on main: the embedded JavaScript extraction for event-handler attributes does not decode HTML entities (`&quot;`, `&#34;`, etc.), so the JS language service sees raw entity text and reports false syntax errors.

## Change summary

HTML event-handler attributes containing `&quot;` entities are flagged as invalid JavaScript because entities are not decoded before JS validation.

## Best solution

Extend the `updateContent` function in `extensions/html-language-features/server/src/modes/embeddedSupport.ts` to decode common HTML entities (`&quot;`/`&#34;`, `&amp;`/`&#38;`, `&apos;`/`&#39;`, `&lt;`/`&#60;`, `&gt;`/`&#62;`) for JavaScript attribute values, mirroring the existing CSS entity-replacement logic.
