# Issue #269346 — [html] Incorrect onclick attribute Parsing

[https://github.com/microsoft/vscode/issues/269346](https://github.com/microsoft/vscode/issues/269346)

## Summary

Still reproduces on main: the embedded JavaScript language service receives raw `&quot;` entities in event handler attributes because `updateContent` only decodes HTML entities for CSS, not JavaScript.

## Change summary

HTML event handler attributes containing `&quot;` entities are incorrectly flagged as invalid JavaScript syntax.

## Best solution

Extend the `updateContent` function in `extensions/html-language-features/server/src/modes/embeddedSupport.ts` to decode `&quot;`/`&#34;` (and potentially other common HTML entities like `&amp;`, `&lt;`, `&gt;`, `&apos;`/`&#39;`) when the embedded language is JavaScript and the region is an attribute value — mirroring the existing CSS entity-decoding logic from commit 4685d759641.
