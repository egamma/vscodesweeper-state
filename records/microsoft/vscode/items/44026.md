# Issue #44026 — [theming] customize file icons in settings

[https://github.com/microsoft/vscode/issues/44026](https://github.com/microsoft/vscode/issues/44026)

## Summary

Keep open as a popular Backlog feature request: the request still describes a core settings-level customization that current VS Code does not appear to provide. Existing support remains centered on selecting a file icon theme or defining associations inside an icon theme contribution.

## Change summary

Allow users to override file icon associations for specific file names, extensions, or language ids from settings without creating or maintaining a file icon theme extension.

## Best solution

The themes area should make an explicit product decision on whether `workbench.iconTheme` should support a user-level overlay for file icon associations, or whether VS Code should continue requiring icon theme extensions and theme-specific customization settings.
