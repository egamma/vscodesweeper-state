# Issue #44026 — [theming] customize file icons in settings

[https://github.com/microsoft/vscode/issues/44026](https://github.com/microsoft/vscode/issues/44026)

## Summary

Legitimate feature request to allow file icon customization via settings without needing a full extension. Not implemented on main; community interest spans years with varied use cases. Parked in Backlog — keep open as an active feature request.

## Change summary

Allow users to override or customize file/folder icons in settings.json without creating a dedicated icon theme extension.

## Best solution

Introduce a `workbench.iconCustomizations` setting (analogous to `workbench.colorCustomizations`) that lets users map file extensions, file names, or folder names to existing icon definitions from the active theme — or provide a path to a custom SVG. This would live in `src/vs/workbench/services/themes/browser/fileIconThemeData.ts` and the theme configuration contribution. Alternatively, accept that third-party themes (Material Icon Theme, Catppuccin Icons) already solve this via their own custom association settings, making this an extension-level concern rather than core.
