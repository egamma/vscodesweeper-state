# Issue #251392 — SCM - Flat Source Control View for Multi-Root Workspaces (Monorepo-Like Behavior)

[https://github.com/microsoft/vscode/issues/251392](https://github.com/microsoft/vscode/issues/251392)

## Summary

Niche but coherent feature request for a flat/unified SCM view across multi-root workspace repositories. Only 3 upvotes after 400+ days — low traction but no grounds to close; keeping open for the community process.

## Change summary

Add an optional flat view mode in the Source Control panel that shows a unified list of changed files across all repositories in a multi-root workspace, with a single commit message input and coordinated commit/push.

## Best solution

Leave for the community process to gauge demand. If traction grows, the SCM team would need to decide whether a unified cross-repo commit/push model belongs in core or is better served by an extension that wraps the SCM API. The multi-repo coordination (single commit message applied to multiple repos, coordinated push) is the hard design question — it changes the mental model of what a commit means.
