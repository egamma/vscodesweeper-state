## Change summary

Ctrl+X then Ctrl+V on a file in the explorer copies instead of moving it, on Windows 11 with WSL.

## Best solution

Ask the reporter to confirm: (1) the file explorer tree has focus when pressing Ctrl+X (the item should visually dim), (2) whether this happens with extensions disabled (`--disable-extensions`), and (3) whether this is in a WSL remote workspace or a local folder. The cut keybinding (`filesExplorer.cut`) requires `FilesExplorerFocusCondition` — if focus is elsewhere, the generic clipboard cut fires and the paste defaults to copy. If the explorer is confirmed focused and the item dims, this would be a real bug in `src/vs/workbench/contrib/files/browser/fileActions.ts`.
