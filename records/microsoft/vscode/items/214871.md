# Issue #214871 — [css] look for variables in all open documents

[https://github.com/microsoft/vscode/issues/214871](https://github.com/microsoft/vscode/issues/214871)

## Summary

Valid feature request acknowledged by maintainer aeschli, who indicated interest in looking at all open CSS editors for variable completions. Still not implemented on main — the CSS language server only processes the current document for variable proposals.

## Change summary

CSS IntelliSense should propose custom properties (variables) from all open CSS documents, not just the current file.

## Best solution

Extend the CSS language server (in `vscode-css-languageservice` or the server wrapper in `extensions/css-language-features/server/src/cssServer.ts`) to gather custom property declarations from all open CSS/SCSS/LESS documents when building completion proposals for `var(--…)`. This likely requires changes in the upstream `vscode-css-languageservice` package to accept an additional document provider.
