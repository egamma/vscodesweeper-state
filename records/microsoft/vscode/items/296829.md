## Summary

Valid feature request for per-agent auto-approve configuration in .agent.md frontmatter. Current auto-approve is global/workspace-wide or session-level; no per-agent granularity exists on main.

## Change summary

Allow specifying auto-approve permissions per custom agent via .agent.md frontmatter rather than relying on the global workspace-wide setting.

## Best solution

Add an `auto-approve` (or similar) frontmatter field to the .agent.md schema that scopes tool auto-approval to a specific custom agent, integrating with the existing permission/approval infrastructure in `src/vs/workbench/contrib/chat/common/constants.ts` and the prompt file attributes parser.
