# Issue #296829 — Per-agent auto-approve for terminal commands in custom agents (.agent.md)

[https://github.com/microsoft/vscode/issues/296829](https://github.com/microsoft/vscode/issues/296829)

## Summary

Legitimate feature request for per-agent auto-approve scoping in .agent.md frontmatter. Current auto-approve is global/workspace-wide via `chat.tools.terminal.autoApprove`; no per-agent override exists on main. Needs product discussion on the trust model implications.

## Change summary

Allow custom agents defined in .agent.md to declare per-agent terminal command auto-approve rules in their frontmatter, rather than relying solely on global/workspace-wide settings.

## Best solution

Add an optional `autoApprove` (or `tools.terminal.autoApprove`) field to the .agent.md frontmatter schema, scoped to the commands that agent is expected to run. This intersects with the existing `CommandAutoApprover` in `src/vs/platform/agentHost/node/commandAutoApprover.ts` and the terminal auto-approve configuration in `src/vs/workbench/contrib/terminalContrib/chatAgentTools/`. Needs a product/security decision on whether per-agent declarations should be able to *expand* approval beyond the user's workspace setting, or only *restrict* it.
