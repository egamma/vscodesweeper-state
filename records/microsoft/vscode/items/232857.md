# Issue #232857 — Incorrect scaling detection on Linux (GNOME)

[https://github.com/microsoft/vscode/issues/232857](https://github.com/microsoft/vscode/issues/232857)

## Summary

Verified duplicate: this reports incorrect multi-monitor scaling on Linux/GNOME under X11/XWayland, which is the same Electron/Chromium upstream limitation tracked by the long-standing canonical issue #41345 (Heterogeneous DPI support on Linux). Both issues share the same root cause and the same workaround (--ozone-platform=wayland).

## Change summary

VS Code applies the wrong scale factor when a different monitor's scaling changes in a multi-monitor GNOME setup, sometimes breaking the window.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/41345, which has tracked heterogeneous DPI on Linux since 2018 with 27 comments and an upstream Chromium/Electron link. The fix lives in Chromium's Ozone/Wayland backend; VS Code cannot resolve this independently.
