# Issue #148034 — Bad behaviour for named pipe (mkfifo) 

[https://github.com/microsoft/vscode/issues/148034](https://github.com/microsoft/vscode/issues/148034)

## Summary

Still reproduces on main: named pipes (FIFOs) are stat'd as FileType.Unknown but VS Code still attempts to open/read them, blocking the UI. Two other issues were closed as duplicates of this one, confirming it as the canonical tracker. The bug is real and growing in relevance (1Password .env pipe use case).

## Change summary

Opening or deleting a named pipe (mkfifo) in the file explorer causes infinite loading/blocking because VS Code tries to read from the pipe.

## Best solution

Guard file reads and deletes against non-regular-file types: in `diskFileSystemProvider.ts`, check `stat.isFIFO()` (and possibly `isSocket()`, `isCharacterDevice()`, `isBlockDevice()`) before reading, and surface a clear label ('Named Pipe') in the explorer instead of 'Unknown File Type'. The fix touches `src/vs/platform/files/node/diskFileSystemProvider.ts` (toType, readFile) and potentially the explorer's open/delete logic.
