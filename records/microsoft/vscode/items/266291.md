# Issue #266291 — Visual issue on the confirm button in chat

[https://github.com/microsoft/vscode/issues/266291](https://github.com/microsoft/vscode/issues/266291)

## Summary

Still present on main: the confirmation button in chat wraps text to multiple lines when using non-English locales because `overflow-wrap: break-word` is set without `white-space: nowrap`. A small CSS fix in the confirmation widget stylesheet would resolve this.

## Change summary

Chat confirm button text wraps to multiple lines with non-English display languages because the button allows word-breaking without preventing wrapping.

## Best solution

Add `white-space: nowrap` to the `.monaco-button` rule inside `.chat-confirmation-widget2 .chat-confirmation-widget-buttons .chat-buttons` in `chatConfirmationWidget.css` (around line 451), as the reporter suggested. This prevents button text from wrapping while still allowing the button to size to its content.
