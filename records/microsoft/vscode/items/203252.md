# Issue #203252 — Running an extension via debugging does not include it in the Issue Reporter Dropdown

[https://github.com/microsoft/vscode/issues/203252](https://github.com/microsoft/vscode/issues/203252)

## Summary

Team-authored issue (MEMBER); must keep open per policy. The bug appears still present on main — the issue reporter extension dropdown iterates `extensionService.extensions` filtering on `!isBuiltin`, but extensions launched via debugging may not appear in the registry depending on how the debug extension host is configured. No fix commit found.

## Change summary

Extensions launched via the debugger (F5) do not appear in the Issue Reporter's extension dropdown, making it harder to test extension-contributed issue reporting.

## Best solution

The issue reporter's extension list (in `issueQuickAccess.ts` and the overlay/wizard) should include extensions running in a debug extension host. This likely requires ensuring the extension registry or the query in the issue reporter accounts for development extensions, possibly by checking `isUnderDevelopment` on extension descriptions.
