# Issue #40233 — [Feature] Local Workspace settings

[https://github.com/microsoft/vscode/issues/40233](https://github.com/microsoft/vscode/issues/40233)

## Summary

Highly upvoted (1021 👍) feature request for local workspace settings that remains unimplemented on main. An open community PR exists but hasn't merged. This should stay open as a well-established backlog item.

## Change summary

Allow per-user, per-workspace settings via a `.vscode/settings.local.json` file (gitignored) that overrides the shared `.vscode/settings.json`.

## Best solution

Implement a local settings layer (e.g. `.vscode/settings.local.json`) in the configuration service that merges on top of folder settings. The open PR https://github.com/microsoft/vscode/pull/269623 is a starting point but needs team review. The work belongs in `src/vs/workbench/services/configuration/` (configuration models, editing service, and the folder configuration class).
