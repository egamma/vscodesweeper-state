# Issue #284279 — Stupidly difficult to select text from the beginning of line; cursor icon inconsistent with behavior

[https://github.com/microsoft/vscode/issues/284279](https://github.com/microsoft/vscode/issues/284279)

## Summary

The editor's CSS cursor type and its hit-testing logic disagree at the margin/content boundary. The I-beam cursor appears (from DOM element boundaries) before the click actually registers as content area (per `contentLeft` in `mouseTarget.ts`). This is still present on current main.

## Change summary

Mouse cursor shows text-selection I-beam in a narrow strip at the left edge of the editor content area where clicking does not actually initiate text selection.

## Best solution

Align the CSS cursor boundary with the hit-test boundary in `src/vs/editor/browser/controller/mouseTarget.ts` (line ~411), or adjust the content area DOM element positioning so the I-beam cursor only appears where clicks are actually handled as content. The fix likely involves the margin/content boundary calculation or the CSS on the `.view-overlays` / `.lines-content` containers.
