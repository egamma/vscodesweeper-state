# Issue #284279 — Stupidly difficult to select text from the beginning of line; cursor icon inconsistent with behavior

[https://github.com/microsoft/vscode/issues/284279](https://github.com/microsoft/vscode/issues/284279)

## Summary

The CSS text cursor appears in the line-decorations zone (between gutter and content) because the scrollbar container starts at `contentLeft`, but mouse interactions in that zone are handled as margin/outside-editor by the hit test — the cursor icon is misleading. Still present on main.

## Change summary

The mouse cursor icon shows a text-selection caret in the narrow zone between line numbers and actual content, but clicking/dragging there doesn't select text as implied.

## Best solution

Either shift the `cursor: text` CSS boundary rightward to exactly match the hit-test content boundary, or expand the hit-test content area leftward a few pixels into the line-decorations zone so the cursor icon matches clickable behavior. The fix likely involves the scrollbar container positioning or a `cursor: default` override on the margin overlay in `src/vs/editor/browser/viewParts/margin/margin.ts` or the line-decorations area.
