# Issue #284279 — Stupidly difficult to select text from the beginning of line; cursor icon inconsistent with behavior

[https://github.com/microsoft/vscode/issues/284279](https://github.com/microsoft/vscode/issues/284279)

## Summary

Keep open and route to the editor area; the report describes a plausible mismatch between the editor text-cursor affordance and the mouse hit-testing path near the left content boundary. I did not find a concrete fixing commit or duplicate in the provided context or local history search, so this should not be closed.

## Change summary

The editor can show a text caret cursor just left of the selectable text area, but dragging from that spot does not start text selection.

## Best solution

Editor hit testing and cursor styling should agree at the content-left boundary: either make the gutter/line-decoration area stop showing the text cursor, or intentionally allow selection to start from the small left-edge tolerance. The likely code is in `src/vs/editor/browser/controller/mouseTarget.ts`, `src/vs/editor/browser/controller/mouseHandler.ts`, and the view-lines cursor styling.
