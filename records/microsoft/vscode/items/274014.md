# Issue #274014 — Accessibility: Editor stops announcing file contents to Voice Over after reaching the end of the file

[https://github.com/microsoft/vscode/issues/274014](https://github.com/microsoft/vscode/issues/274014)

## Summary

Confirmed and still unfixed on main. The screen reader content files in src/vs/editor/browser/controller/editContext/native/ have not been modified since the issue was filed. VoiceOver announces incorrect content near end-of-file, confirmed by a team member and a second reporter.

## Change summary

VoiceOver announces wrong or partial line content when navigating near the end of a file in the editor.

## Best solution

Fix the screen reader content generation in `src/vs/editor/browser/controller/editContext/native/screenReaderSupport.ts` (or the rich/simple content variants) to correctly report line content when the cursor is near or at the last line of the file. Likely a boundary condition in how the accessible text range is computed around EOF.
