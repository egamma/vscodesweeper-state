# Issue #258187 — Open file descriptors are leaked in the integrated terminal

[https://github.com/microsoft/vscode/issues/258187](https://github.com/microsoft/vscode/issues/258187)

## Summary

The upstream node-pty FD leak (microsoft/node-pty#657) was fixed and VS Code now ships node-pty 1.2.0-beta.13, but a commenter still reports FD-related issues on 1.109.5 (which already included the fix at beta.10), suggesting the problem may not be fully resolved or there are additional leak vectors. Keeping open pending confirmation that the latest build resolves the issue.

## Change summary

File descriptors (pipes, eventfds) are leaked to child processes in the integrated terminal, causing issues with CRIU checkpointing and potential resource exhaustion.

## Best solution

Verify on a recent Insiders build (which ships node-pty 1.2.0-beta.13+) whether the FD leak is fully resolved. If the March 2026 reports (comment 3, dev container scenario) still reproduce, there may be a separate leak path in the terminal process spawning code or the remote server that needs investigation in `src/vs/workbench/contrib/terminal/` or the remote extension host.
