# Issue #82623 — "There are task errors" while typing in tasks.json

[https://github.com/microsoft/vscode/issues/82623](https://github.com/microsoft/vscode/issues/82623)

## Summary

Still reproduces on main: the 1-second debounce fix (commit 3404da9137d) was reverted (commit bc995617b0d) because it caused stale task detail, and no replacement was implemented. The notification fires immediately on every config change while typing in tasks.json. Team-authored issue on the On Deck milestone — keeping open.

## Change summary

Editing tasks.json triggers an aggressive 'There are task errors' notification popup on every keystroke before the user finishes typing.

## Best solution

Add a smarter debounce or delay to `_showOutput` when `runSource === TaskRunSource.ConfigurationChange` in `src/vs/workbench/contrib/tasks/browser/abstractTaskService.ts` — e.g. wait several seconds after the last config change before showing the notification, or replace the notification with editor diagnostics (squiggles) as multiple commenters suggested. The original debounce was reverted because it also delayed task detail updates, so the two concerns need to be decoupled.
