## Summary

Team-authored issue on the 'On Deck' milestone — keeping open per policy. The core request (don't show intrusive error popups while the user is still typing in tasks.json) remains valid; a 1s debounce was added but the team consensus is it should align with how language servers handle transient validation errors.

## Change summary

Editing tasks.json triggers an immediate 'There are task errors' notification popup before the user finishes typing, despite an added 1-second debounce.

## Best solution

Likely lives in `src/vs/workbench/contrib/tasks/` task configuration validation logic.
