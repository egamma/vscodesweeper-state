# Issue #241354 — vscode in linux with kde dont use xdg-open or default file-manager

[https://github.com/microsoft/vscode/issues/241354](https://github.com/microsoft/vscode/issues/241354)

## Summary

The "Open Containing Folder" command delegates to Electron's `shell.showItemInFolder()`, so the choice of file manager is entirely upstream. The previous identical report (issue #50386, Dolphin vs Nautilus on KDE) was resolved as an Electron fix. If it has regressed, the fix belongs in Electron again. Keeping open as upstream.

## Change summary

On Linux with KDE and Dolphin configured as the default file manager via xdg-mime, "Open Containing Folder" opens Nemo instead of Dolphin.

## Best solution

Confirm which Electron version introduced or reintroduced the regression, then file (or find) the upstream Electron issue. VS Code's only touchpoint is `shell.showItemInFolder()` in `src/vs/platform/native/electron-main/nativeHostMainService.ts:645` — no VS Code-side change will fix this.
