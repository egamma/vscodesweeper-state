# Issue #301009 — Please have option to include reflog in git graph

[https://github.com/microsoft/vscode/issues/301009](https://github.com/microsoft/vscode/issues/301009)

## Summary

Feature request to add a --reflog option to the SCM Graph view so users can see reflog entries (e.g. detached commits after rebase) in the graph. No duplicates found; the referenced issue #277469 was a different bug (commit not appearing) and is already closed. The request is niche but valid — routing to the git area for triage.

## Change summary

Add an option to include git reflog entries in the SCM Graph view, so orphaned/detached commits are visible and recoverable.

## Best solution

Evaluate whether the SCM history provider API (`ISCMHistoryProvider`) and the git extension's `GitHistoryProvider` can support a reflog mode or toggle. The `--reflog` flag on `git log` changes the commit walk to follow reflog entries instead of parent links — this would need plumbing through the history provider and likely a new UI toggle in the SCM Graph view. Given the niche audience, this could also be an `*extension-candidate` built atop the existing SCM graph API, but the API may not expose enough to do it externally today.
