## Summary

The client-side image resize service caps dimensions at 2048px (targeting OpenAI's algorithm), but Anthropic's API enforces a 2000px limit for many-image requests. When multiple images are attached and at least one exceeds 2000px on either axis, the server returns a 400 error. The resize logic gap is traceable in the current source.

## Change summary

Sending multiple images in a Copilot chat request fails with a 400 error when any image dimension exceeds 2000px, due to Anthropic's stricter many-image dimension limit.

## Best solution

Update the image resize logic in `src/vs/platform/imageResize/browser/imageResizeService.ts` (and the Copilot extension's `resizeImage` path) to account for per-model dimension limits — specifically Anthropic's 2000px cap for many-image requests. The resize should be model-aware or use the lowest common denominator (2000px instead of 2048px). The `imageLimits.ts` in the Copilot extension already handles image count filtering; dimension capping per model could follow a similar pattern.
