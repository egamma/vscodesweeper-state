## Summary

Valid feature request for syntax-aware structural diff (like difftastic). Not implemented on main — the diff editor still uses line/character-level algorithms (Myers, DP). Has 7 upvotes, below the 20-vote backlog threshold but in the Backlog milestone; keeping open for the community process.

## Change summary

Add a syntax-aware diff mode that understands code structure and can distinguish meaningful changes from formatting-only changes.

## Best solution

This could potentially be delivered as an extension via the diff provider API, or as a new diff algorithm option in the core diff editor. The existing `diffAlgorithm` infrastructure in `src/vs/editor/common/diff/` would need an AST-aware variant or a pluggable external tool integration. Given the complexity, an extension wrapping difftastic or tree-sitter-based diffing may be the more natural path.
