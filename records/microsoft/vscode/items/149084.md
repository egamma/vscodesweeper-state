# Issue #149084 — File nesting `expand: false` setting not respected in folders with a single child file nest

[https://github.com/microsoft/vscode/issues/149084](https://github.com/microsoft/vscode/issues/149084)

## Summary

Still reproducible on main: `autoExpandSingleChildren: true` in the explorer tree options unconditionally auto-expands any single-child collapsible node, including file nests, ignoring `explorer.fileNesting.expand: false`. A WIP fix was attempted in 2022 but never merged.

## Change summary

When a folder contains only one nested file group, expanding the folder auto-expands the file nest even though `explorer.fileNesting.expand` is `false`.

## Best solution

Change `autoExpandSingleChildren` in `src/vs/workbench/contrib/files/browser/views/explorerView.ts` from `true` to a function that returns `false` for `ExplorerItem` elements with `hasNests` when the file nesting `expand` setting is `false`, following the pattern attempted in the unmerged WIP commit cc16f8ac25d.
