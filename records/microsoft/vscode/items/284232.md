# Issue #284232 — Git - Blame shows 2 weeks when it is 3 weeks minus 3 hours

[https://github.com/microsoft/vscode/issues/284232](https://github.com/microsoft/vscode/issues/284232)

## Summary

Still reproduces on main: the git extension's `fromNow` in `extensions/git/src/util.ts` uses `Math.floor` for time-unit divisions, while the core `fromNow` in `src/vs/base/common/date.ts` uses `Math.round`. This causes dates just under a unit boundary to round down (e.g. 2.86 weeks → '2 weeks'). Team-authored issue on the On Deck milestone with an open community PR — keeping open.

## Change summary

Git blame relative dates round down instead of rounding to nearest, showing '2 weeks' for a commit nearly 3 weeks old.

## Best solution

Change the six `Math.floor` calls in `extensions/git/src/util.ts` (lines 662–774) to `Math.round`, matching core's `date.ts`. The open PR https://github.com/microsoft/vscode/pull/306558 does exactly this and should be reviewed and landed. Alternatively, consider deduplicating by reusing the core `fromNow` function instead of maintaining a separate copy in the git extension.
