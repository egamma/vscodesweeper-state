# Issue #273535 — Blank Terminal with chrome flag ---enable-features=Vulkan

[https://github.com/microsoft/vscode/issues/273535](https://github.com/microsoft/vscode/issues/273535)

## Summary

Root cause is upstream: forcing Vulkan via the Chromium flag `--enable-features=Vulkan` breaks the WebGL context that xterm.js uses for GPU-accelerated terminal rendering. VS Code already provides a workaround via `terminal.integrated.gpuAcceleration: "off"`. The author account is deleted (ghost), so no further info can be gathered, but the issue documents a real Chromium/Electron interaction worth keeping as a reference.

## Change summary

The integrated terminal renders blank when the Chromium flag `--enable-features=Vulkan` is applied, because Vulkan breaks the WebGL context used by xterm.js for GPU-accelerated rendering.

## Best solution

This is a Chromium-level incompatibility between forced Vulkan and WebGL contexts. The built-in workaround is `terminal.integrated.gpuAcceleration: "off"`. If the team wants to harden against this, xterm.js / the terminal GPU detection in `src/vs/workbench/contrib/terminal/browser/xterm/xtermTerminal.ts` could detect a failed WebGL context and auto-fallback to the canvas or DOM renderer, but that change lives partly in xterm.js upstream.
