# Issue #122775 — Call Stack bounces when short-lived second requests start/stop

[https://github.com/microsoft/vscode/issues/122775](https://github.com/microsoft/vscode/issues/122775)

## Summary

Still present on main. The Call Stack tree restructures its hierarchy when transitioning between 1 and 2+ sessions (line 967 of callStackView.ts), causing visible bouncing. A team member confirmed reproduction in Dec 2022 and no subsequent fix targets this structural-change behavior.

## Change summary

The Call Stack view collapses and restructures when a second debug session briefly appears, causing the user to lose their place in the paused session's stack.

## Best solution

Preserve the expanded/collapsed state of the first session's call stack when transitioning between single-session flat view and multi-session hierarchical view. The fix likely lives in `CallStackDataSource.getChildren()` (around line 967 of `src/vs/workbench/contrib/debug/browser/callStackView.ts`) and the autoexpand logic (lines 206–216). One approach: always use the hierarchical (multi-session) layout even for a single session, or preserve tree expansion state across the structural transition.
