# Issue #153267 — Cannot right click on command palette-like panels

[https://github.com/microsoft/vscode/issues/153267](https://github.com/microsoft/vscode/issues/153267)

## Summary

Still reproduces on main: right-clicking the quick input triggers a blur event that hides the quick pick. The focus tracker in quickInputController.ts calls hide(QuickInputHideReason.Blur) when focus leaves the container, and a native context menu steals focus. Multiple duplicates continue to be filed.

## Change summary

Right-clicking in the command palette or any quick pick dismisses it because the context menu steals focus from the quick input container.

## Best solution

Suppress the blur-to-hide when the focus leaves to a context menu (e.g. detect contextmenu events and set a short ignoreFocusOut window, or render the context menu within the quick input's shadow DOM/container). The fix belongs in src/vs/platform/quickinput/browser/quickInputController.ts around the focusTracker blur handler (line ~333). @sbatten's original suggestion (use the input box as parent container for the context view) is still the best lead.
