# Issue #186018 — TaskProvider not working when Remoting via SSH

[https://github.com/microsoft/vscode/issues/186018](https://github.com/microsoft/vscode/issues/186018)

## Summary

Still appears to be an open bug — task provider activation in remote scenarios (SSH, WSL) fails to surface contributed tasks. The issue is assigned to a team member and was confirmed by a second user in July 2024. No specific fix commit found on main.

## Change summary

Custom TaskProvider tasks are not shown in the task picker when connected via Remote-SSH or WSL, despite working locally.

## Best solution

Investigate whether task provider extension activation events fire correctly on the remote extension host. The `_activateTaskProviders` path in `abstractTaskService.ts` may need to ensure activation reaches the remote extension host, not just the local one. The `hasTaskSystemInfo` gate (line 871) that waits for 2 taskSystemInfos in remote scenarios may be involved.
