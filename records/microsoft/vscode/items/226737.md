[https://github.com/microsoft/vscode/issues/226737](https://github.com/microsoft/vscode/issues/226737)

## Summary

Still unfixed on main: `FileSearchProviderOptions` in the proposed `fileSearchProvider2` API has zero `readonly` modifiers on any of its properties, contrary to VS Code's convention for extension-facing option interfaces.

## Change summary

Properties of `FileSearchProviderOptions` in the proposed file-search API should be marked `readonly` to signal that extensions must not mutate the options object.

## Best solution
