# Issue #35337 — Add control to open file shown in "replace preview" editor when performing project-wide find/replace

[https://github.com/microsoft/vscode/issues/35337](https://github.com/microsoft/vscode/issues/35337)

## Summary

Still valid on current main — no 'Open File' button exists in the replace preview diff editor title bar. An open community PR (#307554) is in progress. The issue is the canonical tracker for this request (multiple duplicates have been closed against it) and carries `help wanted` + `On Deck`, so it should stay open.

## Change summary

Add an 'Open File' button to the replace preview diff editor title bar so users can quickly navigate to the actual file for editing.

## Best solution

Land or iterate on the open community PR https://github.com/microsoft/vscode/pull/307554, which adds the Open File button to the replace preview editor title bar. The implementation should register a menu item in `MenuId.EditorTitle` conditioned on the replace-preview editor context, similar to how SCM diff editors expose their 'Open File' action.
