# Issue #223276 — Potentially FATAL: Invalid `editor.language.brackets` or `editor.language.colorizedBracketPairs`

[https://github.com/microsoft/vscode/issues/223276](https://github.com/microsoft/vscode/issues/223276)

## Summary

Still reproducible on main: `validateBracketPairs` checks array shape but not that each element is a string, so non-string bracket values (e.g. nested arrays) pass through and crash the editor.

## Change summary

Setting `editor.language.brackets` or `editor.language.colorizedBracketPairs` to malformed values (non-string elements) crashes the text editor.

## Best solution

Add `typeof pair[0] === 'string' && typeof pair[1] === 'string'` guards in `validateBracketPairs` (src/vs/editor/common/languages/languageConfigurationRegistry.ts:167–171) so invalid entries are filtered out instead of crashing downstream.
