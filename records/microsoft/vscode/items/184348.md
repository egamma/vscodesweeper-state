# Issue #184348 — Search Editor peek support for notebook search

[https://github.com/microsoft/vscode/issues/184348](https://github.com/microsoft/vscode/issues/184348)

## Summary

Team-authored issue (hard keep-open). The bug likely persists: notebook cell results serialize with cell URIs (cellMatch.cell?.uri), but the peek/double-click handler in searchEditor.ts uses FILE_LINE_REGEX and RESULT_LINE_REGEX patterns that may not correctly resolve notebook cell URIs back to the notebook document, so peek would fail to navigate. The related issue #200172 (notebook results not appearing at all) was fixed, but this peek-navigation bug is a distinct problem.

## Change summary

Double-clicking a notebook search result in the Search Editor does not open a peek preview of the match location.

## Best solution

Fix the peek definition handler in `src/vs/workbench/contrib/searchEditor/browser/searchEditor.ts` to correctly resolve notebook cell URIs from search results back to the parent notebook document and cell, so that `editor.action.peekDefinition` navigates into the right cell. The serialization in `searchEditorSerialization.ts` (line 118, using `cellMatch.cell?.uri`) and the definition provider that backs peek both need to handle the notebook cell URI scheme.
