## Change summary

Warn the user when closing a VS Code window that has uncommitted or unpushed source control changes.

## Best solution

Implement an opt-in `ILifecycleService.onBeforeShutdown` handler in the SCM contribution that checks for dirty/unpushed state via the SCM provider API and shows a confirmation dialog. This would live in `src/vs/workbench/contrib/scm/browser/` as a new file (e.g. `scmLifecycle.ts`) registered in the SCM contribution. A setting like `git.confirmOnClose` would control the behavior.
