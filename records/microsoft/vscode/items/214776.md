# Issue #214776 — [2eg] potential listener LEAK detected, having 203 listeners already. MOST frequent listener (29):

[https://github.com/microsoft/vscode/issues/214776](https://github.com/microsoft/vscode/issues/214776)

## Summary

Team-authored debt issue on 'On Deck' milestone. The listener accumulation in merge editor view zones is architecturally real — each FixedZoneWidget subscribes independently to onDidLayoutChange, and files with many conflicts legitimately produce 200+ listeners across 3–4 editor panes. Keeping open for maintainer judgment.

## Change summary

Merge editor triggers a listener-leak warning (203 listeners on onDidLayoutChange) when many conflict action widgets are created simultaneously.

## Best solution

Refactor FixedZoneWidget to share a single onDidLayoutChange listener across all instances per editor (e.g., via a shared layout-change relay in ConflictActionsFactory), eliminating the per-widget subscription that triggers the leak detector. The disposal logic is already correct; only the listener fan-out pattern needs consolidation.
