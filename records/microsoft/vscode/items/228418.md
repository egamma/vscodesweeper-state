# Issue #228418 — SCM Graph - Make 'Auto' mode select multiple remotes

[https://github.com/microsoft/vscode/issues/228418](https://github.com/microsoft/vscode/issues/228418)

## Summary

Valid feature request for the SCM Graph 'Auto' mode to include refs from multiple remotes (e.g. both origin and upstream). The author found a git-config workaround but the underlying request — smarter auto-detection — remains unimplemented. Low traction (0 upvotes), but from a known contributor with a clear use case. Keeping open in Backlog.

## Change summary

SCM Graph 'Auto' mode only shows the single remote ref from `historyItemRemoteRef`; users with fork-based workflows want it to also include upstream remote refs automatically.

## Best solution

Enhance `_resolveHistoryItemFilter` in `src/vs/workbench/contrib/scm/browser/scmHistoryViewPane.ts` (the `'auto'` case around line 1402) to detect and include refs from additional configured remotes (e.g. upstream). Alternatively, the related issue https://github.com/microsoft/vscode/issues/240489 (On Deck, manually setting merge base) could subsume part of this if it lets users pin multiple refs. The SCM/git area owner should decide whether auto-detection or manual pinning is the right path.
