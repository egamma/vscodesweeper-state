# Issue #324494 — Add a setting to disable the integrated Git worktrees UI (Source Control listing + branch checkout prompts)

[https://github.com/microsoft/vscode/issues/324494](https://github.com/microsoft/vscode/issues/324494)

## Summary

Legitimate feature request for a setting to fully suppress the worktree UI (SCM listing and branch-checkout prompts) even when worktrees exist on disk. The existing `git.detectWorktrees` setting only controls auto-opening of worktrees in separate windows; it does not hide the worktrees artifact group or suppress worktree-aware branch prompts. Related open issues (#313526, #317388) describe symptoms of the same gap — worktrees appearing in Source Control with no way to dismiss them — but none of them cleanly track the opt-out setting request.

## Change summary

Add a setting to fully disable the integrated Git worktrees UI (Source Control artifact listing and worktree-aware branch checkout prompts).

## Best solution

Add a new setting (e.g. `git.showWorktrees`) that, when false, suppresses the worktrees artifact group in the SCM Repositories view and skips the worktree-conflict dialog on branch checkout. The worktree data is fetched in `Repository.updateModelState()` (repository.ts ~line 2881) and surfaced via `GitArtifactProvider`; the setting would gate both. This would also resolve the core complaint in #313526 and #317388 (worktrees reappearing after close).
