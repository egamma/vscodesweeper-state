# Issue #258457 — Shell integration not working correctly for `Pseudoterminal`

[https://github.com/microsoft/vscode/issues/258457](https://github.com/microsoft/vscode/issues/258457)

## Summary

Confirmed bug: `handleCommandExecuted` unconditionally overwrites the command line value (set via 633;E with high confidence) with the prompt input model's buffer-parsed value, which includes the visual continuation prompt prefixes (`... `). The fix should guard the overwrite when `commandLineConfidence` is already 'high'.

## Change summary

When a Pseudoterminal sends the 633;E escape sequence with a properly escaped command, the `onDidStartTerminalShellExecution` event still reports the raw buffer text including visual continuation prompt prefixes.

## Best solution

Guard line 575 in `src/vs/platform/terminal/common/capabilities/commandDetectionCapability.ts` so that `handleCommandExecuted` does not overwrite `currentCommand.command` when `commandLineConfidence` is already 'high' (i.e. when `setCommandLine` was called via 633;E). The prompt input model value should only be used as a fallback when no explicit command line was provided.
