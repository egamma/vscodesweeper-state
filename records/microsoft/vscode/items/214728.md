# Issue #214728 — Unable to Delete Files to Trash: vscode( V > 1.89 )  (on debian kde)

[https://github.com/microsoft/vscode/issues/214728](https://github.com/microsoft/vscode/issues/214728)

## Summary

Already correctly triaged as an upstream Electron issue. The root cause is in Electron's `shell.trashItem` API hanging when spawning `kioclient5` on KDE/Debian. VS Code simply calls `await shell.trashItem(filePath)` and has no control over the underlying trash implementation. The issue remains valid on current main (Electron 42) and should stay open to track the upstream dependency.

## Change summary

Deleting files to trash on Debian KDE hangs indefinitely because Electron's `shell.trashItem` spawns `kioclient5` which never completes in VS Code's process environment.

## Best solution

The fix belongs in Electron's trash implementation (the `shell.trashItem` native call). VS Code's only touchpoint is `src/vs/platform/files/electron-main/diskFileSystemProviderServer.ts` which delegates directly to Electron. An upstream Electron issue should be filed (or linked if one exists) to investigate why `kioclient5` hangs when spawned from Electron's process tree but works from a normal terminal — likely an environment variable or signal handling difference introduced in Electron 29+.
