# Issue #292878 — Not waiting terminal finish and new command cancel the last command

[https://github.com/microsoft/vscode/issues/292878](https://github.com/microsoft/vscode/issues/292878)

## Summary

This report describes the chat terminal tool sending a new command before a long-running build finishes, causing a ^C/SIGINT that cancels the previous command. This is the same underlying problem tracked by issue #271902 (SIGINT from PromptInputModel being incorrectly used in chat terminal tool result), which is already in Backlog and assigned.

## Change summary

The chat agent's terminal tool doesn't wait for long-running commands (>2min) to complete and instead sends the next command, which cancels the running build with SIGINT.

## Best solution

Consolidate this report into https://github.com/microsoft/vscode/issues/271902, which tracks the SIGINT/prompt-detection race condition in the terminal tool. The fix there (correct PromptInputModel state handling) should resolve this scenario.
