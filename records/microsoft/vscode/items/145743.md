# Issue #145743 — Have a `TabKindCustomDiff`

[https://github.com/microsoft/vscode/issues/145743](https://github.com/microsoft/vscode/issues/145743)

## Summary

Valid API gap: the Tab API has no `TabInputCustomDiff` for when two custom editors (e.g. images) are shown in a diff view — the tab's `input` falls to `unknown`. Still unimplemented on current main; keeping open as a tracked Backlog item for the API/tabs area.

## Change summary

Add a `TabInputCustomDiff` class to the Tab API so extensions can identify tabs showing a diff of two custom editors.

## Best solution

Add a `TabInputCustomDiff` class (with `original`/`modified` URIs and `viewType`) to `vscode.d.ts` alongside the existing `TabInputTextDiff` and `TabInputNotebookDiff`, and wire it in the tab model so custom-editor diffs no longer surface as `unknown` input. The work lives in `src/vscode-dts/vscode.d.ts` and the tab API implementation under `src/vs/workbench/api/`.
