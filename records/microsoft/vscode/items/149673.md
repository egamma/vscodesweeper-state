# Issue #149673 — The "$tsc-webpack-watch" from "amodio.tsl-problem-matcher" problem matcher does match, but doesn't mark the task as failed

[https://github.com/microsoft/vscode/issues/149673](https://github.com/microsoft/vscode/issues/149673)

## Summary

Milestone is 'On Deck' (a scheduling milestone), so this must stay open. The bug is also still plausibly present — team members investigated and identified the root cause (multiple problem matchers racing on tryBegin/tryFinish), but no fix commit has landed for this specific interaction.

## Change summary

When multiple problem matchers are active on a watching task, one matcher's tryFinish can consume the end pattern before another matcher processes error lines, causing the task to be marked successful despite compilation errors.

## Best solution

Fix the WatchingProblemCollector in `src/vs/workbench/contrib/tasks/common/problemCollectors.ts` to ensure that when multiple problem matchers share begin/end patterns, error lines are processed by all matchers before any one can trigger tryFinish. The workaround from alexdima (custom background patterns) confirms the timing issue.
