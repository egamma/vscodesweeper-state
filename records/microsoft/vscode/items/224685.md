# Issue #224685 — "Copy Call Stack" in Debug uses the wrong path separator when debugging cross platform

[https://github.com/microsoft/vscode/issues/224685](https://github.com/microsoft/vscode/issues/224685)

## Summary

Still reproduces on main: StackFrame.toString() uses `uri.fsPath` which normalizes path separators to the local OS, producing backslashes for remote Linux paths when the client is Windows. The linked PR #224686 was closed without merging, so no fix has landed.

## Change summary

"Copy Call Stack" renders remote Linux paths with Windows backslash separators when the client is Windows.

## Best solution

In `src/vs/workbench/contrib/debug/common/debugModel.ts` StackFrame.toString(), replace `this.source.uri.fsPath` with `this.source.uri.path` (or use a URI formatter that respects the remote scheme) so the path separators match the remote OS, not the client OS.
