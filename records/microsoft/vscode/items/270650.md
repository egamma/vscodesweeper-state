# Issue #270650 — Copilot Agent prompts fail when chat.useNestedAgentsMdFiles is enabled with SSH FS folders

[https://github.com/microsoft/vscode/issues/270650](https://github.com/microsoft/vscode/issues/270650)

## Summary

Real bug acknowledged by maintainer aeschli — the nested agents MD file search fails silently on schemes without a registered search provider (like SSH FS's `ssh://`), blocking prompt submission entirely. Still unfixed on main.

## Change summary

Copilot Agent prompts fail to send when `chat.useNestedAgentsMdFiles` is enabled and non-file-scheme folders (e.g. SSH FS `ssh://`) are present in the workspace.

## Best solution

Add a fallback in the agent instructions locator (or the search invocation path) so that schemes without a registered search provider are gracefully skipped rather than silently blocking the entire prompt flow. aeschli noted a fallback to regular file access is possible but slow; alternatively, skip unsupported schemes entirely.
