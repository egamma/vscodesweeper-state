# Issue #270650 — Copilot Agent prompts fail when chat.useNestedAgentsMdFiles is enabled with SSH FS folders

[https://github.com/microsoft/vscode/issues/270650](https://github.com/microsoft/vscode/issues/270650)

## Summary

Still reproduces on main: `listNestedAgentMDs` calls `vscode.workspace.findFiles('**/AGENTS.md')` without handling workspaces containing schemes that lack a search provider (like `ssh://`). A team member acknowledged the bug and suggested a fallback but no fix has landed.

## Change summary

Copilot Agent prompts silently fail to send when `chat.useNestedAgentsMdFiles` is enabled and workspace contains folders from SSH FS (which lacks a registered search provider).

## Best solution

In `extensions/copilot/src/platform/promptFiles/vscode-node/agentInstructionsLocator.ts`, the `listNestedAgentMDs` method should either filter workspace folders to only those with a registered search provider before calling `findFiles`, or wrap the call with error handling/timeout so that missing search providers don't block prompt submission entirely. As aeschli noted, a fallback to regular file access (or simply skipping non-file schemes) would resolve this.
