# Issue #194225 — Pick a forever theme color for the quick fix highlight and allow styling both background and borders

[https://github.com/microsoft/vscode/issues/194225](https://github.com/microsoft/vscode/issues/194225)

## Summary

Still valid: the quickfix highlight in codeActionController.ts still borrows `editorFindMatchHighlight` / `editorFindMatchHighlightBorder` instead of having its own dedicated theme color tokens. The related issue #199608 (code action API) was closed but did not address this theming ask. Keeping open as an unresolved polish/feature request.

## Change summary

Register dedicated theme color tokens for the quick fix highlight (background and border) instead of reusing the find-match-highlight colors.

## Best solution

Register new color tokens like `editorCodeActionHighlight.background` and `editorCodeActionHighlight.border` in `src/vs/platform/theme/common/colors/editorColors.ts`, defaulting to the current find-match-highlight values for backward compatibility, and update `src/vs/editor/contrib/codeAction/browser/codeActionController.ts` to use them. This gives theme authors independent control over the quick fix highlight.
