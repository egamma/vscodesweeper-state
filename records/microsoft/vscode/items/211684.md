# Issue #211684 — Accessibility: "Show All Symbols" command doesn't speak file names and paths

[https://github.com/microsoft/vscode/issues/211684](https://github.com/microsoft/vscode/issues/211684)

## Summary

Still reproduces on main: the ariaLabel for workspace symbol picks only includes the symbol name, omitting the container/file path shown visually in the description field.

## Change summary

The 'Show All Symbols' quick pick only speaks the symbol name to screen readers, omitting the file name and path that are displayed visually.

## Best solution

In `src/vs/workbench/contrib/search/browser/symbolsQuickAccess.ts`, change the `ariaLabel` at line 198 to include the `containerLabel` (which holds the container name and file path), e.g. `ariaLabel: containerLabel ? \`${symbolLabel}, ${containerLabel}\` : symbolLabel`.
