# Issue #316307 — Use artifacts for background terminals to improve accessibilty and stability

## Summary

Team-authored feature request with an 'On Deck' milestone — keeping open per policy. The request is to move background terminal status from inline actions to the artifacts area for better accessibility and stability.

## Change summary

Use the artifacts area instead of inline actions to surface background/async terminal status after a turn ends.

## Best solution

Implement artifact-based rendering for background terminal state in the chat-terminal contribution, replacing the current inline 'show' action introduced in PR #316279. This improves screen-reader discoverability and decouples terminal lifecycle UI from inline chat rendering.
