# Issue #104838 — Git - Clone to an alternate directory name

[https://github.com/microsoft/vscode/issues/104838](https://github.com/microsoft/vscode/issues/104838)

## Summary

Valid feature request with strong community support (47 upvotes, promoted to Backlog). The underlying `Git.clone()` method already accepts a `targetName` option, but the clone UI flow in `cloneManager.ts` never prompts for it or passes it through — the feature remains unimplemented.

## Change summary

Allow users to specify a custom directory name when cloning a repository from VS Code, instead of always using the repo's default name.

## Best solution

Add an input box step after the parent-directory picker in `extensions/git/src/cloneManager.ts` that asks for the target folder name (defaulting to the repo name derived from the URL), then pass the user's choice as `targetName` to `this.model.git.clone()`. The low-level plumbing in `extensions/git/src/git.ts` already supports this via `ICloneOptions.targetName`.
