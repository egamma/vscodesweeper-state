# Issue #277853 — Customize Key Binding for default Agent

[https://github.com/microsoft/vscode/issues/277853](https://github.com/microsoft/vscode/issues/277853)

## Summary

Team-authored feature request (MEMBER). The author noted partial support via per-agent keyboard shortcuts (VS Code 1.107), but the core ask — a keybinding that opens the user's configured *default* mode/agent rather than a specific named one — remains unimplemented. Keeping open per hard keep-open rules and because the gap is real.

## Change summary

Allow a keybinding to open the user's configured default chat agent/mode, rather than requiring a binding per specific agent.

## Best solution

Add a 'New Chat (Default Mode)' command that reads the user's configured default mode/agent and opens a new chat session with it, so a single keybinding adapts to whatever the user has set as their default. The mode-picker and chat-action code in `src/vs/workbench/contrib/chat/browser/actions/chatActions.ts` and `src/vs/workbench/contrib/chat/browser/widget/input/modePickerActionItem.ts` are the likely touch points.
