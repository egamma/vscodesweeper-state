# Issue #170001 — Window restores with different bounds each restart

[https://github.com/microsoft/vscode/issues/170001](https://github.com/microsoft/vscode/issues/170001)

## Summary

Confirmed upstream Electron issue: window bounds drift on restart because Electron returns different bounds than what VS Code sets. The maintainer linked this to https://github.com/electron/electron/issues/27651 and labeled it upstream. The root cause persists in current main — the `applyState` workaround in `windowImpl.ts` line 280 only covers macOS/Windows multi-display, not the single-display drift reported here. Keeping open as a valid upstream tracker.

## Change summary

Non-maximized VS Code window slowly grows toward the screen edges on each restart due to Electron returning slightly different bounds than what was set.

## Best solution

The fix lives in Electron (https://github.com/electron/electron/issues/27651). VS Code could potentially mitigate by extending the `setBounds()` workaround in `src/vs/platform/windows/electron-main/windowImpl.ts:applyState()` to also cover single-display scenarios and Linux, but the root cause is upstream. Keep tracking until Electron resolves the bounds inconsistency.
