# Issue #167015 — Disabling Proxy support does not disable proxy agent redirection

[https://github.com/microsoft/vscode/issues/167015](https://github.com/microsoft/vscode/issues/167015)

## Summary

Team-authored issue — keeping open. The bug is still present on current main: the node request service unconditionally applies the proxy agent regardless of the `http.proxySupport` setting value.

## Change summary

Setting `http.proxySupport` to `off` does not prevent VS Code's node request service from using the proxy agent derived from environment variables.

## Best solution

Guard the proxy agent setup in `src/vs/platform/request/node/requestService.ts` (around line 106) to skip `getProxyAgent()` and proxy headers when `http.proxySupport` is `off`. The setting is already registered in `src/vs/platform/request/common/request.ts` but not consulted by the node request path.
