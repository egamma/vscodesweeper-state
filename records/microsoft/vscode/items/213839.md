# Issue #213839 — Window hang when rendering deeply nested trees

[https://github.com/microsoft/vscode/issues/213839](https://github.com/microsoft/vscode/issues/213839)

## Summary

Team-authored issue; the recursive functions in indexTreeModel.ts (lines 673–699) that cause the hang with deeply nested trees remain unrefactored on current main. The bug is still present.

## Change summary

Rendering deeply nested trees (e.g. via Outline view with sticky scroll/indent guides) causes a renderer process hang due to recursive tree traversal functions.

## Best solution

Refactor `hasTreeNode` and `getTreeNode` in `src/vs/base/browser/ui/tree/indexTreeModel.ts` from recursive to iterative implementations, and optimize the sticky-scroll and indent-guide codepaths for deep nesting. The related issue https://github.com/microsoft/vscode/issues/250787 (avoid individual indent guides) addresses part of this.
