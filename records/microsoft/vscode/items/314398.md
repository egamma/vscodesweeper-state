# Issue #314398 — [P1] UI Bug: Connecting line between output and thinking doesn't respect output wrapping

[https://github.com/microsoft/vscode/issues/314398](https://github.com/microsoft/vscode/issues/314398)

## Summary

Still reproduces on main: the curved connector pseudo-element between the thinking collapse header and first thinking item uses a hardcoded `top: 22px`, so it doesn't shift down when a long title wraps. A team member confirmed the expected fix direction.

## Change summary

When the thinking collapse title wraps to multiple lines, the curved connecting line between the header and the first thinking item stays fixed at 22px from the top instead of following the wrapped title.

## Best solution

Replace the hardcoded `top: 22px` on the `::after` pseudo-element in the `.chat-thinking-box` CSS with a value that accounts for the header height — either via a CSS calculation, a layout-aware approach, or by measuring and setting the offset programmatically in `chatThinkingContentPart.ts`.
