# Issue #206959 — multi-diff-editor with 100+ files triggers global event limit

[https://github.com/microsoft/vscode/issues/206959](https://github.com/microsoft/vscode/issues/206959)

## Summary

Still reproduces on main: each file in the multi-diff editor registers a separate `onDidChangeConfiguration` listener on the global text resource configuration service (line 220 of multiDiffEditorInput.ts), easily exceeding the 175-listener threshold with 100+ files. An open PR (#319859) addresses this but has not merged.

## Change summary

Opening 100+ files in the multi-diff editor triggers listener leak warnings because each file adds its own configuration change listener to a global emitter.

## Best solution

Consolidate the per-file configuration listeners into a single listener that dispatches to the relevant documents — the approach taken in the open PR https://github.com/microsoft/vscode/pull/319859. That PR should be reviewed and merged.
