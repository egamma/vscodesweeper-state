# Issue #150976 — Quick Action Undo doesn't work in notebooks.

[https://github.com/microsoft/vscode/issues/150976](https://github.com/microsoft/vscode/issues/150976)

## Summary

Still reproducible on main: when a code action is applied to a notebook cell from the Problems panel, Ctrl+Z doesn't undo it because focus stays in the Problems panel and the notebook undo handler requires `editor.hasEditorFocus()`. Routing to the notebook area for proper ownership.

## Change summary

Pressing Ctrl+Z after applying a quick fix from the Problems panel to a notebook cell does not undo the change unless the user manually focuses the cell first.

## Best solution

The notebook undo/redo contribution in `notebookUndoRedo.ts` gates on `editor.hasEditorFocus()` (reinforced by commit e5b5f2130a9). A fix would need either the code-action application path to return focus to the notebook cell, or the undo handler to recognize a recent code-action edit even without focus. The `notebook` area owner should decide the right approach — this is a focus-management question at the boundary between the Problems panel and notebook editors.

## Evidence

- **undo handler requires focus:** src/vs/workbench/contrib/notebook/browser/contrib/undoRedo/notebookUndoRedo.ts line 25: `if (editor && editor.hasEditorFocus() && editor.hasModel() && viewModel)` — when focus is in the Problems panel, this returns `false` and the notebook undo is skipped.
- **focus requirement tightened:** Commit e5b5f2130a9 (2025-06-18) added the `hasEditorFocus()` guard, making this behavior even more strict than when the issue was filed.
- **regular editors unaffected:** For regular text files, the resource undo service handles undo at the model level regardless of which panel has focus, so the same workflow succeeds.
