# Issue #171865 — Line scrolling stops at folded regions

[https://github.com/microsoft/vscode/issues/171865](https://github.com/microsoft/vscode/issues/171865)

## Summary

Still reproduces on current main: the editorScroll by-line logic doesn't account for folded regions when computing the next model line, causing scroll to get stuck. An open PR (#322302) targets this but hasn't merged yet.

## Change summary

The `editorScroll` command with `by: line` gets stuck when scrolling down past a folded region because it advances by model line count without skipping folded spans.

## Best solution

Merge or iterate on https://github.com/microsoft/vscode/pull/322302 which fixes the by-line scroll to skip over folded regions. The fix belongs in `_computeDesiredScrollTop` in `src/vs/editor/browser/coreCommands.ts`.
