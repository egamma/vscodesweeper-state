# Issue #311920 — Agent profiles and namespacing for Copilot custom agents

[https://github.com/microsoft/vscode/issues/311920](https://github.com/microsoft/vscode/issues/311920)

## Summary

Keep open as a valid chat custom-agent feature request: the current discussion has maintainer interest in qualified enterprise instructions/agents, and I did not find evidence that namespaced agent identity or agent profiles are already implemented on main.

## Change summary

Add namespaced identities and profile-based activation/scoping for user-level Copilot custom agents to avoid filename collisions and noisy global agent pickers.

## Best solution

Have the chat-prompts/customization owners decide the agent identity model, likely by adding publisher/namespace metadata that participates in custom-agent resolution, plus a separate activation/scoping mechanism for user-level agents. If the profile part is too broad, split identity and activation into separate tracked work items.
