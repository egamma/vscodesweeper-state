# Issue #235883 — tab is not aligned for 2 bytes languages like Korean, Chinese, Japanese

[https://github.com/microsoft/vscode/issues/235883](https://github.com/microsoft/vscode/issues/235883)

## Summary

The editor intentionally uses actual font-measured character widths for rendering, which can cause slight tab misalignment with CJK characters. A contributor confirmed this is by design; the cursor model already treats fullwidth chars as width 2, but the renderer uses real metrics for accuracy.

## Change summary

Tab stops don't align perfectly when preceded by varying numbers of CJK fullwidth characters because the editor uses measured font widths rather than assuming exactly 2× monospace width.

## Best solution

Keep the current behavior: the editor's use of measured font widths is intentional and provides better visual accuracy for CJK text overall. The terminal's approach (assume 2× monospace) trades accuracy for alignment; users wanting strict grid alignment can use a font where CJK characters are exactly double the Latin width.
