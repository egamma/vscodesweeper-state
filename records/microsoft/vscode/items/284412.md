# Issue #284412 — "Checkout Branch" in "Source Control" panel does not do what it says.

[https://github.com/microsoft/vscode/issues/284412](https://github.com/microsoft/vscode/issues/284412)

## Summary

Verified duplicate of #286359, which tracks the same problem (checking out a remote branch silently switches to the local tracking branch) with more community discussion and solution proposals.

## Change summary

Checking out a remote branch (e.g. origin/branch) from the Source Control panel silently checks out the local tracking branch instead, even when local and remote point to different commits.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/286359 — the fix belongs in `CheckoutRemoteHeadItem.run()` in `extensions/git/src/commands.ts` (lines 165-184), which unconditionally checks out the local tracking branch when one exists. That issue already has community discussion on the right UX (prompt the user, fast-forward, or detach).
