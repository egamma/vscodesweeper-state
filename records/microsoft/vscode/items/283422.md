# Issue #283422 — Allow specifying MCP server for `Chat › Tools: Eligible For Auto Approval` configuration

[https://github.com/microsoft/vscode/issues/283422](https://github.com/microsoft/vscode/issues/283422)

## Summary

Valid feature request: the auto-approval setting currently keys on bare tool names via `getToolFullReferenceName(toolData)` without a toolSet argument, so MCP tools from different servers sharing a name cannot be distinguished. The infrastructure for server-qualified names exists (`toolSet.referenceName/toolName`) but isn't wired into the eligibility lookup or the settings enum.

## Change summary

Allow the `chat.tools.eligibleForAutoApproval` setting to distinguish same-named tools from different MCP servers.

## Best solution

Wire the toolSet context into `isToolEligibleForAutoApproval` so MCP tools are looked up as `serverName/toolName` in the config, and update `_updateToolReferenceNames` to surface server-qualified names in the settings enum. The plumbing in `getToolFullReferenceName` already supports this shape.
