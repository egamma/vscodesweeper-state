# Issue #145730 — commit message textbox loses focus when scrolled out of view

[https://github.com/microsoft/vscode/issues/145730](https://github.com/microsoft/vscode/issues/145730)

## Summary

Still reproducible on main: the SCM commit message input lives inside a virtualized tree, so scrolling it out of view destroys its DOM element and loses focus. No fix has landed for this specific scroll-driven focus loss.

## Change summary

The commit message textbox loses keyboard focus when the user scrolls it out of the viewport in the Source Control view.

## Best solution

Track whether the input widget had focus before it was unrendered (in `disposeElement`), and restore focus when `renderElement` re-renders it after scrolling back. The fix belongs in `InputRenderer` in `src/vs/workbench/contrib/scm/browser/scmViewPane.ts`, mirroring the existing pattern in `updateChildren` that saves/restores focus across tree refreshes.
