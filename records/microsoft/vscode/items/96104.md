# Issue #96104 — Git - "Stage selected ranges" clobbers line endings for the entire file without any warning (Windows)

[https://github.com/microsoft/vscode/issues/96104](https://github.com/microsoft/vscode/issues/96104)

## Summary

Still reproduces on main — `applyLineChanges` in staging.ts uses `TextDocument.getText()` which returns LF-normalized content, silently converting CRLF files. An open PR (#321284) is actively addressing this.

## Change summary

"Stage Selected Ranges" silently converts the entire file's CRLF line endings to LF when staging partial hunks on Windows.

## Best solution

Land https://github.com/microsoft/vscode/pull/321284 (or an equivalent fix) that preserves original CRLF line endings in `applyLineChanges` before passing the result to `repository.stage()`. The fix lives in `extensions/git/src/staging.ts`.
