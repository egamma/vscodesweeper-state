# Issue #307037 ‚Äî Copilot: Duplicate file context sent to LLM when file is explicitly attached and also the active editor

[https://github.com/microsoft/vscode/issues/307037](https://github.com/microsoft/vscode/issues/307037)

## Summary

Confirmed on current main: the UI layer deduplicates implicit-vs-explicit context for rendering (via `isAttachmentAlreadyAttached`), but the data layer in `getAttachedAndImplicitContext()` does not ‚Äî both entries enter `ChatRequestVariableSet` with mismatched IDs and are sent to the model, wasting context tokens.

## Change summary

When a file is both explicitly attached and implicitly included as the active editor, its content is sent twice to the LLM because the dedup in `ChatRequestVariableSet.add()` compares by `id` only, and implicit entries use static IDs (`vscode.implicit.file`) while explicit attachments use URI-based IDs.

## Best solution

Add URI-based deduplication in `getAttachedAndImplicitContext()` (in `chatInputPart.ts`) before adding implicit entries to the variable set ‚Äî mirror the existing `isAttachmentAlreadyAttached` URI comparison already used by the UI layer. This is cleaner than changing implicit context IDs, which would break consumers relying on the static `vscode.implicit.*` identifiers.
