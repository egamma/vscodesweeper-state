# Issue #81520 — ctrl + click "go to definition" clashes with ctrl + click "follow link"

[https://github.com/microsoft/vscode/issues/81520](https://github.com/microsoft/vscode/issues/81520)

## Summary

Still reproduces on main: both goToDefinitionAtPosition and LinkDetector independently register ClickLinkGesture handlers with no coordination — ctrl+click fires both 'go to definition' and 'open link' when a definition and a document link overlap at the same position. The proposed FollowLinkController (comment 12) was never implemented, and the earlier LinkOpener attempt was reverted.

## Change summary

Ctrl+click on a token that has both a definition provider result and a detected link triggers both 'go to definition' and 'open link' simultaneously.

## Best solution

Implement the FollowLinkController coordination layer proposed by @alexdima: a single controller handles the gesture, collects responses from both the definition provider and the link detector, and either picks the best one or shows a disambiguation picker. The likely files are `src/vs/editor/contrib/gotoSymbol/browser/link/goToDefinitionAtPosition.ts` and `src/vs/editor/contrib/links/browser/links.ts`.
