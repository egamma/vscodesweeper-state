# Issue #160377 — GitHub - can't fork a fork

[https://github.com/microsoft/vscode/issues/160377](https://github.com/microsoft/vscode/issues/160377)

## Summary

Team-authored issue, must keep open. The bug is still present on current main — the pushErrorHandler unconditionally renames the remote to 'upstream' without checking whether an 'upstream' remote already exists. Issue #189101 was closed as a duplicate of this one, confirming this is the canonical tracker.

## Change summary

When forking a fork via the GitHub push error handler, VS Code fails because it tries to rename the origin remote to 'upstream', but an 'upstream' remote already exists from the fork relationship.

## Best solution

Guard the `renameRemote` call at line 224 of `extensions/github/src/pushErrorHandler.ts` — check whether an 'upstream' remote already exists before renaming, and either skip the rename or use a different remote name. The `help wanted` label is already present.
