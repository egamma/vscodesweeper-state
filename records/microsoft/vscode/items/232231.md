# Issue #232231 — Suggest to enable recommended extensions if they are disabled

[https://github.com/microsoft/vscode/issues/232231](https://github.com/microsoft/vscode/issues/232231)

## Summary

Valid feature request; the gap still exists on current main. The workspace recommendation prompt (extensionRecommendationNotificationService.ts) filters out all installed extensions regardless of their enabled/disabled state, so disabled recommended extensions are never surfaced to the user.

## Change summary

Workspace extension recommendations should suggest enabling extensions that are installed but disabled, not only installing missing ones.

## Best solution

Modify `promptWorkspaceRecommendations` in `src/vs/workbench/contrib/extensions/browser/extensionRecommendationNotificationService.ts` to distinguish between installed-and-enabled vs installed-and-disabled extensions, and offer an 'Enable' prompt for the disabled ones. Only 4 upvotes so far — leave it in Backlog for community traction or a maintainer pickup.
