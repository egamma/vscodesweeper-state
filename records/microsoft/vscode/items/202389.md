# Issue #202389 — MiniMap: Side = Left Breaks EditorGroup.Border

[https://github.com/microsoft/vscode/issues/202389](https://github.com/microsoft/vscode/issues/202389)

## Summary

Keep open as a real editor minimap/rendering bug: the report has clear reproduction steps, is already labeled in the right area, and a later commenter confirmed it still reproduces in VS Code 1.114. I did not find a fixing commit or duplicate strong enough to propose closure.

## Change summary

When `editor.minimap.side` is `left`, the `editorGroup.border` separator between side-by-side editor groups is visually covered below the editor header instead of extending to the status bar.

## Best solution

Fix the editor group separator layering so `editorGroup.border` remains visible across the full editor group height regardless of minimap side, likely around the workbench editor grid separator styling and the editor/minimap stacking order. A small visual regression case for split editor groups with left minimap would make this safer.
