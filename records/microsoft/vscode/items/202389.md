# Issue #202389 — MiniMap: Side = Left Breaks EditorGroup.Border

[https://github.com/microsoft/vscode/issues/202389](https://github.com/microsoft/vscode/issues/202389)

## Summary

Still reproduces on main: the splitview separator border and the minimap both use z-index 5, causing the minimap to cover the editor group border when positioned on the left. A prior fix (bumping to z-index 20) was reverted because it broke tab.activeBorderTop rendering.

## Change summary

When editor.minimap.side is set to 'left', the editorGroup.border color stops below the breadcrumbs instead of extending the full height of the editor area.

## Best solution

Resolve the z-index conflict between the splitview separator border (splitview.css line 57) and the minimap (minimap.css line 62) without regressing tab.activeBorderTop (issue #174134). Possible approaches: give the minimap a lower z-index, restructure to avoid the stacking context collision, or clip the separator border's ::before above the minimap's layer only in the editor content area.
