# Issue #240896 — `DebugSession` is never disposed?

[https://github.com/microsoft/vscode/issues/240896](https://github.com/microsoft/vscode/issues/240896)

## Summary

Still unfixed on main: DebugSession.dispose() is never called when a session ends — the commented-out `// session.dispose(); TODO@roblourens` on line 808 of debugService.ts confirms the gap remains. A related commit (8eeb7b940b6) fixed emitter leaks in DebugService itself but did not address session disposal.

## Change summary

DebugSession instances are never disposed when they end, leaking event emitters and other resources held by the session.

## Best solution

Uncomment (and validate) the `session.dispose()` call in `debugService.ts` line 808 when a session ends, ensuring all DebugSession-owned emitters and disposables are cleaned up. The TODO is already in place; the work is confirming that disposing at that point doesn't break restart/reattach flows.
