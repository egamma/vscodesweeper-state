# Issue #226720 — `FileSearchProviderOptions#session` should not be `undefined`

[https://github.com/microsoft/vscode/issues/226720](https://github.com/microsoft/vscode/issues/226720)

## Summary

Already fixed and shipped: commit 0e62f015f7b ensured the session is always a non-undefined object, resolving the WeakMap key error. Shipped in 1.93.0.

## Change summary

FileSearchProviderOptions#session could be undefined, causing WeakMap errors in search providers that cache by session.

## Best solution

Keep the shipped fix; no further change needed. The session is now always an object (via SessionLifecycle) and never undefined.
