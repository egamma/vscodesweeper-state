# Issue #247469 — Starting VS code turns off Bluetooth

[https://github.com/microsoft/vscode/issues/247469](https://github.com/microsoft/vscode/issues/247469)

## Summary

Verified duplicate of the canonical Bluetooth-on-startup issue. The root cause is Electron/Chromium's BlueZ service activation on Linux, already tracked and labeled upstream.

## Change summary

Starting VS Code from the desktop launcher disables or interferes with Bluetooth on Ubuntu Linux.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/134461, which tracks the upstream Electron/Chromium BlueZ activation. The fix needs to land in Electron (e.g. a `--disable-bluetooth` flag), then be wired through VS Code.
