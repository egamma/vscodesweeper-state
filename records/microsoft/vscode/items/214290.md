# Issue #214290 — Expected pattern to be a non-empty string

[https://github.com/microsoft/vscode/issues/214290](https://github.com/microsoft/vscode/issues/214290)

## Summary

Still reproducible on main: an empty string key in `files.exclude` passes through `resolvePatternsForProvider` without being filtered, causing downstream glob matching (picomatch/ripgrep) to throw 'Expected pattern to be a non-empty string'.

## Change summary

Adding an empty string `""` to `files.exclude` causes the Search panel to error with 'Expected pattern to be a non-empty string' when performing text search.

## Best solution

Filter out empty/falsy keys in `resolvePatternsForProvider` in `src/vs/workbench/services/search/common/search.ts` (line ~702), similar to the existing guard in `globExprsToRgGlobs` at `src/vs/workbench/services/search/node/ripgrepFileSearch.ts:132`.
