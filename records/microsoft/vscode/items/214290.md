# Issue #214290 — Expected pattern to be a non-empty string

[https://github.com/microsoft/vscode/issues/214290](https://github.com/microsoft/vscode/issues/214290)

## Summary

Still reproduces on main: an empty-string key in `files.exclude` passes through `resolvePatternsForProvider` unfiltered and reaches ripgrep as an invalid glob, causing the error. The ripgrep file-search path has a separate guard, but the shared `resolvePatternsForProvider` function does not.

## Change summary

Setting `"": true` in `files.exclude` causes project-wide search to fail with "Expected pattern to be a non-empty string" because the empty key is passed to ripgrep as an exclude glob.

## Best solution

Add an empty-string filter in `resolvePatternsForProvider` in `src/vs/workbench/services/search/common/search.ts` (line 702) — change the filter to also require `key` is truthy (`&& key`). This is the central function that feeds exclude/include patterns to both file and text search providers, so fixing it there covers all downstream paths.
