# Issue #232559 — Error: Trying to add a disposable to a DisposableStore that has already been disposed of. The added object will be leaked!

[https://github.com/microsoft/vscode/issues/232559](https://github.com/microsoft/vscode/issues/232559)

## Summary

Still reproduces on main: the async `.then()` in `createOutputChannel` adds to `channelDisposables` without guarding against prior disposal, so calling `dispose()` before the promise resolves triggers the leak warning. A valid debt item worth fixing.

## Change summary

Creating and immediately disposing an output channel triggers a 'DisposableStore already disposed' error because the async channel-creation promise resolves after disposal.

## Best solution

Guard the `channelDisposables.add(...)` calls in the `.then()` callbacks (lines ~169 and ~228 of `src/vs/workbench/api/common/extHostOutput.ts`) with `if (!channelDisposables.isDisposed)` checks, so early disposal doesn't produce the leak warning.
