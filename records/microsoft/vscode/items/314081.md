# Issue #314081 — Hitting the stop button between terminal prompts doesn't result in the command getting killed

[https://github.com/microsoft/vscode/issues/314081](https://github.com/microsoft/vscode/issues/314081)

## Summary

Team-authored bug report about the chat stop button not killing terminal commands between interactive prompts. No fix found on current main; keeping open per team-member authorship rule.

## Change summary

Pressing the chat stop button while a terminal command is waiting between prompts (e.g. `yo code`) does not send SIGINT/Ctrl+C to kill the running process.

## Best solution

The stop-button handler for chat terminal tool invocations should send a SIGINT/Ctrl+C to the active terminal process when cancelled mid-execution. Likely involves the terminal tool execution logic around `chatTerminalToolProgressPart.ts` or the terminal tool's cancellation handling.
