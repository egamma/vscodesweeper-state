# Issue #155000 — Inactive nested breakpoint indicators jump horizontally

[https://github.com/microsoft/vscode/issues/155000](https://github.com/microsoft/vscode/issues/155000)

## Summary

Still reproduces on main: inline breakpoint content widgets are fully disposed and recreated on every state toggle (enable/disable), causing a visible horizontal position jump. No fix has landed.

## Change summary

Inline (column) breakpoint indicators jump horizontally when toggling a breakpoint's enabled/disabled state.

## Best solution

Reuse existing InlineBreakpointWidget instances across state changes instead of disposing and recreating them. Update the CSS class on the existing widget's DOM node rather than tearing down the content widget and re-adding it, which causes the layout jump.
