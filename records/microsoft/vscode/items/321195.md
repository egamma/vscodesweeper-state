# Issue #321195 — editor.findMatch ... Foreground vs HighlightForeground

[https://github.com/microsoft/vscode/issues/321195](https://github.com/microsoft/vscode/issues/321195)

## Summary

Confirmed on main: the CSS rules in findWidget.ts apply editorFindMatchForeground to the non-active decoration class and editorFindMatchHighlightForeground to the active one, swapping the documented semantics. An open PR (#321258) exists but hasn't merged.

## Change summary

editor.findMatchForeground and editor.findMatchHighlightForeground color tokens are applied to the wrong decoration classes, swapping their effect.

## Best solution

Swap the two CSS rules in src/vs/editor/contrib/find/browser/findWidget.ts so that editorFindMatchForeground applies to .currentFindMatchInline (active match) and editorFindMatchHighlightForeground applies to .findMatchInline (other matches), matching the Background token semantics. PR #321258 appears to address this.
