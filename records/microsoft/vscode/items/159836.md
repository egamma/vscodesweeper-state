## Summary

Team-authored debt issue; the startup profiler's `_createPerfIssue` method still uses the clipboard+URL-open approach instead of the built-in issue reporter service. The request remains valid on current main.

## Change summary

The startup profiler creates performance issues by copying data to the clipboard and opening a GitHub URL, rather than using VS Code's built-in issue reporter which would provide a better UX.

## Best solution
