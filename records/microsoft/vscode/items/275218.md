# Issue #275218 — Anthropic BYOK leaves incomplete sentence before tool call

[https://github.com/microsoft/vscode/issues/275218](https://github.com/microsoft/vscode/issues/275218)

## Summary

Team-authored issue (MEMBER); must keep open. The bug involves the linkifier not flushing buffered text before a tool call in BYOK Anthropic mode, leaving an incomplete sentence visible. The `beginToolCalls` signal exists in `messagesApi.ts` and propagates through `pseudoStartStopConversationCallback.ts`, but the linkifier in `responseStreamWithLinkification.ts` may not be flushing on that signal. The issue appears still open on main — no commit was found that specifically addresses the linkifier flush for BYOK tool calls.

## Change summary

BYOK Anthropic chat leaves an incomplete sentence fragment visible before a tool call because the linkifier does not flush its buffered text when tool call arguments begin streaming.

## Best solution

Ensure the linkifier flushes its buffered text when `beginToolInvocation` is called in the BYOK chat response stream, similar to how non-BYOK requests handle it. The fix likely lives in `extensions/copilot/src/extension/linkify/common/responseStreamWithLinkification.ts` and/or the progress callback wiring in `extensions/copilot/src/extension/prompt/node/pseudoStartStopConversationCallback.ts`.
