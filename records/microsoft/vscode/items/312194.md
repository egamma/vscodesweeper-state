# Issue #312194 — Source control menu stack closing on repository visibility toggling

[https://github.com/microsoft/vscode/issues/312194](https://github.com/microsoft/vscode/issues/312194)

## Summary

Valid feature request: the SCM repository visibility submenu closes after each toggle because standard menus dismiss on item execution. No keep-open mechanism exists for regular menu items. This is a UX polish improvement for users with many submodules.

## Change summary

Request to keep the SCM repository visibility submenu open after toggling a repository's visibility, so users can toggle multiple repos without re-navigating the menu stack.

## Best solution

Either implement a keep-open behavior for toggled submenu items in the SCM repositories menu (likely in `src/vs/workbench/contrib/scm/browser/scmViewPane.ts` where `RepositoryVisibilityAction` is registered), or convert the visibility toggles to a quick pick with multi-select. The menu system (`src/vs/base/browser/ui/menu/`) does not currently support keep-open semantics for toggled items, so a broader platform change or a quick-pick approach may be needed.
