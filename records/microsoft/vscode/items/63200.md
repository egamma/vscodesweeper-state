# Issue #63200 — [scss] selector specificity works wrong with nested selectors

[https://github.com/microsoft/vscode/issues/63200](https://github.com/microsoft/vscode/issues/63200)

## Summary

Still reproduces on main — the specificity calculation in vscode-css-languageservice only considers the current selector node, not parent selectors in nested contexts. The issue is correctly labeled and in Backlog with help-wanted.

## Change summary

Nested SCSS/CSS selectors show incorrect specificity on hover because parent selector specificity is not accumulated.

## Best solution

Fix `selectorToSpecificityMarkedString` in `microsoft/vscode-css-languageservice` (selectorPrinting.ts) to walk parent rulesets (similar to how `selectorToElement` already does) and sum their specificities. The `help wanted` label is appropriate since the fix is localized.
