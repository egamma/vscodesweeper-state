# Issue #283286 — Cannot read properties of undefined (reading 'getModelColumnOfViewPosition')

[https://github.com/microsoft/vscode/issues/283286](https://github.com/microsoft/vscode/issues/283286)

## Summary

Still reproducible on main: the access at viewModelLines.ts:837 is unguarded — when modelLineProjections[index] is undefined during a race between _constructLines and event dispatch, the TypeError fires. PR #285602 attempted a fix but was closed without merging.

## Change summary

TypeError crash in convertViewPositionToModelPosition when modelLineProjections is accessed during incomplete reconstruction.

## Best solution

Guard the array access at line 837 in viewModelLines.ts (and the similar unguarded accesses at lines 509/523/818) against undefined, returning a fallback position. Alternatively, defer event emission until _constructLines completes, as PR #285602 attempted. The fix belongs in the editor-core viewModel layer.
