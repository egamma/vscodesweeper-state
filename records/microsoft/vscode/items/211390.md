# Issue #211390 — Discard Changes and Stage Changes buttons are way to close together

[https://github.com/microsoft/vscode/issues/211390](https://github.com/microsoft/vscode/issues/211390)

## Summary

Valid UX concern about the proximity of Stage and Discard buttons in the diff editor gutter toolbar. The buttons are rendered via DiffEditorHunkToolbar menu in gutterFeature.ts and no spacing/separation change has been made since the report. Already labeled under-discussion, which is appropriate.

## Change summary

The Stage Changes and Discard Changes buttons in the diff editor gutter are too close together, leading to accidental clicks that can cause data loss.

## Best solution

Improve the gutter toolbar UX by increasing button spacing, reordering destructive actions, or adding a confirmation for Discard — the implementation lives in `src/vs/editor/browser/widget/diffEditor/features/gutterFeature.ts` and the `DiffEditorHunkToolbar` menu. Alternatively, move the Revert/Discard action into a secondary menu or dropdown as the commenter suggests.
