# Issue #215340 — Skip re-requesting implicit code actions if diagnostics around the cursor have not changed

[https://github.com/microsoft/vscode/issues/215340](https://github.com/microsoft/vscode/issues/215340)

## Summary

The proposed optimization has not been implemented. The `_onMarkerChanges` handler in `codeActionModel.ts` still re-triggers code actions on any marker change in the file, regardless of whether diagnostics near the cursor changed. The `under-discussion` label is appropriate while the design settles.

## Change summary

Avoid re-requesting implicit code actions when diagnostics change elsewhere in the file but not around the cursor position.

## Best solution

In `CodeActionModel`'s `_onMarkerChanges`, cache the markers near the cursor (e.g. on the current line) and only call `_tryAutoTrigger()` when those markers actually differ from the cached set. This would reduce unnecessary code-action requests, especially with incremental diagnostic providers. The fix lives in `src/vs/editor/contrib/codeAction/browser/codeActionModel.ts`.
