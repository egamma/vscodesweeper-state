# Issue #191157 — Recently opened project has the same name

[https://github.com/microsoft/vscode/issues/191157](https://github.com/microsoft/vscode/issues/191157)

## Summary

Valid acknowledged bug — duplicate entries in vscode.dev's recently-opened list due to different URI representations for the same repo (implicit vs explicit default branch). Team confirmed it in 2023 but no fix has landed.

## Change summary

Recently opened projects on vscode.dev show duplicate entries for the same repository when URIs differ only by explicit/implicit default branch encoding.

## Best solution

Normalize vscode-vfs URIs in the recently-opened history so that implicit default-branch URIs and explicit default-branch blob URIs for the same repo are treated as one entry. The fix likely lives in the GitHub embedder's URI handling or in the workspaces history service's deduplication logic. joyceerhl outlined two approaches in the comments — the embedder normalization path seems cleanest.
