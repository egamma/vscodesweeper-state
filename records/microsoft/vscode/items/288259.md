# Issue #288259 — Unable to see git worktree created by copilot

[https://github.com/microsoft/vscode/issues/288259](https://github.com/microsoft/vscode/issues/288259)

## Summary

The hiding of Copilot-created worktrees in Source Control is intentional (PR #284462), but users are requesting a way to opt out or manually access these worktrees. The code explicitly marks copilot/agents worktrees as hidden with no user-facing setting to override this. This is a valid feature request for a configuration option.

## Change summary

Request for a setting or mechanism to show Copilot-created git worktrees in the Source Control view, which are currently hidden by design.

## Best solution

Add a configuration option (e.g. `git.showCopilotWorktrees`) that allows users to override the default hiding behavior in `extensions/git/src/repository.ts` line 979–981, so advanced users who want full SCM visibility of agent-created worktrees can opt in.
