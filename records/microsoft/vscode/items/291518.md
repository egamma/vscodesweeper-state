# Issue #291518 — Sign-in can get stuck from Chat

[https://github.com/microsoft/vscode/issues/291518](https://github.com/microsoft/vscode/issues/291518)

## Summary

Still reproduces on main: the cancellation token is not propagated to chatSetupRunner.run(), so when the user stops the sign-in flow via the Chat stop button, the pendingRun promise never resolves and subsequent sign-in attempts return the stale hung promise. PR #291531 attempted a fix but was closed without merging.

## Change summary

Cancelling a chat sign-in flow via the Stop button leaves sign-in permanently stuck because the pending setup promise is never cleared on cancellation.

## Best solution

Propagate the CancellationToken from the chat agent invoke call through to chatSetupRunner.run() and race it against the setup flow, clearing pendingRun when cancellation wins. The approach in the closed PR #291531 was sound but needs a human-authored version landed on main.
