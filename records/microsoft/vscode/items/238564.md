# Issue #238564 — Braces colouring is off inside Dart multiline strings

[https://github.com/microsoft/vscode/issues/238564](https://github.com/microsoft/vscode/issues/238564)

## Summary

Still present on main: bracket pair colorization can show incorrect colors inside strings due to a race between the initial fast-tokenizer AST and background tokenization completing. The architecture has mitigation (initialAstWithoutTokens + didChange on completion) but edge cases persist where the incorrect state gets stuck.

## Change summary

Bracket pair colorization incorrectly colors braces inside string literals when tokenization hasn't completed, and can sometimes remain stuck in the wrong state.

## Best solution

The race lives in `bracketPairsTree.ts` — when `initialAstWithoutTokens` is built via `FastTokenizer` (which has no string/comment awareness), subsequent token-change events may not fully invalidate the stale bracket state. The fix likely involves ensuring that incremental token updates during the initial-AST phase always trigger a full reparse once string tokens are available, or suppressing bracket colorization decorations until background tokenization reaches the visible range.
