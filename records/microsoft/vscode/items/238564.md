# Issue #238564 — Braces colouring is off inside Dart multiline strings

[https://github.com/microsoft/vscode/issues/238564](https://github.com/microsoft/vscode/issues/238564)

## Summary

The bracket pair colorization race condition is still present on main. The BracketPairsTree uses an `initialAstWithoutTokens` (built without token-type info) that colors brackets inside strings; it should clear once background tokenization finishes, but the transition can fail or flash, leaving brackets incorrectly colored.

## Change summary

Bracket pair colorization incorrectly colors braces inside multiline strings (any language) due to a race between initial bracket detection and background tokenization.

## Best solution

The root cause is in `src/vs/editor/common/model/bracketPairsTextModelPart/bracketPairsTree/bracketPairsTree.ts` — `getBracketsInRange` prefers `initialAstWithoutTokens` (which ignores token types) until background tokenization completes. The fix should ensure the initial AST either suppresses bracket colorization entirely until tokens are available, or more reliably transitions to the token-aware tree. This is a core editor issue, not extension-specific.
