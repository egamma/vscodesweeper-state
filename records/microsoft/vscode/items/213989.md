# Issue #213989 — Bad multi cursor paste behaviour

[https://github.com/microsoft/vscode/issues/213989](https://github.com/microsoft/vscode/issues/213989)

## Summary

Team-authored issue discussing whether `editor.multiCursorPaste` default should change from `spread` to `full`. The discussion is ongoing with suggestions for a separate command or a disambiguation widget. Keeping open per hard keep-open rule (MEMBER author) and because it's an active design discussion.

## Change summary

When pasting multi-line content with multiple cursors and `editor.multiCursorPaste` set to `spread`, content is distributed one-line-per-cursor instead of pasting the full block at each cursor.

## Best solution

Resolve the design discussion: either change the default to `full`, add a separate `pasteWithSpread` command, or show a disambiguation widget when cursor count matches line count. The setting and paste logic live in `src/vs/editor/contrib/multicursor/` and clipboard handling in `src/vs/editor/contrib/clipboard/`.
