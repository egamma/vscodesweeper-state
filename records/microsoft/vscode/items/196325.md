# Issue #196325 — Git - Stage/unstage file, then immediately click it, no diff shown

[https://github.com/microsoft/vscode/issues/196325](https://github.com/microsoft/vscode/issues/196325)

## Summary

Still reproduces on main — the optimistic update moves the resource in the SCM view before `git add` completes, so clicking immediately opens a stale diff. Author is a team member; keeping open for maintainer judgment.

## Change summary

Staging/unstaging a file and immediately clicking it opens a diff with no changes because the index hasn't updated yet.

## Best solution

After an optimistic stage/unstage completes the actual git command, refresh any open diff editor whose resource moved groups — likely in `extensions/git/src/repository.ts` around the `updateModelState` call after `retryRun`, or by having the SCM diff input detect that the index has changed and re-resolve.
