# Issue #291980 — Git Blame editor decoration does not show "Open in Github" option for repos cloned via SSH with a Github enterprise SSH CA 

[https://github.com/microsoft/vscode/issues/291980](https://github.com/microsoft/vscode/issues/291980)

## Summary

Still reproduces on main: the SSH URL regex in getRepositoryFromUrl only matches `git@github.com:` and does not account for GitHub Enterprise SSH CA URLs of the form `org-<number>@github.com:`. The fix is a small regex change.

## Change summary

Git blame 'Open on GitHub' option missing for repos cloned via GitHub Enterprise SSH CA URLs (`org-<number>@github.com:org/repo.git`).

## Best solution

Widen the SSH regex in `extensions/github/src/util.ts` line 67 to accept the `org-<number>@` prefix pattern (e.g. `^[\w-]+@github\.com:`) so Enterprise SSH CA URLs are recognized as GitHub remotes.
