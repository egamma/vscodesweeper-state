# Issue #131728 — 3D webGL plots not visualized after suspension

[https://github.com/microsoft/vscode/issues/131728](https://github.com/microsoft/vscode/issues/131728)

## Summary

This is a known upstream Chromium/GPU-driver issue (WebGL context lost after suspend/resume) already tracked and labeled appropriately. The Chromium bug remains unresolved, so keeping open as an upstream tracker is correct.

## Change summary

3D WebGL plots in notebook outputs become blank after suspending and resuming the computer, requiring a VS Code restart to restore them.

## Best solution

Wait for the upstream Chromium fix (bug 1155863 / related context-loss issues). A VS Code-side mitigation could listen for `webglcontextlost`/`webglcontextrestored` events in the notebook webview's back-layer to prompt re-rendering, but the root cause is Chromium's GPU process not restoring the WebGL context after suspend. The issue is already correctly labeled `upstream-issue-linked` and `chromium`.
