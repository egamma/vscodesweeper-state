# Issue #237431 — Expose `Find in tree` command in Command Palette

[https://github.com/microsoft/vscode/issues/237431](https://github.com/microsoft/vscode/issues/237431)

## Summary

Still valid: `list.find` is registered via `KeybindingsRegistry.registerCommandAndKeybindingRule` (no title/category), so it never appears in the Command Palette. The request to surface it as a discoverable action remains unimplemented.

## Change summary

Make the 'Find in tree' command appear in the Command Palette when a tree widget has focus.

## Best solution

Convert the `list.find` registration in `src/vs/workbench/browser/actions/listCommands.ts` from `KeybindingsRegistry.registerCommandAndKeybindingRule` to a `registerAction2` with a proper title (e.g. 'List: Find') and a `when` precondition so it surfaces in F1 only when a tree has focus.
