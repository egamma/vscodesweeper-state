# Issue #237431 — Expose `Find in tree` command in Command Palette

[https://github.com/microsoft/vscode/issues/237431](https://github.com/microsoft/vscode/issues/237431)

## Summary

Still unimplemented on main: `list.find` is registered as a keybinding-only command (via `KeybindingsRegistry.registerCommandAndKeybindingRule`) and is not exposed in the Command Palette. The request to surface it in F1 when a tree has focus remains a valid gap.

## Change summary

Expose the existing `list.find` command in the Command Palette (F1) so users can discover and invoke tree find without memorizing the keybinding.

## Best solution

Convert `list.find` from a `KeybindingsRegistry.registerCommandAndKeybindingRule` registration to a `registerAction2` with `f1: true` and an appropriate `when` clause (e.g. `RawWorkbenchListFocusContextKey && WorkbenchListSupportsFind`), giving it a proper title like "Find in Tree". The change is localized to `src/vs/workbench/browser/actions/listCommands.ts`.
