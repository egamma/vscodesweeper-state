# Issue #153311 — Git - Lots of vscode socket files left in /tmp when using SSH

[https://github.com/microsoft/vscode/issues/153311](https://github.com/microsoft/vscode/issues/153311)

## Summary

Still reproducible on main: socket files in /tmp are not cleaned up when remote extension host connections end abnormally. The `_cleanResources()` in `extensionHostConnection.ts` does not unlink the pipe created by `_listenOnPipe()`, and the git extension's `IPCServer` only unlinks on orderly dispose.

## Change summary

VS Code remote sessions leave stale `vscode-git-*.sock` and `vscode-ipc-*.sock` files in /tmp that accumulate over time.

## Best solution

Add socket file unlink to `ExtensionHostConnection._cleanResources()` for the pipe created by `_listenOnPipe()`, and ensure the git extension's `IPCServer.dispose()` is reliably called on abnormal disconnects. The pyright directories are owned by Pylance and out of scope for core.

Only the server is closed via `extHostNamedPipeServer.close()` on connection (line 319), but if no connection arrives or the process crashes, the file leaks.
- **git extension does clean on dispose:** extensions/git/src/ipc/ipcServer.ts line 122-124: `dispose()` calls `fs.unlinkSync(this._ipcHandlePath)`, but this only fires on orderly shutdown. Abnormal disconnects (SSH drops, remote kills) will leak the socket file.
- **CLIServer does clean up:** src/vs/workbench/api/node/extHostCLIServer.ts line 187-189: `CLIServerBase.dispose()` properly unlinks the IPC handle path on non-Windows, showing the pattern is established but not applied everywhere.
- **pyright out of scope:** The pyright temp directories are created by the Pylance extension, not VS Code core.
