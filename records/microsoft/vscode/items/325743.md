# Issue #325743 — Toggle maximized terminal causes codex to render glitches

[https://github.com/microsoft/vscode/issues/325743](https://github.com/microsoft/vscode/issues/325743)

## Summary

Real terminal rendering bug: toggling a maximized panel with Ctrl+J sends intermediate resize events that corrupt TUI application rendering (e.g. Codex CLI). The resize debouncer's independent X/Y resize path and the flush-then-resize sequence on visibility change appear to be the root cause. No fix found on current main.

## Change summary

Toggling a maximized terminal panel with Ctrl+J causes TUI applications like Codex CLI to render with visual glitches due to intermediate resize events during the maximize→hidden→visible transition.

## Best solution

Investigate the resize sequence in `terminalInstance.ts` around lines 1425–1437 (visibility change handler) and the `TerminalResizeDebouncer` — when toggling from maximized, the panel goes hidden then visible at the restored size, causing a flush of the old (maximized) dimensions followed by a resize to the new (restored) dimensions. The fix likely needs to coalesce these into a single atomic resize to the final dimensions, avoiding the intermediate state that corrupts TUI layouts.
