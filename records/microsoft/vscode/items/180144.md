# Issue #180144 — Goto line picker should not announce the line content

[https://github.com/microsoft/vscode/issues/180144](https://github.com/microsoft/vscode/issues/180144)

## Summary

The debt is still present on main: the `status()` call in `gotoLocation` (line 155 of editorNavigationQuickAccess.ts) still announces the full line content from the picker rather than letting the editor handle accessibility natively. No progress has been made on the underlying editor-level fix.

## Change summary

The Go to Line picker announces line content via a status hack in the picker instead of letting the editor's own accessibility infrastructure handle it.

## Best solution

The editor should natively announce the current line when cursor position changes via a command (like `setSelection` with JUMP source), removing the need for callers like the quick-access picker and dirtydiffDecorator to manually call `status()`. This likely involves changes to `src/vs/editor/browser/controller/textAreaState.ts` or the editor's accessibility layer to detect programmatic cursor moves and announce accordingly.
