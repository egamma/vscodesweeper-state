# Issue #180144 — Goto line picker should not announce the line content

[https://github.com/microsoft/vscode/issues/180144](https://github.com/microsoft/vscode/issues/180144)

## Summary

The debt is still present on current main: the picker-level `status()` call announcing line content (line 155 of editorNavigationQuickAccess.ts) remains, rather than being handled at the editor level. The discussion concluded that the editor should own this announcement, but no such fix has landed.

## Change summary

Move screen-reader line-content announcement from the Go-to-Line picker into the editor layer so all navigation commands get it for free.

## Best solution

Fix the announcement at the editor level (likely in `textAreaState.ts` or the editor accessibility controller) so that any cursor/selection change — whether from Go to Line, extensions, or commands — correctly announces the target line. Then remove the `status()` hack from `editorNavigationQuickAccess.ts` and the similar pattern in `dirtydiffDecorator.ts`. This requires collaboration with the editor team (as the discussion concluded).
