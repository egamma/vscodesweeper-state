# Issue #163432 — IntlBackslash swapped with Backquote on ISO keyboards on Chrome & Firefox

[https://github.com/microsoft/vscode/issues/163432](https://github.com/microsoft/vscode/issues/163432)

## Summary

Keep open as an upstream browser keyboard-event issue: VS Code Web is relying on browser-reported KeyboardEvent.code, and maintainer analysis already linked matching Chromium and Firefox bugs. There is not enough evidence of a VS Code-side fix on current main, and the existing labels correctly classify it as upstream.

## Change summary

On macOS ISO/British keyboards in VS Code Web, Backquote and IntlBackslash are swapped in Chrome and Firefox, breaking bindings such as Ctrl+Backquote.

## Best solution

Track the Chromium and Firefox fixes as the canonical resolution for web; VS Code should avoid a local browser-specific remap unless the browser APIs still cannot provide correct scan-code data after those upstream fixes. Keep the keyboard.dispatch keyCode workaround documented for affected users.
