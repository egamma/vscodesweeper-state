# Issue #163432 — IntlBackslash swapped with Backquote on ISO keyboards on Chrome & Firefox

[https://github.com/microsoft/vscode/issues/163432](https://github.com/microsoft/vscode/issues/163432)

## Summary

Already correctly triaged as upstream — the root cause is Chromium (issue 1296783) and Firefox (bug 1755096) swapping IntlBackslash/Backquote on ISO keyboards on macOS. VS Code cannot fix this in its web builds without browser-side changes. Keep open to track upstream progress.

## Change summary

On macOS with ISO keyboards (e.g. British layout), Chrome and Firefox swap the IntlBackslash and Backquote scan codes, breaking keybindings like Ctrl+` in VS Code web.

## Best solution

Continue tracking the upstream Chromium (https://issues.chromium.org/issues/41247520) and Firefox (https://bugzilla.mozilla.org/show_bug.cgi?id=1755096) bugs. Once browsers ship the fix, VS Code web keybindings will work correctly on ISO keyboards without any core change. The desktop build already patches this via its native node module.
