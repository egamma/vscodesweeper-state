# Issue #245306 — VSCode Remote Compatibility Issue with Older GLIBC 2.17

[https://github.com/microsoft/vscode/issues/245306](https://github.com/microsoft/vscode/issues/245306)

## Summary

This is a support question about glibc compatibility that is answered by VS Code's documented and deliberate end-of-support policy: glibc >= 2.28 is required as of v1.99 (March 2025). The legacy server workaround the reporter observed in v1.98.2 was intentionally removed. The canonical issues (#203375, #231623) are both closed, and the FAQ documents the migration path.

## Change summary

User asks why VS Code Remote requires glibc >= 2.28 and how to use it on glibc 2.17 systems like CentOS 7.

## Best solution

Close as designed — the glibc >= 2.28 requirement is an intentional platform-support decision. The legacy server mechanism that allowed older glibc was removed in commit 31ec1c528f3 (v1.99). Users on glibc 2.17 systems need to upgrade their OS or use the last supported release (v1.98).

- **Proposed comment:**

  > Thanks for the detailed report and questions. To answer both:
  > 
  > 1. **Why did the official VS Code App (v1.98.2) work with Node 20 on glibc 2.17?** — VS Code shipped a "legacy server" mechanism that bundled a custom sysroot with compatible glibc/libstdc++ libraries, allowing Node 20 to run on older systems. This is why the node binary under `vscode-server` only required glibc 2.17 despite being Node 20 — it was statically linked against the bundled libraries.
  > 
  > 2. **Can this still be used?** — No. As of VS Code **v1.99 (March 2025)**, legacy server support was intentionally removed (https://github.com/microsoft/vscode/commit/31ec1c528f3). The minimum requirement is now **glibc >= 2.28** and **libstdc++ >= 3.4.25**. This was announced and tracked in https://github.com/microsoft/vscode/issues/203375 and https://github.com/microsoft/vscode/issues/231623.
  > 
  > For systems running glibc 2.17 (CentOS 7, Amazon Linux 2), the recommended path is to upgrade the OS to a supported version. CentOS 7 reached end-of-life in June 2024, and Amazon Linux 2 end-of-life is June 2025.
  > 
  > Since this is working as designed — a deliberate platform-support decision — I'm closing this issue. If you have further questions, please visit the [VS Code Remote FAQ](https://code.visualstudio.com/docs/remote/faq#_can-i-run-vs-code-server-on-older-linux-distributions).
