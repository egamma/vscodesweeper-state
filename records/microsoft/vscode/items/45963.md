# Issue #45963 — [icon themes] Expose API to provide a dynamic icon theme.

[https://github.com/microsoft/vscode/issues/45963](https://github.com/microsoft/vscode/issues/45963)

## Summary

Keep open: the original reload pain was partially addressed by an unsupported `_watch` mechanism, but the requested official dynamic icon theme API still is not present on current main. The issue is already routed to `themes` and parked in Backlog, so it should remain for a product/API decision rather than be closed as completed.

## Change summary

Expose an official API or supported contribution mechanism for icon/theme extensions to refresh dynamically generated theme manifests without reloading the VS Code window.

## Best solution

Theme owners should decide whether to turn the existing internal `_watch` behavior into a documented contribution-point contract or add a small public API for theme refresh; otherwise close only after an explicit product decision that the unsupported watcher is the intended final shape.

## Evidence

- **partial implementation:** `src/vs/workbench/services/themes/browser/fileIconThemeData.ts` has a `reload` method and copies `iconTheme._watch`; `src/vs/workbench/services/themes/common/colorThemeData.ts` also sets `themeData.watch = theme._watch === true`, matching the workaround discussed in the issue.
- **public API gap:** A read-only search of `src/vscode-dts/vscode.d.ts` and the workbench API sources did not find a public reload/dynamic theme API; the available mechanism remains an internal contribution property rather than the requested official API.
- **discussion:** The related merged PR https://github.com/microsoft/vscode/pull/66115 unblocked theme reloads, but maintainer follow-up in https://github.com/Microsoft/vscode/pull/66115#issuecomment-456114402 said a final solution was still needed, and the issue remains open with `feature-request`, `themes`, and `Backlog`.
