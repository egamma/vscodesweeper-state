# Issue #201679 — Add vso.* scopes support for Microsoft authentication provider

[https://github.com/microsoft/vscode/issues/201679](https://github.com/microsoft/vscode/issues/201679)

## Summary

Valid upstream dependency — VS Code is waiting on the Azure DevOps team to pre-authorize the granular vso.* scopes for the VS Code client app. A team member confirmed the dependency and no resolution has landed yet.

## Change summary

Extensions needing granular Azure DevOps scopes (vso.*) cannot request them through the Microsoft authentication provider because the VS Code client app hasn't been pre-authorized for those scopes by the ADO team.

## Best solution

Continue tracking until the Azure DevOps team completes the pre-authorization of granular scopes for the VS Code client app registration. No code change in VS Code itself is needed — the fix is an administrative action on the ADO/identity platform side. A periodic check-in with the ADO team (as TylerLeonhardt has been doing) is the right next step.
