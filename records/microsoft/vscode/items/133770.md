# Issue #133770 — Git - VSCode fails to ignore commented lines in commit template when configured comment char

[https://github.com/microsoft/vscode/issues/133770](https://github.com/microsoft/vscode/issues/133770)

## Summary

Still reproduces on main: `stripCommitMessageComments` hardcodes `#` as the comment character, ignoring `core.commentChar`. The code even has a TODO acknowledging this. An open PR (https://github.com/microsoft/vscode/pull/307559) exists but has not merged.

## Change summary

VS Code ignores git's `core.commentChar` config and always strips lines starting with `#`, so custom comment characters in commit templates are not removed.

## Best solution

Review and land the open PR https://github.com/microsoft/vscode/pull/307559, which adds `core.commentChar` support to `stripCommitMessageComments` in `extensions/git/src/git.ts`.
