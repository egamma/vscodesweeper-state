# Issue #189224 — "Unicode Highlight" feature suggests incorrect locale for ignoring common characters

[https://github.com/microsoft/vscode/issues/189224](https://github.com/microsoft/vscode/issues/189224)

## Summary

Valid request: the Unicode Highlight locale data is limited to VS Code localization pack languages, so Ukrainian users get an incorrect 'ru' suggestion. No fix has landed on main — the data in src/vs/base/common/strings.ts still lacks a 'uk' entry.

## Change summary

Unicode Highlight suggests Russian ('ru') locale for Cyrillic characters even when the user's system language is Ukrainian; adding 'uk' to allowedLocales has no effect because the locale data doesn't include it.

## Best solution

Expand the locale data generated by https://github.com/hediet/vscode-unicode-data to include Ukrainian (and potentially other Cyrillic-using locales). The fix touches `src/vs/base/common/strings.ts` (the embedded JSON) and the upstream data-generation tool. The '_os' locale resolution in `src/vs/editor/contrib/unicodeHighlighter/browser/unicodeHighlighter.ts` should also map OS-level Ukrainian locale to the appropriate character set.
