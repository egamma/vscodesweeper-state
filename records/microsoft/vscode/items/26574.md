# Issue #26574 — Find in Files "files to include" override .gitignore

[https://github.com/microsoft/vscode/issues/26574](https://github.com/microsoft/vscode/issues/26574)

## Summary

Still valid on current main: VS Code converts 'files to include' patterns into ripgrep `-g` glob arguments (ripgrepTextSearchEngine.ts:419–431), which causes ripgrep to override .gitignore entries. The upstream ripgrep fix (BurntSushi/ripgrep#829, commit c2f1653) was reportedly merged but a user confirmed in Feb 2026 that the issue persists. With 92 upvotes and active community discussion, this belongs in the Backlog.

## Change summary

When 'files to include' is specified in Find in Files, .gitignore rules are overridden because include patterns are passed as `-g` globs to ripgrep rather than as path arguments.

## Best solution

Either update to a ripgrep version that respects .gitignore even with `-g` include globs, or change VS Code's search layer to pass include paths as ripgrep path arguments (not `-g` globs) when the pattern is a directory path, so .gitignore is still honored. The code path is in `src/vs/workbench/services/search/node/ripgrepTextSearchEngine.ts` around line 419.
