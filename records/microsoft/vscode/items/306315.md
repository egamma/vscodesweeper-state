# Issue #306315 — No display or easy way to see what Repo you are in

[https://github.com/microsoft/vscode/issues/306315](https://github.com/microsoft/vscode/issues/306315)

## Summary

This is a feature request, not a bug. The Source Control view is designed to show the local repository folder name and branch — it does not display the remote URL (e.g., GitHub origin). The user's expectation was set by inaccurate Google/AI-generated answers. The Repositories view does exist and shows repository names, but those are local folder names, not remote URLs.

## Change summary

Request to display the remote repository URL or name (e.g., 'github.com/user/repo') in the Source Control view or status bar, rather than just the local folder name and branch.

## Best solution

Clarify that the current behavior is by design — the SCM view shows local folder names, not remote URLs. If the user wants to see the remote, they can hover over the repository entry in the Source Control Repositories view (the tooltip shows `provider.label: rootUri`), or use the terminal. A feature request to surface the remote URL more prominently could be tracked, but the reported behavior is not a bug. Ask the user to clarify exactly what they'd like to see and where.
