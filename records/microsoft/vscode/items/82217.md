# Issue #82217 — Clarify that proxy settings are only applied to extensions

[https://github.com/microsoft/vscode/issues/82217](https://github.com/microsoft/vscode/issues/82217)

## Summary

The original request — clarify that proxy settings only apply to extensions — is no longer accurate. Since 2019, proxy support has been extended well beyond extensions: `http.fetchAdditionalSupport` and `http.webSocketAdditionalSupport` now route Node.js fetch and WebSocket traffic through the proxy too. Adding the requested 'extensions only' clarification would be misleading on current main.

## Change summary

Clarify in setting descriptions that http.proxy and related settings only apply to extensions.

If further description polish is desired, a new issue scoped to the current proxy architecture would be more appropriate.
