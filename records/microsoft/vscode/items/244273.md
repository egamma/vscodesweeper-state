# Issue #244273 — A single slow Code Action provider can affect all others

[https://github.com/microsoft/vscode/issues/244273](https://github.com/microsoft/vscode/issues/244273)

## Summary

Still reproduces on main: `getCodeActions` in `codeAction.ts` uses `Promise.all(promises)` (line 163), which blocks all results until every provider finishes. A single slow extension provider delays the entire lightbulb/quick-fix experience for all other providers. The issue's request for incremental/streaming results remains unaddressed.

## Change summary

A single slow Code Action provider blocks all others from displaying results because `getCodeActions` awaits `Promise.all` across all providers.

## Best solution

Refactor `getCodeActions` in `src/vs/editor/contrib/codeAction/browser/codeAction.ts` to yield results incrementally as each provider resolves — show available code actions (and the lightbulb) as soon as the first provider returns, appending late arrivals. For `CodeActionTriggerKind.Invoke`, consider a timeout after which available actions are shown with a loading indicator for remaining providers.
