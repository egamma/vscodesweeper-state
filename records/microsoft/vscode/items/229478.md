## Summary

Team-authored accessibility bug about NVDA mouse tracking mode not announcing control names in the menu bar and activity bar. Kept open per hard keep-open rule (MEMBER author) and because it's a legitimate accessibility concern, though mouse tracking is a niche workflow.

## Change summary

NVDA's mouse tracking mode (Insert+M) does not announce control names when hovering over the top menu bar and left activity bar items.

## Best solution

Ensure the menu bar items and activity bar action items expose proper accessible names (aria-label or accessible text) to the platform accessibility tree so that screen reader mouse tracking can read them. Likely involves checking ARIA attributes in `src/vs/workbench/browser/parts/titlebar/menubarControl.ts` and `src/vs/workbench/browser/parts/activitybar/`. However, this is a very niche interaction (mouse tracking mode) and may warrant discussion on whether VS Code should specifically support this NVDA feature or if it's working-as-designed for an Electron app.
