# Issue #250313 — Word-wrap Alt-Z does not work with emojis.

[https://github.com/microsoft/vscode/issues/250313](https://github.com/microsoft/vscode/issues/250313)

## Summary

Still reproduces on main: the monospace line-breaks computer hardcodes surrogate-pair width to 2, but emojis render wider than 2 columns in typical editor fonts, so word-wrap underestimates the line length and fails to break.

## Change summary

Toggling word-wrap (Alt+Z) does not wrap lines containing emoji characters — the emojis overflow past the viewport edge.

## Best solution

The monospace line-breaks computer in `src/vs/editor/common/viewModel/monospaceLineBreaksComputer.ts` should measure or classify emoji surrogate pairs using their actual rendered width (likely via `isFullWidthCharacter` or a dedicated emoji-width lookup) rather than the hardcoded `charWidth = 2`.
