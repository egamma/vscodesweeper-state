# Issue #250313 — Word-wrap Alt-Z does not work with emojis.

[https://github.com/microsoft/vscode/issues/250313](https://github.com/microsoft/vscode/issues/250313)

## Summary

Team-authored issue; the defect is traceable in the source — surrogate pairs (emojis) get a hardcoded charWidth of 2 in the line breaks computer regardless of their actual rendered width, causing word-wrap to underestimate line length and overflow.

## Change summary

Toggling word wrap (Alt+Z) fails to wrap lines of emojis because the monospace line breaks computer hardcodes surrogate-pair width to 2 instead of measuring their actual rendered column span.

## Best solution

In `src/vs/editor/common/viewModel/monospaceLineBreaksComputer.ts`, the surrogate-pair branch (line ~172-176) should compute the character width using the font metrics or `columnsForFullWidthChar` rather than hardcoding `charWidth = 2`. The fix likely involves calling `computeCharWidth` on the full codepoint (or treating astral-plane emojis as full-width via `columnsForFullWidthChar`).
