[https://github.com/microsoft/vscode/issues/160640](https://github.com/microsoft/vscode/issues/160640)

## Summary

Valid feature request to expose git diff's --ignore-matching-lines option in VS Code's SCM diff view. Not implemented on main; a volunteer expressed interest but never submitted a PR. Low traction (3 upvotes) and in Backlog — keeping open for community process.

## Change summary

Add support for git diff's --ignore-matching-lines regex option to filter noise from diffs in VS Code's SCM view.

## Best solution

Implement a setting (e.g. `git.diff.ignoreMatchingLines`) that passes the `--ignore-matching-lines=<regex>` flag to `git diff` calls in the git extension, similar to how `git.blame.ignoreWhitespace` passes `-w` to `git blame`. The work lives in `extensions/git/src/git.ts` where diff commands are built. Issue #87944 is related (diff editor unimportant-difference handling) but distinct — this one is specifically about the git CLI flag.
