# Issue #167781 — Pressing cmd with hover open does not  peek symbol

[https://github.com/microsoft/vscode/issues/167781](https://github.com/microsoft/vscode/issues/167781)

## Summary

Still reproduces on main: when hover is already visible and the user presses Cmd/Ctrl, the hover does not re-render to include the definition preview from goToDefinitionAtPosition. Previous fix attempts were reverted or closed without merging.

## Change summary

Pressing Cmd/Ctrl while hover is open should update the hover to show the definition preview (peek), but currently nothing happens.

## Best solution

In contentHoverController.ts `_onKeyDown`, when the hover is already visible and a trigger modifier is pressed, re-render the hover content to include the definition preview from GotoDefinitionAtPositionEditorContribution. The closed PR #252344 and branch commit c129ea166b5 show prior approaches.
