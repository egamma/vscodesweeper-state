# Issue #178308 — Debug view: Sort "Paused on ..." threads to the top

[https://github.com/microsoft/vscode/issues/178308](https://github.com/microsoft/vscode/issues/178308)

## Summary

Feature request to sort paused threads to the top in the debug Call Stack view. A related PR was closed without merging, and the maintainer expressed reservations about thread reordering but acknowledged room for improvement (e.g. better scroll-to-reveal). No implementation exists on main. Keeping open as under-discussion per the maintainer's feedback.

## Change summary

Sort threads that hit a breakpoint to the top of the Call Stack view's thread list during debugging.

## Best solution

The maintainer suggested an alternative: reveal/scroll more of the focused thread's callstack into view when pausing, rather than reordering threads. A PR for that approach in `src/vs/workbench/contrib/debug/browser/callStackView.ts` would address the core need without the UX churn of reordering. The issue should stay open for the area owner to decide on the approach.
