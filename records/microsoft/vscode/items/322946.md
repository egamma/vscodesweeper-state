# Issue #322946 — Contextual inline code embedding in Copilot Chat

[https://github.com/microsoft/vscode/issues/322946](https://github.com/microsoft/vscode/issues/322946)

## Summary

Feature request to show code selections as inline references in the chat input rather than as opaque file attachments. Related to but not an exact duplicate of #298568, which asks for plain-text insertion — both address the same pain point (context attachments are hard to distinguish) but propose different mechanisms.

## Change summary

When adding a code selection to Copilot Chat, display it as an inline reference (e.g. @file:L22-35) in the input area instead of as a file attachment icon.

## Best solution

Consider consolidating with https://github.com/microsoft/vscode/issues/298568, which addresses the same underlying problem (selections added to chat are opaque attachments rather than visible inline content). If the team decides the two asks are distinct enough, route to the chat area owner for a UX decision on how selections should be represented in the input box.
