# Issue #134415 — Add `!`-prefix parsing to glob util

[https://github.com/microsoft/vscode/issues/134415](https://github.com/microsoft/vscode/issues/134415)

## Summary

Still unimplemented on main — the only attempt (branch `ben/glob-negate` and PR #152225) was closed without merging. The milestone is 'On Deck', a scheduling decision that requires maintainer judgment, so this stays open.

## Change summary

Add support for `!`-prefixed negation patterns in VS Code's glob utility so that `files.exclude` and `search.exclude` can include files that match a negated pattern.

## Best solution

Land the `!`-prefix negation in `src/vs/base/common/glob.ts` using the two-pass approach discussed in the comments (positive patterns first, then negated patterns override). The prior work on `ben/glob-negate` (commit efa5122689e) and the discussion in comments 10–15 provide a concrete starting point. Ripple effects into search (ripgrep already supports negation) and `files.exclude`/`search.exclude` settings need to be addressed.
