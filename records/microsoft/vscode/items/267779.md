# Issue #267779 — Edits should be presented in order

[https://github.com/microsoft/vscode/issues/267779](https://github.com/microsoft/vscode/issues/267779)

## Summary

Valid UX improvement request for edit navigation order in agent mode. A contributor engaged and explained the current cursor-based behavior is consistent with other features (next diff, next find), but the discussion remains open with community support.

## Change summary

After accepting an agent-mode edit in a long file, navigation should go to the next edit in document order from the accepted edit, not from the cursor position.

## Best solution

The team should decide whether edit-accept navigation in agent mode should diverge from the general cursor-based navigation pattern used by next-diff and next-find. If so, the change would likely live in the chat/agent editing contribution's accept-edit handler. A setting could bridge both preferences.
