# Issue #278968 — Response Type of "unknown" almost never emits a /response event for GPT-5 and other models

[https://github.com/microsoft/vscode/issues/278968](https://github.com/microsoft/vscode/issues/278968)

## Summary

Team-authored telemetry instrumentation debt: /response events are missing for turns with responseType 'unknown', creating a blind spot in turn-failure tracking. The issue lives in the copilot-chat extension's fetcher code, not in the core vscode repo. Keeping open for the assigned owner to investigate.

## Change summary

Turns ending with responseType 'unknown' (the 'Sorry, no response was returned' case) emit only /panel.request events but never /response events, causing undercounting of turn failures in telemetry.

## Best solution

Fix the fetcher code path in vscode-copilot-chat (chatMLFetcher.ts around L937) so that a response.error or equivalent /response event is always emitted for the 'unknown' response type, ensuring turn-failure telemetry is complete. The workaround of coalescing across event types is in place but the root cause in the fetcher maze should be resolved.
