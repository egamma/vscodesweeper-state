# Issue #250296 — Panel widths keep resizing when my monitor shuts off

[https://github.com/microsoft/vscode/issues/250296](https://github.com/microsoft/vscode/issues/250296)

## Summary

Genuine bug with multiple confirmations: workbench panel/sidebar widths collapse to minimum when the display changes (monitor sleep, dock switch, wake from sleep). No fix found on main. The `panel-chat` label is too narrow — this is a general layout issue. Normally a bug with `reproductionStatus: unclear` would get `needs-info`, but there are three independent confirmations across different setups, making `keep-open` appropriate while the layout area owner investigates.

## Change summary

Sidebar and auxiliary bar panel widths reset to ~300px (minimum) when the display state changes (monitor off, sleep/wake, dock switch), forcing the user to manually resize each time.

## Best solution

The workbench grid (`src/vs/workbench/browser/layout.ts`, `src/vs/base/browser/ui/grid/gridview.ts`) likely needs to save part sizes before a display-change-induced resize and restore them when the original dimensions return. The non-proportional grid layout (`proportionalLayout: false`) means parts are forced to minimum widths when the window shrinks, but those sizes are not restored when the window expands again. The `layout` area label is more appropriate than `panel-chat`.
