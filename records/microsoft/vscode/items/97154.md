## Summary

Issue carries the 'On Deck' milestone (a scheduling decision), so it must stay open. Substantively, the editor has since adopted the EditContext API with a RichScreenReaderContent class that renders complex HTML for screen readers — the original request's goals (richer accessibility content including line numbers, breakpoints) appear largely addressed by this new architecture, though the milestone indicates maintainers still consider it active.

## Change summary

Use contentEditable (or richer HTML) instead of a plain textarea for the hidden accessibility field to expose line numbers, breakpoints, and other decorations to screen readers.
