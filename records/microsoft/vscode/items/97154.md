# Issue #97154 — Use contentEditable for our hidden accessibility textArea

[https://github.com/microsoft/vscode/issues/97154](https://github.com/microsoft/vscode/issues/97154)

## Summary

The core ask — replacing the plain textarea with a richer input mechanism to enable richer accessibility content (line numbers, decorations, etc.) — has been implemented via the EditContext API adoption and the `renderRichScreenReaderContent` setting, which renders full line content with line-number attributes in a rich DOM structure instead of plain text.

## Change summary

Investigate using contentEditable (or a richer alternative) instead of a plain textarea for the hidden accessibility field, to enable richer screen reader content including line numbers and breakpoints.

## Best solution

The work landed via the EditContext API (a more modern successor to contentEditable) with rich screen reader content support. The `editor.editContext` setting (default: true since 1.94) replaced the textarea, and `editor.renderRichScreenReaderContent` enables rich line rendering with line-number data attributes. No further change is needed — close as completed.
