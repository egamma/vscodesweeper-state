# Issue #295635 — Cmd+enter to approve tool call should work if focus is anywhere in chat view

[https://github.com/microsoft/vscode/issues/295635](https://github.com/microsoft/vscode/issues/295635)

## Summary

Still reproduces on main: the Cmd+Enter keybinding for accepting tool confirmations requires `inChat` context key, which is scoped to the chat widget's DOM element. Clicking the margin puts focus in the chat view but outside the widget's scoped context key service, so the keybinding doesn't fire.

## Change summary

Cmd+Enter to approve tool calls does not work when focus is in the chat view's margin area rather than within the chat widget itself.

## Best solution

Broaden the keybinding's `when` clause to also fire when the chat view has focus (e.g., use a view-level focus context key like `focusedView == 'workbench.panel.chat.view.copilot'` as an OR alternative), or ensure the chat widget's scoped context key service covers the entire chat view container including margins.
