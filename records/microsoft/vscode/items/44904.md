# Issue #44904 — SCM: Align next/previous commands between diff and regular editor

[https://github.com/microsoft/vscode/issues/44904](https://github.com/microsoft/vscode/issues/44904)

## Summary

The keybindings are already aligned (both use Alt+F5/Shift+Alt+F5) and the commands are mutually exclusive via context keys, so the basic UX friction is reduced. However, the commands remain separate (`workbench.action.editor.nextChange` vs `workbench.action.compareEditor.nextChange`) rather than truly unified, so the feature request still has merit. Keeping open as a valid `help wanted` item in the Backlog.

## Change summary

Unify the next/previous change navigation commands so a single command works in both regular editors (dirty diff) and diff editors.

## Best solution

Merge the two command implementations behind a single command ID (or create a dispatcher command) that detects whether the active editor is a diff editor or a regular editor and delegates accordingly, eliminating the need for two parallel registrations with context-guarded preconditions.
