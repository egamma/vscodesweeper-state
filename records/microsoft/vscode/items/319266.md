# Issue #319266 — VS Code's terminal fails to load PowerShell modules yet they successfully load in Windows Terminal (PowerShell 7+)

[https://github.com/microsoft/vscode/issues/319266](https://github.com/microsoft/vscode/issues/319266)

## Summary

The reporter's PATH is missing fzf/oh-my-posh when VS Code is launched from the taskbar vs. a terminal. The `terminal.integrated.inheritEnv` setting has no effect on Windows by design. This appears to be an environment setup issue (tools installed in locations not reflected in the persistent user PATH), but more info is needed to determine whether VS Code should do something differently.

## Change summary

VS Code's terminal on Windows doesn't find fzf and oh-my-posh in PATH when VS Code is launched from the Start menu, though they work in Windows Terminal.

## Best solution

Clarify whether the missing PATH entries are in the user's persistent environment variables (System > Environment Variables) or only injected by the user's PowerShell profile. If the latter, this is expected — VS Code inherits from the Windows environment, not from a shell profile. The reporter can use `terminal.integrated.env.windows` to add paths, or ensure the tools are in their persistent user PATH. If this is actually a gap in how VS Code resolves user PATH on Windows (compared to macOS/Linux login-shell resolution), it may warrant a feature request for parity.

  > This looks like an environment inheritance issue on Windows. When VS Code is launched from the taskbar or Start menu, it inherits the system/user environment variables as Windows knows them — it does not source a PowerShell profile to build PATH (unlike macOS/Linux where `terminal.integrated.inheritEnv` can source a login shell). Tools like `oh-my-posh` and `fzf` may be installed in locations that are only added to PATH by your PowerShell profile or a package manager session, rather than being in your persistent user PATH (System > Advanced > Environment Variables).
  > 
  > Could you check:
  > 1. Are the directories containing `fzf.exe` and `oh-my-posh.exe` listed in your **persistent user PATH** (System Properties > Environment Variables > User variables > Path)?
  > 2. Or are they only available because your PowerShell profile or a tool like Scoop adds them at shell startup?
  > 
  > If (2), the workaround is to either add those directories to your persistent user PATH, or use the `terminal.integrated.env.windows` setting to append them.
