# Issue #292806 — Show the auto selected model name in the chat immediately after choosing

[https://github.com/microsoft/vscode/issues/292806](https://github.com/microsoft/vscode/issues/292806)

## Summary

Valid polish request: the auto-selected model name is only shown after the response completes because `resolvedModelId` is derived from `response.result.metadata.resolvedModel`, which is populated at the end of the request. Showing it earlier during streaming is a reasonable UX improvement for the auto-model experience.

## Change summary

When using the 'Auto' model selection, the resolved model name is not shown until the full response completes — users must wait through thinking/tool-use phases before learning which model was picked.

## Best solution

Surface the resolved model identifier as a progress event early in the response stream (before the final result), so the chat UI can display it immediately. The change would involve the Copilot extension sending the resolved model earlier (e.g., as a progress part) and the chat view model (`src/vs/workbench/contrib/chat/common/model/chatViewModel.ts`) reading it from progress rather than only from the final result metadata.
