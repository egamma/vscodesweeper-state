# Issue #301736 — Unified Agents Bar: switching to Commands or Files removes focus from search input

[https://github.com/microsoft/vscode/issues/301736](https://github.com/microsoft/vscode/issues/301736)

## Summary

Still reproduces on main: the `_switchTab` method in `unifiedQuickAccess.ts` updates the picker value and provider but never calls `picker.focusOnInput()`, so clicking a Radio tab steals focus from the search input.

## Change summary

Switching tabs in the Unified Agents Bar removes focus from the search input, requiring an extra click to resume typing.

## Best solution

Add a `picker.focusOnInput()` call at the end of `_switchTab()` in `src/vs/workbench/contrib/chat/browser/agentSessions/experiments/unifiedQuickAccess.ts` so focus returns to the input after a tab click.
