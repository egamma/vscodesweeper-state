# Issue #311908 — files.watcherExclude setting does not respect relative paths and folders outside the workspace are being watched

[https://github.com/microsoft/vscode/issues/311908](https://github.com/microsoft/vscode/issues/311908)

## Summary

Real bug: when a workspace folder is a subdirectory of a git repository, the git extension watches the entire repo root via `workspace.createFileSystemWatcher(new RelativePattern(Uri.file(repository.root), '**'))`, and `files.watcherExclude` patterns — resolved relative to the workspace folder — don't apply correctly to this broader watcher. Related to but not a duplicate of #305926, which covers worktrees specifically.

## Change summary

Opening a workspace subfolder of a git repo causes VS Code's git extension to watch the entire repo root (including sibling folders), and `files.watcherExclude` relative patterns don't exclude paths from this watcher because the watcher root differs from the workspace folder.

## Best solution

The git extension's repository watcher in `extensions/git/src/repository.ts` (line ~920) should either scope its watcher to the workspace folder when it differs from the repo root, or the exclude-pattern resolution in `extHostFileSystemEventService.ts` should handle watchers whose base path is outside the workspace. This is closely related to https://github.com/microsoft/vscode/issues/305926 and may benefit from a combined fix. The Stripe commenter's workaround confirms real-world impact on large monorepos.
