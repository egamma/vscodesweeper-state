## Summary

The 408 'Timed out reading request body' error (code: user_request_timeout) originates from the GitHub Copilot service infrastructure, not from VS Code's client code. This is a service-side reliability issue that should be reported to GitHub Copilot support, not tracked as a VS Code editor bug.

## Change summary

Copilot chat frequently returns 408 timeout errors ('Timed out reading request body') during agent plan execution across multiple models.

## Best solution

This belongs in https://github.com/github/copilot or through GitHub Support, not the VS Code editor repo. The error is emitted by the Copilot API service layer — VS Code is only surfacing it. Users experiencing persistent timeouts should file through GitHub Support with their request IDs for server-side investigation.
