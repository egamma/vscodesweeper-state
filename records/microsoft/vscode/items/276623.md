# Issue #276623 — Build failed in Electron Tests step due to file locked by another process

[https://github.com/microsoft/vscode/issues/276623](https://github.com/microsoft/vscode/issues/276623)

## Summary

Team-authored CI infrastructure issue about a Windows file-lock race during smoke-test log upload. Hard keep-open because the author is a MEMBER. The issue is a sporadic build flake, not a product bug.

## Change summary

Electron smoke tests fail on Windows CI when Azure DevOps attempts to access a log file that is still locked by the test process.

## Best solution

Investigate whether the smoke-test harness or the Azure pipeline publish step can be made more resilient to file-lock races on Windows — e.g. retry logic on log file upload, or ensuring the Electron process fully exits before the publish step runs. This lives in the build/CI infrastructure (`build/azure-pipelines/`) and the smoke-test runner (`test/smoke/`).
