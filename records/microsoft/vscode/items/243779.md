# Issue #243779 — Checkbox text should be click-able

[https://github.com/microsoft/vscode/issues/243779](https://github.com/microsoft/vscode/issues/243779)

## Summary

Team-authored issue (MEMBER) — keeping open. The gap is real: the issue reporter's checkbox labels are not wired to toggle the associated checkbox on click. bpasero's comment further validates the need, suggesting a reusable label-clickable checkbox widget in `base`.

## Change summary

Clicking the text label next to checkboxes in the issue reporter does not toggle the checkbox; users must click the tiny checkbox target itself.

## Best solution

Create a proper checkbox-with-clickable-label widget in `src/vs/base/browser/ui/toggle/` (or extend the existing `CheckboxActionViewItem` pattern at toggle.ts:450-457 which already wires label clicks). Then adopt it in the issue reporter overlay (`src/vs/workbench/contrib/issue/browser/issueReporterOverlay.ts` around line 1814-1827) and the legacy issue reporter page (`issueReporterPage.ts`). This addresses bpasero's suggestion for a shared widget and the original UX gap in one pass.
