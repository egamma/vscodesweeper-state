# Issue #224554 — activeEditor == 'multiDiffEditor' and multiDiffEditorAllCollapsed not set when expected (context keys)

[https://github.com/microsoft/vscode/issues/224554](https://github.com/microsoft/vscode/issues/224554)

## Summary

Still reproduces on main: the multi-diff editor's context keys (`activeEditor`, `multiDiffEditorAllCollapsed`) are not visible to the keybinding resolver when focus lands on the editor background (outside a specific embedded diff). No fix has been committed for this focus-scoping issue.

## Change summary

Context keys `activeEditor == 'multiDiffEditor'` and `multiDiffEditorAllCollapsed` are not resolved for keybindings when focus is on the multi-diff editor background rather than inside a specific diff.

## Best solution

The multi-diff editor widget or its container needs to ensure that when focus lands on the background/scrollable area (not inside a child diff editor), the DOM element is covered by a context key scope that includes both the group-level `activeEditor` key and the widget-level `multiDiffEditorAllCollapsed` key. Likely fix in `src/vs/editor/browser/widget/multiDiffEditor/multiDiffEditorWidgetImpl.ts` — e.g. making the root element focusable with `tabindex=0` and ensuring the scoped context key service is attached there.
