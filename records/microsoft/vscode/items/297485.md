# Issue #297485 — Yolo but scoped to workspace or chat session

[https://github.com/microsoft/vscode/issues/297485](https://github.com/microsoft/vscode/issues/297485)

## Summary

Valid feature request: yolo/bypass-approvals mode is currently global (`chat.tools.global.autoApprove` is `APPLICATION_MACHINE` scope) and `chat.permissions.default` defaults to window scope. There is no mechanism to scope auto-approve to a specific workspace or to toggle it temporarily for a single chat session without affecting all sessions. The request remains unimplemented on current main.

## Change summary

Allow yolo/bypass-approvals mode to be scoped per workspace or per individual chat session, rather than being a global-only toggle.

## Best solution

Add workspace-level scoping to the auto-approve configuration (e.g. make `chat.permissions.default` support `RESOURCE` scope or add a workspace-aware override), and add a per-session transient bypass toggle (perhaps via the existing `/yolo` slash command) that only applies to the current chat session without persisting globally. The implementation would touch `src/vs/workbench/contrib/chat/browser/chat.shared.contribution.ts` (setting scopes), `src/vs/workbench/contrib/chat/browser/chatSlashCommands.ts` (slash command behavior), and the tool approval logic in `src/vs/workbench/contrib/chat/browser/tools/languageModelToolsService.ts`.
