# Issue #255637 — CTRL + Key commands don't work well on Android with physical keyboard. Deleting a piece of text crashes the browser

[https://github.com/microsoft/vscode/issues/255637](https://github.com/microsoft/vscode/issues/255637)

## Summary

Confirmed bug: editContext feature causes browser crash on Android (Chrome + Gboard) when deleting selected text. Team members identified the likely cause but no fix has landed on main.

## Change summary

On Android with Chrome and Gboard, selecting text and pressing Delete in the Monaco editor crashes the browser tab when editor.editContext is enabled.

## Best solution

Fix the editContext implementation to handle Android IME composition events from Gboard correctly, likely in `src/vs/editor/browser/controller/editContext/`. The crash path needs investigation into how Gboard's composition interacts with the EditContext API on Chrome/Android.
