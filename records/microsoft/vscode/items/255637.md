# Issue #255637 — CTRL + Key commands don't work well on Android with physical keyboard. Deleting a piece of text crashes the browser

[https://github.com/microsoft/vscode/issues/255637](https://github.com/microsoft/vscode/issues/255637)

## Summary

Confirmed bug: the native EditContext API (enabled by default on Chrome) crashes or misbehaves on Android with Gboard when deleting selected text. Team members are already engaged and identified the root cause area. Should remain open for a fix.

## Change summary

Selecting text and pressing Delete on Android Chrome with Gboard crashes the browser tab when the EditContext API is active.

## Best solution

Fix the NativeEditContext input handler to correctly handle selection-delete composition events from Gboard on Android, or disable EditContext on Android Chrome. The fix likely lives in `src/vs/editor/browser/controller/editContext/native/` and the Android-specific paths in `src/vs/editor/browser/controller/editContext/textArea/textAreaEditContextInput.ts`.
