# Issue #305251 — editorContext reports last focused file instead of empty when no editor is open (Copilot Chat)

[https://github.com/microsoft/vscode/issues/305251](https://github.com/microsoft/vscode/issues/305251)

## Summary

This is a narrower case of the stale editorContext injection problem already tracked by the broader, labeled issue #316051. The root cause — the Copilot extension's `TabsAndEditorsServiceImpl.activeTextEditor` fallback to visible editors in recent tab groups, combined with implicit `editorContext` injection — is the same in both. Proposing a duplicate close to consolidate discussion.

## Change summary

When no editor tab is active (e.g. Source Control view is focused), Copilot Chat still injects the last-focused file as `editorContext`, leading to stale/incorrect context in prompts.

## Best solution

Consolidate on https://github.com/microsoft/vscode/issues/316051, which tracks the broader editorContext staleness and cross-session pollution problem. The fix likely lives in `extensions/copilot/src/platform/tabs/vscode/tabsAndEditorsServiceImpl.ts` (the `activeTextEditor` fallback logic) and/or the `chat.agent.currentEditorContext.enabled` injection path — a commenter on #316051 confirmed that disabling that setting eliminates the misbehavior.

## Evidence

- **root cause in source:** `extensions/copilot/src/platform/tabs/vscode/tabsAndEditorsServiceImpl.ts` lines 56-78: when `vscode.window.activeTextEditor` is undefined, the getter falls back to visible editors in the most-recently-used tab group, returning a stale editor reference even when the user has left editor context.
- **editorContext injection:** `extensions/copilot/src/extension/prompt/node/chatParticipantRequestHandler.ts` line 121 calls `IDocumentContext.inferDocumentContext(request, tabsAndEditorsService.activeTextEditor, turns)`, which uses the stale activeTextEditor from the fallback path at `documentContext.ts` line 84.
- **canonical issue:** https://github.com/microsoft/vscode/issues/316051 is open with `bug` and `chat` labels, tracks the same stale-editorContext root cause in a broader multi-session context, and explicitly lists this issue (#305251) as a related report.
- **workaround confirmed:** A commenter on #316051 confirmed that setting `chat.agent.currentEditorContext.enabled: false` eliminates the misbehavior, narrowing the cause to the implicit editorContext injection path controlled by `extensions/copilot/src/platform/configuration/common/configurationService.ts` line 1046.
