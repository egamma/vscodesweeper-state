# Issue #256486 — Add a `contexts` field to the .md metadata for a custom chat mode

[https://github.com/microsoft/vscode/issues/256486](https://github.com/microsoft/vscode/issues/256486)

## Summary

Milestone is 'On Deck' (a real scheduling milestone), so this must stay open regardless of merit. The feature is not yet implemented — the existing `context` header attribute is a scalar for skills only (controls subagent forking), not a file/folder attachment mechanism for chat modes.

## Change summary

Request to add a `contexts` frontmatter field to chat mode .md files that auto-attaches specified files/folders as context without manual selection each time.

## Best solution

Implement a `contexts` (or similar) array attribute in the prompt file header for `PromptsType.prompt` (chat modes) that resolves workspace-relative file/folder paths and attaches them as context to every request. The validator in `promptValidator.ts` and attribute definitions in `promptFileAttributes.ts` would need updating, plus the chat request pipeline would need to resolve these paths at send time.
