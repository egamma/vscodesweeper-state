# Issue #181139 — Accessibility: Make Tab key focus restricted to the currently open view

[https://github.com/microsoft/vscode/issues/181139](https://github.com/microsoft/vscode/issues/181139)

## Summary

Active accessibility feature request with direct team engagement and an unresolved discussion; the author's last comment references an in-person meeting, suggesting ongoing deliberation. Keeping open under its existing `under-discussion` label.

## Change summary

Restrict Tab/Shift+Tab focus cycling to the currently open overlay view (e.g., Ctrl+F find widget) instead of letting it fall through to the editor area, improving navigation for screen reader users.

## Best solution

The team needs to decide whether scoped focus trapping per overlay is the right UX, balancing screen-reader discoverability against the existing tab-through-everything model. If accepted, the implementation would add a focus-trap zone to overlay widgets (find widget, terminal accessibility mode, etc.) that activates when the overlay is open. The decision likely belongs to the accessibility area owners given the prior terminal experiment.
