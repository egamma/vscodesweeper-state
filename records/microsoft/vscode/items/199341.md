# Issue #199341 — Toggle word wrap is not applied to both sides when triggered in inline caused by width

[https://github.com/microsoft/vscode/issues/199341](https://github.com/microsoft/vscode/issues/199341)

## Summary

Still present on main: when toggling word wrap in inline diff mode, only the modified editor receives the toggle state (wordWrapOverride2). Switching back to side-by-side leaves the original editor with stale wrap state. Filed by a team member — keeping open.

## Change summary

Toggle word wrap in inline diff mode doesn't propagate to both editors when switching back to side-by-side view.

## Best solution

In `diffEditorEditors.ts`, when transitioning from inline to side-by-side, synchronize the `wordWrapOverride2` toggle state from the modified editor to the original editor, or have the diff editor's options layer track a unified toggle state that applies to both sides on layout change.
