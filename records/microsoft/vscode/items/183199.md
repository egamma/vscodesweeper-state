# Issue #183199 — Git - The integrated source control (Git) is very very slow in the latest version (1.78.x)

[https://github.com/microsoft/vscode/issues/183199](https://github.com/microsoft/vscode/issues/183199)

## Summary

Broad, long-running Git SCM performance complaint (65 upvotes) with mixed root causes across reporters — some traced to GitLens, some to rev-parse storms on multi-repo workspaces, some to network latency. Recent main commits address specific aspects (rev-parse skip, status throttling) but the umbrella issue remains valid as a tracking point for ongoing git performance work. Keeping open given the community signal and active improvements landing.

## Change summary

Built-in Git integration operations (staging, diffing, committing, pushing) became extremely slow starting around v1.78, with some users seeing 30–100+ second operations that complete in milliseconds from the CLI.

## Best solution

Continue the incremental git-subprocess performance work already underway in `extensions/git/src/model.ts` and `extensions/git/src/git.ts` — the rev-parse skip (commit 34b2db959a4) and status throttling (commit f45c4ba0519) address the most-reported patterns (rev-parse storms and multi-repo fan-out). The remaining reports are diverse enough that a single fix is unlikely; the issue serves as a useful community signal for the git area owner to prioritize further subprocess reduction and caching.
