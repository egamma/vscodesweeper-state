# Issue #302333 — Let CLI know about IME composition started

[https://github.com/microsoft/vscode/issues/302333](https://github.com/microsoft/vscode/issues/302333)

## Summary

Team-authored feature request to expose an event when IME composition starts in the terminal, so CLIs (like Copilot CLI) can hide ghost-text during composition. No implementation exists on main yet; keeping open per team-member authorship.

## Change summary

Expose an `onDidStartTerminalComposition` event (triggered by `compositionstart`) so terminal-hosted CLIs can suppress ghost-text during IME input.

## Best solution

Add a composition-state notification channel from xterm.js/terminal to the CLI process — likely a new event in the terminal capability or extension API surface under `src/vs/workbench/contrib/terminal/`. This improves the IME experience for Copilot CLI and similar tools that render inline suggestions.
