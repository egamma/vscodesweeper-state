[https://github.com/microsoft/vscode/issues/309877](https://github.com/microsoft/vscode/issues/309877)

## Summary

Real bug: the timeline correctly shows commit history across renames (via `git log --follow`), but the diff command always uses the current file URI, so opening a commit from before the rename fails because the old filename isn't resolved.

## Change summary

Clicking a pre-rename commit in the Timeline view fails with 'file not found' because the diff uses the current filename instead of the file's path at that commit.

## Best solution

Extend `logFile` in `extensions/git/src/git.ts` to also return the file path at each commit (e.g. via `--name-status` or `--follow --name-only`), then use that path in `resolveTimelineOpenDiffCommand` instead of the current URI when constructing git URIs for older commits.
