# Issue #231029 — Can no longer install VS Code Server on machines that have no internet access

[https://github.com/microsoft/vscode/issues/231029](https://github.com/microsoft/vscode/issues/231029)

## Summary

Valid request for offline/air-gapped VS Code Server deployment support after the CLI-based server architecture change. The core tracking for this belongs in vscode-remote-release (multiple open issues there), but this issue captures the pain point from the main repo's perspective and has strong community interest (19 upvotes).

## Change summary

Users can no longer manually deploy VS Code Server on machines without internet access after the server deployment architecture changed to CLI-based.

## Best solution

This belongs in the remote development extension ecosystem — the canonical tracking issues are https://github.com/microsoft/vscode-remote-release/issues/9454 (SSH preinstall/offline) and https://github.com/microsoft/vscode-remote-release/issues/1802 (WSL offline package). The CLI's server download/install flow should support a local cache or pre-staged server directory that skips the network fetch. Consider closing as duplicate of the remote-release tracking issues, or routing to the remote team for a documentation/feature response.
