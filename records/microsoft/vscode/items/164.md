# Issue #164 — Better drag and drop / clipboard integration of files across applications

[https://github.com/microsoft/vscode/issues/164](https://github.com/microsoft/vscode/issues/164)

## Summary

Hard keep-open: the issue carries the 'On Deck' milestone, which is a scheduling decision that requires maintainer judgment. The issue also has 166 upvotes and tracks remaining unchecked items (copy files from VS Code explorer to OS clipboard, native file data transfer for DnD, 3rd party tool support) that are still not implemented on main.

## Change summary

Enable full bidirectional clipboard and drag-and-drop file integration between VS Code and the native OS / other applications.

## Best solution

The two checked items (paste from native OS into explorer, drop to desktop) shipped via commits 96b17815bbe and related work. The remaining three unchecked items — copying files from VS Code explorer to the native clipboard, native file data transfer during drag-and-drop (blocked on Electron, see electron/electron#2853), and 3rd party archive tool support — still need implementation. The Electron upstream dependency is the main blocker for the DnD piece; the clipboard copy-out likely also needs Electron API support. Continue tracking here with the upstream label.
