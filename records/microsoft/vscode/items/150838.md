# Issue #150838 — Explorer collapses folder when renamed

[https://github.com/microsoft/vscode/issues/150838](https://github.com/microsoft/vscode/issues/150838)

## Summary

Still reproduces on main: renaming a folder in the Explorer collapses it because the tree identity (based on the resource URI) changes, making the tree treat the renamed folder as a new node rendered collapsed by default.

## Change summary

Renaming an expanded folder in the Explorer causes it to collapse, losing the user's expanded-state context.

## Best solution

After calling `modelElement.rename()` in `explorerService.ts`, preserve the collapsed/expanded state of the renamed node across the parent refresh. One approach: before the refresh, capture whether the node was expanded, and after `updateChildren`, re-expand the node by its new identity. The fix is localized to the rename handling in `explorerService.ts` (lines ~394–400) and possibly `explorerView.ts`'s `refresh` method.
