# Issue #237840 — [Firefox] File tree context menu shows just "Paste" and opens unreliably

[https://github.com/microsoft/vscode/issues/237840](https://github.com/microsoft/vscode/issues/237840)

## Summary

Still reproduces on main. This is a Firefox-specific behavior where the browser shows a native 'Paste' popup when VS Code's web clipboard service attempts to access the Async Clipboard API in a secure (HTTPS) context. The workaround (dom.events.testing.asyncClipboard) confirms this is rooted in Firefox's clipboard permission model. VS Code may need to detect Firefox and avoid eagerly triggering clipboard reads during context menu display, but the root cause is upstream Firefox behavior.

## Change summary

In Firefox on github.dev/vscode.dev, right-clicking the file tree shows a native 'Paste' popup instead of (or alongside) the VS Code context menu.

## Best solution

Investigate whether VS Code's context menu path can avoid triggering `navigator.clipboard.readText()` (or similar) on Firefox during right-click, which is what causes Firefox to inject its native 'Paste' prompt. The clipboard read should be deferred until the user actually invokes a paste action. Likely areas: `src/vs/platform/clipboard/browser/clipboardService.ts` and the context menu trigger path in the explorer tree. Alternatively, track the upstream Firefox bug for Async Clipboard API behavior and mark this as upstream-blocked.

## Evidence

- **upstream root cause:** Firefox shows a native 'Paste' popup whenever a page accesses the Async Clipboard API on HTTPS. Comment 5 confirms that setting dom.events.testing.asyncClipboard=true in Firefox's about:config resolves it (but disables clipboard protection).
- **HTTPS-only:** Related issue https://github.com/microsoft/vscode/issues/239399 confirmed this only occurs over HTTPS, not HTTP — consistent with Firefox's Secure Context clipboard behavior.
- **duplicate consolidated:** https://github.com/microsoft/vscode/issues/239399 was closed as *duplicate of this issue, confirming this is the canonical tracker.
- **no fix on main:** BrowserClipboardService in src/vs/platform/clipboard/browser/clipboardService.ts has no Firefox-specific handling for readText(); the clipboard access path is browser-agnostic and will trigger Firefox's native popup.
- **external report:** Also reported at https://github.com/coder/code-server/issues/7597, confirming the issue affects any VS Code web deployment on Firefox.
