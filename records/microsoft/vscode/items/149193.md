# Issue #149193 — [less] The each function has incorrect linting inside a keyframes at-rule

[https://github.com/microsoft/vscode/issues/149193](https://github.com/microsoft/vscode/issues/149193)

## Summary

Still reproduces on current main. The LESS parser in vscode-css-languageservice (v6.3.4) does not recognize `each()` function calls as valid children inside `@keyframes` bodies — it only expects keyframe selectors. The fix belongs upstream in microsoft/vscode-css-languageservice.

## Change summary

The LESS `each()` function with a detached ruleset argument produces false-positive parse errors when used inside an `@keyframes` at-rule.

## Best solution

The LESS parser's keyframe body parsing in `vscode-css-languageservice` needs to accept function calls like `each(...)` that produce keyframe selectors at compile time. The fix would be in `src/parser/lessParser.ts` in `_parseKeyframeSelector()` to also try `_parseFunction()` or handle `each()` specifically. File an issue or PR against microsoft/vscode-css-languageservice.
