# Issue #149193 — [less] The each function has incorrect linting inside a keyframes at-rule

[https://github.com/microsoft/vscode/issues/149193](https://github.com/microsoft/vscode/issues/149193)

## Summary

Still reproduces on main. The LESS parser in vscode-css-languageservice does not call `_parseFunction()` inside `@keyframes` body parsing, so `each()` triggers false-positive lint errors. The fix belongs in the upstream vscode-css-languageservice repo.

## Change summary

LESS `each()` function used inside `@keyframes` produces false-positive parse errors ('{ expected', '} expected', 'at-rule or selector expected').

## Best solution

In vscode-css-languageservice's `lessParser.ts`, the `_parseKeyframeSelector()` override should also try `_parseFunction()` (which handles `each()`), similar to how `_parseRuleSetDeclaration()` does. The fix is small and localized in the upstream repo.
