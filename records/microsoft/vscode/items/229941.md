# Issue #229941 — Editor `z-index` challenges

[https://github.com/microsoft/vscode/issues/229941](https://github.com/microsoft/vscode/issues/229941)

## Summary

This should stay open as editor debt: the issue is asking for an explicit z-index model across multiple editor features, and the current code still has scattered hard-coded z-index values rather than a single documented ordering contract.

## Change summary

Document and rationalize the editor z-index ordering for minimap, scrollbars, sticky scroll, zone widgets, and related overlays.

## Best solution

Keep this as editor-core debt until the editor owns a documented z-index scale, ideally with named constants or a single reference point in the editor view layer that features such as minimap, sticky scroll, scrollbars, overlays, and zone widgets use consistently.
