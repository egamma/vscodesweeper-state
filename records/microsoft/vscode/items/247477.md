# Issue #247477 — incorrect append double slash text while onEnter called

[https://github.com/microsoft/vscode/issues/247477](https://github.com/microsoft/vscode/issues/247477)

## Summary

Fixed on main by commit 9b43a71f039 which restricts the comment continuation onEnter regex to only fire when `//` is at line start or whitespace-separated — the reported input `[/</g, ` no longer triggers false comment continuation.

## Change summary

Pressing Enter after `[/</g,` in TypeScript incorrectly prepends `//` to the next line due to the onEnter comment continuation rule mismatching regex literal slashes as a line comment.

## Best solution

Keep the shipped fix in commit 9b43a71f039 (PR #321230) which tightens the beforeText regex to `^\s*//|\s//\s`, preventing false matches on non-comment slashes. No further change needed.
