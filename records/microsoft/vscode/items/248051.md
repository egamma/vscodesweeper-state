# Issue #248051 — Explorder: Exclude Git Ignore didn't hornor .gitignore file in ancestor folder

[https://github.com/microsoft/vscode/issues/248051](https://github.com/microsoft/vscode/issues/248051)

## Summary

Still reproduces on main. The author is a team member, so this stays open for maintainer judgment. The explorer's gitignore filter only discovers .gitignore files encountered during tree traversal — ancestor .gitignore files outside the workspace folder root are never visited and thus never loaded into the ignore tree.

## Change summary

When a workspace is opened from a subfolder of a Git repository, the explorer's 'Exclude Git Ignore' setting does not honor .gitignore files in ancestor folders above the workspace root.

## Best solution

The `FilesFilter` in `explorerViewer.ts` should proactively walk up from each workspace folder root to the Git repository root, loading any .gitignore files found along the way (not just those discovered lazily during tree traversal). The ignore tree initialization in `updateConfiguration()` (around line 1316) or a startup hook should seed ancestor .gitignore files by querying the git service for the repo root and reading .gitignore files from ancestor directories.
