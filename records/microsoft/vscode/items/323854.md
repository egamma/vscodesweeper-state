# Issue #323854 — Chat image attachments are silently dropped when not signed in to GitHub, even with a configured multimodal provider

[https://github.com/microsoft/vscode/issues/323854](https://github.com/microsoft/vscode/issues/323854)

## Summary

Already fixed on main: commit f170fd1f23d removed the `isEditorPreviewFeaturesEnabled()` gate from `image.tsx`, which was the exact root cause — `copilotToken?.isEditorPreviewFeaturesEnabled()` evaluated to `undefined` when signed out, causing images to be silently dropped. The fix shipped as part of the "GA vision in vs code" change.

## Change summary

Chat image attachments are silently dropped when not signed in to GitHub, even when a multimodal BYOK/local model is configured, because the image prompt element gates on a Copilot token that is undefined when signed out.

## Best solution

Keep the shipped fix in commit f170fd1f23d; no further change needed. The open PR https://github.com/microsoft/vscode/pull/323856 is now redundant — it proposed a `?? true` fallback for the same guard that this commit removed entirely. The fix will ship in the next Insiders/stable release.
