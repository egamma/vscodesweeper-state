[https://github.com/microsoft/vscode/issues/214925](https://github.com/microsoft/vscode/issues/214925)

## Summary

Valid feature request for a sidebar/panel container-level visibility API. Existing TreeView.onDidChangeVisibility covers individual views but not the overall sidebar open/closed state. Has 22+ upvotes, is in Backlog, and issue #251274 was closed as a duplicate of this — confirming it's the canonical tracker.

## Change summary

Add an extension API to track sidebar/panel visibility state and query whether a specific sidebar container is open or closed.

## Best solution

Implement an extension API (likely on the `window` namespace or as a new event/property) that exposes whether the primary sidebar, secondary sidebar, and panel are visible, plus change events. This fills a real gap between the existing per-view `onDidChangeVisibility` and the container-level state that extensions cannot currently observe programmatically.
