# Issue #157839 — Git - "Add to gitignore" should add to closest gitignore file, not root one.

[https://github.com/microsoft/vscode/issues/157839](https://github.com/microsoft/vscode/issues/157839)

## Summary

Still unimplemented on main — the `ignore()` method hardcodes the root `.gitignore`. An open community PR (#307639) is attempting the fix, so this should stay open pending review of that PR.

## Change summary

The 'Add to .gitignore' command always writes to the root `.gitignore` instead of the closest ancestor `.gitignore` file.

## Best solution

Review and land (or close) the open community PR https://github.com/microsoft/vscode/pull/307639, which implements the closest-gitignore logic in `extensions/git/src/repository.ts`. The approach suggested in the comments — show a quickpick when multiple candidate `.gitignore` files exist — is sound.
