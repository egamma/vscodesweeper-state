# Issue #224813 — Unable to install extension 'vscode.git' because it is not found.

[https://github.com/microsoft/vscode/issues/224813](https://github.com/microsoft/vscode/issues/224813)

## Summary

Team-authored telemetry issue; sandy081 acknowledged the root cause in Sept 2024 but no fix has landed on main. The URL handler tries to install a built-in extension (vscode.git) when handling clone URIs in restricted mode, which fails because built-in extensions aren't in the gallery. Keeping open per hard keep-open rule (MEMBER author) and because the fix is still pending.

## Change summary

Clone-in-VS-Code URL handler errors when it attempts to 'install' the built-in vscode.git extension in a restricted-mode window.

## Best solution

The URL handler in `extensionUrlHandler.ts` should recognize built-in extensions (like `vscode.git`) and skip the gallery install step — instead just activating the already-present extension (or prompting to trust the workspace). sandy081 identified the cause; the fix likely belongs in `src/vs/workbench/services/extensions/browser/extensionUrlHandler.ts` or `src/vs/workbench/contrib/extensions/browser/extensions.contribution.ts`.
