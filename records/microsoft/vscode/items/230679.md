# Issue #230679 — "Save File" dialog is not visible in KDE after upgrade to 1.94.0

[https://github.com/microsoft/vscode/issues/230679](https://github.com/microsoft/vscode/issues/230679)

## Summary

Correctly triaged upstream issue — the root cause is in Electron/Chromium's runtime detection for KDE dialogs (linked by deepak1556 to electron/electron#42045). The bug persists on Kubuntu 24.04 even with current VS Code/Electron versions, as confirmed by the still-open follow-up issue #243575. Already labeled `upstream` and `upstream-issue-linked`; keeping open for upstream resolution.

## Change summary

Native file dialogs (Save, Open) freeze or fail to appear on KDE Plasma (Kubuntu 24.04) due to a MIME info infinite loop triggered by Electron's KDE dialog detection interacting with shared-mime-info shipped by Snap and the system.

## Best solution

The fix must land in Electron's Linux shell-dialog integration (the KDE dialog runtime detection path linked in electron/electron#42045). VS Code should pick it up via an Electron upgrade once resolved upstream. The 'Simple Dialog' setting is a viable workaround for affected users in the meantime. Issue #243575 tracks the same problem and could be consolidated here.
