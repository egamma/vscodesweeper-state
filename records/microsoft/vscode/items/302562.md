# Issue #302562 — Stale output from MCP tool / App in chat window

[https://github.com/microsoft/vscode/issues/302562](https://github.com/microsoft/vscode/issues/302562)

## Summary

The maintainer (connor4312) has explicitly explained this is intentional behavior for transparency and prompt-injection prevention, declined both requested changes, and pointed to an alternative pattern. This can be closed as as-designed.

## Change summary

Request to suppress or update the text display of MCP tool results in the chat window when an MCP App reinvokes a tool with new inputs.

## Best solution

Keep the current behavior where tool output shown to the model is always visible to the user for transparency/security. The reporter should use the alternative pattern linked by the maintainer (https://github.com/modelcontextprotocol/ext-apps/pull/390) to achieve their desired UX.

## Vision fit

- **Fit:** rejected
- **Reason:** Suppressing tool output would conflict with agent security principles — users need transparency into what the model sees to detect prompt injection.
