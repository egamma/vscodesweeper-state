# Issue #176177 — vscode.dev PWA Icon size on Windows 11 is too small

[https://github.com/microsoft/vscode/issues/176177](https://github.com/microsoft/vscode/issues/176177)

## Summary

Still present on main: the PWA icons in resources/server/ have ~7% vertical transparent padding and no maskable icon variant, causing them to appear smaller than the native app icon on Windows 11 taskbar/start menu. The icons and manifest haven't been updated since 2021.

## Change summary

The vscode.dev PWA icon appears smaller than the native VS Code icon on Windows 11 because the PNG has transparent padding and the manifest lacks a maskable icon entry.

## Best solution

Update the PWA icon PNGs in `resources/server/` to remove the transparent padding (or add a separate maskable-purpose icon that fills the safe zone), fix the manifest.json size declarations (currently says 512x512 but the PNGs are 1024x1024), and add a `"purpose": "any maskable"` entry so Windows and Android can crop the icon appropriately.

## Evidence

- **no maskable icon:** The manifest has no `purpose` field on any icon entry — no maskable icon variant exists, which causes platforms like Windows 11 to shrink the icon to fit inside the safe zone.
