# Issue #230518 — Allow grouping of `package.nls*.json` files in a custom folder

[https://github.com/microsoft/vscode/issues/230518](https://github.com/microsoft/vscode/issues/230518)

## Summary

Valid feature request and the canonical issue — #169150 was closed as a duplicate of this one by a team member. Tyler expressed intent to implement but noted marketplace coordination challenges. Low upvotes but has team buy-in; keep open.

## Change summary

Allow extension authors to place package.nls*.json files in a custom subfolder rather than requiring them in the project root.

## Best solution

Implement the approach Tyler outlined: extend the `l10n` property in package.json to support `{ "package": "./path", "bundle": "./l10n" }`, coordinating with the Marketplace team to also support the non-root location. This lives in the extension scanning/resolution layer (webExtensionsScannerService, builtinExtensionsScannerService) and the Marketplace's VSIX packaging.
