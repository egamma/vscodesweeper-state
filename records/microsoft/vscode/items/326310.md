# Issue #326310 — Make command risk assessment explanations respect `github.copilot.chat.localeOverride`

[https://github.com/microsoft/vscode/issues/326310](https://github.com/microsoft/vscode/issues/326310)

## Summary

Valid feature request: the risk assessment prompts in chatToolRiskAssessmentService.ts are hardcoded in English with no locale awareness. The `localeOverride` setting lives in the Copilot extension and is not consulted when building risk assessment prompts. This is a real gap for non-English users relying on safety-critical explanations.

## Change summary

Command risk assessment explanations are always generated in English, ignoring the `github.copilot.chat.localeOverride` setting that controls the language of other Copilot Chat responses.

## Best solution

Modify `buildTerminalPrompt` and `buildGenericToolPrompt` in `src/vs/workbench/contrib/chat/browser/tools/chatToolRiskAssessmentService.ts` to read the locale override setting and include a language instruction in the prompt (e.g. 'Write the explanation in {locale}'). This requires bridging the Copilot extension's `chat.localeOverride` setting into the risk assessment service, likely via `IConfigurationService` or an existing locale-resolution helper. Related to https://github.com/microsoft/vscode/issues/280512 (slash commands not respecting localeOverride) but distinct enough to track separately.
