# Issue #306244 — Rate limit `try again` button doesn't show if you have queued messages

[https://github.com/microsoft/vscode/issues/306244](https://github.com/microsoft/vscode/issues/306244)

## Summary

Still reproduces on main: the `isLast` guard on line 2577 of chatListRenderer.ts prevents confirmation buttons (including 'try again') from rendering when queued messages follow the errored response. The bug is clearly traceable in the source and no fix has landed.

## Change summary

Rate-limit and error 'try again' confirmation buttons are hidden when queued messages are present in the chat widget.

## Best solution

Remove or relax the `isLast` guard at chatListRenderer.ts:2577 so that `errorDetails.confirmationButtons` are rendered regardless of whether the errored response is the last non-pending item. The `isLast` check was likely intended to avoid duplicate button rendering during streaming, but it incorrectly suppresses buttons when queued messages follow the response.
