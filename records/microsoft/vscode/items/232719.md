# Issue #232719 — No close icon when Activity Bar Top

[https://github.com/microsoft/vscode/issues/232719](https://github.com/microsoft/vscode/issues/232719)

## Summary

Current behavior is intentional (close button hidden when activity bar is not 'Default'), but the issue is already labeled `under-discussion` and serves as the canonical tracker for this UX gap. Other issues (#238100, #274349) were closed as duplicates of this one. Keeping open as the active discussion point.

## Change summary

Add a close/hide button to the secondary sidebar when the Activity Bar position is set to 'Top' (or non-default).

## Best solution

The team should decide whether to always show the close button in the auxiliary bar title regardless of activity bar position (removing the `when` clause at line 94 of `auxiliaryBarActions.ts`), or confirm `*as-designed` and document the titlebar toggle as the intended alternative. This is the canonical issue for that decision.
