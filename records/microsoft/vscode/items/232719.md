# Issue #232719 — No close icon when Activity Bar Top

[https://github.com/microsoft/vscode/issues/232719](https://github.com/microsoft/vscode/issues/232719)

## Summary

This is a deliberate design decision: the close button is hidden when Activity Bar is not in its default position (line 94 of auxiliaryBarActions.ts gates it on ActivityBarPosition.DEFAULT). The maintainer (@benibenj) acknowledged the UX concern and multiple duplicates point here as the canonical tracking issue. Keeping open as under-discussion.

## Change summary

Show a close/hide button in the secondary sidebar title bar when Activity Bar position is set to 'top' (or other non-default positions).

## Best solution

Add the close icon to AuxiliaryBarTitle unconditionally (remove the `ActivityBarPosition.DEFAULT` gate at line 94 of `auxiliaryBarActions.ts`), or provide an alternative visible affordance to dismiss the secondary sidebar regardless of activity bar position. The maintainer needs to decide whether the symmetry argument (primary and secondary sidebars looking alike) still outweighs discoverability now that Copilot Chat is the default secondary sidebar occupant.

- **Reason:** Improving discoverability of the Copilot Chat sidebar dismiss action supports the current AI-native editor direction where more users encounter the secondary sidebar via Copilot.
- **Evidence:** Aligns with current agent-experience priorities — making the agentic chat surface intuitive to manage.
