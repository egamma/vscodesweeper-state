# Issue #326302 — [accessibility] [regression]: screen reader cannot announce results while navigating between results in search box

[https://github.com/microsoft/vscode/issues/326302](https://github.com/microsoft/vscode/issues/326302)

## Summary

Credible regression report from the same reporter as the previously verified-fixed #320098. Screen reader (NVDA) cannot announce search results when navigating with arrow keys in the search treeview. The prior fix (commit bdba07cf97f) addressed hover tooltips but the ARIA announcement issue may have a different or recurring root cause. No obvious new regression commit found on main, so this needs hands-on investigation with NVDA.

## Change summary

NVDA cannot announce individual search results when navigating the search results treeview with arrow keys, despite the treeview having a proper accessibility provider with getAriaLabel.

## Best solution

Investigate with NVDA on current main/Insiders to confirm the regression, then bisect to the causing commit. The SearchAccessibilityProvider in searchResultsView.ts provides aria labels correctly in code, so the issue may be in how the tree widget applies them during navigation (aria-activedescendant flow in listWidget.ts) or a rendering/virtualization timing issue where labels aren't applied before focus moves.
