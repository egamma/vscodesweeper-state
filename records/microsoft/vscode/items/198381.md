# Issue #198381 — Provide ability to ignore all line breaks in diff editor (feature request) 

[https://github.com/microsoft/vscode/issues/198381](https://github.com/microsoft/vscode/issues/198381)

## Summary

Valid community-promoted feature request (47 upvotes, in Backlog). No implementation exists on current main — the diff engine only supports `ignoreTrimWhitespace` (leading/trailing whitespace), not ignoring line breaks entirely. Related to but distinct from #43026 (ignore all whitespace).

## Change summary

Add a diff editor option to ignore line break differences so that reformatted code with added/removed newlines does not highlight as changed.

## Best solution

Implement an `ignoreLineBreaks` option in `IDocumentDiffProviderOptions` (src/vs/editor/common/diff/documentDiffProvider.ts) and wire it through the diff editor options/configuration schema, similar to how `ignoreTrimWhitespace` is handled. Could potentially be combined with the broader ignore-all-whitespace work tracked in https://github.com/microsoft/vscode/issues/43026.
