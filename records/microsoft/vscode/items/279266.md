[https://github.com/microsoft/vscode/issues/279266](https://github.com/microsoft/vscode/issues/279266)

## Summary

Confirmed regression on current main: the showRange field is accepted by the API layer but never checked against cursor position in the inline completions model. Milestone 'On Deck' triggers the hard keep-open rule; the community fix PR #279269 remains open.

## Change summary

The showRange field on InlineCompletionItem (proposed API) is ignored — inline completions always display regardless of cursor position relative to the specified range.

## Best solution

Merge or iterate on the community PR https://github.com/microsoft/vscode/pull/279269, restoring cursor-position checks against showRange in the inline completions model (likely in inlineCompletionsSource.ts or inlineCompletionsModel.ts). The InlineSuggestData / InlineCompletionItem types need to carry showRange and the visibility logic must gate on it.
