# Issue #169322 — Search editor context lines is ambiguous and doesn't appear to be accessible to screen readers

[https://github.com/microsoft/vscode/issues/169322](https://github.com/microsoft/vscode/issues/169322)

## Summary

Still reproduces on main: the search editor's context lines number input (InputBox) is created without an `ariaLabel` or hover title, so it's unlabeled for screen readers and has no explanatory tooltip.

## Change summary

The context lines number input in the search editor header has no aria-label or hover title, making it inaccessible to screen readers and ambiguous for sighted users.

## Best solution

Add an `ariaLabel` option to the `contextLinesInput` InputBox constructor in `searchWidget.ts` (e.g. `ariaLabel: localize('contextLines', 'Context Lines')`), and set up a managed hover on the input element with the same label text. The toggle checkbox already has a proper title — the number input just needs parity.
