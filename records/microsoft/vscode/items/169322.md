## Summary

Still reproduces on main: the context lines InputBox in the search editor widget has no `ariaLabel`, `title`, or `placeholder`, making it inaccessible to screen readers and ambiguous visually. Issue is on the 'On Deck' milestone, so it stays open for maintainer scheduling.

## Change summary

The search editor's context lines number input lacks a hover tooltip and ARIA label, making it inaccessible and ambiguous.

## Best solution

Add an `ariaLabel` (and optionally a `title`/`placeholder`) to the `contextLinesInput` InputBox constructor options in `src/vs/workbench/contrib/search/browser/searchWidget.ts` (~line 520) so screen readers announce it as 'Context Lines' and sighted users get a tooltip on hover.
