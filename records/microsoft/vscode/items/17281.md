# Issue #17281 — Add support for capture groups in onEnterRules

[https://github.com/microsoft/vscode/issues/17281](https://github.com/microsoft/vscode/issues/17281)

## Summary

Team-authored feature request (alexdima, MEMBER) for capture group support in onEnterRules — not yet implemented on main, still a real gap. Keeping open per policy.

## Change summary

Allow onEnterRules to reference regex capture groups (e.g. `$1`) in `appendText`, enabling patterns like preserving comment indentation on Enter.

## Best solution

Implement capture group substitution in the onEnter action pipeline — specifically in `src/vs/editor/common/languages/enterAction.ts` and the rule-matching logic in `src/vs/editor/common/languages/autoIndent.ts`. The `appendText` field would resolve `$1`, `$2`, etc. against the matched `beforeText` regex groups. The team discussion (comments 1–4) already converged on `$1`/`$2` syntax (no curly braces). Comment 5 raises broader questions about embedded languages that may warrant scoping down to the simple substitution first.
