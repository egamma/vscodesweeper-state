# Issue #202482 — Code actions menu changes scroll position

[https://github.com/microsoft/vscode/issues/202482](https://github.com/microsoft/vscode/issues/202482)

## Summary

Still present on main: `toCoords()` in `codeActionController.ts` unconditionally calls `revealPosition()` before computing the menu anchor, causing unwanted scroll when the cursor is near the top of the viewport. The author (jrieken) is a team contributor, so this is kept open for maintainer judgment.

## Change summary

Opening the code actions menu via Cmd+. scrolls the editor even when the cursor position is already visible.

## Best solution

Guard the `revealPosition` call in `toCoords()` so it only scrolls when the position is outside the visible viewport (e.g. use `revealPositionInCenterIfOutsideViewport` or check visibility first). The fix is localized to `src/vs/editor/contrib/codeAction/browser/codeActionController.ts` line ~406.
