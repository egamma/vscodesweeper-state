# Issue #193362 — Improve Emitter.toString

[https://github.com/microsoft/vscode/issues/193362](https://github.com/microsoft/vscode/issues/193362)

## Summary

Team-authored engineering improvement request to enhance Emitter debugging by showing the owning class and field name in stack frames. Hard keep-open: author is MEMBER.

## Change summary

Improve Emitter.fire stack frame display to show the owning class and field name (e.g. ElementSizeObserver.onChange.fire) instead of generic Emitter.fire.

## Best solution

Implement an optional `owner` parameter in Emitter options that stores a reference to the containing instance, then use vscode-js-debug's custom `toString` rendering (PR #1802) to resolve the field name at debug time. This is a developer-experience improvement for contributors debugging event flows in the VS Code codebase.
