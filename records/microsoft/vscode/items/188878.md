# Issue #188878 — Can't hover on top few pixels of editor

[https://github.com/microsoft/vscode/issues/188878](https://github.com/microsoft/vscode/issues/188878)

## Summary

The bug is still present on main: the `.scroll-decoration` element (6px tall, position absolute, top 0) lacks `pointer-events: none`, so it intercepts mouse events on the top few pixels of the editor. The maintainer recently asked for a screen recording, so the issue is in an info-needed cycle.

## Change summary

Hovering over text in the top ~6 pixels of the editor viewport is blocked by the scroll-decoration overlay element.

## Best solution

Add `pointer-events: none` to `.monaco-editor .scroll-decoration` in `src/vs/editor/browser/viewParts/scrollDecoration/scrollDecoration.css`. The element is purely decorative (a shadow effect), so it should never consume pointer events.
