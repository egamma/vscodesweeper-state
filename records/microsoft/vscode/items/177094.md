# Issue #177094 — LSP Code Action Requests with wrong types are sent

[https://github.com/microsoft/vscode/issues/177094](https://github.com/microsoft/vscode/issues/177094)

## Summary

Still reproduces on main: `Diagnostic.from` in `extHostTypeConverters.ts` spreads `Range.from(value.range)` without checking for non-finite coordinates, so NaN values serialize as JSON `null` and violate the LSP spec. An open PR (#318520) already addresses this — keeping open pending that PR's review.

## Change summary

When an extension provides a diagnostic with NaN line numbers, VS Code serializes them as `null` in LSP code-action requests, violating the LSP spec and crashing downstream language servers.

## Best solution

Land the sanitization guard proposed in https://github.com/microsoft/vscode/pull/318520, which checks for non-finite coordinates after `Range.from()` in `Diagnostic.from` and replaces them with a safe degenerate range. Review and merge that PR.
