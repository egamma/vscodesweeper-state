# Issue #177094 — LSP Code Action Requests with wrong types are sent

[https://github.com/microsoft/vscode/issues/177094](https://github.com/microsoft/vscode/issues/177094)

## Summary

Still valid on main: NaN line numbers bypass Position validation (`NaN < 0` is false) and serialize as `null` in LSP messages. An open PR (#318520) proposes a fix but hasn't merged yet.

## Change summary

VS Code sends invalid LSP messages with `null` line numbers when an extension creates diagnostics with NaN positions.

## Best solution

Merge or iterate on PR https://github.com/microsoft/vscode/pull/318520 which sanitizes non-finite range coordinates in `Diagnostic.from`. The fix belongs in `src/vs/workbench/api/common/extHostTypeConverters.ts` (Range/Diagnostic conversion) or `src/vs/workbench/api/common/extHostTypes/position.ts` (Position constructor validation).
