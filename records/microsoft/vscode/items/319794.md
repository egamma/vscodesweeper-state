# Issue #319794 — Reasoning/thinking parameters not forwarded to extension-contributed model providers

[https://github.com/microsoft/vscode/issues/319794](https://github.com/microsoft/vscode/issues/319794)

## Summary

Confirmed on current main: `extChatEndpoint.makeChatRequest2` destructures `IMakeChatRequestOptions` but never reads `modelCapabilities`, so `enableThinking` and `reasoningEffort` are silently dropped for extension-contributed model providers. An open PR from the reporter addresses it.

## Change summary

Reasoning/thinking parameters (`enableThinking`, `reasoningEffort`) are silently dropped when requests go through `extChatEndpoint` to extension-contributed model providers, while BYOK and cloud models receive them correctly.

## Best solution

Review and land PR https://github.com/microsoft/vscode/pull/319798, which forwards `enableThinking` and `reasoningEffort` from `modelCapabilities` into the `modelOptions` object in `extChatEndpoint.makeChatRequest2`. The fix is small and localized to `extensions/copilot/src/platform/endpoint/vscode-node/extChatEndpoint.ts`.
