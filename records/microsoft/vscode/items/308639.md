# Issue #308639 — Copilot Chat fails on macOS arm64 with Electron fetcher: “Cannot convert argument to a ByteString … char 65279 (U+FEFF)” (regression between 1.110.1 and 1.115.0)

[https://github.com/microsoft/vscode/issues/308639](https://github.com/microsoft/vscode/issues/308639)

## Summary

Fixed on main: the root cause was a UTF-8 BOM (U+FEFF) returned by @vscode/deviceid that leaked into HTTP request headers via the Copilot extension, causing undici's ByteString validator to reject them. The fix strips the BOM and shipped in 1.121.0.

## Change summary

Copilot Chat fails with a ByteString error because the device ID returned by @vscode/deviceid contains a leading UTF-8 BOM (U+FEFF) that is invalid in HTTP headers.

## Best solution

Keep the shipped fix from commit 15c518a9b04 (strips BOM from device ID in `src/vs/base/node/id.ts` and `src/vs/platform/telemetry/node/telemetryUtils.ts`). Users on VS Code 1.121.0+ should no longer see this error.
