# Issue #153213 — Have `MergeEditorTabInput`-API and `openTab`-API

[https://github.com/microsoft/vscode/issues/153213](https://github.com/microsoft/vscode/issues/153213)

## Summary

Team-authored API proposal issue (jrieken is a core VS Code team member). The `TabInputTextMerge` class was added as a proposed API in commit 785d85a0bd6 but has never been finalized into the stable API (`vscode.d.ts`). The second part — allowing extensions to programmatically open merge editor tabs — also appears unimplemented. Keeping open per the hard keep-open rule for team-authored issues, and because the requested work is genuinely incomplete.

## Change summary

Expose `TabInputTextMerge` as stable API and provide an API for extensions to programmatically open merge editor tabs.

## Best solution

Finalize the `tabInputTextMerge` proposed API into `vscode.d.ts` and implement an extension-facing mechanism (via `vscode.open` command or a dedicated API) to create merge editor tabs. The proposal file at `src/vscode-dts/vscode.proposed.tabInputTextMerge.d.ts` and the implementation in `src/vs/workbench/api/browser/mainThreadEditorTabs.ts` are the starting points.
