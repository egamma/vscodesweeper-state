# Issue #162428 — Make tree view more stable

[https://github.com/microsoft/vscode/issues/162428](https://github.com/microsoft/vscode/issues/162428)

## Summary

Still present on main: the explorer's `setEditable` unconditionally calls `tree.reveal(stat)` when entering edit mode for a new file/folder, which can scroll the viewport even when the item is already visible. The issue is correctly labeled as polish and remains valid.

## Change summary

Creating a new folder in the explorer causes an unnecessary scroll jump when the new item is already visible in the viewport.

## Best solution

Guard the `tree.reveal(stat)` call in `ExplorerView.setEditable()` (line 416 of `src/vs/workbench/contrib/files/browser/views/explorerView.ts`) with a visibility check similar to `selectResource()` — only reveal when `getRelativeTop(stat)` is `null`. This would prevent the scroll jump when the newly created item is already in view.
