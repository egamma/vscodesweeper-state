# Issue #323391 — Search "Files to Include" behaves incorrectly when workspace root and nested file patterns are specified together

[https://github.com/microsoft/vscode/issues/323391](https://github.com/microsoft/vscode/issues/323391)

## Summary

Confirmed bug in `expandSearchPathPatterns` merge logic: when `./` (meaning 'all files') is combined with a more specific pattern targeting the same workspace folder, the merge replaces `undefined` (match-all) with a restrictive pattern set, incorrectly narrowing search scope.

## Change summary

Search 'Files to Include' incorrectly narrows scope when workspace root pattern (`./`) is combined with more specific nested patterns.

## Best solution

Fix the merge logic in `expandSearchPathPatterns` (~line 462-466 of `src/vs/workbench/services/search/common/queryBuilder.ts`): when the existing entry has `pattern: undefined` (match-all), skip adding more specific patterns since they're already included. Also handle the multi-root case where `./` is dropped entirely at line 513-514.
